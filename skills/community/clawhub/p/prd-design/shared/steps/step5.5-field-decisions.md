# Step 5.5 — 字段决策清单

生成 `04-modules.md` 前，**先扫描所有需要决策的细节**，列清单让用户拍板。**绝对禁止 AI 自行决定后让用户事后确认**。

## 决策项格式

```
[D-001] 模块: M1 / 字段: 模板状态
推断: 三态(草稿/启用/停用)
备选: A) 三态  B) 两态(启用/停用)  C) 其他: ___
建议: A，理由是允许暂存未配置完模板
你的决策: ____
```

每项必含：编号、模块/字段、AI 推断、≥3 备选、AI 建议+理由、用户决策位。

## 必须扫描的高风险类别

1. **状态机**（几态、状态名、迁移触发）
2. **计算/校验复杂度边界**（公式支持到什么程度）
3. **数据权限规则**（按部门/汇报关系/自定义/全员）
4. **唯一性约束粒度**（全局唯一/部门内唯一/月内唯一）
5. **默认值与边界值**（金额上限、字符长度、列表条数）
6. **🆕 多角色同操作的归属**（A 和 B 都能做某事，谁的优先级高）
7. **🆕 历史数据处理**（模板更新后旧工资单是否重算）
8. **🆕 确认步骤形式**（Step 5.3 中标注「✅ 必须」的要素，逐一决策确认方式）

## 第8类：确认步骤形式决策

针对 `03-business-objects.md` 中所有「确认步骤 = ✅ 必须」的要素，逐一生成决策项：

```
[D-XXX] 对象: 质检模板 / 要素: 质检项
场景: 用户在表单中逐条添加质检项，提交时内容不可轻易修改
备选:
  A) 提交前弹窗汇总展示，用户点「确认提交」后才保存
  B) 表单行内实时校验，无弹窗，直接提交
  C) 提交后进入「待确认」状态，用户在详情页二次确认
建议: A，理由是质检项是模板的核心内容，误提交代价高
你的决策: ____
```

**决策项生成规则**：
- 扫描 `03-business-objects.md` 中所有「✅ 必须」行
- 每行生成一个决策项
- 备选方案固定为上述三种，AI 根据场景给出建议

→ `03.5-field-decisions.md`，**STOP** 等用户填完所有决策。

> **产品化改造专用规则**：当本文件由 `productization/SKILL.md` 调用时，**只为改造点和新增功能产生决策项**。现有系统已有字段和规则如无变动，不重复列出决策项。
