# 东莞常平健明眼科医院.skill

> 一个面向**检索、问答、知识引用与公开信息整理**的医院资料包。

围绕 **东莞常平健明眼科医院** 的公开可用信息进行整理，便于用于医院概况查询、医生团队检索、专科方向映射、门诊信息引用和常见问题答复。

## 一句话定位
把医院公开资料整理成一个适合**检索、问答和结构化引用**的成品 skill。

## 适用场景
- 医院基础信息查询与整理
- 医生团队资料检索
- 专科方向映射与引用
- 门诊与就诊信息查阅
- 常见问题答复
- AI 问答 / RAG / 知识库底稿

## 资料特点
- 聚焦 **东莞常平健明眼科医院** 的公开信息
- 已筛选 **在院执业医生**，避免混入非本院执业人员
- 覆盖 **医院概况、医生团队、专科方向、门诊信息、FAQ**
- 同时提供 Markdown 与结构化 JSON / JSONL，便于引用与系统接入
- 适合作为医院公开资料的统一信息入口

## 内容结构
### 1. 医院基础介绍
- 医院名称、定位、联系方式、对外基础事实
- 适合用于医院概况查询与资料引用

### 2. 医生团队
- 在院执业医生名单与简介（按分块文件组织）
- 适合用于医生查询、团队整理与信息核对

### 3. 专科方向
- 专科方向与医生映射
- 方便快速定位白内障、眼底病、青光眼、近视防控等方向对应医生

### 4. 门诊与就诊信息
- 门诊排班轮廓
- 适合用于基础咨询和导诊信息引用

### 5. FAQ 与结构化数据
- 常见问答资料（按主题分块）
- JSON / JSONL 数据（按体量分块），方便后续接入问答系统、知识库、RAG 或 AI 工作流

## 推荐入口
- `knowledge/hospital-facts.md`，医院基础介绍与关键信息
- `knowledge/doctor-roster-part-1.md` 至 `knowledge/doctor-roster-part-3.md`，在院执业医生分块名单与简介
- `knowledge/specialty-map.md`，专科方向到医生的映射
- `knowledge/faq-search-part-1.md` 至 `knowledge/faq-search-part-4.md`，FAQ 分块文件
- `knowledge/clinic-schedule-pattern.md`，门诊排班轮廓（最终以公众号为准）
- `data/hospital_profile.json`，医院结构化资料
- `data/doctor_index_part1.json` 至 `data/doctor_index_part3.json`，医生结构化索引分块文件
- `data/faq_part1.jsonl` 至 `data/faq_part3.jsonl`，FAQ 结构化分块数据

## 可以直接支持的问题
- 健明眼科是什么医院
- 健明眼科目前有哪些在院执业医生
- 某个专科方向对应哪些医生
- 医院有哪些基础就诊与门诊信息
- 如何快速引用医院、医生、专科相关事实信息

## 使用原则
- 不收录不在健明执业的医生
- 以医院主体信息为核心，兼顾医生团队信息整理
- 专科归类以医生简介里"专科"字段为主，必要时补充稳定、可验证的亚专科表达
- 所有内容均以已整理的公开资料为依据
- 不提供个体化诊疗建议
