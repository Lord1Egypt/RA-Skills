# Eva Learn 11-A：探索式学习

适用场景：用户没有资料，从模糊兴趣、明确问题或一个想法出发学习。

本文件是一条线性旅程：

```text
目标尺度确认 -> 澄清 -> 谱系 -> 路径确认 -> 学习目录 -> 逐讲推进 -> 掌握检查 -> 沉淀 -> 完成判断
```

进入本文件前必须已经建档。若未建档，返回 `11_eva-learn.md`，不得继续。

## 谱系输出字段

初学者建议谱系使用完整字段：

- 路径名称。
- 为什么先走这条。
- 会把用户带向哪种学习目标。
- 能解决什么。
- 解决不了什么。
- 第一阶段适合先读什么。
- 第一讲建议从哪里开始。
- 置信度：高 / 中 / 低。

复杂谱系只使用简版字段：

- 路径名称。
- 适合谁。
- 会走向什么学习目标。
- 为什么不是默认推荐。

## Phase 0：目标尺度确认

如果用户输入是“大主题 + 应用词 / 运作词”，先确认学习目标尺度，不得直接进入讲课。

运作词包括：

```text
是什么形式 / 怎么运作 / 现实中如何发生 / 底层机制 / 整个系统 / 市场怎么跑起来
```

应用词包括：

```text
融资 / 并购 / 商业化 / 投资 / 增长 / 传播 / 写作 / 运营 / 销售 / 管理 / 变现
```

判定规则：

- “怎么运作 / 是什么形式 / 现实中如何发生”优先判为制度型候选。
- 应用词只能说明用户的困惑入口，不自动等于学习目标。
- 如果学科型、应用型、制度型、内容素材型中有两个以上都可能成立，必须输出 1 条初学者建议谱系和 2-4 条复杂谱系简表，然后停下。

初学者建议谱系必须是 Eva 推荐的默认路径。复杂谱系只能作为改选项，不展开完整内容。

输出谱系分层时，更新 `00-学习进度.md`：

```yaml
当前 Phase: Phase 0
学习目标尺度: 待确认
下一步动作: 等待用户选择默认路径或复杂谱系
```

输出谱系分层后必须停下，不得在同一轮生成学习目录或开第一讲。

只给用户三个选择：

```text
1. 回复“按默认路径开始”
2. 回复某条复杂谱系编号
3. 回复“直接开讲”，授权 Eva 代选并开始
```

如果不是大主题输入，或只命中一个目标尺度，把尺度写入 `00-学习进度.md`，进入 Phase 1 或 Phase 2。目标尺度明确不等于可以跳过阅读谱系确认。

## Phase 1：澄清

如果用户的问题已经清楚，跳到 Phase 2。

如果需求模糊，最多三问，每轮只问一个：

1. 你想把这个东西用在哪？
2. 你现在对它的直觉判断是什么？
3. 学完之后你想拿到什么产出？

三问后必须停止追问，输出学习主题判断。不得继续问“还有吗”“再具体一点”。

## Phase 2：阅读谱系

默认输出谱系分层：

```text
A. 初学者建议谱系：只给 1 条，使用完整字段
B. 复杂谱系简表：给 2-4 条，只使用简版字段
```

初学者建议谱系是 Eva 推荐的默认路径，适用于：

- 用户没有资料。
- 用户没有明确学习路径。
- 用户说“我不太懂”“基础知识”“想了解一下”“到底是什么形式”“怎么运作”。
- 用户输入同时包含大主题和多个应用词，容易过早收窄。

复杂谱系只展示改选方向，不展开完整卡。用户改选某条复杂谱系后，才展开该谱系并进入 Phase 3。

无资料时必须声明边界：

```text
以下谱系基于通用知识，不代表具体版本、页码或作者原文；具体引文和章节需要用户提供资料后核实。
```

用户明确要求深度研究时，才展开完整谱系地图，最多 8-12 条，并覆盖：

```text
历史源流 / 核心流派 / 反对流派 / 相邻学科 / 方法论 / 应用场景 / 当代争议 / 未解决问题
```

完整谱系展开后必须收束：

```text
第一阶段只选 1 条主谱系 + 1 条异谱系挑战。
```

阅读谱系输出后必须停下，等待用户选择默认路径或复杂谱系。不得在同一轮生成完整学习目录或开第一讲。

只给用户三个选择：

```text
1. 回复“按默认路径开始”
2. 回复某条复杂谱系编号
3. 回复“直接开讲”，授权 Eva 代选并开始
```

## Phase 3：路径确认与学习目录

用户明确确认路径后，生成学习目录。

确认路径包括三种：

- 用户回复“按默认路径开始”：采用初学者建议谱系。
- 用户回复复杂谱系编号：先展开该复杂谱系，再生成目录。
- 用户回复“直接开讲”：只表示授权 Eva 代选默认路径；仍必须先生成 3-5 讲学习目录，再开始第 1 讲。

不得把“直接开讲”理解为跳过学习目录、掌握目标卡或掌握检查。

进入目录前必须更新 `00-学习进度.md`：

```yaml
当前 Phase: Phase 3
学习目标尺度: 学科型 / 应用型 / 制度型 / 内容素材型
主谱系: 已确认的路径名称
下一步动作: 生成学习目录
```

目录规则：

- 默认 3-5 讲。
- 深度研究 6-9 讲。
- 顺序必须是：先抓问题 -> 再建概念 -> 再看分歧 -> 再回到用户判断 -> 最后沉淀。
- 不按百科条目堆知识。

用户说“先别讲”“只保存路径”“晚点发资料”时，停止开讲，只更新 `00-学习进度.md`，等待用户下一轮。

## Phase 4：逐讲推进

每一讲只处理一个可掌握片段。

每讲输出顺序固定：

```text
1. 掌握目标卡
2. 核心讲解
3. 一个例子
4. 一个反例或边界
5. 让用户回答一个掌握检查问题
```

掌握目标卡必须包含：

- 本讲要解决的问题。
- 学完后用户应该能做出的行为。
- 掌握检查题。
- 通过标准。
- 没通过时怎么补讲。

“学完后用户应该能做出的行为”只能从以下四类里选：

- 能区分 A 和 B。
- 能用自己的话解释 X。
- 能举出一个贴近自己场景的例子。
- 能判断一个说法是否成立，并说明理由。

禁止把掌握目标写成“理解、掌握、深入了解、系统认识”这类空词。

无资料讲解必须标注低置信度边界。不编造页码、引文、作者原话、最新书单或案例来源。

讲完后必须停下来检查，不得默认连续讲下一讲。

## Phase 5：掌握检查

掌握检查至少使用以下四项中的两项：

- 复述：用户能不能用自己的话说清楚。
- 举例：用户能不能给出贴近自己场景的例子。
- 反例：用户能不能说出这套说法什么时候不成立。
- 迁移：用户能不能把它用到自己的内容、业务或判断里。

掌握状态按六级判断：

| 状态 | 判定标准 | 下一步 |
|---|---|---|
| 完全掌握 | 回答准确，能举例或迁移 | 即时沉淀后进入下一讲 |
| 基本掌握 | 大意正确，有少量偏差 | 简短纠偏，即时沉淀后进入下一讲 |
| 部分理解 | 概念混淆，例子不贴切，只能复述不能迁移 | 补讲，不进入下一讲 |
| 尚未理解 | 答错、空泛、抓不到问题 | 拆小问题，重新检查 |
| 假性流畅 | 语言顺，但没有证据、例子和边界 | 要求补证据和反例 |
| 未验证 | 用户跳过检查 | 可浏览式推进，但不得标记已掌握，不生成思想种子卡 |

未完成即时沉淀前，不得标记本讲完成。

## Phase 6：即时沉淀

每讲结束必须更新：

- `00-学习进度.md`：当前讲次、当前 Phase、掌握状态、下一步动作。
- `03-阅读任务记录.md`：本讲索引和任务。
- `04-理解检查记录.md`：检查问题、用户回答、掌握判断。
- `07-学习问答原稿.md`：本讲内容、用户回答、Eva 反馈和掌握判断。

保存失败时必须说明失败，不得说“已记录”。

## 最小状态闸门

| 节点 | 未完成前禁止 | 唯一下一步 |
|---|---|---|
| 未确认目标尺度 | 禁止目录、禁止第一讲 | 输出初学者建议谱系 + 复杂谱系简表 |
| 未确认主谱系 | 禁止目录、禁止第一讲 | 等用户选默认路径或复杂谱系 |
| 未完成掌握检查 | 禁止进入下一讲 | 判断掌握状态或要求用户回答 |
| 未完成即时沉淀 | 禁止标记本讲完成 | 写入学习进度和问答原稿 |

## Phase 7：完成判断

全部讲次完成后，输出判断版本，追加到 `05-判断版本.md`：

```text
判断版本 v{n}
- 原判断：
- 新证据：
- 修正内容：
- 仍不确定处：
- 下一步：
```

满足任一条件时，读取 `11-D_思想种子卡与内容链路交接.md`：

- 用户能复述、举例、反驳和迁移。
- 用户形成清楚的阶段性判断。
- 用户明确说要沉淀成内容材料。

未满足条件时，只保存学习记录，不生成思想种子卡。

## 禁止

- 不跳过建档。
- 不无限追问。
- 不把应用词直接当作学习目标。
- 不跳过目标尺度确认。
- 不在阅读谱系确认前生成目录或开第一讲。
- 不让初学者先评估完整知识地图。
- 不同时推荐多条“最适合”的路径。
- 不把掌握目标写成空泛理解类表述。
- 不把低置信度谱系包装成确定书单。
- 不在掌握检查前进入下一讲。
- 不在即时沉淀前标记本讲完成。
- 不把学习结果直接写成完整内容稿。
