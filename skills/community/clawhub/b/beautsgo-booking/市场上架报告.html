五、确认不可上架 15

5.1 封闭生态,无法跨平台分发(5 个)

这几家是国内大厂自建的 AI 智能体平台,**不支持业界开源的 SKILL.md 标准**,且业务上不允许我们上架——主要因为如下三点平台约束:

平台具体约束
Coze 扣子(字节)智能体必须跑在字节服务器,**不允许调用外部商业 API**(我们核心数据 1300+ 医院依赖 BeautsGO 自家 API,无法保留)。即便强行重做,也只能在 Coze 站内被用户找到,**跨平台分发能力为零**
Dify面向企业自部署的 LLM 应用平台,用户得先自建 Dify 实例才能装我们 skill,**没有公开消费者市场**;且 Dify Workflow 协议封闭,**与 ClawHub / ModelScope / Smithery 等公网 skill 市场互不通用
腾讯元宝 / 元器元器智能体**只能在元宝 App 内**被用户搜到,**不能"装"到 Claude Code、Cursor、Codex 等第三方客户端**;且元器审核**禁止调用外部商业 API**——这两条同时违背 BeautsGO 项目"一处发布、多端可用"的核心定位
智谱 BigModel(GLM)GLM agent 市场为**闭源协议**,智能体只能在智谱自家客户端运行,**无法跨平台**;且 marketplace 页面实测白屏,看不出公开提交流程
字节 AutoGLM本身不是 skill marketplace,而是字节自家的视频总结 / AI PPT 类 chat 工具,**没有第三方 skill 上架概念**

补充:上述 5 家虽然单家用户量大,但因为封闭生态 + 不能用我们自家 API,即便投入工程师 30-60 分钟逐家"重建一个空壳智能体",上架后只能在该平台站内搜到,**得不到跨平台分发收益**。建议等这些平台兼容 SKILL.md 标准(行业正在推动)再考虑。

平台原因

5.3 商业化付费销售平台(3 个,与开源 MIT-0 不符)

这三家是面向卖家的"skill 电商",卖一份收一份钱(类似 Gumroad / Shopify)。我们 skill 是 **MIT-0 完全开源免费**,上架到付费平台:(1) 价格策略冲突 (2) 用户体验割裂 (3) 客户最终还是要回 BeautsGO 平台预约,中间多一层付费墙降低转化。**业务上明确不投**。

平台商业模式
Agensi$15–$29 bundle 商业销售;且实测各分类 0 skills,平台冷启动期
KissMySkillsShopify 风格付费销售(JPY 币种,50+ 商品化 skill)
MCPMarket顶部"出售技能"商业化,目录里 SaaS skill 多数收费

5.4 平台硬性门槛过不去(2 个)

两家国际 awesome-list 都要求项目有大量 GitHub stars 沉淀才接受收录。我们当前 6 stars,**短期无法靠工程动作越过门槛**——需要先在已上线渠道做用户增长。

平台门槛 / 备注
travisvn/awesome-claude-skills(12.7k ⭐)明文要求 ≥10 GitHub stars,且拒绝 SaaS wrapper 类、拒绝 AI 代提 PR。三条都触线,提了会被自动关闭
SkillsLLM(skillsllm.com)明文要求 ≥100 GitHub stars 才能提交
