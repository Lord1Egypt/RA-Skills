# MyKnowledge 使用指南

## 使用模式

MyKnowledge 支持三种使用模式：

| 模式 | 触发方式 | 适用场景 |
|------|----------|----------|
| 主动模式 | 用户明确指令 | 需要精确控制时 |
| 辅助模式 | 其他 Skill 调用 | 作为 Agent Team 的辅助 |
| 智能任务追踪 | AI 自动检测 | 日常对话中自动管理 |

---

## 主动模式

### 创建知识库

**全局知识库**（跨项目共享）：

```
用户：创建知识库
AI：请选择类型：
     [全局知识库] - 位于 ~/MyKnowledge/global/
     [项目知识库] - 位于当前项目目录

用户：全局知识库
AI：已创建全局知识库 ~/MyKnowledge/global/
     目录结构：
     - README.md
     - requirements/
     - public/
     - archive/
     - PROJECT-STATUS.md
```

**项目知识库**（项目专属）：

```
用户：创建知识库
AI：请选择类型：
     [全局知识库]
     [项目知识库]

用户：项目知识库
AI：已创建项目知识库 {project}/.myknowledge/
```

### 管理需求

**创建需求**：

```
用户：创建一个需求：分析 Q2 销售数据
AI：已创建需求 REQ-20250608-001
     位置：.myknowledge/requirements/REQ-20250608-001/

**查看需求状态**：

```
用户：查看项目状态

## 智能任务追踪

### 自动检测示例

当用户提出复杂任务时，AI 自动创建知识库：

```
用户：帮我分析这个销售数据，统计各区域销售额，
     并预测下季度趋势

AI：🤖 检测到复杂任务（多步骤数据分析）

     已自动创建知识库并记录需求 REQ-20250608-003
     
     任务分解：
     1. 数据清洗和验证
     2. 区域销售额统计
     3. 趋势预测分析
     
     现在开始执行...
```

### 复杂任务特征

AI 会自动检测以下特征：

- **关键词**（满足 3 个及以上触发）：分析、统计、挖掘、开发、设计、调研、整理、清洗、项目
- **多步骤**: 包含多个子任务
- **长期性**: 需要跟踪和记录

### 智能任务追踪配置

在 `settings.yaml` 中调整检测敏感度：

```yaml
complex_task_detection:
  min_keyword_count: 3  # 最少关键词数量（默认 3，可调 2-5）
  keywords:
    - "分析"
    - "统计"
    # ... 更多关键词
```

## 知识库结构

### 全局知识库

```
~/MyKnowledge/global/
├── README.md                 # 知识库说明
├── PROJECT-STATUS.md         # 项目状态快照
├── requirements/             # 需求目录
│   ├── README.md
│   └── REQ-20250608-001/     # 单个需求
│       ├── README.md         # 需求详情
│       └── design.md         # 设计文档（可选，参考 core/templates/design-doc-template.md）
├── public/                   # 公开文档
│   └── README.md
└── archive/                  # 归档需求
    └── README.md
```

### 项目知识库

```
{project}/.myknowledge/
├── README.md
├── PROJECT-STATUS.md
├── requirements/
├── public/
└── archive/
```

## 最佳实践

### 1. 何时使用全局知识库

- 跨项目的通用知识
- 个人学习笔记
- 工具和方法论总结

### 2. 何时使用项目知识库

- 特定项目的需求和文档
- 项目相关的数据分析
- 需要与项目代码一起版本控制的内容

### 3. 需求命名建议

```
好的命名：
- 分析 Q2 销售数据
- 重构用户认证模块
- 调研竞品功能

避免：
- 任务1
- 新需求
- 待办事项
```

### 4. 定期维护

```
用户：归档已完成需求
AI：已归档 3 个已完成需求到 archive/
```

## 平台差异

| 功能 | CodeBuddy | WorkBuddy | OpenClaw | Claude |
|------|-----------|-----------|----------|--------|
| 主动模式 | ✅ | ✅ | ✅ | ✅ |
| 智能任务追踪 | ✅（意图识别） | ✅（意图识别） | ✅（Hook + 意图识别） | ✅（意图识别） |
| 后台运行（操作后告知） | ✅ | ✅ | ✅ | ✅ |

**说明**：
- **CodeBuddy/WorkBuddy**: AI 通过意图识别判断是否创建知识库，会提示用户
- **OpenClaw**: 可通过 Hook 实现后台运行（操作后告知用户），并非"无感知"
- **Claude**: 主要靠意图识别，Hook 支持取决于具体环境

## 快捷键/命令

> 完整命令表见 `modules/commands/main.md`（13 条固定命令 + 同义词映射）。
> 常用高频命令：

| 命令 | 功能 |
|------|------|
| `创建知识库` | 创建新的知识库 |
| `新建需求 {描述}` | 创建新需求 |
| `查看 REQ-001` | 查看某个需求详情 |
| `REQ-001 完成` | 状态转 Done 并归档 |
| `项目进展如何` | 显示 PROJECT-STATUS.md |
| `继续 {项目}` | 恢复上次会话 |
| `搜索 {关键词}` | 跨需求关键词搜索 🚧 计划中（v1.2+） |
| `归档已完成` | 归档已完成需求 |
