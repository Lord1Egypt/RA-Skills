## Description: <br>
Design and implement RESTful API endpoints following FastAPI best practices. Use when creating new API routes, designing request/response schemas, or when the user asks for API design guidance. <br>

This skill is ready for commercial/non-commercial use. <br>

## Use Case: <br>
Developers and backend engineers use this skill to design FastAPI REST endpoints, request and response schemas, pagination, error handling, and security checks for tkms-style projects. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Generated endpoint designs may omit project-specific authentication, authorization, validation, or rate limiting. <br>
Mitigation: Review generated routes and schemas, then add the required controls before deploying them. <br>
Risk: Create, update, and delete endpoint guidance may be unsafe if copied into production without checking business rules and data handling. <br>
Mitigation: Validate mutation behavior through service-layer checks, tests, and code review before release. <br>
