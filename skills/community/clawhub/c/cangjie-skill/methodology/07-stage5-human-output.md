# 阶段 5 — 人性化输出（给人看的笔记+话术库）

> 🆕 此阶段为 cangjie-skill 改造新增。设计来源：08项目任务7「cangjie-skill改造方案」。
> 改造背景：原版 cangjie 只产出给 AI 使用的 skill 包。但一本书的知识不只想「让机器用」——人也需要一份能看懂、能直接消化吸收的笔记。

## 目标

基于阶段0-4的已有分析结果，产出两份给人看的文档：
1. **学习笔记**：大白话+类比+核心论点+批判视角
2. **话术库**：可引用的实用表达+对话场景模板

不额外消耗大量 token —— 复用阶段0的 BOOK_OVERVIEW.md + 阶段1.5高质量单元 + 阶段2的RIA++内容。

---

## 设计哲学

### 为什么不是重新分析，而是复用已有产出？

| 如果重新分析 | 复用已有产出 |
|------|------|
| 推倒重来，token翻倍 | 基于已验证的高质量内容加工 |
| 可能与RIA++结论矛盾 | 与skill包内容完全一致 |
| 额外的主agent推理开销 | 仅需模板化+风格化重构 |

**复用路径**：

```
阶段0 BOOK_OVERVIEW.md 的「结构」「解释」「批判」
  → 学习笔记的骨架 + 批判视角

阶段1.5 通过的单元
  → 学习笔记的核心内容池

阶段2 每个skill的RIA++六段
  → R → 话术库的金句素材
  → I → 学习笔记的骨架段落
  → A1 → 学习笔记的案例素材
  → A2 → 话术库的「什么时候说」
  → E → 话术库的「怎么说」

阶段3 INDEX.md 的引用关系
  → 学习笔记的知识地图
```

## 产出物

### 产出 1：学习笔记（LEARNING_NOTE.md）

**目标读者**：想快速了解这本书核心方法论的普通读者
**写作风格**：
- 大白话开篇（一句话类比）
- 每章/每主题：核心主张 + 关键方法 + 一句话案例
- 批判视角：作者可能没看到的盲点
- 零术语滥用，凡用必解释

**模板**：`templates/LEARNING_NOTE.md.template`

**质量要求**：
- 开篇一句话类比（让人秒懂这本书在讲什么）
- 每条方法论配一句话解释（不看原书也懂）
- 批判视角独立成段（不只是「局限」，是「作者可能错了/漏了」）
- ≤3000字（精炼，不为凑字数）

---

### 产出 2：话术库（TALKING_POINTS.md）

**目标读者**：需要在工作/生活中实际运用书中方法的人
**写作风格**：
- 场景化口语（就像朋友给你支招）
- 每个方法论配「面对XX问题时怎么说」
- 包含具体的表达范例

**模板**：`templates/TALKING_POINTS.md.template`

**质量要求**：
- 每条话术必须有「什么时候用」标签
- 话术字数≤100字/条（真的能拿来就说的长度）
- 话术不等于金句——金句是作者说的，话术是「你」在真实场景中能说的
- ≥10条话术（覆盖书中核心方法论）

## 执行步骤

### 5.1 收集复用素材

从已有产出中提取：

1. **读 BOOK_OVERVIEW.md** → 获取：
   - 一句话主旨
   - 骨架（论点结构）
   - 关键术语（作者用法 vs 常识）
   - 批判（时代局限/立场盲点/未被证明假设/最强反对意见）

2. **读 INDEX.md** → 获取：
   - 各skill之间的引用关系
   - skill总览

3. **读每个通过的skill的 SKILL.md** → 获取：
   - R（原文金句）+ I（自述骨架）+ A1（书中案例）+ A2（触发场景）+ E（可执行步骤）+ B（边界）

### 5.2 写学习笔记

按 `templates/LEARNING_NOTE.md.template` 填充：

1. 开篇类比 —— 从全书主旨提炼一个生活化类比
2. 核心方法论 —— 每个skill对应一段，I段改写为更口语化的版本
3. 关键概念 —— 术语词典（来自BOOK_OVERVIEW.md的关键术语表）
4. 批判视角 —— 作者可能错了/漏了什么（来自BOOK_OVERVIEW.md的批判段）
5. 适合谁读 —— 一句话判读
6. 知识地图 —— 各方法论之间的关系（来自INDEX.md）

### 5.3 写话术库

按 `templates/TALKING_POINTS.md.template` 填充：

1. 对每个skill的A2（触发场景），生成对应话术
2. 话术源自R（原文素材）+ E（可执行步骤转口语）
3. 每条话术配：
   - 什么时候用（场景标签）
   - 怎么说（口语化表达）
   - 关键点（这句的意图是什么）

### 5.4 质量检查

- [ ] 学习笔记开篇有类比
- [ ] 每条方法论有白话解释
- [ ] 批判视角独立且尖锐（不是温和的局限说明）
- [ ] 话术库≥10条
- [ ] 话术≤100字/条
- [ ] 话术有场景标签
- [ ] 两份文档读起来像朋友聊天，不像教科书

## 与阶段4的关系

阶段4（压力测试）验证的是**给机器的skill包**是否精准。

阶段5产出的学习笔记和话术库，质量判断标准不同——「一个没读过原书的人看完后能不能用几句话转述给别人听」。

两个阶段可以并行：阶段4测skill，同时阶段5写人性化输出。但阶段的流水线编号保持不变，阶段5在阶段4之后输出。
