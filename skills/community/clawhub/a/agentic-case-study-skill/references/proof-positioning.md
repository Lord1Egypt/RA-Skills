# CompleteTech LLC Proof Positioning

## Proof Frame

CompleteTech LLC proof assets should package evidence carefully. The goal is to show practical agentic workflow delivery without exposing confidential information or overstating outcomes.

## Evidence Rules

Use only:

- client-approved facts
- approved quotes
- measured outcomes with source/context
- qualitative observations clearly labeled as qualitative
- anonymized details when attribution is not approved
- implementation facts that do not reveal sensitive architecture or client data

Do not invent:

- ROI or savings
- client names
- regulated-use approvals
- legal claims
- production readiness
- evaluation results
- quotes
- before/after metrics

## CompleteTech LLC Themes

Preserve these themes:

- bounded agentic workflow implementation
- workflow discovery and mapping
- tool routing and retrieval
- human approval gates
- evaluation examples and acceptance review
- logs and monitoring
- documentation, support, and handoff

## Anonymization Rules

When public attribution is not approved:

- remove client name
- remove unique project names
- generalize industry and size
- remove exact system names if sensitive
- generalize metrics if not approved
- avoid screenshots or quotes unless approved
- state "anonymized" or "representative" where appropriate

## Outcome Language

Measured:
- "Reduced review time from X to Y based on {source}."
- "Processed X sample cases during evaluation."

Qualitative:
- "Reviewers reported the draft queue was easier to audit."
- "The pilot clarified where human approval was required."

Unknown:
- "Outcome metrics not approved for public use."
- "Client approval required before publication."
