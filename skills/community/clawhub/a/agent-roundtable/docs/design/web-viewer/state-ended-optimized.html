讨论结论

采用混合方案：静态骨架 + SSE 增量更新

  • 首屏使用 SSR/SSG 渲染静态内容骨架，确保首屏加载速度 < 1.5s
  • 后续通过 SSE 通道增量推送新发言和状态变更，实现实时体验
  • 引入断线重连与消息去重机制，保证弱网环境下的可靠性
  • 讨论记录以 JSON 结构化存档，支持后续检索与复盘

技术上我支持混合方案。纯 SSR 没有实时性，纯 SPA 首屏太慢。SSG 出骨架 + SSE 推送增量，既能保证 TTI，又能实现毫秒级更新。

断线重连用指数退避 + Last-Event-ID，消息去重用 message UUID，这套方案在生产环境验证过，稳定性没问题。
