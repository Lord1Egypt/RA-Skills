## 核心理念

**Gemini 文生图，中文直达。** Gemini（Imagen / Gemini 2.0 Flash）是目前 AI 绘图工具中中文文字渲染最强的模型。所有文字在 prompt 中直接指定，出图即成品，无需后期加字。

---

## 一、Gemini 文生图核心优势

| 维度 | Gemini | Midjourney | 其他中文工具 |
|------|--------|------------|-------------|
| 中文渲染 | ⭐⭐⭐⭐⭐ 最强 | ❌ 不可靠 | ⭐⭐⭐-⭐⭐⭐⭐ |
| Prompt 理解 | 自然语言，长段落叙事 | 关键词驱动，逗号分隔 | 各有差异 |
| 布局控制 | 理解位置描述（顶部/左侧/居中） | 自然语言描述流向 | 中等 |
| 文字精准度 | 极高，几乎不出错 | 无法使用 | 偶有错字 |
| 比例控制 | prompt 中明确指定 | `--ar` 参数 | 各有方式 |

**结论**：做中文育儿信息图，Gemini 是唯一不需要后期加字的方案。

---

## 二、Gemini Prompt 优化技巧

### Prompt 三段式结构

每张图的 prompt 按以下结构组织：

```
【第1段：视觉基调】风格 + 配色 + 比例 + 整体氛围（2-3句）
【第2段：逐块布局】从上到下、从左到右逐一描述每个元素（主体篇幅）
【第3段：约束强调】文字渲染要求 + 排除项（1-2句）
```

### 优化要点

| 技巧 | 说明 | 示例 |
|------|------|------|
| **位置用分数** | 用「上1/5处」「距底1/6」代替「顶部」「底部」，更精确 | 「画面上方约1/5处」 |
| **卡片编号** | 多卡片时用「卡片1/卡片2」编号，AI 按序生成更稳定 | 「卡片1（浅蓝色背景）：...」 |
| **宽度用占比** | 用「占画面宽度60%」描述标题大小 | 「占画面宽度约60%的大号标题」 |
| **色彩三要素** | 主色+辅色+底色 分别描述 | 「深海军蓝主色、珊瑚橙点缀、米白底色」 |
| **对比描述** | 用对比帮 AI 理解层级关系 | 「标题深色大字 > 说明浅色小字」 |
| **单 prompt 单图** | 绝不在一段 prompt 里描述多张图 | 每张图一个独立 prompt |

### 图1：封面图 Prompt


## 四、Prompt 编写核心原则

| 原则 | 说明 | 示例 |
|------|------|------|
| **文字用「」标记** | 所有要出现在图上的中文用「」括起 | 标题 —「3岁宝宝一周陪玩指南」 |
| **位置用分数** | 用「上1/5处」「距底1/6」代替「顶部」「底部」 | 「画面上方约1/5处」 |
| **宽度用占比** | 标题大小描述为占画面宽度的百分比 | 「占画面宽度约60%的大号标题」 |
| **卡片编号** | 多卡片时用「卡片1/卡片2」编号，AI 按序生成 | 「卡片1（浅蓝色背景）：左侧emoji，右侧文字」 |
| **色彩三要素** | 主色+辅色+底色分别描述，指定对比关系 | 「深海军蓝主色、珊瑚橙点缀、米白底色」 |
| **末尾加硬约束** | 每段 prompt 结尾强调中文渲染 | 「重要：所有「」中的中文必须100%原样渲染在图片上」 |

## 五、文件输出规范

**每张图一个独立 `.txt` 文件**，文件内容为该图的完整 prompt，可直接复制到 Gemini。

```
02_图1_封面图.txt
02_图2_核心内容图.txt
02_图3_补充展开图.txt
02_图4_误区对比图.txt
02_图5_XXX.txt          ← 可选扩展，从图5开始续号
02_图6_XXX.txt          ← 可选扩展
02_末张_金句收尾图.txt   ← 始终在末尾
```
文件数量：简单选题 5-6 张，复杂选题 8-12 张。

## 六、Gemini Prompt 长度与参数

- **推荐长度**：200-600 中文字（Gemini 擅长处理长段落描述）
- **比例**：在 prompt 开头明确「3:4竖版」（1080×1440px）
- **风格一致性**：多张图共用一个风格描述开头，确保系列统一
- **错误处理**：如果出图中文有误，在原 prompt 基础上追加「请确保所有「」中的中文文字100%正确渲染，不要出现错字、漏字、乱码」

## 七、Prompt 自检清单

### 每张图检查
- [ ] 所有中文文字是否用「」标记？
- [ ] 文字位置是否用分数精确描述（上1/5处、距底1/6）？
- [ ] 标题宽度是否用占比描述（占画面宽度60%）？
- [ ] 多卡片是否编号（卡片1/卡片2）？
- [ ] 主色+辅色+底色是否分别描述？
- [ ] 比例是否明确（3:4竖版）？
- [ ] 年龄段徽章是否在每张内容图出现？
- [ ] prompt 末尾是否加了「重要：所有「」中的中文必须100%原样渲染在图片上」？
- [ ] 是否每个 prompt 只描述一张图（无跨图描述）？

### Carousel 整体检查
- [ ] N张图风格是否统一（同一组配色、同一风格描述开头）？
- [ ] 每张图信息密度是否递增（封面最少 → 金句图留白）？
- [ ] 图4是否有对比/反差设计？
- [ ] 金句收尾图是否在末张？
- [ ] 金句收尾图是否纯金句 + 留白（无任何引导文字）？
- [ ] 文件数量是否在 5-12 之间？
- [ ] 是否每个 prompt 一个独立文件？
