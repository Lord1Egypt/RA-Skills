## V0 需求分析域 范本

### 范本概述
- **域ID**: V0
- **域名称**: 需求分析域
- **任务数量**: 4
- **范本状态**: 可用

### 任务执行范本

#### V0-01 图表类型识别
**执行步骤**:
1. 接收用户需求
2. 分析需求内容
3. 识别图表类型
4. 确认图表类型
5. 输出识别结果

**输入示例**:
- 用户需求：我需要一个展示系统架构的图表
- 需求背景：用于技术文档

**输出示例**:
- 图表类型：结构图/架构图
- 图表特征：展示系统组件和关系
- 识别确认：需要生成架构图

**注意事项**:
- 图表类型要准确，避免误解
- 图表特征要全面，避免遗漏
- 识别确认要明确，避免模糊

#### V0-02 复杂度评估
**执行步骤**:
1. 分析图表内容
2. 评估元素数量
3. 评估关系复杂度
4. 确定复杂度等级
5. 输出评估结果

**输入示例**:
- 图表类型：架构图
- 图表内容：包含10个组件，多个连接关系

**输出示例**:
- 元素数量：10个组件
- 关系复杂度：中等
- 复杂度等级：中等
- 评估确认：中等复杂度图表

**注意事项**:
- 元素数量要准确，避免遗漏
- 关系复杂度要客观，避免主观臆断
- 复杂度等级要合理，避免过高或过低

#### V0-03 交互需求分析
**执行步骤**:
1. 分析用户交互需求
2. 识别交互类型
3. 确定交互级别
4. 确认交互需求
5. 输出分析结果

**输入示例**:
- 图表类型：架构图
- 交互需求：需要点击查看组件详情

**输出示例**:
- 交互类型：点击交互
- 交互级别：简单交互
- 交互需求：点击查看组件详情
- 分析确认：需要HTML格式支持交互

**注意事项**:
- 交互类型要准确，避免误解
- 交互级别要合理，避免过高或过低
- 交互需求要明确，避免模糊

#### V0-04 格式选择
**执行步骤**:
1. 分析图表需求
2. 考虑交互需求
3. 选择输出格式
4. 确认格式选择
5. 输出选择结果

**输入示例**:
- 图表类型：架构图
- 交互需求：需要点击查看组件详情
- 复杂度：中等

**输出示例**:
- 输出格式：HTML
- 选择理由：需要支持交互功能
- 格式确认：HTML格式是最佳选择

**注意事项**:
- 输出格式要合适，避免不合适的选择
- 选择理由要充分，避免牵强
- 格式确认要严格，避免问题

### 范本特征摘要
- **结构特征**: 按需求分析流程组织，从图表类型识别到格式选择，步骤清晰
- **风格特征**: 专业、详细、实用，使用明确的输入输出示例
- **逻辑特征**: 从类型识别到格式选择，逻辑严密，步骤明确
- **格式特征**: 使用Markdown格式，包含步骤、输入输出示例、注意事项
