## 模板2: 功能演示公告

**【功能演示】HR大姐头如何管理你的AI团队**

👥 **场景模拟**：假设你需要管理以下AI团队：
- 客服助手：处理用户咨询
- 运营助理：数据分析报告
- 文案编辑：内容创作
- 技术顾问：代码审查

🔄 **HR大姐头工作流程**：

**第1步：招聘员工**
```
你对HR说："招聘一个客服助手，起名叫客服小美"
```
→ 系统自动创建客服小美的工作空间

**第2步：绑定机器人**
```
你对HR说："帮客服小美绑定机器人，App ID是xxx，Secret是yyy"
```
→ 自动配置飞书机器人，重启生效

**第3步：分配工作**
```
你在飞书中@客服小美："请回答用户关于订单的问题"
```
→ 消息自动路由到客服小美的工作空间

**第4步：团队管理**
```
你对HR说："检查团队状态"
```
→ 显示所有员工状态和绑定情况

📊 **效率对比**：
- 传统方式：30分钟配置一个Agent
- HR大姐头：2分钟完成招聘+绑定
- 效率提升：15倍

```bash

```

🎯 **技巧3：故障恢复**
如果配置出错，使用自动备份恢复：
```bash
# 查找备份文件
ls -la ~/.openclaw/openclaw.json.bak_*

# 恢复备份
cp ~/.openclaw/openclaw.json.bak_xxx ~/.openclaw/openclaw.json
```

🎯 **技巧4：自定义身份**
每个Agent的`IDENTITY.md`、`SOUL.md`、`AGENTS.md`都可以自定义，打造专属人格。

## 模板6: 问题解答公告

**【常见问题解答】feishu-team-manager**

❓ **Q1：安装时遇到VirusTotal警告怎么办？**
A：这是正常现象。技能需要修改系统配置，因此被标记为"可疑"。代码已开源可审查，使用`--force`参数安装。

❓ **Q2：招聘后Agent不响应消息？**
A：请检查：
1. Gateway状态：`openclaw gateway status`
2. 配置正确性：`openclaw doctor`
3. 机器人权限：确保飞书机器人有消息接收权限

❓ **Q3：如何查看团队所有Agent？**
A：使用环境检查脚本：
```bash
cd ~/.openclaw/workspace/skills/feishu-team-manager
python3 scripts/check_env.py
```

❓ **Q4：可以管理多少个Agent？**
A：理论上无限制，但建议根据实际需求配置。每个Agent需要独立的飞书机器人。

❓ **Q5：技能更新后需要重新绑定吗？**
A：不需要。技能更新不会影响现有配置，但建议备份重要数据。

❓ **Q6：如何获取技术支持？**
A：
1. 查看文档：SKILL.md、QUICK_START.md
2. 提交Issue：技能页面
3. 群内咨询：@作者或群内讨论
