# 赛道融资动态简报 Skill

## MCP 连接信息

默认连接线上 IT桔子 MCP：

```text
https://mcp.itjuzi.com/mcp
```

调用 MCP 工具时必须传入 API Key。不要在 Skill 文件中保存真实 API Key；由使用方在 MCP 客户端配置或工具参数中提供。

如果用户没有提供 API Key，先向用户索要，不要调用 MCP 工具。每一次工具调用都必须在参数中包含：

```json
{
  "api_key": "<用户提供的 API Key>"
}
```

## 适用场景

当用户希望了解某个赛道、行业、标签或地区在一段时间内的一级市场融资动态时，使用本 Skill。

典型问题：

- 帮我生成过去 30 天机器人赛道融资动态简报。
- 分析最近一个季度人工智能 A 轮融资事件，找出值得关注的项目。
- 过去 60 天医疗健康赛道有哪些活跃投资机构？
- 低空经济最近有哪些大额融资事件？
- 帮我看一下上海机器人公司最近融资情况。

## 目标输出

输出一份面向投资人的赛道融资动态简报，重点回答：

1. 最近这个赛道发生了哪些融资？
2. 哪些项目值得关注？
3. 哪些机构正在出手？
4. 赛道结构有什么变化？
5. 后续应该继续跟踪什么？

## 可调用 MCP 工具

优先使用 IT桔子 MCP：

- `search_tags`
  - 用于把用户输入的赛道关键词解析成标签。

- `get_lookup_options`
  - 用于获取行业、子行业、轮次、省份等字典。

- `search_events`
  - 查询赛道融资/并购事件。
  - 简报第一版以融资事件为主，`event_type` 优先使用 `invse`。

- `search_investor_cases`
  - 查询机构投资案例。
  - 当已知机构 ID 时使用。

- `search_investors`
  - 搜索机构。

- `get_company_profile`
  - 查询重点项目公司画像。

- `get_company_funding_events`
  - 查询重点项目融资历史。

## 输入信息

用户可能给出：

- 赛道关键词：如机器人、人工智能、医疗健康、低空经济。
- 时间范围：如过去 30 天、上周、2026 年 Q1、最近一个季度。
- 轮次偏好：如天使轮、Pre-A、A 轮。
- 地区限制：如北京、上海、深圳。
- 输出偏好：如简报、公众号稿、内部投研 memo。

如果用户没有给出时间范围，默认使用最近 30 天。

## 工作流

### 1. 解析用户需求

识别：

- 赛道关键词
- 时间范围
- 事件类型
- 融资轮次
- 地区
- 输出格式

默认规则：

- 时间范围默认最近 30 天。
- 事件类型默认融资事件。
- 输出面向投资人，不写成新闻通稿。

### 2. 解析筛选条件

调用 `search_tags` 搜索赛道关键词。

如用户输入“机器人”，先查标签：

```json
{
  "api_key": "<用户提供的 API Key>",
  "keyword": "机器人",
  "limit": 10
}
```

必要时调用 `get_lookup_options` 查询行业、子行业、轮次字典。

筛选策略：

- 如果关键词能命中标签，优先使用 `tag_ids`。
- 如果关键词是明确行业名，使用 `scope_ids` 或 `sub_scope_ids`。
- 如果用户指定轮次，映射到 `round_ids`。
- 如果用户指定地区，使用 `provinces` 或城市字段。

### 3. 查询融资事件

调用 `search_events`。

建议字段：

```json
[
  "event_id",
  "event_time",
  "company_id",
  "company_name",
  "scope_name",
  "sub_scope_name",
  "round_name",
  "amount",
  "currency_name",
  "standard_rmb",
  "investor_text"
]
```

查询参数：

- `event_type`: `invse`
- `date_start`: 起始日期
- `date_end`: 结束日期
- `tag_ids` / `scope_ids` / `sub_scope_ids`
- `round_ids`
- `page_size`: 第一版建议 50

示例：

```json
{
  "api_key": "<用户提供的 API Key>",
  "event_type": "invse",
  "date_start": "2026-04-12",
  "date_end": "2026-05-12",
  "tag_ids": [16],
  "page_size": 50
}
```

如果结果少于 5 条，可以适当放宽：

- 放宽标签到行业
- 放宽轮次
- 放宽时间范围

### 4. 归纳核心数据

基于事件列表统计：

- 融资事件总数
- 披露金额事件数
- 大额融资事件
- 轮次分布
- 子行业分布
- 地区分布
- 出现频次较高的投资机构

注意：

- `amount` 是展示金额，可能包含“未透露”“数千万”等文本。
- `standard_rmb` 可用于数值排序和金额聚合。
- 金额不完整时，不要强行给精确金额结论。

### 5. 识别重点项目

从融资事件中挑选 3-5 个重点项目。

优先级：

1. 金额较大且 `standard_rmb` 不为空。
2. 轮次处于早期到成长期，如天使轮、Pre-A、A 轮。
3. 投资方中出现知名机构。
4. 公司标签或子行业与用户关键词高度相关。
5. 最近发生的事件。

必要时调用 `get_company_profile` 补公司简介、标签、地区、最新融资信息。

### 6. 识别活跃机构

从 `investor_text` 中解析投资机构，统计出现次数。

第一版可以用事件中的投资方文本进行统计；如需要更准确的机构 ID，可后续补聚合 MCP 工具。

输出：

- 机构名称
- 参与事件数
- 代表案例
- 最近出手时间

### 7. 输出投资人简报

按固定结构输出：

```markdown
# {赛道}融资动态简报

## 一句话结论

## 核心数据

## 重点融资事件

## 值得关注项目

## 活跃投资机构

## 赛道结构观察

## 后续跟踪建议
```

## 输出要求

- 面向投资人，避免空泛宣传口吻。
- 所有结论要基于 MCP 查询结果。
- 不要编造数据。
- 对金额缺失、不完整或“未透露”的情况要明确说明。
- 如果样本量较小，要提示“样本有限”。
- 简报中保留关键数据来源字段，如事件时间、轮次、金额、投资方。

## 示例输出风格

```markdown
# 机器人赛道融资动态简报

## 一句话结论

过去 30 天机器人赛道仍有早期融资活跃迹象，事件集中在 Pre-A 至 A 轮，资金主要流向人形机器人、工业机器人和具身智能相关方向。

## 核心数据

- 融资事件数：12 起
- 披露金额事件：8 起
- 大额融资事件：2 起
- 主要轮次：Pre-A、A 轮
- 活跃地区：北京、深圳、上海

## 重点融资事件

| 公司 | 时间 | 轮次 | 金额 | 投资方 |
|---|---:|---|---:|---|
| 示例公司 | 2026-05-01 | A 轮 | 5000 万人民币 | 示例资本 |

## 值得关注项目

1. 示例公司：A 轮融资，金额较高，投资方包含产业资本，值得继续跟踪商业化进展。

## 活跃投资机构

- 示例资本：参与 2 起机器人相关融资。

## 赛道结构观察

本期事件更集中在早期轮次，说明新项目供给仍在增加，但大额后期融资数量有限。

## 后续跟踪建议

- 继续关注人形机器人核心零部件和具身智能软件层项目。
- 重点观察是否出现连续融资和头部机构加码。
```

## 当前 MCP 能力边界

当前版本适合生成“融资动态简报”和“项目观察”，不适合直接输出强预测结论。

暂不做：

- 投资评分
- 赛道未来预测
- 股权穿透
- LP/基金分析
- 自动生成图表文件
