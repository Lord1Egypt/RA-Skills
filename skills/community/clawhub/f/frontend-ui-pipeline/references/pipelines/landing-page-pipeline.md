# Landing Page Pipeline

Use this when the user wants a homepage, launch page, product marketing page, waitlist page, or conversion-focused website.

## Recommended strategy
- Default to **conversion-first**
- Add **brand-expression-first** if visual memorability matters
- Add **mobile-first** if traffic is likely mobile-heavy

## Step 1. Clarify essentials
- What product or offer is this page selling?
- Who is the ideal visitor?
- What is the single main action? sign up / book demo / buy / join waitlist / contact
- What proof exists? testimonials / logos / stats / case studies / screenshots
- Are there reference brands or pages?

## Step 2. Define design direction
- Common style choices: minimalism / bento grid / flat / glass accents
- Typical layout: hero + sections
- Typical navigation: top nav or simplified nav
- Key interaction goal: make CTA obvious and low-friction

## Step 3. Build section plan
Typical sections:
1. Hero
2. Social proof
3. Problem
4. Solution / product overview
5. Key benefits or features
6. Screenshots or demo
7. Testimonials / trust signals
8. FAQ
9. Final CTA

## Step 4. Build plan requirements
- Strong headline and subheadline hierarchy
- One primary CTA per screen region
- Consistent visual rhythm between sections
- Mobile-readable spacing and type sizes
- Loading states only if interactive widgets exist

## Step 5. Review priorities
Focus review on:
- CTA clarity
- Message hierarchy
- Scanability
- Trust and credibility
- Mobile responsiveness
- Accessibility of buttons, contrast, and forms

## Common risks
- Too many competing CTAs
- Pretty visuals but unclear product value
- Weak proof section
- Overlong hero with no immediate action
- Poor mobile spacing
