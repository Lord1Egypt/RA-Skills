## A4 结果汇总域 范本

### 范本概述
- **域ID**: A4
- **域名称**: 结果汇总域
- **任务数量**: 5
- **范本状态**: 可用

### 任务执行范本

#### A4-01 结果收集
**执行步骤**:
1. 界定收集范围
2. 选择收集方法
3. 执行结果收集
4. 确认收集结果
5. 输出收集报告

**输入示例**:
- 收集范围：所有Agent的执行结果
- 收集需求：收集所有任务执行结果

**输出示例**:
- 收集范围：UI设计、前端开发、后端开发、测试
- 收集方法：结果提交、代码审查、测试验证
- 收集结果：
  - UI设计：已完成，设计稿已提交
  - 前端开发：已完成，代码已提交
  - 后端开发：已完成，代码已提交
  - 测试：已完成，测试报告已提交
- 收集确认：所有结果已收集

**注意事项**:
- 收集范围要明确，避免遗漏
- 收集方法要有效，避免无效收集
- 收集结果要完整，避免缺失

**输出示例**:

- 验证方法：功能测试、性能测试、代码审查
- 验证结果：

  - 前端开发：通过，功能完整，性能达标
  - 后端开发：通过，功能完整，性能达标
  - 测试：通过，所有测试用例通过
- 验证确认：所有结果验证通过

**注意事项**:
- 验证标准要严格，避免宽松
- 验证方法要有效，避免无效验证
- 验证结果要准确，避免错误

1. 界定收集范围
2. 选择收集方法
3. 执行反馈收集
4. 分析收集结果
5. 输出反馈报告

**输入示例**:
- 收集范围：用户、团队成员
- 收集需求：收集对协作流程的反馈

**输出示例**:
- 收集范围：用户、团队成员
- 收集方法：问卷调查、访谈、观察
- 收集结果：
  - 用户反馈：满意，希望提高响应速度
  - 团队成员反馈：协作顺畅，希望优化沟通流程
- 分析结果：协作流程整体良好，有优化空间

**注意事项**:
- 收集范围要明确，避免遗漏
- 收集方法要有效，避免无效收集
- 收集结果要准确，避免错误
