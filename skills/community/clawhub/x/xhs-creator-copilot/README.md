## 📦 这是什么

一份运行在 **Cowork / Claude.ai / Claude Code / OpenClaw** 等环境的 Skill,帮小红书创作者:

- 起草笔记内容(3 个候选版本)
- 去 AI 化润色(让文字更像真人写的)
- 起草私信/评论回复
- 批量分诊评论通知
- 合规预检(敏感词 + 免责声明)
- 本地文件归档(按日期 + 规范命名)

**所有产出都以本地文件形式交付**,真人完成最后一步:打开小红书 → 复制粘贴 → 点击发布 / 发送。

---

## ⚖️ 为什么这样设计(必读)

### 平台规则背景

2026-03-10 小红书发布《关于打击 AI 托管运营账号的治理公告》:

- ❌ 禁止 AI 托管发布、互动、注册
- 🔴 主页所有公开笔记均为 AI 代发 → **直接封禁**

2025-09-01《人工智能生成合成内容标识办法》生效:

- AI 生成内容必须显式标识
- 主动声明 AI 内容 → 正常分发
- 未声明 AI 内容 → 限流降权

### 本工具的合规设计

| 维度 | 我们的做法 |
|------|----------|
| 自动登录小红书 | ❌ 不做 |
| 自动发布笔记 | ❌ 不做 |
| 自动回复评论 / 私信 | ❌ 不做 |
| 自动监听通知 | ❌ 不做 |
| 自动浏览检索 | ❌ 不做 |
| 内容生成 | ✅ 本地完成,所有输出强制提示"必须勾选 AI 标识" |
| 真人完成最终发布 / 互动 | ✅ 强制 |

### 为什么这样反而更安全

平台公告原话:**"主动声明 AI 内容,不限分发"**。

- 试图躲过 AI 检测 → 一旦被识别,封禁
- **主动声明 + 真人发布** → 合规且不限流
- 这是**长期可持续**的方式

## 🎯 核心使用场景

### 1. 起草一条笔记

```
你:帮我写一条期货入门笔记,主题是"期货的核心特征",目标引导关注

skill:
  → Step 1 询问素材(可选)
  → Step 2 生成 3 版本候选(干货 A / 故事 B / 播报 C)
  → Step 3 等你选(回复 A/B/C)
  → Step 4 去 AI 化(humanize medium)
  → Step 5 合规预检(自动补免责声明)
  → Step 6 输出到 outputs/2026-04-25/notes/note-tuwen-qihuo-tezheng-final.md
  → Step 7 给你 todo 清单,真人发布
```

最终文件长这样,**全选复制粘贴**到小红书即可:

```markdown
<!-- 复制此分隔线以下的内容到小红书 -->

【标题】
关于期货,我踩过的 3 个坑

【正文】
...

#期货入门 #投资避坑

—————————
以上仅为个人分享,不构成投资建议。

<!-- 复制完成后:必须勾选「设置→内容类型声明→笔记含 AI 合成内容」⚠️ -->
```

### 2. 起草一条私信回复

```
你:粉丝问"看你笔记说期货能做空,棕榈油现在能空吗",
   帮我起草回复

skill:
  → 检索 FAQ 库找类似历史
  → 生成 3 版候选(短/中/长)
  → 去 AI 化 + 合规预检
  → 输出到 outputs/2026-04-25/replies/reply-sixin-粉丝-1430.md
  → 你选 1 条,复制 → 切到小红书 → 粘贴 → 真人发送
```

### 3. 批量分诊每日评论

```
你:[上传 2 张小红书通知页截图]
   帮我整理今天要回的评论清单

skill:
  → 视觉识别截图,提取评论(昵称 / 时间 / 内容 / 笔记上下文)
  → 分类(🔴高优 / 🟡中优 / 🟢低优 / ⚠️引流嫌疑)
  → 为非低优/特殊评论生成回复候选
  → 输出到 outputs/2026-04-25/triage/triage-1900.md
  → 你按优先级在小红书 App 里逐条手动回复
```

### 4. 把 AI 写的东西改得像人写的

```
你:这段是 ChatGPT 写的,帮我改自然点 [paste 文本]
   humanize_level=medium

skill:
  → 识别 24 类 AI 写作痕迹
  → 改写每个命中点
  → 输出对照报告(原文 / 改后 / 改了哪些点)
  → AI 痕迹评分:78 → 32(目标 ≤40)
```

## ❓ 常见问题

**Q1: skill 真的不会接触小红书吗?会不会偷偷登录?**

A: 不会。SKILL.md 明确禁用所有浏览器工具,即使工具可用也不调用。你可以审阅本仓库的全部代码确认,没有任何对 `xiaohongshu.com` 的网络请求。

**Q2: 我已经在 v1.8 时被封过一次,用 v1.0 安全吗?**

A: v1.0 的设计本身合规,但**已封账号建议申诉**。如果账号已被永封,新账号需要换设备 / IP / 浏览器 profile 完全冷启动,避免风控连坐。详见 `references/compliance-guide.md`。

**Q3: 用了 humanizer 是不是就能"骗过"小红书的 AI 检测?**

A: ⚠️ 这不是 humanizer 的目的。humanizer 是为了让文字"读起来更像真人写的",**不是**为了"绕过检测"。**正确流程**:humanize 改完后,你**仍然**在发布时主动勾选"笔记含 AI 合成内容"。这样既合规,又因为内容自然不被限流。

**Q4: skill 能帮我自动监听评论,只读不写,可以吗?**

A: 不可以。哪怕是"只读"的自动化轮询,也属于平台规则禁止的"通过技术手段介入"。本 skill 设计上**不开放**这个功能。请用小红书 App 的原生通知,然后用本 skill 的 reply-triage 模式批量处理。

**Q5: 选题辅助怎么做?需要我手动喂素材?**

A: 是的。skill 不主动抓取小红书数据。建议每天 15 分钟真人浏览 App,把感兴趣的笔记**收藏 / 截图**,粘贴给 skill。这是**目前唯一长期合规**的选题路径。

**Q6: 本地文件输出在哪里?**

A: skill 在你启动它的工作目录下创建 `outputs/YYYY-MM-DD/{notes,replies,triage,humanized}/` 子目录。所有产出物按日期归档,文件名规范,直接打开复制粘贴即可。

## 📜 使用约定

- 任何形式的自动发布
