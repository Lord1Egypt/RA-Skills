# 豆包会话 - Skill 与工作流架构讨论

**时间：** 2026-03-07 16:30  
**主题：** Skill vs 工作流 vs 原子动作的架构定义

---

## 📋 会话内容

### 用户疑问

1. "TTS 工作流 = ATOM-VISUAL-008 + ATOM-DELIVERY-011 + ATOM-DELIVERY-013 你管这个叫工作流，他为什么不会是一个 skill 呢？"

2. "是不是我们不应该提原子级动作这个词语？而我们今天所聊的所有的原则及动作其实都是一个 skill 的颗粒度。"

3. "我现在感觉大概率你是把 tts 工作流写错了。应该是 tts skill。"

### 豆包回复摘要

**Skill（技能）vs 工作流（Workflow）的区别：**

1. **一句话总结**
   - Skill = 单个小能力、小工具
   - 工作流 = 一串按顺序自动执行的任务

2. **核心区别**
   - Skill 是零件，工作流是把零件拼成流水线
   - Skill 手动点一下，工作流自动跑到底
   - Skill 无状态，工作流有步骤、有顺序、有依赖

3. **三层架构**
   - **原子动作层**：最小、不可拆分、单一执行单元
   - **Skill 层**：原子动作的封装与组合，提供可直接调用的能力块
   - **工作流层**：Skill 的有序串联与自动化执行

### 用户结论

**用户质疑完全正确：**
- "TTS 工作流"应该改为"TTS Skill"
- TEMPLATE-* 应该改为 SKILL-*
- 组合模板应该称为 Skill 组合

---

## 📝 关键洞察

**三层架构明确定义：**

| 层级 | 定义 | 组成 | 示例 |
|------|------|------|------|
| 原子动作 | 最小、不可再拆分、单一执行单元 | 单一操作 | ATOM-VISUAL-008 |
| Skill | 多个原子动作 + 简单逻辑的组合 | 2-6 个原子动作 | SKILL-TTS-001 |
| 工作流 | 多个 Skill 的有序串联与自动化执行 | 多个 Skill | WORKFLOW-DOUBAO-001 |

**核心洞察：**
- 原子动作 = 零件（扳手、螺丝刀）
- Skill = 工具（由零件组装）
- 工作流 = 生产线（由工具串联）
