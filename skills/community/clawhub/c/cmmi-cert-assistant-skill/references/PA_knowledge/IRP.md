# IRP — 事件解决与预防（CMMI V3.0 Domain-specific PA）

## PA 基本信息
- **缩写**：IRP
- **英文**：Incident Resolution and Prevention
- **中文**：事件解决与预防
- **所属**：Domain-specific PA（领域特定实践域）
- **适用领域**：IT服务管理、运维管理
- **对应 V2.0**：OT（组织培训）的部分内容

---

## CMMI V3.0 要求要点

### 核心目的
确保及时解决事件，并预防事件复发。

### 关键实践（Specific Practices）
| 实践编号 | 实践内容 | 说明 |
|---------|---------|------|
| IRP 1.1 | 识别事件 | 识别、记录、分类事件 |
| IRP 1.2 | 分析事件 | 分析根本原因 |
| IRP 1.3 | 解决事件 | 实施解决方案 |
| IRP 1.4 | 验证解决方案 | 确认解决方案有效 |
| IRP 2.1 | 预防事件复发 | 实施预防措施 |
| IRP 2.2 | 管理服务事件 | 管理事件生命周期 |
| IRP 3.1 | 持续改进事件管理 | 基于数据分析改进流程 |

### 通用实践（Generic Practices）
- GP 1.1 执行特定实践
- GP 2.1 建立组织政策
- GP 2.2 策划过程
- GP 2.3 提供资源
- GP 2.4 分配责任
- GP 2.5 培训人员
- GP 2.6 管理配置
- GP 2.7 识别并记录干系人需求和接口
- GP 2.8 监督和控制过程
- GP 2.9 客观评估符合性
- GP 2.10 高层审查

## 用户文档 → PA 匹配规则

### 高概率匹配（主要PA）
- 文档名含"事件"、"Incident"、"故障" → **IRP 主要匹配**
- 文档含事件记录、故障报告 → **IRP 主要匹配**

### 中概率匹配（次要PA）
- 文档含服务水平协议 → 同时匹配 **SDM**（服务交付管理）
- 文档含问题管理 → 同时匹配 **CAR**（因果分析与解决）

## 评估访谈常见问题（模式D）

### 针对运维人员的问题
1. "事件是如何识别和记录的？有记录吗？"
2. "事件分析是如何进行的？有根本原因分析吗？"
3. "事件解决后如何验证？有验证记录吗？"
4. "如何预防事件复发？有预防措施吗？"

### 评分要点
- **IRP 1.1-1.4**：是否有系统化的事件识别和解决流程
- **IRP 2.1-2.2**：是否有事件生命周期管理和预防措施
- **IRP 3.1**：是否有基于数据的事件管理改进
