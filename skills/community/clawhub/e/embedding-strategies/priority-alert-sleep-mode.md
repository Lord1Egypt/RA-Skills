# 优先级提醒 - 息屏检测逻辑

**更新时间：** 2026-03-11 01:49  
**更新原因：** 用户被语音播报吵醒，需要添加息屏检测

---

## 📋 **修改内容**

### 原有逻辑
```powershell
# 每小时自动触发优先级提醒
# 1. 生成 TTS 语音
# 2. 发送飞书消息
# 3. 本地播放 MP3
```

### 新增逻辑
```powershell
# 在触发语音播报前，先检测电脑是否息屏
# 如果息屏 → 跳过语音播报，只发送文字消息
# 如果亮屏 → 正常播报（TTS + 文字）
```

---

## 🔧 **息屏检测方法**

### 方法 1：检测最后输入时间
```powershell
# 获取最后用户输入时间
$lastInput = Get-LastInputInfo
$idleTime = (Get-Date) - $lastInput

# 如果空闲时间 > 5 分钟 → 判定为息屏/离开
if ($idleTime.TotalMinutes -gt 5) {
    # 跳过语音播报
    Write-Host "检测到息屏，跳过语音播报"
}
```

### 方法 2：检测显示器电源状态
```powershell
# 检测显示器是否开启
$monitorPower = Get-WmiObject -Namespace root\wmi -Class WmiMonitorBasicDisplayParams
if ($monitorPower.Active -eq $false) {
    # 跳过语音播报
    Write-Host "显示器关闭，跳过语音播报"
}
```
