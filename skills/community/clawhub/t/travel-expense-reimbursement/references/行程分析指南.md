# 行程分析指南

## 第一阶段：生成详细行程安排

1. 读取 `processed/invoice-list.json` 中的所有发票
2. 按时间排序：车票按乘车日期+发车时间，住宿按入住日期
3. 将车站名转换为城市名（如"南京南站"→"南京"）

### 行程识别原则

- 行程是从出发地→目的地→返回地的完整过程
- **短行程**：仅一个目的地（如"南京→北京→南京"）
- **长行程**：多个目的地（如"南京→上海→杭州→北京→南京"）

### 输出格式

在 `output/travel-schedule.md` 中生成：
```
行程1：南京 → 北京 → 上海（短行程、停留住宿）
    分段1交通：南京 → 北京，高铁，出发日期：2026-04-01，结束日期：2026-04-01
    分段1停留：北京，住宿：有，开始：2026-04-02，结束：2026-04-04，住宿方式：宾馆，2天
    分段1行程总天数：3天
    ------------------------------------
    分段2交通：北京 → 上海，高铁，出发日期：2026-04-04，结束日期：2026-04-04
    分段2行程总天数：0天
```

## 第二阶段：完善信息

遇到"待确认"时询问用户补充。

## 第三阶段：生成报销表单数据

### 短行程（往返交通工具相同）
一个行程一条记录。

### 短行程（往返交通工具不同）
分为两条记录。

### 长行程
每个分段一条记录，最后分段单独处理。

### 输出格式

在 `output/travel-detail.md` 中生成：
```
----------------------
行程：南京→北京（出差）
出发日期：2026-04-01
结束日期：2026-04-01
是否结束：否
总天数：3
出发地：南京
出差地：北京
交通：火车
是否享受补贴：是
宾馆天数：2
宿舍天数：0
是否入住协议酒店：否
----------------------
行程：北京→上海（出差）
出发日期：2026-04-04
结束日期：2026-04-04
是否结束：是
...
```

## 检查规则

1. 前一分段结束日期不晚于后一分段出发日期
2. 除最后分段外，其他"是否结束"为"否"
3. 所有字段无缺失
