## 二、合同系统

### 2.1 系统定位

**定位**：权利层 - 签约即确权

**核心功能**：
- 合同模板管理
- 电子签约
- 贡献权重确认
- 合同上链存证

### 2.2 核心流程

```
1. 【选择模板】根据SPV类型选择对应合同模板
        ↓
2. 【填写参数】填写各方贡献权重、分账比例、里程碑
        ↓
3. 【合规审查】合规长老审核合同内容
        ↓
4. 【电子签约】各方在线签署
        ↓
5. 【上链存证】签约信息上区块链存证
        ↓
6. 【生成贡献权重】签约完成，生成初始贡献权重
```

### 2.3 合同类型矩阵

| SPV类型 | 合同类型 | 签约方 | 核心内容 |
|---------|---------|-------|---------|
| 联盟型 | 《社区生意联盟合作公约》 | 联盟成员与盟主 | 权利义务、服务内容、分账比例 |
| 项目型 | 《项目软SPV合作协议》 | 项目成员与牵头方 | 贡献权重、里程碑、验收标准 |
| 国企孵化 | 《项目孵化合作协议》 | 子SPV与母SPV | 服务包、母SPV占股、收益分配 |

### 2.4 合同模板清单

| 模板名称 | 用途 | 使用场景 |
|---------|------|---------|
| 联盟合作公约 | 联盟成员基础协议 | 联盟型SPV |
| 项目合作协议 | 项目成员合作协议 | 项目型SPV |
| 服务采购协议 | 各方与牵头方协议 | 所有类型 |
| 贡献权重协议 | 贡献权重确认协议 | 项目执行中调整 |
| 共管账户协议 | 账户管理协议 | 资金管理 |
| 退出清算协议 | 退出时的清算协议 | 成员退出 |

## 三、账务台账系统

### 3.1 系统定位

**定位**：资产层 - 贡献记录与收益分配

**核心功能**：
- 贡献权重台账
- 收益分配台账
- 成本费用台账
- 权利台账
- 清算总台账

### 3.2 五大台账

#### 3.2.1 贡献权重台账

记录每个成员在项目中的贡献权重配置

| 字段 | 说明 |
|------|------|
| 成员ID | 成员唯一标识 |
| 项目ID | 项目唯一标识 |
| 贡献类型 | 技能/资源/时间/资金 |
| 基础权重 | 约定的权重比例 |
| 信用系数 | 当前信用系数 |
| 有效权重 | 基础权重 × 信用系数 |
| 变更记录 | 权重调整历史 |

#### 3.2.2 收益分配台账

记录每个成员应得和实得收益

| 字段 | 说明 |
|------|------|
| 分配批次 | 批次号 |
| 项目ID | 项目标识 |
| 分配金额 | 总分配金额 |
| 各方金额 | 各成员分配明细 |
| 税费扣除 | 扣除的税费 |
| 平台服务费 | 扣除的平台费 |
| 实际打款 | 实际打款金额 |
| 打款状态 | 待打/已打/失败 |

#### 3.2.3 成本费用台账

记录项目实际发生的各项成本费用

| 字段 | 说明 |
|------|------|
| 费用类型 | 成本类型 |
| 费用金额 | 金额 |
| 费用归属 | 归属成员 |
| 发票状态 | 已开/待开/无需发票 |
| 报销状态 | 待报销/已报销/拒绝 |

#### 3.2.4 权利台账

记录成员在联盟中的各项权利

| 字段 | 说明 |
|------|------|
| 权利类型 | 投票权/知情权/提案权等 |
| 权利状态 | 有效/暂停/丧失 |
| 行使记录 | 权利行使历史 |

#### 3.2.5 清算总台账

项目结束时的最终清算记录

| 字段 | 说明 |
|------|------|
| 项目ID | 项目标识 |
| 总收入 | 项目总收入 |
| 总成本 | 项目总成本 |
| 总税费 | 扣除的税费 |
| 平台服务费 | 平台收取的服务费 |
| 可分配收益 | 最终可分配金额 |
| 各方分配 | 各成员最终分配明细 |
| 清算状态 | 清算中/已完成/有争议 |

### 3.3 共管账户管理

#### 账户开设

| 项目 | 内容 |
|------|------|
| 账户类型 | 银行共管账户 / 第三方支付分账账户 |
| 开户名义 | 牵头方公司名义 |
| 账户命名 | [公司名]-OPC项目专用账户 |
| U盾配置 | 2个U盾（盟主1个+监事1个） |

#### 账户运营规则

| 规则 | 说明 |
|------|------|
| 授权机制 | 2人授权制（发起人+1位核心成员） |
| 大额授权 | ≥5万需3人授权 |
| 通知机制 | 所有操作实时短信/邮件通知全员 |
| 准入白名单 | 仅接受项目收入、合作方服务费 |
| 用途限制 | 仅可用于项目成本、税费、成员分配 |
| 对账机制 | 每日自动对账，每周生成对账单 |

## 四、设计规划系统

### 4.1 系统定位

**定位**：执行层 - 任务分配与里程碑管理

**核心功能**：
- 项目规划
- 任务分解
- 里程碑设定
- 执行跟踪
- 验收确认

### 4.2 核心流程

```
1. 【项目立项】确定项目目标、范围、周期
        ↓
2. 【任务分解】将项目分解为具体任务
        ↓
3. 【任务分配】将任务分配给各成员
        ↓
4. 【里程碑设定】确定关键里程碑节点
        ↓
5. 【执行跟踪】实时跟踪任务进度
        ↓
6. 【验收确认】里程碑完成后验收确认
        ↓
7. 【贡献记录】验收通过后记录贡献
```

### 4.3 里程碑管理

| 里程碑类型 | 说明 | 验收标准 |
|-----------|------|---------|
| 需求里程碑 | 需求确认完成 | 需求文档签字确认 |
| 设计里程碑 | 设计方案确认 | 设计稿通过评审 |
| 开发里程碑 | 功能开发完成 | 测试通过 |
| 交付里程碑 | 成果交付客户 | 客户签字确认 |
| 回款里程碑 | 款项到账 | 银行流水确认 |

### 4.4 贡献记录规则

| 贡献类型 | 记录方式 | 权重影响 |
|---------|---------|---------|
| 技能贡献 | 任务完成验收后记录 | 按任务价值评估 |
| 资源贡献 | 资源落地后记录 | 按资源价值评估 |
| 时间贡献 | 按工时记录 | 按约定单价计算 |
| 资金贡献 | 按实际出资记录 | 1:1记录 |

## 五、审计系统

### 5.1 系统定位

**定位**：合规层 - 全流程审计与合规检查

**核心功能**：
- 合同审计
- 账务审计
- 流程审计
- 合规审计
- 审计报告生成

### 5.2 审计类型

| 审计类型 | 频率 | 执行人 | 审计内容 |
|---------|------|-------|---------|
| 实时审计 | 实时 | 系统自动 | 关键节点自动触发 |
| 日审计 | 每日 | 系统自动 | 日终账务核对 |
| 周审计 | 每周 | 监事长老 | 本周运营复盘 |
| 月审计 | 每月 | 监事长老+合规长老 | 月度全面审计 |
| 里程碑审计 | 按里程碑 | 全体三长老 | 里程碑验收审计 |
| 退出审计 | 按需 | 全体三长老 | 成员退出清算审计 |

### 5.3 审计检查清单

#### 合同审计
```
□ 合同签署流程合规
□ 各方签字完整
□ 签约时间在有效期内
□ 合同内容与实际相符
□ 合同变更有据可查
□ 敏感词检查通过
```

#### 账务审计
```
□ 收入确认依据充分
□ 成本发票齐全
□ 税费计算正确
□ 分账比例与合同一致
□ 打款记录完整
□ 账户流水与账面一致
```

#### 流程审计
```
□ 任务分配有据可查
□ 里程碑验收标准明确
□ 验收流程执行到位
□ 贡献记录真实准确
□ 争议解决程序合规
```

## 六、风控系统

### 6.1 系统定位

**定位**：安全层 - 实时监控与风险预警

**核心功能**：
- 合规监控
- 资金监控
- 信用监控
- 预警机制
- 紧急制动

### 6.2 监控指标体系

| 监控维度 | 关键指标 | 预警阈值 |
|---------|---------|---------|
| **合规** | 敏感词出现次数 | >0 |
| **合规** | 违规操作次数 | >1/月 |
| **资金** | 单笔支出超限 | >5万 |
| **资金** | 账户余额异常 | <预留税费 |
| **资金** | 打款延迟 | >3天 |
| **信用** | 信用分骤降 | 单日降>10分 |
| **信用** | D级成员参与高权重任务 | 触发 |
| **进度** | 里程碑延期 | >7天 |
| **进度** | 任务完成率低 | <60% |

### 6.3 预警机制

| 预警级别 | 触发条件 | 处理方式 |
|---------|---------|---------|
| **蓝色预警** | 轻微偏离 | 系统提示，成员自查 |
| **黄色预警** | 中度偏离 | 盟主关注，要求说明 |
| **橙色预警** | 严重偏离 | 合规长老介入，暂停相关操作 |
| **红色预警** | 紧急风险 | 立即暂停，监事长老紧急处理 |

### 6.4 紧急制动机制

**触发条件**：
- 资金安全受到威胁
- 发生重大违规行为
- 系统检测到欺诈行为

**制动流程**：
```
1. 系统自动触发 / 人工触发
        ↓
2. 相关账户/操作立即暂停
        ↓
3. 全员通知
        ↓
4. 24小时内召开紧急会议
        ↓
5. 形成处理决议
        ↓
6. 执行决议，解除制动
```

## 八、数据流转闭环

### 8.1 完整数据流

```
1. 【合同系统】签约 → 生成贡献权重初始值
        ↓
2. 【设计规划系统】任务执行 → 里程碑验收
        ↓
3. 【账务台账系统】记录贡献 → 更新贡献台账
        ↓
4. 【信用台账】履约情况 → 更新信用分
        ↓
5. 【审计系统】全流程审计 → 生成审计报告
        ↓
6. 【风控系统】实时监控 → 预警/拦截
        ↓
7. 【区块链存证】→ 所有节点上链
```

### 8.2 系统联动关系

```
合同系统 ←→ 账务台账系统
  ↑              ↓
  ↓              ↓
设计规划 ←→ 审计系统
  ↑              ↓
  ↓              ↓
风控系统 ←→ 信用台账
  ↑
  ↓
区块链存证（全系统数据上链）
```
