## Description: <br>
Researches a codebase and creates architectural documentation for features, systems, onboarding, or knowledge transfer. <br>

This skill is ready for commercial/non-commercial use. <br>

## Publisher: <br>
[ziad-hsn](https://clawhub.ai/user/ziad-hsn) <br>

### License/Terms of Use: <br>


## Use Case: <br>
Developers and technical teams use this skill to research a codebase and produce markdown architecture documentation with diagrams, component descriptions, data flow, configuration notes, and stable code references. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Generated architecture documentation may be inaccurate or incomplete if the codebase scope or relevant code paths are misunderstood. <br>
Mitigation: Define the documentation scope before deep research and review the draft against the repository before sharing or relying on it. <br>
Risk: Documentation could be written to an unintended location if the destination is unclear. <br>
Mitigation: Confirm the output path before writing; the skill workflow requires explicit confirmation unless the user already provided a path. <br>
