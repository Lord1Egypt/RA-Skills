# M4 透明度控制域范本

## M4-01 默认隐藏策略

### 输入示例
任务类型：中等复杂度分析任务

### 处理过程
1. 隐藏范围确定：中间分析过程
2. 隐藏级别设定：中等
3. 隐藏原因说明：避免信息过载
4. 用户感知设计：展示关键结果和摘要

### 输出示例
```markdown
## 默认隐藏策略文档

### 隐藏范围
- **隐藏内容**：中间分析过程
- **隐藏级别**：中等
- **隐藏原因**：避免信息过载
- **用户感知**：展示关键结果和摘要

### 隐藏级别
| 级别 | 描述 | 适用场景 |
|------|------|----------|
| 低 | 展示所有细节 | 调试模式 |
| 中 | 展示关键步骤 | 常规模式 |
| 高 | 仅展示结果 | 快速模式 |

### 用户感知设计
- **展示内容**：关键结果和摘要
- **展示格式**：结构化文档
- **展示控制**：用户可切换详细程度
```

## M4-02 用户追溯请求处理

### 输入示例
用户请求：展示分析过程

### 处理过程
1. 请求接收：接收用户追溯请求
2. 请求验证：确认用户需求
3. 请求处理：展示关键步骤和决策点
4. 请求响应：提供详细分析过程

### 输出示例
```markdown
## 用户追溯请求处理流程

### 请求接收
- **请求类型**：展示分析过程
- **请求时间**：YYYY-MM-DD HH:MM:SS
- **请求来源**：用户界面

### 请求验证
- **验证内容**：确认用户需求
- **验证方法**：检查请求参数
- **验证结果**：请求有效

### 请求处理
- **处理内容**：展示关键步骤和决策点
- **处理方法**：调用状态演化追踪模块
- **处理结果**：获取分析过程数据

### 请求响应
- **响应内容**：详细分析过程
- **响应格式**：结构化文档
- **响应时间**：≤5秒
```

## M4-04 细节层级控制

### 输入示例
层级定义：摘要、关键、详细

### 处理过程
1. 层级定义：定义三个细节层级
2. 层级切换设计：用户可切换层级
3. 层级显示设计：不同层级显示不同内容
4. 层级控制实现：实时切换

### 输出示例
```markdown
## 细节层级控制方案

### 层级定义
| 层级 | 名称 | 描述 | 内容 |
|------|------|------|------|
| L1 | 摘要 | 仅展示结果 | 最终输出 |
| L2 | 关键 | 展示关键步骤 | 关键步骤和决策点 |
| L3 | 详细 | 展示所有细节 | 所有步骤和中间状态 |

### 层级切换
- **切换方式**：用户界面切换
- **切换粒度**：步骤级别
- **切换响应**：实时切换

### 层级显示
- **L1显示**：最终输出
- **L2显示**：关键步骤和决策点
- **L3显示**：所有步骤和中间状态

### 层级控制
- **控制方式**：用户界面切换
- **控制粒度**：步骤级别
- **控制响应**：实时切换
```
