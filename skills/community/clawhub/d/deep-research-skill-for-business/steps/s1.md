# S1: 提出核心问题

## 目标

将调研主题拆解为 3-5 个可回答的研究问题，每个问题配备关键词策略。

## ⛔ 红线

- 子问题不少于 3 个，不多于 5 个
- 每个子问题必须是**可被证据回答的**（不是观点、不是假设）
- 每个子问题配备至少 2 组不同关键词

## 操作

### 1. 拆解子问题

将 s0 确认的核心问题拆解为 3-5 个子问题，格式：

```markdown
### 子问题 1: [用问句表述]
- **论证目标**: 这个问题要证明/证伪什么？
- **需要的证据类型**: 数据？案例？政策？学术文献？
- **关键词组 A**: ...
- **关键词组 B**: ...
```

### 2. 输出完整拆解表（示例）

```markdown
## S1: 核心问题拆解

| # | 子问题 | 论证目标 | 证据类型 |
|---|--------|----------|----------|
| 1 | ... | ... | ... |
| 2 | ... | ... | ... |
| 3 | ... | ... | ... |
| 4 | ... | ... | ... |
```

### 3. 自检

- [ ] 每个子问题是否可被证据回答？
- [ ] 关键词组是否覆盖中英文双语？
- [ ] 关键词组是否多样化（不同角度、不同粒度）？

完成后执行：
```bash
bash orchestrator.sh --complete s1 --next s2
```
