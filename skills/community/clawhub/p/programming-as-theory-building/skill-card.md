## Description: <br>
Use when writing, generating, modifying, debugging, reviewing, or refactoring code. Treat code work as rebuilding and preserving the program theory: how real-world affairs map into program behavior, why the current shape exists, which existing facility the new demand resembles, and what observable behavior proves the change belongs. <br>

This skill is ready for commercial/non-commercial use. <br>

## Use Case: <br>
Developers and engineers use this skill to guide coding, debugging, refactoring, and review work by rebuilding the program theory behind a change before editing code. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: The skill influences how an agent approaches code changes and reviews, so poor application could slow trivial edits or over-focus on process. <br>
Mitigation: Apply the theory-building workflow primarily to non-trivial code work, as the artifact itself recommends, and keep summaries concise. <br>
Risk: Guidance-only outputs may still lead to incorrect code changes if the agent misreads the existing program theory. <br>
Mitigation: Require concrete verification through tests, commands, or review evidence before treating a proposed change as complete. <br>
