# ⚡ 5分钟快速上手

跟着这篇指南，你可以在5分钟内创建你的第一本自动化小说！

---

## 🎯 目标

创建一本3章的短篇都市悬疑小说《深夜便利店》

---

## 📋 前置检查

在开始之前，请确认：

- [ ] OpenClaw 已安装并正常运行
- [ ] 默认大模型已配置（推荐使用深度思考模型）
- [ ] 你有文件系统的读写权限
- [ ] 5个智能体已正确配置（见 agents/ 目录）

---

## 🚀 第1步：初始化项目（30秒）

```bash
# 进入技能目录
cd ~/.openclaw/skills/novel-multi-agent-skill/workflow

# 初始化项目
./init-project.sh ../my-first-novel "深夜便利店"
```

**你会看到：**
```
✅ 项目目录创建成功
✅ 总纲模板已复制
✅ 目录结构初始化完成

📁 项目路径: ../my-first-novel
📖 小说名: 深夜便利店
⏰ 创建时间: 2026-05-11

下一步: 编辑 00-总纲.md，确认需求后开始创作
```

---

## ✏️ 第2步：确认总纲（1分钟）

编辑 `../my-first-novel/00-总纲.md`：

```markdown
# 📋 《深夜便利店》创作总纲

## 项目基本信息
- 题材类型：都市悬疑
- 目标篇幅：3章，约6000字
- 时代背景：2024年，现代都市

## 核心设定
- 主线：夜班店员发现便利店的诡异秘密
- 风格：悬疑、惊悚、反转
- 核心悬念：每天凌晨3点，便利店会发生什么？

## 质量标准
1. 每章字数：1800-2200字
2. 逻辑严密，反转合理
3. 氛围营造到位
```

**保存文件，继续下一步。**

## ⚠️ 常见问题排查

### 问题：脚本执行失败
**解决方法：**
1. 检查 OpenClaw 是否正常运行
2. 检查大模型 API 是否可用
3. 检查文件目录权限
4. 查看日志文件定位问题

### 问题：输出内容不符合预期
**解决方法：**
1. 检查总纲的需求描述是否清晰
2. 可以在总纲中增加更详细的要求
3. 手动修改生成的设定，然后继续下一步

### 问题：字数不符合要求
**解决方法：**
1. 检查总纲中的字数要求是否明确
2. 可以在评审阶段要求写作智能体调整字数
3. 手动微调，增加或删减细节
