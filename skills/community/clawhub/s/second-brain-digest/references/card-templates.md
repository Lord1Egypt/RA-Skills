# 卡片模板与格式参考

## 各内容类型的卡片模板

---

### 文章 / 长文

适用于：博客、论文、书籍章节、深度报道

```
标题：[核心论点，陈述句]

核心内容：
作者认为 [核心观点]。
依据是 [主要证据或论证]。
适用于 [场景/条件]，不适用于 [反例/边界]。

来源：[作者]《[标题]》[年份]
标签：#[领域] #[方法论/工具/概念]
强度：[强/中/弱]
```

---

### 想法片段

适用于：碎片灵感、随手记录、聊天中产生的想法

```
标题：[把模糊想法变成清晰陈述]

核心内容：
[补全想法的完整逻辑]
[说明这个想法的适用场景]
[标注：这还是一个待验证的假设 / 已有经验支撑]

来源：用户输入 [日期]
标签：#[领域] #待验证 (如果是假设)
强度：弱（待验证）
```

---

### 会议 / 对话记录

适用于：会议纪要、访谈记录、重要对话

注意：会议记录通常产生多张不同类型的卡片

```
[决策卡片]
标题：[会议名称] 决定了 [具体决策]

核心内容：
决策内容：[是什么]
决策原因：[为什么]
决策条件：[在什么前提下成立，如果前提变了需要重新评估]

来源：[会议名称] [日期]
标签：#决策 #[项目名]
强度：强（已确认决策）

---

[行动卡片]
标题：[负责人] 需要在 [日期] 前完成 [任务]

核心内容：[任务描述 + 成功标准]

来源：[会议名称] [日期]
标签：#行动项 #[项目名]
强度：强
```

---

### 数据 / 研究

适用于：统计数据、调研结果、实验数据

```
标题：[数据说明了什么，不只是数字]
      例："远程工作者的生产力比办公室高13%（斯坦福2015）"

核心内容：
数据：[具体数字]
来源：[研究机构/时间/样本]
适用范围：[这个数据在什么条件下成立]
局限性：[样本偏差/时效性/地区限制等]

来源：[研究名称] [机构] [年份]
标签：#数据 #[领域]
强度：[取决于研究质量]
```

---

### 操作经验

适用于：个人经验总结、最佳实践、踩坑记录

```
标题：[做 X 的时候，Y 方法比 Z 方法更有效]

核心内容：
方法：[具体操作步骤或原则]
适用条件：[什么情况下用这个方法]
为什么有效：[背后的原理]
注意事项：[容易踩的坑]

来源：[个人经验/日期] 或 [来源]
标签：#方法 #[领域] #经验
强度：[中/弱]（个人经验，样本量有限）
```

## 卡片标题写法指南

### 好标题 vs 差标题

| 差标题（话题型）| 好标题（观点型）|
|-------------|-------------|
| 远程办公 | 远程办公在需要深度专注的工作中效率更高 |
| 用户增长 | 冷启动阶段靠社区比靠广告的ROI高3-5倍 |
| 睡眠重要性 | 睡前2小时避免屏幕可使入睡时间缩短40% |
| 读书方法 | 主动提问式阅读比被动阅读记忆留存率高2倍 |

### 判断标准
- 能不能独立成为一个"我知道"的陈述？
- 如果只看标题，能不能知道这张卡片说了什么观点？
- 有没有隐含"因此"或"所以"的逻辑？

## Obsidian / Notion 格式说明

### Obsidian 格式
```markdown
---
tags: [领域, 子话题]
strength: 强/中/弱
source: 来源
date: YYYY-MM-DD
---

# 标题（观点陈述）

核心内容...

## 关联
- [[相关卡片1]]
- [[相关卡片2]]

## 延伸问题
- 问题1
```

### Notion 格式
数据库属性：标题 / 标签（多选）/ 强度（选择）/ 来源 / 日期 / 关联（关系）
