## 技术发明任务清单

技术发明按子域组织的任务清单，附任务间关系和元操作映射提示。

**域间逻辑流**：问题发现 → 方案构思 → 发明实现 → 专利保护

---

### 1 问题发现

| ID | 任务类型 | 说明 | 依赖 | 元操作映射 |
|----|----------|------|------|----------|
| T1-1 | 需求识别 | 识别技术需求，发现技术问题和改进机会 | 无（入口） | S→C |
| T1-2 | 技术痛点 | 分析现有技术的痛点和不足，找到改进方向 | T1-1 | S→C |
| T1-3 | 创新机会 | 综合需求和痛点，识别具体的创新机会 | T1-2 | C |
| T1-4 | 技术趋势 | 调研技术发展趋势，评估创新机会的前景 | T1-3 | S→C |

## 依赖拓扑摘要

### 一、域内链路

**问题发现链路**: T1-1 → T1-2 → T1-3 → T1-4
**方案构思链路**: T2-1 → T2-2 → T2-3 → T2-4
**发明实现链路**: T3-1 → T3-2 → T3-3 → T3-4
**专利保护链路**: T4-1 → T4-2 → T4-3 → T4-4

### 二、跨域协同链路

**完整技术发明链路**: T1-1 → T1-2 → T1-3 → T1-4 → T2-1 → T2-2 → T2-3 → T2-4 → T3-1 → T3-2 → T3-3 → T3-4 → T4-1 → T4-2 → T4-3 → T4-4

**反馈改进链路**: T3-3 → T2-2（功能验证反馈到方案设计）

**专利驱动链路**: T4-1 → T1-3（专利检索反馈到创新机会）

更多组合根据具体任务动态推导。
