# 质检模块

> **路径**：modules/qa/
> **来源**：GitHub - novel-qa

## 模块说明

质检模块负责章节质量检测，确保一致性、逻辑性、可读性。

## 核心功能

| 检查类型 | 检查项 | 严重程度 |
|---------|-------|---------|
| 一致性检查 | 世界观设定一致 | 🔴 高 |
| OOC检查 | 人物性格一致性 | 🔴 高 |
| 伏笔追踪 | 伏笔埋设与回收 | 🟡 中 |
| 逻辑验证 | 情节逻辑链完整性 | 🔴 高 |
| 节奏评估 | 章节节奏是否拖沓 | 🟡 中 |
| 钩子检测 | 悬念设置有效性 | 🟡 中 |
| 风格一致性 | 与Style-DNA匹配度 | 🟢 低 |

## 输出格式

输出为 Markdown 格式的质检报告，保存在 `qa-reports/qa-chapter-{序号}.md`

```markdown
# 质检报告：第X章

## 检查结果汇总
- ✅ 通过项：N项
- ⚠️ 警告项：M项
- ❌ 问题项：K项

## 问题详情

### ❌ 问题1：人物OOC [严重]
**位置**：第3段
**问题描述**：人物A的性格与设定不符
**当前内容**：「......」
**人物设定**：A是冷静理性的性格
**建议修改**：「......」

## 优先修复清单
1. [🔴 高] 人物OOC问题
2. [🔴 高] 逻辑漏洞
3. [🟡 中] 伏笔遗漏
```
