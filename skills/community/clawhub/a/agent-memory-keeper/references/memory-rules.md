# 记忆规则

## 核心原则

### 1. 写下来才算数
- 想记住的任何东西，立刻写文件
- "我记得" ≈ "我没记"
- 文件是记忆，脑子不是

### 2. 分层存储

| 层级 | 存储位置 | 内容 | 更新频率 |
|------|---------|------|---------|
| L1 | MEMORY.md | 长期记忆、精华总结 | 每周 |
| L2 | memory/YYYY-MM-DD.md | 每日原始日志 | 每天 |
| L3 | memory/knowledge/*.md | 领域知识节点 | 按需 |
| L4 | memory/knowledge/rules/rules.md | 行为规则库 | 每次复盘 |

### 3. 信息分级

**必须记：**
- 用户名字、称呼、偏好、红线
- 项目决策和技术选型
- 错误教训和修复方案
- 重要时间节点
- 用户批评和反馈
- 新学到的概念和工具

**按需记：**
- 日常对话内容（记结论不记过程）
- 已完成任务的细节
- 临时想法

**不记：**
- 日常问候
- 一次性聊天
- 不需要复用的信息

### 4. 检索优先
- 不知道就问文件，不靠"我记得"
- 搜索命令：`grep -ri "关键词" memory/`
- 如果搜索没找到，先怀疑自己没记，不是没发生过

### 5. 定期维护
- 每天：写当日日志
- 每周：提炼精华到 MEMORY.md，清理过期信息
- 每次复盘：检查是否有知识缺口
