## 一、互联网行业知识

### 1.1 互联网基础概念

| 概念 | 解释 | 与工作经历的关联 |
|------|------|-----------------|
| B2B | Business to Business，企业间的商业模式 | 平台运营本质是B2B撮合 |
| B2C | Business to Consumer，企业对消费者的模式 | 电商系统服务商家面向C端消费者 |
| SaaS | Software as a Service，软件即服务 | 平台资源池中的产品多为SaaS模式 |
| PaaS | Platform as a Service，平台即服务 | AI中台本质是PaaS，提供模型开发能力 |
| IaaS | Infrastructure as a Service，基础设施即服务 | 云服务器等属于IaaS |
| 平台模式 | 搭建基础设施让多方在上面交易或协作 | 平台运营的核心模式 |
| 生态合作 | 企业之间通过产品、渠道、技术等多维度合作 | 伙伴关系的本质 |

### 1.3 电商行业知识

| 知识点 | 说明 | 关联经历 |
|------|------|----------|
| SKU | Stock Keeping Unit，最小库存单位，如"iPhone 15 Pro 256G 黑色"是一个SKU | 电商PM核心概念 |
| SPU | Standard Product Unit，标准产品单位，如"iPhone 15 Pro"是一个SPU | SKU的上层概念 |
| GMV | Gross Merchandise Volume，商品交易总额 | 电商核心指标 |
| 转化率 | 从浏览到下单的比例，核心衡量用户体验 | 电商系统优化方向 |
| 客单价 | 平均每笔订单的金额 | 商超业态关注指标 |
| 复购率 | 重复购买的比例，衡量用户黏性 | 生鲜业态关注指标 |
| 毛利率 | 销售毛利占销售收入的比例 | 3C卖场关注指标 |
| OMS | Order Management System，订单管理系统 | 电商系统核心模块 |
| WMS | Warehouse Management System，仓储管理系统 | 库存管理核心模块 |
| ERP | Enterprise Resource Planning，企业资源规划系统 | 商超后端系统 |
| POS | Point of Sale，销售终端系统 | 商超线下收银 |
| POS+ERP整合 | 线上线下库存同步、价格统一 | 商超数字化核心需求 |

### 2.2 AI核心概念详解

| 概念 | 通俗解释 | 技术解释 | 面试回答建议 |
|------|----------|----------|------------|
| 大模型(LLM) | 超大规模的AI模型，能理解和生成人类语言 | 基于Transformer架构，参数量达百亿级以上的语言模型 | "大模型是基于Transformer架构的超大规模语言模型，具备强大的理解和生成能力" |
| RAG | 先搜索知识库再让AI回答 | Retrieval-Augmented Generation，检索增强生成 | "RAG通过先检索企业知识库获取相关内容，再让大模型基于检索结果生成回答，解决大模型幻觉问题" |
| Agent | AI自主执行任务的智能助手 | 能感知环境、制定计划、调用工具、执行动作的AI系统 | "Agent是大模型能力的延伸，让AI能自主规划和执行复杂任务，如调用API、查询数据库等" |
| 微调(Fine-tuning) | 给AI模型"补课" | 在预训练模型基础上用特定领域数据继续训练 | "微调是在通用大模型基础上，用行业特定数据进一步训练，让模型更专业" |
| 提示词(Prompt) | 跟AI说话的方式 | 给模型的输入文本，引导模型生成特定输出 | "Prompt是与大模型交互的指令，好的Prompt能显著提升输出质量" |
| Token | AI处理文字的计量单位 | 文本被切分成的基本处理单元（约1个中文字≈1-2个token） | "Token是大模型处理文本的基本单元，影响模型输入输出长度和成本计算" |
| 向量数据库 | AI的"记忆库" | 存储文本向量化的数据库，支持语义搜索 | "向量数据库将文本转化为向量存储，支持语义级别的相似度检索，是RAG的基础" |
| 多模态 | AI同时处理文字、图片、声音 | 能同时理解和生成多种类型数据的模型 | "多模态模型能同时处理文本、图像、语音等，实现更全面的AI理解能力" |

### 2.4 AI产品经理的独特价值

| 维度 | 普通产品经理 | AI产品经理 |
|------|-------------|-----------|
| 核心挑战 | 功能设计 | 效果设计（模型准确率、召回率等） |
| 用户反馈 | 功能好不好用 | 结果准不准、体验好不好 |
| 迭代方式 | 功能优化 | 数据+算法+功能三维优化 |
| 技术理解 | 理解技术可行性 | 理解算法原理和数据依赖 |
| 成果衡量 | 功能上线率 | 模型效果指标+业务价值指标 |
| 风险类型 | 功能bug | 模型偏见、数据安全、伦理问题 |

## 三、求职者使用指引

### 3.1 遇到专业词汇怎么办

1. 先在本文件中搜索对应词汇——咱们已经整理了最常见的术语
2. 如果本文件没有，请向大模型提问："请用通俗语言解释XX术语，就像给一个刚入行的朋友解释一样"
3. 理解术语后，用自己能理解的方式重新表述，确保面试时能自然地说出来，而不是背书式地复述

### 3.2 补充工作细节怎么做

1. 先对照 career-data.md 中的结构化数据，确认自己实际经历中哪些细节缺失
2. 向大模型提问："请根据XX岗位的典型工作内容，帮我补充以下方面的细节：[具体缺失点]"
3. 大模型补充后，务必结合自己实际经历做筛选和调整——补充的是行业通用知识，不是你的真实经历

### 3.3 公司差异提醒

**重要**：各公司的管理流程、制度有差别！本文件中的流程、术语、案例是基于互联网行业通用模式整理的。你在面试时：

1. 不要把本文件的流程当作你所在公司的真实流程——要结合自己实际经历调整
2. 不同公司的伙伴审核流程、供应商审查标准、项目管理制度、AI中台架构都有差异
3. 面试时如果被追问细节，优先用自己真实经历回答，本文件作为补充参考就行
4. 如果面试官问的是其他公司的做法，坦诚说明"我所在公司的做法是XX，不同公司可能有差异"——这种坦诚反而加分 🙌

### 3.4 大模型辅助使用建议

- **简历优化**：把 career-data.md 的经历数据 + interview-guide.md 的STAR模板提供给大模型，让它帮你生成简历初稿
- **面试模拟**：把 interview-guide.md 的问题清单提供给大模型，让它扮演面试官做模拟面试
- **知识补充**：遇到不懂的行业术语或概念，向大模型提问获取解释
- **案例完善**：提供自己的大致经历框架，让大模型帮你补充合理的细节和成果描述
- **注意事项**：大模型补充的内容是行业通用知识，需结合实际经历筛选使用，不可直接照搬
