# 大数据业界洞察需求文档

## 一、信息源分类

| 类型 | 来源 | 优先级 | 说明 |
| --- | --- | --- | --- |
| 学术/研究 | 阿里/腾讯/华为数据中台白皮书、Gartner/IDC 报告 | ⭐⭐⭐⭐⭐ | 权威性强，适合战略参考 |
| 行业实践 | 头部企业技术博客（阿里、字节、美团等） | ⭐⭐⭐⭐⭐ | 实战案例，可借鉴性强 |
| 技术动态 | 开源项目（Apache Flink、Spark、DataHub 等） | ⭐⭐⭐⭐ | 技术选型参考 |
| 新闻资讯 | 36氪、虎嗅、InfoQ、掘金等技术媒体 | ⭐⭐⭐ | 时效性强，捕捉趋势 |
| 会议/峰会 | QCon、ArchSummit、数据智能大会等 | ⭐⭐⭐ | 前沿方向洞察 |

## 二、核心关注方向

### 数据架构演进
- 湖仓一体（Data Lakehouse）
- 实时数仓建设
- 数据编织（Data Fabric）
- 安卓操作系统端侧云侧环境

### 数据治理与质量
- 数据血缘与资产目录
- 数据质量监控
- 隐私计算/数据安全

### AI 与数据中台融合
- 大模型 + 数据中台
- 特征平台
- 数据驱动用户运营、营销增长

### 商业化与数据变现
- 数据产品化
- 数据服务 API
- 外部数据合作

### 荣耀/手机厂商对标
- 华为数据中台实践
- OPPO/vivo 对标分析
- 手机行业互联网业务案例

## 三、工作流程

1. **初步扫描**：快速检索近 6 个月的核心报告/文章
2. **深度筛选**：筛选高价值内容，排除过时/低质信息
3. **整理输出**：结构化整理，形成可执行的洞察
4. **定期更新**：每月全量更新，每天按需更新

## 四、输出形式

| 维度 | 内容 |
| --- | --- |
| 标题 | 清晰概括主题 |
| 来源 | 作者/机构/发布时间 |
| 核心观点 | 3-5 条关键结论 |
| 与我司关联 | 对荣耀数据中台的启示 |
| 行动建议 | 可落地的 1-2 条建议 |
| 原始链接 | 便于进一步查阅 |

## 五、注意事项

- 时间范围：关注近 6 个月内信息，重点关注最近 1 周内信息
- 输出频率：每天推送
- 深度要求：每月深度报告，每天简报
- 特殊关注：重点关注国内外头部大模型发展与应用层信息，以及安卓市场上各终端品牌的动态
