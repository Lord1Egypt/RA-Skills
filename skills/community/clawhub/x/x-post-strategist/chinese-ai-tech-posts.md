# Writing Chinese AI / Tech X Posts


## What Works In Chinese AI X Posts

Top-performing Chinese AI authors are mostly engineers, product builders, tool makers, and research-note writers — not news reporters. That shapes their posts: **conclusion-first, engineering-practical, evidence-backed (usually a screenshot), and list-structured.** Readers stop not for "今天聊聊……" ("today let's talk about…") but for a result you already got, or a sharp problem you just named for them.

Every strong post runs the same four-step motion. Practice making each step shorter and sharper:

1. **Hook (钩子)** — lead with a result, a problem, or a contrast judgement. No warm-up.
2. **Evidence (证据)** — immediately back it: a screenshot, finished product, data, comparison, command, prompt, or "I actually ran this".
3. **Why it matters (为什么重要)** — one layer of meaning for the reader.
4. **Action (动作)** — give a concrete next step: steps, a prompt, a link, "评论区见" ("see the comments"), "后续继续更新" ("I'll keep updating"), or "你怎么看" ("what do you think").

You are not "expressing an opinion" — you are helping the reader move fast through **stop → believe → repost/save**.

## Hook Types

The strongest openers are not topic announcements. Use one of these:

| Hook type | When | One-line example (ZH) |
| --- | --- | --- |
| Year/stage comparison (年份对比) | Trend calls, industry rhythm | 2024 大家在比谁更聪明；2025 大家开始比谁更能把活干完。 |
| Result payoff (结果爆点) | You actually got something working | 我真把 Claude Code + MCP 跑通了。 |
| Problem decomposition (痛点拆解) | A shared confusion you can dissect | 为什么普通用户感觉不到 MCP 的价值？ |
| Tutorial promise (教程承诺) | Prompts, workflows, checklists | 来了。我把「论文 → 图文」整理成一套可复用工作流。 |

Year/stage comparison and result-payoff travel furthest because they are inherently quotable and easy to repost.

## Structures

Pick the skeleton that fits the content type, and name it to yourself before drafting:

- **结论 → 解释 → 推论** (conclusion → explanation → larger inference). Best for trend judgement, product observation, model-release interpretation. Reader feels "you already thought it through for me."
- **痛点 → 方案 → 步骤** (pain point → solution → steps). Best for hands-on content: Claude Code, MCP, prompts, AI tool flows, visual generation. Reader feels "you can get me started right now." Switch to numbered/bulleted lists here.
- **案例/新闻 → 解读 → 立场** (case or news → interpretation → stance). Best for reacting to a paper, official blog, or someone's quote — translate it into 人话 ("plain words"), then take a position.

Tutorials lean on lists; trend posts lean on contrast sentences.

## High-Reuse Sentence Patterns

These are the most transferable moves. Learn the pattern, not the words:

- **Contrast sentence (对比句)** — carries a built-in conclusion and is easy to quote:
  - 不是……而是……（not X, but Y）
  - 2024 是……，2025 是……（in 2024 it was…, in 2025 it is…）
  - 问题不在 X，而在 Y（the problem isn't X, it's Y）
- **Teacher-style transparent reasoning (老师式推理)** — make the thinking visible so the reader keeps up: 因为 / 所以 / 这时候 / 于是 (because / so / at this point / therefore).
- **List compression (列点压缩)** — the moment you hit steps, methods, tools, pitfalls, or principles, switch to numbers or bullets.
- **Analogy (类比)** — turn an abstract concept into a scene the reader grasps instantly: MCP as "USB-C", a shared agent context as "interns sharing one computer", a system-vs-model split as "kitchen and chef", a debugging flow as "solving a case".

## Media And Hashtags

Chinese AI posts that perform do **not** rely on stacked hashtags. They rely on images that prove you actually did the work.

- **Prefer evidence images:** before/after comparison, flowchart, terminal screenshot, generated result, IDE view, finished web page. The image should prove value, not decorate.
- **Hashtags:** default to **0**. Add at most **1–2** relevant tags, and only when you genuinely want search or campaign reach. More than two usually lowers engagement.
- The strongest combination is: **strong opening line + one image that proves you did it + 0–1 necessary tag** — not "three hashtags + a row of emoji".

## Posting Timing (Heuristic — A/B Test It)

Treat these as a starting hypothesis for a Chinese developer / PM / overseas-Chinese tech audience, not a guarantee. Best window is generally **Tuesday–Thursday**, with a morning and an evening peak. Default mapping:

| Content type | Suggested window |
| --- | --- |
| News / launch / hot take | Noon (12:00–18:00) |
| Tutorial / method / prompt / long post | Evening (20:30–21:30) |
| Trend / judgement | Morning (09:00–11:00) |

Test priority for this audience: **morning → noon → evening**, before bothering with weekends. Use native scheduling to run A/B tests instead of guessing.

**Universal one-line skeleton** (especially repostable in the Chinese tech circle):

> 【结论或问题】→【一层证据】→【两到五个要点】→【我后续会更新 / 提示词在下面 / 你怎么看】
> (conclusion or problem → one layer of evidence → 2–5 key points → I'll update later / prompt below / what do you think)

## Pre-Publish Checklist

Run this in the last five minutes before posting:

| Check | Ask yourself | Action |
| --- | --- | --- |
| Hook | Does line 1 already give a result, problem, or judgement? | Cut the warm-up; turn "我最近在想" into "我发现 / 我跑通 / 为什么会这样". |
| Credibility | Did I give evidence, not just an attitude? | Add at least one: screenshot, result image, data, comparison, steps, command, source link. |
| Density | Is complex content broken into short pieces? | More than three layers of logic → switch to a list; split long sentences. |
| Sentence | Is there one quotable judgement sentence? | Deliberately write one "不是……而是……" or "2024……2025……". |
| Media | Does the image prove value, not decorate? | Prefer before/after, flowchart, finished product, terminal screenshot. |
| Tags | Am I stuffing hashtags to "look social"? | Default 0; add 1 only for real search/campaign reach. |
| CTA | What should the reader do next? | Pick one: save / repost / prompt in comments / follow-up / 你怎么看. |
| Tracking | What metric do I watch after posting? | Opinion posts → replies; tutorials → saves/reposts; then likes. |
| Timing | Is this more of a morning, noon, or evening post? | News → noon; tutorial → evening; trend → morning. |
| Threading | Can this lead into a next post? | Leave the full prompt/flow/sequel for the next post or the first reply. |

## A/B Test Variables

Optimize specific variables, not a vague "posting feel":

| Variable | Version A | Version B | Best for |
| --- | --- | --- | --- |
| Hook | Result-first: 我真把 X 跑通了 | Problem-first: 为什么 X 总是失败 | Engineering / tutorial posts |
| First evidence | Finished-product image first | Flowchart first | Prompt / UI / video / web posts |
| Tone | Calm and professional | Slightly excited, spoken | Same topic, twice |
| CTA | 你怎么看 | 提示词/完整流程在下面 | Opinion vs tutorial |
| Tags | No tag | 1 topic tag | Campaign vs normal |
| Length | 180–280 char short post | Thread / long post | Trend posts, paper breakdowns |
| Link position | Link in body | Link in first reply | Distribution-sensitive tutorials |
| Timing | Tue 09:00 | Tue 20:30 | Same topic, two versions |
| Evidence type | Data / report | First-hand experiment / script | Industry judgement vs hands-on |
| Language mix | All Chinese | Chinese-dominant + key English terms | Developer-facing posts |
