# 宪法审计 + Citation Check

> 验证引擎的最后一道关卡。独立于红队攻击，侧重证据可靠性而非逻辑自洽性。

---

## Citation Check（代码级验证）

在宪法审计之前，先对所有引用做硬门控。

### 执行流程

```
报告初稿 → 提取所有引用标注
→ 验证：外部数据→web_search交叉验证 | 内部数据→逻辑一致性检查 | URL→可达性检查
→ 全部通过 → 进入宪法审计
→ 任一失败 → Hard Reject，打回修正引用
```

### 验证规则

| 规则 | 检查内容 | FAIL处理 |
|------|---------|---------|
| **存在性** | 引用来源是否真实存在 | 删除引用，标注"来源不可验证" |
| **时效性** | 数据是否超过2年 | 标注"数据可能已过时" |
| **一致性** | 同一数据多源是否一致 | 标注"数据存在分歧，取保守值" |
| **逻辑性** | 数值是否在合理范围 | 标注"数据存疑，建议人工核实" |

Citation Check 是硬门控。任何引用未通过 → 报告不可输出。

---

## 宪法审计（七维检查）

Citation Check 通过后，执行七维审计：

### 审计维度

| 维度 | 检查内容 | FAIL标准 |
|------|---------|---------|
| **逻辑性** | 结论是否从前提逻辑推出 | 存在逻辑跳跃或自相矛盾 |
| **可执行性** | 行动是否具体到可执行 | 行动描述模糊，无法转化为任务 |
| **代价风险** | 代价是否被充分评估 | P0行动缺代价分析 |
| **战略一致性** | 行动与指导方针是否一致 | 行动在指导方针之外 |
| **数据支撑** | 核心判断是否有数据支撑 | 核心判断仅有逻辑推演无数据 |
| **二阶效应** | 是否考虑了行动的间接影响 | 未分析关键二阶效应 |
| **MECE** | 诊断是否互斥且穷尽 | 诊断维度有遗漏或重叠 |

### 审计执行

审计可委托独立审计专家（use_skill("jianwei-audit-expert")）以避免自审盲区：
- 审计专家不可用 → Chief 自行审计，但必须在报告中标注"自审，未经独立审计"

### 审计结果

| 结果 | 处理 |
|------|------|
| **PASS** | 输出最终报告 |
| **FAIL（1-2维）** | 回溯修正（最多2轮），标注修正记录 |
| **FAIL（3维以上）** | 回溯修正，修正后仍FAIL → 降级整体置信度，标注风险点 |

## 与红队攻击的区别

| | 红队攻击（判断引擎） | 宪法审计（验证引擎） |
|---|---|---|
| **攻击对象** | 逻辑自洽性 | 证据可靠性 |
| **身份** | 最尖锐的反对者 | 最严谨的审计员 |
| **通过标准** | 无致命级攻击 | 无FAIL维度 |
| **失败处理** | 修正逻辑 | 修正数据/证据 |

两套审计并行独立，结果合并。任何一方不通过 → 报告不可输出。
