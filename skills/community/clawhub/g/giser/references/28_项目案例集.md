## 案例 1：新建学校 5 分钟步行圈划定

### 背景
某城市规划局需要评估新建小学选址：确保周边居民区 5 分钟步行可达覆盖率 ≥ 85%。

### 技术路线
```
路网数据(OSM) → 构建网络数据集 → 步行服务区分析 → 叠加人口数据 → 覆盖率计算
```

### 实施步骤

**步骤 1：数据准备**
```
数据获取：
  ├── OSM 路网数据：https://www.openstreetmap.org/ → 导出研究区路网
  ├── 人口数据：第七次人口普查数据（街区级）
  ├── 居民区多边形：从基础地形图中提取
  └── 学校拟选址点

处理工具：
  ArcGIS Pro Network Analyst 扩展模块
```

**步骤 2：构建网络数据集**
```
操作路径：
  1. 创建 File Geodatabase → 导入路网
  2. 右键要素数据集 → New → Network Dataset
  3. 配置：
     - 连通性策略：端点连通（End Point）
     - 成本属性：步行时间 = 长度 / 步行速度
     - 步行速度：5 km/h（≈83.33 m/min）
     - 限制属性：步行可通行的道路（排除高速路/快速路）
  4. Build Network Dataset
```

**步骤 3：服务区分析**
```
ArcGIS Pro → Analysis → Network Analysis → Service Area

参数设置：
  ├── Facilities：学校选址点
  ├── Cutoff Values：5 (分钟)
  ├── Travel Mode：Walking Time (自定义)
  ├── Output：Polygons (非重叠面)
  └── Run

结果：以学校为中心，沿路网的 5 分钟步行可达区域面
```

**步骤 4：人口覆盖分析**
```
1. Spatial Join：将人口数据（面）连接到服务区面 → 分配人口
2. 叠加分析：Intersect(服务区面, 居民区面)
3. 面积加权人口计算：
   服务区内居住面积 / 总居住面积 × 居民区人口
4. 覆盖率 = 覆盖人口 / 总人口 × 100%
```

**步骤 5：调整与优化**
```
如果覆盖率 < 85%：
  → 分析未覆盖区域的空间分布
  → 调整选址位置（靠近人口密集但未覆盖的区域）
  → 重新计算覆盖率
  → 考虑增设人行天桥/地下通道（改善步行可达性）
```

### 关键参数
| 参数 | 值 |
|------|-----|
| 步行速度 | 5 km/h (83.33 m/min) |
| 服务时间 | 5 min |
| 小学服务半径（《城市居住区规划设计标准》）| ≤500m |
| 覆盖率目标 | ≥85% |

### 常见坑点
- 路网孤立路段 → 必须在构建网络前检查和修复
- 步行路径穿越高速路 → 除非有斑马线/天桥/地下通道
- 人口数据空间粒度不足 → 需面积加权分配

## 案例 2：建设用地适宜性评价

### 背景
某县级市需要划定未来5年城市建设用地的适宜开发区域，为国土空间规划提供依据。

### 技术路线
```
确定评价因子 → 因子分级赋值 → AHP法确定权重 → 加权叠加分析 → 适宜性分级
```

### 评价因子体系

| 因子 | 数据来源 | 分级标准 |
|------|---------|---------|
| **地形坡度** | DEM 提取 | 0-8°(5分) / 8-15°(4分) / 15-25°(3分) / 25-35°(2分) / >35°(1分) |
| **交通可达性** | 路网缓冲区 | 0-200m(5分) / 200-500m(4分) / 500-1000m(3分) / 1000-2000m(2分) / >2000m(1分) |
| **水系距离** | 水系缓冲区 | >100m(5分) / 50-100m(3分) / <50m(1分，禁建区) |
| **土地利用现状** | 国土三调数据 | 空闲地(5分) / 其他草地(4分) / 旱地(3分) / 水田(2分) / 水域/林地(1分) |
| **距城镇中心距离** | 多点距离 | 0-1000m(5分) / 1000-3000m(4分) / 3000-5000m(3分) / 5000-10000m(2分) / >10000m(1分) |

### AHP 权重计算

```
构造判断矩阵（1-9标度法）：

           坡度  交通  水系  用地  距离中心
  坡度      1    1/3   3    1/2     2
  交通      3     1    5     2      4
  水系     1/3   1/5   1    1/3    1/2
  用地      2    1/2   3     1      3
  距离中心  1/2  1/4   2    1/3     1

  归一化权重：
   坡度：0.16	交通：0.41  水系：0.06  用地：0.25  距中心：0.12
  CR(一致性比率) = 0.03 < 0.10 ✅ 通过
```

### 实施步骤

```
步骤1：数据预处理
  ├── DEM → Slope（坡度栅格）→ Reclassify（按分级标准赋值）
  ├── 路网 → Multiple Ring Buffer → Feature to Raster → Reclassify
  ├── 水系 → Buffer → Feature to Raster → Reclassify
  ├── 三调数据 → Reclassify（按现状赋值）
  └── 城镇中心 → Euclidean Distance → Reclassify

步骤2：加权叠加
  ArcGIS Pro → Spatial Analyst → Weighted Overlay
  ├── 各图层输入 → 权重填入 → 评估尺度 1-5
  └── 输出：适宜性栅格 (Cell Size: 30m)

步骤3：适宜性分级
  Reclassify:
    1-2：不适宜（生态保护区/水域/陡坡）
    2-3：基本适宜（需工程措施）
    3-4：适宜（优先开发区）
    4-5：最适宜（近期开发首选）

步骤4：约束条件叠加
  ├── 基本农田 → 一票否决
  ├── 生态红线 → 一票否决
  └── 已有建设用地 → 排除
```

### 关键工具

| 步骤 | 工具 |
|------|------|
| 坡度提取 | ArcGIS Pro: Slope (Spatial Analyst) |
| 缓冲区 | ArcGIS Pro: Buffer / Multiple Ring Buffer |
| 距离计算 | ArcGIS Pro: Euclidean Distance |
| 重分类 | ArcGIS Pro: Reclassify |
| 加权叠加 | ArcGIS Pro: Weighted Overlay |
| AHP权重计算 | Python/Excel / yaahp 软件 |

## 案例 3：GEE 土地利用变化监测

### 背景
研究 2015-2025 年某区域土地利用变化，识别城市扩张/耕地减少/林地变化。

### 技术路线
```
GEE平台 → Landsat影像 → 样本标注 → 随机森林分类 → 变化检测 → 可视化
```

### GEE 核心代码

```javascript
// 1. 定义研究区
var roi = ee.Geometry.Rectangle([116.0, 39.0, 117.0, 40.0]);

// 2. 获取影像(2015年 Landsat 8)
var image2015 = ee.ImageCollection("LANDSAT/LC08/C02/T1_L2")
  .filterBounds(roi)
  .filterDate("2015-06-01", "2015-09-30")
  .filter(ee.Filter.lt("CLOUD_COVER", 10))
  .median()
  .clip(roi);

// 3. 获取影像(2025年 Landsat 9)
var image2025 = ee.ImageCollection("LANDSAT/LC09/C02/T1_L2")
  .filterBounds(roi)
  .filterDate("2025-06-01", "2025-09-30")
  .filter(ee.Filter.lt("CLOUD_COVER", 10))
  .median()
  .clip(roi);

// 4. 训练样本（需在Map上手动创建）
// 类型：水体(0)、林地(1)、耕地(2)、建设用地(3)、裸地(4)

// 5. 随机森林分类
var bands = ["SR_B2", "SR_B3", "SR_B4", "SR_B5", "SR_B6", "SR_B7"];
var classifier = ee.Classifier.smileRandomForest(50)
  .train(training, "class", bands);

var classified2015 = image2015.select(bands).classify(classifier);
var classified2025 = image2025.select(bands).classify(classifier);

// 6. 变化检测
var change = classified2025.subtract(classified2015).neq(0);

// 7. 统计各类面积
var area2015 = ee.Image.pixelArea()
  .addBands(classified2015)
  .reduceRegion({
    reducer: ee.Reducer.sum().group(1),
    geometry: roi,
    scale: 30,
    maxPixels: 1e13
  });

// 8. 导出结果
Export.image.toDrive({
  image: classified2025,
  description: "LUCC_2025",
  scale: 30,
  region: roi
});
```

### 关键参数

| 参数 | 值 |
|------|-----|
| 影像源 | Landsat 8 (2015) + Landsat 9 (2025) |
| 分类器 | 随机森林 (n=50) |
| 分辨率 | 30m |
| 分类数 | 5 类（水/林/耕/建/裸） |
| Kappa系数目标 | ≥ 0.80 |

### 常见坑点
- GEE 需科学上网访问
- 样本标注数量不足 → 每类至少 50 个样本
- 云量过高 → 必须设置 CLOUD_COVER < 10%
- 示例中的日期需根据实际影像可用性调整

## 案例 4：连锁门店选址分析

### 背景
某连锁便利店计划在某城市新开 10 家门店，需基于人口密度、交通可达性和竞争分析确定最优位置。

### 评价因子

| 因子 | 数据来源 | 权重 |
|------|---------|------|
| 人口密度 | 人口普查数据 / 手机信令 | 0.35 |
| 交通可达性 | 公交站/地铁站密度 | 0.25 |
| 竞争强度 | 竞品门店分布 | 0.20 |
| 租金水平 | 商业地产数据 | 0.15 |
| 商圈覆盖 | 已有门店服务区 | 0.05（避免重叠） |

### 实施步骤

```
1. 人口密度分析
   人口数据 → Kernel Density → 人口热力图

2. 交通可达性
   公交站/地铁站 → Buffer(200m) → 覆盖范围图

3. 竞争分析
   竞品门店位置 → Kernel Density → 竞争热力图（反向指标）
   已有门店 → Service Area(15min驾车) → 避免重叠

4. 加权叠加
   Weighted Overlay → 选址适宜性图

5. Top 10 候选点筛选
   候选点 = 适宜性高 + 不在已有服务区内 + 满足最小间距(500m)
```

## 案例 6：地籍调查与不动产登记数据生产

### 背景
某县全面开展农村房地一体确权登记，需要完成外业测量→内业建库→不动产登记数据入库全流程。

### 数据流程

```
外业测量（RTK+全站仪）
    ↓ 数据导出(.dat)
CASS 编辑 → 宗地图/房产图
    ↓
iData 质检（拓扑检查/面积校核/编码合规）
    ↓
ArcGIS Pro → GDB数据入库（宗地/自然幢/层/户 四层体系）
    ↓
不动产登记系统对接
```

### 关键标准

| 标准 | 内容 |
|------|------|
| TD/T 1001-2012 | 地籍调查规程 |
| GB/T 37346-2019 | 不动产单元设定与代码编制规则 |
| 自然资源部《不动产登记数据库标准》 | 数据库结构 |

### 四层体系结构

```
宗地（ZD）：地块基本单元
  └── 自然幢（ZRZ）：地块上的建筑物
       └── 层（C）：建筑物的自然层
            └── 户（H）：不动产登记单元
```

## 案例 7：管线探测与三维可视化

### 背景
某城市需要将地下管线探测成果（2D CAD 图）转为三维可视化系统，用于管线管理和应急响应。

### 技术路线

```
管线探测数据 (.dwg)
    ↓ GlobalMapper 格式转换 + 属性清理
管线矢量 (.shp/.gpkg)
    ↓ ArcGIS Pro / SuperMap 拓扑检查
标准化管线数据
    ↓ SuperMap / Cesium 三维发布
管线三维可视化系统
```

### 管线分类标准

| 管线类型 | 颜色 (RGB) | 管径范围(mm) | 埋深范围(m) |
|---------|-----------|-------------|-----------|
| 给水 | 蓝 (0,0,255) | 100-2000 | 0.8-2.5 |
| 排水(污水) | 棕 (139,69,19) | 200-3000 | 1.5-6.0 |
| 排水(雨水) | 青 (0,255,255) | 300-4000 | 1.0-4.0 |
| 燃气 | 黄 (255,255,0) | 50-800 | 0.8-2.0 |
| 电力 | 红 (255,0,0) | 50-200 | 0.5-1.5 |
| 通信 | 绿 (0,128,0) | 50-100 | 0.5-1.5 |
| 热力 | 橙 (255,165,0) | 100-1200 | 1.0-3.0 |

## 案例 8：DOM+DEM 生产全流程（无人机航测）

### 背景
使用大疆无人机对 10km² 区域进行航测，生产 0.05m 分辨率 DOM 和 0.5m 格网间距 DEM。

### 工作流程

```
外业：航线规划 → 像控点布设（RTK测量）→ 无人机航飞
    ↓
内业1：Pix4Dmapper / DJI Terra
  ├── 影像导入 + POS数据
  ├── 空中三角测量（添加像控点平差）
  ├── 密集点云生成
  └── DSM → DOM 输出
    ↓
内业2：GlobalMapper / ArcGIS Pro
  ├── 点云分类（地面点滤波）
  ├── DEM 生成（格网 0.5m）
  ├── DOM 匀色 + 镶嵌
  └── 精度验证（检查点法）
```

### 精度指标

| 产品 | 指标 | 值 |
|------|------|-----|
| DOM | 地面分辨率 | 0.05m |
| DOM | 平面中误差 | ≤0.25m |
| DEM | 格网间距 | 0.5m |
| DEM | 高程中误差（平地） | ≤0.15m |
| DEM | 高程中误差（山地） | ≤0.5m |
