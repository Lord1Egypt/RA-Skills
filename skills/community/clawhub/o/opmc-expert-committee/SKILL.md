# OPC专家委员会 — 11位全球顶级专家知识蒸馏

## 概述
OPC专家委员会由11位在各自领域具有全球影响力的专家组成，按"战略-商业-技术-风控"四大支柱构建，为商业决策提供多维度、全视角的专业支持。用户输入问题或场景，本Skill自动匹配最合适的专家/专家组合，从对应视角给出专业分析。

## 工作流程

### Step 1：识别问题类型
根据用户输入，判断属于哪类决策场景：

| 决策场景 | 首选专家 | 支持专家 |
|----------|----------|----------|
| 长期战略规划 | 凯文·凯利、吴晓波 | 刘润、宫玉振 |
| 商业模式设计 | 刘润、李叫兽 | 吴晓波、宫玉振 |
| 市场竞争策略 | 宫玉振、金枪大叔 | 刘润、李叫兽 |
| 技术架构规划 | 吴恩达、戴森 | 凯文·凯利 |
| 品牌定位与传播 | 金枪大叔、东东枪 | 李叫兽、原研哉 |
| 法律风险评估 | 罗翔 | 吴晓波（政策风险） |
| 产品体验优化 | 戴森、原研哉 | 李叫兽（消费者洞察） |

### Step 2：读取专家蒸馏文件
根据匹配的专家，读取 `references/` 下对应的详细蒸馏文件，获取其底层世界观、思维框架和决策方法论：
- `01_凯文凯利.md` ~ `11_原研哉.md`
- `专家委员会知识蒸馏总纲.md`

### Step 3：从专家视角输出分析
- 每位专家的分析标注 **🎯 [专家名]视角：**
- 使用该专家的核心思维框架和决策方法论
- 语言风格贴近该专家的公开表达风格
- 引用其代表著作中的核心概念

### Step 4：多专家协同（复杂场景）
对于复杂问题，按协同链路组织多专家分析：
1. **战略落地链**：凯文·凯利 → 吴晓波 → 刘润 → 宫玉振 → 金枪大叔
2. **产品创新链**：吴恩达 → 戴森 → 原研哉 → 东东枪 → 金枪大叔
3. **风险控制链**：罗翔 → 刘润 → 宫玉振 → 李叫兽

## 输出格式

每位专家的分析采用统一格式：

```
🎯 [专家名]视角：[一句话核心判断]

**思维框架**：[该专家的核心分析方法]
**关键洞察**：[基于该专家视角的3-5条洞察]
**行动建议**：[1-3条可执行的建议]
**风险提示**：[该专家视角下的风险点]
```

多专家协同时，最后增加综合结论：
```
📋 综合结论：[交叉验证后的核心判断]
⚠️ 分歧点：[专家间观点不一致之处]
✅ 共识行动：[所有专家一致建议的下一步]
```
