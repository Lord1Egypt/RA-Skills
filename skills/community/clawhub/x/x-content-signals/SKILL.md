# X Content Signals

Use this to turn a rough idea into a stronger X post.

## Pick The Primary Signal

Choose one primary signal before rewriting:

- **Dwell**: readers pause, read, save, or think.
- **Reply**: readers naturally want to respond.
- **Repost/share**: readers look smart or useful by sharing it.
- **Profile click**: readers wonder who wrote this.
- **Follow**: readers expect more valuable posts like this.

## Rewrite Rules

- Make the first line specific.
- Replace abstractions with concrete nouns.
- Add a useful payoff, not just a tease.
- Make one clear point.
- Remove generic AI phrasing.
- Avoid rage bait unless the user explicitly wants controversy.
- Add a specific question only when replies are the target.

## Patterns

### Dwell

```text
Most people think X.
But after [specific experience], I think Y.
Here are the 3 reasons:
```

### Reply

```text
Which would you choose for [specific situation]?
A: [tradeoff]
B: [tradeoff]
Context: [constraint]
```

### Share

```text
I made a simple checklist for [painful task]:
1. ...
2. ...
3. ...
Save this before you [common mistake].
```

### Follow

```text
I'm spending the next 30 days doing [specific project].
Today's lesson: [specific insight].
Tomorrow: [next useful topic].
```

## Output

Return:

- **Signal target**: primary + secondary.
- **Final post**: ready to publish.
- **Why it works**: 2-4 bullets.
- **3 hook variants**.
- **Negative-feedback risk**: what could backfire.

## Operating Principle

Treat the public X algorithm as a signal map, not a cheat code. Optimize for posts and replies that earn useful positive actions (dwell, replies, reposts, shares, profile clicks, follows) while avoiding negative actions (not interested, mute, block, report, fast skips).
