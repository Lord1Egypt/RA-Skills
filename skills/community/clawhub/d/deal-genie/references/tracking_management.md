# 寄样管理模板 (tracking_management.md)

## 概述

本文件定义寄样管理（Step 9）的输出格式和模板。寄样是达人分销的隐形卡点——样品寄出去只是开始，后续跟进才是关键。本节将寄样全流程标准化，降低达人流失率。

---

## B. 达人跟进话术（寄样场景）

### 跟进时间线

```
寄出当天 → 签收后1天 → 签收后3天 → 约定推品日 → 签收后7天（长期未推）
    ↓           ↓           ↓           ↓               ↓
  寄出通知    收到确认    推品提醒    推品确认        追加跟进
```

### 场景一：寄出通知（发货当天）

```
时机：发货当天，快递取走后
目标：告知物流、表达期待、埋下"收到后确认"的钩子

话术结构：
1. 通知发货（1句）
2. 表达期待（1句）
3. 埋钩子（1句）← 关键：暗示收到后需要确认

示例：
"宝子，样品已经发走啦，顺丰SF123456789，预计2天到 📦
超期待你试用的感受，这个品我自己用了两周，洗感真的跟之前不一样
你收到以后跟我说一声哈，我把完整的素材包发你~
```

### 场景二：收到确认（签收后1天）

```
时机：快递显示签收后1天
目标：确认收到无误、询问第一感受、约推品时间

话术结构：
1. 确认签收（1句）
2. 询问感受（1句）
3. 约推品时间（1-2句）← 关键：把推品时间确定下来

示例：
"看到快递签收啦，样品都完好吧？ 😊
你用了吗，第一次洗的感觉怎么样？
咱们可以聊聊推品时间了，你看下最近哪天比较空，我这边配合做预热~
```

**分支处理**：
- 达人回复"用了，还不错" → 进入推品时间确认
- 达人回复"还没用" → 不催促，说"没事你先用着，用了跟我说"
- 达人不回复 → 隔2天发"提醒使用"消息

### 场景三：推品提醒（签收后3天）

```
时机：签收后3天，达人未回复推品时间
目标：轻度提醒、了解困难、提供支持

话术结构：
1. 自然切入（1句，不要"你什么时候推品"）
2. 提供素材（1-2句，降低对方的推品门槛）
3. 暗示时机（1句，给个建议时间窗口）

示例：
"对了宝子，最近三八节快到了，洗发水这个品类流量不错的哦～ 
我把素材包整理好了，里面有 B-roll 空镜素材和直播贴片，你直接用就行，不费时间
看看这周末或者下周初方便不？ 😉"
```

### 场景四：推品当天确认

```
时机：约定推品当天
目标：确认推品进行中、提供实时支持、准备转发

话术结构：
1. 确认推品（1句）
2. 提供支持（1-2句）
3. 准备转发素材（1句）

示例：
"今天是不是推我们的洗发水呀？期待！ 🎉
有什么问题随时找我，我在线可以秒回
另外你推完之后，我这边同步在社群和朋友圈转发，帮你加热"
```

### 场景五：长期未推跟进（签收后7天）

```
时机：签收后7天以上，未推品
目标：温和了解原因、提供额外帮助、再不推就撤样

话术结构：
1. 降低压迫感（1句）
2. 了解困难（1句）
3. 提供选项（2句）

示例：
"宝子，没别的意思哈，就是想问问洗发水你用了感觉怎么样？
是觉得品不太合适呢，还是最近太忙排不过来？没事你说实话就行
如果觉得品对不上你的粉丝，那我就安排取回，不耽误你时间。如果能推，你告诉我个时间，我全力配合"
```

## C. 寄样追踪表模板

### HTML 表格结构

生成一个可直接记录和更新的 HTML 表格：

```
┌────────────────────────────────────────────────────────────────────────────┐
│                          寄 样 追 踪 表                                      │
├──────┬──────────┬──────────┬──────────┬──────────┬────────────┬──────────┤
│ 达人 │ 寄出日期  │ 快递单号  │ 签收状态  │ 推品状态  │ 下次跟进    │ 备注     │
├──────┼──────────┼──────────┼──────────┼──────────┼────────────┼──────────┤
│ @A   │ 6.1      │ SF001    │ ✅ 已签收 │ 🟡 已约未推│ 6.8        │ 留意     │
│ @B   │ 6.2      │ SF002    │ ✅ 已签收 │ 🟢 已推  │ —          │ 已出12单 │
│ @C   │ 6.3      │ SF003    │ 🔴 未签收 │ ⚪ 待跟进 │ 6.5        │ 物流延迟 │
│ @D   │ 6.4      │ SF004    │ ⏳ 运输中 │ ⚪ 待跟进 │ 6.6        │          │
```

### 状态标记说明

| 状态 | 标记 | 说明 |
|------|------|------|
| 签收-已签收 | ✅ 已签收 | 达人已收到样品 |
| 签收-运输中 | ⏳ 运输中 | 快递在途 |
| 签收-未签收 | 🔴 未签收 | 快递状态异常 |
| 推品-已推 | 🟢 已推 | 达人已推品 |
| 推品-已约未推 | 🟡 已约未推 | 约了时间但还没到 |
| 推品-待跟进 | ⚪ 待跟进 | 还没有约时间 |
| 推品-确认不推 | 🔴 确认不推 | 达人明确不推 |

## 寄样效率评估

### 关键指标

| 指标 | 计算方式 | 健康值 |
|------|---------|-------|
| 签收率 | 已签收 / 已寄出 | ≥ 95% |
| 推品转化率 | 已推 / 已签收 | ≥ 60% |
| 平均推品周期 | 签收日期 → 推品日期 | ≤ 7天 |
| 样品流失率 | 确认不推 / 已寄出 | ≤ 15% |

### 异常告警

当以下情况出现时，在报告中主动提醒用户：

- 某达人签收后 7 天未联系 → 提醒跟进
- 推品转化率低于 50% → 提醒检查达人匹配是否合理
- 样品流失率高于 20% → 提醒调整寄样策略和达人筛选标准
