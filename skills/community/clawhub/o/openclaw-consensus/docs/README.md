# Docs Index

Use this folder for durable project documentation.

## Core docs
- `PRODUCT_SPEC.md` — one-page product definition
- `PRINCIPLES.md` — hard boundaries and anti-scope-creep rules
- `RUNTIME_CONTRACT.md` — MVP runtime and configuration assumptions
- `ARTIFACTS.md` — run directory and artifact shape
- `PROMPT_CONTRACT.md` — round-1 / round-2 / orchestrator prompt rules
- `OUTPUT_TEMPLATE.md` — final synthesis output structure
- `COMMANDS.md` — only the real command surface
- `AI_BOOTSTRAP.md` — future AI/operator launch guidance

## Repo-shaping docs
- `START_HERE.md` — current repo alignment checklist
- `SKILL_LIFECYCLE.md` — local-dev → public-repo → skill-hub progression rules
- `RUNBOOK.md` — repo-specific implementation order and release-prep notes

## Rule
Prefer a few sharp docs over a bloated doc pile.
If a doc does not help build, validate, or publish the skill, it should justify its existence.
