## T01 冒充/身份错位

**适用类型**：硬核推理、变格、规则型
**核心逻辑**：利用"身份不确定性"制造悬念，让玩家始终怀疑"这个人到底是谁"。

### 设计步骤

```
第一步：设定"冒充动机"
  → 冒充者为什么要冒充这个人？（逃债/复仇/保护某人/完成任务）

第二步：设定"真人去向"
  → 被冒充的人现在在哪里？（已死/被囚禁/主动隐藏/根本不存在）

第三步：设计"露出马脚"的线索
  → 至少 2 条线索让玩家怀疑此人的身份
  → 至少 1 条线索可以"两种解释"（表面是冒名，实际是其他原因）

第四步：设计"配合机制"
  → 如果冒充者需要和其他角色"演对手戏"
  → 确保即使双方不熟，也能通过"角色设定"合理配合
```

### 可套用模板

```
【冒充者设定模板】
真实身份：______
冒充对象：______
冒充动机：______
真人现状：______

【线索设计模板】
线索 A（引发怀疑）：______
线索 B（加深怀疑）：______
线索 C（澄清/反转）：______

【对话配合说明】
冒充者与其他角色的对话，按照"冒充对象的性格"表演，
不需要真正认识对方，只需按照角色卡设定的"公开背景"回应。
```

### 检查清单

- [ ] 冒充动机是否充分？（不能"为了剧情而冒充"）
- [ ] 被冒充者的去向是否有合理解释？
- [ ] 是否有至少 2 条线索让玩家怀疑身份？
- [ ] 冒充者和真正认识冒充对象的人，对话是否有"细微违和感"的线索？

## T02 烟雾弹与全员嫌疑

**适用类型**：硬核推理、本格
**核心逻辑**：每个人都可能有嫌疑，但每个嫌疑都有"可被澄清"的路径，不是强行诬陷。

### 线索比例规范

| 线索类型 | 占比 | 说明 |
|---------|------|------|
| 真实线索（指向真凶，无可辩驳） | 35% | 最终推导链必须用到 |
| 干扰线索（指向好人，但有合理解释） | 35% | 每个被怀疑的人都有澄清方式 |
| 误导线索（表面指向 A，实际指向 B） | 15% | 需要"换个角度"才能看懂 |
| 死胡同线索（有趣但不指向任何人） | 15% | 用来消耗玩家时间，增加沉浸感 |

### 干扰线索设计规范

每条干扰线索必须同时提供：
```
线索描述：______（玩家看到的内容）
表面指向：______（玩家第一反应会怀疑谁）
澄清路径：______（通过哪条线索/对话可以澄清）
澄清后的真实含义：______（这条线索实际说明什么）
```

### 检查清单

- [ ] 每个非凶手角色，是否都有"被怀疑的合理理由"？
- [ ] 每个被怀疑的角色，是否都有"澄清路径"？
- [ ] 最终投票时，是否有至少 2-3 人被认为"有可能"？
- [ ] 有没有"完全无法解释的线索"？（如有，删除或重写）

## T03 关系网三层次设计

**适用类型**：全类型通用（尤其适合情感本、硬核本）
**核心逻辑**（参考东君书院）：不是"关系多"，而是"关系有层次，且层次之间互相印证"。

### 三层次结构模板

```
第一层（公开层）：所有玩家一开始就知道的关系
  格式：A 与 B 是______关系（所有人都知道）
  示例：A 和 B 是夫妻，B 和 C 是同事

第二层（半隐藏层）：需要线索才能发现的关系
  格式：A 与 C 实际是______关系，线索______可以证明
  示例：A 和 C 有私情，线索"情书"可以证明

第三层（深层隐秘）：只有特定角色知道的秘密关系
  格式：只有______知道______和______之间的______
  示例：只有 B 知道 A 和 C 的私情，因为 B 一直在跟踪 A
```

### 关系网交叉验证要求

- 每个角色至少参与 **2 条**关系链
- 关系链之间至少要有 **1 处交集**（不能是独立的故事线）
- 最终真相必须能**同时解释三个层次**

### 检查清单

- [ ] 每个角色是否至少出现在 2 条关系链中？
- [ ] 关系链之间是否有交集？
- [ ] 最终真相是否能同时解释三个层次？
- [ ] 有没有"只为制造悬念而存在"的关系？（如有，删除或赋予实际功能）

## T04 反转与"假平息"节拍

**适用类型**：硬核推理、变格、规则型
**核心逻辑**（参考诡秘之主）：反转不是"突然冒出来"，而是"回头看发现所有线索都有，只是被误解了"。

### 三幕节奏模板（含假平息）

```
第一幕（30%）：建立场景，发现事件，介绍角色
  → 结束点：玩家开始互相怀疑

第二幕（50%）：搜证，推理，指向某个嫌疑人
  → 假平息节点：出现了"决定性证据"指向某人 X
  → 玩家以为：找到了，就是 X 干的
  → 主持人动作：让大家投票（非正式），营造"已经解决"的氛围

第三幕（20%）：新线索出现 → 推翻之前的结论 → 真正真相揭晓
  → 反转触发条件：某条被忽略的线索被重新解读
  → 最终真相：解释所有线索，包括之前"以为是指向 X"的线索
```

### 反转质量检查清单

- [ ] 反转所需的所有信息，是否已经在剧本/线索中给出？
- [ ] 有没有"为了反转而临时加设定"？（如有，删除）
- [ ] 反转后，玩家是觉得"原来如此"还是"被耍了"？
- [ ] 反转是否让前面的推理过程仍然有意义（不是全盘否定）？

### "假平息"设计模板

```
假平息触发线索：______
玩家会误以为：______
实际真相是：______
推翻假平息的新线索：______（可以在第三轮发放，也可以由某个角色的对白触发）
```

## T05 多重胜利条件

**适用类型**：全类型，尤其适合阵营本、情感本
**核心逻辑**：不是"只有找出真凶才算赢"，而是"每个角色都有自己的胜利条件，有些是隐藏的"。

### 胜利条件设计模板

```
【角色：______】

公开胜利条件（所有玩家都知道）：
  ______（通常是"找出真凶"或"让自己的阵营获胜"）

隐藏胜利条件1：
  触发条件：______
  胜利判定：______
  奖励：______（额外剧情揭露/特殊结局）

隐藏胜利条件2（可选，失败了也有剧情）：
  触发条件：______（例如：自己被投票出局，但成功把嫌疑引给某人）
  胜利判定：即使出局，只要满足条件也算胜利
  特殊说明：此条件玩家自己可能不知道，结案时由主持人揭晓

失败条件（触发后进入 Bad End）：
  触发条件：______
  后果：______（提前出局/秘密曝光/目标失败）
```

### "明面上的胜者"设计

```
本局明面上的胜者：______（可以是凶手，也可以是无辜者）
胜者判定条件：
  条件1：______（如：成功让真凶被误判）
  条件2：______（如：自己的秘密没有被曝光）
  条件3：______（如：达成了隐藏目标）

如果胜者条件无法满足：
  → 进入隐藏结局：______（所有人都没有完全胜利，真相更加复杂）
```

### 检查清单

- [ ] 每个角色是否至少有一个"隐藏胜利条件"？
- [ ] 隐藏胜利条件是否与角色的秘密/动机相关联？
- [ ] "明面上的胜者"是否有明确的判定条件？
- [ ] 如果胜者条件无法满足，是否有后备结局？

## T06 角色秘密与甩锅机制

**适用类型**：硬核推理、情感本
**核心逻辑**：A 有一个不能说的秘密，A 把秘密甩锅给 B；B 因为某种原因（不认识 A / 是替身 / 故意配合）背了锅。

### 甩锅设计模板

```
【甩锅者 A】
不能说的秘密：______
甩锅对象 B：______（A 与 B 的关系：熟/不熟/从未见面）
甩锅方式：______（通过线索/对白/行为让其他人以为是 B 干的）

【背锅者 B】
为什么 B 会背锅：______
B 是否知道自己在背锅：______（知道/不知道/半知道）
如果 B 知道，为什么要配合：______（被威胁/有交易/有更深的目的）

【真相揭晓方式】
线索______可以让玩家发现"A 才是真凶，B 是被利用的"
```

### 适用场景变体

| 场景 | A 与 B 关系 | B 的配合方式 |
|------|-------------|-------------|
| B 是替身/冒充者 | 不熟 | B 按照"角色设定"回应，不知道自己在背锅 |
| B 与 A 有交易 | 熟但有利益关系 | B 知道但选择配合，因为有把柄在 A 手里 |
| B 已死/消失 | 无需配合 | A 伪造 B 的"存在证据" |

### 检查清单

- [ ] A 的甩锅是否有"至少 2 条线索"支撑（让玩家怀疑 B）？
- [ ] 是否有"至少 1 条线索"可以让玩家发现真相（A 才是真的）？
- [ ] B 的配合是否有合理的动机（不是莫名其妙配合）？
- [ ] 真相揭晓后，玩家是否会觉得"原来 B 是被利用的"？

## T07 时间线交叉验证

**适用类型**：硬核推理、本格（强制要求）
**核心逻辑**：所有角色的时间线必须能交叉验证；如果有矛盾，必须是"可解释的"。

### 时间线设计规范

```
案发时间窗口：______（精确到小时或半小时）

每个角色在窗口内的行动：
  角色 A：
    - __:__ - __:__：在______（有人证/物证：______）
    - __:__ - __:__：在______（有人证/物证：______）
    - 无法证明的时间段：______（这是嫌疑来源）

交叉验证要求：
  - 同一时间段内，同一角色不能出现在两个地方
  - 每个"不在场证明"必须有验证方式（人证/物证/监控）
  - 案发时间窗口内，至少有 3 个人有独立视角
```

### 矛盾的可解释性模板

```
如果某条时间线有矛盾：
  表面矛盾：______
  真实情况：______
  验证方式：______（哪条线索可以验证真实情况）
  为什么会出现矛盾：______（某人撒谎/记错了/看错了）
```

### 检查清单

- [ ] 同一时间段内，同一角色是否出现在两个地方？（如有，必须有解释）
- [ ] 所有不在场证明是否有验证方式？
- [ ] 案发时间窗口内，是否有至少 3 个人的独立视角？
- [ ] 如果有时间线矛盾，是否有"可解释"的路径？

## T08 线索可解释性（双层次含义）

**适用类型**：全类型通用
**核心逻辑**：每条重要线索至少要有两种解释——表面解释和深层解释。

### 线索设计模板（双层次）

```
【线索编号：______】

线索描述（玩家看到的内容）：
  ______

表面解释（玩家第一眼的理解）：
  → 指向：______（某人或某结论）
  → 理由：______

深层解释（结合隐藏信息后的真实含义）：
  → 实际指向：______（真相）
  → 为什么会有表面解释：______（某人故意布置/巧合/误解）

验证方式（如何让玩家区分两种解释）：
  → 需要线索：______
  → 需要对话：______（问某个人某个问题）
```

### 线索发放节奏规范

| 轮次 | 目的 | 线索类型配比 | 关键动作 |
|-------|------|------------|---------|
| 第一轮（公开） | 建立场景认知，让大家"入戏" | 100% 场景线索（无关键线索） | 角色互相介绍，建立第一印象 |
| 第二轮（半公开） | 推进推理，开始互怀疑 | 40% 指向凶手 / 40% 干扰 / 20% 关系揭露 | 放 1 条"可以被两种解释"的线索 |
| 第三轮（关键） | 决定性线索，引向真相 | 60% 真实线索 / 40% 反转铺垫 | 伏笔回收："回头看才发现早就有了" |

### 检查清单

- [ ] 每条重要线索是否至少有"两种解释"？
- [ ] 第三轮是否有"伏笔回收"的线索（之前出现过但被误解的）？
- [ ] 有没有"只能有一种解释"的线索？（如有，考虑增加可解释性）
- [ ] 线索发放节奏是否和本剧本的节奏设计（T04）匹配？

## T09 唯一真相推导链

**适用类型**：硬核推理、本格（强制要求）
**核心逻辑**：玩家必须能通过逻辑推导"唯一真相"，不能"多种解释都成立"。

### 推导链设计模板

```
【真相：______（一句话）】

推导步骤：
  步骤1：线索______ → 缩小范围到______人
  步骤2：线索______ → 排除其中______人
  步骤3：线索______ → 排除剩余人中的______人
  步骤4：线索______（终极线索）→ 确认______人，且能解释所有疑点

反向验证（必须成立）：
  → 如果凶手是其他人，线索______就无法解释
  → 如果作案时间是其他时间，线索______就无法解释
  → 如果作案手法是其他方式，线索______就无法解释
```

### 防止"多种解释都成立"的检查

- [ ] 如果凶手是 A，整个推导链是否自洽？
- [ ] 如果凶手是 B（其他嫌疑人），推导链是否有断裂？（必须有断裂，否则真相不唯一）
- [ ] 有没有"既可以是 A 也可以是一篇 B"的线索？（如有，增加排他性线索）

## T10 情感递进三幕节奏（情感本专用）

**适用类型**：情感本、古风情感本
**核心逻辑**（参考东君书院）：情感不是"告诉玩家的"，而是"让玩家通过三幕逐步体验到的"。

### 三幕节奏模板

```
第一幕：欢乐/轻松（建立好感）
  目的：让玩家喜欢上自己的角色，喜欢上其他角色
  时长占比：30%
  关键设计：
    - 小剧场演绎：欢乐的对话，建立角色间的"美好关系"
    - 角色卡公开部分：展示角色的可爱/正直/令人同情的一面
  结束节点：事件爆发，美好被打破

第二幕：悬念/撕裂（秘密暴露，关系出现裂痕）
  目的：让玩家开始怀疑"我信任的人到底是谁"
  时长占比：50%
  关键设计：
    - 私密部分逐步揭露：原来 TA 有这个秘密
    - 线索卡：发现"原来我以为的好人其实..."
  结束节点：某个角色的秘密完全暴露，关系破裂

第三幕：爆发/释怀（真相揭晓，情感升华）
  目的：让玩家理解"为什么 TA 会这样做"，产生共鸣
  时长占比：20%
  关键设计：
    - 真相揭秘脚本：不是" TA 是坏人"，而是"TA 也是受害者"
    - 情感高潮：某段回忆/某封信/某个物件，让一切都说得通
```

### 检查清单

- [ ] 第一幕是否让玩家对角色产生了"好感"或"同情"？
- [ ] 第二幕的"撕裂"是否有足够的铺垫（不是突然翻脸）？
- [ ] 第三幕的"释怀"是否让玩家理解了所有人的动机？
- [ ] 有没有"为了催泪而强行写悲情"的段落？（如有，删除或重写）

## T11 小剧场演绎设计

**适用类型**：情感本、古风本、欢乐本
**核心逻辑**：小剧场不是"念台词"，而是"通过对话传递潜台词，让玩家自己体会"。

### 小剧场设计模板

```
【小剧场名称：______】
参与角色：______（2-3 人）
发生时间：______（案发前/案发时/回忆中）

对话设计原则：
  - 每句台词都有"表面意思"和"潜台词"
  - 不能直说秘密，但要让观众（其他玩家）感觉到"有事情"
  - 给每个演员一个"不能说的秘密"，增加表演张力

示例格式：
  角色A："______"（表面：______；潜台词：______；秘密：______）
  角色B："______"（表面：______；潜台词：______；秘密：______）

结束后：
  - 给玩家"私下交流时间"（5-10 分钟）
  - 主持人观察：玩家是否捕捉到了潜台词？
  - 如果没有，通过线索卡在下一轮"暗示"小剧场的真相
```

### 检查清单

- [ ] 每句台词是否都有"潜台词"（不能直白说完）？
- [ ] 每个参与小剧场的角色，是否有"不能说的秘密"？
- [ ] 小剧场结束后，是否有"让玩家消化信息"的时间？
- [ ] 小剧场的真相，是否在后续线索卡中被"回收"？

## T12 规则型剧本的规则设计（变格/规则型专用）

**适用类型**：变格、规则型、恐怖悬疑
**核心逻辑**（参考神秘复苏）：规则不是"游戏机制"，而是"世界观的一部分，且必须精确可验证"。

### 规则设计模板

```
【规则______】

规则描述（对所有玩家公开，或分阶段公开）：
  ______

违反后果（精确，不模糊）：
  ______

验证方式（如何确认有人违反了规则）：
  ______

规则来源（为什么会有这个规则，和剧情的关系）：
  ______

与真相的关系（这条规则如何帮助/干扰推理）：
  ______
```

### 规则与剧情融合检查清单

- [ ] 规则本身是否包含世界观信息（不只是"不能做 XX"）？
- [ ] 违反规则的后果，是否推动剧情发展？
- [ ] 最终真相是否解释了"规则的来源"？
- [ ] 有没有"为了制造恐怖而设定模糊规则"？（如有，改为精确规则）

## T13 明暗双线设计原则

**适用类型**：情感本、硬核推理（反转本）、变格
**核心逻辑**：逻辑第一，明暗双线是手段不是目的。先确保明线自洽，再决定是否需要暗线。

### 什么时候需要明暗双线

| 剧本类型 | 是否需要暗线 | 暗线的作用 |
|---------|------------|---------|
| 情感本 | **需要** | 明线是"发生了什么"，暗线是"为什么会发生"（情感动机）→ 双线交汇时爆发 |
| 硬核反转本 | **需要** | 明线是"大家看到的真相"，暗线是"实际真相"→ 反转靠暗线支撑 |
| 变格/规则型 | **需要** | 明线是"表面规则"，暗线是"规则背后的真相" |
| 纯本格推理 | **不需要** | 明线本身完整（谁→怎么→为什么），加暗线反而增加漏洞 |
| 欢乐本 | **不需要** | 逻辑简单清晰，双线增加理解负担 |

### 设计步骤

```
第一步：先设计明线（逻辑必须自洽）
  → 玩家通过明线推导出的结论，至少 70% 正确
  → 即使没有暗线，明线本身也是完整的、可成立的
  → 这一步先完成，不要跳过

第二步：判断是否需要暗线
  → 问自己三个问题：
    1. 加了暗线之后，明线的逻辑还成立吗？（必须成立）
    2. 暗线的存在，有没有在明线里埋伏笔？（必须有）
    3. 玩家能不能通过推理发现暗线？（最好能推理，不是只能靠揭秘）
  → 三个问题有任何一条答"No"，先不加暗线

第三步：设计暗线（如果需要）
  → 暗线不能和明线矛盾，只能"补充"或"修正"
  → 暗线的伏笔必须在明线里出现过（回头看能找到）
  → 暗线揭秘方式：线索验证 / 角色对白触发 / 主持人引导
```

### 明暗双线模板

```
【明线设计】

明线故事（所有玩家都能看到的叙事）：
  ______

明线逻辑自洽检查：
  → 如果没有暗线，明线能独立成立吗？______（是/否）
  → 明线推导出的结论，有多少比例是正确的？______%

【暗线设计】

暗线内容（隐藏在明线之下的深层叙事）：
  ______

暗线与明线的关系：
  □ 补充（暗线解释明线中没说清楚的部分）
  □ 修正（暗线揭示明线中表面合理但实际错误的部分）
  □ 深层动机（暗线揭示某个角色行为的深层原因）

暗线伏笔（出现在明线中的线索/对话/行为）：
  伏笔1：______（在第______轮/______角色剧本中出现）
  伏笔2：______（在第______轮/______角色剧本中出现）

暗线揭秘方式：
  方式：______（线索验证/对白触发/主持人揭秘）
  触发条件：______
```

### 明暗线关系类型

| 关系类型 | 说明 | 示例 |
|---------|------|------|
| 补充型 | 暗线补充明线没说清楚的部分 | 明线：A 杀了 B。暗线：A 为什么杀 B（深层动机）。|
| 修正型 | 暗线揭示明线中"看起来合理但实际错误"的部分 | 明线：大家都以为 C 当时在客厅。暗线：C 实际上中途离开了。|
| 深层动机型 | 暗线揭示某个角色的深层行为原因 | 明线：D 看起来很可疑。暗线：D 是在保护某个人。|

### 反例（逻辑有问题的暗线）

```
明线：A 在厨房，B 在客厅，C 在卧室
暗线（揭秘时）：其实 B 中途去了厨房，但所有人都没注意到
问题：如果明线里没有任何"可能没注意到 B 离开"的伏笔，
      暗线就是硬加的——逻辑不通

明线：B 说自己在客厅（有证人 D）
暗线：D 其实是 B 的同伙，作伪证
问题改正：线索卡里有"D 欠 B 一大笔钱"
→ 玩家可以提前怀疑 D 的证词，暗线有伏笔支撑
```

### 检查清单

- [ ] 明线本身是否自洽？（即使没有暗线也成立）
- [ ] 暗线是否和明线矛盾？（如有矛盾，必须修复明线或删除暗线）
- [ ] 暗线的每个伏笔是否都出现在明线中？
- [ ] 玩家能否通过推理发现暗线，而不是只能靠主持人揭秘？
- [ ] 暗线揭秘后，回头看是否"原来如此"而不是"硬加的"？

## T14 恐怖氛围营造（变格/恐怖悬疑专用）

**适用类型**：变格、恐怖悬疑、规则型
**核心逻辑**：恐怖不是"吓人"，而是"制造持续的不安感"。用细节和环境说话，不靠jump scare。

### 氛围三层次设计

```
第一层：感知层（开场0-30min）
  → 用感官描写建立"不对劲"的感觉
  → 典型手法：某个物品位置不对 / 某种声音 / 某种气味 / 某种光线变化
  → 关键：玩家第一反应是"我可能看错了"，不是"有鬼"
  → 示例："桌上的水杯比睡前少了一个。没有人动过它。"

第二层：确认层（中段30-80min）
  → 异常现象再次出现，频率加快
  → 玩家开始"确认"这不是错觉
  → DM此时可以主动渲染，但必须保持"可解释"的余地
  → 示例："你又看到了那个影子。它这次没有消失。"

第三层：揭示层（终局80min+）
  → 规则完整揭示，恐怖来源有了解释
  → 恐怖感从"未知"转化为"已知但无法接受"
  → 关键：揭示后玩家感到"终于说清楚了"，而不是"还是一头雾水"
```

### 恐怖节点分布规范

```
每个恐怖节点必须包含以下要素：
  1. 触发条件（什么情况下发生）：______
  2. 表现形式（玩家看到/听到什么）：______
  3. 持续时间（不超过10分钟）：______
  4. 后续喘息（之后至少10分钟无恐怖事件）：______
  5. 与主线的关系（这条恐怖事件是干扰还是线索）：______

⚠️ 禁止：
  - 连续两个恐怖节点无喘息
  - 恐怖事件与主线完全无关（纯粹的惊吓只有负面效果）
  - 恐怖节点让玩家无法继续推理（必须保持推理能力）
```

### 检查清单

- [ ] 恐怖节点之间是否有至少10分钟的喘息时间？
- [ ] 恐怖现象是否都有"可解释"的层面（不是纯粹的灵异）？
- [ ] 最终揭示是否让玩家感到"合理"而不是"被耍了"？
- [ ] 恐怖感是否能让玩家保持推理能力？（不是吓到无法思考）
- [ ] 有没有至少1个恐怖节点让玩家"更想推理"而不是"想逃跑"？

## T15 渐进式规则信息释放（变格/规则型专用）

**适用类型**：变格、规则型、诡秘之主风格
**核心逻辑**：规则不是"一次性告知"，而是"玩家通过收集碎片自己拼出来"。DM控制释出节奏，新手也能跟上。

### 三轮释出节奏设计

```
【第一轮（0-30min）：异常感知阶段】

DM行为：
  → 只说："有些事情……不太对劲。"
  → 发1条含"异常描述"的线索，不解释原因
  → 不主动问玩家"你们觉得规则是什么"

玩家感受到：
  → 有异常，但不知道为什么
  → 开始互相问"你看到了吗"
  → 推理方向：发生了什么？

典型线索示例：
  - "镜子里的裂纹比昨晚多了一道。没有人照过镜子。"
  - "死者手腕上有勒痕，但没有绳索。"

【第二轮（30-80min）：规则拼凑阶段】

DM行为：
  → 主动提及第一轮的异常："有人注意到[现象]吗？"
  → 发第2条线索，这次含"异常+后果"
  → 允许玩家讨论"这是怎么回事"

玩家感受到：
  → 异常开始重复出现，可以总结规律了
  → 推理方向：为什么会发生？和谁有关？

典型线索示例：
  - "第二面镜子也裂了。时间是凌晨三点整。"
  - "死者的勒痕位置，和另一个人手腕上的旧伤疤形状吻合。"

【第三轮（80min+）：规则揭示阶段】

DM行为：
  → 完整规则或核心规则揭示（通过线索卡或DM口述）
  → "现在你们知道了规则。问题是——谁利用了它？"
  → 进入最终推理阶段

玩家感受到：
  → 恍然大悟，之前所有碎片拼成完整图景
  → 推理方向：凶手是如何利用规则的？

典型线索示例：
  - "《镜界录》记载：'午夜独照，镜中现彼。' 触碰镜中人者，将受其伤。"
  - 死者是唯一在午夜独自照镜的人。"
```

### 渐进释出的常见错误

```
❌ 第一轮就全部告诉玩家规则
  → 问题：失去了"拼凑"的乐趣，变成纯推理
  → 修正：把规则拆成至少3个碎片，每轮发一个

❌ 规则释出顺序混乱
  → 问题：玩家拿到第二条线索时还不理解第一条说的是什么
  → 修正：确保第二轮线索是"第一轮的延续"，不是"新的独立线索"

❌ 规则揭示后玩家还是无法推理
  → 问题：规则揭示了，但"凶手如何利用"的路径没有给
  → 修正：规则卡+1条"凶手使用记录"的线索，让玩家能代入推理

❌ 玩家自己总结了规则但DM强行"不对"
  → 问题：破坏玩家的成就感
  → 修正：玩家总结对就给肯定，继续推进；如果总结有漏洞，给下一条线索让他们修正
```

### 检查清单

- [ ] 规则是否被拆成至少3个碎片，分三轮释出？
- [ ] 玩家在第二轮是否已有"规则雏形"的猜测机会？
- [ ] 最终揭示是否让玩家感到"原来如此"而不是"信息过载"？
- [ ] 规则揭示后是否有"凶手如何利用"的推理路径？
- [ ] DM是否有明确的释出节点控制（不是随机释出）？

## T16 搜证文件格式规范（主持人手册/线索卡专用）

**适用类型**：全类型通用
**核心逻辑**：搜证文件是玩家的"第二记忆"。格式好不好，直接影响玩家能不能记住线索、能不能推理。

### 地点分组原则

```
【格式】按地点分组，每组下列出该地点所有线索

地点A
  【线索名1】内容描述（简短，一句话）
  【线索名2】内容描述（简短，一句话）

地点B
  【线索名1】内容描述（简短）
```

### 记忆点命名原则

```
每条线索用【】包裹关键词，关键词=最容易被记住的特征

推荐格式：
  【Rush喷雾】男同性恋间常用辅助药物...（关键词：功能/名字）
  【录音笔】里面有甄机长的坦白录音...（关键词：物品种类）
  【烟头】非常高档的黄鹤楼2916...（关键词：品牌/异常特征）

不推荐：
  【物证1】不知道什么东西的喷雾
  【可疑物品】一个录音的东西
```

### 线索分层标注（让主持人知道哪些该主动说）

```
【线索名】描述内容
  → 表面指向：[玩家第一反应会怀疑谁/什么事]
  → 真实含义：[更深层的解释，只有推理后才能理解]
  → 干扰指数：★☆☆☆ / ★★☆☆ / ★★★☆
  → DM释出时机：[第一轮/第二轮/第三轮]
```

### 检查清单

- [ ] 线索是否按地点分组，结构清晰？
- [ ] 每条线索的【关键词】是否易记（不超过8个字）？
- [ ] 是否有"明显异常"的线索（让玩家第一眼就注意到有问题）？
- [ ] 是否有"表面正常"的线索（仔细看才发现有问题）？
- [ ] 干扰性线索和关键线索的比例是否均衡（约1:1）？

## T17 角色前史三层次（角色卡专用）

**适用类型**：全类型通用
**核心逻辑**：一个角色不是"一个秘密"而是"一条因果链"。前史→现状→行为线，三层逻辑自洽，玩家才能信服角色的每个选择。

### 三层次结构

```
【前史层】（玩家读不到，但决定一切）
  角色在"今天"之前的核心经历
  → 前史决定：这个人的性格从哪里来？为什么会有这个弱点？

  模板：
  [童年/成长经历] → 形成了 [性格特征]
  [某段关键经历] → 埋下了 [今日行为的种子]

【现状层】（角色公开部分可见）
  玩家通过角色卡公开部分看到的信息
  → 现状决定：这个人在"今天"是什么立场？在场是为了什么？

  模板：
  表面身份：[职业/社会角色]
  公开关系：[与死者/其他角色的表面关系]
  公开目标：[玩家自我介绍时说的话]

【行为线层】（角色私密部分设计）
  角色在剧本流程中的行动轨迹
  → 行为线决定：这个人在案发时间段实际做了什么？

  模板：
  案发前：做了 [某事]，为什么做？
  案发时：[在做什么]，[看到了什么]
  案发后：做了 [某事]，[为什么要做]

三层因果检验：
  前史的 [某特质] → 导致现状的 [某行为/选择]
  现状的 [某立场] → 导致行为线的 [某动作]
  每一步都要能回答："为什么他会这样做？"
```

### 前史设计常见错误

```
❌ 前史和现状没有因果关系
  例：前史写"父母早亡"，现状写"性格开朗，朋友很多"
  修正：前史"父母早亡"应该体现在现状中，比如"对人保持距离，害怕失去"

❌ 现状和行為线矛盾
  例：现状写"与死者关系和睦"，行为线写"一直在监视死者"
  修正：要么改现状（关系其实不好），要么改行为线（监视另有目的）

❌ 所有人都是"受害者"前史
  修正：至少一个角色有"自作自受"的前史，增加道德模糊感
```

### 检查清单

- [ ] 每个角色的前史是否都有"事出有因"（不是凭空出现）？
- [ ] 前史是否至少影响现状的1个行为/选择？
- [ ] 现状是否至少对应行为线的1个具体动作？
- [ ] 有没有角色的前史只存在于"为了制造悬念"，而对现状/行为无影响？（删除或赋予功能）
- [ ] 凶手的前史是否有"令人同情但不可原谅"的部分？（情感共鸣关键）

## T18 动态搜证模式设计（参考《大侦探》）

**适用类型**：硬核推理、变格、规则型
**核心逻辑**（参考《大侦探》动态搜证模式）：不是固定搜证顺序，而是"玩家主动探索 → 线索随行为动态激活"，让玩家像真实侦探一样挖掘真相。

### 动态搜证三要素

```
要素一：开放式场景探索
  → 玩家可自主选择探索场景（不按固定顺序）
  → 每个场景包含：基础线索（必看）+ 隐藏线索（需触发条件）
  → 隐藏线索触发条件示例：
    - 优先调查嫌疑人书房 → 触发"隐藏动机信件"
    - 转向案发现场 → 触发"血迹形态分析工具"

要素二：线索动态激活
  → 线索不是"放在那里等发现"，而是"随玩家行为激活"
  → 激活方式：
    - 行为触发：玩家做出某个动作 → 解锁新线索
    - 对话触发：玩家问了某个人某个问题 → 解锁新线索
    - 时间触发：搜证进行到某分钟 → 自动解锁新线索
  → 关键：未发现的线索可能导致推理链断裂（真实感）

要素三：线索投屏共享
  → 发现线索后一键投影至公共屏幕（所有玩家可见）
  → 避免传阅混乱，加速集中推理
  → 隐藏线索的发现：只有触发者自己先看到，然后选择是否投影共享
```

### 动态搜证流程设计模板

```
【搜证阶段：0-60min（可调整）】

第一轮（自主探索，20min）：
  每个玩家选择 1 个场景深入调查
  主持人记录：谁去了哪里，发现了什么
  隐藏线索触发：如果玩家行为符合触发条件，秘密给该玩家一张"隐藏线索卡"

第二轮（信息拼合，20min）：
  玩家自由选择：继续深入 / 交换场景 / 追问某角色
  新线索激活：基于第一轮的发现，解锁"关联线索"
  主持人动作：提醒"你们现在掌握的信息有这些..."（帮助拼合）

第三轮（关键线索，20min）：
  所有玩家集中讨论后，投票决定"最后再查哪里"
  主持人根据投票结果，给出"终极线索"（指向真相的关键证据）
  如果投票分散：DM 可给出"线索提示"（不是直接给答案）
```

### 与传统固定搜证的区别

| 维度 | 传统固定搜证 | 动态搜证模式 |
|------|-------------|-------------|
| 搜证顺序 | 固定时段、固定区域 | 玩家自主选择场景 |
| 线索获取 | 放在那里等发现 | 随玩家行为动态激活 |
| 玩家体验 | "按剧本走流程" | "像真实侦探一样挖掘" |
| DM 工作量 | 低（按流程走） | 中高（需要记录玩家行为并触发对应线索） |

### 检查清单

- [ ] 每个场景是否都有"基础线索"和"隐藏线索"？
- [ ] 隐藏线索的触发条件是否明确（玩家行为/对话/时间）？
- [ ] 是否有"未发现的线索导致推理链断裂"的设计（增加真实感）？
- [ ] 线索投屏机制是否明确（谁发现/谁决定共享）？
- [ ] DM 是否有"线索提示"机制（玩家完全卡住时使用）？

## T19 角色技能系统设计（参考《大侦探》）

**适用类型**：硬核推理、变格、阵营本
**核心逻辑**（参考《大侦探》角色技能系统）：每位玩家根据角色身份拥有"专属能力"，消耗"行动点数"使用，形成"风险与收益并存"的博弈。

### 技能系统设计三要素

```
要素一：技能与角色身份绑定
  → 技能必须"符合角色身份"，不能随便给
  → 示例：
    - 法医 → 对尸体深度鉴定（揭示隐藏死因/时间偏差）
    - 黑客 → 破解电子设备（获取删除记录/聊天记录）
    - 心理学家 → 质疑他人证词逻辑（识别谎言/矛盾）

要素二：行动点数限制
  → 每个玩家有有限行动点数（如 3 点）
  → 每次使用技能消耗 1 点
  → 点数用完后，只能通过"特殊事件"（如帮助他人/被发现秘密）恢复
  → 博弈设计：玩家需权衡"什么时候用技能收益最大"

要素三：技能使用时机
  → 不能在"搜证阶段"随时使用，必须有"触发条件"
  → 触发条件示例：
    - 法医技能：只有在"尸体检验"环节才能使用
    - 黑客技能：只有在"电子设备搜证"环节才能使用
    - 心理学家技能：只有在"讨论阶段"才能使用
```

### 技能卡片设计模板

```
【角色：______】
【专属技能：______】

技能描述：
  ______（这个技能能做什么）

使用条件：
  □ 必须在 [阶段名称] 阶段使用
  □ 必须消耗 [X] 点行动点数
  □ 必须满足 [前置条件]（如：先发现某线索）

技能效果：
  → 成功使用：______
  → 失败后果：______（如果技能使用失败，有什么后果）

行动点数恢复方式：
  ______（如何在游戏中恢复点数）
```

### 技能系统与其他系统的联动

| 联动系统 | 联动方式 |
|---------|---------|
| 搜证系统 | 某些技能可以"解锁隐藏线索" |
| 投票系统 | 技能使用记录可以作为投票参考（"TA 用了技能，说明 TA 知道什么"） |
| 胜负系统 | 技能使用得当可以获得"额外胜利点数" |

### 检查清单

- [ ] 每个角色的技能是否"符合角色身份"？
- [ ] 行动点数是否合理（不能太多也不能太少）？
- [ ] 技能使用时机是否明确（有触发条件）？
- [ ] 技能失败是否有"后果"（不是无条件成功）？
- [ ] 技能系统是否与其他系统（搜证/投票/胜负）有联动？

## T20 线索网状交织设计（参考《大侦探》）

**适用类型**：硬核推理、变格、规则型
**核心逻辑**（参考《大侦探》线索设计）：线索不是"线性排列"，而是"网状交织"——玩家从支线任务（如破解日记密码）可能意外牵出主线关键证据。

### 网状线索 vs 线性线索

```
线性线索（传统）：
  A → B → C → 真相
  问题：玩家按固定顺序搜证，体验单一

网状线索（《大侦探》风格）：
  A ←→ B ←→ C
  ↗    ↘    ↗
  D ←→ 真相
  特点：玩家从不同方向都能接近真相，但路径不同
```

### 网状线索设计三步骤

```
第一步：确定"核心真相链"（3-5 条关键线索）
  → 这些线索是"必须找到"才能推导出真相的
  → 示例：[死者手机记录] + [凶器来源] + [不在场证明破解]

第二步：设计"支线线索网"（每个支线都能"碰触"到核心真相链）
  → 支线1：破解日记密码 → 发现 [凶手动机]（连接到核心链）
  → 支线2：调查嫌疑人背景 → 发现 [不在场证明破绽]（连接到核心链）
  → 支线3：搜证现场细节 → 发现 [凶器来源线索]（连接到核心链）

第三步：设计"意外牵出"机制
  → 玩家在做支线任务时，意外发现核心线索
  → 示例：玩家在破解日记密码时，发现日记里夹着一张 [死者手机记录截图]
  → 关键：这个"意外发现"必须是"合理的新葡萄"，不是硬塞的
```

### 网状线索的"可达性"检查

```
核心真相链的每条线索，是否至少有 2 条不同的"可达路径"？
  → 路径1：直接搜证获得
  → 路径2：通过支线任务意外获得
  → 如果只有 1 条路径：玩家如果错过了，就推导不出真相（设计失败）
```

### 检查清单

- [ ] 核心真相链是否有 3-5 条关键线索？
- [ ] 每条核心线索是否至少有 2 条"可达路径"？
- [ ] 支线线索是否都能"碰触"到核心真相链（不是孤立的）？
- [ ] "意外牵出"机制是否合理（不是硬塞的）？
- [ ] 玩家如果走不同支线，最终都能推导出真相吗？

## T21 童谣杀人模式（参考《恐怖童谣》）

**适用类型**：变格、恐怖悬疑、规则型
**核心逻辑**（参考《明星大侦探》第二季《恐怖童谣》，借鉴阿加莎·克里斯蒂《无人生还》并本土化）：将"童谣歌词"与"死亡事件"一一对应，制造"宿命感"和"不可逆性"。

### 童谣杀人模式设计三要素

```
要素一：童谣歌词设计
  → 歌词数量 = 死亡人数（或略多于死亡人数）
  → 每句歌词暗示：死亡方式 / 死亡场景 / 死亡顺序
  → 关键：歌词不能太直白（"XX 被刀刺死"），要有"隐喻性"
  → 示例（参考《恐怖童谣》）：
    "十个印第安小男孩，砍柴斧下丧性命" → 暗示第一个死者被斧头砍死

要素二：死亡事件与歌词对应
  → 每个死亡事件都必须"精确对应"一句歌词
  → 对应方式可以是：
    - 直接对应：死亡方式完全一致
    - 隐喻对应：死亡方式"像"歌词描述（不是完全一致，但能联想到）
  → 关键：玩家可以通过"解读童谣"来预测下一个死者是谁/怎么死

要素三：童谣的来源与真相
  → 童谣不是"凭空出现"的，必须有来源
  → 来源选项：
    - 凶手编写（为了制造"宿命感"，让其他玩家恐慌）
    - 历史遗留（多年前的一首童谣，凶手按童谣行事）
    - 超自然力量（变格剧本，童谣是"规则"的一部分）
  → 真相揭秘时，必须解释"童谣的来源"和"凶手为什么按童谣行事"
```

### 童谣杀人模式设计模板

```
【童谣名称：______】

童谣歌词（共 ______ 句）：
  第 1 句：______ → 对应死者 ______，死亡方式 ______
  第 2 句：______ → 对应死者 ______，死亡方式 ______
  ...（依此类推）

童谣来源：
  ______（谁写的 / 哪里来的）

凶手动机：
  为什么要按童谣杀人？______（制造恐慌 / 复仇仪式 / 超自然规则）

玩家互动设计：
  → 玩家可以在搜证阶段找到"童谣文本"
  → 玩家可以通过"解读童谣"来预测下一个死者
  → DM 可以根据童谣推进剧情（"第 X 句应验了..."）
```

### 童谣杀人模式的"本土化"要点

```
（《恐怖童谣》的成功在于"本土化"，不是简单照搬《无人生还》）

本土化要点：
  1. 童谣选择：用中国玩家熟悉的童谣（如"两只老虎"/"小星星"），不是西方童谣
  2. 死亡方式：符合中国语境（如"喝农药"比"被手枪射击"更符合本土感觉）
  3. 场景设置：符合中国玩家认知（如"农村老宅"/"工厂宿舍"，不是"英国孤岛"）
```

### 检查清单

- [ ] 童谣歌词是否有 5-10 句（对应死亡人数）？
- [ ] 每句歌词是否"有隐喻性"（不是直白描述死亡）？
- [ ] 每个死亡事件是否都"精确对应"一句歌词？
- [ ] 童谣的来源是否有合理解释（不是凭空出现）？
- [ ] 是否进行了"本土化"处理（符合中国玩家认知）？
