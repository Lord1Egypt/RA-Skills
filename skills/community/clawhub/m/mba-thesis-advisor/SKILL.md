# MBA 论文写作与排版顾问

本 skill 指导你完成 MBA 论文从初稿到优秀论文的全过程——包括诊断问题、重构框架、
提炼贡献、改写关键章节、以及最终排版。基于清华 MBA 论文的实战经验，也适用于
其他中国或国际 MBA 项目。

---

## 核心理念：写法 B（批判性写法）

大多数 MBA 论文初稿采用**写法 A**：描述公司问题 → 套用标准框架（SWOT、PEST、
波特五力）→ 提出泛泛建议。这种写法只能拿到及格分。

**写法 B**（批判性-分析性写法）是区分优秀论文的关键：

- 从一个**反直觉的内部观察**出发，挑战常规认知
- 构建一个**解释机制**的框架，而不仅仅描述现象
- 得出**非显而易见的、可证伪的、有边界条件的**结论
- 读者读完后会想：*"如果不看这篇论文，我不会知道这些"*

**写法 B 的三个标志：**
1. 有数据支撑的反直觉发现
2. 识别出因果机制（不仅仅是相关性）
3. 明确的边界条件（"在X条件下成立，在Y条件下不成立"）

## 阶段一：诊断

### 步骤 1.1 — 通读全文

阅读 `mydata/` 目录下的所有 `.tex` 文件，包括：
`chap01.tex` 到 `chap05.tex`、`abstract.tex`

对每一章评估：
- 提出了什么论断？
- 有什么证据支撑？
- 这些内容是原创的，还是任何行业报告都能写出来？

### 步骤 1.2 — 用 Rubric 评分

使用 7 维度 Rubric 对初稿进行系统评分。完整 Rubric 见
`references/evaluation-rubric.md`。

**7 维度总览：**

| # | 维度 | 权重 | 核心问题 | 危险信号（≤ 1 分） |
|---|------|:----:|---------|-------------------|
| D1 | **原创贡献** | 20% | 说了什么前人没说过的？ | 无法用一句话说出新发现 |
| D2 | **内部视角** | 20% | 是否利用了内部人身份？ | 行业分析师能写出同样内容 |
| D3 | **理论框架** | 15% | 框架是分析利器还是标签？ | SWOT 作为主力分析工具 |
| D4 | **论证连贯** | 15% | RQ→理论→分析→结论是否不断线？ | 存在死理论或无源结论 |
| D5 | **数据证据** | 10% | 论断有可追溯证据吗？ | 声称有问卷但无方法论 |
| D6 | **结论严谨** | 15% | 结论可证伪、有边界吗？ | "企业应该加强……" |
| D7 | **批判深度** | 5% | 有反直觉发现吗？ | 全文发现均"符合预期" |

**底线规则**：D1 或 D2 任一 ≤ 1.5 分，总评封顶"及格"。

**操作要求**：逐一对每个维度的子指标打 1-4 分（见 Rubric 详细标准），
填写评分汇总模板，计算加权总分，并按 ROI 排列改进优先级。

### 步骤 1.3 — 挖掘内部视角

向作者提出五个关键问题，解锁其内部人视角。详见
`references/diagnostic-questions.md`。

这五个问题的回答是整个改写的**原材料**。没有这些回答，不要进入阶段二。

## 阶段二：理论框架升级

### 步骤 2.1 — 选择合适的理论骨架

用更高层次的学术理论替代或补充通用 MBA 框架：

| 研究场景 | 推荐理论框架 |
|----------|-------------|
| 组织变革 / 研发管理 | 动态能力理论（Teece 2007） |
| 跨文化 / 制度环境 | 制度同构理论（DiMaggio & Powell 1983） |
| 战略执行失败 | 委托代理理论 + 路径依赖 |
| 技术采纳 | 吸收能力理论（Cohen & Levinthal 1990） |
| 团队 / 组织设计 | 团队拓扑 + 康威定律 |
| VUCA / 不确定性 | VUCA 框架 + 情景规划 |
| 供应链 / 平台 | 资源基础观（Barney 1991） |

**规则：** PEST 和波特五力可以作为第二、三章的*背景分析工具*，
但绝不能作为第四章的主要分析框架。

### 步骤 2.2 — 构建分析框架图

绘制一张框架图（放在第二章或第一章），展示：
1. 外部环境压力 → 企业应对机制 → 结果
2. 理论视角在框架中的位置
3. 研究问题在框架中的映射

这张图将成为贯穿全文的"骨架"。

## 阶段三：提炼原创贡献

### 步骤 3.1 — 找到机制

从内部视角（阶段一步骤 1.3）中，识别一个**因果机制**：解释为什么会发生，
而不仅仅是发生了什么。

模板：
> "[公司/行业] 面临 [问题]。传统观点认为 [X]。但我们的分析表明，
> 真正的机制是 [Y]：当 [条件] 存在时，[原因] 导致 [结果]，
> 因为 [机制]。这一发现的意义在于 [启示]。"

### 步骤 3.2 — 命名贡献

给原创贡献起一个**令人印象深刻的名字**。例如：
- "运动式研发"
- "本土化悖论"
- "监管驱动的战略漂移"

有名字的概念可以被引用、容易记住，体现学术严肃性。

### 步骤 3.3 — 明确陈述贡献

在结论章节中，添加专门的小节：

```latex
\subsection{理论贡献}
本文的主要理论贡献包括：
\begin{enumerate}
  \item \textbf{概念提出：}...（原创概念名称）...
  \item \textbf{机制识别：}...（因果机制）...
  \item \textbf{框架整合：}...（理论整合方式）...
\end{enumerate}
```

## 阶段四：改写关键章节

### 第四章（核心分析）— 优先改写

第四章承载论文的全部分量。检查清单：
- [ ] 以研究问题开篇，而非背景叙述
- [ ] 每个小节标题都是一个**论断**，而非主题标签
  - 差："4.2 公司战略分析"
  - 好："4.2 监管趋严下CT公司战略漂移的三重机制"
- [ ] 每个小节结构：论断 → 证据 → 机制 → 启示
- [ ] 每个主要部分至少有一个反直觉发现
- [ ] 有量化证据（即使是合理估算也要注明依据）

### 第五章（结论）— 重点改写

用以下结构替代泛泛的建议：
1. **核心发现陈述**（1段）：论文最重要的一个发现
2. **理论贡献**（命名的概念、识别的机制）
3. **管理启示**（针对本案例公司的具体建议，不是万金油建议）
4. **边界条件**：在什么情况下这些发现不适用？
5. **局限性与未来研究方向**

结论中要避免这些表述：
- "企业应加强……"（太泛）
- "建议公司提升……"（没有可操作性）
- "未来可进一步研究……"（太空洞）

### 第二章（文献综述）— 针对性补充

为选定的理论框架（步骤 2.1）补充 3-5 篇基础文献。
结构：已有理论 → 研究空白 → 本文如何填补空白。

## 阶段五：数据与证据

### 步骤 5.1 — 数据来源

中国互联网/科技公司：年报（投资者关系页面）、WIND、Bloomberg
工业/农业公司：CNKI 行业报告、公司投资者关系页面、海关总署数据、农业部数据

### 步骤 5.2 — 最低证据标准

第四章每个重要论断至少需要以下之一：
- 有出处的具体数字
- 具名的内部事件/决策（必要时匿名化）
- 访谈/问卷的直接引用（如进行了一手研究）
- 跨时间段或跨竞争对手的比较

### 步骤 5.3 — 图表

一篇优秀论文建议的最少图表数量：
- 1 张分析框架图（第二章）
- 1-2 张趋势图（第三章：行业背景）
- 1-2 张对比表（第四章：公司 vs. 同行）
- 1 张总结框架图（第五章：贡献可视化）

使用 Python + matplotlib 生成图表，同时保存 PDF 和 PNG 格式。

黑白图表生成的详细规范和代码模板见 `references/formatting-guide.md`。

## 阶段六：质量自查

提交前逐项核查：

### 学术严谨性检查
- [ ] 每个论断都有引用或数据支撑
- [ ] 没有纯描述性段落（每段都要回答"所以呢"）
- [ ] 理论名称正确引用了原始作者和年份
- [ ] 结论与绪论中的承诺一致

### 写法 B 检查
- [ ] 至少有一个让读者感到意外的发现
- [ ] 原创概念或机制已命名并定义
- [ ] "理论贡献"小节明确陈述了创新点
- [ ] 建议是针对本案例公司的，而非通用建议

### LaTeX 排版检查（thuthesis）
- [ ] 所有图表都用 `\ref{}` 交叉引用并有标题
- [ ] 表格有规范的 `\caption{}` 和数据来源脚注
- [ ] 参考文献引用格式统一
- [ ] 摘要清晰陈述了研究问题、方法、发现和贡献
- [ ] 所有图表适配黑白打印（见阶段七）
- [ ] 授权声明页和声明页已插入签名
- [ ] 送审/答辩版的评语页和决议书页只保留标题

## 阶段七：提交前排版

本阶段涵盖论文送审或答辩前的排版工作。详细规范和代码模板见
`references/formatting-guide.md`。

### 步骤 7.1 — 黑白图表转换

所有图表必须在黑白打印时仍可清晰辨认。使用 matplotlib 时：

- 柱状图用**填充纹样**（`//`、`\\`、`..`、`xx`、`++`）代替颜色区分
- 折线图用**标记符号**（`o`、`s`、`^`、`D`、`v`）+ **线型**（`-`、`--`、`-.`、`:`）区分
- **灰度色阶**（`#000`、`#444`、`#888`、`#BBB`）作为辅助手段

**关键规则：** matplotlib `figsize` 宽度必须等于 LaTeX 显示宽度
（thuthesis 为 5.91 英寸），确保字号 1:1 不缩放。LaTeX 端使用
`\includegraphics[width=1\textwidth]`。

**最小字号要求：** 正文文字 10pt，刻度标签 8pt，任何文字不得小于 8pt。

### 步骤 7.2 — 表格脚注

使用 `threeparttable` + `tablenotes` + `\tnote{}` 添加表内脚注。
禁止在 `tabular` 环境中使用 `\footnote{}`。

### 步骤 7.3 — 签名插入

在授权声明页和声明页插入扫描签名，需修改 `thuthesis.cls`：
- 在 `\thu@underline` 内使用 `\includegraphics[height=Xem]{myfigure/sig-name.jpg}`
- 作者签名一般 height=3.0-4.0em，导师签名根据实际调整
- 日期同样填入 `\thu@underline` 中

### 步骤 7.4 — 页面样式配置

通过 `my-thesis.tex` 控制特殊页面的页眉/页脚：
```latex
\statement[page-style=plain]   % 有页眉页脚（硕博推荐）
\statement[page-style=empty]   % 无页眉页脚
```

### 步骤 7.5 — 送审/答辩版特殊页面

送审和答辩版本中，部分页面只显示标题：
- **指导教师评语**（`comments.tex`）：空的 `\begin{comments}\end{comments}`
- **答辩委员会决议书**（`resolution.tex`）：空的 `\begin{resolution}\end{resolution}`

这些环境会自动生成页面标题；空的环境体 = 只有标题的空白页，
答辩后由评委手写填入。

### 步骤 7.6 — 编译

```bash
# macOS (MacTeX) — 需要设置 PATH
export PATH="/Library/TeX/texbin:$PATH"
latexmk -xelatex my-thesis.tex

# 清除缓存重新编译（解决部分编译错误）
latexmk -C && latexmk -xelatex my-thesis.tex
```

开发阶段推荐字体集：`fandol`（跨平台）。
终版提交推荐字体集：`windows`（中文字体渲染最佳）。

## LaTeX 工作流备忘

**论文主文件：** 通常为 `my-thesis.tex`
**章节文件：** `mydata/chap01.tex` 到 `mydata/chap05.tex`
**图表文件：** 放在 `myfigure/` 或 `figures/` 目录，用相对路径引用

**编译命令：**
```bash
cd <论文根目录>
latexmk -xelatex my-thesis.tex
```

**Git 工作流：** 只提交源文件，忽略编译产物。
`.gitignore` 典型配置：`*.pdf`、`*.log`、`*.aux`、`*.synctex.gz`、`*.bbl`、`*.blg`
不要忽略：`myfigure/*.py`、`myfigure/*.png`、`mydata/*.tex`、`refs.bib`、`abstract.tex`
