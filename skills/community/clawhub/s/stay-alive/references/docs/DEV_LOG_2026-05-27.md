## Draft Action State Tracking

Extended the draft command chain so drafts can be reviewed as a small queue instead of a stateless list:

- `review-drafts.mjs`
  - Reads local action artifacts from `runtime/stay-alive/agents/<agent>/actions/`.
  - Hides drafts that already have a dry-run/send action by default.
  - Supports `--include-applied` to audit processed drafts.
  - Computes draft status as `draft`, `applied_dry_run`, `sent`, or `send_failed`.
  - Includes the latest action artifact path when a draft has been handled.
- `apply-draft.mjs`
  - Checks prior action artifacts for the same `run_id` + `draft_index`.
  - Refuses to send a draft if a previous successful send exists.
  - Records `prior_action_count` in each new action artifact.

Validation:


- before action artifact: `draft_count=1`
