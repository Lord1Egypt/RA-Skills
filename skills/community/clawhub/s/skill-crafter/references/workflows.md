# 流程模式与四层骨架

选择结构模式后，用四层骨架来组织它的每一部分。四层骨架是组织原则，结构模式决定操作指南层的内部节奏。

---

## 流程型——操作指南按 Phase 组织

适合有明确步骤顺序的工作流（报告生成、数据分析、代码审查）。

四层骨架如何落地：

```
头部：一句话 + 触发词 + 边界
概述：一句话定位 + 功能范围
操作指南：
  Phase 1: [步骤]
  Phase 2: [步骤]
  ...
  多轮修改：[用户想改某部分时的流程]
补充说明：依赖缺失、超时、边界情况
```

**正确示例（报告生成 Skill 的操作指南层）：**

```markdown
## 使用

### Phase 1：需求理解

1. 从用户描述中提取：主题、目标受众、期望深度
2. 整理为研究框架，一次性让用户确认
3. 确认后进入 Phase 2

### Phase 2：深度搜索

1. 根据研究框架拆分搜索关键词
2. 每个关键词用 search + fetch 收集 3-5 篇核心素材
3. 素材保存到 /sandbox/workspace/outputs/research/

### Phase 3：大纲 + 撰写

1. 基于素材生成大纲（确认门）
2. 用户确认后分章节撰写
3. 整合为完整报告

### 多轮修改

用户说"扩展第三章"或"数据不够新"时：
1. 定位要改的章节
2. 补充搜索或重写
3. 替换原内容
```

## 任务型——操作指南按决策表路由

适合多种独立操作、按意图路由的任务（知识库管理、文件操作、CRUD 工具）。

四层骨架如何落地：

```
头部：一句话 + 触发词 + 边界
概述：一句话定位 + 功能范围（按操作类型分列）
操作指南：
  决策表：用户意图 → 操作
  操作 1: [步骤]
  操作 2: [步骤]
  ...
补充说明：权限问题、冲突处理
```

**正确示例（知识库管理 Skill 的操作指南层）：**

```markdown
## 使用

### 意图路由

| 用户说 | 操作 |
|--------|------|
| 导入/上传/添加文件到知识库 | → 导入操作 |
| 创建/新建知识库 | → 创建操作 |
| 找/搜索知识库里的内容 | → 搜索操作 |
| 移动/重命名/整理 | → 管理操作 |

### 导入操作

1. 用 ima-knowledge skill 获取 kb_id
2. 用 ima_cos_util 上传文件获取 cosKey
3. 调用导入 API

### 创建操作

1. 用 ima-knowledge skill 创建知识库
2. 返回 kb_id 给用户
```

## 规范型——操作指南按规范细则组织

适合标准/规范/指南类 Skill（品牌写作规范、代码规范、审核标准）。

四层骨架如何落地：

```
头部：一句话 + 触发词 + 边界
概述：一句话定位 + 规范覆盖范围
操作指南：
  规范细则（每条 = 场景 + 正误对比）
  检查清单
补充说明：模糊地带的判断原则、规范冲突时的优先级
```

**正确示例（品牌写作规范 Skill 的操作指南层）：**

```markdown
## 使用

### 语气规范

❌ 我们的产品非常优秀，绝对是行业第一
✅ 我们的产品在 XX 场景下表现稳定，客户反馈积极

原则：用事实支撑，不用绝对化表达。

### 用词规范

❌ AI 驱动的智能解决方案
✅ 基于机器学习的自动化处理流程

原则：不用行业黑话，用用户能理解的词。

### 检查清单

写完后逐项检查：
- [ ] 无绝对化表达（"最""绝对""确保"）
- [ ] 无行业黑话
- [ ] 每个结论有数据或案例支撑
```

## 能力集型——操作指南按核心能力组织

适合多个关联功能的集成（客服工作台、开发工具链、多 API 聚合）。

四层骨架如何落地：

```
头部：一句话 + 触发词 + 边界
概述：一句话定位 + 能力列表
操作指南：
  能力 1: [场景 + 步骤]
  能力 2: [场景 + 步骤]
  能力组合: [跨能力的典型工作流]
补充说明：API 限制、配额、降级方案
```

**正确示例（客服工作台 Skill 的操作指南层）：**

```markdown
## 使用

### 工单查询

用户说"查一下工单 #12345"：
1. 调用工单 API 查询详情
2. 格式化返回给用户

### 知识库检索

用户说"退货政策是什么"：
1. 在知识库中搜索相关条目
2. 返回最相关的 3 条结果

### 工单 + 知识库组合

用户说"这个工单的客户问退货怎么处理"：
1. 先查询工单获取上下文
2. 根据工单内容在知识库中检索相关政策
3. 组合为完整回复
```
