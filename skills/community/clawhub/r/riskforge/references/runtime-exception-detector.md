# 运行时异常检测器 (Runtime Exception Detector)

基于.md描述的标准skill，使用大模型进行运行时异常风险检测。

## Skill定义

### 基本信息
- **名称**: 运行时异常检测器
- **描述**: 检测代码中可能导致运行时异常的潜在风险
- **风险类型**: Runtime Exception
- **支持语言**: Java, JavaScript/TypeScript, Python

### 检测规则

#### 空指针异常检测 (NullPointerException)
1. **未检查的对象调用**: 检测未进行null检查就直接调用对象方法或属性的情况
2. **集合操作**: 检测对可能为null的集合进行操作
3. **方法返回值**: 检测未检查方法返回值是否为null
4. **链式调用**: 检测链式调用中可能的空指针风险

#### 数组越界异常检测 (ArrayIndexOutOfBoundsException)
1. **直接索引访问**: 检测数组访问时未进行边界检查的情况
2. **集合方法访问**: 检测通过方法访问时的边界检查缺失
3. **循环边界**: 检测循环中的数组访问边界问题
4. **动态索引**: 检测使用变量作为索引时的边界检查

#### 类型转换异常检测 (ClassCastException/TypeError)
1. **显式转换**: 检测未进行异常处理的类型转换操作
2. **隐式转换**: 检测可能导致转换异常的操作
3. **parse方法**: 检测parseInt、parseFloat等转换函数的使用
4. **泛型转换**: 检测泛型类型转换的安全性

#### 除零异常检测 (ArithmeticException)
1. **直接除法**: 检测除法操作中未检查除数是否为零的情况
2. **变量除法**: 检测使用变量作为除数时的零值检查
3. **表达式除法**: 检测复杂表达式中的除零风险
4. **取模运算**: 检测取模运算中的零值问题

#### 字符串索引异常检测 (StringIndexOutOfBoundsException)
1. **charAt方法**: 检测字符串charAt方法调用的边界检查
2. **substring方法**: 检测substring方法调用的边界检查
3. **动态索引**: 检测使用变量作为字符串索引的情况
4. **循环处理**: 检测循环中字符串处理的边界问题

### 风险等级定义
- **高风险**: 必然导致运行时异常的代码
- **中风险**: 在特定条件下可能导致异常的代码
- **低风险**: 可能性较低的潜在异常风险

## 分析方法

### 分析步骤（每个风险必须经过回溯校验）

**步骤1: 代码理解**
- 理解代码变更的内容和上下文
- 确定涉及的代码段和变量

**步骤2: 模式匹配**
- 根据上述检测规则识别异常风险模式
- 定位可能的异常点

**步骤3: 上下文分析**
- 分析变量的可能取值范围
- 确定异常触发条件

**步骤4: 控制流分析**
- 分析代码执行路径
- 识别可能的异常传播路径

**步骤5: 风险评估**
- 评估异常发生的可能性
- 确定风险等级

**步骤6: 生成初步风险（每个风险单独处理）**
- 对每个识别的风险，生成包含以下字段的初步结果：
  - type, module, file, **line（数字类型）**, description, codeSnippet, suggestion, testCases, level

**步骤7: 回溯校验（强制执行）**
对每个初步风险执行以下校验：

```
## 风险回溯校验

### 待校验风险
- 文件: {file}
- 行号: {line}
- 描述: {description}
- 代码片段: {codeSnippet}
- 建议: {suggestion}

### 校验任务
1. 回溯到{file}第{line}行，验证问题代码
2. 验证描述是否准确反映问题根本原因
3. 验证建议是否可实施且最优
4. 验证行号是否精确（如不准确则调整）
5. 给出置信度：高/中/低

### 校验后处理
- 高置信度：直接输出
- 中置信度：根据corrections修正后输出
- 低置信度：重新分析该风险
```

**步骤8: 输出最终风险**
- 整合校验后的风险信息
- 确保所有字段完整准确

### 输出格式（经过回溯校验）
```json
{
  "type": "Logic Defect",
  "module": "Null Pointer|Array Bounds|Type Cast|Division by Zero|String Index",
  "file": "文件路径",
  "line": 42,
  "description": "风险描述",
  "codeSnippet": "相关代码片段",
  "suggestion": "修复建议",
  "testCases": ["测试用例1", "测试用例2"],
  "level": "high|medium|low",
  "validation": {
    "validated": true,
    "confidence": "high|medium|low",
    "checks": {
      "descriptionAccuracy": true,
      "suggestionFeasibility": true,
      "lineNumberAccuracy": true,
      "impactClarity": true
    },
    "corrections": []
  }
}
```

**注意**: `line`字段必须为数字类型，如 `42`，不能是字符串 `"42"`。

### 行号定位规范
- **必须**：每个风险必须包含具体行号，格式为数字
- **精确性**：行号必须精确到引发风险的具体代码行
- **验证**：行号必须在文件实际行数范围内
- **关联性**：行号必须与codeSnippet中的代码片段完全对应
## 使用说明

### 触发条件
- 当检测到代码变更包含潜在异常风险时触发分析
- 当检测到上述异常模式时触发分析
- 当需要对代码进行质量检查时触发分析

### 配置参数
- **语言支持**: Java (主要), JavaScript/TypeScript, Python
- **文件过滤**: 仅分析源代码文件 (*.java, *.js, *.ts, *.py)
- **忽略规则**: 忽略测试文件、配置文件、构建产物

### 集成方式
此skill通过RiskForge CLI框架自动加载，执行分析时会使用大模型进行智能分析。

### 回溯校验集成
**重要**: 回溯校验已嵌入到分析步骤中，不是独立的后处理环节。

- 每个风险生成后立即执行回溯校验
- 校验内容包括：描述准确性、建议有效性、行号精确性、影响清晰度
- 根据置信度决定是否需要修正
- 低置信度风险需要重新分析

详细校验规则请参考：`references/risk-backtracking-validator.md`

## 注意事项
1. 分析结果依赖于大模型的理解能力
2. 对于复杂的控制流，可能需要人工复核
3. **每个风险必须经过回溯校验才能输出**，不可跳过校验环节
4. **行号必须为数字类型**，如 `42`，不能是字符串 `"42"`
5. 建议结合单元测试验证修复效果
6. 注意处理边界条件和异常情况
7. 低置信度风险应重新分析，不应降低标准输出
### 与风险回溯校验器的集成
- 运行时异常检测器生成的风险会自动传递给风险回溯校验器
- 校验器会对每个风险进行准确性、清晰度和建议有效性验证
- 校验结果会修正原始风险描述，提升风险报告的精准度
