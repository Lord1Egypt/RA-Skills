# 增长策略指南

## 核心目标

找到可重复的增长渠道 + 健康的单位经济模型

## 增长的三大引擎

### 引擎 1: 病毒式增长（Viral Growth）

**核心机制**：用户使用产品时自然带来新用户

**关键指标**：
- K 因子（K-factor）：每个用户带来多少新用户
- K > 1：病毒式增长
- K < 1：需要其他增长渠道

**适用场景**：
- 社交产品（微信、Facebook）
- 协作工具（Figma、Notion）
- 内容平台（抖音、小红书）

**如何提升 K 因子**：
1. 内置分享机制
   - 邀请好友得奖励
   - 分享内容吸引新用户
2. 协作功能
   - 与他人共享文档
   - 团队协作
3. 社交货币
   - 用户愿意在朋友圈晒
   - 形成品牌认知

### 引擎 2: 付费增长（Paid Growth）

**核心机制**：花钱买用户，LTV > CAC

**关键指标**：
- CAC（客户获取成本）：获得一个付费用户的成本
- LTV（客户终身价值）：一个用户带来的总收入
- LTV/CAC > 3：健康的付费增长

**适用场景**：
- SaaS 产品
- 电商
- 在线教育

**如何优化付费增长**：
1. 提升转化率
   - 优化落地页
   - A/B 测试
   - 优化用户引导流程
2. 降低 CAC
   - 优化广告投放
   - 提升广告点击率
   - 找到更便宜的流量渠道
3. 提升 LTV
   - 降低流失率
   - 提升客单价
   - 增加复购

### 引擎 3: 内容/SEO 增长（Content/SEO Growth）

**核心机制**：通过内容吸引自然流量

**关键指标**：
- 自然流量占比
- 内容转化率
- SEO 排名

**适用场景**：
- 信息类产品
- B2B SaaS
- 工具类产品

**如何做内容增长**：
1. 找到目标用户的搜索词
   - 使用 Google Keyword Planner
   - 分析竞争对手关键词
2. 生产高质量内容
   - 解决用户的真实问题
   - 长期价值（不是热点文章）
3. 优化 SEO
   - 关键词布局
   - 内链外链
   - 页面加载速度

## 增长策略选择框架

### 第一步：理解产品属性

| 产品类型 | 推荐增长引擎 |
|----------|--------------|
| 社交产品 | 病毒式增长 |
| SaaS/B2B | 付费增长 + 内容增长 |
| 电商 | 付费增长 + 内容增长 |
| 工具类 | 内容增长 + 病毒式增长 |

### 第二步：找到第一个增长渠道

**原则**：在一个渠道做深，不要同时尝试多个渠道。

**决策流程**：
1. 列出所有可能的增长渠道
2. 评估每个渠道的可行性
   - 能否触达目标用户？
   - 成本是多少？
   - 竞争程度如何？
3. 选择一个最有可能成功的渠道
4. 集中资源把这个渠道做透

### 第三步：建立可重复的增长流程

**流程要素**：
1. 获客：用户从哪里来？
2. 激活：用户如何体验到核心价值？
3. 留存：用户为什么留下来？
4. 变现：用户如何付费？
5. 推荐：用户如何带来新用户？

**文档化**：
- 每个环节的 SOP
- 关键指标仪表盘
- 定期复盘机制

## 增长的常见陷阱

### 陷阱 1: 过早扩张

症状：在 PMF 之前就大规模投放广告。

解药：
- 先验证留存和付费
- 确保 LTV/CAC 健康
- 再大规模扩张

### 陷阱 2: 只关注获客，忽视留存

症状：用户增长快，但流失也快。

解药：
- 留存 > 获客
- 一个留存用户 = 多个新用户
- 关注用户生命周期价值

### 陷阱 3: 追求虚荣指标

症状：关注下载量、注册量，而不是留存和付费。

解药：
- 定义真正的成功指标
- 用北极星指标指导决策
- 不要被虚荣指标迷惑

### 陷阱 4: 增长渠道分散

症状：同时尝试 SEO、广告、内容、社群，没有一个做透。

解药：
- 选择一个最有潜力的渠道
- 集中资源做深
- 做到极致后再考虑其他渠道

## 增长检查清单

在进入 Scale 阶段之前，必须确认：

- [ ] 找到了至少一个可重复的增长渠道
- [ ] LTV/CAC > 3（如果使用付费增长）
- [ ] 留存率稳定且健康
- [ ] 有完整的增长数据分析体系
- [ ] 团队能独立运作增长工作

**如果不确定，继续优化；如果确定，准备 Scale。**
