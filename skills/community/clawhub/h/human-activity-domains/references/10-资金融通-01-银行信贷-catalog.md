## 银行信贷任务清单

银行信贷按子域组织的任务清单，附任务间关系和元操作映射提示。

**域间逻辑流**：信贷申请 → 信贷调查 → 信贷审批 → 信贷管理

---

### 1 信贷申请

| ID | 任务类型 | 说明 | 依赖 | 元操作映射 |
|----|----------|------|------|----------|
| T1-1 | 客户申请 | 接受客户信贷申请，了解客户需求 | 无（入口） | S→I |
| T1-2 | 资料收集 | 收集客户资料，包括身份证明、收入证明、资产证明 | T1-1 | S→A |
| T1-3 | 初步审核 | 初步审核客户资料，评估基本条件 | T1-2 | C→G |
| T1-4 | 受理登记 | 受理信贷申请，登记客户信息 | T1-3 | A→O |

---

|----|----------|------|------|----------|

## 依赖拓扑摘要

### 一、域内链路

**信贷申请链路**: T1-1 → T1-2 → T1-3 → T1-4
**信贷调查链路**: T2-1 → T2-2 → T2-3 → T2-4
**信贷审批链路**: T3-1 → T3-2 → T3-3 → T3-4
**信贷管理链路**: T4-1 → T4-2 → T4-3 → T4-4

### 二、跨域协同链路

**完整银行信贷链路**: T1-1 → T1-2 → T1-3 → T1-4 → T2-1 → T2-2 → T2-3 → T2-4 → T3-1 → T3-2 → T3-3 → T3-4 → T4-1 → T4-2 → T4-3 → T4-4

**反馈改进链路**: T4-4 → T1-1（贷款回收反馈到客户申请）

**风险驱动链路**: T4-3 → T2-3（风险预警驱动风险分析）

更多组合根据具体任务动态推导。
