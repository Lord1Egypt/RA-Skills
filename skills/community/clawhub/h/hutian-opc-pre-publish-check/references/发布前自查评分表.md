# 发布前自查评分表

## 1. 评分体系概述

本评分表用于量化评估Skill/Agent的发布准备度，通过9大检查项的综合评分，得出最终可发布判定。

**评分原则**：
- 量化优于定性
- 可操作优于模糊
- 证据优于陈述

---

## 2. 综合评分计算

### 2.1 各检查项权重

| 序号 | 检查项 | 权重 | 理由 |
|------|--------|------|------|
| 1 | 案例支撑验证 | 15% | 质量基础保障 |
| 2 | 一致性检查 | 20% | 核心质量指标 |
| 3 | 竞品分析与卡位 | 15% | 市场定位保障 |
| 4 | 创新独特点提炼 | 10% | 差异化价值 |
| 5 | Prompt工程文档化 | 10% | 智力贡献证据 |
| 6 | IP与版权检查 | 10% | 法律合规 |
| 7 | 商业钩子检查 | 5% | 商业引流 |
| 8 | 商业化审核 | 10% | 商业合理性 |
| 9 | 合规与安全 | 5% | 底线保障 |
| **合计** | | **100%** | |

### 2.2 综合得分计算

```
综合得分 = Σ(各检查项得分 × 权重)
```

**判定标准**：

| 综合得分 | 判定 | 含义 |
|----------|------|------|
| 90-100分 | ✅卓越 | 可发布，且具有优秀品质 |
| 80-89分 | ✅良好 | 可发布，品质良好 |
| 70-79分 | ⚠️合格 | 可发布，但需关注短板 |
| 60-69分 | ⚠️待改进 | 建议优化后发布 |
| <60分 | 🛘不合格 | 必须重大修改后发布 |

---

## 3. 各检查项评分细则

### 3.1 案例支撑验证（满分100分，权重15%）

| 评分 | 条件 | 说明 |
|------|------|------|
| 100分 | ≥20个案例 | 全覆盖+边界+异常+评分完整 |
| 85分 | 15-19个案例 | 4类场景覆盖，评分完整 |
| 70分 | 10-14个案例 | 基本覆盖，评分完整 |
| 50分 | 5-9个案例 | ⚠️需补充 |
| 20分 | <5个案例 | 🛘阻断发布 |
| 0分 | 无案例 | 🛘阻断发布 |

**场景覆盖要求**：
- 正常场景 ≥5个
- 边界条件 ≥3个
- 异常输入 ≥2个

---

### 3.2 一致性检查（满分100分，权重20%）

| 评分 | 条件 | 偏离度 |
|------|------|--------|
| 100分 | 优秀 | 0-8% |
| 90分 | 良好 | 8-12% |
| 80分 | 通过 | 12-15% |
| 60分 | 待改进 | 15-25% |
| 30分 | 较差 | 25-35% |
| 0分 | 不合格 | >35% |

**评分公式**：
```
一致性得分 = max(0, 100 - 偏离度 × 2.5)
```

---

### 3.3 竞品分析与卡位（满分100分，权重15%）

**卡位评分表**：

| 评分维度 | 权重 | 评分标准 |
|----------|------|----------|
| 卡位清晰度 | 25% | 5分：目标用户非常明确 |
| 差异化程度 | 30% | 5分：与竞品差异明显且可感知 |
| 市场空间 | 20% | 5分：市场足够大 |
| 可执行性 | 15% | 5分：完全可执行 |
| 可防守性 | 10% | 5分：护城河明确 |

**判定标准**：

| 综合得分 | 判定 | 说明 |
|----------|------|------|
| 90-100分 | ✅强卡位 | 可发布 |
| 75-89分 | ✅中等卡位 | 可发布 |
| 60-74分 | ⚠️弱卡位 | 建议优化后发布 |
| <60分 | 🛘无卡位 | 必须重新定位 |

---

### 3.4 创新独特点提炼（满分100分，权重10%）

| 评分 | 条件 |
|------|------|
| 100分 | ≥5个创新点，且都有完整非显而易见性论证 |
| 80分 | 3-4个创新点，且论证完整 |
| 60分 | 3个创新点，论证基本完整 |
| 40分 | <3个创新点，或论证不完整 |
| 20分 | 无明显创新点 |
| 0分 | 抄袭或完全复制 |

---

### 3.5 Prompt工程文档化（满分100分，权重10%）

| 评分 | 条件 |
|------|------|
| 100分 | 完整迭代记录（≥3个版本），有时间线，有效果对比 |
| 85分 | 有迭代记录（2个版本），有时间线 |
| 70分 | 有1个版本记录，有设计思考 |
| 50分 | 无迭代记录，但有完整初始设计 |
| 20分 | 无文档 |
| 0分 | 无法提供Prompt证据 |

---

### 3.6 IP与版权检查（满分100分，权重10%）

| 评分 | 条件 |
|------|------|
| 100分 | 无任何风险，有完整授权记录 |
| 85分 | 低风险，有适当标注 |
| 70分 | 中风险，有标注但需关注 |
| 50分 | 高风险，需修改后发布 |
| 20分 | 明显侵权内容 |
| 0分 | 严重侵权，阻断发布 |

---

### 3.7 商业钩子检查（满分100分，权重5%）

| 评分 | 条件 |
|------|------|
| 100分 | ≥3个有效钩子，互链完整，无孤岛 |
| 85分 | 2个有效钩子，互链基本完整 |
| 70分 | 2个钩子，无互链 |
| 50分 | 1个钩子 |
| 20分 | 无钩子 |
| 0分 | 钩子指向无效内容 |

---

### 3.8 商业化审核（满分100分，权重10%）

**子项评分**：

| 子项 | 权重 | 评分标准 |
|------|------|----------|
| 报价合理性 | 40% | 定价偏离<15%→100分，<30%→70分，>30%→30分 |
| 套餐组合 | 30% | 互补性>70%→100分，>50%→70分，<50%→30分 |
| 引流校正 | 20% | 无问题→100分，有警告→70分，有阻断→30分 |
| 一致性 | 10% | 完全一致→100分，部分不一致→70分，不一致→30分 |

**综合得分** = Σ(子项得分 × 权重)

---

### 3.9 合规与安全（满分100分，权重5%）

| 评分 | 条件 |
|------|------|
| 100分 | 全部合规，无任何风险 |
| 85分 | 有1项轻微风险，已标注 |
| 70分 | 有2-3项轻微风险 |
| 50分 | 有1项中等风险，需关注 |
| 20分 | 有严重风险 |
| 0分 | 有违规内容，阻断发布 |
