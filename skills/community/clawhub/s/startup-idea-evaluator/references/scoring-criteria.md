# 评分标准参考表

> 版本：v1.0  
> 用途：指导创业想法评估器各维度的量化评分

---

## 一、综合评分标准

### 1.1 最终结论档位

| 总分区间 | 结论 | 颜色标识 | 含义 | 建议行动 |
|---------|------|---------|------|---------|
| **80-100** | 可行 | 🟢 | 想法具备较好的基础条件 | 建议立即启动MVP验证 |
| **60-79** | 需调整 | 🟡 | 方向可行但需要优化 | 根据建议调整后可行 |
| **40-59** | 需调整 | 🟡 | 存在较大不确定性 | 需要重新验证核心假设 |
| **20-39** | 不建议 | 🔴 | 风险较高，困难明显 | 建议换方向或深度咨询 |
| **0-19** | 不建议 | 🔴 | 基础条件严重不足 | 强烈建议转介老胡深度咨询 |

### 1.2 评分分布参考

```
优秀分布（总分80+）：
├─ 单项平均：16分
├─ 可能组合：18+16+15+17+14 = 80分
└─ 含义：每个维度都有亮点，没有明显短板

良好分布（总分60-79）：
├─ 单项平均：12-14分
├─ 可能组合：14+12+13+11+10 = 60分
└─ 含义：整体中等，部分维度有短板

风险分布（总分40-59）：
├─ 单项平均：8-11分
├─ 可能组合：12+8+7+10+6 = 43分
└─ 含义：存在明显短板，需要重点关注

高风险分布（总分<40）：
├─ 单项平均：<8分
├─ 可能组合：6+5+4+8+3 = 26分
└─ 含义：多个维度存在严重问题
```

## 二、分项评分详解

### 2.1 市场规模（1-20分）

**评分公式**：

```
最终得分 = 基础分 + 增速加分

基础分判定（TAM/SAM/SOM）：
┌─────────────────────────────────────────────┐
│ SAM规模         │  基础分                   │
├─────────────────────────────────────────────┤
│ ≥100亿          │  15-20分（按实际比例）    │
│ 10-100亿        │  11-15分                  │
│ 1-10亿          │  6-10分                   │
│ <1亿            │  1-5分                    │
└─────────────────────────────────────────────┘

增速加分：
┌─────────────────────────────────────────────┐
│ 年增速         │  加分                      │
├─────────────────────────────────────────────┤
│ >30%           │  +2分                      │
│ 15%-30%        │  +1分                      │
│ 5%-15%         │  0分                       │
│ <5%            │  -1分                      │
│ 负增长          │  -2分                      │
└─────────────────────────────────────────────┘
```

**典型场景评分示例**：

| 场景描述 | 基础分 | 增速 | 最终分 | 说明 |
|---------|-------|------|-------|------|
| AI大模型应用，市场5000亿，年增速50% | 18 | +2 | 20分 | 超级大赛道 |
| 养老服务，市场2000亿，年增速15% | 15 | +1 | 16分 | 大市场，稳定增长 |
| 细分SaaS，市场50亿，年增速30% | 11 | +2 | 13分 | 可行的小而美 |
| 垂直社区，市场5亿，年增速10% | 7 | 0 | 7分 | 细分市场，精耕细作 |
| 小众手工，市场2000万 | 3 | -1 | 2分 | 规模太小 |

### 2.2 竞争强度（1-20分）

**评分逻辑**：

```
最终得分 = 竞争格局分 + 壁垒加分

竞争格局判定：
┌─────────────────────────────────────────────┐
│ 竞争格局         │  分值                    │
├─────────────────────────────────────────────┤
│ 无直接竞品       │  16-20分                 │
│ 3个以内竞品且分散│  13-15分                 │
│ 5-10个竞品      │  10-12分                  │
│ 10+竞品但分散    │  8-10分                   │
│ 头部2-3家集中    │  5-7分                    │
│ 寡头垄断         │  1-4分                    │
└─────────────────────────────────────────────┘

进入壁垒调整：
┌─────────────────────────────────────────────┐
│ 壁垒类型         │  加减分                  │
├─────────────────────────────────────────────┤
│ 高壁垒（资金+技术+资质）│  +2分              │
│ 中等壁垒         │  0分                     │
│ 低壁垒（容易复制）│  -2分                    │
└─────────────────────────────────────────────┘
```

**典型场景评分示例**：

| 场景描述 | 格局分 | 壁垒 | 最终分 | 说明 |
|---------|-------|------|-------|------|
| 全新品类，无人对标 | 18 | +2 | 20分 | 空白市场 |
| 5个竞品但各有侧重 | 13 | 0 | 13分 | 可差异化 |
| 红海但你能做品牌 | 7 | 0 | 7分 | 需要差异化 |
| 大公司垄断的领域 | 4 | -2 | 2分 | 几乎不可能 |

### 2.3 资源匹配度（1-20分）

**评分维度**：

```
资源匹配度 = f(资金匹配, 团队匹配, 能力匹配)

评分维度权重：
├─ 启动资金匹配度：30%
├─ 核心团队完整度：30%
└─ 关键能力具备度：40%

各维度判定标准：
┌─────────────────────────────────────────────┐
│ 单项匹配度         │  维度得分              │
├─────────────────────────────────────────────┤
│ 完全匹配           │  17-20分               │
│ 基本匹配（需补充1-2项）│  12-16分             │
│ 部分匹配（缺多项）  │  7-11分                │
│ 严重不匹配          │  1-6分                 │
└─────────────────────────────────────────────┘

最终得分 = Σ(维度得分 × 权重)
```

**典型场景评分示例**：

| 场景描述 | 资金 | 团队 | 能力 | 最终分 | 说明 |
|---------|-----|-----|------|-------|------|
| 资金充足+完整团队+行业经验 | 20 | 18 | 20 | 19分 | 完美匹配 |
| 资金刚好够+团队缺一人+有经验 | 14 | 12 | 16 | 14分 | 基本匹配 |
| 资金紧张+兼职团队+热情驱动 | 8 | 6 | 8 | 7分 | 资源紧张 |
| 资金不足+无团队+完全外行 | 4 | 3 | 4 | 4分 | 严重不匹配 |

### 2.4 时间窗口（1-20分）

**评分框架**：

```
最终得分 = 市场成熟度分 + 时机调整

市场成熟度判定：
┌─────────────────────────────────────────────┐
│ 市场阶段           │  基础分               │
├─────────────────────────────────────────────┤
│ 正在爆发，窗口刚开  │  16-20分              │
│ 已经成熟，稳定增长  │  13-15分              │
│ 正在成熟，即将爆发  │  11-13分              │
│ 尚未成熟（过早）   │  6-10分               │
│ 红利已过           │  1-5分                │
└─────────────────────────────────────────────┘

时机调整：
┌─────────────────────────────────────────────┐
│ 判断项             │  加减分                │
├─────────────────────────────────────────────┤
│ 政策利好           │  +2分                  │
│ 基础设施完备       │  +1分                  │
│ 资本追捧           │  +1分                  │
│ 政策利空           │  -2分                  │
│ 需要教育市场        │  -2分                  │
│ 资本冷淡           │  -1分                  │
└─────────────────────────────────────────────┘
```

**典型场景评分示例**：

| 场景描述 | 成熟度 | 时机调整 | 最终分 | 说明 |
|---------|-------|---------|-------|------|
| 政策支持+AI应用爆发期 | 18 | +3 | 21分→20分 | 完美时机 |
| 市场已成熟+基础设施完善 | 14 | +1 | 15分 | 好时机 |
| 市场萌芽+需要教育用户 | 8 | -2 | 6分 | 太早了 |
| 红利已过+资本冷淡 | 4 | -3 | 1分 | 错过时机 |

### 2.5 个人优势契合度（1-20分）

**评分框架**：

```
个人优势契合度 = f(经验壁垒, 资源壁垒, 认知壁垒, 动力契合)

评分维度权重：
├─ 行业经验壁垒：30%
├─ 资源/人脉壁垒：25%
├─ 对行业的认知深度：25%
└─ 与个人目标的契合：20%

各维度判定标准：
┌─────────────────────────────────────────────┐
│ 契合程度           │  维度得分              │
├─────────────────────────────────────────────┤
│ 高度契合（护城河）  │  17-20分               │
│ 中度契合（有优势）  │  12-16分               │
│ 轻度契合（热情驱动）│  7-11分                │
│ 不契合（完全外行）  │  1-6分                 │
└─────────────────────────────────────────────┘

最终得分 = Σ(维度得分 × 权重)
```

**典型场景评分示例**：

| 场景描述 | 经验 | 资源 | 认知 | 动力 | 最终分 | 说明 |
|---------|-----|-----|------|-----|-------|------|
| 10年经验+行业人脉+深刻认知 | 20 | 18 | 19 | 18 | 19分 | 完美契合 |
| 相关经验+可调用资源 | 14 | 14 | 13 | 15 | 14分 | 中度契合 |
| 热情驱动+自学能力 | 8 | 6 | 10 | 14 | 9分 | 轻度契合 |
| 完全外行+无资源 | 3 | 2 | 4 | 8 | 4分 | 劣势明显 |

## 三、评分一致性检查

### 3.1 异常评分检测

```
以下情况需要检查评分是否合理：

⚠️ 总分≥80但任一单项<10
   → 需要解释：优势维度足以弥补短板？

⚠️ 总分≤40但任一单项≥15
   → 需要解释：优势维度是否可持久？

⚠️ 五个维度分差>10分
   → 需要验证：是否过于极端？

⚠️ 评分与描述明显不符
   → 需要修正：重新评估
```

### 3.2 评分合理性测试

```
问自己几个问题：

1. 如果这个想法让一个完全外行来做，会得多少分？
   → 对比你给的分，差距是否合理？

2. 如果资金翻倍/减半，分数会怎么变？
   → 资源匹配度评分是否敏感？

3. 如果竞争对手突然减少/增加，分数会怎么变？
   → 竞争强度评分是否动态？

4. 如果你的朋友来做这个，分数会一样吗？
   → 评分是否客观？
```
