# openai-codex/gpt-5.4

**Recommendation:** improve activation first, then launch referrals.

## Assumptions
- The SaaS is bootstrapped, so time and engineering capacity are tight.
- Activation is not yet clearly strong.
- A referral program would take real build and operational effort.

## Facts
- Referral programs amplify the current user experience.
- If activation is weak, referrals usually just send more users into a weak funnel.
- Better activation improves both conversion and referral economics.

## Inference
- For most bootstrapped SaaS teams, fixing activation is the higher-leverage move.
- Referrals work better once users reliably reach value and are happy enough to recommend the product.

## Uncertainty
- If activation is already solid and acquisition is now the main bottleneck, launching referrals sooner may make sense.

## Bottom line
**Wait until activation is stronger.** Launch the referral program after users consistently hit first value, so you scale a working funnel instead of a leaky one.
