# phy-pre-commit-gen â€” Pre-commit Config Generator

Analyzes your project and generates a `.pre-commit-config.yaml` with the right hooks for your specific stack. No generic templates â€” actual configuration based on what your project uses.

## Quick Start

```bash
# Generate config for the current directory
python3 ~/.claude/skills/phy-pre-commit-gen/scripts/pre_commit_gen.py .

# Preview without writing
python3 ~/.claude/skills/phy-pre-commit-gen/scripts/pre_commit_gen.py . --dry-run

# Audit an existing .pre-commit-config.yaml
python3 ~/.claude/skills/phy-pre-commit-gen/scripts/pre_commit_gen.py . --audit

# Overwrite an existing config
python3 ~/.claude/skills/phy-pre-commit-gen/scripts/pre_commit_gen.py . --overwrite

# Write to a specific path
python3 ~/.claude/skills/phy-pre-commit-gen/scripts/pre_commit_gen.py . --output .pre-commit-config.yaml
```

Then activate:
```bash
pip install pre-commit
pre-commit install          # activate for this repo
pre-commit autoupdate       # update all hook revs to latest
pre-commit run --all-files  # run on existing code
```

## Hook Selection Logic

**Python:**
- Has Ruff config â†’ use `ruff` + `ruff-format` (replaces black+isort+flake8)
- Has Black config â†’ use `black` + `isort` (if isort configured) + `flake8` (if configured)
- Neither â†’ add Ruff as recommended default

**JS/TypeScript:**
- Has ESLint â†’ use `eslint` mirror hook
- Has Prettier â†’ use `prettier` mirror hook
- Neither â†’ add Prettier as baseline

**Always included:**
- `pre-commit-hooks` â€” trailing whitespace, EOF fixer, YAML/JSON/TOML check, large files, merge conflict detection, no-commit-to-branch (main/master)
- `detect-secrets` â€” secret baseline scanning
- `markdownlint-cli` â€” Markdown quality

**Conditionally included:**
- Go â†’ `go-fmt`, `go-vet`, `go-unit-tests`
- Rust â†’ `rustfmt` (local), `clippy` (local)
- Terraform â†’ `terraform_fmt`, `terraform_validate`, `terraform_tflint`
- Docker â†’ `hadolint` (Dockerfile security)
- Shell â†’ `shellcheck`
- SQL â†’ `sqlfluff`
- YAML CI present â†’ `yamllint`
- Commitizen/commitlint detected â†’ `commitizen` (commit-msg stage)

## Audit Mode

Checks an existing `.pre-commit-config.yaml` for:
- Unpinned revs (`latest`, `main`, `HEAD`) â€” can break unexpectedly
- No secret detection hook (detect-secrets or gitleaks)
- Missing trailing-whitespace hook
- Local hooks that may not work cross-platform
