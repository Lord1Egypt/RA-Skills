# Code Archaeology - Usage Example

## Basic Invocation
```
Analyze this legacy PHP codebase for security vulnerabilities and technical debt.
```

## Business Intelligence Extraction
```
Perform deep code archaeology to extract business rules, generate PRD documentation, and map business workflows from this legacy system.
```

## Targeted Analysis  
```
Perform code archaeology on the contract management module specifically, focusing on business rule extraction and workflow documentation.
```

## Resume Previous Analysis
```
Continue the code archaeology analysis from where it left off.
```

## Expected Output Structure
When Code Archaeology completes its analysis, you'll receive comprehensive reports covering:

### Business Intelligence Deliverables
- **Business Rules Catalog**: Extracted business rules with implementation details
- **PRD-Style Documentation**: Feature specifications, user stories, and acceptance criteria
- **Process Flow Diagrams**: End-to-end business workflows and state transitions
- **Entity Relationship Maps**: Data models and entity relationships

### Technical Assessment Reports
- **Executive Summary**: Key findings and overall risk assessment
- **Business Domain Map**: Organized view of business capabilities and relationships  
- **Security Assessment**: Detailed vulnerability analysis with severity ratings
- **Performance Profile**: Bottleneck identification and optimization opportunities
- **Technical Debt Inventory**: Maintenance challenges and modernization recommendations
- **Action Plan**: Prioritized next steps with implementation guidance

The analysis is stored in `.code-archaeology/` directory within your workspace for future reference and incremental updates.
