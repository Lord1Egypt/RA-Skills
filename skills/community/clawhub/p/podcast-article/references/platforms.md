# 平台格式规范

## Markdown（默认）

**适用场景**：知识库上传、GitHub Discussions、技术博客、通用存档。

**格式要求**：
- 标题用 `#` / `##` / `###` 层级
- 引用用 `>` 块
- 代码片段用反引号
- 列表用 `-` 或数字编号
- 分隔线用 `---`
- 文件扩展名：`.md`

**示例 Frontmatter（可选）**：
```markdown
---
title: 标题
podcast: 节目名
episode: 第X期
date: YYYY-MM-DD
tags: [播客, 主题词]
---
```

## 微信公众号（wechat）

> ⚠️ **纯文本输出区**：选择此平台时，输出内容中**禁止出现任何 Markdown 符号**，包括 `#`、`**`、`>`、`-`、` ``` `、`---` 等。所有格式均通过汉字标点、空行、emoji 实现。

**适用场景**：公众号推文、服务号消息、企业号内容。

**格式要求**：
- **禁用** Markdown 语法（微信编辑器不渲染）
- 标题：单独成行，加粗或全大写
- 段落：每段不超过 150 字，段与段之间空一行
- 重点用【】或「」包裹，不用 `**` 加粗
- 列表改为"①②③"或"1⃣2⃣3⃣"等 emoji 数字
- 引用改为"— 嘉宾名"跟随在引用文字后
- 结尾必须有「互动引导语」（如：你最认同哪个观点？留言区聊聊～）
- 底部固定格式：
  ```
  ▌来源：《播客名》第X期
  ▌整理：[作者/公众号名]
  ```

**字数建议**：800~1500字（微信用户注意力有限）

## 知乎（zhihu）

**适用场景**：知乎专栏文章、知乎回答引用素材。

**格式要求**：
- 支持标准 Markdown，但图片需上传到知乎图床
- 开头建议用「问题引入」风格，吸引点赞/收藏
- 小标题用 `##` 或直接加粗
- 注重引用来源标注（增加可信度）
- 加入「相关话题标签」建议（在文章末尾注释）
- 结尾固定格式：
  ```
  ---
  原始内容来源：播客《节目名》第X期
  本文为整理总结，欢迎关注原节目。
  ```

**字数建议**：1500~3000字（知乎用户偏好深度内容）

## 企业内刊 / Newsletter（newsletter）

**适用场景**：公司内部邮件、企业 Newsletter、团队周报附件。

**格式要求**：
- 输出为 HTML 或 Markdown（根据用户需求）
- 段落简洁，每段 80~120 字
- 必须包含「本期亮点」板块（3 条要点，置于正文前）
- 引用嘉宾原话时，格式为：
  ```
  ❝ 原话内容 ❞
  ── 嘉宾姓名，职位
  ```
- 结尾包含「推荐指数」（★☆☆☆☆ 到 ★★★★★）和适合人群说明
- 文章顶部加播客基础信息卡片：
  ```
  📻 节目：《节目名》
  🎙 嘉宾：嘉宾姓名
  ⏱ 时长：XX 分钟
  📅 发布：YYYY-MM-DD
  🔗 收听：[链接]
  ```

## 网页（webpage）

**适用场景**：直接在浏览器中打开阅读、分享链接、内嵌到内部平台、本地存档。

**输出规则**：
- 读取 `assets/template.html`，将文章各字段填入对应占位符，输出完整独立的 `.html` 文件
- 所有样式内联在 `<style>` 标签内，**不依赖外部 CDN 或网络资源**，离线可用
- 输出文件命名：`article.html`，保存至产出物目录

**占位符填充对照表**：

| 占位符 | 填入内容 |
|--------|---------|
| `{{TITLE}}` | 文章主标题 |
| `{{SUBTITLE}}` | 副标题（无则留空） |
| `{{PODCAST_NAME}}` | 播客节目名 |
| `{{EPISODE}}` | 期数或发布日期 |
| `{{LEAD}}` | 导语段落（纯文本，100字内） |
| `{{BODY}}` | 正文 HTML（段落用 `<p>`，小标题用 `<h2>`，引用用 `<blockquote>`） |
| `{{QUOTES}}` | 金句列表（每条一个 `<li>`） |
| `{{DATE}}` | 文章生成日期（YYYY-MM-DD） |

**格式要求**：
- 正文中的小标题用 `<h2>`，不用 `<h1>`（页面标题已占用 h1）
- 引用/金句用 `<blockquote>`，嘉宾署名用 `<cite>`
- 段落间距由模板 CSS 控制，正文只需 `<p>` 包裹，不加行内样式
- 不插入任何外链图片；如有需要，说明"此处建议配图：[描述]"

**字数建议**：与所选文章风格一致，无单独限制
