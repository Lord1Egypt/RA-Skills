# Positioning

This skill is not just another Telegram sender.

It is an OpenClaw-focused operational skill for delivering local report artifacts from self-hosted, local, and WSL environments where path handling and host-read policy quirks matter.

## Best fit users

- OpenClaw users running locally or self-hosted
- WSL/Linux + Windows mixed environments
- Agents that generate HTML/PDF/archive reports and need to deliver them into chat

## Key differentiators

- documents real OpenClaw delivery failures and fixes
- trusted tmp workaround for local HTML
- prefers `openclaw message send` CLI when wrapper/tool path misbehaves
- emphasizes absolute paths, MIME verification, and document delivery
