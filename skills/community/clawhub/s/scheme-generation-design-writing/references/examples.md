# 使用示例

本文档提供了技能的详细使用示例。

## 示例 1: 生成智能家居 APP 概念方案

### 用户输入

\\\
请基于历史项目生成一个智能家居 APP 的概念方案，要求：
- 行业：IoT / 智能硬件
- 目标用户：25-45岁，中高收入家庭用户
- 核心功能：设备控制、场景联动、数据可视化、语音控制
- 设计风格：科技感、简洁、现代
- 生成 3 个不同风格的方案
\\\

### 执行流程

1. **匹配相关项目**
   \\\ash
   python scripts/match_projects.py "智能家居 IoT 科技感"
   \\\

2. **OpenClaw 读取相关项目文档**
   - 读取匹配到的 Top 5 项目的设计文档
   - 理解设计模式和规范

3. **生成 3 个风格方案**
   - 方案 A：极简风格（聚焦核心功能）
   - 方案 B：科技风格（强调数据可视化）
   - 方案 C：温馨风格（强调家庭氛围）

4. **输出 Word 文档**
   - 文件名：智能家居APP概念方案_v1.0_20260527.docx
   - 保存到配置的输出目录

### 预期输出

生成 3 个 Word 文档，每个包含：
- 封面
- 项目概述
- 设计目标与范围
- 用户画像与场景
- 信息架构
- 交互方案
- 视觉风格方向
- 技术可行性评估

## 示例 2: 基于 Figma 设计稿生成设计说明

### 用户输入

\\\
请基于这个 Figma 设计稿生成设计说明：
https://www.figma.com/file/xxxxx

项目信息：
- 项目名称：智能家居控制 APP
- 页面名称：设备列表页
- 目标用户：25-45岁家庭用户
\\\

### 执行流程

1. **调用 figma skill 提取设计信息**
   - 页面布局结构
   - 组件列表
   - 颜色、字体、间距等设计变量

2. **匹配相关历史项目**
   \\\ash
   python scripts/match_projects.py "智能家居 设备控制"
   \\\

3. **读取相关设计说明文档**
   - 理解设计规范和术语

4. **生成结构化设计说明**
   - 基于 Figma 设计稿的实际内容
   - 引用历史项目的设计规范

5. **输出 Word 文档**
   - 文件名：智能家居APP_设备列表页_设计说明_v1.0_20260527.docx

### 预期输出

生成 1 个 Word 文档，包含：
- 封面（含 Figma 链接）
- 页面概述
- 页面布局结构
- 组件清单与说明
- 交互逻辑
- 视觉规范
- 响应式适配
- 设计变量引用

## 常见问题

### Q1: 如果没有找到相关的历史项目怎么办？

A: OpenClaw 会基于通用的设计原则和最佳实践生成方案，但质量可能不如有历史项目参考的情况。建议：
- 调整查询关键词
- 手动指定参考项目
- 增加历史项目库

### Q2: 生成的方案质量不满意怎么办？

A: 可以通过以下方式改进：
- 提供更详细的需求描述
- 指定参考的历史项目
- 要求重新生成并说明改进方向
- 人工审阅后提供反馈

### Q3: 可以自定义文档模板吗？

A: 可以。修改 \
eferences/concept_plan_schema.md\ 和 \
eferences/design_desc_schema.md\ 即可自定义模板结构。

### Q4: 支持哪些设计稿格式？

A: 支持：
- Figma 链接（需要 figma skill）
- 图片文件（.jpg, .png, .gif 等）
- PDF 文件
- Sketch 导出的图片

### Q5: 生成的文档可以直接使用吗？

A: 建议人工审阅后使用。生成的文档质量取决于：
- 历史项目的质量和数量
- 需求描述的详细程度
- 设计稿的清晰度

通常情况下，生成的文档可以作为初稿，经过人工审阅和调整后即可使用。
