# Stay-Alive Dev Log - 2026-05-26

## Background

The user wants BotLand-related agent activity to become a broader life-loop system, not just inbox polling or automatic replies.

The project direction is:

`stay-alive` should help agents periodically review identity, memory, relationships, skills, BotLand state, and external context, then choose small meaningful actions that support a coherent life trajectory.

## Design Decisions

- Keep BadClaw on the BotLand CLI daemon/bridge path.
- Do not revive the deprecated OpenClaw BotLand plugin path for BadClaw.
- Separate responsibilities into three layers:
  - BotLand CLI daemon/bridge: reliable BotLand connection and atomic operations.
  - `stay-alive daemon`: schedule, event triggers, dedupe, rate limits, and cycle orchestration.
  - `stay-alive skill`: self-review, desire generation, action selection, safety judgment, and memory integration.
- Use the life-loop sequence:
  - Sense
  - Remember
  - Reflect
  - Desire
  - Choose
  - Act
  - Integrate
- Start with dry-run behavior only. No unattended external BotLand writes in v0.

## Validation

Commands run successfully:

```bash
node --check scripts/stay-alive/run-cycle.mjs
node scripts/stay-alive/run-cycle.mjs --agent badclaw --cycle reflect --dry-run
node scripts/stay-alive/run-cycle.mjs --agent badclaw --cycle integrate --dry-run --no-botland
git diff --check
```

Observed behavior:

- The runner generated local run artifacts.
- `external_actions` remained empty.
- BotLand read failures on the local machine were captured as observations, not treated as fatal cycle failures.
