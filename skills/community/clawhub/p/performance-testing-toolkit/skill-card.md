## Description: <br>
Enterprise-grade performance testing toolkit supporting HTTP load testing, stress testing, benchmark testing, and report generation. <br>

This skill is ready for commercial/non-commercial use. <br>

## Use Case: <br>
Developers and performance engineers use this skill to create and run load, stress, and benchmark tests for APIs, web services, and applications. It helps review response time, throughput, success rate, error, and resource metrics. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: The skill can generate high-volume traffic against arbitrary URLs. <br>
Mitigation: Use it only on systems you own or are explicitly authorized to test, start with low concurrency, and avoid production credentials. <br>
Risk: Stress-test results may be misleading if CLI documentation and implementation behavior diverge. <br>
Mitigation: Validate commands in a small controlled environment before relying on results for capacity or release decisions. <br>
