## 产品创新生成要求

产品创新每种任务类型的"零件清单"——必选/可选组件、组装顺序、领域约束。按清单逐项产出。

---

### 1 创新机会

#### T1-1 趋势洞察

- **必选组件**: 洞察范围（市场/技术/用户/社会）、洞察方法（研究/观察/访谈）、趋势发现、创新启示
- **可选组件**: 趋势报告、专家访谈、行业会议
- **组装顺序**: 确定洞察范围 → 选择洞察方法 → 收集信息 → 分析趋势 → 提炼启示
- **约束**: 范围必须明确；方法必须多样；分析必须深入
- **格式**: 趋势洞察报告（Markdown）

#### T1-2 痛点挖掘

- **必选组件**: 挖掘对象（用户/客户/合作伙伴）、挖掘方法（访谈/观察/数据分析）、痛点清单、痛点优先级
- **可选组件**: 用户旅程图、痛点地图、用户故事
- **组装顺序**: 确定挖掘对象 → 选择挖掘方法 → 执行挖掘 → 整理痛点 → 排序优先级
- **约束**: 对象必须有代表性；方法必须有效；痛点必须真实
- **格式**: 痛点挖掘报告（Markdown）

#### T1-3 机会识别

- **必选组件**: 机会来源（趋势/痛点/技术）、机会描述、机会价值、机会可行性
- **可选组件**: 机会矩阵、机会评估、机会优先级
- **组装顺序**: 汇总机会来源 → 描述机会 → 评估价值 → 评估可行性 → 排序优先级
- **约束**: 来源必须可靠；描述必须清晰；评估必须客观
- **格式**: 创新机会清单（Markdown）

#### T1-4 机会评估

- **必选组件**: 评估维度（市场潜力/技术可行性/竞争壁垒/团队能力）、评估方法、评估结果、评估结论
- **可选组件**: 专家评审、市场调研、技术验证
- **组装顺序**: 确定评估维度 → 选择评估方法 → 执行评估 → 分析结果 → 形成结论
- **约束**: 维度必须全面；方法必须科学；结论必须明确
- **格式**: 机会评估报告（Markdown）

---

### 2 创新设计

#### T2-1 创意生成

- **必选组件**: 创新目标、创意方法（头脑风暴/SCAMPER/类比法）、创意数量、创意草图
- **可选组件**: 创意工作坊、跨领域灵感、创意评估
- **组装顺序**: 明确创新目标 → 选择创意方法 → 执行创意生成 → 记录草图 → 初步筛选
- **约束**: 目标必须明确；数量必须充足（≥50个）；方法必须多样
- **格式**: 创意生成记录（Markdown）

#### T2-2 概念验证

- **必选组件**: 验证目标、验证方法（原型/实验/调研）、验证结果、验证结论
- **可选组件**: MVP、用户测试、技术POC
- **组装顺序**: 确定验证目标 → 选择验证方法 → 执行验证 → 分析结果 → 形成结论
- **约束**: 目标必须明确；方法必须可行；结果必须客观
- **格式**: 概念验证报告（Markdown）

#### T2-3 商业论证

- **必选组件**: 市场规模、收入模式、成本结构、投资回报
- **可选组件**: 竞争分析、风险评估、财务模型
- **组装顺序**: 分析市场规模 → 设计收入模式 → 估算成本结构 → 计算投资回报 → 形成论证
- **约束**: 数据必须可靠；模式必须可行；回报必须合理
- **格式**: 商业论证报告（Markdown）

#### T2-4 创新立项

- **必选组件**: 立项依据、项目目标、项目计划、资源需求
- **可选组件**: 立项评审、利益相关者确认、风险预案
- **组装顺序**: 汇总立项依据 → 设定项目目标 → 制定项目计划 → 评估资源需求 → 获得批准
- **约束**: 依据必须充分；目标必须明确；计划必须可行
- **格式**: 创新立项文档（Markdown）

---

### 3 创新实施

#### T3-1 快速原型

- **必选组件**: 原型目标、原型类型（功能/交互/视觉）、原型工具、原型成果
- **可选组件**: 原型测试、原型迭代、原型文档
- **组装顺序**: 确定原型目标 → 选择原型类型 → 选择工具 → 制作原型 → 测试验证
- **约束**: 目标必须聚焦；类型必须匹配；速度必须快
- **格式**: 快速原型 + 原型说明（Markdown）

#### T3-2 用户验证

- **必选组件**: 验证目标、验证方法（测试/访谈/观察）、用户样本、验证结果
- **可选组件**: 验证脚本、验证环境、验证工具
- **组装顺序**: 确定验证目标 → 设计验证方法 → 招募用户 → 执行验证 → 分析结果
- **约束**: 目标必须明确；方法必须有效；样本必须有代表性
- **格式**: 用户验证报告（Markdown）

#### T3-3 迭代优化

- **必选组件**: 优化依据、优化目标、优化方案、优化效果
- **可选组件**: 优化优先级、优化计划、优化验证
- **组装顺序**: 分析优化依据 → 确定优化目标 → 设计优化方案 → 执行优化 → 验证效果
- **约束**: 依据必须充分；目标必须明确；效果必须可验证
- **格式**: 迭代优化记录（Markdown）

#### T3-4 创新验证

- **必选组件**: 验证指标、验证方法、验证数据、验证结论
- **可选组件**: 验证报告、验证评审、验证决策
- **组装顺序**: 确定验证指标 → 选择验证方法 → 收集验证数据 → 分析结果 → 形成结论
- **约束**: 指标必须可量化；方法必须科学；数据必须真实
- **格式**: 创新验证报告（Markdown）

---

### 4 创新落地

#### T4-1 产品化

- **必选组件**: 产品需求、产品设计、产品开发、产品测试
- **可选组件**: 产品规范、产品文档、产品培训
- **组装顺序**: 整理产品需求 → 完成产品设计 → 执行产品开发 → 进行产品测试 → 发布上线
- **约束**: 需求必须清晰；设计必须完整；质量必须达标
- **格式**: 产品化文档（Markdown）

#### T4-2 市场推广

- **必选组件**: 市场定位、定价策略、推广渠道、营销方案
- **可选组件**: 市场调研、竞品分析、营销预算
- **组装顺序**: 确定市场定位 → 制定定价策略 → 选择推广渠道 → 设计营销方案 → 执行推广
- **约束**: 定位必须准确；策略必须可行；渠道必须有效
- **格式**: 市场推广方案（Markdown）

#### T4-3 规模化

- **必选组件**: 增长目标、增长策略、增长指标、增长计划
- **可选组件**: 增长实验、增长团队、增长工具
- **组装顺序**: 设定增长目标 → 设计增长策略 → 确定增长指标 → 制定增长计划 → 执行增长
- **约束**: 目标必须明确；策略必须可行；指标必须可衡量
- **格式**: 规模化方案（Markdown）

#### T4-4 持续创新

- **必选组件**: 创新机制、创新文化、创新流程、创新评估
- **可选组件**: 创新团队、创新激励、创新平台
- **组装顺序**: 建立创新机制 → 培育创新文化 → 优化创新流程 → 评估创新效果 → 持续改进
- **约束**: 机制必须有效；文化必须支持；流程必须顺畅
- **格式**: 持续创新方案（Markdown）
