**旧称：** 端到端质量追溯系统  
**当前名称：** 供应商直连系统  
**业务场景：** 小米汽车供应链产能管理与质量追溯

**核心业务：**
- 管理 115 家供应商的产能数据
- 24 家试点供应商数据直连（2026 年 4 月底完成）
- 实现从 LCR 场地要求→SDC 签约产能→GP9 实测产能→设计产能的全流程管理

**业务价值：**
- 提升供应链透明度
- 实时监控产能达标情况
- 预警产能风险，保障生产计划

### 业务流程

**产能管理流程（4 阶段）：**
```
LCR 场地要求 → SDC 签约产能 → GP9 实测产能 → 设计产能
```

**核心指标监控（6 大 KPI）：**
1. 决策通过供应商数：115 家
2. SDC 已签约数：87 家（75.7%）
3. SDC 签约及时率：82%
4. GP9 已完成数：62 家（71.3%）

## 2. 豆包会话自动化

### 项目背景

**项目目标：** 自动从豆包读取会话历史，生成专家点评 HTML 网页，实现知识沉淀和学习提效

**业务场景：**
- 用户每天使用豆包进行学习和工作交流
- 会话内容包含大量有价值的知识点
- 需要自动归档和专家点评，形成可复用的知识库

**核心价值：**
- 知识沉淀：避免会话历史丢失
- 学习提效：专家点评帮助知识内化
- 可视化：HTML 专业排版 + Mermaid 架构图

### 业务流程

**4 步自动化流程：**
```
1. 监测触发（"豆包"开头）
   ↓
2. 会话归档（doubao-sessions/ + worklog.txt）
   ↓
3. 专家点评 HTML 生成（科学杂志 + 小红书风格）
   ↓
4. Chrome 自动打开预览 + 飞书推送
```

**文件归档规范：**
- 位置：`doubao-sessions/YYYY-MM-DD (周*) -N worklog.md`
- 专家点评：`html-expert-reviews/expert-review-YYYY-MM-DD-主题.html`
- 统一归档：`html-expert-reviews/` 文件夹

### 当前痛点

**会话语义丢失：**
- 豆包会话历史不互通（飞书 vs Dashboard）
- 会话内容分散，难以统一管理
- 缺乏结构化整理，知识难以复用

**知识内化困难：**
- 被动接受信息，缺乏深度思考
- 知识点孤立，缺乏系统关联
- 缺少专家视角点评和批判性分析

**技术限制：**
- 豆包 API 可能变更
- 会话获取成功率需>95%
- 专家点评生成时间需<30 秒

### IT 解决方案

**技能系统：**
- **doubao-expert-review Skill：** 豆包会话专家点评
- **html-expert-review Skill：** HTML 专家点评生成
- **tts-automation Skill：** TTS 语音自动播放

**脚本工具：**
- `fetch-doubao-history.ps1` - 从豆包读取会话历史
- `fetch-doubao-auto.ps1` - 自动获取豆包内容
- `fetch-doubao-simple.ps1` - 简化版获取脚本
- `save-doubao-automation.ps1` - 保存自动化配置
- `save-doubao-conversation.ps1` - 保存豆包会话

**HTML 标准：**
- 视觉风格：科学杂志排版 + 小红书图标
- 章节结构：6 个必备章节（专家评分/核心观点/深度洞察/知识架构/对比分析/行动建议）
- 图表：Mermaid 专业绘制（知识架构图、Gantt 图）
- 字体：微软雅黑 14px

## 3. 地理知识库

### 项目背景

**项目初衷（个人记忆管理）：**
- **习惯：** 记录去过的每个地方，在 Google Earth 上标记
- **规模：** 已有 500-600 个地标文件
- **目的：** 通过地图记忆，把做过的事情与地点关联，形成时空记忆索引
- **需求：** 结构化导入数据库，结合当地历史、新闻、人文知识

**项目愿景（社会性项目 - Colorful Life）：**
- **项目名称：** Colorful Life（后续核心项目）
- **定位：** 上帝视角的年轻人人生指引（社会性项目）
- **目标用户：** 广大年轻人（高校毕业生、初创者、职场新人）
- **核心功能：** AI 分析年轻人当前处境和需求，从地理维度指引机会
- **示例：** 某科技园创业政策扶持 → 引导年轻人了解

### 业务流程

**时空维度的记忆索引（核心架构）：**
```
地理场景 + 场景关联信息 + 时空印记
```

**索引结构：**
- **基础背景：** 地点基本信息（位置、类型、特征）
- **机会信息：** 创业政策、就业岗位、学习资源、交流活动
- **场景体验：** 基础背景 + 机会信息 + 场景体验（咖啡 + 创业政策）

**示例场景：**
