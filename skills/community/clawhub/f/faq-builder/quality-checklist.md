# FAQ Builder Quality Checklist

## Source Data Quality
- [ ] Support ticket data reviewed (minimum 30 tickets or 90 days of data)
- [ ] Return reason report pulled and top 3 reasons noted
- [ ] Competitor Q&A sections reviewed for category patterns
- [ ] Review mining completed (1-star and 2-star reviews scanned for questions)
- [ ] At least one tier-1 source used (not just assumptions)

## Question Coverage
- [ ] Compatibility / fit questions addressed
- [ ] Shipping and delivery addressed
- [ ] Return and refund policy clearly stated
- [ ] Product specs and materials answered with specific details
- [ ] Top 3 support ticket themes have dedicated FAQ entries
- [ ] Top 2 return reasons have preemptive FAQ entries
- [ ] 8–15 total questions (not fewer, not overwhelmingly more)

## Answer Quality
- [ ] Every answer contains a specific, direct response (no vague or non-answers)
- [ ] Answers include numbers, dimensions, or material names where relevant
- [ ] No answer is shorter than 2 sentences
- [ ] Tone matches brand voice
- [ ] No defensive or corporate-speak language
- [ ] ACES framework applied: Acknowledge → Clarify → Expand → Seal
- [ ] Answers don't create new unanswered questions

## Platform Compliance
- [ ] Answer length matches platform character limits
- [ ] Format matches platform (accordion, plain text, A+ module)
- [ ] Keywords included naturally in 30–40% of answers
- [ ] FAQPage schema markup added (Shopify/WooCommerce)
- [ ] FAQ placement is above the fold or in a prominent section

## Sequence and Organization
- [ ] Purchase-blocker questions appear first (compatibility, returns, shipping)
- [ ] Questions grouped by topic or follow buyer journey order
- [ ] No company background / brand story questions in product FAQ
- [ ] Most important information leads each answer

## Maintenance Setup
- [ ] Support ticket spike alert configured (>30% increase = review trigger)
- [ ] Quarterly review date set in calendar
- [ ] Return analysis trigger: new batch → check if FAQ-addressable
- [ ] Version/update log started for future reference
