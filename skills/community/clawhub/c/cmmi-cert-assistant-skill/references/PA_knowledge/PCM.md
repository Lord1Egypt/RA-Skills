# PCM — 过程质量管理（CMMI V3.0 Core PA）

## PA 基本信息
- **缩写**：PCM
- **英文**：Process Quality Management
- **中文**：过程质量管理
- **所属**：Core PA（核心实践域）
- **对应 V2.0**：OPP（组织过程性能）+ QPM（量化项目管理）的部分内容

---

## CMMI V3.0 要求要点

### 核心目的
通过量化分析和管理过程质量，确保过程能够稳定地交付预期结果。

### 关键实践（Specific Practices）
| 实践编号 | 实践内容 | 说明 |
|---------|---------|------|
| PCM 1.1 | 建立过程性能基线 | 基于历史数据建立过程能力基线 |
| PCM 1.2 | 识别过程质量因素 | 识别影响过程质量的关键因素 |
| PCM 1.3 | 建立过程性能模型 | 建立过程行为与结果的关联模型 |
| PCM 2.1 | 管理过程性能 | 监控过程性能，识别异常 |
| PCM 2.2 | 量化管理项目过程 | 使用统计方法管理项目过程 |
| PCM 3.1 | 持续改进过程性能 | 基于数据分析优化过程 |

### 通用实践（Generic Practices）
- GP 1.1 执行特定实践
- GP 2.1 建立组织政策
- GP 2.2 策划过程
- GP 2.3 提供资源
- GP 2.4 分配责任
- GP 2.5 培训人员
- GP 2.6 管理配置
- GP 2.7 识别并记录干系人需求和接口
- GP 2.8 监督和控制过程
- GP 2.9 客观评估符合性
- GP 2.10 高层审查

## 用户文档 → PA 匹配规则

### 高概率匹配（主要PA）
- 文档名含"过程性能"、"质量指标"、"统计控制" → **PCM 主要匹配**
- 文档含控制图、过程能力分析 → **PCM 主要匹配**

### 中概率匹配（次要PA）
- 文档含缺陷密度趋势 → 同时匹配 **VV**（验证与确认）
- 文档含估算模型 → 同时匹配 **EST**（估算）

## 差距检查清单（用于差距分析）

### PCM 1.x — 过程性能建立
- [ ] 是否有组织级过程性能基线（PPB）？
- [ ] 是否有过程性能模型（PPM）？
- [ ] 是否识别了关键质量子过程？
- [ ] 是否有历史数据支撑（至少6-10个项目）？

### PCM 2.x — 过程性能管理
- [ ] 项目是否使用了量化管理方法？
- [ ] 是否有过程性能监控记录（控制图）？
- [ ] 是否定义了过程异常阈值？

### PCM 3.x — 过程性能改进
- [ ] 是否有基于数据的改进建议？
- [ ] 改进措施是否有跟踪记录？

## 评估访谈常见问题（模式D）

### 针对EPG（过程改进组）的问题
1. "组织级过程性能基线是如何建立的？有哪些指标？"
2. "过程性能模型有哪些？如何验证其有效性？"
3. "如何保证过程性能数据的准确性？"
4. "如何基于过程性能数据推动改进？"

### 针对PM（项目经理）的问题
1. "你的项目使用了量化管理方法吗？具体如何做的？"
2. "你如何知道项目过程是受控的？"
3. "你使用过组织级的过程性能基线吗？"

### 评分要点
- **PCM 1.1-1.3**：是否有系统的过程性能基线和模型
- **PCM 2.1-2.2**：项目是否实际应用了量化管理
- **PCM 3.1**：是否有基于数据的持续改进
