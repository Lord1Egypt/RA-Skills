# reading-profile — 阅读画像

组合行为层数据和驱动层采集，生成穿透「读了什么」抵达「阅读如何塑造这个人」的用户画像。
行为层 = 可采集的阅读数据（读了什么、多久、多少）。驱动层 = 阅读如何塑造了决策、审美、世界观。

## 数据源

| 层 | 来源 | 采集方式 |
|---|---|---|
| 行为 | readdata(overall) | API |
| 行为 | shelf + progress | API |
| 行为 | 本地阅读档案 CSV/md | Read |
| 驱动 | MEMORY context | Read |
| 驱动 | 非微信读书渠道覆盖 | AskUser |
| 驱动 | 阅读驱动力类型 | AskUser |
| 驱动 | 转折书 + 对应决策转变 | AskUser |
| 驱动 | 阅读与生活角色的关系 | AskUser |
| 驱动 | 画像服务场景 | AskUser |

MEMORY 材料优先于 AskUser。已覆盖的维度跳过对应提问。

## AskUser 一次采集

1. 渠道：微信读书之外的阅读渠道和体量 → 判断 WeRead 覆盖率
2. 内驱力：读完一本书最在意的收获 → 框架/模型 | 审美体验 | 行动指南 | 世界观校准（多选）
3. 转折书：哪本书改变了某个重大决策方式？书名 + 改变了什么（开放输入，一次问清）→ 驱动层硬证据
4. 角色关系：阅读和核心生活角色的关系 → 功能定位

## 输出

必选四维度 + 鼓励根据数据丰富度自由扩展：

一个判断句标题**定位读者**

**阅读性格** 从行为推断（重读频次、笔记密度、阅读时段、单次时长），每个特征附证据。

**精神脉络** 按问题意识切分，允许横跨品类。至少一条来自驱动层数据。

**空白区域** 已有脉络的自然延伸方向，每个附具体入口书名，标注「未遇到入口」或「无兴趣」。

可选维度举例：品类 DNA（表格化行为层统计）、驱动层档案（转折书、角色关系，仅「自我认知」场景）。数据充分时自行判断增补。

## 规则

- 微信读书覆盖不全时标注数据口径
- 语言利落，断言直给；用文学笔触书写分析，让洞察自带温度
- 反本质主义：描述读者在具体条件下的功能和偏好，不做本质归因
- 阅读画像做为可复利资产，存为本地 md
- negative(em-dash, not X but Y)

## 迭代触发

- 生成推荐后：将实际阅读反馈回流画像
- 用户报告新转折书：追加驱动层档案
- 空白区域开始涉足：升级为新脉络
