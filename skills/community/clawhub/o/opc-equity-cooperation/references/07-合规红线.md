# 合规红线：非法集资防范、敏感词对照表

> 本文件详细阐述OPC分布式协作中台必须遵守的合规底线，包括非法集资防范、敏感词对照表、风险加固措施。

---

## 一、合规底线声明

> **本体系的运营必须严格遵守国家法律法规，任何情况下都不得触碰合规红线。**

## 二、非法集资防范

### 2.1 非法集资的法律定义

根据《最高人民法院关于审理非法集资刑事案件具体应用法律若干问题的解释》，非法集资需同时具备以下四个特征：

| 特征 | 说明 | OPC场景说明 |
|------|------|------------|
| **非法性** | 未经有关部门依法批准或者借用合法经营的形式吸收资金 | OPC项目不向社会公开募集资金 |
| **公开性** | 通过媒体、推介会、传单、手机短信等途径向社会公开宣传 | OPC在小圈子内运作，不对外公开 |
| **利诱性** | 承诺在一定期限内以货币、实物、股权等方式还本付息或者给付回报 | OPC不承诺任何固定收益 |
| **社会性** | 向社会公众即社会不特定对象吸收资金 | OPC只针对特定合作方 |

### 2.2 绝对禁止行为

| 禁止行为 | 正确做法 |
|---------|---------|
| ❌ 向不特定对象公开募集资金 | ✅ 只对接特定的已有合作基础的人 |
| ❌ 承诺固定收益或保本 | ✅ 只说按实际项目收益分配，风险自担 |
| ❌ 平台归集资金形成资金池 | ✅ 资金走第三方托管或共管账户 |
| ❌ 层级裂变、拉人头返利 | ✅ 每个项目独立核算，不搞层级 |
| ❌ 发行代币、积分分红 | ✅ 不用任何代币化，只做合同约定 |
| ❌ 对外公开招募"投资者" | ✅ 只招募"合作方"或"参与者" |
| ❌ 承诺任何形式的"本金保障" | ✅ 明确告知投资有风险 |
| ❌ 虚假宣传高收益 | ✅ 如实描述，不夸大 |

### 2.3 合规的资金来源

| 资金类型 | 是否允许 | 说明 |
|---------|---------|------|
| 项目回款 | ✅ | 正常的业务收入 |
| 合作方服务费 | ✅ | 正常的服务收入 |
| 平台服务费 | ✅ | 按约定比例收取 |
| 政府补贴 | ✅ | 合法合规的政府补贴 |
| ❌ 社会公开募资 | ❌ | 绝对禁止 |
| ❌ 投资者"投资款" | ❌ | 绝对禁止 |
| ❌ 不特定对象的资金 | ❌ | 绝对禁止 |

## 四、内容合规检查

### 4.1 宣传材料检查清单

```
【必查项】
□ 是否有"保本"、"稳赚"、"100%收益"等承诺
□ 是否有"投资"、"回报率"、"年化"等投资类词汇
□ 是否有"股权融资"、"上市"、"IPO"等内容
□ 是否有"拉人头"、"层级返利"等传销特征
□ 是否对外承诺任何形式的固定收益

【建议检查项】
□ 收益描述是否有"预期"、"参考"等限定词
□ 是否明确"风险自担"原则
□ 是否有免责声明
□ 数据来源是否可靠
□ 是否有夸大宣传
```

### 4.2 对外沟通话术规范

| 场景 | ❌ 禁止说法 | ✅ 正确说法 |
|------|----------|-----------|
| 招募合作方 | "投资我们的项目，保证回报" | "欢迎参与合作，收益按项目实际分配" |
| 描述收益 | "年化收益20%" | "参考历史项目，预期收益可观" |
| 描述风险 | "零风险" | "项目有风险，请谨慎评估" |
| 描述退出 | "随时可以退出拿回本金" | "按协议约定方式退出" |
| 描述前景 | "三年内上市" | "做大做强，争取更好发展" |

## 五、风险加固措施

### 5.1 三大风险点加固

#### 风险点一：非法集资触碰

```
加固措施：
├── 绝对禁止任何"固定收益""保本"承诺
├── 所有分配只写"按实际利润分配"
├── 只在小圈子内运作，不对外公开募资
├── 不搞层级裂变、不搞拉人头
└── 所有资金只接受项目回款，不接受"投资款"
```

#### 风险点二：税务风险

```
加固措施：
├── 所有款项性质都是"服务费""采购款"
├── 每一笔都要有发票对应
├── 牵头方完税凭证公开透明
├── 各成员自行申报个人所得税
└── 系统内置税务计算器，自动算税
```

#### 风险点三：牵头方道德风险

```
加固措施：
├── 账户双人U盾分离保管
├── 所有操作短信通知全员
├── 每日自动对账
├── 监事随时查账
├── 信用分约束，违约成本高
└── 区块链存证，不可篡改
```

## 六、合规审查机制

### 6.1 合规审查触发点

| 审查节点 | 审查内容 | 审查人 |
|---------|---------|-------|
| 项目立项 | 项目合规性审查 | 合规长老 |
| 合作协议签署 | 合同内容合规审查 | 合规长老 |
| 对外宣传材料 | 敏感词检查 | 合规长老 |
| 收益分配 | 分配合规性审查 | 合规长老 |
| 成员加入 | 成员资质审查 | 合规长老 |
| 重大决策 | 决策合规性审查 | 合规长老 |

### 6.2 合规审查流程

```
合规审查流程：

1. 【发起审查】相关方提交审查申请
        ↓
2. 【资料准备】准备相关材料
        ↓
3. 【合规审查】合规长老进行审查
        ↓
4. 【出具意见】出具书面审查意见
        ↓
5. 【问题整改】如有合规问题，整改后复审
        ↓
6. 【审查通过】通过后继续执行
        ↓
7. 【档案归档】审查资料归档保存
```

## 七、违规处罚机制

### 7.1 违规行为分级

| 级别 | 行为类型 | 处罚措施 |
|------|---------|---------|
| **轻微** | 敏感词使用不当、宣传夸大 | 警告、修改内容 |
| **一般** | 未经批准对外合作、违反流程 | 扣信用分10-30分、暂停合作 |
| **严重** | 承诺保本收益、对外募资 | 扣信用分50分、降至D级、除名 |
| **违法** | 涉嫌非法集资、诈骗 | 立即终止合作、移送司法机关 |

### 7.2 违规处理流程

```
违规处理流程：

1. 【发现违规】系统检测/人工举报
        ↓
2. 【调查取证】收集违规证据
        ↓
3. 【出具报告】形成违规调查报告
        ↓
4. 【告知当事人】告知违规事实和处罚
        ↓
5. 【申诉处理】当事人可申诉（3天内）
        ↓
6. 【执行处罚】申诉期满或申诉驳回后执行
        ↓
7. 【公示通报】全员通报处理结果
        ↓
8. 【上链存证】违规记录上链存档
```

## 八、合规敏感词使用指南

### 8.1 日常运营话术规范

| 场景 | 建议话术 |
|------|---------|
| 招募合作方 | "我们正在招募项目合伙人，一起合作共赢" |
| 描述合作模式 | "按贡献权重分配收益，风险共担" |
| 描述项目前景 | "参考同类项目，预期收益情况良好" |
| 描述风险 | "项目存在不确定性，请充分评估后再决定" |
| 描述退出 | "合作期满或项目结束后，按协议约定进行清算" |
| 描述收益 | "历史项目收益仅供参考，不构成任何承诺" |

### 8.2 协议表述规范

| 条款类型 | 规范表述 |
|---------|---------|
| 收益条款 | "各方按贡献权重比例分配项目收益，收益取决于项目实际经营情况" |
| 风险条款 | "本项目存在风险，各方应充分认识并自愿承担" |
| 退出条款 | "退出时按届时实际贡献情况进行清算" |
| 保底条款 | 删除任何形式的保底承诺 |
