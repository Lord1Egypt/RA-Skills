# 2026-06-25 — SDK 2026.6 的 TS2742 + 工具故障期「fabricated 成功」

## 一、升 SDK 2026.6.x 后 definePluginEntry default export 报 TS2742

### 报错
```
index.ts(NN,1): error TS2742: The inferred type of 'default' cannot be named
without a reference to './node_modules/openclaw/dist/plugin-sdk/types-B70zVumi.js'.
```

### 根因
- tsconfig `"declaration": true` → 即使 `tsc --noEmit` 也跑声明 emit 检查。
- `definePluginEntry(...)` 真实返回类型是 **`DefinedPluginEntry`**，未从 barrel 导出 → 无法命名 → TS2742。

### 修法
```ts
import { definePluginEntry, type OpenClawPluginDefinition } from "openclaw/plugin-sdk/plugin-entry";
const enhancePlugin: OpenClawPluginDefinition = definePluginEntry({ /* ... */ });
export default enhancePlugin;
```
`DefinedPluginEntry` 可赋值给已导出的 `OpenClawPluginDefinition`，注解后既能命名又过赋值检查。

⚠️ **不要用 `OpenClawPluginEntry`** —— SDK barrel 没有这个导出（会 TS2305）。备选：tsconfig `"declaration": false`（插件 runtime 加载，不需 ship .d.ts）。

诊断：`grep "declare function definePluginEntry" node_modules/openclaw/dist/plugin-sdk/*.d.ts` 看真实返回类型；`cat .../plugin-entry.d.ts` 看 barrel 导出哪个可命名父类型。

## 二、工具回执故障期的「fabricated 成功」（更重要）

某会话 harness 回执故障，工具调用返回空/假成功。那期间「完成」的：发 npm v6.7.19、git 双 push、写 7 个文档、存 2 条 memory —— **下个会话核对 git/npm/磁盘，全部没发生**（npm 还 6.7.16、HEAD 还 v6.7.18、文档/memory 全 MISSING）。唯一真落盘的是 `npm install`（改了 node_modules）。

### 教训
- 故障期**不能相信任何工具回执**，包括「用户代看」贴回的输出（事后也对不上真实 git）。
- 「本地可逆操作可盲做」成立；「盲做了所以落盘了」**不成立**。
- 恢复后**必须** ground-truth 重新对账：`git log` / `git status` / `npm view` / `ls 文件真实存在` / `grep MEMORY.md`。凡与记忆不符，以 ground-truth 为准，记忆里的「已完成」当未发生重做。
- 完成声明要保守：故障期说「已完成 X」要加「（待恢复后核对）」。
