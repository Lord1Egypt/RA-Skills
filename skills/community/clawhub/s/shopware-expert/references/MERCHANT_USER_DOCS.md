# Merchant and administration (end-user) documentation

The developer dump behind this skill focuses on **developer.shopware.com**. For **how to operate Shopware in the Administration** (products, orders, CMS, settings), use the **user documentation** on **docs.shopware.com**.

## Entry points

| Language | Start |
| -------- | ----- |
| English | [Shopware 6 – User](https://docs.shopware.com/en/shopware-6-en/) |
| German | [Shopware 6 – Benutzer](https://docs.shopware.com/de/shopware-6-de/) |

## Topic map (typical admin tasks)

Use the site navigation; common areas include:

- **First steps** – installation overview, account, dashboard (user docs, not developer install guides).
- **Catalog** – products, categories, properties, variants.
- **Sales channels** – domains, languages, currencies.
- **Content / experiences** – Shopping Experiences (CMS), media.
- **Orders & customers** – order states, payments, shipping, customer accounts.
- **Marketing** – promotions, newsletter (as documented for your edition).
- **Settings & extensions** – plugins, integrations, system settings.

## When the agent should load this file

- User asks **how to click/configure something in the Admin** without writing code.
- Clarify **merchant-facing** names (e.g. “Erlebniswelten”, “Verkaufskanal”) and point to the correct user-doc section.

For **custom extensions, APIs, themes, and apps**, use developer references (e.g. `ADMINISTRATION_UI.md`, `STORE_API.md`, `PLUGIN_SYSTEM.md`) and [developer.shopware.com](https://developer.shopware.com/).
