## Description: <br>
Frontend UI workflow for turning vague ideas, UI drafts, and feedback into buildable frontend plans and iteration steps. <br>

This skill is ready for commercial/non-commercial use. <br>

## Publisher: <br>
[siegelioner](https://clawhub.ai/user/siegelioner) <br>

### License/Terms of Use: <br>
MIT <br>


## Use Case: <br>
External users, product teams, designers, and frontend developers use this skill to turn vague page, app, or product ideas into UI briefs, design direction, build plans, review plans, and iteration steps. It is especially useful before frontend implementation or when an early UI draft needs a prioritized next pass. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: The skill may create or update DESIGN.md when frontend files are created or modified, which can affect repositories with strict documentation or change-control requirements. <br>
Mitigation: Review proposed file changes before applying them and confirm documentation update requirements for the target repository. <br>
