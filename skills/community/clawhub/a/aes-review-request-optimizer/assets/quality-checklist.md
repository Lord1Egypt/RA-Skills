# Review Program Quality Checklist

Run through this list before launching the program. Any "no" is a blocker.

## Audit complete
- [ ] Review rate per SKU calculated
- [ ] Star average per SKU calculated
- [ ] Median time-to-review measured
- [ ] Current templates inventoried
- [ ] Compliance risks flagged

## Customer-experience timing
- [ ] SKU groups defined
- [ ] First-use window estimated per group
- [ ] Opinion-formed window estimated per group
- [ ] Ask trigger documented per group

## Channels
- [ ] Platform-native request enabled
- [ ] In-package insert designed
- [ ] Email program built with correct triggers
- [ ] SMS program (if used) opt-in verified
- [ ] No channel stacking violates 2-asks-in-7-days rule

## Segmentation
- [ ] First-time vs repeat segments separated
- [ ] AOV bands defined
- [ ] New-launch cohort flagged
- [ ] Subscription renewal cohort separate (if applicable)

## Messaging compliance
- [ ] No rating-direction language anywhere
- [ ] No incentive-tied-to-review language anywhere
- [ ] No gating (intercept goes to support, not away from reviews)
- [ ] Product referenced by name in every template
- [ ] Negative-path link present in every channel

## Operational
- [ ] Support team aware of new intercept volume
- [ ] Support SLA agreed
- [ ] Reviewer-status CRM tags configured
- [ ] Reporting dashboard live before launch

## Measurement
- [ ] Baseline metrics frozen
- [ ] A/B test queue defined
- [ ] Cadence for review (weekly / monthly)
- [ ] Stopping criteria for failing tests defined

## Sign-off
- [ ] Brand owner approves messaging
- [ ] Legal/compliance reviewed for marketplace policy
- [ ] Customer-care lead signs off on intercept volume
