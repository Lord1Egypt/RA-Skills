```
1️⃣ 监测触发 → 2️⃣ 会话归档 → 3️⃣ 专家点评 → 4️⃣ 知识库更新
```

### 步骤 1：监测触发

**检测条件：**
- 消息开头两个字是"豆包"
- 自动触发，无需确认

**实现逻辑：**
```powershell
if ($message -match "^豆包") {
    Invoke-Skill -name "DOUBAO-001" -content $message
}
```

### 步骤 2：会话归档

**功能：** 保存豆包会话原始内容到本地

**输出文件：** `doubao-sessions/YYYY-MM-DD (周*) -N worklog.md`

**命名规则：**
- 格式：`YYYY-MM-DD (周*) -N worklog.md`
- 序号 N 自动递增（同一天多次会话）
- 完整保存原始内容，不删减

**文件结构：**
```markdown
# 豆包会话归档

**日期：** 2026-03-07 (周六)
**序号：** -1
**原始时间：** 2026-03-07 16:50
**会话主题：** [自动提取]

---

## 会话内容

[用户粘贴的豆包会话原始内容]

---

_自动归档于 2026-03-07 16:50_
```

**原子动作：**
- ATOM-DATA-001（保存豆包会话）
- ATOM-DATA-002（更新 worklog.txt）

### 步骤 3：专家点评

**功能：** 生成专家级点评 HTML 网页

**输出文件：** `expert-review-YYYY-MM-DD-主题.html`

**HTML 结构（金字塔原理）：**

1. **专家评分** - 完整性/正确性/缺失项（百分比 + 评语）
2. **核心观点** - 结论先行，Critical Thinking
3. **深度洞察** - ≥3 个大观点，MECE 法则
4. **知识架构** - Mermaid 图表
5. **对比分析** - 表格或对比图
6. **行动建议** - 可落地的建议

**视觉风格：**
- 科学杂志 + 小红书视觉
- Nature 杂志排版，微软雅黑字体
- Mermaid 知识架构图
- 专业配色和布局

**交付动作：**
- Chrome 自动打开 HTML（0 步操作）
- 飞书发送文字消息（含 HTML 链接或飞书文档链接）
- ❌ **取消 TTS 语音**（2026-03-07 17:45 起，仅每小时提醒使用）

**原子动作：**
- ATOM-ANALYSIS-017（专家视角分析）
- ATOM-VISUAL-007（生成专家评分）
- ATOM-VISUAL-006（生成 Mermaid 图表）
- ATOM-IO-019/020/021（HTML 模板处理）
- ATOM-VISUAL-009（Chrome 打开文件）
- ATOM-DELIVERY-010（飞书发送文字）
- ❌ ~~ATOM-VISUAL-008（生成 TTS 语音）~~ - 已移除
- ❌ ~~ATOM-DELIVERY-011（飞书发送语音）~~ - 已移除
