## Description: <br>
Helps developers design, orchestrate, and deploy multi-agent collaboration systems with Model Context Protocol (MCP) tool integration, agent communication protocols, task decomposition strategies, and framework selection across LangChain, AutoGen, CrewAI, OpenClaw, and Dify. <br>

This skill is ready for commercial/non-commercial use. <br>

## Use Case: <br>
Developers and engineers use this skill to plan agent architectures, MCP tool integrations, task-routing protocols, and implementation scaffolds for single-agent through multi-agent swarm systems. It is also useful for comparing agent frameworks and estimating cost, latency, observability, and safety considerations before implementation. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Generated architecture, framework, or MCP integration guidance may be incomplete, outdated, or unsuitable for a specific production environment. <br>
Mitigation: Review recommendations against current framework documentation, system requirements, security policies, and operational constraints before implementation. <br>
Risk: Generated tool schemas, manifests, and code snippets could grant excessive permissions or omit validation and audit controls. <br>
Mitigation: Apply least privilege, validate all inter-agent messages with schemas, review tool permissions, and test generated configurations in a sandbox before deployment. <br>
Risk: Cost, token, latency, and benchmark estimates may not match the user's models, workloads, or infrastructure. <br>
Mitigation: Measure estimates with representative workloads and set budget, concurrency, timeout, and monitoring controls before production use. <br>
