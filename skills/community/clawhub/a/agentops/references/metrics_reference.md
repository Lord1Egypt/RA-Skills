# 性能指标说明

本文档说明 AgentOps 使用的各项性能指标及其含义。

## 系统指标

### CPU使用率

- **范围**: 0-100%
- **来源**: /proc/stat
- **健康阈值**: <50%
- **警告阈值**: 50-80%
- **危险阈值**: >80%

### 内存使用率

- **范围**: 0-100%
- **来源**: /proc/meminfo (MemTotal, MemAvailable)
- **健康阈值**: <50%
- **警告阈值**: 50-85%
- **危险阈值**: >85%

### 磁盘使用率

- **范围**: 0-100%
- **来源**: os.statvfs("/")
- **健康阈值**: <70%
- **警告阈值**: 70-90%
- **危险阈值**: >90%

### 系统负载 (Load Average)

- **范围**: 0+ (无上限)
- **来源**: /proc/loadavg
- **含义**: 正在运行或等待CPU的进程数
- **健康阈值**: <CPU核心数
- **警告阈值**: CPU核心数 ~ 2×核心数
- **危险阈值**: >2×核心数

## OpenClaw指标

### 进程状态

- Gateway进程是否存在
- Node.js运行时是否可用

### 会话状态

- Workspace是否存在
- 关键文件是否存在 (SOUL.md, USER.md, AGENTS.md, MEMORY.md)
- Memory目录状态

### 配置状态

- 配置文件是否存在
- JSON格式是否有效
- 关键配置项是否完整

## 告警指标

告警管理器支持的指标：

| 指标 | 单位 | 默认阈值 | 默认操作符 |
|------|------|----------|-----------|
| cpu | % | 90 | gt |
| memory | % | 85 | gt |
| disk | % | 80 | gt |
| load | 数值 | 4 | gt |

## 评分标准

性能报告健康评分 (0-100)：

| 扣分项 | 扣分 |
|--------|------|
| CPU > 90% | -30 |
| CPU > 70% | -10 |
| 内存 > 90% | -30 |
| 内存 > 70% | -10 |
| 磁盘 > 90% | -25 |
| 磁盘 > 80% | -10 |
| 负载 > 8 | -15 |
| 负载 > 4 | -5 |

评分等级：
- **80-100**: 健康 (healthy) ✅
- **50-79**: 警告 (warning) ⚠️
- **0-49**: 严重 (critical) 🔴
