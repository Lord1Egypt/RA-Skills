# data-analyst — 训练数据分析

## 触发场景
- 用户发送训练数据/报告："帮我看看训练数据""我的训练报告"
- 询问训练质量："我怎么练得怎么样""肌群容量分布合理吗"
- App数据报告粘贴到对话中

## 操作规范

### 1. 识别数据来源
- **App报告**：含"训练数据报告""训练密度""每肌群组数""恢复状态"等段落 → 按App指标体系分析
- **口头描述**：用户通过对话告知训练情况 → 基于描述进行定性分析

### 2. 分析维度

| 维度 | 说明 | 关注点 |
|------|------|--------|
| 总训练量趋势 | 组×次×重量之和 | 上升趋势=进步，下降=需关注 |
| 训练密度 | 组数/分钟 | 衡量训练效率，参考值1.5-3组/分钟 |
| 肌群容量分布 | 各肌群训练量占比 | 发现不平衡（某肌群占比过高或过低） |
| 每肌群组数 vs MEV | 实际组数/周MEV(10组) | ✅≥1.0 ⚠️≥0.5 🔴<0.5 |
| 估算1RM | Mayhew公式 TOP10 | 力量评估与渐进超负荷依据 |
| 恢复状态 | 各肌群休息天数 | ✅可练 ⚠️建议再休息 🔴今日刚练 |

### 3. App报告专属指标解读
- **MEV状态标记**：✅训练量充足 → 保持；⚠️偏低 → 关注并考虑补量；🔴严重不足 → 必须增加
- **恢复状态标记**：✅可训练 → 可安排；⚠️建议再休息N天 → 排后面；🔴今日刚训练 → 不安排
- **趋势箭头**：↑>10%增长积极，↓>10%下降需警惕

### 4. 输出规范
- **分析结论**：先总结整体状态（进步/持平/退步）
- **具体发现**：按肌群列出关键数据（用✅⚠️🔴标记）
- **可执行建议**：指向具体调整方向（如"背阔肌组数偏低，建议增加一个划船动作"）
- **不改计划**：建议用户确认后触发 plan-generator 生成新计划
