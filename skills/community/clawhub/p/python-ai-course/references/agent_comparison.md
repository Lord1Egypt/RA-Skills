## 四、核心技术路线对比

### 闭源 API vs 开源自建

| 决策维度 | 闭源 API（OpenAI/Claude/Gemini） | 开源自建（LLaMA/Qwen/DeepSeek） |
|----------|--------------------------------|-------------------------------|
| **前期成本** | 极低（API Key 即用） | 高（GPU 服务器+运维） |
| **长期成本** | 随调用量线性增长 | 固定硬件+边际递减 |
| **模型能力** | 最新 SOTA | 快速追赶中（差距 5-10%） |
| **数据安全** | 数据需出域 ⚠️ | 完全可控 ✅ |
| **定制灵活度** | 受限（仅 Prompt+微调） | 完全自由 |
| **适用** | 快速验证、非敏感数据、中小规模 | 大规模、高安全、深度定制 |

### 低代码 vs 代码框架

| 维度 | 低代码（Coze/Dify/千帆） | 代码框架（LangChain/LangGraph） |
|------|------------------------|-------------------------------|
| 上手速度 | 分钟-小时 | 天-周 |
| 灵活性 | 平台天花板限制 | 无上限 |
| 复杂逻辑 | 困难（可视化混乱） | 原生支持 |
| 可维护性 | 依赖平台 | 代码即资产（Git/CI） |
| 适合 | 原型验证、简单工作流 | 生产系统、复杂业务 |

## 六、四种典型场景推荐

### 场景 1：个人开发者（预算 < ¥500/月）
- **模型**：DeepSeek API（主力）+ Claude 3.5 Sonnet（代码用）
- **框架**：LangChain（入门最快）
- **平台**：Dify Docker 自部署
- **理由**：零前期投入、开源为主、学习曲线平缓

### 场景 2：中小团队（5-20 人，预算 ¥2K-10K/月）
- **模型**：Qwen2.5 14B 自部署（日常）+ Claude API（复杂任务）
- **框架**：LangGraph + Dify（团队 UI）
- **监控**：LangFuse（开源免费）
- **理由**：数据不出域、成本可控、非技术成员友好

### 场景 3：企业级（100+ 人，预算不限）
- **路由**：Haiku（简单）→ GPT-4o（中等）→ o1（复杂）
- **框架**：LangGraph + Semantic Kernel
- **基础设施**：vLLM → Milvus → Kafka → Redis → PostgreSQL
- **可观测**：LangSmith + Prometheus + Grafana + ELK
- **理由**：高可用 99.9%+、全链路可观测、模型路由降本 50-70%

### 场景 4：金融/医疗/政务（高合规）
- **模型**：国产合规模型全私有化、物理隔离
- **框架**：全自研或 LangChain 私有化
- **安全**：Guardrails + 审计 + 脱敏 + 敏感词过滤
- **硬件**：国产 GPU（华为昇腾）或 Nvidia 合规渠道
- **理由**：数据不出域是法律底线、全链路可控可审计

## 八、2026-2027 趋势预判

1. **MCP 成行业标准**：类似 HTTP 之于 Web，所有主流模型和工具将支持 MCP
2. **推理模型驱动 Agent 决策**：o1/R1 成 Agent「大脑」，决策质变
3. **多 Agent 进入生产期**：2025 年 POC，2026-2027 年真正企业级落地
4. **国产方案闭环**：Qwen/DeepSeek + Dify/Coze 可满足 80% 以上企业需求
5. **Agent 即服务（AaaS）**：Agent 成独立产品形态，按需订阅
6. **安全合规优先**：Agent 落地前先定合规框架，而非事后补救

## 教学使用指南

1. **先画图再讲字**：`show_widget` 绘制决策树和产品矩阵图
2. **引导自评打分**：让学员先填五维度评分卡
3. **案例驱动**：每个场景配 1 个真实匿名化企业案例
4. **教选型思维**：核心不是记答案，是学会权衡方法
5. **提醒时效**：Agent 行业 6-12 个月一代，结论需定期重评
