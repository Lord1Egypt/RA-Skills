## Description: <br>
Agentic software development methodology for disciplined AI-assisted development across brainstorming, planning, TDD, debugging, code review, git worktrees, and branch management. <br>

This skill is ready for commercial/non-commercial use. <br>

## Use Case: <br>
Developers and agent operators use this skill to guide an AI coding assistant through structured software-development workflows, including planning, implementation, testing, debugging, review, and branch integration. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: The skill uses strict process-enforcement language that can steer agents toward repository changes, commits, subagents, and worktree actions even for simple tasks. <br>
Mitigation: Install only when strict development workflow enforcement is desired, and require explicit confirmation before commits, pushes, dependency installs, worktree cleanup, or subagent delegation. <br>
