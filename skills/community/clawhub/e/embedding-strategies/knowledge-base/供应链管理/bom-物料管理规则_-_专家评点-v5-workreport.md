委外 BOM 物料管理规则 - 专家深度评点

@Thomas @供应链团队

【评分】70/100

【目标】
通过系统性评估委外 BOM 规则，识别关键缺陷与风险，
避免 ERP 配置错误、明确责任边界、优化成本结构，
预计可降低 15-20% 库存资金占用

【核心发现】

1. 严重错误：C 件定义错误
   - 原文：CSMT 零件 = 采购级零件
   - 纠正：CSMT 零件 = 委外子件（物权归供应商）
   - 业务影响：
     （1）ERP 系统会为 C 生成采购需求计划
     （2）重复计算导致库存积压（C 价值已包含在 HC 中）
     （3）预计增加 15-20% 库存资金占用
   - 建议：立即修正 C 件定义，同步更新 ERP 配置规则

2. 关键缺失：毛需求计算规则
   - 现状：全文未提及毛需求计算规则
   - 风险：
     （1）采购计划错误（多买/少买）
     （2）生产中断或库存积压
     （3）现金流紧张
   - 建议公式：毛需求 = B（外购总成）+ HC（委外总成）+ V（双经销件）
   - 排除项：C/D/S/M 不计入毛需求（子零件/内部生产）

3. 模糊地带：VAA 与 CSMT 区分标准
   - 原文："区别在于商务模式及到货地点"（过于模糊）
   - 纠正：
     （1）VAA：主机厂必须经手（先买入再转卖）
     （2）CSMT：主机厂完全不经手（只收 HC 总成）
   - 财务影响：
     （1）VAA：产生采购成本 + 销售收入
     （2）CSMT：只有 HC 采购成本

【关键欠缺】

1. 质量责任划分规则缺失
   - 问题：C 件质量出问题谁负责？D 件出问题供应商能否免责？
   - 风险：责任不清导致扯皮，影响生产进度
   - 建议：
     （1）C 件：供应商 100% 承担
     （2）D 件：供应商 70% + 主机厂 30%（指定责任）
     （3）V 件：主机厂 50% + 集成商 50%

2. 交付风险管控规则缺失
   - 问题：C 件延迟交付导致 HC 总成无法回厂，如何追责？
   - 风险：供应链中断，整车厂停产（损失约 100-500 万/天）
   - 建议：
     （1）供应商上报 C/D/S 子件的二级供应商清单
     （2）关键 C 件（A 类物料）主机厂备案
     （3）建立供应链预警机制（库存水位监控，提前 4 周预警）

3. 成本核算规则缺失
   - 问题：HC 总成成本如何核算？C 件成本是否透明？
   - 风险：成本黑箱，无法优化，被供应商"卡脖子"
   - 建议：
     （1）HC 成本拆分：C 件成本 + 装配工时 + 管理费 + 利润
     （2）供应商开放 C 件成本结构（至少 A 类物料）
     （3）年度成本审查（对标市场价）
   - 预期收益：预计可降低 5-10% 采购成本

4. 变更管理流程缺失
   - 问题：B 件从 C 类改为 D 类，流程是什么？谁审批？
   - 风险：随意变更导致 BOM 混乱、库存报废
   - 建议：
     （1）变更申请流程：工程→采购→制造三方审批
     （2）变更影响评估：库存、在制、已采购
     （3）变更生效时点控制：断点管理
   - 预期收益：预计可减少 30-50% 变更损失

【专家建议框架】

1. 质量责任矩阵
   - C 件：供应商 100%
   - D 件：主机厂 30% + 供应商 70%
   - V 件：主机厂 50% + 集成商 50%

2. 交付风险管控
   - 二级供应商备案（A 类 C 件）
   - 库存水位监控
   - 预警机制（提前 4 周）

3. 成本核算规则
   - HC 成本拆分：C + 工时 + 管理费 + 利润
   - 年度成本审查
   - 对标市场价

4. 变更管理流程
   - 申请→评估→审批→执行
   - 断点管理
   - 库存处理方案

5. 供应商绩效评估
   - 质量：PPM（百万分之缺陷率）
   - 交付：OTD（准时交付率）
   - 成本：年降
   - 服务：响应速度

6. 毛需求计算规则
   - 公式：毛需求 = B + HC + V
   - 排除：C/D/S/M 不计入
   - 实施：ERP 系统自动识别

【业务价值】

- 避免 ERP 配置错误：修正 C 件定义，防止重复计算
