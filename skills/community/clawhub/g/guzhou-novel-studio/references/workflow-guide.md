## 1. 快速开始

### 1.1 首次使用

```
1. 明确创作类型（短篇/长篇/盐选）
2. 如需市场分析，先执行调研模块
3. 如有风格偏好，先提取或选择风格
4. 开始创作
```

### 1.2 创作类型选择

| 类型 | 字数范围 | 适用场景 |
|------|---------|---------|
| 微小说 | 800-2200字 | 短故事、快速创作 |
| 盐选短篇 | 8000字起 | 知乎等平台付费内容 |
| 长篇 | 3万字+ | 连载、小说出版 |

## 2. 短篇创作流程

### 2.1 流程图

```
┌─────────────────────────────────────────────────────────────────┐
│                        短篇创作流程                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐ │
│  │ 需求确认  │ →  │ 风格设定  │ →  │ 创作输出  │ →  │ 质检修改  │ │
│  └──────────┘    └──────────┘    └──────────┘    └──────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 详细步骤

#### Step 1：需求确认

用户提供以下信息：
- 题材类型（都市、悬疑、科幻、玄幻等）
- 篇幅要求（2000字内、5000字等）
- 核心元素（人设、情节、氛围等）

#### Step 2：风格设定（可选）

```
用户：像猫腻一样写
→ 加载猫腻风格DNA
→ 应用于创作
```

#### Step 3：创作输出

直接输出章节正文，格式：

```markdown
# 章节标题

正文内容...

---

[章节元信息]
- 字数：2000+
- 伏笔：本章埋设的伏笔
- 人物状态：本章结束时的人物状态
```

#### Step 4：质检修改

- 如用户要求质检，执行质检模块
- 根据质检报告修改
- 重复直到满意

## 3. 长篇创作流程

### 3.1 流程图

```
┌─────────────────────────────────────────────────────────────────────────┐
│                          长篇创作流程                                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐          │
│  │ 项目立项  │ →  │ 调研分析  │ →  │ 大纲生成  │ →  │ 风格提取  │          │
│  └──────────┘    └──────────┘    └──────────┘    └──────────┘          │
│        │                                                   │            │
│        ▼                                                   ▼            │
│  ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐          │
│  │ 章节回填  │ ←  │ 质检修改  │ ←  │ 章节创作  │ ←  │ 记忆加载  │          │
│  └──────────┘    └──────────┘    └──────────┘    └──────────┘          │
│        │                                                   │            │
│        └─────────────────────┬─────────────────────────────────┘        │
│                              │                                            │
│                              ▼                                            │
│                     ┌──────────────┐                                      │
│                     │  阶段回顾     │                                      │
│                     │  (每5-10章)   │                                      │
│                     └──────────────┘                                      │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 3.2 详细步骤

#### Step 1：项目立项

```
用户：我要写一个长篇玄幻小说，叫《星际迷途》
→ 创建项目目录结构
→ 初始化project.json
```

**创建的目录结构**：
```
novel-workspace/projects/星际迷途/
├── project.json
├── outline/
├── characters/
├── world/
├── chapters/
├── memory/
├── qa-reports/
└── assets/
```

#### Step 2：调研分析（可选）

```
用户：帮我分析一下玄幻市场
→ 加载调研模块
→ 输出research-report.json
```

#### Step 3：大纲生成

根据设计文档或用户提供的信息，生成：
- `outline/outline.json` - 大纲JSON
- `outline/outline.md` - 大纲Markdown
- `outline/chapters.md` - 章节规划
- `outline/foreshadowing.md` - 伏笔规划

#### Step 4：风格提取（可选）

```
用户：我想参考《斗破苍穹》的风格
→ 加载风格模块
→ 提取Style-DNA
→ 保存到memory/style-dna.json
```

#### Step 5：人物设定

创建人物档案：
```json
{
  "character_id": "uuid",
  "name": "角色名",
  "role": "protagonist",
  "personality": {...},
  "goals": {...},
  "dialogue_samples": [...]
}
```

#### Step 6：章节创作

```
用户：写第一章
→ 加载记忆包（大纲+人物+风格）
→ 创作章节正文
→ 保存到chapters/chapter-01.md
```

#### Step 7：质检修改

```
用户：检查第一章
→ 加载质检模块
→ 读取章节正文和记忆包
→ 输出qa-reports/qa-chapter-01.md
→ 根据报告修改
```

#### Step 8：章节回填

修改完成后，更新记忆系统：
- 更新`memory/memory.json`
- 创建`memory/active/current-plot.json`
- 归档`memory/archive/chapter-01-summary.json`

#### Step 9：循环创作

重复Step 6-8，写完所有章节。

#### Step 10：阶段回顾

每5-10章进行一次：
- 记忆压缩
- 风格漂移校准
- 大纲调整

## 4. 风格管理流程

### 4.1 风格提取

```
用户：提取这段文字的风格
→ 输入风格样本
→ 加载风格模块
→ 输出Style-DNA.json
```

### 4.2 风格应用

```
用户：按照这个风格写
→ 加载Style-DNA
→ 创作时遵循风格特征
```

### 4.3 风格进化

```
用户：调整一下，变得更轻松一些
→ 根据反馈调整Style-DNA
→ 更新memorable_phrases
```

## 5. 质检流程

### 5.1 启动质检

```
用户：帮我检查第三章
→ 加载质检模块
→ 读取章节正文
→ 读取记忆包（人物设定、世界观等）
```

### 5.2 质检维度

| 检查类型 | 说明 |
|---------|------|
| 一致性检查 | 世界观、人物、时间线一致性 |
| OOC检查 | 人物性格行为一致性 |
| 伏笔检查 | 伏笔埋设与回收 |
| 节奏检查 | 章节节奏、冲突密度 |
| 钩子检查 | 悬念设置有效性 |

### 5.3 处理结果

```
优先级排序：
1. 🔴 高 - 必须修复（OOC、逻辑漏洞）
2. 🟡 中 - 建议修复（伏笔遗漏、节奏问题）
3. 🟢 低 - 可选优化（风格细节）

用户确认后，执行修改
```

## 6. 项目恢复流程

### 6.1 恢复现有项目

```
用户：继续写《星际迷途》
→ 定位项目目录
→ 加载project.json
→ 加载最新记忆包
→ 确认当前进度
→ 继续创作
```

### 6.2 记忆同步检查

恢复时检查：
- 当前章节进度
- 人物状态是否更新
- 伏笔是否追踪
- 风格是否漂移

### 6.3 项目状态查看

```
用户：查看项目状态
→ 输出：
  - 项目名称、总章节数
  - 当前章节
  - 未解决的伏笔
  - 人物状态摘要
```
