# 笔记模板

## 永久笔记模板（notes/areas/）

```markdown
# <笔记标题>

> 最后更新：YYYY-MM-DD
> 标签: #永久笔记 #<分类>

## 核心内容
<笔记正文，一个笔记只记录一个主题>

## 关联笔记
- [[相关笔记1]]
- [[相关笔记2]]
```

## 项目笔记模板（notes/projects/）

```markdown
# <项目名称>

> 日期：YYYY-MM-DD
> 状态：进行中/已完成/暂停
> 标签: #项目/<分类>

## 背景
<项目背景和目标>

## 进展
- [x] 已完成事项
- [ ] 待办事项

## 关键决策
<重要决策记录>

## 关联笔记
- [[相关永久笔记]]
- [[任务记录索引]]
```

## 每日笔记模板（notes/daily/）

```markdown
# YYYY-MM-DD Daily Notes

> 标签: #daily #YYYY-MM

## 今日思考
<当天的重要想法和结论，不是原始对话>

## 完成的任务
- <任务简述>

## 明日关注
<需要跟进的事项>

## 关联笔记
- [[相关项目笔记]]
```

## 对话存档模板（references/ai-chats/）

```markdown
# 对话记录：<简短标题>

> 日期：YYYY-MM-DD
> 话题：<主题>
> 标签: #ai-chat #<分类>

## 对话摘要

### 用户主要问题
<用户提出的核心问题>

### 关键结论/决策
<达成的结论和决策>

### 待跟进
- [ ] <待办事项>

## 完整对话关键节点
<重要对话片段，不是全文复制>
```

## 笔记命名规范

- **永久笔记**：`主题名.md`，如 `system-config.md`
- **项目笔记**：`项目名.md`，如 `openclaw-obsidian-integration.md`
- **每日笔记**：`YYYY-MM-DD.md`
- **对话存档**：`YYYY-MM-DD-slug.md`

## 双向链接最佳实践

1. **每个笔记至少有一个出链**：链接到相关笔记，避免孤立
2. **链接到具体笔记**：用 `[[笔记名]]` 而不是 `#标签` 建立强关联
3. **定期用 `--orphans` 检查**：发现并修复孤立笔记
4. **链接即知识**：两个笔记之间的链接本身也是一种知识表达
