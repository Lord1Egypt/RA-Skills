# 8 Phase 详细检查清单

逐 Phase 对照，不要漏项。

## Phase 1 —— 需求澄清 ☐

- [ ] 确认出行日期（精确到星期）
- [ ] 确认人数 & 关系（影响房型、座椅数）
- [ ] 确认机票状态（已定 / 待定 / 已取消）
- [ ] 确认预算范围（人均总预算 / 每晚酒店上限）
- [ ] 确认交通方式偏好（自驾 / 公共 / 跟团）
- [ ] 确认体验项目强需求（潜水 / 看极光 / 极限运动）
- [ ] 确认酒店会籍（IHG / Marriott / Hilton / GHA / Hyatt）
- [ ] 确认作息偏好（早鸟 / 夜猫）—— 决定每日第一个行程起始时间
- [ ] 确认同步房间名（如需跨设备）
- [ ] 确认特殊饮食 / 医疗 / 宠物需求

## Phase 2 —— 资源调研 ☐

- [ ] 小红书 `search_feeds` 至少 3 组关键词
- [ ] 至少查看 5 篇发布 <3 个月的攻略
- [ ] 评论区翻完找真实反馈
- [ ] 地图 API / 手动对比至少 2 条自驾路线
- [ ] 酒店至少 3 方比价（Agoda / Booking / Trip.com / 官网）
- [ ] 渡轮 / 船票 / 火车查官网班次
- [ ] 签证政策查官方或大使馆公告
- [ ] 天气查**历年同期**数据（不是当前预报）

## Phase 3 —— 方案决策 ☐

- [ ] 每晚住宿确认：地点 / 价格 / 确认号 / 停车 / 早餐 / 取消政策
- [ ] 交通方案全链条：机场↔酒店、酒店↔酒店、酒店↔景点
- [ ] 一日游 N 个项目全部选好版本（self-drive / pickup）
- [ ] 预订需要提前锁名额的项目（船潜、热门跳伞、米其林餐厅）
- [ ] 餐厅候选每天午晚各 2 家（主选 + Plan B）
- [ ] 备选方案 Plan B 写入行程书

## Phase 4 —— 行程书 Markdown ☐

- [ ] 文件名带日期和版本号
- [ ] 封面摘要 + 预算总览 + 航班 + 住宿清单 + 交通清单
- [ ] 每天 Day by Day 带 **时间/地点/动作/费用** 四列
- [ ] 关键节点 Plan B
- [ ] 应急联系方式（使领馆、保险、租车）
- [ ] 费用拆分到付款人
- [ ] 早于用户自然醒时间的活动标 ⚠️ 并说明原因

## Phase 5 —— Todo 清单 ☐

- [ ] 三段分区：urgent / important / done
- [ ] 每条带 `data-id` 唯一（u0-uN / i0-iN / d0-dN）
- [ ] 每条带图标（Lucide / Heroicons SVG）
- [ ] 每条副说明包含：去哪办、多少钱、对应行李项
- [ ] 装备 / 行李清单单独一个 `#todoPack`
- [ ] 已完成项保留作为备忘（不删除）
- [ ] Todo 总数合理（10-30 条之间）

## Phase 6 —— 交互式网页 ☐

- [ ] 复制 `index-skeleton.html`
- [ ] 替换目的地、日期、航班
- [ ] 嵌入 Leaflet 地图 + 标点（住宿/景点/餐厅/机场）
- [ ] Todo 列表填充
- [ ] `SYNC_ROOM` 改成本次房间名（即使只用 localStorage 也需要设）
- [ ] 进度条 DOM 就位
- [ ] CSS 用深色主题 + 主色调
- [ ] 本地浏览器测试勾选沉底、进度条、同步徽章

## Phase 7 —— 数据同步 ☐

- [ ] 决定同步后端（localStorage / Gist / JSONBin / Workers KV / 自建）
- [ ] 配置对应凭据（GitHub Token / JSONBin Key / Workers URL）
- [ ] 本地测试：勾选 → 刷新页面 → 状态保留
- [ ] 跨设备测试（电脑 + 手机）：勾选 → 另一端 30s 内可见
- [ ] 断网测试：关 WiFi → 勾选仍可工作 → 联网后自动同步
- [ ] debounce 有效：连勾 5 条只发 1 次网络请求

## Phase 8 —— 部署 & 预览 ☐

- [ ] 选定静态托管平台（Cloudflare Pages / Vercel / Netlify / GH Pages）
- [ ] 首次部署 + 自定义域名（可选）
- [ ] URL 加缓存破坏参数 `?v=<yyyymmdd>`
- [ ] 微信分享链接测试（iOS + Android）
- [ ] 分享前检查：无敏感信息泄漏（确认号、信用卡尾号等）
- [ ] 写经验笔记：本次攻略的独特发现

---

## 🎯 交付标准（DoD）

**合格**：
- 用户能在手机 + 电脑上看同一个指挥页
- Todo 勾选在刷新后保留
- 行程书可独立打印携带（断网可用）
- Plan B 覆盖所有高风险节点

**优秀**：
- 指挥页有地图 + 进度条 + 同步徽章
- Todo 勾选跨设备实时同步
- 每日都有备选方案 + 时间弹性
- 餐厅、加油站、厕所等琐事都预查好
- 行李清单与 Todo 交叉引用

**不及格**：
- 关键节点缺 Plan B
- Todo 没写具体动作（"准备行李"太泛）
- 自驾路线没对比备选
- 酒店没查停车位 / 早餐 / 取消政策
