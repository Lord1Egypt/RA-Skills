## 品牌视觉资产生成要求清单

品牌视觉资产生成技能每种任务类型的"零件清单"——必选/可选组件、组装顺序、领域约束。按清单逐项产出。

---

### BA1 需求分析域

#### BA1-01 信息采集
- **必选组件**: 公司名称（全称）、行业类型、个人信息（姓名/手机/邮箱/网址/地址）、设计偏好
- **可选组件**: 品牌色彩偏好、Logo风格参考、竞争对手Logo
- **组装顺序**: 公司名称确认→行业分析→个人信息采集→设计偏好收集
- **约束**: 个人信息必须使用用户提供的真实信息，不得编造；地址需完整到门牌号；手机格式建议"000-0000-0000"
- **格式**: 公司信息表（Markdown表格）

#### BA1-02 设计方向校准
- **必选组件**: 色彩方案（主色+辅色+hex值）、Logo风格描述、构成思路（图形/文字/组合）、用户确认
- **可选组件**: 色彩心理学分析、行业视觉趋势参考
- **组装顺序**: 行业视觉推导→色彩方案→风格描述→构成思路→用户确认
- **约束**: 必须获得用户明确确认后才进入生成；这是关键校准点，不可跳过
- **格式**: 设计方向确认书（Markdown）

#### BA1-03 资产规格规划
- **必选组件**: 7类资产文件清单、每类文件数量、尺寸规格表、目录结构
- **可选组件**: 设计系统说明、字体规范
- **组装顺序**: 资产分类→文件清单构建→尺寸规格确定→目录结构设计
- **约束**: 文件清单覆盖核心标志(3SVG+21PNG+1ICO)+商标(6SVG+6PNG)+数字化(5SVG+5PNG)+办公(4SVG+4PNG)+宣传(3SVG+3PNG)+规范(3SVG+3PNG+1HTML)+扩展(9SVG+9PNG)。其中：核心标志包=3SVG+21PNG+1ICO=25个；HTML展示页单独产出，不计入核心标志包；其余文件=30SVG+30PNG+1HTML=61个（含商标+数字化+办公+宣传+规范+扩展应用）；全量=33SVG+51PNG+1ICO+2HTML=87个。商标稿件945×945对应8×8cm@300dpi。无后缀的标准版与有后缀的同尺寸版不应同时存在（横版/竖版多尺寸后缀均为非标准尺寸）。技能声称87个文件时必须实际产出87个，禁止数量虚标
- **格式**: 资产规格表（Markdown表格）

### BA2 设计生成域

#### BA2-01 Logo SVG生成
- **必选组件**: Logo SVG源代码、viewBox设置、渐变定义、背景rect
- **可选组件**: 设计理念说明、色彩值标注
- **组装顺序**: AI编写Logo SVG代码→viewBox校准→背景rect匹配
- **约束**: 
  - 必须由AI直接编写SVG矢量代码（SVG是XML格式，非图像生成）
  - viewBox必须覆盖所有元素；背景rect必须覆盖整个viewBox
  - preserveAspectRatio必须设为`xMidYMid meet`
  - **SVG XML特殊字符必须转义**：`<text>`中的`&`→`&amp;`、`<`→`&lt;`、`>`→`&gt;`（仅含英文特殊字符的SVG会触发破碎图标）
  - **文件头部无BOM**：以`<?xml version="1.0" encoding="UTF-8"?>`开头
  - **禁止emoji字符**：📱🌐📍等依赖系统字体，改用SVG `<path>`绘制图标
- **格式**: SVG文件（1个核心Logo）

#### BA2-02 标志形态派生
- **必选组件**: 横版标志SVG、竖版标志SVG、纯图标SVG
- **可选组件**: 每种形态的设计说明
- **组装顺序**: 核心Logo分析→横版布局（图形+文字左右）→竖版布局（图形+文字上下）→纯图标（仅图形）
- **约束**: 3种形态保持设计一致性；纯图标的viewBox需考虑边缘节点padding（通常-15 padding）；SVG XML特殊字符必须转义；`<defs>`禁止嵌套（单层`<defs>`内含所有渐变定义）；文件头部无BOM
- **格式**: SVG文件（3个）

#### BA2-03 SVG资产批量生成
- **必选组件**: 商标注册6个SVG、数字化应用5个SVG、办公文具4个SVG、宣传物料3个SVG（核心VI系统，02~05目录共18个）
- **可选组件**: 品牌规范3个SVG、环境应用3个SVG、产品周边3个SVG、多媒体模板3个SVG（扩展层，06~09目录共12个；核心管线完成后由项目需求决定是否并行生成）
- **组装顺序**: 按目录逐个生成→商标注册（黑白3+彩色3）→数字化应用→办公文具→宣传物料→（可选：品牌规范→环境应用→产品周边→多媒体模板）
- **约束**: 
  - 所有SVG保持设计一致性；商标稿件符合商标局要求（945×945，8cm@300dpi）；黑白稿优先
  - **本任务产出02-09目录全部30个SVG**（商标注册6+数字化5+办公4+宣传3+品牌规范3+环境3+产品3+多媒体3）；02-05为VI系统核心（18个），06-09为扩展层（12个，核心管线完成后可选并行生成）
  - **渐变引用必须在`<svg>`内包含完整`<defs>`定义**，否则PNG渲染失败（某项目中5个SVG最初缺失`<defs>`）
  - **`<defs>`禁止嵌套**：SVG标准中`<defs>`内不得再嵌套`<defs>`（某项目中3个宣传物料SVG最初存在双层结构）
  - **品牌Slogan统一展示规范**：
    - 必须添加Slogan的资产：门头招牌、展会背景板、易拉宝、礼品袋、社交媒体配图、网站Banner、名片背面、A4信纸
    - 不应添加Slogan的资产：核心Logo（横版/竖版/纯图标）、商标注册稿、App图标、社交头像、文档水印
  - **品牌文案全局同步**：所有SVG中`<text>`元素承载的品牌文案必须与HTML中展示的品牌文案保持完全一致；文案变更时需扫描全部SVG用`sed`批量替换
  - **最终交付前必须清除占位符**：`[xxx]`和`{{xxx}}`两种格式（如`[演示标题]`、`{{标题}}`、`[笔记本标题占位]`、`{{副标题}}`等）必须全部替换为实际品牌文案（扫描命令：`grep -rn '[\[\{]' *.svg | grep -v 'xlink:href' | grep -v 'font-size' | grep -v 'stop-color' | grep -v 'fill' | grep -v 'stroke'`）
  - 多类别资产禁止重名：不同目录中的资产名称需区分（如"视频封面模板"≠"PPT封面"）
- **格式**: SVG文件（02~09目录全部共30个）

#### BA2-04 信息注入
- **必选组件**: 替换映射表（占位符→真实信息）、已替换的SVG文件列表
- **替换字段**: 姓名、手机（格式000-0000-0000）、邮箱、网址、地址（长地址分两行，字号14→11）
- **可选组件**: 字号调整记录
- **组装顺序**: 占位符识别→替换映射构建→批量替换→字号适配→替换验证
- **约束**: 
  - 所有涉及个人信息的SVG必须100%替换；长地址需调整字号
  - **品牌信息注入分层原则**：
    | 资产类型 | 姓名 | 电话 | 邮箱 | 网址 | 地址 | 说明 |
    |---------|:---:|:---:|:---:|:---:|:---:|------|
    | 名片正面 | ✓ | ✓ | ✓ | ✓ | ✓ | 完整个人信息 |
    | 员工工牌 | ✓ | ✓ | ✓ | ✓ | △ | 地址可选 |
    | 信封 | ✓ | ✓ | ✓ | ✓ | ✓ | 完整寄信人信息 |
    | 信纸 | △ | △ | ✓ | ✓ | △ | 邮箱+网址必选 |
    | 门头招牌 | ✗ | ✓ | ✗ | ✓ | ✗ | 仅电话+网址 |
    | 背景板/易拉宝 | ✓ | ✓ | ✓ | ✓ | ✗ | 无需地址 |
    | PPT/Banner | △ | △ | △ | ✓ | ✗ | 网址最低要求 |
    | 核心Logo | ✗ | ✗ | ✗ | ✗ | ✗ | 仅公司名 |
  - **禁止emoji字符**：SVG中的emoji（📱✉🌐📍）依赖系统字体，Linux/Mac环境下会显示方块□；改为文本标签（如`-`/`Web:`/`Addr:`）
- **格式**: 替换记录（Markdown表格）

#### BA2-05 小尺寸优化版生成
- **必选组件**: 简化版SVG源代码、移除元素清单、增强元素清单
- **移除规则**: 虚线圆环（stroke-dasharray+stroke-width<2）、细线（stroke-width<1）、小点（r<6）、虚线连接线（stroke-width<1.5）
- **增强规则**: 卫星节点+33%、连接线+50%
- **可选组件**: 简化前后对比
- **组装顺序**: 原SVG分析→不可见元素识别→移除清单→增强清单→简化版生成
- **约束**: 仅用于16-128px；viewBox与完整版一致；SVG XML特殊字符必须转义；`<defs>`禁止嵌套
- **格式**: SVG文件（1个简化版）

### BA3 格式转换域

#### BA3-01 SVG→PNG渲染
- **必选组件**: PNG渲染结果（纯图标1024×1024大图，横版600×240，竖版400×480）、HTML包装文件、渲染参数记录
- **可选组件**: 渲染日志
- **组装顺序**: SVG读取→width/height替换→preserveAspectRatio设置→HTML包装（margin重置）→Edge headless截图
- **约束**: HTML包装必须`*{margin:0;padding:0;box-sizing:border-box}`；必须`preserveAspectRatio="xMidYMid meet"`；Edge headless不按`<img src>`缩放SVG，必须内嵌SVG并设置width/height属性；横版标志渲染规格(600×240 viewBox→600×240渲染)，竖版标志渲染规格(400×480 viewBox→400×480渲染)
- **格式**: PNG文件（纯图标1024×1024大图，横版600×240，竖版400×480，各1个）
- **工具**: `scripts/render_svg_to_png.py --all`（一站式完整渲染）；单独使用时：`render_svg_to_png()` 函数；批量渲染全部SVG：`render_batch_svg()` 函数

#### BA3-02 多尺寸位图生成
- **必选组件**: 纯图标16/32/64/128/256/512px PNG、ICO内嵌PNG提取（16×16/32×32/48×48/64×64 PNG）、每个尺寸的文件大小
- **可选组件**: 缩小质量评估
- **组装顺序**: 大图读取→尺寸枚举→策略选择（小尺寸用简化版/大尺寸用完整版）→LANCZOS缩小→保存→ICO内嵌PNG提取
- **约束**: 必须用Pillow LANCZOS；16/32/64/128px用简化版SVG渲染的大图；256/512px用完整版；输出RGBA格式；ICO构建完成后必须从ICO二进制提取4个内嵌PNG存入01_核心标志目录（`favicon_16x16.png`、`favicon_32x32.png`、`favicon_48x48.png`、`favicon_64x64.png`）
- **格式**: PNG文件（纯图标6尺寸+ICO内嵌4尺寸+1个480展示版=11个；本任务专属产出，01_核心标志合计21 PNG由BA3-01/BA3-02/BA3-02A共同产出）
- **工具**: `scripts/render_svg_to_png.py` 的 `resize_png()` 函数；ICO提取用PIL+struct手动解析ICO二进制

#### BA3-02A 横版/竖版多尺寸生成
- **必选组件**: 横版标志_1200/_300/_150/_75 PNG、竖版标志_960/_240/_120/_60 PNG
- **可选组件**: 每个尺寸的用途说明
- **组装顺序**: 标准PNG读取→尺寸枚举→按比例缩放→LANCZOS缩小→保存
- **约束**: 必须用Pillow LANCZOS；保持纵横比不变（横版2.5:1，竖版5:6）；文件名用_数字后缀（如01_横版标志_1200.png）
- **格式**: PNG文件（横版4尺寸+竖版4尺寸=8个）
- **尺寸用途说明**:
  | 横版 | 尺寸 | 用途 | 竖版 | 尺寸 | 用途 |
  |------|------|------|------|------|------|
  | _1200 | 1200×480 | 高清大屏/印刷海报 | _960 | 800×960 | 高清大屏/竖版海报 |
  | (标准) | 600×240 | 网站页眉/文档封面 | (标准) | 400×480 | 竖版海报/立牌 |
  | _300 | 300×120 | 中等应用/PPT页脚 | _240 | 200×240 | 中等应用/名片 |
  | _150 | 150×60 | 小尺寸水印/信纸 | _120 | 100×120 | 小尺寸图标/头像 |
  | _75 | 75×30 | 极小尺寸/社交媒体 | _60 | 50×60 | 极小尺寸/头像小 |
- **工具**: `scripts/render_svg_to_png.py --all`（一站式）或Pillow `Image.resize((w,h), Image.LANCZOS)`

#### BA3-02B 批量SVG渲染
- **必选组件**: 30个非核心SVG的PNG渲染结果（02_商标注册~09_多媒体模板目录全部SVG：商标注册6+数字化5+办公4+宣传3+品牌规范3+环境3+产品3+多媒体3）
- **可选组件**: 渲染日志、失败重试记录
- **组装顺序**: 目录遍历→SVG文件筛选→逐个渲染→失败重试→结果汇总
- **约束**: 
  - 按viewBox自动计算渲染尺寸（横版600×240、竖版400×480、纯图标480×480、商标945×945、名片1800×1080等）
  - 遍历02_商标注册~09_多媒体模板目录，跳过01_核心标志目录（已由render_svg_to_png.py处理）
  - **增量渲染强制刷新规则**：已存在的PNG自动跳过（增量渲染）；但SVG内容变更后必须先`rm`删除旧PNG再重新渲染，否则旧PNG继续显示。必须强制刷新的场景：品牌文案变更、个人信息替换、颜色/渐变调整、形状/布局修改。流程：`rm <SVG同名>.png` → `python render_svg_to_png.py . --all --timeout 120`
  - Edge headless超时后自动切换`--headless`模式重试
- **格式**: PNG文件（30个VI系统+品牌规范+扩展应用PNG）
- **工具**: `scripts/render_svg_to_png.py --all`（一站式）或`render_batch_svg()` 函数

#### BA3-03 ICO多尺寸构建
- **必选组件**: favicon.ico文件、ICO包含的尺寸清单（16/32/48/64）、ICO内嵌PNG提取（4个PNG：`favicon_16x16.png`/`favicon_32x32.png`/`favicon_48x48.png`/`favicon_64x64.png`）
- **可选组件**: ICO文件大小记录
- **组装顺序**: 源PNG选择→resize到ICO尺寸→PNG编码→ICO二进制构建→文件写入→从ICO二进制提取内嵌PNG
- **约束**: 
  - PIL的`Image.save(ico, sizes=[...])`只保存第一帧，必须手动构建
  - 48px从64px缩小；尺寸≥256填0
  - **ICO内嵌PNG提取**：ICO构建完成后，必须手动解析ICO二进制结构（6字节header + count×16字节entry），检查每个entry的图像数据是否为PNG（`\x89PNG`签名），提取为独立PNG文件存入01_核心标志目录
- **格式**: ICO文件（1个，含4尺寸）+ 4个独立PNG
- **工具**: `scripts/render_svg_to_png.py` 的 `build_ico()` 函数

#### BA3-04 渲染质量校验
- **必选组件**: PNG尺寸验证、内容完整性、边缘截断检查、ICO多尺寸有效性、重复文件检测
- **可选组件**: 四象限像素分布分析、内容中心偏移检查
- **组装顺序**: PNG尺寸读取→边缘像素检查→内容中心检查→ICO结构解析→重复文件检测→报告
- **约束**: PNG四边clean（无截断）；内容中心=画布中心；ICO包含所有声明尺寸；不得存在字节级相同的重复PNG文件
- **格式**: 渲染校验报告（Markdown检查表）
- **工具**: `scripts/verify_assets.py --render-check --dup-check`（`check_render_quality()`检查PNG边缘截断+ICO结构有效性；`check_duplicates()`字节级MD5比对重复PNG）

### BA4 整合交付域

#### BA4-01 HTML展示页生成
- **必选组件**: 完整HTML文件、CSS样式、所有img引用、目录树、场景对照表、色彩规范、使用指南
- **板块清单**: Hero区、品牌概述、标志说明、核心标志、商标注册、数字化应用、办公文具、宣传物料、品牌规范、环境应用、产品周边、多媒体模板、应用场景总览、footer
- **可选组件**: JavaScript交互、动画
- **组装顺序**: HTML骨架→CSS样式→Hero区→品牌概述→标志说明→各板块双格式卡片→应用场景总览→footer
- **约束**: 
  - 禁缓存meta标签（Cache-Control/Pragma/Expires）；favicon引用含ICO+PNG
  - **HTML中SVG显示规范**：禁止使用`max-width`+`max-height`+`object-fit`组合展示SVG，会导致破碎图标。必须使用`width:100%; height:160px; object-fit:contain; display:block; background:#FFFFFF`
  - `img`标签显式设置`display:block`防止底部留白
  - `.tree{white-space:pre}`；底部含公司联系信息
  - **ICO多尺寸展示规范**：ICO构建完成后从ICO二进制提取4个内嵌PNG（16×16/32×32/48×48/64×64）存入01_核心标志目录，HTML中在核心标志区、纯图标多尺寸之后、商标注册之前，添加`▼ 查看全部 favicon.ico 尺寸 (4 PNG)`展开/收起卡片，使用与纯图标多尺寸相同的`png-toggle`+`png-expand`+`png-grid`结构，标签标注`favicon_16×16`等
  - **应用场景表格细化规范**：应用场景表格必须按文件夹顺序（01→09），每行标注分类/文件/尺寸/应用场景四个字段；核心标志部分细化到每种文件（横版/竖版/纯图标/favicon.ico）及每种尺寸；其余类别每个文件一行，标注SVG+PNG双格式及具体应用场景
  - 品牌Slogan、品牌含义文案需与SVG中保持一致
- **格式**: HTML文件（1个）

#### BA4-02 完整性校验
