# 权限错误信号参考

当以下任意信号出现时，立即进入恢复流程。

## 系统级错误信号

| 信号 | 典型场景 | 来源 |
|------|---------|------|
| `Permission denied` | 读写受保护文件 | Unix 系统调用 |
| `EACCES` | 访问被拒绝 | Node.js / Python |
| `EACCES` (misspelled) | 同上，注意拼写变体 | 部分错误日志 |
| `Operation not permitted` | macOS 系统级权限拒绝 | macOS 内核 |
| `EPERM` | 操作不被允许 | Node.js / Python |
| `Access is denied` | 访问被拒绝 | Windows |
| `not permitted` | 系统调用拦截 | 通用 |
| `not allowed` | 系统调用拦截 | 通用 |
| `sudo` 要求但无法执行 | 需要 root 权限 | 命令执行 |

## AI 行为信号（自我检测）

| 信号 | 阈值 | 说明 |
|------|------|------|
| 同一操作连续失败 | ≥ 2 次 | 无论错误类型，一律触发恢复 |
| 工具调用无响应 | > 30 秒 | 从发起调用算起 |
| 重复尝试同一失败步骤 | 检测到循环 | 如：重试 → 同错误 → 重试 |
| 等待用户输入但无交互通道 | 即时 | 微信端无 STDIN 可用 |

## 微信/Claw 特有信号

| 信号 | 说明 |
|------|------|
| 微信回复通道超时 | 发送消息后超过 10 秒无 ack |
| 上一条指令未完成但新指令已到达 | 队列积压，说明卡死 |
| 工具调用返回空结果超过 3 次 | 可能是权限导致静默失败 |
