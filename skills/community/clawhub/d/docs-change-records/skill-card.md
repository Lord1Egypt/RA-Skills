## Description: <br>
Documentation engineer skill for module docs, README updates, architecture notes, API change records, and repository-safe documentation placement. <br>

This skill is ready for commercial/non-commercial use. <br>

## Publisher: <br>
[aiweline](https://clawhub.ai/user/aiweline) <br>

### License/Terms of Use: <br>
MIT-0 <br>


## Use Case: <br>
Developers and documentation maintainers use this skill to update module READMEs, architecture notes, API docs, usage docs, and change records after code changes. It helps keep documentation scoped to the owning module and aligned with validated behavior, design, or interface changes. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Broad documentation-related wording can activate the skill for requests that do not require documentation changes. <br>
Mitigation: Review proposed documentation paths and accept updates only when the underlying change affects usage, architecture, interfaces, or required change records. <br>
Risk: Generated documentation can overstate implementation behavior or validation status. <br>
Mitigation: Cross-check claims against the final implementation and validation evidence before relying on or merging the documentation. <br>
