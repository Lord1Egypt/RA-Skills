# Prompt 模式库

## 一、基础 Prompt 模板结构

```yaml
角色: [定义 AI 的身份和能力范围]
任务: [具体要完成的目标，分步描述]
约束: [格式要求、禁止行为、安全边界]
示例:
  - 输入: "..."
    输出: "..."
输出格式: {json schema 或 格式描述}
动态变量: [需要运行时注入的变量列表]
```

## 二、常用 Prompt 模式

### 2.1 结构化输出模式 (JSON)

```
你是一个 [角色]。请根据以下输入，输出严格符合 JSON 格式的结果。

输入: {{input}}

输出格式:
{
  "field1": "string",
  "field2": ["array of strings"],
  "field3": {
    "subfield": "number"
  }
}

约束:
- 只返回 JSON，不要添加解释
- 字段不能为空
- field3.subfield 必须是 0-100 的整数
```

### 2.2 思维链模式 (Chain of Thought)

```
你是一个 [角色]。请逐步推理以下问题，在给出最终答案前先展示思考过程。

问题: {{question}}

请按以下步骤推理:
1. 理解问题的核心
2. 列出已知信息
3. 分析可能的答案
4. 排除不合理的选项
5. 给出最终答案

在"思考过程"部分详细展开，在"最终答案"部分给出简洁回答。
```

### 2.3 分类/路由模式

```
你是一个智能路由助手。根据用户输入将其分类到以下意图之一:

意图列表:
- order_query: 查询订单状态
- refund_request: 申请退款
- product_info: 询问产品信息
- human_service: 需要人工服务
- other: 其他

用户输入: {{user_input}}

请返回 JSON:
{
  "intent": "意图名称",
  "confidence": 0.0-1.0,
  "reasoning": "分类理由",
  "next_action": "建议的下一步动作"
}
```

### 2.4 Few-Shot 模式

```
你是一个情感分析助手。判断以下文本的情感倾向。

示例1:
文本: "这个产品太好用了，强烈推荐！"
情感: positive
理由: 包含强烈正面评价词

示例2:
文本: "发货太慢，包装也破了"
情感: negative
理由: 包含负面体验描述

示例3:
文本: "还行吧，一般般"
情感: neutral
理由: 没有明显情感倾向

现在请分析:
文本: {{user_input}}
情感:
理由:
```

### 2.5 安全护栏模式

```
你是一个客服助手。

安全规则:
- 不透露内部系统信息
- 不承诺无法兑现的事项
- 遇到价格/退款问题必须转人工
- 检测到辱骂内容时礼貌结束对话

用户: {{user_input}}

请先检查安全规则，再回复。
```

## 三、Agent 工作流设计模式

### 3.1 串行流水线 (Pipeline)

```
Step1: 意图识别 → Step2: 信息检索 → Step3: 内容生成 → Step4: 质量审核
```

### 3.2 并行-汇聚 (Map-Reduce)

```
        ┌→ Worker1: 搜索源A ─┐
Router ─┼→ Worker2: 搜索源B ─┼→ Aggregator: 结果融合 → 输出
        └→ Worker3: 搜索源C ─┘
```

### 3.3 循环迭代 (Iterative)

```
生成 → 自我检查 → [不合格] 修正 → 生成 → [合格] 输出
```

### 3.4 人机协作 (HITL)

```
AI处理 → 置信度检查 → [低] 发送人工审核 → [审核通过] 发布
```

## 四、各场景推荐 Prompt 模式

| 场景 | 推荐模式 | 原因 |
|------|---------|------|
| 数据抽取/结构化 | 结构化输出模式 | 保证程序可解析 |
| 客服对话 | 分类路由 + 安全护栏 | 精准路由 + 安全 |
| 内容生成 | Few-Shot + 思维链 | 提升质量 |
| 复杂推理 | 思维链模式 | 提高准确率 |
| 知识问答 | RAG 模式 (检索增强) | 基于真实数据 |
| 代码生成 | 结构化输出 + 安全护栏 | 格式 + 安全 |

## 五、常见 Prompt 反模式

| 反模式 | 问题 | 改进 |
|--------|------|------|
| "请帮我..." | 角色不明确 | 明确角色设定 |
| "尽可能详细" | 模糊指令 | 给出具体字数/结构 |
| 无输出格式约束 | 输出不可控 | 定义 JSON Schema |
| 无安全规则 | 可能违规 | 添加安全护栏 |
| 单轮对话无上下文 | 缺乏背景 | 注入上下文变量 |
