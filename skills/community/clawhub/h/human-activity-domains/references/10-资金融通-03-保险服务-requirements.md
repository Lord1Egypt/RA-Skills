## 保险服务生成要求

保险服务每种任务类型的"零件清单"——必选/可选组件、组装顺序、领域约束。按清单逐项产出。

---

### 1 保险产品

#### T1-1 产品设计

- **必选组件**: 保障范围、保险责任、除外责任、产品说明
- **可选组件**: 产品调研、产品对比、产品评审
- **组装顺序**: 确定保障范围 → 设定保险责任 → 明确除外责任 → 撰写产品说明
- **约束**: 范围必须明确；责任必须清晰；说明必须完整
- **格式**: 保险产品设计方案（Markdown）

#### T1-2 产品定价

- **必选组件**: 定价依据、保费计算、费率厘定、定价策略
- **可选组件**: 定价模型、定价工具、定价评审
- **组装顺序**: 分析定价依据 → 计算保费 → 厘定费率 → 制定策略
- **约束**: 依据必须充分；计算必须准确；策略必须合理
- **格式**: 产品定价方案（Markdown）

#### T1-3 产品开发

- **必选组件**: 开发计划、条款拟定、系统开发、测试验收
- **可选组件**: 开发工具、开发培训、开发评审
- **组装顺序**: 制定开发计划 → 拟定条款 → 开发系统 → 测试验收
- **约束**: 计划必须可行；条款必须合法；测试必须充分
- **格式**: 产品开发方案（Markdown）

#### T1-4 产品管理

- **必选组件**: 产品上线、产品维护、产品优化、产品评估
- **可选组件**: 管理工具、管理系统、管理报告
- **组装顺序**: 上线产品 → 维护产品 → 优化产品 → 评估产品
- **约束**: 上线必须规范；维护必须及时；优化必须有效
- **格式**: 产品管理方案（Markdown）

### 3 保险理赔

#### T3-1 理赔申请

- **必选组件**: 报案流程、资料清单、案件登记、申请确认
- **可选组件**: 申请工具、申请系统、申请指导
- **组装顺序**: 执行报案流程 → 收集资料 → 登记案件 → 确认申请
- **约束**: 流程必须清晰；清单必须完整；登记必须准确
- **格式**: 理赔申请记录（Markdown）

#### T3-2 理赔调查

- **必选组件**: 调查内容、调查方法、调查结果、调查报告
- **可选组件**: 调查工具、调查系统、调查培训
- **组装顺序**: 确定调查内容 → 选择调查方法 → 执行调查 → 生成报告
- **约束**: 内容必须全面；方法必须科学；报告必须客观
- **格式**: 理赔调查报告（Markdown）

#### T3-3 理赔审核

- **必选组件**: 审核标准、损失核定、赔款计算、审批决策
- **可选组件**: 审核工具、审核系统、审核培训
- **组装顺序**: 确定审核标准 → 核定损失 → 计算赔款 → 做出决策
- **约束**: 标准必须明确；核定必须准确；决策必须公正
- **格式**: 理赔审核记录（Markdown）

#### T3-4 理赔支付

- **必选组件**: 赔款确认、资金划转、支付确认、结案归档
- **可选组件**: 支付工具、支付系统、支付监控
- **组装顺序**: 确认赔款 → 划转资金 → 确认支付 → 结案归档
- **约束**: 确认必须准确；划转必须及时；归档必须完整
- **格式**: 理赔支付记录（Markdown）
