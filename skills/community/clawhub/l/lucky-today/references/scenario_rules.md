# Scenario Rules — 场景化运势规则

在 `fortune_rules.md` 基础分项分之上,叠加场景偏置 `bias_scenario`。同一日不同场景输出不同侧重。

## 通用流程

1. 先按 `fortune_rules.md` §2-5 计算基础 `R, S, wealth, love, career, social`。
2. 根据场景类型,从下面对应小节取 `bias_scenario` 公式,产出"场景分"。
3. 输出时,**只保留场景相关的输出块**,其他块压缩。
4. 强禁忌预警必须针对场景重写(不要复用通用版)。
5. 时间窗默认从"即刻起"到该场景的关键时刻(面试结束 / 考试结束 / 上线时间)。

## 1) 面试运势(interview)

### 触发短语

- 中文:面试运、面试运势、明天面试运怎么样、我今天有面试、interview 准备
- English: "interview luck", "luck for interview", "interview today/tomorrow"

### 偏置公式

```
bias_interview = (career - 50) * 0.5 + (social - 50) * 0.3 + (R mod 5) - 2
interview_score = clamp(S + bias_interview, 0, 100)
```

### 必备输入

- 面试时间(可粗略,如"明天上午")
- 面试形式(线下 / 视频 / 群面),不知道默认"线下 1v1"

### 输出侧重

- **总览**:用 `interview_score` 给评级,1-2 句解释"为什么这个分,career/social 哪项更高/低"
- **着装**:用主色作为内搭推荐,辅色作避雷;若主色为黑/白则直接推荐
- **沟通策略**:基于 `social` 分项 — `social ≥ 70` 主动多说;`50-69` 平衡;`< 50` 多倾听少抢话
- **关键时段**:`(R mod 3)` 映射 — 上午精神最佳 / 下午思维最稳 / 晚间表达流畅
- **专项忌项**(必输出):迟到 / 紧张时口语化 / 谈薪过早 / 否定前公司
- **强禁忌预警**:
  - 若 `interview_score < 50`:警告"今天易出现注意力分散和小失误,务必提前 30 分钟到场 + 反复检查必备文件"
  - 若 `interview_score 50-70`:警告"面试过程中避免主动谈薪 + 避免否定前雇主"
  - 若 `interview_score > 70`:轻量提醒"自信不等于强势,留出对方提问空间"

### 时间窗

`start = 即刻起`,`end = 面试结束时间 + 2 小时`(完整对话效应消化期)。

## 2) 考试运势(exam)

### 触发短语

- 中文:考试运、考试运势、明天考试、我有考试、studying for exam
- English: "exam luck", "test luck", "test today/tomorrow"

### 偏置公式

```
bias_exam = (career - 50) * 0.6 + (love - 50) * 0.1   # love 代表心情稳定度
exam_score = clamp(S + bias_exam, 0, 100)
```

### 必备输入

- 考试日期 + 时段(如"周日上午 9:00")
- 考试类型(笔试 / 机考 / 口试 / 答辩)

### 输出侧重

- **总览**:用 `exam_score` 给评级
- **临考建议**:
  - 考前 24 小时:轻复习 + 8 小时睡眠(强约束)
  - 考前 2 小时:吃饱不撑 + 不碰咖啡若 `(R mod 4) == 0`(咖啡敏感日)
- **应试节奏**:
  - 先易后难(如果 `career < 70`)
  - 全部按顺序(如果 `career ≥ 70`)
- **专项忌项**:熬夜复习 / 临时换笔/换工具 / 拐弯抹角的难题死磕
- **强禁忌预警**:
  - 考前一晚通宵复习(无条件警告)
  - 若 `career < 50`:警告"复盘心态非常重要,任何一题卡住超过预定时间就跳"
  - 若 `(R mod 7) == 0`:警告"今日易粗心,选择题二选一时回头检查"

### 时间窗

`start = 即刻起`,`end = 考试结束时间`。

---

## 3) 相亲 / 约会运势(dating)

### 触发短语

- 中文:相亲运、约会运、约会运势、今天约会、今晚约会、相亲今天
- English: "date today", "first date luck", "blind date", "dating luck"

### 偏置公式

```
bias_dating = (love - 50) * 0.7 + (social - 50) * 0.3
dating_score = clamp(S + bias_dating, 0, 100)
```

### 必备输入

- 约会形式(吃饭 / 喝咖啡 / 看展 / 看电影 / 户外)
- 第几次见面(可省略,默认初次)

### 输出侧重

- **总览**:用 `dating_score` 给评级
- **着装**:
  - 主色作为主搭(衣服)
  - 辅色作为细节(围巾 / 配饰)
  - 若主色冷色(青蓝紫黑)+ 初次:加一件暖色配饰
- **话题建议**(基于 `(R mod 3)`):
  - 0 → 旅行 + 兴趣
  - 1 → 工作 + 近期变化
  - 2 → 美食 + 日常小事
- **沟通避雷**:
  - 不主动谈前任(无条件)
  - 若 `love < 50`:不主动谈未来规划,先建立眼前共鸣
  - 若 `social < 50`:少抢话,多反问
- **强禁忌预警**:
  - 若 `dating_score < 50`:警告"今日易出现尴尬冷场或误解,提前准备 3 个开场话题"
  - 若 `(R mod 8) == 2`:警告"今日易冲动表白/承诺,初次见面避免说'我们以后'"

### 时间窗

`start = 即刻起`,`end = 约会结束 + 4 小时`(消息回复窗)。

---

## 4) 项目上线 / 发布(launch)

### 触发短语

- 中文:上线运、发布运、项目上线运、版本发布运、产品发布运、PR 合并
- English: "launch luck", "release day", "ship today"

### 偏置公式

```
bias_launch = (career - 50) * 0.6 - (10 if (R mod 6) == 0 else 0)   # 故障日扣分
launch_score = clamp(S + bias_launch, 0, 100)
```

### 必备输入

- 上线时间(精确到小时)
- 上线类型(主版本 / 修复 / 灰度)

### 输出侧重

- **总览**:用 `launch_score`
- **窗口建议**(基于 `(R mod 3)`):
  - 0 → 上午上线(回滚窗口长)
  - 1 → 下午早些(避开晚高峰)
  - 2 → 灰度先(小流量验证)
- **必备清单**(无条件输出):
  1. 回滚方案验证可用
  2. 监控告警接收人在岗
  3. 关键链路有备份/降级
- **强禁忌预警**:
  - 若 `launch_score < 50`:警告"今日易出现上线后短期波动,先灰度 + 监控 30 分钟再放量"
  - 若 `(R mod 7) == 0`:警告"今日易'最后一改',冻结代码不要临时改动"
  - 周五下午无条件警告"非必要不在周五下午发主版本,故障消化窗短"

### 时间窗

`start = 即刻起`,`end = 上线时间 + 24 小时`(初期观察窗)。

---

## 5) 出差 / 远行(business-trip)

### 触发短语

- 中文:出差运、远行运、出差今天、明天出差、旅行运势
- English: "business trip", "travel luck", "trip today"

### 偏置公式

```
bias_trip = (social - 50) * 0.4 + (wealth - 50) * 0.2 - (15 if (R + W) mod 6 in {0, 3} else 0)
trip_score = clamp(S + bias_trip, 0, 100)
```

### 必备输入

- 出发时间 + 目的地(粗略即可,如"周一上海")
- 出差时长

### 输出侧重

- **总览**:用 `trip_score`
- **必备清单**:
  1. 充电宝 + 转接头 + 备用充电线
  2. 身份证 / 护照 / 票据电子版备份
  3. 目的地天气查一遍
- **方向建议**:本地幸运方位与目的地方位关系(若反向 → "短期适应需要更慢的节奏";若同向或邻近 → "气场友好,节奏紧凑也无妨")
- **专项忌项**:无条件忌"赶最早班 + 不留缓冲时间"
- **强禁忌预警**:
  - 若 `trip_score < 50`:警告"今日易行程延误,所有交通节点预留 50% 缓冲"
  - 若 `(R mod 5) == 0`:警告"易丢失小件物品(钥匙 / 充电线 / 文件),登记必备清单"
  - 若 `bias_trip` 含 "-15" 触发(即 (R+W) mod 6 ∈ {0,3}):警告"今日不宜远行,如可改期则改期"

### 时间窗

`start = 即刻起`,`end = 返程到家时间`。

## 6) 场景输出模板差异

所有场景模式都使用以下精简模板(去掉与场景无关的块):

```
【<场景名>运势 | YYYY-MM-DD <场景关键时刻>】

✨ 评级:<场景评级 | 分数>

1-2 句解释:用了哪些规则、哪些分项、得出此分的关键原因。

🎨 关键元素
• 主色 / 辅色(着装或工具配色场景才输出)
• 幸运方位 / 出行方向(出行场景才输出)
• 幸运数字 / 物品(常出现,可选)

✅ <场景>宜做
1. <场景特定动作 1>
2. <场景特定动作 2>
3. <场景特定动作 3>

❌ <场景>忌做
1. <场景特定规避 1>
2. <场景特定规避 2>
3. <场景特定规避 3>

⚠️ 强禁忌预警(按场景偏置触发)
- 禁忌事项 / 禁忌时间 / 原因 / 替代方案

💡 关键时段建议
• <早 / 午 / 晚 + 场景关键时刻>
```

通用四元素和详细拆解块在场景模式下默认折叠,除非用户明确说"完整输出"。
