# M0 任务域协调域范本

## M0-01 任务域选择与组合

### 输入示例
用户需求：实现用户登录功能

### 处理过程
1. 分析用户需求的核心目标
2. 扫描可用的任务域
3. 选择合适的任务域组合

### 输出示例
**任务域选择报告**：
- 核心目标：实现用户登录功能
- 选择的任务域：M1功能分析、M2原子方法拆解、M3方法接口设计、M4代码生成
- 组合方案：按顺序执行M1→M2→M3→M4

---

## M0-02 执行顺序确定

### 输入示例
选择的任务域：M1、M2、M3、M4

### 处理过程
1. 分析任务域之间的依赖关系
2. 确定执行顺序
3. 识别关键路径

### 输出示例
**执行顺序**：
1. M1 功能分析 → 理解用户登录功能的需求
2. M2 原子方法拆解 → 识别登录功能的原子方法
3. M3 方法接口设计 → 设计每个原子方法的接口
4. M4 代码生成 → 为每个原子方法生成完整代码

---

## M0-03 结果整合与优化

### 输入示例
各任务域的执行结果

### 处理过程
1. 汇总各任务域的结果
2. 整合为统一输出
3. 优化整体质量

### 输出示例
**整合结果**：
- 功能分析结果：用户登录需要验证用户名和密码
- 方法拆解结果：validateUsername、validatePassword、authenticateUser
- 接口设计结果：每个方法都有明确的输入输出
- 代码生成结果：生成完整的登录功能代码
