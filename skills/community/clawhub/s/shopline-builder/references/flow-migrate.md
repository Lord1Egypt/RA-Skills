## Step 2：搬迁前准备工作

### 引导话术

---

在正式搬迁前，做这几件事能让过程顺很多：

**① 关闭 SHOPLINE 店铺（让买家看不到装修中的店铺）**
路径：SHOPLINE 后台 → 设置 → 基本设置 → 店铺状态 → **关闭**

**② 关闭邮件通知（防止数据迁移时触发大量通知邮件）**
路径：SHOPLINE 后台 → 设置 → 通知 → 关闭所有客户通知和订单通知

**③ 关闭第三方自动化插件**（如邮件营销、ERP 系统等）
→ 防止迁移时触发自动化规则，造成混乱

**④ 盘点要迁移的数据范围**
确认你要迁移哪些：
- ✅ 商品（包括图片、描述、规格）
- ✅ 客户数据
- ✅ 历史订单
- ⬜ 博客文章（可选）
- ⬜ 自定义页面（可选）

**⑤ 广告数据备份（如果你有投放广告）**
→ 如果有 Facebook Pixel / Google Analytics，先备份好像素 ID 和历史数据

准备工作做好了吗？

---

### 常见问题应对

**Q: 一定要关闭店铺状态吗？**
→ 建议关，防止买家在迁移中下单造成订单混乱。迁移完成后再开。

**Q: 广告历史数据会丢失吗？**
→ SHOPLINE 搬迁工具支持同步 Feed ID，保留广告历史表现，具体看迁移工具设置。

## Step 4：平台对接授权

### 引导话术（根据平台分支）

---

### Shopify 用户

现在把你的 Shopify 店铺授权给搬迁工具：

1. 在搬迁工具里选择「Shopify」
2. 输入你的 Shopify 店铺地址（格式：yourstore.myshopify.com）
3. 系统会跳转到 Shopify 授权页面，点击「安装」授权
4. 授权完成后回到搬迁工具

📖 详细步骤：https://help.shopline.com/hc/en-001/articles/4669702695193

授权完成了吗？

---

### WooCommerce 用户

WooCommerce 需要通过 API Key 授权：

1. 在搬迁工具里选择「WooCommerce」
2. 输入你的 WooCommerce 网站地址
3. 系统会要求你提供 API Key 和 API Secret
4. 去 WooCommerce 后台 → WooCommerce → 设置 → 高级 → REST API → 添加密钥
5. 权限选「读取」，生成后复制填入搬迁工具

📖 详细步骤：https://help.shopline.com/hc/en-001/articles/16784282483353

遇到问题了吗？

---

### Shoplazza 用户

Shoplazza 的授权流程：

1. 在搬迁工具里选择「Shoplazza」
2. 输入你的 Shoplazza 店铺地址
3. 使用 API Token 授权（在 Shoplazza 后台 → 应用 → 私有应用 里生成）
4. 填入 Token，完成授权

📖 详细步骤：https://help.shopline.com/hc/en-001/articles/5943463861145

授权完成了吗？

## Step 6：核查迁移数据

### 引导话术

---

迁移完成！现在来核查一下数据，确保没有遗漏 🔍

**核查清单（逐项打勾）：**

**商品核查：**
- [ ] 抽查 10-20 个商品，确认名称/图片/价格/规格正确
- [ ] 检查商品描述格式有没有乱码或排版问题
- [ ] 确认商品分类正确

**客户核查：**
- [ ] 客户数量是否和原平台一致（允许小误差）
- [ ] 抽查几个客户的邮箱和地址是否正确

**订单核查：**
- [ ] 历史订单数量是否对得上
- [ ] 订单状态是否正确显示

**其他：**
- [ ] 自定义页面（关于我们 / 退款政策等）内容是否完整

发现数据问题怎么处理？
- 少量问题：手动修正
- 大量问题：联系 SHOPLINE 客服，看是否需要重新迁移

核查完成了吗？

## Step 8：绑定域名（迁移 DNS）

### 引导话术

---

现在是最关键的一步：域名切换 🌐

**这一步的目标：让你原来的域名指向 SHOPLINE 的服务器。**

**操作步骤：**

1. 在 **SHOPLINE 后台 → 设置 → 域名** 里，输入你的域名并添加
2. 系统给你 DNS 配置参数（IP 地址或 CNAME）
3. 去你的域名注册商，修改 DNS 解析：
   - 删除原来指向旧平台的 A 记录
   - 添加新的 A 记录（指向 SHOPLINE 提供的 IP）
4. 等待 DNS 生效（30 分钟 - 48 小时，通常 2-4 小时）

**⚠️ 重要：DNS 切换期间，原店铺和新店铺可能都无法正常访问（约 1-4 小时）。建议在流量低谷时段操作（凌晨最佳）。**

你的域名在哪里注册的？告诉我，我给你更具体的步骤。

---

### 常见问题应对

**Q: DNS 切换期间订单会丢吗？**
→ DNS 切换期间（一般 1-4 小时）域名可能无法访问，建议在低流量时段操作，提前在社媒告知用户。

**Q: SSL 证书怎么处理？**
→ DNS 生效后，SHOPLINE 会自动为你的域名签发 SSL 证书，无需手动操作。

**Q: 子域名（如 shop.yourdomain.com）也需要迁移吗？**
→ 需要，按同样步骤添加子域名的 CNAME 记录。

## Step 9：开店上线 + 首单冲刺

### 引导话术

---

🎉 最后一步！所有准备工作就绪，现在开店！

**开店 Checklist：**

- [ ] SHOPLINE 后台 → 设置 → 基本设置 → 店铺状态 → **开启**
- [ ] 支付切换为**真实模式**（非测试模式）
- [ ] 重新开启邮件通知（设置→通知）
- [ ] 重新开启第三方插件（如邮件营销工具）
- [ ] 下一个**测试订单**（用真实信用卡，然后取消退款）确认流程正常

**域名切换后的 SEO 注意：**
- 搬迁工具应该已经同步了原有的 URL 和 SEO 数据
- 如果 URL 结构有变化，建议在 SHOPLINE 后台设置 301 重定向
- Google 重新收录新链接可能需要 1-4 周，属于正常现象

**首单冲刺：**
- 给老客户发邮件/短信，告知已迁移到新平台
- 在社媒发布公告，引导老粉到新店
- 设置一个迁移专属折扣（如「搬家庆」8折）

**目标：3 天内出首单！** 你做得到的 💪

### 常见问题应对

**Q: 老客户的账号怎么迁移？**
→ 客户数据已迁移，但密码需要重置。建议发邮件告知客户在新平台设置密码。

**Q: 原平台的订单还会继续收款吗？**
→ 如果已经关闭原平台支付，不会。建议先确认原平台的退款处理流程完成后再关闭。

**Q: 搬迁后 SEO 流量会下降吗？**
→ 短期内可能有波动（1-4 周），但 SHOPLINE 迁移工具保留了 SEO 数据，长期影响很小。如有疑虑，联系 SHOPLINE SEO 支持团队。
