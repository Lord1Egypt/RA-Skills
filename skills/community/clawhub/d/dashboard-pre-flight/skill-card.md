## Description: <br>
Universal React dashboard audit — detect hardcoded values, tier bypasses, stale components, and onboarding inconsistencies before users see them. <br>

This skill is ready for commercial/non-commercial use. <br>

## Publisher: <br>
[tooled-app](https://clawhub.ai/user/tooled-app) <br>

### License/Terms of Use: <br>
MIT-0 <br>


## Use Case: <br>
Developers and engineers use this skill before release, deployment, or auth and onboarding changes to audit React dashboards for hardcoded values, tier handling mistakes, stale components, and inconsistent user flows. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Local search commands can inspect unintended files if project path variables are set too broadly. <br>
Mitigation: Set PROJECT_SRC and related configuration values to the intended project directories before running the audit commands. <br>
