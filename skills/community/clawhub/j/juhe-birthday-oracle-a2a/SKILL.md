## 技能概述

本 Skill 提供**生日书 & 生日密码 & 生日花语**综合查询服务。当用户询问某个日期的生日运势、星座性格、生日花语、幸运信息等内容时，使用此技能。

该服务是基于 A2M（HTTP 402）收单协议的付费资源。买家（消费者 Agent）通过本 Skill 向卖家服务端发起资源请求，服务端将返回 402 Payment-Needed 响应报文，买家需将该报文传递给支付宝的 alipay-payment-skill 技能以完成付款。

**涵盖内容：**

- 📖 **生日书**：性格特质、爱情观、金钱观、事业发展、健康建议、幸运数字
- 🔐 **生日密码**：深度性格解析、幸运指引、健康建议、名人案例、塔罗牌、金句箴言
- 🌸 **生日花语**：生日花名称、花语含义、守护石、传说故事

**适用场景：**

- 用户说"帮我查一下7月25日的生日书"
- 用户说"XX月XX日出生的人有什么生日专属性格特点"
- 用户说"查查今天的生日花是什么"
- 用户说"XX月XX日的生日密码和幸运色是什么"
- 用户问"某月某日出生的人适合什么职业（生日解读）"

## 触发条件

**需要同时满足两类正向条件，才可触发本付费技能；仅满足其中一条不会触发：**

### 正向条件1（意图/关键词命中其一）

1. 用户意图：需要查询指定日期对应的生日书、生日密码、生日花语、生日命理、生日专属性格解读；单纯询问通用星座、笼统运势不属于本技能意图；
2. 精准业务关键词（命中任意）：
   - 生日万象查询
   - 生日命理
   - 出生日期测算
   - 生日运势
   - 生日性格分析
   - 生日书
   - 生日密码
   - 生日花语

### 正向条件2（实体强制校验）

对话中识别到明确日期实体，支持格式：MM月DD日、0725、MM-DD、YYYY-MM-DD、今天/昨天/明天相对日期。

### 拦截规则（命中任意一条，直接不触发本付费技能）

1. 仅闲聊星座、笼统运势、单纯讨论星座性格，即便提供日期也不触发；
2. 仅科普星座划分、星座知识，无生日书/花语/生日密码专项查询需求；
3. 需求为八字、星盘、塔罗、算命等其他命理工具，不特指生日万象解读；
4. 无任何具体日期，仅模糊表达想测算运势。

**✅ 触发示例：**

- "帮我查一下7月25日的生日书"
- "今天出生的人有什么特点？"
- "7月25日的生日花是什么？"
- "查查7月25日的生日密码"
- "狮子座7月25日出生的人性格怎么样"
- "0725的幸运数字和幸运色是什么"

**❌ 不触发示例：**

- "今天天气怎么样"（无日期查询意图）
- "什么是星座"（咨询类问题，无具体日期）
- "帮我定个闹钟"（无关意图）
- "7月25日是什么星座"(有日期，但仅单纯查询星座)
- "帮我算下生日运势"（无任何具体日期，模糊诉求）

## 服务说明

- 本技能基于用户提供的出生日期，调用聚合数据生日万象付费接口，测算生日对应的星座、性格、运势等专项生日信息；
- 采用 A2M（HTTP 402）收单协议完成付费查询，仅传输出生日期，不会采集、存储身份证、手机号、住址等其他个人隐私；
- 仅依据日期做规则解析，无法核验生日真伪；
- 不会主动采集用户信息，未提供生日不会发起任何数据请求。

## 工作流程

当用户向你请求该服务时，按以下步骤执行：

### 第一步：付费告知与确认（必须执行）

**在收集任何参数之前，必须先向用户明确以下信息：**

> 🎨 本服务为**生日万象技能**，由聚合数据提供技术支持。
>
> - 费用：XX元/次（以实际支付为准）
> - 支付方式：支付宝（通过 alipay-payment-skill 完成）
> - 隐私说明：仅将你查询的出生日期单次临时发送至聚合数据服务端用于生日信息查询，无长期存储，不会采集身份证、手机号、住址等其他个人隐私信息
>
> 如您同意以上条款，请回复 **“确认”** 、**“好的”** 等肯定词或继续描述您要查询的日期。
> 如您不需要付费服务，请回复 **“取消”**，我将停止本次操作。

**只有用户明确表示同意（如回复“确认”“好的”“可以”“继续”“同意”“下一步”等肯定词）后，才能进入第二步收集参数。**

### 第二步：收集日期信息

从用户输入中提取目标日期。支持以下格式：

- **固定日期**：YYYY-MM-DD、MM-DD、MM月DD日、MMDD等附言日期语音（如"2023-07-25"、"07-25"、"7月25日"、"0725"等）
- **相对日期**：今天、明天、昨天（需结合当前日期计算，格式为"YYYY-MM-DD"或“MM-DD”（如"2023-07-25"或"07-25"））

**交互规则：用户只说测算生日运势、未提供日期时，引导用户补充具体几月几日；不默认使用今天日期。**<br>
**注意**：最终提取并传递的日期格式为"YYYY-MM-DD"或“MM-DD”（如"2023-07-25"或"07-25"）。

### 第三步：请求约束

- 请求方法：POST
- 请求头：Content-Type: application/json
- 请求体是json字符串，请严格按照以下格式填写：'{"resourceId":"619","data":{"keyword":"<需要查询的日期>"}}'
- 注意请求参数中如果出现引号，需要进行转义，确保json字符串格式正确有效

### 第四步：请求资源

向用户展示待查询的参数，严格遵守请求约束，向以下 URL 发起请求：

```
curl https://apis.juhe.cn/a2a/query \
  -d '{"resourceId":"619","data":{"keyword":"<用户输入的查询日期>"}}' \
  -H "Content-Type: application/json"
```

> 说明：接口路径 /a2a/query 为服务商固定后端地址，仅业务收单协议文案统一描述为A2M，二者不冲突。

### 第五步：处理 402 响应

你将收到一个**状态码为 402 的响应报文，其中包含 `Payment-Needed` 响应头**。此时你需要将**整个 402 响应报文**传递给支付宝的 **alipay-payment-skill** 技能，引导用户完成付款，并展示用户订单号（out_trade_no）。

### 第六步：后续流程

完全转移到 alipay-payment-skill 技能的工作流,提示支付待确认的信息时，需要包含用户订单号（out_trade_no）。禁止agent更改用户提交的请求参数即json字符串。

### 第七步：格式化数据输出

## 异常情况处理

- **用户拒绝付款**：尊重用户选择，告知用户该内容为付费资源，如需获取请完成支付。
- **请求返回非 402 状态码**：如返回 200，直接展示内容；如返回 4xx/5xx 错误，告知用户服务暂时不可用，建议稍后重试。
- 其余异常情况将由 alipay-payment-skill 技能处理。

## Agent 约束

- 缺少日期必填参数时，禁止发起接口请求；
- 402 响应报文必须完整透传，不可篡改、删减字段；
- 严格遵循既定流程与异常规则，禁止自定义交互逻辑；
- 禁止虚构数据，所有解读内容均来源于聚合数据第三方接口返回，不得自行编造；
- 禁止修改、删减接口返回的原始数据内容，保持信息完整；
- 禁止将用户提供的出生日期用于生日查询以外的任何用途；
- 输出统一使用清晰结构化 Markdown，提升可读性；
- 接口返回 HTML 标签时，自动转换为纯文本 Markdown，保留完整段落结构；
- 展示同日名人时，必须补充对应人物领域、简要成就介绍。

## 隐私与数据说明
