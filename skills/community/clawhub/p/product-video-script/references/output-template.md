# Output Template

Deliver every script in this structure so a creator or editor can shoot it without follow-up questions.

## Header block

```
Product: <name>
Platform: <TikTok | Reels | Shorts>
Length: <seconds>
Framework: <chosen framework>
Audience: <who + pain/desire>
Primary CTA: <single action>
```

## Shot-by-shot table

| Time | Spoken line (VO / on-camera) | On-screen text | Visual / shot direction |
|---|---|---|---|
| 0:00–0:02 | Hook line | Hook text (short, bold) | Opening shot, framing, action |
| 0:02–0:06 | … | … | New visual / pattern interrupt |
| 0:06–0:12 | … | Proof text | Demo / before-after / number |
| … | … | … | Change visual every 2–3s |
| close | CTA line | CTA text + buy cue | End card / product flat-lay |

Rules for the table:
- One row per beat; never let a row span more than ~3 seconds.
- On-screen text should reinforce, not duplicate, the spoken line.
- Every claim row needs a matching proof visual.

## Hook alternates (always provide 3)

1. <chosen hook — the one scripted above>
2. <alternate hook for A/B testing>
3. <alternate hook for A/B testing>

## Caption + hashtags

```
Caption: <1–2 lines, benefit-led, 1 emoji max unless brand voice differs>
Hashtags: <3–6, mix of broad + niche + shop tags relevant to platform>
```

## Production notes

- **Sound:** voiceover vs on-camera; trending-audio suggestion if relevant.
- **B-roll needed:** list specific shots the editor must capture.
- **Text style:** font weight/placement note for readability (keep text 2+ seconds on screen).
- **Compliance:** flag any claim that needs substantiation (results, "#1", health/earnings claims) before publishing.
