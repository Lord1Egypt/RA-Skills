# 事实校验判定规则

## 判定分类

### confirmed（无分歧）

**条件**：搜索结果中的关键信息与原文陈述一致。

**一致性判定标准**：
- 数值：完全相同或差异在合理精度范围内（如 "5亿" vs "5.02亿" → confirmed）
- 日期：完全相同或同一自然月内（如 "2025年3月" vs "2025年3月15日" → confirmed）
- 事件：事件存在且核心要素匹配（谁+做了什么）
- 实体：公司名/人名/产品名一致（允许简称/全称差异）

### divergent（有分歧）

**条件**：搜索结果中找到了相关信息，但关键要素与原文不一致。

**分歧类型判定**：

| divergence_type | 判定条件 | 示例 |
|----------------|---------|------|
| `number_mismatch` | 数值差异超出合理精度 | 原文"5亿"，实际"3亿" |
| `date_mismatch` | 日期不在同一自然月 | 原文"3月"，实际"6月" |
| `event_nonexistent` | 搜索结果明确否认或无任何记录 | 原文"收购"，实际无收购记录 |
| `entity_mismatch` | 实体信息不一致 | 原文"A公司"，实际是"B公司"被收购 |

**分歧记录要求**：
- `actual_fact` 必须是搜索结果中的原文摘录，不得改写
- `source_url` 必须是实际访问过的 URL
- `source_snippet` 必须包含足以证明分歧的关键句子

### unverified（待验证）

**条件**：无法找到可验证该事实的搜索结果。

**触发场景**：
- 两次搜索（主查询 + 备选查询）均无相关结果
- 搜索结果全部为引用同一 AI 生成内容（信息源不可靠）
- 搜索结果为付费墙内容，无法获取摘要

**记录要求**：
- `search_queries_tried`：列出所有尝试过的查询
- `note`：说明无法验证的具体原因

## 比对流程

```
1. 从搜索结果中提取关键信息（实体+数值+日期+事件）
2. 与原文的 original_text 逐要素比对
3. 所有要素一致 → confirmed
4. 任一关键要素不一致 → divergent
5. 无法提取关键信息 → unverified
```

## 特殊情况处理

| 情况 | 处理 |
|------|------|
| 多个来源信息矛盾 | 记录为 divergent，actual_fact 列出多个来源的描述，附所有 URL |
| 来源是同一篇文章的镜像 | 视为单一来源，不重复计数 |
| 数值单位换算后一致 | confirmed（如 "5 billion USD" vs "50亿美元"） |
| 来源发布日期早于事件声称日期 | 标记为 unverified，附注"来源早于事件日期" |
| 来源是维基百科 | 可用，但如与其他专业来源矛盾，优先专业来源 |
| 来源是社交媒体/博客 | 可用，但需在 actual_fact 中标注来源类型 |
