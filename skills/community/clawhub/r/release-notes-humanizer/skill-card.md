## Description: <br>
Turn a raw git log, PR title list, or changelog dump into customer-facing release notes that read like a human wrote them. <br>

This skill is ready for commercial/non-commercial use. <br>

## Publisher: <br>
[charlie-morrison](https://clawhub.ai/user/charlie-morrison) <br>

### License/Terms of Use: <br>
MIT-0 <br>


## Use Case: <br>
Product, engineering, developer relations, and marketing teams use this skill to turn raw release artifacts into clear customer-facing release notes. It supports in-app changelogs, email announcements, developer changelogs, marketing posts, status-page notes, and migration guidance for breaking changes. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Generated release notes may overstate, omit, or mischaracterize product changes if the raw changelog input is incomplete or ambiguous. <br>
Mitigation: Review generated drafts against the actual release contents before publishing. <br>
Risk: Inputs may include secrets, unreleased confidential details, or sensitive customer-impacting changes. <br>
Mitigation: Use the skill intentionally for changelog and launch-copy work, and avoid pasting confidential details unless appropriate for the workflow. <br>
