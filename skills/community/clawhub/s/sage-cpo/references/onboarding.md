# CPO Onboarding

当 `~/.sage/product/` 不存在或产品信息为空时，执行本流程。目标是建立最小产品档案，不是填完整产品手册。

## 第一轮：产品基本面

选择 3-5 个问题：

1. 你们现在卖的核心产品或服务是什么？
2. 最典型的客户是谁？他们为什么买？
3. 用户在购买你们之前，通常怎么解决这个问题？
4. 你们最想产品化或标准化的部分是什么？
5. 当前最困扰你的产品问题是什么？

写入：`products_and_services/catalog.md`、`product/users.md`、`product/packaging.md`、`product/open-questions.md`。

## 第二轮：验证与路线图

选择 3-5 个问题：

1. 过去 3 个项目或客户里，哪些需求反复出现？
2. 哪些功能、服务或交付内容最容易失控？
3. 未来 3 个月最想验证的产品假设是什么？
4. 现在有没有明确的产品指标或成功标准？
5. 如果只能做一个产品化动作，你希望先做哪一个？

写入：`product/feedback.md`、`product/experiments.md`、`product/metrics.md`、`product/roadmap.md`。

## 初始诊断

第一轮后必须输出：

1. 当前产品阶段。
2. 最大用户/产品风险。
3. 最值得验证的假设。
4. 建议的最小下一步。

第二轮后必须输出“最小产品操作系统”：

1. 当前北极星指标候选。
2. 本季度产品赌注。
3. Now / Next / Later 初稿。
4. 接下来两周的用户发现动作。
5. 第一个要写入或更新的 `~/.sage/product/` 文件。
