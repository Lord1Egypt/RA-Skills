# 决策检查清单 — 重大决策前用 3Q 审一遍

> 用 12 个决策点 + 三阶段 3Q 检查重大决策。选错了方向，后面再努力也是白费。

---

## 三阶段 3Q 决策流程

### 阶段一：决策前（信息收集）

用六问确保你的决策基础是实的：

| 3Q 维度 | 决策前要问自己 |
|---------|--------------|
| **元认知 Q** | 我真正在做的是哪个决策？还是我在解决错误的问题？ |
| **标杆 Q** | 这个领域最聪明的人面对同样选择会怎么考虑？ |
| **挑战 Q** | 如果我掌握的信息有 30% 是错的呢？ |
| **逻辑 Q** | 我的决策逻辑链条完整吗？有没有跳跃？ |
| **用户 Q** | 这个决策最终影响谁？他们的优先级是什么？ |
| **竞争 Q** | 不走这条路，alternative 是什么？ |

### 阶段二：决策中（方案评估）

用三维检查你的方案空间：

```
阶维 🏛️ 这是执行层决策还是战略层决策？
       → 别用战术勤奋掩盖战略懒惰，也别把执行问题上升到战略高度

纵维 ⏳ 这个决策的后果在 3/6/12 个月后会怎么变？
       → 短期的好处会不会造成长期的麻烦？现在的紧迫感是真的吗？

横维 🌐 除了我想到的选项，还有谁在做类似的事？他们的做法给了我什么启示？
       → 你的方案集够大吗？有没有被你忽略的替代方案？
```

### 阶段三：决策后（执行验证）

用深度 3Q 七维评分（≥ 70/70 通过）检查你的执行计划：

```
① 效果验证：有没有办法在投入最小的情况下先验证这个决策的一部分？
② 重启条件：什么情况下这个决策应该被撤销或调整？
③ 副作用：这个决策会带来什么意想不到的连锁反应？
```

## 12 决策检查点

| # | 检查点 | 具体问题 |
|---|--------|---------|
| 1 | **问题定义** | 我解决的是根因还是症状？重新定义问题后结论会变吗？ |
| 2 | **信息边界** | 我知道什么？我不知道什么？哪里可能是"未知的未知"？ |
| 3 | **假设清单** | 列出支撑这个决策的所有假设。如果任何一个假设不成立呢？ |
| 4 | **反方论点** | 最有说服力的反对意见是什么？我怎么回应？ |
| 5 | **概率思维** | 不要用"会/不会"思考，用"60% 会 / 40% 不会"——你的概率怎么来的？ |
| 6 | **时间贴现** | 这个决策的重要性在 1 天后 / 1 月后 / 1 年后会怎么变？ |
| 7 | **后悔测试** | 如果什么都不做，1 年后我会后悔吗？如果做了，会后悔吗？ |
| 8 | **最坏情况** | 最坏的结果是什么？我能承受吗？怎么防范？ |
| 9 | **机会成本** | 选这条路，放弃了什么？那个放弃的选项未来还会在吗？ |
| 10 | **可逆性** | 这个决策是否可逆？不可逆决策需要更高的证据门槛。 |
| 11 | **执行路径** | 谁来做？什么时候做？第一步是什么？要不要先做一个小验证？ |
| 12 | **退出条件** | 什么情况下我应该承认这个决策是错的并且止损？ |

---

## 快速决策三板斧（没时间走完整流程时）

1. **后悔测试（1 分钟）**：一年后回头看，更后悔做了还是没做？
2. **10/10/10 法（2 分钟）**：这个决策 10 分钟后 / 10 个月后 / 10 年后会怎么看？
3. **最坏情况检查（2 分钟）**：如果全错了，损失在可承受范围内吗？
