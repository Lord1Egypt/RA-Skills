# 续接交接协议 (Continuation Handoff)

> 续接提示词必须基于接受的素材，不只是旧计划。
> 参考 `model-mechanics.md` 机制 5：误差复合。

---

## 必需输入门控

在编写任何续接提示词前，要求：

- `project_id`
- 当前 `clip_id`
- 有效 `parent_clip_id`
- 完整故事目标
- 最终故事结局
- 下一个计划叙事任务
- 接受的上一镜头或接受的最后一帧
- `observed_end_state`（观察到的结尾状态）
- 连贯性锁定
- 精确参考注册表
- 活动平台或保守平台配置

**如果源素材不可用**，说：

> "我有故事计划，但我没有上一次生成的实际结尾。上传镜头或其最后一帧，或准确描述结尾可见内容。我不应该编造续接状态。"

**不要通过编写推测性提示词来隐藏这种不确定性。**

---

## 5 种续接类型

### 1. seamless_continuation（无缝续接）

**适用场景**：
- 同一镜头
- 同一地理
- 同一开放运动
- 相同或动机驱动的镜头续接
- 接受的上一素材作为源

**要求**：
- 观察到的结尾状态完整
- 连贯性锁定明确
- 延续运动矢量

**示例**：
```
上一镜头结尾：角色站在门前，手伸向门把手。
续接：角色推开门，镜头缓慢推进跟随进入房间。
```

---

### 2. intentional_next_shot（有意切换）

**适用场景**：
- 编辑切割合适
- 故事连续性重要
- 不承诺精确帧连续性

**要求**：
- 故事脊椎延续
- 不称为"无缝"
- 新镜头可独立设置

**示例**：
```
上一镜头结尾：角色站在门前。
切换：另一角度，角色已经坐在房间内，开始对话。
```

---

### 3. bridge_between_known_states（桥接已知状态）

**适用场景**：
- 定义的开头状态和结尾状态必须连接
- 平台支持首尾帧生成时，常用 FLF2V

**要求**：
- 开头状态明确（首帧）
- 结尾状态明确（尾帧）
- 只描述过渡过程

**示例**：
```
首帧：角色站在门外。
尾帧：角色坐在房间内。
过渡：角色推门进入，走向椅子坐下，镜头从特写拉远到中景。
```

---

### 4. repair_tail（修复尾部）

**适用场景**：
- 上一次最后几秒失败
- 修复、编辑或重生成尾部
- **在续接前修复**——从失败尾部续接会放大误差

**要求**：
- 识别失败尾部
- 决定修复/编辑/重生成
- 修复后重新评估，再续接

**示例**：
```
上一镜头结尾 3 秒失败（角色脸部模糊）。
修复：重生成最后 3 秒，保留前 12 秒。
修复后观察结尾状态，再续接下一镜头。
```

---

### 5. reanchor_after_drift（漂移后重锚）

**适用场景**：
- 身份、细节、地理、运动、音频或世界连续性降级
- 返回典律身份
- 最强接受的最后一帧
- 稳定源镜头
- 使用典律参考的新有意镜头

**要求**：
- 识别漂移
- 决定重锚策略
- 用原始参考重新锚定，不用上一次生成

**示例**：
```
上一镜头结尾：角色脸部漂移（与参考不符）。
重锚：用原始角色参考图，从稳定最后一帧开始，重新锁定身份。
```

---

## 典律规则

**实际观察状态覆盖计划状态。**

如果计划说主体到达车门，但接受的镜头停在离车门两步远，下一个提示词从离车门两步远开始。不重播终端出口，不假设主体已在车内。

**拒绝的素材永不更新典律，永不成为续接源。**

---

## 续接深度追踪

追踪 `extension_depth`（续接深度）。

- **深度 1**：正常续接，无警告
- **深度 2**：警告——重复续接可能增加连续性风险
- **深度 >=3**：强烈建议重新锚定，不要盲目续接

**当可见漂移开始时**，建议重新锚定而不是再次盲目续接。

---

## 输出合约

返回：

1. 续接类型
2. 使用的源证据
3. 观察到的结尾状态
4. 下一镜头合约
5. 连贯性锁定和允许变化
6. 要排除的已完成节拍
7. 要排除的保留未来节拍
8. 当前镜头的最终自然语言提示词
9. 更新的项目状态胶囊或缺失源证据的请求

## 实战示例

### 示例 1：无缝续接

```
续接类型：seamless_continuation
源证据：clip_01（接受的最后一帧）
观察结尾状态：角色站在门前，手伸向门把手，镜头中景，暖色调室内光

下一镜头合约：
- clip_id: clip_02
- 叙事任务：角色进入房间
- 目标时长：10 秒
- 连贯性锁定：角色外观、场景布局、光照
- 允许变化：角色位置、门的开合

排除已完成节拍：发现门（clip_01）
排除保留节拍：与房间内人物对话（clip_04）

提示词：
角色推开门，镜头缓慢推进跟随进入房间。暖色调室内光从右侧窗户射入，
角色向前走两步，环顾四周。保持角色外观不变。
```

### 示例 2：修复尾部后续接

```
续接类型：repair_tail → seamless_continuation
源证据：clip_01（前 12 秒接受，后 3 秒失败）

修复步骤：
1. 重生成最后 3 秒
2. 观察修复后结尾状态
3. 从修复后状态续接

修复后观察结尾状态：角色站在门前，门已打开，手放在门把手上

下一镜头合约：
- clip_id: clip_02
- 叙事任务：角色进入房间
- 连贯性锁定：角色外观、场景布局、光照
- 排除已完成节拍：发现门、打开门

提示词：
角色推开门，向前走进入房间。镜头跟随平移。暖色调室内光。
```

### 示例 3：漂移后重锚

```
续接类型：reanchor_after_drift
源证据：clip_01（接受的最后一帧，但角色脸部漂移）

漂移识别：角色脸部与参考不符（眼睛颜色变化）

重锚策略：
1. 用原始角色参考图 [Image1]
2. 从稳定最后一帧开始
3. 重新锁定身份

下一镜头合约：
- clip_id: clip_02
- 叙事任务：角色进入房间
- 连贯性锁定：角色身份（重新锚定）、场景布局、光照
- 参考：[Image1] 作为角色参考，保留面部特征，不要转移姿势和背景

提示词：
[Image1] 作为角色参考，保留面部特征。角色推开门，向前走进入房间。
镜头跟随平移。暖色调室内光。
```

## 核心洞察

**续接必须基于实际素材**，不是计划。

**5 种续接类型**：无缝/有意切换/桥接/修复尾部/漂移重锚——选择正确的类型。

**典律规则**：实际观察覆盖计划状态，拒绝素材不更新典律。

**续接深度警告**：>=2 时考虑重新锚定，防止误差复合（机制 5）。
