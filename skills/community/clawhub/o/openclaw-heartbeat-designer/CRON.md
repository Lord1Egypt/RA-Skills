# Cron 配置模板

## 基础语法

```
┌───────────── 分钟 (0-59)
│ ┌───────────── 小时 (0-23)
│ │ ┌───────────── 日 (1-31)
│ │ │ ┌───────────── 月 (1-12)
│ │ │ │ ┌───────────── 周 (0-6, 0=周日)
│ │ │ │ │
* * * * *
```

## 常用模板

### 整点/半点
```bash
# 每30分钟
0,30 * * * *

# 每15分钟
*/15 * * * *

# 每小时整点
0 * * * *
```

### 每天特定时间
```bash
# 早7点、晚7点
0 7,19 * * *

# 早8点、中12点、晚8点
0 8,12,20 * * *
```

### 工作日
```bash
# 周一到周五 早9点
0 9 * * 1-5

# 周一到周五 交易时段
0 10,11,13,14 * * 1-5
```

### 周末降频
```bash
# 周末只检查一次
0 10 * * 0,6
```

## 场景化配置

### 邮件高频（商务人士）
```bash
*/15 * * * *        # 每15分钟
0,30 * * * *        # 每30分钟
```

### 邮件低频（日常用户）
```bash
0 9,12,18 * * *     # 每天3次
```

### 持仓监控（A片区）
```bash
# 交易日 10:00, 11:00, 13:30, 14:30
0 10,11,13,14 * * 1-5
```

### 日历提醒
```bash
# 早晚报
0 8,18 * * *

# 会前30分钟
30 9,14,16 * * 1-5
```

### 新闻轮巡
```bash
# 早中晚3次
0 8,12,20 * * *
```

## OpenClaw cron 配置格式

在 OpenClaw 中配置定时任务时，使用标准 cron 表达式：

```
openclaw cron add "邮件心跳" "0,30 * * * *" --heartbeat
openclaw cron add "日历提醒" "0 8,18 * * *"
```

## ⏰ 常用时区 Cron 示例

### 北京时间配置（换算为UTC）

```bash
# 北京时间早8点 → UTC 00:00
0 0 * * *

# 北京时间早9点 → UTC 01:00
0 1 * * *

# 北京时间中午12点 → UTC 04:00
0 4 * * *

# 北京时间下午6点 → UTC 10:00
0 10 * * *

# 北京时间晚上8点 → UTC 12:00
0 12 * * *
```

### 美股交易时段（北京时间）

```bash
# 美股：北京时间 21:30-次日04:00 (UTC 13:30-20:00)
# 开市后每小时检查
0 13,14,15,16,17,18,19,20 * * 1-5
```

### 港股交易时段（北京时间）

```bash
# 港股：北京时间 09:30-12:00, 13:00-16:00 (UTC 01:30-04:00, 05:00-08:00)
0 1,2,3,5,6,7,8 * * 1-5
```

### 可视化生成方法

当用户配置心跳时，主动生成ASCII图帮助判断：

1. 列出所有心跳的触发时间点
2. 用 ██ 表示触发
3. 同时间有多个心跳时叠加显示
4. 分析密集度，给出评估

---

## 生成建议的原则

1. **不要过频**：邮件15分钟够了，不要5分钟
2. **合并同类**：邮件+日历可以一起检查
3. **工作日区别**：交易、新闻可以分开
4. **留空隙**：给主流程留专注时间，不要全是心跳
5. **可视化检查**：生成ASCII图帮助判断是否过载
