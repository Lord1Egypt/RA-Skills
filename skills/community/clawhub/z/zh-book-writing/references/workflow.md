# 中文小说写作工作流

## 1. 活跃书籍与工作区

先确认当前活跃书籍，只在以下目录工作：

```text
<novelist-workspace>/书库/<作品名>/
```

新书从 `assets/book-scaffold/` 复制模板。书名未定时使用 `未命名-YYYYMMDD`，并在 `00_入口.md` 写明待改名。

不要跨书混用设定、正文、文风、偏好或用户反馈；除非用户明确说明多本书共享世界观或系列主线事实。

## 2. 章节 Preflight

起草或实质修订前读取：

- `00_入口.md`
- `本书记忆/` 下所有记忆文件
- 相关 `背景设定/`、`人物关系/`、`样本文风/`、`大纲/`
- 上一章正式正文，以及当前伏笔需要回读的早前章节
- 活跃 `正文/` 目录列表

不要声称读过不存在的文件。缺口不阻塞时写入章节规划；阻塞连续性时先问用户。

必须输出：

```markdown
## 章节 Preflight
- 活跃书籍：
- 当前正式上一章：
- 下一章编号确认：
- 忽略旧稿：
- 本章阻塞问题：
```

### 章节编号

只统计正式活跃章节：

- 扫描 `正文/` 及卷目录下的正式章节文件。
- 忽略路径包含 `旧稿`、`废稿`、`archive`、`backup` 的文件。
- 文件名与 `00_入口.md`、`剧情进度.md` 冲突时，先指出冲突；优先以已确认的记忆字段作为判断依据。
- 活跃正文出现重复编号时，停止并询问，除非其中一个明确标为旧稿或废稿。
- 写作前明确说出：`下一章编号确认：第X章《标题》`。

## 3. 源书/参考资料模式

用户提供原书、片段、大纲、样本文风、笔记或链接时，先更新 `本书记忆/原作提炼.md`，形成写作者档案：

- 来源与覆盖范围：用户给了什么，实际读了什么。
- 提炼置信度与缺口：确认事实、推断内容、缺失信息、需要追问的问题。
- 核心设定：世界规则、时代质感、阵营、家族/门派/朝堂结构。
- 人物动力与关系张力：欲望、伤口、筹码、羁绊、冲突、秘密。
- 时间线与当前断点：关键事件、正史断点、角色状态。
- 未解冲突与可用钩子：旧怨、隐藏信息、承诺、反转点。
- 文风指纹摘要：节奏、叙事距离、对白密度、古风词感、章末钩子。
- 可分岔点与可写机会：2-4 个有用方向和连续性风险。

随后确认使用方式：续写、番外、改写、平行分岔、灵感借用或其他。平行分岔先更新 `本书记忆/分岔设定.md`。

## 4. 灵感孵化模式

用户给松散想法时，给 2-4 个有差异的方向，不要强行单选。可从这些轴展开：

- 情感线：慢热拉扯 / 先婚后爱 / 宿敌相恋 / 破镜重圆
- 情节骨架：复仇 / 夺权 / 江湖寻仇 / 身份错位 / 师徒禁忌
- 文风：克制古雅 / 高张力网文 / 细腻虐恋 / 爽感强节奏
- 结构：单线推进 / 双视角 / 多阵营权谋 / 单元案件串主线
- 开篇方式：婚宴变故 / 雨夜追杀 / 朝堂赐婚 / 门派灭门 / 重逢试探

把提供过的选项、用户选择和排除方向写入 `本书记忆/灵感选项.md`。

## 5. 章节规划

只追问会实质改变本章写法的缺口。信息足够时，直接输出紧凑思路：

```markdown
## 第X章《标题》思路
- 一句话核心：
- 场景顺序：3-5 个场景/动作节点
- 关系推进：
- 章末钩子：
- 本章写作原则：声音 / 层次 / 细节
- 需要确认：
```

`本章写作原则` 只选 2-3 个最影响本章成败的原则。可以选五层次，也可以选扩展技法，例如：

- 声音：本章需要明确叙事立场，避免摄像头式记录。
- 层次：情绪、权谋或动作压力要有升降，不一味平铺。
- 细节：关键情绪落在动作、物件、触感、光影或停顿上。
- 呼吸：对峙处短句压迫，转折处给一句能承重的长句。
- 活性：用动词、选择和后果替代抽象结论。
- 双线推进：关系线与剧情线互相加压。
- 物象回环：关键物件前后改变含义或成为证据。
- 关系边界：亲密停在试探、逼近、退让或重新划线。
- 对白潜台词：台词表面说事，暗里交换筹码。
- 章末动作钩子：结尾落在动作、物件、命令或外部压力上。

用户要完整规格时再用长版：

```markdown
## 第X章完整规划
- 起点状态：
- 本章目标：
- 视角人物：
- 场景顺序：
- 关键事件：
- 情绪曲线：
- 人物关系变化：
- 伏笔/回收：
- 爽点/虐点/甜点/燃点：
- 章末落点：
- 字数目标：
- 禁忌雷点：
```

章节规划可执行，或用户明确说继续后，再写正文。

## 6. 连续性冲突处理

草稿计划或正文与主线事实、当前连续性或用户偏好冲突时：

- **阻塞冲突**：停止并问用户。例如重复正式章节编号、时间线不可能、秘密边界错误、人物知道不该知道的信息。
- **小冲突但可修**：在章节规划中提出修正，且修正不改变用户意图时再继续。
- **用户有意追认改动**：平行分岔写入 `分岔设定.md`；主线接受的改动写入 `连续性罗盘.md`。

不能为了让草稿成立而悄悄覆盖主线记忆。

## 7. 写作原则、质量门与交付

起草和修订时按 `references/writing-principles.md` 使用五层次原则。原则失误通常归为 P1/P2；只有当它同时造成编号错误、连续性冲突、AI 腔硬伤、用户明确雷点或过期记忆时，才升级为 P0。

交付前执行 `references/quality-gates.md`。P0 必须修正，除非用户明确要求看粗稿。

```markdown
## 交付前自检
- 编号/连续性：
- P0：
- P1：
- 风格命中：五层次命中点 + 本书偏好命中点
- 交付文件：
```

交付默认规则：

- 正文保存为 `.md`。
- 界面支持文件/媒体时，先发送 `.md` 文件，再给简短说明。
- 用户说“发我看一下”“发我 md”“发我文件”或类似表达时，不整章粘贴，优先发送 Markdown 文件。
- 已接受的旧章节不因新增原则被自动改写；只有用户明确要求修旧章时才动 `正文/`。

## 8. 更新本书记忆

草稿通过、修订完成或收到有效反馈后，先更新当前书籍的 `本书记忆/`，再进入下一章规划：

- `剧情进度.md`：正式章节表、当前状态、未解钩子、下一步。
- `连续性罗盘.md`：时间线、伤病、地点、秘密、关系、持有物、承诺、伏笔。
- `章回复盘.md`：用户反馈、问题类型、有效写法、下次自动检查。
- `用户偏好.md`：长期有效的文风、剧情、人物互动、节奏、雷点。
- `文风指纹.md`：可复用技法观察，不保存可识别原文。
- `待问清单.md`：阻塞问题和可边写边确认的问题。

重复经验要合并；过期事实要标注或移除；相对日期要换成绝对日期。

## 版权与风格边界

允许：总结主线事实、提炼世界事实、描述关系机制、识别开放冲突、分析节奏和技法。

避免：复制长段原文、保存可识别段落、近似模仿受保护表达、把样本文字当可复用素材。

推荐记录方式：`该样本常用短句推进动作，情绪在对白后半句翻转`，而不是保存原句。
