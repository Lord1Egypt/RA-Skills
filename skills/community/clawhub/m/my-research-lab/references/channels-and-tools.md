# 推送渠道配置指南

先检测可用性，再引导配置。

**核心：说了发到哪就必须走完配置。配置不完整不能启动。**

## QQ 邮箱（推荐，内容最完整）

```
1. 访问 mail.qq.com
2. 设置 → 账户
3. 开启 POP3/SMTP 服务
4. 获取授权码
5. 在研究院配置中填入：
   - SMTP_USER: 你的QQ邮箱
   - SMTP_PASS: 授权码（不是登录密码）
```

## 企微群（适合快速浏览摘要版）

```
1. 创建/进入企微群
2. 群设置 → 群机器人 → 添加机器人
3. 复制 Webhook URL
4. 在研究院配置中填入 webhookUrl
```

## 公众号 / 小红书

- 有对应 Skill → 引导配置
- 没有则输出排版好的 Markdown/HTML 文档，用户手动发布

## 推送失败处理

研究院有三级降级：
1. 第1次失败 → 等30秒重试
2. 第2次失败 → 尝试备用渠道
3. 都失败 → 存为本地文件 + 下次对话告知用户

绝不让一次推送失败导致简报丢失。

## 沉淀空间配置

### 腾讯文档（优先推荐）
- 检测是否有腾讯文档 MCP/Skill
- 建议建一个独立的研究院专属空间
- 自动追加日报到指定文档，按日期倒序

### WeDoc Skill
- 类似腾讯文档，按用户偏好

### 本地文件（保底）
- 路径：`[工作目录]/research-lab-archive/`
- 目录结构：
  ```
  research-lab-archive/
  ├── daily/2026/05/2026-05-11.md
  ├── brainstorm/2026-05-11-[关键词].md
  ├── projects/[项目名]/
  └── README.md
  ```
- 自动维护 index.md 索引

## 搜索工具

| 工具 | 何时用 | 环境 |
|------|-------|------|
| `web_search` | 广撒网搜索 | 始终可用 |
| `web_fetch` | 知道URL，静态页面 | 始终可用 |
| `web-access` | 需要浏览器的动态页面 | 需安装 |

**web-access 安装：** `openclaw skills install web-access`
- 通过 Playwright 控制真实浏览器
- 支持登录后操作、动态页面渲染、截图
- 推荐时机：web_fetch 抓不到完整内容、需要登录、反爬严格

必须告知用户原理（控制浏览器）和风险（消耗资源），让用户自主决定是否安装，不能静默安装。
