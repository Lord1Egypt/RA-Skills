# 📄 政府公文写作专家 —— 政府材料智能代写引擎 (gov-doc-skill)

> **面向各级政府单位的综合写作助理，不只是格式指导，直接代写完整公文、报告、讲话稿、政策文件等全部内容。**
> 覆盖政府工作报告、领导讲话、政策文件、实施方案、调研报告、述职报告、通知通报、会议纪要等 15+ 类政府文书场景。

## 🎯 功能简介

面向各级政府办公室、政策研究室、业务科室的公文起草人员，**直接代写每一份政府材料的完整文案**，并附写作依据说明：

- **📝 内容代写（核心）**：根据输入参数和素材，直接写出完整材料——标题、主送机关、正文、落款——**复制即用，无需二次重写**
- **🏛 格式合规**：严格遵循《党政机关公文格式》（GB/T 9704-2012）及各文种规范
- **🧠 政治站位对齐**：自动嵌入最新中央精神、政策文件和领导指示，确保政治导向正确
- **📊 数据支撑**：数据陈述规范精准，出处明确，拒绝空泛表述
- **🎯 受众适配**：根据阅文对象自动调整行文语气、信息密度和侧重点
- **🔒 密级意识**：根据密级控制内容范围和用语选择
- **✍️ 用语规范**：使用政府公文标准用语体系，遵循18类禁用/慎用词清单

## 🏛 适用文体

| 类别 | 典型文体 | 主要受众 |
|:---|:---|:---:|
| **上行文** | 请示、报告、意见（上行的） | 上级机关/领导 |
| **下行文** | 通知、通报、批复、决定、命令 | 下级单位 |
| **平行文** | 函、议案 | 平行机关/人大 |
| **会议文书** | 讲话稿、会议纪要、主持词 | 会议参与者 |
| **政务公开文书** | 政府工作报告、公报、公告、通告 | 人大代表/社会公众 |
| **内部管理文书** | 工作总结、述职报告、调研报告、信息简报 | 内部/上级 |
| **政策法规文书** | 实施意见、管理办法、实施方案、行动计划 | 各执行单位 |

## 🚀 快速开始

### 1. 明确需求

告诉 AI 你的写作需求：

> "帮我写一份 **2025 年全区经济工作会议讲话稿**，区委书记用，重点突出**产业升级和招商引资**，字数 4000 字左右。"

### 2. 配置调用

```json
{
  "skill": "gov_doc_skill",
  "inputs": {
    "doc_type": "领导讲话稿/发言稿",
    "title": "在全区经济工作会议上的讲话",
    "issuing_body": "XX 区委书记",
    "government_level": "区县级",
    "audience": "区四套班子领导、各街道负责人",
    "purpose": "部署 2025 年经济工作",
    "key_message": "以科技创新引领产业升级，奋力实现开门红",
    "tone": "鼓舞激励",
    "word_count": 4000
  }
}
```

### 3. 获取输出

AI 会根据你的需求输出完整的材料正文，符合该文种的格式规范和行文风格。

### 💡 建议提供素材，质量翻倍

政府公文的专业性很强，AI 输出的质量直接取决于输入素材的充分程度。**建议您在提问时尽可能提供以下素材：**

| 素材类型 | 示例 | 对质量的提升 |
|:---|:---|:---:|
| **数据材料** | GDP、财税收入、民生支出等关键指标 | 避免空泛定性，让内容有据可查 |
| **工作进展** | 已完成的工作、重点项目、典型做法 | 内容具体真实，避免模板化 |
| **问题清单** | 当前遇到的困难、堵点、短板 | 问题分析精准，建议有的放矢 |
| **上级要求** | 会议精神、领导批示、考核指标 | 政策引用准确，政治站位不偏 |
| **参考稿件** | 过往同类材料、领导的风格偏好 | 风格与本单位一贯表述一致 |
| **特殊要求** | 需突出/回避的内容、敏感问题、特定表述等 | 避免方向偏差，一稿过 |

> 🔑 **核心逻辑**：仅提供主题和文种 → 得到"标准模板换皮"；提供具体素材 → 得到"量身定制的高质量材料"。素材越充分，输出越精准。

### 推荐使用方式

| 场景 | 示例提问 |
|------|---------|
| 撰写讲话稿 | "帮我写一份在教师节表彰大会上的讲话，市长用，2000 字" |
| 起草政策文件 | "起草一份优化营商环境的实施方案，要分任务和完成时限" |
| 修改润色 | "这份工作总结写得太空，帮我补充具体数据和案例" |
| 格式审查 | "检查这份通知的公文格式是否规范" |
| 提炼摘要 | "把这 5000 字的调研报告压缩成 800 字的摘要" |

## 🔗 发布指南

### 发布到 OpenAI GPTs

1. 新建 GPT → 在 **Instructions** 中粘贴 `SKILL.md` 全部内容
2. 在 **Knowledge** 中上传 `knowledge-base.md` 和 `output-templates.md`
3. 配置 Conversation starters 为常用公文写作需求

### 发布到 Dify / Coze

1. 创建 Agent → 系统提示词中粘贴 `SKILL.md`
2. 创建知识库 → 导入 `knowledge-base.md` 和 `output-templates.md`
