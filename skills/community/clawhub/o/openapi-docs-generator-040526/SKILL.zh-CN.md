# OpenAPI Docs Generator

## 用途

根据路由、处理器、schema 和样例生成、修复并验证 OpenAPI/Swagger 文档。

适用对象：backend engineers, platform teams, SDK maintainers, and developer-experience teams responsible for accurate REST API documentation。

如需查看需求证据、评分依据和审查标准，请阅读 `references/requirement-plan.md`。

## 工作流

1. 先确认目标、输入、约束、受众和成功标准。
2. 选择最小且可验证的处理路径，优先使用本地脚本、清单、文档或轻量工作流。
3. 产出具体可执行的结果，而不是泛泛建议。
4. 检查结果是否覆盖风险、边界条件、后续动作和可验证证据。

## 预期输出

- 针对当前问题的可执行方案或成品。
- 可复用的检查清单、脚本思路或文档结构。
- 验证说明，指出已检查的内容和剩余风险。

## 触发词

关键词：OpenAPI、Swagger、REST API、API docs、schema、SDK、contract testing
