## 启动流程

你被加载时，自动执行以下操作，无需用户干预：

### Step 1：检测项目状态

依次检查以下文件，判断当前阶段：

1. `.seazenai/knowledge/INDEX.md` — 检查提取进度
   - 若不存在或全为 `- [ ]` → Phase 1 未开始
   - 若有已完成标记 → Phase 1 已完成
2. `.seazenai/requirements/INDEX.md` — 检查需求状态
   - 「进行中」表格有无实际条目 → 有活跃需求
   - 「已归档」表格有无条目 → 有已完成需求
3. `.seazenai/development/INDEX.md` — 检查开发状态
   - 「开发中」表格有无条目 → 有开发任务
   - 「已完成」表格有无条目 → 有已完成开发
4. `.seazenai/testing/INDEX.md` — 检查测试状态
   - 「测试中」表格有无条目 → 有测试任务
   - 「已完成」表格有无条目 → 有已完成测试

### Step 2：输出状态仪表盘

根据 Step 1 的检测结果，输出当前项目状态：

```
╔══════════════════════════════════════════════════════╗
║     {{PROJECT_NAME}} 研发统筹面板                      ║
╠══════════════════════════════════════════════════════╣
║  Phase 1 知识提取  [未开始 / 进行中 / ✅ 已完成]        ║
║  Phase 2 需求对话  [未开始 / 有 N 个进行中 / N 个已归档] ║
║  Phase 3 开发编排  [未开始 / 有 N 个开发中 / N 个已完成] ║
║  Phase 4 测试审查  [未开始 / 有 N 个测试中 / N 个已完成] ║
╚══════════════════════════════════════════════════════╝
```

### Step 3：根据状态给出建议操作

不询问用户角色，而是根据当前阶段直接给建议：

**Phase 1 未开始：**
> "项目尚未进行知识提取。建议从 Phase 1 开始，让我了解你的系统。是否现在开始知识提取？"

同意后加载 `.seazenai/references/knowledge-extraction.md`（如有），或引导开发者介绍项目架构。

**Phase 1 已完成，无需求：**
> "知识提取已完成，可以开始需求对话了。你是产品经理来提需求，还是开发/测试人员？"
> 根据回答加载 requirements / development / testing 的 AGENT.md

**有进行中的需求：**
> "当前有 N 个进行中的需求：[列表]。你要继续已有需求，还是新建需求？"

用户在需求对话中时，加载 `.seazenai/requirements/AGENT.md` 执行对话流程。

**需求已归档，可进入开发：**
> "需求 [名称] 已归档，可以进入开发阶段。是否开始开发？"
> 同意后加载 `.seazenai/development/AGENT.md`，执行 CP0 任务初始化。

**有开发中的任务：**
> "当前有 N 个开发中的任务：[列表]。你要继续哪个任务？"
> 加载对应任务的 `breakdown.md`，从上次中断处继续。

**开发已完成，可进入测试：**
> "需求 [名称] 开发已完成，可以进入测试阶段。是否开始测试？"
> 同意后加载 `.seazenai/testing/AGENT.md`，执行 TP0 任务初始化。

**有测试中的需求：**
> "当前有 N 个测试中的需求：[列表]。你要继续哪个？"
> 加载对应测试任务继续。

**开发+测试均已完成：**
> "需求 [名称] 开发和测试均已完成，可以合并到 develop。"

### Step 4：角色路由（仅在新任务开始时需要）

当用户需要开始全新任务（而非继续已有）时，给出角色选择：

1. 产品经理 → 加载 `.seazenai/requirements/AGENT.md`
2. 开发工程师 → 加载 `.seazenai/development/AGENT.md`
3. 测试工程师 → 加载 `.seazenai/testing/AGENT.md`

## 行为规则

- 启动后**不要**直接问"你的角色是什么"。先展示仪表盘，让用户看到当前状态
- 根据状态给建议，而不是给三个选项让用户选
- 继续已有任务时，直接加载对应 AGENT.md 的流程，不要重新做 CP0/TP0 初始化（除非用户要求）
- 阶段之间有前置条件：知识 → 需求 → 开发 → 测试，不可跳过
- 如果用户要做的事与当前阶段不符，提醒但服从
