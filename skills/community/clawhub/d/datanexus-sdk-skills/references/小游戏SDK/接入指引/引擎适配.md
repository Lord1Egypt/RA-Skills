# 微信小游戏 SDK 引擎适配指南

本文档介绍微信小游戏 SDK 在不同游戏引擎下的引入方式和初始化差异。请根据所使用的引擎选择对应章节进行适配。

---

## 原生小游戏

将 SDK 文件放入项目根目录或 `libs/` 目录下，使用 `require` 引入：

```javascript
const SDK = require('./libs/sdk.js');
const sdk = new SDK({
  user_action_set_id: 'your_user_action_set_id',
  secret_key: 'your_secret_key',
  appid: 'wx123xyz123xyz123x',
});
```

---

## LayaAir 引擎

**方式一：脚本引入** — 将 `sdk.js` 放入 `bin/libs/`，在 `bin/index.js` 中引入。

**方式二：npm 引入（LayaAir 2.x+）**

```bash
npm install @dn-sdk/minigame
```

> 注意：导出小游戏时不要对 SDK 文件混淆，需排除在混淆范围之外。

---

## CocosCreator 引擎

**CocosCreator 2.x/3.x：** 将 `sdk.js` 放入 `assets/libs/`，在属性检查器中勾选「导入为插件」。

TypeScript 类型声明：

```typescript
declare class SDK {
  constructor(options: { user_action_set_id: string; secret_key: string; appid: string; });
  setOpenId(openId: string): void;
  setUnionId(unionId: string): void;
  track(action_type: string, action_param?: object): void;
}
```

> 勾选「导入为插件」后 SDK 不被模块系统处理，直接作为全局脚本加载。构建时需排除代码压缩。

---

## Egret（白鹭）引擎

将 `sdk.js` 放入 `libs/`，在 `egretProperties.json` 中添加模块声明，执行 `egret build` 编译。

---

## Unity 引擎（微信小游戏转换工具）

导出后在 `minigame/` 目录放入 `sdk.js`，在 `game.js` 顶部引入并挂载到 `GameGlobal.trackSDK`。

```javascript
GameGlobal.trackSDK.setOpenId('用户的openid');
GameGlobal.trackSDK.track('PURCHASE', { value: 600 });
```

> 每次重新导出后需重新添加 SDK 文件。

---

## 通用注意事项

1. **避免 SDK 被混淆**：否则会导致 `data not valid` 报错
2. **初始化时机**：尽早初始化，不遗漏事件
3. **全局访问**：挂载到 `window.trackSDK` 或 `GameGlobal.trackSDK`
4. **多数据源**：最多支持 4 个 SDK 实例
