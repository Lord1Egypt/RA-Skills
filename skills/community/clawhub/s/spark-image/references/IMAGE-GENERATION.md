# 图片生成详细说明

## 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `prompt` | string | 是 | 图片描述文本，最大 2000 字符 |
| `width` | integer | 否 | 宽度，需与支持的 2K 推荐尺寸组合一致 |
| `height` | integer | 否 | 高度，需与支持的 2K 推荐尺寸组合一致 |
| `style` | string | 否 | 附加到提示词中的风格描述，如"油画风格，厚涂笔触" |
| `image_count` | integer | 否 | 生成数量，1-4，默认 1 |

## 推荐提示词技巧

- 运用连贯的自然语言描述画面内容（主体+行为+环境等）
- 用短词语描述画面美学（风格、色彩、光影、构图等）
- 写出图像用途和类型，例如：PPT封面背景图、广告海报设计
- 文字排版描述：将文字内容置于双引号内，通过指令描述文字的大小、字体、颜色、风格和位置

**计费公式**：点数 × 单价（元/点），默认单价 0.01 元/点。

## 图片格式

返回格式为 **PNG**，Base64 编码。需要时可在客户端解码保存为图片文件。

## 多图生成

`image_count` 参数支持 `1-4` 张图片。

- 系统会按张累计计费。
- 多图通常会比单图耗时更久。
- 为保证稳定性，多图请求可能按张生成后再聚合返回，因此结果可能是逐张完成的。
