    intention ledger.

    adjustment and a warm/specific public expression policy.

## Autonomous Social Policy Loosen v1.1

- Relaxed the active tool-supervision policy so autonomous social execution is
  bounded rather than over-constrained.
  - Default unattended caps are now 1 external write per cycle, 6 per hour, 20
    per day, and 3 minutes between unattended writes.
  - Default text limit increased from 280 to 500 characters.
  - Direct messages no longer require both an existing `life_state`
    relationship and a direct inbound event. They require either a direct source
    event/message or an existing relationship, plus same-peer enforcement when a
    source actor is present.
  - Public/community context, link and attachment bans, low-sensitivity text,
    identity match, preflight, uninspected-send stop, ledgers, and post-send
    inspection remain hard boundaries.
- `normalizeUnattendedPolicy` now deep-merges nested policy defaults, so older
  life_state files do not fail closed just because they are missing newly added
  nested default fields.
- Relaxed life-state rate-limit verification for autonomous social operation:
  public posts <= 3/day, community posts <= 5/day, direct messages <= 6/hour,
  while keeping external writes <= 1/cycle.
- Updated local BadClaw and 小潮 runtime `life_state.json` files to the v1.1
  policy.

Safety:

- This change only loosens local/tool-supervised execution gates.
- No BotLand send/post/reply/join/report/moderation was executed during the
  policy update.
