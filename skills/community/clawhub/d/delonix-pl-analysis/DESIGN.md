## 三、财务逻辑框架

### 3.1 财务模型理论基础

#### 收入模型

```
总收入 = RevPAR × 房间数 × 经营天数

其中：
RevPAR = ADR × OCC

客源收入结构：
总客房收入 = Σ(各渠道占比 × 渠道ADR × 已售间夜数)
```

**理论依据**：
- **收益管理理论**（Revenue Management）：源于航空业，核心是"在正确的时间以正确的价格卖给正确的客户"
- **价格歧视策略**：不同渠道差异化定价，最大化收益
- **需求预测模型**：基于历史数据的季节性分析

#### 成本模型

```
总成本 = 固定成本 + 变动成本

固定成本：
- 租金/折旧
- 基础人工成本
- 基础能耗

变动成本：
- 随OCC变化的成本（客房易耗品、布草洗涤、部分人工）
- 渠道佣金（随收入变化）
```

**理论依据**：
- **成本性态分析**（Cost Behavior Analysis）：区分固定/变动成本
- **作业成本法**（Activity-Based Costing）：成本动因分析
- **本量利分析**（CVP Analysis）：保本点测算

#### 利润模型

```
GOP = 总收入 - 运营成本
GOP Margin = GOP / 总收入

EBITDA = GOP - 固定成本摊销
EBITDA Margin = EBITDA / 总收入
```

**理论依据**：
- **杜邦分析法**（DuPont Analysis）：利润率分解
- **经济增加值**（EVA）：考虑资本成本的真实盈利能力

### 3.2 保本点分析模型

#### 基本公式

```
盈亏平衡点（BEP）= 固定成本 / 贡献边际率

其中：
贡献边际 = 单位售价 - 单位变动成本
贡献边际率 = 贡献边际 / 单位售价
```

#### 酒店行业应用

```
保本OCC = 固定成本 / (ADR × 房间数 × 经营天数 × 贡献边际率)

安全边际 = 实际OCC - 保本OCC
安全边际率 = 安全边际 / 实际OCC
```

**理论依据**：
- **本量利分析**（Cost-Volume-Profit Analysis）
- **敏感性分析**：参数变化对结果的影响程度
- **情景分析**：不同假设条件下的多情景预测

### 3.3 敏感性分析方法

#### 单因素敏感性分析

```python
def sensitivity_analysis(base_value, param_name, change_range):
    """
    单因素敏感性分析
    
    参数：
    - base_value: 基准值
    - param_name: 参数名称（如ADR、OCC、人工成本率）
    - change_range: 变化范围（如[-20%, -10%, 0%, +10%, +20%]）
    
    输出：
    - 每个变化幅度对应的利润变化
    - 敏感性系数 = (利润变化率) / (参数变化率)
    """
    results = []
    for change in change_range:
        new_value = base_value * (1 + change)
        new_profit = calculate_profit(new_value)
        profit_change = (new_profit - base_profit) / base_profit
        sensitivity = profit_change / change
        results.append({
            'param_change': change,
            'profit_change': profit_change,
            'sensitivity': sensitivity
        })
    return results
```

#### 多因素交叉分析

```
场景矩阵：
         ADR-10%  ADR 0%  ADR+10%
OCC-10%    场景1    场景2    场景3
OCC 0%     场景4    场景5    场景6
OCC+10%    场景7    场景8    场景9

输出：9种场景下的利润预测
```

### 5.3 异常检测算法

```python
def detect_anomaly(value, historical_data, threshold=2):
    """
    基于Z-Score的异常检测
    
    参数：
    - value: 当前值
    - historical_data: 历史数据序列
    - threshold: Z-Score阈值（默认2，即95%置信区间）
    
    返回：
    - is_anomaly: 是否异常
    - z_score: Z分数
    """
    mean = np.mean(historical_data)
    std = np.std(historical_data)
    z_score = (value - mean) / std
    
    is_anomaly = abs(z_score) > threshold
    
    return {
        'is_anomaly': is_anomaly,
        'z_score': z_score,
        'deviation': f"{abs(z_score):.2f}倍标准差"
    }
```

### 7.2 参数自优化算法

```python
def optimize_parameters(historical_results, target_metric='gop_margin'):
    """
    参数自优化算法
    
    原理：基于历史预测结果与实际结果的偏差，
         自动调整预测模型参数
    
    参数：
    - historical_results: 历史预测与实际数据
    - target_metric: 优化目标指标
    
    返回：
    - optimized_params: 优化后的参数
    """
    
    # 计算预测误差
    errors = []
    for record in historical_results:
        predicted = record['predicted']
        actual = record['actual']
        error = (predicted - actual) / actual
        errors.append(error)
    
    # 计算系统性偏差
    systematic_bias = np.mean(errors)
    
    # 调整参数
    if systematic_bias > 0.05:
        # 预测系统性偏高，下调系数
        adjustment = 1 - systematic_bias * 0.5
    elif systematic_bias < -0.05:
        # 预测系统性偏低，上调系数
        adjustment = 1 - systematic_bias * 0.5
    else:
        adjustment = 1
    
    optimized_params = {
        'coefficient_adjustment': adjustment,
        'confidence_level': 1 - np.std(errors)
    }
    
    return optimized_params
```
