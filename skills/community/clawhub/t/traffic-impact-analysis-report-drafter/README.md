# traffic-impact-analysis-report-drafter

Draft a Traffic Impact Analysis (TIA) — also called a Traffic Impact Study (TIS) — for a proposed land-development project, in the structure that state DOTs and municipal traffic-engineering reviewers expect: scope confirmation → study area and analysis periods → existing conditions → background traffic → trip generation (ITE) → trip distribution and assignment → no-build vs build LOS / queueing / turn-lane warrants → mitigation → conclusions and recommendations.

## When to use

A transportation engineer — civil / traffic engineer, traffic operations PE, site-development consultant, transportation planner, or land-use approval reviewer — has:

- a defined development (land use, size, location, access concept),
- an agency or jurisdiction whose TIA guidelines apply (state DOT, county, or municipality),
- access to count data (existing turning-movement counts, ADT counts, or willingness to commission them), and
- needs a TIA report that supports a site-plan, subdivision, rezoning, special-use permit, or driveway-permit application.

The skill does **not** stamp drawings, model traffic with proprietary software, design geometric improvements, or commit the agency to an approval. It produces a DRAFT report for a licensed Professional Engineer (PE) or licensed Professional Traffic Operations Engineer (PTOE) to verify, stamp, and submit.

## What it produces

- Scope confirmation memo (study intersections, analysis years, peak periods, methodology)
- Existing-conditions summary (geometry, control, current LOS / volume-to-capacity)
- Background traffic forecast (build-out year, annual growth, committed-development trips)
- ITE Trip Generation table (land-use code, independent variable, rate / equation source, AM / PM / Saturday peak, pass-by and internal-capture reductions with rationale)
- Trip distribution and assignment narrative
- Capacity / LOS results table for each study intersection under No-Build, Build, and Build-with-Mitigation scenarios
- 95th-percentile queueing analysis for critical movements
- Auxiliary-turn-lane warrant check (left-turn and right-turn) per AASHTO / NCHRP 745 / agency standard
- Site-access (driveway) sight-distance and spacing review
- Multimodal section: pedestrian, bicycle, transit, ADA compliance
- Crash / safety summary for the study area
- Mitigation recommendations with conceptual cost magnitude
- Conclusions and PE / PTOE sign-off block
- Open-issues log

## Inputs the agent will ask for

One question at a time:

1. Project name, location, and proposed land use (with ITE land-use code if known)
2. Development size (DU, sf GLA, rooms, seats, sf GFA, bays — whatever the ITE independent variable is)
3. Agency / jurisdiction whose TIA guidelines apply (and any specific manual or memo)
4. Build-out year and analysis years (typically opening and opening + 5 / + 10 / + 20)
5. Peak periods to analyse (weekday AM, weekday PM, Saturday midday, school AM / PM, special generator)
6. Trigger threshold result (does the project meet the agency's TIA threshold — typically 100+ new peak-hour trips)
7. Study-area intersections proposed (with rationale tied to agency screening criteria)
8. Count data on hand (turning-movement counts, ADT, dates, source) or whether counts must be commissioned
9. Adjacent committed developments to include in background traffic
10. Site-access concept (full-access driveway, RIRO, signal, median treatment)
11. Multimodal context (bike / ped / transit constraints; ADA hot spots)
12. Approval pathway (site plan, subdivision, rezoning, special-use, driveway permit, NEPA / CEQA tier)
