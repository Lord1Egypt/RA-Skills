# Language Policy

Skill2Team package documentation, generated package guides, runtime manifests, and next-use prompt templates are written in English by default.

## Default Rule

- Use English for startup pages, questions, route prompts, reports, tables, generated package documentation, and final deliverables.
- Keep product names, runtime names, file names, code identifiers, JSON keys, TOML keys, and command examples unchanged.
- Do not introduce translated prompt variants into machine-readable manifests.

## Explicit Override

The user may request another language for a specific conversation response. That override applies only to the requested human-facing response unless the user explicitly asks to change generated package artifacts.

Machine-readable files should remain English for portability.
