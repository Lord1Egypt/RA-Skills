## Known Risks and Mitigations: <br>
Risk: Generated InfluxDB write, retention, or deletion-related commands can affect production data, including intentional expiry of older data. <br>
Mitigation: Review commands before execution, confirm retention windows and target buckets or databases, and test production-impacting changes outside production first. <br>
Risk: InfluxDB 1.x and 2.x use different query languages and resource models, so applying the wrong syntax can cause failed or misleading operations. <br>
Mitigation: Confirm the target InfluxDB version before using generated Flux, InfluxQL, bucket, database, organization, or token guidance. <br>
