## Description: <br>
Audit React, Vue, or Svelte component libraries to find unused components, inconsistent props, missing documentation, accessibility issues, missing tests, and naming convention violations. <br>

This skill is ready for commercial/non-commercial use. <br>

## Publisher: <br>
[charlie-morrison](https://clawhub.ai/user/charlie-morrison) <br>

### License/Terms of Use: <br>
MIT-0 <br>


## Use Case: <br>
Developers and design-system maintainers use this skill to audit frontend component libraries for unused components, documentation gaps, accessibility issues, inconsistent props, missing tests, and naming convention drift. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Generated reports may include private file paths, component names, and source-derived findings. <br>
Mitigation: Run the skill only from the intended project folder and review reports before sharing them outside the project. <br>
Risk: Pattern-based checks for usage and accessibility can produce false positives or miss dynamic behavior. <br>
Mitigation: Treat findings as review prompts and confirm them with project tests, code review, and dedicated accessibility tooling where needed. <br>
