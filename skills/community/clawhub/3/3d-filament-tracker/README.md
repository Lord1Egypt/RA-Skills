# 3D Filament Tracker

把 3D 打印耗材订单截图整理成可交互的 HTML 库存追踪器。

> Claude Code skill，单文件 HTML 输出，无需任何框架。

## 它能干嘛

3D 打印玩家往往一买就是几十卷耗材，散在多个店铺多个订单里。这个 skill：

1. **读图整理** —— 把订单截图丢给 Claude，自动识别材料 / 品牌 / 颜色 / 卷数
2. **生成 HTML 跟踪器** —— 单文件、双击就能用、不联网不需要后端
3. **双步进器交互** —— 每行同时跟踪「在用」+「已用光」，剩下的就是库存
4. **状态持久化** —— localStorage 自动保存，JSON 可导出导入
5. **联动重算** —— 标一卷已用，所有合计自动减

## 关键工程点

### localStorage 按 file URL 隔离

把 HTML 从 `~/Downloads/` 搬到 `~/Claude Code/`，**之前的状态读不到**。所以必须提供 JSON 导出/导入。搬路径前先导出。

### 现有 ≠ 原始

「现有」列 = 原始购买数 − 已用光。步进器分母（`/N`）固定显示原始数，所以代码用 `tr.dataset.total` 锁住原始值，从不依赖 `.qty` 单元格的当前内容。

### 合计永远 JS 算

颜色合计 / 材料小计 / 总计的初始数字只是占位，加载时立刻被 `recomputeAggregates()` 覆盖。这避免了 HTML 写死的数字和实际状态不同步。

### 双步进器约束

`在用 + 已用 ≤ 总数`。任一步进器 +1 前都先检查；超出直接 `return`，不显示报错。
