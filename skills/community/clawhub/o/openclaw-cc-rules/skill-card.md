## Description: <br>
OpenClaw CC Rules provides a structured programming workflow for OpenClaw sessions, covering planning, task tracking, read-only exploration, Git safety, and ordered multi-file changes. <br>

This skill is ready for commercial/non-commercial use. <br>

## Publisher: <br>
[badxtdss](https://clawhub.ai/user/badxtdss) <br>

### License/Terms of Use: <br>
MIT-0 <br>


## Use Case: <br>
Developers use this skill to make OpenClaw follow a conservative software development workflow: explore code before changing it, plan non-trivial work, track tasks, avoid risky Git operations, and verify changes after implementation. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Broad automatic activation may apply the workflow rules in more coding sessions than intended. <br>
Mitigation: Review the activation rules and narrow them if explicit opt-in is preferred. <br>
Risk: Installing from an unreviewed third-party source can introduce rule changes outside the validated release evidence. <br>
Mitigation: Inspect the release contents and rely on the server security summary before deployment. <br>
