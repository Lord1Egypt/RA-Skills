## 一、论文整体结构

### 前置部分（必需）
1. **封面** (cover)
   - 论文题目（中英文）
   - 作者姓名
   - 导师姓名
   - 学科专业
   - 提交日期

2. **原创性声明和使用授权** (declaration)
   - 原创性声明
   - 使用授权书
   - 签名页

3. **摘要** (abstract)
   - 中文摘要（500-1000字）
   - 关键词（3-5个）
   - 英文摘要 (Abstract)
   - Keywords

4. **目录** (toc)
   - 自动生成
   - 包含章节编号和页码

5. **符号对照表**（推荐）
   - 符号说明
   - 缩略语表

### 正文部分

6. **第一章：绪论**
   - 研究背景
   - 研究意义
   - 技术瓶颈/研究空白
   - 科学问题
   - 研究问题/科学问题
   - 主要贡献
   - 论文结构

7. **第二章：相关工作/文献综述**
   - 研究现状分类综述
   - 现有方法对比分析
   - 研究空白识别

8. **第三章至第N-1章：核心内容**
   - 方法/模型/算法
   - 实验设计
   - 结果分析

9. **第N章：总结与展望**
   - 工作总结
   - 贡献闭合
   - 研究局限
   - 未来方向

### 后置部分（必需）

10. **参考文献** (bibliography)
    - GB/T 7714 格式
    - 按引用顺序编号

11. **致谢** (acknowledgment)
    - 感谢导师
    - 感谢合作者
    - 感谢资助机构

12. **攻读学位期间发表论文目录** (resume/publications)
    - 已发表论文
    - 已接收论文
    - 在投论文（可选）

13. **附录**（如有）
    - 补充材料
    - 代码清单
    - 详细推导

## 二、章节编号规范

### 层级结构
```
第1章 绪论
  1.1 研究背景
    1.1.1 问题描述
    1.1.2 挑战分析
  1.2 研究意义
第2章 相关工作
  2.1 ...
```

### 直属小节数量

- 主体章节的直属 `\section` 默认控制在 **5 节以内**。这不是机械字数要求，而是为了让每章形成清晰的论证闭环。
- 推荐结构为：引言、基础理论/问题描述、模型/算法/框架、实验/案例/应用、本章小结。
- "本章小结"默认写成一个自然段，按"问题/目标 -> 本章工作/方法 -> 关键过程/证据 -> 结果价值 -> 对全篇主线的支撑"收束；除非学校模板或导师要求，不拆成多个短段或列表。详见 [`thesis-writing-guide.md`](thesis-writing-guide.md) 的"正文章末小结"一节。
- 当一章超过 5 个直属小节时，优先合并相邻功能：
  - 将“数据预处理”“变量选取”“问题定义”合并为“问题描述与建模基础”。
  - 将“模型框架”“模型建模”“算法设计”整理为一个方法主节，下设 `\subsection`。
  - 将“实验设置”“结果分析”“鲁棒性分析”归入实验或案例主节，下设 `\subsection`。
- 若确有校级模板或导师要求需要超过 5 节，应在章引言或导语中说明各节如何围绕同一章标题展开。

### 章标题与小节标题扣合

- 正文方法章、模型章、算法章和系统应用章的章标题应尽量体现“研究对象 + 问题/任务 + 方法/路径”。
- 小节标题应承接章标题中的对象、问题或方法，不应只写“数据采集”“结果讨论”“模型建立”等可移动到任意章节的泛化标题。
- 泛化标题可以保留，但导语必须说明它如何服务于本章对象和问题。

### 编号格式
- 章：第X章（中文数字或阿拉伯数字）
- 节：X.1
- 小节：X.1.1
- 段：X.1.1.1（尽量避免超过四级）

### 标题后导语规范
- 每一章、每一节、每一个小节，以及需要展开论述的四级标题后，都应先写一段导语。
- 导语至少说明四件事中的两到三件：本层级讨论对象、写作目的、与上一层级的关系、下文组织方式。
- 禁止标题后直接进入列表、公式、图表、算法环境或下一级标题，而没有任何过渡说明。
- 导语宜写成完整段落，而不是一句过短的提示语；推荐使用“本章/本节将……”“下面从……展开……”等显式引导表达。
- 正文各章（第 2 章至结论前）的**章引言**宜写成承上启下两段、约 300~500 字：第①段用章节号（“第 X 章”）承接前章，第②段交代本章问题、思路与各节安排；避免“上一章/上文”相对指代。详见 [`thesis-writing-guide.md`](thesis-writing-guide.md) 的“正文章引言”一节。
- 正文各章的**本章小结**宜写成单段收束：不复述目录，不新增结果，不替代最终结论；它应概括本章问题、方法、证据、价值以及对全文主线的支撑。
- 例外部分通常包括：摘要、目录、参考文献、致谢、附录等非主体论证单元。

## 三、页面设置

### 页边距
| 位置 | 尺寸 |
|------|------|
| 上 | 3.0 cm |
| 下 | 2.5 cm |
| 左 | 2.5-3.0 cm |
| 右 | 2.5 cm |

### 页眉页脚
- 页眉：章标题或"XX大学博士学位论文"
- 页脚：页码（居中）
- 摘要页使用罗马数字
- 正文使用阿拉伯数字

### 字体字号
| 内容 | 字体 | 字号 |
|------|------|------|
| 正文 | 宋体 | 小四/五号 |
| 章标题 | 黑体 | 三号 |
| 节标题 | 黑体 | 四号 |
| 表题/图题 | 宋体 | 五号 |

## 四、常见检查清单

### 前置部分
- [ ] 封面信息完整准确
- [ ] 声明页已签名
- [ ] 摘要字数符合要求
- [ ] 关键词数量合适
- [ ] 目录页码正确
- [ ] 符号表完整

### 正文部分
- [ ] 第一章包含研究背景、意义、贡献
- [ ] 绪论形成“背景 -> 技术瓶颈/研究空白 -> 科学问题 -> 本文贡献 -> 章节安排”的收束链
- [ ] 文献综述覆盖主要相关工作
- [ ] 每一章、节、小节与四级标题后均有导语段落
- [ ] 正文章末"本章小结"用一个自然段完成问题、方法、证据、价值和主线支撑的收束
- [ ] 方法章节说明每个核心模块的动机、设计和技术优势
- [ ] 方法描述清晰完整
- [ ] 实验设计合理
- [ ] 结果分析客观
- [ ] 最后一章回应绪论贡献，有总结、局限和展望

## 五、章节写作主线

做章节级改写或写作方案时，优先读取 [`thesis-writing-guide.md`](thesis-writing-guide.md)。结构检查不只看章节是否存在，还要看正文是否围绕同一科学问题递进展开：

```text
研究背景 -> 技术瓶颈/研究空白 -> 科学问题 -> 本文方法/章节工作 -> 实验证据 -> 贡献闭合 -> 局限与展望
```

若章节只是并列罗列多个工作，应补充“本章基于上一章……”或“围绕同一科学问题，本章进一步……”一类桥接关系。
