# v2.8.28 (2026-05-06)

**修 v2.8.27 位置错：finalize 抽成 helper 覆盖所有 final 早退点**

## 背景

赵博 5/6 01:50 群聊截图：完整 BOT 回复 700+ 字看到了，但**没有立刻显示"内容由AI生成"标识，无法复制**。

排查 gateway.log：

```
01:50:36.880 deliver-start kind=block textLen=563  ← LLM 完整回复在这条
01:50:37.271 deliver-start kind=final textLen=0 mediaCount=0
01:50:37.272 deliver-done final 1ms  ← !!! v2.8.27 finalize log 一次都没出
```

**v2.8.27 修复跑了但没生效**。grep `deliver final 内容空` log 0 次命中。

## 根因

v2.8.27 把 finalize 加在 `if (!finalText)` 早退点（line 528），但 deliver 函数还有**更早的早退点** line 463：

```typescript
const mediaUrls = info.kind === "final" ? mergeDeferredMediaUrls(...) : incomingMediaUrls;
if (!text && mediaUrls.length === 0) {
  return;  // ← !!! v2.8.27 漏修这条，textLen=0 mediaCount=0 在这里就 return 了
}
```

`info.kind === "final" && text === "" && mediaUrls.length === 0` —— 5/6 01:50 完美命中这个分支。stream 永远不被终结，placeholder 卡 last=false 等 120s timeout 才被 v2.8.24 兜底。

## 修复

抽通用 helper `finalizeStreamIfNeeded(reason)`，所有 final 早退点统一调用：

```typescript
const finalizeStreamIfNeeded = async (reason: string): Promise<void> => {
  if (streamSettled || isEvent || !streamId) return;
  try {
    const settleText = computeCurrentPlaceholder();
    await params.client.replyStream(params.frame, streamId, settleText, true);
    streamSettled = true;
    streamFinalized = true;
    stopPlaceholderKeepalive();
    console.log(
      `[wecom-ws] finalizeStreamIfNeeded(${reason}): 用 last=true 终结 streamId=${streamId} 解锁企微 UI（peer=${peerKind}:${peerId}）`,
    );
  } catch (e) {
    console.warn(`[wecom-ws] finalizeStreamIfNeeded(${reason}) 失败...`);
  }
};
```

两处早退点都换成 helper 调用：

```diff
  if (!text && mediaUrls.length === 0) {
+   if (info.kind === "final") {
+     await finalizeStreamIfNeeded("final-empty-textmedia");
+   }
    return;
  }
  ...
  if (!finalText) {
+   if (info.kind === "final") {
+     await finalizeStreamIfNeeded("final-no-text");
+   }
    return;
  }
```

未来再有早退点直接复用 helper，不会再漏。

## 现在 5/6 01:50 场景重现

```
inbound → dispatch (11s)
deliver kind=block textLen=563  ← 完整回复 partial 推送
  → replyStream(streamId, blockText, false) ← last=false
deliver kind=final textLen=0 mediaCount=0
  → 进 line 463 早退分支
  → v2.8.28 finalizeStreamIfNeeded("final-empty-textmedia")
  → replyStream(streamId, computeCurrentPlaceholder(), true) ← !!! last=true 终结
  → 企微 client UI 立刻显示"内容由AI生成" + 解锁交互 ✅
```

## 经验沉淀

**v2.8.27 是经典的 "fix in wrong place"**：

1. 早退点不止一个 —— 加防御代码必须 grep **所有** `return;` 看是否都覆盖
2. 应该一开始就抽 helper，而不是 inline 复制
3. 单测应当覆盖所有 final 路径分支（finalText 空 / mediaCount=0 + textLen=0 / mediaCount>0 + textLen=0 等）

v2.8.28 是用 helper 重构 + 加 grep-able log。下次类似场景一搜 `finalizeStreamIfNeeded(` 就能看到所有调用点。
