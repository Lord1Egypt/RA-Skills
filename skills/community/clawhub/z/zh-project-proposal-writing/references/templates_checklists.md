## 8. 自检清单

### 8.1 形式审查清单

- 是否使用当年模板。
- 是否符合字数页数和格式。
- 是否满足申请资格和限项。
- 是否选择正确项目类别和代码。
- 是否准备全部附件。
- 是否完成签章、日期、承诺和在线确认。
- 是否处理伦理、数据、安全、保密和人工智能使用声明。

### 8.2 内容质量清单

- 标题是否聚焦。
- 摘要是否讲完整故事。
- 立项依据是否推出缺口。
- 目标是否可判断。
- 内容是否递进。
- 方案是否具体。
- 创新是否有比较对象。
- 基础是否直接支撑项目。
- 成果是否可验证。
- 风险是否有替代方案。

### 8.3 科技部与上海清单

- 是否逐条覆盖指南指标。
- 是否写清任务接口和集成验证。
- 是否有场景、用户或示范条件。
- 是否有可验收成果。
- 是否说明合作单位任务和知识产权。
- 是否预算对应任务。

### 8.4 国家自然科学基金清单

- 科学问题是否明确。
- 假设是否有前期依据。
- 内容是否围绕假设递进。
- 方案是否写清关键方法和验证。
- 创新是否不是单纯方法新。
- 前期基础是否服务本项目。
- 预期结果是否写科学认识。

## 9. 评审意见回复模板

针对“【评审意见】”，申请书已在【章节】进行修改：一是补充【证据/说明】，明确【问题】；二是调整【目标/内容/方案】，使其与【指南/科学问题】一致；三是增加【风险控制/验证方式/附件】，提高项目可行性。修改后，项目主线由【原主线】进一步聚焦为【新主线】。

## 10. 可复用提示词

### 10.1 解读指南

请基于我提供的指南和项目想法，生成指南约束矩阵、指标覆盖矩阵、形式审查风险、建议申请书结构和待补充信息。请优先判断项目是否对口，并标出高风险条款。

### 10.2 生成标题

请根据以下项目主线和申报类型，生成十个中文标题，分别给出稳健型、创新型、评审友好型和应用交付型版本。每个标题说明优点、风险和适用条件，并推荐一个首选标题。

### 10.3 改写摘要

请按专家评审视角改写以下摘要。先给一句话主线和六句故事线，再诊断当前摘要的问题，最后给出稳健版、创新版和评审友好版。缺少事实请标“待补充”，不要编造。

### 10.4 诊断故事线

请检查以下申请书是否讲清楚“为什么重要、缺口在哪里、为什么是我们、怎么做、怎么验证、结果有什么价值”。请输出故事线诊断表、主线重构建议和需要同步修改的章节。

### 10.5 改立项依据

请将以下立项依据从“综述式写法”改为“评审式论证”：按背景、已有进展、关键缺口、前期发现、本项目必要性组织，并指出每段应补充的证据。

### 10.6 改技术路线

请诊断以下技术路线是否能支撑目标和指标。请补充输入、关键方法、输出、验证、风险和替代方案，并指出图表应如何重画。

### 10.7 科技部与上海指标覆盖

请把以下指南指标逐条映射到目标、任务、方法、交付物、验收方式、责任单位和预算。任何无法覆盖的指标请标为高风险。

### 10.8 国家自然科学基金精修

请按国家自然科学基金口径重构以下内容，突出科学问题、科学假设、研究内容递进、研究方案可行性、创新点和前期基础。请删除产品化、口号化和无关成果堆砌。

## 12. 每模块写作自检总表

| 模块 | 必须回答 | 常见硬伤 |
|---|---|---|
| 标题 | 研究什么、解决什么 | 大而空、方法堆砌 |
| 关键词 | 如何归类和匹配 | 全是大词或方法词 |
| 摘要 | 完整故事是什么 | 背景泛泛、无缺口 |
| 立项依据 | 为什么值得做 | 综述化、没有问题 |
| 现状 | 别人做到哪里 | 罗列文献、不比较 |
| 科学问题/瓶颈 | 核心卡点是什么 | 问题太大或不可验证 |
| 目标指标 | 完成后判断什么 | 口号化、不可验收 |
| 内容任务 | 做哪几件事 | 并列堆砌、不递进 |
| 方案路线 | 怎么做和怎么验证 | 方法名堆砌、缺对照 |
| 创新 | 新在哪里 | 没有比较对象 |
| 基础可行性 | 为什么能做 | 堆成果、无直接证据 |
| 团队组织 | 谁负责什么 | 挂名、接口不清 |
| 进度里程碑 | 何时完成什么 | 有时间无成果 |
| 成果转化 | 留下什么 | 只写论文专利 |
| 预算 | 钱为什么合理 | 与任务脱节 |
| 风险 | 出问题怎么办 | 口号化无替代 |
| 合规附件 | 能否过审 | 伦理、数据、签章遗漏 |

## 13. 可交付版本终检清单
