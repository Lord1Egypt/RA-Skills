# 提示词质量检查清单

## 功能完整性（40%）
- [ ] 能否明确解决用户痛点
- [ ] 覆盖场景是否完整
- [ ] 边界条件处理是否合理

## 输出稳定性（30%）
- [ ] 执行5次输出结果一致性≥80%
- [ ] 错误输入的容错性
- [ ] 不同模型的适配性

## 可用性（30%）
- [ ] 结构清晰易读
- [ ] 参数说明完整
- [ ] 示例具有代表性

## 评分标准
| 等级 | 分数 | 描述 |
|------|------|------|
| A级 | 90-100 | 优秀，可直接发布 |
| B级 | 70-89 | 良好，微调后发布 |
| C级 | 50-69 | 一般，需较大修改 |
| D级 | <50 | 不合格，不发布 |

## 自检流程
1. 阅读提示词，理解设计意图
2. 代入用户视角，评估实用性
3. 执行3-5次，统计稳定性
4. 检查格式规范
5. 给出最终评级
