# 数字人自动化营销飞书话术

必须保持话术少而准，只在需要用户动作或阶段完成时发送。

产品规则：
- 用户回复 `通过` 时，根据当前状态判断：有人设草案则确认人设；有待确认视频方案则确认方案；有待确认视频时只提示必须回复 `确认发布`，不能把 `通过` 当作发布确认。
- 用户回复 `不通过` 时，根据当前状态进入对应修正：人设修正、方案重做或视频重做。
- 人设确认后必须自动进入形象定制。
- 初次启动时，形象生成后必须立即进入视频生成。
- 用户确认视频方案后必须由后台 job 连续生成视频，先发“老板，正在为您制作视频，请耐心等待～”，生成完成后只发一条标准视频审核话术；不要让 agent 手写长文或补调用低层成片工具。
- 工具已经发出飞书通知后，OpenClaw/其他 agent 不得再额外发送同义消息，避免重复消息。
