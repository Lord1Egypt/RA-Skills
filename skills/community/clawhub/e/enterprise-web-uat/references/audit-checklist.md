# 最终 UAT 审核清单

报告定稿前必须执行本清单。

## 1. 隐私与发布安全

- 不包含凭证、Cookie、Token、密码、浏览器存储状态、私钥、应用密钥或 Authorization 头。
- 不包含客户涉密名称，除非当前用户明确要求写入私有报告。
- 不包含个人本机路径、用户名、桌面路径、机器专属浏览器配置或 workspace 路径。
- 截图如包含隐私数据，必须脱敏。

## 2. 范围与结构

- 已说明范围、假设、排除项、环境、版本、角色和数据规则。
- 必测模块/流程已覆盖，或明确标记为 `OutOfScope` / `Blocked` / `ToConfirm`。
- 报告能支撑明确验收结论。

## 3. RTM 与用例

- 每条验收准则均映射到用例、人工确认、补充验证或明确缺口。
- 用例状态与执行证据一致。
- 阻塞用例没有计入通过。

## 4. 证据质量

- 证据属于对应页面、角色、流程和时间点。
- 页面 URL 与接口 URL 已区分。
- 截图、DOM/控件清单、日志、网络观察均有证据编号。
- 接口/后台成功没有被夸大为 GUI 成功。

## 5. 问题质量

- 每个正式问题都有严重级别、优先级、验收影响、预期结果、实际结果和复现步骤。
- 控件缺失只有在来源资料或用例明确要求时才算正式缺陷，否则标记为 `ToConfirm`。
- 复现步骤是业务可读步骤，不只是技术 API 说明。

## 6. 交付包

- 如要求打包，交付包包含报告、RTM、用例、执行记录、问题、证据索引、审核记录和清单。
- 交付包排除密钥、浏览器存储状态、原始凭证和无关缓存文件。
- 如审核发现问题，修正版本标记为 `review-fixed`。
