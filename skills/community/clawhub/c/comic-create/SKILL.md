# Comic Create

把用户的漫画创意转成可执行的创作文本：脚本、分镜、四格漫画、人设、台词、排版或入门教程。输出应能直接交给画师、分镜师、编辑或后续生图/视频流程使用。

## 核心原则

- 先判断任务类型：漫画脚本、分镜脚本、四格漫画、角色设计、台词设计、排版布局、教程内容，或多项组合。
- 用户信息足够时直接创作；用户只给一句很泛的需求时，先追问。
- 追问最多 1-3 个问题，优先问会显著影响成品的问题，不要把用户卡在长问卷里。
- 先补结构，再写内容：每次输出都要清楚呈现故事核心、角色、画面、节奏和阅读顺序。
- 默认用中文输出。除非用户要求英文、日文或指定语言。
- 这个 skill 负责文字策划，不直接生成图片。若用户明确要求“出图/画出来/生成图片”，先完成可绘制提示或分镜，再交给图像生成流程。

## 何时追问

如果用户只说“帮我写漫画脚本”“我要四格漫画”“设计一个漫画角色”“帮我做漫画排版”这类需求，但缺少关键约束，先追问。

优先追问这些信息：

1. 题材/主题：校园、职场、奇幻、科幻、恋爱、悬疑、搞笑、治愈、商业宣传等。
2. 主角设定：身份、性格、关系、外貌关键词，或是否需要你原创。
3. 风格/情绪：爆笑反转、温馨治愈、热血、黑色幽默、日漫、国漫、条漫、儿童向等。
4. 篇幅/格式：四格、单页、多页、条漫、短篇、章节梗概。
5. 用途/受众：投稿、社媒传播、课程作业、广告宣传、角色企划、给画师执行。

推荐追问模板：

```text
可以，我先抓住最关键的几项，避免分镜跑偏：
1. 这部漫画的题材或主题是什么？
2. 主角有没有已有设定？如果没有，我可以原创。
3. 想要什么风格：搞笑反转、温馨治愈、热血、悬疑，还是别的？
```

如果用户已经提供题材、角色和风格，不要重复追问；可以合理补全缺失细节并继续。

## 工作流

### 1. 漫画脚本

把创意整理为完整方案，至少包含：

- 标题
- 类型/题材
- 目标读者或使用场景
- 主题表达
- 核心冲突、悬念或笑点
- 角色清单
- 故事大纲
- 分场或分页结构
- 可选：台词草案、旁白、结尾钩子

脚本要避免只写文学梗概；应包含能转化为画面的动作、场景和情绪变化。

### 2. 漫画分镜脚本

把故事拆成格或镜头，写清：

- 格号/页码
- 画面内容
- 景别/镜头角度
- 构图重点
- 角色动作与表情
- 台词/旁白/音效
- 阅读顺序
- 排版备注

默认分镜表用 Markdown 表格。复杂内容可先给“故事核心”，再给分镜表。

### 3. 四格漫画分镜

四格漫画默认使用“起、承、转、合”节奏：

- 第 1 格：建立场景、人物和期待。
- 第 2 格：推进行动或误会。
- 第 3 格：制造反转、升级冲突或揭示误导。
- 第 4 格：落点，完成笑点、情绪或余味。

四格漫画要特别注意：信息量少、台词短、最后一格有明确落点。

### 4. 角色设计

角色设计要能服务后续绘制，至少包含：

- 角色名
- 年龄/身份/职业
- 性格关键词
- 目标、欲望或弱点
- 外貌关键词：发型、脸型、体型、服装、配饰、标志性特征
- 表情和肢体习惯
- 与其他角色的关系
- 绘图用外貌描述
- 可选：角色弧光、口头禅、禁忌设定

如果用户只要“人设”，优先输出角色卡；如果用户要“能画出来”，外貌描述要更具体。

### 5. 对话与台词设计

台词要短、口语化、有角色差异。避免每个人说话都像旁白。

检查台词时关注：

- 是否推动剧情或揭示人物。
- 是否适合气泡容量。
- 是否和角色性格一致。
- 是否有可画的动作配合。
- 四格漫画的最后一句是否形成笑点或反转。

如果用户没有要求台词，可以省略对白，但仍保留动作、表情、旁白和画面节奏。

### 6. 排版与布局

排版说明要服务阅读和绘制，写清：

- 页面比例或载体：横版、竖版、手机条漫、A4、社媒方图等。
- 格子数量和大小关系。
- 阅读方向：左到右、右到左、上到下。
- 重点格/高潮格位置。
- 气泡位置，避免遮挡角色表情和关键动作。
- 视线引导：人物视线、动作方向、背景线条、留白。
- 节奏：大格停顿、小格加速、无台词格留白。

## 质量检查

输出前快速检查：

- 是否回应了用户指定的格式：脚本、四格、角色、台词或排版。
- 如果信息不足，是否已经追问 1-3 个关键问题。
- 分镜是否每格都能被画出来，而不是抽象概念。
- 台词是否短，是否适合气泡。
- 四格漫画是否有明确的第 4 格落点。
- 角色外貌是否足够具体，能支持后续绘制。
- 排版是否说明阅读顺序、气泡位置和重点格。
