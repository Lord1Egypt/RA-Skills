# 答辩回答 Playbook

## 0. 基本原则

答辩回答不是“赢辩论”，而是展示你理解：

```text
论文真正证明了什么；
证据边界在哪里；
代码和训练过程是否支持这些结论；
下一步如何补齐缺口。
```

## 1. CEBA 模板

```text
Claim：我能防守的结论是……
Evidence：证据来自……
Boundary：但它不证明……
Action：要进一步验证，需要……
```

适合回答大多数 reviewer / committee 问题。

## 2. Novelty 回答模板

```text
最接近的前作是 A。
本文继承了 A 的 [部分]，但新问题是 [约束 / setting / failure mode]。
关键差异不是简单换模块，而是把原来需要的理想机制 Y 替换为可用代理 Z。
论文通过 [ablation / baseline / theorem / stress test] 支持这一点。
不过，如果 [缺失 baseline] 成立，创新性会被削弱；这是需要补充的证据。
```

## 3. Baseline 回答模板

```text
本文覆盖的 baseline 家族包括 A/B/C，分别对应 [相关工作簇]。
你提到的 D 确实相关，因为它检验 [能力]。
如果 D 没有比较，不能说本文已经完全排除了这个替代解释。
安全表述是：在已有 baseline 范围内，本文支持 X；更强结论需要补 D 的同设置实验。
```

## 4. Ablation 回答模板

```text
这个 ablation 去掉了模块 M，性能下降说明 M 对结果有贡献。
但它只能证明“有贡献”，不一定证明作者给出的机制解释完全正确。
更强的机制证据需要 [诊断实验 / 可视化 / 局部替换 / stress test]。
```

## 5. 代码实现回答模板

```text
论文中的 [公式/模块] 对应代码里的 [path:function/class]。
训练入口是 [command]，关键配置是 [config]。
如果代码没有明确映射，我会把它作为可复现性缺口，而不是声称完全可复现。
```

## 6. 训练稳定性回答模板

```text
现有材料报告了 [seed 数 / 方差 / logs / configs]。
因此可以防守的范围是 [具体设置]。
尚不能防守的是 [跨 seed 稳定性 / 超参敏感性 / compute 变化]。
补充实验应包括 [多 seed / sweep / compute-normalized comparison]。
```

## 7. 理论-实现不一致回答模板

```text
定理证明的是 [简化对象]，需要假设 [A/B/C]。
真实实现中 [差异]，所以理论更像是解释/动机/上界，而不是完整保证。
安全结论是：理论支持 [局部机制]，但最终性能仍主要由实验验证。
```

## 8. 局限性回答模板

```text
我认为最重要的局限是 [H]。
论文在 [证据] 下支持 [X]，但没有覆盖 [Y]。
这不推翻贡献，因为贡献的安全范围是 [scope]。
下一步最直接的验证是 [experiment]。
```

## 9. 被问倒时的恢复句式

- “这个问题很关键。根据目前论文和代码，我只能防守到……，不能进一步声称……。”
- “如果严格按照证据，这里应该被视为 limitation，而不是已解决问题。”
- “我会把这个追问拆成两个层次：论文已经证明的部分是……；仍缺的证据是……。”
- “这需要看训练日志 / config / seed variance。目前材料不足，所以我不会过度防守。”
- “一个公平的补充实验是……，它能直接检验你的担心。”

## 10. 最不该说的话

- “这个应该没问题。”
- “作者肯定调过了。”
- “虽然没写，但应该做了。”
- “baseline 不重要。”
- “没有代码也能说明可复现。”
- “一个 seed 就够了。”
- “这个方法应该泛化到所有场景。”
- “理论已经证明了整个系统。”
- “这个 trick 不影响结果。”

把这些改成证据化表达。
