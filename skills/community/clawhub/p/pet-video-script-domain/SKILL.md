# 萌宠短视频创作知识参考库

## 定位

本技能是 **Universal Task OS 的领域负载物仓库**，不包含任何执行框架。只提供萌宠短视频创作的"是什么"和"长什么样"——执行全部委托UTOS。

| 本技能提供 | UTOS消费方式 |
|-----------|-------------|
| 萌宠短视频任务清单+依赖拓扑 | 内容轴·清单法的任务目录 + 执行轴·管线编排的依赖输入 |
| 分镜脚本要求 | 内容轴·清单法的组件清单 |
| 优秀范本（待填充） | 内容轴·样本法的样本 |

## 三层结构

```
第一层：任务清单与依赖拓扑   →  references/pet-video-catalog.md
第二层：内容要求清单          →  references/pet-video-requirements.md
第三层：优秀范本库            →  references/exemplars.md
```

## 依赖声明

本技能**强依赖** Universal Task OS (universal-task-os)。没有UTOS，本技能只有参考查阅能力，无法执行任何短视频创作任务。

**加载检查流程**（每次激活时执行）：

1. 检测 `universal-task-os` 技能是否已安装
2. **未安装** → 自动安装 `universal-task-os` 技能
3. **安装成功** → 同时加载UTOS，按本技能"使用规则"执行
4. **安装失败** → 降级为**只读参考模式**：
   - ✅ 允许：查阅任务清单、内容要求、范本索引
   - ❌ 拒绝：任何涉及分镜脚本产出、管线编排、提示词生成的任务，并提示"需先安装 Universal Task OS"

**任务模式判定**：

| 任务类型 | 无UTOS | 有UTOS |
|---------|--------|--------|
| 查阅任务清单/要求/范本 | ✅ 只读参考 | ✅ 完整 |
| 按清单法产出分镜脚本 | ❌ 拒绝 | ✅ UTOS编排执行 |
| 依赖拓扑推导创作管线 | ❌ 拒绝 | ✅ UTOS执行轴 |
| 角色一致性校验点插入 | ❌ 拒绝 | ✅ UTOS守护单元 |

## 使用规则

1. **依赖检查**：激活时按上述流程检测并安装UTOS
2. **首次加载**：读取 `references/pet-video-catalog.md`，获取域分类、依赖拓扑、UTOS元操作映射提示
3. **按需深入**：确认目标任务类型后，读取 `references/pet-video-requirements.md` 获取组件清单
4. **委托UTOS**：将任务清单作为清单法输入、范本作为样本法输入、依赖拓扑作为管线编排输入，交给UTOS执行轴+内容轴处理
5. **角色一致性**：萌宠视频的核心约束是角色外观在所有镜头中必须一致，这是贯穿全流程的守护规则

## 萌宠短视频特有维度

| 维度 | 说明 | 影响 |
|------|------|------|
| 角色一致性 | 宠物外观在所有分镜中必须保持一致 | Subject description在所有Seedance prompt中重复 |
| 平台适配 | 不同平台的尺寸/时长/风格要求不同 | 9:16(TikTok/Reels) vs 16:9(YouTube) vs 1:1(小红书) |
| AI视频工具适配 | 不同工具的prompt语法和能力不同 | Seedance 2.0 / Kling / Veo / Pika各有偏好 |
| 情感节奏 | 萌宠视频的核心价值是情感共鸣 | Hook→Build-up→Climax→Resolution的节奏设计 |
| 音效设计 | 萌宠视频高度依赖音效增强感染力 | BGM选择、音效提示、字幕配音 |

### 依赖拓扑摘要

**主链路**: P1-01 → P1-02 → P1-03 → P1-04 → P2-01 → P2-02 → P2-03 → P2-04 → P3-01 → P3-02/P3-03(并行) → P3-04 → P4-01 → P4-02 → P4-03

**角色链**: P1-01 → P1-02 → P1-03 → P1-04 → P4-01（角色一致性锁定）

**故事链**: P2-01 → P2-02 → P2-03 → P2-04（叙事结构设计）

**分镜链**: P3-01 → P3-02 ‖ P3-03 → P3-04（镜头语言编排）

**提示词链**: P4-01 → P4-02 → P4-03（AI视频生成输入）

更多组合由UTOS根据具体任务动态推导。

## 内容要求清单

### P1-01 品种识别
- **必选组件**: 宠物品种名称（中文+英文）、物种类型（猫/犬/兔/鸟/其他）、品种特征关键词
- **可选组件**: 混血比例、年龄阶段（幼年/成年/老年）、体型分类（小型/中型/大型）
- **组装顺序**: 用户输入品种→品种标准化→英文品种名确认→品种特征提取
- **约束**: 品种名必须中英双语（Seedance prompt用英文）；混血品种需标注主要血统
- **格式**: 品种档案卡片（Markdown）

### P1-02 外观描述
- **必选组件**: 毛色（主色+辅色+花纹）、体型（大小+胖瘦）、五官特征（眼睛+鼻子+嘴巴+耳朵）
- **可选组件**: 标志性配饰（项圈/衣服/帽子）、特殊标记（围脖/白手套/异瞳）、参考照片
- **组装顺序**: 毛色描述→体型描述→五官特征→配饰/标记→英文subject description初稿
- **约束**: 描述必须足够具体以在AI视频中保持角色一致性；有照片时以照片为准
- **格式**: 外观描述块（中英双语）

### P1-03 性格标签
- **必选组件**: 主性格标签（1-2个，如戏精/呆萌/高冷/粘人/调皮/傲娇）、性格表现描述
- **可选组件**: 次性格标签、情绪触发场景（什么情况下表现出什么情绪）
- **组装顺序**: 用户描述→性格关键词提取→标签化→表现描述→场景化
- **约束**: 性格标签必须影响后续故事设计——戏精型适合搞笑反差，呆萌型适合温馨日常
- **格式**: 性格档案（Markdown）

### P1-04 行为特征
- **必选组件**: 标志性动作（1-3个，如歪头/打滚/追尾巴/偷吃）、动作触发条件
- **可选组件**: 习惯性行为（每天固定时间做什么）、禁忌行为（不适合拍视频的行为）
- **组装顺序**: 用户描述→行为提取→动作化描述→触发条件标注→可拍性评估
- **约束**: 行为必须是可视化可拍摄的；纯内在状态（如"心情不好"）不可作为行为特征
- **格式**: 行为特征列表（Markdown）

### P2-01 故事前提
- **必选组件**: 主角（宠物名+品种）、场景（在哪）、事件（做什么）、结果（怎么样）
- **可选组件**: 配角（主人/其他宠物/路人）、道具、时间设定
- **组装顺序**: 角色确认→场景设定→事件构思→结果设计→一句话概括
- **约束**: 一句话概括必须≤30字；必须包含冲突或反差（纯记录型不算故事）
- **格式**: 一句话故事前提

### P2-02 情绪风格
- **必选组件**: 风格标签（搞笑反差/温馨治愈/悬疑反转/日常记录/沙雕魔性）、情绪曲线描述
- **可选组件**: 参考视频链接、目标情绪反应（让观众笑/感动/惊讶/治愈）
- **组装顺序**: 故事前提分析→风格匹配→情绪曲线设计→目标确认
- **约束**: 风格必须与宠物性格匹配——高冷猫+搞笑反差=经典组合；粘人狗+温馨治愈=安全组合
- **格式**: 情绪风格卡片（Markdown）

### P2-03 冲突设计
- **必选组件**: 核心矛盾（什么冲突）、升级节奏（矛盾如何加剧）、转折点（矛盾如何变化）
- **可选组件**: 次要矛盾、误导项（让观众以为是A结果是B）
- **组装顺序**: 故事前提提取→矛盾设计→节奏规划→转折点设计→冲突链确认
- **约束**: 30秒视频≤1个冲突；60秒视频≤2个冲突；冲突必须可视化（纯心理冲突不可拍）
- **格式**: 冲突设计表（Markdown）

### P2-04 结局设计
- **必选组件**: 结局类型（反转/温馨/悬念/爆笑/治愈）、最后画面描述、情感落点
- **可选组件**: 彩蛋设计、续集钩子、互动引导（"你家毛孩子也这样吗？"）
- **组装顺序**: 冲突走向分析→结局类型选择→最后画面设计→情感落点确认
- **约束**: 结局必须与情绪风格一致——搞笑风格→爆笑/反转结局；温馨风格→治愈/温馨结局
- **格式**: 结局设计方案（Markdown）

### P3-01 镜头规划
- **必选组件**: 镜头总数、每镜头核心事件（一句话）、镜头顺序
- **可选组件**: 镜头间转场方式、节奏标注（快/中/慢）
- **组装顺序**: 故事拆分→镜头数量确定（按时长：15s→3-4镜/30s→5-7镜/60s→7-10镜）→事件分配→顺序排列
- **约束**: 镜头总数按时长约束；每镜头必须有独立核心事件；事件必须推动故事
- **格式**: 镜头规划表（Markdown表格）

### P3-02 景别设计
- **必选组件**: 每镜头的景别（特写/近景/中景/全景/远景）、景别选择理由
- **可选组件**: 构图要点（三分法/对称/引导线）、画面重心
- **组装顺序**: 镜头事件分析→景别匹配→理由记录→全局景别节奏检查
- **约束**: 景别必须有节奏变化——连续3个同景别会视觉疲劳；Hook镜头建议近景/特写
- **格式**: 景别设计表（Markdown表格）

### P3-03 运镜设计
- **必选组件**: 每镜头的运镜方式（固定/推/拉/摇/跟/升降/环绕）、运镜目的
- **可选组件**: 运镜速度（慢/中/快）、运镜起止点
- **组装顺序**: 镜头事件+景别分析→运镜匹配→目的记录→全局运镜节奏检查
- **约束**: 每镜头≤1种运镜方式（Seedance限制）；运镜必须服务于叙事——推=聚焦/拉=揭示/跟=追踪
- **格式**: 运镜设计表（Markdown表格）

### P3-04 时间轴编排
- **必选组件**: 每镜头时长（秒）、累计时间戳、总时长
- **可选组件**: 节奏标注（快/中/慢）、关键帧时间点
- **组装顺序**: 镜头重要性评估→时长分配→累计计算→总时长校验→节奏调整
- **约束**: 总时长必须匹配目标平台（15s/30s/60s）；Hook镜头≤3秒；高潮镜头给予最多时长
- **格式**: 时间轴表格（Markdown表格）

### P4-01 主体锁定
- **必选组件**: 英文subject description（从P1-02/P1-03生成，所有镜头共用）、角色一致性声明
- **可选组件**: 角色变体描述（不同装扮/场景下的外观微调）
- **组装顺序**: P1外观+性格信息整合→英文描述生成→一致性声明添加→描述锁定
- **约束**: subject description必须在所有镜头prompt中完全一致（或仅允许最小变体）；必须包含"maintain exact appearance across shots"
- **格式**: 英文主体描述块

### P4-02 镜头提示词
- **必选组件**: Seedance格式英文prompt（40-80词）、包含：shot type + subject + action + environment + camera + lighting + style + aspect ratio
- **可选组件**: 负面约束（no human hands visible, no text overlay等）
- **组装顺序**: 景别+运镜+事件+场景信息→prompt公式组装→词数校验→风格一致性检查
- **约束**: 每个prompt必须以shot type开头；主体描述必须与P4-01锁定描述一致；单个prompt≤80词
- **格式**: 英文prompt代码块

### P4-03 音效标注
- **必选组件**: BGM风格建议（如upbeat/lo-fi/dramatic/cute）、每镜头音效提示
- **可选组件**: 具体音乐参考、音效触发时间点、配音建议
- **组装顺序**: 情绪风格→BGM匹配→逐镜头音效标注→音效节奏检查
- **约束**: BGM风格必须与情绪风格一致；音效必须与画面动作同步；Seedance 2.0支持audio generation cues
- **格式**: 音效标注表（Markdown表格）

## Seedance 2.0 Prompt参考

### Prompt公式

```
[Shot type], [subject with consistent visual details], [primary action], [secondary motion], [environment/location], [camera movement + speed], [lighting condition], [visual style + mood], [aspect ratio]
```

**规则：**
- 每个prompt一个主要动作
- 每个prompt一个运镜方式
- 所有镜头保持主体描述一致
- 以shot type开头（Seedance权重靠前的token）
- 以目标平台的aspect ratio结尾

### 景别词汇

| 景别 | 英文 | 最佳用途 |
|------|------|----------|
| 特写 | extreme close-up / close-up | 爪子细节、眼神表情、胡须、鼻子碰碰 |
| 近景 | medium close-up | 脸部反应、吃东西、玩玩具 |
| 中景 | medium shot | 全身动作、宠物+小道具 |
| 全景 | wide shot / full shot | 跑跳、整个房间场景 |
| 远景 | wide establishing shot | 环境揭示、户外风景 |
| 俯拍 | top-down / overhead | 睡姿、食碗、地板玩耍 |
| 仰拍 | low angle / worm's eye | 戏剧性英雄时刻、宠物看起来强大 |
| 跟拍 | tracking shot | 跟随宠物移动 |
| 过肩 | over-the-shoulder | 宠物视角看某物 |
| 偷窥 | peeking / through doorway | 被抓现行时刻、偷偷摸摸行为 |

### 运镜词汇

| 运镜 | 英文 | 情绪/速度 |
|------|------|-----------|
| 固定 | static / locked off | 平静、观察、喜剧节奏 |
| 缓慢推近 | slow dolly in | 建立张力、情感聚焦 |
| 缓慢拉远 | slow dolly out | 揭示惊喜、展示全貌 |
| 轻微手持 | subtle handheld | 自然、TikTok原生感 |
| 跟踪 | tracking at walking pace | 跟随穿越环境 |
| 环绕 | slow orbit | 从各角度展示宠物 |
| 快速推近 | fast zoom in | 惊喜冲击 |
| 快速平移 | quick pan | 捕捉快速动作 |
| 低角度奔跑 | low angle running toward camera | 戏剧性英雄时刻 |

### 灯光关键词

| 场景 | 英文 | 效果 |
|------|------|------|
| 自然光 | natural daylight | 明亮、清新、适合日常 |
| 暖光 | warm golden hour | 温馨、治愈、浪漫 |
| 柔光 | soft diffused light | 柔和、梦幻、可爱 |
| 侧光 | dramatic side light | 戏剧性、神秘 |
| 背光 | backlit / rim light | 轮廓光、神圣感 |
| 环境光 | ambient indoor light | 居家感、舒适 |

### 风格关键词

| 风格 | 英文 | 适合场景 |
|------|------|----------|
| 电影感 | cinematic | 高质感、故事感强 |
| 日常记录 | casual vlog style | 自然、亲近、真实 |
| 治愈系 | soft and dreamy | 温馨、舒适、放松 |
| 搞笑 | comedic, playful | 活泼、有趣、夸张 |
| 纪录片 | documentary style | 专业、严肃、科普 |

### 负面约束

```
no human hands visible, no text overlay, no watermark, no blur, no distortion, maintain exact appearance across shots, no extra limbs, no deformed face
```

## 故事框架模板

### 搞笑/反差模板

**F1: 偷吃被抓**
- Hook (0-3s): 宠物装无辜，主人转身
- Setup (3-10s): 主人放食物在桌上，离开房间
- Build-up (10-20s): 宠物偷偷靠近→偷一口→得逞→再来一次
- Climax (20-25s): 主人意外回来，抓现行
- Punchline (25-30s): 宠物定格，嘴里还叼着食物，一脸心虚

**F2: 期望vs现实**
- Hook (0-3s): 慢动作帅气入场（"期望"）
- Setup (3-12s): 宠物尝试做某件帅气的事（跳跃、接住、装酷）
- Build-up (12-22s): 尝试以可爱的方式出错
- Punchline (22-30s): 现实揭晓——乱七八糟、笨拙，但超可爱

**F3: 复仇计划**
- Hook (0-3s): 主人恶作剧/忽略宠物
- Setup (3-12s): 宠物给一个"你会后悔的"眼神，开始策划
- Build-up (12-22s): 宠物执行复仇计划（藏玩具、坐键盘、打翻东西）
- Punchline (22-30s): 主人崩溃 vs 宠物得意

### 温馨模板

**H1: 日常陪伴**
- Hook (0-3s): 宠物等待主人回家
- Build-up (3-20s): 宠物的等待日常（看窗外、趴门口、听到声音竖起耳朵）
- Climax (20-27s): 主人开门，宠物冲上去
- Ending (27-30s): 依偎在一起

**H2: 成长记录**
- Hook (0-3s): 幼崽时期画面
- Build-up (3-20s): 成长关键时刻蒙太奇
- Climax (20-27s): 成年后同样的动作
- Ending (27-30s): 对比画面，温暖收尾

### Hook公式

| 类型 | 公式 | 示例 |
|------|------|------|
| 悬念钩子 | "你绝对想不到这只[宠物]做了什么" | 镜头对着空房间，传来奇怪声音 |
| 反差钩子 | "看起来[状态]，其实..." | 看起来在睡觉，其实在偷听 |
| 共鸣钩子 | "养[宠物]的都懂" | 宠物盯着食物的渴望眼神 |
| 挑战钩子 | "试试看忍住不笑" | 宠物做出超级搞笑的动作 |

## 景别与运镜选择指南

### 景别选择（按目标情绪）

| 目标 | 推荐景别 | 原因 |
|------|---------|------|
| 让观众笑 | 中景+微低角度 | 捕捉全身喜剧+宠物视角 |
| 制造"好可爱"反应 | 脸部/眼睛特写 | 最大化可爱因子，展示表情细节 |
| 建立张力/悬念 | 从门口/框架偷窥 | 有限视野制造好奇 |
| 展示环境 | 全景建立镜头 | 设定场景，展示宠物与空间的关系 |
| 情感连接 | 平视近景 | 平等化宠物和观众，创造亲密感 |
| 喜剧节奏 | 固定镜头，宠物进出画面 | 经典肢体喜剧技巧 |
| 展示小小体型 | 低角度+超大道具 | 强调小宠物的可爱 |
| 展示力量/骄傲 | 仰视宠物 | 让宠物看起来英勇（对小宠物有反差萌） |
| 吃播内容 | 俯拍 | 清晰展示食物+宠物吃相 |
| 困倦/舒适 | 浅景深特写 | 梦幻、亲密、平静氛围 |

### 运镜选择（按场景能量）

**低能量（平静、困倦、温柔）**
- 固定/锁定 — 让宠物的细微动作成为唯一运动
- 缓慢推近 — 逐渐增加亲密感
- 缓慢平移 — 轻柔揭示场景
- 细微呼吸感 — 像平静的人手持的微小浮动

**中能量（日常、玩耍、探索）**
- 轻柔手持 — 自然、TikTok原生感
- 步行速度跟踪 — 跟随穿越环境
- 缓慢环绕 — 从各角度展示宠物
- 上下倾斜 — 和宠物一起发现什么

**高能量（奔跑、追逐、喜剧）**
- 快速跟踪跟随 — 匹配宠物速度
- 快速平移 — 捕捉快速动作
- 快速推近 — 惊喜冲击
- 充满能量的手持 — 弹跳运动匹配兴奋感
- 低角度向镜头奔跑 — 戏剧性英雄时刻

## 与UTOS的接口

当UTOS处理萌宠短视频领域任务时：

- **Step 0 三轴判定**：萌宠短视频任务通常为中等复杂度+创意型 → 执行轴+内容轴+创新轴
- **Step 1 领域校准**：R1(信息密度)=中（视觉描述为主）+R2(创造性)=极高（故事+镜头语言）+R3(交互性)=中（用户确认角色和方向）+R4(规范性)=中（平台规则约束）+R5(迭代性)=高（同一角色多故事）→ A极高G中循环多
- **Step 2 内容轴**：清单法用本技能的pet-video-requirements；样本法用本技能的exemplars
- **Step 3 执行轴**：管线编排基于本技能的依赖拓扑自动推导元操作序列（角色建档→故事设计→分镜编排→提示词生成）
- **Step 4 交付**：G类守护单元自动插入角色一致性校验点
