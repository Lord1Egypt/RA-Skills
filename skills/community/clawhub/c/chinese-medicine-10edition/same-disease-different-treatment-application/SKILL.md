# Same Disease Different Treatment Application

## When to Use
- 已完成中医辨证，明确疾病的“证”型和病机
- 需要制定个体化治疗方案，避免机械套用病名
- 面对多种慢性病（如水肿、脱肛、子宫下垂、胃下垂等）需判断是否共享相同病机

## Execution Steps

### 1. 判断是否为“同病异治”情形
- **条件**：同一病名（如“水肿病”）但证型不同
- **分析维度**：
  - 脏腑归属（肺、脾、肾等）
  - 虚实性质（虚证或实证）
  - 病因类型（风热、风寒、水湿、邪毒等）
- **执行**：针对不同证型选择相应治法，例如：
  - 宣肺利水（肺失宣降）
  - 健脾渗湿（脾虚湿盛）
  - 温肾化气（肾阳不足）

### 2. 判断是否为“异病同治”情形
- **条件**：不同病名（如脱肛、子宫下垂、胃下垂）但证型相同
- **关键识别**：共同病机为“中气下陷”，表现为脾气虚弱、升举无力
- **执行**：统一采用“升提中气”治法，常用方剂如补中益气汤加减

### 3. 核心逻辑验证
- 验证原则：“相同的病机，其基本治法就相同；不同的病机，其治法也就不相同”
- 若无法明确病机一致性，不得强行归类

### 4. 注意事项
- **严禁**在未辨证情况下仅凭病名或表面症状决定治法
- 慢性病常兼夹复杂病机，需动态评估证型变化
- 治疗策略应包含具体治法名称及代表方药方向

## Output
输出明确的治疗策略，格式为：
- 治则（如同病异治 / 异病同治）
- 对应证型与病机
- 推荐治法（如“健脾渗湿”）
- 代表方剂或药物方向（如“参苓白术散加减”）
