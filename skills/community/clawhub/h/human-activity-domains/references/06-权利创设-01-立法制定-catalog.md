## 立法制定任务清单

立法制定按子域组织的任务清单，附任务间关系和元操作映射提示。

**域间逻辑流**：立法调研 → 立法起草 → 立法审议 → 立法实施

---

### 1 立法调研

| ID | 任务类型 | 说明 | 依赖 | 元操作映射 |
|----|----------|------|------|----------|
| T1-1 | 需求调研 | 调研立法需求，了解社会问题和立法诉求 | 无（入口） | S→C |
| T1-2 | 问题分析 | 分析立法要解决的问题，明确问题性质和范围 | T1-1 | C |
| T1-3 | 比较研究 | 比较国内外相关立法，借鉴有益经验 | T1-2 | S→C |
| T1-4 | 可行性分析 | 分析立法可行性，评估立法条件和时机 | T1-3 | C→G |

---

### 2 立法起草

| ID | 任务类型 | 说明 | 依赖 | 元操作映射 |
|----|----------|------|------|----------|
| T2-1 | 条文起草 | 起草法律条文，明确法律规范和法律后果 | T1-4 | C→A |
| T2-2 | 专家论证 | 组织专家论证，评估条文合理性和可行性 | T2-1 | C→I |
| T2-3 | 公众咨询 | 征求社会公众意见，了解公众诉求 | T2-2 | S→I |
| T2-4 | 草案修改 | 根据专家和公众意见修改草案 | T2-3 | C→A |

---

|----|----------|------|------|----------|

|----|----------|------|------|----------|

## 依赖拓扑摘要

### 一、域内链路

**立法调研链路**: T1-1 → T1-2 → T1-3 → T1-4
**立法起草链路**: T2-1 → T2-2 → T2-3 → T2-4
**立法审议链路**: T3-1 → T3-2 → T3-3 → T3-4
**立法实施链路**: T4-1 → T4-2 → T4-3 → T4-4

### 二、跨域协同链路

**完整立法制定链路**: T1-1 → T1-2 → T1-3 → T1-4 → T2-1 → T2-2 → T2-3 → T2-4 → T3-1 → T3-2 → T3-3 → T3-4 → T4-1 → T4-2 → T4-3 → T4-4

**反馈改进链路**: T4-4 → T1-1（法规修订反馈到需求调研）

**监督驱动链路**: T4-3 → T2-1（效果评估驱动条文修改）

更多组合根据具体任务动态推导。
