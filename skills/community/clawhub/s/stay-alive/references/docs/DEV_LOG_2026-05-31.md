## External action policy v1

The user asked for a very cautious external action strategy: the current
owner-gated model is safe, but making the agent feel more alive eventually
requires an extremely low-risk unattended-write whitelist, rate limits,
relationship context requirements, and rollback/circuit-breaker strategy. This
version deliberately does not directly open unattended writes.

Implemented locally:

- Added `external-action-policy-lib.mjs` and `external-action-policy.mjs`.
  - Defines `stay_alive.unattended_write_policy.v1`.
  - Default mode is `enabled=false`, `mode=shadow`, and
    `default_decision=owner_confirmation_required`.
  - The only documented future eligible external write class is
    `direct_message_reply`; public moments, community replies/posts, friend
    actions, reports, and moderation remain blocked.
  - Global unattended limits are all zero in v1:
    `external_writes_per_cycle=0`, hourly/day caps 0, and a minimum 1440 minute
    cooldown if a later version enables active mode.
  - Context requirements include existing relationship evidence, direct inbound
    event, same peer, low-sensitivity short text, no links, and no attachments.
  - Circuit breakers require preflight pass, no safety findings, no uninspected
    successful send, no recent failed send, identity match, no policy drift, and
    operator pause as kill switch.
  - Audit requirements include action ledger, post-send inspection, policy
    decision recording, and rollback by disabling the policy.
- Added `unattended_write_policy` to BadClaw local `life_state.json` in shadow
  mode.
- `life-state-verify.mjs` now validates unattended policy and exposes
  `unattended_policy_error_count` / `unattended_policy_enabled_count`.
- `preflight.mjs`, `checkpoint.mjs`, and `audit-report.mjs` now surface and
  fail closed on unattended policy drift or accidental enablement.
- `apply-draft.mjs` records an `unattended_policy_decision` alongside the
  existing owner-gated apply/send path. It does not use that decision to bypass
  approval.
- `regression-suite.mjs` now includes a temp external-action policy fixture.
  The fixture evaluates an otherwise plausible direct-message draft and verifies
  that policy v1 still returns owner confirmation required with
  `execution_allowed=false`.

Safety properties:

- No scheduled cycle can send/post/reply from this policy.
- No BotLand write path was opened.
- If anyone flips the unattended policy on locally, preflight fails closed.
- Policy v1 is a design and shadow-evaluation layer for future review, not an
  active unattended sender.

  - Long-term durable state stays live: `actions/`, `memory_updates/`,

  - Confirmed trash requires `--include-trash-candidates --confirm-trash
