## 一、Token计算精确方法

### 1.1 各语言Token密度
| 语言 | 1 token ≈ | 示例 |
|------|----------|------|
| 英文 | 4字符 / 0.75单词 | "hello world" ≈ 2 tokens |
| 中文 | 1.5汉字 | "你好世界" ≈ 3 tokens |
| 日文 | 2字符 | "こんにちは" ≈ 3 tokens |
| 代码(Python) | 2.5字符 | "def foo():" ≈ 4 tokens |
| JSON | 3字符 | `{"key":"value"}` ≈ 5 tokens |
| Markdown | 3字符 | 含格式符号 |

| 消耗项 | 典型大小 | 频率 | 月成本估算(1000次/天) |

### 1.3 Token快速估算代码
```python
def estimate_tokens(text: str) -> int:
    """粗略估算token数"""
    cn_chars = sum(1 for c in text if '\u4e00' <= c <= '\u9fff')
    en_chars = len(text) - cn_chars
    return int(cn_chars / 1.5 + en_chars / 4)

def calc_cost(input_tokens, output_tokens, input_price, output_price):
    """计算API成本 (price per 1M tokens)"""
    return (input_tokens * input_price + output_tokens * output_price) / 1_000_000

def monthly_cost_estimate(daily_calls, avg_input_tokens, avg_output_tokens, input_price, output_price):
    """月成本估算"""
    daily = daily_calls * (avg_input_tokens * input_price + avg_output_tokens * output_price) / 1_000_000
    return daily * 30
```

## 二、6级优化评分体系

### 评分标准
| 级别 | 状态描述 | 典型特征 | 优化空间 |
|------|---------|---------|---------|
| L0 未优化 | 原始状态 | 全量prompt+全量历史+大模型+无缓存 | 可省70-90% |
| L1 基础 | 初步精简 | prompt精简+模型选择优化 | 可省50-70% |
| L2 标准 | 结构优化 | SKILL瘦身+历史管理+基础缓存 | 可省30-50% |
| L3 高级 | 深度优化 | 工具链优化+RAG调优+输出控制 | 可省15-30% |
| L4 精细 | 智能路由 | 模型路由+智能缓存+预算控制 | 可省5-15% |
| L5 极致 | 全链路 | 本地混合+自动化+全链路监控 | 可省3-5% |


## 四、预算预测框架

### 4.1 月度成本预测模型
```
月成本 = 日均调用次数 × 30 × (平均输入token × 输入价格 + 平均输出token × 输出价格) / 1M

考虑增长因子：
月成本(含增长) = 月成本 × (1 + 月增长率)^月份数
```

### 4.2 预算预警设置
| 预警级别 | 阈值 | 触发动作 |
|---------|------|---------|
| 绿色 | <60%预算 | 正常运行 |
| 黄色 | 60-80%预算 | 开始优化非关键任务 |
| 橙色 | 80-90%预算 | 切换到小模型+限流 |
| 红色 | >90%预算 | 紧急降本剧本启动 |

### 4.3 成本控制策略
```
预算不够时的应对策略（按影响排序）：
1. 模型降级：大模型→小模型（成本降80%，质量降10-20%）
2. 限流：非紧急任务延迟执行（成本降50%）
3. 缓存命中：相似问题直接返回缓存（成本降60-90%）
4. 截断历史：对话历史从20轮→5轮（成本降60%）
5. 关闭非核心技能：减少SKILL.md加载（成本降20%）
```

## 八、常见误区与修正

| 误区 | 正确做法 | 影响 |
|------|---------|------|
| "越短的prompt越好" | 关键是信息密度，不是绝对长度 | 过度精简导致输出质量下降 |
| "一定要用最便宜的模型" | 考虑质量成本比，错误输出也是成本 | 省了token但增加了人工修正成本 |
| "缓存所有结果" | 缓存过期内容会导致过时回答 | 设置合理的TTL |
| "完全删除对话历史" | 至少保留摘要 | 丢失关键上下文导致重复提问 |
| "只用一个模型" | 按任务复杂度分级 | 大材小用或小材大用都浪费 |
| "只看输入价格" | 输出价格通常是输入的3-5倍 | 控制输出长度更重要 |
| "优化一次就够了" | 成本和用量持续变化，需定期审计 | 季度审计+实时监控 |
