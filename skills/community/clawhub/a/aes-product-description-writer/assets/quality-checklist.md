# Product Description Writer — Quality Checklist

Use this checklist before delivering any product description output.

---

## Content Quality (9 items)

- [ ] Every bullet leads with a benefit, not a feature
- [ ] No bullet starts with the product name or brand name
- [ ] All claims are specific — no vague adjectives ("great," "amazing," "high-quality") without proof
- [ ] Numbers and measurements used wherever possible (dimensions, weights, capacities, percentages)
- [ ] No two bullets convey the same benefit in different words
- [ ] Short description is 25-50 words and stands alone as a compelling pitch
- [ ] Long description follows the selected framework structure (AIDA / PAS / FAB / BAB)
- [ ] Long description is 150-300 words
- [ ] All product claims are supportable — nothing the seller couldn't prove if challenged

## SEO Compliance (8 items)

- [ ] Primary keyword appears in the title
- [ ] Primary keyword appears in the first bullet point
- [ ] Primary keyword appears in the first paragraph of the long description
- [ ] Each secondary keyword appears at least once in the body copy
- [ ] Keyword density is 1-2% for primary keyword (not over-stuffed)
- [ ] Meta description is ≤155 characters and includes primary keyword
- [ ] No keyword cannibalization with other products in the same catalog
- [ ] Keyword Integration Map completed with all placements documented

## Brand Voice (6 items)

- [ ] Tone matches brand guidelines (if provided) or product positioning
- [ ] Consistent voice across all sections (title, bullets, descriptions)
- [ ] No competitor names mentioned anywhere in the copy
- [ ] No superlative claims without evidence ("best," "most," "#1")
- [ ] Language matches target audience sophistication level
- [ ] Brand name used correctly (spelling, capitalization, trademark symbols if required)

## Technical Accuracy (7 items)

- [ ] All product specifications verified against source data
- [ ] Materials, dimensions, and weights match product listing data
- [ ] Compatibility claims checked (fits Model X, works with System Y)
- [ ] Country of origin correctly stated if included
- [ ] Warranty and guarantee terms match actual policy
- [ ] Care instructions are accurate for the material/product type
- [ ] No claims that could create liability (medical, safety, regulatory)

## Platform Compliance (8 items)

- [ ] Title length within platform limit (Amazon ≤200, Shopify ≤70 recommended, Walmart ≤75, Etsy ≤140)
- [ ] Bullet count matches platform standard (Amazon: 5, Walmart: up to 10)
- [ ] Bullet character length within platform limit (Amazon ≤500, Walmart ≤80)
- [ ] No prohibited language for the target platform (check platform-specific rules)
- [ ] HTML formatting used only where platform supports it (Shopify/Walmart long desc: yes; Etsy: no)
- [ ] Backend search terms provided for Amazon listings (≤250 bytes)
- [ ] Product description length within platform limit
- [ ] Image alt-text suggestions provided if requested

## Formatting (5 items)

- [ ] Consistent bold pattern across all bullets (benefit phrase bolded)
- [ ] No orphan bullets (every bullet has both a benefit headline and supporting detail)
- [ ] Paragraphs in long description are ≤4 sentences each
- [ ] No walls of text — appropriate line breaks and structure
- [ ] Tables render correctly (Product Summary table, Keyword Integration Map)

## Batch-Specific Checks (7 items — Mode B only)

- [ ] Brand voice consistent across all products in the batch
- [ ] No repeated opening phrases across products
- [ ] No keyword cannibalization between products in the batch
- [ ] Each product's unique differentiator is clearly communicated
- [ ] Shared bullet structure pattern followed by all products
- [ ] Cross-Product Consistency Check table completed
- [ ] All products meet every platform constraint

---

**Total items: 50** (43 for Mode A, 50 for Mode B)
**Minimum passing: 40/43 (Mode A) or 45/50 (Mode B)**
