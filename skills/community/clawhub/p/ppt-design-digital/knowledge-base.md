# PPT 设计知识库

> 本知识库为 PPT 智能架构师提供专业的设计原则、配色方案、字体搭配、图表选择等参考信息。

---

## 一、叙事结构详解

### 1.1 SCQA 模型（巴巴拉·明托）

适用于：产品发布、提案、融资路演、咨询报告

| 要素 | 含义 | 示例（AI 报表平台） |
|:---|:---|:---|
| **S**ituation（背景） | 描述现状、已知共识 | "企业报表生成一直依赖财务团队手动处理" |
| **C**omplication（冲突） | 现状中的问题/痛点 | "一个报表平均需要 3 天，错误率高，季度末加班严重" |
| **Q**uestion（问题） | 由此产生的核心问题 | "如何将报表生成时间从 3 天缩短到 3 分钟？" |
| **A**nswer（方案） | 你的解决方案 | "我们的 AI 平台自动接入数据源，秒级生成可视化报表" |

### 1.2 金字塔原理（巴巴拉·明托）

适用于：工作汇报、研究报告、项目提案

```
                结论/中心论点
               /       |       \
          论点 A    论点 B    论点 C
         /  |  \    /  |  \    /  |  \
      论据A1 A2 A3  B1 B2 B3  C1 C2 C3
```

**关键规则**：
- 结论先行（First, the answer）
- 以上统下（上层是下层的总结）
- 归类分组（同一层级的论点属于同一类）
- 逻辑递进（时间/结构/程度顺序）

**MECE 原则**：每一层级的论据要 Mutually Exclusive（相互独立）、Collectively Exhaustive（完全穷尽）

### 1.3 黄金圈法则（Simon Sinek）

适用于：品牌宣传、愿景陈述、公司介绍

```
为什么 (Why) → 核心理念/使命
    ↓
如何做 (How) → 方法/策略/差异化
    ↓
做什么 (What) → 产品/服务/成果
```

> "People don't buy what you do; they buy why you do it."
> —— Simon Sinek

### 1.4 故事化叙事结构（三幕式）

适用于：品牌故事、客户案例、产品旅程

| 幕 | 内容 | 页数占比 |
|:---|:---|:---:|
| 第一幕：建立 | 背景、角色、现状 | ~20% |
| 第二幕：冲突 | 问题、挑战、谷底 | ~30% |
| 第三幕：解决 | 方案、成果、未来 | ~50% |

## 二、各类 PPT 的标准结构

### 2.1 产品发布会

```
1. 封面（标题 + 副标题 + 日期）
2. 痛点/机会（为什么做这个产品）
3. 产品介绍（核心功能 + 亮点）
4. 产品演示（截图/showcase）
5. 技术优势（壁垒/创新点）
6. 市场分析（TAM/SAM/SOM）
7. 商业模式（怎么赚钱）
8. 团队介绍（为什么是我们）
9. 路线图（现在 → 未来）
10. 行动号召（Demo/投资/合作）
```

**建议页数**：10-15 页 | **每页时间**：1.5-2 min

### 2.2 商业计划书（融资路演）

```
1. 封面（公司名 + 定位 + 一句话介绍）
2. 愿景 & 使命
3. 问题定义（痛点验证）
4. 解决方案（产品/服务）
5. 市场机会（TAM/SAM/SOM）
6. 商业模式（收入模型）
7. 获客策略（GTM）
8. 竞争分析（竞品矩阵）
9. 核心优势（护城河）
10. 数据验证（Traction）
11. 财务预测
12. 融资需求 & 用途
13. 团队介绍
14. 未来愿景 & 行动号召
```

**建议页数**：12-15 页 | **最经典的 Pitch Deck 格式：Airbnb / Uber 的早期 Deck**

### 2.3 工作汇报（周报/月报/季报）

```
1. 封面（汇报人 + 时间段）
2. 核心成果一览（KPI 总览）
3. 重点成果详述（数据 + 图表）
4. 项目进展（甘特图/状态）
5. 问题与风险（障碍 + 应对）
6. 不足与反思
7. 下一步计划（时间节点）
8. 需要的支持（资源/决策）
```

**建议页数**：8-12 页 | **建议：每页一个核心数据点**

### 2.4 培训课件

```
1. 封面（课程名称 + 讲师）
2. 课程目标（学完能做什么）
3. 目录/议程
4-每节内容（概念 → 案例 → 练习 → 小结）
  - 4.1 知识点概念
  - 4.2 实际案例
  - 4.3 互动练习
  - 4.4 本节要点
N. 总结回顾（知识地图）
N+1. 参考资料 & 延伸阅读
N+2. Q&A
```

**建议页数**：按每 5 分钟 3-4 页估算 | **关键：每 10 分钟插入互动环节**

### 2.5 学术报告（论文答辩）

```
1. 封面（论文题目 + 答辩人 + 导师）
2. 研究背景 & 意义
3. 文献综述
4. 研究问题 & 假设
5. 研究方法
6. 实验/数据
7. 结果分析
8. 讨论
9. 结论 & 贡献
10. 不足 & 未来工作
11. 致谢 & Q&A
```

**建议页数**：10-15 页 | **关键：方法部分要有流程图，结果部分要有图表**

### 2.6 竞品分析报告

```
1. 封面（分析范围 + 日期）
2. 分析框架说明（评估维度）
3. 竞品全景图（Competitive Landscape）
4. 功能对比矩阵（Feature Matrix）
5. 核心差异分析
6. SWOT 分析
7. 用户评价/反馈对比
8. 定价策略对比
9. 机会与威胁
10. 行动建议
```

**建议页数**：8-12 页 | **关键：用矩阵图而非文字描述做对比**

## 三、配色方案库

### 3.1 基础色彩理论

| 配色类型 | 说明 | 适用场景 |
|:---|:---|:---|
| **单色搭配** | 同一色相不同明度/饱和度 | 专业、极简、正式 |
| **类比色** | 色环上相邻的颜色 | 和谐、温和、品牌感 |
| **互补色** | 色环上相对的颜色 | 高对比、抓眼球、行动号召 |
| **分裂互补** | 一个颜色 + 其互补色的两侧 | 高对比但不刺眼 |
| **三色搭配** | 色环上 120° 三色 | 丰富、活力、创意 |
| **四边形** | 色环上矩形四点 | 复杂、多彩、需要谨慎使用 |

### 3.2 经典配色方案

#### 科技/互联网（专业、创新、信任感）

| 场景 | 主色 | 辅色 | 强调色 | 背景 |
|:---|:---|:---|:---|:---|
| 通用科技 | `#1A73E8` 蓝 | `#34A853` 绿 | `#FBBC04` 黄 | `#FFFFFF` 白 |
| 深色科技 | `#0D47A1` 深蓝 | `#00BFA5` 青绿 | `#FF6D00` 橙 | `#1A1A2E` 深色 |
| 企业级 | `#1565C0` 蓝 | `#424242` 灰 | `#E53935` 红 | `#F5F5F5` 浅灰 |

#### 金融/咨询（稳重、信任、专业）

| 场景 | 主色 | 辅色 | 强调色 | 背景 |
|:---|:---|:---|:---|:---|
| 经典商务 | `#1B3A5C` 藏蓝 | `#B8860B` 金 | `#8B0000` 暗红 | `#FFFFFF` 白 |
| 现代金融 | `#004E89` 深海蓝 | `#FFB400` 金 | `#E8505B` 珊瑚红 | `#F8F9FA` 白灰 |
| 咨询风格 | `#2C3E50` 深灰蓝 | `#E74C3C` 红 | `#3498DB` 亮蓝 | `#ECF0F1` 浅灰 |

#### 教育/培训（温暖、亲和、活力）

| 场景 | 主色 | 辅色 | 强调色 | 背景 |
|:---|:---|:---|:---|:---|
| 教育培训 | `#2E7D32` 绿 | `#F57C00` 橙 | `#42A5F5` 蓝 | `#FFF8E1` 暖白 |
| 儿童教育 | `#FF6B35` 橙 | `#00B4D8` 青 | `#FFD166` 黄 | `#FFFFFF` 白 |
| 学术风格 | `#37474F` 蓝灰 | `#7B1FA2` 紫 | `#00897B` 青绿 | `#F5F5F5` 灰白 |

#### 医疗/健康（洁净、信任、关怀）

| 场景 | 主色 | 辅色 | 强调色 | 背景 |
|:---|:---|:---|:---|:---|
| 医疗健康 | `#00796B` 青绿 | `#4DD0E1` 亮青 | `#E91E63` 粉红 | `#FFFFFF` 白 |
| 制药 | `#1E88E5` 蓝 | `#43A047` 绿 | `#FDD835` 黄 | `#FAFAFA` 白 |

### 3.3 配色避坑指南

| ❌ 避免 | ✅ 建议 |
|:---|:---|
| 高饱和度文字色 + 高饱和度背景色 | 文字色与背景色保持明度差 ≥ 60% |
| 红绿搭配（色盲不友好） | 用红+蓝或蓝+橙替代 |
| 超过 5 种颜色 | 严格控制在 3 色以内 |
| 纯黑色文字（`#000000`） | 使用深灰色（`#333333`）更柔和 |
| 纯白色背景上的荧光色 | 降低饱和度或增大字号 |
| 花色/纹理背景上的文字 | 使用纯色背景或加文字遮罩层 |

### 3.4 无障碍配色参考

确保配色符合 WCAG 2.1 AA 标准（对比度 ≥ 4.5:1）：
- 深色背景上的文字：使用白色或浅色（`#FFFFFF` / `#F5F5F5`）
- 浅色背景上的文字：使用深色（`#333333` 而非 `#000000`）
- 链接文字：在正文基础上加下划线或使用蓝色（`#1A73E8`）
- 图表的色盲友好调色板：`#0072B2`（蓝）`#E69F00`（橙）`#009E73`（绿）`#F0E442`（黄）`#56B4E9`（浅蓝）`#D55E00`（红棕）

## 四、字体搭配指南

### 4.1 中文字体搭配推荐

| 场景 | 标题字体 | 正文字体 | 风格 |
|:---|:---|:---|:---|
| 商务/金融 | 思源黑体 Bold / 微软雅黑 Bold | 思源黑体 Regular / 微软雅黑 Light | 干净、现代、专业 |
| 科技/互联网 | 阿里巴巴 PuHui Bold | 阿里巴巴 PuHui Regular | 圆润、亲和、科技感 |
| 学术/教育 | 思源宋体 Bold | 思源宋体 Regular | 正式、权威、学术感 |
| 时尚/创意 | 汉仪旗黑 Bold / 方正兰亭黑 | 汉仪旗黑 Regular | 简约、现代、设计感 |
| 传统/文化 | 方正清刻本悦宋 Bold | 思源宋体 Regular | 典雅、文化、温润 |

### 4.2 英文字体搭配推荐

| 场景 | 标题字体 | 正文字体 | 风格 |
|:---|:---|:---|:---|
| 经典商务 | Montserrat Bold / Playfair Display | Open Sans / Lato | 专业、优雅 |
| 科技 | Inter Bold / Poppins Bold | Inter Regular / Roboto | 现代、清晰 |
| 创意 | GT Walsheim / Avenir | Source Sans Pro | 时尚、独特 |
| 学术 | Garamond / Times New Roman | Lora / Merriweather | 传统、权威 |
| 极简 | Helvetica Neue / SF Pro | Helvetica Neue Light | 干净、苹果风 |

### 4.3 字体使用规则

| 规则 | 说明 |
|:---|:---|
| 最多 2 种字体 | 标题 1 种 + 正文 1 种 |
| 同一字族不同字重 | 如：思源黑体 Bold（标题）+ 思源黑体 Regular（正文）= 1 种字体 |
| 最小字号 | 投影场景正文字号 ≥ 24pt，小标题 ≥ 28pt |
| 行距 | 1.2-1.5 倍行距 |
| 字间距 | 标题适当增加字距（tracking: 2-5%），正文保持正常 |
| 不要全部大写 | 中文无大写问题，英文标题避免全大写（降低可读性） |

### 4.4 字体嵌入提示

> ⚠️ 如果 PPT 需要在其他电脑上播放，建议：1) 使用系统内置字体（微软雅黑、宋体等）；2) 或将字体嵌入 PPT 文件（文件 → 选项 → 保存 → 将字体嵌入文件）；3) 或导出为 PDF 以防字体丢失。

## 五、图表选择指南

### 5.1 图表类型速查

| 你想表达什么？ | 推荐图表 | 不推荐 |
|:---|:---|:---|
| 不同类别的大小对比 | **柱状图** | 饼图（类别超过 5 个时） |
| 随时间的变化趋势 | **折线图 / 面积图** | 柱状图（数据点超过 15 个时） |
| 各部分占整体比例 | **环形图 / 饼图** | 3D 饼图（失真） |
| 两个变量的相关性 | **散点图 + 趋势线** | 柱状图 |
| 数据分布情况 | **箱线图 / 直方图** | 折线图 |
| 层级/流程关系 | **树状图 / 流程图** | 饼图 |
| 排名 | **条形图**（横向） | 柱状图（类别名过长时） |
| 多维度对比 | **雷达图 / 热力图** | 3D 图表 |
| 时间线 | **甘特图 / 时间轴** | 饼图 |
| 地理分布 | **地图（热力/气泡）** | 柱状图 |

### 5.2 图表设计规范

| 要素 | 规范 |
|:---|:---|
| 标题 | 每个图表必须有标题，位于图表上方或内部 |
| 坐标轴 | 标注清楚 X/Y 轴含义及单位 |
| 数据标签 | 关键数据点直接标注数值 |
| 图例 | 不超过 6 项，直接标注在系列旁 > 单独图例 |
| 配色 | 使用主配色方案中的颜色，避免默认 Excel 配色 |
| 网格线 | 淡化或隐藏，只保留必要的参考线 |
| 3D 效果 | 避免 3D 图表（导致视觉失真） |
| 动画 | 图表入场可分批出现（逐系列/逐数据点） |

### 5.3 数据呈现最佳实践

1. **最高值强调**：柱状图中将最高柱用强调色填充
2. **增量标注**：在趋势图的关键转折点添加文字标注
3. **标出基准线**：添加目标线、行业平均线作为参照
4. **减少小数位**：`3.2M` 而非 `3,214,567`，`53%` 而非 `53.28%`
5. **使用辅助线**：添加 y=0 基准线（避免截断误导）
6. **一致性**：同一组数据在全文中使用相同颜色

### 5.4 常见的图表误导陷阱

| 陷阱 | 说明 | 纠正 |
|:---|:---|:---|
| 截断 Y 轴 | 不从 0 开始的 Y 轴放大差异 | Y 轴必须从 0 开始（或标注截断标记） |
| 不等距刻度 | 时间轴上数据点间距不等 | 确保时间轴刻度均匀 |
| 3D 透视变形 | 3D 效果导致数值难以比较 | 使用 2D 图表 |
| 双 Y 轴滥用 | 左右 Y 轴量纲不同造成混淆 | 拆分两个图或标准化为同一单位 |
| 面积图过度填充 | 面积填充遮盖细节 | 使用折线图 + 透明填充 |

## 六、页面布局与排版

### 6.1 标准页面布局模板

```
┌──────────────────────────────────┐
│         标题区域（~20%）           │
│  完整的陈述句标题（30-36pt）       │
├──────────────────────────────────┤
│                                  │
│         内容区域（~65%）           │
│   - 要点 1 （24-28pt）            │
│   - 要点 2                       │
│   - 要点 3                       │
│                                  │
│        【图表/图片区域】           │
│                                  │
├──────────────────────────────────┤
│         底部信息（~15%）           │
│   页码 | 公司名 | 日期            │
└──────────────────────────────────┘
```

### 6.2 黄金比例版心

页面四周留白区域，遵循 9:16（宽屏）比例：
- 左右边距：画布宽度的 8-10%
- 上下边距：画布高度的 6-8%
- 标题与内容间距：画布高度的 3-4%

### 6.3 常用排版技巧

| 技巧 | 说明 |
|:---|:---|
| **左对齐** | 大多数场景的最佳选择，阅读视线自然 |
| **居中对齐** | 封面/转场页/数据高亮页使用 |
| **Z 字形布局** | 信息按「左上→右上→左下→右下」阅读顺序排列 |
| **F 字形布局** | 用户优先扫描左侧和顶部，重要信息放在 F 路径上 |
| **网格系统** | 将页面分为 12 列网格，按列对齐元素 |
| **对比** | 重要信息用大字号/高亮色/加粗区分 |
| **重复** | 同一层级的信息用相同的视觉元素表示 |
| **亲密性** | 相关元素靠近摆放，不相关元素拉开间距 |
| **对齐** | 所有元素都有意对齐，避免随意放置 |

### 6.4 图片使用规范

| 规则 | 说明 |
|:---|:---|
| 分辨率 | 至少 1920×1080 全屏显示不模糊 |
| 版权 | 使用免费可商用图库（Unsplash / Pexels / Pixabay） |
| 尺寸 | 宽高比尽量与版位匹配，避免拉伸变形 |
| 人物图片 | 人面向文字方向（视线引导） |
| 图标 | 使用 SVG 图标（无损缩放），推荐 FontAwesome / Material Icons |
| 蒙版 | 图片上方加半透明渐变蒙版，确保文字可读 |
| 降噪 | 每页 1 张高质量图片 > 3 张低质量图片 |

## 七、PPT 动画与转场

### 7.1 动画使用原则

| 原则 | 说明 |
|:---|:---|
| **有目的的动效** | 每个动画必须有功能（引导注意力/分层信息） |
| **自然速度** | 入场动画 0.3-0.5 秒，强调动画 0.5-1 秒 |
| **一致性** | 全篇使用同一套动画方案 |
| **克制** | 一页不超过 3 个动画元素 |

### 7.2 推荐的动画方案

| 动画类型 | 推荐效果 | 适用场景 |
|:---|:---|:---|
| 文本入场 | 淡入（Fade In） | 简洁、专业 |
| 图表入场 | 擦除（Wipe）从左到右 | 数据展示 |
| 图片入场 | 缩放（Zoom）1.05→1 | 视觉展示 |
| 强调 | 脉冲（Pulse）/ 变色 | 关键数据 |
| 页面切换 | 淡入淡出（Morph/Fade） | 所有场景 |

### 7.3 不推荐的动画

- ❌ 飞入（Fly In from far）
- ❌ 旋转（Spin）
- ❌ 弹跳（Bounce）
- ❌ 随机线条（Random Bars）

## 八、演讲备注写作指南

### 8.1 每页备注结构

```
┌─ 时间控制 ──────────────────────┐
│ ⏱ 约 90 秒                      │
├─ 过渡语 ────────────────────────┤
│ "刚才我们看到了市场规模的数据，    │
│  接下来我想聊聊在这个市场中，     │
│  用户真正面临什么问题。"           │
├─ 核心要点 ──────────────────────┤
│ • 先说痛点（用故事开头的效果更好） │
│ • 引用具体数据                   │
│ • 展示用户调研结果               │
├─ 手势/视觉提示 ──────────────────┤
│ ↗ 指向图表的高峰点               │
└─────────────────────────────────┘
```

### 8.2 开场白结构（30-60 秒）

```
1️⃣ 破冰/钩子（5-10 秒）
   "我做一个简单的调查..."

2️⃣ 建立共鸣（10-15 秒）
   "在座各位有没有遇到过这个情况..."

3️⃣ 引出主题（10 秒）
   "这就是我们今天要解决的问题。"

4️⃣ 预演议程（10-15 秒）
   "接下来我用 20 分钟分三个部分..."
```

### 8.3 常见过渡语库

| 场景 | 过渡语 |
|:---|:---|
| 切换到问题 | "刚才我们看到现状令人满意，但事实真的如此吗？" |
| 切换到方案 | "那么如何解决这些问题呢？我们来看..." |
| 切换到数据 | "口说无凭，我们来看一组数据。" |
| 切换到案例 | "这个概念听起来抽象，让我用一个真实案例来说明。" |
| 切换到总结 | "我们已经覆盖了三个核心点，快速回顾一下..." |
| 切换到 Q&A | "以上就是我的分享，欢迎各位提问。" |

### 8.4 Q&A 预判准备

提前准备以下三类问题的回答要点：
1. **关键质疑**：对核心论点的挑战（数据来源？竞品也有？）
2. **落地细节**：执行层面的追问（时间表？预算？团队？）
3. **扩展问题**：未来规划的疑问（下一步？长期愿景？）

## 十一、直接可用的文案写作指南

> 本章节指导 AI 如何写出**可直接复制到 PPT 中**的高质量文案。所有规则以「Copy-Paste Ready」为目标。

### 11.1 标题写作公式

| 公式 | 示例 | 适用场景 |
|:---|:---|:---|
| **[数字] + [名词] + [动词/结果]** | "Q2 营收同比增长 35%" | 工作汇报、数据页 |
| **[问题] + [解决方案]** | "如何将 3 天报表缩短到 3 分钟？" | 产品发布、提案 |
| **[动词] + [对象] + [成果]** | "AI 自动生成 CFO 级报表" | 产品介绍、功能页 |
| **[否定现状] + [正面未来]** | "告别手动报表，拥抱智能分析" | 痛点/方案页 |
| **[对比] + [结论]** | "竞品还在爬，我们已经在跑" | 竞争分析 |

**标题三要三不要：**
| ✅ 要 | ❌ 不要 |
|:---|:---|
| 完整的陈述句（有主语有谓语） | 名词短语（"Q2 营收数据"） |
| ≤ 15 个字，一眼读完 | 超过 20 个字，需要聚焦 |
| 包含数字或具体成果 | 空洞的形容词（"显著的""重要的"） |

### 11.2 Bullet Point 写作规范

**每个 bullet 是一条完整的陈述句，不是关键词列表。**

| ❌ 错误写法 | ✅ 正确写法 |
|:---|:---|
| 用户增长快 | MAU 从 1.2 万增长至 3.8 万，+217% |
| 团队经验丰富 | 核心团队来自 BAT，平均 8 年行业经验 |
| 产品性能好 | AI 报表生成仅需 3 分钟，准确率 99.7% |
| 市场空间大 | TAM 达 120 亿，年复合增长率 34% |

**平行结构规则**：同一页的所有 bullet 保持相同语法结构。

| 结构类型 | 示例（同一页） |
|:---|:---|
| **动词开头（推荐）** | • 覆盖 12 个行业、300+ 家企业客户<br>• 实现 99.9% 的系统可用率<br>• 保持 NPS 达 72 分（行业平均 45） |
| **名词开头** | • 行业覆盖：12 个垂直行业<br>• 技术指标：99.9% 可用率<br>• 客户口碑：NPS 72 分 |
| **数字开头** | • 12 个行业，300+ 客户已覆盖<br>• 99.9% 可用率，零宕机事故<br>• 72 分 NPS，领先行业 27 分 |

### 11.3 数据陈述模板

| 场景 | 公式 | 示例 |
|:---|:---|:---|
| 展示增长 | `[指标] 从 [旧值] → [新值]（+[增幅]）` | MAU 从 1.2 万 → 3.8 万（+217%） |
| 展示对比 | `[我方] [指标]，[竞品] [指标]` | 我方获客成本 120 元，竞品 380 元 |
| 展示占比 | `[子项] 占 [整体] 的 [百分比]` | 企业客户贡献 65% 的总营收 |
| 展示趋势 | `连续 [N] 个季度 [上升/下降]，[最新值]` | 连续 6 个季度正增长，最新 Q4 达 8.7M |
| 展示效率 | `从 [旧时长] 缩短到 [新时长]（快 [倍数] 倍）` | 报表从 3 天缩到 3 分钟，快 1440 倍 |

**数据展示铁律：**
1. 数据必须附单位（万/亿/%/元/天）
2. 数据必须有参照系（"从 X 到 Y"，"比上季度增长 Z%"）
3. 数据必须标注时间范围（"Q2 2024""过去 12 个月"）
4. 大数字用缩写（3.2M 而非 3,200,000，8.7 亿而非 870,000,000）

### 11.4 开场白写作框架

**30-60 秒开场白 = 破冰 + 共鸣 + 引出主题**

| 框架 | 示例 |
|:---|:---|
| **数据冲击** | "过去 3 个月，我们调研了 200 家企业，发现 87% 的财务团队还在手动做报表。这意味着什么？" |
| **故事开场** | "上个月，一家客户的 CFO 告诉我，她团队每个季度末平均加班 40 个小时做报表。这不是个例。" |
| **提问互动** | "在座各位，有多少人每个月至少花 1 天时间做报表？——好，几乎所有人。" |
| **惊人事实** | "全球企业每年花在报表制作上的时间超过 50 亿小时。50 亿小时，相当于 57 万年。" |

### 11.5 过渡语库（可直接复制到备注栏）

| 场景 | 过渡语 | 时长 |
|:---|:---|:---:|
| 背景 → 问题 | "看起来一切正常，但事实真的如此吗？" | 5 秒 |
| 问题 → 方案 | "那么，怎么解决这个问题？" | 5 秒 |
| 方案 → 数据 | "口说无凭，来看一组真实数据。" | 5 秒 |
| 数据 → 案例 | "抽象的数据可能不太好理解，我讲一个真实案例。" | 5 秒 |
| 案例 → 总结 | "这个案例说明了什么？[结论]" | 10 秒 |
| 第一部分 → 第二部分 | "好。以上就是市场机会的部分。那我们的方案具体长什么样？" | 10 秒 |
| 内容 → 结尾 | "快速回顾今天的关键信息..." | 10 秒 |

### 11.6 文案精简对照表

| 原文（废话版） | 精简版（可直接用） |
|:---|:---|
| 我们的产品拥有很多非常强大的功能 | 3 个核心功能解决 80% 的报表需求 |
| 用户对我们产品的反馈非常好 | NPS 72 分，推荐率 91% |
| 我们的团队有着非常丰富的行业经验 | 团队平均 8 年行业经验，来自 BAT 与字节 |
| 市场增长空间非常巨大 | TAM 120 亿，年复合增长 34% |
| 我们的产品会通过多种方式帮助客户省钱 | 每客户年均节省 240 人力小时，折合 12 万元 |
| 在未来我们会继续努力做得更好 | Q3 路线图：多语言支持 + API 开放平台 |

**精简口诀：**
- 「非常」→ 删掉（99% 的情况不需要）
- 「很多」→ 换成具体数字
- 「好/坏」→ 换成数据/NPS/对比
- 「未来」→ 换成具体时间节点（Q3/2025/12 月）

### 11.7 不同类型页面的文案要点

#### 痛点页
```
❌ 错误："企业报表制作效率低"
✅ 正确：
• 一份月报平均需要 3 天完成，涉及 4 个部门协作
• 手动报表错误率高达 8%，季度末需反复核对
• 87% 的 CFO 表示"报表耗时太长，影响战略分析"
```

#### 方案页
```
❌ 错误："AI 能帮助提高效率"
✅ 正确：
• AI 自动接入财务/业务数据源，3 分钟生成完整报表
• 自然语言查询：说人话就能出图表，无需 IT 支持
• 一键生成董事会级可视化报告，排版即用
```

#### 数据页（KPI 展示）
```
❌ 错误："各项指标表现良好"
✅ 正确：
• MAU：1.2 万 → 3.8 万（+217%），连续 6 个月正增长
• 营收：Q1=120 万 → Q4=870 万，全年 ARR 破 3000 万
• 留存：D30 留存率 68%，行业平均 42%
```

#### 团队页
```
❌ 错误："团队经验丰富，背景优秀"
✅ 正确：
• CEO：前 Oracle 产品总监，10 年 SaaS 经验
• CTO：ACM 金牌得主，前阿里 P8 架构师

```

#### CTA（行动号召）页
```
❌ 错误："欢迎联系我们"
✅ 正确：
• 今天开放 10 个免费试用名额 → 扫码立即申请
• 首年签约享 5 折优惠 + 专属客户成功经理

```

### 12.3 政务场景配色方案

| 风格 | 主色 | 辅色 | 强调色 | 适用场景 |
|:---|:---|:---:|:---:|:---|

| **科技政务蓝** | `#004098` 科技蓝 | `#FFFFFF` 白 | `#00A8E8` 亮蓝 | 技术方案、智慧城市类项目 |

**政务配色铁律：**
1. 颜色不超过 3 色（主+辅+强调）
2. 对比度要够——政府汇报通常投影仪清晰度不高
3. 红色只用做强调色（不要大面积使用），蓝色做主色最安全
4. 背景使用浅色/白色为主，深色背景在政府场景中慎用
5. **避免使用渐变色模拟"科技感"**——标准政务场景简洁清晰更重要

|:---|:---|:---|:---|
| 正式汇报（局级以上） | 方正小标宋 / 方正大标宋 | 仿宋 / 楷体 | 模仿政府公文风格 |
| 技术方案（信息中心/评审） | 思源黑体 Bold / 微软雅黑 Bold | 思源黑体 Regular / 微软雅黑 | 清晰、现代、通用 |
| 日常汇报 | 微软雅黑 Bold | 微软雅黑 | 最安全的选择 |

> ⚠️ **字体安全**：政务办公电脑通常只预装微软雅黑、宋体、仿宋、楷体。如果使用特殊字体，务必将 PPT 嵌入字体或转 PDF。建议输出时优先推荐**微软雅黑**作为通用安全选择。

### 12.7 三个场景的演讲风格

#### 产品/方案推介（面向客户领导）
```
• 语气：自信、专业、政策性
• 站位：站在政府视角思考（"为全市群众服务"），而非"帮您解决问题"
• 关键句式："结合 XX 省/市在 XX 方面的要求，我们提出..."
• 时间分配：政策+痛点 40% | 方案 40% | 实施+预算 20%
• 切忌：不要从头到尾讲技术细节——讲"业务故事"
```

#### 规划/建设方案（面向评审专家）
```
• 语气：严谨、客观、学术性
• 逻辑：每个结论必须有依据（政策依据、调研数据、国家标准）
• 关键句式："基于 XX 调研发现..."; "参考 XX 标准规范..."
• 时间分配：需求分析 20% | 方案 60% | 实施+投资+风险 20%
• 切忌：不要夸大预期效果，专家最反感官话套话
```

#### 项目建设汇报（面向甲方领导）
```
• 语气：诚实、透明、职业化
• 原则：好事说清楚（完成什么），坏事说透彻（什么问题+怎么解决）
• 关键句式："本期完成了 XX，对比计划..."; "当前存在 XX 问题，我们已采取 XX 措施"
• 时间分配：成果 50% | 问题 20% | 计划 20% | 协调 10%
• 切忌：不要报喜不报忧——领导最关心的是风险是否可控
```

### 12.8 评审 Q&A 预判题库（常用）

| 问题类型 | 常见问题示例 |
|:---|:---|
| **技术** | "这个方案和现有的 XX 系统如何对接？数据标准统一吗？" |
| **安全** | "等保能确保过吗？数据存在哪里？" |
| **信创** | "适配哪些国产数据库？在什么芯片上测试过？" |
| **预算** | "运维费用怎么算的？第一年都包含什么？有没有隐性费用？" |
| **工期** | "6 个月能交付吗？验收标准是什么？" |
| **数据** | "哪些数据需要其他部门提供？协调机制有没有？" |
| **利旧** | "现有硬件/软件怎么利用？会不会重复建设？" |
| **风险** | "最大的技术风险是什么？怎么应对？" |
| **效益** | "建成后怎么考核效果？有没有量化指标？" |
| **运维** | "运营团队多少人？技术支持响应时效？" |

### 12.9 格式规范速查（三统一原则）

> 📋 **本页为 SKILL.md 第 3 步格式硬性标准的速查摘要，便于快速参考**

#### 12.9.1 格式统一（字体 + 字号）

| 层级 | 推荐字体 | 字号范围 |
|:---|:---|---:|
| **PPT 总标题**（封面） | 微软雅黑 Bold / 黑体 | ≥ 30pt |
| **一级标题**（章节页） | 微软雅黑 Bold / 黑体 | 20-28pt |
| **二级标题**（内容小标题） | 微软雅黑 Medium | 16-18pt |
| **正文（文字少时）** | 微软雅黑 / 宋体 | ≥ 14pt |
| **正文（文字多时）** | 微软雅黑 / 宋体 | ≥ 10pt |

**字体规则：**
- 标题：微软雅黑、黑体为主
- 内容：微软雅黑、宋体为主
- 同一页 ≤ 2 种字体（标题 1 种 + 正文 1 种）
- 政务场景优先推荐**微软雅黑**（系统自带，跨电脑兼容好）
- 正文行距：1.2-1.5 倍

#### 12.9.2 颜色统一（主色调 + 每页 ≤ 3 色）

| 规则 | 要求 |
|:---|:---|
| **主色调** | 每份 PPT 必须统一一个主色调，推荐四大政务安全色：`科技蓝` / `活力橙` / `商务灰` / `淡雅绿` |
| **每页颜色数** | ≤ 3 色（主色 + 辅色 + 强调色） |
| **正文颜色** | 黑色或深灰色（推荐 `#333333`，避免纯黑） |
| **重点突显** | 关键数据/关键词用强调色或加粗标亮 |
| **背景** | 浅色背景 + 深色文字 |

#### 12.9.3 布局合理（对齐 + 间距 + 一致性）

| 维度 | 要求 |
|:---|:---|
| **对齐** | 正文左对齐（阅读自然），封面/转场页/数据高亮页居中。同一页所有内容左右边界对齐一致 |
| **边距** | 左右留白 8-10%，上下留白 6-8%，避免内容贴边 |
| **间距** | 标题与正文间距 > bullet 点之间的间距。相关元素靠近，不相关元素拉开 |
| **一致性** | 缩进、序号、段落符号同一层级保持一致 |
