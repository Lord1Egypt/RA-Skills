# 📚 自学教练：一个自考老师用 AI 做的学习闭环，学生通过率提升近 30%

---

## 一句话介绍

**一个 Skill，两步闭环**：把讲义变成可视化学习指南网页，把真题变成沉浸式刷题网页。学生双击就能用，学完就练，练完就过。

---

## 为什么做这个？

我是一名自考专职老师，教《马克思主义基本原理》《习概》《中国近现代史纲要》这些课。

自考的学生有个特点：**基础参差不齐，自学能力差异大，而且大多是边工作边备考**。

每学期我都在做同一件事：
1. 把知识点整理成讲义发给学生
2. 把历年真题汇总发给学生
3. 然后在群里喊："大家记得看讲义！记得刷真题！"

结果呢？

```
👨‍🏫 老师：
  "讲义发了，真题也发了，该做的都做了。"

👨‍🎓 学生：
  "讲义太长了看不下去……"
  "真题做完对完答案就忘了……"
  "老师，这个知识点在讲义哪里？"
  "老师，这道题为什么选 A 不选 B？"

结果：通过率一直上不去。
```

**问题不在内容，在于形式。**

PDF 讲义学生不爱看，Word 真题做完就忘。我需要的不是更好的内容，而是**让内容"活"起来的方式**。

---

## 它是什么？

「自学教练」由两个配合使用的能力组成：

### 📖 能力一：学习指南

**把抽象的讲义知识点，变成可视化、可交互的 HTML 学习网页。**

5 种交互组件，让学生主动探索而不是被动阅读：

| 组件 | 做什么 |
|:---|:---|
| 渐变沉浸式头部 | 打开网页就有视觉冲击，知道这节课很重要 |
| 标签页切换 | 知识点分步展示，不一次性堆出来 |
| 情景连线高亮 | 点击案例，自动高亮对应的理论知识点 |
| 魔法反转 | 一键对比"低分答案"和"满分卷面" |
| 自测翻牌 | 遮住答案，自己先想，再点开核对 |

### ✍️ 能力二：刷题应用

**把真题/模拟题，变成有四种练习模式的交互式刷题网页。**

| 模式 | 怎么用 |
|:---|:---|
| 刷题模式 | 做一题看一题答案+解析，即时反馈 |
| 考试模式 | 全程不看答案，交卷后出成绩报告 |
| 背题模式 | 所有题目+正确答案+解析，快速过一遍 |
| 错题本 | 自动收集做错的题，考前只刷错题 |

### 两个能力怎么配合？

```
老师的讲义              历年真题
    ↓                      ↓
 学习指南               刷题应用
    ↓                      ↓
 可视化学习网页          交互式刷题网页
    ↓                      ↓
    └────── 发给学生 ──────┘
              ↓
       学完就练，练完就过
              ↓
         通过率提升
```

## 真实效果：通过率提升近 30%

这不是假想产品，**这是我的日常教学工具**。

我每上完一个章节，就花几分钟告诉 AI：
- "这是第三章的讲义，帮我做一个学习指南"
- "这是近5年真题，帮我做一个刷题网页"

AI 生成两个 HTML 文件，我直接发到班级群。

学生双击打开，手机/电脑/机房都能用。

**结果**：

```
使用前：
  - 讲义打开率：不到 30%
  - 真题完成率：约 40%
  - 考试通过率：约 50%

使用后：
  - 学习指南打开率：超过 80%（网页比 PDF 有吸引力）
  - 刷题完成率：超过 70%（有即时反馈，愿意刷）
  - 考试通过率：接近 80%（提升近 30%）
```

**学生的反馈**（附截图）：

> "老师这个网页太方便了，我在地铁上都能刷题"

> "错题本功能太香了，考前只看错题就够了"

> "那个魔法反转好好玩，我终于知道满分答案长什么样了"

> "比我买的网课还好用，至少不用等加载"

## 和市面上的工具比，有什么不同？

| 维度 | 商业学习平台 | 普通 PDF/Word | 自学教练 |
|:---|:---|:---|:---|
| 费用 | 几百到几千/年 | 免费 | 免费 |
| 交互性 | 有，但需注册登录 | 无 | 有，双击即用 |
| 可分享性 | 平台内使用 | 可分享但不吸引人 | 单文件，微信/邮件直发 |
| 离线使用 | 需联网 | 可以 | 刷题应用零依赖，断网也能用 |
| 老师修改 | 需登录后台操作 | 直接编辑 | 记事本打开改顶部数据即可 |
| 学生隐私 | 需注册，数据在平台 | 无 | 数据全在本地浏览器 |

**核心差异**：不需要注册、不需要下载 App、不需要付费、不需要联网（刷题应用）。一个 HTML 文件搞定一切。

## 技术亮点

1. **单文件交付**：所有代码（HTML + CSS + JS）封装在一个文件里
2. **零部署成本**：双击打开，或发到群里，学生直接使用
3. **题库数据在文件顶部**：`QUESTION_BANK` 常量在第一行，老师用记事本就能改题目
4. **学习进度持久化**：localStorage 保存，支持导出/导入 JSON 跨设备
5. **响应式设计**：手机/平板/电脑/投影仪都能用
6. **零外部依赖（刷题应用）**：机房断网也不影响

## 创作过程

**起点**：我自己教自考，每学期都为"学生不看讲义、不刷真题"而焦虑。

**第一步**：先做了学习指南，把《习概》主观题做成交互式网页。学生反馈"终于看得下去了"。

**第二步**：发现学生看完指南但不练，又做了刷题应用。四种模式覆盖不同学习阶段。

**第三步**：两个组合使用后发现，通过率明显提升。学生开始主动问我"老师下一章的网页什么时候出"。

**第四步**：固化为 Skill，现在每上完一节课，几分钟就能生成两个网页。
