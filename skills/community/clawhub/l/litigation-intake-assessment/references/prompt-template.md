# 结构化提示词模板

用本模板快速生成《案件初步评估报告》。必要时按行业或案由轻微改写，但不要改动核心步骤。

## 简版模板

```text
请作为中国大陆争议解决律师，基于以下信息生成《案件初步评估报告》：

- 案件基本事实：
- 初步证据材料：
- 我方诉求：
- 管辖法院或地域信息：

请按以下步骤处理：
1. 完成 S1：提取主体、行为、金额、日期、关键条款、关键证据、程序信息，并标记缺失要素。
2. 完成 S4：判断核心/从属法律关系，识别高/中/低风险点，对证据三性作初判。
3. 围绕争议焦点调用司法裁判知识库，检索法条与类案，并说明对我方有利/不利点。
4. 依据 `assessment-metrics.md` 量化估算胜诉概率、诉讼周期、资金成本与时间成本。
5. 给出接案建议，并列出下一步补证与诉讼/谈判动作。

请使用以下固定结构输出：
# 案件初步评估报告
## 一、案件概况与争议焦点
## 二、法律关系与证据评估
## 三、风险识别与裁判检索结论
## 四、量化评估
## 五、接案建议
```

## 扩展模板

```text
你是一名负责争议解决与诉讼管理的中国大陆律师，当前处于接案评估阶段。请根据我提供的案件基本事实、初步证据、我方诉求和管辖法院信息，输出一份《案件初步评估报告》。

处理要求：
- 先做内部结构化提取，不遗漏主体、金额、时间、条款、证据和程序信息。
- 对缺失事实、缺失证据和版本冲突显式标记，不得自行编造。
- 完成法律关系识别、风险分级和证据三性初判。
- 将争议拆解为可检索的焦点，优先检索请求权基础、抗辩事由、责任比例、损失认定、举证责任和当地法院裁判倾向。
- 量化估算胜诉概率、诉讼周期、资金成本和时间成本；每一项都写清测算依据与限制。
- 给出明确的接案建议：建议接案 / 有条件接案 / 谨慎接案 / 不建议接案。

输入信息：
- 案件基本事实：{{facts}}
- 初步证据材料：{{evidence}}
- 我方诉求：{{claims}}
- 管辖法院信息：{{court}}
- 其他补充背景：{{extra}}

输出要求：
- 使用正式、清晰、可供律师复核的中文。
- 先结论、后理由。
- 不引用不存在的法条、案例、案号或数据。
- 若检索受限，单列“检索限制与影响”。
```
