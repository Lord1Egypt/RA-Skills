## A0 协作协调域 范本

### 范本概述
- **域ID**: A0
- **域名称**: 协作协调域
- **任务数量**: 4
- **范本状态**: 可用

### 任务执行范本

#### A0-01 协作需求分析
**执行步骤**:
1. 接收协作请求，明确协作目标
2. 分析协作范围和需求
3. 识别协作约束
4. 评估协作优先级
5. 汇总协作需求

**输入示例**:
- 协作请求：需要多个Agent协作完成软件开发任务
- 协作目标：提高开发效率，保证代码质量
- 协作范围：需求分析、设计、开发、测试

**输出示例**:
- 协作目标：提高开发效率，保证代码质量
- 协作范围：需求分析、设计、开发、测试
- 协作需求：需要Explore Agent进行代码探索，Plan Agent进行架构设计，General-purpose Agent进行开发
- 协作约束：需要在2周内完成，代码质量要高

**注意事项**:
- 协作目标要明确，避免模糊
- 协作范围要合理，避免过宽或过窄
- 协作需求要全面，避免遗漏

#### A0-02 协作域选择
**执行步骤**:
1. 分析协作需求特征
2. 扫描可用协作域列表
3. 匹配协作需求与协作域能力
4. 选择最合适的协作域组合
5. 确定协作域选择理由

**输入示例**:
- 协作需求：需要多个Agent协作完成软件开发任务
- 可用协作域：Agent管理域、任务委派域、并行执行域、结果汇总域

**输出示例**:
- 选择协作域：Agent管理域 + 任务委派域 + 并行执行域 + 结果汇总域
- 选择理由：软件开发需要完整的Agent协作流程
- 协作域组合方案：Agent管理域负责Agent管理，任务委派域负责任务分配，并行执行域负责并行执行，结果汇总域负责结果整合

**注意事项**:
- 协作域选择要基于实际需求
- 避免选择过多或过少的协作域
- 协作域组合要合理，避免重叠

#### A0-03 执行顺序确定
**执行步骤**:
1. 分析协作任务依赖关系
2. 识别关键路径
3. 确定执行顺序
4. 识别并行执行机会
5. 绘制执行顺序图

**输入示例**:
- 协作任务：Agent管理、任务委派、并行执行、结果汇总
- 依赖关系：任务委派依赖Agent管理，并行执行依赖任务委派，结果汇总依赖并行执行

**输出示例**:
- 执行顺序：Agent管理 → 任务委派 → 并行执行 → 结果汇总
- 关键路径：Agent管理 → 任务委派 → 并行执行 → 结果汇总
- 并行机会：Agent管理和任务分析可并行执行

**注意事项**:
- 依赖关系要准确，避免循环依赖
- 关键路径要识别，避免延误
- 并行机会要充分利用，提高效率

#### A0-04 结果整合与优化
**执行步骤**:
1. 收集各协作任务执行结果
2. 验证结果完整性
3. 整合相关结果
4. 优化整体输出
5. 汇总整合报告

**输入示例**:
- Agent管理结果：已注册3个Agent
- 任务委派结果：已委派5个任务
- 并行执行结果：3个任务并行执行完成
- 结果汇总结果：所有任务结果已整合

**输出示例**:
- 整合结果：完整的软件开发协作系统
- 优化建议：优化Agent能力，提高并行效率
- 整合报告：详见附件

**注意事项**:
- 结果收集要全面，避免遗漏
- 结果验证要严格，避免质量问题
- 整合要保持一致性，避免冲突

### 范本特征摘要
- **结构特征**: 按域分组，每个域包含多个任务，每个任务有明确的执行步骤
- **风格特征**: 专业、清晰、结构化，使用表格和列表展示信息
- **逻辑特征**: 从需求分析到结果整合，逻辑清晰，步骤明确
- **格式特征**: 使用Markdown格式，包含标题、列表、表格等元素
