# li-bid-document-maker 사용 가이드

> **입찰 문서 작성 전문가** — 입찰 공고문을 평가 기준에 기반한 전문적인 응찰 문서로 자동 변환합니다

## 설치

```bash
clawhub install li-bid-document-maker
```

## 빠른 시작

설치 후 AI 어시스턴트에게 다음 중 하나를 말하면 스킬이 실행됩니다:

- "입찰 문서를 작성해줘"
- "이 입찰 공고문을 분석해서 응찰 문서를 생성해줘"
- "입찰 제안서 준비를 도와줘"
- "RFP에 대한 응답을 생성해줘"

AI가 자동으로 6단계 워크플로를 실행합니다: **입찰 문서 분석 → 전략 분석 → 개요 생성 → 장별 작성 → 품질 검사 → PDCA 자동 개선**

## 사용 방법

### 방법 1: 파일 업로드

입찰 공고문(PDF 또는 Word)을 채팅창에 드래그 앤 드롭한 후 말씀하세요:
> "li-bid-document-maker를 사용하여 이 문서를 분석해주세요"

### 방법 2: 파일 경로 지정

> "li-bid-document-maker를 사용해주세요. 입찰 파일은 /projects/tender.pdf에 있습니다"

## 시스템 요구사항

| 항목 | 요구사항 |
|------|---------|
| OS | Windows 10/11, macOS, Ubuntu 20.04+ |
| AI 플랫폼 | Claude, OpenClaw, Hermes 또는 파일 I/O를 지원하는 모든 LLM Agent |
| 파일 형식 | PDF(검색 가능 권장) 또는 Word (.docx) |
| 의존성 | `python-docx`, `PyPDF2` (파일 처리용, 선택 사항) |

## 출력

- 완전한 응찰 문서(16장 표준 구조)
- PDCA 품질 개선 보고서
- 평가 기준 및 배점 매핑

## 워크플로

```
단계1: 입찰 문서 분석    → 프로젝트 정보, 기술 요구사항, 평가 기준 추출
단계2: 전략 분석          → 평가 가중치 분석, 경쟁 전략 수립
단계3: 개요 생성          → 평가 기준 기반 개요 (사용자 확인 필요)
단계4: 장별 작성          → 섹션별 내용 작성
단계5: 품질 검사          → 6차원 전체 범위 검사
단계6: PDCA 개선          → 3라운드 자동 개선 후 최종 전달
```
