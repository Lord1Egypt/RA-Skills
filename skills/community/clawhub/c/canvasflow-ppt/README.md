## 📖 简介

**CanvasFlow PPT** —— 流畅的画布，流动的演示。

PPT 的制作依靠审美，可很多初学者需要模板。特别将 10 年设计经验的设计师进行知识、审美蒸馏提炼后，给每个刚入手的兄弟姐妹提供一个快速美观、简便的 PPT Skill。

## 🎨 名字寓意

| 单词 | 含义 |
|------|------|
| **Canvas（画布）** | WebGL 流体背景，像艺术画布一样承载内容，hero 页明显透出 |
| **Flow（流动）** | 横向翻页流畅，主题平滑过渡，视觉节奏呼吸，light/dark 交替 |

---

## ✨ 核心特性

### 视觉设计
- **WebGL 双背景**：dark 页用钛金色散，light 页用银色流动
- **三级字体分工**：衬线标题 / 非衬线正文 / 等宽元数据
- **5 套主题色预设**：墨水经典 / 靛蓝瓷 / 森林墨 / 牛皮纸 / 沙丘
- **10 种布局骨架**：封面 / 幕封 / 数据大字报 / 左文右图 / Pipeline 等

### 交互体验
- **全方位导航**：键盘 ← → / 滚轮 / 触屏滑动 / 底部圆点 / ESC 索引
- **主题平滑切换**：翻到 light 页，背景自动切换为银色流动
- **响应式设计**：桌面 / 移动端自适应

### 工作流规范
- **6 步标准化流程**：环境检测 → 需求澄清 → 主题选择 → 节奏规划 → 填充内容 → 质量自检
- **严格预检机制**：必须先读取 template.html，确认类名存在
- **质量检查清单**：P0/P1/P2/P3 分级，P0 级必须全过

## 🔧 使用方式

### 在 QClaw 中触发

说出以下关键词即可触发：
- "做 PPT"
- "生成演示"
- "CanvasFlow PPT"
- "画布演示"

### 标准工作流

1. **环境检测**：读取 template.html，确认类名存在
2. **需求澄清**：6 问清单（受众/时长/素材/主题色/约束/格式）
3. **主题选择**：从 5 套预设中选择
4. **节奏规划**：规划 light/dark/hero 交替
5. **填充内容**：从 10 种布局骨架中选择，填充内容
6. **质量自检**：对照检查清单，P0 级必须全过

## 🎯 设计哲学

> 继承自 guizang-ppt-skill：

1. **克制优于炫技** — WebGL 背景只在 hero 页透出
2. **结构优于装饰** — 信息靠字号 + 字体对比 + 网格留白
3. **内容层级由字号和字体共同定义**
4. **图片是第一公民** — 只裁底部，顶部和左右完整
5. **节奏靠 hero 页** — hero 和 non-hero 交替
6. **术语统一** — Skills 就是 Skills，不中英混译
