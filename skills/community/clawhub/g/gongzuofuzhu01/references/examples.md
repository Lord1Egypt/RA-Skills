# Personal Assistant 典型使用场景

以下是 7 个典型使用场景，每个场景包含用户输入示例和系统响应示例。

---

## 场景 1: 快速添加任务

**用户输入：**
> 帮我记一下：周五前完成 Q2 述职 PPT，优先级高，大概需要 8 小时

**系统响应：**
```
✅ 任务已创建

ID:      42
标题:    完成 Q2 述职 PPT
状态:    📋 待办
优先级:  🔴 高 (2)
截止:    2026-05-29 (周五)
预估:    8 小时
分类:    -

使用 task update 命令可修改字段。
```

## 场景 4: 设置周期任务

**用户输入：**
> 帮我设置一个周期任务：公司双周例会，每周三下午 3 点，提前 2 天提醒我

**系统响应：**
```
🔄 周期任务已创建

ID:         5
标题:       公司双周例会
类型:       双周 (biweekly)
下次生成:   2026-06-03
提前提醒:   2 天
分类:       会议
优先级:     中 (3)
预估:       1.5 小时 (默认)

下次例会任务将在 2026-06-01 自动生成。
使用 recurring toggle 5 可暂停此周期任务。
```

**用户输入：**
> 给我一些本周的工作建议

**系统响应：**
```
💡 本周工作建议

🔴 紧急处理:
  1. #38 修复登录模块 BUG — 今天截止，建议优先完成
  2. #42 完成 Q2 述职 PPT — 周五截止，还剩 3 天

📋 建议安排:
  周三上午: 集中处理 BUG 修复 (预估 3h)
  周三下午: 公司双周例会 (1.5h)
  周四全天: PPT 制作 — 数据收集 + 大纲 + 初稿 (8h)
  周五上午: PPT 终稿 + 内部审核 (2h)

🔄 周期任务提醒:
  本周五 — 每周代码评审 (未生成，明天自动创建)

⚠️ 阻塞任务:
  #21 周会纪要整理 — 停滞 3 天未更新
```
