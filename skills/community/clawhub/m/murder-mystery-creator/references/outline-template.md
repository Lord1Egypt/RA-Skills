# 大纲生成模板

> 本文件是生成剧本大纲的详细规范。生成大纲时严格按此模板填写每一个字段。

---

## 模板结构

### 【1】故事核心（一句话版本）

> 格式：在[地点]，[人物群体]因[触发事件]聚集，随后[死者姓名]以[死亡方式]死去，所有人都有嫌疑。

示例：
- "暴风雪夜，受邀参加庄园宴会的六人被困山中，而主人竟在深夜被人毒杀，凶手就在宾客之中。"
- "奇幻博物馆一日游变成了惊魂噩梦——马丁先生在展厅密室中离奇死亡，而每个游客都有不在场的漏洞。"

---

### 【2】世界观 & 场景

```
时代背景：[现代/民国/清末/架空/未来/欧洲复古...]
主场景名称：[庄园/别墅/学校/公司/邮轮/雪地旅馆...]
场景封闭原因：[暴雪/孤岛/停电/事故隔离/主动锁门...]

场景设计三问（必答，否则逻辑漏洞）：
  1. 玩家能否自由离开？如不能，理由是？
  2. 场景内能否使用手机/网络？如能，如何限制？
  3. 场景内有监控吗？如有，分布在哪里？（影响凶手行动设计）

场景描述（200字左右）：
  [描写场所的外观、氛围、关键房间/区域的布局，
   用能让玩家代入的语言描写，
   提及1-2个后续将成为关键的地点细节]

可搜证的地点列表：
  - 地点1：[名称]（搜证内容：[会发现什么]）
  - 地点2：[名称]（搜证内容：[会发现什么]）
  - ...（共5-8个地点）
```

> **架空世界观特别说明**：如果选择架空世界，必须在时代背景后补充"科技边界"——即这个世界的最高科技水平是什么，能力上限是什么。开局没规定的超纲能力，中途出现会引起玩家反感。

---

### 【3】事件触发（开场事件）

```
事件发生时间：[具体时间，如"1943年12月24日深夜11时"]
发现者：[由哪位角色或全体玩家一起发现]
事件描述（逐字稿，主持人朗读）：
  [描写发现尸体/异常事件的完整场景，
   包括死者位置、状态、周围环境，
   用第一人称感受或第三人称旁白描写，营造氛围，
   结尾抛出核心悬念："凶手是谁？"]

表面现象（玩家最初获得的信息）：
  - 死亡时间：[大概时段，不精确]
  - 死亡方式：[表面看起来是什么，如"看起来是心脏病"]
  - 死亡地点：[具体位置]
  - 异常发现：[1-3条可疑迹象，足够引起怀疑但不足以确定凶手]
```

---

### 【4】完整时间线

> 这是大纲中最核心的部分。必须精确到小时，并标注每个事件的可见性。

格式：

| 时间 | 事件描述 | 涉及角色 | 可见性 | 备注 |
|------|---------|---------|-------|------|
| XX:00 | [发生了什么] | [谁] | 公开/私密/凶手独知 | [线索关联] |

**可见性说明：**
- **公开**：所有玩家均知晓（通过主持人宣读或公开线索）
- **私密**：通过角色私密剧情或私密线索知晓（仅部分玩家）
- **凶手独知**：只有凶手玩家知道（在凶手私密剧情中揭示）
- **无人知晓→终极线索**：通过第三轮终极线索才揭示

**时间线填写要求：**
- 至少覆盖案发前24小时
- 凶手的行动必须完整（动机→准备→作案→善后）
- 每个非凶手角色在案发时段都必须有自己的行动（用于制造可疑）
- 最少3个时间节点存在"对立叙述"（两人对同一时间段的描述不一致）

---

### 【5】角色关系图谱

> 列出所有角色及其两两之间的关系。关系要有层次感：表面关系 vs 隐藏关系。

格式：

**角色列表：**

| 编号 | 姓名 | 年龄 | 表面身份 | 隐藏秘密（一句话） |
|------|------|------|---------|-----------------|
| 1 | [姓名] | [年龄] | [职业/身份] | [核心秘密] |
| ... | | | | |

**关系矩阵：**

| | 角色1 | 角色2 | 角色3 | ... |
|-|------|------|------|-----|
| 角色1 | — | [与2的关系] | [与3的关系] | ... |
| 角色2 | | — | | |

关系描述格式：`[表面关系] / [隐藏关系]`
例：`老同学 / 实为商业竞争对手，互相了解对方的把柄`

**与死者的关系（重点）：**
每个角色与死者的关系都必须满足：
1. 有接触机会（能接近死者）
2. 有动机（至少一条理由希望死者消失）
3. 有机会（时间线上有单独接近死者的窗口）

---

### 【6】真相 & 凶手逻辑

> 作者视角，只有主持人看到。这是整个剧本的灵魂，必须完全自洽。

```
凶手姓名：[XX]
作案动机：[详细说明，必须有充分的情感或利益根据]

作案完整经过：
  1. [时间] [凶手做了什么准备]
  2. [时间] [凶手如何接触死者]
  3. [时间] [作案过程，包括使用的道具/方法]
  4. [时间] [如何处理现场/制造不在场证明]
  5. [时间] [事后行为，如何在玩家中伪装]

关键证据（可以推导出凶手的必要线索）：
  - 证据1：[证据描述] → 收录于[第X轮线索，线索名称]
  - 证据2：[证据描述] → 收录于[第X轮线索，线索名称]
  - ...（至少3条独立证据，单条不足以确定，联合可确定）

误导线索设计：
  - [角色名]的可疑点：[他/她看起来可疑的理由] → [实际解释（无罪）]
  - ...（每个非凶手角色至少1条误导线索）

叙事反转（可选，强烈建议）：
  [描述游戏中期或结尾的剧情反转，如"死者实际上是故意自杀嫁祸"、
   "以为的凶器其实不是真正的凶器"等]
```

---

### 【7】故事主题

```
核心主题：[用1-2句话说明这个故事想探讨的人性/情感议题]

示例：
- "关于贫富差距下被压抑的愤怒，以及一念之差的悲剧"
- "关于爱与占有欲的边界，以及善意谎言的代价"
- "关于家族秘密和原生家庭创伤如何影响一代人的选择"

情感高光时刻（主持人揭秘时要重点渲染的情节）：
  [描述1-2个让玩家在真相揭晓时会产生强烈情绪反应的情节细节]
```

---

### 【8】叙事节奏设计（三幕结构）

```
第一幕：开场铺垫（约占游戏时间30%）
  目标：建立角色、制造悬念、让玩家对每个角色产生初步判断
  主要活动：自我介绍 → 第一轮线索搜证 → 第一轮讨论
  设计重点：[这一幕需要特别注意的节奏点]

第二幕：中场推理（约占游戏时间50%）
  目标：深化矛盾、揭露私密关系、让玩家怀疑所有人又觉得谁都有理由
  主要活动：第二轮线索搜证 → 质询环节 → 辩论
  关键翻转节点：[在这一幕应该揭露的重大信息，打破玩家的初步判断]

第三幕：终幕反转（约占游戏时间20%）
  目标：终极线索揭示 → 投票 → 真相大白 → 情感收尾
  终极线索设计：[终极线索是什么，为何它能确定凶手]
  情感落点：[真相揭示后，整个故事想留给玩家的最后感受]
```

---

## 大纲自检清单

完成大纲后，逐条确认：

- [ ] 时间线中每个角色在案发时间段都有明确位置
- [ ] 凶手的三要素（动机、机会、手段）均完整
- [ ] 至少3条相互独立的证据指向凶手
- [ ] 每个非凶手角色都有可疑理由（防止凶手太明显）
- [ ] 所有线索在时间线中有合理的存在解释（不能凭空出现）
- [ ] 场景中的可搜证地点覆盖了所有重要线索的存放位置
- [ ] 叙事反转（如有）在逻辑上成立，不违背已知信息
