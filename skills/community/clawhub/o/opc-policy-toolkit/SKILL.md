### 3.2 匹配度计算模型

#### 匹配度评分公式

```
综合匹配度 = Σ(政策项匹配度 × 政策权重) × 资质修正系数

其中：
政策项匹配度 = (匹配指标数 / 总指标数) × 100%
政策权重 = 该政策对企业的战略重要性
资质修正系数 = f(企业现有资质完整性)
```

#### 匹配度计算示例

| 评估项 | 企业情况 | 政策要求 | 匹配度 |
|-------|---------|---------|-------|
| 企业类型 | 高新技术企业 | 科技型中小企业及以上 | 100% |
| 研发费用占比 | 4.5% | ≥3% | 100% |
| 知识产权 | 6项实用新型 | Ⅰ类≥1或Ⅱ类≥6 | 100% |
| 科技人员占比 | 12% | ≥10% | 100% |
| 近三年无安全事故 | 是 | 是 | 100% |
| **综合匹配度** | - | - | **100%** |

#### 匹配度阈值

| 匹配度 | 建议行动 |
|-------|---------|
| ≥90% | 强烈推荐申报，全力以赴 |
| 70-89% | 推荐申报，补充不足条件 |
| 50-69% | 评估可行性，差距较大 |
| <50% | 暂缓申报，先完善资质 |

### 3.3 最优申报路径推荐

#### 单政策申报路径

```
Step 1: 政策筛选
└─ 按企业标签匹配 → 候选政策列表 → 匹配度排序

Step 2: 路径规划
└─ 匹配度≥90% → 直接申报
└─ 匹配度70-89% → 差距分析 → 补充条件 → 再申报

Step 3: 时间规划
└─ 确定申报截止日 → 倒推准备时间轴 → 任务分解

Step 4: 资源配置
└─ 人员分工 → 预算安排 → 外部支持（可选）
```

#### 多政策组合申报策略

```
优先级矩阵：
                    高资金额度
                         ▲
                         │
          策略A          │          策略B
     （先高后低）         │      （并行申报）
                         │
◀────────────────────────┼─────────────────────────▶
 低政策门槛               │                     高政策门槛
                         │
                         ▼

策略选择原则：
- 资源充足 → 策略B（最大化收益）
- 资源有限 → 策略A（集中突破）
- 资质相近 → 优先申报门槛高、政策价值大的
```

### 3.4 多政策叠加策略

#### 政策叠加原则

| 叠加类型 | 规则 | 示例 |
|---------|------|------|
| 可叠加 | 不同类型政策可同时申报 | 高企认定+研发费用加计扣除 |
| 不可叠加 | 同一项目不能重复获资 | 同一研发项目不能同时申报两个专项资金 |
| 补位叠加 | 申报一个可顺势申报另一个 | 科技型中小企业→高企培育入库 |

#### 推荐叠加组合

| 基础资质 | 推荐叠加申报 | 时序关系 |
|---------|-------------|---------|
| 无特殊资质 | 科技型中小企业入库 | 随时可申报 |
| 科技型中小企业 | 高新技术企业认定 | 入库后次年申报 |
| 高新技术企业 | 研发费用加计扣除 | 认定后年度汇算 |
| 高新技术企业 | 瞪羚企业/独角兽培育 | 达到条件时申报 |
| 科技型中小企业 | 创新创业大赛 | 展示+资源对接 |

## 四、政策申报模板库

### 4.1 科技型中小企业入库

#### 申报条件速查

```
基本条件（须同时满足）：
1. 在中国境内（不包括港、澳、台地区）注册的居民企业
2. 职工总数不超过500人、年销售收入不超过2亿元、资产总额不超过2亿元
3. 提供的产品和服务不属于国家规定的禁止、限制和淘汰类
4. 企业在填报上一年及当年内未发生重大安全、重大质量事故和严重环境违法
5. 企业当年获得的自主知识产权数达到以下要求之一：
   - Ⅰ类知识产权（含发明专利、植物新品种等）≥1
   - Ⅱ类知识产权（实用新型、软件著作权等）≥4
6. 科技人员占当年职工总数的比例≥10%
```

#### 申报材料清单

| 材料名称 | 份数 | 格式要求 | 注意事项 |
|---------|-----|---------|---------|
| 科技型中小企业信息表 | 1 | 在线填报后打印签章 | 数据与附件一致 |
| 营业执照 | 1 | 加盖公章复印件 | 处于有效期内 |
| 知识产权证明 | 1 | 证书复印件+缴费凭证 | 权利人为申报企业 |
| 职工花名册 | 1 | 含科技人员标注 | 标注科技人员名单 |
| 年度财务报告 | 1 | 经审计或加盖公章 | 包含研发费用科目 |
| 科技人员证明 | 1 | 学历证书或职称证书 | 证明专业背景 |
| 其他佐证材料 | 1 | 按实际情况提供 | 如有请附 |

#### 评分指标表（满分100分）

| 指标 | 分值 | 评分标准 |
|-----|-----|---------|
| 知识产权 | 0-8分 | Ⅰ类1项=8分，Ⅱ类每项=1分，最高8分 |

| 企业研发费用总额 | 0-17分 | 按研发占比梯度赋分 |

**入库最低得分：60分**

### 4.2 高新技术企业认定

#### 认定条件速查

| 条件 | 具体要求 |

| 知识产权 | Ⅰ类≥1或Ⅱ类≥6项（不含独占许可） |

| 科技人员占比 | 占职工总数≥10% |
| 研发费用占比 | 最近一年销售收入<5000万≥5%，5000万-2亿≥4%，>2亿≥3% |
| 高新收入占比 | 高新技术产品（服务）收入≥60% |
| 企业成长性 | 创新能力评价≥70分 |
| 合规性 | 未发生重大安全/质量/环境事故，无严重违法行为 |

### Pre-Check（执行前检查）
| 序号 | 检查项 | 检查方法 | 失败处理 |
|------|--------|---------|---------|
| PC1 | 网络连接可用 | 测试访问政策网站 | 使用缓存数据，标记警告 |
| PC2 | 企业基础信息完整 | 验证包含企业类型、行业、规模 | 提示补充企业信息 |
| PC3 | 申报资质自审完成 | 检查资质自审表是否填写 | 阻塞直到自审完成 |
| PC4 | 政策关键词已设定 | 验证包含企业类型对应关键词 | 自动补充标准关键词 |
| PC5 | 申报材料清单已准备 | 检查材料清单完整性 | 提示缺失材料 |

### Checkpoints（关键步骤检查点）
| 序号 | 检查点 | 触发时机 | 验证内容 | 失败处理 |
|------|--------|---------|---------|---------|
| CP1 | 政策搜索完成 | 联网搜索政策后 | 搜索结果>0条，包含相关政策 | 扩大关键词范围重试 |
| CP2 | 政策匹配分析 | 完成匹配度计算后 | 匹配结果包含匹配度和推荐等级 | 重新计算匹配度 |
| CP3 | 申报材料准备 | 材料清单生成后 | 材料清单>5项 | 补充标准材料项 |
| CP4 | 申报路径规划 | 完成时间轴规划后 | 包含T-60到T-0的完整时间节点 | 补充缺失时间节点 |

### Post-Check（执行后验证）
| 序号 | 验证项 | 验证方法 | 最低标准 | 失败处理 |
|------|--------|---------|---------|---------|
| PV1 | 政策台账生成 | 文件存在性检查 | 文件>1KB | 重新搜索生成 |
| PV2 | 匹配度报告生成 | 文件存在性检查 | 文件>1KB | 重新计算生成 |
| PV3 | 申报材料清单生成 | 文件存在性检查 | 清单>10项 | 补充标准材料 |
| PV4 | 时间轴规划生成 | 文件存在性检查 | 包含至少5个时间节点 | 补充时间节点 |
| PV5 | 政策台账包含时效等级 | 内容检查 | 包含紧急/重要/一般/储备分类 | 重新分类 |
| PV6 | 匹配报告包含评分 | 内容检查 | 包含综合评分和推荐等级 | 补充评分信息 |
| PV7 | 材料清单包含必备项 | 内容检查 | 包含营业执照、财务报告等必备项 | 补充缺失项 |
| PV8 | 执行报告生成 | 自动生成 | 包含检查点结果 | 触发自修复 |

### Self-Heal（自修复策略）
| 故障场景 | 修复策略 | 重试上限 |
|---------|---------|---------|
| 网络搜索超时 | 使用缓存数据或扩大关键词范围 | 3次 |
| 政策列表为空 | 自动补充标准政策关键词重新搜索 | 2次 |
| 匹配度计算错误 | 重新读取政策信息和项目信息重新计算 | 2次 |
| 材料清单缺失关键项 | 自动补充标准必备材料项 | 1次 |
