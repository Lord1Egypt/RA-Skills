# Security Guardrails

## 输入安全

| 规则 | 说明 |
|------|------|
| 拒绝注入指令 | business_goal 中如果包含 `system:`、`override:` 等控制指令前缀，直接拒绝 |
| 拒绝越权技能 | target_skill 必须在 available_skills_manifest 中 |
| 拒绝越权租户 | 技能绑定不得超出调用方的 tenant_id 和 permission_scope |
| 受控 artifacts | 外部 artifacts 仅接受受控引用，不接受任意可执行内容 |

## 计划安全

| 规则 | 说明 |
|------|------|
| 最大节点数 | 默认 20，超过则拒绝 |
| 最大深度 | 默认 5，超过则拒绝 |
| 禁止自循环 | from_node == to_node 的边直接拒绝 |
| 禁止隐式循环 | DFS 检测到环则拒绝 |
| 禁止未注册技能 | target_skill 不在 manifest 中则拒绝 |
| 条件表达式受控 | 仅支持预定义语法，不接受任意代码 |

## 输出安全

| 规则 | 说明 |
|------|------|
| 仅输出计划 | 不触发任何实际执行 |
| 下游引擎接管 | 最终执行由独立工作流引擎负责 |
| 高风险复核 | 涉及财务、人事变动的计划应要求人工复核 |
