# QQBot 消息示例

## QQ 标题规则

- QQ 在实际渲染里只有两档真正有感知差异的标题。
- `#` / `##` 表现上可视为同一档大标题。
- `###` 到 `######` 表现上可视为同一档小标题。
- 关于标题层级或结构的规则，要单独写成可见区块，不要埋在普通 bullet 里。

## QQ 列表规则

- 把 QQBot 视为不可靠支持原生嵌套 Markdown 列表。
- 原始内容如果有嵌套 bullet，要改写成 QQ 友好结构。
- 主层级保持普通 bullet。
- 子项必须保持可见缩进，不要直接压平成另一个同级 bullet。
- 最里层的子想法，改写成缩进 emoji 引导句，例如 `👉`。

## 任务完成

```text
# 任务完成

- ✅ 审批发起链路已确认接通
- 📌 按钮入口在 `/processes` 页面，不在流程定义页

### 已确认
- 前端页面、弹窗和提交调用都已接上
- 后端存在 `POST /processes` 接口，并会启动流程引擎

### 风险 / 注意
- ⚠️ 如果本地没做 seed，流程下拉可能为空
- ⚠️ 如果服务没启动，页面上也可能看不到预期结果

### 下一步
- 先确认前后端和数据库服务都在运行
  👉 然后再到 `/processes` 页面实际点一次验证
- 如果下拉为空
  👉 优先检查种子数据有没有初始化
```

## 排障说明

```text
# 问题定位

- 📌 页面里只看到“创建流程”，没有“发起审批”按钮

### 初步判断
- 更像是进到了流程定义页，而不是流程实例页

### 可能原因
- 当前访问的不是 `/processes` 页面
- 菜单入口没有配出来
- 本地代码和检查时看到的版本不一致

### 建议操作
- 先确认当前路由和左侧菜单名称
  👉 如果还是找不到，再去代码里搜 `ProcessesPage` 和 “发起审批”
- 再检查本地是否已经拉到最新前端代码
```

## 反例

```text
- 外层列表
  - 里层列表
    - 最里层继续原生嵌套
```

为什么不适合 QQ：
- QQ 不应被视为可靠支持原生嵌套列表。
- 这种结构在 QQ 里通常会越看越糊，而不是更清楚。

```text
- 标题规则
  - QQ 标题只按两档设计
  - # / ## 一档
  - ### / #### / ##### / ###### 一档
```

为什么不适合 QQ：
- 标题规则本身就是重要的排版指导。
- 如果把它埋在普通 bullet 里，可见性会明显下降。

```text
# 当前结论
## 已确认
### 工具层
- xxx
### 输出层
- xxx
### 下一步
- xxx
```

为什么不适合 QQ：
- `#` 和 `##` 在 QQ 里看起来几乎一样，层级会塌。
- `###` 及以下在 QQ 里也会表现成同一档，下面的区块容易糊成一片。
