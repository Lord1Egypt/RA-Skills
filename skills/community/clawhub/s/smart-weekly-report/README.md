# 周报生成器 · Weekly Report Generator

> 一个基于自然语言的 Prompt-based Skill，帮助各职业角色快速生成结构化周报。无需代码，无需依赖，纯 Markdown + AI 驱动。

---

## ✨ 功能特性

- 🎯 **多职业覆盖**：前端/后端/客户端/测试工程师、运营、销售、HR、会计、市场营销、项目管理等
- 📋 **8 套专业模板**：针对不同职业角色深度定制，开箱即用
- 💡 **洞察优先**：不只是数据罗列，自动提炼趋势、异常和行动建议
- 📊 **对比分析**：引导提供上周数据，自动生成环比分析
- ⚡ **零门槛使用**：纯自然语言输入，2 分钟生成专业周报
- 🔧 **灵活定制**：支持调整风格（正式/简洁）、补充数据、修改措辞

---

## 🚀 快速开始

### 第一步：打开 AI 对话

使用任意支持自定义 Prompt 的 AI 工具（如 ChatGPT、Claude、Kimi、通义千问等）。

### 第二步：加载 Skill

将 [SKILL.md](./SKILL.md) 的全部内容复制粘贴到 AI 对话框中发送，或作为系统提示词（System Prompt）加载。

### 第三步：描述你的工作

用自然语言告诉 AI 你本周的工作情况，AI 将自动生成对应的结构化周报。

## 💬 使用示例

### 最简模式（30 秒）
```
我是前端工程师，本周完成了登录页面重构，修复了 3 个 Bug，下周要做首页性能优化，帮我生成周报。
```

### 标准模式（1 分钟）
```
我是运营，本周完成了双十一活动策划方案，DAU 较上周提升 15%，遇到了供应商资源协调问题，下周要推进活动上线，帮我生成周报。
```

### 详细模式（最完整）
```
请帮我生成周报，信息如下：
- 职业角色：后端工程师
- 汇报对象：技术总监
- 本周完成：用户中心微服务拆分（完成度 80%）、接口性能优化（响应时间从 800ms 降至 200ms）、修复线上 P1 Bug 2 个
- 进行中：订单服务重构，预计下周完成
- 遇到的问题：数据库连接池配置问题导致压测失败，已排查解决
- 下周计划：完成订单服务重构、开始支付模块开发
- 关键数据：接口可用率 99.95%，代码提交 23 次
```

## 🎨 输出风格调整

生成周报后，你可以继续告诉 AI 进行调整：

- `"帮我把语气改得更正式一些"` — 适合向高层汇报
- `"帮我精简一下，只保留最重要的内容"` — 适合快速同步
- `"帮我把数据部分补充得更详细"` — 适合数据驱动型团队
- `"帮我加上英文版本"` — 适合国际化团队
- `"帮我转换成邮件格式"` — 适合通过邮件发送

## 💡 最佳实践

1. **提供上周数据**：有对比才有洞察，尽量提供上周的关键数字
2. **量化你的成果**：用数字说话，例如"提升了 30%"比"有所提升"更有说服力
3. **说明阻碍的影响**：不只是描述问题，说明它影响了什么、需要谁帮助
4. **计划要具体**：给出时间节点和负责人，而不是模糊的"下周继续推进"
5. **定期使用**：每周固定时间生成，形成习惯，积累数据对比

## 🔧 进阶用法

### 作为系统提示词使用
将 `SKILL.md` 内容设置为 AI 的系统提示词（System Prompt），之后每次对话直接输入工作内容即可，无需重复加载。

### 团队统一模板
团队 Leader 可以根据 `SKILL.md` 中的模板，定制团队专属的周报格式，统一汇报标准。

### 历史数据对比
将历史周报保存为本地 Markdown 文件，可在下次生成时**有选择地**附上历史数据，辅助多周趋势分析。

> ⚠️ 在向 AI 提供历史周报前，请先**自行脱敏**，只保留分析所必需的最小信息。

## 🔒 数据安全提示（务必阅读）

本 Skill 只是一个 Prompt 模板，**本身不会收集、存储或传输任何数据**。但你在 AI 对话框中输入的任何内容，都会被你所使用的第三方 AI 服务处理。因此：

- **请勿粘贴公司机密**：包括但不限于未公开的财务数据、客户名单、合同金额、源代码、内部架构等。
- **请勿粘贴个人隐私 / HR 敏感信息**：包括但不限于身份证号、手机号、邮箱、薪资、绩效、调薪记录、员工评价等。
- **请对示例数据做脱敏处理**：用"客户 A / ¥XX"、"张某 / P7"等占位符替换真实姓名和金额。
- **遵守公司合规要求**：仅使用所在组织批准的 AI 工具，并遵守对应的数据分级和使用政策。
- **历史周报复用需谨慎**：复用旧报告前请再次确认其中不含敏感字段，必要时只复用模板结构而非原始内容。

本项目按"原样"提供，不对因用户违规输入敏感数据而产生的任何泄露或合规风险承担责任。

## 📦 发布（维护者）

本项目通过 `clawhub publish` 发布到 ClawHub。为保证版本号一致性和可追溯性，请使用仓库自带的发布脚本：

```bash
# 首次使用请赋予执行权限
chmod +x scripts/clawhub-publish.sh

# 发布指定版本
./scripts/clawhub-publish.sh "$(pwd)" --version=1.0.1
```

脚本会自动完成：

1. 校验 SemVer 格式（`x.y.z`），要求新版本严格大于 [VERSION](./VERSION) 文件中的当前版本；
2. 检查 git 工作区是否干净（可用 `--no-git-check` 跳过）；
3. 同步更新 [VERSION](./VERSION)、[SKILL.md](./SKILL.md) 头部版本元注释、[CHANGELOG.md](./CHANGELOG.md)；
4. 调用 `clawhub publish` 并遵守 [.clawhubignore](./.clawhubignore) —— 其中 `scripts/` 已被忽略，**发布脚本本身不会出现在最终制品中**。

支持标志：

- `--dry-run`：仅预演，不写文件、不调用 `clawhub`。
- `--no-git-check`：跳过 git 干净性校验（仅调试用）。

如需直接使用 `clawhub` 原生命令（等效但不自动升版本）：

```bash
clawhub publish /path/to/report-generator --version=1.0.1
```
