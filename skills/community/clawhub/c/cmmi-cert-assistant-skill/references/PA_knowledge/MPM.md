# MPM — 绩效与度量管理（CMMI V3.0 Core PA）

## PA 基本信息
- **缩写**：MPM
- **英文**：Managing Performance and Measurement
- **中文**：绩效与度量管理
- **所属**：Core PA（核心实践域）

---

## CMMI V3.0 要求要点

### 核心目的
确保项目和组织的过程性能被量化管理，基于历史数据做出预测和决策。

### 关键实践（Specific Practices）
| 实践编号 | 实践内容 | 说明 |
|---------|---------|------|
| MPM 1.1 | 建立和维护度量目标 | 与业务目标对齐 |
| MPM 1.2 | 明确度量项 | 过程度量和产品度量 |
| MPM 1.3 | 建立数据收集存储步骤 | 数据管理计划 |
| MPM 2.1 | 分析度量数据 | 统计分析、趋势分析 |
| MPM 2.2 | 存储和管理度量数据 | 组织级度量库 |
| MPM 2.3 | 报告绩效与度量结果 | 定期报告、仪表板 |
| MPM 3.1 | 建立过程性能基线（PPB） | 统计控制图、基线范围 |
| MPM 3.2 | 建立过程性能模型（PPM） | 预测模型、回归分析 |

### 通用实践（Generic Practices）
- GP 1.1 ~ GP 2.10（完整覆盖）

## 差距检查清单（用于差距分析）

### MPM 1.x — 度量策划
- [ ] 是否有度量目标（与业务目标对齐）？
- [ ] 是否有度量项定义（名称、定义、计算方法）？
- [ ] 是否有数据收集存储计划？

### MPM 2.x — 度量分析
- [ ] 是否有定期的度量数据分析记录？
- [ ] 是否有组织级度量库（数据集中存储）？
- [ ] 是否有定期的绩效报告（仪表板/周报）？

### MPM 3.x — 高成熟度（ML4/5）
- [ ] 是否有过程性能基线（PPB）？
- [ ] 基线是否基于至少 6-8 个同类项目数据？
- [ ] 是否有过程性能模型（PPM）用于预测？

## 评估访谈常见问题（模式D）

### 针对EPG/度量负责人的问题
1. "组织的度量目标是什么？如何与业务目标对齐？"
2. "请展示过程性能基线（PPB），它是如何建立的？"
3. "度量数据是如何收集、存储、分析的？"
4. "如何保证度量数据的质量（准确性、完整性）？"
5. "请展示最近一次基于度量数据的决策记录。"

### 评分要点
- **MPM 1.1-1.3**：度量目标是否明确，度量项是否定义清晰
- **MPM 2.1-2.3**：是否有系统化的数据分析和报告
- **MPM 3.1-3.2**（ML4+）：PPB/PPM 是否统计有效
