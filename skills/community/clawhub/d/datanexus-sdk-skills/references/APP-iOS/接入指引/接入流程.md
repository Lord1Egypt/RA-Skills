# iOS SDK 接入流程

## 1. 前置条件

- 已在 DataNexus 平台注册应用并获取 `actionSetId` 和 `secretKey`
- Xcode 开发环境已就绪，最低支持 **iOS 11.0**
- 当前 SDK 版本：**2.1.4**

## 2. 引入 SDK

### 方式一：CocoaPods（推荐）

在 `Podfile` 中添加：

```ruby
pod 'GDTActionSDK'
```

执行安装：

```bash
pod install
```

### 方式二：手动集成

1. 下载 SDK 包，将 `GDTActionSDK.framework` 拖入工程
2. 在 Build Settings 中确认 Framework Search Paths 已包含 SDK 路径
3. 在 Build Phases → Link Binary With Libraries 中添加依赖库：
   - `AdSupport.framework`
   - `CoreTelephony.framework`
   - `SystemConfiguration.framework`
4. 在 Build Settings → Other Linker Flags 中添加 **`-ObjC`**

## 3. 初始化 SDK

在 `AppDelegate` 的 `didFinishLaunchingWithOptions` 方法中完成初始化，需引入 `GDTAction.h` 和 `GDTAction+convenience.h`：

```objectivec
#import <GDTActionSDK/GDTAction.h>
#import <GDTActionSDK/GDTAction+convenience.h>

- (BOOL)application:(UIApplication *)application didFinishLaunchingWithOptions:(NSDictionary *)launchOptions {
    // 步骤 1：初始化
    [GDTAction init:@"YOUR_ACTION_SET_ID" secretKey:@"YOUR_SECRET_KEY"];
    // 步骤 2：启动（v2.1.0 及以上版本必须调用）
    [GDTAction start];
    return YES;
}
```

> **注意：** 从 v2.1.0 起，必须在 `init` 之后调用 `[GDTAction start]`，否则 SDK 不会正常工作。

## 4. 事件上报

### 4.1 自动采集事件

以下事件由 SDK 自动采集，无需手动上报：

| 事件 | 说明 |
|------|------|
| `TICKET` | 凭证事件 |
| `ENTER_FOREGROUND` | 应用进入前台 |
| `ENTER_BACKGROUND` | 应用进入后台 |

> **注意：** iOS 端 SDK v2.1.4+ 已默认自动采集 `START_APP`，无需手动上报。低版本（< v2.1.4）需手动上报（见下文）。

### 4.2 START_APP 上报

#### 常规场景

在 `applicationDidBecomeActive:` 中调用，上报应用启动事件：

```objectivec
- (void)applicationDidBecomeActive:(UIApplication *)application {
    [GDTAction logAction:@"START_APP" actionParam:nil];
}
```

#### URL Scheme 唤起场景

在 `application:openURL:options:` 中上报，并携带 `open_url` 参数：

```objectivec
- (BOOL)application:(UIApplication *)app openURL:(NSURL *)url options:(NSDictionary<UIApplicationOpenURLOptionsKey,id> *)options {
    [GDTAction logAction:@"START_APP" actionParam:@{@"open_url": url.absoluteString}];
    return YES;
}
```

#### Universal Links 唤起场景

在 `application:continueUserActivity:restorationHandler:` 中上报，并携带 `open_url` 参数：

```objectivec
- (BOOL)application:(UIApplication *)application continueUserActivity:(NSUserActivity *)userActivity restorationHandler:(void (^)(NSArray<id<UIUserActivityRestoring>> *))restorationHandler {
    if ([userActivity.activityType isEqualToString:NSUserActivityTypeBrowsingWeb]) {
        [GDTAction logAction:@"START_APP" actionParam:@{@"open_url": userActivity.webpageURL.absoluteString}];
    }
    return YES;
}
```

### 4.3 预定义行为上报（GDTAction+convenience）

SDK 在 `GDTAction+convenience.h` 分类中提供了一系列预定义行为上报方法：

```objectivec
// 注册
[GDTAction reportRegisterActionWithMethod:@"wechat" isSuccess:YES];

// 登录
[GDTAction reportLoginActionWithMethod:@"phone" isSuccess:YES];

// 绑定社交账号
[GDTAction reportBindSocialAccountActionWithType:@"wechat" isSuccess:YES];

// 完成任务
[GDTAction reportFinishQuestActionWithQuestID:@"quest_001"
                                    questType:@"main"
                                    questName:@"新手引导"
                                   questNumer:@(1)
                                  description:@"完成新手引导任务"
                                    isSuccess:YES];

// 创建角色
[GDTAction reportCreateRoleActionWithRole:@"warrior"];

// 升级
[GDTAction reportUpgradeLevelActionWithLevel:@(10)];

// 浏览内容
[GDTAction reportViewContentActionWithContentType:@"article"
                                      contentName:@"活动详情"
                                        contentID:@"content_001"];

// 加入购物车
[GDTAction reportAddingToCartActionWithContentType:@"goods"
                                       contentName:@"商品A"
                                         contentID:@"sku_001"
                                     contentNumber:@(1)
                                         isSuccess:YES];

// 发起结算
[GDTAction reportCheckoutActionWithContentType:@"goods"
                                   contentName:@"商品A"
                                     contentID:@"sku_001"
                                 contentNumber:@(1)
                             isVirtualCurrency:NO
                           virtualCurrencyType:nil
                              realCurrencyType:@"CNY"
                                     isSuccess:YES];

// 付费
[GDTAction reportPurchaseActionWithContentType:@"goods"
                                   contentName:@"商品A"
                                     contentID:@"sku_001"
                                 contentNumber:@(1)
                                paymentChannel:@"wechat_pay"
                                  realCurrency:@"CNY"
                                currencyAmount:@(9900)
                                     isSuccess:YES];

// 添加支付渠道
[GDTAction reportAddPaymentChannelActionWithChannel:@"wechat_pay" isSuccess:YES];

// 评分
[GDTAction reportRateActionWithRate:@(5)];

// 分享
[GDTAction reportShareActionWithChannel:@"wechat_moments" isSuccess:YES];
```

### 4.4 通用上报方法

对于预定义方法未覆盖的自定义行为，可使用通用上报方法：

```objectivec
[GDTAction logAction:@"CUSTOM_ACTION"
         actionParam:@{@"key1": @"value1", @"key2": @"value2"}];
```

### 4.5 特殊 actionParam 参数

通用上报方法支持以下特殊 key：

| Key | 值 | 说明 |
|-----|-----|------|
| `audience_type` | `1` | 标记为回流用户 |
| `outer_action_id` | 自定义字符串 | 外部去重 ID |
| `length_of_stay` | `1` | 次留标记（用于次日留存统计） |

示例 — 上报回流用户的启动事件：

```objectivec
[GDTAction logAction:@"START_APP" actionParam:@{@"audience_type": @(1)}];
```

## 5. CAID 获取

v2.0.11 及以上版本支持获取 CAID，返回值为 NSDictionary：

```objectivec
NSDictionary *caidInfo = [GDTAction getCaid];
```

## 6. 调试验证

SDK 日志 TAG 为 `gdt_action`，日志级别为 info。在 Xcode 控制台中过滤该 TAG 即可查看 SDK 日志：

- 上报成功：`LogAction success xxxx`
- 上报失败：`LogAction failed xxxx`
