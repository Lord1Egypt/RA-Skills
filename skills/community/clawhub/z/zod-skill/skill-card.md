## Description: <br>
Zod v4 best practices, patterns, and API guidance for schema validation, parsing, error handling, and type inference in TypeScript applications. <br>

This skill is ready for commercial/non-commercial use. <br>

## Publisher: <br>
[anivar](https://clawhub.ai/user/anivar) <br>

### License/Terms of Use: <br>
MIT <br>


## Use Case: <br>
Developers and engineers use this skill when writing, reviewing, or debugging Zod v4 schemas in TypeScript projects. It guides runtime validation at API, form, environment, and external-service boundaries, with emphasis on safe parsing, inferred types, error handling, migration from Zod v3, and schema architecture. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Broad activation wording may cause the skill to influence general schema-validation discussions beyond explicit Zod work. <br>
Mitigation: Use the skill when Zod v4 or TypeScript runtime validation is relevant, and review generated guidance against the project dependency version and local code before applying it. <br>
Risk: The guidance targets Zod v4 and may be wrong for projects that still depend on Zod v3 or another validation library. <br>
Mitigation: Confirm the project's validation library and version before applying migration, parsing, enum, string-format, or error-handling recommendations. <br>
