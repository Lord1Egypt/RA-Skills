# 阶段1：解析招标文件

## 目标
对招标文件（PDF/Word）进行全面解析，提取结构化关键信息，为后续标书编写建立完整的信息基础。

## 输入
- 招标文件（已上传的 PDF 或 Word 文档）
- 用户补充的项目背景信息（如有）

## ⚡ 执行指引（Agent必读）

在开始解析前，按以下策略处理文件：

### 文件读取策略

| 文件类型 | 执行方法 |
|---------|---------|
| **PDF** | 1. 优先使用文件读取工具直接提取文本<br>2. 如果返回内容为空或乱码 → 文件是扫描件（图片），需提示用户："此PDF为扫描件，请提供可检索版本的PDF或手动输入招标关键信息"<br>3. 如果内容含有大量表格 → 逐表提取，注意表格可能在PDF中被分页截断 |
| **Word (.docx)** | 1. 使用文件读取工具提取内容<br>2. 注意保留表格结构（Word表格通常能完整保留）<br>3. 页眉页脚中的关键信息（如项目编号）不要遗漏 |

### 跨平台路径处理

| 平台 | 路径格式 | 注意事项 |
|------|---------|---------|
| **Windows** | `C:\Users\name\招标文件.pdf` 或 `C:/Users/name/招标文件.pdf` | 反斜杠需转义或使用正斜杠；路径含空格时用引号包裹 |
| **Mac** | `/Users/name/招标文件.pdf` | 正斜杠；路径含空格时用引号包裹 |
| **Ubuntu/Linux** | `/home/name/招标文件.pdf` | 正斜杠；区分大小写；注意文件权限 |
| **跨平台通用** | 统一使用正斜杠 `/`，所有操作系统都支持 | Agent 应自动将 `\` 转为 `/`；中文路径用引号包裹 |

> Agent 处理路径时自动执行：检测系统类型 → 使用对应路径分隔符 → 检测文件存在 → 读取内容

### 错误处理

- ❌ **文件读取失败**：提示用户检查文件是否存在、文件是否损坏、路径是否正确
- ❌ **内容为乱码**：可能是编码问题，建议用户另存为UTF-8编码后重试
- ❌ **内容过长超上下文**：优先提取关键章节（资格要求、技术参数、评分标准、投标人须知），跳过非关键部分

### 关键章节定位

招标文件中以下章节最为重要，优先提取：
1. **投标人须知前附表** → 项目基本信息、资格要求、时间节点
2. **技术规格/技术要求** → 技术参数清单
3. **评分标准/评标办法** → 评分维度、分值、评分细则
4. **投标文件格式** → 格式要求、需提供的材料清单
5. **合同条款** → 交付要求、付款方式、质保要求

### 步骤3：信息完整性评估
