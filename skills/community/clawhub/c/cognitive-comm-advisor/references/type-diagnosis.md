# Diagnosing Cognitive Preferences

Use this when the user does not know the other person's cognitive type. The goal is a useful
working read of their likely function stack from workplace-visible behavior, not a perfect
diagnosis.

## Observer Lens Calibration

**Critical**: The user's observations are filtered through their own cognitive functions.
Before interpreting answers, consider:

- **Ti-dom observer** may label Fe behavior as "illogical" or "people-pleasing" when it's
  actually sophisticated social reasoning
- **Fe-dom observer** may label Ti behavior as "cold" or "dismissive" when it's actually
  focused analysis
- **Ne-dom observer** may label Si behavior as "rigid" or "unimaginative" when it's actually
  grounded reliability
- **Si-dom observer** may label Ne behavior as "scattered" or "unrealistic" when it's
  actually generative exploration
- **Te-dom observer** may label Fi behavior as "irrational" or "stubborn" when it's actually
  value-driven conviction
- **Fi-dom observer** may label Te behavior as "soulless" or "controlling" when it's actually
  efficient organization

Once you know the user's own Dom/Aux, mentally adjust for their perceptual bias when
interpreting their behavioral observations of the other person. If a description sounds
like a negative mirror of the user's inferior function, probe further before accepting it
at face value.

## Rules

- Use numbered choices throughout.
- Ask 2-4 questions max unless the user wants deeper typing.
- Ask for degree: `strongly`, `slightly`, or `hard to say`.
- Accept shorthand: `1 strong`, `2 slight`, `hard`, `1?`.
- Prioritize T/F and J/P axes because they most change workplace communication.
- If confidence is partial, proceed with a hedge instead of interrogating.
- Always consider observer lens bias (see above) when interpreting answers.

## User-Facing Setup

```markdown
I can work with a rough read — even 2-3 dimensions gives us a solid communication baseline.
Answer with 1 or 2 + degree (`strongly / slightly / hard to say`).
```

## Priority Questions

### 1. T/F — Decision + Pushback Style

Ask this first unless already obvious.

```markdown
1. When they push back on something, is it more:
   1) "The logic/data doesn't support this" / "this doesn't add up"
   2) "This will affect people, morale, trust, or buy-in"

Degree: strongly / slightly / hard to say
```

Alternate versions if the first wording doesn't fit:

```markdown
1. In meetings, do they seem more focused on:
   1) Getting to the right answer, even if it's blunt
   2) Making sure people feel heard and aligned

Degree: strongly / slightly / hard to say
```

```markdown
1. When giving feedback, are they more:
   1) Direct: "here's the issue, fix it"
   2) Careful/softened: context, reassurance, people impact

Degree: strongly / slightly / hard to say
```

### 2. J/P — Structure + Closure Style

Ask second unless already obvious.

```markdown
2. How do they seem to handle work structure?
   1) Prefer plans, agendas, clear decisions, closure
   2) Prefer flexibility, options, adapting as things unfold

Degree: strongly / slightly / hard to say
```

Alternate versions:

```markdown
2. With deadlines, are they more:
   1) Early/planned, wants progress visible before the last minute
   2) Last-minute but adaptive, makes it work when pressure hits

Degree: strongly / slightly / hard to say
```

```markdown
2. When plans change, do they usually:
   1) Get visibly uncomfortable / want to re-stabilize the plan
   2) Roll with it / improvise without much stress

Degree: strongly / slightly / hard to say
```

### 3. E/I — Processing + Communication Volume

Ask if the interaction plan depends on meeting format, pacing, or live discussion.

```markdown
3. How do they process ideas?
   1) Think out loud; talking helps them clarify
   2) Think privately first; they come back with formed thoughts

Degree: strongly / slightly / hard to say
```

Alternate versions:

```markdown
3. In group settings, are they more likely to:
   1) Talk early/often and shape the discussion live
   2) Listen first, then drop a more considered point

Degree: strongly / slightly / hard to say
```

```markdown
3. After long meetings, do they seem:
   1) Energized / wanting to keep discussing
   2) Drained / wanting quiet time or async follow-up

Degree: strongly / slightly / hard to say
```

### 4. S/N — Information Style

Ask when you need to decide whether to lead with concrete details or big-picture framing.

```markdown
4. When you explain something, do they usually want:
   1) Concrete details: examples, steps, data, precedent
   2) Big picture: pattern, vision, implications, possibilities

Degree: strongly / slightly / hard to say
```

Alternate versions:

```markdown
4. Their questions tend to sound more like:
   1) "How exactly does this work? What are the details?"
   2) "Where is this going? What's the broader idea?"

Degree: strongly / slightly / hard to say
```

```markdown
4. Which proof lands better with them?
   1) Specific data, operational facts, examples from reality
   2) Strategic narrative, future impact, conceptual coherence

Degree: strongly / slightly / hard to say
```

## Recommended Diagnosis Flow

1. Ask T/F and J/P first.
2. If both are clear and the scenario is urgent, proceed with an estimated temperament/style.
3. Add E/I if live meeting pacing matters.
4. Add S/N if message framing matters or if the user is preparing a pitch/proposal.
5. Stop once advice would be useful; do not keep typing for its own sake.

## Interpreting Answers

Map choices as:

1. T/F question:
   - `1` = T signal
   - `2` = F signal
2. J/P question:
   - `1` = J signal
   - `2` = P signal
3. E/I question:
   - `1` = E signal
   - `2` = I signal
4. S/N question:
   - `1` = S signal
   - `2` = N signal

Degree handling:

- `strongly`: clear dimension signal
- `slightly`: weak signal; keep hedge
- `hard to say`: unknown; avoid overclaiming

## Confidence Levels

1. **4/4 dimensions clear**: Give specific function-stack-based advice.
2. **3/4 clear**: Give likely stack advice with one hedge: "If their tert is more X than I'm reading..."
3. **2/4 clear**: Give temperament/style-level advice, e.g. Te-dom, FP-ish, NT cluster. This is still useful.
4. **1/4 or less**: Ask one more high-impact question or give scenario-first, type-light advice.

Always output the estimated function profile (e.g. "likely Te-dom, Ni-aux → ENTJ-ish") rather
than leading with the four-letter shorthand.

## Useful Partial Reads

- **TJ-ish**: direct, structured, outcome/logic oriented. Lead with decision, evidence, next steps.
- **TP-ish**: analytical, autonomy-oriented, flexible. Lead with logic and options; avoid pressure tactics.
- **FJ-ish**: harmony/impact oriented, structured. Lead with shared goals, people impact, clear path.
- **FP-ish**: values/autonomy oriented, flexible. Lead with authenticity, choice, meaning; avoid forcing.
- **NT-ish**: strategy, systems, competence. Lead with the model, tradeoffs, and why it matters.
- **NF-ish**: meaning, people, narrative. Lead with purpose, trust, and human stakes.
- **SJ-ish**: reliability, precedent, process. Lead with facts, risk controls, commitments.
- **SP-ish**: immediacy, practicality, action. Lead with concrete reality and what can be done now.

## Keep It Conversational

Good framing:

- "Quick question to help me read their cognitive style better..."
- "One thing that'd change the advice: when they disagree, which version sounds closer?"
- "Rough is fine — we're looking for communication leverage, not a lab result."
- "Keep in mind: what stands out to you about them partly reflects your own cognitive wiring. That's useful data too."

Avoid:

- Long quizzes
- Abstract type jargon before the user has a practical answer
- Pretending certainty from weak evidence
- Taking the user's negative characterizations of the other person at face value without
  considering observer bias
