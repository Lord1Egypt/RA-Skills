# 服务商模式 - 签名与验签规则

> 服务商模式下签名与商户模式的主要区别

## 与商户模式的核心区别

服务商模式下，签名时需要注意：

| 字段 | 说明 |
|------|------|
| mchid | 使用服务商商户号（sp_mchid） |
| serial_no | 服务商证书序列号 |
| 签名私钥 | 服务商的私钥，不是子商户的 |

## 服务商签名示例

```python
class WechatPayPartner:
    """服务商模式微信支付"""
    
    def __init__(self, sp_mchid, sp_serial_no, sp_private_key, sub_mchid):
        self.sp_mchid = sp_mchid      # 服务商商户号
        self.sp_serial_no = sp_serial_no  # 服务商证书序列号
        self.sp_private_key = sp_private_key  # 服务商私钥
        self.sub_mchid = sub_mchid    # 子商户商户号
    
    def get_authorization(self, method, url_path, body=""):
        timestamp = str(int(time.time()))
        nonce = self.generate_nonce()
        
        # 签名串构造（5行）
        sign_str = f"{method}\n{url_path}\n{timestamp}\n{nonce}\n{body}\n"
        
        signature = self.sign(sign_str, self.sp_private_key)
        
        # Authorization头
        return (f'WECHATPAY2-SHA256-RSA2048 '
                f'mchid="{self.sp_mchid}",'
                f'nonce_str="{nonce}",'
                f'timestamp="{timestamp}",'
                f'serial_no="{self.sp_serial_no}",'
                f'signature="{signature}"')
```

## 回调验签（服务商）

服务商接收回调时：

1. **获取回调头信息**
2. **使用服务商证书验签**
3. **解析报文判断子商户**

```python
def handle_partner_notify(body, headers):
    """处理服务商模式回调"""
    
    # 1. 验签（使用服务商公钥）
    if not verify_signature_with_sp_cert(body, headers):
        raise ValueError("验签失败")
    
    # 2. 解密
    data = json.loads(body)
    resource = data.get("resource", {})
    payment_data = decrypt(resource)
    
    # 3. 获取子商户信息
    sub_mchid = payment_data.get("sub_mchid")
    transaction_id = payment_data.get("transaction_id")
    
    # 4. 根据子商户处理业务
    handle_sub_merchant_payment(sub_mchid, transaction_id)
    
    return {"code": "SUCCESS", "message": "成功"}
```

## 注意事项

1. **必须使用服务商私钥签名**，不能用子商户私钥
2. **回调验签使用服务商公钥**
3. **子商户资金结算**由服务商配置分账规则
4. **子商户订单号**需全局唯一
