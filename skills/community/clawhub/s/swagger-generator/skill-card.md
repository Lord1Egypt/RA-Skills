## Description: <br>
Generates OpenAPI 3.0 specs from endpoint descriptions for API validation, documentation, YAML formatting, schema checking, and contract linting. <br>

This skill is ready for commercial/non-commercial use. <br>

## Publisher: <br>
[xueyetianya](https://clawhub.ai/user/xueyetianya) <br>

### License/Terms of Use: <br>
MIT-0 <br>


## Use Case: <br>
Developers and API engineers use this skill to draft, validate, and document REST API contracts as OpenAPI or Swagger specifications. Review generated or stored content carefully before using it for production API documentation. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: The advertised Swagger/OpenAPI generation capability may not match the included script behavior. <br>
Mitigation: Review the installed script before use and require the maintainer to implement the advertised generator or relabel the skill clearly. <br>
Risk: API descriptions, internal endpoints, or contract details may be retained locally under ~/.local/share/swagger-generator. <br>
Mitigation: Avoid entering proprietary API specs, secrets, or unreleased contract details unless local retention is acceptable; prefer opt-in storage, redaction, deletion, and retention controls. <br>
