## Description: <br>
Run a practical two-worker orchestration workflow with Xiaoma for execution and Xiaoniu for QA, using asynchronous CLI agent routing for medium or large tasks that benefit from execute-then-review structure. <br>

This skill is ready for commercial/non-commercial use. <br>

## Publisher: <br>
[wangyangwjy](https://clawhub.ai/user/wangyangwjy) <br>

### License/Terms of Use: <br>
MIT-0 <br>


## Use Case: <br>
Developers and OpenClaw operators use this skill to delegate substantial local work to an execution agent and then route the result to a QA agent for acceptance. It is best suited for workflows where the main session should remain responsive while local workers handle execution, validation, and status reporting. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Bootstrap can broaden local agent-to-agent access and session visibility beyond the two-worker workflow. <br>
Mitigation: Use the bootstrap helper only when local multi-agent orchestration is intended, then replace wildcard agent allow settings with the intended agents and reduce session visibility where possible. <br>
Risk: AutoFixConfig can mutate local OpenClaw configuration and restart the gateway. <br>
Mitigation: Inspect the published bootstrap helper before use, keep or verify the generated backup, and follow the rollback reference if the changed settings are not appropriate. <br>
