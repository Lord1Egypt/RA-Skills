# Resume Structure Guidelines

## Standard Resume Sections

### 1. Contact Information
**Purpose**: Allow employers to contact you
**Required Info**:
- Full name
- Professional email address
- Phone number
- Location (City, State/Country)
- LinkedIn profile URL (optional but recommended)
- Portfolio/Website URL (for relevant roles)

**Optional** (avoid if not requested):
- Photo (not recommended for US/UK resumes)
- Age, gender, marital status (not recommended)
- Physical address (city/state is enough)

**Formatting**:
- Center-aligned or left-aligned
- Use professional email (name@gmail.com, not cute_email@...)
- Include country code for intl phone numbers (+1 for US, +44 for UK, etc.)

### 2. Professional Summary / Objective
**Purpose**: 2-3 sentence overview of qualifications
**When to Use Summary**:
- 3+ years of experience
- Clear career progression
- Applying for similar role

**When to Use Objective**:
- Entry-level / recent graduate
- Career change
- Returning to workforce

**Summary Formula**:
`[Years of experience] + [Key skills/competencies] + [Major achievement] + [What you offer]`

Example:
```
Senior Software Engineer with 8+ years of experience building scalable web applications.
Proficient in Python, React, and AWS. Led team of 10 engineers to deliver 15+ projects
on time and under budget. Seeking to leverage technical leadership skills to drive
innovation at [Company Name].
```

**Objective Formula**:
`[Education/background] + [Relevant skills] + [What you offer] + [Career goal]`

Example:
```
Recent Computer Science graduate with strong Python and machine learning skills.
Completed 3 internships in AI development. Seeking entry-level software engineering
role to apply academic knowledge and internship experience to real-world projects.
```

### 3. Work Experience
**Purpose**: Demonstrate career progression and achievements
**Format**: Reverse chronological (most recent first)

**Each Entry Should Include**:
- Job title
- Company name
- Location (City, State/Country)
- Dates of employment (Month Year – Month Year or "Present")
- 3-5 bullet points describing achievements (not just duties)

**Bullet Point Formula**:
`Action Verb + Task/Project + Quantified Result`

Examples:
- ✅ "Developed Python scripts to automate data processing, reducing manual effort by 20 hours/week"
- ✅ "Led cross-functional team of 12 to deliver project 2 weeks ahead of schedule"
- ✅ "Reduced server costs by 40% ($50K/year) through cloud migration"
- ❌ "Responsible for data processing" (no action verb, no quantification)
- ❌ "Worked on team projects" (vague, no results)

**What to Include**:
- Promotions / job title changes
- Awards / recognition
- Quantifiable achievements
- Relevant projects
- Leadership experiences

**What to Exclude**:
- Reasons for leaving
- Irrelevant part-time jobs (server, retail) for experienced candidates
- Personal information (why you took the job, etc.)

### 4. Education
**Purpose**: Showcase educational qualifications
**Format**: Reverse chronological (most recent first)

**Each Entry Should Include**:
- Degree (B.S., M.S., Ph.D., MBA, etc.)
- Field of study (Computer Science, Business Administration, etc.)
- Institution name
- Graduation date (Month Year)
- GPA (only if >3.5/4.0 or requested)
- Relevant coursework (only for recent graduates)
- Honors/Awards (Summa Cum Laude, Dean's List, etc.)

**When to Include**:
- All degrees (Associate, Bachelor, Master, Ph.D.)
- Relevant certifications (CPA, CFA, etc.)
- Study abroad (if relevant)

**When to Exclude**:
- High school (if you have a college degree)
- Irrelevant coursework
- Low GPA

### 5. Skills
**Purpose**: Highlight technical and professional skills
**Categories** (group skills for readability):

#### Technical Skills
- Programming Languages: Python, Java, C++, etc.
- Frameworks/Libraries: React, Django, Spring, etc.
- Databases: MySQL, PostgreSQL, MongoDB, etc.
- Tools/Platforms: AWS, Docker, Git, etc.

#### Professional Skills
- Languages (if relevant for the job)
- Certifications
- Methodologies (Agile, Scrum, etc.)

**Formatting Tips**:
- Group into categories
- List proficiencies (Expert, Intermediate, Beginner) if relevant
- Don't include obvious skills ("Microsoft Word", "Email")
- Match JD keywords

### 6. Projects (Optional)
**Purpose**: Showcase practical application of skills
**When to Include**:
- Entry-level (little work experience)
- Career change (projects in new field)
- Technical roles (GitHub/portfolio)

**Each Entry Should Include**:
- Project name
- Brief description (1-2 sentences)
- Technologies used
- Your role/contributions
- Link to GitHub/portfolio (if applicable)
- Quantified results (if applicable)

Example:
```
E-Commerce Platform (2023)
• Built full-stack e-commerce web app using React, Node.js, and MongoDB
• Implemented user authentication, payment processing, and order tracking
• Deployed on AWS with 99.9% uptime over 6 months
• GitHub: github.com/username/ecommerce-project
```

### 7. Certifications (Optional)
**Purpose**: Demonstrate specialized knowledge
**When to Include**:
- Industry-recognized certifications (AWS, CPA, PMP, etc.)
- Relevant to target job
- Recent (within 5 years, unless lifetime cert)

**Each Entry Should Include**:
- Certification name
- Issuing organization
- Date obtained (Month Year)
- Expiration date (if applicable)

### 8. Awards & Honors (Optional)
**Purpose**: Highlight recognition and achievements
**When to Include**:
- Professional awards (Employee of the Month, etc.)
- Academic awards (Scholarships, Dean's List, etc.)
- Industry recognition

### 9. Volunteer Experience (Optional)
**Purpose**: Showcase community involvement and transferable skills
**When to Include**:
- Limited work experience (entry-level)
- Career gap (shows productive use of time)
- Relevant skills for target job

## Optional Sections by Career Level

### Entry-Level (0-3 years)
**Must Have**:
- Contact Information
- Education
- Skills
- Projects (if limited work experience)
- Internships (count as work experience)

**Optional**:
- Summary/Objective
- Volunteer Experience
- Awards & Honors
- Relevant Coursework

**Avoid**:
- High school education
- Irrelevant part-time jobs

### Mid-Level (3-10 years)
**Must Have**:
- Contact Information
- Work Experience
- Skills
- Education

**Optional**:
- Professional Summary
- Projects (if notable)
- Certifications
- Awards & Honors

**Avoid**:
- Too much detail on internships (brief mention okay)
- Irrelevant early-career jobs

### Senior-Level (10+ years)
**Must Have**:
- Contact Information
- Work Experience (focus on last 10-15 years)
- Skills
- Education

**Optional**:
- Executive Summary
- Leadership achievements
- Board positions
- Speaking engagements
- Publications

**Avoid**:
- Too much detail on early career
- Every job you've ever had (focus on relevance)

### Entry-Level

2. Objective (or Summary)
3. Education
4. Skills
5. Projects
6. Internships/Work Experience
7. Awards & Honors
8. Volunteer Experience

### Mid-Level
1. Contact Information
2. Summary
3. Work Experience
4. Skills
5. Education
6. Projects (if notable)
7. Certifications
8. Awards & Honors

### Senior-Level
1. Contact Information
2. Executive Summary
3. Work Experience (last 10-15 years)
4. Leadership Achievements

8. Publications (if applicable)

## Content Guidelines by Section

### Work Experience Bullet Points
**DO**:
- Start with action verb
- Include quantification (numbers, percentages, dollar amounts)
- Focus on achievements, not just duties
- Tailor to JD keywords

**DON'T**:
- Use passive voice ("Was responsible for...")
- Include personal pronouns ("I", "me", "my")
- List duties without achievements
- Use generic phrases ("Hard worker", "Team player")

### Skills Section
**DO**:
- Group into categories
- Match JD keywords
- Include proficiency levels (if relevant)
- List most relevant skills first

**DON'T**:
- Include obvious skills ("Email", "Microsoft Word")
- List too many skills (focus on top 10-15)
- Include skills you don't actually have

### Education Section
**DO**:
- List degrees in reverse chronological order
- Include GPA only if >3.5/4.0
- Mention relevant coursework (for entry-level)
- Include honors/awards

**DON'T**:
- Include high school (if you have a degree)
- List every single course
- Include low GPA

## Length Recommendations

### One Page Resume
**For**:
- Entry-level (0-3 years)
- Career changers (early stage)
- Limited work experience

**How to Fit**:
- Focus on most relevant experience
- Remove irrelevant jobs (server, retail)
- Use concise bullet points
- Adjust margins (0.5 inch minimum)
- Adjust font size (10-12pt)

### Two Page Resume
**For**:
- Mid-level (3-10 years)
- Senior-level (10+ years)
- Extensive relevant experience
- Academic/Research positions

**How to Structure**:
- Page 1: Most recent/relevant experience
- Page 2: Earlier career, additional details
- Use consistent formatting
- Include name and "Page 2" header on second page

### More Than Two Pages
**Generally NOT Recommended** (except for academic CVs, federal resumes, etc.)

## Formatting Best Practices

### Fonts
**Recommended**:
- Arial (sans-serif)
- Calibri (sans-serif)
- Helvetica (sans-serif)
- Times New Roman (serif)
- Garamond (serif)

**Avoid**:
- Decorative fonts
- Script fonts
- Comic Sans
- Fonts that are hard to read

### Font Sizes
- Name: 18-24pt
- Section Headings: 14-16pt (bold)
- Job Titles: 12-14pt (bold)
- Body Text: 10-12pt
- Dates: 10-11pt

### Spacing
- Margins: 0.5-1 inch
- Line spacing: 1.0-1.15
- Space between sections: 12-15pt
- Space between bullet points: 3-6pt

### Alignment
- Left-align most text
- Center-align name/contact info (optional)
- Right-align dates (optional, but creates nice visual balance)

### Bullet Points
- Use standard bullets (•, -, *)
- Keep bullet points to 1-2 lines
- Use parallel structure (all start with action verb)
- Limit to 3-5 points per job

## ATS-Friendly Structure

### Section Headings
Use standard headings:
- "Work Experience" (not "Where I've Worked")
- "Education" (not "Schooling")
- "Skills" (not "Things I'm Good At")
- "Projects" (not "Cool Stuff I Built")

### Avoid
- Tables (confuse ATS parsing)
- Text boxes (content often ignored)
- Headers/footers (often skipped)
- Columns (confuse reading order)
- Graphics/icons (not parseable)

### File Format
- Preferred: .docx (Word format)
- Acceptable: .pdf (text-based, not scanned)
- Avoid: .pdf (scanned/image), .pages, .txt

## Industry-Specific Structures

### Technology/Software Development
Emphasize:
- Technical Skills (prominent placement)
- Projects (GitHub/portfolio links)
- Programming languages, frameworks, tools
- Work Experience (focus on technical achievements)

### Data Science/Analytics
Emphasize:
- Technical Skills (Python, R, SQL, etc.)
- Projects (data analysis, machine learning models)
- Tools (Tableau, Pandas, Scikit-learn, etc.)
- Work Experience (focus on data-driven results)

### Product Management
Emphasize:
- Work Experience (cross-functional collaboration, product launches)
- Skills (Agile, roadmap planning, user research)
- Achievements (product metrics, user growth)
- Certifications (PMP, CSPO, etc.)

### Marketing/Sales
Emphasize:
- Work Experience (campaign results, sales numbers)
- Skills (digital marketing, CRM, analytics tools)
- Achievements (ROI, conversion rates, revenue growth)
- Certifications (HubSpot, Google Ads, etc.)

### Finance/Accounting
Emphasize:
- Work Experience (financial analysis, budgeting, forecasting)
- Certifications (CPA, CFA, MBA)
- Skills (Excel, SAP, financial modeling)
- Achievements (cost savings, revenue growth, audit results)

## Checklist Before Submitting

- [ ] Contact information is current and professional
- [ ] Summary/objective is tailored to target role
- [ ] Work experience uses action verbs and quantification
- [ ] Skills match JD keywords
- [ ] Education is accurate and relevant
- [ ] No typos or grammatical errors
- [ ] Length is appropriate for experience level
- [ ] Formatting is consistent and ATS-friendly
- [ ] All dates are accurate
- [ ] No personal information that could lead to discrimination
- [ ] File name is professional (FirstName_LastName_Resume.pdf)
