## Description: <br>
Guide a contributor through the full process of adding a new service to the awesome-agent-native-services catalog: verifying criteria, checking for URL Onboarding, opening an issue, writing the service file, and submitting a PR. <br>

This skill is ready for commercial/non-commercial use. <br>

## Publisher: <br>
[haoruilee](https://clawhub.ai/user/haoruilee) <br>

### License/Terms of Use: <br>
MIT-0 <br>


## Use Case: <br>
External contributors and developers use this skill to prepare a well-evidenced service addition for the awesome-agent-native-services catalog, including eligibility checks, service file drafting, index updates, and PR preparation. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: The skill can help edit a local Git repository and prepare public GitHub issue or PR content. <br>
Mitigation: Review generated diffs, issue text, PR content, and shell or git commands before submission. <br>
Risk: The workflow depends on external onboarding pages and service documentation that may be incomplete, outdated, or misleading. <br>
Mitigation: Check official service sources and verify cited evidence, links, and onboarding instructions before adding the catalog entry. <br>
