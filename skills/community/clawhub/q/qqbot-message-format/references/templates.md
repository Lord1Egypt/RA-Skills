# QQBot 消息模板

## QQ 标题规则

- 把 QQ 视为只有两档真正有用的标题层级。
- `#` / `##` 视为同一档较大的标题。
- `###` / `####` / `#####` / `######` 视为同一档较小的标题。
- 像这种“标题大小或结构本身的规则”，要单独做成可见区块，不要埋在普通 bullet 里面。

## QQ 列表规则

- 把 QQBot 视为不可靠支持原生嵌套 Markdown 列表。
- 如果原始内容有嵌套结构，要改写成批准的 QQ 友好格式，不要直接保留原生嵌套。
- 主层级使用扁平 bullet。
- 子项要保持肉眼可见的缩进，不要直接压成同级普通 bullet。
- 最内层子想法可改写成缩进的 emoji 引导句，例如 `👉`。

## 模板选择

- `short-answer`：用于简单 yes/no、快速澄清或短建议。
- `completion-update`：用于任务完成、检查完成或后台工作结束。
- `troubleshooting`：用于问题诊断、根因解释和操作建议。
- 遇到原本是嵌套列表的内容时，要先改写成 QQ 友好结构。

## short-answer

```text
# 结论

- ✅ {直接答案}
- ⚠️ {如果需要，补一条注意事项}

### 说明
- {只保留 1-2 条短 bullet}
```

## completion-update

```text
# 任务完成

- ✅ {完成了什么}
- 📌 {当前结果}

### 已确认
- {关键信息}
- {关键信息}

### 风险 / 注意
- ⚠️ {风险或依赖项}

### 下一步
- {下一步动作}
  👉 {细节或跟进点}
- {下一步动作}
```

## troubleshooting

```text
# 问题定位

- 📌 {用户当前看到的现象，或主判断}

### 初步判断
- {主要判断}
- {必要时补一条非结论性信息}

### 可能原因
- {原因}
- {原因}
- {原因}

### 建议操作
- {操作建议}
  👉 {下一步检查项或分支条件}
- {操作建议}
```

## 编辑规则

- 用具体事实替换占位符。
- 空区块直接删掉，不要保留空壳。
- 把答案或当前结果放到第一屏，不要先铺垫。
- 不要因为 Markdown 支持更多层级，就硬造第三层、第四层标题。
- 如果某块内容只是从属信息，就留在小标题层，或直接改成引导句。
- 如果消息很短，压缩成一个标题加 2-3 个 bullet 即可。
- 如果原始结构是嵌套的，要改写成 QQ 安全格式，不要保留原生嵌套 bullet。
- 加粗优先只标关键短语，不要整行全粗。
- 不要默认用 `▪️` 或 `└` 作为嵌套标记。
- 如果你在描述一条排版规则、标题规则或结构规则，要把它独立成可见区块，不要藏进通用 bullet 列表。
