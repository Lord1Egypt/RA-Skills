# 负面早期预警库

> 用途：在创意上线前、评测解禁后、评论区导入后，尽早识别可能发酵成公关风险的负面苗头。
> 原则：优先识别“可被放大的真实问题”，不要把所有负面都归为黑粉。

机器可读规则库见 [negative-signal-rules.json](negative-signal-rules.json)。更新关键词、等级或建议动作时，优先更新 JSON；本文档负责解释方法和使用纪律。

样本校准集见 [../evals/negative-signal-samples.md](../evals/negative-signal-samples.md)。新增规则前，先检查是否会误伤 `document` 模式样本。

## 预警分级

| 等级 | 触发条件 | 建议动作 |
|------|----------|----------|
| 低 | 单点吐槽、无数据支撑、未重复出现 | 记录观察，暂不调整主方案 |
| 中 | 同类质疑重复出现，或被参数党/垂直用户指出具体问题 | 调整话术，补充证据，准备回应口径 |
| 高 | 涉及数据造假、产品缺陷、价值观冒犯、KOL背叛、控评、合规问题 | 暂停扩散，复核事实，统一口径，必要时下架物料 |

## 七类高优先级负面信号

| 信号 | 典型表达 | 为什么危险 | 处理原则 |
|------|----------|------------|----------|
| 数据打脸 | `数据哪来的` `实测呢` `虚标` `PPT参数` | 参数党容易把单点质疑做成对比图或长帖 | 立即补来源、测试条件、同源对比 |
| 产品缺陷 | `发热` `断连` `漏音` `续航崩` `品控` `翻车` | 从传播问题转为产品问题，扩散周期长 | 区分个例/批量，补测试边界和售后口径 |
| 价格背刺 | `首发冤种` `背刺` `等降价` `涨价` `国补` | 大促期极易触发普通用户共鸣 | 标注渠道、日期、配置、补贴条件 |
| 信任崩塌 | `恰饭` `软广` `水军` `控评` `删评` | 用户不再讨论产品，转向质疑品牌和KOL | 公开利益关系，避免硬辩，减少水军感话术 |
| 价值观冒犯 | `不尊重` `恶心` `冒犯` `价值观` `低俗` | 会跳出数码圈，进入公共议题 | 下架/改稿优先于解释创作初衷 |
| KOL背叛 | `给黑粉送钱` `骂用户的人也合作` `背叛用户` | 核心粉丝情绪强，容易组织抵制 | 审查历史言论，必要时切割合作 |
| AI空话 | `套壳` `鸡肋` `智障` `PPT功能` `没用` | AI卖点无法验证时会快速变成黑梗 | 落到具体任务、结果和不可用边界 |

## 品类专属早期信号

### 手机

- 性能：`掉帧` `锁帧` `发热` `功耗` `调度保守`
- 影像：`涂抹` `偏色` `鬼影` `拍月亮` `算法味`
- 续航：`尿崩` `虚标` `充电发热`
- 系统：`广告` `杀后台` `断流` `卡顿`

### 耳机

- 音质：`糊` `闷` `刺耳` `底噪` `听个响`
- 降噪：`压耳` `风噪` `人声压不住`
- 佩戴：`夹耳` `掉` `疼` `漏音`
- 连接：`断连` `延迟` `左右不同步`

### 笔记本

- 性能：`残血` `功耗墙` `降频` `跑不满`
- 屏幕：`三低屏` `偏色` `频闪` `亮度不够`
- 散热：`烫手` `风扇吵` `武装直升机`
- 配置：`QLC` `马甲U` `接口少` `缩水`

### 穿戴设备

- 健康：`测不准` `医疗器械吗` `误导健康`
- 续航：`一天一充` `虚标`
- 运动：`GPS飘` `心率飘`
- 生态：`只适合同品牌手机`

### 智能家居

- 清洁：`越拖越脏` `毛发缠绕` `边角扫不到`
- 避障：`撞家具` `识别不了线`
- 安装：`自动上下水不适合我家`
- 隐私：`摄像头隐私` `地图数据`

## 负面识别优先级

优先处理同时满足以下任意两项的负面：

- 来自参数党、专业评测、老用户、核心粉丝，优先级高于纯情绪号。
- 具体到产品型号、配置、场景、时间、测试条件。
- 能被截图、对比图、实测视频复现。
- 触及用户利益：价格、售后、品控、虚假宣传、隐私。
- 触及公共价值观：亲情、性别、地域、职业、未成年人。

## 处理动作库

| 信号 | 立即动作 | 不要做 |
|------|----------|--------|
| 数据打脸 | 补来源、测试条件、同源对比；必要时撤回绝对化结论 | 不要用“实验室数据仅供参考”糊弄 |
| 产品缺陷 | 区分个例/批量，收集型号、批次、环境、复现步骤 | 不要先说“极少数情况” |
| 价格背刺 | 公开渠道、日期、配置、补贴条件；给首发用户权益解释 | 不要把所有差价都说成平台行为 |
| 信任崩塌 | 标注合作关系，减少控评痕迹，统一非水军式回应 | 不要堆正面评论压负面 |
| 价值观冒犯 | 下架或改稿，承认具体问题，说明后续审核机制 | 不要只解释创意初衷 |
| KOL背叛 | 复核历史言论，评估切割合作或降级合作形式 | 不要用粉丝量解释为什么合作 |
| AI空话 | 改成具体任务、用户动作、结果和限制条件 | 不要继续堆“赋能/重塑/生态” |

## 处理时序

| 时间窗口 | 目标 | 动作 |
|----------|------|------|
| 0-2小时 | 判断是否真实问题 | 收集原评论、截图、链接、产品型号、用户身份、是否重复出现 |
| 2-6小时 | 控制扩散 | 暂停争议物料、改口径、准备客服/评论区回应方向 |
| 6-24小时 | 修正认知 | 发布修正信息或补充说明，避免反复解释创作初衷 |
| 24小时后 | 复盘入库 | 更新风险模式、禁用话术、KOL记录、品类索引 |

## 输出格式

```
负面早期预警

| 信号 | 等级 | 依据 | 建议动作 |
|------|------|------|----------|
| {信号类型} | 低/中/高 | {评论/数据/案例依据} | {动作} |

优先处理:
1. {最可能发酵的负面}
2. {需要补证据或改话术的点}
```

## 处理纪律

- 不把“批评产品”自动归类为黑粉。
- 不用水军式正面评论覆盖真实负面。
- 不在事实未核清前承诺“绝无问题”。
- 不把历史案例写成品牌永久标签。
- 先处理可验证问题，再处理情绪问题；价值观冒犯除外，价值观问题先下架/改稿。
