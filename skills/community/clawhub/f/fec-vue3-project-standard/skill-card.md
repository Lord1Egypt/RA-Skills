## Description: <br>
Use when designing or reviewing Vue 3 + TypeScript project structure, SFC/component boundaries, composables organization, route composition, Pinia ownership, API/error/styling defaults, directives, or repository-wide Vue conventions. <br>

This skill is ready for commercial/non-commercial use. <br>

### License/Terms of Use: <br>
MIT-0 <br>


## Use Case: <br>
Developers and engineers use this skill to design or review Vue 3 and TypeScript application structure, component boundaries, composables, routing, Pinia ownership, API conventions, styling defaults, and test coverage. It is suited to repository-wide Vue architecture guidance and should hand off specialized security, accessibility, testing, forms, data-fetching, animation, or performance work to narrower skills. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Auth-token handling examples may be copied into production without enough security review. <br>
Mitigation: Review authentication and token behavior before production use, and use a specialist security-review skill for security-sensitive decisions. <br>
Risk: General Vue architecture conventions may be applied to specialized forms, data-fetching, accessibility, testing, animation, or performance work where narrower guidance is needed. <br>
Mitigation: Use the named specialist skills for those decisions and keep this skill focused on project structure, component boundaries, composables, routing, Pinia ownership, and API conventions. <br>
