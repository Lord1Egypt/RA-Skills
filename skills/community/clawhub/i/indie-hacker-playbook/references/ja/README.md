# インディーハッカー成長戦略 — Build in Public + コミュニティ駆動ディストリビューション

VC、チーム、広告予算なしで成長したいソロファウンダーのためのプレイブック。

## Build in Public（公開で作る）

何を共有し、どのくらいの頻度で、どのプラットフォームが透明性を評価するか：

- **収益マイルストーン**: 月次MRRアップデートが最もエンゲージメントとアカウンタビリティを生む
- **意思決定の舞台裏**: 技術スタック、価格設定、機能選択の理由を共有
- **失敗とピボット**: 本物であることは、磨かれたサクセスストーリーより早く信頼を築く
- **投稿頻度**: 最低週3〜5回; ローンチ期間中は毎日更新
- **BIPを評価するプラットフォーム**: X/Twitter（スレッド）、Indie Hackers（マイルストーン）、LinkedIn（ジャーニー投稿）

## オーディエンス・ファースト

売るプロダクトがある前にフォロワーを増やす：

- **知識を教える**: プロダクト構築前に、自分の専門領域で培った知見を共有
- **旅を記録する**: 売るものができる前から投稿を始める
- **真摯にエンゲージ**: 他者にリプライし、インサイト付きで引用RTし、会話に参加
- **継続は複利**: 100日間の投稿 > バズった10ツイート
- **メール > ソーシャル**: あらゆる機会にフォロワーをメール購読者に転換

## ニュースレターエンジン

有料プロモーションゼロで0から1,000購読者へ：

- **リードマグネット**: メール登録でゲートされた高価値リソース（テンプレート、チェックリスト、データベース）
- **クロスプロモーション**: 隣接ニッチの3〜5ニュースレターと相互紹介
- **ソーシャル→メールファネル**: バズった投稿の最後に「もっと読みたい方は購読を」
- **ウェルカムシーケンス**: 価値を届け関係を構築する3通のオンボーディングメール
- **リファラルプログラム**: SparkLoop等 — 友人を連れてきた購読者に報酬

## コミュニティ駆動ディストリビューション

インディーコミュニティ（IH、Twitter、HN）を効果的に活用：

- **Indie Hackers**: 詳細なマイルストーン投稿、コメントでエンゲージ、他者にフィードバック
- **Hacker News**: ローンチ時にShow HN投稿; 年間を通じて関連スレッドにコメント
- **Product Hunt**: ローンチ前からハンターとの関係構築; コミュニティに定期的に参加
- **Reddit**: ニッチなサブレディットを見つけ、自己宣伝前に数ヶ月間は価値提供ファーストで参加
- **Discord/Slackグループ**: まず頼れるエキスパートになり、関連する時に自然にプロダクトに言及

## マイクロSaaSの経済学

価格設定、マージン、ライフスタイル収入の目標：

- **目標メトリクス**: ソロで持続可能なMRR $5K〜$30K（居住地による）
- **価格設定の心理学**: 最低$29/月からスタート; チャーンが増えるまで四半期ごとに値上げ
- **マージン**: グロスマージン85%+を目標（高コストなインフラを避ける）
- **LTV/CAC**: オーガニックのみ = CAC ≈ ゼロ; 控えめなLTVでも高い収益性
- **イグジットの選択肢**: Build in public → ARR 3〜5倍で複数の買収オファー
