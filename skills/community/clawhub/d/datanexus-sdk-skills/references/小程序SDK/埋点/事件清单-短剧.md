# 短剧小程序 — 必报与选报事件

> 来源：[腾讯广告小程序 SDK 开发指引](https://doc.weixin.qq.com/doc/w3_AJUA1gbWACgBSVyDeMgQ3irunfYyt) — 短剧行为章节
>
> ⚠️ `REGISTER` 的触发判断**必须由服务端完成**，详见 [数据正确性红线](../../通用/埋点/行为类型枚举表.md#️-数据正确性红线务必先看)。

## 必报事件

| action_type | 中文名 | 说明 |
|------------|--------|------|
| `START_APP` | 启动应用 | 用于广告归因、广告效果优化。开启 SDK 自动采集后可不再手动上报 |
| `REGISTER` | 注册 | 用于广告归因、广告效果优化。用户首次打开小程序时上报。⚠️ 服务端判断 |
| `VIEW_DRAMA` | 播放短剧 | 用于广告归因、广告效果优化。用户打开剧目进入播放页，播放每集都需上报。需过滤"暂停再播放"场景（可通过剧集索引变化判断） |
| `SELECT_RECHARGE_LEVEL` | 选择充值档位 | 用于广告归因、广告效果优化。选择充值档位后拉起微信支付页面或关闭充值对话框时上报。需传 `drama_id`、`drama_name`、`episodes` |
| `UNLOCK_PAID_EPISODES` | 解锁免费剧集 | 用于广告归因、广告效果优化。用户观看广告解锁免费剧集，广告曝光即上报（不用等到广告结束） |
| `PURCHASE` | 付费 | 用于广告归因、广告效果优化。在 `wx.requestVirtualPayment` / `wx.requestPayment` 成功回调中上报。需传 `value`、`drama_id`、`drama_name`、`episodes` |
| `LIKE` | 点赞 | 用于广告归因、广告效果优化。用户点赞时上报 |
| `CHECK_IN` | 签到 | 用于广告归因、广告效果优化。用户点击签到按钮时上报 |
| `ADD_TO_WISHLIST` | 收藏小程序 | 用于广告归因、广告效果优化。用户收藏小程序或添加到我的小程序时上报。⚠️ SDK 自动采集仅用于数据校验，**客户仍需手动上报** |
| `SHARE` | 分享小程序 | 用于广告归因、广告效果优化。需区分【转发给朋友】和【分享到朋友圈】。使用播放器插件时需单独监听插件的分享回调。⚠️ SDK 自动采集仅用于数据校验，**客户仍需手动上报** |
| `EXIT_DRAMA` | 结束观看 | 用于广告归因、广告效果优化。用户点击返回退出短剧观看页面或点击右上角退出小程序时上报（可监听 UNLOAD_OR_HIDE 事件） |

## 专用上报方法

| 方法 | 参数 | 对应 action_type |
|------|------|-----------------|
| `sdk.onAppStart()` | 无 | START_APP |
| `sdk.onAppQuit()` | 无 | APP_QUIT |
| `sdk.onPageShow()` | 无 | PAGE_VIEW |
| `sdk.onPageHide()` | 无 | PAGE_LEAVE |

> 小程序 SDK 专用方法较少，大部分行为需使用通用方法 `sdk.track(action_type, action_param)` 上报。
