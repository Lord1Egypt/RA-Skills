# SnapTrade Trading Skill — Setup Guide

This skill lets OpenClaw trade and retrieve account data through the
[SnapTrade API](https://docs.snaptrade.com), which connects to your brokerage
(IBKR, Alpaca, Questrade, and more).

Before using this skill you need four credentials. This guide walks you through
getting them and putting them in the right place.

---

## Step 1 — Create a SnapTrade Account and Get Your API Key

Your `SNAPTRADE_CLIENT_ID` and `SNAPTRADE_CONSUMER_KEY` come from the SnapTrade
partner dashboard. These identify your application to the API.

1. Go to [https://dashboard.snaptrade.com](https://dashboard.snaptrade.com) and create a free account
2. Verify your email
3. Go to **API Key** in the dashboard and generate your free API key
4. You will see two values — copy both:
   - **Client ID** → this is your `SNAPTRADE_CLIENT_ID`
   - **Consumer Key** → this is your `SNAPTRADE_CONSUMER_KEY`

> ⚠️ Your `consumerKey` is sensitive. Never share it, commit it to git, or put
> it in any logs. If it is compromised, regenerate it from the dashboard.

Full guide: https://docs.snaptrade.com/docs/getting-started

---

## Step 2 — Register a SnapTrade User and Get Your User Credentials

SnapTrade uses a per-user credentials system. Even if you are the only person
using this skill, you still need to register a "user" under your API key.
This gives you a `SNAPTRADE_USER_ID` and `SNAPTRADE_USER_SECRET`.

You can do this with a quick Python script once your API key is set up:

```python
from snaptrade_client import SnapTrade

snaptrade = SnapTrade(
    consumer_key="YOUR_CONSUMER_KEY",
    client_id="YOUR_CLIENT_ID",
)

# Pick any unique string as your user ID — do NOT use an email address
response = snaptrade.authentication.register_snap_trade_user(
    body={"userId": "my-openclaw-user"}
)

print("userId:", response.body["userId"])
print("userSecret:", response.body["userSecret"])
```

- **userId** → this is your `SNAPTRADE_USER_ID` (you chose this string)
- **userSecret** → this is your `SNAPTRADE_USER_SECRET` (randomly generated by SnapTrade)

> ⚠️ Save the `userSecret` immediately — SnapTrade will not show it again.
> If you lose it, rotate it using the dashboard or the rotate endpoint.

API reference: https://docs.snaptrade.com/reference/Authentication/Authentication_registerSnapTradeUser

---

## Step 3 — Connect Your Brokerage Account

After registering a user, you need to connect a brokerage account to it.
This is done through the SnapTrade Connection Portal — a secure login flow
hosted by SnapTrade.

```python
response = snaptrade.authentication.login_snap_trade_user(
    user_id="my-openclaw-user",
    user_secret="YOUR_USER_SECRET",
    body={"connectionType": "trade"},  # "trade" enables order placement
)

print("Open this URL in your browser:")
print(response.body["redirectURI"])
```

Open the URL, log in to your brokerage, and authorize SnapTrade. Once done,
your brokerage account is connected and ready to use.

> 💡 For testing without a live account, select **Alpaca Paper** in the
> Connection Portal. This gives you a free simulated trading account with no
> real money at risk.

---

## Step 4 — Set Your Credentials

Once you have all four values, add them to a `.env` file in the same directory
where you run OpenClaw:

```
SNAPTRADE_CLIENT_ID=your_client_id_here
SNAPTRADE_CONSUMER_KEY=your_consumer_key_here
SNAPTRADE_USER_ID=my-openclaw-user
SNAPTRADE_USER_SECRET=your_user_secret_here
```

Or export them directly in your shell session:

```bash
export SNAPTRADE_CLIENT_ID=your_client_id_here
export SNAPTRADE_CONSUMER_KEY=your_consumer_key_here
export SNAPTRADE_USER_ID=my-openclaw-user
export SNAPTRADE_USER_SECRET=your_user_secret_here
```

> ⚠️ Never commit your `.env` file to git. Add it to `.gitignore`.

---

## Step 5 — Run Setup

From the skill directory, run:

```bash
bash scripts/setup.sh
```

This creates a virtual environment and installs the pinned SnapTrade SDK.
Then activate it:

```bash
source .venv-snaptrade/bin/activate
```

You are now ready to use the skill.

---

## Credential Summary

| Variable | Where to get it | Sensitive |
|---|---|---|
| `SNAPTRADE_CLIENT_ID` | SnapTrade Dashboard → API Key | Yes |
| `SNAPTRADE_CONSUMER_KEY` | SnapTrade Dashboard → API Key | Yes — never share |
| `SNAPTRADE_USER_ID` | You choose when registering a user | No |
| `SNAPTRADE_USER_SECRET` | Returned when registering a user | Yes — save immediately |
