# 三层运转体系详细规则

## 第一层 · 每日发现

**目标：帮助用户提升认知，不只是转发信息。**

### 执行流程

```
1. 读取记忆中的研究院配置（方向、信源、大师轮值表、用户反馈规则、领域专属信源 skill）
2. 联网采集（多源并行）：
   · web_search 中文关键词 ×3 组
   · web_search 英文关键词 ×2 组
   · 对高价值链接用 web_fetch 抓全文
   · 信源库中有指定URL的，优先抓取
   · ★ 如有绑定的领域专属信源 skill → 调用其API/方法获取领域数据
     - 调用失败/无返回 → 静默降级，仅用 web_search 结果
     - 调用成功 → 与 web_search 结果合并去重后进入质检
3. 排重检查（读取推送历史，过滤重复内容）
4. 内容质检（见质检机制，含反馈规则过滤）
5. 为每条入选内容生成：
   · 摘要（必须过摘要质检）
   · 院长点评（这对用户意味着什么，基于用户memory的关联分析）
   · 当日轮值大师点评（有绑定大师时）
6. 院长精选（选1条最值得关注的高亮）
7. 生成简报（两种格式同时生成）
8. 推送到所有已配置渠道
9. 沉淀到长期存储空间（完整版）
10. 更新推送历史
11. 自进化（强制执行）→ 详见 references/self-evolving.md
```

### 三层质检机制

```
第一层：内容质检
  评分 = 重要性×0.4 + 相关性×0.35 + 时效性×0.25
  ≥3.5 入选，<3.5 淘汰
  
  自进化规则：
  · 读取记忆中的反馈规则（用户曾说"不想看XX类内容"）
  · 匹配反馈规则的内容，即使评分达标也降权或淘汰
  · 用户曾点赞/展开阅读的类型，加权

第二层：摘要质检
  □ 包含至少1个具体数据/事实/人名/机构名？
  □ 包含"这件事为什么重要"的判断？
  □ 与标题有信息增量（不能是标题换说法）？
  □ 2-3句话，≤100字？
  不通过 → 重写直到通过

第三层：关联质检
  □ 这条内容与用户的实际工作/生活有什么关联？
  □ 读取用户 memory，找到关联点
  □ 附带"对你的意义"说明（1-2句）
  找不到关联 → 降低排序优先级，但不淘汰
```

### 简报格式

**简报必须包含概览区：**

```
概览区（必须）：
  · 今日采集 [X] 条 → 质检通过 [Y] 条 → 入选 [Z] 条
  · 今日轮值大师：[大师A、大师B]
  · 本周累计发现：[N] 条

每条内容（必须）：
  · 来源（信源名称 + 原文链接）
  · 评分（数字 + 颜色标签）
  · 标题
  · 详细摘要（有干货的2-3句）
  · 院长点评（结合用户实际情况的判断）
  · 大师点评（当日轮值大师，各2-3句，标注是谁说的）
  · 对你的意义（基于用户memory的关联分析）
  · 原文链接按钮

底部（必须）：
  · 沉淀位置链接
  · 头脑实验候选（评分≥4.5的条目）
  · "回复反馈给院长"
```

#### IM 渠道（企微等）：摘要版
```
摘要版只包含：概览 + 每条的标题/评分/一句话摘要 + 原文链接
末尾附：完整版在 [沉淀位置链接]
```

#### 邮件渠道：完整版（HTML 格式）
```
完整版包含上述所有字段，用卡片式布局，大师点评用紫色背景区分
HTML 模板包含概览区和"对你的意义"字段
```

#### 沉淀存储：完整版（Markdown）
```
永久存档，包含所有字段 + 元数据（评分、来源、大师、推送状态）
```

### 原文链接（强制）

每条推荐必须有可点击的原文链接。没有原文链接的内容不进简报。

## 第二层 · 头脑实验

**目标：AI基于用户实际情况，主动发起深度探讨。用户不回复 AI 也可以做。**

```
触发方式：
  · 定时任务自动触发（每周/每3天）
  · 用户主动说"脑爆 [话题]"/"头脑实验"/"大师们讨论一下"
  · 院长在每日发现中发现高价值内容，主动建议

选题逻辑（AI主动选题时）：
  1. 回顾本周每日发现中评分最高的 3-5 条内容
  2. 读取用户 memory，判断哪些话题用户会感兴趣
  3. 选择 1 个最值得深挖的话题
  4. 如果用户没回复，AI自行完成头脑实验并存档，下次对话告知用户

执行流程：
  第1步：确定话题（院长选题或用户指定）
  第2步：院长分析（结合用户背景做初步分析）
  第3步：召集相关大师（根据话题选 2-3 位最匹配的大师，不一定是轮值的）
    · 每位大师从自己的认知框架出发
    · 3-5 句话，聚焦核心观点
    · 不同观点有冲突 → 保留冲突
  第4步：院长综合
    · 共识点
    · 分歧点
    · 对用户的实际意义（基于 memory）
    · 下一步设想（可能发展为动手实践项目）
  第5步：存档到沉淀空间
  第6步：推送给用户（如有推送渠道）

输出格式：
  # 🧪 头脑实验 · [日期] · [话题]
  
  ## 话题来源
  [来自哪天的每日发现 / 用户提出]
  
  ## 院长分析
  [结合用户背景的分析]
  
  ## 大师视角
  ### [大师A]
  [分析]
  ### [大师B]
  [分析]
  
  ## 综合
  - **共识**：...
  - **分歧**：...
  - **对你的意义**：...
  
  ## 下一步设想
  - [ ] 值得发展为动手实践项目？
  - [ ] 需要更多信息？（加入每日发现搜索词）
  - [ ] 可以直接行动的小事？
```

## 第三层 · 动手实践

**目标：从头脑实验中选一个可以去做的项目，院长做 PM 跟踪到完成。**

```
项目来源：
  · 头脑实验中产出的"下一步设想"
  · 用户主动说"这个我想动手做"
  · 院长基于用户 memory 主动建议

项目启动流程：
  第1步：院长提案
    "基于 [头脑实验/用户需求]，建议启动项目：[项目名]
     目标：[一句话]
     预计周期：[X天/周]"
  
  第2步：任务拆解（院长做PM的核心工作）
    把项目拆成具体任务，每个任务必须标注：
    
    | 任务 | 谁做 | 状态 | 截止 |
    |------|------|------|------|
    | 写初稿 | 🤖 AI | 待开始 | 4/10 |
    | 找3个案例 | 👤 用户 | 待开始 | 4/12 |
    | 整合成文 | 🤖 AI | 待开始 | 4/13 |
    | 发布到公众号 | 🤖 AI（需授权） | 待开始 | 4/14 |
    | 收集反馈 | 👤 用户 | 待开始 | 4/21 |
    
    🤖 = AI可以独立完成
    👤 = 用户需要在现实中做
    🤖+👤 = 需要协作
  
  第3步：用户确认
    "以上任务拆解OK吗？AI的部分我现在就开始，还是等你确认？"
    首次问"以后AI的部分自动做还是每次确认？" → 记住偏好
  
  第4步：执行与跟踪
    AI的任务 → 直接执行，完成后告知用户
    用户的任务 → 到期前主动提醒，完成后请用户反馈
    协作任务 → 先做AI的部分，等用户输入后继续
    
    进度跟踪（院长每次被唤起时检查）：
      · 有哪些任务逾期了？→ 提醒用户
      · 有哪些AI任务可以提前做？→ 主动做
      · 项目整体进度如何？→ 简要汇报
  
  第5步：验证与迭代
    · 项目完成后，如何验证效果？（院长要主动设计验证方式）
    · AI可以做的验证（数据分析、效果跟踪）→ 自动执行
    · 需要用户做的验证 → 提醒并跟踪
    · 根据验证结果迭代下一步

项目不一定每周换：
  · 一个项目没完成，不急着换新的
  · 院长持续跟踪直到项目完成或用户主动中止
  · 可以同时有多个项目，但院长要管好优先级
```
