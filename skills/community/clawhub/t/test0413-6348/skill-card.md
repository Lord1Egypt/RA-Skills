## Description: <br>
Security audit + append-only logging + monitoring for OpenClaw skills, including file-level diff, baseline approval, and SHA-256 integrity checks. <br>

This skill is ready for commercial/non-commercial use. <br>

## Publisher: <br>
[ucloud-sec](https://clawhub.ai/user/ucloud-sec) <br>

### License/Terms of Use: <br>
MIT-0 <br>


## Use Case: <br>
Developers and operators use this skill to audit local OpenClaw skill directories, record append-only scan logs, review file changes, and set up user-approved change monitoring. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: The skill reads local OpenClaw skill directories and stores audit logs, state, and git snapshots that may contain sensitive diff content. <br>
Mitigation: Keep secrets out of skill files, limit access to ~/.openclaw/skills-audit, and review stored diff contents before sharing them. <br>
Risk: Scheduled notifications can expose change details to an unintended channel or recipient. <br>
Mitigation: Enable cron only after confirming the notification channel, recipient, and timezone; send summaries by default and provide raw diffs only on explicit request. <br>
Risk: The release carries wallet and OAuth capability tags, but the security guidance says the artifacts do not need those credentials. <br>
Mitigation: Do not grant wallet or OAuth credentials to this skill; run it with only the local filesystem and git permissions needed for auditing. <br>
