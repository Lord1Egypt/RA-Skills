# 文档生成

## 适用场景

这个能力用于生成需求文档、设计文档、构建说明、能力说明、操作手册和最终 Skill 文档。

## 选择条件

- 需要把对话和搜索结果整理成正式文档。
- 需要输出结构稳定、可继续引用的材料。
- 需要把方案写成用户能直接阅读的说明。
- 需要把模板骨架填成完整产物。

## 输入

- 目标文档类型
- 受众
- 内容范围
- 语气要求
- 结构约束

## 输出

- 成文文档
- 标题结构
- 模块划分
- 缺口标记

## 边界

- 不把草稿当最终稿，必要时保留待确认项。
- 不把内部思考过程写进最终文档。
- 不把文档生成和真实执行混为一谈。

## 降级策略

- 信息不足时，先输出提纲，再补正文。
- 结构不稳定时，先固定章节，再填内容。
- 如果用户还没定稿，只生成可讨论版本。

## 与主流程的关系

这个能力常用于把调研阶段、设计阶段和构建阶段的结果串成统一文档链。
