## 一、AI Agent工程核心

### Harness工程五步循环
```
① 发现（Discover）→ 观察到需要执行的任务
② 交付（Execute）→ 调用工具/模型完成任务
③ 验证（Verify）→ 检查执行结果是否正确
④ 持久化（Persist）→ 经验和结果写入记忆
⑤ 调度（Schedule）→ 决定下一步做什么
```

**核心认知**：模型决定Agent的上限，Harness决定Agent的下限。同一模型在不同Harness下表现差异可达数十倍。

### 六核心工程零件
| 零件 | 功能 |
|:-----|:------|
| Automations | 触发执行的条件 |
| Worktrees | 并发执行隔离 |
| Skills | 可复用能力包 |
| Plugins | 外部连接器 |
| Sub-agents | 子智能体 |
| Memory | 状态持久化 |

**Loops数量决定Agent成熟度**：Hermes Agent有8个完整Loop（标杆），大多数框架只有1-2个。

### 记忆三个进化阶段
| 阶段 | 机制 | 突破 |
|:-----|:------|:------|
| Memory 1.0 | 向量数据库/知识库 | 被动存储事实 |
| Memory 2.0 | Memory Nudge/Skill Nudge | 事后反思总结 |
| Memory 3.0 | 动态上下文图谱+夜间异步巩固 | 自主进化 |

**五层分层记忆架构**：工作记忆→情景记忆→语义记忆→程序记忆→元记忆

### 六条Agent进化路径
| 路径 | 代表项目 |
|:-----|:---------|
| 输出自审 | Meta自审Agent |
| 持久记忆 | Hermes Agent |
| 进化搜索 | AutoGPT进化版 |
| 对抗训练 | 阿里巴巴双Agent |
| 自我修改 | OpenAI Codex |
| 编排自优化 | LangChain优化版 |

### 三层工程演进
```
Prompt Engineering（静态输入优化）
  ↓
Context Engineering（动态上下文管理）
  ↓
Harness Engineering（完整生存系统）
```
大多数团队在第1层，优秀团队在第2层，少数团队到达第3层。

### 6条洞察
1. **模型上限论 & Harness下限论**：模型决定上限，Harness决定下限
2. **闭环复利效应**：Agent价值随使用时间指数增长
3. **薄Harness + 厚Skills**：未来趋势
4. **App→Skill的生态转向**：从"安装→使用→卸载"到"发现→调度→组合"
5. **记忆是Agent核心护城河**：同模型同工具下，拉开差距的是记忆
6. **渐进式复杂度**：最小可用=1个Loop+3个工具+1层记忆

### 5条法则
1. **能力与生存分离**：优先构建生存系统再提升能力
2. **闭环思维**：没有闭环的Agent不是Agent
3. **复利设计**：为进化留出空间比当前性能更重要
4. **分层解耦**：任何复杂系统都可以通过分层变得可控
5. **渐进式复杂度**：先让它跑起来，再让它变聪明

### 多智能体架构
| 架构 | 适用场景 |
|:-----|:---------|
| 集中式 | 流程确定型 |
| 分布式 | 开放探索型 |
| **混合式⭐** | **大多数场景** |

**核心信令协议**：MCP（模型上下文协议）+ A2A（Agent-to-Agent）

### Agent落地八大挑战
流程理解建模→知识库与RAG→多步推理可靠性→工具编排→错误恢复→安全控制→成本控制→效果评估

## 二、CBI产品创新（4阶段门控）

```
Discover → Gate① → Create → Gate② → Develop → Gate③ → Deploy → Gate④
```

### Phase 1：Discover（发现）
**核心活动**：项目立项→痛点识别（六维度排序矩阵）→竞品分析→价值主张定义→商业论证
**门①评审**：痛点真实可量化？价值主张经验证有差异化？财务预测合理？

### Phase 2：Create（创建）
**核心活动**：结构化头脑风暴→用户验证→产品规格定义→GTM 5P策略→更新商业论证
**门②评审**：可持续差异化？客户验证付费意愿？GTM可行？

### Phase 3：Develop（开发）
**核心活动**：工程设计+原型→供应链落地→测试验证→上市准备（受控上市三步法）
**受控上市**：Crawl（1-2个区域试点2-3月）→ Walk（优化后扩展3-6月）→ Run（全面推广）
**门③评审**：通过全部测试验证？供应链就绪？客户确认？

### Phase 4：Deploy（部署）
**核心活动**：产能建设→团队组建→交付→追踪目标→上市后评审（30/90/180/365天）
**门④评审**：达成收入和利润率目标？经验教训总结？

### 商业论证8大要素
战略适配性→TAM/SAM/SOM→竞品分析→痛点清单→客户价值主张→产品目标→风险评估→财务测算

## 三、FTB 80/20 战略（9步法）

### Phase 1：识别80（Step 1-4）
**Step 1 Quad象限分析**（核心工具）：
```
         客户高价值   客户低价值
产品高价值   AA（核心）   AB（简化）
产品低价值   BA（培养）   BB（放弃）
```
**Step 2** 市场细分（≤10个，可执行，实用）
**Step 3** 细分P&L（管理费用聚焦在哪）
**Step 4** 行动计划与战略意图（Where we focus? How we win? Full potential?）

### Phase 2：聚焦80（Step 5-8）
**Step 5** Clean Sheet组织评估

**Step 8** USa：U理解（Brown Paper占70%精力）→ S简化（VA/NVA/NVAR）→ A行动

### Step 9：三大有机增长杠杆
| 杠杆 | 核心 |
|:-----|:------|
| Lever 1：80聚焦销售 | 向现有80客户卖更多80产品 |
