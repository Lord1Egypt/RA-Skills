# 人格分析维度规范

## 分析触发条件

- **轮次触发**：每 5 轮对话自动触发一次后台分析
- **实时触发**：检测到情绪突变（如从平稳突然到崩溃）时立即触发
- **静默运行**：分析过程对用户完全不可见，不阻塞主对话流

## 分析维度

| 维度 | 量表 | 说明 |
|------|------|------|
| 表达风格 | 1-10（内敛→外放） | 用户倾向于隐藏还是直接表达情绪 |
| 归因方式 | 1-10（外归因→内归因） | 用户把问题归因于外部还是自身 |
| 冲突模式 | 1-10（回避→对抗） | 面对冲突时的本能反应 |
| 自我觉察 | 1-10（低→高） | 对自身行为模式的认知程度 |
| 情绪调节 | 1-10（压抑→宣泄） | 处理情绪的方式 |
| 依恋模式 | 焦虑型 / 回避型 / 安全型 / 混乱型 | 在关系中的依恋倾向 |
| 改变意愿 | 1-10（抗拒→迫切） | 对改变的准备程度 |

## 输出格式

分析结果必须以 JSON 格式输出，包含：

```json
{
  "expression_style": {"score": 4, "observation": "..."},
  "locus_of_control": {"score": 3, "observation": "..."},
  "conflict_pattern": {"score": 2, "observation": "..."},
  "self_awareness": {"score": 5, "observation": "..."},
  "emotional_regulation": {"score": 3, "observation": "..."},
  "attachment_style": {"type": "焦虑型", "observation": "..."},
  "change_readiness": {"score": 3, "observation": "..."},
  "recommended_counselor": "烈风",
  "counselor_reason": "...",
  "personality_summary": "用2-3句话总结这个人的核心模式"
}
```

## 咨询师推荐规则

| 用户模式 | 推荐咨询师 | 原因 |
|----------|-----------|------|
| 回避冲突 + 低改变意愿 | 烈风 | 需要被点醒 |
| 情绪崩溃 + 高痛苦 | 暖光 | 需要先被接住 |
| 前后矛盾 + 无觉察 | 镜子 | 需要看见自己的模式 |
| 防御心重 + 轻松风格 | 老友 | 消解防御才能深入 |
| 其他情况 | 静水 | 保持中性引导 |

## 人格画像输出

退出时向用户展示的格式：

1. **一句话总结**：直击核心模式
2. **模式公式**：如「回避型冲突处理 × 外部归因 × 焦虑型依恋」
3. **维度评分条**：7个维度用进度条直观展示
4. **关键洞察**：指出用户没意识到的重复模式

禁止输出：诊断标签（如抑郁症）、建议鸡汤、道德评判
