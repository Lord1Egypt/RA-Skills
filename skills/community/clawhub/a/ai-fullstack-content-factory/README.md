# AI全链路内容创作工厂 (AI Full-Stack Content Factory)

> 从热点发现到多平台发布，一站式完成 AI 内容生产

## 业务场景

一位科技内容创作者，需要持续产出 AI / 大模型领域的深度文章，发布到微信公众号获取流量，同时希望在抖音/小红书平台分发短视频脚本和图文内容。

**痛点：**
- 热点转瞬即逝，人工搜索 + 写作耗时太长
- 多平台内容格式各异，改写成本高
- 封面图设计是瓶颈，经常卡在配图上

## 核心技能

| 技能名 | 用途 | 入口 |
|--------|------|------|
| **Agent-Reach** | 全网多平台搜索（Twitter/X、Reddit、YouTube、微信公众号、小红书） | 热点发现 |
| **公众号助手** | AI 生成 3000-5000 字深度文章 + 封面图，刘润风格 | 文章创作 |
| **card-renderer** | 渲染 Mac Pro / 赛博朋克 / 包豪斯 等风格知识卡片 | 视觉附图 |
| **nano-banana-pro** | Gemini 3 Pro Image 生成高质量配图 | 封面/配图 |
| **xiaohongshu-mcp** | 小红书内容发布、图片/图文/视频发布 | 多平台分发 |

## 工作流详解

### Phase 1：热点发现（Agent-Reach）
```bash
# 搜索 AI 热点
使用 Agent-Reach 搜索 Twitter/Reddit/YouTube 关键词
整合搜索结果，提取核心热点话题
```

### Phase 2：选题确认
助手展示热点列表，用户选择方向或调整，助手据此聚焦写作角度。

### Phase 3：文章创作（公众号助手）
```bash
# 调用公众号助手生成文章
自动撰写 3000-5000 字深度文章
自动生成配套封面图
```

### Phase 4：视觉资产（nano-banana-pro + card-renderer）
```
使用 nano-banana-pro 生成封面图
使用 card-renderer 渲染风格卡片（可选 Mac Pro/赛博朋克/包豪斯）
```

### Phase 5：多平台分发（xiaohongshu-mcp）
```
将公众号文章改写为小红书图文
发布到小红书
```

## 使用示例

### 场景：写一篇 GPT-5 最新进展的文章

**用户输入：**
```
帮我写一篇关于 GPT-5 最新进展的公众号文章，并发布到小红书
```

**助手自动执行：**
1. `Agent-Reach` 全网搜索 GPT-5 相关讨论
2. `公众号助手` 生成完整深度文章 + 封面图
3. `nano-banana-pro` 生成多个风格的封面备选
4. `card-renderer` 渲染知识卡片作为文内附图
5. `xiaohongshu-mcp` 改写为小红书版本并发布

**耗时：** 约 5-10 分钟（人工仅需确认选题和最终发布）

## 适用人群

- 科技 / AI 类公众号博主
- 内容工作室（批量生产内容）
- 想快速追热点的内容创作者
- 跨境内容运营者（英文热点 → 中文输出）

## 局限性

- 小红书发布需要有效的 Cookie 会话
- 部分平台（如微信公众号）需要手动确认发布
- 热点信息的时效性依赖搜索结果质量
