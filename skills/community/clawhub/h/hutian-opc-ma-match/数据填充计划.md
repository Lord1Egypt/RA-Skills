# 数据填充计划

> 本文档为上市公司并购数据库的数据填充提供分步实施计划，按赛道优先级分批填充，确保数据质量和覆盖完整性。

---

## 一、填充目标

### 1.1 总体目标

| 目标项 | 目标值 | 说明 |
|-------|-------|------|
| A股覆盖 | 500家 | 各赛道代表性公司 |
| 港股覆盖 | 200家 | 重点公司和壳资源 |
| 赛道覆盖 | 12个一级赛道 | 全覆盖 |
| 画像覆盖 | 100% | 入库公司均需画像 |

### 1.2 质量目标

| 质量维度 | 目标值 |
|---------|-------|
| 基础信息完整率 | ≥99% |
| 财务数据完整率 | ≥95% |
| 动力分类准确率 | ≥90% |
| 画像覆盖率 | 100% |
| 数据时效性 | ≤3个月 |

## 三、数据采集清单

### 3.1 单家公司采集字段

| 字段类别 | 采集字段 | 数据来源 | 采集方式 |
|---------|---------|---------|---------|
| **基本信息** | 公司名、代码、市场、上市时间 | 交易所网站 | 自动 |
| | 主营业务描述 | 公司官网/年报 | 半自动 |
| **财务数据** | 总市值、流通市值 | 东方财富/同花顺 | 自动 |
| | 营业收入、净利润 | 公司财报 | 自动 |
| | PE、PB | 行情数据 | 自动 |
| | 资产负债率 | 财报计算 | 半自动 |
| | 现金储备 | 财报提取 | 半自动 |
| | 研发投入比 | 财报计算 | 半自动 |
| **赛道分类** | 一级赛道 | 自主判断 | 人工 |
| | 二级赛道 | 自主判断 | 人工 |
| | 核心关键词 | 年报提取 | 半自动 |
| **动力分类** | 动力类型 | 判定模型 | 人工复核 |
| | 判定依据 | 分析结论 | 人工 |
| | 置信度 | 判定把握 | 人工 |
| **并购历史** | 并购次数 | 公告统计 | 半自动 |
| | 最近并购时间 | 公告记录 | 半自动 |
| | 并购标的赛道 | 公告提取 | 半自动 |
| **画像信息** | 偏好赛道 | 分析结论 | 人工 |
| | 财务门槛 | 分析结论 | 人工 |
| | 并购风格 | 历史行为 | 人工 |

### 3.2 数据采集工具推荐

| 数据类型 | 推荐工具 | 说明 |
|---------|---------|------|
| 财务数据 | 东方财富 | 数据全、更新快 |
| 并购公告 | 巨潮资讯网 | A股公告权威来源 |
| 港股数据 | 披露易 | 港交所公告 |
| 市值行情 | 同花顺/雪球 | 实时行情 |
| 行业分类 | 申万行业分类 | 标准分类参考 |

## 四、质量控制

### 4.1 采集质量检查

| 检查项 | 检查标准 | 不合格处理 |
|-------|---------|-----------|
| 必填字段 | 100%完整 | 补采 |
| 数据时效 | 财务数据≤3个月 | 更新 |
| 格式规范 | 符合字段格式要求 | 修正 |
| 逻辑一致 | 数据逻辑自洽 | 复核 |

### 4.2 动力分类质量检查

| 检查项 | 检查标准 | 不合格处理 |
|-------|---------|-----------|
| 分类依据 | 有明确判定依据 | 补充 |
| 置信度 | 与分类依据匹配 | 调整 |
| 一致性 | 同类公司分类一致 | 复核 |

### 4.3 画像质量检查

| 检查项 | 检查标准 | 不合格处理 |
|-------|---------|-----------|
| 画像完整性 | 关键字段均有 | 补填 |
| 画像准确性 | 与公司实际情况匹配 | 修正 |
| 画像时效 | 重大变化后更新 | 触发更新 |

## 五、实施保障

### 5.1 人员分工

| 角色 | 职责 | 人数 |
|-----|------|-----|
| 数据负责人 | 整体统筹、质量把控 | 1人 |
| 数据采集员 | 基础数据采集 | 2-3人 |
| 数据审核员 | 数据质量审核 | 1人 |
| 分析师 | 动力分类、画像分析 | 1-2人 |

### 5.2 进度跟踪

| 跟踪维度 | 跟踪频率 | 跟踪指标 |
|---------|---------|---------|
| 每日进度 | 每日 | 采集数量、完成率 |
| 每周汇总 | 每周 | 批次完成情况 |
| 质量检查 | 每批次 | 合格率、问题率 |
| 整体进度 | 每阶段 | 总目标完成率 |

### 5.3 风险管理

| 风险类型 | 风险描述 | 应对措施 |
|---------|---------|---------|
| 数据源不稳定 | 部分网站访问受限 | 准备备选数据源 |
| 分类标准争议 | 动力类型判断分歧 | 建立案例库、统一标准 |
| 工作量超预期 | 采集工作量大 | 优化工具、分批推进 |
| 数据质量不达标 | 采集数据质量差 | 加强培训、质量检查 |
