# 商业模式分析套件 SKILL.md

本技能为技术成果转化咨询师（OPC导师）设计，系统化分析商业计划书的商业模式可行性。

## 模块一：文件解析与信息提取

### 1.1 支持的文件格式
- PDF文件：使用PDF解析工具提取文本
- Word文件：使用docx解析工具提取文本
- 纯文本：直接读取文本内容

### 1.2 信息提取范围

#### PESTL维度提取
| 维度 | 提取内容 | 关键词 |
|------|----------|--------|
| P（政治） | 政策导向、政府态度、监管趋势、产业政策 | 政策扶持、补贴、政府合作、监管 |
| E（经济） | GDP增速、行业增长率、消费能力、融资环境 | 市场规模、增长率、投资、融资、盈利 |
| S（社会） | 人口结构、消费习惯、文化趋势、社会认知 | 目标用户、年龄层、消费行为、市场接受度 |
| T（技术） | 技术成熟度、替代技术、技术壁垒、研发投入 | 技术优势、专利、研发、创新、壁垒 |
| L（法律） | 法律法规、合规要求、知识产权、行业标准 | 合规、许可、资质、知识产权、标准 |

#### 波士顿矩阵维度提取
| 维度 | 提取内容 |
|------|----------|
| 市场份额 | 各产品/业务线的市场占有率数据 |
| 市场增长率 | 目标市场的年复合增长率 |
| 产品线分布 | 商业计划书中提到的所有产品/业务线 |

#### SWOT维度提取
| 维度 | 提取内容 |
|------|----------|
| S（优势） | 技术优势、资源优势、品牌优势、团队优势 |
| W（劣势） | 资金不足、人才缺失、经验欠缺、渠道薄弱 |
| O（机会） | 市场空白、政策利好、技术突破、合作伙伴 |
| T（威胁） | 竞争加剧、政策变化、技术替代、市场风险 |

#### 商业模式画布九宫格提取
| 维度 | 提取内容 |
|------|----------|
| 客户细分 | 目标客户群体、客户画像特征 |
| 价值主张 | 产品/服务价值、差异化卖点 |
| 渠道通路 | 销售渠道、获客方式、服务网络 |
| 客户关系 | 客户服务策略、留存机制 |
| 收入来源 | 盈利模式、定价方式、收入结构 |
| 核心资源 | 关键资产、技术能力、人才资源 |
| 关键业务 | 核心业务活动、运营重点 |
| 重要合作 | 合作伙伴、供应商、战略联盟 |
| 成本结构 | 主要成本项、成本结构特征 |

### 1.3 提取结果标注
- 记录每项提取信息的原文引用
- 标注信息来源位置（页码/章节）
- 找不到的信息标记为"未提取到"

### 1.4 关键规则
**严禁编造**：所有提取结果必须基于原文，无法提取的信息必须如实标注，不给出任何推测或假设值。

## 模块二：信息缺口识别与交互确认

### 2.1 缺口识别逻辑
```
完整分析所需信息 - 已提取信息 = 信息缺口
```

### 2.2 优先级排列
| 优先级 | 维度类别 | 具体维度 |
|--------|----------|----------|
| P0（必须） | 画布核心 | 价值主张、客户细分、收入来源 |
| P1（重要） | 画布支撑 | 成本结构、核心资源、关键业务 |
| P2（一般） | 画布渠道 | 渠道通路、客户关系 |
| P3（补充） | PESTL/SWOT | 各子维度补充 |

### 2.3 交互提问策略

#### 第1轮提问（画布核心维度）
最多3个问题，聚焦：价值主张、客户细分、收入来源

#### 第2轮提问（画布支撑维度）
最多3个问题，聚焦：核心资源、关键业务、重要合作

#### 第3轮提问（画布渠道维度）
最多3个问题，聚焦：渠道通路、客户关系

#### 第4轮提问（PESTL/SWOT补充）
最多3个问题，补充其他缺口信息

### 2.4 提问原则
- 简洁明确，每个问题不超过50字
- 提供选项或示例，便于用户回答
- 不给默认值，不编造答案

### 2.5 回答处理
- 用户回答直接补充到分析数据
- 无法回答的问题，保留"待补充"标记
- 缺失信息在最终报告中明确标注

## 模块三：PESTL分析

### 3.1 P-政治环境分析
**评估维度**：
- 政策导向：国家/地方政策支持程度
- 政府态度：政府采购、国有资本参与可能性
- 监管趋势：行业监管政策走向
- 产业政策：专项扶持政策、补贴政策

**输出格式**：
```
P（政治）：
- 关键发现：[基于原文的关键发现]
- 影响评估：[正面/负面/中性]
- 影响程度：[高/中/低]
```

### 3.2 E-经济环境分析
**评估维度**：
- GDP增速：宏观经济增长预期
- 行业增长率：细分市场增速数据
- 消费能力：目标客户支付能力
- 融资环境：资本市场活跃度、投资热度

### 3.3 S-社会文化分析
**评估维度**：
- 人口结构：目标人群规模、年龄分布
- 消费习惯：购买频次、决策周期
- 文化趋势：社会价值观变化
- 市场认知：行业接受度、教育成本

### 3.4 T-技术环境分析
**评估维度**：
- 技术成熟度：技术所处阶段（实验室/中试/量产）
- 替代技术：同类技术竞争情况
- 技术壁垒：专利布局、核心技术控制
- 研发投入：研发投入占比、技术迭代速度

### 3.5 L-法律合规分析
**评估维度**：
- 法律法规：行业相关法规要求
- 合规要求：资质许可、认证要求
- 知识产权：专利布局、商标保护
- 行业标准：国家标准、行业规范

| T | | | |

## 模块四：波士顿矩阵分析

### 4.1 四象限定义
| 象限 | 市场增长率 | 相对市场份额 | 战略建议 |
|------|------------|--------------|----------|
| 明星（Star） | 高（>10%） | 高（>1） | 加大投入，保持增长优势 |
| 现金牛（Cash Cow） | 低（<10%） | 高（>1） | 维持现状，贡献现金流 |
| 问题（Question Mark） | 高（>10%） | 低（<1） | 选择性投资，寻求突破 |
| 瘦狗（Dog） | 低（<10%） | 低（<1） | 收缩或退出 |

### 4.2 产品线分类
基于商业计划书中提到的产品/业务线，按市场份额和增长率分类。

### 4.3 推断规则
如果商业计划书中没有明确的产品线划分，根据业务描述进行合理推断，并在报告中标注"【推断】"。

### 4.4 波士顿矩阵图（文本版）
```
                    高增长率
                       ↑
                       │
      问题业务         │        明星业务
      （高增长         │       （高增长

      ══════════════════════════════→ 高市场份额
       低增长          │        低增长
```

- 战略建议
- 资源配置优先级

## 模块五：SWOT分析

### 5.1 S-优势（Strengths）
**定义**：内部有利因素，核心竞争力

**评估维度**：
- 技术优势：核心技术、专利壁垒
- 资源优势：资金、渠道、品牌
- 能力优势：团队能力、运营经验
- 定位优势：市场定位、差异化

**输出格式**：
```
S（优势）：
1. [优势1] - [重要性评级：高/中/低]
2. [优势2] - [重要性评级：高/中/低]
...
```

### 5.2 W-劣势（Weaknesses）
**定义**：内部不利因素，短板差距

**评估维度**：
- 资源不足：资金、人才、渠道
- 能力缺失：运营、研发、管理
- 市场劣势：品牌认知、份额占比
- 成本劣势：生产成本、获客成本

### 5.3 O-机会（Opportunities）
**定义**：外部有利因素，市场窗口

**评估维度**：
- 市场机会：市场空白、需求增长
- 政策机会：政策利好、扶持补贴
- 技术机会：技术突破、跨界融合
- 合作机会：战略伙伴、生态协同

### 5.4 T-威胁（Threats）
**定义**：外部不利因素，风险挑战

**评估维度**：
- 竞争威胁：竞争对手、替代产品
- 政策威胁：政策变化、监管收紧
- 技术威胁：技术替代、颠覆创新
- 市场威胁：需求变化、进入门槛

### 5.5 SWOT交叉策略

| 组合 | 策略名称 | 策略描述 |
|------|----------|----------|
| S×O | 增长策略 | 利用优势抓住机会 |
| W×O | 改善策略 | 弥补劣势抓住机会 |
| S×T | 防御策略 | 利用优势规避威胁 |
| W×T | 避险策略 | 弥补劣势规避威胁 |

**输出格式**：
```
SWOT交叉策略：

SO增长策略：[具体行动建议]
WO改善策略：[具体行动建议]
ST防御策略：[具体行动建议]
WT避险策略：[具体行动建议]
```

## 模块六：商业模式画布九宫格

### 6.1 客户细分（Customer Segments）
**定义**：企业瞄准的消费者群体

**核心问题**：
- 我们为谁创造价值？
- 谁是我们最重要的客户？

**评估标准**（1-10分）：
| 分数 | 等级 | 描述 |
|------|------|------|
| 1-3 | 不及格 | 客户定义模糊，未明确目标群体 |
| 4-6 | 及格 | 有基本客户画像，但细分不足 |
| 7-8 | 良好 | 客户细分清晰，有差异化定位 |
| 9-10 | 优秀 | 精准客户画像+可量化市场规模+验证数据 |

**评分依据**：
- 客户画像完整度
- 市场细分合理性
- 市场规模可量化
- 客户验证情况

### 6.2 价值主张（Value Propositions）
**定义**：解决客户问题或满足客户需求的系列产品和服务

**核心问题**：
- 我们给客户带来什么价值？
- 解决了客户的什么问题？
- 满足客户的哪些需求？

**评估标准**（1-10分）：
| 分数 | 等级 | 描述 |
|------|------|------|
| 1-3 | 不及格 | 价值主张不清，无法回答"为什么选你" |
| 4-6 | 及格 | 有基本价值主张，但差异化不明显 |
| 7-8 | 良好 | 价值主张清晰，有明确差异化 |
| 9-10 | 优秀 | 强价值主张+验证数据+护城河 |

**评分依据**：
- 价值主张清晰度
- 差异化程度
- 客户需求匹配度
- 竞争壁垒

### 6.3 渠道通路（Channels）
**定义**：如何接触、抵达客户并传递价值

**核心问题**：
- 如何触达客户？
- 如何销售产品/服务？
- 如何交付价值？

**评估标准**（1-10分）：
| 分数 | 等级 | 描述 |
|------|------|------|
| 1-3 | 不及格 | 无明确渠道规划 |
| 4-6 | 及格 | 有基本渠道，但依赖单一渠道 |
| 7-8 | 良好 | 多渠道布局，有优先级 |
| 9-10 | 优秀 | 全渠道体系+获客成本数据+渠道验证 |

**评分依据**：
- 渠道完整性
- 渠道多元化
- 获客成本
- 渠道效率

### 6.4 客户关系（Customer Relationships）
**定义**：与客户建立的关系类型

**核心问题**：
- 如何建立和维护客户关系？
- 如何提升客户忠诚度？
- 如何提高客户留存？

**评估标准**（1-10分）：
| 分数 | 等级 | 描述 |
|------|------|------|
| 1-3 | 不及格 | 未考虑客户关系维护 |
| 4-6 | 及格 | 有基本客户服务，但无体系 |
| 7-8 | 良好 | 客户关系分层管理，有留存策略 |
| 9-10 | 优秀 | 客户生命周期管理+LTV数据+NPS指标 |

**评分依据**：
- 客户关系深度
- 留存策略有效性
- 客户价值指标（LTV、CAC）
- 客户满意度

### 6.5 收入来源（Revenue Streams）
**定义**：从客户群体中获得的收入

**核心问题**：
- 如何赚钱？
- 客户愿意为什么付费？
- 收入结构如何？

**评估标准**（1-10分）：
| 分数 | 等级 | 描述 |
|------|------|------|
| 1-3 | 不及格 | 盈利模式不清 |
| 4-6 | 及格 | 有基本收入模式，但单一 |
| 7-8 | 良好 | 多元收入来源，定价合理 |
| 9-10 | 优秀 | 收入模型验证+单位经济数据+增长飞轮 |

**评分依据**：
- 盈利模式清晰度
- 收入来源多元化
- 定价合理性
- 单位经济模型

### 6.6 核心资源（Key Resources）
**定义**：执行商业模式所需的关键资产

**核心问题**：
- 需要哪些关键资源？
- 如何获取和保有这些资源？
- 资源的稀缺性和壁垒如何？

**评估标准**（1-10分）：
| 分数 | 等级 | 描述 |
|------|------|------|
| 1-3 | 不及格 | 核心资源未识别 |
| 4-6 | 及格 | 识别了部分资源，但不够系统 |
| 7-8 | 良好 | 核心资源清晰，有获取路径 |
| 9-10 | 优秀 | 资源壁垒+独占性+可持续性 |

**评分依据**：
- 资源识别完整性
- 资源壁垒强度
- 资源独占性
- 资源可持续性

### 6.7 关键业务（Key Activities）
**定义**：执行商业模式所需的关键活动

**核心问题**：
- 必须做好哪些事？
- 业务优先级如何？
- 业务的可扩展性如何？

**评估标准**（1-10分）：
| 分数 | 等级 | 描述 |
|------|------|------|
| 1-3 | 不及格 | 关键业务不明确 |
| 4-6 | 及格 | 识别了部分关键业务 |
| 7-8 | 良好 | 关键业务清晰，有优先级和里程碑 |
| 9-10 | 优秀 | 业务闭环+效率指标+可扩展性 |

**评分依据**：
- 业务识别清晰度
- 业务优先级合理性
- 业务闭环完整性
- 业务可扩展性

### 6.8 重要合作（Key Partnerships）
**定义**：让商业模式运作所需的关键合作伙伴

**核心问题**：
- 需要哪些关键伙伴？
- 合作伙伴的价值是什么？
- 合作模式和绑定机制如何？

**评估标准**（1-10分）：
| 分数 | 等级 | 描述 |
|------|------|------|
| 1-3 | 不及格 | 未考虑合作伙伴 |
| 4-6 | 及格 | 有基本合作意识，但无具体方案 |
| 7-8 | 良好 | 合作伙伴明确，有合作框架 |
| 9-10 | 优秀 | 战略联盟+生态协同+绑定机制 |

**评分依据**：
- 合作伙伴识别
- 合作模式清晰度
- 生态协同效应
- 绑定机制有效性

### 6.9 成本结构（Cost Structure）
**定义**：执行商业模式所产生的全部成本

**核心问题**：
- 主要成本项是什么？
- 成本结构特征？
- 如何优化成本？

**评估标准**（1-10分）：
| 分数 | 等级 | 描述 |
|------|------|------|
| 1-3 | 不及格 | 成本结构不清 |
| 4-6 | 及格 | 有基本成本估算，但不够精细 |
| 7-8 | 良好 | 成本结构清晰，有优化空间 |
| 9-10 | 优秀 | 精细成本管理+边际成本递减+盈利拐点 |

**评分依据**：
- 成本识别完整性
- 成本结构合理性
- 边际成本趋势
- 盈利模型清晰度

## 关键原则

1. **不编造**：信息提取严格基于输入文件，找不到的标记"未提取到"
2. **交互确认**：缺口信息必须向用户提问，不给默认值
3. **量化评估**：九宫格每维度1-10分，有明确评分标准
4. **OPC视角**：站在技术成果转化咨询师角度，关注技术成果的商业化可行性
5. **去夸张底线**：严禁"颠覆性""全球领先"等表述
6. **数据支撑**：评分必须有依据，引用原文或用户确认
