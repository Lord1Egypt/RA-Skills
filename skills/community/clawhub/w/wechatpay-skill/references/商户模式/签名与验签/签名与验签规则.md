# 签名与验签规则（V3 API）

> 基于微信支付官方文档精简，专注Python/Node.js实现要点

## 核心概念

| 概念 | 说明 |
|------|------|
| **商户API证书** | 商户在商户平台申请的证书，包含公钥和私钥 |
| **商户API私钥** | 用于生成签名，**仅商户持有，不可泄露** |
| **证书序列号** | 标识使用哪个证书，放在HTTP头中 |
| **微信支付公钥** | 用于验证微信响应和回调签名的真实性 |
| **APIv3密钥** | 用于回调通知中敏感信息的AES-256-GCM解密（32字节） |

## 签名类型对比

| 版本 | 签名算法 | 私钥类型 | 适用接口 |
|------|----------|----------|----------|
| V2 | MD5/HMAC-SHA256 | API密钥 | 付款码支付（V2专用） |
| V3 | SHA256-RSA | API证书私钥 | JSAPI/小程序/Native/APP/H5 |

## 接口请求签名（V3）

### 签名串构造（5行格式）

```
HTTP请求方法\n
URL（不含域名）\n
请求时间戳\n
请求随机串\n
请求报文主体（GET为空）\n
```

### 示例：JSAPI下单签名

```
POST
/v3/pay/transactions/jsapi
1600000000
C5BE8E6B1B7A5D3E9F2A8C4B6E1D9F3A
{"amount":{"total":1},"currency":"CNY","appid":"wxa5f5c1d6e8f9a2b3","mchid":"1234567890","notify_url":"https://example.com/notify","out_trade_no":"ORDER20201010001","description":"测试商品"}
```

### Authorization头格式

```
Authorization: WECHATPAY2-SHA256-RSA2048 mchid="{商户号}",nonce_str="{随机串}",timestamp="{时间戳}",serial_no="{证书序列号}",signature="{签名值}"
```

### 签名算法

1. 使用商户API私钥（PKCS8格式）
2. 对签名串进行SHA256摘要
3. 用私钥对摘要进行RSA签名
4. 签名结果Base64编码

## 调起支付签名（前端）

### JSAPI/小程序支付

签名串（4行）：
```
{appId}\n
{timeStamp}\n
{nonceStr}\n
{package}\n
```

**package值**：`prepay_id=wx201410272009395522657a690389285100`

### APP支付

签名串（4行）：
```
{appId}\n
{timeStamp}\n
{nonceStr}\n
{prepayId}\n
```

**prepayId值**：纯prepay_id，不带`prepay_id=`前缀

### 签名算法

与接口请求签名相同，使用商户API私钥。

## 回调通知验签

### 验签步骤

1. **获取响应头**：
   - `Wechatpay-Signature`：签名值
   - `Wechatpay-Nonce`：随机串
   - `Wechatpay-Timestamp`：时间戳
   - `Wechatpay-Serial`：证书序列号

2. **构造待验签串**：
   ```
   {timestamp}\n
   {nonce}\n
   {body}\n
   ```

3. **获取微信支付公钥**：
   - 从证书序列号匹配商户平台配置的微信支付公钥
   - 或使用微信支付平台证书中的公钥

4. **验签**：
   - 使用微信支付公钥解密签名
   - 比对解密结果与待验签串的SHA256

## 敏感信息解密

回调通知中的敏感字段（如真实姓名、身份证号）使用AES-256-GCM解密：

```
APIv3密钥 + 加密算法AES-256-GCM + 附加数据（nonce+associated_data）
```

## 常见签名错误

| 错误 | 原因 |
|------|------|
| SIGN_ERROR | 签名串格式错误、签名算法错误、私钥不匹配 |
| 调起失败 | appId填错、package值带前缀/不带前缀混淆 |
| 回调验签失败 | 用了错误的公钥、时间戳格式错误 |

## Python实现要点

```python
# 签名关键代码
from cryptography.hazmat.primitives import hashes, serialization
from cryptography.hazmat.primitives.asymmetric import padding
from cryptography.hazmat.backends import default_backend
import base64, json

def sign(sign_str: str, private_key: str) -> str:
    """V3签名"""
    private_key_obj = serialization.load_pem_private_key(
        private_key.encode(),
        password=None,
        backend=default_backend()
    )
    signature = private_key_obj.sign(
        sign_str.encode(),
        padding.PKCS1v15(),
        hashes.SHA256()
    )
    return base64.b64encode(signature).decode()
```

## Node.js实现要点

```javascript
// 签名关键代码
const crypto = require('crypto');

function sign(signStr, privateKey) {
    const sign = crypto.sign('RSA-SHA256', Buffer.from(signStr), privateKey);
    return sign.toString('base64');
}
```
