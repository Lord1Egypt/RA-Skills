# record2note

An AI agent skill that turns voice recordings into structured Markdown notes.

Compatible with Claude, ChatGPT, Gemini, OpenCode, and other AI agents.

## Features

- Speech transcription with whisper.cpp
- Optional speaker diarization with pyannote-audio
- LLM-generated summaries, key points, and todos
- Markdown notes compatible with Obsidian, Notion, and Logseq
- Two workflows: automatic monitoring and manual processing
- Supports macOS and Windows

## Installation

### Option 1: Install through an agent (recommended)

Tell your AI agent:

> Install the `record2note` skill from this directory.

The agent will handle setup and configuration.

### Option 2: Manual install

**macOS / Linux:**

```bash
chmod +x install.sh
./install.sh
```

**Windows:**

```powershell
powershell -ExecutionPolicy Bypass -File install.ps1
```

`install.sh` installs to `~/.config/record2note/`. If your agent uses a custom skill directory, copy the files there after installation.

After installation, tell the agent: `Use record2note`.

## Dependencies

| Dependency | macOS | Windows |
|---|---|---|
| whisper-cli | `brew install whisper-cpp` | [GitHub release](https://github.com/ggerganov/whisper.cpp/releases) |
| ffmpeg/ffprobe | `brew install ffmpeg` | `choco install ffmpeg` |
| Python 3 | Included with macOS | `choco install python` |
| pyannote-audio (optional) | `pip install pyannote-audio torch` | Same |
| fswatch | `brew install fswatch` | Not needed, use FileSystemWatcher |

## Usage

After installation, tell the agent:

- `Use record2note` - start the skill and enter setup or mode selection
- `Enable automatic monitoring` - watch the folder in the background
- `Process recording` - process a file manually

## Configuration

Configuration lives in the skill root:

- `config.json` - generated by setup
- `config.example.json` - example settings
