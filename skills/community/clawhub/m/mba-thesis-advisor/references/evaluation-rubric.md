# MBA 论文评价 Rubric v2.0

本 Rubric 用于对 MBA 论文进行系统性评价，识别改进优先级。
基于"写法 B"理念设计，侧重区分优秀论文与及格论文。

---

## 总体规则

- 每个子指标打 **1-4 分**（1=不及格 / 2=及格 / 3=良好 / 4=优秀）
- 维度得分 = 子指标加权平均
- 总分 = 7 个维度加权求和，满分 4.0
- **底线规则**：D1（原创贡献）或 D2（内部视角）任一维度 ≤ 1.5，总评封顶"及格"
- **等级划分**：
  - 优秀 ≥ 3.5（且 D1、D2 均 ≥ 3）
  - 良好 3.0 – 3.49
  - 及格 2.0 – 2.99
  - 不及格 < 2.0

---

## D1 原创贡献（总权重 20%）

核心问题：**论文提供了什么前人没有的知识增量？**

### D1a 知识增量的可辨识性（维度内权重 35%）

| 分数 | 标准 |
|:----:|------|
| 1 | 通读全文后无法用一句话说出"这篇论文告诉了我什么新东西" |
| 2 | 能提取出 1-2 条新信息，但本质是"旧理论 + 新场景"的组合 |
| 3 | 有明确的新发现（新机制/新解释/新关系），读者能感受到认知更新 |
| 4 | 新发现令人"意外但事后合理"——读前不知道，读后觉得"原来如此" |

**快速检测**：读完第五章后闭卷写一句话——"这篇论文最重要的新发现是___"。
写不出 = 1 分；能写出但觉得"常识" = 2 分。

### D1b 概念化程度（维度内权重 30%）

| 分数 | 标准 |
|:----:|------|
| 1 | 发现停留在现象描述层面，未进行任何概念抽象 |
| 2 | 对现象进行了归类或命名，但定义模糊，不具备可迁移性 |
| 3 | 提出了命名的概念，有清晰定义，可在其他案例中被识别和应用 |
| 4 | 概念有名称 + 定义 + 操作化指标 + 边界条件，形成"可引用的学术工具" |

**快速检测**：论文中是否有加粗/引号标注的原创术语？该术语有定义段落吗？
能否在另一家公司案例中"拿来用"？

### D1c 贡献的显式陈述（维度内权重 20%）

| 分数 | 标准 |
|:----:|------|
| 1 | 全文无任何关于"本文贡献"的明确陈述 |
| 2 | 在结论中笼统提及"本文丰富了 XX 理论"，但未说明如何丰富 |
| 3 | 有专门的"理论贡献"小节，逐条列出贡献内容 |
| 4 | 贡献陈述采用对比结构："已有文献认为 X，本文发现 Y，差异原因是 Z"——每条贡献有对标物 |

**快速检测**：在第五章搜索"理论贡献"或"原创性贡献"，检查是否有独立小节；
每条贡献是否有"区别于已有研究"的对比句。

### D1d 贡献的层次多样性（维度内权重 15%）

| 分数 | 标准 |
|:----:|------|
| 1 | 仅有一种类型的贡献（如仅提出了建议） |
| 2 | 有两种类型（如新发现 + 新框架），但层次单一 |
| 3 | 覆盖概念/机制/框架中的至少两种，且各自独立有价值 |
| 4 | 形成递进层次：提出原创概念 → 识别因果机制 → 构建分析框架 → 提出可检验命题 |

**快速检测**：列出论文声称的所有贡献，分类为概念类/机制类/框架类/方法类/命题类。
类别数 ≥ 3 = 良好。

## D2 内部视角利用度（总权重 20%）

核心问题：**作者是否利用了只有内部人才能获得的信息？**

### D2a 信息独特性（维度内权重 40%）

| 分数 | 标准 |
|:----:|------|
| 1 | 全部信息可通过搜索引擎/年报获得 |
| 2 | 有少量非公开信息（如"据了解"），但缺乏具体细节 |
| 3 | 包含具体的内部制度名称、决策节点、人物行为——公开渠道查不到 |
| 4 | 提供了制度运作的"黑箱"逻辑：不仅说了"发生了什么"，还解释了"内部规则为什么导致它发生" |

**快速检测**：遮住作者姓名，问："一个行业分析师能写出这段话吗？"
能 = 1-2 分；不能 = 3-4 分。

### D2b 信息的证据化程度（维度内权重 30%）

| 分数 | 标准 |
|:----:|------|
| 1 | 内部信息仅以"据笔者观察"出现，无任何结构化呈现 |
| 2 | 有基本事实陈述，但未转化为支撑论断的证据 |
| 3 | 内部信息被结构化为"现象 → 制度原因 → 影响"的论证链，直接支撑核心论断 |
| 4 | 内部信息经三角验证（多个信息源/时间点交叉印证），且作者显式说明了信息来源的可靠性与局限 |

**快速检测**：找出所有"基于内部观察"的段落，检查每段是否包含：
①具体事实 ②制度解释 ③对核心论断的支撑关系。三者齐全 = 3 分以上。

### D2c 覆盖面与贯穿度（维度内权重 30%）

| 分数 | 标准 |
|:----:|------|
| 1 | 内部视角仅出现在某一小节的"附带提及" |
| 2 | 集中在一个章节或一个论题上 |
| 3 | 覆盖论文的 2-3 个核心论题 |
| 4 | 贯穿全文主要章节——从问题诊断到机制分析到改进方案都有内部信息支撑 |

**快速检测**：在每一章标记"此处使用了内部视角"的段落数。
仅 1 章有 = 2 分；3 章以上有 = 4 分。

## D3 理论框架的选择与使用（总权重 15%）

核心问题：**框架是匹配问题的分析利器，还是套用的标签？**

### D3a 框架与研究问题的匹配度（维度内权重 35%）

| 分数 | 标准 |
|:----:|------|
| 1 | 研究问题问 A，框架回答 B；明显错位 |
| 2 | 框架能部分覆盖研究问题，但有明显遗漏维度 |
| 3 | 框架与研究问题高度匹配——用该框架可以系统回答提出的问题 |
| 4 | 匹配度高且具有不可替代性：换一个框架会损失关键分析维度 |

**快速检测**：做"研究问题 → 框架"映射表。每个 RQ 能对应到框架吗？
有无 RQ 找不到对应框架？有无框架找不到对应 RQ？

### D3b 框架在分析章节的实质使用（维度内权重 35%）

| 分数 | 标准 |
|:----:|------|
| 1 | 框架仅在第二章出现，第四章分析未使用任何理论术语 |
| 2 | 第四章提及了框架名称，但仅为"贴标签"式引用 |
| 3 | 框架的核心概念被用于**结构化**第四章的分析（如按框架维度组织小节） |
| 4 | 框架不仅结构化了分析，还产生了**新发现**——不用该框架就不会识别出的洞见 |

**快速检测**：在第四章搜索第二章每个理论的关键术语。
出现 0 次 = 死理论；仅出现在段首"根据 XX" = 贴标签；用于组织论证结构 = 实质使用。

**附：死理论检测清单模板**

对第二章中引入的每一个理论/框架，逐一填写：

```
| 理论名称 | 第四章是否实质使用？ | 第五章结论是否涉及？ | 判定 |
|----------|:------------------:|:------------------:|------|
| [理论A]  | 是/否              | 是/否              | 核心框架 / 辅助工具 / 死理论 |
| [理论B]  | ...                | ...                | ...  |
```

判定规则：
- **核心框架**：第四章实质使用 + 第五章涉及
- **辅助工具**：第四章实质使用，第五章未单独提及（可接受，但需控制数量 ≤ 2）
- **死理论**：第四章未实质使用 → 应删除或压缩至 1-2 句背景介绍

**达标标准**：死理论数量 = 0 得 4 分；1 个得 3 分；2-3 个得 2 分；≥ 4 个得 1 分。

### D3c 理论的批判性使用（维度内权重 30%）

| 分数 | 标准 |
|:----:|------|
| 1 | 理论被当作真理直接套用，未讨论任何适用条件 |
| 2 | 在文献综述中提及了理论的一般局限，但未与本案例关联 |
| 3 | 明确讨论了该理论在本案例情境中的适用性边界，做了情境化调整 |
| 4 | 不仅情境化修正，还反向贡献：用本案例发现指出了该理论的盲区或修正方向 |

**快速检测**：搜索"但需要注意""在本研究情境中""该框架的局限"等表述。
找到 + 与本案例关联 = 3 分以上。

## D4 论证链条连贯性（总权重 15%）

核心问题：**从研究问题 → 理论 → 分析 → 结论，是否形成不断线的逻辑链？**

### D4a 研究问题的贯穿性（维度内权重 35%）

| 分数 | 标准 |
|:----:|------|
| 1 | 研究问题仅在第一章出现，后续章节未回应 |
| 2 | 研究问题在第四/五章被部分回应，但存在遗漏（某个 RQ 未被回答） |
| 3 | 每个研究问题在分析章节和结论中都有明确的回应段落 |
| 4 | 每个 RQ 的回应可追溯：RQ → 理论工具 → 证据 → 发现 → 结论，形成完整"证据链" |

**快速检测**：列出绪论中所有研究问题，在第四/五章逐一搜索对应回答。
每个 RQ 是否有 ≥ 1 段明确回应？

**附：研究问题追溯表模板**

```
| 研究问题 | 对应理论工具 | 第四章回应段落 | 第五章结论段落 | 证据链完整性 |
|----------|------------|:------------:|:------------:|:-----------:|
| RQ1      | [框架名]   | §4.X.X       | §5.1 第X段   | 完整/有断裂  |
| RQ2      | ...        | ...          | ...          | ...         |
```

### D4b 跨章衔接（维度内权重 30%）

| 分数 | 标准 |
|:----:|------|
| 1 | 各章"硬切换"，无过渡段落 |
| 2 | 有基本过渡句（"在上一章基础上"），但未说明逻辑关系 |
| 3 | 每章开头/结尾有明确衔接段：上一章解决了什么 → 本章要解决什么 → 为什么需要本章 |
| 4 | 衔接段不仅说"做什么"，还说"为什么这个顺序"——读者能理解论证递进是唯一合理路径 |

**快速检测**：只读每章第一段和最后一段，能否还原全文论证脉络？
能 = 3-4 分；不能 = 1-2 分。

### D4c 无"死理论"与无"无源结论"（维度内权重 35%）

| 分数 | 标准 |
|:----:|------|
| 1 | ≥ 4 个死理论，或 ≥ 2 个无源结论 |
| 2 | 2-3 个死理论，或 1 个无源结论 |
| 3 | ≤ 1 个死理论，且无无源结论 |
| 4 | 零死理论、零无源结论——综述中每个框架在分析中有着落，结论中每个论断可追溯至分析 |

**快速检测**：
- **死理论**：用 D3 的检测清单统计
- **无源结论**：对第五章每个结论性论断，在第四章标注其支撑段落；
  找不到支撑 = "无源结论"

**附：无源结论检测表模板**

```
| 第五章结论论断（摘要） | 第四章支撑段落 | 判定 |
|----------------------|:------------:|------|
| [结论1]              | §4.X.X       | 有源 / 弱源 / 无源 |
| [结论2]              | ...          | ...  |
```

判定规则：
- **有源**：分析章节有 ≥ 1 段专门论述，且证据充分
- **弱源**：分析章节有提及但篇幅不足以支撑结论的力度
- **无源**：分析章节完全未涉及，结论"凭空出现"

## D5 数据与证据质量（总权重 10%）

核心问题：**每个关键论断是否有可追溯、可验证的证据支撑？**

### D5a 关键论断的证据支撑率（维度内权重 30%）

| 分数 | 标准 |
|:----:|------|
| 1 | > 50% 的关键论断无数据/引用支撑 |
| 2 | 多数关键论断有引用，但来源单一（如均为同一份报告） |
| 3 | 每个关键论断至少有 1 个可查证的数据源，且来源多元 |
| 4 | 核心论断有 ≥ 2 个独立来源交叉印证（三角验证） |

**快速检测**：标记第四章所有关键论断（通常以"本文发现/认为/表明"开头），
检查每个论断下方 100 字内是否有 `\cite{}` 或具体数字。

### D5b 一手数据方法论（维度内权重 30%）

| 分数 | 标准 |
|:----:|------|
| 1 | 声称有一手数据（问卷/访谈）但完全无方法论描述 |
| 2 | 有基本方法描述（如"进行了访谈"），但缺样本量、问题设计 |
| 3 | 方法论包含：数据收集时间/方式、样本量/选取标准、分析方法；**或**明确标注"XX 数据为基于参与观察的主观判断，非统计结果" |
| 4 | 一手数据有完整方法论（样本量、抽样方式、信效度、分析方法），附录可查原始工具 |

**快速检测**：搜索"问卷""访谈""调查"关键词，每处是否有方法论说明？
若声称有问卷但无样本量 = **硬伤**（封顶 1.5 分）。

**重要提示**：诚实标注 > 伪装严谨。如果确实没有做正式问卷/访谈，
把"基于问卷调查和访谈数据"改为"基于作者在 XX 部门 XX 年的参与观察和公开数据研判"，
反而能拿到 3 分——因为方法论描述与实际做法一致。

### D5c 数据的时效性与可比性（维度内权重 20%）

| 分数 | 标准 |
|:----:|------|
| 1 | 使用 5 年以上旧数据作为当前分析依据 |
| 2 | 数据基本为近 3 年内，但缺乏时间序列或横向对比 |
| 3 | 有时间序列数据（展示趋势）或横向对比数据（竞对/行业基准） |
| 4 | 同时具备时间序列 + 横向对比 + 对比基准 |

**快速检测**：检查数据表格，有 ≥ 2 年纵向对比？有 ≥ 2 家企业横向对比？

### D5d 图表规范性（维度内权重 20%）

| 分数 | 标准 |
|:----:|------|
| 1 | 无图表，或图表无标题/无数据来源 |
| 2 | 有图表且有标题，但数据来源标注不完整或图表与正文脱节 |
| 3 | 图表有规范标题、数据来源脚注、`\ref{}` 交叉引用；与正文论述紧密配合 |
| 4 | 图表规范且具有"一图胜千言"的信息密度——看图即可理解核心发现 |

**快速检测**：每张图表是否有 `\caption`？有数据来源？正文有 `\ref` 引用？
去掉正文只看图表，能否理解核心观点？

## D6 结论严谨性与边界感（总权重 15%）

核心问题：**结论是否可证伪、有边界、不过度推广？**

### D6a 结论的可证伪性（维度内权重 30%）

| 分数 | 标准 |
|:----:|------|
| 1 | 结论为不可证伪的"应该/建议"句式 |
| 2 | 结论为事实性总结，正确但不可被检验 |
| 3 | 结论可以被后续研究验证或反驳（给出了条件和预测方向） |
| 4 | 明确提出了形式化的可检验命题，包含变量、阈值或时间窗口 |

**快速检测**：对每个核心结论问——"什么样的证据能证明这个结论是错的？"
答不出来 = 不可证伪 = 1 分。

### D6b 边界条件的明确性（维度内权重 25%）

| 分数 | 标准 |
|:----:|------|
| 1 | 结论被表述为普适真理，无适用范围限定 |
| 2 | 在"局限性"节提及了一般性外推限制 |
| 3 | 在结论本身中嵌入了边界条件（"在 X 条件下成立，在 Y 条件下可能不成立"） |
| 4 | 边界条件区分了"确定不适用"和"可能不适用但需检验"两类情形 |

**快速检测**：在结论段搜索"但""前提是""不适用于""需注意"等限定词。
找不到 = 1-2 分；嵌入在结论句中 = 3-4 分。

### D6c 替代性解释的回应（维度内权重 25%）

| 分数 | 标准 |
|:----:|------|
| 1 | 未考虑任何替代性解释 |
| 2 | 在局限性中提及"可能有其他解释"，但未展开 |
| 3 | 主动提出 1-2 种替代性解释，并给出不采纳的理由 |
| 4 | 替代性解释构成独立小节，论证有力且承认部分替代解释的合理成分 |

**快速检测**：搜索"替代性解释""另一种可能""也有人认为"等段落。

### D6d 局限性的诚实度（维度内权重 20%）

| 分数 | 标准 |
|:----:|------|
| 1 | 无局限性节，或局限性为空话 |
| 2 | 列出 2-3 条常规局限，但未与核心论断关联 |
| 3 | 局限性直接指向核心论断的薄弱环节，承认"什么问题本文回答不了" |
| 4 | 不仅指出局限，还说明"为什么在这些局限下结论仍然成立"——有补救论证 |

**快速检测**：局限性节中每一条是否能对应到论文的某个核心论断？
还是只是泛泛而谈？

## D7 批判性深度 / 写法 B 程度（总权重 5%）

核心问题：**论文是否挑战了常规认知，提供了"不看就不会知道"的洞见？**

### D7a 反直觉发现（维度内权重 40%）

| 分数 | 标准 |
|:----:|------|
| 1 | 全文发现均为"符合预期"（如"经济下行导致裁员"） |
| 2 | 有 1-2 处"有意思但不意外"的观察 |
| 3 | 有明确的反直觉发现——挑战了行业常识或理论预期 |
| 4 | 反直觉发现有数据支撑，且提供了"为什么反直觉但实际合理"的解释 |

**快速检测**：用一句话概括论文核心发现，问一个业内人士"你觉得意外吗？"
不意外 = 1-2 分。

### D7b 因果机制识别（维度内权重 35%）

| 分数 | 标准 |
|:----:|------|
| 1 | 仅描述相关性（A 与 B 同时出现） |
| 2 | 提出了因果方向（A 导致 B），但未解释传导路径 |
| 3 | 识别了因果传导路径（A → 通过 C 机制 → 导致 B） |
| 4 | 因果机制可操作化：指出了干预 A 的哪个环节可以改变 B 的结果 |

**快速检测**：核心发现是否有"因为""导致""机制在于"等因果表述？
该表述是否指向可干预环节？

### D7c 小节标题的论断性（维度内权重 25%）

| 分数 | 标准 |
|:----:|------|
| 1 | 小节标题全为主题标签（如"4.2 公司战略分析"） |
| 2 | 部分标题有论断倾向，但仍偏描述性 |
| 3 | 主要小节标题是论断而非主题（如"4.3.1 边界扩大 ≠ 机制重构"） |
| 4 | 所有核心分析章节的标题读完后，即可还原论文的主要论点 |

**快速检测**：只读第四章所有小节标题（不读正文），能否说出 3 个核心观点？
能 = 4 分。

## 改进优先级排序方法

评分完成后，按以下逻辑排列改进优先级：

1. **P0（必须修复）**：任何子指标 ≤ 1.5 分——这是"硬伤"，不修复会被评审直接质疑
2. **P1（高ROI修复）**：对总分影响最大的短板维度（权重 × 差距）
3. **P2（锦上添花）**：已达 3 分的维度冲击 4 分

优先级公式：

```
改进 ROI = 维度权重 × 子指标维度内权重 × (目标分 - 当前分)
```

按 ROI 从高到低排序，优先修复排名前 3 的子指标。
