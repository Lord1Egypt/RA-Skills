# 市场评估方法论

> 来源：创业方法论综合  
> 用途：系统评估市场规模、竞争格局和进入时机

---

## 一、市场规模评估（TAM/SAM/SOM）

### 1.1 概念定义

```
┌─────────────────────────────────────────────────────────────┐
│  三层市场模型                                                │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│                         ┌─────────────┐                      │
│                         │    TAM      │                      │
│                         │  总潜在市场  │                      │
│                         │ (100%)      │                      │
│                         └──────┬──────┘                      │
│                                │                             │
│                    ┌───────────┴───────────┐                 │
│                    │        SAM            │                │
│                    │   可服务市场          │                │
│                    │   (你的能力范围)       │                │
│                    └───────────┬───────────┘                 │
│                                │                             │
│                    ┌───────────┴───────────┐                 │
│                    │        SOM           │                 │
│                    │   可获得市场          │                 │
│                    │   (实际可获取)         │                 │
│                    └───────────────────────┘                │
│                                                              │
│  示例：在线教育                                              │
│  - TAM：全球教育市场 6万亿美金                                │
│  - SAM：中国K12在线教育 5000亿                               │
│  - SOM：你聚焦的STEAM编程教育 100亿                          │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 计算方法

#### 方法一：自上而下（Top-Down）

```
步骤1：确定总体市场容量
   → 行业报告、政府数据、公开研究

步骤2：识别目标细分
   → 按地域/用户类型/产品类型细分

步骤3：计算可服务比例
   → 基于你的能力范围做减法

示例计算：
─────────────────────────────────────
总体市场：2025年中国预制菜市场 5,000亿
  ├─ B端市场：3,000亿 → 排除（不是你的战场）
  └─ C端市场：2,000亿
      ├─ 一线城市：800亿 → 重点目标
      └─ 新一线城市：600亿 → 潜在扩展
─────────────────────────────────────
TAM：2,000亿（C端预制菜）
SAM：800亿（一线城市C端）
SOM：50亿（你有信心拿到的市场份额）
```

#### 方法二：自下而上（Bottom-Up）

```
步骤1：确定目标用户数量
   → 用户画像 × 可触达渠道

步骤2：确定用户价值
   → 单用户收入（ARPU）× 用户生命周期（LTV）

步骤3：计算市场规模
   → 用户数量 × 用户价值

示例计算：
─────────────────────────────────────
目标用户：一线城市25-40岁白领（8,000万）
  × 目标渗透率：5%（400万）
  × 年消费额：2,400元
─────────────────────────────────────
可服务市场：96亿
预期可获得（5年）：20亿
```

### 1.3 市场评估评分标准

| TAM规模 | 评分 | 说明 |
|--------|------|------|
| ≥100亿 | 16-20分 | 大市场，足以支撑野心 |
| 10-100亿 | 11-15分 | 可行的小而美市场 |
| 1-10亿 | 6-10分 | 细分市场，需要精耕细作 |
| <1亿 | 1-5分 | 市场可能太小 |

| 市场增速 | 调整分 |
|---------|-------|
| 年增速>30% | +2分 |
| 年增速15-30% | +1分 |
| 年增速5-15% | 0分 |
| 年增速<5% | -1分 |
| 负增长 | -2分 |

### 2.3 竞争强度评分标准

| 竞争格局 | 评分 | 判定标准 |
|---------|------|---------|
| 空白市场 | 16-20分 | 无直接竞品，或大玩家不屑于做 |
| 分散市场 | 11-15分 | 多个小玩家，没有绝对垄断 |
| 集中市场 | 6-10分 | 头部2-3家占据60%+市场 |
| 寡头市场 | 1-5分 | 1-2家占据80%+市场，后来者很难 |

| 进入壁垒 | 调整分 |
|---------|-------|
| 高壁垒（资金/技术/资质） | +2分 |
| 中等壁垒 | 0分 |
| 低壁垒（容易复制） | -2分 |


### 3.2 时机判断检查清单

```
时机评估检查表：

【市场准备度】
□ 用户对这个问题有认知吗？
  - 有认知，有解决方案 → 成熟
  - 有认知，没有好方案 → 机会窗口
  - 没有认知，需要教育 → 太早

□ 付费意愿如何？
  - 用户已经在付费 → 成熟
  - 愿意付费但没渠道 → 机会
  - 不愿意付费 → 太早

【基础设施】
□ 支撑这个业务的基础设施完备了吗？
  - 支付系统
  - 物流系统
  - 云计算/CDN
  - 移动设备渗透率

【政策环境】
□ 政策是支持还是限制？
  - 鼓励政策 → +加分
  - 无政策 → 中性
  - 限制政策 → 风险

【资本热度】
□ 资本是否在关注这个赛道？
  - 资本追捧 → 证明市场热度
  - 资本冷淡 → 需要谨慎
```

### 3.3 时机评分标准

| 时机判断 | 评分 | 说明 |
|---------|------|------|
| 市场正在爆发，窗口刚开 | 16-20分 | 最佳时机 |
| 市场已成熟，稳定增长 | 11-15分 | 好时机 |
| 市场尚未成熟 | 6-10分 | 风险较高 |
| 市场红利已过 | 1-5分 | 困难重重 |

## 五、数据来源推荐

| 数据类型 | 推荐来源 |
|---------|---------|
| 行业报告 | 艾瑞咨询、36氪研究院、头豹研究院、麦肯锡报告 |
| 市场规模 | 国家统计局、行业协会、上市公司财报 |
| 竞品信息 | 天眼查、IT桔子、Crunchbase、Product Hunt |
| 趋势判断 | Google Trends、百度指数、微博热搜 |
| 政策信息 | 国务院官网、各部委公告、地方政府政策 |
