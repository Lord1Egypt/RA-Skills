# 山海经童话机

## 概述

这是一个山海经童话故事生成器。用户给出提示词（如主角、场景、主题），技能生成模仿安徒生童话结构的幽默故事，并生成播客音频播放出来。

## 故事风格规范

### 目标受众
- 适合 8 岁左右女孩子听
- 语气温暖、亲切、有童心

### 结构要求（模仿安徒生童话）
1. **开场**：用一句有趣的画面或问题引入，立刻抓住注意力
2. **建立**：介绍主角和世界观，让读者有代入感
3. **转折**：出现一个问题或困难
4. **高潮**：危机时刻（可加入适度神秘/恐怖元素，增加吸引力）
5. **解决**：主角用智慧、勇气或善良化解问题
6. **结局**：温暖的道理或开放式美好结局

### 幽默设计
- 主角的行为和想法要有点傻萌、认真得可爱
- 可以让配角（如配角妖怪）吐槽主角，增加喜感
- 对话要口语化、有节奏感
- 安徒生式的幽默：不是爆笑，而是会心一笑

### 恐怖/惊悚元素（适度）
- 只在故事高潮或转折点出现
- 用"声音从深处传来"、"影子变大了"、"空气突然变冷"等氛围描写
- 不要血腥、不要真正的怪物吓人
- 恐怖过后要有化解，保持温暖底色

### 山海经元素融入
- 主角：精卫、刑天、白泽、鲲、鹏、九尾狐、烛龙、应龙、狡、乘黄等
- 配角：可以用其他山海经妖怪做朋友/帮手
- 保持山海经角色的原始气质（如九尾狐的聪明狡黠、白泽的博学、烛龙的神秘）
- 但赋予角色更丰富的性格和对话

## 播客生成要求

### 工具使用
调用 `create_podcast` 工具生成音频。

### 参数选择
- **mode**：`solo`（单人播客）
- **voice**：
  - 推荐 `大壹`：温暖、有故事感，适合讲童话
  - 也可选 `咪仔`、`流畅女声`
- **user_prompt**：填写完整的故事脚本

### 恐怖元素播音技巧（写在user_prompt末尾提示播音员）
- 讲到恐怖/神秘段落时，稍微压低声音、放慢语速
- 用"停顿"和"低声"制造悬念
- 但结尾要温柔、充满希望，不要真的吓到小朋友

## 工作流程

### 步骤1：理解用户提示词
用户可能给出：
- 主角名称（如"精卫"、"九尾狐小九"）
- 场景（如"东海"、"深山"）
- 主题（如"友谊"、"勇气"）
- 其他要求

如果用户只给了一句话提示词，自己补充完整故事背景和配角。

### 步骤2：生成故事
根据提示词生成完整故事脚本（1500-2500字），包含：
- 有趣的开场句
- 完整的起承转合
- 对话和描写
- 温馨或开放式的结局

### 步骤3：生成播客音频
调用 `create_podcast`，选择合适的音色，将故事转化为音频。

### 步骤4：反馈
告知用户：
- 故事的核心创意
- 播客正在生成中（异步）
- 完成后的使用方式

## 故事示例结构参考

```
【开场】用画面或问题引入
【建立】主角登场 + 她的愿望/问题
【转折】出现困难或意外
【高潮+惊悚元素】危机时刻（可加适度恐怖）
【解决】智慧/勇气/善良化解
【结局】温暖道理 + 开放美好
```

## 边界情况

- 用户没给提示词 → 随机选一个山海经主角，写一个温馨小故事
- 用户指定了具体情节 → 在保持主角设定的基础上融入用户需求
- 音频生成中 → 告知"稍等，音频马上就好"
- 音频生成失败 → 重新生成一次，告知用户
