## 一、每日开工三步走

```
□ 步骤1：打开项目总览看板
   → 看所有项目状态，找到需要推进的项目

□ 步骤2：检查哪个项目可以推进
   → 当前阶段是否完成？
   → 下阶段前置条件是否满足？
   → 对应调用哪个Skill？

□ 步骤3：执行并记录
   → 调用对应Skill
   → 记录输入输出到项目档案
   → 做阶段性考评
   → 更新看板
```

## 三、常见问题处理

**Q：阶段2尽调评分<3.0怎么办？**
A：→ 检查是哪个维度拉低总分
   → 如果是数据不全，补充数据后重新评分
   → 如果是硬伤（技术不靠谱/市场太小），建议终止项目，走终止流程

**Q：项目卡在某个阶段不动了？**
A：→ 检查阻塞原因（缺人/缺资料/缺决策）
   → 记录阻塞原因到风险看板
   → 如果是等待决策，升级到三会治理系统处理

**Q：突然来了紧急新项目？**
A：→ 先快速评估优先级
   → 如果P0，可以并行推进现有项目
   → 如果是P0以上，暂停低优先级项目，先处理急事
