## Description: <br>
Secrets guides agents through end-to-end secrets lifecycle governance, including classification, secure storage, rotation, least privilege, developer ergonomics, audit, and incident response. <br>

This skill is ready for commercial/non-commercial use. <br>

## Use Case: <br>
Developers, security reviewers, and infrastructure teams use this skill to remove hardcoded secrets, design secret storage and injection patterns, plan rotations, and review CI/CD or runtime secret handling. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Users may disclose live credentials, private keys, exact secret paths, or production tokens while asking for help. <br>
Mitigation: Use placeholders for secret types and systems, and do not paste live API keys, passwords, private keys, production tokens, exact secret paths, or other sensitive access details into chat. <br>
Risk: Security workflow recommendations may be applied to production credential systems without enough organization-specific review. <br>
Mitigation: Review storage, access, rotation, and incident response changes with the responsible security or platform owners before changing production secret handling. <br>
