## 科学研究生成要求

科学研究每种任务类型的"零件清单"——必选/可选组件、组装顺序、领域约束。按清单逐项产出。

---

### 1 课题设计

#### T1-1 文献调研

- **必选组件**: 调研范围、检索策略、文献清单、文献综述
- **可选组件**: 文献管理工具、引文分析、研究前沿图
- **组装顺序**: 确定调研范围 → 设计检索策略 → 检索文献 → 筛选文献 → 撰写综述
- **约束**: 范围必须明确；策略必须系统；综述必须全面
- **格式**: 文献综述（Markdown）

#### T1-2 问题定义

- **必选组件**: 研究背景、研究问题、研究目标、研究意义
- **可选组件**: 研究假设、研究范围、研究限制
- **组装顺序**: 分析研究背景 → 提出研究问题 → 明确研究目标 → 阐述研究意义
- **约束**: 问题必须明确；目标必须可衡量；意义必须充分
- **格式**: 问题定义文档（Markdown）

#### T1-3 假设提出

- **必选组件**: 假设内容、假设依据、假设可验证性、假设变量
- **可选组件**: 假设推导过程、假设文献支持
- **组装顺序**: 基于已有知识 → 推导假设 → 明确变量 → 验证可验证性
- **约束**: 假设必须可验证；依据必须充分；变量必须明确
- **格式**: 研究假设文档（Markdown）

#### T1-4 方案设计

- **必选组件**: 研究方法、技术路线、实验设计、进度计划
- **可选组件**: 研究工具、研究资源、风险预案
- **组装顺序**: 选择研究方法 → 设计技术路线 → 制定实验设计 → 安排进度计划
- **约束**: 方法必须匹配问题；路线必须可行；设计必须科学
- **格式**: 研究方案（Markdown）

---

### 2 实验研究

#### T2-1 实验准备

- **必选组件**: 实验材料清单、实验设备清单、实验环境要求、操作规程
- **可选组件**: 预实验、安全培训、设备校准
- **组装顺序**: 列出材料清单 → 准备设备 → 检查环境 → 制定规程 → 预实验验证
- **约束**: 材料必须齐全；设备必须正常；环境必须符合；规程必须详细
- **格式**: 实验准备清单（Markdown）

#### T2-2 实验执行

- **必选组件**: 实验步骤、操作记录、观察记录、异常处理
- **可选组件**: 实验助手、实验监控、实验录像
- **组装顺序**: 确认准备就绪 → 按步骤执行 → 记录操作 → 记录观察 → 处理异常
- **约束**: 步骤必须严格；记录必须完整；异常必须处理
- **格式**: 实验执行记录（Markdown）

#### T2-3 数据记录

- **必选组件**: 数据表格、数据格式、数据备份、数据说明
- **可选组件**: 数据采集工具、数据管理系统
- **组装顺序**: 设计数据表格 → 确定数据格式 → 记录数据 → 备份数据 → 编写说明
- **约束**: 表格必须规范；格式必须统一；备份必须及时
- **格式**: 数据记录表（Markdown/Excel）

#### T2-4 质量控制

- **必选组件**: 质量标准、质量检查、质量问题、质量改进
- **可选组件**: 质量工具、质量审核、质量报告
- **组装顺序**: 确定质量标准 → 执行质量检查 → 发现质量问题 → 实施质量改进
- **约束**: 标准必须明确；检查必须全面；问题必须解决
- **格式**: 质量控制报告（Markdown）

---

### 3 数据分析

#### T3-1 数据清洗

- **必选组件**: 清洗规则、缺失值处理、异常值处理、重复值处理
- **可选组件**: 清洗工具、清洗脚本、清洗日志
- **组装顺序**: 制定清洗规则 → 处理缺失值 → 处理异常值 → 处理重复值 → 验证清洗结果
- **约束**: 规则必须合理；处理必须有依据；结果必须可追溯
- **格式**: 数据清洗报告（Markdown）

#### T3-2 统计分析

- **必选组件**: 分析方法、分析工具、分析结果、结果解释
- **可选组件**: 分析脚本、分析图表、分析报告
- **组装顺序**: 选择分析方法 → 准备分析工具 → 执行分析 → 解释结果 → 生成图表
- **约束**: 方法必须匹配数据；工具必须可靠；解释必须准确
- **格式**: 统计分析报告（Markdown）

#### T3-3 假设验证

- **必选组件**: 验证方法、验证标准、验证结果、验证结论
- **可选组件**: 验证工具、验证图表
- **组装顺序**: 确定验证方法 → 设定验证标准 → 执行验证 → 分析结果 → 形成结论
- **约束**: 方法必须科学；标准必须明确；结论必须客观
- **格式**: 假设验证报告（Markdown）

#### T3-4 结论形成

- **必选组件**: 研究结论、结论依据、结论意义、研究局限
- **可选组件**: 结论讨论、未来研究方向
- **组装顺序**: 综合分析结果 → 提炼研究结论 → 阐述结论依据 → 讨论结论意义 → 指出研究局限
- **约束**: 结论必须基于数据；依据必须充分；局限必须诚实
- **格式**: 研究结论（Markdown）

---

### 4 成果发布

#### T4-1 论文撰写

- **必选组件**: 论文结构（引言/方法/结果/讨论/结论）、论文内容、参考文献、格式规范
- **可选组件**: 论文模板、写作工具、写作指导
- **组装顺序**: 确定论文结构 → 撰写引言 → 撰写方法 → 撰写结果 → 撰写讨论 → 撰写结论 → 整理参考文献
- **约束**: 结构必须规范；内容必须完整；引用必须准确
- **格式**: 研究论文（Markdown/Word）

#### T4-2 同行评审

- **必选组件**: 评审邀请、评审标准、评审意见、评审回复
- **可选组件**: 评审平台、评审流程、评审时间
- **组装顺序**: 邀请评审专家 → 提供评审标准 → 收集评审意见 → 分析意见 → 准备回复
- **约束**: 专家必须相关；标准必须明确；意见必须尊重
- **格式**: 同行评审记录（Markdown）

#### T4-3 论文修改

- **必选组件**: 修改依据、修改内容、修改说明、修改版本
- **可选组件**: 修改对照、修改日志
- **组装顺序**: 分析评审意见 → 确定修改内容 → 执行修改 → 撰写说明 → 版本管理
- **约束**: 依据必须充分；修改必须到位；说明必须清晰
- **格式**: 修改版论文 + 修改说明（Markdown）

#### T4-4 成果发表

- **必选组件**: 目标期刊、投稿材料、投稿流程、发表跟进
- **可选组件**: 发表策略、发表宣传、发表影响
- **组装顺序**: 选择目标期刊 → 准备投稿材料 → 提交投稿 → 跟进审稿 → 处理发表事宜
- **约束**: 期刊必须匹配；材料必须完整；流程必须规范
- **格式**: 发表记录（Markdown）
