# Deep Research Skill V3.1

商业决策级深度调研 Skill——多步骤编排 + 断点强制校验 + Agent 执行指令。

## 这是什么

一个结构化的深度调研流程 Skill，通过编排脚本强制 Agent 按 10 个步骤逐步执行，确保调研报告的质量和可追溯性。

核心设计理念：**不让 Agent 读完所有资料后一次性写报告**，而是通过断点校验和强制检查表，保证每一步都扎实。

## 核心特性

- **编排脚本强制推进**：通过 `orchestrator.sh` 控制执行顺序，不可跳步
- **三个强制断点**：S0（范围确认）、S2（搜索前反思）、S9（交付前自检）
- **信源五级分级**：T1 一手权威 → T5 不可引用，核心论据必须 T1-T2
- **三源印证规则**：核心数据必须有 3 个及以上独立来源
- **仅标注推测**：事实/分析/判断自然行文，仅推测内容标注置信度
- **16 项交付自检**：自动 6 项 + 手动 10 项，全部通过才能交付

## 文件结构

```
deep-research-v3/
├── SKILL.md              ← Agent 阅读的 Skill 定义
├── orchestrator.sh       ← 编排脚本（入口）+ Agent 指令 + 自检
├── steps/                ← 10 个步骤文件（s0-s9）
│   ├── s0.md             ← 确认范围与深度
│   ├── s1.md             ← 提出核心问题
│   ├── s2.md             ← 搜索前反思校验（断点）
│   ├── s3.md             ← 第一轮搜索
│   ├── s4.md             ← 深度抓取
│   ├── s5.md             ← 第二轮验证搜索
│   ├── s6.md             ← 证据整理与信源分级
│   ├── s7.md             ← 展开论述
│   ├── s8.md             ← 交汇洞察
│   └── s9.md             ← 生成结论 + 自检
├── templates/
│   └── report-template.md ← 报告模板
└── rules/
    ├── source-grading.md  ← 信源分级标准
    └── writing-style.md   ← 写作风格规范
```

## 使用方式

### 使用方式

将整个目录放入 Agent 的 skills 目录，Agent 收到调研任务后会自动触发：

```bash
bash orchestrator.sh "调研主题"
```

本 Skill 只定义调研方法论（流程、校验、分级、写作规范），不限定具体搜索工具——Agent 使用当前环境可用的搜索和抓取工具即可。

## 依赖
