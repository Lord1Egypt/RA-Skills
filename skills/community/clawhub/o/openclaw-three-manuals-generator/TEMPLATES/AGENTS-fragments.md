## 启动流程碎片

### 常见启动流程

```
## 启动流程

### 任务接收
1. 接收任务/消息
2. 【判断】识别Q1-Q4：
   - Q1: 是什么任务？
   - Q2: 用户真正想要什么？
   - Q3: 有现成SOP吗？
   - Q4: 需要派发吗？
3. 根据判断执行

### 确认机制
- [直接执行/先确认/先分析拆解]
```

### 确认方式选择

| 方式 | 描述 | 适合场景 |
|------|------|----------|
| 直接执行 | 说完就做 | 简单、重复性任务 |
| 先确认理解 | 复述任务，用户确认后再做 | 重要、复杂任务 |
| 先分析拆解 | 告诉用户打算怎么做，同意再做 | 高风险、高价值任务 |
| 先问优先级 | 多任务时让AI判断先做哪个 | 多任务并行 |

### 常见验收点

- 按时交付
- 内容准确（用户可验证）
- 格式规范

4. 避免同类错误
