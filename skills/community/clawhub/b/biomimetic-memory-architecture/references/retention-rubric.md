# BMA Retention Rubric

The audit estimates lasting value for aged memory archive files.

## High-Value Signals

- User decisions or preferences
- Durable project facts
- Configuration changes with lasting operational meaning
- Failure lessons with root cause
- Reusable workflows or runbooks
- Contacts, commitments, deadlines, or external obligations
- System architecture changes

## Low-Value Signals

- Pure tool output
- Empty debriefs or “no new content” reports
- Repeated dreaming diary or literary reflection
- Old transient status snapshots
- Duplicated material already captured in project/runbook files
- Temporary troubleshooting notes without lasting cause or fix

## Suggested Buckets

- **retain-summary**: keep compressed residue in active memory and cite cold source
- **cold-archive-only**: move source to cold archive; no active residue needed
- **review-manual**: ambiguous; human or stronger model should inspect
- **skip-active**: not old enough or protected path

## Compression Principle

Old memories should shrink aggressively. Preserve the “why it matters later,” not the full event transcript.
