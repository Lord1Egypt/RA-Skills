## T1 任务创建域 范本

### 范本概述
- **域ID**: T1
- **域名称**: 任务创建域
- **任务数量**: 4
- **范本状态**: 可用

### 任务执行范本

#### T1-01 任务识别
**执行步骤**:
1. 接收任务请求
2. 分析任务来源和背景
3. 识别任务类型和特征
4. 确认任务目标和范围
5. 输出任务识别结果

**输入示例**:
- 任务请求：创建一个博客系统
- 任务来源：客户
- 任务背景：客户需要一个个人博客网站

**输出示例**:
- 任务类型：软件开发任务
- 任务特征：中等复杂度，需要前后端开发
- 任务目标：创建一个功能完整的个人博客系统
- 任务范围：包含用户管理、文章管理、评论管理等功能

**注意事项**:
- 任务识别要准确，避免误解
- 任务特征要全面，避免遗漏
- 任务目标要明确，避免模糊

#### T1-02 任务定义
**执行步骤**:
1. 设置任务标题（subject）
2. 编写任务描述（description）
3. 配置可选元数据
4. 验证任务定义完整性
5. 确认任务定义

**输入示例**:
- 任务识别结果：软件开发任务，创建博客系统
- 任务目标：创建一个功能完整的个人博客系统

**输出示例**:
- subject：创建个人博客系统
- description：开发一个包含用户管理、文章管理、评论管理等功能的个人博客系统，支持响应式设计，提供良好的用户体验
- activeForm：正在创建个人博客系统
- owner：开发团队
- metadata：priority=high, deadline=YYYY-MM-DD

**注意事项**:
- subject要简洁明确，避免冗长
- description要详细完整，避免遗漏
- 元数据要符合规范，避免随意

#### T1-03 任务分类
**执行步骤**:
1. 评估任务复杂度
2. 判断任务类型
3. 设定任务优先级
4. 添加任务标签
5. 确认任务分类

**输入示例**:
- 任务定义：创建个人博客系统
- 任务特征：中等复杂度，需要前后端开发

**输出示例**:
- 复杂度分类：复杂（预估10+步骤）
- 任务类型：软件开发任务
- 优先级：高
- 标签：Web开发、全栈、博客系统
- 分类依据：任务涉及多个技术栈，需要完整开发流程

**注意事项**:
- 复杂度评估要客观，避免低估
- 任务类型要准确，避免混淆
- 优先级要合理，避免过高或过低

#### T1-04 任务注册
**执行步骤**:
1. 准备任务注册信息
2. 执行任务注册
3. 确认注册状态
4. 记录注册时间
5. 完成任务注册

**输入示例**:
- 任务定义：创建个人博客系统
- 任务分类：复杂软件开发任务，高优先级

**输出示例**:
- 注册信息：任务ID=TASK-001, 标题=创建个人博客系统, 状态=pending
- 注册状态：成功
- 注册时间：YYYY-MM-DD HH:mm:ss
- 注册确认：任务已成功注册到任务管理系统

**注意事项**:
- 注册信息要完整准确
- 注册状态要及时更新
- 注册时间要准确记录

### 范本特征摘要
- **结构特征**: 按任务流程组织，从识别到注册，步骤清晰
- **风格特征**: 规范、标准、易于操作，使用明确的输入输出示例
- **逻辑特征**: 从任务识别到任务注册，逻辑连贯，步骤明确
- **格式特征**: 使用Markdown格式，包含步骤、输入输出示例、注意事项
