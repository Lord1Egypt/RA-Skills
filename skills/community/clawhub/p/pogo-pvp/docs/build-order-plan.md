# 培养顺序 — build-order 设计文档

## 命令

```
/pvp 培养顺序
```

## 用途

从用户所有联盟的库存中，输出全局最值得优先培养的宝可梦 Top N。

## 输入

- `my_pokemon.json`（全部联盟）
- 各联盟 rankings 数据
- 配队推荐结果

## 排序逻辑

跨联盟统一排序，考虑因素：

### 权重

| 因子 | 权重 | 说明 |
|------|------|------|
| 最高物种排名 | 30% | 该宝可梦在任一联盟的最高排名 |
| 最佳 IV 排名 | 20% | 在所有联盟中最好的 IV 排名 |
| 是否已培养 | 20% | 未培养优先 |
| 是否出现在推荐配队 | 15% | 如果某联盟配队需要它，加分 |
| 覆盖联盟数 | 15% | 能在几个联盟使用（1500/2500/master） |

## 推荐投入文案

| 状态 | 推荐投入 |
|------|---------|
| 未培养 | 星尘 / 糖果 |
| 未培养 + 高排名 + IV好 | 星尘 / 糖果 / XL糖 |
| 缺配招 | 精英学习器 |
| 已培养 | 无（已可用） |

## 合并规则

同一个宝可梦可能有多个联盟条目，合并规则：

- 取最高排名
- 取最佳 IV
- 如果任一联盟未培养 → 标记为未培养
- 列出所有适用联盟

## 实现位置

- `src/buildorder.ts` — `/pvp 培养顺序` 命令逻辑

## 不做

- 不输出具体星尘/糖果数量
- 不分析 XL 糖缺口
- 不做跨物种对比的精确数值（只取排名层）
