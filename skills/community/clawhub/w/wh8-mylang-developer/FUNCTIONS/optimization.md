## 一、枚举

### 适用场景
- 策略研发初期
- 参数范围较小
- 需要了解参数收益分布

### 操作方法
```
主图回测后右键菜单 → 枚举
```

### 枚举原理
- 按步长遍历范围内所有参数组合
- 全排列计算
- 计算量大但全面

### 参数关系设置
```mql
// 示例：两个均线参数
N1 < N2;  // 设置参数关系，减少计算量
```
- 每配置一个参数关系，计算量减少一半
- 设置4个关系，减少到原来的1/16

### 寻找稳定参数
1. 枚举计算后点击【寻找稳定参数】
2. 进入参数分布热力图
3. 颜色渐变越不明显，参数越稳定

## 二、遗传

### 适用场景
- 参数范围较大
- 需要精调参数
- 定期检验模型有效性

### 操作方法
```
主图回测后右键菜单 → 参数优化 → 遗传
```

### 遗传原理
- 模拟自然进化过程
- 遗传、交叉、变异
- 随机全局优化

### 参数设置

| 参数 | 说明 |
|------|------|
| 族群大小 | 初始种群数量 |
| 最大遗传代数 | 迭代次数 |
| 参考标准 | 排序依据（收益率/胜率等） |

### 优势
- 避免陷入局部最优
- 有变异性，覆盖面广
- 计算效率高

## 三、滚动调参

### 适用场景
- 短线震荡类模型
- 需要定期调参
- 评估动态调参效果

### 操作方法
```
主图回测后右键菜单 → 参数优化 → 滚动调参
```

### 参数设置

| 参数 | 说明 |
|------|------|
| 调参间隔 | 多久调参一次 |
| 回测数据长度 | 每次用多长数据优化 |
| 参数优化方法 | 枚举/遗传 |
| 参数筛选标准 | 参考标准 |

### 机制说明
```
示例：调参间隔30日，回测数据长度60日

第1阶段：用前60日数据优化 → 用于后30日
第2阶段：用前60日数据优化 → 用于后30日
...依此类推
```

### 回测报告
- 各阶段合并的总体回测报告
- 分析定期调参方案可行性

## 四、优化原则

### 追求参数高原，而非参数孤岛

```
┌─────────────────────────────────────┐
│           参数高原（好）              │
│   ┌───┬───┬───┬───┬───┐           │
│   │ 6 │ 7 │ 8 │ 7 │ 6 │           │
│   ├───┼───┼───┼───┼───┤           │
│   │ 7 │ 8 │ 9 │ 8 │ 7 │  ← 稳定   │
│   ├───┼───┼───┼───┼───┤           │
│   │ 6 │ 7 │ 8 │ 7 │ 6 │           │
│   └───┴───┴───┴───┴───┘           │
└─────────────────────────────────────┘

┌─────────────────────────────────────┐
│           参数孤岛（坏）              │
│   ┌───┬───┬───┬───┬───┐           │
│   │ 1 │ 2 │ 1 │ 2 │ 1 │           │
│   ├───┼───┼───┼───┼───┤           │
│   │ 2 │ 5 │10 │ 5 │ 2 │  ← 不稳定 │
│   ├───┼───┼───┼───┼───┤           │
│   │ 1 │ 2 │ 1 │ 2 │ 1 │           │
│   └───┴───┴───┴───┴───┘           │
└─────────────────────────────────────┘
```

### 参数优化注意事项

1. **综合长期和短期数据**
   - 用近期数据优化 → 得到短期参数表
   - 用长期数据优化 → 得到长期参数表
   - 取交集 → 长期短期都表现好的参数

2. **避免参数过多**
   - 参数越多，过度拟合风险越大
   - 需要同时满足多个参数条件才有效
   - 市场波动可能导致黑天鹅

3. **评价标准**
   - 盈亏比 × 胜率 > 0.6 可盈利
   - 乘积越大，模型越好
   - 结合夏普比率、回撤等综合评估

### 敏感性测试
```
回测后右键 → 敏感性测试
```
- 分析滑点/手续费变化对收益的影响
- 滑点增大模型失效 → 不适合实盘
- 评估策略实战能力
