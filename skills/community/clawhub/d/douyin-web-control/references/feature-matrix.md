# 抖音网页版 CLI 功能矩阵

`douyin-web` 是抖音网页版的底层动作库。每个命令尽量只做一件小事，输出 JSON，并且页面动作默认附带截图，方便上层 Agent 判断动作是否真的生效。

## 不适合直接放进底层 CLI 的功能

| 功能 | 原因 |
| --- | --- |
| 完整发布视频/图文 | 多步骤创作流程，涉及上传、草稿、校验和账号风险。底层 CLI 只适合做到入口导航，完整发布应单独设计创作者中心工具。 |
| 通知/私信逐条处理 | 涉及隐私，而且更像收件箱工作流，需要单独的消息/通知动作模型。 |
| 创作者数据分析流程 | 有价值，但属于创作者中心能力，不应混进“播放器/浏览控制”底层动作库。 |
| 判断视觉状态是否绝对正确 | 例如爱心是否真的变红、收藏是否真的高亮。这需要视觉/OCR 校验层。CLI 可以负责点击和截图，但不应假装自己一定看懂了视觉状态。 |
| 在桌面自动化环境里精确录到抖音窗口 | AVFoundation 录的是物理屏幕。某些测试环境会把前台拉回自动化工具，所以可以验证 MP4 生成，但不一定能稳定证明录到的是抖音画面。 |

## 下一批值得补的能力

- `follow-author`：关注当前视频作者，并用截图/视觉层确认。
- `copy-link`：打开分享面板并复制当前视频链接。
- `quality VALUE`：选择 `1080P`、`720P`、`540P` 或自动清晰度。
- `later`：点击播放器里的“稍后再看”。
- `pip`：点击“小窗模式”。
- `creator-*`：把创作者中心拆成单独动作组，比如上传入口、作品管理入口、数据入口，但不在底层库里做完整发布流程。
