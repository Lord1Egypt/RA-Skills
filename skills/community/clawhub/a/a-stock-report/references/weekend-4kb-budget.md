# 财经周末版 4KB 字节预算实战记录（v3.4.x）

**实战触发**：2026-06-16 周二 cron 实跑（异常周——本周交易日 Tue 6/9 → Mon 6/15，跳过 Sat/Sun 6/13-6/14 视为周末，5d 数据完整 7 字段）。

**目标**：把 8 条要闻 + 5d 情绪表格 + 解读 + 3 个关注板块 + 免责声明塞进企业微信 webhook `msgtype=text` 的 4096 字节硬上限。

## 关键发现（不是模板里写明的，是实跑摸出来的）

### 1. 标题会被脚本自动覆盖
`send_weekend_news.py` 第 187-192 行强制把首行标题改为 `📰 【财经周末要闻】{sat} - {sun}`，其中：
- `sat = today - 1 day`
- `sun = today`

**今日（2026-06-16 周二）算出来 =** `📰 【财经周末要闻】2026年06月15日 - 2026年06月16日`（66 字节）

**LLM 写的原始标题** `📰 【财经周末要闻】2026年6月14日（周日）`（57 字节）

**delta = +9 字节**。LLM 排版时必须按 66 字节预占位，不能按 57 字节算余量。

### 2. 注入正则不匹配 markdown 表格
`_build_and_push_report()` 第 215 行正则：`•\s*{label}[：:].*?(?=[\n•])`

这只匹配 `• 标签：值` 这种 bullet 写法。**如果 LLM 用 markdown 表格**（v3.4.1 立推荐的格式），注入静默跳过（`injected: 0/7`）。

**含义**：表格里的 7 字段值必须由 LLM 直接从 `/tmp/_weekend_5d.json` 填入，不能指望脚本注入覆盖。脚本注入仅对传统 bullet 排版生效。

### 3. 字节预算实操（3968 字节样本）

| 段 | 字节占比 | 备注 |
|---|---|---|
| 标题 | 66 | 必占 |
| 8 条要闻（标题+摘要+来源+空行） | ~2200 | 每条 ~275B，含 1 空行 |
| 段标题 + 5d 表头 + 7 行 | ~900 | 表格紧凑化（周X 名缩掉、列名精简） |
| 一周解读 | ~210 | ≤ 50 字硬约束的 ~3.5 倍，OK |
| 段标题 + 关注板块 3 条 | ~430 | |
| 免责声明 | ~50 | |

**总 3968 + 9 标题 delta = 3977 字节** ≤ 4096，安全 ~120 字节 buffer。

**踩坑样本**：第一稿 4856 字节（超 758B）→ 第二稿 4194B（仍超 98B）→ 第三稿 4121B（仍超 25B）→ 第四稿 4067B（标题 swap 后 4076B，仍卡边）→ 第五稿 3968B（**达标**）。

### 4. msgtype=text 的隐性约束

`wx()` 内部固定 `msgtype: text`（不是 markdown）。这意味着：
- markdown 表格**不会渲染**，只显示为原始 `| ... |` 文本
- 表情 ✅/❌/⚠️ 可正常显示（Emoji 不在 markdown 语法范围内）
- 4KB 限制是**text 字段的字节长度**，不是渲染后字符数

**含义**：表格只是给用户**可读性**（微信对齐显示），不是真正的 markdown 渲染。优化字节用 markdown 表格（v3.4.1 决策）依然成立。

## 与模板约束的关系

| 模板约束 | 字节预算约束 | 冲突时 |
|---|---|---|
| ≤ 1500 字 | ≤ ~3900 字节 | **字节是硬墙**（4KB 微信），字数是软目标 |
| 6-8 条要闻 | 实测 8 条 ~2200B | 8 条安全，9 条必爆 |
| 每条 ≤ 70 字 | 实测 ~50-60 字 ~270B | OK |
| 关注板块 ≤ 100 字/段 | 实测 ~80 字/段 ~140B | OK |

## 实战建议（下次跑 cron 时）

1. **目标 ≤ 3900 字节**（不是 4096）—— 留 200B buffer 给标题 delta + 编码波动
2. **5d 表格用紧凑列名**：`06/09` 而非 `06/09 周二`、`情绪打分` 而非 `量化情绪打分`、`两融(亿)` 而非 `两融余额(亿)`、`两融占比` 而非 `两融/流通市值`
3. **IF 基差裸值不带"升水/贴水"标签**——升贴水从符号自明（+ 升水 / - 贴水）
4. **关注板块段每条 ≤ 80 字**——3 条 = ~430B 安全
5. **要闻段每条 ≤ 60 字摘要**——8 条 + 空行 = ~2200B
6. **必填免责声明**（v3.3.5+3 fix 的目标）—— 50 字节不可砍

## 异常周（Dragon Boat 等）日期处理

2026-06-09（Tue）→ 2026-06-12（Fri）→ 2026-06-15（Mon）是 5 个交易日，6/13-6/14 视为周末，**6/16 周二 cron 触发**产生"周末"报告。

**脚本标题 swap 行为**：`sat = 6/15`, `sun = 6/16`，生成 `2026年06月15日 - 2026年06月16日` —— 视觉上不是"周末"日期范围但脚本不感知，按 `today-1` 硬算。

**LLM 端的妥协**：把 5d 表格的日期列名按 `MM/DD` 简写（不带周X），让用户自己脑补日期对应星期。
