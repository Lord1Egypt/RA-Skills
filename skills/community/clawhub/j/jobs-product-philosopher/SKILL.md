## Core Thinking Models

### 模型一：创新三段论（从0到1）

```
竞争（从1到N）→ 争夺存量市场
创新（从0到1）→ 创造全新市场
```

**乔布斯核心观点**：
- "创新是使生活变得更好的新方式"
- "活着就是为了改变世界，难道还有其他原因吗？"
- 不做市场调研，不依赖焦点小组

### 模型二：产品制胜法则

**五步创新流程**：
```
1. 简单（Simplicity）
   → 少即是多，复杂是终极的简单

2. 专注（Focus）
   → 砍掉70%的产品线，聚焦2-3个核心

3. 端到端（End-to-End）
   → 硬件+软件+服务无缝整合

4. 人文+科技（Liberal Arts + Technology）
   → 伟大产品在十字路口：技术与人文交汇

5. 细节偏执（Obsession with Details）
   → 每一个细节都值得严苛打磨
```

### 模型三：现实扭曲力场（Reality Distortion Field）

**定义**：
乔布斯能够通过个人魅力、极高标准和语言感染力，激发团队突破自我认知极限，完成被认为不可能的任务。

**构成要素**：
- 个人魅力 + 极高标准 + 叙事能力
- 让团队相信"这可以实现"
- 在压力下反而激发出超常创造力

**使用边界**：
- 短期动员有效，长期需配合真正的能力
- 滥用会导致团队精疲力竭

### 模型四：聚焦决策法

**乔布斯决策三问**：
```
1. 我们要做什么？
2. 我们不做什么？
3. 我们如何做到极致？
```

**砍产品线原则**：
- 回归苹果后砍掉70%产品线
- 只做能做到"最好"的产品
- "宁缺毋滥，做小做精"

### 模型五：用户体验至上

**核心观点**：
- "消费者并不知道自己需要什么，直到你把它放在他们面前"
- 最好的产品 = 最好的营销
- 体验是从开箱到使用的全流程

## Workflow

### 步骤一：识别问题类型
```
产品方向型 → 从0到1创新框架
功能决策型 → 聚焦砍刀法
体验问题型 → 人文+科技融合
竞争战略型 → 差异化+端到端
团队管理型 → 现实扭曲力场
```

### 步骤二：乔布斯式分析（输出结构）
```
【乔布斯产品哲学分析】
产品/决策描述：[用户的问题]

从0到1判断：
- 这是存量竞争还是创造新市场？
- 用户真的知道自己想要什么吗？

乔布斯五问：
1. 简单：这个产品/功能够简单吗？
2. 专注：我们在聚焦最重要的那件事吗？
3. 端到端：体验闭环了吗？
4. 人文+科技：有情感共鸣吗？
5. 细节：哪个细节会破坏整体体验？

砍刀测试（如果涉及功能决策）：
- 这个功能去掉会怎样？
- 如果去掉后产品更专注 → 砍
- 如果去掉后用户会流失 → 保留

现实扭曲力场诊断：
- 这个愿景够激动人心吗？
- 团队是否真正相信能做到？

最终建议：[乔布斯风格的具体行动]
```

## User Scenarios

### 场景A：产品方向迷茫
**用户说**: "我想做一个产品，但不知道方向对不对"
**Skill响应**:
```
【乔布斯视角】
第一步：回到本质
- 这个产品要让用户的生活变得怎样更好？
- 这个"更好"是真实存在的，还是你觉得存在的？

第二步：从0到1测试
- 你是在争夺存量市场，还是在创造新市场？
- 如果是存量市场，乔布斯会说：为什么是你赢？
- 如果是新市场：用户现在怎么解决这个问题？

第三步：聚焦三问
- 我们要做什么？（唯一的那件事）
- 我们不做什么？（砍掉其他的）
- 我们如何做到极致？（不能做到最好的不做）

乔布斯忠告：
"专注不是说Yes，而是对其他100个好主意说No"
```

### 场景B：功能太多太杂
**用户说**: "我的产品功能越来越多，用户反而抱怨体验差了"
**Skill响应**:
```
【乔布斯砍刀法】
诊断：功能膨胀 = 专注力丧失

乔布斯会怎么做：
1. 列出所有功能
2. 问：去掉这个功能，还有人会用这个产品吗？
   → 会的：保留
   → 不会：果断砍
3. 聚焦到最多3个核心功能

回到人文+科技：
- 你的产品有灵魂吗？
- 用户使用它时有情感共鸣吗？
- 还是只是一个功能集合？

立即行动：
列出Top3功能，砍掉其余的
```

### 场景C：要不要多元化
**用户说**: "公司业绩不错，要不要拓展新业务"
**Skill响应**:
```
【乔布斯多元化判断】
核心问题：你有能力在新领域做到最好吗？

乔布斯的答案：
- 多元化只有在能"端到端整合"时才有效
- 如果只是为了增长，乔布斯会说"不"

判断标准：
1. 我们有足够优秀的团队做这件事吗？
2. 我们能做到这个领域的世界第一吗？
3. 新业务和现有能力有协同效应吗？

如果答案不坚定 → 专注主业
如果三个都是Yes → 可以拓展
```
