# First Response Protocol / 首轮响应协议

用途：用户第一次丢来混乱、不完整、有情绪的复盘问题时，控制 Skill 的第一轮反应。目标为少问、先跑、先命中。

## 首轮判断

先把用户输入分成三类：

| 输入类型 | 信号 | 处理 |
| --- | --- | --- |
| 一句话问题 | “如果当年去了北京会怎样” | 最多问 3 个问题；用户不答就跑材料有限版 |
| 中等材料 | 有年份、城市、实际选择、想复盘的假设结果、情绪 | 直接跑中材料版，结尾问 3 个补充问题 |
| 高材料 | 有职业、家庭、现金流、健康、外部环境 | 直接跑深度版，附录保留多视角依据 |

## 问题预算

首轮问题预算按材料量决定：

| 材料量 | 允许先问 | 交付规则 |
| --- | ---: | --- |
| 一句话 | 1-3 个问题 | 用户不答或表达不想填时，立刻材料有限版运行 |
| 中等材料 | 0 个问题 | 先交付中材料报告，3 个补充问题放结尾 |
| 高材料 | 0 个问题 | 直接深度版，补充问题放继续解释段 |

硬规则：

- 用户已经给出年份、实际选择、想复盘的假设结果和当前纠结时，禁止先追问。
- 用户表达“别问太多”“先跑”“懒得填”“直接来”时，禁止继续问问题。
- 补充问题只用于升级报告，不能阻塞第一版报告。
- 首轮所有问题合计最多 3 个，不能拆成多段连续追问。

## 首轮必须先做的内部动作

用户没给完整材料时，也要先内部识别：

1. 岔路动作。
2. 系统切换。
3. 主张力轴。
4. 主原型。
5. 隐秘交换。
6. 首屏判词草稿。

## 用户拒绝填表时

用户说“懒得填”“先跑”“别问太多”“你直接来”时，使用这个回应：

```text
可以，我先跑材料有限版。下面会把已知事实和关键假设分开，未确认内容只按情景处理。结尾给你 5 个最值得补的信息，你后面只补 1-2 个也能升级报告。
```

然后直接输出：

- 首屏判词。
- 一句人话张力判断。
- 最像哪类人生遗憾。
- 三种结果。
- 4 张轻量事件冲击卡。
- 30 天验证实验。
- 5 个补充问题。

## 首轮最多问 3 个问题

只有在用户没有拒绝补充时才问。问题必须高信息密度：

```text
我先问 3 个最关键的：
1. 当年大概是哪一年，你几岁，在哪个城市？
2. 你最后选了什么，最想复盘哪个没走的选择？
3. 今天反复想起它，最卡的是职业、钱、关系、健康、身份，还是不甘？
```

如果用户已经给出中等材料，跳过这三个问题，直接输出报告，并在报告结尾写：

```text
如果你愿意继续校准，只补这 3 个信息就够：
1. {{最影响判词的信息}}
2. {{最影响事件冲击卡的信息}}
3. {{最影响 30 天实验的信息}}
```

## 首轮禁区

- 不要求用户填 intake 长表。
- 不展示 Agent、质量评分、矩阵、事件库。
- 不先讲 Skill 能力。
- 不输出传播稿。
- 不用命理化表达。
- 不把猜测写成事实。

## 材料有限版首屏要求

材料有限版判词也必须通过 Magic Score：

- 有承载物。
- 有年份或阶段。用户没给年份时，用“如果发生在 2015-2019 平台增长期”这种条件性写法。
- 有隐秘交换。
- 有可回收资产。

例：

```text
如果那次选择发生在 2015-2019 的平台增长期，北京在你心里就像一张更早上牌桌的入场券。你想问“会不会更好”，其实在问自己有没有把更早被看见的训练，换成更稳、更少折腾的生活。
```

## 升级规则

用户后续只补 1-2 条信息时，优先升级：

1. 判词。
2. 事件冲击卡。
3. 30 天验证实验。

不需要重写完整报告，除非用户要求深度版。
