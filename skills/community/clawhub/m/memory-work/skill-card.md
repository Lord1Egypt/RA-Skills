## Use Case: <br>
External users and developers use Memory Work to run a local, AI-assisted personal knowledge vault that structures weekly work, maintains long-term memory files, and helps archive or evolve reusable materials and skills. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Broad local vault access may expose or modify personal memory, profile, weekly workspace, material, tool, and skill files. <br>
Mitigation: Install it only in a dedicated vault that the user is comfortable letting an agent read, summarize, and update. <br>
Risk: Some actions may lack clear consent boundaries, especially deletes, archives, cross-zone searches, and persistent memory or profile writes. <br>
Mitigation: Require explicit confirmation for deletes, archives, cross-zone searches, and persistent profile or memory writes before use. <br>
Risk: The artifact makes a sensitive-access audit claim that security evidence says is unsupported. <br>
Mitigation: Remove or correct the unsupported claim before publication or installation guidance. <br>
