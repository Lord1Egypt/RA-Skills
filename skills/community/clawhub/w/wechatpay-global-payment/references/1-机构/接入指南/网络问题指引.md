# 境外网络接入指引

本文档介绍**机构**境外接入微信支付的**域名选择**、**加速点**、**部署建议**，以及**网络问题排查**。

---

## 1. 可用的 API 域名

境外业务统一使用**境外域名 + `/v3/global/` 路径前缀**。

### 1.1 主域名

| 域名 | 部署位置 | 适用 |
|------|--------|------|
| `apihk.mch.weixin.qq.com` | **香港加速点（默认）** | ✅ 境外业务**强烈推荐**，所有 `/v3/global/` 接口走此域名 |

### 1.2 其他可用加速域名

部分场景可使用其他加速点：

| 域名 | 加速点 | 适用区域 |
|------|--------|---------|
| `apius.mch.weixin.qq.com` | 美国 | 美洲（如机构服务器在美洲） |
| `apieu.mch.weixin.qq.com` | 欧洲 | 欧洲 / 中东 / 非洲 |

> ⚠️ **境外业务不要用境内域名** `api.mch.weixin.qq.com`，可能会存在网络不稳定，请求耗时增加的情况。

### 1.3 选择建议

```
机构服务器部署位置
    │
    ├── 全球区域均可使用 → apihk.mch.weixin.qq.com（默认）
    │
    ├── 美洲 → apius.mch.weixin.qq.com
    │
    └── 欧洲 / 中东 / 非洲 → apieu.mch.weixin.qq.com
```

## 2. 域名的功能

### 2.1 请求路由
所有境外加速域名最终都指向同一套微信支付境外后端服务，功能完全相同：
- 接口能力：完全一致
- 数据：完全一致（不同域名同一订单可见）
- 证书与 APIv3 密钥：同一套（境外体系，与境内独立）

### 2.2 为什么需要多域名
- **降低延迟**：跨境网络延迟高，加速点就近部署
- **灾备**：单域名故障时可切换
- **合规**：部分地区有网络访问限制

## 3. 部署架构建议

### 3.1 单区域部署

```
机构服务器（某地）
    │
    │ HTTPS
    ▼
最近的境外加速域名（如 apihk）
    │
    │ 微信内部专线
    ▼
微信支付境外后端
```

### 3.2 多区域部署（大型机构）

```
亚太机构节点 ───► apihk.mch.weixin.qq.com
欧洲机构节点 ───► apieu.mch.weixin.qq.com
美洲机构节点 ───► apius.mch.weixin.qq.com
          │
          │ 同一微信支付境外后端
          ▼
      微信支付主服务（数据一致）
```

建议：多区域部署时，**每个区域用对应的加速域名**，获得最佳延迟。

### 3.3 故障切换

建议欧洲或北美机构实现**自动切换**：
```java
// 伪代码
String[] domains = {
    "apihk.mch.weixin.qq.com",  // 首选
    "apieu.mch.weixin.qq.com"   // 备选
};

for (String domain : domains) {
    try {
        return callApi(domain);
    } catch (NetworkException e) {
        log.warn("Domain {} failed, trying next", domain);
    }
}
throw new RuntimeException("All domains failed");
```

## 4. IP 白名单（出口 + 入口 IP 列表）

> ✅ **微信支付境外业务提供官方 IP 白名单**，用户合规要求需要 IP 白名单时可直接使用以下列表。

### 4.1 出口 IP（微信支付 → 机构服务器，回调使用）

机构需在防火墙的**入站规则**中放行这些 IP，以确保能收到微信支付的异步回调（支付通知 / 退款通知 / 委托代扣通知 / H5 权限审核通知 等）。

**📚 官方权威列表**：[Outbound IP List](https://pay.weixin.qq.com/doc/global/v3/en/4016687294)

### 4.3 部署建议

- **全列表配置**：建议机构对以上列表内IP均配置进白名，以符合容灾要求。微信支付会在部分IP出现故障时调整IP路由，若未将全部IP配置进白名单则可能出现IP访问不通的情况
- **白名单 + 域名解析**：实际请求时仍通过域名 `apihk.mch.weixin.qq.com`（DNS 会解析到这些 IP，默认情况下会根据机构服务区地址就近解析），白名单只在防火墙层面控制

### 4.4 IP 与域名的对应关系

| 域名 | 对应入口 IP |
|---------|------------|
| `apihk.mch.weixin.qq.com` | 全部IP|
| `apius.mch.weixin.qq.com` | 美国硅谷 IP（`43.130.30.190` / `43.130.30.86`）|
| `apieu.mch.weixin.qq.com` | 法兰克福 IP（`43.131.7.109` / `162.62.97.190`）|


#### 入站防火墙规则（微信支付 → 机构 notify_url）

参照 [官方出口 IP 列表](https://pay.weixin.qq.com/doc/global/v3/en/4016687294)，将所有出口 IP 加入入站白名单，开放 **443 端口**给这些 IP 访问 `notify_url` 路径。

### 4.6 注意事项

1. **白名单 ≠ 安全保障**：即使配置了 IP 白名单，仍**必须**做 V3 签名验证 + AEAD_AES_256_GCM 解密 + 幂等去重 — 这是微信支付安全模型的**核心**，IP 白名单只是网络层的辅助
2. **IP 列表会更新**：建议机构每月对照一次官方文档，及时同步最新 IP
3. **不要把 IP 当作长期不变的常量硬编码**：实际调用仍走域名 + DNS 解析，IP 白名单只用于防火墙层
4. **如发现接口连接超时但 DNS 正常**：可能是新 IP 未加入白名单，先临时放开调试，再向微信支付获取最新列表

## 5. 回调 URL 配置

### 5.1 回调地址要求

- 必须是 **HTTPS**
- 必须是**公网可访问**
- **不能带参数**（URL query string）
- 域名可以是境外域名
- 建议配置**备份 URL**

### 5.2 典型问题

| 问题 | 原因 | 解决 |
|------|------|------|
| 微信服务器访问不到回调 URL | 防火墙拦截微信 IP | 开放 80/443 端口给所有 IP，按域名而非 IP 限流 |
| 回调 5 秒超时 | 机构处理太慢 | 先验签 → 异步队列 → 立即应答 |
| SSL 证书错误 | 使用自签证书 | 使用 CA 签发的有效证书 |
| HTTP 重定向到 HTTPS | 微信可能不跟随 | 直接配 HTTPS URL |
| 反向代理过滤了签名头 | Nginx / CDN 默认丢弃 `Wechatpay-*` 自定义头 | 显式放行 4 个签名头：`Wechatpay-Signature` / `Wechatpay-Timestamp` / `Wechatpay-Nonce` / `Wechatpay-Serial` |

### 5.3 回调可达性测试

在配置前可用以下方式验证 URL 可达性：
```bash
# 从境外服务器测试（如 AWS 东京节点）
curl -X POST https://your-domain.com/wechat/callback -v
```

## 6. 网络问题排查

### 6.1 症状：调用接口超时

**检查清单**：
- [ ] DNS 解析是否正常
- [ ] 是否选择了**最近的加速域名**（默认 apihk）
- [ ] 路径前缀是否带 `/v3/global/`（境外特有）
- [ ] 服务器出口网络是否通畅
- [ ] 防火墙是否放行 443 端口
- [ ] HTTP Client 超时配置（建议连接超时 5s，读超时 30s）
- [ ] 服务器时间与 NTP 同步（误差 > 5 分钟会签名失败）

**诊断命令**：
```bash
# DNS 解析
dig apihk.mch.weixin.qq.com

# 连接测试
curl -v https://apihk.mch.weixin.qq.com/v3/global/certificates \
  -H "Authorization: WECHATPAY2-SHA256-RSA2048 ..."

# 网络延迟
ping apihk.mch.weixin.qq.com
traceroute apihk.mch.weixin.qq.com

# TCP 连通性
nc -zv apihk.mch.weixin.qq.com 443
```

### 6.2 症状：SSL 握手失败

**可能原因**：
- 机构服务器 SSL 版本过低（必须支持 TLS 1.2+）
- 证书链不完整
- 机构本地时间与服务器差异过大（> 5 分钟）

**诊断**：
```bash
# 检查 SSL 握手
openssl s_client -connect apihk.mch.weixin.qq.com:443 -tls1_2
```

### 6.3 症状：间歇性 502 / 503

**原因**：
- 微信侧临时故障
- 机构 → 微信之间网络抖动
- 被 CDN 拦截（如机构用 CF 等中间代理）

**处理**：
- 实现**重试逻辑**（指数退避）
- 收到 5xx 时重试 2-3 次
- 仍失败则记录日志，后续兜底查单

### 6.4 症状：回调收不到

**排查步骤**：
1. **可达性**：用 `curl` 从公网测试回调 URL
2. **SSL**：确保证书有效
3. **应答速度**：服务器日志看处理耗时（必须 < 5s）
4. **防火墙**：检查是否拦截了微信 IP
5. **格式**：响应必须是 `{"code":"SUCCESS","message":"成功"}`
6. **状态码**：200 + 正确的 body 才算成功应答
7. **签名头丢失**：检查反向代理 / WAF 是否吞掉了 `Wechatpay-*` 头

## 7. 常见问题

### Q1：境外业务可以用境内主域名 `api.mch.weixin.qq.com` 吗？
**A**: **不可以**。境外接口路径是 `/v3/global/...`，境内域名不识别这个前缀。

### Q2：不同加速域名返回的数据一致吗？
**A**: 完全一致。后端是同一套服务，只是加速点不同。

### Q3：能否硬编码加速域名的 IP？
**A**: **代码中不推荐硬编码 IP**（应通过域名 + DNS 解析），但**防火墙白名单层面可以使用 §4.2 提供的官方 IP 列表**。两者区别：
- ❌ 不要在代码里 `connect(43.154.252.25, 443)` — 应该 `connect(apihk.mch.weixin.qq.com, 443)`
- ✅ 防火墙规则按 IP 配置 OK — 每月对照 §4.2 官方列表更新即可

### Q4：用 CDN 代理微信支付 API 可以吗？
**A**: **不推荐**：
- CDN 可能修改请求/响应，影响签名
- 增加故障点
- 建议直接调用微信域名

### Q5：回调地址可以用非标准端口吗（如 8443）？
**A**: 不推荐。微信默认只访问 **443 端口**。非标准端口可能被拦截。

### Q6：微信支付有双活、灾备吗？
**A**: 有。微信支付本身实现了多地域多机房，机构只需调加速域名即可。

### Q7：境外服务器调用境内域名走专线还是公网？
**A**: 走公网。微信支付未提供公网外的专线接入。
