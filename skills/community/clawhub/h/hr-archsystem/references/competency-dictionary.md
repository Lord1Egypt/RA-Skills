# 胜任力模型词典

## 输入与输出

**前置依赖：** job-level 产出的职级序列表
**输入：** 职类序列、关键岗位清单、高绩效员工样本（BEI访谈）
**输出：** 《胜任力词典》——能力维度表 × 分级标尺，供 job-evaluation 使用

---

## Step 1：胜任力建模整体流程

```
Step 1: 选取样本岗位（每个序列选2-3个代表性岗位）
Step 2: BEI访谈高绩效员工（每个岗位至少3人）
Step 3: 能力编码（从访谈文本中提取行为指标）
Step 4: 能力维度整合（归类合并，删除重叠）
Step 5: 定义分级标尺（每项能力4-5级描述）
Step 6: AI协同能力等级内置（每级都要有）
Step 7: 验证词典（与现有高绩效员工匹配测试）
```

## Step 2：能力维度选择框架

### 2.1 通用能力（所有序列、所有职级）

| 能力维度 | 定义 | 为什么必要 |
|---------|------|---------|
| 批判性思维 | 基于证据进行逻辑推理，识别假设和偏见 | AI时代信息过载，判断力比记忆更重要 |
| 持续学习 | 主动获取新知识并转化为行动 | AI加速知识更新，半衰期缩短 |
| 人际协作 | 与他人有效合作，包括跨层级跨部门 | 组织的基础设施 |
| 沟通表达 | 清晰传递信息，并根据受众调整 | 专业价值的放大器 |
| 问题解决 | 定义问题、分析根因、设计并执行解决方案 | 专业序列的核心交付 |
| 结果导向 | 设定目标、持续追踪、交付可衡量成果 | 组织存在的理由 |
| 适应韧性 | 在不确定性中保持效能，快速调整 | VUCA时代的生存能力 |

### 2.2 按序列的差异化能力

| M管理序列额外要求 | P专业序列额外要求 | T技术序列额外要求 |
|-----------------|-----------------|-----------------|
| 团队领导 | 专业深度 | 技术敏锐度 |
| 战略规划 | 创新贡献 | 细节精确 |
| 决策担当 | 知识传播 | 系统思维 |
| 资源调配 | 跨团队影响 | 工具效率 |

### 2.3 AI时代新增能力（所有序列都要）

| AI相关能力 | 定义 | Level 1-5 简述 |
|-----------|------|------------|
| AI工具应用 | 选择、配置、使用AI工具提升效能 | L1:基础→L5:系统集成 |
| 人机协作判断 | 知道什么时候用AI，什么时候不用 | L1:被动→L5:战略配置 |
| AI输出评估 | 批判性审视AI生成内容的准确性和适用性 | L1:信任→L5:审计优化 |
| AI伦理意识 | 理解AI的偏见局限和合规要求 | L1:了解→L5:风控把关 |

## Step 3：BEI访谈方法

### 3.1 访谈框架

BEI（Behavioral Event Interview）基于"STAR"原则：

```
S - Situation（情境）: 当时是什么情况？
T - Task（任务）: 你面临的任务/挑战是什么？
A - Action（行动）: 你具体做了什么？为什么？
R - Result（结果）: 结果如何？你学到了什么？
```

### 3.2 访谈问题示例（高绩效样本）

```
针对"问题解决"能力的访谈问题：
1. 给我讲一个你遇到过的最复杂的问题，当时你怎么处理的？
2. 什么时候你需要在信息不完整的情况下做决策？你怎么做？
3. 讲一个你最初解决方案失败但最终成功的例子。
4. 你用什么方法确保你解决了问题的根本原因而不是表面症状？

针对"AI协作"的补充问题：
5. 你目前使用哪些AI工具？你如何判断什么任务适合用AI？
6. 举一个你发现AI输出有问题的例子，你是怎么处理的？
7. 你的工作中哪些部分你觉得AI永远无法替代？
```

### 3.3 每岗位建议样本量

```
最少：3人（高绩效）+ 1人（中等绩效，用于对比）
最佳：5人（高绩效）+ 2人（中等绩效）
```

## Step 4：能力编码与整合

### 4.1 编码流程

从访谈文本中提取"行为片段"，每个片段标注：
- 对应的能力维度
- 该行为的复杂度/影响范围
- 出现频率

### 4.2 能力整合原则

```
合并条件：
- 两个能力在>60%的访谈中表现相同行为 → 合并
- 两个能力在>80%的岗位上都要求 → 保留更通用的

删除条件：
- 只有<20%的访谈样本提到该能力 → 删除或降为次要能力
- 该能力无法用行为区分高绩效和中等绩效 → 删除
```

### 4.3 典型岗位的能力数量控制

```
建议数量：
- 每个岗位 6-10 项能力（含3-5项通用 + 3-5项专业）
- 能力过多 → 评价时权重分散，失去焦点
- 能力过少 → 无法区分不同岗位的差异
```

## Step 5：分级标尺设计

### 5.1 分级原则

每项能力的分级描述必须满足：
1. **行为锚点** — 有具体可见的行为描述，不是程度词
2. **递进清晰** — 级别之间的差异是质变，不是量变
3. **可观察** — 他人能识别和评估该行为
4. **AI感知** — 每级都要标注AI协同行为（新增要求）

### 5.2 分级模板

```
能力名称：[能力维度名称]
能力定义：[一句话定义]

Level 1 - 基础
  行为描述：
  AI协同表现：

Level 2 - 发展
  行为描述：
  AI协同表现：

Level 3 - 熟练
  行为描述：
  AI协同表现：

Level 4 - 精通
  行为描述：
  AI协同表现：

Level 5 - 卓越
  行为描述：
  AI协同表现：
```

### 5.3 完整示例："问题解决"能力

```
能力名称：问题解决
能力定义：通过系统性地定义、分析和解决问题，交付可衡量成果的能力。

Level 1 - 基础
  行为描述：识别明确的、重复出现的问题；在指导下执行已知解决方案；
            将无法解决的问题上报给相关人员。
  AI协同表现：使用AI工具（如ChatGPT）查询常见问题的标准答案；
              对AI给出的建议进行基础验证。

Level 2 - 发展
  行为描述：独立分析问题的多个可能原因；提出2-3个可行方案并做简要对比；
            在标准流程内优化解决方案。
  AI协同表现：能够编写有效Prompt获取问题分析框架；
              用AI辅助收集数据，但自己判断哪些数据相关。

Level 3 - 熟练
  行为描述：定义复杂、非结构化问题，主动收集信息；设计并执行新解决方案；
            评估方案效果并迭代；影响他人认同问题优先级。
  AI协同表现：使用AI做根因分析，并能识别AI分析中的逻辑漏洞；
              训练或微调过领域专用AI工具。

Level 4 - 精通
  行为描述：解决跨部门、跨系统的复杂问题；建立问题解决的方法论和工具；
            推动组织层面的流程改进；预防未来类似问题的发生。
  AI协同表现：设计人机协作的问题解决流程，明确AI和人的分工边界；
              建立团队AI工具使用规范并推广。

Level 5 - 卓越
  行为描述：解决没有先例的战略性难题；创造新的问题解决范式；
            在行业内分享方法论；培养他人的问题解决能力的系统性方法。
  AI协同表现：主导AI问题解决系统的设计（如专家系统、知识库）；
              对AI问题解决能力进行伦理和安全审查。
```

## Step 7：词典维护机制

```
更新触发条件：
- 每半年一次例行review
- 业务战略重大调整
- AI工具重大变化（出现新一代工具）
- 评价数据发现能力要求与实际脱节

更新流程：
评估新版词典 → 校准会审议 → 更新词典版本号 → 通知使用者
```
