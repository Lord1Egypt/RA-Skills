# Agentic Development Invoice Positioning

## Service Language

Use invoice descriptions that match the underlying agentic development offer:

- Agentic workflow discovery and mapping
- Pilot architecture and implementation
- Tool routing and retrieval setup
- Approval-gate design
- Human-in-the-loop workflow configuration
- Evaluation cases and regression examples
- Prompt-injection and tool-permission checks
- Logging and monitoring setup
- Documentation and administrator handoff
- Post-handoff support and managed monitoring
- Change-order implementation for additional workflows, integrations, or deployment targets

## Billing Categories

Professional services:

- discovery
- workflow design
- prototype implementation
- evaluation and test design
- integration work
- documentation
- training
- support
- advisory review

Pass-through or reimbursable:

- API usage
- model usage
- hosting
- storage
- third-party software
- travel
- printing or mailing
- procurement performed with written approval

Adjustments:

- discounts
- credits
- refunds
- late fees
- partial payments
- holdbacks

## Boundaries

Do not imply:

- regulated compliance certification
- autonomous production decisioning
- legal, tax, accounting, medical, financial, employment, housing, credit, biometric, surveillance, or safety-critical approval
- guaranteed savings or performance outcomes

Useful wording:

- "bounded agentic workflow pilot"
- "human approval gates"
- "evaluation examples and acceptance review"
- "non-production prototype unless production deployment is separately authorized"
- "support period per agreement"
- "third-party charges billed at cost unless otherwise agreed"

## Line Item Style

Good line items are specific and auditable:

- "Discovery workshop and workflow map for support triage pilot"
- "Prototype implementation: intake classification, retrieval, draft response, approval queue"
- "Evaluation set: 25 representative examples and regression checklist"
- "Handoff documentation and administrator walkthrough"
- "Post-handoff support retainer, June 2026"

Avoid vague line items:

- "AI stuff"
- "Consulting"
- "Automation"
- "Miscellaneous"

## Payment Note Defaults

Use only if the user has not supplied their own terms:

- "Payment due according to the governing agreement or accepted statement of work."
- "Please reference the invoice number with payment."
- "Third-party pass-through charges are billed only when authorized and documented."
- "Taxes, if applicable, should be calculated according to the payer's verified tax setup."
