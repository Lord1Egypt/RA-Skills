# 订单状态流转

## 订单状态一览

| 状态 | 说明 | 是否终态 |
|------|------|----------|
| **SUCCESS** | 支付成功 | ✅ 是 |
| **REFUND** | 转入退款 | ✅ 是 |
| **NOTPAY** | 未支付 | ❌ 可关单 |
| **CLOSED** | 已关闭 | ✅ 是 |
| **PAYER_ERROR** | 用户支付失败 | ❌ 可重试 |

## 状态流转图

```
创建订单
    │
    ▼
NOTPAY（未支付）
    │
    ├───[用户取消/超时]──→ CLOSED（终态）
    │
    ├───[支付失败]──────→ PAYER_ERROR（可重试）
    │
    └───[支付成功]──────→ SUCCESS（终态）
                              │
                              └───[全额退款]──→ REFUND（终态）
                              │
                              └───[部分退款]──→ SUCCESS + 有退款记录
```

## 各支付方式订单有效期

| 支付方式 | 订单有效期 | 过期后 |
|----------|------------|--------|
| JSAPI/小程序 | 2小时（prepay_id） | 需重新下单 |
| Native | 无明确过期时间 | 建议24小时内 |
| APP | 2小时（prepay_id） | 需重新下单 |
| H5 | 5分钟（h5_url） | 需重新下单 |
| 付款码 | 无 | 扫一次即消费 |

## 订单状态查询

### 商户订单号查单
```
GET /v3/pay/transactions/out-trade-no/{out_trade_no}?mchid={商户号}
```

### 微信订单号查单
```
GET /v3/pay/transactions/id/{transaction_id}?mchid={商户号}
```

### 合单订单号查单（服务商）
```
GET /v3/combine-transactions/out-trade-no/{combine_out_trade_no}
```

## 关闭订单

```
POST /v3/pay/transactions/out-trade-no/{out_trade_no}/close
```

**适用场景**：
- 用户取消支付
- 订单超时未支付
- 商品售罄需要关闭

**注意**：
- 已支付订单无法关闭
- 关单后订单金额会释放
- 关单是幂等操作，重复关单返回成功

## 回调通知状态

微信支付会在支付成功、退款完成时发送回调通知。

| 通知类型 | 触发时机 |
|----------|----------|
| **支付成功回调** | 用户支付成功后 |
| **退款成功回调** | 退款审核通过并完成 |
| **退款失败回调** | 退款失败 |

**回调报文解密后**：
- 基础支付：`trade_state` 字段表示支付状态
- 合单支付：`sub_orders[]` 数组，每个子单有自己的状态
