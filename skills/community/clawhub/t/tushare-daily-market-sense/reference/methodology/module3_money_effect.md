# 模块 3：赚钱效应与上涨主线

使用 `module3_money_effect.json`。

候选池：`pct_chg >= 7%` 且成交额 `>= 2 亿`，按成交额降序排列。

工作步骤：

1. 概括候选数、合计成交额、成交额占比、收益中位数和近似涨停数。
2. 按业务事实、产品环节、事件催化和同步量价行为归纳临时主题，作为内部判定步骤。
3. 每组至少 3 只股票；不足 3 只归入“其他/孤立强势股”，但不要输出单独的主题分组陈列表。
4. 每组计算代表股、入选数、组合计成交额和组内成交额占比；这些指标只进入主线判定、领导股/弹性股和总结句。
5. 评定主线确认度：
   - ★★★：组成交额占比 >= 30%，5 日涨幅中位 > 0，5 日相对收益中位 > 0，连续放量天数中位 >= 2。
   - ★★：满足 4 条中的 3 条。
   - ★：满足 2 条或更少，或缺少组内扩散。
6. 上涨主线表只展示 ★★ 与 ★★★ 主题；★ 主题只用于“资金轮动/局部异动”结论，不进入主线表。
7. 评定位置判断：基于组内代表股的 `close_position_120d`、`drawdown_120_high`、`ret_5d`、`rel_ret_5d`、`sustained_volume_days_5` 综合判断，只输出“高位趋势”或“低位启动”。无法充分证明低位启动时，默认写“高位趋势”。
8. 对每个 ★★ / ★★★ 主线做组内分层：列出领导股与弹性股。领导股优先看成交额承载、市场辨识度、产业链核心地位和收盘承接；弹性股优先看主线相关性、换手/放量活跃、5 日相对强、收盘承接、市值不过分笨重、股性活。

组内分层口径：

- 领导股不是涨幅最高股，而是最能承载主线共识的股票。优先级为：组内成交额厚度 > 市场辨识度 > 产业链核心地位 > 收盘承接。
- 弹性股不是杂毛补涨股，必须仍属于该主线。优先选择 `elasticity_hint_score` 较高、`turnover_rate`/`amount_ratio_20d` 活跃、`rel_ret_5d` 为正、`close_to_high` 较好的样本。
- 先后强弱基于成交额、放量、相对收益、换手率、收盘贴近日高和 120 日位置等日线字段判断。
- 产业链地位不能由脚本硬判。模型只能基于可确认的主营、公告、公开业务事实或常识性产业链位置判断；不确定时写“产业链地位待核验”。

写作要求：

- 成交额占比是第一门槛；不要因为股票名称看起来有想象力就提升低成交额主题的评级。
- 主题分组是内部工作，不作为独立小节输出；报告从赚钱效应总览后直接进入主线判定。
- 候选稀疏或没有主题达到足够成交额厚度时，定性为资金轮动，而不是确认主线。
- 若没有 ★★ 或 ★★★ 主题，主线表下写“无二星/三星主线，赚钱效应偏资金轮动”。
- 输出清晰的 ★★★ 领导股名单，供最终聚合做交叉检查；若没有 ★★★ 主线，写“今日无三星主线领导股”。
- 每个 ★★ / ★★★ 主题最多列领导股 1-3 只、弹性股 1-3 只；如果证据不足，不要硬凑数量。
