# 📊 PPT 智能架构师 —— 政务数字化内容代写引擎 (ppt-skill)

> **面向政务数字化建设领域，不只是指导你怎么做，直接帮你写每页 PPT 的完整文案。**
> 专注三大场景：给政府客户推介产品方案 / 编写规划建设方案 / 汇报项目建设进展

## 🎯 功能简介

面向政务数字化领域的售前顾问、解决方案架构师、项目经理，**直接代写每一页 PPT 的完整文案** + 提供政务风视觉设计指导：

- **📝 内容代写（核心）**：根据你的需求和素材，直接写出每页的标题、要点、数据陈述——**你只需要复制粘贴到 PPT 中**
- **🧠 政策对齐**：自动关联国家/省/市级政策文件，确保方案与政策方向一致
- **🏛 政务语言规范**：使用政务场景标准用语（群众/服务对象、集约建设、赋能、协同联动）
- **🔒 安全合规融入**：等保、密评、数据安全、信创适配贯穿方案全文
- **🎨 政务风视觉指导**：配色（红蓝主色调）、字体（微软雅黑安全方案）、图表（扁平简洁）
- **🎙 演讲赋能**：逐页备注文案 + 过渡语 + 评审 Q&A 预判，可直接复制到 PPT 备注栏

## 🏢 适用场景

| 场景 | 典型受众 | 页数 | 核心目标 |
|:---|:---|:---:|:---|
| **产品/方案推介** | 区县/市领导、局办负责人、业务科室 | 12-15 页 | 获得认可，建立信任 |
| **规划/建设方案** | 评审专家、业主单位、财政局 | 18-22 页 | 通过评审，获得立项 |
| **项目建设汇报** | 甲方分管领导、监理、项管办 | 10-12 页 | 展示进展，获取信任 |

## 🚀 快速开始

### 1. 明确需求

告诉 AI 你的 PPT 类型和目标：

> "我要做一个**产品发布会**的 PPT，向投资人展示我们的 SaaS 平台，核心卖点是 AI 自动生成报表。"

### 2. 配置调用

```json
{
  "skill": "ppt_skill",
  "inputs": {
    "topic": "AI SaaS 报表平台产品发布会",
    "ppt_type": "产品发布会",
    "audience": "投资人 / 潜在客户",
    "duration_minutes": 20,
    "key_message": "AI 自动生成报表，将 CFO 团队从3天缩短到3分钟",
    "tone": "专业 + 创新感",
    "brand_colors": "#1A73E8, #34A853"
  }
}
```

### 3. 获取输出

AI 会根据你的需求输出完整的内容架构、逐页设计和演讲备注。

### 推荐使用方式

| 场景 | 示例提问 |
|------|---------|
| 快速出大纲 | "我要做一份新员工培训的PPT，帮我出大纲" |
| 单页优化 | "这页数据太多，帮我精简" |
| 整体设计咨询 | "给投资者的 Deck 应该用什么配色和字体？" |
| 演讲排练 | "帮我为这8页PPT写演讲备注" |

## 📁 文件结构

```
ppt-skill/
├── README.md                   # 本文档
├── SKILL.md                    # 核心：AI PPT 架构师提示词
├── knowledge-base.md           # 知识库：设计原则、配色、图表、排版
├── output-templates.md         # 输出模板：大纲/逐页/备注等多种模板
├── schema.json                 # 输入参数定义
└── examples.json               # 测试用例
```

## 🔗 发布指南

### 发布到 OpenAI GPTs

1. 新建 GPT → 在 **Instructions** 中粘贴 `SKILL.md` 全部内容
2. 在 **Knowledge** 中上传 `knowledge-base.md` 和 `output-templates.md`
3. 配置 Conversation starters 为常见 PPT 需求

### 发布到 Dify / Coze

1. 创建 Agent → 系统提示词中粘贴 `SKILL.md`
2. 创建知识库 → 导入 `knowledge-base.md` 和 `output-templates.md`
