# Paper Logic

## 功能

深挖论文章节之间的内在逻辑：递进关系、过渡策略、创新点定位、核心干货提炼。输出逻辑链而非结构清单，便于复习回看。

## 输入

用户粘贴文献文本（或指定已有文件路径）。

## 输出格式

```
# [论文标题] — 逻辑拆解

## 一、逻辑主线
[一句话概括论文从开头到结论的核心论证路径]

## 二、章节递进关系
1. [板块A] → [板块B]：[过渡策略描述]
2. [板块B] → [板块C]：[过渡策略描述]
...

## 三、关键节点
- **创新点落点**：X 板块 / 作者通过 X 方式建立创新性
- **实验核心落点**：X 板块 / 核心发现是什么
- **核心结论落点**：X 板块 / 最关键的一句话结论

## 四、每章干货
- **Title + Abstract**：研究问题 + 核心结论（1-2句）
- **Introduction**：为什么要做这个研究（1-2句）
- **[其他板块]**：......（简洁提炼，不复述原文）

## 五、逻辑可视化（可选）
[Introduction → Literature Review → Hypotheses → Method → Results → Discussion] 的递进箭头图，标注每段的核心功能
```

---

## 逻辑梳理步骤

1. **识别文献类型**，匹配对应逻辑模板
2. **划分章节边界**，找出每个板块的核心功能
3. **串联递进关系**，标注相邻章节间的过渡策略
4. **定位三个落点**：创新点 / 实验核心 / 核心结论
5. **提炼每章干货**，控制字数，不复述原文
6. **输出逻辑链**，简洁、可存档

---

## 核心原则

- **递进 > 并列**：不孤立描述每个板块，要说清楚"为什么先写A再写B"
- **创新点定位**：心理学/HCI/社科类论文通常在 Introduction 末尾或 Discussion 前半部建立创新性
- **实验核心聚焦**：Results 部分只提炼最核心的1-3个发现，不做完整统计报告
- **干货控制**：每章不超过3句话，避免写成缩写版原文
- **回看友好**：输出结果应能在3个月内快速回忆起论文全貌

---

## 文献类型适配

| 类型 | 逻辑特征 |
|------|----------|
| 实证论文 | 问题→假设→方法→结果→讨论，线性递进 |
| 综述 | 文献量→归类→分歧→研究空白→研究问题 |
| 理论论文 | 现有理论批评→新理论构建→论证→边界讨论 |
| 元分析 | 方法→结果汇总→异质性分析→理论更新 |

若文献类型不符标准结构，在输出开头注明实际格式来源。
