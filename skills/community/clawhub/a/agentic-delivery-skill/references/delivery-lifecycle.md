# Agentic Development Delivery Lifecycle

## Flow

1. Kickoff
   - Confirm scope, owners, cadence, approval gates, access, milestones, and next steps.

2. Access and readiness
   - Collect systems, docs, APIs, sample inputs, reviewer lists, and sandbox constraints.

3. Planning and tracking
   - Maintain project plan, milestone tracker, dependencies, decisions, and risks/issues.

4. Build and review
   - Implement approved scope, review prototype behavior, and manage change requests.

5. Evaluation and acceptance
   - Run evaluation examples, summarize results, resolve defects, and package acceptance evidence.

6. Launch or handoff
   - Use readiness checklists, monitoring plan, runbook, quickstart, and handoff checklist.

7. Support and closeout
   - Manage support tickets, escalation, post-launch review, lessons learned, and closeout summary.

## Handoff To Other Skills

- `agentic-email-skill`: status updates, review requests, acceptance reminders, support responses.
- `agentic-proposal-skill`: change-order or expansion proposal from delivery facts.
- `agentic-contract-skill`: scope, acceptance, support, or change-order terms.
- `agentic-invoice-skill`: milestone, change-order, retainer, support overage, or final invoices.
- `agentic-discovery-skill`: revisit discovery if scope or workflow assumptions are invalid.
- `agentic-certificate-skill`: training attendance certificates when training is delivered.

## Minimum Project Controls

- project owner
- technical owner
- reviewer/approver
- milestone status
- decision log
- risk/issue log
- dependency tracker
- change request process
- evaluation evidence
- acceptance status
- handoff checklist
