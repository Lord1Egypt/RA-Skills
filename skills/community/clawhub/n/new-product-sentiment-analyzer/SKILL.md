

### 名称
`new-product-insight-analyzer`

### Slug
`new-product-insight-analyzer`

### 定位（用户可见）
聚焦企业最新发布会及官方发声，精准提炼其新一代产品的核心技术与参数，快速评估其市场竞争壁垒。

### 描述（LLM识别）
洞察力敏锐的产品战略专家及硬科技行业分析师。当用户要求总结某企业刚刚发布的新品核心亮点，询问某产品线近期的升级迭代情况及其技术参数，或在进行竞品对比前要求快速建立目标企业最新产品的能力基线时，激活此技能。

执行逻辑：
1. 产品实体定位：从指令中提取“企业名称”及具体的“产品/系统/解决方案名称”。若未指定具体产品，则默认搜索最近一次发布的新品。
2. 触发联网搜索：定向检索垂直科技媒体、行业论坛及企业官网发声。搜索逻辑包含：“企业名称” AND “产品名称” AND (“发布会” OR “核心参数” OR “白皮书” OR “技术架构” OR “首发”)。
3. 硬核数据提取：过滤公关形容词，专门捕获可量化的指标（如：处理速度提升百分比、功耗降低数值、首发搭载的特定芯片架构或算法引擎）。
4. 壁垒评估：识别该新品解决的核心行业痛点，并判断其宣传的卖点是属于“微创新（UI/体验优化）”还是“底层颠覆（核心组件自研突破）”。
5. 输出格式：《新品竞争力评估简报》，包含：核心参数与卖点清单、首发技术亮点/代际差异、对标市场与目标客群。

### 适用场景
- 招商团队评估目标企业技术迭代能力及产品竞争力
- 投资人判断新品是否能形成新的增长曲线
- 企业自身做竞品分析前的信息收集
- 采购方评估供应商产品升级的实际价值

### 输入参数
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `company_name` | string | 是 | 目标企业名称 |
| `product_name` | string | 否 | 具体产品/系统名称，若不提供则自动提取最近一次发布的新品 |
| `time_range` | int | 否 | 搜索时间范围（月），默认3个月（聚焦最新发布） |
| `baseline_compare` | string | 否 | 对比基准，可选`previous_gen`（上一代）、`industry_avg`（行业均值）、`both`，默认`both` |

### 输出格式
输出一份《新品竞争力评估简报》，包含以下模块：

1. **新品基本信息**：产品名称、发布时间、发布场合（发布会/线上/展会）
2. **核心参数与卖点清单**（硬核指标，无废话Bullet Points或表格）  
   - 性能指标（如算力、功耗、精度、速度）  
   - 架构/算法创新（如自研芯片、全新框架）  
   - 兼容性/生态特性  
   - 价格定位（如有）
3. **首发技术亮点/代际差异**  
   - 对比上一代产品的跨越式升级点  
   - 对比行业同价位/同类型产品的差异优势  
   - 标注“微创新”还是“底层颠覆”
4. **对标市场与目标客群**  
   - 细分市场定位（如下沉市场、高端政企、特定工业场景）  
   - 目标客群画像  
   - 解决的行业核心痛点
5. **竞争壁垒评估**（低/中/高）  
   - 技术可复制性  
   - 专利/供应链/生态护城河

### 执行流程（内部步骤）

1. **产品实体定位**  
   - 从用户输入中提取 `company_name` 和 `product_name`。  
   - 若无 `product_name`，搜索“{company_name} 最新发布 产品 2026”或“{company_name} 新品 发布会”，确定最近一款主打新品。  
   - 若企业近期发布多款产品，默认选择曝光度最高（媒体报道量最大）的一款，并告知用户。

2. **联网搜索策略**  
   - 关键词组合：  
     - `"{company_name} {product_name} 发布会"`  
     - `"{product_name} 核心参数"`  
     - `"{product_name} 白皮书"`  
     - `"{product_name} 技术架构"`  
     - `"{product_name} 首发 评测"`  
   - 优先来源：企业官网新闻中心、垂直科技媒体（如36氪、虎嗅、IT之家、极客公园）、知乎产品评测、B站发布会录播文字稿。  
   - 时间限定在 `time_range` 内。

3. **硬核数据提取**  
   - 忽略以下词汇：“震撼”、“革命性”、“极致”、“完美”、“卓越”。  
   - 重点捕获：  
     - 数字指标：XX TOPS、XX nm、XX GHz、XX 毫秒、XX 瓦、提升 X%  
     - 技术名词：自研NPU、Chiplet架构、AI大模型、端侧推理引擎  
     - 认证/标准：通过XX认证、符合XX协议  
   - 若参数缺失，标注“公开资料未披露该参数”。

4. **代际差异与行业对比**  
   - 若用户提供上一代产品名称或自动识别，搜索“上一代产品 参数”进行对比。  
   - 行业均值可基于已知公开数据或合理推断（如“行业主流算力为10TOPS”）。  
   - 判断创新层级：  
     - **微创新**：优化体验、小幅度性能提升、外观改动。  
     - **底层颠覆**：自研核心芯片、全新架构、解决行业长期痛点（如功耗减半）。  

5. **市场定位与壁垒评估**  
   - 根据产品定价、性能、适用场景推断目标客群（如“面向中小企业的轻量版”）。  
   - 壁垒评估：  
     - **高**：自研IP、独家供应链、专利墙、生态锁定  
     - **中**：性能领先但可追赶、有先发优势  
     - **低**：参数与竞品持平、无独特护城河  

6. **输出报告**  
   - 参数清单简洁明了。  
   - 代际差异用对比表格或箭头说明。  
   - 壁垒评估给出理由。

### 错误处理与注意事项

| 错误类型 | 处理方式 |
|---------|----------|
| 未找到任何新品发布信息 | 返回“近{time_range}个月内未发现该企业公开新品发布，请扩大时间范围或确认企业名称” |
| 产品参数未披露 | 标注“官方未公布详细参数，以下为媒体推测或行业对比”，不虚构数据 |
| 无上一代对比数据 | 仅输出行业均值对比或跳过该部分 |
| 多款新品同时发布 | 默认选择主推款（媒体提及最多），并在报告中说明“另有X款新品，本报告聚焦主推产品” |

### 扩展能力
- **竞品对比表**：输入竞品产品名称，输出多维度参数对比矩阵。  
- **供应链溯源**：分析新品核心零部件供应商，评估供应链风险。  
- **用户口碑聚合**：在发布会后1-3个月，搜索用户评测和开发者反馈，补充实际体验数据。
