# 热点内容创作助手

> 触发词：「热点创作」「追热点」「创作方向」「小红书方向」「抖音方向」
> 适用场景：需要从热点中挖掘内容创作方向，并生成配图
> 凭证：由用户自行配置API密钥（参考TOOLS.md配置方式）

---

---

## 执行流程

```
Step1 抓热点 → Step2 筛选5个方向 → 用户选择
→ Step3 生成封面 → Step4 生成虚拟形象
→ Step5 选完图后 → 去AI味写作（自动触发）
```

---

## Step1：抓取热点

### ⚠️ 自检机制（必须全部成功才进入Step2）

**6个平台必须全部返回有效数据后，才能发出热点报告。**

如果某个平台失败/无数据：
→ 标记该平台为「❌ 失败」，立即重试（最多3次）
→ 重试仍然失败则在报告中显示「⚠️ 暂无数据」，其他平台正常展示
→ 任意平台连续3次失败，立即用小红书搜索API补充该平台内容
→ **只有拿到至少4个平台的有效数据，才进入Step2**

### 平台配置

| 平台 | API类型 | 说明 |
|------|---------|------|
| 微博 | GET `/misc/hotboard?type=weibo` | 实时热搜 |
| 知乎 | GET `/misc/hotboard?type=zhihu` | 知乎热榜 |
| 抖音 | GET `/misc/hotboard?type=douyin` | 抖音热榜 |
| 虎扑 | GET `/misc/hotboard?type=hupu` | 虎扑热搜 |
| 36氪 | GET `/misc/hotboard?type=36kr` | 36氪科技 |
| 小红书 | 搜索 POST `/search/aggregate` | 补充热门话题 |

### 自检通过标准

✅ 全部6个平台有数据 → 进入Step2  
⚠️ 4-5个平台有数据 → 进入Step2（报告中标注缺失平台）  
❌ 3个以下平台有数据 → **暂停**，告知用户「热点抓取异常，重新尝试中」

## Step2：筛选创作方向

从所有热点中，筛选出**5个创作方向**：

### 输出格式

```
🔥 今日热点速览（YYYY-MM-DD）

[平台名]热搜 Top5：
1. [标题]（热度值）
...

---

📋 5个创作方向候选

【方向1】🔴 [领域]｜[核心爆点]
平台：[小红书/抖音/微博/知乎]
切入点：[一句话说明为什么适合做]
热点关联：[对应哪个热点]
建议形式：大字报封面 + 虚拟形象出镜解读
预估效果：🔥🔥🔥🔥🔥（5个🔥为最高）

【方向2】...
...

请选择方向（输入数字1-5）：
```

### 筛选原则

1. **热度高** — 有流量基础
2. **有话题性** — 能引发讨论
3. **适合视觉呈现** — 能配图/出镜
4. **有差异化** — 不要重复同质内容
5. **时效性** — 只选近3天内的事件，超过5天一律排除

## Step3：生成大字报封面（通用工作流）

**前提：** 用户已选择方向

### ⚠️ 询问用户封面风格

**先询问用户对封面的要求，不得自行决定。**

```
封面风格想怎么定？
  1）你帮我推荐一个（AI根据热点主题推荐）
  2）我来描述我想要的风格（请说明背景色/字体/构图偏好）
  3）直接用你默认的风格（简洁大字报）
```

### 选择「1」（AI推荐）或「3」（默认）时的流程

**Step 3a：AI推荐封面方案**

根据热点主题，从以下风格库中选择最合适的：

| 风格 | 背景 | 字体 | 适用场景 |
|------|------|------|---------|
| 科技大事件 | 深色径向渐变 | 超大白字 | 科技/商业热点 |
| 社会事件 | 纯色+线条 | 大字+副标题 | 社会/民生热点 |
| 娱乐/明星 | 亮色渐变 | 柔体大字 | 娱乐/影视热点 |
| 节日/假期 | 暖色调 | 活泼字体 | 节日/出行热点 |
| 国际/争议 | 深蓝/深红 | 严肃大字 | 国际/政治热点 |

> AI推荐后，将风格方案发给用户确认。

**Step 3b：标题候选**

根据热点内容，拟出3个候选标题发给用户：

```
【候选A】标题文字（引发好奇/疑问/冲突）
【候选B】标题文字（引发好奇/疑问/冲突）
【候选C】标题文字（引发好奇/疑问/冲突）

请选择一个标题，或提出修改意见：
```

### 选择「2」（用户描述）时的流程

用户描述风格偏好后：
1. 理解用户需求，撰写封面提示词
2. 将提示词发给用户确认
3. 用户确认后才生成

### 通用提示词模板（内部使用）

```
[风格描述]大字报封面，
[背景描述]，
[主标题：超大加粗白字，带阴影]，
[副标题：小号细体字]，
[右下角署名位置]，
无水印，高品质
```

### 禁止事项

- 用户未确认风格方案前生成图片
- 用户未确认标题前生成图片
- 生成过程不展示提示词，用户只看结果图

## Step4：生成虚拟形象（通用工作流）

**前提：** 用户选择了方向，需要生成人物出镜

### ⚠️ 第一步：询问用户是否需要虚拟形象

### 询问话术

```
这个方向需要出镜人物吗？
  1）需要生成一个虚拟形象（请告诉我你想要的形象要求）
  2）不需要出镜人物，用纯大字报封面即可
```

### 选择「1」的完整流程

**收集用户要求：**
```
虚拟形象需要什么风格？
  A）二次元动漫风（类似日系番剧）
  B）3D渲染风（类似游戏角色）
  C）写实风（接近真人）
  D）我用自己的参考图（请附上图片）

虚拟形象的外貌要求：
  → 性别：
  → 年龄感：
  → 脸型：
  → 发型/发色：
  → 眼睛颜色：
  → 表情/眼神：
  → 其他特征：

服装要求：
  → 颜色：
  → 风格：
  → 是否有品牌/制服等特殊要求：

背景要求：
  → 纯色背景（什么颜色）：
  → 场景背景（什么场景）：

是否有参考图需要提供？（是/否）
```

> 若用户表示「你帮我选」，则由AI根据热点主题推荐最合适的外貌特征，用户确认后执行。

**生成并发送：**
- 图片生成后，通过消息工具发送到用户
- 附文字：「虚拟形象V1，请确认是否需要调整？」
- 收到修改意见后，重新生成（最多3次）

**发布规范：**
- 右下角水印：「图片由AI生成，注意甄别」
- 正文末尾加：「图片由AI生成」

---

## 错误处理

- 某个平台API失败：跳过该平台，其他平台继续
- 封面生成失败：提示用户重新描述热点标题，重新生成
- 形象生成风格偏离：根据用户反馈重新生成

---

## Step5：去AI味写作（封面+形象图选完后自动触发）

用户确认封面图和形象图后，无需额外指令，自动进入Step5。

**必读技能：** humanize_write（说人话）

### 流程

```
用户选完图片
  → 用户选择发布平台：小红书 / 抖音 / 微博 / 知乎
  → 生成AI初稿
  → 执行去AI味处理
  → 输出最终文案给用户确认
  → 用户修改或直接使用
```

### 用户可选的风格

- 日常分享（朋友聊天感，轻微吐槽）
- 专业干货（口语化，不用术语堆砌）
- 情感共鸣（偏情绪化，用"我"经历打动人）
- 搞笑吐槽（夸张自嘲，网感强）
