# 综合诊断框架

## 概述

综合诊断是基于PESTL分析、波士顿矩阵、SWOT分析、商业模式画布的交叉分析，识别商业模式的核心风险、机会和改进建议。

## 交叉分析逻辑

### 核心公式
```
核心风险 = 画布短板维度 × SWOT威胁 × PESTL负面因素
核心机会 = 画布优势维度 × SWOT机会 × PESTL正面因素
```

### 分析矩阵
```
                    PESTL正面              PESTL负面
                  ┌────────────────────┬────────────────────┐
    画布优势      │  ★★★ 黄金机会       │  ◆◆ 优势受限       │
                  │  (S/O + PESTL+)    │  (S/O + PESTL-)    │
    画布短板      ├────────────────────┼────────────────────┤
                  │  ▲▲ 待提升机会      │  ⚠⚠⚠ 核心风险     │
                  │  (W/T + PESTL+)    │  (W/T + PESTL-)    │
                  └────────────────────┴────────────────────┘
```

## 风险识别

### 风险类型
| 风险类型 | 定义 | 识别来源 |
|----------|------|----------|
| 战略风险 | 商业模式层面的根本性问题 | 画布评分低分维度 |
| 环境风险 | 外部环境变化带来的威胁 | SWOT威胁 + PESTL负面 |
| 执行风险 | 落地执行中的问题 | 关键业务、核心资源缺失 |
| 财务风险 | 收入、成本相关问题 | 收入来源、成本结构 |

### 风险等级
| 等级 | 标准 | 响应策略 |
|------|------|----------|
| 高 | 画布<5分 + SWOT威胁 + PESTL负面 | 立即解决 |
| 中 | 画布<6分 + 任一威胁或负面 | 优先解决 |
| 低 | 单一维度问题 | 持续关注 |

### 风险识别流程
```
1. 找出画布评分≤5的维度
2. 对应SWOT中的威胁因素
3. 对应PESTL中的负面因素
4. 交叉比对，确定核心风险
5. 评估风险等级和影响
```

## 机会识别

### 机会类型
| 机会类型 | 定义 | 识别来源 |
|----------|------|----------|
| 市场机会 | 市场空白或增长带来的机会 | SWOT机会 |
| 政策机会 | 政策红利带来的机会 | PESTL正面 |
| 能力机会 | 核心能力转化带来的机会 | 画布高分维度 |
| 生态机会 | 合作伙伴带来的机会 | 重要合作完善度 |

### 机会等级
| 等级 | 标准 | 把握策略 |
|------|------|----------|
| 高 | 画布≥7分 + SWOT机会 + PESTL正面 | 重点投入 |
| 中 | 画布≥6分 + 任一机会或正面 | 积极把握 |
| 低 | 单一维度优势 | 适度关注 |

### 机会识别流程
```
1. 找出画布评分≥7的维度
2. 对应SWOT中的机会因素
3. 对应PESTL中的正面因素
4. 交叉比对，确定核心机会
5. 评估机会价值和可行性
```

## 改进优先级

### 优先级排序原则
| 优先级 | 标准 | 行动时间 |
|--------|------|----------|
| P0 | 高风险 + 高影响 | 立即（1个月内） |
| P1 | 中风险 + 高价值 | 短期（3个月内） |
| P2 | 中风险 + 中价值 | 中期（6个月内） |
| P3 | 低风险 + 低价值 | 长期（12个月内） |

### 优先级矩阵
```
            高价值                低价值
         ┌─────────────────┬─────────────────┐
    高   │    ★ P0         │    ◇ P1         │
   紧   │   立即行动        │   短期行动       │
   急   ├─────────────────┼─────────────────┤
         │    ◆ P2         │    ○ P3         │
    低   │   计划行动        │   持续观察       │
         └─────────────────┴─────────────────┘
```

## OPC建议框架

### OPC视角说明
作为技术成果转化咨询师（OPC导师），关注点：
1. 技术成熟度与商业化节奏匹配
2. 技术壁垒的可持续性
3. 核心技术的知识产权布局
4. 技术团队与商业团队的协同
5. 技术产品化路径的可行性

## 注意事项

1. **数据来源**：所有分析基于提取数据和用户确认信息
2. **交叉验证**：不同维度的同一信息相互印证
3. **保守评估**：无法确认的信息采用保守估计
4. **可操作性**：建议必须可执行、可落地
5. **OPC视角**：始终站在技术成果转化角度
6. **去夸张**：严禁使用"颠覆性""全球领先"等表述
