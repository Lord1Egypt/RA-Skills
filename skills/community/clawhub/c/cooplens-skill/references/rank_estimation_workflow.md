# 位次预估：统计、分析与预测工作流

## 定位

本文件用于所有涉及分数线、投档线、最低位次、候选排序或新项目门槛判断的任务。位次预估必须作为独立分析章出现，不能只在结论里顺带给一个数字。

## 重要声明

```text
重要声明：CoopLens Skill 产出的内容由 AI 生成，真实性需要使用者自行核实，不代表任何官方意见，不能作为任何决策依据或参考。
```

不得另写位次预估专用声明或兜底式来源承诺。人工确认内容写在数据卡片或人工确认清单中。任何位次预估都必须同时说明：

- 数据来源是否能打开；
- 链接内容是否对应目标年份、目标省份、科类/选科、招生批次、专业组、计划类型和项目名称；
- 数字是否来自官方/权威原文，还是由一分一段表换算；
- 人工确认动作是什么；
- 哪些数据没有核到，因此会降低置信度。

## 统一顺序：先结论，再统计，再分析，再预测

### 零、先给结论

位次章的第一屏必须让家长看到结论，不要先铺长篇方法。先写：

- 参考区间：约 X–Y 位；
- 冲/稳/保判断：冲 / 边界冲 / 稳 / 保 / 不建议冲；
- 置信度：强 / 中 / 弱；
- 当前学生位次的安全边际或差距；
- 会推翻本结论的条件。

随后再进入统计、分析和预测推导。

### 一、先统计

把可验证数据收集成窄卡片或短表，不使用没有来源链接的关键数字。至少统计：

1. 当年是否招生核验（入场闸门）：学校本科招生网/招生办官网、当年招生章程/招生简章、目标省招生计划/专业目录、教育部中外合作办学监管工作信息平台/CRS 或教育部官网项目身份来源；先判断继续招生、未在当年招生计划中、当年停招/暂停招生，或未核到当年招生。
2. 当前年份招生信息：招生简章、招生章程、招生计划、专业目录、招生省份、科类/选科、计划人数、收费和证书规则。
3. 最近完成年度录取信息：最低分、最低位次、投档线或专业线，优先省考试院和学校本科招生网；必须标注招生批次和院校专业组。
4. 一分一段表：用于分数与位次互相换算，必须对应同省、同年、同科类/选科。
5. 同校普通专业基线：同校同专业、相近专业组或同层次普通招生专业。
6. 同校合作/国际路径样本：同校其他合作项目、国际学院或高收费专业组。
7. 同省同档样本：同省、同层次、相近专业、相近学费和培养模式的合作项目。
8. 影响需求的数据：学费、总费用、是否出国、证书、留服认证、城市/校区、计划变化、专业热度、招生人数。

每个数字卡片必须包含：

```text
数据项：...
数值：约...
来源链接：...
来源类型：省考试院 / 学校官网 / CRS / 排名官方 / 权威平台 / 其他
人工确认：请打开链接核对年份、省份、科类、批次、专业组、项目名称和数值是否一致。
质量等级：A / B / C / D
```

质量等级：

- A：两个以上官方来源一致，或省考试院与学校官网互相印证。
- B：一个官方来源加一个权威平台一致。
- C：只有一个可打开来源，或多个来源有小幅差异。
- D：只有转载、机构或公开讨论，不能支撑排序和计算。### 二、再分析

统计完成后再解释数据关系，至少分析：

1. 绝对分差：普通专业最低分 - 合作项目最低分。
2. 绝对位差：合作项目最低位次 - 普通专业最低位次。
3. 相对位差：绝对位差 / 普通专业最低位次 × 100%。
4. 线差：项目录取分 - 批次线。
5. 同省同档样本中位数：用来降低单个样本异常的影响。
6. 招生状态与计划变化：继续招生、当年停招/暂停招生、未在当年招生计划中、扩招、缩招、首次招生、专业组调整。停招或未核到当年招生的项目不能进入本年位次公式或硬排序。
7. 需求调整方向：学费、证书、留服认证、是否必须出国、专业热度、城市和公开讨论信号对报考意愿的影响。
8. 异常值：年份、科类、批次、专业组、位次单位、提前批/本科批混淆等。

计算公式必须展示给用户：

```text
绝对位差 = 合作项目位次 - 普通基线位次
相对位差 = 绝对位差 / 普通基线位次 × 100%
线差 = 录取分 - 同招生批次控制线
```

### 三、再预测

预测只在统计和分析完成后进行。输出必须是区间，不是单点。

#### 有历史线项目

```text
趋势参照位次 = 最近完成年度项目位次 × (1 + 年均变化率)
调整后参考区间 = 趋势参照位次 ± 招生计划/专业热度/费用/证书/校区/样本波动调整
```

如果历史线少于三年，必须降低置信度，并把同校普通基线和同省同档样本纳入交叉参照。

#### 新项目、首年招生或本省无历史线项目

使用三情景，不得给单点：

```text
中性情景参考位次 = B普通招生基线位次 + A同省同档样本中位位次差
门槛偏高情景参考位次 = B普通招生基线位次 + A样本较小位次差
门槛偏宽情景参考位次 = B普通招生基线位次 + A样本较大位次差
```

首年项目可以加入“信息不对称”方向判断，但只有在有同类历史样本时才能量化；没有样本时只作定性说明。

#### 独立法人中外合作大学

如果不存在同校普通专业基线，改用：

- 同省同分段普通高校；
- 同省其他独立法人合作大学；
- 同省同费用段国际化本科路径；
- 学校自身近年分省分专业线。

必须写清楚“没有普通基线，因此用同省同分段和同类机构作参照”。

## 功能覆盖要求

- 单项目分析：位次章必须先说明“这个项目相对学生分数/位次是冲、稳、保还是不适合”，再展示统计和分析。
- 多项目对比：每个项目都要进入同一口径的统计、分析、预测；证据弱的项目不能参与硬排序。
- 按省份/分数/位次找候选：先给候选分组和每个候选的可能位次结论，再统计同省候选分布，再分析位差、证书/费用、升学就业和风险。
- 位次预估独立功能：第一屏给参考区间、冲稳保、置信度和推翻条件；随后展示统计样本、公式、情景区间、人工确认清单。

## 禁止项

- 禁止没有来源链接的关键数字。
- 禁止只写“预计多少位”而不展示统计样本和计算链。
- 禁止把“计算过程”和最终结论混在同一段；计算过程必须可被 HTML 样式识别并以黑底红字呈现。
- 禁止把历史参照写成官方预测。
- 禁止用公开讨论、机构宣传或无法打开的页面支撑分数和位次。
- 禁止把数据质量为 D 的数字用于排序。


## 招生批次拆分规则

位次章必须把 `招生批次` 当作公式输入的一部分。处理顺序如下：

1. 先从省考试院专业目录/招生计划锁定批次、院校专业组和计划类型。
2. 同一项目在不同批次或不同专业组招生时，拆成多条数据卡；每条卡单独列招生人数、最低分、最低位次和批次控制线。
3. 当年计划、历史录取线、普通基线、同省样本和一分一段表口径不一致时，不能直接合并或取平均；只能写成弱参照，并说明为什么没有进入公式。
4. 批次未核到、专业组未核到或计划类型未核到时，不能参与硬排序；如必须展示，只能放在“仅关注/待核验，不参与排序”。
5. 父母最容易看错的是“同一项目不同批次看起来名字一样，但招生人数和分数线不同”，报告必须主动提醒并列出核对问题。
