## Known Risks and Mitigations: <br>
Risk: Local REST API memory can accumulate project context that may include sensitive design details if the user records them. <br>
Mitigation: Enable memory only when useful, keep entries concise, and avoid storing secrets, tokens, customer data, or unrelated personal information in ~/rest-api/memory.md. <br>
Risk: API design, security, migration, or deployment guidance may be incomplete for a specific production environment. <br>
Mitigation: Review generated contracts, auth boundaries, migration plans, tests, observability, and release checklists with the responsible engineering and security owners before deployment. <br>
