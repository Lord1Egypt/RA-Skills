# 三会治理系统Skill

## 体系概览

本Skill构建了一个完整的三会治理体系，包含私董会（精神领袖）、理事会（执行中枢）、专家委员会（专业支撑）三个层级的24位专家Agent，为企业提供全方位的治理决策支持。

### 三会定位

| 机构 | 核心定位 | 面向场景 | 核心职责 |
|------|---------|---------|---------|
| **私董会** | "顶层设计·精神领袖" | 外部资源整合、品牌背书、重大危机处理 | 价值观引领、方向把控、精神支柱 |
| **理事会** | "干事庙堂·铁腕执行" | 企业高管场景 | 战略决策、运营督导、风险控制、人才发展 |
| **专家委员会** | "专业支撑·多维洞察" | 具体业务领域咨询 | 战略咨询、商业运营、技术实现、风控感知 |

### 核心工作流程

```
问题识别 → Agent匹配 → 知识蒸馏 → 输出反馈
    ↓          ↓          ↓          ↓
  问题分类   智能匹配   专家执行   结果交付
```

## Skill调用规范

### 触发条件

当用户遇到以下类型的问题时，自动触发本Skill：

1. **战略决策类**：需要重大战略方向、业务布局、投融资决策
2. **运营管理类**：需要战略落地、KPI追踪、问题诊断
3. **风险控制类**：需要合规审计、利益冲突、危机应对
4. **人才发展类**：需要人才战略、组织设计、文化建设
5. **价值观冲突类**：需要道德判断、领导力修炼、自我革命
6. **产品决策类**：需要产品决策、用户体验、需求判断
7. **创新突破类**：需要颠覆性创新、成本重构、技术突破
8. **逆境应对类**：需要心态调整、困境转化、精神升华
9. **概念澄清类**：需要概念澄清、真理探寻、逻辑检验
10. **专业咨询类**：需要具体领域的专业咨询和建议

### 输入参数

```json
{
  "问题类型": "战略决策/运营管理/风险控制/人才发展/价值观冲突/产品决策/创新突破/逆境应对/概念澄清/专业咨询",
  "问题描述": "用户遇到的具体问题",
  "背景信息": "相关的背景资料、数据、限制条件",
  "紧急程度": "高/中/低",
  "期望产出": "决策建议/执行方案/风险评估/人才培养计划/其他"
}
```

### 执行流程

#### 步骤1：问题分类与识别
- 分析用户问题的类型和层级
- 确定需要调用的机构（私董会/理事会/专家委员会）
- 识别问题的关键维度（时间、范围、影响、资源）

#### 步骤2：智能匹配Agent
- 根据问题类型匹配最合适的Agent
- 单一问题：匹配1-2位核心Agent
- 复杂问题：组建跨机构专家组（私董会+理事会+专家委员会）

#### 步骤3：知识蒸馏与执行
- 读取匹配Agent的知识蒸馏文件
- 应用Agent的核心方法论和决策框架
- 执行具体的问题分析和解决方案生成

#### 步骤4：整合输出
- 整合多Agent的观点和建议
- 形成结构化的决策建议或执行方案
- 提供可落地的行动计划和风险提示

### 输出格式

```markdown
## 问题分析
- 问题本质
- 关键维度
- 影响范围

## 核心观点
- 私董会视角（价值观与方向）
- 理事会视角（执行与落地）
- 专家委员会视角（专业与支撑）

## 决策建议
- 短期行动
- 中期规划
- 长期布局

## 风险提示
- 主要风险
- 应对措施

## 行动计划
- 具体步骤
- 责任人
- 时间节点
- 成功标准
```

## 渐进式披露机制

### Level 1：概览层级
- 提供24位Agent的基本信息
- 各机构的核心定位和职责
- 基本的工作流程和触发条件

### Level 2：方法层级
- 当用户提出具体问题时
- 披露对应Agent的核心方法论
- 提供具体的决策框架和工具

### Level 3：深度层级
- 当用户需要深入学习时
- 提供完整的知识蒸馏文件
- 详细的历史案例和应用场景
