# /qa-case 测试用例设计示例 — 订单改价功能

> 本示例与 [prd-demo.md](./prd-demo.md) 联动，展示评审问题如何转化为用例。

## 原始输入

```
/qa-case

产品/模块：电商平台 - 订单管理模块
需求描述：
  - 客服在后台对"已支付未发货"订单进行改价（仅下调，≤30%）
  - 改价后自动触发差额退款
  - 改价记录展示：操作人、时间、原金额、新金额、差额
  - 仅有权限的客服可操作

评审问题清单（来自 /qa-prd）：
  1.（高/完整性）未定义退款失败时的处理策略
  2.（高/边界）并发改价场景未定义
  3.（高/安全）客服权限校验规则不明确
  4.（中/边界）下调30%边界值不明确

关联系统：Jira 项目 ORDER
```

## AI 输出（精简版，完整格式见 login-demo.md）

### 需求分析
- 需求编号：ORDER-2024-003
- 功能点拆解：
  1. 客服对符合条件的订单发起改价（金额下调 ≤30%）
  2. 系统校验客服权限
  3. 改价成功 → 触发退款 → 记录操作日志
- 业务价值：解决用户因促销/差价/补偿等原因要求降价时，客服无法直接处理的痛点
- 验收标准：
  1. 改价成功后订单金额更新，差额发起退款
  2. 退款失败时订单金额回滚，改价操作撤回
  3. 并发改价时仅第一个操作成功，后续返回冲突提示
  4. 无权限客服看不到改价入口

### 简明摘要

- 核心层:6 / 体验层:5 / 增值层:1

## 合规场景：ERP 采购审批 + 行业合规要求

> 展示当用户提供行业/合规要求时，AI 如何引用 `team/` 配置生成合规检查点。

### 原始输入

```
/qa-case

产品/模块：ERP 系统 - 采购审批模块
需求描述：
  - 采购员发起采购申请（填写金额、用途、供应商）
  - 部门主管一级审批（金额 ≤ 5万事前自动通过）
  - 财务主管二级审批（所有申请均需经财务）
  - 审批记录完整留痕（操作人、时间、意见），不可删除

行业/合规要求：SOX 404（财务内控）、ISO 27001-7.1.2（访问控制）

关联系统：Jira 项目 FINANCE
```

### AI 输出（精简，仅展示合规相关部分）

> AI 从 `team/standards.json` 匹配到 ERP 行业标准：
> - SOX 404：财务报告内部控制
> - ISO 27001 7.1.2：访问控制策略
>
> 从 `team/roles.json` 匹配到 ERP 行业角色：
> - 财务主管、审计员、业务员、采购员

#### FINANCE-2024-008-TC01：标准采购申请两级审批通过
- **业务分层**：核心层
- **类型**：功能测试
- **设计方法**：场景法
- **优先级**：P0
- **前置条件**：采购员、部门主管、财务主管账号均可用；采购金额 100,000 元
- **测试数据**：采购员=采购员001 / 部门主管=主管001 / 财务主管=财务001
- **测试步骤**：
  1. 使用采购员账号登录，填写采购申请（金额 10万），提交
  2. 使用部门主管账号登录，进入待审批列表，查看申请详情
  3. 部门主管点击"通过"，填写审批意见
  4. 使用财务主管账号登录，进入待审批列表
  5. 财务主管点击"通过"
- **预期结果**：
  1. 采购申请提交成功，状态为"待一级审批"
  2. 部门主管通过后，状态变为"待二级审批（财务）"
  3. 财务主管通过后，状态变为"已批准"，生成完整审批记录
- **合规检查点**：SOX 404 — 审批权限分离（发起人≠审批人≠终审人），审批记录不可删除

## 使用技巧

1. **评审问题清单直接粘贴**：AI 会自动将"未定义退款失败处理"转为 TC02、"并发改价"转为 TC05——不用人手动想
2. **业务分层帮助排列执行优先级**：核心层 6 条必须全过，体验层 5 条尽量过，增值层 1 条有时间再过
3. **测试数据的边界值要具体**：TC03 "刚好 30%" 和 TC04 "30.1%" 来自黑盒方法自动匹配，边界值分析方法论保证了这一点
4. **行业合规要求是可选的，但填了就生效**：输入行业/合规要求后，AI 会从 `team/` 目录自动匹配角色和标准，在相关用例末尾追加「合规检查点」。不需要合规时留空即可，不扫兴
5. **team/ 配置可自由扩展**：如果你的行业不在预设列表中，在 `team/roles.json` 和 `team/standards.json` 中新增即可，AI 下次就会引用
