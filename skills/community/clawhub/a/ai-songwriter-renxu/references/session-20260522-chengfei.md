## 项目背景

- **客户**：成飞（成都飞机工业集团）

## 调研结果

- 1958年创立（原国营132厂，黄田坝）
- 1964年首飞歼5甲
- 1980年代承担歼10研制
- 歼10获国家科技进步奖特等奖
- 2011年歼20（威龙）首飞 — 中国首款五代隐身战机
- 2024.12.26 歼36（六代机）首飞

### 六代机技术特征
- 无尾三角翼、三发动机（双涡扇+火箭/冲压）
- 全向隐身（360°无死角）
- AI智能座舱、忠诚僚机协同
- 作战半径 ~3000km（超歼20 50%+）
- 速度可能达 Mach 2.5-3+
- 代号"白帝"

## 踩坑记录

### 坑1: 尾字重复（征×2）
- **现象**：S4L4「长征」和 S9L1「长征」尾字重复
- **根因**：S9L1 错用了「征」而非指定的「程」
- **修复**：S9L1「六十八载风雨路再踏新程」

### 坑2: 生僻字发音不准
- **现象**：v1 中「筚路蓝缕」「峥嵘」「旌」发音不清晰
- **根因**：music-2.6 对乙级/丙级字发音不稳定
- **修复**：
  - 筚路蓝缕 → 披荆斩棘（中间位替换）
  - 峥嵘 → 光荣（尾字ong组替换）
  - 蕴藏 → 深藏（中间位替换）
  - 旌 → 灯（尾字eng组替换）
  - 保留词加 `--extra` 拼音提示：雷霆、铭、崇、凌、芳华、铸成

### 坑3: hardcoded 韵母判断不靠谱
- **结论**：必须用 pypinyin 逐字验证，不能用硬编码集合
- **示例**：iong/ing/ueng → eng 组；ong（无介音）→ ong 组

## 关键经验

1. **四步法必须严格按序执行**：大纲→韵脚表→造句→验证，不能边想边写
2. **44字全局唯一**靠韵脚预分配实现，不是后验修正
3. **结构标签双轨制**：交付用户纯文本，传 API 时保留标签
4. **发音三层防线**：事前扫描 → 事中 `--extra` 提示 → 事后听审
5. **「成飞」只出现一次**：在 S1L1 Intro 中低调出现，Chorus 不重复
