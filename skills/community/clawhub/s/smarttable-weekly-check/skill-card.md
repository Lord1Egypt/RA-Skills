## Description: <br>
Enterprise WeCom Smart Table weekly report checking tool that uses Puppeteer and internal table APIs to extract structured data, evaluate team weekly reports against four quality rules, and generate reports and project summaries. <br>

This skill is ready for commercial/non-commercial use. <br>

## Use Case: <br>
Employees and team leads use this skill to extract authorized WeCom Smart Table weekly-report data, check report quality across multiple weeks, and produce Markdown summaries for follow-up. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: The skill stores employee weekly-report data locally under smartsheet_data/. <br>
Mitigation: Run it only for authorized reports, keep the output directory private, and delete exported data when it is no longer needed. <br>
Risk: The browser login profile is persisted in ~/.smarttable-check-browser. <br>
Mitigation: Protect the local profile directory as sensitive authentication state and remove it after use on shared or temporary machines. <br>
Risk: Optional scheduled execution can repeatedly access and analyze internal report links. <br>
Mitigation: Review the cron message, file paths, and authorization scope before enabling recurring checks. <br>
