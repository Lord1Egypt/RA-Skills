## Description: <br>
Generate structured changelogs from git history using conventional commits, semantic versioning, multiple output formats, and optional AI-enhanced release descriptions. <br>

This skill is ready for commercial/non-commercial use. <br>

## Publisher: <br>
[charlie-morrison](https://clawhub.ai/user/charlie-morrison) <br>

### License/Terms of Use: <br>
MIT-0 <br>


## Use Case: <br>
Developers and release engineers use this skill to turn repository commit history into release notes, changelogs, or GitHub release text. It is useful when preparing tagged releases, summarizing unreleased changes, or documenting project history. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Generated changelogs can expose private commit details, issue references, authors, or misleading AI-enhanced summaries if published without review. <br>
Mitigation: Review generated changelogs before publishing, use explicit project-local output paths, and avoid including private repository or issue-tracker details unless intended. <br>
