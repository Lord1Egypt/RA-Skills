## 二、MVP功能详细规格

### 2.1 阶段1：猎手功能（完整实现）

#### 功能点

| 功能 | 描述 | 验收标准 |
|------|------|---------|
| 需求理解 | 理解用户模糊输入，提炼核心诉求 | 能准确复述用户需求 |
| 趋势扫描 | 扫描相关领域趋势（内置知识） | 输出3个趋势要点 |
| 机会识别 | 识别市场空白和差异化机会 | 输出2-3个机会点 |
| 灵感激发 | 提供类比案例和跨界灵感 | 输出2个参考案例 |
| 方向提案 | 生成3-5个创意方向提案 | 每个方向包含6要素 |

#### 限制

| 限制项 | 值 | 说明 |
|--------|---|------|
| 最大方向数 | 5 | 避免输出过多增加用户决策负担 |
| 趋势扫描深度 | 基础 | 仅基于内置知识，不调用外部搜索 |
| 生成耗时 | <30秒 | MVP性能目标 |

---

### 2.2 阶段2：策划师功能（完整实现）

#### 功能点

| 功能 | 描述 | 验收标准 |
|------|------|---------|
| 矛盾识别 | 识别技术矛盾和物理矛盾 | 能识别并明确定义矛盾 |
| 矛盾矩阵 | 39×39矛盾矩阵查表 | 推荐2-3个发明原理 |
| 原理应用 | 将发明原理转化为解决方案 | 每个原理有具体应用说明 |
| DIKWP审视 | 五层价值审视 | 每层有分析结论 |
| 方案设计 | 输出完整创新方案 | 包含技术路径+差异化+MVP |
| 方案裁剪 | 按核心/增强/可选三级裁剪 | 裁剪后逻辑清晰 |
| 方案衍生 | 生成1-2个变体方案 | 变体有明确差异点 |

#### 限制

| 限制项 | 值 | 说明 |
|--------|---|------|
| 最大TRIZ原理数 | 5 | 避免过度发散 |
| 方案版本数 | 3 | 核心/标准/完整 |
| 衍生方案数 | 2 | 避免过多选择 |
| 生成耗时 | <60秒 | MVP性能目标 |

---

### 2.3 阶段3：分析师功能（完整实现）

#### 功能点

| 功能 | 描述 | 验收标准 |
|------|------|---------|
| 市场分析 | 市场规模和竞争格局分析 | 输出TAM/SAM/SOM |
| 商业模式 | 商业模式画布九宫格 | 每格有内容填充 |
| 财务测算 | 三年财务预测 | 输出收入/成本/利润/ROI |
| 风险评估 | 四类风险识别和应对 | 输出风险矩阵 |
| 综合结论 | 可行性评估和建议 | 输出评分和建议 |

#### 限制

| 限制项 | 值 | 说明 |
|--------|---|------|
| 财务预测年数 | 3年 | MVP仅支持3年预测 |
| 竞品分析深度 | 基础 | 仅基于内置知识 |
| 生成耗时 | <90秒 | MVP性能目标 |

#### 金融合规 ⚠️

MVP版本必须在所有财务相关输出中包含免责声明：

```
【免责声明】
本报告中的财务预测和市场评估基于公开信息和合理假设，仅供参考。
实际结果可能因市场变化、企业执行力等因素而有所不同。
投资决策请咨询专业金融顾问。
```

### 2.4 阶段4：知识产权师功能（简化版）

#### 功能点

| 功能 | 描述 | 验收标准 | 实现方式 |
|------|------|---------|---------|
| 创新点提取 | 从方案中提取创新点 | 输出完整清单 | 内置逻辑 |
| IP分类 | 分类为专利/商标/软著 | 分类合理 | 内置逻辑 |
| IP策略建议 | 建议保护类型和时机 | 建议清晰 | 内置逻辑 |
| 材料清单 | 生成申请材料清单 | 清单完整 | 内置逻辑 |
| 查新检索 | ⚠️ 简化版 | 定性评估 | 内置知识，暂不调用API |

#### 简化说明

**查新检索简化版**：
- 不调用外部专利数据库API
- 基于内置专利知识进行初步评估
- 输出「建议进一步查新」提示
- v0.2将接入智慧芽等API实现完整功能

#### 限制

| 限制项 | 值 | 说明 |
|--------|---|------|
| 创新点数量 | 不限 | 按需提取 |
| IP类型 | 专利/商标/软著 | 暂不支持其他类型 |
| 生成耗时 | <60秒 | MVP性能目标 |

### 2.5 阶段5：设计师功能（完整实现）

#### 功能点

| 功能 | 描述 | 验收标准 |
|------|------|---------|
| 方案整合 | 整合四个阶段的产出 | 结构完整、无遗漏 |
| 格式美化 | 规范化输出格式 | 格式统一、易读 |
| 行动清单 | 生成AI任务+人工任务清单 | 任务清晰、有时间节点 |

#### 限制

| 限制项 | 值 | 说明 |
|--------|---|------|
| 输出格式 | Markdown | 暂不支持Word/PDF导出 |
| 视觉设计 | 文字为主 | v1.0增加可视化 |
| 生成耗时 | <30秒 | MVP性能目标 |

## 三、技术规格

### 3.1 性能指标

| 指标 | 目标值 | 测量方法 |
|------|-------|---------|
| 首次响应时间 | <3秒 | 用户输入到首次响应 |
| 单阶段生成时间 | <2分钟 | 触发到输出完成 |
| 完整流程时间 | <10分钟 | 用户输入到方案包输出 |
| 系统可用性 | 99.5% | 月度正常运行时间 |

### 3.2 容量规格

| 规格项 | 值 | 说明 |
|--------|---|------|
| 并发用户数 | 100 | MVP目标 |
| 会话长度 | 不限 | 无会话长度限制 |
| 上下文保留 | 当前项目 | 不保留跨项目上下文 |
| 存储需求 | 无 | 纯会话交互，无持久化 |

### 3.3 限制清单

| 限制项 | 值 | 超出处理 |
|--------|---|---------|
| 单次输入长度 | 2000字 | 提示用户精简 |
| 方案方向数 | 5个 | 超出则合并 |
| 方案版本数 | 3个 | 超出则提示精简 |
| 衍生方案数 | 2个 | 超出则提示精简 |
| 用户反馈等待 | 30分钟 | 超时发送提醒 |

## 六、风险与应对

### 6.1 技术风险

| 风险 | 等级 | 应对措施 |
|------|------|---------|
| TRIZ分析质量不稳定 | 🟡中 | 内置质量检查，增加案例库 |
| 财务预测偏差大 | 🟡中 | 明确免责声明，提供估算范围 |
| 响应时间过长 | 🟢低 | 优化提示词，控制输出长度 |

### 6.2 业务风险

| 风险 | 等级 | 应对措施 |
|------|------|---------|
| 用户需求过于模糊 | 🟡中 | 提供需求澄清引导 |
| 用户期望过高 | 🟡中 | 明确MVP范围和限制 |
| 知识产权建议不准确 | 🔴高 | 简化版明确提示「建议专业咨询」 |

### 6.3 合规风险

| 风险 | 等级 | 应对措施 |
|------|------|---------|
| 投资建议合规 | 🔴高 | 强制包含免责声明 |
| 知识产权建议合规 | 🟡中 | 明确「非法律意见」 |
| 数据安全 | 🟢低 | 不存储敏感信息 |

## 七、后续规划

### 7.1 短期规划（v0.2）

1. **接入查新API**
   - 接入智慧芽开放平台
   - 实现深度专利检索
   - 生成专业查新报告

2. **用户档案系统**
   - 支持用户注册登录
   - 保存项目历史
   - 支持会话延续

3. **输出增强**
   - 支持Word导出
   - 支持PDF导出
   - 支持一键分享

### 7.2 中期规划（v0.3）

1. **竞品深度分析**
   - 接入天眼查、企查查等数据源
   - 自动抓取竞品信息
   - 生成竞争分析报告

2. **案例库建设**
   - 积累创意孵化成功案例
   - 建立案例标注和检索
   - 支持案例推荐

3. **行业模板**
   - 针对不同行业定制模板
   - 优化各行业分析维度
   - 提供行业Benchmark

### 7.3 长期规划（v1.0）

1. **可视化升级**
   - 增加雷达图、趋势图等可视化
   - 支持图表交互
   - 提供数据仪表盘

2. **生态对接**
   - 与技术经理人事务所API对接
   - 支持项目一键移交
   - 建立服务闭环

3. **团队协作**
   - 支持多人协作
   - 支持权限管理
   - 支持任务分配
