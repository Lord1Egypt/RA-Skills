## 一、核心原则：六大共识

### 共识1：功能吉凶因盘而异（V.K. Choudhry）

**不存在"天然吉星/凶星"这种东西。**

传统文本说"木星是吉星"——但如果木星掌管6宫+3宫（如Taurus上升），它在功能上是凶星。
传统文本说"土星是凶星"——但如果土星入庙7宫+Sasa Yoga（如Leo上升），它是好事。

**规则**：每颗星的吉凶必须先判断功能性质，再判断天然性质。功能性质 = 宫主关系决定。

### 共识2：单一技法不做结论（Marc Boney / K.N. Rao）

**任何单一条规则都不能直接得出结论。**

- DK=Mars ≠ 配偶是战士型（必须加上：Mars是什么宫主？落什么宫？什么状态？）
- 7宫主入庙 ≠ 婚姻一定好（必须加上：D9呢？7宫受什么相位？DK呢？）
- 三系统收敛 ≠ 高置信度（三个弱信号加起来 ≠ 一个强信号）

**规则**：至少3个独立信号指向同一结论，才做判断。且3个信号必须来自不同维度（如：Dasha维度+Transit维度+静态Yoga维度）。

### 共识3：规则的前提条件先查（Marc Boney）

**不是每条规则适用于每个盘。**

- Shasti Hayani Dasha 要求太阳在第1宫 → 不满足则不适用
- Neecha Bhanga 要求定位星在Kendra → 不满足则不成立
- Vargottama 对凶星放大的是凶性，对吉星放大的是吉性
- Yoga的"有"不等于"强"——强度取决于组成行星的整体力量

**规则**：应用任何技法之前，先检查该技法的前提条件是否满足。

### 共识4：案例验证 > 经典引述（K.N. Rao）

**"给我看技术，给我看方法，给我看结果。"**

- 一条规则必须通过至少30-50个真实案例验证才能被采纳
- Rao的《婚姻时间》用了500个案例验证，达到99%成功率
- 经典文本本身可能是后人篡改的（Shyamasundara Dasa的BPHS考证）

**规则**：标注每条结论的证据类型——"经典记载"/"案例验证"/"推断"——权重不同。

### 共识5：先整体后细节（Hart de Fouw / Rao PACDARES）

**先建立盘的整体叙事，再用细节验证。**

类比医学：一个症状可以指向100种疾病，但只有综合所有症状才能确诊。

**步骤**：
1. 先读盘的"大故事"——这个盘的核心主题是什么？
2. 再读具体领域的"小故事"——婚姻/事业/财富各自有什么承诺？
3. 最后用推运验证静态分析——什么时候兑现？

### 共识6：先验证过去，再预测未来（K.N. Rao 核心方法论）

**发展技能的最好方法是不断收集你亲密认识的人的星盘，反复验证。**

- 先用星盘"倒推"用户已经经历过的3-5件关键事件
- 每一个倒推命中，用户对后续预测的信任度翻倍
- 如果倒推失败，必须回到方法论找问题，而不是硬解释

**规则**：在预测未来之前，必须先用已知的过去事件做验证校准。

## 二、PACDARES 本命盘分析框架（K.N. Rao）

这是Rao系统化的本命盘分析方法论，是所有分析的地基。

### P — Position（位置）

对每颗行星检查：
- 落在什么星座？星座性质（火/土/风/水）？主宰什么宫？
- 落在什么宫位？这个宫位的天然征象是什么？
- 度数位置？在星座的前1/3、中1/3还是后1/3？
- 功能性质：这颗星在这个上升盘里是功能吉星还是凶星？

### A — Aspect（相位）

对每颗行星检查：
- 受到哪些行星的相位？（7/4/8/5/9/12宫相位）
- 哪些相位是吉的（木星/金星）？哪些是凶的（土星/火星）？
- 相位的紧密程度（orb）如何？
- 精确Drishti度数分析（特殊度数如Pushkara Navamsha是否被精确相位触及）

### C — Conjunction（合相）

对每颗行星检查：
- 与哪些行星合相？合相的orb如何？
- 合相行星的功能性质是什么？（功能吉星合功能凶星 = 污染）
- 合相产生的特殊Yoga（如Mercury+Venus紧密合相 = 智慧+美学融合）

### D — Dhana Yoga（财富瑜伽）

检查所有财富相关组合：
- 2宫主+11宫主的联系
- 1宫主+2宫主+11宫主的联系
- Jupiter/Venus/Mercury在财富宫的配置
- 注意：财富瑜伽的强度取决于组成行星的力量

### A — Arishta Yoga（灾厄瑜伽）

检查所有困难组合：
- 凶星在角宫/Kendra的影响
- 6宫主+8宫主+12宫主的相互联系
- Kemadruma Yoga（月亮2/12宫无星）
- 关键：**这一步不能跳过**——每个盘都有"问题区域"，必须明确标出

### R — Raja Yoga（皇家瑜伽）

检查所有地位/权力相关组合：
- 1/4/7/10宫主与5/9宫主的联系
- 特殊Raja Yoga（如Sasa Yoga、Neecha Bhanga Raja Yoga）
- 注意区分"有Yoga"和"Yoga强"——后者需要额外条件

### E — Exchange（互换/Parivartana）

检查所有宫位主星互换：
- 哪些宫位的主星互换了星座？
- 互换的效果（吉宫互换 vs 凶宫互换 vs 混合互换）
- 互换带来的特殊能量流动

### S — Special Features（特殊特征）

识别每个盘的独特之处：
- Vargottama行星
- Pushkara Navamsha行星
- Atmakaraka的特殊位置
- 逆行行星
- 燃烧行星
- 特殊度数（精确入旺/落陷度、Sandhi交界点）

## 三、九层复合方法（K.N. Rao Composite Approach）

**做到第六层，平均85%的预测成功率。**

| 层级 | 内容 | 输出 |
|------|------|------|
| **L1** | PACDARES本命盘分析 | 静态承诺清单：每个领域"有什么/没什么/有什么问题" |
| **L2** | 分盘分析（至少D9+D10，推荐6-16个） | 分盘层面的承诺验证/矛盾 |
| **L3** | **综合L1+L2** | 矛盾检查——D1说好但D9说差的地方必须解释 |
| **L4** | Vimshottari Dasha | 大方向——哪个时间段激活什么领域 |
| **L5** | Ashtakavarga + Transit | 精确过境评分——Transit的好坏量化 |
| **L6** | 条件Dasha（Chatursheeti/Dwisaptati/Shodashi等） | 缩小到年/月级精度 |
| **L7** | Jaimini Chara Dasha | 独立验证层 |
| **L8** | 其他Jaimini条件Dasha | 更精细的时间锚点 |
| **L9** | Tajika年度盘确认 | 年/月/周级最终确认 |

### L3 矛盾检查协议（最容易被跳过、也最重要）

**原则**：当不同层面的指示矛盾时，不回避，而是解释。

**矛盾类型**：
1. **D1 vs D9矛盾**：D1说"婚姻好"但D9说"婚姻有挑战"→ 解释：结构上好，但过程中有曲折
2. **吉Yoga vs 凶相位矛盾**：有Raja Yoga但受土星相位→ 解释：成功来得晚且伴随压力
3. **DK vs 7宫主矛盾**：DK指向一种配偶类型，7宫主指向另一种→ 解释：两个维度需要同时满足
4. **静态 vs 推运矛盾**：静态承诺好但推运窗口弱→ 解释：承诺在更远的未来兑现

**处理方式**：
```
矛盾发现 → 记录矛盾 → 解释矛盾 → 标注置信度降低 → 在输出中如实告知用户
```

## 四、验证驱动的分析流程（Verification-Driven Analysis）

### 4.1 为什么要验证过去

Rao的方法论核心：**用户的真实经历是最好的校准数据**。

如果倒推失败率 > 30%，说明：
1. 出生时间可能需要矫正
2. Ayanamsa可能需要调整
3. 技法选择可能有问题
4. 分析逻辑有偏差

### 4.2 倒推验证协议

**步骤1：收集已知事件**
- 至少收集3-5件有明确时间的重大人生事件
- 事件类型：感情、事业变动、搬迁、健康、重大决定
- 时间精度：精确到月份最佳，季度可接受

**步骤2：事件→Dasha映射**
- 每个事件对应什么Dasha（MD/AD/PD）？
- 事件发生时是否有Transit触发？
- Dasha/Transit的征象是否与事件性质匹配？

**步骤3：评估匹配度**
- 5/5事件匹配 → 引擎和分析方法论高度可靠
- 4/5事件匹配 → 基本可靠，1个偏差需要解释
- 3/5事件匹配 → 需要检查出生时间或调整分析参数
- <3/5事件匹配 → 必须停止预测，先解决校准问题

**步骤4：校准偏差**
- 如果系统性偏差（如所有事件都早了1年）→ 调整出生时间
- 如果特定领域偏差（如感情事件都不匹配）→ 检查该领域的征象星选择
- 如果偏差无规律 → 回到PACDARES重新做静态分析

### 4.3 正推预测协议

**只有在倒推验证通过后，才进入正推预测。**

**步骤1：确认承诺**（从PACDARES得出）
- "这个盘有没有婚姻承诺？" → 有/没有/有但受限
- "承诺的形态是什么？" → 早婚/晚婚/困难但最终成功

**步骤2：定位窗口**（从L4-L9得出）
- 大方向（哪个10年区间）
- 中方向（哪个2-3年区间）
- 小方向（哪个季度/月份）

**步骤3：描述形态**（从矛盾检查得出）
- 不仅仅是"什么时候"，还要描述"什么样的"
- 包含矛盾的解释——"会有XX挑战，但最终YY"

## 五、置信度分级系统

### 5.1 结论的三级分类

| 级别 | 定义 | 标记 | 示例 |
|------|------|------|------|
| **A-已验证** | 经过用户已知事件倒推验证的结论 | ✅ [A] | "2014年感情经历与木星入旺过境DK火星吻合" |
| **B-强推断** | 3个以上独立维度指向同一方向，但未经事件验证 | ⭐ [B] | "2027-2030是强婚姻窗口（5/6 Dasha系统支持）" |
| **C-假设** | 单一维度或理论推导，缺乏多系统支撑 | ⚡ [C] | "配偶可能是科技行业（Rahu在7宫）" |

### 5.2 输出规范

**每条关键结论必须标注置信度级别**。格式：

```
结论：XXXXX
置信度：[A/B/C]
依据：PACDARES分析→L3矛盾检查→L4-6推运
矛盾：有/无（如有，简述）
```

### 5.3 预测的诚实边界

**Rao的原话**："在物理学中90%的结果就称之为伟大。在医学中60%就批准药物。"

- ✅ 可以做的：描述趋势、给出窗口、提示风险
- ⚠️ 需要谨慎的：精确到月份、具体事件描述
- ❌ 不应该做的：绝对断言、确定性预言、具体到日期

## 七、经典文献的审慎使用（Shyamasundara Dasa）

### 7.1 BPHS的真实性问题

现代BPHS**不是原始文本**，包含大量19-20世纪后人插入：
- 不平等宫位制（Sripati系统）——后来插入
- Jaimini体系内容——BPHS中的Jaimini被正统Jaimini导师告诫"忽略它"
- 补救措施中敬拜各种天神——与Parashara已知的生平和教导矛盾

### 7.2 使用原则

1. **BPHS不能作为独立的权威来引用**，尤其当它与其他古代文献矛盾时
2. 当BPHS与Brihat Jataka矛盾时，优先采用Brihat Jataka
3. 任何技法应当通过实际案例验证，而非仅因"经典记载"就被接受
4. 南印度传统使用的"五大经典"（Brihat Jataka、Khadaka Paddhati、Phaladipika、Saravali、Jataka Parijata）应与BPHS对照参考

### 7.3 推荐的文献优先级

1. **Brihat Jataka**（Varahamihira，6世纪）——最可信的古典基础
2. **Saravali**（Kalyana Varma，10世纪）——系统化解读
3. **Phaladipika**（Mantreswara，13世纪）——实战导向
4. **BPHS**（核心部分，需区分原始与插入）
5. **Jaimini Sutras**（独立于BPHS学习，不从BPHS中学习Jaimini）

## 八、实战精准确课五步法

### 总结：从静态到动态的完整流程

```
Step 1: PACDARES静态分析
   → 建立"承诺清单"：每个领域 有/没有/有但受限
   → 识别特殊特征和矛盾

Step 2: 分盘验证+矛盾检查（L2-L3）
   → D9/D10/D7等分盘验证或否定D1的结论
   → 矛盾必须明确记录和解释

Step 3: 倒推验证
   → 用用户已知的3-5件重大事件做校准
   → 匹配度 <60% 则停止，回到Step 1

Step 4: 推运定位（L4-L9）
   → Vimshottari → 条件Dasha → Transit → Jaimini → Tajika
   → 逐层缩小时间窗口
   → 每层独立结论 → 检查一致性

Step 5: 输出
   → 三级置信度标注
   → 包含矛盾解释
   → 诚实标注预测边界
```
