# swagger-skills

这是一个独立的 Swagger/OpenAPI skill 生成目录。

## 安装依赖

推荐使用稳定版 Python 3.10+，并在本地虚拟环境或当前 Python 环境中安装依赖后执行生成器。

如果本机缺少稳定版 Python 3.10+，脚本不会自动安装 Python 本体，只会返回安装指引。可选方式：

```powershell
winget install Python.Python.3.12
uv python install 3.12
```

也可以从 Python 官网下载安装。

也可以手动提前安装依赖：

```powershell
cd <swagger-skills目录>
python -m pip install -r requirements.txt
```

## 配置接口文档

复制示例配置：

```powershell
Copy-Item config\sources.example.json config\sources.json
```

编辑 `config/sources.json`，将 `swagger_url` 替换为真实 Swagger UI 页面链接。如果已知真实 spec 地址，也可以填写 `spec_url`。

## 生成接口 Skill 内容

默认 `--output-mode auto`。当 `sources.json` 里只有一个链接时，会产出 1 个独立 skill；当有多个链接时，会询问你选择：

- `combined`：多个接口文档合并产出 1 个 skill。
- `separate`：每个接口文档分别产出 1 个 skill。

```powershell
python scripts\build_swagger_skill.py --clean-generated
```

也可以直接指定产出模式，避免交互询问：

```powershell
python scripts\build_swagger_skill.py --output-mode combined --clean-generated
python scripts\build_swagger_skill.py --output-mode separate --clean-generated
```

默认生成结果会直接放在 `swagger-skills` 的同级目录下，避免把生成器 skill 和业务产物 skill 放在父子目录中被重复扫描注册：

- 合并模式：`..\combined-swagger-skills\`
- 分开模式：`..\<source_id>-swagger-skills\`

例如生成器位于 `<parent>\swagger-skills`，`admin-config` 的产物会位于 `<parent>\admin-config-swagger-skills`。

`--clean-generated` 只会清理同级目录下带有 `Generated by swagger-skills` 标记的产物 skill，不会清理任意同级目录。

如需自定义产物根目录：

```powershell
python scripts\build_swagger_skill.py --output ..\my-swagger-skills-root --clean-generated
```

生成的文件夹和文件名只使用英文、数字和下划线。生成器会优先使用英文 tag、operationId、path 片段或 `source_id`；中文名称不会转换为拼音，也不会直接进入路径。

进入具体产出的 skill 后，从该目录内的 `SKILL.md` 查看业务接口覆盖范围，从 `FEATURES.md` 查找功能，再进入对应 `api_clients/` 调用文件和 `references/` 字段说明文件。

产物运行时架构：

- 域名：`scripts/skill_common.py`（配置源 `config/domains.json`，支持 `<DOCUMENT_ID>_ORIGIN` 环境变量覆盖）
- HTTP + 字段映射：`scripts/skill_http.py`（默认开启映射，索引来自 `openapi_field_index.json`）
- 字段查询：`python scripts/openapi_fields.py lookup <字段名>`
