## Description: <br>
Performs OpenClaw system health checks, security audits, version checks, and scheduled monitoring configuration for machines running OpenClaw. <br>

This skill is ready for commercial/non-commercial use. <br>

## Use Case: <br>
Developers and operators use this skill to assess OpenClaw host health, review security posture, inspect service and dependency status, and configure recurring local checks. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: The cron option can configure recurring local checks that continue after the setup session. <br>
Mitigation: Enable scheduled checks only when recurring monitoring is desired, and review local cron configuration after setup. <br>
Risk: Healthcheck audit output and logs may contain sensitive local system, network, or service details. <br>
Mitigation: Treat reports and logs as sensitive operational data and limit sharing to trusted reviewers. <br>
Risk: Repair or hardening recommendations may affect host configuration. <br>
Mitigation: Review suggested remediation before applying changes, and confirm the local openclaw CLI is the expected trusted binary. <br>
