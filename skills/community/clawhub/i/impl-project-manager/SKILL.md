# 实施项目经理工具箱

IT/软件交付项目的全流程管理 Skills，以里程碑为核心驱动，飞书在线表格为数据源和交付物。

## 核心模式

**PM 在飞书表格录入 → Skills 读取分析 → 对话确认 → 自动更新飞书表格**

飞书表格是唯一数据源，不维护本地文档副本。

## 核心模块

1. 项目管理（立项、里程碑、风险、变更）
2. 主合同签订与回款跟踪
3. 采购分包与付款跟踪

## 飞书对接

### 配置

应用凭证见 [references/feishu-config.md](references/feishu-config.md)

### 脚本

| 脚本 | 用途 |
|------|------|
| `scripts/feishu_api.py` | 飞书 API 基础（token 获取、通用请求） |
| `scripts/feishu_sheet.py` | 飞书表格读写操作 |
| `scripts/calc_engine.py` | 项目管理计算引擎（里程碑推算、金额计算、风险识别、预警判断） |

### 使用方式

1. PM 在飞书表格中录入/修改数据（立项信息、里程碑日期、回款比例等）
2. 调用 `feishu_sheet.py read` 读取表格数据
3. 调用 `calc_engine.py` 自动计算（金额、状态、风险、预警）
4. 通过对话呈现分析结果，PM 确认后
5. 调用 `feishu_sheet.py write` 更新飞书表格中的计算字段

## 工作流程

### 新项目启动

1. PM 在飞书表格中填写立项信息（项目名称、客户、预算、干系人、范围）
2. PM 填写需求确认日期，回款比例
3. Skills 读取表格 → 自动推算里程碑日期 → 计算回款金额 → 写回表格

**里程碑推算规则：**
- M1 需求确认：PM 提供
- M2 试运行完成：M1 + 3个月
- M3 验收通过：M2 + 半个月
- M4 质保完成：M3 + 1年

**干系人角色：** 客户方对接人 + 我方交付负责人 + 我方实施经理

### 日常更新

- PM 在飞书更新里程碑实际完成日期、回款/付款实际数据
- Skills 读取 → 自动计算状态、预警、风险 → 写回表格
- 变更记录：PM 提供变更描述，Skills 评估三维度影响后写入表格
- 分包录入：PM 提供分包商信息 + 各里程碑付款比例，Skills 计算付款金额并写入

### 文档生成

按需从飞书表格数据生成中文 markdown 文档：

| 文档 | 触发时机 |
|------|----------|
| 项目计划书 | 项目启动 |
| 进度周报 | 定期/按需 |
| 合同回款跟踪表 | 合同/回款变更 |
| 分包付款跟踪表 | 分包/付款变更 |
| 项目状态快照 | 随时 |
| 结项复盘报告 | 项目关闭 |

## 自动规则

### 风险自动识别

| 规则 | 触发条件 | 等级 |
|------|----------|------|
| 里程碑延期 | 计划日期已过且未完成 | 🔴高 |
| 里程碑即将到期 | ≤7天且未完成 | 🟡中 |
| 需求变更频发 | 单里程碑变更≥3条 | 🟡中 |
| 预算超支风险 | 已发生成本>预算×80% | 🔴高 |

### 回款/付款预警

| 级别 | 触发条件 | 标记 |
|------|----------|------|
| 即将到期 | 距计划日期≤7天且未完成 | 🟡 |
| 已逾期 | 超过计划日期且未完成 | 🔴 |

### 里程碑状态自动流转

- 前置完成 → 后继自动进入"进行中"
- 填写实际完成日期 → 标记"已完成"
- 计划日期已过无实际日期 → 标记"已延期"

## 变更管理

每条变更必须评估三维度影响：
1. **成本**：预算变化及金额
2. **工期**：工期变化及天数
3. **范围**：交付范围变化及描述

## 多项目管理

采用每个项目独立表格模式，通过项目总目录统一管理。

### 项目总目录
- **总目录表格：** https://my.feishu.cn/sheets/PkIasY146hifjptNC69cdk6knEb
- 记录所有项目的名称、客户、预算、状态、表格链接
- 新项目自动在此登记

### 新项目流程
1. PM 说「新建项目」→ 逐项提问收集立项信息
2. 确认需求确认日期 → 自动推算里程碑
3. 确认回款比例 → 自动计算金额
4. AI 创建新飞书表格（5个Sheet：项目总览、合同回款、分包付款、变更记录、风险清单）
5. **自动设置编辑权限**（组织内可编辑）
6. 写入项目数据 → 在总目录登记
7. 告诉 PM 新表格链接

### 日常操作
- PM 说「看XX项目」→ AI 从总目录定位表格 → 读取数据
- PM 说「更新XX项目」→ AI 定位表格 → 读取分析 → 对话确认 → 更新飞书表格

## 数据边界

**不包含：** 签约方银行账户、个人隐私信息、分包商绩效评价。
**数据源：** 飞书在线表格（唯一数据源，不维护本地副本）。
**默认权限：** 所有新建表格自动设置为组织内可编辑（`tenant_editable`）。
