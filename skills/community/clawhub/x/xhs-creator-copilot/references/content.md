# content.md — 笔记内容生成与选题辅助

> **文档类型**:Skill 参考文档(笔记起草)
> **适用 Skill**:`xhs-creator-copilot` v1.1+
> **目标路径**:`references/content.md`
> **版本**:1.1.0

---

## 0. 本模块作用

负责两件事:

1. **选题辅助**:根据用户提供的素材(官方后台截图、收藏笔记、跨平台指数等),帮用户选话题
2. **三版本笔记起草**:干货教学型 / 故事叙述型 / 数据播报型,起草后自动走 humanize → 合规预检 → 本地输出

**本模块不会**:
- ❌ 自动抓取小红书任何数据
- ❌ 联网访问平台
- ❌ 调用任何浏览器工具

**素材完全由真人提供**(粘贴文本 / 上传截图 / 描述记忆)。

## 1. 选题辅助

### 1.1 输入源(全部由真人提供)

| 素材类型 | 用户操作 | skill 处理 |
|---------|---------|----------|
| 官方后台热点截图 | 上传创作服务平台首页截图 | 视觉识别活动列表 + 笔记灵感 |
| 个人收藏清单 | 粘贴收藏笔记的标题 | 聚类分析话题方向 |
| 跨平台趋势 | 粘贴微信指数 / 百度指数截图 | 提取热度关键词 |
| 用户记忆 | 文字描述"我最近想到 X 话题" | 直接基于此扩展 |
| 历史数据 | skill 读 `data/publish_log.json` | 识别用户已发过什么 |

### 1.2 选题分析输出

```markdown
# 选题建议(2026-04-25)

## 输入素材汇总
- 官方热点活动:RED新生代创作大赛、星布谷地
- 你的收藏:5 条笔记(主题集中在止损纪律、保证金管理)
- 微信指数热词(期货相关):豆粕、棕榈油、降息预期
- 你近期已发:3 篇都是干货型

## 推荐选题(5 个候选)

### 选题 1 ⭐(推荐度最高)
**主题**:豆粕这波怎么看 — 我的判断逻辑
**类型**:故事叙述型(B 版)
**理由**:
- 豆粕是当前微信指数热词
- 你近期 3 篇都是干货型,B 版能换换节奏
- 故事型适合你想引导关注的 goal

### 选题 2
...

### 选题 3
...
```

### 1.3 选题红线

skill **不**推荐以下类型选题(即使热度高):
- ❌ "稳赚 / 翻倍 / 暴赚"叙事 → 合规违规
- ❌ "代客理财 / 跟单 / 喊单"导向 → 平台禁止
- ❌ 模仿其他博主的具体笔记 → 同质化风险
- ❌ 平台敏感事件(政策不确定的话题)→ 限流风险

## 2. 三版本笔记起草

每次起草**必须生成 3 个版本**(A 干货 / B 故事 / C 播报),让用户选。这是从 v1.8 沿用并强化的设计。

### 2.1 版本 A:干货教学型

**定位**:结构化知识输出,高收藏率,适合 `引导收藏` / `引导关注` 类目标。

**结构骨架**:

```
[钩子句 ≤30字]              ← 提出问题或反常识
     ↓
[核心概念定义 ≤100字]        ← 1 句话讲清是什么
     ↓
[3 个要点](或 2 个、4 个,避免每篇都是 3 个)
  要点1:标题 + 50字解释 + 具体例子
  要点2:同上
  要点3:同上
     ↓
[对比/总结 ≤80字]            ← 可选
     ↓
[免责声明]                   ← 强制
     ↓
[行动引导] ≤30字              ← 关注/收藏导向
```

**语言风格**:
- 第二人称为主("你""我们")
- 关键术语首次出现加括注:`保证金(交易所要求的履约担保金)`
- 禁用绝对化表述

**Humanize 处理**:起草完成后自动走 medium 等级,重点处理模式 #06(三段排比)、#10(总分总死结构)、#21(缺具体细节)。

### 2.2 版本 B:故事叙述型

**定位**:情感共鸣,评论区活跃,适合 `引导关注` / `引导私信` 类目标。

**结构骨架**:

```
[个人经历引子 ≈100字]        ← 时间、地点、具体场景
     ↓
[转折点 ≈80字]              ← 意料之外发生了什么
     ↓
[思考/复盘 ≈150字]           ← 学到什么
     ↓
[结论升华 ≈50字]             ← 与主题呼应(避免空洞升华)
     ↓
[互动问题 ≤30字]             ← "你有过类似经历吗?"
     ↓
[免责声明]                   ← 强制
```

**语言风格**:
- **必须**第一人称("我""我当时")
- **必须**有具体时间锚点("2024 年 4 月""上周三")
- **必须**有具体数字("亏了 2 万 8""扛了 5 分钟")
- 适度自嘲(期货圈有效)

**Humanize 处理**:走 medium 等级。故事型最容易出 AI 味(因为 AI 倾向"完美叙事"),humanize 时重点处理模式 #20(完美主义文风)、#19(客观中立腔)、#15(升华癖)。

### 2.3 版本 C:数据播报型

**定位**:信息密度高,低情感,适合每日快讯 / 高频发布场景。

**结构骨架**:

```
[数据标题 ≤20字]             ← 包含核心数字
     ↓
[核心数据 3 条]
  📊 数据1:指标 + 数值 + 环比/同比
  📊 数据2:同上
  📊 数据3:同上
     ↓
[简短分析 ≤150字]            ← 数据背后的原因/影响
     ↓
[后市关注点 ≤80字](可选)
     ↓
[免责声明]                   ← 强制
     ↓
[信源标注]                   ← "数据来源:上期所"
```

**语言风格**:
- 客观中性,不带情绪词
- 数字精确到合适量级("上涨 12.3%" 而非"大涨")
- 使用 emoji 分隔数据条目

**Humanize 处理**:走 light 等级(播报型本身就该是相对客观的;过度 humanize 反而失专业感)。

## 3. 字数与标题规则

### 3.1 字数硬上限

| publish_type | 标题硬上限 | 正文建议 |
|-------------|-----------|---------|
| 图文 | 20 字 | 500-1000 字 |
| 长文 | 64 字 | 1500+ 字(享受"字数和时长"徽标)|

### 3.2 标题公式

| 版本 | 公式 | 示例 |
|------|------|------|
| A 干货型 | `[数字]+[核心概念]+[价值承诺]` | "3个指标看懂期货保证金" |
| A 干货型 | `[核心概念]+[对比/区别]` | "期货和股票的 3 个本质区别" |
| B 故事型 | `[时间/数字]+[身份]+[转折]` | "10 万亏 18% 那周,我才懂..." |
| B 故事型 | `[场景]+[顿悟]` | "K 线盯到凌晨 3 点,我才懂杠杆" |
| C 播报型 | `[日期]+[品种]+[数字]` | "4.25 期货收盘 \| 沪铜+0.8%" |
| C 播报型 | `[数字]+[品种]+[动向]` | "3 大能化品普跌,沥青领跌 1.2%" |

### 3.3 标题红线

- ❌ 具体收益数字("我靠期货赚了 50 万")
- ❌ 诱导性承诺("看完必赚")
- ❌ 独家/内幕词汇

## 5. 合规预检(强制)

每个版本生成后,**必须**过合规扫描。**期货/财经赛道是平台 2026 年点名的"审核加严类目"**,任何高风险词都立即拦截(不留余地)。

### 5.1 敏感词扫描(立即拦截)

发现以下词,**强制替换或拒绝**:

```
稳赚、必涨、必跌、保本、内幕消息、翻倍、跟单、喊单、居间、
一夜暴富、保证盈利、稳赢、零风险、无风险、必胜、暴富
```

### 5.2 高风险表述(标注提醒)

- 具体收益率数字("月收益 30%")→ 必须删数字或加风险提示
- 确定性预测("一定会涨")→ 改概率性表述
- 承诺性收益 → 全部删

### 5.3 免责声明(必须存在)

每个版本结尾必须含**至少**一句:
- 不构成投资建议
- 仅供参考
- 理财有风险
- 投资有风险
- 入市需谨慎

如缺失,skill 自动追加默认句:`以上仅为个人分享,不构成投资建议。`

### 5.4 AI 内容标识(v1.1.0 强制)⭐

每个版本生成后,skill **自动在正文末尾追加**:

```
本文部分内容由 AI 辅助生成。
```

这是 2026-02-12 小红书 AI 标识公告 + GB 45438-2025 国标的强制要求,
**用户不得删除该标识**(humanizer 处理时也不得移除)。

最终发布时,真人**仍需**在小红书后台勾选「设置 → 内容类型声明 → 笔记含 AI 合成内容」(双重标识)。

### 5.5 故事营销红线(v1.1.0 新增)⭐

针对**版本 B 故事叙述型**,skill **严禁**生成以下内容:

- ❌ 编造**具体对话**("我朋友老张跟我说...")
- ❌ 伪造**聊天记录截图**(skill 不生成此类素材)
- ❌ 伪装**特定身份**("我是 XX 公司前研究员")
- ❌ 伪造**客户反馈**("有学员跟我说...")

**允许的故事内容**:
- ✅ 用户**自己真实的经历**(用户口述,skill 复述)
- ✅ 公开的**真实事件**(有出处可查)
- ✅ 明确标注的**虚构案例**("举个例子,假设有人...")

### 5.6 矩阵账号风险提示(v1.1.0 新增)⭐

如果 skill 检测到用户**最近 3 天内已生成 ≥ 5 篇笔记**,主动提示:

> ⚠️ 你最近 3 天已生成 5 篇笔记。平台 2026 新规对"批量发相似内容"识别加强,
> 建议:① 错峰发布(每天 ≤2 篇)② 加入更多个人真实经历 ③ 内容结构错开

## 7. 用户选择后的流程

用户回复 A / B / C 后:

1. skill 把选中的版本调用 `local-output.md` 输出到 `outputs/YYYY-MM-DD/`
2. 文件命名:`note-{type}-{topic-slug}-v{ABC}.md`
3. 输出末尾**必须**包含 §3.4 提示(下一节)

### 7.1 必含的发布提示

```markdown
---

## 📌 真人发布提示

✅ 内容已保存到:`outputs/2026-04-25/note-image-text-baozhengjin-vA.md`

发布步骤(由你手动完成):
1. 打开小红书 App 或 Web 创作服务平台
2. 选择对应类型(图文/长文)
3. 复制本文件内容粘贴
4. 上传配图(如有)
5. 🏷️ **必须**:在「设置 → 内容类型声明」勾选「**笔记含 AI 合成内容**」
6. 检查预览,点击发布

⚠️ 节奏提醒:
- 你最近 7 天发布了 N 篇,频次正常 / 需放缓
- 同主题内容已有 M 篇,建议下次换方向

依据《人工智能生成合成内容标识办法》,
未主动标识 AI 内容可能被平台限流。
```
