# 版本存档模板

> 本模板定义了版本存档的标准流程与输出结构。存档时参照此模板执行，生成新的 `[小说标题]_vN/` 目录。

---

## 存档触发条件

仅在用户**明确要求**时执行存档：
- 用户说"存成 vN"、"这是终稿了"、"存档"等类似表述
- 按创建顺序递增命名：`[小说标题]_v1/`、`[小说标题]_v2/`…

---

## 命名规则（文件编号 = 章编号）

**所有章节文件名使用全局连续三位数编号，不随卷重置：**

| 卷 | 章节范围 | 文件名 |
|----|---------|--------|
| 卷 1 | 第 1-30 章 | `chapter-001.md` ~ `chapter-030.md` |
| 卷 2 | 第 31-60 章 | `chapter-031.md` ~ `chapter-060.md` |
| 卷 3 | 第 61-100 章 | `chapter-061.md` ~ `chapter-100.md` |
| … | … | … |
| 卷 N | 最后一卷 | `chapter-XXX.md` |

**关键规则：**
- ❌ 禁止每卷内从第 1 章重新开始编号（不要出现卷 1 有 1-30 章、卷 2 也有 1-30 章的情况）
- ✅ 文件名直接反映全局章号：`chapter-031.md` = 第 31 章，`chapter-271.md` = 第 271 章
- ✅ 三位数补零：`chapter-001.md`、`chapter-300.md`

## 执行步骤

### 步骤 1：创建存档目录

```bash
N=下一个序号（从现有最大序号 +1）
mkdir -p "[小说标题]_v${N}/"
```

### 步骤 2：复制只读文件

将 `在编辑稿/` 下的非正文文件原样复制到存档目录：

```bash
cd "[小说标题]_v${N}/"
# 复制根文件
cp ../在编辑稿/01-story-premise.md ./
cp ../在编辑稿/02-world-setting.md ./
cp ../在编辑稿/03-chapters-written.md ./

# 复制大纲目录
cp -r ../在编辑稿/03-outline/ ./

# 复制参考文件
cp -r ../在编辑稿/references/ ./
cp -r ../在编辑稿/media/ ./

# 复制报告
cp -r ../在编辑稿/05-reports/ ./
```

### 步骤 3：复制正文为 txt

`在编辑稿/04-chapters/` 下所有章节已使用全局连续三位数编号，直接转为 txt 复制：

```bash
mkdir -p chapters
for vol_dir in ../在编辑稿/04-chapters/vol-*; do
    vol_num=$(basename "$vol_dir")
    mkdir -p "chapters/${vol_num}"
    for chapter_file in "$vol_dir"/*.md; do
        # 文件名已是全局连续三位数（chapter-NNN.md），直接转换格式
        sed 's/^#\+ //; s/`//g; s/\*\*/\n/g' "$chapter_file" > "chapters/${vol_num}/$(basename "$chapter_file" .md).txt"
    done
done
```

### 步骤 4：生成全文合并版

将 `chapters/vol-*/chapter-*.txt` 按**全局章节号**顺序合并：

```bash
# 按卷 + 章号排序后合并（文件名已是三位数全局连续编号）
cat chapters/vol-*/chapter-*.txt > [小说标题]_全文.txt
```

**文件名 = 章编号 = 全局连续三位数**：`chapter-001.txt`（第 1 章）、`chapter-300.txt`（第 300 章），不随卷重置。

### 步骤 5：生成目录

生成 `[小说标题]_目录.txt`，格式：

```
# [小说标题] 目录 — vN

## 第 001 章 [章节标题] → chapters/vol-1/chapter-001.txt
## 第 002 章 [章节标题] → chapters/vol-1/chapter-002.txt
……
## 第 300 章 [章节标题] → chapters/vol-10/chapter-300.txt
```

**关键：章节号全局连续编号，不随卷重置。**
- 卷 1 第 1 章 = 第 001 章
- 卷 2 第 1 章 = 第 031 章（如果卷 1 有 30 章）
- 卷 10 第 1 章 = 第 271 章（以此类推）

### 步骤 6：生成人物画像 JSON

扫描所有章节，提取**主要角色画像**，生成 `char_data.json`：

```json
{
  "project": "[小说标题]",
  "version": "vN",
  "total_chapters": 300,
  "characters": [
    {
      "name": "主角名",
      "role": "主角",
      "portrait": "media/comfyui/images/主角图片.png",
      "description": "主角的简要描述",
      "key_arc": "成长线 1 → 成长线 2",
      "first_appearance": "第 1 章",
      "affinity": "主角简称"
    }
  ]
}
```

**生成规则：**
- **主要角色**：指在正文中出现频率高、推动主线/关键剧情的角色
- 至少包含：主角、女主/重要配角、核心灵体、关键引导者
- 次要角色（每卷单元灵）可在 `char_data.json` 中以 `minor_characters` 字段补充

### 步骤 7：生成 HTML 导引文件

生成 `[小说标题]_导引.html`，要求：

#### 页面结构

1. **页面头部**
   - 小说标题 + 版本信息 + 统计数据
   - 示例：`[小说标题] v3 · 12 角色 · 300 章`

2. **全文简要介绍**
   - 一段对整个小说的简要概述（200-300 字）
   - 包含核心主题、故事走向、整体氛围
   - 从 `01-story-premise.md` 和 `02-world-setting.md` 提取

3. **人物卡区域**
   - 响应式网格布局（CSS Grid）
   - 每人一张卡片，包含：
     - 角色图片（`portrait` 字段）
     - 角色名 + 角色定位（如"主角"、"灵体"）
     - 简要描述（1-2 句话）
   - 暗色主题，hover 高亮边框

4. **章节目录区域**
   - 按卷分组，每卷一个区块
   - **章节号全局连续编号**（关键要求）
   - 每章一个超链接指向 `chapters/vol-X/chapter-NNN.txt`
   - 格式：`第 001 章 [标题]`

5. **页脚**
   - 版本信息、生成时间、统计数据

#### 章节编号规则（非常重要）

- ❌ **禁止**：每卷内从第 1 章重新开始编号
- ✅ **必须**：全局连续编号（卷 1 第 1 章 = 第 001 章，卷 10 第 1 章 = 第 271 章）
- 编号格式：三位数（第 001 章、第 300 章）
- HTML 目录中显示为 `第 001 章 · 章节标题`

#### 样式要求

- 暗色主题（背景 `#0a0a0f`，卡片 `#12121a`）
- 响应式布局（CSS Grid，移动端适配）
- 字体：`PingFang SC, Microsoft YaHei, sans-serif`
- 章节链接 hover 效果

### 步骤 8：完成检查

存档完成后验证：
- [ ] 所有章节已提取为 txt
- [ ] 全文合并版完整（无缺失、无重复）
- [ ] 目录章节号全局连续
- [ ] HTML 导引页人物卡完整
- [ ] HTML 导引页章节链接全部有效
- [ ] 全文介绍已添加
- [ ] 存档目录内无可编辑文件（只读确认）

## 反向合并

当存档版本与 `在编辑稿` 不一致时：

1. 先确认以哪个版本为准
2. **以 `在编辑稿` 为准**：不需要反向合并
3. **以存档版为准**：将存档版全文按章节拆分，覆盖 `在编辑稿/04-chapters/` 中的对应文件

---

## 注意事项

- ❌ 不要在存档版本上做任何编辑
- ❌ 不要在 `在编辑稿/` 之外创建或修改内容
- ✅ 生成存档时确保所有章节已正确排序和合并
- ✅ 如果用户没有明确要求存档，不要擅自创建 vN 目录
- ✅ **文件编号 = 章编号 = 全局连续三位数**，不要每卷从 1 开始
- ✅ HTML 导引文件必须包含：人物卡 + 全文介绍 + 连续编号章节目录
