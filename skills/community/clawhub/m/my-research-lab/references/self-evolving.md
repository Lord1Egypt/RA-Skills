# 自进化引擎详细规则

**关键设计原则：自进化不是"写在 SKILL.md 里等着被执行"的被动规则，而是必须写进定时任务 prompt 中的强制步骤。**

## 为什么之前不触发？

```
❌ 定时任务 prompt 里只写了"采集→推送"，没有"自进化"步骤
❌ AI 运行定时任务时只看 prompt，不会自动去读 SKILL.md 的自进化章节

现在的解法：
✅ 定时任务 prompt 中显式包含自进化步骤（步骤11-15）
✅ 每日发现结束后强制执行信源调权+反馈验证+写入记忆
✅ 每周头脑实验结束后输出信源健康周报
✅ 即使前面步骤失败，自进化中的"写入记忆"仍然执行
```

## 1. 信源库自动维护

```
执行时机：每次每日发现运行后

维护逻辑：
  □ 本次命中高价值内容（评分≥4.0）的信源 → weight += 0.1（上限2.0）
  □ 本次没有命中的信源 → 连续计数+1
  □ 连续3次没命中 → weight -= 0.2（下限0.1）
  □ 连续5次没命中 → 标记为"待审核"，下次对话告知用户是否保留
  □ 用户反馈"这个来源不错" → 该信源 weight = max(weight, 1.5)
  □ 用户反馈"这个来源不好" → 该信源 weight = 0.1

验证方式：
  每周在头脑实验前，输出信源健康报告（静默记录到记忆）：
  "本周信源健康：[X]个活跃 / [Y]个待审核 / [Z]个低权重"
```

领域专属信源 skill 的命中也参与调权统计。

## 1.5 多领域覆盖自检（用户配置了多个 domains 时启用）

```
执行时机：每次每日发现运行后

检查逻辑：
  □ 如果用户配置了多个 domains（研究子方向）
  □ 统计本期简报覆盖了哪些 domain
  □ 连续3期只聚焦某一个 domain → 自动在下一期调整搜索关键词偏向
  □ 不强制每期覆盖所有 domain（自然波动是正常的）

未配置 domains 时 → 跳过此检查（向下兼容）
```

## 2. 反馈驱动的规则进化

```
执行时机：每次用户给出反馈时立刻执行

反馈类型和处理：
  "这条没用" → 
    · 记录到反馈规则库
    · 分析为什么没用（话题不对？太浅？已知？）
    · 生成过滤规则，写入记忆
    · 下次质检时应用这条规则
  
  "这条很好" →
    · 记录到正向反馈库
    · 分析为什么好（话题命中？深度够？实用？）
    · 提高相关信源权重
    · 调整搜索关键词偏好
  
  "简报太长/太短" →
    · 调整 maxItemsPerDay
    · 记录偏好变化
  
  "大师点评没用" →
    · 记录具体哪位大师的点评没用
    · 调整该大师的参与频率
    · 严重时从轮值表移除

验证方式：
  每条反馈必须有 actionTaken 字段，不能只记录不行动
  下次运行时，检查反馈规则是否被应用
```

## 3. 定时任务健康检查

```
执行时机：每次院长被唤起对话时

检查清单：
  □ 每日发现的定时任务是否在运行？
  □ 头脑实验的定时任务是否在运行？
  □ 最近一次执行是否成功？
  □ 推送渠道是否还在可用？（邮箱授权码会过期）
  □ 沉淀空间是否还在可写入？
  □ 有没有未处理的错误日志？

发现问题 → 自动修复或立刻告知用户
绝对不能：等用户自己发现问题
```

## 4. 推送失败自动重试

```
第1次失败 → 等30秒重试
第2次失败 → 尝试备用渠道
都失败 → 存为本地文件 + 下次对话告知：
  "昨天的简报推送失败了（原因：[XXX]）。内容已存在 [路径]。
   需要我排查修复吗？"
```

## 5. 错误日志标准格式

```json
{
  "timestamp": "2026-04-08T09:15:00",
  "category": "push|search|qc|cron|archive",
  "step": 6,
  "severity": "warning|error|critical",
  "message": "邮件发送失败：SMTP连接超时",
  "autoFixAttempted": true,
  "autoFixResult": "重试后成功",
  "needsUserAction": false
}
```

连续3天同类错误 → 升级为 critical，对话开头优先提醒。

## 大师轮值排班

```
执行时机：每次每日发现运行后

逻辑：
  · 大师 ≥3 位时启用轮值
  · 每日发现：每天轮值 2 位大师
  · 每周自动轮换轮值表
  · 用户反馈"这周让 X 多参与"→ 调整轮值优先级
  · 大师点评长期被反馈"没用"→ 降低参与频率，严重时移除

每次运行后更新：
  · 记录今日参与了哪些大师
  · 计算下次轮值名单
  · 写入 lab-config.json 的 masterRotation 字段
```
