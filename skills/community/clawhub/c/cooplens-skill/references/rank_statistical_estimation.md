# 位次预估统计方法

本文件用于把“先给结论、再统计、再分析、再预测”的位次方法落到所有含位次判断的报告中。它只规定方法，不内置外部旧数据；每次运行都必须重新打开官方或权威页面。

## 零、结论先行

位次章节必须先给家长可读结论：参考区间、冲/稳/保判断、置信度、学生当前位次与区间的差距/安全边际、推翻条件。结论不能代替数据证明；结论之后必须继续展示统计、分析和预测方法。

## 一、先统计

输出 `可验证数据统计`，至少覆盖以下数据，并在数值旁提供可点击链接：

1. 当年招生计划：学校本科招生网、招生简章、招生章程、省教育考试院专业目录。
2. 最近完成年度录取结果：最低分、最低位次、投档线、专业线；优先省考试院与学校本科招生网。
3. 一分一段表：同省、同年、同科类/选科、同批次。
4. 同校普通专业基线：同批次、同科类、同专业组或尽量相近专业。
5. 同省同档样本：同层次学校、相近学费、相近培养模式、相近专业热度的合作项目。
6. 项目变化项：招生计划增减、首次招生、证书变化、学费变化、校区变化、选科变化、专业组变化。

每条统计数据必须写清：年份、省份、科类/选科、招生批次、专业组、计划类型、专业名称、招生计划人数、最低分、最低位次、同批次控制线、来源链接、人工确认要点、质量等级。没有可打开来源的数字不能进入公式。

### 批次口径先行

先统计阶段必须先分批次。同一项目在提前批、本科批、综合评价批、专项计划、中外合作单列批或不同院校专业组中出现时，视为不同数据记录。不得把不同批次的招生人数相加作为同一项目计划数，不得把不同批次的最低分/最低位次取最低或平均作为项目门槛。批次未核到的数字只能作为弱参照。


## 二、再分析

在 `位差与线差分析` 中，先处理口径，再处理关系：

```text
绝对位差 = 合作项目位次 - 普通基线位次
相对位差 = 绝对位差 / 普通基线位次 × 100%
线差 = 录取分 - 同招生批次控制线
相对线差 = 线差 / (满分 - 批次控制线) × 100%
```

分析必须说明异常值：招生批次混淆、专业组变动、位次单位误读、只找到分数没有位次、计划数大幅变化、冷门/热门专业替换、首年信息不对称、官方页面与第三方页面冲突。

## 三、再预测或预估

预测只能在统计和分析之后出现，且输出区间而不是单点。

### 有历史线项目

```text
趋势参照位次 = 最近完成年度项目位次 × (1 + 年均变化率)
调整后参考区间 = 趋势参照位次 ± 招生计划/专业热度/费用/证书/校区/样本波动调整
```

历史不足三年时必须降级置信度，并引入同校普通基线和同省同档样本。

### 新项目、首年招生或本省无历史线项目

```text
中性情景参考位次 = B普通招生基线位次 + A同省同档样本中位位次差
门槛偏高情景参考位次 = B普通招生基线位次 + A样本较小位次差
门槛偏宽情景参考位次 = B普通招生基线位次 + A样本较大位次差
```

若没有普通基线，使用同省同分段学校、同类机构、同费用段路径和学校自身近年分省线作为替代，并明确“无普通基线”。

## 四、重要声明和人工确认

位次章必须包含：

- `位次预估结论先行`：参考区间、冲/稳/保、置信度、推翻条件；

```text
重要声明：CoopLens Skill 产出的内容由 AI 生成，真实性需要使用者自行核实，不代表任何官方意见，不能作为任何决策依据或参考。
```

`位次预估重要声明` 只能放置上面的固定重要声明，不能另写自定义声明。

人工确认清单必须证明数据仍需人工确认，而不是宣称已经全部核实。

## 五、计算过程显示规范

位次预估中的 `计算过程` 不能和最终结论混在同一段。报告应把公式链、情景区间换算和调整项单独放在 `计算过程` 下，优先使用 fenced code block；移动端 HTML 页面必须把这块渲染为黑底红字，提醒家长这是可复核的估算链条，不是官方结论。
