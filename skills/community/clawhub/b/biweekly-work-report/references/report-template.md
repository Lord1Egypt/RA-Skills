# Report Generation Template

This document defines the canonical output format for weekly and biweekly
reports generated by this skill.

## Report Structure

```markdown
# 工作周报 — YYYY.MM.DD ~ YYYY.MM.DD

> 报告周期：YYYY年M月D日（DayOfWeek）— YYYY年M月D日（DayOfWeek）| 共N个工作日

## 一、本周重点工作

- **【Theme 1】** Description (耗时N天)
- **【Theme 2】** Description (耗时N天)
- ...

## 二、详细工作记录

### YYYY-MM-DD（DayOfWeek）
- [HH:MM] 【Category】Description
- ...

### YYYY-MM-DD（DayOfWeek）
*无工作记录*

## 三、下周计划

- [ ] Actionable item 1
- [ ] Actionable item 2
- ...

## 四、问题与风险

- Issue 1
- Issue 2

---
*本报告由 AI 周报助手自动生成 | 数据来源: ~/.workbuddy/work-logs/*
*生成时间: YYYY-MM-DD HH:MM*
```

## Section Guidelines

### 一、本周重点工作

**Purpose:** Give a high-level summary for managers/peers.

**How to write:**
- Group entries by **business topic**, not category tag.
- Merge related entries across days into 2-5 themes.
- Each theme: bold keyword + one-sentence summary + time spent (days).
- Order by importance (most impactful first).
- Skip themes with only one minor entry.

### 二、详细工作记录

**Purpose:** Provide a complete, verifiable record.

**How to write:**
- List days chronologically (Monday → Friday).
- Copy entries verbatim from the daily log files.
- Days with no entries: write `*无工作记录*`.
- Days with no file: write `*无记录*`.

### 三、下周计划

**Purpose:** Show forward momentum.

**How to write:**
- Infer 2-4 items from the current work trajectory.
- Use `- [ ]` checklist format.
- If unclear, add `- [ ] 待补充 — 请根据实际安排填写` and mention to user.

### 四、问题与风险

**Purpose:** Surface blockers and risks.

**How to write:**
- Extract from entries tagged `【修bug】`.
- Flag unresolved issues mentioned in descriptions.
- For resolved issues: note as `(已修复)`.
- If nothing to report: write `暂无显著风险`.

## Date Range Rules

| User Input | Start | End |
|-----------|-------|-----|
| 本周周报 / 周报 | This Monday | Today |
| 上周周报 | Last Monday | Last Friday |
| 双周报 | 14 days ago | Today |
| 双周报 (alt) | Monday 2 weeks ago | Last Friday |
| Custom dates | As specified | As specified |

## File Naming

```
工作周报_YYYY-MM-DD~YYYY-MM-DD.md
```

Write to the current workspace directory, or a path the user specifies.
