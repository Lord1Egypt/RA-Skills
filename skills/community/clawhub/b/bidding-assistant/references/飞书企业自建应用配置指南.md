# 飞书企业自建应用配置指南

## 概述

本指南详细介绍如何配置飞书企业自建应用，用于发送PDF报告到飞书群聊。

---

## 配置步骤

### 第一步：创建飞书企业自建应用

1. **访问飞书开放平台**
   - 打开浏览器，访问：https://open.feishu.cn/

2. **登录飞书账号**
   - 使用飞书管理员账号登录

3. **创建应用**
   - 点击"创建企业自建应用"
   - 选择"从零创建"
   - 填写应用信息：
     - 应用名称：招投标信息推送
     - 应用描述：自动推送招投标信息日报
     - 应用图标：上传应用图标（可选）
   - 点击"创建"

4. **获取应用凭证**
   - 创建成功后，进入应用详情页
   - 在左侧菜单中，点击"凭证与基础信息"
   - 复制以下信息：
     - App ID
     - App Secret

---

### 第二步：配置应用权限

1. **添加权限**
   - 在左侧菜单中，点击"权限管理"
   - 点击"添加权限"
   - 搜索并添加以下权限：
     - `im:chat`：获取群聊信息
     - `im:message`：发送消息
     - `im:message:send_at_all`：发送@所有人消息（可选）
     - `drive:drive`：上传文件到云空间

2. **申请权限**
   - 添加权限后，点击"申请权限"
   - 填写申请理由：
     - 应用用途：推送招投标信息日报
     - 使用场景：每日自动推送PDF报告到指定群聊
   - 提交申请，等待管理员审批

---

### 第三步：添加机器人到群聊

1. **获取机器人配置**
   - 在左侧菜单中，点击"机器人"
   - 记录机器人信息：
     - 机器人名称
     - 机器人App ID

2. **将机器人添加到群聊**
   - 打开目标群聊
   - 点击群设置 → 群成员 → 添加成员
   - 搜索并添加机器人（招投标信息推送）
   - 设置机器人权限：
     - 允许机器人发送消息
     - 允许机器人上传文件

3. **获取群聊ID**
   - 在群聊中，点击右上角"..."
   - 点击"复制群ID"或"查看群信息"
   - 记录 `open_chat_id`

---

## API接口说明

### 1. 获取 tenant_access_token

**接口地址**：`POST https://open.feishu.cn/open-apis/auth/v3/tenant_access_token/internal`

**请求参数**：
```json
{
  "app_id": "cli_xxxxxxxxxxxxxxxx",
  "app_secret": "xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
}
```

**返回参数**：
```json
{
  "code": 0,
  "msg": "success",
  "tenant_access_token": "t-xxxxxxxxxxxxxxxxxxxxxxxx",
  "expire": 7200
}
```

---

### 2. 上传文件

**接口地址**：`POST https://open.feishu.cn/open-apis/drive/v1/medias/upload_all`

**请求头**：
```
Authorization: Bearer {tenant_access_token}
```

**请求参数**（multipart/form-data）：
- `file_name`: 文件名
- `parent_type`: 父节点类型（默认：explorer）
- `size`: 文件大小
- `file_type`: 文件类型
- `file`: 文件内容

**返回参数**：
```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "file_key": "file_v2_XXXXXXXXXXXXXXXXXXXXXXXX"
  }
}
```

---

### 3. 发送文件消息

**接口地址**：`POST https://open.feishu.cn/open-apis/im/v1/messages`

**请求头**：
```
Authorization: Bearer {tenant_access_token}
Content-Type: application/json; charset=utf-8
```

**请求参数**：
```json
{
  "receive_id_type": "chat_id",
  "receive_id": "oc_xxxxxxxxxxxxxxxx",
  "msg_type": "file",
  "content": "{\"file_key\":\"file_v2_XXXXXXXXXXXXXXXXXXXXXXXX\"}"
}
```

**返回参数**：
```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "msg_id": "om_xxxxxxxxxxxxxxxx"
  }
}
```

## 使用示例

```python

from datetime import datetime
```

## 常见问题

### Q1: 如何获取群聊ID（open_chat_id）？

**A**: 有以下几种方式：

**方式1：通过群聊设置获取**
1. 打开群聊
2. 点击右上角"..."
3. 点击"复制群ID"

**方式2：通过API获取**
```python
from scripts.feishu_enterprise_app import FeishuEnterpriseApp

app = FeishuEnterpriseApp()
chat_info = app.get_chat_info('oc_xxxxxxxxxxxxxxxx')
print(f"群聊名称: {chat_info['name']}")
```

**方式3：通过开发者工具获取**
1. 打开飞书网页版
2. 按F12打开开发者工具
3. 切换到Network标签
4. 在群聊中发送一条消息
5. 查看请求的URL，找到chat_id参数

---

### Q2: tenant_access_token 过期了怎么办？

**A**: tenant_access_token 的有效期为2小时（7200秒），客户端需要定期刷新。

代码中已自动处理：
```python
# 每次调用API时都会检查token
if not self.tenant_access_token:
    self.get_tenant_access_token()
```

如果手动处理：
```python
import time
from datetime import datetime

# 记录token获取时间
token_time = datetime.now()

# 检查token是否过期（2小时）
if (datetime.now() - token_time).seconds > 7200:
    app.get_tenant_access_token()
```

### Q3: 权限申请被拒绝怎么办？

**A**:

1. **检查申请理由**
   - 是否清楚说明了应用用途
   - 是否说明了使用场景
   - 是否说明了数据来源

2. **联系管理员**
   - 请企业飞书管理员审批权限申请
   - 提供详细的申请说明和使用场景

3. **使用最小权限原则**
   - 只申请必要的权限
   - 避免申请敏感权限

---

### Q4: 文件上传失败怎么办？

**A**: 检查以下几点：

1. **文件大小限制**
   - 单个文件大小不能超过100MB
   - 如果文件过大，考虑压缩或拆分

2. **文件类型限制**
   - 支持的文件类型：PDF、DOC、DOCX、XLS、XLSX、PPT、PPTX、PNG、JPG、JPEG、GIF、TXT
   - 确保文件类型在支持范围内

3. **网络问题**
   - 检查网络连接是否稳定
   - 尝试重试上传

4. **权限问题**
   - 确保应用有上传文件的权限（`drive:drive`）
   - 检查权限是否已审批通过

---

### Q5: 消息发送失败怎么办？

**A**: 检查以下几点：

1. **群聊ID是否正确**
   - 确认使用的是 `open_chat_id`，不是 `chat_id`
   - 确认机器人已添加到该群聊

2. **机器人权限**
   - 确认机器人有发送消息的权限
   - 确认群主没有禁用机器人

3. **消息格式**
   - 检查消息内容是否符合格式要求
   - 检查JSON格式是否正确

4. **请求频率限制**
   - 飞书API有请求频率限制
   - 避免短时间内发送大量消息

---

### Q6: 如何查看API调用日志？

**A**: 飞书开放平台提供详细的API调用日志：

1. **访问开放平台**
   - 打开 https://open.feishu.cn/

2. **进入应用详情**
   - 点击你的应用

3. **查看日志**
   - 在左侧菜单中，点击"数据概览"
   - 查看API调用次数、成功率、错误率等

## 安全建议

1. **保护凭证**
   - 不要将 App ID 和 App Secret 提交到代码仓库
   - 使用环境变量或配置文件存储敏感信息
   - 定期更换 App Secret

2. **权限控制**
   - 只授予应用必要的权限
   - 限制机器人可以操作的群聊
   - 定期审计应用权限使用情况

3. **错误处理**
   - 添加重试机制
   - 记录详细的错误日志
   - 设置告警阈值

4. **监控与告警**
   - 监控API调用成功率
   - 监控消息发送成功率
   - 异常情况及时告警
