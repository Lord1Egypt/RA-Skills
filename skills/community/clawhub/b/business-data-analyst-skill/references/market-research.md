# 市场调研方法手册

## 适用场景

市场调研适用于以下场景：

- 进入新市场或新赛道前评估机会
- 分析行业规模、增长趋势与竞争格局
- 了解目标客户需求、痛点与决策路径
- 对标竞争对手的产品、价格、渠道与品牌
- 验证或修正业务假设（市场规模、客户意愿、定价空间）
- 为战略分析（SWOT、PESTEL、3C、五力）提供外部事实输入

## 调研流程

### 1. 定义调研问题

回答三个问题：

- **决策问题**：调研结果要支持什么决策？（进入/不进入、定价多少、优先攻哪个客群）
- **关键假设**：隐含了哪些需要验证的假设？（市场够大、客户愿意付这个价、竞品没有壁垒）
- **输出形式**：报告、打分卡、市场地图还是数据表？

### 2. 设计调研策略

根据问题类型选择调研路径：

| 问题类型 | 首选路径 | 辅助路径 |
|---|---|---|
| 市场有多大、增长多快 | 二手数据 + 行业报告 | 专家访谈验证 |
| 客户想要什么、痛点在哪 | 用户访谈 + 问卷 | 社交媒体/评论挖掘 |
| 竞争对手在做什么 | 二手数据 + 产品体验 | 渠道走访 |
| 定价是否合理 | 竞品价格映射 + 问卷 | 价格敏感度测试 |

### 3. 执行信息收集

按以下优先级收集：

1. **公开权威数据**：国家统计局、央行、行业协会、白皮书、上市公司财报
2. **行业研究**：艾瑞、QuestMobile、IDC、Gartner、Forrester、36Kr、亿欧等
3. **竞品公开信息**：官网、产品文档、定价页、用户评价、招聘岗位、融资新闻
4. **用户生成内容**：社交媒体、论坛、评论区、电商评价
5. **一手调研**：问卷、访谈、焦点小组、神秘访客

### 4. 交叉验证与质量评估

每条信息必须回答：

- **来源是否可靠？**（一手 vs 二手、权威 vs 自媒体、有数据支撑 vs 主观判断）
- **数据是否过时？**（行业报告的有效期通常不超过 12 个月）
- **口径是否匹配？**（市场规模的统计范围、用户数的定义是否与我的问题一致）
- **是否存在利益偏向？**（咨询公司可能乐观、竞品可能夸大、媒体可能放大风险）

不一致的信息标注为 `待验证`，不做确定性结论。

### 5. 综合输出

输出结构建议：

1. **调研结论摘要**：3-5 条最关键发现
2. **市场规模与趋势**：TAM、SAM、SOM、增长率、关键趋势
3. **竞争格局**：主要玩家、市场份额、定位差异
4. **客户与需求洞察**：目标客群画像、核心需求、未被满足的痛点
5. **关键假设验证结果**：哪些假设被支持、哪些被推翻、哪些待验证
6. **信息缺口与后续调研建议**

## 搜索工具使用规范

### 何时使用搜索工具

当以下条件之一成立时，应启动外部搜索：

- 用户明确要求调研、查行业、看竞品、看市场
- 当前分析缺少关键外部事实（市场规模、竞品数据、行业基准）
- 涉及战略/竞争判断需要外部输入（SWOT 的机会/威胁、五力的行业结构）
- 需要验证或反驳一个关于市场的假设

### 搜索策略

1. **关键词结构化**：从问题中提取 3-5 组搜索关键词，每组包含：
   - 核心概念（行业/市场/产品名）
   - 限定词（规模、报告、趋势、竞争格局、市场份额）
   - 中英文各一组（国内用中文，国际用英文）

   示例：
   ```
   "中国 SaaS CRM 市场规模 2025"
   "SaaS CRM market size 2025 forecast"
   "CRM 行业竞争格局 市场份额"
   ```

2. **来源优先级**：
   - 一级来源：国家统计局、央行、行业协会、公司财报、招股书
   - 二级来源：知名咨询公司报告（Gartner、IDC、Frost & Sullivan、McKinsey）
   - 三级来源：行业媒体（36Kr、晚点、虎嗅、TechCrunch）、券商研报
   - 四级来源：自媒体、社区讨论、个人博客（标注置信度低）

3. **搜索轮次**：
   - 第一轮：宽泛搜索，了解行业整体概貌
   - 第二轮：定向搜索，针对关键缺口补充信息
   - 第三轮：验证搜索，用不同来源交叉验证核心数字

### 信息筛选规则

- 优先使用含发布时间、数据来源、统计口径的信息
- 单一来源的数据标注为 `单来源`，不做核心判断依据
- 不同来源给出差异超过 30% 的数据，认定为 `口径不一致`，注明分歧
- 无日期、无来源、明显过时（超过 18 个月）的信息不采纳

### 搜索工具纪律

- 每次搜索前明确搜索目标，不要漫无目的浏览
- 同一问题最多搜索 3 轮，每轮 2-3 组关键词
- 搜索结果优先使用权威来源摘要，非必要不打开全文
- 搜索结束后立即整理关键发现，不把搜索结果直接堆入输出
- 若搜索结果与已有假设矛盾，优先记录矛盾，不要选择性忽略

## 市场调研方法

### 1. 市场规模估算

#### 方法一：自上而下法（Top-down）

```
TAM（总可用市场） = 目标人群总数 × 年人均消费/年客单价
SAM（可服务市场） = TAM × 可达比例（渠道覆盖 × 地区覆盖 × 场景匹配）
SOM（可获得市场） = SAM × 预期渗透率（初期 1-3 年）
```

适用：行业成熟、有权威宏观数据时优先使用。

风险：容易偏乐观，建议用不同口径交叉验证。

#### 方法二：自下而上法（Bottom-up）

```
销售额 = 销售团队人数 × 人均产出 × 客户数增长
或
销售额 = 渠道网点数 × 网点平均产出
```

适用：有可参考的运营数据或竞品公开数据时。

优势：更贴近实际，适合早期验证。

#### 方法三：类比法（Analogy）

```
目标市场 = 相似成熟市场 × 人口/经济/渗透率调整系数
```

适用：全新市场缺乏直接数据时，参考相似市场的发展路径。

风险：类比的有效性取决于可比程度，标注关键差异。

### 2. 竞争分析调研

需要收集的信息：

- **产品与定价**：功能对比表、价格分层、免费/付费策略
- **渠道与交付**：销售模式（直销/渠道/自营/平台）、物流、服务网络
- **客户与口碑**：目标客群、客户规模、NPS、评价关键词
- **组织与资源**：团队规模、融资历史、股东背景、招聘方向
- **市场表现**：市场份额趋势、增长率、活跃用户、GMV（如公开）

信息获取渠道：

- 竞品官网、产品文档、Help Center
- 第三方评测平台（G2、Capterra、Trustpilot、大众点评端）
- 招聘网站（JD 看方向、规模、薪资）
- 融资新闻、财报（上市或披露公司）
- 用户社区、社交媒体、投诉平台

### 3. 客户与需求调研

#### 用户画像调研

至少覆盖：

- 人口属性：年龄、地域、职业、收入
- 行为特征：使用频率、使用场景、决策路径
- 需求与痛点：核心需求、未被满足的需求、切换成本
- 信息渠道：通过什么渠道了解产品、信任什么来源

#### 需求验证方法

- **问题访谈**：了解用户在什么场景下遇到什么问题，现在怎么解决
- **解决方案测试**：用户是否愿意为方案付钱/花时间
- **价格敏感度测试**：Van Westendorp 或 Gabor-Granger 法

### 4. 行业与趋势调研

重点覆盖：

- **政策环境**：监管态度、行业标准、补贴/限制政策
- **技术驱动**：关键技术成熟度、渗透率、对成本/效率的影响
- **社会变化**：消费习惯、人口结构、工作方式的变化
- **资本流向**：赛道融资总额、头部项目、估值变化

来源举例（中国）：

- 国家统计局、工信部、商务部
- 36Kr、晚点、虎嗅、甲子光年、亿欧
- 上市公司财报、招股书
- 券商行业研报（东方财富、Wind）
- 咨询公司报告（McKinsey、BCG、Bain、Deloitte、PwC、KPMG）

来源举例（国际）：

- World Bank、IMF、OECD、UN
- Gartner、IDC、Forrester、Frost & Sullivan
- Crunchbase、PitchBook、CB Insights
- SEC filings (10-K, 10-Q)、annual reports

## 输出质量检查清单

输出前逐项检查：

- [ ] 调研问题是否明确对应一个决策场景
- [ ] 每条关键发现都有来源标注（来源 + 时间）
- [ ] 未验证的关键假设已标注 `待验证` 或 `假设`
- [ ] 分歧数据已标注不同口径和出处
- [ ] 市场规模的统计口径已说明（TAM/SAM/SOM、时间段、包含范围）
- [ ] 竞品信息标注了收集时间和可靠程度
- [ ] 没有把搜索结果直接堆砌到输出中，而是做了综合提炼
- [ ] 调研结论直接回答了起始的决策问题

## 与现有分析框架的关系

市场调研为其他分析框架提供外部输入：

- `SWOT`：调研结果为机会和威胁提供事实依据
- `PESTEL`：调研覆盖政策、经济、社会、技术维度
- `3C`：调研覆盖客户和竞争对手维度
- `五力`：调研覆盖行业结构、替代品、新进入者
- `STP`：调研覆盖市场细分、客户画像和定位依据
- `4P`：调研覆盖竞品定价、渠道和促销策略

市场调研本身不替代上述框架，而是在框架需要外部数据时提供输入。调研结果进入分析流程后，仍需遵守 `SKILL.md` 的分析纪律。
