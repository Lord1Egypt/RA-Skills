# interview-synthesis

Synthesizes qualitative user research data—interviews, usability tests, focus groups, or survey open-ends—into a structured report with named themes, supporting quotes, insights, and prioritized recommendations.

## When to Use

- You have raw interview transcripts or session notes and need to find patterns
- You need a stakeholder-ready synthesis report quickly
- You want to ensure no strong observation is missed in the analysis

## What It Does

1. Routes analysis by research method (user interviews, usability tests, focus groups, surveys)
2. Extracts atomic observations from the raw data
3. Clusters observations into named themes with prevalence counts
4. Selects the best supporting verbatim quote for each theme
5. Writes insights (what it means) and prioritized recommendations (what to do)
6. Produces a stakeholder-ready synthesis report

## Input

- Raw transcripts, session notes, or open-ended survey responses (pasted as text)
- Research question or goal
- Number of participants
- Research method type

## Output

A structured Markdown report with named themes, prevalence counts, verbatim quotes, insights, and recommendations rated by priority (🔴 Critical / 🟡 Important / 🟢 Consider), plus a top-insights summary and suggested next steps.

## Privacy

All participant names and identifiers are anonymized to P1, P2, etc. in the output. Sensitive data is never reproduced verbatim in the final report.
