## Description: <br>
Helps coding agents apply branch discipline, structured commit messages, and safety checks around commits, pushes, merges, rebases, and destructive Git operations. <br>

This skill is ready for commercial/non-commercial use. <br>

## Publisher: <br>
[yiiknow](https://clawhub.ai/user/yiiknow) <br>

### License/Terms of Use: <br>
MIT-0 <br>


## Use Case: <br>
Developers and coding agents use this skill while working in Git repositories to decide when to create branches, commit completed work, structure commit messages, and pause for safety checks before risky Git operations. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: The skill shapes real Git workflow decisions, including branch, stash, rebase, reset, and push behavior in shared repositories. <br>
Mitigation: Review proposed Git actions before approval, especially destructive operations or changes that affect shared branch history. <br>
