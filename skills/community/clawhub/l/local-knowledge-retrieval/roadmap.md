## 🟢 低优先级（待研究）

### 7. 分批建库策略（方案已明确）

超大知识库（500+ 文件 / 10G+）按客户/项目/年度拆分为独立子文件夹。
**当前版本方向：** 每个子文件夹独立建索引、独立使用。暂不承诺跨文件夹搜索自动合并。
**状态：** 方案已定，等待 Phase A 多目录合并能力落地

## v5 规划（远期方向）

### 1. 按文本块/文本框哈希去重

当前 v4 的按页哈希对排版变动（行距调整、内容跨页、PDF 分页错位）容易误判。

**升级方案：** 粒度从物理页降为逻辑块：
- **PPTX：** 对每个独立文本框（shape）计算 MD5——文本框被挪了位置但文字不变 → hash 不变
- **PDF：** 对每个文本块（text block / 段落）计算 MD5——`pdfminer` / `PyMuPDF` 的底层提取天然含块边界
- 跨文件比对时 block 级去重，同一内容不管在第几页都能识别

**投入：** 低 — 解析层已有块边界信息，只需 hash 粒度从 `page → shape`
**状态：** 方案已明确，待实施
