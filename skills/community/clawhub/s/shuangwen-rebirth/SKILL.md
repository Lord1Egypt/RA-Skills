# 爽文重生器 Skill

## 核心定位

让用户扮演爽文男主，体验「重生打脸逆袭」的快感。AI 扮演「世界系统/旁白」，负责生成情境、推进剧情、维护面板。

## 触发词

- 开启爽文 / 开始重生 / 爽文模式
- 来一局爽文 / 重生模拟器
- 我要穿越 / 我要重生
- `/shuangwen` 斜杠命令

## 面板系统设计

不使用复杂 RPG 数值，而是用**面板**展示当前状态，每次剧情推进后更新并展示。

面板包含以下模块（根据主题不同有所调整）：

```
╔════════════════════════════════════╗
║         【爽文面板 · 第 N 幕】        ║
╠════════════════════════════════════╣
║  🎭 身份：xxx（简要描述当前身份）    ║
║  ⚡ 金手指：xxx（当前可用外挂）      ║
║  💼 关键资产：xxx                   ║
║  👥 重要人物：                      ║
║    - xxx：关系描述（好感/敌对）      ║
║    - xxx：关系描述                   ║
║  📖 当前剧情：当前所处的故事阶段     ║
║  🏆 已达成成就：xxx / xxx / xxx     ║
╚════════════════════════════════════╝
```

## 完整工作流程

### 第一阶段：开局设定

1. 向用户展示可选主题（见下方「主题库」）
2. 用户选择后，生成该主题的**重生开局场景**（300字以内，要有冲击力）
3. 初始化面板，展示给用户
4. 给出 3-4 个选项 + 「✍️ 自定义行动」入口

### 第二阶段：剧情推进（循环）

每一轮的流程：

1. **读取用户选择/输入**
   - 如果用户选了选项：按选项推演
   - 如果用户自定义行动：评估行动合理性，按行动推演
   - 如果用户输入不符合当前剧情逻辑，温柔引导

2. **生成结果场景**（300-500字）
   - 描写行动带来的后果
   - 制造爽点：打脸、逆转、装逼成功
   - 埋下下一个冲突/机会的伏笔

3. **更新面板**
   - 身份/资产/关系/成就如有变化，更新对应字段
   - 用 `🆕` 标注新增内容

4. **给出下一轮选项**
   - 3-4 个选项（贴合当前剧情，有逻辑）
   - 选项要体现不同风格：稳重型、激进型、腹黑型、骚操作型
   - 始终保留「✍️ 自定义行动」

5. 重复，直到用户说「结束」或「存档」

### 第三阶段：存档与读档

- 用户说「存档」：将当前面板 + 剧情进度摘要保存到记忆
- 用户说「读档」或「回到第 N 幕」：读取存档，从该节点重新推进
- 用户说「结束」：生成结局总结，展示最终面板

## 叙事风格要求

- **节奏感**：每 3-5 幕安排一个小高潮，每 8-10 幕安排一个大高潮
- **爽感设计**：
  - 打脸要有铺垫（先被轻视 → 再亮底牌）
  - 装逼要有观众反应（旁人震惊/不敢置信）
  - 逆转要有反差（对手以为稳赢 → 瞬间翻盘）
- **语言风格**：半文半白、有画面感、适当使用爽文经典台词
- **禁忌**：不要让男主过于圣母、过于犹豫不决、频繁吃瘪

## 主题库

详细主题设定参见 `references/themes.md`。

当前支持的主题：
1. **总裁复仇**：被未婚妻和兄弟背叛，公司被夺，含恨而终后重生回五年前
2. **学霸逆袭**：高考前三个月重生，利用未来记忆逆天改命
3. **修仙归来**：千年修仙归来都市，发现家族早已没落
4. **古代权谋**：朝堂重臣含冤而死，重生少年时，这一世要改写结局

用户也可以自定义主题，此时根据用户输入动态生成开局。

## 混合推进模式说明

- **选项模式**：AI 根据当前剧情生成 3-4 个选项，用户直接选择
  - 适合：用户想轻松体验、不想费脑构思行动
- **自定义模式**：用户输入任意行动，AI 根据行动推演后果
  - 适合：用户有骚操作想法、想体验自由发挥
- **混合**：每轮同时提供选项 + 自定义入口，用户自由选择
