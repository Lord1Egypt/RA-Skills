# AI 评论生成 v2.0 — Claude 直接生成

## 一、生成策略

Claude 直接根据笔记内容和评论区风格生成评论。不调用外部模型。

### 1.1 生成 Prompt 模板

```
你正在小红书浏览一篇关于 {笔记主题} 的帖子。
帖子标题：{标题}
帖子核心内容：{内容摘要，50-100字}
评论区风格：{观察到的评论风格描述}

请生成一条自然的中文评论，要求：
1. 15-40字之间
2. 口语化，像真人随手打的
3. 必须引用笔记中的一个具体细节（数字/品种名/方法名等）
4. 语气从以下随机选择：{随机风格}
5. emoji 最多1个，可以没有
6. 不要用"学到了""受教了""干货"等AI常见评论词
7. 不要用感叹号开头
```

### 1.2 风格变体

在生成前用 `javascript_tool` 随机选择风格：
```
javascript_tool("['casual','professional','curious','humorous'][Math.floor(Math.random()*4)]")
```

| 风格 | 特征 | 示例方向 |
|------|------|---------|
| casual | 口语化，简短 | "这个xx我也在研究" |
| professional | 有见地，提出观点 | "xx方面确实，不过yy也值得考虑" |
| curious | 提问式 | "xx的话，zz情况怎么处理？" |
| humorous | 轻松调侃 | "看完突然觉得自己xx" |

## 二、安全过滤

### 2.1 屏蔽模式（生成后自检）

生成评论后，Claude 必须自检是否包含以下内容：

**硬性屏蔽**（命中则必须重新生成）：
- 任何电话号码格式（11位数字、区号+号码）
- 微信号/QQ号（"加我""私我""vx""wx"）
- 任何 URL 或网址
- 明显的营销语（"免费领""限时优惠""加入我们"）
- 投资建议性用语（"保证收益""稳赚""推荐买入"）
- AI 特征词（"作为AI""我认为""首先...其次...最后"）

**软性屏蔽**（命中则建议重新生成）：
- 过于正式的书面语
- 过长的评论（>60字）
- 明显与笔记内容无关
- 与最近的评论过于相似

### 2.2 多样性检查

在写入评论前，检查近期评论历史：
```
Bash: grep '"action":"comment"' data/nurture-log/$(date +%Y-%m-%d).jsonl 2>/dev/null | tail -10
```

对比当前评论与最近10条评论，如果主题/结构高度相似 → 重新生成。

Claude 可以直接判断两条评论是否相似，不需要计算 Jaccard 系数。

## 三、降级模板库

当 AI 生成连续 3 次未通过安全检查时，从模板库随机选取。

### 3.1 通用正面模板

```yaml
templates:
  generic_positive:
    - "写得清楚，正好在学这块"
    - "终于有人把这个讲明白了"
    - "实操性很强，先收藏了"
    - "这个思路可以借鉴一下"
    - "跟我最近在看的方向一致"

  agreement:
    - "确实是这样，很多人忽略这点"
    - "深有同感，之前踩过坑"
    - "终于有人说了大实话"

  curious:
    - "这个后续有更新吗？想继续跟"
    - "请问这个适合新手上手吗？"
    - "想了解更多细节"
```

### 3.2 期货/财经专属模板

```yaml
  trading_specific:
    - "风控做得好才是真本事"
    - "仓位管理确实是关键"
    - "轻仓试错的思路不错"
    - "基本面和技术面结合看更全面"
    - "止损纪律比预测准确率更重要"
    - "做过才知道心态有多重要"
    - "这个品种波动确实大"
```

### 3.3 模板使用规则

```
1. 用 javascript_tool 随机选择模板类别和条目
2. 如果模板包含 {占位符}，替换为笔记中的实际内容
3. 同一会话内不重复使用同一模板
4. 模板评论同样需要通过安全自检
```

## 四、评论执行流程

### 4.1 完整流程

```
1. 判断是否应该评论：
   - 评论配额未满
   - javascript_tool("Math.random()") < 0.10
   - 笔记赞数 > 500（热门帖评论区活跃）
   - 应在浏览+点赞之后再评论（不要一进笔记就评论）

2. 浏览评论区（必须）：
   - scroll down 到评论区
   - wait 随机 3-8 秒，读几条热评
   - 观察评论区的语气和长度

3. 生成评论：
   - 使用 1.1 的 Prompt 模板
   - 将观察到的评论区风格融入生成

4. 安全自检（2.1）

5. 多样性检查（2.2）

6. 输入评论：
   - find("说点什么") 或 find("评论输入框")
   - left_click 输入框
   - wait 随机 1-2 秒
   - computer(action="type", text=评论全文)

7. 发送前确认（MANDATORY）：
   - wait 随机 1-4 秒（模拟审视自己写的评论）

8. 发送：
   - find("发送") 或 find("发布")
   - left_click 发送按钮
   - wait 随机 2-3 秒

9. 验证发送成功：
   - 确认"评论成功"提示 或 评论数+1

10. 日志记录：
    - 追加到 JSONL，包含评论内容
```

### 4.2 评论发送取消

如果发送前发现问题：
```
1. 找到"取消"按钮并点击
2. 或者：全选(Cmd+A) → 删除(Backspace) → 点击输入框外部
3. 记录跳过原因到日志
```
