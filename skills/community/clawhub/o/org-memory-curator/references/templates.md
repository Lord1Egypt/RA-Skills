# 模板

## 1. 结构化归档判断输出模板

当使用本 skill 做归档判断时，默认输出应尽量接近以下结构：

```yaml
archive_decision: <archive|do_not_archive>
reason: <为什么归档或为什么不归档>
project_detected: <yes|no>
project_id: <若识别到则填写，否则为 none>
target_files:
  - <目标文件路径 1>
  - <目标文件路径 2>
write_mode: <append|create|no_write>
content_summary: <提炼后的归档摘要>
promotion_level: <project|lesson|engineering-standard|none>
```

若信息不足, 必须明确输出:
信息不足，暂不归档；请补充 project-id 或项目名称。

## 11. 使用模板时的注意事项

- 模板是约束结构，不是强制逐字照抄

- 能简洁表达就不要写冗余文字

- 原始聊天不能直接原样进入长期记忆

- 归档时优先保留结论、边界、风险、里程碑、经验

- 不要把暂时猜测写成正式事实
