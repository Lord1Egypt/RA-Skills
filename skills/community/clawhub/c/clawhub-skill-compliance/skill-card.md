## Description: <br>
Pre-flight checklist for ClawHub skill publishing, focused on metadata completeness, dependency transparency, and security scope documentation. <br>

This skill is ready for commercial/non-commercial use. <br>

## Publisher: <br>
[tchen6500](https://clawhub.ai/user/tchen6500) <br>

### License/Terms of Use: <br>
MIT-0 <br>


## Use Case: <br>
Skill authors use this checklist before publishing or republishing ClawHub skills to verify metadata, dependency declarations, environment variable handling, security scope, and clear package guidance. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Examples or placeholders could accidentally include real secrets when authors adapt the checklist. <br>
Mitigation: Use annotated placeholders such as <your-api-key> and keep real secrets out of skill examples. <br>
Risk: Future versions could add executable behavior, downloads, install commands, or tool use outside the current documentation-only scope. <br>
Mitigation: Re-review and scan future versions before deployment if scripts, tools, downloads, or install commands are added. <br>
Risk: Guidance could be misused to make malicious skills appear compliant. <br>
Mitigation: Use the checklist only for legitimate publishing preparation and keep security scope and dependency information visible for reviewers. <br>
