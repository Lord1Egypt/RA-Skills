## SOP概述

### 定位

本SOP是以**听心格爱五步法**为框架，以**阳明心学**为内核，以**十大核心轴**为诊断维度，以**6大解题模型**为解决方案的专业夫妻关系咨询解题系统。

### 适用范围

- 夫妻关系卡点诊断
- 夫妻关系优化咨询
- 夫妻关系危机干预
- 夫妻关系进化指导

### 核心原则

| 原则 | 说明 |
|-----|-----|
| **阳明心学** | 心即理、致良知、知行合一 |
| **东方心理学五原则** | 致良知、尊重接纳、内发自觉、和合共生、知行合一 |
| **身心共治** | 身体、情绪、关系三位一体 |

## 第一步：问心（标准操作）

### 目标

1. 建立信任连接
2. 澄清主诉问题
3. 明确咨询目标
4. 激活动机意愿

### 核心操作

#### 1.1 接纳情绪（5分钟）

**操作要点**：
- 表达理解和接纳
- 不评判、不建议
- 创造安全空间

**话术模板**：
```
"谢谢你愿意分享这些。"
"我能感受到这对你很重要。"
"谢谢你信任我，愿意说出这些。"
```

**观察要点**：
- 用户的情绪状态（焦虑/愤怒/悲伤/迷茫）
- 用户的表达模式（流畅/断续/回避）
- 用户的肢体语言（如果可见）

#### 1.2 具象化问题（10分钟）

**操作要点**：
- 从抽象描述到具体例子
- 了解问题的时间线
- 了解已尝试的方法

**核心问题**：
```
"能给我一个具体的例子吗？"
"这个问题是从什么时候开始的？"
"它对你的生活造成了什么影响？"
"你已经尝试过什么方法？效果如何？"
```

**观察要点**：
- 问题的具体表现
- 问题的严重程度
- 问题的持续时间
- 之前尝试的效果

#### 1.3 明确目标（10分钟）

**操作要点**：
- 从问题聚焦到愿景
- 将模糊愿望转化为具体目标
- 确认双方对目标的共识

**核心问题**：
```
"如果这个问题得到解决，你希望看到怎样的自己？"
"你的关系/生活会有什么不同？"
"对你来说，真正重要的是什么？"
"所以你的目标是...（总结），对吗？"
```

**观察要点**：
- 用户真正想要的是什么
- 目标的清晰度和可衡量性
- 用户的动力强度

### 输出物

| 输出项 | 内容 |
|-------|------|
| 主诉问题 | 具体描述的问题 |
| 影响范围 | 问题对生活的影响 |
| 时间线 | 问题开始和持续时间 |
| 已有尝试 | 之前尝试的方法和效果 |
| 咨询目标 | 清晰、具体、可衡量的目标 |
| 能量状态 | 用户当前的情绪能量状态 |

### 检查清单

- [ ] 是否建立了信任连接
- [ ] 是否澄清了具体问题
- [ ] 是否了解了时间线
- [ ] 是否了解了已有尝试
- [ ] 是否明确了咨询目标
- [ ] 用户是否有改变的动机

## 第二步：听心（标准操作）

### 目标

1. 深度聆听内在声音
2. 识别情绪和需求
3. 洞察心格特质
4. 发现深层动机

### 核心操作

#### 2.1 情绪识别（10分钟）

**操作要点**：
- 识别表层情绪下的深层情绪
- 区分情绪和需求
- 找到情绪背后的需求

**核心问题**：
```
"在这个情况下，你最强烈的感受是什么？"
"这个情绪背后，你真正需要的是什么？"
"如果你的需求得到满足，你会有什么感觉？"
```

**情绪识别工具**：
| 表层情绪 | 深层情绪 | 对应需求 |
|---------|---------|---------|
| 愤怒 | 恐惧/受伤 | 安全/被理解 |
| 冷漠 | 失望/无力 | 被重视/被需要 |
| 指责 | 受伤/渴望 | 被理解/被接纳 |
| 抱怨 | 渴望/无力 | 被关注/被支持 |

#### 2.2 心格洞察（15分钟）

**操作要点**：
- 识别用户的心格类型
- 理解心格优势与挑战
- 关联心格与关系模式

**心格洞察问题**：
```
"在关系中，你通常扮演什么角色？"
"当你感到受伤时，你通常怎么反应？"
"什么会让你在关系中感到特别不安？"
"你倾向于主动沟通还是等待对方？"
```

**心格类型参考**（详见heart-grid-theory.md）：
- 立春型、雨水型、惊蛰型...
- 清明型、谷雨型、立夏型...
- 小满型、芒种型、夏至型...
- 小暑型、大暑型、立秋型...
- 处暑型、白露型、秋分型...
- 寒露型、霜降型、立冬型...
- 小雪型、大雪型、冬至型...
- 小寒型、大寒型

#### 2.3 需求发现（10分钟）

**操作要点**：
- 识别表层需求和深层需求
- 区分自己未满足的需求
- 理解需求与行为的关系

**核心问题**：
```
"在这段关系中，你最渴望得到什么？"
"有什么是你一直想要但没有说出口的？"
"如果可以改变伴侣的一件事，你会选择什么？"
```

### 输出物

| 输出项 | 内容 |
|-------|------|
| 表层情绪 | 当前直接表达的情绪 |
| 深层情绪 | 情绪背后的真实感受 |
| 核心需求 | 未被满足的深层需求 |
| 心格类型 | 用户的心格特质 |
| 心格优势 | 在关系中的天然优势 |
| 心格挑战 | 需要注意的发展方向 |

### 检查清单

- [ ] 是否识别了表层和深层情绪
- [ ] 是否理解了情绪背后的需求
- [ ] 是否洞察了用户的心格类型
- [ ] 是否发现了核心需求
- [ ] 是否理解了需求与关系模式的关系

## 第三步：见心（标准操作）

### 目标

1. 穿透表象看见本质
2. 识别核心信念和模式
3. 追溯生成因素
4. 连接过去与现在

### 核心操作

#### 3.1 信念识别（15分钟）

**操作要点**：
- 识别关于自我的核心信念
- 识别关于关系的基本假设
- 识别限制性信念

**信念探索问题**：
```
"你认为好的关系应该是什么样的？"
"你觉得在关系中，什么是'应该'的？"
"你认为男人/女人在关系中应该怎么表现？"
"你觉得付出和回报应该是什么关系？"
```

**常见限制性信念**：
| 信念类型 | 具体表现 |
|---------|---------|
| 关于自己 | "我不值得被爱"、"我不够好" |
| 关于伴侣 | "他/她应该知道我在想什么" |
| 关于关系 | "真正的爱不应该有争吵" |
| 关于付出 | "我付出足够多，他/她就应该..." |

#### 3.2 模式发现（15分钟）

**操作要点**：
- 识别重复出现的行为模式
- 识别关系中的互动循环
- 理解模式的形成原因

**模式探索问题**：
```
"这种问题在你们之间是第一次发生吗？"
"在之前的关系中，有没有类似的情况？"
"你和父母的相处模式，和现在的关系有什么相似？"
"每次冲突后，通常是怎么结束的？"
```

**常见关系模式**（详见blockage-taxonomy.md）：
| 模式类型 | 具体表现 |
|---------|---------|
| 追逃模式 | 一方追，一方逃 |
| 攻击-防御 | 一方攻击，一方防御 |
| 冷漠循环 | 双方都变得冷漠 |
| 控制-顺从 | 一方控制，一方顺从 |

#### 3.3 溯源连接（10分钟）

**操作要点**：
- 追溯模式的原生家庭根源
- 理解依恋风格的形成
- 连接过去经历与现在关系

**溯源问题**：
```
"你的父母是怎么处理冲突的？"
"你小时候在家庭中是如何表达需求的？"
"你最早的记忆中，亲密关系是什么样的？"
"这些经历，对你现在有什么影响？"
```

**代际传递分析**（详见intergenerational-transmission.md）：
- 模式传递
- 信念传递
- 依恋传递
- 创伤传递

### 输出物

| 输出项 | 内容 |
|-------|------|
| 核心信念 | 关于自我、关系、伴侣的信念 |
| 限制性信念 | 需要突破的信念 |
| 行为模式 | 重复出现的互动模式 |
| 模式根源 | 模式的形成原因 |
| 代际影响 | 原生家庭的影响 |
| 链路追溯 | 从表层到深层的因素链 |

### 检查清单

- [ ] 是否识别了核心信念
- [ ] 是否发现了限制性信念
- [ ] 是否识别了关系模式
- [ ] 是否追溯了模式根源
- [ ] 是否连接了过去与现在
- [ ] 是否理解了代际影响

## 第四步：格爱（标准操作）

### 目标

1. 理解爱的结构和本质
2. 分析爱的供需关系
3. 匹配双方的爱之语
4. 重塑爱的蓝图

### 核心操作

#### 4.1 愛字解构（15分钟）

**操作要点**：
- 从"愛"字四维度分析
- 理解冲突应对（爫）
- 理解安全感给予（冖）
- 理解情感沟通（心）
- 理解认可支持（夂）

**愛字分析问题**：
```
"在冲突中，你通常怎么反应？（爫）"
"当伴侣需要安全感时，你会怎么做？（冖）"
"你是怎么表达情感的？（心）"
"你是怎么给予伴侣认可和支持的？（夂）"
```

**愛字四维度**（详见love-character-analysis.md）：

| 维度 | 含义 | 正面表现 | 负面表现 |
|-----|------|---------|---------|
| 爫（冲突） | 冲突应对方式 | 建设性面对 | 逃避/攻击 |
| 冖（安全） | 安全感的给予与获得 | 保护/被保护 | 控制/依赖 |
| 心（情感） | 情感表达与沟通 | 亲密/共鸣 | 疏离/冷漠 |
| 夂（认可） | 认可与支持 | 欣赏/鼓励 | 忽视/贬低 |

#### 4.2 供需分析（15分钟）

**操作要点**：
- 识别自己的爱之语
- 识别伴侣的爱之语
- 分析供需错位
- 寻找供需平衡

**爱之语分析**（详见love-character-analysis.md）：
| 爱之语 | 表现 | 需求 |
|-------|------|-----|
| 肯定的言语 | 赞美、鼓励 | 被认可 |
| 精心的时刻 | 全情陪伴 | 被重视 |
| 接受礼物 | 礼物、惊喜 | 被惦记 |
| 服务的行动 | 帮忙做事 | 被照顾 |
| 身体的接触 | 拥抱、牵手 | 被亲密 |

**供需分析问题**：
```
"你用什么方式表达爱？（爱之语）"
"你通常用什么方式感受被爱？"
"你觉得伴侣能读懂你的爱之语吗？"
"伴侣的爱之语是什么？你能接收到吗？"
```

#### 4.3 爱的蓝图重塑（10分钟）

**操作要点**：
- 重新定义爱的期待
- 校准爱的标准
- 建立新的互动模式
- 提升爱的能力

**重塑问题**：
```
"基于我们今天的发现，你对爱有什么新的理解？"
"你愿意尝试用伴侣的爱之语来表达吗？"
"你希望这段关系在未来3个月有什么变化？"
"你愿意为此付出什么？"
```

### 输出物

| 输出项 | 内容 |
|-------|------|
| 愛字四维度分析 | 爫/冖/心/夂的现状 |
| 爱的供需图 | 双方的爱之语和需求 |
| 供需错位点 | 不匹配的地方 |
| 爱的蓝图 | 重塑的方向和目标 |
| 爱的能力提升 | 需要发展的方面 |

### 检查清单

- [ ] 是否完成了愛字四维度分析
- [ ] 是否识别了双方的爱之语
- [ ] 是否发现了供需错位
- [ ] 是否制定了爱的蓝图
- [ ] 用户是否理解了爱的本质

## 第五步：解题（标准操作）

### 目标

1. 制定具体可行的方案
2. 预设可能的障碍
3. 建立追踪机制
4. 确保行动落地

### 核心操作

#### 5.1 方案制定（20分钟）

**操作要点**：
- 选择匹配的解题模型
- 制定具体的行动计划
- 确保行动可执行
- 从"极小化行动"开始

**解题模型匹配**（详见solution-models.md）：

| 卡点类型 | 推荐模型 | 核心机制 |
|---------|---------|---------|
| 沟通型 | SRC模型 | 暂停→映射→连接 |
| 亲密型 | 3R模型 | 修复→重连→深化 |
| 权力型 | GIVE模型 | 协商→投入→验证→调整 |
| 成长型 | 共同进化模型 | 共同目标→协同行动 |
| 信任型 | TRUST模型 | 透明→责任→理解→持续→信任 |
| 价值型 | ALIGN模型 | 评估→学习→整合→协商→落地 |

**行动计划模板**：
```
【极小化行动】
- 行动内容：___
- 开始时间：___
- 持续时间：___
- 成功标准：___

【障碍预设】
- 可能遇到的障碍：___
- 应对策略：___

【支持系统】
- 需要谁的帮助：___
- 需要什么资源：___
```

#### 5.2 障碍预设（10分钟）

**操作要点**：
- 识别可能的内部障碍
- 识别可能的外部障碍
- 制定应对策略
- 建立支持系统

**障碍探索问题**：
```
"你觉得执行这个计划最大的挑战是什么？"
"当你想要放弃时，你会怎么提醒自己？"
"谁可以支持你执行这个计划？"
"你需要一个什么样的"锚点"来提醒自己？"
```

**常见障碍及应对**（详见action-guide.md）：

| 障碍类型 | 具体表现 | 应对策略 |
|---------|---------|---------|
| 旧习惯 | 自动化反应 | 暂停+深呼吸+选择 |
| 旧信念 | "我做不到" | 认知重构+证据法 |
| 外部压力 | 工作/家庭 | 边界设定+优先级 |
| 伴侣抵抗 | 不配合 | 耐心+以身作则 |

#### 5.3 追踪机制（10分钟）

**操作要点**：
- 建立周度追踪
- 设置里程碑
- 定期回顾调整
- 庆祝小成功

**追踪机制**（详见effect-tracking.md）：

| 频率 | 内容 | 时长 |
|-----|------|-----|
| 每日 | 简单记录 | 1分钟 |
| 每周 | 回顾与计划 | 5分钟 |
| 每月 | 深度评估 | 15分钟 |
| 每季 | 系统复盘 | 30分钟 |

**里程碑设置**：
```
【第1周里程碑】
- 完成一次深度对话
- 识别一次旧模式
- 执行一个极小化行动

【第1月里程碑】
- 正负互动比改善
- 冲突处理模式改善
- 亲密连接频率提升

【第3月里程碑】
- 核心卡点明显改善
- 关系能量状态提升
- 建立新的互动模式
```

### 输出物

|-------|------|

| 极小化行动 | 具体可执行的第一步 |

| 追踪机制 | 周/月/季度追踪计划 |

2. **见心（10分钟）**：识别核心模式
3. **解题（15分钟）**：极小化行动 + 追踪机制
