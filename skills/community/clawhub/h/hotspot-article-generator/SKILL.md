# 热点爆款文章生成器

需要执行**完整流程**或涉及数据来源、步骤分支、自检清单时，**必须先读取**与本技能同目录下的 `references/core_workflow.md`，并完整遵循其中的触发规则、术语规范、数据来源约束、操作步骤与注意事项。

## 简介

面向新媒体运营、内容营销与自媒体创作者，在可控合规前提下，把「热点/选题 → 风格对齐 → 可发布成稿」压缩为可重复执行的工作流。

通过本技能，你可以：

- **热点与对标**：按场景选择「追热点」或「直给话题」，并联合同类热门内容做风格与传播要素提取
- **成稿与差异化**：输出情感共鸣 / 实用干货 / 观点评论等方向的成稿，并支持精写单篇与多篇矩阵两种节奏
- **交付闭环**：可选封面说明或规格、爆款理由与发布建议；内置质量检查、去 AI 化表达与 SEO 布局思路

适用于要抢时效的热点稿、要「像爆款」但需平台适配的专栏、以及已有草稿需质检与优化等场景。

## 功能特性

### 核心功能

- **热点追踪**：全网热搜采集、时效分级与筛选（具体来源与约束以 `core_workflow.md` 为准）
- **风格分析**：热门/竞品文章检索，提炼标题、结构、情绪与可复用要素
- **爆款生成**：模仿并强化差异点，支持多版本并行；可按用户指定平台语气调整
- **封面与视觉**：按需提供封面图方案或符合项目内封面规格的输出（见 `references/cover-image-spec.md` 等引用文档）
- **质量控制**：违禁与风险表述检查、事实性核对提示、评分与修复建议
- **SEO 与分发**：关键词与标题变体、摘要与平台侧搜索友好建议

### 特色亮点

- **起点智能判断**：直接话题、追热点、仅优化成稿三种入口，避免重复做无效采集
- **模板与规范对齐**：成稿结构可对齐 `assets/article-template.md` 与 `references/viral-article-guide.md`（由工作流何时引用以 `core_workflow.md` 为准）
- **可审计**：爆款理由、改动说明与自检项可追溯，减少「只有正文没有依据」的交付

## 使用指南

### 基础使用（4 步）

#### 第 1 步：确认模式

根据用户说法判断入口，缺信息时追问，勿臆测赛道或日期范围。

- **直给话题**：用户已给出明确主题或关键词 → 可走快速生成路径（工作流中可能跳过部分热点采集）
- **追热点**：用户要「今日热点」「最近热议」等 → 走完整采集与分析
- **优化成稿**：用户粘贴或上传现有正文 → 走质检与改写分支

> 用户：帮我写一篇小红书风格的「春季露营清单」，要实用一点
> 助手：确认平台（小红书）、体裁（清单/笔记）、字数或字数区间；然后按 `core_workflow.md` 执行竞品与生成步骤。

> 用户：今天有什么热点能写成公众号推文？
> 助手：确认领域偏好或排除项；再按完整流程拉热点并给出可选选题与成稿。

#### 第 2 步：对齐输出规格

追问或确认：目标平台、篇数（精写 1 篇 / 多篇矩阵）、是否需要封面、是否需要 SEO 关键词表、禁忌词或品牌调性。

若用户未指定平台：**列出默认假设并征求确认**，避免误用版式与话术。

#### 第 3 步：执行生成与质检

严格遵循 `references/core_workflow.md` 中的步骤顺序、并行/分支规则及自检清单；涉及封面、版式或爆款拆解细节时，按该文档指向加载 `references/` 下对应文件。

#### 第 4 步：交付与说明

交付正文（及备选标题/摘要如需），并给出简短爆款理由与发布建议；若某步数据不可用，**说明原因与替代方案**，不得编造热搜排名或虚假互动数据。

### 常用口令速查

| 命令示例                          | 功能                   |
| --------------------------------- | ---------------------- |
| `基于今日热点写一篇 [平台] 推文`  | 完整热点流程 + 成稿    |
| `话题：[主题]，要 [风格]，给一篇` | 快速模式，跳过全量热点 |
| `优化下面这篇文章：[正文]`        | 质检、改写与结构建议   |
| `同一选题给我 3 个不同角度的版本` | 多篇差异化输出         |
| `加上封面说明 / 封面规格`         | 触发封面相关交付       |
| `顺带做 SEO：关键词 + 标题备选`   | 强化搜索侧建议         |

### 失败或无法执行时

向用户说明是缺参数、数据源不可用、还是环境限制（如无法联网检索）；**禁止虚构榜单、阅读量与引用来源**；合规边界以 `core_workflow.md` 及平台规范为准。

## 使用场景

| 场景         | 角色        | 需求描述                       | 使用方式                                         |
| ------------ | ----------- | ------------------------------ | ------------------------------------------------ |
| 热点响应发稿 | 新媒体编辑  | 短时间要把热议话题写成可发稿件 | 选追热点模式；并行竞品；出主稿+备选标题；过质检  |
| 账号日常更新 | 自媒体博主  | 稳定产出「像爆款」的笔记或软文 | 直给话题；对齐平台语气；可选封面与标签建议       |
| 投放前润色   | 内容营销    | 成稿需风险词检查与 SEO 加固    | 粘贴原文；走优化分支；输出修改说明与关键词       |
| 多平台矩阵   | 运营负责人  | 同一选题要多端适配的若干版本   | 指定平台列表；多篇模式；统一事实口径与差异化开头 |
| 内部复盘     | 策划 / 增长 | 需要可汇报的「为什么这么写」   | 交付正文附爆款理由、结构映射与自检结果           |

## 注意事项与边界

- **数据真实性**：热搜、案例与引用须与工作流允许的来源一致；抓取或检索失败时如实告知，不补齐虚假数据。
- **版权与引用**：不擅自大段复制受版权保护原文；需要时建议用户取得授权或改为概述。
- **合规**：避免违规医疗理财承诺、恶意引流、个人隐私与非授权肖像描述；各平台社区规范以最新官方规则为准。
- **能力边界**：自动化程度、是否可出图、是否可访问外网等依赖实际运行环境；若不支持应在回复中明确，并给出手动替代步骤（如用户自行配图）。
