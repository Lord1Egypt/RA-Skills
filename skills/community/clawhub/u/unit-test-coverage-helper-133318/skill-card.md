## Description: <br>
Helps developers, QA engineers, maintainers, and product teams add useful unit tests and raise coverage in existing codebases with practical workflows, checklists, code changes, and verification notes. <br>

This skill is ready for commercial/non-commercial use. <br>

## Publisher: <br>
[kyro-ma](https://clawhub.ai/user/kyro-ma) <br>

### License/Terms of Use: <br>
MIT-0 <br>


## Use Case: <br>
Developers, software maintainers, QA engineers, open-source contributors, and product teams use this skill to plan and produce targeted unit tests, coverage-improvement workflows, validation notes, and follow-up risks for existing codebases. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Broad trigger wording and implicit invocation can activate the skill outside a focused unit-testing task. <br>
Mitigation: Invoke it explicitly for testing and coverage work, and check that the proposed workflow matches the current codebase before applying changes. <br>
Risk: Generated test code or coverage advice can miss project conventions or introduce incorrect assertions. <br>
Mitigation: Review proposed code changes and run the project's normal test and coverage commands before merging. <br>
