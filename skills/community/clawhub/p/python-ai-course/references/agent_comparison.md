## 二、核心技术路线对比

### 2.1 闭源 API 路线 vs 开源自建路线

| 决策因素 | 闭源 API（OpenAI/Claude/文心等） | 开源自建（LLaMA/Qwen/DeepSeek） |
|----------|-------------------------------|-------------------------------|
| **前期成本** | 低（按调用付费，零运维） | 高（GPU 服务器 + 运维人力） |
| **长期成本** | 随规模线性增长 | 固定成本 + 边际递减 |
| **数据安全** | 数据需出域（有合规风险） | 数据不出域，完全可控 |
| **模型能力** | 最新最强（SOTA） | 接近但略弱（快速追赶中） |
| **定制灵活性** | 受限（只能调 Prompt 和微调） | 完全自由（可改架构、蒸馏、量化） |
| **延迟** | 依赖网络，有时不稳定 | 本地推理，延迟可控 |
| **适用场景** | 快速验证、中小规模、通用场景 | 大规模、高安全、深度定制 |

### 2.2 低代码平台 vs 代码框架

| 对比维度 | 低代码（Coze/Dify/千帆） | 代码框架（LangChain/AutoGen） |
|----------|-------------------------|------------------------------|
| 上手速度 | 分钟级 | 天到周级 |
| 灵活性 | 受限于平台能力 | 完全自由 |
| 复杂逻辑 | 困难（条件分支多时变「意大利面」） | 灵活（完整编程能力） |
| 可维护性 | 依赖平台，迁移困难 | 代码即资产，Git 管理 |
| 团队要求 | 产品/运营友好 | 需要专业工程师 |
| 适合 | 原型验证、简单工作流 | 生产系统、复杂业务 |

## 四、典型场景推荐方案

### 场景 1：个人开发者快速验证（预算 < ¥500/月）

**推荐方案**：
- 模型：Claude 3.5 Sonnet（代码能力最强）+ DeepSeek（便宜兜底）
- 框架：LangChain（入门最友好）
- 平台：Dify 自部署（Docker Compose 一条命令启动）
- 工具集成：MCP Server 生态

**理由**：成本最低、学习资源最多、社区最活跃

---

### 场景 2：中小团队内部工具（5-20 人，预算 ¥2000-10000/月）

**推荐方案**：
- 模型：Qwen2.5（自部署 7B/14B）+ Claude API（复杂任务备用）
- 框架：LangGraph（工作流编排）
- 平台：Dify（提供前端 UI，团队非技术成员也能用）
- 评估监控：LangFuse（开源免费）

**理由**：数据不出域、成本可控、非技术成员友好

---

### 场景 3：企业级生产系统（100+ 人，预算不限）

**推荐方案**：
- 模型：多层路由（简单任务用 Haiku/DeepSeek，复杂任务用 Claude 3.5/GPT-4o）
- 框架：LangGraph + Semantic Kernel（多语言支持）
- 平台：自研 + Dify/LangSmith
- 基础设施：vLLM（推理服务）+ Milvus（向量库）+ Kafka（事件驱动）
- 可观测性：LangSmith + Prometheus + Grafana

**理由**：高可用、可观测、多模型路由降本

---

### 场景 4：高安全合规场景（金融/医疗/政务）

**推荐方案**：
- 模型：国产合规模型（Qwen/DeepSeek/百川），完全私有化部署
- 框架：LangChain/LangGraph（私有化部署）
- 平台：全自研，不依赖任何外部 SaaS
- 安全层：Guardrails + 自建审计日志
- 硬件：国产 GPU（华为昇腾 + 寒武纪）或 Nvidia 合规渠道

**理由**：数据不出域是底线，全链路可控

## 五、选型五维度打分卡

教学时让学员根据自身情况，给以下五个维度打分（1-5 分），再匹配推荐方案：

| 维度 | 1 分 | 3 分 | 5 分 |
|------|------|------|------|
| **团队规模** | 个人 | 5-20 人 | 100+ 人 |
| **预算** | < ¥500/月 | ¥2000-10000/月 | 不限 |
| **数据安全** | 无特殊要求 | 客户数据需保护 | 金融/医疗合规 |
| **定制需求** | 标准化功能 | 部分定制 | 深度定制 |
| **延迟要求** | >5s 可接受 | 2-5s | <1s 实时 |

**得分与方案对应**：
- 5-10 分 → 场景 1（低成本快速验证）
- 11-17 分 → 场景 2（中小团队）
- 18-22 分 → 场景 3（企业级）
- 23-25 分 → 场景 4（高合规）

## 六、2025-2026 年 Agent 趋势判断

1. **MCP 协议将成为标配**：类似 HTTP 之于 Web，MCP 会成为 AI 工具交互的通用语言
2. **推理模型驱动 Agent**：o1/R1 类推理模型大大提升 Agent 规划和决策质量
3. **多 Agent 系统主流化**：单 Agent 的「全知全能」不现实，分工协作是必然
4. **国产模型+国产框架闭环**：Qwen/DeepSeek + Dify/Coze 的国内生态日趋成熟
5. **Agent 即服务（AaaS）**：Agent 会像 SaaS 一样成为标准的企业服务形态
6. **安全合规推动私有化部署**：数据安全是 Agent 落地的第一道门槛，不是最后一道

## 教学使用指南

1. **先画图再讲文字**：用 `show_widget` 绘制「大厂 Agent 产品矩阵图」「选型决策树」「技术路线对比图」
2. **引导学员自评**：让学员先完成「五维度打分」，再给推荐方案，更有代入感
3. **强调变化性**：提醒学员 Agent 领域变化极快，选型结论有 6-12 个月的有效期，定期重新评估
4. **案例驱动**：每个典型场景配 1 个真实案例（可用匿名化的企业案例）
5. **不下定论**：不要宣称「XX 方案最好」，而是「在 XX 条件下，XX 方案更合适」
