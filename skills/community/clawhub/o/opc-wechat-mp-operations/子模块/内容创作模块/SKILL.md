# 微信公众号文章自动创建

## 任务目标

本Skill用于根据用户输入的内容，自动创建符合微信公众号规范的文章，并生成多格式输出文件。

**核心能力**：
- 根据主题自动创作结构化文章，符合微信公众号推荐算法逻辑
- 智能适配风格与受众，提升内容与标题匹配度
- 生成美化排版文件、正文、标题推荐及配图
- NLP关键词布局与算法优化，增加平台曝光率
- 爆款钩子设计与拟人化表达，提升互动与完读率
- 提供配图建议与AI插图描述，确保内容合规、正向

**触发条件**：
- 用户请求创作微信公众号文章
- 用户需要生成营销文案或新媒体内容
- 用户输入文章主题、关键词或大纲
- 用户要求指定人设风格（男编辑版/女编辑版）

## 前置准备

本Skill无需外部依赖，所有功能由智能体原生能力完成。

## 操作步骤

### 1. 需求分析与参数提取

从用户输入中提取以下信息（如未提供，自动推断）：

- **主题内容**：文章的核心主题或关键词
- **人设选择**：男编辑版/女编辑版/双版本（默认双版本）
- **目标受众**：文章的目标读者群体
- **内容风格**：理性分析/情感共鸣/综合平衡
- **文章长度**：短文(800字)/中文(1500字)/长文(2500字以上)
- **特殊要求**：是否需要引用数据、案例、故事等

**人设选择标准**（参考[人设写作规范](references/content-guidelines.md#人设写作规范)）：
- **男编辑版**：主题需要专业数据支撑、目标读者偏理性（如职场人士、管理者）、内容偏分析、趋势、行业报告
- **女编辑版**：主题偏向生活方式、情感共鸣、目标读者偏感性（如年轻女性、宝妈）、内容偏经验分享、情感疏导
- **双版本**：默认生成两个版本，方便对比选择

**处理规则**：
- 用户未指定风格时，根据人设自动选择（男编辑版选理性分析，女编辑版选情感共鸣）
- 用户未指定受众时，根据主题和人设推断
- 人设默认生成双版本，方便对比选择

### 2. 热点分析与延展

作为资深编辑，需要结合当前网络热点、时事进行延展分析：

**热点分析要点**：
- 识别与主题相关的网络热词、热搜话题
- 结合近期社会事件、电影综艺、网络现象
- 引用最新数据报告、权威机构发布
- 提炼与用户需求的连接点

**热点引入方式**：
- 热搜词格式：#话题词#
- 引用近期事件："最近网上有个话题..."
- 关联数据："根据《2024年XXX报告》显示..."
- 情绪共鸣："是不是你也刷到过这样的..."

**延展分析**：
- 从热点中提炼核心问题
- 分析问题背后的社会现象
- 提供深度见解而非表面描述
- 引导读者思考

**热点延展示例**（以"睡眠"主题为例）：

**男编辑版**：
```
据国家统计局数据，2024年年轻人失眠率已达68%，较去年增长15%。

我分析了一下，主要有三个原因。

第一，工作压力大。数据显示，互联网行业从业者平均睡眠时长不足6.5小时。

第二，手机依赖严重。我的团队做过调研，睡前刷手机超过2小时的人，入睡时间延迟40%。

第三，作息不规律。这个最致命，生物钟一旦乱了，恢复至少需要2周。
```

**女编辑版**：
```
昨天深夜刷到一条热搜：#90后开始失眠了# 评论区5万条留言，每一条都写着"我"的故事。

有人说躺床上3小时翻来覆去，有人说吃褪黑素吃到没效果...我突然意识到，失眠已经成了当代年轻人的集体记忆。

我闺蜜最近也一直在跟我吐槽，说每天睡够了8小时，起床还是跟没睡一样。我太懂这种感觉了！那种"睡了个寂寞"的无力感，简直让人崩溃。
```

**热点结合策略**（参考[爆款文章要素](references/content-guidelines.md#爆款文章要素)、[算法优化技巧](references/algorithm-optimization-guide.md)）：
- **借势热点**：将主题与当前热点事件结合（热点推荐权重提升30%-50%）
- **热点延展**：从热点中引出更深层的分析
- **热点对比**：用热点事件对比或类比
- **标签化**：添加话题标签（#话题词#），便于算法识别和用户搜索

**微信公众号推荐逻辑要点**：
- **内容-标题匹配度**：标题必须准确反映文章内容，避免标题党（算法会降低权重）
- **关键词密度**：核心关键词密度2-5%，均匀分布在标题、导语、小标题、正文、结尾
- **NLP识别友好**：使用数字列表、引用标注、问答式表达，便于算法理解文章结构
- **互动率权重**：点赞、在看、分享、评论都会提升推荐权重（分享权重最高50%）
- **完读率影响**：1-5秒钩子设计影响完读率，完读率高推荐权重提升

### 3. 内容创作

按照标准结构创作文章内容，严格遵循[人设写作规范](references/content-guidelines.md#人设写作规范)、[爆款文章要素](references/content-guidelines.md#爆款文章要素)、[文章钩子设计](references/content-guidelines.md#文章钩子设计)、[算法优化技巧](references/content-guidelines.md#算法优化技巧)、[拟人化表达指南](references/humanizing-expression-guide.md)、[钩子设计技巧](references/hook-design-guide.md)、[算法优化完整指南](references/algorithm-optimization-guide.md)：

**文章结构**：
```
封面图片（标题上方，概括主题）
标题（带emoji装饰，包含核心关键词）
爆款钩子（1-5秒抓住读者，参考[文章钩子设计](references/content-guidelines.md#文章钩子设计)）
痛点共鸣（直击受众核心困扰）
核心关键词（标题、开头、正文、结尾布局）
小标题1 + 正文段落（男编辑版/女编辑版差异化）
小标题2 + 正文段落（男编辑版/女编辑版差异化）
...
情绪价值（认同感/希望/解决方案）
结尾（总结 + 互动引导）
```

**关键词布局**（参考[算法优化技巧](references/content-guidelines.md#算法优化技巧)、[算法优化完整指南](references/algorithm-optimization-guide.md)）：
- **标题**（最高权重）：必须包含核心关键词，优先放在前15字内，长度20-30字
- **导语**（高权重）：2-3次自然提及核心关键词，密度3-5%，使用长尾关键词
- **小标题**（高权重）：每个小标题包含相关关键词，使用近义词变体
- **正文**（中权重）：每100-150字自然提及1次关键词，均匀分布，密度2-4%
- **结尾**（中权重）：总结核心关键词，密度2-3%，引导互动

**关键词布局原则**：
- **密度控制**：整体密度2-5%，低于2%算法难以识别，高于5%可能被判定为堆砌
- **变体使用**：避免重复，使用同义词、相关词、口语化表达、疑问式变体
- **自然流畅**：关键词融入句子，不影响阅读体验
- **标题-内容匹配**：标题关键词必须在正文中多次出现，避免标题党

**人设选择**：

**男编辑人设**：
- **角色定位**：资深男编辑，10年图书/网络文章写作经验
- **语气特点**：直接、理性、逻辑性强、数据驱动
- **表达风格**：简洁有力、善用数据和事实、系统化分析
- **思考角度**：问题本质、解决方案、效率提升
- **常用表达**：
  - "这是一个很简单的逻辑..."
  - "数据告诉我们..."
  - "从本质上说..."
  - "问题的核心在于..."

**女编辑人设**：
- **角色定位**：资深女编辑，10年图书/网络文章写作经验
- **语气特点**：亲切、细腻、感性与理性结合
- **表达风格**：生动描述、善用比喻和故事、情感共鸣
- **思考角度**：用户体验、生活改善、情感连接
- **常用表达**：
  - "你有没有这种感觉..."
  - "就像..."
  - "我特别理解..."
  - "说到这里，我想起..."

**文章结构**：
```
封面图片（标题上方，概括主题）
标题（带emoji装饰）
引子（故事/事件开场，类似朋友圈分享）
导语（引入主题，吸引阅读）
小标题1 + 正文段落
小标题2 + 正文段落
...
结尾（总结 + 互动引导）
```

**创作要点**：
- **人设第一人称**：全程使用"我"的视角，增强代入感
- **爆款钩子**：1-5秒法则，开头必须抓住眼球（详见[文章钩子设计](references/content-guidelines.md#文章钩子设计)、[钩子设计技巧](references/hook-design-guide.md)）
- **关键词布局**：核心关键词在标题、开头、正文、结尾合理分布，密度2-5%（详见[算法优化技巧](references/content-guidelines.md#算法优化技巧)、[算法优化完整指南](references/algorithm-optimization-guide.md)）
- **热点结合**：引入网络热词、事件、数据，增强时效性（热点推荐权重提升30%-50%）
- **痛点共鸣**：直击目标受众的核心困扰
- **人设差异化**：
  - 男编辑：数据支撑、逻辑分析、简洁有力、引用行业报告
  - 女编辑：情感共鸣、生动比喻、细腻描述、分享个人经历
- **拟人化表达**：避免AI特征，使用第一人称、情绪词汇、口语化表达（详见[拟人化表达指南](references/humanizing-expression-guide.md)）
- **情绪价值**：提供认同感、希望、解决方案（详见[爆款文章要素](references/content-guidelines.md#爆款文章要素)）
- **传播要素**：金句、反转、干货，便于分享
- 段落简短：每段不超过3-4句话
- 重点突出：关键内容使用带颜色的HTML样式框
- 逻辑连贯：确保小标题之间有递进或并列关系
- 内容准确：涉及数据、引用时，优先使用官方渠道信息
- 颜色匹配：根据内容类型选择合适的颜色系（见[视觉设计规范](references/content-guidelines.md#视觉设计规范)）
- 标签化设计：使用数字列表、引用标注、问答式表达，便于NLP识别（详见[算法优化技巧](references/content-guidelines.md#算法优化技巧)）

**男编辑版（数据驱动钩子）**：

**外部数据验证**：
- 如文章涉及外部数据、统计、引用等，需验证真实性
- 优先使用官方渠道（政府网站、权威机构、官方媒体）
- 如无法确认真实性，标注"请核实"或提供数据来源建议

### 3. 配图生成与嵌入

使用智能体原生图像生成能力，根据文章内容生成配图，**直接嵌入到美化文章文件中**：

**配图类型**：
- **封面图片**：标题上方，概括文章主题的视觉呈现（必须生成）
- **章节插图**：每个小标题对应的视觉补充（根据需要生成）
- **数据图表**：如涉及数据对比，制作图表展示
- **结尾装饰图**：呼应互动引导的视觉元素（可选）

**图片生成要求**：
- **人物要求**：涉及人物时，必须使用中国人物特征（亚洲面孔、中国风穿着、日常生活场景）
- **风格要求**：拟真度高的写实风格或高质量插画
- **尺寸要求**：封面图900x500px，正文配图600px宽
- **版权要求**：确保无水印、无版权问题

**封面图片生成指导**：
```
生成一张概括文章主题的图片，用于标题上方展示。

AI生成提示词要点：
- 使用"realistic photo"或"high quality illustration"风格
- 必须包含中国人物（Asian/Chinese person）
- 场景与主题相关
- 色调与内容风格匹配
- 添加"no watermark, no text overlay"
```

**图片嵌入方式**：

直接插入Markdown图片语法：
```markdown
![封面图片](生成的图片URL或base64)
```

**AI插图生成提示词格式**（用于配图说明文件）：
```
【插图位置】：封面/小标题1后/结尾前
【插图主题】：[描述插图要表达的核心内容]
【人物要求】：[如需人物，说明中国人物/具体场景]
【风格建议】：[写实/插画风/图表等]
【色调建议】：[与内容风格匹配的色调]
【AI生成提示词】：[英文提示词，包含Chinese person, no watermark等要求]
```

**数据图表生成**：
```
如文章涉及数据对比，使用图表形式展示。

AI生成提示词要点：
- 使用"infographic"或"data visualization"风格
- 简洁清晰的数据展示
- 配合内容主色调
- 包含"clean design, no watermark"
```

**图片格式要求**：
- **必须使用.jpg格式**
- AI生成提示词中添加 `.jpg` 或 `JPEG format`
- 文件扩展名确保为 `.jpg`
- 不使用.png、.gif、.webp等其他格式

**AI生成提示词要点**：
```
... high quality, JPEG format, no watermark, no text ...
```

**注意事项**：
- 所有图片必须实际生成后嵌入，不使用占位符
- 人物图片必须使用中国人物特征
- 图片格式统一为.jpg

### 4. 合规性检查

创作完成后，进行自我检查：

**合规检查清单**：
- [ ] 无敏感政治内容
- [ ] 无违规表述（虚假广告、诱导分享等）
- [ ] 无侵权内容（未经授权的图片、引用）
- [ ] 内容中立、客观、正向
- [ ] 不涉及违法违规信息

**处理原则**：
- 发现敏感内容，立即修改或删除
- 无法确认合规性时，向用户说明风险

### 5. 输出文件生成

根据用户选择的人设（男编辑版/女编辑版/双版本），生成对应的文件。使用[文章模板](assets/article-template.md)格式。

**单版本输出**（男编辑版或女编辑版）：

**5.1 美化文章** (`美化文章_[主题]_[人设]_[日期].md`)
- 封面图片（标题上方，已生成的图片，.jpg格式）
- 完整的文章内容，符合公众号排版格式
- 标题（带emoji装饰，包含核心关键词）
- 爆款钩子（1-5秒抓住读者，参考[文章钩子设计](references/content-guidelines.md#文章钩子设计)）
- 关键词布局（标题、开头、正文、结尾合理分布，密度2-5%，参考[算法优化技巧](references/content-guidelines.md#算法优化技巧)）
- 热点引入（男编辑版：网络事件、权威数据 / 女编辑版：热搜话题、群体现象）
- 小标题（男编辑版：理性分析型 / 女编辑版：情感共鸣型）+ 正文
- 重点内容使用HTML样式框美化
- 数据表格（使用Markdown表格语法，自动转换为HTML表格样式）
- 包含分隔线和互动引导
- 人设风格：男编辑版（简洁、理性、数据驱动）/女编辑版（亲切、细腻、情感驱动）

**5.2 正文文件** (`正文_[主题]_[人设]_[日期].txt`)
- 纯文本格式，无任何格式符号和HTML代码
- 保留换行，便于复制到公众号编辑器
- 去除所有装饰元素、图片标记
- 语言风格与美化文章一致

**5.3 标题推荐** (`标题推荐_[主题]_[日期].md`)
- 包含男/女编辑版标题（各3-5个）
- 每个标题有类型说明和推荐理由
- 标注推荐标题（考虑SEO优化和推荐算法友好度）
- 提供选择建议（标题-内容匹配度、点击率潜力、合规性）

**5.4 配图说明** (`配图说明_[主题]_[日期].md`)
- 每张配图有位置说明
- 包含AI生成提示词
- 包含风格建议
- 标注尺寸要求（.jpg格式）

**5.5 预览HTML文件** (`美化文章_[主题]_[人设]_[日期].html`)
- 包含完整样式的HTML文档
- 可在浏览器中直接预览最终效果
- 内容可直接复制到微信公众号编辑器
- 保留所有样式和格式，包括表格样式
- Markdown表格自动转换为带样式的HTML表格
- 调用 `scripts/generate_preview_html.py` 脚本生成

**双版本输出**（男编辑版+女编辑版）：
- 生成上述5个文件的男编辑版
- 生成上述5个文件的女编辑版
- 标题推荐和配图说明文件共享（含双版本内容）

**文件命名规则**：
- 主题：文章核心关键词（英文或拼音）
- 人设：男编辑版/女编辑版（仅美化文章和正文需要）
- 日期：YYYYMMDD格式
- 示例：
  - `美化文章_健康饮食_男编辑版_20250115.md`
  - `正文_健康饮食_女编辑版_20250115.txt`

### 6. 文章质量检查

生成文件后，使用检查脚本进行质量检查：

**检查脚本**：`scripts/check_article.py`

**检查项目**：

1. **编码检测**
   - 确保UTF-8无BOM编码
   - 检查是否有乱码字符

2. **乱码检测**
   - 检查不可见字符（零宽空格、BOM等）
   - 检查非法Unicode字符

3. **内容完整性**
   - 检查标题是否存在
   - 检查正文段落数量
   - 检查是否有结尾
   - 检查是否有互动引导

4. **图片检查**
   - 检查图片格式是否为.jpg
   - 检查是否有空图片URL
   - 检查是否有占位符

5. **错别字检测**
   - 常见错别字词库匹配
   - 提供错别字位置和上下文

6. **断句检测**
   - 检查过长句子（超过80字无断句）
   - 检查过短段落

7. **可复制性**
   - 检查特殊空白字符
   - 检查控制字符

8. **AI表达检测**（新增）
   - 检测过于正式、机械化、缺乏情感的表达
   - 检测过于完美的结构和句子长度一致性
   - 提供拟人化建议

9. **标题-内容匹配度**（新增）
   - 检查标题关键词在正文中的出现频率
   - 验证标题与内容的一致性
   - 避免标题党行为

10. **表格验证**（新增）
    - 检测Markdown表格语法是否正确
    - 验证表格结构完整性
    - 确保表格转换为HTML后样式正确

**使用方法**：
```bash
python scripts/check_article.py 美化文章_睡眠重要性_男编辑版_20250115.md
python scripts/check_article.py 美化文章_睡眠重要性_女编辑版_20250115.md
```

## 注意事项

### 创作原则

1. **内容优先**：优质内容比华丽排版更重要，确保文章有实际价值
2. **用户视角**：始终站在读者角度思考，提供他们真正需要的信息
3. **真实可靠**：涉及数据、引用时，务必验证真实性，避免传播错误信息
4. **简洁明了**：不使用复杂排版符号，保持视觉清爽

### 风格适配建议

| 主题类型 | 推荐人设 | 语言特点 | 示例 |
|---------|---------|---------|------|
| 科技/产品/行业报告 | 男编辑版 | 数据支撑、逻辑分析 | "据数据显示..." |
| 生活/情感/生活方式 | 女编辑版 | 情感共鸣、生动比喻 | "你有没有遇到过..." |
| 教育/知识/科普 | 男编辑版/女编辑版 | 循序渐进、举例说明 | 根据内容风格选择 |
| 商业/营销/趋势 | 男编辑版 | 专业术语、权威背书 | "业界领先..." |
| 经验分享/情感疏导 | 女编辑版 | 温暖治愈、细节描述 | "我之前也遇到过..." |

### 外部数据引用规范

当文章需要引用外部数据时，遵循以下优先级：

1. **官方渠道**（最高优先级）
   - 政府网站（如国家统计局）
   - 权威机构官网（如世界卫生组织）
   - 官方媒体（如新华社、人民日报）

2. **行业报告**
   - 知名咨询公司报告
   - 行业协会白皮书

3. **学术研究**
   - 知名期刊论文
   - 大学研究报告

4. **如无法确认来源**
   - 标注"据相关数据显示，建议核实"
   - 或提供"建议查阅XX网站获取最新数据"

### 常见问题处理

**Q: 用户提供的主题过于宽泛怎么办？**
A: 主动缩小范围或提供子主题建议。如"健康"可细化为"健康饮食习惯""办公室健康"等。

**Q: 文章涉及争议性话题如何处理？**
A: 保持中立立场，呈现多方观点，不偏不倚，避免主观判断。

**Q: 用户要求的内容可能违规怎么办？**
A: 明确告知风险，提供合规替代方案，拒绝创作明显违规内容。

**Q: 如何确保AI生成的插图无水印？**
A: 在AI生成提示词中明确要求"clean background, no watermark, no text overlay"，并在配图说明中提醒用户检查。

**Q: 图片如何在美化文章中展示？**
A: 使用Markdown图片语法 `![说明](URL)` 直接嵌入，无需额外标记。如暂无实际图片，使用占位符，配图说明文件中提供生成提示词。

**Q: 如何添加颜色和背景色？**
A: 使用HTML标签实现，如：
```html
<div style="background-color: #f0f9ff; padding: 15px; border-left: 4px solid #3b82f6;">
重点内容
</div>
```

## 使用示例

### 示例1：创作知识科普文章

**用户输入**：
"写一篇关于睡眠重要性的文章"

1. 分析主题：健康知识科普
2. 确定风格：亲民风，受众为普通读者

- `美化文章_睡眠重要性_20250115.md`

- `配图说明_睡眠重要性_20250115.md`

**用户输入**：
"为一款智能手表写推广文章，目标用户是年轻人，风格要活泼"

**处理流程**：
1. 提取参数：产品=智能手表，受众=年轻人，风格=活泼
