# 第十三篇：质量检查验收标准体系

> 核心标准：GB/T 18316-2008、GB/T 24356-2023、GB/T 33176-2016、GB/T 39610-2020、GB/T 22483-2008
> 补充来源：湖北省DLG技术规程(2022)、湖北省DEM技术要求(2022) 质量控制章节
> **贯穿性主题**——本体系中的精度指标与第一篇的坐标系基准、第七篇的数据转换精度、第九篇的制图规范直接关联。

## 13.1 二级检查一级验收制度

这是中国测绘成果质量控制的核心制度，适用于数字测绘成果、模拟测绘成果等所有类型。

| 环节 | 执行主体 | 检查方式 | 关键要求 |
|------|---------|---------|---------|
| **过程检查** | 测绘单位作业部门 | 全数检查（逐单位成果详查） | 不作等级评定；自查互查后方可进行；错误修改后复查至无误 |
| **最终检查** | 测绘单位质量管理部门 | 全数检查（逐单位成果详查） | 逐幅评定质量等级；野外检查项可抽样；审核过程检查记录 |
| **验收** | 生产委托方 | 抽样检查 | 评定单位和批成果等级；审核最终检查记录；不合格退回重验需重新抽样 |

**核心原则**：各级检查独立进行，不应省略或代替。

> **神经连接**：二级检查一级验收 → 第十七篇 GDB→DWG 转换后验证（转换结果质量检查）、第九篇制图规范（图面质量）、第六篇 CASS 质量检查功能。

## 13.3 质量评定指标体系

### 计分方法（三类检查项）

| 检查项类型 | 适用场景 | 计分公式 |
|-----------|---------|---------|
| **符合性检查** | 空间参考系、逻辑一致性概念/格式项 | 符合 s=100，不符合 → 该质量元素不合格 |
| **中误差检查** | 位置精度 | m ≤ 0.3m₀ → s=100；m > 0.3m₀ → s=60+40/(0.7m₀)×(m₀-m) |
| **错误率检查** | 属性精度、完整性、拓扑一致性、表征质量 | r ≤ r₀ → s=60+40/r₀×(r₀-r) |

### 错误率限值（r₀）

| 要素类别 | 属性精度/完整性 | 拓扑一致性 | 表征质量 |
|---------|---------------|-----------|---------|
| **极重要要素** | 0% | — | 0% |
| **重要要素** | 0.05% | 0.07% | 0.07% |
| **一般要素** | 0.3% | 0.4% | 0.4% |

**极重要要素定义**：国界、国界界桩界碑及其注记。
**重要要素定义**：境界及界桩界碑、县级以上地名/居民地、县级以上公路及桥梁、测量控制点、干线铁路及桥梁、高速公路及桥梁、六级以上河流及相通湖泊水库、重要管线、一/二级山脉名称。

### 特别不合格情形

- 整体或普遍问题明显大于限值 → 质量元素直接不合格
- 数据处理导致影像裂缝/漏洞 → 影像/栅格质量不合格
- 地物地貌特征严重失真 → 表征质量不合格
- 图号错 → 附件质量不合格
- 一组等高线/要素层多余或遗漏 → 完整性不合格

## 13.4 单位成果与批成果质量评定

### 单位成果质量等级

**质量得分**：S = min(Sᵢ)，取所有质量元素得分的最小值（短板效应）。

| 得分范围 | 质量等级 |
|---------|---------|
| 90 ≤ S ≤ 100 | **优级品** |
| 75 ≤ S < 90 | **良级品** |
| 60 ≤ S < 75 | **合格品** |
| 质量元素不合格/粗差比例>5% | **不合格品** |

> **注意**：附件质量可不参与计算，其他8个质量元素全部参与。

### 批成果合格判定

| 等级 | 判定条件 | 后续处理 |
|------|---------|---------|
| 批合格 | 样本中未发现不合格单位成果，且概查未发现不合格 | 验收中发现的质量问题均应修改 |
| 批不合格 | 样本中发现不合格单位成果/概查中发现不合格/不能提交技术性文档 | 逐一查改合格后重新提交验收 |

## 13.A 逐质量元素详细检查清单

### A.1 空间参考系（代码01）

| 检查内容 | 检查方法 | 常用工具 | 常见错误 |
|---------|---------|---------|---------|
| 大地基准是否为CGCS2000 | 自动：读取数据坐标系定义 | ArcGIS 属性→坐标系；FME CoordinateSystemValidator | 误用WGS84或北京54 |
| 高程基准是否为1985国家高程基准 | 目视+自动：检查元数据与高程值范围 | ArcGIS 元数据查看器；CASS图面信息 | 使用1956黄海高程系未转换 |
| 投影参数是否正确（3°带/6°带） | 自动：检查投影定义与中央经线 | ArcGIS Project工具验证；Proj4库 | 中央经线选错或带号不一致 |
| 高斯-克吕格投影带号与图幅是否匹配 | 自动：图幅编号反算带号对比 | 自定义脚本；FME AttributeManager | 1:2000图幅跨带未处理 |

### A.2 位置精度（代码02）

| 检查内容 | 检查方法 | 常用工具 | 常见错误 |
|---------|---------|---------|---------|

| 高程中误差是否超限 | 抽样：水准检测点对比 | ArcGIS 3D Analyst；CASS高程检查 | 等高线高程偏差>0.7m（1:2000丘陵） |

## 13.B CASS 11.0 质量检查实操 SOP

### B.1 图形检查

**操作步骤**：

1. 打开CASS 11.0，加载待检查DWG文件
2. 菜单：`检查` → `图形检查` → `重叠线检查`
   - 设置容差（建议0.001m），执行检查
   - 检查结果在"检查结果"面板中列出，双击定位到问题位置
3. 菜单：`检查` → `图形检查` → `缝隙检查`
   - 选择面层（如房屋面层JMD），设置缝隙阈值（0.01m）
4. 菜单：`检查` → `图形检查` → `自相交检查`
   - 选择所有线层（道路DLSS、水系SXSS等），执行检查
5. 菜单：`检查` → `图形检查` → `微小面检查`
   - 面积阈值设置：1:500→0.5m²，1:1000→2m²，1:2000→8m²

| 检查项 | 容差/阈值 | 适用图层 |
|--------|----------|---------|
| 重叠线 | 0.001m | 所有线层 |
| 缝隙 | 0.01m | 房屋面层JMD |
| 自相交 | — | 道路DLSS、水系SXSS |
| 微小面 | 按比例尺设定 | 所有面层 |
| 悬挂线 | 0.01m | 道路、水系线层 |

### B.2 属性检查

**操作步骤**：

1. 菜单：`检查` → `属性检查` → `编码检查`
   - 系统自动对比CodeField.db编码字典，列出非法编码
   - 非法编码标红显示，可直接修改
2. 菜单：`检查` → `属性检查` → `必填字段检查`
   - 配置必填字段列表：编码(CODE)、图层(LAYER)、名称(NAME)
   - 缺失字段要素高亮显示
3. 菜单：`检查` → `属性检查` → `编码-图层对应检查`
   - 比对layer.def映射表，检查编码与图层是否匹配
   - 不匹配项自动生成错误清单

**编码-图层典型对应规则（CASS 11.0）**：

| 编码前缀 | 应归图层 | 常见错误 |
|---------|---------|---------|
| 1311xx | JMD（居民地） | 误归入DLSS（道路） |
| 1621xx | DLSS（道路） | 误归入JMD |
| 2111xx | SXSS（水系） | 编码前缀写错归入GXYZ |
| 3111xx | GXYZ（地貌） | 等高线编码归入ZBTZ |
| 4111xx | ZBTZ（植被土质） | 与GXYZ混淆 |

### B.3 拓扑检查

**操作步骤**：

1. 菜单：`检查` → `拓扑检查` → `面层闭合检查`
   - 选择面层（JMD/SXSS），检查是否所有面均为闭合多边形
   - 未闭合面标黄色，可一键闭合修复
2. 菜单：`检查` → `拓扑检查` → `线层连通检查`
   - 选择道路网络，检查端点是否正确连接
   - 悬挂节点（dangle）标红，伪节点（pseudo）标蓝
3. 菜单：`检查` → `拓扑检查` → `重叠面检查`
   - 检查面层是否存在空间重叠
   - 适用于JMD层房屋面重叠检测

### B.4 输出检查报告

**操作步骤**：

1. 完成所有检查后，菜单：`检查` → `输出检查报告`
2. 选择报告格式：HTML（推荐）或TXT
3. 报告内容自动汇总：
   - 检查项目清单及结果统计
   - 各类问题数量及占比
   - 问题要素坐标定位信息
4. 报告保存路径：项目目录/质检/检查报告_日期.HTML
5. 附带问题要素图层（单独DWG），便于修改定位

```python
# CAL脚本：检查JMD层编码前缀
import arcpy

def check_jmd_code_prefix(feature_class, code_field="CODE"):
    """检查JMD层编码是否以131开头"""
    errors = []
    with arcpy.da.SearchCursor(feature_class, ["OID@", code_field, "SHAPE@"]) as cursor:
        for oid, code, shape in cursor:
            if code and not str(code).startswith("131"):
                errors.append({"OID": oid, "CODE": code, "X": shape.centroid.X, "Y": shape.centroid.Y})
    return errors
```

### D.4 PythonCaller 自定义检查逻辑

**示例1：检查高程点密度**

```python
import fme
import fmeobjects

class ElevationDensityChecker(object):
    """检查高程点密度是否满足规范要求"""
    def __init__(self):
        self.min_density = 10  # 每km²最少10个高程点（1:2000平地）
        self.grid_size = 1000  # 1km格网

    def input(self, feature):
        x = float(feature.getAttribute('fme_geometry{0}.x'))
        y = float(feature.getAttribute('fme_geometry{0}.y'))
        grid_key = f"{int(x/self.grid_size)}_{int(y/self.grid_size)}"
        if not hasattr(self, 'grid_counts'):
            self.grid_counts = {}
        self.grid_counts[grid_key] = self.grid_counts.get(grid_key, 0) + 1
        self.features.append(feature)

    def close(self):
        for key, count in self.grid_counts.items():
            if count < self.min_density:
                # 输出密度不足的格网
                output_feature = fmeobjects.FMEFeature()
                output_feature.setAttribute('grid_key', key)
                output_feature.setAttribute('point_count', count)
                output_feature.setAttribute('min_required', self.min_density)
                output_feature.setAttribute('issue_type', '高程点密度不足')
                self.pyoutput(output_feature)
```

**示例2：检查编码-图层映射一致性**

```python
import fme

# CASS编码前缀-图层映射表
CODE_LAYER_MAP = {
    '131': 'JMD',  # 居民地
    '162': 'DLSS', # 道路
    '211': 'SXSS', # 水系
    '311': 'GXYZ', # 地貌
    '411': 'ZBTZ', # 植被土质
    '111': 'CLKZD',# 测量控制点
    '511': 'GX',   # 管线
    '611': 'JJTJZQ',# 境界
}

class CodeLayerChecker(object):
    def __init__(self):
        pass
    def input(self, feature):
        code = str(feature.getAttribute('CODE') or '')
        layer = str(feature.getAttribute('LAYER') or '')
        prefix = code[:3]
        expected_layer = CODE_LAYER_MAP.get(prefix)
        if expected_layer and layer != expected_layer:
            feature.setAttribute('_issue_type', '编码图层不匹配')
            feature.setAttribute('_expected_layer', expected_layer)
            feature.setAttribute('_actual_layer', layer)
            feature.setAttribute('_code', code)
        self.pyoutput(feature)
```

## 13.F 常见质量缺陷案例库

### 缺陷1：平面精度超限（中误差>限值）

| 项目 | 内容 |
|------|------|
| **缺陷描述** | 1:2000 DLG平地平面中误差实测值1.5m，超限值1.2m |
| **产生原因** | ①像控点布设间距过大（>800m）；②像控点刺点误差偏大；③空三解算精度不足 |
| **检查方法** | 野外散点检测：选取30+均匀分布检测点，RTK测量坐标与图上坐标对比，计算中误差 |
| **修改方法** | ①补测像控点并重新空三解算；②对超限区域重新采集矢量；③增加检测点验证 |
| **预防措施** | 像控点间距控制在600m以内；空三解算后先做精度预评估再采集矢量 |

### 缺陷2：属性编码错误（CASS编码与要素不符）

| 项目 | 内容 |
|------|------|
| **缺陷描述** | 建筑物编码131100误写为13110（少一位），导致CASS符号化错误 |
| **产生原因** | ①作业员手动输入编码出错；②批量赋值时编码字典版本不一致 |
| **检查方法** | CASS编码检查功能自动比对CodeField.db；FME AttributeValidator正则校验 |
| **修改方法** | 在CASS中选中错误要素→修改属性→输入正确编码131100→重新符号化 |
| **预防措施** | 使用CASS编码下拉选择而非手动输入；统一编码字典版本并锁定 |

### 缺陷3：拓扑错误（面层未闭合/线层自相交）

| 项目 | 内容 |
|------|------|
| **缺陷描述** | 房屋面层存在12处未闭合多边形，道路线层5处自相交 |
| **产生原因** | ①矢量化时终点未捕捉到起点；②编辑时误操作产生折返线段 |
| **检查方法** | ArcGIS Topology规则/GeometryValidator自动检测；CASS拓扑检查功能 |
| **修改方法** | 未闭合面：CASS一键闭合修复或手动捕捉闭合；自相交线：删除多余节点重新绘制 |
| **预防措施** | 编辑时开启节点捕捉（容差0.001m）；面层编辑完成后立即运行闭合检查 |

### 缺陷4：高程点密度不足

| 项目 | 内容 |
|------|------|
| **缺陷描述** | 1:2000平地区域高程点密度仅6点/km²，低于规范要求的10点/km² |
| **产生原因** | ①丘陵区域按山地标准采集；②部分区域影像模糊导致高程点遗漏 |
| **检查方法** | 建立1km格网统计高程点数量；FME PythonCaller自定义密度检查 |
| **修改方法** | 在密度不足区域补采高程点（间隔100m）；引用DEM辅助内插 |
| **预防措施** | 采集前设定密度指标并实时统计；使用FME自动化密度检查工作流 |

### 缺陷5：要素遗漏（重要地物缺失）

| 项目 | 内容 |
|------|------|
| **缺陷描述** | 某图幅内一条四级公路及一座小桥被遗漏，属于重要要素遗漏 |
| **产生原因** | ①影像分辨率不足导致细小地物无法辨识；②作业员疏忽漏绘 |
| **检查方法** | 与最新影像叠加目视检查；与上一版本数据对比变更检测 |
| **修改方法** | 参考更高分辨率影像或野外补测，补充遗漏要素 |
| **预防措施** | 采用多源影像交叉验证；作业完成后必须进行影像叠加自查 |

### 缺陷6：符号化错误

| 项目 | 内容 |
|------|------|
| **缺陷描述** | 1:2000图中独立树符号使用了1:500的符号规格，符号偏大 |
| **产生原因** | ①符号库文件未按比例尺切换；②引用了错误版本的符号库 |
| **检查方法** | 对照GB/T 20257图式逐要素目视检查；符号尺寸自动量测对比 |
| **修改方法** | 加载正确比例尺的符号库文件，重新符号化 |
| **预防措施** | 项目启动前统一符号库版本并锁定；建立符号库版本管理制度 |

### 缺陷7：图廓整饰不规范

| 项目 | 内容 |
|------|------|
| **缺陷描述** | 图廓内比例尺、坐标系信息缺失，接合表内容不完整 |
| **产生原因** | ①图廓模板版本过旧；②批量出图时图廓变量未正确替换 |
| **检查方法** | 对照图廓整饰模板逐项目视检查；CASS图廓检查功能 |
| **修改方法** | 使用CASS标准图廓功能重新生成图廓，确认所有字段正确填充 |
| **预防措施** | 使用项目统一的图廓模板；出图前增加图廓字段自动检查流程 |

### 缺陷8：元数据不完整

| 项目 | 内容 |
|------|------|
| **缺陷描述** | 元数据中数据源信息、坐标系统描述、数据精度等必填项为空 |
| **产生原因** | ①元数据采集与数据生产脱节；②元数据编辑工具操作不便 |
| **检查方法** | ArcGIS元数据验证功能；FME XMLValidator按Schema校验 |
| **修改方法** | 在ArcGIS元数据编辑器中补填缺失项；或通过Python脚本批量写入 |
| **预防措施** | 数据生产过程中同步填写元数据；建立元数据自动提取脚本 |

### 缺陷9：坐标系统错误（用了WGS84而非CGCS2000）

| 项目 | 内容 |
|------|------|
| **缺陷描述** | 数据坐标系定义为WGS84 UTM 49N，应使用CGCS2000 3度带39带 |
| **产生原因** | ①GPS原始数据采用WGS84未转换；②数据导入时系统默认WGS84 |
| **检查方法** | ArcGIS坐标系属性查看；FME CoordinateSystemValidator；对比中央经线 |
| **修改方法** | ArcGIS Project工具将WGS84转为CGCS2000（注意WGS84与CGCS2000差异约0.1mm，通常可直接定义投影） |
| **预防措施** | 项目技术设计中明确坐标系要求；数据入库前强制坐标系校验 |

### 缺陷10：图纸间叠盖处理错误

| 项目 | 内容 |
|------|------|
| **缺陷描述** | 相邻图幅接边处房屋面重叠5cm，水系线偏移8cm |
| **产生原因** | ①各图幅独立作业，接边未统一处理；②接边容差设置不合理 |
| **检查方法** | CASS接边检查功能；ArcGIS EdgeMatch工具；FME接边验证 |
| **修改方法** | 以一幅为基准，相邻幅调整接边要素坐标，偏差<0.2mm图上距离可强制吻合 |
| **预防措施** | 接边时指定接边优先级（通常东/南幅为准）；作业完成后立即执行接边检查 |

## 13.G 新型成果质检专项

### G.1 倾斜摄影模型质检SOP

**适用标准**：GB/T 39610-2020、GB/T 24356-2023

| 检查项 | 检查方法 | 合格标准 | 工具 |
|--------|---------|---------|------|
| 影像质量 | 目视+自动 | 无明显模糊、曝光异常 | ContextCapture质量报告 |
| 航向重叠度 | 自动 | ≥60%（山地≥70%） | Pix4D/CC重叠度统计 |
| 旁向重叠度 | 自动 | ≥30% | Pix4D/CC重叠度统计 |
| 旋偏角 | 自动 | ≤1像元（最大≤1.5像元） | 航摄质量检查软件 |
| 模型完整性 | 目视 | 无裂缝、漏洞、拉伸 | 三维浏览软件漫游检查 |
| 纹理质量 | 目视 | 无明显拉伸、扭曲、模糊 | 三维浏览软件局部放大 |
| 模型精度 | 野外检测 | 平面≤3倍中误差，高程≤3倍中误差 | RTK检测点对比 |

**操作步骤**：

1. 导入空三成果，检查相机参数和POS精度
2. 生成重叠度统计图，标注不满足要求的区域
3. 生成三维模型，进行全区域漫游目视检查
4. 选取20+检测点，RTK实测坐标与模型量测坐标对比
5. 输出精度统计报告（中误差、最大误差、粗差率）

### G.2 实景三维模型质检SOP

**适用标准**：GB/T 24356-2023、CH/T 9015-2012

| 检查项 | 检查方法 | 合格标准 | 工具 |
|--------|---------|---------|------|
| 模型几何精度 | 野外检测 | 平面≤0.5m，高程≤0.5m（1:1000） | RTK+模型量测对比 |
| 模型细节层次(LOD) | 目视 | 符合设计要求的LOD级别 | 三维浏览器 |
| 纹理分辨率 | 自动 | 像素地面分辨率≤0.1m | 元数据检查 |
| 模型拓扑 | 自动 | 无自相交、法线反转 | MeshLab/FME检查 |
| 模型接缝 | 目视 | 相邻模型无可见缝隙 | 三维浏览器 |
| 空间参考系 | 自动 | CGCS2000，高程85基准 | 坐标系属性验证 |

### G.3 点云数据质检SOP

**适用标准**：GB/T 24356-2023

| 检查项 | 检查方法 | 合格标准 | 工具 |
|--------|---------|---------|------|
| 点云密度 | 自动统计 | 地面点≥4点/m²（1:1000） | CloudCompare/FME |
| 点云精度 | 野外检测 | 平面≤0.3m，高程≤0.2m | RTK检测点对比 |
| 点云分类正确率 | 抽样目视 | 地面点分类正确率≥95% | CloudCompare/GlobalMapper |
| 点云完整性 | 目视+自动 | 无大面积漏洞 | FME点密度栅格化 |
| 噪声点比例 | 自动 | ≤1% | FME统计离群点 |
| 点云配准精度 | 自动 | 检查扫描站间配准误差 | 扫描仪配套软件 |

**操作步骤**：

1. 加载点云数据，生成密度栅格图，标注密度不足区域
2. 统计总点数、各类别点数及占比
3. 选取地面点抽样区域（每km²取5个1m×1m窗口），目视检查分类正确性
4. 选取15+检测点，RTK实测坐标与点云插值坐标对比
5. 使用统计法检测噪声点（邻域距离法，阈值3倍标准差）
