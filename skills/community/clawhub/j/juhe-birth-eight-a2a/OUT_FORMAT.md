# 生辰宝典输出格式

支付成功并拿到接口 JSON 后渲染。**仅使用接口返回字段，禁止编造内容。**

## 数据路径

业务数据取自响应体的 `result` 或 `data` 字段（优先 `result`，不存在则用 `data`）。外层 `reason`、`error_code`、`code`、`message` 不展示给用户。

下文统一记为 `payload.*`。

---

## 输出结构（固定）

```markdown
# 🧧 生 辰 档 案

---

## 📋 信息总览

{信息总览表}

---

## 🗓️ 历法对照

{历法对照表}

---

## 🐾 生肖与干支

{生肖与干支表}

> 节气：{节气展示}

---

## 🔮 八字 · 五行

{八字五行表}

{日主所缺表}

---

> 📌 档案生成日期：{当前日期}

---

_以上内容仅供娱乐参考，不具备科学依据。_
```

---

## 渲染模板

### 信息总览

```markdown
## 📋 信息总览

| 类别 | 详情 |
|:---:|:---|
| **公历日期** | {cYear}年 {cMonth}月 {cDay}日 |
| **出生时辰** | {hour 展示} |
| **农历日期** | {gzYear}年 {ImonthCn} {IDayCn} |
| **星期** | {ncWeek} |
| **生肖** | {AnimalEmoji} {Animal} |
| **星座** | {AstroEmoji} {astro} |
| **日主五行** | {ShuEmoji} {eightAll.shu} |
| **五行所缺** | {LoseEmojiLine} |
```

- `{hour 展示}`：取自用户本次查询的 `hour` 参数，格式如「13时」；用户以时辰称谓查询时可保留称谓（如「午时」）。
- `{LoseEmojiLine}`：将 `fiveAll.lose` 逐字拆分，每字加五行 emoji，以 ` · ` 连接，如 `木火` → `🌳 木 · 🔥 火`；`lose` 为空时整行跳过。

### 历法对照

```markdown
## 🗓️ 历法对照

| 公历 | 农历 |
|:---:|:---:|
| {cYear}年 {cMonth}月 {cDay}日 | {gzYear}年 {ImonthCn} {IDayCn} |
| {ncWeek} | {闰月展示} |
```

- `{闰月展示}`：`isLeap` 为 `true` 时展示「闰月」，否则展示「—」。

### 生肖与干支

```markdown
## 🐾 生肖与干支

| 属相 | 纪年 | 纪月 | 纪日 |
|:---:|:---:|:---:|:---:|
| {AnimalEmoji} {Animal} | {gzYear} | {gzMonth} | {gzDay} |
```

### 节气

紧跟「生肖与干支」表格后以引用块展示：

```markdown
> 节气：{Term}
```

| 条件 | `{节气展示}` |
|------|-------------|
| `isTerm` 为 `true` 且 `Term` 非空 | `{Term}` 原文 |
| 其他 | `无` |

### 八字 · 五行

```markdown
## 🔮 八字 · 五行

| 柱位 | 年柱 | 月柱 | 日柱 | 时柱 |
|:---:|:---:|:---:|:---:|:---:|
| **干支** | {eightAll.eight[0]} | {eightAll.eight[1]} | {eightAll.eight[2]} | {eightAll.eight[3]} |
| **五行** | {fiveAll.five[0]} | {fiveAll.five[1]} | {fiveAll.five[2]} | {fiveAll.five[3]} |

| 日主 | 所缺 |
|:---:|:---:|
| {ShuEmoji} {eightAll.shu} | {LoseEmojiLine} |
```

- 「五行所缺」行在 `fiveAll.lose` 为空时，「所缺」列展示「—」，不删行。

### 档案生成日期

```markdown
> 📌 档案生成日期：{YYYY}年{M}月{D}日
```

取渲染时的当前本地日期，非接口字段。

---

## 字段映射

| 展示项 | JSON 字段 | 说明 |
|--------|-----------|------|
| 公历年月日 | `payload.cYear` / `cMonth` / `cDay` | 数字间加空格，如 `2021年 1月 15日` |
| 农历中文 | `payload.ImonthCn` + `IDayCn` | 如「腊月 初三」 |
| 农历干支年 | `payload.gzYear` | 如「庚子」，用于农历日期前缀 |
| 星期 | `payload.ncWeek` | 如「星期五」 |
| 生肖 | `payload.Animal` | 如「鼠」 |
| 星座 | `payload.astro` | 如「魔羯座」 |
| 是否闰月 | `payload.isLeap` | 布尔 |
| 年/月/日柱 | `payload.gzYear` / `gzMonth` / `gzDay` | |
| 节气 | `payload.isTerm` / `Term` | |
| 八字四柱 | `payload.eightAll.eight[]` | 年、月、日、时 |
| 日主五行 | `payload.eightAll.shu` | 如「水」 |
| 四柱五行 | `payload.fiveAll.five[]` | 与八字四柱一一对应 |
| 五行缺失 | `payload.fiveAll.lose` | 如「木火」 |

## 渲染规则

1. **固定版式**：严格按「信息总览 → 历法对照 → 生肖与干支 → 八字·五行 → 档案生成日期 → 免责」顺序输出；模块间用 `---` 分隔。
2. **表格对齐**：类别列居中（`:---:`），详情列左对齐（`:---`）。
3. **空值跳过**：字段缺失、空字符串、`null` 时跳过对应行；「五行所缺」为空时信息总览中该行跳过，八字表「所缺」列展示「—」。
4. **数组字段**：`eightAll.eight`、`fiveAll.five` 固定按年、月、日、时顺序；长度不足时仅渲染已有列，不臆造。
5. **文本清洗**：去除 HTML/script 标签；保留中文标点与换行。
6. **禁止臆造**：不生成「宜忌」「流年」「大运」等接口未返回的内容。
7. **页脚免责**（固定追加）：

```markdown
---

_以上内容仅供娱乐参考，不具备科学依据。_
```
