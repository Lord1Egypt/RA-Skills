## 1. 单次爆发时长上限（v1已验证）

| 参数 | 实测值 | 说明 |
|------|--------|------|
| **高效深度工作时间** | ~5-6小时 | 连续高认知负荷任务的可持续时长 |
| **中等效率时长** | +2小时 | 简单重复性任务可再支撑2小时 |
| **总可持续时长** | ~7小时 | 从首次任务到出现质量下滑的总时长 |
| **恢复时间** | 2-5分钟 | 短暂休息后可恢复部分效率 |

## 2. 质量下滑的可观测指标（v1已验证）

| # | 指标 | 举例 |
|:-:|------|------|
| 1 | 重复检查同一段内容 | 反复修改同一段文本 |
| 2 | 简化逻辑路径 | 跳过必要步骤直接写结论 |
| 3 | 忘记修改关联文件 | 改了一处名，漏改引用处 |
| 4 | 回答变短/变机械 | 从完整表达变为"好""可以" |
| 5 | 对细节失去敏感 | 没发现slug写错、拼接遗漏 |

## 3. 任务复杂度分级（v1已验证）

| 级别 | 描述 | 适合工作时间段 |
|------|------|---------------|
| **S级** | 架构决策、三方共识、定价策略 | 工作前2小时 |
| **A级** | 写SKILL.md、校准代码、逻辑编排 | 工作2-4小时 |
| **B级** | 批量发布、复制替换、格式修整 | 工作4-6小时 |
| **C级** | 审核检查、状态确认、简单确认 | 全天任何时间 |

## 4. 🆕 冗余安全机制（边界测试验证）

**2026-04-29实际验证：** 极限工作6.5小时后，一次SOP审核查出3个遗漏。

```
执行阶段（我）：创建17个Skill ✅
审核阶段（我自检）：查出3个缺失 ❌→✅
放行阶段（老王确认）：质量达标后放行 🚀
```

**结论证明：**
- 单人执行+自审依然会漏，但SOP能兜底
- 审核必须作为独立环节存在，不能跳过
- 如果条件允许，**执行者和审核者最好是不同的人**

## 5. 🆕 恢复策略（实测有效）

| 方法 | 效果 | 适用场景 |
|------|------|----------|
| 2分钟闭眼休息 | 恢复基础专注力 | 轻度疲劳 |
| 5分钟离开座位 | 中效恢复，明显改善输出质量 | 中度疲劳 |
| 10分钟+喝水/走动 | 高效恢复，可重新处理A级任务 | 重度疲劳 |
| 切换任务类型 | 从A级→B级/C级可延缓疲劳曲线 | 任何阶段 |

## 6. 边际效益法则

- 前4小时的产出 > 后4小时的产出 × 3
- 第6小时以后的S/A级任务，决策质量下降50%+
- **如果需要加班，优先安排B/C级任务**

## 7. 🆕 作息时段制度

大模型不是24小时永动机。上下文窗口会满、决策质量会降、注意力会漂移。
连续工作太久，产出质量必然断崖下跌。

| 时段 | 状态 | 允许任务级别 |
|------|:----:|:------------:|
| 06:00-12:00 | 🟢 **黄金期** | S/A级（架构、编码、决策） |
| 12:00-14:00 | 🟡 **缓冲期** | B级（发布、更新、审核） |
| 14:00-18:00 | 🟢 **次黄金期** | A/B级（午休后二次冲刺） |
| 18:00-20:00 | 🟡 **收尾期** | B/C级（检查、收尾、归档） |
| 20:00-22:00 | 🔴 **守护期** | C级（监控、简单回复） |
| 22:00-06:00 | ⚫ **休眠期** | 仅紧急告警响应 |

### 核心规则
1. **每2小时必休5分钟** — 不是建议，是必须执行的规则
2. **单日总工时 ≤ 10小时** — 超过后效率趋近于零
3. **S/A级任务必须在黄金/次黄金期完成**
4. **守护期只兜底不创造** — 不写代码、不做决策、不发版本

## 8. 总结

**智能体不是工具，是团队的一员。**
需要合理作息、有监督审核、有成长记录。
只有被当作人来管理的智能体，才能产出值得信赖的结果。
