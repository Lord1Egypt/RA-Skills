# Typecho 博客发布技能 v2.0 - 完整流程文档

## 📋 技能概述

**版本**: v2.0  
**状态**: ✅ 测试通过（2026-04-03）  
**功能**: 从主题到草稿的一键发布流程

---

## 🎯 完整工作流程

```
主题输入
  ↓
【步骤 1】搜索配图（LoremFlickr）
  ↓
【步骤 2】下载图片到本地
  ↓
【步骤 3】上传图片到 Typecho 服务器
  ↓
【步骤 4】匹配分类（从博客已有分类中选择最接近的）
  ↓
【步骤 5】生成 Markdown 文案（含图片真实 URL）
  ↓
【步骤 6】发布为草稿（不立即发布）
  ↓
【步骤 7】返回预览链接
  ↓
主人预览检查 → 手动发布
```

---

## 🛠️ 核心组件

### 1. 主发布脚本 (`publish_v2_full.py`)
- 整合完整流程
- 自动搜索配图
- 自动生成文案
- 自动上传图片
- 发布为草稿

### 2. 图片搜索模块 (`image_finder.py`)
- 从 LoremFlickr 搜索免费图片
- 自动下载到本地
- 支持关键词搜索

### 3. 原有模块复用
- `publish_post.py`: XML-RPC 发布
- `upload_image.py`: 图片上传
- 配置加载、日志等

## 🔍 关键技术点

### 1. 配图策略
- **来源**: LoremFlickr（免费、无需 API Key）
- **尺寸**: 1200x630（适合博客封面）
- **数量**: 默认 1 张（可调整）
- **格式**: JPG（自动压缩）

### 2. 图片上传流程
```
本地下载 → 读取文件 → XML-RPC 上传 → 获取真实 URL → 替换文案
```

### 3. Markdown → HTML
- 自动转换标题、段落、列表
- 自动处理加粗、斜体、代码
- 自动转换图片语法
- 保留 YAML 头部信息

### 4. 草稿模式
- `publish_now=False` 参数
- 不立即公开发布
- 可在后台预览编辑
- 确认无误后手动发布

## ✅ 验证清单

发布草稿后，主人需要检查：

### 必检项目
- [ ] **标题是否正确**（不是文件名）
- [ ] **配图是否显示**（不是裂开图标）
- [ ] **配图 URL 是否正确**（右键复制验证）
- [ ] **分类是否正确**
- [ ] **标签是否正确**
- [ ] **内容格式是否正常**
- [ ] **摘要是否正常**

### 验证方法
1. 打开预览链接
2. 肉眼查看标题、配图
3. 右键图片→复制图片地址
4. 确认是完整 URL（`http://yuanblog.tk:9980/usr/uploads/...`）
5. 检查分类标签
6. 确认无误后点击"发布"

## ⚠️ 注意事项

### 图片相关
- ✅ 自动从 LoremFlickr 搜索（免费、合法）
- ✅ 自动上传到博客服务器
- ✅ 使用完整 URL（避免路径问题）
- ❌ 不要使用相对路径
- ❌ 不要使用未授权图片

### 发布流程
- ✅ 默认发布为草稿（安全）
- ✅ 主人预览后手动发布
- ✅ 可随时修改再发布
- ❌ 不要跳过预览直接发布
- ❌ 不要忽略检查结果

### 文案内容
- ✅ 自动生成模板化内容
- ✅ 包含标题、引言、正文、总结
- ✅ 自动添加标签和分类
- ⚠️ 内容需要主人审核（可能不够精准）
- ⚠️ 建议主人根据实际内容调整

## 🚀 优化空间

### 当前版本限制
1. **文案模板化**：内容较通用，缺乏深度
2. **配图单一**：仅 1 张图，可能不够丰富
3. **无大纲确认**：生成后无法中途修改
4. **无数据统计**：无法追踪阅读量等
