# v1.9.0: README Formatter, Repo Init, Dev Guide overhaul

**Date:** 2026-03-11
**Author:** cc-mini
**Branch:** cc-mini/repo-init-tool

## What changed

### New tool: wip-readme-format (tool #13)

Auto-generates READMEs following the WIP Computer standard. Key design decision: generates separate section files instead of one monolithic README.

Section files: `README-init-badges.md`, `README-init-title.md`, `README-init-teach.md`, `README-init-features.md`, `README-init-coverage.md`, `README-init-more-info.md`, `README-init-license.md`, `README-init-technical.md`.

Modes: default (generate staging files), `--deploy` (assemble + move into place), `--dry-run` (preview), `--check` (validate existing README).

Toolbox mode aggregates sub-tool interfaces for badges, generates interface coverage table using SKILL.md `name:` frontmatter.

### New tool: wip-repo-init (tool #12)

Scaffolds the standard `ai/` directory in any repo. Dogfooded on the toolbox itself. Parker sorted files into the new structure, then we filled in real product docs (roadmap, readme-first, plan stubs).

### Dev Guide overhaul

Both guides now cross-reference each other at the top. Both explicitly document the RELEASE-NOTES-v{version}.md workflow. This was missing: the other CC session read the Dev Guide and still didn't know to write release notes on the branch.

### SKILL.md name fixes

All 8 existing tool SKILL.md files had their `name:` frontmatter fixed. Interface coverage table now shows "Release Pipeline" instead of "wip-release", "Identity File Protection" instead of "wip-file-guard", etc.

### Other fixes

- Interface coverage table: "OpenClaw" column renamed to "OC Plugin"
- License Guard row corrected (was claiming Module + CC Hook, only has CLI)
- Skill deployment added to wip-install (deploys to `~/.openclaw/skills/<tool>/`)
- Amalgamated interface system notes into one reference document
- Plan stubs created for all roadmap items

## Key decisions

- Section-based README formatter (Parker's design). Edit any section independently, deploy assembles them. Same pattern as release notes.
- Release notes on the branch is now documented as mandatory in both Dev Guides.
- Tagline renders as a paragraph, not an h2.
