## 一、开平仓信号

### 基本语法
```mql
条件, 信号;
```
**重要**：信号必须和条件一起使用，不能单独写信号！

### 信号类型表

| 信号 | 含义 | 使用场景 |
|------|------|----------|
| BK | 买开 | 做多开仓 |
| SK | 卖开 | 做空开仓 |
| BP | 买平 | 平空仓 |
| SP | 卖平 | 平多仓 |
| BPK | 反手买 | 平空仓+等量买开（空转多） |
| SPK | 反手卖 | 平多仓+等量卖开（多转空） |

### 详细说明

#### BK - 买开
```mql
// 价格上穿10日均线，买开
CROSS(CLOSE, MA(C,10)), BK;
```
- 作用：买入开仓，做多
- 执行后：产生多头持仓（BKVOL > 0）

#### SK - 卖开
```mql
// 价格下穿10日均线，卖开
CROSSDOWN(CLOSE, MA(C,10)), SK;
```
- 作用：卖出开仓，做空
- 执行后：产生空头持仓（SKVOL > 0）

#### BP - 买平
```mql
// 条件满足时，买入平空
条件, BP;
```
- 作用：平掉空头持仓
- 注意：只有在有空头持仓时才能执行

#### SP - 卖平
```mql
// 条件满足时，卖出平多
条件, SP;
```
- 作用：平掉多头持仓
- 注意：只有在有多头持仓时才能执行

#### BPK - 反手买
```mql
// 反手买：平空仓+等量买开
条件, BPK;
```
- 等价于：`BP;` + `BK;`（同一行内）
- 用于：空头持仓时，看多

#### SPK - 反手卖
```mql
// 反手卖：平多仓+等量卖开
条件, SPK;
```
- 等价于：`SP;` + `SK;`（同一行内）
- 用于：多头持仓时，看空

### 使用建议

| 场景 | 推荐信号 |
|------|----------|
| 一直持仓，非多即空 | `BPK` / `SPK` |
| 可能空仓 | `BK` / `SK` + `BP` / `SP` |
| 需要分批建仓 | `BK` + `ADD_LONG` |
| 需要止损/止盈 | `SP` / `BP` |

## 二、加减仓指令

### 基本语法
```mql
条件, 指令(手数);
```
**重要**：指令必须带手数！

### 指令类型表

| 指令 | 含义 | 语法 |
|------|------|------|
| ADD_LONG | 多头加仓 | `条件, ADD_LONG(N);` |
| LOWER_LONG | 多头减仓 | `条件, LOWER_LONG(N);` |
| ADD_SHORT | 空头加仓 | `条件, ADD_SHORT(N);` |
| LOWER_SHORT | 空头减仓 | `条件, LOWER_SHORT(N);` |

### 详细说明

#### ADD_LONG(N) - 多头加仓
```mql
// 盈利2%时，加仓2手
CLOSE > BKPRICE*(1+2/100), ADD_LONG(2);
```
- 作用：在多头持仓基础上增加持仓
- 要求：必须有多头持仓（BKVOL > 0）

#### LOWER_LONG(N) - 多头减仓
```mql
// 亏损1%时，减仓1手
CLOSE < BKPRICE*(1-1/100), LOWER_LONG(1);
```
- 作用：减少多头持仓
- 要求：持仓手数 >= N

#### ADD_SHORT(N) - 空头加仓
```mql
// 盈利时，空头加仓
条件, ADD_SHORT(2);
```
- 作用：在空头持仓基础上增加持仓
- 要求：必须有空头持仓（SKVOL > 0）

#### LOWER_SHORT(N) - 空头减仓
```mql
// 条件满足时，空头减仓
条件, LOWER_SHORT(1);
```
- 作用：减少空头持仓
- 要求：持仓手数 >= N

### 加减仓特点
- **不视为交易结束**：加减仓后仍是同一笔交易
- **必须先开仓**：没有持仓时不能加减仓
- **可以连续出信号**：使用 `TRADE_AGAIN(N)` 允许同一行加减仓连续N次

### TRADE_AGAIN 函数
```mql
TRADE_AGAIN(N);  // 允许同一行加减仓指令连续出现最多N次
```
- 作用：解决盈利后想分批加仓的需求
- 示例：
```mql
TRADE_AGAIN(3);  // 同一行加减仓可连续3次
CLOSE > BKPRICE*1.02, ADD_LONG(BKVOL/2);  // 每次盈利2%加仓一半
```

## 三、信号执行机制

### 执行原则

1. **信号优先级高于指令**
   - BK/SK/BP/SP/BPK/SPK > ADD_LONG/LOWER_LONG/ADD_SHORT/LOWER_SHORT

2. **从开仓到平仓是一个完整交易**
   - 减仓不视为交易结束
   - 必须出现平仓信号（SP/BP）才算结束

3. **开平仓信号一一对应**
   - BK/SK/BPK/SPK 开仓后只能出现加减仓或平仓信号
   - BP/SP 平仓后只能出现开仓信号

4. **K线执行时机**
   - 默认：K线最后一笔价格确定信号，下一根K线开始下单
   - 优化：使用 `SIGCHECK` 可在K线走完前下单

### 执行流程图
```
开仓信号(BK/SK)
     │
     ▼
加减仓指令(ADD/LOWER) ──可重复N次──┐
     │                               │
     │ (不结束交易)                   │
     ▼                               │
平仓信号(SP/BP) ◄───────────────────┘
     │
     ▼
  交易结束
```

## 四、信号过滤机制

> ⚠️ 详细过滤规则见官方文档：https://www.wenhua.com.cn/popwin/guolvmx3.htm

### AUTOFILTER 必要性
```mql
AUTOFILTER;  // 必须写！过滤重复信号
```

### 一开一平过滤
- 开仓后不再出开仓信号
- 平仓后不再出平仓信号
- 同一笔交易内只保留第一个有效开仓信号

### 加减仓过滤
- 同一行加减仓指令一轮只出一次
- 需要先出减仓才能再出同一行加仓
- 使用 `TRADE_AGAIN(N)` 允许连续N次

### 示例：完整的交易模型
```mql
// 基础框架
条件A, BK;           // 开多
条件B, SP;           // 止损/平多
条件C, SP;           // 平多，结束交易

条件A1, SK;          // 开空
条件B1, BP;          // 止损/平空
条件C1, BP;          // 平空，结束交易

AUTOFILTER;          // 必须写！
```

## 常见错误

### ❌ 错误1：信号不带条件
```mql
// 错误
BK;  // 信号不能单独使用！

// 正确
CROSS(CLOSE, MA(C,10)), BK;
```

### ❌ 错误2：加减仓不带手数
```mql
// 错误
ADD_LONG;  // 缺少手数！

// 正确
ADD_LONG(2);  // 加仓2手
```

### ❌ 错误3：忘记写AUTOFILTER
```mql
// 错误
CROSS(CLOSE,MA), BK;
CROSSDOWN(CLOSE,MA), SP;
// 缺少 AUTOFILTER!

// 正确
CROSS(CLOSE,MA), BK;
CROSSDOWN(CLOSE,MA), SP;
AUTOFILTER;
```

### ❌ 错误4：没有持仓时加减仓
```mql
// 假设当前没有持仓
ADD_LONG(1);  // 无效！必须先BK开仓
```

### ❌ 错误5：平仓信号写在开仓前面
```mql
// 错误写法
SP;     // 平仓写在前面
BK;     // 开仓写在后面

// 正确写法
BK;     // 开仓
SP;     // 平仓
```
