# M0 任务域协调域范本

## M0-01 任务域选择与组合

### 输入示例
用户任务：分析用户行为数据并生成报告

### 处理过程
1. 任务需求分析：需要数据分析和报告生成
2. 可用任务域扫描：数据分析域、报告生成域
3. 任务域选择：选择数据分析域和报告生成域
4. 任务域组合：数据分析域 → 报告生成域

### 输出示例
```markdown
## 任务域选择报告

### 任务需求分析
- **任务目标**：分析用户行为数据并生成报告
- **任务范围**：数据采集、数据清洗、数据分析、报告生成
- **任务复杂度**：中等

### 可用任务域
| 任务域 | 功能 | 适用性 |
|--------|------|--------|
| 数据分析域 | 数据统计分析 | 高 |
| 报告生成域 | 生成可读报告 | 高 |

### 任务域选择
- **选择结果**：数据分析域、报告生成域
- **选择理由**：覆盖任务全部需求

### 任务域组合
- **组合方案**：数据分析域 → 报告生成域
- **依赖关系**：报告生成域依赖数据分析域的结果
```

## M0-02 执行顺序确定

### 输入示例
任务域：数据分析域、报告生成域

### 处理过程
1. 任务域依赖关系分析：报告生成域依赖数据分析域
2. 执行顺序方案：数据分析域优先
3. 关键路径识别：数据分析 → 报告生成

### 输出示例
```markdown
## 执行顺序图

### 任务域依赖关系
| 任务域 | 依赖任务域 | 依赖类型 |
|--------|------------|----------|
| 数据分析域 | 无 | 无 |
| 报告生成域 | 数据分析域 | 结果依赖 |

### 执行顺序方案
```
数据分析域 → 报告生成域
```

### 关键路径
- **路径**：数据分析域 → 报告生成域
- **关键节点**：数据分析域（决定整体进度）
- **并行可能**：无（严格依赖）
```

## M0-03 结果整合与优化

### 输入示例
各任务域执行结果：
- 数据分析域：用户活跃度80%、行为模式A、转化率5%
- 报告生成域：用户行为分析报告

### 处理过程
1. 各任务域执行结果汇总：收集两个任务域的结果
2. 整合方案设计：将分析结果整合到报告中
3. 优化建议：优化报告结构和可读性

### 输出示例
```markdown
## 整合报告

### 各任务域执行结果
| 任务域 | 输出 | 质量 | 状态 |
|--------|------|------|------|
| 数据分析域 | 用户活跃度80%、行为模式A、转化率5% | 高 | 完成 |
| 报告生成域 | 用户行为分析报告 | 高 | 完成 |

### 整合方案
- **整合内容**：将数据分析结果整合到报告中
- **整合格式**：结构化数据表格 + 可视化图表
- **整合效果**：报告包含完整分析结果

### 优化建议
- **建议1**：增加数据可视化图表
- **建议2**：优化报告章节结构
- **建议3**：增加结论和建议章节
```

## M0-04 深化路由

### 输入示例
深化需求：需要深入分析用户画像

### 处理过程
1. 深化需求分析：需要用户画像分析
2. 任务类型识别：数据分析类任务
3. 深化路由方案：路由到数据分析域
4. 目标方法论域确定：数据分析域

### 输出示例
```markdown
## 深化路由报告

### 深化需求分析
- **深化目标**：深入分析用户画像
- **深化范围**：用户特征、行为偏好、消费习惯
- **深化深度**：详细分析

### 任务类型识别
- **任务类型**：数据分析类
- **任务复杂度**：中等
- **任务依赖**：需要用户行为数据

### 深化路由方案
- **路由目标**：数据分析域
- **路由原因**：数据分析域具备用户画像分析能力
- **路由预期**：生成详细用户画像报告

### 目标方法论域
- **域名称**：数据分析域
- **域功能**：用户画像分析
- **域能力**：数据统计、特征提取、画像生成
```
