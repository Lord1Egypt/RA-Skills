## Description: <br>
Generates open-source project README and changelog content from project details, update notes, or commit-style text. <br>

This skill is ready for commercial/non-commercial use. <br>

## Publisher: <br>
[moodsir](https://clawhub.ai/user/moodsir) <br>

### License/Terms of Use: <br>
MIT-0 <br>


## Use Case: <br>
Developers use this skill to draft standard README.md and CHANGELOG.md content for software projects, including feature lists, usage notes, categorized changes, and semantic version suggestions. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Generated README or changelog content may be inaccurate, incomplete, or misleading for the target repository. <br>
Mitigation: Review generated documentation before saving, committing, or publishing it. <br>
Risk: Documentation includes package-style installation examples even though security evidence describes this as a lightweight local Python text generator. <br>
Mitigation: Run the artifact as `python app.py` unless a separately packaged CLI is verified. <br>
Risk: Broad trigger phrases in platform examples may invoke the skill unintentionally. <br>
Mitigation: Use more specific trigger phrases when adapting the publishing examples. <br>
