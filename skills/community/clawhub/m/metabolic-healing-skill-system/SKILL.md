# 代谢慢病"非药而愈"十大功能集群技能体系

## 核心理念

每个能力单元是一个**可独立执行、可自由组合**的原子任务。AI拿到输入就能执行，输出可被任意其他单元消费。

代谢慢病的"非药而愈"不是替代医疗，而是通过**精准评估→营养干预→运动处方→行为改变→持续监测**的闭环，逆转代谢紊乱的根本原因。本技能体系覆盖从个体健康评估到群体健康管理的全链路。

## ⚠️ 领域边界声明

**本技能体系是健康管理辅助工具，不构成医疗诊断、处方或治疗方案。**

1. 本技能提供的所有评估、干预建议均为**参考性质**，不能替代执业医师的临床诊断和处方
2. 涉及用药调整（停药、减药、换药）的任何建议，**必须由执业医师确认后方可执行**
3. 当触发C6-23医疗转诊红线时，**必须立即就医**，不得依赖本技能的任何输出延误治疗
4. 出现急性症状（胸痛、意识障碍、严重低血糖、酮酸中毒迹象等），**立即拨打120或前往急诊**
5. 本技能的安全审核（C6集群）是**辅助筛查**，不能替代医疗安全评估的全部专业判断
6. 孕妇、儿童、老年人及多病共存患者的干预方案，**必须经相关专科医师审核**

## 能力单元Schema

每个单元包含5个字段：

| 字段 | 说明 |
|------|------|
| **输入** | 启动所需的数据/上下文，标注必选/可选 |
| **输出** | 产物格式与颗粒度 |
| **依赖** | 前置必须完成的单元ID，无依赖=入口单元 |
| **AI自治度** | ⬛全自动 / 🟨半自动(需人工确认) / ⬜辅助(人工主导) |
| **组合接口** | 输出可被哪些单元消费（→单元ID列表） |

## 执行规则

1. **必读加载**：首次使用本技能时，必须用Read工具读取 `references/index-and-pipelines.md` 获取完整单元索引和Pipeline编排，这是执行所有后续操作的前提
2. **按需加载**：识别集群后，用Read工具读取对应参考文件获取完整单元规格
3. **依赖检查**：执行前确认依赖单元的输出已就绪，缺失则先执行依赖
4. **自由组合**：无依赖冲突的单元可并行；有依赖的按序执行
5. **跨集群流转**：一个单元的输出可作为另一集群单元的输入，按组合接口串联
6. **安全优先**：涉及健康干预的单元，必须先经过C6安全审核集群的风险评估后方可执行
7. **转诊优先**：当C6-23判定触发转诊红线时，暂停所有非药干预流程，优先启动医疗转诊Pipeline(P25)
