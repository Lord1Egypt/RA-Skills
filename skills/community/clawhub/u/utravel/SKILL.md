# UTravel 技能

## 技能概述

UTravel 是一个面向普通游客的智能旅行规划工具。它能够根据用户指定的目的地（精确到市）、旅行天数、个人偏好（如网红打卡、悠闲程度等）生成定制化行程，并智能推荐每日住宿位置。同时，它会记录用户类型（成人/学生）、旅行偏好和住宿偏好，以便提供更精准的服务。

---

## 触发方式

用户可以通过以下方式调用本技能（自然语言交互）：

- “帮我规划一次去成都的旅行，玩3天。”
- “我想去北京玩2.5天，我是学生，喜欢历史文化，住宿要地铁附近。”
- “计划去杭州，4天，带父母（老年人），不要太累。”

技能会自动解析用户输入，提取关键参数，若信息不全则主动询问。

---

## 输入参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `destination` | string | ✅ | 旅行目的地，需精确到市级行政区，如“北京市”、“成都市” |
| `duration` | number | ✅ | 旅行天数，支持整数或小数（如 3.5 天表示三天半） |
| `user_type` | string | ❌ | 用户类型：`adult`（成年人，默认）或 `student`（学生） |
| `user_age_group` | string | ❌ | 年龄段：`young`（年轻人）、`middle`（中年人）、`elderly`（老年人），用于推荐住宿便利性 |
| `travel_style` | string | ❌ | 旅行风格：`intensive`（紧凑打卡）、`balanced`（均衡惬意，默认）、`leisurely`（悠闲慢游） |
| `preferences` | array | ❌ | 兴趣偏好列表，可选值：`历史文化`、`自然风光`、`美食探店`、`网红打卡`、`购物`、`亲子` 等 |
| `must_visit` | array | ❌ | 用户明确指定的必打卡景点或网红项目（如“故宫”、“洪崖洞”） |
| `accommodation_preference` | string | ❌ | 住宿位置偏好：`metro`（地铁附近，默认）、`scenic`（景区附近）、`quiet`（安静区域）、`central`（市中心） |
| `accommodation_budget` | string | ❌ | 住宿预算：`economy`（经济型）、`mid-range`（中档，默认）、`luxury`（高档） |
| `special_requirements` | array | ❌ | 特殊需求，如 `轮椅通道`、`家庭房`、`宠物友好` 等 |

## 核心功能与工作流程

### 1. 信息收集与确认
- 若用户未提供 `destination` 或 `duration`，技能会主动询问。
- 询问用户是否有旅游偏好（兴趣类型），是否要打卡网红项目，以及希望行程紧凑还是惬意。
- 询问用户是否为学生（以便提供学生票价），以及同行人是否有老年人（影响住宿安排）。
- 询问住宿偏好（地铁附近或景区附近等）。

### 2. 景点筛选与标注
- 根据 `destination` 和 `preferences` 调用景点数据库，提取符合要求的景点列表。
- 对每个景点标注：
  - **是否必玩**（根据 `must_visit` 或热度自动标记）
  - **是否需要提前预约**（及预约链接）
  - **成人票价** 和 **学生参考价**（若 `user_type=student` 则重点显示学生价）

### 3. 行程生成
- 根据 `duration` 和 `travel_style` 生成每日行程（上午、下午、晚间），确保节奏符合选择（紧凑/均衡/悠闲）。
- 行程中会包含餐饮推荐和当日注意事项（如闭馆时间、交通提醒）。
- 所有景点会按地理位置优化顺序，减少往返时间。

### 4. 住宿智能推荐
- 分析每天 **晚间最后一个景点** 和 **次日上午第一个景点** 的位置。
- 计算两者之间的几何中心，并筛选周边酒店。
- 若用户为 **老年人**（`user_age_group=elderly`），优先推荐位于同一区域、步行可达的酒店，避免频繁更换。
- 若无特殊需求，默认推荐 **地铁站附近 500 米内** 的酒店。
- 输出酒店名称、地址、距离各景点的距离、评分、价格区间、配套设施及推荐理由。

### 5. 记录用户画像
- 每次规划完成后，将本次输入的 `user_type`、`travel_style`、`preferences`、`accommodation_preference` 等信息存入用户档案，以便后续推荐更个性化。

## 输出示例（对话式）

**用户**：帮我规划一次去西安的旅行，4天，我是学生，喜欢历史古迹，想打卡兵马俑和城墙，住宿要经济型，最好在地铁旁。

**UTravel**：
好的，已收到您的需求：

目的地：西安市

天数：4天

用户类型：学生

偏好：历史文化

必打卡：兵马俑、西安城墙

住宿：经济型，地铁附近

我正在为您生成行程...

【Day 1】
上午：西安城墙（成人票54元，学生票27元，需现场购票）
下午：钟楼、鼓楼（联票成人50元，学生25元，无需预约）
晚上：回民街（美食探店，免费）
住宿推荐：西安湘子门国际青年旅舍（地铁2号线永宁门站旁，经济型，评分4.5，距城墙步行5分钟）

【Day 2】
上午：陕西历史博物馆（免费但需提前7天预约，务必抢票）
下午：大雁塔（成人票40元，学生票20元，无需预约）
晚上：大唐不夜城（免费，网红打卡）
住宿推荐：同一家酒店（不换宿，方便）

【Day 3】
全天：秦始皇兵马俑博物馆（成人票120元，学生票60元，建议提前3天预约）
晚上：返回市区，自由活动
住宿推荐：仍住同一家酒店（交通便利）

【Day 4】
上午：小雁塔（免费，人少清静）
下午：整理行李，返程

注意事项：

兵马俑距离市区较远，建议早出发，可乘地铁+公交。

陕博预约非常抢手，请定好闹钟。

已将您的偏好记录在案，下次可直接说“再规划一次”快速生成。

## 注意事项

- 本技能依赖准确的景点数据库，若目的地信息不全，会提示用户补充或建议调整。
- 门票价格仅供参考，实际价格以景区当日公布为准。
- 预约信息会尽量提供官方渠道，但请用户自行核实。
- 对于老年人，住宿建议会优先考虑同一酒店连住，避免搬动。
- 所有行程安排均会考虑景点开放时间，避免安排已关闭的景点。
