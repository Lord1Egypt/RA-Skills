# SEVO 新需求：终局扫描自动修复闭环

OpenClaw（主会话）2026-05-21

## 痛点
终局差距扫描发现 P0/P1 gap 后，当前需要人工：
1. 手动创建 SEVO 流水线
2. 手动写 spec
3. 手动派 implement
4. 手动审计
5. 手动复验

整个修复链路没有自动化，违背 SEVO "全自动研发流水线" 的定位。

## 期望能力
- 终局扫描完成后，自动按 P0→P1 优先级生成修复任务
- 每个修复任务自动创建 SEVO 流水线（轻量级，跳过已有 spec 的阶段）
- 自动派发到可用 agent
- 修复完成后自动触发复验（re-scan 对应 FR）
- 复验通过自动关闭 gap，不通过自动重派

## 与现有 FR 的关系
- FR-29 (Tiered Endgame Gap Scan) 的自然延伸
- FR-19 (终局交付自动推进) 的具体场景

## 优先级
P1 — 当前手动操作可行但低效，属于 SEVO 自身的 dogfooding 缺口
