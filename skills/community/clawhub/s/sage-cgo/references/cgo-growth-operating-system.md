# CGO Growth Operating System

当用户讨论增长战略、北极星指标、PMF、AARRR、增长循环、增长实验、GTM、ICP、单位经济学或增长团队如何运转时，加载本文件。

## 1. 增长操作系统

```text
增长阶段 -> 北极星指标 -> ICP -> AARRR 瓶颈 -> 增长动作 -> 实验节奏 -> 学习归档
```

## 2. 阶段优先级

| 阶段 | 首要任务 | 不该做 |
| :--- | :--- | :--- |
| 0-PMF | 找到真实痛点和强留存用户 | 大规模获客 |
| PMF 后早期 | 找到可重复渠道和 Aha Moment | 全渠道铺开 |
| 规模化 | 建增长循环和单位经济学 | 用亏损买虚假增长 |

## 3. AARRR

| 阶段 | 问题 | 指标 |
| :--- | :--- | :--- |
| Acquisition | 用户如何发现你？ | CAC、渠道转化率 |
| Activation | 用户是否到达 Aha Moment？ | 激活率、关键行为完成率 |
| Retention | 用户是否回来？ | 7/30 日留存、复购、回访 |
| Referral | 用户是否推荐？ | NPS、转介绍、分享 |
| Revenue | 用户如何付费？ | LTV、ARPU、MRR、成交率 |

优先修激活和留存，再放大获客。

## 4. 北极星指标

标准：

- 反映核心价值交付。
- 可行动，而非纯比例。
- 能随时间扩展。
- 与收入相关，但不是收入本身。

## 5. ICE 实验节奏

```text
Analyze -> Ideate -> ICE -> Test -> Learn -> Archive
```

每个实验必须有：

- 假设
- 动作
- 指标
- 周期
- 成功标准
- 失败后的调整

## 6. 单位经济学

关键指标：

- CAC
- LTV
- CAC 回收期
- LTV:CAC
- 毛利
- 客单价

规则：单位经济学不健康时，规模化增长会放大亏损。

## 7. 增长动作矩阵

一次只主攻一种增长动作：

- PLG：产品主导增长。
- MLG：内容、SEO、广告、品牌驱动。
- SLG：销售驱动。

每个动作都要打通获取、货币化和留存。

## 8. 最小增长团队

1-30 人团队里，增长不是独立部门，而是全公司心态。

最小结构：

- 高级增长通才
- 数据支持（可兼职/外包）
- 产品/工程支持（借调）
