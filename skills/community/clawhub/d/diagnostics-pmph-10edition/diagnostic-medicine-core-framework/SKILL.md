# 诊断学核心框架与学习方法

## 何时使用
- 医学生开始接触临床实践
- 需系统掌握诊断基本技能而非深入疾病机制
- 面对复杂或非典型表现需启动整体性临床思维
- 病史采集后需整合多源信息形成初步诊断

## 核心流程

### 1. 掌握诊断五大基础支柱
- **病史采集**：通过结构化问诊获取主诉、现病史、既往史等主观资料
- **常见症状识别**：理解疼痛、心悸、眩晕等主观感受作为早期诊断线索
- **体格检查**：规范运用视、触、叩、听、嗅发现客观体征（如黄疸、啰音）
- **实验室检查**：合理选择血液、体液等标本分析，聚焦临床应用而非技术细节
- **辅助检查**：在问诊与体检基础上，针对性选用影像、内镜、心电图等

### 2. 构建正确临床诊断思维
- **整体观**：整合症状、体征、检查结果，避免仅凭单一指标武断诊断
- **动态发展观**：考虑疾病在不同个体的差异性与演变过程
- **诊断层级追求**：优先争取病理学、细胞学或病原学诊断，不满足于功能或影像诊断
- **循证医学整合**：结合最佳证据、临床经验与患者意愿共同决策

### 3. 采用有效学习方法
- **多元教学结合**：课堂讲授 + 案例讨论 + 模拟训练 + 床旁实践
- **角色扮演与标准化病人**：提升实操体验与沟通能力
- **反复床旁练习**：诊断技能无法仅靠理论掌握，必须在真实场景中积累
- **培养自学能力**：从诊断学起养成终身学习习惯

### 4. 避免常见误区
- 盲目依赖高精尖检查而忽视基本问诊与体检
- 过早陷入专科疾病细节而忽略基础能力构建
- 忽视病史完整性导致漏诊或误诊

> **关键原则**：所有检查必须建立在系统问诊与体格检查基础上，不可本末倒置。
