# A股财报审阅助手（Earnings Reviewer CN）

基于 Anthropic `earnings-analysis` 原版架构本土化改造，适配A股上市公司财报审阅场景。

## Overview

创建专业的**A股财报更新报告**，分析已覆盖上市公司的季度/半年/年度业绩，
遵循主流国内券商研究报告格式（中金、中信、国君风格）。

**报告规格：**
- 长度：6-10页
- 字数：2,500-4,000字
- 表格：1-3个摘要表（关键财务指标对比）
- 图表：6-10张
- 交付时间：财报发布后24-48小时内
- 受众：已了解该公司基本面的专业投资者/研究员
- 重点：超预期/未达预期分析、盈利预测修订、投资逻辑更新

---

## When to Use

**适用场景：**
- "帮我做[公司名]2024年三季报分析"
- "分析[公司名]Q3业绩"
- "这家公司年报发布了，帮我做业绩点评"
- "财报季[公司名]业绩超预期/不及预期怎么看"

**不适用场景：**
- 用户请求"首次覆盖报告"→ 需要独立的深度研究模板
- 公司尚未被覆盖→ 需先建立覆盖体系
- 用户需要"快评/flash note"→ 简化为1页格式

## Process

### 阶段1：数据收集（30-60分钟）

> ⚠️ 重要：训练数据存在时效滞后，必须实时搜索最新财报数据。

**启动前必须按顺序完成以下4步：**

1. **确认今日日期** — 记录当前日期，作为时效性基准
2. **搜索最新财报** — 使用以下方式之一：
   - 搜索关键词："[公司名/股票代码] 最新财报 业绩"
   - 直接访问巨潮资讯（cninfo.com.cn）、上交所（sse.com.cn）、深交所（szse.cn）
   - 使用 Wind/Choice/同花顺 查询最新财报文件
3. **验证财报期间** — 确认是最新报告期（非历史数据）
4. **核对披露日期** — 确认披露日期与财报期一致（如2024Q3对应2024年10月末）

**必须收集的文件清单：**

| 文件类型 | 来源平台 | 强制程度 |
|---------|---------|---------|
| 业绩快报/业绩预告 | 巨潮资讯 / 上交所 / 深交所 | ✅ 必须 |
| 季报/半年报/年报正文 | 巨潮资讯（PDF） | ✅ 必须 |
| 业绩说明会/电话会议纪要 | 公司IR / 各大券商 | ✅ 强烈建议 |
| 投资者关系问答记录 | 深交所互动易 / 上证e互动 | 📌 建议 |
| 一致预期数据（Wind/Choice） | Wind资讯 / Choice金融 | ✅ 必须（用于超预期分析） |
| 前期业绩指引 | 上季度/上年度财报 | ✅ 必须 |

**国内数据源优先级：**
1. 巨潮资讯（cninfo.com.cn）— 官方披露平台，优先级最高
2. Wind 资讯 — 专业数据库，一致预期/财务数据
3. Choice 金融（东方财富）— 替代Wind的中低成本选项
4. 同花顺 iFinD — 财务数据/研报
5. 上证e互动 / 深交所互动易 — 管理层Q&A

---

### 阶段2：业绩分析（2-3小时）

**核心分析维度（对应国内研究员标准）：**

#### 2.1 营收分析
- 总营收 vs 一致预期（量化差异：金额 + 百分比）
- 主营业务收入拆分（按产品线/业务板块/地区）
- 同比/环比增速变化及原因解释
- 核心业务量价分析（量增/价增/量价齐升）

#### 2.2 盈利质量分析
- 毛利率变化（同比/环比，解释驱动因素）
- 期间费用率变化（销售费用率/管理费用率/研发费用率）
- 归母净利润 vs 扣非归母净利润（差异说明）
- 经营性现金流质量（与净利润比较，警示"增收不增现"）
- ROE趋势（杜邦拆解：净利率×资产周转率×杠杆）

#### 2.3 关键运营指标
- 根据行业特性选择核心KPI（如：
  - 制造业：产能利用率、库存周转天数、在手订单
  - 消费品：同店增速、新开门店数、客单价
  - 科技：MAU/DAU、ARPU、研发投入占比
  - 金融：不良率、净息差、规模增速）

#### 2.4 指引与前景
- 公司给出的下期指引（如有）
- 指引变化 vs 前期（上调/下调/维持）
- 管理层在业绩说明会上的关键表态

#### 2.5 超预期/未达预期判断
- 与Wind/Choice一致预期对比
- 量化超预期/未达预期幅度
- 归因分析：为什么结果与预期不同？（主动性因素 vs 外部因素）

---

### 阶段3：投资逻辑更新（1小时）

**对照前期投资逻辑，评估本期财报的影响：**

| 投资逻辑要点 | 本期验证情况 | 影响评估 |
|------------|------------|---------|
| [逻辑1] | 已验证/未验证/部分验证 | 正面/中性/负面 |
| [逻辑2] | ... | ... |

**标记重要变化的红绿灯系统：**
- 🟢 超预期变化（利好投资逻辑）
- 🟡 符合预期（维持原判）
- 🔴 不及预期/风险信号（需重新评估逻辑）

---

### 阶段4：盈利预测修订（1小时）

更新未来2-3年盈利预测，清晰展示修订前后对比：

| 指标 | 修订前预测 | 修订后预测 | 变化幅度 | 修订原因 |
|-----|----------|----------|---------|---------|
| 营业收入（亿元） | | | | |
| 归母净利润（亿元） | | | | |
| EPS（元/股） | | | | |
| PE（当前股价/新EPS） | | | | |

---

### 阶段5：报告生成与质检（30分钟）

**报告结构（6-10页DOCX）：**

```
第1页：摘要页
  - 评级（买入/增持/中性/减持）+ 目标价
  - 关键要点（3-5条，每条不超过30字）
  - 核心数据表（营收/净利润/EPS 实际vs预期vs指引）

第2-3页：业绩详细分析
  - 营收拆分分析
  - 利润率分析
  - 现金流分析

第4页：关键运营指标
  - 行业特性KPI
  - 管理层指引

第5-6页：投资逻辑更新
  - 超预期/未达预期汇总
  - 前期逻辑验证
  - 观点变化说明

第7-8页：盈利预测修订
  - 修订前后对比表
  - 关键假设变化

第9-10页：估值
  - PE/PB/EV-EBITDA历史区间
  - 目标价测算
  - 可比公司估值对比

附录（可选）：完整财务报表摘要
```

**质检清单（交付前必须完成）：**
- [ ] 所有数据标注来源（巨潮/Wind/Choice，含日期）
- [ ] 超预期/未达预期分析引用了一致预期来源及日期
- [ ] 盈利预测修订含修订前后对比及修订原因
- [ ] 投资逻辑更新明确给出观点变化（维持/上调/下调）
- [ ] 财务数据来源为最新报告期（非训练数据中的历史数据）
- [ ] 报告末尾列出完整数据来源清单

## Common Rationalizations（常见借口警示）

| 借口 | 正确做法 |
|-----|---------|
| "训练数据里有这家公司的财报" | 必须实时搜索最新数据，不得使用训练集历史数据 |
| "一致预期不好找，估算一下" | 必须查Wind/Choice实际一致预期，不得估算 |
| "管理层没给明确指引" | 在报告中如实说明"管理层未给出量化指引" |
| "数据有点旧但差不多" | 财报时效性是核心要求，过时数据=错误报告 |

---

## Red Flags（执行异常警告）

- 🚨 财报数据与搜索结果不一致 → 停止，核实数据来源
- 🚨 找不到一致预期数据 → 说明来源缺失，不得捏造
- 🚨 季报披露日期超过90天前 → 可能不是最新报告期，重新确认
- 🚨 营收/利润出现明显异常值 → 核实是否为非经常性损益

---

## Verification（可验证的证据要求）

完成后必须能回答以下问题：
1. 本期财报的具体披露日期是？来源链接是？
2. 一致预期数据来自哪个平台？查询日期？
3. 营收/净利润超预期幅度是多少？计算依据？
4. 本期财报对投资逻辑的影响：维持/强化/削弱？
5. 盈利预测修订了哪些年份的数字？修订幅度？
