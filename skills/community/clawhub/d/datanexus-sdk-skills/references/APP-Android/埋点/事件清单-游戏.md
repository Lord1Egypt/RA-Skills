# Android 游戏 — 必报与选报事件

> 来源：[腾讯广告-游戏行业安卓SDK开发指引](https://docs.qq.com/doc/DU29kS3Jsd05yWUxO)

## 必报事件

| action_type | 中文名 | 说明 |
|------------|--------|------|
| `START_APP` | 启动应用 | 用于广告归因、广告效果优化。**必须手动上报**（冷启+热启）。建议在 init + start 之后立即上报一条，在主 Activity 的 onResume 中再上报一条 |
| `REGISTER` | 注册 | 用于广告效果优化。用户完成账号注册（手机号/微信/QQ/TAPTAP 等）时上报。如需实名认证才能游玩，在实名认证通过后上报 |
| `LOGIN` | 登录 | 用于广告效果优化。用户每次完成账号登录时上报 |
| `PURCHASE` | 付费 | 用于广告归因、广告效果优化。**必须传 `value`（金额，单位：分）**，完整上报所有用户所有支付方式的全部付费行为 |

## 条件必报事件（无该行为可不报）

| action_type | 中文名 | 说明 |
|------------|--------|------|
| `CREATE_ROLE` | 创建角色 | 用于广告归因、广告效果优化。无该行为可不报 |
| `TUTORIAL_FINISH` | 完成新手教程 | 用于广告归因、广告效果优化。无该行为可不报 |
| `VIEW_CONTENT` | 浏览商城/活动 | 用于广告效果优化。无该行为可不报 |
| `UPDATE_LEVEL` | 游戏等级提升 | 用于广告效果优化。无该行为可不报 |

## 选报事件

> 根据业务场景选择上报，上报越完整，广告优化效果越好。

| action_type | 中文名 | 专用方法 |
|------------|--------|---------|
| `BINDACCOUNT` | 绑定社交账号 | `ActionUtils.onBindAccount(type, success)` |
| `QUEST_FINISH` | 完成任务 | `ActionUtils.onQuestFinish(id, type, name, number, desc, success)` |
| `SHARE` | 分享 | `ActionUtils.onShare(channel, success)` |
| `RATE` | 评分 | `ActionUtils.onRateApp(value)` |
| `ADD_TO_CART` | 加入购物车 | `ActionUtils.onAddToCart(type, name, id, number, success)` |
| `CHECKOUT` | 提交订单 | `ActionUtils.onCheckout(type, name, id, number, isVirtual, virtualType, currency, success)` |
| `ADD_PAYMENT_CHANNEL` | 添加支付渠道 | `ActionUtils.onAddPaymentChannel(channel, success)` |

## 专用上报方法（ActionUtils 类）

| 方法 | 对应 action_type |
|------|-----------------|
| `ActionUtils.onRegister(method, success)` | REGISTER |
| `ActionUtils.onLogin(method, success)` | LOGIN |
| `ActionUtils.onPurchase(type, name, id, number, channel, currency, amount, success)` | PURCHASE |
| `ActionUtils.onCreateRole(roleName)` | CREATE_ROLE |
| `ActionUtils.onUpdateLevel(level)` | UPDATE_LEVEL |
| `ActionUtils.onViewContent(type, name, id)` | VIEW_CONTENT |
| `ActionUtils.onShare(channel, success)` | SHARE |

> 建议优先使用专用方法，包含优化广告投放所需参数。
>
> Android 端 `START_APP` 没有专用方法，使用 `GDTAction.logAction("START_APP")` 上报。
