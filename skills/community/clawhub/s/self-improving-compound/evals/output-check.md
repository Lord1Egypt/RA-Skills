# Output Check — Self-Improving Evaluation

## Purpose
Verify that logged learnings are actionable and well-structured.

## Checklist

- [ ] Entry has a unique ID (e.g., COR-20260508-001)
- [ ] Entry is stored in SQLite and visible through `search` or `export`
- [ ] Entry has a Pattern-Key if applicable
- [ ] Entry states what was wrong / missing / improved
- [ ] Entry includes the correct answer or prevention rule
- [ ] Entry is concise and avoids incident-log sprawl
- [ ] No secrets, tokens, or private data leaked

## Threshold
Pass if ≥6/7 checks are true.
