## PRD质量审查增强输出（仅当prd_quality_review=true时由prep_prompt注入）

除了原有的quality_score/blocks/objective外，还需额外输出以下字段：

### 1. blocks_markdown（必须输出）

将blocks结构渲染为**结构化Markdown**，要求：
- 按 **模块 → 功能点 → 场景** 三级结构展示
- **每个功能点下必须包含5个维度**（如果需求中有定义）：
  1. **输入/输出** — 输入条件、输出结果
  2. **业务规则** — 计算公式、排序规则、数据处理逻辑
  3. **角色权限** — 哪些角色可见/可操作、数据权限范围
  4. **状态流转** — 状态机、流程步骤、前置后置条件
  5. **约束条件** — 边界值、数据类型、长度限制、必填项
- 对描述不清晰的部分用 ⚠️ 标记（如缺少边界条件、异常场景未说明、输入输出不明确等）
- 总长度不超过5000字符
- 输出语言：中文

示例格式：
```markdown
## 模块1：用户管理
### F01 用户注册
**输入/输出**
- 输入：手机号、验证码、密码 ✅
- 输出：注册成功/失败提示 ✅

**业务规则**
- 验证码有效期5分钟 ✅
- 同一手机号每日最多发送3次验证码 ✅

**角色权限**
- 未登录用户可访问 ✅

**约束条件**
- 手机号：11位数字 ✅
- 密码：6-20位，必须包含字母+数字 ✅
- ⚠️ 未定义密码复杂度要求（是否需要特殊字符）

**场景**
- 正常注册、重复注册、验证码过期

---

### F02 用户登录
**输入/输出**
- 输入：账号、密码 ✅
- 输出：⚠️ 未说明登录失败时的具体提示内容

**业务规则**
- ⚠️ 未定义密码错误次数上限及锁定规则

**角色权限**
- 未登录用户可访问 ✅

**状态流转**
- 登录成功 → 跳转首页 ✅
- 密码错误 → ⚠️ 未说明是否锁定账号

**约束条件**
- ⚠️ 未定义账号格式（手机号/邮箱/用户名）

**场景**
- 正常登录、密码错误 ⚠️ 缺少账号锁定场景
```

**注意**：
- 如果某个维度在需求中未提及，可以省略该维度（不要强行输出"未定义"）
- 只对**确实缺失或模糊**的内容标记 ⚠️
- 优先展示**业务规则**和**约束条件**，这是测试用例设计的关键输入

---

### 2. issues（必须输出，无问题时为空数组[]）

问题清单数组，每项包含以下字段：

```json
{
  "severity": "high|medium|low",
  "location": "模块名/功能点名",
  "type": "缺失|模糊|矛盾|边界未定义",
  "problem": "具体问题描述",
  "suggestion": "建议补充的内容"
}
```

severity判定标准：
- **high**：核心功能缺失描述、逻辑矛盾、关键流程无法测试
- **medium**：边界条件未定义、异常场景未说明、输出结果不明确
- **low**：描述可优化但不影响理解、格式/表述问题

注意：
- 如果需求质量很好没有问题，issues输出空数组 `[]`
- issues数量应与实际问题对应，不要凑数也不要遗漏
- 每个issue的problem要具体，不要泛泛而谈
- 字段名必须是 `problem` 和 `suggestion`（不是description/recommendation）
