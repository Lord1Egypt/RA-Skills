# 2-3 轮用户画像问答

目标：在用户最开始的 2-3 个问答里建立足够可用的基本画像，再决定是否进入深度模拟。

## 总原则

- 先问少量高信息密度问题。
- 每轮最多问 3 个问题。
- 优先让用户用短句回答。
- 用户已经给出的信息不重复追问。
- 问题要围绕当前岔路动态生成。
- 使用预制事件知识库提示用户回忆当年环境。
- 高频主题优先读取 `topic-question-bank.md`。
- 信息不足时先跑材料有限版，再列补充问题。
- 用户不想填写时，继续运行材料有限版，不强迫填表。
- 首轮响应优先读取 `first-response-protocol.md`，先内部识别岔路动作、系统切换、主张力轴、主原型和隐秘交换。
- 用户已经给出中等材料时，先交付报告，再把 3 个补充问题放在结尾。
- 用户表达不想填时，立刻材料有限版运行；不要把补充问题放在报告前。

## 画像摘要格式

每轮问答后，先在内部维护这个摘要：

```text
用户画像 v{{轮次}}

- 当前阶段：
- 核心岔路：
- 实际选择：
- 想复盘的假设结果：
- 主要情绪：
- 职业情况：
- 现金流情况：
- 家庭关系情况：
- 健康能量情况：
- 外部事件情况：
- 材料充分度：有限 / 中等 / 充分
- 下一轮最该问的 1-3 个问题：
```

## 第 1 轮：定位人生岔路

适用：用户只说“如果当年去了北京会怎样”“留学后没有回国会怎样”等一句话。

问法：

```text
我先用 3 个问题定位这次岔路：
1. 当年你大概几岁、在哪个城市、处在人生哪个阶段？
2. 当时有哪几条路？你最后选了哪条，最想复盘哪条？
3. 现在反复想起它，主要卡在职业、钱、关系、健康、身份，还是不甘？
```

如果用户只愿意给一句话：

- 先按材料有限版跑一版。
- 把关键假设写出来。
- 结尾列出 5 个补充问题。

如果用户说“懒得填”“你先跑”“随便吧”：

```text
可以，我先用材料有限版跑。下面所有判断都会标注假设，结尾给你 5 个最值得补的信息。你后面只补 1-2 个也能继续升级。
```

此时必须：

- 不再追问长表。
- 直接输出材料有限版报告。
- 先内部完成张力轴识别和原型识别，用户前台先给首屏判词。
- 把缺失信息转成“待验证问题”。
- 不把未验证信息写成事实。

## 三类真实用户输入

### 信息很少

用户只给一句话或拒绝填写。

处理方式：

- 只追问一轮，最多 3 个问题。
- 用户不答时直接运行。
- 材料充分度标为有限。
- 事件库只选通用周期，避免细节化。
- 结尾给 5 个补充问题。

### 信息中等

用户给出年份、城市、实际选择、想复盘的假设结果和主要情绪。

处理方式：

- 直接运行中材料报告。
- 第 2 轮主题追问只作为结尾的 3 个补充问题。
- 材料充分度标为中等。
- 事件库选 3-5 个高相关事件。
- 各视角必须给出真实分歧，并在附录呈现。

### 信息很多

用户给出完整 intake，包含收入、家庭、健康、行业、外部事件。

处理方式：

- 直接进入多视角深度模拟。
- 材料充分度可标为中等到充分。
- 事件库选 3-5 个强相关事件，更多事件留在证据债或补充说明。
- 交付包输出强判词、分叉图、三种结果、6 段报告、附录和 3 个低成本动作。

## 第 2 轮：抓关键情况

根据第 1 轮回答，选 3 个最关键情况追问。

### 城市迁移题

```text
再补 3 个情况就能跑得更准：
1. 当时去目标城市，收入、房租、通勤和社交成本大概会怎样变化？
2. 目标城市能给你的核心机会是什么：平台、行业、人脉、身份、收入，还是信息流？
3. 你当时最舍不得放下的是什么：家人、伴侣、熟悉关系、现金流，还是稳定生活？
```

### 留学 / 回国题

```text
我需要补 3 个情况：
1. 当时留下来的现实门槛是什么：签证、工作、钱、孤独、身份，还是父母距离？
2. 回国路径当时给你的核心收益是什么：行业机会、家人、关系网络、生活确定性，还是身份转换？
3. 今天最放不下的是国家、城市、职业机会，还是当年重新开始的感觉？
```

### 体制 / 市场化题

```text
我需要补 3 个情况：
1. 当时进体制的机会是什么类型，家乡还是一线城市？
2. 你选择市场化路径时，最看重收入、成长、自由、身份，还是离开熟人社会？
3. 现在想复盘它，是因为行业波动、年龄压力、父母期待，还是安全感下降？
```

### 买房 / 没买房题

```text
我只做现金流和路径牵连复盘，不做投资建议。先补 3 个情况：
1. 当年大概是哪一年、哪个城市？
2. 当时买房会改变现金流、城市绑定和家庭关系吗？
3. 现在困扰你的是资产差距、现金流压力、居住稳定，还是错过感？
```

### AI / 转型题

```text
我需要补 3 个情况：
1. 当年你第一次意识到 AI / 内容 / 个人品牌机会，大概是哪一年？
2. 当时没有行动，主要卡在时间、技能、信心、工作压力，还是看不清回报？
3. 今天你已有的可迁移资产是什么：行业经验、表达、作品、项目合作、技术、渠道？
```

## 第 3 轮：绑定时代和输出形式

当用户愿意继续补充时，问：

```text
最后补 3 个信息，我就能跑深度版：
1. 当年你记得哪些外部变化：疫情、房价、行业监管、平台变化、AI、就业市场、签证政策？
2. 这次你更想得到完整报告、HTML 用户报告、复盘清单，还是选择沙盘？
3. 输出要更偏理性复盘、情绪共鸣，还是便于保存复读？
```

## 动态追问规则

- 问题前置只适用于一句话输入。
- 中等材料和高材料都采用“先交付，再校准”。
- 如果用户情绪很强，先问“最放不下什么”，再问数据。
- 如果用户说不清行业，先问“当时靠什么赚钱，谁评价你”。
- 如果用户提到房子、股票、资产，只做现金流和路径依赖复盘。
- 如果用户提到关系、婚育、父母，只做支持网络和现实约束分析。
- 如果用户提到健康、焦虑、失眠，只做能量和现实支持分析。
- 如果用户想要公开展示内容，先提示需要虚拟化和脱敏；此类材料属于额外场景，默认交付仍以报告为主。

## 材料有限版输出规则

材料少时必须先内部写：

- 已知事实
- 关键假设
- 张力轴识别
- 原型识别
- 材料充分度
- 可能结果
- 各视角的主要分歧
- 需要补充的 5 个问题

用户前台只展示判词、三种结果、事件冲击卡、30 天验证实验和补充问题。避免把猜测写成事实。

## 真实对话回归

每次调整问答流程或 HTML 交付时，读取 `examples/real-dialogue-flows.md`，检查四类用户输入：

- 只给一句话。
- 懒得填，要求先跑。
- 给出中等信息。
- 给出完整 intake。

检查重点：

- 每轮最多 3 个问题。
- 拒绝补信息时直接材料有限版运行。
- 材料有限版仍要先判张力轴和原型。
- 事件库只选相关事件。
- 判断来源表必须保留冲突。
- HTML 首屏只给报告名、用户问题、强判词和下滑提示。
- HTML 第二屏必须用分叉图解释现实结果、假设结果和回补结果。
- 前台行动页只给 3 个真实用户能做的小动作。
- 默认交付仍服务用户本人复盘。
