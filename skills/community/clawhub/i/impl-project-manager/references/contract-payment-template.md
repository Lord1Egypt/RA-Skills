# 合同与回款模板

## 合同生命周期

```
NEGOTIATING (谈判中) → SIGNED (已签约) → EFFECTIVE (生效)
→ FULFILLING (履约中) → ACCEPTED (验收) → SETTLED (结算) → CLOSED (关闭)
```

## 回款节点默认模板

基于 IT/软件交付里程碑：

| 里程碑 | 默认回款比例 | 说明 |
|--------|-------------|------|
| 需求确认 | 30% | 需求基线确认后 |
| 试运行完成 | 40% | 系统试运行通过 |
| 验收通过 | 25% | 正式验收签字 |
| 质保完成 | 5% | 质保期满 |

**计算规则：** 计划回款金额 = 合同金额 × 回款比例

## 预警规则

| 预警级别 | 触发条件 | 标记 | 建议动作 |
|----------|----------|------|----------|
| 即将到期 | 距计划回款日期 ≤7 天且未回款 | 🟡 | 提前准备回款材料，联系客户方 |
| 已逾期 | 超过计划回款日期且未回款 | 🔴 | 立即跟进，升级处理 |

## 回款状态

| 状态 | 条件 |
|------|------|
| 未回款 | 实际回款金额 = 0 |
| 部分回款 | 0 < 实际回款金额 < 计划回款金额 |
| 已回款 | 实际回款金额 ≥ 计划回款金额 |
