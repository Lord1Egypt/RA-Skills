# 胡田-SPAC资本路径规划 Skill

## 技能定位

这是「胡田-OPC导师-股权合作机制」的**姊妹篇**：

| 维度 | 姊妹Skill（OPC股权合作机制） | 本Skill（SPAC资本路径规划） |
|------|-----------------------------|-----------------------------|
| **解决的问题** | 现在的合作与分钱 | 未来的成长与资本化 |
| **时间跨度** | 当下项目周期 | 3-5年中长期规划 |
| **核心关注** | 分钱机制、治理结构、SPV内部运营 | 资本化路径、上市准备、估值提升 |
| **适用阶段** | SPV虚拟共同体阶段 | 实体公司→上市阶段 |

> **关系说明**：姊妹Skill跑通单个项目，本Skill跑通集群资本化。两者无缝衔接，构成OPC体系从"第一桶金"到"资本证券化"的完整闭环。

## 核心适用场景

- 已有1-2个成功SPV项目，考虑规模化复制
- 有多个项目形成集群，考虑装入母公司统一运作
- 对接国企/产业资本，打造OPC项目集群
- 规划3-5年香港上市（18C章/SPAC/传统主板）
- 需要将SPV贡献权重折算为工商股权

### 阶段1：SPV虚拟共同体（当前）

| 特征维度 | 具体描述 |
|----------|----------|
| **组织形态** | 无工商注册，纯协议约束 |
| **合作模式** | 技能交换、贡献权重分配 |
| **权益载体** | 贡献权重（非股权） |
| **灵活性** | 高度灵活，随时加入退出 |
| **合规要求** | 低，主要依赖自愿遵守 |

**触发升级条件**：
- 连续3个月盈利
- 年营收突破500万
- 核心3人愿意全职投入

### 阶段2：实体公司（双轨制）

| 特征维度 | 具体描述 |
|----------|----------|
| **组织形态** | 成立有限合伙企业或有限责任公司 |
| **工商注册** | 正式法律实体，可开具发票 |
| **权益折算** | SPV贡献权重折算为工商股权 |
| **治理结构** | 股东会+董事会/执行事务合伙人 |
| **资本运作** | 可正式引入外部投资 |

**触发升级条件**：
- 5个以上成熟SPV项目
- 年营收>5000万
- 研发投入占比≥15%
- 具备独立运营能力

### 阶段3：SPAC上市

**可选路径**：

| 路径 | 适用场景 | 优缺点 |
|------|----------|--------|
| **路径A：18C章特专科技公司** | 无盈利但有潜力的科技公司 | ✅ 门槛低 ✅ 政策支持 ❌ 流动性较弱 |
| **路径B：SPAC并购上市** | 快速上市、减少审批风险 | ✅ 周期短 ✅ 确定性高 ❌ 稀释较大 |
| **路径C：传统主板上市** | 盈利达标、规模较大 | ✅ 流动性好 ✅ 品牌效应 ❌ 门槛高周期长 |

## 与SPV体系的关系

- **不是替代，而是延伸和升级**
- 姊妹Skill「OPC股权合作机制」聚焦SPV层面的合作机制
- 本Skill聚焦从项目孵化到资本证券化的成长路径
- 完整链条：**贡献权重 → 工商股权 → 上市股份**

```
贡献权重（SPV层）
     ↓ 折算
工商股权（实体公司层）
     ↓ 稀释/增发
上市股份（SPAC层）
```


## 常见问题

**Q: SPV和SPAC有什么区别？**
A: SPV是"特殊目的载体"，当前阶段的虚拟合作共同体；SPAC是"特殊目的收购公司"，是成熟的上市公司形态。两者是不同阶段的不同工具。

**Q: 为什么不需要"Spark"这个中间阶段？**
A: 经过优化设计，直接从"实体公司"过渡到"SPAC上市"更高效，路径更清晰，避免不必要的中间层架构。

**Q: 从SPV到实体公司的权重折算怎么算？**
A: 参考`references/08-与SPV体系对接.md`，按历史贡献、持续投入、未来价值三维度综合评估。

**Q: 什么时候开始准备上市？**
A: 建议在实体公司成立后就启动上市规划，倒推3-4年做准备。参考`references/07-时间节点倒推表.md`。
