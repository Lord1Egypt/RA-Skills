## Description: <br>
AI-powered cloud service mapping and provider recommendation across Azure, AWS, and Google Cloud. <br>

This skill is ready for commercial/non-commercial use. <br>

## Use Case: <br>
Cloud architects, DevOps teams, and organizations evaluating migrations or multi-cloud strategies use this skill to analyze workload requirements and compare AWS, Azure, and Google Cloud service recommendations. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Workload descriptions may contain confidential infrastructure details, compliance-sensitive information, or migration plans. <br>
Mitigation: Avoid sending secrets, exact internal architecture, compliance-sensitive details, or confidential migration plans unless the API operator is trusted. <br>
Risk: Session and user identifiers may expose user or tenant information. <br>
Mitigation: Use pseudonymous session and user identifiers where possible. <br>
Risk: Cloud recommendations can be incomplete or unsuitable for a specific organization's constraints. <br>
Mitigation: Have cloud, security, and compliance owners review recommendations before using them for architecture or migration decisions. <br>
