# 输出格式 (v2.0.1)

## 报告结构

报告按以下顺序输出：

```
📋 执行信息（元信息区块，必填）
📰 今日快讯（速览区）
⚡ 今日信号（详细分析）
🚫 避坑提醒
🔍 值得留意
🧊 冰山一角
📡 待观察
🔇 无信号（仅当上述全部为空时）
```

---

## 📋 执行信息（必填）

报告最顶部的元信息区块，让读者了解报告的执行环境。

格式：
```
---
📋 执行信息
- 执行时间：{YYYY-MM-DD HH:mm} {时区}
- 执行模型：{provider/model-id}
- 执行方式：{子代理隔离 / 主会话直接执行}
- 搜索轮次：web_search × {N}, web_fetch × {N}
- 信号筛选：候选 {N} 条 → 通过门控 {N} 条
---
```

**字段说明**：
- **执行时间**：报告生成的时间，使用用户本地时区
- **执行模型**：实际生成报告的模型全名（如 `xiaomi-coding/mimo-v2.5-pro`）
- **执行方式**：`子代理隔离` 或 `主会话直接执行`（降级时）
- **搜索轮次**：实际执行的 web_search 和 web_fetch 次数
- **信号筛选**：候选信息总数和通过门控的数量

**获取方式**：
- 执行模型：子代理通过 `session_status` 获取当前模型；主会话直接读取 agent 配置
- 执行方式：根据 Step 2 是否 spawn 了子代理判断
- 搜索轮次：执行过程中计数
- 信号筛选：Step 5 质量门控前后分别计数

---

## 📰 今日快讯（必填，3-8 条）

报告最顶部的速览区，让读者 10 秒掌握今天发生了什么。

每条一句话，不展开分析，不标注置信度。包含来源 URL。

```
📰 今日快讯
- xAI 解散，并入 SpaceX → 来源URL
- Anthropic 估值超 OpenAI → 来源URL
- DeepSeek 获国家大基金首轮融资 → 来源URL
```

即使没有达到⚡/🔍标准的信号，快讯区也要输出（汇总当天所有值得知道的事）。

---

## ⚡ 今日信号（0-3 条）

判定标准：同时满足"需要行动"+"三天后仍有意义"+"有具体所以呢"

没有同时满足三条的 = 留空。空是正常结果。

**每条必须包含至少一个来源 URL。**

格式：
```
⚡ [事件] 一句话事实陈述
  所以呢：跟用户的关系
  行动：具体可执行的建议（P0/P1/P2）
  时效：🔴24h紧急/🟡7天中期/🟢30天长期
  来源：URL（必须）
  置信度：高/中
```

> 低置信度信息不能进入此区。

---

## 🚫 避坑提醒（0-2 条）

判定标准：明确告知用户不该做什么，避坑价值大于机会价值

**每条必须包含至少一个来源 URL。**

格式：
```
🚫 [事件] 一句话事实陈述
  避坑原因：为什么不要做
  建议：具体的规避动作
  来源：URL（必须）
  置信度：高/中
```

---

## 🔍 值得留意（3-7 条）

判定标准：满足"具体性门控"，但不需要满足"持久性门控"。跟用户有关联，但可能三天后就不重要了。

**每条必须包含至少一个来源 URL。**

格式：
```
🔍 [简述] — 所以呢：一句话
  来源：URL（必须） | 置信度：高/中
```

---

## 🧊 冰山一角（0-2 条）

判定标准：必须有明确的三步推理链条。缺一步就不报。

推理结构：大家关注 X → X 会导致 Y → Y 对用户意味着 Z

**每条必须包含至少一个来源 URL。**

格式：
```
🧊 [表面] 大家都在关注什么
  [冰山] 真正值得关注的是什么
  [推理] X → Y → Z
  来源：URL（必须）
  置信度：高/中
```

---

## 📡 待观察（0-5 条）

判定标准：置信度低但潜力大的早期信号。连续 2 天有佐证升级为正式信号，7 天无进展移除。

**每条必须包含至少一个来源 URL。**

格式：
```
📡 [事件简述] — 当前佐证：[来源URL]
  后续关注：需要验证什么信息
```

---

## 🔇 无信号输出

当上述五类有效信号均为 0 条时的默认输出。

```
🔇 今日无关键信号。

以下为背景信息供了解：
- 一句话快讯 1 → 来源URL
- 一句话快讯 2 → 来源URL
- 一句话快讯 3 → 来源URL
```

这不是失败，是高质量筛选的结果。

---

## 来源 URL 规则

- **每条信号必须包含至少一个来源 URL**，无 URL 的信号不得输出
- URL 必须是可访问的原始链接（不是搜索引擎结果页）
- 优先使用一手来源 URL（官方博客、GitHub、论文）
- 多个来源用逗号分隔
- 低置信度信号标注"待验证"并附来源 URL
