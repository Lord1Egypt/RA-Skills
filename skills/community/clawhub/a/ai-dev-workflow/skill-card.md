## Description: <br>
AI Dev Workflow provides a standardized AI-assisted programming workflow that turns a rough software idea into structured requirements, a method-call architecture blueprint, and complete project code. <br>

This skill is ready for commercial/non-commercial use. <br>

## Publisher: <br>
[wangjiaocheng](https://clawhub.ai/user/wangjiaocheng) <br>

### License/Terms of Use: <br>
MIT-0 <br>


## Use Case: <br>
Developers and engineers use this skill to guide an AI coding agent through a repeatable three-step process: expand requirements, design cross-layer method-call structure, and generate project code, configuration, and deployment files. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Generated application code may include insecure or unsuitable sharing links, upload flows, notifications, deployment files, or privacy-sensitive behavior. <br>
Mitigation: Review generated app code before real deployment, with extra attention to sharing, uploads, notifications, deployment configuration, and privacy-sensitive features. <br>
Risk: The workflow includes a default example stack that may not match the user's intended implementation. <br>
Mitigation: State the preferred technology stack before code generation when the default example stack is not desired. <br>
