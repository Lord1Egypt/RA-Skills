# Live Host Delivery Guide

A great script fails with poor delivery. Use these rules to deliver Q&A answers that convert.

## Pacing

- Match the room: when comments flood, use short variants; when it's calm, use long variants with demos.
- Never read robotically. Glance at the cue card, then say it in your own rhythm.
- Pause for one beat after a CTA so viewers can act.

## CTA Discipline

- End every answer with a next step: "tap the cart", "use the code", "before the timer".
- Restate price + code every 3–5 minutes for new viewers.
- Tie CTAs to scarcity (stock, timer) without inventing fake urgency.

## Comment Management

- Pin the product and the active code at all times.
- Batch repeat questions: answer "price?" once to the whole room.
- Acknowledge by name when possible — it builds trust and keeps people watching.
- Ignore trolls after one factual reply; don't let conflict derail momentum.

## Demos Beat Description

- Whenever a question is about quality or function, show it live.
- Pre-stage demo props so you can grab them instantly.
- Narrate what the camera sees so clip-watchers later understand.

## Multi-Host Handoffs

- Use a clear handoff line so the stream doesn't go quiet.
- The incoming host restates the offer and code to reset context.
- Keep a shared cue card so answers stay consistent between hosts.

## Compliance On Air

- Stick to substantiated claims; if unsure, don't say it.
- Use the approved wording for guarantees and policies verbatim.
- For sensitive categories, use the safe general line and redirect to support.

## Pre-Stream Checklist

- Cue cards loaded and grouped by theme.
- Product pinned, code live, timer set.
- Demo props staged.
- Returns/shipping policy memorized.
- Handoff lines agreed (if multi-host).
