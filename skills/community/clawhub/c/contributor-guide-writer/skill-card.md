## Description: <br>
Analyze the current project structure, package manager, and build tools to generate a comprehensive, standardized CONTRIBUTING.md file. <br>

This skill is ready for commercial/non-commercial use. <br>

## Publisher: <br>
[sunny0826](https://clawhub.ai/user/sunny0826) <br>

### License/Terms of Use: <br>
MIT-0 <br>


## Use Case: <br>
Developers and open source maintainers use this skill to inspect a repository and draft a tailored CONTRIBUTING.md guide with local setup, development workflow, test, lint, pull request, and code of conduct sections. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: The skill may create or update CONTRIBUTING.md in the workspace. <br>
Mitigation: Review the planned file changes or a diff before accepting them, especially when the repository already has a contributor guide. <br>
Risk: Repository-specific setup, test, or lint commands can be inaccurate if the project structure is ambiguous. <br>
Mitigation: Confirm generated commands against detected project files before publishing the guide. <br>
