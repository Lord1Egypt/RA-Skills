# ziwei_verify 配置说明

> **版本**：1.0  
> **日期**：2026-05-04  
> **适用范围**：`ziwei_verify` 模块的所有配置项

---

## 📋 配置总览

`ziwei_verify` 支持三种配置方式（按优先级从高到低）：

1. **函数参数**（最高优先级） - 每次调用时传入
2. **环境变量**（次优先级） - 进程级别全局配置
3. **配置文件**（最低优先级） - 默认值

**推荐做法**：
- 简单场景：使用环境变量
- 复杂场景：通过 `config.set_config()` 动态调整
- 生产部署：环境变量 + 配置文件结合

---

## 🔧 环境变量配置

在 shell 中设置（`~/.bashrc`、`~/.zshrc` 或 systemd service 文件）：

```bash
# 校正阈值
export ZIWEI_VERIFY_MAX_SHIFTS=2
export ZIWEI_VERIFY_CONFIDENCE_THRESHOLD_AUTO=0.7
export ZIWEI_VERIFY_CONFIDENCE_THRESHOLD_INTERACTIVE=0.3

# 功能开关
export ZIWEI_VERIFY_SIMULATION_MODE=false
export ZIWEI_VERIFY_ENABLE_CACHE=true

# 缓存配置
export ZIWEI_VERIFY_CACHE_TTL=3600

# 日志级别
export ZIWEI_VERIFY_LOG_LEVEL=INFO
```

**生效**：重启 Python 进程或 OpenClaw Gateway。

---

### 环境变量详表

| 变量名 | 类型 | 默认值 | 说明 | 示例 |
|--------|------|--------|------|------|
| `ZIWEI_VERIFY_MAX_SHIFTS` | `int` | `2` | 最大校正偏移（1 或 2 个时辰） | `2` |
| `ZIWEI_VERIFY_CONFIDENCE_THRESHOLD_AUTO` | `float` | `0.7` | 自动校正阈值（≥此值自动接受） | `0.75` |
| `ZIWEI_VERIFY_CONFIDENCE_THRESHOLD_INTERACTIVE` | `float` | `0.3` | 交互模式阈值（≥此值显示候选） | `0.5` |
| `ZIWEI_VERIFY_SIMULATION_MODE` | `bool` | `false` | 模拟模式（不真实计算） | `true` |
| `ZIWEI_VERIFY_ENABLE_CACHE` | `bool` | `true` | 启用结果缓存 | `false` |
| `ZIWEI_VERIFY_CACHE_TTL` | `int` | `3600` | 缓存 TTL（秒） | `1800` |
| `ZIWEI_VERIFY_LOG_LEVEL` | `str` | `INFO` | 日志级别（DEBUG/INFO/WARN/ERROR） | `DEBUG` |
| `ZIWEI_VERIFY_PROMPT_TEMPLATE` | `str` | `default` | 提示词模板名称 | `倪海厦专用` |

## 📁 配置文件（`config.py`）

### 配置类结构

```python
# ziwei_verify/config.py

class Config:
    """配置容器（所有属性可动态修改）"""

    # 校正参数
    max_shifts: int = 2
    confidence_threshold_auto: float = 0.7
    confidence_threshold_interactive: float = 0.3

    # 功能开关
    simulation_mode: bool = False
    enable_cache: bool = True

    # 缓存配置
    cache_ttl_seconds: int = 3600
    cache_maxsize: int = 512

    # 提示词
    prompt_template: str = "default"

    # 日志
    log_level: str = "INFO"

    # 调试
    debug_mode: bool = False
```

### 读取配置

```python
from ziwei_verify.config import get_config

config = get_config()

print(f"最大偏移：{config.max_shifts}")
print(f"自动阈值：{config.confidence_threshold_auto}")
```

---

### 修改配置（运行时）

```python
from ziwei_verify.config import set_config

# 修改单个配置
set_config(max_shifts=1)

# 批量修改
set_config(
    confidence_threshold_auto=0.75,
    confidence_threshold_interactive=0.5,
    enable_cache=False
)
```

**注意**：修改后立即生效，无需重启。

---

### 重置配置

```python
from ziwei_verify.config import reset_config

reset_config()  # 恢复为默认值
```

## 🎛️ 函数参数覆盖

所有核心函数支持参数覆盖配置（优先级最高）：

### `calibrate()` 参数

```python
result = calibrate(
    packet=original,
    birth_dt=birth_dt,
    max_shifts=1,              # 覆盖 config.max_shifts
    interactive=True,          # 覆盖默认 False
    # 注意：confidence_threshold_* 不在此函数，需改 config
)
```

**有效参数**：
- `max_shifts`：`int`，1 或 2
- `interactive`：`bool`

**示例**：

```python
# 临时使用交互模式（不改变全局配置）
result = calibrate(packet, birth_dt, interactive=True)

# 临时限制偏移为 1 时辰
result = calibrate(packet, birth_dt, max_shifts=1)
```

## 🧪 配置测试

### 测试脚本：`test_config.py`

```python
#!/usr/bin/env python3
"""配置加载测试"""

import os
from ziwei_verify.config import get_config, set_config, reset_config

def test_env_vars():
    """测试环境变量覆盖"""
    os.environ["ZIWEI_VERIFY_MAX_SHIFTS"] = "1"
    os.environ["ZIWEI_VERIFY_LOG_LEVEL"] = "DEBUG"

    # 重新加载配置（需重启进程，此处仅演示）
    config = get_config()
    print(f"max_shifts（应=1）: {config.max_shifts}")
    print(f"log_level（应=DEBUG）: {config.log_level}")

def test_runtime_modify():
    """测试运行时修改"""
    config = get_config()
    print(f"修改前 max_shifts: {config.max_shifts}")

    set_config(max_shifts=1, enable_cache=False)
    print(f"修改后 max_shifts: {config.max_shifts}")
    print(f"修改后 enable_cache: {config.enable_cache}")

    reset_config()
    print(f"重置后 max_shifts: {config.max_shifts}")

if __name__ == "__main__":
    test_runtime_modify()
```

**运行**：

```bash
python test_config.py
```

**预期输出**：

```
修改前 max_shifts: 2
修改后 max_shifts: 1
修改后 enable_cache: False
重置后 max_shifts: 2
```

## ⚠️ 配置陷阱与规避

### 陷阱1：环境变量类型错误

**问题**：

```bash
export ZIWEI_VERIFY_MAX_SHIFTS=true  # 字符串 "true"，不是布尔
```

`config.max_shifts` 会读取为字符串 `"true"`，导致类型错误。

**解决**：环境变量始终是字符串，`config.py` 内部会转换：

```python
self.simulation_mode = os.getenv("ZIWEI_VERIFY_SIMULATION_MODE", "false").lower() == "true"
```

---

### 陷阱2：修改配置后未生效

**问题**：调用 `set_config(max_shifts=1)` 后，`calibrate()` 仍使用旧值。

**原因**：`calibrate()` 函数参数未传递 `max_shifts`，读取的是全局 `config`，但 `config` 对象已被修改，应该生效。若不生效，可能是其他地方缓存了旧配置。

**解决**：

```python
# 方式1：直接改 config 对象属性
from ziwei_verify.config import get_config
get_config().max_shifts = 1

# 方式2：传递函数参数（最高优先级）
result = calibrate(packet, birth_dt, max_shifts=1)
```

---

### 陷阱3：缓存配置不生效

**问题**：设置 `enable_cache=False` 后，缓存依然命中。

**原因**：`calibrator.py` 中的 `@lru_cache` 装饰器在模块加载时已绑定，配置修改时缓存未清空。

**解决**：

```python
from ziwei_verify.config import set_config
from ziwei_verify.calibrator import clear_cache

set_config(enable_cache=False)
clear_cache()  # 手动清空已有缓存
```

## 🔐 安全配置建议

### 1. 关闭敏感日志

```bash
export ZIWEI_VERIFY_LOG_LEVEL=WARN  # 不输出 DEBUG 级别的详细数据
```

### 2. 禁用缓存（隐私场景）

```bash
export ZIWEI_VERIFY_ENABLE_CACHE=false
```

### 3. 限制最大偏移（避免过度校正）

```bash
export ZIWEI_VERIFY_MAX_SHIFTS=1  # 最多偏移1个时辰（更保守）
```

## 📈 性能调优配置

| 目标 | 配置项 | 推荐值 |
|------|--------|--------|
| 减少计算量 | `max_shifts` | `1`（而非 2） |

## 🔄 配置热更新

`ziwei_verify` 支持**运行时动态调整**，无需重启：

```python
from ziwei_verify.config import set_config

# 在 Web 服务中根据用户偏好动态调整
@app.route("/api/set_threshold", methods=["POST"])
def set_threshold():
    data = request.json
    set_config(
        confidence_threshold_auto=data["auto"],
        confidence_threshold_interactive=data["interactive"]
    )
    return {"status": "ok", "current": get_config().__dict__}
```

## 📚 配置项详解

### `max_shifts`（最大偏移）

**类型**：`int`  
**默认**：`2`  
**范围**：`1` 或 `2`  
**说明**：

- `1`：生成 ±1 时辰的 2 个候选（共 2 个）
- `2`：生成 ±1、±2 时辰的 4 个候选（共 4 个），加上基准共 5 个

**性能影响**：

| 值 | 候选数 | 预计耗时 |
|----|--------|---------|
| 1 | 2 | ~0.03s |
| 2 | 4 | ~0.05s |

**建议**：
- 普通用户：`2`（更全面）
- 批量处理：`1`（更快）
- 特殊案例：`2`（避免遗漏）

---

### `confidence_threshold_auto`（自动阈值）

**类型**：`float`  
**默认**：`0.7`  
**范围**：`0.0 - 1.0`

**作用**：校正后置信度 ≥ 此值 → 自动接受，无需交互

**调优建议**：

| 场景 | 推荐值 | 理由 |
|------|--------|------|
| 保守派（不轻易校正） | `0.8` | 只有非常确定时才自动接受 |
| 平衡派（默认） | `0.7` | 较好的置信度即接受 |
| 激进派（追求校正） | `0.6` | 较低分数也接受 |

---

### `confidence_threshold_interactive`（交互阈值）

**类型**：`float`  
**默认**：`0.3`  
**范围**：`0.0 - 1.0`

**作用**：最优候选分数 ≥ 此值 → 显示候选列表供用户选择

**调优建议**：

| 场景 | 推荐值 | 说明 |
|------|--------|------|
| 高要求 | `0.5` | 只有较好的候选才展示 |
| 平衡 | `0.3` | 默认，适度展示 |
| 宽松 | `0.2` | 所有候选都展示（包括差的） |

---

### `simulation_mode`（模拟模式）

**类型**：`bool`  
**默认**：`false`

**作用**：
- `true`：使用模拟数据，不调用真实 `ziwei` 技能（用于单元测试）
- `false`：正常调用 `ziwei` 技能

**使用场景**：
- 单元测试：`True`
- 开发调试：`True`（若无 ziwei 环境）
- 生产环境：`False`

---

### `enable_cache`（缓存开关）

**类型**：`bool`  
**默认**：`true`

**缓存内容**：
- 排盘结果（`packet`）
- 候选评分（`match_score`）

**建议**：
- 生产环境：`True`
- 内存紧张：`False`
- 批量一次性处理：`False`（避免缓存污染）
