# 认知复盘引擎（v3.3 Pro 新增）

## 定位

**不评估"候选人说了什么"，评估"候选人如何思考"。**

v3.2 解决"包装过的简历如何穿透审计"，v3.3 解决"面试口述如何拆解真实认知过程"。

输入源优先顺序：飞书妙记转录文本 > 手动面试记录 > 面试官笔记。

---

## 模块 1：矛盾检测引擎（Contradiction Engine）

### 核心洞察

AI 生成内容通常"过于光滑"。真实经历存在模糊性、例外情况、资源限制、失败痕迹。矛盾检测不是"找茬"，是**识别信号衰减**。

### 检测维度

| 维度 | 检测目标 | 示例信号 | 置信度 |
|------|---------|---------|--------|
| **时间矛盾** | 时间线不自洽 | "2023年同时负责两个异地全职项目" | 高 |
| **数据矛盾** | 数字/指标前后冲突 | 先说"团队5人"后说"我带了20人" | 高 |
| **角色矛盾** | 职责描述不一致 | "我主导架构" vs "需要请示技术总监" | 中 |
| **逻辑跳跃** | 结论与前提不匹配 | "因为用了AI，所以效率提升300%"（缺中间链路） | 中 |
| **决策模糊** | 关键决策无权衡过程 | "选了方案A"但没有提方案B/C或取舍理由 | 低-中 |
| **过度光滑** | 全程无失败/挫折描述 | 所有项目都顺利推进，无例外情况 | 低-中 |

### 判定规则

```
矛盾发现 → 分类 → 置信度标注 → 输出
```

- **高置信度矛盾**：直接标注，要求面试官追问
- **中置信度矛盾**：标注"存疑"，建议但不强制追问
- **低置信度矛盾**：仅内部记录，不主动暴露（避免误报消耗面试官信任）

### ⚠️ 防误报铁律

1. **口语转写失真不计为矛盾**：飞书妙记可能断句错误、同音词替换
2. **记忆模糊≠说谎**：候选人记不清具体数字是正常现象
3. **不同粒度的描述≠矛盾**：宏观概述和细节描述可能表面冲突但实质一致
4. **必须标注"可能是转写/记忆误差"**：每条矛盾输出必须带这一免责声明

## 模块 2：认知解析层（Cognitive Parsing Layer）

### 解析维度

| 维度 | 评估内容 | 关键信号 | 反信号 |
|------|---------|---------|--------|
| **问题拆解** | 接到需求后如何结构化 | 先问边界→拆子问题→排序 | 直接跳答案/复述需求 |
| **抽象能力** | 能否从具体案例提炼本质 | "核心矛盾是X""本质上是Y问题" | 只讲细节不提规律 |
| **逻辑一致性** | 前后论述是否自洽 | 论点→论据→结论链条完整 | 前后立场漂移无解释 |
| **真实性纹理** | 是否存在真实细节 | 提到了意外/bug/临时妥协 | 所有描述都是"最佳实践" |
| **不确定性处理** | 面对未知的反应 | 坦诚"这个我不确定，但我的推测是..." | 硬编答案/含糊其辞 |
| **修正能力** | 被challenge后如何调整 | 接受质疑→重新思考→给出新视角 | 防御性反驳/死不承认 |
| **Ownership** | 是否主动推进问题 | "后来我发现还有一个问题..." | 等面试官问才说 |
| **AI协同** | 如何描述AI使用过程 | 提到了prompt迭代/失败尝试/边界判断 | 笼统说"用AI辅助" |

### 评分方法

每个维度不按"1-4分"，而是按**行为频率**标注：

| 标注 | 含义 |
|------|------|
| **持续出现** | 面试中多次（≥3次）表现出该特征 |
| **偶尔出现** | 1-2次，可能是偶发 |
| **未观察到** | 没有足够数据判断 |
| **反向出现** | 表现出相反特征（需标注） |

> 为什么不用1-4分？因为认知过程是**模式**不是**分数**。"这个人遇到challenge总是防御"比"修正能力=2.5分"有价值得多。

## 模块 3：深挖追问 Agent（Deep Drill Agent）

### 核心原则

**不是固定题库，是动态生成。** 基于矛盾引擎和认知解析的发现，实时生成追问。

### 追问触发规则

| 触发条件 | 追问模板 |
|---------|---------|
| 矛盾引擎发现时间矛盾 | "你刚才提到A和B同时发生，具体是怎么协调的？" |
| 决策模糊（无tradeoff） | "当时有没有考虑过其他方案？为什么排除？" |
| 缺乏失败描述 | "这个项目遇到过最大的意外是什么？" |
| 抽象词汇无细节 | "你提到'效率大幅提升'，具体从多少到多少？怎么测的？" |
| 被challenge后防御 | "我理解你的做法，但如果从X角度看，会不会有不同结论？" |
| AI协同描述笼统 | "你能举一个AI帮上忙和一个AI帮不上忙的具体例子吗？" |

### 追问深度评估

| 层级 | 候选人表现 | 评估 |
|------|-----------|------|
| **第1层（表层）** | 复述/美化既有叙述 | 未穿透 |
| **第2层（细节）** | 能补充具体细节和数字 | 初步验证 |
| **第3层（反思）** | 承认不足、复盘决策过程 | ✅ 真实能力暴露 |
| **第4层（重构）** | 推翻原有叙述，给出新视角 | ✅ 高阶认知能力 |

> 三层以后才能暴露真实能力。如果候选人停留在第1-2层，标注"认知深度不足"。

## 模块 4：认知画像层（Cognitive Embedding）

### 输出格式

不输出"综合评分：82"，输出**能力拓扑图**：

```markdown
## 认知画像

| 维度 | 特征 | 证据 |
|------|------|------|
| 决策风格 | 数据驱动（偶尔经验补充） | "先看数据再判断"出现3次 |
| 思维结构 | 框架型（先搭骨架再填充） | 每次回答问题都先说"我分三个方面" |
| AI协同 | 高度增强型（深度迭代prompt） | 描述了3次prompt调优过程 |
| 复杂度承载 | 中高（能处理3层嵌套问题） | 成功拆解了跨部门流程重构 |
| Ownership | 强（主动延伸话题） | 4次主动补充面试官未问到的信息 |
| 风险偏好 | 保守（偏好验证后再推进） | 多次提到"先小范围试点" |
| 修正能力 | 快（被challenge后1轮内调整） | 2次被追问后快速修正观点 |
| 真实性风险 | 低（全程有失败/例外描述） | 主动提到2个项目的教训 |
```

### 画像可信度

| 数据质量 | 画像可信度 | 标注 |
|---------|-----------|------|
| 完整面试转录（>30分钟） | 高 | 可直接用于定级辅助 |
| 面试记录（>15分钟） | 中 | 建议补充更多信息 |
| 简要笔记（<15分钟） | 低 | 仅做参考，不做强推断 |

## 防幻觉铁律（认知复盘专属）

| 规则 | 说明 |
|------|------|
| 不推断未表达的内容 | 候选人没说"我想过方案B"，就不能写"候选人考虑过多种方案" |
| 标注转写质量 | 飞书妙记转录可能有误差，必须在报告开头标注"本分析基于语音转写文本" |
| 矛盾需标注置信度 | 每条矛盾必须带"高/中/低置信度"标签 |
| 不替代面试官判断 | 输出是"建议"不是"结论"，最终判断由面试官做出 |
| 认知模式≠能力定级 | 认知画像是定级的辅助证据，不能替代六维度打分 |

---

## 与 v3.2 的关系

```
v3.2（简历审计 → 六维度打分 → 定级）
  ↑ 补充
v3.3 Pro（面试认知复盘 → 矛盾检测 → 认知画像 → 追问建议）
```

- v3.3 **不替代** v3.2，是**叠加层**
- 输入：同一份面试记录，v3.2 做"交叉验证 + 打分"，v3.3 做"认知过程解析"
- 输出：v3.2 的定级报告 + v3.3 的认知复盘报告
- 定级时：v3.3 的认知画像可作为六维度打分的补充证据
