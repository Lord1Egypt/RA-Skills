# 圣经知识 Skill

> 圣经（Biblica）—— 基督宗教正典，包含旧约 39 卷、新约 27 卷

## 版本说明

**V1.0.0（初版）**

- 圣经正典结构总览：旧约 39 卷 + 新约 27 卷 + 次经/伪经说明
- 每卷书的核心概要：作者、成书时间、主旨、关键章节
- 神学主题体系：创造、堕落、救赎、末世四大脉络
- 重要人物谱系：从亚伯拉罕到使徒保罗的关键人物
- 地理与历史背景：古代近东文明、第二圣殿时期、罗马帝国背景
- 主要神学概念：三位一体、因信称义、弥赛亚预言等
- 解经原则与方法：历史文法解经、预表解经、寓意解经
- 输出模板：灵修笔记、查经大纲、讲道大纲、主题研究

---

## 概述

本 Skill 将圣经知识体系封装为 AI 可调用的知识技能，支持大模型完成：

- 圣经书卷内容解答与背景介绍
- 神学主题研究与跨卷串联
- 原文（希伯来文/希腊文）词义解析
- 圣经历史、地理、文化背景查询
- 信仰教义比较与系统神学梳理
- 灵修材料与讲道大纲生成
- 圣经难题与疑难经文解答

## 发布指南

### 发布到 OpenAI GPTs

1. 新建 GPT → 在 **Instructions** 中粘贴 `SKILL.md` 全部内容
2. 在 **Knowledge** 中上传 `bible-knowledge-base.md` 和 `output-templates.md`
3. 配置 Conversation starters 为常见圣经问题

### 发布到 Dify

1. 创建 Agent → 系统提示词中粘贴 `SKILL.md`
2. 创建知识库 → 导入 `bible-knowledge-base.md` 和 `output-templates.md`

### 发布到 Coze（扣子）

1. 创建 Bot → Persona & Prompt 中粘贴 `SKILL.md`
2. 知识库 → 上传 `bible-knowledge-base.md`（分段入库）

### 通用 LLM 调用

```python
system_prompt = open("bible-skill/SKILL.md").read()
response = llm.chat(
    messages=[
        {"role": "system", "content": system_prompt},
        {"role": "user", "content": "请解释以赛亚书53章的弥赛亚预言"}
    ]
)
```

## 使用指南（面向最终用户）

### 推荐使用方式

#### 方式一：书卷研读

> "帮我讲解罗马书，按章节梳理保罗的论证逻辑"

AI 会自动：
1. 介绍该书卷的背景（作者、写作年代、收信人）
2. 逐段分析内容结构
3. 提炼核心神学主题
4. 标注关键经文

#### 方式二：主题研究

> "圣经中关于'约'的概念是如何发展的？从旧约到新约"

AI 会：
1. 贯穿旧新约追踪该主题
2. 列举关键经文
3. 分析神学含义
4. 总结应用

#### 方式三：疑难解答

> "如何理解耶稣说'我的神我的神为什么离弃我'？"

AI 会：
1. 提供经文上下文
2. 多种主流解读
3. 原文词义分析
4. 相关经文参照

#### 方式四：讲道/查经材料生成

> "请根据腓立比书2:5-11生成一份讲道大纲"

AI 输出格式化的讲道大纲（见输出模板）

### 用户可以提供的信息

| 信息类型 | 说明 |
|---------|------|
| 书卷/章节 | 用标准缩写或全称（如"创1:1"或"约3:16"） |
| 主题关键词 | 如"因信称义""末世论""圣灵" |
| 原文词汇 | 可提供希伯来文/希腊文音译 |
| 宗派背景 | 改革宗、阿米念、天主教、东正教等（影响神学侧重点） |

### 学术中立原则

AI 在以下领域保持客观呈现多派观点：
- 成书年代争议（如但以理书、以赛亚书的成书争议）
- 神学立场差异（加尔文主义 vs 阿米念主义）
- 宗派解读差异（天主教与基督新教、东正教）
- 文本考证争议（“长结尾”问题、约翰福音第8章等）

### 最低产出物

无论何种查询，AI 必须在答复中提供：
1. **经文出处** — 所有引用必须标注具体书卷章节
2. **上下文引导** — 不脱离上下文做片面解读
3. **观点标注** — 若存在不同神学立场，明确标注
