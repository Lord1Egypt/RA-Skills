# Branch H: Qualitative Study — Detailed Rules

This document contains the full Branch H specification for qualitative research, including Thematic Analysis, IPA, Grounded Theory, and Constructivist Grounded Theory subtypes.

---

## Core Qualitative Rules (H1–H9)

### H1: Qualitative Results Full-Scan Rule

Do not stop scanning at Figure 1 or theme framework. Continue through the entire Results/Findings section until Discussion/Limitations/Conclusion.

**Prohibited:** "Themes 3–5 only appear in supplementary appendices" / "current PDF does not include detailed quotations" without full scan.

### H2: Qualitative Theme Heading Detection Rule

Recognize standalone theme labels as explicit headings: `Personalisation and informed choice`, `Simplifying the process`, `Feeling abandoned`, etc.

Output: "Results has no numbered subsections, but is organized by 5 standalone theme titles which should be treated as explicit Results structure."

### H3: Supplementary Appendix Statement Rule

"Detailed quotations available in Supplementary Appendices 2–6" does NOT mean main text lacks narrative or representative quotes. Main text provides summary narrative + selected quotes; appendix provides additional detail.

### H4: Reflexivity Assessment Grading Rule

Three-level assessment:

| Level | Criteria |
|-------|----------|
| Absent | No mention of researcher position/identity/experience/power relations |
| Partial/brief | Mentions researcher position in Limitations or elsewhere, but no standalone statement |
| Explicit and systematic | Standalone reflexivity section per COREQ standards |

Do not label as "absent" if researcher position is mentioned anywhere.

### H5: Intercoder Reliability Epistemology-Sensitive Rule

Do not mechanically demand quantified intercoder reliability. Align expectations with epistemological stance:
- Positivist/content analysis → quantified reliability expected
- Reflexive TA / interpretivist → quantifying agreement may contradict epistemological stance

### H6: Demographic Table Audit Rule (8 items)

1. Per-row counting, no impression-based estimation
2. Each categorical variable counts must sum to N
3. Composite label protection (e.g., "Chinese Eurasian" as independent label, not forcibly categorized)
4. Diagnosis multi-label rule (comorbidity allows occurrence counts > N)
5. Percentage labeling: manual calculations must be tagged `[教学性计算]`
6. Count-percentage mismatch: retain raw count, flag for manual verification
7. Prohibit causal inference from demographic table
8. Downstream Module consistency: use audit-corrected values throughout

### H6a: Diagnosis Counting Rule (5 items)

1. Distinguish participant count (unique N) from diagnosis occurrence count (may exceed N due to comorbidity)
2. Per-row diagnosis label counting
3. Do not exaggerate: "MDD = 16/21 (76.2%)" → "most common", not "almost all"
4. Label occurrence counts with annotations
5. Audit-vs-suggestion priority: row-by-row audit overrides prior suggestions/old values

### H7: Theme Count Inconsistency Rule

When Abstract, Results, and Figure report different theme/subtheme counts → flag as high-priority source inconsistency.

### H8: Module C Recommended Structure

1. Thematic overview (citing Figure 1)
2. Theme 1 definition + representative quotes
3. Theme 2 definition + representative quotes
4. Theme 3–N (cover ALL main-text themes)
5. Cross-theme relationships `[教学性综合]`
6. Discussion transition

**Quote extraction requirement:** Standard mode: ≥1 representative quote per high-order theme. Close-reading: 2–3 quotes per theme with participant ID + subtheme + interpretive note.

### H9: G0 Source Verification Fields

participant N, focus group N, sampling strategy, recruitment source, data collection method, interview duration, analysis approach, epistemological stance, saturation statement, coding process, member checking/focus group validation, number of themes, number of subthemes, theme names, representative quote source (text vs supplement), reflexivity status (H4), intercoder reliability (H5), author-reported limitations, internal inconsistencies (H6–H7), diagnosis occurrence vs participant count (H6a).

## IPA Subtype Rules (H10–H16)

### H10: Subtype Identification

Label as: Branch H — Qualitative Study / IPA subtype.

IPA-specific Study Profile fields: idiographic focus, lived experience, sense-making, double hermeneutic, epistemological stance (Heidegger/critical realism/contextualism), super-ordinate themes, sub-ordinate themes, case-by-case/purposive sample logic, researcher interpretation layer.

### H11: IPA Quality Criteria

IPA-specific dimensions (not generic TA standards): purposive small sample, depth of interview data, idiographic interpretation, participant meaning-making, reflexive role of researcher, member checking, peer review of themes, transparency of theme generation, preservation of participant context.

**Saturation rule:** "Not reported" should be noted but NOT judged as major flaw for IPA (idiographic depth > frequency coverage).

### H12: IPA Findings Structure Rule

Module B for IPA should recognize: "Findings" (not "Results"), super-ordinate themes, sub-ordinate themes, asymmetrical theme structures (not all themes need equal sub-themes), representative quotes, interpretive commentary, links to prior literature/theory, transition to methods/implications/conclusion.

### H13: Interpretive Synthesis Labelling Rule

Cross-theme relationships, bifurcation points, feedback loops, narrative progressions — all must be labeled `[教学性综合 / interpretive synthesis by skill]`, not attributed as author-proposed models.

### H14: Non-Causal Progression Rule

Theme sequences organized as progression (T1→T2→T3) must add: "This is a presentation-friendly narrative progression, not a causal pathway tested or claimed by the authors."

### H15: IPA Quote-Interpretation-Literature Pattern Rule

Classic three-segment structure: Participant quote → IPA interpretation (double hermeneutic) → Theoretical/literature integration. Module C must label each segment separately. High-value phenomenological quotes spanning multiple theme levels must be tagged `[high-value phenomenological quote]`.

### H16: Table 1 Terminology Precision Rule

Do not call thematic framework tables "statistical tables" / "factor matrices" / "correlation matrices." Use: "thematic framework table" / "super-ordinate/sub-ordinate theme table" / "theme overview table."

## Grounded Theory Subtype Rules (H17–H23)

### H17: GT Subtype Identification

Distinguishing features vs IPA/TA: theoretical output (core category + conceptual model), coding process (open→axial→selective), core concern (process/social mechanism/explanatory model), typical terms ("core category" / "emerged" / "process").

### H18: GT Results Heading Hierarchy Rule

Conceptual Model is the upper Results heading; Core Category is an internal subsection within it. Do not list Core Category as the first independent Results subsection.

### H19: GT Hierarchy Distinction Rule

Five levels must be distinguished: Core category → Category → Subcategory → Explanatory factor → Taxonomy frequency count. Do not conflate levels. Frequency n is theme coverage, not effect size.

### H20: GT Evidence Type Refinement Rule

Evidence type must specify: Grounded Theory conceptual-model evidence, based on N participant interviews via systematic coding (open/axial/selective), producing core category → categories/subcategories → properties/dimensions → conditions/factors → explanatory model → conceptual integration.

### H21: GT Coding Quality Distinction Rule

Separate dimensions: quantified intercoder reliability, consensus coding, auditor role, peer debriefing, memoing/audit trail, theoretical sampling, theoretical saturation. Do not collapse to "intercoder reliability not reported."

### H22: GT Saturation Severity Rule

GT typically requires saturation discussion more than IPA/TA. Severity grading: explicit saturation declared (normal), saturation not declared but coding process described + adequate N (Important), neither saturation nor coding process described (Important+), N clearly insufficient (<10 without discussion) (Important+).

### H23: GT Conceptual Model Arrow Interpretation Rule

Arrows in conceptual model figures (e.g., Figure 2) are "conceptual/explanatory relations grounded in patient narratives" — not causal effects, statistical pathways, or mediation models.

## Constructivist Grounded Theory (CGT)

CGT subtype applies when Charmaz (2014) methodology is identified. All GT rules H17–H23 apply, plus CGT-specific epistemological considerations (constructivist, co-constructed meaning, researcher-participant relationship). H6/H6a demographic audit applies with full force.
