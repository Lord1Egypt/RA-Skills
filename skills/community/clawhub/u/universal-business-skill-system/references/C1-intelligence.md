# C1 情报采集与监测 — 详细单元规格

## C1-01 行业市场数据标准化收集

| 字段 | 内容 |
|------|------|
| **输入** | 必选：行业名称、目标市场范围；可选：数据源偏好、时间范围 |
| **输出** | 标准化市场数据报告（市场规模、增长率、份额、趋势），Markdown格式 |
| **依赖** | 无 |
| **AI自治度** | 🟨 半自动（需人工确认数据源和范围） |
| **组合接口** | →C1-07, →C1-12, →C4-01, →C4-05 |

**执行要点**：
1. 基于行业校准确定核心数据维度（如：电商→GMV/客单价/复购率；SaaS→ARR/Churn/NRR）
2. 从公开数据源、行业报告、统计数据采集
3. 数据标准化处理（统一单位、货币、时间口径）
4. 输出结构化报告，标注数据来源和时效

| 字段 | 内容 |
|------|------|
| **输入** | 必选：行业名称、关注地区；可选：特定政策领域 |
| **输出** | 政策法规动态清单（政策名称、发布机构、影响评估、生效日期），Markdown |
| **依赖** | 无 |
| **AI自治度** | 🟨 半自动（需人工确认关注范围） |
| **组合接口** | →C1-08, →C1-15, →C6-10, →C9-19 |

**执行要点**：
1. 基于行业校准确定关键监管机构和政策类别
2. 追踪近期发布和即将生效的政策法规
3. 评估每项政策对业务的直接影响
