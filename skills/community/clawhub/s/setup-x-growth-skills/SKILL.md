# Setup X Growth Skills

Run this once before using the other skills.

## Ask The User

1. What X account are we growing?
2. Who should follow it?
3. What should followers get from it every week?
4. What proof, experience, or projects can the account show?
5. What topics should the account become known for?
6. How often can the user post and reply?
7. What metric matters most right now: followers, replies, leads, profile clicks, newsletter clicks, sales?
8. Where should we save the account notes?

## Create The Growth Brief

Write a concise brief with:

- **Account promise**: one sentence.
- **Target audience**: specific people, not demographics.
- **Retrieval neighborhood**: adjacent accounts, topics, phrases, communities.
- **Content pillars**: 3-5 repeatable themes.
- **Proof bank**: projects, numbers, screenshots, stories, lessons.
- **Cadence**: posts/day and replies/day.
- **Metrics**: what to track weekly.
- **Voice rules**: tone, banned phrases, examples of good posts.

## Save It

If the user has a repo or notes folder, save the brief as `X_GROWTH_CONTEXT.md`. If not, show the brief and tell them to paste it into their agent/project notes.

## Then Recommend

- Use `/x-growth-strategy` for the first plan.
- Use `/x-content-calendar` for the first week.
- Use `/x-post-audit` before publishing important posts.
- Use `/x-analytics-loop` after 7 days.

## Operating Principle

Treat the public X algorithm as a signal map, not a cheat code. Optimize for posts and replies that earn useful positive actions (dwell, replies, reposts, shares, profile clicks, follows) while avoiding negative actions (not interested, mute, block, report, fast skips).
