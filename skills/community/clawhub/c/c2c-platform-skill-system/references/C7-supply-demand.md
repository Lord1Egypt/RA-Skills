## 步骤7.1 服务商管理 (C7-01 to C7-04)

### C7-01 服务商入驻审核

- **输入**: 服务商入驻申请（基本信息/实名认证C6-04/营业执照/行业许可证/保证金缴纳C6-28）、入驻指南标准(C5-03)、条目发布规范(C5-15)、服务商分级标准(C7-02)、审核通过率目标、历史驳回原因分布
- **输出**: 入驻审核结果（通过/驳回+原因/待补充材料），审核评分（资质完整度/行业匹配度/区域供需匹配度），初始等级建议，审核时效SLA（≤2工作日）
- **依赖**: 无
- **AI自治度**: 🟨
- **组合接口**: → C7-02(分级管理初始等级)、C7-03(技能标签初始设定)、C7-19(扶持方案新服务商识别)

### C7-02 服务商分级管理

- **输入**: 服务商绩效数据(C7-04)、信用分(C7-07)、历史订单数据（完成率/好评率/投诉率/完成码确认率）、保证金缴纳等级、平台战略品类需求
- **输出**: 分级标准与结果（等级定义：青铜/白银/黄金/铂金/钻石→权益差异：佣金折扣/流量倾斜/提现速度/专属客服），分级调整规则（升降级条件/周期/申诉）
- **依赖**: 无
- **AI自治度**: 🟨
- **组合接口**: → C7-22(核心服务商运营分级依据)、C7-19(扶持方案分级差异化)、C7-01(入驻审核分级建议)

### C7-03 服务商技能标签管理

- **输入**: 服务商条目数据（品类/服务类型/技能描述）、条目发布规范(C5-15)中的品类分类体系、服务商自选标签、用户评价关键词提取、平台标准技能标签字典（按品类→子品类→技能三级）
- **输出**: 技能标签体系（标准标签字典+服务商标签映射+标签权重），标签管理规则（服务商自选→平台验证→用户反馈修正），供需匹配标签使用规则
- **依赖**: 无
- **AI自治度**: ⬛
- **组合接口**: → C7-11(供需匹配技能维度)、C7-21(能力认证标签验证)、C7-03(搜索排序标签权重)

### C7-04 服务商绩效评估

- **输入**: 订单完成数据（接单率/完成率/完成码确认率）、评价数据（好评率/差评率/C7-09评价体系）、投诉数据(C6-10)、收入数据（易点收入/提现频率）、履约质量(C7-17)、信用分(C7-07)
- **输出**: 绩效评估报告（综合评分→维度评分→排名→趋势→改进建议），评估周期（周/月/季度），绩效等级划分与权益对应
- **依赖**: 无
- **AI自治度**: ⬛
- **组合接口**: → C7-02(分级管理绩效维度)、C7-15(流失预警绩效输入)、C7-19(扶持方案绩效诊断)

## 步骤7.2 买家运营 (C7-05 to C7-06)

### C7-05 买家画像与分层

- **输入**: 用户行为数据（浏览/搜索/下单/支付/评价/复购）、消费数据（易点消费金额/频次/品类偏好/客单价）、3级分销参与度（L1邀请人数/分润收入/邀请活跃度）、信用分(C7-07)、完成码使用习惯
- **输出**: 买家画像（基础属性+消费偏好+行为特征+价值等级+生命周期阶段），分层模型（新客/活跃/忠诚/沉睡/流失→对应运营策略），RFM评分
- **依赖**: 无
- **AI自治度**: ⬛
- **组合接口**: → C7-06(生命周期管理分层输入)、C7-16(流失预警画像维度)、C7-11(供需匹配需求侧画像)

### C7-06 买家生命周期管理

- **输入**: 买家画像与分层(C7-05)、生命周期阶段定义（认知→注册→首单→复购→忠诚→沉睡→流失）、阶段转化率数据、关键行为事件（首次下单/首次评价/首次邀请/首月复购）、易点奖励触达规则
- **输出**: 生命周期管理方案（阶段定义→转化目标→触达策略→易点激励→关键指标），含自动化运营规则（如：注册3天未下单→推送新人券+易点奖励）
- **依赖**: 无
- **AI自治度**: ⬛
- **组合接口**: → C7-16(流失预警生命周期数据)、C7-05(画像更新生命周期行为)

## 步骤7.3 信用体系 (C7-07 to C7-10)

### C7-07 信用分计算与更新

- **输入**: 信用分计算规则(C5-19)、行为数据（履约率/完成码确认率/评价分/投诉率/违规记录/退款率）、时间衰减因子、3级分销行为（邀请质量/分润异常/传销违规指标）、刷单识别结果(C6-14)、虚假交易识别结果(C6-15)
- **输出**: 信用分更新（当前分值→变动值→变动原因→历史趋势），触发式实时更新+定时批量更新双模式，异常变动预警（单次变动>50分触发人工复核）
- **依赖**: 无
- **AI自治度**: ⬛
- **组合接口**: → C7-08(等级划分分值区间)、C7-24(信用修复机制输入)、C7-02(分级管理信用维度)

### C7-08 信用等级划分

- **输入**: 信用分分布数据(C7-07)、等级定义规则(C5-19)（差:350-449/一般:450-549/良好:550-649/优秀:650-799/卓越:800-950）、等级权益差异（提现速度/佣金折扣/流量权重/活动资格/保证金减免）、等级分布健康度目标
- **输出**: 信用等级划分结果（等级分布统计→权益矩阵→等级变更通知），等级调整策略（分布偏移时调整阈值）
- **依赖**: C7-07
- **AI自治度**: 🟨
- **组合接口**: → C7-24(信用修复等级目标)、C7-02(分级管理信用等级映射)

### C7-09 评价体系管理

- **输入**: 评价数据（星级/文字/图片/标签）、评价场景（订单完成后/完成码确认后）、评价维度（服务态度/专业能力/准时性/性价比）、评价防刷机制（限制同一买家对同一服务商评价频率）、易点激励评价规则
- **输出**: 评价体系方案（评价规则→展示逻辑→防刷策略→计算权重→申诉机制），含服务商评价看板与趋势分析
- **依赖**: 无
- **AI自治度**: ⬛
- **组合接口**: → C7-10(差评预警评价数据源)、C7-04(绩效评估评价维度)、C7-07(信用分评价分维度)

### C7-10 差评预警与处理

- **输入**: 评价数据(C7-09)、差评阈值（≤2星/含敏感投诉关键词）、投诉数据(C6-10)、服务商历史差评率趋势、差评响应时效要求（≤4小时首次响应）、完成码争议关联差评
- **输出**: 差评预警与处理方案（预警触发→通知→响应→处理→回访→统计），含差评原因分类、自动回复模板、升级条件、服务商改进建议
- **依赖**: C7-09
- **AI自治度**: 🟨
- **组合接口**: → C7-07(信用分差评扣分)、C6-10(投诉处理差评联动)、C7-17(履约质量差评关联)

## 步骤7.4 供需匹配与预警 (C7-11 to C7-13)

### C7-11 供需匹配优化策略

- **输入**: 买家需求画像(C7-05)、服务商供给画像（技能标签C7-03/等级C7-02/位置/可用时段）、匹配维度（品类/位置/价格/时间/技能/信用分/等级）、匹配算法参数（权重配置/排序策略/去重规则）、3级分销推荐链关系
- **输出**: 供需匹配优化方案（匹配算法→权重配置→排序策略→A/B测试方案→效果指标），含搜索排序规则（信用分权重≥20%/等级权重≥15%/距离权重≥15%/价格权重≤10%）
- **依赖**: 无
- **AI自治度**: 🟨
- **组合接口**: → C7-25(供需平衡调度匹配引擎)、C7-20(冷启动匹配策略)

### C7-12 供给不足品类预警

- **输入**: 品类需求数据（搜索量/浏览量/下单量/未成交订单）、品类供给数据（在架条目数/活跃服务商数/可接单容量）、供需比阈值（需求/供给<0.6触发预警）、区域×品类×时段三维分析
- **输出**: 供给不足预警报告（预警品类→缺口量化→区域分布→时段分布→招商建议→易点激励方案），按日/周自动生成
- **依赖**: 无
- **AI自治度**: ⬛
- **组合接口**: → C7-25(供需平衡调度供给维度)、C7-19(扶持方案供给不足品类优先)

### C7-13 需求溢出品类预警

- **输入**: 品类需求数据（搜索量趋势/下单失败率/平均等待时长）、服务商负载率（接单率/饱和度/拒绝率）、需求溢出阈值（下单失败率>15%或等待时长>2h触发）、品类×区域交叉分析
- **输出**: 需求溢出预警报告（溢出品类→溢出量→区域热点→时段分布→扩容建议→临时调价方案），含服务商招募与激励建议
- **依赖**: 无

## 步骤7.5 流失与活跃度管理 (C7-14 to C7-16)

### C7-15 服务商流失预警

- **输入**: 服务商绩效评估(C7-04)、活跃度数据(C7-14)、收入趋势（易点收入连续下降/提现频率降低）、投诉趋势、等级变动、竞品信息（如可获取）

### C7-16 买家流失预警

- **输入**: 买家画像(C7-05)、生命周期数据(C7-06)、消费行为趋势（下单频率下降/易点余额增长不消费/搜索不下单）、3级分销活跃度（邀请停止/分润收入下降）、差评后无复购
- **输出**: 买家流失预警结果（流失风险等级→预警原因→干预方案→易点召回激励→渠道选择），含召回效果评估框架
- **依赖**: C7-05, C7-06
- **AI自治度**: ⬛
- **组合接口**: → C7-06(生命周期管理召回策略)、C7-05(画像更新流失特征)

- **AI自治度**: 🟨

### C7-25 供需平衡调度

- **输入**: 供需匹配策略(C7-11)、供给不足预警(C7-12)、需求溢出预警(C7-13)、服务商活跃度(C7-14)、易点补贴预算、价格弹性数据、区域×品类×时段供需矩阵
- **输出**: 供需平衡调度方案（调度策略→易点激励分配→价格调整建议→服务商调度→流量分配→效果预测），含自动化调度规则与人工干预条件
- **依赖**: C7-11, C7-12, C7-13
- **AI自治度**: 🟨
- **组合接口**: → C7-11(匹配算法调度参数)、C7-20(冷启动调度特殊规则)

### C7-26 双边网络效应评估

- **输入**: 买家规模与增长、服务商规模与增长、供需匹配数据(C7-11)、跨边网络效应指标（买家增长→服务商供给增长弹性/服务商增长→买家需求增长弹性）、3级分销裂变对双边增长的贡献、易点体系对留存的贡献、完成码体验对复购的影响
- **输出**: 双边网络效应评估报告（效应强度→拐点分析→品类差异→区域差异→增长建议），含关键假设验证与A/B测试建议
- **依赖**: 无
- **AI自治度**: ⬛
- **组合接口**: → C7-11(匹配算法网络效应权重)、C7-20(冷启动网络效应阈值)
