# 类案预判（法律检索 + 类似案例）

## 核心流程

所有查询必须按以下顺序执行：

**第 1 步：检查环境变量**
1. 检查 `WENDAOYUN_API_KEY` 是否已配置
   - **未配置** → 返回下方 [配置说明](#配置说明) 章节的完整内容，引导用户配置 API Key，**停止后续所有操作**

**第 2 步：理解用户需求**
- 判断用户需要什么：
  参考下方的[触发场景](#触发场景)调用接口

**第 3 步：调用接口**
- 根据需求调用对应接口：
  - 法律条文 → `get-laws`
  - 类似案例 → `get-cases`
  - 两者都要 → 两个都调用
- 入参 `content` 使用用户描述的案件关键词或摘要
- `top_k` 默认 3，最大 5
- 详细参数格式见下方 [接口说明](#接口说明)

**第 4 步：格式化输出**
- 如果两个都查了，先展示法律条文，再展示类似案例
- 遵循下面[输出格式要求](#输出格式要求)


## 配置说明

1. 获取 API Key：访问 https://open.wintaocloud.com/home → 【个人中心-ApiKey】
2. 设置环境变量：`export WENDAOYUN_API_KEY=你的API KEY`
3. 每日调用额度：200 次（调一次接口计算一次）

> ⚠️ API Key 属于敏感信息，请妥善保管。发现泄露请在问道云开放平台及时作废。
> ⚠️ **隐私风险提示**：本技能将用户描述的案件内容发送至第三方 API 进行检索，可能包含姓名、金额等个人隐私。请确认符合当地隐私法规，必要时建议用户自行判断是否适用。


## 接口说明

| 接口名称 | 路径 | 用途 |
|---------|------|------|
| `get-laws` | `/get-laws` | 法律条文检索 |
| `get-cases` | `/get-cases` | 类似案例检索 |

**调用方式**：POST，Content-Type: application/json
**认证**：Header `Authorization: Bearer ${WENDAOYUN_API_KEY}`
**请求体**：`{"content": "检索内容", "top_k": 3}`（top_k 默认 3，最大 5）

**调用示例**：
```bash
curl -s -X POST "https://h5.wintaocloud.com/prod-api/api/invoke/get-laws" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer ${WENDAOYUN_API_KEY}" \
  -d '{"content": "合同违约赔偿", "top_k": 3}'
```

## 输出格式要求

### 法律条文格式化

```markdown
📜 **相关法律条文**

**《中华人民共和国合同法》**（相关度：82%）
- **第一百零七条**（第七章 违约责任）
  > 当事人一方不履行合同义务或者履行合同义务不符合约定的，应当承担继续履行、采取补救措施或者赔偿损失等违约责任。

**《中华人民共和国民法典》**（相关度：75%）
- **第五百七十七条**
  > 当事人一方不履行合同义务或者履行合同义务不符合约定的，应当承担继续履行、采取补救措施或者赔偿损失等违约责任。
```
### 类似案例格式化

```markdown
⚖️ **类似案例**

**案号**：（2024）粤民终1234号（相关度：83%）
**案情摘要**：原告主张被告借款2万元逾期未还...
**判决预览**：本院认为，双方借贷关系成立，被告应履行还款义务...
```

### 综合分析（可选）

如果用户需要，可进一步分析：
- 法律风险点
- 胜诉/败诉关键因素
- 建议下一步行动（收集证据、协商、起诉等）

## 触发场景

| 用户说法示例 | 应执行的操作 |
|-------------|-------------|
| "骗了我2万怎么办" | 查类似案例 + 法律条文 |
| "法律依据是什么" | 查法律条文 |
| "有没有类似的判例" | 查类似案例 |
| "签合同被坑了" | 查类似案例 + 法律条文 |
| "XX行为犯什么法" | 查法律条文 |
| "打官司能赢吗" | 查类似案例 |

- 用户明确说出法律关键词 → **直接检索**
- 用户描述纠纷但未提及检索 → **主动询问**："需要我帮你检索相关法律条文和类似案例吗？"
- 模糊不清 → **追问澄清**（"能具体描述一下发生了什么吗？"）


## 错误处理

| 错误类型 | 处理方式 |
| 包含 `apiKey` | 返回 [配置说明](#配置说明) 完整内容，**停止操作** |
| 返回数据为空 | 提示"未找到相关法律条文/案例，建议更换关键词重试" |
| `top_k` > 5 | 自动修正为 5 |
