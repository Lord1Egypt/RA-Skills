# 专家访谈方法论

## 访谈前：结构化提纲

### 问题设计原则

1. **开放式为主**：避免是非题，引导专家自由表达
2. **追问预案**：每个问题预设2层追问（"为什么会这样？→ 那又是因为什么？"）
3. **避免引导性提问**：不说"业内都说XX产品更好，您同意吗？"，改问"您怎么看不同产品之间的差异？"
4. **5-8个核心问题**：每个指向一个关键假设的验证

### 迂回问法（应对敏感问题）

| 敏感问题 | 迂回问法 |
|---------|---------|
| 直接问毛利率 | "同行为什么有的赚30%有的亏钱？差异在哪？" |
| 直接问竞争劣势 | "如果让您给竞争对手提建议，您会说什么？" |
| 直接问行业黑幕 | "消费者最常误解这个行业的哪些事？" |
| 直接问未来战略 | "三年前行业最重要的事是X，您觉得三年后会是什么？" |

### 提纲模板

```
1. 行业概况
   - 这个行业过去3年最大的变化是什么？
   - 追问：这个变化对产业链哪些环节影响最大？

2. 竞争格局
   - 如果行业只剩3家公司，您觉得会是哪三家？为什么？
   - 追问：XX公司最大的短板是什么？

3. 商业模式
   - 这个行业赚钱最核心的能力是什么？
   - 追问：现在最赚钱的环节和5年前一样吗？

4. 趋势判断
   - 3年后这个行业最可能被颠覆的是什么？
   - 追问：什么信号出现时您会改变这个判断？

5. 开放收尾
   - 有什么我没问到但您觉得很重要的事？
```

## 访谈中：偏差控制

### 识别被试者偏差

| 偏差类型 | 表现 | 对策 |
|---------|------|------|
| 幸存者偏差 | 只看到成功案例，忽略失败者 | 追问"您见过的失败案例中，最常见原因是什么？" |
| 近因偏差 | 最近经历被过度放大 | 追问"去年/前年这个时候，行业在讨论什么？" |
| 立场偏差 | 供应商/客户/竞争者立场影响判断 | 标注信源角色，交叉验证 |
| 确认偏差 | 专家只说支持自己观点的信息 | 追问"反对您这个判断的人，通常会说什么？" |
| 社会期望偏差 | 给出"正确"而非"真实"的答案 | 用迂回问法，降低防御心理 |

### 关键数字交叉验证

同一问题问3个不同信源。如果3个答案：
- 基本一致 → A级
- 2个一致 + 1个偏离 → B级（标注分歧）
- 3个完全不同 → 追问每个信源"为什么您的数字跟别人差这么多？"

## 访谈后：信息分级

### A级 — 可直接引用
- 经过交叉验证的数据和事实
- 多个独立信源互相印证的判断
- 信源本人有直接操作经验（非道听途说）
- **引用格式**：直接引述数据/判断

### B级 — 引用需标注
- 单一信源但逻辑自洽的判断
- 信源权威性高但未经第三方验证的数据
- **引用格式**："据业内人士透露……" / "有行业专家认为……"

### C级 — 仅供背景理解，不引用
- 未经验证的观点、传闻
- 信源有明确利益立场的评价性判断
- 信源本人也说明"这是猜测"的信息
- **使用方式**：帮助建立认知框架，但绝不出现在报告中

### 信息分级检查清单

- [ ] 这个数字有第二个独立来源确认吗？→ A级
- [ ] 信源在这个判断上有利益立场吗？→ 降一级
- [ ] 信源是直接参与者还是转述者？→ 转述降一级
- [ ] 这个判断形成的时间？超过6个月需更新验证

## 常见访谈错误

| 错误 | 纠正 |
|------|------|
| 问"您同意XX观点吗？" | 问"您怎么看这个问题？" |
| 只是"嗯嗯"不做笔记 | 关键数字当场复述确认 |
| 接受模糊数字 | 追问"大概在什么区间？上限/下限是多少？" |
| 只访谈一个信源就下结论 | 至少3个不同立场信源交叉验证 |
| 访谈后凭记忆整理 | 24小时内完成访谈笔记结构化 |
