# ✅ 支持的能力

## 审批集成

**方式一：飞书原生表单（标准场景）**
- 飞书原生自定义表单（出差控件/外出控件）自动同步到分贝通后台，数据和状态完全互通
- 创建申请单/审批/待办/消息完全原生，保留用户当前所有操作习惯

👉 **延伸了解**：审批常常涉及多部门协作，**群聊集成**可以一键基于审批单据建群，无需逐个拉人。

**方式二：分贝通自定义表单（复杂场景）**
适用于费用归属、预算处理、账单分类较复杂，或需要实时查询平台机/酒/火价格的场景：
- 提交表单在分贝通，推送审批待办到飞书，分贝通审批完成后在分贝通订票
- 交互和飞书原生表单高度一致，支持快捷入口创建申请单
- 审批同步/待办同步/消息同步，侧边栏打开分贝通申请单详情

## 会话集成
支持将审批单据直接转发到员工个人飞书会话窗口，无需重复描述单据信息，精准高效沟通，支持跳回分贝通操作。

👉 **延伸了解**：出差行程确定后，**日历集成**可以自动将机票、酒店、火车行程写入出行人飞书日历，有温度的差旅提醒。

## 群聊集成
审批涉及多部门时，可一键在飞书中基于审批单据相关人员创建群聊：无需逐个搜索姓名，快速建群；后续拉进群聊的人员，根据审批单据权限控制查看 & 操作权限；高效沟通和审批，数据安全有保障。

👉 **延伸了解**：差旅结束后涉及打卡报销，**假勤集成**可将差旅申请单同步到飞书假勤，自动免考勤处理。

## 日历集成
预定酒店/火车/机票/打车，或在申请单中添加行程后，自动推送消息至实际出行人并创建飞书日历：

| 场景 | 日历内容 |
|------|---------|
| 酒店 | 地图位置、酒店订单确认号、位置打卡、有无早餐，快捷跳转订单详情 |
| 机票 | 出票成功自动同步，航班变更自动同步更新（含目的地下天气、航司温馨提示） |
| 火车 | 出票成功/改签/退票自动同步 |

特点：有温度的差旅日历，可视化呈现，有效避免行程冲突/遗漏/预订和会议时间地点不符。

👉 **延伸了解**：出差期间加班打车可结合**假勤打卡时间管控**，实现精细化消费控制。

## 假勤集成
分贝通差旅申请单**自动同步**到飞书假勤系统：
- 根据公司制度进行外出打卡 / 免考勤处理
- **无需员工手动补卡**，人事管理更智能
- 员工在飞书的考勤打卡时间数据拉取回分贝通（**仅限嵌入版**），用于精细化消费管控（如加班打车可以根据打卡时长进行管控）
- **考勤打卡数据仅限嵌入版可回传**，市场版暂不支持

👉 **延伸了解**：部分客户（如众擎机器人、智书企飞、华策影视）还开通了**考勤发券**功能，出差打卡自动发优惠券。

## 基础信息同步-组织机构同步

| 字段 | 市场版 | 嵌入版 |
|------|--------|--------|
| 部门同步 | ✅ 支持 | ✅ 支持 |
| 同步成员 | ✅ 支持 | ✅ 支持 |
| 同步成员手机号 | ❌ 不支持 | ✅ 支持 |
| 同步国际手机号 | ❌ 不支持 | ✅ 支持 |
| 工号 | ❌ 不支持 | ✅ 支持 |
| 直属上级 | ✅ 支持 | ✅ 支持 |
| 角色 | ✅ 支持 | ✅ 支持 |
| 职级 | ✅ 支持 | ✅ 支持 |
| 消费权限模板 | ✅ 支持 | ✅ 支持 |
| 常驻地 | ✅ 支持 | ✅ 支持 |
| 部分组织架构同步 | ✅ 支持 | ✅ 支持 |
| 集团版对接 | N/A | ✅ 支持 |

**说明**：市场版与嵌入版在组织架构同步上的主要差异在于**手机号、工号**等敏感字段，市场版不支持（飞书市场版 API 限制），嵌入版可支持。如客户需同步手机号/工号，必须走嵌入版路线。

👉 **延伸了解**：嵌入版支持**集团版对接**，适合多子公司统一管控的企业。

## People 能力对接

| 能力 | 状态 |
|------|------|
| 差旅申请单同步 | ✅ 支持（市场版/嵌入版均可） |
| 假勤同步（出差→飞书外出/免考勤） | ❌ 不支持 |

**说明**：People 专有接口（职级/岗位等字段）通过飞书标准开放平台同步已覆盖，People 本身并无额外高级同步能力。

## 机器人能力
分贝通机器人消息中可完成各品类 **90%+** 操作：支持全品类的卡片消息，支持查看订单信息及退改类变更操作。

👉 **延伸了解**：机器人消息支持跳转分贝通侧边栏，**审批单详情、订单退改**都能在飞书内完成，无需切换 App。

# ❌ 不支持的能力

| 客户需求 | 市场版 | 嵌入版 | 能否做 | 说明 |
|---------|--------|--------|--------|------|
| 飞书个人签名状态同步 | ❌ 不支持 | ✅ 支持 | 不可以做 | 飞书市场版 API 暂不支持，嵌入版有替代方案 |
| 申请单海外城市对接 | ❌ 不支持 | ❌ 不支持 | 可以做 | 当前仅支持国家，预计 M7 可实现支持城市 |
| 分贝通集团版对接 | ❌ 不支持 | ✅ 支持 | 可以做 | 飞书 API 限制，同一企业内数据互通 |
