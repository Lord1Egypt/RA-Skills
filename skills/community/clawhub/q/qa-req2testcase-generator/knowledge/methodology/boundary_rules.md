# 金融场景边界值规则

> 适用范围：券商交易系统中涉及价格、数量、金额、时间的边界值测试规则，覆盖 A 股（沪/深/北交所）、港股通、期权等品种。

## 1. 精度截断规则

### 定义

金融系统中价格、数量、金额的小数位精度有严格规定。不同市场、不同品种的精度要求不同，系统必须正确处理精度截断（而非四舍五入），避免因浮点数精度问题导致资金差异。

### 适用场景

- 委托价格输入校验
- 成交金额计算
- 手续费/印花税计算
- 资金余额展示
- 清算对账

### 券商业务精度规则表

| 市场/品种 | 字段 | 精度要求 | 截断规则 | 示例 |
|-----------|------|---------|---------|------|
| 沪A股票 | 委托价格 | 2位小数 | 截断（不四舍五入） | 15.567 → 15.56 |
| 深A股票 | 委托价格 | 2位小数 | 截断 | 23.999 → 23.99 |
| 北交所股票 | 委托价格 | 2位小数 | 截断 | 8.125 → 8.12 |
| 沪深基金 | 委托价格 | 3位小数 | 截断 | 1.2345 → 1.234 |
| 沪深债券 | 委托价格 | 3位小数 | 截断 | 100.1256 → 100.125 |
| 港股通 | 委托价格 | 3位小数 | 截断 | 68.1259 → 68.125 |
| 期权 | 权利金 | 4位小数 | 截断 | 0.35678 → 0.3567 |
| 所有品种 | 成交金额 | 2位小数 | 截断 | 180000.567 → 180000.56 |
| 所有品种 | 手续费 | 2位小数 | 向上取整（进一法） | 4.501 → 4.51 |
| 所有品种 | 印花税 | 2位小数 | 向上取整（进一法） | 1.001 → 1.01 |

### 工程实现：精度截断

#### Java BigDecimal 实现

```java
import java.math.BigDecimal;
import java.math.RoundingMode;

public class PriceUtil {
    
    /**
     * 价格截断（不四舍五入）
     * @param price 原始价格
     * @param scale 小数位数
     * @return 截断后的价格
     */
    public static BigDecimal truncatePrice(BigDecimal price, int scale) {
        return price.setScale(scale, RoundingMode.DOWN);
    }
    
    /**
     * 手续费向上取整（进一法）
     */
    public static BigDecimal ceilFee(BigDecimal fee) {
        return fee.setScale(2, RoundingMode.UP);
    }
    
    // 示例
    public static void main(String[] args) {
        // 沪A股票价格截断：15.567 → 15.56
        System.out.println(truncatePrice(new BigDecimal("15.567"), 2));  // 15.56
        
        // 基金价格截断：1.2345 → 1.234
        System.out.println(truncatePrice(new BigDecimal("1.2345"), 3)); // 1.234
        
        // 手续费进一法：4.501 → 4.51
        System.out.println(ceilFee(new BigDecimal("4.501")));           // 4.51
        
        // ⚠️ 常见错误：使用 double 导致精度丢失
        double wrongPrice = 0.1 + 0.2;  // 0.30000000000000004
        BigDecimal correctPrice = new BigDecimal("0.1").add(new BigDecimal("0.2")); // 0.3
    }
}
```

#### Python Decimal 实现

```python
from decimal import Decimal, ROUND_DOWN, ROUND_UP, getcontext

# 设置全局精度
getcontext().prec = 28

def truncate_price(price: str, scale: int) -> Decimal:
    """价格截断（不四舍五入）"""
    d = Decimal(price)
    quantize_str = '0.' + '0' * scale
    return d.quantize(Decimal(quantize_str), rounding=ROUND_DOWN)

def ceil_fee(fee: str) -> Decimal:
    """手续费向上取整（进一法）"""
    d = Decimal(fee)
    return d.quantize(Decimal('0.01'), rounding=ROUND_UP)

# 示例
print(truncate_price("15.567", 2))   # 15.56（沪A股票价格）
print(truncate_price("1.2345", 3))   # 1.234（基金价格）
print(ceil_fee("4.501"))             # 4.51（手续费）

# ⚠️ 常见错误：使用 float
wrong = 0.1 + 0.2          # 0.30000000000000004
correct = Decimal("0.1") + Decimal("0.2")  # 0.3

# ⚠️ 常见错误：Decimal(0.1) vs Decimal("0.1")
bad = Decimal(0.1)    # 0.1000000000000000055511151231257827021181583404541015625
good = Decimal("0.1") # 0.1
```

## 2. 最大委托量与最小委托金额

### 定义

交易所和券商对单笔委托的数量和金额设有上下限，防止异常委托影响市场秩序。不同市场、不同品种的限制不同。

### 适用场景

- 委托下单时的数量/金额校验
- 大宗交易的特殊限制
- 零股处理（卖出时允许不足一手）

### 券商业务规则表

| 市场/品种 | 最小委托量 | 最大委托量（单笔） | 步长 | 最小委托金额 | 备注 |
|-----------|-----------|-------------------|------|-------------|------|
| 沪A股票（买入） | 100股（1手） | 99,999,900股 | 100股 | 无限制 | 买入必须整手 |
| 沪A股票（卖出） | 1股 | 99,999,900股 | 1股 | 无限制 | 卖出允许零股（不足1手的部分） |
| 深A股票（买入） | 100股 | 99,999,900股 | 100股 | 无限制 | 同沪A |
| 深A股票（卖出） | 1股 | 99,999,900股 | 1股 | 无限制 | 同沪A |
| 北交所股票（买入） | 100股 | 99,999,900股 | 1股 | 无限制 | 步长为1股（非100整数倍） |
| 港股通 | 1股 | 按个股规定 | 1股 | 无限制 | 每手股数因个股而异 |
| 沪深ETF | 100份 | 99,999,900份 | 100份 | 无限制 | |
| 沪深可转债 | 10张 | 10,000张 | 10张 | 无限制 | 1张=100元面值 |
| 期权 | 1张 | 按品种规定 | 1张 | 无限制 | 单笔限额因品种而异 |
| 大宗交易（沪A） | — | — | — | 200万元或50万股 | 需满足金额或数量门槛 |

## 3. 数量上下限（不同市场/品种）

### 定义

除单笔委托限制外，还存在账户级别的持仓限制、单日累计委托限制等。这些限制因市场、品种、账户类型而异。

### 适用场景

- 持仓集中度风控
- 单日累计委托量监控
- 信用账户担保比例
- 期权持仓限额

### 券商业务规则表

| 限制类型 | 规则 | 触发条件 | 处理方式 |
|---------|------|---------|---------|
| 单一股票持仓比例 | 不超过该股票总股本的5% | 买入后持仓超限 | 拒绝委托 |
| 信用账户维持担保比例 | 不低于130% | 担保比例低于阈值 | 触发追保通知 |
| 信用账户平仓线 | 低于130%（各券商不同） | 担保比例持续低于平仓线 | 强制平仓 |
| 期权单品种持仓限额 | 权利仓≤5000张（初级） | 买入后超限 | 拒绝委托 |
| 单日撤单次数 | 部分券商限制≤500次/日 | 累计撤单超限 | 拒绝撤单 |
| 新股申购上限 | 按市值计算（每1万元市值=1000股额度） | 申购数量超过额度 | 拒绝或按上限申购 |

## 4. 时间边界（交易时段边界值）

### 定义

证券交易有严格的时间窗口，不同阶段（集合竞价、连续竞价、盘后交易）的规则不同。时间边界是高频出现缺陷的区域。

### 适用场景

- 委托时间校验
- 集合竞价/连续竞价切换
- 盘前/盘后委托处理
- 跨日清算时间点
- 节假日休市处理

### 券商业务时间规则表（A股）

| 时段 | 时间范围 | 允许操作 | 特殊规则 |
|------|---------|---------|---------|
| 盘前委托 | 前一交易日清算后 ~ 09:15 | 限价委托（暂存） | 委托暂存，09:15后统一报盘 |
| 开盘集合竞价 | 09:15 ~ 09:25 | 限价委托 | 09:20~09:25不可撤单（沪市） |
| 集合竞价撮合 | 09:25 ~ 09:30 | 不接受委托 | 系统撮合产生开盘价 |
| 上午连续竞价 | 09:30 ~ 11:30 | 限价/市价委托 | 正常交易 |
| 午间休市 | 11:30 ~ 13:00 | 限价委托（暂存） | 委托暂存，13:00后报盘 |
| 下午连续竞价 | 13:00 ~ 14:57 | 限价/市价委托 | 正常交易 |
| 收盘集合竞价 | 14:57 ~ 15:00 | 限价委托 | 不可撤单，产生收盘价 |
| 盘后固定价格交易 | 15:05 ~ 15:30 | 限价委托（收盘价） | 仅沪市，按收盘价成交 |
| 清算时段 | 15:30 ~ 次日（各券商不同） | 不接受委托 | 系统清算，通常20:00后可接受次日委托 |
