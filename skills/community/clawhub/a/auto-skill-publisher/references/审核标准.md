# ClawHub技能审核标准

## 一、内容审核清单

### 敏感内容检查
- [ ] 无政治敏感内容（领导人、政党、敏感历史事件）
- [ ] 无色情低俗内容（性暗示、裸露内容）
- [ ] 无暴力血腥内容（虐待、凶杀、恐怖内容）
- [ ] 无虚假诈骗信息（虚假广告、钓鱼内容）
- [ ] 无侵犯版权内容（盗版软件、未授权内容）
- [ ] 无恶意代码/病毒（木马、蠕虫、恶意脚本）

### 合规性检查
- [ ] 符合目标平台当地法律法规
- [ ] 无歧视性内容（种族、性别、宗教歧视）
- [ ] 无垃圾信息特征（过度重复、关键词堆砌）

## 二、格式审核清单

### SKILL.md 格式
- [ ] 包含 YAML frontmatter
- [ ] frontmatter 包含 `name` 字段
- [ ] frontmatter 包含 `description` 字段
- [ ] description 长度在 50-200 字符之间

### 文件结构
- [ ] SKILL.md 位于文件夹根目录
- [ ] 无二进制文件（图片、视频、可执行文件）
- [ ] 无版本控制文件（.git、.svn）
- [ ] 无隐藏文件（.DS_Store、Thumbs.db）
- [ ] 无许可证文件（LICENSE、COPYING）

### Slug 规范
- [ ] 仅包含小写字母
- [ ] 仅包含数字
- [ ] 仅包含短横线 `-`
- [ ] 不以短横线开头或结尾
- [ ] 长度在 3-50 字符之间

## 三、质量审核清单

### 描述质量
- [ ] 功能描述清晰易懂
- [ ] 使用场景明确
- [ ] 核心功能突出

### 文档质量
- [ ] 使用方法完整
- [ ] 示例可实际操作
- [ ] 无明显语法错误
- [ ] 无乱码或格式错误

### 技术质量
- [ ] 脚本无明显错误
- [ ] 依赖明确说明
- [ ] 错误处理完善

## 五、常见问题修复指南

### 问题1: 描述过长
- **现象**: description 超过 200 字符
- **修复**: 精简描述，突出核心功能

### 问题2: 描述过短
- **现象**: description 少于 50 字符
- **修复**: 补充功能说明和使用场景

### 问题3: 包含二进制文件
- **现象**: 文件夹包含 .png、.jpg、.exe 等文件
- **修复**: 删除所有二进制文件，仅保留文本文件

### 问题4: Slug 包含大写字母
- **现象**: Slug 如 `My-Skill-Name`
- **修复**: 改为 `my-skill-name`

### 问题5: 缺少必要字段
- **现象**: frontmatter 缺少 name 或 description
- **修复**: 添加完整 frontmatter
