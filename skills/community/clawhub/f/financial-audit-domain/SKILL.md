# 财报排雷审计分析知识参考库

## 定位

本技能是 **Universal Task OS 的领域负载物仓库**，不包含任何执行框架。只提供财报排雷的"是什么"和"长什么样"——执行全部委托UTOS。

| 本技能提供 | UTOS消费方式 |
|-----------|-------------|
| 排雷任务清单+依赖拓扑 | 内容轴·清单法的任务目录 + 执行轴·管线编排的依赖输入 |
| 50项指标要求 | 内容轴·清单法的组件清单 |
| 优秀范本（待填充） | 内容轴·样本法的样本 |

## 三层结构

```
第一层：任务清单与依赖拓扑   →  references/audit-catalog.md
第二层：指标要求清单          →  references/audit-requirements.md
第三层：优秀范本库            →  references/exemplars.md
```

## 依赖声明

本技能**强依赖** Universal Task OS (universal-task-os)。没有UTOS，本技能只有参考查阅能力，无法执行任何排雷分析任务。

**加载检查流程**（每次激活时执行）：

1. 检测 `universal-task-os` 技能是否已安装
2. **未安装** → 自动安装 `universal-task-os` 技能
3. **安装成功** → 同时加载UTOS，按本技能"使用规则"执行
4. **安装失败** → 降级为**只读参考模式**：
   - ✅ 允许：查阅任务清单、指标要求、范本索引
   - ❌ 拒绝：任何涉及排雷计算、管线编排、报告生成的任务

**任务模式判定**：

| 任务类型 | 无UTOS | 有UTOS |
|---------|--------|--------|
| 查阅任务清单/指标要求/范本 | ✅ 只读参考 | ✅ 完整 |
| 按清单法产出排雷报告 | ❌ 拒绝 | ✅ UTOS编排执行 |
| 依赖拓扑推导分析管线 | ❌ 拒绝 | ✅ UTOS执行轴 |
| 舞弊红线检查点插入 | ❌ 拒绝 | ✅ UTOS守护单元 |

## 使用规则

1. **依赖检查**：激活时按上述流程检测并安装UTOS
2. **首次加载**：读取 `references/audit-catalog.md`，获取域分类、依赖拓扑
3. **按需深入**：确认目标任务类型后，读取 `references/audit-requirements.md` 获取50项指标的详细要求
4. **数据源优先级**：TDX MCP优先→THS降级→手动输入兜底

## ⚠️ 适用范围限制

**本排雷框架仅适用于非金融类 A 股企业**（制造业、消费、能源、运输、地产、科技等）。

**明确不适用于以下行业：**
- 🏦 银行（现金流量表结构完全不同）
- 🛡️ 保险（收入确认和准备金计提逻辑特殊）
- 📈 证券/券商（商业模式和财务指标无意义）

## 财报排雷特有维度

| 维度 | 说明 | 影响 |
|------|------|------|
| 合并报表口径 | 必须使用合并报表，严禁母公司报表 | 母公司报表会遗漏子公司数据，导致分析失真 |
| 4期全周期 | 连续4期数据对比，不遗漏任何一期 | 单期数据无法判定趋势，必须全周期分析 |
| 风险升级规则 | 单期→2期→4期逐级升级 | 持续异常比单期异常风险更高 |
| 舞弊红线 | 5条红线触发任一直接【坚决回避】 | 优先级最高，不可妥协 |
| 数据源双轨 | TDX优先，THS降级 | 不同数据源覆盖范围不同，需注明实际使用 |

## 指标要求清单

所有指标均需完整对比4期全周期数据，分四期单独列示，不遗漏任何一期。

### F1-01 货币资金分析
- **必选组件**: 货币资金期末余额(亿元)、利息收入(亿元)、货币资金收益率(%)、逐期变动趋势
- **可选组件**: 受限资金说明、银行函证结果
- **组装顺序**: 资产负债表取数→利润表利息收入取数→平均余额计算→收益率计算→趋势判定
- **约束**: 收益率<0.15%→标记异常；持续2期→升级高风险；4期全周期→升级极高风险
- **格式**: 四期对比表（Markdown）

### F1-04 货币资金覆盖比
- **必选组件**: 覆盖比=货币资金/短期有息负债、四期数据、判定结果
- **可选组件**: 趋势分析
- **组装顺序**: F1-01/F1-02取数→除法计算→四期列示→判定
- **约束**: ≥1→安全；0.5-1→谨慎；<0.5→高危；持续2期<0.5→极高风险
- **格式**: 四期对比表（Markdown）

### F1-06 存贷双高判定
- **必选组件**: 三条件判定结果（①货币资金≥10%资产 ②有息负债≥20%资产 ③利息费用/利息收入>3）
- **可选组件**: 判定理由
- **组装顺序**: 三条件逐项判定→同时满足→标记存贷双高
- **约束**: 单期触发→资金真实性异常；任意2期→极高舞弊风险
- **格式**: 四期判定表（Markdown）

### F2-01 应收账款+应收票据分析
- **必选组件**: 应收净额(亿元)、占营收比例(%)、增速(%)、周转率
- **可选组件**: 坏账准备、账龄分析
- **组装顺序**: 资产负债表取数→营收取数→占比计算→增速计算→周转率计算→趋势判定
- **约束**: <20%→健康；20-40%→一般；>40%→高风险
- **格式**: 四期对比表（Markdown）

### F2-08 存货利润操纵判定
- **必选组件**: 三信号逐期判定（①存货增速>营收增速30% ②毛利率逆势上涨≥0.5pct ③存货周转天数拉长≥10%）
- **可选组件**: 行业对比数据
- **组装顺序**: 三信号逐期判定→同时触发3个→舞弊极高风险
- **约束**: 单期触发2个→关注；触发3个→舞弊极高风险；持续2期≥2个→极高舞弊风险
- **格式**: 四期判定表（Markdown）

### F4-07 供应链议价能力差额
- **必选组件**: 供应链净占款差额(亿元)、占营收比例(%)、逐期判定
- **可选组件**: 行业对比
- **组装顺序**: 应付+合同负债取数→应收+预付+合同资产取数→差额计算→占比计算→判定
- **约束**: 差额>0→强议价；差额<0→资金被占用；持续2期为负且扩大→高风险
- **格式**: 四期对比表（Markdown）

### F5-01 营业收入分析
- **必选组件**: 营收(亿元)、增速(%)、销售收现比
- **可选组件**: 收入结构、行业对比
- **组装顺序**: 利润表取数→增速计算→收现比计算→趋势判定
- **约束**: 增速±20%→健康；波动>50%无理由→异常；收现比<0.9→收入质量异常
- **格式**: 四期对比表（Markdown）

### F5-07 净利润与扣非净利润
- **必选组件**: 净利润(亿元)、扣非净利润(亿元)、扣非占比(%)、反向变动判定
- **可选组件**: 非经常性损益明细
- **组装顺序**: 利润表取数→扣非占比计算→反向变动判定
- **约束**: 扣非占比≥80%→高质量；<60%→低质量；扣非连续3期为负→坚决回避
- **格式**: 四期对比表（Markdown）

### F5-09 归母与合并净利润偏离度
- **必选组件**: 偏离度=归母/合并净利润×100%、四期数据、判定结果
- **可选组件**: 少数股东损益明细
- **组装顺序**: 归母取数→少数股东损益取数→合并=归母+少数→偏离度计算→判定
- **约束**: 90%-110%→正常；超出→利润操纵嫌疑；持续2期→极高舞弊风险
- **格式**: 四期对比表（Markdown）

### F6-01 三大现金流与类型判定
- **必选组件**: CFO/CFI/CFF(亿元)、正负标注、类型判定（优质现金牛/扩张成长型/烧钱续命型/困境变卖型）
- **可选组件**: 各现金流结构分析
- **组装顺序**: 现金流量表取数→正负判定→类型匹配→四期标注
- **约束**: 类型判定必须基于CFO/CFI/CFF正负组合
- **格式**: 四期判定表（Markdown）

### F6-02 净利润现金比率
- **必选组件**: 比率=CFO/净利润、四期数据、判定结果
- **可选组件**: 趋势分析
- **组装顺序**: CFO取数→净利润取数→除法计算→四期列示→判定
- **约束**: 连续≥1→利润真实；<0.5→极高造假嫌疑；连续2期<0.5→极高舞弊风险
- **格式**: 四期对比表（Markdown）

### F7-01 舞弊红线检查
- **必选组件**: 5条红线逐条判定结果（通过/不通过）
- **可选组件**: 触发依据、证据链
- **组装顺序**: 红线1-5逐条检查→结果汇总→触发判定
- **约束**: 触发任一→直接判定【坚决回避标的】；优先级最高
- **格式**: 舞弊红线检查表（Markdown检查表）

### F7-02 风险等级判定
- **必选组件**: 风险等级（正常/谨慎关注/关注/规避/坚决回避）、判定依据
- **可选组件**: 异常项汇总、修正建议
- **组装顺序**: 异常项统计→风险矩阵匹配→等级判定→依据记录
- **约束**: 按排雷判定矩阵执行；舞弊红线优先级最高
- **格式**: 风险判定报告（Markdown）

## 与UTOS的接口

当UTOS处理财报排雷领域任务时：

- **Step 0 三轴判定**：财报排雷任务通常为复杂+高度结构化+合规驱动 → 执行轴+内容轴+守护轴
- **Step 1 领域校准**：R1(信息密度)=极高（三大报表数据）+R2(创造性)=零（纯计算+规则判定）+R3(交互性)=低（数据输入后自动执行）+R4(规范性)=极高（会计准则+审计规范）+R5(迭代性)=低（一次性分析）→ S极高C深A标准G极高
- **Step 2 内容轴**：清单法用本技能的audit-requirements；样本法用本技能的exemplars
- **Step 3 执行轴**：管线编排基于本技能的依赖拓扑自动推导元操作序列
- **Step 4 交付**：G类守护单元自动插入舞弊红线检查点+数据口径校验
