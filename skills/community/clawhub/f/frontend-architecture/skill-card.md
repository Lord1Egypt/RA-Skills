## Description: <br>
Designs scalable frontend architecture for large applications, including project structure, module boundaries, dependency management, code splitting strategy, and technical debt prevention. <br>

This skill is ready for commercial/non-commercial use. <br>

Developers and frontend teams use this skill to plan maintainable frontend architectures, define module boundaries, select technologies, create code-splitting strategies, and manage technical debt for growing applications. <br>

## Known Risks and Mitigations: <br>
Risk: Generated frontend architecture and configuration examples may not fit a project's constraints if applied without review. <br>
Mitigation: Review generated ESLint, Vite, Next.js, monorepo, and ADR examples before applying them to a real project. <br>
Risk: Capability tags in server evidence include crypto and purchase-related capabilities that do not match the documentation-only behavior. <br>
Mitigation: Treat the skill as guidance-only and correct the mismatched capability tags during release maintenance. <br>
Risk: Broad trigger wording may activate the skill when general frontend discussion was intended. <br>
Mitigation: Narrow trigger wording if accidental activation becomes disruptive. <br>
