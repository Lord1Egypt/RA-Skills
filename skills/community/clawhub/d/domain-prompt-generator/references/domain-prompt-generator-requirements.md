# 领域提示词生成器方法论要求清单

## M0 任务域协调域

### M0-01 任务域选择与组合
- **必选组件**: 需求分析、域扫描、域选择、组合方案
- **可选组件**: 域优先级评估、域依赖分析
- **组装顺序**: 需求分析→域扫描→域选择→组合方案
- **约束**: 域选择必须基于任务需求，不得随意组合
- **格式**: 任务域选择报告（Markdown）

### M0-02 执行顺序确定
- **必选组件**: 依赖分析、顺序方案、关键路径、执行计划
- **可选组件**: 并行执行机会、风险评估
- **组装顺序**: 依赖分析→顺序方案→关键路径→执行计划
- **约束**: 执行顺序必须尊重依赖关系
- **格式**: 执行顺序图（Mermaid/Markdown）

### M0-03 结果整合与优化
- **必选组件**: 结果汇总、整合方案、优化建议、整合报告
- **可选组件**: 质量评估、改进建议
- **组装顺序**: 结果汇总→整合方案→优化建议→整合报告
- **约束**: 整合必须保持各域结果的完整性
- **格式**: 整合报告（Markdown）

### M0-04 深化路由
- **必选组件**: 需求分析、类型识别、路由方案、路由报告
- **可选组件**: 路由优先级、路由优化
- **组装顺序**: 需求分析→类型识别→路由方案→路由报告
- **约束**: 路由必须准确匹配任务类型
- **格式**: 深化路由报告（Markdown）

## M1 领域分析域

### M1-01 领域识别
- **必选组件**: 主题解析、核心内容识别、领域分类、领域整理
- **可选组件**: 领域来源分析、领域重要性评估
- **组装顺序**: 主题解析→核心内容识别→领域分类→领域整理
- **约束**: 领域识别必须基于用户提供的主题
- **格式**: 领域识别报告（Markdown）

### M1-02 领域范围
- **必选组件**: 范围定义、边界确定、范围验证、范围报告
- **可选组件**: 范围可视化、范围案例
- **组装顺序**: 范围定义→边界确定→范围验证→范围报告
- **约束**: 范围必须合理，不宜过大或过小
- **格式**: 领域范围文档（Markdown）

### M1-03 领域特征
- **必选组件**: 特征识别、特征分类、特征分析、特征报告
- **可选组件**: 特征可视化、特征案例
- **组装顺序**: 特征识别→特征分类→特征分析→特征报告
- **约束**: 特征必须具有代表性
- **格式**: 领域特征报告（Markdown）

### M1-04 领域边界
- **必选组件**: 边界识别、边界定义、边界验证、边界报告
- **可选组件**: 边界可视化、边界案例
- **组装顺序**: 边界识别→边界定义→边界验证→边界报告
- **约束**: 边界必须清晰明确
- **格式**: 领域边界文档（Markdown）

### M1-05 领域报告
- **必选组件**: 报告目标、报告内容、报告格式、报告生成
- **可选组件**: 报告可视化、报告分发策略
- **组装顺序**: 报告目标确定→报告内容整理→报告格式设计→报告生成
- **约束**: 报告内容必须完整；报告格式必须清晰
- **格式**: 领域分析报告（Markdown）
