# 接入质量检查清单

> 本清单用于 SDK 接入完成后、上线前的质量评估。按环节逐项检查，确保接入规范、数据准确。

## 一、前置准备

- [ ] **已阅读对应端的 SDK 接入说明文档**
- [ ] **数据源为专用数据源**（建议新建）
- [ ] **数据源已绑定正确的 AppID**
- [ ] **分发开关已按应用类型正确开启**（参考[SDK选型决策树](../选型/SDK选型决策树.md)第 3 步）
- [ ] **安全域名已配置**（小程序/小游戏端）：`https://api.datanexus.qq.com`
- [ ] **AppID 关联已完成**（小程序/小游戏端）

## 二、SDK 版本与初始化

### 2.1 SDK 版本

- [ ] **SDK 版本为满足监管新规的最新版本**

### 2.2 初始化规范

- [ ] **尽早初始化**
- [ ] **隐私协议前置**：用户**同意隐私协议后**再调用 `init()`
- [ ] **避免重复初始化**
- [ ] **参数正确性**

### 2.3 启动方法（Android / iOS / 鸿蒙端）

- [ ] **init 后立即调用 start**

## 三、必报事件完整性

> 详细清单参考[行为类型枚举表](../埋点/行为类型枚举表.md)第一节。

### 3.1 APP 端（Android / iOS / 鸿蒙）

- [ ] **START_APP**（启动应用）
- [ ] **REGISTER**（注册）
- [ ] **LOGIN**（登录）
- [ ] **CREATE_ROLE**（创建角色）
- [ ] **PURCHASE**（付费），**必须传 value（金额，单位：分）**

### 3.2 START_APP 上报规范（Android / iOS 端重点）

- [ ] **冷启动上报**
- [ ] **热启动上报**
- [ ] **鸿蒙/小程序/小游戏**：START_APP 为自动采集，**不要手动上报**

### 3.3 付费事件上报规范

- [ ] **value 必传**：类型为 number，单位为分

## 四、事件上报规范

- [ ] **上报时机准确**
- [ ] **专用方法优先**
- [ ] **用户 ID 设置**（小程序/小游戏端）
- [ ] **actionParam 键值规范**

## 五、平台特有检查项

### 5.1 Android 端

- [ ] **权限配置**
- [ ] **混淆配置**
- [ ] **OAID / 信通院 SDK**：版本建议 **2.8.0 及以上**

### 5.2 iOS 端

- [ ] **Linker Flags**：`-ObjC`
- [ ] **ATT 权限**（可选）

### 5.3 鸿蒙端

- [ ] **权限配置**：`INTERNET` 和 `GET_NETWORK_INFO`

### 5.4 小程序 / 小游戏端

- [ ] **安全域名**
- [ ] **SDK 不跨端使用**

## 六、数据合规

- [ ] **隐私政策已披露 SDK 信息**
- [ ] **用户同意在前，初始化在后**
- [ ] **权限最小必要**
- [ ] 详细合规要求参考[数据合规指引](../合规/数据合规指引.md)

## 七、数据验证

### 7.1 开发阶段

- [ ] **开启调试模式**
- [ ] **验证上报接口返回码**：返回 `code: 0` 即为成功
- [ ] **使用真机测试**

### 7.2 联调阶段

- [ ] **DataNexus 日志查询**

### 7.3 上线后

- [ ] **质量看板检查**：数据为 **T+1 更新**
