# 轻量版动漫语义推荐方法论

## 核心判断

当前阶段先不把重点放在数据库、向量索引和自动打分上。更有价值的是把推荐过程稳定成一套方法：

```text
理解用户喜欢的体验
  -> 抽取相似维度
  -> 分层给候选
  -> 解释像在哪里和哪里不同
  -> 根据反馈收窄
```

LLM 的优势在于理解自然语言里的细腻口味，而不是机械执行标签检索。

## 1. 先抓锚点作品

用户说“类似 X”时，先判断 X 的媒介和核心体验。

不要只问：

```text
X 是什么题材？
```

要问：

```text
用户喜欢 X 的哪种观看体验？
```

例子：《和班上第二可爱的女孩成为朋友》

```text
校园恋爱
轻小说感
普通/阴角男主
班级人气女生
从朋友开始慢慢变暧昧
低压力日常糖
不希望太后宫
```

## 2. 抽体验维度

推荐前先在脑内生成一个 taste profile：

```text
关系模式：
  朋友到恋人、女主主动靠近、互相发现对方价值、慢热暧昧

情绪口感：
  甜、害羞、轻松、低压力、日常感

叙事节奏：
  小事件推进、互动累积、低冲突、恋爱线清晰

雷点：
  后宫过强、党争太重、卖肉太多、误会拖太久
```

不同题材换不同维度。比如“芙莉莲系”要看时间感、关系余韵、战斗是否主轴；“校园恋爱系”要看关系结构、暧昧密度、女主主动性、后宫度。

## 3. 分层推荐

不要给一个平铺榜单。分三层：

```text
直球相似：
  核心体验最接近。

相邻相似：
  某些体验相似，但表面题材或情绪强度不同。

拓展尝试：
  有一个关键点很像，但整体方向有偏移。必须说明这是 stretch pick。
```

例子：用户喜欢《和班上第二可爱的女孩成为朋友》

```text
直球相似：
  我心里危险的东西、久保同学不放过我、邻家天使

相邻相似：
  堀与宫村、放学后失眠的你、更衣人偶坠入爱河

拓展尝试：
  义妹生活、败犬女主太多了、月色真美
```

## 4. 每个推荐必须解释“像在哪里”

不要只说：

```text
同为校园恋爱，所以推荐。
```

要说：

```text
像的是女主主动靠近男主，以及两个人从日常互动里逐渐建立亲密感。
不同的是它更偏纯糖/更偏群像/更偏青春文学。
```

推荐解释建议包含：

```text
像在哪里
哪里不同
可能不适合点
```

## 5. 反馈驱动收窄

用户反馈比第一次推荐更重要。

```text
“这些都看过”
  -> 下潜到中腰部、旧作、漫画/轻小说原作。

“太后宫”
  -> 降低多女主、党争、卖肉和暧昧对象数量。

“太甜腻”
  -> 加一点现实感、成长线或轻微苦味。

“想要女主更主动”
  -> 提高主动女主、调侃男主、关系推进明确的作品。

“想要更纯爱”
  -> 降低三角关系和党争，优先 1v1。

“想要更有剧情”
  -> 从日常糖转向青春剧情、社团、悬疑或成长主线。
```

## 6. 什么时候用工具/数据

默认推荐可以不联网，尤其是用户只是要“像 X 的口味”。但只要使用了搜索/浏览，或者作品事实会影响推荐，就必须先按高质量数据源核实，再做语义判断。

需要工具的情况：

```text
作品很新或很冷门，需要确认基本事实。
用户要求来源链接、年份、评分、平台。
需要维护看过列表或长期反馈。
需要批量扩充某个题材候选池。
```

强制源顺序：

```text
1. 官方动画/出版社/流媒体页面
   用途：播出状态、改编状态、制作方、集数、staff、cast、版权上线情况。

2. Bangumi
   用途：中文名、中文 ACG 语境、评分、标签、社区热度、条目关系。

3. AniList
   用途：结构化标签、格式、人气、跨语种标题、API 友好元数据。

4. MAL/Jikan
   用途：国际热度、英文标题、补充元数据。

5. TMDb
   用途：海报、图片、通用媒体资料；不作为动漫口味判断主源。
```

使用原则：

```text
搜索引擎只能用来找入口，不能把搜索摘要当事实。
如果要查锚点作品，至少先看 Bangumi 或 AniList；当前/冷门/改编状态不清的作品还要看官方源。
不要依赖随机 SEO 站、搬运简介站、AI 生成榜单、无来源博客做事实依据。
如果指定高质量源不可用，要明确说“未能从指定高质量源核实”。
推荐理由可以来自 LLM 的语义判断，但事实字段要尽量来自高质量数据源。
回复过程描述里要说清楚查的是“Bangumi/AniList/官方信息”，不能只说“搜索了网页”。
```

本地 CLI 是实验工具，不是默认推荐路径。它适合测试方法是否能工程化，不适合替代 LLM 的即时语义判断。

## 7. 推荐输出模板

```text
你喜欢的可能是这几个点：
- ...
- ...
- ...

1. 作品名
   像在哪里：...
   不同/注意：...

2. 作品名
   像在哪里：...
   不同/注意：...

如果你反馈“这些都看过/太后宫/太甜/想更冷门”，下一轮我会按这个方向收窄。
```
