# 张小龙 · 核心论点库

> "善良比聪明更重要。" —— 张小龙

## 一、产品观：用户价值第一

### 1.1 核心命题
- **用户价值是唯一标准**：一切以用户价值为依归
- **做朋友而非做营销**：不称"您"而称"你"，把用户当朋友
- **不骚扰用户**：不允许把用户不需要的东西推给用户

### 1.2 产品价值观
```
微信价值观
    ↓
好的产品是创意的
    ↓
好的产品是有用的
    ↓
好的产品是美的
    ↓
好的产品是容易使用的
    ↓
好的产品是含蓄不招摇的
    ↓
好的产品是诚实的
    ↓
好的产品经久不衰
    ↓
好的产品不放过任何细节
    ↓
好的产品是环保的
    ↓
好的产品尽可能少的设计（少即是多）
```

### 1.3 KPI悖论
- KPI是产品成功的副产品
- 当KPI变成目标时，产品会变形
- 好的产品自然带来KPI

## 二、极简主义：少即是多

### 2.1 核心命题
- **减法原则**：不做什么比做什么更重要
- **克制之道**：微信从不做节日运营或logo变化
- **简约美学**：微信界面八年几乎没变

### 2.2 "少"的价值
```
少 = 聚焦
    ↓
聚焦 = 核心功能
    ↓
核心功能 = 用户真正需要的
    ↓
真正需要的 = 长期价值
```

### 2.3 拒绝诱惑
- 拒绝流量变现的诱惑
- 拒绝过度运营的冲动
- 拒绝功能堆砌

## 三、洞察人性：理解用户

### 3.1 核心命题
- **人是社会动物**：社交的本质是找到同类
- **理解惰性**：用户是懒惰的，不要让用户思考
- **尊重习惯**：改变用户习惯是危险的

### 3.2 人性洞察
| 人性特点 | 产品应用 |
|----------|----------|
| 社交需求 | 朋友圈、附近的人 |
| 表达需求 | 公众号、视频号 |
| 效率需求 | 小程序、支付 |
| 安全感需求 | 隐私保护 |

### 3.3 用户不会告诉你他们需要什么
- 乔布斯：用户不知道他们想要什么
- 张小龙：需要观察而非调研
- 数据可以参考，但不能主导

## 四、去中心化：让创造者自主

### 4.1 核心命题
- **平台思维**：微信是水电煤基础设施
- **去中心化**：不给中心化流量，让用户自己发现
- **赋能而非控制**：提供工具，不做裁判

### 4.2 去中心化结构
```
中心化 = 流量集中
    ↓
流量集中 = 少数玩家受益
    ↓
去中心化 = 流量分散
    ↓
流量分散 = 生态繁荣
```

### 4.3 小程序的去中心化
- 微信不设中心化入口
- 用户通过搜索、扫码等自然方式发现
- 让好的小程序自然脱颖而出

## 五、善良比聪明更重要

### 5.1 核心命题
- **AI比你更聪明但可能不如你善良**
- **善良是底线**：不欺骗用户、不利用用户
- **长期主义**：善良的产品有长期价值

### 5.2 善良的具体体现
| 场景 | 善良的选择 |
|------|------------|
| 广告 | 不做开屏广告 |
| 推送 | 不主动推送用户不需要的 |
| 隐私 | 不读取用户聊天记录 |
| 运营 | 不做诱导分享 |

### 5.3 善良的回报
- 用户信任带来长期价值
- 用户愿意花更多时间在微信上
- "你舍不得在你最好的朋友脸上贴广告"

## 六、团队管理：警惕KPI和流程

### 6.1 核心命题
- **小团队效率更高**：150人是上限
- **警惕KPI**：当KPI变成目标时，产品会变形
- **警惕流程**：大团队带来大流程，大流程带来低效率

### 6.2 微信风格
```
小团队 = 高效率
    ↓
高效率 = 快速迭代
    ↓
快速迭代 = 更好的产品
    ↓
更好的产品 = 用户价值
```

### 6.3 领导力原则
- 给方向，不给具体任务
- 让团队自己定目标
- 容忍失败，鼓励创新
