## 一、CLM 核心框架

### 1.1 客户生命周期 5 阶段

```
认知(Awareness) → 考虑(Consideration) → 购买(Purchase) → 留存(Retention) → 推荐(Advocacy)
                                                                      ↓
                                                                   流失(Churn)
                                                                      ↓
                                                                   召回(Win-back)
```

| 阶段 | 核心目标 | 关键指标 | 适用 Agent |
|------|---------|---------|----------|
| 认知 | 品牌曝光 + 人群资产沉淀 | Reach / SOV / AIPL-A 人群 | media / creative |
| 考虑 | 种草 + 互动 + 留资 | CTR / 互动率 / AIPL-I 人群 | kol / content |
| 购买 | 首次转化 | CAC / CVR / AIPL-P 人群 | media / livestream |
| 留存 | 复购 + 会员升级 | 复购率 / 频次 / LTV | data / strategy |
| 推荐 | UGC + 转介绍 | NPS / 推荐率 / K-Factor | kol / content |
| 召回 | 唤醒沉睡用户 | 召回率 / 流失率 | data / media |

## 二、RFM 用户分群模型

### 2.1 RFM 定义

| 维度 | 含义 | 计算方式 | 权重 |
|------|------|---------|:---:|
| **R**ecency | 最近一次购买距今天数 | 当前日期 - 最后订单日期 | 30% |
| **F**requency | 时间窗口内购买次数 | COUNT(订单) | 30% |
| **M**onetary | 时间窗口内消费金额 | SUM(金额) | 40% |

### 2.2 RFM 分群矩阵(8 大人群)

| R | F/M | 人群标签 | 占比参考 | 运营策略 | 触达频率 |
|:-:|:---:|---------|:------:|---------|:------:|
| 高 | 高 | **重要价值用户(VIP)** | 5-10% | 专属服务 / 限量权益 / 1对1 | 月 1-2 次 |
| 高 | 低 | **重要保持用户** | 10-15% | 提频 / 关联推荐 / 会员日 | 周 1 次 |
| 低 | 高 | **重要发展用户** | 10-15% | 流失预警 / 个性化召回 / 大额券 | 触发式 |
| 低 | 低 | **重要挽留用户** | 15-20% | 流失预警 / 重激活 / 问卷调查 | 季度 1 次 |
| 高 | 高(小额) | **一般价值用户** | 10-15% | 提客单 / 升级引导 | 月 2 次 |
| 高 | 低(小额) | **一般保持用户** | 15-20% | 场景拓展 / 内容种草 | 月 1 次 |
| 低 | 高(小额) | **一般发展用户** | 5-10% | 激活 / 新品试用 | 触发式 |
| 低 | 低(小额) | **流失用户** | 15-25% | 低成本触达 / 大促唤醒 | 大促期 |

### 2.3 RFM 评分规则(5 分制)

```python
# R 评分(天数越少分越高)
R_score = 5 if R <= 30 else 4 if R <= 60 else 3 if R <= 90 else 2 if R <= 180 else 1

# F 评分(次数越多分越高，需按品类调整)
# 以美妆为例(年复购 4 次)
F_score = 5 if F >= 8 else 4 if F >= 5 else 3 if F >= 3 else 2 if F >= 2 else 1

# M 评分(金额越高分越高，需按客单价调整)
# 以客单价 200 元为例
M_score = 5 if M >= 3000 else 4 if M >= 1500 else 3 if M >= 800 else 2 if M >= 300 else 1

# RFM 总分 = 0.3*R + 0.3*F + 0.4*M
```

### 2.4 行业 RFM 基准(年)

| 行业 | 平均 R(天) | 平均 F(次) | 平均 M(元) | 优秀 F | 优秀 M |
|------|:--------:|:--------:|:--------:|:-----:|:-----:|
| 美妆护肤 | 90 | 4 | 800 | 8+ | 3000+ |
| 食品饮料 | 30 | 12 | 960 | 24+ | 3000+ |
| 服饰鞋包 | 90 | 4 | 1600 | 8+ | 5000+ |
| 母婴用品 | 60 | 6 | 1800 | 12+ | 5000+ |
| 3C 数码 | 180 | 0.5 | 1500 | 2+ | 5000+ |
| SaaS(B2B) | 365 | 1 | 5万 | 2+ | 15万+ |

## 三、用户标签体系

### 3.1 标签分类框架

```
├── 基础属性标签 (静态)
│   ├── 人口统计: 性别 / 年龄 / 城市 / 收入
│   ├── 设备属性: 机型 / 系统 / 网络
│   └── 来源渠道: 首单渠道 / 注册渠道
│
├── 行为标签 (动态)
│   ├── 浏览行为: 偏好品类 / 浏览时段 / 停留时长
│   ├── 购买行为: RFM / 复购周期 / 客单价
│   ├── 互动行为: 点赞 / 收藏 / 分享 / 评价
│   └── 退货行为: 退货率 / 退货原因
│
├── 价值标签 (计算)
│   ├── LTV 分层: 高 / 中 / 低
│   ├── CAC 回收期: 已回收 / 未回收
│   └── 预测价值: CLV_next_12m
│
└── 心理标签 (推测)
    ├── 价格敏感度: 高 / 中 / 低
    ├── 品牌忠诚度: 忠诚 / 游离 / 价格型
    └── 决策风格: 冲动 / 理性 / 深度研究
```

### 3.2 标签应用场景

| 场景 | 使用标签 | 触发条件 | 推荐动作 |
|------|---------|---------|---------|
| 大促推送 | 价格敏感=高 + 购买频次=低 | 大促前 3 天 | 满减券推送 |
| 新品推荐 | 品类偏好 + LTV=高 | 新品上市 | 优先体验邀请 |
| 流失预警 | R > 2x 平均周期 | 触发式 | 个性化召回 |
| 会员升级 | F 提升 + M 提升 | 月度计算 | 升级提醒 + 权益 |
| 内容推荐 | 浏览偏好 + 互动偏好 | 实时 | 千人千面 |

## 四、会员体系设计

### 4.1 会员等级模型(积分制 / 成长值制)

| 等级 | 门槛 | 权益 | 心理诉求 |
|------|------|------|---------|
| **注册会员** | 注册即得 | 新人礼包 / 生日券 | 归属感 |
| **银卡会员** | 年消费 ≥500 | 9.5 折 / 积分 1.2x | 省钱 |
| **金卡会员** | 年消费 ≥2000 | 9 折 / 积分 1.5x / 专属客服 | 优越感 |
| **铂金会员** | 年消费 ≥5000 | 8.5 折 / 积分 2x / 优先发货 / 限量产品 | 尊贵感 |
| **钻石会员** | 年消费 ≥20000 或邀请制 | 1对1 顾问 / 线下活动 / 定制产品 | 身份认同 |

### 4.2 积分体系设计原则

| 要素 | 建议 | 案例 |
|------|------|------|
| 获取方式 | 购买(1元=1分) + 签到 + 评价 + 转介绍 | 多元化 |
| 使用方式 | 抵现 + 换购 + 抽奖 | 灵活 |
| 过期机制 | 年底清零 / 滚动 12 月 | 制造紧迫感 |
| 积分价值 | 100 积分 ≈ 1 元 | 清晰可感知 |
| 权益感知 | 列表明确展示 | 避免用户无感 |

### 4.3 付费会员(DM/PLUS 模式)

| 要素 | 京东 PLUS | 淘宝 88VIP | Costco |
|------|---------|----------|--------|
| 年费 | 149 元 | 88 元(淘气值≥1000) | 299 元 |
| 核心权益 | 免邮 / 折扣 / 联名 | 优酷 / 饿了仔 / 折扣 | 2% 返现 |
| 目标 | 提频次 + 锁定 | 生态协同 | 复购 + 忠诚 |
| 续费率 | ~80% | ~75% | ~90% |

**付费会员适用条件**:
- ✅ 高频消费品(年消费 ≥10 次)
- ✅ 客单价稳定(非冲动消费)
- ✅ 有差异化权益(非单纯打折)
- ❌ 不适合: 低频 / 高客单 / 一次性消费

## 五、流失预警与召回

### 5.1 流失预警模型

#### 流失定义(按品类)

| 品类 | 正常复购周期 | 预警周期 | 流失判定 |
|------|:---------:|:------:|:------:|
| 美妆 | 60-90 天 | >120 天 | >180 天 |
| 食品 | 20-30 天 | >45 天 | >60 天 |
| 服饰 | 60-120 天 | >150 天 | >240 天 |
| SaaS | 按年 | 续约前 60 天 | 续约失败 |

#### 流失评分模型(Logistic Regression 示例)

```python
import pandas as pd
from sklearn.linear_model import LogisticRegression

# 特征工程
features = df[['recency', 'frequency', 'monetary', 
               'sessions_last_30d', 'email_open_rate',
               'last_login_days', 'complaint_count']]

# 标签: 1=已流失, 0=活跃
target = df['is_churned']

# 训练
model = LogisticRegression()
model.fit(features, target)

# 预测流失概率
df['churn_probability'] = model.predict_proba(features)[:, 1]

# 分级
df['churn_risk'] = pd.cut(df['churn_probability'],
                           bins=[0, 0.3, 0.6, 1.0],
                           labels=['低风险', '中风险', '高风险'])
```

### 5.2 召回策略矩阵

| 流失类型 | 触发条件 | 召回方式 | 成本 | 预期召回率 |
|---------|---------|---------|:---:|:--------:|
| 短期沉睡 | R > 1x 周期 | 个性化短信 / 邮件 | 低 | 15-25% |
| 中期沉睡 | R > 2x 周期 | 优惠券 + 推送 | 中 | 8-15% |
| 长期沉睡 | R > 3x 周期 | 大额券 + 电话 | 高 | 3-8% |
| 彻底流失 | R > 4x 周期 | 放弃 / 换新客 | - | <3% |

#### 召回文案模板

```markdown
### 短期沉睡(温馨)
"亲爱的 [昵称]，我们已经 [X] 天没见了。你之前关注的 [品类/产品] 
正在限时优惠，[券金额] 元专属券已发放到你的账户。"

### 中期沉睡(利益)
"[昵称]，你的会员等级即将在 [X] 天后降级。
立即下单保级，还可使用 [大额券] 元回归专属券。"

### 长期沉睡(恳求)
"我们想你啦！
[昵称]，回归即送 [大额券] + 免邮特权 + [热门单品] 1 元购。"

### 调研型召回(诊断)
"我们想做得更好。
花 3 分钟告诉我们为什么不再下单，即送 [券金额] 元感谢礼。"
```

## 六、NPS & CSAT 客户满意度体系

### 6.1 NPS (Net Promoter Score)

#### 计算公式
```
NPS = % 推荐者(9-10分) - % 贬损者(0-6分)
     (忽略 7-8 分的被动者)
```

#### NPS 等级标准

| NPS 范围 | 等级 | 行业参考 |
|:--------:|------|---------|
| >50 | 卓越 | Apple(72), Amazon(73) |
| 30-50 | 优秀 | 银行(35), 电商(40) |
| 10-30 | 良好 | SaaS(30), 电信(15) |
| 0-10 | 一般 | 快递(5), 保险(10) |
| <0 | 危险 | 航空(-5) |

#### NPS 调研问卷模板

```markdown
Q1: 你有多大意愿向朋友/同事推荐 [品牌]？(0-10 分)

Q2(推荐者 9-10): 您最看重 [品牌] 的什么？
Q2(被动者 7-8): 我们怎么做能让您更愿意推荐？
Q2(贬损者 0-6): 您不满意的主要原因是什么？(多选)
  □ 产品质量  □ 服务态度  □ 价格  □ 物流  □ 售后
  □ 品牌形象  □ 其他: ___
```

### 6.2 CSAT (Customer Satisfaction Score)

#### 应用场景

| 触点 | 调研时机 | 问题 | 等级 |
|------|---------|------|:---:|
| 购买后 | 订单完成 7 天 | "对本次购物体验满意吗？" | 1-5 星 |
| 售后后 | 客服关闭工单 | "问题是否解决？" | Y/N + 1-5 星 |
| 交付后 | 收货 3 天 | "产品是否符合预期？" | 1-5 星 |
| 内容后 | 文章/视频播放完 | "内容有帮助吗？" | Y/N |

### 6.3 CES (Customer Effort Score)

```
CES = 平均分(1-7) — 越低越好
"为了完成 [任务]，你付出了多少努力？"
1=非常轻松  7=非常困难
```

**适用**: 评估服务体验(售后 / 退换货 / 找客服)。

## 七、LTV 计算模型

### 7.1 简单 LTV(历史法)

```
LTV = 平均客单价 × 年复购次数 × 平均生命周期(年) × 毛利率
```

### 7.2 预测 LTV(Beta-Geometric / BG/NBD 模型)

```python
# 使用 lifetimes 库
from lifetimes import BetaGeoFitter
from lifetimes.utils import summary_data_from_transaction_data

# 数据准备
data = summary_data_from_transaction_data(
    df, 
    customer_id_col='user_id',
    datetime_col='order_date',
    observation_period_end=max(df['order_date'])
)

# 训练模型
bgf = BetaGeoFitter(penalizer_coef=0.0)
bgf.fit(data['frequency'], data['recency'], data['T'])

# 预测未来 12 个月购买次数
data['predicted_purchases_12m'] = bgf.conditional_expected_number_of_purchases_up_to_time(
    365, data['frequency'], data['recency'], data['T']
)

# 预测 LTV
from lifetimes import GammaGammaFitter
ggf = GammaGammaFitter(penalizer_coef=0.0)
ggf.fit(data['frequency'], data['monetary_value'])

data['predicted_ltv_12m'] = ggf.customer_lifetime_value(
    bgf, data['frequency'], data['recency'], data['T'],
    data['monetary_value'], time=12, discount_rate=0.01
)
```

### 7.3 LTV 应用场景

| 场景 | LTV 用途 | 决策阈值 |
|------|---------|---------|
| 投放获客 | LTV/CAC ≥ 3:1 | <3 暂停渠道 |
| 会员升级 | LTV 提升 ≥20% | <20% 不值得投入 |
| 流失预警 | 召回成本 ≤ LTV×10% | >10% 放弃召回 |
| 品类拓展 | LTV 预测提升 | 评估可行性 |
| 渠道优化 | 按渠道 LTV 分配预算 | 高 LTV 渠道加权 |

## 八、私域 LTV 计算实操

### 8.1 私域 vs 公域 LTV 差异

| 维度 | 公域 LTV | 私域 LTV |
|------|---------|---------|
| 触达成本 | 每次付费 | 0 边际成本 |
| 复购率 | 行业基准 | 公域的 2-3x |
| 客单价 | 基准 | +20-40% |
| LTV 周期 | 1-3 年 | 3-5 年 |

### 8.2 私域 LTV 计算公式

```
私域 LTV = (客单价 × 年复购次数 + 关联消费 + 转介绍价值) × 客户生命周期 × 毛利率

其中:
- 关联消费 = 客单价 × 交叉销售率(20-30%)
- 转介绍价值 = 新客 CAC × 年转介绍数 × 转化率
```

### 8.3 私域价值评估表

| 用户类型 | 年贡献 | 3 年 LTV | 运营成本 | 净 LTV |
|---------|:-----:|:-------:|:------:|:-----:|
| 普通群成员 | 800 | 2400 | 50 | 2350 |
| 活跃群成员 | 2000 | 6000 | 100 | 5900 |
| 朋友圈私域 | 3500 | 10500 | 200 | 10300 |
| 1对1 专属 | 8000 | 24000 | 500 | 23500 |

## 九、CDP 选型与集成

### 9.1 CDP 核心能力

```
数据采集 → 身份合并 → 标签计算 → 人群圈选 → 触达执行 → 效果回流
```

### 9.2 CDP vs CRM vs DMP

| 维度 | CRM | DMP | CDP |
|------|:---:|:---:|:---:|
| 数据来源 | 一手交易 | 三方 cookie | 一手 + 行为 |
| 用户类型 | 已购客户 | 匿名 | 全量(已知+匿名) |
| 实时性 | T+1 | T+1 | 实时 |
| 应用场景 | 销售/服务 | 广告投放 | 全场景 |

### 9.3 主流 CDP 对比(2025-2026)

| 厂商 | 适合规模 | 核心优势 | 价格 |
|------|---------|---------|:---:|
| 阿里数据银行 | 天猫生态 | 电商原生 | 按消耗 |
| 神策数据 | 中大型 | 私有化部署 | 20-100万/年 |
| 火山引擎 | 抖音生态 | 字节系打通 | 按消耗 |
| GrowingIO | 中型 | 行为分析强 | 10-50万/年 |
| 易观方舟 | 中型 | 性价比 | 5-30万/年 |
| Segment(海外) | 全球化 | 集成丰富 | $120k+/年 |

### 9.4 CDP 集成建议

| 企业阶段 | 建议 |
|---------|------|
| 初创(<1000 万 GMV) | 不需要 CDP，用 Excel + 企业微信即可 |
| 成长(1000 万-1 亿) | 用 SaaS 型(易观方舟 / GrowingIO) |
| 成熟(1-10 亿) | 私有化部署(神策) + 阿里数据银行 |
| 大型(10 亿+) | 自建 CDP + 多源数据融合 |
