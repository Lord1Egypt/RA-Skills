# Example: Landing Page UI Brief

## Prompt

"I want a landing page for an AI note-taking tool for consultants. The goal is demo bookings, but I do not know how to organize the page."

## Recommended Route

- Scenario: landing page
- Stage: UI Brief
- Optional companion: `ui-ux-pro-max` for stronger conversion and style direction

## Example Output Shape

### Project Summary
- **Product / page name:** AI note-taking tool landing page
- **Product type:** B2B SaaS landing page
- **Platform:** Responsive web

### Target User
- **Primary audience:** Consultants who run many client calls
- **User maturity:** Professional
- **Main user need:** Capture, summarize, and share meeting outcomes with less manual cleanup

### Main Goal
- **Primary outcome:** Book qualified demos
- **Primary CTA:** Book a demo
- **What success looks like:** Visitors quickly understand the product, trust it, and take action

### Core Content
- Hero with clear value proposition
- Social proof or trust indicators
- Problem section
- Product workflow overview
- Benefits for consultants
- Screenshots or demo preview
- FAQ
- Final CTA

### Recommended Next Step
- Move to Design Direction with a conversion-first strategy.
- If implementation begins, create `DESIGN.md` to record the selected direction.
