# 商业模式画布 🎯

## 核心流程（5 Steps）

### Step 1: 商业模式诊断
- 了解产品/服务核心：你在卖什么？解决什么问题？
- 明确当前阶段：想法验证/已有用户/规模化
- 识别商业模式类型：SaaS/平台/电商/广告/订阅/交易抽佣
- 初步判断：这个生意最大的风险在哪？

### Step 2: 九大模块填充
按顺序构建商业模式画布：
1. **客户细分**（Customer Segments）：为谁创造价值？
2. **价值主张**（Value Propositions）：解决什么痛点？
3. **渠道通路**（Channels）：怎么触达客户？
4. **客户关系**（Customer Relationships）：如何获取/留存/增长？
5. **收入来源**（Revenue Streams）：怎么赚钱？
6. **核心资源**（Key Resources）：需要什么资产？
7. **关键业务**（Key Activities）：必须做好什么？
8. **重要伙伴**（Key Partnerships）：谁帮你完成？
9. **成本结构**（Cost Structure）：花钱在哪？

### Step 3: 商业模式验证
- 检查九大模块一致性（价值主张↔客户细分是否匹配）
- 识别关键假设（哪些是未验证的猜想）
- 评估可行性：市场/技术/财务三维验证
- 标注风险等级：高/中/低

### Step 4: 优化建议
- 根据商业模式类型提供优化方向
- 识别可砍掉的成本和可放大的收入
- 建议MVP验证方案（最小成本验证核心假设）
- 提供对标案例参考

### Step 5: 画布输出
- 完整商业模式画布（9模块表格）
- 核心假设清单 + 验证计划
- 商业模式类型标签
- 一句话商业逻辑总结

## 常见商业模式类型速查

| 类型 | 收入模型 | 关键指标 | 代表企业 |
|------|----------|----------|----------|
| SaaS订阅 | 月/年费 | MRR/Churn/NDR | Notion/Figma |
| 平台抽佣 | 交易额% | GMV/Take Rate | 美团/Airbnb |
| 免费增值 | 高级版付费 | 转化率/DAU | Dropbox/Zoom |
| 广告模式 | 广告展示/点击 | DAU/eCPM | Google/抖音 |
| 电商直营 | 商品差价 | 客单价/毛利率 | SHEIN/安克 |
| 内容付费 | 课程/会员 | 付费率/ARPU | 得到/Netflix |
| API经济 | 按调用量计费 | 调用量/留存 | Stripe/Twilio |

## 边界约束
1. **诚实评估**：不回避商业模式的致命缺陷
2. **数据支撑**：尽量引用同类企业数据做参考
3. **可执行性**：建议需匹配当前团队资源和阶段
4. **动态视角**：商业模式需要持续迭代而非一成不变

## 单元经济模型（Unit Economics）

### 核心公式
- **LTV（客户终身价值）**= ARPU × 毛利率 × 平均生命周期
- **CAC（获客成本）**= 营销总费用 ÷ 新增付费用户数
- **LTV/CAC ≥ 3** 为健康值
- **回收期** = CAC ÷ (ARPU × 毛利率)，≤12个月为佳

### 各模式关键指标
| 模式 | 北极星指标 | 健康基准 |
|------|-----------|----------|
| SaaS | NDR（净收入留存率）| >110% |
| 平台 | Take Rate + GMV增速 | 3-25% / >30% YoY |
| 免费增值 | Free→Paid转化率 | >5% |
| API经济 | 月调用量留存 | >85% |

## 竞争护城河分析（5种类型）
1. **网络效应**：用户越多价值越大（微信/Uber）
2. **转换成本**：用户迁移代价高（Salesforce/Oracle）
3. **规模经济**：成本随规模递减（AWS/京东物流）
4. **品牌资产**：心智占位+溢价能力（Apple/茅台）
5. **数据壁垒**：独有数据+算法飞轮（Google/字节）

**评估方法**：你的项目命中几种？0种=无壁垒，1种=有基础，2种+=有竞争力

## Anti-rationalization

| 借口 | 正确做法 |
|------|----------|
| "9个模块我只需要填几个关键的就行" | 必须完整填充全部9个模块，每个模块都有实质内容，不能留空或写"暂无" |
| "单元经济模型算不出来，没有数据" | 必须基于行业基准数据做合理估算，标注为"估算值"，不能跳过LTV/CAC/LTV-CAC比率计算 |
| "护城河分析写'技术领先'就行了" | 必须从5种护城河类型（网络效应/转换成本/规模经济/品牌/数据壁垒）逐一评估，命中几种说几种 |
| "成本结构写个总数就行，不用细分" | 成本结构必须区分固定成本和可变成本，标注各项占比，识别最大成本项 |
| "关键假设列一两条就够了" | 必须识别至少3-5个关键假设，每个标注风险等级和验证方法，高风险假设必须有MVP验证方案 |
| "对标案例随便提一个名字就行" | 对标案例必须说明可借鉴的具体点是什么，与当前项目的相似性和差异性 |
| "商业模式类型不用标，一目了然" | 必须明确标注商业模式类型标签（SaaS/平台/电商等），并用一句话总结商业逻辑 |
