# Competitor Radar — Quality Checklist

Use this checklist before delivering any competitor analysis output.

---

## Data Collection (8 items)

- [ ] Competitor identity confirmed — correct product, variant, and seller matched
- [ ] All data points timestamped with collection date (UTC)
- [ ] Platform identified and platform-specific analysis applied
- [ ] Organic vs sponsored listings distinguished in search visibility analysis
- [ ] Data collection method stated (browser observation, user-provided, third-party tool)
- [ ] Number of data points per competitor stated explicitly
- [ ] Stale data (>7 days old) flagged with freshness warning
- [ ] Your baseline position documented before competitor analysis begins

## Evidence Integrity (7 items)

- [ ] Every claim tagged with evidence strength: observed / inferred / estimated
- [ ] No fabricated BSR, sales velocity, conversion rate, or ad spend data
- [ ] No fabricated price history or trend claims without supporting data
- [ ] "Full market" language avoided unless coverage demonstrated
- [ ] Partial data acknowledged with explicit coverage estimate
- [ ] Each factual claim traceable to a specific, timestamped observation
- [ ] Snapshot data not presented as trend data

## Price Normalization (6 items)

- [ ] All prices converted to same currency with rate stated
- [ ] Multi-pack prices divided to per-unit cost
- [ ] Shipping costs included in landed price calculations
- [ ] Subscribe & Save / auto-replenishment prices noted separately
- [ ] Coupons and promotions reflected in effective price
- [ ] New vs refurbished/used listings not mixed in comparisons

## Competitive Context (7 items)

- [ ] Competitor rating distribution included (not just average)
- [ ] Review velocity calculated and stated (reviews per month)
- [ ] Seller maturity and market tenure noted
- [ ] Brand positioning differences acknowledged (premium vs budget vs generic)
- [ ] Active promotions flagged as temporary conditions
- [ ] Fulfillment method differences accounted for in price comparisons
- [ ] At least 3 competitors included for Mode B landscape scans

## Analysis Quality (8 items)

- [ ] All 6 analysis categories addressed (listing, pricing, reviews, promotions, search, brand)
- [ ] Competitive Positioning Map completed with scores for all dimensions
- [ ] Threat Assessment table completed with rationale for each category
- [ ] Each recommendation is specific and actionable (not "monitor the market")
- [ ] Recommendations ranked by Impact / Effort / Urgency
- [ ] Margin floor respected — no recommendation breaches floor without explicit warning
- [ ] Competitive response modeled for pricing recommendations
- [ ] Whitespace opportunities identified and validated

## Output Quality (5 items)

- [ ] Correct output template used (Mode A or Mode B)
- [ ] Executive summary is 5 lines or fewer
- [ ] All tables properly formatted with no missing cells
- [ ] Next review date and trigger events specified
- [ ] Anomalies and data limitations documented

## Safety and Honesty (4 items)

- [ ] No claims of access to private marketplace analytics
- [ ] Competitor actions described factually without value judgments
- [ ] Platform-specific data limitations acknowledged
- [ ] Recommendations account for competitor response scenarios

---

**Total items: 45**
**Minimum passing: 40/45 checked**
