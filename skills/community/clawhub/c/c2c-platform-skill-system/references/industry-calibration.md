# C2C本地生活服务平台 — 行业校准

校准机制：本技能专为C2C本地生活服务平台设计，校准维度覆盖平台运营全链路。当平台模式存在差异时（如B2C、O2O、纯信息撮合等），按以下流程调整：

1. 识别平台类型（纯C2C / B2C2C / C2B2C / 混合）
2. 识别交易模式（虚拟币 / 法币直付 / 混合）
3. 识别增长模式（裂变分润 / 投放拉新 / 自然增长）
4. 识别履约模式（实物 / 线下服务 / 线上服务 / 混合）
5. 按差异调整对应集群的术语、指标和流程

## 变体校准

### 变体A：B2C2C混合平台（有入驻商家+个人服务商）

| 差异维度 | 调整内容 |
|----------|----------|
| C1 情报 | 增加：商家竞品分析、供应链价格监测 |
| C2 内容 | 增加：商家店铺装修、商品详情页规范 |
| C3 活动 | 增加：商家联合营销、品牌日活动 |
| C4 分析 | 增加：商家GMV排名、商家流失预警 |
| C6 合规 | 增加：商家资质审核、食品/特种行业许可 |
| C7 供需 | 增加：商家分级管理、商家培训体系 |
| C9 准入 | 增加：商家入驻审核流程、行业特殊许可 |

### 变体B：纯信息撮合平台（不涉及资金流转）

| 差异维度 | 调整内容 |
|----------|----------|
| C4 分析 | 移除：交易闭环指标，替换为：联系转化率、线索质量评分 |
| C6 合规 | 移除：支付合规、反洗钱，替换为：信息安全等级保护 |
| C7 供需 | 移除：钱包/冻结/结算，替换为：线索分配、联系促成 |
| C9 准入 | 移除：EDI许可证(非交易类不需要)，替换为：ICP备案+公安备案 |
| C10 保障 | 移除：支付系统，替换为：线索防爬虫、联系信息安全 |

### 变体C：垂直品类平台（仅做某一品类如家政/家教）

| 差异维度 | 调整内容 |
|----------|----------|
| C1 情报 | 聚焦：垂直品类竞品、品类专业标准 |
| C2 内容 | 深化：品类专业知识库、服务项目标准化 |
| C4 分析 | 增加：品类季节性分析、服务时长分析 |
| C5 知识 | 深化：品类专业技能认证、行业资质要求 |
| C6 合规 | 增加：品类特殊合规(如家政需家政服务员证、家教需教师资格) |
| C7 供需 | 增加：品类技能认证、资质验证 |

## 自定义校准流程

当平台模式不在以上变体中时：

1. **就近匹配**：找到最接近的变体（A/B/C）作为基础
2. **差异识别**：列出与基础变体的3-5个核心差异点
3. **维度调整**：按差异点修改对应集群的校准维度
4. **验证覆盖**：确认11个维度均已覆盖，无遗漏
5. **迭代修正**：首次执行后根据实际业务反馈微调
