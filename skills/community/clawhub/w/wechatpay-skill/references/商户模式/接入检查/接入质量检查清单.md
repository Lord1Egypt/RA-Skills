# 接入质量检查清单

> 上线前必查！按模块逐项核对

## 🔴 致命问题（必须修复）

| 检查项 | 说明 |
|--------|------|
| ✅ **签名私钥不泄露** | 私钥必须通过环境变量或密钥管理服务加载，禁止硬编码在代码中 |
| ✅ **订单号全局唯一** | out_trade_no每笔都要生成唯一值，避免重复下单 |
| ✅ **退款幂等** | 退款失败重试时必须复用原out_refund_no，换新号会退两次 |
| ✅ **金额用整数（分）** | 金额必须用整数（单位：分），禁止用浮点数计算 |
| ✅ **签名算法正确** | JSAPI调起签名第四行是`prepay_id=xxx`，APP是纯prepay_id |

## 必须实现的功能

### 基础支付

| 功能 | 接口 | 说明 |
|------|------|------|
| 下单 | POST /v3/pay/transactions/{type} | 根据支付方式选jsapi/native/app/h5 |
| 调起支付 | 前端调用 | 签名串构造必须正确 |
| 支付成功回调 | notify_url | 必须验签后处理业务 |
| 商户订单号查单 | GET /v3/pay/transactions/out-trade-no/{} | 必须实现 |
| 退款申请 | POST /v3/refund/domestic/refunds | 必须验重 |
| 退款回调 | 接收退款状态通知 | 必须处理 |
| 查询退款 | GET /v3/refund/domestic/refunds/{} | 必须实现 |
| 关闭订单 | POST /v3/pay/transactions/.../close | 超时订单必须关单 |

### 小程序支付专项

| 检查项 | 说明 |
|--------|------|
| ✅ AppID已认证 | 个人小程序不支持支付 |
| ✅ mchid与AppID绑定 | 商户平台手动绑定 |
| ✅ openid获取正确 | 通过wx.login获取code再换openid |
| ✅ 调起签名正确 | 使用小程序AppID |

## 回调处理检查

| 检查项 | 必须做到 |
|--------|----------|
| 验签 | 收到回调后先验签，验签失败不处理业务 |
| 幂等 | 同一通知只处理一次，避免重复发货/加款 |
| 金额校验 | 回调金额必须与本地订单金额一致 |
| 状态校验 | 只处理SUCCESS状态的支付回调 |

## 安全检查

| 检查项 | 要求 |
|--------|------|
| 私钥加载 | 从环境变量或KMS加载，不硬编码 |
| 证书序列号 | 使用当前有效的证书序列号 |
| 回调IP白名单 | 只处理来自微信支付IP段的回调 |
| 日志脱敏 | 日志中禁止打印敏感信息（密钥、完整卡号等） |

## 分账检查（如使用）

| 检查项 | 说明 |
|--------|------|
| 下单时传profit_sharing=true | 否则无法分账 |
| 支付成功后1分钟才能分账 | 需等待资金到位 |
| 分账总额不超过可分账金额 | 需扣除微信手续费 |
| 不分账也要调用解冻 | 否则资金冻结30天 |
