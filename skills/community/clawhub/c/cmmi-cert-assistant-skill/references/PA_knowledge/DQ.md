# DQ — 数据质量（CMMI V3.0 Domain-specific PA）

## PA 基本信息
- **缩写**：DQ
- **英文**：Data Quality
- **中文**：数据质量
- **所属**：Domain-specific PA（领域特定实践域）
- **适用领域**：数据管理、信息服务
- **与DM关系**：DQ是DM的具体实践，专注于数据质量测量和改进

---

## CMMI V3.0 要求要点

### 核心目的
确保数据满足其预期用途的质量要求。

### 关键实践（Specific Practices）
| 实践编号 | 实践内容 | 说明 |
|---------|---------|------|
| DQ 1.1 | 定义数据质量需求 | 基于业务需求定义数据质量维度 |
| DQ 1.2 | 测量数据质量 | 客观测量数据质量 |
| DQ 1.3 | 分析数据质量结果 | 识别根本原因和改进机会 |
| DQ 1.4 | 定义数据质量改进措施 | 制定纠正和预防措施 |
| DQ 2.1 | 实施数据质量改进 | 执行改进措施 |
| DQ 2.2 | 验证数据质量改进效果 | 确认改进措施的有效性 |
| DQ 3.1 | 建立数据质量文化 | 培养数据质量意识和责任 |

### 通用实践（Generic Practices）
- GP 1.1 执行特定实践
- GP 2.1 建立组织政策
- GP 2.2 策划过程
- GP 2.3 提供资源
- GP 2.4 分配责任
- GP 2.5 培训人员
- GP 2.6 管理配置
- GP 2.7 识别并记录干系人需求和接口
- GP 2.8 监督和控制过程
- GP 2.9 客观评估符合性
- GP 2.10 高层审查

## 用户文档 → PA 匹配规则

### 高概率匹配（主要PA）
- 文档名含"数据质量"、"Data Quality"、"数据清洗" → **DQ 主要匹配**
- 文档含数据质量测量记录、质量报告 → **DQ 主要匹配**

### 中概率匹配（次要PA）
- 文档含数据治理策略 → 同时匹配 **DM**（数据管理）
- 文档含过程质量指标 → 同时匹配 **PCM**（过程质量管理）

## 差距检查清单（用于差距分析）

### DQ 1.x — 数据质量策划
- [ ] 是否定义了数据质量维度（准确性、完整性等）？
- [ ] 是否有数据质量测量计划？
- [ ] 是否有数据质量测量方法（规则、工具）？
- [ ] 是否识别了关键数据质量风险？

### DQ 2.x — 数据质量改进
- [ ] 是否有数据质量测量记录？
- [ ] 是否有数据质量分析报告？
- [ ] 是否有数据质量改进措施？
- [ ] 是否有改进效果验证记录？

### DQ 3.x — 数据质量文化
- [ ] 是否有数据质量培训？
- [ ] 是否定义了数据质量角色和职责？
- [ ] 是否有数据质量激励机制？

## 1. 引言

### 1.3 术语与缩略语

（数据在业务决策中的作用）

（准确性、完整性、一致性、及时性、唯一性、有效性）

## 3. 数据质量维度定义
| 质量维度 | 定义 | 测量方法 | 可接受标准 | 测量频率 |

## 4. 数据质量测量计划

（需要测量的数据资产清单）

（每个数据资产的质量责任人）

|--------|---------|---------|---------|--------|--------------|--------|

## 7. 数据质量报告

## 8. 数据质量文化

（数据质量意识和技能培训）
### 8.2 角色与职责
（数据质量相关的角色和职责）
### 8.3 激励机制
（数据质量改进的激励措施）

## 9. 持续改进计划
（基于测量结果和业务变化的持续改进）

## 10. 版本记录
（版本号、修订日期、修订内容、修订人、审批人）

## 评估访谈常见问题（模式D）

### 针对数据管理人员的问题
1. "你们如何定义数据质量？有哪些质量维度？"
2. "数据质量是如何测量的？有测量记录吗？"
3. "发现数据质量问题后如何改进？有跟踪记录吗？"
4. "如何验证数据质量改进的效果？"

### 针对数据用户的问题
1. "你使用的数据质量如何？有没有质量问题？"
2. "发现数据质量问题后向谁反馈？如何处理？"
3. "数据质量对业务决策有什么影响？"

### 评分要点
- **DQ 1.1-1.4**：是否有系统的数据质量定义和测量计划
- **DQ 2.1-2.2**：数据质量改进是否有措施和效果验证
- **DQ 3.1**：是否建立了数据质量文化和责任意识
