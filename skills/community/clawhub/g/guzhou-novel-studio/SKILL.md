# 孤舟小说工作室（Guzhou Novel Studio）

> **版本**：1.0.0  
> **更新日期**：2026-04-29  
> **定位**：整合创作生态的智能路由系统

---

## 1. 系统概述

孤舟小说工作室是一个统一的创作技能体系，整合了调研、创作、风格、质检、长篇管理五大核心模块。通过智能路由机制，自动识别用户意图并调度最合适的工具。

### 1.1 核心能力

| 模块 | 能力 | 数据格式 |
|------|------|---------|
| **调研模块** | 开题调研、续写调研、改写调研 | JSON |
| **创作模块** | 短篇、中篇、长篇、技法应用 | Markdown |
| **风格模块** | 风格提取、风格保持、风格进化 | JSON |
| **质检模块** | 一致性检查、逻辑验证、问题追踪 | Markdown |
| **长篇管理** | 项目初始化、章节追踪、记忆同步 | JSON/Markdown |

## 2. 触发词与路由规则

### 2.1 场景识别矩阵

| 触发关键词 | 识别场景 | 路由目标 |
|-----------|---------|---------|
| "写一个故事"、"短篇"、"2000字内" | 短篇创作 | 创作模块-短篇 |
| "中篇"、"8000字"、"1万字" | 中篇创作 | 创作模块-中篇 |
| "长篇"、"连载"、"第X章"、"章节" | 长篇创作 | 长篇管理+创作模块 |
| "续写"、"继续写"、"下一章" | 续写场景 | 长篇管理→创作模块 |
| "改写"、"修改"、"润色" | 改写优化 | 质检模块→创作模块 |
| "风格"、"模仿"、"像...一样" | 风格设定 | 风格模块 |
| "开题"、"调研"、"卡文" | 调研场景 | 调研模块 |
| "检查"、"质检"、"问题" | 质检场景 | 质检模块 |
| "人物"、"设定"、"大纲" | 设定场景 | 长篇管理模块 |

### 2.2 优先级规则

```
优先级1：长篇创作（明确包含"长篇"、"章节"）
    ↓
优先级2：续写/续章（明确包含"续写"、"继续"、"下一章"）
    ↓
优先级3：改写优化（明确包含"改写"、"修改"）
    ↓
优先级4：质检审核（明确包含"检查"、"审核"、"质检"）
    ↓
优先级5：其他场景
```

### 2.3 手动切换命令

| 命令 | 效果 | 示例 |
|------|------|------|
| `/调研 <主题>` | 切换到调研模块 | `/调研 都市言情` |
| `/创作 <类型>` | 切换到创作模块 | `/创作 短篇` |
| `/风格 <风格名>` | 切换到风格模块 | `/风格 说书人` |
| `/质检` | 切换到质检模块 | `/质检` |
| `/长篇` | 切换到长篇管理 | `/长篇` |
| `/记忆` | 查看/操作记忆系统 | `/记忆 查看` |
| `/状态` | 查看当前项目状态 | `/状态` |

## 3. 智能路由逻辑

### 3.1 路由决策流程

```
用户输入
    │
    ▼
┌─────────────────┐
│  关键词匹配分析  │
└─────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────┐
│                  场景判断                        │
├─────────────────────────────────────────────────┤
│                                                 │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐         │
│  │ 短篇   │  │ 中篇    │  │ 长篇    │         │
│  │ 场景    │  │ 场景    │  │ 场景    │         │
│  └────┬────┘  └────┬────┘  └────┬────┘         │
│       │            │            │               │
│       ▼            ▼            ▼               │
│  ┌─────────────────────────────────┐            │
│  │       路由目标确定              │            │
│  ├─────────────────────────────────┤            │
│  │ 短篇 → 创作模块(短篇)           │            │
│  │ 中篇 → 创作模块(中篇)           │            │
│  │ 长篇 → 长篇管理 + 创作模块       │            │
│  └─────────────────────────────────┘            │
│                      │                          │
└──────────────────────┼──────────────────────────┘
                       │
        ┌──────────────┼──────────────┐
        ▼              ▼              ▼
   ┌─────────┐   ┌─────────┐   ┌─────────┐
   │ 调研模块 │   │ 风格模块 │   │ 质检模块 │
   │ (可选)   │   │ (可选)   │   │ (可选)   │
   └─────────┘   └─────────┘   └─────────┘
```

### 3.2 复合场景处理

当用户输入包含多个关键词时，按优先级执行：

```
用户：先做个市场调研，然后写一个都市言情的短篇
→ [路由分析] 检测到调研+短篇创作场景
→ 先执行调研模块 → 再路由到创作模块

用户：第一章写完了，帮我检查一下
→ [路由分析] 检测到质检场景
→ 路由到质检模块 → 输出质检报告

用户：质检通过了吗？我想继续写第二章
→ [路由分析] 质检完成，检测到续写场景
→ 加载记忆包 → 路由到创作模块写第二章
```

## 4. 模块调用说明

### 4.1 调研模块 (modules/researcher/)

```markdown
路径：modules/researcher/
功能：市场调研、竞品分析、题材研究
输入：用户创作需求
输出：research-report.json
```

### 4.2 创作模块 (modules/creator/)

```markdown
路径：modules/creator/
功能：短篇/长篇/盐选创作、技法应用
输入：需求+风格设定+大纲
输出：章节正文.md

引用工具：
- novel-creation skill（短篇/长篇创作）
- qingshan-novel-writing skill（技法支持）
```

### 4.3 风格模块 (modules/stylist/)

```markdown
路径：modules/stylist/
功能：风格提取、风格保持、风格进化
输入：风格样本
输出：Style-DNA.json

子模块：
- novel-style-reference/（风格参考工具）
- yunhui-style/（云辉风格工具）
```

### 4.4 质检模块 (modules/qa/)

```markdown
路径：modules/qa/
功能：一致性检查、OOC检测、逻辑验证
输入：章节正文+记忆包
输出：qa-report.md
```

### 4.5 长篇管理模块 (modules/studio/)

```markdown
路径：modules/studio/
功能：项目初始化、章节追踪、记忆同步
输入：大纲+章节
输出：追踪文件.md

参考：references/file-format-spec.md
```

## 5. 数据流转规范

### 5.1 模块间数据格式

| 流转方向 | 数据格式 | 说明 |
|---------|---------|------|
| 调研→创作 | JSON | 调研报告 |
| 创作→质检 | Markdown | 章节正文 |
| 质检→修改 | Markdown | 质检报告 |
| 创作→长篇管理 | JSON | 章节摘要 |

### 5.2 核心规则

- **JSON优先**：模块间数据传递使用JSON格式
- **Markdown输出**：面向用户的输出使用Markdown
- **记忆同步**：每个创作循环后必须同步更新记忆系统
- **版本记录**：每次重大修改记录版本号
