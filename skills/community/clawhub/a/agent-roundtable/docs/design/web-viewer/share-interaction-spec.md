## 1. 分享链接区域

### 位置

在页面顶部导航栏右侧（桌面端）或底部操作栏内（移动端）。

### 布局

```
┌─────────────────────────────────────────────────────┐
│  [LIVE]  圆桌讨论                        [🔗 分享]  │
└─────────────────────────────────────────────────────┘
```

桌面端：分享按钮位于 top-bar 右侧。
移动端：分享按钮固定在底部操作栏。

## 2. 链接复制交互

### 2.1 触发方式

点击「分享」按钮，弹出分享面板（Popover / Bottom Sheet）。

### 2.2 分享面板 — 桌面端（Popover）

```
┌──────────────────────────────────────────┐
│  📋 分享链接                              │
│  ┌──────────────────────────┐ [复制]     │
│  │ http://host:8199/r/abc123│            │
│  └──────────────────────────┘            │
│  任何人通过此链接可查看讨论                 │
│                                          │
│  ⚠️ [撤销链接]                            │
└──────────────────────────────────────────┘
```

- 宽度：360px
- 弹出方向：从分享按钮下方弹出
- 点击面板外区域关闭
- ESC 键关闭

### 2.3 分享面板 — 移动端（Bottom Sheet）

```
┌──────────────────────────────────┐
│         ━━━ (拖拽条)              │
│                                  │
│  📋 分享链接                      │
│  ┌──────────────────────────┐    │
│  │ http://host:8199/r/abc123│    │
│  └──────────────────────────┘    │
│  [        复制链接        ]       │
│                                  │
│  任何人通过此链接可查看讨论        │
│                                  │
│  ⚠️ 撤销链接                     │
└──────────────────────────────────┘
```

- 从底部滑入
- 最大高度：60vh
- 向下拖拽或点蒙层关闭

### 2.4 复制成功反馈

点击「复制」按钮后的交互流程：

```
状态流转：
[复制] --点击--> [✓ 已复制] --2s后--> [复制]
         (颜色变绿)           (恢复默认)
```

- 按钮文字：「复制」→「✓ 已复制」
- 按钮颜色：品牌蓝 `#4F46E5` → 成功绿 `#22C55E`
- 持续时间：2 秒后恢复
- 使用 `navigator.clipboard.writeText()` API
- 降级方案：`document.execCommand('copy')` (for 微信内置浏览器)

## 3. 撤销链接确认流程

### 3.1 触发方式

在分享面板中点击「撤销链接」，弹出确认对话框。

### 3.2 确认对话框

```
┌──────────────────────────────────────────┐
│                                          │
│  ⚠️ 确认撤销链接？                        │
│                                          │
│  撤销后，所有通过此链接的访问              │
│  将在 5 秒内失效，且不可恢复。             │
│                                          │
│  [取消]                    [确认撤销]     │
│                                          │
└──────────────────────────────────────────┘
```

### 3.3 视觉规范

| 元素 | 样式 |
|------|------|
| 背景蒙层 | `rgba(0, 0, 0, 0.6)` + `backdrop-filter: blur(4px)` |
| 对话框背景 | `#1E293B`（与卡片同色） |
| 对话框圆角 | 16px |
| 对话框阴影 | `0 16px 48px rgba(0, 0, 0, 0.5)` |
| 最大宽度 | 380px（桌面）/ calc(100% - 32px)（移动端） |
| 居中方式 | viewport 居中（桌面）/ 底部弹出（移动端） |

### 3.4 按钮样式

| 按钮 | 背景 | 文字 | 圆角 | 高度 |
|------|------|------|------|------|
| 取消 | `transparent` + 1px border `#334155` | `#94A3B8` | 8px | 40px |
| 确认撤销 | `#EF4444` (danger red) | `#FFFFFF` | 8px | 40px |

- 确认撤销按钮 hover：`#DC2626`
- 按钮间距：12px
- 按钮区 padding-top：24px，上方 border-top：1px solid `#334155`

### 3.5 撤销成功反馈

确认撤销后的交互流程：

```
[确认撤销] --点击--> [对话框关闭]
                   [分享面板显示 "已撤销" 状态]
                   [3s后面板自动关闭]
                   [页面 5s 内变为失效提示页]
```

### 3.6 撤销后 — 失效提示页

```
│                                                      │
│                      🔒                               │
│                                                      │
│            此讨论链接已失效                            │
│                                                      │
│      创作者已撤销了此链接的访问权限                     │
│                                                      │
│        Roundtable AI · 圆桌讨论                       │
│                                                      │

```

| 元素 | 样式 |
|------|------|
| 居中布局 | 全屏居中，max-width 480px |
| 图标 | 🔒，48px，`--rt-text-muted` |
| 标题 | 20px / 700 / `--rt-text-primary` |
| 副标题 | 15px / 400 / `--rt-text-muted` |
| 底部品牌 | 同等待中页面底部 |

## 4. 交互时序总览

```
分享流程：
  [点击分享] → [面板弹出 200ms] → [显示链接] → [点击复制] → [✓ 已复制 2s]

撤销流程：
  [点击撤销] → [确认对话框 200ms] → [确认撤销] → [面板更新] → [失效页 5s]

降级说明：
  复制 API：navigator.clipboard → execCommand fallback
  微信环境：Bottom Sheet 强制使用（Popover 定位不可靠）
```

---

## 5. 给码飞的实现提示

1. 分享面板 Popover 用 `position: absolute` + `transform-origin: top right`
2. 移动端检测 `window.innerWidth < 768px` 切换 Bottom Sheet
3. 复制链接用 `navigator.clipboard.writeText()`，catch 里降级到 textarea + execCommand
4. 撤销确认对话框用 `<dialog>` 或自定义 modal，注意 `aria-modal="true"` 无障碍
5. 撤销后 SSE 连接会收到 revoke 事件，前端切换到失效页
6. 所有动画 200ms ease-out，与现有动效体系一致
