# Taiyi Checklists / 太一清单

## Before Acting

- Goal clear?
- Success gate known?
- Risk class assigned?
- Correct tool/skill selected?
- Smallest useful action identified?
- Confirmation needed?

## Before Claiming Success

- Actual state inspected?
- Readback/status/diff/test run?
- Version/path/source checked?
- Remaining risks disclosed?
- No hidden external action taken?

## Clean Staging

Exclude:

- `.git/`, `node_modules/`, caches;
- `state/`, logs, pids;
- `backups/`, archives;
- private memory, secrets, tokens;
- screenshots or raw personal data unless required.

Include:

- `SKILL.md`;
- necessary scripts;
- necessary references;
- assets needed by the skill.

## Workflow Checkpoints

Confirm before accidental-impact operations:

- public posting/publishing;
- sending messages;
- deleting or overwriting important files;
- restarting/stopping services;
- spending money;
- credential/account changes.

This is not an extra safety layer; it is an anti-misclick workflow checkpoint. Host policy and tool permissions decide what is allowed.

## Strong Skill Quality

- Clear trigger description.
- Bilingual name if public audience matters.
- Real executable helpers when useful.
- Verification gates.
- Clear host-policy delegation and low-friction workflow checkpoints.
- Clean file structure.
- No noisy docs.
- Publishable from temp directory.
