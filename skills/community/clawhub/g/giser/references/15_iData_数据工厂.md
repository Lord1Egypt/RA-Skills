## 一、软件定位

### 1.1 基本信息

| 项目 | 内容 |
|------|------|
| **开发商** | 广东南方数码科技股份有限公司（SouthGIS） |
| **软件定位** | 测绘数据一体化生产平台（数据采集→编辑→质检→入库 全周期） |
| **核心特色** | 一个平台、一套数据、一体化生产 |
| **目标用户** | 测绘院/地理信息中心/规划院/勘察设计单位 |
| **授权方式** | 商业授权（可申请试用） |
| **下载地址** | https://o.southgis.com/download/init（南方数码生态圈） |

### 1.2 iData 与 CASS 的关系

| 维度 | CASS | iData 数据工厂 |
|------|------|---------------|
| 定位 | 数字化地形地籍成图 | 测绘数据一体化平台 |
| 数据模型 | DWG为中心 | GIS数据库为中心 |
| 生产模式 | 单机/图幅为单元 | 协同/数据库为单元 |
| 质检能力 | 基础 | 模板化质检引擎 |
| 三维支持 | 基础 | 三维采编一体化 |
| 数据库对接 | 通过转换 | 原生对接（ArcSDE/PostGIS） |
| **关系** | iData 可视为 CASS 的升级与互补，同属南方数码生态 |

## 二、核心功能模块

### 2.1 数据采集

| 功能 | 说明 |
|------|------|
| **外业数据导入** | 全站仪/RTK数据直接导入 |
| **航测立体采集** | 支持立体像对采集（配合MapMatrix） |
| **点云辅助采集** | LiDAR点云数据加载与辅助绘图 |
| **DOM底图数字化** | 正射影像加载，叠加采集 |
| **CASS数据衔接** | .DWG直接读取，继承CASS编码体系 |

### 2.2 数据编辑

| 功能 | 说明 |
|------|------|
| **符号化编辑** | 所见即所得的图式符号化，编辑即入库 |
| **属性编辑** | 属性面板/批量赋值/字段计算 |
| **拓扑编辑** | 实时拓扑检查与修复（支持自定义规则） |
| **注记编辑** | 自动注记生成/手动注记/批量修改 |
| **接边处理** | 图幅自动接边/手动接边/属性一致性检查 |

### 2.3 质检引擎（核心差异化）

| 功能 | 说明 |
|------|------|
| **模板化质检** | 预制质检方案模板（按比例尺/项目类型），一键运行 |
| **图形检查** | 几何合法性/拓扑正确性/悬挂点/伪节点/面闭合 |
| **属性检查** | 字段必填/值域范围/编码合法性/属性一致性 |
| **逻辑检查** | 要素间关系（如道路与桥梁的连通性） |
| **自动修复** | 可配置的自动修复规则（如自动去除重复点） |
| **质检报告** | 自动生成质检报告（HTML/Excel） |

> → 质检标准引用 `07_质量检查与验收标准.md`

### 2.4 数据入库

| 功能 | 说明 |
|------|------|
| **ArcSDE** | 企业级空间数据库入库 |
| **PostGIS** | 开源空间数据库入库 |
| **File GDB** | 文件地理数据库导出 |
| **GeoPackage** | 现代格式导出 |
| **方案管理** | 入库方案模板（配置图层映射/字段映射/编码转换） |

### 2.5 三维采编（新版本增强）

| 功能 | 说明 |
|------|------|
| **点云加载** | LAS/LAZ 加载与可视化 |
| **三维采集** | 直接在点云/倾斜模型上采集DLG要素 |
| **DEM编辑** | TIN构建与DEM编辑 |

## 三、典型工作流

### 工作流 1：全野外数字测图

```
RTK/全站仪外业采集
    ↓ (导出 .dat / .csv)
iData 导入展点
    ↓
草图绘制（符号化编辑）→ 实时质检
    ↓
内业编辑（拓扑修复/属性录入/注记配置）
    ↓
成果检查（模板化质检引擎）
    ↓
数据入库（ArcSDE / PostGIS / GeoPackage）
```

### 工作流 2：航测内业采集

```
立体像对 + 空三成果
    ↓
iData 立体采集模块
    ↓（要素采集+编码+属性）
DLG 初编
    ↓
外业调绘成果内业修改
    ↓
质检 → 入库
```

### 工作流 3：已有数据更新

```
旧版数据 (DWG / GDB)
    ↓
iData 导入 & 数据探查
    ↓
叠加最新影像/点云
    ↓（变化区域识别与更新）
要素编辑 → 质检 → 入库
```

## 四、关键操作速查

### 4.1 编码体系

iData 兼容 CASS 编码体系，同时支持 GB/T 13923-2022 国家标准编码。

| 操作 | 方法 |
|------|------|
| 查看要素编码 | 选中要素 → 属性面板 → "编码"字段 |
| 编码对照 | 方案管理 → 编码方案 → 导入 CASS/国标编码表 |
| 自定义编码 | 方案管理 → 新增编码 → 配置符号和属性 |

### 4.2 快捷键（常用）

| 快捷键 | 功能 |
|--------|------|
| F2 | 绘制线 |
| F3 | 绘制面 |
| F8 | 正交开关 |
| F11 | 对象捕捉设置 |
| Ctrl+S | 保存 |
| Ctrl+Z | 撤销 |

### 4.3 质检方案配置

```
1. 质检菜单 → 方案管理 → 新建方案
2. 选择检查类型：图形检查 / 属性检查 / 逻辑检查
3. 配置具体规则（如"面必须闭合"、"编码必须在GB/T 13923范围内"）
4. 设置严重级别（错误/警告/提示）
5. 保存方案 → 一键运行 → 查看质检报告
```

## 五、模板化质检引擎深度 ⭐ V4.0深度扩充

### 5.1 质检规则库
- 图形规则：面闭合、线自相交、悬挂节点、伪节点、面重叠、窄面、碎面
- 属性规则：字段必填、值域范围、编码合法性、属性一致性、关联属性逻辑
- 拓扑规则：面必须被面覆盖、线必须被面边线覆盖、点必须在线端点
- 逻辑规则：道路与桥梁的连通性、等高线与高程点的矛盾

### 5.2 质检方案模板示例
```json
{
  "scheme_name": "1:500地形图质检",
  "rules": [
    {"type": "geometry", "rule": "polygon_must_close", "level": "error"},
    {"type": "attribute", "rule": "code_in_range", "params": {"min": 100000, "max": 999999}, "level": "error"},
    {"type": "topology", "rule": "no_dangle_point", "tolerance": 0.001, "level": "warning"},
    {"type": "logic", "rule": "contour_elevation_consistent", "level": "error"}
  ]
}
```

### 5.3 自动修复规则配置
- 自动去除重复点
- 自动闭合微开口 (<0.01m)
- 自动删除零长度线

## 八、数据互操作

### 8.1 iData ↔ ArcGIS Pro

```
iData 方案导出为 File GDB
    ↓
ArcGIS Pro 加载 File GDB
    ↓（空间分析/制图/发布服务）
ArcGIS Pro 结果导出
    ↓
iData 导入用于出图/质检
```

### 8.2 iData ↔ CASS

```
iData 直接打开 CASS .DWG（继承所有编码和扩展属性）
CASS 也可打开 iData 导出的 .DWG
两者编码体系互认，基本无缝
```

### 8.3 iData ↔ FME

```
iData → File GDB → FME（用于复杂ETL转换）
FME 处理结果 → File GDB/GeoPackage → iData 入库
```

## 九、常见问题 (V4.0 验证)

| 问题 | 原因 | 方案 | V4.0 验证 |
|------|------|------|-----------|
| 打开CASS DWG后符号丢失 | 符号库路径未配置 | 将CASS符号库目录添加到iData符号搜索路径 | ✅ 2026.06 |
| 质检报告误报太多 | 质检规则过于严格 | 调整检查规则严重级别和容差值 | ✅ 2026.06 |
| 数据入库慢 | 网络延迟或数据量大 | 使用批处理模式，分批入库 | ✅ 2026.06 |
| 编码与国标不一致 | 使用旧版CASS编码 | 方案管理中加载GB/T 13923编码对照表 | ✅ 2026.06 |
