## 1. 条件Dasha系统概述

BPHS记载超过40种Dasha系统，其中**条件Dasha**（Conditional Dasha）是只在特定命盘条件下才适用的推运系统。其核心原则是：**推运系统的选择本身即是命局的诊断过程**——特定命盘特征激活特定推运系统，从而在该盘上获得更高预测精度。

## 2. 八大条件Dasha系统

### 2.1 Vimshottari Dasha（通用基准，120年）

虽然不是"条件性"系统，但作为所有Dasha分析的基准，必须首先确认。

- **计算基础**：月亮所在Nakshatra
- **周期**：太阳6→月亮10→火星7→Rahu 18→木星16→土星19→水星17→Ketu 7→金星20=120年
- **适用**：所有命盘（无特殊条件）
- **地位**：通用性最强、文献最丰富、同行交流基础

### 2.2 Ashtottari Dasha（108年，条件性）

**适用条件**（必须同时满足）：
- 出生时Rahu位于Lagna Lord的Kendra(1/4/7/10)或Trikona(1/5/9)
- 出生于Krishna Paksha（下弦月期）

| 行星 | 太阳 | 月亮 | 火星 | 水星 | 土星 | 木星 | Rahu | 金星 |
|:---:|:---:|:---:|:---:|:---:|:---:|:---:|:---:|:---:|
| 年限 | 6 | 15 | 8 | 17 | 10 | 19 | 12 | **21** |
| 合计 | **108**（排除Ketu） | | | | | | | |

**特色**：灵性觉醒时机、深层业力显化、Rahu相关重大转折

### 2.3 Shodshottari Dasha（116年）

**适用条件**：出生在Shukla Paksha（上弦月期）且上升点位于Rahu-Ketu轴线的主宰星座或其三方宫

- 周期：116年
- 特色：与Ashtottari互补，一个适用于上弦月期，一个适用于下弦月期

### 2.4 Dwadashottari Dasha（112年）

**适用条件**：上升主星位于第2宫或第12宫

- 周期：112年
- 特色：专注于财富和支出领域的业力展开

### 2.5 Panchottari Dasha（105年）

**适用条件**：出生在白天（日出到日落）且上升位于太阳的主宰星座（狮子座）

- 周期：105年
- 特色：与太阳能量强关联的命盘

### 2.6 Shatabdika Dasha（100年）

**适用条件**：上升点位于Sunrise方向（即出生时太阳在地平线以上）且特定行星配置

- 周期：100年（世纪周期）
- 特色：对应百年周期的人生节奏

### 2.7 Chaturashiti Sama Dasha（84年）

**适用条件**：第10宫主星位于第10宫（在自身星座或擢升星座）

- 周期：84年
- 特色：专用于职业和公共事业预测
- **84年周期**：与天王星公转周期(84年)惊人巧合

### 2.8 Shashtihayani Dasha（60年）

**适用条件**：太阳位于上升点（Lagna）

- 周期：60年
- 特色：最短的条件Dasha之一，与太阳的60年周期对应
- 特别适合寿限和人生重大转折的判定

## 3. 条件判定流程图

```
开始 → 检查Vimshottari（通用基准）
  │
  ├─ 太阳在上升点？ → 是 → 加算Shashtihayani
  │
  ├─ 10宫主在10宫？ → 是 → 加算Chaturashiti Sama
  │
  ├─ Rahu在Lagna Lord的Kendra/Trikona
  │   且Krishna Paksha？ → 是 → 加算Ashtottari
  │
  ├─ Shukla Paksha + 上升在Rahu-Ketu主宰星座
  │   或三方宫？ → 是 → 加算Shodshottari
  │
  ├─ 上升主星在2或12宫？ → 是 → 加算Dwadashottari
  │
  ├─ 白天出生 + 上升狮子座？ → 是 → 加算Panchottari
  │
  └─ 其他特殊配置？ → 检查对应的条件Dasha
```

## 4. 条件Dasha的实战应用策略

### 何时使用条件Dasha

| 情境 | 推荐操作 |
|:---|:---|
| Vimshottari Dasha预测结果模糊或与实际不符 | 检查是否满足条件Dasha适用条件 |
| 命盘特征明显（如Rahu强势、太阳在上升等） | 优先使用对应条件Dasha进行交叉验证 |
| 特定领域需要精准timing（职业→Chaturashiti Sama） | 使用对应领域专项Dasha |

### 与通用Dasha的整合方法

1. **基准层**：Vimshottari Dasha始终作为第一层分析
2. **验证层**：检查条件Dasha适用性，如满足则加算
3. **收敛判断**：当Vimshottari和条件Dasha指向同一时间窗口→置信度大增
4. **领域专项**：对特定问题（职业、灵性等）使用对应的条件Dasha精细化

### 注意事项

- **不要过度使用**：不是每个命盘都需要条件Dasha，过度叠加反而增加混乱
- **精度要求**：条件Dasha的计算对出生时间精度要求更高
- **文献基础**：部分条件Dasha的文献记载不如Vimshottari完整，需结合实践经验
