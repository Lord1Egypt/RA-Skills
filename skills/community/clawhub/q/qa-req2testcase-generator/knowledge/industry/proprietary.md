# 自营域（Proprietary Trading Domain）

> 本文档覆盖券商自营投资核心业务对象的状态机、操作规则、异常模式与测试点，涵盖自营投资决策、交易组合管理、风控指标、交易策略等子域。

---

## 自营投资决策流程

### 业务背景
自营投资是券商以自有资金进行证券投资的业务，是券商重要的收入来源之一。自营业务受到严格的监管约束，包括规模限制、投资范围限制、风控指标等。投资决策需经过投资委员会审批，实行严格的授权管理。

### 投资决策状态全集
- 研究建议（ResearchProposal）：投研团队提出投资建议
- 策略审核（StrategyReview）：投资经理审核投资策略
- 风控预审（RiskPreCheck）：风控部门进行事前风控审查
- 投委会审批（ICApproval）：投资委员会审批（大额/重大投资）
- 已授权（Authorized）：投资决策获得授权，可执行交易
- 执行中（Executing）：交易员正在执行投资指令
- 已完成（Completed）：投资指令执行完毕（终态）
- 已否决（Rejected）：投资决策被否决（终态）
- 已撤回（Withdrawn）：投资建议被撤回（终态）

### 状态转移规则
| 当前状态 | 触发条件 | 目标状态 | 是否允许 |
|---------|---------|---------|--------|
| 研究建议 | 投资经理采纳 | 策略审核 | ✅ |
| 研究建议 | 投资经理不采纳 | 已否决 | ✅ |
| 研究建议 | 研究员撤回 | 已撤回 | ✅ |
| 策略审核 | 审核通过 | 风控预审 | ✅ |
| 策略审核 | 审核不通过 | 已否决 | ✅ |
| 风控预审 | 风控通过（小额） | 已授权 | ✅ |
| 风控预审 | 风控通过（大额） | 投委会审批 | ✅ |
| 风控预审 | 风控不通过 | 已否决 | ✅ |
| 投委会审批 | 审批通过 | 已授权 | ✅ |
| 投委会审批 | 审批不通过 | 已否决 | ✅ |
| 已授权 | 交易员开始执行 | 执行中 | ✅ |
| 已授权 | 授权过期（通常T+3） | 已撤回 | ✅ |
| 执行中 | 执行完毕 | 已完成 | ✅ |
| 已完成 | 重新执行 | — | ❌ |
| 已否决 | 直接执行 | — | ❌ |

### 允许操作
- 提交投资建议：投研团队基于研究成果提出投资方向和标的建议
- 策略审核：投资经理评估投资建议的可行性和预期收益
- 风控预审：风控部门检查投资是否符合风控指标和合规要求
- 投委会审批：重大投资决策需投资委员会集体决策
- 执行交易：获得授权后由交易员在授权范围内执行

### 禁止操作（反例知识）
- 未经风控预审直接交易：所有自营交易必须经过事前风控审查
- 超授权范围交易：交易金额/标的超出授权范围
- 授权过期后执行：授权有时效性，过期需重新申请
- 投资经理自行执行：投资决策与交易执行必须分离（前中后台分离）
- 投资禁止名单标的：投资被列入禁止名单的标的（如内幕信息相关）

### 异常模式
- 授权过期：投资授权在有效期内未执行 → 测试点：验证授权过期后系统自动失效，交易员无法下单
- 审批流程超时：投委会审批超过规定时限 → 测试点：验证审批超时的自动提醒和升级机制
- 风控指标变动：审批通过后、执行前风控指标发生变化 → 测试点：验证执行前的二次风控校验
- 市场异常波动：执行期间市场剧烈波动 → 测试点：验证紧急止损机制和授权变更流程

### 测试点模板
- TP-PROP-001：投资建议提交，验证必填字段（标的、方向、金额、理由）完整性校验（优先级：P0）
- TP-PROP-002：大额投资（>5000万），验证必须经过投委会审批（优先级：P0）
- TP-PROP-003：授权过期（T+3），验证系统自动失效并通知相关人员（优先级：P0）
- TP-PROP-004：交易员超授权金额下单，验证系统拦截（优先级：P0）
- TP-PROP-005：禁止名单标的投资，验证风控预审阶段拦截（优先级：P0）
- TP-PROP-006：投委会审批超过2个工作日，验证自动提醒机制（优先级：P1）

### 中间态处理规则
- 「风控预审」为中间态，应在4小时内完成
- 「投委会审批」为中间态，应在2个工作日内完成
- 「执行中」为中间态，执行期限由授权文件指定（通常T+1至T+5）
- 授权有效期内未执行完毕，需申请延期或重新授权

### 幂等/重试规则
- 投资建议使用 proposalId + version 作为幂等键
- 授权指令使用 authorizationId + tradeDate 作为幂等键
- 审批流程不可重复提交，同一建议在审批中不可再次提交
- 交易执行使用 authorizationId + orderSeqNo 作为幂等键

## 交易组合管理

### 业务背景
券商自营投资通常按资产类别或策略类型划分为多个交易组合（Trading Book），每个组合有独立的投资策略、风控指标和绩效考核。

### 组合类型

#### 股票组合
- 投资范围：A股、港股通、科创板
- 策略类型：价值投资、成长投资、事件驱动
- 风控指标：单票集中度≤5%、行业集中度≤20%、总仓位≤80%
- 止损规则：单票止损-8%、组合止损-5%

#### 债券组合
- 投资范围：国债、金融债、企业债、可转债
- 策略类型：持有到期、交易型、套利
- 风控指标：久期限制、信用评级下限（AA-）、集中度限制
- 止损规则：组合回撤-2%触发预警，-3%触发止损

#### 基金组合
- 投资范围：公募基金、ETF、LOF
- 策略类型：FOF配置、ETF套利、指数增强
- 风控指标：单只基金持仓≤10%、同一管理人≤20%

#### 衍生品组合
- 投资范围：股指期货、国债期货、期权、互换
- 策略类型：对冲、套利、方向性交易
- 风控指标：保证金占用≤净资本30%、Greeks限额
- 特殊要求：衍生品交易需单独授权，交易员需持有相关资质

### 组合状态全集
- 筹建中（Preparing）：组合正在设立，配置参数和风控指标
- 运作中（Active）：组合正常运作，可进行交易
- 限制交易（Restricted）：因风控触发或合规要求限制部分交易
- 清盘中（Winding）：组合正在清盘，仅允许减仓操作
- 已清盘（Closed）：组合已清盘完毕（终态）

### 异常模式
- 组合间关联交易：不同组合之间的对倒交易 → 测试点：验证组合间交易的关联交易检测
- 被动超限：市场波动导致集中度被动超限 → 测试点：验证被动超限的宽限期（通常5个交易日）和强制减仓
- 跨组合风控：多个组合合计持有同一标的超过总限制 → 测试点：验证跨组合的合并风控计算
- 组合绩效归因异常：资金调拨导致绩效计算偏差 → 测试点：验证资金调拨对组合收益率计算的影响

### 测试点模板
- TP-PROP-010：股票组合单票买入超过净值5%，验证系统拦截（优先级：P0）
- TP-PROP-011：债券组合买入AA-以下评级债券，验证系统拦截（优先级：P0）
- TP-PROP-012：组合状态为「限制交易」时尝试加仓，验证系统拒绝（优先级：P0）
- TP-PROP-013：被动超限后第6个交易日仍未调整，验证系统强制限制交易（优先级：P1）
- TP-PROP-014：跨组合合计持有同一股票超过总限制，验证合并风控告警（优先级：P1）

## 风控指标体系

### VaR（Value at Risk）
- 定义：在给定置信水平和持有期下，组合可能的最大损失
- 计算方法：历史模拟法、参数法（方差-协方差）、蒙特卡洛模拟
- 参数设置：置信水平99%，持有期1天/10天
- 限额管理：各组合VaR不超过分配的VaR限额
- 回测验证：定期回测VaR模型的准确性（突破次数不超过理论值）

### 压力测试
- 历史情景：重现历史极端行情（如2015年股灾、2020年疫情）
- 假设情景：自定义市场变动幅度（如沪指下跌10%、利率上升100bp）
- 频率：至少每月一次，极端行情时每日执行
- 结果应用：压力测试损失超过阈值需调整仓位或补充资本

### 集中度控制
- 单票集中度：单一股票持仓市值/自营权益类证券总市值 ≤ 5%
- 行业集中度：单一行业持仓/自营权益类证券总市值 ≤ 20%
- 发行人集中度：同一发行人债券/自营固收类证券总市值 ≤ 10%
- 对手方集中度：同一对手方场外交易敞口/净资本 ≤ 15%

### 止损线管理
- 个股止损：单只股票浮亏达到买入成本的-8%触发止损
- 组合止损：组合净值回撤达到-5%触发止损
- 年度止损：年度累计亏损达到年初预算的-10%触发全面止损
- 止损执行：触发后T+1日内完成减仓/平仓
- 止损豁免：特殊情况可申请止损豁免（需投委会审批）

### 异常模式
- VaR模型失效：极端行情下VaR严重低估实际损失 → 测试点：验证VaR回测突破次数超过阈值时的模型修正触发
- 止损线触发但无法执行：跌停/停牌导致无法卖出 → 测试点：验证无法止损时的风险敞口计算和升级处理
- 集中度计算延迟：实时行情波动导致集中度计算滞后 → 测试点：验证集中度的计算频率和延迟容忍度
- 压力测试数据缺失：部分标的无历史数据无法参与压力测试 → 测试点：验证数据缺失时的替代处理方案

### 测试点模板
- TP-PROP-020：VaR计算，验证99%置信水平、1天持有期的VaR值准确性（优先级：P0）
- TP-PROP-021：VaR超限，验证系统触发预警并限制新增头寸（优先级：P0）
- TP-PROP-022：个股浮亏达-8%，验证系统触发止损预警通知（优先级：P0）
- TP-PROP-023：组合净值回撤-5%，验证系统限制该组合交易并通知投委会（优先级：P0）
- TP-PROP-024：压力测试（沪指-10%情景），验证组合预估损失计算准确性（优先级：P1）
- TP-PROP-025：止损豁免申请，验证需投委会审批且有时效限制（优先级：P1）

## 交易策略

### 量化交易
- 策略类型：统计套利、因子选股、高频做市、CTA趋势跟踪
- 技术架构：策略引擎+风控引擎+执行引擎，低延迟要求
- 回测要求：策略上线前需通过历史回测和模拟盘验证
- 监控指标：策略收益、最大回撤、夏普比率、换手率

### 套利策略
- ETF套利：一二级市场价差套利（折价申购/溢价赎回）
- 期现套利：股指期货与现货指数的基差套利
- 跨市场套利：A/H股价差、沪深跨市场套利
- 可转债套利：转股溢价率套利、条款博弈

### 对冲策略
- 股指期货对冲：持有股票多头+股指期货空头
- 期权对冲：买入看跌期权保护股票持仓
- 配对交易：同行业股票多空配对
- Beta对冲：调整组合Beta至目标水平

### 异常模式
- 策略失效：市场环境变化导致策略持续亏损 → 测试点：验证策略监控指标（连续亏损天数、最大回撤）触发策略暂停
- 套利机会消失：价差收敛速度超预期导致套利失败 → 测试点：验证套利策略的止损和退出机制
- 量化策略异常交易：程序bug导致异常大量下单 → 测试点：验证量化交易的熔断机制（单策略单日交易量上限）
- 对冲比例偏移：对冲工具与标的相关性下降 → 测试点：验证对冲有效性监控和再平衡触发

### 测试点模板
- TP-PROP-030：量化策略上线，验证必须通过回测+模拟盘验证（优先级：P0）
- TP-PROP-031：单策略单日交易量超过上限，验证系统自动暂停策略（优先级：P0）
- TP-PROP-032：ETF套利，验证折溢价计算准确性和交易执行时效（优先级：P1）
- TP-PROP-033：策略连续亏损5天，验证系统触发策略审查流程（优先级：P1）
- TP-PROP-034：对冲比例偏移超过阈值，验证系统提示再平衡（优先级：P1）

## 合规要求

### 自营规模限制
1. **权益类证券**：自营权益类证券及衍生品规模不超过净资本的100%
2. **固定收益类**：自营固定收益类证券规模不超过净资本的500%
3. **总规模**：自营业务总规模不超过净资本的一定倍数
4. **单一标的**：持有单一上市公司股票不超过其总股本的5%（举牌线）

### 关联交易管理
1. **识别**：识别自营与资管、投行等业务之间的关联交易
2. **审批**：关联交易需经合规部门审批
3. **披露**：重大关联交易需向监管报告
4. **公平定价**：关联交易价格不得偏离市场公允价格

### 防火墙制度
1. **信息隔离**：自营部门与投行、研究所之间的信息隔离
2. **物理隔离**：办公区域物理分隔
3. **系统隔离**：交易系统、研究系统的权限隔离
4. **人员隔离**：关键岗位人员不得兼任
5. **跨墙审批**：必要的信息传递需经合规审批（跨越防火墙）

### 内幕交易防控
1. **内幕信息管理**：内幕信息知情人登记和管理
2. **限制交易名单**：涉及内幕信息的标的列入限制交易名单
3. **交易监控**：自营交易与内幕信息的关联性监控
4. **举报机制**：内幕交易举报和调查机制

## 常见缺陷模式

1. **授权管理漏洞**：交易员可绕过授权直接下单
2. **风控指标计算延迟**：集中度/VaR计算不实时，导致超限未及时发现
3. **止损执行不及时**：触发止损后未在规定时间内完成减仓
4. **关联交易未识别**：自营与资管同时交易同一标的未触发关联交易检测
5. **防火墙失效**：自营人员可访问投行项目信息
6. **组合间对倒**：不同自营组合之间的对倒交易未被检测
7. **策略监控缺失**：量化策略异常运行未触发熔断
8. **禁止名单更新延迟**：内幕信息标的未及时加入禁止名单

## 边界值与特殊场景

| 场景 | 边界条件 | 预期行为 |
|------|---------|---------|
| 单票集中度 | 恰好5% | 允许（≤5%） |
| 单票集中度 | 5.01% | 拦截新增买入 |
| 止损线 | 浮亏恰好-8% | 触发止损预警 |
| 止损线 | 浮亏-7.99% | 不触发 |
| 授权有效期 | T+3日收盘 | 授权失效 |
| 授权金额 | 恰好等于授权上限 | 允许执行 |
| 授权金额 | 超过授权上限0.01元 | 拦截 |
| 举牌线 | 持有恰好5%总股本 | 触发举牌披露义务 |
| VaR限额 | 恰好等于限额 | 预警（≥限额） |
| 策略日交易量 | 恰好等于上限 | 允许（≤上限） |
| 策略日交易量 | 超过上限1笔 | 暂停策略 |
| 被动超限宽限期 | 第5个交易日 | 最后调整日 |
| 被动超限宽限期 | 第6个交易日 | 强制限制交易 |
