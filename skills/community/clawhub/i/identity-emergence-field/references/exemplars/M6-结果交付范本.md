# M6 结果交付域范本

## M6-01 结果完整性验证

### 输入示例
任务结果：身份涌现分析报告

### 处理过程
1. 完整性标准制定：报告是否包含所有必要部分
2. 验证方法选择：检查报告结构和内容
3. 验证执行：逐项检查报告内容
4. 验证结论：结果完整

### 输出示例
```markdown
## 结果完整性验证报告

### 完整性标准
| 标准 | 描述 | 权重 |
|------|------|------|
| 结构完整性 | 报告是否包含所有必要部分 | 40% |
| 内容完整性 | 报告内容是否完整 | 30% |
| 格式完整性 | 报告格式是否规范 | 20% |
| 逻辑完整性 | 报告逻辑是否清晰 | 10% |

### 验证方法
- **方法**：逐项检查法
- **检查项**：涌现场识别、身份场激活、共识网络、共识收敛、透明度控制
- **检查标准**：是否完整、准确、清晰

### 验证结果
| 检查项 | 状态 | 说明 |
|--------|------|------|
| 涌现场识别 | ✓ | 完整 |
| 身份场激活 | ✓ | 完整 |
| 共识网络 | ✓ | 完整 |
| 共识收敛 | ✓ | 完整 |
| 透明度控制 | ✓ | 完整 |

### 验证结论
**结论**：结果完整，符合完整性标准
```

## M6-02 响应速度优化

### 输入示例
当前响应时间：60秒

### 处理过程
1. 速度目标设定：在45秒内生成报告
2. 优化点识别：涌现场识别和共识网络构建步骤
3. 优化方案设计：并行处理，优化算法
4. 优化效果评估：响应时间从60秒降至40秒

### 输出示例
```markdown
## 响应速度优化报告

### 速度目标
- **目标**：在45秒内生成报告
- **当前**：60秒
- **差距**：15秒

### 优化点识别
| 优化点 | 当前耗时 | 优化目标 | 优化方法 |
|--------|----------|----------|----------|
| 涌现场识别 | 20秒 | 15秒 | 并行处理 |
| 共识网络构建 | 25秒 | 15秒 | 优化算法 |
| 共识收敛 | 15秒 | 10秒 | 优化算法 |

### 优化方案
- **涌现场识别**：采用并行处理
- **共识网络构建**：优化网络构建算法
- **共识收敛**：优化收敛算法

### 优化效果
- **响应时间**：从60秒降至40秒
- **优化幅度**：33%
- **达标情况**：达标（40秒 < 45秒）
```
