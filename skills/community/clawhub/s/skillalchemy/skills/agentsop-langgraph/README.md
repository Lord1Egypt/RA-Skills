# langgraph-sop-skill

A skill that captures the **operating system** behind LangGraph — the de facto standard for production agent workflows in 2026 (used by Klarna, Replit, Uber, LinkedIn, AppFolio, Elastic). Unlike API docs, this skill encodes the *decision patterns* a coder-agent needs: when to reach for StateGraph vs. the Functional API, when supervisor beats swarm, when to wrap a flow in a subgraph, when to interrupt vs. let the agent run, and where the framework actually breaks down. It is meant to be activated by a coder agent that is being asked to build, extend, debug, or migrate a stateful LLM workflow, and it explicitly cites its sources (LangChain docs, official blog, GitHub issues, production case studies, community gotchas) so the agent can verify claims before acting.
