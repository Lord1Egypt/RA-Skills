# Step 7: Talk And Q&A Rehearsal

Use this step when the user wants to explain the paper in a group meeting, journal club, class, lab meeting, defense, or PPT.

## Talk blueprint

Create a slide/talk blueprint derived from the authoritative detailed report.

Default structure:

1. Motivation: why this paper matters.
2. Problem setting: task, inputs, outputs, constraints.
3. Broken assumption: what prior work needed but the target setting lacks.
4. Core insight: unavailable ideal `Y` and surrogate `Z`.
5. Method overview: one architecture/pipeline figure.
6. Formula walkthrough: one central objective/update/bound.
7. Algorithm toy example: concrete mini-case.
8. Experiments: what each block tests.
9. Evidence audit: what is strongly vs weakly supported.
10. Limitations and reviewer concerns.
11. Future directions and boundary-pushing ideas.
12. Three takeaways.

## Q&A rehearsal

For each likely question:

- classify audience: beginner / peer / advisor / reviewer / practitioner;
- provide an answer;
- cite the report section or paper evidence;
- state confidence;
- state what the paper does not prove.

## Defense rule

Do not over-defend. If the paper lacks evidence, answer with a balanced defense:

```text
The paper supports X through [evidence].
It does not fully establish Y because [missing evidence].
A fair follow-up would test Z.
```
