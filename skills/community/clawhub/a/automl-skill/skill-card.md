## Description: <br>
Automl Skill helps data scientists, developers, and machine learning engineers build low-code PyCaret workflows for classification, regression, clustering, anomaly detection, time series forecasting, NLP, association rules, and statistical inference. <br>

This skill is ready for commercial/non-commercial use. <br>

## Use Case: <br>
External users, developers, data scientists, and machine learning engineers use this skill to quickly prototype, compare, tune, evaluate, explain, save, and deploy PyCaret-based machine learning workflows with supporting statistical tests and model diagnostics. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: AutoML outputs and statistical tests can be misleading when datasets are biased, sensitive, poorly sampled, or incorrectly preprocessed. <br>
Mitigation: Review dataset quality, model sensitivity, validation splits, and statistical assumptions before relying on generated workflows or conclusions. <br>
Risk: Deployment and app/API generation examples can create files, containers, or cloud resources with unintended access or sensitive data exposure. <br>
Mitigation: Choose output paths intentionally, inspect generated app/API/Docker files, and use least-privilege AWS, GCP, or Azure credentials. <br>
