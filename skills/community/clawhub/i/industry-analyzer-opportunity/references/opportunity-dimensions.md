# 行业机会 4 维度框架

> 这是 行业机会分析器 写 `04-opportunity.md` 时的"4 维度"标准框架。每个维度有自己的写作要点。

---

## 维度 ① 需求场景

### 写作要点

- 3-5 个典型场景，覆盖不同客户类型
- 每个场景 = {目标客户} × {典型动作} × {期望结果}
- 现有方案缺口 = 这个场景下，市场上的方案"做不到什么"或"做不好什么"

### 检索模板

```
"{行业}" 典型应用场景
"{行业}" 客户案例
"{行业}" 数字化 场景
"{行业}" 业务 流程
```

### 模板片段

```markdown
### 1.1 场景名

**目标客户**：{客户类型，如三甲医院 / 制造企业 / 中小商家}

**典型动作**：{客户在这个场景下做什么}

**现有方案缺口**：
- 缺口 A：{具体不足}
- 缺口 B：{具体不足}

**机会洞察**：{这个缺口背后意味着什么市场机会}
```

---

## 维度 ② 需求痛点

### 写作要点

- 3-7 个痛点，按"客户分层"组织
- 分层示例：
  - 终端用户（C 端）
  - 行业厂商（B 端）
  - 渠道 / 服务商
  - 监管 / 政府
- 痛点要具体，**不是"成本高"这么宽泛**，而是"三甲医院 IT 系统每年维护成本占预算 X%"

### 检索模板

```
"{行业}" 行业痛点
"{行业}" 客户调研 痛点
"{行业}" 满意度 调研
"{行业}" 痛点 报告
```

### 模板片段

```markdown
### 2.1 痛点名（{客户分层}）

**痛点描述**：{具体描述}

**影响范围**：{X% 客户 / X 万家 / X 亿元}

**量化（如有）**：{具体数字}

**根因（可选）**：{为什么存在这个痛点}
```

## 维度 ③ 政策机会

### 写作要点

- 3-5 项政策 / 规划 / 文件
- 必须含：**发布时间 + 发布部门 + 资金规模 + 影响**
- 来源要可追溯（如"国务院 2024 年印发的《XX 规划》"）

### 检索模板

```
"{行业}" 国家政策 2024
"{行业}" 国家政策 2025
"{行业}" 国务院 规划
"{行业}" 部委 文件
"{行业}" 十四五 规划
"{行业}" 地方政府 补贴
"{行业}" 行动计划
```

### 模板片段

```markdown
### 3.1 政策名

**政策全名**：{全名}

**发布时间**：{YYYY-MM}

**发布部门**：{国务院 / XX 部委 / XX 地方政府}

**资金 / 规模**：{如有}

**核心内容**：
- 要点 1
- 要点 2

**影响**：
- 对行业：{...}
- 对厂商：{...}
- 对客户：{...}

**机会窗口**：{政策落地的 1-2 年时间窗}
```

---

## 维度 ④ 潜在未来发展

### 写作要点

- 3-5 个趋势，覆盖短期（1 年内）/ 中期（1-3 年）/ 长期（3+ 年）
- 每个趋势必须有**驱动因素**——不能凭空预测
- 潜在赢家 = 谁可能抓住这个趋势

### 检索模板

```
"{行业}" 未来 趋势
"{行业}" 行业 报告 2025 2026
"{行业}" 战略 趋势
"{行业}" 麦肯锡 报告
"{行业}" Gartner 报告
"{行业}" IDC 报告
"{行业}" 艾瑞 报告
```

### 模板片段

```markdown
### 4.1 趋势名

**时间窗**：短期（< 1 年）/ 中期（1-3 年）/ 长期（3+ 年）

**驱动因素**：
- 因素 1：{技术 / 政策 / 需求 / 资本}
- 因素 2

**具体表现**：
- 表现 1
- 表现 2

**潜在赢家**：
- 类型 1：{如：云大厂 / 垂直龙头 / 新创公司}
- 类型 2

**风险（可选）**：
- 风险 1：{可能让趋势不成立的因素}
```
