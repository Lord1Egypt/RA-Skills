## Description: <br>
Use when users need visual direction, interface hierarchy, layout decisions, design specifications, or prototypes before implementing a Web or mini program UI. <br>

This skill is ready for commercial/non-commercial use. <br>

## Publisher: <br>
[binggg](https://clawhub.ai/user/binggg) <br>

### License/Terms of Use: <br>
MIT-0 <br>


## Use Case: <br>
Developers and design-focused agents use this skill to define visual direction, interface hierarchy, layout strategy, color, typography, and prototype expectations before implementing web or mini program UI. It is intended for UI design decisions rather than backend-only work or routine implementation of an already-approved interface. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Rigid default style rules can conflict with an approved brand system, accessibility requirements, or routine UI implementation tasks. <br>
Mitigation: Tell the agent to prioritize approved design-system, brand, and accessibility constraints and to override the skill's default font or color bans only as narrowly as needed. <br>
Risk: The skill can over-constrain visual choices when the user only needs straightforward implementation of an already-approved UI. <br>
Mitigation: Use the skill for visual direction, hierarchy, layout, or prototype decisions; skip it or narrow its scope when the design direction is already settled. <br>
