## Description: <br>
Ui Design Guide helps agents choose visual direction, interface hierarchy, layouts, design specifications, and prototypes before implementing web or mini program UIs. <br>

This skill is ready for commercial/non-commercial use. <br>

## Publisher: <br>
[binggg](https://clawhub.ai/user/binggg) <br>

### License/Terms of Use: <br>
MIT-0 <br>


## Use Case: <br>
Developers and coding agents use this skill to establish a deliberate UI design direction before creating frontend interfaces. It is suited for web and mini program work that needs explicit decisions about aesthetics, hierarchy, layout, typography, color, icons, and prototype quality. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: The skill can steer visual choices aggressively, which may conflict with enterprise, accessibility, or brand-constrained projects. <br>
Mitigation: State required design-system, accessibility, and brand constraints explicitly so they override the skill's stylistic defaults. <br>
Risk: Generated UI guidance or prototype code may not match the target project's approved design direction. <br>
Mitigation: Review the design specification and self-audit outputs before implementation or release. <br>
