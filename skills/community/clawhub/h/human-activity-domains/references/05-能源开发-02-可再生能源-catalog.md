## 可再生能源任务清单

可再生能源按子域组织的任务清单，附任务间关系和元操作映射提示。

**域间逻辑流**：资源评估 → 技术开发 → 项目建设 → 运营维护

---

### 1 资源评估

| ID | 任务类型 | 说明 | 依赖 | 元操作映射 |
|----|----------|------|------|----------|
| T1-1 | 资源调查 | 调查可再生能源资源分布，包括太阳能、风能、水能等 | 无（入口） | S→C |
| T1-2 | 资源分析 | 分析资源质量、资源稳定性、资源可利用性 | T1-1 | C |
| T1-3 | 资源规划 | 规划资源开发利用方案，确定开发规模和时序 | T1-2 | C→A |
| T1-4 | 资源优化 | 优化资源配置，提高资源利用效率 | T1-3 | C→A |

## 依赖拓扑摘要

### 一、域内链路

**资源评估链路**: T1-1 → T1-2 → T1-3 → T1-4
**技术开发链路**: T2-1 → T2-2 → T2-3 → T2-4
**项目建设链路**: T3-1 → T3-2 → T3-3 → T3-4
**运营维护链路**: T4-1 → T4-2 → T4-3 → T4-4

### 二、跨域协同链路

**完整可再生能源链路**: T1-1 → T1-2 → T1-3 → T1-4 → T2-1 → T2-2 → T2-3 → T2-4 → T3-1 → T3-2 → T3-3 → T3-4 → T4-1 → T4-2 → T4-3 → T4-4

**反馈改进链路**: T4-3 → T2-4（性能监测反馈到技术优化）

**资源驱动链路**: T1-4 → T3-1（资源优化驱动项目规划）

更多组合根据具体任务动态推导。
