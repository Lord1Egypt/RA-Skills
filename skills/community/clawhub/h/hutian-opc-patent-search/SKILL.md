# 查新检索 Skill

> **命名渊源**：「查新」源自科技查新（Novelty Search），是申请前对新颖性/创造性进行验证的标准化流程。本Skill将AI能力注入这一传统流程，并延伸至申请中、申请后全生命周期，实现"事前预防→事中合规→事后维护"的闭环价值。

---

## 核心使命

**全生命周期AI护航——申请前预审·申请中合规·申请后维护**

| 阶段 | 核心价值 | 用户痛点解决方案 |
|------|---------|-----------------|
| 申请前 | AI预审 | 省钱（防驳回）、省时（预审替代反复修改）、省心（替代不靠谱事务所） |
| 申请中 | 流程合规 | 选对代理所、签好合同、备齐材料、不被坑 |
| 申请后 | 维护运营 | 年费/续展不遗漏、使用合规不踩坑、权利运营变现 |

## 模块一：专利查新（Patent Novelty Search）

**目标**：避免冲突，不是简单的"做交底文件"，而是实打实的冲突排查+相似度分析+风险定价。

### 功能1：技术特征提取与权利要求构建

**用户输入**：
- 技术方案描述（自然语言）
- 技术交底书（已有文档）
- 权利要求书（已有草案）

**AI处理**：

| 处理步骤 | 操作内容 |
|---------|---------|
| 独立权利要求解析 | 提取必要技术特征（特征A/B/C...）及对应技术效果 |
| 从属权利要求解析 | 提取附加技术特征及引用关系 |
| 技术特征矩阵构建 | 特征编号|特征描述|实施方式|技术效果|可专利点类型 |
| 可专利点识别 | 标记方法/产品/用途/系统等不同类型创新点 |

**输出**：
```
技术特征矩阵表
┌────────┬────────────────┬──────────────┬──────────────────┬──────────────┐
│特征编号│    特征描述    │   实施方式   │    技术效果      │ 可专利点类型 │
├────────┼────────────────┼──────────────┼──────────────────┼──────────────┤
│ TF-001 │ 降噪处理模块   │ 自适应滤波器 │ 提升语音识别率   │ 方法+装置    │
│ TF-002 │ 意图识别引擎   │ 深度学习模型 │ 精准理解用户指令 │ 方法          │
│ TF-003 │ 响应生成模块   │ 模板+NLG     │ 快速响应         │ 方法+系统    │
└────────┴────────────────┴──────────────┴──────────────────┴──────────────┘

建议权利要求框架：
├── 独立权利要求1：一种语音控制方法（覆盖TF-001+TF-002+TF-003）
├── 从属权利要求2：一种语音控制方法，其特征在于...
├── 从属权利要求3：一种语音控制方法，其特征在于...
└── 独立权利要求N：一种语音控制系统（系统类型权利要求）
```

### 功能2：相似专利检索与相似度分析

**基于功能1输出的技术特征矩阵，执行多维度检索**：

**检索策略**：

| 检索维度 | 数据来源 | 检索方法 |
|---------|---------|---------|
| 语义检索 | 智慧芽PatSnap / Google Patents / CNIPA | 自然语言描述匹配技术方案 |
| 分类号检索 | IPC / CPC分类库 | 定位技术领域对应分类号 |
| 关键词检索 | 关键词+同义词+上下位概念 | 扩展检索词提高召回率 |
| 发明人追踪 | 同一技术领域活跃发明人 | 追踪其相关申请 |
| 申请人追踪 | 竞争对手/行业龙头申请人 | 监控其专利布局 |

**相似度分析**：

对每个相似专利，执行逐项权利要求比对：

| 对比项 | 分析内容 |
|-------|---------|
| 特征覆盖分析 | 申请方案的TF-001是否被X专利的Y权利要求覆盖 |
| 相似度评分 | 技术特征重合度 = (重合特征数 / 申请方案总特征数) × 100% |
| 覆盖类型判定 | 完全覆盖 / 部分覆盖 / 技术路径不同 |

**输出**：
```
相似专利清单与逐特征比对表

┌────────────┬────────────┬─────────────────────┬────────────┬────────────────────┐
│  专利号    │  申请日    │     技术方案摘要    │  相似度    │      覆盖类型      │
├────────────┼────────────┼─────────────────────┼────────────┼────────────────────┤
│ CN113XXXXX │ 2022-03-15 │ 一种语音控制系统... │    87%    │   ⚠️部分覆盖        │
│ CN112XXXXX │ 2021-08-22 │ 智能家居语音助手... │    65%    │   🔶技术路径不同    │
│ US10XXXXX  │ 2020-11-10 │ Voice control...    │    72%    │   ⚠️部分覆盖        │
└────────────┴────────────┴─────────────────────┴────────────┴────────────────────┘

逐特征比对详情（以CN113XXXXX为例）：
┌────────┬────────────────┬──────────────────────────────┬───────────────┐
│ 特征   │   申请方案     │     对比专利CN113XXXXX        │    比对结果   │

### 功能3：同族专利追踪

**对每个高相似度专利（相似度≥40%），追踪其同族专利**：

**同族追踪范围**：

| 同族类型 | 追踪内容 | 数据来源 |
|---------|---------|---------|
| 中国同族 | 同一优先权在中国的其他申请/分案 | CNIPA |
| PCT国际同族 | 同一优先权在其他国家的申请 | WIPO PATENTSCOPE |
| 分案/续案 | 分案申请、部分续案追踪 | 各国家局数据库 |
| 引用/被引 | 审查员引用的对比文件/后续改进 | 专利数据库 |

| 分析维度 | 具体内容 |

| 地域覆盖 | 哪些国家/地区已有布局 |
| 时间线 | 申请日→公开日→授权日→到期日 |

### 功能4：冲突危险性评估

**基于功能2+3的相似度分析和同族追踪，评估冲突风险**：

**冲突等级定义**：

| 风险等级 | 标识 | 定义 | 驳回概率 |
|---------|------|------|---------|
| 直接冲突 | ⛔ | 权利要求被完全覆盖，无区别特征 | >90% |
| 间接冲突 | 🔴 | 部分特征重叠，可能构成等同侵权 | 60-90% |
| 潜在冲突 | 🟡 | 技术路径相近但特征不同，审查员可能引用 | 30-60% |
| 无冲突 | 🟢 | 技术路径显著不同，特征有本质区别 | <15% |

**冲突风险矩阵**：

| 专利号 | 相似度 | 同族覆盖 | 冲突等级 | 驳回概率区间 | 风险理由 |
|-------|--------|---------|---------|-------------|---------|
| CN113XXXXX | 87% | CN/US/EP/JP | 🔴间接冲突 | 65-80% | TF-001被等同覆盖，TF-002有差异但不足以克服 |
| CN112XXXXX | 65% | CN | 🟡潜在冲突 | 35-50% | 技术路径不同但存在交叉引用风险 |
| US10XXXXX | 72% | US | 🔴间接冲突 | 55-70% | US市场存在等同侵权风险 |

**风险量化评估**：

| 风险类型 | 评估内容 |
|---------|---------|
| 驳回风险 | 申请被驳回的概率区间及主要驳回理由预测 |
| 授权后无效风险 | 若授权，他人提出无效宣告的成功概率 |
| 侵权诉讼风险 | 若实施该技术方案，面临专利权人起诉的可能性 |

**输出**：
```
冲突危险性评估报告

一、总体风险等级：🔴高风险

二、核心冲突专利分析
┌────────────┬────────────────┬────────────────────────────┐

### 功能5：规避创新路线图

**不是空泛的"建议修改"，而是具体可操作的规避方案**：

**方案类型与内容**：

| 方案类型 | 具体操作 | 适用场景 |
|---------|---------|---------|
| 方案A：权利要求差异化 | 识别被覆盖特征，提供替代特征及技术效果论证 | 特征等同但可技术改进 |
| 方案B：技术路径替代 | 使用TRIZ矛盾矩阵找替代技术路线 | 技术路径被占但原理可替代 |
| 方案C：从属权利要求加固 | 缩小独立权利要求保护范围，加固外围从属权利要求 | 需平衡保护范围与授权风险 |
| 方案D：分案申请策略 | 将宽泛申请拆分为多个窄申请，分别检索规避 | 存在多个独立创新点 |

**TRIZ矛盾矩阵规避示例**（TF-001降噪处理被覆盖）：

| 矛盾类型 | 问题描述 | TRIZ解决原理 | 替代方案 |
|---------|---------|-------------|---------|
| 技术矛盾 | 降噪精度与计算速度冲突 | #15-动态特性原理 | 采用分阶段降噪策略 |
| 物理矛盾 | 需要宽频降噪又需要低延迟 | #1-分割原理 | 分割频段处理后合成 |
| 技术矛盾 | 滤波器参数自适应与稳定性冲突 | #35-参数变化原理 | 多级固定滤波器+智能切换 |

**规避方案对比表**：

| 方案 | 修改内容 | 修改后风险再评估 | 预估授权率 | 保护范围影响 |
|-----|---------|-----------------|-----------|-------------|
| 方案A：权利要求差异化 | 将TF-001替换为"多级分频降噪" | 🟡潜在冲突 | 60-70% | 保护范围缩小15% |
| 方案B：技术路径替代 | 改用"频谱分析+特征提取"替代滤波 | 🟢无冲突 | 75-85% | 保护范围基本持平 |
| 方案C：从属权利要求加固 | 保留原独立权利要求，增加TF-004强化从属 | 🔴间接冲突 | 50-60% | 保护范围不变但不稳定 |
| 方案D：分案申请 | 将TF-002/TF-003独立分案，TF-001做另一分案 | 取决于各分案检索结果 | 混合 | 多个窄保护范围 |

**输出**：
```
规避创新路线图

推荐方案：方案B（技术路径替代）

【方案B详细说明】
1. 原技术路径：自适应滤波器降噪 → 深度学习意图识别 → 模板响应
2. 替代技术路径：频谱分析降噪 → 注意力机制意图识别 → 生成式响应
3. 核心差异：
   ├── 降噪原理：频域处理替代时域滤波
   ├── 识别机制：注意力机制替代传统深度学习
   └── 响应生成：生成式替代模板式
4. TRIZ原理应用：
   ├── 分割原理：将降噪任务分割为多个频段处理
   └── 动态特性原理：注意力权重自适应调整
5. 修改后风险：🟢无冲突
6. 预估授权率：75-85%
```

## 模块二：商标可注册性（Trademark Registrability）

**目标**：图形+名称+分类+防冲突+费用优化，不是简单的"评估可注册性"，而是完整的保护方案设计。

### 功能1：图形近似比对

**用户输入**：
- 商标图形设计文件（JPG/PNG/AI/PSD等格式）
- 产品/服务领域描述

**AI处理**：

| 处理步骤 | 操作内容 |
|---------|---------|
| 图形要素识别 | 使用read_image工具识别构图/线条/色彩/形状/意象 |
| 视觉特征提取 | 分析主色调/图形元素/整体结构/设计风格 |
| 同类别检索 | 搜索中国商标网同类别+关联类别已注册商标图形 |
| 逐项比对 | 构图相似/线条相似/色彩混淆/整体印象相似 |

## 模块四：申请中流程合规（Application Compliance）

**核心功能**：申请过程中的流程合规与代理管理

### 4.1 申请流程指南

**专利申请三轨制**：

| 类型 | 保护对象 | 审查周期 | 保护期限 | 官费区间 |
|------|---------|---------|---------|---------|
| 发明专利 | 产品/方法 | 2-3年 | 20年 | 3450-5000元 |
| 实用新型 | 产品形状/结构 | 6-8月 | 10年 | 500-1000元 |
| 外观设计 | 产品外观 | 6-8月 | 15年 | 500-1000元 |

**专利申请时间节点**：
```
提交申请 → 受理（1周）→ 初审（3-12月）→ 公布（申请日18月）→ 实审（2-3年）→ 授权
```

**商标申请流程**：
```
提交申请 → 受理（1月）→ 初审公告（3月）→ 注册公告 → 颁发证书
异议期：初审公告日起3个月内可被他人提出异议
```

**软著登记流程**：
```
提交材料 → 受理（3-5工作日）→ 审查（60自然日）→ 登记（公告）
加急：30自然日
```

### 4.2 代理所选择标准

**专利代理机构选择checklist**：

| 评估项 | 权重 | 评估要点 |
|-------|-----|---------|
| 资质证书 | 必选 | 专利代理机构注册证 |
| 专业领域匹配 | 高 | 是否专注你的技术领域 |
| 代理师经验 | 高 | 执业年限+类似案例 |
| 口碑评价 | 中 | 客户评价+驳回率统计 |
| 服务透明度 | 中 | 流程可追溯+节点通知 |
| 费用结构 | 中 | 明确报价+无隐藏费用 |

**避坑指南：识别不良代理**：
- ⚠️ 承诺100%通过率（不可能）
- ⚠️ 价格极低（如发明专利代理费<2000元）
- ⚠️ 不分析技术方案直接报价
- ⚠️ 要求一次性付清全款且不退
- ⚠️ 频繁要求补充材料（赚手续费）
- ⚠️ 软著代理声称可以代签经办人（违规）
- ✅ 正规代理会先技术沟通、明确风险、合理报价

### 4.3 代理合同关键条款

**专利代理委托合同必备条款**：

| 条款类型 | 关键内容 | 谈判要点 |
|---------|---------|---------|
| 费用结构 | 官费+代理费分开列明 | 明确哪些包含/不包含 |
| 修改次数 | 初稿修改次数限制 | 争取≥3次免费修改 |
| 驳回责任 | 驳回后的处理方式 | 是否退费/部分退费 |
| 退费机制 | 什么情况下可退费 | 明确退费比例和时间 |
| 保密条款 | 技术资料保密义务 | 必须有保密协议 |
| 交付物 | 具体交付哪些文件 | 权利要求书+说明书+附图 |

**合同谈判要点**：
```
✅ 优先选择"包授权"模式（价格较高但风险代理承担）
✅ 明确"一次通过"vs"按次收费"模式
✅ 设置质量保证金（通过后再付尾款）
✅ 约定响应时间（工作日内几小时回复）
```

### 4.4 申请材料清单

**发明专利必备材料**：
- [ ] 请求书（专利名称+申请人+发明人）
- [ ] 说明书（技术领域+背景+内容+实施例）

## 模块五：申请后维护运营（Post-Grant Management）

**核心功能**：授权后维护与权利运营

### 5.1 法律状态跟踪

**专利审查进度关键节点**：

| 阶段 | 正常时长 | 异常预警 |
|------|---------|---------|
| 受理 | 1周内 | 超2周未受理→联系代理所 |
| 初审 | 3-12月 | 超12月→主动查询 |
| 公布 | 申请日18月 | 提前公开可请求提前公布 |
| 实审 | 2-3年 | 可提加快审查（专利审查高速路PPH） |
| 授权 | 收到授权通知后2月内缴费 | 逾期未缴视为撤回 |

**审查意见通知书（OA）应对**：

| 通知书类型 | 回复时限 | 应对策略 |
|-----------|---------|---------|
| 初审意见 | 2个月 | 补正形式问题 |
| 实审意见 | 3个月 | 认真分析，必要时修改权利要求 |
| 驳回决定 | 3个月内可复审 | 评估是否值得复审 |

**常见驳回理由应对模板**：

| 驳回理由 | 应对策略 | 答复要点 |
|---------|---------|---------|
| 缺乏新颖性 | 修改权利要求，增加区别特征 | 具体指出与对比文件的区别技术特征 |
| 缺乏创造性 | 技术效果论证+修改 | 论证预料不到的技术效果 |
| 说明书不支持 | 缩小保护范围 | 删除超范围的权利要求 |
| 单一性缺陷 | 分案申请或修改 | 保留核心权利要求 |

### 5.2 授权后维护规则

**专利年费管理**：

| 年度 | 年费标准(发明) | 年费标准(实用/外观) | 滞纳金(超期每月+5%) |
|------|---------------|-------------------|-------------------|
| 1-3年 | 900/900/500 | 600/600 | 逾期1个月起加收 |
| 4-6年 | 1200/1200/600 | 900/900 | ... |
| 7-9年 | 2000/2000/1000 | 1200/1200 | ... |
| 10+年 | 4000/4000/2000 | 2000/2000 | ... |

**专利维持决策矩阵**：
```
商业价值高 + 技术领先 → 按时缴费，维持保护
商业价值低 + 技术落后 → 评估放弃，节省成本
商业价值不确定 → 部分放弃/专利开放许可
```

**商标注册后维护**：

| 规则 | 说明 | 后果 |
|------|------|------|
| 3年使用义务 | 注册后3年内必须实际使用 | 可被他人申请"撤三"撤销 |
| 10年有效期 | 到期前12个月可续展 | 逾期6个月宽展期，逾期则注销 |
| 变更备案 | 注册人名义/地址变更需备案 | 未备案可能导致权利不稳定 |
| 使用证据 | 保留发票/合同/广告等证据 | 应对撤三申请 |

**软著登记后维护**：

| 规则 | 说明 |
|------|------|
| 保护期限 | 自然人：终身+50年；法人：50年 |
| 版本更新 | 重大版本更新建议重新登记 |
| 变更登记 | 软件名称/著作权人变更需办理变更 |

## 费用预警机制

| 场景 | 预估费用 | 说明 |
|------|---------|------|
| 直接申请（不做预审） | 官费+代理费 | 遭遇驳回后需重新申请 |
| 预审后申请（低风险） | 预审+官费+代理费 | 一次通过，总成本更低 |
| 预审后申请（中风险） | 预审+调整费+官费+代理费 | 修改成本可控 |
| 驳回后重申请 | 预审+官费+代理费×2 | 双倍成本，预审价值体现 |
| 申请中代理合同陷阱 | 额外手续费+时间成本 | 规避要点见模块四 |
| 软著补正超2次驳回 | 重新缴费+重新排队 | 60自然日重新计算 |

**经济决策建议**：当驳回概率>40%时，预审费用 < 驳回后重申请费用，帮助用户做出最优选择。
