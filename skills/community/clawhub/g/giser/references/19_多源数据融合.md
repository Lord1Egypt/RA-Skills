## 一、多源数据融合概述

多源数据融合是 GIS 工程中最容易出问题的环节——不同来源的数据在**坐标系、格式、精度、语义、时间**五个维度上存在差异，融合前必须逐维度对齐。

| 差异维度 | 典型场景 | 风险等级 | 根因分析 | 解决方案 |
|---------|---------|---------|---------|---------|
| **坐标系差异** | 规划数据(CGCS2000) vs 测绘数据(XA80/WGS84) | 🔴 最高 | 基准不同（椭球参数+投影方式） | 七参数转换+统一投影 |
| **格式差异** | CAD(.dwg) + GIS(.gdb) + BIM(.rvt) + 点云(.las) | 🟡 中等 | 数据结构不兼容 | 需要中间格式桥接（GML/GeoJSON） |
| **精度差异** | 1:500地形图 + 1:10000规划图 | 🔴 最高 | 比例尺差异→要素泛化不同 | 精度标注+优先级排序+重采样 |
| **语义差异** | 国标分类 vs 地方分类 vs 行业分类 | 🟡 中等 | 编码体系不一致 | 建立编码映射表+人工校验 |
| **时间差异** | 2020年影像 + 2025年矢量 | 🟡 中等 | 地物变化→位置偏移 | 变化检测+时效性标注+版本管理 |
| **尺度差异** | 无人机(cm级) + 卫星(m级) + 规划(km级) | 🔴 高 | 空间分辨率不在一个数量级 | 多尺度融合算法（小波/金字塔） |

### 1.1 融合质量评估指标体系

| 指标类型 | 具体指标 | 合格标准 | 优良标准 | 测量方法 |
|---------|---------|---------|---------|---------|
| **几何精度** | 平面中误差(RMSE) | ≤0.5m | ≤0.2m | 检查点比对 |
| **几何精度** | 高程中误差(RMSE) | ≤0.3m | ≤0.1m | 检查点比对 |

| **拓扑一致性** | 面闭合合格率 | ≥98% | ≥99.5% | 拓扑检查工具 |
| **语义一致性** | 编码映射准确率 | ≥90% | ≥95% | 人工抽样核查 |
| **时间时效性** | 数据新鲜度标注率 | 100% | 100% | 元数据检查 |

## 二、坐标系统一实战 ⭐

### 2.1 多源数据坐标系冲突典型场景

```
场景1：城市规划项目（最常见）
  来源A: 地形图 → 西安80 3度带 38带 (EPSG:2349)
  来源B: 影像 → WGS84 (EPSG:4326, 无投影)
  来源C: 管线CAD → 地方独立坐标系 (未知EPSG)
  来源D: 宗地数据 → CGCS2000 3度带 38带 (EPSG:4547)
  → 统一目标：CGCS2000 (EPSG:4547)
  → 动作：B→A→D 直接投影；C→D 需要地方参数或仿射变换

场景2：跨省数据拼接
  来源A: 广东省 → CGCS2000 3度带 38带 (中央经线114°E)
  来源B: 广西省 → CGCS2000 3度带 37带 (中央经线111°E)
  → 统一目标：CGCS2000 6度带 19带 (中央经线111°E)
  → 动作：先转地理坐标(WGS84/CGCS2000地理)，再投6度带

场景3：国际项目（一带一路）
  来源A: 中国数据 → CGCS2000
  来源B: 当地数据 → WGS84 UTM zone XX
  来源C: 历史数据 → 当地旧坐标系（参数未知）
  → 统一目标：WGS84 UTM（国际通用）
  → 动作：CGCS2000→WGS84（七参数）；当地坐标系→WGS84（三参数或仿射）
```

### 2.2 坐标系统一操作流程（七步法）

```
第1步：识别所有输入数据的坐标系
  ├ ArcGIS: 图层属性 → Source → Spatial Reference
  ├ QGIS: 图层属性 → Information → CRS
  ├ DWG: 检查有无 .prj 伴随文件，否则需询问提供方
  ├ LAS: 头部信息 LAS FCTR 记录
  └ 无坐标系标注 → ⚠️ 拒绝处理，要求提供方确认

第2步：选择目标坐标系
  ├ 优先使用项目甲方指定坐标系
  ├ 其次使用主要数据的坐标系（占比>60%）
  ├ 国家项目默认 CGCS2000
  └ 国际项目默认 WGS84 UTM

第3步：获取转换参数
  ├ 同一基准（CGCS2000内部各带）→ 直接投影变换（无需参数）
  ├ 不同基准（XA80→CGCS2000）→ 需要七参数/三参数
  │   ├ 有已知参数 → 直接使用（记录来源）
  │   ├ 有公共点 → 计算参数（至少3个均匀分布公共点）
  │   └ 无公共点 → 使用全国统一转换模型（有2-5m误差）
  ├ 地方独立坐标系 → 必须向测绘单位索要转换关系
  └ 未知坐标系 → 使用 QGIS 插件 "Proj Finder" 或人工判断

第4步：执行批量转换
  ├ 矢量 → ArcGIS Project工具 / ogr2ogr
  ├ 栅格 → ArcGIS 投影栅格 / gdalwarp
  ├ CAD → 先导入GIS再投影（自动强制加.prj）
  ├ 点云 → GlobalMapper / PDAL / LAStools
  └ 影像 → GDAL Warp (重采样方法：双线性/立方卷积)

第5步：验证转换精度
  ├ 检查公共点残差（平面<0.2m, 高程<0.3m）
  ├ 与已知参考数据叠加检查（视觉检查）
  ├ 边界处检查数据接边（无缝隙/无重叠）
  └ 转换后拓扑检查（面闭合/线连续）

第6步：处理接边差异
  ├ 接边误差<0.1m → 直接拼接
  ├ 接边误差0.1~0.5m → 橡皮页变换（Rubbersheet）
  ├ 接边误差0.5~2.0m → 仿射变换（Affine）
  └ 接边误差>2.0m → 重新检查坐标转换参数

第7步：记录转换元数据
  ├ 源坐标系 + 目标坐标系
  ├ 转换方法（七参数/三参数/投影变换）
  ├ 转换参数来源（官方/计算/估计）
  ├ 精度评估结果（RMSE）
  └ 处理人 + 日期 + 审核人
```

### 2.3 批量坐标系转换脚本（生产级）

```python
# ============================================================
# 生产级批量坐标系转换脚本
# 功能：批量转换目录下的所有矢量/栅格数据到目标坐标系
# 依赖：GDAL 3.8+ / Fiona 1.9+ / Rasterio 1.3+
# ============================================================

from osgeo import gdal, osr, ogr
from osgeo import rasterio as rio
from osgeo import fiona
from pathlib import Path
import glob
import logging

# 配置日志
logging.basicConfig(level=logging.INFO, format='%(asctime)s - %(levelname)s - %(message)s')
logger = logging.getLogger(__name__)

class CoordinateConverter:
    """坐标系批量转换器"""
    
    def __init__(self, target_epsg=4547):
        """
        初始化转换器
        :param target_epsg: 目标坐标系 EPSG 代码 (默认 CGCS2000 3度带38带)
        """
        self.target_srs = osr.SpatialReference()
        self.target_srs.ImportFromEPSG(target_epsg)
        self.target_epsg = target_epsg
        
    def convert_raster(self, src_path, dst_path=None):
        """
        转换栅格数据坐标系
        :param src_path: 源栅格路径
        :param dst_path: 输出路径（默认加后缀）
        """
        if dst_path is None:
            p = Path(src_path)
            dst_path = str(p.parent / f"{p.stem}_EPSG{self.target_epsg}{p.suffix}")
        
        try:
            # 自动检测源坐标系
            src_ds = gdal.Open(src_path)
            src_srs = osr.SpatialReference()
            src_srs.ImportFromWkt(src_ds.GetProjection())
            
            # 执行转换
            gdal.Warp(
                dst_path, src_path,
                dstSRS=self.target_srs.ExportToWkt(),
                srcSRS=src_srs.ExportToWkt(),
                resampleAlg=gdal.GRA_Bilinear,  # 双线性重采样
                targetAlignedPixels=True,  # 对齐像素
                callback=self._gdal_progress
            )
            
            logger.info(f"✅ 栅格转换完成: {src_path} → {dst_path}")
            return dst_path
            
        except Exception as e:
            logger.error(f"❌ 栅格转换失败: {src_path} - {e}")
            return None
    
    def convert_vector(self, src_path, dst_path=None):
        """
        转换矢量数据坐标系
        :param src_path: 源矢量路径
        :param dst_path: 输出路径
        """
        if dst_path is None:
            p = Path(src_path)
            dst_path = str(p.parent / f"{p.stem}_EPSG{self.target_epsg}{p.suffix}")
        
        try:
            # 使用 ogr2ogr 命令行（最稳定）
            cmd = (
                f'ogr2ogr -f "{p.suffix.upper()[1:]}" "{dst_path}" "{src_path}" '
                f'-t_srs EPSG:{self.target_epsg} -s_srs EPSG:auto -overwrite'
            )
            os.system(cmd)
            
            logger.info(f"✅ 矢量转换完成: {src_path} → {dst_path}")
            return dst_path
            
        except Exception as e:
            logger.error(f"❌ 矢量转换失败: {src_path} - {e}")
            return None
    
    def batch_convert_directory(self, src_dir, pattern="*.tif,*.shp,*.gpx"):
        """
        批量转换目录下所有匹配的文件
        :param src_dir: 源目录
        :param pattern: 文件模式（逗号分隔）
        """
        src_dir = Path(src_dir)
        patterns = [p.strip() for p in pattern.split(",")]
        
        converted = []
        failed = []
        
        for pat in patterns:
            for src_file in src_dir.glob(f"**/{pat}"):
                src_path = str(src_file)
                
                if src_path.endswith(('.tif', '.tiff', '.img', '.dem')):
                    result = self.convert_raster(src_path)
                elif src_path.endswith(('.shp', '.gpx', '.geojson', '.kml')):
                    result = self.convert_vector(src_path)
                else:
                    continue
                
                if result:
                    converted.append(result)
                else:
                    failed.append(src_path)
        
        logger.info(f"批量转换完成: 成功 {len(converted)} 个, 失败 {len(failed)} 个")
        return converted, failed
    
    def _gdal_progress(self, complete, message, unknown):
        """GDAL 进度回调"""
        if int(complete * 100) % 10 == 0:
            logger.info(f"转换进度: {complete*100:.0f}%")
        return 1

# 使用示例
if __name__ == "__main__":
    converter = CoordinateConverter(target_epsg=4547)  # CGCS2000 3度带38带
    converted, failed = converter.batch_convert_directory(
        src_dir=r"D:\YourGISData\Raw",
        pattern="*.tif,*.shp"
    )
    print(f"完成！成功: {len(converted)}, 失败: {len(failed)}")
```

### 2.4 七参数计算完整 Python 实现

```python
# ============================================================
# 七参数计算（Bursa-Wolf 模型）
# 输入：至少3个公共点（源坐标 + 目标坐标）
# 输出：七参数 (dx, dy, dz, rx, ry, rz, s)
# ============================================================

import numpy as np
from scipy.optimize import least_squares

def compute_7param(source_pts, target_pts):
    """
    计算七参数
    :param source_pts: 源坐标点列表 [(x1,y1,z1), ...] (单位: m)
    :param target_pts: 目标坐标点列表 [(x1,y1,z1), ...] (单位: m)
    :return: (dx, dy, dz, rx, ry, rz, s) 单位: m, rad, ppm
    """
    assert len(source_pts) >= 3, "至少需要3个公共点！"
    
    def residuals(params, src, tgt):
        """残差函数"""
        dx, dy, dz, rx, ry, rz, s = params
        s = 1 + s * 1e-6  # ppm → 比例因子
        
        # 旋转矩阵 (小角度近似)
        R = np.array([
            [1, -rz, ry],
            [rz, 1, -rx],
            [-ry, rx, 1]
        ])
        
        residuals = []
        for (xs, ys, zs), (xt, yt, zt) in zip(src, tgt):
            v = np.array([xs, ys, zs])
            v_t = s * R @ v + np.array([dx, dy, dz])
            residuals.extend([v_t[0]-xt, v_t[1]-yt, v_t[2]-zt])
        
        return np.array(residuals)
    
    # 初始值
    x0 = np.array([0, 0, 0, 0, 0, 0, 0])
    
    # 最小二乘求解
    result = least_squares(
        residuals, x0,
        args=(np.array(source_pts), np.array(target_pts)),
        method='lm'
    )
    
    params = result.x
    print(f"七参数计算结果：")
    print(f"  dx={params[0]:.4f}m, dy={params[1]:.4f}m, dz={params[2]:.4f}m")
    print(f"  rx={params[3]:.6f}rad, ry={params[4]:.6f}rad, rz={params[5]:.6f}rad")
    print(f"  尺度={params[6]:.2f} ppm")
    
    # 计算残差RMSE
    r = residuals(params, np.array(source_pts), np.array(target_pts))
    rmse = np.sqrt(np.mean(r**2))
    print(f"  拟合残差 RMSE = {rmse:.4f}m")
    
    return params

# 使用示例
source = [
    (365874.12, 3456789.45, 45.12),
    (365912.34, 3456823.56, 45.23),
    (365845.67, 3456756.78, 44.98),
]
target = [
    (365874.56, 3456789.89, 45.15),
    (365912.78, 3456823.98, 45.26),
    (365846.10, 3456757.21, 45.01),
]

params = compute_7param(source, target)
```

### 2.6 跨带投影转换与变形分析

```
3度带 vs 6度带 选择原则：
  - 测区跨度 < 1.5° → 使用3度带（变形小）
  - 测区跨度 1.5°~3° → 使用6度带（减少分带）
  - 测区跨度 > 3° → 使用自定义中央经线（最小变形）

跨带转换最佳实践：
  1. 先将源数据投影到地理坐标系（WGS84/CGCS2000地理）
  2. 再从地理坐标系投影到目标带的投影坐标系
  
  原因：直接从带A投影到带B会因为投影参数不同导致误差放大。
  中间经过地理坐标系（球面坐标）是最安全的路径。

长度变形对照表（高斯-克吕格投影）：
  距中央经线距离   | 3度带长度变形 | 6度带长度变形
  -----------------|-----------------|-----------------
  0km (中央经线)  | 0mm/km        | 0mm/km
  30km            | 0.07mm/km     | 0.28mm/km
  60km            | 0.28mm/km     | 1.12mm/km
  90km            | 0.63mm/km     | 2.52mm/km
  120km           | 1.12mm/km     | 4.48mm/km (超限！)
  
  结论：距中央经线 >90km 时，6度带变形可能超过测绘规范允许值（1:5000 允许 2.5mm/km）
```

### 2.7 高程基准统一（1985高程 vs 大地高）

```
高程系统差异：
  - 1985国家高程基准：正常高系统，以黄海平均海平面为基准
  - 大地高（GPS测得）：椭球高，以参考椭球面为基准
  - 两者差异：ξ = H_大地 - H_正常 （ξ为大地水准面差距，中国境内 -30m ~ +70m）

转换方法：
  1. 使用 EGM2008 重力场模型（精度 ~10-30cm）
     >>> pip install pygnss
     >>> from pygnss import geoid
     >>> xi = geoid.egm2008(lat, lon)  # 获取大地水准面差距
     >>> H_normal = H_ellipsoid - xi
  
  2. 使用国家高程异常格网模型（精度 ~5-10cm）
     - 下载：SGC2000 高程异常格网（官方）
     - 双线性内插获取 ξ
  
  3. 使用已知水准点联测（精度最高，cm级）
     - 至少3个均匀分布的水准点
     - 计算区域平均 ξ，或建立 ξ 曲面模型
```

## 三、点云与矢量配准（套合）⭐

### 3.1 点云↔矢量套合流程（标准SOP）

```
LAS/LAZ 点云 + SHP 矢量套合标准流程：

  第1步：坐标系一致性检查（必须相同！）
    ├ 点云: lasinfo 查看 VLR 中的坐标系信息
    ├ 矢量: 图层属性查看 CRS
    └ 不一致 → 先执行坐标系统一（见第二章）

  第2步：点云预处理
    ├ 去除噪声点（SOR滤波 / Statistical Outlier Removal）
    ├ 分类（Ground / Building / Vegetation / Noise）
    ├ 地面点提取（用于DEM生成）
    └ 建筑物点提取（用于建筑轮廓提取）

  第3步：特征提取
    ├ 建筑物角点 → 与矢量建筑轮廓对比
    ├ 道路边线 → 与矢量道路中线对比
    ├ 高程点 → 与矢量高程注记对比
    └ 边界线 → 与矢量地块边界对比

  第4步：计算偏移量
    ├ 方法A：手动选取同名点（至少3对）
    ├ 方法B：ICP自动配准（点云↔点云）
    ├ 方法C：Hough变换提取线特征配准
    └ 方法D：RANSAC平面拟合配准

  第5步：矢量平移/仿射变换
    ├ 平移（1参数）：整个测区统一偏移
    ├ 仿射（6参数）：允许旋转+缩放+剪切
    └ 多项式（n阶）：复杂非线性变形

  第6步：验证套合精度
    ├ 检查点法：独立检查点验证（不参与配准）
    ├ 重叠区分析法：检查叠加后差异
    └ 剖面法：沿剖面线检查高程/平面差异

  第7步：迭代优化
    ├ 残差 > 0.5m → 重新选取同名点
    ├ 残差 0.2~0.5m → 可接受（1:500标准）
    └ 残差 < 0.2m → 优秀（1:500高精度）
```

### 3.2 常用套合工具与参数对比

| 工具 | 适用场景 | 关键操作 | 精度 | 批量能力 |
|------|---------|---------|------|---------|

### 3.3 CloudCompare ICP 配准实战（详细步骤）

```
CloudCompare ICP 配准详细步骤：

  准备工作：
    - 确保 reference 和 aligned 点云坐标系一致
    - 确保两个点云有重叠区域（至少30%重叠）
    - 预处理：去噪声、降采样（加速计算）

  第1步：加载点云
    File → Open → 选择 reference.ply / aligned.ply
    （可以同时加载多个点云，按住 Ctrl 多选）

  第2步：手动大致对齐（非常重要！）
    - ICP 需要初始对齐（<10°旋转，<1m平移）
    - 选中两个点云 → Edit → Translate/Rotate
    - 使用鼠标拖拽，使重叠区域大致对齐
    - 技巧：打开 "View → Toggle Centered Perspective"
    
  第3步：运行 ICP
    - 选中两个点云（先选 reference，再 Ctrl+选 aligned）
    - Tools → Registration → Fine Registration (ICP)
    
  第4步：参数设置（关键！）
    - RMS Difference: < 0.01 (收敛阈值，单位m)
    - Final overlap: 80%+ (预期重叠率）
    - Random sampling limit: 50000 点 (多→准但慢）
    - Enable farthest point removal: ON (去除离群点）
    - Adjust scale: OFF (除非有尺度差异）
    - Rotation/Translation filtering: ON (过滤大变形）
    
  第5步：查看配准报告
    - RMS (Root Mean Square) → 应 < 0.05m（优秀）
    - 旋转矩阵 → 检查是否合理（不应有大角度旋转）
    - 平移向量 → 检查是否合理
    - 收敛迭代次数 → >50次可能初始对齐不好
    
  第6步：应用变换
    - 如果结果满意 → Apply Transform
    - 导出配准后的点云：File → Save
    
  第7步：验证精度
    - 在重叠区域选取检查点
    - 测量对应点距离 → 应 < 0.1m
    - 剖面检查：沿剖面线检查配准质量
```

### 3.4 点云与影像配准（全景影像 + LiDAR）

```
应用场景：移动测量车（MLS）同时采集 LiDAR 点云 + 全景影像
需要：将点云和影像精确配准，实现"点云着色"或"影像点云融合"

配准方法：
  方法A：GNSS/IMU 辅助配准（最佳）
    - 移动测量车自带 GNSS/IMU 轨迹
    - 点云每个点有时间戳 → 线性内插位姿
    - 影像每个帧有时间戳 → 直接获取位姿
    - 配准精度：cm级（取决于GNSS/IMU精度）

  方法B：特征点匹配配准
    - 从点云生成强度图像（Intensity Image）
    - 从全景影像提取 SIFT/SURF 特征点
    - 特征点匹配 → 计算变换矩阵
    - 配准精度：dm~m级（取决于纹理）

  方法C：人工选同名点
    - 在点云强度图中选点
    - 在全景影像中对应位置选点
    - 至少6对点（求解投影矩阵）
    - 配准精度：m级

Python 实现（点云着色）：
```

```python
# 点云与全景影像配准 + 着色
import numpy as np
import open3d as o3d
from PIL import Image
import pyntcloud

# 1. 加载点云
pcd = o3d.io.read_point_cloud("point_cloud.ply")
points = np.asarray(pcd.points)

# 2. 加载全景影像
pano_img = np.array(Image.open("panorama.jpg"))

# 3. 位姿数据（从GNSS/IMU轨迹获取）
poses = load_poses("trajectory.txt")  # 每个点对应位姿

# 4. 球面投影（点→像素坐标）
for i, (x, y, z) in enumerate(points):
    # 获取该点的位姿
    pose = poses[i]
    # 转换到相机坐标系
    p_cam = pose.R @ np.array([x, y, z]) + pose.t
    # 球面投影到全景
    theta = np.arctan2(p_cam[1], p_cam[0])  # 水平角
    phi = np.arcsin(p_cam[2] / np.linalg.norm(p_cam))  # 俯仰角
    # 映射到像素坐标
    u = int((theta + np.pi) / (2*np.pi) * pano_img.shape[1])
    v = int((np.pi/2 - phi) / np.pi * pano_img.shape[0])
    # 获取颜色
    color = pano_img[v, u] / 255.0
    # 赋值给点
    pcd.colors[i] = color

# 5. 保存着色点云
o3d.io.write_point_cloud("colored_point_cloud.ply", pcd)
```

### 3.5 精度评估与残差分析

| 评估方法 | 原理 | 优点 | 缺点 | 适用场景 |
|---------|------|------|------|---------|
| **检查点法** | 独立检查点比对 | 最客观，无偏差 | 需要额外采集检查点 | 高精度项目 |
| **重叠区分析法** | 重叠区差异统计 | 不需要额外数据 | 只能评估重叠区 | 任意项目 |
| **剖面法** | 沿剖面线检查差异 | 直观，易于发现系统误差 | 只能检查剖面线 | 道路/管线 |
| **DEM差分法** | 生成DEM后差分 | 全面覆盖 | 受DEM内插影响 | 地形数据 |
| ** Hausdorff距离** | 最大最近点距离 | 严格，捕捉离群误差 | 对噪声敏感 | 高精度要求 |

```python
# 配准精度评估 Python 实现
def evaluate_registration_accuracy(ref_pts, aligned_pts, transform_matrix):
    """
    评估配准精度
    :param ref_pts: 参考点 (N×3)
    :param aligned_pts: 待配准原始点 (N×3)
    :param transform_matrix: 4×4 变换矩阵
    :return: 精度评估报告
    """
    # 应用变换
    aligned_homo = np.hstack([aligned_pts, np.ones((len(aligned_pts), 1))])
    aligned_transformed = (transform_matrix @ aligned_homo.T).T[:, :3]
    
    # 计算残差
    residuals = np.linalg.norm(ref_pts - aligned_transformed, axis=1)
    
    # 统计
    report = {
        'mean_error': np.mean(residuals),
        'rmse': np.sqrt(np.mean(residuals**2)),
        'max_error': np.max(residuals),
        'median_error': np.median(residuals),
        'std_error': np.std(residuals),
        'percentile_95': np.percentile(residuals, 95),
    }
    
    # 输出报告
    print("=" * 50)
    print("配准精度评估报告")
    print("=" * 50)
    for k, v in report.items():
        print(f"  {k}: {v:.4f}m")
    print("=" * 50)
    
    # 可视化残差分布
    import matplotlib.pyplot as plt
    plt.hist(residuals, bins=50, edgecolor='black')
    plt.xlabel('残差 (m)')
    plt.ylabel('频次')
    plt.title('配准残差分布')
    plt.show()
    
    return report
```

# 1. 栅格格式转换 (GDAL)
from osgeo import gdal

def raster_convert(src_path, dst_path, dst_format="GTiff"):
    """
    栅格格式转换
    :param src_path: 源路径
    :param dst_path: 目标路径
    :param dst_format: 目标格式驱动名
    """
    src_ds = gdal.Open(src_path)
    driver = gdal.GetDriverByName(dst_format)
    dst_ds = driver.CreateCopy(dst_path, src_ds, strict=0)
    dst_ds.FlushCache()
    print(f"✅ 转换完成: {src_path} → {dst_path}")
    return dst_ds

# 2. 矢量格式转换 (OGR)
from osgeo import ogr

def vector_convert(src_path, dst_path, dst_format="ESRI Shapefile"):
    """
    矢量格式转换
    """
    src_ds = ogr.Open(src_path)
    src_layer = src_ds.GetLayer()
    
    driver = ogr.GetDriverByName(dst_format)
    dst_ds = driver.CreateDataSource(dst_path)
    dst_layer = dst_ds.CopyLayer(src_layer, src_layer.GetName())
    
    print(f"✅ 转换完成: {src_path} → {dst_path}")
    return dst_ds

# 3. 坐标系转换 + 格式转换 一步完成
def convert_with_reprojection(src_path, dst_path, dst_epsg=4547):
    """
    格式转换 + 坐标系统一
    """
    cmd = f'ogr2ogr -f "ESRI Shapefile" "{dst_path}" "{src_path}" -t_srs EPSG:{dst_epsg}'
    os.system(cmd)
    print(f"✅ 转换+投影完成")
