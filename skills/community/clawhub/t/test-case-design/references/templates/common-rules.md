## 零、测试用例编写通用规则（所有测试类型必须遵守）

1. **步骤必须给出具体数据**：测试步骤中必须直接给出具体的输入值、参数、操作对象，让测试人员可以照抄执行，不得使用描述性语言代替。
   - ❌ 不合格：`输入超长字符串`、`输入特殊字符`、`输入测试攻击向量`、`填写表单必填字段`、`发送带错误参数的请求`
   - ✅ 合格：`输入 ' OR '1'='1`、`输入 <script>alert(1)</script>`、`输入 用户名:testuser, 密码:Test@123456`、`发送请求，参数 name 设为空字符串 ""`
2. **大量数据例外**：当测试需要输入大量内容时，允许用描述性语言 + 明确参数说明，无需完整写出全部内容。大量数据的判定标准：
   - 文本输入 > 1000 个字符
   - 文件上传 > 1MB
   - 列表/表格数据 > 50 行
   - 其他明显不适合完整写出的场景
   - ✅ 合格：`在文本框中粘贴一段 10000 个汉字的文本（可重复"测试"二字 5000 次）`
   - ✅ 合格：`上传一个大小为 500MB 的压缩包文件（内容为随机生成数据）`
   - ✅ 合格：`在搜索框中输入一个包含 2000 个字符的随机字符串（可用 Python: "A"*2000 生成）`

## 一、测试类型分类

### 1. 功能测试
验证功能是否按需求规格正常工作，确保所有功能点正确实现。

**子类型**：增删改查（CRUD）、表单验证、状态流转、业务规则、数据校验

**示例**：登录功能、表单提交、数据查询、列表操作、状态流转

---

### 3. 兼容性测试
验证系统在不同环境下的兼容性，确保一致体验。

**子类型**：平台兼容、浏览器兼容、设备兼容、版本兼容

**示例**：不同机型适配、不同浏览器布局、版本升级兼容

---

### 4. UI 测试
验证界面显示和交互是否正确，确保用户体验。

**子类型**：布局测试、样式测试、交互测试、主题测试

**示例**：页面错位、样式不一致、动画卡顿

---

### 5. 性能测试
验证系统的性能指标是否达标，确保流畅体验。

**子类型**：响应时间、并发能力、资源占用、稳定性

**示例**：首屏加载慢、卡顿闪退、高并发崩溃

---

### 6. 可用性测试
验证系统的易用性和用户体验，确保易学易用。

**子类型**：导航测试、表单测试、帮助测试、可访问性

**示例**：操作繁琐、提示不明确、无法键盘操作

---

### 7. 联动测试
验证不同模块/组件之间的数据同步和状态联动。

**子类型**：表单联动、列表联动、搜索联动、状态联动、数据联动

**示例**：省市区联动、筛选条件联动、跨页面状态同步

---

### 8. 路由测试
验证页面路由跳转、参数传递和权限控制。

**子类型**：导航跳转、浏览器导航、路由参数、深链接

**示例**：直接访问 URL、404 处理、登录拦截

## 二、优先级判定规则

### P0 - 致命（Critical）
- 核心功能
- 影响主流程
- 导致系统崩溃
- 数据丢失

**处理**：立即修复，阻塞发布

---

### P1 - 严重（Major）
- 重要功能
- 影响用户体验
- 功能错误
- 界面严重问题

**处理**：尽快修复，影响发布

---

### P2 - 一般（Minor）
- 次要功能
- 轻微体验问题
- 界面小问题
- 提示信息不准确

**处理**：排期修复，可带病发布

---

### P3 - 建议（Trivial）
- 优化建议
- 界面美化
- 不影响功能

**处理**：有空再改

---

## 三、用例编号规则

### 编号格式

```
[平台]_[模块]_[维度]_[序号]
```

### 平台前缀

| 平台 | 前缀 |
|-----|------|
| 移动端 App | APP |
| 桌面端 | DESKTOP |
| 小程序 | MP |
| 移动 Web | H5 |
| PC Web 端 | WEB |
| 通用功能测试 | FUNC |
| 通用联动测试 | LINKAGE |
| 通用路由测试 | ROUTING |
| Agent 测试 | AGENT |

| 维度 | 缩写 |
|-----|------|
| 手势操作 | GESTURE |
| 屏幕适配 | SCREEN |
| 中断恢复 | INTERRUPT |
| 网络切换 | NETWORK |
| 设备兼容 | COMPAT |
| 权限管理 | PERMISSION |
| 系统交互 | SYSTEM |
| 性能体验 | PERFORMANCE |
| 生命周期 | LIFECYCLE |
| 接口测试 | API |
| UI 视觉 | UI |
| 联动测试 | LINKAGE |
| 路由测试 | ROUTING |
| 支付测试 | PAYMENT |
| 数据同步 | SYNC |
| 授权管理 | AUTH |
| 浏览器兼容 | BROWSER |

### 简化格式（通用测试用例）

对于通用测试用例，可使用简化格式：

```
TC_[测试类型前缀]_[场景关键词]_[序号]
```

简化前缀对照：

| 完整前缀 | 简化前缀 | 适用场景 |
|---------|---------|---------|
| FUNC_CRUD | FUNC | 功能测试-增删改查 |
| FUNC_LIST | FUNC | 功能测试-列表功能 |
| FUNC_FORM | FUNC | 功能测试-表单验证 |
| FUNC_STATUS | FUNC | 功能测试-状态管理 |
| LINKAGE | LINKAGE | 联动测试 |
| ROUTING | ROUTING | 路由测试 |
| API | API | 接口测试 |
| UI | UI | UI 视觉测试 |
| AGENT | AGENT | Agent 测试 |

简化格式示例：

```
TC_FUNC_CRUD_001     - 功能测试-新增数据第1个用例

TC_API_GET_001       - 接口测试-GET请求第1个用例

TC_AGENT_TASK_001    - Agent测试-任务完成度第1个用例
```
