# 中国法律法规合规检查要点

本文档列出中国主要法律法规的软件合规检查要点，供合规审查时参考。
每部法律均含：适用范围、合规检查点、代码中需检查项、**法律责任（行政+刑事）**。

---

## 1. 《中华人民共和国网络安全法》（2017年6月1日生效）

### 适用范围
在中华人民共和国境内建设、运营、维护和使用网络，以及网络安全的监督管理。

### 关键合规检查点

#### 网络安全等级保护（第21条）
- [ ] 是否按照网络安全等级保护制度的要求，履行安全保护义务
- [ ] 是否对系统进行等级保护备案（二级以上系统）
- [ ] 是否定期开展等级保护测评

#### 用户信息保护（第22条、第41-44条）
- [ ] 收集用户信息是否明示目的、方式和范围
- [ ] 是否经被收集者同意
- [ ] 是否提供删除/更正个人信息的功能
- [ ] 是否存在泄露、篡改、毁损个人信息的行为

#### 网络安全事件应急预案（第25条）
- [ ] 是否制定网络安全事件应急预案
- [ ] 是否定期演练

#### 关键信息基础设施保护（第31-38条）
- [ ] 如属于关键信息基础设施运营者（CIIO），是否履行了额外义务
- [ ] 是否进行每年一次的安全检测评估

#### 数据境内存储（第37条）
- [ ] 如为CIIO，是否在境内存储个人信息和重要数据
- [ ] 确需向境外提供的，是否通过安全评估

### 代码中需检查项
- 用户数据收集是否明确告知并获同意
- 是否有数据加密存储（传输中+静态）
- 是否有访问日志和审计日志
- 是否有数据泄露检测机制
- 是否有安全配置（密码策略、会话管理）

### ⚖️ 法律责任（网络安全法）

#### 行政责任（第59-75条）

| 违法行为 | 处罚主体 | 行政处罚内容 |
|----------|----------|-------------|
| 未按规定履行网络安全保护义务（第59条） | 网信部门/有关部门 | 警告 + 最高10万元罚款；拒不改正或导致危害网络安全后果：最高100万元罚款 + 停业整顿 |
| 关键信息基础设施运营者未履行义务（第59条第2款） | 网信部门/有关部门 | 最高100万元罚款；拒不改正或导致危害后果：最高1000万元罚款 + 停业整顿 |
| 个人信息被窃取/泄露（第64条） | 网信部门/有关部门 | 没收违法所得 + 最高100万元罚款；情节严重：最高1000万元罚款 + 停业整顿 |
| 非法出售个人信息（第64条第2款） | 网信部门/有关部门 | 没收违法所得 + 并处违法所得1-10倍罚款；无违法所得或不足100万元：最高1000万元罚款 |
| 危害网络安全活动（第63条） | 公安机关 | 行政拘留（5-15日）+ 罚款（最高50万元）|
| 网络运营者拒不改正或情节严重（多项条款） | 网信部门/有关部门 | 最高100万元/1000万元罚款 + 停业整顿 + 吊销营业执照 |

**对直接负责主管人员的处罚（第64、65、66、68、69、70条）：**
- 个人罚款：1万元以上10万元以下
- 可并处：行政拘留、禁止一定期限内担任相关企业董监高

#### 刑事责任（刑法相关条款）

网络安全法本身不直接规定刑事责任，但以下《刑法》条款适用于网络安全违法行为：

| 罪名 | 刑法条款 | 量刑标准 |
|------|----------|----------|
| **非法侵入计算机信息系统罪** | 第285条第1款 | 有期徒刑/拘役（3年以下）|
| **非法获取计算机信息系统数据、非法控制计算机信息系统罪** | 第285条第2款 | 情节严重：3年以下；情节特别严重：3-7年 + 罚金 |
| **提供侵入、非法控制计算机信息系统程序、工具罪** | 第285条第3款 | 情节严重：3年以下；情节特别严重：3-7年 + 罚金 |
| **破坏计算机信息系统罪** | 第286条 | 5年以下；后果特别严重：5年以上 |
| **拒不履行信息网络安全管理义务罪** | 第286条之1 | 拒不改正致违法信息大量传播/用户信息泄露：3年以下 + 罚金 |
| **侵犯公民个人信息罪** | 第253条之1 | 情节严重：3年以下 + 罚金；情节特别严重：3-7年 + 罚金 |

> **⚠️ 提示：** 网络运营者拒不履行安全管理义务，致使用户信息泄露、造成严重后果的，可同时追究单位的行政责任和直接责任人的刑事责任。

## 2. 《中华人民共和国数据安全法》（2021年9月1日生效）

### 适用范围
在境内开展数据处理活动及其安全监管。

### 关键合规检查点

#### 数据分类分级（第21条）
- [ ] 是否建立数据分类分级管理制度
- [ ] 是否对核心数据、重要数据、一般数据进行分级
- [ ] 重要数据处理者是否明确数据安全负责人和管理机构

#### 重要数据出境（第31条）
- [ ] 重要数据出境是否通过安全评估
- [ ] 是否向境外司法/执法机构提供境内存储的数据（需经主管机关批准）

#### 数据安全事件应急预案（第29条）
- [ ] 是否制定数据安全事件应急预案
- [ ] 是否及时处置系统漏洞、计算机病毒、网络攻击等风险

#### 数据交易（第33-35条）
- [ ] 如从事数据交易中介服务，是否审核数据来源和交易双方的身份
- [ ] 是否留存审核记录

### 代码中需检查项
- 数据分级标签是否在代码中体现
- 重要数据出境是否有审批流程
- 是否有数据访问权限控制
- 是否有数据操作审计日志

### ⚖️ 法律责任（数据安全法）

#### 行政责任（第44-52条）

| 违法行为 | 处罚主体 | 行政处罚内容 |
|----------|----------|-------------|
| 未按规定履行数据安全保护义务（第45条） | 网信部门 | 警告 + 最高50万元罚款；拒不改正或造成大量数据泄露：最高200万元罚款 + 停业整顿 |
| 危害国家安全和利益的数据处理活动（第46条） | 国家安全机关/公安机关 | 最高500万元罚款 + 停业整顿；对责任人最高100万元罚款 |
| 从事数据交易中介服务未履行审核义务（第48条） | 网信部门 | 最高100万元罚款 + 停业整顿 |
| 拒不配合数据安全审查（第49条） | 网信部门 | 最高100万元罚款 + 停业整顿 |
| 将核心数据/重要数据非法提供给境外（第50条） | 网信部门 | 最高500万元罚款 + 停业整顿；对责任人最高100万元罚款 |
| 未经安全评估向境外提供重要数据（第46条参照） | 网信部门 | 最高500万元罚款 |

**对直接负责主管人员的处罚：**
- 个人罚款：1万元以上10万元以下；情节严重的：10万元以上100万元以下
- 可并处：禁止一定期限内担任相关企业董监高

#### 刑事责任

| 罪名 | 刑法条款 | 量刑标准 |
|------|----------|----------|
| **为境外窃取、刺探、收买、非法提供国家秘密、情报罪** | 第111条 | 情节较轻：5-10年；情节严重：10年以上或无期；特别严重时可判死刑 |
| **侵犯商业秘密罪** | 第219条 | 情节严重：3年以下 + 罚金；情节特别严重：3-10年 + 罚金 |
| **侵犯公民个人信息罪** | 第253条之1 | 情节严重：3年以下 + 罚金；情节特别严重：3-7年 + 罚金 |
| **拒不履行信息网络安全管理义务罪** | 第286条之1 | 3年以下 + 罚金 |

> **⚠️ 高危提示：** 向境外非法提供核心数据/重要数据，可能构成《刑法》第111条「为境外窃取、刺探、收买、非法提供国家秘密、情报罪」，最高可判**死刑**。

## 3. 《中华人民共和国个人信息保护法》（PIPL，2021年11月1日生效）

### 适用范围
在境内处理自然人个人信息的活动。

### 关键合规检查点

#### 处理个人信息的合法性基础（第13条）
- [ ] 是否基于同意处理个人信息（查看是否有同意记录）
- [ ] 是否为订立/履行合同所必需
- [ ] 是否为履行法定职责所必需

#### 告知-同意规则（第17-23条）
- [ ] 处理个人信息前是否以显著方式告知
- [ ] 处理敏感个人信息是否取得单独同意
- [ ] 向其他个人信息处理者提供个人信息是否取得单独同意

#### 个人信息主体权利（第44-47条）
- [ ] 是否提供查阅、复制个人信息的途径
- [ ] 是否提供更正、补充个人信息的途径
- [ ] 是否提供删除个人信息的途径
- [ ] 是否提供撤回同意的途径

#### 跨境提供个人信息（第38-43条）
- [ ] 跨境提供是否通过安全评估/认证/标准合同
- [ ] 是否告知个人信息主体境外接收方的身份、联系方式、处理目的等
- [ ] 是否取得个人信息主体的单独同意

#### 自动化决策（第24条）
- [ ] 利用个人信息进行自动化决策是否明示
- [ ] 是否提供不针对个人特征的选项

#### 未成年人个人信息（第31条）
- [ ] 处理14周岁以下未成年人个人信息是否取得监护人同意

### 代码中需检查项
- 隐私政策是否完整、易读、可访问
- 是否有Cookie/追踪技术的告知和同意机制
- 是否有"撤回同意"的功能入口
- 是否有数据主体权利行使的API（查阅/删除/更正）
- 敏感个人信息处理是否有单独授权
- 数据跨境传输是否有合规机制
- 日志中是否意外记录个人敏感信息（明文密码、身份证号等）

### ⚖️ 法律责任（个人信息保护法）

#### 行政责任（第66-71条）

| 违法行为 | 处罚主体 | 行政处罚内容 |
|----------|----------|-------------|
| 处理个人信息侵害个人信息权益，未造成损害（第66条第1款） | 网信部门 | 警告 + 最高100万元罚款；责令改正 |
| 违反规定处理个人信息，拒不改正或情节严重（第66条第2款） | 网信部门 | 最高5000万元或上一年度营业额5%罚款 + 停业整顿 + 吊销营业执照 |
| 个人信息处理者未尽到保护义务，造成信息泄露等后果（第69条） | 网信部门 | 按第66条处罚；同时承担民事赔偿责任 |
| 拒绝、阻碍个人信息保护监督检查（第70条） | 网信部门 | 最高100万元罚款；对责任人最高10万元罚款 |

**对直接负责主管人员的处罚（第66条第2款）：**
- 个人罚款：1万元以上10万元以下；情节严重的：10万元以上100万元以下
- 可并处：禁止一定期限内担任相关企业董监高

> **💰 罚款力度说明：** PIPL 第66条第2款的罚款上限为「5000万元以下或上一年度营业额5%」，是迄今为止中国法律中对违法行为罚款力度最大的条款之一（超过GDPR的2000万欧元或全球营业额4%）。

#### 刑事责任

| 罪名 | 刑法条款 | 量刑标准 |
|------|----------|----------|
| **侵犯公民个人信息罪** | 第253条之1 | 情节严重：3年以下 + 罚金；情节特别严重：3-7年 + 罚金 |
| **拒不履行信息网络安全管理义务罪** | 第286条之1 | 致个人信息泄露造成严重后果：3年以下 + 罚金 |
| **非法获取计算机信息系统数据罪** | 第285条第2款 | 情节严重：3年以下；情节特别严重：3-7年 + 罚金 |

**「情节严重」认定标准（司法解释）：**
- 出售/提供行踪轨迹信息、通信内容、征信信息、财产信息50条以上
- 出售/提供住宿信息、通信记录、健康生理信息、交易信息等其他可能影响人身、财产安全的个人信息500条以上
- 出售/提供前两项规定以外的个人信息5000条以上
- 违法所得5000元以上

> **⚠️ 高危提示：** 非法出售/提供公民个人信息达到「情节特别严重」标准（一般为上述数量10倍），可判处**3-7年有期徒刑**。单位犯罪同时对单位判处罚金，并对直接责任人员判处刑罚。

## 4. 《网络安全审查办法》（2022年2月15日修订生效）

### 适用范围
关键信息基础设施运营者采购网络产品和服务，网络平台运营者开展数据处理活动。

### 关键合规检查点
- [ ] 如掌握超过100万用户个人信息，赴国外上市是否申报网络安全审查
- [ ] 采购网络产品和服务是否可能影响国家安全的，是否申报网络安全审查

### ⚖️ 法律责任

#### 行政责任
- 应当申报网络安全审查而未申报的：由网信部门责令限期改正；拒不改正或造成严重后果的，处最高100万元罚款，对责任人处最高10万元罚款
- 提交虚假材料或隐瞒有关情况的：处最高100万元罚款

#### 刑事责任
- 如隐瞒情况导致国家安全受到危害，可能构成《刑法》第111条「为境外窃取、刺探、收买、非法提供国家秘密、情报罪」

## 5. 《计算机软件保护条例》（2002年1月1日生效，2013年修订）

### 适用范围
软件和软件开发相关的著作权保护。

### 关键合规检查点
- [ ] 是否使用了未经授权的第三方库/代码
- [ ] 开源协议（GPL等）是否合规使用
- [ ] 是否保留了第三方库的版权声明
- [ ] 自研代码是否有著作权声明和版本记录

### 代码中需检查项
- 第三方依赖的许可证类型（LICENSE文件）
- 是否有GPL等强Copyleft许可证的依赖
- 代码注释中是否包含他人版权声明被盗用

### ⚖️ 法律责任

#### 行政责任（第24条、第30条）
- 未经许可复制/发行/表演/放映/广播/汇编/通过信息网络向公众传播作品的：
  - 责令停止侵权 + 没收违法所得 + 没收侵权复制品 + 罚款（非法经营额1-5倍，无非法经营额或难以计算的最高25万元）

#### 刑事责任（刑法第217条 侵犯著作权罪）
- **情节严重**（非法经营额5万元以上/复制品数量1000份以上/其他严重情节）：
  - 3年以下有期徒刑 + 罚金
- **情节特别严重**（非法经营额25万元以上/复制品数量5000份以上）：
  - 3-10年有期徒刑 + 罚金

> **⚠️ 提示：** 在软件中使用盗版/未授权代码，不仅面临民事赔偿，金额巨大时（5万元以上）可追究刑事责任。

## 6. 《中华人民共和国电子商务法》（2019年1月1日生效）

### 适用范围
电子商务经营者（如平台、商家）。

### 关键合规检查点（如适用）
- [ ] 是否依法办理市场主体登记（除外情形除外）
- [ ] 是否明示用户信息查询、更正、删除以及用户注销的方式
- [ ] 是否保障网络安全，依法采取加密等安全技术措施

### ⚖️ 法律责任

#### 行政责任（第76-88条）
- 未明示用户信息查询、更正、删除以及用户注销的方式：最高50万元罚款
- 不履行网络安全保障义务：最高200万元罚款；情节严重的停业整顿
- 电子商务平台经营者未按规定履行审核义务：最高200万元罚款

#### 刑事责任
- 明知平台内经营者侵犯知识产权，未采取必要措施的，与侵权人承担连带责任；情节严重的，可能构成《刑法》第218条「销售侵权复制品罪」（5年以下+罚金）

## 7. 《关键信息基础设施安全保护条例》（2021年9月1日生效）

### 适用范围
关键信息基础设施运营者（公共通信、能源、交通、水利、金融、公共服务、电子政务等）。

### 关键合规检查点
- [ ] 是否设置专门安全管理机构
- [ ] 是否每年至少进行一次网络安全检测和风险评估
- [ ] 是否优先采购安全可信的网络产品和服务

### ⚖️ 法律责任（第39-43条）

#### 行政责任
- 未按规定履行安全保护义务：最高100万元罚款；拒不改正或导致重大网络安全事件的：最高1000万元罚款 + 停业整顿
- 对直接负责主管人员：最高10万元罚款；情节严重的：10万-100万元罚款

#### 刑事责任
- 造成重大网络安全事件，致使国家利益遭受重大损失的，可能构成《刑法》第286条「破坏计算机信息系统罪」（5年以下；后果特别严重：5年以上）

## 8. 《中华人民共和国刑法》相关条款汇总（软件合规重点）

> 本节汇总与软件/数据合规直接相关的主要刑法罪名，审查时应重点关注。

| 罪名 | 条款 | 构成要件（简要） | 量刑 |
|------|------|----------------|------|
| **侵犯公民个人信息罪** | 第253条之1 | 违反国家规定，向他人出售/提供公民个人信息，情节严重 | 3年以下+罚金；情节特别严重：3-7年+罚金 |
| **非法侵入计算机信息系统罪** | 第285条第1款 | 侵入国家事务、国防建设、尖端科学技术领域的计算机信息系统 | 3年以下+拘役 |
| **非法获取计算机信息系统数据罪** | 第285条第2款 | 侵入除第285条第1款规定以外的计算机信息系统，获取该系统中存储、处理或传输的数据，情节严重 | 3年以下+罚金；情节特别严重：3-7年+罚金 |
| **破坏计算机信息系统罪** | 第286条 | 对计算机信息系统功能进行删除、修改、增加、干扰，造成系统不能正常运行，后果严重 | 5年以下；后果特别严重：5年以上 |
| **拒不履行信息网络安全管理义务罪** | 第286条之1 | 网络服务提供者不履行法律、行政法规规定的信息网络安全管理义务，经监管部门责令采取改正措施而拒不改正，致使用户信息泄露/违法信息大量传播等 | 3年以下+罚金 |
| **侵犯著作权罪** | 第217条 | 未经著作权人许可，复制发行/通过信息网络向公众传播其作品，违法所得数额较大或有其他严重情节 | 3年以下+罚金；情节特别严重：3-10年+罚金 |
| **为境外窃取、刺探、收买、非法提供国家秘密、情报罪** | 第111条 | 为境外的机构、组织、人员窃取、刺探、收买、非法提供国家秘密或情报 | 5-10年；情节特别严重：10年以上或无期；可判死刑 |
| **非法利用信息网络罪** | 第287条之1 | 设立用于实施诈骗、传授犯罪方法、制作或销售违禁物品等法律法规禁止的物品、物品的网站、通讯群组，情节严重 | 3年以下+罚金 |
| **帮助信息网络犯罪活动罪** | 第287条之2 | 明知他人利用信息网络实施犯罪，为其提供互联网接入、服务器托管、网络存储、通讯传输等技术支持，或提供广告推广、支付结算等帮助，情节严重 | 3年以下+罚金 |

### 单位犯罪（第30-31条）
- 单位犯上述罪行：**对单位判处罚金**，并对其直接负责的主管人员和其他直接责任人员**判处刑罚**。
- 对于软件公司而言，公司本身可能因合规失职被处罚金，同时法定代表人、CTO、产品经理等负责人可能被追究刑事责任。

## 通用合规建议

1. **隐私政策**：确保隐私政策完整、易懂、可访问，涵盖PIPL要求的所有内容
2. **用户协议**：确保用户协议公平，不含霸王条款
3. **数据加密**：传输中（TLS 1.2+）和静态（AES-256等）均需加密
4. **访问控制**：基于最小权限原则，实施RBAC或ABAC
5. **审计日志**：记录所有敏感操作，保留至少6个月
6. **数据保留**：建立数据保留和销毁策略，不无限期保留个人信息
7. **供应商管理**：对处理数据的供应商进行尽职调查，签署DPA（数据处理协议）
8. **安全培训**：对开发人员和运维人员进行数据安全培训
9. **刑事风险排查**：定期排查是否存在可能被认定为「侵犯公民个人信息罪」「拒不履行信息网络安全管理义务罪」的行为
10. **应急预案**：制定数据安全事件应急预案，确保72小时内报告（PIPL要求）
