## 一、成果汇交制度概述

### 1.1 法律依据

- **《测绘法》第三十三条**：测绘项目完成后应向测绘地理信息主管部门汇交成果
- **《测绘成果管理条例》**：规定汇交范围、程序和时间

### 1.2 汇交范围

```
基本汇交内容：
├── 测绘成果数据（DLG/DEM/DOM/DRG/三维模型）
├── 测绘成果文档（技术设计书/技术总结/检查报告/验收报告）
├── 元数据文件
├── 控制测量成果
└── 专业技术总结
```

## 二、文件命名规范

### 2.1 DLG 成果命名

```
格式：{项目编号}_{图幅编号}_{比例尺代码}_{要素类型}_{版本号}.{扩展名}

示例：
  BJ2025-K50E001001-1K-DLG-V01.gpkg
  └─┬──┘ └───┬────┘ └┬┘ └┬┘ └┬┘ └─┬─┘
    项目    图幅编号   比例尺 类型 版本  格式

比例尺代码：
  500  → 1:500
  1K   → 1:1,000
  2K   → 1:2,000
  5K   → 1:5,000
  10K  → 1:10,000
  50K  → 1:50,000
```

### 2.2 DEM/DOM 成果命名

```
DEM：{项目编号}_{图幅编号}_{分辨率}m_DEM_{版本号}.tif
     示例：BJ2025-K50E001001-2m-DEM-V01.tif

DOM：{项目编号}_{图幅编号}_{分辨率}m_DOM_{版本号}.tif
     示例：BJ2025-K50E001001-0.2m-DOM-V01.tif
```

### 2.3 图幅编号规则

采用 **国家基本比例尺地形图分幅与编号标准（GB/T 13989）**：

```
1:100万  → J-50（字母+数字，4°×6°分幅）
1:50万   → J-50-A（16幅）
1:25万   → J-50-[1]~[16]（96幅）
1:10万   → J-50-1~144（144幅）
1:5万    → J-50-1-A~D（576幅）
1:2.5万  → J-50-1-A-1~4（2304幅）
1:1万    → J-50-1-(1)~(64)（9216幅）
1:5000   → J-50-1-(1)-a~d（36864幅）
```

> 实际项目中常用编号软件：ArcGIS Pro 地图系列 / 国家测绘局分幅工具

## 四、技术报告模板

### 4.1 技术设计书（纲要）

```
一、项目概况
  - 项目名称、目的、范围（含项目区位图）
  - 任务来源与依据
  - 测区概况（地理位置、地形地貌、交通条件）

二、已有资料分析
  - 已有控制成果
  - 已有地形图/影像
  - 资料可用性评估

三、技术方案
  - 技术路线：控制→航摄→空三→立体采集→DEM/DOM→DLG→质检
  - 坐标系统：CGCS2000 / 1985高程基准
  - 投影方式：高斯-克吕格3°带 中央子午线×××°
  - 软件配置：ArcGIS Pro 3.4 / CASS 11.0 / GlobalMapper v26

四、精度指标
  - 平面位置中误差：≤0.××m（平地）
  - 高程中误差：≤0.××m（平地）
  - DOM分辨率：0.××m

五、质量控制措施
  - 二级检查一级验收制度
  - 过程检查节点

六、进度计划与人员组织
  - 甘特图
  - 岗位职责

七、安全保障
  - 数据保密措施
  - 外业安全预案
```

### 4.2 技术总结（纲要）

```
一、项目概况（同设计书，补充最终完成情况）
二、技术设计执行情况（偏离项说明及原因）
三、成果精度统计
  - 控制点精度
  - DLG平面/高程精度（中误差统计表）
  - DEM高程精度（检查点对比表）
  - DOM平面精度
四、质量评价（附质量检查报告结论）
五、存在问题与处理
六、成果清单
七、结论与建议
```

### 4.3 元数据编制

按照 GB/T 19710（ISO 19115）标准，至少包含：

| 元数据项 | 说明 | 示例 |
|---------|------|------|
| 数据集名称 | 成果标题 | "北京市某区1:500 DLG" |
| 数据集日期 | 生产日期 | "2026-05" |
| 数据集格式 | 格式名称+版本 | "GeoPackage 1.3" |
| 空间参考系 | EPSG代码 | "EPSG:4490 (CGCS2000)" |
| 空间范围 | 四角坐标 | `[116.2,39.8,116.5,40.0]` |
| 数据生产者 | 单位名称 | "××测绘院" |
| 数据质量 | 精度信息 | "平面中误差0.25m" |
| 数据志 | 生产工艺 | "航测立体采集" |

## 六、常见交付问题与对策

| 问题 | 原因 | 对策 |
|------|------|------|
| 成果被退回 | 缺元数据/格式不对 | 按本规范逐项核对 |
| 中文乱码 | 编码非UTF-8 | 统一转UTF-8，ArcGIS Pro设置环境变量 |
| 文件命名混乱 | 无命名规范 | 执行2.1节命名规则 |
| 坐标系错误 | 使用WGS84而非CGCS2000 | 全流程统一EPSG:4490 |
| 精度超标 | 生产中未做精度控制 | 生产过程设置检查点→参见06_数据生产流程规范.md |
