# Z1 记忆代谢协议 V1
# 日期：2026-04-08
# 状态：ACTIVE
# 级别：长期运行协议

## [0] 定义
记忆代谢，指的是：
**项目运行结果如何从任务卡与项目房间，进入 Reflection Wing，被编译、筛选、升级、归档。**

目标不是保存一切，而是：
- 保存可复用的
- 升级稳定出现的
- 归档已完成但低频使用的
- 淘汰无增量的噪声

---

## [1] 三层去向规则
### 1.1 只留在 Project Room 的内容
满足以下任一条件：
- 只对单一项目有效
- 主要是事实记录或样本展示
- 没有跨任务复用价值
- 只是阶段性过程痕迹

结论：
保留在 `project_rooms/*`，不进入 Reflection Wing。

### 1.2 进入 Reflection Wing 的内容
满足以下条件之一：
- 对两个及以上任务复现有效
- 能压缩为原则、失败模式、提示词内核、思考路径
- 对调度、验收、生产效率有明显增益
- 能降低未来 token 消耗或减少错误率

结论：
进入 `reflection_wing/*`

### 1.3 升级为 Constitution Candidate 的内容
必须同时满足：
1. 已在多个任务中被验证稳定有效
2. 违反它会明显带来质量下降、链路混乱或 token 浪费
3. 适用于 Z1 或全矩阵的长期默认行为
4. 不是局部技巧，而是结构性规则

结论：
进入 `reflection_wing/constitution_candidates/`
等待 Z1 抽检决定是否升为宪法。

---

## [2] 五类标准产物
项目完成后，Librarian / Z1 应判断是否生成以下五类产物：
1. principle card
2. prompt kernel
3. failure pattern
4. thinking path
5. constitution candidate

不是每个项目都要五项全出。
原则：
**只提炼有复用价值的，不制造伪沉淀。**

---

## [3] 升级门禁
### 3.1 Principle 门禁
需要至少满足：
- 可复用
- 有证据支持
- 可压成一句话版本

### 3.2 Prompt Kernel 门禁
需要至少满足：
- 可直接复用于未来任务指令或生成流程
- 能减少解释成本
- 有明确输入输出逻辑

### 3.3 Failure Pattern 门禁
需要至少满足：
- 故障已真实发生
- 能描述症状、触发条件、修复法
- 对未来有预防价值

### 3.4 Thinking Path 门禁
需要至少满足：
- 不是结果，而是路径
- 对未来同类问题有迁移价值
- 结构清晰，可压缩为步骤链

### 3.5 Constitution Candidate 门禁
需要至少满足：
- 已跨多个任务验证
- 属于高层结构性规则
- 偏离它的代价显著
- 与 USER 偏好、AGENTS 不可变原则一致

---

## [4] 反向淘汰规则
以下内容不得升级：
- 一次性灵感
- 无证据支撑的抽象大道理
- 仅在单一任务有效的技巧
- 只为凑数量而写的空洞总结
- 对未来不能减少错误或提升效率的漂亮话

一句话：
**没有复用价值的经验，不配进入长期层。**

---

## [5] 项目收尾标准动作
每个重要项目完成后，默认执行以下判断：
1. 完成 Corridor 卡归档
2. 更新 Project Room 索引
3. 判断是否需要 Reflection 编译
4. 若需要，写入对应原则/内核/失败卡
5. 若达到长期结构性级别，送 Constitution Candidate

---

## [6] Z1 抽检职责
Z1 负责最终抽检：
- 该经验是否真的跨任务稳定成立
- 是否符合长官偏好
- 是否值得占用长期记忆位
- 是否应升级为底层宪法

---

## [7] 一句话总纲
**项目负责沉淀事实，反思负责提炼逻辑，宪法只接纳被反复验证的硬规则。**
