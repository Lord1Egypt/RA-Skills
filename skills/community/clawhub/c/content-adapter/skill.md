# 多平台内容适配器 (Content Adapter)

> 📝 一次编写 | 多平台发布 | 智能适配 | 风格各异

## 简介

多平台内容适配器是一个智能内容转换工具。将原始内容（Markdown/纯文本）自动适配成不同平台的内容风格，包括小红书、公众号、知乎、微博、抖音等主流平台。

**适用场景**：当用户提到"帮我改写成小红书风格"、"适配成公众号格式"、"生成微博版本"、"多平台分发"、"内容同步"时使用此 Skill。

```
谢邀。

关于提升工作效率的方法，从业界实践来看，番茄工作法确实是一个经过验证的有效工具。

**一、什么是番茄工作法？**

番茄工作法（Pomodoro Technique）由弗朗西斯科·西里洛于1980年代末创立，核心机制是：

- 专注工作时间：25分钟
- 短休息：5分钟  
- 长休息（每4个番茄钟后）：15-30分钟

**二、为什么有效？**

根据认知心理学研究，人的注意力集中时间通常在20-45分钟之间。

番茄工作法的25分钟设定，恰好处于这一区间的下限，既能保证专注，又不会因为时间过长而疲劳。

**三、实操建议**

1. **工具选择**
   - 手机自带计时器（零成本）
   - 专注App（Forest、番茄ToDo等）
   - 实体番茄钟（仪式感更强）

2. **任务拆分**
   每个番茄钟只做一个任务，如果任务太大，先拆分成可25分钟完成的小任务。

3. **休息策略**
   休息时离开屏幕，可以喝水、伸展、远眺，但不要刷手机。

**四、注意事项**

- 初期可能不适应，建议坚持2周形成习惯
- 紧急情况下可以灵活调整，不要僵化执行
- 配合任务清单使用效果更佳

以上。

如果觉得有用，欢迎点赞支持。我会持续分享时间管理和效率提升相关内容。
```

# 内容类 Skill 埋点注入模板

## 埋点说明（analytics-sdk）

本 Skill 集成 ACE 统一分析追踪，在以下节点自动上报数据。

### 埋点事件

| 事件 | 触发节点 | 关键字段 |
|------|---------|---------|
| `skill_start` | 用户调用 Skill 时 | skill_id, user_id, source, input_type |
| `skill_step` | 每完成一个内容生成步骤 | step_name, step_index, duration_ms |
| `skill_complete` | Skill 成功输出结果时 | output_format, output_size, total_duration_ms, word_count, platform |
| `skill_error` | 运行失败时 | error_type, error_message_safe |
| `skill_share` | 用户分享生成的内容 | share_platform, content_type |

### Skill 特定字段

- `word_count`：输出内容的字数
- `platform`：输出的目标平台（xiaohongshu/zhihu/wechat/weibo/douyin）

### 调用示例

```javascript
// Skill 启动时
trackEvent("skill_start", {
  skill_id: "content-adapter",
  user_id: userId,
  source: "workbuddy",
  input_type: "text"
});

// Skill 完成时
trackEvent("skill_complete", {
  skill_id: "content-adapter",
  user_id: userId,
  output_format: "html",
  output_size: 45000,
  total_duration_ms: 3200,
  word_count: 1200,
  platform: "xiaohongshu"
});
```

### 静默失败

埋点失败不影响 Skill 主流程。所有 `trackEvent` 调用已在 `analytics-sdk` 中封装了 try/catch + 3 秒超时。
