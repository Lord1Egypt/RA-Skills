# 学习阶段计划生成器 (Study Phase Planner)

## Overview

为任意教育阶段（小学/初中/高中/大学）的任意学科生成系统化的三阶段学习计划（基础→强化→冲刺），包含可量化的里程碑检查点，输出交互式 HTML 可视化报告。

## 触发条件

当用户提出以下任一意图时使用本技能：

- 要求制定某学科的学习/复习/备考计划
- 提到"基础/强化/冲刺"分阶段安排
- 期中/期末/升学（中考/高考/考研）复习规划
- 需要学习里程碑或检查点
- 询问"XX学科怎么复习/怎么备考"
- 要求生成学习计划报告

## 核心能力

1. **智能阶段分析** — 根据用户输入自动识别教育阶段、学科、目标类型、可用时间
2. **三阶段拆分** — 按黄金比例（基础40-50% / 强化30-40% / 冲刺15-25%）分配时间
3. **里程碑设计** — 为每个阶段生成必达/期望/挑战三级里程碑
4. **可视化报告** — 输出含时间线、进度条、检查点追踪表的交互式 HTML 报告

## 工作流程

### Step 1: 需求信息收集

从用户输入中提取以下信息，如有缺失则主动询问：

| 维度 | 必填/可选 | 说明 |
|------|-----------|------|
| 教育阶段 | 必填 | 小学 / 初中 / 高中 / 大学 |
| 年级 | 必填 | 具体年级（如"初三""高二""大二"） |
| 学科 | 必填 | 具体学科名称（可多学科，如"数学+物理"） |
| 目标类型 | 必填 | 日常学期 / 期中备考 / 期末备考 / 中考 / 高考 / 考研 / 单科突破 / 证书考试 |
| 可用时间 | 必填 | 从哪天开始到哪天结束，或"距离考试还有X天" |
| 当前水平 | 可选 | 基础薄弱 / 中等 / 较好（影响内容深度和节奏） |
| 每日可用时长 | 可选 | 周一至周五X小时，周末X小时 |
| 薄弱环节 | 可选 | 具体知识模块/题型（优先安排到强化阶段） |
| 目标分数 | 可选 | 期望达到的分数/等级 |
| 教材版本 | 可选 | 人教版/北师大版/苏教版等（影响内容细节） |

**询问策略**：第一轮只问必填项，可选信息在后续对话中根据情况补充。若用户说"随便"或未指定，按以下默认值处理：
- 当前水平：中等
- 每日可用时长：工作日2h / 周末4h
- 薄弱环节：不特别安排，均匀覆盖

### Step 2: 验证与计算

1. **加载参考数据**：打开 `references/education_stages.md`，对照用户的教育阶段和学科，确认学科是否属于该阶段的常规课程，并获取该阶段的学科特点和大纲框架。

2. **计算时间参数**：
   - 计算总可用天数（不含无计划日）
   - 根据目标类型确定三阶段时间分配比例（参考 `references/education_stages.md` 第五节"不同目标的计划差异化"）
   - 将总天数按比例分配到三个阶段

3. **日期规划**：
   - 从起始日期开始，逐段分配
   - 每个阶段之间保留1-2天缓冲/回顾日（冲刺阶段除外）
   - 为长周期计划（≥3个月）插入2-3个休息/调整周

**分配比例速查表**：

| 目标类型 | 基础 | 强化 | 冲刺 | 缓冲日 |
|----------|------|------|------|--------|
| 日常学期 (4-5月) | 45% | 35% | 15% | 5% |
| 期中备考 (2-4周) | 35% | 40% | 20% | 5% |
| 期末备考 (4-6周) | 35% | 40% | 20% | 5% |
| 中考/高考 (6-12月) | 45% | 30% | 15% | 10% |
| 考研 (6-10月) | 45% | 30% | 15% | 10% |
| 单科突破 (1-3月) | 30% | 45% | 20% | 5% |

### Step 3: 三阶段计划内容设计

为每个阶段生成详细计划内容。

#### 基础阶段 (Foundation)

设计原则：
- 搭建系统知识框架，按章节/模块顺序推进
- 重点：概念理解、公式推导、基本题型训练
- 产出：知识框架图（文字描述）、基础题正确率目标
- 节奏：先慢后快，前期给适应期

输出内容模板：
```
阶段目标：[具体目标描述]
核心内容：
1. [模块/章节名] — [知识点列表]
2. ...
每日节奏：[工作日安排] / [周末安排]
阶段测验：[X]次，每次覆盖[Y]个模块
```

#### 强化阶段 (Reinforcement)

设计原则：
- 专题突破 + 综合题型训练
- 重点：高频考点、易错题型、解题技巧、速度训练
- 产出：专题总结笔记、易错题汇编
- 节奏：按专题集中突破，每周1-2个专题

输出内容模板：
```
阶段目标：[具体目标描述]
专题列表：
1. [专题名] — [计划天数] — [突破要点]
2. ...
综合训练：[X]套综合卷，间隔[Y]天
重点突破：[根据用户薄弱环节优先安排]
```

#### 冲刺阶段 (Sprint)

设计原则：
- 全真模拟 + 查漏补缺
- 重点：限时训练、错题清零、心态调整
- 产出：模拟成绩曲线、考前备忘录
- 节奏：模拟考→复盘→针对性补漏→再模拟

输出内容模板：
```
阶段目标：[具体目标描述]
模拟安排：[X]次全真模拟，时间间隔[X]天
错题清零：[时间段]内完成所有错题回顾
考前备忘录：考试前[X]天完成
作息调整：考前[X]天开始调整
```

### Step 4: 里程碑设计

为每个阶段设计3类里程碑（参考 `references/education_stages.md` 第六节）：

1. **必达项 (Must/P0)**：必须完成的硬性指标，未达成则计划失败
   - 示例：完成全部教材一遍精读、基础题正确率达70%、完成≥3次全真模拟
2. **期望项 (Should/P1)**：尽力完成的目标，未达成需调整后续计划
   - 示例：综合题正确率达85%、完成所有专题笔记整理
3. **挑战项 (Could/P2)**：超额完成的加分项
   - 示例：提前预习下一章节、完成额外真题训练

**里程碑总数建议**：
- 短周期（≤4周）：5-8个里程碑
- 中周期（1-3月）：10-15个里程碑
- 长周期（≥3月）：15-20个里程碑

### Step 5: 生成 HTML 报告

使用 `assets/report_template.html` 作为输出模板。

#### 5.1 读取模板

```
Read assets/report_template.html 获取模板内容
```

#### 5.2 填充模板变量

模板使用 `{{VARIABLE_NAME}}` 占位符，需填充以下变量：

**头部信息**：
- `{{PLAN_TITLE}}` — 计划标题，格式："[阶段][学科]三阶段学习计划"（如"初三数学中考三阶段备考计划"）
- `{{STAGE_BADGE}}` — 阶段标签（如"初中 · 中考备考"）
- `{{DATE_RANGE}}` — 日期范围（如"2026.03.01 - 2026.06.15"）
- `{{TOTAL_DAYS}}` — 总天数
- `{{SUBJECT}}` — 学科名称

**摘要卡片**：
- `{{FOUNDATION_DAYS}}`, `{{REINFORCEMENT_DAYS}}`, `{{SPRINT_DAYS}}` — 各阶段天数
- `{{FOUNDATION_PCT}}`, `{{REINFORCEMENT_PCT}}`, `{{SPRINT_PCT}}` — 各阶段百分比
- `{{FOUNDATION_DATE_RANGE}}`, `{{REINFORCEMENT_DATE_RANGE}}`, `{{SPRINT_DATE_RANGE}}` — 各阶段日期范围

**阶段详情 `{{PHASE_DETAILS}}`**：
为每个阶段生成以下 HTML 结构（阶段类型用 `foundation` / `reinforcement` / `sprint`）：

```html
<div class="phase-detail {{PHASE_CLASS}} open">
  <div class="phase-header">
    <span class="expand-icon">▶</span>
    <div class="phase-number">{{PHASE_NUMBER}}</div>
    <div class="phase-info">
      <div class="phase-title">{{PHASE_TITLE}}</div>
      <div class="phase-subtitle">{{PHASE_DATE_RANGE}} · {{PHASE_DAYS}}天</div>
    </div>
  </div>
  <div class="phase-body">
    <p style="margin-bottom:12px;font-size:14px;"><strong>阶段目标：</strong>{{PHASE_GOAL}}</p>
    <div class="tasks">
      {{TASK_ITEMS}}
    </div>
  </div>
</div>
```

每个 `TASK_ITEM` 格式：
```html
<div class="task-item">
  <div class="task-check"></div>
  <span>{{TASK_CONTENT}}</span>
</div>
```

**里程碑 `{{MILESTONE_CARDS}}`**：
每个里程碑使用 `priority-must` / `priority-should` / `priority-could` 作为 CSS class：

```html
<div class="milestone-card {{PRIORITY_CLASS}}">
  <div class="ms-icon">{{MS_ICON}}</div>
  <div class="ms-content">
    <div class="ms-title">{{MS_TITLE}}</div>
    <div class="ms-desc">{{MS_DESC}}</div>
    <div class="ms-meta">
      <span>📅 {{MS_DATE}}</span>
      <span>📊 {{MS_METRIC}}</span>
      <span class="ms-priority">{{MS_PRIORITY_LABEL}}</span>
    </div>
  </div>
</div>
```

MS_ICON 映射：must → 🚩 / should → 🎯 / could → ⭐
MS_PRIORITY_LABEL 映射：must → 必达 / should → 期望 / could → 挑战

**检查点追踪表 `{{CHECKPOINT_ROWS}}`**：
```html
<tr>
  <td>{{INDEX}}</td>
  <td>{{NAME}}</td>
  <td>{{DEADLINE}}</td>
  <td>{{PHASE}}</td>
  <td><span class="status-badge status-pending">待完成</span></td>
</tr>
```

**学习建议 `{{TIPS_ITEMS}}`**：
```html
<div class="tip-item">
  <div class="tip-num">{{N}}</div>
  <span>{{TIP_CONTENT}}</span>
</div>
```

**其他**：
- `{{GENERATED_DATE}}` — 生成日期（格式 YYYY.MM.DD）

#### 5.3 输出位置

将填充后的 HTML 写入当前工作目录：
```
<workspace>/output/study_plan_<stage>_<subject>.html
```

filename 格式：`study_plan_<stage>_<subject>.html`（如 `study_plan_初三_数学.html`），使用中文无空格命名。

#### 5.4 呈现报告

生成完成后，使用 `present_files` 展示 HTML 报告。

### Step 6: 计划调整与迭代

若用户在查看报告后提出调整需求（如"基础阶段太长了""加一个专题"），直接在原 HTML 文件中修改对应部分并重新呈现。修改时：

1. 保持三阶段结构不变，只调整内容/时间/里程碑
2. 时间调整后重新计算所有百分比和日期
3. 告知用户变更影响（如"基础阶段从45天缩短到35天，冲刺阶段相应增加10天"）

## 注意事项

1. **计划合理性**：确保每日任务量适中，不给用户造成过大压力。长周期计划务必插入休息日。
2. **阶段过渡**：每个阶段结束后保留1-2天回顾/过渡期，让用户有时间消化和调整。
3. **里程碑可执行性**：每个里程碑必须有明确的"如何验证"方法（测试/自评/外部检查）。
4. **学科差异**：
   - 理科（数学/物理/化学）：侧重题型训练和解题速度
   - 文科（语文/英语/历史/政治）：侧重积累和记忆，基础阶段占比可适当提高
   - 语言类（英语）：听说读写四维并行，而非线性推进
5. **大学阶段特殊性**：大学以自学为主，需考虑课程冲突，并引导用户自行整理专业课大纲。
6. **报告输出**：HTML 报告始终为中文，日期格式统一用 YYYY.MM.DD，学科名保持用户原始表述。

## 示例对话模式

```
用户：帮我制定初三数学中考备考计划，现在到中考还有100天

助手分析：
- 教育阶段：初中（初三）
- 学科：数学
- 目标类型：中考
- 总天数：100天
- 分配比例：基础45% / 强化30% / 冲刺15% / 缓冲10%
- 基础：45天 | 强化：30天 | 冲刺：15天 | 缓冲：10天
→ 加载 references/education_stages.md 获取初中数学大纲
→ 设计三阶段内容 + 里程碑
→ 填充 HTML 模板
→ 输出报告
```
