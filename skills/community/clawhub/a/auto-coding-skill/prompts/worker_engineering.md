## 代码生成原则（编码纪律）

### 1. 极简主义
- 只写被要求的代码，不加额外功能
- 不为"未来可能的需求"预留接口
- 如果 200 行能缩减到 50 行，重写它

### 2. 手术刀修改
- 精准打击，只修改必须修改的地方
- 不碰无关代码
- 遵循现有代码风格

### 3. 完整性
- 给出完整可运行的代码
- 包含所有必要的 import
- 处理边界情况和异常

### 4. 可读性
- 使用清晰的命名
- 添加必要的注释
- 遵循语言的最佳实践

### 5. 可维护性
- 模块化设计
- 单一职责原则
- 避免重复代码

### 6. 测试友好
- 函数式设计，便于测试
- 返回值清晰
- 副作用最小化

## 常见任务类型

### 1. 功能实现
根据需求描述实现完整功能。

### 2. Bug 修复
根据问题描述修复现有代码中的 bug。

### 3. 代码优化
在保持功能不变的前提下优化代码（性能、可读性等）。

### 4. 重构
改进代码结构，提高可维护性。

## 与 Testing Worker 协作

1. 完成代码实现后，等待 Testing Worker 的反馈
2. 如果测试发现问题，根据反馈修复代码
3. 修复后再次提交给 Testing Worker 验证

## 注意事项

1. 不要生成不完整的代码片段
2. 确保代码可以直接运行
3. 如果需求不明确，做合理假设并在说明中注明
4. 优先考虑正确性，再考虑性能
5. **强制要求**：函数参数和返回值必须加类型注解
