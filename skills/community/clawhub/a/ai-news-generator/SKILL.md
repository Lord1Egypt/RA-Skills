# AI 资讯生成器

## 功能定位

本技能专注于搜索、汇总和解读 AI 领域的最新资讯，并通过图表和配图增强信息呈现，包括但不限于：

- **大模型发布与更新**：GPT、Claude、Gemini、Llama、Qwen 等主流模型的新版本
- **研究突破**：顶会论文（NeurIPS、ICML、ICLR、ACL 等）、arXiv 重要预印本
- **产业动态**：OpenAI、Anthropic、Google DeepMind、Meta AI、百度、阿里、腾讯等公司动态
- **政策与监管**：国内外 AI 相关法规、政策、标准
- **应用落地**：AI 在各行业的实际应用案例
- **融资与市场**：AI 领域的重要融资、并购、市值变化

## 触发场景

- 用户询问「最近有什么 AI 新闻」「今天 AI 有什么大事」
- 用户要求生成「AI 周报」「AI 月度资讯简报」
- 用户想了解某个特定 AI 产品或公司的最新动态
- 用户询问某个 AI 研究方向的进展
- 用户希望对比不同 AI 模型或产品的最新能力

## 工作流程

### 1. 理解用户意图

收到请求后，首先明确：
- **时间范围**：今天、本周、本月、最近 N 天？（默认：最近 7 天）
- **领域范围**：全局 AI 资讯 / 特定公司 / 特定方向（如多模态、Agent、推理模型）？
- **输出格式**：简报 / 深度分析 / 对比报告 / 可视化摘要？
- **是否需要图表**：默认在数据类内容（评测对比、融资趋势、市场份额）中自动配上图表
- **语言**：中文（默认）/ 英文 / 双语？

### 2. 信息采集策略

使用 WebSearch 工具，按以下顺序执行搜索，每次搜索控制在 1–3 个查询词组：

#### 核心搜索源（优先级由高到低）

| 来源 | 搜索方式 | 适合内容 |
|------|---------|---------|
| 36氪 / 机器之心 / 量子位 | `site:36kr.com AI` | 中文产业资讯 |
| OpenAI Blog / Anthropic / Google DeepMind | `site:openai.com blog` | 官方公告 |
| arXiv | `site:arxiv.org cs.AI` | 学术研究 |
| TechCrunch / VentureBeat / The Verge | `AI news site:techcrunch.com` | 英文产业资讯 |
| Twitter/X / LinkedIn | 关键人物动态 | 行业意见领袖观点 |

#### 推荐搜索策略

- **中文资讯**：搜索词加「最新」「2026」等时间词提升时效性
- **英文资讯**：在 WebSearch 中设置 `topic: "technology"` 或 `topic: "news"`
- **模型相关**：同时搜索中英文名称（如「Gemini 2.5」与「Google Gemini」）
- **并行搜索**：无依赖的搜索词组可并行发起，节省时间

### 3. 内容整理与去重

- 合并重复报道，保留信息密度最高的来源
- 区分「一手信息」（官方博客、论文）和「二手报道」（媒体解读）
- 标注信息来源和大致日期
- 标记未经核实的传言（如「据悉」「消息人士称」）

### 4. 输出格式规范

#### 标准资讯简报格式

```markdown
# AI 资讯简报 · [日期/周期]

## 本期重点
[2–3 句话总结本期最重要的 1–2 条新闻]

## 模型与产品
- **[产品名]**：[核心更新] · [来源]

## 研究进展
- **[论文/研究名称]**：[主要发现] · [机构] · [arXiv/来源链接]

## 公司动态
- **[公司名]**：[事件摘要] · [来源]

## 政策与监管
- [摘要] · [地区] · [来源]

## 值得关注
[编辑推荐的深度观点或值得持续跟踪的趋势]

---
*资讯来源：[来源列表]*
*整理时间：[时间]*
```

#### 深度分析格式

当用户需要对某一话题深度解读时：
1. 背景：该话题的来龙去脉
2. 事件：核心事实（what happened）
3. 影响：对行业/技术/用户的潜在影响
4. 争议：不同观点和质疑
5. 展望：未来走向预测

#### 对比报告格式

当用户需要对比多个模型或产品时，使用表格形式，参考 `references/model_compare_template.md`。

### 5. 可视化与图片

在文本简报之外，主动使用 WorkBuddy 的可视化能力增强资讯呈现效果。

#### 内联图表（show_widget）

在简报中嵌入实时渲染的图表，让数据一目了然。适用于以下场景：

| 场景 | 推荐图表类型 | 说明 |
|------|------------|------|
| 模型评测得分对比 | 柱状图 / 雷达图 | 多模型在 MMLU、HumanEval 等基准上的得分 |
| 融资规模趋势 | 折线图 / 面积图 | 某时段内 AI 领域融资总额变化 |
| 市场份额分布 | 饼图 / 环形图 | 各大模型/API 的市场份额 |
| 事件时间线 | SVG 流程图 | 重大发布、政策事件按时间排列 |
| 新闻热度词云 | HTML 交互词云 | 本周期高频关键词可视化 |

**使用方法：**
1. 先调用 `read_me`（modules: "chart"）加载图表设计规范
2. 调用 `show_widget` 生成内联 Chart.js 图表或 SVG 图
3. 在图表前后配合文字说明，不要堆叠多个 widget 不加解释

**Chart.js 图表示例结构（HTML widget）：**
```html
<canvas id="myChart" width="680" height="400"></canvas>
<script src="https://cdn.jsdelivr.net/npm/chart.js"></script>
<script>
  new Chart(document.getElementById('myChart'), {
    type: 'bar',
    data: { labels: [...], datasets: [...] },
    options: { ... }
  });
</script>
```

详细代码模板参见 `references/visualization_examples.md`。

#### AI 配图（ImageGen）

当需要为简报生成封面配图或概念示意图时，使用 `ImageGen` 工具：

1. 先通过 `ToolSearch` 加载 `ImageGen` 工具 schema
2. 再通过 `DeferExecuteTool` 调用，传入中文/英文 prompt

**适用场景：**
- 简报封面图：如「AI 行业周报封面，科技感蓝色调」
- 概念示意图：如「大模型参数规模对比可视化概念图」
- 趋势预景图：如「2026 年 AI 发展趋势展望」

**注意：** ImageGen 生成的是图片文件，通过 `present_files` 展示给用户。

#### 引用外部图片

搜索到的新闻中如果包含关键图片（如产品截图、架构图、评测图表），使用 WebFetch 抓取原始页面，在简报中描述图片内容并标注来源链接。不直接嵌入外部图片 URL（可能有版权和访问限制），而是：
- 描述图片核心信息
- 提供来源链接供用户点击查看
- 若为公开官方图片（如 OpenAI 官博配图），可直接引用 URL

### 6. 质量检查

发布前检查：
- [ ] 所有重要数据点是否有来源支撑
- [ ] 时间信息是否准确（避免将旧新闻当新闻）
- [ ] 是否区分了事实与观点
- [ ] 是否有明显遗漏的重大事件
- [ ] 图表数据与文字内容一致，无矛盾
- [ ] 配图 prompt 是否准确描述了需求

## 注意事项

- AI 领域变化极快，务必在搜索时加入时间词（如年份、「最新」）确保时效
- 大模型测评数据来源众多，务必注明基准测试来源，避免混淆不同评测体系
- 区分「发布」（released）和「公告」（announced）— 公告不代表已上线
- 对于未经官方确认的传言，明确标注「据报道」或「未经证实」
