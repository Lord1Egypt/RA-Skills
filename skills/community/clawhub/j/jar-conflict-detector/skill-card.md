## Description: <br>
Detects version conflicts, known incompatible dependency pairs, and duplicate JARs in Spring Boot Maven or Gradle projects, then produces conflict reports and resolution guidance. <br>

This skill is ready for commercial/non-commercial use. <br>

## Publisher: <br>
[yssnb](https://clawhub.ai/user/yssnb) <br>

### License/Terms of Use: <br>
MIT-0 <br>


## Use Case: <br>
Developers and engineers use this skill to inspect Spring Boot microservice dependency trees, troubleshoot classpath errors such as NoSuchMethodError or ClassNotFoundException, and prepare actionable Maven or Gradle dependency fixes. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: The bundled checker can run Maven or Gradle dependency commands from the selected project, and those commands may execute project build logic. <br>
Mitigation: Run the skill only on trusted Java projects, or execute it inside a container or sandbox for third-party repositories. <br>
Risk: Generated reports may be written into the selected workspace path. <br>
Mitigation: Choose an explicit output path and review generated report files before sharing or committing them. <br>
Risk: Static POM parsing is less complete when Maven or Gradle cannot run, so transitive dependency findings may be incomplete. <br>
Mitigation: Treat fallback-mode results as preliminary and rerun with Maven or Gradle available before using the report as a CI gate. <br>
