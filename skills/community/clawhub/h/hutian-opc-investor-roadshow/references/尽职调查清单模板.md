# 尽职调查清单模板

## 四层金字塔尽职调查体系

本清单按照「材料核查→技术评估→商业逻辑→风险量化」四层金字塔结构设计，确保尽职调查全面、系统、深入。

---

## 第一层：材料完整性核查

### 1.1 基础证照

| 序号 | 材料名称 | 核查要点 | 核验方式 | 重要性 |
|------|----------|----------|----------|--------|
| 1 | 营业执照 | 法人、注册资本、经营范围 | 国家企业信用信息公示系统 | 🔴必查 |
| 2 | 组织机构代码证 | 代码一致性 | 官网核验 | 🔴必查 |
| 3 | 税务登记证 | 纳税人类型、税率 | 税务系统 | 🔴必查 |
| 4 | 开户许可证 | 基本户信息 | 银行核验 | 🟡重要 |
| 5 | 社保登记证 | 员工参保情况 | 社保系统 | 🟡重要 |

### 1.2 财务报表

| 序号 | 材料名称 | 核查要点 | 核验方式 | 重要性 |
|------|----------|----------|----------|--------|
| 1 | 审计报告（近3年） | 营收、利润、资产、负债 | 事务所盖章核验 | 🔴必查 |
| 2 | 财务报表（近3年） | 科目余额、明细账 | 抽查原始凭证 | 🔴必查 |
| 3 | 银行流水（近6个月） | 资金流向、收支核实 | 银行对账单 | 🔴必查 |
| 4 | 纳税申报表 | 税务合规性 | 税局系统 | 🟡重要 |
| 5 | 社保缴费记录 | 用工合规性 | 社保系统 | 🟡重要 |

### 1.3 知识产权

| 序号 | 材料名称 | 核查要点 | 核验方式 | 重要性 |
|------|----------|----------|----------|--------|
| 1 | 专利证书（全部） | 专利号、权利人、法律状态 | 国家专利局官网 | 🔴必查 |
| 2 | 软件著作权 | 著作权人、登记号 | 中国版权保护中心 | 🟡重要 |
| 3 | 商标注册证 | 商标类别、有效期 | 国家商标局 | 🟡重要 |
| 4 | 技术秘密资料 | 保密措施、权属清晰 | 实地核查 | 🟡重要 |
| 5 | 集成电路布图设计 | 登记证书 | 国家知识产权局 | 🟢可选 |

### 1.4 融资历史

| 序号 | 材料名称 | 核查要点 | 核验方式 | 重要性 |
|------|----------|----------|----------|--------|
| 1 | 融资协议（全部） | 估值、轮次、特殊条款 | 工商变更核验 | 🔴必查 |
| 2 | 投资方尽调报告 | 尽调结论、风险点 | 原件核对 | 🟡重要 |
| 3 | 股东协议 | 股权结构、控制权 | 工商登记核对 | 🔴必查 |
| 4 | 董事会决议 | 重大决策记录 | 原件核对 | 🟡重要 |
| 5 | 优先清算权协议 | 退出条款 | 律师审核 | 🟡重要 |

### 1.5 合同订单

| 序号 | 材料名称 | 核查要点 | 核验方式 | 重要性 |
|------|----------|----------|----------|--------|
| 1 | 前五大客户合同 | 客户名称、金额、期限 | 抽查验证 | 🔴必查 |
| 2 | 前五大供应商合同 | 供应商稳定性 | 抽查验证 | 🟡重要 |
| 3 | 代理/经销协议 | 渠道稳定性 | 抽查验证 | 🟡重要 |
| 4 | 重大采购合同 | 采购价格、履约情况 | 抽查验证 | 🟡重要 |
| 5 | 战略合作协议 | 合作内容、期限 | 抽查验证 | 🟢可选 |

## 第二层：技术可行性评估

### 2.1 技术参数验证

| 序号 | 验证项目 | 验证方法 | 评估标准 |
|------|----------|----------|----------|
| 1 | 参数真实性 | 与行业公开数据对比 | 不超出物理极限 |
| 2 | 参数可复现性 | 第三方测试验证 | 可重复实现 |
| 3 | 测试报告 | 第三方机构出具 | CMA/CNAS认证 |
| 4 | 竞品对比 | 横向技术对比 | 优势明确 |
| 5 | 行业标准符合性 | 对标国家标准 | 符合或超越 |

### 2.2 TRL等级评定

| TRL等级 | 定义 | 评估要点 | 适用技术类型 |
|---------|------|----------|--------------|
| TRL 1 | 基本原理 | 原理明确 | 概念阶段 |
| TRL 2 | 概念形成 | 理论验证 | 论文阶段 |
| TRL 3 | 实验验证 | 关键功能验证 | 实验室验证 |
| TRL 4 | 实验室验证 | 原理样机 | 原型机 |
| TRL 5 | 相关环境验证 | 模拟环境验证 | 工程样机 |
| TRL 6 | 系统模型验证 | 原理样机集成 | 演示样机 |
| TRL 7 | 系统原型验证 | 实际环境验证 | 工程化样机 |
| TRL 8 | 系统完成 | 实际系统调试完成 | 量产前 |
| TRL 9 | 系统成功运行 | 实际系统运行成功 | 量产 |

### 2.3 技术壁垒分析

| 壁垒类型 | 评估维度 | 评估标准 |
|----------|----------|----------|
| 专利壁垒 | 专利数量、质量、布局 | 发明专利>5项+国际布局 |
| 技术秘密 | 工艺参数、配方 | 保密措施完善 |
| 人才壁垒 | 核心技术人员 | 股权绑定+竞业协议 |
| 品牌壁垒 | 市场认知度 | 客户认可度高 |
| 成本壁垒 | 规模效应 | 成本优势明显 |

### 2.4 竞品技术对比

|----------|------------|-------|-------|-------|

| 性能指标2 | | | | |
| 成本 | | | | |
| 稳定性 | | | | |
| 交付能力 | | | | |
| 售后支持 | | | | |

## 第三层：商业逻辑评估

### 3.1 市场规模测算

| 层次 | 定义 | 测算方法 | 典型数据来源 |
|------|------|----------|--------------|
| TAM（总可触达市场） | 理论最大市场 | 行业报告+宏观数据 | Gartner, IDC, 行业协会 |
| SAM（可服务市场） | 目标区域+行业 | TAM×目标占比 | 细分市场报告 |
| SOM（可获得市场） | 3年内可获得 | 竞争分析+渗透率 | 市占率预测 |

### 3.2 竞争格局分析

| 分析维度 | 内容 | 输出物 |
|----------|------|--------|
| 直接竞争者 | 同类业务企业 | 竞品清单 |
| 间接竞争者 | 替代方案 | 替代品分析 |
| 潜在进入者 | 可能进入的巨头 | 威胁评估 |
| 替代威胁 | 技术替代风险 | 替代路径分析 |
| 供应商议价 | 核心部件供应商 | 供应链依赖度 |
| 客户议价 | 主要客户集中度 | 客户依赖度 |

### 3.3 商业模式画布

| 模块 | 核心问题 | 企业填写 |
|------|----------|----------|
| 客户细分 | 我们为谁创造价值？ | |
| 价值主张 | 我们为客户带来什么？ | |
| 渠道通路 | 如何触达客户？ | |
| 客户关系 | 如何维护客户？ | |
| 收入来源 | 如何从客户获取收益？ | |
| 核心资源 | 需要哪些关键资源？ | |
| 关键业务 | 需要做哪些关键业务？ | |
| 重要合作 | 需要哪些合作伙伴？ | |
| 成本结构 | 需要付出哪些成本？ | |

### 3.4 增长逻辑分析

| 增长维度 | 驱动因素 | 扩展路径 |
|----------|----------|----------|
| 产品扩展 | 新产品研发 | 产品线延伸 |
| 市场扩展 | 新市场开拓 | 区域扩张 |
| 客户扩展 | 新客户获取 | 客户群拓展 |
| 模式扩展 | 商业模式升级 | 平台化转型 |

## 第四层：风险识别与量化

### 4.1 风险分类矩阵

| 风险类型 | 风险因素 | 可能性 | 影响程度 | 风险等级 |
|----------|----------|--------|----------|----------|
| 技术风险 | 技术迭代 | 🟡中 | 🔴高 | 🟠中高 |
| 技术风险 | 技术替代 | 🟡中 | 🔴高 | 🟠中高 |
| 技术风险 | 技术泄密 | 🟢低 | 🔴高 | 🟡中 |
| 市场风险 | 需求变化 | 🟡中 | 🔴高 | 🟠中高 |
| 市场风险 | 竞争加剧 | 🔴高 | 🟡中 | 🟠中高 |
| 市场风险 | 价格战 | 🟡中 | 🟡中 | 🟡中 |
| 政策风险 | 监管变化 | 🟢低 | 🔴高 | 🟡中 |
| 政策风险 | 补贴退坡 | 🟡中 | 🟡中 | 🟡中 |
| 运营风险 | 团队流失 | 🟡中 | 🔴高 | 🟠中高 |
| 运营风险 | 资金链断裂 | 🟡中 | 🔴高 | 🟠中高 |
| 运营风险 | 供应链中断 | 🟢低 | 🔴高 | 🟡中 |
| 法律风险 | 知识产权纠纷 | 🟡中 | 🟡中 | 🟡中 |
| 法律风险 | 合同纠纷 | 🟡中 | 🟡中 | 🟡中 |

### 4.2 风险等级说明

| 等级 | 标识 | 定义 | 应对策略 |
|------|------|------|----------|
| 高风险 | 🔴 | 可能性高+影响大 | 规避或转移 |
| 中高风险 | 🟠 | 可能性中+影响大 或 可能性高+影响中 | 加强监控+预案 |
| 中风险 | 🟡 | 可能性中+影响中 | 日常管理 |
| 低风险 | 🟢 | 可能性低+影响中/小 | 接受风险 |

### 4.3 综合风险评级

| 评级 | 条件 | 建议 |
|------|------|------|
| 🟢低风险 | 5项以下风险，且无🔴项 | 可正常推进 |
| 🟡中风险 | 1-2项🟠，或3项以上🟡 | 需重点关注，制定预案 |
| 🟠中高风险 | 3项以上🟠，或1项以上🔴 | 需深度评估，谨慎决策 |
| 🔴高风险 | 2项以上🔴，或多项🟠集中爆发 | 建议暂缓，重新评估 |

- 商业逻辑：✅清晰/⚠️待完善/❌不成立
- 综合风险：🟢低/🟡中/🟠中高/🔴高

## 二、材料核查明细
[详细记录每项材料的核查结果]

## 三、技术评估详情
[技术参数、TRL等级、竞品对比]

## 四、商业逻辑分析
[市场、竞争、商业模式、增长]

## 五、风险量化矩阵
[风险清单、等级、应对措施]

## 六、综合建议
[是否推荐招商、关键条件、注意事项]
