# 基于诊断假设的重点问诊与体格检查流程

## 何时使用
- 患者就诊时有明确主诉（如胸痛、气短、腹痛、呕吐等）；
- 场景为急诊或门诊，时间资源有限；
- 医生已具备基础疾病知识和临床推理能力；
- **不适用于**无明确主诉、需全面筛查或健康体检的场景。

## 如何执行
### 第一阶段：重点问诊（驱动诊断假设）
1. **明确主诉**，推断可能受累的器官系统（如“气短”→心血管/呼吸系统）。
2. **围绕该系统深入问诊**，收集：
   - 症状的时间演变（起病、性质、强度、频率）；
   - 加重/缓解因素；
   - 相关伴随症状（如气短需问端坐呼吸、夜间阵发性呼吸困难、踝水肿等）。
3. **主动记录阴性症状**（即预期但未出现的症状），这对鉴别诊断至关重要。
4. **选择性采集其他病史**：
   - 过去史：仅限与当前问题相关者，必问用药史、过敏史，育龄女性必问妊娠可能；
   - 个人史：根据假设选择（如吸烟史对呼吸系统疾病）；
   - 家族史：仅当怀疑遗传性疾病时询问；
   - 系统回顾：仅总结阳性发现，用于修正假设。

### 第二阶段：重点体格检查（验证/修正假设）
1. **基于问诊形成的诊断假设**，设计针对性查体方案。
2. **保持逻辑顺序但大幅精简内容**（如从头到脚，但跳过无关系统）。
3. **优先检查主诉相关系统**，同时兼顾关联体征（如心衰患者查颈静脉、肝颈回流征、下肢水肿）。
4. **避免“头痛查头、脚痛查脚”**：重点检查≠局部检查，必须体现系统性临床思维。
5. **减少患者不适**：尽量减少体位变动，高效完成关键项目。

### 输出结果
- 形成支持或否定诊断假设的关键阳性/阴性体征列表；
- 明确下一步诊疗方向（如安排特定辅助检查、启动经验性治疗或转诊）。

> **关键原则**：诊断假设是动态工具，不是先入为主；问诊与查体是不断交互验证的过程。
