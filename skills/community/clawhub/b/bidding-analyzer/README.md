## 🎯 功能简介

专为政企/企业采购投标场景设计。上传招标文件并提供企业知识库后，本技能将自动输出：

| 输出模块 | 说明 |
|:---|:---|
| ✅ **投标准备事项清单** | 资质/商务/技术/报价/流程管控，含时间节点倒推 |
| 📄 **预算驱动型技术方案** | 按预算档位自动下钻至可执行级细节，支持100~300+页标准Word排版 |
| 📊 **评分模拟分析** 🆕 | 基于评分办法自动建模仿真，量化评估得分区间与风险点 |
| 🥇 **竞争优势评估** 🆕 | 多维度对比竞品，识别优势卖点与潜在失分项 |
| 💡 **投标策略建议** 🆕 | 从"保本"/"冲高"/"守成"等多策略维度给出组合建议 |
| ⚠️ **风险与注意事项清单** | 废标条款识别、评分短板预警、合规澄清建议 |

## 🆕 V1.2.0 新功能详解

### 📊 评分模拟分析

基于招标文件的评分办法，自动构建评分模型，模拟不同策略下的得分区间：

```
┌─────────────────────────────────────────────┐
│  📊 评分模拟分析报告                        │
├─────────────────────────────────────────────┤
│  【技术分】                                  │
│  ├─ 技术方案完整度        22-25分 (权重25%)│
│  ├─ 项目团队配置          13-15分 (权重15%)│
│  └─ 实施计划可行性         8-10分 (权重10%)│
│                                              │
│  【商务分】                                  │
│  ├─ 价格得分              26-30分 (权重30%)│
│  └─ 售后服务               8-10分 (权重10%)│
│                                              │
│  📈 预估总分范围：77-90分                    │
│  📉 最低分风险：低于80分概率≈15%             │
│  ⚡ 关键变量：价格策略(±4分)                  │
│  💡 建议：优先优化技术方案完整度              │
└─────────────────────────────────────────────┘
```

通过蒙特卡洛仿真（1000次迭代），量化评估不同投标策略下的得分分布，让您提前了解**分数天花板**与**失分风险点**。

### 🥇 竞争优势评估

| 维度 | 我方 | 竞品A | 竞品B | 优势判定 |
|:---|:---:|:---:|:---:|:---:|
| 🏗️ 技术架构 | ★★★★★ | ★★★★☆ | ★★★☆☆ | ✅ 显著优势 |
| 💰 价格竞争力 | ★★★☆☆ | ★★★★☆ | ★★★☆☆ | ⚠️ 需关注 |
| 🛡️ 资质案例 | ★★★★☆ | ★★★☆☆ | ★★★★☆ | ⚖️ 持平 |
| ⏱️ 交付能力 | ★★★★★ | ★★★☆☆ | ★★★★☆ | ✅ 显著优势 |
| 🔧 售后服务 | ★★★★☆ | ★★★★☆ | ★★★☆☆ | ✅ 微优势 |

自动结合招标评分权重，输出**差异化竞争策略**，帮助您在评审中最大化得分。

### 💡 投标策略建议

根据评分模拟与竞争分析结果，智能推荐三种策略路径：

| 策略 | 适用场景 | 核心打法 | 预估得分 |
|:---|:---|:---|:---:|
| 🛡️ **保本策略** | 竞争激烈、预算有限 | 控制成本+基础合规 | 75-82分 |
| 🚀 **冲高策略** | 技术领先、资质充足 | 拉满技术分+展示亮点 | 85-93分 |
| ⚖️ **守成策略** | 关系型项目、议价空间小 | 平衡各维度+稳中求胜 | 80-88分 |

## 👥 典型用户场景

### 🏢 场景一：中小型IT企业投标经理
> *"公司准备投一个350万的政务云项目，评分办法很复杂，竞争对手信息零散。我需要快速知道：我们胜算多大？哪个维度会丢分？技术方案写到什么深度才够？"*

**本技能解决方案：** 输入招标文件+企业知识库 → 自动输出评分模拟分析（预估85-90分）、竞争优势评估（技术架构领先+价格弱势）、投标策略建议（建议"冲高策略"拉满技术分）→ 同时生成约200页技术方案草稿。

### 🏗️ 场景二：大型集成商售前团队
> *"每周投3-5个项目，每个项目评分标准不同，全靠人工翻标书找重点太慢了。想统一投标模板，又能按项目自动适配。"*

**本技能解决方案：** 标准化输入接口 → 自动提取评分权重并排优先级 → 按预算档位生成对应深度的方案框架 → 全员统一输出格式，质量可控。

### 🆘 场景三：投标新手/跨行业拓展
> *"第一次投教育行业的标，对这个行业的评分偏好和竞争对手完全没概念。"*

**本技能解决方案：** 提供招标内容 → 技能自动解析评分体系 → 结合公用知识库给出行业典型对标分析 → 输出"新手避坑指南"，降低试错成本。

### 📈 场景四：投标决策层快速判断
> *"明天要决定是否参与某个大项目的投标，今天必须出判断依据。"*

**本技能解决方案：** 输入招标核心内容 → 10分钟内输出投标准备清单+评分预估+策略建议 → 量化数据辅助决策，告别"拍脑袋"。

## 🚀 快速开始

### 1. 准备输入材料
- 📄 粘贴招标文件核心章节（采购需求/评分办法/资质要求）
- 📚 提供企业知识库（产品手册/历史中标方案/资质证书，建议RAG检索后传入）
- 📊 （可选）提供竞品知识库，启用竞争优势评估

### 2. 配置调用

```json
{
  "skill": "bidding_document_analyzer",
  "inputs": {
    "bidding_content": "<招标文件全文或核心章节>",
    "company_kb": "<企业知识库/产品方案/资质案例>",
    "company_name": "XX科技有限公司",
    "project_type": "信息化/云平台",
    "budget_override": "300-500万",
    "scoring_mode": "detailed",
    "competitor_kb": "<已知竞品信息/行业报告>"
  }
}
