# Proof Asset Use-Case Decision Table

| Situation | Template | Why |
|---|---|---|
| Starting proof collection | `case-study-intake-questionnaire` | Collects facts, approvals, and constraints. |
| Interviewing stakeholders | `client-outcome-interview-guide` | Structures evidence gathering. |
| Internal sales enablement only | `internal-case-study-brief` | Keeps sensitive details internal. |
| Public attribution not approved | `anonymized-case-study` | Protects identity and confidential details. |
| Public attribution approved | `public-named-client-case-study` | Uses approved named-client story. |
| Show workflow change | `before-after-workflow-summary` | Compares current and improved workflow. |
| Tell delivery narrative | `implementation-story` | Explains project arc. |
| Technical audience | `technical-implementation-note` | Summarizes architecture and controls. |
| Governance proof | `risk-control-summary` | Shows risk/control approach. |
| Approval workflow proof | `human-approval-gate-summary` | Shows humans stayed in control. |
| Evaluation proof | `evaluation-results-summary` | Packages test/evaluation evidence. |
| Ask client for quote | `testimonial-request` | Requests approval tactfully. |
| Draft quote for review | `testimonial-draft` | Gives client editable starting point. |
| Reusable proof snippets | `proof-point-library` | Stores approved proof. |
| Sales handout | `sales-one-pager` | Condenses proof for buyer conversations. |
| Website story | `website-case-study` | Public web format. |
| LinkedIn content | `linkedin-post` | Social proof post. |
| Email nurture | `email-nurture-story` | Turns proof into email story. |
| Referral enablement | `referral-enablement-blurb` | Helps client introduce CompleteTech LLC. |
| Portfolio listing | `portfolio-entry` | Short public/internal portfolio entry. |
| Speaking/podcast outreach | `speaker-podcast-pitch` | Turns project into pitch. |
| Award submission | `award-submission-draft` | Formats story for awards. |
| Press announcement | `press-release-draft` | Announcement draft with approval caveats. |
| Quote approval | `customer-quote-approval-checklist` | Prevents unapproved quote use. |
| Confidentiality review | `confidentiality-and-anonymization-checklist` | Checks what must be removed. |
| Very short proof | `micro-proof-snippet` | Reusable two-to-four sentence proof. |
| Proof governance | `proof-approval-status-tracker` | Tracks what can be used where. |

Selection rule: choose by approval status first, then channel. If facts are unapproved, stay internal or anonymized. If metrics are not verified, use qualitative language or mark approval required.
