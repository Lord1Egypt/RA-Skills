# 机构模式 签名与验签规则（境外）

> 本文档**完全按照境外（跨境）官方文档独立编写**，不沿用境内模板。
>
> 文档来源：[境外签名生成](https://pay.weixin.qq.com/doc/global/v3/zh/4012354988.md) / [境外签名验证](https://pay.weixin.qq.com/doc/global/v3/zh/4012354989.md) / [境外回调报文解密](https://pay.weixin.qq.com/doc/global/v3/zh/4012354990.md) / [境外敏感信息加解密](https://pay.weixin.qq.com/doc/global/v3/zh/4012354992.md) / [证书与回调验签机制](https://pay.weixin.qq.com/doc/global/v3/zh/4012354982.md)。
>
> ‼️ 境外接入只有"机构"这一种角色，请求体必带 `sp_mchid` + `sp_appid` + `sub_mchid`；本 Skill 不覆盖直连模式（`mchid` + `appid`）。

## 一、机构维度的核心约束

| 约束 | 说明 |
|---|---|
| **签名主体是机构** | 所有 API 请求用**机构** `sp_mchid` / `serial_no` / `private_key` 签名，**不是子商户的** |
| **子商户用请求体字段表达** | 在请求体中用 `sub_mchid`（必填）+ `sub_appid`（部分产品必填）传递子商户上下文 |
| **统一路径前缀 `/v3/global/`** | 境外所有接口都以 `/v3/global/` 开头（**不是境内的 `/v3/pay/partner/`**） |
| **回调机构维度配置** | 通过设置事件通知地址 API 配置，所有子商户共用同一回调，**业务侧必须按 `sub_mchid` 路由** |
| **APIv3 密钥共用** | 机构维度配置一把 32 字节密钥，机构所有子商户产生的回调都用这把密钥 AEAD 解密 |

## 二、为什么签名 / 为什么验签

| 场景 | 谁签 | 谁验 | 用什么 |
|---|---|---|---|
| 请求签名 | 机构用**机构私钥**签 | 微信支付用机构**公钥**验 | RSA-SHA256 |
| 应答验签 | 微信支付用**平台私钥**签 | 机构用**平台证书**验 | RSA-SHA256 |
| 回调验签 | 微信支付用**平台私钥**签 | 机构用**平台证书**验 | RSA-SHA256 |
| 调起支付签名（JSAPI / APP / 小程序）| 机构用**机构私钥**签 | 微信支付前端 / 客户端验 | RSA-SHA256 |
| 回调解密 | 微信支付用 APIv3 密钥加密 | 机构用 APIv3 密钥解密 | AEAD-AES-256-GCM |

> ‼️ **境外认证类型**：`Authorization: WECHATPAY2-SHA256-RSA2048 ...`，与境内字面值相同，但机构号 / 私钥 / APIv3 密钥 / 平台证书是**两套独立体系**，请勿混用。

## 三、请求签名（机构 → 微信支付）

### 3.1 拼接 5 行签名串

```
{HTTP方法}\n
{URL（去域名+完整 query）}\n
{时间戳}\n
{随机串}\n
{请求体}\n
```

每行末尾必须含 `\n`，**最后一行也保留 `\n`**（GET 等 body 为空时，最后一行只剩 `\n`）。

#### 字段取值

| 字段 | 取值 | 注意事项 |
|---|---|---|
| HTTP 方法 | `GET` / `POST` 等大写 | 不要小写 |
| URL | 从 `https://apihk.mch.weixin.qq.com/v3/global/transactions/jsapi` 取 `/v3/global/transactions/jsapi` | 境外**统一前缀 `/v3/global/`**；带 query 的接口要包含 `?key1=val1&key2=val2` |
| 时间戳 | UNIX 秒 | 不是毫秒；时差与微信支付服务器超过 **5 分钟**直接拒签 |
| 随机串 | 32 位以下随机字符串 | 仅数字 + 字母；与 `Authorization` 头里的 `nonce_str` 字节级一致 |
| 请求体 | 完整请求 body 字符串（**含 `sp_mchid` / `sub_mchid`** 等机构字段）| GET / DELETE 无 body 接口填 `""`；序列化前后字节级一致 |

### 3.2 签名计算

使用**机构** API 证书私钥 `private_key`，对上面拼接出来的 5 行签名串做 RSA-SHA256 签名，再 Base64 编码：

```
signature = Base64( SHA256-RSA( signMessage, 机构私钥 ) )
```

‼️ 不要用子商户私钥签名——子商户在 V3 体系中**没有自己的私钥**。

### 3.3 拼装 `Authorization` 头

```
Authorization: WECHATPAY2-SHA256-RSA2048 mchid="<机构号>",nonce_str="<随机串>",timestamp="<时间戳>",serial_no="<机构证书序列号>",signature="<签名值>"
```

| 字段 | 来源 |
|---|---|
| `mchid` | **机构号**（即 `sp_mchid`，不是子商户号） |
| `nonce_str` | 与签名串第 4 行相同 |
| `timestamp` | 与签名串第 3 行相同 |
| `serial_no` | **机构** API 证书序列号（**不是平台证书序列号、不是子商户的**） |
| `signature` | 上一步 Base64 后的签名值 |

> ‼️ **必须头**：调用 V3 接口必须显式设置 `Content-Type: application/json` / `Accept: application/json` / `User-Agent`。

### 3.4 子商户字段在请求体中的位置

机构所有支付类接口请求体必带：

```json
{
  "sp_mchid": "<机构号>",
  "sp_appid": "<机构 AppID>",
  "sub_mchid": "<子商户号，必填>",
  "sub_appid": "<子商户 AppID，部分产品必填>",
  "out_trade_no": "...",
  "amount": { "total": 100, "currency": "USD" },
  "payer": { "sub_openid": "<子商户 AppID 下的 OpenID>" }
}
```

‼️ **不要把 `sp_mchid` 与 `sub_mchid` 弄混**：`sp_*` 是机构（"sp" = service provider），`sub_*` 是子商户。

## 四、应答验签（微信支付 → 机构）

### 4.1 取应答头

| Header | 含义 |
|---|---|
| `Wechatpay-Timestamp` | 应答时间戳 |
| `Wechatpay-Nonce` | 应答随机串 |
| `Wechatpay-Signature` | 应答签名值（Base64）|
| `Wechatpay-Serial` | 平台证书序列号（用于本地匹配公钥）|

### 4.2 拼接 3 行验签串

```
{Wechatpay-Timestamp}\n
{Wechatpay-Nonce}\n
{响应体}\n
```

最后一行（响应体）末尾**也必须保留 `\n`**。

### 4.3 验签计算

1. 用 `Wechatpay-Serial` 在本地缓存中查找**平台证书**；查不到时调用 `GET /v3/global/certificates` 重新拉取并缓存。
2. 从平台证书中提取**平台公钥**。
3. 用平台公钥对 `Wechatpay-Signature` 做 RSA-SHA256 验签。

> ‼️ 机构平台证书与境内**完全独立**，不要把境内证书拿来验境外应答。

## 五、调起支付签名（JSAPI / APP / 小程序）

### 5.1 JSAPI / 小程序

4 行签名串（注意大小写：`timeStamp`，与前端 `timestamp` 不同）：

```
{appId}\n
{timeStamp}\n
{nonceStr}\n
{package}\n
```

| 字段 | 来源（机构维度） |
|---|---|
| `appId` | 与下单时传入 `sub_openid` 时使用的 AppID 一致：用 `sub_openid` 时填 `sub_appid`，用 `sp_openid` 时填 `sp_appid` |
| `timeStamp` | UNIX 秒 |
| `nonceStr` | 32 位以下随机串 |
| `package` | `prepay_id=wx20...` 形式 |

签名值 = `Base64( SHA256-RSA( signMessage, 机构私钥 ) )`。

‼️ **签名用机构私钥，但 `appId` 必须与下单时使用的 OpenID 来源 AppID 配套**——这是非常容易出错的地方。

### 5.2 APP

签名串与 JSAPI 类似（4 行），第一行用与下单 AppID 配套的移动应用 AppID（一般是 `sub_appid`），第四行 `package` 固定为 `Sign=WXPay`。

## 六、敏感信息加解密

部分接口（分账接收方添加 / 子商户进件 / 退款 / 委托代扣等）需要在请求体中传递敏感字段（姓名、身份证号、手机号、银行账号）。

### 6.1 加密算法

`RSA / ECB / OAEPWithSHA-1AndMGF1Padding` —— 用**平台公钥**加密：

```
encryptedField = Base64( RSA_OAEP( plaintext, platformPublicKey ) )
```

请求体中字段填密文；HTTP 头加：

```
Wechatpay-Serial: <用于加密的平台证书序列号>
```

> ‼️ 子商户进件接口对图片以外的所有敏感字段都要加密，且头里 `Wechatpay-Serial` 不能少。

### 6.2 解密算法

应答里的敏感字段（如签约通知中的 OpenID 等）用**机构私钥**解密：

```
plaintext = RSA_OAEP_decrypt( Base64Decode( ciphertext ), 机构私钥 )
```

## 七、境外签名 6 大重点核对

1. **认证类型**：`Authorization: WECHATPAY2-SHA256-RSA2048 mchid="..."`，`mchid` 是**机构号**（即 `sp_mchid`）。
2. **5 行签名串**：每行末尾必须含 `\n`，包括最后一行。
3. **URL 去域名**：境外路径全部以 `/v3/global/` 开头（**不要使用境内的 `/v3/pay/partner/`**）。
4. **时间戳精度**：UNIX 秒；时差超过 5 分钟拒签，**跨境部署特别注意服务器时区与 NTP 同步**。
5. **私钥**：机构 API 证书私钥（**不是子商户私钥、也不是平台证书私钥**）。
6. **`serial_no`**：机构 API 证书序列号（**不是平台证书序列号**）。

## 九、跨境部署常见踩雷

| 项 | 说明 |
|---|---|
| **服务器时区与 NTP** | 跨境节点容易出现 5 分钟时差，签名直接拒签；建议统一 UTC + 强制 NTP |
| **代理 / CDN 过滤签名头** | CloudFront / Cloudflare 默认会过滤 `Wechatpay-*` 4 个头 → 显式放行，或应用层直连 |
| **多语言错误描述** | 显式声明 `Accept-Language: en` / `zh-CN` / `zh-HK` / `zh-TW` |
| **境内境外密钥混用** | 境内 API 证书 / APIv3 密钥与境外**完全独立**，禁止共用一份变量 |
| **机构 / 子商户字段混用** | `sp_*` 是机构、`sub_*` 是子商户；调起支付的 `appId` 必须与下单 OpenID 来源 AppID 一致 |
| **路径前缀错写** | 境外所有路径必须 `/v3/global/...`，不是境内的 `/v3/pay/partner/...` |

## 十、与境内的差异速查

| 项 | 境内 | 境外 |
|---|---|---|
| 域名 | `api.mch.weixin.qq.com` | `apihk.mch.weixin.qq.com` |
| 路径前缀 | `/v3/pay/partner/...` 等 | **统一 `/v3/global/...`** |
| 平台证书路径 | `/v3/certificates` | `/v3/global/certificates` |
| 跨境分账 | 无 | `/v3/global/profit-sharing/...` |
| `Accept-Language` | 默认中文 | 显式声明 |
| `detail.sign_information` | 不返回 | 签名错误**返回**辅助排查 |
| 多币种 | 默认 CNY | 必填 `amount.currency` |
