# Output Template — Supplier Scorecard

One scorecard per supplier per review period. For comparisons, produce the table side-by-side and one shared recommendation section.

## 1. Header

```
Supplier: [name / code]
Product(s): [SKUs covered]
Review period: [quarter / date range]
Orders in evidence log: [n]
Scored by: [name]   Date: [date]
Previous score: [n or "first scorecard"]
```

## 2. Score Table

| Dimension | Weight | Score (1–10) | Weighted | Evidence (cite log entries) |
|---|---|---|---|---|
| Quality consistency | 30% | | | [defect %, spec deviations, batch variance] |
| Reliability | 25% | | | [complete-and-correct order ratio] |
| Lead time accuracy | 20% | | | [avg deviation from promised dates] |
| Communication | 15% | | | [response times, proactive disclosures] |
| Cost competitiveness | 10% | | | [landed cost vs. market, terms, MOQ] |
| **Total** | 100% | | **/100** | |

Weights may be adjusted only with a written reason stated here.

## 3. Band & Trend

```
Band: [Strategic partner 85+ / Solid 70–84 / Improvement plan 55–69 / Transition <55]
Trend vs. last period: [+/− points, which dimensions moved, why]
Leading indicators flagged: [e.g., response-time slippage, unannounced substitutions]
```

## 4. Recommendation (the decision, not a summary)

One of: consolidate volume / hold and monitor / improvement plan / qualify backup / begin transition — with the specific next action, owner, and deadline. For comparisons: which supplier gets which share of volume and why the score gap does or doesn't justify switching (include switching-cost estimate).

## 5. Actions for Next Period

| Action | Owner | Deadline | Success measure |
|---|---|---|---|
| [e.g., attach ±3-day lead-time target to Q3 POs] | | | |

Max 4 actions. Each measurable.

## 6. Re-score Trigger

Next scheduled re-score date, plus ad-hoc triggers (failed inspection, missed launch, >X% defect order).

## Quality gates

- Every score cites log evidence — no impression scores
- Weights sum to 100%; deviations from defaults justified in writing
- Band and recommendation consistent with the banding rules
- Switching recommendations include switching-cost estimate and sustained-gap evidence
- Actions ≤4, each with owner, deadline, and measure
