## 三、类比库（B+A 反讽系核心训练）⭐

### 抽象 → 具体类比对照

| 抽象状态 | 类比 | 出处 |
|---------|------|------|
| 焦虑 | 后台程序太多 / CPU 占用 100% | — |
| 社交 | 信号搜索 / 蓝牙配对 | — |
| 学习 | 代餐 / 不耐受 | B+A 反讽系代表 |
| 早晨 | 开机 / 系统加载 | B+A 反讽系代表 |
| 离开 | 延迟加载 / 缓存清理 | — |
| 拖延 | 心电图 / 进度条卡住 | — |
| 选择困难 | 路由表混乱 | — |
| 通勤 | 数据传输 | — |
| 学历 | 驾照 | B+A 反讽系代表 |
| 人脉 | 信号塔 | — |
| 情绪低落 | 电量耗尽 | — |
| 自我怀疑 | Bug 报告 | — |
| 内心戏 | 后台进程 | — |
| 假精致 | UI 渲染但逻辑空洞 | — |
| 装忙 | CPU 风扇响但没在跑任务 | — |

### 类比库使用法

1. **找你最常感受的 5 个情绪** → 各配一个类比
2. **写标题前先问：能否替换成类比？** （"我焦虑" → "我的后台程序太多了"）
3. **类比要"有反应"**：技术词配生活情绪 / 物件配抽象状态最有反差
4. **避免烂梗**：不用"emo"、"破防"、"yyds" 这种已经被烂用的，用计算机/医学/物流术语反而新鲜

## 四、温度判断速查

`reverse_engineer.detect_temperature(notes)` 的输出可以直接对照下表：

| 温度 | 对应路线 | 适合博主类型 | 用户获得感 |
|------|---------|-----------|-----------|
| 治愈 | `B_HEALING` | B 型纯诗意命名 | 被理解 + 被命名 |
| 反讽 | `BA_IRONY` | B+A 混合（B+A 反讽系）| 被解构 + 被治愈反讽地 |
| 锋利 | `C` | C 现实策略 | 被激活 + 被赋能 |

**自动判断信号词：**
- 治愈：接纳 / 理解 / 陪伴 / 温柔 / 如灯塔 / 暖到了 / 加 `*` 符号
- 反讽：塌房 / 审判 / 破碎 / 装 / 驾照 / 代餐 / 开机慢 / 不耐受 / 加 `:` 冒号
- 锋利：骗子 / 不要脸 / 现实 / 潜规则 / 策略 / 拿回 / 穷 / 加 `！?`

## 五、迁移操作流程（4 步）

```
Step 1. 跑 reverse_engineering 拿到温度判断
   ↓
Step 2. detect_route() 决定 B_HEALING / BA_IRONY / A / C
   ↓
Step 3. 调用对应路线的 6 模型 + 隐形公式 + 30 选题
   ↓
Step 4. 用改写示例对照表 + 类比库做内容生产
```

### 代码示例

```python
from xhscosmoskill import (
    detect_temperature, detect_route,
    BA_IRONY_MODELS, generate_formula_report
)

# 拿到博主温度
temp = detect_temperature(notes)
# 决定路线
route = detect_route(archetype, temp["primary"])
# 拿到对应路线的 6 模型
models = get_models_for_route(route)
# 生成完整公式报告
md = generate_formula_report(notes, name, archetype, temp["primary"])
```

## 六、混合博主特殊处理

### 同 B+A 但温度不同（B+A 反讽系 vs B 治愈系）

| 维度 | B 治愈系代表 (B+A 治愈系)| B+A 反讽系代表 (B+A 反讽系) |
|------|---------------------|-----------------|
| 招牌技 | 给情绪命名（Odyssey 时期）| **A 像 B 类比 + 病理化命名** |
| 品牌符号 | `*` 显式 | 无（极简短句即识别）|
| 用户得到 | 被理解 + 被命名 | 被解构 + 被治愈反讽地 |
| 一句话 | 让你"被她说出来" | 让你"看见自己被他解构" |

**规则：** 同型不同温度，应走不同路线，不要强套同一套模板。

### 单一博主多路线信号

如果 `detect_temperature` 返回多个温度都接近（is_hybrid=True），可以：
1. 主路线按主温度走
2. 副路线作为"扩展模型"补充进 6 模型（例如 5 个主路线 + 1 个副路线）
