# 输出模板

## 1. 可追踪市场 — 初始快照报告

```markdown
# 结算追踪报告

**市场：** {event_title}
**链接：** {market_url}
**生成时间：** {YYYY-MM-DD HH:MM:SS} UTC
**结算日期：** {end_date}
**剩余时间：** {hours} 小时

## 结算数据源

| 字段 | 值 |
|------|------|
| 数据源 URL | {resolution_url} |
| 指标 | {metric_name} |
| 可追踪性 | {完全/部分/手动} |
| 数据方法 | {html_table/embedded_json/api} |

## 当前状态

### 结算数据

| 排名 | 候选项 | 分数 | 置信区间 | 模型/条目 |
|------|--------|------|----------|-----------|
| {rank} | {candidate} | {score} | ±{ci} | {model_name} |
| ... | ... | ... | ... | ... |

### 市场价格

| 候选项 | 市场价格 | 结算数据排名 |
|--------|---------|-------------|
| {candidate} | {price}% | #{rank} |
| ... | ... | ... |

## 对齐分析

**数据领先者：** {leader}（分数：{score}）
**市场领先者：** {market_leader}（{price}%）
**状态：** {对齐 / 偏离}

## 风险评估

| 因素 | 评估 |
|------|------|
| 领先差距 | {gap} 分 |
| 置信区间重叠 | {是/否}（领先者 CI: ±{ci1}，第二名 CI: ±{ci2}） |
| 距结算时间 | {hours} 小时 |
| 市场确信度 | {leader_price}%（高/中/低确信度） |

## 监控配置

| 设置 | 值 |
|------|------|
| 检查间隔 | {interval} 分钟 |
| 状态目录 | {state_dir} |
| 告警日志 | {alert_log_path 或 "仅终端"} |
```

## 2. 不可追踪市场报告

```markdown
# 结算追踪报告 — 不可追踪

**市场：** {event_title}
**链接：** {market_url}
**生成时间：** {YYYY-MM-DD HH:MM:SS} UTC

## 评估结果

**可追踪等级：** {手动/不可}

### 原因

{列出无法自动追踪的具体原因}

1. {原因_1}
2. {原因_2}

### 结算条件（来自描述）

> {引用市场描述的相关部分}

## 建议

{如果为手动等级：}
- 按以下计划手动检查 {source_url}：
  - {根据距结算时间制定的计划}
- 关键数据点：{检查内容}

{如果为不可等级：}
- 该市场结算条件具有主观性，无法可靠追踪。
- 建议改为监控市场价格变动作为替代。
```

## 3. 监控告警格式

### 终端输出格式

```
══════════════════════════════════════════════════════
[{YYYY-MM-DD HH:MM} UTC] 结算追踪器已启动
市场：{event_title}
数据源：{resolution_url}
结算时间：{end_date}（剩余 {hours} 小时）
──────────────────────────────────────────────────────
[HH:MM] #{rank} {leader} {score}±{ci} | #{rank} {second} {score}±{ci} | 差距: {gap}
        市场: {leader} {price}% | {second} {price}% | ...
        状态: 对齐 ✓
──────────────────────────────────────────────────────
[HH:MM] 无变化。下次检查在 {interval} 分钟后。
[HH:MM] WARNING: {org} 分数变化 {old}→{new}，差距缩小至 {gap}
[HH:MM] CRITICAL: 领先者变更！{new_leader} {score} > {old_leader} {score}
══════════════════════════════════════════════════════
```

### 告警严重等级

| 等级 | 颜色 | 触发条件 |
|------|------|----------|
| **CRITICAL** | 红色加粗 | 领先者变更；结算结果将翻转 |
| **WARNING** | 黄色 | 差距缩小；置信区间重叠；分数显著变化 |
| **ALERT** | 品红色加粗 | 市场价格与结算数据不一致 |
| **INFO** | 灰色 | 无变化；微小更新；状态报告 |

### JSON 告警格式（日志文件用）

```json
{
  "type": "CRITICAL",
  "message": "领先者变更！Google 1504 > Anthropic 1502",
  "data": {
    "new_leader": "Google",
    "old_leader": "Anthropic",
    "new_score": 1504,
    "old_score": 1502
  },
  "timestamp": "2026-02-24T12:00:00+00:00"
}
```

## 4. 监控间隔指南

| 距结算时间 | 建议间隔 |
|-----------|---------|
| > 7 天 | 360 分钟（6 小时） |
| 2-7 天 | 120 分钟（2 小时） |
| 1-2 天 | 60 分钟（1 小时） |
| < 24 小时 | 30 分钟 |
| < 6 小时 | 15 分钟 |
