## Agent协作方法论要求清单

Agent协作领域每种任务类型的"零件清单"——必选/可选组件、组装顺序、领域约束。按清单逐项产出。

---

### Agent数据结构规范

#### Agent字段定义

| 字段 | 类型 | 必选 | 说明 |
|------|------|------|------|
| id | string | 是 | Agent唯一标识符，格式：AGENT-{序号} |
| name | string | 是 | Agent名称，最大50字符 |
| type | string | 是 | Agent类型：Explore/Plan/General-purpose/Custom |
| capabilities | array | 是 | Agent能力列表 |
| tools | array | 是 | Agent可用工具 |
| status | string | 是 | Agent状态：available/busy/offline/error |
| current_task | string | 否 | 当前执行的任务 |
| performance | object | 否 | Agent性能指标 |
| metadata | object | 否 | 自定义元数据 |

#### Agent类型说明

| 类型 | 说明 | 适用场景 |
|------|------|----------|
| Explore | 探索型，快速搜索能力 | 搜索/探索任务 |
| Plan | 规划型，架构设计能力 | 规划/设计任务 |
| General-purpose | 通用型，完整执行能力 | 复杂任务 |
| Custom | 自定义型，根据需求定制 | 特定需求 |

#### 任务委派字段定义

| 字段 | 类型 | 必选 | 说明 |
|------|------|------|------|
| delegation_id | string | 是 | 委派唯一标识符 |
| task_id | string | 是 | 任务ID |
| agent_id | string | 是 | Agent ID |
| delegation_type | string | 是 | 委派类型：sync/async |
| delegation_mode | string | 是 | 执行模式：foreground/background |
| priority | string | 否 | 优先级：high/medium/low |
| timeout | number | 否 | 超时时间（秒） |
| max_retries | number | 否 | 最大重试次数 |
| status | string | 是 | 委派状态：delegated/executing/completed/failed |

### A0 协作协调域

#### A0-01 协作需求分析
- **必选组件**: 协作目标、协作范围、协作需求、协作约束
- **可选组件**: 协作优先级、协作时间要求
- **组装顺序**: 目标确定→范围界定→需求分析→约束识别→需求汇总
- **约束**: 协作目标必须明确；协作范围必须合理；协作需求必须全面
- **格式**: 协作需求分析报告（Markdown）

#### A0-02 协作域选择
- **必选组件**: 可用协作域列表、协作域选择理由、协作域组合方案
- **可选组件**: 协作域依赖关系说明
- **组装顺序**: 需求分析→协作域扫描→选择决策→组合设计
- **约束**: 协作域选择必须基于协作需求，不可强行套用
- **格式**: 协作域选择报告（Markdown）

#### A0-03 执行顺序确定
- **必选组件**: 协作任务依赖关系分析、执行顺序方案、关键路径识别
- **可选组件**: 并行执行可能性分析
- **组装顺序**: 依赖分析→顺序确定→关键路径识别
- **约束**: 执行顺序必须尊重协作任务依赖关系
- **格式**: 执行顺序图（Markdown流程图）

#### A0-04 结果整合与优化
- **必选组件**: 各协作任务执行结果汇总、整合方案、优化建议
- **可选组件**: 整合效果评估
- **组装顺序**: 结果收集→整合分析→优化建议
- **约束**: 整合必须保持各协作任务输出的完整性
- **格式**: 整合报告（Markdown）
