# PQA — 过程质量保证（CMMI V3.0 Core PA）

## PA 基本信息
- **缩写**：PQA
- **英文**：Process Quality Assurance
- **中文**：过程质量保证
- **所属**：Core PA（核心实践域）
- **对应 V2.0**：相当于 PPQA

---

## CMMI V3.0 要求要点

### 核心目的
确保项目和组织过程的正确执行，提供客观的过程和工件可见性。

### 关键实践（Specific Practices）
| 实践编号 | 实践内容 | 说明 |
|---------|---------|------|
| PQA 1.1 | 客观评价过程 | 评价项目过程是否符合组织标准 |
| PQA 1.2 | 客观评价工作产品 | 评价工作产品是否符合标准 |
| PQA 2.1 | 沟通质量保证结果 | QA检查单、审计报告 |
| PQA 2.2 | 确保问题被解决 | 问题跟踪、闭环验证 |
| PQA 3.1 | 策划和实施质量保证 | QA计划 |

### 通用实践（Generic Practices）
- GP 1.1 ~ GP 2.10（完整覆盖）

## 用户文档 → PA 匹配规则

### 高概率匹配（主要PA）
- 文档名含"QA检查"、"审计"、"质量保证" → **PQA 主要匹配**
- 文档含检查单、审计报告 → **PQA 主要匹配**

### 中概率匹配（次要PA）
- 文档含过程合规检查 → 同时匹配 **GOV**（治理）
- 文档含问题跟踪 → 同时匹配 **CAR**（因果分析）

## 差距检查清单（用于差距分析）

### PQA 1.x — 评价
- [ ] 是否有QA计划（覆盖哪些过程、哪些工作产品）？
- [ ] 是否有定期的过程审计记录（QA检查单）？
- [ ] 是否有定期的工作产品审计记录？

### PQA 2.x — 沟通与跟踪
- [ ] QA发现的问题是否有记录并跟踪闭环？
- [ ] QA结果是否定期向项目团队和高层报告？
- [ ] QA是否有独立的上报渠道（不受PM干扰）？

## 评估访谈常见问题（模式D）

### 针对QA/EPG的问题
1. "QA的独立汇报渠道是什么？如何保证客观性？"
2. "请展示最近一次QA审计的检查单和报告。"
3. "QA发现的问题是如何跟踪闭环的？"
4. "QA策划是如何做出的（覆盖哪些过程/产品）？"

### 评分要点
- **PQA 1.1-1.2**：过程和产品评价是否有记录
- **PQA 2.1**：QA结果是否沟通到相关干系人
- **PQA 2.2**：问题是否有跟踪并验证闭环
