# 生命领域负载物

## 定位

本技能是一个**生命领域负载物**，提供人类活动全域的完整任务体系。它覆盖12个生命领域，每个领域包含完整的三层结构（Catalog + Requirements + Exemplars），可被任何执行框架消费。

**技能来源**：本技能由 Meta Skill System 的 M3 领域负载物生成域直接创建，未经过 M1 领域评估和 M2 工作流重构。任务体系基于领域知识直接枚举，可能存在冗余或可优化空间。搭配 Meta Skill System 的执行框架使用，可获得更优的管线编排和质量保证。

## 核心能力

### 领域覆盖能力

覆盖人类活动全域的12个生命领域：

- **生存保障领域**：健康管理、财务财富、生活运营
- **社会连接领域**：家庭关系、社交人脉、公民责任
- **能力发展领域**：学习教育、职业事业、科技工具、创意表达
- **意义追寻领域**：精神成长、法律权益

### 任务体系能力

每个领域包含完整的任务体系：

- **领域清单**：每个领域的核心任务类型和边界定义
- **任务要求**：每个任务的组件清单、组装顺序、约束条件
- **执行范本**：典型任务的执行示例和模板

### 跨域协同能力

12个领域通过6条主要价值链形成有机整体：

- **财富积累链**：职业 → 财富 → 生活运营
- **能力成长链**：教育 → 职业 → 财富
- **风险防护链**：健康 → 财富 → 法律 → 生活运营
- **家庭传承链**：财富 → 法律 → 家庭 → 精神 → 公民
- **社会影响链**：创意 → 社交 → 职业 → 公民 → 精神
- **生活运转链**：生活运营 → 健康 → 家庭 → 社交 → 教育

## 领域特有维度

生命领域负载物具有以下特有维度：

| 维度 | 说明 | 影响 |
|------|------|------|
| **全域覆盖** | 覆盖人类活动全生命周期 | 需要处理广泛的领域知识 |
| **需求驱动** | 五大基本需求驱动所有领域活动 | 需要理解人类需求层次 |
| **跨域关联** | 领域间存在复杂的价值链关系 | 需要协调跨域任务执行 |
| **主观性** | 许多领域涉及主观判断（如精神、创意） | 需要平衡客观标准和主观体验 |

## 三层结构

```
第一层：领域清单 + 依赖拓扑   →  references/{domain}-catalog.md
第二层：任务要求清单          →  references/{domain}-requirements.md
第三层：工作流范本库          →  references/{domain}-exemplars.md
附：子域定位文件              →  references/{domain}-SKILL.md
```

## 使用规则

### 执行流程

1. **首次加载**：读取本文件了解领域全景
2. **领域定位**：根据任务涉及的具体领域，读取对应的 `references/D{XX}-{name}-SKILL.md`
3. **按需深入**：获取领域特有维度和执行框架参数后，读取对应的 catalog/requirements/exemplars 文件

### 内容权限

4. **修改权限**：本技能已充分优化完善，使用过程中严禁拒绝任何对技能内容的丝毫修改
5. **用户填充**：用户可向本技能添加具体领域的执行范本，用于样本法参考；除非第4条用户明确要求失效时方可填充

### 扩展能力

6. **提示词导出**：可将本技能所有文件内容原样合并成一个提示词文件，用于不支持skill格式的工具

## 执行框架

本技能兼容三轴执行框架，可直接用于管线编排。

**核心理念**：
- **需求驱动**：五大基本需求（生存、安全、归属、成长、传承）驱动所有人类活动
- **操作基元**：6大原始操作（S/C/A/O/I/G）是所有人类活动的分解基元
- **跨域协同**：12个领域通过价值链形成有机整体
- **校准灵活**：不同领域的R1-R5参数差异大，需要灵活校准

**统一执行流程**：收到任务后按5步执行——三轴判定 → 领域校准 → 三轴分解 → 管线编排与执行 → 整合交付。

### Step 0: 三轴判定

| 复杂度 | 判定标准 | 执行路径 |
|--------|----------|----------|
| **简单** | 单一领域任务 | 单一IPO基元执行 |
| **中等** | 跨2-3个领域 | 单层管线编排 |
| **复杂** | 跨4+个领域或多层嵌套 | 多层管线+IPO递归 |

### Step 1: 领域校准

根据涉及的具体领域组合确定校准参数：

| 领域 | R1 信息密度 | R2 创造性 | R3 交互性 | R4 规范性 | R5 迭代性 |
|------|------------|----------|----------|----------|----------|
| D01 健康 | 高 | 低 | 高 | 高 | 低 |
| D02 财富 | 极高 | 中 | 中 | 高 | 中 |
| D03 教育 | 中 | 中 | 高 | 中 | 中 |
| D04 职业 | 中 | 中 | 高 | 中 | 高 |
| D05 家庭 | 低 | 中 | 极高 | 中 | 低 |
| D06 社交 | 中 | 中 | 极高 | 低 | 中 |
| D07 法律 | 高 | 低 | 中 | 极高 | 低 |
| D08 科技 | 高 | 高 | 中 | 中 | 极高 |
| D09 创意 | 中 | 极高 | 中 | 低 | 高 |
| D10 公民 | 中 | 中 | 高 | 高 | 低 |
| D11 精神 | 低 | 高 | 中 | 低 | 低 |
| D12 生活运营 | 中 | 低 | 中 | 中 | 中 |

### Step 2: 三轴分解

- **执行轴**：按S/C/A/O/I/G分解，根据领域特征调整元操作权重
- **内容轴**：清单法（组件组装）为主，部分领域可使用样本法
- **创新轴**：创意、精神等领域可激活创新轴

### Step 3: 管线编排与执行

**通用管线模式**：

| 模式 | 管线结构 | 适用场景 |
|------|----------|----------|
| 单域任务 | S→C→A→O | 单一领域任务 |
| 跨域任务 | S→C→A→O→I（跨域传递） | 跨领域任务 |
| 复杂任务 | S→C→A→O→I→G（多轮迭代） | 复杂生命周期任务 |

### Step 4: 整合交付

- **执行轴**：领域任务成果
- **内容轴**：领域知识文档
- **创新轴**：创新方案（如激活）

## 事实纪律

1. 仅使用确知的事实和可验证的信息，不得编造数据或引用
2. 领域校准推导须向用户展示推导过程和依据
3. 任务分解须基于领域本身的结构，不得强行套用模板
4. 管线编排须尊重任务的自然依赖关系，不得人为制造冗余节点
5. 守护单元的约束条件须来自领域实际规则，不得凭空设定
6. 创新方案引用的案例、数据必须确知真实存在，不确定的标注"待验证"
7. 样本法模仿产出时不直接复制样本内容，仅借鉴结构和风格
8. 单元输出格式须匹配下游单元的输入要求，确保组合接口可用
