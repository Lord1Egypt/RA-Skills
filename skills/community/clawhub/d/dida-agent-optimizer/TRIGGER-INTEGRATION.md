# Agent 架构与工程优化师 - 触发机制集成

## 自动触发条件

以下场景自动触发 agent-optimizer skill：

### 1. Cron 执行失败
- 任务超时
- Payload schema 错误
- Session target 不匹配
- 连续 2 次以上失败

### 2. 工具调用异常
- API 返回错误状态码
- 权限不足
- 网络连接失败
- 超时或限流

### 3. 工作流中断
- 子 Agent 异常退出
- 消息发送失败
- 会话状态丢失

### 4. 性能退化
- 响应时间显著增加
- Token 消耗异常增长
- 错误率上升

## 触发方式

### 自动触发
当检测到上述问题时，小七应：
1. 读取 `skills/agent-optimizer/SKILL.md`
2. 按照诊断工作流执行
3. 输出修复建议
4. 用户确认后执行修复
5. 验证并记录

### 手动触发
用户明确要求时：
- "检查一下系统问题"
- "优化一下配置"
- "诊断一下为什么失败"

## 集成到现有流程

### Heartbeat 集成
在 HEARTBEAT.md 中添加优化师巡检：

```markdown
# 🔄 Agent 健康度检查（每月一次）
- 读取 learnings/error-log.md，分析错误趋势
- 检查 cron 任务成功率
- 评估 token 使用效率
- 输出优化建议
```

### 错误处理集成
在工具调用失败后：
1. 记录错误到 learnings/error-log.md
2. 判断是否触发优化师
3. 执行诊断和修复

## 权限与安全

### 自动执行范围
- ✅ 读取日志和状态
- ✅ 临时文件清理
- ✅ 重试限流错误（≤3次）
- ✅ 更新心跳文件

### 需确认操作
- ⚠️ 修改 cron 配置
- ⚠️ 重启服务
- ⚠️ 修改系统配置

### 禁止自动执行
- ❌ 修改核心身份文件（SOUL.md/IDENTITY.md）
- ❌ 发送外部消息
- ❌ 删除不可恢复数据
