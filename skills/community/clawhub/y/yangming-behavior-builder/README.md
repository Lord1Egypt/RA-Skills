# 阳明造人术 · 行为操作系统

> "知是行的主意，行是知的功夫。知行合一，不是先知后行，而是未知便未行。"

## 简介

阳明造人术是一个 AI Agent 技能创建引擎，专注于"行为-执行-成果"研究框架。

输入任何人物的名字，系统会自动通过 6 个并行 Agent 研究其行为模式、执行特征和成果验证，生成可运行的行为视角 skill。

核心创新：**知行摩擦诊断机制** —— 帮助用户识别自身行为与目标人物之间的差距，从而改进行为。

## 与女娲 nuwa-skill 的区别

| 维度 | 女娲 nuwa-skill | 阳明造人术 |
|------|----------------|----------|
| 核心焦点 | 思维框架 / 怎么说 | 行为执行 / 怎么做 |
| 研究对象 | 他们想什么 / 说什么 | 他们做什么 / 结果如何 |
| 输出维度 | 思维模型 + 决策启发 + 表达DNA | 行为指纹 + 执行特征 + 成果验证 |
| 特色机制 | 提取思维框架 | 知行摩擦诊断 |
| 应用场景 | 分析判断 / 角色扮演 | 行为校准 / 执行改进 |

## 核心哲学

**知行合一**：真正的"知道"体现在"做到"。只研究人说了什么是浅层的，研究人实际做了什么、结果如何、能否复制，才是真正的学习。

> "阳明造人术，不是造人，是借镜照己。"

## 工作流程

```
阶段0：确认需求（30秒）
  → 筛选研究价值，决定是否继续

阶段0.5：创建技能目录
  → 建立 research/sources/logs 目录结构

阶段1：6个并行Agent行为数据采集
  → B1执行模式 / B2成果追踪 / B3压力反应
  → B4协作模式 / B5执行节奏 / B6失败记录

阶段1.5：研究质量汇总
  → 展示信息质量表，用户确认后继续

阶段2：行为框架提取
  → 行为模型（3-7个）+ 执行特征（5-10个）
  → 成果验证 + 知行差距分析

阶段3：技能建设
  → 创建 SKILL.md + 知行摩擦诊断脚本
  → 可复用模块检查

阶段4：质量验证
  → 已知行为测试 + 边缘行为推断
  → 知行摩擦测试 + 质量评分

阶段5：周期性执行日志
  → 使用记录 + 摩擦点趋势 + 行为改进跟踪
```

## 适用场景

- **投资知行**：创建投资经理的行为视角 skill，诊断知行摩擦
- **职场顾问**：创建企业家/高管的执行模式参考
- **技能提炼**：将某个人的核心能力结构化、可操作化
- **自我校准**：通过知行摩擦诊断识别自身行为改进方向

## 触发词

当用户表达以下意图时，优先使用本引擎：

- 「创建XX的知行顾问」
- 「创建XX的行为视角」
- 「研究XX的行为」
- 「知行摩擦诊断」
- 「我想分析某人的行为模式并生成skill」

## 输出示例

每个生成的 skill 包含：

- **身份激活规则**：以目标人物视角回应
- **行为指纹档案**：6维度行为研究结果
- **行为模型**：3-7个可观察的行为模式
- **执行特征**：5-10个快速决策规则
- **知行摩擦诊断**：内置诊断脚本 + 使用示例
- **执行日志**：持续跟踪使用效果
