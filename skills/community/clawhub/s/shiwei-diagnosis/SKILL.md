# 施炜企业管理诊断方法论

以三个已交付的完整诊断报告（巴奴、沃森生物、法兰泰克）为方法论来源，提炼可复制的诊断框架和报告输出规范。

## 核心理念

诊断不是列举问题清单，而是建立问题之间的因果结构。从财务表征出发，通过三角验证锁定真问题，再逐层归因到战略、组织、文化，最终落到"业务-能力"、"成长-治理"、"环境-进化"三个基本矛盾。

## 诊断工作流

### 阶段一：资料收集与三角验证

执行以下三个并行的数据采集动作，不允许跳过任何一项：

1. **资料分析**：公司发展历程/大事记、年报/经营分析报告、主要领导讲话、行业文献/网站资料
2. **深度访谈**：中高层管理人员一对一访谈（覆盖率≥80%关键岗位）、员工代表座谈会、分子公司走访
3. **问卷调研**：面向中高层干部的经营管理调查问卷（5点李克特量表）、覆盖战略/运营/组织/人才/文化五维度、回收率≥70%

**调研规模参考基准**（基于三个已交付项目的实际数据）：

| 企业规模 | 访谈人数 | 问卷回收 | 覆盖范围 |
|---------|---------|---------|---------|
| 中型（1000-3000人） | 35-50人 | 150-200份 | 高管→中层，跨2-4城 |
| 中大型（3000-8000人） | 50-75人 | 200-250份 | 高管→基层，跨4-10城 |
| 大型（>8000人） | 70-100人 | 240-300份 | 全层级，跨多省/国际 |

实际参考：巴奴73人访谈+240份问卷（跨4城10店），法兰泰克69人访谈+165份问卷（多分子公司），沃森38人访谈+3场座谈会45人+204份问卷（跨上海/玉溪/北京）。

三角交叉验证原则：访谈中的感知判断必须用问卷数据验证，问卷中的异常打分必须用访谈内容解释。资料中的财务数据是问题表征的起点。

### 阶段二：诊断框架推演

按以下顺序进行：财务表征 → 问题现象 → 原因分析 → 根因诊断 → 变革建议。

先读详细框架参考 `references/diagnosis-framework.md` 再开始推演。

### 阶段三：输出诊断MD文档

按 `references/md-template.md` 的模板结构输出。MD文档是中间产物，供后续PPT生成使用。

### 阶段四：输出诊断PPT

以MD文档为内容源，按 `references/ppt-structure.md` 的PPT结构生成幻灯片。

**生成前必读**：`references/ppt-structure.md` 末尾的「PPT生成技术规范」章节（7条强制规范），包括：
- 装饰线条必须画成横条（禁止画成竖线）
- 卡片高度与内容密度匹配（禁止太空旷）
- 箭头必须用PPT形状（禁止用Unicode字符）
- 飞轮/环形布局用行列+中心方案（禁止角度算法）
- 颜色值必须是6位hex
- 内容稀少页面不拉大卡片
- 脚本禁止引入未使用依赖

**生成工具**：使用 `pptx` skill（Node.js + pptxgenjs），参考 `references/ppt-structure.md` 的页面设计规范。

**生成后验证**（必须执行，审计不通过禁止交付）：

执行 `references/ppt-structure.md` 末尾的「PPT生成审计检查清单」，共7大类21项。有两种执行方式：

- **方式A（推荐）**：运行同目录下的 `audit_ppt.js` 自动化审计脚本，输出 PASS/FAIL 报告
- **方式B（无脚本时）**：逐页打开PPT，对照检查清单人工核验

**门禁规则**：FAIL项 > 0 时，必须先修复（自动修复或手动修改 `generate_ppt.js`），重新生成后重新审计，直到全部 PASS 才可调用的 `present_files` 交付给用户。

审计检查清单详见 `references/ppt-structure.md` 末尾章节，包含：颜色对比度、布局溢出、形状箭头、页面结构、飞轮特殊页、脚本质量、生成后验证等7大类21项具体检查标准。

## 分析框架总览

### 理论基石

诊断方法论建立在以下核心理论模型之上：

1. **企业成长五阶段模型**（施炜《中国企业成长导航》）：创业期→机会成长期→系统成长期→分蘖成长期→整合成长期。用于判断企业发展阶段，定位问题性质。
2. **"一心开二门"模型**：分析战略与组织的互动关系。一心（心愿/心力/心识）→ 战略之门（机会洞察/价值创造/资源整合）→ 组织之门（架构流程/团队建设/企业文化）。
3. **三大基本矛盾**（根因分析层）："业务—能力"矛盾、"成长—治理"矛盾、"环境—进化"矛盾。

### "一心开二门"模型（独立框架）

这是施炜诊断体系中的核心分析模型，用于系统化审视企业的战略端和组织端：

```
              ┌─────────────────┐
              │     心（一心）    │
              │ 心愿·心力·心识   │
              └───────┬─────────┘
                      │
        ┌─────────────┴─────────────┐
        ▼                           ▼
┌───────────────┐           ┌───────────────┐
│  战略之门      │           │  组织之门      │
│ · 机会洞察     │           │ · 架构流程     │
│ · 价值创造     │           │ · 团队建设     │
│ · 资源整合     │           │ · 企业文化     │
└───────────────┘           └───────────────┘
```

**使用场景**：
- 在诊断的第二部分（问题原因分析），用此模型将问题归位——哪些属于战略之门没打开？哪些属于组织之门没打开？
- 在第三部分（变革建议），设计"一心开二门"跃迁蓝图——从当前状态到目标状态的战略和组织双重升级路径。

### 六大诊断维度

> ⚠️ 命名说明：此处"六大诊断维度"覆盖诊断报告的全部维度（战略/运营、组织、人才、文化、领导/决策、财务/经营），与巴奴案例中"组织能力六维模型"（凝聚力/协同力、战斗力、生命力、领导）是不同层级的概念。后者是"组织能力"这个维度的深钻子框架——详见 `references/diagnosis-framework.md` 维度2。

| 维度 | 核心问题 | 典型现象指标 |
|------|----------|------------|
| 战略/运营 | 战略管理是否闭环？运营是否高效？ | 战略解码缺失、资源投向分散、研发-生产-营销脱节 |
| 组织能力 | 组织架构是否匹配战略？平台能力是否发育？ | 职能小散弱、缺乏中枢机构、专业职能断层 |
| 人才 | 人才密度和梯队是否充足？激励是否有效？ | 后备干部断层、考核激励失焦、人才逆淘汰 |
| 文化 | 文化是否支撑战略？是否出现稀释？ | 创业精神流失、官僚化倾向、信任不足 |
| 领导/决策 | 高管团队和决策机制是否成熟？ | 企业家单核驱动、缺乏民主决策机制 |
| 财务/经营 | 财务指标揭示什么趋势？ | 增收不增利、周转率下降、主力品种集中度风险 |

### 三个基本矛盾（根因分析层）

所有企业问题最终归结为以下三个矛盾之一：

1. **"业务——能力"的矛盾**：业务增长在前，组织能力成长滞后。当能力不能有效支撑时，业务增长出现瓶颈。能力的核心是技术资源+组织资源+人力资源。
2. **"成长——治理"的矛盾**：企业规模已进入新阶段，但治理方式仍停留在创业期——企业家个人驱动、缺少高管团队、缺少民主决策、缺少组织规则。
3. **"环境——进化"的矛盾**：核心价值观和认知方式（战略思维）不适应环境变化趋势。需要文化选择和战略选择的双重进化。

### 企业成长阶段锚定

用五个阶段判断企业所处位置：

- **创业期**（创新驱动、商业模式验证）
- **机会成长期**（抓住机遇、市场拉动、领导人驱动、非规范成长）— 问卷得分≥3.5
- **系统成长期**（战略意图成长、要素支撑、管理含量）— 问卷得分≥4.2
- **分蘖成长期**（业务多元化、能力成长、并购整合）
- **整合成长期**（新战略重构、新模式验证）

绝大多数中国企业的问题本质是：处于从机会成长向系统成长跨越的阶段，但治理方式和能力建设拖后腿。

## 诊断报告写作原则

### 结构原则

1. 先肯定后批判：第一部分必须是"历史回顾和成功经验"，展示对企业的尊重和理解
2. 访谈和结论双栏对照：每项问题分析必须包含"访谈整理"和"事实和结论"两部分
3. 问卷数据交叉验证：每个分析维度必须引用问卷打分数据，做层级交叉分析（高管vs中基层、不同部门的打分差异）
4. 文化归因先行：在成功经验部分，关键成功要素必须追溯到文化归因（企业家精神和文化品格）
5. 根因不止于表象：最终必须上升到三个基本矛盾

### 语言原则

- 直接但不冒犯——说真话，但给台阶
- 用企业的语言——引用访谈原话、内部术语
- 结论有判断——不说"可能""大概"
- 数据说话——每个判断都要有数据来源

### 变革建议原则

- 必须是系统性方案，不是散点建议
- 明确转型方向（机会成长→能力成长、中小企业→中大型、国内→全球）
- 给出可操作的实施路径（3年为期，逐层落地）
- 起点通常是：建设高管团队和民主决策机制

## 使用方式

1. 当用户提供访谈记录、问卷数据、企业资料时，自动进入诊断工作流。
2. 先输出诊断MD文档供用户审阅确认。
3. 确认后，基于MD文档输出PPT。MD文档是PPT的内容来源，PPT是MD的视觉呈现。
4. 如用户只提供部分材料（如只有访谈），先追问缺失的数据来源，但可以在已有材料基础上给出初步判断框架。

详细的框架细节、MD模板和PPT结构，分别在 `references/` 目录下的三个文件中。

## 配套脚本（scripts/）

| 脚本 | 用途 | 使用方法 |

每次生成PPT后**必须**运行审计，未通过则禁止交付：

```bash
node scripts/audit_ppt.js scripts/generate_ppt.js
echo $?  # 0=通过，1=有FAIL项
```

审计覆盖7大类21项检查，完整清单见 `references/ppt-structure.md` 末尾。
