# Archetype Composer / 原型组合器

用途：当 `life-archetype-library.md` 没有强匹配时，使用本文件从人生迁移语法、张力轴和事件冲击生成“临时人生原型”。目标是避免每遇到新题就新增固定案例。

## 使用时机

满足任一条件时启用：

- 用户问题不明显属于城市、海外、AI、买房、体制、大厂、旧公司、小城或创业。
- 固定原型只能覆盖题目表层，无法解释隐藏不甘。
- 用户的痛点来自两个以上系统交叉，例如家庭照护 + 职业扩张、健康中断 + 技能转向、关系变化 + 城市迁移。
- 判词删掉主题词后仍能套到多个题，说明原型命中不足。

## 输入来源

组合器只接收已经完成的语法结果：

```text
岔路动作：{{留下/离开/回流/错过/延迟/押注/撤退/转向/被迫中断}}
系统切换：{{系统 A}} vs {{系统 B}}
自我叙事断裂：{{用户真正痛的地方}}
主张力轴：{{张力轴 1}} / {{张力轴 2}} / {{张力轴 3}}
承载物：{{城市/组织/关系/身体/角色/机会/技能/资产/身份}}
可回收资产：{{2-5 项}}
事件锚点：{{3-6 个已发生事件或个人冲击}}
```

## 临时原型输出格式

```text
临时原型名：{{承载物}} + {{自我叙事断裂}}
匹配证据：{{来自用户原话的 2-4 个证据}}
主动作：{{岔路动作}}
系统切换：{{系统 A}} vs {{系统 B}}
主张力轴：{{2-3 个}}
隐藏不甘：{{用户没有直说的交换}}
判词母句：{{承载物}}在你心里，像{{隐喻}}。你反复想起它，指向{{隐秘交换}}。
幻想校正：{{假设结果成立需要哪些条件，也会拿走什么}}
事件冲击候选：{{4-6 个事件锚点}}
可回收资产：{{今天还能回收什么}}
```

## 组合公式

### 1. 承载物

承载物必须具体，优先从用户原话提取：

- 地点：北京、英国、家乡、一线城市。
- 组织：旧公司、大厂、体制、团队、平台。
- 关系：父母、伴侣、同事、客户、导师。
- 身体：睡眠、恢复力、长期压力、健康中断。
- 资产：房子、现金流、身份、签证、作品。
- 机会：AI 窗口、创业念头、转岗入口、海外岗位。

承载物过泛时，回到用户原话找更具体的物件或年份。

### 2. 隐秘交换

用主张力轴写交换：

| 主张力轴 | 交换写法 |
| --- | --- |
| 熟悉环境 vs 陌生竞争 | 把低摩擦和熟人支持，换成外部评价和陌生比较 |
| 安全感 vs 可见度 | 把稳和省力，换成更早被看见和被比较 |
| 国际身份 vs 中文舒适区 | 把熟悉语言和家庭靠近，换成海外独立训练 |
| 作品资产 vs 收藏观察 | 把低风险学习感，换成可展示作品和真实反馈 |
| 家庭托底 vs 个人扩张 | 把家庭距离和情绪底座，换成个人试炼空间 |
| 现金流底盘 vs 高波动机会 | 把可控生活，换成上限试错 |
| 组织背书 vs 独立判断 | 把组织资源和标签，换成独立验证 |
| 身体节奏 vs 快速跃迁 | 把身体持续性，换成高强度机会 |
| 固定身份 vs 新技术身份 | 把熟悉岗位安全，换成新技能定价 |
| 城市锚点 vs 流动弹性 | 把居住底盘和家庭理解，换成迁移弹性 |

### 3. 判词母句

```text
{{承载物}}在你心里，像{{隐喻}}。你反复想起它，是因为你怀疑自己把{{换出的训练/身份/机会}}，换成了{{保住的安全/熟悉/低摩擦}}。今天还能回收的，是{{可回收资产}}。
```

判词必须有：

- 一个具体承载物。
- 一个年份、阶段或生活场景。
- 一个隐秘交换。
- 一个今天能回收的方向。

### 4. 幻想校正

假设结果不可写得太亮。必须补三类条件：

- 现实条件：钱、时间、签证、月供、家人距离、岗位入口、身体恢复。
- 承受条件：孤独、被评价、空窗、睡眠、关系压力、失败暴露。
- 留下条件：作品、反馈、关系、授权、外部定价、持续投入。

### 5. 事件冲击

每个临时原型至少选 4 张深卡：

1. 当年选择点附近的事件。
2. 后来改变成本和回报的事件。
3. 当前重新触发遗憾的事件。
4. 一个个人冲击：家庭、身体、公司、关系、现金流或贵人机会。

每张卡仍按 `event-shock-engine.md` 六段写：事件锚点、生活场景、现实代价、情绪代价、误判点、今天验证。

## 失败信号

- 临时原型名只是主题名，缺少隐藏不甘。
- 只说机会和代价，缺少承载物。
- 判词没有年份、阶段或生活场景。
- 事件冲击卡只写宏观事件，缺少用户体感。
- 今天验证只给建议，缺少找谁、写什么、发给谁、看什么。

## 回到固定原型的条件

如果临时原型连续 3 次在真实用户报告中高频出现，并且都有稳定的承载物、隐藏不甘、事件冲击和可回收资产，才考虑沉淀为 `life-archetype-library.md` 的固定原型。
