# Nano Banana Prompt 指南

## Prompt 结构

每页 prompt = 全局风格前缀 + 页面具体内容

### 全局风格前缀

STYLE = """你是一名顶级PPT设计师。请生成一张16:9比例的专业演示幻灯片图片。

设计风格要求：
- 背景：[具体描述]
- 内容卡片：[颜色 hex + 圆角 + 阴影]
- 主标题：[颜色 hex]，粗体大字
- 强调色：[颜色 hex]
- 正文：[颜色 hex]
- 字体：现代无衬线体
- 所有中文文字必须清晰可读，不能模糊或乱码
- 请只生成一张图片"""

### 页面 Prompt 模板

这是一张[页面类型]。[背景和卡片简述]。

标题（[颜色 hex]，大号粗体）：[精确文字]

[区域描述]：
- [具体内容]

[底部]：
- [锚点/数据/注脚]

## 9 条铁律

### 1. 文案原文复制
从 content.md 复制，一字不改。

### 2. 颜色用 hex
`深墨绿色(#1f4a3c)` 不是 `深绿色`

### 3. UI mockup 详细描述
不要"一个聊天截图"。要描述每个气泡、头像、时间戳。

### 4. 中文名强制指令
**CRITICAL**: 名字必须是「[中文名]」。不能用英文。注意用中文引号「」不用"".

### 5. 表格逐行列出
不说"7 行表格"。写出每一行的每一列。

### 6. 避免 prompt 泄漏
第一句不写样式指令（会被渲染为标题）。

- ❌ `[标题关键词]。[背景样式]。`
- ✅ `这是一张关于[主题]的页面。`

### 7. 人物一律火柴人（NEW · 强制）
Nano Banana 画写实脸部 80% 翻车（崩坏的五官、AI 油画感）。所有人物强制用火柴人：
- 头：深色填充圆 + 两点眼睛 + 一条弯嘴（表达情绪）
- 身体：简几何（梯形/矩形）
- 四肢：粗线 3-4px 描边
- 服饰：实色色块填充（暖橙/薄荷绿/米白）

Prompt 写法：
```
一个火柴人风格的[角色]：
- 头部：深蓝圆形 + 两点眼睛 + 一条弯嘴（表达[情绪]）
- 身体：简梯形 + 实色[颜色]填充
- 粗线 3-4px 深蓝描边
- 风格：Stripe illustration / Notion illustration
```

### 8. AI Slop 黑名单（NEW · 强制禁绘）
以下元素必须在 prompt 明确禁止，否则 AI 会自动塞进去：
- **商务俗套**：商务握手、西装领带人物、会议桌大特写
- **科技俗套**：齿轮+火箭组合、闪电图标、水晶球、无限符号
- **AI 油画感**：彩虹渐变、光效粒子、3D 阴影、金属反光
- **装饰垃圾**：花朵/蝴蝶/星星/烟花点缀、动物吉祥物
- **中式滥用**：龙/凤/祥云（除非内容明确需要）、印章滥用

Prompt 结尾必须加一段：
```
【禁绘】
- 禁止：商务握手、齿轮火箭、水晶球、闪电、3D 阴影、渐变光效、写实照片、彩虹、花朵装饰、动物吉祥物、AI slop
```

### 9. 明确 reference 品牌（NEW · 提升成功率 10×）
泛泛说"极简现代"无效。要明指 reference：
- 扁平插画：`Stripe illustration` / `Notion illustration` / `Dropbox illustration` / `Linear design`
- 信息图：`The Economist infographic` / `FT graphic` / `McKinsey report`
- 极简：`Dieter Rams` / `Muji` / `Apple Keynote`
- 中式古典：`宋代文人画` / `宣纸书法` / `荣宝斋文创`（避免"中国风"这种模糊词）

## Visual Strategy Cheatsheet（NEW）

**A+B 混合策略**（推荐 80% 商务 deck）：
- A 信息图：工具页 / 矩阵 / 流程 / 数据表（图文比 60/40）
- B 扁平插画：章节扉页 / 概念页 / 隐喻页（图文比 70/30）
- 一份 deck 内混用，不纯用其中一种

**分层派文字密度**（避免"每页都堆满"或"每页都空"）：
- 扉页/金句/过渡：≤ 15 字
- 工具页/信息图：≤ 60 字正文（表格标注独立计）
- 封面：可稍放宽到 30 字

**色彩扩展建议**：
- 纯商务（navy + accent）扁平插画容易"失色"
- 一定要加第 3 色（薄荷绿 / 淡杏 / 灰粉）给插画做服饰/辅助元素
- 3 色是插画生存线，纯 2 色画不出生动场景

## 中文渲染 Fallback

重跑 2 次仍乱码 → HTML + Playwright 单独生成该页：
1. 写 HTML（Tailwind CSS + 系统中文字体）
2. Playwright 截图 1920×1080
3. 替换该页 JPG

## 参考链模式

生成 Slide N 时传入 Slide N-1 的 JPG 作为参考，提升风格一致性。
首次全量生成可不用，QA 修复单页时建议启用。
