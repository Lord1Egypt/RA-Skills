## Description: <br>
Use when facing 2+ independent tasks that can be worked on without shared state or sequential dependencies. <br>

This skill is ready for commercial/non-commercial use. <br>

## Use Case: <br>
Developers and agent operators use this skill to split independent coding or debugging work into focused parallel subagent tasks, then review and integrate the returned results. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Parallel subagents can produce conflicting or incorrect recommendations if tasks are not truly independent. <br>
Mitigation: Use the skill only for clearly independent work, provide each subagent minimal necessary context, and review all returned changes before integrating them. <br>
