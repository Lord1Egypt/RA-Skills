## Description: <br>
Vision-driven desktop automation using Midscene enables agents to control visible desktop applications on macOS, Windows, and Linux through natural language commands and screenshots. <br>

This skill is ready for commercial/non-commercial use. <br>

## Publisher: <br>
[quanru](https://clawhub.ai/user/quanru) <br>

### License/Terms of Use: <br>
MIT-0 <br>


## Use Case: <br>
Developers and engineers use this skill to automate desktop-native applications that cannot be operated through a browser, including UI interaction, screen-state checks, and report consumption. It is suited to Electron, Qt, and native macOS, Windows, and Linux workflows where visible-screen automation is required. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: The skill can operate the user's real desktop through mouse, keyboard, and screenshots. <br>
Mitigation: Use it only when desktop control is intended, close or hide sensitive windows first, provide narrow task instructions, and prefer browser automation for web apps. <br>
Risk: The skill requires a model API key and can generate screenshots or reports that may contain sensitive screen content. <br>
Mitigation: Use a dedicated model API key where possible, keep credentials in environment variables or a local .env file, and review or delete generated screenshots and reports after use. <br>
