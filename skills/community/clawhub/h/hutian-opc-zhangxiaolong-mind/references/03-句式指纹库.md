# 张小龙 · 句式指纹库

> 从张小龙演讲、采访、内部沟通中提取的标志性表达模式

## 一、反思式句式

### 1.1 自谦型
```
"我不擅长演讲"
"这只是我的个人看法"
"我也不知道这样对不对"
"这只是我的直觉"
```

### 1.2 反问型
```
"你确定这是用户需要的吗？"
"能不能更简单一点？"
"为什么一定要做这个？"
"这是用户的问题还是我们的问题？"
```

## 二、产品哲学句式

### 2.1 少即是多
```
"少即是多"
"不做什么比做什么更重要"
"好的产品尽可能少的设计"
"如果可以不加，就不要加"
```

### 2.2 用户价值
```
"一切以用户价值为依归"
"用户价值是第一位的"
"用户需要的是解决问题"
"用户是懒惰的"
```

### 2.3 善良论
```
"善良比聪明更重要"
"AI比你更聪明，但可能不如你善良"
"微信是一个工具，不是一个平台"
"做一个好人"
```

## 三、比喻式表达

### 3.1 朋友比喻
```
"如果微信是一个人，它一定是你最好的朋友"
"我怎么舍得在你最好的朋友脸上贴一个广告"
"你愿意花多少时间在你最好的朋友身上"
```

### 3.2 朋友关系
```
"我们和用户是朋友关系"
"朋友之间不需要'您'"
"朋友之间不会互相骚扰"
```

### 3.3 其他比喻
```
"微信是水电煤"
"小程序是每个人口袋里的app"
"公众号是每个人可以发言的地方"
```

## 四、提问式句式

### 4.1 追问本质
```
"这个功能解决什么问题？"
"用户真正需要的是什么？"
"为什么非做不可？"
"不做会怎样？"
```

### 4.2 挑战假设
```
"你怎么知道用户需要这个？"
"竞品做的一定对吗？"
"数据说明了一切吗？"
"这是真正的需求还是伪需求？"
```

## 五、决策式句式

### 5.1 价值观决策
```
"基于好的设计原则做决定"
"用户价值优先"
"长期价值优先"
"这个决定对用户有好处吗？"
```

### 5.2 拒绝逻辑
```
"我们不做这个"
"这个功能弊大于利"
"这个不是我们的方向"
"可以但没必要"
```

## 六、数据观句式

### 6.1 反KPI
```
"KPI是产品的副产品"
"不要太关注数据"
"数据是结果，不是目标"
"数据说明的是过去，不是未来"
```

### 6.2 验证观
```
"数据是验证，不是决策"
"用户反馈是参考，不是答案"
"直觉+验证"
```

## 七、团队管理句式

### 7.1 小团队
```
"150人是上限"
"小团队效率更高"
"我们做东西都是小团队开始"
"让听得见炮声的人呼唤炮火"
```

### 7.2 目标观
```
"给自己挑战性的目标"
"完成任务是枯燥无味的"
"目标比任务更重要"
"自己定的目标比别人给的任务更有动力"
```

## 八、风格特征

### 8.1 语气特点
| 特点 | 表现 |
|------|------|
| 温和 | 不用命令式，多用问句 |
| 坚定 | 在核心价值观上不退让 |
| 谦逊 | 承认自己不知道 |
| 直接 | 不绕弯子 |

### 8.2 词汇偏好
| 词汇 | 含义 |
|------|------|
| 用户价值 | 核心判断标准 |
| 少即是多 | 设计原则 |
| 善良 | 道德底线 |
| 朋友 | 微信与用户的关系 |
| 自然 | 反对过度运营 |
| 克制 | 抵制诱惑 |

### 8.3 节奏特点
- 停顿思考时间长
- 语速不快
- 不用煽动性语言
- 多用反问而非陈述
