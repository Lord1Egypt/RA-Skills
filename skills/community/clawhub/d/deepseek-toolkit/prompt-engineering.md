# DeepSeek Prompt 工程技巧

> DeepSeek 模型有自己的特性，针对性地优化 Prompt 效果会显著不同。

## 核心差异：DeepSeek vs OpenAI vs Claude

| 特性 | DeepSeek | OpenAI (GPT-4o) | Claude |
|------|---------|-----------------|--------|
| 中文 System Prompt | ✅ 效果更好 | 中性 | 中性 |
| System Prompt 敏感度 | 高（微调效果明显） | 中 | 高 |
| JSON Mode | ✅ 原生支持 | ✅ | ✅ |
| 思考链 (CoT) | R1/Reasonix 内置 | o1/o3 内置 | 需手动提示 |
| 长文档处理 | V4 Pro 128K | 128K | 200K |
| 代码能力 | V4 Pro 极强 | 极强 | 极强 |
| 创意写作 | V3 偏理性 | 均衡 | 偏细腻 |
| 指令遵循 | 高 | 极高 | 极高 |

## 一、System Prompt 优化

### 原则：中文场景用中文

```python
# ❌ 中文任务用英文 system prompt
system = "You are a helpful AI assistant."

# ✅ 中文任务用中文 system prompt（DeepSeek 对中文 system 更敏感）
system = "你是一个专业的AI助手，回答准确、简洁、有逻辑。"
```

### 原则：角色要具体

```python
# ❌ 模糊角色
system = "你是一个编程助手。"

# ✅ 具体角色 + 能力边界
system = """你是一个资深 Java 后端开发专家，专长：
- Spring Boot / Spring Cloud 微服务架构
- MySQL 性能调优和索引优化
- 大数据量下的系统设计
回答时优先给出代码示例和最佳实践。如果问题超出 Java 后端范围，明确说明。"""
```

### 原则：输出格式在 system 中定义

```python
system = """你是一个代码审查助手。

输出格式：
## 问题等级：🔴严重 / 🟡警告 / 🟢建议
## 问题描述
[1-2句话]
## 修复建议
[代码示例]
## 影响范围
[说明]

每次审查输出 3-5 个问题，按严重程度排序。"""
```

## 二、结构化 Prompt 模板

### 模板 1：任务分解型

```
# 任务
[清晰描述要做什么]

# 背景
[为什么做这个，提供上下文]

# 约束
- [约束条件1]
- [约束条件2]

# 输出格式
[期望的输出结构]

# 示例
[1-2个输入输出示例]
```

### 模板 2：Few-Shot 学习型

```
根据以下示例，完成新任务：

示例1：
输入：[...]
输出：[...]

示例2：
输入：[...]
输出：[...]

新任务：
输入：[...]
输出：
```

### 模板 3：链式思考型（V3/V4 Pro 使用）

```
请一步步思考以下问题：

1. 首先，[分析问题的第一步]
2. 然后，[第二步]
3. 接着，[第三步]
4. 最后，[得出结论]

问题：[你的问题]
```

> 注意：R1 和 Reasonix 内置了推理能力，不需要手动加 CoT 提示，直接问即可。

### 模板 4：JSON 结构化输出

```python
prompt = f"""从以下文本中提取信息，以 JSON 格式输出。

文本：{text}

输出格式：
{{
    "人物": ["名字1", "名字2"],
    "地点": ["地点1"],
    "时间": "具体时间",
    "事件": "一句话概括",
    "情绪": "正面/负面/中性"
}}

只输出 JSON，不要其他内容。"""

# 配合 response_format 参数效果更好
response = client.chat.completions.create(
    model="deepseek-chat",
    messages=[{"role": "user", "content": prompt}],
    response_format={"type": "json_object"}
)
```

## 三、DeepSeek 特有技巧

### 技巧 1：利用 V4 Pro 的代码能力

```python
# V4 Pro 对「先分析再编码」的提示响应极好
prompt = """分析以下需求，然后实现：

1. 需求分析（列出要点）
2. 技术方案（选择最合适的方案）
3. 代码实现（带注释）
4. 测试用例（至少3个）

需求：实现一个 LRU 缓存，支持 get/put 操作，O(1) 时间复杂度。"""
```

### 技巧 2：R1/Reasonix 不需要 CoT

```python
# ❌ 对 R1 使用 CoT 提示（多余，浪费 token）
prompt = "请一步一步思考：如果 x² + 5x + 6 = 0，求 x。"

# ✅ 直接问（R1 会自动推理）
prompt = "解方程 x² + 5x + 6 = 0"
```

### 技巧 3：中文翻译优化

```python
# DeepSeek 的中文翻译能力很强，但需要给上下文
prompt = """这是一篇关于 AI Agent 的技术博客，请翻译为中文。

术语对照：
- Agent → 智能体
- Prompt → 提示词
- Fine-tuning → 微调
- Token → Token（不翻译）

翻译要求：保持技术准确性，语言自然流畅。"""
```

### 技巧 4：长文档处理

```python
# V4 Pro 支持 128K 上下文，处理长文档时：
prompt = """以下是完整文档，请完成指定任务。

## 任务
[具体任务]

## 文档内容
{long_document}

## 注意
- 基于文档内容回答，不要编造
- 引用原文时标注章节号
- 如果文档中没有相关信息，明确说明"""
```

## 四、高级 Prompt 模式

### 模式 A：自我校验

```python
prompt = f"""完成以下任务，然后检查你的答案：

任务：{task}

请按以下格式输出：
1. 初步答案：[你的回答]
2. 检查过程：[验证答案是否正确]
3. 修正（如果需要）：[修正后的答案]
4. 最终答案：[确认后的回答]"""
```

### 模式 B：对比分析

```python
prompt = f"""对比分析以下两个方案：

方案A：{option_a}
方案B：{option_b}

分析维度：
| 维度 | 方案A | 方案B |
|------|-------|-------|
| 成本 | ... | ... |
| 性能 | ... | ... |
| 可维护性 | ... | ... |
| 推荐场景 | ... | ... |

结论：[你的推荐]"""
```

### 模式 C：渐进式优化

```python
# 第一轮：生成初版
v1 = call_api("写一个 Python 函数计算斐波那契数列")

# 第二轮：优化
v2 = call_api(f"""优化以下代码：
{v1}

优化方向：
1. 性能（时间复杂度）
2. 可读性
3. 边界处理
输出优化后的代码和说明。""")

# 第三轮：最终打磨
v3 = call_api(f"""对以下代码做最终审查：
{v2}

检查：性能、正确性、Python 风格、文档字符串。
输出最终版本。""")
```

## 五、常见问题与解决方案

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 输出太啰嗦 | 未限制长度 | 加 `max_tokens` + Prompt 中说"简洁回答" |
| 格式不对 | 指令不够明确 | 给出明确的输出格式模板 |
| 中英文混杂 | 未指定语言 | system prompt 明确"用中文回答" |
| 代码不完整 | 截断 | 增加 `max_tokens`，或要求"先说思路再写代码" |
| 幻觉/编造 | 缺乏约束 | 加"仅基于提供的信息回答，不要编造" |
| 跟不上指令 | 指令太复杂 | 拆分为多步，逐步执行 |
| 数学错误 | 用了非推理模型 | 数学/推理任务用 R1 或 Reasonix |
