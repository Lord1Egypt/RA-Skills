# 常见功能模式参考

编写功能型 PRD 时，直接复用以下标准模式的描述框架。

---

## 1. CRUD 模式（增删改查）

**适用场景：** 数据管理、列表维护、配置管理

**PRD 必写要点：**
- 列表展示字段、排序方式、分页规则
- 新增字段（哪些必填、默认值）
- 编辑（哪些字段可改、哪些不可改）
- 删除（软删除还是硬删除、关联数据如何处理）
- 搜索/筛选条件

**字段描述模板：**
```
| 字段 | 类型 | 新增必填 | 编辑可改 | 列表展示 | 校验规则 |
```

---

## 2. 表单提交模式

**适用场景：** 信息录入、设置配置、申请审批

**PRD 必写要点：**
- 每个字段的类型、校验规则、错误提示
- 提交前的前端校验 vs 后端校验
- 提交成功后的跳转/反馈
- 重复提交防护
- 草稿保存（是否支持）

**校验规则模板：**
```
| 规则 | 校验时机 | 正则/条件 | 错误提示 |
| 前端 | 失焦/提交 | | |
| 后端 | 提交 | | |
```

---

## 3. 列表/表格模式

**适用场景：** 数据展示、用户管理、订单列表

**PRD 必写要点：**
- 列字段（哪些展示、宽度、对齐方式）
- 排序（默认排序、可排序字段）
- 筛选（筛选条件、联动关系）
- 分页（每页条数、总数展示）
- 空状态（无数据时展示什么）
- 批量操作（是否支持、最大批量数）
- 行操作（每行的操作按钮）

---

## 4. 状态机模式

**适用场景：** 订单流程、审批流程、任务流转

**PRD 必写要点：**
- 所有状态枚举
- 每个状态可执行的操作
- 状态流转条件（谁触发、什么条件）
- 每个状态对应的展示/行为差异
- 逆向流转（是否允许回退）

**状态机描述模板：**
```
状态枚举：
  S1 - 初始态
  S2 - 处理中
  S3 - 已完成
  S4 - 已取消

流转规则：
  S1 →(用户提交)→ S2
  S2 →(系统处理成功)→ S3
  S2 →(用户取消)→ S4
  S2 →(系统处理失败)→ S1  (是否允许？)
```

---

## 5. 权限控制模式

**适用场景：** 角色管理、功能权限、数据权限

**PRD 必写要点：**
- 角色定义（有哪些角色）
- 功能权限（谁能看什么、谁能操作什么）
- 数据权限（能看到哪些数据范围）
- 越权处理（无权限时的行为）
- 权限变更的生效时机

**权限矩阵模板：**
```
| 功能/角色 | 角色A | 角色B | 角色C |
| 查看列表  | ✅    | ✅    | ❌    |
| 新增      | ✅    | ❌    | ❌    |
| 删除      | ✅    | ❌    | ❌    |
```

---

## 6. 搜索/筛选模式

**适用场景：** 列表筛选、全文搜索、条件查询

**PRD 必写要点：**
- 搜索方式（精确/模糊/全文）
- 筛选条件（单选/多选/范围/日期）
- 搜索范围（搜哪些字段）
- 排序规则（相关性/时间/自定义）
- 空结果处理（无匹配时展示什么）
- 搜索历史（是否保存、清除规则）

---

## 7. 导入/导出模式

**适用场景：** 批量数据操作、报表下载

**PRD 必写要点：**
- 导入：
  - 文件格式（xlsx/csv）、大小限制
  - 模板下载
  - 字段映射规则
  - 校验失败处理（跳过/中断/标记错误行）
  - 导入结果反馈（成功N条/失败N条/失败原因）
- 导出：
  - 导出字段（与列表是否一致）
  - 文件格式
  - 大数据量处理（异步下载/分页导出）
  - 导出权限

---

## 8. 通知/消息模式

**适用场景：** 站内信、邮件通知、推送提醒

**PRD 必写要点：**
- 触发条件（什么事件触发通知）
- 通知渠道（站内/邮件/短信/Push）
- 通知内容（模板、变量）
- 接收人规则（固定人员/动态计算）
- 去重规则（同一天不重复/同类型不重复）
- 用户可配置项（是否可关闭某类通知）

---

## 9. 定时任务模式

**适用场景：** 定时同步、定期检查、定时清理

**PRD 必写要点：**
- 执行频率（多久执行一次）
- 触发条件（是否需要满足前置条件）
- 执行逻辑（具体做什么）
- 执行结果（成功/失败如何记录）
- 失败重试（是否重试、重试次数、重试间隔）
- 并发控制（上一次未完成时如何处理）
- 手动触发（是否支持）

---

## 功能型 PRD 质量检查清单

- [ ] 每个功能点都有明确的输入/输出/行为描述
- [ ] 所有字段都标注了类型和校验规则
- [ ] 核心流程画了完整的步骤（含分支）
- [ ] 异常/边界情况有明确的处理方式
- [ ] 给出了具体的数值（不是"适当"、"合理"）
- [ ] 状态流转画了完整的状态机
- [ ] 权限控制有明确的矩阵
- [ ] 接口定义了请求/响应/错误码
- [ ] 明确列出了本期不做的内容
