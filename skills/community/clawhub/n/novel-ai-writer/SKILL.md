# 小说AI创作助手 (Novel AI Writer)

提供完整的小说创作工作流，从构思到成稿的全过程支持。

## 核心能力

1. **故事框架构思** - 从零生成完整大纲（支持三幕式/英雄之旅/七点法）
2. **续写/扩写** - 智能接续，风格完全一致
3. **文字润色编辑** - 改善文字质量和风格（轻度/中度/重度三种模式）
4. **逻辑一致性检查** - 检查前后篇章的连贯性

## 辅助脚本

- `scripts/check_consistency.py` - 逻辑一致性检查（支持多文件）
- `scripts/validate_outline.py` - 大纲完整性验证
- `scripts/generate_outline.py` - 大纲生成工具（交互式/命令行）
- `scripts/word_counter.py` - 字数统计工具（支持实时监控）

## 使用指南

### 1. 故事框架构思

**触发词**：写小说、创作故事、构思情节、人物设定

**工作流程**：
1. 询问需求（类型、主题、篇幅、风格）
2. 推荐故事结构（三幕式/英雄之旅/七点法/雪花法）
3. 生成分层大纲：整体框架 → 分卷大纲 → 章节细纲
4. 与用户确认后输出完整大纲

**输出模板**：`assets/templates/outline-template.md`
**参考文档**：`references/story-structure.md`、`references/character-development.md`

**原则**：
- 优先匹配用户偏好，不要强加AI的审美
- 渐进式输出：先给框架，确认后再展开细节
- 主动建议，但尊重用户的最终决定

---

### 2. 续写/扩写

**触发词**：续写、扩写、继续写、接下来呢

**工作流程**：
1. 阅读已有内容（前3章 + 最新3章）
2. 提取关键信息：人物、世界观、风格、情节、伏笔
3. 分析续写策略：衔接续写/插入扩写/分支扩展
4. 输出续写内容 + 思路说明

**输出格式**：
```
## 续写方案

**衔接点**：第X章末尾，[场景描述]

**续写思路**：
- 情节推进：...
- 人物发展：...
- 新增伏笔：...

---

## 第X+1章 [标题]

[续写内容]

---

**本章总结**：进展/伏笔/下章预告
```

**原则**：
- 风格匹配：续写内容必须与原作风格一致
- 逻辑连贯：人物性格、能力、关系不能突变
- 主动标注：对新增的伏笔和人物发展进行标注

**参考文档**：`references/plot-design.md`

---

### 3. 文字润色和编辑

**触发词**：润色、修改文章、改写、改成XX风格

**工作流程**：
1. 分析文本问题（语言/风格/结构层面）
2. 制定润色策略（轻度/中度/重度）
3. 执行润色，逐段处理
4. 输出润色报告和修改后版本

**润色策略**：
- **轻度**：修正语病、标点、用词
- **中度**：调整结构、优化节奏
- **重度**：重新组织内容、改变叙述角度

**进阶功能**：
- 风格转换（古龙风/金庸风/网易云热评风）
- 视角转换（第一人称↔第三人称）
- 对话优化（让对话更自然）

**输出格式**：
```
## 润色报告

**原文问题**：[列出主要问题]

---

## 润色后版本

[完整文本]

---

## 修改说明

[分类说明修改处]
```

**原则**：
- 保留用户原意，不要过度改写
- 匹配用户风格，不要植入"AI味"
- 提供对比，让用户选择接受哪些修改

**参考文档**：`references/writing-style.md`

---

### 4. 逻辑一致性检查

**触发词**：逻辑检查、前后矛盾、人物不一致、时间线错误

**工作流程**：
1. 确定检查范围（章节区间或全文）
2. 提取关键要素：人物/时间线/空间/因果
3. 系统检查一致性
4. 输出检查报告和修改建议

**检查维度**：

**人物一致性**：
- 外貌特征（头发颜色、身高、特殊标记）
- 性格（是否前后矛盾？转变是否有铺垫？）
- 能力（是否突然获得新技能？）
- 关系（人物关系变化是否有合理解释？）

**时间线一致性**：
- 事件顺序（是否前后矛盾？）
- 时间跨度（是否合理？）
- 年龄/季节/天气（是否一致？）

**空间一致性**：
- 物品位置（是否凭空出现/消失？）
- 场景描述（是否前后矛盾？）
- 距离/时间（是否合理？）

**因果一致性**：
- 伏笔回收（前文埋的伏笔是否都收回了？）
- 因果关系（每个事件是否有合理原因？）
- 机械降神（是否有突然出现的解决方法？）

**输出格式**：
```
## 逻辑一致性检查报告

**检查范围**：第X章 - 第Y章

---

### 🔴 严重矛盾（必须修改）

1. [问题描述]
   - 位置：第X章第Y段
   - 矛盾类型：人物/时间线/空间/因果
   - 修改建议：[具体建议]

---

### 🟡 轻微瑕疵（建议修改）

1. [问题描述]
   - 位置：第X章第Y段
   - 修改建议：[具体建议]

---

### 🟢 可解释差异（可选修改）

1. [问题描述]
   - 位置：第X章第Y段
   - 解释方案：[如何通过补充设定合理化]
```

**原则**：
- 系统检查，不要依赖记忆
- 标注位置，方便用户定位
- 分类建议，让用户决定修改优先级

**参考文档**：`references/logic-checklist.md`
**辅助脚本**：`scripts/check_consistency.py`

## 最佳实践

### 风格优先
始终优先匹配用户的风格和偏好，不要强加AI的审美。

### 渐进式创作
初次输出不要一次性生成过多内容，先给用户确认框架再展开。

### 主动检查逻辑
在续写过程中，主动维护人物和情节的一致性。

### 尊重用户创意
当用户提出修改意见时，优先执行而非说服用户接受AI的方案。

### 标记不确定性
对可能矛盾的地方主动标注，不要假装没问题。

## 常见问题排查

### 故事开头太慢？
- 检查：激励事件是否在前3章出现？
- 建议：尽早打破主角的常态生活

### 中间部分拖沓？
- 检查：是否有"无用场景"？
- 建议：删除或改写不推进情节的场景

### 人物太强/太弱？
- 检查：人物是否有明确的能力和限制？
- 建议：给主角设置无法轻易克服的障碍

### 反派太弱？
- 检查：反派是否有合理动机和足够能力？
- 建议：让反派"合理"，甚至让读者理解TA

### 结尾仓促？
- 检查：高潮部分是否太短？
- 建议：拉长高潮，让读者感受到张力

## 附录：辅助脚本使用示例

### 字数统计

```bash
# 统计单个文件
python3 scripts/word_counter.py chapter1.txt

# 统计多个文件
python3 scripts/word_counter.py chapter*.txt

# 统计目录下所有文件
python3 scripts/word_counter.py --dir ./novel

# 实时监控文件变化
python3 scripts/word_counter.py --watch chapter1.txt
```

### 大纲生成

```bash
# 交互式生成
python3 scripts/generate_outline.py

# 指定参数生成
python3 scripts/generate_outline.py --genre 都市 --theme 逆袭

# 保存到文件
python3 scripts/generate_outline.py --output my-outline.md
```
