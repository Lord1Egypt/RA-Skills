# 反幻觉核查手册

面向AI Agent质检场景的幻觉识别与防范实战指南。

## 一、幻觉的六大类型

| 类型 | 表现 | 高发场景 | 检测方法 |
|------|------|---------|---------|
| **假引用** | 编造不存在的论文/作者/DOI | 学术写作、文献综述 | 逐一核验DOI/标题 |
| **假数据** | 凭空编造统计数据、百分比 | 论文结果、分析报告 | 要求提供原始数据来源 |
| **逻辑幻觉** | 结论与论据不匹配 | 论证过程 | 逐段检查论点→论据对应关系 |
| **一致性幻觉** | 前后矛盾（第3章说A，第5章说非A） | 长文本 | 全文一致性扫描 |
| **格式幻觉** | 三线表画错、公式编号跳号 | 学术论文 | 按规范逐项检查 |
| **过度自信** | 对不确定的事给出笃定回答 | 普通问答 | 要求标注置信度 |

## 二、假文献检测（最关键）

### 2.1 识别信号

假文献的高频特征：
- **DOI不存在**或格式错误（10.xxxx/xxxxx应为固定格式）
- **期刊名模糊**（如"中国某某研究"而非完整刊名）
- **作者名异常**（张XX、李XX这类泛化名字）
- **年份不合理**（2026年引用了"2028年"的论文）
- **标题太完美**（与论文主题过于契合，像定制）
- **引用格式不统一**（同一篇论文里混用APA/GB/T格式）

### 2.2 核验流程

```
对每篇参考文献：
1. 提取DOI → search_semantic_paper_match按标题查
2. 无DOI → 按作者+年份+标题关键词查
3. Google Scholar交叉验证
4. 确认：作者/年份/标题/期刊全部匹配
```

### 2.3 我们踩过的坑

- 智慧医院论文(26/27号材料)参考文献4-10全是假文献
- 论文1初稿中"访谈32人"与第五章"模拟数据"前后矛盾
- Devil审稿时自己也幻觉，把不存在的期刊当成真的

## 三、假数据检测

### 3.1 识别信号

- **数据过于圆整**（恰好50%、正好100万）
- **数据过于完美**（所有指标都在预期范围内）
- **无法溯源**（无法提供原始数据文件或出处）
- **前后不一致**（摘要说N=171，方法部分说N=200）
- **统计结果不合理**（p值全是<0.001，没有不显著的结果）

### 3.2 核验流程

```
1. 所有数据必须有标注来源（真实数据/公开数据/假设数据）
2. 交叉验证：同一数据在不同章节出现时必须一致
3. 抽查法：随机选3-5个关键数据点，要求agent提供原始出处
4. 总量校验：各分项相加必须等于总和
```

## 四、LLM幻觉的学术发现（2024-2026）

### 4.1 核心结论

来自UniFact (2025)、HalluMat (2025)、Bi'an (2025)等最新研究：

1. **幻觉检测和事实验证是互补的**——两个范式结合起来效果最好（UniFact）
2. **混合方法优于单一方法**——检索+生成+NLI的多阶段验证比纯LLM判断更可靠（HalluMat降低30%幻觉率）
3. **自一致性检查有效但不够**——让模型多次回答取一致结果能抓到部分幻觉，但有上限
4. **小模型微调可超越大模型**——Bi'an的14B微调模型超过70B基座模型
5. **幻觉频率与查询复杂度正相关**——高熵查询更容易产生幻觉

### 4.2 对我们的启示

- L4双审（编辑部主任+学术副主编）= 混合方法，符合学术结论
- 质检agent用不同模型交叉 = 避免单一模型的系统性幻觉
- 对高复杂度任务（如论文创新性论证）应提高审查标准

## 五、实战核查清单

### 论文核查（逐项）

- [ ] 所有参考文献DOI可验证
- [ ] 假文献（张XX/李XX/泛化名）已清除
- [ ] 数据来源已标注（真实/模拟/公开）
- [ ] 前后数据一致性（N值、比例、统计结果）
- [ ] 摘要与正文一致（结论、样本量、方法）
- [ ] 论点与论据对应（每段核心主张有支撑）
- [ ] 三线表格式正确（无竖线、表头在上方）
- [ ] 引用格式统一（全文同一种格式）
- [ ] 基金项目编号为真实编号或占位符已标注
- [ ] 作者信息完整（姓名/职称/单位/联系方式）

### 公文核查

- [ ] 无乱码/占位符/XXX未填
- [ ] 文号格式正确
- [ ] 引用的政策文件名称准确
- [ ] 数据与原始来源一致
- [ ] 称谓/职务正确（如"副主任"不是"主任"）

### 报告核查

- [ ] 数据有来源标注
- [ ] 图表数据与正文一致
- [ ] 结论有数据支撑
- [ ] 预测有依据说明

## 六、防御性写作规则

1. **不确定就标注**：用"待验证""初步分析"而非笃定语气
2. **数据必须溯源**：每个数字都要能追溯到原始文件
3. **引用必须真实**：宁可少引用也不编造
4. **交叉引用要一致**：同一信息在文中多次出现必须相同
5. **让两个agent分别写，再交叉审**：比一个人写一个人审更可靠
