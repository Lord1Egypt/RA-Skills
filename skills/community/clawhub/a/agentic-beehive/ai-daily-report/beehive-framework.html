三层架构
L1 范式层 🧠 任务交付 vs 状态维持
📦 任务交付型
CrewAI / AutoGen 范式。核心抽象是任务管道，执行器生命周期=任务生命周期。做完就结束，平时没有活的执行器。
🔄 状态维持型 ✦ 本系统
LangGraph / 蜂巢范式。核心抽象是状态机+事件触发器。执行器常驻但 idle，有事件才唤醒。状态机存在是为了 trigger task，不是为了 close task。
🔑 核心差异
流水线思维 vs 反应式思维。蜂巢不停运行——市场不停，系统不停；无信号时 idle，有事件时唤醒。
L2 生态层 🌿 蜂巢与外在群落
🏠 蜂巢核心
Operator（红小臣）是态势感知入口，掌握工具架构，按需调用。白板唯一管理者。
🌼 油菜花群落
数据源。AI 市场日报本身就是油菜花群落——从大模型花园中采蜜。另有欧洲电力价格群落。
🌿 麦卢卡树丛
专业知识。法规、标准、行业报告。需要深度解读时采蜜。
🌊 河流
流式信息。新闻、监控、告警。持续流过，蜂巢实时过滤。
🐝 另一个蜂巢
AutoGen Studio、LangGraph Studio。不是"另一个工具"，是整片信息生态中的协作蜂巢。
L3 分支层 🔀 四大分支
💻 coding
契约驱动开发（PACT），Claude Code + subagent 协作。任务交付型。
🎵 composing
DeepSeek→MiniMax 双模型管线。思想空间→内容实现。任务交付型。
👁️ supervising
LangGraph 编排模式。Supervisor/Debate/Event-Driven。状态维持型。
📈 量化交易 ✦ 本系统
事件驱动交易系统。多 Agent 辩论+风控审批。AI 日报是其数据感知层。

酿蜜公式
性价比分
CE = capability² × log₁₀(1000 / price)
能力²做乘数——弱模型即使免费也不排第一。对数缩放避免超低价爆炸。
¥0.1/M → CE × 4.0 ｜ ¥1/M → × 3.0
¥10/M → × 2.0 ｜ ¥100/M → × 1.0
综合排名
Score = 0.35×Cap + 0.30×CE + 0.20×Eco + 0.15×Mom
Cap = 能力分（MMLU/C-Eval）
CE = 性价比分（归一化）
Eco = 生态分（API/文档/社区）
Mom = 势头分（近期增长/热度变化）
权重可演化：早期重能力，成熟期重性价比
能力门槛：综合排名要求 capability ≥ 0.70
Bloom 衰减
bloom(n+1) = bloom(n) - 0.05 + signal × 0.3
每日自然衰减 0.05（无新信号时）。
信号越强，增幅越大。
新发现厂商初始加成 +0.15。
