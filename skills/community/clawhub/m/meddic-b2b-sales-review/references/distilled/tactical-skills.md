# 战术技能（Tactical Skills）

## 用途
从销售 VP 的 review 会议中提取的实战战术技能，包括信息展示技巧、对话控制方法、工具使用和风险控制。这些不是理论框架，而是"怎么在实际操作中做得更好"的具体方法。

---

## 技能一：中性对比表技术

### 什么是中性对比表
一张看起来客观、中立的功能对比表，帮助客户理解不同方案的差异。重点不是"我们最好"，而是"我们最适合你"。

### 设计原则
1. **列维度要选对我们有利的**：不是所有维度都列，选我们有优势且客户关心的
2. **对手的信息要准确**：不要瞎编对手的短板，客户会验证
3. **用符号而非文字**：✅/❌/⚠️ 比"好/差/一般"更有冲击力
4. **留一个"待确认"列**：对不确定的信息标注"需与厂商确认"，比直接写"不支持"更安全
5. **最后一列是"你最看重的"**：让客户自己标注权重，引导客户按对我们有利的维度思考

### 使用时机
- 客户在多家比选时
- 需要帮客户建立评估框架时
- 客户的决策标准还不清晰时

### 注意事项
- 不要贬低对手，只展示事实
- 表格要看起来中性，不能一眼看出是我们的销售材料
- 如果客户问"这个表是谁做的"，诚实回答

## 技能二：场判断对话脚本

### 开场阶段
> "在我们深入讨论之前，我想先了解一下你们的背景，这样我能给更有针对性的建议。"

### 发起方判断
> "这个项目最初是怎么启动的？是某个业务问题触发的，还是整体规划的一部分？"

### 竞争格局
> "你们目前在看哪些方案？我们好有针对性地展示我们的差异化。"

### 采购动机
> "你们评估这些方案，最看重的是什么？是解决具体问题，还是做整体升级？"

### 预算
> "预算大概什么范围？我们好推荐最合适的配置。"

### 时间压力
> "有没有时间节点？比如什么时候要上线？"

### 关键原则
- 不要一次问完，穿插在自然对话中
- 每个问题后根据回答追问
- 记录关键信息，后续方案中体现

## 技能三：把 LLM 当同事用

### 适用场景
- 需要快速生成方案初稿
- 需要分析客户资料
- 需要准备会议材料
- 需要模拟客户可能的提问

### 使用方法
1. **给背景**：把客户信息、行业背景、我们的方案特点喂给 LLM
2. **给角色**：让 LLM 扮演客户/竞争对手/评审专家
3. **给任务**：明确要输出什么（方案、对比表、FAQ、话术）
4. **迭代优化**：对输出不满意就追问、修改、补充

### 具体用法
- "假设你是客户的 CTO，你会问我们什么问题？"
- "帮我写一份针对 XX 行业的方案摘要，重点突出 YY"
- "对比我们和竞争对手 A 的方案，列出优劣势"
- "帮我想 5 个在拜访时可以问的问题"

### 注意事项
- LLM 输出需要人工审核，不能直接用
- 敏感信息不要喂给公开 LLM
- 把 LLM 当加速器，不是替代品

## 技能四：投入前风险控制

### 投入前必问的五个问题
1. **这个机会的 ACV 是多少？** → 判断值不值得投入
2. **我们赢的概率有多大？** → 基于 gate 判断，不是感觉
3. **需要投入多少售前资源？** → 人力、时间、高层介入
4. **如果输了，损失是什么？** → 沉没成本、机会成本
5. **有没有更低投入的验证方式？** → 先用小投入验证再加码

### 风险控制策略
- **分阶段投入**：不要一开始就全力投入，先用小投入验证机会真实性
- **设置止损点**：在投入前就定好"如果 XX 没有达到，就停止投入"
- **高层介入要谨慎**：高层时间是稀缺资源，用在确定性高的机会上
- **POC 要有标准**：不是"做做看"，而是"达到什么标准才算成功"

## 技能五：资源分配铁律

### 铁律一：二八法则
- 80% 的资源投入 20% 的高价值机会
- 不要平均撒网

### 铁律二：确定性优先
- 资源优先投入确定性高的机会
- 不确定的机会先用小投入验证

### 铁律三：回报率思维
- 每个机会的资源投入要有回报率预期
- 回报率 = ACV × 赢率 ÷ 预计投入

### 铁律四：机会成本意识
- 投入 A 意味着不投入 B
- 选择做一件事的同时，也在选择不做另一件事

### 铁律五：及时止损
- 发现投入方向错了，立刻止损
- 沉没成本不是继续投入的理由

## 技能六：不值得 review 的项目信号

### 信号一：信息严重缺失
- 连客户是谁、谁在决策都不知道
- 没有任何一次有效拜访
- **处理**：先补信息，不值得 review

### 信号二：长期无进展
- 超过 60 天没有实质性推进
- 每次 review 都是"下周再看看"
- **处理**：降级或关闭，释放资源

### 信号三：场判断为负
- 明显是陪标项目
- 客户采购动机不纯
- **处理**：轻投入或放弃

### 信号四：资源严重不匹配
- 项目需要的资源远超我们能提供的
- 项目规模与我们的能力不匹配
- **处理**：评估是否值得投入，或找合作伙伴

### 信号五：客户关系已经破裂
- 客户明确表示不考虑我们
- 之前合作有严重问题未解决
- **处理**：暂停，修复关系后再考虑

## 在 Review 中的使用

1. **对比表**：在方案阶段用，帮客户建立评估框架
2. **对话脚本**：在 Discovery 阶段用，系统获取信息
3. **LLM 工具**：在方案准备阶段用，提升效率
4. **风险控制**：在投入决策前用，避免资源浪费
5. **资源分配**：在 pipeline review 时用，审视资源配置
6. **不值得 review 的信号**：在每个 review 开始前用，快速筛除

---

## 关键原则

1. **工具服务于判断**：所有工具和技巧都是为了帮助做出更好的判断
2. **不要过度依赖技巧**：真诚和专业比任何技巧都重要
3. **实践出真知**：这些技能要在实战中不断打磨
4. **因地制宜**：不是每个场景都要用所有技能
5. **持续迭代**：好的销售会不断优化自己的战术库
