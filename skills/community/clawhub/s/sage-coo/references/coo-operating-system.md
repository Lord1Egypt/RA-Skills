# COO Operating System

当用户讨论经营节奏、OKR、指标、周会、复盘、组织操作系统、创始人时间分配或“公司怎么运转起来”时，加载本文件。

## 1. 操作系统定义

Sage COO 的核心产物不是建议，而是公司操作系统：

```text
愿景 -> 优先级 -> owner -> 节奏 -> 指标 -> 复盘 -> 机制更新
```

一个 1-30 人团队的操作系统应该轻，但不能空。它至少包含：

- 当前最重要目标
- 每周优先级
- 明确 owner
- 指标和阈值
- 决策记录
- open loops
- SOP 与授权边界

## 2. 四层运营节奏

| 频率 | 活动 | 目的 | 产出 |
| :--- | :--- | :--- | :--- |
| 每周 | 指标回顾、解除阻碍、做决定 | 防止漂移 | 决定、owner、截止时间 |
| 每月 | 路线图、预算偏差、流程复盘 | 纠偏 | 更新计划和 open loops |
| 每季度 | OKR、优先级、组织设计 | 重新校准 | 下季度重点 |
| 每年 | 战略、资本、招聘计划 | 方向确认 | 年度目标 |

## 3. 60 分钟周度经营会

原则：不做 PPT，只看数据、阈值和决定。

推荐议程：

| 时间 | 内容 |
| :--- | :--- |
| 0-5 分钟 | CEO / COO 说出本周最重要的 3 个数字 |
| 5-25 分钟 | 增长、销售或客户漏斗 |
| 25-40 分钟 | 产品 / 交付进度 |
| 40-50 分钟 | 财务 KPI、现金、消耗率、跑道 |
| 50-60 分钟 | 记录决定、owner、截止日期 |

规则：

- 指标连续 2 周红灯，必须停下来解决根因。
- 每个决定必须有 owner 和截止时间。
- 周会不是同步会，是决策会。

## 4. CEO / COO 仪表盘

根据阶段选择 5 个以内关键数字。

### Pre-PMF

- 激活率
- 早期用户留存
- 每周假设测试数量
- 客户访谈数量
- 学习速度

### Post-PMF

- Pipeline Coverage
- 新增收入 / 新增项目额
- 毛利或交付利润
- Burn Multiple 或月度净消耗
- 现金跑道
- 客户留存 / 复购 / NRR

对内容、服务或项目制团队，可替换为：

- 新增有效商机
- 方案转化率
- 项目准时交付率
- 客户修改轮次
- 人均可交付产能
- 应收账款回款周期

## 5. OKR 执行系统

OKR 层级：

```text
公司 OKR（季度）
  -> 团队 OKR（季度）
      -> 个人每周优先级
          -> 每日站会焦点
```

好的 KR 必须满足：

1. 具体，有单一方程式。
2. 有明确 owner。
3. 每周能观察。
4. 能向上追溯到公司战略。

COO 的职责：

- 控制目标数量。
- 把 KR 接入周会。
- 防止团队写“看起来努力”的目标。
- 让每周动作能追溯到季度目标。

## 6. RACI 授权系统

每个重要项目、流程或跨职能事项都应该有轻量 RACI。

| 角色 | 含义 | 规则 |
| :--- | :--- | :--- |
| R | Responsible，实际执行者 | 只能有 1 个 |
| A | Accountable，最终负责人 | 只能有 1 个 |
| C | Consulted，被咨询者 | 控制数量 |
| I | Informed，被告知者 | 事后同步 |

如果一件事没有 A，它迟早会变成创始人的事。
