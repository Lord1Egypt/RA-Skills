# 古董鉴定估价 (Antique Appraisal)

## Overview

本技能提供完整的古董/藏品在线鉴定估价服务。用户描述或上传藏品信息后，自动完成信息结构化采集、AI辅助初步评估、鉴定服务报价、支付引导、最终生成交互式HTML可视化鉴定报告。同时支持微信小程序和Web浏览器双环境自适应输出。

## 环境检测

在技能启动时，首先检测当前运行环境：

- **微信小程序环境**：检测到微信小程序特征（如 `wx` 对象、小程序 UA 标识）时，所有输出使用简化的卡片式格式，图片尺寸优化，交互精简，适合小程序内展示。
- **非微信终端（Web/WorkBuddy）**：使用完整HTML交互式报告，支持全部可视化图表和动画效果。

环境检测逻辑：询问用户当前在哪个终端使用，或通过上下文判断。默认为非微信终端（Web/WorkBuddy环境）。

## 工作流决策树

```
用户输入（含触发词）
    │
    ├── 描述藏品但未给图片 → 进入 Phase 1：信息采集（含图片引导）
    ├── 上传了藏品图片 → 进入 Phase 1：信息采集（图片已满足，补全其他信息）
    ├── 直接问"值多少钱" → 先引导信息采集，再估价
    └── 查询已有订单 → 进入 Phase 4 直接输出鉴定结果
```

## Phase 1: 信息采集

### 采集维度

按优先级依次询问以下信息，每次最多问2个问题：

1. **藏品名称/类型**（必填）
   - 陶瓷（青花/粉彩/单色釉/紫砂等）
   - 书画（国画/书法/油画/版画等）
   - 玉器（翡翠/和田玉/独山玉/岫玉等）
   - 青铜器/金银器
   - 钱币（古钱币/银元/纸币等）
   - 木器/家具
   - 杂项（文房四宝/鼻烟壶/铜镜等）
   - 其他：用户自行描述

2. **藏品图片**（至少1张，建议3-5张多角度）
   - 正面全景图
   - 底部/背面细节图
   - 落款/印章特写
   - 瑕疵/修复痕迹特写
   - 尺寸参照物对比图
   - 如用户未上传，主动引导：格式支持 JPG/PNG/WebP，建议光线充足、背景简洁

3. **年代判断**（选填，用户自述）
   - 可选择朝代/时期，或"不确定，需鉴定"

4. **尺寸规格**（选填）
   - 长度 × 宽度 × 高度（厘米）
   - 重量（如有必要）

5. **品相描述**（选填）
   - 完好 / 微瑕疵 / 有修复痕迹 / 残损
   - 让用户描述具体瑕疵位置和程度

6. **来源/传承**（选填）
   - 家传 / 购买 / 馈赠 / 出土 / 其他
   - 是否有来源证明/鉴定证书

7. **参考信息**（选填）
   - 过往鉴定结果
   - 类似藏品参考价
   - 用户期望估价区间

### 采集完成确认

```
确认无误？(输入"确认")  /  需要修改？(说明修改项)
```

## Phase 2: 初步评估与报价

### 2.1 AI初步分析

基于用户提供的藏品信息，结合 references/antique-knowledge.md 中的知识库，进行初步分析：

- **类型特征**：描述该类型藏品的典型特征和鉴别要点
- **年代判断线索**：根据描述和图片中的风格、工艺、材质等线索推测年代
- **品相影响评估**：评估品相对价值的影响程度
- **初步估价区间**：给出市场参考价范围（分低/中/高三个等级）

### 2.2 鉴定服务报价

根据藏品类型和用户选择的鉴定深度，提供分级服务：

| 服务等级 | 内容 | 价格 | 周期 | 适合人群 |
|---------|------|------|------|---------|
| 🔍 基础鉴定 | AI+初级鉴定师，出具基础鉴定报告 | ¥99 | 2小时 | 好奇了解、入门收藏者 |
| ⭐ 专家鉴定 | 高级鉴定师，详细鉴定报告含市场分析 | ¥299 | 24小时 | 有一定价值的藏品 |
| 👑 权威鉴定 | 专家团队联合鉴定，权威证书+拍卖建议 | ¥999 | 3-5个工作日 | 高价值藏品、拍卖/保险用途 |

推荐用户根据藏品预估价值选择合适等级。如用户不确定，推荐专家鉴定。

### 2.3 用户选择确认

```

1. 基础鉴定（¥99）— 适合初步了解
2. 专家鉴定（¥299）— 推荐，性价比最高
3. 权威鉴定（¥999）— 高价值藏品首选

请回复数字 1 / 2 / 3
```

## Phase 3: 支付与订单

### 3.1 订单生成

用户选择服务等级后，生成订单：

- 订单编号：ANTQ-YYYYMMDD-XXXXXX（6位随机）
- 藏品信息摘要
- 服务等级和价格
- 预计完成时间

### 3.2 支付引导

根据运行环境生成支付方式：

**微信小程序环境**：
- 直接拉起微信支付
- 提供微信支付小程序码

**非微信终端（Web）**：
- 生成模拟支付二维码（使用 SVG 内嵌二维码，可选）
- 引导用户扫描二维码完成支付
- 提供支付完成后的确认按钮

支付二维码生成方式：使用 `show_widget` 工具生成内嵌 SVG 二维码，或提示用户在正式环境中扫码支付。

### 3.3 支付确认

无论哪种环境，最终都需要确认支付状态。在技能演示/模拟场景中，提供"模拟支付完成"选项。

## Phase 4: 鉴定报告输出

支付完成后，生成最终鉴定报告。

### 4.1 报告内容结构

鉴定报告包含以下模块：

**微信小程序版本**（简化卡片式）：
1. 藏品基本信息卡片
2. 年代鉴定结论
3. 保存状况评级
4. 市场参考价区间
5. 收藏建议

**非微信终端（Web完整版）**：
1. 📋 藏品信息总览
2. 🔍 年代鉴定分析（含朝代特征匹配）
3. 🎨 风格/工艺特征分析
4. 📐 品相与保存状况评分（5维雷达图）
5. 💰 市场估价（对比图表）
6. 📈 投资收藏建议
7. ⚠️ 注意事项与风险提示
8. 📚 参考资料与出处

### 4.2 报告生成方式

使用 `assets/report-template.html` 作为模板，替换其中的占位符为实际鉴定数据。

模板支持的占位符：
- `{item_name}` - 藏品名称
- `{item_type}` - 藏品类型
- `{era_judgment}` - 年代判断
- `{condition_rating}` - 品相评分
- `{price_low}` - 估价下限
`{price_high}` - 估价上限
- `{appraisal_details}` - 详细鉴定文字
- `{market_analysis}` - 市场分析
- `{expert_advice}` - 收藏建议
- `{risk_notes}` - 风险提示
- `{report_date}` - 报告日期
- `{order_id}` - 订单编号

### 4.3 环境自适应输出

- **微信小程序**：输出精简文本+卡片，适合小程序页面渲染
- **Web/WorkBuddy**：生成完整HTML文件，使用 `present_files` 呈现

## 常见鉴定品类速查

## 注意事项

1. **免责声明**：所有鉴定结果为AI辅助分析参考，不构成法律或交易依据，重大交易请委托权威机构
2. **图片质量**：图片质量直接影响鉴定准确度，对模糊/低质量图片要提前说明局限性
3. **文字描述优先**：当图片信息不足以判断时，优先依据文字描述，同时标注不确定性
4. **价格参考**：估价基于公开市场数据和个人藏品销售记录，实际成交价受拍卖行、市场波动等因素影响
5. **文物保护**：如发现疑似文物（根据《文物保护法》标准），提示用户遵守相关法律法规，不得私自买卖
