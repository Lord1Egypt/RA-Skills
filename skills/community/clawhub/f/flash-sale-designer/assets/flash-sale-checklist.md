# Flash Sale Quality Checklist

Use this checklist before, during, and after every flash sale event.

---

## Pre-Launch (T-48 Hours)

### Strategy & Mechanics
- [ ] Event rationale documented (clearance / launch / revenue acceleration / loyalty)
- [ ] Discount depth confirmed and margin calculated at minimum expected sell-through
- [ ] Flash allocation quantity confirmed with fulfillment/3PL
- [ ] Duration set and end time communicated in primary audience timezone
- [ ] Platform(s) confirmed: Shopify / TikTok Shop / Amazon / WooCommerce / other

### Inventory & Operations
- [ ] Flash allocation reserved in inventory system (separate from main stock)
- [ ] Fulfillment team notified of expected volume surge
- [ ] Packing and shipping capacity confirmed for 2–3x normal daily volume
- [ ] SKU(s) active, priced correctly, and displaying properly on all devices
- [ ] Payment processor alerted if single-event volume is unusual

### Technical Setup
- [ ] Countdown timer installed and server-side (not client-side clock)
- [ ] Timer tested on mobile Safari, Chrome, and Firefox
- [ ] Stock visibility thresholds configured in inventory app
- [ ] Cart page shows timer and discounted price correctly
- [ ] Checkout flow tested end-to-end on mobile and desktop
- [ ] Site load test performed (or CDN/caching confirmed active)

### Communication Assets
- [ ] T-72h teaser email drafted and scheduled
- [ ] T-24h reveal email drafted and scheduled
- [ ] T-1h reminder SMS drafted and scheduled
- [ ] T-0 launch email drafted and scheduled
- [ ] Mid-event and final push emails drafted (to be manually triggered if conditions met)
- [ ] Social posts (launch, mid-event) drafted and saved
- [ ] TikTok live outline prepared (if running live selling)

---

## Pre-Launch (T-2 Hours)

- [ ] All scheduled emails confirmed in ESP (not stuck in draft)
- [ ] Discount code / automatic discount confirmed active on correct products
- [ ] Timer is counting down correctly on live product page
- [ ] Mobile UX reviewed one final time
- [ ] Customer service team briefed on event details and common questions
- [ ] Analytics / pixel confirmed firing on product page and checkout

---

## During Event

### First 30 Minutes
- [ ] Launch email confirmed delivered (check ESP dashboard)
- [ ] Social posts published
- [ ] Site performance monitored (no 5xx errors, page load <3s)
- [ ] Orders flowing through correctly in admin panel
- [ ] Stock counts updating in real-time

### Ongoing
- [ ] Monitor cart abandonment rate — if >80%, check for checkout friction
- [ ] Watch for social media mentions and respond to questions
- [ ] Mid-event email triggered if velocity is high (optional)
- [ ] Customer service queue monitored for order issues

### Final 30–60 Minutes
- [ ] Final push email and SMS triggered to non-purchasers
- [ ] If near sellout, activate waitlist CTA messaging

---

## Post-Event (Within 2 Hours)

- [ ] "Event ended" / waitlist email sent to non-buyers
- [ ] "Thank you" confirmation email sent to buyers (or confirm automated flow triggered)
- [ ] Discount deactivated / automatic discount turned off
- [ ] Timer removed or replaced with "sale ended" messaging
- [ ] Social post published with results summary (optional but good for community)
- [ ] Waitlist landing page live for next event

---

## Post-Event (Within 48 Hours) — Analysis

- [ ] Revenue captured during event vs. forecast
- [ ] Conversion rate during event vs. site baseline
- [ ] Average order value during event vs. baseline
- [ ] Email open and click rates for each touchpoint
- [ ] SMS opt-out rate
- [ ] Units sold vs. allocation (sell-through rate)
- [ ] New customers acquired vs. repeat buyers
- [ ] Return/refund rate of flash-sale orders (track over 30 days)
- [ ] Lessons documented for next event

---

## Repeat Event Guardrails

- [ ] Confirmed at least 30 days since last flash sale on same product
- [ ] Confirmed no competing sale events in calendar (avoid stacking promotions)
- [ ] Full-price conversion rate reviewed — no structural decline from over-discounting
- [ ] Waitlist from previous event will receive early access messaging
