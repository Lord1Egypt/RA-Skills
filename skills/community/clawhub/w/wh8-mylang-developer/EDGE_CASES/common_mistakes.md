## 一、语法错误

### ❌ 错误1：信号不带条件
```mql
// 错误写法
BK;

// 正确写法
CLOSE > MA(C,10), BK;
```

### ❌ 错误2：加减仓不带手数
```mql
// 错误写法
ADD_LONG;  // 缺少手数
ADD_SHORT; // 缺少手数

// 正确写法
ADD_LONG(2);   // 加仓2手
ADD_SHORT(1);  // 加仓1手
```

### ❌ 错误3：忘记写AUTOFILTER
```mql
// 错误写法
CROSS(MA5,MA10), BK;
CROSSDOWN(MA5,MA10), SP;
// 模型结束，缺少 AUTOFILTER

// 正确写法
CROSS(MA5,MA10), BK;
CROSSDOWN(MA5,MA10), SP;
AUTOFILTER;  // 必须写！
```

### ❌ 错误4：条件写成中文
```mql
// 错误写法
收盘价>均线, BK;

// 正确写法
CLOSE > MA(C,10), BK;
```

### ❌ 错误5：中文字符混用
```mql
// 错误写法
N:=10；  // 全角分号
// 条件A，BK；  // 全角逗号和分号

// 正确写法
N:=10;  // 半角分号
条件A, BK;  // 半角逗号和分号
```

## 二、逻辑错误

### ❌ 错误6：没有持仓时加减仓
```mql
// 错误场景：假设当前无持仓
ADD_LONG(1);  // 无效！必须先BK开仓
LOWER_LONG(1); // 无效！
```

### ❌ 错误7：减仓超过持仓
```mql
// 假设当前持仓3手
LOWER_LONG(5);  // 错误！减仓手数不能超过持仓
// 正确：LOWER_LONG(2);  最多减2手
```

### ❌ 错误8：同时有BK和SK信号
```mql
// 错误写法
CLOSE > MA(C,10), BK;   // 买开
CLOSE < MA(C,10), SK;   // 卖开（同时满足会冲突）

// 正确写法：分开条件
CLOSE > MA(C,10), BK;
CLOSE < MA(C,10), BP;   // 用买平而不是卖开
```

### ❌ 错误9：开仓条件写在平仓条件后面
```mql
// 错误写法（信号顺序）
SP;     // 平仓写在前面
BK;     // 开仓写在后面
// 这会导致逻辑混乱

// 正确写法
BK;     // 开仓
SP;     // 平仓
```

### ❌ 错误10：BPK/SPK和BK/SK混用
```mql
// 错误写法
BPK;           // 反手买
BK;            // 再买开（会冲突）

// 正确写法：只用一种
BPK;           // 反手买
// 或
BK;            // 买开
SP;            // 卖平
```

## 三、边界条件

### ⚠️ 边界1：加减仓过滤
```mql
// 同一行加减仓指令一轮只出一次
TRADE_AGAIN(3);  // 允许连续3次

// 如果没有TRADE_AGAIN，同一行加减仓只会出一次
```

### ⚠️ 边界2：信号消失（忽闪）
```
K线走完前条件满足 → 出信号 → K线走完条件不满足 → 信号消失
```
- 使用 `SIGCHECK` 可以进行信号复核
- 消失的信号会被恢复持仓

### ⚠️ 边界3：主连链回测换月
```
主力切换时：
1. 旧主力清仓
2. 新主力重新计算信号
3. 资金延续
```
- 换月时新旧合约趋势可能相反
- 需要关注换月点附近的信号

### ⚠️ 边界4：持仓成本变化
```mql
// 加仓后BKPRICE会变化（加权平均）
ADD_LONG(2);           // 加仓
BKPRICE;               // 现在是加仓后的平均价
BKPRICE2 := BKPRICE;  // 如果需要记录原开仓价，要单独保存
```

### ⚠️ 边界5：权益计算时机
```mql
// MONEYTOT是实时权益（含浮动盈亏）
// 持仓为0时的权益才是真正的盈亏
// 权益最大回撤和损益最大回撤计算方式不同
```

### ⚠️ 边界6：信号计算起始时间
```
数据起始 ≠ 信号计算起始

例：沪铜2201合约
- 挂牌时间（数据起始）：2021年1月18日
- 开始交易时间（信号计算起始）：2021年11月1日
- 设置信号计算起始时间应该：2021年11月1日
```

## 四、实盘问题

### ⚠️ 问题1：滑点影响
```mql
// 回测假设以收盘价成交
// 实盘可能滑点较大

// 敏感性测试：
// 回测后右键 → 敏感性测试
// 分析滑点增大时收益变化
```

### ⚠️ 问题2：信号不成交
```mql
// 原因：
// 1. 价格快速波动，委托价格不利
// 2. 流动性不足
// 3. 网络延迟

// 解决方案：使用SETSIGPRICETYPE
SETSIGPRICETYPE(BK, 2);   // 买开用对手价
SETSIGPRICETYPE(SP, 4);   // 止损用市价
```

### ⚠️ 问题3：隔夜跳空
```mql
// 期货隔夜有跳空风险
// 模型需要考虑：
// 1. 隔夜止损设置
// 2. 仓位控制
// 3. 使用TRADE_SMOOTHING消除跳空
```

### ⚠️ 问题4：换月跳空
```mql
// 主力合约切换时价格不连续
// 解决方案：
// 1. 使用主连链回测
// 2. 主力切换时手动平移
// 3. 使用TRADE_OTHER('ZHULIAN')自动换月
```

### ⚠️ 问题5：流动性风险
```mql
// 小品种流动性差
// 建议：
// 1. 优先选择主力合约
// 2. 设置最小成交量条件
VOL>1000, BK;  // 成交量大于1000才开仓
```

## 快速检查清单

### 写完模型后检查：
- [ ] 信号都带条件了吗？
- [ ] 加减仓都带手数了吗？
- [ ] 写了 `AUTOFILTER;` 吗？
- [ ] 没有用中文标点吗？
- [ ] 平仓写在开仓后面了吗？
- [ ] 减仓手数不超过持仓吗？
