# 周报检查规则详解

## 规则1：内容变动

**目的：** 检测是否存在每周复制粘贴相同内容的情况。

**检查方法：**
- 对比同一人最近4周的"本周工作内容"字段
- 如果文本完全一致 → 标记为不合格
- 例外：单一工作职责（如"运维值班"、"日常维护"），在报告中单独标注但不判定不合格
- 实施建议：使用 `===` 精确匹配，忽略空格差异（先 trim 再比较）

**判定标准：**
- 连续2周完全相同 → ⚠️ 需关注
- 连续3-4周完全相同 → ❌ 不合格（除非是明确的单一职责）
- 部分相似（如开头相同、大部分内容相同）→ ⚠️ 需关注

**典型问题案例：**
```
W1: 维护系统，处理日常工单，修复bug
W2: 维护系统，处理日常工单，修复bug
W3: 维护系统，处理日常工单，修复bug
```

## 规则2：描述清晰度

**目的：** 确保工作内容描述足够具体，能支撑一周的工作量。

**检查方法：**
- 统计任务条目数（按序号如 `1.` `1、` `1）` 或换行分隔）
- 检查是否有进度标注（百分比、时间节点如 "2025-01-15"）
- 是否有项目名称或模块标识

**判定标准：**
- 任务条目 < 3 条 → ⚠️ 需关注
- 无任何进度/百分比/时间标注 → ⚠️ 需关注
- 内容全为泛泛描述（如"处理日常事务"、"完成领导安排工作"）→ ⚠️ 需关注

**良好的周报示例：**
```
【项目A】
1、完成用户模块接口开发 100%
2、配合前端联调支付流程 80%
3、修复生产环境登录异常问题 100%

【项目B】
1、参加需求评审会议 100%
2、编写技术方案文档 50%
```

## 规则3：文字数量

**目的：** 通过字数检测态度敷衍的周报。

**检查方法：**
- 计算本周工作内容的有效字符数
- 有效字符：中文字（\u4e00-\u9fff）、英文字母（a-zA-Z）、数字（0-9）
- 排除：纯标点符号、空格、换行符

**判定标准：**
- **< 50 个有效字符 → ❌ 直接不合格（态度问题）**
- 实施严格，不设例外
- 注意：`休假` 2字也触发规则3，但标注"休假"说明情况，不按态度问题处理

**代码实现：**
```javascript
const workChars = (text.match(/[\u4e00-\u9fff\w]/g) || []).length;
```

## 规则4：必填与重复

**目的：** 确保关键字段有填写，且计划有变化。

**检查方法：**
- 检查"本周工作内容"是否为空
- 检查"下周工作计划"是否为空
- 检查同一人近4周的"下周工作计划"是否完全相同

**判定标准：**
- 本周工作内容为空 → ❌ 不合格
- 下周工作计划为空 → ❌ 不合格
- 近4周下周工作计划完全相同 → ❌ 不合格
- 字段有值但内容无实质意义（如"无"、"待定"）→ ❌ 不合格

**特殊情况处理：**
- `休假` → 不判定不合格，但在报告中标注
- `待领导安排工作` / `等待领导安排工作` → 虽然字段非空，但连续多周出现时视为态度问题

## 综合判定逻辑

```
对于每个人员:
  issues = []   # 直接不合格项
  warnings = [] # 需关注项

  # 规则3 (优先级最高)
  if countChars(work) < 50:
    if work == '休假' or work.contains('休假'):
      warnings.add('本周休假')
    else:
      issues.add('规则3: 工作内容<50字，态度问题')

  # 规则4 必填
  if work.trim() == '':
    issues.add('规则4: 本周工作内容为空')
  if plan.trim() == '':
    issues.add('规则4: 下周工作计划为空')

  # 规则1 内容变动 (需多周数据)
  if len(weeks) >= 2:
    uniqueWorks = set(w.work for w in weeks)
    if len(uniqueWorks) == 1:
      warnings.add('规则1: 内容无变化')

  # 规则2 描述清晰度
  taskCount = countTasks(work)
  if taskCount < 3:
    warnings.add('规则2: 仅XX条任务')

  # 规则4 计划重复 (需多周数据)
  if len(weeks) >= 2:
    uniquePlans = set(w.plan for w in weeks)
    if len(uniquePlans) <= 1:
      issues.add('规则4: 多周计划完全相同')

  # 最终判定
  if len(issues) > 0: return ❌ 不合格
  if len(warnings) > 0: return ⚠️ 需关注
  return ✅ 合格
```
