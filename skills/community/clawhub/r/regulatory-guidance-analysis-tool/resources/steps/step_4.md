## Step 4 — 法规分析

依次执行以下子任务。每项子任务执行前，读取 `resources/prompts/` 下对应的 prompt 文件，按其中指令分析，严格使用用户原文。

**每完成一个子任务，立即将分析结果写入对应的 Markdown 文件，保存到当前工作目录，再继续下一个子任务。**

| 顺序 | 子任务 | Prompt 文件 | 输出文件 |
|------|--------|-------------|----------|
| 4.1 | 法规&指南元素采集 | `4.1_元素采集.md` | `{原始文档stem}_4.1_元素采集.md` |
| 4.2 | 法规&指南定性 | `4.2_定性.md` | `{原始文档stem}_4.2_定性.md` |
| 4.3 | 法规&指南重点内容 | `4.3_重点内容.md` | `{原始文档stem}_4.3_重点内容.md` |
| 4.4 | 法规&指南生命周期图 | `4.4_生命周期图.md` | `{原始文档stem}_4.4_生命周期图.md` |
| 4.5 | 法规&指南红黄绿蓝灯图 | `4.5_红黄绿蓝灯.md` | `{原始文档stem}_4.5_红黄绿蓝灯.md` |

### 断点续跑

每个子任务执行前，检查对应输出文件是否已存在且内容有效：

- 若文件不存在：正常执行分析并写入
- 若文件已存在且内容 >10 有效字符（去格式后）：跳过该子任务，提示"4.X 已有缓存，跳过"

### 4.6 落盘与校验

全部子任务完成后，运行校验：

```bash
python scripts/validate.py {原始文档stem} --step 4
```

校验通过（exit 0）则继续，否则根据报错定位缺失文件并补全。
