# 🎯 九、开发流程总结

### 9.0 唯一信息源（生成 Custom.vue）

**页面稿源只允许：** 当前视图的 `index.vue` + **由 index.vue 引用**的样式文件（常见为 `assets/index.css`，以实际引用为准）+ 上述文件中的图片等资源路径。

**禁止：** 读取或对照本仓库其他文件来生成页面（如其他文件夹的 `Custom.vue`、`index.vue`、`create.md`、路由、`package.json` 等），不得以已有页面为模板拼代码。规范仍以 SKILL.md 与 `data/` 为准。

### 9.1 完整开发流程

```
阶段1: 准备工作（10分钟）
├─ 安装依赖（element-ui，按项目既有配置；生成页面时不依赖读取其他工程文件作稿源）
├─ 读取源文件（仅 index.vue + index.vue 所引用的 CSS）
├─ 理解页面结构和功能
└─ 创建Custom.vue基础结构

阶段2: 页面分析（10分钟）
├─ 分析页面布局
├─ 识别重复结构
├─ 识别Element UI替换点
└─ 输出分析报告

阶段3: 循环渲染（20分钟）
├─ 任务2: 筛选项开发（可选）
└─ 任务3: 列表开发（必须）

阶段4: 组件替换（30分钟）
├─ 任务4: Element UI组件替换
├─ 同步实现template、data、methods
├─ 覆盖组件样式
└─ 测试组件功能

阶段5: 完善页面（20分钟）
├─ 任务5: 其他页面部分渲染
├─ 补充所有静态内容
├─ 匹配所有样式
└─ 重命名所有class

阶段6: 验证测试（10分钟）
├─ 样式检查（开发者工具对比）
├─ 功能检查（测试所有交互）
├─ 代码检查（命名、结构、完整性）
└─ 运行测试（npm run serve）

总计: 约90-120分钟
```

### 9.2 检查点设置

**每个阶段结束时的检查点：**

```
✅ Checkpoint 1 - 准备工作完成
- [ ] Element UI已安装
- [ ] 源文件已读取
- [ ] Custom.vue已创建
- [ ] 基础结构已搭建

✅ Checkpoint 2 - 分析完成
- [ ] 页面结构已理解
- [ ] 重复结构已识别
- [ ] 组件替换点已标记
- [ ] 分析报告已输出

✅ Checkpoint 3 - 循环渲染完成
- [ ] 筛选项已实现（如有）
- [ ] 列表已实现（如有）
- [ ] v-for循环正常
- [ ] 样式完全一致

✅ Checkpoint 4 - 组件替换完成
- [ ] 所有组件已替换
- [ ] data和methods已实现
- [ ] 组件样式已覆盖
- [ ] 组件功能正常

✅ Checkpoint 5 - 页面完成
- [ ] 所有内容已渲染
- [ ] 所有class已重命名
- [ ] 所有样式已匹配
- [ ] 页面完整显示

✅ Checkpoint 6 - 验证通过
- [ ] 视觉效果一致
- [ ] 功能完全正常
- [ ] 代码质量合格
- [ ] 无错误无警告
```
