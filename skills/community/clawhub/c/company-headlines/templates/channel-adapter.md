# 渠道版转换清单

从母版（叙事骨架填完的完整稿）到三渠道版本的转换清单。

---

## 母版 → 公众号版

### 加什么
- [ ] 标题悬念化/观点化/数字化（提供3个候选）
- [ ] 开篇改场景镜头或问题切入
- [ ] 至少1处有画面感的细节描写
- [ ] 至少1句可截图转发的金句
- [ ] 段落拆分更短（手机屏3-4段）

### 减什么
- [ ] 内部知道的细节（人名、金额可能需模糊）
- [ ] 过于敏感的数字
- [ ] 正式公文式的套话

### 检查
- [ ] 篇幅600-1200字
- [ ] 读出来像一个人在聊天，不像公告
- [ ] 配图建议写了

---

## 母版 → 官网版

### 加什么
- [ ] 标准5W开篇（时间+主体+事件+地点+背景）
- [ ] 完整时间戳
- [ ] 正式全称+职务
- [ ] 可供媒体直接引用的段落

### 减什么
- [ ] 第一人称（"我们" → "雄韬股份"或"公司"）
- [ ] 口语表述
- [ ] 主观评价
- [ ] 场景描写和情感渲染

### 检查
- [ ] 篇幅300-500字
- [ ] 信息完整可存档
- [ ] 语言客观严谨
- [ ] 适合被外部引用

---

## 母版 → 内部版

### 加什么
- [ ] 第一人称（"我们"）
- [ ] 团队背后的故事和努力
- [ ] 具体人名（谁做了什么，点名感谢）
- [ ] 对公司/团队的意义
- [ ] 下一步行动和对团队的要求

### 减什么
- [ ] 对外营销语言
- [ ] 竞争比较
- [ ] 正式公文套话

### 检查
- [ ] 篇幅200-400字
- [ ] 真诚直接，有温度
- [ ] 团队能感受到被认可
- [ ] 有明确的下一步

---

## 通用检查（三版都做）

- [ ] 公司名是否正确
- [ ] 日期是否正确
- [ ] 禁用词检查通过
- [ ] 数字已核实
- [ ] 没有编造的信息
