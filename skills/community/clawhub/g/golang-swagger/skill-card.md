## Description: <br>
Golang OpenAPI/Swagger documentation with swaggo/swag, including annotation comments, swag init code generation, framework integrations, security definitions, and struct tags for adding or maintaining Swagger/OpenAPI docs in Go projects. <br>

This skill is ready for commercial/non-commercial use. <br>

## Use Case: <br>
Developers and engineers use this skill to add, audit, and maintain OpenAPI/Swagger documentation in Go services that use swaggo/swag and common Go HTTP frameworks. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Generated or edited Swagger documentation can diverge from implementation details or expose routes incorrectly. <br>
Mitigation: Review proposed source changes, generated docs, and Swagger UI route exposure before accepting them, especially for production services. <br>
Risk: Using an unpinned swag CLI install can reduce build reproducibility. <br>
Mitigation: Pin the swag CLI version when reproducible installs are required. <br>
