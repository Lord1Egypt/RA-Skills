# Mistakes-v2.md — 可追溯、可校验的行为纠错基因库

---

## 格式规范

```markdown
## Mistake-YYYY-MM-DD-XX
- **信号**：[触发场景] + [核心场景]，禁止模糊
- **错误**：[一句话说清楚是什么]
- **根因**：[技术层面，≤2行，禁止心理叙事]
- **验证状态**：[待复核/✅已确认/❌已作废]
- **风险级别**：[🟥严重/🟧中等/🟨轻微]
- **关联ID**：[同根因的旧ID，可选]
- **升级记录**：[已升级为规则 → 关联工作手册条款ID，可选]
```

## 字段填写标准

| 字段 | 标准 |
|------|------|
| **信号** | 必须写清触发条件+核心场景 |
| **根因** | 技术层面的底层原因，禁止人类心理叙事，不超过2行 |
| **风险级别** | 🟥严重=信任受损/违反明确原则；🟧中等=同一模式第3次触发；🟨轻微=偶发偏差 |
| **验证状态** | 新条目默认「待复核」，用户确认后改为「✅已确认」 |

## 流转规则

- 第1次：生成 Mistake 条目，标「待复核」
- 第2次：关联旧ID，触发+1
- 第3次：根因第3次出现 → 整理时自动生成「规则草案」→ 汇报确认后写入 AGENTS.md
