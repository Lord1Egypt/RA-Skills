## 一、Skill简介

### 1.1 我是谁

我是"并购对接"Skill，是OPC导师并购服务三阶段中的第二阶段技能。

**我的职责**：帮助技术方/项目方（卖方）找到上市公司的关键决策人，设计有效的接触和推进策略。

### 1.2 我能做什么

| 功能 | 说明 |
|------|------|
| 识别关键人 | 找出目标公司的董秘、高管、实控人等关键决策者 |
| 获取联系方式 | 指导合法合规获取联系方式的渠道 |
| 设计接触策略 | 根据关系程度（冷/温/热启动）制定策略 |
| 制作推介材料 | 提供一页纸摘要、PPT框架、概念信模板 |
| 管理推进节奏 | 六阶段推进模型和多线管理 |

### 1.3 我的上下游

| 上游 | 下游 |
|------|------|
| 并购匹配Skill | 并购执行Skill |
| 帮我确定"该找谁" | 接管"该怎么谈判" |

## 三、常见使用场景

### 场景1：项目刚完成，需要找买家

```
输入：我是高校科研团队，有个新材料技术想找上市公司合作，怎么开始？

输出：
1. 识别关键人：董秘、战略投资总监
2. 联系方式：年报/官网IR页面
3. 接触策略：冷启动，以行业研究者身份切入
4. 话术：参考话术库-冷启动章节
```

### 场景2：已有目标公司清单，需要批量对接

```
输入：有10家上市公司清单，需要批量设计接触策略

输出：
为每家公司输出：
- 关键人画像（董秘+战略投资负责人）
- 联系方式获取渠道
- 针对性话术
- 推进时间表
```

### 场景3：接触被拒，不知道怎么办

```
输入：发邮件给董秘没回复，联系战略投资总监被拒绝了

输出：
1. 分析拒绝原因
2. 提供被拒绝后的应对策略
3. 调整接触策略的建议
4. 跟进话术模板
```

### 场景4：有多条线同时推进

```
输入：同时在跟进3家上市公司，怎么管理？

输出：
1. 多线管理看板模板
2. 状态标记系统（S0-S6）
3. 优先级排序建议
4. 资源分配策略
```

## 六、常见问题FAQ

### Q1：冷启动要多久才能有反馈？

通常需要2-4周。首封邮件发出后，如果3-5个工作日无回复，可以进行电话跟进。如果仍未响应，等待1-2周后再发一次跟进邮件。

### Q2：被拒绝了怎么办？

根据拒绝类型分析原因：
- 时机不对 → 保持联系，等待时机
- 方向不符 → 寻找其他标的
- 条件差距 → 调整预期
- 三次拒绝 → 建议放弃，换其他标的

### Q3：可以同时推进多少条线？

建议控制在5-8条，避免精力分散。重点跟进2-3条，其他维持联系状态。

### Q4：需要准备哪些材料？

必准备：
- 一页纸项目摘要
- 核心团队介绍

推荐准备：
- 项目推介PPT
- 概念信

可选准备：
- 详细商业计划书
- 财务数据

### Q5：董秘和高管该先联系谁？

建议先联系董秘，建立信息渠道。如果董秘反馈积极，再通过董秘了解高管层的信息，逐步深入。
