# Report Formatter — 专业报告排版与格式化

## 描述
将原始内容快速格式化为专业级别的报告/文档/邮件，适配多种输出场景。内置主题风格，支持一键排版。

## 触发条件
- 用户要求"排版"、"格式化"、"整理成报告"、"做成PPT大纲"
- 生成报告、会议纪要、周报、月报、提案等结构化文档后
- 用户说"太乱了"、"整理一下"、"美化"

## 支持的输出格式

├─────────────────────────────────────────┤
│                                          │
│  一、摘要/Executive Summary              │
│  （3-5行核心结论）                        │

## 主题风格

### 商务简约（默认）
- 字体层级清晰，留白充足
- 蓝色系标题，灰色正文
- 适合：报告、邮件、提案

### 科技感
- 深色背景（如果输出HTML/PDF）
- 渐变色标题
- 适合：技术报告、产品方案

### 温暖亲和
- 圆角、暖色调
- 更多表情符号
- 适合：公众号、内部通知

### 极简
- 纯文字，无装饰
- 适合：正式公文、法律文件

## 处理流程

### 第一步：识别内容类型
判断输入内容的性质（数据/观点/会议记录/周报素材等）

### 第二步：匹配最佳模板
根据内容类型+用户场景，选择最合适的输出格式

### 第三步：结构化重组
- 提取核心信息（结论、数据、行动项）
- 按模板结构重新组织
- 补充缺失的元数据（日期、署名等）

### 第四步：排版美化
- 应用主题风格
- 统一标题层级
- 优化表格/列表格式
- 控制段落长度（每段不超过5行）

### 第五步：质量检查
- [ ] 标题层级是否正确
- [ ] 数据是否准确保留
- [ ] 行动项是否有负责人和截止日期
- [ ] 字数是否适合场景（邮件<500字，报告<3000字）
- [ ] 格式是否一致

## 输出

```markdown
## 排版完成 ✅

**格式**：[报告/会议纪要/周报/邮件/提案/公众号/PPT大纲]
**主题风格**：[商务简约/科技感/温暖亲和/极简]
**字数**：XXX
**预计阅读时间**：X分钟

[格式化后的完整内容]
```

## 快捷命令

| 用户说 | 触发格式 |
|--------|---------|
| "排版成报告" | 内部报告 |
| "整理会议纪要" | 会议纪要 |
| "写周报" / "周报格式" | 周报/月报 |
| "格式化邮件" | 正式邮件 |
| "做成方案" / "提案格式" | 提案/方案书 |
| "公众号排版" | 微信/公众号文章 |
| "PPT大纲" | PPT大纲 |
| "整理一下" | 自动判断最佳格式 |

## 注意事项
- 保持原文的核心信息和数据不变
- 如果信息缺失（如会议纪要缺少参会人），用 [待补充] 标记
- 输出格式可根据用户指定调整
- 如果内容不适合某种格式（如技术方案硬套周报模板），主动建议更合适的格式
