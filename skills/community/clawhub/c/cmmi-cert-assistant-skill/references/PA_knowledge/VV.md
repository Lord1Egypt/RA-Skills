# VV — 验证与确认（CMMI V3.0 Core PA）

## PA 基本信息
- **缩写**：VV
- **英文**：Verification and Validation
- **中文**：验证与确认
- **所属**：Core PA（核心实践域）
- **对应 V2.0**：相当于 VER（验证）+ VAL（确认）

---

## CMMI V3.0 要求要点

### 核心目的
确保工作产品满足指定需求（验证），最终产品满足预期用途（确认）。

### 关键实践
| 实践编号 | 实践内容 |
|---------|---------|
| VV 1.1 | 策划验证活动 |
| VV 1.2 | 策划确认活动 |
| VV 1.3 | 选择验证与确认方法 |
| VV 2.1 | 实施验证 |
| VV 2.2 | 实施确认 |
| VV 3.1 | 确保验证与确认结果可追溯 |

### 通用实践
- GP 1.1 ~ GP 2.10（完整覆盖）

## 用户文档 → PA 匹配规则#

### 高概率匹配（主要PA）
- 文档名含"测试计划"、"测试用例"、"验收测试" → **VV 主要匹配**
- 文档含测试报告、缺陷清单 → **VV 主要匹配**

### 中概率匹配（次要PA）
- 文档含同行评审记录 → 同时匹配 **PR**
- 文档含需求追溯矩阵 → 同时匹配 **RDM**
