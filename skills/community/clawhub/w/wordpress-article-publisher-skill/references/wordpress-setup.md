# WordPress 应用程序密码设置教程

## 什么是应用程序密码？

应用程序密码（Application Password）是 WordPress 提供的一种认证方式，允许第三方应用在不暴露主密码的情况下访问 WordPress REST API。

**优点：**
- ✅ 安全性高（可随时撤销）
- ✅ 无需暴露登录密码
- ✅ 可以限制权限
- ✅ 独立于主密码

## 如何生成应用程序密码

### 步骤 1：登录 WordPress 后台

访问你的 WordPress 管理后台：
```
https://你的域名/wp-admin/
```

### 步骤 2：进入用户设置

点击左侧菜单「用户」→「个人资料」

或者直接访问：
```
https://你的域名/wp-admin/profile.php
```

### 步骤 3：找到应用程序密码设置

在个人资料页面，向下滚动找到「应用程序密码」部分。

**注意：** 如果看不到这个选项，可能是因为：
1. 你的 WordPress 版本低于 5.6
2. 你的角色不是管理员
3. 网站使用了限制 REST API 访问的插件

### 步骤 4：生成新密码

1. 在「应用程序密码」部分，找到名称输入框
2. 输入应用程序名称（如 "OpenClaw助手"、"手机APP" 等）
3. 点击「添加新应用程序密码」按钮

### 步骤 5：复制密码

WordPress 会生成一个 16 位的密码，格式类似：
```
xxxx xxxx xxxx xxxx
```

**重要：**
- 密码只显示一次，请立即复制保存
- 密码中包含空格，请完整复制（包括空格）

### 步骤 6：使用应用程序密码

将以下信息提供给助手：
```
用户名：你的WordPress用户名
应用程序密码：xxxx xxxx xxxx xxxx
```

## 常见问题

### Q1：应用程序密码和普通密码有什么区别？

| 对比项 | 普通密码 | 应用程序密码 |
|--------|----------|--------------|
| 用途 | 登录后台 | API访问 |
| 可撤销 | 不能 | 可以，随时撤销 |
| 暴露风险 | 高 | 低 |
| 多设备管理 | 不支持 | 支持，每个设备独立密码 |

### Q2：应用程序密码安全吗？

应用程序密码本身是安全的，但需要注意：
- ✅ 使用 HTTPS 传输（防止中间人攻击）
- ✅ 不要在不受信任的设备上保存
- ✅ 定期检查和撤销不用的密码
- ✅ 发现异常立即撤销密码

### Q3：忘记了应用程序密码怎么办？

应用程序密码无法找回，只能重新生成：
1. 进入「用户」→「个人资料」
2. 找到已生成的应用程序密码
3. 点击密码旁边的「撤销」按钮
4. 重新生成新的密码

### Q4：可以生成多个应用程序密码吗？

可以，而且建议为不同用途生成不同的密码：
- 一个用于桌面助手
- 一个用于手机应用
- 一个用于自动化脚本

这样如果某个密码泄露，只需撤销该密码，不影响其他用途。

### Q5：应用程序密码不工作怎么办？

排查步骤：
1. ✅ 确认用户名和密码正确
2. ✅ 确认密码格式正确（包含空格）
3. ✅ 确认网站使用 HTTPS
4. ✅ 确认 REST API 未被阻止
5. ✅ 确认应用程序密码未被撤销

### Q6：网站使用双因素认证可以使用应用程序密码吗？

可以。应用程序密码是独立于双因素认证的，不会触发二次验证。

## 安全建议

### ✅ 推荐做法
- 为每个应用生成独立的应用程序密码
- 使用强随机密码（WordPress 自动生成的就是强密码）
- 定期审查和清理不用的应用程序密码
- 在 HTTPS 网站上使用

### ❌ 不推荐做法
- 在多个应用间共享同一个密码
- 在不安全的网络环境下使用
- 将密码保存在明文文件中
- 使用简单或重复的密码名称
