# 质量检验清单

## 生成前检验（Phase Gate）
在调用generate_batch.py之前，逐项确认：

### 需求完整性
- [ ] 5维度需求全部确认（内容定位/章节架构/页面规模/数据精度/风格偏好）
- [ ] 风格预览已通过用户选择
- [ ] 如用户要求排除/增加特定内容，已记录

### Prompt质量
- [ ] 每页标题带观点（非纯描述性）
- [ ] 使用对应风格的标准化prompt模板
- [ ] 文字用引号包裹
- [ ] 无拟人暗示词汇（商务/现代/专业等）
- [ ] 无颜色描述（由风格模板控制）
- [ ] 布局有足够变化（非连续同布局）
- [ ] 单页信息量适中（1-3个核心点）
- [ ] 4+分点用横向/矩阵布局

---

## 生成后检验（交付前必须执行）

### 文件基础检验
```bash
# 检验脚本模板
FILE="./{标题}.pptx.html"
echo "1. 文件存在: $(test -f $FILE && echo 'PASS' || echo 'FAIL')"
echo "2. 文件大小: $(ls -la $FILE | awk '{print $5}') bytes"
echo "3. SLIDE SPLIT数: $(grep -c '<!-- SLIDE SPLIT -->' $FILE)"
echo "4. DOCTYPE数: $(grep -c '<!DOCTYPE html>' $FILE)"
echo "5. html标签数: $(grep -c '<html' $FILE)"
echo "6. 控制字符: $(python3 -c \"print(len(open('$FILE','rb').read().translate(None,b'\\x00\\x01\\x02\\x03\\x04\\x05\\x06\\x07\\x08\\x0b\\x0c\\x0e\\x0f\\x10\\x11\\x12\\x13\\x14\\x15\\x16\\x17\\x18\\x19\\x1a\\x1b\\x1c\\x1d\\x1e\\x1f')))\")"
```

- [ ] 文件存在且大小>1KB
- [ ] DOCTYPE数量 = 页数（每页必须是独立HTML文档）
- [ ] 无控制字符（0个）
- [ ] 无Unicode替换字符(�)

### 文字防乱码检验（图文分离模式）
- [ ] 每页有text-layer HTML层覆盖在bg-image之上
- [ ] 中文文字全部通过HTML渲染，未依赖AI生图渲染中文
- [ ] 生图prompt中中文文字量符合决策表限制（封面≤3个短词，内容页0个）
- [ ] 逐页视觉检查：无缺笔画/虚构字/乱码（打开文件翻看）
- [ ] 数据数字精确（与源数据对照）
- [ ] 文字位置与生图设计协调
- [ ] 文字可读性：对比度足够，半透明底色无冲突

### 内容逻辑检验
- [ ] 章节架构与需求一致
- [ ] 关键数据准确（与源数据对照抽查）
- [ ] 无遗漏案例/章节
- [ ] 特定排除/增加内容已执行（grep验证）
- [ ] 页码连续正确

### 风格统一性检验
- [ ] 配色方案整体一致
- [ ] 字体风格无跳跃
- [ ] 布局类型有变化但不违和
- [ ] 封面与内容页风格呼应

## 常见问题速查

| 问题 | 根因 | 解决 |
|------|------|------|
| 只显示1页封面 | HTML结构不合规，所有slide在一个HTML里 | 每个SLIDE SPLIT间必须是完整独立HTML |
| 字体太小 | CSS手写模式信息密度过高 | 用生图模式，prompt中强调"大号""中号" |
| 乱码 | 编码问题或特殊字符 | UTF-8编码，grep检查控制字符 |
| 风格不统一 | 混用不同风格的prompt | 严格使用对应风格模板，不要混搭除非用户要求 |
| 内容遗漏 | 未按用户要求排除/增加 | 生成后grep验证关键词 |
