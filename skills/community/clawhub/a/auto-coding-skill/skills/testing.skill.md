# Testing：测试策略

## 1. Overview（概述）

代码跑通 ≠ 测试足够。本技能定义从正常路径到安全漏洞的全覆盖测试策略，确保改动不仅行为正确，而且在边界、异常和安全维度上没有缺口。

**核心原则**：
- 测试优先级：正确性 > 边界 > 错误 > 安全 > 性能
- 每新增功能覆盖至少：1 个正常路径 + 2 个边界 + 1 个错误路径
- 回归测试不能有新的失败（新旧测试全部通过）

## 2. When to Use（使用条件）

**触发条件**：
- Step 4 测试阶段自动注入
- B 级 / C 级任务强制执行
- A 级任务简化版（至少每个函数一个正常路径测试）

**可跳过特定维度**：
- 纯工具函数（无外部输入）→ 跳安全维度
- 无状态的格式化函数 → 跳边界维度

## 3. Process（执行流程）

### 3.1 测试五维度

```
维度 1: 正常路径（Happy Path）— 最优先
  ├─ 输入正常值 → 验证输出符合预期
  ├─ 覆盖所有功能点
  └─ ❌ 禁止：只测正常路径就说"做完了"

维度 2: 边界条件（Boundary）— 次优先
  ├─ 空值：None/""/[]/{}/0
  ├─ 极值：最大整数、最长的合法字符串、0 和负数边界
  ├─ 边界值：刚好等于限制值、刚好超过限制值
  ├─ 特殊值：空格的字符串、空字节的二进制
  └─ 原则：如果你写了 if 检查某个值 → 必须有测试

维度 3: 错误路径（Error Path）— 不可跳过
  ├─ 无效输入 → 预期抛出/返回错误
  ├─ 依赖故障：数据库不可用、网络超时、文件不存在
  ├─ 并发冲突：竞态条件、重复提交
  └─ 原则：每个错误处理分支 → 至少一个对应的测试

维度 4: 安全漏洞（Security）— 存在用户输入时必须检查
  ├─ SQL 注入：输入 "; DROP TABLE users; --
  ├─ XSS：输入 <script>alert(1)</script>
  ├─ 未验证输入：直接使用用户提供的数据作为文件路径/命令参数
  ├─ 权限绕过：未认证用户访问受保护资源
  ├─ 敏感数据泄露：密码/Token 出现在日志或响应中
  └─ 原则：只要接收外部输入，安全测试不可跳过

维度 5: 回归测试（Regression）
  ├─ 运行项目全部已有测试
  ├─ 确认没有新的失败
  ├─ 确认之前的 bug 没有复活
  └─ ❌ 禁止：只跑新增的测试，跳过回归
```

### 3.2 测试覆盖决策树

```
有外部输入？
  ├─ 是 → 维度 4（安全）不可跳过
  └─ 否 → 维度 4 可跳过（但需说明理由）

有 if/switch/条件分支？
  ├─ 是 → 维度 2（边界）覆盖所有分支
  └─ 否 → 维度 2 至少一个边界测试

有异常处理？
  ├─ 是 → 维度 3（错误路径）覆盖所有 catch
  └─ 否 → 维度 3 至少一个（确认不抛异常就是预期行为）
```

## 4. Risk Scorecard（反借口表）

| # | 纪律 | 常见借口 | 反驳 |
|---|------|---------|------|
| T1 | 正常 + 边界 + 错误都必须测 | "正常能跑就行了" | 正常能跑 ≠ 边界不崩 ≠ 错误不泄露信息 |
| T2 | 安全测试不可跳过 | "这个输入肯定是安全的" | 防御性编程的前提：所有输入都是不安全的 |
| T3 | 回归测试必须全跑 | "就改了一行，不用全跑" | 一行可能影响你没想到的模块 |
| T4 | 测试可独立运行 | "要先跑 test_a 再跑 test_b" | 有顺序依赖的测试 = 不稳定 = 没人跑 |
| T5 | 覆盖不等于质量 | "覆盖率 90% 了" | 90% 覆盖可能只测了 happy path 90 次 |

## 5. Red Flags（危险信号）

- 🚩 新增代码 > 100 行但新增测试 = 0 — 丢失覆盖
- 🚩 测试耗时翻了 ≥ 2 倍 — 可能有循环导入或死循环
- 🚩 覆盖率下降 > 5% — 新增了未测试的路径
- 🚩 安全检查被标记为"可跳过"但有 Web/API 入口 — 绝对不可
- 🚩 回归测试有未知的新失败（不是预期的行为变更）— 引入了回归 bug
- 🚩 测试之间共享可变状态 — 破坏独立运行性

## 6. Verification（自检清单）

**必检（≤5 项）**：
- [ ] 正常路径测试通过（所有核心功能点）
- [ ] 边界条件覆盖：空值、极值、边界值（至少 3 种）
- [ ] 错误路径测试通过：每个错误分支有测试
- [ ] 无安全漏洞：SQL 注入/XSS/未验证输入已测试
- [ ] 回归测试全部通过（无新的失败）

**抽检（≤3 项）**：
- [ ] 测试耗时未翻倍（对比上次运行时间）
- [ ] 覆盖率无明显下降（对比上次 > 5% 下降触发警告）
- [ ] 测试可独立运行（随机顺序通过、无共享状态依赖）

**免检**：测试框架选择、断言库偏好、测试文件组织结构
