# Harness Dev Standards — 你的 AI 代码交付质检官

> ai-acheng 出品 | 每个 AI 开发者必备的质量标准工具

---

## 🦆 一句话介绍

AI 写代码的时代，效率已经不是问题。
问题是 —— 你敢把 AI 写的代码直接交付吗？

**Harness Dev Standards — 给你的代码做一次全面体检，只需要 30 秒。**

---

## ✨ 核心特性

### 🛡️ 6 道质量门禁，一道都不能少

| 门禁 | 检查内容 |
|-----|---------|
| 📋 需求门禁 | 需求完整清晰，无模糊点 |
| 🏗️ 架构门禁 | 文件结构标准化，依赖最小化 |
| 💻 编码门禁 | 语法类型正确，命名规范 |
| 📦 依赖门禁 | 依赖完整无冗余，无安全漏洞 |
| 🌍 环境门禁 | .env.example 完整，敏感信息保护 |
| ✅ 交付门禁 | 项目可构建，README 完整 |

---

### 🤖 10+ 常见问题自动修复

遇到问题不用慌，策略库告诉你怎么修：

- ✅ 依赖版本冲突 → 3 种解决方案
- ✅ TypeScript 类型错误 → 标准修复模板
- ✅ Import 路径错误 → 自动排查流程
- ✅ 启动/构建失败 → 根因定位指南

**超过 100+ 实际项目踩坑总结的修复策略。

---

### 🛠️ 2 个一键运行脚本

```bash
# 依赖检查 — 发现未使用依赖 + 安全漏洞
./scripts/depcheck.sh

# 完整质量扫描 — 5 大项一键跑完
./scripts/quality-scan.sh
```

**30 秒出结果，零配置，谁都会用。**
