## Description: <br>
Responsive Layout helps agents design, implement, and review responsive frontend layouts across mobile-first breakpoints, container queries, fluid grids, data-dense screens, touch targets, and cross-device behavior. <br>

This skill is ready for commercial/non-commercial use. <br>

## Publisher: <br>
[bovinphang](https://clawhub.ai/user/bovinphang) <br>

### License/Terms of Use: <br>
MIT-0 <br>


## Use Case: <br>
Developers, designers, and frontend reviewers use this skill to choose responsive layout strategies, define component behavior across viewports, handle dense UI surfaces, and verify touch, keyboard, safe-area, and orientation behavior. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Optional related packages are separate from this guidance skill and may introduce their own dependencies or behavior if installed. <br>
Mitigation: Evaluate and scan optional related packages independently before installation. <br>
Risk: Responsive layout guidance can still lead to incorrect or unusable UI if applied without viewport, input, and dense-state validation. <br>
Mitigation: Review proposed layout changes and test common breakpoints, touch and keyboard paths, safe areas, orientation changes, and data-dense states before release. <br>
