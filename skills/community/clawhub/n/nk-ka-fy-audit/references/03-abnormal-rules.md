# 异常表备注规则（参考）

对`问题`/`异常`表，逐行扫描备注列，按以下优先级判定。

## 关键词优先级

| 优先级 | 关键词 | 处理 |
|:------:|--------|:----:|
| **P1** | `不返佣` / `不涉及返佣` / `暂不返佣` / `本月不涉及返佣` / `sql交易金额有问题` 或 `有疑问`+`待确认`（同时出现） | ❌ 排除 |
| **P2** | `补发` / `回算` / `补充返佣` / `调账` | ✅ 历史差额调整 |
| **P3** | `本月确认` / `手工核算` / `人工核算` | ✅ 历史差额调整 |
| **P4** | `上月待确认` / `系统未维护` / `手工修改` / `遗留` / `补算` / `已补协议` / `人工补算` / `发放历史分润` / `返佣流程已补` / `sp722` | ✅ 历史差额调整 |
| **默认** | 无匹配（含完全无备注） | ❌ 排除 |

> `有疑问*待确认`：同时包含"有疑问"和"待确认"两个词才匹配，单独"待确认"不匹配。

## 区块标题继承机制

部分月份异常/问题表有多个数据块，第一列（A列）非空且品牌列为空 → 区块标题行 → 下方数据行继承该分类。

| 月份 | Sheet | 区块标题 | 位置 | 继承效果 |
|:----:|-------|:--------:|:----:|---------|
| 202405 | 异常及回算 | `回算` | A12 | 下方莲花物业等继承→包含 |
| 202407 | 异常 | `回算` | A16 | 下方华祥苑等继承→包含 |
| 202411 | 异常 | `补充返佣` | A16 | 下方EP/天天惠等继承→包含 |
| 202501 | 异常 | `调整费率导致的人工核算` | A16 | 下方灵山景区继承→包含 |
| 202509 | 问题 | `特殊补发` | A12 | 下方达慧堂医药继承→包含 |

**解析实现要点：**
1. 先检查A列+品牌列（品牌为空且A列非空→区块标题，更新section_note）
2. 第二次表头行（品牌="品牌"）不更新section_note
3. 数据行有自己备注时，用自己的（优先级高于section_note）
4. section_note如"回算"/"补充返佣"/"特殊补发"→P2；含"人工核算"→P3
5. 不能仅凭关键词"调账"在sheet名中出现就判定全表调账
