# 进化方向总览

10 类技能类型，每类 4-5 个进化方向。每个方向附带典型症状描述，帮用户判断是否该选。

---

## 类型1: 工具封装

封装 CLI/API/系统工具，提供可调用接口。如 `cli-anything-wps`、`a-stock-data`、`futuapi`。

| 方向 | 目标 | 典型症状 |
|------|------|---------|
| **覆盖面** | 增加命令/接口/参数组合 | "这个 skill 不认识某个命令/接口" |
| **鲁棒性** | 异常重试、超时降级、格式兼容 | "遇到网络波动/格式变化就挂了" |
| **精准度** | 参数校验更严格，减少误调用 | "我输入模糊参数它也瞎执行" |
| **速度** | 减少冗余步骤，一次调用解决 | "同样的事要跑好几步才完成" |
| **文档化** | 改善输出格式、错误提示清晰度 | "报错了根本看不懂哪里错" |

---

## 类型2: 领域知识

专业知识判断、准则引用、异常识别。如 `tianchuan-audit-perspective`、`mu-youzhi-perspective`、`financial-calculation-verify`。

| 方向 | 目标 | 典型症状 |
|------|------|---------|
| **深度** | 增加判断点，更细粒度识别 | "skill 覆盖的科目/场景不够全" |
| **精准** | 减少幻觉，降低误判率 | "你反复纠正过 skill 的结论" |
| **边界** | 覆盖更多模板/格式/准则版本 | "换了模板格式 skill 就认不出" |
| **可解释** | 不仅说结论，说推理链和准则依据 | "输出太干，只说有没有问题不说为什么" |
| **时效性** | 更新到最新法规/准则 | "引用的准则还是旧的" |

---

## 类型3: 流程编排

多步骤工作流串联、阶段化任务分解。如 `audit-articulation-review`、`trading-analysis`。

| 方向 | 目标 | 典型症状 |
|------|------|---------|
| **检查点** | 关键节点加用户确认 | "跑偏了才发现，中间没机会纠正" |
| **并行度** | 可并行步骤拆开 | "明明互不依赖的步骤却串行走" |
| **容错** | 某步挂了不影响其他 | "一小步出错整个流程全完蛋" |
| **输出结构化** | 更清晰的结构化结果 | "结果混在一起，没法对比" |
| **进度可视** | 展示当前进度和预估剩余 | "不知道还要跑多久" |

---

## 类型4: 分析评估

评分、审计、合规检查、质量评估。如 `darwin-skill`、`financial-report-minesweeper`。

| 方向 | 目标 | 典型症状 |
|------|------|---------|
| **评分校准** | 权重/阈值与实际反馈对齐 | "打分经常偏高/偏低" |
| **测试覆盖** | 测试用例命中最薄弱点 | "测不出问题但实际用着别扭" |
| **报告质量** | 更清晰的对比和追溯 | "报告太粗糙，看不到细节" |
| **误报控制** | 减少假阳性 | "老报一些不是问题的问题" |
| **漏报控制** | 减少假阴性 | "真有问题的反而没发现" |

---

## 类型5: 创意生成

内容生成、占卜测算、文本创作。如 `liuyao-divination`、`ziwei-doushu`、`bazi`。

| 方向 | 目标 | 典型症状 |
|------|------|---------|
| **多样性** | 避免输出套路化 | "每次结果都差不多" |
| **忠实度** | 更准确复现传统规则 | "推导过程和原规则有偏差" |
| **解释力** | 不仅给结果，说清推导 | "给了结果但不知道为什么是这个结果" |
| **个性化** | 记住用户偏好 | "每次都问同样的偏好设定" |
| **可控性** | 用户能调节参数 | "想换个风格没法调" |

---

## 类型6: 数据采集

爬取、抓取、数据采集与清洗。如 `wechat-article-spider`。

| 方向 | 目标 | 典型症状 |
|------|------|---------|
| **反爬对抗** | 应对目标站点改版 | "站点改版后采不到了" |
| **数据清洗** | 输出更结构化 | "采回来还要手工整理半天" |
| **速率控制** | 不触发限流 | "跑一半被限流了" |
| **覆盖面** | 更多数据源 | "只能从一个地方采" |
| **增量更新** | 只采新增/变化部分 | "每次都全量重采太慢了" |

---

## 类型7: 格式转换

文件格式互转。如 Markdown->Word、PDF->Excel、HTML->结构化数据。

| 方向 | 目标 | 典型症状 |
|------|------|---------|
| **保真度** | 不丢信息、格式不乱 | "转出来表格全错位了" |
| **模板库** | 更多输出样式预置 | "没有我要的输出格式" |
| **批处理** | 一次处理整个文件夹 | "只能一个一个转" |
| **格式检测** | 自动识别输入格式 | "手动指定格式太麻烦" |

---

## 类型8: 安全合规

风险扫描、合规检查、漏洞审计。安全审计、合同审查、隐私合规。

| 方向 | 目标 | 典型症状 |
|------|------|---------|
| **规则库** | 更多检查项、更新法规 | "新出的规定 skill 没覆盖" |
| **误报率** | 减少假阳性 | "老报不是问题的问题" |
| **漏报率** | 减少假阴性 | "明显的问题没检出来" |
| **严重度分级** | 致命/高危/中/低/建议 | "所有问题混在一起没有优先级" |
| **修复建议** | 给具体修复方案 | "只说有问题不说怎么改" |

---

## 类型9: 交互教学

引导式学习、问答训练、渐进教学。

| 方向 | 目标 | 典型症状 |
|------|------|---------|
| **适应性** | 根据回答动态调整难度 | "难度要么太简单要么太难" |
| **反馈精度** | 不仅说对错，说错在哪 | "只说错了没说为什么" |
| **进度追踪** | 记录薄弱点，下次续接 | "不记得上次学到哪了" |
| **知识密度** | 减少废话，提高信息密度 | "讲了一大堆没用的" |

---

## 类型10: 集成桥接

多平台数据流转、服务编排、MCP 封装。如 MCP connector、跨 service 工作流。

| 方向 | 目标 | 典型症状 |
|------|------|---------|
| **可靠性** | 连接超时重试、断点续传 | "偶尔连不上就彻底挂了" |
| **组合度** | 多 connector 串联成流水线 | "每次都要手动分步调" |
| **监控** | 每步状态可追踪 | "不知道哪一步卡住了" |
| **数据一致性** | 跨平台数据对齐 | "两个平台的数据对不上" |

---

## 不确定类型时的诊断问题

当用户不清楚目标 skill 属于哪类时，问：

1. "这个 skill 的主要输出是什么？"（命令结果/专业判断/多步报告/评分/文本/数据/文件/检查项/教学反馈/数据流转）
2. "用户最常纠正这个 skill 的什么问题？"（对应上述症状）
3. 根据回答推荐 1-2 个最匹配的类型，让用户确认。
