# 华为流程管理体系：IPD / LTC / ITR

## 概述

华为有4个一级运作流程，构成以"客户为中心"的价值创造闭环：

```
MTL (市场到线索) → LTC (线索到回款) → ITR (问题到解决)
        ↑                                   |
        └─────── IPD (需求到产品) ←──────────┘
```

## IPD — 集成产品开发 (Integrated Product Development)

**定位**：从客户需求到产品上市的端到端流程

**核心理念**：
- 产品开发是投资行为
- 基于市场的创新
- 跨部门协同（打破部门墙）
- 结构化流程
- 异步开发与重用

**关键阶段**：
1. 概念阶段 — 需求分析、可行性评估
2. 计划阶段 — 项目计划、资源分配
3. 开发阶段 — 产品设计开发
4. 验证阶段 — 测试验证
5. 发布阶段 — 产品上市
6. 生命周期管理 — 持续优化

**关键价值**：
- 将A产品的经验教训制度化传递给B产品
- 避免重复犯错，降低研发成本
- 从职能型组织→流程型组织

## LTC — 线索到回款 (Lead to Cash)

**定位**：端到端的主业务流程，承载公司最大的物流、资金流和人力投入

**核心理念**：
- 从客户视角出发的业务流程重构
- "铁三角"是LTC的核心作战单元
- 将销售从"个人英雄"变为"组织能力"

**三大阶段**：

### ML — 管理线索 (Manage Lead)
- 重点角色：AR（客户经理）牵头
- 客户关系拓展、线索培育
- 线索验证与转化

### MO — 管理机会点 (Manage Opportunity)
- 重点角色：SR（解决方案经理）牵头
- 方案引导、投标管理
- AR持续夯实客户关系
- FR配合输出交付方案

### MCE — 管理合同执行 (Manage Contract Execution)
- 重点角色：FR（交付经理）牵头
- 合同交付、验收管理
- AR和SR配合支持

## ITR — 问题到解决 (Issue to Resolution)

**定位**：从客户问题到彻底解决的服务流程

**核心理念**：
- 某代表处的问题解决了叫"解决"
- 全球此类问题都根治了才叫"关闭"
- 形成服务质量改进闭环

## 三大流程的关系

```
IPD  ─→ 把产品开发出来（从概念到面市）
LTC  ─→ 把产品变成收入（从线索到回款）
ITR  ─→ 把问题彻底解决（从问题到关闭）
```

三者日复一日运行，形成公司年度业绩（财务三张表）。

## 流程管理的核心原则

1. **流程反映业务本质**：流程不是画图，是还原业务本质
2. **以客户为中心**：流程从客户中来，到客户中去
3. **流程Owner**：业务比重最大者成为流程Owner
4. **一张皮运作**：质量、运营、内控、授权、财经要素融入流程
5. **持续改进**：流程改进一小步，绩效改进一大步
6. **模板化+IT化**：简单、海量、重复的工作先流程化再IT支撑
