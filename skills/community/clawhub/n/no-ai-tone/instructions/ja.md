# 日本語：AI生成文体を避けるガイド

> 調査元：note、X（旧Twitter）、Togetter、LifeHacker Japan、AI総合研究所、北京語言大学 ACL論文（多言語比較）、ユーザーによる実例まとめ。

核心的な問題：AI文章の問題は特定の語彙だけではなく、**一定の温度感、過剰な整然さ、断定を避ける態度**にある。日本語のAI文体はさらに「丁寧すぎる」「説明書のよう」という独自の問題を抱えている。

---

## 1. 語彙・表現ブラックリスト

### 1.1 接続詞・繋ぎ言葉（過剰使用のサイン）

これらの語は単独では問題ないが、AI文章では毎段落・高密度で出現する。

**即削除推奨（出現頻度が異常に高い）**
- また、さらに、そして（連続3回以上で即バレ）
- したがって、そのため、結果として
- なお、ちなみに（情報の付け足しに乱用）
- しかしながら、一方で（硬すぎる逆接）
- それでは、続いて（説明書的な進行）
- と同時に（曖昧な同時性、ほぼ何も言っていない）
- つまり（直前と同じことを言い換えるだけのとき）
- 言い換えると（「つまり」の同義語；説明のふりをした繰り返し）

**1記事1回まで**
- 以上が〜です / 以上のように
- 最終的には / 結論として / まとめると
- 前述のように / 上記の通り

### 1.2 語尾パターン（最も目立つAIの特徴）

AI文章は語尾の種類が極端に少ない。「〜です。〜ます。〜です。〜ます。」の連続がテンポを単調にする。

**過剰出現する語尾**
- 〜です。〜ます。（2文以上連続で使わない）
- 〜と言えるでしょう（断定を避ける典型表現）
- 〜と考えられます（責任回避の丁寧語）
- 〜することができます（「〜できます」で十分）
- 〜が重要です / 〜が大切です（空洞的強調）
- 〜ではないでしょうか（疑問形で断定を回避）
- 〜してみてください（指示の弱い言い回し）
- 〜かもしれません（多用すると全部が曖昧になる）
- 〜ようです（伝聞・推測を重ねると全文が霧の中になる）
- 〜らしいです（根拠のない見聞き表現；出典を明記するか削る）

**改善の方向性**
- 体言止め、倒置、短文を積極的に混ぜる
- 「〜できます」「〜します」など直接的な語尾
- 会話的な「〜ですよね」「〜なんです」を適所で使う

### 1.3 責任回避フレーズ（事なかれ主義ブラックリスト）

AI が断定を避けるために多用するパターン。Twitterユーザーが「何か言っているようで何も言っていない」と批判する表現群。

```
❌ 一概には言えませんが
❌ ケースバイケースです
❌ 場合によります
❌ 価値観によって異なります
❌ メリットもあればデメリットもあります
❌ 一般的には〜と言われています
❌ 〜と言われています（出典なし）
❌ 専門家によると（誰の専門家か不明）
❌ あまり推奨されません（誰が推奨しないのか？）
❌ 注意が必要です（何に、どう注意するのか？）
```

### 1.4 ビジネス抽象語（「本質感」「最適化感」問題）

```
本質的・本質を押さえる
最適化・最適解
価値を最大化する
革新・イノベーション
エンゲージメント
インサイト・アウトプット
スケーラビリティ・オンボーディング
包括的・網羅的
持続可能な
シナジー
DX（デジタルトランスフォーメーション）← 「何を変えるのか」を書く
カスタマージャーニー ← 具体的な接点と行動を書く
フレームワーク ← 何のフレームワークか、中身を書く
ステークホルダー ← 誰のことか名指しする
アジリティ・レジリエンス ← 何に対して速く/強くなるのか書く
```

→ これらは具体的な動詞と数字で置き換える。「最適化します」→「処理時間を半分にします」。

### 1.5 定型比喩（AI御用達の比喩ブラックリスト）

AIが好む比喩は種類が極端に少なく、繰り返し使われることで「既製品感」を醸し出す。

```

扉を開く（機会の比喩；使われすぎ）
種をまく（将来投資の比喩；使われすぎ）
```

## 2. 文構造の問題

### 2.1 「3点列挙」中毒

日本語AI文章の最大の特徴のひとつ。「3つのポイント」「5つのステップ」という整然とした構造が、説明書感を生む。

```
❌ 本サービスの特徴は以下の3点です。
   1. 使いやすさ
   2. 高い信頼性
   3. 充実したサポート

✓ 一番使ってもらっているのは、通知が来ない点だと思います。
  ユーザーが自分のペースで確認できるので。
```

**チェックポイント**：「〜の3つ」「以下のN点」という表現が出たら、本当に列挙が必要かを確認する。

### 2.2 「〜については以下の通りです」進行

```
❌ 使い方については以下の手順を参照してください。
   ステップ1：〜
   ステップ2：〜
   ステップ3：〜

❌ それでは、次のセクションでは〜について詳しく解説します。
```

→ 内容をそのまま書く。「次に〜を解説します」は読者の時間を奪うだけ。

### 2.3 文末の単調なリズム（節拍器問題）

すべての文が同じ長さ・同じ語尾・同じテンションで並ぶ。

```
❌ AIは近年、急速に発展しています。多くの企業が活用を始めています。
   業務効率が大きく改善されています。今後もさらなる進化が期待されます。

✓ AIが現場に入ってきた。想像していたより静かに。
  「気づいたら使っていた」という声が一番多い。
```

→ 短文・長文・体言止めを意識的に交互に混ぜる。

### 2.4 前置き長すぎ問題

```
❌ 近年、デジタル化の急速な進展に伴い、多くの企業において業務効率化が
   重要な課題となっています。このような背景の中、本記事では〜について
   詳しく解説いたします。

✓ 結論から言います：週次レポートを3時間から20分に短縮できます。
```

## 3. 語気・スタンスの問題

### 3.1 「同じ温度」問題（最も根本的な違和感）

Twitterユーザーの言葉：「スクロールしても、どこを読んでも同じ温度で、感情の起伏がない」

AI文章は全体を通じて：
- 重要なことも、どうでもいいことも、同じ調子で書かれる
- 驚きがない、失望がない、怒りがない
- 褒めるときも「〜は優れています」という一定の温度

→ 本当に大事なことには、他と違う書き方をする。短くする、改行を変える、語気を変える。

### 3.2 「事なかれ」スタンス

AI は批判・反論・失敗談を避ける。

```
❌ この方法にはメリットもあればデメリットもあります。（何も言っていない）
❌ 個人の状況によって最適な選択は異なります。（逃げている）

✓ 正直、最初の設計は失敗でした。
  ユーザーに怒られてから作り直しました。
```

→ 何かに反対している、何かを選んでいる、何かに失敗したことがある文章は信頼できる。

### 3.3 「やたら持ち上げる」問題

Togetterまとめで指摘された特徴：「やたら相手を持ち上げる」

```
❌ 素晴らしいご質問ありがとうございます。
❌ おっしゃる通り、その観点は非常に重要です。
❌ 大変興味深いテーマを取り上げていただきました。
```

→ 反応そのものは省略する。内容から始める。

### 3.4 過剰な正確さが逆効果

「誤字がない」「文法が完璧」「句読点が必ずある」——これらが人間味のなさのサインになっている。

→ 一人称の小さなズレ、やや崩した表現、文章として完成していない一文は「人がいる」感を生む。

## 4. レイアウト・記号の問題（「豆腐体」）

日本語AI文章独自のビジュアル特徴：

```
❌ 各段落のはじめに絵文字
❌ **太字強調** の多用（Markdownがそのまま出る）
❌ 全ての見出しが「○○とは何か」「○○する3つの方法」形式
❌ 箇条書きが完璧に揃っている（同じ文字数、同じ語尾）
❌ 「以上」で締める
❌ コロン「：」の英語的な使い方
```

**「説明書感」の正体**：箇条書きのすべての項目が同じ長さで、同じ語尾で終わっている。人が書く箇条書きは長さがバラバラで、語尾もまちまち。

## 5. 内容の問題

### 5.1 一次体験の欠如

Togetter・noteユーザーが最も指摘する特徴：「体験談がない」「人の温度がない」

```
❌ このサービスを使うことで、多くのユーザーが生産性の向上を実感しています。

✓ 使い始めた月、残業が週4時間減りました。
  理由はよくわかりません。でも減りました。
```

感情も抽象的に書かれる：
```
❌ 大変感動しました。心から嬉しく思います。
✓ 胸が熱くなって、思わず画面から目を離した。
```

→ 「〜と感じた」ではなく「〜した」という動作で感情を表す。

### 5.2 「空洞的強調」問題

```
❌ これは非常に重要なポイントです。
❌ 大きなメリットがあります。
❌ 特に注目すべき点があります。
```

→ 強調するなら、なぜ重要かを具体的に書く。「重要です」と言わずに、重要だとわかる内容を書く。

### 5.3 カタカナ語の多用

AI生成の日本語には不自然なカタカナ語が増える傾向がある：

```
インサイト・アウトプット・フィードバック・ソリューション・
エンゲージメント・オンボーディング・スケーラビリティ・
コンバージョン・モニタリング・プロアクティブ
```

→ 日本語で言えるものは日本語で書く。カタカナは固有名詞や定着した略語に限定する。

### 5.4 完璧な論理が「迷い」のなさを露呈する

「1段落削除してもスムーズに読める」——これが構造が完璧すぎる証拠。

人が書く文章には：「脱線」「補足しすぎた一文」「言い直し」がある。完璧に整理されすぎた文章は、生きていない。

## 6. 改稿チェックリスト

日本語文案の修正時、以下の順序で確認する：

1. **語尾スキャン** ── 「〜です。〜ます。」が3連続していないか確認し、体言止め・短文で割る
2. **接続詞削除** ── 「また」「さらに」「したがって」を数え、半数を削除する
3. **断定チェック** ── 「〜でしょう」「〜と考えられます」を「〜です」「〜します」に変える、または削る
4. **体験確認** ── 具体的な数字・エピソード・失敗談が1箇所でもあるか確認する
5. **比喩チェック** ── §1.5 の定型比喩リストに該当がないか確認する
6. **視点統一** ── 「私」「私たち」「あなた」が一文中で混在していないか確認する
7. **音読テスト** ── 声に出して読み、「説明書みたい」と感じたら書き直す

## 7. 改稿例

### サービス紹介

```
❌ AI腔の原文：
本サービスは、最新のAI技術を活用し、業務効率化を包括的にサポートします。
使いやすいインターフェースと充実したサポート体制により、
導入から運用まで安心してご利用いただけます。
また、柔軟なカスタマイズ機能により、さまざまなニーズに対応可能です。

✓ 改稿後：
導入した翌週、「会議の前に読む資料」が消えました。
理由はシンプルで、前日夜に自動で全員の手元に届くからです。
```

### ブランドメッセージ

```
❌ AI腔の原文：

✓ 改稿後：
前職で、同じ問い合わせに1日30回答えていました。
そのとき作ったのが、このツールの原型です。
```

## 8. 自己チェック三問

文章を書いたら、この三問で確認する：

1. **「読める」か「感じられる」か？** ── 情報として読めるが何も残らない文章より、一文だけ心に刺さる文章の方が価値がある
2. **誰が言っている？** ── この文章は、具体的な誰かが言いそうか。「関係者一同」が書いたような文章なら書き直す
3. **何を諦めたか？** ── 良い文章はすべてを言わない。何かを言わないことで、残ったことが際立つ
