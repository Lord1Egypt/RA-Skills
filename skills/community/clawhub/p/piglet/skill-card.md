## Description: <br>
Practical Python craftsmanship guidance based on One Python Craftsman for writing, refactoring, and reviewing Python code. <br>

This skill is ready for commercial/non-commercial use. <br>

## Publisher: <br>
[PsiACE](https://clawhub.ai/user/PsiACE) <br>

### License/Terms of Use: <br>
MIT-0 <br>


## Use Case: <br>
Developers and engineers use this skill to apply practical Python craftsmanship guidance during code creation, refactoring, and review. It focuses on naming, branching, data handling, functions, exceptions, loops, decorators, imports, file I/O, edge cases, SOLID design, and modern syntax choices. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Suggested code changes may be incorrect or unsuitable for a specific codebase. <br>
Mitigation: Review recommendations before applying them, and run the project's normal tests and code review checks. <br>
Risk: Examples involving file writes, renames, network requests, or database-like operations could be copied into sensitive contexts without adaptation. <br>
Mitigation: Check side effects, permissions, paths, network behavior, and data handling before using such examples in production code. <br>
