## Draft Lookup Window Fix

While validating BadClaw after adding checkpoint verification, `status.mjs --limit 10` exposed a useful edge case: the latest run was healthy, but the real pending draft was older than the 10-run health window, so compact status could report zero visible drafts even though `review-drafts --limit 50` still showed the pending draft.

Updated:

- `status.mjs` now supports `--draft-limit <n>`
- default draft lookup window is `max(limit, 50)`
- health/latest-run inspection can stay compact while draft queue inspection remains wide enough to catch older pending drafts
- `operator-console.mjs` now supports `--draft-limit <n>` and passes that wider window to both status and draft-packet lookup

This keeps operator output from accidentally hiding older pending drafts while still preserving the recent-run health window.

## Operator Preflight Gate

Added `scripts/stay-alive/preflight.mjs` as a one-command operator preflight gate.

Behavior:

- runs `status.mjs`, `operator-console.mjs`, and `audit-report.mjs`
- creates one local-only checkpoint by default for traceability

- JSON output is compact by default; `--include-raw` can embed full underlying status/operator/audit/verify payloads

Also updated `checkpoint.mjs` checkpoint ids to keep millisecond precision, avoiding same-second checkpoint overwrites during repeated preflight runs.
