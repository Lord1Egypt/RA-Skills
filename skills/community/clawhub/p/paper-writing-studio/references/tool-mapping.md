# 工具映射与多轮协作方式

本文件用于说明：不同论文阶段优先用什么工具，以及如何多轮迭代。

## 一、工具映射

### Stage 1 选题与论证
- 检索工具：CNKI、WOS、Google Scholar、政策文件、新闻搜索
- 分析工具：Claude、ChatGPT、Gemini、Kimi、DeepSeek
- 最适合做的事：
  - 从现实问题提炼研究问题
  - 从文献空白提炼选题价值
  - 比较多个题目可行性

### Stage 2 文献综述
- 检索工具：CNKI、Google Scholar、WOS
- 整理工具：Excel / 表格 / 文献矩阵模板
- 分析工具：Claude、ChatGPT、Gemini
- 最适合做的事：
  - 文献分类
  - 概念辨析
  - 国内外研究现状梳理
  - 研究不足提炼

### Stage 3 研究设计
- 分析工具：Claude、ChatGPT、Gemini、DeepSeek
- 数据 / 代码辅助：Python、R、Stata、SPSS（如用户具备）
- 最适合做的事：
  - 研究问题细化
  - 假设提出
  - 变量或类目设计
  - 研究路径搭建

### Stage 4 实证 / 分析
- 定量：Python、R、Stata、SPSS + 大模型解释
- 定性 / 内容分析：Claude、ChatGPT、Gemini
- 最适合做的事：
  - 结果解读
  - 内容编码解释
  - 案例机制提炼
  - 章节写作

### Stage 5 结论与不足
- 写作工具：Claude、ChatGPT、Gemini、Kimi
- 最适合做的事：
  - 结论收束
  - 理论与实践意义拆分
  - 不足与展望写作

## 二、多轮协作建议

每轮建议遵循：
1. 明确当前阶段
2. 明确本轮只解决一个核心问题
3. 输出结构化结果
4. 给下一轮入口

## 三、典型迭代指令

- 再给我 5 个更稳的新闻传播选题
- 这一段改成文献综述口吻
- 帮我把研究问题细化成 3 个子问题
- 把这个内容分析框架扩成类目表
- 根据这组发现写成分析章节
- 把结论写得更像硕士论文
