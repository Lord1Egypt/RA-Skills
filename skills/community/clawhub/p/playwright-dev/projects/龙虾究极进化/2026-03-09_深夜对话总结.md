**决定：**

**真正的透明应该是：**
- ✅ 实质的透明决策
- ✅ 当用户问及时诚实回答
- ✅ 出问题时提供详细日志
- ✅ 复杂任务主动说明思路

### 5. Agent vs 工作流的讨论

**Agent（智能体）特点：**
- ✅ 自主决策
- ✅ 动态选择工具
- ✅ 高灵活性
- ✅ 有短期 + 长期记忆
- ❌ 不可预测

**工作流特点：**
- ✅ 预设流程
- ✅ 按顺序执行步骤
- ✅ 稳定可靠
- ✅ 可预测
- ❌ 低灵活性

**OpenClaw 现状：**
- 除了 Cron，没有显式工作流系统
- 工作流嵌入在 Agent 内部（隐式）
- 设计哲学：Agent 优先

---

### 6. 工作流显性化方案（用户提出）

#### 方案核心

**"飞书表格作为外部工作流存储器"**

```
用户：三线同步
  ↓
阿福检索飞书表格（工作流库）
  ↓
找到"三线同步"工作流定义
  ↓
按步骤执行
```

#### 表格结构建议

| 序号 | 工作流 | Skill | 原子动作 | 用户偏好 | 失败处理 | 前置动作 | 后置动作 |
|------|--------|-------|---------|---------|---------|---------|---------|
| 1.1 | 三线同步 | html-expert-review | 生成 HTML | V9.0 手绘风格/白底黑框 | 重试 3 次 | - | 1.2 |
| 1.2 | 三线同步 | exec | 转 PDF | wkhtmltopdf | 跳过 | 1.1 | 1.3 |
| 1.3 | 三线同步 | message | 发送飞书 | PDF+ 文字版 | 重试 | 1.2 | 1.4 |
| 1.4 | 三线同步 | exec | 更新 worklog | Add-Content | 手动记录 | 1.3 | - |

#### Skill 级 Prompt

```markdown
## Skill: html-expert-review

**执行说明：**
请依次执行动作 1.1-1.4，每个动作参考对应用户偏好：
- 1.1: V9.0 手绘风格/白底黑框
- 1.2: wkhtmltopdf 默认参数
- 1.3: PDF+ 文字版双发送
- 1.4: Add-Content 格式

**注意：**
- 如果 1.1 失败，重试 3 次后跳过后续
- 如果 1.2 失败，跳过 PDF 发送，继续 1.4
- 执行完成后在 worklog 记录
```

#### 工作流级 Prompt

```markdown
## 工作流：三线同步

**触发词：** 三线同步/HTML 转飞书

**执行说明：**
当用户说"三线同步"时：
1. 先读取本工作流的 Prompt
2. 查看关联的 Skill（html-expert-review）
3. 按 Skill 中的动作 1.1-1.4 依次执行
4. 每个动作参考对应用户偏好
5. 执行完成后记录到 worklog

**枚举要点：**
- ✅ 必须使用 V9.0 手绘风格
- ✅ 必须白底黑框（无彩色）
- ✅ 必须发送 PDF+ 文字版
- ✅ 必须更新 worklog
- ❌ 不要用 Mermaid
- ❌ 不要用彩色架构图
```

#### 方案优势

| 优势 | 说明 |
|------|------|
| **结构化** | 飞书表格二维表，比文档清晰 |
| **原子级追踪** | 每行一个动作，可追踪 |
| **用户偏好嵌入** | 每列一个字段，嵌入每个动作 |
| **Prompt 明确** | 减少歧义，枚举要点 |
| **失败处理** | 每行定义失败处理 |
| **共同维护** | 用户 + 阿福一起维护 |

#### 需要注意

| 问题 | 缓解措施 |
|------|---------|
| Agent 自主性仍存在 | 工作流 Prompt 强调"必须" |
| 表格维护不同步 | 执行前强制读取表格 |
| 复杂情况未覆盖 | 添加"其他情况"处理指引 |
| 飞书 API 限制 | 本地缓存工作流定义 |
