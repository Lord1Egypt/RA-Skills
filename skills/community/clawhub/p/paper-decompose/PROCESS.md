# paper-decompose 处理流程

> **版本**：v1.0.0 | 2026-06-26
> **目的**：详细描述 7 步处理流程，确保输出质量下限

---

## 完整处理流程

### Step 1：识别输入类型

**目标**：判断输入属于哪种模式，选择对应的处理策略。

| 模式 | 触发条件 | 处理重点 |
|------|---------|----------|
| `cn-report` | 中文研报（券商/行业研究/白皮书） | 重投资视角；Layer 2 强调"对行业逻辑的改变" |
| `en-paper` | 英文论文（arXiv/Nature/ACL 等） | 重方法论创新；Layer 2 强调"技术机制改进" |
| `article` | 公众号长文/技术博客 | 重观点提炼；Layer 2 强调"作者的核心论点" |

**识别技巧**：
- 看 URL 域名（券商网站/arxiv.org/公众号）
- 看文档格式（PDF 含数据图表 vs 文本含代码）
- 看引用风格（学术引用 vs 媒体引用）

---

### Step 2：缺口层（GAP）

**目标**：明确文档要解决的核心问题，梳理该领域 3-5 个既往方案的不足。

#### 2.1 识别核心问题

问自己：
- 这篇文档第一段/摘要讲的核心问题是什么？
- 作者为什么认为这个问题重要？
- 如果这个问题不解决，会发生什么？

**输出格式**：
```
问题：[用一句话陈述核心问题]
重要性：[为什么这个问题值得关注]
不解决的后果：[现实世界中会怎样]
```

#### 2.2 梳理既往方案

找 3-5 个该领域的主流方案/方法/思路。来源：
- 论文本身的 Related Work 部分
- 该领域的综述文章
- 行业里的主流做法

**输出格式**：
```
既往方案1：[方案名] + [核心思路] + [不足]
既往方案2：[方案名] + [核心思路] + [不足]
既往方案3：[方案名] + [核心思路] + [不足]
```

#### 2.3 战略位置

明确本文填的是哪个具体缺口：
```
本文战略位置：[在 [X 问题] 的 [Y 子方向] 上，针对 [Z 缺陷] 提出 [W 方案]
```

**示例**：
> 本文战略位置：在"大模型推理速度"问题上，针对"自回归生成的串行瓶颈"，提出"投机解码 + 草稿模型"方案。

---

### Step 3：增量层（DELTA）

**目标**：用一句话结构化表达作者的核心创新，**聚焦"挪动了什么"而非"做了什么"**。

#### 3.1 一句话结构化公式

```
核心增量 = 在 [X 问题] 上，将 [Y 组件/步骤/机制] 从 [A 旧方案] 改为 [B 新方案]，通过 [M 机制] 实现 [N 收益]
```

**正例**：
- ✅ "在大模型推理问题上，将生成方式从『自回归逐 token』改为『草稿验证并行』，通过小型草稿模型预生成 + 大模型批量验证，实现 2-3 倍速度提升"
- ✅ "在量化投资选股问题上，将因子筛选从『单因子打分』改为『多因子正交降秩』，通过 PCA 提取 3-5 个独立驱动要素，降低过拟合风险"

**反例**：
- ❌ "这篇论文用了一个新模型"（太模糊）
- ❌ "作者做了很多实验"（没有结构化）
- ❌ "效果比之前好"（没有对比基线）

#### 3.2 核心机制

解释这个增量背后的关键技术/方法（2-3 句话）：
```
核心机制：
1. [机制 1 的关键点]
2. [机制 2 的关键点]
3. [机制之间的协同]
```

#### 3.3 对比维度

相对前作的提升幅度（用数字说话）：
```
对比维度：
- 精度：+X%
- 速度：+X 倍
- 成本：-X%
- 可解释性：[改善/未变/变差]
```

### Step 4：Napkin Sketch 层（架构对比图）

**目标**：用 ASCII 双列图直观对比旧方法 vs 新方法，让读者一眼看见架构差异。

#### 4.1 ASCII 图规范

**必含要素**：
- 双列布局（旧方法 vs 新方法）
- 3-7 个步骤的流程对比
- 关键差异用 `→` 或 `*` 标注
- 边框用 `┌─┐ │ └─┘` 字符

**模板**：
```
  旧方法                      新方法
┌──────────────┐          ┌──────────────┐
│   步骤1      │          │   步骤1'     │ *
├──────────────┤          ├──────────────┤
│   步骤2      │    →     │   步骤2'     │
├──────────────┤          ├──────────────┤
│   步骤3      │          │   步骤3'     │ *
└──────────────┘          └──────────────┘
```

#### 4.2 画图步骤

1. **提取流程**：从文档中提取 3-7 个关键步骤
2. **标差异**：找出哪些步骤被修改（标 `'`）
3. **画连线**：用 `→` 表达整体架构方向
4. **标星号**：关键创新点用 `*` 标记

#### 4.3 复杂场景处理

**场景 A：架构完全不同**
```
  旧方法（流水线）         新方法（端到端）
┌──────────────┐        ┌──────────────────┐
│  模块 A      │        │                  │
├──────────────┤        │  端到端模型      │
│  模块 B      │   →    │  (A+B+C 融合)    │
├──────────────┤        │                  │
│  模块 C      │        │                  │
└──────────────┘        └──────────────────┘
```

**场景 B：关键模块替换**
```
  旧方法                      新方法
┌──────────────┐          ┌──────────────┐
│  输入预处理  │          │  输入预处理  │
├──────────────┤          ├──────────────┤
│  特征工程    │    →     │  特征工程    │
│  (手工)      │          │  (自动学习)  │ *
├──────────────┤          ├──────────────┤
│  分类器      │          │  分类器      │
└──────────────┘          └──────────────┘
```

### Step 5：启发层（INSPIRATION）

**目标**：通过 Inversion（反转）和 Mapping（迁移）两种思路，把核心机制跨域复用。

#### 5.1 Inversion（反转）

**问题**：如果把核心机制反向用，会发生什么？

**思路**：
- 把"加速"反转成"减速"会怎样？
- 把"并行"反转成"串行"会怎样？
- 把"加法"反转成"减法"会怎样？
- 把"集中"反转成"分散"会怎样？

**输出**：
```
Inversion 1：[反向应用 1] + [会发生什么]
Inversion 2：[反向应用 2] + [会发生什么]
```

**示例**（针对"投机解码"）：
> Inversion：把"大模型验证草稿"反转为"草稿验证大模型"——即用小模型反向筛选大模型的输出，可能发现大模型的"自信错误"。

#### 5.2 Mapping（迁移）

**问题**：把核心机制搬到其他领域，能怎么用？

**思路**：
- 这个机制的本质是什么？（抽象出来）
- 在其他领域有没有类似结构？
- 跨领域套用会怎样？

**输出**：
```
Mapping 1：搬到 [领域 A]，[应用场景]
Mapping 2：搬到 [领域 B]，[应用场景]
```

**示例**（针对"多因子降秩"）：
> Mapping 1：搬到"个人技能评估"——把多项技能降维到 3-5 个核心能力，找到杠杆点
> Mapping 2：搬到"决策分析"——把多因素决策降维到 2-3 个关键变量，避免决策瘫痪

### Step 6：善人特化 - 投资影响评估（可选）

**目标**：在 Layer 2 和 Layer 4 之间，评估本文对投资决策的影响。

#### 6.1 影响维度评估

| 维度 | 评估问题 | 影响等级 |
|------|---------|----------|
| **基本面** | 是否影响持仓标的基本面（行业逻辑改变）？ | 强 / 中 / 弱 / 无 |
| **估值** | 是否改变市场对该标的的估值方法？ | 强 / 中 / 弱 / 无 |
| **资金面** | 是否引发资金流入/流出（北向/机构调仓）？ | 强 / 中 / 弱 / 无 |
| **政策面** | 是否涉及监管/政策风险？ | 强 / 中 / 弱 / 无 |

#### 6.2 持仓建议

```
影响标：[具体股票/行业代码]
当前持仓：[有/无/关注中]
建议操作：加仓 / 持有 / 减仓 / 观望
依据：[基于上述评估的具体原因]
```

**触发条件**：
- 仅当用户启用"投资特化模式"或文档为投资类研报时执行
- 默认关闭，保持通用性

---

### Step 7：输出校验

**目标**：确保 4 层都非空，ASCII 图必须包含新旧对比。

#### 7.1 必检项

- [ ] Layer 1（缺口）：核心问题 + 至少 3 个既往方案
- [ ] Layer 2（增量）：一句话结构化 + 核心机制 + 对比维度
- [ ] Layer 3（Napkin）：ASCII 双列图，必须新旧对比
- [ ] Layer 4（启发）：Inversion 至少 1 个 + Mapping 至少 2 个
- [ ] （可选）投资影响评估

#### 7.2 质量校验

- [ ] 一句话结构化清晰（能用一句话说清增量）
- [ ] ASCII 图可读（3-7 个步骤，不超过 10 行）
- [ ] Inversion 不是简单"反过来"（要有具体场景）
- [ ] Mapping 跨域有效（不是同义改写）

#### 7.3 失败回退

如果某层无法填充：
- Layer 1 缺失 → 文档可能不是研究类，提示用户
- Layer 2 缺失 → 文档可能没有实质创新，提示"无明显增量"
- Layer 3 缺失 → 文档结构化程度低，强制画架构图
- Layer 4 缺失 → 文档应用性弱，简化处理
