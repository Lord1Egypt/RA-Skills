# 金融分析师 Skill | Financial Analyst Skill

版本：`1.0.0`

## 简介

这是一个面向金融投资分析与决策判断的技能包。

它适用于：

- 市场方向判断与趋势分析
- 行业/板块研究与赛道选择
- 个股/标的估值与基本面分析
- 持仓复盘与组合诊断
- 交易策略评估与仓位管理
- 风险管理与杠杆使用评估

## 设计思想

本项目融合了两套核心方法论：

### 1. 尉缭子分析法的结构力

来自 [尉缭子分析法 Skill](https://github.com/phoenixlucky/weiliaozi-skill)，贡献了：

- **按顺序想，不跳步** 的分析纪律
- **本质 → 条件 → 得失 → 先后 → 对手** 的固定分析顺序（金融版本扩展为六栏）
- **系统拆解** 视角：先看底层结构，再判断动作
- **条件式判断**：所有结论必须建立在条件和约束之上
- **博弈意识**：市场对手会动，最大的对手是周期

### 2. 商业数据分析师的严谨性

来自 [商业数据分析师 Skill](https://github.com/phoenixlucky/business-data-analyst-skill)，贡献了：

- **先定义问题，再分析** 的工作习惯
- **统一口径** 的指标纪律
- **先验数据，后解释** 的质量意识
- **事实与推断分开放** 的准确规则
- **结论必须对应动作** 的输出要求

### 3. 金融投资的周期思维

本项目独有的核心框架：

- **趋势优先，叙事优先，共识优先** — 但必须在周期定位和风险约束下使用
- **三变量检查**：流动性 × 叙事 × 盈利增长
- **周期定位优先**：先判断牛/熊/震荡市，再决定策略
- **仓位管理 > 方向判断**：方向错了还能活，仓位错了会出局
- **杠杆纪律**：杠杆前的正确顺序是 周期 → 策略 → 仓位 → 杠杆

## 核心分析框架

### 六栏分析

1. **周期定位** — 先看宏观、市场和估值水位
2. **本质与驱动** — 三变量检查：流动性 × 叙事 × 盈利
3. **条件与约束** — 自身条件、外部条件、硬约束
4. **得失与风险** — 收益、成本、最坏情况
5. **先后与节奏** — 顺序、仓位、杠杆纪律
6. **博弈与对手** — 机构、散户、周期、博弈路径

### 领域模块

根据问题类型，可引入以下领域模块：

- **宏观与市场方向分析** — 经济周期、利率、资金流向
- **行业/板块分析** — 生命周期、渗透率、竞争格局
- **个股/标的分析** — 财务健康、估值、护城河
- **持仓与组合复盘** — 归因分析、集中度、纪律检查
- **交易策略评估** — 策略类型、胜率盈亏比、执行纪律

## 适合什么场景

这个 Skill 适合：

- "现在能不能买 / 卖？"
- "AI 板块还值不值得追？"
- "这只股票 PE 这么高，贵不贵？"
- "我的持仓该怎么调整？"
- "要不要加杠杆？"
- "大盘现在是顶还是底？"
- "这个行业值不值得研究？"
- "帮我复盘一下最近的交易。"

它尤其适合下面这种情况：

- 市场情绪高涨，不知道该不该跟
- 持盈怕回撤、空仓怕踏空
- 想在趋势策略和价值判断之间找到平衡
- 需要把投资问题从情绪驱动切换为分析驱动

## 输出格式

默认输出六栏结构，每栏 3-5 个关键点：

- **周期定位**
- **本质与驱动**
- **条件与约束**
- **得失与风险**
- **先后与节奏**
- **博弈与对手**
- **判断一句**
- **建议动作**
- **关键风险**

## 快速模式

如果用户只需要快速判断，最少保留四段：

1. **周期定位** — 一句话周期位置
2. **驱动检查** — 三变量检查
3. **关键风险** — 最需要注意的风险
4. **动作建议** — 做/不做/怎么做

## 使用方式

```text
帮我分析一下 AI 板块现在还能不能追，用金融分析师框架分析。
```

```text
某公司 PE 80 倍，帮我用六栏框架分析一下估值是否合理。
```

```text
我重仓了某科技股，浮盈 40%，帮我做持仓复盘，给调仓建议。
```

```text
现在能不能加杠杆买纳指？
```

## 回答语言

- 默认根据用户的提问语言回答
- 中文问用中文，英文问用英文

## 局限

- 不提供具体价格目标或买卖点预测
- 不替代专业投资顾问的法律义务
- 需要足够的市场数据才能做高质量判断，信息不足时应标注信息缺口
- 依赖用户提供的基础数据质量（财报数据、持仓数据等）
