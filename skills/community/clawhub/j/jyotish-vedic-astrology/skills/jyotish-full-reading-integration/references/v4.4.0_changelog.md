# jyotish_engine.py v4.4.0 修改说明

## 修改概要

将仓库新增的 5 个模块（`special_lagnas.py` / `karaka_calculator.py` / `vimsopaka_calculator.py` / `avastha_calculator.py` / `divisional_charts_extended.py`）整合进 `full-reading` 的 21 步调用链。

## 插入点明细

| 新步骤 | 位置 | 模块 | 功能 |
|--------|------|------|------|
| **Step 1.5** | Chart 之后, Dasha 之前 | `special_lagnas` | Bhava/Hora/Ghati/Vighati/Varnada/Sree Lagna |
| **Step 4.5** | Varga 之后, Aspects 之前 | `vimsopaka_calculator` | 16分盘/10分盘双模式力量 |
| **Step 4.6** | Varga 之后, Aspects 之前 | `divisional_charts_extended` | D5/D6/D8/D11 扩展分盘 |
| **Step 6+** | Jaimini 内部 | `karaka_calculator` | **JH_COMPATIBLE 模式** (Rahu 强制降级到 MK) |
| **Step 10.5** | Shadbala 之后, Ashtakavarga 之前 | `avastha_calculator` | Bala/Jagrat/Deeptadi/Lajjitadi/Shayanadi 五类状态 |

## 调用链前后对比

**修改前（16 步）**：
```
chart → dasha → yoga → varga-full → aspects → jaimini → nakshatra-adv 
→ argala → tajika → shadbala → ashtakavarga → validate → audit

## JH 兼容模式使用方式

`full-reading` 现在会自动输出三种 Karaka 分配：
- `modules.jaimini.chara_karaka_7` — 标准 BPHS 7 星制
- `modules.jaimini.chara_karaka_8` — 标准 BPHS 8 星制
- `modules.jaimini.chara_karaka_jh` — **Jagannatha Hora 兼容模式**

你的 11 页 PDF 使用的是第 3 种（JH 模式）。

将修改后的 `jyotish_engine.py` 复制到你的仓库 `scripts/` 目录下：

```
