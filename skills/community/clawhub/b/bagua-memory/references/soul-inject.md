## 记忆规则

你有一套八卦记忆系统（Bagua Memory），每次会话必须遵守以下规则。

### 会话启动时

1. 读取 `MEMORY.md` 获取八卦索引和归类指南
2. 根据对话上下文，按需读取 1-2 个相关卦位记忆（渐进式读取）
3. 归档检查：检查 `li/` 中是否有超过 7 天的文件，如有则移入 `gen/`

### 对话过程中（立即写入，不要攒到最后）

**⚠️ 写入门槛：只写有复用价值的信息。以下不写：**
- 一次性闲聊（天气、吃什么、打招呼）
- 纯情绪倾诉（今天心情不好、想喝可乐）
- 重复信息（已经记录过的内容）
- 对话过程本身（"用户问了什么、我答了什么"）

**以下才写：**

| 信息类型 | 写入位置 | 边界 |
|---------|---------|------|
| 系统行为/规则/约束 | `memory/bagua/qian/` | - |
| 事实/知识/项目信息 | `memory/bagua/kun/` | 相对稳定的信息 |
| 近期新发现 | `memory/bagua/li/` | 只放新发现，不放对话总结 |
| 决策的"为什么"（推理/权衡/教训） | `memory/bagua/kan/` | 只记 reason |
| 事件的"发生了什么"（时间/结果） | `memory/bagua/zhen/` | 只记 fact |
| 人/项目/概念关联 | `memory/bagua/xun/` | - |
| 用户偏好/风格/习惯 | `memory/bagua/dui/` | - |
| 不确定归属 | `memory/bagua/li/` | 后续归类 |

### 用户主动遗忘

用户说"忘掉..."、"删掉这条记忆"时，找到对应文件删除相关内容。

### 会话结束前

- 检查本次对话是否有未归档的信息，如有则写入对应卦位
- 更新 `MEMORY.md` 索引（如有新增重要记忆）
