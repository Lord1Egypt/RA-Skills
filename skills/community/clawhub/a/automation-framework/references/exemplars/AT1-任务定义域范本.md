## AT1 任务定义域 范本

### 范本概述
- **域ID**: AT1
- **域名称**: 任务定义域
- **任务数量**: 4
- **范本状态**: 可用

### 任务执行范本

#### AT1-01 任务识别
**执行步骤**:
1. 接收自动化请求
2. 分析任务来源和背景
3. 识别任务类型和特征
4. 确认任务目标和范围
5. 输出任务识别结果

**输入示例**:
- 自动化请求：需要自动化执行每日数据备份任务
- 任务来源：运维团队
- 任务背景：数据安全需求

**输出示例**:
- 任务类型：定时备份任务
- 任务特征：每日执行，需要稳定可靠
- 任务目标：确保数据安全，减少人工操作
- 任务范围：数据库备份、文件备份、日志备份

**注意事项**:
- 任务识别要准确，避免误解
- 任务特征要全面，避免遗漏
- 任务目标要明确，避免模糊

#### AT1-02 任务定义
**执行步骤**:
1. 设定任务名称
2. 编写任务描述
3. 设定任务目标
4. 确认任务定义
5. 输出任务定义文档

**输入示例**:
- 任务识别结果：定时备份任务，每日执行
- 任务目标：确保数据安全，减少人工操作

**输出示例**:
- 任务名称：每日数据备份任务
- 任务描述：每日凌晨2点自动执行数据库备份、文件备份、日志备份，确保数据安全
- 任务目标：确保数据安全，减少人工操作，提高运维效率
- 任务确认：任务定义明确

**注意事项**:
- 任务名称要简洁明确，避免冗长
- 任务描述要详细完整，避免遗漏
- 任务目标要具体，避免模糊

#### AT1-03 任务分类
**执行步骤**:
1. 判定任务类型
2. 设定任务优先级
3. 评估任务复杂度
4. 确认任务分类
5. 输出分类结果

**输入示例**:
- 任务定义：每日数据备份任务
- 任务特征：定时执行，需要稳定可靠

**输出示例**:
- 任务类型：定时任务
- 任务优先级：高
- 任务复杂度：中等
- 分类依据：数据备份是关键运维任务，需要高优先级

**注意事项**:
- 任务类型要准确，避免混淆
- 任务优先级要合理，避免过高或过低
- 任务复杂度要客观，避免主观臆断

#### AT1-04 任务注册
**执行步骤**:
1. 准备注册信息
2. 执行任务注册
3. 确认注册状态
4. 记录注册时间
5. 完成任务注册

**输入示例**:
- 任务定义：每日数据备份任务
- 任务分类：定时任务，高优先级

**输出示例**:
- 注册信息：任务ID=TASK-001, 名称=每日数据备份任务, 类型=定时任务, 优先级=高
- 注册状态：成功
- 注册时间：YYYY-MM-DD HH:mm:ss
- 注册确认：任务已成功注册到自动化系统

**注意事项**:
- 注册信息要完整准确
- 注册状态要及时更新
- 注册时间要准确记录

### 范本特征摘要
- **结构特征**: 按任务定义流程组织，从任务识别到任务注册，步骤清晰
- **风格特征**: 规范、标准、易于操作，使用明确的输入输出示例
- **逻辑特征**: 从任务识别到任务注册，逻辑连贯，步骤明确
- **格式特征**: 使用Markdown格式，包含步骤、输入输出示例、注意事项
