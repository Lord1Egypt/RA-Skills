# 行业校准指南

## 校准机制

当用户首次使用本技能时，需识别其所属行业并注入行业特定知识。校准不修改框架，只在执行时替换行业术语、数据维度、法规引用和流程特征。

## 校准流程

1. **识别行业**：从用户输入中提取行业关键词
2. **匹配校准模板**：在下表中找到对应行业
3. **注入校准上下文**：将行业特定信息作为执行上下文的一部分
4. **自动适配**：后续所有单元自动使用校准后的术语和维度

## 已内置校准模板

> **分组逻辑**：按商业模式共性归类。未覆盖行业可先找最近分组做近似校准，再微调。

| 分组 | 行业 | 共性逻辑 |
|------|------|----------|
| 消费与零售 | 电商、快消/零售、餐饮 | C端驱动、SKU管理、复购逻辑 |
| 平台与数字 | 互联网/平台、SaaS | 双边市场/订阅制、留存为王、迭代快 |
| 专业与知识 | 咨询、教育 | 人即产能、知识即产品、项目制 |
| 制造与工业 | 制造、汽车 | 产业链长、重资产、产能/良率核心 |
| 强监管 | 医药、金融 | 合规即命门、牌照/审批驱动、风控先行 |
| 资产运营 | 房地产 | 周期长、资金密集、政策敏感 |

## 自定义行业校准

如果用户行业不在已内置列表中，按以下流程生成校准：

1. **就近匹配**：先在上表分组中找到商业模式最接近的分组
2. **询问行业名称和3-5个核心业务场景**
3. 基于场景和近似分组推断各校准维度
4. 生成校准模板并注入上下文
5. 在执行中持续校准和修正
