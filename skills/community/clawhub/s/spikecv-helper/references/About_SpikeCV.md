# More About SpikeCV

*last updated: 2026-04-22*

**SpikeCV** is an open-source project for `spike camera`, mainly including spiking datasets, algorithms library and hardware interface.

---

## Introduction

### spike camera

Mimicking the sampling mechanism of the fovea of primate retina, a retina-inspired camera, named **spike camera**, outputs asynchronous binary spike streams by continuously capturing photons. The photon accumulation reaches a preset threshold, a spike stream is generated. Generally, 1 means a spike is fired, and 0 means no spike. The difference from the traditional camera imaging. The traditional digital camera records scenes with a constant shutter speed (e.g., 33 ms), which loses much visual information and suffers motion blur. The spike camera can record the external information with a sampling rate of 40 Khz, and retains high-speed spatio-temporal information for both moving and static objects, which can be applied to tasks as acquisition, detection, tracking and recognition of ultra-high-speed scenes.

### Architecture of SpikeCV

SpikeCV mainly consists of three parts：

1. **Datasets**：
   - `Raw Spikes`: Off-line files of spike streams that generated by spike cameras or simulators.
   - `HFR Videos`: High-Frame-Rate  (HFR) Videos at the same sampling frequency as the spike camera,  which can be used to test the ultra-high-speed scene reconstruction algorithm based on the spike camera
   - `Optical Flow`: Ground truth optical flow information of the simulated scenes
   - `Stereo Depth`: Depth information of scenes captured by binocular and monocular spike cameras
   - `Object Labels`: Class labels of objects in the scene
   - `Bounding Boxes`: Candidate boxes represent the location and size of moving/foreground objects, which can be used for tasks such as detection and tracking
2. **Algorithm Library**：
   - `SpkData`: Mainly used to load data generated by the spike camera, Ground Truth (GT) information corresponding to spike stream, and simulate spikes generation
   - `SpkProc`: Models and algorithms for spike cameras, including *Filters*, *Reconstruction algorithm*, *Optical Flow (OF) Estimation*, *Depth Estimation*, *Object Detection*, *Multi-target Tracking* and *Recognition* and other modules
   - **Tool**：
     - `Visualization` —— Visualization of spike flow or results of various tasks
     - `Metrics` —— Quantitative Evaluation Metrics for the algorithms
     - `Utils` —— Auxiliary development tools, such as data path processing, etc.
3. **Device Interface**: Providing spike camera driver and supporting data acquisition, real-time data processing interface. Supported for the currently released spike cameras, or use offline files for simulated camera data debugging.

> **Further Reading:** For a comprehensive overview, see the SpikeCV team's publication: [*SpikeCV: open a continuous computer vision era*](https://link.springer.com/article/10.1007/s11432-023-4565-6).


## Resources

* **Official webpage** of SpikeCV: [spikecv](https://spikecv.github.io/index.html) . You would need it if:
   + You want to know more or get in touch with the SpikeCV team
   + You wanna check out the spikecv-related events or competitions hold by SpikeCV team
   + You wanna learn more about SpikeCV and spike camera, including a further understanding of the project architecture, research topics, and their publications

* **Joining and Contributing** SpikeCV community: you should check out the contributing workflow on the [webpage](https://spikecv.github.io/join.html) 

* **Publications**: If you want further links to **publications and researches** conducted by SpikeCV team, check out [this](https://spikecv.github.io/publications.html). **This official publication list is a comprehensive and up-to-date source**; any other lists found elsewhere may be incomplete or outdated. You would need this when confronting some difficult question about certain algorithm that required in-depth reading of the Paper (you can further user request to the publication listed out in this webpage).

* **Code Repository**: You can check out the `spikecv` python library from [**github repo**](https://github.com/Zyj061/SpikeCV), where integrate many classical implementation of Algorithm and provided Device interface and Data processing Modules that help you read spike data and perform visual task on spike data.

* **Documentation**: To learn more about the SpikeCV library, please refer to the **documentation** at [readthedocs](https://spikecv.readthedocs.io) . You would need it if:
+ You want to know the API of a certain function/class/method provided in `spikecv` library, which can be downloaded through GitHub
+ You want to take a look at some code example of the `spikecv` library 

* **Datasets**: You might want to check out **spike datasets** provided by the team, which can be download directly from [OpenI](https://openi.pcl.ac.cn/Cordium/SpikeCV/datasets) . You can also take a look at the datasets they provided for the competitions WUJI_INF (which hold in IJCAI2025) at [OpenI](https://openi.pcl.ac.cn/IJCAl2025)

* **Hardware Acquisition**: If you wonder "where to buy a spike camera?", you should check out [spikesee](https://spikesee.com/), a company producing high quality **spike camera**.
