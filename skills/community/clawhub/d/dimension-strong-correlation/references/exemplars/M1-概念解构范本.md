# M1 概念解构域范本

## M1-01 概念分析

### 输入示例
概念A：创新，概念B：传统

### 处理过程
1. 概念定义：创新=引入新事物，传统=长期存在的做法
2. 内涵分析：创新强调新颖性，传统强调稳定性
3. 外延分析：创新包括技术创新、管理创新等，传统包括文化传统、工艺传统等
4. 概念特征：创新vs传统的对立统一

### 输出示例
```markdown
## 概念分析报告

### 概念定义
| 概念 | 定义 | 核心特征 |
|------|------|----------|
| 创新 | 引入新事物 | 新颖性、突破性 |
| 传统 | 长期存在的做法 | 稳定性、延续性 |

### 内涵分析
- **创新**：强调新颖性、突破性、变革性
- **传统**：强调稳定性、延续性、传承性

### 外延分析
- **创新**：技术创新、管理创新、模式创新
- **传统**：文化传统、工艺传统、习俗传统

### 概念特征
- **对立性**：创新vs传统
- **统一性**：创新基于传统，传统需要创新
```

## M1-02 维度识别

### 输入示例
概念：创新、传统

### 处理过程
1. 维度识别：时间维度、空间维度、主体维度、内容维度、方式维度
2. 维度特征分析：每个维度的特征
3. 维度分类：核心维度、辅助维度

### 输出示例
```markdown
## 维度识别报告

### 维度列表
| 维度ID | 维度名称 | 维度类型 | 维度特征 |
|--------|----------|----------|----------|
| D1 | 时间维度 | 核心维度 | 新旧程度 |
| D2 | 空间维度 | 核心维度 | 地域范围 |
| D3 | 主体维度 | 辅助维度 | 实施主体 |
| D4 | 内容维度 | 核心维度 | 具体内容 |
| D5 | 方式维度 | 辅助维度 | 实施方式 |

### 维度特征
- **时间维度**：衡量新旧程度，创新偏新，传统偏旧
- **空间维度**：衡量地域范围，创新可能全球，传统可能本地
- **主体维度**：衡量实施主体，创新可能是企业，传统可能是社区
- **内容维度**：衡量具体内容，创新是新事物，传统是旧事物
- **方式维度**：衡量实施方式，创新是突破，传统是延续
```

## M1-03 维度定义

### 输入示例
维度：时间维度、空间维度、主体维度、内容维度、方式维度

### 处理过程
1. 维度名称确定：时间维度、空间维度等
2. 维度定义：明确定义每个维度
3. 维度范围确定：确定每个维度的范围
4. 维度度量设定：设定度量标准

### 输出示例
```markdown
## 维度定义报告

### 维度定义
| 维度 | 定义 | 范围 | 度量 |
|------|------|------|------|
| 时间维度 | 新旧程度 | 0-100 | 0=最旧，100=最新 |
| 空间维度 | 地域范围 | 本地-全球 | 本地、区域、国家、全球 |
| 主体维度 | 实施主体 | 个人-组织 | 个人、团队、企业、社会 |
| 内容维度 | 具体内容 | 技术-文化 | 技术、管理、文化、制度 |
| 方式维度 | 实施方式 | 渐进-突破 | 渐进、改良、革命、颠覆 |

### 维度关系
- **时间维度与空间维度**：新事物可能全球传播
- **主体维度与内容维度**：不同主体关注不同内容
```

## M1-04 维度关系

### 输入示例
维度：时间、空间、主体、内容、方式

### 处理过程
1. 关系识别：识别维度间的关系
2. 关系类型：相关关系、因果关系、对立关系
3. 关系强度：强、中、弱
4. 关系图：绘制维度关系图

### 输出示例
```markdown
## 维度关系报告

### 关系识别
| 源维度 | 目标维度 | 关系类型 | 关系强度 |
|--------|----------|----------|----------|
| 时间 | 空间 | 相关关系 | 中 |
| 主体 | 内容 | 因果关系 | 强 |
| 内容 | 方式 | 相关关系 | 中 |
| 时间 | 方式 | 对立关系 | 强 |

### 关系分析
- **时间与空间**：新事物可能快速传播到全球
- **主体与内容**：不同主体决定创新内容
- **内容与方式**：内容决定实施方式
- **时间与方式**：新事物需要新方式

### 关系图
```
时间 ──→ 空间
  │
  ↓
方式 ←── 内容 ←── 主体
```
```

## M1-05 解构报告

### 输入示例
解构结果：创新和传统解构为5个维度

### 处理过程
1. 报告目标：汇总概念解构结果
2. 报告内容：概念分析、维度识别、维度定义、维度关系
3. 报告格式：结构化Markdown
4. 报告生成：生成最终报告

### 输出示例
```markdown
## 解构报告

### 概念分析
- **概念A**：创新（新颖性、突破性）
- **概念B**：传统（稳定性、延续性）

### 维度识别
- **维度数量**：5个
- **核心维度**：时间、空间、内容
- **辅助维度**：主体、方式

### 维度定义
- **时间维度**：新旧程度（0-100）
- **空间维度**：地域范围（本地-全球）
- **主体维度**：实施主体（个人-组织）
- **内容维度**：具体内容（技术-文化）
- **方式维度**：实施方式（渐进-突破）

### 维度关系
- **主要关系**：主体→内容→方式
- **次要关系**：时间→空间

### 结论
**结论**：创新和传统可以解构为5个维度，形成维度矩阵
```
