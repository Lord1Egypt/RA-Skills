# Weekly Report Generator - 周报自动生成

## 技能描述

基于 worklog.txt 工作日志，每周三下午自动生成正式周报（workreport.txt），支持增量更新、历史保留、格式标准化。

## 触发条件

- **定时触发**：每周三 15:00（Cron 任务）
- **手动触发**：用户说"生成周报"、"写周报"、"本周总结"
- **更新触发**：用户说"更新周报"、"刷新周报"

## 核心能力

1. **worklog 解析** - 提取本周工作内容（按日期过滤）
2. **周报生成** - 按标准格式生成 workreport.txt
3. **增量更新** - 保留历史周报，追加新周报
4. **豆包会话整合** - 从豆包会话提取工作内容
5. **项目知识库同步** - 刷新项目 workload
6. **格式标准化** - 统一语言风格、信息颗粒度

## 周报结构标准

### 固定栏目

**供应商保供数据运营**（固定更新内容）：
- SDC（Supplier Daily Capacity）数据收集
- GP9 实际产能 vs 计划产能
- LCR（本地化率）统计
- 产能达标率预警

**本周新做事项**：
- 来源 1：豆包会话（会议记录、对话总结）
- 来源 2：worklog.txt 日常记录
- 来源 3：项目知识库更新

**Previous Workload**（历史工作参考）：
- 参考上周周报内容
- 保持信息颗粒度一致
- 语言风格统一

### 标准格式

```markdown
## 2026 年第 X 周 (2026-03-XX)

### 本周完成

1. [项目名] 具体工作内容 - 成果/进展
2. [项目名] 具体工作内容 - 成果/进展

### 进行中

1. [项目名] 工作内容 - 当前状态 - 下一步计划

### 下周计划

1. [项目名] 计划工作内容 - 预期成果

### 问题与风险

1. [项目名] 问题描述 - 影响 - 建议方案
```

## 工作流程

### 标准流程（周三 15:00）

```
1. 读取 worklog.txt - 提取本周记录
2. 读取豆包会话 - 提取会议记录（如有）
3. 读取项目知识库 - 提取项目进展
4. 生成周报草稿 - workreport.txt
5. 增量追加 - 保留历史周报
6. 发送通知 - 飞书提醒用户审阅
7. 用户编辑 - 手动完善措辞
8. 提交领导 - 发送正式周报
```

### 豆包会话整合流程

```
1. 读取豆包会话 - doubao-sessions/日期-序号.md
2. 提取工作内容 - 会议记录、决策、行动项
3. 分类整理 - 按项目/任务分类
4. 更新 worklog.txt - 添加豆包提取的内容
5. 同步项目知识库 - 刷新项目 workload
6. 生成周报 - 包含豆包内容
```

## 输入源

### 主要输入

| 来源 | 文件/位置 | 说明 |
|------|----------|------|
| **日常记录** | `worklog.txt` | 每日工作内容（主要来源） |
| **豆包会话** | `doubao-sessions/*.md` | 会议记录、对话总结 |
| **项目知识库** | `tasks/projects/*.md` | 项目进展、workload 更新 |
| **历史周报** | `workreport.txt` | Previous Workload 参考 |

### 豆包会话提取规则

**触发词：** "豆包" 开头的消息

**自动流程：**
1. 保存原始内容 → `doubao-sessions/日期 - 序号.md`
2. 提取工作内容 → 更新 `worklog.txt`
3. 同步项目知识库 → 刷新对应项目 workload
4. 生成专家点评 → `expert-review-日期 - 主题.html`

## 增量更新策略

### worklog.txt 更新

**规则：**
- ✅ 追加模式 - 不覆盖已有内容
- ✅ 按日期组织 - 每天一个章节
- ✅ 状态标记 - [完成]/[进行中]/[待办]

**示例：**
```markdown
### 2026-03-07 (周六)
- [完成] 周报 Skill 创建
- [进行中] 豆包会话整合
- [待办] 领导审阅
```

### workreport.txt 更新

**规则：**
- ✅ 保留历史周报 - 不删除旧内容
- ✅ 追加新周报 - 添加到文件开头
- ✅ 分隔线区分 - `---` 分隔不同周报

**示例：**
```markdown
# Work Report - 周报

---

## 2026 年第 10 周 (2026-03-05)
[新周报内容]

---

## 2026 年第 9 周 (2026-02-26)
[历史周报内容]
```

## 信息颗粒度规范

### 语言风格

**正式但清晰：**
- ✅ "完成产能监控系统架构设计"
- ❌ "弄了个产能监控的东西"

**具体可量化：**
- ✅ "创建 11 个 Feishu 任务（t100042-t100052）"
- ❌ "创建了一些任务"

**项目导向：**
- ✅ "[感知与行动中心] 完成 v2.1 版本修订"
- ❌ "修改了文档"

### 内容标准

**必须包含：**
- 项目名称/标识
- 具体工作内容
- 成果/进展状态
- 量化指标（如有）

**可选包含：**
- 遇到的问题
- 下一步计划
- 需要支持的事项

## Cron 任务配置

### 周三周报生成

**配置：**
```json
{
  "name": "周三周报生成",
  "schedule": {
    "kind": "cron",
    "expr": "0 15 * * 3",
    "tz": "Asia/Shanghai"
  },
  "payload": {
    "kind": "systemEvent",
    "text": "生成周报"
  },
  "sessionTarget": "main"
}
```

**触发时间：** 每周三 15:00（Asia/Shanghai）

**执行流程：**
1. 读取 worklog.txt（本周内容）
2. 读取豆包会话（如有）
3. 生成周报草稿
4. 飞书发送通知："周报已生成，请审阅"
5. 用户手动编辑完善
6. 提交领导

## 项目知识库同步

### 刷新项目 workload

**流程：**
1. 识别本周涉及的项目
2. 读取项目卡片（`tasks/projects/*.md`）
3. 更新「项目进度」章节
4. 添加本周工作内容
5. 更新「最后更新时间」

**示例：**
```markdown
### 项目进度

**2026-03-07 更新：**
- 完成周报 Skill 创建
- 整合豆包会话流程
- 配置 Cron 定时任务
```

2. **历史保留** - 所有历史周报都要保留
3. **豆包整合** - 豆包会话内容要同步到 worklog
4. **项目同步** - 项目 workload 要及时刷新
5. **周三定时** - 每周三 15:00 自动生成
