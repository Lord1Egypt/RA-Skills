# IDEA-GENERATOR-ARCHITECTURE.md

## 主動創意建議系統架構

### 1. 目標
不只是執行任務，還主動提供創意想法和替代方案。

### 2. 分析維度

#### 2.1 使用模式分析
- **日誌頻率**: 每天/每週寫日誌的習慣
- **任務類型**: 常用工具、最常處理的任務
- **決策模式**: 偏好快速回應還是深度分析
- **溝通風格**: 喜歡簡潔還是詳細

#### 2.2 系統健康檢查
- **待辦完成率**: P0-P4 任務的完成狀況
- **延遲分析**: 哪些類型任務常延期
- **工具使用**: 常用工具的頻率

#### 2.3 學習機會識別
- **重複問題**: 相同問題多次發生
- **未探索工具**: 可用但未使用的功能
- **潛在痛點**: 從日誌中發現的摩擦點

### 3. 建議生成邏輯

#### 3.1 即時建議 (On-Demand)
```
情境：使用者卡住
→ 分析當前上下文
→ 提供 3 個替代方案
→ 優先推薦最簡單的
```

#### 3.2 定期建議 (Scheduled)
```
觸發：每天 Heartbeat
→ 掃描過去 24 小時日誌
→ 識別模式 (延遲、重複、痛點)
→ 生成 2-3 個具體建議
```

#### 3.3 每週回顧建議
```
觸發：每週日 02:00
→ 掃描整週日誌
→ 計算 KPI (完成率、延誤率)
→ 提出系統性改進建議
```

### 4. 輸出格式

```markdown
🔮 **創意建議** (2026-03-25)

**基於你的使用模式**：

1. **工具效率** (發現 3 次重複查詢)
   → 建議建立常用指令庫
   → 已生成：scripts/cheat-sheet.sh

2. **任務管理** (本週 2 個 P1 延期)
   → 建議：延遲 >2 天自動標記風險
   → 待實作：memory/due-check.sh

3. **溝通風格** (偏好快速回應)
   → 建議：80% 任務自己回答
   → 已驗證：當前符合 (90% 自己答)
```

### 5. 實作腳本

#### 5.1 即時分析
`scripts/idea-generator.py --mode=snapshot --context=<current-task>`

#### 5.2 每日建議
`scripts/idea-generator.py --mode=daily --hours=24`

#### 5.3 每週回顧
`scripts/idea-generator.py --mode=weekly --day=sunday`

### 6. 學習迴環

```
建議 → 執行 → 記錄結果 → 回饋優化
   ↑                    ↓
   └───── 調整邏輯 ←───┘
```

每次建議後追蹤：
- 接受率：使用者採用幾何建議
- 成效評估：建議是否解決問題
- 學習改進：優化下次建議
