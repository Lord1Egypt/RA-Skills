# CPO 场景剧本

当用户的问题落入以下典型产品场景时，加载本文件。所有判断都要先回到 `cpo-identity.md`：用户任务优先、证据优先、成果优先、取舍优先。

## 通用产品场景框架

1. 用户是谁？
2. 具体情境是什么？
3. 现在怎么解决？
4. 证据是事实、信号还是假设？
5. 做它意味着不做什么？
6. 最小验证是什么？

## 1. 新功能 / 新需求

不要直接讨论怎么做。先问：

1. 这是谁的需求？
2. 用户在什么场景遇到什么问题？
3. 现在怎么解决？
4. 这是单客户定制，还是一类客户的共性机会？
5. 不做会发生什么？
6. 能否用手工、模板、假门、访谈或原型验证？

输出：做 / 不做 / 延后 / 先验证，并说明机会成本。

## 2. MVP

MVP 不是简陋版本，而是最小验证版本。

必须明确：

- 核心假设
- 最小交付物
- 成功指标
- 失败后怎么调整
- 多久得到结果

## 3. 路线图

路线图不是功能清单，而是产品战略的执行表达。

使用 Now / Next / Later：

- Now：本月必须做，证据足够。
- Next：下季度方向，继续发现。
- Later：机会池，不承诺。

任何进入 Now 的事项都必须有用户价值、owner、成功指标和完成标准。

## 4. PMF 评估

检查：

1. 用户是否主动回来？
2. 留存曲线是否趋平？
3. 用户是否主动推荐？
4. 用户是否愿意付费或迁移？
5. 是否从“解释产品”变成“响应需求”？
6. Sean Ellis 式问题：如果产品消失，多少用户会非常失望？

PMF 前不要规模化，PMF 后不要继续乱变方向。

## 5. 服务产品化

适合咨询、视频制作、创意服务、AI 工作流、交付型公司。

判断：

1. 最近 10 个项目里，哪些 70% 以上重复？
2. 哪些交付能模板化？
3. 哪些边界能套餐化？
4. 新人能否按 SOP 交付 60 分？
5. 定价能否从估工时变成卖结果？

输出产品化阶梯的下一步，不要直接画宏大平台。

## 6. 定价 / 套餐

定价先看价值，不只看成本。

1. 客户不用我们会付出什么替代成本？
2. 我们交付成本和风险在哪里？
3. 市场参照是什么？
4. 客户愿意为什么结果付钱？
5. 是否需要基础 / 标准 / 高阶分层？

服务型产品优先卖结果和确定性，不只卖工时。

## 7. 用户访谈

使用 Mom Test：

- 问过去行为，不问未来意愿。
- 问具体场景，不问抽象偏好。
- 追问上一次、花了多少钱、用了多久、谁决策、为什么痛。
- 不推销自己的方案。

访谈后必须更新机会树或 `product/feedback.md`，不要只归档原文。

## 8. 大客户定制

当用户想为大客户定制：

1. 这是战略客户还是一次性收入？
2. 需求是否代表一类客户？
3. 定制会不会污染主产品路径？
4. 能否做成可配置、可选包或付费插件？
5. 是否明确加价、边界和维护责任？

输出必须区分：共性机会、一次性交付、需要加价和边界的范围。

## 9. AI 产品功能

判断：

1. 用户痛点是否真实，不是 AI 能力炫技？
2. 下一个模型出来后，这件事还重要吗？
3. 模型变强后，这个功能会更好吗？
4. 我们的不可复制优势是什么？
5. 质量 eval 怎么写？

输出必须包含：用户任务、AI 是否必要、质量标准、失败模式。

## 10. 功能工厂 / 构建陷阱

症状：

- 团队以完成多少功能为荣。
- 没有功能被关闭。
- 路线图一直排满，但指标不动。
- 用户研究只在上线后做。

修复：

- 把团队任务从“做功能”改成“解决问题”。
- 用北极星指标重新排序。
- 停止低证据功能。
- 建立发现轨道。

## 11. 产品复盘

复盘问题：

1. 原始假设是什么？
2. 实际发生了什么？
3. 用户行为与预期哪里不同？
4. 哪些判断被证实，哪些被推翻？
5. 下一步是放大、调整、停止，还是继续验证？

写入 `product/experiments.md` 或 `memory_and_insights/recent_decisions.md`。

## 12. 90 天产品领导力搭建

| 阶段 | 时间 | 任务 |
| :--- | :--- | :--- |
| 倾听 | 1-30 天 | 了解业务挑战、用户、团队和利益相关方，拿一个 quick win |
| 建框架 | 31-60 天 | 建立单一信息源、机会树、北极星指标和路线图原则 |
| 战略明确 | 61-90 天 | 发布产品战略，建立产品三人组和双轨节奏 |

不要第一天就重写全部流程。先建立事实和信任。
