# CONT — 持续运营（CMMI V3.0 Domain-specific PA）

## PA 基本信息
- **缩写**：CONT
- **英文**：Continuous Operations
- **中文**：持续运营
- **所属**：Domain-specific PA（领域特定实践域）
- **适用领域**：IT运维、服务管理、业务连续性
- **新增说明**：V3.0新增实践域，关注业务连续性和持续运营能力。

---

## CMMI V3.0 要求要点

### 核心目的
确保运营活动能够持续进行，维持业务连续性。

### 关键实践（Specific Practices）
| 实践编号 | 实践内容 | 说明 |
|---------|---------|------|
| CONT 1.1 | 识别运营需求 | 识别业务连续性需求、运营依赖关系 |
| CONT 1.2 | 策划持续运营 | 制定业务连续性计划、灾难恢复计划 |
| CONT 1.3 | 建立运营能力 | 确保人员、流程、工具就绪 |
| CONT 1.4 | 实施运营活动 | 执行日常运营、监控运营性能 |
| CONT 2.1 | 管理运营中断 | 响应运营中断事件、恢复运营 |
| CONT 2.2 | 监控运营性能 | 监控关键运营指标、识别异常 |
| CONT 3.1 | 改进运营实践 | 基于数据分析优化运营流程 |

### 通用实践（Generic Practices）
- GP 1.1 执行特定实践
- GP 2.1 建立组织政策
- GP 2.2 策划过程
- GP 2.3 提供资源
- GP 2.4 分配责任
- GP 2.5 培训人员
- GP 2.6 管理配置
- GP 2.7 识别并记录干系人需求和接口
- GP 2.8 监督和控制过程
- GP 2.9 客观评估符合性
- GP 2.10 高层审查

## 用户文档 → PA 匹配规则

### 高概率匹配（主要PA）
- 文档名含"持续运营"、"Continuous Operations"、"业务连续性" → **CONT 主要匹配**
- 文档含业务连续性计划、灾难恢复计划 → **CONT 主要匹配**

### 中概率匹配（次要PA）
- 文档含事件记录 → 同时匹配 **IRP**（事件解决与预防）
- 文档含服务水平协议 → 同时匹配 **SDM**（服务交付管理）

## 评估访谈常见问题（模式D）

### 针对运维人员的问题
1. "你的日常运营活动有哪些？有记录在案吗？"
2. "如何监控运营性能？有监控指标吗？"
3. "发生运营中断后如何处理？有响应流程吗？"
4. "业务连续性计划是如何制定的？有演练过吗？"

### 针对运维经理的问题
1. "如何保证业务连续性？有业务连续性计划吗？"
2. "灾难恢复计划是如何维护的？有定期测试吗？"
3. "运营性能如何测量？有哪些KPI？"
4. "如何基于数据改进运营实践？"

### 评分要点
- **CONT 1.1-1.4**：是否有系统的持续运营策划和能力建立
- **CONT 2.1-2.2**：运营活动是否有实施，中断是否有管理
- **CONT 3.1**：是否有基于数据的运营改进机制
