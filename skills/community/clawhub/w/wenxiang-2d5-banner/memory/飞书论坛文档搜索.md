### 1.4 关键安装步骤摘要（来自飞书官方教程）

**前提条件：**
- OpenClaw 版本：Linux/MacOS ≥ 2026.2.26，Windows ≥ 2026.3.2
- 查看版本：`openclaw -v`
- 升级命令：`npm install -g openclaw`

**安装流程：**
1. 创建飞书机器人（飞书开放平台 → 创建企业自建应用）
2. 批量导入权限（使用官方提供的 JSON 权限模板）
3. 配置事件订阅（使用长连接接收事件，添加 `im.message.receive_v1`）
4. 发布应用并获取 App ID 和 App Secret
5. 安装插件：`npx -y https://sf3-cn.feishucdn.com/obj/open-platform-opendoc/879b06f872058309ef70f49bcd38a71f_Pr8pNIJ9J9.tgz install`
6. 运行 OpenClaw：`openclaw gateway run`
7. 完成配对：`openclaw pairing approve feishu --notify`

**诊断命令：**
- `/feishu start` - 确认是否安装成功
- `/feishu doctor` - 检查配置是否正常
- `/feishu auth` - 批量完成用户授权

### 3.2 工作流配置示例

**三种工作流自动化场景（来自 xugj520.cn）：**

1. **消息→表格自动归档**
   - 飞书群消息 → OpenClaw 提取关键信息 → 自动填充到飞书表格
   - 适用场景：客户反馈、问题跟踪、日报收集

2. **定时任务→数据更新**
   - Cron 定时任务（如每日 10:00）→ 自动抓取销售数据 → 填充飞书表格
   - 适用场景：销售报表、库存管理、项目进度跟踪

3. **条件触发→自动通知**
   - 飞书表格数据变化（如库存<50）→ 自动标红 → 发送飞书消息通知负责人
   - 适用场景：库存预警、任务截止提醒、异常监控

**配置命令：**
```bash
# 开启流式输出
openclaw config set channels.feishu.streaming true

# 开启多任务并行、独立上下文
openclaw config set channels.feishu.threadSession true

# 设置@机器人回复策略
openclaw config set channels.feishu.requireMention true

# 查看飞书插件状态
/feishu start
```

### 3.3 飞书官方插件 vs OpenClaw 内置插件对比

| 维度 | 飞书官方插件 | OpenClaw 内置插件 |
|------|-------------|------------------|
| **操作身份** | 以你本人身份（OAuth） | 以机器人身份 |
| **消息** | 读取、发送、回复、搜索 | 读取、发送、回复 |
| **文档** | 创建 + 编辑 + 读取 | 读取为主 |
| **多维表格** | ✅ 完整操作（表、字段、记录、视图） | ✅ 基础读写 |
| **日历日程** | ✅ 创建/查询/修改/搜索/忙闲查询 | ❌ |
| **任务管理** | ✅ 创建/查询/更新/完成/子任务 | ❌ |
| **知识库** | ✅ 完整读写 | ✅ 只读 |
| **安装复杂度** | 需要额外安装 CLI 工具 + OAuth 授权 | OpenClaw 内置，`openclaw channels add` 即可 |
| **维护方** | 飞书团队 | OpenClaw 社区 |

**选择建议：**
- 选**飞书官方插件**：想让 AI 帮你操作飞书（写文档、建表、约会议），需要日历、任务管理等高级功能

### 4.2 常见问题排查清单

**问题 1：飞书插件加载失败**
- 检查事件订阅是否配置（使用长连接，添加 `im.message.receive_v1`）
- 确认网关已启动后再配置事件订阅
- 查看日志：`openclaw logs --follow`

**问题 2：API 配额耗尽**
- 原因：OpenClaw health check 每 60 秒调用一次 bot/v3/info API
- 解决：禁用不需要的飞书渠道 `openclaw config set channels.feishu.enabled false`

**问题 3：图片/文件上传失败**
- 检查权限：必须有 `im:resource` 权限
- 在飞书开放平台补权限后，创建新版本 → 发布
- 重启网关：`openclaw gateway restart`

**问题 4：机器人不回复**
- 默认需要@机器人才会回复
- 确认机器人已被添加到群聊
- 检查 groupPolicy 配置

**问题 5：WebSocket 频繁断开**
- 网络波动导致，确保 open.feishu.cn 走直连（不走代理）
- 检查 DNS 配置
- 日志报 400 错误时，通过 NO_PROXY 环境变量排除飞书域名
