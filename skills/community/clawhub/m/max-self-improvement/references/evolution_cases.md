## 案例 1: 成功模式提取

**类型**: success
**时间**: 2026-04-15
**任务**: 为用户创建数据可视化报告

### 情境
用户请求创建一个销售数据月度报告，包含图表和趋势分析。之前没有为该用户处理过此类任务。

### 动作
1. 读取 `/memories/user_preferences.md` → 发现用户偏好详细图表
2. 读取 `/memories/workflow_library.md` → 找到"数据报告创建"流程
3. 应用模式：为报告类任务先确认数据源和期望格式
4. 使用 Python + matplotlib 生成图表
5. 交付前检查图表可读性和数据准确性

### 结果
报告成功交付，用户满意度高。

### 模式提取

```markdown
## Pattern: 报告类任务确认清单

**置信度**: [high]
**验证次数**: 3
**触发条件**: 用户请求创建任何类型的报告或数据文档

### 模式描述
在开始报告类任务前，先确认：
1. 数据源位置和格式
2. 期望的报告结构和长度
3. 图表类型偏好
4. 输出格式 (PDF/HTML/PPT)

### 效果
减少返工次数，提升用户满意度
```

## 案例 2: 失败教训记录

**类型**: failure
**时间**: 2026-04-18
**任务**: 部署 Web 应用到生产环境

### 情境
用户要求部署一个 Node.js 应用到云服务器。使用 `npm run build` 后直接部署，但应用无法启动。

### 动作
```json
{
  "reasoning": "直接执行 npm run build，然后上传到服务器",
  "tool_calls": ["bash: npm run build", "bash: scp dist/* server:/var/www/app"],
  "output": "构建成功，但部署后应用崩溃 - 缺少环境变量"
}
```

### 根因分析
1. 未检查生产环境与开发环境的差异
2. 缺少环境变量配置步骤
3. 未在部署前进行完整性检查
4. 忽略了 `.env` 文件的处理

### 教训提取
1. 部署前必须确认目标环境配置
2. 环境变量需要显式传递或配置
3. 部署流程必须包含健康检查步骤

```markdown
## Lesson: 生产部署环境检查

**严重程度**: [high]
**相关Pattern**: N/A (新建)

### 预防措施
- [x] 部署前检查目标环境
- [x] 确认环境变量配置
- [x] 添加部署后健康检查
- [ ] 使用部署清单脚本
```

## 案例 3: 置信度升级

**类型**: pattern_validation
**时间**: 2026-04-20
**模式**: "长文本任务先规划后执行"

### 验证历史
- **2026-04-10** [unconfirmed]: 首次应用于代码重构任务，成功
- **2026-04-15** [low]: 应用于文档编写任务，成功
- **2026-04-18** [medium]: 应用于复杂数据分析，成功
- **2026-04-20** [high]: 再次应用于多文件重构，验证通过

### 最终模式

```markdown
## Pattern: 复杂任务先规划后执行

**置信度**: [high]
**验证次数**: 4
**首次记录**: 2026-04-10
**最后验证**: 2026-04-20

### 触发条件
任务涉及：
- 多个文件/模块
- 超过 3 个步骤
- 不确定的技术实现
- 用户未明确期望顺序

### 模式描述
1. 分解任务为子任务列表
2. 识别任务间的依赖关系
3. 确定执行顺序
4. 预估每个子任务的复杂度
5. 先完成高风险/不确定的子任务

### 效果
显著减少返工，提高任务完成率
```

## 案例 4: 反例推翻模式

**类型**: counter_example
**时间**: 2026-04-22
**原有模式**: "文件操作前先备份"

### 情境
用户要求批量重命名 200 个图片文件。应用"先备份"模式后，发现：
1. 图片总大小 5GB，备份耗时过长
2. 重命名操作本身是幂等的
3. 用户明确表示不需要备份

### 分析
对于幂等操作且用户明确放弃备份的情况，"先备份"模式反而降低效率。

### 模式更新

```markdown
## Pattern: 文件操作先备份 (修订版)

**置信度**: [medium] (原 [high])

### 触发条件
- 非幂等操作 (删除、覆盖)
- 用户未明确表示不需要备份
- 操作涉及重要数据

### 不适用场景
- [x] 幂等操作 (重命名、复制)
- [x] 用户明确放弃备份
- [x] 临时/测试文件

### 新增规则
用户可以显式声明 "不需要备份"，此时跳过备份步骤。
```

  - [x] 分析现有代码结构
  - [x] 设计新架构

  - [ ] 更新单元测试
  - [ ] 更新集成测试

1. 读取 `session_notes.md`
2. 读取项目上下文

- 阻塞项必须清晰标注以便针对性解决
- 状态摘要帮助快速定位继续点

## 案例 6: 元认知审查触发

**类型**: metacognitive_review
**时间**: 2026-04-28

### 触发条件
- 完成第 5 个任务
- 用户纠正次数: 4 次 (阈值: 3)

### 指标摘要
```
任务成功率: 80% (↓10%)
平均完成轮次: 3.2 (↑0.5)
用户纠正频率: 0.8次/任务 (↑0.3)
```

### 根因分析
1. 最近 3 个任务涉及新领域，模式匹配不足
2. 响应长度偏短，用户需要更多解释
3. 工具选择有时不是最优

### Self-Improvement Directives

1. **[优先级-High]**: 新领域任务先确认用户期望的技术深度
2. **[优先级-High]**: 响应增加 "解释推理过程" 部分
3. **[优先级-Medium]**: 复杂决策前列出可选方案及其权衡
```
