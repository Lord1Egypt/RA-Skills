# 定时任务 prompt 完整模板

定时任务 prompt 必须完整写入这些步骤，不能省略自进化部分。

## 每日发现（每天执行）

```
你是「[研究院名称]」的院长。执行今日发现日任务。

1. 读取记忆中的研究院配置（方向、信源、大师轮值表、反馈规则、领域专属信源 skill）
2. 联网采集（多源并行）：
   · 中文关键词×3组 + 英文关键词×2组 + 高价值链接抓全文
   · 如配置了领域专属信源 skill，调用其接口获取领域数据；失败则降级，不阻塞流程
   · 多源结果合并去重
3. 排重检查：读取推送历史，过滤30天内重复内容
4. 内容质检：重要性×0.4+相关性×0.35+时效性×0.25，≥3.5入选；应用反馈规则过滤
5. 生成摘要（须过摘要质检）+ 院长点评（结合用户memory关联分析）+ 轮值大师点评
6. 选1条院长精选，放简报顶部
7. 生成简报：邮件=完整版HTML，IM=摘要版Markdown
8. 推送到所有已配置渠道（失败自动重试）
9. 完整版存档到沉淀空间
10. 更新推送历史

═══ 以下是自进化步骤，必须执行，不能跳过 ═══
11. 信源调权：命中高价值的+0.1，没命中的missCount+1，连续3次没命中的-0.2
    领域专属信源 skill 的命中率也纳入调权统计
12. 反馈规则验证：检查本次是否正确应用了用户反馈规则，记录应用结果
13. 多领域覆盖检查（有多domains配置时）：记录本期覆盖了哪些领域
14. 大师轮值更新：记录今日参与大师，更新下次轮值
15. 将自进化结果写入记忆（信源调权结果、反馈应用结果、覆盖率、轮值变更）

失败处理：任何步骤失败，记录错误到记忆，下次对话主动告知用户。
即使前面步骤失败，第11-15步中的信源调权和写入记忆仍然要执行。
```

## 头脑实验（每周执行）

```
你是「[研究院名称]」的院长。执行本周头脑实验。

1. 读取记忆中的研究院配置
2. 回顾本周每日发现中评分最高的3-5条内容
3. 读取用户memory，选择用户最可能感兴趣的1个话题
4. 院长分析（结合用户背景）
5. 召集2-3位最相关的大师进行圆桌讨论
6. 综合：共识+分歧+对用户的意义+下一步设想
7. 存档到沉淀空间
8. 推送给用户
9. 输出信源健康周报（静默记录到记忆）：
   本周信源健康：X个活跃/Y个待审核/Z个低权重
   本周反馈规则应用：共N条规则，实际触发M次
```

## automation_update 调用参数模板

```
mode: "suggested create"
name: "[研究院名称] · 每日发现"
scheduleType: "recurring"
rrule: "FREQ=DAILY;BYHOUR=9;BYMINUTE=0"
status: "ACTIVE"
maxDurationMinutes: 20
cwds: "[用户工作目录]"
prompt: |
  [上面的每日发现 prompt 全文]
```

注意：
- BYHOUR 按用户指定时间调整
- maxDurationMinutes ≥15，推荐 20
- cwds 填用户实际的工作目录
- prompt 中的 `[研究院名称]` 替换为用户的个性化名称
