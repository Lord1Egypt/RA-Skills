# 评估方法论详解

## 核心原则

1. **保守估算** — 初次评估时倾向于高估难度和耗时
2. **分层拆解** — 大任务拆成小任务分别评估
3. **明确假设** — 估算基于什么假设要说清楚
4. **保留调整空间** — 评估不是承诺，是参考

## 数据规模估算

### 如何判断数据量级

| 数据类型 | 估算方法 |
|---------|---------|
| 文件数 | `ls` 或 `dir` 统计 |
| 文本长度 | 按千字符估算 |
| 数据库行数 | 按批量查询估算 |
| 股票/公司数 | 直接查数量（A股约5300家）|

### 经验系数

- 读取操作： ~1KB/条
- 写入操作： ~2KB/条
- 分析操作： ~5KB/条
- 复杂计算： ~10KB/条

## 复杂度加成规则

在基础级别上，以下情况需要升级：

| 情况 | 加成 |
|------|------|
| 需要数据清洗 | +0.5级 |
| 跨平台/跨系统 | +1级 |
| 需要人工审核 | +1级 |
| 涉及保密/敏感信息 | +0.5级 |
| 依赖外部API且不稳定 | +1级 |
| 任务边界模糊 | +0.5级 |

## 时间估算公式

```
基础时间 = 数据量 × 单条处理时间
总时间 = 基础时间 × (1 + 复杂度加成)
缓冲时间 = 总时间 × 1.2 (预留20%)
```

### 示例

任务：分析100只股票的基本面

```
数据量：100只
基础单条时间：5分钟
基础时间：100 × 5 = 500分钟

复杂度加成：
+ 需要数据清洗 (+0.5级 → ×1.5)
+ 跨平台（需要调用多个API）(+1级 → ×2)

修正后：500 × 1.5 × 2 = 1500分钟
缓冲后：1500 × 1.2 = 1800分钟 ≈ 30小时

结论：L4，建议分批执行
```
