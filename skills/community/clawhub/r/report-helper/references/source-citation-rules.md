# 信息来源编号规则

成稿必须让正文事实与末尾来源一一对应。先建来源台账，再写正文；不要写完后临时补编号。

## 1. 正文前声明

在目录之后、第一章正文之前，必须加入：

```markdown
## 关于本报告的信息来源

正文中每条关键事实判断均以上标标注信源等级和编号，如 <sup>a1</sup> 表示 A 级官方/学术一手来源，<sup>b4</sup> 表示 B 级权威媒体原创报道。四级分类：a = 官方/学术一手，政府机构、监管机构、公司年报和原始研究报告均归入此类；b = 权威媒体原创报道；c = 行业媒体/工具商博客；d = 软文/服务商自述/不可验证。编号对应末尾「信息来源与分级」章节，可按编号查找具体出处。本报告目前不使用 c、d 级来源支撑正文判断。
```

如果本报告没有 c 或 d 级来源，末尾来源列表省略对应小节；正文声明仍保留完整四级规则。

## 2. 来源台账

写正文前，从中间资料整理一份来源台账：

- 每一个编号只对应一个具体来源。
- 同一来源重复使用时复用同一个编号，不重复造号。
- 编号全报告唯一，按首次进入台账的顺序递增，不按分组重新编号。
- 每条来源必须记录：编号、等级、标题或来源名称、URL 或出处说明、访问时间、支撑的事实。
- 没有 URL 的离线来源必须写清来源性质和可披露边界；不能披露或不能核验的材料降级处理，必要时不进入正文。

## 3. 正文上标

正文中每条关键事实、数据、时间、人物身份、产品发布、融资、政策、论文结论、媒体报道和关键判断后面都要放上标：

```markdown
Perplexity AI 在 2022 年 12 月上线对话式答案引擎，每个回答带来源引用<sup>a3</sup>。
QuestMobile 口径显示某 App 月活为 530 万，低于公司对外包含 App 内调用的宣传口径<sup>b6</sup>。
```

多来源共同支撑同一句判断时，连续放多个上标：

```markdown
这一变化同时来自搜索入口产品化和内容平台分发规则变化<sup>a4</sup><sup>b11</sup>。
```

存疑材料只可作为背景或风险提示，不支撑正文判断：

```markdown
某服务商自报客户案例显示转化率提升，但未披露样本口径，本文仅作为未证实案例记录<sup>d20</sup>。
```

不要把一整段只在末尾挂一个来源。段内不同事实来自不同来源时，分别标注。

## 4. 末尾分级列表

报告末尾只保留「信息来源与分级」章节，不再另开「参考文献」「方法论说明」。

```markdown
## 八、信息来源与分级

### a · 官方/学术一手（x 条）

1. {来源标题或文件名}<br>{URL 或出处说明}

### b · 权威媒体原创报道（x 条）

6. {来源标题或媒体名}<br>{URL 或出处说明}

### c · 行业媒体/工具商博客（x 条）

8. {来源标题或站点名}<br>{URL 或出处说明}

### d · 软文/服务商自述/不可验证（x 条）

20. {来源标题或材料说明}<br>{URL 或出处说明}

> 所有来源访问时间：{YYYY-MM-DD}
```

如果某一级没有来源，可以省略该小节；没有 c 和 d 就不写 c、d 小节。只要正文出现对应上标，末尾必须有同编号条目。

访问时间使用本次报告实际访问日期，例如 `> 所有来源访问时间：2026-04-30`。

## 5. 自检规则

- 正文出现的每个 `<sup>[a-d]数字</sup>` 都必须能在末尾找到同级同号来源。
- 末尾来源条目原则上都应在正文出现；只作为背景阅读的来源要标明「背景参考」，不要混进事实支撑列表。
- 关键事实判断必须由 a 或 b 级来源支撑；c、d 级来源不支撑正文判断。
- d 级来源不得支撑结论，只能支撑「某方这样声称」这一事实。
- 任何无法找到来源的事实，要么补搜，要么删掉，要么改写为「未搜到 / 存疑」。
