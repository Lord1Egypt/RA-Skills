This skill is ready for commercial/non-commercial use. <br>

## Use Case: <br>
Developers and engineering teams use this skill to create, debug, and maintain Cypress end-to-end and component tests, including selectors, assertions, auth flows, network mocks, CI setup, and test stability patterns. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Install examples include a curl-to-bash command that can execute unreviewed remote code. <br>
Mitigation: Install through ClawHub/OpenClaw or a reviewed pinned source, and review installation scripts before execution. <br>
Risk: Authentication, CI secret, database reset, and file deletion examples can affect real data or expose credentials if copied into shared or production environments. <br>
Mitigation: Use these examples only in isolated test environments with test-only credentials, protected CI secrets, validated paths, and explicit guards that prevent staging or production use. <br>
