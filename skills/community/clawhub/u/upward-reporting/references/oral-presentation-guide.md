# 口头汇报策略

文档是材料，口头才是决胜。汇报场景中**怎么讲**和**写什么**同样重要——同一份文档，不同的讲述方式可能产生截然不同的效果。

---

## 核心原则：文档是入场券，口头是决胜场

- 文档的作用：让读者在会前建立基本认知，确认"这件事值得花时间听你讲"
- 口头的作用：在有限时间内传递判断力、建立信任、推动决策
- 不要在口头汇报中重复文档内容——口头应该传递文档无法传递的东西（判断的温度、推理的过程、决策的紧迫性）

---

## 时间长度适配

不同时长需要不同的内容密度和结构，**千万不要把30分钟版压缩成5分钟版——而是重新设计结构**。

### 5分钟版（电梯汇报 / 会议间隙）

**结构**：TL;DR 直接讲出来，不展开

```
1句话：核心判断
1句话：战略对齐（这件事对老板的目标意味着什么）
2-3个数字：关键效果或差距
1句话：需要决策的事项
```

**关键**：5分钟版不是"30分钟版的压缩"，而是只讲TL;DR——判断、数字、决策诉求。其余全部省略。

**适合场景**：老板在会议间隙问"你那个项目怎么样了"、微信发语音汇报

---

### 15分钟版（例行同步 / 定期规划汇报）

**结构**：

```
开场30秒：核心判断 + 战略对齐（2句话）
3分钟：核心挑战与解法思路（只讲2-3个最重要的，用思辨叙事而非编号堆叠）
5分钟：关键数据和效果验证（2-3个最能说服人的数据，配归因说明）
3分钟：系统架构或方案实质（用1张图快速展示系统骨架）
2分钟：下一步方向 + 需要决策的事项
2分钟：Q&A
```

**关键**：15分钟版要取舍——只讲最重要的2-3个问题和解法，其余用"详见文档"带过。

**适合场景**：周/月规划汇报、项目进展同步

---

### 30分钟版（战略评审 / 项目启动 / 关键决策会）

**结构**：

```
开场1分钟：核心判断 + 战略对齐 + 闭环逻辑框架（用1张图）
8分钟：核心挑战与应对（3-4个问题，每个用思辨叙事展开，包含论证和战略选择）
5分钟：系统架构（用2-3张图展示架构、流程、上下游）
5分钟：实践与验证（关键数据+归因+认知反思）
3分钟：竞品对标与时机（如果相关）
3分钟：下一步方向 + 降级方案 + 需要决策的事项
5分钟：Q&A
```

**关键**：30分钟版可以完整展开，但仍要用思辨叙事而非编号堆叠。开场1分钟是最重要的——听众在这里决定是否认真听后面的内容。

**适合场景**：战略评审会、项目方向批准会、跨团队对齐会

---

## 开场30秒：决定听众是否认真听

开场30秒是最关键的——听众在这里判断"这个人的思路清楚不清楚、值不值得继续听"。

**开场公式**：

```
[核心判断] + [战略对齐] + [今天要讲什么]
```

例："AI知识供给的系统性闭环，是部门AIDevOps规模化落地的核心执行单元。今天我讲三件事：核心挑战和解法思路、系统架构、实践验证效果。"

**避免的开场方式**：
- ❌ 从背景讲起（"去年我们做了…"）——听众在等你给结论
- ❌ 先讲概念定义（"什么是知识供给闭环…"）——听众不需要科普
- ❌ 先感谢或寒暄过多——30秒很宝贵，不要浪费

## Q&A准备

### 常见追问类型与应对

| 追问类型 | 听者意图 | 应对策略 |
|---------|---------|---------|
| "凭什么这么说？" | 验证论证可信度 | 1句数据/逻辑支撑，然后"详细论证见文档附录A" |
| "为什么不做X而做Y？" | 验证战略选择合理性 | 简述排除路径和理由，参考技法6（战略选择论证） |
| "如果做不成怎么办？" | 验证底线思维 | 简述降级方案和底线目标 |
| "这件事和我/我的团队有什么关系？" | 确认利益相关 | 直说对对方的影响和收益，不要回避 |
| "这个数据口径是什么？" | 验证数据可信度 | 简述口径定义和计算方式，"详细见附录" |
| "时间怎么排？" | 确认执行可行性 | 给里程碑节点而非任务级别排期 |

### 追问应对原则

1. **给1句直接回答，不要绕弯**——追问者需要的是确定性，不是铺垫
2. **承认不确定，但给出方向**——"这个数据我们还在验证中，初步判断是X方向"
3. **需要详细论证的，指向文档**——"这个的完整论证链路见文档附录A.2，我们可以会后细看"
4. **不要被追问带偏节奏**——如果追问涉及非核心内容，快速回答后回到主线

---

## 讲述节奏控制

### 节奏原则

- **每3-5分钟切换一次内容维度**（问题→解法→数据→架构→方向），避免听众疲劳
- **关键判断处放慢**——"这里我想强调一个判断…"，让听众有时间消化
- **数据处加速**——数据本身不需要展开，快速念完数字+一句归因结论
- **架构处用图**——口头描述架构是灾难，投影一张架构图，用30秒讲核心组件和接口

### 语速与停顿

- 关键判断前：停顿1秒（制造期待感）
- 数据后：停顿1秒（让听众消化数字）
- 转折后：停顿2秒（让听众跟上思路切换）
- 非关键段落：正常语速，不刻意放慢

## 不同受众的口头适配

### 高层/大老板

- 开场直接给判断，不讲背景铺垫
- 全程用业务语言而非技术术语
- 数据只说结论+1句归因，不展开过程
- 主动说"需要您决策的事项是X"
- Q&A重点准备：ROI质疑、时机质疑、降级方案

### 直属上级

- 开场1句判断，然后展示推导过程
- 可以用技术术语，但关键判断仍要用业务语言
- 数据展开到归因层级
- 主动暴露假设和局限
- Q&A重点准备：执行可行性、资源合理性、风险应对

### 平级/协作方

- 开场说"我们需要联合做X"而非"我要做X需要你们配合"
- 强调共同目标和各方收益
- 明确角色和决策机制
- Q&A重点准备：对方投入和收益、协作摩擦点、升级路径
