## 一、GIS 定义与核心特征

**GIS（Geographic Information System，地理信息系统）**：在计算机硬、软件系统支持下，对地球表层空间中的地理数据进行**采集、存储、管理、运算、分析、显示和描述**的技术系统。

### 核心特征

| 特征 | 说明 |
|------|------|
| **空间性（Spatial）** | 所有数据都关联空间位置（坐标），这是GIS区别于普通信息系统的本质 |
| **时序性（Temporal）** | 地理数据随时间变化，支持动态分析和历史回溯 |
| **多维性（Multi-dimensional）** | 2D→2.5D(DEM)→3D(真三维)→4D(3D+时间) |
| **关联性（Relational）** | 要素之间存在空间关系（拓扑/距离/方向/包含） |

### 系统组成（五要素）

```
GIS 系统
  ├── 硬件（服务器/工作站/GPS/无人机/扫描仪/绘图仪）
  ├── 软件（ArcGIS/QGIS/SuperMap/CASS/FME/...）
  ├── 数据（核心资产，占系统价值的70%+）
  ├── 人员（GIS分析师/开发人员/测绘工程师/管理者）
  └── 方法（工作流程/标准规范/分析模型/管理制度）
```

## 二、发展历程

| 年代 | 阶段 | 特征 |
|------|------|------|
| 1960s | **萌芽期** | 加拿大GIS(CGIS)是世界上第一个GIS系统 |
| 1980s | **商业软件崛起** | Esri ArcInfo、MapInfo、Intergraph相继推出 |
| 1990s | **桌面GIS时代** | ArcView、MapInfo Professional 普及 |
| 2000s | **WebGIS时代** | Google Maps(2005)、OpenStreetMap(2004)、天地图 |
| 2010s | **云GIS + 大数据** | ArcGIS Online、GEE云平台、空间大数据 |
| 2020s | **GeoAI + 数字孪生** | 深度学习遥感解译、实景三维、SuperMap AgentX |

> **重要节点**：Esri 于 2024 年停止支持 ArcMap，所有新项目必须使用 **ArcGIS Pro**。

## 三、主流应用领域

| 领域 | 典型应用场景 |
|------|-------------|
| **自然资源** | 土地利用监测、矿产资源调查、林业资源管理 |

| **交通运输** | 路径规划、物流配送优化、交通流量分析 |
| **公共安全** | 应急响应、灾害评估、疫情防控 |
| **商业金融** | 选址分析、市场区域划分、客户画像 |
| **农业** | 精准农业、作物估产、土壤分析 |
| **水利** | 流域分析、洪水模拟、水资源管理 |
| **电力/管线** | 电网管理、管线信息系统 |

### 7.2 路径安全规范（建议全局遵守）

```
✅ 推荐：D:/Projects/GIS_Data/2026/landuse.shp
❌ 避免：D:/项目/数据/2026年/土地利用#1.shp
❌ 避免：D:\Projects\GIS Data\land use (2026).shp
❌ 避免：\\network\共享文件夹\数据.gdb

通用规则：
1. 项目根目录可用中文：D:/项目/
2. 数据子目录尽量英文：/data/vector/
3. 文件名 100% 英文 + 下划线 + 数字
4. 日期格式：YYYYMMDD（非 YYYY-MM-DD）
5. Python操作路径：使用 r"raw string" 或正斜杠 /
```

### 7.3 GDAL/Python 安全路径模板

```python
# ✅ 推荐写法
import os
base = r"D:/Projects/GIS_Data"
shp = os.path.join(base, "2026", "landuse.shp")  # 自动处理分隔符

# ✅ 中文路径（GDAL 3.1+）
gdal.Open(r"D:/项目/数据/土地利用.tif")  # raw string + 正斜杠

# ❌ 避免
gdal.Open("D:\\项目\\数据\\土地利用.tif")  # 反斜杠可能转义
gdal.Open("D:/项目/数 据/土地#1.shp")       # 空格+特殊字符
```
