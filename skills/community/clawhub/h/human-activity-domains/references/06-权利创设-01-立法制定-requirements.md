## 立法制定生成要求

立法制定每种任务类型的"零件清单"——必选/可选组件、组装顺序、领域约束。按清单逐项产出。

---

### 1 立法调研

#### T1-1 需求调研

- **必选组件**: 调研范围、调研方法、调研数据、调研报告
- **可选组件**: 调研团队、调研工具、调研预算
- **组装顺序**: 确定调研范围 → 选择调研方法 → 收集调研数据 → 分析数据 → 形成报告
- **约束**: 范围必须明确；方法必须科学；数据必须可靠
- **格式**: 需求调研报告（Markdown）

#### T1-2 问题分析

- **必选组件**: 问题描述、问题性质、问题范围、问题影响
- **可选组件**: 问题调研、问题分类、问题优先级
- **组装顺序**: 描述问题 → 分析性质 → 界定范围 → 评估影响
- **约束**: 描述必须清晰；性质必须准确；范围必须明确
- **格式**: 问题分析报告（Markdown）

#### T1-3 比较研究

- **必选组件**: 比较对象、比较维度、比较方法、比较结论
- **可选组件**: 比较资料、比较工具、比较报告
- **组装顺序**: 确定比较对象 → 设计比较维度 → 选择比较方法 → 形成比较结论
- **约束**: 对象必须可比；维度必须全面；方法必须科学
- **格式**: 比较研究报告（Markdown）

#### T1-4 可行性分析

- **必选组件**: 分析维度、分析方法、分析数据、分析结论
- **可选组件**: 分析工具、分析模型、分析报告
- **组装顺序**: 确定分析维度 → 选择分析方法 → 收集分析数据 → 形成分析结论
- **约束**: 维度必须全面；方法必须科学；结论必须客观
- **格式**: 可行性分析报告（Markdown）

### 2 立法起草

#### T2-1 条文起草

- **必选组件**: 条文结构、条文内容、条文语言、条文逻辑
- **可选组件**: 条文体例、条文注释、条文说明
- **组装顺序**: 设计条文结构 → 撰写条文内容 → 规范条文语言 → 检验条文逻辑
- **约束**: 结构必须清晰；内容必须完整；语言必须规范
- **格式**: 法律草案（Markdown）

#### T2-2 专家论证

- **必选组件**: 论证范围、论证专家、论证意见、论证结论
- **可选组件**: 论证会议、论证报告、论证记录
- **组装顺序**: 确定论证范围 → 邀请论证专家 → 收集论证意见 → 形成论证结论
- **约束**: 范围必须明确；专家必须权威；意见必须充分
- **格式**: 专家论证报告（Markdown）

#### T2-3 公众咨询

- **必选组件**: 咨询范围、咨询方式、咨询意见、咨询总结
- **可选组件**: 咨询平台、咨询工具、咨询报告
- **组装顺序**: 确定咨询范围 → 选择咨询方式 → 收集咨询意见 → 总结咨询结果
- **约束**: 范围必须明确；方式必须便捷；意见必须真实
- **格式**: 公众咨询报告（Markdown）

#### T2-4 草案修改

- **必选组件**: 修改依据、修改内容、修改理由、修改版本
- **可选组件**: 修改对照、修改说明、修改记录
- **组装顺序**: 分析修改依据 → 确定修改内容 → 阐述修改理由 → 形成修改版本
- **约束**: 依据必须充分；内容必须准确；理由必须合理
- **格式**: 修改版法律草案（Markdown）

### 3 立法审议

- **约束**: 内容必须完整；理由必须充分；说明必须清晰
- **格式**: 立法提案（Markdown）

#### T3-2 立法审议

- **组装顺序**: 确定审议程序 → 审议法律内容 → 收集审议意见 → 形成审议结论
- **约束**: 程序必须合法；内容必须完整；意见必须充分
- **格式**: 立法审议记录（Markdown）

#### T3-3 立法表决

- **约束**: 程序必须合法；方式必须公正；结果必须准确

#### T3-4 立法公布

- **必选组件**: 公布内容、公布方式、公布时间、施行日期

- **组装顺序**: 确定公布内容 → 选择公布方式 → 确定公布时间 → 明确施行日期
- **约束**: 内容必须完整；方式必须正式；时间必须明确
- **格式**: 立法公布公告（Markdown）
