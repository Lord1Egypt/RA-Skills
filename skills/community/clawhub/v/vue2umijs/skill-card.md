## Description: <br>
Migrate Vue 2/3 projects to React on UmiJS (@umijs/max) with conventions, constraints, stack mappings, and syntax examples for .vue to Umi pages and data flow. <br>

This skill is ready for commercial/non-commercial use. <br>

## Publisher: <br>
[aiswot](https://clawhub.ai/user/aiswot) <br>

### License/Terms of Use: <br>
MIT-0 <br>


## Use Case: <br>
Developers and engineers use this skill to migrate Vue 2/3 applications to UmiJS, React 18, and antd while preserving routing, data flow, styling, request handling, accessibility, and deployment behavior. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Migration changes can alter routing, authentication, request handling, environment variables, or deployment paths. <br>
Mitigation: Review generated application changes, run project tests, and verify behavior parity for routing, auth, request flows, environment mapping, and deployment assets. <br>
Risk: Framework conversion guidance can produce subtle UI or state-management drift from the original Vue application. <br>
Mitigation: Compare critical forms, tables, i18n flows, state models, and accessibility behavior against the legacy application before release. <br>
