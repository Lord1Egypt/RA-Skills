## Description: <br>
React and Next.js performance optimization guidelines from Vercel Engineering. This skill should be used when writing, reviewing, or refactoring React/Next.js code to ensure optimal performance patterns. Triggers on tasks involving React components, Next.js pages, data fetching, bundle optimization, or performance improvements. <br>

This skill is ready for commercial/non-commercial use. <br>

## Use Case: <br>
Developers and AI coding agents use this skill when writing, reviewing, or refactoring React and Next.js code for performance. It provides guidance for data fetching, rendering, bundle size, server behavior, client behavior, and JavaScript optimization patterns. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Security-sensitive examples in React and Next.js guidance may be copied into application code without enough safeguards. <br>
Mitigation: Review generated changes that touch cookies, sessions, localStorage, inline scripts, dangerouslySetInnerHTML, or background logging; use non-sensitive identifiers and CSP, nonce, and untrusted-data safeguards for inline script patterns. <br>
