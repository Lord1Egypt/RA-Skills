# Lesson Plan Architect

**Platforms:** Claude · Openclaw · Codex
**Domain:** Education

## Purpose

Generates a complete, standards-aligned K-12 lesson plan from a teacher's intake. Covers learning objectives, a gradual-release instructional sequence (I-do / We-do / You-do), tiered differentiation, materials list, formative assessment, and a standard-alignment table linking each activity to the target standard and Bloom level.

## When to Use

- Daily or weekly lesson planning for K-12 classrooms
- Building substitute-teacher-ready plans
- Producing differentiated versions for ELL, IEP/504, and gifted learners
- Aligning an existing lesson to a specific standard (CCSS, NGSS, state, IB, or local framework)
- Generating a formative assessment plus exit ticket for an existing lesson topic

## What It Does

**Phase 1: Intake**
1. Collects subject, grade level, lesson length, target standard(s), and learner profile
2. Confirms instructional modality (in-person, hybrid, asynchronous) and available materials/tech
3. Asks for prior knowledge anchor and the misconception(s) the teacher anticipates

**Phase 2: Design**
4. Writes 1–3 I-can objectives mapped to the target standard(s) and Bloom level
5. Builds the gradual-release sequence: hook/warm-up → I-do (model) → We-do (guided practice) → You-do (independent practice)
6. Designs formative checks for understanding at each stage
7. Drafts tiered differentiation: support, on-level, and stretch versions of the core task; ELL and IEP/504 accommodations

**Phase 3: Assessment and Verification**
8. Drafts an exit ticket aligned to the I-can objectives
9. Builds a standard-alignment table mapping each activity to the standard code and Bloom level
10. Produces a materials and prep checklist and a homework/extension option

## Output

A complete lesson plan with header (subject, grade, duration, standards), I-can objectives, materials list, vocabulary, gradual-release activity sequence with time blocks, differentiation table (Support / On-level / Stretch + ELL + IEP/504 accommodations), formative checks, exit ticket, homework/extension, and a standard-alignment table. Substitute-teacher-ready when the teacher confirms.

## Notes

The skill never fabricates a standard code that does not exist in the named framework. If the user names a standard the skill cannot verify in the conversation, it asks the user to paste the standard text. Student names, IEP/504 specifics, and any personally identifiable learner information shared during the session must not be used in examples, tool calls, or external searches.
