# Event Shock Engine / 事件冲击引擎

用途：把人生岔路里的抽象判断，转成用户能感受到的具体冲击。每次生成报告时，从 `era-event-timeline-2008-2026.md` 中筛选 4-6 个高相关事件或周期，再写成深度事件冲击卡。

## 基本流程

1. 识别主题：城市、海外、买房、职业、AI/技术转向，或混合主题。
2. 锁定时间：当年选择点、之后 3-5 年、当前触发年份。
3. 检索事件：只选和用户生活有直接关系的 4-6 个事件或周期。
4. 转成生活场景：写清这一事件在用户生活里长什么样。
5. 写出现实代价：钱、时间、健康、关系、签证、房贷、工作稳定性、机会窗口。
6. 写出情绪代价：不甘、焦虑、孤独、松一口气、安全感、错过感、羞耻感。
7. 写出误判：用户当年可能高估或低估了什么。
8. 写出验证：今天能用 7-30 天低风险动作校准什么判断。

## 事件冲击卡字段

每张卡固定六段：

| 字段 | 写法 |
| --- | --- |
| 事件锚点 | 年份 / 阶段 / 周期名，必须能回到用户当年的选择点。 |
| 生活场景 | 写城市、公司、家庭、租房、通勤、求职、会议、项目、病痛、关系中的具体画面。 |
| 现实代价 | 写钱、时间、健康、关系、签证、房贷、岗位、家庭照护或机会窗口怎样变化。 |
| 情绪代价 | 写焦虑、不甘、孤独、松一口气、安全感、错过感、羞耻感等真实感受。 |
| 误判点 | 写用户当年高估或低估了什么，不审判用户。 |
| 今天验证 | 写 7-30 天低风险动作：找谁聊、看什么材料、写什么案例、发给谁、记录什么反馈。 |

## 事件体感转换

每次从事件库取出事件后，先转成用户体感解释，再决定是否进入主报告。

| 字段 | 写法 |
| --- | --- |
| 事件 | 事件名、时间范围、适用主题。 |
| 用户当时可能看到什么 | 用户能直接观察到的场景，例如岗位减少、房租上涨、航班难买、同事开始试工具。 |
| 它真正改变了什么 | 它改变了成本、机会、身份、关系、健康或现金缓冲中的哪一项。 |
| 它可能让用户产生什么情绪 | 不甘、焦虑、孤独、松一口气、安全感、错过感等。 |
| 它最容易被误判成什么 | 例如把城市光环当能力光环，把收藏工具当入局，把房价涨跌当个人眼光。 |
| 今天还能怎么验证 | 7-30 天内的小动作，能让用户拿到真实反馈。 |

无法转成用户体感场景的事件，不放入主报告，只能进入附录。

## 主题映射

### 1. 城市选择类

适用输入：如果去了北京/上海/深圳/杭州、如果回老家、如果留在一线城市。

优先筛选：

- 2015-2019 移动互联网和内容平台成熟。
- 2016-2020 短视频和直播增长。
- 2020-2022 疫情、通勤和城市支持网络变化。
- 2021-2024 平台监管、裁员和行业波动。
- 2023-2026 AI 工具、远程协作和作品可见度变化。

必须转成：

- 机会密度：会议、项目、同行、老板、客户、行业活动如何改变被看见概率。
- 生活成本：房租、通勤、社交成本、储蓄率。
- 圈层进入门槛：别人为什么愿意带你进局。
- 家庭距离：父母、伴侣、照护和熟人支持。
- 身体消耗：睡眠、通勤、压力恢复。
- 作品可见度：项目如何从公司内部经验变成外部能看懂的案例。

### 2. 海外 / 留学 / 回国类

适用输入：留学后没回国、留英国/美国/澳洲/新加坡/香港、后来想出海。

优先筛选：

- 2016-2020 Brexit、签证和留学生就业环境变化。
- 2020-2022 疫情、边境、远程办公和家庭距离。
- 2021-2023 全球通胀、房租和生活成本。
- 2023-2026 AI、远程协作、出海公司和跨境内容机会。

必须转成：

- 签证现实：雇主、身份、求职空窗、可承受月数。
- 孤独感：陌生城市、语言会议、社交支持。
- 家人距离：父母期待、照护、回国频率。
- 海外通胀：租金、工资、储蓄压力。
- 远程机会：今天是否还能补英文作品、国际同行反馈和跨境项目。

### 3. 买房 / 没买房类

适用输入：那年买了房/没买房、如果早买/晚买、如果没有背房贷。

优先筛选：

- 2008-2016 城市化和房价预期变化。
- 2016-2020 热点城市限购、信贷和首付窗口。
- 2020-2022 疫情对收入稳定和居住安全感的影响。
- 2021-2026 地产周期、利率、就业波动和现金流压力。

必须转成：

- 首付窗口：当年能拿出多少钱，家庭是否支持。
- 月供压力：收入下降、岗位波动和健康成本。
- 城市绑定：换城自由、通勤、孩子和父母距离。
- 家庭安全感：父母认可、伴侣关系、居住稳定。
- 房价预期变化：只做复盘，不给投资建议。
- 现金流弹性：应急金、储蓄率、可承受空窗。

### 4. 职业 / 体制 / 大厂 / 离职类

适用输入：进体制、进大厂、离开上一家公司、转岗、创业、没离职。

优先筛选：

- 2015-2019 移动互联网增长和大厂扩张。
- 2020-2022 疫情和组织远程化。
- 2021-2024 平台监管、裁员和岗位重估。
- 2023-2026 AI 和工作流对知识岗位的重估。

必须转成：

- 稳定感：收入波动、身份确定、父母安心。
- 成长速度：项目密度、上级认可、组织能见度。
- 裁员周期：公司调整、岗位收缩、年龄压力。
- 技能迁移：旧项目能否写成外部案例。
- 健康消耗：睡眠、会议、通勤、压力恢复。

### 5. AI / 技术转向类

适用输入：如果早点做 AI、如果转技术、如果系统做 Skill/工作流。

优先筛选：

- 2023-2024 生成式 AI 工具扩散。
- 2023-2024 内容平台和个人作品分化。
- 2025-2026 Skill、工作流和自动化流程扩散。
- 2025-2026 就业市场技能重估。

必须转成：

- 工具收藏：收藏、试用、看教程和真实作品之间的差距。
- 作品资产：别人能否打开、试用、评价和复述。
- 工作流封装：输入、步骤、输出、边界。
- 公开输出：案例页、同行反馈、用户试用。
- 岗位重估：市场、内容、产品、运营经验如何被重新包装。
- 技术门槛：先用文档和手动流程验证，再考虑自动化。
- 现金流安全：业余时间实验，避免高风险转向。

## 深卡写作标准

- 每份报告只选 4-6 张事件冲击卡。
- 每张卡建议 160-260 字；手机阅读时每段 1-2 句。
- 事件锚点必须包含具体年份、年份区间、明确阶段或可核对周期，例如“2016 签证身份支撑”“2021-2024 裁员和岗位重估”“2023-2026 AI 作品窗口”。
- 避免把“作品资产”“现金流”“熟人支持”“身体恢复力”“当年某某”单独当作事件锚点；这些可以放在生活场景或误判点里。
- 卡片要写具体时间、具体场景、现实代价、情绪代价、误判点和验证动作。
- 避免堆很多小卡；宁可少选事件，也要把冲击写深。
- 如果一张卡删掉主题名后可以套到任何案例，重写。

## 合格示例

主题：2023 年没有认真做 AI。

| 字段 | 示例 |
| --- | --- |
| 事件锚点 | 2023-2024 生成式 AI 工具扩散。 |
| 生活场景 | 你开始收藏工具、试用插件、看别人做工作流，但真实工作里仍然靠手动写方案、整理素材和改稿。 |
| 现实代价 | 差距没有马上体现在收入上，先体现在作品空白上：别人能展示一个可复用流程，你只能说自己关注过。 |
| 情绪代价 | 这种错过感会带来被同龄人甩开的焦虑，也会让你怀疑过去的市场和内容经验有没有过期。 |
| 误判点 | 你可能把“知道工具”当成“进入新赛道”，低估了作品、复盘和外部试用反馈。 |
| 今天验证 | 选一个每周重复出现的工作问题，用 AI 做最小流程，找 3 个同事独立跑一遍，看他们能否复述价值和卡点。 |

## 使用边界

- 已发生事件用于复盘，未来只做情景假设。
- 宏观事件不能直接推出个人成败。
- 买房主题禁止输出投资建议。
- 职业主题禁止替用户做离职或转行决定。
- 海外主题禁止替代签证、法律或移民咨询。
- 健康主题只讨论生活消耗和恢复，不做诊断。
