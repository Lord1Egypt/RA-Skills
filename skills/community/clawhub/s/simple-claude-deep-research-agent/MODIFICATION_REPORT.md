## 执行摘要

经过详细分析原始需求文档（`research_lead_agent.md`, `research_subagent.md`, `citations_agent.md`）与当前实现的对比，发现当前系统存在**严重不完善**的问题。当前实现仅为原始需求的**约 30% 完成度**，缺少大量关键功能和细节。

## 一、需求分析总结

### 1.1 原始需求的核心组件

#### Research Lead Agent 应具备的能力：
1. **完整的四步研究流程**：评估分解 → 查询类型判定 → 详细计划制定 → 方法化执行
2. **智能查询分类**：深度优先/广度优先/直接查询，每种类型有明确的处理策略
3. **复杂的子代理管理**：
   - 使用 `run_blocking_subagent` 工具（不是普通 Task）
   - 依赖优先级排序
   - 并行部署策略
   - 明确的任务分配原则
4. **内部工具集成**：Asana, Slack, Github, GDrive, Gmail, Gcal 等
5. **严格的质量控制**：源质量评估、冲突解决、贝叶斯推理更新
6. **完整的输出格式化**：使用 `complete_task` 工具，明确不含引用

#### Research Subagent 应具备的能力：
1. **OODA 研究循环**：观察 → 定位 → 决策 → 行动
2. **工具使用策略**：
   - 内部工具优先（GDrive, Gmail, Slack 等）
   - web_search → web_fetch 核心循环
   - 并行工具调用
3. **严格预算管理**：
   - 简单任务：<5 次调用
   - 中等任务：~10 次调用
   - 复杂任务：~15 次调用
   - 绝对上限：20 次调用
4. **源质量批判性思维**：识别投机、营销语言、数据选择等问题
5. **密集报告格式**：简洁、信息密集的报告

#### Citations Agent 应具备的能力：
1. **引用添加**：为报告添加准确的引用
2. **严格约束**：不修改原文，仅添加引用
3. **引用策略**：避免过度引用、有意义语义单元、最小化句子碎片化
