### 环节1：项目挂榜

#### 1.1 谁可以发布项目

| 发布者类型 | 权限 | 审核要求 |
|-----------|------|---------|
| **OPC导师** | 可发布任意规模项目 | 需提交客户需求确认 |
| **OPC成员** | 可发布≤5万项目 | 需导师背书 |
| **外部客户** | 可发布需求等待报价 | 需社区审核 |

#### 2.5 挂帅机制

**挂帅人职责**：
- 作为项目牵头方，对外代表项目组
- 负责项目整体统筹和进度把控
- 主导项目分包和团队组建
- 承担项目交付的主要责任

**挂帅人权益**：
- 基础贡献权重 + 10%挂帅补贴
- 优先获得新项目信息
- 可担任三长老角色（需信用A级以上）

#### 2.6 流标处理

| 流标原因 | 处理方式 |
|---------|---------|
| 竞标方均不达标 | 重新发布，降低门槛 |
| 竞标方数量不足 | 自动延长竞标期3天 |
| 客户取消项目 | 项目终止，公示原因 |
| 恶意竞标 | 取消竞标资格，扣信用分 |

### 环节3：项目分包

#### 3.1 拆解逻辑

```
项目拆解四步法：

Step 1：需求分析
└── 分析客户需求，确定核心交付物

Step 2：模块划分
└── 按功能/流程拆解为独立模块
    • 每个模块有明确边界
    • 每个模块可独立验收

Step 3：技能映射
└── 为每个模块匹配所需技能
    • 核心技能（必备）
    • 辅助技能（可选）

Step 4：分包分配
└── 根据能力画像分配模块
    • 优先匹配度高
    • 考虑负载均衡
```

#### 3.2 拆解示例

**案例：品牌全案营销项目（30万）**

| 模块 | 内容 | 预算 | 所需技能 | 推荐分包 |
|------|------|------|---------|---------|
| M1 | 品牌策略定位 | 3万 | 品牌策划/市场分析 | 策略导师 |
| M2 | 视觉设计（VI） | 6万 | LOGO设计/VI设计 | 设计OPC |
| M3 | 官网开发 | 10万 | 前端+后端+运维 | 开发团队 |
| M4 | 内容创作 | 4万 | 文案/编辑/排版 | 内容OPC |
| M5 | 社交媒体运营 | 4万 | 运营/投放/客服 | 运营OPC |
| M6 | 项目管理 | 3万 | 项目统筹/质量管理 | 挂帅人 |

#### 3.3 分包规则

| 规则类型 | 具体内容 |
|---------|---------|
| **分包门槛** | 单模块金额≥1000元才能分包 |
| **分包比例** | 挂帅人保留模块价值≥30% |
| **分包审核** | C级以上项目需三长老审批 |
| **接包限制** | 同一成员承接模块数≤3个 |
| **利益回避** | 挂帅人不得承接自己分包的项目 |

#### 3.4 接包人匹配

**匹配算法**：

```
匹配得分 = 技能匹配度×40% + 信用等级×30% + 负载状态×20% + 历史好评×10%

• 技能匹配度：根据能力画像技能标签匹配
• 信用等级：S/A/B/C/D对应系数1.2/1.0/0.9/0.7/0.5
• 负载状态：当前项目数≤3为正常
• 历史好评：最近5个项目平均好评率
```

## 五、关键决策点

### 5.1 决策点清单

| 决策点 | 决策内容 | 决策者 | 决策规则 |
|-------|---------|-------|---------|
| **D1** | 项目发布审核 | 平台/导师 | 格式+合规双审 |
| **D2** | 项目分级确认 | 平台 | 按金额+复杂度自动分级 |
| **D3** | 竞标资质预审 | 系统 | 能力标签+信用等级筛选 |
| **D4** | 评标结果确认 | 评标委员会 | 综合得分最高者 |
| **D5** | 挂帅人任命 | 三长老 | 评标结果确认后自动任命 |
| **D6** | 分包方案审批 | 平台/三长老 | 审核分包合理性和利益回避 |
| **D7** | SPV协议签署 | 全体成员 | 各方签字确认 |
| **D8** | 里程碑验收 | 客户+三长老 | 验收通过方可分账 |
| **D9** | 分账方案审批 | 三长老 | 审核分账计算正确性 |
| **D10** | 纠纷裁决 | 三长老/仲裁 | 按争议等级处理 |
