# 首次启用 memory-lancedb-pro（与 `SKILL.md`「首次启用记忆」对应）

当用户**第一次**使用本 Skill 做邮件检索 / 船货盘查询，或助手检测到**尚无** `memory_store`、`memory_search` 及 memory-lancedb-pro **等价能力**时，**必须先**走本节；**禁止**未获同意即静默安装或改配置。

## A. 说明用途（口语，约 5 句话内）

- 邮件多、若每次提问都让模型通读最近一段时间的全部邮件，会消耗大量 token、变慢。  
- 记忆插件把邮件存入本机向量库后，可按问题只取相关片段给模型，一般更省 token。  
- 需要安装 **memory-lancedb-pro**，并配置 **Embedding**（以插件文档为准）。  
- 最后一句：请用户确认是否同意继续安装与配置。  
- 勿夸大「完全免费」「100% 不漏邮件」；勿暗示可替代专业合规审查。

## B. 征求明确同意（opt-in）

必须使用户能**拒绝**。示例话术：

```text
是否同意我带您完成 memory-lancedb-pro 的安装，并按插件说明配置 Embedding？
回复「同意」继续；回复「不同意」则暂不启用记忆检索（后续可能更费 token 或能力受限）。
```

- **同意** → 执行 **C**。  
- **拒绝** → 执行 **D**。

## C. 用户同意后的推荐顺序（以官方文档为准）

1. **安装插件**：优先插件市场；命令示例（以 OpenClaw 为准）：`openclaw plugins install memory-lancedb-pro@beta`；须在用户回复「同意」后代执行。  
2. **配套 Skill**：若 ClawHub 单独提供 memory-lancedb-pro Skill，按官方说明核对是否需同时安装。  
3. **配置 Embedding**：在 OpenClaw 或插件设置中完成；零基础用分步引导。  
4. **验证**：轻量 `memory_search` 试检索后进入 **Workflow 1（邮箱配置）**。

## D. 用户拒绝或环境不具备

- 说明不启用时查邮仍可能可行，但更易重复读大量正文、token 更高；无替代能力则如实说明。  
- **不得**反复催促开通；用户日后说「装记忆」「省 token」等可再次走 A–C。
