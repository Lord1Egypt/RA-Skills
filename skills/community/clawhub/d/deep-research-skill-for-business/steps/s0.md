# S0: 确认范围与深度

## 目标

在开始搜索前，与用户对齐调研边界，确保不跑偏、不返工。

## ⛔ 红线

- **必须输出 6 要素确认表**（见下方模板）
- 缺少任何一要素则不可进入 s1
- 如用户未提供足够信息，**主动提问澄清**，不可猜测

## 操作

### 1. 提取或追问 6 要素

| # | 要素 | 说明 |
|---|------|------|
| ① | **核心问题** | 用户到底要回答什么问题？一句话说清楚 |
| ② | **边界** | 什么在范围内？什么明确不涉及？ |
| ③ | **已知约束** | 已有哪些已知条件、限制、前提？ |
| ④ | **关键未知** | 哪些信息缺失？是否需要先向用户确认？ |
| ⑤ | **交付物** | 最终要产出什么？报告结构？格式要求？ |
| ⑥ | **深度要求** | 需要到什么深度？快速简报 / 标准分析 / 全深度？ |

### 2. 输出确认表（必须）

```markdown
## S0: 调研范围确认

| 维度 | 确认 |
|------|------|
| **核心问题** | ... |
| **边界** | ... |
| **已知约束** | ... |
| **关键未知** | ... |
| **交付物** | ... |
| **深度要求** | ... |

### 待澄清问题（如有）
1. ...
2. ...
```

### 3. 获取用户确认

将确认表发给用户，等待明确确认或修正后再进入 s1。

### 4. 标记完成

完成后执行：
```bash
bash orchestrator.sh --complete s0 --next s1
```

## 校验标准

- [ ] 6 要素全部有明确内容
- [ ] 关键未知已向用户提问
- [ ] 用户已确认（或"无需补充"）
