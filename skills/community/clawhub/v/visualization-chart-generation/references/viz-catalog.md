## 可视化图表生成清单与依赖拓扑

可视化图表生成按使用流程组织的任务清单，附任务间关系和执行映射提示。

**域间逻辑流**：V0 → V1 → V2/V3/V4 → V5（按需调用）

---

### V0 需求分析域

| ID | 任务类型 | 说明 | 依赖 | 执行映射 |
|----|---------|------|------|----------|
| V0-01 | 图表类型识别 | 识别用户需要的图表类型：流程图/结构图/说明图/数据图 | 无（入口） | S |
| V0-02 | 复杂度评估 | 评估图表复杂度：简单/中等/复杂 | V0-01 | C |
| V0-03 | 交互需求分析 | 分析交互需求：静态/交互/动态 | V0-02 | C |
| V0-04 | 格式选择 | 根据需求选择输出格式：SVG/HTML/HTML+JS | V0-03 | C→A |

### V1 设计规范域

| ID | 任务类型 | 说明 | 依赖 | 执行映射 |
|----|---------|------|------|----------|
| V1-01 | 颜色系统应用 | 选择合适的色系和层级，应用到图表 | 无（入口） | C→A |
| V1-02 | 字体系统应用 | 应用标题/正文/辅助文字规范 | V1-01 | A |
| V1-03 | 布局规则应用 | 应用圆角、间距、安全区域规范 | V1-02 | A |
| V1-04 | 组件样式应用 | 应用预设类（box/node/arr/leader等） | V1-03 | A |

### V2 SVG生成域

| ID | 任务类型 | 说明 | 依赖 | 执行映射 |
|----|---------|------|------|----------|
| V2-01 | 静态图表生成 | 生成架构图、关系图、对比图 | V1-04 | A |
| V2-02 | 流程图生成 | 生成顺序过程、因果关系、决策树 | V1-04 | A |
| V2-03 | 结构图生成 | 生成包含关系、物理/逻辑层次 | V1-04 | A |
| V2-04 | 说明图生成 | 生成建立直觉、物理截面或抽象隐喻 | V1-04 | A |

| V3-03 | 表单控件生成 | 生成按钮、输入框、下拉菜单 | V1-04 | A |

---

### 依赖拓扑摘要

以下为可视化图表生成任务的主要依赖链路：

#### 一、域内链路

**需求分析链路**: V0-01 → V0-02 → V0-03 → V0-04
**设计规范链路**: V1-01 → V1-02 → V1-03 → V1-04
**SVG生成链路**: V1-04 → V2-01/V2-02/V2-03/V2-04 → V5-01
**HTML生成链路**: V1-04 → V3-01/V3-02/V3-03/V3-04 → V5-02
**复杂可视化链路**: V1-04 → V4-01/V4-02/V4-03 → V5-02

#### 二、跨域协同链路

**完整图表生成链路**: V0-01 → V1-01 → V2-01/V3-01/V4-01 → V5-01/V5-02 → V5-03

**格式选择链路**: V0-04 → V2-01/V3-01/V4-01（格式决定生成方式）
**规范应用链路**: V1-04 → V2-01/V3-01/V4-01（规范应用到生成）
**输出保存链路**: V5-01/V5-02 → V5-03（输出后添加保存功能）

#### 三、格式选择决策树

```
用户需求
├── 静态图表
│   ├── 简单 → SVG（V2-01）
│   └── 复杂 → SVG（V2-02/V2-03/V2-04）
├── 交互功能
│   ├── 简单交互 → HTML（V3-01）
│   ├── 动态效果 → HTML（V3-02）
│   └── 表单控件 → HTML（V3-03）
├── 数据可视化
│   ├── 标准图表 → HTML+JS+Chart.js（V4-01）
│   ├── 复杂关系 → HTML+JS+D3.js（V4-02）
│   └── 自定义 → HTML+JS（V4-03）
└── 保存为图片
    └── SVG或HTML + 保存按钮（V5-03）
```

#### 四、设计规范依赖

**颜色系统**：V1-01 → 所有生成任务
**字体系统**：V1-02 → 所有生成任务
**布局规则**：V1-03 → 所有生成任务
**组件样式**：V1-04 → 所有生成任务
