# 🚀 即兴写作模式 (Instant Mode) 完整协议

> 从 SKILL.md 提取。当用户想要快速产出、不想走完整 Q&A 流程时触发。创意驱动,2 问定调直接出稿。

### 触发条件

检测到以下信号之一 → **Creator 主动询问**是否启用即兴模式(不再自动激活):

- 用户明确说:"快快快" / "随便写" / "速写" / "即兴"
  - ⚠️ 移除: "直接写" / "别问了直接写" / "不用问了" — 这些短语在日常对话中太常见
- 用户在**同一句话中**同时提供了完整的主题+平台+字数+风格(≥4个维度)
  - ⚠️ 移除: "≥40字" 自动激活 — 字数不是可靠的同意信号
- 用户在 Phase 1 中说"不用再问了，直接开始写"
  - ⚠️ 移除: MEMORY.md ≥3次历史 → 自动激活。过去行为不代表当前意愿。
- 用户说:"按上次的来" + 明确提到具体的写作类型/主题 → Creator 确认沿用后进入即兴模式
  - ⚠️ 移除: 单独的"按上次的来"不经确认就激活

**Creator 确认话术:**
```
⚡ 检测到你可能想快速出稿。要切换到即兴模式吗？
- 即兴模式：最多2轮问题 + 直接出稿，85%置信度，够用就好
- 完整模式：深度挖掘需求，经读者评审，质量更有保障

你可以随时在两种模式间切换。
```

### 核心原则

> **"2 问直出" - 最多 2 轮问题,每轮 ≤4 问,然后直接产出终稿。**

### 工作流

```
用户输入 → Creator 轻量意图捕获(≤2轮) → 配方自动匹配 → Executor 双温单版直出 → Creator Lite 审查 → 交付 → 后台进化
```

### Phase I: 轻量意图捕获 (Creator, ≤2 轮)

**第 1 轮(必问,4 个关键维度)**:
- 目标受众 + 平台/场景
- 语调/风格方向(提供 2-3 选项)
- 长度预期
- 特殊限制(有什么绝对不能出现的?)

**第 2 轮(选问,仅当第 1 轮信息不足时)**:
- 补充 1-2 个关键缺口
- 确认模板/配方(从 MEMORY.md 自动推荐)

**第 2 轮后**:无论信息是否完美,强制执行"信息够用就写"原则。Creator 输出一张**轻量意图卡**:

```markdown
## 📋 即兴写作意图卡

- 主题:[一句话]
- 受众:[谁在看]
- 语调:[风格方向]
- 字数:[目标范围 ±20%]
- 模板:[自动匹配的最佳模板]
- 配方:[如有融合,标注比例]
- 禁止项:[从 MEMORY.md 自动提取]
- 必须包含:[从 MEMORY.md 自动提取]

🧠 自动注入了你的历史偏好:
- 语调:[从 MEMORY 提取]
- 禁用词:[从黑名单提取]
- 偏好结构:[从历史高分模板提取]

⚡ 正在直接产出...
```

### Phase II: 双温单版直出 (Executor)

**核心设计**:Executor 分两遍写作--第一遍自由释放创造力,第二遍精密校准。两遍内化到同一个版本中,省去 v1/v2 对比 + Reader 评审。

#### 双温分写协议

**第 1 遍 - 创意爆发 (temperature ≈ 0.7-0.8)**
- 自由创作,允许创造力溢出
- 不受字数约束
- 只关注内容质量而非格式

**第 2 遍 - 精密校准 (temperature ≈ 0.2-0.3)**
- 字数拉入目标 ±20% 柔区
- AI 痕迹自查与消除(对照模板专属疲劳词表)
- 结构优化、逻辑修复
- 偏好对齐(对照意图卡中的禁止项/必须项)

#### 字数柔区 (±20%)

- `--words` 为**目标中心**,允许 ±20% 浮动
- 超出 → 1 次 Normalizer pass 纠偏
- 仍超出 → 保存 + 标注字数 warning(不硬截断)

**Executor spawn 协议**(即兴版,精简于完整版 Phase 3):

```
sessions_spawn:
  task: |
    [即兴写作模式 - 双温分写]

    意图卡:[轻量意图卡完整内容]

    适用模板骨架:[从模板库提取]

    用户历史偏好(自动注入):
    - 语调偏好:[从 MEMORY 全局 + 类型匹配]
    - 禁用词:[从黑名单]
    - 偏好表达:[从白名单]
    - 模板专属 AI 疲劳词:[从模板库的词表提取]

    写作分两遍:
    第 1 遍:自由创作,不被字数限制,输出完整正文
    第 2 遍:精密校准--字数拉入目标范围(±20%)、AI 痕迹自查、结构优化

    输出:仅输出校准后的最终版本(不输出第 1 遍草稿)
  model: "deepseek/deepseek-v4-flash"
  context: "isolated"
```

### Phase III: Creator Lite 审查 (Creator, 不 spawn Reader)

不同于完整模式的 Phase 4(spawn Reader 做 6 维评分),即兴模式中 Creator 自己做快速审查:

**Lite 审查清单(5 项硬性检查 - v2.8 从 3 项升级)**:
1. ✅ 是否触犯了意图卡中的**禁止项**?
2. ✅ 是否命中了**模板专属 AI 疲劳词**?(扫描全文,发现 >3 个 → 发回 Executor 修正)
3. ✅ 字数是否在 ±20% 柔区内?
4. ✅ 🆕 **被动语态密度**:中文被动语态(「被」「受」「得到」)是否 >3 次/500 字?超过 → 标记修正
5. ✅ 🆕 **结尾升华检测**:最后一段是否存在「综上所述」「未来展望」「具有重要意义」等 AI 套话?存在 → 直接砍掉

通过 → 直接交付。不通过 → 发回 Executor 单次修正(**仅 1 轮修正权,不进入完整 Mode A/B/C 循环**)。

### Phase IV: 交付 + 后台进化

- 输出终稿 + 意图卡摘要(不输出评分表,保持简洁)
- **进化引擎仍然运行**:即兴写作的所有偏好数据照常写入 MEMORY.md
- 用户反馈后 → 如果涉及风格调整,可以升级到完整模式(回 Phase 2 重来)

### 即兴模式 vs 完整模式对比

| 维度 | 完整模式 (Phase 0-5.5) | 即兴模式 (v2.7) |
|------|----------------------|----------------|
| Q&A 轮数 | ≤4 轮,逐层推进 | ≤2 轮,关键维度 |
| Executor 输出版本 | ≥2 版 | 1 版(双温内化) |
| Reader 参与 | 完整 6 维评分 | 跳过(Creator Lite 审查) |
| 迭代能力 | Mode A/B/C 多轮 | 仅 1 次修正权 |
| 适合场景 | 重要内容、初次尝试、高要求 | 熟悉主题、紧急需求、"随便写" |
| 质量标准 | 95% 置信度 | 85% 置信度(够用就好) |
| 进化参与 | 完整进化分析 | 进化记录(轻量) |

### 即兴模式的守护规则

1. **用户随时可升级**:说"太随意了,来正式的" → 自动切回完整模式,从 Phase 1 开始
2. **不强制即兴**:Creator 检测到触发信号后,可以主动确认:"检测到你想要快速产出,切即兴模式(2问完事直接写)?"
3. **1 次修正权只用 1 次**:不给修改无限循环(即兴模式的核心价值是"快")
4. **AI 疲劳词表自动注入**:每个模板维护专属高频 AI 词汇表(见 template-library.md),Executor prompt 自动注入
