# FAQ 风格库 — 我的回复模式沉淀

> **文档类型**:用户运行数据(初始为空模板)
> **维护方式**:由 skill 在用户主动确认后追加
> **本文位置**:`data/faq-library.md`

---

## 这是什么

这是你的**个人回复风格档案**。每次你用 skill 起草私信/评论回复后,如果你告诉 skill"我用了候选 X"或"我自己改成了这样",skill 会把这条问答**追加**到本文件。

随着积累,skill 会越来越懂你的语气、习惯用词、对不同问题的标准应对方式。**它学的是风格,不是事实判断**(具体市场观点不学,因为天天变)。

---

## 如何让 skill 沉淀 FAQ

任务结束时 skill 会问:

```
本次任务你最终用了哪一条?(可选回答)
A. 候选 X(skill 直接存)
B. 我自己改写了(把最终发的内容粘给我)
C. 我没回(不存,但记录"暂未应对此类问题")
D. 跳过(不沉淀)
```

只有 A 或 B 时才会更新本文件。

## 库内格式约定

每个条目长这样(skill 自动生成):

```markdown
## [类别] 问题原文

**首次记录**:YYYY-MM-DD HH:MM
**最近一次出现**:YYYY-MM-DD HH:MM
**累计遇到次数**:N

### 我的标准回复
[最终发出的回复全文]

### 风格特征
- 长度:约 X 字
- 语气:[亲切 / 专业 / 边界明确 / ...]
- 关键句式:[如 "看到啦~"、"我自己的判断是" 等]
- 是否含免责声明:✓ / ✗

### 类似问题(用户后续遇到时,skill 优先匹配)
- 变体 1:...
- 变体 2:...
```

## 类别枚举(skill 自动归类用)

skill 会把每条问答归到以下类别之一:

| 类别 | 说明 | 频率预估 |
|------|------|---------|
| `新手入门` | 期货是什么、怎么开始、看什么书 | 高 |
| `具体操作` | 现在能做空 X 吗、X 价位能进吗 | 高(都需高强度合规处理)|
| `风险求助` | 我亏了 / 爆仓 / 帮我看 | 中 |
| `组队拉群` | 加微信、建群、找搭子 | 中(标准拒绝模板)|
| `求带单` | 求带、求暗号、求大佬 | 低(标准拒绝)|
| `表达赞赏` | 好棒、学到了、辛苦 | 高(简短回应) |
| `催更/闲聊` | 怎么不更新、最近忙吗 | 中 |
| `质疑挑战` | 你这观点不对、你也亏过 | 中(理性回应)|
| `求合作/广告` | 商务合作、推广 | 低(走联系方式) |
| `其他` | 兜底 | — |

## 隐私与脱敏

skill 在沉淀时**自动脱敏**:

- 粉丝昵称 → 替换为 "粉丝" 或 "@用户" (除非你坚持保留)
- 具体日期 → 保留"YYYY-MM-DD"
- 手机号 / 微信号 / 身份证号 → 一律脱敏为 `[已脱敏]`
- 你提到的具体品种代码 + 价位 → 保留(这些是公开信息)

## 风格指纹提取(skill 用)

skill 在每次起草新回复前,会从本库提取你的**整体风格指纹**:

```yaml
风格指纹(自动计算,每次更新):
  开头偏好:["看到啦~", "谢谢呀~", "好问题"]  # top 3
  结尾偏好:["🌱", "(不构成投资建议)", "评论区见"]
  平均长度:48 字
  emoji 使用:中等(每条 0-2 个)
  口语词频率:"呀/嘛/呢/哈" 出现率 30%
  对边界问题的态度:温和但坚定
  对求带单的标准回应:已记录 N 次,套路稳定
```

新生成的候选会**自动贴近**这个风格,不需要你每次重新教 skill。

## 维护建议

- **每月扫一次**本文件,看看有没有"已经过时"或"风格已变"的条目
- 想删除某条 → 直接编辑此文件删除即可,skill 下次读时自动忽略
- 想更新某条标准回复 → 找到对应 `## [类别] 问题原文`,改"我的标准回复"段
- 文件**不会自动备份**,建议定期手动备份此文件(放 iCloud / OneDrive)
