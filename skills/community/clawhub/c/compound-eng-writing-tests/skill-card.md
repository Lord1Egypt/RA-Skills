## Description: <br>
Generic test writing discipline for test quality, real assertions, anti-patterns, and rationalization resistance when writing tests, adding coverage, or fixing failing tests across languages and frameworks. <br>

This skill is ready for commercial/non-commercial use. <br>

## Use Case: <br>
Developers and engineers use this skill to write, improve, or repair tests with behavior-focused assertions, realistic test data, appropriate mocking boundaries, and resistance to common excuses for skipping coverage. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Cleanup guidance such as database resets or Docker volume teardown can affect real data if applied outside a test environment. <br>
Mitigation: Confirm the target is a test environment before applying destructive cleanup commands or state resets. <br>
