## Usage Guide

### When to Load
- User explicitly names a brand: "Make it look like Stripe" / "Linear style"
- User describes a style matching a category: "Dark developer tool" → Linear/Supabase
- User provides a reference URL that maps to a known brand

### How to Load
1. Read the corresponding `assets/design-md/<brand>/DESIGN.md` file
2. Extract the design tokens (colors, typography, components, shadows)
3. Apply them as the foundation for Phase 1 Design System in the XDesign workflow
4. Skip manual brand material extraction — go directly to Phase 2

### Style-to-Brand Mapping
When user says:
- "clean and minimal" → Vercel, Cal, Ollama, or Linear
- "dark mode" → Linear, Supabase, Resend, Framer, Raycast, Warp, or Spotify
- "warm and friendly" → Claude, Notion, Airbnb, Lovable, or Zapier
- "professional / fintech" → Stripe, Coinbase, Revolut, or Wise
- "developer tool" → Linear, Supabase, Cursor, Vercel, or Raycast
- "documentation" → Mintlify
- "luxury / premium" → Apple, Tesla, BMW, Ferrari, or SpaceX
- "creative / playful" → Figma, Airbnb, Spotify, or Clay
- "technical / engineering" → Vercel, Supabase, Linear, or ClickHouse
- "enterprise" → IBM, HashiCorp, Airtable, or Cohere
- "AI platform" → Claude, Cohere, Mistral, or Replicate
- "automotive" → BMW, Tesla, Ferrari, Lamborghini, or Renault
- "crypto / blockchain" → Coinbase, Kraken, or Revolut
- "automation / workflow" → Zapier, Composio, or Webflow

### Limitations
- DESIGN.md files capture visual style only, not brand assets (logos, photography)
- Custom fonts may require web font subscriptions — always provide fallbacks
- These are style references, not 1:1 copies. Apply the design language, don't replicate the brand identity
- For brands not in this catalog, fall back to web_fetch or screenshot extraction
