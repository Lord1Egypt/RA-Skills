# /qa-report 报告生成示例 — 三段话变日报

> 本示例展示 `/qa-report` 如何处理非结构化输入——测试员随口一段话，AI 提取数据填入标准模板。

## 场景一：一句话生成日报

### 原始输入

```
/qa-report

报告类型：日报
日期：2026-06-22
姓名：张三

数据来源：
  今天主要测了订单改价功能，跑了 18 条用例，P0 的 3 条全过了，
  P1 7 条过了 5 条，有 2 条涉及并发改价的用例需要开发配合补充测试数据。
  发现 2 个 Bug，一个退款金额计算错误（提给李四了），
  一个是页面在 Safari 浏览器改价按钮错位（提给王五了）。
  明天继续测 P2 的用例，还有性能那条。
  对了，支付宝的测试环境今天又挂了半天，下午三点才恢复。
```

### AI 输出

#### 测试日报
| 项目 | 内容 |
|------|------|
| 日期 | 2026-06-22 |
| 姓名 | 张三 |
| 模块 | 订单改价功能 |
| 今日执行用例 | 18 条（P0:3 全通过、P1:5/7 通过、P2:0） |
| 今日发现缺陷 | 2 个（严重:1「退款金额计算错误」、一般:1「Safari 按钮错位」） |
| 进行中 | 订单改价 P2 用例、性能测试用例 |
| 阻塞问题 | P1 并发改价用例（2 条）等待开发补充测试数据；支付宝测试环境不稳定（今日 3 小时不可用） |
| 明日计划 | 完成订单改价 P2 用例 + 性能测试用例 |

#### 数据来源标注
- 文本输入（张三口述），无文件校验

## 使用技巧

1. **日报不用填表格**：像聊天一样说一段话就行，AI 负责提取和排版
2. **关键数字要说清楚**：P0/P1/P2 分布、致命/严重/一般/建议 分布——这些是 AI 无法编造的，但可以从你口述中提取
3. **文件 > 口述**：CSV 导出的数据比口述精准，建议阶段报告和季度报告优先用文件输入
4. **混合输入最省事**：主要数据用 CSV，临时补充用口述——AI 会交叉验证
