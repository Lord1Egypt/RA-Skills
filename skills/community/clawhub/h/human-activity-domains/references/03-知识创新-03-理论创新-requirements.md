## 理论创新生成要求

理论创新每种任务类型的"零件清单"——必选/可选组件、组装顺序、领域约束。按清单逐项产出。

---

### 1 理论研究

#### T1-1 文献综述

- **必选组件**: 检索范围、检索策略、文献清单、文献综述
- **可选组件**: 文献管理工具、引文分析、研究前沿图
- **组装顺序**: 确定检索范围 → 设计检索策略 → 检索文献 → 筛选文献 → 撰写综述
- **约束**: 范围必须全面；策略必须系统；综述必须深入
- **格式**: 文献综述（Markdown）

#### T1-2 理论分析

- **必选组件**: 理论梳理、理论评价、理论空白、研究方向
- **可选组件**: 理论对比、理论图谱、理论演进
- **组装顺序**: 梳理现有理论 → 评价理论优劣 → 识别理论空白 → 确定研究方向
- **约束**: 梳理必须完整；评价必须客观；空白必须明确
- **格式**: 理论分析报告（Markdown）

#### T1-3 理论批判

- **必选组件**: 批判对象、批判角度、批判依据、批判结论
- **可选组件**: 批判方法、批判标准、批判证据
- **组装顺序**: 确定批判对象 → 选择批判角度 → 收集批判依据 → 形成批判结论
- **约束**: 对象必须明确；角度必须多元；依据必须充分
- **格式**: 理论批判报告（Markdown）

#### T1-4 理论综合

- **必选组件**: 综合对象、综合方法、综合视角、综合结论
- **可选组件**: 综合框架、综合模型、综合图谱
- **组装顺序**: 确定综合对象 → 选择综合方法 → 形成综合视角 → 得出综合结论
- **约束**: 对象必须全面；方法必须科学；视角必须创新
- **格式**: 理论综合报告（Markdown）

### 2 理论构建

#### T2-1 概念定义

- **必选组件**: 核心概念、概念内涵、概念外延、概念体系
- **可选组件**: 概念演进、概念对比、概念图谱
- **组装顺序**: 确定核心概念 → 定义概念内涵 → 界定概念外延 → 建立概念体系
- **约束**: 概念必须清晰；内涵必须准确；外延必须明确
- **格式**: 概念定义文档（Markdown）

#### T2-2 理论框架

- **必选组件**: 理论要素、关系结构、框架图示、框架说明
- **可选组件**: 框架演进、框架对比、框架应用
- **组装顺序**: 确定理论要素 → 分析关系结构 → 绘制框架图示 → 撰写框架说明
- **约束**: 要素必须完整；结构必须合理；图示必须清晰
- **格式**: 理论框架文档（Markdown）

#### T2-3 理论模型

- **必选组件**: 模型变量、模型关系、模型公式、模型说明
- **可选组件**: 模型假设、模型推导、模型验证
- **组装顺序**: 确定模型变量 → 分析模型关系 → 建立模型公式 → 撰写模型说明
- **约束**: 变量必须明确；关系必须合理；公式必须准确
- **格式**: 理论模型文档（Markdown）

#### T2-4 理论假说

- **必选组件**: 假说内容、假说依据、假说预测、假说验证
- **可选组件**: 假说推导、假说对比、假说应用
- **组装顺序**: 提出假说内容 → 阐述假说依据 → 明确假说预测 → 设计假说验证
- **约束**: 内容必须明确；依据必须充分；预测必须可验证
- **格式**: 理论假说文档（Markdown）

- **格式**: 理论阐述文档（Markdown）
