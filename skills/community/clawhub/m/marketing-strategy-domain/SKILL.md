# 品牌营销战略决策知识参考库

## 定位

本技能是 **Universal Task OS 的领域负载物仓库**，不包含任何执行框架。只提供品牌营销的"是什么"和"长什么样"——执行全部委托UTOS。

| 本技能提供 | UTOS消费方式 |
|-----------|-------------|
| 营销任务清单+依赖拓扑 | 内容轴·清单法的任务目录 + 执行轴·管线编排的依赖输入 |
| 分析模型要求 | 内容轴·清单法的组件清单 |
| 优秀范本（待填充） | 内容轴·样本法的样本 |

## 三层结构

```
第一层：任务清单与依赖拓扑   →  references/marketing-catalog.md
第二层：分析模型要求清单      →  references/marketing-requirements.md
第三层：优秀范本库            →  references/exemplars.md
```

## 依赖声明

本技能**强依赖** Universal Task OS (universal-task-os)。没有UTOS，本技能只有参考查阅能力，无法执行任何营销分析任务。

**加载检查流程**（每次激活时执行）：

1. 检测 `universal-task-os` 技能是否已安装
2. **未安装** → 自动安装 `universal-task-os` 技能
3. **安装成功** → 同时加载UTOS，按本技能"使用规则"执行
4. **安装失败** → 降级为**只读参考模式**：
   - ✅ 允许：查阅任务清单、模型要求、范本索引
   - ❌ 拒绝：任何涉及策略产出、管线编排、分析报告的任务，并提示"需先安装 Universal Task OS"

**任务模式判定**：

| 任务类型 | 无UTOS | 有UTOS |
|---------|--------|--------|
| 查阅任务清单/模型要求/范本 | ✅ 只读参考 | ✅ 完整 |
| 按清单法产出营销策略 | ❌ 拒绝 | ✅ UTOS编排执行 |
| 依赖拓扑推导分析管线 | ❌ 拒绝 | ✅ UTOS执行轴 |
| 数据验证校验点插入 | ❌ 拒绝 | ✅ UTOS守护单元 |

## 使用规则

1. **依赖检查**：激活时按上述流程检测并安装UTOS
2. **首次加载**：读取 `references/marketing-catalog.md`，获取域分类、依赖拓扑、UTOS元操作映射提示
3. **按需深入**：确认目标任务类型后，读取 `references/marketing-requirements.md` 获取组件清单
4. **委托UTOS**：将任务清单作为清单法输入、范本作为样本法输入、依赖拓扑作为管线编排输入

## 品牌营销特有维度

| 维度 | 说明 | 影响 |
|------|------|------|
| 分析模型驱动 | 每个任务对应1-3个经典分析模型 | SWOT/五力/PESTEL/4P/AARRR等模型选择是核心决策 |
| 数据依赖 | 分析结论必须有数据支撑 | 无数据的分析是猜测，不是策略 |
| 多模型交叉验证 | 关键决策需多个模型交叉验证 | 单一模型可能有盲区 |
| 行业差异 | 不同行业的模型适用性不同 | B2B/B2C/快消/SaaS的模型选择差异大 |
| 时间维度 | 营销策略有短期/中期/长期之分 | 战略(1-3年)/战术(季度)/执行(月度/周)层次不同 |

## 分析模型要求清单

### S1-01 宏观环境扫描
- **必选组件**: PESTEL六大维度分析（政治/经济/社会/技术/环境/法律）、关键影响因素、对业务的影响程度
- **可选组件**: 趋势预测、风险评级、机会窗口
- **组装顺序**: 数据收集→六维度逐一分析→影响程度评估→关键因素排序→结论
- **约束**: 每个维度至少3个影响因素；结论必须有数据/案例支撑
- **格式**: PESTEL分析矩阵（Markdown表格）

### S1-02 行业竞争分析
- **必选组件**: 五力评分（供应商议价力/买方议价力/替代品威胁/新进入者威胁/竞争强度）、行业吸引力结论
- **可选组件**: 各力量的关键驱动因素、竞争格局图
- **组装顺序**: 行业定义→五力逐一评估→评分→吸引力结论→战略启示
- **约束**: 每个力量必须有具体证据；评分1-5分制
- **格式**: 五力分析报告（Markdown）

### S1-03 内部能力评估
- **必选组件**: SWOT四象限（优势/劣势/机会/威胁）、SO/WO/ST/WT策略组合
- **可选组件**: 优先级排序、关键成功因素
- **组装顺序**: 内部调研→外部扫描→四象限填写→策略组合推导→优先级排序
- **约束**: 每象限≤5条；策略组合必须基于SWOT交叉
- **格式**: SWOT矩阵+策略组合（Markdown）

### S2-02 差异化策略
- **必选组件**: 差异化维度（产品/服务/渠道/品牌/技术）、差异化价值主张、成本分析
- **可选组件**: 价值创新机会、蓝海战略画布
- **组装顺序**: 竞争分析→差异化维度识别→价值创新分析→成本评估→策略设计
- **约束**: 差异化必须可感知（用户能区分）；成本必须可控
- **格式**: 差异化策略方案（Markdown）

### S3-01 品牌识别设计
- **必选组件**: Kapferer六维度（体格/关系/反射/个性/文化/自我形象）、品牌金字塔
- **可选组件**: 品牌原型、品牌故事、品牌声音
- **组装顺序**: 品牌现状分析→六维度设计→品牌金字塔→品牌声音定义
- **约束**: 六维度必须一致（不可互相矛盾）；品牌个性必须可执行
- **格式**: 品牌识别文档（Markdown）

### S3-02 价值主张设计
- **必选组件**: 客户画像、客户痛点列表、客户收益列表、产品服务列表、收益创造方式
- **可选组件**: 价值主张画布、竞争对比
- **组装顺序**: 客户调研→痛点排序→收益排序→产品服务匹配→画布填写→验证
- **约束**: 痛点必须来自客户调研（不可AI编造）；收益必须可量化
- **格式**: 价值主张画布（Markdown）

### S4-01 市场细分与目标
- **必选组件**: 细分变量（地理/人口/心理/行为）、细分市场描述、目标市场选择理由、定位声明
- **可选组件**: 市场规模估算、竞争密度分析
- **组装顺序**: 市场调研→细分变量选择→市场细分→评估吸引力→选择目标→定位
- **约束**: 细分必须可衡量/可进入/足量/可操作；目标市场≤3个
- **格式**: STP分析报告（Markdown）

### S4-03 定价策略
- **必选组件**: 定价方法（成本加成/竞争定价/价值定价）、价格区间、定价理由
- **可选组件**: 价格弹性分析、心理定价技巧、折扣策略
- **组装顺序**: 成本分析→竞品价格调研→价值评估→定价方法选择→价格确定→弹性测试
- **约束**: 定价必须覆盖成本；价格必须与品牌定位一致
- **格式**: 定价方案（Markdown）

### S5-01 增长漏斗设计
- **必选组件**: AARRR五阶段定义、各阶段核心指标、当前基线数据、目标值
- **可选组件**: 漏斗可视化、瓶颈识别、优化策略
- **组装顺序**: 用户旅程梳理→五阶段定义→指标设定→基线采集→瓶颈识别→优化策略
- **约束**: 每阶段必须有可量化指标；瓶颈必须有数据支撑
- **格式**: 增长漏斗文档（Markdown）

### S6-01 用户画像设计
- **必选组件**: ICP描述（人口特征/行为特征/心理特征/场景描述）、用户分层
- **可选组件**: 用户访谈记录、数据来源说明
- **组装顺序**: 数据收集→特征提取→画像构建→分层→验证
- **约束**: 画像必须基于真实数据；不可凭空编造用户特征
- **格式**: 用户画像卡片（Markdown）

### S6-03 客户旅程地图
- **必选组件**: 旅程阶段（认知→考虑→购买→使用→推荐）、每阶段触点、情绪曲线、痛点/机会点
- **可选组件**: 后台支持系统、改进方案
- **组装顺序**: 阶段划分→触点识别→情绪标注→痛点标注→机会点识别→改进方案
- **约束**: 每阶段至少1个触点；痛点必须有情绪依据
- **格式**: 客户旅程地图（Markdown）

### S7-01 指标体系设计
- **必选组件**: HEART五维度指标（愉悦度/参与度/采纳率/留存率/任务完成率）、数据采集方式
- **可选组件**: 指标权重、目标值、仪表盘设计
- **组装顺序**: 业务目标→指标映射→HEART框架填写→采集方式→仪表盘设计
- **约束**: 每个维度至少1个可量化指标；指标必须可采集
- **格式**: 指标体系文档（Markdown）

### S8-03 OKR目标设定
- **必选组件**: 目标(Objective)、关键结果(Key Results，≤5个)、时间表、负责人
- **可选组件**: 依赖关系、风险预案
- **组装顺序**: 策略分解→目标设定→关键结果定义→时间表→责任分配
- **约束**: 目标必须有挑战性但可达成；关键结果必须可量化
- **格式**: OKR文档（Markdown）

### S9-01 销售漏斗设计
- **必选组件**: 漏斗阶段（线索→MQL→SQL→商机→成交）、转化率目标、各阶段动作
- **可选组件**: 漏斗周期、SLA定义、自动化策略
- **组装顺序**: 销售流程梳理→阶段定义→转化率设定→动作设计→自动化
- **约束**: 每阶段必须有明确的进入/退出标准；转化率必须基于历史数据
- **格式**: 销售漏斗文档（Markdown）

## 与UTOS的接口

当UTOS处理品牌营销领域任务时：

- **Step 0 三轴判定**：营销任务通常为复杂+结构化+数据驱动 → 执行轴+内容轴
- **Step 1 领域校准**：R1(信息密度)=高（数据+模型）+R2(创造性)=中（策略设计）+R3(交互性)=高（多方协调）+R4(规范性)=中（商业规范）+R5(迭代性)=高（持续优化）→ S高C高A中I高G中循环高
- **Step 2 内容轴**：清单法用本技能的marketing-requirements；样本法用本技能的exemplars
- **Step 3 执行轴**：管线编排基于本技能的依赖拓扑自动推导元操作序列
- **Step 4 交付**：G类守护单元自动插入数据验证校验点（结论必须有数据支撑）
