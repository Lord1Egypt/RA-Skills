# 交付场景模板

## 进度报告

```markdown
# 进度报告 | {项目名称}

**报告周期**：{起始日期} — {截止日期}
**项目状态**：🟢 正常 / 🟡 有风险 / 🔴 延期

## 一、本期进展

### 已完成
- ✅ {完成事项1} — {交付成果}
- ✅ {完成事项2} — {交付成果}

### 进行中
- 🔄 {事项3} — 进度 **{X}%**，预计 {日期} 完成

### 待启动
- ⏳ {事项4} — 计划 {日期} 开始

## 二、关键数据

| 指标 | 计划 | 实际 | 状态 |
|:---|:---|:---|:---|
| 整体进度 | {X}% | {X}% | ✅/⚠️ |
| 里程碑节点 | {计划日期} | {实际日期} | ✅/⚠️ |

## 三、风险与问题

⚠️ {风险描述} — 影响：{说明} | 应对：{措施}

## 四、下期计划
1. **[优先]** {计划事项1}
2. {计划事项2}

## 五、需要甲方配合
- {需要客户做的事1}
- {需要客户做的事2}
```

💡 进度报告的核心价值：让客户感知你在推进，降低焦虑。定期发送比一次性交付更容易建立信任。

## 验收报告

```markdown
# 验收报告 | {项目名称}

**甲方**：{客户名} | **乙方**：{你的名字} | **验收日期**：{日期}

## 一、交付物清单

| 序号 | 交付物 | 合同约定 | 实际交付 | 状态 |
|:---|:---|:---|:---|:---|
| 1 | {交付物1} | {规格} | {实际规格} | ✅ 符合 |
| 2 | {交付物2} | {规格} | {实际规格} | ✅ 符合 |

## 二、验收结果

- 功能完整性：✅ 符合合同约定
- 质量标准：✅ 符合验收标准
- 交付时间：✅ 按时 / ⚠️ 延期 {X} 天

## 三、遗留事项

| 事项 | 说明 | 预计完成时间 |
|:---|:---|:---|
| {遗留项1} | {说明} | {日期} |

## 四、验收确认

甲方确认以上交付物符合合同约定，同意验收通过。

| | 甲方 | 乙方 |
|:---|:---|:---|
| 签名 | | |
| 日期 | | |
```

💡 验收报告签署后是催款的直接法律凭证。务必让甲方签字确认。
⚠️ 遗留事项必须列清，否则甲方会以此拖延验收。

## 满意度调查

```markdown
# 满意度调查 | {项目名称}

**项目**：{名称} | **完成日期**：{日期}

## 评分（请勾选）

| 维度 | 1⭐ 很不满意 | 2⭐ 不满意 | 3⭐ 一般 | 4⭐ 满意 | 5⭐ 非常满意 |
|:---|:---|:---|:---|:---|:---|
| 整体满意度 | | | | | |
| 沟通响应速度 | | | | | |
| 交付质量 | | | | | |
| 专业程度 | | | | | |
| 性价比 | | | | | |

## 开放问题
1. 本次合作中您最满意的是什么？
2. 有什么是我们可以改进的？
3. 近期是否有其他合作需求？（如有，请简要描述）

---

🎁 **老客户专属福利**：回复此调查即享下次合作 **9折** 优惠（有效期 {X} 个月）
```

💡 发送时机：验收通过后 3-5 个工作日。目的不仅是收集反馈，更是铺垫复购和转介绍。
