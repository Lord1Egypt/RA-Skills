# Eva-skill 资产状态归一表

本文件是主动回捞、表达资产状态和素材降级规则的唯一真源。

使用方式：

- `01b` 第一次主动回捞：只回捞 `idea-card`、经历卡和 `persona-card`，判断标题承诺能否被真实素材兑现。
- `02a` 第二次主动回捞：回捞 `persona-card`、`voice-card`、`idea-card` 和经历卡，确认表达资格、用户文风和正文素材厚度。
- 主动回捞只读，不保存、不推断、不编造；保存仍必须由用户明确触发。

## 主动回捞最小协议

1. 只扫描当前运行项目的 `./eva-memory/`，不得扫描 Eva-skill 仓库本体或无关目录。
2. 优先扫描 `idea-cards/`、`persona/`、`voice/`。
3. 匹配依据只使用文件名、frontmatter、keywords、原文摘要和卡片里的真实字段。
4. 默认只返回 1-3 张最相关资产。
5. 未扫描、未命中、已命中必须如实区分。
6. 未扫描不得写成未命中；未命中不得推断、补全或编造。

## 状态归一

| 状态类型 | 包含状态 | 处理 |
|---|---|---|
| 已命中类 | 已提供 / 主动回捞-已命中 / 已授权回捞-已命中 | 按已提供处理，只使用真实字段 |
| 未命中类 | 主动回捞-未命中 / 已授权回捞-未命中 / 未命中 | 不得当作已有素材；需要真实经历、身份、文风或数据的位置占位或降级 |
| 未扫描类 | 未扫描-无目录 / 未扫描-无权限 / 未扫描-未执行 | 不得声称已回捞；如需继续，只能按无素材降级 |
| 未提供类 | 未提供 / 暂无 / 待补充 | 需要真实经历、身份、文风或数据的位置占位或降级 |
| 裸授权 | 已授权回捞 | 必须归一成已命中或未命中；无法确认时按未命中处理 |

## persona-card 状态

允许值：

```text
已提供
主动回捞-已命中
主动回捞-未命中
未扫描-无目录
未扫描-无权限
未扫描-未执行
已授权回捞-已命中
已授权回捞-未命中
未提供-需占位
未提供-低个性化继续
```

处理：

- 命中或已提供：可用于表达资格、身份来源、选择代价和隐私边界。
- 未命中、未扫描或未提供：涉及“我为什么能讲”、经历、真实体验的位置必须用占位符或降级。
- 不得写“未提供-中性口语”或“未提供-沿用本轮原稿”。

## voice-card 状态

允许值：

```text
已提供
主动回捞-已命中
主动回捞-未命中
未扫描-无目录
未扫描-无权限
未扫描-未执行
已授权回捞-已命中
已授权回捞-未命中
未提供-中性口语
未提供-沿用本轮原稿
```

处理：

- 命中或已提供：正文撰写必须优先保护用户表达文风。
- 未命中、未扫描或未提供：只能用中性口语或沿用本轮原稿，不得默认写成 Eva 腔或璐璐腔。
- 不得写“未提供-需占位”或“未提供-低个性化继续”。

## 是否阻塞

确认资产状态是必须的，但不等于每次都要追问用户。

| 场景 | 做法 |
|---|---|
| 高个性化、高信任感、强表达资格、像自己说的、用我的风格写、需要大量个人经历 | 只停下来确认一个最高优先级资产；没有对应资产时不进入高置信度正文 |
| 普通写稿、低置信度试结构、局部改稿、只优化开头 | 内部标注两卡状态，继续低个性化写作，不连续追问 |

默认低个性化继续状态：

```text
persona-card 状态：未提供-低个性化继续
voice-card 状态：未提供-中性口语
```

只优化开头不因缺两卡而阻塞。
