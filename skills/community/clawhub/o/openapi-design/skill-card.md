## Description: <br>
API design specification and best practices skill for designing new APIs, reviewing existing APIs, generating API documentation, and standardizing interface specifications. <br>

This skill is ready for commercial/non-commercial use. <br>

## Publisher: <br>
[sydpz](https://clawhub.ai/user/sydpz) <br>

### License/Terms of Use: <br>
MIT-0 <br>


## Use Case: <br>
Developers and API reviewers use this skill to design REST-style APIs, standardize endpoint and error conventions, generate OpenAPI documentation, and review existing API designs. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Example tokens, passwords, URLs, and shell commands may be copied without adaptation. <br>
Mitigation: Treat bundled values as examples only, replace placeholders, and run optional documentation-tool commands only when those tools are intentionally needed. <br>
