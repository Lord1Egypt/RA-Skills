# AgentMem — 项目介绍

> **市场上首个同时管理「Agent记忆」+「Agent调度」的开源方案**

---

## 一句话

AgentMem 是一个为多Agent系统设计的**记忆+调度闭环框架**——Agent不只是记住，还知道该派谁去干、以及上次干得怎么样。

---

## 为什么要做这个

现在市面上的AI记忆系统（Mem0、MemOS、Memori）解决的是**「记住」**的问题。但在真实的多Agent场景中，还有一个同样重要的问题——**「谁该干什么」**。

举个常见的场景：用户说「帮我查一下这个数据，然后分析一下趋势」——

| 问题 | 对应能力 |
|------|---------|
| 哪个Agent该接这个任务？ | **调度** |
| 之前有没有聊过这个话题的上下文？ | **记忆** |
| 上次派给谁了？他干得怎么样？ | **调度+记忆联动** |

现有方案只解决了记忆这一半。调度靠硬编码if-else、靠人工指定。而且记忆和调度完全不互通——Agent记住了但不知道怎么用，调度决策了但不知道历史表现。

**这就是AgentMem要解决的问题。**

---

## 核心创新

### 1. 四层记忆分级（HOT→WARM→COLD→ARCHIVE）
按时间自然衰减，不像其他方案所有记忆一视同仁。2小时内的事随时取，30天前的旧事自动归档。

### 2. 跨通道互通
同一Agent在不同通道（webchat、飞书、Slack、Telegram）的记忆自动共享。
用户在A通道聊完，B通道的Bot也能接上话——不用重复说一遍。

### 3. 调度+记忆闭环
```
用户请求 → 意图识别 → Agent调度 → 执行 → 日志 → 优化下次调度
```
每次调度自动记录完整链路：谁在哪个通道发起了什么任务、派给了谁、结果如何、花了多久。
下次调度优先选历史上成功率高的Agent，越用越准。

### 4. 17个记忆模块，开箱即用
事实提取、BM25+向量融合检索、矛盾检测、知识图谱因果链、遗忘机制、
主动召回、记忆反馈修正、自我审查规律库……覆盖记忆的全生命周期。

## 适合谁用

- **多Agent系统开发者** — 需要管理多个AI Agent的记忆和调度
- **Agent平台/框架开发者** — 需要内置记忆+调度能力
- **AI创业团队** — 需要开箱即用的记忆系统，不想从零造轮子
- **个人开发者/研究者** — 学习AI记忆系统的完整实现

---

## 不适合谁用

- ❌ 需要云端SaaS服务 — AgentMem纯本地，不上云
- ❌ 需要百万级文档RAG — 小规模向量库，够用但不够大
- ❌ 需要GPU加速 — 全CPU推理，够用但不算快

---

## 技术栈

- Python 3.10+
- chromadb（向量存储，本地SQLite）
- 纯CPU推理（内置embedding模型）
- **零外部API依赖，完全本地运行**
- 依赖只有 chromadb 一个包

---

## 快速开始

```bash
pip install -e .

# 写入记忆
python -m agent_mem.core.hot_cache write --agent main --channel webchat --text "用户偏好简洁回答" --importance 7

# 跨通道查询（查所有通道）
python -m agent_mem.core.hot_cache query --agent main --limit 5

# 调度日志
python -m agent_mem.core.dispatch_logger stats

# 运行记忆引擎
python -m agent_mem.core.engine_v2 --mode daily
```
