## Description: <br>
Sets up or standardizes a fresh OpenClaw workspace as the Fire Dragon Fruit Architecture, with one strong main agent, isolated rescue, durable file memory, project truth files, heartbeat, cron, official Feishu integration, Ollama memory embeddings, and an explicit evolution loop. <br>

This skill is ready for commercial/non-commercial use. <br>

## Use Case: <br>
Developers and OpenClaw operators use this skill to install or normalize a durable personal operating setup with file-based memory, project source-of-truth documents, scheduled maintenance, and recovery topology. It is intended for system-level OpenClaw refactors rather than small one-file changes or prompt-only tweaks. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: The skill can make broad, durable changes to an OpenClaw workspace, including memory, project, scheduling, and integration files. <br>
Mitigation: Review target files before applying changes, keep version control or backups, and merge existing project truth instead of overwriting it. <br>
Risk: Heartbeat, cron, isolated cron, Feishu, or other external outputs may read, write, or publish information over time. <br>
Mitigation: Confirm scheduled jobs, file targets, and external-output approvals before enabling automation. <br>
