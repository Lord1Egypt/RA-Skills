## 📨 场景1：邮件

**适用人群**：商务人士、跨境工作者

**推荐频率**：每15-30分钟

**推荐理由**：
- 重要紧急：客户邮件不能漏
- 有明确判断标准：已读/未读/重要
- 适合批量处理：攒一波再处理

**Cron 示例**：
```bash
# 每30分钟
0,30 * * * *
# 每15分钟（高频）
*/15 * * * *
```

**心跳检查清单**：
```markdown
## 邮件检查
- [ ] 检查收件箱（最近30分钟）
- [ ] 标记重要邮件（发件人/主题判断）
- [ ] 汇总紧急事项
```

---

## 📅 场景2：日历/日程

**适用人群**：会议多、时间碎

**推荐频率**：每天2-3次（早报+晚报+会前提醒）

**推荐理由**：
- 时间敏感：错过会议就是事故
- 可提前处理：知道会议内容可以准备
- 定期回顾：避免遗忘待办

**Cron 示例**：
```bash
# 每天早7点、午12点、晚7点
0 7,12,19 * * *
# 会前30分钟提醒（单独定时）
30 9,14,16 * * 1-5
```

**心跳检查清单**：
```markdown
## 日历检查
- [ ] 今天日程概览
- [ ] 未来3天重要事件
- [ ] 待准备事项（如PPT）
```

---

## 📈 场景3：持仓监控

**适用人群**：投资者、持有股票/基金者

**推荐频率**：交易时段每小时

**推荐理由**：
- 时效性强：涨跌需要及时知道
- 情绪管理：避免过度盯盘
- 可设阈值告警：大幅波动时主动提醒

**Cron 示例**：
```bash
# A股：周一到周五 9:30-15:00，每小时
0 10,11,13,14 * * 1-5
# 港股：10:00-12:30, 14:00-16:00
0 10,11,14,15 * * 1-5
```

**心跳检查清单**：
```markdown
## 持仓监控
- [ ] 主要持仓涨跌
- [ ] 大盘概况（沪深300/恒生）
- [ ] 有无异常波动（>5%）
```

## 📊 心跳日程可视化

帮助用户直观看到每天的心跳分布，避免过于密集。

### 文本可视化示例

```
08:00 ██ 邮件 + 日历
09:00 ██ 代码质量
10:00 ██ 持仓监控
11:00 ██ 持仓监控
12:00 ██ 邮件 + 新闻
13:00 ██ 持仓监控
14:00 ██ 持仓监控
15:00 ██ 持仓监控
18:00 ██ 日历 + 待办
19:00 ██ 新闻
20:00 ██ 邮件

图例：██ = 心跳触发点
```

### 密集度评估

| 类型 | 描述 | 建议 |
|------|------|------|
| **稀疏** | 每小时0-1次 | 可以增加 |
| **适中** | 每小时1-2次 | 最佳状态 |
| **密集** | 每小时2-3次 | 注意过载 |
| **过载** | 每小时3次以上 | 必须精简 |

### 如何调整

如果可视化显示太密集：
1. **合并同类**：邮件+日历一起检查
2. **降频**：每小时→每2小时
3. **砍掉**：优先级最低的砍掉

## ⚠️ 心跳过载警示

**问题**：心跳太多 = 焦虑 = 忽略所有

**建议**：
- 同时激活的心跳项目 ≤ 5个
- 频率高的项目：选最关键的
- 合并同类项：邮件+日历可以合并检查

**心法**：心跳是助手巡逻，不是主人焦虑源
