# 📐 数据结构说明

> 理解数据结构有助于理解日报记录如何流转。

---

## 📊 数据存储架构

```
┌─────────────────┐     写入      ┌─────────────────────────┐
│   本地记忆文件    │ ──────────→  │  腾讯文档共享多维表格     │
│ (memory/)       │              │  （每日进展镜像表）       │
│                 │ 读取/查询     │                         │
│ user-config.md  │ ←───────────  │  教研人员表              │
│ daily-records/  │              │  项目信息表              │
│                 │              │  每日进展镜像表（共享）   │
└─────────────────┘              └─────────────────────────┘
```

---

## 📋 字段结构

### 每日日报记录字段

| 字段 | 类型 | 说明 | 示例 |
|---|---|---|---|
| 日期 | string | 毫秒时间戳 | "1748601600000" |
| 姓名 | string | record_id（教研人员表） | "rpeiUU" |
| 项目名称 | string | record_id（项目信息表） | "rL5tMS" |
| 工时 | number | 当日该项目工时（小时） | 8 |
| 工作内容 | string | 具体工作描述 | "完成图形化课程第3节课件" |
| 类型 | option | 开发/非研发 | "开发" |
| 请假工时 | number | 当日请假小时数 | 0 |
| 请假说明 | string | 请假原因 | "" |
| 加班工时 | number | 当日加班小时数 | 0 |
| 加班说明 | string | 加班原因 | "" |

### 请假/加班特殊处理

请假和加班不归入项目，而是在月底单独汇总：

| 类型 | 存储方式 | 月报体现位置 |
|---|---|---|
| 请假 | 单独字段，记录日期+说明 | 2026年X月课程研发投入 表 |
| 加班 | 单独字段，记录日期+说明 | 2026年X月课程研发投入 表 |
| 普通工作 | 归入对应项目 | 教研老师-XXX 页面 |

---

## 🔗 表关联关系

```
教研人员表
  └── record_id = 每日记录.姓名

项目信息表
  └── record_id = 每日记录.项目名称

每日进展表（共享）
  └── 姓名 → 教研人员表（lookup）
  └── 项目名称 → 项目信息表（lookup）
  └── 日期 → 用于月度汇总筛选
```

---

## 📁 本地存储文件

| 文件 | 内容 |
|---|---|
| `memory/user-config.md` | 用户配置（姓名、授权状态、record_id） |
| `memory/daily-records/YYYY-MM.md` | 每月日报摘要缓存（用于快速查询） |

## 🔄 月报数据转换逻辑

### 日报 → 月报"教研老师-XXX"页

```
输入：某人员当月所有日报记录
处理：按项目聚合工时，合并同类工作内容
输出：
  序号 | 项目 | 研发阶段 | 部门 | 年 | 月 | 工时 | 备注
```

### 日报 → 月报"课程研发投入"页

```
输入：某人员当月请假/加班记录
处理：按日期排序，生成说明文字
输出：
  姓名 | 请假工时 | 请假说明 | 加班工时 | 加班说明
```

---

## ⚙️ 项目名匹配规则

当用户说"做了AI课程"时，系统按以下顺序匹配项目名：

1. **模糊关键词匹配** — "AI课程" → "AI微信小程序"
2. **同义词映射** — "机器狗" → "灵芯派机器狗V1.0"
3. **未知项目处理** — 记录为"教研内部工作"，提示用户确认
