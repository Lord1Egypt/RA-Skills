# 视觉策略指南

> 决定"一份 deck 长什么样"的核心策略。配合 `Prompt指南.md` 使用。

---

## 策略 1：A+B 混合（推荐 80% 商务 deck）

**一份 deck 不要纯用一种视觉风格**。纯信息图显得冷，纯插画显得幼稚。

### 分工
| 流派 | 用于哪种页 | 图文比 |
| --- | --- | --- |
| **A · 信息图** | 工具页、矩阵、流程、数据表、表单 | 60/40 |
| **B · 扁平插画** | 章节扉页、概念页、隐喻页、互动页 | 70/30 |
| **纯排版** | 封面、过渡、仪式、金句 | 80/20（文字即主视觉） |

### 实战分页决策表
| 页面类型 | 推荐流派 | 例子 |
| --- | --- | --- |
| 封面 | 纯排版 | 大字 + 竖线 + 胶囊 tag |
| 章节扉页 | B 插画 | 冰山 / 三岔路口 / 主管分工 / 齿轮 / 回顾书桌 |
| 工具方法（SMART / RACI / DECIDE） | A 信息图 | 字母 badge + 表格 + 图标 |
| 流程图（PDCA / 5 Why） | A 信息图 | 循环圆 / 阶梯 / 箭头链 |
| 矩阵（授权四象限 / 情境领导） | A 信息图 | 2×2 色块 / 多行标签 |
| 过渡（中场休息 / 上下半场） | 纯排版深色 | 深色底 + 大字 + 几何装饰 |
| 互动/自测 | B 插画 + A 结构 | 小插画 + 问题列表 + 得分卡 |
| 收官（工具箱 / 谢谢） | A 网格 / 纯排版 | 6 列工具名 / 大字谢谢 + 签名 |

---

## 策略 2：分层派文字密度（硬性上限）

不同页面有不同功能，**文字密度必须分层**，不能一刀切。

| 模板 | 主正文上限 | 允许额外元素 |
| --- | --- | --- |
| 封面 | 30 字 | 副标 + 小标签 |
| 章节扉页 | 15 字（副标句） | Tag 胶囊（不计字数） |
| 过渡页 | 15 字 | 时钟 / 几何（不计） |
| 工具信息图 | 60 字 | 表格标签 / 数据数字（不计） |
| 互动仪式 | 60 字正文 | 得分卡 / 承诺卡（独立计） |

### 超标判断
- Prompt 里写满 5 行中文说明 → 大概率超标
- 一页有 3 个以上"一句话金句" → 砍 2 个
- 底部"收尾金句"超过 25 字 → 改短

### 反例（v1 失败）vs 正例（v2 成功）
| 反例 | 正例 |
| --- | --- |
| 每页塞表格 + 说明 + 注释 | 扉页只放章节数字 + 5 字标题 + 副标 |
| 过渡页塞时长 + 3 句提示 | 过渡只放"中场休息" + "15 min" + 1 句 |
| 工具页硬挤 Excel 样式表格 | 工具页用 badge + 色块 + 图标化表格 |

## 策略 3：色彩 3 色生存线（扁平插画必须）

**纯 2 色（主墨 + 强调）不够画扁平插画**。插画里的人物、服饰、道具、场景都要区分层次，只有 2 色会挤成一团。

### 标准 3 色体系
- 主墨色（深蓝/黑/深绿）· 做线条与主标题
- 强调色 1（暖橙/朱砂/荧光黄）· 做关键数据、CTA、第一重突出
- 辅助色（薄荷绿/奶茶/雾粉）· 做插画人物服饰、次要元素、第二重对比

### 实战配色对照
| 风格方向 | 主墨 | 强调 | 辅助 | 背景 |
| --- | --- | --- | --- | --- |
| 工具派商务 | `#1E3A5F` 深海军蓝 | `#E8763A` 暖橙 | `#7CB9A0` 薄荷绿 | `#FAFAF7` 暖米 |
| 中式古典 | `#1A1A1A` 墨黑 | `#A63A2A` 朱砂 | `#B88A3E` 古金 | `#F7F1E3` 宣纸 |
| 科技极简 | `#0A0A0A` 近黑 | `#FF6B35` 荧光橙 | `#00D4FF` 赛博蓝 | `#FAFAFA` 冷白 |
| IDEO 工作坊 | `#2C3E2E` 墨绿 | `#D67B3E` 烧橘 | `#F4D6A8` 杏黄 | `#FFF8EC` 奶油 |

## 策略 4：视觉 DNA 三件套（每份 deck 必须有）

一份 deck 要有**视觉 DNA**——用户看完第 3 页就能认出"这是同一份 deck"。三件套：

### ① 标志性装饰元素
- v1 工具派：左上角橙色 3-4px 竖线 + 小标签
- 中式：左侧古金色竖线 + 右上角朱砂印章
- 科技：顶部单根彩色发丝线

**强制**：所有页都要有，不能只用在封面。

### ② 胶囊 Tag 系统
- 胶囊形（圆角 999px）
- 3 色循环：浅蓝 / 浅橙 / 浅绿
- 扉页右下、工具页顶部、收官全图都用同一套

### ③ 页脚统一
- 一条 1px 暖浅灰横线
- 左端：deck 标题缩写
- 右端：页码 / 章节

**强制**：除了封面和过渡页，其他所有页都带统一页脚。

## 策略 5：插画隐喻库（B 流派专用）

每章扉页必须有一个**定制隐喻插画**，不能空排版。以下是商管 PPT 的插画隐喻对照表：

| 主题 | 隐喻 |
| --- | --- |
| 看全局 / 系统思考 | 火柴人站海边看冰山 |
| 做决策 / 选择 | 火柴人站三岔路口，正确路径尽头亮灯泡 |
| 分任务 / 派人 | 主管捧任务卡，面前 4 个下属 |
| 盯过程 / 改进 | 火柴人调齿轮 / 推小车上坡 |
| 带人 / 差异化 | 主管面前 4 个不同姿态下属（D1-D4） |
| 复盘 / 回顾 | 火柴人坐书桌前，台灯 + 笔记本 |
| 协同 / 跨部门 | 中心圆 + 放射小圆 + 连接线 |
| 危机 / 风险 | 火柴人站边缘望悬崖 |
| 成长 / 进阶 | 火柴人爬阶梯，每阶一个图标 |
| 选择悖论 | 火柴人拿 5 张问号牌，头顶问号气泡 |

**禁止**的隐喻（AI slop）：
- 握手、西装、会议桌 → 太俗
- 齿轮 + 火箭 → 所有科技公司都用烂
- 水晶球、灯泡、闪电 → AI 第一反应，避开
- 爬山顶插旗 → 励志毒鸡汤
- 天平 → 司法意味太重

---

## 策略 6：字体与排版的 3 条铁律

1. **禁衬线**：PPT 正文禁用宋体/楷体（Nano Banana 渲染衬线中文 50% 崩字）
2. **大字 + 大留白**：主标题至少 ≥ 72pt，与下一元素间距 ≥ 24pt
3. **行距宽松**：正文行距 1.5-1.7，不用单倍行距

---

## 自测清单（Phase 2 Spec 审查必用）

每份 deck 结束 Spec 时，过一遍：

- [ ] 是否纯一种视觉流派？（纯 A 或纯 B 都是红灯）
- [ ] 扉页/过渡/工具页字数是否分层？（一刀切红灯）
- [ ] 配色是否只有 2 色？（插画会发腻）
- [ ] 是否定义了标志性装饰元素？（没有则视觉 DNA 不成立）
- [ ] 是否每个扉页都有具体的插画隐喻？（空排版是浪费）
- [ ] 页脚是否统一？（不统一就是散乱）

有任一 NO → 回 Spec 补。
