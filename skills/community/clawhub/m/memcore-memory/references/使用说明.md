# MemCore 使用说明

## 这是什么

MemCore 是一个记忆自动进化系统，让 AI 不再每次启动都要读 3500+ tokens 的全量记忆，改为按需检索，节省 92% 上下文。同时它会自动从对话中归纳模式、发现故障规律。

## 核心能力

| 能力 | 说明 |
|------|------|
| 记忆压缩 | MEMORY.md(3500 tokens) → 简报(≤500 tokens)，启动自动加载 |
| 5层检索 | 精确搜索 → 语义搜索 → 原则搜索 → 同义词扩展 → 原文grep兜底 |
| 模式归纳 | 从日志中自动发现"哪根管子反复爆" |
| 故障诊断 | 早上9点报告系统状态+历史故障参考方案（只报不修） |
| 自进化 | 用过的记忆升值，没用过的衰减 |

## 日常使用

### 启动时
AI 自动加载 `MEMORY_BRIEF.md`（≤500 tokens），包含：
- 最近活跃的故障模式
- 近7天重要教训
- 待办任务

需要查历史时，让 AI 用：
```
mos search "关键词"
```

### 散会时
说"散会"后，AI 自动执行：
1. 总结今天的结论
2. 提炼教训写入 MEMORY.md
3. MemCore 自动索引新日志 + 归纳新模式 + 衰减旧记忆
4. 更新任务板 + 归档 + 重启

### 每天早上9点
自动收到健康报告，格式如下：

```
📋 EXO2 每日健康报告 (2026-05-16)

🟢 正常项：cron全绿 | 同步正常

🔴 异常项：
  - MiniMax 401 token失效
  - 历史同类故障: 3次
  - 建议方案: 从Token Plan Key换新key
  - 置信度: 100%
```

AI 不会自动修，等你决定。

## 自动维护

每天 04:00 和 16:00 自动运行，不打扰：
- 索引新日志
- 归纳新模式
- 衰减旧记忆
- 刷新简报

## 备份与回退

```bash
cp memcore_backup_<日期>/AGENTS.md ~/.openclaw/workspace/
cp memcore_backup_<日期>/MEMORY.md ~/.openclaw/workspace/
```

原始 MEMORY.md 从未被修改。MemCore 所有数据存在独立 SQLite 文件里。

## 安装到 EXO1

1. 通过 Syncthing 同步 `memcore.skill`
2. 运行 `bash skills/memcore/scripts/install.sh`
3. 修改 AGENTS.md 启动流程（参考 `references/upgrade-guide.md`）
4. 添加 04:00 和 16:00 cron
