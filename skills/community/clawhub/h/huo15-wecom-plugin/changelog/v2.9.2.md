# v2.9.2 (2026-05-07)

**派生 workspace 从 main 种子复制身份文件 — 跳过 OpenClaw 核心 BOOTSTRAP 引导**

## 用户反馈

赵博 v2.9.1 部署后立刻反馈：「我跟 BOT 私聊为什么提示这个？我早就初始化了」，截图显示 BOT 在新派生 workspace 里跑「Hey. I just came online. Who am I? Who are you?」5 步引导（OpenClaw 核心 `BOOTSTRAP.md` 写出来的），让用户填名字 / 风格 / emoji。

## 根因

OpenClaw 核心的 SessionStart 看到 fresh workspace（IDENTITY.md / SOUL.md 不存在）会自动写 `BOOTSTRAP.md` 触发"我是谁/你是谁"对话。这是为陌生用户首次接触 BOT 设计的，**对老用户跨 group/DM 派生 workspace 的场景是误触发**。

v2.9.0/v2.9.1 派生 workspace 是全新空目录 → 老用户每个新群/DM 都被引导一次。

## 修复

`ensureDynamicAgentListed` 在 mkdir workspace 后加一步**种子复制**：

```ts
await seedWorkspaceFromMain({
  targetWorkspace: workspace,
  mainWorkspace: cfg.agents.defaults.workspace,  // 默认 ~/.openclaw/workspace
  seedFiles: ["IDENTITY.md", "SOUL.md", "USER.md", "AGENTS.md"],
});
```

复制策略：

| 文件 | 复制 | 理由 |
|---|---|---|
| `IDENTITY.md` | ✅ | 名字 / 创造物 / vibe / emoji — 跨 agent 共享 |
| `SOUL.md` | ✅ | 用户偏好 / 风格 / 边界 — 跨 agent 共享 |
| `USER.md` | ✅ | 用户名字 / 时区 / 备注 — 跨 agent 共享 |
| `AGENTS.md` | ✅ | 项目规则 / 工作约定 — 跨 agent 共享 |
| `BOOTSTRAP.md` | ❌ | **跳过！** 不复制 = 不触发引导。OpenClaw 核心会检测到 IDENTITY 已存在跳过写 BOOTSTRAP |
| `MEMORY.md` | ❌ | 每个 agent 独立记忆，不污染主工作区 |
| `HEARTBEAT.md` | ❌ | per-agent 心跳节奏 |
| `DREAMS.md` | ❌ | per-agent 计划想法 |
| `TOOLS.md` | ❌ | per-agent 工具偏好 |

## 配置

```yaml
channels:
  wecom:
    dynamicAgents:
      seedFromMainWorkspace: true                                    # v2.9.2 默认 true
      seedFiles: ["IDENTITY.md", "SOUL.md", "USER.md", "AGENTS.md"]  # 默认值
      mainWorkspacePath: "~/.openclaw/workspace"                     # 默认从 agents.defaults.workspace 读
```

## 容错

- main workspace 缺失 → skip 不报错
- 源文件缺失 → skip 不报错
- 目标已存在 → skip（幂等 + 不覆盖用户后续修改）
- 任何 fs 错误 → console.warn 不阻塞 dispatch

## 老 workspace 批量回填命令

v2.9.0/v2.9.1 期间已派生但缺身份文件的老 workspace 一次性回填：

```bash
MAIN=~/.openclaw/workspace
for ws in ~/.openclaw/workspace-wecom-*; do
  for f in IDENTITY.md SOUL.md USER.md AGENTS.md; do
    [ -f "$MAIN/$f" ] && [ ! -f "$ws/$f" ] && cp "$MAIN/$f" "$ws/$f" && echo "seeded: $ws/$f"
  done
  # 删 BOOTSTRAP.md（已派生 workspace 残留的引导脚本）
  [ -f "$ws/BOOTSTRAP.md" ] && rm "$ws/BOOTSTRAP.md" && echo "removed BOOTSTRAP: $ws"
done
```
