# 问题诊断框架与改善方向库

## 端到端诊断模型：ODP-I²（订单-交付-问题-改善）

采用"从接单到交付"全流程扫描法，按照4大维度对企业进行诊断。

### 诊断4大维度

| 维度 | 权重 | 描述 |
|:----:|:----:|------|
| **流程(Process)** | 30% | 业务流程完整性、标准化、执行落地 |
| **系统(System)** | 25% | 信息化覆盖率、系统集成度、数据质量 |
| **组织(Organization)** | 20% | 职责清晰、KPI合理、协同效率 |
| **能力(Capability)** | 25% | 人员技能、管理方法、改善文化 |

## 端到端流程诊断速查

接单→计划→采购→仓储→生产→质检→发货

| 流程节点 | 检查要点 | 常见问题 | 改善方向 |
|---------|---------|---------|---------|
| **接单** | 交期承诺依据、订单录入方式、变更处理 | 销售乱承诺、手工录入出错、变更无流程 | 建立ATP承诺、订单系统化、ECN流程 |
| **计划** | MPS/MRP、排产方式、计划频率 | 无滚动计划、排产靠经验、计划孤岛 | 建立S&OP、导入APS、滚动排产 |
| **采购** | 采购模式、供应商管理、到货准时率 | 多头采购、供应商无评估、到货波动大 | 品类管理、供应商分级、VMI |
| **仓储** | 库位管理、盘点、配送方式 | 物无定位、账实不符、领料制 | WMS、条码化、配送至工位 |
| **生产** | 5S/标准化/OEE/异常管理 | 现场混乱、SOP形同虚设、异常响应慢 | 精益现场导入、TPM、Andon |
| **质检** | 检验流程、追溯、质量数据 | 质量孤岛、追溯困难、不良重复 | SPC、批次追溯、质量数据平台 |
| **发货** | 发货流程、准确率 | 发货出错、物流跟踪缺失 | 扫码出库、物流跟踪 |

## 改善方向速查库（按专业领域）

### 精益生产方向
| 改善方向 | 适用场景 | 潜在价值 |
|---------|---------|:--------:|
| 5S+目视化管理 | 现场混乱、SOP执行差 | 效率提升5-10% |
| VSM价值流优化 | 交付周期长、WIP高 | 交付周期缩短30-50% |
| TPM自主维护 | 设备故障多、OEE低 | OEE提升15-25% |
| SMED快速换模 | 换线时间长、批量大 | 换型时间缩短50-80% |
| 看板拉动系统 | 库存高、计划波动大 | WIP降低30-50% |
| 标准化作业 | 质量波动大、新人上手慢 | 效率稳定+质量提升 |
| 线平衡改善 | 瓶颈工序突出 | 产线效率提升10-20% |
| Poka-Yoke防错 | 人为错误多、不良率波动 | 不良率降低50-90% |

### 数字化方向
| 改善方向 | 适用场景 | 潜在价值 |
|---------|---------|:--------:|
| MES实施 | 生产过程不透明、追溯困难 | 实时透明+可追溯 |
| WMS/条码系统 | 库存不准、找货难、收发错 | 库存准确率>99% |
| APS高级排程 | 排产靠经验、订单延误多 | 排产效率+交期改善 |
| QMS质量系统 | 质量数据孤岛、追溯难 | 质量数据闭环 |
| 数据采集(IoT) | 设备数据不透明、OEE靠估算 | 数据驱动决策 |
| 数字孪生 | 工艺复杂、需模拟仿真 | 优化产线+预防问题 |

### 组织流程方向
| 改善方向 | 适用场景 | 潜在价值 |
|---------|---------|:--------:|
| S&OP产销协同 | 产销脱节、库存波动大 | 供需平衡+库存优化 |
| 计划集控中心 | 计划分散、执行监控缺失 | 计划一盘棋 |
| KPI体系设计 | 考核与战略脱节、数据不透明 | 管理抓手+持续改善 |
| 流程再造(BPR) | 流程冗长、审批多、决策慢 | 效率提升30-50% |
| 组织架构优化 | 职责重叠、部门墙 | 协同效率提升 |
| 供应链协同 | 供应商管理弱、供应链响应慢 | 供应链韧性+成本优化 |

## 改善方向评价矩阵

| 维度 | 评价标准 | 权重 |
|:----:|---------|:----:|
| **紧迫度** | 问题对业务的冲击程度 | 30% |
| **可行性** | 技术/资源/时间可行性 | 25% |
| **价值量** | 预期效益（效率/质量/成本） | 30% |
| **扩展性** | 能否横向复制推广 | 15% |

## 改善项目卡片输出格式

每个改善项目包含：
1. **项目名称** — 简明扼要
2. **问题描述** — 业务痛点+数据支撑
3. **改善方向** — 精益/数字化/组织流程
4. **实施方案** — 关键步骤+里程碑
5. **预期效益** — 量化指标（效率/质量/成本/交期）
6. **投入估算** — 时间/资金/资源
7. **优先级** — P1(高优)/P2(中优)/P3(长期)
8. **关联系统** — ERP/MES/WMS/PLM等
