# Brief 输出规范 (v5.0)

> **本文档定义所有 Type Agent 交付物的统一规范**。各 Agent 通过 `<skill-base>/references/_shared/brief-output-spec.md` 引用。

---

## 🎯 设计原则

1. **用户视角优先**：交付物是给用户看的，不是给系统看的
2. **结构一致性**：所有领域交付物遵循同一框架，降低用户认知成本
3. **可读性 > 完整性**：宁缺勿滥，避免"看起来很全但读不下去"
4. **可执行性**：每个交付物必须包含"用户下一步该做什么"

---

## 📐 统一交付物结构

每个 Type Agent 的交付物遵循 **5 段式结构**：

```
1. 📋 执行摘要     — 1 分钟看懂
2. 🔍 核心分析     — 5 分钟理解
3. 🎯 方案/策略    — 主体内容
4. 📅 执行路线图   — 时间+动作+责任
5. ⚠️ 风险与建议   — 边界与下一步
```

### 1. 执行摘要（≤300字）

回答用户最关心的 3 个问题：
- **结论是什么**（一句话核心建议）
- **为什么**（2-3 个关键论据）
- **下一步**（用户该做什么）

```markdown
## 📋 执行摘要

**核心建议**：[一句话]

**关键依据**：
1. [依据1]
2. [依据2]
3. [依据3]

**建议下一步**：
- [ ] [动作1]
- [ ] [动作2]
```

### 2. 核心分析

**只放用户决策必须知道的信息**，详细数据放附录或单独文档。

```markdown
## 🔍 核心分析

### 现状诊断
[3-5 个关键发现，每个发现含数据/证据]

### 关键洞察
[2-3 个反直觉或有价值的洞察]

### 决策要点
[用户需要做出的关键决策，附选项对比]
```

### 3. 方案/策略（主体）

按各 Type Agent 领域专属模板产出。**统一规则**：
- 方案 ≥2 个候选时，必须附对比表
- 关键决策点用 ✅/⚠️/❌ 标注
- 引用数据/基准时必须标来源（如"基于 benchmark-database.md 美妆行业 Q2 数据"）

### 4. 执行路线图

```markdown
## 📅 执行路线图

| 阶段 | 时间 | 关键动作 | 责任方 | 交付物 | 成功标准 |
|------|------|---------|--------|--------|---------|
| 准备 | Week 1 | [动作] | [角色] | [产出] | [指标] |
| 启动 | Week 2-3 | [动作] | [角色] | [产出] | [指标] |
| 推进 | Week 4-6 | [动作] | [角色] | [产出] | [指标] |
| 评估 | Week 7 | [动作] | [角色] | [产出] | [指标] |
```

### 5. 风险与建议

```markdown
## ⚠️ 风险与建议

### 主要风险
| 风险 | 概率 | 影响 | 应对 |
|------|------|------|------|
| [风险1] | 高/中/低 | [描述] | [应对] |

### AI 能力边界声明
> 本方案由 AI 基于 [方法论/数据源] 产出，建议在以下环节寻求人类专家验证：
> - [环节1]
> - [环节2]

### 下一步建议
1. **立即可做**：[动作]
2. **本周内决策**：[决策点]
3. **本月内推进**：[里程碑]
```

## 🎨 格式规范

### 视觉层次

| 元素 | 用途 | 示例 |
|------|------|------|
| `## H2` | 五段式主标题 | `## 📋 执行摘要` |
| `### H3` | 段内子标题 | `### 现状诊断` |
| **粗体** | 关键结论/数字 | **核心建议**：… |
| `> 引用` | 重要提醒/边界声明 | `> AI 能力边界：…` |
| 表格 | 对比/矩阵/路线图 | 见各模板 |
| `- [ ]` | 用户待办 | `- [ ] 与团队对齐` |

### 数据呈现

- 数字带单位（"提升 23%"而非"提升 23"）
- 区间用范围（"3-5%"而非"约 4%"）
- 来源必标（数据/基准/假设）
- 不确定项标 ⚠️

### 长度控制

| 交付物类型 | 建议长度 | 上限 |
|----------|---------|------|
| 单一交付物（如品牌定位） | 800-1500 字 | 3000 字 |
| 完整全案（如 Campaign） | 3000-5000 字 | 8000 字 |
| 执行摘要 | ≤ 300 字 | 500 字 |

超过上限时**必须拆分**为多个交付物，并附导航。

---

## 🔗 与用户确认节点的关系

每个交付物完成后，触发 ⛔ 用户确认：

```
✅ [交付物名称] 已完成

📋 执行摘要：[一句话]

是否进入下一步？
- ✅ 确认无误，继续
- ✏️ 需要修改：[用户反馈]
- 🔄 整体重来：[原因]
```

---

## 🚫 反模式（禁止）

| 反模式 | 为什么禁止 |
|--------|----------|
| "以下是完整方案：…" 然后罗列所有内容 | 用户读不完，找不到重点 |
| 用大量行业黑话但不解释 | 增加用户认知负担 |
| 只给方案不给执行路线 | 用户不知道下一步 |
| 罗列 10+ 风险但不排优先级 | 用户无法决策 |
| 数据无来源标注 | 用户无法验证可信度 |
| AI 自我吹捧"这是最佳方案" | 失去中立性 |
