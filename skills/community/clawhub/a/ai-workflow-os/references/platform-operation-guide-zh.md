# 平台操作指南（中文）

本 Skill 可用于知识库接入治理，但不会默认替用户上传资料到任何云端平台。

## 飞书 / Feishu

适合保存市场情报、政策监控、食品安全风险监控、资料索引和审核队列。

建议结构：

```text
知识库首页
├── 项目状态
├── 研究资料暂存区
├── 已批准资料库
├── 审计日志
└── 主题索引
```

## Obsidian / 本地 Markdown

适合本地知识库和可审计资料归档。建议保留文件名、来源、时间和标签。

## NotebookLM / 资料阅读

适合对已确认资料进行阅读和问答。不建议把未确认或敏感资料自动上传。

## 通用原则

- 未确认资料先暂存。
- 敏感资料不自动上传。
- 所有入库行为保留审计记录。
