# 供应链需求诊断指南

## 1. 需求层级判断

### 1.1 五级需求体系

| 层级 | 阶段名称 | 阶段特征 | 典型需求 | 优先循环 |
|------|---------|---------|---------|---------|
| **L1** | 概念验证 | 想法验证阶段 | 手板、mockup、3D打印 | 内循环 |
| **L2** | 原型开发 | 原理样机阶段 | 功能样机、首批小样 | 内循环+外循环 |
| **L3** | 工程验证 | 设计定型阶段 | 工程样机、小批试制 | 外循环 |
| **L4** | 生产验证 | 工艺定型阶段 | 中批试产、工艺验证 | 外循环 |
| **L5** | 量产阶段 | 规模生产阶段 | 大批量生产、产能爬坡 | 外循环+战略合作 |

### 1.2 层级判断流程

```
┌─────────────────────────────────────────────────────────────────┐
│                        需求层级判断流程                          │
├─────────────────────────────────────────────────────────────────┤
│  Step 1: 项目处于什么阶段？                                       │
│     ↓                                                           │
│  Step 2: 需要什么类型的能力？                                     │
│     ↓                                                           │
│  Step 3: 需求量级是多少？                                         │
│     ↓                                                           │

│  Step 5: 综合判断 → 确定需求层级 → 选择内/外循环                  │
└─────────────────────────────────────────────────────────────────┘
```

## 2. 内/外循环判断标准

### 2.1 决策矩阵

| 判断维度 | 内循环优先 | 外循环优先 |
|---------|-----------|-----------|
| **需求量级** | <100件 | ≥100件 |
| **技术标准化** | 非标定制 | 标准工艺 |
| **社区能力** | 有现成能力方 | 无或能力不足 |
| **保密要求** | 高（不适合外发） | 一般 |
| **成本考量** | 不追求最低成本 | 成本敏感 |
| **时间要求** | 紧急、快速响应 | 有一定缓冲时间 |

### 2.2 特殊情况处理

**可混合使用**：
- 同一项目不同阶段可能需要内+外循环
- 例：L2阶段，内循环做结构设计，外循环做手板制作

**内循环优先但外循环补充**：
- 紧急小批量需求：内循环优先，外循环兜底
- 技术难点突破：内循环高手+外循环专业设备

**外循环优先但需社区背书**：
- 首次对接某类供应商，平台协助谈判
- 大额订单需要集群议价支持

## 3. 需求画像要素

### 3.1 必填要素

| 要素 | 说明 | 示例 |
|------|------|------|
| 产品名称 | 项目/产品名称 | 智能水质监测仪 |
| 需求类型 | 需要的制造环节 | PCB制板+整机组装 |
| 技术参数 | 关键技术指标 | 4层板、10cm×10cm |
| 需求量级 | 单次订单数量 | 首批50件 |
| 预算范围 | 可接受价格区间 | 5000-8000元 |
| 时间要求 | 交付时间要求 | 15个工作日内 |
| 质量要求 | 验收标准 | 符合IPC二级标准 |

### 3.2 选填要素

| 要素 | 说明 | 何时填写 |
|------|------|---------|
| 设计文件 | CAD图纸、BOM清单等 | 有现成设计时 |
| 样品参考 | 参照样品或图片 | 需要供应商参考时 |
| 认证要求 | CE/FCC/ROHS等 | 涉及出口或特殊市场时 |
| 特殊工艺 | 电镀、喷涂、特殊材料等 | 有特殊要求时 |
| 供应商偏好 | 指定或排除的供应商 | 有历史合作或顾虑时 |

## 4. 需求描述规范

### 4.1 清晰度要求

好的需求描述：
- ✅ 技术参数具体、可测量
- ✅ 数量明确，无歧义
- ✅ 交期清晰，有缓冲
- ✅ 验收标准可执行

不好的需求描述：
- ❌ "差不多就行"
- ❌ "看着办"
- ❌ "做好一点"
- ❌ "尽快"

### 4.2 常见问题清单

填写需求诊断单时，检查以下问题：

| 问题类型 | 具体表现 | 正确做法 |
|---------|---------|---------|
| 参数不明确 | 只说"好质量" | 提供具体参数或参照样品 |
| 数量模糊 | "几百件左右" | 给出明确数量或范围 |
| 交期太紧 | "明天就要" | 合理评估时间，必要时分段 |
| 预算不符 | 预算远低于市场价 | 重新评估预算或调整需求 |
| 标准缺失 | 无验收标准 | 明确执行标准或验收方法 |
