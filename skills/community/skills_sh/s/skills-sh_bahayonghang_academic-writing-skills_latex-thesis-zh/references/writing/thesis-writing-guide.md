# 中文学位论文章节写作主线指南

本指南用于在已有中文 LaTeX 学位论文中做章节级写作优化。它不是英文会议论文 section-writing 的直接翻译；优先服从学位论文的章节结构、模板规范、标题后导语、GB/T 7714 引用和答辩可解释性要求。

## 使用边界

- 用于：绪论、文献综述、方法章节、实验章节、总结与展望、摘要/创新点/结论对齐。
- 不用于：从零代写学位论文、编造文献、编造实验、绕过学校模板要求。
- 默认保留 `\cite{}`、`\ref{}`、`\label{}`、数学环境、参考文献键和模板宏命令。

## 总体主线

一篇学位论文应形成一条可答辩的主线：

```text
研究背景 -> 技术瓶颈/研究空白 -> 科学问题 -> 本文方法/章节工作 -> 实验证据 -> 贡献闭合 -> 局限与展望
```

修改章节时先写出这条主线，再判断每一章是否在推进它，而不是并列堆放多个工作。

## 绪论

推荐结构：

1. 研究背景：说明问题出现的领域语境和应用价值。
2. 技术瓶颈/研究空白：从已有方法或实践限制中推出未解决问题。
3. 科学问题：把空白收束为本文要回答的核心问题。
4. 本文工作：说明方法或系统如何回应该问题。
5. 主要贡献：每条贡献都应能映射到后文方法或实验。
6. 章节安排：说明各章如何递进，而不是机械罗列标题。

检查点：

- 是否从背景直接跳到本文工作，缺少前人不足或技术原因。
- 是否把“研究意义”写成口号，而没有对应的技术瓶颈。
- 是否每条贡献都有实验、分析或章节证据支撑。

## 正文章引言（承上启下两段式）

第 2 章到结论前的每一章，章标题之后应先写一段**章引言**再进入小节，与章末"本章小结"首尾呼应。它不同于第 1 章绪论的三层漏斗式引言，更短、更聚焦于"承上启下"。

**篇幅**：一般 1~2 个自然段、约 300~500 字。它不是摘要，也不要重复绪论文献综述里已讲过的内容。

**两段角色**：

1. **承上（第①段）**：点明前一章对本章的价值——前一章解决了什么、得出什么结论，由此引出本章为何要继续。用**章节号**（"第 2 章"）指代，避免"上一章 / 上文 / 前一章"这类相对表述（便于跳读定位）。
2. **启下（第②段）**：交代本章针对什么问题、核心思想是什么、相对已有方法的优势在哪；段末视学科需要，用一两句预告本章各节安排（"本章组织如下：…"，工科常配一张本章框架图）。路标段非硬性要求，可结合学科与导师习惯取舍。

**可套用模板**：

```text
第 X 章……解决了……，并得出……，但在……方面仍存在……。
针对这一问题，本章提出……，其核心思想是……，相比……的优势在于……。
本章组织如下：N.1 节……，N.2 节……，N.3 节……。
```

**正反例**：

| 写法 | 判定 |
|------|------|
| "第 2 章建立了基线模型，但其在长序列上开销过大。针对这一瓶颈，本章提出稀疏注意力机制……本章组织如下：3.1 节……" | 通过（承上有章节号、启下有问题+思路+路标） |
| 章标题后直接 `\section` 或公式 | 缺章引言（既缺承上也缺启下） |
| "上一章讨论了基线方法。" 单句即转入小节 | 相对指代 + 过简（应扩成两段、改用章节号） |
| 把整章方法细节写进章引言 | 过长（细节应下沉到对应小节） |

> 边界：绪论（第 1 章）按"绪论"一节的漏斗式写法处理，不套用本节两段式；本节只面向正文各章。

## 正文章末小结（单段收束式）

第 2 章到结论前的每一章，章末"本章小结"应与章首引言形成首尾呼应：章引言负责说明"为什么写本章"，本章小结负责说明"本章解决了什么、如何证明、对全篇有什么支撑"。它不是最终"总结与展望"的缩写，也不是把各节标题重新罗列一遍。

**篇幅**：默认写成 **1 个自然段**。除非学校模板、导师要求或用户明确要求列点，不拆成多个短段或列表。工科学位论文中可使用"首先、其次、然后、最后"串联阶段性工作，但这些连接词应服务于同一个连续段落。

**单段角色顺序**：

1. **问题/目标**：开头点明本章围绕的核心问题、评价对象或技术瓶颈。
2. **工作/方法**：概括本章提出、构建、设计或分析了什么，不展开公式推导细节。
3. **过程/证据**：压缩说明关键步骤、实验对象、案例、指标或验证路径。
4. **结果/价值**：只写原文已有证据能够支撑的结论；缺少数据时标为证据不足，不补造结果。
5. **主线支撑**：收束到本章对后续章节、总体贡献或全文科学问题的支撑作用。

**可套用模板**：

```text
本章围绕……问题，针对……不足，提出/构建了……。首先，……；其次，……；然后，……；最后，……。以……为实验/案例对象的验证表明，……，从而为后续……/全文……提供了支撑。
```

**正反例**：

| 写法 | 判定 |
|------|------|
| "本章围绕非平稳工业过程运行工况评价中的异常刻画与动态响应不足问题，提出……首先……其次……然后……最后……验证表明……为后续优化调控提供支撑。" | 通过（单段、问题-方法-证据-价值闭合） |
| 连续写三段："本章首先介绍……。本章其次介绍……。本章最后介绍……。" | 过散（应压成一个自然段，避免目录式复述） |
| 用条目列出"1. 完成了…… 2. 完成了…… 3. 完成了……" | 非默认形态（除非模板/导师要求，否则改为连续段落） |
| "本章提出的方法达到国际领先水平，填补国内空白。" 但正文没有对应证据 | 证据越界（应删除或改为可验证表述） |

> 边界：最终"结论/总结与展望"应综合全文贡献、局限与未来方向；本章小结只做章内证据收束，不机械重复每章"第 X 章……"句式。

## 文献综述

推荐链条：

```text
共识 -> 分歧 -> 局限 -> 空白 -> 本文切入点
```

每个主题组应先说明该组文献解决了什么，再比较方法假设、机制、适用范围或失败模式，最后指出与本文问题相关的局限。不要按作者年份逐条点名。

边界：

- 不新增未在原文出现的引用。
- 不把“相关研究较少”写成“尚无人研究”，除非已有证据。
- 如果研究空白缺少引用支撑，标为“证据不足”，不要强行写成确定结论。

## 方法章节

每个核心章节或模块应回答三件事：

1. **动机**：为什么这一章/模块必须存在，它回应了哪个瓶颈。
2. **设计**：输入、处理流程、关键结构、输出是什么。
3. **优势**：相对可比方案有什么技术收益，后文如何验证。

章节之间需要显式递进：

```text
基于上一章提出的 ...，本章进一步解决 ...
```

若多项工作只是并列放置，需要补“共同科学问题”和“递进关系”。

## 实验与讨论

实验章节不应像项目汇报。推荐层次：

1. 实验设置：数据、指标、基线、协议公平性。
2. 有效性：主结果与强基线对比。
3. 消融/敏感性：验证关键模块、参数或设计选择。
4. 机理解释：解释结果原因，而非复述表格。
5. 文献回溯：与文献综述中的代表性工作比较。
6. 局限与启示：说明适用范围和后续改进。

禁止把未报告的数据写成结论。若缺少基线、消融或显著性信息，应明确标记缺口。

## 摘要、创新点与总结

摘要、创新点和结论应像“三方闭合”：

| 位置 | 应回答 |
| --- | --- |
| 摘要 | 研究问题、方法、结果、意义 |
| 创新点/主要贡献 | 本文相对已有工作的增量 |
| 总结与展望 | 已验证的贡献、局限、未来方向 |

检查每条创新点是否在结论中被回应，且是否有实验或章节证据支撑。

## 输出建议格式

```latex
% THESIS-WRITING（第 N 行）[Severity: Major] [Priority: P1]: 章节主线断裂
% 问题：绪论提出了“提高鲁棒性”，但实验章节没有对应验证。
% 建议：补充鲁棒性实验，或将绪论贡献改为当前结果能支持的范围。
% 证据状态：needs evidence
```
