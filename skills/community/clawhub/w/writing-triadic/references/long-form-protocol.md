# 📖 长文档分章协同 (Long-Form Chapter Manager) 完整协议

> 从 SKILL.md 提取。适用场景: 毕业论文(1-5万字)、书籍、长篇报告、系列文章。将大文档拆分为独立章节,分章写作,全局一致。

### 核心设计

```
长文档 = 全局大纲 + N 个独立章节
每个章节 = 一次独立的 Phase 1→5 写作会话
章节间 = Creator 维护一致性(术语/语调/引用/交叉引用)
```

### Chapter Manifest(章节清单)

每次长文档写作任务开始时,Creator 创建 `写作计划.md` 增强版:

```markdown
## 📖 Chapter Manifest

| 章 | 标题 | 目标字数 | 状态 | 依赖 | 完成日期 |
|----|------|---------|------|------|---------|
| 1 | 绪论 | 3000 | ✅ 完成 | - | 2026-05-18 |
| 2 | 文献综述 | 5000 | 🔄 进行中 | 章1 | - |
| 3 | 研究方法 | 4000 | ⏳ 待写 | 章2 | - |
| 4 | 结果与分析 | 6000 | ⏳ 待写 | 章3 | - |
| 5 | 讨论 | 4000 | ⏳ 待写 | 章4 | - |
| 6 | 结论 | 2000 | ⏳ 待写 | 章5 | - |

### 全局约束 (跨章节一致性)
- 术语统一: [术语A] / [术语B] 全文使用中文术语
- 人称: 全文使用第一人称「本研究」/「我们」
- 引用格式: GB/T 7714-2015
- 图表编号: 按章编号(图1-1, 表2-3)
```

### 一致性看门狗

Creator 在每章写作前后执行:

| 检查项 | 频率 | 说明 |
|--------|------|------|
| 术语一致性 | 每章完成后 | 扫描本章,对照全局术语表,发现不一致→标注 |
| 引用编号 | 每章完成后 | 检查交叉引用是否跳号/重复 |
| 语调漂移 | 每 3 章 | 对比各章正式度/句长,漂移 >20% → 提醒用户 |
| 字数累计 | 每章完成后 | 更新累计字数 vs 总目标 |
| 依赖检查 | 新章开始前 | 确保依赖章节已完成 |

### 长文档模式 vs 普通模式

| 维度 | 普通模式 | 长文档模式 |
|------|---------|-----------|
| 写作单元 | 一次一篇 | 一次一章 |
| 大纲 | 写作计划.md | Chapter Manifest(全局大纲+跨章约束) |
| 一致性 | 无 | 术语/引用/语调看门狗 |
| 进化 | 单篇进化 | 分章进化+跨章同步 |
| 会话 | 一次会话完成 | 跨多次会话(可中断续写) |
| 适用 | 500-5000 字 | 1万-10万+ 字 |

### 触发条件

- 用户明确说「写论文」「写书」「长篇」「大稿子」
- 预估总字数 ≥8000 字
- 用户要求分章节/分部分写作

### 跨会话续写

长文档跨多次会话时:
1. 新会话开始 → Creator 读取 MEMORY.md + Chapter Manifest
2. 确认当前进度:「上次写到第3章,今天继续第4章?」
3. 继续前一章的上下文自动注入 Phase 2 写作规则
