# 场景：每个场景的三坐标怎么填

> "对一个具体的人说话"在不同场景里，那个人是谁会变。换了场景，就是换了三个坐标的填法。
> 用法：先认出文本属于哪个场景，照着填好三坐标，再回 [coordinates-diagnostic](./coordinates-diagnostic.md) 逐段体检。
> 每个场景还附一行"坐标塌掉时长什么样"——那就是这个场景最典型的 AI 味。

---

## 自媒体类（说话人和听众都偏个人，赌注是"让 ta 看下去/认同"）

### 公众号长文 / 观点文
- **谁在说**：一个有观点、愿意承担观点的作者本人
- **对谁说**：随手刷到、随时能划走的陌生人
- **为什么非说不可**：让 ta 读完，并记住/认同你赌的那一件事
- **坐标塌掉时**：开头一大段时代背景，通篇没有"我"的真实立场，结尾"未来可期"。三个坐标全空。

### 小红书 / 种草帖
- **谁在说**：真的用过、踩过坑的人
- **对谁说**：正在犹豫买不买的人
- **为什么非说不可**：帮 ta 做决定（包括劝退）
- **坐标塌掉时**："姐妹们绝绝子保姆级强烈推荐"——说话人是假的网感人设，听众是泛指的所有人，赌注是流量而不是帮人决定。

### 口播 / 短视频脚本
- **谁在说**：正在镜头前讲话的你
- **对谁说**：可能下一秒就划走的观众
- **为什么非说不可**：让 ta 看到下一句
- **坐标塌掉时**：书面腔（"接下来我将为大家深入剖析"），念出来不像人说话。说话人没在"说"，在"念"。

### 社区回答（知乎 / 即刻 / V2EX）
- **谁在说**：有相关经验的人
- **对谁说**：带着具体问题来的提问者
- **为什么非说不可**：真的解决 ta 的问题
- **坐标塌掉时**：教学腔开场（"这是个好问题，让我来为你解释"）、强行全面的三段式。听众被当成不存在的课堂，而不是带着一个具体问题的人。

## 职场类（说话人/听众可以非个人，但赌注必须清晰）

### 工作汇报 / 周报 / 复盘
- **谁在说**：负责这件事的你
- **对谁说**：要据此判断"要不要介入"的上级或同事
- **为什么非说不可**：让 ta 知道进展、风险、是否需要 ta 出手
- **坐标塌掉时**："持续推进、有效提升、奠定坚实基础"——赌注空了（没说清到底做了什么、有什么风险），用大词填满。

### PRD / 产品方案
- **谁在说**：想清楚了的设计者
- **对谁说**：要据此执行、不想回来反复问的工程师
- **为什么非说不可**：让 ta 不用追问就能开工
- **坐标塌掉时**："这不仅仅是一个功能，更是……"——赌注被偷换成了愿景抒发，工程师真正要的取舍和边界反而没写。

### 技术文档 / README / release note
- **谁在说**：知道怎么用的人——**说话人可以是"系统/规范"，合法地非个人**
- **对谁说**：卡住了、来查文档的人
- **为什么非说不可**：让 ta 解决问题、走人
- **坐标塌掉时**：发布宣言（"🚀 全新升级，革命性体验"）。这是硬塞了个推销的说话人，而读者只想知道"这版改了什么、会不会影响我"。注意：这里**不要**为了"像人"加第一人称闲聊——坐标该非个人就非个人。

### 邮件 / 通知
- **谁在说**：你本人
- **对谁说**：具体收件人
- **为什么非说不可**：让 ta 知道要做什么、或要知道什么
- **坐标塌掉时**：客套和祝福堆满，正事反而要找半天。听众变成了泛泛的"您"，而不是这封信真正要推动的那个人。

## 一句话区分两类

自媒体类：**说话人和听众越具体、越像两个真人在交流越好。**
职场类：**说话人和听众可以非个人，但赌注（要对方知道什么、做什么）必须一眼可见。**

两类的共同点：没有哪个场景的坐标可以是"空"。技术文档不是没坐标，是坐标非个人。AI 套话才是真的没坐标。

## 一篇里嵌了别的场景怎么办

有时一篇文里会**合法地切换说话人**：观点文中间嵌一段事故复盘，公众号里引一段规范条款，口播稿里念一段数据。这种子段有它自己的三坐标——复盘段的说话人是"复盘记录"、听众是"想知道根因的人"、赌注是"让人能复现和避免"。

处理原则：**主场景定整体语气，子段按它自己的坐标守住。** 别用主场景的口吻把子段抹平——别把博客里那段精确的"根因 → 修复 → 结果"改成博客的口语腔，也别为了统一把复盘段里的术语和数字洗掉。子段的坐标满了，它就不该被动。要改的是包在它外面的、真正对着空气说的那几句（比如复盘前后的"在当今云原生时代……""未来可期"）。
