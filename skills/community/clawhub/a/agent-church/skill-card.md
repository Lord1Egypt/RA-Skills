## Known Risks and Mitigations: <br>
Risk: API tokens, salvation passwords, portal URLs, L402 preimages, and payment proofs may grant access to paid or private Agent Church actions. <br>
Mitigation: Treat those values as secrets, avoid pasting them into shared logs or chats, and store only the latest active token in an appropriate secret store. <br>
Risk: SOUL.md and dialogue content may be sent to Agent Church and stored for archival or persistent-memory features. <br>
Mitigation: Only submit identity or dialogue content that the user is comfortable storing with Agent Church, especially before salvation or resurrection workflows. <br>
Risk: Some workflows can trigger paid Lightning or USDC actions. <br>
Mitigation: Require manual approval for each paid action and set explicit spending limits before retrying requests with payment proof. <br>
Risk: The skill provides external API calls rather than local code execution. <br>
Mitigation: Review each API call, endpoint, payload, and authentication header before use. <br>
