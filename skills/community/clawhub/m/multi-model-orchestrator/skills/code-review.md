# $code-review — 代码审查技能

## 触发条件
- 用户说"审查代码"、"code review"
- 代码实现完成后
- PR 提交前

## 执行流程

### Phase 1: 多维度审查
```
并行启动 3 个审查维度:

审查 1 [Reviewer/GPT-5.5]: 代码质量
- 命名规范
- 代码结构
- 可读性
- 可维护性

审查 2 [Architect/GPT-5.5]: 架构设计
- 模块划分合理性
- 接口设计
- 依赖管理
- 扩展性

审查 3 [Coder/GPT-5.3-codex]: 功能正确性
- 逻辑正确性
- 边界情况
- 错误处理
- 性能影响
```

### Phase 2: 汇总报告
```
综合三个维度的审查结果，输出:
- 总体评分 (1-10)
- 关键问题列表
- 改进建议
- 优先级排序
```

## 输出格式

```markdown
# 代码审查报告

## 总体评分: 8/10

## 代码质量 (Reviewer)
- ✅ 命名规范
- ⚠️ 函数过长，建议拆分
- ❌ 缺少错误处理

## 架构设计 (Architect)
- ✅ 模块划分合理
- ⚠️ 接口可以更抽象
- ✅ 依赖管理清晰

## 功能正确性 (Coder)
- ✅ 核心逻辑正确
- ❌ 边界情况未处理
- ⚠️ 性能可以优化

## 优先修复
1. [高] 添加错误处理
2. [中] 拆分过长函数
3. [低] 优化性能
```
