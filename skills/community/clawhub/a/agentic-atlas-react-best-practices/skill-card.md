## Description: <br>
Audits React code for performance, bundle size, and best practices when reviewing React code, auditing bundle size, finding performance issues, checking React 18+ patterns, or evaluating a React codebase. <br>

This skill is ready for commercial/non-commercial use. <br>

## Publisher: <br>
[asimons81](https://clawhub.ai/user/asimons81) <br>

### License/Terms of Use: <br>
MIT-0 <br>


## Use Case: <br>
Developers and engineers use this skill to review React applications for performance, bundle size, accessibility, and React 18+ best-practice issues. It is intended for React codebases, not backend Node.js, non-React, or CSS-only reviews. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: The agent may propose commands, package installs, or repository-wide scans while auditing a React project. <br>
Mitigation: Review and approve commands, installs, and broad scans before execution, especially in private repositories. <br>
Risk: Audit findings and code fixes may be incomplete or incorrect for a specific application. <br>
Mitigation: Treat the output as review guidance and validate suggested changes with tests, linting, and human code review. <br>
