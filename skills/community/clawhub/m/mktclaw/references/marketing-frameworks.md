## 一、战略与定位框架

### 1. STP 模型 (Segmentation-Targeting-Positioning)

**适用 Agent**: strategy / brand / creative

```
S (Segmentation) 市场细分
    ↓ 按 [地理/人口/心理/行为] 维度细分市场
T (Targeting) 目标选择
    ↓ 评估各细分市场的: 规模/增长/竞争/匹配度
P (Positioning) 定位确立
    → 在目标市场的心智中确立独特位置
```

**输出格式**:
```markdown
## STP 分析

### S - 市场细分
| 细分维度 | 子市场 | 规模 | 增速 | 竞争强度 |
|---------|--------|------|------|---------|
| [维度] | [子市场1] | [大/中/小] | [%] | [高/中/低] |

### T - 目标选择
| 优先级 | 目标市场 | 选择理由 | 风险 |
|:------:|---------|---------|------|
| P0 | [市场] | [理由] | [风险] |

### P - 定位
- 定位陈述: "For [目标用户], [品牌] is the [品类] that [差异化价值]"
- 定位支撑: [3 个 RTB]
```

### 2. 特劳特定位理论 (Trout Positioning)

**核心思想**: 定位不是你对产品做什么，而是你对用户心智做什么。

**4 种定位方法**:
| 方法 | 适用场景 | 示例 |
|------|---------|------|
| **领导者定位** | 品类第一 | "今年过节不收礼，收礼只收脑白金" |
| **对立定位** | 老二挑战老大 | "XXX，不是 XX 的 XX" |
| **重新定位** | 颠覆既有认知 | "XX 不止是 XX" |
| **品类创造** | 创造新品类 | "XX，开创 XX 新品类" |

**定位有效性的 4 个测试**:
1. 是否在用户心智中找到了一个空位？
2. 是否足够简单(一句话能说清)？
3. 是否与竞品有明显区隔？
4. 是否能长期持有？

## 三、消费者旅程框架

### 1. AIDA 模型

**适用 Agent**: media / creative / content

```
Attention(注意) → Interest(兴趣) → Desire(欲望) → Action(行动)
```

**应用场景**: 传统广告/销售漏斗/简单决策品类

| 阶段 | 关键问题 | 适合渠道 | KPI |
|------|---------|---------|-----|
| Attention | 如何让用户看到？ | OOH/开屏/信息流 | 曝光/Reach |
| Interest | 如何让用户感兴趣？ | 短视频/KOL/内容 | 点击/停留 |
| Desire | 如何激发购买欲？ | 测评/直播/详情页 | 收藏/加购 |
| Action | 如何促成转化？ | 搜索/促销/CTA | 下单/GMV |

---

### 2. AISAS 模型(电通)

**适用 Agent**: media / creative / kol

```
Attention(注意) → Interest(兴趣) → Search(搜索) → Action(行动) → Share(分享)
                                                          ↑              │
                                                          └──────────────┘
                                                          (社交化延伸)
```

**应用场景**: 数字时代消费者决策/社交媒体营销

**关键差异**: 相比 AIDA，增加了 **Search(搜索)** 和 **Share(分享)** 环节，反映消费者主动获取信息和口碑传播的行为。

| 阶段 | 平台适配 | 关键内容形式 |
|------|---------|------------|
| Attention | 抖音开屏/小红书信息流 | 短视频/海报 |
| Interest | 小红书种草/B站深度 | 图文笔记/中长视频 |
| Search | 小红书搜索/百度/微信 | SEO 内容/品牌专区 |
| Action | 天猫/抖音电商/小程序 | 详情页/直播 |
| Share | 微信朋友圈/小红书 | UGC/转发有礼 |

---

### 3. AIPL 模型(阿里系)

**适用 Agent**: media / data / kol(电商导向)

```
A(Awareness 认知) → I(Interest 兴趣) → P(Purchase 购买) → L(Loyalty 忠诚)
```

**应用场景**: 阿里系电商(天猫/淘宝)运营/数据银行(品牌数据银行)

| 阶段 | 数据银行定义 | 转化路径关键动作 |
|------|------------|---------------|
| A | 被品牌触达过(15天内) | 触达曝光/进店 |
| I | 与品牌有过互动 | 点击/收藏/加购 |
| P | 购买过品牌商品 | 下单成交 |
| L | 复购/忠诚用户 | 二次购买/AIPL流转 |

**FAST 方法论**(阿里人群资产):
- **F**(Fertility): AIPL 人群总量
- **A**(Advancing): AIPL 转化效率
- **S**(Superiority): 高价值人群占比
- **T**(Thriving): 人群活跃度

---

### 4. 5A 模型(科特勒)

**适用 Agent**: strategy / brand

```
Aware(知晓) → Appeal(诉求) → Ask(询问) → Act(行动) → Advocate(倡导)
```

**应用场景**: 数字时代的品牌资产衡量

| 阶段 | 衡量指标 |
|------|---------|
| Aware | 知名度/品牌搜索量 |
| Appeal | 偏好度/考虑购买率 |
| Ask | 主动搜索/咨询/进店 |
| Act | 购买/复购 |
| Advocate | NPS/UGC/推荐率 |

## 四、媒介规划框架

### 1. Reach & Frequency(到达率与频次)

**适用 Agent**: media

**核心公式**:
```
有效到达率 = 触达目标受众的比例 × 暴露频次

行业基准:
- 品牌认知建立: Reach 70%+ × Frequency 3+
- 产品上市引爆: Reach 50%+ × Frequency 5+
- 促销活动告知: Reach 60%+ × Frequency 2+
- 竞品防御: Reach 40%+ × Frequency 4+
```

**Reach 三种计算**:
| 类型 | 定义 | 用途 |
|------|------|------|
| Reach 1+ | 至少看到 1 次 | 知名度建设 |
| Reach 3+ | 至少看到 3 次 | 信息有效传递 |
| Reach 7+ | 至少看到 7 次 | 行动转化 |

---

### 2. GRP / TRP(总评分点)

**适用 Agent**: media

```
GRP (Gross Rating Points) = Reach(%) × Frequency
TRP (Target Rating Points) = 目标受众内的 GRP

例: 触达 60% 目标人群，平均看到 4 次
   GRP = 60 × 4 = 240

行业基准:
- OOH 月度 campaign: 100-300 GRP
- 电视黄金时段: 200-500 GRP/周
- 信息流广告: 难以直接计算 GRP，用 OTS 替代
```

---

### 3. OTS (Opportunity To See)

**适用 Agent**: media

```
OTS = 总曝光次数 / Reach 人数
    = 平均每个用户看到广告的次数

OTS 与有效频次关系:
- OTS 1-2: 知名度建立
- OTS 3-5: 信息理解
- OTS 6+: 转化触发
```

---

### 4. 媒介渠道选择决策树

**适用 Agent**: media

```
营销目标是什么？
│
├─ 品牌认知 (Awareness)
│   ├─ 预算充足 → 抖音开屏 + 小红书信息流 + OOH
│   ├─ 中等预算 → 小红书 KOL 种草 + 抖音信息流
│   └─ 低预算 → 小红书 KOC + 抖音自然流量
│
├─ 品牌考虑 (Consideration)
│   ├─ 高客单价 → 小红书深度种草 + B站测评 + 微信深度
│   └─ 低客单价 → 抖音短视频 + 直播
│
├─ 销售转化 (Conversion)
│   ├─ 电商导向 → 抖音/小红书信息流 + 电商内投 + KOL 直播
│   ├─ 线下导向 → LBS 定向 + OOH + 抖音同城
│   └─ 私域导向 → 微信朋友圈 + 视频号 + 公众号
│
└─ 用户留存 (Retention)
    ├─ 高频品类 → 微信社群 + 短信 + 小程序
    └─ 低频品类 → 微信公众号内容 + 会员体系
```

## 六、长期/短期投资框架

### Binet & Field 60:40 法则

**适用 Agent**: strategy / media / data

**核心结论**:
> 长期品牌建设(Brand)与短期销售激活(Sales)的理想预算比例是 **60:40**。

| 维度 | 品牌建设(60%) | 销售激活(40%) |
|------|--------------|--------------|
| 目标 | 心智占领/溢价能力 | 即时转化/促销 |
| 时间 | 长期(1年+) | 短期(1周-3月) |
| KPI | 知名度/考虑度/搜索指数 | GMV/ROAS/CPA |
| 内容 | 情感向/故事化/大制作 | 功能向/促销/简单直接 |
| 渠道 | TV/OTV/OOH/视频号 | 搜索/信息流/直播/电商 |

**按品类调整**:
| 品类 | 推荐比例(Brand:Sales) | 原因 |
|------|:--------------------:|------|
| 享乐品类(奢侈品/酒) | 70:30 - 80:20 | 情感驱动 |
| 高频快消(食品/日化) | 60:40 | 平衡 |
| 耐用消费品(家电/汽车) | 50:50 | 决策周期长 |
| B2B/SaaS | 40:60 - 30:70 | 理性决策 |
| 新品牌(0-1阶段) | 50:50 → 60:40 | 平衡投入 |

### 3. ROI / ROAS / CAC / LTV 计算框架

**适用 Agent**: data / strategy / media

```markdown
## 核心公式

### ROAS (Return on Ad Spend)
ROAS = 广告带来的收入 / 广告花费
健康值: 因品类而异(美妆 3-5x, 家电 2-3x, 服饰 4-8x)

### CAC (Customer Acquisition Cost)
CAC = 总营销+销售费用 / 新客数量
健康值: CAC < LTV/3

### LTV (Lifetime Value)
LTV = 平均客单价 × 年购买频次 × 客户生命周期(年) × 毛利率
健康值: LTV/CAC ≥ 3:1

### ROI (Return on Investment)
ROI = (收益 - 投入) / 投入 × 100%
健康值: > 0%
```

## 十、Byron Sharp — 品牌增长理论 (How Brands Grow)

**适用 Agent**: strategy / brand / media

> **v5.2.1 新增**：补齐 Agency CEO 评估中指出的"缺失 Byon Sharp / IPA Databank 等 Agency 日常使用的基础工具"问题。

### 10.1 核心贡献

Byron Sharp 教授（Ehrenberg-Bass Institute）的研究颠覆了传统品牌营销认知：

| 传统认知 | Sharp 实证 | 对策略的影响 |
|---------|-----------|-------------|
| 品牌忠诚是核心资产 | **轻度买家是增长核心** | 广告应追求"轻度买家触达"而非"深度种草" |
| 细分市场是增长路径 | **大规模渗透是增长第一定律** | 品类渗透率 > 细分深耕 |
| 差异化是核心策略 | **差异化不重要，关键是"可记忆的独特性"** | 建立"独特品牌资产"（Distinctive Assets） |
| 品牌建设 vs 效果 | **两者互补而非冲突** | Binet & Field 60:40 实证支撑 |

### 10.2 品牌增长的 7 条法则

| # | 法则 | 核心含义 | 实践应用 |
|:-:|------|---------|---------|
| 1 | **渗透法则** (Law of Doubling) | 增长来自**扩大用户基数**，而非提高忠诚度 | KPI 应以 Reach / 渗透率为核心 |
| 2 | **忠实用户法则** (Retention Double Jeopardy) | 大品牌的忠实用户更多，但"非常忠实"的比例与品牌大小成正比 | 不要只 Focus 忠实用户 |
| 3 | **市场份额法则** (Share of Market) | 市场份额越大，用户在各类购买行为中的占比越接近市场比例 | 品牌的关键对手是"不买我的人" |
| 4 | **可记忆性法则** (Memory) | 购买决策依赖**品牌记忆结构** (Mental Structures) | 广告目标应是有效刷新品牌记忆 |
| 5 | **注意力法则** (Salience) | 品牌被想到的"易得性" (Mental Availability) 是关键 | 在目标受众的"触发场景"中建立品牌连接 |
| 6 | **差异化法则** (Differentiation) | 差异化重要性被高估；**独特性** (Distinctiveness) 更重要 | 建立独特的颜色/音乐/视觉符号 |
| 7 | **创意法则** (Creativity) | 创意表达影响品牌记忆的强度 | 广告要有"品牌编码"（Branding） |

### 10.3 Mental Availability vs Physical Availability

```markdown
## Mental Availability（心智可得性）
品牌在消费者"购买时刻"被想起的概率和容易程度。

衡量指标：
1. **品牌知名度** (Brand Awareness) — 自发提及 / 辅助提及
2. **品类关联** (Category Entry Points, CEPs) — 
   用户在不同消费场景下会想起哪些品牌
3. **独特性资产** (Distinctive Assets) — 
   看到颜色/音乐/字体就能联想到品牌的能力

优化策略：
- 广告内容应以"品类触发场景"为核心
- 建立多 CEPs 覆盖（不同消费场景都能想起你的品牌）
- 持续曝光，而非短时间爆发（Mental Availability 需要持续投入）

## Physical Availability（物理可得性）
消费者在购买渠道中能够实际找到和购买到品牌的容易程度。

衡量指标：
1. **分销密度** — 线上线下铺货率
2. **购买便利性** — 下单/配送/支付的流畅度
3. **货架/页面占比** — 在终端口的"物理曝光"

优化策略：
- 全渠道覆盖（线上+线下+社交电商）
- 降低购买门槛（首单优惠/试用装）
- 确保终端(货架/搜索结果)的"产品可见度"
```

### 10.4 应用建议

```yaml
brand_growth_strategy:
  primary_goal: "渗透率提升"
  kpi_focus:
    - "category_penetration"    # 品类渗透率
    - "mental_availability"     # 心智可得性分数
    - "distinctive_asset_recognition"  # 独特性资产识别率
    - "share_of_voice"          # 声量占比（与市场份额的关系）
  
  media_strategy:
    - "Reach > Frequency"       # 广度优先
    - "持续投放 > 脉冲投放"      # 避免大面积断档
    - "Multi-channel > 单渠道"   # 多渠道触达
    - "轻度买家 > 重度买家"      # 广告是给"记得不太清楚的人"看的
```

## 十一、IPA Databank（英国广告从业者协会数据库）

**适用 Agent**: strategy / media / data

### 11.1 背景

IPA Databank 收录了 1400+ 个真实营销 Campaign 的效果数据，是**全球最大规模的营销效果实证研究数据库**。

### 11.2 关键发现

| 发现 | 数据支撑 | 策略含义 |
|------|---------|---------|
| 品牌建设预算 ROI 更高 | 品牌建设 Campaign 的利润回报是效果 Campaign 的 **1.8x** | Binet & Field 60:40 的实证基础 |
| 电视+数字组合最优 | TV + Online Video 组合的 ROI 比纯数字高 **38%** | 不要盲目砍掉传统媒介 |
| 创意质量驱动效果 | 高创意质量 Campaign 的 ROI 是低质量的 **11x** | 不要在创意上省钱 |
| 效果广告有天花板 | 效果广告的 ROI 在 6 个月后显著递减 | 品牌建设是持续增长的前提 |
| 长期 vs 短期 ROI 曲线 | 品牌建设 ROI 在 Year 3 超越效果广告 | CMO 需平衡长期与短期 |

### 11.3 IPA 效果评估框架

```markdown
## IPA 效果金字塔

    → 利润 / 市场份额 / 定价权
品牌效果 (Brand Effects)
    → 知名度 / 品牌偏好 / 品牌联想
传播效果 (Communications Effects)
    → 广告认知 / 信息理解 / 品牌记忆
媒介效果 (Media Effects)
    → Reach / GRP / 点击 / 互动
```

## 十二、VBFS — 基于价值的收费框架 (Value-Based Fee Structure)

**适用 Agent**: strategy / media

### 12.1 什么是 VBFS

Value-Based Fee Structure（基于价值的收费结构）是 Agency 对客户进行项目定价时，**将费用与服务创造的价值挂钩**，而非按工时或固定费率计费。

```markdown
## 三种收费模式对比

| 模式 | 定义 | 适用场景 | Agency 偏好 | 客户偏好 |
|------|------|---------|:----------:|:--------:|
| 工时费(Time & Materials) | 按投入人天收费 | 执行型项目 | ❌ 收入上限 | ✅ 可控 |
| 固定费(Fixed Fee) | 事先约定一口价 | 清晰 scope | ❌ 风险高 | ✅ 预算确定 |
| 价值费(VBFS) | 按创造的价值收费 | 策略型项目 | ✅ 收入无上限 | ⚠️ 需信任 |
```

### 12.2 VBFS 定价模型

```yaml
VBFS_pricing:
  common_models:
    - name: "收入分成" (Revenue Share)
      formula: "Agency 费用 = 用户收入增量 × 约定百分比(5-15%)"
      best_for: "Campaign 效果可精确追踪的品类"
    - name: "奖金池" (Bonus Pool)
      formula: "基础费 + 超目标 KPI 奖金"
      best_for: "有明确 KPI 的项目"
    - name: "股權置换" (Equity Swap)
      formula: "Agency 费用部分以股权支付"
      best_for: "初创品牌/新业务线"
    - name: "混合模式" (Hybrid)
      formula: "基础费(60-80%) + 效果奖金(20-40%)"
      best_for: "大多数品牌-Agency 合作"
  
  pricing_factors:
    - "品牌当前营收规模"
    - "项目对营收的预期影响"
    - "Agency 的参与度(策略 vs 执行)"
    - "行业利润率基准"
    - "品牌-Agency 关系阶段"
```

### 12.3 Agency CEO 视角

```markdown
## VBFS 实施要点

### 对 Agency 有利
- 好方案获得好回报，激励创作更优方案
- 与客户形成"共赢"关系，而非零和博弈
- 吸引愿意投资增长的高质量客户

### 对 Agency 有风险
- 若项目效果差（非 Agency 可控因素），收入可能低于工时费
- 归因争议（客户可能归功于自己的产品团队）
- 现金流不稳定（奖金池部分到项目结束时才结算）
```

## 十三、Effie 评审框架

**适用 Agent**: creative / strategy / data

### 13.1 Effie Awards 评审标准

Effie（艾菲奖）是全球最具影响力的营销效果奖项，其评审框架可反向用于方案的自检质量评估：

| 评审维度 | 权重 | 核心评估问题 |
|---------|:---:|-------------|
| **挑战 (Challenge)** | 15% | 品牌面临的核心商业/营销挑战是什么？挑战是否有足够的说服力和重要性？ |
| **策略 (Strategy)** | 20% | 策略是否清晰？是否基于扎实的洞察？目标的设定是否 Smart？ |
| **创意 (Idea/Creative)** | 20% | Big Idea 是否与策略紧密关联？创意是否有突破性和传播力？ |
| **执行 (Execution)** | 15% | 媒介策略是否匹配？渠道组合是否合理？整合度如何？ |
| **效果 (Results)** | 30% | KPI 是否达到了？效果是否与挑战匹配？是否有"归因可信度"？ |

### 13.2 Effie 式自检模板

Agent 可将 Effie 框架用于交付物交付前的自检：

```markdown
## Effie 式质量自检

| 维度 | 自检评分(1-10) | 评语 | 改进空间 |
|------|:-----------:|------|---------|
| **挑战 (Challenge)** | /10 | 是否精准定义了核心挑战？ | [改进建议] |
| **策略 (Strategy)** | /10 | 策略是否有清晰逻辑链？ | [改进建议] |
| **创意 (Idea)** | /10 | Big Idea 是否有突破性？ | [改进建议] |
| **执行 (Execution)** | /10 | 渠道组合是否有效？ | [改进建议] |
| **效果 (Results)** | /10 | KPI 预期是否合理且有依据？ | [改进建议] |
| **总分** | **/50** | **Effie 金奖通常 ≥ 42/50** | |
```

### 13.3 Effie 的"效果链"要求

> Effie 特别重视"效果归因"的可信度——方案的 KPI 承诺必须有清晰的因果链条。

```markdown
## 效果链 (The Chain of Evidence)

### 一级：直接效果
投入(Spend) → 曝光(Reach) → 点击(CTR) → 转化(CVR) → 销售(GMV)
证据要求：平台报告 / GA 数据 / 归因模型

### 二级：品牌效果
广告活动 → 品牌知名度提升 → 品牌偏好提升 → 溢价意愿
证据要求：Brand Lift Study / 消费者追踪

### 三级：商业效果
品牌效果 → 市场份额增长 → 定价权增强 → 利润提升
证据要求：尼尔森数据 / 财报 / 行业报告

## 自检问题
1. 我的效果链的每一步都有证据支撑吗？
2. 我能区分"归因于我的Campaign"和"归因于外部因素"吗？
3. 我的 KPI 目标是否足够有野心（Effie 金奖通常要求超过行业均值 2x）？
```
