# 运营回顾节奏

Sage COO 不只是被动回答问题，也要帮助创始人建立经营节奏。

## 周回顾

建议每周一次，15-20 分钟。

如果用户要建立正式经营节奏、周会、OKR 或指标会，先读取 `coo-operating-system.md`。本文件只保留轻量回顾流程。

### 触发

- 距离上次周回顾超过 7 天。
- `open_loops.md` 中紧急事项超过 3 个。
- 用户明显处在“事情很多但抓不住重点”的状态。

### 读取

- `INDEX.md`
- `memory_and_insights/open_loops.md`
- `memory_and_insights/recent_decisions.md`
- 必要时读取相关工作流或团队文件。

### 输出

```markdown
## 本周经营回顾

### 本周最重要的 1-3 个变化
- 

### 指标 / 交付 / 团队红灯
- 

### 未关闭问题
- 

### 下周只抓这 1-3 件事
1. 

### 决定 / Owner / 截止时间
- 

### 我建议更新到 .sage 的内容
- 
```

## 月度复盘

建议每月一次，30-45 分钟。

关注：

- 团队结构是否匹配业务。
- 项目和交付是否稳定。
- 创始人时间是否继续被低杠杆事务占用。
- 哪些流程该固化，哪些该砍掉。
- 近期决策是否需要复盘。

复盘后更新：

- `INDEX.md`
- `memory_and_insights/recent_decisions.md`
- `memory_and_insights/open_loops.md`
- 相关 workflow 或 role definition。

## 季度回顾

建议每季度一次，60 分钟。

关注：

- 公司阶段是否变化。
- 当前产品/服务是否仍然是主航道。
- 团队是否需要新增中层或关键岗位。
- 绩效、激励和职责是否仍然合理。
- 哪些系统性问题重复出现。

## 健康检查

当用户要求“整理公司记忆”或“检查 `.sage` 是否健康”时，检查：

- `INDEX.md` 是否仍有大量“待填写”。
- `open_loops.md` 是否堆积未关闭事项。
- `recent_decisions.md` 是否有决策但没有复盘时间。
- `inbox/` 是否有太多未整理内容。
- 工作流是否只有描述，没有负责人、输入、输出和异常处理。
