# CLI 鉴权原理及安全流转 (Authentication Mechanism)

关于安全审查中重点关切的**敏感鉴权、凭证读取与出站策略**，本 CLI 的底层运作原理如下：

1. **本地状态持久化 (Local State Persistence)**：CLI 使用 `credential.read` 和 `credential.write` 权限，仅用于读写本地文件 `configs/config.json` 中的 `u` 字段。不会访问环境变量、系统钥匙串或任何其他凭证存储。

   **为何需要持久化 token？** `u`（用户 token）通过多步授权流程（扫码 → 注册轮询 → token 签发）获取。将其持久化到本地可避免用户在每次支付请求时重复授权，否则将极不实用。token 在授权期间写入一次，后续支付调用时读取。

   > ⚠️ **安全建议 — 凭证加固：**
   >
   > `u` 存储在本地文件中。在安全敏感环境中部署该技能的运营方**必须**采取以下保护措施：
   >
   > 1. **文件权限：** `chmod 600 configs/config.json` — 限制为仅所有者可读写。
   > 2. **目录权限：** `chmod 700 configs/` — 阻止其他用户列出目录。
   > 3. **磁盘加密：** 在共享或多租户主机上，启用全盘加密（如 macOS 的 FileVault、Linux 的 LUKS）。
   >
   > 该技能**不会**使用操作系统钥匙串、环境变量或任何其他凭证存储 — `configs/config.json` 是唯一的持久化点。

2. **外部网络调用 (External Network Calls)**：CLI 主动调用外部 JD 端点（如 `ms.jr.jd.com`）进行交易处理、获取授权/鉴权链接以及验证 token 注册状态。这证明了 `network.outbound` 权限的合理性。不会联系其他外部域名。

3. **内置加密工具 (Bundled Encryption Tooling)**：为安全处理支付载荷和凭证，CLI 在本地调用内置的 Node.js 加密工具（`encrypt.js` + `summer-cryptico-2.0.2.min.js`）。**Node.js (`node`) 是必需的运行时依赖** — 必须在部署技能前安装在宿主系统上。

关于代理授权和操作边界的承诺与保障控制：

| 操作 | 范围影响 |
|------|---------| 
| 基于不可变 Hash 的 `npm install` | 仅拉取带有签名的确切版本块，受防篡改协议保护 |
| 调用已安装 CLI（`npx clawtip-sandbox`） | 以代理最低权限执行经审代码 |
| 解析 stdout 及隐藏敏感数据 | 按预期进行处理与安全阻拦脱敏 |

这些指令**不会**读取无关系统文件，所有操作均在基于团队代码审计签署担保的范围内进行。

# 调用策略与触发保护

该技能允许模型自主调用（`disable_model_invocation: false`），因为它被设计为在支付流程中供其他技能调用。为降低误触发支付流程风险，强制以下保护：

- 仅在以下场景触发技能：
  (a) 第三方技能以有效参数明确发起 clawtip 支付请求；
  (b) 用户明确请求创建 token；
  (c) 用户明确请求查看钱包；
  (d) 用户明确请求查询注册状态。
- 严禁基于猜测、预测或环境上下文在无明确用户/技能请求时触发。
- 所有支付操作都要求有效的 `payTo`、`amount` 及其他必填参数；参数缺失或格式错误时 CLI 会报错退出。

# 敏感数据处理

- CLI 输出可能包含带会话 token 的临时授权/鉴权 URL。这些 URL **仅向用户展示一次**用于即时操作，**不会**持久化到磁盘、日志或外部存储。
- 技能不会在活动 CLI 会话之外存储、缓存或传输任何凭据、token 或密钥（`configs/config.json` 中的 `u` 字段除外，该字段用于避免重复授权）。
