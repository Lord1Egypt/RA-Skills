# 🔄 协作流程规范

## 🎯 核心原则

**串行执行，闭环确认**
- 上一步不验收通过，绝不进入下一步
- 每一步完成后，必须通知总策划验收
- 验收不通过，打回修改后重新提交

---

## 📋 六步标准流程

### 流程图示

```
用户需求
    ↓
📋 第0步：总策划制定总纲
    ↓ ✅ 用户确认
🌍 第1步：世界观构建
    ↓ ✅ 总策划验收
👥 第2步：角色设计
    ↓ ✅ 总策划验收
📑 第3步：情节规划（第一批）
    ↓ ✅ 总策划验收
✍️ 第4步：正文创作（第一批）
    ↓
🔍 第5步：质量评审
    ↓ ✅ 按建议优化
📑 第6步：情节规划（下一批）
    ↓
    🔄 循环...
```

---

## 📝 每步详细规范

### 第0步：总策划制定总纲

**负责人**：总策划
**输入**：用户需求
**输出**：《项目总纲》
**验收人**：用户

**流程**：
1. 总策划与用户充分沟通，明确需求
2. 总策划制定详细的项目总纲
3. 用户确认总纲，项目正式启动

**注意事项**：
- 需求不明确，绝不进入下一步
- 总纲必须包含：题材、篇幅、核心设定、质量标准
- 用户确认后，总纲原则上不再大幅修改

---

### 第1步：世界观构建

**负责人**：世界观构建智能体
**输入**：项目总纲
**输出**：《世界观设定》
**验收人**：总策划

**流程**：
1. 总策划下达任务，明确要求
2. 世界观构建智能体完成设定
3. 智能体提交给总策划验收
4. 总策划验收：
   - ✅ 合格 → 进入下一步
   - ❌ 不合格 → 打回修改，重新提交

**验收标准**：
- 设定清晰，无模糊地带
- 逻辑严密，无明显矛盾
- 细节丰富，有足够深度
- 符合题材和时代特征

---

### 第2步：角色设计

**负责人**：角色设计智能体
**输入**：项目总纲 + 世界观设定
**输出**：《角色档案》
**验收人**：总策划

**流程**：
1. 总策划下达任务，明确要求
2. 角色设计智能体完成角色设计
3. 智能体提交给总策划验收
4. 总策划验收：
   - ✅ 合格 → 进入下一步
   - ❌ 不合格 → 打回修改，重新提交

**验收标准**：
- 人物立体，不是工具人
- 性格鲜明，有辨识度
- 动机合理，行为符合逻辑
- 关系复杂，有戏剧张力

---

### 第3步：情节规划

**负责人**：情节策划智能体
**输入**：项目总纲 + 世界观 + 角色档案
**输出**：《分章大纲》（一批10章）
**验收人**：总策划

**流程**：
1. 总策划下达任务，明确批次（如第1-10章）
2. 情节策划智能体完成大纲
3. 智能体提交给总策划验收
4. 总策划验收：
   - ✅ 合格 → 进入下一步
   - ❌ 不合格 → 打回修改，重新提交

**验收标准**：
- 结构完整，起承转合清晰
- 节奏合理，张弛有度
- 爽点密集，钩子有力
- 伏笔巧妙，前后呼应

---

### 第4步：正文创作

**负责人**：章节写作智能体
**输入**：世界观 + 角色 + 对应章节大纲
**输出**：正文初稿（一批5-10章）
**验收人**：质量评审官

**流程**：
1. 总策划下达任务，明确章节范围
2. 章节写作智能体按章创作
3. 完成后提交给质量评审官

**创作标准**：
- 字数：1800-2200字/章
- 严格遵循大纲情节
- 人物性格保持一致
- 文笔流畅，无AI痕迹

---

### 第5步：质量评审与优化

**负责人**：质量评审官 + 章节写作智能体
**输入**：正文初稿
**输出**：评审报告 + 优化后的定稿
**验收人**：总策划

**流程**：
1. 质量评审官完成评审，给出评分和修改建议
2. 章节写作智能体根据评审意见优化
3. 提交给总策划最终验收
4. 总策划验收：
   - ✅ 合格 → 定稿，归档
   - ❌ 不合格 → 再次优化或重写

**评审标准**：
- 按《质量评分标准》五维度评分
- 总分 ≥ 70分通过
- 红线内容零容忍

---

### 第6步：下一批规划与循环

**负责人**：总策划
**输入**：上一批完成情况
**输出**：下一批的任务指令

**流程**：
1. 总策划总结上一批的经验教训
2. 情节策划智能体规划下一批大纲
3. 重复第3-5步，直到完成全部章节

---

## ⚠️ 任务下达与通知规范

### 任务下达规范

总策划下达任务时，必须明确说明：
1. **任务内容**：具体做什么
2. **输入文件**：需要参考哪些文件
3. **输出要求**：输出格式、文件名、位置
4. **质量标准**：验收标准是什么
5. **完成时间**：预计什么时候完成
6. **⚠️ 重要提醒**：完成后必须立即发消息通知总策划！

### 完成通知规范

智能体完成任务后，必须立即发送通知：
```
【任务完成通知】
我是：XXX智能体
完成的任务：XXX
输出文件：XXX.md
请总策划验收！
```

### 超时处理

1. 任务超过预期时间50%未完成 → 总策划主动询问
2. 任务超过预期时间100%未完成 → 发警告，排查原因
3. 连续3次超时 → 考虑调整智能体配置或提示词

## 📊 进度汇报规范

总策划必须定期向用户汇报进度：

### 阶段汇报
- 每完成一个阶段（如世界观、角色、大纲完成）
- 必须汇报：完成内容、质量评价、下一步计划

### 批量汇报
- 每完成一批章节（如10章）
- 必须汇报：完成章节数、质量评分、样本章节、下一步计划

### 异常汇报
- 遇到任何问题导致进度延迟或质量问题
- 必须立即汇报：问题描述、原因分析、解决方案、预计延迟时间

---

## 🔒 权限与责任

### 总策划权限
- 制定项目总纲和质量标准
- 调度所有智能体的工作
- 验收每一步的输出
- 决定是否打回修改
- 向用户汇报进度

### 各智能体责任
- 按要求完成自己的任务
- 严格遵守质量标准
- 完成后立即通知总策划
- 认真对待修改意见，持续改进

### 用户权限
- 提出需求和修改意见
- 确认项目总纲
- 随时了解项目进度
- 对最终成品有最终决定权

---

## 🎯 关键决策点

以下节点必须由总策划决策，不能跳过：

| 决策点 | 决策内容 |
|--------|---------|
| 总纲确认 | 《项目总纲》是否符合用户需求 |
| 世界观验收 | 世界观设定是否达标、逻辑自洽 |
| 角色验收 | 角色设定是否立体、有辨识度 |
| 大纲验收 | 情节大纲是否精彩、节奏合理 |
| 章节终验 | 每一章最终是否可以通过 |
| 最终交付 | 整部小说是否可以交付 |

---

**生效时间**：永久生效
**适用范围**：所有小说项目，所有智能体
**最终解释权**：总策划
