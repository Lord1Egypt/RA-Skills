## Use Case: <br>
Developers and engineers use this skill to diagnose slow systems, design profiling and load-testing workflows, optimize application and database bottlenecks, plan capacity, and add performance gates to CI/CD. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Profiling, benchmarking, or load-testing commands can affect running systems if applied without review. <br>
Mitigation: Review commands before execution and prefer staging or otherwise controlled environments. <br>
Risk: Load testing can disrupt systems or services outside the user's authority. <br>
Mitigation: Only test systems that the user owns or is authorized to test. <br>
Risk: Generated traces, heap snapshots, or memory snapshots can contain application data. <br>
Mitigation: Protect generated performance artifacts as sensitive data and share them only through approved channels. <br>
