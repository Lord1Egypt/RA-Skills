## Description: <br>
Context Driven Development helps teams create and maintain structured project context documents for AI-assisted development and team alignment. <br>

This skill is ready for commercial/non-commercial use. <br>

## Publisher: <br>
[wpank](https://clawhub.ai/user/wpank) <br>

### License/Terms of Use: <br>


## Use Case: <br>
Developers and engineering teams use this skill to establish product, technical, workflow, and work-tracking context artifacts so AI-assisted sessions and team onboarding stay aligned. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Generated or updated context files can shape future AI-assisted development if they contain stale or inaccurate assumptions. <br>
Mitigation: Review generated product, tech-stack, workflow, and tracking documents before relying on them and update them as project decisions change. <br>
Risk: The README includes a moving GitHub branch install command while server-resolved provenance is unavailable. <br>
Mitigation: Prefer the ClawHub install command or inspect and pin any external source before installation. <br>
