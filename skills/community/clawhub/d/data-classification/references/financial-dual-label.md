# 金融数据双标签体系

## 双标签定义

对金融字段同时输出两套标签：

1. **通用数据标签**：依据 GB/T 43697-2024，给出行业领域、描述对象/数据主体、内容类别、通用级别（一般/重要/核心）。
2. **金融数据标签**：依据 JR/T 0197-2020，给出金融业务分类（一级/二级/三级/四级子类）和最低安全级别（1-5）。

推荐格式：

`金融数据 / 用户数据-个人信息-个人身份信息 / 一般数据 + 客户-个人-个人身份鉴别信息-传统鉴别信息 / JR/T 4级`

## JR/T 0197-2020 要点

- 等级：5级 > 4级 > 3级 > 2级 > 1级。
- 影响对象：国家安全、公众权益、个人隐私、企业合法权益。
- 影响程度：严重损害、一般损害、轻微损害、无损害。
- 定级依据：保密性、完整性、可用性；保密性优先。
- 重要数据不低于5级。
- 个人金融信息可参考 JR/T 0171 C1/C2/C3 分类；认证、账户、交易、信用等从严。

## JR/T 0197 附录 A 优先规则

金融字段分类时，优先检索并匹配：

- `jrt0197-appendix-a-full.csv`：完整机器可读表，适合脚本/grep 检索。
- `jrt0197-appendix-a-compact.md`：紧凑 Markdown 表，适合人工阅读。

匹配顺序：

1. 字段注释/中文字段名精确命中四级子类或三级子类。
2. 表名/业务上下文命中二级或三级子类，再用字段名细化到四级子类。
3. 多个候选都合理时，输出“推荐金融标签”和“候选金融标签”，推荐项按语义贴近度和就高从严选择。
4. 附录 A 无清晰匹配时，才使用下面的启发式。

## 审核提醒

- 字段名与表名冲突时，以更具体的字段注释优先，其次字段名，再其次表名。
- `id`、`code`、`status`、`type` 这类字段必须结合表名/注释，否则低置信度。
- 若 SQL 中含表注释如 `客户`、`账户`、`交易`、`贷款`，可提升相关字段的金融语义置信度。
- 字段存在多个合理 JR/T 落点时，应输出“推荐金融标签”和“候选金融标签”；例如“企业借款金额”推荐 `客户-单位-单位资讯信息-企业信贷信息/JR-T 3级`，同时列出 `业务-合约协议-贷款业务信息-放还款信息/JR-T 2级` 作为候选。
- 对批量结果，建议把 `置信度 < 0.6` 或命中 `需确认` 的字段单独列为复核清单。
