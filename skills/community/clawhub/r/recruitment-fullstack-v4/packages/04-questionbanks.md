# 📚 7 大岗位差异化题库

## 统一结构

每个题库包含：
1. 核心维度与权重
2. 必问主题题（含考察目标 + 红灯）
3. A/B/C/D 追问模板（针对该岗位）
4. 协作模拟题
5. Deal Breakers
6. 特殊评估提示

---

# 🎯 题库 1：销售岗

## 1.1 核心维度与权重

| 维度 | 权重 | 说明 |
|------|------|------|
| 业绩真实性 | 30% | 销售岗最大造假风险 |
| 客户获取与转化 | 20% | 核心能力 |
| 抗压与韧性 | 15% | 销售本质是被拒绝 |
| 资源积累 | 10% | 行业人脉 / 方法论 |
| 团队协作 | 10% | 与售前 / 交付 / 客户成功 |
| 学习与复盘 | 10% | 顶级销售必有方法论 |
| 商业敏感度 | 5% | 大客户岗位更重要 |

## 1.2 必问主题题（10 题）

| # | 问题 | 考察 | 🔴 红灯 |
|---|------|------|--------|
| S1 | 描述过去 3 年业绩：年度指标 / 实际完成 / 排名 / Top 3 大单的客户金额周期 | 业绩真实性 | 含糊带过 / 说不清排名 |
| S2 | 用 PEARL 完整复盘你做过最有难度的一单 | 方法论沉淀 | 只讲结果不讲过程 |
| S3 | 你从 0 接触新客户到签约的典型流程？平均周期？转化率？ | 流程化能力 | 说不出转化漏斗 |
| S4 | 讲一个你以为能签但最终没签的大单。事后怎么复盘？ | 学习 + 诚实 | "没有失过单" |
| S5 | 经历过最差的一个季度 / 年是怎样的？怎么应对？ | 韧性 | 全推外部 |
| S6 | 你目前能直接拨通电话的客户决策人有多少？最近沟通了什么？ | 资源真实性 | 说有但举不出近期案例 |
| S7 | 最近一次和竞品同台竞争的输 / 赢案例 | 竞争意识 | "从未输给竞品" |
| S8 | 客户砍价砍到你不能接受时怎么处理？最近案例 | 谈判技巧 | 只会让价 / 只会强硬 |
| S9 | 上一次和售前 / 交付 / 产品冲突？怎么解决？ | 协作 | 从无冲突 |
| S10 | 你上家公司业绩排名第几？真实计算方式？为什么离开 Top 位置？ | 动机真实性 | 排名靠前但解释不清离职 |

## 1.3 销售岗 A/B/C/D 追问模板

**触发场景**：候选人说"我业绩 1000 万"

```
A-归因：1000 万里多少是你自己开发？多少是分配 / 老客户续约？团队同岗位平均业绩多少？你排名？
B-方法：Top 3 单子的客户来源分别是？（陌拜 / 转介绍 / 市场线索）你的销售方法论是什么？画一下漏斗？
C-度量：1000 万是签约额、回款额还是确认收入？统计周期？提成多少？
D-反事实：如果不在这家公司，独立创业也能做到这个数吗？哪些单子有"运气"成分？
```

## 1.4 协作模拟题

> "假设我们要进入 [目标行业] 市场，你作为销售负责人，第一周做什么？给我 30/60/90 天计划。"

## 1.5 销售岗 Deal Breakers

- ❌ 简历显示业绩 Top 但说不清排名计算方式
- ❌ 离职时正值业绩高峰且无合理解释
- ❌ 客户资源都在"前公司客户"，无私人迁移性
- ❌ 拒绝提供 reference（销售岗必做背调）

## 1.6 特殊评估提示

- **销售岗 TORC 必问**（业绩造假率高，背调可参考行业TORC框架）
- **重点验证 Top 3 大单的真实性**

# 🎯 题库 2：技术岗

## 2.1 核心维度与权重

| 维度 | 权重 | 说明 |
|------|------|------|
| 技术深度 | 30% | 核心能力 |
| 系统设计 | 20% | 高级岗位更重要 |
| 问题解决与 debug | 15% | 真实场景能力 |
| 工程素养 | 10% | 代码 / 测试 / 文档 |
| 学习能力 | 10% | 技术更新快 |
| 协作沟通 | 10% | 跨团队 / 向上汇报 |
| 业务理解 | 5% | 资深技术必备 |

## 2.2 必问主题题（10 题）

| # | 问题 | 考察 | 🔴 红灯 |
|---|------|------|--------|
| T1 | 选最有技术含量的项目，从架构讲起，逐层追问到代码细节 | 技术深度真实性 | 只能讲到框架名称 |
| T2 | [项目中] 为什么选 A 技术而非 B？当时还考虑哪些方案？做了什么权衡？ | 决策深度 | "大家都用"/"老板让用" |
| T3 | 设计 [千万级用户消息推送系统]，怎么设计？ | 架构能力 | 直接给方案不问需求 |
| T4 | 处理过最严重的线上故障？从发现到解决的完整过程 | 实战能力 | 从未遇到故障 |
| T5 | 接手过最烂的代码？怎么处理？最终效果？ | 工程素养 | 直接重写不考虑业务连续性 |
| T6 | 性能优化的具体案例？优化前后数据？瓶颈怎么定位的？ | 底层能力 | 只会用工具不懂原理 |
| T7 | 坚持技术正确性、PM/老板要快速上线的冲突案例 | 协作 + 判断 | 完全妥协或完全坚持 |
| T8 | 过去 6 个月学的新技术？怎么学的？学到什么程度？我能考你吗？ | 学习深度 | 只能说"看过文档" |
| T9 | 最近一次 review 别人代码给出的最有价值反馈？被 review 收到的？ | 工程文化 | 不做 review / 只看格式 |
| T10 | 需要其他团队配合但他们没动力时怎么推动？具体案例 | 影响力 | 只会找老板 / 只会硬上 |

## 2.3 技术岗 A/B/C/D 追问模板

**触发场景**：候选人说"我做了 XX 系统的架构设计"

```
A-归因：哪些是你设计的？哪些是 follow 已有规范？团队 senior 是谁？他做了什么？
B-方法（6 层深技术追问）：
  整体架构 → 关键模块 → 核心算法/数据结构
  → 实现细节 → 边界处理 → 优化方向
C-度量：QPS / 延迟 / 可用性具体数字？怎么测的？上线后实际数据 vs 设计目标差距？
D-反事实：重新设计会改什么？当时选另一个方案结果会怎样？
```

## 2.4 协作模拟题（按子方向）

| 子方向 | 题目 |
|--------|------|
| 后端 | 设计分布式 ID 生成器 |
| 前端 | 设计无限滚动列表组件 |
| 算法 | 给定业务场景设计推荐系统 |
| 大数据 | 设计实时数仓架构 |
| 移动端 | 设计离线优先的数据同步机制 |
| 安全 | 设计零信任架构 |

## 2.5 技术岗 Deal Breakers

- ❌ 简历技术栈与项目深度严重不符
- ❌ 无法讲清自己声称掌握技术的底层原理
- ❌ 故障案例细节经不起追问
- ❌ 完全无法做系统设计题（高级岗）

## 2.6 特殊评估提示

- **6 层深追问必做**（区分真深度和包装）
- **现场写代码或白板设计可选项**（高级岗推荐）
- **GitHub / 技术博客是加分参考**

# 🎯 题库 3：产品岗

## 3.1 核心维度与权重

| 维度 | 权重 | 说明 |
|------|------|------|
| 用户洞察 | 25% | 产品经理灵魂 |
| 商业判断 | 20% | 资深 PM 关键 |
| 项目落地能力 | 15% | 推动力 |
| 数据分析 | 10% | 决策依据 |
| 跨团队协作 | 10% | 影响力 |
| 战略思维 | 10% | 总监级必备 |
| 产品 sense | 10% | 难量化但关键 |

## 3.2 必问主题题（10 题）

| # | 问题 | 考察 | 🔴 红灯 |
|---|------|------|--------|
| P1 | 你做过最得意的产品 / 功能是？为什么是它？ | 自我认知 + 产品观 | 选择标准是"老板表扬" |
| P2 | 最近一个产品决策，背后的用户洞察怎么获得？做了几次访谈？看了什么数据？ | 调研深度 | 拍脑袋 / 只看老板想法 |
| P3 | 用 PEARL 复盘你主导的产品项目 | 方法论 | — |
| P4 | 讲一个你做砸了的产品。砸在哪？为什么？ | 诚实 + 反思 | 全推开发 / 声称"还在迭代" |
| P5 | 上次 ABC 三个需求要砍 1 个，怎么决策？现在回看对吗？ | 判断框架 | 没有框架 / 完全按老板 |
| P6 | 举一个你用数据推翻了直觉判断的案例 | 数据敏感度 | 只看 DAU |
| P7 | 和开发 / 设计 / 运营冲突时怎么处理？最近案例 | 影响力 | 靠老板 / 靠 deadline |
| P8 | 评价你常用 App 的最大问题？让你改你怎么改？ | 产品 sense | 只能说 UI / 没有用户视角 |
| P9 | 你的产品怎么赚钱？关键商业指标？指标和体验有冲突吗？ | 商业意识 | 完全不关心商业 |
| P10 | 你做 0-1 多还是 1-N 多？哪个更难？为什么？ | 经验识别 | 说不清自己擅长哪个 |

## 3.3 产品岗 A/B/C/D 追问模板

**触发场景**：候选人说"我做了 XX 功能，提升留存 15%"

```
A-归因：15% 里这个功能贡献多少？怎么归因？同期还有什么变化？（运营活动 / 外部因素）
B-方法：用户洞察怎么得来的？看了多少用户 / 数据？为什么是这个方案？砍掉了哪些方案？
C-度量：留存的定义？次留 / 7留 / 30留？基线多少？置信区间？长期效果（3 个月后还提升吗）？
D-反事实：不做这个功能，留存会自然变化吗？是否带来了负向指标？（如时长下降）
```

## 3.4 协作模拟题

**题目 A**：给你 [某产品]，假设你接手做产品负责人，第一周做什么？前 3 个月规划？
**题目 B**：白板题——设计一个 [岗位相关] 产品的核心功能

## 3.5 产品岗 Deal Breakers

- ❌ 说不清自己产品的核心指标
- ❌ 从未做过用户访谈 / 调研
- ❌ 简历里全是"参与"没有"主导"（资深岗）
- ❌ 没有产品观点 / 只会执行

## 3.6 特殊评估提示

- **产品 sense 难量化但关键** — 必出现场题
- **资深 PM 必问商业判断** — 区分"功能 PM"和"业务 PM"
- **数据驱动能力必验证** — 直接问指标定义

# 🎯 题库 4：管理岗

## 4.1 核心维度与权重

| 维度 | 权重 | 说明 |
|------|------|------|
| 团队搭建 | 25% | 招 / 留 / 育人 |
| 战略思维 | 20% | 方向判断 |
| 业绩达成 | 15% | 结果交付 |
| 领导力 | 15% | 影响他人 |
| 决策能力 | 10% | 关键时刻 |
| 跨部门协同 | 10% | 横向影响 |
| 危机处理 | 5% | 应变能力 |

## 4.2 必问主题题（12 题）

| # | 问题 | 考察 | 🔴 红灯 |
|---|------|------|--------|
| M1 | 目前管多少人？团队结构？直接汇报几个？各负责什么？ | 基本盘 | 说不清结构 |
| M2 | 管理生涯招过最满意的一个人是谁？为什么选他？后来发展？ | 识人能力 | — |
| M3 | Fire / 劝退过下属吗？最近一次的情况？决策过程？反思？ | 管理勇气 | 从未开过人（管理 5 年以上） |
| M4 | 如果你明天离开，谁接替？团队 high / low performer 分别是？怎么处理？ | 人才盘点 | 说不出名字 |
| M5 | 接手团队时业绩如何？任内最好成绩？关键动作？ | 交付能力 | — |
| M6 | 团队没达目标的季度。怎么向上汇报？怎么调整？ | 诚实 + 应变 | 从未失败 / 推团队 |
| M7 | 最近为团队制定战略？过程？现在回看对错？ | 战略能力 | — |
| M8 | 和平级负责人最严重的冲突？怎么解决？现在关系？ | 横向影响 | 靠老板裁决 |
| M9 | 和老板最近一次重大分歧？怎么处理？ | 向上沟通 | 从无分歧 |
| M10 | 你的团队文化是怎样的？怎么塑造的？有什么具体动作？ | 领导力深度 | 只能说"开放""信任"空话 |
| M11 | 讲一个管理生涯的至暗时刻。怎么挺过来的？ | 韧性 | — |
| M12 | 如果电话你随机一位下属，他会怎么评价你？最不满意你哪点？ | 自我认知 + TORC | 完美主义型回答 |

## 4.3 管理岗 A/B/C/D 追问模板

**触发场景**：候选人说"我带领团队完成了 XX 目标"

```
A-归因（管理岗尤其重要）：团队里谁是真正关键贡献者？没有他能做成吗？你个人做了什么独特的事？哪部分功劳应归你前任/继任者？
B-方法：做了哪些具体管理动作？（OKR / 1-on-1 / review 机制）激励机制是什么？谁设计的？关键决策时刻你做了什么判断？
C-度量：团队人数变化？人均产出变化？离职率？关键人才保留率？业绩 vs 公司平均 vs 同行业？
D-反事实：你不做这个团队，谁能做到同样结果？团队成功多少是市场红利 / 公司平台？
```

## 4.4 协作模拟题

**情境**："你接手一个 [团队规模] 的团队，现状是 [问题描述]，给你 90 天，前 30/60/90 天做什么？"

变体：接手低绩效团队 / 接手内部冲突严重的团队 / 接手快速扩张中的团队 / 接手核心人才即将离职的团队

## 4.5 管理岗 Deal Breakers

- ❌ 说不清直接下属的优劣势
- ❌ 从未做过 fire 决策（管理 5 年以上）
- ❌ 团队离职率显著高于行业且无反思
- ❌ 全员都"特别好"的回避型回答

## 4.6 特殊评估提示

- **管理岗必做反向 reference**（联系前下属）
- **必问 fire 经历**（管理勇气是核心）
- **团队盘点能力**是高级管理岗硬指标

| 跨部门服务 | 15% | 内部客户思维 |

| 数据敏感度 | 10% | 现代职能必备 |
| 项目推动 | 5% | 职能项目化 |

## 5.2 必问主题题（10 题）

| # | 问题 | 考察 | 🔴 红灯 |
|---|------|------|--------|
| F1 | 日常 80% 时间花在什么事情上？哪些是你最擅长的？ | 专业聚焦 | — |
| F2 | 你服务的业务部门做什么？他们核心目标？你的工作怎么支持？ | 业务理解 | 说不清业务 |
| F3 | 主导 / 参与过的体系化建设项目？讲最有代表性的 | 建设能力 | 只会执行 |
| F4 | 最近一次合规 / 风险问题？怎么处理？ | 风险意识 | 从未遇到 |
| F5 | 业务提了一个你认为不合规 / 不合理的需求，怎么处理？案例 | 沟通 + 原则 | 硬拒或全顺从 |
| F6 | 用什么数据 / 指标衡量自己工作？数据推动改进的案例？ | 数据思维 | 完全无数据 |
| F7 | 工作需要哪些部门配合？最难推动的？怎么搞定？ | 影响力 | — |
| F8 | 用工具 / 系统 / 流程优化提升效率的案例？ | 现代素养 | 还在纯人力处理 |
| F9 | 你专业领域过去 1 年有什么新变化？怎么跟进？ | 专业发展 | 完全没关注 |
| F10 | 为什么离开上一家？职能岗在那里话语权如何？新岗位看重什么？ | 动机 | 因"业务看不起职能"且无反思 |

## 5.3 职能岗 A/B/C/D 追问模板

**触发场景**：候选人说"我搭建了 XX 体系"

```
A-归因：你主导还是参与？角色具体是？决策者是谁？你的输入哪些被采纳？
B-方法：搭建前做了什么调研？参考了什么标杆？落地遇到什么阻力？怎么解决？如何让业务部门接受这个体系？
C-度量：上线后效果？什么指标改善？业务满意度怎么评估？怎么知道这个体系是"好"的？
D-反事实：不做这个体系业务会怎样？重做会改什么？
```

## 5.4 协作模拟题（按子专业）

| 子专业 | 题目 |
|--------|------|
| HR | 快速扩张 50→200 人，HR 三大块怎么搭？ |
| 财务 | 0-1 业务线的财务体系怎么建？ |
| 法务 | 业务做新产品涉及 [合规场景]，如何评估和推进？ |
| 行政 | 多地办公（北上深）的行政体系怎么建？ |
| IT | 公司从 100 人扩到 500 人，IT 基础设施怎么规划？ |

## 5.5 职能岗 Deal Breakers

- ❌ 完全不懂所服务的业务
- ❌ 没有体系化思维（高级岗）
- ❌ 缺乏风险意识（涉合规岗位）
- ❌ "等业务来找我"的被动模式

## 5.6 特殊评估提示

- **判断"懂业务的职能" vs "纯支持型职能"是高级岗关键**
- **现代职能岗必须有数据化和工具化能力**
- **合规类岗位必问"红线坚持"案例**

# 🎯 题库 6：转行候选人

## 6.1 核心维度与权重（与标准岗位不同）

| 维度 | 权重 | 说明 |
|------|------|------|
| 转行动机真实性 | 25% | 防"逃避型转行" |
| 能力迁移性 | 25% | 过往经验如何用 |
| 学习能力 | 20% | 决定能否快速补齐 |
| 心理预期管理 | 10% | 是否接受降薪 / 降级 |
| 行业理解深度 | 10% | 是否真了解新行业 |
| 抗压与稳定性 | 10% | 转行适应期压力 |

## 6.2 必问主题题（10 题）

| # | 问题 | 考察 | 🔴 红灯 |
|---|------|------|--------|
| TR1 | 为什么想转行？具体是因为什么触发了这个决定？ | 动机真实性 | 抱怨前行业/逃避问题 |
| TR2 | 你认为过去的工作中，哪些能力可以迁移到新岗位？ | 能力迁移 | 说不出具体技能名称 |
| TR3 | 你为这次转行做了什么准备？（课程 / 项目 / 自学） | 主动规划 | 什么都没做 |
| TR4 | 你预计转行后需要多久才能达到独当一面？你怎么算出来的？ | 自我认知 | "很快"无依据 |
| TR5 | 你能接受薪资调整吗？幅度多少可以接受？ | 预期管理 | 不能低于现有薪资 |
| TR6 | 转行后你最大的短板是什么？你打算怎么补？ | 自我认知 | 回避问题 |
| TR7 | 新行业你最想深入了解的是什么？你打算怎么了解？ | 行业理解 | 没有具体想法 |
| TR8 | 讲一个你在原行业做得好的案例，说明其中哪些方法论可以复用 | 能力迁移 | 无法举出具体案例 |
| TR9 | 转行初期遇到最大挑战是什么？你怎么应对？ | 抗压能力 | 回避适应期困难 |
| TR10 | 家人 / 伴侣支持你转行吗？（如有） | 稳定性 | 家人强烈反对且无独立判断 |

## 6.3 转行候选人 A/B/C/D 追问模板

**触发场景**：候选人说"我在金融行业做了 5 年，想转行做产品经理"

```
A-归因：为什么是产品经理而不是其他岗位？你在金融行业做得好为什么想离开？
B-方法：你认为金融背景具体能怎么用在产品工作中？举一个具体例子？
C-度量：你学过哪些产品相关的技能？到什么程度？做过什么项目？有数据吗？
D-反事实：如果转行后 6 个月发现不适合，你会怎么办？重来一次你会改变什么？
```

## 6.4 协作模拟题

**题目**：假设你已有 [新行业岗位] 的基础技能，现在接手一个 [具体业务场景]，你在前 90 天会怎么做？重点是如何利用你原来的背景优势。

## 6.5 转行候选人 Deal Breakers

- ❌ 动机是"逃避"前行业问题而非"追求"新机会
- ❌ 对新行业完全不了解，没有任何准备
- ❌ 薪资预期不切实际（不能低于原薪资）
- ❌ 说不清自己能力的迁移路径

## 6.6 特殊评估提示

- **转行候选人必须叠加主岗位题库**（转行题库 40% + 主岗位题库 60%）
- **重点考察学习能力和主动规划**，而非已有技能对口度
- **心理预期管理**是转行成功的关键预测因子

# 🎯 题库 7：实习/应届

## 7.1 核心维度与权重

| 维度 | 权重 | 说明 |
|------|------|------|
| 学习能力 | 30% | 应届最核心能力 |
| 潜力识别 | 25% | 成长型思维 |

| 专业基础 | 15% | 与岗位相关的基础知识 |
| 稳定性 | 10% | 是否能坚持 |

## 7.2 必问主题题（10 题）

| # | 问题 | 考察 | 🔴 红灯 |
|---|------|------|--------|
| Y1 | 你最自豪的学习经历是什么？是怎么学的？ | 学习能力 | 只会跟着学校走 |
| Y2 | 遇到过完全没有头绪的问题吗？怎么解决的？ | 问题解决 | 从未遇到困难 |
| Y3 | 你对我们公司/岗位了解多少？为什么想来这里？ | 动机 + 准备 | 一无所知 |
| Y4 | 你做过最有挑战的项目/作业是什么？怎么完成的？ | 抗压 + 执行 | 没有什么挑战 |
| Y5 | 你的职业规划是什么？这个岗位在你的规划中处于什么位置？ | 自我认知 | 没有规划 |
| Y6 | 你和同学 / 组员合作项目时，角色是什么？发生过冲突吗？ | 协作能力 | 从无冲突 / 全是自己的功劳 |
| Y7 | 你有什么短板？你打算怎么改进？ | 自我认知 | "我太完美主义了" |
| Y8 | 最近一次主动学习新技能是什么时候？怎么学的？ | 学习主动性 | 学校没教就不会 |
| Y9 | 你对薪资的期望是多少？为什么？ | 预期管理 | 超出市场应届水平 |
| Y10 | 你还有什么想让我们知道但还没问到的吗？ | 自我推介 | "没有了" |

## 7.3 实习/应届 A/B/C/D 追问模板

**触发场景**：候选人说"我虽然没有经验，但我学习能力很强"

```
A-归因：学习能力很强是你自己评价的还是有人这样评价过你？举一个具体例子？
B-方法：你具体是怎么学习的？看书/课程/项目/向人请教？哪个方法对你最有效？
C-度量：学过到什么程度？能用它解决什么问题？有没有具体的成果？
D-反事实：如果让你重新学一个技能，你会改变什么学习方法？
```

## 7.4 协作模拟题

**题目**：给你一个 [具体任务，如：分析某款 App 的用户留存问题]，你完全没有相关经验，你打算怎么做？前 3 天你会做什么？

## 7.5 实习/应届 Deal Breakers

- ❌ 对公司和岗位完全不了解，没有任何准备
- ❌ 职业规划空白或完全不现实
- ❌ 薪资预期严重超出市场水平
- ❌ 完全没有任何主动学习的经历

## 7.6 特殊评估提示

- **实习/应届必须叠加主岗位题库**（实习题库 40% + 主岗位题库 60%）
- **潜力比经验更重要** — 重点考察学习能力和成长型思维
- **文化匹配在应届阶段更为关键** — 决定能否融入团队
