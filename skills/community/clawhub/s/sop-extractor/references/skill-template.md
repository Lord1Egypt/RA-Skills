## 异常处理

### 工具调用失败

| 失败类型 | 处理方式 |
|---------|---------|
| 工具未安装 | 自动尝试安装（pip install / apt install），失败则降级为人工 |
| API 超时/网络抖动 | 自动重试 2 次，间隔 5 秒 |
| 凭证过期/权限不足 | 提示用户检查凭证，给出具体排查指引 |
| 接口变更/不兼容 | 记录错误，降级为 🟡 半自动模式 |

### 人工中转超时

> 🔴 人工环节如果用户超过 30 分钟未回来，主动提醒一次。
> 超过 2 小时未回来，保存当前进度，告诉用户如何恢复。

### 凭证配置

> 首次使用需要凭证的工具时，按以下流程处理：

1. 检查环境变量是否存在
2. 缺失 → 告知用户需要什么、为什么需要
3. 引导用户获取凭证（给具体步骤和链接）
4. 用户提供后，写入环境变量并立即验证
5. 验证通过 → 继续执行，以后不再询问
6. 验证失败 → 告知原因和解决方法

> 详细凭证模板见 `references/credential-setup-guide.md`
