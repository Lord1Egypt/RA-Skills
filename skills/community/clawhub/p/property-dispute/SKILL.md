# 物业纠纷全流程维权助手

## 概述

此技能将物业纠纷维权过程转化为结构化的6阶段工作流。业主只需描述遇到的问题，
系统自动输出：权利评估、法律依据、升级路径、投诉文件、谈判话术、财务审计
方案——全部汇集在一份交互式HTML可视化报告中。

## 工作流程

当业主描述物业纠纷问题时，按以下6个阶段依次执行：

### 阶段1：信息采集与问题分类

首先向业主确认以下关键信息（如用户已提供则跳过）：

1. **问题类型**（可多选）：
   - 设施维修（电梯/屋顶/外墙/管道/消防设施等）
   - 物业费争议（涨价/乱收费/服务质量不达标）
   - 公共收益被侵占（广告费/停车费/场地租金等）
   - 账目不透明（拒绝公开/虚假公示）
   - 服务质量（卫生/安保/绿化等不达标）
   - 相邻侵权（噪音/违建/占用通道等）
   - 人身/财产损害（高空抛物/设施脱落致损等）

2. **时间跨度**：问题持续多久
3. **已尝试的沟通**：与物业/业委会/街道办的沟通历史
4. **现有证据**：合同、缴费记录、照片、视频、聊天记录等
5. **所在城市**（用于匹配地方性法规）

### 阶段2：权利评估

基于业主描述的问题，从以下维度进行权利评估：

1. **实体权利**：
   - 查 `references/laws.md` 中民法典第271-287条，确认业主对共有部分的权利
   - 确认物业的对应义务（民法典第942条服务义务、第943条公开义务）
   - 确认是否构成紧急情况（民法典第281条）

2. **程序权利**：
   - 监督权（民法典第285条）
   - 知情权/查账权（民法典第943条）
   - 解聘权（民法典第946条）
   - 索赔权（物业管理条例第36条）

3. **评分体系**（0-100分）：
   - 法律依据明确性：30分
   - 证据充分度：25分
   - 问题严重性：25分
   - 维权紧迫性：20分

4. **紧迫程度判定**：
   - 🔴 紧急（24小时内）：涉及人身安全（电梯故障/消防隐患/高空坠物风险）
   - 🟡 较急（7天内）：严重影响生活（漏水/停水停电/房屋受损）
   - 🟢 常规（30天内）：费用争议/服务不达标

### 阶段3：法律条款速查

从 `references/laws.md` 中调取对应的法律条款，按优先级排列：

1. **与业主问题直接相关的核心条款**（1-3条）
2. **支撑性条款**（2-5条）
3. **处罚条款**（1-3条）

每个条款输出格式：
- 条款编号和名称
- 内容概要
- 与当前问题的关联说明
- 物业不遵守的法律后果

### 阶段4：升级流程导航

参考 `references/complaint_flow.md`，为业主设计五级升级路径：

1. **级别1：物业直接沟通** — 生成书面整改通知
2. **级别2：业委会介入** — 如需解聘物业，说明表决程序
3. **级别3：街道办/12345投诉** — 投诉要点
4. **级别4：多部门联合举报** — 住建局+市场监管局+税务局+消防组合策略
5. **级别5：法律诉讼** — 诉讼方案和成本估算

每个级别都标注：
- 预计耗时
- 需要的材料清单
- 成功概率评估
- 如果不成功，升级到下一级的触发条件

### 阶段5：投诉文件生成

根据业主的具体问题，生成以下文件：

1. **《整改通知书》**（致物业公司）— 含法律依据引用、整改期限、逾期后果
2. **《投诉举报函》**（致住建局/街道办）— 含事实描述、证据清单、诉求
3. **《信息公开申请书》**（致物业公司）— 要求公开账目、公共收益等
4. **证据收集清单** — 告知业主还需要补充哪些证据

文件格式参考 `references/negotiation_scripts.md` 中的风格：
- 引用具体法条
- 明确期限
- 告知后果
- 保持专业冷静

### 阶段6：公共收益与税务审计

分析物业潜在的财务违规点：

1. **公共收益审计清单**：
   - 电梯广告收入
   - 地面/地下停车费（业主共有部分）
   - 公共区域摊位/场地租赁
   - 快递柜/自动售货机场地费
   - 通信基站场地租金

2. **物业费收支核对要点**：
   - 人工成本是否虚高
   - 外包服务价格是否合理
   - 维修支出是否有报价单+验收报告
   - 代收代缴水电费差额

3. **税务举报要点**（如发现违规）：
   - 不开发票收费
   - 收入申报不实
   - 员工工资私发
   - 举报渠道：税务局官网实名举报/12366热线

## 输出格式

完成6阶段分析后，按以下结构生成交互式HTML维权报告：

### 报告结构

1. **顶部仪表盘**：权利指数/紧迫程度/胜诉概率/可追回金额 4个核心指标
2. **权利评估**：业主依法享有的各项权利清单
3. **法律依据**：适用的法律条款卡片墙
4. **升级流程**：时间线式的逐级维权路径
5. **投诉文件**：可直接打印/发送的投诉文书
6. **谈判脚本**：多场景谈判话术
7. **税务审计**：公共收益核查清单和税务举报策略
8. **联系方式**：当地各部门联系方式

### HTML生成规则

- 使用 `assets/report_template.html` 作为模板
- 将所有 `{{VARIABLE}}` 占位符替换为实际内容
- 输出的HTML文件保存到当前工作目录，命名为 `物业维权报告_[问题概要]_[日期].html`
- 用 `present_files` 展示最终报告

## 注意事项

1. **法律边界**：报告末尾必须标注"以上内容为法律信息参考，不构成正式法律意见。建议在采取重大法律行动前咨询专业律师。"
2. **隐私保护**：不要求业主提供身份证号、详细住址等非必要个人信息
3. **地方差异**：尽量获取业主所在城市，匹配地方性法规（如深圳的i深圳APP查账、杭州的第三方云账本等）
4. **证据为王**：反复向业主强调证据收集的重要性，提供具体的证据收集清单
5. **冷静理性**：所有输出文本保持专业、客观、冷静的语调，用事实和法律说话，不煽动情绪
