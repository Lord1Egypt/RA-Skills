# 评价沉淀与信用更新

> 本文档详细说明双方互评机制、信用更新规则、知识入库流程等评价沉淀相关内容。

---

## 一、评价机制

### 1.1 双向评价体系

| 评价方 | 被评价方 | 评价权重 | 评价时间 |
|--------|----------|----------|----------|
| **甲方** | 乙方（执行方） | 70% | 项目结束/里程碑验收后 |
| **乙方** | 甲方（需求方） | 30% | 项目结束/里程碑验收后 |

### 1.2 评价维度（5分制）

| 维度 | 说明 | 优秀(5分) | 良好(4分) | 一般(3分) | 较差(2分) | 很差(1分) |
|------|------|-----------|-----------|-----------|-----------|-----------|
| **交付质量** | 成果物质量 | 超预期 | 符合要求 | 基本达标 | 部分不达标 | 完全不达标 |
| **时效性** | 是否按时完成 | 提前完成 | 按期完成 | 轻微延期 | 明显延期 | 严重延期 |
| **沟通协作** | 沟通效率 | 非常顺畅 | 比较顺畅 | 一般 | 沟通困难 | 无法沟通 |
| **专业能力** | 解决问题的能力 | 游刃有余 | 能解决问题 | 基本能解决 | 解决困难 | 无法解决 |
| **服务态度** | 配合意愿 | 非常积极 | 比较积极 | 一般 | 比较消极 | 非常消极 |

### 1.3 甲方评乙方（执行方）

| 评价维度 | 权重 | 评价要点 |
|----------|------|----------|
| **交付质量** | 25% | 成果物是否满足要求，是否有创意 |
| **时效性** | 25% | 是否按时交付，是否有延误 |
| **专业能力** | 20% | 技术水平，问题解决能力 |
| **沟通协作** | 15% | 响应速度，沟通效率 |
| **服务态度** | 15% | 配合意愿，责任心 |

### 1.4 乙方评甲方（需求方）

| 评价维度 | 权重 | 评价要点 |
|----------|------|----------|
| **配合程度** | 30% | 是否积极配合工作 |
| **需求清晰度** | 25% | 需求是否明确、及时反馈 |
| **付款及时性** | 25% | 是否按时付款 |
| **沟通效率** | 20% | 是否及时响应 |

## 二、信用评级

### 2.1 评级体系

| 评级 | 分值区间 | 权益影响 | 说明 |
|------|----------|----------|------|
| **S级** | 95-100分 | 收益分配+20%，优先参与新项目 | 卓越表现 |
| **A级** | 85-94分 | 正常权益 | 优秀表现 |
| **B级** | 70-84分 | 收益分配-10%，观察期1个月 | 良好表现 |
| **C级** | 55-69分 | 收益分配-30%，降级警告 | 需改进 |
| **D级** | <55分 | 取消参与资格，启动退出程序 | 不合格 |

### 2.2 信用分计算

```
信用分 = 基础分100分 + 加分项 - 扣分项
```

**加分项**：

| 加分项 | 分值 | 说明 | 最高累计 |
|--------|------|------|----------|
| 提前完成交付物 | +3分/次 | 需验收通过 | 无上限 |
| 超额完成目标 | +5分/次 | 需量化证明 | 无上限 |
| 提出有效改进建议 | +2分/次 | 被采纳并实施 | 无上限 |
| 主动承担额外工作 | +3分/次 | 经三长老确认 | 无上限 |
| 获得客户点名表扬 | +5分/次 | 有书面记录 | 无上限 |

**扣分项**：

| 扣分项 | 分值 | 说明 | 备注 |
|--------|------|------|------|
| 交付物延期 | -3分/次 | 每延期1天 | 最多扣30分 |
| 低于承诺80%投入 | -5分/次 | 按月统计 | 无上限 |
| 违反保密义务 | -10分/次 | 造成损失另赔 | 无上限 |
| 重大失误 | -10分/次 | 影响项目进展 | 无上限 |
| 被投诉 | -8分/次 | 经核实有效投诉 | 无上限 |
| 质量问题返工 | -5分/次 | 需返工修复 | 无上限 |

### 2.3 评级调整规则

| 调整类型 | 触发条件 | 调整方式 |
|----------|----------|----------|
| **自动升级** | 连续3个项目评价4.5分以上 | 自动升级1级 |
| **自动降级** | 连续2个项目评价2.5分以下 | 自动降级1级 |
| **强制降级** | 单次严重违规 | 直接降至D级 |
| **观察期** | 降级后进入1个月观察期 | 表现良好可恢复 |

## 三、评价流程

### 3.1 评价流程

```
┌─────────────────────────────────────────────────────────────┐
│                    评价流程                                  │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  【评价触发】                                                │
│  ├── 项目结项后自动触发                                      │
│  ├── 或里程碑验收后触发                                      │
│  └── 向各方发送评价邀请                                      │
│                                                              │
│  【评价填写】                                                │
│  ├── 各方填写评价表                                          │
│  ├── 填写5个维度打分                                        │
│  ├── 撰写评价评语                                            │
│  └── 提交评价                                                │
│                                                              │
│  【评价汇总】                                                │
│  ├── 系统计算综合得分                                        │
│  ├── 计算信用分变化                                          │
│  └── 生成评价报告                                            │
│                                                              │
│  【评价公开】                                                │
│  ├── 向各方反馈评价结果                                      │
│  ├── 公开评价统计（匿名）                                    │
│  └── 更新信用台账                                            │
│                                                              │
│  【申诉处理】                                                │
│  ├── 如有异议可申诉                                          │
│  ├── 三长老审核申诉                                          │
│  └── 最终裁决                                                │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 评价时限

| 阶段 | 时限 | 逾期处理 |
|------|------|----------|
| **评价填写** | 收到邀请后7天内 | 系统自动提醒 |
| **申诉期** | 评价公开后7天内 | 可申诉 |
| **申诉处理** | 申诉后5个工作日内 | 三长老处理 |

### 3.3 评价公开规则

| 公开内容 | 公开范围 | 说明 |
|----------|----------|------|
| **综合评分** | 全社区公开 | 公开显示 |
| **各维度评分** | 项目成员可见 | 参与方可见 |
| **评价评语** | 双方互见 | 可选择匿名 |
| **信用等级** | 全社区公开 | 公开显示 |

## 四、知识入库

### 4.1 知识沉淀内容

| 知识类型 | 具体内容 | 入库格式 |
|----------|----------|----------|
| **项目案例** | 成功案例、失败案例 | 项目结项报告 |
| **经验总结** | 执行经验、教训反思 | 经验文档 |
| **解决方案** | 常见问题解决方案 | 标准答案 |
| **模板工具** | 文档模板、执行工具 | 模板库 |
| **最佳实践** | 优秀执行方案 | 案例库 |

### 4.2 项目结项报告模板

详细模板请参见 `contracts/10-项目结项报告.md`

### 4.3 知识入库流程

```
1. 项目结项 → 触发知识沉淀
2. 挂帅人提交结项报告
3. 经验总结提炼
4. 三长老审核通过
5. 知识入库存档
6. 社区共享
```

### 4.4 知识复用激励

| 激励类型 | 条件 | 奖励 |
|----------|------|------|
| **优秀案例奖** | 被选为优秀案例 | +10信用分 |
| **知识贡献奖** | 被多次引用 | +5信用分 |
| **模板贡献奖** | 贡献实用模板 | +3信用分 |

## 五、信用台账

### 5.1 信用台账内容

| 字段 | 内容 | 更新频率 |
|------|------|----------|
| 成员信息 | 姓名、联系方式 | 变更时更新 |
| 信用等级 | S/A/B/C/D级 | 实时更新 |
| 信用分 | 0-100分 | 实时更新 |
| 历史评价 | 评价记录 | 每项目更新 |
| 加分记录 | 加分项明细 | 触发时更新 |
| 扣分记录 | 扣分项明细 | 触发时更新 |

### 5.2 信用查询

| 查询方式 | 权限 | 内容 |
|----------|------|------|
| **自我查询** | 全部 | 完整信用档案 |
| **项目成员查询** | 项目相关 | 对方信用等级 |
| **公开查询** | 全社区 | 等级显示 |

## 六、快速检查清单

评价环节，确认以下事项：

```
□ 项目已结项
□ 验收已完成
□ 双方已完成评价
□ 信用分已更新
□ 结项报告已提交
□ 经验教训已总结
□ 知识已入库
```
