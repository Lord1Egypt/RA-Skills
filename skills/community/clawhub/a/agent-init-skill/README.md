# agent-init

A skill that initializes or updates **AGENTS.md / CLAUDE.md** for any project using a two-phase workflow:
1. Run the built-in `/init` command to produce raw project intelligence
2. Restructure the output into a 9-section universal template, adapting all tech-specific references to what the project actually uses

The template covers: project overview, quick commands, backend/frontend architecture, key conventions, local dev & verification flow, quality checks, reference projects, and document navigation.

## Installation

### Claude Code

**Method 1 — Marketplace install (recommended)**

```bash
claude plugins install agent-init
```

**Method 2 — Manual install**

Copy the skill folder into your Claude Code skills directory:

```bash
# Linux / macOS
cp -r agent-init ~/.claude/skills/

# Windows (PowerShell)
Copy-Item -Recurse agent-init $env:USERPROFILE\.claude\skills\
```

**Method 3 — `.skill` package install**

If you have a `.skill` file (generated by the skill-creator's `package_skill.py`):

```bash
claude plugins install ./agent-init.skill
```

### Codex (OpenAI)

Codex reads project-level instruction files. Since this skill is a Claude Code skill (not a Codex-native format), use it as a **reference template**:

1. Copy the skill's 9-section template into your Codex project's `AGENTS.md`
2. Or, run this skill inside Claude Code to generate the file, then use the output with Codex:

```bash
# In Claude Code, with agent-init installed:
# Just say: "Initialize AGENTS.md for this project"
# Claude Code will generate a complete AGENTS.md using the skill.
# The resulting file works with any AI coding tool.
```

The `AGENTS.md` generated by this skill follows a universal format that works across AI coding tools — it's tool-agnostic documentation.

### OpenCode

OpenCode supports `CLAUDE.md` and `AGENTS.md` as project-level instruction files:

```bash
# 1. Use Claude Code + agent-init to generate the file for your project
#    Just say: "Create a CLAUDE.md for this project"
#
# 2. The generated CLAUDE.md / AGENTS.md will be picked up by OpenCode automatically
```

### Generic — Any AI Coding Tool

The output of this skill is a plain Markdown file (`AGENTS.md` or `CLAUDE.md`) placed in the project root. Most AI coding tools (Cursor, Windsurf, Aider, Continue, Cline, etc.) natively read these files. You can:

1. Install the skill in Claude Code (any method above)
2. Use Claude Code to generate the `AGENTS.md` for your project
3. The file will be picked up by whatever AI coding tool you use next

## Usage

Once installed, invoke the skill inside Claude Code with natural language:

```
"Initialize AGENTS.md for this project"
"Create a CLAUDE.md for my project"
"Set up the agent guidelines for this repo"
"Generate project documentation for AI agents"
```

The skill automatically:
- Detects the actual tech stack (not hardcoded to Spring Boot + React)
- Adapts the template for single-stack, full-stack, or monorepo projects
- Preserves any existing hand-written CLAUDE.md/AGENTS.md content
- Marks unavailable sections with `TODO` instead of fabricating content
- Verifies all referenced file paths and commands

## Template Sections

| # | Section | Purpose |
|---|---------|---------|
| 1 | Project Overview | One-paragraph summary with detected tech stack |
| 2 | Quick Commands | Build, start, test, format, lint cheat sheet |
| 3 | Backend Architecture | Package tree with annotations, core subsystems |
| 4 | Frontend Architecture | Tech stack, routing, API layer, component library |
| 5 | Key Conventions | Enforced rules with doc links |
| 6 | Local Dev & Verification | Edit → Build → Start → Verify loop with curl examples |
| 7 | Quality Checks | Lint / format / build / test command reference |
| 8 | Reference Projects | Sibling repos or template projects |
| 9 | Document Navigation | Index of all existing docs |
