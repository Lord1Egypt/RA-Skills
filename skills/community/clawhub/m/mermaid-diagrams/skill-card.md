## Description: <br>
Create software diagrams using Mermaid syntax for class diagrams, sequence diagrams, flowcharts, ERDs, C4 architecture diagrams, state diagrams, git graphs, and other diagram types. <br>

This skill is ready for commercial/non-commercial use. <br>

## Publisher: <br>

### License/Terms of Use: <br>


## Use Case: <br>
Developers and technical writers use this skill to create maintainable Mermaid diagrams for software architecture, data models, workflows, and system interactions. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Installation examples include npx, npm, and manual copy commands that can modify local agent skill directories. <br>
Mitigation: Review installation commands before running them and execute only the commands appropriate for the target agent environment. <br>
Risk: Mermaid diagrams can disclose private architecture, database, workflow, or business details when rendered in hosted editors. <br>
Mitigation: Use local rendering for diagrams that contain sensitive system or business information. <br>
Risk: Incorrect or stale diagrams can mislead implementation, onboarding, or architecture decisions. <br>
Mitigation: Pair generated diagrams with prose context and update them when the underlying system changes. <br>
