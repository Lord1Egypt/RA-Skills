## Description: <br>
SeisBench Model API explains how agents can use SeisBench's model interface to annotate and classify seismic waveforms with pretrained models for phase picking, earthquake detection, denoising, and depth estimation. <br>

This skill is ready for commercial/non-commercial use. <br>

## Use Case: <br>
Developers and seismology practitioners use this skill to understand how to install SeisBench, load pretrained waveform models, and apply annotate or classify workflows to ObsPy streams. It supports seismic tasks such as phase picking, earthquake detection, waveform denoising, and depth estimation. <br>

## Known Risks and Mitigations: <br>
Risk: The workflow may install Python packages from PyPI and download pretrained model weights for local caching. <br>
Mitigation: Use a controlled Python environment and review the package and model-weight sources before running the installation or pretrained-model loading steps. <br>
Risk: Very small waveform amplitudes can create numerical instability or poor normalization behavior. <br>
Mitigation: Inspect waveform scale before inference and apply appropriate normalization or scaling when values are extremely small. <br>
Risk: Seismic phase picks, detections, denoising, and depth estimates may be model-dependent and should not be treated as definitive without review. <br>
Mitigation: Validate outputs against domain expectations, input data quality, and any operational requirements before using results in downstream decisions. <br>
