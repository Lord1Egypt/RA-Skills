# 常见问题 FAQ

## 安装与环境

### Q: 运行报错 "ModuleNotFoundError: No module named 'requests'"
**A:** 需要安装依赖：
```bash
pip install requests beautifulsoup4
```

### Q: 提示 "没有权限" 或 "Permission denied"
**A:** 给脚本添加执行权限：
```bash
chmod +x scripts/monitor.py
```

### Q: Python 版本有要求吗？
**A:** Python 3.6 及以上版本均可。

## 使用问题

### Q: 为什么有些学校查不到数据？
**A:** 可能原因：
1. **网站动态加载** — 使用 Vue/React 等前端框架的网站，内容需要通过浏览器渲染
2. **网站改版** — 学校更换了网站系统，页面结构变了
3. **访问限制** — 触发了反爬机制（已内置重试应对）

**解决方法：**
- 手动打开该校网站，确认是否正常
- 检查 schools.json 中的链接是否最新
- 尝试换个时间段再查

### Q: "请求超时"是什么意思？
**A:** 脚本连接该网站超过 20 秒无响应。通常原因：
- 学校网站暂时故障或维护中
- 网络波动
- 目标网站服务器响应慢

**建议：**
- 手动访问该网址确认状态
- 过 1-2 小时再试
- 如果持续超时，可能是网站已变更，需要更新链接

### Q: "被禁止访问(403)" 怎么解决？
**A:** 网站开启了反爬机制。脚本已尝试更换浏览器标识，但如果网站防护严格仍可能被拦截。可以尝试：
- 换一个网络环境再试
- 该网站可能只允许浏览器访问

### Q: "页面内容过少" 是什么情况？
**A:** 说明网站可能使用了 JavaScript 动态加载技术（如 Vue.js、React 等），普通 HTTP 请求只能获取到空框架。这类网站必须用浏览器才能查看完整内容。

**目前限制：** 脚本无法抓取动态页面的内容。

### Q: 一次性查太多学校会不会有问题？
**A:** 不会出大问题，但建议一次不超过 6 所，否则耗时较长（每所学校 20-60 秒）。

## 功能相关

### Q: 怎么设置定时自动监控？
**A:** 告诉 AI 你要的监控时间即可。例如：
- "设置每周三和周六下午6点自动监控"
- "每天上午9点查一次"
AI 会帮你配置 cron 定时任务，到点自动运行并发送结果。

### Q: 结果能自动保存到文件吗？
**A:** 可以。在命令后加 `--save` 参数即可保存到 `output/` 目录。也可以通过 AI 直接发送给你。

### Q: 怎么添加新的目标学校？
**A:** 两种方式：
1. **自己编辑** — 修改 `schools.json`，按格式添加学校名称和网站链接
2. **让 AI 帮忙** — 告诉 AI "帮我加一个XX大学"，AI 会帮你配置并更新

### Q: 同一个学校有多个网站（研究生院+学院）怎么加？
**A:** 在 `schools.json` 中同一个学校的 `sites` 列表里添加多个网站即可：
```json
{
  "name": "南昌大学",
  "sites": [
    {"site_name": "研究生院", "url": "https://yjsy.ncu.edu.cn/..."},
    {"site_name": "数学与计算机学院", "url": "https://smcs.ncu.edu.cn/..."}
  ]
}
```

### Q: 能监控考研调剂信息吗？
**A:** 可以。脚本会自动扫描页面上包含"招生""调剂""复试"等关键词的通知。只要目标学校的调剂通知发布在公开页面上，就能抓取到。

## 错误码速查

| 错误信息 | 含义 | 解决方案 |
|----------|------|---------|
| 请求超时 | 网站响应超过20秒 | 检查网络，稍后重试 |
| 被禁止访问(403) | 触发了反爬机制 | 更换网络环境 |
| 页面不存在(404) | 网址已失效 | 更新 schools.json 链接 |
| 服务不可用(503) | 服务器临时故障 | 稍后重试 |
| 网络连接失败 | 域名无法解析或网络不通 | 检查网络连接 |
| 页面内容过少 | 动态页面需浏览器渲染 | 手动访问查看 |
| 未找到相关通知 | 页面已正常但无匹配内容 | 可能网站改版 |
| 解析失败 | HTML结构异常 | 可能网站改版 |

---

## 注意事项

1. 脚本只会抓取**公开页面**的内容，无法获取需要登录的页面
2. 通知发布日期是脚本**从页面文本提取**的，部分网站可能没有显示日期
3. 如果学校网站使用了 PDF 附件发布通知，标题中可能不包含关键词
4. 不同学校的网站更新频率不同，建议定期关注（每周2-3次）
