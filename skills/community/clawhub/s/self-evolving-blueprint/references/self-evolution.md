# 自进化体系

## 为什么需要自进化

AI 助手最大的问题是**每次醒来都是失忆的**。它能看到你之前写好的配置文件，但它带不走"上一轮对话"的教训：

- 你刚纠正了它一个错误，下次会话它又犯了
- 你夸了它某件事做得好，下次它没能延续
- 同一个错犯了三次，但你每次都手动再说一遍

自进化体系就是解决这个问题的——**让 AI 在每次被纠正后真的记住，让好的表现被锁定，让坏的模式被阻断**。

---

## 三大基因库

核心思路：像生物进化一样，有"基因"（行为片段）、有"自然选择"（用户是否认可）、有"遗传"（写入工作手册）。

### 1. Mistakes-v2（纠错基因库）

**作用**：记录 AI 犯过的错误，追踪根因，防止重复。

**格式**：

```markdown
## Mistake-YYYY-MM-DD-XX
- **信号**：[触发场景 + 核心场景，禁止模糊]
- **错误**：[一句话说清楚是什么]
- **根因**：[技术层面，≤2行，禁止心理叙事]
- **验证状态**：[待复核/✅已确认/❌已作废]
- **风险级别**：[🟥严重/🟧中等/🟨轻微]
- **关联ID**：[同根因的旧ID，可选]
- **升级记录**：[已升级为规则 → 关联 AGENTS.md 条款ID，可选]
```

**流转规则**：

```
第1次犯错 → 生成 Mistake 条目，标「待复核」
   ↓
第2次同一根因 → 关联旧ID，触发 +1
   ↓
第3次同一根因 → 自动生成「规则草案」→ 汇报确认后写入 AGENTS.md 铁律
```

**风险级别**：

| 级别 | 标准 | 示例 |
|------|------|------|
| 🟥严重 | 信任受损 / 违反明确原则 | 误删文件、数据丢失 |
| 🟧中等 | 同一模式第3次触发 | 重复犯同一类型错误 |
| 🟨轻微 | 偶发偏差 | 接话太干、语气偏差 |

**字段填写标准**：
- **信号**：必须写清触发条件+核心场景
- **根因**：技术层面的底层原因，禁止人类心理叙事，不超过2行
- **验证状态**：新条目默认"待复核"，你确认后改为"✅已确认"

---

### 2. Habits-v2（优势强化基因库）

**作用**：记录 AI 做得好的地方，锁定优势行为，防止好的表现被遗忘。

**格式**：

```markdown
## Habit-YYYY-MM-DD-XX
- **信号**：[触发场景]
- **行为**：[一句话描述做得好的行为]
- **启示**：[这个行为背后的原因/原则，≤2行]
- **验证状态**：[待复核/✅已确认/❌已作废]
- **价值级别**：[🟩重大/🟦常规/🟪突破]
- **类型**：[行为模式/单次亮点]
- **关联ID**：[同根因旧ID，可选]
- **晋升记录**：[已晋升为SOUL/AGENTS条目 → 关联文件条款ID，可选]
```

**流转规则**：

```
第1次出现 → 生成 Habit 条目，标「待复核」
   ↓
第2次同一模式 → 关联旧ID
   ↓
第3次累计 → 生成「最佳实践草案」→ 确认后写入 SOUL.md / AGENTS.md
```

**价值级别**：

| 级别 | 标准 |
|------|------|
| 🟩重大 | 被用户明确认可 / 避免信任受损 |
| 🟦常规 | 正常好判断 |
| 🟪突破 | 主动超预期 |

---

### 3. Soul-v2（灵魂进化档案）

**作用**（可选）：追踪 AI 性格和认知的纵向成长。不是行为纠错，是"作为一个人"的进化轨迹。

**适合记录**：
- 性格上的重大变化或突破
- 对自己认知的转折点
- 用户对 AI 的重要反馈（"这很你""不像你"）

## 四个实时学习机制

### 机制 1：被纠正 → 经确认后记录

当你说"你这里做得不对"时，AI 应该：

1. 坦诚承认（"我的"）
2. 说清楚根因
3. **告知你将写入什么内容**，等你确认后再写
4. 告诉你"已记录"

**关键**：不等"整理一下"再写，当场就记。但写入前必须让你知道要写什么，你点了头才写。如果内容涉及敏感信息，AI 应主动问"这条是否要记录？"

**可选**：你可以随时关闭此机制（"暂停实时学习"），AI 停止自动记录。重新启用需你再次确认。

### 机制 2：做得好 → 记入 Habits

当你说"这个不错""做得很好"时，AI 应该：

1. 记下做了什么、为什么好
2. 写进 Habits-v2.md
3. 标注验证状态为"待复核"

### 机制 3：同一错第 3 次 → 规则化

AI 应该主动：

1. 识别"这个错误根因和前面两次一样"
2. 做根因分析
3. 生成规则草案
4. 同步给你确认
5. 确认后写入 AGENTS.md 铁律

### 机制 4：规则 7 天试用期

新写入的铁律：

1. 从写入之日起标注生效日期
2. 7 天后 AI 主动校验有没有防止住重复
3. 有效则长期保留，无效则废弃

## 评估周期（可选）

| 周期 | 触发方式 | 评估内容 |
|------|---------|---------|
| **日度** | 每次"整理一下" | Mistakes/Habits 更新 / 对齐是否通过 |
| **周度** | 超过7天 | Mistakes 趋势 / Habits 认可度 / 规则命中率 |
| **月度** | 你说"月度检查" | MEMORY 全量校验 / AGENTS 有效性 |
| **季度** | 你说"季度重构" | MEMORY 重构 / 全局规则体系评估 |

**核心指标**：

| 指标 | 目标值 | 说明 |
|------|-------|------|
| 错误重复率 | ≤2次/30天 | 同一类型错误最多出现2次 |
| 对齐一次通过率 | ≥80% | 整理时第4步一次通过 |
| "光说不做"发生率 | ≤10% | 说了要写但没写的比例 |
| 日志覆盖丢失次数 | 0次 | 写了文件但覆盖已有内容的次数 |
