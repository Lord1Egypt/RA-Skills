# 复杂度分级与路由

---

## 分级标准

| 等级 | 触发条件 | 双引擎配置 |
|------|---------|-----------|
| **简单** | 具体事实查询（行业规模/某公司数据） | 直接回答，不走双引擎 |
| **中等** | 问题明确但需多维度分析（竞对分析+市场评估） | 引擎A精简（2个假说）+ 引擎B定向验证 |
| **复杂** | 模糊且多维度（战略转型/行业剧变） | 完整双轨+交叉对抗+红队攻击 |
| **S级** | 高度不确定性+多利益相关方 | 3假说并行+情景规划+双审计 |

---

## 各路径详细配置

### 简单路径

- 不触发任何引擎
- 直接搜索回答
- 不生成报告

### 中等路径

```
Phase 0 战略意图 → Phase 1 坏战略检查 → Phase 2 假说生成（2个）
→ Phase 3 假说筛选 → Phase 4 洞察生成（简化：只做类比+反常信号）
→ Phase 5 定向验证（只验证P0子命题）→ Phase 6 前提假设检验
→ Phase 7 诊断+指导方针 → Phase 8 可行性检查（精简）
→ Phase 9 连贯性行动（3个）→ Phase 10 红队攻击（跳过宪法审计）
→ 输出（3000-5000字）
```

### 复杂路径

```
Phase 0-10 完整走完
→ 双审计并行（红队+宪法）
→ 输出（5000-8000字）
```

### S级路径

```
Phase 0-2（3个互斥假说）→ Phase 3 筛选 → Phase 4 洞察生成（完整四机制）
→ Phase 5 验证+假说对抗辩论 → Phase 6 前提假设检验（加倍严格：用户假设必须有数据）
→ Phase 7-10 完整流程 + 情景规划 + 置信区间标注
→ 输出（8000-12000字）
```

## 维度裁剪

用户可以通过命令指定维度：

| 命令 | 启用引擎 | 验证框架 |
|------|---------|---------|
| `/见微 竞争` | A+B | 五力+战略群组+Revenue Bridge |
| `/见微 市场` | A+B | 五看三定+TAM-SAM-SOM |
| `/见微 组织` | A+B | VRIO+价值链+BLM一致性 |
| `/见微 战略` | A+B | 安索夫+情景规划+BLM业务设计 |

单维度模式下：
- 假说只从对应维度生成
- 跳过宪法审计
- 报告按目标维度输出，不需要五维度完整结构
- 可组合：`/见微 竞争 市场`

## 隐式触发

用户未用 `/见微` 但表达了战略决策意图时：

> "这个问题涉及多个维度的战略判断。需要我用见微模式走一遍系统分析吗？回复'是'或直接 /见微 即可。"

以下情况**不触发**：
- 行业数据查询
- 公司信息查询
- 简单概念解释
- 泛泛讨论
