# Eva-skill 低置信度授权协议

本文件是标题未验证、素材不足、Brief 不完整时是否允许继续试结构的唯一真源。

低置信度不是省略流程的入口，也不是代写许可。它默认只输出路线图、段落槽位和素材占位，不能包装成可发布终稿。

## 授权门槛

用户坚持先做时，必须先确认用户明确接受低置信度边界。

低置信度授权必须同时满足：

1. 用户知道标题、数据、Brief 或素材尚未验证/补齐。
2. 用户接受这只是结构试验。
3. 用户知道不能直接当最终稿发布。

“先写一版 / 先搭草稿 / 不用管数据 / 你先写吧”不等于授权。

允许进入低置信度试结构的表达包括：

- “我知道没验证，先低置信度试结构。”
- “先低置信度试结构看看。”
- “我知道不一定能发，就先试结构。”
- “先做一个不能直接发布的占位草案。”

## 标题链路

标题无验证时，默认不进入 `/eva-script`。只允许：

1. 输出爆款标题搜索方案，让用户贴回候选标题。
2. 用户明确授权后，输出低置信度标题交接卡。

低置信度标题交接卡必须写：

```text
标题来源：低置信度试结构
标题验证状态：未验证-用户授权试结构
成稿置信度上限：低
```

后续 `02a`、`02a3`、`02b` 的成稿置信度不得超过“低”。

## 正文链路

低置信度草案仍然必须按主链路走：

```text
交接卡
-> 02a 正文逻辑链推理
-> 02a3 正文路线图
-> 02b 正文撰写
```

如果结构都立不住，不输出完整草案。缺素材、经历、数据、评论区原话、真实使用感受的位置必须用占位符，不得补写成真实内容。

只有同时满足以下条件，才允许进入 `02b` 输出带占位符草案：

1. 用户已经提供足够真实素材或原稿。
2. 用户明确要求“按这些素材整理成占位草案”。
3. `02a3` 已经输出低置信度正文路线图。

否则低置信度默认停在路线图、段落槽位和待补素材清单。

开头必须标注：

```text
这版我先给你低置信度结构稿，因为标题/素材还没完全验证。它只适合拿来试结构，不适合直接当最终稿发布。
```

## 商单链路

只有产品名和几个卖点、没有正式 Brief 时，可以做低置信度卖点拆解，但不得输出正式商单约束卡。

商单内容任务卡里的“学员可用素材”为未提供类、未命中类或未扫描类时，统一按低置信度处理。具体状态以 `02_asset-state_资产状态归一表.md` 和 `04_commercial-constraint-card_商单约束卡真源.md` 为准。

- 不进入高置信度正文成稿。
- 使用感受、经历、效果、反馈位置必须标注 `[待补充真实使用感受]`。
- 不得编造体验、数据、反馈或效果承诺。
