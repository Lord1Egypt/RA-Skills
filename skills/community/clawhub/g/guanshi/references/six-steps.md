# 六步洞察法详细规则

> 本文件包含观势六步洞察法（含 Step 5.5 宪法审计 + Step 6.5 逻辑自检）的详细规则。Chief 在 Step 2-6 阶段只加载问题树中标记为"必须"或"部分"的框架。

## 总体逻辑：BLM 主线驱动 → 五看三定洞察 → 五维度固定输出

```
时间轴：  过去                      现在                       未来

        BLM预诊断    ──→   五看三定   ──→   差距分析   ──→   BLM重构
        (Step 1A)         (Step 2)        (Step 2末尾)      (Step 5-6)

        回顾过去战略-      看行业→市场     业绩差距：现状       新战略意图
        组织-能力一致性     →客户→竞争     vs 目标             新业务设计
        (框架性假设，       →自己          机会差距：现有       关键任务→组织
        低置信度，随       定控制点→目标   能力 vs 新需求       →人才→文化
        资料迭代更新)      →策略                                一致性验证(6.5)
             │                                                      │
             └────────────────── BLM 脊梁 ──────────────────────────┘
```


**核心原则**：分析过程的严谨性留给自己，输出结构的可读性留给客户。MECE 问题树决定分析深度（哪些框架必须/部分/跳过），但最终报告按 PART 1-4 叙事结构输出，以"成功经验 → 核心矛盾 → 解决方案 → 实施路线"为主线层层递进。分析框架的产出物（五看三定、VRIO、7S、ADKAR、OHI、安索夫等）按叙事线整合进对应 PART，而非按框架维度平铺。BLM 模型作为贯穿主线，在 PART 1 战略维度和 PART 3 重构后追加一致性校验。

**框架覆盖率检查（Step 1 判定表最终化前强制执行）**：在框架判定表完成后，必须逐类别检查——核心分析类别（市场洞察/竞争定位/内部诊断/战略选项/情景规划）之下，至少有一个核心框架被标记为"必须"。若某类别的所有框架均为"部分"或"跳过"，须将该类别最相关的一个框架从"部分"提升为"必须"。检查完成后判定表方可锁定。

**每步输出要求**：标题即结论 + 数据源标注 + "但"字转折衔接下一步。

## Step 1：问题界定与 MECE 解构

**目标**：把模糊的商业焦虑翻译成结构化的战略命题，并产出问题树驱动后续框架选择。

**金字塔原理应用**：从顶层问题出发，逐层分解为 MECE（互斥且穷尽）的子问题。每个子问题映射到对应的分析框架。这是整个诊断的**议程设置**——决定了报告分析什么、不分析什么。

**执行**：

1. **界定核心问题**：用一句话定义战略问题（不是现象，是问题本身）
2. **Cynefin 域判断**：Simple / Complicated / **Complex** / Chaotic
3. **明确分析边界**：时间范围 / 地理范围 / 业务范围
4. **MECE 问题树**：将核心问题分解为 3-5 个互斥且穷尽的子问题。推荐分解维度：
   - 分支 1：市场/外部/机会（WHERE — 增量在哪）
   - 分支 2：竞争/定位/威胁（WHO — 对手在做什么）
   - 分支 3：组织/能力/路径（HOW — 我们能否抓住）
   - 分支 4：时间/节奏/风险（WHEN — 窗口期和风险）
5. **框架选择映射表**：每个子问题 → 对应该加载哪些框架，标注（必须 / 部分 / 跳过）。是必需的必须写"必须"，有更优但非必需的写"部分"，不需要的写"跳过"并写一句理由。

**输出格式**（标题即结论）：

```
A. 战略问题界定
   > **战略问题**：[一句结论性定义]
   > **分析边界**：时间 | 地理 | 业务

B. MECE 问题树（金字塔结构）
   核心问题：[一句顶层问题]
   ├─ 分支 1：[子问题，MECE 维度一]
   │   ├─ 关键未知 1.1：[具体需要回答什么]
   │   ├─ 关键未知 1.2：[具体需要回答什么]
   │   └─ 对应框架：框架X（必须）/ 框架Y（部分）
   ├─ 分支 2：[子问题，MECE 维度二]
   │   ├─ 关键未知 2.1：[具体需要回答什么]
   │   └─ 对应框架：框架Z（必须）
   ├─ 分支 3：[子问题，MECE 维度三]
   │   ├─ 关键未知 3.1：[具体需要回答什么]
   │   └─ 对应框架：框架A（必须）/ 框架B（部分）
   └─ 分支 4（可选）：[子问题，MECE 维度四]
       └─ 对应框架：情景规划（必须）

C. 框架适用性判定表
   | 框架 | 适用？ | 理由 |
   |------|:---:|------|
   | PESTEL（完整） | 部分/跳过 | [哪些因子和核心问题直接相关] |
   | 波特五力 | 必须/部分/跳过 | [为什么] |
   | 安索夫矩阵 | 必须/部分/跳过 | [为什么] |
   | BCG 矩阵 | 必须/部分/跳过 | [为什么] |
   | VRIO | 必须/部分/跳过 | [为什么] |
   | 7S | 必须/部分/跳过 | [为什么] |
   | ADKAR | 必须/部分/跳过 | [为什么] |
   | OHI | 必须/部分/跳过 | [为什么] |
   | 情景规划 | 必须/部分/跳过 | [为什么] |
   | 宪法审计 | 必须 | 硬门控，不可跳过 |
```

**门控**：问题界定不清晰 → 必须追问，不得跳过。

**禁止**：问题树写完后，后续步骤绕过映射表，全量加载所有框架。

## Step 1A：BLM 预诊断（过去战略-组织一致性回顾）

**目标**：企业在"战略执行—业务设计—关键任务—正式组织—人才—氛围文化"六个维度上的历史一致性水平，建立初步框架性假设。作为后续五看三定洞察和差距分析的"过去参考基线"。

**定位**：BLM 预诊断是框架性假设，基于当前已有信息（公开资料+行业常识+用户可能提供的内容数据）。置信度标注为低或中，随后续资料迭代更新。不走苏格拉底门控——信息不足不阻塞，标注"框架性推断，待验证"即可。

**预诊断输出格式**：

```
BLM 预诊断（过去模式一致性假设 — 低/中置信度，随资料迭代更新）

A. 过去战略意图回顾

   > 过去战略一致性自评：高 / 中 / 低
   > 关键证据：[2-3条公开/已知信息]

B. 过去业务设计回顾
   | 要素 | 过去状态 | 证据 | 置信度 |
   |------|---------|------|:---:|
   | 客户选择 | [过去的客户选择] | | |
   | 价值主张 | [过去的定位] | | |
   | 盈利模式 | [过去的商业模式] | | |
   | 战略控制点 | [过去的差异化来源] | | |
   | 活动范围 | [价值链覆盖] | | |

C. 过去组织-能力一致性评估（BLM 执行要素）
   | 要素 | 过去状态 | 与战略意图的对齐度 | 证据 | 置信度 |
   |------|---------|:---:|------|:---:|
   | 关键任务 | [过去核心任务] | ✓/△/✗ | | |
```

**置信度迭代规则**：
- 初始标注为"低"或"中"，标清哪些是公开信息、哪些是推断
- 五看三定完成后用市场数据回检——如"看自己"发现过往毛利率趋势验证了预诊断的效率假设，升级置信度
- Step 4 内部诊断完成后用 VRIO/7S 数据再次回检——如组织诊断发现架构与战略的错位与预诊断一致，升级置信度
- 最终在 BLM 重构时，将预诊断中已验证的部分合并到新战略设计的"继承/抛弃"清单

## Step 1.5：信息审计（苏格拉底门控）

**前置条件**：Step 2 不可跳过。

**三维自检**：
1. **事实数据**：有行业规模/增长率/市场份额/财务数据吗？
2. **多视角**：有客户/竞争对手/内部至少两方信息吗？
3. **时间纵深**：了解过去 3-5 年的关键变化节点吗？

**任一维度完全缺失 → STOP**：
1. 一句话告知局限性
2. 输出 3 个精准问题（分别覆盖事实/视角/背景缺口），标注"问这个是为了什么"
3. **禁止输出任何分析/建议**

## Step 2：五看三定（市场洞察核心方法）

**目标**：借用华为五看三定方法，系统扫描外部环境并形成内部锚定，识别机会与威胁，产出战略控制点、目标和策略方向。五看三定替代原 Step 2-3 的环境扫描和市场分析，成为市场洞察的统一入口。

**五看**：

```
一看行业（宏观趋势）→ 二看市场（客户需求）→ 三看客户（购买行为）
    → 四看竞争（对手动态）→ 五看自己（内部审视）
```

**三定**：

```
定战略控制点（护城河在哪）→ 定目标（打哪里）→ 定策略（怎么打）
```

**框架加载规则**：五看三定是默认骨架，但框架适用性判定表中应标注哪些传统框架被五看覆盖：
- 一看行业 → 替代 PESTEL（PESTEL 标记为"部分"或"跳过"，但核心因子渗入一看输出）
- 四看竞争 → 为 Step 3 竞争定位提供输入，但不是 Step 3 本身
- 五看自己 → 为 Step 4 内部诊断提供输入，但不是 Step 4 本身
- TAM-SAM-SOM → 嵌入二看市场（市场量化）

**PESTEL 降级规则**：五看三定激活后，PESTEL 退化为"渗入模式"——不再输出完整 PESTEL 表。一看行业时，若某 PESTEL 因子与核心战略问题高度相关（框架判定表标注为"必须"），则在一看中展开该因子；否则只在一看中一句带过。

**输出格式**：

```

   行业增长驱动力：[2-3个核心驱动因素]
```

## Step 3：竞争定位（竞对深度画像补充）

**目标**：在五看三定"四看竞争"的基础上，对核心竞对做深度画像和心理博弈推演。

**定位**：四看竞争已覆盖竞争格局、对手近期动作和战略意图。Step 3 不重复四看内容，而是做竞对画像的"深度补充"——竞对惯性约束、决策心理、反应模式、博弈推演。

**框架加载规则**：查阅 Step 1 产出的框架适用性判定表——仅加载竞对画像、价值曲线等标记为"必须"或"部分"的框架。

**输出格式**：

**A. 竞争格局总览**（标题即结论）

## Step 5.5：宪法审计（内部硬门控）

**前置条件**：Step 5 完成。Step 6 不可跳过此步。

**目标**：在综合建议之前，对 Step 5 产出的战略选项进行系统性的逻辑与可行性审计。此为 **Chief 内部质检流程**，审计结果不写入最终报告——客户端报告的五维度结构和情景规划弹性测试已经内化了逻辑验证，不需要额外展示"我们自查了"。

**审计维度**（七维检查）：

| 维度 | 检查问题 | 判定 |
|------|---------|:---:|
| **逻辑一致性** | 选项推导链条有无断裂？安索夫四象限的选项生成是否合理？ | PASS / FAIL |
| **可执行性** | 选项要求的能力与 Step 4 诊断的能力缺口是否矛盾？ | PASS / FAIL |
| **成本与风险** | 每个选项的全周期成本是否被估算？风险是否在容忍度内？ | PASS / FAIL |
| **战略匹配度** | 选项是否与 Step 1 定义的核心问题对齐？有无跑题？ | PASS / FAIL |
| **数据支撑** | 关键假设是否有 Step 2-4 的数据支撑？标注置信度 | PASS / FAIL |
| **二阶效应** | 如果执行该选项，会触发竞争对手什么反应？会产生什么意料之外的副作用？ | PASS / FAIL |
| **MECE 覆盖** | 回到 Step 1 的问题树——每个关键未知都被至少一个选项覆盖了吗？ | PASS / FAIL |

**审计结果**：
- 任一维度 FAIL → 回到对应步骤修正，最多迭代 2 次
- 全部 PASS → 进入 Step 6

**输出格式**（内部质检，不写入报告）：
```
宪法审计结果（内部）
├─ 七维判定：逻辑 ✓ / 执行 ✓ / 成本 ✓ / 匹配 ✓ / 数据 ✓ / 二阶 ✓ / MECE ✓
├─ 修正记录（如有）：[修正了什么，为什么]
└─ 审计结论：[通过 / 部分通过 / 不通过]
```
