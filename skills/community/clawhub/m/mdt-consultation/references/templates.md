# MDT 会诊模板

## 合同审查会诊 (contract-review)

**适用场景：** 重要合同签署前、金额超过50万、涉及多方权益

### 会诊科室
| 角色 | Agent | 审查重点 |
|------|-------|----------|
| 🛡️ 法律审核 | shield | 合同结构、条款完整性、风险点 |
| ⚖️ 条文对照 | law | 法律条文引用、合规性、法规匹配 |
| 😈 风险找茬 | devil | 常见陷阱、隐性风险、对方优势条款 |

### 并行任务
- shield: 逐条审核合同文本，标注风险等级(🔴高🟡中🟢低)
- law: 对照相关法律法规，检查合规性
- devil: 站在对方角度，找出合同漏洞

### 综合输出格式
```markdown
# 📋 合同MDT审查报告

## 基本信息
- 合同名称：
- 涉及金额：
- 对方单位：

## 风险总评：[高/中/低]

## 🔴 高风险项（必须修改）
1. ...
2. ...

## 🟡 中风险项（建议修改）
1. ...

## 🟢 合规确认
1. ...

## 💡 谈判建议
1. ...

## 会诊签名
- 法律审核(Shield)：...
- 条文对照(Law)：...
- 风险找茬(Devil)：...
```

## 重要公文明代会诊 (document-collaborative)

**适用场景：** 重要公文、领导讲话稿、向上级汇报材料、对外发布文稿

### 会诊科室
| 角色 | Agent | 审查重点 |
|------|-------|----------|
| ✍️ 初稿撰写 | content | 结构、逻辑、格式规范 |
| 📜 文学润色 | sage | 文风、遣词造句、修辞 |
| 😈 挑刺审核 | devil | 逻辑漏洞、表述歧义、敏感点 |
| ⚖️ 合规审核 | shield | 政治性、规范性、保密审查 |

### 并行任务
- content: 结构优化+内容充实
- sage: 文风润色+修辞提升
- devil: 逻辑审查+表述挑刺
- shield: 合规性+政治性检查

### 综合输出格式
```markdown
# 📝 公文MDT会诊报告

## 基本信息
- 文稿标题：
- 文种：
- 紧急程度：

## ✍️ 撰写意见(Content)
- 结构建议：...
- 内容补充：...

## 📜 润色建议(Sage)
- 文风调整：...
- 词句优化：...

## 😈 挑刺意见(Devil)
- 逻辑问题：...
- 表述风险：...

## ⚖️ 合规意见(Shield)
- 政治性检查：...
- 格式规范：...

## 📊 综合评估
- 修改优先级：...
- 终稿建议：...
```

## 品牌宣传MDT会诊 (brand-campaign)

**适用场景：** 品牌宣传活动、多平台内容发布、重要节点宣传

### 会诊科室
| 角色 | Agent | 职责 |
|------|-------|------|
| 🎬 统筹策划 | director | 整体方案、时间线、资源协调 |
| 📱 小红书 | writer-xhs | 小红书文案+选题 |
| 📰 公众号 | writer-mp | 公众号长文+深度内容 |
| 🎨 视觉设计 | designer | 封面、海报、视觉风格 |
| 📊 数据分析 | analyzer | 效果预测、投放建议、KPI设定 |

### 并行任务
- director: 制定整体宣传方案和时间节点
- writer-xhs: 小红书3篇文案（标题+正文+标签）
- writer-mp: 公众号1篇深度文章
- designer: 视觉方案（风格定义+封面建议）
- analyzer: 数据预测和效果评估方案

## 战略决策MDT会诊 (strategic-decision)

**适用场景：** 重大决策、新项目立项、资源分配、风险评估

### 会诊科室
| 角色 | Agent | 职责 |
|------|-------|------|
| 🧠 深度分析 | brain | 多维度分析、利弊权衡 |
| 😈 反向思考 | devil | 最坏情况推演、盲点发现 |
| 📊 数据支撑 | analyzer | 数据分析、趋势研判 |
| ⚖️ 合规评估 | law | 法规政策、风险边界 |

### 并行任务
- brain: 系统性分析框架（SWOT/决策树/成本效益）
- devil: 反向推演最坏N种情况
- analyzer: 相关数据和趋势支撑
- law: 政策法规约束和风险边界

### 综合输出格式
```markdown
# 🎯 决策MDT分析报告

## 决策事项
- 背景：
- 选项：

## 🧠 分析框架(Brain)
- SWOT分析：...
- 成本效益：...

## 😈 风险推演(Devil)
- 最坏情况Top3：...
- 盲点提醒：...

## 📊 数据支撑(Analyzer)
- 关键数据：...
- 趋势研判：...

## ⚖️ 合规评估(Law)
- 政策约束：...
- 风险边界：...

## 🎯 综合建议
- 推荐方案：...
- 行动步骤：...
- 关键节点：...
```

## 技术方案MDT会诊 (technical-review)

**适用场景：** 技术选型、架构设计、代码审查、安全评估

### 会诊科室
| 角色 | Agent | 职责 |
|------|-------|------|
| 🧠 技术分析 | brain | 架构设计、技术选型分析 |
| 💻 开发实现 | game-dev / claude-code | 可行性评估、开发量估算 |
| 🔒 安全评估 | tester-security | 安全风险、漏洞分析 |
| ⚡ 性能评估 | tester-perf | 性能瓶颈、扩展性 |

### 并行任务
- brain: 技术方案多维度分析
- game-dev/claude-code: 开发可行性+工时估算
- tester-security: 安全风险清单
- tester-perf: 性能基线+扩展性评估
