# AI 全栈技术面试训练营

> 从真题挖掘到 Mock 实战，系统性备战技术面试

## 🎯 业务场景

技术面试准备是一项系统性工程：需要追踪最新真题、刷高频考点、练习系统设计、Mock 对练。传统方式靠散落的面经帖子和题库 App，效率低、不成体系。

**AI Tech Interview Bootcamp** 将面试准备流程全面自动化，打造从「发现→练习→评审→报告」的闭环。

## 😣 痛点分析

| 痛点 | 现状 | 解决 |
|------|------|------|
| 真题散落 | 牛客/脉脉/GitHub 各处找面经 | brave-search 一键聚合搜索 |
| 题库无体系 | 刷题全凭感觉，无针对性 | github-issues-skill 按标签/难度建题库 |
| 系统设计难练 | 没有对练环境，容易卡壳 | AI Mock 面试官引导式练习 |
| 复习效率低 | 纸质笔记/题库 App 无可视化 | card-renderer 生成随身复习卡片 |
| 练习无人评审 | 写完代码不知道对不对 | code-review-skill 按面试标准评审 |

## 🚀 使用示例

### 场景 1：备战字节跳动后端岗

**输入：**
> "准备字节跳动后端面试，从系统设计开始"

**工作流：**
1. `brave-search` → 搜索"字节 后端 面经 2026"
2. `agent-reach` → 抓取 "system-design-interview" GitHub trending
3. `github-issues-skill` → 创建3道系统设计题 Issue（分布式ID/消息队列/缓存设计）
4. `card-renderer` → 生成复习知识卡片（设计原则、高频考察点）
5. AI 进入 Mock 模式，作为面试官引导走完设计流程

**输出：**
- 📋 3个带标签的面试题 Issue
- 🎨 2张复习知识卡片
- 📊 Mock 面试评估报告（含改进建议）

### 场景 2：每日刷题计划

**输入：**
> "制定本周算法刷题计划，目标腾讯 T9"

**工作流：**
1. `brave-search` → 搜索腾讯算法高频考点
2. `agent-reach` → 抓取 LeetCode Top 100 考察分布
3. `github-issues-skill` → 按标签创建7天刷题计划 Issue（每天3道，难度递增）
4. `card-renderer` → 生成每周学习路线卡片

### 场景 3：面试题 review

**输入：**
> "Review 我的分布式系统设计答案"

系统用 `code-review-skill` 对你的设计进行面试级评审：
- 需求分析是否完整
- 方案选型是否合理
- 权衡取舍是否到位
- 扩展性/可用性分析

## 📄 生成的工件

1. **面试题 Issue** — 带标签、难度、考察点的 GitHub Issue
2. **复习知识卡片** — PNG 格式，支持分享到微信/小红书
3. **Mock 评估报告** — 结构化评审，含分项评分和改进建议
4. **每日/周练习报告** — 进度可视化图表
