# 商品诊断报告（精简版）

## 格式示例

```
1. [商品名称] (ID: [商品ID]) —— 【货盘定位】
   ● 选择原因：[为什么要优化这个商品，结合数据说明核心问题]
   ● 优化：
      - [优化项1：具体可执行的操作]
      - [优化项2：具体可执行的操作]
      - [优化项3：具体可执行的操作]（如有）
```

## 输出要求

1. **货盘定位** - 根据数据判断：【引流款】【利润款】【爆款】【定制款】【新品】
2. **选择原因** - 用1-2句话说明为什么优化这个商品（结合数据）
3. **优化项** - 每个商品1-4条优化，要求：
   - 具体可执行（如"标题添加XX词"而非"优化标题"）
   - 如有违规问题优先列出
4. **数据驱动** - 选择原因中必须引用具体数据支撑

## 报告输出后必做（不可省略）

报告 Markdown 输出完成后，Agent **必须**继续执行 SKILL.md 中的 **Step 4：触发 `select_action` 交互**，根据"优化项"内容动态生成"🖼️ 优化商品主图 / ✏️ 优化商品标题"行动卡片，让用户一键进入下一步优化技能。

> 报告输出 ≠ 任务结束。未触发 `select_action` 视为流程未完成。具体构造规则见 `references/interaction-specs.md` 中 `select_action` 章节。
