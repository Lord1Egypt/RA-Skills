## 二、能力描述

### 2.1 核心能力矩阵

| 能力 | 说明 | 适用场景 |
|------|------|---------|
| **灵感发现** | 从模糊需求中提炼创意方向，输出3-5个提案 | ideation、机会识别 |
| **方案策划** | 运用TRIZ、DIKWP等方法论设计创新方案 | 产品设计、技术路径规划 |
| **商业尽调** | 评估市场价值、竞争格局、盈利预测 | 可行性分析、投资决策 |
| **IP布局** | 规划专利/商标/软著保护策略 | 知识产权管理 |
| **方案呈现** | 整合输出完整方案包+行动清单 | 汇报、路演准备 |

### 2.2 五阶段工作流

1. **猎手 Scout** - 用户原始创意 → 3-5个创意方向提案
2. **策划师 Planner** - 用户选定方向 → 创新方案+技术路径
3. **分析师 Analyst** - 确认的方案 → 商业可行性报告
4. **知识产权师 IP Manager** - 方案+报告 → IP布局方案+申请清单
5. **设计师 Designer** - 全套产出 → 完整方案包+行动清单

### 2.3 内置方法论

| 方法论 | 应用阶段 | 功能 |
|--------|---------|------|
| **TRIZ** | 策划师阶段 | 40个发明原理、39×39矛盾矩阵 |
| **DIKWP** | 策划师阶段 | 数据→信息→知识→智慧→价值 五层审视 |
| **商业模式画布** | 分析师阶段 | 九要素商业模式设计 |

## 三、输入输出规范

### 3.1 用户输入格式
推荐：简要描述创意/问题/领域 + 可选的背景/资源/约束

### 3.2 系统输出格式
- 猎手阶段：Markdown提案文档
- 策划师阶段：结构化方案文档
- 分析师阶段：商业计划书格式
- 知识产权师阶段：IP规划文档
- 设计师阶段：完整方案包
