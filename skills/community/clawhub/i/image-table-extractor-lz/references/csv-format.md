# CSV 格式规范

## 概览
CSV（Comma-Separated Values，逗号分隔值）是一种简单的表格数据交换格式，广泛用于数据导入导出。

## 基本规则

### 1. 字段分隔
- 使用逗号（`,`）分隔字段
- 每行一条记录
- 第一行通常为表头

### 2. 字段引用
当字段包含以下字符时，必须用双引号（`"`）包裹：
- 逗号（`,`）
- 双引号（`"`）
- 换行符

### 3. 引号转义
双引号字段内的双引号需转义为两个双引号：
```
原始数据：He said "Hello"
CSV格式："He said ""Hello"""
```

## 示例

### 简单表格
```csv
姓名,年龄,城市
张三,25,北京
李四,30,上海
王五,28,广州
```

### 包含特殊字符
```csv
产品名称,价格,描述
"iPhone 15 Pro",7999,"最新款智能手机,配备A17芯片"
"MacBook Pro",14999,"14英寸笔记本,性能强劲"
"AirPods Pro",1899,"主动降噪,沉浸体验"
```

### 包含引号
```csv
书籍,作者,评价
"《三体》","刘慈欣","被誉为""中国科幻里程碑"""
"《百年孤独》","马尔克斯","魔幻现实主义经典"
```

## 验证规则

### 必须满足
1. 每行字段数量一致（与表头一致）
2. 引号成对出现
3. 使用 UTF-8 编码
4. 换行符统一（建议使用 `\n`）

### 常见错误
- 字段内逗号未加引号
- 引号不成对
- 字段数量不一致
- 使用了错误的编码

## 数据提取注意事项

### 数据完整性
- 保持原始数据不变
- 不修改数字格式
- 不移除有效数据

### 过滤规则
- 仅移除明确为"pass"（不区分大小写）的数据
- 空单元格保留为空，不移除
- 其他数据全部保留

### 输出要求
- 表头必须保留
- 列顺序与原始表格一致
- 数据行顺序可按原表格顺序
