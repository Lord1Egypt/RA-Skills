# sofagent-fde

> FDE 企业部署的十步标准化流程——带着 AI Agent 进企业跑落地，人走后 Agent 依然守规矩。
>
> v0.95 · 孔放勋

**问题**：你带着 AI Agent 进企业跑通了落地，部署完人走了。两周后回来一看——Agent 改了不该改的文件、踩过的坑没记住、产出没人审。**人治有你，人走没纪律。**

**sofagent-fde 解决的是**：给你一个标准化的十步部署流程，确保你交付的不是一个「能跑的 Agent」，而是一套「有纪律的、可持续的治理体系」——你走之后，Agent 该守的规矩守着，该记的教训记着，该审的产出审着。

---

## 谁用这个

**你（部署专家）用。** 不是企业终端用户用。

| 你可能是 | 场景 |
|------|------|
| 外部 FDE 顾问 | 带着部署专家和 AI Agent 进驻企业现场，完成交付后离场 |
| 企业 CIO / 网管 / IT 负责人 | 自己用 AI Agent 做内部摸底和节点规划，不请外部顾问 |

> sofagent-fde 是部署专家的**操作手册**，不是企业的对话机器人。你负责和人聊，fde 负责帮你把每一句话变成可执行的方案。

### 你交付什么（不是什么）

| 你交付的 | 你不交付的 |
|------|------|
| 有纪律的 Agent 节点（守规矩、能审计） | 运维系统（监控/告警/重启/日志轮转） |
| 部署方案书（一份 .md，企业看得懂） | 万能 AI 平台 |
| 知识库（5 个静态文档 + 后续自动生长） | 持续驻场支持 |
| 能自己运转的治理体系 | — |

> **范围声明**：sofagent 是 FDE 的**纪律底座**，不是**运维底座**。覆盖 FDE 问题空间约 20%（Agent 质量层：纪律 + 审计 + 经验沉淀），运维层约 80% 不在范围内。简单说：管 Agent 干活守不守规矩，不管 Agent 服务器开不开机。

## 十步部署流程

每一步有明确的**做什么 → 产出什么**。不是建议，是操作清单——跳了哪步，交付就有漏洞。

| 步骤 | 做什么 | 产出什么 |
|:----:|--------|------|
| §一 | 确定企业场景：行业、规模、痛点 | 企业画像（一段话 + 部门/岗位清单） |
| §二 | 盘点平台与工具：协同平台、业务系统、数据可达性、知识库 | 技术环境清单 |
| §三 | 工作流深挖：逐岗位、逐流程，五要素全填 | 完整工作流节点图 |
| §四 | 识别 AI 节点：对每个节点判 🔄自动执行 / ⚡强化岗位 / 👤不动 | 节点分类清单（含判定理由） |
| §五 | 价值量化：替人成本 + 时间成本 + 隐性成本，算回本周期 | 节点价值清单（老板看这个买单） |
| §六 | 出方案：每个节点定版本/铁律/审计/输入输出/企业 Skill | 节点→方案对照表 |
| §七 | 部署与交付：五个阶段（部署 → 知识库 → 检查点 → 教会一人 → 独立运行） | 在跑的 AI 节点 + 知识库 + 检查点 |
| §八 | 交付物确认：部署方案书 + 知识库 + 运行时三样 | 交付清单（逐条打勾） |
| §九 | 企业画像速采模板 | 标准化采集表（§一/§二 用） |
| §十 | 离场检查清单：9 项全部打勾才能走 | 离场确认 |

> 十步详情见 [SKILL.md](./SKILL.md)——316 行的现场操作手册，每步都有追问话术和产出模板。

## 装好就能用

### 1. 装 sofagent 完整版（底座）

sofagent-fde 需要 sofagent 完整版做底座——fde 管部署流程，sofagent 管 Agent 纪律。两者是流程和规则的关系：fde 告诉你「怎么部署」，sofagent 让 Agent「部署后守规矩」。

```bash
git clone https://github.com/KongFangXun/sofagent.git
cd sofagent && bash sofagent/scripts/install.sh
```

### 2. 加载 fde Skill

fde 只有一个文件（`SKILL.md`，316 行）。用你的 Agent 客户端加载这个 Skill，进企业现场时 Agent 会按十步流程引导你。

```bash
# fde 已在仓库内，不需要单独安装
ls sofagent/sofagent-fde/SKILL.md
```

### 3. 验证

加载后问你的 Agent：「我们准备进一家企业做 AI 部署，按 fde 流程走第一步。」如果 Agent 开始问你「企业叫什么、做什么行业、多少人」——说明 fde 已生效。

### 配套依赖

| 组件 | 角色 | 必须？ |
|------|------|:--:|
| [sofagent 完整版](../README.md) | 纪律层底座——4 底线 + 6 则铁律 + 编排 + 审计 | ✅ fde 管流程，sofagent 管纪律 |

## 一个具体场景

你是 FDE 顾问，带着 Agent 进一家 50 人的电商公司。

- **§一**：确认这是一家做母婴用品的电商，运营团队 15 人，客服 8 人，供应链 10 人。
- **§二**：他们用钉钉协同，ERP 是自研的，知识库在语雀。
- **§三**：深挖运营岗——每天早上拉昨天的销售数据、做日报、发到钉钉群。五要素全填。
- **§四**：这个节点判 🔄（输入可自动取、规则明确、输出可自动推）。
- **§五**：这个岗位月薪 8K，时薪约 45 元，每天花 1.5 小时做这个。AI 接管后年省约 1.6 万 × 15 人 = 24 万。一次部署成本 5 万，**三个月回本**。
- **§六**：这个节点配 full 版 sofagent，审计重点 A7 不存盲改（要读 ERP 数据），审计周期日报，输入 ERP API，输出钉钉群。
- **§七**：部署 sub-agent、建知识库、配检查点（10% 抽检）、教会运营主管、跑两天无人问 → 独立运行。
- **§八-§十**：交付方案书、确认 9 项离场清单、走人。

你走之后：运营的 Agent 每天自动拉数据、做日报、发钉钉群。sofagent-audit 每周扫一次 git diff，确保 Agent 没越界。运营主管是节点主人，有问题找知识库的运维手册。**人治在你驻场时，纪律在你离场后。**
