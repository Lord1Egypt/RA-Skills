# OPC行业深度分析技能

## 概述

本技能为OPC创业者提供**系统化行业深度分析能力**，从6维竞品对标升级为**10维度完整行业分析**，新增客单价量本利模型、合规风险评估、风险变化预期等关键维度。输出专业Word文档报告，适用于战略决策、投资判断、市场进入等场景。

**核心价值**：让OPC用最小成本获得接近咨询公司级别的行业洞察。

**数据源保障**：内置50+权威数据源，覆盖宏观/行业/投资/工商/专利/政策/技术/国际8大类，强制引用规范确保每个数据点有据可查。

### 引用铁律（不可违反）
1. **核心论据双源验证**：TAM/SAM/SOM、CAGR、市场规模等核心数据必须有≥2个A级源交叉验证
2. **单一B级源限制**：单一B级源不能作为核心论据的唯一支撑，必须找A级源补强
3. **C级源禁用**：媒体报道、匿名来源、未经证实数据禁止作为论据，仅作搜索线索
4. **逐数据标注来源**：报告中每个数据点必须标注来源（机构名+年份），无来源的数据视为无效
5. **引用格式**：正文中 `(机构名，年份)`；表格中增设"数据来源"列；附录按分类列出完整引用

### 数据源选择优先级
1. 官方统计（统计局/央行/海关）→ 2. 国际权威（Gartner/IDC/McKinsey）→ 3. 国内头部研究（艾瑞/信通院/赛迪）→ 4. 专业垂直（清科/智慧芽/IT桔子）→ 5. 行业媒体/咨询（36氪/前瞻/头豹）→ 6. 搜索发现的新来源（需验证资质）

## 10步工作流程

### 步骤1：需求确认与赛道界定
- 解析用户输入，明确分析范围（行业边界、细分赛道）
- 搜索最新市场数据和行业动态
- 确认分析重点维度（可跳过非关键步骤）
- **数据源指引**：优先搜索艾瑞咨询、IDC、前瞻产业研究院的行业概览报告，确定赛道边界和市场规模量级

### 步骤2：PEST宏观环境分析
- **政策(Policy)**：搜索相关政策文件，评估政策影响（🟢利好/🟡中性/🔴利空）
  - **必搜数据源**：中国政府网(gov.cn)、发改委(ndrc.gov.cn)、工信部(miit.gov.cn)、北大法宝(pkulaw.com)
  - **引用要求**：政策文件必须标注文号和发布日期，地方政策需标注省份
- **经济(Economy)**：搜索市场规模、增速、付费意愿数据
  - **必搜数据源**：国家统计局(stats.gov.cn)、央行(pbc.gov.cn)、海关总署(customs.gov.cn)
  - **引用要求**：经济数据必须标注统计年份和发布机构
- **社会(Social)**：搜索市场认知度、用户痛点、需求变化
  - **必搜数据源**：艾瑞咨询用户调研、百度指数、央视调查数据
  - **引用要求**：社会趋势数据需多源交叉，至少2个来源验证
- **技术(Technology)**：搜索技术成熟度、AI应用现状、关键突破
  - **必搜数据源**：Gartner Hype Cycle、中国信通院(caict.ac.cn)白皮书、赛迪顾问
  - **引用要求**：技术成熟度评估需标注判断依据（Gartner曲线位置/TRL等级）

### 步骤3：市场规模与增速分析
- 测算**TAM/SAM/SOM**三层市场空间
- 收集CAGR数据，分析增长驱动因素
- 全球对标分析，引用权威数据源
- **必搜数据源**：
  - 国内市场：艾瑞咨询、IDC、前瞻产业研究院、沙利文
  - 全球市场：Gartner、Statista、IMF、世界银行
  - 投融资数据：IT桔子、清科研究中心、CB Insights
- **引用要求**：
  - TAM/SAM/SOM每个测算必须有≥2个A级源交叉验证
  - CAGR数据必须标注测算期间和来源
  - 自行测算需说明假设前提和推算逻辑

### 步骤4：产业链分析
- 绘制上游/中游/下游三层产业链图谱
- 分析各环节核心玩家、价值分配、整合难度
- 识别产业链关键节点和机会点
- **必搜数据源**：天眼查/企查查（企业画像）、Wind（上市公司财务）、前瞻产业研究院（产业链图谱）
- **引用要求**：产业链企业信息需标注数据来源（天眼查/企查查/Wind），价值分配数据需标注来源

### 步骤5：竞争格局分析
- 识别四类竞品：传统服务商/数据平台/AI工具/垂直专家
- 构建六维对比矩阵：技术深度/AI能力/行业Know-How/交付效率/成本/可定制化
- 定位市场空白区域，发现差异化机会
- **必搜数据源**：
  - 企业信息：天眼查、企查查、巨潮资讯（上市公司年报）
  - 融资情况：IT桔子、Crunchbase、36氪
  - 专利布局：智慧芽(zhihuiya.com)、国知局(cnipa.gov.cn)
- **引用要求**：竞品数据需标注信息来源和获取时间，融资数据需标注轮次/金额/投资方

### 步骤6：技术路线与趋势分析
- 分析当前主流技术路线及成熟度
- 识别关键技术瓶颈及解决路径
- 预测未来3年技术演进方向
- **必搜数据源**：
  - 技术成熟度：Gartner Hype Cycle、中国信通院白皮书
  - 专利趋势：智慧芽、Google Patents、Espacenet
  - 学术前沿：arXiv、Semantic Scholar、中国知网
- **引用要求**：技术成熟度判断需标注来源（Gartner定位/TRL等级），专利数据需标注检索范围和时间

### 步骤7：客户画像与痛点分析
- 识别6类目标客户并绘制画像
- 分析决策链路和付费意愿
- 构建客户价值矩阵（付费能力×战略价值）
- **必搜数据源**：
  - 客户画像：清科研究中心（投资机构画像）、艾瑞咨询（用户调研）
  - 付费意愿：PitchBook、投中研究院（行业客单价）
- **引用要求**：客户画像数据需标注调研来源和样本量，付费数据需标注年份

### 步骤8：客单价分析与量本利模型
- 收集行业客单价数据，按客户类型分层
- 测算固定成本/变动成本/边际贡献
- 计算盈亏平衡点和目标利润所需业务量
- **必搜数据源**：
  - 行业定价：清科研究中心、PitchBook、艾瑞咨询
  - 上市公司成本：Wind、巨潮资讯（年报披露）
  - 行业基准：头豹研究院、前瞻产业研究院
- **引用要求**：客单价数据需标注来源和年份，自行测算需说明假设前提

### 步骤9：合规风险与变化预期
- 识别数据合规/责任边界/资质要求等风险点
- 标注风险等级：🟢低风险 / 🟡中风险 / 🔴高风险
- 预测未来1-3年风险变化趋势和监管走向
- **必搜数据源**：
  - 法规文件：北大法宝(pkulaw.com)、威科先行(wkinfo.com.cn)
  - 监管动态：市场监管总局(samr.gov.cn)、工信部、网信办
  - 行业合规：信通院合规报告、律所合规指南
- **引用要求**：法规引用必须标注文号和生效日期，风险预测需标注政策依据

### 步骤10：结论与建议
- 基于10维分析形成核心结论
- 提出差异化定位和竞争策略
- 制定短期/中期/长期目标（2025/2026/2027+）
- **引用要求**：结论中每个关键判断需回溯到前述章节的具体数据来源，不能出现无依据的新论断

## 输出格式要求

### Word文档生成
- 使用docx-js库生成标准.docx文件
- 专业蓝配色：`#2563eb`作为主色调
- 中文字体：`微软雅黑`，英文字体：`Arial`

### 表格规范
- 使用docx的`Table`对象，禁止使用ASCII线条绘制表格
- 表头背景色：`#2563eb`，白色字体
- 交替行背景色：`#f8fafc` / `#ffffff`
- 风险等级列：🟢绿色背景 / 🟡黄色背景 / 🔴红色背景

### 数据可视化
- 使用**表格+文字描述**实现图表效果
- 产业链图用层级表格表示
- SWOT矩阵用2×2表格呈现
- 增长趋势用数据表格+趋势描述

### 文档结构
```
📄 行业深度分析报告
├── 执行摘要
│   ├── 核心结论（3-5条）
│   ├── 关键数据点（TAM/SAM/CAGR等）
│   └── OPC机会窗口
├── 一、PEST宏观环境分析
├── 二、市场规模与增速
├── 三、产业链分析
├── 四、竞争格局分析
├── 五、技术路线与趋势
├── 六、客户画像与痛点
├── 七、客单价与量本利分析
├── 八、合规风险与变化预期
├── 九、结论与建议
└── 附录（数据来源/术语解释）
```

## 质量检查清单

```
✅ 数据源引用合规（强制项，不通过则报告无效）
   □ 核心论据（TAM/SAM/SOM/CAGR/市场规模）有≥2个A级源交叉验证
   □ 每个数据点标注来源（机构名+年份）
   □ 无C级源（媒体报道/匿名来源）作为核心论据
   □ 单一B级源无A级源补强的数据已标注为"待验证"
   □ 法规文件标注文号和生效日期
   □ 自行测算数据说明假设前提和推算逻辑

✅ 数据准确性
   □ TAM/SAM/SOM测算有明确依据
   □ 市场数据标注来源和置信度等级
   □ 风险等级标注有数据支撑
   □ 避免"据传""据说"等模糊表述

✅ 分析完整性
   □ 10个维度全覆盖（或说明跳过原因）
   □ 产业链上中下游完整
   □ 竞品覆盖主要类型
   □ 客户画像不少于4类
   □ 每步使用的数据源记录在附录

✅ 输出规范性
   □ Word文档格式正确
   □ 表格使用Table对象，含"数据来源"列
   □ 专业蓝配色统一
   □ 目录结构清晰
   □ 附录含完整数据源引用清单（按8大类分类）

✅ 建议可执行性
   □ 结论有数据支撑，可回溯到具体章节
   □ 策略可落地
   □ 目标有时间节点
   □ 风险有应对策略
```
