## Description: <br>
Reviews Swift Testing code for proper use of #expect/#require, parameterized tests, async testing, and organization. <br>

This skill is ready for commercial/non-commercial use. <br>

## Publisher: <br>
[anderskev](https://clawhub.ai/user/anderskev) <br>

### License/Terms of Use: <br>
MIT-0 <br>


## Use Case: <br>
Developers and engineers use this skill to review Swift files that use Swift Testing APIs such as import Testing, @Test, #expect, #require, @Suite, parameterized tests, async confirmations, and related organization patterns. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Review guidance could be inaccurate or applied outside the Swift Testing files in scope. <br>
Mitigation: Require explicit Swift file scope, confirm Swift Testing markers before critique, and cite file-line evidence for each reported issue. <br>
Risk: The agent could perform actions beyond review assistance. <br>
Mitigation: Treat the skill as documentation-only; it should not run code, modify files, access credentials, or contact external services. <br>
