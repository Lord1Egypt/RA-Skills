# 批量企业信息导出

## 适用场景

当用户需要一次性查询**多家企业**的**基本信息**时，使用批量导出功能，而非逐个调用单个查询工具。

**触发条件**：
- 用户提供了一批企业名称（≥5家），且需要查询的是企业本身的工商基础信息
- 用户需要批量获取联系方式（手机、固话、邮箱）

**不适用场景**：
- 查询司法风险、经营信息、知识产权等非基本信息 → 使用 `find_tool.js` + `call.js` 逐个查询
- 仅查询1-2家企业的某项信息 → 使用 `find_tool.js` + `call.js` 单独查询

## 支持导出的字段

批量导出仅支持企业基本信息，包括：

| 类别 | 字段 |
|------|------|
| 工商登记 | 统一社会信用代码、注册号、组织机构代码、成立日期、核准日期、登记状态 |
| 人员资本 | 法定代表人、注册资本（万元）、企业类型、营业期限 |
| 地址经营范围 | 注册地址、经营范围 |
| 区域行业 | 所属省份、所属城市、所属区县、国标行业门类、国标行业大类 |
| 其他 | 曾用名、企业英文名 |
| 社保 | 参保人数、参保人数所属年报 |
| 联系方式-手机 | 精选手机、更多手机、严选手机、去除年报手机 |
| 联系方式-固话 | 精选固话、更多固话、严选固话、去除年报固话 |
| 联系方式-邮箱 | 精选邮箱、更多邮箱 |

## 操作流程

### 步骤1：整理企业名称为 txt 文件

将企业名称保存为 `.txt` 文本文件，每行一个企业名称：

```
小米科技有限责任公司
华为技术有限公司
腾讯科技（深圳）有限公司
```

**用户提供的文件格式转换**：

用户提供的可能是任意格式的文件（如 `.xlsx`、`.csv`、`.docx`、`.pdf` 等），也可能直接在对话中列出企业名称。agent 需要读取文件内容，提取其中的企业名称，生成符合上传规范的 `.txt` 文件：

1. 读取用户提供的文件，提取企业名称列（若为表格类文件，需识别包含企业名称的列）
2. 将提取的企业名称逐行写入新的 `.txt` 文件，每行一个企业名称
3. 确保生成的 `.txt` 文件符合以下格式要求

**文件格式要求**：
- 必须为 `.txt` 文本文件，UTF-8 编码
- 每行一个企业名称，不允许空行或仅含空白字符的行
- 单行长度不超过 10000 字符
- **单个文件最多 10000 行**（即最多查询 10000 家企业）
- 文件大小不超过 5MB

**超过 10000 家企业时**：需将企业名称拆分到多个 txt 文件中，每个文件不超过 10000 行，然后逐个上传。

### 步骤2：上传文件

使用 `upload_file.js` 脚本上传 txt 文件：

```bash
node scripts/upload_file.js "<txt文件路径>"
```

示例：

```bash
node scripts/upload_file.js "D:\download\enterprises.txt"
```

脚本会自动执行文件校验和上传，校验不通过时会列出具体错误信息。

### 步骤3：获取结果

上传成功后，脚本返回 JSON 结果：

```json
{
  "code": "200",
  "message": "SUCCESS",
  "result": {
    "file_link": "https://..../企业批量查询导出(匹配结果-2条)-202605211410162rV5.xlsx",
    "matched_count": 1,
    "remaining_count": 4999
  },
  "error": null
}
```

**返回字段说明**：

| 字段 | 说明 |
|------|------|
| `code` | 状态码，`"200"` 表示成功 |
| `message` | 状态描述 |
| `result.file_link` | Excel 文件下载地址，包含所有匹配企业的基本信息 |
| `result.matched_count` | 成功匹配的企业数量 |
| `result.remaining_count` | 剩余可导出条数额度 |
| `error` | 错误信息，成功时为 `null` |

将 `result.file_link` 提供给用户，用户可通过该链接下载 Excel 文件。同时告知用户剩余可导出条数（`remaining_count`）。

**导出条数不足时的响应**：

当剩余导出条数不足时，返回状态码 `238`：

```json
{
  "code": "238",
  "message": "批量导出条数不足",
  "result": {
    "remaining_count": 0
  },
  "error": null
}
```

遇到此情况时：需根据剩余额度减少查询企业数量，并告知用户。

### 步骤4：清理临时文件

批量查询完成后，需删除批量查询过程中生成的临时文件，不删除用户的输入文件。

## 完整示例

用户说："帮我查一下这三家公司的基本信息：小米科技有限责任公司、华为技术有限公司、腾讯科技（深圳）有限公司"

操作步骤：

1. 将三家企业名称写入 txt 文件（如 `companies.txt`）
2. 运行上传命令：

```bash
node scripts/upload_file.js "companies.txt"
```

3. 将返回的 `file_link` 链接提供给用户

## 多文件处理示例

当企业数量超过 10000 家时，需拆分为多个文件：

```
batch_01.txt   # 第 1-10000 家企业
batch_02.txt   # 第 10001-20000 家企业
batch_03.txt   # 第 20001-25000 家企业
```

然后依次上传每个文件：

```bash
node scripts/upload_file.js "batch_01.txt"
node scripts/upload_file.js "batch_02.txt"
node scripts/upload_file.js "batch_03.txt"
```

每次上传都会返回独立的 Excel 下载链接。

## 注意事项

- 批量导出仅返回企业基本信息，不包含司法风险、知识产权、经营信息等
- `matched_count` 可能小于文件中的企业数量，部分企业名称可能无法匹配
- 上传后 Excel 文件生成需要一定时间，链接有效期有限，建议及时下载
- 如需查询非基本信息，请使用 `find_tool.js` 查找对应工具后通过 `call.js` 逐个查询
- 批量查询完成后，需删除批量查询过程中生成的临时文件，不删除用户的输入文件。
- 每次导出成功后，响应中的 `remaining_count` 表示剩余可导出条数，额度不足时需提醒用户
