# 星座运势输出格式

支付成功并拿到接口 JSON 后渲染。**仅使用接口返回字段，禁止编造内容。**

## 输出顺序（固定）

**星座档案始终放在首位。** 完整输出结构：

```markdown
{星座档案模板} ← 有 result 档案字段时必渲染，排第一

---

{运势模板} ← 按 type 选日/周/月/年

---

_以上内容仅供娱乐参考，不具备科学依据。_
```

仅有档案、无运势数据时，只输出档案 + 页脚免责。

---

## 星座档案模板

接口返回 `result` 对象且含 `name`、`range` 等档案字段时使用，**固定排在全文最前**。

数据路径：`result.*`（外层 `reason`、`error_code` 不展示给用户）。

### 渲染结构

```markdown
# ✨ {name} · 星座档案

> 📅 出生日期：{range 格式化}

## 基础信息

| 项目     | 内容    |
| -------- | ------- |
| 星座     | {name}  |
| 日期区间 | {range} |
| 星座特点 | {zxtd}  |
| 元素属性 | {sssx}  |
| 掌管宫位 | {zggw}  |
| 阴阳性   | {yysx}  |
| 最大特征 | {zdtz}  |
| 主管行星 | {zgxx}  |

## 幸运属性

| 项目     | 内容   |
| -------- | ------ |
| 幸运色   | {xyys} |
| 幸运石   | {jssw} |
| 幸运数字 | {xyhm} |
| 开运金属 | {kyjs} |

## 性格标签

**基本表现**：{bx}

## 性格解读

### 优点

{yd}

### 缺点

{qd}

### 基本特质

{jbtz}

### 具体特质

{jttz}

### 行事风格

{xsfg}

### 个性盲点

{gxmd}

### 综合总结

{zj}
```

### 字段映射

| 展示项   | JSON 字段      | 说明                                            |
| -------- | -------------- | ----------------------------------------------- |
| 星座名称 | `result.name`  | 如「金牛座」                                    |
| 日期区间 | `result.range` | 如 `4.20-5.20`，可格式化为「4月20日 - 5月20日」 |
| 星座特点 | `result.zxtd`  | 如「稳健固执」                                  |
| 元素属性 | `result.sssx`  | 火/土/风/水                                     |
| 掌管宫位 | `result.zggw`  | 如「第二宫」                                    |
| 阴阳性   | `result.yysx`  | 阴性 / 阳性                                     |
| 最大特征 | `result.zdtz`  | 如「财富」                                      |
| 主管行星 | `result.zgxx`  | 如「金星」                                      |
| 幸运色   | `result.xyys`  |                                                 |
| 幸运石   | `result.jssw`  | 如「翡翠、玉」                                  |
| 幸运数字 | `result.xyhm`  |                                                 |
| 开运金属 | `result.kyjs`  |                                                 |
| 基本表现 | `result.bx`    | 短标签，逗号分隔                                |
| 优点     | `result.yd`    |                                                 |
| 缺点     | `result.qd`    |                                                 |
| 基本特质 | `result.jbtz`  | 长文                                            |
| 具体特质 | `result.jttz`  | 长文；空值跳过                                  |
| 行事风格 | `result.xsfg`  |                                                 |
| 个性盲点 | `result.gxmd`  |                                                 |
| 综合总结 | `result.zj`    | 优先于 jbtz 作「一句话画像」                    |
| 图标     | `result.icon`  | URL 非空时可作标题装饰，否则跳过                |

日运「幸运参数」中的 `color`、`number` 与档案 `xyys`、`xyhm` 重复时，**以运势接口当日字段为准**；档案区仍展示本命幸运属性。

## 通用页眉

所有周期共用以下标题区（字段取自接口 `name`、`date` / `datetime`）：

```markdown
# ✨ {name} · {周期标签}运势

> 📅 {date 或 datetime}
```

| 周期     | `{周期标签}` | 日期字段                                         |
| -------- | ------------ | ------------------------------------------------ |
| today    | 今日         | `datetime`                                       |
| tomorrow | 明日         | `datetime`                                       |
| week     | 本周         | `date`（区间，如 2014年06月29日-2014年07月05日） |
| month    | 本月         | `date`（如 2016年12月）                          |
| year     | 年度         | `date` 或 `{year}年`                             |

## 日运模板 (`today` / `tomorrow`)

日运/明日运含**数值指数**，用五星制可视化（金实心 / 空心）；文本概述单独成段。

### 渲染结构

```markdown
# ✨ {name} · {今日|明日}运势

> 📅 {datetime}

## 运势指数

| 维度 | 指数     | 星级                   |
| ---- | -------- | ---------------------- |
| 综合 | {all}    | {all 对应星级 HTML}    |
| 爱情 | {love}   | {love 对应星级 HTML}   |
| 工作 | {work}   | {work 对应星级 HTML}   |
| 财运 | {money}  | {money 对应星级 HTML}  |
| 健康 | {health} | {health 对应星级 HTML} |

> **星级换算**（满分 100，五星制，按分数区间取整星，不使用半星）：
>
> | 关键词 / 分数区间 | 星级 |
> | ----------------- | ---- |
> | 极佳、小吉、90-100 分 | ★★★★★ |
> | 较好、80-89 分 | ★★★★☆ |
> | 一般、中平、56-79 分 | ★★★☆☆ |
> | 较差、小凶、40-55 分 | ★★☆☆☆ |
> | 很差、大凶、0-39 分 | ★☆☆☆☆ |
>
> **星级样式**（必须用 HTML 着色，共 5 颗星位从左到右排列）：
>
> | 类型     | 写法                                   | 说明             |
> | -------- | -------------------------------------- | ---------------- |
> | 实心整星 | `<span style="color:#FFD700">★</span>` | 金色实心         |
> | 空心空星 | `☆`                                    | 不着色，保持空心 |
>
> **拼装示例**：
>
> - 90 分 → ★★★★★ → 5 颗金 `★`
> - 85 分 → ★★★★☆ → 4 颗金 `★` + 1 颗 `☆`
> - 65 分 → ★★★☆☆ → 3 颗金 `★` + 2 颗 `☆`
> - 54 分 → ★★☆☆☆ → 2 颗金 `★` + 3 颗 `☆`
> - 39 分 → ★☆☆☆☆ → 1 颗金 `★` + 4 颗 `☆`
```

```markdown
### 💰 财运
```

## 年运模板 (`year`)

年运含年度主题 + 多维长文；`mima.text`、`career`、`love`、`finance` 为**数组**，逐项渲染为列表或分段。

### 渲染结构

```markdown
# ✨ {name} · {year} 年度运势

> 📅 {date}

## 年度主题

**{mima.info}**

{mima.text 数组：每项一段，段间空行}

## 分项运势

### 💼 事业运

{career 数组：每项以 `- ` 列表项或独立段落呈现}

### ❤️ 感情运

{love 数组}

### 💰 财运

{finance 数组}
```

### 字段映射

| 展示项   | JSON 字段       | 说明                 |
| -------- | --------------- | -------------------- |
| 年份     | `year` / `date` |                      |
| 年度主题 | `mima.info`     | 短标题               |
| 年度说明 | `mima.text[]`   | 字符串数组，逐项输出 |
| 事业     | `career[]`      | 字符串数组           |
| 感情     | `love[]`        | 字符串数组           |
| 财运     | `finance[]`     | 字符串数组           |

## 渲染规则

1. **档案首位**：有 `result` 档案字段时，必须先渲染 [星座档案模板](#星座档案模板)，再渲染运势。
2. **按 type 选运势模板**：日/周/月/年四选一，不与档案模块混用字段。
3. **空值跳过**：字段缺失、空字符串、`null` 时不展示该行/小节（含 `icon`、`jttz`），不填「暂无」。
4. **数组字段**：`mima.text`、`career`、`love`、`finance` 每项独立成段或列表，保持原文。
5. **文本清洗**：去除 HTML/script 标签；保留中文标点与换行。
6. **禁止臆造**：不生成「建议」「宜忌」等接口未返回的内容。
7. **页脚免责**（固定追加）：

```markdown
---

_以上内容仅供娱乐参考，不具备科学依据。_
```
