# 投资分析工作流（Investment Analysis Workflow）

> 核心目标：所有新分析必须建立在已存档报告的既有逻辑和数据之上，只做增量修正，禁止另起炉灶。

---

## 一、与已有 Skill 的协同关系

| Skill | 角色 | 何时调用 |
|:---|:---|:---|
| **本 Skill** | **流程规范层** | 每次收到新材料时，规定"先读什么、后做什么、输出到哪里" |
| `prosperity-investing` | **分析方法论** | 判断产业周期、四浪定位、大开门筛选、选股纪律；**行业全景扫描**（8层产业链框架） |
| `optical-comm-investing` | **行业知识库** | 分析光通信/光模块/光芯片标的时，提供技术路线和供应链细节 |
| `upstream-material-analysis` | **行业知识库** | 分析上游材料标的时，提供量价测算和传导链分析 |

**三层嵌套（公司分析场景）**：
```
本 Skill（流程规范）
    ↓ 规定 Step 1-6
prosperity-investing（方法论）
    ↓ 提供四浪理论、大开门六维筛选、选股纪律、三维度分析法
optical-comm-investing / upstream-material-analysis（行业知识）
    ↓ 提供技术路线、供应链、竞争格局
```

**行业研究场景的嵌套（新增）**：
```
prosperity-investing（行业全景框架）
    ↓ 第1-8层：行业空间→产业链拆解→竞争格局→传导链→景气度→变量跟踪→标的扫描→投资结论
本 Skill（流程规范）
    ↓ 规定读取哪些产业跟踪文件、输出到哪里
optical-comm-investing / upstream-material-analysis（行业知识）
    ↓ 提供该行业的技术路线、供应链、竞争格局细节
```

## 二、核心原则（不可违反）

### 原则1：存档报告是主信息源
- 任何新分析的**第一信息源**是 `investment/` 目录下已存档的报告
- 外部搜索仅用于**补充数据缺口**（找不到的旧数据、最新股价、最新公告）
- **禁止**：用临时搜索的新信息**替代**已存档的核心逻辑

### 原则2：增量分析，不重复造轮子
- 已有内容（公司背景、行业框架、六维打分）**不再重写**
- 新材料只做三件事：
  1. **验证**已有假设 → 更新验证状态
  2. **修正**已有假设 → 更新业绩预估/关键变量（附理由）
  3. **新增**信息 → 追加到跟踪清单

### 原则3：连续性优先
- 每次分析必须是**上一次分析的延续**，而非独立事件
- 业绩预估数字必须能追溯到之前的逻辑链条，修改要说明理由
- 四浪定位、大开门判定等结论变更要解释"什么新信息改变了判断"

### 原则4：单一标的，单一主文件
- 每个标的只有一个主分析文件（如 `永鼎股份_主分析.md`）
- 新材料不再创建独立新文件，而是**追加到主文件末尾**（Append模式）
- 只有**全新标的/行业**才创建新文件

## 三、标准操作流（SOP）：6步

### Step 1: 新材料分类（10秒判断）

```
Q1: 这个材料涉及哪个已存档标的/行业？
    ├── 是已有标的 → Step 2 (增量更新)
    ├── 是已有行业的新标的 → Step 2 + Step 5 (关联行业逻辑)
    └── 完全全新 → Step 6 (全新建档)

Q2: 这个材料的性质是什么？
    ├── 公司公告/财报 → 验证/修正业绩预估
    ├── 行业新闻 → 验证/修正行业景气判断
    ├── 调研/交流 → 补充关键变量/新增跟踪项
    └── 政策/宏观 → 修正行业/多标的逻辑
```

### Step 2: 读取相关存档（必须执行）

**读取清单**（按优先级排序）：

| 优先级 | 文件 | 读取内容 |
|:---|:---|:---|
| **P0** | 该标的的**主分析文件** | 核心逻辑、业绩预估、关键变量、上次判断 |
| **P1** | `大开门股票池综合评估.md` | 该标的在池中的排序和判定 |
| **P2** | `大开门股票池.md` | 标的的基础信息 |
| **P3** | 相关行业框架文件 | 行业景气判断、产业链位置 |
| **P4** | `大开门股票池_研报跟踪存档.md` | 该标的的历史跟踪记录 |

**禁止**：未读取P0-P2就直接分析新材料。

**如何读取**：使用 `read` 工具读取上述文件，将关键信息（当前业绩预估、上次判定、跟踪清单）作为本次分析的基底。

### Step 3: 增量分析（核心步骤）

**3.1 关联强度量化标准**

收到新材料后，先判断与股票池标的的关联强度。判断标准唯一：**利润影响的可量化性**。

| 关联强度 | 判断标准 | 盈利预测动作 | 自检问题 |
|:---|:---|:---|:---|
| 🔴 **强** | 新材料中的数字**可直接代入**该标的利润公式（产能×单价×净利率、订单×单价×净利率等） | **必须修正**盈利预测和PE | 如果无法写出具体的数字修正，不是强关联 |
| 🟡 **中** | 新材料提供验证/新增变量，但需**额外推导**才能影响利润（如行业涨价→该标的是否受益需看份额/客户结构） | **新增**跟踪指标，不直接修正利润 | 如果写不出"新增跟踪指标"是什么，实际应为⚪弱 |
| ⚪ **弱** | 新材料仅验证行业逻辑，与该标的**无直接数字关联** | 明确标注"无直接关联"，不写 | 禁止为了凑数写弱关联 |

> **执行检查**：如果判定为🟡中但写不出新增跟踪指标，降级为⚪弱。如果判定为🔴强但无法写出修正公式，降级为🟡中。

**3.2 增量映射**

将新材料中的每条信息，对照已有逻辑做**增量映射**：

| 新材料信息类型 | 对照已有逻辑 | 输出动作 |
|:---|:---|:---|
| 业绩数据（如Q1利润） | 对比已有业绩预估 | 更新预估数字，说明偏差原因 |
| 产能/订单数据 | 对比已有产能假设 | 修正或确认产能爬坡节奏 |
| 客户/合作信息 | 对比已有客户列表 | 新增客户质量评估 |
| 价格/毛利率变化 | 对比已有利润率假设 | 修正利润模型 |
| 行业景气信号 | 对比已有行业判断 | 确认或修正行业周期位置 |
| 政策/宏观变化 | 对比已有多标的影响 | 批量更新相关标的逻辑 |

**3.3 产业信息冲突处理**

当新材料中的产业信息（产能、价格、需求、技术等）与已存档产业跟踪文件中的信息冲突时：

| 冲突类型 | 判断规则 | 处理动作 |
|:---|:---|:---|
| **同一指标，不同数字** | ① 公司公告/官方数据优先<br>② 无官方数据，取**最新时间**<br>③ 时间相同，取**一手信息**（专家访谈>券商研报>自媒体） | 更新产业跟踪为最新数字，保留旧数字作为对比，标注来源和时间 |
| **同一指标，方向相反** | 拆解到**具体产品型号/规格**，可能都是对的 | 细分指标，不笼统合并（如200μm涨价 vs G.652D降价） |
| **时间线冲突** | 取**最新信息**覆盖旧信息 | 更新指标，注明"原预期[旧]，最新[新]" |
| **个案 vs 总量冲突** | 个案是某公司的进展，总量是行业整体 | 不冲突，个案追加为"进展"，不修改总量判断 |
| **客户/订单信息冲突** | 以公司公告/合同披露为准 | 纪要中的"大概率"保留为待验证，不写入确认项 |

**禁止**：强行合并两个不同来源的数字，取平均或简单覆盖。必须标注来源和时间，让用户看到差异。

**示例**：
- 永鼎纪要说"国内光纤需求2.5亿芯公里"，但产业跟踪记录"2.0亿"→更新为2.5亿（最新），注明"原2.0亿（2026-05-28），更新为2.5亿（2026-06-04，永鼎纪要/天风）"
- 永鼎纪要说"200G EML已在客户端验证"，但产业跟踪记录"200G EML 2027年缺口20-30%"→不冲突。追加"鼎芯光电200G EML进入客户端验证（2026-06-04）"，不修改缺口20-30%的总量判断。

**增量分析模板**（必须按此格式输出）：

```markdown
## [增量更新] YYYY-MM-DD [材料来源]

### 新增/修正信息
- [信息1]：具体内容 → 影响：[验证/修正/新增]了[哪个假设]
- [信息2]：具体内容 → 影响：[验证/修正/新增]了[哪个假设]

### 业绩预估修正（如有）
| 年份 | 原预估 | 新预估 | 修正原因 |
|:---|:---|:---|:---|
| 2026E | X亿 | Y亿 | 新材料显示... |

### 关键变量更新
- [变量1]：原"[状态]" → 新"[状态]"（原因：新材料中的具体信息）
- [变量2]：原"[状态]" → 新"[状态]"（原因：新材料中的具体信息）

### 跟踪清单更新
- [x] 已完成：[事项]（验证结果：[结论]）
- [ ] 新增跟踪：[新事项]
- [ ] 状态变更：[原事项]从[待验证]变为[已验证/证伪]

### 结论变更（如有）
- 原结论：[X]
- 新结论：[Y]
- 变更原因：[新材料中的具体信息]
```

### Step 4: 调用分析方法论 Skill（按需）

根据标的所属行业，调用对应的方法论 Skill：

| 标的类型 | 调用 Skill | 解决的问题 |
|:---|:---|:---|
| 任意景气股 | `prosperity-investing` | 四浪定位、大开门六维筛选、选股纪律 |
| 光通信/光模块/光芯片 | `optical-comm-investing` | 技术路线、供应链位置、九字诀评估 |
| 上游材料（铜箔/衬底/晶体等） | `upstream-material-analysis` | 量价测算、传导链、利润结构影响 |

**调用方式**：读取对应 Skill 的 SKILL.md 和关键 reference 文件，将方法论应用到增量分析中。

**注意**：本 Skill 负责"什么时候调用、调用后怎么用"，方法论 Skill 负责"具体怎么判断"。

### Step 5: 一致性检查

- [ ] 新的业绩预估是否与之前的逻辑链条自洽？
- [ ] 如果修正了预估，是否解释了修正原因？
- [ ] 大开门判定是否仍然成立？如果不成立，说明了什么新信息导致变更？
- [ ] 四浪定位是否需要调整？调整依据是什么？
- [ ] 该标的的结论变化是否影响了 `大开门股票池综合评估.md` 中的排序？

### Step 6: 输出与更新（强制步骤）

**6.1 追加到主文件**
- 使用 `edit` 工具的 Append 模式，在主文件末尾添加 Step 3 的增量分析内容
- 追加内容必须包含**时间戳**和**材料来源**
- 追加内容必须明确引用之前的哪个结论被修正了
- 追加内容不能重复主文件中已有的背景信息

**6.2 更新综合评估**
- 分析完成后，**必须**更新 `大开门股票池综合评估.md` 中对应标的的条目
- 更新项：业绩预估数字、前瞻PE、是否大开门判定、四浪位置、时间戳

**6.3 研报跟踪存档（仅研报场景）**
- 如果新材料是研报/调研纪要，追加写入产业跟踪文件和个股跟踪文件
- **一源多归**：一份研报可能涉及多个产业方向（如CPO同时涉及光通信+半导体设备+材料），允许同时追加到多个产业文件，但个股层只写一次
- 格式遵循 `prosperity-investing` 中的研报跟踪工作流（Step 0-5）

**6.4 回执自动生成（仅研报场景）**
- 回执**从已归档文件中自动提取**，禁止手打两遍
- 提取路径：
  - 产业层面：从 `[产业跟踪文件]` 提取"新增产业指标"和"对产业的影响"
  - 个股层面：从 `[个股跟踪文件]` 提取"关联强度"、"关键发现"、"盈利预测修正"、"新增跟踪指标"
- 固定格式：

```
📄 研报分析回执
━━━━━━━━━━━━━━━━━━━━━━
研报来源：[从文件名提取]

【个股层面】（仅写相关标的）
```

> **禁止**：回执内容和归档文件内容不一致。回执是归档文件的**摘要提取**，不是独立创作。

## 四、文件管理规范

### 4.1 目录结构

**标的主文件**（不分季度，持续追加）：
```
investment/
├── [标的名称]_主分析.md          ← 每个标的只有一个主文件，持续追加
├── 大开门股票池综合评估.md         ← 统一排序表（持续更新）
└── archive/                       ← 历史版本存档
    └── [标的名称]_主分析_YYYYMMDD.md  ← 重大修订前的备份
```

**产业跟踪文件**（按季度分片，避免单文件膨胀）：
```
investment/产业跟踪/
├── [产业名称]_跟踪_2025Q4.md      ← 归档，只读
├── [产业名称]_跟踪_2026Q1.md      ← 归档，只读
├── [产业名称]_跟踪_2026Q2.md      ← 当前活跃文件（本季度）
└── [产业名称]_跟踪_索引.md         ← 汇总所有季度的累计指标总表
```

**个股研报跟踪文件**（不分季度，持续追加）：
```
investment/研报跟踪/
├── [标的名]_[代码]_研报跟踪.md     ← 持续追加，记录所有关联研报
└── [标的名]_[代码]_业绩重估_YYYYMMDD.md  ← 重大业绩修正独立文件
```

**分片规则**：
- 每季度初新建产业跟踪文件（如2026年7月1日新建`光通信_跟踪_2026Q3.md`）
- 旧季度文件冻结，只读，不再追加
- 索引文件保留所有季度的P0/P1/P2累计指标总表，不保留全文
- 查找历史跨文件搜索：`grep "关键词" investment/产业跟踪/*`
- 当前活跃文件行数控制：每季<<200行，超过则提前分片

## 六、错误示例 vs 正确示例

### ❌ 错误：独立分析，不读存档

```
用户：永鼎交流会纪要
我：读取交流会 → 临时搜索永鼎信息 → 独立写一份《永鼎交流会分析.md》
      → 不引用之前对永鼎的任何判断
      → 不关联之前光模块行业分析中的InP芯片逻辑
      → 不开主文件，另起炉灶
```

**违反原则**：原则1（存档不是主信息源）、原则4（未追加到主文件）。

### ✅ 正确：基于存档做增量

```
用户：永鼎交流会纪要
我：Step1: 判断→已有标的（永鼎）
    Step2: 读取《永鼎股份_主分析.md》+《大开门股票池综合评估.md》
    Step3: 交流会信息增量映射
           - Q1利润1.5亿 → 修正2026E预估（原5-8亿→新11亿）
           - Coherent合作 → 验证客户质量提升假设
           - 第二次扩产 → 新增关键变量
    Step4: 调用 prosperity-investing → 四浪定位、大开门判定
    Step5: 一致性检查 → 2026E PE 35x仍不满足；2027E乐观情景接近边界
    Step6: 追加到《永鼎股份_主分析.md》末尾 + 更新综合评估
```

**结果**：用户看到的是连续的逻辑演进，之前的跟踪清单被更新，大开门判定一致性得到维护。

## 八、参考来源

本 Skill 基于用户对分析连续性的硬性要求设计，解决以下痛点：
1. 分析碎片化：每次新材料独立成篇，无连续性
2. 逻辑断裂：不引用之前判断，导致结论前后矛盾
3. 重复劳动：公司背景、行业框架每次都重写
4. 跟踪清单丢失：之前的跟踪项被遗忘，未更新状态

## 九、景气投资深度周报标准（新增）

> 适用于周期性输出的产业链/行业深度周报，与"增量分析"（Step 1-6）形成互补——增量分析是"点"，周报是"面"。

### 9.1 周报8模块标准结构（顺序固定）

| 模块 | 内容要求 | 禁止 |
|:---|:---|:---|
| **1. 核心判断** | 一句话定调（钱往哪走）+ 3条逻辑 + 时间窗口 | 不写"多个可能"，必须给明确判断 |
| **2. 产业链利润分配推演** | 价值链拆解（利润占比/毛利率/定价权）+ 为什么A能赚钱B不能 + 利润迁移路径（2025→2026→2027） | 不罗列数据，每段必须有"我的判断" |
| **3. 技术路线/产品路线胜负手** | 当前状态对比表 + 我的判断（哪条路线会赢）+ 时间窗口 + 对投资的映射 | 不写"两种都有可能" |
| **4. 预期差挖掘** | 市场定价了什么 / 我认为没定价什么 / 如果兑现利润怎么变 | 不写"跟踪观察"，只写有判断的标的 |
| **5. 利润预估与增长逻辑** | 表格（标的/代码/2026E/2027E/增速/利润弹性来源/大开门状态/关键变量）+ 每个标的利润预估逻辑 | **不写目标市值、不写买入卖出价位、不写潜在空间** |
| **6. 风险警示** | 风险事件 / 对核心判断的影响 / 修正方案 | 不写泛泛的"风险提示"，必须写"如果XX错了，我怎么做" |
| **7. 附录** | 价格走势、产能数据、跟踪指标清单（P0/P1/P2） | 数据来源标注 [^n] |
| **8. 尾部声明** | 信息来源说明 + 非独立调研声明 + 判断错误风险提示 | — |

### 9.2 估值红线（不可违反）

**只做利润预估，不做市值预测**：

| ✅ 可以做 | ❌ 不做 |
|:---|:---|
| 2026E/2027E利润预估 | 目标市值 |
| 利润弹性来源分析 | 买入/卖出价位 |
| 大开门状态判定（PE≤10x/15x/20x） | 潜在上涨空间 |
| 关键变量对利润的影响（如果XX→利润从A→B） | 市值回调预测 |
| 当前前瞻PE（作为参考，不用于预测） | 目标估值倍数 |
| 买入/卖出**区间**（基于利润预估+市场定价，但明确说明是"用户自行判断"） | — |

**用户原话**："市值算不准，直接给出未来两年的业绩情况就好了，不用给出市值。"

**原因**：市值受情绪/流动性/板块轮动影响，利润预估准确性远高于市值预测。给出利润后，用户自行结合市场定价做决策。

### 9.3 周报文件命名与存放

- **存放**：`investment/05_周报/`
- **命名**：`景气投资深度周报_YYYY第WW周_{主题}_YYYYMMDD.md`
- **示例**：`景气投资深度周报_2026第23周_光通信_20260606.md`

### 9.4 与增量分析的关系

| 场景 | 使用 |
|:---|:---|
| 收到新材料（研报/纪要/公告） | **增量分析**（Step 1-6）→ 追加到主文件/产业跟踪 |
| 周期性输出产业链全景判断 | **周报**（8模块）→ 新建周报文件，引用增量分析的结果 |
| 既有增量分析，又要写周报 | 先执行增量分析（更新存档），再基于更新后的存档写周报 |

**关键**：周报中的利润预估必须来源于已存档的增量分析，不能凭空写。如果存档中缺少某标的的利润预估，周报中标注"待单独深度分析"。
