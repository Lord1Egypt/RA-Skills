## 示例 1：v2.4+ 增强格式（含约束标注、5W2H、核心句）

### 输入

> "这个PPT太难搞了，我想让你为我做个PPT，明天下午3点前必须交，尽量用公司模板，就介绍钛合金马扎产品，我们研发了200多天，开了200次产品会就为了精益求精，他主要成分就是钛合金，和常见的马扎功能一样，但是防水、耐磨、不易损坏，还很轻，你为我做PPT吧。"

### 注意力锚定（内部，不输出）

- `[CRITICAL]` 明天下午3点前必须交 → 🔴 硬约束
- `[CORE]` 做PPT → 核心目标
- `[ENTITY]` 钛合金马扎、200多天、200次、下午3点 → 实体
- `[EXAMPLE]` （无）
- `[RESISTANCE]` PPT太难搞了 → 阻力
- 核心句重构：「用户 要 做PPT 于 明天下午3点前」

### 隐式约束升级（内部，不输出）

- "尽量用公司模板" → 初判🟡 → 上下文复核：属组织规范 → **升级为 🟡🔴**

## 示例 2：v2.3 基础格式（无约束标注）

### 输入

> "这个PPT太难搞了，我想让你为我做个PPT，明天要介绍产品的，就一个钛合金马扎，我们研发了200多天，开了200次产品会就为了精益求精，他主要成分就是钛合金，和常见的马扎功能一样，但是防水、耐磨、不易损坏，承担了这些优势的情况下还很轻，你为我做PPT把。"

### 拆分结果

```
【拆分结果】

## 块 1：用户（"我"）
  角色：requester（需求方）
  目的：明天要介绍产品
  诉求：
    1. 做一个PPT（核心交付物）
    2. 介绍钛合金马扎产品
    3. 研发周期：200多天
    4. 产品会议：200次
    5. 品质追求：精益求精
    6. 主要成分：钛合金
    7. 功能：与常见马扎同质
    8. 优势：防水、耐磨、不易损坏
    9. 附加优势：轻便（同比质量小）
  动机：用户觉得PPT太难做

## 块 2：执行者（"你"）
  角色：executor（执行方）
  目的：制作一个符合用户诉求的PPT
  诉求：
    1. 使用技能或MCP等进行完整的PPT制作
    2. 涵盖产品核心卖点（钛合金材质、轻便耐用）
    3. 体现研发投入（200天、200次会议）
    4. 突出精益求精的品质追求
  动机：块1的所有要求

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【确认询问】
以上拆分是否完整准确？如有遗漏或错误，请补充或纠正。
```
