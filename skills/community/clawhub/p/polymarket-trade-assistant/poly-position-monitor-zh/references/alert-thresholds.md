# 告警阈值指南

## 价格变化阈值

所有价格变化均为**相对变化**（非绝对百分点）。

公式：`change = (current_price / price_N_min_ago) - 1`

示例：价格从 0.50 变为 0.515 = +3% 相对变化。

### 默认阈值

| 时间窗口 | 阈值 | 告警级别 | 说明 |
|----------|------|----------|------|
| 5 分钟  | 3%  | WARNING  | 短期闪动，可能回弹 |
| 15 分钟 | 15% | WARNING  | 显著趋势变化 |
| 60 分钟 | 10% | ALERT    | 持续方向性变动 |
| 240 分钟| 20% | CRITICAL | 重大重定价事件 |

### 调参建议

**更敏感**（活跃交易者、小市场）：
```json
{"5m": 0.02, "15m": 0.05, "60m": 0.07, "240m": 0.15}
```

**更迟钝**（长期持有者、高交易量市场）：
```json
{"5m": 0.05, "15m": 0.20, "60m": 0.15, "240m": 0.30}
```

低价市场（< $0.10）天然有更大的百分比波动，必要时考虑按市场单独设置阈值。

## 成交量阈值

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `volume_spike_ratio` | 2.0 | 当前间隔成交量 > 滚动均值 2 倍时告警 |
| `volume_drop_ratio` | 0.3 | 当前间隔成交量 < 滚动均值 30% 时告警 |

成交量比较以上一检查周期的 `interval_volume_usd` 作为基线。经过多个周期后，形成滚动对比。

## 仓位变化阈值

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `position_change_pct` | 0.20 | 仓位规模变化 > 20% 时告警 |
| `min_inflow_usd` | 1000 | 新仓位价值 > $1,000 时告警 |

## 告警级别

| 级别 | 颜色 | Telegram | 邮件 | 适用场景 |
|------|------|----------|------|----------|
| INFO | 灰色 | 否（默认） | 否 | 状态更新、微小变化 |
| WARNING | 黄色 | 是 | 否（默认） | 中等异常、早期信号 |
| ALERT | 紫色 | 是 | 是 | 需要关注的重要事件 |
| CRITICAL | 红色加粗 | 是 | 是 | 重大事件、可能影响结算 |

每个通知渠道的 `min_level` 可在 config.json 中单独配置。
