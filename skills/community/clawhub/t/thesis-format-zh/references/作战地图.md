# 论文排版作战地图

> 2026-05-09 | 从两次翻车中提取的系统性改进方案

## 一、诊断：慢在哪、错在哪

| 环节 | 当前做法 | 根因 | 技术名称 |
|------|---------|------|---------|
| 读规范 | 人脑记参数 | 靠记忆 | 缺"规范即代码"（spec as code） |
| 写代码 | 每次手写 fix 脚本 | 重复造轮子 | 缺样式注册层（style registry） |
| 应用格式 | 逐 run 设置字体/间距 | 脆弱易漏 | direct formatting 而非 style-based |
| 验证 | 跑完抽查几段 | 没有完整对比 | 缺 format auditor + diff 工具 |

## 二、竞品武器库

### 武器 1：python-docx-template（模板驱动）⭐

**地址**: https://docxtpl.readthedocs.io/

**核心思路**: 让业务人员在 Word 里设计好模板（样式全在里面），开发者只管运行时注入数据。

```python
# 现在（慢，6行动作）
p.runs[0].font.name = "宋体"
p.runs[0].font.size = Pt(12)
p.paragraph_format.line_spacing = 1.5
p.paragraph_format.first_line_indent = Pt(24)
p.paragraph_format.alignment = WD_ALIGN_PARAGRAPH.JUSTIFY
for r in p.runs:  # 还要处理中英文字体...

# 竞品（快，1行）
tpl = DocxTemplate("thesis_template.docx")  # 样式全在模板里
tpl.render({"title": "...", "abstract": "...", "body": "..."})
tpl.save("output.docx")
```

**我们的场景怎么用**: 把河北科技大学模板 .docx 变成 docxtpl 模板，标记好占位符（`{{title}}`, `{{abstract}}`, `{{body}}` 等），格式化=渲染模板。

### 武器 2：Style API（样式注册层）⭐

**地址**: https://python-docx.readthedocs.io/en/latest/user/styles-understanding.html

**核心思路**: Word 有内置样式体系（Normal、Heading 1-9、List Bullet 等），定义一次，全文复用。改样式=改全文。

```python
# 样式注册（做一次，放进 style_registry.py）
style = doc.styles.add_style('论文正文', WD_STYLE_TYPE.PARAGRAPH)
style.font.name = '宋体'
style.font.size = Pt(12)
style.paragraph_format.line_spacing = 1.5
style.paragraph_format.first_line_indent = Pt(24)

# 到处复用（1 行替代 6 行）
p.style = doc.styles['论文正文']
```

**关键概念**: 样式继承链。`基于 Normal → 改不同部分 → 创建子样式`，不用每个样式从头定义。

### 武器 3：Redlines（格式 diff）⭐

**地址**: https://github.com/JSv4/Python-Redlines

**核心思路**: 格式化前后对比，生成 Word Track Changes 标注版，直接看改了什么、漏了什么。

```bash
python3 -m redlines 原始.docx 格式化后.docx 差异报告.docx "AI Auditor"
# 输出：Word 修订模式文档，可直接接受/拒绝每处修改
```

替代我们现在的"跑完脚本肉眼抽查几段"。

### 武器 4：GitHub 开源方案（参考架构）

| 项目 | 关键词（搜 GitHub） | 价值 |
|------|-------------------|------|
| MDPI 投稿格式化 | `docx-template mdpi manuscript-formatting` | Word→期刊格式自动化 |
| Elsevier 投稿脚本 | `docx-template elsevier` | 同上，多期刊适配 |
| Amberg/DocxTemplater | C# 但思路通用 | 占位符/循环/表格/图表/Markdown嵌入 |

### 武器 5：Spire.Doc CompareOptions（审计）

**地址**: https://www.e-iceblue.com/

商业/免费限制版，可以忽略格式差异只看文字变化，或把 tracked changes 提取成文本日志做自动化审计报告。

## 三、改造路线图

```
现在（慢且错）                  →  目标（快且准）
──────────────────────────────────────────────────────
凭记忆读规范                    →  spec_config.yaml（规范即配置）
手写 direct formatting          →  style_registry.py（一次注册）
逐 run 设置                     →  p.style = styles['论文正文']
抽查验证                        →  format_auditor.py + Redlines diff
每次写新脚本                    →  python docxtpl 渲染模板
```

## 四、立即可拆解的 3 个资源

1. **样式体系权威文档**: https://python-docx.readthedocs.io/en/latest/user/styles-understanding.html
   → 看懂 style inheritance（继承链）

2. **模板驱动标准做法**: https://github.com/elapouya/python-docx-template
   → 看怎么把样式锁在 .docx 模板文件里

3. **diff 工具**: https://github.com/JSv4/Python-Redlines
   → 跑完格式化自动生成红线版对比
