# ERRORS — 执行失误

记录事实错误、路由误判、地址输出失败、画像应用偏差等失败事件。

- **写入时机**：用户明确指出"你弄错了"、给出了错误信息、分析结论与事实不符
- **写入格式**：严格按下方模板
- **核心字段**：`失败现象` / `失败原因` / `正确做法` / `如何复现`

---

## 条目模板

```
## [YYYY-MM-DD] <一句话标题（描述错误现象）>
- **area**: <路由规则 / 指数数据 / 大师框架 / 表达风格 / 用户画像>
- **priority**: <high | medium | low>
- **status**: new
- **recurrence-count**: 1

**触发情境**：<用户问了什么、当时在做什么分析>
**失败现象**：<错误的具体表现，如"把中证500路由到了巴菲特框架">
**失败原因**：<根因分析，不只是描述现象>
**正确做法**：<下次遇到同类情境时应该怎么做>
**如何复现**：<最小可复现步骤，便于反思时验证>
```
