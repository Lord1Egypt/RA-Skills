# {项目名} — 轻量工作流

> 适用于简单项目（单文件 HTML、配置文件等）。
> 3 步流程：规划 → 实施+自审 → 交付

---

## 流程

```
规划（说清楚改什么）→ 实施+自审（改完自查）→ 交付（git commit + 报告）
```

### 第 1 步：规划

收到改动需求后，先回答 3 个问题：

1. **改哪里**：涉及哪个/哪些文件？
2. **改什么**：具体要做什么改动？
3. **风险点**：可能影响哪些现有功能？

得到确认后进入第 2 步。

### 第 2 步：实施 + 自审

修改完成后，运行自审 Checklist。

### 第 3 步：交付

- git commit（语义化提交：type: 描述）
- 报告：改了什么、在哪里、效果如何

---

## 变更目录（可选）

对于较大改动：

```
.harness/changes/{type}-{简述}-{日期}/
└── summary.md    # 一句话记录：改了什么、为什么
```

对于小改动，直接 git commit 即可。
