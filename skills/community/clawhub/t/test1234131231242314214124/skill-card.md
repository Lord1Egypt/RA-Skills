## Use Case: <br>
Developers and engineers use this skill to review code changes before submission or after task completion, focusing on correctness, readability, maintainability, tests, and embedded-system concerns. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Code review guidance may produce incorrect or misleading findings. <br>
Mitigation: Treat findings as recommendations and manually verify important issues before changing or merging code. <br>
Risk: Code snippets or diffs submitted for review may contain secrets. <br>
Mitigation: Avoid including secrets in review inputs and redact credentials before sharing code with an agent. <br>
