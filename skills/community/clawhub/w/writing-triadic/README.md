## ✨ 核心理念

传统的 AI 写作就是"你说一句，它写一段"。Writing Triadic 把写作拆成三个角色 + 一个进化引擎：

| 角色 | 中文名 | 职责 |
|---|---|---|
| 🧠 **Creator** | 创作者 / 内容架构师 | 深层挖掘用户意图（4问/轮），智能配方匹配（跨模板融合），驱动进化引擎 |
| ✍️ **Executor** | 执行者 / 精密写手 | 按规则+配方产出 ≥2 版有本质差异的初稿；**v2.6: 支持差异修改（Diff-Mode）+ v3 特征合成** |
| 👁️ **Reader** | 读者 / 灵魂受众 | 代入目标读者身份，自适应 100 分加权评分，选出最佳版本 |
| 🧬 **Evolution Analyst** | 进化分析师 v2 | 每次写作后自动提炼偏好 + 全局统计分析（漂移/否决权/采纳率） |

## v2.6 升级 (2026-05-15)

### Executor 多模态迭代
- **三种修改模式** — Mode A: 差异修改（只改指定段落） / Mode B: v3 特征合成（取两版最优融合） / Mode C: 全量重写

- **变动标注** — 每段 `[保留]` / `[已修改]` / `[新增]` 标注，变更摘要一目了然
- **v3 合成度报告** — v1 基因 X% + v2 基因 X% + 全新 X%

### 守护规则
- Mode A/B 不丢原版本（v1/v2 保持可回溯）
- 同一篇最多 3 轮迭代，超出建议 Mode C 重启
- 修改记录自动纳入进化引擎学习

### 写作类型 × 维度矩阵记忆

```
[商业文案] 宣传语 → "不要硬推，太像广告" → 语调偏软，禁用促销词
[博客文章] 技术分享 → "代码块太多重点不突出" → 3-4技巧上限，代码精简
[毕业论文] 开题报告 → "文献综述太薄" → 每个小节必须有前人不足评述
[朋友圈文案] 日常分享 → "太像工作汇报" → 取消序号、用口语化叙事
...
```

**不同写作类型的偏好相互隔离，不会串味。** 写宣传语学到的规则不会错误应用到技术博客上。

直接说你想写什么：

- "帮我写一篇关于 AI 的博客"

- "帮我写个工作总结"

- "写个朋友圈文案"

### 工作流程

```
用户提需求
    ↓
Phase 0: 读取你的风格进化档案（知道你讨厌什么、喜欢什么）🔒 仅本地
    ↓
Phase 1: 创作者挖掘意图（最多4问/轮，基于历史偏好精问）
    ↓
Phase 1 增强: 🆕 智能配方推荐（跨模板融合方案）
    ↓
Phase 1.5: 联网调研 ✋ 征求同意后执行 + 更新知识库
    ↓
Phase 1.6: SEO 分析（公开发布 + SEO 适用模板时启用）
    ↓
Phase 2: 匹配模板（16选1）→ 制定规则（含历史偏好注入）
    ↓
Phase 3: 执行者产出 ≥2版初稿
    ↓
Phase 4: 读者以人类视角评审 → 选出最佳
    ↓
Phase 5: 呈现给你 → 你的指正即时记忆
    ↓
Phase 5.5: 进化分析师 v2 自动提炼 + 全局统计 🧬
    ↓
    💡 偏好漂移检测 + 否决权审查 
```

## 🛡️ AI 痕迹检测

内置完善的 AI 痕迹避坑指南，涵盖：
- **词汇警戒线**：40+ 中英文高危 AI 词汇
- **结构性模式**：对仗强迫症、结尾升华综合征、破折号滥用
- **内容空洞信号**：端水大师、时间遁词、模糊归因
- 🆕 **中英双语混淆**：英文思维写中文检测 + 中文思维硬译英文检测 + 回译测试法
- **历史禁忌检查**：Reader 评审时优先检查用户历史偏好，触犯额外扣分
