## Description: <br>
Install, verify, and manage OpenClaw memory with integrated daily distillation, weekly synthesis, procedural learning, retention, and cold archiving workflows. <br>

This skill is ready for commercial/non-commercial use. <br>

## Use Case: <br>
Developers and OpenClaw operators use this skill to install and govern a structured agent-memory workspace. It replaces separate OpenCortex and Lesson-Imprint flows with daily distillation, weekly synthesis, retention audit, procedural safeguards, and cold archiving. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Scheduled automation can persist, reorganize, and move private memory files with under-disclosed or contradictory approval boundaries. <br>
Mitigation: Use a test workspace first, run audit and dry-run modes, review generated reports, and back up memory/ before allowing migration or scheduled maintenance. <br>
Risk: Installer output can propose gateway config.patch commands and cron changes that affect OpenClaw memory behavior. <br>
Mitigation: Review each proposed configuration patch and scheduled job before applying it, then run the verification script after changes. <br>
Risk: Optional voice profiling, infrastructure collection, metrics, and git backup can collect or persist sensitive workspace context. <br>
Mitigation: Keep those features disabled unless explicitly needed, and review vault and git-backup settings before enabling them. <br>
