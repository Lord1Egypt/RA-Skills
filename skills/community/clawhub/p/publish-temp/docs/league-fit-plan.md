# 联盟适配 — league-fit 设计文档

## 问题

当前 `/pvp 七夕青鸟 2500` 只返回：

```
在 高级联盟 中未找到 七夕青鸟
```

这个信息太干，用户想知道的是"能用还是不能用"。

## 目标

改为多联盟自动判断，输出每个联盟的适配状态。

## 判断逻辑

### Step 1: 查宝可梦基础数据

从 gamemaster 获取：

- 种族值（atk / def / sta）
- 类型

### Step 2: 对每个联盟判断

#### 超级联盟（CP ≤ 1500）

- 计算最大 CP（IV15/15/15, Lv51）
- 如果最大 CP ≥ 1400 → ✅ 活跃
- 如果最大 CP ≥ 1000 → ⚠️ 可用但较弱
- 如果最大 CP < 1000 → ❌ 不适合

#### 高级联盟（CP ≤ 2500）

- 如果最大 CP ≥ 2300 → ✅ 活跃
- 如果最大 CP ≥ 1800 → ⚠️ 可用
- 如果最大 CP < 1800 → ❌ 不适合

#### 大师联盟

- 如果该宝可梦在 master rankings 中 → ✅ 活跃
- 否则根据 CP 判断 → ⚠️ / ❌

### Step 3: 读取 rankings 确认

即使 CP 允许，也要确认 PvPoke 排名：

- 在 rankings 中 → 显示实际排名
- 不在 rankings 中 → 显示"未进入排名"（但仍标注适配性）

## 输出格式

```
七夕青鸟 联盟适配

超级联盟    ✅ 推荐（PvPokeTW #10）
高级联盟    ⚠️ 较少（未进入排名，CP上限不足）
大师联盟    ❌ 不适合

推荐：超级联盟
```

```
固拉多 联盟适配

超级联盟    ❌ 不适合（CP超标）
高级联盟    ❌ 不适合（CP超标）
大师联盟    ✅ 推荐（PvPokeTW #24）

推荐：大师联盟
```

## 展示方式

- 作为 `/pvp 培养` 命令的补充输出
- 当用户在指定联盟未找到时触发自动跨联盟查询
- 或者在 `/pvp 七夕青鸟 适配` 子命令中独立触发

## 实现位置

- `src/leaguefit.ts` — 联盟适配判断逻辑
- `src/query.ts` — 在未找到时调用跨联盟查询
- `src/index.ts` — 新增路由

## 不做

- 不做 CP 精确计算（复用 fetcher.ts 的 calcCP）
- 不展示 IV 排名以外的百分比
- 不判断非标准联盟（小小杯等）
