## 专利申请任务清单

专利申请按子域组织的任务清单，附任务间关系和元操作映射提示。

**域间逻辑流**：专利准备 → 专利撰写 → 专利申请 → 专利授权

---

### 1 专利准备

| ID | 任务类型 | 说明 | 依赖 | 元操作映射 |
|----|----------|------|------|----------|
| T1-1 | 技术交底 | 收集技术交底书，了解发明创造的技术方案 | 无（入口） | S→C |
| T1-2 | 专利检索 | 检索相关专利文献，评估新颖性和创造性 | T1-1 | S→C |
| T1-3 | 专利分析 | 分析检索结果，确定专利申请的保护范围 | T1-2 | C |
| T1-4 | 申请策略 | 制定专利申请策略，包括申请类型、申请时机、申请地域 | T1-3 | C→A |

---

## 依赖拓扑摘要

### 一、域内链路

**专利准备链路**: T1-1 → T1-2 → T1-3 → T1-4
**专利撰写链路**: T2-1 → T2-2 → T2-3 → T2-4
**专利申请链路**: T3-1 → T3-2 → T3-3 → T3-4
**专利授权链路**: T4-1 → T4-2 → T4-3 → T4-4

### 二、跨域协同链路

**完整专利申请链路**: T1-1 → T1-2 → T1-3 → T1-4 → T2-1 → T2-2 → T2-3 → T2-4 → T3-1 → T3-2 → T3-3 → T3-4 → T4-1 → T4-2 → T4-3 → T4-4

**反馈改进链路**: T3-4 → T2-1（答复审查意见反馈到权利要求书）

**检索驱动链路**: T1-2 → T1-3（专利检索驱动专利分析）

更多组合根据具体任务动态推导。
