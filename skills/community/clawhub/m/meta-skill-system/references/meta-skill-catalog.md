### 依赖拓扑摘要

以下为方法论执行任务的主要依赖链路，执行框架可据此自动编排管线：

#### 一、域内链路

**任务域协调链路**: M0-01 → M0-02 → M0-03 → M0-04
**领域评估链路**: M1-01 → M1-02 → M1-03 → M1-04 → M1-05
**工作流重构链路**: M2-01 → M2-02 → M2-03 → M2-04 → M2-05 → M2-06
**领域负载物生成链路**: M3-01 → M3-02 → M3-03/M3-04/M3-05/M3-06 → M3-07

**执行轴·主链路**: M4-01 → M4-02 → M4-03 → M4-04 → M4-07 → M4-08 → M4-12
**执行轴·守护链路**: M4-01 → M4-06 → M4-02 → M4-06 → M4-03 → M4-06 → M4-04（守护贯穿全程）
**执行轴·交互链路**: M4-01 → M4-05 → M4-02 → M4-05 → M4-03（交互贯穿S/C/A）
**执行轴·三轴流程链路**: M4-09 → M4-10 → M4-11 → M4-07

**内容轴·清单法链路**: M5-01 → M5-03 → M5-04 → M5-05
**内容轴·样本法链路**: M5-02 → M5-06 → M5-07
**内容轴·搜索链路**: M5-08 → M5-03/M5-06（搜索结果用于构建清单或获取样本）

**创新轴·模式直用链路**: M6-01/M6-02/M6-03/M6-04 → M6-05 → M6-06 → M6-07
**创新轴·模式改进链路**: M7-01/M7-02/M7-03/M7-04/M7-05/M7-06/M7-07/M7-08/M7-09/M7-10（各自独立）
**创新轴·模式迁移链路**: M8-01/M8-02/M8-03/M8-04 → M8-05
**创新轴·模式构建链路**: M9-01 → M9-02 → M9-03 → M9-04 → M9-05 → M9-06

#### 二、跨域协同链路

**完整领域处理链路**: M0-01 → M1-01 → M2-01 → M3-01 → M4-01

**评估→重构链路**: M1-05 → M2-01（评估结论决定是否重构）
**重构→负载物链路**: M2-06 → M3-01（重构结果简化负载物生成）
**负载物→执行链路**: M3-07 → M4-01（负载物作为执行框架输入）

**创新→执行链路**: M6-07/M7-10/M8-05/M9-06 → M4-11（创新方案输入三轴分解）
**内容→执行链路**: M5-05/M5-07 → M4-12（内容成品输入整合交付）

**深化路由链路**:
- M0-04 → M1-01（评估深化）
- M0-04 → M2-01（重构深化）
- M0-04 → M3-01（负载物深化）
- M0-04 → M6-01/M7-01/M8-01/M9-01（创新深化）

#### 三、三轴协同链路

**执行轴+内容轴**: M4-07 → M5-01/M5-02 → M4-12
**执行轴+创新轴**: M4-07 → M6-01/M7-01/M8-01/M9-01 → M4-11
**三轴全开**: M4-09 → M6-01/M7-01/M8-01/M9-01 → M4-11 → M5-01/M5-02 → M4-12

#### 四、知识图谱

将依赖拓扑扩展为知识图谱，支持更复杂的任务关系推理：

**实体节点类型**:
- **任务节点**: M0-01~M9-06（70个任务）
- **域节点**: M0~M9（10个域）
- **元操作节点**: S/C/A/O/I/G（6个元操作）
- **领域负载物节点**: 生成的技能

**关系边类型**:
- **依赖边**: 任务间的前后依赖关系（→）
- **组合边**: 任务间的组合关系（⇉/⇇）
- **输入边**: 任务的输入关系
- **输出边**: 任务的输出关系
- **深化边**: 深化路由关系（M0-04 → 各域）
- **包含边**: 域包含任务的关系（M1 → M1-01~M1-05）

**推理能力**:
- **路径推导**: 从起点任务到终点任务的最优路径
- **任务推荐**: 基于当前任务推荐下一步任务
- **依赖分析**: 分析任务的前置依赖和后续影响
- **并行识别**: 识别可并行执行的任务组合

更多组合根据具体任务动态推导。
