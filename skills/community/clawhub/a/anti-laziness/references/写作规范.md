# 写作场景专项规范

> 写作是思想的表达，必须言之有物、逻辑清晰、表达准确

---

## 一、内容创作（写什么）

### 1.1 绝对禁止

| 禁止 | 说明 | 反例 |
|-----|------|------|
| AI 套话 | 正确的废话 | 「随着XX的发展...」「这是一个复杂的问题...」 |
| 空洞总结 | 没有信息量的总结 | 「综上所述...」 |
| 正确的废话 | 说了等于没说 | 「要重视加强...」 |
| 正确的重复 | 同义反复 | 「重要的是...关键在于...核心是...」 |
| 虚假具体 | 假装具体 | 「据不完全统计...」「很多人...」 |

### 1.2 必须做到

1. **开门见山**
   - 第一句就要切入正题
   - 删除所有过渡句

2. **言之有物**
   - 每段至少有一个具体信息点
   - 删除所有「假大空」

3. **观点明确**
   - 立场清晰，不模棱两可
   - 结论明确，不含糊其辞

4. **证据支撑**
   - 观点要有依据
   - 数据要有来源

### 1.3 内容检查清单

- [ ] 第一句是否切入正题？
- [ ] 每段是否有具体信息？
- [ ] 观点是否有支撑？
- [ ] 是否有正确的废话？
- [ ] 是否有空洞的总结？

## 二、结构逻辑（怎么写）

### 2.1 禁止的套路

| 套路 | 说明 | 替代 |
|-----|------|------|
| 三段式 | 现象-原因-对策 | 按逻辑自然分段 |
| 堆砌小标题 | 为形式而拆分 | 逻辑连贯优先 |
| 伪分层 | 强行套用层级 | 实际内容分层 |
| 虎头蛇尾 | 开头详细结尾简略 | 前后呼应 |
| 车轱辘话 | 来回说同一件事 | 一次性说清 |

### 2.2 必须遵循的原则

1. **层次分明**
   - 论点 → 论据 → 结论
   - 每一层都有明确职责

2. **前后呼应**
   - 开头提出的问题
   - 结尾要有回应

3. **逻辑连贯**
   - 段落之间有逻辑连接
   - 跳转要有过渡

4. **详略得当**
   - 重点详细
   - 已知从简

### 2.3 结构检查清单

- [ ] 是否用了三段式套路？
- [ ] 小标题是否为了形式而拆分？
- [ ] 开头和结尾是否呼应？
- [ ] 段落之间是否有逻辑连接？

## 三、表达风格（怎么说话）

### 3.1 禁止的词汇/表达

#### AI 味词汇

| 禁用 | 说明 | 替代 |
|-----|------|------|
| 主打 | 滥用词汇 | 突出、主要 |
| 搭子 | 网络用语 | 伙伴、搭档 |
| GPU干烧 | 网络用语 | 思考、费力 |
| 赋能 | 滥用词汇 | 支持、帮助 |
| 抓手 | 滥用词汇 | 切入点、关键点 |
| 闭环 | 滥用词汇 | 完整、完结 |
| 落地 | 滥用词汇 | 实现、执行 |
| 沉淀 | 滥用词汇 | 积累、存储 |

#### 过度连接词

| 禁用 | 说明 | 替代 |
|-----|------|------|
| 首先/其次/最后 | 机械套用 | 自然过渡 |
| 综上所述 | 空洞总结 | 总结具体内容 |
| 从某种程度上说 | 正确的废话 | 直接说 |
| 有一说一 | 口语 | 删除 |
| 有一说一 | 口语 | 删除 |

#### 无意义反问

| 禁用 | 说明 |
|-----|------|
| 你觉得呢？ | 制造对话感但无意义 |
| 难道不是吗？ | 强加认同 |
| 是不是？ | 无意义确认 |

### 3.2 必须的风格

1. **口语化**
   - 用生活化比喻
   - 用简单直白的词
   - 删除学术腔

2. **准确**
   - 用词精准
   - 不夸张
   - 不夸大

3. **简洁**
   - 能删则删
   - 短句优先
   - 删除修饰

### 3.3 表达检查清单

- [ ] 是否有 AI 味词汇？
- [ ] 是否有过度连接词？
- [ ] 是否有无意义反问？
- [ ] 是否足够口语化？

## 四、自检清单（写作后）

### 4.1 删改类

- [ ] 删除所有「的」字超过3个的冗长句子
- [ ] 删除所有「随着...」「在...背景下」等套话开头
- [ ] 删除所有「综上所述」「总的来看」等空洞总结
- [ ] 删除所有正确的废话

### 4.2 检查类

- [ ] 检查是否有连续3句以上句式相同
- [ ] 检查是否有连续3句以上以同一字开头
- [ ] 检查每段是否都有具体信息点
- [ ] 检查开头和结尾是否呼应

### 4.3 朗读类

- [ ] 朗读一遍，拗口的地方重写
- [ ] 朗读一遍，不自然的地方重写
- [ ] 朗读一遍，删除重复表达

## 五、常见问题修改

### 5.1 开头问题

| 原句 | 修改 |
|-----|------|
| 「随着XX的发展...」 | 直接说事 |
| 「在当前背景下...」 | 直接说事 |
| 「相信大家都有这种感觉...」 | 删除 |
| 「首先...其次...最后...」 | 自然过渡 |

### 5.2 中间问题

| 原句 | 修改 |
|-----|------|
| 「从某种程度上说...」 | 删除或具体说 |
| 「某种意义上...」 | 删除或具体说 |
| 「基本上...」 | 删除或具体说 |
| 「据不完全统计...」 | 删除或给出具体数 |

### 5.3 结尾问题

| 原句 | 修改 |
|-----|------|
| 「综上所述...」 | 总结具体内容 |
| 「总而言之...」 | 总结具体内容 |
| 「希望...」 | 删除或具体说 |
| 「让我们一起...」 | 删除 |

### 5.4 词汇问题

| 原词 | 修改 |
|-----|------|
| 主打 | 突出、主要 |
| 赋能 | 支持、帮助 |
| 抓手 | 关键、切入点 |
| 沉淀 | 积累、存储 |
| 落地 | 实现、执行 |
| 闭环 | 完整、完结 |

## 七、写作流程建议

### 第一步：列提纲

```
主题：
核心观点：
结构：
- 第一点：
- 第二点：
- 第三点：
```

### 第二步：写初稿

- 不要追求完美
- 先写出来
- 不要边写边改

### 第三步：删改

- 先删后改
- 大刀阔斧
- 每句都要问"这句有用吗"

### 第四步：朗读

- 朗读检查
- 拗口必改
- 不自然必改

### 第五步：定稿

- 最终检查
- 确认无误
- 可以交付
