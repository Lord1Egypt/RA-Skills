# NeuroSync: 移動端 FPS 遊戲底層手感系統識別與靈敏度自動化校準工具

NeuroSync 是一個將控制工程理論引入電競手感標定的自動化框架。本工具透過電腦視覺（OpenCV）毫秒級追蹤 iOS 標準化硬體手勢，並採用非線性最小二乘法解耦遊戲引擎黑盒子中的**低速線性常數 ($K_{sys}$)** 與 **高速加速度平方律常數 ($K_{accel}$)**，達成跨設備手感 100% 完美遷移。

---

## 🛠️ 核心實驗運行指南

### 1. iOS 端環境建置與標準化訊號錄製
> ⚠️ **重要限制**：本工具高度依賴 iOS 獨有的物理輔助錄製機制，**不適用於 Android 系統**。

1. 打開 iOS 設備的 **設定 -> 輔助使用 -> 觸控 -> 輔助觸控 (AssistiveTouch/俗稱小白點)**。
2. 進入 **建立新手勢**。
3. 在純黑背景中，利用自訂手勢錄製 **3 段完全水平、滑行距離與滑行速度截然不同** 的滑屏手勢。
   * *注意：請務必確保滑屏軌跡保持在中央，滑動過程絕對不能觸碰到螢幕邊界，否則會造成訊號截斷與速度突變。*
4. 將錄製好的手勢指派給小白點快速鍵。

### 2. 影片上傳與檔名規範
1. 開啟螢幕錄影（建議保持背景乾淨，或啟用本程式自帶的下半部隱私遮罩）。
2. 在同一個跑道下依序觸發這 3 段小白點手勢，將錄製好的影片檔案移至與本程式相同的資料夾下。
3. 請將這 3 部影片嚴格修改命名為： ( 211行左右，變數名稱和影片名稱可一併修改 )
   * `gesture_6_2cm.mp4` (短距離手勢) 
   * `gesture_10_5cm.mp4` (長距離甩槍手勢)
   * `gesture_6_5cm.mp4` (中距離變速手勢)

### 3. DMZ 模式交叉參數全矩陣測試
為了打破參數間的共線性（Collinearity），我們必須建立具備高正交性的相異測試矩陣。請進入遊戲的 **DMZ 模式或自由訓練場**，保持視角水平，分別在以下 **4 種參數組合** 下，依序重複調用上述 3 種手勢（共 12 筆交叉測資）：

| 測試組別 | 靈敏度值 (S) | 加速度值 (A) | 對應影片循環 | 實測轉動角度差 ($\Delta\theta$) |
| :--- | :---: | :---: | :---: | :--- |
| **組合一** | 130 | 130 | 影片 1, 2, 3 | 記錄遊戲中實際旋轉的角度度數 |
| **組合二** | 130 | 65 | 影片 1, 2, 3 | 記錄遊戲中實際旋轉的角度度數 |
| **組合三** | 65 | 65 | 影片 1, 2, 3 | 記錄遊戲中實際旋轉的角度度數 |
| **組合四** | 65 | 130 | 影片 1, 2, 3 | 記錄遊戲中實際旋轉的角度度數 |

將你在 DMZ 模式量測到的 12 筆端點角度差，同步輸入到 `neurosync_calibration.py` 中的 `real_angles` 陣列中即可。

## 📊 物理換算報告輸出說明

執行主程式後，系統除了會自動完成大一統微積分擬合，還會依據強制物理定錨基準（$K_{sys} = 1.0$ / $K_{accel} = 0.003$）為使用者印出直觀的手感報告：
1. **低速平穩跟槍手感**：手指在螢幕上滑動 1 公分，畫面將穩定轉動多少度。
2. **高速暴力甩槍手感**：爆發流速下，加速度平方律提供了百分之多少的甩頭角動能增幅。
3. **觸控層細緻靈敏度解析度**：手指滑過最小物理刻度 1 像素（Pixel）時，遊戲視角產生無法分割的極微小位移度數（$\text{deg/pixel}$）。
