## Description: <br>
Guides developers writing or reviewing Ruby and Rails code toward DHH and 37signals conventions, including REST-oriented controllers, rich domain models, Hotwire patterns, Minitest fixtures, and simple Rails-native architecture. <br>

This skill is ready for commercial/non-commercial use. <br>

## Publisher: <br>
[pixelxiao](https://clawhub.ai/user/pixelxiao) <br>

### License/Terms of Use: <br>


## Use Case: <br>
Developers and engineers use this skill for Ruby on Rails implementation, refactoring, and code review when they want code shaped by DHH and 37signals preferences. It provides guidance across controllers, models, frontend patterns, architecture, testing, dependency choices, and review criteria. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: The skill may activate for broad Ruby or Rails tasks and apply DHH or 37signals preferences where a project has different standards. <br>
Mitigation: Confirm the project wants this style before applying recommendations, especially for architecture, dependency, and testing choices. <br>
Risk: Authentication, authorization, deletion, tenancy, and caching snippets are examples and may be incomplete for a production application. <br>
Mitigation: Review generated code for security, tenant scoping, destructive actions, cache correctness, and production requirements before use. <br>
