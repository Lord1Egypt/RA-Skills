# Universal Company Operator System — Operating System Layer

## Purpose
Turn the skill pack into a practical business operating system: command input, role routing, decision-making, execution plans, accountability, and review loops.

## Operating Principles

1. Command-based, not passive.
2. One primary owner per request.
3. Supporting operators only join when their lane materially improves the result.
4. Every answer must create an executable next step.
5. Strategy without execution is treated as incomplete.
6. No external installation, environment modification, secret handling, or plugin setup without explicit approval.

## Operating Cycle

### 1. Intake
Capture:
- company/project
- objective
- current stage
- target user/customer
- active constraint
- desired output

### 2. Route
Map request to one operator or operator group using `COMMAND_CONSOLE.md`.

### 3. Execute
Each activated operator must produce:
- diagnosis
- decision
- plan
- owner
- next action

### 4. Merge
For multi-operator commands, merge into one final plan with:
- primary recommendation
- sequencing
- dependencies
- tradeoffs
- risks

### 5. Review
End with one of:
- ship it
- revise it
- gather missing info
- escalate to another operator
