# 内容生产 (Content Production)

> "Give so much value that it feels like theft." — Alex Hormozi

## 内容生产的三种模式

### 模式 A：价值输出型（Alex Hormozi 式）
- **核心**：每条内容必须给观众一个可执行的工具/框架/洞察
- **节奏**：「我在3分钟里，给你一个能立刻用的东西」
- **例子**：「做自媒体的5个坑，第3个90%的人都会犯」
- **适合**：商业/成长/教育类内容
- **你的使用场景**：产品经理视角的干货拆解

### 模式 B：故事叙事型（Dan Koe 式）
- **核心**：用个人故事传递价值观和思考
- **节奏**：「我来告诉你一个关于...的故事」
- **例子**：「为什么我放弃高薪选择做自由职业」
- **适合**：个人品牌/生活类内容
- **你的使用场景**：职场转型叙事

### 模式 C：体验/挑战型（MrBeast 式）
- **核心**：创造值得观看的体验
- **节奏**：「你绝对想不到接下来会发生什么」
- **例子**：「一天之内从0粉丝涨到1000」
- **适合**：短视频/娱乐类内容
- **你的使用场景**：涨粉挑战/实验

## 脚本写作模板

### 15秒短视频脚本（抖音/小红书/Reels）

```
【前3秒】Hook: 一句话让人停下来
【2-8秒】痛点/问题: 你知道他们说的那个XXX吗？
【8-13秒】解决方案: 但实际不是那样的，真相是...
【13-15秒】Call to Action: 关注我，下期讲...
```

### 10分钟长视频脚本（B站/YouTube）

```
【0-30秒】Hook: 让观众决定不划走
【30-90秒】预告: "今天我会讲3个方法，其中第2个..."
【90秒-8分】主体: 3-5个模块，每个模块一个核心观点+案例
【8-9分】总结: 三个要点回顾
【9-10分】CTA: "关注我，下周出..."
```

## Justin Welsh 的内容生产系统

他最出名的是 **"One-to-Many" 内容复用系统**：

```
一条核心内容（长文/LinkedIn长帖/YouTube视频）
       ↓
拆成5-10条Twitter/Threads
       ↓
再浓缩成1-2条LinkedIn帖
       ↓
再降到小红书图文
       ↓
再拆成短视频脚本
```

**一条内容，5-7个平台，10倍曝光**。

## 发布频率建议

| 阶段 | 最低频率 | 理想频率 |
|------|---------|---------|
| 0-3个月 | 每周3条 | 每天1条 |
| 3-6个月 | 每周2条 | 每周3-4条 |
| 6个月+ | 每周1条 | 每周2-3条 |

**关键**：前3个月别管质量，保证输出。量变引起质变。
