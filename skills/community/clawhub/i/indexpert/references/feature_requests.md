# FEATURE_REQUESTS — 扩展需求

记录用户提出的"还能不能…"、"要是能 X 就好了"、"下次能不能…"类期望。反思模式聚合后，转为 Skill 能力建设的方向建议。

- **写入时机**：用户表达了当前 Skill 无法满足的需求，但不属于当次对话的紧急任务
- **是否在边界内**：如新增大师、新增指数数据 → 可以实现；如接实时行情 API → 当前边界外

---

## 条目模板

```
## [YYYY-MM-DD] <一句话标题>
- **area**: <路由规则 / 指数数据 / 大师框架 / 表达风格 / 用户画像 / 其他>
- **priority**: <high | medium | low>
- **status**: new

**触发情境**：<用户说了什么、在什么对话场景下提出>
**具体期望**：<用户希望 Skill 能做什么>
**是否在当前能力边界内**：<是 / 否 / 部分可行>
**初步评估**：<实现难度、需要改哪些文件>
```
