# 反模式与常见错误

## AP-01: 超大数据量导致性能下降

**错误做法：**
直接将 scan_rate 设得很大（如 > 500 pts/min），或用超长时间范围（如 60 min × 500 pts/min = 30000 点），导致生成缓慢、图片文件过大。

**正确做法：**
- 短时（5-10 min）用 80-120 pts/min
- 长时（30+ min）用 50-70 pts/min
- 总点数超过 20000 时可适当降低 scan_rate 或缩短时间窗口
- 先使用推荐表确认合理参数范围

**深层原因：**
扫描速率越高，生成的信号数组越大，Matplotlib 渲染时间呈线性增长。大图片文件也不便于分享和预览。

---

## AP-02: 峰重叠导致标注混乱

**错误做法：**
将多个峰故意设为相同的 RT（保留时间）以制造重叠，或者峰间距极小（< 0.05 min），导致标注完全重叠不可读。

**正确做法：**
- 单峰之间的 RT 差建议 ≥ HWHM × 2
- 需要紧凑排列时改用 `cluster`（簇峰，各子峰独立标注）或 `merged`（融峰，单一标注）类型
- 簇峰的子峰 RT 间隔建议 0.2-0.5 min
- 可用 `annotate: false` 跳过不需要标注的峰

**深层原因：**
碰撞避让算法将 1 min 内的峰分为一组进行分布排列，但极端重叠（RT差 < 0.05 min）会让标注文本物理上无法避免重叠。

---

## AP-03: 负峰配置不正确

**错误做法：**
将负峰的 HWHM 设为负数，或者试图用 baseline 为负来实现倒峰。

**正确做法：**
- 负峰的 `height` 设为负数，`HWHM` 仍为正数
- 示例：`{"name": "负峰", "RT": 8.0, "height": -1200, "HWHM": 0.1}`
- baseline 也可设为 0 配合负峰使用

**深层原因：**
高斯峰公式中 height 控制信号幅度的方向（正/负），而 HWHM 永远是半高半宽（正数）。baseline 是整体偏移量，不受峰方向影响。

---

## AP-04: 忽略点数推荐导致图形锯齿

**错误做法：**
不调整 scan_rate 就使用默认值，或配置的 scan_rate 太低（< 50 pts/min），导致峰形严重锯齿状、不美观。

**正确做法：**
- 使用 `--show-recommendations` 查看推荐表
- 或使用交互式模式，按推荐扫描速率操作
- 确保总点数 ≥ 500（脚本会自动对此设下限）
- 峰越尖锐（HWHM 越小），所需点数越多

**深层原因：**
scan_rate 决定时间轴分辨率。低扫描速率时，尖锐峰的采样点不足，高斯曲线被"截断"为少数离散点，呈现锯齿。

---

## AP-05: CSV 列索引配置错误

**错误做法：**
导入 CSV 数据时不检查其列结构，直接用默认的 x_col=0, y_col=1 导致数据错乱。

**正确做法：**
- 先查看 CSV 文件的前几行，确认列顺序
- 如果 X 数据在第 2 列，Y 数据在第 3 列，配置 x_col=1, y_col=2
- 如果 CSV 文件没有表头，使用 `--no-header` 参数
- 脚本会打印检测到的列名，用来确认配置是否正确

**深层原因：**
CSV 导入默认假设 X 在第一列、Y 在第二列，但设备导出的 CSV 列顺序不同。列索引从 0 开始计数。

---

## AP-06: JSON 配置文件编码错误

**错误做法：**
用 Windows 记事本编辑 JSON 配置文件并另存为 UTF-8 with BOM，导致脚本无法解析。

**正确做法：**
- 使用支持纯 UTF-8 的编辑器（VS Code、Sublime 等）
- 保存时选择"UTF-8 without BOM"编码
- 或者直接使用交互式模式生成配置，避免手动编辑

**深层原因：**
Python 的 json 模块不支持 BOM 前缀（\ufeff），会导致 json.JSONDecodeError。BOM 是 Windows 旧版记事本的默认行为。

---

## AP-07: 忽略循环依赖的峰配置

**错误做法：**
在簇峰/融峰的 `peaks` 数组中使用重复的子峰引用，或创建自身引用的子峰配置。

**正确做法：**
- 每个子峰独立定义其 RT/height/HWHM
- 不要尝试在子峰中嵌套另一个复合峰（脚本不支持递归展开）
- 所有子峰参数必须是简单数值（不能是对象或数组）

**深层原因：**
复合峰的子峰只支持简单的高斯峰参数（RT, height, HWHM），不支持嵌套复合。脚本的 generate_composite_peak() 只处理一级展开。

---

## AP-08: 交互模式中断后不保存配置

**错误做法：**
在交互式配置进行到一半时按 Ctrl+C 中断，发现已输入的参数全部丢失，只能重新开始。

**正确做法：**
- 现在 Ctrl+C 会优雅退出并提示，不会报错崩溃
- 在输入之前，先用 JSON 文件预先写好常用配置
- 使用 `--config config.json` 方式运行，避免重复输入

**深层原因：**
交互式配置是一次性的内存操作，参数尚未持久化。脚本会捕获 Ctrl+C 信号并给出友好提示而非直接崩溃。
