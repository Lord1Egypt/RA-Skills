# 三一闭环四问 (Three-One Reflection Protocol)

> 来源：葛军老师教学实践——每道题做完后，让学生自己回答四个问题。
> 这四个问题直接映射三一思维(求本→发散→创造)，是从"被动做题"到"主动建构"的关键闭环。
> v3.0 — 三一闭环四问 + GO-ON策略体系 + 引导式交互 + 计算验证 + 实战校准

---

## 核心理念

做题不是目的，思考才是。每道题做完了，不算完。

葛军老师说：**"琢磨拆解你的思维体系，数学水平自然会提升。"**

三一闭环四问就是让学生"琢磨"自己思维过程的工具。不反思的做题是低效重复，反思后的做题是思维升级。

**四问与三一思维的映射：**

| # | 问题 | 对应三一 | 层次跃迁 |
|---|------|---------|---------|
| ① | 本题的核心原理是什么？ | 一解多题(求本) | 找到"一"的根源 |
| ② | 它是由课本上的哪道"种子"题演化而来的？ | 一解多题(从一到多) | 理解演化路径 |
| ③ | 还能用其他方法解答吗？ | 一题多解(发散) | 拓展思维广度 |
| ④ | 试着变换条件，能不能编出一道新题？ | 一题多变(创造) | 从解题者到探索者 |

**关键原则：不是列出问题让学生自己想，而是引导学生一步步做出答案，再点评、总结、得出结论。**

**引导铁律（实战校准 v3.0）：**
1. **点评后必须追问**——不直接说"你漏了XXX"，而是追问让学生自己发现
2. **每问结束后学生必须有一次修正机会**——AI总结前，让学生先修正自己的回答
3. **GO-ON策略是"引导"不是"演示"**——学生卡住时给策略提示和命名，让学生选方向，而非AI自己出变式
4. **语音伴随全程**——四问的每一步（提问、点评、总结）都应配语音
5. **Q①必须追问关键步骤和子问题关系**——从具体步骤到抽象原理，而非跳过直接问原理

## 三一闭环四问详解

### 问题①：本题的核心原理是什么？

- **本质**：识别题目背后的核心数学原理（"一"的根源），而非仅仅罗列解题步骤
- **目标**：让学生从"知道怎么做"升级到"知道为什么这样做"
- **引导方式**：
  1. AI先问："这道题的核心原理是什么？换句话说，如果只允许用一个数学概念来概括这道题，你选哪个？"
  2. 学生回答后，AI**追问关键步骤和子问题**：
     - 学生只说了原理名称 → "你说核心原理是XX。那我追问：解答中有哪几个关键步骤体现了这个原理？每个步骤分别对应原理的哪个部分？"
     - 学生说了步骤但没上升到原理 → "你列了这些步骤。但我想让你再想一层：这些步骤背后，有没有一个统一的东西把它们串起来？那个东西才是核心原理。"
     - 学生说了原理但太笼统 → "你说核心原理是XX。那我问你：XX里面的哪个具体性质在这道题里起决定性作用？"
     - 学生基本正确 → "你抓住了核心原理。再追问：这个原理在这道题的几个子问题中，分别以什么形式出现？它们是什么关系？"
  3. **学生修正后**，AI再总结：明确核心原理+关键步骤+子问题关系，形成"解题骨架图"

- **层次递进**：
  - 浅层：能列出关键步骤，但说不清背后的原理
  - 中层：能说出核心原理，并将步骤与原理对应
  - 深层：能说出原理在不同子问题中的表现形式，理解因果链

### 问题②：它是由课本上的哪道"种子"题演化而来的？

- **本质**：追溯题目到课本基础题，理解"从一到多"的演化路径
- **目标**：让学生理解复杂题都是由简单题"生长"出来的，降低畏难情绪
- **引导方式**：
  1. AI先问："这道题如果追溯到课本，你能想到哪道最基础的题？换句话说，这道题的'种子'是什么？"
  2. 学生回答后，AI**追问**：
     - 找不到种子题 → "这道题用了XX原理。课本上讲XX原理时，最基础的例题是什么？那道例题和这道题有什么关系？"
     - 种子题找错了（太远）→ "你说的那道题确实用了XX，但和这道题的演化路径不太一样。想想：这道题的核心变换是什么？课本上哪个基础题做了同样的变换？"
     - 找到了但不理解演化 → "你说种子题是XX，对。那从XX到这道题，中间发生了什么变化？条件怎么被'加厚'的？"
     - 基本正确 → "你找到了种子题。再想：从种子题到这道题，经历了哪几步'生长'？每一步加了什么？"
  3. **学生修正后**，AI再总结：种子题→演化路径→当前题，形成"生长链"

- **层次递进**：
  - 浅层：能说出相关知识点，但找不到具体种子题
  - 中层：能找到种子题，但不理解演化路径
  - 深层：能完整描述"种子题→生长过程→当前题"的演化链

### 问题③：还能用其他方法解答吗？

- **本质**：从不同角度解同一道题，拓展思维广度
- **目标**：训练一题多解能力，理解不同方法揭示不同层面
- **引导方式**：
  1. AI先问："这道题，你能想到几种不同的解法？"
  2. 学生回答后，AI**追问**：
     - 只会一种方法 → "你用了XX法。我给你一个提示：如果换个工具呢？比如代数法→几何法，或正推→倒推。试试看？"
     - 两种方法但本质相同 → "你说的两种方法，核心思路是不是一样的？真正的一题多解，不是换符号而是换思路。比如，能不能从几何角度重新看这道题？"
     - 有多种方法但不理解差异 → "你列了3种方法。我问你：每种方法分别揭示了这道题的哪个侧面？哪个方法最'本质'？"
     - 基本正确 → "你已经能用不同方法了。最后一个问题：哪种方法最容易迁移到新题？为什么？"
  3. **学生修正后**，AI再总结：不同方法对比+各方法揭示的侧面+迁移性

- **层次递进**：
  - 浅层：只会一种方法
  - 中层：能想到2种以上方法，但说不清差异
  - 深层：能对比不同方法的本质差异，并判断迁移性

### 问题④：试着变换条件，能不能编出一道新题？

- **本质**：从做题人到出题人，从解题者到探索者
- **目标**：训练一题多变能力，理解问题的本质结构
- **引导方式**：
  1. AI先问："你能把这道题改一改，出一道看起来不同但考法一样的题吗？"
  2. 学生尝试后，AI**不直接出变式**，而是用GO-ON策略命名和引导：
     - 学生只会换数字 → "你换了数字，这叫同构变式。但其实有更狠的招——我给你四个方向，你选一个试试：**Op**换运算、**Ob**换对象、**N**变元素个数、**G**画图。你最想试哪个？"
     - 学生出的题太简单/太复杂 → "你用了XX策略出了这道题，有意思。再试一个方向？比如N策略——原题有3个元素，你的题有几个？试着调整元素个数，看规律是否还成立。"
     - 学生卡住 → "出题卡住了？别急，我用四个方向帮你破局：Op是换运算（加减变乘除），Ob是换对象（集合变函数），N是变个数（3个变n个），G是画图（代数变几何）。你觉得哪个方向最有可能？"
     - 学生想可视化 → "你选G策略？好，把这道题画出来，条件和结论在图上是什么关系？换一个几何情境，能编出类似的题吗？"
  3. **学生修正/再尝试后**，AI再总结：点评学生出的题，指出哪些抓住了本质、哪些偏离了；如果学生确实出不来，AI再给示范性变式，但必须标注用了哪个GO-ON策略

- **层次递进**：
  - 浅层：只会换数字（同构变式）
  - 中层：能改变对象或条件，但保持核心考法（同质变式）
  - 深层：能一般化，提炼出问题的本质结构（抽象变式），从解题者成为探索者

## GO-ON策略体系

> 提出问题的四种基本策略，用于引导第四问"编新题"环节。

当你引导学生变换条件编新题时，如果学生卡住或方向单一，用以下四种策略拓展思路：

### Op策略：四则运算策略

**核心思想**：改变题目中的运算，看结构如何变化。

| 原运算 | 变换方向 | 举例 |
|--------|---------|------|
| 加法 → 乘法 | 运算升级 | a+b=c → ab=c，封闭性问题变开放 |
| 等式 → 不等式 | 弱化条件 | a²=b² → a²≥b²，解集扩大 |
| 加法 → 减法 | 方向反转 | 求和 → 求差，顺序敏感 |
| 离散 → 连续 | 域的推广 | 数列求和 → 积分求面积 |

**引导语**："原题用的是什么运算？把那个运算换一种，问题会变成什么样？"

### Ob策略：对象变化策略

**核心思想**：保持逻辑结构不变，替换数学对象。

| 原对象 | 替换对象 | 举例 |
|--------|---------|------|
| 集合 | 函数定义域 | A=B → f(A)=f(B) |
| 整数 | 实数/复数 | Z → R → C |
| 数列 | 函数 | aₙ → f(n) |
| 三角形 | 多边形 | n=3 → n=k |
| 数值 | 向量/矩阵 | 标量运算 → 线性运算 |

**引导语**："题目里的对象是XXX，如果把XXX换成YYY，但保持条件推导的逻辑不变，题目会怎么变？"

### N策略：元素个数策略

**核心思想**：增减元素个数，观察规律是否成立、如何变化。

| 变化方向 | 操作 | 举例 |
|---------|------|------|
| 减少元素 | n→n-1 | 3个集合相等→2个集合相等，问题简化 |
| 增加元素 | n→n+1 | 2元不等式→3元不等式，对称性是否保持？ |
| 极端化 | n→1 或 n→∞ | 1个元素时退化成什么？无穷时极限是什么？ |
| 一般化 | 固定n→任意n | 3项求和→n项求和，归纳法出场 |

**引导语**："原题有3个元素。如果只有2个，还成立吗？如果有n个呢？规律还在吗？"

### G策略：图策略（图表/图形/图像/示意图）

**核心思想**：将代数关系可视化，或从几何情境出题。

| 图类型 | 用法 | 举例 |
|--------|------|------|
| 数轴 | 标注集合/区间 | 集合相等→数轴上区间重合 |
| 坐标系 | 函数图像 | 不等式→图像上下方区域 |
| 韦恩图 | 集合关系 | A=B→两圆完全重合 |
| 树状图 | 分类讨论 | 分类计数问题 |
| 示意图 | 流程/逻辑 | 解题步骤流程化 |

**引导语**："把这道题画出来，条件和结论在图上是什么关系？换一个几何情境，能编出类似的题吗？"

### GO-ON策略组合使用

四种策略可以组合，不限于单用：

- **Op+Ob**：换运算+换对象 → 结构性变式（如：集合相等→函数值域相等）
- **Ob+N**：换对象+增元素 → 一般化（如：3元对称不等式→n元对称不等式）
- **Op+G**：换运算+画图 → 几何解释（如：代数不等式→面积比较）
- **全组合**：终极一般化 → 这类问题的最一般形式是什么？

**引导语**："你已经换了对象（Ob），那运算（Op）能不能也换？两个一起换，问题会变成什么样？"

## 集成方式

### 在各场景中的触发时机

| 场景 | 何时触发三一闭环四问 | 触发方式与侧重 |
|------|---------------------|--------------|
| **S1 拆解** | 不触发（尚未解题） | — |
| **S2 解题** | 解答完成后 | 引导回答四问，重点①② |
| **S3 多解** | 所有解法对比后 | 重点是③：不同解法拓展思维广度，用GO-ON拓展④ |
| **S4 变式** | 变式链结束后 | 重点是④：用GO-ON策略引导学生自己出变式题 |
| **S5 通法** | 通法提炼后 | 重点是①②：溯源核心原理和种子题，提炼通性通法 |
| **S6 诊错** | 处方给出后 | 重点是①：错误往往源于对核心原理理解偏差 |
| **S7 引导** | 学生独立解出后 | **必须触发完整四问**，四问都需引导式交互 |

## 反思质量的评价标准

| 层次 | 问题①核心原理 | 问题②种子题溯源 | 问题③多解 | 问题④编新题 |
|------|-------------|---------------|---------|-----------|
| **浅层** | 列出步骤但说不清原理 | 能说知识点但找不到种子题 | 只会一种方法 | 只会换数字 |
| **中层** | 能说出原理并对应步骤 | 能找到种子题但说不清演化 | 能想到2种以上方法 | 能改变对象或条件(Ob/Op) |
| **深层** | 能说出原理在不同子问题中的表现形式 | 能完整描述种子题→当前题的演化链 | 能对比不同方法的本质差异+迁移性 | 能一般化+说出适用边界(全GO-ON) |

**AI 行为**：学生回答后，AI 根据层次给出具体化的引导和点评：
- 浅层："你列了很多步骤，但它们背后有没有一个统一的东西？找到那个，才是核心原理。"
- 中层："你找到了核心原理/种子题，这很好。再想想：从种子题到这道题，具体走了哪几步？"
- 深层："你已经能一般化并说出边界了。这就是从解题者到探索者的升级。"

## 与"一听就懂一做就废"专项的衔接

三一闭环四问是根治"一听就懂一做就废"的核心工具：

1. "听懂了"只到了Q①的浅层（知道用了什么原理，但说不清为什么）
2. "会做了"到了Q②③的中层（能溯源种子题、能换方法，理解"从一到多"）
3. "能迁移了"才到Q④的深层（能出题、能创造，从解题者变成探索者）

四问做完，才是从"听懂"→"会做"→"能迁移"的完整闭环。

- **v3.0** 三一对齐升级：①反思三问→三一闭环四问（核心原理/种子题溯源/其他方法/编新题）②四问直接映射三一思维(求本→发散→创造) ③Q①融入原版三问的"关键点"和"子问题"作为追问 ④Q②新增"种子题溯源"——连接"一解"与"多题"的桥梁
- **v2.1** 实战校准：①点评后必须追问让学生自己发现（不直接说"你漏了"）②每问结束后学生必须修正一次 ③GO-ON策略改为引导式（命名策略让学生选方向，不自己出变式）④语音伴随反思全程
- **v2.0** 葛军原版三问 + GO-ON策略体系(Op/Ob/N/G) + 引导式交互
- **v1.0** 初始版本：自创三问 + 集成方式 + 输出模板 + 评价标准
