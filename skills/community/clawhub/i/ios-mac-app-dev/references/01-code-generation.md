# 智能代码生成

> 基于结构化提示词 + 生成-测试-修复闭环的生产级 Swift/SwiftUI 代码生成。

## 代码生成工作流

```
需求分析 → 结构化提示词 → 代码生成 → 单元测试 → 测试通过？→ 代码审查 → 合并
                                                ↓ 否
                                           差异修复 → 重新生成
```

## 结构化提示词框架

使用四层结构：

```
┌─────────────────────────────────────────────┐
│  【技术栈层】语言版本、框架、最低部署目标     │
│  例：Swift 6.0, SwiftUI, iOS 16+           │
├─────────────────────────────────────────────┤
│  【功能层】核心业务逻辑描述                   │
│  例：JWT令牌生成与用户认证                   │
├─────────────────────────────────────────────┤
│  【约束层】性能指标、安全要求、边界条件        │
│  例：QPS≥5000, 防止SQL注入, 事务一致性       │
├─────────────────────────────────────────────┤
│  【格式层】输出结构、命名规范、设计模式         │
│  例：MVVM架构, 使用依赖注入                  │
└─────────────────────────────────────────────┘
```

### 模板

**SwiftUI 视图生成：**
> 你是有5年SwiftUI开发经验的iOS专家。
> 技术栈：Swift 6.0, SwiftUI, iOS 16+, Observation
> 功能：实现 `{feature_name}`
> 约束：
> - 支持深色模式
> - 支持动态字体
> - 响应时间 < 100ms
> - 使用 Observation 框架
> - 遵循 MVVM 架构
> 输出：完整的 View + ViewModel 代码

**网络层生成：**
> 技术栈：Swift 6.0, URLSession, async/await
> 功能：实现 `{api_name}` REST API 网络请求层
> 约束：
> - 请求重试（3次）
> - 超时 10s
> - 错误统一处理（Error 枚举）
> - 请求/响应日志
> 输出：Service + Model + Error 枚举

## 必须人工编写的代码

以下代码类型 AI 只生成框架/接口定义，核心逻辑必须人工实现：

| 代码类型 | 原因 | 策略 |
|----------|------|------|
| 支付逻辑 | 业务风险高，错误代价大 | 生成协议接口，逻辑留空 |
| 权限判定 | 安全漏洞不可接受 | 生成安全规范检查清单 |
| 加密/鉴权 | 安全漏洞不可接受 | 生成接口定义 + 安全规范 |
| 核心算法 | AI可能生成低效实现 | 生成性能要求文档 |
| 货币计算 | 精度要求严格 | 生成 Decimal 使用规范 |

## 代码质量门禁

- ✅ 避免紫蓝渐变背景
- ✅ 所有颜色支持深色模式
- ✅ NSLocalizedString 本地化
- ✅ 遵循 Human Interface Guidelines
- ✅ 使用系统字体（SF Pro / PingFang）
- ✅ Dynamic Type 支持

## 多模型协同 (Pro)

三层模型架构：
1. **通用代码模型** — 网络层、数据持久化、UI 布局
2. **iOS领域模型** — 用 iOS/macOS 历史代码微调，SwiftUI 模式、Apple API
3. **项目定制模型** — 结合具体技术栈和业务逻辑
