# PI — 产品集成（CMMI V3.0 Domain-specific PA）

## PA 基本信息
- **缩写**：PI
- **英文**：Product Integration
- **中文**：产品集成
- **所属**：Domain-specific PA（领域特定实践域）
- **适用领域**：硬件工程、软件工程、系统集成
- **对应 V2.0**：PI（产品集成）

---

## CMMI V3.0 要求要点

### 核心目的
将产品组件组装成完整产品，确保产品作为一个整体正确运行。

### 关键实践（Specific Practices）
| 实践编号 | 实践内容 | 说明 |
|---------|---------|------|
| PI 1.1 | 制定产品集成策略 | 定义集成顺序、接口兼容性检查策略 |
| PI 1.2 | 确保接口兼容性 | 验证接口定义与实现的一致性 |
| PI 1.3 | 组装产品组件 | 按计划组装组件 |
| PI 1.4 | 评估组装的组件 | 验证组装后的组件功能 |
| PI 2.1 | 管理产品集成风险 | 识别并管理集成风险 |
| PI 2.2 | 建立产品集成基线 | 控制集成版本 |

### 通用实践（Generic Practices）
- GP 1.1 执行特定实践
- GP 2.1 建立组织政策
- GP 2.2 策划过程
- GP 2.3 提供资源
- GP 2.4 分配责任
- GP 2.5 培训人员
- GP 2.6 管理配置
- GP 2.7 识别并记录干系人需求和接口
- GP 2.8 监督和控制过程
- GP 2.9 客观评估符合性
- GP 2.10 高层审查

## 用户文档 → PA 匹配规则

### 高概率匹配（主要PA）
- 文档名含"集成"、"Integration"、"系统集成" → **PI 主要匹配**
- 文档含集成计划、集成测试报告 → **PI 主要匹配**

### 中概率匹配（次要PA）
- 文档含接口定义 → 同时匹配 **II**（接口管理）
- 文档含测试用例 → 同时匹配 **VV**（验证与确认）

## 评估访谈常见问题（模式D）

### 针对技术人员（开发/测试）的问题
1. "你们的产品是如何集成的？有集成计划吗？"
2. "如何保证接口兼容性？有接口测试吗？"
3. "集成过程中遇到过什么问题？如何解决的？"
4. "集成测试是如何进行的？有测试报告吗？"

### 评分要点
- **PI 1.1-1.2**：是否有系统化的产品集成策略和接口兼容性检查
- **PI 1.3-1.4**：产品集成是否按计划实施，是否有测试记录
- **PI 2.1-2.2**：集成风险是否有管理，是否有集成基线控制
