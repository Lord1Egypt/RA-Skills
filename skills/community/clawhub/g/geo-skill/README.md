## 简介

### 什么是GEO

**GEO（Generative Engine Optimization，生成式引擎优化）** 是一套帮助AI系统（如DeepSeek、豆包、Kimi等）准确理解、信任并优先推荐您内容的优化技术。

### 为什么需要GEO.SKILL

```
┌────────────────────────────────────────────────────────────────┐
│                        AI搜索 vs 传统搜索                        │
├────────────────────────────────────────────────────────────────┤
│                                                                 │
│   传统搜索 (Google/Baidu)         AI搜索 (DeepSeek/豆包)         │
│   ┌─────────────────┐            ┌─────────────────┐           │
│   │ 返回10条网页    │            │ 直接给出答案    │           │
│   │ 用户自己判断    │     →      │ AI替你整合信息  │           │
│   │ 需点击访问      │            │ 来源可追溯      │           │
│   └─────────────────┘            └─────────────────┘           │
│                                                                 │
│   园区面临的挑战：                                                │
│   • AI搜索时园区信息可能缺失或被遗漏                            │
│   • 官网缺乏结构化数据，AI难以解析                               │
│   • 内容碎片化，无法形成可信答案                                 │
│   • 竞品园区在AI中曝光度更高                                     │
│                                                                 │
└────────────────────────────────────────────────────────────────┘
```

## 快速开始

### 第一步：准备园区信息

在开始之前，请准备以下信息：

```yaml
必填信息:
  - 园区官方名称
  - 园区简称/别名
  - 详细地址
  - 联系电话
  - 官方邮箱
  - 官网URL
  
可选信息:
  - 产业定位
  - 核心优势
  - 产品类型
  - 价格区间
  - 优惠政策
```

### 第二步：生成llms.txt

1. 打开 `templates/llms.txt` 模板
2. 根据模板格式填写园区信息
3. 将文件上传至网站根目录
4. 访问 `https://yourpark.com/llms.txt` 验证

### 第三步：部署Schema

1. 选择需要的Schema类型：
   - **Organization** - 园区基本信息
   - **Product** - 厂房/办公室产品
   - **Event** - 招商活动

2. 在 `templates/schema/` 目录找到对应模板
3. 填写JSON-LD参数
4. 将代码添加到页面 `<head>` 区域

### 第四步：生产内容

1. 使用**问题图谱生成器**创建核心问题列表
2. 使用**倒金字塔写作模板**生产内容
3. 添加表格、清单等结构化元素
4. 在多平台分发

### 第五步：启动监控

1. 配置核心监控关键词
2. 设置周/月监控报告
3. 建立问题修复流程

## 使用指南

### 模板使用

#### llms.txt模板

```markdown
# [园区名称]

## 关于我们
[园区定位、核心产业、地理优势描述]

## 核心产业
- [产业1]
- [产业2]
- [产业3]

## 产品类型
- 独栋办公楼
- 标准厂房
- 定制化空间

## 联系方式
- 地址：[详细地址]
- 电话：[联系电话]
- 邮箱：[邮箱]
- 官网：[URL]

## 最新动态
[最近3个月的重要更新]

## 数据摘要
- 占地面积：[X]亩
- 入驻企业：[X]家
- 年产值：[X]亿元
```

#### Schema模板

根据您的需求选择对应模板：
- `organization.json` - 园区整体信息
- `product.json` - 单个厂房/办公室
- `event.json` - 招商活动

#### 问题图谱模板

自动生成50-100个核心问题，覆盖：
- 基础认知问题
- 招商入驻问题
- 政策优惠问题
- 配套设施问题
- 区域竞品问题

#### 倒金字塔写作模板

```
## [核心问题]

[直接答案 - 40-60字，必须包含关键数据]

[详细说明 - 100-200字，补充背景]

### 关键参数
| 参数 | 数值 | 说明 |
|------|------|------|

### 申请条件
- 条件1
- 条件2

### 相关问题
- [关联问题1]
- [关联问题2]
```

### 脚本使用

#### llms.txt生成器

```bash
python scripts/generator.py --input data.json --output llms.txt
```

#### 诊断工具

```bash
python scripts/diagnostic.py --url https://yourpark.com
```

#### 监控工具

```bash
python scripts/monitor.py --keywords "苏州生物医药园,厂房出租"
```

## 最佳实践

### ✅ 推荐做法

| 实践 | 说明 |
|------|------|
| 定期更新 | llms.txt建议每月更新一次 |
| 数据精确 | 所有数字使用精确值，避免"大概" |
| 一致性 | 联系方式保持全网一致 |
| 答案前置 | 核心答案放在段落开头 |
| 主动分发 | 在知乎等平台主动回答问题 |
| 持续监控 | 建立周/月监控机制 |

### ❌ 避免做法

| 实践 | 说明 |
|------|------|
| 信息过时 | 避免使用过期租金和政策 |
| 模糊描述 | 避免"环境优美"等空洞词 |
| 主观对比 | 避免与其他园区主观比较 |
| 信息矛盾 | 避免多平台发布不同信息 |
| 忽视反馈 | 忽视用户真实问题 |

### 关键指标

| 指标 | 目标值 | 测量频率 |
|------|--------|----------|
| AI引用率 | >80% | 每周 |
| 信息准确率 | >95% | 每月 |
| 前3排名率 | >60% | 每周 |
| 内容覆盖率 | >90% | 每季度 |

## 常见问题

### Q1: llms.txt和robots.txt有什么区别？

| 对比项 | robots.txt | llms.txt |
|--------|------------|----------|
| 目标用户 | 搜索引擎爬虫 | AI系统 |
| 内容 | 爬取规则 | 内容摘要 |
| 位置 | 网站根目录 | 网站根目录 |
| 格式 | TXT | Markdown |

### Q2: Schema和llms.txt必须同时部署吗？

建议同时部署，二者互补：
- **llms.txt** 提供整体概述，适合AI快速了解
- **Schema** 提供结构化数据，适合精确信息提取

### Q3: 多久需要更新一次内容？

| 内容类型 | 更新频率 |
|----------|----------|
| llms.txt | 每月 |
| Schema产品 | 按实际变化 |
| 招商文章 | 每周 |
| 活动信息 | 活动前2周 |
| 政策解读 | 政策发布后1周内 |

### Q4: 如何验证GEO效果？

1. 在DeepSeek/豆包/Kimi搜索核心关键词
2. 检查园区信息是否出现
3. 验证信息是否准确
4. 追踪排名变化

### Q5: 遇到负面信息怎么处理？

1. 评估问题严重程度
2. 制定修复方案
3. 更新llms.txt和内容
4. 发布正面内容
5. 持续监控效果
