# 📝 古典智慧实战复盘库 (Practical Wisdom Case Log)
版本: v1.0
用途: 实现 `apply-china-classics` 的自进化闭环。记录【方案 $\rightarrow$ 执行 $\rightarrow$ 结果 $\rightarrow$ 修正】的全过程。

---

## 📂 复盘记录格式 (Case Entry Template)

每条复盘记录必须包含以下字段：

- **Case-ID**: `[YYYYMMDD-S-01]` (日期-场景类别-序号)
- **场景描述**: 简述当时面对的具体困境与资源状态。
- **应用心法**: 记录使用了哪个【战略】+【战术】+【现代映射行为】。
- **执行结果**: $\mathbf{Success\ / \ Partial\ / \ Fail}$ (及具体表现)。
- **偏差分析**: 
  - 预期结果与实际结果的差异点。
  - 是否是因为【时位分析】判定失误？还是【现代映射】在具体环境下不适用？
- **修正法则 (New Rule)**: 
  - 总结出一条可复用的“微心法”。
  - 格式：`在 [特定条件] 下，使用 [计谋X] 需注意 [修正点Y]，否则会导致 [后果Z]`。

---

## 🛠️ 自进化更新协议 (Evolution Protocol)

当用户提供反馈时，AI 必须执行以下流程：
1. **录入复盘**: 按照上述模板将案例写入 `KNOWLEDGE/case-log.md`。
2. **更新映射**: 如果发现 `modern-mapping.md` 中的行为在实际中失效，应在映射表下方增加 `[实战修正]` 备注。
3. **优化索引**: 如果出现一个全新的高频场景，应在 `a-list-classics.md` 中新增场景标签。
