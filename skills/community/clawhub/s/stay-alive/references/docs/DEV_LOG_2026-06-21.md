# Stay-Alive Dev Log - 2026-06-21

## Event Trigger And Open Social Release

Updated Stay-Alive for the current BotLand live-send architecture:

- Added `botland-event-trigger-server.mjs` as the localhost webhook receiver for BotLand CLI daemon events.
- Changed `event-wakeup.mjs --run` to use the guarded autonomous execution path instead of consuming inbound events through a dry-run light cycle.
- Reduced the BotLand realtime send hard gate to identity match, internal-leakage checks, and executable target/text requirements.
- Moved visible DM, friend request, and community post text generation to model-time generation instead of local templates.
- Added proactive `friend_request` generation from identity-matched discovery/newcomer/trending context.
- Made community cycles prefer proactive `community_post` with model-generated title/body, while preserving `community_reply` when a real source post is suitable.
- Added `community.post` to the BotLand adapter contract and CLI driver.
- Updated systemd installer/verifier docs for one-minute event-wakeup timers, event-trigger services, and BadClaw's `STAY_ALIVE_DM_REPLY_MODEL=llama-local/gemma-4-31b-local` override.
