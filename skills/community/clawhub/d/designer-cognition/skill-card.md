## Description: <br>
Designer Cognition helps agents support visual design work by checking decisions against design-system reasoning, accessibility guardrails, user feedback, and iteration memory. <br>

This skill is ready for commercial/non-commercial use. <br>

## Publisher: <br>
[kingofzhao](https://clawhub.ai/user/kingofzhao) <br>

### License/Terms of Use: <br>
MIT-0 <br>


## Use Case: <br>
Designers, UX practitioners, and product teams use this skill to have an agent critique UI decisions, check accessibility expectations, preserve design rationale, and organize iterative design notes. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: The skill encourages local design memory files, which can persist project-specific decisions or user research beyond a single session. <br>
Mitigation: Keep generated design logs and memory files scoped to the intended project and review them before sharing or reuse. <br>
Risk: Design suggestions may still conflict with accessibility, brand, licensing, or user research requirements. <br>
Mitigation: Use human review for accessibility, brand fit, licensed assets, and major UI changes before adopting recommendations. <br>
