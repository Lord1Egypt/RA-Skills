# Job Screener Engine — 通用岗位筛选与评分引擎

[![ClawHub](https://img.shields.io/badge/ClawHub-发布-4B44C0)](https://clawhub.ai/CalmDownTR/job-screener-engine)

对用户提供的岗位机会进行结构化评估，输出多维量化评分和行动建议。

---

## 这是什么

一个通用 AI Agent 技能（Skill），帮助求职者理性评估工作机会。适用于支持 `npx skills` 标准的各类 AI 编程助手。通过 6 个维度对岗位进行量化评分（0-100 分），并给出投递建议。

**核心设计理念：**
- **量化代替直觉**：每个评分维度都有明确标准，减少主观偏见
- **一票否决**：不满足底线的岗位直接标记，节省精力
- **面试/入职解耦**：即使不建议入职，也可能值得面试练手
- **信息透明**：信息不足的维度诚实标注，不猜测不编造

## 何时使用

初期筛选阶段建议依赖个人快速判断，以保持投递效率；获得面试邀请或 Offer 后再用本引擎进行深度评估，此时投入的每一分钟都对应一个真实推进中的机会，投入产出比更高。

## 安装

```bash
npx skills add CalmDownTR/job-screener-engine
```

安装后自动配置为 AI 可用技能。

## 首次使用（必须配置）

第一次使用前，需要填写个人求职信息。有两种方式：

### 方式一：AI 引导配置（推荐）

告诉 AI：

> "帮我用 job-screener-engine 看看这个岗位"

AI 会自动检测到没有 `user_profile.md`，然后逐一提问（薪资底线、技术方向、公司偏好等），完成后自动生成配置文件。

### 方式二：自行填写

```bash
cp references/user_profile.TEMPLATE.md references/user_profile.md
# 然后编辑所有 [占位符]
```

## 使用

配置完成后，直接告诉 AI：

| 你说 | AI 会做 |
|------|---------|
| "帮我看下这家的岗位" + 公司名、岗位名 | 搜索信息 → 评分 → 输出报告 |
| "这个岗位值得投吗" + JD 链接 | 解析 JD → 搜索补充 → 评分 |
| "帮我对比这两个机会" + 岗位信息 | 分别评分 → 同分排序 → 对比输出 |

### 输出示例

```
## 📋 岗位评估 | XX科技有限公司 - 后端开发工程师

### 基本信息
- 薪资：20K-30K
- 公司规模：C轮 / 500人
- 地点：上海
- 信息来源：用户提供 + 搜索补充

### 评分明细
| 维度 | 分值 | 说明 |
|------|------|------|
| 薪资匹配度 | 13/18 | 在期望范围内 |
| 公司规模 | 14/20 | C轮融资，有一定规范度 |
| 技术成长 | 9/15 | 后端技术栈匹配，但方向偏业务 |
| 工作节奏 | 11/15 | 弹性工作，偶有加班 |
| 稳定性 | 14/20 | C轮融资，收入来源清晰 |
| 区域可行性 | 10/12 | 通勤和配套均属合理水平 |
| **总分** | **71/100** | |

### 判定：✅ 推荐投递
### 面试练手建议：中等价值

岗位与自身技术方向部分相关，可投递并了解市场行情。
```

## 自定义配置

所有评估参数通过 `references/user_profile.md` 配置：

| 配置项 | 说明 |
|--------|------|
| Base 底线/期望 | 最低接受与理想薪资 |
| 技术方向 | 核心赛道（Agent/LLM / 后端等） |
| 公司偏好 | 目标公司类型 |
| 工作节奏 | 能接受的强度 |
| 维度权重 | 各维度重要性 |
| 一票否决 | 直接跳过的条件 |
| 同分排序 | 分数相同时的优先级 |
