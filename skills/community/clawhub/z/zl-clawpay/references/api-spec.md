| 接口代码 | 接口名称 | 触发词 | 请求方法 |
| -------- | -------- | -------- | -------- |
| C00003 | 绑定子钱包 | 绑定子钱包、验证钱包凭据 | POST |
| L00001 | 查询子钱包 | 查询子钱包、查询绑定状态 | local |

| C00011 | 撤销绑定 | 撤销钱包绑定、撤销绑定 | POST |

## 0. 前置条件：凭据配置

> ⚠️ **apiKey（SM2 私钥）和 subWalletId（子钱包ID）不通过对话框传递**（会永久留在聊天记录中）。首次使用时通过 **C00003 绑定子钱包** 将凭据加密保存到本地 `~/.zl-claw-pay/state.json`，后续接口自动读取。

| 配置项 | 存储方式 | 说明 |
|--------|----------|------|
| apiKey（SM2 私钥） | 本地加密存储 `~/.zl-claw-pay/state.json` | 证联 APP 获取，仅用于客户端 SM2 签名，不传给服务器 |
| subWalletId（子钱包ID） | 本地加密存储 `~/.zl-claw-pay/state.json` | 证联 APP 获取，用作请求头 appId |

> 🔐 `apiKey` 和 `subWalletId` 使用 AES-256-GCM 加密存储，密钥由机器特征派生。
>
> **seqId vs orgSeqId**：发起支付 (C00009) 响应包含 `seqId`，智能体保存到会话上下文；查询支付 (C00005) 请求使用 `orgSeqId`，传入已保存的 `seqId`。
>
> 🔒 **传输安全**：所有 API 请求必须使用 HTTPS 协议，严禁使用 HTTP。

## 1. 查询子钱包 (L00001)

### 1.1 基本信息

| 项目 | 内容 |
| -------- | -------- |
| **接口代码** | L00001 |
| **请求方法** | local |
| **说明** | 查询当前子钱包绑定状态 |

#### 响应参数

| 参数 | 类型 | 说明 |
|--------| -------- | -------- |
| resCode | string | S010000 表示成功 |
| resMsg | string | 绑定状态描述 |
| subWalletId | string | 子钱包ID（已绑定时返回，未绑定为 null） |
| bindStatus | string | 绑定状态：已绑定/未绑定 |

### 1.2 AI 使用指南

#### 触发词

- "查询子钱包"、"查询绑定状态"、"query wallet"

#### 使用场景

用户想确认当前是否已绑定子钱包。

#### 引导话术

**已绑定**：
> "当前已绑定子钱包，ID：[subWalletId 前4位]***[后4位]，状态：已绑定"

**未绑定**：
> "当前未绑定子钱包。请先通过 C00003 绑定子钱包。"

#### 调用示例

```bash
node {baseDir}/scripts/skill.js call -interfaceId=L00001 -method=local
```

## 2. 解绑子钱包 (L00002)

### 2.1 基本信息

| 项目 | 内容 |
| -------- | -------- |
| **接口代码** | L00002 |
| **请求方法** | local |
| **说明** | 本地解绑子钱包，清除本地凭据（不通知服务端） |

#### 响应参数

| 参数 | 类型 | 说明 |
|--------| -------- | -------- |
| resCode | string | S010000 表示成功，F010001 表示未绑定 |
| resMsg | string | 解绑结果描述 |
| subWalletId | string | 被解绑的子钱包ID |
| unbindStatus | string | 解绑状态：已解绑/解绑失败 |

### 2.2 AI 使用指南

#### 触发词

- "解绑子钱包"、"清除本地凭据"、"unbind wallet"

#### 使用场景

用户想本地清除凭据，但不希望服务端撤销绑定（区别于 C00011 撤销绑定）。

> **注意**：此操作仅清除本地凭据（Memory + StateStore），不会通知服务端。如需彻底撤销绑定，请使用 `C00011` 撤销绑定接口。

#### 引导话术

**解绑成功**：
> "子钱包已解绑，本地凭据已清除。如需继续使用支付功能，请重新绑定。"

**未绑定**：
> "当前未绑定子钱包，无需解绑。"

#### 调用示例

```bash
node {baseDir}/scripts/skill.js call -interfaceId=L00002 -method=local
```

## 3. 绑定子钱包 (C00003)

### 3.1 基本信息

| 项目 | 内容 |
| -------- | -------- |
| **接口代码** | C00003 |
| **接口地址** | /post/claw/bind-sub-wallet |
| **请求方法** | POST |

#### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|--------| -------- | -------- |------|
| apiKey | string | 是 | API Key（64位 hex，SM2 私钥，验证通过后加密保存到本地 state.json） |
| subWalletId | string | 是 | 子钱包 ID（32位 hex，作为 appId 传入报文头） |
| subWalletName | string | 是 | 子钱包名称（AI 从 OpenClaw user.md 自动获取，用于核对智能体名称与 APP 申报名称是否一致） |

#### 响应参数

| 参数 | 类型 | 说明 |
|--------| -------- | -------- |
| subWalletId | string | 子钱包标识符 |

### 3.2 AI 使用指南

#### 触发关键词

- "绑定子钱包"、"验证钱包凭据"、"bind wallet"

#### 使用场景

当用户首次使用 ZLPay 并需要绑定其子钱包时。这是其他功能的前提条件。

#### 引导话术

**步骤 1 - 请求信息**：
> "请提供您的子钱包信息以完成绑定：
> - 子钱包名称：[{subWalletName}]（从您的智能体名称自动获取）
> - API Key（64位 hex，SM2 私钥）：请在证联 APP 中获取
> - 子钱包 ID（32位 hex）：请在证联 APP 中获取"

**步骤 2 - 确认信息**：
> "请确认绑定信息：
> - 子钱包名称：[{subWalletName}]
> - API Key：[已提供]
> 确认绑定？(y/n)"

**绑定成功**：
> "子钱包绑定成功！
> - 钱包名称：[{subWalletName}]
> - 钱包 ID 已加密保存至本地
> 
> 现在可以使用支付、查询订单、查询交易流水等功能。"

**绑定失败**：
> "绑定失败：{resMsg}
> 请检查：
> 1) 子钱包名称是否正确？
> 2) API Key 是否正确且未过期？
> 3) 该钱包是否已被其他账户绑定？"

#### 调用示例

```bash
node {baseDir}/scripts/skill.js call -interfaceId=C00003 -method=POST -endpoint=/post/claw/bind-sub-wallet --apiKey=<64位hex密钥> --subWalletName=<子钱包名称> --subWalletId=<32位hex子钱包ID>
```

> "支付状态查询结果：
> - 状态：[orderStatus]
> - 金额：[amount÷100] 元

## 6. 发起支付 (C00009)

### 6.1 基本信息

| 项目 | 内容 |
| -------- | -------- |
| **接口代码** | C00009 |
| **接口地址** | /post/pay-claw/payment |
| **请求方法** | POST |

#### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|--------| -------- | -------- |------|
| amount | string | 是 | 金额（**分**） |
| merApiKey | string | 是 | 商户技能绑定的 apiKey，用于后续查询和对账 |
| seqId | string | 是 | 商户交易号（流水号），**每个商户必须唯一**，用于对账和查询支付状态 |
| orderDetail | string | 否 | 订单详情（商品名称），最多 256 字符 |

> **注意**：`subWalletId` 自动从本地 state.json 读取，无需传参。

#### 响应参数

| 参数 | 类型 | 说明 |
|--------| -------- | -------- |
| resCode | string | 响应代码，S010000 表示成功 |
| resMsg | string | 响应消息 |
| resData | object | 响应数据对象 |
| resData.seqId | string | 实际使用的交易 ID（系统生成或商户提供） |
| resData.orderStatus | string | 订单状态：成功/处理中/失败 |

### 6.2 AI 使用指南

#### 触发词

- "发起订单支付"、"执行订单付款"、"提交支付订单"、"确认订单支付"、"完成账单支付"、"initiate payment"

#### 使用场景

用户使用商户技能且需要收费时，通过 OpenClaw 协议调用此技能完成支付。

#### 执行逻辑

1. **判断支付意图**：详见 [SKILL.md - 规则0](../SKILL.md#规则0-支付意图判断最重要)
   - 核心原则：仅解析用户原生文本，禁用模糊口语触发词
   - 合法触发：组合式精准指令（如"发起订单支付""执行订单付款"）
   - 判断原则：宁可多问一句，不可误付一笔
2. 确认已绑定子钱包（未绑定则引导用户先执行 C00003）
3. 调用 C00009 执行支付
4. 成功后将 seqId 保存到会话上下文
5. 可通过 seqId 使用 C00005 查询支付状态

> **安全提示**：支付接口禁止自动重试，避免重复扣款。

#### 引导话术

**意图清晰时 — 直接执行并告知**：
> "正在为您支付 [amount÷100] 元给 [商户]，订单：[orderDetail]..."
>
> 支付完成后：
> "支付完成！金额 [amount÷100] 元，交易号：[seqId]，状态：[orderStatus]"
>
> **注意**：智能体必须将 `seqId` 保存到会话上下文，以便后续查询支付状态。

**意图模糊时 — 先确认再执行**：
> "确认一下：您要支付 [amount÷100] 元给 [商户]，订单：[orderDetail]。确认支付吗？"

**商户技能调用时 — 确认后执行**：
> 当商户技能返回支付指令（amount、merApiKey、seqId），智能体先向用户展示订单信息（商户、金额、订单），用户确认后再执行支付。
> 执行后告知："已为您的订单 [orderDetail] 支付 [amount÷100] 元，交易号：[seqId]"

**模糊口语触发时 — 引导使用标准指令**：
> 用户说"我要支付""帮我付款"等模糊口语时，提示："请使用标准支付指令，如'发起订单支付'或'执行订单付款'。"

**未绑定用户提示**：
> "您尚未绑定子钱包，无法发起支付。
> 请先在证联 APP 中创建子钱包，然后通过 C00003 绑定子钱包。
> 绑定需要提供：API Key（SM2 私钥）、子钱包 ID、子钱包名称。"

#### 常见错误码

| 错误码 | 说明 | 处理建议 |
|--------|------|----------|
| `F010001` | 未绑定子钱包 | 提示用户先执行 C00003 绑定子钱包 |
| `F010002` | 余额不足 | 提示用户充值或更换支付方式 |
| `F010003` | 商户不存在或 merApiKey 错误 | 检查 merApiKey 是否正确 |
| `F010004` | seqId 重复 | 生成新的唯一流水号重试 |
| `F010005` | 支付限额超出 | 提示用户在 APP 中调整子钱包支付限额 |
| `F010006` | 子钱包已冻结 | 提示用户联系客服解冻 |
| `F010007` | 网络超时 | 使用 C00005 查询支付状态确认结果，不要自动重试 |

### 6.3 支付参数关联关系

| 参数 | 说明 | 必填 | 来源 |
|--------|------|------|------|
| **amount** | 支付金额（人民币，分） | 是 | 商户技能提供 |
| **merApiKey** | 商户身份验证 | 是 | 商户技能提供 |
| **seqId** | 商户流水号 | 是 | 商户技能提供 |
| **subWalletId** | 子钱包标识，作为 appId | 是 | 本地 state.json 自动读取 |
| **orderStatus** | 支付状态结果 | 是 | 接口返回 |

## 7. 撤销绑定 (C00011)

### 7.1 基本信息

| 项目 | 内容 |
| -------- | -------- |
| **接口代码** | C00011 |
| **接口地址** | /post/pay-claw/unbind-sub-wallet |
| **请求方法** | POST |

#### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|--------| -------- | -------- |------|
| subWalletName | string | 是 | 子钱包名称（AI 从 OpenClaw user.md 自动获取，用于核对智能体名称与 APP 申报名称是否一致） |

> **说明**：只需要传 `subWalletName`，其他凭据自动从本地 state.json 读取。用户在 APP 创建子钱包时已上送此名称，此接口用于核对智能体名称与申报名称是否一致。

#### 响应参数

| 参数 | 类型 | 说明 |
|--------| -------- | -------- |
| unbindStatus | string | 解绑状态：已解绑/解绑失败 |
| resMsg | string | 解绑结果描述 |

### 7.2 AI 使用指南

#### 触发词

- "撤销钱包绑定"、"撤销绑定"、"revoke binding"

#### 使用场景

用户需要更换子钱包或不再使用支付功能。

#### 执行逻辑

1. AI 从 OpenClaw `user.md` 自动获取 `subWalletName`（与 APP 申报时上送的名字一致）
2. **二次确认**：告知操作不可逆（API key 将被永久禁用，本地凭据将被清除）
3. 调用 C00011 执行解绑（核对智能体名称与申报名称是否一致）
4. 成功后自动清除本地所有凭据数据（Memory + StateStore）

> ⚠️ **重要**：此操作不可逆。撤销后 API key 将被永久禁用，本地凭据将被清除。

#### 引导话术

**撤销前确认**：
> "⚠️ 撤销绑定是**不可逆操作**。撤销后，此 API key 将被永久禁用，无法再次使用。
>
> 将撤销子钱包 [{subWalletName}] 的绑定，确认撤销吗？"

**撤销成功**：
> "✅ 子钱包 [{subWalletName}] 的绑定已撤销。此 API key 已被永久禁用，本地凭据已清除。
>
> 如需继续使用支付功能，请先在 APP 中创建子钱包，然后重新绑定。"

**撤销失败**：
> "撤销绑定失败：[错误原因]。请稍后重试。"

#### 常见错误码

| 错误码 | 说明 |
|--------|------|
| `S010101` | API key 错误 |
| `S010102` | 子钱包不存在或已解绑 |
| `S010103` | 存在未完成交易 |
| `S010104` | 子钱包已冻结 |

#### 调用示例

```bash
node {baseDir}/scripts/skill.js call \
  -interfaceId=C00011 \
  -method=POST \
  -endpoint=/post/pay-claw/unbind-sub-wallet \
  --subWalletName=<AI自动获取的智能体名称>
```

> **注意**：`apiKey` 和 `subWalletId` 从本地 state.json 读取，无需传入。`subWalletName` 由 AI 从 user.md 自动获取。
