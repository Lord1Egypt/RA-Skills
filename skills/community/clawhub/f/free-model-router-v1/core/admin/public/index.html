API Key

安装时，系统会自动向 model-server 注册一个唯一的 API Key（存储在 ~/.free-model-router/data/config.json 中）。调用 OpenAI 和 Anthropic 兼容接口时，需要在请求头中携带该 API Key：

Authorization: Bearer <你的 API Key>
  • 可在管理面板概览页面查看 API Key 状态
  • 如果 API Key 显示为「未注册」，请检查网络连接或重启服务以触发重新注册
  • API Key 与设备绑定，请勿泄露给他人

API 端点

格式 方法 路径 说明
OpenAI POST /v1/chat/completions 兼容 OpenAI 格式
Anthropic POST /v1/messages 兼容 Anthropic 格式

模型名称

调用时请使用模型名称 model-router，系统将自动选择当前可用的最优免费模型，无需关心底层具体模型。

OpenAI 格式调用示例

curl -X POST http://127.0.0.1:5678/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <你的 API Key>" \
  -d '{
    "model": "model-router",
    "messages": [{"role": "user", "content": "你好"}],
    "stream": false
  }'

Anthropic 格式调用示例

curl -X POST http://127.0.0.1:5678/v1/messages \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <你的 API Key>" \
  -d '{
    "model": "model-router",
    "messages": [{"role": "user", "content": "你好"}],
    "max_tokens": 1024
  }'

流式响应示例

curl -X POST http://127.0.0.1:5678/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <你的 API Key>" \
  -d '{
    "model": "model-router",
    "messages": [{"role": "user", "content": "讲个故事"}],
    "stream": true
  }'
