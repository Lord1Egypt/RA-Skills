## 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | string | 是 | 唯一标识，格式: req_[随机8位] |
| createdAt | string | 是 | 创建时间，ISO 8601格式 |
| userId | string | 是 | 用户唯一标识 |
| userName | string | 是 | 用户昵称 |
| competition.name | string | 是 | 比赛名称 |
| competition.category | string | 否 | 竞赛分类 |
| role | string | 是 | 角色: 队长/队员/均可 |
| skills | array | 是 | 用户具备的技能标签 |
| skillsNeeded | array | 否 | 需要队友具备的技能 |
| freeTime | string | 否 | 空闲时间描述 |
| contact | string | 是 | 原始联系方式 |
| contactMasked | string | 是 | 遮蔽后的联系方式 |
| teamSize.current | number | 否 | 当前团队人数 |
| teamSize.target | number | 否 | 目标团队人数 |
| deadline | string | 是 | 组队截止日期 |
| status | string | 是 | 状态: active/matched/closed |
| notes | string | 否 | 备注说明 |

## 匹配规则

### 优先级排序

1. **比赛匹配**（必须）
   - 比赛名称完全一致
   
2. **技能互补**（高分优先）
   - 我需要的技能 = 对方具备的技能
   - 对方需要的技能 = 我具备的技能
   
3. **时间匹配**（加分项）
   - 空闲时间有交集

4. **截止日期**（筛选项）
   - 组队截止日期未过

### 匹配评分

```
匹配分 = 比赛匹配(50分) + 技能互补(最多30分) + 时间匹配(最多20分)

技能互补评分：
- 每匹配一项技能 +10分，上限30分

时间匹配评分：
- 完全匹配 +20分
- 部分匹配 +10分
- 无交集 0分
```

### 推荐输出规则

- 每次推荐2条最佳匹配
- 按匹配分从高到低排序
- 同分时优先推荐最新发布的需求
- 匹配分低于30分的不推荐
- 无匹配时输出提示信息

## 隐私保护

### 联系方式遮蔽规则

**手机号**：保留前3后4位
- 原始：13812345678
- 遮蔽：138****5678

**微信号**：保留前3后3位
- 原始：wxid_abc123xyz
- 遮蔽：wxi****xyz
- 原始：abc123（短号）
- 遮蔽：abc***123

**QQ号**：保留前3后2位
- 原始：12345678
- 遮蔽：123***78

**邮箱**：保留前2字符和@后域名
- 原始：zhangsan@qq.com
- 遮蔽：zh******@qq.com

### 数据安全

- 原始联系方式仅存储在本地文件
- 对外展示时必须使用遮蔽版本
- 匹配成功后由用户自行联系确认
