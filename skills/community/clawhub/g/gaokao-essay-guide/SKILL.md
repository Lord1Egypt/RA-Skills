# 高考作文写作全流程指导

## 任务目标
- 本 Skill 用于：高考作文（全国卷及各省卷）的全流程写作指导与诊断改进
- 能力：
  1. 审题立意：拆解题目、识别题型、精准立意
  2. 结构搭建：生成文章框架、推荐最优结构模式
  3. 素材推荐：匹配主题的鲜活素材与名言金句
  4. 语言润色：具象化表达、修辞提升、金句打造
  5. 评分预测：多维度诊断与分数预估
  6. 范文生成：生成完整范文供参考学习
- 触发：用户提到"高考作文""作文指导""审题""立意""作文评分""作文诊断""范文"等

## 前置准备
- 使用前先读取评分标准参考文档，了解阅卷要求
- 涉及结构生成时调用结构脚本，涉及诊断评分时调用诊断脚本

## 操作步骤

### 场景一：从零开始全程引导写作

#### 模式A：分步引导（默认）

**第1步：审题立意**

审题立意的输出**必须严格遵循以下格式模板**，不可省略或替换任何环节：

```
**题目类型：[材料作文/任务驱动型作文/命题作文/话题作文]**

一、审题分析
【此处根据题目类型，运用审题三法分析材料，必须显式标注使用了哪种方法】

二、任务要素（仅任务驱动型作文需要）
【身份/对象/场景/文体，并说明任务驱动型作文的特征】

三、立意方向
【2-3个立意方向，必须使用"浅层/中层/深层"标准深度等级标注，并按"个人层面/社会层面/国家层面"框架组织】

四、推荐立意
【推荐最优立意并说明理由，若题目文体不限，需推荐适合不同文体的立意方向】
```

具体执行规则：
1. 用户提供作文题目/材料
2. 智能体识别题目类型，**首行必须输出"题目类型：XX"**，四大标准术语为：`材料作文`、`任务驱动型作文`、`命题作文`、`话题作文`。严禁使用非标准表述
3. 运用审题三法分析材料，**必须显式标注使用了哪种方法**（如"运用关键词法分析：……"），不可仅做自主分析而不标注方法名称
4. 若为任务驱动型作文，拆解四大任务要素（身份、对象、场景、文体），并说明任务驱动型作文的特征
5. 给出2-3个立意方向，**必须使用"浅层/中层/深层"标准深度等级标注**，**必须按"个人层面/社会层面/国家层面"框架组织**
6. 推荐最优立意并说明理由。若题目"文体不限"，**必须为议论文、记叙文、散文等不同文体分别推荐适合的立意方向**
7. 等用户确认立意后进入下一步

**第2步：结构搭建**
1. 根据立意和文体，推荐最优结构模式（递进式/并列式/对比式）
2. 调用 `scripts/structure.py` 生成结构框架
3. 若为应用文体（演讲稿/倡议书/辩论稿/书信），**必须标注完整的格式要求**，包括：
   - 演讲稿/发言稿：称呼、问候语、正文、结束语、字数建议
   - 倡议书：标题、称呼、正文、号召语、署名、日期
   - 辩论稿：称呼、开篇立论、攻辩/论证、总结陈词、结束语、字数/时间限制、辩手分工
   - 书信：标题、称呼、问候、正文、此致敬礼、署名、日期
4. 等用户确认结构后进入下一步

**第3步：素材推荐**
1. 从素材库中匹配与立意相关的素材（时代人物/文化传承/科技前沿）
2. 推荐贴切的名言引用（古今中外结合，2-3处为宜）
3. 推荐金句模板，供用户化用
4. 提醒避免陈旧素材（司马迁、爱迪生等）
5. 等用户确认素材后进入下一步

**第4步：范文生成**
1. 基于已确认的立意、结构、素材，生成完整范文
2. 范文中标注关键写作技法（过渡词、金句、修辞等）
3. 提醒用户：范文供参考学习，需根据自身情况调整

**第5步：评分预测**
1. 调用 `scripts/diagnose.py` 对范文进行量化诊断
2. 智能体结合立意深度、思辨水平等主观维度给出综合评分
3. 给出进一步优化建议

#### 模式B：一次性完整方案

1. 用户提供作文题目/材料
2. 智能体一次性输出，**首行必须输出标准术语定性**，且各环节严格遵守第1步的格式模板和方法论要求
3. 用户自行参考写作

### 场景二：对已有作文诊断改进

**第1步：量化诊断**
1. 用户提供已有作文内容
2. 调用 `scripts/diagnose.py` 进行字数、标题、结构、格式、语言等多维度诊断
3. 输出诊断报告与扣分汇总

**第2步：深度分析**

语言诊断**必须覆盖以下五个维度**，不可遗漏：
1. **空泛表达**：是否全是口号和套话，缺乏具体内容
2. **口语化问题**：是否使用了过于口语化的表述，缺乏书面语的凝练与正式感
3. **句式单一**：是否句式结构雷同，缺乏长短交替和节奏变化
4. **修辞缺失**：是否缺少比喻、排比等修辞手法
5. **逻辑跳跃**：句与句之间是否存在逻辑断裂

其他维度：
- 立意是否准确深刻
- 思辨水平（是否有辩证思考、认知跃迁）
- 素材是否贴切新鲜

**第3步：改进方案**
1. 按优先级排列问题（偏题>结构>素材>语言）
2. 针对每个问题给出具体修改建议和示例
3. 给出预估分数与提分空间

**第4步：修改示范**
1. 对关键段落给出修改前后的对比示范
2. 标注修改理由和技法说明

## 资源索引

### 脚本工具
- **[scripts/diagnose.py](scripts/diagnose.py)**
  - 用途：对已有作文进行多维度量化诊断，输出评分预测和改进建议
  - 触发时机：当需要诊断已有作文或评估范文质量时调用
  - 脚本使用说明：
    - 功能1：诊断作文文件 → `python ./scripts/diagnose.py --file 作文.txt`
    - 功能2：诊断作文内容 → `python ./scripts/diagnose.py --essay "作文内容"`
    - 功能3：指定题目诊断 → `python ./scripts/diagnose.py --essay "作文内容" --topic "题目"`
    - 功能4：JSON格式输出 → `python ./scripts/diagnose.py --essay "作文内容" --json`

- **[scripts/structure.py](scripts/structure.py)**
  - 用途：根据题目和立意生成文章结构框架
  - 触发时机：当需要搭建文章结构时调用
  - 脚本使用说明：
    - 功能1：生成递进式框架 → `python ./scripts/structure.py --topic "题目" --thesis "立意" --type 递进式`
    - 功能2：生成并列式框架 → `python ./scripts/structure.py --topic "题目" --thesis "立意" --type 并列式`
    - 功能3：生成对比式框架 → `python ./scripts/structure.py --topic "题目" --thesis "立意" --type 对比式`
    - 功能4：指定文体格式 → `python ./scripts/structure.py --topic "题目" --thesis "立意" --format 演讲稿`
    - 可选文体：议论文、演讲稿、倡议书、辩论稿、书信

## 适用范围与局限

### 适用场景
- 高考语文作文（全国卷及各省卷）全流程指导
- 高三学生、复读生的作文训练与提升
- 语文教师的教学辅助与参考
- 模拟考试作文的评估与改进

### 不适用场景
- 初中及以下年级作文指导（评分标准和要求不同）
- 英语作文或其他语种写作
- 创意写作、小说创作、诗歌创作
- 公务员考试申论写作（评分体系不同）
- 已定稿作文的代写或抄袭用途

### 能力局限性说明
- 评分预测基于可量化的表层指标（字数、结构、过渡词等），无法完全替代阅卷教师对立意深度、思辨水平的主观判断
- 素材库需定期更新以保持时效性，超过三年的素材可能不再"新鲜"
- 本技能为辅助工具，不能替代真实写作练习和教师面批
- 生成的范文仅供参考学习，直接照搬可能被判为套作

## 注意事项
- **附件读取规则**：每次指导开始前，先读取 `references/scoring-criteria.md` 了解评分标准；审题立意时读取 `references/writing-methodology.md`；推荐素材时读取 `references/essay-materials.md`
- **审题输出规范（硬性要求）**：审题输出必须严格遵循格式模板——首行标准术语定性、显式标注审题三法、深度等级用"浅层/中层/深层"标注、立意按"个人/社会/国家"层面组织、文体不限时需适配不同文体。违反任何一项即不合格
- **脚本调用规则**：结构搭建时调用 `scripts/structure.py`；诊断评分时调用 `scripts/diagnose.py`；两者可独立使用
- **范文使用规则**：生成的范文仅供参考学习，提醒用户根据自身情况调整，不要照搬
- **素材新鲜度**：避免推荐司马迁、爱迪生、牛顿等陈旧素材，优先推荐近三年时代人物和热点
- **文体格式规范**：结构搭建时必须标注完整格式要求，辩论稿需包含字数/时间限制、辩手分工等细节
- **语言诊断五维度**：诊断语言问题时必须覆盖空泛表达、口语化、句式单一、修辞缺失、逻辑跳跃五个维度，不可遗漏
- **立意优先**：审题立意是作文成败的关键，偏离题意全盘皆输，务必在审题环节充分分析
- **辩证思维**：高分作文必须有辩证思考，避免非黑即白的简单判断，体现思辨深度

## 反模式（千万别这么用）

| 错误用法 | 为什么错 | 正确做法 |
|----------|----------|----------|
| 把范文直接背诵套用到考场 | 套作会被阅卷老师识别并降档，高考明确惩罚套作 | 参考范文的结构和技法，用自己的语言和素材重新写 |
| 跳过审题直接让生成结构 | 审题立意是作文成败关键，偏题全盘皆输 | 必须完成审题立意后再搭建结构 |
| 只看评分不看诊断建议 | 分数只是参考，真正有价值的是具体的改进方向 | 重点关注各维度的问题分析和改进建议 |
| 把素材库当万能模板直接堆砌 | 阅卷负面清单明确惩罚"堆砌事例" | 根据立意精选1-2个最贴切的素材，深入分析 |
| 每次都用同一种结构模式 | 不同题目适合不同结构，生搬硬套是"八病"之一 | 根据题目特点和立意选择最优结构 |
| 忽略脚本的评分局限性 | 脚本只检测表层指标，不能评判思想深度 | 把脚本评分作为下限参考，结合主观维度综合判断 |

## FAQ（常见问题）

**Q1：评分预测的准确度如何？**
诊断脚本基于字数、标题、结构、过渡词、语言五个可量化维度评分，能反映作文的基本水准。但立意深度、思辨水平、文采等主观维度需结合智能体的深度分析。预估分数误差范围约为±5分。

**Q2：素材库中的人物已经被大量使用了怎么办？**
素材不在于新旧，在于角度和分析深度。即使是常见素材（如张桂梅），只要切入角度独特、分析深入，同样可以得高分。建议关注事件的新角度，而非一味追求冷门。

**Q3：一次性完整方案和分步引导哪个更好？**
分步引导（模式A）适合训练阶段，能确保每一步都经确认后再进入下一步，有助于培养写作思维。一次性方案（模式B）适合时间紧迫或已有较强基础的用户快速参考。

**Q4：能否用于其他类型的考试作文？**
本技能严格按照高考评分标准设计。中考、四六级、考研作文的评分体系不同，直接使用可能产生偏差。公务员申论更是完全不同的文体。

**Q5：诊断脚本需要什么运行环境？**
Python 3.6 及以上版本，仅使用标准库（argparse、json、re、sys），无需安装任何第三方包。

**Q6：如何让作文从"二类文"提升到"一类文"？**
一类文（55-60分）的核心标准是：精准把握关键词深层内涵 + 辩证思考 + 结合当下现实 + 必须结合材料文本。建议重点关注"认知跃迁"（从现象→本质→规律的思维升级）和"金句密度"（每200字1个哲学化表达）。
