# 商业化审核与校正手册

## 1. 目的与适用范围

本手册用于对Skill/Agent的商业化要素进行系统性审核与校正，确保定价、套餐、引流的合理性和一致性。

**适用场景**：
- 新Skill商业化上线前审核
- 现有Skill定价/套餐调整
- 引流链路优化
- 商业化一致性检查

---

## 2. 报价合理性审核

### 2.1 定价公式

```
合理价格 = 算力要求系数 × 使用频率系数 × 价值密度系数
```

**各系数详细定义**：

| 系数类型 | 等级 | 系数值 | 判定条件 |
|----------|------|--------|----------|
| 算力要求 | 简单 | 1.0 | 单次调用<1秒，无复杂计算 |
| | 中等 | 1.3 | 单次调用1-3秒，有一定计算 |
| | 复杂 | 1.8 | 单次调用>3秒，有复杂推理 |
| | 极高 | 2.0 | 需要多轮调用或外部API |
| 使用频率 | 低频 | 0.8 | 用户平均每月使用<5次 |
| | 中频 | 1.0 | 用户平均每月使用5-20次 |
| | 高频 | 1.2 | 用户平均每月使用20-50次 |
| | 极高频 | 1.5 | 用户平均每月使用>50次 |
| 价值密度 | 通用 | 1.0 | 解决通用问题，非刚需 |
| | 实用 | 1.5 | 解决常见问题，有一定价值 |
| | 专业 | 2.0 | 解决专业问题，价值较高 |
| | 高端 | 3.0 | 解决核心痛点，不可替代 |

**计算示例**：

| Skill类型 | 算力系数 | 频率系数 | 价值密度 | 计算值 | 建议定价 |
|-----------|----------|----------|----------|--------|----------|
| 轻量工具 | 1.0 | 0.8 | 1.0 | 0.8 | 1-3元 |
| 标准工具 | 1.3 | 1.0 | 1.5 | 1.95 | 4-6元 |
| 专业工具 | 1.8 | 1.2 | 2.0 | 4.32 | 7-9元 |
| 高端方案 | 2.0 | 1.5 | 3.0 | 9.0 | 9元+或套餐 |

### 2.2 梯度校验

| 定价区间 | 类型 | 特征 | 合理系数组合 |
|----------|------|------|--------------|
| 1-3元 | 轻量级 | 单一功能、快速执行、通用场景 | 算力1.0 × 频率0.8 × 价值1.0 ≈ 0.8 |
| 4-6元 | 标准级 | 多功能、中等复杂、实用场景 | 算力1.3 × 频率1.0 × 价值1.5 ≈ 1.95 |
| 7-9元 | 专业级 | 复杂功能、专业场景、高价值 | 算力1.8 × 频率1.2 × 价值2.0 ≈ 4.32 |

### 2.3 偏离度计算与校正

**偏离度计算**：
```
定价偏离度 = |实际定价 - 公式计算定价| / 公式计算定价 × 100%
```

**判定标准**：

| 偏离度 | 判定 | 处置 |
|--------|------|------|
| <15% | ✅合理 | 可发布 |
| 15-30% | ⚠️偏高/偏低 | 建议调整 |
| >30% | 🛘偏离过大 | 必须校正 |

**校正方案**（当偏离度过大时）：

#### 方案A：调整功能复杂度
- 定价偏高 → 简化功能或降低算力要求
- 定价偏低 → 增加功能或提升价值密度

#### 方案B：调整目标定位
- 定价偏高 → 扩大目标用户群或提升价值感知
- 定价偏低 → 聚焦高端用户或提升专业度

#### 方案C：调整使用频次假设
- 定价偏高 → 证明高频使用价值
- 定价偏低 → 降低频次假设或提升单次价值

## 3. 套餐组合审核

### 3.1 套餐折扣合理性

**折扣范围**：8-9折（最优解通常在8.5折）

| 折扣 | 评价 | 适用场景 |
|------|------|----------|
| <7折 | 🔴过低 | 损害利润，除非引流目的明确 |
| 7-8折 | ⚠️偏低 | 可用于限时促销 |
| 8-9折 | ✅合理 | 标准套餐折扣 |
| >9折 | ⚠️意义不大 | 缺乏吸引力 |

### 3.2 功能互补性评估

**互补性计算**：
```
功能互补性 = 1 - (重叠功能数 / 总功能数)
```

**判定标准**：

| 互补性 | 判定 | 处置 |
|--------|------|------|
| >70% | ✅强互补 | 适合打包 |
| 50-70% | ⚠️中等 | 评估是否值得打包 |
| <50% | 🔴重叠高 | 🛘建议拆分或重组 |

**功能重叠矩阵**：

| 功能 | Skill A | Skill B | Skill C | 重叠判定 |
|------|---------|---------|---------|----------|
| 功能1 | ✅ | ✅ | ⚡ | 重叠 |
| 功能2 | ✅ | ❌ | ✅ | 部分重叠 |
| 功能3 | ✅ | ✅ | ❌ | 重叠 |
| 功能4 | ⚡ | ✅ | ✅ | 部分重叠 |
| 功能5 | ✅ | ❌ | ❌ | 不重叠 |

### 3.3 场景覆盖完整性

**检查项**：
- [ ] 套餐是否覆盖用户的主要使用场景
- [ ] 套餐内Skill是否形成完整工作流
- [ ] 用户购买套餐后是否还需要其他Skill

## 4. 关联引流校正

### 4.1 引流路径校验

**校验流程**：

```
A Skill引流到B Skill
    ↓
检查1：B Skill是否存在？ → 不存在🛘阻断
    ↓ 存在
检查2：引流话术与实际功能是否匹配？ → 不匹配⚠️校正
    ↓ 匹配
检查3：引流路径是否自然（非硬广）？ → 太硬⚠️优化
    ↓ 自然
✅ 有效引流
```

### 4.2 引流话术自然度评估

| 评分 | 描述 | 示例 |
|------|------|------|
| 5分 | 场景自然延伸 | "查新完成后，推荐使用技术尽调深度分析" |
| 4分 | 有逻辑关联 | "需要专利申请？试试这个工具" |
| 3分 | 关联一般 | "相关推荐：XXX" |
| 2分 | 略显突兀 | "你可能还需要：XXX" |
| 1分 | 生硬广告 | "限时优惠：XXX，只需X元" |

**判定标准**：
- ≥4分：✅ 可用
- 3分：⚠️ 建议优化
- <3分：🔴 需要重新设计

### 4.3 互链闭环检查

**检查公式**：
```
互链完整度 = 实际互链数 / 理论互链数 × 100%
```

**示例**：
- A→B, A→C, B→A, C→A
- 理论互链数：4（两两互链）
- 实际互链数：4
- 完整度：100% ✅

**孤岛节点判定**：

| 节点类型 | 判定 | 处置 |
|----------|------|------|
| 入链=0且出链=0 | 完全孤岛 | 🛘必须添加引流 |
| 入链=0 | 无人引流 | ⚠️建议增加入口 |
| 出链=0 | 无法引流 | ⚠️建议增加出口 |

### 4.4 转化漏斗通畅性

**漏斗分析**：
```
用户进入A Skill
    ↓
使用A功能
    ↓
触发引流钩子（需评估触发时机是否合适）
    ↓
点击引流链接
    ↓
进入B Skill
```

**优化点**：
- 引流时机：功能完成后立刻展示 vs 使用中途展示
- 引流位置：输出末尾 vs 侧边栏 vs 弹窗
- 引流形式：文字链接 vs 按钮 vs 卡片

## 5. 商业化一致性校正

### 5.1 定价与定位一致性

| 定位层级 | 价格范围 | 一致性判定 |
|----------|----------|------------|
| 高端/专业 | 7-9元 | ✅ 高端产品高价 |
| | 4-6元 | ⚠️ 高端产品中价，需强化价值感知 |
| | 1-3元 | 🛘 定位矛盾，需重新定义 |
| 标准/实用 | 4-6元 | ✅ 标准产品中价 |
| | 1-3元 | ⚠️ 标准产品低价，可接受 |
| | 7-9元 | 🛘 定位矛盾，需重新定义 |
| 入门/轻量 | 1-3元 | ✅ 入门产品低价 |
| | 4-6元 | ⚠️ 入门产品中价，需简化功能 |
| | 7-9元 | 🛘 定位矛盾，需升级功能 |

### 5.2 免费与付费边界

**边界清晰度检查**：

| 场景 | 判定标准 | 处置 |
|------|----------|------|
| 完全免费 | 无任何付费限制 | ✅ OK |
| 完全付费 | 所有功能均需付费 | ✅ OK，需明确 |
| 基础免费+高级付费 | 免费功能明确，付费功能清晰 | ✅ OK |
| 时间限制 | 免费试用期明确（7-30天） | ✅ OK |
| 模糊边界 | 无法区分免费/付费功能 | 🛘 必须明确 |

### 5.3 试用期设计合理性

| 试用设计 | 评价 | 建议 |
|----------|------|------|
| 7天试用+明确付费引导 | ✅合理 | 可用 |
| 无限期免费（功能受限） | ⚠️需评估 | 限制条件需明确 |
| 试用后强制付费 | ⚠️需谨慎 | 需充分告知 |
| 无试用期直接付费 | 🔴体验差 | 建议增加试用 |

## 7. 常见问题与解答

### Q1: 定价公式计算值与实际定价差距30%，如何选择校正方案？
A: 优先选择**调整功能复杂度**（方案A），因为：
- 最直接有效，用户能感知价值变化
- 不影响定位和目标用户
- 风险最小

### Q2: 套餐内两个Skill功能重叠60%，但用户确实需要打包购买？
A: 
- 如果重叠功能是核心功能，考虑拆分
- 如果重叠功能是辅助功能，可保留但需优化话术
- 建议增加差异化功能减少重叠感

### Q3: 发现引流目标不存在（Skill未创建）？
A: 
- 🛘阻断发布
- 方案1：暂缓发布，先完成被引流Skill
- 方案2：移除该引流路径，补充其他有效引流

### Q4: 如何判断引流话术是否"自然"？
A: 自问："用户看到这个引流，会觉得被推销吗？"
- 如果是 → 不够自然
- 如果否 → 较自然
- 好的引流是"用户觉得是为自己好，而不是在推销"
