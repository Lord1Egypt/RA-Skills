# Systematic Literature Review

**Platforms:** Claude · Openclaw · Codex
**Domain:** Academic Research

## Purpose

Drives a researcher through a PRISMA-aligned systematic literature review. Covers protocol definition, search strategy construction, screening decisions on title/abstract and full text, structured data extraction, quality appraisal, and narrative synthesis — always recording inclusions, exclusions, and assumptions so the review is reproducible.

## When to Use

- Graduate student or postdoc starting a systematic review or scoping review
- Researcher preparing a PRISMA-compliant methods section for a journal submission
- Evidence synthesis team building an extraction table from a candidate paper set
- Anyone organizing a defensible, transparent review process across many studies

## What It Does

**Phase 1: Protocol & Search**
1. Captures the review question in a structured framework (PICO, PEO, SPIDER, or CIMO) — one question at a time
2. Defines explicit inclusion and exclusion criteria, databases to search, date range, and language scope
3. Drafts a reproducible search string with Boolean operators and field tags

**Phase 2: Screening**
4. Records every screening decision (Include / Exclude / Unclear) with a reason code drawn from the exclusion list
5. Tracks counts at every PRISMA stage: identified, deduplicated, title/abstract screened, full-text assessed, included
6. Flags conflicts (Unclear or contradictory decisions) for the reviewer to resolve before progressing

**Phase 3: Extraction, Appraisal & Synthesis**
7. Builds a per-study extraction table aligned to the review question
8. Applies a quality appraisal checklist appropriate to the study designs included
9. Synthesizes results narratively (themes, gaps, contradictions) and produces the PRISMA flow diagram description

## Output

A complete review packet: protocol summary, search strategy, PRISMA flow counts, screening log with reason codes, extraction table, quality appraisal table, narrative synthesis, and a limitations / open-questions section. Ready to drop into a methods chapter or supplementary appendix.

## Safety Notes

The skill never invents study data, citations, DOIs, or author names — every extracted field must trace back to text the user supplied. If a paper's full text is not provided, the skill marks the relevant cells "not assessable from abstract" rather than guessing. Unpublished manuscripts shared during the session are treated as confidential and never reused in examples or external lookups.
