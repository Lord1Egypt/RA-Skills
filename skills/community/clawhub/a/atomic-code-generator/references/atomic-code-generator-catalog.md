# 原子代码生成清单与依赖拓扑

> 🧩执行协议等待指令：复杂功能分拆成原子方法，原子方法生成完整代码，默认只展示方法调用结构但用户可以要求展示完整代码。

**域间逻辑流**：M0 → M1 → M2 → M3 → M4 → M5

---

| ID | 任务类型 | 说明 | 依赖 | 元操作映射 |
|----|---------|------|------|-----------|
| M0-01 | 任务域选择与组合 | 根据任务需求选择合适的任务域组合 | 无（入口） | S→C |
| M0-02 | 执行顺序确定 | 确定任务域的执行顺序和依赖关系 | M0-01 | C |
| M0-03 | 结果整合与优化 | 整合各任务域的执行结果，优化整体输出 | M0-02 | C→A |
| M0-04 | 深化路由 | 根据任务类型将深化请求路由到相应的方法论域 | M0-03 | C→I |

## 依赖拓扑摘要

**任务域协调链路**: M0-01 → M0-02 → M0-03 → M0-04
**功能分析链路**: M1-01 → M1-02 → M1-03 → M1-04 → M1-05
**原子方法拆解链路**: M2-01 → M2-02 → M2-03 → M2-04 → M2-05 → M2-06
**方法接口设计链路**: M3-01 → M3-02 → M3-03 → M3-04 → M3-05
**代码生成链路**: M4-01 → M4-02 → M4-03 → M4-04 → M4-05
**结果交付链路**: M5-01 → M5-02 → M5-03 → M5-04 → M5-05

**完整处理链路**: M0-01 → M1-01 → M2-01 → M3-01 → M4-01 → M5-01
