**用户主权说明**:

## P1 商业定位

### EX-02 垂直利基定位一页纸

**对应任务**: P1-01

**来源**: OPC实战方法论总结

**适用场景**: 0→1阶段，需要确定自己的垂直利基定位

**定位一页纸模板**:

```
## 我的垂直利基定位

**一句话定位**：我帮 [具体人群] 解决 [具体问题]，实现 [可量化成果]

**目标客户画像**：
- 行业：________（如：牙科诊所、独立律所、跨境电商）
- 城市/区域：________
- 规模：________（如：年营收100-500万）
- 决策者：________（如：诊所老板/创始人）

**核心痛点**：
1. ________（最痛的那个，客户愿意为此付费）

**我的解决方案**：
- 不是说「我能帮你导入AI」
- 而是说「我能在___天内帮你增加___%的___」

**为什么选我**：
1. 行业经验：________年
2. 独特能力：________
3. 差异化：________（与市面方案的1-3个核心差异）

**验证方式**：
- 3个愿意付费的潜在客户是否确认
```

**关键设计决策**:
- 一句话定位强制包含"谁+什么问题+什么成果"三要素
- 禁止"AI"出现在定位中——客户买单的是成果不是技术
- 必须有3个潜在客户确认才算定位通过

**可复用要素**:
- 一句话定位模板可适配任何垂直领域
- 验证方式是通用的OPC需求验证标准

## P3 产品与交付

### EX-06 MVP一页纸

**对应任务**: P3-01

**来源**: OPC产品化方法论

**适用场景**: 需要定义最小可行产品/服务

**MVP一页纸模板**:

```
## MVP定义

**核心价值（仅1个）**：________

**全功能清单→砍到MVP**：

| 功能/服务 | 保留？ | 理由 |
|----------|--------|------|
| ________ | ✅保留 | 核心价值 |
| ________ | ❌砍掉 | V2.0再说 |
| ________ | ❌砍掉 | 非核心 |

**最小交付标准**：
- 客户能获得核心价值的最低配置
- 不追求完美，追求"能用"

**验证指标**：
- __%的客户表示满意 = 验证通过
- ___个付费客户 = 需求确认

**交付周期**：≤2周

**砍掉清单回顾**（V2.0候选）：
1. ________
2. ________
```

**关键设计决策**:
- MVP只能有1个核心价值点，多了就不是MVP
- 砍掉清单必须保留——是产品迭代路线图的输入

**可复用要素**:
- "保留/砍掉"决策模板适用于任何OPC产品/服务设计

```
| 标准化→产品化 | 可独立交付的产品 | 识别自动化点位+搭建AI Agent | 产品交付耗时降低50% | ___周 |
| 产品化→SaaS化 | 订阅制软件 | 封装为自助服务+订阅管理 | MRR≥¥___ | ___月 |

### 每步的验证标准
1. 标准化通过：不依赖特定人即可交付
2. 产品化通过：AI自动化覆盖≥70%流程
3. SaaS化通过：客户可自助使用，MRR持续增长
```

**关键设计决策**:
- 必须按Dan Martell路径逐步迁移，不可跳步
- 每步都有明确的验证标准——不可"差不多就行"

**可复用要素**:
- 三阶段迁移模板适用于任何OPC服务产品化
