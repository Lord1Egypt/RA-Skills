# 拆解卡输出模板

每条拆解结果用本模板。所有字段都要保留，没线索的写「待补充」。

---

```markdown
# {任务编号} | {一句话标题}

> 参考链接：{原始链接}
> 拆解时间：{YYYY-MM-DD HH:mm}
> 平台：{小红书 / 抖音 / 视频号}
> 内容类型：{视频 / 图片}

## 元数据
- **作者**：{nickname}
- **发布时间**：{YYYY-MM-DD}
- **互动**：👍 {liked} ｜ ⭐ {collected} ｜ 💬 {comment} ｜ 📤 {share}
- **note_id**：`{note_id}`

### 1. 内容目标
{用户输入的目标，如「门店引流 + 私域加微」；没填写「未指定」}
```

# 字段定义详解 + Anti-Pattern

## 字段速查表

| 字段 | 必填 | 长度 | 形式 | 示例 |
|---|---|---|---|---|
| 任务编号 | ✅ | 短 | `AIC-{YYMMDD}-{序号}` | `AIC-260426-001` |
| 内容目标 | 🟡 | 一句话 | 名词短语 | 门店引流 + 私域加微 |
| 平台 | ✅ | 短 | 单选 | 小红书 / 抖音 / 视频号 / 全平台 |
| 目标客群 | ✅ | 30–80 字 | 画像描述（带证据） | 30-45 岁、重品质与审美、愿为材质买单的女性 |
| 爆款主题 | ✅ | 30–60 字 | 解释机制的完整句 | 借画面美感把"看视频"变成"想象自己穿上"，激活高保存意愿 |
| 参考内容拆解 | ✅ | 长 | 结构化（按时间段聚合） | 见 SKILL.md Step 4b |
| 风格标签 | ✅ | 3–6 个 | 形容词/品类（标已有/新增） | 老钱风、松弛感、贵气感、亚麻 |
| 场景标签 | ✅ | 2–4 个 | 名词短语（环境） | 度假海边、都市通勤、茶室 |
| 情绪钩子 | ✅ | 核心 + 佐证 | 双段（带 graph 命中） | 反差感｜年龄 × 成就 \| 命中 graph hooks |
| 评论关键词 | 🟡 | 3–8 个 | 关键词 + **原始评论佐证** | 怎么买（评论：「绿色裤子怎么买」） |
| 参考视频旁白文案 | 视频必填 | 原文 | 逐字稿 / 「无」 | 我有一个不一样的老板，叫古姐… |

| 参考封面文案 | ✅ | 一行 | 文案 / 「无」 | 把江南春色穿在身上是什么体验？ |
| 参考发布主题 | ✅ | 标题 | 原文 | 深圳新中式｜… |
| 参考发布文案 | ✅ | 正文 | **原文**（含 emoji + 换行，禁改写） | 谁懂啊家人们…💚 |
| 参考热门标签 | ✅ | 全部 | `#xxx` 拼接（来自 `hash_tag[].name`） | #新中式 #深圳南山 #国风穿搭 |
| 可学习要点 | ✅ | 2–5 条 | bullet（mode 决定形式） | 见下方 mode 说明 |

## 主观字段 · Anti-Pattern（关键）

### 爆款主题
说**为什么火**（机制），不说**是什么**（描述）。

| ❌ 描述型 | ❌ 营销型 | ✅ 洞察型 |
|---|---|---|
| 展示新中式服装新款 | 东方美学高级感种草 | 借画面美感把"看视频"变成"想象自己穿上"，激活高保存意愿 |
| 主理人讲创业故事 | 75 后女老板创业经历传递匠心 | 用 75 后 × 2000 平的数字反差建立"匠心 × 规模"信任锚点 |

**自检问句**：你写的句子能不能解释"为什么这条能让人收藏而不是只点赞"？不能 → 重写。

### 风格 / 场景 / 情绪钩子（最容易混淆）

| 维度 | 定义 | 形式 |
|---|---|---|
| **风格** | 看起来/感觉起来是什么 | 形容词、流派词 |
| **场景** | 发生在哪儿 | 名词短语，环境 |
| **情绪钩子** | 让用户内心起什么波澜 | 「{情绪类型}｜{激发手法}」双层 |

例（AIC-001 见花开陈列）：
- 风格：温婉 / 东方美学 / 清新雅致 / 沉浸式陈列
- 场景：门店陈列区 / 木质衣架 / 玻璃橱窗
- 情绪钩子：**向往感｜画面氛围代入** + 一句佐证

| ❌ 差（孤词、形容词堆） | ✅ 好（双层 + 机制） |
|---|---|
| 美感 | **向往感｜画面氛围代入**（无人声陈列靠视觉建立想拥有的情绪）|
| 高级 | **身份感｜植入"穿上 = 成为某种人"**（"江南温婉大小姐既视感"）|
| 反差 | **反差感｜年龄 × 成就**（具体数字制造身份反差）|

### 旁白逻辑分析

必须**分层**写，不能流水段。每层标时间段 + 一句话功能。

| ❌ 流水段 | ✅ 分层结构 |
|---|---|
| 视频开头讲了创业经历，中间讲匠心，结尾讲共鸣，整体很完整 | 第一层 · 建立反差与好奇（0-12 秒）：用 75 后 + 2000 平的数字反差激发好奇<br>第二层 · 传递匠心与价值（12-28 秒）：通过"亲自审样"等细节塑造专业匠人<br>第三层 · 落脚情感与共鸣（28-39 秒）：从产品上升到情感连接，完成升华 |

末尾必须有**关键设计** 1-3 条：本条文案"好"在哪个具体设计选择上（反差数字 / 工艺名词承载 / 时间锚点等）。

### 目标客群

必须**带证据**，不能只丢画像。

| ❌ 模糊 | ✅ 带证据 |
|---|---|
| 30-45 岁有审美的女性 | 30-45 岁追求东方美学的女性，看重材质和工艺（佐证：评论里"怎么买/绿色马甲怎么买"高频；desc 强调"祖母绿真丝、手工钉珠"等材质细节；藏赞比 70% 说明决策周期长） |

**品牌视角模式下**：必须显式标注"对标 graph/audience/segments 中的 {客群层名}"。

### 可学习要点（mode-aware）

#### 模式 A · graph 已填充（品牌视角）
写"我方做同主题怎么用 X"——具体到品牌动作。

| ❌ 通用 | ✅ 品牌化 |
|---|---|
| 数字反差是好钩子 | 我方"{{N 年成立 / N 年专注 / N+ SKU}}"可作为类似数字反差使用，配合主理人出镜（参考 graph/brand/brand-story 中已记录的关键经历素材） |

#### 模式 B · graph 未填充（客观拆解）
写客观要点 + 行动建议（不针对具体品牌）。每条 bullet 含「现象 → 推论 → 建议」三段。

```
✅ 例：
- 无人声陈列也能拿高互动（藏赞比 70%）→ 前提是产品视觉力够强 +
  发布文案足够画面化能补足画面缺失的信息 → 高客单 / 高审美品类适用此公式
```

## 「评论关键词」语义分类与防伪（agent 自己做，不用 regex）

> 旧版本曾用 regex 预提取关键词写到 `keywords.json`，但语言无穷变体（"怎么卖" / "啥价" / "贵不贵"），regex 永远漏；regex 也分不清语义（"价格不是问题" 不是询价）。已彻底废弃。
>
> agent 直接读 `comments.json` 做语义分类——LLM 比 regex 强 100 倍，零盲点零维护成本。

### 数据源

`comments.json`（已经 parser 过滤——`is_pinned=True` 的商家置顶 / 反诈骗自动标了，可以跳过；其余都是用户真实评论）。

### 分类四类

| 类别 | 抓什么 | 典型表达 |
|---|---|---|
| **ask** 问 | 问购买路径 / 价格 / 地址 / 时间 / 渠道 | 怎么买、多少钱、啥价、怎么卖、店在哪、几点关门、线上有吗、贵不贵 |
| **request** 求 | 主动提需求（强意向） | 求微信、还有吗、断码了吗、求联系、补货吗 |
| **praise** 夸 | 共鸣 / 喜欢的具体表达 | 好美、想要、高级感、显气质、心动、绝美 |
| **objection** 异议 | 纠正 / 反对（**不是中性疑问**） | 请不要叫它X、这是 A 不是 B、不应该这么贵 |

### 输出格式（每条带证据）

```
- {关键词}（{N} 条原评：「原文1」「原文2」「原文3」）— {一句解读 / 转化信号判断}
```

### 防伪约束（硬性）

1. 每个关键词必须附 1-3 条**原始评论原文**（直接从 comments.json 复制，不许改写）
2. 找不到原文证据的关键词不允许出现——禁止凭空生成
3. **疑问句不算异议**：「不是真丝吗？」是中性询问，归 ask；「请别叫它新中式」才归 objection
4. 同一原评可同时归到多类——「怎么买啊好喜欢绿色」既是 ask 又是 praise，分别引用
5. comments.json 是 `[]` 或带 `_error` → 写「⚠️ 评论数据未获取」，**不允许从 desc 推测**

### 好坏对照

```
✅ 好（带证据 + 解读）：
- 怎么买（5 条原评：「绿色裤子怎么买」「怎么购买？线上」「怎么买呢 好喜欢绿色」）—
  最高频转化信号
- 怎么卖 / 啥价（2 条原评：「怎么卖啊」「这套啥价」）— 询价的另一种说法
- 异议·满族服饰（1 条原评：「这是新宾满族服饰，请不要叫它新中式！」👍 1）—
  虽只 1 条但获赞，提示标签使用边界

❌ 差（无证据 / 编造）：
- 怎么买、多少钱、求链接（光列词，无原文，禁止）

❌ 差（误判疑问为异议）：
- 异议（评论：「这是真丝吗」）  ← 这是疑问不是异议
```

## 「参考热门标签」清理规则

**来源**：API 返回的 `hash_tag[].name`，不是 `topics[]`（topics 是算法精选，通常 1-2 个，是 hash_tag 的子集）。

**清理**：
- 用 `hash_tag` 字段直接拿，结果是干净的字符串数组（不带 `[话题]` 渲染标记）
- **不要**从 desc 文本 grep `#xxx[话题]#`——desc 里的渲染标记需要 `sed 's/\[话题\]//g'` 清除
- 顺序保留 API 返回的原顺序（≈ 作者优先级）
- 全部用 `#` 拼接成一行

```
✅ #新中式 #深圳南山 #国风穿搭
❌ #新中式[话题]# #深圳南山[话题]#
```
