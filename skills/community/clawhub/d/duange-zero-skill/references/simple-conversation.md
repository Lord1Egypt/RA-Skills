# 零基础对话流程

## 目标

让用户不需要理解任何专业概念，也能把想法说清楚。

## 开场

```text
你想做一个什么小助手？不用说专业话，像平时聊天一样说就行。
```

如果用户沉默或说不知道：

```text
我给你几个例子，你选一个最像的：
1. 帮我改文字
2. 帮我整理文件
3. 帮我生成图片提示词
4. 帮我写文章/脚本
5. 帮我检查东西有没有问题
6. 我想做别的
```

## 追问顺序

每次只问一个问题。

1. 它帮你做什么？
2. 你会给它什么？
3. 你想拿到什么？
4. 它做事时最不能错什么？
5. 你以后会怎么叫它出来？

## 自动整理格式

```text
我听懂的大概是：

你想做的小助手：……
你会给它：……
它要输出：……
它要注意：……
触发方式：……

我这样理解对吗？
```

## 用户表达很乱时

不要批评用户表达。先提炼，再确认。

```text
我先帮你翻译成人能执行的版本：
……
你看看是不是这个意思？
```

## 用户目标太大时

把大目标拆成第一个可完成版本。

```text
这个想法可以做得很大。我们先做第一个简单可用版：……
后面再加更多功能。
```
