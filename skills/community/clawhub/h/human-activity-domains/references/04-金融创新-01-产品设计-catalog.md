## 金融产品设计任务清单

金融产品设计按子域组织的任务清单，附任务间关系和元操作映射提示。

**域间逻辑流**：需求分析 → 产品设计 → 产品开发 → 产品运营

---

### 1 需求分析

| ID | 任务类型 | 说明 | 依赖 | 元操作映射 |
|----|----------|------|------|----------|
| T1-1 | 客户需求 | 调研客户需求，了解客户的金融需求和痛点 | 无（入口） | S→I→C |
| T1-2 | 市场调研 | 调研金融市场趋势、行业动态、竞争格局 | T1-1 | S→C |
| T1-3 | 竞品分析 | 分析竞争对手的金融产品特点、优劣势、市场表现 | T1-2 | S→C |
| T1-4 | 监管要求 | 研究金融监管政策、合规要求、准入门槛 | T1-3 | S→C→G |

## 依赖拓扑摘要

### 一、域内链路

**需求分析链路**: T1-1 → T1-2 → T1-3 → T1-4
**产品设计链路**: T2-1 → T2-2 → T2-3 → T2-4
**产品开发链路**: T3-1 → T3-2 → T3-3 → T3-4
**产品运营链路**: T4-1 → T4-2 → T4-3 → T4-4

### 二、跨域协同链路

**完整金融产品设计链路**: T1-1 → T1-2 → T1-3 → T1-4 → T2-1 → T2-2 → T2-3 → T2-4 → T3-1 → T3-2 → T3-3 → T3-4 → T4-1 → T4-2 → T4-3 → T4-4

**反馈改进链路**: T4-3 → T2-1（产品优化反馈到产品架构）

**监管驱动链路**: T1-4 → T2-4（监管要求驱动合规设计）

更多组合根据具体任务动态推导。
