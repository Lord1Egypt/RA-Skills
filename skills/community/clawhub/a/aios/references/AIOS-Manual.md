## 第一章：投资哲学

### 1.1 核心信念

**世界上没有一套被验证的"选股法则"。**

**但是，有一套被几十年、几百年反复验证的"风险管理法则"。**

无论是巴菲特、索罗斯、达利欧、文艺复兴、趋势交易员还是量化基金，策略各不相同，但在风险管理上却有高度共识。

### 1.2 我的能力圈

- ✅ 对产业的深度理解
- ✅ GTM（Go-To-Market）和商业模式判断
- ✅ 对 AI 长期趋势的敏感度
- ❌ 短线技术分析（不是我的优势）

**结论：** 用 AI 发现未来 5 年的产业变化，用交易手册控制风险，让市场决定收益。

### 1.3 什么钱赚，什么钱不赚

| 赚 ✅                                  | 不赚 ❌                    |
| -------------------------------------- | ------------------------- |
| 产业趋势带来的长期价值增长                | 短线投机、追涨杀跌          |
| 被市场低估的优质资产                      | 没有研究过的"热门股"        |
| 有明确催化剂的成长股                      | 杠杆交易、衍生品投机        |
| AI 时代基础设施和应用层机会               | 别人推荐但自己不懂的标的     |

### 1.4 三大铁律

1. **本金永远第一** — 赚钱需要无数次，爆仓只需要一次
2. **永远先考虑亏多少，再考虑赚多少**
3. **没有完美的系统，只有不断进化的系统**

## 第二章：十大核心原则

### 原则一：本金永远第一

> 任何一次交易，都不能让整个账户受到致命伤害。

- 单笔最大亏损：账户总额的 **1-2%**
- 单月最大回撤：账户总额的 **5-8%**
- 触发熔断线：暂停交易，重新审视系统

### 原则二：先决定退出，再决定买入

每次买入前，必须先写好：

```
1. 我为什么买？（买入逻辑）
2. 什么情况下我错了？（失效条件）
3. 如果错了怎么办？（退出计划）
```

**回答不了这三个问题 → 不买。**

### 原则三：永远分批建仓

```
❌ 一次性 100% 建仓

✅ 第一次：20-30%
✅ 加仓条件（满足其一）：
   - 公司基本面变好
   - 财报超预期
   - 市场开始验证你的判断
```

**绝对不是：跌了所以补。**

### 原则四：永远不要因为亏损补仓

补仓前必须回答：

> 如果今天没有持仓，现在这个价格，我会买吗？

- 答案是「不会」→ 禁止补仓
- 答案是「会」→ 可以考虑，但仍需走完检查流程

### 原则五：仓位比价格重要

> 方向错一点不要命，仓位错了要命。

| 标的类型               | 最大仓位   |
| ---------------------- | ---------- |
| ETF                    | 20-25%     |
| 核心龙头（长期看好）    | 10-15%     |
| 高风险成长股            | 5-8%       |
| 试错仓                 | 1-3%       |

### 原则六：单笔亏损固定（Position Sizing）

**先确定允许亏损金额，再反推仓位大小。**

```
示例：
账户总额：100 万
单笔最大亏损：1% = 1 万元
止损距离：5%

可买金额 = 1 万 ÷ 5% = 20 万
可买股数 = 20 万 ÷ 当前股价
```

### 原则七：盈利可以加仓（加赢家，不救输家）

```
❌ 普通人：亏损 → 加仓（摊薄成本）

✅ 专业：赚钱 → 确认趋势 → 再加仓
```

### 原则八：不要预测，改为应对

```
❌ "我觉得一定涨"

✅ "如果涨，怎么办？"
✅ "如果跌，怎么办？"
```

**所有答案提前写好。交易当天：不用思考，执行。**

### 原则九：交易日志

记录什么：

- 为什么买？为什么卖？
- 当时情绪是什么？
- 后来看哪里判断错了？
- 有没有违反规则？

> 半年以后你会发现：最大的敌人不是市场，是自己。

### 原则十：定期推翻自己

每半年重新回答：

```
我的交易系统还有没有漏洞？
```

**不要相信任何一条规则，包括自己的。**

## 第三章：交易规则（SOP）

### 3.1 买入流程

```
┌─────────────────────────────────────┐
│  Step 1: 研究阶段                    │
│  - 产业分析（AI 辅助）               │
│  - 公司基本面（财报、护城河、管理层） │
│  - 估值判断                          │
└──────────────┬──────────────────────┘
               ▼
┌─────────────────────────────────────┐
│  Step 2: 写交易计划                   │
│  - 买入理由（3 条以上）              │
│  - 失效条件（什么情况下我错了）       │
│  - 止损价格                          │
│  - 目标价格区间                       │
│  - 计划仓位（分批方案）              │
└──────────────┬──────────────────────┘
               ▼
┌─────────────────────────────────────┐
│  Step 3: 执行检查（运行 aios 工具）  │
│  - 通过所有检查项 → 允许买入         │
│  - 未通过 → 回到 Step 2 补充         │
└──────────────┬──────────────────────┘
               ▼
┌─────────────────────────────────────┐
│  Step 4: 执行交易                     │
│  - 按计划执行第一批仓位（20-30%）    │
│  - 记录到交易日志                     │
└─────────────────────────────────────┘
```

### 3.2 加仓流程

加仓前必须满足以下条件之一：

- ✅ 公司基本面变好（有具体证据）
- ✅ 财报超预期（数据支持）
- ✅ 市场开始验证你的判断（价格向有利方向运动）

同时必须：

- ✅ 当前持仓未触及仓位上限
- ✅ 走完交易检查流程

### 3.3 减仓/卖出流程

| 触发条件               | 动作                    |
| ---------------------- | ----------------------- |
| 触及止损价             | 立即卖出全部            |
| 失效条件被触发         | 卖出全部或大部分        |
| 达到目标价             | 分批减仓（先回本）      |
| 基本面恶化             | 逐步减仓至清仓          |
| 发现更好的机会         | 减仓腾出资金            |

### 3.4 熔断机制

| 条件                     | 动作                           |
| ------------------------ | ------------------------------ |
| 单笔亏损 > 2%            | 暂停该标的后续操作              |
| 单月回撤 > 5%            | 暂停所有新开仓，只允许减仓      |
| 连续 3 笔亏损            | 暂停交易 1 周，复盘系统         |
| 情绪失控（愤怒/恐惧）    | 当天不交易                      |

## 第四章：风险控制体系

### 4.1 仓位管理

```
总仓位建议：
- 满仓上限：80%（永远保留 20% 现金）
- 正常仓位：50-70%
- 防守仓位：30-50%（市场不明朗时）
```

### 4.2 止损规则

| 止损类型     | 适用场景         | 止损幅度  |
| ------------ | ---------------- | --------- |
| 硬止损       | 所有持仓         | -8% 到 -10% |
| 技术止损     | 趋势交易         | 跌破关键支撑 |
| 时间止损     | 试错仓           | 持有 30 天无表现 |
| 逻辑止损     | 所有持仓         | 买入逻辑失效 |

### 4.3 风险回报比

**最低要求：1:2**

```
潜在收益 ≥ 2 × 潜在亏损

示例：
买入价：100 元
止损价：92 元（亏损 8%）
→ 目标价至少 116 元（盈利 16%）
```

### 4.4 相关性控制

- 同一行业持仓不超过总仓位的 **30%**
- 同一主题（如 AI）持仓不超过总仓位的 **40%**
- 避免持仓之间高度相关（如同一供应链上下游）

### 4.5 现金管理

- 永远保留至少 **20%** 现金
- 现金不是"闲置"，是"弹药"
- 市场恐慌时，现金是最有价值的资产

## 第五章：AI 辅助决策系统

### 5.1 AI 负责什么

| AI 的职责                | 说明                              |
| ------------------------ | --------------------------------- |
| 产业趋势扫描             | 发现未来 3-5 年的结构性变化        |
| 财报数据整理             | 快速提取关键财务指标               |
| 估值模型计算             | DCF、可比公司分析                  |
| 新闻情绪分析             | 监控市场情绪变化                   |
| 风险信号预警             | 异常波动、负面新闻提醒             |

### 5.2 AI 不负责什么（必须由人做）

| 人的职责           | 说明                              |
| ------------------ | --------------------------------- |
| 最终买入决策       | AI 提供信息，人做判断              |
| 仓位大小决定       | 必须由人根据风险偏好决定           |
| 情绪管理           | AI 无法替代纪律执行                |
| 战略方向判断       | 产业理解需要人的直觉和经验         |
| 止损执行           | 必须由人下决心                     |

### 5.3 AI 工作流

```
每日：
  - 市场概览扫描
  - 持仓标的关键新闻
  - 异常信号检测

每周：
  - 产业趋势更新
  - 持仓估值复核
  - 机会池更新

每月：
  - 组合绩效分析
  - 风险指标检查
  - 系统规则审视

每半年：
  - 交易系统全面复盘
  - 规则推翻与重建
  - 能力圈重新评估
```

```markdown
- 占总仓位：____%

1. 
2. 
3. 

### 失效条件

### 止损价：
### 目标价：
### 风险回报比：

### 当时情绪状态
- [ ] 冷静理性
- [ ] 有些焦虑
- [ ] FOMO（害怕错过）
- [ ] 恐慌
- [ ] 过度自信

### 是否违反规则
- [ ] 没有违反
- [ ] 违反了：__________

### 事后复盘（卖出后填写）
- 结果：盈利/亏损 ____%

```

### 6.2 复盘框架

**月度复盘：**

1. 本月交易笔数
2. 胜率（盈利笔数 / 总笔数）
3. 平均盈亏比
4. 最大单笔盈利 / 亏损
5. 是否有违反规则的交易
6. 情绪管理评分（1-10）
7. 下月改进点

**半年度复盘：**

1. 交易系统是否还有效？
2. 哪些规则需要修改？
3. 能力圈是否有变化？
4. 市场环境是否变化？
5. 下半年重点方向

## 附录：检查清单

### A. 买入前检查清单

- [ ] 我已经完成了充分的研究

- [ ] 我知道什么情况下我的判断是错的

- [ ] 当前总仓位未超过 80%
- [ ] 同行业仓位未超过 30%

- [ ] 如果今天没有持仓，这个价格我会买吗？（答案必须是"会"）

- [ ] 当前仓位未触及上限

### C. 卖出前检查清单

- [ ] 是否触及止损价？
- [ ] 买入逻辑是否还成立？

### D. 每周检查清单

- [ ] 审视所有持仓的买入逻辑是否还成立
- [ ] 检查总仓位和行业集中度
- [ ] 更新交易日志
- [ ] 记录本周学到的一件事
