# 互联网终端域（Digital Channel Domain）

> 本文档覆盖券商APP、小程序、线上运营等核心业务对象的状态机、规则引擎与测试点。

---

## 一、APP核心功能

### 功能模块
| 模块 | 核心功能 | 优先级 |
|------|---------|--------|
| 开户 | 在线开户、身份认证、视频见证、风险测评 | P0 |
| 交易 | 股票/基金/债券买卖、撤单、持仓查询 | P0 |
| 行情 | 实时行情、K线图、自选股、涨跌排行 | P0 |
| 资讯 | 7×24新闻、公告、研报 | P1 |
| 账户 | 资金查询、交割单、银证转账 | P0 |
| 消息 | 交易通知、活动推送、系统公告 | P1 |

### 开户流程（状态机）
| 状态 | 说明 | 下一状态 |
|------|------|---------|
| 未开始 | 用户进入开户页面 | 身份验证 |
| 身份验证 | OCR识别身份证+活体检测 | 基本信息确认 |
| 基本信息确认 | 确认个人信息 | 风险测评 |
| 风险测评 | 评估投资者风险等级 | 视频见证 |
| 视频见证 | 与客服视频确认开户意愿 | 协议签署 |
| 协议签署 | 电子签名签署开户协议 | 审核 |
| 审核 | 后台审核开户材料 | 开户成功/开户失败 |
| 开户成功 | 账户开通 | — |
| 开户失败 | 材料不合格，需重新提交 | 身份验证（重新开始） |

### 交易下单流程
| 步骤 | 操作 | 校验 |
|------|------|------|
| 1. 选择股票 | 搜索/自选股点击 | 股票代码有效性 |
| 2. 输入委托 | 价格+数量 | 涨跌停校验、资金校验、最小单位(100股) |
| 3. 确认下单 | 确认弹窗 | 二次确认 |
| 4. 提交委托 | 发送到柜台 | 委托校验+风控检查 |
| 5. 返回结果 | 成功/失败 | 委托号/错误原因 |

## 二、小程序

### 平台支持
| 平台 | 功能范围 | 限制 |
|------|---------|------|
| 微信小程序 | 行情+资讯+账户查询 | 不支持交易下单 |
| 支付宝小程序 | 行情+资讯 | 功能子集 |

### 授权登录
- **静默授权**：获取用户OpenID（无需用户确认）
- **显式授权**：获取用户昵称、头像（需用户确认）
- **手机号授权**：绑定手机号（需用户点击按钮确认）

### 小程序与APP数据同步
- 自选股：小程序修改后实时同步到APP
- 账户信息：登录后自动同步
- 设置偏好：通知设置、主题等

## 三、线上运营

### 活动管理
| 状态 | 说明 | 操作 |
|------|------|------|
| 草稿 | 活动创建未发布 | 编辑/删除 |
| 待审核 | 提交审核 | 审批 |
| 已通过 | 审核通过待上架 | 上架 |
| 进行中 | 活动进行中 | 监控/暂停 |
| 已结束 | 活动到期 | 统计/归档 |

### 用户画像
| 维度 | 数据来源 | 用途 |
|------|---------|------|
| 基本信息 | 注册信息 | 客户分层 |
| 交易行为 | 交易记录 | 活跃度分析 |
| 浏览行为 | 页面埋点 | 兴趣偏好 |
| 资产情况 | 账户数据 | 资产分层 |

### 精准推送
- **触发式推送**：交易成交通知、资金变动通知
- **定时推送**：每日早报、盘后总结
- **条件推送**：自选股涨跌幅超阈值
- **运营推送**：活动通知、产品推荐

### AB测试
| 要素 | 说明 | 示例 |
|------|------|------|
| 页面布局 | 不同UI方案对比 | 列表 vs 卡片 |
| 推荐算法 | 不同推荐策略对比 | 协同过滤 vs 内容推荐 |
| 运营文案 | 不同话术对比 | 利益点 vs 功能点 |
| 按钮颜色 | 不同视觉方案 | 红色 vs 蓝色 |

## 四、用户体验要求

### 性能指标
| 指标 | 要求 | 测试方法 |
|------|------|---------|
| 首屏加载 | <2秒 | 弱网环境测试 |
| 页面切换 | <500ms | 动画流畅度 |
| 接口响应 | <1秒 | API压测 |
| 内存占用 | <200MB | 长时间运行监控 |
| 崩溃率 | <0.1% | 线上监控 |

### 适配性
- **iOS**：iPhone 8及以上，iOS 14+
- **Android**：主流机型（华为/小米/OPPO/vivo），Android 9+
- **屏幕尺寸**：4.7寸~6.9寸全覆盖
- **横竖屏**：行情K线支持横屏

## 五、安全要求

### 登录认证
| 方式 | 安全等级 | 场景 |
|------|---------|------|
| 密码登录 | 基础 | 首次登录 |
| 短信验证码 | 中等 | 快捷登录 |
| 指纹/面容 | 高 | 生物识别 |
| 手势密码 | 中等 | APP解锁 |

### 交易安全
- **交易密码**：下单时必须输入（可设置免密额度）
- **设备绑定**：新设备首次交易需短信验证
- **异常登录检测**：异地/IP变化触发二次验证
- **会话超时**：5分钟无操作自动锁定

### 数据安全
- 传输加密：HTTPS/TLS 1.2+
- 本地存储加密：敏感信息加密存储
- 截屏防护：交易页面禁止截屏
- 离场清除：APP切后台后模糊遮罩

## 六、常见测试场景

### 开户测试
- 完整开户流程端到端验证
- 身份证OCR识别准确性
- 活体检测（照片攻击/视频攻击）
- 开户中断后恢复（断网/杀进程）
- 风险测评超时处理

### 交易测试
- 正常下单全流程
- 涨跌停价格限制
- 资金不足时下单拦截
- 撤单成功/失败
- 网络异常时重试机制

### 推送通知测试
- 触发式推送即时性（<5秒）
- 推送内容准确性
- 推送开关控制
- 角标数字更新

### 多设备测试
- 同一账号多设备登录互踢
- 设备解绑流程
- 自选股跨设备同步

### 边界值
- 首屏加载（1.9s vs 2.0s）
- 交易密码错误次数上限（4次 vs 5次）
- 自选股数量上限（99 vs 100）
- 单次银证转账金额上限
- APP后台存活时间

## 七、常见缺陷模式

1. **弱网超时白屏**：网络不稳定时页面加载失败无友好提示
2. **K线数据闪烁**：行情更新时K线图频繁重绘
3. **推送延迟**：交易成交通知延迟超30秒
4. **内存泄漏**：长时间使用后APP卡顿/崩溃
5. **生物识别降级**：指纹/面容不可用时未降级到密码
6. **自选股同步丢失**：APP和小程序自选股不一致
7. **截屏防护失效**：交易页面可被系统截屏
8. **热更新异常**：发版后部分用户未获取最新版本
9. **银证转账重复提交**：网络超时重试导致重复转账
