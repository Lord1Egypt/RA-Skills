## Use Case: <br>
Developers and engineers use this skill when building or reviewing Remix v2 applications that need correct document head metadata, route links, cookie sessions, authentication checks, and CSRF protection. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Auth, session, and CSRF examples can weaken an application if copied without adapting cookie flags, secret rotation, validation coverage, library versions, or locale settings. <br>
Mitigation: Review the generated guidance before production use and verify secure cookie settings, secret rotation, CSRF validation on mutating actions, current Remix-related library versions, and app-specific locale values. <br>
