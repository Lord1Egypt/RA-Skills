## Description: <br>
Interactive project scaffolding skill that creates Node.js, Python, Go, or Rust project structures with README, CI/CD, Docker, linting, testing, license, and editor configuration files. <br>

This skill is ready for commercial/non-commercial use. <br>

## Publisher: <br>
[ryudi84](https://clawhub.ai/user/ryudi84) <br>

### License/Terms of Use: <br>
MIT <br>


## Use Case: <br>
Developers and engineering teams use this skill to bootstrap runnable application or library projects with common repository files, build automation, container files, and starter tests. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: The skill writes many files and directories into the chosen output location. <br>
Mitigation: Run with --dry-run first and choose a simple project name and output directory before allowing writes. <br>
Risk: The skill may initialize a Git repository and copy configured git name or email into generated project files. <br>
Mitigation: Use --no-git-init when repository initialization is not wanted and review generated author metadata before publishing. <br>
