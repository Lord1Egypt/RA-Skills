# 我的宝可梦库 v1 — 设计文档

```
/pvp 添加 <宝可梦> <联盟> <IV> <CP> <等级>
/pvp 我的
/pvp 我的 <联盟>
/pvp 评估我的 <联盟>
```

---

## 目标

记录用户已有的宝可梦，作为后续培养、配队、评估的基础数据层。IV 排名模块提供「理论最佳」，宝可梦库提供「我有什么」，两者结合才能回答「值不值得培养」。

## 数据文件

`data/my_pokemon.json`

结构：

```json
[
  {
    "name": "弃世猴",
    "speciesId": "annihilape",
    "league": "1500",
    "iv": [2, 15, 15],
    "cp": 1497,
    "level": 17,
    "moves": {
      "fastMove": "踢倒",
      "chargedMove1": "近身战",
      "chargedMove2": "愤怒之拳"
    },
    "built": false,
    "note": "今天抓的，还没二招",
    "createdAt": "2026-05-19T20:00:00+08:00",
    "updatedAt": "2026-05-19T20:00:00+08:00"
  }
]
```

### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| name | string | 是 | 中文宝可梦名，显示用 |
| speciesId | string | 是 | PvPoke 英文 ID，用于对接 gamemaster |
| league | string | 是 | `1500` / `2500` / `master` |
| iv | [number, number, number] | 是 | `[攻击IV, 防御IV, 生命IV]` |
| cp | number | 是 | 当前 CP |
| level | number | 是 | 当前等级 |
| moves | object | 否 | `{ fastMove, chargedMove1?, chargedMove2? }` |
| built | boolean | 否 | 是否已培养完成（默认 false） |
| note | string | 否 | 备注 |
| createdAt | string | 是 | ISO 8601 |
| updatedAt | string | 是 | ISO 8601 |

### 约束

- 同 `speciesId + league` 不重复（一只宝可梦在一个联盟只能有一条）
- 如需记录同一宝可梦在不同联盟（如 1500/2500），允许重复 speciesId，但 league 不同
- 大师联盟不录入 IV 排名，但可以记录已有宝可梦（用于配队）

## 命令设计

### 1. `/pvp 添加 <宝可梦> <联盟> <IV> <CP> <等级> [招式] [--built] [--note]`

录入宝可梦。

```
/pvp 添加 弃世猴 1500 2/15/15 1497 17 招式=踢倒/近身战/愤怒之拳 --built --note=二招已开
```

**参数解析规则：**
- `<宝可梦>` — 中文名，通过 `findSpeciesId` 转换为 speciesId
- `<联盟>` — `1500` / `2500` / `master`
- `<IV>` — `a/d/h` 格式
- `<CP>` — 纯数字
- `<等级>` — 纯数字（支持小数，如 17 / 45.5）
- `<招式>` — 可选，`fast/charged1/charged2` 或 `招式=踢倒/近身战/愤怒之拳`
- `--built` — 可选标记，表示已培养完成
- `--note=<...>` — 可选备注

**验证逻辑：**
- IV 范围 0~15，超出返回错误
- CP 必须为正整数，不超过联盟上限
- 等级 1~51（含小数）
- 重复 `speciesId + league` 时更新已存在记录（覆盖 IV/CP/Level/招式/备注），不新增

### 2. `/pvp 我的`

列出所有已录入宝可梦，按联盟分组。

```
/pvp 我的
```

输出示例：

```
我的宝可梦 (共 6 只)

=== 超级联盟 ===
弃世猴  2/15/15  CP1497  Lv17    ✅已培养
玛力露丽  0/14/15  CP1500  Lv46    ⏳未培养  距最佳差-1防御

=== 高级联盟 ===
烈咬陆鲨  2/15/11  CP2496  Lv26    ⏳未培养

=== 大师联盟 ===
帝牙卢卡  15/15/15  CP4273  Lv40    ✅已培养
```

**显示规则：**
- 联盟分组排序：超级联盟 → 高级联盟 → 大师联盟
- 每组内按 `built` 降序（已培养优先）
- 已培养显示 ✅，未培养显示 ⏳
- 未培养且 IV 排名可查时，显示**距最佳 IV 的差值**（来自 `/pvp iv` 模块）

### 3. `/pvp 我的 <联盟>`

筛选指定联盟的宝可梦。输出格式同上，只显示该联盟。

```
/pvp 我的 1500
```

### 4. `/pvp 评估我的 <联盟>`

评估指定联盟下已有宝可梦的培养优先级。

```
/pvp 评估我的 1500
```

输出结构（按优先级排序）：

```
=== 超级联盟 培养优先级 ===

1. 玛力露丽 0/14/15  → 距最佳差-1防御，优先补
2. 胖嘟嘟 1/14/14  → 距最佳差-1生命，次优先
3. 弃世猴 2/15/15  → 已是最佳，跳过
```

**评估逻辑：**
对每个宝可梦：
1. 调用 `/pvp iv` 计算该宝可梦在该联盟下的最佳 IV 列表
2. 查找用户 IV 在排名中的位置
3. 按 **排名靠后（即偏差大）优先** 排序
4. 已 built 的排在最后（或标注已培养）

**不适合评估的情况（直接跳过）：**
- 大师联盟（无 IV 排名）
- 该联盟无生效排名数据（未抓到/未缓存）
- 已标记 built

## 后续扩展用途

### 1. 查已捕

```
/pvp 我的 → 快速看有什么
```

### 2. 判断接近最佳 IV

录入时自动对比 `/pvp iv` 结果，提示「距最佳差 X」。

### 3. 培养推荐

评估功能输出优先级。

### 4. 配队前置

后续 `/pvp team` 配队命令中，优先从已有宝可梦库中选。

## 不做的（v1 红线）

- ❌ 不提供图形化界面（纯文本 discord/微信）
- ❌ 不批量导入/导出
- ❌ 不自动同步游戏数据（无 PoGo API）
- ❌ 不提供删除命令（覆盖更新即可）
- ❌ 不提供搜索/过滤（后续可加）
- ❌ 不提供「进化后」级联处理
