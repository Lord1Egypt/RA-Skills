# 两层过滤器 v2.0

## 设计理念

过滤分两层执行，与浏览流程对应：
- **Tier 1（粗筛）**：在搜索结果/Feed 页面完成，不需要进入笔记
- **Tier 2（精筛）**：进入笔记详情页后执行

这种分层设计减少了不必要的页面导航，提高了效率。

---

## Tier 1：粗筛（搜索结果 / Feed 页面）

在笔记卡片上可见的信息进行快速判断。

### 1.1 点赞数检查

从卡片上读取点赞数，判断是否在范围内：

```
点赞数提取方式：
  方式1（推荐）：截图后视觉识别卡片上的数字
  方式2：javascript_tool 提取（DOM结构可能变化）

判断：
  min_likes <= 点赞数 <= max_likes
  
特殊格式处理（通过 javascript_tool）：
  javascript_tool(`
    var t = "LIKE_TEXT";
    t.includes('万') ? parseFloat(t) * 10000 :
    t.toLowerCase().includes('w') ? parseFloat(t) * 10000 :
    parseInt(t.replace(/,/g, '')) || 0
  `)
```

### 1.2 标题关键词检查

从卡片上可见的标题文字判断：

```
通过条件：标题包含 config.filters.note.keywords_include 中至少一个词
排除条件：标题包含 config.filters.note.keywords_exclude 中任一词 → 跳过

注意：
- 标题可能被截断，只能看到前20-30字
- 标题不含目标关键词不一定要跳过（搜索结果本身已按关键词过滤）
- 但标题含排除词的一定要跳过
```

### 1.3 去重检查

```
Bash: grep -q "NOTE_ID_OR_URL" data/interacted-notes.txt && echo "DUP" || echo "NEW"

NOTE_ID 提取：
  - 从笔记卡片的链接中提取（如 /explore/687858880000000010027e9c）
  - 或从 URL 路径中截取唯一标识

如果是 "DUP" → 跳过，不进入
```

### 1.4 粗筛决策

```
通过粗筛的条件（全部满足）：
  ✅ 点赞数在范围内
  ✅ 标题不含排除词
  ✅ 未在去重列表中

任一条件不满足 → 跳过此笔记，继续浏览下一个
```

## Tier 2：精筛（笔记详情页）

进入笔记后，利用可见的完整信息做精确判断。

### 2.1 发布时间检查

```
在笔记详情页查找发布日期：
  - 通常在作者名下方或笔记末尾显示（如 "04-14"、"2025-07-20"）
  - 通过截图视觉识别或 find 搜索日期文本

判断：发布时间在 config.filters.detail.publish_within_days 天以内
  如果超出 → navigate("back")，跳过
```

### 2.2 内容关键词复检

```
通过 read_page 或截图阅读笔记正文内容：
  - 确认正文中包含目标关键词（至少一个）
  - 确认正文不含排除词

这一步在已经搜索目标关键词时通常能通过，
主要用于发现页模式(discover_feed)的筛选。
```

### 2.3 内容质量快检

```
以下情况视为低质量，建议跳过：
  - 正文极短（< 50 字），只有标题没有实质内容
  - 明显的广告软文（大量产品链接、引导关注公众号）
  - 内容与标题严重不符
  
判断方式：Claude 在阅读内容时直接判断
```

### 2.4 作者检查（仅在考虑关注时）

```
如果本次可能执行"关注"动作，需额外检查作者：
  - 作者有头像（非默认头像）
  - 作者昵称非纯数字/纯字母（机器人特征）
  - 如果能看到粉丝数：在合理范围内

判断方式：观察笔记顶部的作者信息区域
```

### 2.5 精筛决策

```
通过精筛 → 执行互动动作
未通过精筛 → 记录原因，关闭笔记，继续下一个

注意：即使通过精筛，也有 20% 概率故意不互动
（config.anti_detection.skip_qualifying_note_probability）
这是为了避免"每个合格笔记都互动"的机器人特征
```

## 搜索策略

### 关键词轮换

```
从 config.targets.keywords 中选择关键词：
  如果 keyword_rotation == "random":
    javascript_tool("KW_LIST[Math.floor(Math.random() * KW_LIST.length)]")
  如果 keyword_rotation == "sequential":
    按列表顺序依次使用

每个关键词使用条件（满足任一则换词）：
  - 已在该关键词下处理 max_notes_per_keyword 篇笔记
  - 连续 max_scroll_without_match 屏无合格笔记
  - 已浏览该关键词搜索结果超过 3 页
```

### 搜索执行

```
1. 在搜索框输入关键词
2. 等待结果加载（2-4秒）
3. 浏览第一屏结果，做 Tier 1 粗筛
4. 找到合格笔记 → 进入，做 Tier 2 精筛
5. 处理完毕 → 返回搜索结果，继续浏览
6. 需要更多结果 → scroll down 加载
7. 达到换词条件 → 更换关键词
```

## 特殊情况处理

### 合格笔记不足

```
如果搜索多个关键词后仍难以找到 min_likes >= 100 的笔记：
  - 考虑降低 min_likes 阈值（临时降至 50）
  - 或切换到发现页模式（首页推荐流通常有更多高赞内容）
  - 或搜索更宽泛的关键词（如"期货"替代"期货技术分析"）
```

### 商业推广笔记

```
以下特征的笔记应跳过（即使点赞数合格）：
  - 标题含"开户""手续费""返佣""福利"
  - 作者名含"期货公司""XX证券""XX营业部"
  - 正文大量引导添加联系方式
  
这类笔记互动可能导致账号被关联为"水军"
```
