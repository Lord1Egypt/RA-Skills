# WT8麦语言量化交易助手 - 快速入门

## 简介

这是一个基于文华财经WT8软件的麦语言量化交易AI助手。它可以帮助你：

- 🔍 **快速分类需求**：自动识别是行情查询、历史数据、账户查询、下单交易、模拟运行还是回测分析
- 📝 **生成正确代码**：使用真实存在的函数和示例，避免泛化错误
- ⚠️ **避开陷阱**：明确标注边界条件和常见错误
- 📤 **即取即用**：可直接分发给其他AI使用

## 快速开始

### 1. 判断需求类型

当你提出问题时，AI会自动判断属于哪种类型：

| 类型 | 关键词 | 说明 |
|------|--------|------|
| 行情 | 价格、行情、当前价、开盘、最高、最低 | 获取实时行情数据 |
| 历史数据 | 历史、K线数据、收盘价、成交量 | 获取历史K线数据 |
| 账户查询 | 账户、资金、权益、持仓、盈亏 | 查询账户信息 |
| 下单 | 买开、卖开、买平、卖平、BK、SK | 交易指令 |
| 模拟 | 模拟、下单、信号、页面盒子 | 模拟实盘运行 |
| 回测 | 回测、测试、收益、回撤、胜率 | 历史数据回测 |

### 3. 基础模型框架

```mql
// 条件A，买开
条件A, BK;
// 多头止损
条件B, SP;
// 条件C，卖平，结束本次交易
条件C, SP;
// 条件A1，卖开
条件A1, SK;
// 条件B1，空头止损
条件B1, BP;
// 条件C1，买平，结束本次交易
条件C1, BP;
AUTOFILTER;  // 必须写，过滤信号
```

## 常见问题

### Q: 为什么模型没有执行信号？
A: 检查是否写了 `AUTOFILTER;`，这是必须的。

### Q: 为什么信号忽闪？
A: K线走完前满足条件出信号，K线走完条件不满足信号消失。使用 `SIGCHECK` 函数可以进行信号复核。

### Q: 如何避免过度拟合？
A: 参数优化时追求"参数高原"而非"参数孤岛"，选择收益稳定、适合多种趋势场景的参数。
