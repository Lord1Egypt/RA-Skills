# Seedream 4.0-5.0 提示词工程规范 (Layer 1 — 按需加载)

> 本文件包含 Seedream 5.0 lite / 4.5 / 4.0 的提示词方法论，补充 Seedance 2.0 规范中未覆盖的文生图、图像编辑、参考图生图、多图输入/输出等能力。
> **加载条件**: 用户使用 Seedream 4.x/5.x 平台，或涉及文生图、图像编辑、参考图生成、多图操作、知识可视化等场景时加载。

---

## 1. 通用规则 (General Rules)

### 1.1 自然语言清晰描述
用**简洁连贯**的自然语言写明 **主体 + 行为 + 环境**，对美学有要求时补充 **风格、色彩、光影、构图**。

| 推荐 | 避免 |
|------|------|
| 一个穿着华丽服装的女孩，撑着遮阳伞走在林荫道上，莫奈油画风格 | 一个女孩，撑伞，林荫街道，油画般的细腻笔触 |

### 1.2 明确应用场景
有明确应用场景时，在提示中写明图像用途和类型。

| 推荐 | 避免 |
|------|------|
| 设计一个游戏公司的 logo，主体是一只在用游戏手柄打游戏的狗，logo 上写有公司名 "PITBULL" | 一张抽象图片，狗拿着游戏手柄，狗狗上写 PITBULL |

### 1.3 风格渲染
有明确风格需求时，使用精准的**风格词**或提供**参考图像**。

### 1.4 文字渲染
将要生成的**文字内容**放在**双引号**中。

| 推荐 | 避免 |
|------|------|
| 生成一张海报，标题为 "Seedream 4.5" | 生成一张海报，标题为 Seedream 4.5 |

### 1.5 图像编辑目标明确
使用**简洁明确的指令**，说明需要修改/参考的对象及具体操作，避免指代模糊的代词；希望保持不变的部分需强调。

| 推荐 | 避免 |
|------|------|
| 让图中最高的那只熊猫穿上粉色的京剧服饰并戴上头饰，并保持动作不变 | 让它穿上粉色衣服 |

## 2. 文生图 (Text-to-Image)

### 2.1 核心原则
Seedream 5.0 lite / 4.5 / 4.0 对文本理解能力更强，**简洁精确的提示通常优于重复堆叠华丽复杂的词汇**。画面不再泛白。

### 2.2 高密度细节描述
对于细节丰富的图像，通过详细的文本描述精准控制画面细节。每个元素逐一描述其位置、状态、内容。

**示例结构**:
```
[场景概述]。[元素1位置]: [元素1描述]；[元素2位置]: [元素2描述]；... 背景是[背景描述]。[光影描述]。
```

### 2.3 知识可视化
可将知识与推理结果转化为高密度图像内容（公式、图表、教学插图等）。

**关键要求**:
- 使用**专业术语**确保知识点表达准确
- 写清对生成图像的具体要求（可视化形式、版式、风格）

**示例**:
- 在黑板上画出二元一次方程组及其相应的解法步骤
- 绘制一张信息图，展示通货膨胀的成因，每条成因独立呈现，并配有简洁图标

## 3. 图像编辑 (Image Editing)

### 3.1 四种编辑操作

| 操作 | 指令要点 | 示例 |
|------|---------|------|
| **增加** | 明确指出增加的对象和位置 | 给图中女生增加相同款式的银色耳线和项链 |
| **删除** | 明确指出删除的对象 | 去掉女生的帽子 |
| **替换** | 指明替换对象 + 替换为 + 保持不变的部分 | 把最大的面包人换成牛角包形象，保持动作和表情不变 |
| **修改** | 指明修改对象 + 修改属性 + 保持不变的部分 | 让图中三个机器人变成透明水晶材质，颜色从左到右分别变成红黄绿 |

### 3.2 视觉信号控制
当画面复杂难以通过文本准确描述编辑对象时，可采用以下视觉信号：

| 信号类型 | 用法 | 示例 |
|---------|------|------|
| **涂鸦** | 用颜色涂抹标记位置，在 prompt 中指明颜色对应的操作 | 将房间内红色涂抹位置放入电视，蓝色涂抹位置放入沙发，不改变其他布局 |
| **线框** | 用线框标记区域和大小，在 prompt 中指明线框对应的操作 | 放大图中的标题至红框大小，并改成和萨克斯图案一样的颜色和字体风格 |
| **箭头** | 用箭头指示方向和目标 | — |

## 4. 参考图生图 (Reference-to-Image)

### 4.1 四种参考类型

| 参考类型 | 提取信息 | 提示词结构 |
|---------|---------|-----------|
| **参考人物形象** | 角色外貌特征 | 参考图中的人物形象 + [生成画面描述] |
| **参考风格** | 艺术风格/视觉语言 | 参考图的[风格描述] + [生成内容描述] |
| **参考虚拟实体形象** | 3D/IP/虚拟角色 | 图中的形象变成[目标材质/形态] + [场景描述] |
| **参考款式** | 产品设计/服装款式 | 衣服款式与图中一致 + [变化描述] |

### 4.2 提示词双要素
参考图生图的提示词必须包含两部分：
1. **指明参考对象**: 清晰描述希望从参考图中提取并保留的元素
2. **描述生成画面**: 具体说明希望生成的画面内容、场景等细节

### 4.3 设计草图→高保真
从设计草图生成高保真效果图时：
1. 提供清晰的原始图片，若图中有文字说明则注明"遵循图中文字内容进行生成"
2. 明确主体与要求（如高保真 UI 界面、现代简约风格客厅实景图）
3. 明确指出需与参考图保持一致的关键要求（如家具位置与参考图一致）

## 5. 多图输入 (Multi-Image Input)

### 5.1 三种复合操作

| 操作 | 提示词结构 | 示例 |
|------|-----------|------|
| **替换** | 将图一的[元素]替换为图二的[元素] | 将图一的主体替换为图二的主体 |
| **组合** | 让图一[元素]穿上/使用图二的[元素] | 让图一人物穿上图二的服装 |
| **迁移** | 参考图二的风格，对图一进行风格转换 | 参考图的风格，对图一进行风格转换 |

### 5.2 多图引用规范
- 使用"图一/图二/图三"明确指代不同输入图像
- 每张图的参考要素需明确（人物/服装/风格/产品）
- 多图之间需兼容：光源逻辑、造型、质量水平

---

## 6. 多图输出 (Multi-Image Output)

### 6.1 触发方式
通过以下提示词触发组图生成：
- "一系列"/"一套"/"组图"
- 具体数字表明图片数量

### 6.2 适用场景
| 场景 | 说明 |
|------|------|
| 分镜序列 | 角色连贯、风格统一的分镜图 |
| 漫画创作 | 多格漫画的连续画面 |
| IP 产品 | 统一视觉风格的成套设计 |
| 表情包 | 同一角色的多种表情/动作 |

## 7. Seedream 4.x/5.x 与 Seedance 2.0 差异速查

| 维度 | Seedance 2.0 (视频) | Seedream 4.x/5.x (图片) |
|------|---------------------|--------------------------|
| 核心输出 | 视频分镜 | 静态图片/组图 |
| 提示词结构 | 5层: Subject→Action→Camera→Style→Constraints | 3层: 主体+行为+环境 + 美学补充 |
| 文字渲染 | 不涉及 | 双引号包裹文字内容 |
| 图像编辑 | 无 | 增加/删除/替换/修改 + 视觉信号 |
| 参考图 | Reference-to-Video | Reference-to-Image (4种类型) |
| 多图输入 | 无 | 替换/组合/迁移 |
| 多图输出 | 多镜头序列 | 组图 (分镜/漫画/IP/表情包) |
| 知识可视化 | 无 | 公式/图表/教学插图 |
| Negative Prompt | 必须 | 不适用 (图片模式) |
| Motion Strength | 必须 | 不适用 |

---

## 8. 平台适配提示

当 MadStory 检测到用户使用 Seedream 4.x/5.x 平台时：
- 自动切换为图片生成模式，移除 Camera/Motion 相关参数
- 应用双引号文字渲染规则
- 启用图像编辑/参考图/多图操作语法
- 知识可视化场景使用专业术语
- 不注入 Negative Prompt（图片模式不需要）

## 9. 实战示例 (Prompt Examples)

> 以下示例来自 Seedream 4.0-5.0 官方指南，展示不同场景下的高质量提示词写法。

### 9.1 文生图 — 高密度细节描述

**示例：凌乱办公桌**
```
一个凌乱的办公桌桌面。桌面上有一台开着的笔记本电脑，屏幕显示绿色代码；旁边一个马克杯，杯上写着"Developer"，杯口冒出热气；一本摊开的书，页面是维恩图，展示三个圆的嵌套关系，三个圆分别为灰色、蓝色和浅绿色；一个便签贴，上面画着一个思维导图，思维导图是上下结构，分为3层结构；一支钢笔，笔帽掉在旁边；钢笔旁边是一个手机，屏幕显示一条新消息通知，桌子角落是一小盆多肉植物。背景是模糊的书架。阳光从右侧照射，在桌上形成光影。
```

**示例：冰箱内部视图**
```
冰箱打开的内部视图：上层: 左边放着一盒牛奶，牛奶盒上绘制了三只大小不一的奶牛，在草原上吃草，右边是一个鸡蛋支架，里面放着八个鸡蛋。中层: 一个盘子，里面装着吃剩的烤鸡，烤鸡上插着一个红色的小旗帜，旁边是一个装满草莓的透明保鲜盒，盒子上绘制有菠萝、草莓和橙子的图案。下层: 蔬菜抽屉里有生菜、胡萝卜和西红柿。冰箱门后面的置物架里放着番茄酱和蛋黄酱。
```

### 9.2 文生图 — 知识可视化

**示例：数学方程**
```
在黑板上画出下列二元一次方程组及其相应的解法步骤：5x + 2y = 26；2x - y = 5。
```

**示例：信息图**
```
绘制一张信息图，展示通货膨胀的成因，每条成因独立呈现，并配有简洁图标。
```

### 9.3 图像编辑 — 替换+保持不变

**示例：替换面包人**
```
把最大的面包人换成牛角包形象，保持动作和表情不变。
```

**示例：修改材质+颜色**
```
让图中三个机器人变成透明水晶材质，颜色从左到右分别变成红黄绿。并让绿色的机器人跑起来，黄色的走路，红色的站立。
```

### 9.4 参考图生图 — 参考人物形象

**示例：角色手办**
```
参考图中的人物形象，做一个动漫人物手办，放在桌上，后面放置一个印有角色图像的生日礼物包装盒，盒子下面有一本书，在包装盒前面，添加一个圆形塑料底座，角色手办站在上面，将场景设置在室内，尽可能真实；生成尺寸和现在图一样；手办在图片的左边；整个图片的风格和原始图一样，大片摄影感。
```

### 9.5 参考图生图 — 参考风格

**示例：应用 icon 设计**
```
参考图标的线性简约风格，设计9个不同场景下的应用 icon，包括：音乐、天气、日历、相机、聊天、地图、闹钟、购物车、笔记本，保持一致的配色。
```

### 9.6 多图输入 — 组合

**示例：换装**
```
让图一人物穿上图二的服装。
```

### 9.7 设计草图→高保真

**示例：手绘原型→UI**
```
这是一个 Web 端房屋租赁软件的详情页手绘图，请根据图中文字示意，将其渲染成高保真的 UI 界面，并在画廊区域增加一些示例图片，在房屋信息和预定信息区域增加一些示例的文字信息。
```
