# Generated Specialized Figure Skill Output Spec

Version: 1.0.1

A specialized figure-making skill generated by this factory must be self-contained, publishable, and reusable for arbitrary target papers of the same figure class.

## Required Generated Workflow

Generated skills must include this production workflow or a stricter domain-specific equivalent:

| Step | Mode | Required Meaning |
|---|---|---|
| P1 | TEXT_ONLY | Intake material and optional references |
| P2 | TEXT_ONLY | Diagnose figure need and subtype routing |
| P3 | TEXT_ONLY | Produce 4-6 text candidates, normally 6 |
| P4 | TEXT_ONLY | Set up the visual candidate board |
| P5 | IMAGE_ONLY | Generate/display 4-6 candidate images or schematic candidates, normally 6 |
| P6 | TEXT_ONLY | Record image batch, compare candidates, and lock/revise direction |
| P7 | TEXT_ONLY | Build final image brief/prompt |
| P8 | IMAGE_ONLY | Generate formal figure candidate or revision batch |
| P9 | TEXT_ONLY | Review, caption, legend, body text, and handoff |

P4/P5/P6 are mandatory. They are the explicit candidate-image bridge that prevents generated skills from stopping at text candidates.

## Required Content

The generated skill must include:

- first-trigger startup gate;
- visible plan in every text reply;
- current state/output footer in every text reply;
- all steps and current position in every text footer;
- final next-question help in every text reply, including the unknown-next prompt;
- strict text/image separation;
- explicit response mode on every step;
- default recommendation when presenting multiple options;
- 4-6 text candidates, normally 6;
- 4-6 image candidates, normally 6;
- mandatory visual-candidate setup -> image-only candidate generation -> candidate-review bridge after multi-option text;
- optional sample/reference-image protocol with multiple images and per-image attribute preferences;
- rendering route: ChatGPT web Create image through ChatGPT Images 2.0; Codex `$imagegen` first; if unavailable, ChatGPT Images 2.0 API or another approved image-generation API;
- prohibition against SVG, Mermaid, TikZ, Graphviz, HTML/CSS, canvas, plotting-library exports, filesystem/code-drawn images, or code-rendered visual candidates/drafts/finals/fallbacks;
- lawful source/corpus provenance if the skill depends on a corpus;
- full-feasible-corpus coverage notes when local PDFs or a paper index were available;
- multi-label evidence coverage when figures/PDFs can belong to multiple taxonomy classes;
- release checklist with candidate-bridge tests.

## Minimum Validation Tests

Before locking the specialized skill, run or simulate:

1. First-trigger startup response.
2. First-trigger image request boundary: the first reply remains text-only.
3. State-footer completeness after a text turn.
4. Automatic current-status/current-output listing.
5. Default recommendation after multiple figure schemes.
6. Text candidate turn with 4-6 candidates, normally 6.
7. Candidate bridge setup: after text candidates, the next step is visual candidate-board setup, not final prompt/final image/caption.
8. `IMAGE_ONLY` candidate-board generation with 4-6 candidates, normally 6.
9. Candidate-review turn records the previous image batch and asks the user to select/revise.
10. Formal prompt construction after direction lock.
11. Formal `IMAGE_ONLY` figure generation boundary.
12. Caption/body-text package output.
13. Optional reference-image handling.
14. Corpus coverage and evidence-lineage checks.

Fail validation if any example, starter message, or workflow lets the user go from text candidates directly to final prompt, final image generation, or caption without a candidate-image board, unless it explicitly records a user-requested text-only skip.

```yaml
  status: generated | tested | patched | locked
  lock_grade: none | production_grade | limited | pilot | fallback
  lock_basis: none | full_taxonomy | user_approved_thin | user_approved_pilot | user_approved_fallback
  slug: <skill-slug>
  version: <semver>
  package_path: <path-or-link>
  included_files: []
  known_limitations: []
  test_results:
    startup_gate: passed | failed
    state_footer: passed | failed
    default_recommendation: passed | failed
    candidate_image_bridge: passed | failed
    rendering_boundary: passed | failed
    next_question_help: passed | failed
```

Production-grade lock requires full-taxonomy evidence sufficiency and full-feasible corpus coverage or a recorded justified exception.
