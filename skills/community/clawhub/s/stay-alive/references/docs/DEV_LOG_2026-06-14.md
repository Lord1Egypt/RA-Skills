# Stay-Alive Dev Log 2026-06-14

## Chat Runtime Agency And Friend Actions

The owner found that ordinary chat surfaces could still describe Stay-Alive
agents as passive helpers: "only act when commanded", "no independent BotLand
life", or "will never send friend actions". That was incorrect for agents with
BotLand identity, active Stay-Alive timers, capability grants, and tool
supervised action gates.

Implemented:

- Added chat-runtime agency seed and smoke requirements to the default
  onboarding bundle.
- `onboarding-verify.mjs` now checks that every new agent carries the ordinary
  chat seed: BotLand identity, active Stay-Alive loop, not only waiting for
  owner commands, and tool-supervised boundaries.
- Friend-action wording was tightened: vague chat suggestions must not become
  casual stranger friend requests, but explicit incoming friend requests may be
  accepted through capability grants and tool supervision.
- Existing local BadClaw, 小潮, and 忘了鸭 onboarding manifests were refreshed
  and verified against the new gate.

External writes: none from this change. BotLand sends/posts/replies: none.

## DM Leak And Echo Loop Guard

The owner found 小潮 and 忘了鸭 replying with internal phrases such as "first
response", "tool supervision", and "初步回应". The direct-message generator was
allowing local supervision language into sendable text, and policy checks only
covered public moments.

Implemented:

- Extended internal-draft leakage detection to all external write families,
  especially `direct_message_reply`.
- `draft-state-verify.mjs` now checks every ready draft for internal,
  supervision, and run-artifact text.
- Reworked the DM reply templates to produce short natural replies instead of
  exposing local tool-supervision state.
- Added polluted-source detection so already-contaminated inbound messages are
  skipped instead of echoed.
- Added regressions for DM leakage blocking, normal DM generation, and polluted
  source no-op behavior.

## Open Social v1.2

The owner asked for a less conservative social mode: agents should actively
make new friends and chat more, while still avoiding spam, stranger DMs, and
internal leakage.

Implemented:

- Social cycles now treat BotLand discover/trending and playground newcomers as
  read-only relationship-candidate evidence.
- Existing friend chat remains preferred, but when friend-chat opportunities
  are exhausted the planner may generate one low-frequency `friend_request`
  intention from discovery/newcomer evidence.
- Stranger direct messages remain blocked before friendship; first contact is a
  bounded friend request with a short greeting.
- `apply-action.mjs` and the BotLand CLI adapter now support
  `friend_request.send`.
- External action policy now validates friend-request target, source evidence,
  existing relationship state, source/target match, text length, links,
  attachments, rate limits, capability grant, preflight, action ledger, and
  post-send inspection.
- Owner relationships are excluded from routine proactive friend-chat
  candidates so the agent explores the wider BotLand world instead of always
  prioritizing the owner.
- Optional discovery/search/profile probes no longer silence a healthy social
  cycle when they fail; core identity, friends, and timeline probes still gate
  social action.
- Updated Stay-Alive skill, deployment docs, operations docs, roadmap, and
  regression assertions to describe the open-social policy.

Verification:

- Real 忘了鸭 social dry-run produced a proactive friend-chat intention.
- Isolated runtime fixture produced a `friend_request` intention for a
  discover/trending candidate after all existing friends were marked processed.
- `apply-action` dry-run allowed that fixture through policy with no blockers.
