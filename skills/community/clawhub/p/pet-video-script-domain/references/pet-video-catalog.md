# 萌宠短视频任务清单与依赖拓扑

萌宠短视频按价值链组织的任务清单，附任务间依赖关系和UTOS元操作映射提示。

**域间逻辑流**：P1 角色建档 → P2 故事设计 → P3 分镜编排 → P4 提示词生成

---

## P1 角色建档

| ID | 任务类型 | 说明 | 依赖 | UTOS映射提示 |
|----|---------|------|------|-------------|
| P1-01 | 品种识别 | 确定宠物品种/混血类型，作为视觉描述的基准 | 无（入口） | S |
| P1-02 | 外观描述 | 提取毛色、花纹、体型、五官、配饰等视觉特征 | P1-01 | S→C |
| P1-03 | 性格标签 | 确定宠物性格特征（戏精/呆萌/高冷/粘人等） | P1-01 | S→C |
| P1-04 | 行为特征 | 记录宠物的标志性动作和习惯性行为 | P1-02, P1-03 | S→C→A |

## P2 故事设计

| ID | 任务类型 | 说明 | 依赖 | UTOS映射提示 |
|----|---------|------|------|-------------|
| P2-01 | 故事前提 | 一句话概括故事核心——谁+在哪+做什么+结果 | P1-04 | C→A |
| P2-02 | 情绪风格 | 确定故事基调（搞笑反差/温馨治愈/悬疑反转/日常记录） | P2-01 | C |
| P2-03 | 冲突设计 | 设计故事中的矛盾点和升级节奏 | P2-01, P2-02 | C→A |
| P2-04 | 结局设计 | 确定结局类型（反转/温馨/悬念/爆笑）和情感落点 | P2-03 | C→A |

## P3 分镜编排

| ID | 任务类型 | 说明 | 依赖 | UTOS映射提示 |
|----|---------|------|------|-------------|
| P3-01 | 镜头规划 | 按故事节奏拆分为3-10个镜头，确定每镜头的核心事件 | P2-04 | C→A |
| P3-02 | 景别设计 | 为每个镜头确定景别（特写/近景/中景/全景/远景） | P3-01 | C→A |
| P3-03 | 运镜设计 | 为每个镜头确定运镜方式（固定/推/拉/摇/跟/升降） | P3-01, P3-02 | C→A |
| P3-04 | 时间轴编排 | 为每个镜头分配时长，确保总时长匹配目标平台 | P3-01, P3-02, P3-03 | C→A→G |

## P4 提示词生成

| ID | 任务类型 | 说明 | 依赖 | UTOS映射提示 |
|----|---------|------|------|-------------|
| P4-01 | 主体锁定 | 从P1档案生成统一的英文subject description，所有镜头共用 | P1-02, P1-03 | A |
| P4-02 | 镜头提示词 | 为每个镜头生成Seedance格式英文prompt（40-80词） | P4-01, P3-02, P3-03 | A |
| P4-03 | 音效标注 | 为每个镜头标注BGM风格和音效提示 | P3-01, P3-04 | A |

---

## 依赖拓扑摘要

**主链路**: P1-01 → P1-02 → P1-03 → P1-04 → P2-01 → P2-02 → P2-03 → P2-04 → P3-01 → P3-02/P3-03(并行) → P3-04 → P4-01 → P4-02 → P4-03

**角色链**: P1-01 → P1-02 → P1-03 → P1-04 → P4-01（角色一致性锁定）

**故事链**: P2-01 → P2-02 → P2-03 → P2-04（叙事结构设计）

**分镜链**: P3-01 → P3-02 ‖ P3-03 → P3-04（镜头语言编排）

**提示词链**: P4-01 → P4-02 → P4-03（AI视频生成输入）

更多组合由UTOS根据具体任务动态推导。
