# 路线 B：高精度图表数据提取（后续规划）

## 目标
将当前“图表类型识别+初步数据提取”能力，升级为“高精度图表数据还原”能力。

## 当前状态（v1 增强版）
- ✅ 图表类型识别：pie / bar / line / table
- ✅ 多区域检测：dashboard 分块识别
- ✅ 初步数据提取：能抽到部分标签/数值/坐标
- ✅ 回归测试：test_chart_ocr.py 5/5 通过

## 当前局限
- 饼图：图例文本与数值配对不稳定，易混入无关文字
- 柱状图：类目与数值分离不准，易混进页面标题/KPI
- 折线图：依赖 OCR 文本，几何估计仅作兜底，点位数量有限
- 表格：仅支持简单表格，复杂表格效果一般
- 精度边界：当前更适合“类型识别+初步提取”，不适合“高精度还原原始数据”

## 路线 B 阶段规划

### 阶段 1：数据提取稳定性提升
**目标**：在当前启发式方案基础上，提升数据提取稳定性

#### 饼图
- [ ] 图例 marker 与文本配对
  - 基于颜色/位置/几何关系匹配 marker 与文本
  - 支持常见图例布局（右/下/内嵌）
- [ ] 数值与标签对齐
  - 基于位置关系对齐数值与标签
  - 支持百分比/绝对值混合场景

#### 柱状图
- [ ] 类目与数值分离
  - 基于坐标轴位置分离类目与数值
  - 支持类目在 X 轴、数值在柱体上的场景
- [ ] 坐标轴检测
  - 检测 X/Y 轴及刻度
  - 基于坐标轴映射数值

#### 折线图
- [ ] 坐标轴检测与标尺映射
  - 检测 X/Y 轴及刻度
  - 基于坐标轴映射点位
- [ ] 线条几何解析
  - 基于线条像素点估计数据点
  - 支持多条线场景

#### 表格
- [ ] 单元格分割
  - 基于线条/空白分割单元格
  - 支持合并单元格场景
- [ ] 表头与数据行分离
  - 基于位置/格式分离表头与数据行

#### 评估指标
- 饼图：图例配对准确率 > 80%
- 柱状图：类目-数值配对准确率 > 80%
- 折线图：数据点数量 > 5 个/图
- 表格：单元格识别准确率 > 80%

### 阶段 2：几何分析增强
**目标**：引入更重的几何分析，提升数据提取精度

#### 图像处理
- [ ] 图表主体检测
  - 基于边缘/颜色/纹理检测图表主体
  - 支持多图表场景
- [ ] 透视矫正
  - 基于角点检测进行透视矫正
  - 支持倾斜/变形图表

#### 几何分析
- [ ] 线条检测
  - 基于 Hough 变换检测线条
  - 支持曲线/折线/柱体
- [ ] 形状检测
  - 基于轮廓检测形状
  - 支持圆形/矩形/多边形

#### 数据映射
- [ ] 坐标轴映射
  - 基于坐标轴刻度映射数值
  - 支持线性/对数坐标轴
- [ ] 颜色映射
  - 基于颜色映射数据
  - 支持图例/热力图

#### 评估指标
- 图表主体检测准确率 > 90%
- 坐标轴映射误差 < 5%
- 颜色映射准确率 > 80%

### 阶段 3：专用模型引入
**目标**：引入专用图表解析模型，实现高精度数据提取

#### 模型选择
- [ ] 图表解析模型
  - 基于深度学习检测图表元素
  - 支持饼图/柱状图/折线图/表格
- [ ] 文字识别模型
  - 基于专用 OCR 模型识别文字
  - 支持中文/英文/数字

#### 模型部署
- [ ] 模型轻量化
  - 模型量化/剪枝
  - 支持本地部署
- [ ] 模型集成
  - 与现有 OCR 流程集成
  - 支持 fallback 机制

#### 评估指标
- 图表元素检测准确率 > 95%
- 文字识别准确率 > 90%
- 模型推理时间 < 5s/图

## 路线 B 实施建议

### 优先级
1. **阶段 1**：数据提取稳定性提升
   - 风险低，收益高
   - 可在当前启发式方案基础上实现

2. **阶段 2**：几何分析增强
   - 风险中等，收益中等
   - 需要引入更重的图像处理

3. **阶段 3**：专用模型引入
   - 风险高，收益高
   - 需要引入深度学习模型

### 资源需求
- **阶段 1**：1-2 周
  - 主要工作：图例配对、类目分离、坐标轴检测
  - 依赖：现有 OCR 流程

- **阶段 2**：2-3 周
  - 主要工作：几何分析、透视矫正
  - 依赖：OpenCV、图像处理

- **阶段 3**：4-6 周
  - 主要工作：模型选择、训练、部署
  - 依赖：深度学习框架、训练数据

### 风险控制
- **阶段 1**：保持向后兼容
  - 不破坏现有 API
  - 支持 fallback 机制

- **阶段 2**：模块化设计
  - 几何分析模块独立
  - 支持插件式扩展

- **阶段 3**：渐进式引入
  - 先支持部分图表类型
  - 逐步扩展到所有图表类型

## 路线 B 预期成果

### 短期（阶段 1）
- 图表数据提取稳定性提升
- 支持更多图表类型
- 精度提升到可用水平

### 中期（阶段 2）
- 几何分析能力增强
- 支持复杂图表场景
- 精度提升到专业水平

### 长期（阶段 3）
- 专用模型支持
- 高精度数据提取
- 支持所有图表类型

## 路线 B 后续步骤

### 下一步
1. **确定阶段 1 优先级**
   - 选择最急需提升的图表类型
   - 制定详细实施计划

2. **准备测试数据**
   - 收集更多图表样本
   - 标注数据用于评估

3. **实施阶段 1**
   - 按优先级逐步实现
   - 每步都进行回归测试

### 后续
- 根据阶段 1 效果，决定是否进入阶段 2
- 根据阶段 2 效果，决定是否进入阶段 3

## 路线 B 总结

路线 B 是一个渐进式提升过程，从当前“类型识别+初步提取”逐步升级到“高精度数据还原”。

- **阶段 1**：快速见效，风险低
- **阶段 2**：中等风险，中等收益
- **阶段 3**：高风险，高收益

建议按阶段逐步推进，每阶段都进行充分评估和回归测试。
