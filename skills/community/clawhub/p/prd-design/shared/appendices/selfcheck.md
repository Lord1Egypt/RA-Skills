## 六、原型交互自检

> 逐页逐元素执行，禁止带已知缺陷交付。

### 6.1 建立可点击元素清单

扫描原型中**所有**以下类型的元素，逐一列出并标注 ✅/❌：
- 所有 `<button>` 元素
- 所有 `<a>` 链接元素
- 所有带 `onclick` 属性的元素
- 所有 Tab 切换控件（含导航 Tab、页内 Tab、角色切换 Tab）
- 所有下拉选择器（`<select>`）中有联动逻辑的
- 所有表单提交路径

输出格式：
```
页面/模块：[模块名]
  元素1：[元素描述] → 绑定函数：[函数名或inline代码] → ✅/❌
  元素2：[元素描述] → 绑定函数：[函数名或inline代码] → ✅/❌
```

### 6.2 逐项验证交互正确性

对清单中每个元素，验证以下 4 点：

| 验证点 | 标准 |
|--------|------|
| **有绑定** | onclick/事件监听器存在，不为空函数 `function(){}` |
| **函数存在** | 绑定的函数在 JS 中有完整实现，不是 `undefined` |
| **有可见反馈** | 点击后有明确的界面变化（弹窗/抽屉/toast/内容切换/状态变化），不是静默执行 |
| **反馈正确** | 反馈内容与该元素的语义一致（「新建」→弹出新建表单，「删除」→二次确认弹窗+行消失） |

额外检查：
- [ ] 操作类图标（编辑/删除/查看）全部绑定交互，不允许装饰性摆放
- [ ] 无 `href="#"` 死链

### 6.3 Tab 专项检查（历史高发问题）

对每一组 Tab 控件：
- [ ] 所有 Tab 按钮均有 onclick 绑定
- [ ] 每个 Tab 对应的内容区域有唯一 ID
- [ ] 切换函数能正确找到目标内容区域（getElementById/querySelector 不返回 null）
- [ ] 切换后 active 状态正确转移（旧 Tab 取消 active，新 Tab 获得 active）
- [ ] 默认激活的 Tab 对应内容默认可见，其余内容默认隐藏
- [ ] 切换后内容区域实际渲染了对应内容，不是空白

### 6.4 表单交互专项检查

对每个表单/抽屉/弹窗：
- [ ] 「提交」按钮有完整的校验逻辑（必填项为空时有提示，不是直接提交）
- [ ] 「取消」/「×」按钮能关闭弹窗/抽屉
- [ ] 有联动字段的（如选A显示X，选B显示Y）联动逻辑正确实现
- [ ] 提交成功后有 toast 提示，内容与操作一致
- [ ] 提交成功后弹窗/抽屉关闭，列表有刷新或新增记录的视觉反馈

### 6.5 原型与 PRD 一致性校验

- [ ] Layer 3 的 ASCII 草图与原型实际界面一致（字段位置、控件类型、按钮布局）
- [ ] PRD 描述的按钮文案 = 原型按钮文案（一字不差）
- [ ] PRD 描述的必填字段 = 原型必填标记（* 标注）
- [ ] PRD 描述的二次确认行为 = 原型实际弹窗内容
- [ ] PRD 描述的 loading/成功/失败反馈 = 原型实际行为
- [ ] PRD 描述的默认值 = 原型打开时的初始状态
- [ ] 任一不一致 → 列出具体位置 → 由用户裁决以谁为准 → 同步修改

### 6.6 原型数据驱动校验

- [ ] 统计数字、列表行数、下拉选项全部通过 JS 变量或 mock 数组驱动，不可硬编码在 HTML 中
- [ ] 原型中 mock 数据对象贯穿全流程：步骤 A 提交写入的字段，步骤 B 页面从同一对象读取
- [ ] 依赖前置条件才显示的区域已体现空态（前置未满足）和有数据态（前置满足后）两种状态

### 6.7 修复与重检规则

**问题修复对照表**：

| 问题类型 | 修复要求 |
|---------|---------|
| Tab 点击无响应 | 补全 onclick 绑定和切换函数，确保内容区域正确渲染 |
| 按钮点击无反馈 | 补全对应函数实现，至少有 toast 反馈 |
| 函数未定义 | 在 `<script>` 中补全函数体，不允许空函数 |
| 表单无校验 | 补全必填项校验逻辑 |
| 联动逻辑缺失 | 补全 onchange/oninput 事件处理 |
| 角色切换后页面无变化 | 确保角色切换后重新渲染当前页面 |
| 原型与PRD描述不一致 | 以用户裁决结果为准，同步修改原型和PRD |

发现任何缺陷修复后，**必须重新执行 6.1-6.6 的完整自检**，全部通过后才允许交付。
