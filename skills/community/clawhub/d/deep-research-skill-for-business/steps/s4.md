# S4: 深度抓取

## 目标

对 s3 中筛选出的高价值 URL（P0-P2）执行全文抓取，获取完整内容。不止于搜索摘要。

## ⛔ 红线

- 必须抓取至少 3 个 P0-P2 来源的全文
- 抓取失败不可跳过——记录失败原因，找替代来源
- 知乎等反爬平台无法抓取的，不推送，优先找替代来源
- 抓取内容必须足够支撑后续论证

## 操作

### 1. 筛选待抓取 URL

从 s3 结果中选出：
- 所有 P0 来源（全部抓取）
- P1 来源（至少抓取 3 个）
- P2 来源（至少抓取 2 个）

### 2. 执行全文抓取

对每个选中的 URL，使用当前环境的网页抓取工具获取全文内容（maxChars=50000）。

### 3. 记录抓取结果

```markdown
## S4: 深度抓取结果

| # | 标题 | URL | 抓取状态 | 关键发现 |
|---|------|-----|----------|----------|
| 1 | ... | ... | ✅ 成功 / ❌ 失败(原因) | ... |
| ... | ... | ... | ... | ... |

### 失败 URL 及替代方案
- URL1: 失败原因 → 替代来源：...
```

### 4. 标记完成

完成后执行：
```bash
bash orchestrator.sh --complete s4 --next s5
```

## 校验标准

- [ ] 至少 3 个 P0-P2 来源被抓取全文
- [ ] 抓取内容中有具体数据/案例/文件编号
- [ ] 失败 URL 记录了原因和替代方案
- [ ] 抓取内容足够支撑后续论证
