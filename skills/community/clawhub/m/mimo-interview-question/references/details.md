# 面试题生成器 - 详细参考

## 一、各岗位题库模板

### 1.1 后端工程师面试题（中级）

**技术题示例：**
```
Q1: 请解释MySQL索引的B+树结构，为什么不用B树或红黑树？[难度: ⭐⭐⭐]
参考答案：
- B+树叶子节点形成有序链表，适合范围查询；B树非叶子也存数据，扇出更大但范围查询慢
- 红黑树不适合磁盘IO（树太高，每次查询多次磁盘随机IO）
- B+树矮胖，3-4层可存储千万级数据，减少磁盘IO次数
得分点：1.B+树vs B树区别 2.磁盘IO优化 3.范围查询优势
加分项：提及聚簇索引vs非聚簇索引、页分裂/合并
常见错误：混淆B树和B+树、不知道页的概念
追问方向：什么情况下索引会失效？如何排查慢查询？
```

```
Q2: 设计一个支持百万并发的消息推送系统 [难度: ⭐⭐⭐⭐]
参考答案：
- 接入层：WebSocket长连接 + 负载均衡（Nginx/HAProxy）
- 消息层：Kafka/RabbitMQ消息队列解耦
- 存储层：Redis缓存在线状态 + MySQL持久化
- 推送策略：在线实时推/离线存储+上线拉取
得分点：1.长连接管理 2.消息队列选型 3.在线/离线双通道
加分项：连接数水平拆分方案、消息ACK和重试机制
常见错误：只考虑在线不考虑离线、没有考虑消息丢失
追问方向：如何处理消息顺序性？如何保证消息不丢失？
```

### 1.2 前端工程师面试题（高级）

```
Q1: React/Vue的虚拟DOM Diff算法有什么差异？如何优化渲染性能？[⭐⭐⭐⭐]
参考答案：
- React: Fiber架构，时间切片，可中断渲染；diff基于key和类型
- Vue: 依赖收集+精确更新，编译时优化（静态提升/补丁标记）
- 优化：React.memo/useMemo/useCallback, Vue的computed/v-once
得分点：1.Diff算法原理 2.框架差异 3.优化手段
加分项：React Concurrent Mode、Vue 3.0编译器优化
```

### 1.3 产品经理面试题

```
Q1: 如何从0到1设计一个新功能？以"短视频评论功能"为例 [⭐⭐⭐]
参考答案：
- 需求分析：用户为什么要评论？（表达观点/互动/社交）
- 竞品分析：抖音/快手/B站评论功能差异
- MVP定义：发表评论+列表展示+点赞
- 数据指标：评论率/回复率/评论留存
- 迭代方向：评论区推荐/置顶/筛选
得分点：1.需求分析框架 2.竞品对比深度 3.MVP思维
加分项：提及评论治理（敏感词/反垃圾）
```

### 1.4 数据分析师面试题

```
Q1: 某电商App日活下降5%，如何排查原因？[⭐⭐⭐]
参考答案：
- 数据验证：确认数据准确性，排除技术问题
- 拆解维度：新用户vs老用户/平台(iOS/Android)/地区/渠道
- 时间对比：环比/同比/节假日因素
- 归因分析：产品变更/竞品活动/外部环境/季节性
- 输出建议：定位根因后给出改善方案
得分点：1.分析框架完整 2.维度拆解能力 3.数据敏感度
```

## 二、行为面试STAR题库

### 2.1 通用行为题（按能力维度）

**团队协作类：**
- 描述一次你与团队成员产生分歧的经历，如何解决的？
- 讲一个你推动跨部门合作的案例
- 你如何处理团队中的"搭便车"现象？

**问题解决类：**
- 描述一次你在紧急情况下解决问题的经历
- 讲一个你通过分析数据做出决策的案例
- 你如何在信息不完整的情况下做决策？

**学习能力类：**
- 最近一年你学到的最有价值的新技能是什么？
- 描述你快速掌握一个陌生领域知识的方法
- 你如何保持技术敏感度？

**领导力类（高级岗位）：**
- 讲一个你带领团队度过困难的案例
- 你如何培养团队成员？
- 描述一次你做出不受欢迎但最终正确的决策

### 2.2 STAR评分标准细化

| 维度 | 1分（差） | 3分（合格） | 5分（优秀） |
|------|----------|------------|------------|
| Situation | 描述模糊 | 背景清晰 | 背景+数据支撑 |
| Task | 职责不清 | 明确自己的角色 | 角色+目标量化 |
| Action | 只说"我们" | 清晰描述个人行动 | 行动+决策逻辑 |
| Result | 无结果 | 有定性结果 | 量化结果+复盘 |

### 3.2 系统设计评估框架
```
评估步骤（30-45分钟）：
1. 需求澄清（5min）：功能需求+非功能需求（QPS/延迟/可用性）
2. 容量估算（5min）：DAU/存储/带宽预估
3. 高层设计（10min）：核心组件+数据流
4. 详细设计（10min）：数据模型/API/核心算法
5. 扩展讨论（5min）：扩展性/监控/故障处理
6. 总结（5min）：tradeoff分析/未来优化方向
```

## 五、面试官追问策略

### 5.1 深挖追问（发现真实水平）
- "能具体说一下你是怎么实现的吗？"
- "如果数据量增大100倍，方案还能work吗？"
- "你提到用了XX技术，为什么选它而不是YY？"
- "如果这个功能出问题了，你会怎么排查？"

### 5.2 压力追问（高级别候选人）
- "如果时间只有一半，你会砍掉哪些功能？为什么？"
- "这个方案有什么明显缺陷？"
- "CTO质疑你的方案，你怎么说服他？"

### 5.3 识别虚假回答的信号
- 全程说"我们"不说"我"→ 可能没深度参与
- 回答过于完美/教科书式 → 可能是背题
- 无法回答追问细节 → 可能只了解表面
- 结果全部量化且完美 → 可能是编造
