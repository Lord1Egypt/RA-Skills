# 上报参数说明

## 一、通用上报方法（按端区分）

各端均提供通用上报方法，结构一致但语法不同：

| 端 | 方法 | 示例 |
|----|------|------|
| Android | `GDTAction.logAction(String actionType, JSONObject actionParam)` | `GDTAction.logAction("PURCHASE", json)` |
| iOS | `[GDTAction logAction:(NSString *)actionName actionParam:(NSDictionary *)actionParam]` | `[GDTAction logAction:@"PURCHASE" actionParam:@{@"value":@600}]` |
| 鸿蒙 | `logAction(actionType: string, actionParam?: Object)` | `logAction("PURCHASE", {"value": 600})` |
| 小游戏 | `sdk.track(action_type: string, action_param?: object)` | `sdk.track("PURCHASE", { value: 600 })` |
| 小程序 | `sdk.track(action_type: string, action_param?: object)` | `sdk.track("PURCHASE", { value: 600 })` |

## 二、actionParam 键值约束

| 约束项 | Android / iOS | 鸿蒙 | 小程序 / 小游戏 |
|--------|:------------:|:----:|:--------------:|
| Key 类型 | 仅 String | 仅 String | 仅 String |
| Key 命名规则 | 字母、数字、下划线；字母开头；最大 255 字符 | 字母、数字、下划线；字母开头；最大 64 字符 | 字母、数字、下划线；字母开头；最大 255 字符 |
| Key 禁止前缀 | `ams_reserved_` | `ams_reserved_` | — |
| Value 类型 | String / Number / Boolean / JSONArray（元素类型需统一） | String / Number / Boolean / JSONArray（元素类型需统一） | String / Number / Boolean / Object |

## 三、常用 actionParam 字段

| 字段 | 类型 | 适用行为 | 说明 |
|------|------|---------|------|
| `value` | number | PURCHASE / COMPLETE_ORDER / CREDIT / AD_PURCHASE | **金额，单位：分**。付费类行为必传 |
| `length_of_stay` | number | START_APP | 留存天数。7日留存传 `7`，1日留存传 `1`（iOS 端支持） |
| `audience_type` | number | 任意事件 | 标记为回流用户激活时传 `1`（iOS 端） |
| `outer_action_id` | string | 任意事件 | 去重 ID。若上报可能重复请填写，系统按此 ID 去重 |
| `open_url` | string | START_APP | iOS 端通过 URL Scheme / Universal Links 唤起时传入（表示外部唤起来源） |
| `custom_action` | string | CUSTOM | 自定义行为子类型，如 `"UV_CORE_ACTION"` |
| `method` | string | REGISTER / LOGIN | 注册/登录方式，如 `"Mobile"`、`"QQ"` |
| `success` | boolean | REGISTER / LOGIN / BINDACCOUNT / ADD_TO_CART / CHECKOUT / PURCHASE / ADD_PAYMENT_CHANNEL | 是否成功 |
| `roleName` | string | CREATE_ROLE | 角色名称 |
| `level` | number | UPDATE_LEVEL | 升级后等级 |
| `contentType` | string | VIEW_CONTENT / ADD_TO_CART / CHECKOUT / PURCHASE | 内容/商品类型 |
| `contentName` | string | VIEW_CONTENT / ADD_TO_CART / CHECKOUT / PURCHASE | 内容/商品名称 |
| `contentID` | string | VIEW_CONTENT / ADD_TO_CART / CHECKOUT / PURCHASE | 内容/商品 ID |
| `contentNumber` | number | ADD_TO_CART / CHECKOUT / PURCHASE | 商品数量 |
| `channel` | string | SHARE / PURCHASE / ADD_PAYMENT_CHANNEL | 分享渠道 / 支付渠道 |
| `rate` | number | RATE | 评分值 |
| `questID` | string | QUEST_FINISH | 任务 ID |
| `questType` | string | QUEST_FINISH | 任务类型 |
| `questName` | string | QUEST_FINISH | 任务名称 |

> ⚠️ 使用**专用上报方法**时，上述参数已内置在方法签名中，无需手动构造 actionParam。

## 四、用户标识设置（按端区分）

### Android 端

| 方法 | 说明 |
|------|------|
| `GDTAction.setUserUniqueId(String)` | 设置业务自定义用户 ID |
| `GDTAction.getClickID(Context)` | 获取点击 ID（仅游戏场景） |
| `GDTAction.getChannelID(Context)` | 获取渠道包标识 |
| `GDTAction.setAnidEnable(boolean)` | 开关 ANID 采集（默认开启） |

### iOS 端

| 方法 | 说明 |
|------|------|
| `[GDTAction getCaid]` | 获取设备 CAID 信息（v≥2.0.11） |

### 鸿蒙端

| 方法 | 说明 |
|------|------|
| `setUserUniqueId(String)` | 设置业务自定义用户 ID |

### 小游戏 / 小程序端

| 方法 | 说明 | 优先级 |
|------|------|--------|
| `sdk.setOpenId(openid)` | 设置微信 OpenId | **优先使用**（建议在上报注册事件前设置） |
| `sdk.setUnionId(unionid)` | 设置微信 UnionId | 仅在无法获取 OpenId 时使用 |

也可在初始化时直接传入：

```javascript
const sdk = new SDK({
  user_action_set_id: '数据源ID',
  secret_key: '密钥',
  appid: 'wx开头的AppID',
  openid: '用户openid',       // 可选，也可后续通过 setOpenId 设置
  unionid: '用户unionid',     // 可选
  user_unique_id: '业务用户ID', // 可选
});
```

## 五、初始化方法对比

| 端 | 初始化方法 | 启动方法 | 说明 |
|----|----------|---------|------|
| Android | `GDTAction.init(context, userActionSetId, appSecretKey)` 或带 `ChannelType, channelId` 的重载 | `GDTAction.start()` | init 后必须调用 start 才开始采集 |
| iOS | `[GDTAction init:actionSetId secretKey:secretKey]` | `[GDTAction start]`（v≥2.1.0 必调） | init 后调用 start |
| 鸿蒙 | `init(context, user_action_set_id, secret_key)` 可选 `show_log`, `user_unique_id` | `start()` | init 后调用 start |
| 小游戏 | `new SDK({ user_action_set_id, secret_key, appid })` | — | 构造即初始化，尽早调用 |
| 小程序 | `new SDK({ user_action_set_id, secret_key, appid })` | — | 构造即初始化，尽早调用 |

> ⚠️ Android/iOS/鸿蒙端的 `init()` 和 `start()` 是**两步操作**，缺少 `start()` 调用将无法正常采集数据。

## 七、常见参数错误

| 错误现象 | 原因 | 修正方式 |
|---------|------|---------|
| 金额数据为 0 | `value` 字段未传或传了字符串 | 传入 number 类型的金额值（单位：分） |
| 去重导致数据丢失 | `outer_action_id` 重复 | 确保每条行为的去重 ID 唯一 |
| 自定义行为未识别 | `action_type` 传了 `CUSTOM` 但未传 `custom_action` | 两个字段必须同时传入 |
| Key 命名报错 | Key 以数字开头或包含特殊字符 | Key 必须以字母开头，仅包含字母/数字/下划线 |
| Key 使用了保留前缀 | Key 以 `ams_reserved_` 开头 | 避免使用此前缀 |
| init 后数据未上报 | 未调用 `start()` | Android/iOS/鸿蒙端 init 后必须调用 start |
| JSONArray 报错 | JSONArray 中元素类型不统一 | 数组元素须全部为 String、全部为 Number 或全部为 Boolean |
