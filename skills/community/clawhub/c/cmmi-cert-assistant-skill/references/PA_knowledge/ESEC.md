# ESEC — 网络安全工程（CMMI V3.0 Domain-specific PA）

## PA 基本信息
- **缩写**：ESEC
- **英文**：Cybersecurity Engineering
- **中文**：网络安全工程
- **所属**：Domain-specific PA（领域特定实践域）
- **适用领域**：网络安全、信息安全、软件工程
- **新增说明**：V3.0新增实践域，关注网络安全工程实践

---

## CMMI V3.0 要求要点

### 核心目的
确保产品、服务、系统在面临网络威胁时具有足够的韧性。

### 关键实践（Specific Practices）
| 实践编号 | 实践内容 | 说明 |
|---------|---------|------|
| ESEC 1.1 | 识别网络安全需求 | 识别保密性、完整性、可用性需求 |
| ESEC 1.2 | 分析网络威胁 | 威胁建模、攻击面分析 |
| ESEC 1.3 | 设计网络安全机制 | 安全架构、防御深度策略 |
| ESEC 1.4 | 实施网络安全控制 | 编码安全、安全测试 |
| ESEC 2.1 | 验证网络安全机制 | 渗透测试、漏洞扫描 |
| ESEC 2.2 | 管理网络安全风险 | 风险处置、持续监控 |
| ESEC 3.1 | 响应网络安全事件 | 事件响应、恢复计划 |

### 通用实践（Generic Practices）
- GP 1.1 执行特定实践
- GP 2.1 建立组织政策
- GP 2.2 策划过程
- GP 2.3 提供资源
- GP 2.4 分配责任
- GP 2.5 培训人员
- GP 2.6 管理配置
- GP 2.7 识别并记录干系人需求和接口
- GP 2.8 监督和控制过程
- GP 2.9 客观评估符合性
- GP 2.10 高层审查

## 用户文档 → PA 匹配规则

### 高概率匹配（主要PA）
- 文档名含"网络安全"、"Cybersecurity"、"渗透测试" → **ESEC 主要匹配**
- 文档含威胁模型、安全架构设计 → **ESEC 主要匹配**

### 中概率匹配（次要PA）
- 文档含安全需求 → 同时匹配 **ESAF**（确保安全性）
- 文档含安全测试记录 → 同时匹配 **VV**（验证与确认）

## 差距检查清单（用于差距分析）

### ESEC 1.x — 网络安全策划
- [ ] 是否识别了网络安全需求（CIA三性）？
- [ ] 是否进行了威胁建模？
- [ ] 是否有安全架构设计？
- [ ] 是否有网络安全控制计划？

### ESEC 2.x — 网络安全实施
- [ ] 是否实施了安全编码实践？
- [ ] 是否有安全测试（渗透测试、漏洞扫描）？
- [ ] 是否有网络安全风险管理的记录？

### ESEC 3.x — 网络安全事件响应
- [ ] 是否有网络安全事件响应流程？
- [ ] 是否有安全事件记录？
- [ ] 是否有恢复计划和演练记录？

## 评估访谈常见问题（模式D）

### 针对安全工程师的问题
1. "你们的网络安全需求是什么？如何管理的？"
2. "威胁建模是如何进行的？"
3. "网络安全控制是如何设计和实施的？"
4. "网络安全测试是如何进行的？有测试记录吗？"
5. "发生网络安全事件后如何处理？"

### 评分要点
- **ESEC 1.1-1.4**：是否系统识别网络安全需求并设计安全机制
- **ESEC 2.1-2.2**：网络安全控制是否实施，是否有验证记录
- **ESEC 3.1**：是否有网络安全事件响应流程
