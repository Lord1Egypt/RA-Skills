# M2 处理策略选择域范本

## M2-01 简单任务策略

### 输入示例
任务类型：简单任务（概念密度低，结构清晰）

### 处理过程
1. 简单任务特征确认：概念数量≤3，层级关系≤2，无复杂依赖
2. 执行策略设计：直接执行，无中间过程展示
3. 响应时间目标：≤10秒
4. 输出完整性要求：完整结果

### 输出示例
```markdown
## 简单任务策略文档

### 任务特征
- **概念数量**：2个
- **层级关系**：1层
- **依赖关系**：无复杂依赖
- **复杂度等级**：简单

### 执行策略
- **策略名称**：直接执行策略
- **执行方式**：一步到位，无中间过程
- **响应时间**：≤10秒
- **输出完整性**：完整结果

### 执行步骤
1. 接收任务输入
2. 直接执行处理
3. 输出完整结果
```

## M2-02 中等任务策略

### 输入示例
任务类型：中等任务（概念密度中等，结构复杂度中等）

### 处理过程
1. 中等任务特征确认：概念数量4-6，层级关系3-4，有中等依赖
2. 关键步骤识别：概念分析、结构分析、上下文分析、综合判定
3. 展示策略设计：关键步骤展示，中间过程可选展示
4. 响应时间目标：30-60秒

### 输出示例
```markdown
## 中等任务策略文档

### 任务特征
- **概念数量**：5个
- **层级关系**：3层
- **依赖关系**：中等依赖
- **复杂度等级**：中等

### 关键步骤
| 步骤 | 内容 | 输入 | 输出 |
|------|------|------|------|
| 1 | 概念分析 | 任务描述 | 概念密度报告 |
| 2 | 结构分析 | 概念关系 | 结构复杂度报告 |
| 3 | 上下文分析 | 外部信息 | 上下文依赖报告 |
| 4 | 综合判定 | 各项指标 | 密度等级判定 |

### 展示策略
- **展示内容**：关键步骤和决策点
- **展示级别**：中等细节
- **展示格式**：结构化文档
- **展示控制**：用户可切换详细程度
```

## M2-03 复杂任务策略

### 输入示例
任务类型：复杂任务（概念密度高，结构复杂度高）

### 处理过程
1. 复杂任务特征确认：概念数量≥7，层级关系≥5，有复杂依赖

3. 透明度控制策略设计：默认隐藏，用户可要求追溯
4. 响应时间目标：60-120秒

### 输出示例
```markdown
## 复杂任务策略文档

### 任务特征
- **概念数量**：8个
- **层级关系**：5层
- **依赖关系**：复杂依赖
- **复杂度等级**：复杂

### 状态演化追踪策略
- **追踪内容**：关键状态快照、演变路径、决策点
- **追踪频率**：每个关键步骤
- **追踪格式**：时间序列数据
- **追踪存储**：内存缓存

### 透明度控制策略
- **默认隐藏**：中间过程默认隐藏
- **用户追溯**：支持用户要求展示
- **展示级别**：可切换详细程度
- **展示格式**：结构化文档

### 执行流程
1. 接收任务输入
2. 执行状态演化追踪
3. 控制透明度
4. 交付最终结果
```

## M2-04 策略切换条件

### 输入示例
任务复杂度变化：简单任务 → 中等任务

### 处理过程
1. 切换条件识别：概念数量从3增加到5
2. 切换阈值设定：概念数量>3时切换到中等任务策略
3. 切换流程设计：自动切换，无需用户干预
4. 切换验证：验证切换后策略有效性

### 输出示例
```markdown
## 策略切换条件文档

### 切换条件
| 条件类型 | 当前值 | 阈值 | 切换方向 |
|----------|--------|------|----------|
| 概念数量 | 5 | >3 | 简单→中等 |
| 层级关系 | 3 | >2 | 简单→中等 |
| 依赖关系 | 中等 | 复杂 | 中等→复杂 |

### 切换阈值
- **简单→中等**：概念数量>3 或 层级关系>2
- **中等→复杂**：概念数量>6 或 层级关系>4 或 依赖关系复杂

### 切换流程
1. 监测任务特征
2. 与阈值比较
3. 自动切换策略
4. 验证切换有效性
```

## M2-05 策略优化

### 输入示例
策略执行反馈：响应时间过长

### 处理过程
1. 策略执行反馈收集：响应时间45秒，超过目标30秒
2. 优化点识别：数据处理和分析步骤耗时过长
3. 优化方案设计：并行处理数据，优化分析算法
4. 优化效果评估：响应时间从45秒降至25秒

### 输出示例
```markdown
## 策略优化报告

### 策略执行反馈
- **响应时间**：45秒（目标30秒）
- **问题**：数据处理和分析步骤耗时过长

### 优化点识别
| 优化点 | 当前耗时 | 优化目标 | 优化方法 |
|--------|----------|----------|----------|
| 数据处理 | 20秒 | 10秒 | 并行处理 |
| 分析步骤 | 25秒 | 15秒 | 优化算法 |

### 优化方案
- **数据处理**：采用并行处理，将数据分片处理
- **分析步骤**：优化分析算法，减少计算复杂度

### 优化效果
- **响应时间**：从45秒降至25秒
- **优化幅度**：44%
```

## M2-06 策略验证

### 输入示例
策略：中等任务策略

### 处理过程
1. 验证标准制定：策略是否适合任务复杂度
2. 验证方法选择：对比分析法
3. 验证执行：对比策略效果与预期目标
4. 验证结论：策略选择合理，执行有效

### 输出示例
```markdown
## 策略验证报告

### 验证标准
| 标准 | 描述 | 权重 |
|------|------|------|
| 适配性 | 策略是否适合任务复杂度 | 40% |
| 效率 | 响应时间是否满足目标 | 30% |
| 完整性 | 输出是否完整 | 20% |
| 可用性 | 用户体验是否良好 | 10% |

### 验证方法
- **方法**：对比分析法
- **对比对象**：策略效果 vs 预期目标
- **对比维度**：适配性、效率、完整性、可用性

### 验证结果
| 维度 | 预期 | 实际 | 达标 |
|------|------|------|------|
| 适配性 | 中等任务 | 中等任务 | ✓ |
| 效率 | ≤30秒 | 25秒 | ✓ |
| 完整性 | 完整 | 完整 | ✓ |
| 可用性 | 良好 | 良好 | ✓ |

### 验证结论
**结论**：策略选择合理，执行有效
```
