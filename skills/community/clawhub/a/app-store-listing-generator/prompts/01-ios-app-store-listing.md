# Prompt 1: iOS App Store Listing

Copy and paste this prompt into Claude. Replace all [BRACKETED] fields with your app's details.

---

```
You are an App Store Optimization (ASO) expert. Generate a complete iOS App Store listing for the following app. Strictly enforce all character limits — show the character count after every field.

App Details:
- Name: [APP_NAME]
- Category: [CATEGORY — e.g., Productivity, Health & Fitness, Finance, Games, Social]
- What it does: [ONE_SENTENCE — what the app does, who it's for, main outcome]
- Core features: [FEATURE_1], [FEATURE_2], [FEATURE_3]
- Target user: [WHO — e.g., "remote workers who struggle with focus", "parents tracking kids' screen time"]
- Problem solved: [SPECIFIC PROBLEM — be concrete]
- Competitors (optional): [COMPETITOR_1], [COMPETITOR_2], [COMPETITOR_3]
- Tone: [TONE — professional / friendly / bold / minimal / playful]

---

Generate the complete iOS App Store listing:

**TITLE** (max 30 characters — include primary keyword naturally, not forced)
[title]
Character count: X/30

**SUBTITLE** (max 30 characters — secondary benefit, keyword, or differentiator)
[subtitle]
Character count: X/30

**KEYWORD FIELD** (max 100 characters — comma-separated, NO spaces after commas, NO words already in title/subtitle, NO competitor names)
[keyword1,keyword2,keyword3,keyword4,keyword5...]
Character count: X/100

**DESCRIPTION** (max 4,000 characters)

Write the description in this structure:
- Lines 1-3 (above the fold): Hook, primary benefit, proof/credibility. These 3 lines are all most users read before deciding. Make them count.
- Body: 3 sections, one per core feature. Each section: feature name as natural header, 2-3 sentences of benefit-focused copy. No bullet lists — flowing paragraphs read better.
- Closing: Social proof line + clear CTA + download prompt.

Use line breaks between sections. Write for humans first, keywords second.

[description — 4,000 chars max]
Character count: X/4,000

**A/B TITLE VARIANTS** (3 alternatives to split test)
1. [title variant A] (X/30) — angle: [benefit-led / keyword-led / curiosity]
2. [title variant B] (X/30) — angle: [benefit-led / keyword-led / curiosity]
3. [title variant C] (X/30) — angle: [benefit-led / keyword-led / curiosity]

**ABOVE-THE-FOLD PREVIEW**
Copy-paste the first 3 lines of the description here:
[lines 1-3]
(This is what users see before tapping "more" — verify it stands alone as a compelling pitch)

**LOCALIZATION NOTES**
- If targeting Spanish-speaking US market: [keyword translation suggestions for keyword field]
- If targeting UK App Store: [spelling/phrasing adjustments]
- High-priority alternative markets for this category: [list 2-3 countries to consider localizing for]
```

---

## Tips for Best Results

- **Primary keyword in title** is the #1 ASO ranking factor on iOS. Don't sacrifice it for branding.
- **Keyword field:** Never repeat words already in your title or subtitle — Apple's algorithm already indexes those. Every character in the keyword field is wasted if it duplicates metadata.
- **Subtitle** shows in search results below the title. Treat it like a second title, not a tagline.
- **Above-the-fold test:** Read just lines 1-3 of the description with fresh eyes. Would you download? If not, rewrite.
- **A/B testing:** Upload Variant A to App Store Connect, run for 2 weeks, switch to Variant B if conversion drops.
