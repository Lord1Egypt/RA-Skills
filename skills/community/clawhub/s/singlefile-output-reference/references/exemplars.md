# 优秀范本库

## 更多范本说明

### 范本扩展指南

以上提供了两个完整的、可直接运行的范本（俄罗斯方块和计算器）。其余范本标记为`[待补充]`。

**填充新范本时需遵循以下规范**：

1. **单文件原则**：所有HTML/CSS/JS内联在一个`.html`中；所有Python逻辑在一个`.py`中；所有Shell逻辑在一个`.sh`中；所有SQL逻辑在一个`.sql`中
2. **零外部依赖**：
   - HTML：仅用CDN白名单资源（cdnjs.cloudflare.com / cdn.jsdelivr.net / unpkg.com）
   - Python：仅用标准库
   - Shell：仅用coreutils及常见系统工具
   - SQL：标准SQL优先，方言差异注释标注
3. **可直接运行**：
   - HTML：双击即可在浏览器中正常工作
   - Python：`python file.py` 即可运行（含 `--help`）
   - Shell：`chmod +x file.sh && ./file.sh` 即可运行（含 `--help`）
   - SQL：`mysql/psql/sqlite3 < file.sql` 即可执行
4. **代码质量**：
   - 变量命名清晰
   - 关键逻辑有注释
   - 错误处理完善
   - 边界情况考虑
5. **文件命名**：`exemplars/<kebab-case-name>.html` / `.py` / `.sh` / `.sql`

### 范本作为UTOS样本法的使用方式

当用户要求产出某种单文件应用时：

1. UTOS从exemplars中找到最接近的范本
2. 分析范本的代码结构（作为"样本特征摘要"）
3. 按用户需求修改/替换内容部分
4. 保持结构框架不变，仅替换差异化内容
5. 输出新的完整可运行单文件

### 范本优先级

当多个范本可用于同一类产出时，选择优先级：

1. **同类同域**：S2游戏优先用S2范本
2. **结构相似**：卡片式UI优先用Portfolio范本
3. **交互模式匹配**：拖拽交互优先用拖拽排序题范本
4. **降级使用**：无精确匹配时，取最接近的范本并标注差异

```
决策树: 可视化数据→图表展示→S4-01仪表盘 或 S1-10报告可视化
选型: S1-10报告可视化页(含目录导航+章节+图表嵌套)
组件清单 → 侧边导航 + 章节内容区 + S4-01图表嵌入 + 目录自动生成
联调检查 → 图表数据联动、响应式、空数据处理
质量等级 → L3交付（精品级，含暗色模式）
```

### 示例3：管线模式——"帮我做日志分析并生成可视化报告"

```
决策树 → Python+HTML管线
选型 → S5-B(日志解析) → S5-B(CSV过滤) → S4-A(图表) → S1-10(报告)
组装 → 逐个组装4个产物，按接口规范设计数据流
     → 日志解析输出JSON → CSV过滤读取JSON输出CSV → 图表读取CSV → 报告嵌入图表
联调 → 管线端到端测试: mock日志→解析→过滤→图表→报告
质量等级 → L2交付（各产物独立可用 + 管线串联可用）
```

### 示例4：Shell脚本模式——"我需要一键部署生产环境"

```
决策树: 部署发布→S7-C(部署发布脚本)
组件清单 → Shebang→安全设置→环境检测→备份当前版本→部署新版本→健康检查→回滚准备→通知
联调检查 → dry-run模式验证 / 健康检查超时处理 / 回滚路径验证
质量等级 → L3交付（含dry-run + 彩色日志 + 通知集成）
```

### 示例5：SQL脚本模式——"我需要建一套用户系统的表"

```
决策树: 定义表结构→S8-A(建表脚本)
组件清单 → 数据库声明→DROP IF EXISTS→事务BEGIN→CREATE TABLE→约束→索引→注释→COMMIT→验证查询
联调检查 → 重复执行(幂等性) / 约束有效性 / 索引创建成功
质量等级 → L2交付（含方言注释 + 验证查询 + 回滚语句）
```
