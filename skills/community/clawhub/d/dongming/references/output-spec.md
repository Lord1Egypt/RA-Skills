# 输出规范 v2.0

## 一、核心准则（客户视角）

### CEO 扫描测试（强制执行）

**执行摘要必须能在 90 秒内读完。** 读完要知道三件事：
1. 问题是什么？
2. 怎么做？
3. 为什么这样做是对的？

如果 90 秒读不完 → 太长了，删。

### 每章首段即结论

不是背景介绍，不是"随着市场环境的变化……"，不是"在当今竞争日益激烈的……"。

每章第一段直接给出该章的核心判断。

### 数字优先

任何能用数字表达的判断，不许用形容词：
- ❌ "市场很大" → ✅ "TAM 320 亿，CAGR 18%"
- ❌ "增长强劲" → ✅ "过去 3 年复合增长率 27%"
- ❌ "竞争激烈" → ✅ "Top 5 集中度仅 23%，新进入者年增 15%"

### 禁止句式

报告中禁止以下表述：
- "鉴于此" → 直接说结论
- "综上所述" → 已经说过了，不需要总结
- "值得注意的是" → 如果值得注意，直接说
- "关键在于" → 直接说关键是什么
- "更重要的是" → 直接说更重要的

### 禁止引用方法论原话

报告中禁止出现：
- ❌ "Rumelt 在《好战略坏战略》中说……"
- ❌ "根据 BCG 假说驱动方法论……"
- ❌ "按照好战略的定义……"

方法论是幕后逻辑，不是台前台词。用自然语言表达判断。

### 去 AI 味

| 检测项 | 问题特征 | 修正 |
|--------|---------|------|
| 夸大宣告 | "无疑将""必将""史诗级""颠覆性" | 改为量化判断或标注置信度 |
| 破折号滥用 | 每段超过 1 个"——" | 每段最多 1 个 |
| 三段式法则 | "首先……其次……最后……" | 按逻辑自然分段 |
| 否定式排比 | "不是……不是……而是……" | 直接陈述判断 |
| 模糊归因 | "业内人士""普遍认为""众所周知" | 改为具体来源或标注"推断" |
| 修辞膨胀 | 形容词堆叠、比喻泛滥 | 删除修饰词，保留事实和判断 |

### 锋利性底线

- 执行摘要：一页内完成
- 连贯性行动：3-5 个，每个写明代价
- 指导方针必须有排他性（明确说不做什么）
- 标题即结论：每节标题必须是完整判断，不是主题标签

## 二、报告结构（Rumelt 内核）

```
封面 + 机密声明
 └─ 执行摘要（90秒可读完：诊断 + 指导方针 + 连贯性行动 + 核心代价）
     │
     ├─ 战略地图（Mermaid 图表 — 强制）
     │   └── 情势 → 诊断 → 指导方针 → 连贯性行动 → 预期效果
     │
     ├─ 零、战略意图（方向/刻度/底线/时间窗 + 坏战略检查）
     │
     ├─ 一、诊断（情势的重新框定 + 外部证据 + 内部证据）
     │
     ├─ 二、指导方针（应对思路 + 排他性 + 不做清单）
     │
     ├─ 三、连贯性行动（3-5个 + 增强关系 + 代价 + 甘特图）
     │
     ├─ 四、验证（可验证假说 + 证伪条件 + 退出路线）
     │
     └─ 附录：假说生成/筛选记录 + 红队攻击记录 + 框架分析详情
```

### 执行摘要格式

```
| 诊断 | 指导方针 | 连贯性行动 | 核心代价 |
|------|---------|-----------|---------|
| [情势本质的重新框定] | [应对思路+排他性] | [3-5个相互强化的动作] | [最大的代价是什么] |
```

### 诊断章节格式

```
## 一、诊断

> **诊断**：[对情势本质的重新框定，不是现象描述]

### 外部证据
[市场/竞争分析 → 对诊断的支撑]

### 内部证据
[组织/能力分析 → 对诊断的支撑]
```

### 指导方针章节格式

```
## 二、指导方针

> **指导方针**：[应对诊断的整体思路]
> **排他性**：因此我们不做 [X]

### 不做清单
| 看起来应该做 | 但与指导方针矛盾 | 所以不做 |
|-----------|-------------|--------|
| [X] | [矛盾点] | [理由] |
```

### 连贯性行动章节格式

```
## 三、连贯性行动

### 行动 1：[标题]
**做什么**：[具体动作]
**增强关系**：被[行动X]强化 / 强化[行动Y]
**代价**：[资金/时间/风险]

### 增强回路图
[行动间相互强化关系]

### 实施计划（甘特图）
| 行动 | 启动 | 完成 | 依赖 | 里程碑 |
|------|------|------|------|--------|
```

### 验证章节格式

```
## 四、验证

| 核心判断 | 可验证指标 | 时间窗 | 证伪条件 |
|---------|-----------|--------|---------|
| [判断] | [指标] | [时间] | [什么证据能推翻它] |

### 退出路线
如果[证伪条件]触发 → [退出方案]
```

## 三、数据呈现规范

### 图表使用

| 情景 | 图表类型 |
|------|---------|
| 市场规模/增长 | 柱状图 |
| 市场份额 | 饼图或对比表 |
| 二维定位 | 矩阵/坐标图 |
| 实施计划 | 甘特图 |
| 增强回路 | 回路图 |

### 数据标注

每条关键数据标注来源格式：`资料来源：XX报告；客户数据；洞明分析`

---

## 四、置信度体系

| 等级 | 定义 |
|------|------|
| **高** | 多源交叉验证 |
| **中** | 两个来源或单一来源+逻辑推演 |
| **低** | 推断或类比，需验证 |

S 级问题：核心判断标注置信区间（如"60-80% 概率"），而非二元高/中/低。

---

## 五、复杂度对应

| 等级 | 输出要求 |
|------|---------|
| 简单 | 直接回答 |
| 中等 | 精简报告（执行摘要 + 诊断 + 指导方针 + 连贯性行动，省略附录） |
| 复杂 | 完整报告 |
| S级 | 完整报告 + 情景规划 + 置信区间 |
