# 各 Phase 引导内容

## Phase 1：知识注入（2-3 周）

**目标**：让智能体拥有系统的完整理解。

**操作**：
1. 按 knowledge-extraction.md 的 10 步手册执行
2. 从核心模块开始提取
3. 开发者参与知识验证

**验收**：
- [ ] 任一模块：智能体能回答它做什么、输入输出、依赖关系
- [ ] 任一张表：智能体能回答字段含义、关系、哪里读写
- [ ] 任一 API：智能体能回答路径、参数、返回值、权限
- [ ] pitfalls.md 包含近 6 个月 80%+ 线上 bug 涉及的模块
- [ ] 至少 80% 核心模块知识通过人工验证（Go/No-Go #1）

## Phase 2：需求对话（3-4 周）

**目标**：需求方通过与智能体对话产出需求文档。

**操作**：
1. 先跑 3 个需求训练期（见 cold-start.md）
2. 每次对话后复盘、调优 prompt
3. 积累否定和决策知识

**验收**：
- [ ] 至少 3 个真实需求产出文档
- [ ] 澄清轮次减少 ≥ 70%（训练期 50% 也算进步）
- [ ] 边界条件覆盖率 > 90%
- [ ] 需求方主观评价 ≥ 3.5/5

## Phase 3：开发编排（6-8 周 MVP）

**前置条件**：
- [ ] Go/No-Go #2 通过
- [ ] Phase 3 验证实验完成且数据记录完整
- [ ] 硬性安全边界已设置

**验收**：
- [ ] 从需求文档到 Review 通过时间缩短 50%+
- [ ] CP2 决策质量被认可（≥80% 不需要额外补充选项）
- [ ] AI 漏掉的问题 < 20%
- [ ] 需求理解偏差导致的 Bug 为零

## Phase 4：测试审查（与 Phase 3 并行）

**目标**：测试人员从"编写用例"转为"审查策略 + 探索性测试"。

**验收**：
- [ ] CP4 盲区审查流程正常运行
- [ ] 每次漏测后 blindspot-checklist.md 有更新
- [ ] 测试人员 50%+ 时间投入探索性测试
