# 示例：创业方案 Red Team

## 用户提交的方案

```
我打算做一个面向自由职业者的 AI 记账工具，
自动读取微信/支付宝账单，用 AI 分类收支，
一键生成税务报告。定价 39元/月，目标第一年做到1万付费用户。
```

## Step 1 背景问询回答

```
目标：帮自由职业者省税务时间，一年1万付费用户
最担心：用户获取成本太高
阶段：想法阶段，还没开始开发
```

## Step 2 攻击阵型

攻击者 A：老股东 — 财务假设 + 增长逻辑
攻击者 B：竞争对手 PM — 市场竞争 + 用户切换成本

---

## Step 3 三轮攻击

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🔴 第一轮：逻辑漏洞
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

[老股东] 质疑：
- 你假设自由职业者需要税务报告，但大多数人没有主动申报的习惯，痛点多真实？
- 微信/支付宝没有开放 API，数据怎么读取？截图 OCR 的准确率能保证吗？

[竞对 PM] 补充：
- 用友、金蝶个人版早有类似功能且有品牌背书，你的差异化具体是什么？

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🟠 第二轮：执行风险
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

依赖风险：微信/支付宝平台政策不在你控制之内，一旦封堵读取路径核心功能失效
法务风险：税务报告出错谁负责？有没有免责机制？
最脆弱节点：数据获取方式——这个走不通，整个产品就不存在

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
⚫ 第三轮：最坏情景
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

情景1：微信上线3个月后更新反爬策略，数据读取全面失效，
       付费用户要求退款，你没有备用方案。

情景2：AI 分类出错导致用户少报收入被税务局查，用户起诉你承担责任。

致命问题：整个商业模式依赖非官方数据渠道，平台可随时单方面终止。
```

## Step 5 综合评估

```
整体韧性：弱

必须解决：
1. 数据获取方式——找到合规稳定的数据来源，否则产品基础不成立

建议解决：
1. 访谈20个自由职业者验证痛点真实性
2. 补充法律免责条款

一句话结论：数据获取问题不解决，不建议进入开发阶段。
```
