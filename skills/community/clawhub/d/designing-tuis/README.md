# designing-tuis

Skill-related documentation for dialogue-driven TUI design, screenshot-to-code workflows, and `.tui` schema-based design.

## Contents
- `SKILL.md`: primary skill entrypoint — dual-route architecture, 3-phase decomposition protocol, proportion mechanism, comparison/iteration protocol, and accuracy expectations
- `references/`: supporting material loaded only when needed
  - `components-cheatsheet.md`: component defaults, visual signatures, and disambiguation rules
  - `layout-inference-rules.md`: layout direction, component identification, spacing, proportion, border, and color inference rules
  - `color-accessibility.md`: WCAG color standards, color-blind safe palettes (Okabe-Ito, IBM), contrast requirements, and TUI-specific accessibility guidelines
  - `schema-to-code-mapping.md`: .tui Schema → framework code (BubbleTea/Ink/Textual) conversion reference
  - `known-quirks.md`: tui-studio pitfalls and workarounds
  - `workflow-examples.md`: dual-route workflow walkthroughs with 3-phase protocol examples
