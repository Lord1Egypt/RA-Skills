# 筛选项折行与高度自适应（筛选行 / el-radio-group）

适用于：将静态「标签 + 横向一排选项」改造为 `el-radio-group` + `el-radio`（或同类横向筛选 chip）后，在**视口变窄或选项较多**时需要**折行**，且须避免与下一行筛选、列表区域**重叠**。

---

## 1. 现象与根因

| 现象 | 常见根因 |
|------|------------|
| 折行后选项与下一行文字叠在一起 | 筛选**外层卡片**或**每一筛选行**仍使用与设计稿一致的**固定 `height`**，多出行被裁切在盒内或溢出后与下方重叠 |
| 同一行内标签与选项重叠 | 筛选行 `flex` 子项未设 `min-width: 0`，`el-radio-group` 被压窄后内部仍单行排列，与左侧标签区抢占空间 |
| 折行后行距过密、仍像「糊成一团」 | 仅用 `flex-wrap: wrap` 但未设 **`row-gap` / `column-gap`（或子项 margin 策略）**，换行后垂直方向无间距 |
| 选项 pill 内文字被裁切 | `.el-radio` 或 `__label` 写死 `height: 22px` 且无 `min-height` / `padding`，与 `line-height` 冲突 |

根因归纳：**把「设计稿单行高度」当成了布局的 `height`，而不是 `min-height`；间距仍依赖逐项 `margin-left`，换行后「首项/次行」边距规律失效，应用 `gap` 统一控制。**

---

## 2. 识别步骤

1. 在 `Custom.vue` 中定位**筛选区域**（通常对应 index 中带边框、圆角的白底容器，内含「课程 / 年级 / 学期」等行）。
2. 在 `<style scoped>` 中搜索这些容器及子行上的 **`height:`**（如整卡 `164px`、每行 `22px`）。
3. 检查筛选行是否为 `display: flex` + `flex-wrap: wrap`（或应折行却为 `nowrap`）。
4. 检查 `el-radio-group` 是否 **`flex: 1`** + **`min-width: 0`**，选项间距是否仅靠 **`margin-left`** 链式控制。
5. 在浏览器中缩小宽度或增加选项文案长度，确认是否出现**与下一行重叠**或**垂直方向裁切**。

---

## 3. 修复方法（通用）

### 3.1 筛选外层容器（卡片）

- 将固定 **`height`** 改为 **`min-height: [原设计高度]`**，并设 **`height: auto`**（或不再声明 `height`）。
- 增加适量 **`padding-bottom`**，避免最后一行贴底边框过紧。
- 保持 **`box-sizing: border-box`**。

### 3.2 每一筛选行（标签 + 选项组）

- 行容器：`flex-wrap: wrap`，**`align-items: flex-start`**（或 `align-content: flex-start`），**`gap: [行间距] [列间距]`**（如 `8px 16px`）。
- 行宽：在保留设计 **`max-width`** 的前提下，使用 **`width: 100%`**，避免沿用稿中较窄的固定 `width` 导致过早换行或布局异常（可通过更高优先级选择器覆盖 index 拷贝来的宽度）。
- 左侧标签：`flex-shrink: 0`，**`height: auto`** + **`min-height: [原行高]`**，**`line-height`** 与选项一致，避免固定高度裁切。

### 3.3 `el-radio-group` 与 `el-radio`

- `el-radio-group`：`display: flex`，**`flex-wrap: wrap`**，**`row-gap` / `column-gap`** 与稿面间距大致一致；**`flex: 1`**、**`min-width: 0`** 以占据标签右侧剩余空间。
- 去掉（或归零）子项 **`margin-left` 链**，改由父级 **`gap`** 控制，避免换行后第一列出现异常大空白或间距不一致。
- `.el-radio`：使用 **`min-height: [原高度]`**，**`height: auto`**；`__label` 可 **`padding: 2px 8px`**（示例）+ **`line-height`** 与稿一致，保证略增高时仍垂直居中。

### 3.4 「学期」类窄行升级为全宽折行

若 index 中该行容器为固定 **宽度 + 高度**（如 `386px` × `22px`），改造后应改为与上方筛选行一致的 **全宽策略**：**`width: 100%` + `max-width` 与面板一致**，**`height: auto` + `min-height`**，**`flex-wrap: wrap`**，**勿**依赖 `justify-content: space-between` 强行撑满单行（窄屏下易与标签错位）。

### 3.5 名称行（标签 + 搜索框）

若该行存在固定 **`height: 28px`**，在需要与上行折行结果共存时，改为 **`min-height` + `height: auto`**，并允许 **`flex-wrap: wrap`** + **`gap`**，避免搜索框与换行后的选项重叠。

---

## 4. 需求文案（可复制）

**筛选区域折行**

- 筛选卡片在选项折行时**高度随内容增高**，不得固定为设计稿单行高度而导致裁切或与下方模块重叠。
- 每一筛选行在折行时，行与行之间、选项与选项之间须有**明确间距**（如通过 `gap` 实现），**不得出现文字或控件互相重叠**。
- 「标签 + 选项组」布局在宽度不足时允许换行；左侧标签与右侧选项组**不得互相压盖**；`el-radio-group` 应能占满标签右侧剩余宽度并正确换行。
- 选项样式在折行后仍保持可读，选中态/主题色逻辑不变。

---

## 5. 自检清单

- [ ] 筛选外层容器为 **`min-height` + 自动增高**，非固定 `height` 裁切多行内容  
- [ ] 各筛选行 **`flex-wrap: wrap`** + **`gap`**（或等价垂直间距）  
- [ ] **`el-radio-group`** 使用 **`flex: 1`**、**`min-width: 0`**、**`flex-wrap: wrap`**  
- [ ] 选项间距以 **`gap`** 为主，非长链 **`margin-left`**  
- [ ] `.el-radio` / `__label` 使用 **`min-height`**，避免固定 `height` 导致裁切  
- [ ] 窄屏/多选项下无与**下一筛选行、列表表头**的**重叠**  
- [ ] PC 设计宽度下视觉与稿面**单行时**仍基本一致（误差可接受范围内）
