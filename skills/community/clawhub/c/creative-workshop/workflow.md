### 1.2 Stage-Gate 模型

创意造物局采用5阶段4门控的Stage-Gate模型：

```
┌────────┐     ┌────────┐     ┌────────┐     ┌────────┐     ┌────────┐
│        │     │ Gate 1 │     │ Gate 2 │     │ Gate 3 │     │ Gate 4 │
│ Stage 1│────→│方向确认│────→│方案确认│────→│报告确认│────→│IP策略  │────→ Output
│ 猎手   │     │        │     │        │     │        │     │确认    │
└────────┘     └────────┘     └────────┘     └────────┘     └────────┘
                    │              │              │              │
               用户选择方向     用户确认方案    用户确认报告   用户确认IP策略
```

#### Gate 评审规则

| Gate | 评审内容 | 通过标准 | 失败处理 |
|------|---------|---------|---------|
| Gate 1 | 方向选择 | 用户选择1个方向 | 返回猎手重新扫描 |
| Gate 2 | 方案质量 | 方案满足基本要求 | 返回策划师调整 |
| Gate 3 | 商业可行 | 可行性评分≥5 | 建议重新设计或放弃 |
| Gate 4 | IP策略 | 用户确认IP策略 | 调整策略或简化 |

#### 流程图

用户输入原始创意
    │

    ▼
┌─────────────────────────────────────────────────────────────────────┐

│ ├─ 行业趋势分析（3年内趋势）                                        │
│ ├─ 热点话题追踪（近6个月）                                          │
│ └─ 政策环境扫描（相关政策）                                          │
└─────────────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────────────┐
│ Step 3：机会识别 ✅ AI自动执行                                       │
│ ├─ 市场空白点分析                                                   │
│ ├─ 用户痛点挖掘                                                     │
│ └─ 差异化机会判断                                                   │
└─────────────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────────────┐
│ Step 4：灵感激发 ✅ AI自动执行                                       │
│ ├─ 类比案例引入（2-3个相关案例）                                     │
│ ├─ 跨界灵感迁移（相邻领域灵感）                                      │
│ └─ TRIZ进化趋势扫描（可选）                                         │
└─────────────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────────────┐
│ Step 5：方向提案生成                                                 │

## 四、并行与串行调度

### 4.1 调度模式选择

| 场景 | 调度模式 | 说明 |
|------|---------|------|
| 单一角色任务 | 串行执行 | 按顺序执行各步骤 |
| 独立分析任务 | 并行执行 | 市场分析+竞品分析同时进行 |
| 全流程任务 | 流水线 | 按阶段顺序执行，Gate控制 |

### 4.2 并行执行示例（分析师阶段）

```
串行执行（默认）：
    市场分析 → 商业模式 → 竞品分析 → 财务测算 → 风险评估 → 结论

并行执行（可选）：
    ┌─────────────────────────────────────────────────────┐
    │                                                     │
    │  市场分析 ──┐                                       │
    │             │                                       │
    │  竞品分析 ──┼──→ 综合汇总 → 结论                   │
    │             │                                       │
    │  财务测算 ──┘                                       │
    │                                                     │
    │  风险评估 ──────────────────────────────────→ 结论  │
    │                                                     │
    └─────────────────────────────────────────────────────┘
```
