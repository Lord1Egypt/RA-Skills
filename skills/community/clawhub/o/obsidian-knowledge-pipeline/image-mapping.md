# Image Mapping — 配图生成与映射规则

## 配图生成

### 方案选择逻辑

```
if DASHSCOPE_API_KEY 环境变量存在且非空:
    方案A: Wan2.7 脚本生成
    参考: ~/.agents/skills/wan2.7-image-skill/SKILL.md
else:
    方案B: image_generate 工具生成
```

### 方案A: Wan2.7 脚本

```bash
# 1. 构造 prompt
PROMPT="{核心概念}的概念插画，科技风格，扁平化设计，16:9比例，高清"

# 2. 调用 Wan2.7 脚本
# 具体调用方式参考 wan2.7-image-skill 的 SKILL.md
# API Key 通过 $DASHSCOPE_API_KEY 环境变量传递

# 3. 保存输出
# 输出文件保存到: $IMAGES_DIR/{主题}-配图.png
```

### 方案B: image_generate 工具

```
调用 image_generate 工具:
  prompt: "{核心概念}的概念插画，科技风格，扁平化设计，16:9比例"
  size: "1536x1024" (或 "1024x1024")
  outputFormat: "png"
  filename: "{主题}-配图"
```

### Prompt 构造规范

| 分类 | Prompt 模板 | 色调建议 |
|------|------------|----------|
| AI应用 | "{主题}AI应用场景概念图，科技蓝色调，扁平化设计" | 蓝色系 |
| AI技术 | "{主题}技术架构示意图，深紫色科技感，信息图表风格" | 紫色系 |
| 基础设施 | "{主题}基础设施拓扑图，深色背景，节点连线风格" | 深灰/绿 |
| 金融科技 | "{主题}金融科技数据可视化，金色蓝色渐变" | 金色/蓝色 |
| 人物观点 | "{人物名}技术理念可视化，现代简约风格" | 中性色 |
| 商业与消费 | "{主题}商业趋势信息图，暖色调渐变" | 暖色系 |

### Prompt 约束

- 必须包含主体描述 + 风格 + 色调
- 禁止使用真实人名出现在生成 prompt 中（人物观点类使用抽象表达）
- 中文 prompt 即可，工具会自动处理

## 配图保存规则

### 路径

```
$IMAGES_DIR/{主题}-配图.png
```

其中：
- `$IMAGES_DIR` = `$OBSIDIAN_VAULT_PATH/raw/images/`（默认 `~/Documents/Obsidian Vault/raw/images/`）
- `{主题}` 从文章核心概念提取，中英文均可
- 文件名中禁止包含：空格、`/`、`:`、`*`、`?`、`"`、`<`、`>`、`|`
- 多个词用 `-` 连接

### 命名示例

```
✅ Vibe-Trading-配图.png
✅ LangChain技术架构-配图.png
✅ K8s-服务网格-配图.png
❌ Vibe Trading 配图.png （包含空格）
❌ Vibe:Trading配图.png （包含冒号）
❌ assets/vibe-trading.png （禁止使用 assets 目录）
```

## 配图引用格式

### 在 raw 文章中

```markdown
![配图](images/{主题}-配图.png)
```

- 使用相对于 `raw/` 目录的路径
- 固定使用 `images/` 前缀
- 禁止使用绝对路径

### 在 Wiki 编译节点中

```markdown
![{主题}配图](../../raw/images/{主题}-配图.png)
```

- 使用相对于 Wiki 节点位置的相对路径
- 从 `wiki/知识透视/{分类}/` 回溯到 vault 根需两级 `../../`
- 禁止使用绝对路径

## 映射关系文件

### 文件位置

```
$IMAGES_DIR/关联关系.md
```

即 `raw/images/关联关系.md`

### 文件初始化模板

若文件不存在，创建时写入：

```markdown
# 配图关联关系

> 本文件记录所有文章与配图的映射关系
> 由 OpenClaw Agent 自动维护
> 格式标准化，请勿手动修改结构

---
```

### 追加条目格式

每处理一篇文章，在文件末尾追加一个条目：

```markdown
### {文章标题}

| 字段 | 值 |
|------|-----|
| 文章路径 | `raw/{文章标题}.md` |
| 配图路径 | `images/{主题}-配图.png` |
| 配图Prompt | {生成时使用的完整 prompt} |
| 生成方案 | {Wan2.7 \| image_generate} |
| 生成时间 | YYYY-MM-DD HH:mm |
| 分类 | {自动分类结果} |
| 状态 | ✅ 已完成 |

---
```

### 映射文件维护规则

1. **仅追加，不删除**：映射条目是历史记录，即使文章被删除也保留映射
2. **状态标记**：正常完成标记 `✅ 已完成`，配图生成失败标记 `❌ 配图缺失`
3. **禁止重复**：处理前先检查映射文件中是否已存在同标题条目
4. **格式一致**：所有条目使用统一的表格格式
