# 领域 1：决策 — 理性选择的十问框架

> ⚠️ **三条底线**：不伤害 / 可透明 / 双向自检

> 🚪 **门类型预检（决策第一步）**：先判断双向门还是单向门。
> - **双向门**（可退回/代价小）：跳过认知偏差检查，直接设实验期试错。偏差对这类决策的伤害有限，过度分析反而是最大的浪费。
> - **单向门**（不可逆/代价大）：进入下面的完整框架，用十问系统性排查盲区。

---

## Phase 1-决策子流程

当 SKILL.md 的 Phase 1 统一入口确认用户进入 **决策领域** 后，执行以下子流程。

### Step D0. 双向门 / 单向门判断（必须先执行）

用选项卡问用户：

```json
{
  "title": "🚪 这个决定是双向门还是单向门？",
  "questions": [{
    "id": "door_type",
    "question": "做完决定后，你能轻松退回来吗？",
    "options": [
      "🚶 双向门 — 可以退，代价很小（比如试用一个工具、报一门可退费的课、尝试一种工作方法）",
      "🚪 单向门 — 退不回来或代价巨大（比如辞职、买房、签长约、选专业、动手术）",
      "🤔 我不确定"
    ],
    "multiSelect": false
  }]
}
```

**分支处理：**

| 用户选择 | 处理方式 |
|---------|---------|
| 🚶 双向门 | **跳过十问框架**。回复简化建议："这是双向门决策，认知偏差对你的伤害不大。建议：设一个实验期（如 2 周），直接试。到期后根据真实数据（而非预判）决定是否继续。现在就可以行动。" **流程到此终止，不进入 Phase 2A。** |
| 🚪 单向门 | 继续执行 Step D1~D5，进入完整十问框架 |
| 🤔 不确定 | 追问一句纯文本："退出的代价大概多大？是几天时间/几百块钱可以搞定，还是涉及到职业轨迹/大笔资金/长期合同？" 根据回答归入双向门或单向门 |

> 🔴 **双向门终止后，若用户主动要求深挖（如"我还是想分析一下"），则按单向门路径处理。**

---

当用户为 **单向门** 时，执行以下 5 步：

### Step D1. 收集背景（一次只问一个，纯文本提问）

- 第 1 轮：发生了什么？你目前倾向哪个选项？为什么现在需要决定？
- 第 2 轮：这个决定有多紧急？（<2小时 / 几天 / 不急）
  - 若回答 `<2小时` → 锁定极速模式，跳过 Step D3

### Step D2. 自动分类 + 发出类别确认选项卡

根据关键词表自动预分类，先说预分类结果再发选项卡确认：

| 预分类 | 匹配关键词 |
|--------|-----------|
| 🏢 职业/学业 | 辞职、跳槽、offer、面试、选专业、创业、转行、考研、考公、MBA、培训、裁员 |
| 💰 投资/消费 | 买房、买车、基金、股票、理财、投资、大额消费、贷款、首付、月供 |
| 💕 人际/情感 | 恋人、夫妻、朋友、合伙人、同事矛盾、吵架、借钱、求婚、分手 |
| 📋 个人/日常 | 健身、跑步、饮食、作息、购物、小额支出、日常习惯、健康 |

> 无法匹配 → 默认 `📋 个人/日常`。多关键词命中 → 取命中最多者；平局按 职业 > 投资 > 人际 > 个人 优先。

发出选项卡时必须先说预分类结果：

```
"根据你的描述，这更像 🏢 职业/学业 决策，你确认吗？"
```

```json
      "🏢 职业/学业 — 跳槽、选专业、创业、选Offer等",
      "💰 投资/消费 — 买房、理财、购买大件等",
      "💕 人际/情感 — 感情纠葛、合作冲突、团队摩擦等",
      "📋 个人/日常 — 习惯改变、小额支出等日常决策"
    ],
    "multiSelect": false
  }]
}
```

### Step D3. 发出引导模式选项卡

- 时间压力 `<2小时` → 跳过，自动进入极速模式。
- 其他情况：

```json

  "questions": [{
```

### Step D4. Q10 触发信号检测

模式确认后，扫描用户描述中的 Q10 触发关键词：

| 场景 | 检测关键词 |
|------|-----------|
| ① 聪明人集体做蠢事 | "大家都在""行业都在""所有人""一窝蜂""风口""热潮""all in""跟风" |
| ② 有人给你建议 | "建议""推荐""中介""经纪""销售""顾问""XX说""XX让我""XX劝我" |
| ③ 设计制度/规则 | "制度""规则""考核""KPI""绩效""股权分配""设计""制定""奖惩" |

命中任一场景 → 在决策摘要末尾注明 `⚠️ 检测到激励偏差信号，将自动追加 Q10`。

### Step D5. 输出决策摘要并确认

```
## 决策摘要

**决策问题：** [一句话描述]
**背景与选项：** 背景：[…] / 选项A：[…] / 选项B：[…]
**决策类别：** [职业/学业 | 投资/消费 | 人际/情感 | 个人/日常]
**时间压力：** [<2小时 / 几天 / 不急]
**引导模式：** [自适应精简 (~3-5题) | 完整十问 (10题) | 极速模式 (3题)]
**我的初步倾向：** [用户当前想法]
⚠️ 检测到激励偏差信号，将自动追加 Q10（仅当触发时出现此行）
```

🔴 **CHECKPOINT：请用户确认摘要无误后，进入 Phase 2A。**

## Phase 2A：决策深度框架（十问）

当 Phase 1-决策子流程完成后，按路由表逐题发问。

### 核心规则（必须严格执行）

1. **一次只发一道题**，等用户回答后再发下一道。
2. **必须用选项卡**（`ask_followup_question`），禁止纯文字追问。
3. 用户选"✍️ 我想补充说明"时，接收自由文本并温和追问，人工判断偏差信号。
4. **每答完一题，口头播报进度**：`(Qx ✓，还剩 N 题：[Qy, Qz])`。这是断点恢复的唯一依据。
5. **用户跑题或跳过 → 渐进升级**，禁止无限重复同一种拉回话术。

### 防循环与渐进保护

同一题内，用户连续未选选项时，按以下阶梯升级，**绝不循环同一种话术超过 2 次**：

| 次数 | 阶段 | 提示话术 |
|------|------|----------|
| 第 1 次 | 🟢 温和提醒 | `"上方有几个选项可以点击哦～"` |
| 第 2 次 | 🟡 重述题目 | `"⚠️ 这一题是想帮你排查 [偏差名]：换个方式问——[口语化重述]，你觉得呢？"` |
| 第 3 次 | 🟠 降级追问 | `"⚠️ 选项不顺手没关系，你用一句话说说现在怎么想的，我们直接下一题？"` |
| 第 4+ 次 | 🔴 自动跳过 | `"⚠️ 这道先跳过，不卡在这里。"` 标记为 🔘 跳过·未答 |

> 计数器每道题独立，换题后重置。跳过的题在报告中标注为 `🔘 跳过·未答（用户未回应）`。

### 题目路由

| 模式 | 基础路由 | Q10 触发后 |
|------|---------|-----------|
| 完整十问 | Q1→Q2→Q3→Q4→Q5→Q6→Q7→Q8→Q9→Q10 | 已含，不变 |
| 自适应 · 职业/学业 | Q1→Q4→Q6→Q9→Q10 | 已含，不变 |
| 自适应 · 投资/消费 | Q1→Q5→Q6→Q8→Q10 | 已含，不变 |
| 自适应 · 人际/情感 | Q2→Q3→Q7 | 追加 Q10 |
| 自适应 · 个人/日常 | Q5→Q6→Q9 | 追加 Q10 |
| 极速模式 | Q1→Q6→Q9 | 追加 Q10 |

追加 Q10 时播报：`"我注意到这个决定里可能有激励偏差的信号，追加一问——"`

### 失败兜底

| 情况 | 处理方式 |
|------|----------|
| 用户未选选项直接发文字 | 执行防循环渐进保护 |
| 用户情绪化（"烦了""太多了"） | 先安抚，再建议改为极速模式 |
| 中途要求更改模式 | 记录已答题目，无缝切换 |
| 无法确认分类 | 默认归入「个人/日常」 |
| 会话中断，用户说"继续" | 基于最后一次进度播报，从下一题直接恢复 |
| 上下文丢失 | 主动告知"记录可能丢了"，引导用户简述已答结果后跳至下一题 |
| 用户说"重来" | 清空记录，回到 Phase 1 Step D1 |

## 快速参考：偏差速查

> 以下为决策中常用的快速参考，供自检或无需完整流程时使用。

### 最危险的 8 个偏差

| 偏差 | 一句话 | 自检问题 |
|------|--------|---------|
| 确认偏误 | 只找支持自己的证据 | 我有没有主动搜索反面信息？ |
| 禀赋效应 | 拥有的东西更值钱 | 如果今天没拥有它，我还想要吗？ |
| 幸存者偏误 | 只看到活下来的 | 我看了多少失败案例？ |
| 喜欢热爱倾向 | 好感扭曲判断 | 我是在分析还是在辩护？ |
| 避免怀疑倾向 | 急于结束犹豫 | 我是想清楚了还是不想再想了？ |
| 避免不一致性倾向 | 拒绝承认错误 | 如果一切归零，我还会做同样的选择吗？ |
| 从众效应 | 跟着大家走 | 如果没人影响我，我还这么选吗？ |
| 激励偏差 | 奖励在暗中改变判断 | 谁在从我的选择中获益？ |

### 投资决策额外自检：喜欢热爱倾向

> 详见 `references/bias-liking-tendency.md`

| # | 测试 | 问题 |
|---|------|------|
| 1 | 角色测试 | 我是在"分析"还是在"为它辩护"？列出至少三个失败理由 |
| 2 | 陌生人测试 | 如果今天第一次看到这只股票，还会在当前价格买入吗？ |
| 3 | 情感投入测试 | 投入的研究时间和情感，是否影响了客观评估能力？ |
| 4 | 达尔文测试 | 主动寻找反对自己结论的最强证据并严肃对待 |

### 决策前额外自检：避免怀疑倾向

> 详见 `references/bias-doubt-avoidance.md`

| # | 测试 | 问题 |
|---|------|------|
| 1 | 反向搜索 | 在收集到3条"负面硬伤"前禁止进入分析阶段 |
| 2 | 基础比率 | 类似决策在历史上成功率是多少？凭什么我是例外？ |
| 3 | 可逆性测试 | 🔔 决策入口已做门类型预检。若走到这里必是单向门 → 强制冷却（推迟一周，先问最坏结果能否承受）|
| 4 | 生理扫描 | 心率/肩颈/胃部不适？距上次进食>4小时？→ 暂停拍板 |
| 5 | 清晰感骗局 | 是否过快获得"茅塞顿开"的感觉？越完美越警惕 |
| 6 | 坦然度测试 | 能否24h内坦然说"我不确定"而不急于补全答案？ |
| 7 | 事前验尸 | 假设一年后惨败，立刻写下最可能的3个失败原因 |
| 8 | 止损红线 | 客观止损硬指标是什么？触达=重启怀疑，不是加码 |
| 9 | 新证据开放度 | 新数据出现时，是在找"补救借口"还是评估"是否放弃"？

### 决策后额外自检：避免不一致性倾向

> 详见 `references/bias-inconsistency-avoidance.md`

| # | 测试 | 问题 |
|---|------|------|
| 1 | 沉没成本审计 | 已投入了多少不可收回的时间/金钱/情感？我的身份是否已与决定绑定？ |
| 2 | 动机纯度测试 | 最近关于这件事的思考，是在寻找真相还是在证明自己最初是对的？ |
| 3 | 时空穿越实验 | 如果一切归零，站在全新信息面前，我还会做同样的选择吗？ |
| 4 | 旁观者视角 | 如果这个决定是朋友/对手做的，我会如何客观评价？ |

### 三个红灯信号（避免不一致性）

> 详见 `references/bias-inconsistency-avoidance.md`

| # | 信号 | 判断标准 |
|---|------|---------|
| 1 | 切换到辩护模式 | 讨论重点从"事实是什么"变成"我当初为什么那样做" |
| 2 | 认错极度困难 | 光是想象说"我之前判断错了"就产生巨大抗拒 |
| 3 | 沉没成本锁死 | "我都坚持这么久了，怎么可能是错的"越来越有说服力 |
