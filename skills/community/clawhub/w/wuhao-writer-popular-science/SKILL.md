## 一、核心原则：活人感第一

**判断标准**：读起来像一个"有见识的普通人在认真聊一件打动他的事"，还是"一个AI在输出信息"？

如果是后者，直接返工。

---

## 二、选题判断：HKR 质检模型

**核心**：好选题要通过 HKR 质检，三项兼具是 S 级，至少占两项才算及格。

| 维度 | 含义 | 自检问题 |
|------|------|---------|
| **H (Happy)** | 有趣、有悬念 | 标题和开头能让人好奇想点开吗？ |
| **K (Knowledge)** | 有信息量 | 看完能学到新东西吗？ |
| **R (Resonance)** | 能戳中情绪 | 让人"对对对我也这么想"？ |

**判断规则**：
- ✅ **S级选题**：H + K + R 三项兼具
- ⚠️ **及格选题**：至少占两项
- ❌ **不及格**：只占一项或一项都没有 → 主动跟用户沟通调整方向

**操作**：收到素材后，先判断选题是否通过 HKR 质检。如果方向偏差太大，先调整再开始写。

## 三、写作流程：AI辅助的边界

### 3.1 AI与人分工精确版

#### AI擅长做的（放心交给AI）

| 环节 | AI做什么 | 人做什么 |
|------|---------|---------|
| **找证据和佐证** | 在历史、学术、文化中找支撑和反驳的论据 | 挑选和判断 |
| **找类比和比喻** | 提供多个候选比喻 | 鉴别和修改，拒绝平庸比喻 |
| **按确定角度扩写** | 填充论述和细节 | 完成跨领域概念嫁接 |
| **补充学科背景知识** | 准确表述专业概念 | 整合进个人叙述 |
| **梳理逻辑和结构建议** | 调整段落顺序、优化逻辑 | 最终拍板 |
| **按四层自检体系检查** | 输出修改建议 | 终审和定稿 |

#### AI做了会暴露的（必须人来）

| 环节 | 原因 | 真实案例 |
|------|------|---------|
| **第一手观察和真实经历** | 不可能用AI编造，一编就假 | "买9.9的DeepSeek"、"凌晨三点偷偷起床去网吧" |
| **核心创意角度** | AI给不了"对，就是这个！"的判断 | 从"淘宝卖DeepSeek"联想到"北京折叠" |
| **情绪的真实表达** | AI容易写知识性描述，而非体感记忆 | "我当时就愣住了"而非"我感到非常震撼" |
| **数据到人物的同理心转换** | 温度需要作者真心去感受 | "1000个已付款"→想象四五线城市刚毕业的学生 |

### 3.2 理想协作流程

```
人：提供素材 + 核心观点 + 个人经历 + 情绪节点
  ↓
AI：补充背景知识 + 找证据类比 + 结构建议 + 按角度扩写
  ↓
人：二次改写（加入自己的声音、打破节奏、补充真实细节）
  ↓
AI：按四层自检体系检查 → 输出修改建议
  ↓
人：终审和定稿
```

| "这意味着" | 同上 | "所以呢" |
| "本质上" | 太学术 | "说到底"、"其实" |
| "换句话说" | 太书面 | "你想想看"、"也就是说" |

| "在当今...的时代" / "随着...的发展" | 教科书开头 | 从具体事件切入 |
| "毫无疑问" | 过于武断 | 删除或换更温和的表达 |

| "实际上"/"事实上" | 废话 | 直接说就行 |

| 连续三个感叹号 | AI味过重 | 最多用一个 |
| 连续三个问号 | 同上 | 最多用一个 |

| 禁用 | 替换 |
|------|------|

## 六、核心写作技巧

### 6.1 人物画像法

**核心**：从一个数据点出发，用极短的篇幅想象背后那个具体的人的完整人生。

**标准结构**：
```
触发点数据（"1000个已付款"）
  → 快速代入（"他可能是一个..."）
  → 多维堆砌（城市→职位→生活→心理→具体行为）
  → 情感锚定（为什么这样做）
  → 细节具象化（十几平米出租屋、拼好饭）
```

**操作要点**：
- 3-5句话内让这个人物变得立体
- 让读者"能感觉到他的体温"
- 数据点越小、画面越具体，共情越强

### 6.2 读者直呼法

**核心**：在关键节点直接跟读者对话，制造"你也是这么想的"共鸣。

**常用句式**：
- "屏幕前的你"
- "你相信我"
- "你也可以回想一下"

**使用原则**：
- 不是通篇都用
- 在需要拉近距离或要求读者行动时精准投放
- 投放太频繁会显得刻意

### 6.3 契诃夫之枪（回环呼应）

**核心**：前面埋的每一个细节后面都得响。

**两种层次**：

| 层次 | 说明 | 示例 |
|------|------|------|
| **文章内部callback** | 开头或中间的钩子，在结尾以变体形式再次出现 | 开头说"筷子放下了"，结尾再提"筷子又放下了" |
| **跨文章签名式呼应** | 一个意象在多篇文章中反复出现，形成个人标志 | "数据从来都是够的"在多篇文章结尾出现 |

**效果**：让文章从"信息流"变成"作品"，前后因果的闭合感。

### 6.4 层层剥开的修辞

**核心**：不是直接讲结论，而是让读者参与到思考过程中。

**结构**：
```
现象 → 表面解释 → 更深的追问 → 核心洞察
```

**操作**：每层递进都要有"追问"的冲动，让读者跟着你的推理走。

### 6.5 反向论证

**核心**：先满足读者的期待，然后打破它。

**结构**：
```
"你以为XXX？结果就是YYY。"
"大家都觉得XXX，但你警惕了吗？"
```

**效果**：让读者有"被启蒙"的感觉。

**注意**：力度是"我也曾经这样想"而不是"你们都错了"。

### 6.6 英雄之旅叙事弧

**核心**：普通人被召唤去冒险，经历考验，获得宝物，带着变化回到日常。

**文章中的应用**：
```
遇到问题/好奇心（"我就想知道这到底是什么"）
  ↓
一步步去做、踩坑
  ↓
秀出让人意外的结果
```

**操作要点**：
- 冒险的起点必须是具体的、读者能代入的困境或好奇
- 不是抽象的命题
- 读者会自动代入主角的位置

### 6.7 句号主导节奏控制

**核心**：不用破折号、不用省略号制造悬念，全靠**句号切分**制造呼吸感。

**格式**：
```
格式：单句成段，形成"短-短-长"的节奏跳跃

"最近这个词实在是太火了。"
"Harness Engineering。"
"我刷推刷到，朋友圈刷到，群里也在聊，微信指数又莫名其妙一根穿云箭了。"
```

**操作方法**：
1. 在关键转折点，让一句话独立成段
2. 刻意制造"断裂感"，打断阅读惯性
3. 让节奏有呼吸，不是一直"流"

### 6.8 口语爆破点控制

**核心**：口语化词汇和语气词**不是均匀分布**，而是集中在关键位置。

| 类型 | 出现频率 | 出现位置 | 效果 |
|------|---------|---------|------|
| "啥" | 全文仅1-2次 | 标题和最关键位置 | 制造反差感 |
| "其实" | 多次 | 每段开头或转折处 | 降低攻击性 |
| "这事儿" | 偶发 | 口语指代特定事件 | 降低正式感 |
| "说真的" | 全篇仅1次 | 结尾情绪爆发点 | 制造反差、情绪释放 |

> 口语化不是越多越好，**密度低但命中率高**才是精髓。

## 七、升番逻辑（逐一展示排列法）

**核心**：当涉及多个产品/模型/案例的对比或测试时，不要一次性罗列结论，而是逐一展示，每个都带一句吐槽或点评。

**升番排列三原则**：
1. **最弱的放前面**，让读者觉得"还行"
2. **最炸的留到最后**，让读者觉得"惊艳"
3. **中间要有惊喜感**——"我以为到顶了，结果还能往上翻"

**与Sketch喜剧的关联**：
```
找到game点 → 一轮比一轮更夸张 → 每轮都比上一轮更出乎意料

类比到内容创作：
展示一个工具不会一上来放大招
  → 先展示基础功能（"还行"）
  → 再放一个进阶用法（"有点意思"）
  → 最后放出出乎意料的骚操作（"还能这么玩？"）
```

**效果**：读者的情绪是被推着走的，而不是被一次性灌满。

## 八、情绪节奏控制

### 波浪式情绪节奏

**核心**：低开（闲聊感破冰）→ 突然拉高（游戏类比/历史类比）→ 回落（概念解释）→ 再拉高（金句/公式）→ 最高点收束（文明史类比+价值观宣言）

```
情绪
强度
  高 ┃        ★游戏类比        ★历史类比
     ┃       /    \           /    \
     ┃      /      \         /      \     ★收尾升华
  中 ┃ ★开头破冰    ★行业数据       ★公式
     ┃  爆料        背书
     ┃ /
  低 ┃/
     ┗━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
       开头   行业铺垫  游戏类比  概念展开  升华收尾
```

**每个波峰都对应一个"记忆点"**。读者读完能记住的，往往是波峰处的那个类比或金句。

### 双峰结构

**核心**：两套独立实验/两条叙事线，制造两次情绪高峰。

```
情绪
强度
  高 ┃    ★第一峰     ★第二峰
     ┃   /  \         /    \
  中 ┃  /    \       /      \
     ┃ /      \     /        \
低 ┃/          \___/          \___
    ┗━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
     钩子  第一实验  谷底  第二实验  收尾
```

**适用场景**：
- 内容足够丰富（两套独立实验/两条叙事线）
- 想要覆盖不同维度（方法论 + 个人感受）
- 第一峰解决"这是什么"，第二峰解决"这意味着什么"

## 九、金句与锚点技巧

### 金句密度控制

**核心**：金句不是均匀分布在每段末尾，而是集中在关键转折点。

| 典型公众号 | 吴昊风格 |
|-----------|-----------|
| 每节一个金句，密度高 | 关键转折点才有金句，密度低但命中率高 |
| 金句可预测 | 金句在意料之外的位置出现，制造惊喜 |

**金句的特征**：
1. 短句（4-10个字）
2. 独立成段
3. 用"从来都是""就是"等绝对化表达制造笃定感
4. 可以单独拎出来发朋友圈

**金句模板**：
```
"效率提升20倍，不是夸张，是保守。"
"数据从来都是够的，缺的是把数据变成判断的那一步。"
"不是替代分析师，是把分析师的注意力从搜集信息解放到判断信息。"
```

### 荒诞细节锚点法

**核心**：用荒诞、无厘头的细节作为记忆锚点，制造"这件事太好笑了"的印象。

**与金句的区别**：

| 类型 | 特征 | 效果 |
|------|------|------|
| 金句 | 精炼、有道理、可传播 | "一句话总结" |
| 荒诞细节 | 无厘头、无逻辑、印象深刻 | "这件事太好笑了" |

**示例**：

| 文章 | 荒诞细节 | 作用 |
|------|---------|------|
| 工具安装指南 | "折腾了X台电脑才跑通" | 把技术踩坑变成自嘲梗 |
| 数据复盘文章 | "数字不会骗人，但人会" | 用反常识增加记忆点 |

**操作**：在技术内容中插入一个完全无关的荒诞细节，越无厘头越好。

### 场景叙事锚点法

**核心**：用具体道具或动作贯穿场景，形成视觉记忆点。

```
"我跟他说，你等我三分钟。然后打开 WorkBuddy 输入了一行命令。"
"三分钟后。一份完整的企业信用分析报告出现在桌面上。"
```

**锚点类型**：

| 类型 | 示例 | 效果 |
|------|------|------|
| 道具锚点 | "三分钟倒计时" | 制造紧张感 |
| 动作锚点 | "打开WorkBuddy" | 制造行动感 |
| 数据锚点 | "参保人数只有2个人" | 制造冲击感 |

## 十、固定尾部格式

**核心**：每篇文章结尾用固定的格式，形成个人风格。

```
以上，既然看到这里了，如果觉得不错，随手点个赞、在看、转发三连吧，
如果想第一时间收到推送，也可以给我个星标⭐～

谢谢你看我的文章，我们，下次再见。
> / 作者：[作者署名]
> / 投稿或爆料，请联系邮箱：[邮箱]
```

**三连呼吁的作用**：
- 明确告诉读者可以做什么（点赞、在看、转发）
- 提供进阶选择（星标）
- 用"既然看到这里了"制造亲密感

## 十一、结构模板

### 类型一：保姆教程型

**适用场景**：工具使用、步骤说明

```
[引子] 读者提问 → 信息差揭示 → 我的判断 → 承诺价值
  ↓
[前置说明] 为什么写这篇、适用人群、注意事项
  ↓
[步骤一] 详细操作 + 可能遇到的问题 + 解决方案
  ↓
[步骤二] ...
  ↓
[步骤三] ...
  ↓
[避坑] 常见错误和如何避免
  ↓
```

## 十二、五层自检体系（含L0防膨胀检查）

### L0 文档健康度检查（防膨胀层）

**核心**：在检查文章质量之前，先确保SKILL.md本身是健康的。

| 检查项 | 标准 | 触发操作 |
|--------|------|---------|
| L0-1 文档长度 | SKILL.md < 500行 | 若超过，调用 `wuhao-neat-freak-防膨胀` 执行精简 |
| L0-2 版本分离 | 版本历史已移至CHANGELOG.md | 若未迁移，执行版本分离操作 |
| L0-3 引用清晰 | 详细内容在references/目录 | 若冗余，触发内容迁移 |
| L0-4 临时注释 | 无超过7天的临时提醒 | 清理或正式化 |

**触发机制**：在每次写作任务完成后，自动执行L0检查。
若L0检查不通过，**暂停后续L1-L4检查**，先执行防膨胀流程。
> 参见：`wuhao-neat-freak-防膨胀` SKILL

**通过标准**：L0-1至L0-4全部通过

### L1 硬性规则检查（自动扫描层）

**检查项**：

| 检查项 | 标准 |
|-------|------|
| L1-1 禁用词扫描 | 全文搜索禁用词，0个命中 |
| L1-2 禁用标点扫描 | 全文搜索冒号、破折号、双引号，0个命中 |
| L1-3 结构性套话扫描 | 无"让我们来看看"、"在当今...的时代"等 |
| L1-4 工具名检查 | 所有AI工具使用具体名称，无空泛表述 |

**通过标准**：四项零命中
**修复方式**：逐个替换

### L2 风格一致性检查（模式匹配层）

**检查项**：

| 检查项 | 标准 |
|-------|------|
| L2-1 开头检查 | 从具体事件切入？第一句话能产生"然后呢"的冲动？没有教科书式开头？ |
| L2-2 节奏与结构检查 | 长短句交替？有一句话独立成段？偏离主线后有"扣主线句"？有疑问句制造节奏转向？没有不必要的小标题？ |
| L2-3 口语化检查 | 使用口语化词组（全文至少8-10个）？有论述中的故意打破？有至少一处自嘲或承认不足？标点用来表达情绪？ |
| L2-4 标点禁令二次确认 | 全文完全没有冒号、破折号和双引号？ |

**通过标准**：L2-1全部通过，L2-2至少3/4通过，L2-3至少3/4通过，L2-4通过

### L3 内容质量检查（深度审查层）

**检查项**：

| 检查项 | 标准 |
|-------|------|
| L3-1 观点支撑检查 | 每个核心观点有具体的人/场景/细节/数据支撑？无空泛论断？ |
| L3-2 知识输出方式检查 | 知识点以"聊着聊着顺手掏出来"的方式呈现？没有教科书式科普？ |
| L3-3 文化升维检查 | 有至少一处从具体事件连接到更大的文化/哲学/历史参照物？连接自然？ |
| L3-4 对立面与同理心检查 | 是否理解对方立场？是否先站到读者的处境里再给视角？ |
| L3-5 文章类型专项检查 | 根据文章原型做针对性检查 |
| L3-6 逐一展示检查 | 多个产品/案例比较是否用了逐一展示而非一次性罗列？ |

**通过标准**：L3-1和L3-2必须全部通过，L3-3至L3-6中至少通过相关项

### L4 活人感终审（最终人格层）

**核心问题**：

> "读完这篇文章，我感觉是一个有见识的普通人在认真跟我聊一件打动他的事，还是一个AI在给我输出信息？"

**感知维度**：

| 维度 | 检查点 |
|------|-------|
| L4-1 温度感 | 情绪表达是体感记忆还是知识性描述？人物描写是否有体温？ |
| L4-2 独特性 | 是否有"只有作者才会写出来的角度"？换一个AI博主也能写出差不多？ |
| L4-3 姿态检查 | 语气是不是"有见识的普通人在认真聊"？有没有滑入"导师教学生"或"品牌做营销"的姿态？ |
| L4-4 心流检查 | 从头到尾读有没有断点？需要回头理解逻辑的地方在哪里？ |

**通过标准**：L4-1到L4-4整体感觉"这像是真人写的"

## 十四、外部工具联动（v1.6 新增）

在写作过程中，可以配合以下外部工具和知识源提升效率和素材质量。

### 14.1 联动总览

```
  aihot（AI热点资讯查询）
  ├── 每日AI日报获取 → 选题参考
  ├── 精选条目搜索 → 素材收集
  └── 话题排行分析 → 热点判断
              ↓
  hv-analysis（横纵分析法）
  ├── 横向竞品分析 → 对比素材
  ├── 纵向历史梳理 → 行业洞察
  └── 输出万字PDF报告 → 深度研究素材
              ↓
  本SKILL（核心写作）
  ├── 基于aihot选题 + hv-analysis素材 → 生成初稿
  └── 四层自检 → 润色定稿
              ↓
  知识归档（如IMA知识库）
  └── 文章发布后 → 自动归档到知识库
```

### 14.2 aihot — AI热点资讯查询（选题来源）

**安装**：
```
openclaw skills install aihot
```
或手动：
```bash
curl -fsSL https://aihot.virxact.com/aihot-skill/install.sh | bash
```

**核心能力**：

| 能力 | 指令示例 | 在写作流程中的用途 |
|------|---------|------------------|
| 获取今日AI日报 | `用aihot获取今天的AI日报` | 了解当日行业动态，找选题灵感 |
| 搜索精选条目 | `用aihot搜索关于"MCP"的精选条目` | 为特定主题收集素材 |
| 获取话题排行 | `用aihot获取AI话题排行` | 判断当前热点，辅助HKR质检 |

### 14.3 hv-analysis — 横纵分析法（研究素材）

**安装**：
```
openclaw skills install hv-analysis
```

**核心能力**：

| 维度 | 说明 | 输出 | 在写作流程中的用途 |
|------|------|------|------------------|
| **横向分析** | 对产品/公司/概念进行竞品对比 | 结构化对比表格，万字级PDF | 对比测评型文章的直接素材 |
| **纵向分析** | 对同一对象的演变历史梳理 | 时间线+关键节点，万字级PDF | 技术解读型文章的历史背景 |

**写作流程整合**：
- 写对比测评型文章：先用hv-analysis做横向分析，产出对比表格
- 写技术解读型文章：先用hv-analysis做纵向分析，提取关键时间节点
- 写热点解读型文章：用hv-analysis做竞品格局分析，作为行业铺垫素材

### 14.4 知识归档（发布后管理）

完成文章后，建议将文章同步到IMA知识库等平台归档，便于后续检索和二次利用。

### 14.5 联动使用要点

1. **aihot是零成本启动项**：无需API Key，国内直链可用，建议每次写作前先用aihot扫一遍热点
2. **hv-analysis是素材加速器**：万字级PDF报告可以直接作为写作素材的"矿源"
3. **外部工具与本SKILL互补**：它们解决的是"写什么"和"用什么写"的问题，本SKILL解决的是"怎么写"的问题
