# Platform Rules & Evidence Standards

Mechanics differ by platform; the physics are constant: deadlines are absolute, evidence formats are prescribed, and reviewers spend seconds per case. Verify current policies in the seller center — rules change. Patterns below are the audit framework.

## 1. Marketplace dispute patterns (TikTok Shop, Shopee, Lazada, Amazon-style)

**What counts as proof of delivery:** carrier scan to the correct address is baseline; GPS-stamped or photo delivery confirmation is the gold standard; for high-value orders, signature. A scan to the wrong zip is not delivery proof — check before citing it.

**INR cases:** platforms increasingly auto-refund if the seller doesn't respond with tracking evidence inside the SLA (often 24-72h). Response template: tracking number + carrier + delivery timestamp + address-match statement + photo/GPS if available.

**INAD/SNAD cases:** the listing AS THE BUYER SAW IT is the contract. Screenshot listings at every revision (or use version history where the platform keeps it). Your defense is listing accuracy: specs, photos, and the claim that the delivered item matches them — serial/batch photos and outbound QC records carry this.

**Return-condition disputes:** most platforms allow refund deduction or rejection for used/damaged/incomplete returns IF documented: unboxing video of the return package (continuous, showing the shipping label), photos of condition, weight comparison. No video → it's your word against theirs → you lose.

**Seller metrics interaction:** every dispute outcome feeds account health. Sometimes refunding to protect the metric (e.g., during a promotion-eligibility window) is rational — note when this consideration changes the recommended strategy.

**Review extortion:** "refund or bad review" violates marketplace policies broadly; report through the dedicated channel BEFORE resolving the claim — the report timestamps the threat and typically neutralizes the review. Never reference the threat in the buyer-facing message.

## 2. Payment chargebacks (PayPal, Stripe, card networks)

Different game: the adjudicator is a bank, not the marketplace, and the doc is called representment.

- **Always respond.** Non-response is automatic loss AND your chargeback ratio counts it identically. 
- **Reason code first.** The chargeback reason code (fraud / product-not-received / not-as-described / subscription-cancel) dictates which evidence wins. Match the packet to the code — generic packets lose.
- **Winning packets:** AVS/CVV match + device/IP history (fraud codes); delivery confirmation to AVS-verified address (PNR codes); listing screenshots + correspondence showing the buyer acknowledged the product (INAD codes); cancellation-policy screenshot + usage logs (subscription codes).
- **Format:** banks process these as documents — one PDF, chronological, cover summary up top, labeled exhibits. Plain, dated, boring wins.
- **Ratios:** card networks penalize sellers above ~0.9-1% chargeback rates regardless of win rate. If ratio is the problem, prevention (clear descriptors, proactive refunds for likely-loss cases, fraud screening) beats representment heroics.
- **Double-dipping:** buyer files marketplace claim AND chargeback — flag it in both venues; platforms have procedures and it discredits the claim.

## 3. Evidence quality standards (all venues)

| Evidence | Standard that makes it usable |
|---|---|
| Tracking | Number + carrier + event timestamps; screenshot from carrier site, not your dashboard |
| Delivery photo/GPS | From carrier record with coordinates/address visible |
| Outbound QC | Photo/video tied to order number, timestamped before shipment |
| Return unboxing | Continuous video: label visible → opening → contents; no cuts |
| Listing state | Dated screenshot incl. price, variant, claims as displayed |
| Chat logs | Platform-native screenshots with names/dates visible; never edited excerpts |
| Weights | Carrier intake weight vs. product spec weight (catches empty-box and swap claims) |

Chain rule: every artifact must connect to THIS order (order number, label, serial). Generic product photos prove nothing.

## 4. Deadline discipline

At triage, record: platform SLA for first response, evidence-submission window, appeal window, and chargeback representment due date (typically 7-21 days from notice, earlier in the processor dashboard). Work backward; submit at 80% of the window, never the last hour (uploads fail, time zones bite). A calendar entry per dispute is not optional — missed windows are the #1 self-inflicted loss.

## 5. Appeals

Appeal only with: (a) new evidence not in the original submission, (b) demonstrable misapplication of the platform's own policy, or (c) procedural failure (evidence not reviewed). Re-arguing the same facts louder fails and consumes the one appeal most platforms allow. Appeal text: what's new, in the first line.
