## 一、工程智慧纲领（11条通用原则）

### 1️⃣ 三棱镜原理
任何复杂系统的问题诊断，只需**三个视角**：激励（源）→ 传递（路径）→ 响应（表现）。
- **遇到问题第一问**：它属于三个视角中的哪一个？
- **优先法则**：Always work on the source first!
- **跨域示例**：电机NVH（电磁激励→结构路径→人耳感知）；汽车异响（路面激励→悬架/车身→乘员感知）

### 2️⃣ 症状与根因分离原则
> "噪声不是问题，噪声是症状。问题藏在产生噪声的地方。"
- 不要问"怎么做消音"——要问"为什么会产生这个声音？"

### 3️⃣ 80/20分配算法
在任何资源有限的问题中，找到20%的"关键少数"投入80%的精力。
- **诊断**：先查概率最高的根因
- **设计**：先做核心20%功能
- **决策**：如果有10件待办，先问"哪2件做完能让剩下8件失去优先级？"

### 4️⃣ 快速失败原则
错误的代价与发现错误的时间成正比。第一个目标不是"做出完美产品"，而是"用最小成本验证最大的风险假设"。

### 5️⃣ 闭环自检原则
任何单一方向的优化都会在其他方向产生副作用，只有闭环反馈才能发现。优化的衡量指标至少3个：效率 + 质量 + 副作用。

### 6️⃣ 三环递进决策原则
```
外环（战略）→ 回答"该做什么" — 80/20分析
  ↓
中环（验证）→ 回答"做对的事" — CBI创新流程
  ↓
内环（执行）→ 回答"把事做对" — 工程设计
```
决策前先问：我在哪个环？不要从外环直接跳到内环。

### 7️⃣ 跨域连接密度 = 系统成熟度
真正的高手不是知道最多的人，而是连接最多的人。

### 8️⃣ 系统级最优原则
**任何工程优化的目标不是单一组件的性能最优，而是整个系统的综合最优。**

### 9️⃣ 救火与布局双模式
| 模式 | 场景 | 节奏 |
|:-----|:-----|:-----|
| 救火模式 | 产线危机/批量失效 | 48小时出临时方案 |
| 布局模式 | 新项目规划/前瞻研发 | 2-4周完成评估 |

收到问题先判断：这是救火还是布局？

### 🔟 先问再答
```
第一问：应用场景是什么？（客户/环境/工况）
第二问：关键参数有哪些？（负载/空间/性能/成本）
第三问：约束条件是什么？（法规/安全/材料/时间）
```

### 1️⃣1️⃣ 知懂分离
| 层级 | 特征 |
|:-----|:------|
| **知** | 书本能查到，能复述标准答案 |
| **懂** | 实践验证过，能解决实际问题 |
| **悟** | 打破常规创新，能创造新方法 |

## 二、跨域洞察图谱（12条）

### 两域交叉（6个）
1. **电机+NVH** → 电机特征频率与结构固有频率差值在50Hz以内必然产生可感知噪声
2. **齿轮+NVH** → 所有齿轮噪声问题本质归结为传递误差TE
3. **电机+齿轮** → 电磁转矩波动与齿轮啮合冲击存在固定相位关系
4. **CBI+NVH** → 声学目标应从"客户可感知的差异"反推，不是从国标
5. **FTB+工程** → 80/20是通用资源分配算法，跨领域适用
6. **CBI+FTB** → 创新与运营是螺旋上升的闭环

### 三域交叉（3个）
7. **全链路能量传递模型**（电机+齿轮+NVH）：电→磁→机械→声，问题定位 = f(源频率, 路径特性, 响应阈值)
8. **三环递进决策模型**（FTB+CBI+工程）：外环→中环→内环，不可跳级
9. **最小化浪费的三种策略**（80/20+快速失败+修形）：核心是"把资源从低价值耗散中释放出来"

### 涌现原则（3条）
A. 三棱镜原理（见上）
B. 噪声不是问题，噪声是症状
C. 系统成熟度=跨域连接密度

## 四、Eric个人经验层

### 9个核心项目案例速查

| 项目 | 问题 | 根因 | ⭐教科书上没有的 |
|:-----|:-----|:-----|:----------------|
| BMW IBK2 5379Hz | 高频刺耳音 | 机壳5300Hz模态与电机5379Hz共振 | 高频刺耳音根因在电磁参数与结构耦合，碳刷改变比隔振更有效 |
| BMW蛐蛐声 | 大批量异响 | 蜗杆刀纹频率=3517Hz | 刀纹噪声易误诊为齿轮啮合，区分：刀纹频率=转频×刀纹数 |
| 路虎XVS3循环音 | 0~10Hz周期波动 | 马达特征频率被蜗轮频率调制 | 循环音先查电气参数再查机械，成本路径正好反过来 |
| Mazda J72Y Buzzing | 800Hz内低频 | 软轴装配过松 | Buzzing 80%是软轴问题，不要上来就拆电机 |
| 通用GEM响度 | 3.33sone>目标2.5 | 软轴过松+齿轮箱啮合不良双重源 | 响度超标拆解到多个源，按成本排序处理 |
| Mazda GEN7敲击 | 9.13Hz规律敲击 | 盖板孔径过大→行星架晃动 | 行星齿轮敲击频率=行星架转频，低到常人会忽视 |
| BMW 223Hz | 苍蝇叫异音 | 衬套打滑 | 223Hz落人耳最敏感范围(200-500Hz)，幅度不高也优先处理 |
| BMW 3.5Hz晃动 | 高调晃动 | 齿轮箱运转频率与结构耦合 | 低频振动(<10Hz)优先查ISO 2631，不是查噪声 |
| MQB 524.1Hz超标 | 1/3倍频程超标 | 电机525Hz激励支架共振 | 共振差值<1Hz，加橡胶隔振最快最便宜 |

### 7条工艺窍门
1. 任何电机噪声：先空载听→再加载听→比较差异
2. 低频Buzzing：先目视检查软轴装配是否过松（80%是软轴问题）
3. 有刷电机哒哒声：先拆碳刷，看端面磨损状态
4. 齿轮噪声初诊：用手机录10秒音频，数每秒"哒"几下估算频率
5. 3D图谱分析：Ctrl+Shift+↑↓调到5-20Hz带宽
6. 响度超标：先锁定额外的贡献频带再动手
7. 人体感知振动：优先做速度一次积分，不要只看加速度

### 9条决策准则（IF-THEN）
```
IF 电机异响 AND 空载出现 THEN 优先查碳刷/电刷
IF 电机异响 AND 加载出现 THEN 优先查齿轮/负载
IF 低频Buzzing(<800Hz) THEN 先查软轴装配
IF 循环音(0-10Hz波动) THEN 先优化电气参数再改机械
IF 高频刺耳音(>3000Hz) THEN 查碳刷参数+电磁力波+机壳模态
IF 行星齿轮敲击 THEN 查行星架与盖板配合间隙
IF 电机共振 THEN 先试橡胶隔振垫
IF VDI评级<7 THEN 必须处理
IF VDI评级≥7 THEN 与客户Workshop确认是否接受
```

## 五、电子五层诊断模型

```
1️⃣ 感知层：传感器输出是否正常（电压/电流/信号质量）
2️⃣ 信号调理层：放大/滤波后信号质量（示波器/频谱分析）
3️⃣ 控制核心层：时序图、输入输出、定时器、联锁逻辑
4️⃣ 执行层：驱动电路电压/电流、继电器/接触器
5️⃣ 反馈层：阶跃信号响应（超调/稳态误差/响应时间）
```

### 电磁噪声 vs 机械噪声区分
1. **断电测试**：断电后噪声是否立即消失 → 电磁噪声 vs 机械惯性
2. **变电压测试**：改变电压看频率是否变化 → 电磁噪声（跟转速）vs 共振（固定频率）
3. **示波器+频谱仪联用**：观察电流谐波和噪声频谱对应关系
