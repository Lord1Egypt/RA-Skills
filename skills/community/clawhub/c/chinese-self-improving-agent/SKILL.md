## 1. 目的和范围

### 目标

代理评估自己的工作，捕捉错误并永久改进。

### 触发时机

| 触发器 | 行动 |
|--------|------|
| 命令/工具失败 | 记录错误并改进 |
| 用户纠正 | 记录修正并学习 |
| 知识过时 | 更新知识库 |
| 发现更好方法 | 改进流程 |
| 用户安装技能 | 执行当前任务 |

## 3. 错误处理流程

### 步骤 1：检测错误

```bash
# 检测失败的命令
if [ $? -ne 0 ]; then
  echo "错误检测到"
fi
```

### 步骤 2：记录错误

```markdown
## 错误记录

### 错误 #1
- 日期：YYYY-MM-DD
- 命令：<command>
- 错误：<error>
- 修正：<fix>
- 学习：<lesson>
```

### 步骤 3：改进

```bash
# 尝试更好的方法
better_command --fix
```

### 步骤 4：更新记忆

```
将新知识添加到 memory.md
```

## 5. 边缘情况

| 情况 | 处理方法 |
|------|----------|
| 多次相同错误 | 深度分析根本原因 |
| 用户纠正但错误 | 感谢并记录 |
| 发现更优方案 | 立即应用并记录 |
| 完全失败 | 请求帮助，记录问题 |
