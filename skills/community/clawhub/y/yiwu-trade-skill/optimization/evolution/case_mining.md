## 二、成功案例提取规则

### 2.1 成功案例定义

```
成功案例标准：

1. 结果成功
   ├── 订单成交（金额≥$1000）
   ├── 问题解决（客户投诉处理）
   └── 目标达成（选品成功/开发成功）

2. 过程有价值
   ├── 有可借鉴的方法
   ├── 有可复制的经验
   └── 有独特的洞察

3. 信息可获取
   ├── 用户授权分享
   ├── 信息脱敏可行
   └── 细节可还原
```

### 2.2 提取触发条件

| 触发类型 | 触发条件 | 提取方式 |
|----------|----------|----------|
| 行为触发 | 用户完成关键动作 | 自动记录 |
| 结果触发 | 达成特定目标 | 自动标记 |
| 反馈触发 | 用户表达满意 | 主动邀请 |
| 人工触发 | 运营人员识别 | 主动提取 |

### 2.3 自动提取机制

```
┌─────────────────────────────────────────────────────────────────┐
│                    成功案例自动提取机制                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  信号识别                                                       │
│  ├── 用户完成关键动作（成交/解决问题等）                         │

│  └── 业务数据达标（转化率/满意度等）                            │
│      ↓                                                          │

│  ├── 生成案例摘要                                               │

│  人工确认                                                       │

## 三、失败案例归因分析

### 3.1 失败案例定义

```
失败案例标准：

1. 结果失败
   ├── 订单流失
   ├── 问题未解决
   └── 目标未达成

2. 有分析价值
   ├── 失败原因可分析
   ├── 有防控措施可借鉴
   └── 教训可避免重复

3. 信息可获取
   ├── 用户授权分享
   ├── 信息脱敏可行
   └── 细节可还原
```

### 3.2 失败分类体系

```
┌─────────────────────────────────────────────────────────────────┐
│                      失败案例分类体系                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│                        按原因分类                                 │
│         ┌──────────┬──────────┬──────────┬──────────┐           │
│         │ 能力不足 │ 判断失误 │ 执行偏差 │ 外部因素 │           │
│         ├──────────┼──────────┼──────────┼──────────┤           │
│         │专业知识 │ 市场判断 │ 操作失误 │ 政策变化 │           │
│         │沟通技巧 │ 客户判断 │ 服务失误 │ 汇率波动 │           │
│         │谈判技巧 │ 趋势预测 │ 跟进失误 │ 物流延误 │           │
│         └──────────┴──────────┴──────────┴──────────┘           │
│                                                                  │
│                        按阶段分类                                 │
│         ┌──────────┬──────────┬──────────┬──────────┐           │

### 3.3 归因分析方法

```
归因分析框架：

1. 多因素分解
   ├── 内部因素（可控）
   │   ├── 能力因素（知识、技能）
   │   ├── 态度因素（重视度、执行力）
   │   └── 方法因素（策略、流程）
   │
   └── 外部因素（不可控）
       ├── 市场因素（竞争、需求）
       ├── 客户因素（预算、决策）
       └── 环境因素（政策、汇率）

2. 归因判定
   ├── 主要归因：最主要失败原因
   ├── 次要归因：辅助失败原因
   └── 直接/间接：直接原因 vs 根本原因

3. 责任划分
   ├── 我方责任：可以改进的部分
   ├── 客户责任：需要客户配合
   └── 外部责任：不可控因素

4. 教训提取
   ├── 认知教训：对什么有了新认识
   ├── 方法教训：以后应该怎么做
   └── 机制教训：需要建立什么机制
```

### 3.4 归因分析模板

```
失败案例归因分析模板：

【案例概况】
- 案例ID：XXX
- 案例类型：订单流失
- 发生时间：XXX
- 涉及金额：$10,000

【结果描述】
- 预期结果：客户下单
- 实际结果：客户取消询盘
- 损失评估：直接损失$0，机会损失$10,000

【归因分析】
- 主要归因：价格谈判失败
  - 具体表现：客户要求降价15%，拒绝后客户流失
  - 根因分析：未能有效传递价值，过度依赖价格竞争
  - 我方责任：70%
  
- 次要归因：跟进时机不当
  - 具体表现：跟进频率过高，客户感到压力
  - 根因分析：未能准确判断客户意向阶段
  - 我方责任：30%

【教训提取】
- 认知教训：
  - 客户询价≠意向客户
  - 价格不是唯一的竞争维度
  
- 方法教训：
  - 谈判前先确认客户核心需求
  - 提供价值而非降低价格
  - 判断客户意向后再决定跟进频率

- 机制教训：
  - 建立客户意向评估机制
  - 完善价格谈判话术库
```

## 四、案例标准化格式

### 4.1 案例标准格式

```
案例标准格式（Template）：

┌─────────────────────────────────────────────────────────────────┐
│                        案例文档模板                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  【案例标题】                                                    │
│  一句话概括核心故事，如："$50万大单谈判：如何用价值而非价格赢单"  │
│                                                                  │
│  【案例概览】                                                    │
│  ├── 案例类型：成交案例/跟进案例/谈判案例/风控案例               │
│  ├── 涉及金额：$50,000                                        │
│  ├── 时间周期：2个月                                           │
│  ├── 结果：成功成交                                           │
│  └── 亮点标签：#大单谈判 #价值传递 #长期跟进                    │
│                                                                  │
│  【背景】                                                        │
│  交代案例发生的情境：                                           │
│  - 客户背景：谁、什么公司、采购什么产品、规模如何               │
│  - 市场背景：什么市场、什么时机、竞争情况                       │
│  - 挑战背景：遇到什么困难或挑战                                 │
│                                                                  │
│  【过程】                                                        │
│  按时间线描述关键步骤：                                         │
│  Step 1：...（时间、动作、关键决策点）                          │
│  Step 2：...（时间、动作、关键决策点）                          │
│  Step 3：...（时间、动作、关键决策点）                          │
│                                                                  │
│  【关键转折】                                                    │
│  描述案例中最关键的转折点：                                     │
│  - 转折1：... → 结果...                                        │
│  - 转折2：... → 结果...                                        │
│                                                                  │
│  【成功要素】                                                    │
│  总结案例成功的关键因素：                                       │
│  1. ...                                                        │
│  2. ...                                                        │
│  3. ...                                                        │
│                                                                  │
│  【可复制经验】                                                  │
│  提炼可复制的经验和方法：                                       │
│  - 方法1：具体怎么做                                           │
│  - 方法2：具体怎么做                                           │
│  - 方法3：具体怎么做                                           │
│                                                                  │
│  【注意事项】                                                    │
│  提醒可能的风险和陷阱：                                         │
│  - 风险1：...                                                  │
│  - 风险2：...                                                  │
│                                                                  │
│  【适用场景】                                                    │
│  明确案例适用的情境：                                          │
│  - 适用：...                                                   │
│  - 不适用：...                                                 │
│                                                                  │
│  【延伸阅读】                                                    │
│  相关案例、知识点、工具推荐                                     │
│                                                                  │

```

## 七、质量保障

### 7.1 质量标准

| 维度 | 标准 | 检查项 |
|------|------|--------|
| 准确性 | 内容真实可靠 | 事实核对、数据验证 |
| 实用性 | 可操作可复制 | 步骤清晰、方法可行 |
| 可读性 | 易于阅读理解 | 结构清晰、语言流畅 |
| 时效性 | 内容保持新鲜 | 定期复审、及时更新 |
| 隐私性 | 信息脱敏合规 | 脱敏处理、授权确认 |

### 7.2 质量审核

```
审核流程：

提交 → AI预审 → 人工审核 → 质量评级 → 入库/打回
  ↓         ↓          ↓          ↓
内容检查  自动检测  专家评审  ABC评级

质量评级：
├── A级：可直接使用，优秀案例
├── B级：可用，有小问题需修改
├── C级：需大改，内容价值一般
└── D级：不入库，无参考价值
```
