## [0.1.0] - 2026-05-25
Initial release. Seven-phase workflow aligned to the AIAG-VDA FMEA Handbook seven-step PFMEA approach: Phase 1 planning and preparation (project name, customer, regulatory frame, IATF 16949 / AS9100 / 21 CFR 820 / customer-specific applicability, cross-functional team roster, scope, prior PFMEA / DFMEA / control plan / process-flow inputs); Phase 2 process-structure analysis (process item → process step → process work element decomposition with 4M categorisation of Man / Machine / Material / Method / Measurement / Environment); Phase 3 function analysis (process-item function, process-step function and product characteristic, process-work-element function and process characteristic — three-tier function tree); Phase 4 failure analysis (failure effects to next-level process / plant / end customer, failure mode of the process step, failure cause from the process work element — failure chain populated row-by-row); Phase 5 risk analysis (Severity 1–10 of effect at the end customer, Occurrence 1–10 of cause, Detection 1–10 of current detection control with current prevention control and current detection control identified separately, Action Priority assigned per the AIAG-VDA AP table — High / Medium / Low — replacing RPN); Phase 6 optimisation (recommended preventive and detective actions with single named owner and target completion date, re-evaluation of S / O / D and AP after action effectiveness verification); Phase 7 results documentation (DRAFT PFMEA worksheet, Top-N High-AP action list, optimisation action plan, reanalysis-trigger list, and a cross-functional-team review and sign-off block) — for quality / manufacturing / supplier-quality engineering review before any control-plan, PPAP, or customer Source-Inspection submission. Never finalises a PFMEA, never signs PPAP, never overrides the cross-functional team's judgement, and never substitutes for the customer's PFMEA-format requirements.
