# 路径决策（A / B / 拒绝）

## A 路径（新建 Wiki）

满足其一即可：
- `related_wiki` 为空或目标不存在
- 目标 Wiki 存在但章节命中极低，硬塞会导致主题边界模糊

## B 路径（补充现有 Wiki）

满足全部：
- 有明确目标 Wiki
- 新内容属于该 Wiki 范围
- 扫描命中关键章节或可自然落到现有结构

## 拒绝入库

满足其一：
- 与现有内容重叠率极高且无新增
- 信息密度低，无法形成可复用知识

## 必答三问（防误入）

1. 这个内容放进目标 Wiki 会不会让主题边界变糊？
2. 如果今天新建词条，是否比补充更清晰？
3. 有无历史矛盾或冲突结论需要先标注？

## 决策记录

> 决策日志格式见 workflow.md Step 3，写入当天日志 `## 决策` 板块下；不等待确认。

## checkpoint 约定

路径决策完成后固定写：

```bash
bash {baseDir}/scripts/wiki_entry_step_checkpoint.sh \
  --step 3 --status done --note "路径已决策，已写入决策日志"
```
