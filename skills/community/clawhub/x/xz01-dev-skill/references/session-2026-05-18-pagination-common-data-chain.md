# Session note: pagination chain requires common_data include

## Trigger

User reported that real list pages still had no bottom pagination:

- `https://www.900az.com/zoszx/`
- `https://www.900az.com/gzos/`

and asked to compare with `xz01_demo` to see what was missing.

## Root cause

The templates had a bottom pagination output location:

```html
<ul class="pagination">
    {$page_code|raw}
</ul>
```

but the rendered HTML was empty:

```html
<ul class="pagination">
</ul>
```

The missing piece was not CSS and not the bottom output tag. The current theme was missing the demo data-preparation chain:

```html
{include file="./themes/default/common_data/_list_news_common.html"/}
{include file="./themes/default/common_data/_list_game_common.html"/}
```

The affected templates were using `getAllData(...)` for the main list. `getAllData()` returns data but does not create the global/template `$page_code` value, so `{$page_code|raw}` rendered empty.

## Correct demo chain

For list pages, `xz01_demo` uses this chain:

1. The list template includes the corresponding `common_data/_list_*_common.html` file.
2. The common data file calls `getPcPageData(...)` or `getMobilePageData(...)` depending on device.
3. Those helpers generate `$page_code = $list->render()`.
4. The template outputs `{$page_code|raw}` at the bottom.

For game list pages, demo also saves/restores `$page_code` around secondary data fetches to avoid another pagination query overwriting the main list pager.

## Durable rule

When xz01 pagination is missing, do not stop at checking whether `{$page_code|raw}` exists. Verify the full chain:

```text
list template
  -> includes matching common_data/_list_*_common.html
  -> common_data calls getPcPageData/getMobilePageData for the main list
  -> $page_code is generated by the helper
  -> template outputs {$page_code|raw}
```

If the page has `<ul class="pagination"></ul>`, treat it as a missing upstream pagination data-chain issue, not a CSS issue and not a reason to invent a custom pager.

## Validation

A PASS requires:

- the target list template includes the relevant `common_data/_list_*_common.html` file;
- the common data file calls `getPcPageData` / `getMobilePageData` for the main list;
- rendered PC pages contain non-empty numeric pagination;
- rendered mobile pages contain non-empty previous/next pagination;
- page 2 links return HTTP 200 and mark/navigate page 2 correctly;
- no custom request-param/page-count/pager_i/ellipsis/disabled-prev-next pager algorithm is introduced.
