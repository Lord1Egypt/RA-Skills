# OPC领域要求清单

一人公司按任务类型定义的组件槽位——每个任务的"零件清单"。按清单逐项产出，确保不遗漏。

---

## P0 工作流重构

### P0-01 传统工作流识别
- **必选组件**: 目标领域名称、传统工作流全链路描述（环节→角色→文档→协作点）、环节数、参与角色数、中间文档数、协作节点数
- **可选组件**: 各环节耗时占比、传递损耗率、返工率
- **组装顺序**: 领域确认→全链路梳理→环节标注→角色标注→文档标注→协作点标注→数据汇总
- **约束**: 必须完整列出传统工作流的每个环节，不可跳过"理所当然"的步骤；量化数据缺失时标注"待测"
- **格式**: 工作流全景图（Markdown表格+流程标注）

### P0-02 环节存在理由分析
- **必选组件**: 每个环节的存在理由（事情本身需要 / 人的局限需要）、环节类型标记（✅核心 / 🔶校准 / ❌传递 / ❌协调 / ⚡校验 / ❌格式）、标记理由
- **可选组件**: 人的局限类型细分（脑容量/传递/协作/记忆/出错）、可消除程度评估、校准价值评估（该环节的中间产出物是否起纠偏作用）
- **组装顺序**: 逐环节追问→存在理由判定→类型标记→校准价值评估→理由记录→汇总统计
- **约束**: 每个环节必须追问"如果执行者是一个拥有无限知识能力和零协作损耗的AI，这个环节还需要吗？"；判定必须基于事情本身逻辑，不可因"行业惯例"保留；对中间产出物必须追问"这个产出物是否在过程中起校准作用？"——起校准作用的标记为🔶校准
- **格式**: 环节分析表（Markdown表格）

### P0-03 人的局限补偿层消除
- **必选组件**: 消除清单（哪些环节消除、为什么可消除）、保留清单（✅核心环节+🔶校准环节+⚡关键校验节点）、消除后的信息传递方式（IPO链自动传递）
- **可选组件**: 每个消除环节的风险评估、消除后需新增的关键校验点
- **组装顺序**: ❌标记环节逐一评估→消除决策→🔶校准环节确认→信息传递替代方案→保留环节确认→关键校验点插入→消除清单+保留清单
- **约束**: 消除传递/协调/格式环节不可犹豫——这些是人的局限产物不是事情本身需要；校验环节精简为关键节点但涉及合规的不可消除；**校准环节不可消除**——中间产出物起纠偏作用的必须保留为基元内分步校准点
- **格式**: 消除决策表（Markdown表格）

### P0-04 重整为IPO基元链
- **必选组件**: 重构后的工作流形态（单步IPO/简短基元链/IPO+人工决策）、每个基元的I/P/O定义、基元间传递关系、基元内分步校准点（如有）、总环节数（目标≤传统工作流的30%）
- **可选组件**: 每个基元的能力需求标注、AI自动化程度评估、关键人工决策节点说明
- **组装顺序**: 保留环节排序→IPO基元划分→基元内分步校准点插入→基元I/P/O定义→传递关系确认→形态选择→能力需求标注
- **约束**: 基元数≤5（超过5说明还没充分消除）；每个基元必须可由AI辅助一人完成；合规节点必须有人工介入；🔶校准环节作为基元内分步校准点保留，不是独立基元
- **格式**: IPO基元链图（Markdown流程图+表格）

### P0-05 重构验证
- **必选组件**: 七项验证结果（事情完整性/补偿层消除/校准不丢失/端到端可执行/复杂度回归/质量守恒/合规不跳过）、验证不通过项的修正方案
- **可选组件**: 重构前后对比数据（环节数/中间文档/参与角色/耗时/传递损耗）、风险残留项
- **组装顺序**: 逐项验证→记录结果→不通过项修正→二次验证→通过
- **约束**: 七项必须全部通过；"质量守恒"不可妥协——重构是简化流程不是降低质量；"合规不跳过"是硬约束；"校准不丢失"防止过度合并
- **格式**: 验证清单（Markdown检查表）

### P0-06 执行形态选择
- **必选组件**: 选定执行形态及理由、执行方式说明、关键人工决策节点（如有）
- **可选组件**: 形态切换条件、后续迭代方向
- **组装顺序**: 重构结果评估→三种形态适配判断→选定→理由记录
- **约束**: 选择最简形态——能单步IPO的不用基元链，能基元链的不加人工决策；人工决策节点必须标注具体决策内容和触发条件
- **格式**: 形态选择记录（Markdown）

## P1 商业定位

### P1-01 垂直利基定位
- **必选组件**: 目标客户画像（具体到行业+城市+规模）、核心痛点（1个）、解决的具体成果（可量化的商业成果，不是"导入AI"）
- **可选组件**: 次级客户群、痛点优先级排序、客户支付能力评估
- **组装顺序**: 目标客户画像→核心痛点→具体成果→垂直利基一句话
- **定位约束**: 禁止出现"所有人"/"所有行业"/"AI技术"等泛化表述；必须窄到一句话说清
- **格式**: 一页纸（Markdown）

### P1-02 市场扫描与竞品分析
- **必选组件**: 目标市场规模（TAM/SAM/SOM）、3-5个直接竞品、竞品定价区间、行业趋势判断
- **可选组件**: 替代方案分析、政策/监管趋势、技术演进路线
- **组装顺序**: 市场规模→竞品列表→竞品分析→趋势判断→机会窗口
- **时间约束**: 调研不超过3天，超时强制进入下一任务
- **格式**: 竞品对比表+趋势总结（Markdown）

### P1-03 差异化价值主张
- **必选组件**: 与竞品的核心差异（1-3点）、差异化优势的证据（案例/数据/逻辑）、客户选择你的理由
- **可选组件**: 护城河评估、差异化可持续性分析
- **组装顺序**: 竞品弱点→自身优势→差异点→证据→价值主张声明
- **定位约束**: 差异必须与客户痛点直接相关，不是"我们用更好的AI"
- **格式**: 价值主张画布（Markdown表格）

### P1-04 个人能力盘点
- **必选组件**: 专业领域知识(A1)、技术能力(A2)、人脉资源(A3)、个人品牌(A4)、行业经验年限(A5)的自我评估、能力-工具交叉矩阵（5×5）
- **可选组件**: 能力差距分析、补强计划、AI工具使用熟练度
- **组装顺序**: A1-A5自评→AI工具B1-B5评估→交叉矩阵→优势区识别→OPC可能性发现
- **约束**: 自评必须诚实，不可高估；识别出的OPC可能性须通过P2-01模式选择验证
- **格式**: 能力评估表+维度矩阵（Markdown表格）

## P2 商业模式

### P2-01 模式选择
- **必选组件**: 四大模式适配评估（AI原生机构/AI顾问/AI数字产品/AI SaaS）、选择理由、预期毛利率、启动难度自评
- **可选组件**: 混合模式设计、模式切换时机
- **组装顺序**: 个人能力盘点→四模式逐个评估→匹配度打分→选定模式→切换条件
- **约束**: 必须基于P1-04的个人能力盘点做匹配，不可脱离实际；初次创业优先选启动难度低的模式
- **格式**: 模式对比表+选择结论（Markdown表格）

### P2-02 服务包/产品定义
- **必选组件**: 服务/产品名称、解决的具体商业成果、交付物清单、交付周期、收费模式（建设费+月维护费/一次性/订阅）
- **可选组件**: 增值服务、阶梯版本（基础/专业/企业）
- **组装顺序**: 商业成果→服务内容→交付物→交付周期→收费模式→服务包命名
- **定位约束**: 服务名称和描述禁止出现"AI"二字（卖成果不卖技术）；交付物必须具体可见
- **格式**: 服务方案一页纸（Markdown）

### P2-03 定价策略设计
- **必选组件**: 成本计算（AI工具+时间+外包）、竞品定价区间、价值锚定（客户因此获得的商业价值）、最终定价
- **可选组件**: 阶梯定价、促销策略、涨价路线图
- **组装顺序**: 成本底线→竞品定价→价值锚定→定价方案→验证（3个潜在客户反馈）
- **约束**: 定价必须高于成本×3（OPC覆盖波动）；MRR目标≥3×月固定成本
- **格式**: 定价表（Markdown表格）

### P2-04 收入结构规划
- **必选组件**: MRR/ARR目标（3月/6月/12月）、break-even月份、runway计算（月储备/月净支出）、收入来源占比
- **可选组件**: 情景分析（乐观/基准/悲观）、收入增长模型
- **组装顺序**: 月固定成本→MRR目标→break-even→runway→收入结构→增长路径
- **约束**: runway安全线≥6个月（OPC建议12个月）；break-even超过12个月需重新评估模式
- **格式**: 财务预测表（Markdown表格+关键公式）

## P3 产品与交付

### P3-01 MVP最小方案设计
- **必选组件**: 核心功能/服务（仅1个核心）、砍掉清单（砍了什么、为什么砍）、最小交付标准、验证指标（什么算成功）
- **可选组件**: V2.0规划、技术选型、AI辅助方案
- **组装顺序**: 全功能清单→逐项砍→剩核心功能→定义最小交付→设定验证指标
- **约束**: MVP交付周期≤2周；功能砍到只剩1个核心价值点；验证指标必须可量化
- **格式**: MVP一页纸（Markdown）

### P3-02 服务流程标准化
- **必选组件**: 交付SOP（步骤→输入→输出→负责人/AI）、质量控制点、客户沟通节点
- **可选组件**: 交付模板库、异常处理流程
- **组装顺序**: 完整交付经验→步骤拆解→质量控制点插入→标准化模板→SOP文档
- **约束**: 只有跑通3次以上的流程才能标准化；SOP必须含异常处理
- **格式**: SOP文档（Markdown流程图+表格）

### P3-03 产品化路径规划
- **必选组件**: 当前状态（人工服务流程）、目标状态（产品化/SaaS化）、迁移步骤（3-5步）、每步的验证指标
- **可选组件**: 自动化点位识别、技术栈选择、时间估算
- **组装顺序**: 当前服务流程→自动化点位→产品化拆解→迁移步骤→验证指标
- **约束**: 迁移路径必须按Dan Martell路径：服务→标准化→产品化→SaaS化，不可跳步
- **格式**: 迁移路线图（Markdown时间线）

### P3-04 AI辅助交付方案
- **必选组件**: 交付流程是否已通过P0判定（前置条件：如需重构则必须完成P0-04重整为IPO基元链后再进入；如无需重构则跳过P0直接进入）、重构后的工作流中AI可自动化的环节、AI自治度评估（⬛全自动/🟨半自动/⬜辅助）、人机协作接口
- **可选组件**: AI工具选型、成本估算、备选方案（AI失败时的人工兜底）
- **组装顺序**: P0判定结果确认（如需重构则基于P0-04重构结果；如无需重构则基于原始工作流）→流程拆解→逐环节评估AI可行性→自治度标注→人机接口定义→兜底方案
- **约束**: 必须先通过P0判定再评估AI自动化——如需重构则在旧流程上做AI化只是在低效基础上自动化低效，必须先完成P0-04重整后再进入；任何⬛全自动环节必须有⬜人工兜底；合规相关环节不可全自动
- **格式**: AI辅助方案表（Markdown表格）

## P4 获客与品牌

### P4-01 内容获客体系设计
- **必选组件**: 目标客户聚集的渠道（2-3个）、内容类型与频率、AI内容生产流程、转化路径（曝光→信任→试用→付费）
- **可选组件**: SEO策略、社区运营、转介绍机制
- **组装顺序**: 渠道选择→内容类型→生产流程（AI自动化）→分发流程→转化路径→数据监测
- **约束**: 内容必须围绕垂直利基定位，不做泛内容；内容生产优先AI自动化，人审核即可
- **格式**: 获客飞轮设计（Markdown流程图）

### P4-02 冷启动获客方案
- **必选组件**: 首批3个付费客户的具体目标名单、联系方式与触达方式、价值主张话术
- **可选组件**: 免费/低价案例交换方案、推荐人策略
- **组装顺序**: 目标名单→触达方式→话术准备→首次沟通→成交→案例积累
- **约束**: 必须找到3个愿意付费的潜在客户才能进入下一阶段；免费案例不超过3个
- **格式**: 冷启动行动计划（Markdown清单）

### P4-03 个人IP与品牌策略
- **必选组件**: 个人品牌定位（行业专家定位）、内容输出计划、信任建设路径（案例/数据/背书）
- **可选组件**: 媒体曝光计划、行业活动参与、合作伙伴策略
- **组装顺序**: 品牌定位→内容输出→案例积累→信任建设→品牌迭代
- **约束**: 品牌定位必须与垂直利基一致；品牌是长期资产，不可短视
- **格式**: 品牌策略一页纸（Markdown）

### P4-04 客户转化与留存
- **必选组件**: 转化漏斗（曝光→线索→成交）、留存机制（复购/续费/推荐）、LTV/CAC目标
- **可选组件**: 客户分层策略、流失预警指标
- **组装顺序**: 漏斗设计→转化率基准→留存机制→LTV/CAC计算→优化循环
- **约束**: LTV/CAC≥3才可持续；月流失率≤5%为健康
- **格式**: 转化漏斗表（Markdown表格）

## P5 运营与自动化

### P5-01 AI工具栈选型

### P5-02 Agent工作流搭建
- **必选组件**: 目标流程是否已通过P0判定（前置条件：如需重构则必须完成P0-04重整为IPO基元链后再进入；如无需重构则跳过P0直接进入）、Agent功能定义、触发条件、输入/输出、人机协作接口、异常处理
- **可选组件**: 多Agent协作设计、性能监控指标

### P5-03 日常运营SOP

- **约束**: 日检5分钟内可完成；周检30分钟内；月检2小时内

### P5-04 系统维护与迭代

## P6 财务与合规

### P6-01 主体选择与注册
- **必选组件**: 主体类型选择（个体工商户/一人有限公司/合伙）、注册流程、年成本对比
- **可选组件**: 注册地选择、经营范围设计、银行开户
- **组装顺序**: 需求分析→主体对比→选择→注册→银行开户→税务登记
- **合规约束**: 无3个稳定付费客户前建议个体工商户；法人财产与个人财产必须严格分离
- **格式**: 主体对比表+注册清单（Markdown）

### P6-02 现金流管理
- **必选组件**: 月度收支记录、runway计算、安全线（6-12个月）、预警机制
- **可选组件**: 情景分析、现金流优化方案
- **组装顺序**: 收入记录→支出分类→runway计算→安全线判断→预警→优化
- **约束**: runway<3个月=红色预警（必须立即行动）；3-6个月=黄色预警
- **格式**: 现金流表（Markdown表格+关键公式）

### P6-03 税务规划
- **必选组件**: 主体对应的税种与税率、申报周期与方式、合规节税项（研发费用抵扣等）
- **可选组件**: 税收优惠政策、跨期策略
- **组装顺序**: 税种识别→税率计算→申报安排→节税项→合规验证
- **合规约束**: 不得违规节税；关键决策建议咨询专业会计师（标注"不构成税务意见"）
- **格式**: 税务规划表（Markdown表格）

### P6-04 合同与法律
- **必选组件**: 服务合同模板（范围/付款/责任/知识产权）、NDA模板、关键条款审查清单
- **可选组件**: 外包合同模板、劳动合同（如有）
- **组装顺序**: 合同类型识别→模板选择/起草→关键条款审查→签署→归档
- **合规约束**: 合同必须经过审核；关键条款（责任限制/知识产权/付款条件）不可省略
- **格式**: 合同模板+审查清单（Markdown）

### P6-05 数据安全与隐私
- **必选组件**: 数据采集范围与合法性、存储方案与加密、用户隐私声明、数据跨境评估
- **可选组件**: 数据备份方案、安全审计、合规认证
- **组装顺序**: 数据盘点→采集合法性→存储方案→隐私声明→跨境评估→安全措施
- **合规约束**: 用户数据采集必须有合法依据；敏感数据必须加密存储；如涉及跨境须评估合规
- **格式**: 数据安全方案（Markdown）
