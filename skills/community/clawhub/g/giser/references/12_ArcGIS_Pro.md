## 7.5 转为 CAD 工具集详解（★核心）

**2个工具：**
- **添加CAD字段**：一步添加 Layer/Color/LineType/Elevation/CADType/DocPath/RefName 等保留字段
- **导出至CAD**：要素导出为 DWG/DXF/DGN（支持14种格式版本，DWG_R14~R2018、DXF_R14~R2018、DGN_V8）

**字段映射规则：**

| 字段 | 控制内容 | CASS应用 |
|------|----------|----------|
| Layer | CAD图层名 | CASS编码分层 |
| CADType/Entity | 实体类型(TEXT/INSERT/POINT等) | 注记→TEXT、符号→INSERT |
| Elevation | Z坐标 | 高程点赋值 |
| Contour | 线Z坐标 | 等高线高程 |
| RefName | 块插入名 | CASS符号块名 |
| DocPath | 输出文件路径 | 分割输出多个DWG |
| LTScale/LineWt | 线宽控制 | 线型显示 |
| Color | 实体颜色 | 图层颜色 |

**种子文件机制：**
- 模板DWG预定义块、图层、线型、符号系统
- 输出继承种子文件的CAD标准
- RefName/Layer字段可按名引用种子文件定义
- 种子文件格式版本覆盖输出类型参数

**输出格式列表：**
DWG_R2018(默认)/R2013/R2010/R2007/R2005/R2004/R2000/R14、DXF_R2018/R2013/R2010/R2007/R2005/R2004/R2000/R14、DGN_V8

## 7.11 GDB ↔ CASS/DWG 转换实战

### GDB导出到CAD流程
1. 运行"添加CAD字段"（可选）
2. 填充Layer/Color/LineType/Elevation/CADType/RefName等字段
3. 运行"导出至CAD"，选择DWG_R2018格式
4. 可选：指定种子文件预设图层和符号标准
5. 可选：DocPath字段分割输出到多个DWG文件

### CAD导入到GDB流程
1. 运行"CAD至地理数据库"工具（批量）
2. 或使用"导出要素"工具（单个要素类）
3. 确保CAD文件有.prj投影文件定义坐标系
4. 输出：点/线/面要素类 + DWG附加MSC要素类

### CASS特殊处理
- CASS实体编码(XData)需要通过Add CAD Fields预设对应字段
- CASS六位编码→Layer字段实现分层导出
- CASS注记和符号使用INSERT实体→RefName控制符号块
- CASS等高线→Contour字段存储高程
- CASS宗地/房屋面状要素→CADType=INSERT保留符号样式

## 7.16 CASS转换Python代码示例

### 7.16.1 CAD批量导入GDB

```python
import arcpy, os

arcpy.env.workspace = "C:/data/CAD"
output_gdb = "C:/data/GIS.gdb"

if not arcpy.Exists(output_gdb):
    arcpy.management.CreateFileGDB(os.path.dirname(output_gdb),
                                   os.path.basename(output_gdb))

for dwg in arcpy.ListFiles("*.dwg"):
    print(f"处理: {dwg}")
    arcpy.conversion.CADToGeodatabase(dwg, output_gdb, "CAD_Data", 1000)
print("批量转换完成")
```

### 7.16.2 GDB导出CAD（带字段控制）

```python
import arcpy

input_features = "C:/data/GIS.gdb/Buildings"
output_cad = "C:/data/Buildings.dwg"

# 添加CAD保留字段
for field in ["Layer", "Color", "Linetype", "Elevation"]:
    if field not in [f.name for f in arcpy.ListFields(input_features)]:
        arcpy.management.AddField(input_features, field, "TEXT", field_length=50)

# 填充字段值
with arcpy.da.UpdateCursor(input_features, ["Layer", "Color"]) as cursor:
    for row in cursor:
        row[0] = "Buildings"  # CAD图层名
        row[1] = "1"          # CAD颜色
        cursor.updateRow(row)

arcpy.conversion.ExportCAD(input_features, "DWG_R2018", output_cad,
                           "Ignore_Filenames_in_Tables", "Overwrite_Existing_Files")
```

### 7.16.3 坐标系转换

import arcpy

## 7.17 ModelBuilder 可视化建模

> 来源：ArcGIS Pro 3.6 官方帮助 — ModelBuilder

### 定义

ModelBuilder 是一种**可视化编程语言**，用于构建地理处理工作流。模型将一系列流程和地理处理工具串联为示意图，一个流程的输出用作另一个流程的输入。

### 五大核心功能

| 功能 | 说明 | 测绘应用 |
|------|------|----------|
| **构建模型** | 通过添加和连接数据和工具来构建模型 | 串联多步骤地理处理流程 |
| **迭代处理** | 迭代处理每个要素类/栅格/文件/工作空间中的表 | 批量坐标转换、批量裁剪 |
| **可视化** | 以易于理解的图表显示工作流顺序 | 流程文档化、方案评审 |
| **分步运行** | 分步运行模型，一直到所选步骤，或运行整个模型 | 调试复杂流程 |
| **作为工具共享** | 将模型设为地理处理工具，可在 Python 脚本和其他模型中调用 | 标准化流程复用 |

### 3.6 版界面更新

- 元素节点改为**圆角矩形**
- 变量显示数据类型图标，工具显示工具类型图标（内置/模型/脚本）
- 运行时显示**进度条**，完成后显示绿色✓或红色✗
- 柔和配色方案区分准备运行/等待/已完成状态

### 典型示例：野生动物栖息地分析模型

```
[Vegetation图层] → 按属性选择图层 → 输出选择
[主要道路] → 缓冲区(1500英尺) → 道路缓冲区
[输出选择 + 道路缓冲区] → 擦除 → 擦除结果
[擦除结果 + 坡度 + 高程 + 气候] → 相交 → 满足所有条件的区域
```

### 迭代器类型

| 迭代器 | 遍历对象 | 典型场景 |
|--------|----------|----------|
| **For** | 指定次数 | 重复运行N次 |
| **While** | 满足条件时 | 阈值迭代优化 |
| **要素选择** | 要素类中的要素 | 逐行处理 |
| **行选择** | 表中的行 | 逐记录处理 |
| **要素类** | 工作空间中的要素类 | 批量处理多图层 |
| **栅格** | 工作空间中的栅格 | 批量栅格计算 |
| **表** | 工作空间中的表 | 批量表操作 |
| **工作空间** | 文件夹中的工作空间 | 跨GDB批量处理 |
| **文件** | 文件夹中的文件 | 批量文件转换 |
| **字段值** | 字段中的唯一值 | 按分类字段处理 |
| **多值** | 输入值列表 | 多参数组合 |
| **时间** | 时间序列 | 时序分析 |

### 模型验证与调试

- **模型验证**：运行前自动检查参数有效性、数据存在性、数据类型兼容性
- **中间数据管理**：中间输出可标记为"中间"，运行后自动清理

> 来源：ArcGIS Pro 3.6 官方帮助 — Tasks

### 什么是任务？

任务是**预配置的交互式步骤集**，引导用户按标准化流程完成工作。核心价值：
- 实施最佳实践工作流，确保操作一致性
- 降低操作门槛，减少犹豫和错误
- 可作为新员工培训教程

### 任务组成结构

| 组件 | 说明 | 测绘应用 |
|------|------|----------|
| **任务项** | 存储容器，在目录窗格"任务"文件夹下管理 | 按业务模块分组（外业入库/质检/制图） |
| **单个任务** | 一组连续步骤，可独立运行或归入任务组 | 对应具体工序（如"RTK点导入"） |
| **任务步骤** | 核心执行单元 | 自动运行工具、设置视图、控制图层 |
| **任务窗格** | 操作界面，侧边栏显示步骤列表 | 清晰、不可跳过的操作面板 |

### 步骤配置能力

| 步骤类型 | 可执行操作 | 测绘场景 |
|----------|-----------|----------|
| **运行工具/命令** | 自动打开地理处理工具并填入预设参数 | 预置坐标转换参数→一键执行 |
| **设置地图视图** | 自动缩放/平移至预定义范围或图层 | 自动定位到测区范围 |
| **控制图层状态** | 设置可见性、可选性、可捕捉性 | 质检步骤切换图层组合 |

### 记录操作自动创建步骤

通过**"记录操作"**功能，设计者手动执行一遍操作流程，系统自动记录为任务步骤——零代码创建标准化流程。

### 共享与分发

```
导出 → .esriTasks 文件 → 团队成员导入
```

**测绘场景**：技术负责人一次性设计好SOP，导出为任务文件分发→所有内外业人员加载即用→统一流程、统一成果。

## 9.1 场景（Scenes）—— 3D 制图

### 9.1.1 场景类型对比

| 维度 | 全球场景 (Global) | 局部场景 (Local) |
|------|-------------------|------------------|
| 适用场景 | 跨越地球的数据；需查看地球曲率 | 小范围（城市/工地）；编辑/分析/测量 |
| 坐标系 | 仅 WGS84 / CGCS2000 地理坐标系 | 支持地理/投影/自定义坐标系 |
| 注意 | 非 WGS84/CGCS2000 建议用局部场景避免重投影畸变 | 基于 ArcGIS Maps SDKs 的移动端不支持局部场景 |

### 9.1.2 场景创建方式

1. 插入 → 新建地图 → 新建全球/局部场景
2. 视图选项卡切换全球/局部模式
3. 目录窗格右键场景 → "作为新的全球/局部视图打开"（同时查看两种模式）
4. 导入旧文档：`.3dd`(ArcGlobe)→全球 / `.sxd`(ArcScene)→局部

### 9.1.3 绘制模式

| 模式 | 特点 | 适用场景 |
|------|------|---------|
| 透视绘制 | 近大远小，人眼观察习惯 | 通用3D可视化 |
| 平行绘制 | 相同尺寸要素渲染一致 | 建筑绘图、3D统计可视化 |

### 9.1.4 场景内容

- **2D图层**：无z坐标，地面叠加

- **高程表面**：默认"地面"，可自定义多个表面（地质层/臭氧层等），数据源支持栅格/TIN/高程服务

## 9.6 标注系统

### 9.6.1 标注核心概念

| 标注（动态） | 注记（静态） |
|-------------|-------------|
| 位置自动生成 | 人工放置 |
| 不可选择 | 可独立选择 |
| 无法编辑单个显示属性 | 可逐个编辑 |

### 9.6.2 标注引擎对比

| 引擎 | 特点 |
|------|------|
| **Maplex Label Engine**（默认） | 高级放置+智能冲突检测 |
| Standard Label Engine | 基本放置 |

> ⚠️ Maplex→Standard 切换会**丢失所有高级放置设置**，不可恢复。

### 9.6.3 Maplex四大标注放置属性

**1. 位置 (Position)** — 标注如何放置？
- 点/线/面各有独特位置选项
- 线要素：弯曲标注或平直标注
- 支持字间距、字符间距、投影刻度对齐

**2. 自适应策略 (Adaptive Strategies)** — 如何自适应地图版面？
- 可用策略：标注堆叠、要素超限、减小字型、标注缩写、键编号
- 可指定策略优先级顺序

**3. 冲突解决 (Conflict Resolution)** — 多标注竞争谁胜出？
- 要素权重：0~1,000（0=可用空间，1,000=不可被压盖）
- 内部/边界权重（面要素）、背景标注、标注密度控制
- 大要素可放置多个标注、移除重复标注

**4. 权重和障碍** — 标注优先放自由空间，如需压盖选总权重最低的位置

### 9.6.4 地图级别标注属性

| 工具 | 功能 |
|------|------|
| 暂停 | 暂停标注绘制 |
| 锁定 | 锁定当前位置和大小 |
| 查看未放置 | 显示无法放置的标注 |
| 优先级/权重 | 调整标注顺序 |
| 缩写字典 | 创建/管理缩写（仅Maplex） |
| 键编号组 | 创建/管理键编号（仅Maplex） |
| 草稿质量 | 速度更快但标注更少（仅Maplex） |

### 9.6.5 标注管理

| 操作 | 方法 |
|------|------|
| 开启标注 | 标注选项卡 → Label按钮 |
| 标注类视图 | 内容面板→按标注列出→批量管理/复制粘贴属性 |

# 第十一篇：ArcGIS Pro 3.6 数据管理与编辑

> 来源：ArcGIS Pro 3.6 官方帮助 — 地理数据库、数据设计工具、编辑功能、数据验证

## 11.1 地理数据库（GDB）基础

### 定义与架构

地理数据库是 ArcGIS 中组织地理信息的核心容器，所有元素以表形式存储在 DBMS 中。

| 层次 | 说明 |
|------|------|
| 基本数据集 | 要素类（Feature Class）、栅格数据集（Raster Dataset）、表（Table） |
| 存储形式 | 要素类每行一个要素，`Shape` 字段存储几何 |
| 方案 (Schema) | 包含坐标系、坐标分辨率、要素类、拓扑、网络、关系、属性域定义 |
| 空间表达 | 矢量：有序 x,y 坐标 + 可选 m/z 值；栅格：波段+像素数组 |

### 三种地理数据库对比

| 特性 | 文件 GDB (.gdb) | 移动 GDB (.geodatabase) | 企业级 GDB |
|------|:--:|:--:|:--:|
| 存储形式 | 文件夹，每数据集独立文件 | 基于 SQLite 单文件 | 存储在 DBMS（Oracle/SQL Server/PostgreSQL 等） |
| 数据集大小上限 | 默认 1TB（可配置至 256TB） | 2TB | 由 DBMS 决定（几乎无限） |
| 多用户编辑 | ✗ | ✗ | ✓ |
| 版本管理 | ✗ | ✗ | ✓ |
| 编辑者追踪/附件 | ✗ | ✓ | ✓ |
| 条件值 | ✗ | ✓ | ✓ |
| 网络数据集/Terrain | ✓ | ✗ | ✓ |
| 存储效率 | 约为 shapefile 的 1/3 | — | — |

> 选用决策：小项目→文件 GDB；便携交换→移动 GDB；多用户协作→企业级 GDB。

## 11.2 GDB 数据集类型（20种）

| 数据集 | 说明 |
|--------|------|
| 表 (Table) | 行列组成的存储对象；含空间属性的表 = 要素类 |
| 要素类 (Feature Class) | 同几何类型、同属性、同空间参考的要素集合 |

### 扩展类型

|--------|------|----------|
| 3D 对象 | 高级 3D 几何，支持 PBR 和动画 | — |
| 注记 (Annotation) | 可独立选择/定位/修改的文本图形 | — |
| 要素数据集 | 共用坐标系的主题相关要素类集合 | — |
| 拓扑 | 定义要素间空间关系规则，强制执行数据完整性 | — |
| 网络数据集 | 构建交通网模型 | Network Analyst |
| 宗地结构 | 宗地数据管理框架 | — |
| Terrain 数据集 | 多分辨率 TIN 表面 | 3D Analyst |
| 镶嵌数据集 | 大规模影像管理+动态镶嵌 | — |

| 尺寸 (Dimension) | 特殊注记，显示长度/距离 | — |

## 11.3 数据设计工具（六大组件）

### 11.3.1 字段 (Fields)

| 参数 | 说明 |
|------|------|
| Field Name / Alias | 正式名称 / 别名 |
| Data Type | 数据类型（文本/数值/日期等） |
| Allow NULL | 是否允许空值 |
| Domain / Default | 绑定的属性域 / 默认值 |
| Precision / Scale | 数值最大位数 / 小数位数 |

**属性窗格字段颜色代码**：橙色=子类型字段、紫色=符号化字段、黄色=高亮字段、灰色=只读、绿色=未使用默认值、蓝色=使用默认值、红色=严重无效值（违反属性域）

### 11.3.2 属性域 (Domains)

| 类型 | 说明 | 适用字段 |
|------|------|----------|
| 范围域 (Range) | 指定最小值~最大值范围 | 数值、日期 |
| 编码值域 (Coded Value) | 指定离散值列表（代码+描述） | 所有类型 |

### 分割/合并策略

| 操作 | 策略 | 说明 |
|------|------|------|
| 分割 | 默认值 / 复制 / 几何比 | 几何比仅限数值字段 |
| 合并 | 默认值 / 总和值 / 几何加权 | 总和值和几何加权仅限数值字段 |

### 11.3.3 子类型 (Subtypes)

五大作用：自动默认值、属性域约束、规则关联、性能优化、自定义规则。

> 示例：城市街道 → local（地方）/ collector（集散）/ arterial（主干），每个子类型不同默认值和属性域。

### 11.3.4 条件值 (Contingent Values)

字段组内字段间的**联动值约束**——在某字段选择值后，系统自动强制执行关联限制。

| 概念 | 说明 |
|------|------|
| 字段组 | 指定字段集合，条件值建立在其上 |
| 限制性字段组 | 严格限制为指定条件值（默认） |
| 非限制性字段组 | 允许提交未指定的值 |
| `<ANY>` | 特殊值，该字段任何值都有效 |
| 停用值 | 历史数据仍可见，但不可用于新编辑 |

**生成模式**：数据模式（按实际数据定义有效组合）/ 方案模式（按属性域创建所有可能组合）

### 11.3.5 属性规则 (Attribute Rules)

| 规则类型 | 目的 | 评估方式 |
|----------|------|----------|
| 计算规则 | 根据编辑模式自动填充属性字段 | 实时或批处理 |
| 约束规则 | 控制数据输入，防止无效数据 | 实时评估 |
| 验证规则 | 高亮显示现有数据错误 | 批处理（延迟） |

**前置条件**：数据集需全局 ID；企业级 GDB 需数据所有者身份；批处理规则需编辑者追踪。
**表达式语言**：ArcGIS Arcade。**兼容性**：添加属性规则后不再兼容 ArcMap 及 ArcGIS Pro ≤2.0。

### 11.3.6 编辑者追踪 (Editor Tracking)

自动记录的 4 个字段：

| 字段 | 内容 |
|------|------|
| `created_user` | 创建者用户名 |
| `created_date` | 创建日期时间 |
| `last_edited_user` | 最后编辑者用户名 |
| `last_edited_date` | 最后编辑日期时间 |

> 时间标准：跨时区数据必须使用 **UTC**。支持的数据集：表、要素类、注记类、尺寸类、要素数据集、镶嵌数据集（移动 GDB 除外）、属性关系类。

## 11.9 数据设计五层金字塔架构

> 来源：ArcGIS Pro 3.6 官方帮助 — 数据设计

### 核心理念

"成功的 GIS 构建于精心设计的地理数据集合之上，该集合可有效地对世界建模并可进行简单而高效的显示、编辑和分析。"

### 五层架构（由基础到高级）

```
         ┌──────────────┐
         │ 5.属性规则    │ ← 自动计算+约束+验证
         ├──────────────┤
         │ 4.条件值      │ ← 字段间联动约束
         ├──────────────┤
         │ 3.子类型      │ ← 要素分类+默认值配置
         ├──────────────┤
         │ 2.属性域      │ ← 字段有效值限制
         ├──────────────┤
         │ 1.字段        │ ← 最基本结构单元
         └──────────────┘
```

| 层级 | 名称 | 核心机制 | 复杂度 | 测绘应用示例 |
|------|------|----------|:---:|-------------|
| **L1** | 字段 (Fields) | 定义数据类型、长度、别名、是否可空 | ★ | 建筑物名称(Text)/层数(Short)/面积(Double) |
| **L2** | 属性域 (Domains) | 范围域(数值范围)或编码值域(离散列表) | ★★ | 地类编码域：011=水田/012=水浇地/013=旱地 |
| **L3** | 子类型 (Subtypes) | 按分类字段分组，每组不同默认值和域 | ★★★ | 房屋→砼房/砖房/土木房，各不同层数范围 |
| **L4** | 条件值 (Contingent Values) | 字段间联动值约束（选A字段→B字段可选值联动变化） | ★★★★ | 选"道路"→路面类型可选沥青/水泥；选"河流"→不可选路面类型 |
| **L5** | 属性规则 (Attribute Rules) | 计算规则(自动填充)+约束规则(阻止无效)+验证规则(事后检查) | ★★★★★ | 面积=Shape_Area自动同步；宗地号唯一性校验 |

### 操作入口

```
目录窗格 → 右键要素类/表 → 数据设计 → 字段/属性域/子类型/属性规则/条件值
```

### 设计原则

1. **自底向上构建**：先完成字段→再定义域→再划分子类型→最后加规则
2. **域优先原则**：能用属性域解决的问题不用条件值；能用条件值的不用属性规则
3. **最小复杂度原则**：仅在业务需求明确需要时才升级层次
4. **测绘场景优先级**：字段(L1)+属性域(L2)覆盖80%场景；子类型(L3)覆盖地类分类需求；属性规则(L5)用于面积/编号自动计算

## 12.4 arcpy.da 数据访问模块

### 游标对比

| 游标 | 访问 | 核心方法 | 适用场景 |
|------|------|----------|----------|
| **SearchCursor**(table, fields, {where}) | 只读 | reset() | 查询/遍历数据 |
| **InsertCursor**(table, fields) | 写入 | insertRow(row) → OID | 批量添加记录 |
| **UpdateCursor**(table, fields, {where}) | 读写 | updateRow(row), deleteRow() | 批量修改/删除 |

### 重要令牌

| 令牌 | 返回 |
|------|------|
| `SHAPE@XY` | (X, Y) 元组 |
| `SHAPE@` | 完整 Geometry 对象 |
| `SHAPE@WKT` | WKT 字符串 |
| `OID@` | ObjectID 字段 |
| `SHAPE@AREA` | 面积（面要素） |
| `SHAPE@LENGTH` | 长度（线要素） |

### 完整代码模式

```python
# SearchCursor — 查询（with 语句自动关闭）
with arcpy.da.SearchCursor(fc, ['NAME', 'SHAPE@XY', 'POPULATION'], "POP > 10000") as cursor:
    for name, xy, pop in cursor:
        print(f"{name}: {pop} at {xy}")

# InsertCursor — 插入
with arcpy.da.InsertCursor(fc, ['NAME', 'SHAPE@XY']) as cursor:
    cursor.insertRow(('新点', (120.5, 30.2)))

# UpdateCursor — 批量更新
with arcpy.da.UpdateCursor(fc, ['STATUS', 'AREA']) as cursor:
    for row in cursor:
        if row[1] > 10000:
            row[0] = '大型'
            cursor.updateRow(row)
```
