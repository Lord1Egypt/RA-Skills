## 多轮优化原理

### 多轮优化的核心逻辑

```
初始方案 → 执行 → 反馈 → 分析 → 调整 → 优化方案 → 执行 → ...
                                    ↑
                                    └──────────────┘
```

### 关键原则

| 原则 | 说明 |
|-----|-----|
| 反馈驱动 | 根据执行反馈调整，而非固定方案 |
| 小步快跑 | 每次调整一小步，持续优化 |
| 保持方向 | 调整方法，不调整目标 |
| 记录学习 | 每次调整都是学习机会 |

## 反馈识别机制

### 反馈类型

| 类型 | 信号 | 说明 |
|-----|-----|-----|
| **正向反馈** | 效果良好，用户满意 | 继续当前方案，可考虑强化 |
| **中性反馈** | 效果一般 | 需要微调 |
| **负向反馈** | 效果不好，用户不满 | 需要调整 |
| **停滞反馈** | 没有进展 | 需要分析原因 |

### 反馈识别信号

**正向反馈信号**：

```
- 能量评分提升
- 正负比改善
- 里程碑达成
- 用户表达"感觉好多了"
- 冲突减少
- 亲密增加
```

**负向反馈信号**：

```
- 能量评分下降
- 正负比恶化
- 计划无法执行
- 用户表达"没有用"
- 冲突增加
- 退回到旧模式
```

### 反馈对话模板

**收集反馈**：

```
"这周执行得怎么样？
 和上周比有什么变化？"

"你觉得方案有帮助吗？
 具体是哪部分有帮助？"

"有没有什么执行起来特别困难的？"

"如果10分是满分，你给这个方案打几分？"
```

**深度反馈**：

```
"你说'有帮助'，具体是哪里有帮助？"

"你说'没帮助'，具体是哪里没帮助？"

"执行中遇到的最大挑战是什么？"

"如果可以改变方案的一个部分，你想改变什么？"
```

## 障碍分析框架

### 障碍类型

| 类型 | 说明 | 示例 |
|-----|-----|-----|
| **执行障碍** | 想做但做不到 | 太忙、忘了我、不知道怎么做 |
| **意愿障碍** | 不想做 | 不认同、不相信、没动力 |
| **匹配障碍** | 方案不适合 | 方法不对、节奏不对、不适合双方 |
| **时机障碍** | 时机不对 | 太早/太晚、缺乏准备、外部压力 |

### 障碍诊断流程

```
步骤1：识别障碍类型
↓
步骤2：深入了解具体情况
↓
步骤3：确定根本原因
↓
步骤4：选择对应解决方案
```

### 障碍诊断问题

**诊断执行障碍**：

```
"你是想执行但没执行，还是执行了但效果不好？"

"执行中遇到的具体困难是什么？"

"是不知道怎么做，还是知道但做不到？"

"是时间不够，还是动力不够？"
```

**诊断意愿障碍**：

```
"你觉得这个方法不适合你吗？"

"你对这种方法有什么顾虑？"

"你不认同这个方向的哪一点？"

"你有没有更好的想法？"
```

**诊断匹配障碍**：

```
"你觉得这个方案哪里不适合？"

"是节奏太快还是太慢？"

"是你还是伴侣的问题？"

"如果改变某个部分，你愿意尝试吗？"
```

## 方案调整策略

### 调整类型

| 调整类型 | 适用情况 | 调整方式 |
|---------|---------|---------|
| **简化** | 方案太复杂 | 删除次要内容，保留核心 |
| **分解** | 步骤太多 | 拆成更小的步骤 |
| **替代** | 方法不对 | 换成其他方法 |
| **强化** | 力度不够 | 增加频率或深度 |
| **暂停** | 时机不对 | 暂停一段时间 |
| **升级** | 进展良好 | 挑战更高目标 |

### 调整策略详解

**策略1：简化**

```
适用：方案太复杂，无法执行

操作：
1. 识别方案中最核心的一个行为
2. 先只做这一个
3. 等稳定后再加其他

示例：
原来："每天check-in + 每周深度对话 + 每月约会"
调整后："先只做每天check-in"
```

**策略2：分解**

```
适用：步骤太难

操作：
1. 识别哪一步最难
2. 把这一步拆成更小的子步骤
3. 从最简单的一步开始

示例：
原来："进行一次深度对话"
调整后："先复述对方说的话3次"
```

**策略3：替代**

```
适用：方法不适用

操作：
1. 识别不适用的具体部分
2. 寻找替代方法
3. 尝试新方法

示例：
原来："每周五约会"
不适用："周五总加班"
替代："改为周六上午"
```

**策略4：强化**

```
适用：效果良好，可以更进一步

操作：
1. 识别效果最好的部分
2. 增加频率或深度
3. 扩展到其他领域

示例：
原来："每周一次约会"
效果良好，调整为："每周两次约会"
```

**策略5：暂停**

```
适用：时机不对或有阻力

操作：
1. 承认当前时机不对
2. 暂停该方案
3. 约定重新开始的时机

示例：
"最近工作特别忙，我们暂停每周约会，
 等这阵子忙完再恢复，下个月怎么样？"
```

**策略6：升级**

```
适用：基础目标达成，能力允许

操作：
1. 评估当前进展
2. 设定更高的目标
3. 制定升级计划

示例：
原来："进行每日check-in"
达成后，调整为："增加每周一次感恩表达"
```

### 调整对话模板

**方案太复杂**：

```
"我理解信息量有点大。
 我们先简化一下，专注于最简单的一件事。"

"这周我们只做...，其他的先放一放。
 等这个稳定了，我们再加其他的。"

"你觉得哪一部分最简单、最容易执行？"
```

**方法不适用**：

```
"听起来这个方法不太适合你们的情况。"

"我们来换一个方法...
 你觉得这样可以吗？"

"如果是这样（新的方案），你能接受吗？"
```

**时机不对**：

```
"我觉得可能时机不太对。
 你们最近压力比较大。"

"我们可以先暂停这个计划，
 等压力小一点再继续。"

"下个月再重新开始，好吗？"
```

**效果很好**：

```
"进展得不错！"

"既然效果很好，我们来升级一下。
 尝试...，你觉得怎么样？"

"这证明了你们是可以做到的。
 接下来我们挑战一个更高的目标？"
```

## 迭代升级路径

### 迭代层级

| 层级 | 适用情况 | 调整范围 |
|-----|---------|---------|
| **微调** | 小问题 | 调整话术、时机、频率 |
| **调整** | 中等问题 | 更换方法、简化步骤 |
| **重构** | 大问题 | 更换模型、改变方向 |
| **重启** | 完全失败 | 从头开始 |

### 迭代路径选择

```
情况分析
    ↓
┌─────────────────────────────────┐
│ 效果良好 → 继续 + 强化           │
│ 效果一般 → 微调                 │
│ 效果不好 → 分析原因              │
│ 完全失败 → 重启                  │
└─────────────────────────────────┘
    ↓
原因分析
    ↓
┌─────────────────────────────────┐
│ 执行问题 → 简化/分解            │
│ 意愿问题 → 沟通/替代            │
│ 匹配问题 → 调整/替代            │
│ 时机问题 → 暂停/等待            │
└─────────────────────────────────┘
    ↓
选择调整策略
    ↓
新方案
```

### 迭代对话引导

**启动迭代**：

```
"看起来当前的方案遇到了一些挑战。
 让我们来分析一下问题在哪里。"

"我们来调整一下，让它更适合你们的情况。"
```

**迭代中**：

```
"这周我们试着用新的方式...
 感觉怎么样？"

"和之前比，有什么不同？"

"这样的调整适合你们吗？"
```

**迭代成功**：

```
"效果不错！
 看来调整是对的。"

"我们继续用这个新方案，
 下周看看效果能不能保持。"
```

**迭代失败**：

```
"这次调整好像效果也不理想。

让我们重新分析一下问题。
 可能是...（新的假设）"

"我们来尝试另一种方式..."
```

## 持续优化机制

### 周度优化

```
每周五：
1. 回顾本周执行情况
2. 识别问题或亮点
3. 做小调整（如果需要）
4. 计划下周
```

### 月度优化

```
每月末：
1. 回顾本月整体进展
2. 评估方案有效性
3. 做中等调整（如果需要）
4. 制定下月计划
```

### 季度优化

```
每季度末：
1. 深度复盘
2. 评估是否需要换模型
3. 制定下季度目标
4. 必要时重构方案
```

### 优化决策树

```
本周效果如何？
    ↓
┌──────────────┬──────────────┐
│ 效果好        │ 效果不好      │
↓              ↓
├──────────────┴──────────────┐
│                        │
↓                          ↓
可以加强或扩展         分析原因
    ↓                      ↓
┌──────────────┐    ┌──────────────┐
│ 增加频率    │    │ 执行问题    │ → 简化
│ 增加深度    │    │ 意愿问题    │ → 沟通
│ 扩展领域    │    │ 匹配问题    │ → 调整
└──────────────┘    │ 时机问题    │ → 暂停
                    └──────────────┘
                          ↓
                       调整后继续
```

### 优化清单

```
【持续优化清单】

□ 每周五：周度回顾和微调
□ 每周：记录关键事件和感受
□ 每月末：月度评估
□ 每月：与AI进行月度对话
□ 每季度：深度复盘
□ 每个里程碑达成后：庆祝+新目标
□ 遇到障碍时：障碍分析+调整
□ 连续2周无进展：方案重构
□ 重大变化时：从头评估
```
