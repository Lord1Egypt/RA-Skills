# 断点续跑

项目中断后重新启动时，按以下顺序检查文件状态，从第一个缺失/未完成的环节继续。

---

## 检查链

```
1.  无项目文件夹且无创意草稿？ → 从立项开始（第1步：信息采集）
2.  有创意草稿但无项目文件夹？ → 读取草稿恢复上下文，继续立项流程
3.  无创意原点.md？ → 继续立项流程
4.  无基础信息.md 或 世界观与运行规则.md？ → 立项整理输出
5.  无人物小传？ → 生成人物小传
6.  人物小传质检≠通过？ → 人物小传质检/修改
7.  无故事概要？ → 生成故事概要
8.  故事概要质检≠通过？ → 故事概要质检/修改
9.  逐集概要批次数不足？ → 继续生成缺失批次
10. 逐集概要质检≠通过？ → 逐集概要质检/修改
11. 剧本EP数 < 总集数？ → 继续生成缺失集数
12. 剧本质检≠通过？ → 剧本质检/修改
13. 以上全部完成？ → 项目交付
```

---

## 判断规则

- 文件不存在 → 生成
- 质检报告不存在 → 执行质检
- 最后状态为打回 → 修改 → 重新质检
- 最后状态为通过 → 检查下一环节

断点续跑同样遵循当前会话的工作模式（子代理模式或主代理直出模式）。

---

## 批量环节续跑

逐集概要、剧本等批量/逐集环节，续跑时：

- 计算已完成批次数/集数，从下一个未完成的开始
- 批次计算规则：每批10集，批次数 = `ceil(总集数 / 10)`
- 总集数读取自 `{剧名}/基础信息.md`
