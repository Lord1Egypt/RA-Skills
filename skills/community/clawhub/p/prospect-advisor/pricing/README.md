# 裂变定价技术方案 — 高考志愿填报Skill

## 核心逻辑

**差价 = 推广者的利润空间 = 裂变动力**

| 套餐 | 单人支付 | 推广者转售价 | 推广者单笔利润 |
|------|----------|-------------|---------------|
| 20人次(3万) | 1,500/人 | 1,999/人 | +499/人 (+33%) |
| 50人次(5万) | 1,000/人 | 1,999/人 | +999/人 (+99%) |
| 100人次(8.9万) | 888/人 | 1,999/人 | +1,111/人 (+125%) |

## 三级定价架构

```
┌─────────────────────────────────────┐
│           用户端（小程序内）          │
│  ├─ 免费版：3次查询（引流）          │
│  ├─ 单人次：1,999元（零售）          │
│  └─ 推广者入口：批量购买              │
├─────────────────────────────────────┤
│           推广者端（代理商后台）        │
│  ├─ 20人次套餐：30,000元              │
│  ├─ 50人次套餐：50,000元              │
│  └─ 100人次套餐：88,880元            │
├─────────────────────────────────────┤
│           惠迈端（结算后台）           │
│  ├─ 订单管理                          │
│  ├─ 推广者分成结算                    │
│  └─ 数据分析                          │
└─────────────────────────────────────┘
```

## 裂变路径设计

```
普通用户（免费版体验）
    │
    ├─ 体验满意 → 转单人次 1,999元
    │
    └─ 发现商机 → 联系惠迈 → 成为推广者
        │
        ├─ 买20人次(3万) → 熟人圈推广
        ├─ 买50人次(5万) → 班级/年级推广
        └─ 买100人次(8.9万) → 多班/校级推广
            │
            └─ 推广者的客户 → 体验免费版 → 转化付费
                ↑ (推广者拉人头，客户从免费版入口进来)
```

## 技术实现要点

### 1. 免费版限制
- 同一手机号/微信号限3次免费查询
- 数据量限制：仅查3所学校的基础分数线
- 不提供完整推荐报告

### 2. 套餐核销机制
- 推广者购买套餐后，获得N个核销码
- 每个核销码对应1次完整查询
- 核销码可分发、可批量导入

### 3. 微信支付集成
- 商户号接入
- 套餐购买走B端支付
- 单人次查询走C端支付

### 4. 佣金结算
- 推广者购买套餐时自动记录
- 推广者的客户通过推广链接进入
- 系统自动统计推广业绩
