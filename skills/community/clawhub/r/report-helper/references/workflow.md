# 研究与写作 Workflow

本文件承接 `SKILL.md` 的详细流程。

## 第一步：并行联网搜集

用 3 路 research worker 搜集资料。若当前 Agent 支持 subagent，可以并行；若不支持，就在主会话里按三轮顺序执行。每路 prompt 必须遵循 `references/subagent-research-prompt.md` 的模板。

分工：

- **Agent 1 — 时间线**：起源、创始人、发展历程、关键事件、版本迭代、危机
- **Agent 2 — 同期格局**：竞品识别、各竞品特点、用户口碑、对比评测、市场与商业数据
- **Agent 3**（复杂对象才启用）：补充信息，创始人深度背景、社区讨论、行业环境、最新数据校验

所有研究任务都必须做一次**最新数据校验**：在进入关卡二前，用当前日期附近的关键词重新查找近 12-24 个月的官方披露、监管文件、财报、原始研究报告、公司公告、权威媒体原创报道或行业数据。研究公司时，这一步还要覆盖收入/ARR、用户或客户、利润率、现金流、融资、估值、可比公司指标等可公开获得的定量材料。

宁可多搜，不要因为信息不够导致分析浮于表面。

### 中间资料落盘

3 路 research worker 返回后，主会话必须立即把各路产出落盘成独立中间文档，保存到配置的中间资料目录。默认目录是：

```text
./output/intermediate/{topic-slug}-{YYYY-MM-DD}/
├── 00-信息充分性审计.md
├── 01-时间线资料.md
├── 02-同期格局资料.md
└── 03-补充资料.md
```

如果 `config.local.json` 或 `REPORT_HELPER_INTERMEDIATE_DIR` 配置了其他目录，使用配置目录，并保持 `{topic-slug}-{YYYY-MM-DD}` 子目录结构。

每份中间文档最小结构：

```markdown
# {维度名}资料 —— {研究对象}

> 搜集时间：YYYY-MM-DD HH:MM | 搜集者：Agent N | 子任务：{时间线 / 同期格局 / 补充}

## 关键信息清单
[按时间线 / 按竞品 / 按主题分组的有序列表]

## 原始引用与来源
[每条关键事实：引用片段 + URL + 访问时间 + 来源类型 + 判定理由]

## 未搜到 / 存疑
[搜不到、来源单一、跨来源矛盾的项]
```

交付简报里必须告诉用户中间资料路径。

## 关卡二：信息充分性审计

搜完后，在 chat 里输出：

```markdown
【关卡二 · 信息充分性审计】

发展脉络覆盖：
- {已覆盖的关键节点逐条列出}
- 缺口：{明确写出没搜到的断层}

同期格局覆盖：
- 识别到的竞品：{A / B / C}
- 每个竞品的信息密度：{足够 / 薄弱，具体说明}
- 缺口：{明确写出}

来源质量：
- 一手源 N 条
- 二手源 M 条
- 存疑来源 K 条
- 统计依据：各路资料中 `来源类型` 字段
- 单一来源支撑的关键判断：{列出来，或写「无」}

最新数据校验：
- 已核查的最新资料：{列出最新官方/财报/监管/原始研究/权威报道及发布日期}
- 距今 12-24 个月关键数据是否覆盖：{是 / 否，缺什么}
- 公司对象定量数据：{收入/ARR、用户/客户、利润率、现金流、融资、估值、可比公司指标；没有公开数据就写缺口}

我的建议：{继续写作 / 先补 XXX 再开始}
```

交互模式停住等用户；auto 模式按建议执行。若关键一手源缺失、发展脉络断层明显、单一来源关键判断超过 3 条，或最新数据校验未完成，auto 模式也必须补搜。公司对象缺少成长性和估值定量材料时，必须补搜；若确实没有公开数据，写入缺口和替代指标。

同时把这份审计落盘为 `00-信息充分性审计.md`。

## 关卡三：写作准备审计

正文结构由 `references/report-template.md` 决定。本关卡只判断材料和结论是否足以进入写作，不规定正文目录和章节写法。生成核心判断时，可使用 `report-template.md` 的「写作判断工具」。

进入正文写作前，在 chat 里输出：

```markdown
【关卡三 · 写作准备审计】

成品体裁：
- 使用框架：{report-template.md 的 A. 正式研究报告结构 / B. 叙事型深度研究结构 / 用户指定结构}
- 判断依据：{用户原话中的体裁要求}

核心判断候选：
1. {判断 1：一句话结论 + 证据链出处}
2. {判断 2：一句话结论 + 证据链出处}
3. {判断 3：一句话结论 + 证据链出处}

自评：
- 以上判断是否只是资料复述？{否 / 是，若是则列出要重写的判断}
- 证据链是否够扎实？{薄弱项列出来}
- 是否还有关键事实只有单一来源支撑？{列出来，或写「无」}
- 最新数据是否已进入核心判断？{是 / 否，若否列出原因}
- 公司对象是否包含成长性和估值的定量判断？{是 / 否 / 不适用；若否列出要补的指标}
- 成品体裁是否匹配用户要求？{是 / 否}

建议：{进入 report-template.md 写作 / 先补 XXX 再写}
```

交互模式停住等用户；auto 模式不等确认，但若核心判断只是资料复述、体裁路由不清、证据链薄弱、最新数据未校验，或公司对象缺少成长性/估值定量判断，必须先补搜或重写判断。

## 第四步：按模板写作

关卡三通过后，进入正文写作。此处只规定写作动作，不展开正文目录：

1. 读取 `references/report-template.md`，按用户要求选择 A. 正式研究报告结构、B. 叙事型深度研究结构，或用户指定结构。
2. 读取 `references/writing-style.md`，统一语气、标点和表达风格。
3. 读取 `references/source-citation-rules.md`，先从中间资料建立来源台账，分配 a/b/c/d 等级和全报告唯一编号；正文判断只使用 a、b 级来源支撑。
4. 对象是产品、公司、概念或人物时，再读取 `references/adaptations-by-type.md`，调整侧重点。
5. 使用 `report-template.md` 的「写作判断工具」组织材料和生成判断，但不要把工具标题写成正文标题。
6. 把关卡二的来源质量、存疑项和单一来源判断带入正文处理：能补搜的先补搜，不能确认的写「未搜到 / 存疑」。
7. 把关卡二的最新数据校验带入正文：如果最新公开数据会改变判断，必须改写正文，不允许沿用旧叙事。
8. 公司对象必须写成长性和估值定量分析：收入/ARR、用户或客户、利润率、现金流、融资、估值、可比公司倍数等，按公开资料可得性选择；缺失数据要说明缺口和替代观察指标。
9. 在目录之后、第一章正文之前加入「关于本报告的信息来源」，声明编号规则。
10. 正文每条关键事实判断后添加 `<sup>a1</sup>` 这类来源上标；同一句有多来源时连续标注，不能只在段末笼统挂一个来源。
11. 正文末尾只保留「信息来源与分级」，按 a/b/c/d 分组列出完整来源；没有 c 和 d 就不写对应小节，c、d 级不支撑正文判断。

写作完成后，先保存一份内部 Markdown 构建稿，再进入审核。默认构建稿路径建议：

```text
./output/work/{YYYY-MM-DD}-{topic-slug}-report.md
```

如果 `config.local.json` 或 `REPORT_HELPER_WORK_DIR` 配置了其他目录，构建稿保存到配置目录。该 Markdown 只作为 PDF 渲染输入。

## 第五步：交付前审核

正文完成后，读取 `references/review-checklist.md` 做审核。审核不是抽查，命中问题必须改完再重扫。

审核顺序：

1. 体裁检查：正文结构是否与 `report-template.md` 选定框架一致；若用户指定目录，以用户指定为准。
2. 来源检查：正文前是否声明编号规则；正文上标是否与末尾「信息来源与分级」一一对应；是否存在孤儿上标、无正文引用的来源，或 c、d 级来源支撑正文判断。
3. 内容检查：关键判断是否有证据链；是否存在编造、单一来源硬判断、存疑事实未标注。
4. 最新性检查：是否在写作前完成最新数据校验，且最新资料已进入相关判断。
5. 公司定量检查：公司对象是否包含成长性、盈利质量、融资/现金流、估值或可比公司等定量分析；公开数据缺失时是否标注缺口。
6. 风格检查：按 `review-checklist.md` 清理套话、标点、句式和空泛表达。
7. 出稿检查：确认 H1、目录、信息来源与分级、引用标注、字数范围和存疑事实处理满足 `report-template.md`、`source-citation-rules.md` 与 `review-checklist.md`。
8. 回炉：任一项不通过，回到对应写作段落重写，再重新审核。

审核通过后，回到 `SKILL.md` 的生成交付物步骤，读取 `references/delivery.md`，生成 PDF。
