# Comparison Page Template

Use this template when the user wants to compare multiple algorithms in one page.

## Table of Contents
1. Goal
2. Section structure
3. Required comparison dimensions
4. Output checklist

---

## 1. Goal

A comparison page should help the learner understand:
- what the algorithms have in common
- where they differ mathematically
- where they differ operationally
- how their outputs differ on the same setup
- when each algorithm is more suitable

---

## 2. Section Structure

### Recommended structure
1. hero intro
2. shared task definition
3. algorithm summary cards
4. shared data and assumptions
5. formula comparison section
6. parameter comparison section
7. chart comparison section
8. result comparison table
9. final recommendation section

---

## 3. Required Comparison Dimensions

When possible, compare:
- core intuition
- target function or objective
- update rule
- parameters
- computational structure
- output behavior
- strengths and limitations

If the algorithms are in the same family, explicitly show:
- what is inherited
- what is newly added
- what is optimized or changed

---

## 4. Output Checklist

Before finalizing a comparison page, verify:
- the comparison uses a shared setup where possible
- formulas are not presented as isolated fragments
- differences are visible, not implied
- tables and charts reinforce the comparison
- the final recommendation is grounded in the content shown earlier
