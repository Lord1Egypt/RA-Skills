# MBA 论文学术规范检查清单（v1.7）

> 本文档是 **Guardrails 校验项**的人工对照清单。
> 自动化脚本：`scripts/loop_self_check.py`
> 使用方式：每个 Phase 退出前对照本清单 + 运行脚本，确保 100% 通过。

---

## 一、格式维度（18 项）

### 1.1 封面

- [ ] 包含：学校代码
- [ ] 包含：分类号
- [ ] 包含：密级
- [ ] 包含：UDC
- [ ] 包含：论文题目
- [ ] 包含：作者姓名
- [ ] 包含：专业
- [ ] 包含：研究方向
- [ ] 包含：导师
- [ ] 包含：答辩日期

### 1.2 原创性声明

- [ ] 包含声明正文
- [ ] 包含签名栏
- [ ] 包含日期
- [ ] 位置：封面之后、摘要之前

### 1.3 中文摘要

- [ ] 标题"摘要"两字间空两格
- [ ] 标题：黑体 16 磅加粗居中，单倍行距
- [ ] 标题段前 24 磅、段后 18 磅
- [ ] 内容：宋体 12 磅（小四）
- [ ] 内容：行距 20 磅，段前段后 0 磅
- [ ] **内容字数 ≥ 800 字** ⚠️
- [ ] 关键词 3-5 个
- [ ] 关键词格式：分号分隔，"关键词"三字加粗

### 1.4 英文摘要

- [ ] 标题"Abstract"（Arial 16 磅加粗居中）
- [ ] 内容：Times New Roman 12 磅
- [ ] 行距 20 磅
- [ ] 关键词 3-5 个，"Key Words"加粗
- [ ] 中英文关键词一一对应

### 1.5 目录

- [ ] 标题"目录"两字间空两格（黑体 16 磅加粗居中）
- [ ] 从绪论开始列出
- [ ] 一级节：宋体 12 磅加粗
- [ ] 二级节：仿宋 10.5 磅
- [ ] 页码右对齐
- [ ] 纳入 1-3 级标题

### 1.6 正文各章标题

- [ ] 格式：`# 第X章 ×××`
- [ ] 黑体 16 磅加粗居中
- [ ] 单倍行距，段前 24 磅、段后 18 磅
- [ ] 章序号与章名称之间空两格
- [ ] **每章另起一页（分页符）** ⚠️

### 1.7 标题层级

- [ ] 一级节：`## 1.2 ×××`（黑体 14 磅）
- [ ] 二级节：`### 1.2.1 ×××`（黑体 13 磅）
- [ ] 三级节：`### （1）×××`（宋体 12 磅）
- [ ] 禁止使用 `## 第X章` 混合格式 ⚠️
- [ ] 禁止使用 `### 第X.X` 混合格式

### 1.8 正文段落

- [ ] 宋体 12 磅（英文 Times New Roman 12 磅）
- [ ] 两端对齐，首行左缩进 2 个汉字符
- [ ] 段前段后 0 磅
- [ ] **行距 20 磅** ⚠️
- [ ] **正文中无 `**加粗**` 强调术语** ⚠️

### 1.9 图号图题

- [ ] 图号 + 图题置于图**下方**
- [ ] 宋体 10.5 磅加粗居中
- [ ] 单倍行距，段前 6 磅、段后 12 磅
- [ ] 图注左缩进两字
- [ ] 图中文字宋体 10.5 磅

### 1.10 表号表题

- [ ] 表号 + 表题置于表**上方**
- [ ] 宋体 10.5 磅加粗居中
- [ ] 单倍行距，段前 6 磅、段后 6 磅
- [ ] 表注左缩进两字
- [ ] 表格标题不应加粗

### 1.11 表格

- [ ] **三线表**（顶线、表头底线、底线）⚠️
- [ ] 无竖线
- [ ] 表格内无 `**加粗**` 残留
- [ ] 表头与数据行分明

### 1.12 图表编号

- [ ] 阿拉伯数字分章依序连续编码
- [ ] 格式示例："图 1.1"、"表 3.2"
- [ ] 引用格式自然句式："如图 1.1 所示"（非 `**图 1.1**` 加粗）

### 1.13 注释

- [ ] **页下注**（非上标形式）⚠️
- [ ] 编号：①②③……连续
- [ ] 不得使用上标加序号

### 1.14 参考文献

- [ ] 正文引用：**作者年制**（不得使用上标加序号）⚠️
- [ ] 文后"参考文献"标题左对齐
- [ ] **中英文分编** + **字母排序** ⚠️
- [ ] 中文：宋体小四号全角标点
- [ ] 英文：Times New Roman 小四号
- [ ] 英文作者姓前名后缩写
- [ ] 杂志名书名斜体
- [ ] 同一作者同年多篇加 a/b/c 区分

### 1.15 页码

- [ ] 前置部分（封面/声明/摘要/目录）：罗马数字
- [ ] 正文部分（从绪论起）：阿拉伯数字

### 1.16 附录

- [ ] 标题格式同章标题
- [ ] 内容格式同正文
- [ ] 大写字母 A/B/C 编序号（如"附录 A"）
- [ ] 只有一个附录名为"附录"不编号
- [ ] 图表编号与正文分开（如"图 A.1"）

### 1.17 致谢

- [ ] 标题"致谢"两字间空两格
- [ ] 格式同章标题
- [ ] 内容格式同正文

### 1.18 写作语法

- [ ] 正文段落中无 `**xxx**` 加粗术语 ⚠️
- [ ] 列表项小标题用 `###` 升为正式节标题
- [ ] 表格标题行是普通段落文字
- [ ] 引用图表用自然句式
- [ ] 过渡句式自然，不大量使用 `**xxx领域**` 加粗术语

---

## 二、大纲维度（2 项）

- [ ] 每章内容不超出原定大纲范围（超出章节标★）
- [ ] **每章 ≥ 2 节，每节 ≥ 2 点，每点 ≥ 5 行** ⚠️

---

## 三、内容准确性维度（2 项）

- [ ] 每引用一个理论/数据必须标注来源（无来源结论标⚠️）
- [ ] 正文数据前后无矛盾（矛盾处标⚠️）

---

## 四、查重风险维度（1 项）

- [ ] **连续 ≥ 30 字与参考文献重合标🔴，需 paraphrase** ⚠️

---

## 五、学术规范维度（6 项）

- [ ] **引用标注完整率必须达到 100%** ⚠️
- [ ] 术语首次出现须全称 + 后续简称
- [ ] 正文中文格式：1 人 / 2 人 / 3+ 人
- [ ] 正文英文格式：1 人 / 2 人 / 3+ 人
- [ ] 文后中文格式：期刊 / 专著 / 论文集
- [ ] 文后英文格式：作者姓写在前名缩写在后，杂志名书名斜体

---

## 六、章节完整性（10 项 Guardrails 自动化校验项）

> 以下 10 项由 `scripts/loop_self_check.py` 自动校验。

- [ ] **章节完整性**：必须包含 `# 第1章` 到 `# 第7章` 全部 7 章
- [ ] **字数门槛**：每章 ≥ 100 行（全文 ≥ 700 行）
- [ ] **参考文献存在**：包含 `## 参考文献` 标题
- [ ] **标题层级**：无非法的 `## 第X章` 混合格式
- [ ] **正文加粗**：无 `**xxx**` 残留
- [ ] **引用完整性**：至少 10 处 `(作者，年份)` 模式
- [ ] **三线表**：无多余竖线
- [ ] **表格标题**：表格标题在表上方
- [ ] **合并残留**：无 `===END===` 等残留标识
- [ ] **核心章节关键词**：
  - 第 5 章包含：战略选择 / 竞争战略 / 差异化 / 集中化 / 成本领先 / QSPM
  - 第 6 章包含：实施 / 保障 / 组织 / 人才 / 财务 / KPI / 考核

---

## 七、Word 输出校验（Verification Loop）

> Word 文档输出后由 `loop_self_check.py --verify-docx` 自动校验。

- [ ] **分页符**：每章后有分页符
- [ ] **三线表**：表格为三线表样式
- [ ] **标题层级**：黑体 16/14/13 磅
- [ ] **正文行距**：20 磅
- [ ] **加粗已过滤**：无 `**残留**`
- [ ] **参考文献分编**：中文 + 英文分两块
- [ ] **段落数量**：≥ 500 段
- [ ] **表格数量**：> 0

## 九、违规等级与处理

| 等级 | 标记 | 处理方式 |
|------|------|---------|
| 🔴 P0 | 致命错误（章节缺失、参考文献缺失、合并残留） | **必须修复**，否则阻断 Phase 推进 |
| 🟡 P1 | 严重问题（标题层级错误、加粗残留、字数不足） | **必须修复**，累计 3 项打回 Phase |
| ⚠️ P2 | 一般问题（个别引用缺失、图表编号不规范） | **建议修复**，可在 Phase 5 终审前统一修订 |
| 💡 建议 | 优化项（措辞、过渡句式） | **可选**，可放入 Phase 5.1 去 AI 味 |
