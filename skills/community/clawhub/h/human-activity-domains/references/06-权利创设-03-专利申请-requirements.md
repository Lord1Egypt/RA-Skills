## 专利申请生成要求

专利申请每种任务类型的"零件清单"——必选/可选组件、组装顺序、领域约束。按清单逐项产出。

---

### 1 专利准备

#### T1-1 技术交底

- **必选组件**: 技术方案、技术背景、技术效果、技术附图
- **可选组件**: 技术资料、技术访谈、技术验证
- **组装顺序**: 收集技术资料 → 了解技术背景 → 描述技术方案 → 说明技术效果
- **约束**: 方案必须完整；背景必须清晰；效果必须明确
- **格式**: 技术交底书（Markdown）

#### T1-2 专利检索

- **必选组件**: 检索范围、检索策略、检索结果、检索结论
- **可选组件**: 检索工具、检索数据库、检索报告
- **组装顺序**: 确定检索范围 → 设计检索策略 → 执行检索 → 分析结果 → 形成结论
- **约束**: 范围必须全面；策略必须有效；结论必须客观
- **格式**: 专利检索报告（Markdown）

#### T1-3 专利分析

- **必选组件**: 分析对象、分析维度、分析方法、分析结论
- **可选组件**: 分析工具、分析模型、分析报告
- **组装顺序**: 确定分析对象 → 设计分析维度 → 选择分析方法 → 形成分析结论
- **约束**: 对象必须明确；维度必须全面；方法必须科学
- **格式**: 专利分析报告（Markdown）

#### T1-4 申请策略

- **必选组件**: 申请类型、申请时机、申请地域、申请预算
- **可选组件**: 策略分析、策略评估、策略调整
- **组装顺序**: 确定申请类型 → 选择申请时机 → 确定申请地域 → 编制申请预算
- **约束**: 类型必须匹配；时机必须合适；地域必须明确
- **格式**: 专利申请策略（Markdown）

---

### 2 专利撰写

#### T2-1 权利要求书

- **必选组件**: 独立权利要求、从属权利要求、权利要求布局、权利要求语言
- **可选组件**: 权利要求分析、权利要求优化、权利要求对比
- **组装顺序**: 撰写独立权利要求 → 撰写从属权利要求 → 设计权利要求布局 → 规范权利要求语言
- **约束**: 独立权利要求必须完整；从属权利要求必须引用正确；布局必须合理
- **格式**: 权利要求书（Markdown）

#### T2-2 说明书

- **必选组件**: 技术领域、背景技术、发明内容、具体实施方式
- **可选组件**: 说明书附图、说明书摘要、说明书说明
- **组装顺序**: 撰写技术领域 → 描述背景技术 → 说明发明内容 → 描述具体实施方式
- **约束**: 领域必须明确；背景必须清晰；内容必须完整；实施方式必须详细
- **格式**: 说明书（Markdown）

#### T2-3 摘要

- **必选组件**: 技术方案、技术效果、摘要文字、摘要附图
- **可选组件**: 摘要分析、摘要优化、摘要对比
- **组装顺序**: 提炼技术方案 → 说明技术效果 → 撰写摘要文字 → 选择摘要附图
- **约束**: 方案必须准确；效果必须明确；文字必须简洁
- **格式**: 摘要（Markdown）

#### T2-4 附图

- **必选组件**: 附图类型、附图内容、附图标记、附图说明
- **可选组件**: 附图绘制、附图审核、附图优化
- **组装顺序**: 确定附图类型 → 绘制附图内容 → 标记附图标记 → 撰写附图说明
- **约束**: 类型必须匹配；内容必须清晰；标记必须准确
- **格式**: 附图（图片+Markdown说明）

---

### 3 专利申请

#### T3-1 申请提交

- **必选组件**: 申请文件、申请费用、申请方式、申请确认
- **可选组件**: 申请代理、申请咨询、申请记录
- **组装顺序**: 准备申请文件 → 缴纳申请费用 → 选择申请方式 → 确认申请
- **约束**: 文件必须完整；费用必须缴纳；方式必须正确
- **格式**: 专利申请记录（Markdown）

#### T3-2 形式审查

- **必选组件**: 审查内容、审查标准、审查结果、补正通知
- **可选组件**: 审查咨询、审查沟通、审查记录
- **组装顺序**: 了解审查内容 → 确认审查标准 → 接受审查结果 → 处理补正通知
- **约束**: 内容必须了解；标准必须符合；结果必须接受
- **格式**: 形式审查记录（Markdown）

#### T3-3 实质审查

- **必选组件**: 审查内容、审查标准、审查意见、审查结论
- **可选组件**: 审查咨询、审查沟通、审查记录
- **组装顺序**: 了解审查内容 → 确认审查标准 → 接受审查意见 → 处理审查结论
- **约束**: 内容必须了解；标准必须符合；意见必须答复
- **格式**: 实质审查记录（Markdown）

#### T3-4 答复审查意见

- **必选组件**: 审查意见、答复策略、答复文件、答复提交
- **可选组件**: 答复咨询、答复沟通、答复记录
- **组装顺序**: 分析审查意见 → 制定答复策略 → 准备答复文件 → 提交答复
- **约束**: 意见必须分析透彻；策略必须可行；文件必须规范
- **格式**: 审查意见答复（Markdown）

---

### 4 专利授权

#### T4-1 专利授权

- **必选组件**: 授权通知、授权费用、授权确认、授权证书
- **可选组件**: 授权咨询、授权记录、授权管理
- **组装顺序**: 接收授权通知 → 缴纳授权费用 → 确认授权 → 获取授权证书
- **约束**: 通知必须接收；费用必须缴纳；确认必须及时
- **格式**: 专利授权记录（Markdown）

#### T4-2 专利维护

- **必选组件**: 维护计划、维护费用、维护记录、维护提醒
- **可选组件**: 维护工具、维护团队、维护预算
- **组装顺序**: 制定维护计划 → 安排维护费用 → 执行维护 → 设置提醒
- **约束**: 计划必须可行；费用必须到位；记录必须完整
- **格式**: 专利维护记录（Markdown）

#### T4-3 专利运用

- **必选组件**: 运用方式、运用策略、运用合同、运用收益
- **可选组件**: 运用平台、运用团队、运用预算
- **组装顺序**: 确定运用方式 → 制定运用策略 → 签订运用合同 → 获取运用收益
- **约束**: 方式必须合法；策略必须可行；合同必须规范
- **格式**: 专利运用方案（Markdown）

#### T4-4 专利保护

- **必选组件**: 保护策略、保护措施、保护行动、保护效果
- **可选组件**: 保护团队、保护预算、保护工具
- **组装顺序**: 制定保护策略 → 采取保护措施 → 执行保护行动 → 评估保护效果
- **约束**: 策略必须可行；措施必须有效；行动必须及时
- **格式**: 专利保护方案（Markdown）
