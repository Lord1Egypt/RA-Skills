## Description: <br>
Design and verify API contracts for backward compatibility. Use when handling development work that needs repeatable steps, quality gates, and practical outputs. <br>

This skill is ready for commercial/non-commercial use. <br>

## Use Case: <br>
Developers and teams use this skill to plan and review API contract changes with measurable success criteria, quality gates, and risk-focused next actions. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: The skill is a generic planning aid and should not be the only basis for API backward-compatibility review. <br>
Mitigation: Ask explicitly for schema diffs, breaking-change classification, versioning and deprecation review, consumer impact checks, and compatibility test evidence. <br>
