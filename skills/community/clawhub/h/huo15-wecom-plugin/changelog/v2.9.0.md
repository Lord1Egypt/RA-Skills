# v2.9.0 (2026-05-07)

**动态 agent 派生补全 — 每个企微群/DM 独立 workspace + agentDir**

## 背景

赵博 5/7 反馈："每次企微对话，OpenClaw 都找不到用户的工作目录。"

诊断（详见复盘段）：

- v2.8.x 已经实现动态 agent 派生（`generateAgentId / shouldUseDynamicAgent / ensureDynamicAgentListed`，被 stream-orchestrator / routing-bridge / agent/handler 三处调用）
- **但 `ensureDynamicAgentListed` 实现严重缺陷**：只写 `cfg.agents.list` 加 `{ id: agentId }`，**没写 workspace、没写 agentDir、没写 cfg.bindings**
- 默认 `enabled: false` + 用户配置也没显式启用 → `shouldUseDynamicAgent` 永远返 false
- 即使启用了，缺 `cfg.bindings` 让 OpenClaw 核心 routing 找不到该派生 agent → 整条 inbound 全部 fallback 回 `main`，共用 `~/.openclaw/workspace`

历史现场证据：`ls ~/.openclaw/`：
```
workspace                                                          ← 默认（4/5 - 当下都用这个）
workspace-wecom-default-group-wrgzumeqaa2fbtwcfhmmox2x12t9rlew     ← 4/5 mtime（旧版兜底建过空壳）
workspace-wecom-default-group-wrgzumeqaa88wpxbtrftwzgcgohbxd6g     ← 同上 8 个空壳
...
```

那些群目录是 OpenClaw `resolveAgentWorkspaceDir` 兜底分支（`~/.openclaw/workspace-${id}`）建的，但 binding 缺失没人 dispatch 到——空壳目录。

## 根因

`@wecom/aibot-node-sdk` 与 OpenClaw 核心已经原生支持 `dynamicAgentCreation`（参考 `node_modules/openclaw/dist/extensions/feishu/monitor-DDkD5r4p.js` 内置实现）：

```js
// feishu 派生 agent 的"完整"姿势：
const workspace = resolveUserPath(workspaceTemplate.replace("{userId}", uid).replace("{agentId}", aid));
const agentDir = resolveUserPath(agentDirTemplate.replace("{userId}", uid).replace("{agentId}", aid));
await fs.mkdir(workspace, { recursive: true });
await fs.mkdir(agentDir, { recursive: true });
cfg.agents.list = [...list, { id, workspace, agentDir }];
cfg.bindings = [...bindings, { agentId, match: { channel:"feishu", peer: { kind:"direct", id:senderOpenId } } }];
await runtime.config.writeConfigFile(cfg);
```

wecom 之前写一半就停了，三件不可缺的元素：

| 元素 | v2.8.x | v2.9.0 |
|---|---|---|
| `cfg.agents.list[i].id` | ✅ | ✅ |
| `cfg.agents.list[i].workspace` | ❌ | ✅ |
| `cfg.agents.list[i].agentDir` | ❌ | ✅ |
| `cfg.bindings[]` 路由 entry | ❌ | ✅ |
| mkdir workspace/agentDir | ❌ | ✅ |

## 修复

### 1. `ensureDynamicAgentListed` 升级签名

```diff
- export async function ensureDynamicAgentListed(agentId: string, runtime: any): Promise<void>
+ export async function ensureDynamicAgentListed(params: {
+   agentId: string;
+   chatType: "dm" | "group";
+   peerId: string;
+   accountId: string;
+   runtime: any;
+ }): Promise<void>
```

写入逻辑（`upsertAgentEntryAndBinding`）：

```ts
// cfg.agents.list 写完整 entry
list.push({
  id: agentId,
  workspace: resolveUserPath(renderTemplate(workspaceTemplate, vars)),
  agentDir: resolveUserPath(renderTemplate(agentDirTemplate, vars)),
});

// cfg.bindings 写路由 entry —— 不可缺！
bindings.push({
  type: "route",
  agentId,
  match: {
    channel: "wecom",
    accountId,
    peer: { kind: chatType === "group" ? "group" : "direct", id: peerId },
  },
});
```

### 2. 新增 `channels.wecom.dynamicAgents` 配置

```yaml
channels:
  wecom:
    dynamicAgents:
      enabled: true                                                    # v2.9.0 默认 true（之前默认 false 是 bug）
      workspaceTemplate: "~/.openclaw/workspace-{agentId}"              # 默认值，与 feishu 对齐
      agentDirTemplate: "~/.openclaw/agents/{agentId}/agent"            # 默认值
      maxAgents: 100                                                    # 上限防 cfg 无限膨胀
      adminUsers: []                                                    # 管理员 bypass 派生用 main
      groupEnabled: true
      dmCreateAgent: true
```

模板支持 5 个占位符：`{agentId}` / `{userId}`（同 peerId）/ `{accountId}` / `{peerId}` / `{chatType}`。

### 3. 默认翻 `enabled: true`

之前默认 false → 90% 用户没在 `~/.openclaw/openclaw.json` 显式开启过——生产环境完全没在跑动态 agent。v2.9.0 翻成 true，**老用户升级即用**。

要保持旧行为（所有消息汇到 main）→ 显式配 `channels.wecom.dynamicAgents.enabled: false`。

### 4. 旧 entry 自动迁移

如果 `cfg.agents.list[i]` 已存在但缺 `workspace/agentDir`（v2.8.x 写的旧数据），v2.9.0 第一次入站时自动补齐字段并写 binding。零手工迁移。

### 5. 三个调用点同步改签名

- `src/runtime/routing-bridge.ts:42`
- `src/capability/bot/stream-orchestrator.ts:206`
- `src/agent/handler.ts:740`

全部从 `ensureDynamicAgentListed(targetAgentId, core)` 改成完整 object。

## 复盘 — 为什么 v2.8.x 写一半就停了

v2.8.x 实现 `ensureDynamicAgentListed` 时只写了 `{id}` 没写 binding，注释说 "只把 ID 添加到 OpenClaw 配置"。看上去是参考了**不完整的内部 ad-hoc 实现**（`openclaw-plugin-wecom/dynamic-agent.js` 这个旧 fork），没对照 OpenClaw 核心当下原生支持的 feishu 内置实现 schema。

教训：**派生 agent 是 cfg.agents.list + cfg.bindings 双写**，少一边 OpenClaw 核心都不会 route 过去。今后任何"动态 agent"功能，参考必须是 OpenClaw 核心 `node_modules/openclaw/dist/extensions/<channel>/monitor-*.js` 的内置实现，**不是抄外部 fork**。
