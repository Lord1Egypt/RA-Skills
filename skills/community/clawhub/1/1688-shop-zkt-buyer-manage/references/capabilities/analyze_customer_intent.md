# analyze_customer_intent

分析单个客户的采购意图，包括近期沟通需求分析、采购决策依据，以及根据客户数据自动追加流失分析或商机理由。

## 前置条件

- 已配置 AK
- 推荐传 `--buyer-id-list` 一次查多个客户；仅查一位且只有昵称时，传 `--nick-name` 作为兜底

## 参数

| 参数 | 类型 | 必传 | 说明 |
|------|------|------|------|
| `buyerIdList` | Array<string> | 二选一 | 客户 ID **字符串**数组，多人一次查（首选），如 `["abc","def"]`；后端类型为 `Array<String>`，加密格式 ID 原样透传即可 |
| `nickName` | string | 二选一 | 买家昵称，单查兜底，模糊查询 |

## 返回字段

`data.data[0]` 为匹配到的买家详情，关键字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| `nickName` | string | 买家昵称 |
| `buyerType` | string | `lostRiskType` / `wakeUpType` |
| `recentChatNeedsAna` | JSON | 近期沟通需求分析 |
| `purchaseDecisions` | JSON | 采购决策依据 |
| `lostAnalysis` | JSON | 流失风险分析（仅当 `awakenReason` 为空且 `lostAnalysis` 不为空时展示） |
| `awakenReason` | JSON | 商机唤醒理由（`awakenReason` 不为空时优先展示此区块，不再展示流失分析） |

## 典型用法

```bash
# 多人批量（首选）
python3 {baseDir}/cli.py analyze_customer_intent --buyer-id-list '["id1","id2","id3"]'

# 单人查询（兜底）
python3 {baseDir}/cli.py analyze_customer_intent --nick-name 张三
```

## 展示规则（必须有串场话语，禁止直接堆数据）

### 头部引入语（必须）

markdown 渲染开头必须有一段引导语，告诉用户"接下来要从哪几个维度分析"。推荐模板：

> 先带你从「最近在聊什么」入手，再看「决策关注点」，最后结合客户当前的跟进阶段，给出可参考的判断 👇

### 区块间串场话语（必须）

每个区块在展示数据前，都要有一句"桥梁语"说明这部分数据来自哪里、能帮商家看清什么，避免显得突兀：

| 区块 | 串场话语模板 |
|------|-------------|
| 🟦 近期沟通需求分析 | "这部分来自最近聊天记录的结构化提炼，用来看客户目前关心的核心诉求：" |
| 🟨 采购决策依据 | "接下来看看客户下单前最在意的几个决策点，这些是跟进时优先回应的方向：" |
| 🟥 流失风险分析（仅 `awakenReason` 空且 `lostAnalysis` 不空时展示） | "系统识别到这位客户存在一些流失信号，我们一起看看背后的原因（风险程度已用等级描述，便于快速判断）：" |
| 🟩 商机唤醒理由（`awakenReason` 不空时优先出） | "另外，这位客户在以下几个维度具备唤醒价值，可以作为主动触达的切入点：" |

### 尾部收束语（必须）

数据展示完毕后，追加一句收束语提醒商家：

> 🎯 以上分析都来自最近一段时间的真实互动数据，建议结合客户当前跟进阶段选择最合适的动作。

### 颜色区块规范

- 以 HTML 颜色区块形式展示，不同模块用不同背景色区分
- 🟦 **近期沟通需求分析** — 浅蓝色 `#e0f2fe`
- 🟨 **采购决策依据** — 浅黄色 `#fef3c7`
- 🟩 **商机唤醒理由** — 浅绿色 `#dcfce7`（`awakenReason` 不为空时优先展示，展示后不再展示🟥）
- 🟥 **流失风险分析** — 浅红色 `#fee2e2`（仅当 `awakenReason` 为空且 `lostAnalysis` 不为空时展示）

### 优先级规则（强制，与 SKILL.md 铁律 A6 一致）

**🟩 商机唤醒理由 > 🟥 流失风险分析**，两者即使同时有值也只展示一个：
```
if awakenReason 不为空:
    只渲染 🟩 商机唤醒理由
elif lostAnalysis 不为空:
    只渲染 🟥 流失风险分析
```
- ❌ 禁止两个区块同时渲染
- ❌ 禁止向用户解释"为什么只看到商机理由"

### 数据渲染规范（禁止裸 JSON）

- **数组对象（如 `demands` / `decision_points`）必须渲染为 HTML 表格**，表头使用中文字段名
  - `demands` → 摘要 / 预算 GMV / 需求规模 / 物流要求 / 定制要求
  - `decision_points` → 关注点（`title`）/ 摘要（`summary`）/ 建议动作（`next_step`）/ 重要性（`importance`）
- **嵌套字典按「中文字段名：值」分条输出**，左侧 `<b>...</b>` 加粗
- **纯字符串正文直接输出**，禁止包装成「内容：xxx」这种字段名=值形式
- 空值统一展示为 `—`
- **流失风险分 / 风险分 / riskScore** 等数值字段必须由 cmd.py 自动转换为「极高 / 较高 / 中等 / 较低 / 很低」的程度描述，禁止直接透出原始分数
