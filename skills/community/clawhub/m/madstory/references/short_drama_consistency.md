# 短剧全流程一致性管控方案 (Layer 2 — 深度加载)

> **适用场景**: 单集时长 3~5 分钟的短剧，单条拍摄脚本有效录制时长 > 15 秒时必须启用本方案。
> **加载条件**: 用户选择 Mode 8 短剧创作，且涉及多场景/多集/长时长戏份时加载。
> **核心目标**: 确保全片人物外观/行为逻辑、场景空间/环境参数、跨片段拼接后观感无任何一致性漏洞。

---

## 阶段 A: 前期筹备 — 标准化一致性管控台账

### A1. 人物设定专属档案 (Character Dossier)
每位登场角色必须建立独立档案，包含以下五项不可变记录：

| 档案维度 | 记录内容 | 校验方式 |
|---------|---------|---------|
| 外观细节 | 发型、发色、妆容、服饰搭配、配饰位置、穿着状态 | 逐场次对照档案原文 |
| 性格逻辑 | 核心性格标签（≤3个）、行为倾向、情绪反应模式 | 与台词/动作逻辑交叉验证 |
| 行为习惯 | 标志性动作、口头禅、下意识小动作 | 全片至少出现 2 次以建立角色锚 |
| 台词口径 | 语速特征、用词偏好、禁忌用语 | 跨场次台词风格一致性 |
| 时间线坐标 | 该角色每段戏份的绝对时间位置（集数+秒数范围） | 与全片时间线轴对齐 |

**档案模板**:
```
角色名: [name]
视觉标记: [唯一辨识特征，如"左眼下方泪痣"]
发型/发色: [exact description]
妆容: [exact description]
服饰搭配: [上衣/下装/鞋/外套，含颜色材质]
配饰位置: [每件配饰的精确佩戴位置]
穿着状态: [如"外套披肩不穿袖""袖口挽至肘部"]
性格标签: [≤3个标签]
行为习惯: [标志性动作+频率]
台词口径: [语速/用词特点]
出镜时间线: [Ep1@0:15-0:45, Ep1@2:10-2:55, ...]
```

### A2. 场景全景清单 (Scene Inventory)
每个拍摄场景建立独立编号档案：

| 档案维度 | 记录内容 |
|---------|---------|
| 空间布局 | 房间/场景的平面描述，关键参照物位置 |
| 陈设摆放 | 桌面物件、墙面装饰、地面物品的精确位置与排列 |
| 环境参数 | 光线方向、色温(K值)、天气状态、背景杂音属性 |
| 道具状态 | 每件关键道具的物理状态：磨损程度、摆放角度 |
| 参考素材 | 该场景各角度的实拍参考图编号 |

**场景编号规则**: `SC-[集数]-[场景序号]` 如 `SC-01-02`

### A3. 全片时间线轴 (Master Timeline)
- 将每一段超过 15 秒的长时长戏份按时间节点拆分标记
- 每个标记节点包含：绝对时间位置、关联角色、所在场景、相邻戏份
- 关联前后相邻戏份的一致性要求

**时间线标记格式**:
```
[Ep1@1:30-2:45 | 时长75s | 角色:甲/乙 | 场景:SC-01-02 | 前接:Ep1@0:30-1:25 | 后接:Ep1@2:50-3:30]
  校验节点@10s: 甲右腕手链可见 + 乙外套穿着状态 + 桌面水杯位置(距桌沿5cm)
  校验节点@20s: ...
```

## 阶段 B: 拍摄执行 — 长时长戏份现场一致性核查机制

### B1. 开拍前对照核查 (Pre-Shoot Verification)
每次长时长戏份（>15s）开拍前，逐项核对：
1. 对照人物档案：当前演员外观是否与前次同角色出镜完全匹配
2. 对照场景清单：场景陈设、道具位置是否与同场景上一次记录一致
3. 对照时间线轴：确认当前戏份与前后段的一致性约束要求
4. 全部核对通过后签署"开拍确认"，方可正式录制

### B2. 分片段录制 + 连贯校验流程
- 每 **10~12 秒** 设置一个校验节点（checkpoint）
- 录制第一段时留存该节点的标准参考画面（Reference Frame）
- 后续分段录制前，先对齐 Reference Frame 的所有视觉要素
- 确认对齐后启动下一段录制

### B3. 现场环境参数同步留存
- 每条长戏份全程录制素材时，同步记录每一时间节点的环境参数：
  - 光线色温（K值）、光源方向
  - 现场背景噪音属性（dB 参考值 + 噪音类型）
- 现场快速回放核查：人物动作连贯性、场景元素稳定性
- 发现偏差立即重拍该段，不进入下一段

## 阶段 C: 后期制作 — 跨片段一致性精修校验

### C1. 剪辑阶段核查
- 将拆分录制的长戏份素材拼接后，逐帧核查：
  - 人物动作衔接是否连续（无跳帧）
  - 服饰状态是否一致（扣子/袖口/衣领位置）
  - 场景光影是否统一（同场景光源一致）

### C2. 调色阶段统一
- 同场景所有戏份统一以下参数：
  - 色温（白平衡锁定到同一 K 值）
  - 亮度曲线（同场景使用同一 LUT）
  - 滤镜参数（饱和度/对比度/色相偏移 完全一致）

### C3. 音轨整合阶段对齐
- 同场景的背景杂音参数对齐（噪音类型、音量级）
- 环境音参数一致（混响、空间感）
- 跨片段音画同步检查：无口型延迟、无音效错位

## 阶段 D: 最终验收标准 (Final Acceptance Criteria)

短剧全片一致性验收须满足以下四项硬性要求，任一项不通过即视为不合格：

1. **人物外观无矛盾**: 全片所有 >15s 长戏份中，同一角色的发型/发色/妆容/服饰/配饰/穿着状态完全一致
2. **行为逻辑无冲突**: 同一角色的性格、行为习惯、台词风格在跨场次间保持统一
3. **场景空间无冲突**: 同一场景的空间布局、陈设位置、道具状态全片一致
4. **跨时段拼接自然**: 所有拆分录制的素材拼接后，观感连贯自然

> 验收可借助 `tools/mad-story-scripts/` 中的独立校验工具

# 短剧核心创作方法论 (从 ai-short-drama 迁移)

> 以下方法论适用于 Mode 8 短剧创作的全流程，与上述 A-D 一致性管控方案配合使用。

---

## 阶段 E: 剧本结构化设计

### E1. 剧本黄金结构

短剧为"流量逻辑"服务：强节奏、高密度反转、每帧留钩子、适配竖屏观看、符合平台完播机制。

| 节奏层 | 时长 | 核心要求 |
|--------|------|----------|
| 黄金3秒 | 0-3秒 | 直接抛冲突：危机、羞辱、背叛、绝境、天降机遇。拒绝铺垫 |
| 高频反转 | 每30秒 | 完成一次剧情拐点，"压制→反击→再压制→翻盘"循环 |
| 钩子结尾 | 最后2-3秒 | 预埋悬念：未揭晓身份、即将危机、隐藏伏笔 |

单集规格：1-3分钟，标准为3分钟（约12段×15秒）。

### E2. 剧本输出格式

```
短剧《[剧名]》
题材：[题材类型]
集数：第X集 / 共X集
单集时长：[X]分钟

## 人物表
| 角色 | 身份 | 核心特征 | 视觉标记 |
|------|------|----------|----------|
| [角色名] | [身份] | [一句话人设] | [辨识度标记：泪痣/疤痕/发色等] |

## 第X集 剧本

### 镜头1（0-3秒）[钩子]
- 景别：[特写/近景/中景/全景]
- 画面：[画面描述]
- 台词/旁白：[内容]
- 情绪：[情绪关键词]

### 镜头N（最后2-3秒）[钩子结尾]
- 画面：[悬念画面]
- 情绪：[让观众必须看下一集的情绪]
```

### E3. 输入校验与降级策略

| 用户提供了什么 | 缺失项 | 降级策略 |
|----------------|--------|----------|
| 只有题材关键词 | 集数、时长、角色、冲突 | 读取 `references/short_drama_genres.md` 匹配题材模板，补全标准结构，向用户确认后继续 |
| 有题材+大致剧情描述 | 角色细节 | 自动生成角色卡草案，标注"待确认"，向用户展示后继续 |
| 有完整剧本 | 无 | 直接进入阶段 F（角色卡） |
| 一段小说/故事文本要求改成短剧 | 需要提取剧情骨架 | 执行"小说改编流程"（见 E4） |
| 只说"帮我写个短剧"，无任何信息 | 全部缺失 | 向用户提出2个关键问题：①什么题材/调性？②大概讲什么故事？不自行猜测题材 |
| 提供了现有提示词要求优化 | 提示词质量不明 | 按八要素公式逐项检查，输出"缺失项清单+优化建议" |

### E4. 小说改编流程

当用户提供一段小说/故事文本要求改成短剧时：

1. **提取剧情骨架**：从原文中提取核心冲突线、关键转折点、人物关系，忽略环境描写和内心独白（这些在短剧中靠画面表现）
2. **压缩到短剧节奏**：将小说的缓慢铺压缩为黄金3秒冲突+30秒反转结构，保留最核心的3-5个反转点
3. **转化台词**：将小说的长对话压缩为短剧台词（每句≤15字），大段内心独白转化为旁白或画面情绪
4. **进入阶段 F**：为提取出的角色创建角色卡，后续流程不变

### E5. 题材适配要点

- 重生/逆袭/复仇：突出阶层落差、身份反转、强弱互换的爽感递进
- 甜宠/爱情：每段必有肢体或情感推进，拒绝原地踏步
- 悬疑/推理：每段揭示一个线索同时制造两个新疑问
- 古风/仙侠：世界观设定在黄金3秒内通过画面交代
- 穿越/异能：现代知识碾压异世界的反差感，开场必须秒交代"穿越"设定
- 豪门/商战：密集的利益博弈和权力更迭，每集至少一次身份反转

## 阶段 F: 角色人设与视觉建模

### F1. 角色一致性核心规则

角色"跳脸"（同一角色在不同镜头长相变了）是 AI 短剧最致命的问题：

- 每个主角必须有至少一个**视觉辨识标记**（泪痣、疤痕、固定发色、标志性配饰）——这是给 AI 模型的锚点
- 全剧服装固定，不同场景只换背景不换衣服
- 提示词中每段都重复写"同一角色，服装一致，发型不变"——重复不是啰嗦，是因为 AI 模型在长提示词中对开头信息的注意力会衰减

### F2. 角色卡模板

```
## 角色卡：[角色名]

### 基础设定
- 性别/年龄：
- 身份/职业：
- 性格关键词（3个）：

### 外观描述（AI绘图用）
- 面部：[脸型、五官特征、辨识标记如泪痣/疤痕]
- 发型：[长度、颜色、造型]
- 体型：[身高区间、体态特征]
- 服装：[标志性穿搭，全剧不变]
- 配饰：[固定配饰]

### 三视图关键词
正面：[描述]
侧面：[描述]
背面：[描述]

### 参考图提示词（Seedance/Seedream 格式）
[完整的角色定妆照提示词，包含八要素]
```

### F3. 角色卡示例

```
## 角色卡：陆辰

### 基础设定
- 性别/年龄：男/28岁
- 身份/职业：隐藏身份的顶级家族继承人，目前以普通外卖员身份生活
- 性格关键词：隐忍、冷静、外冷内热

### 外观描述（AI绘图用）
- 面部：棱角分明的国字脸，剑眉星目，右眉角一道一厘米浅疤，肤色偏白
- 发型：黑色短发，利落侧分，露出额头
- 体型：身高185cm，肩宽体挺，精壮不胖
- 服装：全剧统一穿黑色高领毛衣+深灰色西装裤（无论什么场景）
- 配饰：左手腕一块银色机械表

### 三视图关键词
正面：28岁东亚男性，黑色短发侧分，右眉角浅疤，黑色高领毛衣，银色机械表，眼神冷厉
侧面：下颌线清晰，鼻梁高挺，黑色高领毛衣领口可见，短发利落
背面：宽肩窄腰，黑色高领毛衣贴合身形，短发露出后颈

### 参考图提示词（Seedance/Seedream 格式）
（28岁东亚男性，棱角分明的国字脸，剑眉星目，右眉角一道一厘米浅疤，黑色短发侧分露出额头，
黑色高领毛衣，深灰色西装裤，左手腕银色机械表，身高185cm肩宽体挺），
正面半身照，自然光，中景 medium shot，固定镜头 locked-off，
电影感写实风格，4K高清细节丰富，
同一角色服装一致发型不变保持人物样貌与服装一致，面部稳定不变形五官清晰
```

## 阶段 G: 分镜脚本切割

### G1. 分镜核心规则

- 每段只做一个主动作。"缓慢转身微笑"比"转身然后微笑然后挥手"稳定十倍
- 复杂交互动作（开门、上下楼梯、打斗）目前 AI 处理不好，用剪辑衔接绕开
- 动作用"慢""连续""稳定"修饰。速度越慢出片越稳
- 情绪戏→特写+微表情+慢动作；推进戏→推镜+平移+跟拍；高光戏→慢放+景深虚化

### G2. 分镜表格式

| 要素 | 说明 | 示例 |
|------|------|------|
| 段号+时间戳 | 精确到秒 | 镜头3（30-45秒） |
| 景别 | 只选一个 | 特写/近景/中景/全景 |
| 运镜 | 只选一个方向 | 缓推/慢拉/平移/跟拍/固定 |
| 画面内容 | 谁在哪做什么 | 具体到表情和动作 |
| 台词 | 1-2句 | 不超过15字 |
| 情绪 | 关键词 | 震惊/隐忍/得意 |

## 阶段 H: AI 视频/图片提示词生成

### H1. 提示词八要素公式

```
主体（谁）+ 动作（做什么）+ 场景（在哪）+ 光影（什么光）+ 镜头语言（怎么拍）+ 风格（什么调性）+ 画质（多清晰）+ 约束（稳定兜底）
```

这个顺序经过大量实测验证：主体放前面给模型重心，动作第二位提供动态锚点，镜头趁模型锁死构图前到位，风格后面加风味，约束最后兜底。调换顺序会导致模型注意力分配错乱。

### H2. 五种提示词模板

**基础模板**:
```
[角色完整外观描述]，[单一主动作，带速度修饰词]，[具体场景环境]，[光影方向]，
[景别]，[运镜方向+速度]，[风格]，[画质]，同一角色服装一致发型不变保持人物样貌与服装一致，
面部稳定不变形五官清晰动作流畅
```

**单角色情绪镜头**:
```
[角色完整外观：年龄+性别+发型发色+五官标记+服装+配饰]，
[单一微动作，如：缓缓转头看向镜头，嘴角微微上扬]，
[具体场景：如暖光咖啡馆靠窗位置窗外雨天街道]，
[光影：如暖黄柔光golden hour]，
[景别：近景close-up]，[运镜：缓慢推进slow push-in]，
[风格]，[画质]，
同一角色服装一致发型不变保持人物样貌与服装一致，
面部稳定不变形五官清晰动作流畅，画面不抖动不跳帧
```

**角色出场/亮相镜头**:
```
[角色完整外观]，
[出场动作：如缓步走入画面，驻足环顾]，
[具体场景+氛围]，
[光影]，
[景别：中景→全景]，[运镜：缓慢拉远reveal]，
[风格]，[画质]，
同一角色服装一致发型不变保持人物样貌与服装一致，
面部稳定不变形五官清晰动作流畅
```

**双人对话镜头**:
```
[角色A外观]和[角色B外观]，
[互动动作：如面对面站立，角色A微微皱眉，角色B嘴角带笑]，
[具体场景]，
[光影]，
[景别：过肩镜头OTS]，[运镜：固定locked-off]，
[风格]，[画质]，
两个角色各自保持样貌服装一致不混淆，
面部稳定不变形五官清晰，动作流畅
```

**悬念/反转镜头**:
```
[角色外观]，
[关键动作：如猛然睁眼/嘴角勾起冷笑/缓缓转身]，
[场景+氛围变化]，
[光影变化：如从暖光切冷光]，
[景别：特写extreme close-up]，[运镜：极慢推进imperceptible push-in]，
[风格]，[画质]，
同一角色服装一致发型不变保持人物样貌与服装一致，
面部稳定不变形五官清晰动作流畅，画面不抖动不跳帧
```

### H3. 约束兜底词

根据镜头类型选择：

**基础版（3-5秒）**: 同一角色服装一致发型不变，面部稳定不变形五官清晰，动作流畅

**完整版（5-15秒）**: 同一角色服装一致发型不变保持人物样貌与服装一致，面部稳定不变形五官清晰动作流畅，画面不抖动不跳帧，人体结构正常，动作连贯时序清晰

**多角色版**: 两个角色各自保持样貌服装一致不混淆，面部稳定不变形五官清晰，动作流畅，画面不抖动不跳帧

**漫剧版（3-5秒）**: 同一角色服装一致发型不变，线条稳定不抖动，画风统一色彩一致

**漫剧完整版（5-15秒）**: 同一角色服装一致发型不变保持人物样貌与服装一致，线条稳定不抖动画风统一色彩一致，比例协调不变形，画面流畅不跳帧动作连贯时序清晰

## 阶段间状态传递契约

每个阶段的输出必须满足下一阶段的输入要求，不合格则在本阶段内修正：

| 上游→下游 | 输出物 | 下游输入要求 | 校验点 |
|-----------|--------|-------------|--------|
| 剧本→角色 | 人物表 | 每个角色有视觉标记列 | 视觉标记列是否为空 |
| 角色→分镜 | 角色卡 | 外观描述完整（面部+发型+服装+配饰） | 四项是否齐全 |
| 分镜→提示词 | 分镜表 | 每镜头六要素齐全（景别+运镜+画面+台词+情绪+时间戳） | 是否有空项 |
| 提示词→用户 | 提示词 | 八要素齐全，约束兜底词存在 | 逐项核对 |

## 输出质量校验

每完成一个阶段后执行校验，不通过则修正，不进入下一阶段：

- **剧本**：黄金3秒是否有冲突？每30秒是否有反转？结尾是否有钩子？人物表是否有视觉标记？
- **角色卡**：外观描述是否足够具体（不是"漂亮女孩"而是"22岁丹凤眼右眉泪痣黑长直白衬衫"）？三视图是否覆盖？
- **分镜**：每段是否只有一个主动作？是否只指定了一种运镜？段间情绪是否衔接？
- **提示词**：八要素是否齐全？约束兜底词是否存在？主体描述是否和角色卡一致？

## 视觉类型选择

| 维度 | 真人短剧 | 漫剧/动漫短剧 |
|------|----------|---------------|
| 风格关键词 | cinematic realism, 写实 | anime style, cel shading, 漫画风 |
| 约束词重点 | 面部稳定不变形、五官清晰 | 线条稳定、色彩一致、画风统一 |
| 角色参考图 | 真人三视图或定妆照 | 角色设定图（立绘） |
| 常见翻车点 | 换脸、面部变形、肢体异常 | 画风跳变、线条抖动、比例失调 |
| 适用工具 | Seedance 全能参考模式 | Seedance 文生视频 / 漫剧专用Agent |

如果用户没有指定视觉类型，默认询问偏好。如果用户提到"漫剧""动漫风""二次元"，直接走漫剧路线。

## 多集连续剧规则

每集继承上一集的：所有角色卡（外观不变）、风格词（光影/画质/调性统一）、场景资产（同一地点参数一致）。只在分镜层面更新剧情推进。

## 阶段判断逻辑

1. **有创意/题材但没剧本** → 从阶段 E（剧本设计）开始
2. **有剧本需要分镜** → 从阶段 G（分镜切割）开始
3. **有分镜需要提示词** → 从阶段 H（提示词生成）开始
4. **有小说/故事文本** → 执行"小说改编流程"（E4）
5. **需要优化现有提示词** → 按八要素公式逐项检查补全
6. 不确定时直接询问用户需要从哪个阶段开始
