🔧 错误纠正

错误 1：C 件（CSMT）的定义错误

❌ 原文："CSMT 零件 - 采购级零件，因采购商务模式、采购策略、制造场地等原因将装配任务外包至某一供应商处"
✅ 纠正："CSMT 零件 - 委外子件，物权归供应商，由供应商自行采购/管理，用于装配成 HC 总成后交付主机厂"
📖 专家分析：
• Consignment 的本质是"寄售/委外"，物权从头到尾在供应商手中
• 如果 C 是采购级，主机厂就要为 C 做需求计划、承担库存风险——这违背了委外初衷
• 主机厂要的是 HC（委外总成），不是 C（中间散件）
• 正确理解：C 是供应商的子件，HC 才是主机厂的采购级总成

错误 2：毛需求计算规则缺失

❌ 原文：全文未提及毛需求计算规则，容易导致 ERP 配置错误
✅ 补充："毛需求计算规则：只计算 B（外购总成）+ HC（委外总成）+ V（双经销件），C/D/S/M 不计入毛需求"
📖 专家分析：
• 毛需求是 ERP/MRP 系统的核心输入，规则缺失会导致采购计划错误
• C/D/S 都是子零件，其价值已包含在 HC 或上级总成中，重复计算会导致库存积压
• M（自制件）走内部生产计划，不计入外部采购需求
• 建议公式：毛需求 = B + HC + V（或归入 B 类）

错误 3：VAA 与 CSMT 的区分标准模糊

❌ 原文："与 CSMT 零件的区别在于其商务模式及到货地点"
✅ 纠正："VAA 与 CSMT 的核心区别：
• VAA：主机厂必须经手（先买入再转卖），解决专利/商务壁垒问题
• CSMT：主机厂完全不经手（只收 HC 总成），纯装配外包"
📖 专家分析：
• 原文"商务模式及到货地点"太模糊，无法指导实际操作
• 判断标准：主机厂是否需要做 V 的采购计划？是否需要经手买卖？
• VAA：需要（主机厂先买 A，再卖给集成商）
• CSMT：不需要（供应商自己搞定 C，只交付 HC）
• 财务影响：VAA 会产生主机厂的采购成本 + 销售收入，CSMT 只有 HC 的采购成本

⚠️ 关键欠缺

欠缺 1：质量责任划分规则

  • 问题：C 件质量出问题，谁负责？D 件是主机厂指定的，出问题了供应商能否免责？
  • 风险：责任不清导致扯皮，影响生产进度
  • 建议补充：
    • C 件质量责任：100% 由供应商承担（主机厂只验收 HC 总成）
    • D 件质量责任：供应商负责集成质量，主机厂承担指定责任（可按比例分摊）
    • V 件质量责任：主机厂承担采购责任，集成商承担装配责任

欠缺 2：交付风险管控规则

  • 问题：C 件供应商延迟交付，导致 HC 总成无法按时回厂，如何追责？
  • 风险：供应链中断，整车厂停产
  • 建议补充：
    • 供应商必须上报 C/D/S 子件的二级供应商清单
    • 关键 C 件（A 类物料）需要主机厂备案
    • 建立供应链预警机制（供应商库存水位监控）

欠缺 3：成本核算规则

  • 问题：HC 总成的成本如何核算？C 件成本是否透明？
  • 风险：成本黑箱，无法进行成本优化
  • 建议补充：
    • HC 总成成本拆分：C 件成本 + 装配工时 + 管理费 + 利润
    • 供应商需开放 C 件成本结构（至少 A 类物料）
    • 年度成本审查机制（对标市场价）

欠缺 4：变更管理流程

  • 问题：B 件从 C 类改为 D 类（或反之），流程是什么？谁审批？
  • 风险：随意变更导致 BOM 混乱、库存报废
  • 建议补充：
    • 物料类型变更申请流程（工程→采购→制造三方审批）
    • 变更影响评估（库存、在制、已采购）
    • 变更生效时点控制（断点管理）

申请→评估→审批→执行
