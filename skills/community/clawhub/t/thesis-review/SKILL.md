# Thesis Review — 学位论文评审意见生成器

生成专业、结构化、可操作的硕士学位论文评审意见。

## 触发条件

用户满足以下任一情况时自动启用：
- 提供论文 PDF/文本，要求"评审意见"、"评阅"、"评阅书"
- 要求"按照评审模板"、"做一份评审"
- 说"帮我审一下这篇论文"
- 提供论文内容并要求评价/点评

## 工作流

### Step 1：信息收集

从论文中提取/确认以下信息：

| 字段 | 来源 |
|------|------|
| 论文题目 | 封面/第一页 |
| 作者姓名 | 封面 |
| 学校/院系 | 封面 |
| 学位类型 | 硕士/博士/工程硕士 |
| 学科方向 | 摘要/关键词 |
| 页数 | PDF 元数据 |
| 参考文献数 | 参考文献部分统计 |

### Step 2：深度阅读

按以下顺序阅读论文关键部分：

1. **摘要 + 关键词** → 把握核心问题和创新点
2. **绪论/引言** → 了解研究背景、文献综述、论文结构
3. **各章小结** → 了解每章做了什么
4. **结论与展望** → 了解作者自述的贡献和不足
5. **代表性章节** → 选 1-2 章深入阅读理论/方法部分
6. **实验/仿真章节** → 检查实验设计和结果分析
7. **参考文献** → 检查引用质量和规范性

### Step 3：生成评审意见

严格按照以下模板格式输出。

```markdown

**论文题目：** [论文完整题目]
**作者：** [作者姓名]
**学校：** [学校名称]
**学科方向：** [学科/研究方向]
**论文页数：** [X] 页，参考文献 [Y] 篇

---

## 1 论文优点 / 创新点 / 亮点

理论/方法创新：[一句话概括理论或方法上的创新点]
实验/验证充分：[一句话评价实验设计的合理性]
工程应用价值：[一句话评价实际应用价值，填具体解决了什么问题]
写作与逻辑：[一句话评价写作质量]

---

## 2 存在的问题与修改建议（核心部分）

### 关于创新性与贡献

问题描述：[具体问题描述，指出创新点与已有工作的边界不清、贡献不够明确等]
建议：[具体可操作的修改建议]

### 关于理论/方法的严谨性

问题描述：[具体问题，如证明不完整、假设过强、推导有跳跃、缺少必要/充分条件讨论等]
建议：[具体可操作的修改建议，指出应补充什么内容]

### 关于实验/仿真设计

问题描述：[具体问题，如缺少对比实验、参数选择无依据、缺少定量指标、样本量不足等]
建议：[具体可操作的修改建议]

### 关于结果分析与讨论

问题描述：[具体问题，如仅描述现象不分析原因、缺少物理意义解释、数据与结论不对应等]
建议：[具体可操作的修改建议]

### 关于论文写作与规范性

问题描述：[具体问题，如符号不一致、英文摘要错误、图表缺标注、引用不规范等]
建议：[具体可操作的修改建议]

### 其他问题

| 类型 | 位置 | 问题描述 | 修改建议 |
|------|------|----------|----------|
| [错别字/公式编号/格式/单位等] | [页码或章节] | [具体问题] | [具体建议] |

---

## 3 给作者的附加提问

1. [问题 1 — 检验对核心方法的理解]
2. [问题 2 — 检验对实验结果的理解]
3. [问题 3 — 检验对未来工作的思考]

---

## 总体评价

[2-3 句总结性评价，包含：学术水平判断、主要优点、主要不足、是否建议修改后答辩/直接答辩]

**建议结论：** □ 同意答辩  □ 修改后答辩  □ 重大修改后重审  □ 不同意答辩
```

## 评审要点清单

生成评审时，逐项检查以下维度：

### A. 创新性（权重 30%）
- [ ] 是否提出了新概念/新方法/新模型？
- [ ] 与已有文献的边界是否清晰？
- [ ] 创新点是"发现"还是"组合"？
- [ ] 是否填补了某个研究空白？

### B. 学术水平（权重 25%）
- [ ] 文献综述是否全面、时效性好？
- [ ] 理论推导是否严密、假设是否合理？
- [ ] 证明是否完整（充分/必要条件讨论）？
- [ ] 数学/算法描述是否可复现？

### C. 实验验证（权重 20%）
- [ ] 实验设计是否合理？
- [ ] 是否有对比实验（baseline）？
- [ ] 是否报告定量指标（而非仅定性描述）？
- [ ] 参数选择是否有依据？
- [ ] 结果是否可复现？

### D. 结果分析（权重 10%）
- [ ] 是否深入分析了结果背后的机理？
- [ ] 异常现象是否有合理解释？
- [ ] 结论是否与实验数据一致？
- [ ] 局限性是否有讨论？

### E. 写作规范（权重 15%）
- [ ] 结构是否清晰、逻辑是否连贯？
- [ ] 图表是否规范（编号、标题、图例、单位）？
- [ ] 符号是否前后一致？
- [ ] 参考文献格式是否统一？
- [ ] 中英文摘要是否对应？
- [ ] 是否有错别字、格式错误？

---

## 评审语气指南

- **问题描述**：客观陈述，避免主观判断词（如"很差"、"不行"），用"存在不足"、"有待完善"等
- **建议**：具体可操作，避免空话（如"应进一步完善"），用"应补充 XX 实验/分析/对比"
- **总体评价**：先肯定优点，再指出不足，最后给出明确建议
- **提问**：针对论文核心内容，能检验作者真正理解程度，而非泛泛而问

---

## 输出格式要求

1. 输出为 Markdown 格式
2. 问题描述和建议必须一一对应
3. 每个问题至少给出一条具体建议
4. 其他问题用表格列出，便于作者逐条修改
5. 总体评价不超过 200 字
6. 明确给出答辩建议（四选一）

---

## 特殊场景处理

### 论文质量极高
- 减少问题数量（3-4 条即可），增加提问深度
- 总体评价明确"达到优秀硕士学位论文水平"

### 论文质量较差
- 问题分类写满（每个维度都写）
- 总体评价明确"存在较大不足"
- 建议结论选"重大修改后重审"

### 论文非本专业领域
- 在总体评价中注明"非本人研究方向，仅从方法论和规范性角度评阅"
- 减少理论深度方面的批评

### 用户提供的是摘要/片段而非全文
- 在开头注明"仅基于提供的摘要/片段进行初步评审，建议提供全文以获得完整评价"
- 聚焦于可见内容的评审，标注"未看到 XX 章节，无法评价"

## 输出约定

### 必须输出 Word 文档（.docx）

评审意见生成后，**必须**同时输出为 `.docx` 文件，保存到工作目录。

**文件命名规则：**
```
评审意见_[作者姓名]_[YYYY-MM-DD].docx
```
示例：`评审意见_代常乐_2026-05-19.docx`

**Word 文档格式要求：**

| 元素 | 格式 |
|------|------|
| 主标题 | 黑体 二号 居中 |
| 一级标题 | 黑体 三号 左对齐 |
| 二级标题 | 黑体 小三号 左对齐 |
| 正文 | 宋体 小四号 1.5 倍行距 |
| 表格 | 三线表，表头黑体五号，内容宋体五号 |
| 论文信息区 | 宋体 小四号，键值对格式 |

**生成步骤：**
1. 先用 `docx` 技能生成 Word 文档
2. 设置标题样式、正文字体、行距
3. 表格使用三线表格式
4. 保存到工作目录
5. 通知用户文件已生成

**注意：** Markdown 预览仍可在聊天中显示，但 `.docx` 文件是最终交付物。
