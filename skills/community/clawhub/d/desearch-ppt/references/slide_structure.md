# PPT 结构与大纲提炼指南 (v2.0)

当你拿到了经过数据验证和概念创新的 `research_report_final.md` 后，你的任务是将其重构为一份页数合理、重点突出、思想深刻的 PPT 大纲。

---

## ⚠️ 绝对红线：100% 忠于最终报告

**PPT 的所有文字内容必须 100% 来源于 `research_report_final.md`！**
- 绝不允许自行编造内容、数据或案例。
- 绝不允许套用固定的行业分析模板（如强行插入"产业链上中下游"或"技术底座"），必须**完全根据最终报告的实际目录和内容**来组织 PPT 的章节，包括新增的原创概念章节。
- 如果报告是关于"AI教育"，PPT 就必须全是关于"AI教育"；如果报告是关于"新媒体发展"，PPT 就必须全是"新媒体发展"。

---

## 核心任务：如何将任意报告扩展为一份重点突出的 PPT？

### 1. 智能弹性页数 (v2.0 新增)

不再强制固定 100 页。请根据报告内容的丰富程度，智能地决定一个合理的总页数。

**定页原则**：
- **宁缺毋滥**：与其用空洞的内容凑数，不如让每一页都言之有物。
- **不堆砌附录**：不要在报告末尾添加大量的"附录"、"参考文献列表"、"术语表"、"致谢"等填充页面。如果需要参考文献，用 1-2 页即可。
- **参考范围**：通常在 50-80 页之间。如果报告内容极其详实（超过 2 万字），可以规划到 70-80 页；如果报告较为精炼（1 万字左右），50-60 页即可。

**定页方法**：
1. 快速阅读报告，统计其主要章节数量和每章的篇幅。
2. 按以下比例分配页数：

| 模块 | 建议占比 | 说明 |
|------|----------|------|
| 开篇与定调 | 约 10% | 封面、目录、执行摘要、核心结论速览 |
| 背景与现状 | 约 15% | 行业背景、政策环境、市场概况 |
| 核心章节（报告主体） | 约 50% | 报告的核心内容，按章节拆解 |
| 原创概念 | 约 10% | 3-5 个原创概念，每个概念 1-2 页 |
| 挑战与展望 | 约 10% | 风险分析、未来趋势 |
| 结语 | 约 5% | 总结建议、结尾页 |

### 2. 内容拆解法则：一页讲透一个细分点

不要在一页 PPT 里塞入太多内容。

- **错误做法**：一页 PPT 讲完"AI教育的三大应用场景"。
- **正确做法**：第 31 页讲"应用场景总览"，第 32-34 页详细讲"场景一：个性化学习"，第 35-37 页详细讲"场景二：智能批改"，第 38-40 页讲"场景三：虚拟教师"。
- 每页必须有 100-150 字的完整中文句子（3-4 段完整的论述），将原报告中的详细段落直接提炼过来。

### 3. 原创概念页的特殊处理 (v2.0 重点)

为报告中生成的 3-5 个原创概念，**必须**单独设置专门的页面。这些页面应该是整个 PPT 的**思想高地**。

**原创概念页的设计要求**：
- **布局**：优先使用 `Hero Image` 或 `Pyramid Architecture` 布局，使其最具视觉冲击力。
- **内容结构**：
  - 大标题：概念名称（如"技能幻觉"）
  - 副标题：概念的核心定义或公式（如"技能幻觉 = 语言幻觉 + 技能不可用 + 认知错位"）
  - 正文：2-3 个分点，详细阐述概念的内涵
  - 插图：一个极具创意的、能代表这个概念核心思想的手绘插图
- **标记**：在页面右上角或显著位置标注"团队首创概念"字样，突出原创性。

### 4. 数据页的特殊处理 (v2.0 新增)

对于包含经过 `.gov` 验证的数据的页面：
- **布局**：优先使用 `Data Dashboard` 布局。
- **数据来源**：在页脚明确标注数据来源的 `.gov` 网址。
- **视觉设计**：使用仪表盘、趋势图、柱状图等数据可视化元素。

### 5. 布局多样性映射

你必须为每一页指定一种特定的视觉布局。**相邻的两页尽量不要使用相同的布局**。从《prompt_design.md》中的 8 大布局模板中挑选最合适的一个：

| 场景 | 推荐布局 |
|------|----------|
| 封面/封底/大场景展示 | **Hero Image (全景大图)** |
| 每个大章节的开头 | **Chapter Transition (章节过渡)** |
| 发展历史/未来预测 | **Timeline / Roadmap (时间线)** |
| 两种路线对比/中美对比 | **Split-Screen Comparison (对比博弈)** |
| 技术架构/成本结构 | **Pyramid Architecture (金字塔架构)** |
| 市场规模/ROI数据 | **Data Dashboard (数据仪表盘)** |
| 商业闭环/生态系统 | **Circular Flywheel (生态飞轮)** |
| 竞争格局/分类罗列 | **Matrix / Grid (矩阵分布)** |
