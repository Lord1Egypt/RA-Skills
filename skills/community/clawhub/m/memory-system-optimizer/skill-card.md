## Description: <br>
Memory System Optimizer configures an OpenCLAW memory workflow with layered memory, self-reflection, emotion-aware response behavior, task planning, and supporting maintenance scripts. <br>

This skill is ready for commercial/non-commercial use. <br>

## Publisher: <br>
[githubxiaohei](https://clawhub.ai/user/githubxiaohei) <br>

### License/Terms of Use: <br>


## Use Case: <br>
Developers and OpenCLAW users can use this skill to add a file-backed memory system with logging, retention, task context, and operational guidance for agent workspaces. It is most relevant when an agent needs persistent working notes and structured memory maintenance. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: The server security review reports undisclosed payment code in a release that describes itself as free to use. <br>
Mitigation: Review or remove the billing script before installation, and do not configure payment API credentials unless the payment behavior is intended and disclosed. <br>
Risk: The server security review reports broad proactive-agent instructions that can persist conversation-derived data and periodically check personal sources. <br>
Mitigation: Install only in workspaces where persistent memory is expected, and review the memory, autonomy, and security configuration files before enabling automatic reads or writes. <br>
