# 职级评价（任职资格评定）

## 输入与输出

**前置依赖：** competency-dictionary 产出的胜任力词典
**输入：** 被评价人信息、目标职级、行为证据、词典
**输出：** 《个人评价报告》——三因子评分、职级建议、能力差距分析

> **前置检查：** 没有胜任力词典就不做评价。评价必须基于明确的能力标准，否则是无效评价。

```

## Step 2：证据收集

### 2.1 证据类型与来源

| 证据类型 | 来源 | 权重 |
|---------|------|------|
| **行为证据** | BEI访谈、360反馈、项目文档 | 最高（直接行为）|
| **成果证据** | 绩效数据、项目产出、OKR完成情况 | 高（可衡量）|
| **AI效能证据** | AI工具使用日志、项目中AI贡献比例 | 中（AI时代新增）|
| **他人评价** | 直属上级、同事、下游协作方 | 参考 |
| **自我评估** | 被评价人自述 | 参考 |

### 2.2 证据收集模板

对词典中的每一项能力，收集以下证据：

```
能力：[从词典中选择一项能力]
需要达到的Level：[根据目标职级确定]

证据记录表：
┌─────────────────────────────────────────────────────────┐
│ 证据1                                                      │
│ 情境（S）：                                            │
│ 任务（T）：                                            │
│ 行动（A）：                                            │
│ 结果（R）：                                            │
│ 证据类型：□ 行为 / □ 成果 / □ AI效能 / □ 他人评价       │
│ 置信度：□ 高 / □ 中 / □ 低（是否有文档支撑）            │
├─────────────────────────────────────────────────────────┤
│ 证据2 ...                                               │
│ ...                                                      │
└─────────────────────────────────────────────────────────┘
```

### 2.3 AI效能证据的特殊收集方式

AI协同能力是新增维度，需要特别记录：

```
AI效能证据模板：
□ 使用了哪些AI工具（列出工具名）
□ 在该任务中，AI完成了多少比例的工作（估计：__%）
□ 人类在该任务中的不可替代贡献是什么
□ 是否存在AI误用/过度依赖的情况
□ AI输出的质量由谁把关、如何验证

注意：AI效能证据不用于"证明人不如AI"，而是用于
"评估人在人机协作中的判断力和把关能力"
```

## Step 3：Korn Ferry Hay 三因子评分

> 三因子是全球最广泛使用的岗位评价方法。这里将其改造用于**人的评价**（而非岗位评价）。

### 3.1 三因子定义（人的维度）

| 因子 | 人的维度含义 | 对应词典能力 |
|------|------------|------------|
| **Know-How** | 知识、技能、经验的综合水平 | 批判性思维 + 持续学习 + 对应专业能力 |
| **Problem Solving** | 在不确定环境中思考和决策的深度 | 问题解决 + 适应韧性 + AI输出评估 |
| **Accountability** | 对他人和组织的实质影响范围 | 结果导向 + 人际协作（影响范围）|

### 3.2 三因子评分量表

```
Know-How（KH）
  等级   分值    描述
  -------- ----   ----------------------------------------
  KH1     50     执行标准化任务，需要基础指导
  KH2     87     执行多样化任务，需要定期指导
  KH3     125    执行复杂任务，偶尔需要指导
  KH4     165    执行高度复杂任务，几乎不需要指导
  KH5     200    创造新方法，领导领域发展
  [注：数字为Hay标准点值，可用于后续薪酬映射]

Problem Solving（PS）
  PS1 = 15% of KH  （高度结构化环境，重复性问题）
  PS2 = 25% of KH  （有结构但需要判断，常见问题）
  PS3 = 33% of KH  （复杂环境，复杂问题）
  PS4 = 50% of KH  （极度复杂，战略性未知问题）
  PS5 = 75-100% of KH （前沿领域，无先例问题）

Accountability（AC）
  AC1     10%   个人操作员：对自己任务负责
  AC2     25%   小组长：对自己+1-2人负责
  AC3     50%   经理：对自己团队负责
  AC4     75%   总监：跨多个团队/部门负责
  AC5     100%+  VP+：对事业部/公司整体负责
```

### 3.3 综合评分

```
总因子分 = KH点值 + (KH × PS%) + (KH × AC%)

该总分对应到内部职级体系的哪个级别？
→ 通过与同职级同事的分布比较来判断最终职级建议
```

## Step 4：校准会机制

> 评价必须经过校准，单人评价无效。

### 4.1 校准会参与方

```
必须参加：
□ 直属上级（评价第一负责人）
□ 隔级上级（评价监督）
□ HRBP（标准一致性把关）

可选参加：
□ 合作紧密的平级同事
□ 被评价人本人（只汇报结果，不参与讨论）
```

### 4.2 校准会流程

```
时长：每人15-20分钟（如果评价10人，会议约3小时）
流程：

1. 评价人陈述（5min）
   - 评价人陈述三因子评分及依据
   - 朗读关键行为证据

2. 参会人质疑（5min）
   - 其他参会人提问："这个评分是否低估/高估了？"
   - 所有质疑必须有证据支撑，不能是主观印象

3. 讨论并确认（5min）
   - 达成一致评分
   - 记录分歧及理由（如有）

4. 职级建议（2min）
   - 根据校正后评分，对照职级序列表给出职级建议
   - 明确是否建议晋升/维持/降级
```

### 4.3 校准会公平性检查清单

```
□ 是否有同序列跨部门的比较？（防止单一领导"注水"）
□ 证据是否充分（至少有2-3个有效证据才能支持某一能力的某一Level）？
□ AI效能证据是否被适当纳入？（不能因为"用了AI就扣分"）
□ 历史评价数据是否被正确使用？（避免今年和去年矛盾）
```

## Step 6：评价后的反馈机制

### 6.1 向被评价人反馈

```
反馈原则：
□ 反馈在评价完成后5个工作日内进行
□ 反馈必须包含：评分+证据，不只是结论
□ 反馈必须给出具体的1-3条发展建议
□ 允许被评价人提出申诉（有正式申诉通道）
□ AI协同能力的反馈要特别说明"把关能力"，而非"替代人的价值"
```

### 6.2 申诉机制

```
申诉触发条件：被评价人认为评价有明显偏差
申诉时效：收到反馈后10个工作日内
申诉受理：隔级上级 + HRBP 联合重审
申诉结果：维持原评价 / 修改评价 / 新评价（须全体同意）
```

评价数据流向：
┌──────────────┐
│ 个人评价报告  │→ ① 薪酬调整依据
│              │→ ② 晋升决策依据

│              │→ ④ 团队能力地图建设
│              │→ ⑤ 词典本身的有效性验证（聚类分析）

```

---

## 关键原则（红线）

1. **没有词典不评价** — 能力标准必须先于评价存在
2. **没有证据不评分** — 每项能力的评分必须有具体行为证据
3. **没有校准不确认** — 单人评价无效
4. **AI不是扣分项** — 用AI完成工作是现代职场正常能力，不因此扣分
5. **评价公开标准** — 评价标准应对被评价人透明（词典即标准）
6. **评价有保鲜期** — 评价结果在12个月内有效，超期需重评
