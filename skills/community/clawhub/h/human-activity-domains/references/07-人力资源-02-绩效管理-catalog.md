## 绩效管理任务清单

绩效管理按子域组织的任务清单，附任务间关系和元操作映射提示。

**域间逻辑流**：绩效计划 → 绩效实施 → 绩效考核 → 绩效改进

---

### 1 绩效计划

| ID | 任务类型 | 说明 | 依赖 | 元操作映射 |
|----|----------|------|------|----------|
| T1-1 | 目标设定 | 设定绩效目标，包括组织目标、部门目标、个人目标 | 无（入口） | C |
| T1-2 | 指标设计 | 设计绩效指标，包括KPI、OKR、能力指标 | T1-1 | C→A |
| T1-3 | 计划制定 | 制定绩效计划，包括考核周期、考核方式、权重分配 | T1-2 | C→A |
| T1-4 | 计划沟通 | 与员工沟通绩效计划，达成共识 | T1-3 | I |

---

### 2 绩效实施

|----|----------|------|------|----------|

| T2-2 | 绩效辅导 | 辅导员工提升绩效，包括工作指导、能力提升 | T2-1 | I |

| T2-4 | 绩效沟通 | 与员工沟通绩效情况，及时反馈 | T2-3 | I |

## 依赖拓扑摘要

### 一、域内链路

**绩效计划链路**: T1-1 → T1-2 → T1-3 → T1-4
**绩效实施链路**: T2-1 → T2-2 → T2-3 → T2-4
**绩效考核链路**: T3-1 → T3-2 → T3-3 → T3-4
**绩效改进链路**: T4-1 → T4-2 → T4-3 → T4-4

### 二、跨域协同链路

**完整绩效管理链路**: T1-1 → T1-2 → T1-3 → T1-4 → T2-1 → T2-2 → T2-3 → T2-4 → T3-1 → T3-2 → T3-3 → T3-4 → T4-1 → T4-2 → T4-3 → T4-4

**反馈改进链路**: T4-4 → T1-1（改进评估反馈到目标设定）

**持续优化链路**: T4-4 → T1-2（改进评估反馈到指标设计）

更多组合根据具体任务动态推导。
