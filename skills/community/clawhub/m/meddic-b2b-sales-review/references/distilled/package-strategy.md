# 套餐策略与底线思维

## 用途
指导销售团队设计分层套餐（高/中/低档）和思考价格底线。核心不是"怎么打折"，而是"怎么设计每个档位都值得买"。同时建立底线意识——什么情况下宁可不卖。

---

## 为什么要设计套餐

### 对客户的价值
- 降低决策难度（选比不选容易）
- 提供参照物（有了高价锚定，中价显得合理）
- 尊重预算差异（不同客户买得起不同的东西）

### 对我们的价值
- 提高成交率（总有一个档位适合）
- 保护利润率（高价档有溢价，低价档有底线）
- 管理客户预期（套餐 = 边界 = 不是所有东西都包含）
- 避免无底线砍价（"这个价格对应这个方案"）

---

## 套餐设计原则

### 原则一：至少三个档位
- **高档**：全功能 + 优先服务 + 定制化（锚定价，让中档显得合理）
- **中档**：标准功能 + 标准服务（主推款，利润最高）
- **低档**：基础功能 + 自助服务（入门款，降低门槛）

### 原则二：每个档位独立可行
- 每个档位都要能让客户获得明确价值
- 不是"把功能砍到不能用"来区分档位
- 低档不是"阉割版"，是"精简版"

### 原则三：档位差异要可感知
- 客户能清楚说出"高/中/低档的区别是什么"
- 差异不是隐藏在技术细节里，而是在使用体验上
- 最好能用一句话说清楚区别

### 原则四：中档是利润中心
- 高档用来锚定和筛选高价值客户
- 中档是大多数客户的选择，利润率最高
- 低档用来获客和培养未来升级客户

---

## 底线设计

### 金额底线
- 低于这个价格，项目不值得做
- 计算方式：交付成本 + 最低利润率 + 机会成本
- **不是"再低一点也能做"，而是"低于这个线不如不做"**

### 场景底线
- 某些场景天然不适合我们，宁可不接
- 例如：纯价格导向的招标、明显是陪标的项目、客户行业我们不了解
- **不是"什么钱都要赚"，而是"赚该赚的钱"**

### 毛利率底线
- 每个项目/套餐有最低毛利率要求
- 低于底线需要特殊审批
- **不是"打折打到底"，而是"打折要有理由和边界"**

### 用户底线
- 某些类型的客户不值得服务
- 例如：没有决策权的联系人、明显是竞争对手的探子、历史合作有严重问题的客户
- **不是"所有客户都要争取"，而是"选对客户"**

---

## 减法原则

### 做减法比做加法难，但更重要

**减法一：功能减法**
- 不是客户要什么就加什么
- 问"如果不做这个功能，客户能不能用？"如果能，先不做
- 核心功能做到极致 > 什么功能都有但都一般

**减法二：客户减法**
- 不是所有客户都要服务
- 聚焦在最匹配的客户群体
- 服务 10 个好客户 > 服务 100 个差客户

**减法三：承诺减法**
- 不是答应客户所有要求
- 每个承诺都有成本，都有风险
- 少承诺多交付 > 多承诺少交付

**减法四：价格减法**
- 不是越便宜越好
- 价格战是最后手段，不是第一选择
- 价格反映价值，降价 = 降价值认知

## 讨论模板

在设计套餐或讨论定价时，用以下模板：

### 模板一：套餐设计讨论
```
1. 目标客户分几类？每类的核心需求是什么？
2. 每个套餐包含什么？不包含什么？
3. 三个套餐之间的价格梯度是多少？（建议 2-3 倍）
4. 客户选每个套餐的理由是什么？
5. 如果客户选了最低档，我们的利润率是多少？
```

### 模板二：底线讨论
```
1. 这个项目的最低报价是多少？怎么算出来的？
2. 低于什么价格我们就不做了？为什么？
3. 如果客户要求大幅砍价，我们能砍什么？不能砍什么？
4. 这个项目的最低可行方案是什么？客户能接受吗？
5. 如果赢了但利润很低，对我们的长期影响是什么？
```

### 模板三：价格谈判准备
```
1. 客户的预算范围是多少？我们怎么知道的？
2. 我们的报价在客户预算的什么位置？
3. 如果要降价，能降多少？用什么换？（功能、服务、周期）
4. 降价值不值得？换来的客户价值是什么？
5. 底线在哪？突破底线的后果是什么？
```

---

## 在 Review 中的使用

当 review 定价和套餐策略时：

1. "这个客户的套餐是哪个档位？为什么选这个？"
2. "报价的底线在哪？你有没有突破底线？"
3. "如果客户要砍价，你的策略是什么？"
4. "这个项目的利润率是多少？符合我们的要求吗？"
5. "如果这个价格接了，对后续同类客户有什么影响？"

---

## 关键原则

1. **套餐不是打折工具**：每个档位都是完整的价值交付
2. **底线不是谈判起点**：底线是最后防线，不是报价策略
3. **做减法比做加法更重要**：少做、做精、做到位
4. **价格反映价值**：降价 = 降价值认知，慎重使用
5. **每个决策都有连锁反应**：这个客户的价格会影响所有同类客户的预期
