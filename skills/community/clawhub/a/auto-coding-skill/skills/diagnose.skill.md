# Diagnose：系统化调试

## 1. Overview（概述）

调试不是猜谜。本技能用 6 阶段系统化流程替代「我感觉是这里」的随机猜测。每个阶段建立可验证的反馈循环，直到根因被确认。

**核心原则**：
- 没有可复现 = 没有 bug（先建反馈循环）
- 假设 → 验证 → 排除 → 下一个假设（不可跳步）
- 修复前先写回归测试（先再现 bug）
- 调试代码必须清理（不能留在生产里）

## 2. When to Use（使用条件）

**触发条件**：
- 测试失败且原因不明
- 行为不符合预期且不是明显的语法/类型错误
- Reviewer 否决权触发重写时
- 用户报告 bug 需要复现

**不可用调试**：
- 明显的语法/类型错误（直接修，不需要假设）
- 已知的错误模式（有现成方案）

## 3. Process（执行流程）

### 3.1 六阶段流程图

```
阶段 1: 建立反馈循环（5 秒内可复现 → 进入阶段 2）
  ├─ 找到最小复现步骤
  ├─ 自动化复现（一行命令或一个测试）
  ├─ ❌ 禁止：描述现象后直接猜测原因
  └─ 🚩 如果 5 分钟内无法复现 → 报告「不可复现」并终止

阶段 2: 精确复现（确认输入 → 输出 → 期望的差异）
  ├─ 记录：输入数据、环境状态、预期输出、实际输出
  ├─ 缩小范围：二分法定位（哪一行/哪个条件开始出错）
  └─ ❌ 禁止：范围没缩小就列假设

阶段 3: 列出假设（≥2 个，排优先级）
  ├─ 假设 A: {根因} → 概率 {高/中/低} → 验证方法
  ├─ 假设 B: {根因} → 概率 → 验证方法
  ├─ 排序原则：1) 常见度 2) 验证成本 3) 影响范围
  └─ ❌ 禁止：只有一个假设就开始插桩

阶段 4: 插桩验证（每个假设用最简单的方法验证）
  ├─ 打印变量值（不是用 debugger 逐步跟踪）
  ├─ 对比预期值 vs 实际值
  ├─ 每个假设验证后明确结论：确认 / 排除 / 不确定
  └─ ❌ 禁止：同时验证多个假设（混淆结果）

阶段 5: 修复（确认根因后）
  ├─ 先写回归测试（证明修复后 bug 不会再出现）
  ├─ 用最少代码修复（不顺手重构）
  ├─ 运行全量测试（确认修复没有引入回归）
  └─ ❌ 禁止：修复 + 重构混在一起

阶段 6: 清理 + 记录
  ├─ 删除所有调试代码：grep "[DEBUG-] -- 确认无残留
  ├─ 记录根因（写入提交信息或文档）
  └─ 复盘：为什么第一次没发现？（改进预防）
```

### 3.2 反例：不该怎么做

```
❌ "我觉得是空指针，我先加个 null check 试试"
   → 跳过阶段 1-3，直接猜 + 改 = 可能修了症状没修根因

❌ "让我用 debugger step through 整个流程"
   → 没有假设的逐步跟踪 = 浪费时间，不知道在找什么

❌ 加上 try-catch 掩盖错误后说"修好了"
   → 掩盖 ≠ 修复，根因还在
```

## 4. Risk Scorecard（反借口表）

| # | 纪律 | 常见借口 | 反驳 |
|---|------|---------|------|
| D1 | 先建反馈循环 | "我大概知道问题在哪，直接改" | 不建循环就猜 = 瞎蒙 |
| D2 | 列 ≥2 个假设 | "只有一个可能的原因" | 只有一个可能 = 你没想够 |
| D3 | 假设优先于插桩 | "打日志太慢了，直接 debugger" | 没假设的 debugger = 大海捞针 |
| D4 | 先写回归测试 | "改一行而已，不用测试" | 一行也能引入新 bug |
| D5 | 清理调试代码 | "这个 print 说不定以后有用" | 调试代码不是日志；下次会误导 |

## 5. Red Flags（危险信号）

- 🚩 5 分钟内无法复现 → 可能不是代码 bug（环境/数据问题）
- 🚩 同一个假设验证了 ≥3 次 → 你可能在确认偏见而非验证
- 🚩 修复方案影响 > 3 个文件 → 可能是设计问题而非实现 bug
- 🚩 "我觉得这样就能修" → 没有验证的"觉得" = 没有修复
- 🚩 调试时间 > 15 分钟 → 召回主会话，可能需要重新设计方案

## 6. Verification（自检清单）

**必检（≤5 项）**：
- [ ] 反馈循环已建立（bug 可在 5 秒内复现）
- [ ] 假设已列出并排优先级（≥2 个，有验证方法）
- [ ] 回归测试已编写（修复前，先复现 bug）
- [ ] 调试代码已清理（grep "[DEBUG-]" 无残留、无注释掉的打印语句）
- [ ] 根因已明确记录（提交信息或阶段日志）

**抽检（≤3 项）**：
- [ ] 调试耗时 < 5 分钟（否则应报告主会话）
- [ ] 复盘结论已输出（为什么第一次测试没抓到？）
- [ ] 修复只涉及最小范围（未顺手重构）

**免检**：调试工具选择、Log 格式
