## Core Accounting Logic

### 1. 凭证生成规则

```
【采购入库】
借：库存商品        贷：应付账款/银行存款

【销售出库】
借：应收账款/现金    贷：主营业务收入
    应交税费-销项税

同时结转成本：
借：主营业务成本    贷：库存商品

【费用支出】
借：管理费用/销售费用  贷：银行存款/现金

【收款到账】
借：银行存款/现金    贷：应收账款
```

### 2. 增值税计算

```
【一般纳税人】
应交增值税 = 销项税额 - 进项税额
销项税额 = 不含税销售额 × 税率（13%/9%/6%）
进项税额 = 不含税采购额 × 税率

【小规模纳税人】
应交增值税 = 销售额 × 征收率（1%/3%）
```

### 3. 成本核算方法

```
【加权平均法】
加权平均单价 = (期初金额 + 本期入库金额) / (期初数量 + 本期入库数量)
发出成本 = 发出数量 × 加权平均单价

【毛利率法】
预计毛利率 = 毛利 / 销售收入
销售成本 = 销售收入 × (1 - 毛利率)

【个别计价法】
成本 = Σ(每批发出数量 × 该批实际单价)
```

### 4. 往来账龄分析

| 账龄 | 计提比例 | 备注 |
|------|----------|------|
| 1-6个月 | 0% | 正常 |
| 7-12个月 | 5% | 关注 |
| 1-2年 | 20% | 预警 |
| 2-3年 | 50% | 风险 |
| 3年以上 | 100% | 坏账 |

### 5. 银行对账步骤

```
第一步：获取银行对账单与企业账数据
第二步：标记双方已相符记录
第三步：识别未达账项：
  - 企业已收银行未收（+）
  - 企业已付银行未付（-）
  - 银行已收企业未收（+）
  - 银行已付企业未付（-）
第四步：编制余额调节表
第五步：追踪未达账项原因
```

## Execution Workflow

### 【处理一笔采购业务的完整流程】

```
Step 1: 接收采购信息
  - 供应商名称
  - 商品名称、数量、单价
  - 是否开票（税率）

Step 2: 生成采购入库凭证
  借：库存商品        XX,XXX（不含税金额）
      应交税费-进项税  XXX（含税金额-不含税）
      贷：应付账款      XX,XXX（含税金额）

Step 3: 登记进货录入表
  - 日期、单号、供应商
  - 商品、数量、单价、金额
  - 已付款、未付款

Step 4: 处理进项发票
  - 发票号码、金额、税率
  - 更新认证状态

Step 5: 如付款，生成付款凭证
  借：应付账款        XX,XXX
      贷：银行存款      XX,XXX

Step 6: 更新供应商对账表
  - 增加本期应付
  - 记录付款情况
```

## Data Validation Rules

### 必须校验的数据关系
```
✓ 出库数量 ≤ 当前库存
✓ 发票价税合计 = 不含税金额 + 税额
✓ 税额 = 不含税金额 × 税率
✓ 付款金额 ≤ 应付余额
✓ 收款金额 ≤ 应收余额
✓ 每笔凭证借贷必相等
```

### 异常情况识别
```
⚠️ 同一供应商连续大额采购
⚠️ 客户集中度超过30%
⚠️ 毛利率异常波动超过±20%
⚠️ 库存周转天数超过行业平均2倍
⚠️ 费用占比超过收入10%
⚠️ 银行流水与账面不符
```

## Common Excel Formulas Reference

```excel
// 库存实时计算
=期初库存+SUMIF(进货!商品列,商品,进货!数量列)-SUMIF(出货!商品列,商品,出货!数量列)

// 应收账款余额
=期初应收+SUMIF(出货!客户列,客户,出货!金额列)-SUMIF(收支!往来列,客户,收支!收入列)

// 销售毛利
=销售收入-销售数量*VLOOKUP(商品,商品资料表,参考进价列,FALSE)

// 税前利润
=营业收入-营业成本-销售费用-管理费用-财务费用

// 杜邦ROE
=(净利润/营业收入)*(营业收入/总资产)*(总资产/所有者权益)

// 资产负载率
=总负债/总资产

// 流动比率
=流动资产/流动负债
```

## Notes

1. **借贷必相等**：每笔凭证借方合计 = 贷方合计
2. **发票时效**：进项发票须在360天内认证
3. **成本方法**：确认后在年度内不得随意变更
4. **审计留痕**：所有调整须附调整说明
5. **数据备份**：月末关账前务必备份数据
6. **上市公司合规**：须符合财政部会计准则要求
