# Eric's Second Brain

> 左脑工程 × 右脑人生 × 创造脑 AI：一个把 Eric 已有技能体系融会贯通的元认知操作系统。

Eric's Second Brain 不是单一领域专家，而是一个元技能：它负责把问题拆成工程、人生认知、AI/创新三个维度，按需路由到对应知识层，再把多个视角合成为一个可执行答案。

## 核心能力

| 能力 | 说明 |
|------|------|
| 三脑协同 | 左脑负责工程诊断，右脑负责人性与认知判断，创造脑负责 AI/创新方案 |
| 跨域路由 | 判断问题属于纯工程、纯人生、AI/创新，还是跨域混合问题 |
| 合成输出 | 使用主辅、并行、递进、反馈四种合成方式，把多域答案编织成统一建议 |
| Eric 经验层 | 注入 Eric 12 年汽车零部件研发、AI 实践、内容运营和技能工程经验 |
| 技能体系地图 | 连接 Engineering Wisdom、Life Wisdom、AI Agent、CBI、FTB 与 ima 工具链 |

## 适用场景

- 工程问题不只是技术问题，还牵涉客户、老板、资源和决策。
- 人生选择不只是情绪问题，还牵涉资产配置、时代趋势和能力结构。
- AI 工具不只是效率工具，还牵涉流程重构、技能化、自动化和内容生产。
- 用户的问题描述模糊，需要先判断“这到底是哪类问题”。

典型触发：

```text
这个技术路线到底该不该继续？
我想把工程经验做成 AI 技能，该怎么拆？
团队效率低，是流程问题还是人的问题？
我该继续做汽车零部件，还是转向 AI+制造业？
这个产品创新方向怎么判断？
```

## 安装

### ClawHub

```bash
clawhub install erics-second-brain
```

### 手动安装

```bash
git clone https://github.com/Shun1989/erics-second-brain.git ~/.workbuddy/skills/erics-second-brain
```

重启 WorkBuddy 后即可使用。

## 与前两个技能的关系

- `erics-engineering-wisdom`：左脑，处理工程技术问题。
- `erics-life-wisdom`：右脑，处理人生、财富、职业、认知问题。
- `erics-second-brain`：总控入口，负责判断问题类型、调用多脑视角、合成答案。
