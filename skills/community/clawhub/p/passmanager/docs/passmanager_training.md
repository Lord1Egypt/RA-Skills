## 🔒 密码管理原则

### 2.1 核心原则
#### 2.1.1 零信任原则
- **默认不信任**: 所有请求都需要验证
- **最小权限**: 按需分配访问权限
- **持续验证**: 每次访问都需要验证

#### 2.1.2 加密存储原则
- **本地加密**: 所有密码本地AES-256加密存储
- **无明文存储**: 禁止在任何地方存储明文密码
- **定期轮换**: 重要密码定期更新

### 2.2 禁止行为
🚨 **严格禁止以下行为**:
- ❌ 在代码中硬编码密码
- ❌ 在配置文件中存储明文密码
- ❌ 通过聊天工具发送密码
- ❌ 共享个人账户密码
- ❌ 使用弱密码（如123456）
- ❌ 重复使用相同密码
- ❌ 将密码记录在纸质文档

### 2.3 安全要求
✅ **必须遵守的要求**:
- ✅ 所有密码必须通过PassManager管理
- ✅ 使用后立即清除内存中的密码
- ✅ 记录所有密码访问日志
- ✅ 定期审计密码使用情况
- ✅ 发现安全问题立即报告

## 📖 操作指南

### 4.1 安装与初始化

#### 4.1.1 环境检查
```bash
# 检查Python版本
python3 --version  # 需要3.8+

# 检查SQLite版本
sqlite3 --version  # 需要3.35+

# 检查OpenClaw版本
openclaw --version  # 需要2026.3.7+
```

#### 4.1.2 系统初始化
```bash
# 1. 初始化数据库
python3 passmanager.py init

# 2. 添加管理员助手
python3 passmanager.py add-assistant --name "管理员" --level admin

# 3. 添加初始密码
python3 passmanager.py add \
  --service "email" \
  --key "example@company.com" \
  --value "[示例密码]" \
  --notes "示例邮箱账户" \
  --expires-days 180

# 4. 验证安装
python3 passmanager.py status
```

### 4.2 日常操作

#### 4.2.1 密码查询（最常用）
```bash
# 查询单个密码
python3 passmanager.py get --service "email" --key "example@company.com"

# 查询服务下所有密码
python3 passmanager.py list --service "email"

# 搜索密码
python3 passmanager.py search --query "api"

# 查看密码详情
python3 passmanager.py detail --service "email" --key "example@company.com"
```

#### 4.2.2 密码管理
```bash
# 添加新密码
python3 passmanager.py add \
  --service "new_service" \
  --key "username" \
  --value "[示例密码]" \
  --tags "production,api" \
  --notes "生产环境API密钥"

# 更新密码
python3 passmanager.py update \
  --service "email" \
  --key "example@company.com" \
  --value "[新密码]"

# 删除密码
python3 passmanager.py delete \
  --service "old_service" \
  --key "deprecated_key"

# 标记密码过期
python3 passmanager.py expire \
  --service "email" \
  --key "old_account"
```

#### 4.2.3 助手管理
```bash
# 查看自己权限
python3 passmanager.py my-permission

# 查看所有助手
python3 passmanager.py list-assistants

# 添加新助手
python3 passmanager.py add-assistant \
  --name "新助手" \
  --level user \
  --notes "新加入的助手"

# 修改助手权限
python3 passmanager.py update-assistant \
  --name "新助手" \
  --level admin

# 禁用助手
python3 passmanager.py disable-assistant --name "离职助手"
```

### 4.3 高级操作

#### 4.3.1 批量操作
```bash
# 批量导入密码（CSV格式）
python3 passmanager.py import \
  --file "passwords.csv" \
  --format csv

# 批量导出密码
python3 passmanager.py export \
  --output "passwords_backup.json" \
  --format json \
  --encrypted

# 批量更新过期密码
python3 passmanager.py update-expired \
  --days 90 \
  --notify
```

#### 4.3.2 审计与监控
```bash
# 查看访问日志
python3 passmanager.py logs \
  --days 7 \
  --assistant "管理员"

# 生成审计报告
python3 passmanager.py audit-report \
  --output "weekly_audit.md" \
  --period week

# 检查密码强度
python3 passmanager.py check-strength

# 检查过期密码
python3 passmanager.py check-expiry
```

#### 4.3.3 系统维护
```bash
# 备份数据库
python3 passmanager.py backup \
  --comment "日常备份"

# 恢复数据库
python3 passmanager.py restore \
  --backup-file "backup_示例日期.db"

# 优化数据库
python3 passmanager.py optimize

# 查看系统状态
python3 passmanager.py status --detail
```

## 🔍 常见问题解答

### 7.1 操作问题
#### Q1: 查询密码返回"权限不足"
**A**: 检查助手权限级别，联系管理员提升权限

#### Q2: 添加密码失败
**A**: 检查服务名和键名是否已存在，或联系管理员

#### Q3: 系统响应缓慢
**A**: 可能是数据库需要优化，运行`passmanager.py optimize`

### 7.2 安全问题
#### Q1: 怀疑密码泄露怎么办？
**A**: 立即运行应急处理流程，更改密码并报告管理员

#### Q2: 发现异常访问日志？
**A**: 立即报告管理员，暂停相关账户访问

#### Q3: 如何确保密码安全？
**A**: 遵守安全规范，定期更换密码，监控访问日志

### 7.3 技术问题
#### Q1: 数据库损坏怎么办？
**A**: 使用最新备份恢复，联系技术支持

#### Q2: 如何迁移到新服务器？
**A**: 使用备份恢复功能，确保加密密钥一致

#### Q3: 如何扩展系统？
**A**: 支持集群部署，参考部署指南
