## AT0 自动化协调域 范本

### 范本概述
- **域ID**: AT0
- **域名称**: 自动化协调域
- **任务数量**: 4
- **范本状态**: 可用

### 任务执行范本

#### AT0-01 自动化需求分析
**执行步骤**:
1. 接收自动化请求，明确自动化目标
2. 分析自动化范围和需求
3. 识别自动化约束
4. 评估自动化优先级
5. 汇总自动化需求

**输入示例**:
- 自动化请求：需要自动化执行每日数据备份任务
- 自动化目标：确保数据安全，减少人工操作
- 自动化范围：数据库备份、文件备份、日志备份

**输出示例**:
- 自动化目标：确保数据安全，减少人工操作
- 自动化范围：数据库备份、文件备份、日志备份
- 自动化需求：需要定时执行备份任务，支持多种备份方式
- 自动化约束：需要在业务低峰期执行，不能影响正常业务

**注意事项**:
- 自动化目标要明确，避免模糊
- 自动化范围要合理，避免过宽或过窄
- 自动化需求要全面，避免遗漏

#### AT0-02 自动化域选择
**执行步骤**:
1. 分析自动化需求特征
2. 扫描可用自动化域列表
3. 匹配自动化需求与自动化域能力
4. 选择最合适的自动化域组合
5. 确定自动化域选择理由

**输入示例**:
- 自动化需求：需要自动化执行每日数据备份任务
- 可用自动化域：任务定义域、调度设置域、执行策略域、监控管理域

**输出示例**:
- 选择自动化域：任务定义域 + 调度设置域 + 执行策略域 + 监控管理域
- 选择理由：数据备份需要完整的自动化流程
- 自动化域组合方案：任务定义域负责任务定义，调度设置域负责调度配置，执行策略域负责执行策略，监控管理域负责监控管理

**注意事项**:
- 自动化域选择要基于实际需求
- 避免选择过多或过少的自动化域
- 自动化域组合要合理，避免重叠

#### AT0-03 执行顺序确定
**执行步骤**:
1. 分析自动化任务依赖关系
2. 识别关键路径
3. 确定执行顺序
4. 识别并行执行机会
5. 绘制执行顺序图

**输入示例**:
- 自动化任务：任务定义、调度设置、执行策略、监控管理
- 依赖关系：调度设置依赖任务定义，执行策略依赖调度设置，监控管理依赖执行策略

**输出示例**:
- 执行顺序：任务定义 → 调度设置 → 执行策略 → 监控管理
- 关键路径：任务定义 → 调度设置 → 执行策略 → 监控管理
- 并行机会：任务定义和需求分析可并行执行

**注意事项**:
- 依赖关系要准确，避免循环依赖
- 关键路径要识别，避免延误
- 并行机会要充分利用，提高效率

#### AT0-04 结果整合与优化
**执行步骤**:
1. 收集各自动化任务执行结果
2. 验证结果完整性
3. 整合相关结果
4. 优化整体输出
5. 汇总整合报告

**输入示例**:
- 任务定义结果：已定义数据备份任务
- 调度设置结果：已配置每日凌晨2点执行
- 执行策略结果：已配置重试策略和超时控制
- 监控管理结果：已配置状态监控和告警通知

**输出示例**:
- 整合结果：完整的数据备份自动化系统
- 优化建议：优化备份策略，提高备份效率
- 整合报告：详见附件

**注意事项**:
- 结果收集要全面，避免遗漏
- 结果验证要严格，避免质量问题
- 整合要保持一致性，避免冲突

### 范本特征摘要
- **结构特征**: 按域分组，每个域包含多个任务，每个任务有明确的执行步骤
- **风格特征**: 专业、清晰、结构化，使用表格和列表展示信息
- **逻辑特征**: 从需求分析到结果整合，逻辑清晰，步骤明确
- **格式特征**: 使用Markdown格式，包含标题、列表、表格等元素
