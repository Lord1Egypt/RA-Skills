# 科研课题成果汇编 Skill

## 一、技能定位

本技能专门用于**科研课题成果的系统性汇编整理**。用户提供课题资料（或仅提供课题名称），Skill 将：

1. **分析课题性质**，识别成果类型
2. **构建总体框架**，提交用户确认
3. **逐项拆分深化**，网络资料补充
4. **全文合并整合**，输出 Word 文档

语言原则：**简洁、精准、聚焦创新性**，不堆砌废话，每句话都承载实质内容。

---

## 二、五阶段工作流

---

### Phase 1 — 课题信息摄入与解析

收到用户资料后，完成以下解析：

#### 1.1 基本信息提取

```
【课题基本信息】
- 课题名称：
- 课题类别：（国家级 / 省部级 / 市厅级 / 校级 / 企业委托 / 其他）
- 所属领域：（自然科学 / 社会科学 / 工程技术 / 人文艺术 / 交叉学科）
- 研究周期：
- 主持单位 / 主持人：
- 已有资料清单：（用户提供的文件、数据、报告等）
```

#### 1.2 成果类型识别

根据课题性质，自动识别本课题可能产出的**成果类型**：

| 成果类型 | 适用场景 | 对应框架 |
|---------|---------|---------|
| 研究报告 | 所有课题必备 | 标准学术研究报告框架 |
| 政策建议报告 | 社会科学、应用研究 | 政策简报框架 |
| 技术报告 / 规范 | 工程技术类 | 技术文件框架 |
| 学术论文摘要集 | 有发表成果 | 摘要汇编框架 |
| 专著 / 编著章节 | 理论研究类 | 学术专著框架 |
| 数据集说明文档 | 有数据成果 | 数据集元数据框架 |
| 专利说明摘要 | 有发明专利 | 专利摘要框架 |
| 成果转化说明 | 有应用落地 | 应用案例框架 |

#### 1.3 网络资料补充策略

- 在线检索同类课题的公开成果报告（中国知网、万方、国家社科基金数据库等）
- 参考本领域权威机构发布的成果框架模板
- 补充与课题相关的最新政策文件、行业标准、统计数据

---

### Phase 2 — 总体框架构建（⚠️ 须用户确认后方可进入 Phase 3）

#### 2.1 框架呈现格式

为课题所有成果构建**总体框架清单**，格式如下：

```
═══════════════════════════════════════
【课题名称】成果汇编总体框架
═══════════════════════════════════════

成果一：研究报告
├── 第一章 绪论
│   ├── 1.1 研究背景与问题提出
│   ├── 1.2 研究目的与意义
│   ├── 1.3 国内外研究综述
│   └── 1.4 研究方法与技术路线
├── 第二章 [课题核心内容章节]
│   ├── 2.1 ...
│   └── 2.2 ...
├── 第三章 研究发现与创新点
│   ├── 3.1 主要研究发现
│   └── 3.2 理论创新 / 方法创新 / 实践创新
├── 第四章 对策建议 / 应用方案
└── 第五章 结论与展望
    ├── 5.1 主要结论
    ├── 5.2 研究局限
    └── 5.3 未来研究方向

成果二：政策建议报告（如适用）
├── 执行摘要（300字）
├── 问题分析
├── 政策建议（3-5条，每条含依据）
└── 预期效果

成果三：...
───────────────────────────────────────
【框架说明】
- 共 X 项成果，总计约 X 个板块
- 预计总字数：约 X 万字
- 建议撰写顺序：...
═══════════════════════════════════════

⚠️ 请确认以上框架：
   A. 框架已确认，开始拆分深化
   B. 需要调整（请说明调整内容）
   C. 删除/合并某些成果项
```

**Phase 2 结束后，必须等待用户确认，收到确认指令后才进入 Phase 3。**

---

### Phase 3 — 逐项拆分与内容深化（循环执行）

每次只处理**一个板块**，完成后提示用户是否继续下一板块。

#### 3.1 单板块处理流程

```
Step A：定位板块在总框架中的位置与功能
Step B：提取用户已有资料中与本板块相关的内容
Step C：在线补充资料（政策文件、统计数据、文献引用）
Step D：按以下原则撰写内容：
         - 每段首句即核心观点，不铺垫废话
         - 数据/事实优先，论证简洁有力
         - 创新点须显式标注（可用"【创新】"标签）
         - 引用须注明来源
Step E：输出本板块完整内容
Step F：自查：是否偏题？是否有冗余表达？
```

#### 3.2 内容撰写铁律

1. **首句即核心**：每段第一句直接切入主题，禁止"众所周知…""随着…时代的到来…"式开头
2. **创新优先**：凡涉及研究创新点的内容，须在段落中显式标注 `【理论创新】`/`【方法创新】`/`【实践创新】`
3. **数据说话**：能用数字说清楚的不用形容词，如"提升效率约37%"而非"大幅提升效率"
4. **无废话原则**：删除所有"本研究认为"、"综上所述，不难看出"等无实质内容的表达
5. **引用规范**：中文文献按 GB/T 7714 格式，外文文献按 APA 格式

#### 3.3 各成果类型撰写要点

**研究报告**（见 `references/outcome_frameworks.md` → 章节 1）

**政策建议报告**：
- 执行摘要：问题+建议核心，不超过400字
- 政策建议：每条建议用"建议X：[标题]"格式，含背景、措施、预期效果三要素
- 语言：面向政府决策者，平实直接，避免学术术语堆砌

**技术报告**：
- 摘要须含技术指标、适用范围、主要创新
- 技术路线图应有文字说明版本
- 结果部分须含量化指标对比

**学术论文摘要**：
- 结构化摘要：目的-方法-结果-结论，各要素独立成句
- 关键词 5-8 个，按重要性排序

---

### Phase 4 — 全文合并与整合

所有板块完成后，执行合并：

#### 4.1 合并检查清单

```
□ 各成果项按总框架顺序排列
□ 同一数据/事实在全文只出现一次（重复的合并）
□ 章节编号连续无跳号
□ 所有"【创新】"标注已在摘要/引言中有对应呼应
□ 参考文献列表已去重、统一格式
□ 图表编号统一（表1、图1起）
□ 各成果项字数符合预定目标
```

#### 4.2 全文一致性检查

- **术语统一**：同一概念全文用同一术语，不混用别名
- **时态统一**：研究报告用过去时描述已完成工作
- **字体/格式**：为 Word 输出做准备（标题层级、段落格式）

---

### Phase 5 — Word 文档输出

#### 5.1 输出前询问

```
即将生成 Word 文档，请确认以下信息：
1. 成果文件名：[默认：课题名称_成果汇编_年份]
2. 是否需要封面页？（含：课题名称/主持人/单位/日期）
3. 目标期刊/提交机构的格式要求？（如有，请告知）
4. 是否需要目录页？
```

#### 5.2 Word 格式规范

| 元素 | 格式要求 |
|------|---------|
| 页面 | A4，上下 2.54cm，左右 3.17cm |
| 正文字体 | 中文宋体 12pt，英文 Times New Roman 12pt |
| 一级标题 | 黑体 16pt，加粗，居中 |
| 二级标题 | 黑体 14pt，加粗，左对齐 |
| 三级标题 | 宋体 12pt，加粗，左对齐 |
| 行距 | 1.5 倍行距 |
| 段间距 | 段前 6pt，段后 6pt |
| 页眉 | 课题名称，居中 |
| 页码 | 底部居中，阿拉伯数字 |
| 参考文献 | 宋体 10.5pt，悬挂缩进 |

#### 5.3 成果文件组织

若有多项成果，生成**一个合并文件**，各成果项之间用分页符隔开，并在目录中各自列出：

```
目录
一、研究报告 ............................  1
二、政策建议报告 ........................ 35
三、技术报告摘要 ........................ 42
四、学术论文摘要集 ...................... 48
附录 A：数据说明 ........................ 55
附录 B：参考文献总表 .................... 60
```

> 使用 `docx` skill 生成最终 Word 文档

## 五、交互规范

1. **Phase 2 框架确认是强制节点**：未收到用户明确确认指令（A/B/C）前，不得进入 Phase 3
2. **Phase 3 逐板块推进**：每完成一个板块，询问"继续下一板块（[板块名]）？还是需要先调整当前内容？"
3. **资料不足时**：主动告知用户缺少哪类资料，并给出两个选项：（1）用网络资料补充，（2）等待用户补充后继续
4. **字数透明**：每个板块完成后报告当前已完成字数和总体进度
5. **保存进度**：建议用户将每个 Phase 的输出保存，防止意外中断
