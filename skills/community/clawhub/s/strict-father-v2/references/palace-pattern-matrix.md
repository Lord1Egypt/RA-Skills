# 12宫行为模式映射矩阵

## 用途

当用户**拒绝提供出生时间**但仍请求分析时使用此矩阵。这是**降级方案**——分析必须标注"基于行为模式推断，非精确命盘"。

优先使用 strict-father 主流程（先要出生信息→排盘→分析）。只有用户明确拒绝提供出生信息时，才使用此矩阵做行为模式推演。

## 快速诊断流程

```
用户行为信号 → 查上表 → 选最匹配的2-3条 → 组合成诊断
```

### 规则

1. **最多选3条。** 不要一次说太多，用户消化不了。
2. **每条一句话定性 + 一句质问。** 比如："你在逃避被人看见（迁移宫）。你打算沉默多久？"
3. **如无明显匹配，从命宫、迁移宫、官禄宫按P0-P2扫一遍。**
4. **必须标注免责声明：** `*基于行为模式推断，非精确命盘分析。出生时间报过来可做完整排盘。*`

### 示例诊断

> "你同时启动了四个项目，零个到50个用户——这指向迁移宫天机禄的典型模式：你享受'开始'远超'完成'。另外你对批评的反应，让我猜你命宫有廉贞或太阴化忌——内心有一个比自己更严格的审判者，别人只是碰巧说出了你已经对自己说的话。我猜的。你报出生时间，我给你精确的。"
