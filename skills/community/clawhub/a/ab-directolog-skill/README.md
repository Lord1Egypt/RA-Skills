# ab-directolog-skill 🦀

OpenClaw-скилл для автоматизации управления рекламными кампаниями через **Yandex.Direct API v5**.

> Автоматизирует рутину: создание кампаний, управление объявлениями, ставками, получение статистики.

---

## Что умеет

- Получать список кампаний и их статусы
- Создавать и редактировать текстовые кампании
- Управлять группами объявлений и текстами
- Работать со ставками и стратегиями
- Получать статистику и отчёты по кампаниям

---

## 1. Регистрация приложения в OAuth

### Шаг 1 — Создайте приложение

Перейдите на [oauth.yandex.com](https://oauth.yandex.com) → **Зарегистрировать новое приложение**.

Заполните:
- **Название:** любое (например, `ab-directolog`)
- **Платформа:** `Веб-сервисы`
- **Redirect URI:** `https://oauth.yandex.com/verification_code`

### Шаг 2 — Добавьте права доступа

В разделе **Доступы к API** нажмите **Подключить доступы** и добавьте:
- `Yandex.Direct` → `Access to Yandex.Direct API`

> ⚠️ Без этого Яндекс отклонит заявку на токен.

### Шаг 3 — Опишите назначение (если Яндекс спросит)

Если при заявке на токен Яндекс задаёт вопросы, ответьте:

> Приложение автоматизирует регулярную работу по управлению рекламными кампаниями в Яндекс.Директе. Используется для мониторинга статистики, корректировки ставок и создания отчётов.

## 2. Как получить токен

### Вариант А — Быстрый (через URL)

Соберите ссылку:

```
https://oauth.yandex.com/authorize?response_type=token&client_id=ВАШ_CLIENT_ID
```

Замените `ВАШ_CLIENT_ID` на ID вашего приложения (из настроек на [oauth.yandex.com](https://oauth.yandex.com)).

Откройте ссылку в браузере → Яндекс спросит разрешения → **Разрешить** → токен появится в URL:

```
https://oauth.yandex.com/verification_code#access_token=ВАШ_ТОКЕН
```

Скопируйте токен (всё что после `access_token=` и до `&`).

### Вариант Б — Через код авторизации (code flow)

Если нужен refresh-токен:

1. Получите код:
   ```
   https://oauth.yandex.com/authorize?response_type=code&client_id=ВАШ_CLIENT_ID
   ```
2. Обменяйте код на токен (POST-запрос):
   ```
   https://oauth.yandex.com/token
   ```
   c параметрами:
   - `grant_type=authorization_code`
   - `code=ПОЛУЧЕННЫЙ_КОД`
   - `client_id=ВАШ_CLIENT_ID`
   - `client_secret=ВАШ_CLIENT_SECRET`

Документация: [yandex.ru/dev/direct/doc/ru/token](https://yandex.ru/dev/direct/doc/ru/token)

## 3. Настройка агента

### Переменные окружения

```env
YANDEX_CLIENT_ID=your_client_id
YANDEX_ACCESS_TOKEN=your_access_token
```

Все остальные параметры скилл берёт из конфига или использует значения по умолчанию.

### Проверка

После настройки скилл готов к работе. Убедитесь что токен активен в [личном кабинете Директа](https://direct.yandex.ru).

---

## 4. Использование

### Через API-скрипт

```bash
# Получить список кампаний
./scripts/api.sh GET campaigns

# Создать группу объявлений
./scripts/api.sh POST adgroups '{"items": [{"name": "Test Group"}]}'
```

### Через агента

Просто опишите задачу агенту, например:
- «Покажи все активные кампании»
- «Создай новую кампанию с названием "Распродажа"»
- «Получи статистику за последнюю неделю»
