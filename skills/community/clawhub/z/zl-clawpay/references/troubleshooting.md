# 故障排除与问题反馈

## 常见问题

### 平台配置缺失

**问题**：遇到 `config/.env not found` 错误。

**原因**：打包时未包含平台配置文件。

**解决**：检查 pack-skill.ps1 是否包含 `config` 目录。

---

## 问题反馈流程

当用户遇到以下情况且无法自行解决时，执行问题反馈：

| 场景 | 示例 |
|------|------|
| 支付反复失败 | 多次尝试支付，每次都失败 |
| CLI 返回未知错误码 | 返回非预期的错误码 |
| 查询支付状态异常 | 查询支付状态时返回异常结果 |
| 用户明确要求反馈 | 用户说"我要反馈问题" |

### 反馈步骤

**Step 1：确认问题无法自行解决**

检查是否可以通过重试、检查网络、更换 seqId 等方式解决。如果可以，不执行反馈。

**Step 2：告知用户反馈渠道**

告知用户可将问题反馈至 **rd@zlrt.net**，由用户自行决定是否发送：

> "如需反馈问题，请发送邮件至 rd@zlrt.net，邮件中请勿包含 apiKey、subWalletId、交易流水号、金额等敏感信息。"

> **⚠️ 敏感信息过滤**：如果用户选择发送邮件，提醒其移除以下内容：
> - **apiKey**：任何形式的 API Key（完整值或部分内容）
> - **subWalletId**：子钱包 ID
> - **seqId**：交易流水号
> - **金额信息**：具体支付金额
> - **商户信息**：商户 API Key（merApiKey）
> - **个人数据**：用户账号、姓名等
>
> 只保留错误码、错误描述、操作步骤等非敏感信息。

**Step 3：确认反馈**

告知用户："问题已记录，如需反馈请发送邮件至 rd@zlrt.net。"
