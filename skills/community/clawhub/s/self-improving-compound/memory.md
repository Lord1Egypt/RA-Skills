# Memory Quick Reference

> ⚠️ **Source of truth**: All entries live in `learning/memory_tree/chunks.db`.
> Edit / search / export via `python3 scripts/learnings.py --root <workspace>`.
> This file is a reference card only — entries written here are NOT durable.

## Entry Types
| Prefix | Type | When |
|--------|------|------|
| COR | Correction | User corrected a fact or preference |
| LRN | Learning | Discovered a convention or transferable lesson |
| ERR | Error | Non-obvious failure or tool quirk |
| FTR | Feature request | Missing capability the user wants |

## ID Format
- Human ID: `TYPE-YYYYMMDD-XXX` (e.g., `COR-20260508-001`)
- Internal: content-addressed SHA256 (auto-generated by SQLite)

## Pattern-Key Rules
- **Must use namespaced format**: `domain:key` (e.g., `api:rate-limit`, `docker:layer-cache`)
- Plain keys like `rate-limit` will trigger a warning
- Link related entries with `See Also: [Pattern-Key]`

## Lifecycle (HOT / WARM / COLD)
| Transition | Threshold |
|---|---|
| HOT → WARM | 30 days unused |
| WARM → COLD | 90 days unused |
| WARM → HOT | 3 uses within 7 days |
| → Project memory | Proven + broadly applicable |

## Rules
1. Search before log (`learnings.py search` first).
2. Only log durable lessons — skip one-off noise.
3. Use namespaced Pattern-Keys.
4. Promote when a rule is proven.
5. Never log secrets (tokens, passwords, API keys).
6. Use `learnings.py` CLI for all operations — don't edit this file.
