# Customer Journey Map Quality Checklist

Validate every journey map before delivery. Mark items [x] complete or [ ] incomplete.

---

## Data Completeness (8 items)

- [ ] **DC-01:** At least 90 days of quantitative analytics data collected
- [ ] **DC-02:** Data from all active marketing channels included (paid, organic, email, SMS, social, marketplace)
- [ ] **DC-03:** Ecommerce platform analytics pulled and reviewed (Shopify, Amazon, TikTok Shop)
- [ ] **DC-04:** Customer feedback data collected (reviews, surveys, NPS, support tickets)
- [ ] **DC-05:** Email/SMS performance data included for automated flows and campaigns
- [ ] **DC-06:** Post-purchase data gathered (delivery times, return rates, repeat purchase rates)
- [ ] **DC-07:** Qualitative data supplements quantitative (session recordings, interviews, or ticket analysis)
- [ ] **DC-08:** Data gaps explicitly documented with plan to fill in next iteration

## Touchpoint Coverage (8 items)

- [ ] **TC-01:** All seven stages mapped: awareness, consideration, evaluation, purchase, onboarding, retention, advocacy
- [ ] **TC-02:** Each stage includes at least 3 specific touchpoints from actual behavior data
- [ ] **TC-03:** Touchpoints name the specific channel (e.g., "Instagram Reels ad" not just "social media")
- [ ] **TC-04:** Cross-channel handoffs documented (ad to landing page, email to product page)
- [ ] **TC-05:** Physical touchpoints included where applicable (packaging, product inserts, unboxing)
- [ ] **TC-06:** Customer-initiated touchpoints mapped (support contacts, search queries, review reading)
- [ ] **TC-07:** Platform-specific touchpoints identified (Buy Box, Shopify checkout, TikTok Shop tab)
- [ ] **TC-08:** Third-party touchpoints included (review sites, comparison blogs, YouTube, Reddit)

## Friction Analysis (8 items)

- [ ] **FA-01:** At least 2 friction points identified per stage
- [ ] **FA-02:** Each friction point is specific and measurable (affected users, drop-off rate)
- [ ] **FA-03:** Friction categorized by type: technical, informational, emotional, procedural
- [ ] **FA-04:** Severity rated consistently (frequency x impact x revenue at stake)
- [ ] **FA-05:** Root causes identified, not just symptoms
- [ ] **FA-06:** Platform-specific friction flagged (Amazon policy limits, Shopify constraints, TikTok trust gaps)
- [ ] **FA-07:** Cross-channel friction identified (messaging inconsistencies, broken handoffs)
- [ ] **FA-08:** Each friction point has at least one proposed solution

## Emotion Mapping (6 items)

- [ ] **EM-01:** Customer emotions documented for each of the seven stages
- [ ] **EM-02:** Both positive and negative emotions captured
- [ ] **EM-03:** Emotions are specific (e.g., "commitment anxiety at payment" not "negative feeling")
- [ ] **EM-04:** Emotional states connected to behavioral outcomes
- [ ] **EM-05:** Emotional peaks (delight) and valleys (frustration) identified
- [ ] **EM-06:** Trust level assessed at each stage

## Optimization Recommendations (7 items)

- [ ] **OR-01:** At least one optimization per stage
- [ ] **OR-02:** Each recommendation is specific and actionable
- [ ] **OR-03:** Expected impact estimated (e.g., "15-25% improvement in landing page conversion")
- [ ] **OR-04:** Effort classified: quick win (under 1 week), medium (1-4 weeks), strategic (1-3 months)
- [ ] **OR-05:** Prioritized action plan uses impact-effort framework
- [ ] **OR-06:** Each action plan item has assigned owner, timeline, and success metric
- [ ] **OR-07:** At least 3 quick wins identified for immediate implementation

## Cross-Channel Consistency (5 items)

- [ ] **CC-01:** Messaging consistency evaluated across channels
- [ ] **CC-02:** Pricing and promotion consistency verified
- [ ] **CC-03:** Product information consistency checked (descriptions, images, availability)
- [ ] **CC-04:** Cross-device journey continuity evaluated (mobile to desktop)
- [ ] **CC-05:** Channel transition quality documented with friction at each handoff

## Actionability (5 items)

- [ ] **AC-01:** Journey map answers the primary business question stated in objective
- [ ] **AC-02:** Findings understandable by non-technical stakeholders
- [ ] **AC-03:** Action plan achievable with current team resources
- [ ] **AC-04:** Success metrics defined for each recommended optimization
- [ ] **AC-05:** Review date set for refreshing the map (typically 90 days)

## Presentation Quality (6 items)

- [ ] **PQ-01:** Header complete (brand, platform, segment, date, data period, objective)
- [ ] **PQ-02:** Journey summary provides clear executive-readable narrative
- [ ] **PQ-03:** Tables and structured formats used consistently
- [ ] **PQ-04:** Data presented with specificity (percentages, dollar amounts, time periods)
- [ ] **PQ-05:** Benchmarks provided alongside current metrics to show gaps
- [ ] **PQ-06:** All placeholder text replaced with actual findings

---

## Scoring Guide

| Score | Rating | Interpretation |
|---|---|---|
| 48-53 | Exceptional | Comprehensive, data-rich, ready for executive presentation |
| 40-47 | Strong | Solid map with minor gaps that do not undermine analysis |
| 30-39 | Adequate | Usable but missing meaningful detail in some areas |
| 20-29 | Needs Work | Significant gaps reducing reliability and actionability |
| Below 20 | Incomplete | Major sections missing; do not deliver in current state |

## Post-Checklist Actions

1. Address any unchecked items in FA, OR, and AC categories (critical)
2. Document reasons for intentionally skipped items
3. Note items to address in next iteration
4. Confirm scope and depth meet stakeholder needs before delivery
