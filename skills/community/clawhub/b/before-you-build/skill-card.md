## Description: <br>
Use before building or changing a product, feature, SaaS, AI app, side project, or startup idea to run a short demand, distribution, and failure-pattern reality check. <br>

This skill is ready for commercial/non-commercial use. <br>

## Publisher: <br>
[bin1874](https://clawhub.ai/user/bin1874) <br>

### License/Terms of Use: <br>
MIT <br>


## Use Case: <br>
External users, developers, founders, indie hackers, AI builders, and small teams use this skill before implementation to challenge product demand, distribution, monetization, and failure-pattern assumptions. It produces short product-risk reviews for new ideas, feature changes, and already-launched projects. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Optional Case Memory lookup may send idea details to an external endpoint if a compatible agent performs the lookup. <br>
Mitigation: Ask for explicit user permission first and send only a short, non-confidential idea summary with no secrets, customer names, private financials, credentials, private user data, or unreleased confidential details. <br>
Risk: The skill gives product-risk guidance and may be mistaken for code, security, architecture, or implementation review. <br>
Mitigation: Invoke it explicitly for pre-build product review and use other review processes for code quality, security, architecture, migrations, or infrastructure decisions. <br>
