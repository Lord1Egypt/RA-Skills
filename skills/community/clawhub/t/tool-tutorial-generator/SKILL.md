# 工具软件教程自动生成器

## 概述

用户输入任意工具/软件/语言/框架名称（如"Figma"、"PyTorch"、"微信小游戏开发"、"Cursor编辑器"），自动搜索中文互联网教程资源，聚合、去重、结构化整理，生成一份完整精美的交互式HTML教程报告。

## 触发条件

以下任意短语出现时立即激活：
- "生成XX教程"、"XX教程"、"XX使用指南"、"XX入门"
- "怎么用XX"、"如何学习XX"、"帮我学XX"
- "给XX做个教程"、"XX软件学习报告"
- "tutorial for XX"、"how to use XX"

## 工作流

### 第一步：需求确认

1. 提取用户输入中的工具/软件名称
2. 如果名称不明确或太宽泛，追问一次：例如"你说的是Figma设计工具还是Figma插件生态？"
3. 确认后立即开始，不复述「正在为你搜索」

### 第二步：多平台搜索（并行）

同时进行以下搜索，每个方向至少2个关键词变体，topic分别设定为programming或general：

```
搜索维度：
1. 知乎/CSDN - 教程合集、入门指南
   关键词：「XX入门教程」「XX教程 2025」「XX使用指南」
   
2. B站/视频资源 - 实战项目、看视频教程
   关键词：「XX视频教程」「XX实战」「XX手把手」
   
3. 百度/通用 - 官方文档、安装配置
   关键词：「XX安装配置」「XX官方文档」「XX最新教程」
   
4. 微信公众号/掘金 - 进阶技巧、最佳实践
   关键词：「XX进阶技巧」「XX最佳实践」「XX踩坑」
   
5. GitHub/开源 - 项目示例、脚手架
   关键词：「XX GitHub项目」「XX awesome」「XX starter」
```

使用 WebSearch 工具，每组搜索用 `query_keyword_groups` 参数同时搜索不同角度。

### 第三步：内容聚合与结构化

从搜索结果中提取以下信息（按优先级）：

| 板块 | 优先级 | 内容来源 |
|------|--------|----------|
| 工具概览 | P0（必填） | 官方描述+百科+社区共识 |
| 安装配置 | P0（必填） | 官网+CSDN/掘金安装教程 |
| 基础用法 | P0（必填） | 教程文章+视频教程摘要 |
| 进阶技巧 | P1（尽量） | 掘金/知乎/公众号深度文章 |
| 常见问题 | P1（尽量） | Stack Overflow/知乎问答/CSDN踩坑 |
| 学习路线图 | P2（可选） | 知乎/掘金学习路线文章 |
| 资源汇总 | P2（可选） | 聚合以上所有来源链接 |

**内容合成原则**：
- 不要机械罗列搜索结果，要用自己的话重新组织
- 安装配置步骤具体到命令/操作，不是泛泛而谈
- 基础用法至少包含2-3个可操作的示例
- 如果搜索结果不足，基于自身知识补充
- 每个板块保持在200-800字，总报告2500-5000字

### 第四步：生成HTML报告

使用 `assets/tutorial_template.html` 作为报告模板参考。报告需包含：

**视觉设计**：
- 现代中文排版风格，配色干净专业
- 左侧固定目录导航（TOC），支持点击跳转
- 顶部 Hero 区域：工具名称、一句话简介、标签
- 卡片式内容布局，板块清晰
- 代码块带语法高亮样式
- 移动端响应式

**核心板块（7个）**：
1. 📋 工具概览 - 一句话介绍、适用人群、学习难度评级
2. 🔧 安装配置 - 分平台（Windows/macOS/Linux）安装步骤
3. 🎯 基础入门 - 核心概念+2-3个实操示例
4. 🚀 进阶技巧 - 效率提升技巧、工作流优化
5. ❓ 常见问题 - FAQ格式，至少5个高频问题
6. 🗺️ 学习路线图 - 从入门到精通的阶段性路径
7. 📚 资源汇总 - 分类推荐链接（官方/社区/视频/书籍）

**技术要求**：
- 纯HTML+CSS+JS，零外部依赖
- 内嵌CSS和JS，单文件可离线使用
- 目录导航自动高亮当前阅读位置
- 平滑滚动
- "返回顶部"浮动按钮
- 支持打印（@media print）

生成后将HTML文件保存到工作区根目录，文件命名：`{工具名}_教程报告.html`

### 第五步：交付

使用 present_files 展示生成的HTML报告。附加简短总结：报告包含多少板块、内容字数、推荐下一步学习建议。

## 模板使用

在 `assets/tutorial_template.html` 中提供了完整的HTML模板骨架（CSS样式+交互JS）。生成报告时：
1. 参考模板的CSS变量和布局结构
2. 根据实际内容调整板块（不是所有工具都需要所有7个板块）
3. 保持视觉风格一致
4. 如果工具特别简单（如小工具），可精简为5个板块
