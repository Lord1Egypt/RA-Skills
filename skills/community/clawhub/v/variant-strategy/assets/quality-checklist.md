# Variant Strategy Quality Checklist

## Data Collection (8 items)
- [ ] Minimum 90 days of variant-level sales data collected
- [ ] Units sold, revenue, and conversion rate captured per variant
- [ ] Return rate and top return reasons documented per variant
- [ ] Current inventory levels and days of supply calculated
- [ ] Holding cost / storage fees quantified per variant
- [ ] Customer review sentiment analyzed by variant
- [ ] Platform and target audience clearly identified
- [ ] Seasonal context and planned launches noted

## Performance Analysis (7 items)
- [ ] All variants plotted on Revenue Contribution vs. Growth Trend matrix
- [ ] Stars, Cash Cows, Question Marks, and Dogs clearly classified
- [ ] Classification thresholds documented and justified
- [ ] Trend calculation uses consistent time periods
- [ ] Outlier events (promotions, stockouts) flagged and adjusted for
- [ ] Conversion rate differences between variants analyzed
- [ ] Variant-level advertising spend and ROAS included if available

## Economic Analysis (6 items)
- [ ] Gross margin calculated per variant (COGS + fulfillment)
- [ ] Return processing cost deducted from revenue per variant
- [ ] Storage/holding cost allocated per variant
- [ ] Net margin calculated after all variable costs
- [ ] Inventory turnover rate computed per variant
- [ ] Negative-margin variants explicitly flagged

## Competitive Analysis (5 items)
- [ ] Top 3-5 competitors identified in same category and price range
- [ ] Competitor variant offerings mapped and compared
- [ ] Gap opportunities identified (variants competitors offer that you lack)
- [ ] Differentiation opportunities noted (unique variants you carry)
- [ ] Search volume data gathered for potential new variants

## Recommendation Quality (7 items)
- [ ] Every variant has a clear action: Keep, Promote, Retire, Add, or Test
- [ ] Retirement candidates have sell-through strategy with timeline
- [ ] New variant additions validated by demand signals (not just intuition)
- [ ] Bundle opportunities analyzed using cart co-occurrence data
- [ ] Recommended variant count is within 4-12 range (or justified if outside)
- [ ] Inventory allocation percentages sum to 100% and reflect demand weighting
- [ ] Cannibalization risk assessed for similar variants

## Implementation Plan (5 items)
- [ ] Phased rollout timeline with specific week targets
- [ ] Sell-through pricing schedule for retiring variants
- [ ] Test quantities and go/no-go criteria defined for new variants
- [ ] Budget allocated for promotion of Question Mark variants
- [ ] Responsible owner assigned for each phase

## Monitoring & Sustainability (5 items)
- [ ] Weekly, monthly, and quarterly review cadence defined
- [ ] KPI thresholds for triggering variant action established
- [ ] Dashboard specifications provided for ongoing tracking
- [ ] Seasonal re-evaluation triggers documented
- [ ] Competitive monitoring schedule set (quarterly minimum)
