# Go / No-Go 与 Forecast 校准参考

## 用途

当任务需要判断"是否继续投入""预测应不应该上调/下调""这单要不要放弃"时读取本文件。

## Go 的最低条件

一般至少要满足：
- EB 可触达
- 有真实 Champion
- 痛点与业务结果绑定
- 有预算可能
- 决策流程可预见

## 常见 No-Go / 暂缓信号

- 找不到 EB
- 没有真实 Champion
- 痛点不急，不值得预算支持
- 决策流程不清
- 客户没有明确下一步承诺

## Gate 规则

- 用客户行为判断阶段，不用销售感觉
- gate 没过，不要直接套该阶段赢率
- gate 不成立时，优先阶段回退，而不是轻微打折

## Forecast 校准

### Commit
- EB 基本可触达
- Champion 真实
- 决策流程较清晰
- 客户承诺明确

### Best Case
- 方向正确
- 仍缺 1-2 个关键证据
- 存在推进机会，但不够硬

### Pipeline
- 还在探索
- 关键证据不足
- 不能提前美化

## 管理原则

- 硬证据不够，不要高报预测
- 两周内关键短板补不齐，考虑降预测
- 机会质量低于门槛时，优先止损而不是继续堆资源
- 缺硬证据时，不允许上报高预测
