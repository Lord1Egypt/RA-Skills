## 6.2 功能模块（六层结构 v2.2 🆕）

读取 `03.5-field-decisions.md` 用户决策，生成。**每个操作节点必须含 6 层，缺一不可**。

详细模板和每层示例见 `../INTERACTION-SPEC.md`（共享交互规格文档）。

每个操作节点的 7 层：

```
## 模块 MX: <模块名>
**关联用户故事**: US-XXX
**关联实体**: 引用 ER 模型实体名

### 操作节点 MX.Y: <操作名>

#### Layer 1 — 入口与触发
- 入口位置 / 触发控件 / 可见条件 / 可用条件 / 是否需要二次确认

#### Layer 2 — 完整交互流（🆕 v3.1 新增，必须逐步描述）
> 描述用户从触发入口到操作完成的每一步交互动作和系统响应。
> **格式**：`[用户动作] → [系统响应]`，按操作顺序逐条列出，不允许跳步或合并。

示例（新建质检模板）：
- 用户点击「新建模板」按钮 → 右侧滑出「新建质检模板」抽屉（宽480px）
- 用户填写「模板名称」输入框 → 实时字数统计显示（已输入X/20字）
- 用户选择「质检类型」下拉 → 选中「AI质检」后，下方动态显示「AI置信度阈值」字段
- 用户点击「+ 添加质检项」 → 当前表单末尾插入新的质检项行（输入框 + 删除按钮）
- 用户点击某质检项的「删除」图标 → 弹出行内确认「确认删除该质检项？[取消][确认]」
- 用户点击确认删除 → 该行动画消失，剩余行自动重新编号
- 用户点击「提交」按钮 → 按钮变为 loading 态，禁止重复点击
- 提交成功 → 抽屉关闭，列表刷新，顶部 toast 提示「创建成功」（2秒消失）
- 提交失败（名称重复）→ 名称字段下方显示红色错误文案「模板名称已存在，请修改」，焦点自动回到该字段

**写作要求**：
- 每条必须有明确的「用户动作」和「系统响应」两侧，不允许只写一侧
- 条件分支（如选A显示X、选B显示Y）必须分别写出，不可省略
- 所有按钮（包括取消、关闭×）的点击结果都必须描述
- 异常路径（校验失败、网络错误）必须单独列条，不可合并到成功路径

#### Layer 3 — 界面形态（ASCII 草图）
- 画出主要界面结构（字段位置/控件类型/按钮布局）
- 说明：形态（弹窗/抽屉/页面内嵌）、尺寸、关闭方式

#### Layer 4 — 字段表（7 列）
> 来源列直接复用 `03-business-objects-er.md` 中对应要素的标注。

| 字段名 | 类型 | 来源 | 必填 | 默认值 | 约束/边界 | 备注 |

#### Layer 5 — 字段交互生命周期（每个字段 4 阶段）
每字段独立一小节：前置填充 / 输入中 / 失焦时 / 提交时

#### Layer 6 — 提交与反馈
- 提交按钮文案 / 校验顺序 / loading 态 / 成功反馈 / 失败反馈 / 自动关闭

#### Layer 7 — 边界与异常清单
- 默认值 / 必填可空边界 / 数值边界 / 业务逻辑边界 / 并发边界 / 异常边界
```

字段类型限定：string/number/bool/enum/date/datetime/array/object/file/reference

→ `04-modules.md`，**STOP** 等确认。

> **产品化改造专用规则**：当本文件由 `productization/SKILL.md` 调用时：
> - **只对改造/新增的操作节点做7层详述**
> - 现有系统不变的操作节点，在模块下用一行标注 `#### [沿用现有实现: <操作名，简述不变的原因>]`，不展开7层
> - Layer 4 字段表只列新增字段和改造字段，不变字段一行标注 `（其余字段沿用现有，共 N 个，无变动）`
