## 概述

### 什么是GEO

GEO（Generative Engine Optimization，生成式引擎优化）是一套帮助AI系统准确理解、信任并优先推荐您内容的技术体系。与传统SEO不同，GEO的目标用户是AI，而非人类。

## 🏗️ 第一阶段：基础设施层

### 1.1 llms.txt 部署工具

#### 什么是llms.txt

`llms.txt` 是一个专门为AI系统设计的文本文件，放置在网站根目录下，用于提供网站的核心信息和结构化描述。

#### 标准格式

```markdown
# [园区官方名称]

## 关于我们
[2-3句话描述园区定位、核心产业、地理优势]

## 核心产业
- [产业1]
- [产业2]
- [产业3]

## 产品类型
- 独栋办公楼
- 标准厂房
- 定制化空间
- 研发实验室

## 联系方式
- 地址：[详细地址]
- 电话：[联系电话]
- 邮箱：[官方邮箱]
- 官网：[网站URL]

## 最新动态
[最近3个月的重要更新、政策、活动]

## 数据摘要
- 占地面积：[X]亩
- 入驻企业：[X]家
- 产业集聚度：[X]%
- 年产值：[X]亿元
```

#### 部署检查清单

- [ ] 文件放置在网站根目录：`https://yourpark.com/llms.txt`
- [ ] 响应类型为 `text/plain`
- [ ] 字符编码为 UTF-8
- [ ] 内容不超过 50,000 字符
- [ ] 定期更新（建议每月）

### 1.2 Schema 结构化数据工具

#### 支持的类型

| 类型 | 用途 | 关键字段 |
|------|------|----------|
| Organization | 园区基本信息 | name, description, url, contact |
| Product | 厂房/办公室 | name, price, area, floorHeight |
| Event | 招商活动 | name, date, location, organizer |

#### Organization Schema 示例

```json
{
  "@context": "https://schema.org",
  "@type": "Organization",
  "name": "[园区名称]",
  "alternateName": "[别名/简称]",
  "description": "[园区描述，100-200字]",
  "url": "[官网URL]",
  "logo": "[Logo图片URL]",
  "image": "[园区全景图URL]",
  "address": {
    "@type": "PostalAddress",
    "streetAddress": "[街道地址]",
    "addressLocality": "[城市]",
    "addressRegion": "[省份]",
    "postalCode": "[邮编]",
    "addressCountry": "CN"
  },
  "telephone": "[联系电话]",
  "email": "[邮箱]",
  "areaServed": {
    "@type": "City",
    "name": "[城市名]"
  },
  "knowsAbout": ["生物医药", "智能制造", "数字经济"],
  "hasOfferCatalog": {
    "@type": "OfferCatalog",
    "name": "园区产品",
    "itemListElement": [
      {
        "@type": "Offer",
        "itemOffered": {
          "@type": "Product",
          "name": "[产品名称]"
        }
      }
    ]
  }
}
```

#### Product Schema 示例（厂房/办公室）

```json
{
  "@context": "https://schema.org",
  "@type": "Product",
  "name": "[产品名称，如：XX园区A栋标准厂房]",

      "@type": "Organization",

    }

  "additionalProperty": [
    {
      "@type": "PropertyValue",
      "name": "建筑面积",
      "value": "[面积]",
      "unitCode": "MTK"
    },
    {
      "@type": "PropertyValue",

      "value": "[层高]",
      "unitCode": "MTR"
    },
    {
      "@type": "PropertyValue",
      "name": "承重",
      "value": "[承重]",

    {
```

#### Event Schema 示例（招商活动）

```json

    "@type": "Organization",

    "url": "[主办方网站]"

    "@type": "Organization",
    "name": "[承办方名称]"
  },

    "@type": "Offer",
```

## 最佳实践

### DO（推荐）

- ✅ 保持llms.txt每月更新
- ✅ 所有数字使用精确值，避免"大概"、"左右"
- ✅ 联系方式保持与官网一致
- ✅ 核心答案放在每个段落的开头
- ✅ 主动在知乎等平台回答相关问题
- ✅ 监控AI搜索结果，及时修复问题

### DON'T（避免）

- ❌ 避免过时的租金和政策信息
- ❌ 避免模糊的描述如"环境优美、交通便利"
- ❌ 避免与其他园区对比时使用主观评价
- ❌ 避免在多个平台发布矛盾信息
- ❌ 避免忽视用户真实问题

### 关键指标

| 指标 | 目标值 | 测量方式 |
|------|--------|----------|
| AI引用率 | >80% | 核心问题AI回答中提及园区 |
| 信息准确率 | >95% | AI提供信息与实际一致 |
| 前3排名率 | >60% | 核心关键词排名前3 |
| 内容覆盖率 | >90% | 问题图谱覆盖率 |
