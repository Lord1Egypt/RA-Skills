# AI 产品经理方法论参考手册

## 一、AI PM 五大核心能力体系

基于腾讯云开发者社区 2026 年研究，AI PM 需要具备五大核心能力:

### 能力模型

```
模型边界管理 → 判断"能不能用 AI"
Prompt 工程   → 定义"怎么跟 AI 说"
Agent 编排    → 设计"AI 怎么干活"
RAG 数据工程  → 确保"AI 说真话"
评估监控      → 验证"AI 干得好不好"
```

## 二、AI PM 能力三角模型

```
          技术理解力
              △
             /|\
            / | \
           /  |  \
          /   |   \
         /    |    \
  产品设计力 ─── 商业敏感度
```

### 2.1 技术理解力
- 基础原理: 监督学习/无监督学习/强化学习
- 模型生命周期: 数据→训练→部署→监控
- 大模型技术栈: Prompt、LoRA微调、KV缓存优化、推理加速

### 2.2 产品设计力
- 概率化设计思维 (降级机制)
- 人机协同流程设计
- 数据驱动迭代机制 (A/B → 评估 → 优化)
- 可解释性 UI 设计

### 2.3 商业敏感度
- ROI 计算模型
- AI 产品定价策略 (按Token/按效果/订阅制)
- 伦理风险管理 (数据偏见检测、内容过滤)
- 合规架构设计 (GDPR、AI法案)

## 五、成本估算公式

```
日Token消耗 = DAU × 每次平均调用次数 × (AvgInputTokens + AvgOutputTokens)
日成本 = 输入Token × 输入单价 + 输出Token × 输出单价
月成本 = 日成本 × 30
```

### 缓存策略节省

```
无缓存日成本 = FullCost
语义缓存日成本 = FullCost × (1 - CacheHitRate × 0.7)
精确缓存日成本 = FullCost × (1 - CacheHitRate)
```

## 六、RAG 方案设计参数参考

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| Chunk Size | 500-1500 tokens | 代码500，长文档1000-1500 |
| Chunk Overlap | 10-20% | 减少信息断裂 |
| Embedding维度 | 1024/1536/3072 | 高维更精确但成本高 |
| 向量检索 Top-K | 10-20 | 初筛阶段 |
| Rerank Top-N | 3-5 | 最终返回给模型 |
| 混合检索权重 | 向量0.7 + 关键词0.3 | 根据场景调整 |

## 七、Eval 评估指标速查

| 指标 | 公式 | 适用场景 |
|------|------|---------|
| 准确率 (Accuracy) | (TP+TN)/(TP+TN+FP+FN) | 分类任务 |
| 精确率 (Precision) | TP/(TP+FP) | 宁缺毋滥场景 |
| 召回率 (Recall) | TP/(TP+FN) | 宁滥勿缺场景 |
| F1 Score | 2×P×R/(P+R) | 综合平衡 |
| BLEU | n-gram 重合度 | 翻译/摘要 |
| ROUGE | 召回导向 | 摘要评估 |
| NDCG | 排序质量 | 搜索/推荐 |

## 八、常见 AI 产品模式

### 8.1 对话式 (Conversational)
- 代表: ChatGPT、智能客服
- 关键设计: 多轮对话记忆、上下文窗口管理、降级到人工

### 8.2 嵌入式 (Embedded)
- 代表: Notion AI、Copilot
- 关键设计: 场景感知的触发时机、结果预览与确认、撤销能力

### 8.3 自主式 (Agentic)
- 代表: AutoGPT、Devin
- 关键设计: 任务拆解透明度、中间结果可审查、安全边界与熔断
