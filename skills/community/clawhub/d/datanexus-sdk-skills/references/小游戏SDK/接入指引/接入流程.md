# 微信小游戏 SDK 接入流程

## 1. 前置条件

- 已完整阅读本 SDK 接入流程文档
- 在微信公众平台「开发管理 → 开发设置 → 服务器域名」中，将 `https://api.datanexus.qq.com` 添加到 **request 合法域名**
- IAA 小游戏需额外填写接入申请表（由对接运营提供）

## 2. 下载 SDK

### 方式一：npm 下载

```bash
npm i @dn-sdk/minigame
```

引用方式：

```javascript
import { SDK } from '@dn-sdk/minigame';
```

### 方式二：链接下载

从 DataNexus 文档站下载 SDK 文件（含 index.js 和类型声明），放入项目目录直接引用。

## 3. 引用 SDK 并初始化

根据游戏引擎引用方式不同（详见[引擎适配](./引擎适配.md)），以下为通用初始化示例：

```javascript
const sdk = new SDK({
  user_action_set_id: 'your_user_action_set_id',
  secret_key: 'your_secret_key',
  appid: 'wx123xyz123xyz123x',
});
```

### 初始化参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `user_action_set_id` | string | 是 | 数据源 ID，在 DataNexus 数据接入页面获取 |
| `secret_key` | string | 是 | 数据源密钥 |
| `appid` | string | 是 | 当前小游戏的 AppID（wx 开头），不可填写其他应用的 |
| `openid` | string | 否 | 微信 OpenId |
| `unionid` | string | 否 | 微信 UnionId |
| `user_unique_id` | string | 否 | 业务自定义用户 ID |
| `auto_track` | boolean | 否 | 自动采集开关，默认 `true` |

## 4. 设置用户 ID

初始化后尽早设置用户标识，**建议在上报注册事件前完成**：

```javascript
sdk.setOpenId('用户的openid');
sdk.setUnionId('用户的unionid');
```

> ⚠️ 未设置 openid 或 unionid 时，SDK 会暂停数据上传。

## 5. 事件上报

### 5.1 通用上报方法

```javascript
sdk.track(action_type, action_param);
```

示例：

```javascript
sdk.track("PURCHASE", { value: 600 });
sdk.track("VIEW_CONTENT", { contentName: "商城页" });
```

### 5.2 专用上报方法

| 方法 | 对应 action_type | 参数 |
|------|-----------------|------|
| `sdk.onRegister()` | REGISTER | 无 |
| `sdk.onCreateRole(roleName)` | CREATE_ROLE | 角色名 |
| `sdk.onTutorialFinish()` | TUTORIAL_FINISH | 无 |
| `sdk.onPurchase(value)` | PURCHASE | 金额(分) |
| `sdk.onAppStart()` | START_APP | 无 |
| `sdk.onAppQuit()` | APP_QUIT | 无 |
| `sdk.onEnterForeground()` | ENTER_FOREGROUND | 无 |
| `sdk.onEnterBackground()` | ENTER_BACKGROUND | 无 |
| `sdk.onAddToWishlist()` | ADD_TO_WISHLIST | 无 |

## 6. 调试查看事件信息

```javascript
SDK.setDebug(true);
```

控制台 TAG：`[@dn-sdk/minigame v1.x.x]`。Network 面板监控 `api.datanexus.qq.com/data-nexus-cgi/miniprogram`，返回 `code: 0` 即成功。

## 7. 兼容性测试

在微信开发者工具、Android 真机、iOS 真机上测试。

## 8. 数据对账

DataNexus 平台 → 日志查询 → 输入 AppID 或数据源 ID。上线后通过质量看板查询数据质量。

## 9. 高级功能

### 9.1 设置上报请求并发数

```javascript
SDK.setRequestConcurrency(5);  // 最小 1，默认 4
```

### 9.2 自动采集能力

默认开启，自动采集：TICKET / START_APP / ENTER_FOREGROUND / ENTER_BACKGROUND / LOGIN / SHARE / START_PAY / FINISH_PAY / ADD_TO_WISHLIST / CREATE_GAME_ROOM

关闭方式：初始化时设置 `auto_track: false`。
