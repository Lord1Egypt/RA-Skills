# 人设引导脚本

## 这份文件是什么

当用户进入三库内容工作流但**没有可用人设卡**时，AI 第一句话该说什么、怎么把用户引导到一张可用的人设卡。

`content-writer.workflow.md` 第 0 步会引用本文件。人设统一走 CSO 平台。

## 触发条件

进入工作流后，按以下顺序判断：

1. 用户本次对话直接提供了人设卡文件 / 内容 → 读取后跳过引导，进入"导入路径"
2. CSO 平台已有人设（`siluzan-cso persona list`）→ 列出供用户选择
3. 以上都没有 → 触发本引导脚本

## 第一句话（AI 主动开口）

> 在开始之前，我需要先和你确认"人设"——也就是这个账号是个什么样的"人"。
>
> 为什么这一步不能省：三库（流量因子 / 产品资产 / 烹调方法）是通用骨架，同一个"强观点开场"模板，配不同腔调写出来完全是两篇东西。差别全在人设。
>
> 三种方式，挑一个：
>
> **① 用 3 篇代表作品反推（推荐）**
> 你贴 3 篇想延续风格的作品（你自己写的、或者想对标的账号都行），我读完会给你一张反推出来的人设卡。
>
> **② 你直接描述**
> 如果脑子里已经很清楚账号长什么样，跟我说定位 / 受众 / 风格 / 口头禅就行，我边问边整理。
>
> **③ 你已经有现成的人设卡**
> .md / 文本 / 截图，发给我，我结构化后保存到 CSO 平台。
>
> 你想走哪条？

## 兜底分支：用户什么都不想填

如果用户表达"我先随便写一篇看看""我不想搞这么麻烦""我还没想好"——**不要硬卡**，给临时人设路径：

> 你现在就想写一篇？也可以——你先告诉我这次想写什么主题、给谁看、希望什么腔调。我用一个**临时人设**给你生成。
>
> 写完之后，如果你喜欢这版的腔调，我们就把它保存成正式人设；不喜欢就丢掉重来。

临时人设的处理：

- 仅在会话内存中，**不**保存到 CSO 平台
- 各章节按"待确认"对待，写完再请用户拍板
- 写完后主动问一次："这版的腔调你喜欢吗？要不要保存？"
- 用户说"保存"才入库（`siluzan-cso persona create`）

## "我不知道什么叫人设" 的解释

如果用户提出这个疑问：

> 你可以把它想成演员的角色卡：这个号是个什么样的"人"，聊什么、不聊什么、说话什么腔。我们后面写出来的所有内容，都是这个"人"在说话。不用一开始就完美，写几篇会越来越准。

## 三个分支的执行细节

### 分支 ① 反推路径（推荐）

**AI 说：**

> 好，把 3 篇贴给我（或者上传文件）。每篇之间用 `---` 分隔，或者一篇一条消息发都行。

**收到后：**

- 调用 `persona-reverse-sop.md` 流程
- 输出 styleGuide（符合 `persona-schema.md` 的 6 章节）
- 进入 **A1 确认环节**（见下方"落盘前确认流程"）

**作品数量兜底：**

- 只给 1–2 篇：

  > 可以，但 1–2 篇里有些章节我没法稳定推出来（口头禅、禁忌、平台适配这些）。这些我会标"待确认"，最后我们一条条过一遍。你愿意补充更多作品吗？或者直接进确认环节，缺的部分你来填？

- 给超过 5 篇：
  > 这么多篇我可以全读，但建议挑你最想延续风格的 3–5 篇就好——多了反而难找共性。你来挑，还是我读完所有的之后告诉你哪几篇最有代表性？

### 分支 ② 描述路径

**AI 主动按 6 章节顺序问，不要把空表格甩给用户：**

```
Q1（→ 一句话定位）: 先说定位——这个账号是干嘛的？想解决什么问题、给受众什么价值？
Q2（→ 目标受众）: 写给谁看？他们的职业 / 阶段（新手 / 老手 / 决策者 / 围观者）、最关心什么问题？
Q3（→ 内容 DNA·风格&立场）: 想让人感觉这个号是什么风格？给几个倾向参考（可多选）：
    - 强判断（先下结论，立场鲜明）
    - 务实步骤（教程感，可执行）
    - 故事感（有画面，节奏慢）
    - 互动型（引发讨论）
    - 反差颠覆（打脸式认知）
    - 温和分享（生活化）
    - 专业权威（知识密度高）
    - 接地气（口语化）
   再补一句：这个号始终在「捍卫」或「反对」什么？（→ 价值观锚点）
Q4（→ 内容 DNA·节奏）: 节奏偏好——快（句句推进）、中等、还是稳一点的慢节奏？
Q5（→ 内容 DNA·开场&口头禅）: 习惯怎么起开头？有没有口头禅、固定开场、标志性比喻？想到几条算几条。
Q6（→ 推荐用词）: 有哪些你常用、希望多出现的词？尽量给 6 个以上具体词。
Q7（→ 禁用表达 / 禁忌）: 有什么是你绝对不讲的话题？或者绝不能用的表达？（没有就说"无"，我按常识补红线再跟你确认）
Q8（→ <平台> 平台适配建议）: 这个号主要发哪个平台？（抖音 / 视频号 / 公众号 / 小红书…）我按该平台写适配建议。
```

> 这 8 问对应 styleGuide 的 6 个章节（Q3+Q4+Q5 一起喂「内容 DNA」的开场 / 句式 / 节奏 / 立场）。常讲话题、行业范围不单独设字段——从 Q1/Q2 的定位与受众里推断即可。

问完按 `persona-schema.md` 模板整理成 6 章节 styleGuide → **A1 确认环节**。

### 分支 ③ 已有卡路径

**AI 说：**

> 发给我，任何格式都行。我读完按标准 6 章节重排版，缺的章节补问你，然后保存到 CSO 平台。

**收到后：**

- 解析内容，重排版到 `persona-schema.md` 的 6 个章节
- 缺失的必填章节（一句话定位 / 目标受众 / 内容 DNA / 推荐用词 / 禁用表达·禁忌 / 平台适配）→ 单独问用户
- 章节信息不全但能从已有内容推断 → 写推断版并标"待确认"
- 完全没线索的章节 → 留占位 + 标"待确认"
- 进入 **A1 确认环节**

## 落盘前确认流程（A1）

所有路径最终都汇到这里。**强制执行**，不允许跳过。

**AI 说：**

> 我整理了一份 styleGuide，咱们一条条过一遍。我会列出每个章节和我的判断，你确认 / 修改 / 补充。
>
> 过完之后保存到 CSO 平台，下次写稿可以直接用 `siluzan-cso persona list` 取回。

**逐条过的展示形式（按 6 个章节）：**

```
✅ 已确认 [章节名]: <内容>
       —— 来源：作品 A 反复出现 / 你刚才描述的 / 你提供的卡
       —— 要改吗？

⚠️ 高概率 [章节名]: <内容>
       —— 来源：作品 B 出现 1 次，需要确认
       —— 保留 / 修改 / 删除？

❓ 待确认 [章节名]: <内容>
       —— 我推不出来，需要你确认
       —— 你的回答？
```

每个章节三选一：保留 / 改 / 删。过完后 6 个章节都为"已确认"，才允许保存到平台。

**最后一步：命名 + 保存**

> 给这个人设起个名字（英文/拼音/中文皆可，会作为平台 personaName）。

AI 把确认好的 markdown 写到临时文件，调用：

```
siluzan-cso persona create --name "<name>" --style-guide-file <临时文件路径>
```

保存成功后反馈：

> 已保存到 CSO 平台（人设 ID：<id>）。下次写稿直接用 `<name>` 引用就行。
>
> 接下来回到主流程，开始写稿。

## 临时人设的写完追问

如果用户走的是"临时人设"路径，写完正文后必须主动问一次：

> 这版稿子你看下——
>
> - 腔调对吗？
> - 这次用的人设我没保存。你喜欢这版的话，我可以保存成正式人设（写到 CSO 平台），下次直接用。要保存吗？

- 用户说要 → 进入 A1 确认流程 → `siluzan-cso persona create`
- 用户说不要 → 不保存，结束
