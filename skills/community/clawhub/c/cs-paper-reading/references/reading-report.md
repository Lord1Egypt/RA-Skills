# Reading Report Template for Computer Science Papers

## Purpose
Integrate findings from all three reading passes into a comprehensive report that synthesizes insights, evaluations, and actionable conclusions.

## Report Structure

### 1. Paper Metadata
```markdown
## Paper Information
- **Title**: [Full title]
- **Authors**: [Key authors]
- **Venue**: [Conference/Journal and year]
- **DOI/URL**: [If available]
- **Reading Date**: [YYYY-MM-DD]
- **Reader**: [Your name/ID]
```

### 2. Executive Summary
```markdown
## Executive Summary
[1-2 sentence summary of the paper's core contribution and significance]
```

### 3. First Pass Integration
```markdown
## First Pass Overview
**Category**: [Paper type]
**Core Problem**: [1 sentence]
**Main Contribution**: [1 sentence]
**Key Related Work**: [2-3 papers/authors]
**Assumptions Validity**: [Yes/No/Partial]
**Relevance Decision**: [Proceed/Stop]
```

### 4. Second Pass Integration
```markdown
## Content Analysis
**Core Contribution**: [2-3 sentences]
**Key Innovation**: [What's new]
**Main Results**: [Quantitative summary]
**Strengths**: [2-3 points]
**Weaknesses**: [2-3 points]
**Related Papers to Read**: [List]
**Relevance Score**: [High/Medium/Low]
```

### 5. Third Pass Integration
```markdown
## Deep Analysis
### Technical Assessment
- **Algorithm/Method**: [Detailed analysis]
- **Complexity**: [Time/Space]
- **Correctness**: [Proof/Empirical]
- **Robustness**: [Sensitivity analysis]

### Experimental Assessment
- **Reproducibility**: [Score 1-5]
- **Statistical Rigor**: [Score 1-5]
- **Baseline Fairness**: [Score 1-5]
- **Evaluation Completeness**: [Score 1-5]

### Innovation Assessment
- **Novelty**: [Score 1-5]
- **Significance**: [Score 1-5]
- **Impact**: [Score 1-5]
- **Generality**: [Score 1-5]
```

### 6. Critical Synthesis
```markdown
## Critical Synthesis
### Strengths
1. [Strength 1 with justification]
2. [Strength 2 with justification]
3. [Strength 3 with justification]

### Weaknesses
1. [Weakness 1 with justification]
2. [Weakness 2 with justification]
3. [Weakness 3 with justification]

### Missing References
1. [Paper 1 - why it's relevant]
2. [Paper 2 - why it's relevant]

### Experimental Issues
1. [Issue 1]
2. [Issue 2]
```

### 7. Future Directions
```markdown
## Future Work Directions
1. [Direction 1 - brief description]
2. [Direction 2 - brief description]
3. [Direction 3 - brief description]
```

### 8. Personal Assessment
```markdown
## Personal Assessment
- **Implementation Feasibility**: [Easy/Medium/Hard]
- **Extension Potential**: [High/Medium/Low]
- **Teaching Value**: [Could be used for teaching?]
- **Collaboration Potential**: [Worth contacting authors?]
- **Overall Quality Score**: [1-10]
```

### 9. Reviewer Perspective (Optional)
```markdown
## Reviewer Perspective
### Overall Assessment
- **Accept/Weak Accept/Weak Reject/Reject**
- **Confidence**: [High/Medium/Low]
- **Soundness**: [Score 1-5]
- **Significance**: [Score 1-5]
- **Clarity**: [Score 1-5]

### Review Summary
- **Main Contribution**: [1-2 sentences]
- **Strengths**: [3-4 bullet points]
- **Weaknesses**: [3-4 bullet points]
- **Questions for Authors**: [2-3 questions]
- **Suggestions for Improvement**: [2-3 suggestions]
```

### 10. References and Further Reading
```markdown
## References
- [List of key references cited in the paper]
- [Additional papers to read based on this paper]
```

## Usage Guidelines

### When to Use This Template
- After completing all three reading passes
- For papers that require comprehensive analysis
- When preparing for literature review, paper review, or research implementation

### How to Fill the Template
1. **First Pass Integration**: Copy summary from first-pass.md
2. **Second Pass Integration**: Copy summary from second-pass.md
3. **Third Pass Integration**: Copy analysis from third-pass.md
4. **Critical Synthesis**: Combine and synthesize insights from all passes
5. **Future Directions**: Identify from third-pass analysis
6. **Personal Assessment**: Add your own evaluation

### Quality Checklist
- [ ] All sections are filled with specific, actionable content
- [ ] Scores and ratings are justified with evidence
- [ ] Strengths and weaknesses are balanced and well-supported
- [ ] Future directions are concrete and feasible
- [ ] Personal assessment is honest and constructive

## Example Usage

### For Literature Survey
- Create one report per paper
- Use reports to compare papers and identify trends
- Synthesize multiple reports into a literature review

### For Paper Review
- Use as basis for conference/journal review
- Focus on reviewer perspective section
- Provide constructive feedback

### For Research Implementation
- Emphasize technical assessment and implementation feasibility
- Identify extension potential and collaboration opportunities
- Use future directions for research planning

## Integration with Three-Pass Method

This template integrates outputs from all three reading passes:

1. **First Pass**: Provides overview and relevance assessment
2. **Second Pass**: Provides content analysis and initial evaluation
3. **Third Pass**: Provides deep technical analysis and critical assessment

The reading report synthesizes these into a comprehensive analysis that supports decision-making and further research.
