# 知识库助手参考文档

查询 1688 分销知识库，获取发货流程、铺货操作、订单管理等分销相关文档。支持按渠道（21 个）和工具（24 个）筛选。

**完整流程：解析意图 → 确认渠道/工具 → 检索知识库 → 判断召回 → 总结输出**

---

## 一、CLI 调用

```bash
# 默认查询（不限渠道和工具）
python3 scripts/cli.py knowledge_helper query --query="铺货流程"

# 指定渠道和工具
python3 scripts/cli.py knowledge_helper query --query="发货" --channel="抖音" --business="自动分销"

# 列出所有支持的渠道
python3 scripts/cli.py knowledge_helper channels

# 列出所有支持的工具
python3 scripts/cli.py knowledge_helper businesses
```

### 接口工具名

`distribution_knowledge_tool`

### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `query` | string | 是 | 用户查询问题，**原封不动传递用户原话，严禁修改** |
| `channel` | string | 否 | 渠道中文名（如 "抖音"），未指定时传 `"default"` |
| `business` | string | 否 | 工具中文名（如 "逸淘分销铺货"），未指定时传 `"default"` |

### 返回结构

`data` 为精排后的文档数组（最多 20 条），每条包含：

| 字段 | 说明 |
|------|------|
| `score` | 相关度分数（0-1，越高越相关） |
| `chunking_val` | 文档正文内容 |
| `source_url` | 原始文档链接 |

```json
{
  "success": true,
  "data": {
    "data": [
      {
        "score": 0.99,
        "chunking_val": "工具名称：逸淘分销铺货;内容：...",
        "source_url": "https://peixun.1688.com/article/xxx"
      }
    ]
  }
}
```

## 二、query 原则（必须严格遵守）

- **不要去掉任何词语**
- **不要改写、精简或翻译用户问题**
- 原封不动地将用户原话作为 query 参数传递
- **唯一例外**：召回结果与用户问题完全无关时，才允许对 query 做极小调整（仅补充 1-2 个关键词），重试仅允许 1 次

## 四、召回结果判断

拿到 `data` 数组后，在总结之前先做相关性判断：

**直接进入总结（满足任一条件）：**
- 最高 `score` ≥ 0.3
- `chunking_val` 内容与用户问题有明显关联（哪怕 score 较低）
- 有任意文档内容提到了用户问题中的核心关键词

**允许调整 query 重试 1 次（需同时满足）：**
- 所有文档的 `score` 都低于 0.15
- 且 `chunking_val` 内容与用户问题完全无关

**重试规则：**
- 仅允许重试 **1 次**，不得多次重试
- query 调整方式：仅提取用户问题中最核心的 1-2 个关键词，不得大幅改写
- 重试后无论结果如何，直接进入总结

## 五、结果展示格式

基于检索到的文档，按以下格式输出：

### 查询总结（详细版）

```
## 查询总结

根据知识库，关于「[用户问题]」的完整操作指南如下：

### 操作步骤

**第 1 步：[步骤名称]**
[详细说明这一步要做什么，在哪里操作]
[如果文档中有图片，必须展示：![操作示意图](图片URL)]

**第 2 步：[步骤名称]**
[详细说明]
[相关图片]

...

### 注意事项
- [从文档中提取的重要提示]
- [常见问题或特殊情况说明]
```

### 详细参考

列出相关度较高的文档（score > 0.3），每篇包含：
- 关键内容（不要过度精简，保留操作细节）
- 文档中的所有图片（使用 markdown 图片语法展示）
- 来源链接（`source_url`）
- 相关度评分（`score`）

### 总结要求

1. **精准解决问题**：直接回答用户的问题，不能泛泛而谈
2. **保留图片信息**：文档中的图片链接必须完整保留，使用 `![图片说明](图片URL)` 格式展示
3. **步骤详细列举**：操作步骤逐步编号，每步清晰说明
4. **内容不能过于简短**：至少包含完整的操作流程，不能省略关键细节
