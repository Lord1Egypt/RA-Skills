# 顾问会诊质量保障体系

## 三层校验机制

```
┌─────────────────────────────────────┐
│  第一层：AI 自我验证 (自动)          │
│  - 检查顾问核心理念匹配度            │
│  - 关键词/问题匹配评分               │
│  - 输出前自检                        │
├─────────────────────────────────────┤
│  第二层：用户反馈 (每次会诊后)        │
│  - 满意度评分 (1-5 星)               │
│  - 准确性评价                        │
│  - 实用性评价                        │
├─────────────────────────────────────┤
│  第三层：效果追踪 (定期回顾)          │
│  - 行动执行率                        │
│  - 决策效果评估                      │
│  - 模型选择优化                      │
└─────────────────────────────────────┘
```

---

## 第一层：AI 自我验证

### 验证逻辑

每次会诊输出前，自动检查：

```python
for 顾问 in 参与会诊的顾问:
    检查回答是否包含:
    - 该顾问的核心问题 (至少 1 个)
    - 该顾问的关键词 (至少 2 个)
    - 该顾问的思维特征
    
计算匹配分数:
- 80-100 分：✅ 通过
- 60-79 分：⚠️ 警告（需要改进）
- <60 分：❌ 失败（重新生成）
```

## 第二层：用户反馈

### 每次会诊后自动询问

```
━━━━━━━━━━━━━━━━━━━━━
会诊完成！请评价：

⭐️ 满意度：[1⭐️ 2⭐️ 3⭐️ 4⭐️ 5⭐️]
📊 准确性：[低 中 高]
💡 实用性：[低 中 高]
🎯 模型选择：[合适 太强 太弱]

[直接回复评分或跳过]
━━━━━━━━━━━━━━━━━━━━━
```

### 反馈记录

保存到 `memory/顾问会诊/feedback-log.md`：

```markdown
## 反馈记录

| 日期 | 主题 | 满意度 | 准确性 | 实用性 | 模型 | 备注 |
|-----|------|-------|-------|-------|------|------|
| 03-26 | 快团团决策 | ⭐️⭐️⭐️⭐️ | 高 | 高 | Sonnet | 马斯克部分很到位 |
| 03-25 | 产品设计 | ⭐️⭐️⭐️ | 中 | 高 | Qwen | Jobs 部分不够深入 |
```

### 负面反馈处理

如果评分 <3 星：

```
感谢反馈！请问：
1. 哪个顾问的观点最不满意？
2. 希望加强哪方面？
3. 下次应该用哪个模型？（Qwen/Sonnet/Opus）

[帮助我改进]
```

- [ ] 行动 3

- 全部完成

## 持续优化循环

```
会诊输出
    ↓
用户反馈
    ↓
记录分析
    ↓
发现模式（如"马斯克部分总被说太温和"）
    ↓
调整提示词
    ↓
下次改进
```

### 示例优化

**问题发现：**
```
反馈显示：马斯克的部分经常被评价为"不够激进"
```

**优化行动：**
```
调整马斯克的提示词：
- 增加"本周必须开始"的紧迫感
- 强调"第一性原理"的拆解
- 加入更多挑战性提问
```

**验证：**
```
下次会诊后检查马斯克部分的评分是否提升
```

## 特殊情况处理

### 顾问观点冲突

```
⚠️ 检测到观点冲突

德鲁克：建议聚焦核心客户
巴菲特：建议等待更好时机
马斯克：建议立即开始

冲突处理：
1. 识别冲突点
2. 分析各自前提
3. 给出平衡建议
4. 让用户选择倾向

你的倾向是？
- 稳健优先（听巴菲特的）
- 行动优先（听马斯克的）
- 平衡方案
```

### 信息不足

```
⚠️ 信息不足，无法准确分析

缺少的关键信息：
- [信息 1]
- [信息 2]

建议：
1. 补充信息后重新会诊
2. 先做轻量级分析
3. 安排信息收集行动

选择？[补充信息] [继续分析] [取消]
```
