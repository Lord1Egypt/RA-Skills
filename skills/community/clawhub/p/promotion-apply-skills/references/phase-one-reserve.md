# 阶段一：排期预占

本文件用于创建推广的第一阶段：从运营/申请人自然语言中尽量自动解析字段，完成创建和预占。主目标是减少运营补填项。

## 运营/申请人必须提供

| 字段 | 说明 |
|---|---|
| 项目名称 | 从运营/申请人消息中提取核心项目名 |
| 项目等级 | S+ / S / A / B |
| 时间段 | 起止时间 |

## 数字员工自动解析

新流程中，推广名称由后台第一步勾选 4 个前缀字段后自动生成。数字员工不要让运营/申请人手工拼完整推广名称，只需要把字段选对；如果当前接口仍要求 `name`，也应由数字员工按规则生成，不让运营/申请人填完整字符串。

| 字段 | 解析方式 |
|---|---|
| 项目类型 | 按关键词映射 |
| 投放资源 | 按关键词映射；仅决定名称前缀，不决定配置方式 |
| 内容大类 / 版权类型 / 战略类型 | 见“名称前缀”规则 |

默认值：

| 字段 | 默认值 |
|---|---|
| 项目负责人 | 项目申请人 + 王可；如果接口只支持单个 `projectOwner`，用项目申请人，王可通过管理员补权限或服务端支持字段处理 |
| 是否保密 | 否 |
| 创建人二级部门 | 内容运营部 |

## 时间段与多天排期

排期预占只有跨自然日时才可能拆分。运营/申请人给出同一天内的小时段（例如 8:00-12:00）时，必须保持为一条排期，不要拆，也不用就拆分问题追问。运营/申请人给出跨自然日时间段时，数字员工先给出推荐拆分方案，并让运营/申请人确认后再创建/预占；不能未经确认直接拆，也不能把跨天区间静默当作一条连续排期提交。

拆分规则：

1. 同一天：直接使用用户给出的开始时间和结束时间；例如 `2026-06-13 08:00:00` 至 `2026-06-13 12:00:00` 只创建 1 条排期。只给日期时按 `00:00:00` 至 `23:59:59`。
2. 跨自然日：先按自然日给出推荐拆分方案，等运营/申请人确认后再执行。
   - 首日：使用用户给出的开始时间，到当天 `23:59:59`。
   - 中间日：`00:00:00` 到 `23:59:59`。
   - 末日：`00:00:00` 到用户给出的结束时间；如果只给日期，则到 `23:59:59`。
3. 如果运营/申请人明确要求“每天都投同一个时间段”，例如 6月13日到15日每天 8:00-12:00，则按每天 `08:00:00` 至 `12:00:00` 生成逐日排期。没有这种明确要求时，推荐使用首日开始时间、中间全天、末日结束时间的拆法。
4. 最多拆 7 个自然日。只有跨自然日时才应用这个限制；同一天内任意小时段都算 1 天。超过 7 个自然日时不要创建，提示“当前一次最多支持 7 天排期，请缩短时间或拆成多次”。
5. 轻量确认时只展示推荐拆分后的日期摘要，不要输出冗长 JSON；必须问“是否按这个拆分创建？如果要每天同一时段或不拆，请直接说明”。

示例：

```text
用户：6月13日 8:00 到 6月15日 23:59:59
拆分：
- 2026-06-13 08:00:00 至 2026-06-13 23:59:59
- 2026-06-14 00:00:00 至 2026-06-14 23:59:59
- 2026-06-15 00:00:00 至 2026-06-15 23:59:59
```

## 项目名提取规则

从运营/申请人消息中提取核心项目名：

1. 去掉时间段，包括数字-数字、MM.DD 等格式。
2. 去掉端口词，例如全端、移动端。
3. 去掉等级词，例如 S级、S+、A级、B级。
4. 去掉尾部动词，例如推广、投放、上线。
5. 剩下的作为核心项目名。

## 名称前缀

空降 / 有票（云村有票）四个前缀字段固定，运营/申请人不需要选择：

| 前缀字段 | 固定值 |
|---|---|
| 内容大类 | 潜力 |
| 版权类型 | 全网 |
| 战略类型 | 其他 |
| 投放资源 | 其他 |

名称前缀固定为 `【潜力/全网/其他/其他】`。

其它资源类型（歌曲 / 普通专辑 / 数专 / H5 / MV / 歌单）：

| 前缀字段 | 来源 |
|---|---|
| 内容大类 | 运营/申请人选：头部 / 战略 / 潜力 / 品牌。数字员工不自动判，只给规则提示 |
| 版权类型 | 运营/申请人提供；可半自动抓取运营/申请人消息里的独家 / 首发 / 全网 / 自制 / 后发 |
| 战略类型 | 运营/申请人选：艺粉 / Kpop / 嘻哈 / 欧西 / 日音/二次元 / 其他 |

内容大类提示：

- 品牌：品牌合作（云音乐 x 外部 IP/品牌联动）；或品牌内容（年度音乐奖、半年/年度说唱奖、硬地系列、音乐人荣誉里程碑、音乐人特写、云村乐话、云上叨叨等品牌向内容）。
- 头部：艺人等级=A 且区域等级>A；或艺人/音乐人未入驻站内；或项目等级>A 的版权合作内容。
- 战略：重点垂类（欧西/Kpop/日音&二次元/嘻哈/艺粉）中的非头部艺人内容。
- 潜力：艺人等级<A 且非重点垂类的潜力内容。

## 推广名称生成

格式：

```text
【内容大类/版权类型/战略类型/投放资源】项目核心名 –【内容运营部】
```

示例：

```text
【头部/全网/其他/单曲】周杰伦新歌 –【内容运营部】
```

| 段 | 来源 |
|---|---|
| 内容大类 | 名称前缀规则 |
| 版权类型 | 名称前缀规则 |
| 战略类型 | 名称前缀规则 |
| 投放资源 | 投放资源标签映射 |
| 项目核心名 | 运营/申请人消息提取 |
| `–【内容运营部】` | 固定默认 |

## 轻量补问模板

其它资源类型：预填投放资源 / 版权（抓到时），运营/申请人补内容大类 + 战略类型：

```text
收到，确认以下预占信息：
• 推广名称：【{内容大类?}/{版权类型/抓到则填}/{战略类型?}/{投放资源}】{核心项目名} –【内容运营部】
• 项目等级：{S+/S/A/B}
• 项目类型：{枚举值}
• 时间段：{起} - {止}
• 客户端：{按资源类型自动选择}

请补充：
① 内容大类：头部 / 战略 / 潜力 / 品牌（选一个）
   提示：品牌=品牌合作或品牌向内容；头部=A级且区域>A / 未入驻 / 项目>A的版权合作；战略=重点垂类非头部；潜力=非A且非重点垂类
② 战略类型：艺粉 / Kpop / 嘻哈 / 欧西 / 日音/二次元 / 其他（选一个）
③ 版权类型：自制 / 独家 / 首发 / 全网 / 后发（上方未填或需更正时）
④ 关联艺人ID：（如有）※仅歌曲版权宣发/艺人合作宣发时出现
```
