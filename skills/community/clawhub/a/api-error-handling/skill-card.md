## Description: <br>
Guides agents through API error-handling patterns across languages and layers, including operational versus programmer errors, retries, circuit breakers, error boundaries, HTTP responses, graceful degradation, and structured logging. <br>

This skill is ready for commercial/non-commercial use. <br>

## Known Risks and Mitigations: <br>
Risk: Guidance may be applied incorrectly to production APIs, causing misleading error responses or weak failure handling. <br>
Mitigation: Review recommendations against service requirements, test failure paths, and keep sensitive internals out of client-facing responses. <br>
Risk: Installing from a moving remote source can change skill content over time. <br>
Mitigation: Review the source before installation and pin to a trusted commit or release when installing globally. <br>
