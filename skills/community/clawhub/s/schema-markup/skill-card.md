Developers, SEO practitioners, and site maintainers use this skill to add, fix, audit, and validate schema.org structured data for search rich results across static sites, React/Next.js applications, and CMS or WordPress sites. <br>

## Known Risks and Mitigations: <br>
Risk: Optional install commands and copied examples may be run or adapted without reviewing their source or runtime context. <br>
Mitigation: Review the source before running optional npx install commands and adapt examples only after confirming they match the target site and deployment workflow. <br>
Risk: JSON-LD examples embedded in application code can expose untrusted or incorrectly serialized page data. <br>
Mitigation: Use trusted or safely serialized values when adapting React and JSON-LD examples, and validate output with rich-result and schema validators before deployment. <br>
Risk: Structured data that does not match visible page content can create misleading search markup. <br>
Mitigation: Keep schema aligned with visible page content and validate required properties, URLs, dates, prices, and supported rich-result types before release. <br>
