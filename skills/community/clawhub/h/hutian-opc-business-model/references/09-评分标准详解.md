# 评分标准详解

## 概述

本文档提供商业模式画布九维度评分的详细标准，每个维度包含5个分数段（1-2分、3-4分、5-6分、7-8分、9-10分）的具体描述。

---

## 1. 客户细分（Customer Segments）

### 定义
企业想要接触和服务的不同人群或组织。

### 评分维度
- 客户画像完整度
- 市场细分合理性
- 市场规模可量化
- 客户验证情况

### 分数段标准

#### 1-2分（极差）
```
- 客户定义模糊或混乱
- 完全没有明确的目標群体
- "所有人都可以是客户"
- 无客户画像
- 无市场规模估算
```

#### 3-4分（较差）
```
- 有基本客户概念
- 但细分非常粗糙
- 客户画像不完整
- 无明确的市场规模数据
- 客户定位模糊
```

#### 5-6分（及格）
```
- 有基本的客户画像
- 能区分1-2个客户群体
- 有定性的市场描述
- 无精准的量化数据
- 客户差异化定位不够
```

#### 7-8分（良好）
```
- 客户细分清晰
- 有2-3个明确的客户群体
- 差异化定位明确
- 有定量的市场规模
- 客户画像包含多个维度
```

#### 9-10分（优秀）
```
- 精准客户画像
- 多维度细分（如行业+规模+行为）
- 可量化的市场规模（含TAM/SAM/SOM）
- 有客户验证数据（如已有用户）
- 清晰的客户优先级排序
- 明确的客户获取策略
```

## 2. 价值主张（Value Propositions）

### 定义
解决客户问题或满足客户需求的系列产品和服务。

### 评分维度
- 价值主张清晰度
- 差异化程度
- 客户需求匹配度
- 竞争壁垒

### 分数段标准

#### 1-2分（极差）
```
- 价值主张完全不清晰
- 无法回答"为什么选你"
- 没有解决明确的问题
- 与竞品无差异化
- 客户看不到价值
```

#### 3-4分（较差）
```
- 有基本的价值概念
- 但表达混乱或模糊
- 无法清晰描述差异化
- 客户需求匹配度不明
- 价值主张缺乏支撑
```

#### 5-6分（及格）
```
- 有基本的价值主张
- 能说清解决什么问题
- 有一定的差异化描述
- 客户需求匹配度一般
- 但缺乏数据支撑
```

#### 7-8分（良好）
```
- 价值主张清晰明确
- 有明确的差异化优势
- 与客户需求高度匹配
- 有案例或数据支撑
- 能回答"为什么选你"
```

#### 9-10分（优秀）
```
- 强价值主张，差异化明显
- 有护城河（专利/技术/品牌）
- 有验证数据支撑
- 客户复购或推荐意愿高
- 可持续的竞争优势
```

---

## 3. 渠道通路（Channels）

### 定义
如何接触、抵达客户并传递价值。

### 评分维度
- 渠道完整性
- 渠道多元化
- 获客成本
- 渠道效率

### 分数段标准

#### 1-2分（极差）
```
- 完全没有渠道规划
- 不知道如何触达客户
- 没有销售能力
- 依赖客户自己找上门
- 无任何交付渠道
```

#### 3-4分（较差）
```
- 有基本的触达想法
- 但没有具体方案
- 依赖单一渠道
- 渠道效率未知
- 无获客成本概念
```

#### 5-6分（及格）
```
- 有基本的渠道规划
- 有1-2种明确的渠道
- 依赖单一渠道为主
- 有基本的获客思路
- 渠道效率一般
```

#### 7-8分（良好）
```
- 多渠道布局
- 渠道优先级明确
- 有获客成本数据
- 渠道效率可衡量
- 有渠道优化计划
```

#### 9-10分（优秀）
```
- 全渠道体系完善
- 线上线下渠道协同
- 有详细的CAC数据
- 渠道ROI可优化
- 有渠道验证案例
- 边际获客成本递减
```

## 4. 客户关系（Customer Relationships）

### 定义
与客户建立的关系类型。

### 评分维度
- 客户关系深度
- 留存策略有效性
- 客户价值指标（LTV、CAC）
- 客户满意度

### 分数段标准

#### 1-2分（极差）
```
- 完全不考虑客户关系
- 卖完就不管
- 无任何留存意识
- 无客户维护机制
- 客户流失无应对
```

#### 3-4分（较差）
```
- 有基本的客户服务
- 但无系统性维护
- 客户关系策略缺失
- 无留存数据
- 客户满意度未知
```

#### 5-6分（及格）
```
- 有基本的客户服务
- 有基本的客户维护意识
- 客户关系分层不足
- 有基本的复购策略
- 客户满意度一般
```

#### 7-8分（良好）
```
- 客户关系分层管理
- 有明确的留存策略
- 有复购或续费机制
- 有NPS或满意度数据
- 客户生命周期管理思路
```

#### 9-10分（优秀）
```
- 完善的客户生命周期管理
- 有LTV/CAC数据
- 有NPS或客户满意度指标
- 客户推荐机制
- 社群或会员体系
- 客户成功团队
```

---

## 5. 收入来源（Revenue Streams）

### 定义
从客户群体中获得的收入。

### 评分维度
- 盈利模式清晰度
- 收入来源多元化
- 定价合理性
- 单位经济模型

### 分数段标准

#### 1-2分（极差）
```
- 盈利模式完全不清
- 不知道怎么赚钱
- 没有收入来源
- 没有定价策略
- 商业模式不可持续
```

#### 3-4分（较差）
```
- 有模糊的盈利概念
- 但无法说清具体方式
- 收入来源单一
- 定价无依据
- 单位经济为负
```

#### 5-6分（及格）
```
- 有基本的盈利模式
- 能说清收入来源
- 有基本的定价策略
- 收入来源相对单一
- 单位经济勉强平衡
```

#### 7-8分（良好）
```
- 盈利模式清晰
- 多元收入来源
- 定价策略合理
- 有单位经济数据
- 收入增长可预期
```

#### 9-10分（优秀）
```
- 收入模型经过验证
- 多元收入来源组合
- 定价策略精细化
- 有详细的单位经济模型
- 边际收益为正
- 收入增长飞轮
```

## 6. 核心资源（Key Resources）

### 定义
执行商业模式所需的关键资产。

### 评分维度
- 资源识别完整性
- 资源壁垒强度
- 资源独占性
- 资源可持续性

### 分数段标准

#### 1-2分（极差）
```
- 核心资源未识别
- 不知道自己有什么
- 无独特资源
- 资源可被轻易复制
- 资源不可持续
```

#### 3-4分（较差）
```
- 有基本的资源意识
- 但识别不完整
- 资源壁垒弱
- 资源可替代性强
- 资源可持续性差
```

#### 5-6分（及格）
```
- 识别了主要资源
- 但不够系统
- 有一定的资源壁垒
- 部分资源独占
- 资源可持续性一般
```

#### 7-8分（良好）
```
- 核心资源识别清晰
- 有明确的获取路径
- 有资源壁垒
- 资源有一定独占性
- 资源可持续
```

#### 9-10分（优秀）
```
- 核心资源体系完整
- 强资源壁垒
- 资源完全独占
- 资源可持续性强
- 资源难以复制
```

## 7. 关键业务（Key Activities）

### 定义
执行商业模式所需的关键活动。

### 评分维度
- 业务识别清晰度
- 业务优先级合理性
- 业务闭环完整性
- 业务可扩展性

### 分数段标准

#### 1-2分（极差）
```
- 关键业务不明确
- 不知道该做什么
- 业务眉毛胡子一把抓
- 无业务优先级
- 无可扩展性
```

#### 3-4分（较差）
```
- 有基本的业务方向
- 但关键业务不清晰
- 业务优先级混乱
- 业务闭环不完整
- 可扩展性差
```

#### 5-6分（及格）
```
- 识别了部分关键业务
- 有基本的业务优先级
- 业务闭环基本完整
- 可扩展性一般
- 业务效率待提升
```

#### 7-8分（良好）
```
- 关键业务清晰
- 业务优先级明确
- 业务闭环完整
- 有业务效率指标
- 可扩展性较好
```

#### 9-10分（优秀）
```
- 关键业务突出
- 业务优先级科学
- 业务闭环完善
- 有详细的效率指标
- 可扩展性强
- 业务可复制
```

#### 3-4分（较差）

- 有基本的成本概念
- 但识别不完整
- 成本结构不合理
- 边际成本高
- 盈利模型不清
```

#### 5-6分（及格）
```
- 有基本的成本估算
- 主要成本项识别
- 成本结构基本合理
- 边际成本一般

## 综合评级标准

| 评级 | 分数区间 | 含义 | 建议 |
|------|----------|------|------|
| A | 8-10分 | 商业模式成熟 | 具备规模化条件，可加速扩张 |
| B | 6-7分 | 商业模式基本成立 | 需补短板，优化薄弱环节 |
| C | 4-5分 | 商业模式存在重大缺陷 | 需重新设计商业模式 |
| D | 1-3分 | 商业模式不成立 | 建议推翻重来，重新思考 |

---

## 评分原则

1. **客观公正**：严格按标准评分，不主观拔高
2. **数据支撑**：评分必须有原文或用户确认依据
3. **综合判断**：结合多个维度综合评估
4. **保守原则**：无法确认的信息取较低分
5. **一致性**：同类型内容评分标准一致
