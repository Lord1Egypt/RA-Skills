# Automation Safety Rules

所有负责维护或扩展此自动化脚本的开发者，必须严格遵守以下安全边界。这不仅是为了保护用户账号安全，也是为了符合 OpenClaw / ClawHub 的合规要求。

## 1. 绝对禁止的操作 (NEVER)
- **绝对禁止点击“发布”按钮**。
- **绝对禁止**在后台（headless=True）执行发文操作。
- **绝对禁止**读取、保存、打印、导出、持久化任何形式的用户凭证（Cookie、LocalStorage、SessionStorage、Token、Password）。
- **绝对禁止**使用 `browser_context.storage_state` 记录和恢复登录态。
- **绝对禁止**调用非头条官方公开的私有 API 进行“黑客式发文”。
- **绝对禁止**编写任何绕过验证码（CAPTCHA）、滑块验证或风控提示的代码。
- **绝对禁止**读取 `/output/toutiao/` 以外的任何系统文件或敏感路径。
- **绝对禁止**执行任何自动评论、点赞、私信、关注等互动操作。

## 2. 必须保证的操作 (MUST)
- 浏览器必须在用户面前可见 (`headless=False`)。
- 当出现登录框、安全验证时，脚本必须暂停并交由人工处理。
- 脚本结束时，必须停留在编辑页面，强制用户进行“人工最后审阅 (Human Review)”。
- 如果设置了尝试保存草稿（非默认），也必须确保不触发发布。
- 脚本必须在文件头部显式声明：本工具仅供草稿填写辅助，非自动化发布工具。
