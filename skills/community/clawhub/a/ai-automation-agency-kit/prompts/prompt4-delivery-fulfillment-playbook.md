# Prompt 4 — Delivery & Fulfillment Playbook

## Purpose
Build the complete operational infrastructure to deliver projects professionally, get great testimonials, and set up recurring retainer revenue.

---

## The Prompt

```
You are an agency operations consultant. Help me build a complete delivery and fulfillment playbook for my AI automation agency.

My agency:
Agency name: [AGENCY NAME]
Primary tools I use: [e.g., "Zapier, Make, Google Workspace, Airtable, Slack"]
My niche: [YOUR NICHE]

Current client:
Client name: [CLIENT NAME or "Client A" for the template]
Client industry: [INDUSTRY]
Project type: [e.g., "operations reporting automation, 5 workflows"]
Contract value: [$AMOUNT]
Start date: [DATE]
Agreed timeline: [e.g., "4 weeks"]

Build me a complete delivery playbook:

1. CLIENT ONBOARDING SEQUENCE
a) Welcome email (send same day as signed contract):
   - Warm opener (specific to their company)
   - What happens next (numbered, 5 steps)
   - What you need from them in the first 48 hours
   - Calendly/booking link for kickoff call
   - P.S. with one reassurance sentence

b) Discovery questionnaire (15 questions in 4 sections):
   Section 1: Business context (company size, team structure, key stakeholders)
   Section 2: Current process (step-by-step of what's being automated, who does it, how long it takes)
   Section 3: Tools and access (every system we'll touch, who has admin access)
   Section 4: Success definition (what does "done well" look like to them?)

c) Kickoff call agenda (60 minutes):
   - Min 0-5: introductions + tone setting
   - Min 5-20: problem walkthrough (let them talk)
   - Min 20-40: solution walkthrough (you present the plan)
   - Min 40-55: access and logistics (accounts, credentials, shared drive)
   - Min 55-60: next steps and weekly check-in schedule

d) Shared folder structure (Google Drive or Notion):
   - Folder names and what goes in each
   - File naming convention
   - Permission levels (client view vs. edit)

2. PROJECT DOCUMENTATION TEMPLATES
a) Scope of Work document:
   - Project overview (2 paragraphs)
   - Deliverables list (numbered, specific)
   - Out of scope list (numbered, specific)
   - Timeline with milestones
   - Acceptance criteria (how you both know it's done)
   - Signature block

b) Change Order Form:
   - Request description
   - Why it falls outside original scope
   - Additional time required
   - Additional cost
   - Impact on existing timeline
   - Client signature line

c) Weekly Progress Update Format (email template, send every Friday):
   - Completed this week (bullets)
   - In progress (bullets with % complete)
   - Blocked / waiting on client (bullets with specific ask)
   - Next week plan (bullets)
   - Any decisions needed before Monday

3. THREE-PHASE DELIVERY FRAMEWORK
For each phase (Discovery / Build / Launch):
- Duration (weeks)
- Owner (you vs. client)
- Deliverables list
- Sign-off gate (what the client approves before next phase starts)
- Common mistakes to avoid in this phase

Phase 1 — Discovery (Week 1-2)
Phase 2 — Build (Week 2-3)
Phase 3 — Launch & Training (Week 4)

4. SOP PACK — 5 STANDARD OPERATING PROCEDURES
For each SOP, write a complete step-by-step document:

SOP 1: Multi-Step Zapier Workflow Setup
SOP 2: Google Sheets Auto-Populate Dashboard
SOP 3: Slack/Email Alert Integration
SOP 4: Scheduled Report Automation (daily/weekly digest)
SOP 5: Data Pipeline Between Two Systems (e.g., CRM → Sheets → Airtable)

Each SOP format:
- Purpose (one sentence)
- Prerequisites (access, accounts, data)
- Step-by-step instructions (numbered, include screenshots note)
- Testing checklist (5 steps to verify it works)
- Common errors and fixes
- Maintenance notes (what to check monthly)

5. RESULTS REPORTING TEMPLATE
End-of-project report for client:
- Project overview (what was built)
- Before state (quantified — hours/week, error rate, etc.)
- After state (quantified — new time, error reduction)
- ROI calculation (formula + filled example)
- Screenshots of live workflows
- Recommendations for next phase (2-3 ideas)
- Testimonial request section (specific, easy, not generic)

6. OFFBOARDING + RETENTION SYSTEM
a) Project close email (send day of final delivery):
   - Celebrate the win (specific results from the results report)
   - Provide everything: links, credentials, documentation folder
   - Set expectation for 30-day check-in
   - Plant the seed for next project (one natural sentence, not a pitch)

b) 30-day check-in email:
   - How is everything running?
   - One value-add: share something they can use
   - Soft retainer offer: "If you want me keeping an eye on the automations..."

c) Retainer pitch script (after 30-day check-in or at project close):
   - Monitoring tier ($300/month): error alerts, monthly review, 2 minor updates
   - Active tier ($750/month): monitoring + 5 new workflow updates + priority response
   - VIP tier ($1,500/month): monitoring + unlimited updates + new project each quarter
   - How to frame: "You've invested $X in automations — makes sense to protect that."

d) Referral ask email (send at 30-day check-in if results are strong):
   - Specific, easy ask
   - Give them the exact words to use when referring you
   - Offer incentive if appropriate (one month free, gift card)
```

---

## Example Output (condensed)

**Welcome Email Opening:**  
"Welcome to the team, Marcus — really looking forward to what we're building together at NV Tech Automation. Your ops team is about to get 15 hours per week back. Here's what happens next..."

**Discovery Questionnaire — Process Section:**  
Q6: Walk me through the nightly F&B reconciliation from start to finish — what's the first action and what's the last?  
Q7: Which systems are involved? (POS, PMS, spreadsheet name, email?)  
Q8: Who does this task today, and what happens if they're out sick?  
Q9: What's the most error-prone step — where do mistakes usually happen?

**Phase 2 Build Sign-off Gate:**  
"Before moving to Launch, client approves: (1) all workflows run in test mode with no errors, (2) dashboard shows correct data for past 7 days, (3) client admin credentials are transferred and tested."

**Retainer Pitch:**  
"You've invested $5,500 in these five automations. The monitoring tier at $300/month means I catch any errors before they become problems, handle small updates as your business changes, and check in monthly to make sure everything's still running clean. Most clients find it pays for itself the first time something breaks before they'd have noticed."
