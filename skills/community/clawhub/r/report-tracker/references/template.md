# 研报跟踪存档文件模板与格式规范

## 文件头（每份存档文件开头）

```markdown
# 大开门股票池·研报跟踪信息存档
> 规则：每次收到研报，提取关键信息后，关联到现有大开门股票池的标的，将关联信息追加到此文件。
> 股票池认准：[当前池子标的列表，以 investment/大开门股票池.md 为准]
> 格式：时间戳 | 研报来源 | 关联标的 | 关联强度 | 关键信息 | 新增跟踪指标 | 对盈利预测/PE的影响
```

## 单份研报关联记录格式

```markdown
---

## [日期] [时间] | [研报标题/来源]

### 关联标的：[标的名称] [代码]（[大开门状态]）

| 维度 | 内容 |
|------|------|
| **关联强度** | [🔴 强 / 🟡 中 / ⚪ 弱] — [一句话说明原因] |
| **关键信息** | [分点列出研报中与该标的直接相关的要点，①②③格式] |
| **新增跟踪指标** | [分点列出因这份研报新增需要跟踪的指标] |
| **对盈利预测/PE的影响** | [明确：上调/下调/维持/无影响 + 具体数字变化] |
| **来源文件** | [workspace/reports/下的研报分析文件路径] |
```

### 关联强度定义（严格执行）

| 强度 | 判定标准 | 盈利预测动作 |
|:---|:---|:---|
| 🔴 **强** | 研报逻辑**直接改变**该标的的收入/成本/利润模型。例如：产品涨价直接传导至公司主营产品、产能缺口直接对应公司扩产计划、客户订单直接锁定公司出货量。 | **必须修正**盈利预测和PE |
| 🟡 **中** | 研报提供**基本面验证**或**新增需跟踪的变量**。例如：行业景气度获额外证据、技术路线变化需关注但不立即改变利润模型、竞争对手动态需跟踪。 | **新增跟踪指标**，不修正盈利预测 |
| ⚪ **弱** | 研报内容与该标的**无直接关联**。例如：光芯片研报对覆铜板标的、CCL研报对算力租赁标的。 | 明确标注"无直接关联" |

## 多份研报合并格式

同一天多份研报，可合并为一条记录，每个标的一个子节：

```markdown
---

## [日期] | [研报1标题] + [研报2标题]

### 关联标的：[标的A]
[关联强度/关键信息/新增指标/盈利影响 表格]

### 关联标的：[标的B]
[同上]
```

## 累计跟踪指标总表格式（文件末尾）

```markdown
---

## 📋 累计跟踪指标总表（按标的分类）

### [标的名称] [代码]

| 优先级 | 跟踪指标 | 首次提出时间 | 来源研报 | 当前状态 |
|:---|:---|:---|:---|:---|
| 🔴 P0 | [指标1] | [日期] | [研报来源] | [状态] |
| 🟡 P1 | [指标2] | [日期] | [研报来源] | [状态] |
| 🟢 P2 | [指标3] | [日期] | [研报来源] | [状态] |
```

### 优先级定义

| 优先级 | 含义 | 跟踪频率 |
|:---|:---|:---|
| 🔴 P0 | 直接影响盈利预测的核心变量 | 每季度/每月 |
| 🟡 P1 | 影响估值判断或需要持续观察的变量 | 每季度/事件驱动 |
| 🟢 P2 | 远期风险或备选情景 | 每半年/年度复盘 |

### 当前状态填写规范

| 状态值 | 含义 |
|:---|:---|
| 跟踪中 | 已纳入常规跟踪，等待新数据 |
| 待[时间] | 明确等待某个时间点（如"待2026年7月Q2财报"） |
| **新增** | 本次研报新增，首次纳入跟踪 |
| 已验证/已失效 | 指标已被证实或证伪，可归档 |

## 盈利预测修正记录格式

当关联强度为🔴强、需要修正盈利预测时，用以下格式记录：

```markdown
### 盈利预测修正记录：[标的名称]

| 项目 | 原预测 | 修正后 | 修正原因 |
|:---|:---|:---|:---|
| 2027E利润 | [原数字] | [新数字] | [研报具体逻辑] |
| 2027E PE | [原数字] | [新数字] | 利润上修/下修 |
| 目标市值 | [原数字] | [新数字] | [分部估值调整说明] |
| 修正时间 | — | [日期] | [研报来源] |
```

## 文件管理规范

1. **追加不覆盖**：所有新记录写在文件末尾，`---` 分隔。
2. **定期归档**：当文件超过500行时，将历史记录（>3个月前的）归档到 `investment/archive/研报跟踪存档_[年份].md`。
3. **同步更新**：修改存档文件后，同步检查 `investment/大开门股票池.md` 中的盈利预测是否需要同步修正。
4. **最后更新行**：文件末尾保留 `*最后更新：[日期]*` 标记。

## 自检清单模板（每次存档后必须附上）

在存档文件每次更新后，在文件末尾保留：

```markdown

*存档规则：每次新研报提取关联信息后，在此文件末尾追加，并更新累计跟踪指标总表。*
*最后更新：[日期]*
```

每次分析完研报，必须向用户发送以下格式的回执：

```
📄 研报分析回执
━━━━━━━━━━━━━━━━━━━━━━
研报来源：[文件名/URL]
分析时间：YYYY-MM-DD HH:MM

关联标的：
├─ [标的名] [代码] — [🔴强/🟡中/⚪弱]
│   关键发现：[1-2句话核心发现]
│   盈利预测修正：[原X亿 → 新Y亿] / 无
│   新增跟踪指标：[指标名1]、[指标名2] / 无
└─ [标的名] [代码] — [🔴强/🟡中/⚪弱]
    关键发现：...
    盈利预测修正：无
    新增跟踪指标：无

存档位置：investment/大开门股票池_研报跟踪存档.md
━━━━━━━━━━━━━━━━━━━━━━

✅ 自检清单
□ 只关联了股票池中的标的？ 是
□ 追加了存档（追加模式）？ 是
□ 写了"对盈利预测/PE的影响"？ 是
□ 更新了累计跟踪指标总表？ 是
```

**回执原则**：
- 每只标的一行，关联强度一眼看清
- 盈利预测修正用"原→新"格式，无修正写"无"
- 新增指标用顿号分隔，无新增写"无"
- 存档位置明确写出文件路径

## 禁止事项清单

| 禁止行为 | 正确替代 |
|:---|:---|
| 不写"关联强度"列 | 必须明确 🔴/🟡/⚪ |
| 不写"对盈利预测/PE的影响" | 必须明确：上调/下调/维持/无影响 |
| 用"可能"、"大概"模糊表述 | 用具体数字范围或"维持不变" |
| 覆盖历史记录 | 追加模式，写在文件末尾 |
| 遗漏"新增跟踪指标" | 每份研报至少列出1-3个新增跟踪点 |
| 混淆不同标的的关联 | 每只标的单独一个小节 |
| 遗漏Step 3存档 | 强制写入大开门股票池_研报跟踪存档.md |
| 不发回执 | 必须发结构化回执 |
| 不勾选自检清单 | 强制勾选，未勾=未完成 |
