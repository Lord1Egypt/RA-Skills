# 8 Phase 详细检查清单（v2）

逐 Phase 对照，不要漏项。迭代时回溯某个 Phase，只需重新过该 Phase 的检查清单。

## Phase 1 —— 需求澄清 ☐

- [ ] 确认出行日期（精确到星期）
- [ ] 确认人数 & 关系（影响房型、座椅数）
- [ ] 确认机票状态（已定 / 待定 / PNR 号）
- [ ] 确认预算范围（人均总预算 / 每晚酒店上限）
- [ ] 确认交通方式偏好（自驾 / 公共 / 跟团）
- [ ] 确认体验项目强需求（潜水 / 看极光 / 极限运动）
- [ ] 确认酒店会籍（IHG / Marriott / Hilton / GHA / Hyatt）
- [ ] 确认作息偏好 —— **精确到小时**（如 10:00 自然醒），早于此标 ⚠️
- [ ] 确认同步需求（是否跨设备共享 Todo）
- [ ] 确认特殊需求（饮食 / 医疗 / 宠物 / 无障碍）

## Phase 2 —— 资源调研 ☐

### 小红书 MCP
- [ ] 至少 3 组关键词搜索
- [ ] 至少查看 5 篇发布 <3 个月的攻略
- [ ] **评论区翻完**找真实反馈（不是只看正文）
- [ ] 识别商业推广帖 vs 真实体验帖

### 交叉验证
- [ ] 🔴 **每条关键信息 2+ 来源确认**
- [ ] 自驾距离/时间以 **Google Maps 实测**为准（不信小红书文字描述）
- [ ] 渡轮/船票班次查**官网/电话**确认
- [ ] 配额制项目查**运营方**确认（不只信小红书）
- [ ] 景点坐标从 **Google Maps / Wanderlog** 官方链接提取

### 路线对比
- [ ] 至少对比 2 条自驾路线
- [ ] 标注每条路线的 km 数 + 预计耗时 + 路况
- [ ] 山区/岛屿特别注意：海岸线绕远可能反而更快

### 酒店 & 服务
- [ ] 酒店至少 3 方比价
- [ ] 渡轮/船票查官网班次（记录电话/购票方式）
- [ ] 签证查官方（不只信小红书）
- [ ] 天气查**历年同期**数据

## Phase 3 —— 方案决策 ☐

- [ ] 每晚住宿确认：地点 / 价格 / 确认号 / **停车** / 早餐 / 取消政策
- [ ] 交通方案全链条：机场↔酒店、酒店↔酒店、酒店↔景点
- [ ] 自驾保险方案确定（第三方 vs 租车行）
- [ ] 一日游项目选好版本（self-drive / pickup）+ 沟通关键词
- [ ] 配额制项目是否提前锁名额
- [ ] 餐厅候选每天午晚各 2 家（主选 + Plan B）+ 选择理由
- [ ] 所有 Plan B 精确到**替代班次/路线/费用**

## Phase 4 —— 行程书 + 话术 ☐

### 行程书
- [ ] 文件名带日期和版本号
- [ ] **顶部版本变更日志**（每次改动写 WHY）
- [ ] 封面摘要 + 预算总览 + 航班 + 住宿清单 + 交通清单
- [ ] 每天 Day by Day 带 **时间/地点/动作/费用** + 导航关键词
- [ ] 🔴 每个 POI 有**选择理由** + **避坑提示**
- [ ] 🔴 Plan B 精确到班次/时间/路线/费用
- [ ] 应急联系方式（使领馆、保险、租车）
- [ ] 费用拆分到付款人
- [ ] 早于用户自然醒时间的活动标 ⚠️ 并说明原因

### 实操话术
- [ ] 🔴 租车取还车话术（拒绝加保险、争取新车、验车签字）
- [ ] 🔴 一日游/活动运营商沟通关键词
- [ ] 事故处理 SOP（报警→拍照→通知→索赔）

## 🎯 交付标准（DoD）

**合格**：
- 用户能在手机 + 电脑上看同一个指挥页
- Todo 勾选在刷新后保留
- 行程书可独立打印（断网可用）
- Plan B 覆盖所有高风险节点，精确到班次/路线

**优秀**：
- 多页架构 + 三链路部署
- 每个 POI 有选择理由 + 避坑 + 导航关键词
- 关键场景有英文话术（照念即用）
- 版本日志追溯所有决策变更
- 坐标全部经过卫星图验证
- Todo 跨设备实时同步
- 图标混搭（Noto 3D + Lucide）视觉精致

**不及格**：
- 关键节点缺 Plan B 或 Plan B 是"改室内"式废话
- POI 只有"地点+时间"没有选择理由
- 坐标估算/复用，没验证
- 地图只有大头针没有路线
- 自驾距离引用小红书没经 Google Maps 验证
- 单页 HTML 超 80KB 没分页
