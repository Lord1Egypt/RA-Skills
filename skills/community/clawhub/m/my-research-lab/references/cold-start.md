# 冷启动完整流程

冷启动有 5 个步骤，每一步都不能跳过。

## 步骤 1：确认研究方向

```
读取用户记忆 → 获取背景、关注领域、已有配置

记忆中已有方向 → "我看到你关注 [XXX]，围绕这个方向？"
记忆中没有 → "你希望研究院的研究方向是什么？"
用户回"对"或说新方向即可。
```

## 步骤 2：建立两条生命线（强制）

**没有推送渠道 + 沉淀空间 = 研究院不算启动。**

### 生命线一：推送渠道

自动检测（静默执行，不问用户）：
- 记忆中是否已有配置好的推送渠道？→ 有则直接复用
- email-sender Skill / 邮箱 MCP → 可用？
- 企微 Webhook → 可用？
- 公众号 Skill → 可用？
- 其他 IM 推送 → 可用？

检测结果：
- 已有可用渠道 → 告知用户 "推送将发送到 [渠道]"
- 没有可用渠道 → 必须引导配置：
  ```
  研究院需要一个推送渠道才能运转。你希望简报发到哪里？
   1. 邮箱（推荐，内容最完整）
   2. 企微群（适合快速浏览）
   3. 其他
  ```
- 配置完成前不进入下一步。

### 生命线二：沉淀空间

自动检测（静默执行）：
- 记忆中是否已有配置好的沉淀空间？→ 有则直接复用
- 腾讯文档 MCP/Skill 可用？→ 优先推荐
- WeDoc Skill 可用？→ 推荐
- 本地文件系统 → 始终可用（保底）

检测结果：
- 发现腾讯文档可用 → "检测到你有腾讯文档，建一个独立空间保存研究院所有产出？"
- 发现其他存储平台 → 类似推荐
- 都没有 → "研究院产出将存在本地 [路径]，后续你装了云端存储我会自动迁移"

沉淀空间需要存储的内容：
- 每日发现简报（完整版）
- 头脑实验记录
- 动手实践项目文档
- 大师点评存档
- 蒸馏的大师 skill 本身（如有）

## 步骤 3：大师扫描与排班（静默执行）

```
扫描路径：
  1. ~/.openclaw/workspace/skills/nuwa-skill/examples/*-perspective/SKILL.md
  2. ~/.codebuddy/skills/*-perspective/SKILL.md
  3. 用户记忆中手动配置的大师列表

扫描流程：
  □ 列出所有 *-perspective 目录
  □ 读取每个 SKILL.md 的 description
  □ 与研究方向做语义匹配，≥0.7 的自动绑定

大师排班（大师 ≥3 位时启用轮值）：
  · 院长作为PM，安排大师参与哪些环节：
    - 每日发现点评：每天轮值 2 位大师（避免每天都是同一批声音）
    - 头脑实验：根据话题选择最相关的 2-3 位大师
    - 动手实践：选择与项目最匹配的 1-2 位大师做顾问
  · 轮值表写入配置，每周自动轮换
  · 用户可手动调整："这周让 [大师X] 多参与"

输出：
  有匹配 → "发现 [N] 位大师可以加入顾问团：[列表]。已安排轮值排班。"
  无匹配 → 静默跳过
```

## 步骤 3.5：领域专属信源 skill 扫描（按需推荐）

详见 `references/source-skills-registry.md`。

简要：根据用户研究方向匹配领域，命中则推荐安装该领域的信源 skill；不匹配则静默跳过。

## 步骤 4：创建定时任务（强制）

**不创建定时任务 = 研究院没有启动。**

必须创建的定时任务：

1. **每日发现（每天执行）**
   - 名称：`[研究院名称] · 每日发现`
   - 频率：每天 [用户指定时间，默认 9:00]
   - 超时：≥20 分钟（采集+质检+大师点评+生成+推送+沉淀+自进化）
   - prompt：见 `references/cron-prompts.md`

2. **头脑实验（每周执行，可选每3天）**
   - 名称：`[研究院名称] · 头脑实验`
   - 频率：每周 [用户指定，默认周三]
   - 超时：≥15 分钟
   - prompt：见 `references/cron-prompts.md`

创建后验证：
- 定时任务确实已创建？（不是"说了会创建"而是真的创建了）
- 超时设置足够长？
- 推送渠道配置正确？
- 沉淀空间可写入？

全部验证通过 → 进入步骤 5
任何一项失败 → 修复后重新验证，不能带病启动

## 步骤 5：立刻执行第一次采集

启动后直接执行，不等明天。让用户立刻看到研究院在运转。
