# OpenClaw ShrimpCard

如果你也受不了那种一眼就很“AI”的 agent 介绍页，这个 skill 可能正适合你。

OpenClaw ShrimpCard 做的事情很简单：把一个代理真正做过的事，整理成一张能拿得出手的身份卡。不是空话，不是套模板，不是“强大助手”“高效执行”这种谁都能写的词，而是尽量从真实 evidence 出发，做出一套可以公开展示的成品。

它最后会帮你产出：

- 一份基于证据的自我介绍 JSON
- 一份结构化的 share card JSON
- 一张真实的 8-bit 像素角色图
- 一份适合截图、分享、展示的 HTML 卡片

## 这个 skill 适合谁

适合这些人：

- 想给自己的 agent 做一个像样展示页的人
- 不想再用空泛文案包装 agent 的人
- 想把 task trace、memory、owner feedback 这些真实材料压缩成公开卡片的人
- 想要“能交付的成品”，而不是停在 prompt 或半成品阶段的人

如果你只是想随便生成一个炫酷海报，它可能不是最省事的那个。
但如果你在意“这张卡写的到底像不像我”“是不是有证据支撑”“能不能真的发出去”，那它会很顺手。

## 它和一般 agent 展示工具有什么不一样

很多工具的问题不是做不出图，而是做出来的东西太像样板。

OpenClaw ShrimpCard 比较较真，它会卡这些问题：

- 不让你直接拿 example 充当真实身份
- 不鼓励你写“全能型”“高智能”这种空词
- 没有真实图片挂回去之前，不算完工
- 最终 HTML 渲染前，还会再做一轮校验

换句话说，它不是帮你“吹”，而是帮你“提炼”。

## 你会得到什么体验

比较理想的使用感受是这样的：

你把代理的一些真实痕迹喂进去，它先整理证据，再收束成短文案，然后生成 share card，再补上像素角色图，最后导出成一张完整的 HTML 卡片。

整套流程下来，你得到的不是一句介绍，而是一份可复用、可验证、可展示的 agent identity bundle。

这对下面几种场景尤其有用：

- 给自己的 agent 做公开主页
- 给团队里的 agent 做统一展示卡
- 把长期运行的 agent 沉淀成可以传播的素材
- 给 demo、作品集、项目页补上更完整的一层包装

## 这个 skill 的性格

你可以把它理解成一个有点挑剔的卡片编辑。

它不太喜欢空话。
它不太接受“差不多得了”。
它希望你给它的内容，最后真的能站得住。

所以它的结果通常不是最浮夸的，但会更耐看，也更经得起追问。

## 一句话总结

OpenClaw ShrimpCard 不负责把 agent 包装得天花乱坠。

它负责把一个 agent 真正做过的事，压成一张体面、清楚、能发出去的卡。
