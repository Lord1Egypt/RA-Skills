## Description: <br>
Improve code readability, structure, and maintainability by cleaning up duplication, simplifying flow, renaming, or reorganizing without changing behavior. <br>

This skill is ready for commercial/non-commercial use. <br>

## Publisher: <br>
[sf0799](https://clawhub.ai/user/sf0799) <br>

### License/Terms of Use: <br>

## Use Case: <br>
Developers and engineers use this skill to refactor local code for clearer structure, simpler control flow, reduced duplication, and better naming while preserving existing behavior and compatibility. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Refactoring can unintentionally change behavior or compatibility. <br>
Mitigation: Use version control, review diffs, preserve public interfaces unless explicitly allowed, and run relevant tests or checks after changes. <br>
Risk: The agent may need read and edit access to the target repository. <br>
Mitigation: Install only when that repository access is acceptable and review any proposed or applied changes before relying on them. <br>
