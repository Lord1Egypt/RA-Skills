# 数仓与多维表格同步

## 同步类型

两种方向：

- **数仓→多维表格**：数仓表数据每日同步至多维表格，同步前清空多维表格原数据
- **多维表格→数仓**：多维表格数据每日同步至数仓表，全量覆盖（分区表同步至对应业务日期分区）

## 申请表单填写指南

### 核心配置类

| 字段 | 填写规则 | 说明 |
|:-----|:---------|:-----|
| 需求类型 | 「数仓_to_多维表格」或「多维表格_to_数仓」 | 根据方向选择 |
| 多维表格管理员 | 仅数仓→多维表格需要 | 明确当前多维表格的管理员名单 |
| 文档密级确认 | 需选「是」 | 仅支持L4及以上密级 |
| 上游任务节点ID | 可选填`default`或具体节点ID | `default`不校验数据就绪状态；多个ID用分号空格分隔 |
| 记录容量限制 | 2万行（默认）或5万行 | 超5万行会报错 |
| 数仓表筛选条件 | 可选 | 不填`where`，不填分区信息；多个条件用`and`连接 |

### 业务标识类

- **需求概要**：格式 `数仓表{表名}同步至多维表格【{多维表格名称}】`
- **需求详细说明**：描述同步业务场景及用途
- **业务线**：选择归属业务线

### 数据源配置类

**多维表格URL**：打开目标多维表格的数据表页面，复制浏览器地址栏完整URL

**权限配置**（必须配置「可管理权限」）：
- 方式一（推荐）：点击「･･･」→「更多」→「添加文档应用」，搜索「数仓与多维表格同步」配置可管理权限
- 方式二（不推荐）：点击「分享」→「管理协作者」，搜索「数仓与多维表格同步」群组

**ODPS表名**：带空间名的完整表名，格式 `{项目名}.{表名}`

> ⚠️ 仅支持生产环境表，测试环境表无法通过审批

### 映射关系配置

**字段映射格式**：
```
{多维表格字段1:数仓表字段1,多维表格字段2:数仓表字段2,...}
```

> ⚠️ 注意事项：
> 1. 字段名禁止使用特殊符号（@、#、¥等）
> 2. 配置完成后**禁止修改或删除**对应字段（新增字段不影响现有任务）
> 3. 多维表格字段类型需与数仓表字段数据类型大致匹配
> 4. 多维表格**不支持的字段类型**：人员、单向关联、双向关联、查找引用、公式、地理位置、群组、创建人、修改人
> 5. 多维表格→数仓方向不支持的字段类型会以JSON文本格式同步

### 调度配置

- **调度时间描述**：格式如 `每天早上5:30同步数据`
- 分区表规则：仅支持单分区同步，分区值格式`yyyyMMdd`，默认同步前一日（d-1）数据

## 审批流程

**数仓→多维表格**：直属上级审批 → 数仓同学校验数据敏感性 → 敏感数据需额外加签业务负责人

**多维表格→数仓**：按常规流程审批

## 注意事项

1. 提交前确认多维表格容量是否够用（最高5万行）
2. 映射关系确认后不要随意修改字段
3. 如需变更历史同步任务配置，联系黄家杰
4. 分区表每日同步最新分区，非分区表同步全量数据
