## Known Risks and Mitigations: <br>
Risk: Wallet-connected blockchain templates and snippets can create real financial risk if copied as-is. <br>
Mitigation: Review blockchain code before use, replace mock data with verified on-chain reads, test on testnets, simulate transactions, and require clear user confirmation before any mainnet wallet signature or transaction. <br>
Risk: Generated transaction scaffolds may contain unverified payment targets, calldata, or token approval behavior. <br>
Mitigation: Verify Relayr payment targets and calldata, avoid unlimited token approvals unless explicitly intended, and keep user confirmation clear before execution. <br>
