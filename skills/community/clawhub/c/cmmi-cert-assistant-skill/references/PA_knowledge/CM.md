- **英文**：Configuration Management
- **中文**：配置管理

## CMMI V3.0 要求要点

### 核心目的
确保项目的配置项（代码、文档、环境）被规范管理，变更受控，基线可回溯。

### 关键实践（Specific Practices）
| 实践编号 | 实践内容 | 说明 |
|---------|---------|------|
| CM 1.1 | 建立配置管理计划 | 配置管理计划文档 |
| CM 1.2 | 识别配置项 | 配置项清单、基线定义 |
| CM 1.3 | 建立配置管理系统 | 版本控制工具、权限管理 |
| CM 2.1 | 跟踪配置项变更 | 变更请求、审批记录 |
| CM 2.2 | 控制配置项变更 | 变更影响分析、审批流程 |
| CM 2.3 | 建立和维护基线 | 基线发布记录 |
| CM 3.1 | 发布配置审计报告 | 配置审计记录、一致性报告 |

### 通用实践（Generic Practices）

## 用户文档 → PA 匹配规则

### 高概率匹配（主要PA）
- 文档名含"配置管理"、"版本控制"、"变更记录" → **CM 主要匹配**
- 文档含代码仓库截图、分支策略 → **CM 主要匹配**

### 中概率匹配（次要PA）
- 文档含变更审批流程 → 同时匹配 **RDM 2.3**（需求变更管理）
- 文档含基线发布记录 → 同时匹配 **PLAN**（计划基线）

## 差距检查清单（用于差距分析）

### CM 1.x — 策划
- [ ] 是否有配置管理计划？
- [ ] 是否识别了所有配置项（代码/文档/环境）？
- [ ] 是否有版本控制工具且权限配置正确？

### CM 2.x — 控制
- [ ] 配置项变更是否有审批记录？
- [ ] 是否有基线（Baseline）建立记录？
- [ ] 变更影响分析是否有记录？

### CM 3.x — 审计
- [ ] 是否定期发布配置审计报告？
- [ ] 配置项与实际交付物是否一致？

## 文档转换指导（模式C）

### 转换后的文档结构（配置管理计划模板）
```markdown
# 配置管理计划

## 1. 概述
（目的、范围、引用标准）

## 2. 配置管理工具
（工具名称、版本、访问地址、权限策略）

## 3. 配置项识别
| 配置项ID | 名称 | 类型 | 负责人 | 基线策略 |
|----------|------|------|--------|----------|

## 4. 变更控制流程
（变更请求 → 影响分析 → 审批 → 实施 → 验证）

## 5. 基线管理
（基线名称、建立条件、发布流程、回溯策略）

## 6. 配置审计
（审计频率、审计内容、审计报告模板）

## 7. 版本记录
（计划版本、修订日期、修订内容、审批人）
```

## 评估访谈常见问题（模式D）

### 针对配置管理员/SCM的问题
1. "你们的代码是怎么管理的？用什么工具？"
2. "配置项变更的流程是什么？有审批记录吗？"
3. "请展示最近一次基线发布的记录。"
4. "配置审计是怎么做的？频率如何？"

### 评分要点
- **CM 1.1-1.3**：是否有系统化的配置管理策划
- **CM 2.1-2.3**：变更是否受控，基线是否建立
- **CM 3.1**：是否定期审计并发布报告
