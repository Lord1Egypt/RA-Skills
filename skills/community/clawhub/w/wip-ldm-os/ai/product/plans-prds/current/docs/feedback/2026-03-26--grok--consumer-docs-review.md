# Grok Feedback: Consumer Docs Review

**Date:** 2026-03-26
**Source:** Grok 4.1
**Context:** Review of docs.wip.computer as consumer-facing docs (not technical GitHub docs)

## Summary

The docs already do a solid job staying high-level, benefit-focused, and jargon-light. Sells the dream without overwhelming people with code. Tone is friendly and approachable.

## Recommendations

### 1. Fill the Getting Started Section (Biggest Win)

Currently empty. Should be a 3-5 minute onboarding flow:
- What you will need (just a Mac? 1Password?)
- One-click or copy-paste install steps (with screenshots/GIFs)
- First 60 seconds checklist: open the app, see first AI message, confirm Memory Crystal is live
- End with You are done! and link to Next: Try these 3 everyday things
- Should be default landing experience or most prominent button

### 2. Add Real User Stories and Everyday Use-Cases

Current content is feature-list heavy. Consumers connect with I want to do X, how does this help?
- New top-level Use Cases or For You section with short scenarios:
  - I have 5 different AIs. How do they share the same long-term memory?
  - I am writing a novel. Watch Dream Weaver turn scattered notes into chapters.
  - I want my AI to text me updates. Here is how AI Messages + iMessage works.
- Each use case gets 1-2 screenshots or a 10-second GIF + Try it now button
- Turns docs from product brochure into here is how my life gets better

### 3. Heavy Visuals + Simple Explanations

Site is still text-heavy. Consumers scan, they do not read walls.
- One big hero diagram on homepage (you + your AIs + Memory Crystal + Dream Weaver)
- Screenshots/GIFs on every feature page
- Short 15-30 second videos for trickier flows
- Keep GitHub technical diagrams out. Just the friendly consumer view.

### 4. Clear Bridges to Technical GitHub Docs

Zero links to GitHub currently.
- Bottom of every section: Want the full technical details? -> Link to repo
- For Developers and Tinkerers footer link to wip-ldm-os repo
- Technical Deep-Dive callout boxes on pages like Memory Crystal

### 5. Small Consumer-Friendly Polish

- Audience note at top: These docs are for regular users. Developers: full specs live in GitHub.
- FAQ section (Is my data private? Do I need to be online? What if I switch computers?)
- Troubleshooting page with common fixes in simple language
- Footer links: wip.computer, community, GitHub org, Report an issue
- Pre-load Ask AI with consumer questions

## Bottom Line

The bones are excellent. Clean Mintlify design + Ask AI is a killer feature for consumer docs. Just needs more hand-holding, visuals, and explicit this is the friendly version, tech stuff is over there signposting.
