> 自动解析论文核心信息构建 PPT，无需用户重新描述论文内容。

# 学术汇报 PPT 生成技能 (Academic PPT Builder)

当用户提供学术汇报需求时，自动进入以下流程。

## 输入模式识别

收到用户输入后，先判断来源类型：

| 输入特征 | 触发模式 |
|----------|----------|
| 包含 `## 汇报素材摘要` 标题块 | **摘要直填模式**（推荐，来自 paper-reader） |
| 包含 `---` 分隔符 + 表格 | **摘要直填模式** |
| 论文标题 + 背景描述，无结构化数据 | **从零构建模式** |
| 仅有"帮我生成 PPT"等简短指令 | **从零构建模式** |

---

## 模式 A：摘要直填模式（优先）

当用户粘贴了 paper-reader 的"汇报素材摘要"时：

### Step 1：解析摘要结构

读取以下字段：

```
论文标题（中文）/（英文）
发表期刊 + 年份
一句话叙事弧
核心创新点 ×3
关键证据链（对应图号 → 结论）
数据摘要（样本量、统计方法、p 值）
局限性 + 未来方向
```

### Step 2：补充汇报信息

摘要缺少以下字段时，主动询问：

| 字段 | 说明 |
|------|------|
| 汇报类型 | 论文答辩 / 顶会 Oral / 基金汇报 |
| 时长 | 分钟数（影响页数） |
| 目标受众 | 领域专家 / 基金委 / 答辩委员会 |

### Step 3：直接生成 PPT 结构

叙事弧和创新点已存在于摘要中，可直接映射到 PPT 页面：

```
封面页 ← 论文标题 + 作者 + 期刊
↓ ↓ ↓
背景页 ← 叙事弧"背景"部分
↓ ↓ ↓
问题页 ← 叙事弧"问题"部分
↓ ↓ ↓
假设页 ← 核心创新点（创新点即假设）
↓ ↓ ↓
方法页 ← 从摘要"关键证据链"推断实验设计
↓ ↓ ↓
结果页 ×N ← 每个关键证据链对应一页
↓ ↓ ↓
意义页 ← 叙事弧"意义"部分 + 局限性
↓ ↓ ↓
讨论/未来 ← 未来方向
```

## 模式 B：从零构建模式

当用户未提供结构化摘要时，按以下流程构建。

### 一、基础信息收集

确认以下信息（若用户未提供则主动询问）：

| 字段 | 说明 |
|------|------|
| 汇报题目 | 中文 + 英文 |
| 汇报类型 | 论文答辩 / 顶会 Oral / 基金汇报 |
| 时长 | 分钟数（对应页数 16-20 页） |
| 目标受众 | 领域专家 / 基金委 / 答辩委员会 |

### 二、叙事主线构建（核心）

先于任何页面设计，构建全篇一句话叙事弧：

```
格式：背景 → 问题 → 假设 → 方法 → 结论 → 意义
字数：30-50 字
检验：去掉所有术语后，普通人能否听懂你在做什么？
```

**快速构建法（用户可填写也可让 AI 推断）：**
```
我研究 [领域] 中的 [现象A]
但 [痛点：现有理论无法解释什么]
因此我提出 [核心假设]
用 [方法/模型] 验证
最终发现 [结论]
这意味着 [领域意义]
```

### 三、故事弧标签分配

| 页码 | 阶段 | 叙事功能 |
|------|------|----------|
| 1 | Why（痛点）| 领域存在什么未解决的科学空白 |
| 2 | What（假设）| 核心创新点，一句话概括 |
| 3-4 | How（方法）| 实验范式设计逻辑 |
| 5-8 | So What（结果）| 关键证据链，逐步论证 |
| 9 | Now What（意义）| 贡献、局限、下一步 |
| 10+ | 附录 | 方法细节、补充数据 |

### 四、每一页详细内容

#### 结果页（核心，每页 ≤6 行正文）

```markdown
## [页码] [叙事标题]（中文 / English）

故事弧位置：结果 - 第 X 个关键证据

布局：☐ 图表主导   ☐ 左右分栏   ☐ 上下对比   ☐ 流程图

核心可视化：
┌─────────────────────────────────────────────┐
│  图表类型   │  如：散点图 + 线性回归          │
│  数据维度   │  n=42 mice, 3 conditions, 重复3次│
│  统计元素   │  ***p<0.001, 两因素 ANOVA, SEM  │
│  视觉引导   │  箭头标注 A→B，高亮框圈出关键区  │
│  图表叙事   │  观众应看到：___________________ │
└─────────────────────────────────────────────┘

正文要点：
• 关键结论 1（与图表直接对应，附统计量）
• 关键结论 2（与图表直接对应，附统计量）
• 控制实验 / 阴性对照说明
• 统计方法与效应量（Cohen's d / η²）

动画：淡入关键标注（如 ***p<0.001），时长 ≤1.5s
```

#### 方法页（必须可复现）

```markdown
## [页码] [方法标题]

实验设计要素：
┌─────────────────────────────────────────────┐
│  样本量        │  n = ? / 组，共 ? 组          │
│  纳入标准      │  具体描述                      │
│  排除标准      │  具体描述（无则写"无预设排除"）│
│  对照设计      │  同窝对照 / 双盲 / 随机         │
│  干预细节      │  剂量、途径、时间、频率         │
│  统计方法      │  检验方法 + 校正方式            │
│  软件版本      │  Python 3.9 / R 4.2 / ImageJ   │
│  可重复性声明  │  原始数据已上传 DOI: xxx        │
└─────────────────────────────────────────────┘

伦理：IACUC #2023-0128 / 知情同意声明
```

#### 封面页

```
布局：左文字区 + 右主图（水印效果）
元素：主标题 / 副标题 / 作者 / 单位 / 基金标注 / 日期
风格：标题带故事感（如"看见未来的海马"）
```

### 五、硬性规范（每页必检）

#### 图表自包含清单

- [ ] 图标题（英文 below figure，≤15 words）
- [ ] 轴标签（中文 + 单位）
- [ ] 图例（双语或英文）
- [ ] 统计标注（p 值 + 检验方法）
- [ ] 误差棒（SEM 或 SD，说明）
- [ ] 显著性标记（\*p<0.05, \*\*p<0.01, \*\*\*p<0.001）
- [ ] n 标注（每组样本量）

#### 方法页最低标准

- 至少 3-4 页（含 1 页统计方法专页）
- 必须覆盖：动物模型 / 被试描述、干预方案、数据采集、分析流程
- 提供试剂/设备品牌型号
- 伦理审批声明（IACUC / IRB 编号）

#### 结果页叙事规则

- 文字只提炼最强结论，不复述图注
- 每张图配一句话"观众应看到什么"
- 区分"作者结论"vs."推测性解读"
- 禁止把相关性表述为因果性

#### 视觉风格

- 配色：Nature/Science 学术蓝灰（#2C3E50 / #34495E / #ECF0F1）
- 字体：Helvetica / Arial，正文不加粗，标题加粗
- 色盲友好：禁用红绿对比，改用蓝橙 / 灰绿
- 留白：信息密度高但呼吸感充足

### 六、输出结构

```
1. 整体叙事弧（一句话 + 故事弧标签表）
2. 封面页（标题 + 代表图描述）
3. 结果页 1-N（按证据链顺序）
4. 方法页（至少 3 页）
5. 讨论/意义页
6. 致谢 + Q&A
7. 补充材料（可选）
```

### 七、可选追加内容

根据用户需求补充：

- ☐ 数据可用性声明
- ☐ 参考文献格式（本领域 top journal）
- ☐ 作者贡献声明（CRediT 格式）
- ☐ 利益冲突声明
- ☐ 补充数据页

### 八、核心原则

> **让每一页都为最终结论服务。** 不是堆砌内容，而是用故事线串联证据链。
> 优先接收 paper-reader 的结构化摘要，减少用户重复劳动。
