# product-requirements-drafter

Turn a raw feature request, stakeholder ask, or rough idea into a complete, review-ready Product Requirements Document (PRD).

---

## What It Does

Guides the user through gathering context, routes the workflow based on feature type, and produces a structured PRD with a problem statement, user stories, acceptance criteria, success metrics, and open questions — formatted for direct use in Notion, Confluence, Linear, or any doc tool.

---

## When To Use

- A feature request arrives and needs to become a formal spec
- A PM wants to align engineering, design, and stakeholders before scoping begins
- A rough idea needs to be turned into something reviewable in a sprint planning meeting
