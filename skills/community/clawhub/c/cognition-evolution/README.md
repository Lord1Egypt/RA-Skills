# 认知进化系统 · 使用说明

## 功能

- ✅ 会话结束自动复盘，无需手动总结
- ✅ 识别关键决策、教训、洞见
- ✅ 沉淀为可复用原则，下次遇到同类问题自动调用
- ✅ 区分已解决和未解决的问题

## 安装

```bash
cp -r cognition-evolution ~/.openclaw/skills/
```

## 使用

**全自动，无需干预。** 每次会话结束后自动触发。

**手动触发：**
```
复盘一下这次对话
这次有什么认知升级？
总结我们刚才的讨论
```

## 复盘输出示例

```markdown
## 认知沉淀 2026-05-03

### 决策
- 选用 PostgreSQL 而非 MongoDB ← 因为事务需求高、团队熟悉

### 教训
- 遇到技术选型时，先分析业务场景，再看流行度

### 洞见
- 关系型数据库在复杂查询上的性能经常被低估

### 未解决
- 部署方案待定 ← 等运维同事确认服务器环境
```

## 与 longterm-memory 的区别

| 系统 | 侧重点 | 写入时机 |
|------|-------|---------|
| 永久记忆 | 事实+偏好 | 随时 |
| 认知进化 | 原则+教训 | 会话结束 |

两者配合使用：永久记忆记住"是什么"，认知进化记住"怎么做更好"。
