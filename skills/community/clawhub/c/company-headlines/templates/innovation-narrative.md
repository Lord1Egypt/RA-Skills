# 创新叙事模板

## 适用场景
技术升级、研发成果、产品迭代、工艺突破

## 事件级别
L2 标准（日常迭代）或 L3 重要（重大突破）

## 结构骨架

```
问题引入 → 技术解耦 → 数据验证 → 价值落地
```

## 填写表

| 段落 | 写什么 | 字数 | 铁律 |
|------|--------|------|------|
| 问题引入 | 行业/客户遇到的真实问题 | 40-60字 | 写具体痛点，不泛泛 |
| 技术解耦 | 做了什么，分层讲 | 80-120字 | 让外行也能看懂方向 |
| 数据验证 | 1-2个关键指标对比 | 40-60字 | **必带数字**，升级前 vs 升级后 |
| 价值落地 | 已经在哪里用了 | 60-80字 | 有场景，有真实使用 |

## 变量槽

| 变量 | 说明 | 示例 |
|------|------|------|
| {{PAIN_POINT}} | 具体痛点 | 30家门店月底对账要3天 |
| {{SOLUTION}} | 解决方案概述 | 4层校验对账逻辑 |
| {{BEFORE}} | 升级前指标 | 对账3天 |
| {{AFTER}} | 升级后指标 | 对账40分钟 |
| {{DEPLOY_SCENE}} | 落地场景 | 17家连锁零售客户 |

## 自检

- [ ] 问题引入是写具体场景还是泛泛的"效率低"？
- [ ] 技术解耦让外行看懂了吗？（给非技术同事读一遍）
- [ ] 数据对比有了吗？（升级前 vs 升级后）
- [ ] 价值落地是"已经在用"还是"未来将"？（要前者）
- [ ] 禁用词检查通过？

## 常见问题

| 问题 | 修正 |
|------|------|
| 堆术语 | 先用一句话概括，再展开 |
| "性能显著提升" | 给具体数字和场景 |
| "未来将广泛应用于..." | 写已经在哪用了 |

## 示例

> **问题引入**：财务月底关账，最怕Excel公式错一行。连锁超市的会计，30家门店数据要对三天。
>
> **技术解耦**：我们把对账逻辑拆成4层——数据清洗、匹配校验、差异标记、自动冲销。每层只做一件事。
>
> **数据验证**：对账时间从3天缩到40分钟。错误率从1/300降到1/5000。
>
> **价值落地**：这套方案已经在17家连锁零售客户跑了半年。上个月，他们中最早接入的那家，第一次实现了当月0差异关账。
