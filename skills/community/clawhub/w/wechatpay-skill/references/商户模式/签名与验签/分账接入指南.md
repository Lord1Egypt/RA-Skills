# 分账接入指南

> 本文档说明微信支付分账功能的基本概念和接入要点

## 分账概述

分账是指将一笔支付订单的资金分配给多个接收方（如平台佣金、服务费、多商户结算等）。

**适用场景**：
- 电商平台分账给商家
- 外卖平台分账给骑手/商家
- 会员费分成
- 多方服务费结算

## 分账前提

### 1. 下单时必须传分账标识

```json
{
    "profit_sharing": true
}
```

### 2. 资金到位

支付成功后需等待至少1分钟才能发起分账

### 3. 配置分账接收方

分账接收方必须先添加：
- 个人：需微信支付实名
- 商户：需已在微信支付开户

## 分账流程

```
1. 添加分账接收方
       │
       ▼
2. 请求分账（资金分配）
       │
       ▼
3. 查询分账结果
       │
       ├─── 成功 → 完成
       │
       └─── 等待 → 轮询查询
```

## 添加分账接收方

```http
POST /v3/profitsharing/receivers/add
```

```json
{
    "appid": "wx1234567890",
    "type": "MERCHANT_ID",  // 或PERSONAL_WECHATID
    "account": "1234567890",
    "relation_type": "DISTRIBUTOR"  // 或STORE_OWNER等
}
```

## 请求分账

```http
POST /v3/profitsharing/orders
```

```json
{
    "transaction_id": "微信订单号",
    "out_order_no": "分账单号（全局唯一）",
    "receivers": [
        {
            "type": "MERCHANT_ID",
            "account": "子商户号",
            "amount": 100,
            "description": "分账描述"
        }
    ]
}
```

**注意**：
- 分账总额不超过可分账金额（扣除微信手续费后）
- 单次最多50个接收方
- out_order_no必须全局唯一

## 查询分账结果

```http
GET /v3/profitsharing/orders/{transaction_id}
```

**结果判断**：
- `result=SUCCESS`：接收方分账成功
- `result=WAITING`：等待处理
- `result=CLOSED`：已关闭

## 解冻剩余资金

如果订单**不需要分账**，必须调用解冻接口：

```http
POST /v3/profitsharing/orders/unfreeze
```

否则资金会冻结30天（自动解冻）。

## 分账回退

退款前需先回退已分出的资金：

```http
POST /v3/profitsharing/returnorders
```

**注意**：
- 只能回退分账给商户类型的接收方
- 必须原分账接收方回退
- 只能回退剩余可用资金

## 常见问题

| 问题 | 解决方案 |
|------|----------|
| 无法分账 | 检查下单时是否传profit_sharing=true |
| 金额超限 | 计算实际可分账金额（扣除手续费） |
| 资金冻结 | 调用解冻接口或等待30天 |
| 分账失败 | 检查接收方是否已添加、是否微信支付实名 |
