# 需求评审清单

## 1. 完整性
- [ ] 是否有业务背景？
- [ ] 是否有目标用户？
- [ ] 是否有需求范围和非范围？
- [ ] 是否有主流程和异常流程？
- [ ] 是否有字段清单？
- [ ] 是否有业务规则？
- [ ] 是否有权限说明？
- [ ] 是否有验收标准？

## 2. 一致性
- [ ] PRD、原型、字段清单是否一致？
- [ ] 流程与页面跳转是否一致？
- [ ] 业务规则与验收标准是否一致？

## 3. 可实现性
- [ ] 数据来源是否明确？
- [ ] 接口依赖是否明确？
- [ ] 技术路径是否可落地？

## 4. 可验收性
- [ ] 是否存在无法测试的描述？
- [ ] 是否存在“智能、便捷、高效”等泛化词但无指标？

## 5. AI 风险
- [ ] 是否定义准确率/召回率/采纳率等指标？
- [ ] 是否有人工兜底？
- [ ] 是否有错误样本回流？
- [ ] 是否有低置信度处理？
