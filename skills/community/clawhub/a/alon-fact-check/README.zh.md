# alon-fact-check

[English](./README.md)

`alon-fact-check` 是一个事实核查 skill，用于核验 URL 或粘贴文本中的事实性声明。

它会提取明确、可验证的声明，用权威来源核查，并返回包含结论、可信度评分、来源链接和关键限定条件的结构化报告。

## 快速安装

```bash
npx skills add alondotsh/alon-skills --skill alon-fact-check
```

## 适用场景

当用户想要完成以下任务时，适合使用这个 skill：

- 判断某个事实性声明是真的、假的、误导性的，还是证据不足。
- 检查 URL、文章、帖子或粘贴段落的事实可靠性。
- 将声明与官方、原始、同行评审或专业事实核查来源进行比对。
- 理解过度概括或误导性声明中更准确的窄版本。

不要用于观点润色、一般研究摘要、生活方式建议，或个性化医疗/法律/金融建议。

## 输入类型

### URL

对于 URL 输入，skill 会先读取并总结来源页面，然后再进行事实核查。

最终报告包含：

- `来源摘要`：页面本身主张或声称的内容。
- `摘要`：核查证据显示的结论。

这样可以避免外部搜索结果污染声明提取。

### 粘贴文本

对于长文本、类似文章的文本、访谈稿或多段内容，skill 会先写 `内容摘要`，再提取声明。

对于简短直接声明，会省略摘要字段，让报告保持紧凑。

## 工作流程

1. 将输入解析为 URL 或文本。
2. 当输入是 URL 或长文本时，先理解来源内容。
3. 提取 1-5 条明确、可验证的声明。
4. 搜索权威来源，并对争议声明加入反向或辟谣取向搜索。
5. 将每条声明判定为真实、误导、错误或未证实。
6. 最后一次性组装干净报告，避免重复声明段落或来源错配。

## 输出结构

报告采用结论优先结构：

```text
事实核查报告

来源：...
来源摘要 / 内容摘要：...
总体评估：...
摘要：...
来源说明：...

声明 #1
声明：...
结论：...
可信度：...
说明：...
来源：
- 来源标题 [来源类型 / 为什么可信]
  URL
```

对于非中文输出，标签和措辞应与用户语言保持一致。

## 摘要质量

顶部摘要应该有用，而不是只追求短。

相关时应包含：

- 条件和范围。
- 基线或比较对象。
- 时间范围。
- 近似幅度。
- 机制。
- 会改变结论的例外情况。

示例：

```text
“咖啡会升高 LDL”过于笼统。更准确的说法是：未过滤咖啡，尤其是每天数杯时，可能因保留更多 cafestol/kahweol 而升高 LDL；纸滤咖啡会去除大部分这类二萜，对 LDL 的影响通常较小。
```

## 来源处理

来源标签说明来源类型或为什么可信。它们不是引用编号，也不能替代声明级可信度评分。

每个来源必须直接支持或反驳它所在的具体声明。
