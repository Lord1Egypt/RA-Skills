## 金融产品设计生成要求

金融产品设计每种任务类型的"零件清单"——必选/可选组件、组装顺序、领域约束。按清单逐项产出。

---

### 1 需求分析

#### T1-1 客户需求

- **必选组件**: 客户画像、需求调研、需求分析、需求优先级
- **可选组件**: 客户访谈、客户问卷、客户数据分析
- **组装顺序**: 确定客户画像 → 设计调研方案 → 执行调研 → 分析需求 → 排序优先级
- **约束**: 画像必须清晰；调研必须全面；分析必须深入
- **格式**: 客户需求报告（Markdown）

#### T1-2 市场调研

- **必选组件**: 调研范围、调研方法、市场数据、趋势分析
- **可选组件**: 行业报告、专家访谈、市场测试
- **组装顺序**: 确定调研范围 → 选择调研方法 → 收集市场数据 → 分析趋势 → 形成报告
- **约束**: 范围必须明确；数据必须可靠；分析必须客观
- **格式**: 市场调研报告（Markdown）

#### T1-3 竞品分析

- **必选组件**: 竞品清单、分析维度、分析结果、竞争策略
- **可选组件**: 竞品购买、竞品体验、竞品拆解
- **组装顺序**: 确定竞品清单 → 设计分析维度 → 收集竞品信息 → 分析对比 → 制定策略
- **约束**: 竞品必须有代表性；维度必须全面；分析必须客观
- **格式**: 竞品分析报告（Markdown）

#### T1-4 监管要求

- **必选组件**: 监管政策、合规要求、准入门槛、合规检查
- **可选组件**: 监管解读、合规咨询、监管沟通
- **组装顺序**: 收集监管政策 → 分析合规要求 → 确定准入门槛 → 执行合规检查
- **约束**: 政策必须准确；要求必须完整；检查必须严格
- **格式**: 监管要求分析（Markdown）

---

### 2 产品设计

#### T2-1 产品架构

- **必选组件**: 产品定位、产品结构、功能模块、服务流程
- **可选组件**: 产品原型、产品文档、产品评审
- **组装顺序**: 确定产品定位 → 设计产品结构 → 规划功能模块 → 设计服务流程
- **约束**: 定位必须清晰；结构必须合理；流程必须顺畅
- **格式**: 产品架构文档（Markdown）

#### T2-2 定价模型

- **必选组件**: 成本分析、收益测算、价格策略、定价公式
- **可选组件**: 定价测试、价格敏感度分析、竞争定价分析
- **组装顺序**: 分析成本结构 → 测算收益 → 制定价格策略 → 建立定价公式
- **约束**: 成本必须准确；收益必须合理；策略必须可行
- **格式**: 定价模型文档（Markdown）

#### T2-3 风控设计

- **必选组件**: 风险识别、风险评估、风控措施、风控流程
- **可选组件**: 风控模型、风控工具、风控测试
- **组装顺序**: 识别风险 → 评估风险 → 设计风控措施 → 制定风控流程
- **约束**: 识别必须全面；评估必须客观；措施必须有效
- **格式**: 风控设计方案（Markdown）

#### T2-4 合规设计

- **必选组件**: 合规要求、合规检查、合规整改、合规报告
- **可选组件**: 合规咨询、合规培训、合规监控
- **组装顺序**: 梳理合规要求 → 执行合规检查 → 进行合规整改 → 形成合规报告
- **约束**: 要求必须完整；检查必须严格；整改必须到位
- **格式**: 合规设计报告（Markdown）

### 3 产品开发

#### T3-1 系统开发

- **必选组件**: 开发需求、技术方案、开发计划、代码实现
- **可选组件**: 开发工具、开发规范、代码审查
- **组装顺序**: 明确开发需求 → 设计技术方案 → 制定开发计划 → 执行开发 → 代码审查
- **约束**: 需求必须清晰；方案必须可行；代码必须规范
- **格式**: 系统开发文档（Markdown）

#### T3-2 测试验证

- **必选组件**: 测试方案、测试用例、测试环境、测试结果
- **可选组件**: 自动化测试、性能测试、安全测试
- **组装顺序**: 制定测试方案 → 编写测试用例 → 准备测试环境 → 执行测试 → 分析结果
- **约束**: 方案必须全面；用例必须覆盖；环境必须稳定
- **格式**: 测试验证报告（Markdown）

#### T3-3 上线准备

- **必选组件**: 上线方案、部署计划、数据迁移、应急预案
- **可选组件**: 上线检查、上线演练、上线监控
- **组装顺序**: 制定上线方案 → 安排部署计划 → 执行数据迁移 → 制定应急预案
- **约束**: 方案必须可行；计划必须详细；预案必须完备
- **格式**: 上线准备文档（Markdown）

#### T3-4 运营准备

- **必选组件**: 运营团队、运营流程、运营工具、运营培训
- **可选组件**: 运营手册、运营指标、运营监控
- **组装顺序**: 组建运营团队 → 制定运营流程 → 配置运营工具 → 执行运营培训
- **约束**: 团队必须专业；流程必须清晰；工具必须完备
- **格式**: 运营准备文档（Markdown）

---

### 4 产品运营

#### T4-1 产品推广

- **必选组件**: 推广策略、推广渠道、推广预算、推广效果
- **可选组件**: 推广活动、推广素材、推广合作
- **组装顺序**: 制定推广策略 → 选择推广渠道 → 安排推广预算 → 评估推广效果
- **约束**: 策略必须可行；渠道必须有效；预算必须合理
- **格式**: 产品推广方案（Markdown）

#### T4-2 客户服务

- **必选组件**: 服务标准、服务流程、服务渠道、服务质量
- **可选组件**: 服务培训、服务工具、服务监控
- **组装顺序**: 制定服务标准 → 设计服务流程 → 建立服务渠道 → 监控服务质量
- **约束**: 标准必须明确；流程必须顺畅；质量必须保证
- **格式**: 客户服务方案（Markdown）

#### T4-3 产品优化

- **必选组件**: 优化需求、优化方案、优化计划、优化效果
- **可选组件**: 优化测试、优化验证、优化监控
- **组装顺序**: 收集优化需求 → 设计优化方案 → 制定优化计划 → 评估优化效果
- **约束**: 需求必须真实；方案必须可行；效果必须可验证
- **格式**: 产品优化方案（Markdown）

#### T4-4 产品退出

- **必选组件**: 退出策略、退出计划、客户通知、资产处置
- **可选组件**: 退出审批、退出执行、退出监控
- **组装顺序**: 制定退出策略 → 安排退出计划 → 通知客户 → 处置资产
- **约束**: 策略必须合理；计划必须详细；通知必须及时
- **格式**: 产品退出方案（Markdown）
