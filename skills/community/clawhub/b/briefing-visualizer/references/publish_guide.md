## 二、上架流程

### Step 1：整理技能包

确认以下文件齐全：

```
briefing-visualizer/
├── SKILL.md                    ← 核心说明文件
├── scripts/
│   └── process_briefing.py    ← 核心处理脚本
└── references/
    └── html_template.md        ← HTML 模板参考
```

### Step 2：登录 ClawHub

访问 https://www.codebuddy.cn/work/，使用腾讯云账号登录。

### Step 3：提交技能

- 点击「发布技能」或「+ 新建」
- 填写技能信息（见下节）
- 上传技能文件夹（压缩 zip 或直接导入文件夹）
- 提交审核

### Step 4：等待审核

- 官方审核周期：1-3 个工作日
- 审核通过后自动上架
- 可在「我的技能」中查看状态

### 简介（一句话）

```

```

### 详细介绍

```
【简报转手机长图 · Turn Briefing into Visual】

将结构化文字简报（行业简报、市场动态、竞品情报、招商资料等）
转化为精美的手机端长图，一键发送到微信群/朋友圈/小红书。

▎ 核心能力
1. 资讯抓取与整理：用户直接提供内容或让 AI 辅助抓取
2. 自动排版生成 HTML：390px 手机宽度，暖橙色主题，适配微信阅读习惯
3. Chrome headless 截长图：自动识别并裁剪底部空白
4. 底部通栏广告位：自动拼接品牌/产品/联系方式图片

▎ 适用场景
- 行业简报（海参/食品/零售/金融等行业周报）
- 市场简报（竞品动态、价格行情、趋势汇总）
- 招商简报（招商政策、合作优势、联系方式）
- 公司动态（新品发布、活动预告）
- 任何需要快速生成精美长图进行传播的场景

▎ 使用方法
1. 安装本技能
2. 提供：标题 / 资讯内容 / 底部通栏图（可选）
3. AI 自动生成 HTML → 截图 → 拼接底部图 → 输出长图

▎ 核心优势
- 全自动：一条指令完成从文字到长图的全流程
- 适配手机：390px 宽度，微信/小红书直接发送
- 品牌统一：底部通栏广告位，自定义品牌露出
- 零门槛：无需设计技能，AI 自动排版

▎ 关于底部通栏图
支持用户自定义品牌/产品/联系方式图片，宽度 1080px 以上效果最佳。
无底部图需求时可跳过。
```

## 四、收益模式（重要说明）

### 当前情况

根据腾讯官方公开信息：

1. **ClawHub 技能市场**已上线，支持开发者提交技能
2. **多端分发**：上架后可在 QClaw / OpenClaw / WorkBuddy 等多个产品中安装使用
3. **变现机制**：腾讯目前仍在完善技能市场的商业模式

### 建议的收益策略

#### 短期策略（当前可操作）

1. **免费 + 引流**
   - 技能免费发布，积累安装量和好评
   - 在技能说明中引流到你的私域（微信/公众号/小程序）
   - 通过技能触达精准用户，再转化其他产品/服务

2. **模板付费**
   - 提供多套 HTML 配色模板（行业版/招商版/活动版）
   - 在技能说明中提示「进阶模板请联系作者获取」

3. **定制服务**
   - 在技能中留联系方式（底部通栏图 / 技能说明）
   - 用户使用技能生成后，有定制需求可联系你

#### 中期策略（市场成熟后）

1. **技能内购**
   - 主技能免费 + 专业版功能解锁（如多套模板、批量处理）
   - 参考 App Store 的 Free + IAP 模式

2. **订阅制**
   - 「每月行业简报模板订阅」，每月更新配色/版式
   - 适合内容创作者、KOL 团队

3. **B2B 授权**
   - 企业定制版技能（替换品牌色/底部图/模板）
   - 按企业/团队授权收费

#### 长期策略

1. **技能矩阵**
   - 开发系列技能：「简报转长图」「长图发小红书」「小红书数据分析」
   - 形成内容营销自动化工具链，用户在你这里一站式解决

2. **平台分成**
   - 腾讯后续若推出技能内购分成，按安装量/付费量结算
   - 提前积累优质技能和口碑，等待平台开放

## 七、常见问题

**Q：技能市场对个人开发者开放吗？**
A：是的，个人开发者可用腾讯云账号直接提交。

**Q：审核需要多久？**
A：通常 1-3 个工作日。

**Q：技能可以收费吗？**
A：目前以免费为主；付费机制（内购分成）腾讯正在完善，建议先做免费版积累口碑。

**Q：Chrome 路径不对怎么办？**
A：脚本会自动检测 Mac/Windows 路径；Linux 用户需手动配置 `CHROME` 变量。
