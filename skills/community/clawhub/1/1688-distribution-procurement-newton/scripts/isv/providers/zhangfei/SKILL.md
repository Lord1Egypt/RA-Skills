## 五、CLI 规范

- 形态：`python3 cli.py <command> [options]`
- **平台路由**：`cli.py` 与 `scripts/procurement.py` 为**唯一固定入口**；执行侧**必须先**按用户已选平台决定子命令。**淘宝** 使用本节列出的淘宝命令名；**抖音** 使用与淘宝**不同**的独立命令名。**禁止** 跨平台混用子命令。
- **鉴权**：所有子命令仅通过 **`--token`** 传 ISV Token；未传或空 → JSON `error: missing_token`。
- **stdout**：JSON（`success`、`markdown`、`data`；失败可有 `error`）。
- **环境**：`SKILL_API_BASE` 默认 `https://spmiddle.1tsoft.com/skill/skill`。

子命令与参数示例见 **§六**（本节不重复贴 bash）。**区分**：淘宝 `link-source` 与抖音对应独立命令均走 `daifa/relation`（分销关系）；SKU 关联分别使用 **§六 · 能力三** 中当前平台对应的预览 / 保存关联命令。

## 六、能力一 / 二 / 三 — 技术说明与 CLI（按平台分流）

**适用**：已选定平台；**§3.3** 占位平台 **禁止** 执行本节命令。

### 6.1 淘宝

#### 能力一 · 自动下单

##### 店铺列表

```bash
python3 cli.py list-shops --token "<ISV_TOKEN>"
```

多店铺时列表选定后再继续；记下 `shopId`、店名。

##### 采购账号

```bash
python3 cli.py list-distributors --token "<ISV_TOKEN>" --shop-id "<shop_id>"
```

在 **第 2 步 · 采购账号与开分销范围** 中：自动下单账号用于 `enable-auto-order`；`enable-link-type` 的 `--purchasing-accounts` 对应 `{all_accounts}`：用户约定「全部」或子集时展开为 `accounts[].name` 英文逗号串；**用户未说明开分销范围时默认仅 `{purchase_account}` 一个账号**。

##### 商品 ID

逗号分隔，最多 5 个。可选在售：

```bash
python3 cli.py list-products --token "<ISV_TOKEN>" --shop-nick "<店铺nick或ID>"
```

##### 确认后 · 开分销

```bash
python3 cli.py enable-link-type \
  --token "<ISV_TOKEN>" \
  --item-ids "123456,789012" \
  --shop "<店铺名称>" \
  --purchasing-accounts "<账号1,账号2,...>"
```

按 `data.fail_items` 处理失败项。

##### 确认后 · 保存自动下单

```bash
python3 cli.py enable-auto-order \
  --token "<ISV_TOKEN>" \
  --shop-name "<店铺名称>" \
  --purchase-account "<用户选中的采购账号>"
```

失败时对用户输出 **§3.2 淘宝** 第 6 步「失败」句，将响应 `error` 或 `markdown` 中的业务说明填入 `{失败原因}`。

#### 能力二 · 售后处理

##### 店铺列表

```bash
python3 cli.py list-shops --token "<ISV_TOKEN>"
```

能力二对用户 **不收集** 生效店铺范围；调用 `enable-auto-after-sale` 时 **`--shop-names` 固定** `"全部"`。

##### 售后采购账号

```bash
python3 cli.py list-after-sale-accounts --token "<ISV_TOKEN>" --shop-id "<shop_id>"
```

`--shop-id` 仅用于中台解析 sid，**不**作为上述 URL 的 query。

##### 保存

```bash
python3 cli.py enable-auto-after-sale \
  --token "<ISV_TOKEN>" \
  --shop-id "<shop_id>" \
  --purchase-account "<账号1,账号2,...>" \
  --shop-names "全部"
```

用户确认后再执行。失败时对用户输出 **§3.2 淘宝** 能力二 · 第 4 步「失败」句，将响应 `error` 或 `markdown` 中的业务说明填入 `{失败原因}`。

#### 能力三 · 关联货源

##### 店铺列表

```bash
python3 cli.py list-shops --token "<ISV_TOKEN>"
```

##### 本店商品 ID

可选 `list-products`（同能力一）。

##### 1688 货源链接

`https://detail.1688.com/offer/....html`；执行侧解析 **offerId**，与 `--source-offer-url` 一致；**不对用户**单独核对货源编号。

##### 采购账号

```bash
python3 cli.py list-distributors --token "<ISV_TOKEN>" --shop-id "<shop_id>"
```

**禁止** `list-after-sale-accounts`。

##### SKU 预览

```bash
python3 cli.py link-preview \
  --token "<ISV_TOKEN>" \
  --shop-id "<shopId>" \
  --shop-nick "<店铺名称>" \
  --local-item-id "<本店商品ID>" \
  --source-offer-url "https://detail.1688.com/offer/xxxx.html" \
  --purchase-account "<采购账号>"
```

匹配说明：`本店SKU---上家SKU --匹配度：xx%`（整数）；用户认可后再执行保存。

##### 保存关联

```bash
python3 cli.py confirm-link \
  --token "<ISV_TOKEN>" \
  --shop-id "<shopId>" \
  --shop-nick "<店铺名称>" \
  --local-item-id "<本店商品ID>" \
  --source-offer-url "https://detail.1688.com/offer/xxxx.html" \
  --purchase-account "<采购账号>" \
  [--limit-rows "<仅提交前 N 条 SKU 映射，可选>"]
```

用户确认后再执行。失败时对用户输出 **§3.2 淘宝** 能力三 · 第 6 步「失败」句，将响应 `error` 或 `markdown` 中的业务说明填入 `{失败原因}`。

##### 可选：分销关系（非 SKU）

```bash
python3 cli.py link-source \
  --token "<ISV_TOKEN>" \
  --downstream-item-id "<本店商品ID>" \
  --shop "<店铺名称>" \
  --purchasing-accounts "<采购账号>" \
  [--source-offer-id "<offerId>"]
```

  --source-offer-url "https://detail.1688.com/offer/xxxx.html" \
  --purchase-account "<采购账号>" \

## 七、助手执行协议与流程锁死

以下约束 **优先于** 模型默认习惯；违反即视为未按本 Skill 执行。

### 流程锁死（话术原文级）

- **锁0 · 全文严格合规**：对用户的每条消息，可见正文 **必须** 同时满足上款 **「每次回答（总则 · 全文合规）」** 与 **锁1～锁8**、下述「编号条款」；**禁止**出现任何未获本文件某表「对用户说的话」或 **锁4** 列表规则 **明示授权** 的字句（含自撰提醒、操作建议、安全提示、接口长文复述等）。
- **锁1 · 未说明销售平台**：用户可见内容 **仅允许** §3.1「未说明销售平台」引号内 **整句**，**句前句后不得**加任何字。
- **锁2 · 全局固定句**：§3.1 中「用户选定淘宝」「用户选定抖音」「非支持平台」「业务失败」「鉴权失败」「已选平台但说不清办哪项」各 **对用户说的话** 与引号内 **逐字一致**；仅 `{平台名}`、`{店铺名称}` 等占位替换。
- **锁3 · 分卷业务隔离**：淘宝分卷（§3.2） 各表「对用户说的话」**必须原文**；**禁止** 用 §3.2以外的占位句办理淘宝业务；**禁止** 在 **未选淘宝** 时使用 §3.2 主句。抖音分卷（§3.3） 各表「对用户说的话」**必须原文**；**禁止** 用 §3.3以外的占位句办理抖音业务；**禁止** 在 **未选抖音** 时使用 §3.3 主句。
- **锁4 · 列表辅助**：需展示店铺 / 采购账号 / 在售列表时，**仅允许**在 **主句原文之前** 增加 **列表块**。**店铺** 编号列表：该列表 **紧上一行** 必须为 **店铺列表：**（四字加中文冒号，独占一行，字面逐字），其下为编号分行（序号、店名等）。**采购账号** 编号列表（含自动下单 `list-distributors`、售后 `list-after-sale-accounts`、关联货源第 4 步之分销账号）：该列表 **紧上一行** 必须为 **采购账号列表：**（六字加中文冒号，独占一行，字面逐字），其下为编号分行。**在售** `list-products` 若编号展示：**不**使用「店铺列表：」「采购账号列表：」。无店铺 / 无采购账号因而 **不展示** 编号列表时：**禁止** 单独输出上述标题行。列表块后 **必须紧跟** 该步主句（或用户已给出唯一可核对选项时直接进入下一步主句）。**禁止**用自撰问法替代主句。
- **锁5 · 禁止抢答解释**：**禁止**向用户解释 Skill 未提供的背景说明；**禁止**把内部路由、附录代号、执行提示全文复述给用户。
- **锁6 · 店铺步骤**：进入 **能力一 / 二 / 三 的第 1 步 · 店铺** 时，执行侧 **必须先** 拉取店铺列表，再按锁4 输出（含 **有店 / 无店** 分支锁死句）；**禁止**未拉表仅让用户凭空报店名（用户已给出唯一可核对店名除外）。
- **锁8 · 鉴权**：线上由平台自动注入 Token；**禁止**在任何步骤向用户索要、展示或复述 Token。
- **锁7 · 平台隔离**：非淘宝（§3.2）**只** 输出对应 **占位 · 唯一收口句**；非抖音（§3.3）**只** 输出对应 **占位 · 唯一收口句****禁止**调用 **§六** CLI；**禁止**将淘宝/抖音锁死句改写到占位节「代替」落地。

### 编号条款

1. **单步输出**：每次回复只推进 **当前一步**；**禁止**同条消息列全流程清单。  
2. **对用户不展示实现细节**：**禁止**命令行、代码块、JSON、路径、接口字段名、环境变量名（助手后台执行即可）。**禁止**在任何轮次索要或复述 Token 内容。  
3. **列表类信息**：须用户从店铺 / 账号 / 商品中选择时，**先**拉列表再编号展示；**禁止**展示原始接口数据。  
4. **禁止伪造列表**：未取得对应查询命令的真实结果前，**禁止**输出「店铺列表：」「采购账号列表：」及任何编号行；**禁止**输出“店铺A/店铺B/店铺C”等示例占位内容充当真实列表。  
5. **话术禁区**：**禁止**承诺未实现能力；失败时对用户 **仅** 输出当前平台分卷已收录的「成功/失败」分支（含能力一 · 第 6 步、能力二 · 第 4 步、能力三 · 第 6 步之 `{失败原因}`）或 §3.1「业务失败」句，**禁止**自撰。  
6. **鉴权失败**：只对用户说 §3.1「鉴权失败」一句；**禁止**展示 Token 内容、禁止索要阿里用户 ID。  
7. **店铺列表**：勿向用户索要阿里用户 ID。
