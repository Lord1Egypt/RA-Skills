# 一致性检测方法

## 1. 方法论概述

一致性检测是保障Skill/Agent输出稳定性的核心机制。通过对同一输入执行多次完整流程，对比输出在结论、结构、数据、建议四个维度的偏离程度，量化评估系统的可靠性。

**核心假设**：如果一个Skill在不同时间对相同输入产生差异巨大的输出，说明其逻辑存在随机性或不稳定因素，无法为用户提供可信赖的服务。

---

## 2. 偏离度计算框架

### 2.1 四维度权重分配

| 维度 | 权重 | 理由 |
|------|------|------|
| 结论一致性 | 40% | 核心结论是最重要的输出，偏差影响最大 |
| 结构一致性 | 25% | 框架稳定性影响可读性和可预期性 |
| 数据一致性 | 20% | 数据偏差影响决策准确性 |
| 建议一致性 | 15% | 行动建议的方向性一致即可 |

### 2.2 各维度计算方法

#### 维度1：结论一致性（权重40%）

**计算规则**：
- 核心结论完全相同：0分
- 核心结论部分相同（>50%重合）：25分
- 核心结论大部分不同（30-50%重合）：50分
- 核心结论完全不同（<30%重合）：100分

**判定方法**：
1. 提取每次输出的核心结论（通常在摘要或结论段）
2. 使用关键词重叠法或语义相似度判断
3. 人工复核AI判断结果

#### 维度2：结构一致性（权重25%）

**计算公式**：
```
结构偏离度 = (差异段落数 / 总段落数) × 100%
```

**判定方法**：
1. 统计每次输出的总段落数
2. 对比段落结构（标题层级、内容分布）
3. 标记存在差异的段落

**示例**：
- 第一次输出：5段（背景→方法→结果→讨论→结论）
- 第二次输出：6段（背景→方法→结果→讨论→建议→结论）
- 第三次输出：5段（背景→方法→结果→讨论→结论）
- 差异段落：第2次多了"建议"段落
- 结构偏离度 = 1/6 × 100% ≈ 17%

#### 维度3：数据一致性（权重20%）

**计算公式**：
```
数据偏离度 = Σ(|实测值 - 平均值| / 平均值) / n × 100%
```

**判定方法**：
1. 提取关键数值数据（百分比、排名、数量等）
2. 计算3次输出的平均值
3. 计算每个实测值与平均值的偏差百分比
4. 取平均值作为数据偏离度

**示例**：
- 输出1数据：提升30%
- 输出2数据：提升28%
- 输出3数据：提升35%
- 平均值：(30+28+35)/3 = 31
- 数据偏离度 = (|30-31|+|28-31|+|35-31|)/3/31 × 100% = 7.5%

#### 维度4：建议一致性（权重15%）

**计算规则**：
- 行动建议方向完全相同：0分
- 行动建议方向部分相同：25分
- 行动建议方向相反：50分
- 无建议或建议混乱：100分

**判定方法**：
1. 提取每次输出的行动建议
2. 判断建议方向（推荐/不推荐/中性/需进一步分析）
3. 对比方向是否一致

---

## 3. 综合偏离度计算

### 3.1 计算公式

```
综合偏离度 = 结论偏离度×40% + 结构偏离度×25% + 数据偏离度×20% + 建议偏离度×15%
```

### 3.2 阈值判定

| 综合偏离度 | 判定 | 处置建议 |
|------------|------|----------|
| 0-15% | ✅优秀 | 可发布 |
| 15-25% | ⚠️良好 | 建议优化后发布 |
| 25-35% | 🔴一般 | 必须优化 |
| >35% | 🛘不合格 | 重新调试 |

---

## 4. 操作流程

### Step 1: 选择测试输入

**选择原则**：
- 覆盖主要功能场景
- 包含典型边界条件
- 选取3个不同类型的输入

**示例**：
- 输入A：专业领域术语查询
- 输入B：模糊概念查询
- 输入C：多维度组合查询

### Step 2: 执行多次测试

```
for each 测试输入:
    for i in [1, 2, 3]:
        执行完整流程
        记录输出结果
        截图保存
```

### Step 3: 逐维度对比

| 维度 | 输入A对比 | 输入B对比 | 输入C对比 |
|------|-----------|-----------|-----------|
| 结论一致性 | ... | ... | ... |
| 结构一致性 | ... | ... | ... |
| 数据一致性 | ... | ... | ... |
| 建议一致性 | ... | ... | ... |

### Step 4: 计算综合偏离度

取3个输入的综合偏离度平均值作为最终得分。

## 5. 示例：查新检索Skill一致性测试

### 测试输入
> "TRIZ技术创新方法"

### 第一次输出
```
【结论】TRIZ是一种有效的技术创新方法
【结构】5段：背景→方法介绍→应用场景→案例分析→结论
【数据】提升效率30%，降低成本25%
【建议】推荐企业在研发中应用TRIZ方法
```

### 第二次输出
```
【结论】TRIZ能够有效支持技术创新
【结构】5段：背景→方法介绍→核心原理→应用建议→结论
【数据】提升效率28%，降低成本23%
【建议】可考虑在产品开发中试点应用
```

### 第三次输出
```
【结论】TRIZ方法对技术创新有显著促进作用
【结构】6段：背景→方法介绍→核心原理→实施步骤→案例分析→结论
【数据】提升效率35%，降低成本28%
【建议】建议在研发团队中推广TRIZ培训
```

### 偏离度计算

| 维度 | 偏离度 | 权重 | 加权得分 |
|------|--------|------|----------|
| 结论一致性 | 0%（核心结论一致） | 40% | 0% |
| 结构一致性 | 20%（1/5段落差异） | 25% | 5% |
| 数据一致性 | 8%（数据偏差平均） | 20% | 1.6% |
| 建议一致性 | 0%（方向一致） | 15% | 0% |
| **综合偏离度** | - | 100% | **6.6%** |

### 判定结果
✅ **通过**（综合偏离度6.6% < 15%阈值）

## 6. 异常情况处理

### 6.1 某次输出完全失效
- 如果某次输出出现明显错误（如崩溃、无输出），该次偏离度计100%
- 这种情况必须触发🛘阻断

### 6.2 多次输出质量差异大
- 如果3次输出中某次质量明显优于其他（如结构清晰度、数据准确性），说明存在不稳定性
- 触发⚠️警告，需优化

### 6.3 结论相反的情况
- 如果出现结论完全相反的输出（如一次推荐、一次不推荐），结论偏离度计100%
- 这种情况下综合偏离度通常>35%，触发🛘不合格

---

## 7. 常见不稳定因素及解决方案

| 不稳定因素 | 表现特征 | 解决方案 |
|------------|----------|----------|
| 随机性Prompt | 每次输出框架不同 | 固定输出结构模板 |
| 缺少示例 | 边界情况处理不一致 | 增加Few-shot示例 |
| 上下文累积 | 历史对话影响后续输出 | 定期清理上下文 |
| 温度参数过高 | 词汇选择变化大 | 降低temperature至0.3-0.5 |
| 逻辑分支多 | 不同分支结果差异大 | 简化逻辑或增加条件判断 |
