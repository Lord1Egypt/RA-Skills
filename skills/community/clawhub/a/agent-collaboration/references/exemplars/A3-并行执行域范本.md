## A3 并行执行域 范本

### 范本概述
- **域ID**: A3
- **域名称**: 并行执行域
- **任务数量**: 5
- **范本状态**: 可用

### 任务执行范本

#### A3-01 并行机会识别
**执行步骤**:
1. 分析任务依赖关系
2. 识别无依赖关系的任务
3. 确认并行条件
4. 规划并行执行
5. 确认并行识别结果

**输入示例**:
- 任务列表：UI设计、前端开发、后端开发、测试
- 依赖关系：前端开发依赖UI设计，后端开发依赖UI设计，测试依赖开发完成

**输出示例**:
- 可并行任务：前端开发、后端开发
- 并行条件：UI设计完成后，前端和后端可同时开发
- 并行限制：需要足够的开发资源
- 并行执行计划：UI设计完成后，同时启动前端和后端开发

**注意事项**:
- 并行任务要无依赖关系
- 并行条件要满足
- 并行限制要考虑

#### A3-02 并行任务分配
**执行步骤**:
1. 制定分配标准
2. 选择分配方法
3. 执行任务分配
4. 确认分配结果
5. 输出分配报告

**输入示例**:
- 可并行任务：前端开发、后端开发
- 可用Agent：General-purpose Agent
- 分配标准：任务特征与Agent能力匹配

**输出示例**:
- 分配标准：任务特征与Agent能力匹配
- 分配方法：能力匹配、资源匹配、负载均衡
- 分配结果：
  - 前端开发：General-purpose Agent 1
  - 后端开发：General-purpose Agent 2
- 分配确认：任务分配成功

**注意事项**:
- 分配标准要客观，避免主观臆断
- 分配方法要有效，避免无效分配
- 分配结果要准确，避免错误

#### A3-03 并行执行监控
**执行步骤**:
1. 设定监控指标
2. 选择监控方法
3. 执行并行监控
4. 分析监控结果
5. 输出监控报告

**输入示例**:
- 并行任务：前端开发、后端开发
- 监控需求：监控并行任务执行进度

**输出示例**:
- 监控指标：任务状态、进度、问题、资源使用
- 监控方法：实时监控、定期检查、问题反馈
- 监控结果：
  - 前端开发：进行中，进度60%，CPU 40%，内存 50%
  - 后端开发：进行中，进度55%，CPU 35%，内存 45%

**注意事项**:
- 监控指标要全面，避免遗漏
- 监控方法要有效，避免无效监控
- 监控结果要准确，避免错误

#### A3-04 并行结果收集
**执行步骤**:
1. 界定收集范围
2. 选择收集方法
3. 执行结果收集
4. 确认收集结果
5. 输出收集报告

**输入示例**:
- 并行任务：前端开发、后端开发
- 收集需求：收集并行任务执行结果

**输出示例**:
- 收集范围：前端开发、后端开发
- 收集方法：结果提交、代码审查、测试验证
- 收集结果：
  - 前端开发：已完成，代码已提交
  - 后端开发：已完成，代码已提交
- 收集确认：所有并行任务结果已收集

**注意事项**:
- 收集范围要明确，避免遗漏
- 收集方法要有效，避免无效收集
- 收集结果要完整，避免缺失

#### A3-05 并行结果合并
**执行步骤**:
1. 制定合并标准
2. 选择合并方法
3. 执行结果合并
4. 确认合并结果
5. 输出合并报告

**输入示例**:
- 并行结果：前端开发结果、后端开发结果
- 合并需求：合并前端和后端开发结果

**输出示例**:
- 合并标准：代码兼容、功能完整、性能达标
- 合并方法：代码合并、接口联调、集成测试
- 合并结果：
  - 代码合并：成功
  - 接口联调：成功
  - 集成测试：通过
- 合并确认：并行结果合并成功

**注意事项**:
- 合并标准要合理，避免过度严格或宽松
- 合并方法要有效，避免无效合并
- 合并结果要准确，避免错误
