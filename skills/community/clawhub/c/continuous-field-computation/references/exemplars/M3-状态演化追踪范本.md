# M3 状态演化追踪域范本

## M3-01 状态快照记录

### 输入示例
任务执行过程：分析用户行为数据

### 处理过程
1. 状态定义：定义关键状态节点
2. 快照时机确定：每个关键步骤完成后
3. 记录格式设计：时间序列数据
4. 存储方案制定：内存缓存

### 输出示例
```markdown
## 状态快照记录规范

### 状态定义
| 状态ID | 状态名称 | 触发条件 | 记录内容 |
|--------|----------|----------|----------|
| S0 | 初始状态 | 任务开始 | 任务输入 |
| S1 | 数据清洗完成 | 清洗步骤完成 | 清洗后数据 |
| S2 | 特征提取完成 | 提取步骤完成 | 特征数据 |
| S3 | 分析完成 | 分析步骤完成 | 分析结果 |
| S4 | 最终状态 | 任务结束 | 最终输出 |

### 快照时机
- **时机1**：数据清洗完成后
- **时机2**：特征提取完成后
- **时机3**：分析完成后
- **时机4**：任务结束时

### 记录格式
```json
{
  "state_id": "S1",
  "timestamp": "YYYY-MM-DDTHH:MM:SSZ",
  "content": {
    "data": "清洗后数据",
    "metrics": {"rows": 1000, "columns": 10}
  }
}
```
```

## M3-02 演变路径追踪

### 输入示例
状态序列：S0 → S1 → S2 → S3 → S4

### 处理过程
1. 路径定义：从初始状态到最终状态的完整路径
2. 追踪方法：时间序列记录
3. 路径记录：记录每个状态转换
4. 路径分析：分析路径特征

### 输出示例
```markdown
## 演变路径追踪报告

### 路径定义
**路径**：S0 → S1 → S2 → S3 → S4

### 追踪方法
- **方法**：时间序列记录
- **记录频率**：每个状态转换
- **记录内容**：状态内容、转换时间、转换原因

### 路径记录
| 转换 | 时间 | 源状态 | 目标状态 | 转换原因 |
|------|------|--------|----------|----------|
| T1 | 00:01 | S0 | S1 | 数据清洗完成 |
| T2 | 00:02 | S1 | S2 | 特征提取完成 |
| T3 | 00:03 | S2 | S3 | 分析完成 |
| T4 | 00:04 | S3 | S4 | 任务结束 |

### 路径分析
- **路径长度**：4个状态转换
- **路径特征**：线性演化，无循环
- **路径效率**：平均每个转换1秒
```

## M3-05 状态分析

### 输入示例
状态序列：S0 → S1 → S2 → S3 → S4

### 处理过程
1. 分析目标：分析状态演化的模式和规律
2. 分析方法：模式识别
3. 分析执行：识别线性演化模式
4. 分析报告：生成分析报告

### 输出示例
```markdown
## 状态分析报告

### 分析目标
- **目标**：分析状态演化的模式和规律
- **范围**：S0到S4的状态序列
- **维度**：模式、规律、异常

### 分析方法
- **方法**：模式识别
- **技术**：时间序列分析
- **工具**：状态机分析

### 分析结果
| 分析维度 | 结果 | 说明 |
|----------|------|------|
| 演化模式 | 线性 | 状态按顺序演化 |
| 演化规律 | 依赖驱动 | 每个状态依赖前一个状态 |
| 异常情况 | 无 | 无异常状态 |

### 分析结论
- **模式**：线性演化，无循环
- **规律**：每个状态依赖前一个状态
- **异常**：无异常状态
- **建议**：保持当前演化模式
```
