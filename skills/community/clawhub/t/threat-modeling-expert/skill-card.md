## Description: <br>
Threat modeling with STRIDE, PASTA, and attack trees. Analyze architectures for security gaps, extract security requirements, build data flow diagrams, and prioritize risks. For secure-by-design planning and security architecture reviews. <br>

This skill is ready for commercial/non-commercial use. <br>

## Publisher: <br>
[solomonneas](https://clawhub.ai/user/solomonneas) <br>

### License/Terms of Use: <br>
MIT-0 <br>


## Use Case: <br>
Developers, security engineers, and architecture reviewers use this skill to structure threat modeling sessions, identify attack paths, extract security requirements, and prioritize mitigations for systems they are authorized to review. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Threat modeling can expose sensitive architecture and security details. <br>
Mitigation: Use an agent environment and storage controls appropriate for sensitive design information before including detailed system data. <br>
Risk: Security review outside an agreed scope can produce inappropriate or misleading guidance. <br>
Mitigation: Use the skill only for systems the user is authorized to review, and define scope and trust boundaries up front. <br>
