# config-matrix.md — 配置方式主表（唯一真相源）

> ⚠️ **本文件是 5 种配置方式全字段的唯一权威定义。** 阶段一、阶段二、hard-rules 全部引用此处，不在别处重复定义。
>
> **分工铁律：**
> - 数字员工（脑）：识别配置方式 → 照本表算好"业务正确值" → 喂给技术那套
> - 技术那套（手）：拿数字员工喂的规范值，构造 promotion-add / promotion-update-creative，**人群 JSON（strategyInfo/crowdList/plan-check）由技术那套自己组**，数字员工不碰
>
> **禁猜必问：** 本表标「PM提供」或本表没写的，一律问 PM，**禁止脑补**（见 hard-rules 原理五）。

---

## 1. 全字段总表（创建技术字段 + 推广名前缀 + 项目类型）

> ⚠️ **v3 策略（彻底修复"项目类型/前缀反复出错"）：** 推广名 4 前缀（内容大类/版权类型/战略类型/投放资源）+ 项目类型，由**数字员工**在阶段一统一处理 — 能死值就查表死值（§1.2），需 PM 选的就由数字员工**展示选项让 PM 选**，**整包传给技术那套，技术那套不再补问 PM**。
>
> v2 旧策略错在让数字员工不传这 6 字段——技术那套收不到 PM 原话"空降"时，它的死映射触发不了，会填默认"其他类型"。v3 由数字员工兜底所有 PM 沟通，技术那套退化为纯写后台。

## 2. 阶段收料边界（错4/错5 的根治）

| 配置方式 | 阶段一必收（创建必填，不收就建错） | 阶段二必收（素材） |
|---|---|---|
| 标准 | 等级 / 时段 / **艺人ID** / 资源子类型(歌曲/歌单/专辑/MV) | **歌曲/歌单/专辑/MV ID**（update-creative `resourceInfo`） / 图片 / 角标 |
| 数专 | 等级 / 时段 / **艺人ID** / 是数字专辑还是数字单曲(定角标) | **数专ID**(拼链接) / 图片 |
| H5 | 等级 / 时段 | H5链接 / 图片 / 角标(三选一) |
| 空降 | 等级 / 时段 / **艺人ID** （人群66594/不包含固定，不用问） | **歌曲/专辑ID或链接**(拼orpheus) / 大卡图 |
| 有票 | 等级 / 时段 / **艺人ID** / 人群包ID + 包含 | 购票链接 / 大卡图 |

> 💡 **所有资源ID/链接统一归阶段二**（promotion-update-creative）：
> - **标准方式**：歌曲/歌单/专辑/MV ID → 阶段二直接作为 `resourceInfo` 传入 update-creative
> - **空降/数专**：歌曲/数专ID → 阶段二拼跳转链接（orpheus/music.163.com），`resourceInfo` 传空串
> - 阶段一只需知道资源子类型（歌曲/activity等），不收任何资源ID
>
> 错5 场景变体：Agent 在阶段一问了歌曲ID → 收下时注明"阶段二用"，**绝不填进 promotion-add 创建参数**。空降阶段一只收：等级/时段/艺人ID。

## 3. 物料 + 业务正确值（阶段二用）

| 配置方式 | 必传图片 | 角标 | 跳转链接（业务正确值算法） |
|---|---|---|---|
| 标准 | 大卡607 + 小卡400 + 蓝框420(可选) | PM提供；用户没给时可由技术那套取资源默认角标 | 技术那套必须按资源 ID 调默认值接口重新获取；Web 必须 http/https，非 Web 用资源/模板返回的客户端默认链接 |
| 数专 | 大卡607 + 小卡400 + 蓝框420(可选) | 数字专辑 / 数字单曲(二选一) | **按端拼3组**(见下) |
| H5 | 大卡607 + 小卡400 + 蓝框420(可选) | 独家策划 / 活动 / 演出(PM三选一) | PM提供 H5链接 |
| 空降 | **仅大卡607**(必填；420可不要) | 活动(固定) | **拼 orpheus**(见下) |
| 有票 | **仅大卡607** | 演出(固定) | PM提供购票链接 |

**图片尺寸：** 大卡=1080×607 / 小卡=1080×400 / 蓝框小卡=1080×420

### 3.1 数专跳转链接（按端拼，3 组）
- 移动端/荣耀白牌/iPad新版/AndroidPad：`https://music.163.com/st/album/m/album/{数专ID}?nm_style=sbt`
- PC/Web：`https://music.163.com/store/newalbum/detail?id={数专ID}`
- iPadHD旧版/Mac：`https://music.163.com/#/album?id={数专ID}`

### 3.1.1 标准资源默认值

标准歌曲/歌单/专辑/MV 的跳转链接和默认角标不由数字员工猜，也不能从旧创意或上游入参直接沿用，更不能用资源 ID 手拼 `https://music.163.com/...`。交接② 时把资源类型和资源 ID 传给技术那套，并明确要求技术那套调用资源默认值获取逻辑重新计算：

- Web：必须使用 `pcUrl` 或其它 http/https 链接；如果资源只返回 `orpheus` 或没有 http/https，Web 链接视为缺失，必须问 PM 手工提供。
- 移动端/荣耀白牌/PC/Mac/iPad/AndroidPad/iPadHD：优先使用资源或模板返回的客户端默认链接（如 `composedData.commonComposedData.url`、`orpheus` 或 `data-template-extract-private` 的 `preData`）；没有时不能用 Web 链接兜底，必须让 PM 提供端维度链接。
- 特例：技术那套已经调用 `plan-pack-resource-list` 但返回空时，只有标准歌曲可以兜底：非 Web 端 `orpheus://song/{歌曲ID}`，Web 端 `https://music.163.com/#/song?id={歌曲ID}`。其它标准类型（专辑/MV/歌单等）先不使用默认兜底，仍问 PM 端维度链接。
- 用户本轮显式提供的端维度链接优先，但 Web 仍必须是 http/https。
- 保存前技术那套必须把最终端维度链接发给 PM 确认；PM 确认后才能真正提交 `promotion-update-creative`。

### 3.2 空降跳转链接（拼 orpheus）
PM 给歌曲/专辑的 https 链接或资源ID，**只提取 ID**，套进模板，其余字符原样不动：
- 空降歌曲：`orpheus://comment?threadId=R_SO_4_{歌曲ID}&autoPlay=1`
- 空降专辑：`orpheus://comment?threadId=R_AL_3_{专辑ID}&autoPlay=1`

> ⚠️ 这是防技术那套默认链接出错的核心：空降/数专若不喂拼好的链接，技术那套会用错误默认值。数字员工必须先算好正确链接写入追踪表 J 列，交接时一并喂。

## 5. 人群：数字员工喂业务值，技术那套构造（不碰 JSON）

数字员工只需把**两个业务值**喂技术那套：
- 人群包 ID（空降固定 66594；有票 PM 提供）
- 方向：包含 / 不包含（空降固定"不包含"；有票一般"包含"）

技术那套自己负责：`plan-check` 校验人群包 → 构造 `strategyInfo` / `crowdList`（包含=contain/include，不包含=notcontain/exclude）→ 随 promotion-add 提交。**数字员工不写、不传 strategyInfo JSON。**
