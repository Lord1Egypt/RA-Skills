# Size Guide Builder — Quality Checklist

Verify every item before delivering a size guide. Each item should be checked off.

## 1. Completeness (8 items)

- [ ] Every size offered in the product catalog has a row in the table
- [ ] All critical measurement points for the product category are included
- [ ] Dual measurement units (inches AND centimeters) are provided
- [ ] International size conversions cover all target markets
- [ ] Fit type is explicitly stated (slim, regular, relaxed, oversized)
- [ ] Fabric composition and stretch information is included
- [ ] Self-measurement instructions are provided for each measurement point
- [ ] Between-sizes guidance is included

## 2. Accuracy (7 items)

- [ ] Unit conversions are mathematically correct (1 in = 2.54 cm)
- [ ] Size progression is consistent (each size is larger than the previous in all dimensions)
- [ ] Measurements match the source data (manufacturer specs, fit samples, or user-provided data)
- [ ] Garment vs. body measurement distinction is clearly labeled
- [ ] International conversions are verified against at least 2 reference sources
- [ ] Fit notes match known customer feedback
- [ ] No copy-paste errors between sizes or measurement columns

## 3. Clarity (6 items)

- [ ] Table headers are unambiguous (no abbreviations without definition)
- [ ] Measurement points are described with anatomical landmarks, not vague terms
- [ ] Measurement instructions use action verbs and specific locations ("wrap tape around the fullest part of your bust")
- [ ] Fit descriptions are quantified where possible ("runs 0.5 size small" not just "runs small")
- [ ] Ease is explained if garment measurements are used
- [ ] Disclaimer about measurement tolerances is included

## 4. Inclusivity (5 items)

- [ ] Body-type recommendations cover at least 3 scenarios (athletic, petite, plus)
- [ ] Gender-neutral language is used where appropriate
- [ ] Extended sizes (plus, petite, tall) are included if the product offers them
- [ ] Width options are documented (for footwear: narrow, standard, wide)
- [ ] Measurement instructions work for people measuring themselves alone

## 5. Formatting (6 items)

- [ ] Tables render correctly in the target platform format (Shopify, WooCommerce, etc.)
- [ ] Tables are readable on mobile devices (scrollable or responsive)
- [ ] Size label column is first and visually distinct
- [ ] Alternating row shading or borders for readability
- [ ] Consistent number formatting throughout (decimal places, units)
- [ ] No broken characters, encoding issues, or HTML rendering problems

## 6. Actionability (5 items)

- [ ] A customer with only a measuring tape can determine their correct size
- [ ] The guide addresses the most common sizing complaints for this product category
- [ ] Contact/support path is mentioned for edge cases
- [ ] Return/exchange policy reference is included or suggested
- [ ] Guide includes a "when in doubt" recommendation (size up or size down)

## 7. Brand Consistency (4 items)

- [ ] Multi-brand stores have separate tables per brand or deviation notes
- [ ] Brand-specific fit notes are attributed to the correct brand
- [ ] Sizing data source is traceable (manufacturer spec sheet, fit sample, etc.)
- [ ] Guide matches the current season/production run, not an outdated version

## 8. SEO and Discoverability (4 items)

- [ ] Product category name appears in the guide title
- [ ] Common search terms are included naturally (e.g., "how to measure for [product]")
- [ ] Guide can be linked directly from product pages
- [ ] Alt text descriptions are provided for any visual measurement diagrams

## 9. Maintenance (5 items)

- [ ] Date of creation or last update is noted
- [ ] Process for updating the guide when new products/brands are added is defined
- [ ] Customer feedback loop is in place (sizing complaints → guide updates)
- [ ] Seasonal or limited-edition variations are flagged for separate handling
- [ ] Return data analysis cadence is recommended (quarterly minimum)
