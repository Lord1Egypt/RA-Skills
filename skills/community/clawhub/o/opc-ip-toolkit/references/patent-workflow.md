# 专利申请详细流程

## 一、专利类型选择

### 三种专利对比

| 对比项 | 发明专利 | 实用新型专利 | 外观设计专利 |
|-------|---------|-------------|-------------|
| 保护对象 | 产品/方法 | 产品形状/结构 | 产品外观 |
| 保护期限 | 20年 | 10年 | 15年 |
| 审查周期 | 2-3年 | 6-8个月 | 4-6个月 |
| 授权率 | ~70% | ~90% | ~95% |
| 官费(个人) | ~1000元 | ~500元 | ~500元 |
| 技术门槛 | 高 | 中 | 低 |
| 保护力度 | 强 | 中 | 弱 |

### 选择建议

**优先考虑发明专利的情况**：
- 核心技术创新，技术壁垒高
- 竞争对手技术实力强，需要强保护
- 计划进行专利许可或诉讼
- 技术生命周期较长（5年以上）

**优先考虑实用新型的情况**：
- 结构改进型创新
- 快速需要专利证书（用于高企认定、招投标等）
- 技术生命周期短（2-3年）
- 预算有限但需要保护

**考虑外观设计的情况**：
- 产品的独特外观是竞争优势
- 需要保护产品的造型、图案、色彩
- 用于电商平台防伪或品牌建设

**建议组合**：
- 核心发明 + 配套实用新型（同时申请）
- 产品发明 + 外观设计（全方位保护）

## 二、查新检索

### 目的
确认技术方案是否具备**新颖性**，避免重复申请浪费时间和费用。

### 新颖性判断标准
- 在申请日前没有同样的发明或实用新型在国内外出版物上公开发表过
- 没有在国内公开使用过
- 没有其他单位或者个人就同样的发明或实用新型在申请日前提出过申请

### 免费检索工具

#### 1. 国家知识产权局官网
**地址**：https://pss-system.cponline.cnipa.gov.cn/

**使用方法**：
1. 进入专利检索界面
2. 选择"发明专利/实用新型"检索
3. 输入关键词（技术领域、功能描述）
4. 组合检索式：关键词 AND 技术特征

#### 2. SooPAT专利搜索
**地址**：http://www.soopat.com/

**特点**：
- 界面友好
- 支持中英文检索
- 可查看专利全文

#### 3. CNIPA.AI
**地址**：https://cnipa.ai/

**特点**：
- AI辅助理解专利内容
- 免费使用
- 支持多语言

### 检索策略

#### 关键词选取
```
1. 技术领域词：新能源汽车、无人机、智能家居
2. 功能词：控制、优化、检测、识别
3. 结构词：传感器、电机、芯片、算法
4. 近义词扩展：电池→蓄电池→储能装置
```

#### 检索式构建
```python
# 示例检索式
(无人机 OR 无人飞行器 OR UAV) AND (避障 OR 障碍物检测 OR 路径规划) AND NOT (植保 OR 农业 OR 喷洒)
```

### 检索结果分析

**判断是否为现有技术**：
1. 技术方案完全相同 → 缺乏新颖性
2. 技术方案实质相同 → 缺乏新颖性
3. 技术方案不同但解决同样问题 → 可能具备创造性（需进一步分析）

**风险等级评估**：
| 风险等级 | 描述 | 建议 |
|---------|------|------|
| 🟢 低风险 | 未发现相同或相近技术 | 可继续申请 |
| 🟡 中风险 | 发现部分相近技术 | 聚焦差异点，强调创新 |
| 🔴 高风险 | 发现相同技术方案 | 考虑改进或放弃 |

## 三、创意点挖掘（TRIZ应用）

### 什么是TRIZ
TRIZ（俄语"发明问题解决理论"的缩写）是前苏联发明家阿奇舒勒创立的创新方法论，包含40个发明原理和矛盾矩阵。

### 快速应用TRIZ的步骤

#### 步骤1：识别技术矛盾
```
描述：我想改进技术的哪个参数？
例如：我想让设备更轻（改善参数），但不能降低强度（恶化参数）
```

#### 步骤2：查找矛盾矩阵
| 改善参数 \ 恶化参数 | 强度 | 重量 | 速度 | 精度 |
|-------------------|------|------|------|------|
| 重量 | 15,8 | - | 2,6 | 3,32 |
| 速度 | 2,6 | 2,6 | - | 13,27 |
| 精度 | 3,32 | 3,32 | 13,27 | - |

> 注：数字对应40个发明原理

#### 步骤3：应用发明原理
推荐参考 `references/triz-innovation.md` 获取完整的40个发明原理和推荐组合。

### 常用TRIZ原理速查

| 创新方向 | 推荐原理 |
|---------|---------|
| 减小体积 | 1.分割、4.不对称、7.嵌套、17.维度变化 |
| 提高强度 | 8.重量补偿、13.反向作用、14.曲面化 |
| 加快速度 | 2.抽取、13.反向作用、19.周期性动作 |
| 提高精度 | 3.局部质量、6.通用性、13.反向作用 |
| 降低成本 | 15.动态化、19.周期性动作、35.参数变化 |
| 简化结构 | 1.分割、6.通用性、15.动态化 |

## 四、技术交底书撰写

### 技术交底书的重要性
技术交底书是发明人与代理人沟通的桥梁，质量直接影响专利文件质量。

### 标准模板

```markdown
# 技术交底书

## 一、基本信息
- 发明名称：[简洁、准确的技术名称]
- 申请类型：[发明/实用新型]
- 技术领域：[所属技术领域]

## 二、背景技术（现有技术）
### 2.1 现有技术描述
[描述该技术领域的现有技术现状]

### 2.2 现有技术存在的问题
[指出现有技术的不足或缺陷]

### 2.3 现有技术存在的根本原因
[分析问题产生的技术原因]

## 三、技术方案
### 3.1 总体技术方案
[概述本发明的技术方案，不涉及具体实现细节]

### 3.2 具体技术特征
[详细描述本发明的技术特征，可包括：
- 结构特征
- 方法步骤
- 参数范围
- 材料组成等]

### 3.3 技术方案的关键创新点
[列出2-5个关键创新点]

## 四、有益效果
[描述本发明相对于现有技术的有益效果：
- 性能提升
- 成本降低
- 效率提高
- 精度改善等]

## 五、附图说明
[如果有附图，列出各附图的简要说明]

## 六、最佳实施方式
[描述实现本发明的最佳具体实施方案]

## 七、补充信息
- 是否需要保密处理：[是/否]
- 是否有在先申请：[是/否，日期]
- 其他需要说明的事项：
```

### 撰写要点

**❌ 避免**：
- 技术方案公开不充分（代理人无法理解）
- 过于笼统，缺乏具体技术特征
- 使用商业宣传语言而非技术语言
- 遗漏关键创新点

**✅ 推荐**：
- 使用"本发明"、"所述"、"包括"等技术语言
- 结构类专利配合附图说明
- 方法类专利按步骤详细描述
- 数值参数给出具体范围

## 五、权利要求书撰写

### 权利要求的结构

```markdown
1. 一种[产品/方法名称]，其特征在于：[技术特征1]，所述[技术特征1]与[技术特征2]连接/配合/关联...
```

### 独立权利要求
- 覆盖最大保护范围
- 包含解决技术问题的全部必要技术特征
- 格式："一种XXX，其特征在于..."

### 从属权利要求
- 引用独立权利要求或其他从属权利要求
- 附加技术特征进一步限定
- 格式："根据权利要求1所述的XXX，其特征在于..."

### 撰写示例

```markdown
## 示例：智能水杯

### 独立权利要求1
一种智能水杯，包括杯体（1）和杯盖（2），其特征在于：
所述杯体（1）内设置有温度传感器（3）和控制器（4）；
所述控制器（4）与温度传感器（3）电连接；
所述杯体（1）外壁设置有显示屏（5），所述显示屏（5）与控制器（4）电连接；
所述温度传感器（3）用于检测杯体内液体的温度，
并将温度信号传输至控制器（4），
控制器（4）控制显示屏（5）显示温度值。

### 从属权利要求2
根据权利要求1所述的智能水杯，其特征在于：
所述杯体（1）内还设置有液位传感器（6），
所述液位传感器（6）与控制器（4）电连接，
用于检测杯体内液体的液位。

### 从属权利要求3
根据权利要求2所述的智能水杯，其特征在于：
所述显示屏（5）还用于显示液位信息。
```

### 撰写技巧

| 技巧 | 说明 |
|-----|------|
| 布局合理 | 独权从宽，从权逐步收窄 |
| 多方案保护 | 包含多个并列技术方案 |
| 功能性描述 | "用于"、"适于"等表述 |
| 参数范围 | 给出数值范围而非单一值 |
| 避免非必要限定 | 不要加入与解决技术问题无关的特征 |

## 六、说明书写法要点

### 说明书结构

1. **技术领域**：本发明涉及XXX领域
2. **背景技术**：描述现有技术和存在的问题
3. **发明内容**：技术方案+有益效果
4. **附图说明**：各附图的简要说明
5. **具体实施方式**：最佳实施例

### 各部分写法

#### 技术领域
```markdown
本发明涉及[技术领域]，特别涉及一种[具体应用场景]的[产品/方法]。
```

#### 背景技术
```markdown
[行业背景介绍]
目前，...[描述现有技术]
然而，现有技术存在以下问题：
（1）...[问题1]
（2）...[问题2]
```

#### 发明内容
```markdown
本发明的目的是提供一种[产品/方法]，以解决...[现有技术问题]。

为实现上述目的，本发明采用以下技术方案：
一种[产品/方法]，包括...[主要组成]，其特征在于：...

本发明具有以下有益效果：
（1）...[效果1]
（2）...[效果2]
```

#### 具体实施方式
```markdown
以下结合附图和具体实施例对本发明进行详细说明。

实施例1：
如图1所示，一种智能水杯，包括杯体1、杯盖2...
具体使用时，用户...
```

### 注意事项

- **充分公开**：说明书应当对发明作出清楚、完整的说明
- **支持权利要求**：每个实施例都应能支持对应的权利要求
- **避免"大概"**：使用准确的技术术语
- **逻辑连贯**：从问题→方案→效果形成完整逻辑链

## 七、审查意见答复策略

### 常见审查意见类型

| 类型 | 说明 | 答复策略 |
|-----|------|---------|
| 新颖性问题 | 与对比文件相同或相近 | 强调区别技术特征 |
| 创造性问题 | 技术方案显而易见 | 论证技术效果和技术问题 |
| 清楚性问题 | 权利要求不清楚 | 修改表述或补充说明 |
| 支持问题 | 得不到说明书支持 | 调整范围或补充实施例 |
| 公开不充分 | 说明书公开不充分 | 补充说明或修改 |

### 答复模板

```markdown
## 对审查意见的答复

### 一、修改说明
[说明对申请文件的修改内容]

### 二、对审查意见的答复

#### 1. 关于新颖性
[不同意/同意审查意见]
[论述本申请与对比文件的区别技术特征]
[说明区别技术特征带来的技术效果]

#### 2. 关于创造性
[不同意/同意审查意见]
[论证技术方案的非显而易见性]
[强调发明的有益效果]

### 三、结论
恳请审查员在充分考虑上述意见后，批准本申请。
```

### 答复原则

| 原则 | 说明 |
|-----|------|
| 逐条答复 | 对每条审查意见都要回应 |
| 事实为依据 | 以技术事实为基础论证 |
| 修改适度 | 修改不应超出原申请范围 |
| 积极态度 | 展现配合审查的诚意 |

## 八、时间节点提醒

| 阶段 | 时间节点 | 注意事项 |
|-----|---------|---------|
| 申请 | 提交日 | 确定申请日和优先权日 |
| 初审 | 3个月内 | 补正形式问题 |
| 公布 | 申请日18个月后 | 提前公开可缩短审查周期 |
| 实审请求 | 申请日3年内 | 需缴纳实质审查费 |
| 授权 | 审查通过后 | 缴纳当年年费 |
| 年费 | 每年到期日前 | 逾期6个月可补缴但有滞纳金 |
