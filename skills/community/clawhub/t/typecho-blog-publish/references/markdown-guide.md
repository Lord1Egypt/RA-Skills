# Typecho Markdown 渲染优化指南

## 问题分析

Typecho 的 Markdown 渲染可能遇到的问题：

### 1. 代码块问题
**问题**：代码块可能不显示或格式混乱

**解决方案**：
- 使用标准的三反引号语法
- 指定语言标识（如 ```bash）
- 确保代码块前后有空行

### 2. 列表缩进问题
**问题**：列表项可能不对齐或不显示

**解决方案**：
- 确保列表项前没有多余的空格
- 列表前后保留空行
- 避免在列表中嵌套复杂内容

### 3. 标题问题
**问题**：标题可能不显示或格式不对

**解决方案**：
- 标题前后保留空行
- 使用标准的 # 语法
- 避免在标题中使用特殊字符

## 优化后的文章模板

```markdown
---
title: 文章标题
categories: 分类 1, 分类 2
tags: 标签 1, 标签 2
---

# 主标题（H1）

简短的介绍段落。

## 一级小节（H2）

正文内容。

### 二级小节（H3）

更多内容。

#### 列表示例

无序列表：
- 项目一
- 项目二
- 项目三

有序列表：
1. 第一项
2. 第二项
3. 第三项

#### 代码示例

行内代码：`code here`

代码块：
```bash
# 命令示例
echo "Hello"
```

#### 引用

> 这是引用内容
> 可以有多行

---

*文章结尾*
```

## 最佳实践

1. **标题**：前后保留空行
2. **列表**：简洁明了，避免嵌套
3. **代码**：使用标准三反引号
4. **段落**：段落之间保留空行
5. **特殊字符**：避免使用可能冲突的字符
