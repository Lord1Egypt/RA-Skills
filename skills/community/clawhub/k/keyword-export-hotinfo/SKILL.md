# 搜爆款作品小助手

面向内容创作者与运营人员，在**合规数据源约束**下快速看清话题热度并收敛可复用的爆款线索。

通过本 Skill，你可以：

- **全景热度**：围绕同一话题在三平台侧对齐爆款样本与互动信号（非「全网」口径）。
- **结构化交付**：关键词扩展、平台分层结论与聚合词表，便于选题与复盘。
- **流程可追溯**：术语、平台边界与自检规则集中在 workflow，执行路径一致。

适用于新媒体运营、选题策划、营销复盘与 MCN 内容研究等角色。

## 功能特性

### 核心功能

- **话题与关键词**：从用户描述抽取核心词与关联词，生成可检索的组合词。
- **三平台爆款检索**：仅公众号、抖音、小红书；按 workflow 中的爆款阈值与搜索模板执行。
- **校验与清洗**：平台来源、时效、互动与可信度分层校验，排除非允许平台结果。
- **关键词聚合**：在多平台样本上汇总高频主题词与可传播角度线索（细则见 `references/`）。

### 特色亮点

- **口径清晰**：用户说「全网」时仍只覆盖上述三平台，并在输出中写明范围。
- **一事一职**：完整步骤、术语表与清单在 `references/core_workflow.md`，正文保持提纲便于模型跟随。

## 使用指南

### 基础使用（4 步）

#### 第 1 步：加载完整规则

用户意图涉及**完整流程**（热点盘点、指定话题深挖、跨平台对比、聚合交付）时：

1. **必须先读取**与本 Skill 同目录下的 `references/core_workflow.md`。
2. **完整遵循**其中的触发规则、术语规范、数据来源约束、操作步骤、自检清单与注意事项。

按需参阅： `references/platform-standards.md`、`references/keyword-extraction-guide.md`、`references/data-confidence.md`、`references/angle-mining-guide.md`。

#### 第 2 步：发起热点或话题任务

用户可提供话题名称、事件描述或「看看最近什么火」类需求。

> 用户：三大平台最近关于 XX 有什么爆款？
> 助手：先按 workflow 提取与扩展关键词，再分平台检索；说明数据仅含公众号 / 抖音 / 小红书。

若用户只说「查热点」而未限定主题：**可先做广义热点盘点**，或追问是否聚焦某一话题 / 行业，避免编造具体条目。

#### 第 3 步：确认输出形态

按 workflow 交付结构化结果（如关键词块、分平台摘要、聚合词表）；若用户需要表格或简报体例，在不超过 workflow 约束的前提下对齐格式。

#### 第 4 步：复核边界

输出前对照 workflow 自检：是否混入非允许平台、是否虚构链接或互动数据、时效是否符合分级规则。

### 常用指令速查

| 命令示例                    | 功能                                          |
| --------------------------- | --------------------------------------------- |
| 「最近有什么热点 / 热搜」   | 触发场景 1：三平台热点盘点（口径见 workflow） |
| 「分析 XX 话题热度 / 爆款」 | 触发场景 2：指定话题检索与校验                |
| 「XX 的关键词 / 选题方向」  | 在 validated 样本上做聚合与角度提炼           |
| 「只看成小红书 / 公众号」   | 单平台子集分析，仍需遵守数据来源与校验规则    |

**无法执行或数据不可用**：说明原因（缺关键输入、检索失败、环境无法访问来源等），**不得编造**爆款条目、链接或精确互动数；可给出下一步建议（补全话题、收窄时间范围、改为人工复核渠道等）。

## 使用场景

| 场景            | 角色         | 需求描述                                   | 使用方式                                               |
| --------------- | ------------ | ------------------------------------------ | ------------------------------------------------------ |
| 热点响应        | 新媒体运营   | 需快速判断某类话题在三平台的爆款密度与语气 | 发起热点或指定话题查询；输出分平台摘要与聚合词         |
| 选题策划        | 内容策划     | 要把抽象想法落成可搜、可写的关键词与角度   | 走关键词提取与扩展；结合 angle 指南给出可复制切入点    |
| 战役复盘        | 品牌营销     | 活动结束后比对话题在各平台的爆款形态       | 限定话题与时间窗口；按 workflow 校验后对比结构与关键词 |
| 竞品 / 事件追踪 | 研究员 / MCN | 跟踪单一事件或名称的传播样本               | 固定核心词组合检索；清洗非三平台噪声后归档结论         |

## 注意事项与边界

- **平台**：允许数据来源仅限公众号、抖音、小红书；快手、B 站、视频号、知乎、微博等均不纳入，即使在搜索结果中出现也应排除。
- **数据真实性**：互动量、链接与发布时间须来自可核验线索；不可用惯性填充数字或虚构 URL。
- **合规表述**：遵守各平台社区规范与版权边界；对外分享时注意个人隐私与引流合规。
- **与时效**：workflow 对发布时间分级有强制规则，输出中应反映样本时效层级（如实时 / 近三日等），避免把过期内容当作当前热点主推。
