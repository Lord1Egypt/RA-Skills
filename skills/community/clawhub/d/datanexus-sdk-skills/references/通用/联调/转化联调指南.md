# 转化联调指南

## 概述

转化联调是验证 SDK 数据上报链路是否正常的关键步骤。通过模拟线上广告场景，测试从广告点击 → 数据上报 → 转化归因的完整链路。

> ⚠️ **鸿蒙端暂不支持转化联调**，鸿蒙 SDK 接入后请通过 DataNexus 日志查询和质量看板验证数据上报的正确性，参考[数据对账](./数据对账.md)。

## 一、整体流程

```
1. 创建转化规则
   └→ 投放平台 → 工具箱 → 转化归因 → 新建转化
       ├── 选择方式（SDK / API）
       ├── 填写基本信息
       ├── 配置监测链接（曝光/点击）
       ├── 通过联调
       └── 激活转化

2. 新建广告使用转化
   └→ 新建广告 → 选择转化 → 提交
```

## 二、联调步骤

### 2.1 前置准备

1. 确认数据源已创建，获取 `user_action_set_id` 和 `secret_key`
2. 确认数据源已绑定对应的 AppID
3. 确认 DataNexus 平台已开启相应的分发开关（参考[SDK选型决策树](../选型/SDK选型决策树.md)第 3 步）
4. 确认账户已通过审核，且具备联调所需的版位权限（见下表）

#### 联调版位权限要求

| 应用类型 | 联调所需版位权限 |
|---------|----------------|
| Android / iOS APP | **腾讯平台与内容媒体**版位 |
| 微信小程序 / 微信小游戏 | **微信朋友圈**版位 |
| 鸿蒙 | 暂不支持联调 |

> 如账户不具备对应版位权限，联调将无法进行。请联系行业运营确认版位权限开通情况。

### 2.2 创建联调任务

1. 进入投放平台 → 工具箱 → 转化归因
2. 选择对应数据源，创建转化规则
3. 进入联调页面，系统会基于对应版位创建联调广告

### 2.3 执行联调

1. 点击联调预览广告
2. 在应用内触发对应的转化行为（如注册、付费等）
3. 等待系统检测上报数据
4. 联调有效时间为 **1 小时**，请在有效时间内完成

### 2.4 确认联调结果

- ✅ 检测到转化数据 → 联调通过 → 激活转化规则
- ❌ 未检测到数据 → 参考[联调排障](../排障/联调排障.md)

## 三、监测链接说明

| 类型 | 作用 | Android / iOS | 小程序 / 小游戏 |
|------|------|:------------:|:--------------:|
| 点击监测链接 | 用户点击时下发信息到广告主服务器 | **必填** | 选填 |
| 曝光监测链接 | 用户曝光时下发信息，用于曝光归因 | 选填 | 选填 |

> **注意**：
> - **Android / iOS APP 的点击监测链接为必填项**，需在创建转化规则时配置
> - 曝光监测数据仅用于曝光归因，不支持其他用途
> - 点击监测和曝光监测无需重复配置，避免数据重复下发
> - 点击监测 URL 中的 `__CALLBACK__` 字段需要 URLDecode 后使用

## 四、注意事项

1. **版位权限**：联调所需版位因应用类型而异（见 2.1 节），无对应版位权限的账户无法联调
2. **鸿蒙暂不支持**：鸿蒙端暂不支持转化联调，请通过日志查询和质量看板验证
3. **新旧链路不可混用**：数据源（旧链路）和新版转化归因是不同链路，不能交叉联调
4. **状态更新延迟**：转化规则状态更新有约 **5 分钟延迟**
