## Known Risks and Mitigations: <br>
Risk: Generated tests, test plans, and security probes may be incorrect or unsuitable for the target system. <br>
Mitigation: Review generated guidance and code before using it, and run tests only against authorized local or staging environments. <br>
Risk: Database cleanup, load testing, or security testing examples could disrupt systems or data if pointed at production. <br>
Mitigation: Use synthetic accounts and data, isolate test environments, and confirm cleanup and load-test scope before execution. <br>
Risk: Strict TDD guidance may suggest deleting or rewriting code before the team has accepted that workflow. <br>
Mitigation: Treat TDD recommendations as reviewable guidance and preserve intentional implementation work unless a human owner approves changes. <br>
