## 二、分析框架详解

### 2.1 波特五力量化评估
| 力量 | 评估维度 | 评分标准(1-5) | 行业案例 |
|------|---------|--------------|---------|
| 供应商议价力 | 供应商集中度/转换成本/前向整合威胁 | 5=极强 1=极弱 | 芯片厂对手机厂(4-5) |
| 买家议价力 | 买家集中度/价格敏感度/后向整合 | 5=极强 1=极弱 | 企业客户对SaaS(3-4) |
| 新进入者威胁 | 资本需求/规模经济/品牌壁垒/政策门槛 | 5=极高 1=极低 | 社交APP进入壁垒(2) |
| 替代品威胁 | 替代品性价比/转换成本/用户意愿 | 5=极高 1=极低 | AI替代传统搜索(4) |
| 同业竞争 | 竞争者数量/行业增速/差异化程度/退出壁垒 | 5=极激烈 1=温和 | 外卖平台(5) |

### 2.2 定位图谱绘制方法
```
选择两个最关键维度作为XY轴：
常用组合：
1. 价格(低→高) × 功能丰富度(基础→全面)
2. 专业度(小白→专家) × 价格(低→高)  
3. ToB(轻) × ToB(重)
4. 通用(大众) × 垂直(小众)
5. 国内(本土化) × 国际(全球化)

绘制步骤：
1. 确定XY轴维度（必须对用户决策最重要）
2. 标注各竞品位置（圆圈大小=市场份额）
3. 标注我方当前位置和理想位置
4. 识别空白象限（无竞品覆盖的区域）
```

## 三、行业竞品分析差异化

### 3.1 SaaS/B2B产品分析要点
- **功能深度比宽度重要**：同一功能做到什么程度
- **集成生态**：API开放程度/第三方集成数量/工作流自动化
- **客户成功**：实施周期/培训成本/留存率/扩展收入占比
- **定价阶梯**：免费版功能/入门版价格/企业版溢价/按量计费模式
- **安全合规**：SOC2/ISO27001/GDPR/等保/私有化部署

### 3.2 ToC APP产品分析要点
- **用户体验**：首次使用体验/核心路径步数/上手时间
- **留存曲线**：次日/7日/30日留存率对比
- **获客渠道**：应用商店ASO/社媒投放/KOL合作/裂变增长
- **变现模式**：广告/订阅/内购/电商/增值服务
- **社区运营**：UGC活跃度/创作者激励/话题运营

### 3.3 电商平台分析要点
- **供给侧**：SKU数量/品类覆盖/自营vs平台/独家品牌
- **物流履约**：配送时效/仓储网络/退换货体验
- **价格竞争力**：补贴力度/会员价/比价优势
- **用户LTV**：复购率/客单价/会员体系/跨品类渗透
- **内容生态**：直播带货/短视频种草/买家秀/达人合作

### 3.4 内容平台分析要点
- **内容生态**：创作者数量/头部集中度/内容多样性/更新频率
- **算法推荐**：推荐精准度/信息茧房/新内容曝光机会
- **变现效率**：创作者分成/广告加载率/电商转化率
- **社区氛围**：评论质量/互动率/社区规则执行/恶意内容治理
- **差异化壁垒**：独家内容/IP/社区文化/创作者关系

## 六、常见分析误区

| 误区 | 正确做法 |
|------|---------|
| 只分析功能不对比体验 | 功能相同但体验差异可能决定胜负 |
| 只看国内忽略全球 | 海外竞品可能即将进入国内市场 |
| 静态分析忽略动态 | 竞品在变化，要做时间维度追踪 |
| 只盯直接竞品 | 替代品和潜在进入者更危险 |
| 过度模仿头部竞品 | 头部竞品的策略不一定适合你 |
| 只看产品不看商业 | 商业模式差异可能决定谁能活到最后 |
| 忽略用户真实反馈 | 数据+用户原声结合才能看到全貌 |

## 七、输出质量检查清单
- [ ] 竞品范围是否完整（直接+间接+潜在至少各1个）
- [ ] 数据来源是否标注且可溯源
- [ ] 功能对比是否有深度评分而非仅有/无
- [ ] 定价分析是否包含隐性成本（实施/培训/迁移）
- [ ] SWOT是否交叉输出4种策略而非仅列4象限
- [ ] 建议是否具体可执行（有数字/有步骤/有负责人）
- [ ] 差异化方向是否避免了"更好"而是"不同"
- [ ] 是否包含持续监控机制（不是一次性分析）
