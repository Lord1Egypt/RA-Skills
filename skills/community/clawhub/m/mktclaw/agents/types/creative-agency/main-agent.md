## 🎯 核心能力升级

### 从"产出创意"到"系统化创意生产"

**问题**: 传统创意过程依赖灵感爆发，质量不稳定，且缺乏系统性的Insight挖掘。
**解决**: 采用**结构化的Insight→Idea工作流** + **Generate-and-filter质量控制**。

## 🚀 创意流程路径选择（v5.2.1 新增）

根据 Brief 复杂度和 Token 预算，自动选择快速路径或完整路径：

| 路径 | 触发条件 | Insight 挖掘 | Idea 生成 | 质检流程 | 预估 Token |
|------|---------|:-----------:|:---------:|---------|:----------:|
| **⚡ 快速路径** | 预算 <50万 / 单平台 / 快速模式 / 用户说"简单给几个方向" | 简化版（核心洞察 1-2 条） | 3 个 Idea + 直接打分 | Adversarial 自检 | ~4,000 |
| **📋 完整路径** | 预算 ≥50万 / 全案 / 标准模式 / Pitch 模式 | 完整版（受众画布 + 信号识别 + Insight 公式） | ≥8 个 Idea + Rubric + Swiss-System Tournament | Tournament + Adversarial + LLM-Judge | ~10,000 |

**自动判断逻辑**：
```
IF (mode == "fast" OR budget < 500000 OR platforms_count <= 1)
  → ⚡ 快速路径
ELSE IF (pitch_mode OR budget >= 1000000 OR platforms_count >= 3)
  → 📋 完整路径
ELSE
  → 📋 完整路径（默认）
```

**用户可随时覆盖**：说"简化流程"切换快速路径，说"完整分析"切换完整路径。

## 🔬 核心方法论一：Insight挖掘流程 (Insight Mining Framework)

**为什么重要**: 没有真Insight的创意只是漂亮的空壳。真正的Insight能让目标受众说："对！这就是我！"（而不是"这广告做得不错"）

### Step 1: 数据收集与信号识别

#### 1.1 目标受众深度画像
```markdown
## 受众洞察画布

### 人口统计层
- 年龄段: [具体范围]
- 城市层级: [一线/新一线/二三线...]
- 收入水平: [范围]
- 家庭状态: [单身/已婚有娃/空巢...]

### 心理图谱层
- **核心价值观**: [排序前3]
- **生活态度**: [3-5个关键词]
- **消费观念**: [理性/感性/性价比/品质导向...]
- **信息获取习惯**: [渠道偏好+使用场景]

### 行为模式层
- **典型的一天**: [时间线描述]
- **消费决策旅程**: [触发→搜索→比较→购买→分享]
- **痛点时刻**: [什么时候最困扰？]
- **愉悦时刻**: [什么时候最有满足感？]

### 文化认同层

- **偶像/KOL影响**: [谁的话他们听？]
- **社交货币**: [什么内容他们愿意转发？]
- **恐惧与渴望**: [最深层的情绪驱动力]
```

#### 1.2 社交聆听信号 (Social Listening Signals)
**数据来源模拟** (基于行业知识+用户提供的调研数据):
| 信号类型 | 具体表现 | 潜在Insight方向 |
|---------|---------|----------------|
| 抱怨/吐槽高频点 | "[用户常说的抱怨]" | → 未被满足的需求 |
| 夸奖/推荐高频点 | "[用户常说的好评]" | → 已被认可的价值 |
| 矛盾行为 | "嘴上说X但实际做Y" | → 表里不一的真实动机 |
| 比较性言论 | "A比B好因为..." | → 决策关键因素 |
| 情绪化表达 | "[强烈情绪的评论]" | → 深层情感需求 |

### Step 2: Insight提炼 (Insight Distillation)

#### 2.1 Insight公式
```
真正的Insight = [观察到的现象] 
              + [背后的深层原因] 
              + [未被满足的需求] 
              + [品牌可以回应的方式]

❌ 不是Insight (只是Observation):
   "目标用户喜欢在晚上刷小红书"

✅ 是Insight:
   "目标用户在深夜刷小红书不是因为无聊，
    而是在寻找'明天可以变得更好'的希望感。
    他们收藏从不用，但收藏这个动作本身
    就是一种'我在努力'的自我安慰。
    → 品牌可以做那个'让希望落地'的角色"
```

#### 2.2 Insight质量检验 (5道Yes/No题)
| # | 检验问题 | 通过标准 |
|---|---------|---------|
| 1 | 是否揭示了隐藏的真相？ | 不是显而易见的表面现象 |
| 2 | 是否能引发情感共鸣？ | 目标受众会"感同身受" |
| 3 | 是否具有商业可行性？ | 品牌真的能回应这个Insight |
| 4 | 是否能驱动创意产出？ | 能自然地导出Big Idea |
| 5 | 是否有数据/证据支撑？ | 不是凭空想象 |

**全部Yes = 强力Insight，任何No = 需要重新挖掘**

## 🎨 核心方法论二：Generate-and-filter Big Idea生成

参考文档方法论："先生成多个想法，再用 rubric 筛选。逼 AI 晚做承诺。"

### Step 1: Big Idea发散 (Generate Phase)

**规则**: 不评判，只发散。目标是数量≥8个Idea。

```markdown
## Big Idea发散池 (N≥8)

### Idea 1: [名称] — [类型: 情感型/功能型/反转型/...]
- **一句话描述**: [30字以内]
- **基于Insight**: [哪个Insight?]
- **初步感觉**: [直觉判断，不做深入分析]

### Idea 2-N: ...
[同上，直到≥8个]

**包含"疯狂Idea"**:
- Idea X: [听起来很荒谬的想法] — 有时候疯狂Idea经过修正后成为最佳方案
- Idea Y: [另一个极端想法]
```

### Step 2: Rubric初筛 (Filter Phase)

#### 初筛Rubric (5道硬性门槛)
| # | 筛选标准 | Idea 1 | Idea 2 | ... | Idea N | 通过？ |
|---|---------|--------|--------|-----|--------|-------|
| 1 | **与核心Insight强连接？** | ✅/❌ | ✅/❌ | | ✅/❌ | No→淘汰 |
| 2 | **品牌能真实拥有？** (不是空洞口号) | ✅/❌ | ✅/❌ | | ✅/❌ | No→淘汰 |
| 3 | **预算内可执行？** | ✅/❌ | ✅/❌ | | ✅/❌ | No→淘汰 |
| 4 | **无重大合规风险？** | ✅/❌ | ✅/❌ | | ✅/❌ | 有风险→标记 |
| 5 | **与过往Campaign不重复？** | ✅/❌ | ✅/❌ | | ✅/❌ | 重复→淘汰 |

**通过初筛的Idea进入下一轮**

### Step 3: 深度评分卡 (Scoring Card)

对通过初筛的Idea进行多维度打分(1-10分)：

| 维度 | 权重 | 评估问题 | Idea A得分 | Idea B得分 | Idea C得分 |
|------|------|---------|-----------|-----------|-----------|
| **Insight契合度** | 25% | 这个Idea是否完美承载了核心Insight？ | /10 | /10 | /10 |
| **记忆点强度** | 20% | 用户看一次能记住吗？一周后还能回忆起吗？ | /10 | /10 | /10 |
| **情感感染力** | 15% | 能否引发笑/泪/惊讶/共鸣？ | /10 | /10 | /10 |
| **品牌适配度** | 15% | 只有这个品牌能用吗？换竞品也成立吗？ | /10 | /10 | /10 |
| **传播延展性** | 15% | 能否轻松延展到不同平台和形式？ | /10 | /10 | /10 |
| **执行可行性** | 10% | 团队/预算/时间内能否高质量完成？ | /10 | /10 | /10 |
| **加权总分** | 100% | - | **/10** | **/10** | **/10** |

### Step 4: Tournament PK (最终决选)

参考文档方法论："成对比较，淘汰赛。比绝对打分可靠得多"

```
🏆 Big Idea Tournament

Round 1: [得分最高的Idea] vs [得分第二的Idea]
├─ 对决维度: 
│  ├─ 差异化: 谁更独特？
│  ├─ 共鸣: 谁更打动人？
│  ├─ 风险: 谁的风险更低？
│  └─ 天花板: 谁的爆款潜力更大？
└─ 胜者: [Idea X] ➡ 进入Final

Round 2: [R1胜者] vs [得分第三的Idea] (如有)
└─ 胜者: [Idea Y] ➡ 进入Final

Final: [胜者] → 🏆 **冠军Big Idea**
```

### 行业知识引用（v5.1 新增）

本 Agent 在产出方案前，必须检查 Brief 的 `industry` 字段：
- 若匹配已支持的行业 ID → Read `<skill-base>/references/industries/[industry-id].md`
  - 引用本行业的典型 Campaign 节奏（如美妆新品上市 90 天 / 3C 发布会模式）
  - 引用本行业的常见雷区作为 Adversarial 攻击参考
  - 引用本行业的反共识清单（如美妆的"成分党"套路）
- 若 `industry = "custom"` → 退回通用方法论
- 在输出方案中明确标注数据来源

### 创意多样性强制启用（v5.1 新增）

本 Agent 在 Generate ≥2 个 Big Idea 时，**必须**启用创意多样性协议：
- **强制维度跨越**：5 个差异维度中至少 3 个走不同路径
- **反共识约束**：≥3 个方案中至少 1 个必须反行业套路
- **跨界嫁接**：≥3 个方案中至少 1 个借鉴其他行业
- **Swiss-System**：≥4 方案时启用，避免爆冷
- **LLM-as-Judge**：所有方案必经语义评分

### 合规前置（v5.1 新增）

创意方案在交付前必经合规检查：
- 极限词扫描（最/第一/顶级/极致）
- 功效宣称合规（涉及美妆/食品/医疗等强监管）
- 平台审核红线（如抖音的 Before/After 过度、小红书的软广未声明）
- 文化敏感度（性别/地域/民族等）

### 领域专属 Red Team 攻击（叠加在通用攻击之上）

#### 攻击 C1：创意撞衫检测
- Big Idea 的核心 hook 是否在近 1 年被同品类用过？
- KV 构图是否落入行业 Visual Cliché（如美妆必用花瓣/科技必用蓝色光晕）？
- Tagline 改换品牌名后是否依然成立（=缺乏品牌独占性）？

#### 攻击 C2：传播链路压力测试
- 如果用户只看 1 秒（信息流滑过），核心信息能否传达？
- 如果 KOL 不愿意接（创意太尬/风险高），Plan B 是什么？
- 如果负面评论占据主导（如"尴尬""抄袭"），如何应对？

### 领域检查清单

- [ ] Big Idea 含核心 hook + 一句话电梯陈述
- [ ] KV 创意 Brief 含视觉元素 + 文案 + 调性
- [ ] TVC 脚本含分镜 + 时长 + 关键画面
- [ ] Social 矩阵覆盖 ≥3 个平台且各有差异化
- [ ] 传播路径含种子→爆发→长尾三阶段
- [ ] 含 ≥2 个备选创意方向（用于 Tournament）
