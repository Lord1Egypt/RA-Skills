# 概念八维度解剖 · concept-anatomy

> **定位**：把"学懂一个概念"这件事，从"百度一下"提升到"八维手术刀式理解"。
> **方法论**：ljg-learn 八维度解剖模型 + 善人「投资概念特化」。
> **版本**：v1.0.0 | 2026-06-26

---

## 🎯 技能目标

**核心价值**：用 **8 个独立维度**对单一概念进行**完整解剖**，最终浓缩为一句不超过 30 字的核心顿悟。

**8 个维度**：
1. **历史** — 概念诞生的背景、起源、解决什么问题
2. **辩证** — 概念的正面/反面/对立面
3. **现象** — 日常生活中的 3 个具体表现
4. **语言** — 字/词根、同义词/反义词
5. **形式** — 抽象为数学/逻辑/图形表示
6. **存在** — 什么条件下存在/消失
7. **美感** — 美学/艺术/设计中的对应物
8. **元反思** — 常见误解、自身理解的盲区

**与"百度一下"的核心差异**：
- ❌ 百度：定义 + 例子的扁平信息
- ✅ concept-anatomy：8 维度的立体结构 + 一句话核心顿悟

## 🧠 认知框架：八维度解剖模型

> **核心命题**：真正"学懂"一个概念 = 用 8 个不同视角同时审视它。
> 每个维度揭示概念的**不同侧面**，8 个维度合起来就是**概念的完整 DNA**。

### 为什么是 8 个维度？

| 维度 | 解决的问题 | 缺失的后果 |
|------|----------|----------|
| **历史** | 这东西怎么来的？ | 只知道"是什么"，不知道"为什么是" |
| **辩证** | 它和谁对立？ | 看不到概念边界，混淆相似概念 |
| **现象** | 它在生活中长什么样？ | 抽象但无法落地 |
| **语言** | 它的字面意思是什么？ | 误读概念，错用术语 |
| **形式** | 能用数学/逻辑表达吗？ | 无法精确推理 |
| **存在** | 什么条件下成立？ | 误用场景，张冠李戴 |
| **美感** | 它美在哪？ | 缺乏深层共鸣，记忆不牢 |
| **元反思** | 我可能错在哪？ | 自我盲区，发现不了误解 |

### 维度顺序的逻辑

```
1. 历史（起源）→ 2. 辩证（对立）→ 3. 现象（落地）
        ↓
4. 语言（字面）→ 5. 形式（抽象）→ 6. 存在（条件）
        ↓
7. 美感（情感）→ 8. 元反思（自省）
```

**前半段（1-3）**：从外部视角理解（背景/对立/现实）
**中段（4-6）**：从内部视角理解（语言/形式/边界）
**后半段（7-8）**：从情感/反思视角（美感/盲区）

## 🔄 处理流程（8 步）

详细流程见 [`PROCESS.md`](./PROCESS.md)。简要步骤：

```
Step 1: 历史维度 → 概念起源、背景、解决的问题
Step 2: 辩证维度 → 找对立面，正反合
Step 3: 现象维度 → 3 个日常具体表现
Step 4: 语言维度 → 字根、词源、同义/反义词
Step 5: 形式维度 → 数学/逻辑/图形表示
Step 6: 存在维度 → 出现条件 / 消失条件
Step 7: 美感维度 → 美学/艺术/设计的对应物
Step 8: 元反思维度 → 常见误解 + 自省盲区
        ↓
Final: 浓缩为 1 句核心顿悟（≤ 30 字）
```

## 🚧 边界

| 不做 | 原因 |
|------|------|
| 工具/方法（不是概念） | 如"SOP"、"OKR"偏方法论，应用 `thinking-toolbox` |
| 极宽泛概念（如"哲学"、"科学"） | 应拆分为子概念分别解剖 |
| 纯技术术语（只关心定义） | 应用 `deep-questioning` 即可 |
| 主观感受类（如"幸福"） | 可解剖但侧重辩证/元反思维度 |

## 🔗 上下游串联

### 上游
- **`deep-questioning`**：用 5 层追问初步了解概念
- **`shanren-lens`**：善人视角筛选

### 下游
- **`essence-trace`**：对概念做更深的本质追问
- **`shanren-investment-decision`**：投资概念特化后做决策
- **`article-writing`**：把解剖结果转为文章

### 预拼装工作流

**`concept-anatomy-flow`（深度学懂一个概念）**：
```
deep-questioning (5层初步了解)
  → concept-anatomy (8维度解剖)
    → essence-trace (追本溯源)
      → article-writing (输出文章)
```

**`investment-concept-flow`（投资概念特化）**：
```
concept-anatomy -mode=investment (投资特化解剖)
  → investment-analysis-kit (指标化)
    → shanren-investment-decision (决策)
```

## 🎯 与现有技能的关系

| 现有技能 | 关系 | 互补点 |
|---------|------|--------|
| `cognitive-os` | 体系级 | cognitive-os 是"哲学工具箱"，concept-anatomy 是"工具实例" |

**核心理念区分**：
- `cognitive-os`：**OS 层**（遇到问题如何思考）
- `thinking-toolbox`：**工具层**（有哪些思维工具）
- `concept-anatomy`：**实例层**（对一个概念怎么解剖）
- `deep-questioning`：**方法层**（如何提问）
