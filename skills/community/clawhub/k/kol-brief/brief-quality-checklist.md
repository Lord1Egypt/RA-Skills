# KOL Brief Quality Checklist

Review every brief against this list before sending. Categories 1–4 are blocking: fix failures before delivery.

## 1. Positioning
- [ ] Exactly one positioning angle
- [ ] Angle names the audience's reason to buy, not a brand slogan
- [ ] Angle matched to THIS creator's niche and audience demographics
- [ ] Angle is one readable sentence
- [ ] No second angle smuggled into talking points

## 2. Talking points
- [ ] Exactly 3 talking points
- [ ] Every point has attached proof (demo moment, stat, or comparison)
- [ ] Stats are real and traceable to brand materials
- [ ] At least one point is an on-camera demonstrable moment
- [ ] No point duplicates the angle verbatim

## 3. Deliverables
- [ ] Content type and count specified
- [ ] Minimum duration stated
- [ ] Exact posting date window stated
- [ ] Required hashtags and @mentions listed
- [ ] Link placement specified (anchor/bio/pin)
- [ ] Draft review deadline and turnaround stated
- [ ] Revision policy stated (rounds + timeframe)

## 4. Compliance
- [ ] Category-specific banned claims listed VERBATIM in don'ts
- [ ] Compliant alternative phrasing provided
- [ ] Disclosure method specified (toggle + verbal/caption)
- [ ] Price/discount claims bounded to campaign dates
- [ ] Required disclaimers included where applicable
- [ ] Dos ≤7 and don'ts ≤7

## 5. Compensation & rights
- [ ] Structure with actual numbers (flat/commission/hybrid)
- [ ] Payment trigger and schedule stated
- [ ] Usage rights: ad re-use, duration, whitelisting addressed
- [ ] Exclusivity window stated or explicitly "none"
- [ ] Free product / sample policy stated

## 6. Timeline
- [ ] Ship date → draft → feedback → posting → payment chain complete
- [ ] Dates are realistic (shipping buffer, review turnaround)
- [ ] Campaign window matches any discount validity

## 7. Format & tone
- [ ] Total length ≤2 pages
- [ ] Scannable: tables and short lists, no paragraph walls
- [ ] Mandatory elements clearly separated from creator-choice elements
- [ ] No word-for-word script (unless LIVE run-of-show)
- [ ] No internal notes, placeholder brackets, or "TBD" remaining
- [ ] Named contact with response-time expectation included

## 8. LIVE-specific (when applicable)
- [ ] Run-of-show with timed segments
- [ ] Price/code repetition cadence specified
- [ ] Pinned-product and link logistics covered
- [ ] Q&A FAQ sheet attached
