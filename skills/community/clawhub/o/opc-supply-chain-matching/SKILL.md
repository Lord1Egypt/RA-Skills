## 一、Skill概述

### 1.1 定位与价值

OPC供应链对接平台是胡田OPC导师体系中的**上游能力赋能工具**，定位为社区生产制造能力的统一对接平台。它不是简单的供应商黄页，而是扎根于OPC导师社区的**能力协作网络**，通过内循环撮合+外循环集采两大核心机制，帮助社区成员突破"单体OPC生产制造能力不足"的瓶颈。

**一句话定位**：社区内部协作共享 + 外部供应商集群议价。

### 1.2 核心解决的问题

| 痛点 | 解决方案 |
|------|----------|
| OPC单体对接供应商，议价能力弱 | 社区统一谈判，集群采购议价 |
| 小批量打样成本高、周期长 | 内循环成员协作，分摊成本 |
| 缺乏供应商筛选和评估能力 | 平台建立供应商准入标准和评估体系 |
| 跨环节协调难、管理成本高 | 平台统一对接，全流程服务支持 |

### 1.3 双层循环架构

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                        OPC供应链对接平台 双层循环架构                        │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  【内循环】社区分包协作                                                        │
│  ┌─────────────────────────────────────────────────────────────────┐        │
│  │  场景：小样、demo、样机                                          │        │
│  │  机制：成员之间撮合，走分包网                                    │        │
│  │  优势：信任基础、低成本、响应快                                  │        │
│  │  代表：设计外包、算法外包、小批量试制                            │        │
│  └─────────────────────────────────────────────────────────────────┘        │
│                              ↓ 上游能力不足时                               │
│  【外循环】平台集采对接                                                        │
│  ┌─────────────────────────────────────────────────────────────────┐        │
│  │  场景：中试、量产、产品化                                        │        │
│  │  机制：平台统一对接外部供应商                                    │        │
│  │  优势：规模议价、专业筛选、全流程支持                            │        │
│  │  代表：模具制造、PCB制板、整机组装、认证检测                    │        │
│  └─────────────────────────────────────────────────────────────────┘        │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 1.4 与其他Skill的关系

```
获客分包平台（接单） ←→ 供应链对接（本Skill，生产） ←→ 市场销售渠道对接（出货）
        ↓                         ↓                          ↓
      客户需求                 上游能力                   销售渠道
```

三者构成OPC供需闭环，但各自独立运作——项目方不一定需要全闭环。

## 二、供应链需求诊断

### 2.1 需求层级判断

根据项目阶段判断需要的制造能力层级：

| 层级 | 阶段 | 典型需求 | 内/外循环 |
|------|------|----------|-----------|
| **L1 概念验证** | 想法验证 | 手板、mockup、3D打印 | 内循环优先 |
| **L2 原型开发** | 原理样机 | 功能样机、首批小样 | 内循环+外循环 |
| **L3 工程验证** | 设计定型 | 工程样机、小批试制 | 外循环为主 |
| **L4 生产验证** | 工艺定型 | 中批试产、工艺验证 | 外循环 |
| **L5 量产** | 规模生产 | 大批量生产、产能爬坡 | 外循环+战略合作 |

### 2.2 内/外循环判断标准

**优先内循环的场景**：
- 单次需求量小（<100件）
- 技术要求非标准化，需要深度协作
- 社区内有现成能力方
- 保密要求高，不适合外发

**优先外循环的场景**：
- 单次需求量中等及以上（≥100件）
- 工艺要求标准化，市场成熟
- 需要专业资质（认证、检测）
- 成本敏感，需要规模效应

### 2.3 需求画像模板

**供应链需求诊断单**

| 字段 | 内容 | 示例 |
|------|------|------|
| **项目名称** | 需求所属项目 | 智能水质监测仪 |
| **当前阶段** | 所处研发阶段 | L2原型开发 |
| **需求类型** | 需要的制造环节 | PCB制板+整机组装 |
| **技术参数** | 关键技术指标 | 4层板、10cm×10cm |
| **需求量级** | 单次订单数量 | 首批50件 |
| **预算范围** | 可接受价格区间 | 5000-8000元 |
| **时间要求** | 交付时间要求 | 15个工作日内 |
| **质量要求** | 验收标准 | 符合IPC二级标准 |
| **特殊要求** | 资质/保密等 | 需要ROHS认证 |
| **循环判断** | 内/外循环 | 外循环优先 |

**详细文档**：`references/01-供应链需求诊断指南.md`

## 三、内循环匹配机制

### 3.1 社区分包网定位

社区分包网是OPC社区内部的能力共享网络，解决"有人需要、有人能供"的信息撮合问题。

```
┌─────────────────────────────────────────────────────────────┐
│                    社区分包网 运作机制                        │
├─────────────────────────────────────────────────────────────┤
│  【能力登记】           【需求发布】          【撮合成交】      │
│      │                     │                    │           │
│      ▼                     ▼                    ▼           │
│  OPC能力展示          需求方发布需求        平台智能匹配       │
│  能力标签化          社区内部分发          双方确认合作       │
│  状态同步            响应方接单            进入执行交付       │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 内循环能力分类

| 能力类别 | 典型服务 | 适用场景 |
|---------|---------|---------|
| **设计类** | 外观设计、结构设计、UI设计 | 产品概念阶段 |
| **研发类** | 硬件开发、嵌入式软件、算法 | 原型开发阶段 |
| **加工类** | 3D打印、CNC加工、手板制作 | 样机制作阶段 |
| **电子类** | PCB设计、焊接贴片、测试 | 电路板阶段 |
| **软件类** | APP开发、小程序、Web后台 | 数字化阶段 |
| **认证类** | 安规测试、EMC测试、认证咨询 | 产品认证阶段 |

### 3.3 内循环协作流程

```
Step 1：能力登记
└── OPC在平台登记能力，设置可承接范围和价格区间

Step 2：需求发布
└── 有需求的OPC发布分包需求，说明需求详情和预算

Step 3：智能匹配
└── 平台根据能力标签、负载状态、历史评价推荐候选方

Step 4：双方对接
└── 需求方与能力方直接沟通，确认技术细节和合作条件

Step 5：签约执行
└── 通过分包协议正式确立合作，进入执行交付

Step 6：验收结算
└── 交付完成后按协议分账，更新双方信用记录
```

### 3.4 内循环定价参考

| 能力类型 | 定价方式 | 参考区间 |
|---------|---------|---------|
| 设计类 | 按项目/按页/按套 | 500-5000元/项目 |
| 研发类 | 按人天/按项目 | 800-2000元/人天 |
| 加工类 | 按件/按工时 | 50-500元/件 |
| 电子类 | 按项目/按点位 | 1000-10000元/项目 |
| 软件类 | 按功能点/按人天 | 500-3000元/功能点 |

## 四、外循环供应商管理

### 4.1 供应商分类体系

按制造环节分类，不做复杂的分级体系——不同供应商解决不同问题。

| 环节类别 | 涵盖范围 | 典型供应商类型 |
|---------|---------|---------------|
| **ID设计** | 外观造型、CMF设计 | 工业设计公司、独立设计师 |
| **结构设计** | 结构工程、堆叠设计 | 结构工程师、ID工作室 |
| **手板制作** | 3D打印、CNC手板、硅胶复模 | 手板厂、模型工作室 |
| **模具制造** | 注塑模具、冲压模具、压铸模具 | 模具厂、模具服务商 |
| **PCB制造** | PCB制板、SMT贴片 | PCB厂商、SMT工厂 |
| **整机组装** | 焊接、装配、测试 | 组装厂、EMS厂商 |
| **认证检测** | 安规认证、EMC测试、性能测试 | 检测机构、认证咨询公司 |
| **包装供应** | 包装设计、包装材料 | 包装厂、包材供应商 |
| **量产代工** | 大批量生产、产能外包 | 代工厂、OEM/ODM厂商 |

### 4.2 供应商准入标准

所有入库供应商必须通过以下审核：

| 审核维度 | 审核内容 | 通过标准 |
|---------|---------|---------|
| **资质合规** | 营业执照、行业资质 | 证照齐全、合法经营 |
| **能力验证** | 设备清单、技术能力 | 设备与描述匹配 |
| **质量体系** | 质量管控、检验流程 | 有基本质量文件 |
| **配合意愿** | 响应速度、沟通态度 | 3个工作日内响应 |
| **价格水平** | 报价单、对比市场 | 不高于市场均值20% |
| **信用背调** | 工商信息、司法风险 | 无重大风险记录 |

### 4.3 供应商评估体系

按四大维度评估，不设等级，只有"可用/慎用/淘汰"标签：

| 评估维度 | 评估指标 | 评估方法 |
|---------|---------|---------|
| **交付质量** | 产品合格率、返工率 | 批次抽检、客户反馈 |
| **成本控制** | 报价竞争力、性价比 | 多供应商比价 |
| **交期保障** | 准时交货率、周期波动 | 交期记录统计 |
| **配合度** | 响应速度、问题解决 | 沟通记录评价 |

**供应商标签**：
- 🟢 **可用**：四维评估均合格，持续合作
- 🟡 **慎用**：某维度存在问题，谨慎使用
- 🔴 **淘汰**：存在严重问题，不再推荐

### 4.4 供应商库管理

```
┌─────────────────────────────────────────────────────────────┐
│                    供应商库 运营机制                          │
├─────────────────────────────────────────────────────────────┤
│  【入库】新供应商申请 → 资质审核 → 能力验证 → 试单合作        │
│     ↓                                                     │
│  【在库】正常合作 → 定期评估 → 动态调整                     │
│     ↓                                                     │
│  【出库】严重问题 → 淘汰处理 → 档案留存                     │
└─────────────────────────────────────────────────────────────┘
```

## 五、供应商对接SOP

### 5.1 完整对接流程（8步）

```
Step 1 需求确认
   ↓ 明确供应链需求、参数、数量、预算、时间
Step 2 循环判断
   ↓ 判断内循环还是外循环
Step 3 供应商匹配
   ↓ 根据需求匹配候选供应商
Step 4 需求描述发送
   ↓ 使用标准模板向供应商发送需求
Step 5 报价收集与比较
   ↓ 收集至少2家供应商报价，进行比价分析
Step 6 商务谈判
   ↓ 基于量价关系、付款条件进行议价
Step 7 合同签署
   ↓ 使用标准合同模板，审核关键条款
Step 8 交付验收
   ↓ 按验收标准执行交付，保留验收记录
```

## 六、成本控制与议价策略

### 6.1 集群采购逻辑

社区通过聚合多个OPC的同类需求，实现规模效应：

```
┌─────────────────────────────────────────────────────────────┐
│                    集群采购 运作模式                          │
├─────────────────────────────────────────────────────────────┤
│  【分散需求】           【聚合采购】          【规模议价】      │
│      │                     │                    │           │
│      ▼                     ▼                    ▼           │
│  OPC A: 50件            聚合: 500件           单价降低       │
│  OPC B: 100件     →    统一谈判         →    15-30%        │
│  OPC C: 200件          匹配供应商             交期优化       │
│  OPC D: 150件          签订框架协议            账期改善       │
└─────────────────────────────────────────────────────────────┘
```

### 6.2 集群采购触发条件

| 条件类型 | 触发标准 | 执行策略 |
|---------|---------|---------|
| **数量聚合** | 同类需求≥3家OPC，总量≥200件 | 统一谈判 |
| **周期聚合** | 季度内同类需求≥500件 | 签订季度框架 |
| **年度聚合** | 年度同类需求≥2000件 | 签订年度框架 |

### 6.3 议价策略

#### 策略1：量价关系

| 订单量级 | 预期单价降幅 | 谈判目标 |
|---------|-------------|---------|
| 100-500件 | 5-10% | 争取10% |
| 500-1000件 | 10-15% | 争取15% |
| 1000-5000件 | 15-25% | 争取20% |
| >5000件 | 25-35% | 争取30% |

#### 策略2：付款条件优化

| 付款方式 | 供应商偏好 | 社区争取 |
|---------|-----------|---------|
| 预付100% | 高风险 | 不接受 |
| 预付30%+尾款到付 | 中等风险 | 可接受 |
| 预付30%+70%到付 | 低风险 | 争取此条件 |
| 月结30天 | 最优条件 | 需要量支撑 |

#### 策略3：账期争取

| 合作时长 | 可争取账期 | 条件 |
|---------|-----------|------|
| 首次合作 | 预付为主 | 无账期 |
| 3个月内 | 30%预付+月结 | 按时付款记录 |
| 6个月以上 | 月结30天 | 持续稳定订单 |
| 战略合作 | 月结60天 | 年度框架协议 |

### 6.4 成本透明化

帮助OPC理解成本构成，避免被高价或低价误导：

| 成本构成 | 占比参考 | 说明 |
|---------|---------|------|
| **材料成本** | 40-60% | 原材料采购费用 |
| **加工成本** | 20-35% | 人工、设备折旧 |
| **管理成本** | 5-10% | 工厂运营费用 |
| **利润** | 10-15% | 合理利润空间 |
| **物流税费** | 3-5% | 运输和税费 |

**成本预警线**：
- 低于成本构成下限：可能存在偷工减料风险
- 高于成本构成上限：议价空间较大

**详细文档**：`references/05-成本控制与议价策略.md`
