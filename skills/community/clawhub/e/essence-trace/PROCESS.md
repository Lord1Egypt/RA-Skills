# essence-trace 处理流程

> **版本**：v1.0.0 | 2026-06-26
> **目的**：详细描述 6 步深钻流程 + 终止条件

---

## 完整处理流程（6 步）

### Step 1：初始问题陈述

**目标**：用 1 句话清晰陈述待溯源的问题/观点/现象。

#### 关键问题
- 用户给的输入是"模糊"还是"清晰"？
- 模糊 → 先用一句具体的话重述
- 清晰 → 直接进入 Step 2

#### 输出格式
```
初始问题：[一句话清晰陈述]
输入类型：[观点 | 现象 | 问题 | 决策疑问]
```

#### 重要原则
- **避免复杂的复合问题**（如"A 和 B 哪个好，又如何如何"）
- 拆分为多个单一问题，分别溯源
- 用"是什么 / 为什么 / 怎么" 明确问题类型

---

### Step 2：第 1 层追问

**目标**：找到问题的"前置问题"——这个问题之所以存在的更基础问题。

#### 关键问法
- "这个问题为什么会存在？"
- "要回答这个问题，需要先回答什么？"
- "这个问题的提出者，背后假设了什么？"

#### 追问技巧
- **剥假设**：列出问题中的所有假设，逐一剥离
- **找前置**：问"在 X 之前的 Y 是什么"
- **反向问**：问"如果不是 X，会怎样"

#### 输出格式
```
Q1（前置问题）：[问题]
  假设：[问题中隐含的假设]
  剥离假设后的本质：[剥假设后的核心]
```

---

### Step 3：连续追问（5 Whys）

**目标**：基于 Step 2 的回答，继续追问，直到找到根因。

#### 追问公式
```
Q(n+1) = 为什么 Q(n) 的答案是这样？
```

#### 追问技巧
- **不停留在表象**：每次追问都试图**深入一层**
- **关注机制**：问"通过什么机制"，而不是"是什么"
- **挑战前提**：每层都问"这个前提成立吗"

#### 典型深度
| 深度 | 状态 | 例子 |
|------|------|------|
| L1 | 表象 | "房价会涨，因为政策支持" |
| L2 | 机制 | "政策要稳增长" |
| L3 | 根因 | "稳增长靠投资驱动" |
| L4 | 基础 | "投资靠土地信用扩张" |
| L5 | 第一性 | "土地是财富的锚 + 政府信用背书" |

#### 输出格式
```
Q1: [问题 1]
  → Q2: [问题 2]
    → Q3: [问题 3]
      → Q4: [问题 4]
        → Q5: [问题 5]
          → [不可分节点]: [终止]
```

---

### Step 4：识别不可分节点

**目标**：找到"物理/逻辑层面无法再分"的节点，标记为深钻终点。

#### 终止判定（详见 SKILL.md 终止条件）

**类型 1：物理/数学不可分**
- 物理定律（重力、相对论、热力学）
- 数学公理（欧氏几何、集合论）
- 物质基本单位（原子、夸克）

**类型 2：逻辑/概念不可分**
- 不可定义的基本概念（意识、存在、时间）
- 公理化体系起点
- 自明之理

**类型 3：经验/实证终止**
- 实验事实（光速 30 万 km/s）
- 统计规律（大数定律）

**类型 4：选择/价值终止**
- 个人选择/偏好
- 价值判断（善恶/美丑）

#### 识别技巧
- 追问到"它就是这样" → 物理/经验终止
- 追问到"这是定义" → 概念终止
- 追问到"我想要" → 价值终止
- 追问到循环论证 → 已经触及框架边界

#### 输出格式
```
不可分节点：[节点]
终止类型：[物理 | 逻辑 | 经验 | 价值]
原因：[为什么这是不可分的]
```

---

### Step 5：回溯链条

**目标**：从不可分节点反向画出"问题 → 子问题 → ... → 不可分节点"的完整链条。

#### 链条画法

**线性链条**（最常见）：
```
Q1 → Q2 → Q3 → Q4 → Q5 → 不可分节点
```

**树状链条**（问题有多个分支）：
```
       Q1
      /  \
   Q2a   Q2b
   |      |
  Q3a   Q3b
   \      /
    不可分节点
```

**循环链条**（少见但可能出现）：
```
Q1 → Q2 → Q3 → Q1（循环）→ 终止于价值判断
```

#### 输出格式
```
追问链：
[从下往上/从左往右画]
  Q1: [问题 1]
    → Q2: [问题 2]
      → Q3: [问题 3]
        → [不可分节点]: [终点]

每个 Q 的简要回答（可选）：
  Q1 答：[简答]
  Q2 答：[简答]
  Q3 答：[简答]
```

---

### Step 6：杠杆点识别

**目标**：在追问链中找到 2-3 个**高杠杆节点**——小改变带来大影响的位置。

#### 杠杆点判定标准

| 标准 | 解释 |
|------|------|
| **多节点影响** | 这点改变 → 多个上下游节点都变？ |
| **长期/短期双重** | 短期见效 + 长期有效？ |
| **整体效率** | 提升系统整体效率？ |
| **不可逆性低** | 错了可以低成本回退？ |

#### 12 个杠杆点层级（Donella Meadows）

```
12. 参数（最低杠杆）— 数字调整
11. 缓冲器大小
10. 库存-流量关系
9. 延迟
8. 负反馈回路
7. 驱动增长的正反馈
6. 信息流结构
5. 系统规则
4. 自组织能力
3. 系统目标
2. 范式/思维方式（最高杠杆）
1. 超越范式（最高）
```

**实战中常用的杠杆层级**：
- **L5-L7**：具体可操作的杠杆点（规则、反馈、增长驱动）
- **L2-L3**：思维方式和目标（高杠杆但难改变）

#### 输出格式
```
关键杠杆点：
① [杠杆点 1] — 杠杆层级：[L1-L12]
   影响力：[影响哪些节点]
   可操作性：[短期可做 / 长期可做 / 需改变思维方式]

② [杠杆点 2] — 杠杆层级：[L1-L12]
   影响力：[影响哪些节点]
   可操作性：[短期可做 / 长期可做 / 需改变思维方式]

③ [杠杆点 3] — 杠杆层级：[L1-L12]
   影响力：[影响哪些节点]
   可操作性：[短期可做 / 长期可做 / 需改变思维方式]
```

## Final：本质输出

**目标**：用一句话表达"这个问题的本质是 ______"。

### 公式

```
本质 = [问题的根因/核心机制] = [最简化的洞察]
```

### 评判标准
- [ ] 不超过 30 字
- [ ] 包含追问链的核心
- [ ] 揭示了"为什么是这样"的机制
- [ ] 可作为"问题名片"使用
- [ ] 暗含解决方案的指引

### 反例 vs 正例

- ❌ "房价是综合因素决定的"（空洞）
- ❌ "很多人买不起房"（没说本质）
- ✅ "房价 = 土地财政 + 货币信用 + 供需失衡的共振"（机制清晰）
- ✅ "创业失败率高 = 现金流断裂 + 市场假设错的叠加"（机制清晰）

## 5 Whys 变体：7 Whys / 3 Whys

**不是机械地问 5 次**，而是问**到根因**。

### 何时 3 Whys 足够
- 简单的工程问题（机器故障）
- 经验上已有明确因果链
- 时间紧迫，需要快速决策

### 何时需要 7-10 Whys
- 复杂的社会/经济/系统问题
- 表面答案都"对但浅"
- 涉及多因素叠加

### 何时根本问不下去
- 进入哲学终极问题
- 进入价值判断（"你想要什么"）
- 进入不可证伪命题

**处理方式**：
- 3 Whys 足够 → 直接输出
- 7+ Whys → 标记为"复杂问题"，输出完整链条
- 问不下去 → 标记为"已触及认知边界"

## 投资问题特化

**触发**：用户启用 `-mode=investment` 或问题为投资类

### 投资常见追问路径
```
价格涨跌 → 业绩驱动 → 行业景气 → 货币环境 → 经济周期 → 政策周期
→ 国际资本流动 → 美元周期 → 全球信用体系 → [不可分：人性 + 资源稀缺]
```

### 投资杠杆点
- **高杠杆**：货币政策、监管政策、产业链关键节点
- **中杠杆**：行业格局、公司治理、估值切换
- **低杠杆**：短期资金面、技术指标、消息面

### 善人特化
- 杠杆点必须能"为我所用"——能转化为具体投资策略
- 强调"找到那个 2-3 个高杠杆点"，避免陷入无尽追问
- 配合 `shanren-investment-decision` 输出持仓建议

---

## 失败回退

如果某步无法推进：
- **Step 2 失败**：问题太模糊，要求用户澄清
- **Step 3 失败**：进入价值/偏好领域，标注"溯源到价值判断终止"
- **Step 4 失败**：进入哲学终极问题，标注"溯源到认知边界"
- **Step 5 失败**：问题过于复杂，输出简化版
- **Step 6 失败**：杠杆点不明确，建议"重新审视问题本身"

任何失败**不影响整体输出**，但要诚实标注。
