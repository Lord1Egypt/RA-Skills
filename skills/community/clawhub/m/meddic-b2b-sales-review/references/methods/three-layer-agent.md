# 三层 Agent 范式

## 用途
将任何岗位/角色拆解为"老板意志 → Boss Agent → 员工 Agent"三层结构，用于理解组织内 AI Agent 的定位、执行边界与汇报关系。在销售 review 中帮助判断：这个角色/项目处在哪一层，谁是真正的 Boss，核心动作流是什么。

---

## 核心模型

```
老板意志（战略意图/业务目标）
    ↓ 拆解下发
Boss Agent（战略拆解 / 异常识别 / 意志下发）
    ↓ 驱动执行
员工 Agent（承接执行 / 驱动一线 / 汇报结构化）
```

### 第一层：老板意志
- **定义**：组织最高层的战略意图与业务目标，通常不可直接执行
- **特征**：方向性、抽象性、结果导向
- **举例**：拿下某行业 TOP 10 客户、Q3 营收翻倍、建立标杆案例
- **关键问题**：这个意志有没有被清晰拆解成可执行的子目标？

### 第二层：Boss Agent
- **定义**：将老板意志拆解为可执行任务、识别异常并纠偏的中间层
- **核心动作**：
  - **战略拆解**：把大目标拆成阶段性里程碑和可执行动作
  - **异常识别**：监控执行过程中的偏差信号（进度、质量、方向）
  - **意志下发**：将拆解后的任务以明确指令传递给员工 Agent
- **关键能力**：判断力 > 执行力。能区分"该做"和"不该做"，能识别什么时候需要升级汇报
- **常见角色**：销售总监、区域经理、项目负责人

### 第三层：员工 Agent
- **定义**：承接 Boss Agent 下发的具体任务，在一线执行并结构化汇报
- **核心动作**：
  - **承接执行**：按指令完成具体动作（拜访、方案、谈判）
  - **驱动一线**：直接面对客户/用户，产生一手信息
  - **汇报结构化**：将执行结果按 Boss Agent 需要的格式汇报
- **关键能力**：执行力 > 判断力。能把确定性任务高质量完成
- **常见角色**：一线销售、客户成功、售前顾问

---

## 如何拆解任何角色

### 步骤一：定位层级
问三个问题：
1. 这个角色的 **直接产出** 是什么？（决策 vs 执行）
2. 这个角色 **向谁汇报**？汇报内容是什么？
3. 这个角色 **能调动多少资源**？（预算、人力、时间）

### 步骤二：识别 Boss
- 每个员工 Agent 都有且只有一个 Boss Agent
- Boss 不一定是组织架构上的上级，而是 **真正分配任务、评估结果的人**
- 如果一个角色有多个 Boss，说明职责边界不清晰

### 步骤三：定义核心动作流
```
输入（Boss 下发什么）→ 核心动作（做什么）→ 输出（汇报什么）→ 异常处理（什么情况升级）
```

---

## 在销售 Review 中的应用

| 场景 | 要判断的问题 | 对应层级 |
|------|-------------|---------|
| 项目推进卡住 | 卡在 Boss 层（没拆解清楚）还是员工层（执行不到位）？ | 定位瓶颈层 |
| 资源分配 | 谁是这个项目的真正 Boss？Boss 有没有在做异常识别？ | 确认责任人 |
| AI Agent 落地 | 这个 Agent 替代的是哪一层？替代 Boss 需要判断力，替代员工需要执行力 | 产品定位 |
| 销售赋能 | 一线销售是需要更好的执行工具（员工层），还是更好的判断辅助（Boss层）？ | 方案设计 |

---

## 关键原则

1. **层级不可跳过**：不能让员工 Agent 直接接老板意志，中间必须有拆解和判断
2. **Boss Agent 的核心价值是判断力**：如果只是传话，不是真正的 Boss
3. **一个角色可以同时是两层**：小团队里，中层管理者既是 Boss（对下）又是员工（对上）
4. **AI Agent 的定位决定产品设计**：替代员工层 → 工具型产品；替代 Boss 层 → 决策辅助型产品
5. **退化信号**：当 Boss Agent 只在做信息搬运而没有判断，它在退化为员工 Agent

---

## 讨论模板

在 review 中使用此框架时，按以下顺序讨论：

1. **这个角色/项目处在哪一层？**（定位）
2. **谁是它的 Boss？Boss 在做该做的事吗？**（责任）
3. **核心动作流有没有断点？**（瓶颈）
4. **如果要用 AI Agent，替代哪一层的什么动作？**（机会）
