# 设计说明 - 文档结构

设计说明（Design Description）的 .docx 文档应包含以下章节结构。内容基于 Figma 设计稿的实际信息 + 过往项目设计规范生成。

## 必选章节

### 1. 封面
- 项目名称
- 文档类型：「设计说明」
- Figma 设计稿链接（如有）
- 版本号与日期

### 2. 页面概述
- 页面名称与所属模块
- 页面核心功能描述
- 页面入口与跳转关系

### 3. 页面布局结构
- 整体布局描述（如「顶部导航 + 左侧筛选 + 右侧内容区」）
- 各区域功能说明

## 可选章节

### 4. 组件清单与说明
- 页面使用的关键组件列表
- 每个组件的：
  - 名称与用途
  - 交互状态（默认 / 悬停 / 点击 / 禁用 / 加载 / 空状态）
  - 数据来源

### 5. 交互逻辑
- 用户操作流程（按步骤描述）
- 表单验证规则
- 错误提示与边界情况
- 权限与可见性控制

### 6. 视觉规范
- 色彩应用（主色、辅色、功能色）
- 字体层级（标题 / 正文 / 辅助文字 / 字号 / 行高）
- 间距系统
- 圆角与阴影规范
- 图标规范

### 7. 响应式适配
- 断点定义
- 各断点下的布局变化

### 8. 设计变量引用
- 从 Figma 提取的 Design Tokens（如有）
- 颜色变量映射
- 间距/字体变量映射

## 撰写原则

- 所有描述指向设计稿中的具体画板/帧
- 交互说明覆盖完整生命周期（正常流程 + 异常状态）
- 视觉规范优先引用过往项目中的已有规范，确保一致性
- 使用中文撰写，术语保持行业规范
- 语言专业、简洁、有说服力
- 每个章节 200-500 字

## 示例结构

`
【封面】
项目名称：智能家居控制 APP - 设备列表页
文档类型：设计说明
Figma 链接：https://www.figma.com/file/xxxxx
版本：v1.0
日期：2026-05-27

【页面概述】
页面名称：设备列表页
所属模块：设备管理
核心功能：展示用户所有智能设备，支持快速控制和状态查看
页面入口：从首页点击"设备"Tab 进入

【页面布局结构】
整体布局：顶部搜索栏 + 设备分类 Tab + 设备卡片列表
- 顶部搜索栏：支持按设备名称搜索
- 设备分类 Tab：全部、客厅、卧室、厨房、浴室
- 设备卡片列表：瀑布流布局，每个卡片展示设备图标、名称、状态

【组件清单与说明】
1. 搜索框组件
   - 用途：快速查找设备
   - 状态：默认（灰色占位符）、聚焦（蓝色边框）、输入中（显示清除按钮）
   - 数据来源：用户输入

2. 设备卡片组件
   - 用途：展示单个设备信息和快速控制
   - 状态：
     - 默认：白色背景，显示设备图标、名称、状态
     - 离线：灰色背景，显示"离线"标签
     - 点击：跳转到设备详情页
   - 数据来源：设备 API

【交互逻辑】
1. 用户进入页面 → 加载设备列表 → 显示设备卡片
2. 用户点击搜索框 → 输入关键词 → 实时过滤设备列表
3. 用户点击分类 Tab → 切换到对应分类 → 显示该分类下的设备
4. 用户点击设备卡片 → 跳转到设备详情页
5. 用户长按设备卡片 → 显示快捷操作菜单（编辑、删除）

异常处理：
- 设备列表为空：显示空状态插图 + "暂无设备，点击添加"
- 网络错误：显示错误提示 + "重试"按钮
- 搜索无结果：显示"未找到相关设备"

【视觉规范】
色彩应用：
- 主色：#1890FF（品牌蓝，用于按钮、链接）
- 辅色：#52C41A（成功绿，用于在线状态）
- 功能色：#FF4D4F（错误红，用于离线状态）
- 背景色：#F5F5F5（页面背景）、#FFFFFF（卡片背景）

字体层级：
- 页面标题：PingFang SC Medium, 20px, 行高 28px
- 设备名称：PingFang SC Regular, 16px, 行高 24px
- 状态文字：PingFang SC Regular, 14px, 行高 22px
- 辅助文字：PingFang SC Regular, 12px, 行高 20px

间距系统：
- 页面边距：16px
- 卡片间距：12px
- 卡片内边距：16px
- 组件间距：8px

圆角与阴影：
- 卡片圆角：8px
- 按钮圆角：4px
- 卡片阴影：0 2px 8px rgba(0, 0, 0, 0.08)

【响应式适配】
断点定义：
- 移动端：< 768px
- 平板：768px - 1024px
- 桌面端：> 1024px

布局变化：
- 移动端：单列布局，卡片宽度 100%
- 平板：双列布局，卡片宽度 48%
- 桌面端：三列布局，卡片宽度 32%

【设计变量引用】
颜色变量：
- --primary-color: #1890FF
- --success-color: #52C41A
- --error-color: #FF4D4F
- --bg-color: #F5F5F5

间距变量：
- --spacing-xs: 4px
- --spacing-sm: 8px
- --spacing-md: 12px
- --spacing-lg: 16px
- --spacing-xl: 24px
`

## 注意事项

1. **基于实际设计稿**：所有描述必须基于实际的设计稿内容，不要臆造
2. **引用历史规范**：视觉规范优先引用历史项目中的已有规范
3. **完整性**：确保覆盖所有交互状态和异常情况
4. **可执行性**：开发人员应能根据设计说明直接实现
5. **一致性**：术语、格式、风格与历史项目保持一致
6. **精确性**：尺寸、颜色、字体等参数应精确到具体数值
