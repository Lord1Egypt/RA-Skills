Read references/principles.md before starting.

<process>
1. Read the current AGENTS.md content and any nearby AGENTS.md files that merge into scope.
2. Split each instruction into one of four buckets:
   - root-essential
   - topic-specific
   - conflicting
   - delete
3. If you find conflicts, stop and present the exact conflict to the user before rewriting anything.
4. If there are no blockers, draft a minimal root AGENTS.md structure and list the follow-up docs that should hold the removed guidance.
5. Call out redundant, vague, or stale lines explicitly so the user can decide whether to delete them.
</process>

<success_criteria>
This workflow is complete when the user has either:
- a clear audit of what to keep, move, and delete, or
- a concise proposed file structure for a rewrite.
</success_criteria>
