# Cybernetic Evolver — 工作流程详述

## 完整进化循环流程图

```
┌─────────────────────────────────────────────────────────────────┐
│                        启动初始化                                │
│  1. 定义目标 target                                             │
│  2. 初始化参数 θ、经验缓冲、探索率 ε                              │
│  3. 初始化环境                                                   │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                    STEP 1: 感知 (Perceive)                       │
│  输入: raw_state（环境原始状态）                                  │
│  输出: state（系统内部特征向量）                                   │
│  控制论对应: 系统辨识 + 信息处理（《工程控制论》第三章）            │
│  功能: 归一化 → 降维（如需）→ 特征提取                             │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                    STEP 2: 评价 (Evaluate)                       │
│  输入: current_performance（当前性能）                           │
│  输出: error = target - performance（误差信号）                   │
│  控制论对应: 第十八章"误差的控制"                                 │
│  功能: 实时量化性能，驱动后续决策和调整方向                         │
└─────────────────────────────────────────────────────────────────┘
                              │
          ┌───────────────────┴───────────────────┐
          ▼                                       ▼
  ┌───────────────────┐                 ┌───────────────────┐
  │ error < 阈值      │                 │ error ≥ 阈值      │
  │ → 系统稳定        │                 │ → 触发自适应     │
  │ → 可选: 降低探索率│                 │ → 准备参数/结构  │
  └───────────────────┘                 │   调整            │
          │                             └───────────────────┘
          ▼                                       │
┌─────────────────────────────────────────────────────────────────┐
│                    STEP 3: 决策 (Decide)                        │
│  输入: ε（探索率）、Q值表                                        │
│  输出: action（动作选择）                                         │
│  控制论对应: 第十五章"最优控制"（多种策略中选择）                  │
│  策略选择:                                                       │
│    - rand() < ε → explore()（探索新策略）                        │
│    - else → exploit()（利用已知最优）                            │
│  探索方法: ε-greedy / UCB / 玻尔兹曼                            │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                    STEP 4: 执行 (Act)                            │
│  输入: action（动作索引）                                         │
│  输出: next_state, reward, done                                   │
│  控制论对应: 第六章"随动系统"（输出跟踪目标）                      │
│  功能: 动作映射 → 环境交互 → 新状态获取                           │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                    STEP 5: 反馈 (Feedback)                      │
│  输入: (state, action, reward, next_state, done)                 │
│  输出: 记录经验至缓冲，更新在线学习模型                           │
│  控制论对应: 维纳反馈控制原理（闭环反馈）                          │
│  功能: 完成闭环；触发稳定性检查                                   │
└─────────────────────────────────────────────────────────────────┘
                              │
          ┌─────────────────┴─────────────────┐
          ▼                                   ▼
  ┌───────────────────┐               ┌───────────────────┐
  │ 经验缓冲写入     │               │ 在线学习模型更新  │
  │ buffer.append()  │               │ learner.update()  │
  └───────────────────┘               └───────────────────┘
          │                                   │
          └─────────────────┬─────────────────┘
                            ▼
┌─────────────────────────────────────────────────────────────────┐
│                 STEP 6: 稳定性检查 (Stability Check)           │
│  输入: 当前误差 error、参数向量 θ                                │
│  输出: True（通过）/ False（失稳，拒绝更新）                    │
│  控制论对应: Lyapunov稳定性判据 + Popov超稳定性（《工程控制论》第十七章）│
│  计算: V = error² + λ·||θ-θ*||²                                │
│  判断: ΔV = V(k+1) - V(k) < 0 → 稳定                           │
│  保护: 若失稳，回滚参数，触发保护模式                            │
└─────────────────────────────────────────────────────────────────┘
                              │
          ┌─────────────────┴─────────────────┐
          ▼                                   ▼
  ┌───────────────────┐               ┌───────────────────┐
  │ 稳定性检查通过   │               │ 稳定性检查失败   │
  │ → 继续进化       │               │ → 回滚参数        │
  │                   │               │ → 可选: 降低学习率│
  └───────────────────┘               └───────────────────┘
          │                                   │
          └─────────────────┬─────────────────┘
                            ▼
┌─────────────────────────────────────────────────────────────────┐
│              STEP 7: 自适应调整 (Adapt) — 参数层                 │
│  触发条件: error ≥ adaptation_threshold                         │
│  控制论对应: 第十七章"自适应控制"                               │
│  方法:                                                        │
│    - 梯度下降: θ ← θ - α·∇J(θ)                                 │
│    - 策略搜索: 在参数空间中进行采样优化                          │
│  目标: 减少误差 e，使性能向目标收敛                              │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│            STEP 8: 结构变异 (Mutate Structure) — 结构层           │
│  触发条件: 连续 adaptation_failures ≥ mutation_trigger 次        │
│  控制论对应: 1980修订版"大系统理论" + 协同学（哈肯）              │
│  变异类型:                                                      │
│    - add_node: 添加新状态节点                                    │
│    - add_action: 添加新动作选项                                  │
│    - rewire: 重连边（改变因果关系）                              │
│    - prune: 删除冗余节点                                        │
│  目标: 突破参数优化的局部最优，实现结构层面进化                    │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│              STEP 9: 探索率衰减 (Decay Exploration)               │
│  功能: ε ← ε * epsilon_decay                                     │
│  钱学森原理: 初期需要探索（高ε），随学习深入逐渐利用已知最优（低ε）│
│  平衡: 探索-利用平衡是动态的，随系统成熟度调整                      │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                 STEP 10: 循环或终止                               │
│  条件: step < max_steps 且 not done                             │
│  → 继续: state = next_state，回到 STEP 1                         │
│  → 终止: 输出最终参数、进化历史、最优策略                         │
└─────────────────────────────────────────────────────────────────┘
```

## 各步骤时间尺度

| 步骤 | 时间尺度 | 控制论对应 |
|------|---------|-----------|
| 感知 Perceive | 微秒级 | 系统辨识 |
| 评价 Evaluate | 微秒级 | 误差控制 |
| 决策 Decide | 微秒级 | 最优控制 |
| 执行 Act | 毫秒~秒级 | 随动系统 |
| 反馈 Feedback | 毫秒级 | 闭环控制 |
| 稳定性检查 | 微秒级 | Lyapunov |
| 自适应 Adapt | 毫秒~秒级 | 自适应控制 |
| 结构变异 Mutate | 秒~分钟级 | 自组织系统 |

## 异常处理流程

```
异常: 稳定性失稳（ΔV > 阈值）
  → 拒绝本次参数更新
  → 回滚到上一个稳定状态
  → 降低学习率（α ← α * 0.5）
  → 可选: 进入保护模式，暂停探索

异常: 连续多次自适应失败
  → 触发结构变异
  → 重置 adaptation_failures 计数器

异常: 环境剧烈变化（误差突变）
  → 立即进入自稳定模式
  → 紧急参数调整（大学习率）
  → 快速恢复稳定后再进入正常自适应

异常: 探索率降至过低（陷入局部最优）
  → 周期性提升探索率（ε ← min(ε * 1.1, ε_max)）
  → 强制结构变异
```
