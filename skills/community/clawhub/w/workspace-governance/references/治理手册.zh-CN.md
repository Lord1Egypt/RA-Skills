## 定位

本手册是 `SKILL.md` 的补充说明，不重复定义执行规则。  
执行语义以 `SKILL.md` 为准，本手册只回答三件事：如何接入、如何运维、如何排错。

## 真源规则

- 单一真源：`SKILL.md`
- 本文档只放补充信息（示例、FAQ、运维建议）
- 若与 `SKILL.md` 冲突，以 `SKILL.md` 为准

## 快速接入

1. 把 `SKILL.md` 放到 Agent 可读目录
2. 在项目根目录放 `SKILL_ADAPT.yaml`
3. 运行契约检查脚本：`bash scripts/check_skill_contract.sh`
4. 可选运行文档同步检查：`bash scripts/check_docs_sync.sh`

## 目录说明（实践导向）

- `SKILL.md`：治理规范真源
- `SKILL_ADAPT.yaml`：项目级策略参数
- `scripts/`：自动化校验脚本
- `tools/`：互操作能力矩阵与工具映射
- `references/`：非规范类说明文档

## Agent Profile 合并（新增）

当存在平台 profile（如 `tools/adapt-profiles/openclaw.yaml`）时，采用 base + overlay：

- base: `SKILL_ADAPT.yaml`
- overlay: 平台 profile

合并规则：

- `immutable_dirs`：并集（只增不减）
- `protected_files`：并集（只增不减）
- `destructive_guard`：仅允许更严格覆盖，禁止放宽

## 常见运维动作

### 1) 规范变更发布

先改 `SKILL.md`，再同步 `references/`，最后执行：

```bash
bash scripts/check_skill_contract.sh
bash scripts/check_docs_sync.sh
```

### 2) 非交互任务上线前检查

- 确认有 `Non-Interactive Safety Policy`
- 确认 `ask-user` 场景会 `blocked`
- 确认 `SKILL_ADAPT.yaml` 存在且可解析

### 3) 互操作依赖缺失排查

- 检查 `tools/interop-capabilities.yaml`
- 若用户意图依赖外部能力（如 git/github），但运行时不存在，返回 `blocked` 并提示缺失依赖

## FAQ（增量）

**Q1：为什么手册不再写完整规则？**  
为了避免 `SKILL.md` 与手册双份维护导致语义漂移。

**Q2：脚本校验失败先看哪里？**  
先看 `SKILL.md` 标题段落是否缺失，再看 `references/` 是否同步更新。

**Q3：中文用户只看哪里最有效？**  
先看 `README.zh-CN.md`，再看本文档；执行规则最终以 `SKILL.md` 为准。
