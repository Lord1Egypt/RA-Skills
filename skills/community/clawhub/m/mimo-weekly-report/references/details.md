# Weekly Report 详细参考

## 四种模板详细结构

### 1. 简洁版（适合站会/Slack/微信）
**适用场景：** 快速同步、团队内部、非正式汇报
**篇幅：** 200-400字
**结构：** 完成(3-5条) + 进行中(1-2条) + 卡点(0-1条) + 下周(2-3条)

### 2. 详细版（适合正式邮件/领导汇报）
**适用场景：** 周报邮件、月度review素材、跨部门同步
**篇幅：** 500-1000字
**结构：** 本周摘要 + 详细完成 + 数据分析 + 进行中 + 风险 + 下周计划 + 需要的支持

### 3. OKR版（适合OKR团队）
**适用场景：** OKR进度同步、季度中review
**篇幅：** 400-800字
**结构：** 每个O的KR进度 + 信心指数 + 风险 + 下周重点

### 4. 项目制版（适合多项目并行）
**适用场景：** PM/项目经理、同时负责多个项目
**篇幅：** 按项目数量灵活
**结构：** 每个项目独立一个section（进展/风险/下周）+ 整体概览

## 行业风格适配

### 互联网行业
- 风格：简洁、数据驱动、使用OKR
- 常用指标：DAU/MAU、转化率、留存率、NPS
- 关键词：迭代、灰度、AB测试、用户反馈、上线
- 示例：本周完成V2.3迭代，DAU提升12%，转化率从3.2%→3.8%

### 金融行业
- 风格：严谨、注重合规、强调风控
- 常用指标：AUM、不良率、合规检查通过率
- 关键词：风控、合规、审计、尽调、策略
- 示例：完成3笔信贷尽调，风控模型准确率提升至96.5%，合规检查零异常

### 制造业
- 风格：注重进度和良率、强调执行力
- 常用指标：良品率、产能利用率、交付准时率
- 关键词：产线、工艺、良品率、排产、供应商
- 示例：A产线良品率从94%→96.2%，完成B产品试产，交付准时率98.5%

### 教育行业
- 风格：注重教学效果、关注学生反馈
- 常用指标：完课率、满意度、通过率、续费率
- 关键词：课程、教案、学情、互动、辅导
- 示例：本周完成3个章节教案，学生完课率85%，满意度4.6/5.0

## 量化表达转化表

| 模糊描述 | 量化表达 |
|----------|----------|
| 完成了X个功能 | 完成X个功能，覆盖Y%需求，提前Z天交付 |
| 优化了性能 | 响应时间从Xms降至Yms，提升Z%，超过目标值 |
| 修复了bug | 修复X个bug（P0:1个，P1:3个），线上故障率降低Y% |
| 做了用户调研 | 完成X份用户访谈，提炼Y个痛点，输出Z页报告 |
| 写了文档 | 产出X篇技术文档，覆盖Y个核心模块，团队阅读率Z% |
| 开了很多会 | 参与X场评审/对齐，推动Y个决策落地，节省Z天等待 |
| 带新人 | 完成X次code review，新人独立承担Y个需求，效率提升Z% |
| 做了数据分析 | 分析X万条数据，发现Y个关键洞察，推动Z个策略调整 |

## 不同角色的周报侧重

### 工程师
- 重点：代码产出、技术难点攻克、代码质量、技术分享
- 量化指标：提交数、PR数、bug率、测试覆盖率、文档数
- 避免：只写"写代码"，要写具体功能和效果

### 产品经理
- 重点：需求推进、用户反馈、数据分析、竞品动态
- 量化指标：需求完成率、用户满意度、功能使用率、AB测试结果
- 避免：只写"写PRD"，要写推动了什么决策

### 设计师
- 重点：设计产出、设计评审、用户体验提升
- 量化指标：页面数、设计稿版本数、用户测试反馈、设计系统组件数
- 避免：只写"画图"，要写设计解决了什么问题

### 运营
- 重点：活动效果、用户增长、内容产出、渠道数据
- 量化指标：转化率、ROI、新增用户、内容阅读量、活动参与率
- 避免：只写"做活动"，要写活动带来了什么结果

### 管理者
- 重点：团队进展、资源协调、风险管理、战略推进
- 量化指标：团队产出、项目健康度、人员成长、跨部门协作
- 避免：只写"开会管理"，要写推动了什么关键决策

## 周报常见问题与解决方案

**问题1: 觉得这周没干什么，没啥可写**
- 解法：关注过程指标（沟通/评审/调研/学习）
- 技巧：回顾日历和聊天记录，找出被忽略的工作
- 底线：即使真的空闲，写学习和准备也是内容

**问题2: 工作太碎太杂，不知道怎么组织**
- 解法：按项目归类，提炼共性
- 技巧：用"支持了X个需求"统一归类碎片工作
- 模板：70%主要项目 + 20%支持性工作 + 10%学习成长

**问题3: 报喜不报忧，卡点不敢写**
- 解法：卡点也是信息，主动暴露
- 技巧：卡点+你的尝试+需要的支持（显示你在积极解决）
- 格式：⚠️ {问题}，已尝试{方案}，需要{支持}

**问题4: 每周内容差不多，感觉在凑字数**
- 解法：加入趋势对比（本周vs上周vs目标）
- 技巧：用"连续X周保持..."或"本周首次突破..."
- 进阶：每周加一个"本周洞察/反思"section

**问题5: 给不同领导看的周报不知道怎么写**
- 解法：同一内容不同详略
- 直属领导：详细版，包含具体数据和技术细节
- 高层领导：简洁版，只关注结果和趋势
- 跨部门：项目制版，突出接口和依赖

## 周报质量自检清单

1. ✅ 成果是否有数据支撑（不是"完成了"而是"完成了X，提升了Y%"）
2. ✅ 是否有重点（最多5条核心成果）
3. ✅ 进行中事项是否有预计完成时间
4. ✅ 卡点是否说明了需要的具体支持
5. ✅ 下周计划是否具体可执行（不是"继续推进"）
6. ✅ 篇幅是否合适（简洁版不超1页，详细版不超2页）
7. ✅ 是否有上周计划的闭环（上周说要做的，本周有交代）
8. ✅ 语气是否积极正向（即使有卡点也在寻找解决方案）

## 时间范围识别

| 用户输入 | 识别为 |
|----------|--------|
| 本周/这周 | 本周一至今 |
| 上周 | 上周一至上周日 |
| 最近两周 | 14天内 |

| 未说明 | 默认本周 |
