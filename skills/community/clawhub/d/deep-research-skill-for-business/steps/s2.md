# S2: 搜索前反思校验（断点强制检查）

## 目标

在第一轮搜索前，反思搜索策略是否合理。这是 V3 最关键的"断点"——一旦跳过，后续搜索可能跑偏而无法发现。

## ⛔ 红线

- **必须输出反思校验表**，缺一项则不可进入 s3
- 如果发现关键词/方向有问题，立即修正并重新输出

## 操作

### 1. 输出反思校验表

```markdown
## S2: 搜索前反思校验

### 关键词检查
| 子问题 | 关键词组 A | 关键词组 B | 是否足够多元？ | 是否中英双语？ |
|--------|-----------|-----------|---------------|---------------|
| 1 | ... | ... | ✅/❌ | ✅/❌ |
| 2 | ... | ... | ✅/❌ | ✅/❌ |
| ... | ... | ... | ✅/❌ | ✅/❌ |

### 覆盖面检查
- [ ] 是否覆盖了不同信源类型（官方/学术/媒体/行业报告）？
- [ ] 是否考虑了不同立场/利益相关方的视角？
- [ ] 是否有明显遗漏的角度？

### 潜在偏差检查
- [ ] 关键词是否隐含了先入为主的假设？（如"AI必然提升效率"）
- [ ] 是否设计了"反向关键词"来验证对立观点？（如"AI落地失败案例"）

### 修正（如有）
- 原关键词 X → 修正为 Y，原因：...
- 新增关键词 Z，原因：...
```

### 2. 修正关键词（如需要）

如果检查发现问题，修正后再输出一次确认表。

### 3. 标记完成

完成后执行：
```bash
bash orchestrator.sh --complete s2 --next s3
```

## 校验标准

- [ ] 每个子问题至少 2 组关键词
- [ ] 关键词多元且中英双语
- [ ] 覆盖了不同信源类型
- [ ] 检查了潜在偏差
- [ ] 如有修正，已明确记录
