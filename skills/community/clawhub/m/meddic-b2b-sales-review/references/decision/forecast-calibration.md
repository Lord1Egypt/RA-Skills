# Forecast 校准参考

## 用途

用于判断当前机会应放在 Commit / Best Case / Pipeline 哪一档。

## 基本规则

### Commit
- EB 基本可触达
- Champion 真实
- 决策流程较清晰
- 客户承诺明确

### Best Case
- 方向正确
- 仍缺 1-2 个关键证据
- 存在推进机会，但不够硬

### Pipeline
- 还在探索
- 关键证据不足
- 不能提前美化

## 管理原则

- 缺硬证据时，不允许上报高预测
- 如果关键短板两周内补不齐，应考虑下调 forecast
- 机会质量低于门槛时，优先止损而不是继续堆资源

## Forecast 校准检查

### 从 MEDDIC 角度校准
- EB 可触达 → Commit 基础
- Champion 真实 → Commit 基础
- 痛点紧急 + Metrics 量化 → Best Case 基础
- 决策流程清晰 → Commit 基础
- 以上都缺 → Pipeline

### 从客户行为校准
- 客户有明确下一步承诺 → 支持 Commit
- 客户有验证行为 → 支持 Best Case
- 客户只有口头兴趣 → Pipeline

## 常见 forecast 虚高原因

- 销售把"客户有兴趣"当成"客户要买"
- 销售把"开始谈价格"当成"接近成交"
- 销售把"做了演示"当成"验证通过"
- 销售把"客户说汇报领导"当成"在推进"
- 管理者没有硬证据就接受销售的乐观判断
