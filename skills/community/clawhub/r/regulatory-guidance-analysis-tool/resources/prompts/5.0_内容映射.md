# 5.0 分析数据 → HTML 内容映射

读取 5 个独立分析文件（`{原始文档stem}_4.1_元素采集.md` ~ `{原始文档stem}_4.5_红黄绿蓝灯.md`），逐文件映射为 slide。**不得删减实质性数据**，只压缩冗余措辞。

## 映射表

| 分析文件 | 最少页数 | 映射规则 |
|----------|:------:|------|
| `{stem}_4.1_元素采集.md` | 1 | 表中每一行都必须在 slide 中出现。如内容过多，允许 2 页 |
| `{stem}_4.2_定性.md` | 1 | 为什么/适用范围/适用对象三栏，每栏至少保留完整句子 |
| `{stem}_4.3_重点内容.md` | 不限，全量输出 | 每个领域必须保留：领域名+条款范围+高风险原因+管控方式，至少各一句完整话 |
| `{stem}_4.4_生命周期图.md` | 1~2 | 所有阶段必须出现，每个阶段至少一行说明 |
| `{stem}_4.5_红黄绿蓝灯.md` | 不限，全量输出 | ① 统计 overview（4 个数字+含义+特征总结）1 页 ② 红灯条款按条款原始顺序平铺，每页至多 8 条，优先填满 ③ 黄灯/绿灯/蓝灯全部列出 |

## 不得删减项

- 分析报告 MECE 表中每一行的核心信息不得丢弃
- 红黄绿蓝灯统计中 4 个数字及含义必须全部出现
- 红灯条款必须全部列出（不得只取示例），内容多时按每页至多 8 条拆分
- 绿灯、蓝灯、黄灯必须全部列出（具体数量以 `{stem}_4.5_红黄绿蓝灯.md` 为准）
- 4.3 重点内容中每个领域的条款范围+高风险原因+管控方式必须保留，不得缩为一句

## 分页规则

### 4.1 元素采集
- 默认 1 页，全表罗列。行数 > 10 时可拆为 2 页，每页保留完整表头。

### 4.2 定性
- 固定 1 页。

### 4.3 重点内容（不限页数，全量输出）

**领域划分**：以分析文件中 `## N. 领域名` 标题为准，每个标题区域为一个领域。若无 `##` 标题，以表格中"领域"列每一行为一个领域。

**分配规则：**

| 条件 | 页数 |
|------|:--:|
| 每个领域（默认） | 独占 1 页 |
| 两个条件 **同时** 满足时，至多 2 个领域可合并为 1 页：(a) 每个领域表格行数 ≤ 3（不含表头）；(b) 每个领域表格外说明文字 ≤ 2 句 | 2 领域/页 |
| 单领域表格行数 > 8 | 按 **每页 5~8 行** 拆为多页 |

每页必须保留：领域名、条款范围、高风险原因、管控方式，至少各一句完整话。**不按领域类型分组**，一律按上表逐领域分配。

### 4.4 生命周期图
- 1~2 页，所有阶段必须出现，每个阶段至少一行说明。

### 4.5 红黄绿蓝灯（不限页数，全量输出）

**默认布局：三列网格微卡**（Layout B）。每条红灯/黄灯/绿灯使用独立的 `.hand-box` 卡片，按 3 列 `flex-wrap` 网格排列。

**每张卡片的 HTML 结构（强制）：**

```html
<div class="hand-box" style="flex:0 0 calc(33% - 8px);padding:8px 10px">
  <span class="num-circle" style="width:28px;height:28px;font-size:13px;line-height:28px;margin-bottom:4px">条文号</span>
  <p style="font-size:13px;color:var(--text-2);margin:0;line-height:1.5">红灯内容摘要</p>
</div>
```

**网格容器（强制）：**

```html
<div style="display:flex;flex-wrap:wrap;gap:12px;margin-top:18px">
  <!-- .hand-box 卡片在此 -->
</div>
```

**布局规则：**
- 卡片为白底 `.hand-box`（不外加 `background`、不外加 `border-left`）
- `.num-circle` 固定 28px，条文号用原文条款号（如 `1`、`14a`、`27c`）
- 红灯卡片内如混有黄灯/绿灯关键词，在 `<p>` 末尾用 `<span style="color:var(--warn);font-weight:700"> 🟡</span>` 或对应色标记
- 黄灯/绿灯的 `.num-circle` 保持默认配色（不额外设 `background:var(--good)`），靠外层 `.hand-box` 的底色区分灯色

**分页及排序：**
- 第 1 页固定为统计 overview：4 色 KPI 数字卡（横排 flex）+ 特征总结。
- 红灯按章节分组，每页 9~12 张卡片，每章若条件允许合并于同一页。
- 红灯共 9 页（页码标注 `红灯 N/9`）。
- 黄灯 ≤ 4 条独占 1 页，绿灯 7 条独占 1 页（可与黄灯合并页以省空间）。
- 全文条款全量列出，不得遗漏任何一条。
- 每页顶部标注 `4.5 红黄绿蓝灯 — 红灯 1/9`。

#### pptx-model 专用布局

**默认布局：`.ts-grid-3` 三列卡片网格**。每条红灯/黄灯/绿灯使用独立的 `.ts-card`，按 `.ts-grid-3` 排列。

**每张卡片的 HTML 结构（强制）：**

```html
<div class="ts-card" style="padding:10px 12px">
  <b style="font-size:14px;color:var(--bad);display:block">条文号</b>
  <p style="font-size:13px;color:var(--text-2);margin:4px 0 0;line-height:1.5">红灯内容摘要</p>
</div>
```

**骨架（每页必须）：**
```html
<section class="slide">
  <div class="ts-stripe"></div>
  <div class="ts-chrome">
    <span class="ts-alert-tag">红灯 N/9</span><span class="ts-page">NN / TT</span>
  </div>
  <p class="ts-kicker" style="margin-top:6px;color:var(--bad)">章节名（条款范围）</p>
  <h2 class="ts-h2 mt-s">4.5 红黄绿蓝灯 — 红灯 N/9</h2>
  <div class="ts-grid-3" style="gap:10px">
    <!-- .ts-card 卡片在此 -->
  </div>
  <div class="ts-stripe-b"></div>
  <div class="ts-footer"><span>文档名</span><span>NN / TT</span></div>
</section>
```

**布局规则：**
- 红灯 `.ts-card` 内条文号 `<b>` 用 `color:var(--bad)`，黄灯用 `color:var(--warn)`，绿灯用 `color:var(--good)`
- 红灯卡片内如混有黄灯/绿灯关键词，在 `<p>` 末尾标注 `🟡` / `🟢`
- 赤红之间可混一个特殊处理的绿灯卡片，使用 `class="ts-card"` + `border-top-color:var(--good)`
- 统计 overview 页用 `.ts-grid-4` + `.ts-card` 展示 4 色 KPI
