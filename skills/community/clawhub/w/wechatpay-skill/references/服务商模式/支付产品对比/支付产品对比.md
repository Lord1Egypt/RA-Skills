# 服务商模式 - 支付产品对比

> 服务商模式下各支付方式与商户模式的区别

## 与商户模式的核心区别

| 对比项 | 商户模式 | 服务商模式 |
|--------|----------|------------|
| 商户号 | 自己申请的商户号 | 帮助子商户接入 |
| 子商户 | 无 | 有sub_mchid |
| 分账 | 自己收款 | 可能需要分账给子商户 |
| 进件 | 无 | 支持子商户进件 |

## 服务商特有的支付方式扩展

### 子商户支付

服务商模式下，所有支付方式都需要传子商户参数：

| 参数 | 说明 |
|------|------|
| sp_mchid | 服务商商户号 |
| sub_mchid | 子商户商户号 |
| sub_appid | 子商户AppID |
| sub_openid | 子用户openid |

## 子商户openid获取

### 服务商模式获取子商户openid

```python
# 服务商模式下，需要指定子商户AppID获取openid
def get_sub_openid(code, sub_appid, sub_secret):
    """获取子商户用户的openid"""
    url = "https://api.weixin.qq.com/sns/jscode2session"
    params = {
        "appid": sub_appid,        # 子商户AppID
        "secret": sub_secret,      # 子商户AppSecret
        "js_code": code,
        "grant_type": "authorization_code"
    }
    resp = requests.get(url, params=params)
    return resp.json().get("openid")
```

## 服务商模式下单示例

```python
# 服务商JSAPI下单
body = {
    "appid": sp_appid,           # 服务商AppID
    "mchid": sp_mchid,           # 服务商商户号
    "description": "商品描述",
    "out_trade_no": out_trade_no,
    "notify_url": notify_url,
    "amount": {
        "total": total_fee,
        "currency": "CNY"
    },
    "payer": {
        "sp_openid": sp_openid,   # 服务商用户openid
        # 或
        "sub_openid": sub_openid  # 子商户用户openid
    },
    "sub_mchid": sub_mchid,      # 子商户商户号
    "sub_appid": sub_appid       # 子商户AppID（可选）
}
```

## 合单支付（服务商）

服务商合单支付支持更多子单（最多50个）：

```python
body = {
    "combine_appid": sp_appid,
    "combine_mchid": sp_mchid,
    "combine_out_trade_no": combine_out_trade_no,
    "sub_orders": [
        {
            "mchid": sub_mchid1,
            "out_trade_no": sub_order_no1,
            "amount": {...}
        },
        {
            "mchid": sub_mchid2,
            "out_trade_no": sub_order_no2,
            "amount": {...}
        }
    ]
}
```

## 注意事项

1. **子商户必须先进件**：服务商需要先帮子商户完成进件
2. **子商户必须绑定**：子商户AppID和mchid必须已绑定
3. **权限申请**：服务商需要申请相关接口权限
