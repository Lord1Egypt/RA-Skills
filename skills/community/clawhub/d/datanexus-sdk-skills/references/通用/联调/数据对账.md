# 数据对账

## 概述

数据对账用于验证 SDK 上报的数据是否准确完整到达 DataNexus 平台。建议在联调通过后、正式上线前进行数据对账。

## 一、DataNexus 日志查询

### 操作步骤

1. 使用初始化 SDK 时的数据源 ID 所属账号登录 [DataNexus 平台](https://datanexus.qq.com)
2. 进入 **日志查询** 页面 → 选择 **「用户」** 子菜单
3. 在右侧输入 **微信小程序/小游戏 AppID** 或 **数据源 ID**
4. 查看实时上报的原始数据

### 对账维度

逐条对比以下字段：

| 对账项 | 预期 | 核实方式 |
|--------|------|---------|
| 行为类型（action_type） | 与实际触发行为一致 | 比对上报的 action_type 值 |
| 行为参数（action_param） | 金额、自定义参数完整 | 比对 value 等字段值 |
| 行为时间（action_time） | 与实际发生时间一致 | 比对时间戳 |
| 用户标识 | OpenId/UnionId 正确 | 比对用户 ID 字段 |
| 数据条数 | 与客户端触发次数一致 | 统计日志查询结果条数 |

## 二、SDK 数据质量看板

### 入口

DataNexus 平台 → 工具 → [质量看板](https://datanexus.qq.com/web/tool/quality-panel)

### 检查内容

1. **基础信息**：SDK 版本、AppID、数据源 ID 是否正确
2. **必报字段校验**：START_APP、REGISTER、LOGIN、CREATE_ROLE、PURCHASE 是否完整上报
3. **数据量趋势**：上报数据量是否符合预期
4. **异常检测**：是否存在数据断崖、突增等异常

### IAA 小游戏专项

- 质量看板的「基础信息」区域可查看「必报字段校验」结果
- 支持【复审】提交功能，质量不达标时可修复后提交复审

## 三、常见对账问题

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 日志查询无数据 | SDK 未正确初始化或域名未配置 | 检查初始化参数和安全域名 |
| 数据条数不匹配 | 开启了自动采集导致重复上报 | 开启自动采集后不再手动上报 START_APP |
| action_type 不对 | 使用了错误的行为类型 | 参照[行为类型枚举表](../埋点/行为类型枚举表.md) |
| 金额为0 | value 字段未传 | 在 action_param 中添加 value 参数 |
| 时间偏差大 | action_time 传了错误值 | 使用服务器当前时间戳 |

## 四、对账流程建议

```
1. 开发阶段 → 开启 SDK 调试模式 → 查看事件触发日志
2. 联调阶段 → 通过 DataNexus 日志查询实时验证
3. 上线前   → 通过质量看板做全面检查
4. 上线后   → 持续监控质量看板数据量趋势
```
