# Git差异提取器 (GitDiffSkill)

基于.md描述的标准skill，使用大模型进行Git差异分析和变更提取。

## Skill定义

### 基本信息
- **名称**: Git差异提取器
- **描述**: 提取Git仓库中两个分支之间的差异，识别变更的代码文件及其具体的变更行号
- **风险类型**: Git Analysis
- **支持语言**: 所有语言

### 检测规则

#### 差异提取规则
1. **分支存在性检查**: 验证基础分支和目标分支是否存在
2. **文件类型过滤**: 仅提取代码文件，忽略二进制文件和文档
3. **变更类型识别**: 识别新增、修改、删除的文件
4. **行号映射**: 精确提取每个文件的变更行号
5. **差异范围**: 支持两点式/三点式差异对比

#### 文件过滤规则
1. **包含文件**: 源代码文件 (*.java, *.js, *.ts, *.py, *.cpp, *.c, *.go, *.rb, *.php)
2. **排除文件**: 测试文件、配置文件、构建产物、文档文件

### 输出格式定义
- **diff_files**: 变更的文件列表
- **change_lines_map**: 每个文件的变更行号映射
- **base_branch**: 基础分支
- **target_branch**: 目标分支
- **change_summary**: 变更摘要

## 分析方法

### 分析步骤
1. **仓库验证**: 验证Git仓库的有效性
2. **分支解析**: 获取基础分支和目标分支的commit对象
3. **差异计算**: 计算两个分支之间的文件差异
4. **文件过滤**: 根据规则过滤相关代码文件
5. **行号提取**: 提取每个文件的变更行号
6. **结果格式化**: 将结果格式化为标准JSON

### 输出格式
```json
{
  "diff_files": ["文件路径列表"],
  "change_lines_map": {
    "文件路径": [变更行号数组]
  },
  "base_branch": "基础分支名称",
  "target_branch": "目标分支名称",
  "change_summary": {
    "total_files": 总文件数,
    "added_files": 新增文件数,
    "modified_files": 修改文件数,
    "deleted_files": 删除文件数
  }
}
```

## 使用说明

### 触发条件
- 当需要分析Git分支间差异时触发
- 当需要获取代码变更信息时触发
- 当需要为其他skill提供变更数据时触发

### 配置参数
- **基础分支**: 默认 "origin/master"
- **目标分支**: 需要指定的目标分支
- **文件过滤**: 可配置的包含/排除规则
- **差异类型**: 两点式/三点式差异

### 集成方式
此skill通过RiskForge CLI框架自动加载，执行分析时会使用大模型进行智能分析。

## 注意事项
1. 确保Git工作区干净，避免未提交文件干扰
2. 分支名称支持特殊字符和复杂命名
3. 对于大型仓库，分析可能需要较长时间
4. 结果依赖于Git仓库的完整性
