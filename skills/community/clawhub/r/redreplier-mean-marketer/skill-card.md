## Description: <br>
A background accountability loop that surfaces high-scoring RedReplier lead opportunities and relays server-generated accountability messages when prior leads have not been approved or rejected. <br>

This skill is ready for commercial/non-commercial use. <br>

## Publisher: <br>
[tarasshyn](https://clawhub.ai/user/tarasshyn) <br>

## Use Case: <br>
External users and developers use this skill to run scheduled RedReplier lead checks, receive concise opportunity notifications, and triage leads through approve or reject commands. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: The skill is intended to run on a recurring schedule and may continue sending accountability messages until stopped. <br>
Mitigation: Confirm the 5-minute schedule is acceptable, use the server configuration to pause the loop, or stop the scheduled job when automatic nudges are no longer wanted. <br>
Risk: The skill uses a RedReplier API key for API access. <br>
Mitigation: Use a dedicated RedReplier token and keep it limited to the intended host environment. <br>
Risk: Each polling run can consume a lead and advance server-side notification state. <br>
Mitigation: Deliver any non-empty notify output to the user and avoid polling only to inspect output. <br>
Risk: The skill can relay mean or profanity-enabled accountability copy generated by RedReplier. <br>
Mitigation: Use the profanity configuration setting to keep messages swear-free and limit delivery to channels where the user expects this tone. <br>
