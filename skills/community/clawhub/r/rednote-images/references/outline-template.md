# RedNote Outline Template

Use `outline.md` to decide the card sequence before writing final prompts.

## Recommended Structure

```markdown
# RedNote Outline

**Theme**: <main topic>
**Audience**: <target audience>
**Style**: <style>
**Layout**: <layout>
**Language**: <zh|en|ja|ko>
**Aspect**: <3:4|1:1|9:16>
**Card Count**: <number>
**Series Reference**: references/series-reference.png
**Series Anchors**:
- <overall palette / brand colors>
- <repeating subject / mascot / product / presenter>
- <shared typography, framing, and spacing rules>

## Card 1
**Role**: cover
**Filename**: 01-cover.md
**Goal**: Hook the viewer with the main promise or thesis
**Visual Focus**: <main visual focus>
**Continuity Anchors**:
- <what must stay consistent from previous cards: palette, mascot, outfit, prop, typography, layout rhythm>
**Key Points**:
- <point 1>
- <point 2>

## Card 2
**Role**: content
**Filename**: 02-content-problem.md
**Goal**: Introduce the first key idea
**Visual Focus**: <main visual focus>
**Key Points**:
- <point 1>
- <point 2>
- <point 3>
```

## Rules

- Card 1 should almost always be the cover card.
- Keep each card focused on one main takeaway.
- `Filename` should refer to the prompt filename, not the final image path.
- `Goal` should describe what the card is doing in the series.
- `Visual Focus` should describe the dominant visual idea, not just the topic.
- `Series Reference` should point to the shared visual reference image for the whole set.
- Use `Series Anchors` for global consistency, and `Continuity Anchors` for card-to-card continuity.

## Recommended Card Rhythm

- `01-cover` -> hook and positioning
- `02-content` -> first key idea
- `03-content` -> second key idea
- `04-content` -> comparison, checklist, or example
- `05-ending` -> closing takeaway or CTA
