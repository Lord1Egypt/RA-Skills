# IP资产盘点模板

## 用途
提供6列标准化矩阵模板，帮助企业系统盘点所有知识产权资产，识别保护缺口和行动优先级。

## 使用场景
- 初创企业/科技公司首轮融资前IP尽调
- 并购交易中的目标公司IP资产估值与风险排查
- 企业年度知识产权审计
- IPO前知识产权合规自查（科创板/创业板硬门槛）

### 确权状态判断标准

| 状态 | 含义 | 典型问题 |
|------|------|---------|
| ✅ 已确权 | 已获得授权证书/登记证书 | 注意年费缴纳状态、权利稳定性（是否有无效风险） |
| ⏳ 审查中 | 已申请未授权 | 关注审查意见答复期限、授权前景预判 |
| ❌ 未申请 | 存在可保护客体但未启动申请 | 评估是否已过申请窗口（新颖性丧失）、成本收益比 |
| ⚠️ 不完整 | 权利有但保护范围小于应有 | 商标核心类别未注册、专利权利要求过窄 |
| 📄 协议依赖 | 权利来自许可合同/委托开发合同 | 审查许可期限、范围限制、终止条款、改进成果归属 |

### 风险等级判定

| 等级 | 标准 |
|------|------|
| 🔴 高 | 核心IP未确权/即将到期/存在权属争议/已被起诉侵权 |
| 🟡 中 | IP已确权但保护范围不足/年费即将逾期/发现潜在侵权行为 |
| 🟢 低 | IP确权完整且保护期限>5年/无已知争议 |

### 常见行动建议速查

| 发现的问题 | 建议行动 |
|-----------|---------|
| 核心技术未申请专利 | 评估是否以商业秘密保护更优；若需公开使用则立即申请（注意新颖性宽限期） |
| 商标核心类别未注册 | 尽快提交注册申请（防止恶意抢注）；同步做市场检索排除侵权 |
| 员工离职带走代码/客户 | 检查竞业限制协议是否有效（是否支付补偿金）；完善代码管理+访问权限 |
| 开源代码合规风险 | 进行开源合规扫描（工具：FOSSology/BlackDuck）；建立开源使用审批流程 |
| 委托开发成果归属不明 | 梳理历史合同；优先通过补充协议确权到公司名下 |
| IP许可即将到期 | 评估续约必要性；核心许可建议签订"永久许可+合理条件" |
| 域名即将到期 | 立即续费（设置自动续费）；检查注册邮箱是否仍可访问 |

## 使用说明

1. **全员参与**：IP盘点不是法务一个部门的事——需研发（专利/代码）、市场（商标/域名）、HR（竞业/成果归属）协同
2. **优先级排序**：先盘核心IP（直接影响营收的），再盘边缘IP
3. **定期更新**：每半年更新一次矩阵，特别是专利申请状态和商标续展日期
4. **文档留痕**：盘点过程中发现的权属瑕疵，需书面记录处理过程和结果
5. **外部支撑**：技术类IP（特别是软件专利/AI算法）建议委托专利代理机构做专业检索和稳定性分析
