### 3.2 百科/资料采集（虚构角色 & 历史人物）

#### 虚构角色采集流程

**Step 1: 百科信息采集**

```bash
# 萌娘百科（动漫角色首选，内容最详细）
curl -sL "https://zh.moegirl.org.cn/{角色名}" -H "User-Agent: Mozilla/5.0" | \
  python3 -c "import sys,re; h=sys.stdin.read(); t=re.sub(r'<[^>]+>',' ',re.sub(r'<(script|style)[^>]*>.*?</\\1>','',h,flags=re.DOTALL)); print(re.sub(r'\s+',' ',t)[:15000])"

# 百度百科
curl -sL "https://baike.baidu.com/item/{角色名}" -H "User-Agent: Mozilla/5.0"

# Fandom Wiki（海外作品）
curl -sL "https://{作品名}.fandom.com/zh/wiki/{角色名}" -H "User-Agent: Mozilla/5.0"
```

**需要提取的关键信息**：

| 维度 | 说明 | 在百科中的位置 |
|------|------|--------------|
| 基本设定 | 年龄/身份/种族/能力 | 信息框/基本资料 |
| 性格描述 | 百科的"性格"章节 | 通常在"角色介绍"下 |
| ⭐ 经典台词 | **最重要** — 说话风格的唯一来源 | "经典语录"/"台词"章节 |
| ⭐ 口头禅 | 标志性用语 | 散落在角色描述中 |
| 人际关系 | 跟谁关系好/不好/暧昧 | "人际关系"章节或角色关系图 |
| 关键剧情 | 影响性格形成的转折事件 | "剧情"章节 |
| 角色评价 | 侧面了解角色特点 | "角色评价"/"人气投票" |

**Step 2: 台词/语录采集**

如果百科中台词不足（< 10 条），额外搜索：

```bash
# 搜索台词库
搜索: "{角色名}" 经典台词 语录
搜索: "{作品名}" "{角色名}" 名场面 台词
搜索: "{角色名}" 口头禅

# 常见台词来源
# - 百度知道/知乎 "XX有哪些经典台词"
# - 哔哩哔哩专栏 (bilibili.com/read)
# - 各种语录整理帖
```

> ⚠️ 采集到的台词需要**标注出处（哪一集/哪个场景）**，用于理解台词的语境。

**Step 2.5: B站原作视频字幕提取（动漫/影视角色专属）**

百科台词通常只有十几条精华，而B站上的原作动画/电视剧有**完整的AI字幕**——几十集的全部对话内容。这是虚构角色说话风格分析的**最佳数据源**。

**适用场景**：
- 动漫角色 → B站上有对应动画（番剧/国创）
- 影视角色 → B站上有对应电视剧/电影片段
- 游戏角色 → B站上有游戏CG/实况视频

**采集流程**：

```
1. 搜索原作视频
   B站搜索API: keyword="{作品名}" + search_type=video
   或直接搜: "{作品名}" + "{角色名}"

2. 筛选目标集数
   - 动漫: 优先选角色戏份多的集数（名场面/角色主线集）
   - 影视: 优先选角色的高光片段/经典场景
   - 不需要全部集数，选 5-10 集角色重点集即可

3. 提取字幕
   走 Section 3.3 的 B站字幕提取标准流程:
   view API → player/v2 → 字幕JSON

4. 筛选角色台词
   ⚠️ 关键步骤: B站字幕是全部角色的对话，需要筛选出目标角色的台词
   方法:
   - 结合百科已知的口头禅/称呼习惯进行初步过滤
   - 利用 LLM 对话分析: 把字幕文本交给 AI，要求识别并提取特定角色的台词
   - 如有弹幕/评论提到"XX名台词在x:xx"，辅助定位

5. 口语风格分析
   用 scripts/speech_analysis.py 分析筛选后的角色台词
```

**B站搜索番剧/影视的技巧**：

```bash
# 搜索番剧
curl -s "https://api.bilibili.com/x/web-interface/search/type?\
keyword={作品名}&search_type=media_bangumi&page=1" \
  -H "User-Agent: Mozilla/5.0" -H "Referer: https://www.bilibili.com/"
# 返回 season_id, media_id 等

# 搜索影视剧
curl -s "https://api.bilibili.com/x/web-interface/search/type?\
keyword={作品名}&search_type=media_ft&page=1" \
  -H "User-Agent: Mozilla/5.0" -H "Referer: https://www.bilibili.com/"

# 搜索角色名场面/台词合集（用户投稿，通常有完整字幕）
curl -s "https://api.bilibili.com/x/web-interface/search/type?\
keyword={角色名}+名场面+台词&search_type=video&order=click&page=1" \
  -H "User-Agent: Mozilla/5.0" -H "Referer: https://www.bilibili.com/"
```

**⚠️ 番剧字幕 vs 用户投稿字幕**：
- **番剧/正版影视**（`bangumi.bilibili.com`）：字幕通常是官方翻译，质量最高。但 player/v2 API 对番剧可能返回不同格式的字幕URL。
- **用户投稿**（如"XX全集台词合集"、"XX名场面"）：用标准的 view → player/v2 → 字幕JSON 流程，最容易获取。
- **建议**：优先搜索用户投稿的"台词合集"/"名场面"视频，这些通常已经帮你筛选好了角色台词。

**台词角色识别（LLM辅助）**：

字幕中无法直接区分说话人。用以下 prompt 让 LLM 帮忙筛选：

```
以下是动画《{作品名}》第{N}集的字幕文本。
请识别并提取角色「{角色名}」的台词。

已知「{角色名}」的特征:
- 口头禅: {口头禅}
- 称呼习惯: {称呼}
- 说话风格: {风格描述}

只输出「{角色名}」说的话，每行一句。不确定的用 [?] 标注。

字幕文本:
{字幕内容}
```

> 💡 如果一部动画有 12 集，每集字幕约 300 行，选 5 集角色重点集 = 1500 行字幕。
> 经 LLM 筛选后，目标角色通常占 30-50% = **450-750 行角色台词**，比百科的 10 条台词丰富 50 倍。

**Step 3: 人际关系建模**

从百科中提取角色关系，生成关系图：

```
{角色名} 的人际关系:
  ❤️ {恋人/暧昧对象}: {关系描述}
  🤝 {挚友/搭档}: {关系描述}
  👨‍👩‍👧 {家人}: {关系描述}
  ⚔️ {对手/敌人}: {关系描述}
  👑 {上级/师父}: {关系描述}
```

**关系信息在 MENTOR.md 中的作用**：
- 决定角色对不同人的态度（如"对{恋人}温柔，对{对手}冷酷"）
- 提供互动模式的参考（"面对{挚友}会开玩笑，面对{上级}恭敬"）

#### 历史/名人采集流程

**Step 1: 百科生平**

```bash
# 维基百科
curl -sL "https://zh.wikipedia.org/wiki/{人名}" -H "User-Agent: Mozilla/5.0"
# 百度百科
curl -sL "https://baike.baidu.com/item/{人名}" -H "User-Agent: Mozilla/5.0"
```

重点提取：生平大事、思想主张、性格评价、争议事件。

**Step 2: 名言语录（最重要）**

```bash
# 中文名人
搜索: "{人名}" 名言 经典语录
搜索: "{人名}" 最有名的话
# 来源: 名言网/语录大全/知乎整理帖

# 英文名人
搜索: "{English name}" quotes
# 来源: goodreads.com/quotes, brainyquote.com
```

语录采集目标：**至少 20 条**，覆盖不同主题和情境。

**Step 3: 演讲/著作文本**

```bash
# 近现代名人
搜索: "{人名}" 演讲 全文 文字稿
搜索: "{人名}" 采访 原文

# 如果有视频（B站/YouTube搜索）
# 走 Section 3.3 的视频字幕提取流程
```

**Step 4: 古人的特殊处理**

古代人物需要额外处理文言文：
- 收集原文（诗词/奏疏/书信）+ 白话翻译
- MENTOR.md 中提供**双模式**：
  - `文言模式`：用古文风格回答（适合角色扮演）
  - `现代模式`：用现代口语但保留性格特征（适合日常对话）
- 标注哪些是有据可查的原话，哪些是基于性格推断的
