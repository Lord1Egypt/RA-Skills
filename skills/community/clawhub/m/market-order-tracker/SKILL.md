# 专业市场跟单Agent

## 敌人

> "订单多了就乱，漏单错单断货全靠嘴问"

当订单从每月10单增长到50单时，传统跟单模式的弊端开始暴露——采购说没收到订单，生产说交期太短，仓库说包装不对，货代说文件不全，客户催单时你才发现进度落后了半个月。每个环节都在"以为别人知道"，最后都是你的责任。

## 记忆点

**从接单到交付的全生命周期追踪**

本技能帮助外贸跟单人员建立系统化的订单管理体系，让每一个订单都清晰可见、每一个节点都有人负责、每一次异常都能及时处理。

## 第一章：订单全生命周期管理框架

### 1.1 订单生命周期阶段

每一个外贸订单都会经历以下阶段：

```
[询盘] → [报价] → [接单] → [定金确认] → [生产] → [验货] → [出货] → [发货] → [到港] → [尾款] → [交付] → [售后] → [复购]
```

### 1.2 阶段定义与标准

**阶段1：询盘阶段**
- 状态：INQ（Inquiry）
- 负责人：业务员
- 完成标志：收到客户询盘
- 下一步：提供报价

**阶段2：报价阶段**
- 状态：QUO（Quotation）
- 负责人：业务员
- 完成标志：发送正式报价
- 下一步：等待客户确认

**阶段3：接单阶段**
- 状态：ORD（Order）
- 负责人：业务员+跟单员
- 完成标志：客户确认PO/合同
- 下一步：收取定金

**阶段4：定金确认**
- 状态：DEP（Deposit）
- 负责人：财务+跟单员
- 完成标志：定金到账
- 下一步：下达生产工单

**阶段5：生产阶段**
- 状态：PRD（Production）
- 负责人：工厂/PMC
- 完成标志：产品完成
- 下一步：申请验货

**阶段6：验货阶段**
- 状态：INS（Inspection）
- 负责人：QC/第三方
- 完成标志：验货通过
- 下一步：安排出货

**阶段7：出货准备**
- 状态：SHP（Shipment Prep）
- 负责人：仓库+货代
- 完成标志：货物入仓
- 下一步：提交报关

**阶段8：发货阶段**
- 状态：FLT（Freight）
- 负责人：货代
- 完成标志：货物离港
- 下一步：发送单据

**阶段9：到港阶段**
- 状态：ARR（Arrival）
- 负责人：客户/清关行
- 完成标志：清关完成
- 下一步：收取尾款

**阶段10：尾款确认**
- 状态：BAL（Balance）
- 负责人：财务
- 完成标志：尾款到账
- 下一步：交付提单

**阶段11：交付阶段**
- 状态：DLV（Delivery）
- 负责人：跟单员
- 完成标志：客户签收
- 下一步：售后跟进

**阶段12：售后阶段**
- 状态：SRV（Service）
- 负责人：客服
- 完成标志：问题解决
- 下一步：复购邀请

### 2.1 接单核查清单

收到客户订单时，必须完成以下核查：

**客户资质核查**：
- [ ] 客户已完成KYC审核
- [ ] 客户信用额度充足
- [ ] 客户付款记录良好
- [ ] 无历史违约或纠纷

**订单信息核查**：
- [ ] 产品规格清晰完整
- [ ] 数量和单价明确
- [ ] 交期满足要求
- [ ] 贸易术语明确
- [ ] 包装要求确认

**商务条款核查**：
- [ ] 付款方式合理
- [ ] 质保条款明确
- [ ] 验货标准约定
- [ ] 违约责任清晰

## 第六章：异常处理与升级机制

### 6.1 异常类型定义

| 异常级别 | 定义 | 影响 | 响应时限 |
|---------|------|------|---------|
| P0 | 严重异常 | 交期延误/客户流失 | 2小时内 |
| P1 | 重要异常 | 成本增加/小延误 | 4小时内 |
| P2 | 一般异常 | 需协调处理 | 24小时内 |
| P3 | 轻微异常 | 记录跟踪 | 72小时内 |

### 6.2 常见异常与处理

**异常1：生产延期**
```
问题：工厂通知原材料短缺，生产延期5天
影响：原定5/21完成→5/26完成
处理：
1. 评估影响：可能赶不上5/25船期
2. 备选方案：
   - 方案A：换船（5/28开船，+3天）
   - 方案B：加急空运部分货物
3. 客户沟通：提前告知，争取理解
4. 升级判断：需业务员介入谈判
```

**异常2：验货不合格**
```
问题：验货发现5%产品有外观瑕疵
影响：需返工或补货
处理：
1. 评估不合格率：5%在AQL范围内
2. 备选方案：
   - 方案A：全检后出货，不良品返工
   - 方案B：按比例折扣出货
3. 客户沟通：如实告知，提供方案
4. 预防措施：加强QC管控
```

**异常3：货物破损**
```
问题：到港后发现部分货物破损
影响：客户可能索赔
处理：
1. 拍照留证：第一时间拍照
2. 通知货代：启动保险理赔
3. 客户沟通：说明情况，提供补货/折扣方案
4. 责任界定：根据提单条款处理
```

## 使用指南

### 快速开始

1. **新建订单**：输入客户名称、产品信息、交期要求
2. **系统核查**：自动完成核查清单检查
3. **生成工单**：自动创建内部工单并分配任务
4. **跟踪进度**：按节点跟踪，接收自动提醒
5. **客户通知**：系统自动发送标准化通知
6. **异常处理**：发现问题及时升级处理

### 常见问题

**Q: 如何处理多个订单同时跟单？**
A: 使用看板视图，按状态分组，优先处理紧急和即将到期的订单。

**Q: 客户要求变更订单怎么办？**
A: 在"订单变更"功能中记录变更内容，评估影响，重新计算交期和成本。

**Q: 如何处理货代拖延？**
A: 在"物流管理"中记录延误，评估备用方案，必要时升级更换货代。

## 适用场景

**最适合**：
- 订单量大的贸易公司（20单/月以上）
- 有自己生产线的工厂外贸部门
- 需要管理多个供应商的跟单团队
- 客户对交期敏感度高

**不太适合**：
- 订单量很小的SOHO（Excel足够）
- 纯代理不参与生产的业务
- 交期非常宽松的非紧急业务
