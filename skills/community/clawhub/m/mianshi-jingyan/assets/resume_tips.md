# 简历优化建议 / Resume Optimization Tips

---

## 一、简历基本结构 / Resume Structure

### 1.1 中文版 / CN Version

```
1. 基本信息
   - 姓名、联系方式、邮箱
   - 工作年限、学历
   - 所在地（可选）

2. 自我介绍（可选，但推荐）
   - 1-2句话概括核心能力
   - 突出差异化优势

3. 工作经历
   - 按时间倒序
   - 每段：公司、岗位、时间
   - 每段2-4个要点

4. 项目经历（核心）
   - 选择最有代表性的项目
   - 用数据量化成果
   - 体现方法论

5. 技能证书
   - 数据工具
   - 分析方法
   - 证书资质
```

### 1.2 English Version / EN Structure

```
1. Header
   - Name, Phone, Email, Location (city/country)
   - LinkedIn URL (recommended)
   - Optional: Portfolio / GitHub / Personal site

2. Professional Summary (2-3 sentences)
   - Years of experience + core specialty
   - Key methodologies / tools
   - One quantified achievement as a hook

3. Work Experience
   - Reverse chronological order
   - 2-4 bullet points per role
   - Format: Action verb + what + how + quantified result

4. Key Projects (highlight 2-3 most relevant)
   - Context + method + quantified impact
   - Show methodology (RFM, PSM, DID, A/B test)

5. Skills
   - Tools: SQL, Python, BI tools
   - Methodologies: RFM, PSM, DID, A/B testing, etc.
   - Languages + Certifications

6. Education
   - Degree, university, year
```

## 二、项目描述公式 / Project Description Formula

### 2.1 公式 / Formula

<!-- ZH-CN -->
```

```

<!-- EN -->


---

### 2.2 示例 / Examples

<!-- ZH-CN -->
**❌ 不好的写法**：
```
负责数据分析工作，用SQL取数，用Python做分析，
给业务提供数据支持。
```

**✅ 好的写法**：
```
【企微AI选房工具分析】
- 独立负责企微项目的数据分析，从Hive取数、Python清洗到策略建议
- 发现工具被动使用率40%、公域转私域仅28%的问题
- 用RFM+K-means做经纪人分层，差异化运营策略带动：
  * 商机转化率提升57%（28%→44%）
  * 新房业绩增长37.5%
  * 人均商机增长111.9%
- DID验证剔除自然增长，效果置信
```

<!-- EN -->
**❌ Weak example**:
```
Responsible for data analysis work, used SQL to extract data,
Python to analyze, provided data support to the business.
```

**✅ Strong example**:
```
[AI Property Tool Impact Assessment]
- Led end-to-end data analysis: Hive extraction → Python cleaning → strategy recommendations
- Identified key issues: 40% passive tool usage, public-to-private conversion only 28%
- Built RFM + K-means segmentation model → targeted operational strategies:
  * Lead conversion rate: +57% (28%→44%)
  * New property revenue: +37.5%
  * Leads per agent: +111.9%
- Validated with DID to remove seasonality and organic growth effects
```

## 三、量化成果的表达 / Quantifying Results

### 3.1 量化原则 / Quantification Principles

<!-- ZH-CN -->
```
✅ 量化维度：
- 提升了多少（百分比）
- 带来了多少价值（金额/数量）
- 效率提升（时间缩短、流程简化）
- 覆盖了多少（用户量、订单量）

❌ 避免：
- "效果不错"
- "有明显提升"
- "得到业务认可"
```

<!-- EN -->
```
✅ Quantify by:
- % improvement (e.g., +57% conversion rate)
- Absolute value (e.g., $X revenue generated)
- Time saved (e.g., reporting from 2 days → 2 hours)
- Scale (e.g., covered 10,000+ agents)

❌ Avoid:
- "Results were good"
- "Noticeable improvement"
- "Stakeholders were satisfied"
```

---

### 3.2 量化示例 / Quantification Examples

| 场景 / Scenario | 模糊表达 / Vague | 量化表达 / Quantified |
|----------------|-----------------|----------------------|
| 转化提升 / Conversion lift | "提升了转化" | "商机转化率提升57%（28%→44%）" |
| 业绩增长 / Revenue growth | "带动了业务增长" | "新房业绩增长37.5%" |
| 效率提升 / Efficiency gain | "提升了效率" | "取数时间从2天→2小时" |
| 用户分层 / User segmentation | "做了用户分层" | "RFM分层覆盖1万+经纪人，8类精细化运营" |
| Retention lift | "improved retention" | "Day-2 creator retention: +15%" |
| Tool adoption | "increased adoption" | "Tool adoption: 60% → 80% (+33%)" |

## 四、常用动词 / Action Verbs

### 4.1 中文 / CN

<!-- ZH-CN -->
```
✅ 推荐：
- 搭建（指标体系、分析体系）
- 设计（埋点方案、评估框架）
- 主导/独立负责（核心项目）
- 推动（业务落地）
- 优化（转化链路、模型效果）
- 验证（DID、AB测试）
- 发现（问题、机会）
- 沉淀（方法论、SOP）

❌ 少用：
- 参与、协助、支持、负责（太泛）
```

<!-- EN -->
```
✅ Strong verbs:
- Built / Designed (frameworks, systems, dashboards)
- Led / Owned (projects, analysis end-to-end)
- Drove (business impact, strategy execution)
- Optimized (funnels, model performance)
- Validated (with DID, A/B tests, regression)
- Identified (opportunities, risks, root causes)
- Developed (methodologies, SOPs, training materials)

❌ Weak verbs to avoid:
- Assisted, participated in, supported, helped (too vague)
```

## 七、注意事项 / Key Principles

### 7.1 常见问题 / Common Mistakes

<!-- ZH-CN -->
```
❌ 问题1：项目描述太泛 → 要具体：用什么方法、发现什么、带来什么价值
❌ 问题2：没有数据 → 要量化："转化率提升57%（28%→44%）"
❌ 问题3：技术堆砌 → 要结合业务："用SQL从Hive取数，Python清洗建模"
❌ 问题4：简历太长 → 2页以内，精选2-3个核心项目，每段3-5个要点
```

<!-- EN -->
```
❌ Mistake 1: Vague project descriptions → Be specific: what method, what did you find, what was the impact?
❌ Mistake 2: No data → Quantify: "Conversion rate +57% (28%→44%)"
❌ Mistake 3: Tech dumping → Tie to business: "SQL/Hive extraction + Python cleaning + PSM/DID validation"
❌ Mistake 4: Too long → 2 pages max, 2–3 key projects, 3–5 bullets per role
```

---

### 7.2 面试官想看到什么 / What Interviewers Look For

<!-- ZH-CN -->
```
✅ 业务理解：知道为什么要做这个项目，清楚项目对业务的价值
✅ 技术能力：会用合适的工具和方法，知道方法的原理和局限性
✅ 落地能力：能推动业务方执行，有可视化的交付
✅ 成长性：主动发现问题，持续学习新技术
```

<!-- EN -->
```
✅ Business impact: Knows why the project was done and its business value
✅ Technical skill: Uses the right tools, understands principles and limitations
✅ Execution: Can drive stakeholder buy-in, has tangible deliverables
✅ Growth mindset: Proactively identifies issues, continuously learns new skills
```

### 8.2 英文简历注意事项 / EN Resume Tips

```
✅ Dos:
- Keep to 1–2 pages (US: 1 page typical; EU/International: 2 pages OK)
- Use metrics: %, ×, dollar amounts — never vague statements
- Lead each bullet with a strong action verb
- Tailor the Professional Summary to each role
- Include LinkedIn URL — always

❌ Don'ts:
- Don't list every tool you know — only what's relevant
- Don't use Chinese company names without context (use English equivalents)
- Don't include a photo (unless applying in Asia specifically)
- Don't use tables or columns — ATS-friendly plain text format works best
- Don't use "I" — resume bullets are implied first person (or drop subject)
```
