# 错误码手册（universal-file-ops v1.0.0）

为普通大模型/智能体用户提供**通俗易懂**的错误提示。
每个错误都包含：错误码、脚本名称、发生行号、问题描述、解决方案。

---

## 错误码分类

| 前缀 | 类别 | 说明 |
|------|------|------|
| `UFO-1XXX` | 文件/路径错误 | 文件不存在、路径错误等 |
| `UFO-2XXX` | 语法/编码错误 | Python 语法错误、编码问题等 |
| `UFO-3XXX` | 环境错误 | Python 未安装、pip 未找到等 |
| `UFO-4XXX` | 网络/包管理错误 | 网络超时、包安装失败等 |
| `UFO-9XXX` | 未知错误 | 未分类的意外错误 |

## UFO-1XXX：文件/路径错误

### UFO-1001：文件不存在
- **脚本**：`py_tools.py` / `python_env.py`
- **发生行号**： varies（文件读取处）
- **问题描述**：找不到你要操作的文件。可能是路径拼写错误，或文件被删除/移动了。
- **解决方案**：
  1. 检查文件路径是否正确（绝对路径 vs 相对路径）
  2. 检查文件是否被其他程序占用
  3. 使用 `ls` / `dir` 确认文件存在

### UFO-1002：非 .py 文件
- **脚本**：`py_tools.py`
- **发生行号**：140~150 行附近
- **问题描述**：你指定的文件不是 Python 脚本（不是 `.py` 结尾）。
- **解决方案**：请选择以 `.py` 结尾的 Python 脚本文件。

### UFO-1003：venv 路径不存在
- **脚本**：`python_env.py`
- **发生行号**：220~230 行附近
- **问题描述**：venv（Python 虚拟环境）不存在。你需要先创建 venv。
- **解决方案**：运行 `python python_env.py setup` 创建 venv。

## UFO-2XXX：语法/编码错误

### UFO-2001：Python 语法错误
- **脚本**：`py_tools.py`
- **发生行号**：360~375 行（`cmd_review` 函数）
- **问题描述**：Python 文件有语法错误（括号不匹配、缩进错误、拼写错误等），无法解析。
- **解决方案**：
  1. 查看错误提示中的行号和列号
  2. 检查该行是否有：括号不匹配、冒号缺失、缩进错误
  3. 可运行 `python py_tools.py review --file <文件>` 查看详细错误位置

### UFO-2002：缩进包含 Tab
- **脚本**：`py_tools.py`
- **发生行号**：205~215 行（`cmd_normalize` 函数）
- **问题描述**：Python 文件缩进包含 Tab 符，这会导致在不同环境下缩进显示不一致。
- **解决方案**：
  1. 运行 `python py_tools.py normalize --file <文件> --fix` 自动修复
  2. 或手动将 Tab 替换为 4 个空格

### UFO-2003：编码声明错误
- **脚本**：`py_tools.py`
- **发生行号**：70~110 行（`cmd_normalize` 函数）
- **问题描述**：文件编码声明不是 `utf-8`，或缺少编码声明。
- **解决方案**：
  1. 运行 `python py_tools.py normalize --file <文件> --fix` 自动修复
  2. 或手动在第一行添加 `# -*- coding: utf-8 -*-`

### UFO-2004：行长度超过 120 字符
- **脚本**：`py_tools.py`
- **发生行号**：280~285 行（`cmd_normalize` 函数）
- **问题描述**：某一行代码太长（超过 120 字符），建议拆分成多行。
- **解决方案**：手动在合适位置换行（在逗号、运算符处换行）。

## UFO-3XXX：环境错误

### UFO-3001：指定的 Python 版本未安装
- **脚本**：`python_env.py`
- **发生行号**：210~220 行（`cmd_setup` 函数）
- **问题描述**：你要使用的 Python 版本（如 3.11）没有安装。
- **解决方案**：
  1. Windows：从 python.org 下载安装 Python 3.11
  2. Linux：运行 `apt install python3.11`
  3. macOS：运行 `brew install python@3.11`

### UFO-3002：pip 未找到
- **脚本**：`python_env.py`
- **发生行号**：260~270 行附近
- **问题描述**：venv 中的 pip 未找到，可能是 venv 创建不完整。
- **解决方案**：运行 `python python_env.py clean-reinstall` 重新创建 venv。

### UFO-3003：venv 创建失败
- **脚本**：`python_env.py`
- **发生行号**：220~230 行（`cmd_setup` 函数）
- **问题描述**：无法创建 venv，可能是 Python 安装不完整。
- **解决方案**：重新安装 Python（推荐 3.11 版本）。

## UFO-4XXX：网络/包管理错误

### UFO-4001：网络连接超时
- **脚本**：`python_env.py`
- **发生行号**：120~150 行（`_pip_install_with_retry` 函数）
- **问题描述**：安装包时网络连接超时，无法下载。
- **解决方案**：
  1. 检查网络连接是否正常
  2. 脚本会自动切换镜像源重试（最多 3 次）
  3. 如果仍然失败，可手动指定镜像：`pip install -i https://pypi.tuna.tsinghua.edu.cn/simple <包名>`

### UFO-4002：所有镜像源均连接失败
- **脚本**：`python_env.py`
- **发生行号**：120~150 行（`_pip_install_with_retry` 函数）
- **问题描述**：尝试了所有镜像源（清华、阿里、中科大），均无法连接。
- **解决方案**：
  1. 检查网络连接（防火墙、代理设置）
  2. 稍后重试
  3. 手动下载包的离线安装文件（.whl）进行安装

### UFO-4003：包名格式错误
- **脚本**：`python_env.py`
- **发生行号**：260~270 行（`cmd_install` 函数）
- **问题描述**：你指定的包名格式不正确（可能包含空格、特殊字符等）。
- **解决方案**：检查包名是否正确（可在 https://pypi.org 搜索确认）。

### UFO-4004：沙箱运行测试失败
- **脚本**：`py_tools.py`
- **发生行号**：sandbox-test 子命令
- **问题描述**：生成的测试在沙箱中运行失败（断言未通过、导入错误、语法错误等）。
- **解决方案**：
  1. 查看失败输出，定位失败用例和错误信息
  2. 如果导入错误（ModuleNotFoundError），需要安装缺失依赖后用 `--require` 参数重试
  3. 如果断言失败，检查测试逻辑或目标代码是否正确
  4. 修复后重新运行 `sandbox-test`

### UFO-4005：沙箱环境创建失败
- **脚本**：`py_tools.py`
- **发生行号**：sandbox-test 子命令
- **问题描述**：无法创建沙箱虚拟环境或安装依赖，可能是 Python 环境或网络问题。
- **解决方案**：
  1. 检查系统 Python 是否可用（运行 `python --version`）
  2. 检查网络连接，确保可以下载 pip 包
  3. 重试，如果持续失败，检查防病毒软件是否阻止了临时目录创建

## 如何使用错误码？

### 1. 查看错误响应
所有错误响应都是标准化 JSON 格式：
```json
{
  "status": "error",
  "error_code": "UFO-2001",
  "script": "py_tools.py",
  "line": 365,
  "message": "Python 语法错误：expected ':' (第 365 行，第 10 列）",
  "suggestion": "请检查第 365 行是否有拼写错误、括号不匹配、缩进错误等问题",
  "detail": "expected ':' (<unknown>, line 365)"
}
```

### 2. 根据错误码解决问题
1. 查看 `error_code` → 在本手册中查找对应错误
2. 查看 `line` → 定位到具体行号
3. 查看 `suggestion` → 按照建议操作
4. 查看 `detail` → 获取详细错误信息（方便调试）

### 3. 智能体自查修复
如果你是大模型/智能体，收到错误响应后：
1. 读取错误码对应的手册章节（本文件）
2. 根据 `line` 和 `message` 定位问题
3. 调用对应的修复工具（如 `py_tools.py normalize --fix`）
4. 重新执行原任务
