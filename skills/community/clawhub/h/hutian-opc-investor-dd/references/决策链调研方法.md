# 决策链调研方法

## 一、投资机构决策链概述

### 1.1 典型决策层级

```
┌─────────────────────────────────────────────────────────────┐
│                        投决会（最高决策层）                    │
│  ├── 表决权：过半数/2/3多数通过                              │
│  ├── 否决权：可一票否决                                       │
│  └── 特点：不定人不定时，重大事项才召开                        │
└─────────────────────────────────────────────────────────────┘
                              ↑
┌─────────────────────────────────────────────────────────────┐
│                    合伙人与管理委员会                         │
│  ├── 日常决策权：日常事项直接决定                              │
│  ├── 提案权：向投决会提交项目                                  │
│  └── 影响力：项目成败直接影响团队声誉                          │
└─────────────────────────────────────────────────────────────┘
                              ↑
┌─────────────────────────────────────────────────────────────┐
│                    MD/ED/总监（项目负责人）                    │
│  ├── 主导推进：全流程跟进项目                                  │
│  ├── 资源协调：内部团队和外部资源                               │
│  ├── 报告撰写：投资备忘录主笔                                   │
│  └── 维护关系：被投企业和合伙人关系                             │
└─────────────────────────────────────────────────────────────┘
                              ↑
┌─────────────────────────────────────────────────────────────┐
│                    VP/SVP（高级投资经理）                      │
│  ├── 执行尽调：协调各条线尽调工作                               │
│  ├── 报告撰写：协助撰写投资备忘录                               │
│  ├── 渠道维护：与项目方日常对接                                 │
│  └── 推荐权：对项目有初步推荐权                                 │
└─────────────────────────────────────────────────────────────┘
                              ↑
┌─────────────────────────────────────────────────────────────┐
│                Associate/Analyst（分析师）                    │
│  ├── 项目筛选：初步过滤不符合标准的项目                          │
│  ├── 信息收集：行业研究、数据整理                               │
│  ├── 辅助尽调：协助高级经理执行尽调                             │
│  └── 观察学习：参与项目讨论但不主导                             │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 决策链类型

| 类型 | 特点 | 适用机构 | 对接策略 |
|-----|------|---------|---------|
| **合伙人主导型** | 合伙人个人喜好决定一切 | 早期/精品基金 | 搞定合伙人就成功80% |
| **团队决策型** | MD+VP+Analyst集体评估 | 中型VC | 各层级都需维护 |
| **投决会主导型** | 委员会投票决定 | 大型基金/国资 | 准备充分的路演 |
| **双GP决策型** | 两位合伙人共同决定 | 双创始人基金 | 同时维护两位 |

## 二、关键人物识别方法

### 2.1 人物识别四步法

#### Step 1: 公开信息收集

```markdown
信息源清单：
1. 机构官网
   - 团队页面：照片、职位、背景
   - 投资理念：通常有合伙人署名文章
   - 新闻动态：近期主导的项目

2. 第三方平台
   - IT桔子/天眼查：投资事件中的角色
   - 领英：工作经历、关联关系
   - 36氪：报道中的引言人物

3. 媒体内容
   - 采访报道：谁在接受采访
   - 播客节目：谁在发声
   - 行业大会：谁在演讲

4. 社交媒体
   - 微博/雪球：个人IP展示
   - 公众号：文章作者
   - 个人微信：行业人脉
```

#### Step 2: 角色层级判断

| 职位 | 决策影响力 | 识别特征 |
|-----|----------|---------|
| **创始合伙人** | 90-100% | 公司创始人，官网首位，常有署名文章 |
| **管理合伙人** | 80-90% | 负责管理，公开露面多 |
| **合伙人** | 60-80% | 有独立判断权，可发起项目 |
| **MD/ED** | 40-60% | 执行层面，项目主负责人 |
| **VP** | 20-40% | 执行层面，撰写报告 |
| **Associate** | 5-15% | 初筛层面，辅助角色 |

#### Step 3: 影响力验证

```markdown
验证方法：
1. 查看历史项目中谁出现在签约现场
2. 搜索"主导"的新闻报道
3. 通过FA/其他创业者打听
4. 查看股权结构中的GP持股

判断信号：
✅ 确认信号：
- 创始人亲自跟进项目
- 多次出现签约/官宣
- 投决会上做汇报
- 有独立的项目推荐权

⚠️ 存疑信号：
- 始终是下属出面
- 官网介绍靠后
- 媒体引言较少
```

#### Step 4: 个人风格画像

```markdown
风格评估维度：
1. 背景专业度
   - 金融背景/产业背景/学术背景
   - 是否有相关赛道经验

2. 沟通风格
   - 进攻型/防守型/平衡型
   - 喜欢数据/喜欢故事/喜欢逻辑

3. 关注重点
   - 市场规模/商业模式/团队背景
   - 技术壁垒/竞争格局/退出路径

4. 决策风格
   - 快速决策/深思熟虑/集体决策
   - 感性/理性/直觉型

5. 合作倾向
   - 强势主导/平等合作/跟随支持
```

## 三、决策流程拆解

### 3.1 标准决策流程

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  初次接触   │ →  │   内核/立项  │ →  │    尽调     │ →  │   投决会    │
│             │    │             │    │             │    │             │
│  1-2周      │    │  1-2周      │    │  2-4周      │    │  1-2周      │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
       ↓                                                  ↓
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  条款谈判   │ ←  │     TS      │ ←  │   法律尽调  │ ←  │   通过      │
│             │    │             │    │             │    │             │
│  2-4周      │    │  1-2周      │    │  1-2周      │    │             │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
       ↓
┌─────────────┐    ┌─────────────┐
│   打款      │    │   交割      │
│             │    │             │
│  1-2周      │    │  1周内      │
└─────────────┘    └─────────────┘
```

### 3.2 各阶段详解

#### 3.2.1 初次接触阶段

| 要素 | 说明 | 评估 |
|-----|------|-----|
| **来源判断** | FA推荐/ cold call/ 主动BD/ 朋友引荐 | FA推荐最有效 |
| **响应速度** | 多久回复/是否热情 | 1周内正常 |
| **初次沟通** | 是否约见/电话/仅邮件 | 面谈最好 |
| **初步意向** | 是否有明确兴趣 | 关键判断 |

**优化建议：**
```
✅ 提高初次接触效率：
- 优先FA推荐（有背书）
- 朋友引荐（信任传递）
- 参加路演活动（批量接触）
- 精准BD（研究后定制）

❌ 低效方式：
- 盲目群发BP
- 非相关领域BD
- 过于频繁跟进
```

#### 3.2.2 内核/立项阶段

| 要素 | 说明 | 评估重点 |
|-----|------|---------|
| **立项标准** | 机构内部立项门槛 | 赛道/阶段/规模 |
| **立项材料** | 需要准备的材料清单 | 完整度 |
| **决策周期** | 多久给反馈 | 正常1-2周 |
| **立项会** | 谁参加/问什么 | 关键评估点 |

**立项常见问题：**
```
典型问题清单：
1. 你们解决什么痛点？
2. 市场规模有多大？
3. 为什么是现在？
4. 你们的竞争优势是什么？
5. 团队为什么能做这个？
6. 财务预测的假设是什么？
7. 估值是怎么计算的？
8. 打算拿这笔钱做什么？
```

#### 3.2.3 尽调阶段

| 尽调类型 | 内容 | 周期 |
|---------|------|-----|
| **业务尽调** | 财务、法务、业务、市场 | 2-4周 |
| **法务尽调** | 法律文件、股权结构、诉讼 | 1-2周 |
| **财务尽调** | 财务报表、现金流、税务 | 1-2周 |
| **技术尽调** | 专利、技术壁垒、团队 | 1-2周 |
| **背景调查** | 创始人履历、诚信调查 | 1周 |

**配合技巧：**
```
✅ 高效配合尽调：
- 提前准备材料清单
- 指定专人对接
- 及时响应补充材料
- 诚实披露问题

❌ 尽调受阻信号：
- 要求补充的材料不断变化
- 尽调时间无故延长
- 对接人频繁更换
- 突然冷淡
```

#### 3.2.4 投决会阶段

| 要素 | 说明 |
|-----|------|
| **投决会频率** | 每周/双周/月度 |
| **参会人员** | 合伙人+外部专家 |
| **上会材料** | 投资备忘录+尽调报告 |
| **表决规则** | 多数/全票/一票否决 |
| **结果反馈** | 当场/次日/一周内 |

**投决会应对：**
```
✅ 准备充分：
- 了解可能提问的问题
- 准备补充材料和数据
- 核心创始人出席
- 良好心态

常见提问：
1. 如果腾讯/阿里也做这个怎么办？
2. 为什么不先做小市场验证？
3. 创始团队为什么没有CTO？
4. 你们的单位经济模型怎么算的？
5. 怎么看待XX竞品？
```

## 四、决策周期分析

### 4.1 周期评估表

| 机构类型 | 初次到打款 | 周期说明 |
|---------|-----------|---------|
| **精品小基金** | 1-2个月 | 决策快，合伙人说了算 |
| **中型VC** | 2-4个月 | 标准流程，VP-MD-合伙人 |
| **大型VC** | 3-6个月 | 流程复杂，多层级审批 |
| **国资基金** | 6-12个月 | 多级审批，可能需上级单位 |
| **CVC** | 2-4个月 | 产业协同评估 |

### 4.2 周期异常分析

```markdown
周期过长的原因识别：

1. 兴趣不足
   信号：跟进不积极/回复慢
   应对：评估是否继续投入精力

2. 内部争议
   信号：反复要求补充材料/约见不同人
   应对：找出争议点，针对性解决

3. 流程复杂
   信号：法务/合规要求多/审批环节多
   应对：耐心配合，提前了解流程

4. 资金问题
   信号：款项拖延/反复确认LP
   应对：评估资金来源可靠性

5. 条款分歧
   信号：TS到SPA阶段反复修改
   应对：明确底线，必要时放弃
```

### 4.3 时间管理策略

```
融资时间表规划：

假设目标融资时间：3个月

Week 1-2: 机构筛选与接触
- 确定目标机构名单（20-30家）
- 通过FA/朋友开始引荐

Week 3-4: 第一轮对话
- 优先接触5-10家最有希望的
- 评估兴趣度

Week 5-8: 深度跟进
- 重点跟进3-5家明确兴趣的
- 准备尽调材料

Week 9-12: 尽调与谈判
- 配合尽调
- 条款谈判

Week 13: 签约打款
- 最终选择
- 签约打款
```

## 五、一票否决权识别

### 5.1 有一票否决权的人物类型

| 角色 | 否决权范围 | 识别方法 |
|-----|----------|---------|
| **创始合伙人** | 几乎所有事项 | 官网首位/创始人 |
| **管理合伙人** | 多数事项 | 负责管理事务 |
| **特定MD** | 其负责领域 | 需打听 |
| **LP代表** | 国资基金/大LP | LP背景 |
| **投决会成员** | 投票时行使 | 投决会成员名单 |

### 5.2 识别与应对

```
识别方法：
1. 查看官网团队介绍顺序
2. 搜索"合伙人/创始人"+表态
3. 通过FA打听
4. 观察签约现场的出席人

应对策略：
✅ 对于有一票否决权的合伙人：
- 重点维护关系
- 确保获得个人认可
- 提前了解其关注点
- 做好充分准备

⚠️ 风险信号：
- 对接人始终无法安排与合伙人见面
- 合伙人态度模糊
- 合伙人频繁更换
```

## 七、实战技巧

### 7.1 加速决策的方法

```
✅ 加速策略：
1. 搞定关键人物
   - 找到有一票否决权的人
   - 直接对话
   - 获得个人背书

2. 提供完整材料
   - 一次性提供所有尽调材料
   - 减少反复补充
   - 提高效率

3. 制造紧迫感
   - 其他机构的竞争态势
   - 融资窗口期
   - 业务发展需求

4. 展示确定性
   - 已有收入/客户
   - 团队完整度
   - 里程碑达成

5. 借助FA
   - FA的专业背书
   - 熟悉流程
   - 帮助协调
```

### 7.2 判断决策进展

```
决策阶段判断：

初期接触：
- 信号：约见/电话沟通积极
- 信号：要求BP/材料
- 信号：提出深入问题

立项：
- 信号：要求更详细材料
- 信号：安排尽调
- 信号：讨论条款框架

尽调：
- 信号：进入正式尽调
- 信号：签署TS
- 信号：法务介入

投决：
- 信号：确定上会时间
- 信号：要求补充材料
- 信号：与合伙人会面

签约：
- 信号：SPA谈判
- 信号：最终条款确认
- 信号：打款时间表
```

### 7.3 失败原因识别

```
融资失败信号：

❌ 兴趣丧失：
- 回复变慢/不回复
- 问题变少/表面化
- 不再约见

❌ 内部阻力：
- 反复要求同样材料
- 换不同人了解情况
- 提到"需要讨论"

❌ 条款僵局：
- 条款分歧无法调和
- 反复修改TS
- 拖延签署

❌ 资金问题：
- LP审批拖延
- 反复确认资金
- 突然提出新条件

识别后的应对：
- 及时止损，转向其他机构
- 了解真实原因
- 评估是否可补救
```

## 八、数据来源

### 8.1 信息获取渠道

| 渠道 | 可获取信息 | 可靠性 |
|-----|-----------|-------|
| 机构官网 | 组织架构、团队信息 | 高 |
| 新闻报道 | 项目报道中的角色 | 高 |
| FA机构 | 内部流程、决策人 | 高 |
| 已投企业 | 真实合作体验 | 高 |
| 领英 | 团队背景、离职信息 | 中 |
| 行业社群 | 口碑、传闻 | 中 |
| 求职平台 | 团队规模、招聘 | 中 |

### 8.2 交叉验证方法

```
验证原则：
- 单来源 → 存疑
- 两个独立来源 → 基本可信
- 三个以上来源 → 可信

典型验证场景：
1. 确认关键人物
   - 官网 + 新闻 + FA确认 → 一致则可信

2. 确认决策周期
   - FA说法 + 已投企业说法 + 历史项目 → 综合判断

3. 确认条款风格
   - FA了解 + 协议文本 + 创业者反馈 → 全面了解
```
