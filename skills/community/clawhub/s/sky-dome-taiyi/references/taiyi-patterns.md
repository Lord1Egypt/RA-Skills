# Taiyi Patterns / 太一模式库

## Mission Command Pattern

1. State the deliverable.
2. Define success gate.
3. Inspect actual state.
4. Identify risk class.
5. Plan 3-7 steps.
6. Execute active step.
7. Verify.
8. Adapt.
9. Deliver.
10. Consolidate.

## Research Pattern

1. Prefer primary sources.
2. Use search only when no known URL exists.
3. Fetch static pages before browser automation.
4. Cross-check surprising claims.
5. Separate fact, inference, and recommendation.
6. Cite source names or URLs.

## Debug Pattern

1. Reproduce or inspect exact error.
2. Check simplest config first.
3. Change one variable at a time.
4. Run smallest smoke test.
5. Record root cause only after evidence.

## Skill Forge Pattern

1. Choose slug and public bilingual name.
2. Metadata must clearly describe trigger cases.
3. SKILL.md stays procedural; optional depth goes to references.
4. Scripts only for deterministic repeated value.
5. Avoid README/CHANGELOG clutter.
6. Stage cleanly before publishing.
7. Inspect version after publish.

## Release Pattern

1. Inspect working files.
2. Remove state, backups, logs, caches, secrets.
3. Bump version.
4. Create clean staging directory.
5. Publish only after confirmation when external.
6. Inspect metadata/page/download command.
7. Report exact version and evidence.

## Swarm Pattern

1. Split into independent tasks.
2. Spawn isolated sub-agents with narrow deliverables.
3. Continue main work if safe.
4. Merge findings.
5. Verify before final claim.

## Archive Pattern

1. Summarize goal and current state.
2. List decisions and changed files.
3. Capture evidence and open risks.
4. Convert reusable lessons into memory/docs/skills.
5. Prune noisy transient state.
