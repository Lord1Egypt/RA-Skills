# SoloForge — 一人公司商务助手

**越用越聪明，不是装上就聪明。** 数据积累越多，自动化程度越高。

面向中国市场的一人公司商务助手。合同条款基于中国法律，定价为人民币。

帮一个不具备商务团队的人，完成原本需要商务团队协作的文档和流程。不是替代团队，是让一个人不缺位。

## 为什么不用 ChatGPT 配一个 prompt？

能用。但每次都要重新描述项目、重新输入客户信息、重新报价。

SoloForge 的差异是**跨文档数据流**——项目数据持久化在本地 JSON 文件中，从提案→合同→对账→催款，数据自动流转，不需要重复填表。

| | 第 1 单 | 第 10 单 |
|:---|:---|:---|
| 写提案 | 和 ChatGPT 一样，需要你描述客户和需求 | 从 clients.json 读客户信息和历史合作记录，自动填充 |
| 报价 | 问你预算范围 | 参考历史客单价和客户风险等级，自动匹配首付比例 |
| 写合同 | 手动填金额、条款 | 提案数据自动携带，合同条款从 config.json 读取默认值 |
| 催款 | 你得记得谁欠钱 | 遍历 deals.json 逾期检查，按天数自动匹配催款语气 |
| 经营复盘 | 没有数据，纯靠记忆 | 汇总 deals.json 全部项目，自动计算回款率、客户集中度、现金流 |

第 1 单的体验和 ChatGPT 没区别。第 10 单的时候，它知道你的客户、你的定价、你的回款节奏——这才是真正的差异化。

## 使用示例（第 10 单场景）

你已经在 SoloForge 中管理了 5 个客户、完成了 7 个项目。clients.json 里有历史数据，deals.json 里有 3 个进行中的项目。

```
你：帮我给华创科技写个提案，他们想做一个小程序

SoloForge：
  [读取 clients.json → 华创科技：零售行业，黄灯客户（预算模糊），曾合作1次]
  [读取 deals.json → 历史客单价 ¥25,000 / 当前3个进行中项目 / 回款率73%]
  [客户集中度检测 → 最大客户占比 52% ⚠️ 超过40%安全线]
  [首付建议 → 黄灯客户，建议50%]
  [web search → 小程序开发最新市场价格]

  生成提案（三档报价：基础版 ¥18,000 / 推荐版 ¥32,000 / 旗舰版 ¥58,000）
  ⚠️ 当前最大客户占总营收52%，建议拓展更多客户降低依赖。
  💡 华创科技曾表达过二期需求，是否同时准备SOW？
  是否保存为项目？确认后写入 deals.json，后续合同和对账自动携带数据。

你：推荐版，帮我写合同

SoloForge：
  [自动携带：华创科技 / 小程序开发 / ¥32,000 / 首付50%=¥16,000]
  [加载 config.json：验收期5天 / 违约金上限30% / 修改3次]

  生成服务合同（展开为正式法律文书格式）...
  💡 建议先签NDA保护方案细节。
```

第一单时没有这些数据，SoloForge 会问你——和 ChatGPT 一样。但每次保存后，下一单就少问一点。

## 数据驱动的全生命周期

```
  获客          成单           签约            交付            收款            复盘
   ↓             ↓              ↓              ↓              ↓              ↓
 开发信  →  提案/报价单  →  合同/NDA/SOW  →  进度/验收报告  →  对账单/催款函  →  月/季/年分析
   │             │              │              │              │              │
   └─────────────┴──────────────┴──────────────┴──────────────┴──────────────┘
                                    ↕ 读写
                         data/deals.json
                         data/clients.json
                         data/config.json
```

## 模块架构

```
solo-forge/
├── SKILL.md                        # 入口：意图识别 + 路由 + 首次引导
├── data/                           # 业务数据（.gitignore 排除真实数据）
│   └── example/                    # schema 示例（仅供结构参考，不作为业务数据）
│       ├── deals.json
│       ├── clients.json
│       └── config.json
└── references/
    ├── proposal.md                  # 提案/获客策略 + 客户信号扫描
    ├── proposal-templates.md        # 提案/开发信/案例/服务介绍模板
    ├── contract.md                  # 合同保护条款解读
    ├── contract-templates.md        # 合同/NDA/SOW 模板
    ├── quotation.md                 # 三档定价法/定价方法论/催款策略
    ├── quotation-templates.md       # 报价单/对账单/催款函模板
    ├── negotiation.md               # 谈判决策树（攻防场景）
    ├── tracking.md                  # 项目 CRUD / 计费类型 / 逾期检查
    ├── clients.md                   # 客户档案管理 / 调研搜索策略
    ├── business-review.md           # 经营分析决策矩阵（集中度/现金流/回款率）
    ├── review-templates.md          # 月/季/年分析模板
    ├── delivery.md                  # 交付状态判断 / 验收策略 / 满意度调查策略
    └── delivery-templates.md        # 进度报告/验收报告/满意度调查模板
```

每个模块 <100 行，按需加载，不全量注入上下文。

## 安装

SoloForge 是 SKILL.md 格式，支持所有兼容 [ClawHub](https://clawhub.ai) 生态的 Agent（[龙虾 OpenClaw](https://github.com/openclaw/openclaw)、QClaw、WorkBuddy 等）和 [Claude Code](https://docs.anthropic.com/en/docs/claude-code)。

```bash
# ClawHub 生态（龙虾 / QClaw / WorkBuddy）
npx clawhub@latest install Qiu-Difeng/solo-forge

# Claude Code
git clone https://github.com/Qiu-Difeng/solo-forge.git
cd solo-forge
claude
```

首次启动时 SoloForge 会引导你确认默认配置（首付比例、验收期、违约金等），确认后存入 `data/config.json`，之后即可正常使用。

> 没装 Agent？先装 [龙虾（OpenClaw）](https://github.com/openclaw/openclaw) 或 [Claude Code](https://docs.anthropic.com/en/docs/claude-code)。龙虾需要先装 [Node.js](https://nodejs.org/)（LTS 即可）。

## 快速上手

直接用自然语言对话：

**获客**："帮我给XX公司写封冷邮件" / "写个服务介绍"
**成单**："帮我写个提案" / "做个报价单"
**签约**："帮我写合同" / "出个NDA" / "写个SOW"
**谈判**："客户说太贵了怎么回" / "客户要求加需求怎么办"
**管理**："添加一个项目" / "客户列表" / "经营概览" / "逾期检查"
**复盘**："写个月报" / "做个季度复盘" / "帮我分析一下经营状况"
**交付**："写个进度报告" / "出个验收报告" / "满意度调查"

不知道说什么就说"帮我"，SoloForge 会引导你。

💡 已有客户和项目？先说"帮我添加3个客户"或"录入我的历史项目"，跳过冷启动阶段。

## 覆盖的 15+ 文档类型

| 阶段 | 文档 |
|:---|:---|
| 获客 | 冷邮件（A/B 测试）、服务介绍、案例包装、商业计划书 |
| 成单 | 提案（含三档报价） |
| 签约 | 服务合同、NDA（双向保密）、SOW（项目确认书） |
| 交付 | 进度报告、验收报告 |
| 收款 | 报价单、对账单、催款函（按逾期天数自动升级语气） |
| 复盘 | 月度分析、季度复盘、年度总结、满意度调查 |

## 做不到什么

**主动承认边界比过度承诺更可信。**

- 不生成 PDF / Word —— 输出 Markdown，你需要自己复制到 Word 排版并导出 PDF
- 不连接银行 / 税务 / 开票系统 —— 数据仅在本地 JSON 文件中
- 不替代律师 —— 合同模板基于中国商业惯例，重大金额建议咨询专业律师
- 不替用户做商业决策 —— 提供建议和风险识别，最终决策权在用户
- 风险感知依赖数据积累 —— 如果不使用项目管理功能记录数据，预警功能无法生效

## 技术细节

- **零依赖**：纯 SKILL.md prompt 架构，不需要 Python / Node / 任何运行时
- **数据持久化**：直接读写本地 JSON 文件（deals / clients / config）
- **按需加载**：模块 <100 行，根据意图只加载相关模块，不全量注入
- **中国商业文化适配**：付款条款、违约金上限（民法典第585条）、催款语气分级
