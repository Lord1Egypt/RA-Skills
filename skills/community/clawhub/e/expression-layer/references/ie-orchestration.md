# 投教工作流 × 表达层 技能编排分析

**分析时间**：2026-04-24
**分析对象**：`investor-education-workflow` (IE) + `expression-layer` (EL)

---

## 一、定位与边界

| 维度 | 投教工作流 (IE) | 表达层 (EL) |
|------|----------------|------------|
| **核心定位** | 知识检索 + 意图 framing + 合规 + 反馈入库 | 内容生成 + 格式转换 + 可视化 + 发布 |
| **输入** | 用户问题/知识点/客户场景 | 结构化内容 + 输出意图 |
| **输出** | 投教策略/内容框架/合规审查/入库标记 | Markdown/PNG/HTML/公众号推文 |
| **依赖** | 知识库/搜索源/数据层/EL | ljg-skills/humanizer/wechat-publisher |
| **边界** | 不直接生成最终格式，负责“教什么、怎么教” | 不负责知识准确性，负责“怎么表达、怎么呈现” |

**关系**：IE 是“大脑/策略层”，EL 是“双手/执行层”。IE 决定内容方向与合规，EL 负责多形式交付。

---

## 二、编排流程图

```
用户提问/知识点触发
        │
        ▼
[IE 阶段①] 检索知识库 → 命中/未命中
        │
        ▼
[IE 阶段②] 定向搜索（官方源）→ 原始素材
        │
        ▼
[IE 阶段③] 内容框架生成 + 意图标记
   ├── 意图：plain / writes / card / wechat / multi
   └── 内容：结构化文本/数据/链接
        │
        ▼
[EL 路由编排] 按意图路由至 ljg-skills
        │
        ▼
[EL 执行] 生成/转换/可视化
        │
        ▼
[IE 阶段④] 合规审查 + 多形式交付
        │
        ▼
[IE 阶段⑤] 反馈入库（使用频次/难度/迭代标记）
```

---

## 三、技能调用映射

| IE 阶段 | 调用 Skill | 职责 | 输出给 EL |
|---------|-----------|------|----------|
| ① 检索 | `ljg-learn` | 概念解剖/知识库检索 | 结构化知识点 + 意图标记 |
| ② 搜索 | `searxng` + `url-to-markdown` | 官方源补充 | 原始素材 + 来源标注 |
| ③ 框架 | `decision-checklist` | 合规/偏误识别 | 内容框架 + 风险提示 |
| ④ 交付 | `expression-layer` | 路由/生成/可视化/发布 | 最终成品（Markdown/PNG/HTML/公众号） |
| ⑤ 入库 | `task-state-tracker` | 反馈/版本控制 | 入库记录 + 迭代建议 |

## 四、编排模式示例

### 示例 1：单步路由（直接问答）
```
IE: 客户问“什么是PE？” → 检索知识库 → 标记意图 plain
EL: ljg-plain → 输出大白话（≤200字）
IE: 合规检查 → 交付客户 → 标记“理解难度⭐”入库
```

### 示例 2：串联路由（论文解读）
```
IE: 用户提供论文链接 → 标记意图 paper_flow
EL: ljg-paper → 生成解读 → ljg-card(-c) → 生成漫画 PNG
IE: 合规检查 → 交付（Markdown+PNG） → 入库
```

### 示例 3：并行路由（多形式输出）
```
IE: 投教活动需要“定投”素材 → 标记意图 multi (plain + card + writes)
EL: 并行执行 ljg-plain + ljg-card(-i) + ljg-writes
IE: 整合输出 → 合规检查 → 交付（文本+卡片+文章） → 入库
```

### 示例 4：发布路由（公众号）
```
IE: 生成完整投教文章 → 标记意图 wechat
EL: wechat-publisher → 格式转换+封面匹配+合规检查 → 推送草稿
IE: 确认发布 → 记录阅读量/互动数据 → 入库
```

## 五、优势与改进点

### ✅ 优势
1. **解耦设计**：IE 专注知识/合规/策略，EL 专注表达/格式/发布，职责清晰
2. **灵活编排**：支持单步/串联/并行/发布，覆盖全场景
3. **可扩展**：新增 skill 只需更新 EL 路由矩阵，不影响 IE 逻辑
4. **闭环反馈**：IE 阶段⑤ 持续收集使用数据，驱动知识库进化

### ⚠️ 改进点
1. **意图识别自动化**：当前依赖人工标记，后续可加入 NLU 模块自动识别意图
2. **上下文传递**：IE → EL 需传递结构化上下文（内容+意图+格式），避免信息丢失
3. **错误降级**：EL 某 skill 失败时，IE 需提供降级策略（如 card 失败 → 降级为纯文本）
4. **性能优化**：并行路由需控制并发数，避免 Token 超限或 API 限流
