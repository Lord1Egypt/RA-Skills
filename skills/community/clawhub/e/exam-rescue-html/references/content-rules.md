# 抢救清单内容规则

## 一、核心公式速记（formula_cards）

### 筛选来源
从 knowledge_map.points 中找出满足以下任一条件的考点：
- point.name 或 point.evidence 含"公式"/"定理"/"法则"/"定律"/"性质"/"推论"
- RAG 检索返回的原文中含数学表达式、化学方程式、物理公式等

### 每张卡片字段
| 字段 | 要求 |
|------|------|
| name | 公式/定理/法则名称，≤20字 |
| formula | 公式或定义正文；数学公式用 $LaTeX$ 格式；非数学学科写文字公式/原文 |
| condition | 适用条件/前提/范围，≤30字；无则 null |
| memory_hook | 口诀/关键词，≤10字；无则 null，不强制 |

### 写法规范
- formula 直接写内容，不写"公式为："等引导语
- 数学：`$f'(x) = \lim_{h\to0}\frac{f(x+h)-f(x)}{h}$`
- 物理：`$F = ma$`，condition 填"质量恒定，合外力=F"
- 文科/法律：用简洁文字原文，不强行 LaTeX

## 二、高频考点清单（hotspot_cards）

### 筛选规则
```
主条件：importance ≥ 0.75
降级（筛选后 < 5 个时）：importance ≥ 0.6
按 importance 降序，上限 20 个
```

### 每张卡片字段
| 字段 | 要求 |
|------|------|
| name | 考点名，与 knowledge_map 保持一致 |
| importance | 取自 knowledge_map |
| is_weak_point | 取自 knowledge_map |
| core_note | ≤2句话，30–80字，直接陈述；公式>条件>步骤>概念 |

### core_note 按知识类型写法
**公式类**：写公式 + 使用条件
> "sin²x + cos²x = 1；变体：1 + tan²x = sec²x（x ≠ kπ + π/2）。"

**条件判断类**：写充分/必要条件 + 典型反例
> "f'(x₀)=0 是极值必要条件，非充分条件；x³ 在 x=0 处导数为零但非极值。"

**步骤类**：写关键步骤，跳过显然步骤
> "1. 令分母=0求奇点；2. 验证左右极限是否相等；3. 与函数值比较。"

**概念辨析类**：写最精简的区分点
> "充分条件→推出结论；必要条件←结论倒推；充要条件←→双向。"

### 优先级颜色（imp-dot class）
| importance 范围 | class |
|----------------|-------|
| ≥ 0.8 | high |
| 0.6–0.79 | mid |
| < 0.6 | low |

## 三、高频错点清单（mistake_cards）

### 筛选来源
- knowledge_map.points 中 is_weak_point = true 的所有考点
- 补充：RAG 检索返回的原文中含"错误"/"误区"/"注意"/"容易混淆"/"陷阱"的考点
- 上限 15 个，按 importance 降序

### 每张卡片字段
| 字段 | 要求 |
|------|------|
| name | 错点名，描述错误本质，如"极限与连续的混淆"，≤20字 |
| wrong | 常见错误理解/做法，以"❌"开头，≤40字 |
| right | 正确理解/做法，以"✅"开头，≤40字 |
| tip | 辨别/记忆技巧，≤20字；无则 null |

### 写法规范
- wrong 和 right 必须形成对比，不是错误的解释而是正确做法的呈现
- 示例：
  - wrong: `❌ 极限存在则函数在该点连续`
  - right: `✅ 极限存在且等于函数值，才连续`
  - tip: `可导必连续，连续不必可导`

## 四、无内容兜底规则

| 场景 | 处理 |
|------|------|
| 该学科无公式（如纯文科） | formula_cards = []，`{{formula_count}}` = 0，隐藏该分区（Claude 在 HTML 中不输出该 section） |
| 筛选后考点数 < 5 | 触发降级条件，降至 importance ≥ 0.6 |
| 无薄弱点且无错误线索 | mistake_cards = []，`{{mistake_count}}` = 0，隐藏该分区 |
| 知识库某考点无返回 | 仅凭 point.name + point.evidence 生成，继续 |
