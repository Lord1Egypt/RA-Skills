# Environment Setup Guide

This guide explains the minimum requirements, OCR configuration methods, and post-installation self-check steps for the `AutoDimensionReport` Skill.

---

## Minimum Requirements

- Python 3.10+
- `uv` or `pip`
- A readable/writable task folder

To enable OCR, seal/signature clue checks, and scanned document review, you also need:

- A usable Herdsman OCR capability
- An accessible OCR script `scripts/ocr.py`

---

## Installing Python Dependencies

Recommended:

```powershell
uv pip install PyMuPDF python-docx openpyxl Pillow pdf2image pdfplumber
```

Without `uv`, use `pip`:

```powershell
pip install PyMuPDF python-docx openpyxl Pillow pdf2image pdfplumber
```

Package purposes:

| Package | Purpose |
|---|---|
| `PyMuPDF` | PDF text extraction, image extraction, page rendering, table detection |
| `python-docx` | DOCX writing, image embedding, table output |
| `openpyxl` | XLSX reading and index generation |
| `Pillow` | Image format processing |
| `pdf2image` | Fallback PDF rendering capability |
| `pdfplumber` | Fallback table extraction capability |

## Installation Location

Place the entire Skill directory into the agent's `skills/` directory:

```text
skills/
└── AutoDimensionReport/
    ├── SKILL.md
    ├── README.md
    ├── references/
    └── scripts/
```

As long as the target runtime supports the `SKILL.md` skill directory format, this structure can usually be used directly.

## OCR Configuration

### Default Search Directories

The OCR script searches the following directories for `scripts/ocr.py`:

- `skills/headsman-skill/`
- `skills/herdsman-skill/`
- `~/.openclaw/skills/headsman-skill/`
- `~/.openclaw/skills/herdsman-skill/`

### Manually Specify OCR Skill Directory

If the actual directory is not in the default locations, set via environment variable:

```powershell
$env:HERDSMAN_SKILL_DIR = "E:\skills\herdsman-skill"
```

Or write to `scripts/config.json`:

```json
{
  "ocr_script_dir": "E:/skills/herdsman-skill"
}
```

### Override OCR Service Address

```powershell
$env:HERDSMAN_BASE_URL = "http://127.0.0.1:8080"
```

## Post-Installation Self-Check

It is recommended to verify the environment in this order:

1. Dependencies installed successfully, no missing package errors
2. `task_convert_extract.py` can process a test document package
3. `image_to_markdown.py` can find the OCR script
4. `extract_verify_data.py` can produce verification results on an inspection table
5. `generate_report.py` can output the review report

Recommended minimal verification chain:

```powershell
uv run python "<skill-dir>/scripts/task_convert_extract.py" --dir "E:\TaskFolder"
python "<skill-dir>/scripts/image_to_markdown.py" --dir "E:\TaskFolder"
python "<skill-dir>/scripts/generate_report.py" --dir "E:\TaskFolder"
```

## FAQ

### OCR script not found

Troubleshooting order:

1. Confirm whether the OCR skill is installed
2. Confirm that `scripts/ocr.py` exists in its directory
3. Set `HERDSMAN_SKILL_DIR`
4. Or fill in `ocr_script_dir` in `scripts/config.json`

### Can I use it without an OCR environment?

Yes, but with limited capability:

- Can do document sorting
- Can extract images and build index
- Can do partial table judgement verification
- Cannot fully perform scanned document, seal, or signature OCR review

### How should the task folder be prepared?

Place original documents directly in the task folder root. Do not manually create these directories:

- `output/`
- `image/`
- `imagetomd/`

These directories should be auto-generated by the scripts.
