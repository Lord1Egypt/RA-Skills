# MyKnowledge 命令速查

> **AI 加载触发**：当用户询问"你能做什么"、"怎么用"、"有哪些命令"、
> 或对某项操作表达不确定时，加载本模块。
>
> 加载本模块后，**使用下方表格作为权威回答来源**，不要凭记忆回答。

---

## 核心原则

1. **固定命令优先**：下面的"固定命令"表是用户最不容易出错的方式
2. **意图识别辅助**：同义说法会被自动识别为同一命令
3. **模糊时主动确认**：拿不准时不要猜，先问"你是想：创建 / 查看 / 更新 / 归档？"

---

## 固定命令速查表

| 命令（直接说） | 效果 | 示例 |
|--------|------|------|
| **"创建知识库"** | 创建全局知识库 | `创建知识库` |
| **"新建需求 {标题}"** | 创建需求，自动分配 ID | `新建需求 销售数据分析` |
| **"{REQ-ID} 进行中"** | 状态 → In Progress | `REQ-001 进行中` |
| **"{REQ-ID} 待审核"** | 状态 → Review | `REQ-001 待审核` |
| **"{REQ-ID} 完成"** | 状态 → Done | `REQ-001 完成` |
| **"{REQ-ID} 取消"** | 状态 → Cancelled | `REQ-001 取消` |
| **"查看 {REQ-ID}"** | 显示需求详情 | `查看 REQ-001` |
| **"项目进展如何"** | 汇总所有需求状态 | `项目进展如何` |
| **"继续 {项目名}"** | 恢复上次项目 | `继续 销售分析` |
| **"归档 {REQ-ID}"** | 移到 archive/ | `归档 REQ-001` |
| **"导出项目 {名}"** | 打包为 zip | `导出项目 销售分析` |
| **"导入知识库"** | 从 zip 导入 | `导入知识库` |
| **"开启/关闭自动记录"** | 切换智能任务追踪 | `开启自动记录` |
| **"重新初始化"** | 重置配置 | `重新初始化` |
| **"调整检测灵敏度"** | 修改自动触发阈值 | `调整检测灵敏度` |

## 意图识别（同义表达映射）

当用户说的话**不在固定命令表里**，但意图明显时，按下方映射处理：

| 用户意图 | 触发词示例 | 对应操作 |
|----------|-----------|----------|
| 创建类 | "记录"、"加个"、"写下"、"建一个"、"我要做" | → 新建需求 |
| 更新类 | "改"、"更新"、"编辑"、"备注"、"补充" | → 更新现有 REQ |
| 完成类 | "完事"、"OK 了"、"好了"、"交付了"、"结了" | → 状态转 Done |
| 查看类 | "看"、"显示"、"列出来"、"现在怎样"、"多少个" | → 读取状态 |
| 归档类 | "归档"、"收起来"、"放 archive" | → 移到 archive/ |
| 搜索类 | "搜"、"找"、"有没有"、"在哪" | → 关键词检索 |

## 模糊时的确认话术

**不要瞎猜**。如果用户的话不在以上任何一类，按下方模板追问：

```
🤔 你的意思是？

[创建]   创建一个新的 REQ 记录
[查看]   查看某需求的当前状态
[更新]   更新某需求的内容或状态
[归档]   把已完成的需求移到 archive/

或者你可以直接说："新建需求 xxx" / "查看 REQ-001" / "项目进展如何"
```

## 能力范围速览（避免越界）

**本 Skill 擅长**：
- 创建和管理本地知识库目录结构
- 跟踪 REQ-* 需求的全生命周期
- 自动把对话追加到当前活跃需求
- 在 PROJECT-STATUS.md 维护项目快照

**本 Skill 不做**（不要尝试）：
- ❌ 联网/同步到云端
- ❌ 多用户协作
- ❌ 修改 ~/.myknowledge/ 之外的文件
- ❌ 执行任意 shell 命令
- ❌ 富文本/所见即所得编辑

详见 README.md 的"⚠️ 能力边界"小节。
