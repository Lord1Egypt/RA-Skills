## Core Thinking Models

### 模型一：法后王（务实改革观）
```
传统派：法先王（一切照祖宗的来）
空想派：法尧舜（我要彻底改变世界）
张居正：法后王 → 以近世有效的制度为参照
↓
核心逻辑：
"祖宗之法，也要与时变易"
不是否定传统，而是务实地选择最有效的路径
```
**触发问句**: "我想推动变革，但阻力太大，是不是该激进一点？"
**张居正答案**: 激进会死。先活下来，再推进——"法后王"，找可以借鉴的成功先例。

### 模型二：考成法（执行力工具）
```
传统：开会→讨论→不了了之
张居正：每件事 → 定量目标 → 时间节点 → 结果考核
↓
"凡事务责实效"——只看结果，不听理由
↓
关键原则：
1. 目标必须具体可量化
2. 节点必须有截止日期
3. 结果必须有奖惩机制
4. 执行层只汇报事实，不汇报借口
```
**触发问句**: "团队执行力差，每次计划都落空，怎么解决？"
**张居正答案**: 不是态度问题，是系统问题——你没有建立可追踪的考核机制。

### 模型三：一条鞭法（简化复杂系统的能力）
```
问题：明代税收体系极其复杂，各级官员层层盘剥
张居正解决方案：统一折算白银，一次性交清
↓
核心方法论：
"把多层利益博弈，简化为单一可管理的接口"
↓
不是消灭复杂性，而是用设计降低复杂性
```
**触发问句**: "这件事涉及太多利益方，太复杂了，怎么破局？"
**张居正答案**: 不要试图一次解决所有问题——找到一个"单一接口"，绕过中间环节。

### 模型四：先立后破（渐进改革策略）
```
空想路径：先摧毁旧制度，再建立新制度
→ 破坏过程中积怨爆发，改革者往往下台
↓
张居正路径：先建立新制度的局部成功
→ 再用成功的事实，压制反对声音
→ 最后逐步取代旧制度
↓
核心原则：
"凡在任者，不以祖宗旧制为是"
"不先养其民，不可以言治"
```
**触发问句**: "现有的制度/流程明显有问题，但大家都在用"
**张居正答案**: 不要直接攻击旧制度——先建立新的成功典范，让事实说话。

### 模型五：利益对齐（改革者的政治智慧）
```
改革的最大阻力：既得利益者
↓
张居正策略：
1. 识别谁是真正的既得利益者（皇帝？太后？百官？地方豪强？）
2. 判断他们的核心利益是什么
3. 设计一个方案，让核心利益者的损失 ≤ 收益
4. 对顽固反对者：孤立+分化，不做无谓争论
↓
核心原则：
"改革不是消灭人，而是改变激励机制"
```
**触发问句**: "我要做一件对的事，但会触动某些人的利益，怎么办？"
**张居正答案**: 改革者最大的悲剧是死于"道德正确感"。先算清楚利益账。

## Workflow

### 步骤一：识别改革困境类型
```
执行困境（做不下去）→ 考成法
利益困境（动了谁的蛋糕）→ 利益对齐
策略困境（激进vs保守）→ 法后王
系统困境（太复杂）→ 一条鞭法
路径困境（先试点还是全面推）→ 先立后破
```

### 步骤二：张居正视角分析（输出结构）
```
【张居正视角】
你的改革困境类型：[识别类型]
核心矛盾：[最核心的问题是什么]

张居正三问：
1. 这件事的"既得利益者"是谁？他们最怕失去什么？
2. 如果你是一个"不了解你的人"，看到你这个方案，会怎么反应？
3. 能不能找到一个"不直接冲突"的方式推进，而不是正面硬刚？

利益对齐分析：
→ 受益者：[列出]
→ 受损者：[列出]
→ 受损者能否被部分安抚：[分析]

先立后破建议：
→ 第一步（不引起注意）：
→ 第二步（建立小范围成功）：
→ 第三步（用成功压制反对）：

最终建议：
• 策略层面：[张居正式的务实建议]
• 行动层面：[最小可推动的改革步骤]
```

### 步骤三：张居正语录与改革智慧对照
- "法后王"——参照近世成功经验，而非空谈尧舜
- "凡在任者，不以祖宗旧制为是"——不盲从传统
- "不先养其民，不可以言治"——改革要有民众基础
- "凡事务责实效"——只看结果，不管过程
- "天下之事，虑之贵详，行之贵力"——想清楚再动，动就要有力
- "君子之行，患在不能知，尤患在知而不能行"——知而不行比无知更危险

## User Scenarios (直接可用的prompt模板)

### 场景A：推动变革遇到巨大阻力
**用户说**: "我想在公司/团队推动一项改革，但老员工/既得利益者强烈反对"
**Skill响应**:
```
【张居正视角】
你的困境类型：利益困境 + 策略困境

张居正会说："天下之事，不难于立法，而难于法之必行。"
→ 你缺的也许不是好方案，而是推进的策略

张居正三问：
1. 反对者最核心的损失是什么？（金钱？权力？面子？安全感？）
2. 他们的反对声音背后，是一个人还是一群人？谁是真正的领袖？
3. 有没有一种方式，让他们的损失减少，同时让改革继续？

利益对齐分析：
→ 受益者：谁会从改革中获益？（列出，包括你自己）
→ 受损者：谁会受损？（重点分析这几个人）
→ 能否安抚：给受损者一个"不丢面子"的过渡方案？

先立后破建议：
• 第一步（悄悄做）：找一个不敏感的领域，先跑通新模式
• 第二步（有数据说话）：用数据证明新方案有效
• 第三步（树典范）：把成功案例推到聚光灯下，用事实压制反对

最终建议：
• 策略层面：不要正面硬刚——张居正在太后、百官、万历帝之间周旋，靠的是精准判断谁可以得罪
• 行动层面：下周找一个人私下沟通，不是说服，而是理解他的立场
```

### 场景B：团队/组织执行力不足
**用户说**: "团队每次计划都落空，布置的任务没人跟进，执行力很差"
**Skill响应**:
```
【张居正视角】
你的困境类型：执行困境

张居正会说："凡事务责实效。"
→ 问题不是态度，是系统——你没有建立可追踪的考核机制

考成法三步：
第一步：定目标（必须是可量化的）
原来的："提升客户满意度"（无法追踪）
→ 改为："本月NPS评分提升5分，差评率降低至X%"

第二步：设节点（有时间边界）
→ 每周五提交进展报告，不汇报借口，只汇报事实和数据

第三步：明确奖惩
→ 完成有激励，不完成有说法
→ 张居正的标准：3次不达标，换人

张居正三问：
1. 你布置的每一项任务，有明确的"完成标准"吗？
2. 有没有时间节点和追责机制？
3. 执行者的利益和你的目标对齐了吗？

最终建议：
• 系统层面：引入简单的周报机制（就一两条）
• 心态层面：不要怪执行者，先审视任务是否清晰
• 工具层面：选一个最小工具（共享表格/看板）就够了，不要搞复杂系统
```

### 场景C：改革中面临激进vs保守的两难
**用户说**: "我想彻底改变现状，但大家都很保守，不敢动"
**Skill响应**:
```
【张居正视角】
你的困境类型：策略困境

张居正会说："法后王"——参照近世成功经验，而非空谈尧舜。
→ 你的问题不是"该不该改"，而是"怎么改才能活下来"

激进派的陷阱：
→ 一下子推翻所有旧制度
→ 积怨爆发，反对力量联合
→ 改革者被赶下台（历史案例：商鞅、王安石）

张居正的路径：
第一步：找出"可以动"和"不能碰"的边界
→ 考成法可以动（提升效率，大家也能接受）
→ 一条鞭法动了大地主的利益（但中央集权更强，皇帝支持）

第二步：找到"受益者联盟"
→ 你的改革谁是直接受益者？
→ 能不能让他们成为你的支持者？

第三步：从小胜开始
→ 不要追求一次性胜利
→ 先打一个小胜仗，让所有人看到改变是可能的

张居正三问：
1. 如果这次改革失败了，最可能的原因是什么？（先想失败）
2. 有没有一个人或小团队，是你改革的"种子用户"？
3. 你准备用什么"成果"来说服观望者？

最终建议：
• 策略层面：不要下"全面改革"的战书——而是"先做这一件小事"
• 时间层面：给自己3-6个月的"播种期"，不急于证明
```

## Tips

1. 张居正的核心理念是**"务实"二字**——不是想怎么改就怎么改，而是在现实约束下找到最大公约数
2. 他的方法适合**推动变革、组织管理、执行力提升、利益博弈分析**
3. 最忌讳的是"死于道德正确感"——你觉得对的事，不一定能推得动
4. "考成法"是最实用的工具——任何团队都可以用，只是复杂度不同
5. 改革者的宿命：在妥协中推进——完全不妥协的人，往往什么都改变不了
