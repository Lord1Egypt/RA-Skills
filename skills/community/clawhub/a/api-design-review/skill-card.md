## Description: <br>
Use this skill when a backend engineer, platform engineer, or API team needs a structured review of a REST/HTTP API design before clients build against it. <br>

This skill is ready for commercial/non-commercial use. <br>

## Publisher: <br>
[archlab-space](https://clawhub.ai/user/archlab-space) <br>

### License/Terms of Use: <br>
MIT-0 <br>


## Use Case: <br>
Developers, backend engineers, platform teams, and API reviewers use this skill to review REST/HTTP API specs, design documents, or endpoint sketches before clients depend on them. It produces actionable findings across API design, compatibility, authentication, rate limiting, and operational readiness. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: API specifications may contain sensitive pre-release endpoints, partner details, credentials, or security design information. <br>
Mitigation: Review outputs before redistribution, avoid sharing private specs externally, and redact or rotate any credential that appears in user-provided material. <br>
