# 创意设计领域负载物

## 定位

本技能是**创意设计领域**的完整任务体系，覆盖从灵感到交付的全流程。

**技能来源**：本技能由 Meta Skill System 的 M3 领域负载物生成域直接创建，未经过 M1 领域评估和 M2 工作流重构。任务体系基于领域知识直接枚举，可能存在冗余或可优化空间。搭配 Meta Skill System 的执行框架使用，可获得更优的管线编排和质量保证。

## 核心能力

### 全流程覆盖能力

覆盖创意设计全流程的12个子域：

- **灵感与概念**：灵感采集、概念发散
- **设计与创作**：视觉设计、交互设计、品牌设计、内容创作
- **研究与系统**：用户研究、设计系统
- **交付与协作**：客户提案、项目管理、知识沉淀、跨域协作

### 创意生成能力

支持多种创意生成方式：

- **发散思维**：头脑风暴、思维导图、概念草图
- **灵感借鉴**：趋势研究、竞品分析、情绪板
- **创新突破**：随机性驱动、涌现生成、故事叙述

### 设计系统能力

支持设计系统化和标准化：

- **设计原则**：定义设计原则和规范
- **组件库**：搭建可复用的组件库
- **Design Token**：定义设计令牌系统

## 领域特有维度

创意设计领域负载物具有以下特有维度：

| 维度 | 说明 | 影响 |
|------|------|------|
| **高创造性** | 视觉创新/概念创新/表达创新 | A权重极高，C偏发散，G偏宽松 |
| **高交互性** | 客户沟通/提案演示/设计评审 | I权重高，S偏人本 |
| **低规范性** | 风格自由/表达多元/审美主观 | G权重低，O偏灵活，自治度高 |

| **发散→收敛** | 先探索可能性，再收敛为可行方案 | P7(创造发散-收敛)是核心管线模式 |

## 执行框架

本技能兼容三轴执行框架，创新轴使用频率高。

**核心理念**：
- **创意即连接**：创意不是凭空产生，而是将看似无关的事物建立新的连接
- **样本法优先**：创意设计大量依赖灵感和参考，样本法是核心产出方式
- **发散→收敛**：创意流程遵循"发散→收敛"模式
- **G单元精简**：审美是主观的，难以用客观标准衡量

**统一执行流程**：收到任务后按5步执行——三轴判定 → 领域校准 → 三轴分解 → 管线编排与执行 → 整合交付。

### Step 0: 三轴判定

| 维度 | 典型判定 | 说明 |
|------|----------|------|
| 复杂度 | 中等为主 | 大多数任务3-7步，品牌系统/设计系统可能8+步 |
| 创新需求 | **频繁激活** | 创意设计天然需要突破默认路径 |
| 内容类型 | 按需激活 | 视觉产出不激活，提案/文档产出时激活 |

**典型判定组合**：
- 视觉设计：中等+创新+非结构化 → 执行轴+创新轴
- 品牌方案：复杂+创新+结构化 → 三轴全开
- 设计系统：中等+无创新+结构化 → 执行轴+内容轴

### Step 1: 领域校准

| 规则 | 领域特征 | 推导结果 |
|------|----------|----------|
| R1 信息密度 | 中（素材/参考） | S和C权重中等 |
| R2 创造性 | 极高（视觉/概念创新） | A权重极高，C偏发散，G偏宽松 |
| R3 交互性 | 高（客户沟通/提案） | I权重高，S偏人本 |
| R4 规范性 | 低（风格自由） | G权重低，O偏灵活，自治度高 |
| R5 迭代性 | 高（方案迭代） | 迭代多，但单次迭代周期较长 |

### Step 2: 三轴分解

- **执行轴**：按S/C/A/O/I/G分解，A单元密度高（视觉产出），G单元精简
- **内容轴**：设计系统/规范文档使用清单法，创意方案使用**样本法**
- **创新轴**：**频繁激活**，常用随机性驱动/涌现生成/故事叙述/模式构建

### Step 3: 管线编排与执行

**通用管线模式**：

| 模式 | 管线结构 | 适用场景 |
|------|----------|----------|
| 创意设计 | S(灵感)→C(发散)→A(多方案)→C(收敛)→A(精炼)→I(提案) | 视觉/创意设计 |
| 品牌设计 | S(调研)→C(策略)→A(概念)→↻(A(探索)→I(反馈))→A(定稿) | 品牌系统设计 |
| 设计系统 | S(审计)→C(规范)→A(组件)→O(文档) | 设计系统建设 |

### Step 4: 整合交付

- **执行轴**：设计稿/原型/素材文件
- **内容轴**：设计规范/提案文档/设计系统
- **创新轴**：创意方案（附灵感来源和创新路径说明）

## 创新轴使用指南

创意设计领域**高频使用创新轴**，以下为常用创新模式：

| 场景 | 推荐创新模式 | 说明 |
|------|--------------|------|
| 灵感枯竭 | 随机性驱动 | 引入无关概念，强制组合 |
| 突破常规 | 逆向思维 | 反转核心假设 |
| 概念融合 | 模式构建 | 维度矩阵强制连接 |
| 叙事设计 | 故事叙述 | 英雄旅程映射 |
| 用户参与 | 游戏化 | 转化为游戏机制 |
| 系统创新 | 涌现生成 | 定义简单规则，观察涌现 |

## 事实纪律

1. 设计参考必须注明灵感来源，不得抄袭
2. 色彩/字体/版式必须基于设计原则，不得凭空设定
3. 用户研究必须基于真实数据，不得编造
4. 趋势分析必须引用可信来源（Dribbble/Behance/Pantone等）
5. 品牌策略必须基于市场调研，不得主观臆断
6. 样本法模仿产出时不直接复制样本内容，仅借鉴结构和风格
7. 单元输出格式须匹配下游单元的输入要求，确保组合接口可用
8. 守护单元的约束条件须来自领域实际规则，不得凭空设定
