## 一、功能测试

| 测试场景 | 测试要点 |
|---------|---------|
| GET请求 | 正常查询单条/列表数据；带参数查询；参数为空/类型错误处理 |
| POST请求 | 正常创建资源；必填字段校验；字段格式校验；重复创建处理 |
| PUT/PATCH请求 | 完整更新/部分更新；更新不存在资源；并发更新处理 |
| DELETE请求 | 正常删除；删除不存在资源；级联删除；软删除 |
| 批量操作 | 批量创建/更新/删除；部分成功处理；事务回滚 |

## 二、状态码验证

| 状态码 | 说明 |
|-------|------|
| 200 OK | 正常响应 |
| 201 Created | 创建成功 |
| 204 No Content | 删除成功 |
| 400 Bad Request | 参数错误 |
| 401 Unauthorized | 未认证 |
| 403 Forbidden | 无权限 |
| 404 Not Found | 资源不存在 |
| 409 Conflict | 资源冲突 |
| 422 Unprocessable | 校验失败 |
| 500 Internal Error | 服务器错误 |

## 五、安全测试

| 测试场景 | 测试要点 |
|---------|---------|
| 注入攻击 | SQL注入（参数化查询验证）、NoSQL注入、命令注入 |
| 越权访问 | 水平越权（IDOR，修改资源ID访问他人数据）、垂直越权（普通Token访问管理接口） |
| 批量赋值 | 请求体中添加只读字段（如 role、isAdmin、balance），验证是否被拒绝 |
| API限流 | 短时间内大量请求（>100次/秒），验证是否返回 429 |
| 敏感信息泄露 | 响应中是否返回密码哈希、Token、内部IP等敏感字段；错误堆栈是否暴露框架版本 |
| 安全头检查 | X-Content-Type-Options、X-Frame-Options、CSP、HSTS 等安全响应头 |
| JWT安全 | 无签名Token（alg=none）、过期Token、篡改Payload、密钥爆破 |
| 参数污染 | 重复参数（?id=1&id=2）、数组注入（?id[]=1&id[]=2）、HTTP Parameter Pollution |
