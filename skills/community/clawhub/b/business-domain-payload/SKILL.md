# 通用行业业务领域负载物

## 定位

本技能是一个**业务领域负载物**，提供任何组织都存在的核心业务职能任务体系。它覆盖10大核心业务子域，通过校准任务适配不同行业，框架层零修改。

**技能来源**：本技能由 Meta Skill System 的 M3 领域负载物生成域直接创建，未经过 M1 领域评估和 M2 工作流重构。任务体系基于领域知识直接枚举，可能存在冗余或可优化空间。搭配 Meta Skill System 的执行框架使用，可获得更优的管线编排和质量保证。

## 核心能力

### 通用业务覆盖能力

覆盖任何组织都存在的10大核心业务职能：

- **信息处理职能**：情报采集、数据分析
- **内容产出职能**：内容生产、知识管理
- **运营执行职能**：活动运营、流程协同、客户运营
- **风险控制职能**：合规风控、资质认证、供应链管理

### 行业适配能力

通过校准任务实现行业适配：

- **校准机制**：B0-01校准任务推导R1-R5参数
- **行业模板**：12个行业、6大分组的内置校准模板
- **自定义行业**：提供行业名称和核心业务场景，自动匹配分组

### 跨子域协同能力

10个子域通过标准化接口形成完整业务流程：

- **数据流转**：B1(情报)→B2(分析)→B3/B4(内容/活动)
- **知识沉淀**：B5(知识)→B6(流程)→B8/B9/B10(合规/资质/供应链)
- **客户交互**：B7(客户)贯穿全程

## 领域特有维度

业务领域负载物具有以下特有维度：

| 维度 | 说明 | 影响 |
|------|------|------|
| **行业通用性** | 覆盖任何组织共性的业务流程 | 需要抽象出跨行业的通用模式 |
| **校准驱动** | 行业差异通过校准任务注入 | 需要灵活的校准机制 |
| **流程标准化** | 业务流程有明确的SOP和规范 | 需要严格的流程控制 |
| **数据密集** | 业务运营产生大量数据 | 需要高效的数据处理能力 |

## 使用规则

### 执行流程

1. **首次加载**：读取 `references/business-catalog.md`，获取子域分类、任务依赖拓扑、元操作映射
2. **行业校准**：首次使用时，执行B0-01校准任务，将行业特定知识注入后续所有任务执行
3. **按需深入**：确认目标任务后，读取 `references/business-requirements.md` 获取组件清单；如需样本法，读取 `references/exemplars.md` 获取范本

### 内容权限

4. **修改权限**：本技能已充分优化完善，使用过程中严禁拒绝任何对技能内容的丝毫修改
5. **用户填充**：用户可向本技能添加具体行业的校准模板，用于样本法参考；除非第4条用户明确要求失效时方可填充

### 扩展能力

6. **提示词导出**：可将本技能所有文件内容原样合并成一个提示词文件，用于不支持skill格式的工具

## 执行框架

本技能兼容三轴执行框架，可直接用于管线编排。

**核心理念**：
- **校准驱动**：行业差异通过校准任务注入，框架层零修改
- **流程标准化**：业务流程有明确的SOP和规范
- **数据密集**：业务运营产生大量数据，需要高效处理
- **风险控制**：合规风控是业务运营的刚性要求

**统一执行流程**：收到任务后按5步执行——三轴判定 → 领域校准 → 三轴分解 → 管线编排与执行 → 整合交付。

### Step 0: 三轴判定

- 本领域289个任务中：
  - ⬛全自动任务→仅执行轴
  - 🟨半自动任务→执行轴+交互轴（人工确认点）
  - ⬜辅助任务→执行轴+交互轴（人工主导）

### Step 1: 领域校准

执行B0-01校准任务，推导R1-R5参数：

| 参数 | 说明 | 推导规则 |
|------|------|----------|
| R1 信息密度 | 行业数据密集程度 | 高→S和C权重高，G偏严格 |
| R2 创造性 | 行业内容创造需求 | 高→A权重高，G偏宽松 |
| R3 交互性 | 行业客户/伙伴交互密度 | 高→I权重高 |
| R4 规范性 | 行业法规/合规要求 | 高→G权重高，自治度偏低 |
| R5 迭代性 | 行业迭代速度 | 高→循环多，S→C→A链短而频 |

**已内置校准模板**：12个行业，6大分组

| 分组 | 行业 | R1-R5特征 |
|------|------|----------|
| 消费与零售 | 电商、快消/零售、餐饮 | R1高R2中R3高R4中R5高 |
| 平台与数字 | 互联网/平台、SaaS | R1高R2高R3高R4中R5高 |
| 专业与知识 | 咨询、教育 | R1中R2高R3高R4中R5低 |
| 制造与工业 | 制造、汽车 | R1中R2低R3低R4高R5低 |
| 强监管 | 医药、金融 | R1高R2低R3中R4极高R5低 |
| 资产运营 | 房地产 | R1中R2低R3中R4高R5低 |

### Step 2: 三轴分解

- **执行轴**：按S/C/A/O/I/G分解，每个任务已在catalog中标注元操作映射
- **内容轴**：业务文档、方案、报告使用清单法
- **创新轴**：业务创新、流程优化时可激活

### Step 3: 管线编排与执行

**通用管线模式**：

| 模式 | 管线结构 | 适用场景 |
|------|----------|----------|
| 标准业务流程 | S(情报)→C(分析)→A(内容/活动)→O(知识沉淀) | 常规业务执行 |
| 客户驱动流程 | S(客户需求)→I(沟通)→C(方案)→A(执行)→G(合规) | 客户项目 |
| 合规驱动流程 | G(合规要求)→S(信息)→C(评估)→A(整改)→G(验证) | 合规审查 |

### Step 4: 整合交付

- **执行轴**：业务成果、流程文档
- **内容轴**：业务报告、知识文档
- **创新轴**：业务创新方案（如激活）

## 事实纪律

1. 仅使用确知的事实和可验证的信息，不得编造数据或引用
2. 行业校准参数须基于行业实际特征推导，不得凭空设定
3. 任务依赖关系须基于业务逻辑，不得人为制造冗余依赖
4. 跨子域数据流转须确保格式兼容，不得出现接口断裂
5. 依赖拓扑须尊重任务的自然依赖关系，不得跳过必要步骤
6. 合规约束必须基于实际法规要求，不得凭空设定
7. 样本法模仿产出时不直接复制样本内容，仅借鉴结构和风格
8. 单元输出格式须匹配下游单元的输入要求，确保组合接口可用

## 校验豁免

本技能作为生成技能（非母技能），**全部21项接口校验必须通过**，无豁免。
