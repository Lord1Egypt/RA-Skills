## A2 任务委派域 范本

### 范本概述
- **域ID**: A2
- **域名称**: 任务委派域
- **任务数量**: 5
- **范本状态**: 可用

### 任务执行范本

#### A2-01 任务分析
**执行步骤**:
1. 接收任务，明确任务目标
2. 分析任务特征
3. 识别任务需求
4. 确定任务约束
5. 输出任务分析结果

**输入示例**:
- 任务：开发用户登录功能
- 任务目标：实现用户登录、注册、密码重置功能
- 任务来源：产品经理

**输出示例**:
- 任务特征：中等复杂度，需要前后端开发
- 任务需求：需要UI设计、前端开发、后端开发、测试
- 任务约束：需要在1周内完成，代码质量要高
- 任务分析结果：任务需要多个Agent协作完成

**注意事项**:
- 任务特征要准确，避免误解
- 任务需求要全面，避免遗漏
- 任务约束要明确，避免模糊

#### A2-02 Agent匹配
**执行步骤**:
1. 制定匹配标准
2. 选择匹配方法
3. 执行Agent匹配
4. 确认匹配结果
5. 输出匹配报告

**输入示例**:
- 任务需求：需要UI设计、前端开发、后端开发、测试
- 可用Agent：Explore Agent、Plan Agent、General-purpose Agent
- 匹配标准：任务特征与Agent能力匹配

**输出示例**:
- 匹配标准：任务特征与Agent能力匹配
- 匹配方法：能力匹配、场景匹配、资源匹配
- 匹配结果：
  - UI设计：Plan Agent（规划能力强）
  - 前端开发：General-purpose Agent（执行能力强）
  - 后端开发：General-purpose Agent（执行能力强）
  - 测试：Explore Agent（搜索能力强）

**注意事项**:
- 匹配标准要客观，避免主观臆断
- 匹配方法要有效，避免无效匹配
- 匹配结果要准确，避免错误

#### A2-03 委派策略制定
**执行步骤**:
1. 选择委派方式
2. 设定委派参数
3. 制定委派策略
4. 确认委派策略
5. 输出委派策略文档

**输入示例**:
- 任务：开发用户登录功能
- 匹配结果：UI设计=Plan Agent，前端开发=General-purpose Agent，后端开发=General-purpose Agent，测试=Explore Agent
- 委派需求：需要合理的委派策略

**输出示例**:
- 委派方式：同步委派（需要立即结果）
- 委派参数：超时时间=30分钟，重试次数=3次
- 委派策略：
  - UI设计：Plan Agent，同步委派
  - 前端开发：General-purpose Agent，异步委派
  - 后端开发：General-purpose Agent，异步委派
  - 测试：Explore Agent，同步委派

**注意事项**:
- 委派方式要合适，避免不合适的方式
- 委派参数要合理，避免不合理的参数
- 委派策略要明确，避免模糊

#### A2-04 任务委派执行
**执行步骤**:
1. 准备委派目标
2. 准备委派内容
3. 执行任务委派
4. 确认委派状态
5. 输出委派结果

**输入示例**:
- 委派目标：Plan Agent、General-purpose Agent、Explore Agent
- 委派内容：UI设计、前端开发、后端开发、测试任务
- 委派策略：同步委派、异步委派

**输出示例**:
- 委派目标：Plan Agent、General-purpose Agent、Explore Agent
- 委派内容：已委派UI设计、前端开发、后端开发、测试任务
- 委派状态：成功
- 委派结果：所有任务已成功委派

**注意事项**:
- 委派目标要明确，避免错误
- 委派内容要准确，避免虚假
- 委派状态要及时更新，避免延迟

#### A2-05 委派状态跟踪
**执行步骤**:
1. 设定跟踪指标
2. 选择跟踪方法
3. 执行状态跟踪
4. 分析跟踪结果
5. 输出跟踪报告

**输入示例**:
- 委派任务：UI设计、前端开发、后端开发、测试
- 跟踪需求：跟踪任务委派状态

**输出示例**:
- 跟踪指标：任务状态、进度、问题
- 跟踪方法：实时跟踪、定期检查、问题反馈
- 跟踪结果：
  - UI设计：进行中，进度50%，无问题
  - 前端开发：进行中，进度30%，遇到技术问题
  - 后端开发：进行中，进度40%，无问题
  - 测试：未开始，等待开发完成

**注意事项**:
- 跟踪指标要全面，避免遗漏
- 跟踪方法要有效，避免无效跟踪
- 跟踪结果要准确，避免错误
