# 人设卡 Schema（标准契约）

## 这份文件是什么

人设卡（Persona Card）是三库内容工作流的**唯一变量**。三库（流量因子 / 产品资产 / 烹调方法）是通用骨架，人设卡是注入它的"风味与边界"。

人设卡即 **CSO 平台的 styleGuide**（一段 Markdown）。本文件定义这份 styleGuide 的**标准结构**——它与 CSO 平台 AI 生成人设时使用的章节**完全一致**。所有创建方式（反推 / 描述 / 导入 / 临时）的产物，落盘前都必须符合这套章节结构，否则主 SOP 无法稳定消费。

> ⚠️ 关键约束：styleGuide 是**松散 Markdown 章节**，不是结构化字段。**不要**在文档或 SOP 里引用形如 `persona.style_tags`、`persona.core_task`、`persona.opening_preferences` 这种平台并不存在的"字段路径"。一切信息都从下面 6 个章节里**读取或推断**。

## 存储位置（统一走 CSO 平台）

- **正式人设卡**：保存在 CSO 平台。
  - 拉取：`siluzan-cso persona list --name <名称> --json-out ./snap-cso`，落盘后脚本读 `styleGuide` 字段（见 `references/core/tips.md`）
  - 保存：先把符合本结构的 markdown 写到临时文件，再 `siluzan-cso persona create --name <名称> --style-guide-file <path>`
  - 命令细节见 `references/persona.md`
- **临时人设**（单次写稿用，不落盘）：仅存在会话内存中，不写平台

> 平台只有 `persona create`（新建），**没有原地 update**。修改 / 反哺已有人设时，合并内容后**存为新版本**（名称可加版本后缀）。

## styleGuide 标准结构（6 个章节）

styleGuide 必须包含以下 6 个 `##` 章节，**章节标题不可省略、不可改名**（与 CSO 平台生成口径一致）：

| 章节                          | 内容                                                                                                           | 必填 |
| ----------------------------- | -------------------------------------------------------------------------------------------------------------- | ---- |
| `## 一句话定位`               | 账号核心定位 / 价值主张（一句话）                                                                              | ✅   |
| `## 目标受众`                 | 受众画像：职业 / 阶段 / 关心的问题                                                                             | ✅   |
| `## 内容 DNA`                 | 4 条要点：开场惯用方式 / 标志性表达 · 口头禅 / 节奏 / 价值观锚点                                               | ✅   |
| `## 推荐用词（≥6 个）`        | 高频/优先使用的词条（注入正文用）                                                                              | ✅   |
| `## 禁用表达 / 禁忌（≥3 条）` | 绝不能用的词、句式、话题禁区                                                                                   | ✅   |
| `## <平台> 平台适配建议`      | 针对主运营平台的内容形态、长度、互动、SEO/封面要点；标题里的 `<平台>` 替换为真实平台名（如「抖音」「公众号」） | ✅   |

### `## 内容 DNA` 的 4 条要点（固定）

```markdown
## 内容 DNA

- 开场惯用方式：<这个号习惯怎么起开头>
- 标志性表达 / 口头禅：<反复出现的句式、口头禅、标志性比喻>
- 节奏：<快 / 中 / 慢；句子长短与推进密度>
- 价值观锚点：<这个号始终在捍卫 / 反对的立场>
```

> 「内容 DNA」是腔调注入与质检的核心来源：语气、句式、节奏、立场都从这 4 条读取。

## 派生维度：从章节里"推断"，不要当字段

主 SOP 的很多机制（风格倾向选模板、结构推进、质检尺子、群体共鸣）需要的信息，**真实人设里没有独立字段**，必须从上面 6 个章节**推断**，推不出来就用库内**通用默认**：

| 主 SOP 需要的维度                | 从哪个章节推断                                      | 推不出来时                                    |
| -------------------------------- | --------------------------------------------------- | --------------------------------------------- |
| 风格倾向（选产品资产模板）       | 「内容 DNA」+「一句话定位」                         | 用通用模板，提示用户                          |
| 核心任务（判断/方法/灵感/转化…） | 「一句话定位」（定位即任务）                        | 从定位提炼                                    |
| 语气 / 节奏                      | 「内容 DNA · 标志性表达·口头禅」「内容 DNA · 节奏」 | 库内中性默认                                  |
| 开场偏好                         | 「内容 DNA · 开场惯用方式」                         | 库内通用开场                                  |
| 中段推进 / 结尾偏好              | 「内容 DNA · 节奏」（无显式结尾字段）               | 库内通用结构（现象→本质 + 行动提醒）          |
| 常用句式 / 比喻                  | 「内容 DNA · 标志性表达·口头禅」                    | 中性默认句式 + 标"建议补充"                   |
| 高频关键词                       | 「推荐用词」                                        | —                                             |
| 选题方向 / 行业                  | 「一句话定位」+「目标受众」                         | 不限，提示风险                                |
| 选题禁区 / 禁用表达              | 「禁用表达 / 禁忌」                                 | 无（按通用红线）                              |
| 质检维度                         | 「价值观锚点」+「一句话定位」提炼                   | 用通用质检清单（见 `sop.md` 第 10 步）        |
| 群体共鸣锚点（TF-0028）          | 无独立字段                                          | 写稿时跑实时三步生成法（见 `sop.md` 第 5 步） |
| 主战平台                         | 「`## <平台> 平台适配建议`」的章节标题              | 写稿前问用户本次发哪个平台                    |

## 完整 styleGuide 模板

```markdown
## 一句话定位

<一句话写清账号核心定位 / 价值主张>

## 目标受众

<受众画像：职业 / 阶段 / 关心的问题>

## 内容 DNA

- 开场惯用方式：<…>
- 标志性表达 / 口头禅：<…>
- 节奏：<快 / 中 / 慢，及句子特征>
- 价值观锚点：<…>

## 推荐用词（≥6 个）

- <词1>
- <词2>
- <…共 ≥6 个>

## 禁用表达 / 禁忌（≥3 条）

- <禁忌1>
- <禁忌2>
- <禁忌3>

## <平台> 平台适配建议

<针对该平台的内容形态、长度、互动建议、SEO/封面要点>
```

约束（与 CSO 平台口径一致）：

1. 整体不超过 ~800 字（太长会让宿主 system_prompt 占用爆炸）。
2. 风格指纹要具体，不要"专业、有趣、深度"这种空话。
3. 推荐用词与禁用表达给到具体词条，不要给抽象类目。
4. 用第二人称「你」对写作者讲话。
5. 直接 markdown，不要 code fence 包裹。

> 落盘时把以上 markdown 写入临时文件，用 `siluzan-cso persona create --name <name> --style-guide-file <path>` 保存。

## 与主 SOP 的章节映射

| 主 SOP 步骤        | 读取的 styleGuide 章节（或推断来源）                                                  |
| ------------------ | ------------------------------------------------------------------------------------- |
| 第 1 步 加载人设   | 全部 6 章节（复述）                                                                   |
| 第 2 步 定目标     | 「一句话定位」                                                                        |
| 第 2 步 加载库     | 「`## <平台> 平台适配建议`」标题 → 决定挂主库还是赛道包                               |
| 第 3 步 选题       | 「一句话定位」+「目标受众」（推断方向）；「禁用表达 / 禁忌」（排除）                  |
| 第 5 步 找流量因子 | 「目标受众」；TF-0028 用「价值观锚点」+「目标受众」实时推导                           |
| 第 6 步 配产品资产 | 「内容 DNA」+「一句话定位」（推断风格倾向）                                           |
| 第 7 步 选烹调方法 | 「内容 DNA · 开场惯用方式 / 节奏」（推断结构偏好）                                    |
| 第 9 步 写正文     | 「内容 DNA · 标志性表达·口头禅 / 节奏 / 价值观锚点」+「推荐用词」+「禁用表达 / 禁忌」 |
| 第 10 步 质检      | 「价值观锚点」+「一句话定位」提炼质检维度；无则通用清单                               |
