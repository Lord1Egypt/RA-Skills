# Usage Examples / 使用示例

## 1. New Project Intake / 新项目需求访谈

User:

> I want to build a small AI tool for my team. I am not technical. Ask me questions first.

Expected skill behavior:

- Enter New Project Intake mode.
- Ask 6–8 first-round questions.
- Do not write code yet.
- After answers, summarize requirements, identify missing information, define MVP, and produce a development plan.

用户：

> 我想做一个内部 AI 工具，但我不懂技术，你先问我问题。

期望行为：

- 进入“新项目需求访谈”模式。
- 先问 6–8 个第一轮问题。
- 不直接写代码。
- 用户回答后，总结需求、识别缺失信息、定义 MVP，并生成开发计划。

---

## 2. Mid-Project Realignment / 项目中期方向校准

User:

> The project is halfway done. I keep adding new ideas and now I feel the MVP is too big. Help me recalibrate.

Expected skill behavior:

- Enter Mid-Project Realignment mode.
- Ask about current progress, original goal, drift, new ideas, core user, and core use case.
- Challenge assumptions.
- Identify what to keep, simplify, delay, delete, or validate.
- Generate updated Coding Agent instructions.

用户：

> 项目做到一半，我不断加新想法，现在感觉 MVP 太大了，帮我重新校准。

期望行为：

- 进入“项目中期方向校准”模式。
- 询问当前进度、原始目标、偏离感、新想法、核心用户和核心场景。
- 敢于挑战假设。
- 判断哪些保留、降级、推迟、删除或验证。
- 生成更新后的 Coding Agent 指令。

---

## 3. Code Review & Upgrade Plan / 代码审查与升级优化

User:

> I uploaded a repository. Review it for security, dead code, performance, architecture, and produce an upgrade plan.

Expected skill behavior:

- Enter Code Review & Upgrade Plan mode.
- Inspect project structure first.
- State review scope and blind spots.
- Identify P0/P1 issues first.
- Provide specific fixes, tests, rollback plans, and Coding Agent task queue.

用户：

> 我上传了一个项目代码，帮我检查漏洞、冗余代码、性能、架构，并生成升级计划。

期望行为：

- 进入“代码审查与升级优化”模式。
- 先理解项目结构。
- 说明审查范围与盲区。
- 优先输出 P0/P1 问题。
- 提供具体修复步骤、测试方法、回滚方案和 Coding Agent 任务队列。
