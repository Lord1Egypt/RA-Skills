## 可视化图表生成要求清单

可视化图表生成每种任务类型的"零件清单"——必选/可选组件、组装顺序、设计约束。按清单逐项产出。

---

### 输出格式规范

#### SVG代码结构

```svg
<?xml version="1.0" encoding="UTF-8"?>
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 680 H" width="100%">
  <defs>
    <marker id="arrow" viewBox="0 0 10 10" refX="8" refY="5"
      markerWidth="6" markerHeight="6" orient="auto-start-reverse">
      <path d="M2 1L8 5L2 9" fill="none" stroke="context-stroke"
        stroke-width="1.5" stroke-linecap="round" stroke-linejoin="round"/>
    </marker>
  </defs>

  <!-- 标题 -->
  <text x="340" y="30" text-anchor="middle" class="th">图表标题</text>

  <!-- 图表内容 -->
  <!-- ... -->

</svg>
```

#### HTML代码结构

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>图表标题</title>
    <style>
        body {
            font-family: system-ui, -apple-system, sans-serif;
            margin: 0;
            padding: 20px;
            background: #ffffff;
        }
        /* 样式 */
    </style>
</head>
<body>
    <h1>图表标题</h1>

    <!-- 图表内容 -->
    <div id="chart">
        <!-- ... -->
    </div>

    <script>
        // 交互逻辑
    </script>
</body>
</html>
```


#### V1-02 字体系统应用
- **必选组件**: 标题字体（h1=15px/h2=14px/h3=13px）、正文字体（13px）、辅助文字（12px）、字重（400常规/500粗体）、行高（1.6）
- **字体规范**:

| 元素 | 大小 | 字重 | 行高 |
|------|------|------|------|
| h1 | 15px | 500 | 1.6 |
| h2 | 14px | 500 | 1.6 |
| h3 | 13px | 500 | 1.6 |
| 正文 | 13px | 400 | 1.6 |
| 辅助文字 | 12px | 400 | 1.6 |

- **字体族**:
  - `--font-sans`: system-ui, -apple-system, sans-serif
  - `--font-serif`: Georgia, serif
  - `--font-mono`: Consolas, monospace
- **可选组件**: 字体加载方案、字体回退方案
- **组装顺序**: 元素类型识别→字体大小选择→字重选择→行高设置→字体族应用
- **约束**: 字体大小最小11px；字重只使用400和500
- **格式**: 字体应用方案（Markdown表格）

#### V1-03 布局规则应用
- **必选组件**: 圆角设置（8px/12px/16px）、间距设置（24px内边距/60px最小间距）、安全区域（x=40~640）、viewBox设置（"0 0 680 H"）
- **布局规范**:
  - **圆角**：`--border-radius-md` (8px), `--border-radius-lg` (12px), `--border-radius-xl` (16px)
  - **SVG viewBox**：固定宽度680，高度H根据内容计算
  - **安全区域**：x=40到x=640，y=40到y=(H-40)
  - **最小间距**：节点间60px，节点内24px
- **可选组件**: 响应式布局方案
- **组装顺序**: 图表尺寸计算→viewBox设置→安全区域确定→间距规划→圆角应用
- **约束**: viewBox宽度必须为680；必须在安全区域内绘制；圆角必须使用规范值
- **格式**: 布局方案（Markdown）

#### V1-04 组件样式应用
- **必选组件**: 组件类型选择（box/node/arr/leader）、样式类应用、自定义样式补充
- **预设样式类**:

| 类名 | 用途 | 样式 |
|------|------|------|
| `t` | 主要文字 | sans 14px primary |
| `ts` | 次要文字 | sans 12px secondary |
| `th` | 粗体标题 | sans 14px medium (500) |
| `box` | 中性矩形 | bg-secondary fill, border stroke |
| `node` | 可点击节点 | 带hover效果 |
| `arr` | 箭头线 | 1.5px, 开放箭头 |
| `leader` | 虚线引线 | tertiary stroke, 0.5px |
| `c-{ramp}` | 彩色节点 | 应用到g/rect/circle |

- **箭头标记**（必须包含在SVG的defs中）:
```svg
<defs>
  <marker id="arrow" viewBox="0 0 10 10" refX="8" refY="5"
    markerWidth="6" markerHeight="6" orient="auto-start-reverse">
    <path d="M2 1L8 5L2 9" fill="none" stroke="context-stroke"
      stroke-width="1.5" stroke-linecap="round" stroke-linejoin="round"/>
  </marker>
</defs>
```
- **可选组件**: 自定义样式类、样式覆盖方案
- **组装顺序**: 组件类型识别→样式类选择→自定义样式补充→样式应用
- **约束**: 必须使用预设样式类；箭头标记必须包含在defs中；描边0.5px
- **格式**: 组件样式方案（Markdown）

### V2 SVG生成域

#### V2-01 静态图表生成
- **必选组件**: SVG代码、viewBox设置、元素绘制、文字标注、颜色应用
- **SVG规则**:
  - viewBox固定为 `"0 0 680 H"`
  - width="100%" 在根SVG上
  - H = 最底部元素 + 20px
  - 安全区域：x=40到x=640，y=40到y=(H-40)
  - 背景透明
  - 每个SVG一个工具调用
  - 不旋转文字
  - 使用0.5px描边
  - 连接线必须 `fill="none"`
  - 每个 `<text>` 需要 `dominant-baseline="central"`
- **可选组件**: 图表标题、图例、注释
- **组装顺序**: 图表结构设计→元素定位→颜色应用→文字标注→代码生成
- **约束**: 必须遵循SVG规则；必须应用设计规范
- **格式**: SVG代码文件（.svg）

#### V2-02 流程图生成
- **必选组件**: 流程步骤、步骤间连接、条件分支、开始/结束节点
- **流程图规范**:
  - 单行节点高度：44px
  - 双行节点高度：56px
  - 节点间距：60px最小
  - 节点内边距：24px
  - 文字与边缘间距：12px
- **可选组件**: 循环路径、并行流程、注释说明
- **组装顺序**: 流程步骤识别→节点设计→连接线绘制→条件分支处理→代码生成
- **约束**: 必须遵循流程图规范；节点高度必须统一
- **格式**: SVG代码文件（.svg）

#### V2-03 结构图生成
- **必选组件**: 层级结构、包含关系、连接线、标签
- **结构图规范**:
  - 最外层容器：大圆角矩形，rx=20-24，最浅填充（50层级），0.5px描边
  - 内部区域：中等圆角矩形，rx=8-12，次浅填充（100-200层级）
  - 最小内边距：20px
  - 最多2-3层嵌套
- **可选组件**: 图例、说明文字
- **组装顺序**: 层级识别→容器设计→包含关系绘制→标签添加→代码生成
- **约束**: 必须遵循结构图规范；嵌套层级不超过3层
- **格式**: SVG代码文件（.svg）

#### V2-04 说明图生成
- **必选组件**: 说明对象、视觉元素、标注线、说明文字
- **说明图规范**:
  - 物理主题：绘制简化版本（截面、剖面）
  - 抽象主题：发明空间隐喻
  - 颜色编码强度，不是类别
  - 标签放在绘制对象外部，用细引线连接
- **可选组件**: 交互控制、动画效果
- **组装顺序**: 说明对象分析→视觉元素设计→标注线绘制→说明文字添加→代码生成
- **约束**: 必须遵循说明图规范；标签必须在外部
- **格式**: SVG代码文件（.svg）

- **必选组件**: 动画效果、过渡效果、时间控制、触发条件
- **动画类型**:

- **组装顺序**: 动画需求分析→动画类型选择→关键帧设计→时间控制→代码生成

#### V3-03 表单控件生成
- **必选组件**: 控件类型（按钮/输入框/下拉菜单/复选框/单选按钮）、控件样式、事件处理

- **组装顺序**: 控件需求分析→控件类型选择→样式设计→事件处理→代码生成
- **约束**: 控件必须符合标准；必须有焦点状态和悬停状态

#### V3-04 复杂布局生成

#### V5-03 图片保存功能
- **必选组件**: 保存按钮、Canvas绘制逻辑、图片导出逻辑、下载触发
- **实现方式**:
  - **SVG转图片**：SVG → Canvas → PNG
  - **HTML转图片**：html2canvas库 → Canvas → PNG
  - **Canvas直接导出**：canvas.toDataURL('image/png')
- **保存按钮代码**:
```javascript
function saveAsImage() {
    const canvas = document.createElement('canvas');
    const ctx = canvas.getContext('2d');
    // 绘制图表到canvas
    // ...
    const link = document.createElement('a');
    link.download = '图表.png';
    link.href = canvas.toDataURL('image/png');
    link.click();
}
```
- **可选组件**: 图片质量设置、图片尺寸设置
- **组装顺序**: 保存按钮创建→Canvas绘制逻辑→图片导出逻辑→下载触发→代码集成
- **约束**: 必须使用标准API；必须支持主流浏览器
- **格式**: JavaScript代码（集成到HTML文件中）
