# 客户管理模块

管理客户档案、调研客户时加载此模块。数据读写 `data/clients.json`。

## 操作映射

| 用户说 | 执行 |
|:---|:---|
| "添加客户" | 追加 client |
| "客户列表" | 列出所有客户及状态 |
| "更新客户" | 找到 client 更新字段 |
| "调研客户/了解XX公司" | 客户调研（见下方策略） |

## client 关键字段

`name`, `industry`, `contact`, `status`(待评估/优质/一般/高风险/已合作/已流失), `signal`(绿灯/黄灯/红灯), `notes`, `research`{company_intro, products, competitors, recent_news, pain_points, decision_maker}

**客户名规范化**：写入 name 时统一去掉前后空格，全角转半角。查询匹配时忽略空格和大小写，避免"北京XX公司"和"北京 XX 公司"被当成两个客户。

## 客户调研策略

用户要求调研某公司时，按以下顺序搜索：
1. **公司基础**："{公司名} 官网" / "{公司名} 公司简介" → 规模、业务、行业
2. **近期动态**："{公司名} 新闻" / "{公司名} 最新动态" → 融资、扩张、人事变动
3. **痛点推断**："{行业名} 数字化转型" / "{行业名} 营销痛点" → 行业共性需求
4. **决策人**："{公司名} {职位关键词}" → 谁是拍板人

搜索结果整理后保存到 `research` 字段。
