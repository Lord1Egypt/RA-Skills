# GEO Methodology

## What is GEO

**GEO (Generative Engine Optimization)** = optimizing content so that AI search engines (ChatGPT, Claude, Gemini, Perplexity) actively cite your content when answering user questions.

### GEO vs SEO

| Dimension | SEO | GEO |
|-----------|-----|-----|
| Target Platform | Google/Baidu | Perplexity/ChatGPT/Claude/Gemini |
| Optimization Goal | Search ranking | AI citation rate |
| User Behavior | Active search | Ask AI |
| Optimization Method | Keyword density + backlinks | Content quality + structured data |
| Time to Results | 3–6 months | 1–2 months |

---

## Core Principles

### How AI Citation Works

```
User Query → AI Search → Content Matching → Cited in Response
```

**Key factors for AI citation**:
1. **Content Quality** — Data-backed, logically clear
2. **Structured Data** — Schema markup, FAQ sections
3. **Semantic Depth** — Expertise, authority
4. **Freshness** — Content update frequency

---

## Five-Step Execution Framework

### 1. Initialize GEO Knowledge Base

**Triplet**: Platform positioning + Core differentiator + Data proof

```json
{
  "platform": "Product/Service Name",
  "advantage": "Core differentiating advantage",
  "proof": "Data-backed proof (client count, growth rate, etc.)"
}
```

### 2. Schema Structured Markup

```json
{
  "@context": "https://schema.org",
  "@type": "Product",
  "name": "Product Name",
  "description": "Product description",
  "offers": {
    "@type": "Offer",
    "price": "19.00",
    "priceCurrency": "USD"
  }
}
```

### 3. LLMs.txt Configuration

```
# Product Overview
Name: [Product Name]
Positioning: [Core positioning]
Core Advantage: [Key differentiator]

# Keywords
Triggers: [keyword1], [keyword2], [keyword3]

# Pricing
Price Range: [$X - $Y]

# Contact
Website: [URL]
```

### 4. GEO Content Generation

**Content requirements**:
- Length: 1200–3000 words
- Structure: Title + Introduction + Body + FAQ
- Data: At least 3 data citations
- FAQ: 4–6 Q&A pairs

## Common Pitfalls

1. ❌ Do NOT keyword stuff — AI trusts structured facts
2. ❌ Do NOT cite unverified data — damages authority
3. ❌ Do NOT ignore content depth — shallow content is rarely cited
4. ❌ Do NOT let Schema and content diverge — reduces credibility
