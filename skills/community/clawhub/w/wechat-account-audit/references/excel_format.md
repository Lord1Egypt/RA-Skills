# 微信公众平台 Tendency Excel 格式参考

## 导出方式

微信公众平台后台 → 统计 → 内容分析 → 导出 Excel

选择时间范围后，下载「数据趋势」Excel 文件。

## 常见列结构

微信的 Excel 列名会根据后台版本和语言设置有所不同。以下是已验证的列名变体：

### 核心列（几乎总是存在）

| 中文列名 | 可能变体 | 说明 |
|---------|---------|------|
| 文章标题 | 标题 | 文章名称 |
| 发布时间 | 发表时间、时间 | 发布日期 |
| 阅读人数 | 阅读量、阅读、阅读次数 | 文章阅读量 |
| 分享人数 | 分享量、分享、分享次数 | 分享次数 |
| 收藏人数 | 收藏、收藏量 | 收藏次数 |

### 渠道相关列

| 中文列名 | 说明 |
|---------|------|
| 阅读渠道 | 各渠道阅读分布，格式如「公众号消息500次,朋友圈300次」 |
| 渠道 | 简化版渠道信息 |
| 主要来源 | 阅读量占比最高的渠道 |

### 可选列

| 中文列名 | 说明 |
|---------|------|
| 送达人数 | 推送到达的粉丝数 |
| 阅读率 | 阅读人数/送达人数 |
| 点赞人数 | 点赞数 |
| 留言人数 | 评论数 |
| 在看人数 | 「在看」点击数 |
| 链接 | 文章永久链接 |
| 数据时间 | 统计数据的时间范围 |

## 版本差异

### 新版后台（2024年后）

- 列名更完整，通常包含「阅读渠道」
- 日期格式：YYYY-MM-DD
- 数字字段为纯数字（不带千分符）

### 旧版后台

- 「阅读渠道」列可能不存在
- 某些列可能合并（如阅读和分享在同一列用换行分隔）
- 日期格式可能为 YYYY/MM/DD

## 已知陷阱

1. **空行**：Excel 中间可能有空行，解析时跳过
2. **汇总行**：末尾可能有「总计」行，需要在分析时排除（标题包含「总计」「合计」「汇总」）
3. **千分符**：某些导出会带逗号（如 1,234），解析时需要去除
4. **多篇文章合并**：如果同一日期发布多篇，可能会有重复日期行
5. **隐藏行**：旧版 Excel 可能包含隐藏行（如图文消息被删除的记录）

## 用户画像数据

用户画像数据不在 tendency Excel 中，需要从公众号后台「用户分析 → 用户属性」页面截图获取。截图包含：

- 性别分布（男/女/未知）
- 年龄分布（分段：18岁以下、18-25、26-35、36-45、46-60、60岁以上）
- 地域分布（省份/城市）
- 语言分布（简体中文、繁体中文、英文、其他）

建议让用户提供这几张截图，手动提取数据填入分析报告。
