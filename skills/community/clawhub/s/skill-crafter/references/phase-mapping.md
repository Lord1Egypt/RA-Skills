# Phase 1 → Phase 2 映射规则

Phase 1 产出的信息怎么落到四层骨架的每一层？以下映射表让 Agent 从"理解完需求"到"开始写"不再需要猜。

---

## 映射总表

| Phase 1 产出 | 映射到哪一层 | 怎么写 |
|-------------|-------------|--------|
| 技能目标（这个 Skill 做什么） | **头部 description 第一句** | 用一句话说清做什么，不展开 |
| 用户可能的触发短语 | **头部 description 触发词区** | `Use when user asks to` + 列出所有可能说法 |
| 不适用边界 | **头部 description 边界区** | `不适用于 XXX` + 穷举排除项 |
| 能力范围（能做什么的列表） | **第二层：概述 - 功能范围** | 每条一行，只列"能做什么" |
| 一句话定位 | **第二层：概述 - 一句话定位** | "对 XX 的封装，聚焦 YY" 或 "做 ZZ 类型的任务" |
| 典型使用场景（用户会在什么情况下用） | **第三层：操作指南 - 各场景** | 每个场景 = 一个"意图 → 完整操作"对 |
| 操作步骤（用什么工具、执行什么命令） | **第三层：操作指南 - 场景内的步骤** | 按场景给完整命令/步骤，不给参数说明 |
| 默认值和可选参数 | **第三层：操作指南 - 场景注释** | 在场景示例中用注释标明默认值 |
| 依赖和前置条件 | **第四层：补充说明** | 写依赖缺失时的兜底方案 |
| 已知的坑和边界情况 | **第四层：补充说明** | 每个坑 = 一个"症状 → 具体处理"对 |
| 输出格式和存放位置 | **第三层：操作指南 - 场景注释** + **第四层：补充说明** | 操作指南中写输出位置，补充说明中写冲突处理 |

---

## 映射流程

Phase 1 完成后，Agent 手里有以下信息：

```
1. 技能名称
2. 技能目标
3. 触发短语（用户可能的说法）
4. 不适用边界
5. 能力范围
6. 典型场景（2-4个）
7. 每个场景的操作步骤
8. 依赖和前置条件
9. 已知坑点
10. 输出格式
```

按以下顺序写入四层骨架：

```
Step 1: 写头部
  - 用 #2（技能目标）写 description 第一句
  - 用 #3（触发短语）写触发词区
  - 用 #4（不适用边界）写边界区

Step 2: 写概述
  - 用 #2 + #5（能力范围）组合一句话定位
  - 用 #5 列功能范围

Step 3: 写操作指南
  - 用 #6（典型场景）创建场景小节
  - 用 #7（操作步骤）填充每个场景
  - 用 #10（输出格式）在场景中标注输出
  - 用 #8（默认值）在场景注释中标明

Step 4: 写补充说明
  - 用 #8（依赖和前置条件）写依赖兜底
  - 用 #9（已知坑点）写异常处理
  - 用 #10（输出格式）写冲突处理
```

## 常见错误

### 错误1：把能力范围写进了头部

```
❌ description: 支持单篇抓取、专辑下载、列表获取、多格式输出。
   Use when user asks to 抓取文章.
```

能力范围是概述的内容，头部只说"做什么"。触发词也没覆盖多种表达。

```
✅ description: 将网页文章抓取并转换为 Markdown 的 Skill。
   Use when user asks to 抓取网页文章、URL转Markdown、保存网页、下载文章.
   不适用于本地文件转换、PDF处理.
```

### 错误2：把操作步骤写进了概述

```
❌ ## 概述
   使用 python3 scripts/cli.py URL 命令抓取文章，支持 --format 参数指定输出格式。
```

概述只画边界。"怎么做"归操作指南。

### 错误3：把依赖信息写进了操作指南

```
❌ ## 使用
   先确保 WeSpy 已安装：pip install wespy
   然后执行：python3 scripts/cli.py URL
```

操作指南只写"正常情况怎么用"。依赖缺失的处理归补充说明。

### 错误4：补充说明只写"处理异常"

```
❌ ## 补充说明
   - 处理网络异常
   - 处理依赖缺失
```

没有具体方案，Agent 遇到问题还是得自己猜。

```
✅ ## 补充说明
   - WeSpy 未安装：优先用本地源码 ~/Documents/project/WeSpy；不存在则 git clone；clone 失败则报错提示
   - 网络超时：单次 30 秒，重试 2 次；仍失败则跳过，报告失败列表
```
