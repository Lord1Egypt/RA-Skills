# 公众号排版与审美设计规范

本文件定义公众号文章的视觉呈现标准。排版不是装饰，而是阅读体验的一部分——好的排版让人「不知不觉就读完了」，差的排版让人「看了一眼就不想读」。

---

## 1. 排版核心原则

### 1.1 三感法则

| 感官 | 目标 | 实现方式 |
|------|------|---------|
| **呼吸感** | 让眼睛不累 | 充分的留白和段间距 |
| **层次感** | 一目了然知道内容结构 | 大小标题+不同样式区隔 |
| **节奏感** | 读起来有起伏 | 长短段落交替+配图打断 |

### 1.2 对比原则（CRAP原则）

公众号排版可以简化为四法则：
- **对比**（Contrast）：标题/正文/引用区别明显
- **重复**（Repetition）：同层级元素样式统一
- **对齐**（Alignment）：全文左对齐，无左右混排
- **邻近**（Proximity）：相关元素靠近，不相关的分开

## 2. 字体与字号

### 2.1 字号体系

| 元素 | 字号 | 颜色 | 字重 | 说明 |
|------|------|------|------|------|
| 标题 | 18-20px | `#333333` | 600 | 全文统一，可加品牌色 |
| 二级标题 | 17-18px | `#1a1a1a` | 700 | 可用品牌左边框/背景色块 |
| 正文字体 | 15-16px | `#3f3f3f` | 400 | 正文深度灰，纯黑太刺眼 |
| 小标题/引用 | 14-15px | `#555555` | 500 | 次级信息 |
| 注释/来源 | 12-13px | `#999999` | 400 | 非核心信息 |
| 页边距 | — | — | — | 12-15px |

### 2.2 字体选择

```
手机端首选：
  -apple-system, "PingFang SC", "Noto Sans SC", "Microsoft YaHei", sans-serif
```

**原则**：
- 使用系统字体，不强制加载外部字体（加载慢、不稳定）
- 英文和数字使用系统中合适的西文字体
- 不要使用书法体/艺术体作为正文字体

### 2.3 禁止的字体操作

- ❌ 全文使用一种颜色（单调）
- ❌ 全文超过3种颜色（花哨）
- ❌ 使用艳色做正文（如大红、亮蓝）
- ❌ 正文使用斜体（中文斜体不易读）
- ❌ 字号小于14px（手机端太费眼）

## 3. 颜色体系

### 3.1 品牌配色

建立统一的配色系统（建议）：

```
主色（品牌色）: #4E79A7（钢蓝）
强调色:        #F28E2B（琥珀橙）——用于重点标注、数据高亮
正色:          绿色 #59A14F —— 增长、正面情绪
负色:          红色 #E15759 —— 警示、下降
正文色:        #3f3f3f（深灰）—— 比纯黑柔和
辅助色:        #718096（中灰）—— 次要信息
边框色:        #e8ecf0（浅灰）—— 分割线、边框
背景色:        #f7f8fa（极浅灰）—— 引用框背景
```

### 3.2 配色比例

```
60% 主色（标题/关键索引）
30% 中性色（正文/描述）
10% 强调色（重点标注/CTA按钮）
```

### 3.3 颜色使用场景

| 元素 | 推荐配色 |
|------|---------|
| 大标题文字 | 品牌色 或 #333 |
| 小标题标识 | 品牌色左边框（4px） |
| 正文加粗 | 品牌色 或 #333 |
| 引用/重点 | 品牌色左边框 + 品牌色浅背景 |
| 数据高亮 | 强调色（琥珀橙） |
| 链接文字 | 品牌色 |
| 分割线 | #e8ecf0 |

### 3.4 配色禁忌

- ❌ 正文使用亮色（红/橙/蓝/绿）
- ❌ 大段文字使用浅色（灰 < #666）
- ❌ 同一段落使用3种以上颜色
- ❌ 黄底白字/蓝底红字等低对比组合
- ❌ 彩虹色渐变做排版

## 4. 留白与间距

### 4.1 间距规范

| 间距 | 标准 | 说明 |
|------|------|------|
| 段落间距 | 15-20px | 一个空行 |
| 标题与正文间距 | 12-15px | 标题下方留出视线空间 |
| 页边距 | 12-15px | 左右留白，不让文字贴边 |
| 行间距 | 1.6-1.8倍 | 低于1.5行太挤，高于2.0行太散 |
| 图片上下间距 | 16-20px | 图片与文字之间留空间 |
| 图注与图片间距 | 6-8px | 图注紧贴图片下方 |

### 4.2 段落控制

- **单段最长**：手机屏幕展示不滚动即为一屏（约150-200字）
- **长段落信号**：超过5行时尝试拆分为2-3段
- **短段技巧**：1-2句也可以为一段，用于重点内容的强调

### 4.3 留白的价值

```
不善于留白的排版:
"这段话很长很多内容很丰富读者看了会觉得累而且没有重点..."
→ 读者感受：好累，不想看

善于留白的排版:
"这段话不长。"

"留出空间。"

"读者舒服了，就愿意读了。"
→ 读者感受：轻松，有节奏
```

## 5. 图文混排

### 5.1 配图密度

| 文章长度 | 最少配图 | 最佳配图 |
|---------|---------|---------|
| 800-1200字 | 2张 | 3张 |
| 1200-2000字 | 3张 | 4-5张 |
| 2000-3000字 | 4张 | 6-8张 |
| 3000字以上 | 6张 | 8-10张 |

### 5.2 配图位置

```
位置1：文章前1/3处（首屏配图，制造视觉锚点）
位置2：第1个核心观点后（打断文字节奏）
位置3：文章中间（视觉休息点）
位置4：第2-3个核心观点之间（过渡）
位置5：结尾附近（引导前配图）
```

### 5.3 配图类型

| 类型 | 适用场景 | 来源 | 风格 |
|------|---------|------|------|
| 概念图 | 抽象概念、方法论 | 自制（Canva/稿定） | 简洁扁平风 |
| 数据图 | 数据分析、对比 | 自制（可视化工具） | 品牌配色 |
| 场景图 | 故事、案例 | 高质量图库 | 干净写实风 |
| 截图 | 教程、操作说明 | 真实截图 | 加圆角边框 |
| 金句卡片 | 核心观点强调 | 自制 | 品牌色背景+白字 |
| 表情包 | 轻松内容 | 自制/合规 | 不超过2张 |

### 5.4 配图质量检查

```
□ 清晰度 ≥ 1080px（不模糊）
□ 无拉伸变形（宽高比正确）
□ 无水印（无版权风险）
□ 风格统一（全文同系色调）
□ 图注完善（居中，13px #999）
□ 图片底色与排版背景协调
□ 非纯装饰图（每张图都有信息价值）
```

### 5.5 图注规范

```
每张图下方建议加图注：

[图片]
📌 图注内容（居中，13px，#999，斜体可选）
```

## 6. 样式组件

### 6.1 标题样式

```
① 纯文字标题（最简洁）
  ## 这里是大标题
  （字重700，字号18px，品牌色或深色）

② 品牌色左边框
  ## 这里是大标题
  （左侧4px品牌色实线边框，如本文件标题所示）

③ 色块底纹标题
  ## 这里是大标题
  （背景色#f0f4f8，左右内边距12px，圆角4px）
```

**每篇文章统一使用一种标题样式，不要混用。**

### 6.2 引用样式

```
> 这是引用内容
> 可以用来展示案例、金句、重要提示

使用场景：
- 名人名言/金句
- 关键数据
- 案例背景
- 注意事项

样式：左侧4px品牌色边框 + 浅色背景
```

### 6.3 分割线

```
---（三个短横线）

使用位置：
- 正文与结尾之间
- 重大话题转换时
- 案例与方法论之间

样式：浅灰#e8ecf0，1px，两端留白
```

### 6.4 重点标注

```
加粗 + 品牌色：强调核心结论
加粗（不变色）：强调关键词
加粗 + 背景：非常关键的信息（慎用，全文≤1次）
```

## 7. 封面图设计标准

### 7.1 尺寸规范

| 类型 | 尺寸 | 比例 | 注意 |
|------|------|------|------|
| 首条大图 | 900 × 383 px | 2.35:1 | 最常用，必须适配 |
| 次条方形 | 300 × 300 px | 1:1 | 次条默认为方形 |
| 朋友圈分享 | 400 × 400 px | 1:1 | 分享时的缩略图 |

### 7.2 封面设计原则

```
文字覆盖图：
┌───────────────────────────────────────────┐
│                                           │
│   [背景图 - 干净/模糊或有纹理的抽象背景]     │
│                                           │
│       大号加粗标题文字（居左/居中）          │
│          副标题或推荐语（较小）              │
│                                           │
│       [品牌logo或公众号名称右下角]           │
│                                           │
└───────────────────────────────────────────┘
```

**原则**：
- 文字与背景对比度 ≥ 3:1（保证可读性）
- 文字不超过图面积的40%
- 标题不超过15字（否则太小）
- 留白充足，不拥挤
- 全系列风格统一（字体/配色/排布方式）

### 7.3 封面图禁用手法

- ❌ 过多文字堆砌（像通知公告）
- ❌ 低分辨率或模糊图片
- ❌ 与文章内容无关的图片
- ❌ 带水印的图片
- ❌ 过于花哨的背景（干扰标题阅读）

## 8. 手机端预览检查

发布前，务必在手机上检查：

```
□ 标题是否被截断（<30字最安全）
□ 封面图是否完整显示
□ 正文首屏能否吸引人（前200字）
□ 段落长度是否一屏可读
□ 图片加载是否正常
□ 引用/加粗在手机上效果
□ 底部引导是否清晰可见
□ 全文是否需要"太长了"的滑动量
```

## 9. 不同内容类型的排版差异

| 类型 | 标题效果 | 配图策略 | 特殊样式 |
|------|---------|---------|---------|
| 干货教程 | 沉稳品牌色 | 操作截图+流程图 | 步骤编号、tips框 |
| 行业分析 | 数据可视化 | 数据图表+趋势图 | 信息图为主 |
| 个人故事 | 有温度 | 场景配图+金句卡 | 大图、文字留白多 |
| 观点文章 | 强对比 | 金句卡为主 | 引用多、加粗多 |
| 盘点清单 | 数字突出 | 每项配图 | 项目符号+强调数字 |
