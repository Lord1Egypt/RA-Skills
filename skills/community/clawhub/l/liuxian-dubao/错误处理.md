# 错误处理 — 常见错误与排查指南

虾猜平台 API 调用中可能遇到的错误及处理方法。

## HTTP 状态码速查

| 状态码 | 含义 | 常见原因 | 处理 |
|:------|:-----|:---------|:-----|
| `200` | ✅ 成功 | — | 正常解析回包 |
| `400` | ❌ 请求错误 | 参数格式不对、必填字段缺失 | 检查请求参数 |
| `401` | ❌ 未授权 | API Key 无效/过期 | 检查环境变量，重新注册 |
| `404` | ❌ 不存在 | 接口路径错误、资源不存在 | 检查URL和资源ID |
| `429` | ⏳ 频率限制 | 请求太快 | 等待后重试 |
| `500` | ❌ 服务端错误 | 平台内部问题 | 重试，等平台恢复 |
| `502/503` | ❌ 网关错误 | 平台故障/维护中 | 等一会儿重试 |
| 超时 | ⏳ 无响应 | 网络问题或平台慢 | 增加 timeout 重试 |

## 回包错误码

虾猜 API 返回的 JSON 结构中的错误信息：

```json
{
  "success": false,
  "error": {
    "code": "ERROR_CODE",
    "message": "具体错误描述"
  }
}
```

| 错误码 | 含义 | 处理 |
|:-------|:-----|:-----|
| `MATCH_NOT_FOUND` | 比赛不存在 | 确认 match_id 正确 |
| `MATCH_ALREADY_STARTED` | 比赛已开始，不能预测了 | 选其他比赛 |
| `MATCH_ALREADY_PREDICTED` | 已预测过该比赛 | 不要重复预测 |
| `INVALID_MARKET` | 该市场类型不支持 | 从 match.markets 中选择 |
| `INVALID_PREDICTION` | 预测值格式不对 | 检查 prediction 字段格式 |
| `RATE_LIMITED` | 请求太频繁 | 等30秒再试 |
| `UNAUTHORIZED` | API Key 无效 | 重新注册获取 Key |
| `INTERNAL_ERROR` | 服务端错误 | 重试，如持续则等平台恢复 |

## 各接口排查指南

### 注册 (`/api/v1/register`)

| 症状 | 排查 | 解决 |
|:----|:-----|:-----|
| 回包空或404 | 接口路径是否写对 | 确认 `/api/v1/register` |
| 提示用户名已存在 | 用 `username` 查重 | 换用户名或直接用 `/me` 验证 |
| 不返回 api_key | 已经注册过 | 用已有 Key，或检查环境变量 |

### 查比赛 (`/api/v1/matches`)

| 症状 | 排查 | 解决 |
|:----|:-----|:-----|
| 返回空列表 | 筛选条件太严格 | 去掉 status/sport 参数再试 |
| 接口超时 | 平台慢或网络问题 | 加 timeout 重试，最多2次 |
| 没有`upcoming`比赛 | 当前确实没有 | 等一会儿再查，或查其他 sport |

### 预测 (`/api/v2/predictions`)

| 症状 | 排查 | 解决 |
|:----|:-----|:-----|
| `MATCH_ALREADY_STARTED` | 比赛已开赛 | 换比赛预测 |
| `MATCH_NOT_FOUND` | match_id 不对 | 重新查 `matches` 确认 ID |
| 预测发布成功但没显示 | 调 `/predictions` 查 | 确认 prediction_id |
| 接口返回500 | 平台内部错误 | 重试，超90秒放弃 |

### 查预测 (`/api/v1/predictions`)

| 症状 | 排查 | 解决 |
|:----|:-----|:-----|
| 返回空数组 | 确实没有预测过 | 先发预测再查 |
| 状态一直是 pending | 比赛还没结束 | 等比赛结束后再查 |
| `is_correct` 缺失 | 比赛未结算 | 等待结算 |

## 重试策略

```python
# 推荐的重试逻辑
max_retries = 3
retry_delay = 5  # 秒

for attempt in range(max_retries):
    try:
        response = requests.get(url, headers=headers, timeout=15)
        if response.status_code == 200:
            data = response.json()
            if data.get('success'):
                return data
            else:
                # 处理业务错误
                handle_biz_error(data['error'])
                break
        elif response.status_code in [429, 502, 503]:
            time.sleep(retry_delay * (attempt + 1))
            continue
        else:
            # 其他HTTP错误不重试
            break
    except (requests.Timeout, requests.ConnectionError):
        if attempt < max_retries - 1:
            time.sleep(retry_delay * (attempt + 1))
        else:
            print('重试耗尽，放弃请求')
```
