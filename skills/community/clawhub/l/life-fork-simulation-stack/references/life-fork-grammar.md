# Life Fork Grammar / 人生迁移语法

用途：把任何“如果当年……”复盘问题，先拆成通用迁移结构，再进入事件冲击和报告生成。此文件优先于主题案例库使用。

## 总流程

1. 识别岔路动作。
2. 识别系统切换。
3. 找到自我叙事断裂。
4. 生成首屏判词。
5. 生成三种结果：现实结果 / 假设结果 / 回补结果。
6. 选择 3-6 个高相关事件或周期。
7. 转成事件冲击卡。
8. 找到今天还能回收的资产。
9. 生成 30 天验证实验。
10. 把多视角、事件表、已知信息程度和补充问题放到继续解释段。

## 1. 岔路动作层

先判断用户当年做了什么动作。动作可多选，但首个动作决定报告主叙事。

| 动作 | 识别信号 | 复盘重点 |
| --- | --- | --- |
| 留下 | 当年没有离开某个城市、组织、关系或路径 | 留下给了什么托底，也拿走了哪些外部碰撞 |
| 离开 | 当年离开了某个系统 | 离开换来了什么新定价，也失去了哪些熟人支持 |
| 回流 | 曾经出去，后来回来 | 回来保住了什么，出去那段经历是否继续被使用 |
| 错过 | 看见过机会，但没有行动 | 当年缺行动、缺入口、缺资源，还是缺承受力 |
| 延迟 | 想做，但一直拖到窗口变弱 | 窗口变弱前本可留下什么作品或能力证据 |
| 押注 | 把资源压在房子、行业、公司、城市、关系或技术上 | 押注带来的稳定、绑定、弹性和机会损失 |
| 撤退 | 从高压、高风险、高可能性路径退回低风险路径 | 撤退保住了什么，也让哪个更大的自己停了下来 |
| 转向 | 从一个身份、行业、城市或系统切换到另一个系统 | 新系统如何重新定价用户，旧系统资产是否迁移 |
| 被迫中断 | 疫情、家庭、健康、裁员、签证、政策、现金流等外部冲击中断 | 中断来自外部冲击，个人选择只承担其中一部分 |

输出时用一句话写明：

```text
岔路动作：{{动作}}。当年用户实际做了{{具体行动}}，放下了{{未走动作}}。
```

## 2. 系统切换层

识别用户当年真正进入和离开的系统。系统切换比主题名更重要。

| 系统对照 | 代表问题 | 常见代价 |
| --- | --- | --- |
| 大城市 vs 小城市 | 去北京、回老家、留深圳 | 机会密度、生活成本、家庭距离、身体消耗 |
| 海外 vs 国内 | 留学未回国、回国、转新加坡/香港 | 签证、语言、孤独、家人距离、身份支撑 |
| 市场化 vs 体制 / 国企 | 进体制、去市场公司 | 稳定、收入弹性、组织规则、成长速度 |
| 大厂 / 平台 vs 小公司 / 创业 / 自由职业 | 进大厂、离开平台、创业 | 资源、授权、外部定价、现金流波动 |
| 买房绑定 vs 租房流动 | 买房、没买房、提前上车 | 月供、换城自由、家庭安全感、现金流弹性 |
| 熟人系统 vs 陌生竞争系统 | 留旧公司、去新城市、海外求职 | 熟悉协作、被重新评价、关系重建 |
| 稳定现金流 vs 高波动机会 | 留稳岗、转高增长行业 | 空窗承受力、储蓄率、机会窗口 |
| 旧技能路径 vs 新技术路径 | 早点做 AI、转技术 | 作品资产、工具门槛、岗位重估 |
| 家庭责任 vs 个人扩张 | 回家、陪伴父母、婚育节奏 | 照护、距离、个人试错空间 |
| 公司内部成果 vs 外部可见作品 | 工作很多但外部看不见 | 案例、公开表达、同行反馈 |

输出时用一句话写明：

```text
系统切换：从{{系统 A}}进入{{系统 B}}，核心交换为{{得到}}换{{放下}}。
```

## 3. 自我叙事断裂层

首屏判词必须从这一层生成。先命名用户真正痛的地方，再解释原因。

| 断裂 | 判词方向 |
| --- | --- |
| 失败感 | 把失败感拆成结果、环境和当时承受力 |
| 错过更大的自己 | 命名那个更早被看见、更独立或更有野心的自己 |
| 太早求稳 | 写清安全感换走了什么训练 |
| 逃离竞争 | 区分撤退、承受力和当时的现实约束 |
| 自由换安全 | 写出自由背后的现金流、身份和关系成本 |
| 成长换舒服 | 写出舒服给的恢复，也写出成长被延迟的部分 |
| 国际化身份放弃感 | 区分地点、语言、岗位和国际作品资产 |
| 没有被看见 | 把被看见拆成作品、网络、授权和外部反馈 |
| 被时代甩下 | 区分时代窗口、行动缺口和今天可补动作 |
| 经验没有变成资产 | 指向案例、公开表达、流程和作品封装 |

判词生成公式：

```text
{{承载物}}在你心里，像{{隐喻}}。你反复想起它，指向{{自我叙事断裂}}：你怀疑自己把{{放下的东西}}，换成了{{保住的东西}}。
```

## 4. 事件冲击层

事件库不可只当背景。每个事件都要转成用户能感受到的冲击卡。

| 字段 | 写法 |
| --- | --- |
| 事件锚点 | 哪一年 / 哪个阶段 / 发生了什么 |
| 生活场景 | 这个事件在用户生活里长什么样 |
| 现实代价 | 钱、时间、健康、关系、身份、工作、家庭、机会窗口 |
| 情绪代价 | 不甘、焦虑、孤独、安全感、错过感、羞耻感、松一口气 |
| 误判点 | 用户当年高估或低估了什么 |
| 今日验证 | 7-30 天内可以做的现实动作 |

事件筛选优先级：

1. 和当年选择点直接相关。
2. 和后来遗憾被重新触发相关。
3. 能转成生活场景。
4. 能支持今天验证动作。

## 5. 三种结果层

所有报告统一使用三种结果。

| 结果 | 定义 | 写法 |
| --- | --- | --- |
| 现实结果 | 用户实际走过的结果 | 它给你什么 / 它拿走什么 / 最大误判 |
| 假设结果 | 用户没走或中断的结果 | 条件成立时可能给什么 / 会拿走什么 / 最大误判 |
| 回补结果 | 今天还能补回、回收或重新验证的结果 | 今天可回收什么 / 会拿走什么舒服感 / 最大误判 |

禁用命名：

- 把三种结果写成某某线。
- 把报告包装成多宇宙设定。
- 按主题自造结果名称。

## 6. 可回收资产层

每份报告最后必须回答：今天还能回收什么。

| 可回收资产 | 回收方式 |
| --- | --- |
| 城市经验 | 写成城市、行业、项目和用户样本的判断 |
| 海外经历 | 转成英文案例、跨文化判断和国际反馈 |
| 行业判断 | 封装成案例、清单、流程或 Skill |
| 组织经验 | 写出预算、授权、协作和复盘能力 |
| 大厂 / 平台经历 | 提炼成方法、数据意识和跨团队案例 |
| 体制经验 | 回收规则意识、长期节奏和低波动安排 |
| 房产带来的稳定性 | 转成现金流边界和生活底盘 |
| 现金流安全感 | 做成应急金、空窗月数和选择边界 |
| 家庭支持 | 转成明确边界、照护安排和情绪底座 |
| 语言能力 | 写成双语案例和跨境沟通样本 |
| 作品案例 | 用一页说明问题、判断、动作、结果、边界 |
| 公开表达 | 用低风险发布获取同行反馈 |
| AI 工具能力 | 选真实问题，做最小工作流 |
| 工作流封装能力 | 写输入、步骤、输出、失败条件 |
| 关系网络 | 找 3-5 个真实人做信息校准 |
| 身体和生活节奏 | 记录睡眠、通勤、恢复和持续投入能力 |

## 7. 30 天验证实验语法

验证实验要回答：今天能用什么小动作回收资产。

| 时间 | 默认动作 | 目标 |
| --- | --- | --- |
| 第 1 周 | 找真实人或真实材料 | 校准假设结果的真实代价 |
| 第 2 周 | 写一页案例或做最小流程 | 把经验变成外部可看见材料 |
| 第 3 周 | 发给 3 个可信对象 | 观察别人能否复述价值 |
| 第 4 周 | 根据反馈加码、缩小或换题 | 决定继续回收哪项资产 |

## 8. 语法回归表

| 主题 | 岔路动作 | 系统切换 | 自我叙事断裂 | 主要可回收资产 |
| --- | --- | --- | --- | --- |
| 去北京 | 留下 / 错过 | 深圳熟人系统 vs 北京陌生竞争系统 | 更早被看见的自己 | 城市经验、作品案例、关系网络 |
| 留学未回国 | 回流 / 撤退 | 海外 vs 国内 | 国际化身份和独立感 | 海外经历、语言能力、英文作品 |
| 买房 / 没买房 | 押注 / 错过 | 买房绑定 vs 租房流动 | 自由和安全的交换 | 现金流安全感、生活底盘 |
| 进体制 / 市场化 | 转向 / 撤退 | 体制低波动 vs 市场化高波动 | 稳定感来源 | 组织经验、现金流边界、健康节奏 |
| 离开上一家公司 | 离开 / 留下 | 熟人系统 vs 陌生竞争系统 | 外部定价和低摩擦的交换 | 组织经验、作品案例、关系网络 |
| 2023 年认真做 AI | 错过 / 延迟 | 旧技能路径 vs 新技术路径 | 被时代甩下和经验资产化 | AI 工具能力、工作流封装、公开案例 |

使用时先套语法，再读主题资料。主题资料只补充场景，不决定报告结构。
