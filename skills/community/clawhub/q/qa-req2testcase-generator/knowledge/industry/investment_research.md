# 投研域（Investment Research Domain）

> 本文档覆盖券商投资研究核心业务对象的状态机、操作规则、异常模式与测试点，涵盖研究所管理、研报管理、数据投研等子域。

---

## 研究所管理

### 业务背景
券商研究所是证券公司的核心部门之一，负责宏观经济、行业和公司的深度研究，为内部投资决策和外部客户提供研究服务。研究所的收入模式主要包括佣金分仓（机构客户交易佣金分配）和研究服务费。

### 研究团队管理

#### 团队架构
- 首席经济学家：宏观经济研究，对外发声
- 行业首席分析师：各行业研究负责人
- 研究员：具体行业/公司的研究执行
- 研究助理：数据收集、模型维护、报告撰写辅助
- 量化研究员：因子研究、模型开发、策略回测

#### 覆盖行业
- 大消费：食品饮料、医药生物、家电、零售
- TMT：电子、计算机、通信、传媒
- 周期：有色金属、钢铁、化工、建材
- 金融地产：银行、非银金融、房地产
- 制造：机械、电力设备、汽车、军工
- 其他：农林牧渔、交通运输、公用事业

#### 研究方法论
- 自上而下：宏观→行业→公司
- 自下而上：公司基本面→行业比较→宏观验证
- 量化研究：因子模型、统计分析、机器学习
- 产业链研究：上下游关系、供需分析、竞争格局

### 分析师管理状态全集
- 入职培训（Training）：新入职分析师培训期
- 正常执业（Active）：分析师正常开展研究工作
- 静默期（QuietPeriod）：因参与投行项目等进入静默期
- 暂停执业（Suspended）：因合规问题暂停研究发布
- 离职交接（Handover）：分析师离职，进行覆盖交接
- 已离职（Left）：分析师已离职（终态）

### 状态转移规则
| 当前状态 | 触发条件 | 目标状态 | 是否允许 |
|---------|---------|---------|--------|
| 入职培训 | 培训考核通过 | 正常执业 | ✅ |
| 正常执业 | 参与投行项目 | 静默期 | ✅ |
| 正常执业 | 合规违规 | 暂停执业 | ✅ |
| 正常执业 | 提出离职 | 离职交接 | ✅ |
| 静默期 | 项目结束/公告发布 | 正常执业 | ✅ |
| 暂停执业 | 整改完成 | 正常执业 | ✅ |
| 离职交接 | 交接完成 | 已离职 | ✅ |
| 静默期 | 发布研报 | — | ❌ |
| 暂停执业 | 发布研报 | — | ❌ |

### 异常模式
- 分析师离职未交接：覆盖行业/公司无人跟踪 → 测试点：验证离职触发的覆盖交接流程和客户通知
- 静默期违规发布：分析师在静默期内发布相关研报 → 测试点：验证静默期标的的发布拦截机制
- 覆盖重叠：多个分析师覆盖同一标的产生冲突 → 测试点：验证覆盖分配的唯一性校验

### 测试点模板
- TP-IR-001：分析师进入静默期，验证相关标的研报发布被拦截（优先级：P0）
- TP-IR-002：分析师离职，验证覆盖行业/公司的交接流程触发（优先级：P0）
- TP-IR-003：新分析师入职，验证培训考核通过后方可发布研报（优先级：P1）
- TP-IR-004：分析师覆盖分配，验证同一标的不可重复分配（优先级：P1）

## 研报管理

### 业务背景
研究报告是研究所的核心产出，包括深度报告、点评报告、行业报告、晨会纪要等。研报从撰写到发布需经过严格的合规审查流程，发布后通过多渠道推送给机构客户。

### 研报类型

| 类型 | 说明 | 篇幅 | 发布频率 |
|------|------|------|---------|
| 深度报告 | 公司/行业深度研究 | 20-50页 | 不定期 |
| 首次覆盖 | 首次覆盖某公司的深度报告 | 30-60页 | 不定期 |
| 点评报告 | 事件/业绩点评 | 3-10页 | 事件驱动 |
| 行业周报/月报 | 行业跟踪 | 10-20页 | 周/月 |
| 晨会纪要 | 每日研究观点汇总 | 2-5页 | 每日 |
| 策略报告 | 市场策略和配置建议 | 10-30页 | 周/月/季 |
| 量化报告 | 因子/策略/模型研究 | 10-30页 | 不定期 |

### 研报生命周期状态全集
- 撰写中（Drafting）：分析师正在撰写研报
- 内部评审（InternalReview）：研究所内部评审（质量审核）
- 合规审查（ComplianceReview）：合规部门审查内容合规性
- 待发布（PendingPublish）：审查通过，等待发布时间窗口
- 已发布（Published）：研报已发布并推送（终态）
- 已驳回（Rejected）：审查不通过，需修改（可重新提交）
- 已撤回（Withdrawn）：已发布研报因错误被撤回（终态）
- 已更正（Corrected）：已发布研报发布更正版本（终态）

### 状态转移规则
| 当前状态 | 触发条件 | 目标状态 | 是否允许 |
|---------|---------|---------|--------|
| 撰写中 | 提交评审 | 内部评审 | ✅ |
| 内部评审 | 评审通过 | 合规审查 | ✅ |
| 内部评审 | 评审不通过 | 已驳回 | ✅ |
| 合规审查 | 合规通过 | 待发布 | ✅ |
| 合规审查 | 合规不通过 | 已驳回 | ✅ |
| 已驳回 | 修改后重新提交 | 内部评审 | ✅ |
| 待发布 | 到达发布时间 | 已发布 | ✅ |
| 待发布 | 分析师撤回 | 撰写中 | ✅ |
| 已发布 | 发现重大错误 | 已撤回 | ✅ |
| 已发布 | 发布更正版 | 已更正 | ✅ |
| 已发布 | 未经流程直接修改 | — | ❌ |
| 已撤回 | 直接重新发布 | — | ❌（需重新走审核流程） |

### 合规审查要点
- **投资评级合规**：评级变更需有充分依据，不得频繁变更
- **目标价合理性**：目标价需有估值模型支撑，偏离度需合理
- **利益冲突披露**：披露公司与研究标的的业务关系
- **信息来源合规**：不得使用内幕信息或未公开信息
- **敏感词检查**：不得包含夸大、误导性表述
- **数据准确性**：财务数据、行情数据需核实准确

### 推送分发

#### 分发渠道
- 机构客户邮件：按客户订阅偏好推送
- 研究平台：Wind、Choice、iFinD等第三方平台
- 自有APP/小程序：面向零售客户的精简版
- 路演/电话会：重要报告配合路演推介

#### 分发规则
- 公平披露：同一研报需同时向所有订阅客户发布
- 提前发布禁止：不得向特定客户提前泄露研报内容
- 分级发布：深度报告可设置VIP客户优先阅读期（通常24小时）
- 外部平台延迟：自有渠道发布后，外部平台可延迟发布（通常1-3天）

### 异常模式
- 研报数据错误：发布后发现财务数据引用错误 → 测试点：验证研报撤回和更正流程
- 合规审查超时：合规审查超过规定时限（通常4小时） → 测试点：验证审查超时的自动提醒和升级
- 提前泄露：研报发布前内容被泄露 → 测试点：验证研报访问权限控制和操作审计
- 推送失败：邮件/平台推送失败 → 测试点：验证推送失败的重试机制和补发流程
- 评级冲突：同一标的不同分析师给出不同评级 → 测试点：验证评级冲突的检测和协调机制

### 测试点模板
- TP-IR-010：研报提交合规审查，验证敏感词检查+利益冲突披露+数据准确性校验（优先级：P0）
- TP-IR-011：研报发布，验证同时向所有订阅客户推送（公平披露）（优先级：P0）
- TP-IR-012：已发布研报撤回，验证撤回通知发送和平台下架（优先级：P0）
- TP-IR-013：合规审查超过4小时，验证自动提醒审查人员（优先级：P1）
- TP-IR-014：VIP客户优先阅读期，验证24小时后向普通客户开放（优先级：P1）
- TP-IR-015：研报推送至外部平台失败，验证重试3次后告警（优先级：P1）

### 中间态处理规则
- 「内部评审」为中间态，应在1个工作日内完成
- 「合规审查」为中间态，应在4小时内完成（紧急报告2小时）
- 「待发布」可持续至指定发布时间，但不超过3个工作日
- 审查中的研报不可被其他人修改，需加编辑锁

### 幂等/重试规则
- 研报使用 reportId + version 作为幂等键
- 推送使用 reportId + channelId + pushTime 作为幂等键
- 合规审查结果使用 reportId + reviewRound 作为幂等键
- 推送失败重试：最多3次，间隔5min/15min/30min

## 数据投研

### 业务背景
数据投研是利用大数据、人工智能等技术手段辅助投资研究的新兴领域。包括另类数据采集、因子分析、量化模型构建、策略回测等环节。

### 数据采集

#### 数据源类型
- 基础数据：财务报表、公告、行情数据（Wind/Choice/聚源）
- 另类数据：卫星图像、社交媒体、电商数据、招聘数据
- 产业数据：行业协会数据、政府统计数据、产业链调研
- 实时数据：新闻舆情、分析师预期、资金流向

#### 数据采集状态全集
- 待采集（PendingCollect）：数据采集任务已创建
- 采集中（Collecting）：正在从数据源拉取数据
- 清洗中（Cleaning）：数据清洗和标准化处理
- 已入库（Stored）：数据已入库可供使用（终态）
- 采集失败（CollectFailed）：数据源不可用或格式异常（终态，可重试）
- 质量异常（QualityIssue）：数据质量检查不通过

#### 数据质量规则
- 完整性：关键字段不可为空，缺失率<1%
- 准确性：数值范围校验，异常值检测
- 时效性：数据更新延迟不超过规定时限
- 一致性：同一指标不同数据源的交叉验证

### 因子分析

#### 因子类型
- 价值因子：PE、PB、PS、PCF、股息率
- 成长因子：营收增速、利润增速、ROE变化
- 动量因子：过去N日收益率、相对强弱
- 质量因子：ROE、毛利率、资产负债率
- 波动因子：历史波动率、Beta、特质波动
- 流动性因子：换手率、成交额、买卖价差
- 另类因子：舆情情绪、分析师预期变化、资金流向

#### 因子研究流程
1. 因子定义：明确因子计算公式和参数
2. 因子计算：基于历史数据计算因子值
3. 因子检验：IC/IR分析、分组回测、衰减分析
4. 因子筛选：选择有效因子纳入模型
5. 因子组合：多因子加权组合优化
6. 样本外验证：在样本外数据上验证因子有效性

### 模型构建

#### 模型类型
- 多因子选股模型：基于因子打分的股票筛选
- 机器学习模型：随机森林、XGBoost、深度学习
- 时间序列模型：ARIMA、GARCH、LSTM
- NLP模型：研报文本分析、舆情情感分析
- 知识图谱：产业链关系、事件传导路径

#### 模型生命周期状态全集
- 开发中（Developing）：模型正在开发和调试
- 回测中（Backtesting）：模型在历史数据上回测验证
- 评审中（Reviewing）：模型提交评审委员会审核
- 模拟运行（PaperTrading）：模型在模拟环境中实盘验证
- 已上线（Production）：模型正式上线使用
- 已下线（Offline）：模型因失效或升级被下线（终态）
- 回测失败（BacktestFailed）：回测结果不达标（终态，可修改后重新回测）

#### 状态转移规则
| 当前状态 | 触发条件 | 目标状态 | 是否允许 |
|---------|---------|---------|--------|
| 开发中 | 提交回测 | 回测中 | ✅ |
| 回测中 | 回测达标 | 评审中 | ✅ |
| 回测中 | 回测不达标 | 回测失败 | ✅ |
| 回测失败 | 修改后重新回测 | 回测中 | ✅ |
| 评审中 | 评审通过 | 模拟运行 | ✅ |
| 评审中 | 评审不通过 | 开发中 | ✅ |
| 模拟运行 | 模拟期表现达标（通常3个月） | 已上线 | ✅ |
| 模拟运行 | 模拟期表现不达标 | 开发中 | ✅ |
| 已上线 | 模型失效/升级替换 | 已下线 | ✅ |
| 已上线 | 未经评审直接修改 | — | ❌ |
| 已下线 | 直接重新上线 | — | ❌（需重新走评审流程） |

### 回测验证

#### 回测要求
- 回测区间：至少覆盖3年历史数据（含牛熊市）
- 样本划分：训练集70%、验证集15%、测试集15%
- 避免过拟合：交叉验证、正则化、样本外测试
- 交易成本：回测需考虑佣金、滑点、冲击成本
- 基准比较：与基准指数和同类策略比较

#### 回测达标标准
- 年化收益率：超过基准N个百分点（根据策略类型）
- 最大回撤：不超过设定阈值（如-15%）
- 夏普比率：≥1.0
- 信息比率：≥0.5
- 换手率：在合理范围内（避免过度交易）

### 异常模式
- 数据源中断：关键数据源不可用导致因子计算中断 → 测试点：验证数据源故障的降级策略和告警
- 因子失效：已上线因子IC持续为负 → 测试点：验证因子监控指标触发因子下线/替换
- 模型过拟合：模型在样本内表现优异但样本外失效 → 测试点：验证样本外验证的强制执行
- 回测数据泄露：未来数据泄露到历史回测中 → 测试点：验证回测框架的时间戳严格控制
- 模型漂移：上线模型随时间推移性能下降 → 测试点：验证模型性能监控和自动告警机制

### 测试点模板
- TP-IR-020：数据采集任务，验证数据完整性检查（缺失率<1%）（优先级：P0）
- TP-IR-021：因子计算，验证IC/IR指标计算准确性（优先级：P0）
- TP-IR-022：模型回测，验证必须覆盖3年以上历史数据（优先级：P0）
- TP-IR-023：模型上线，验证必须通过回测+评审+3个月模拟运行（优先级：P0）
- TP-IR-024：数据源中断，验证降级策略和告警通知（优先级：P0）
- TP-IR-025：已上线模型性能下降（夏普比率<0.5持续30天），验证自动告警（优先级：P1）
- TP-IR-026：回测框架时间戳校验，验证不可使用未来数据（优先级：P1）
- TP-IR-027：因子IC连续3个月为负，验证触发因子审查流程（优先级：P1）

## 知识管理

### 研究数据库
- 公司数据库：覆盖公司的基本面数据、研究笔记、调研纪要
- 行业数据库：行业数据、产业链图谱、竞争格局分析
- 宏观数据库：宏观经济指标、政策跟踪、国际比较
- 会议纪要库：路演纪要、电话会纪要、调研纪要

### 标签体系
- 行业标签：申万一级/二级/三级行业分类
- 主题标签：新能源、AI、消费升级等投资主题
- 事件标签：业绩预告、股权激励、并购重组等事件类型
- 情绪标签：看多/看空/中性
- 时效标签：短期（1个月内）/中期（3-6个月）/长期（1年以上）

### 知识图谱
- 实体类型：公司、行业、产品、人物、事件
- 关系类型：供应链关系、竞争关系、投资关系、人事关系
- 应用场景：关联分析、事件传导、风险预警
- 更新机制：自动抽取（NLP）+人工审核

### 异常模式
- 知识过期：研究数据库中的信息未及时更新 → 测试点：验证数据时效性标记和过期提醒
- 标签不一致：同一公司在不同报告中标签不一致 → 测试点：验证标签体系的一致性校验
- 知识图谱错误：自动抽取的关系存在错误 → 测试点：验证知识图谱的人工审核流程和纠错机制

### 测试点模板
- TP-IR-030：研究数据库检索，验证按行业/主题/时间的多维检索准确性（优先级：P0）
- TP-IR-031：知识图谱关系查询，验证供应链上下游关系的完整性（优先级：P1）
- TP-IR-032：标签自动打标，验证NLP模型的标签准确率≥90%（优先级：P1）
- TP-IR-033：数据过期提醒，验证超过6个月未更新的公司数据触发提醒（优先级：P1）

## 合规要求

### 静默期管理
1. **IPO静默期**：参与承销的分析师在IPO前后40天内不得发布相关研报
2. **再融资静默期**：参与再融资项目期间不得发布相关研报
3. **重大事项静默期**：知悉未公开重大信息后进入静默期
4. **静默期管理**：系统自动标记静默期标的，拦截相关研报发布

### 研报合规
1. **客观公正**：研报内容应客观公正，不得有误导性陈述
2. **信息来源**：引用数据需注明来源，不得使用内幕信息
3. **评级体系**：使用统一的投资评级体系，评级变更需有依据
4. **风险提示**：研报必须包含风险提示章节
5. **免责声明**：研报必须包含标准免责声明
6. **发布审批**：所有研报发布前需经合规审查

### 利益冲突管理
1. **业务关系披露**：披露公司与研究标的的投行、做市等业务关系
2. **持仓披露**：披露分析师及公司对研究标的的持仓情况
3. **薪酬独立**：分析师薪酬不得与特定投行项目挂钩
4. **独立性声明**：研报需包含分析师独立性声明

## 常见缺陷模式

1. **静默期研报泄露**：静默期内研报内容通过非正式渠道泄露
2. **合规审查绕过**：分析师通过微信/邮件直接发送未审核研报给客户
3. **数据引用错误**：财务数据、行情数据引用错误未被发现
4. **评级频繁变更**：短期内频繁变更投资评级缺乏合理依据
5. **推送不公平**：VIP客户提前获取研报内容
6. **模型回测作弊**：使用未来数据或过度优化参数导致回测结果失真
7. **因子计算错误**：因子公式实现与定义不一致
8. **数据质量问题**：数据清洗不充分导致异常值影响模型
