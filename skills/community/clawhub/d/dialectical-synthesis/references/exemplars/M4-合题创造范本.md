# M4 合题创造域范本

## M4-01 合题策略

### 输入示例
正方论证、反方论证、局限识别结果

### 处理过程
1. 分析正反方的核心价值
2. 识别可以整合的维度
3. 设计合题创造策略
4. 评估策略的可行性

### 输出示例
**合题策略**：
- 策略名称：优势整合策略
- 核心思路：整合远程办公和传统办公的优势
- 关键维度：工作性质、员工特点、技术条件
- 实施路径：先试点后推广

## M4-02 合题构建

### 输入示例
合题策略

### 处理过程
1. 设计合题框架
2. 填充合题内容
3. 梳理合题逻辑
4. 优化合题结构

### 输出示例
**合题**：
核心立场：混合办公模式

合题内容：
1. 正题价值：远程办公提高员工满意度和效率
2. 反题价值：传统办公促进团队协作和创新
3. 合题整合：根据工作性质和员工特点，灵活选择办公模式

---

## M4-03 合题验证

### 输入示例
合题

### 处理过程
1. 制定验证标准
2. 选择验证方法
3. 执行验证
4. 分析验证结果

### 输出示例
**验证结果**：
| 验证项 | 结果 | 说明 |
|--------|------|------|
| 包含正题价值 | ✅ 通过 | 保留了远程办公的优势 |
| 包含反题价值 | ✅ 通过 | 保留了传统办公的优势 |
| 超越正反方 | ✅ 通过 | 创造了新的办公模式 |
| 可实施性 | ✅ 通过 | 有明确的实施路径 |

## M4-04 合题优化

### 输入示例
验证后的合题

### 处理过程
1. 确定优化目标
2. 制定优化策略
3. 执行优化
4. 评估优化结果

### 输出示例

## M4-05 合题整合

### 输入示例
优化后的合题

### 处理过程
1. 制定整合策略
2. 选择整合方法
3. 执行整合
4. 验证整合结果

### 输出示例
**整合后的合题**：
混合办公模式 v2.0

核心原则：
1. 工作性质决定办公模式
2. 员工特点影响办公选择
3. 技术条件支撑办公实现

实施框架：
1. 评估阶段：分析工作性质和员工特点
2. 选择阶段：确定合适的办公模式
3. 实施阶段：提供必要的技术支持
4. 评估阶段：持续优化办公模式
