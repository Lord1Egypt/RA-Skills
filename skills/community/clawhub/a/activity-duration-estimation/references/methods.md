# 四种估算方法详解

> 本文件详细说明四种活动历时估算方法的公式、计算步骤和适用场景。

---

## 方法一：直接估算法（Direct Average）

### 公式

$$
\text{预期工期} = \frac{O + M + P}{3}
$$

其中：
- **O** = 乐观时间（Optimistic）
- **M** = 最可能时间（Most Likely）
- **P** = 悲观时间（Pessimistic）

### 适用场景

| 场景 | 说明 |
|------|------|
| 简单/单阶段任务 | 只有一个活动，无复杂依赖 |
| 数据对称 | O和P对M大致对称分布 |
| 快速粗略估算 | 不需要高精度，快速出数 |
| 重复性工作 | 历史数据表明O/M/P接近均匀分布 |

### 局限性

- 忽略O和P概率权重差异
- 对极端值敏感
- 不提供方差/标准差信息

---

## 方法二：β分布（PERT）估算法

### 公式

$$
\text{预期工期} = \frac{O + 4 \times M + P}{6}
$$

$$
\text{标准差} \sigma = \frac{P - O}{6}
$$

$$
\text{方差} \sigma^2 = \left(\frac{P - O}{6}\right)^2
$$

### 理论基础

PERT（Program Evaluation and Review Technique）估算基于β分布假设。β分布能够灵活地描述偏态分布——现实中多数任务的悲观侧往往比乐观侧延伸更远。

### 多阶段总工期

当有n个阶段（活动）时：

$$
\text{总预期工期} = \sum_{i=1}^{n} \text{预期工期}_i
$$

$$
\text{总标准差} = \sqrt{\sum_{i=1}^{n} \sigma_i^2}
$$

### 概率工期计算

基于正态分布近似（中心极限定理）：

| 概率 | Z值 | 工期公式 |
|------|-----|---------|
| P50 | Z=0.00 | 预期工期 |
| P68 | Z=1.00 | 预期工期 + σ |
| P75 | Z=0.67 | 预期工期 + 0.67σ |
| P85 | Z=1.04 | 预期工期 + 1.04σ |
| P90 | Z=1.28 | 预期工期 + 1.28σ |
| P95 | Z=1.64 | 预期工期 + 1.64σ |
| P99 | Z=2.33 | 预期工期 + 2.33σ |

### 适用场景

| 场景 | 说明 |
|------|------|
| 多阶段项目 | 3个以上活动串联 |
| 非对称分布 | O和P到M的距离明显不等 |
| 项目管理 | PMP/PMP标准实践 |
| 建筑/软件 | 行业标准方法 |

## 方法三：正态分布估算法

### 公式

$$
\text{均值} \mu = \frac{O + M + P}{3}
$$

$$
\text{方差} \sigma^2 = \left(\frac{P - O}{6}\right)^2
$$

> 注意：这里借用β分布的标准差公式作为正态分布的方差估计

### 使用条件

正态分布估算有三个前提条件：
1. **数据足够多**：通常需要 >30 个样本点
2. **对称性**：O和P到M的距离大致相等（|M-O| ≈ |P-M|）
3. **独立性**：各活动之间相互独立

### 注意事项

- 条件满足时，β分布和正态分布的结果接近
- 条件不满足时（如严重偏态），优先使用β分布
- 正态分布更适合大数据量的统计回归

### 适用场景

| 场景 | 说明 |
|------|------|
| 历史数据充分 | 有大量同类任务数据 |
| 重复性生产 | 制造/生产线任务 |
| 对称分布 | OMP值大致等距 |
| 统计检验 | 需要假设检验场景 |

## 方法四：蒙特卡洛模拟法（Monte Carlo Simulation）

> 当有多个阶段时，蒙特卡洛模拟是精度最高的方法。

### 基本原理

蒙特卡洛模拟通过对每个阶段进行大量随机抽样（通常≥2000次），模拟项目总工期的不确定性分布，而不是仅输出一个点估计值。

### 步骤详解

#### Step 1：计算各阶段的α、β参数（PERT-Beta分布）

$$
\alpha = 1 + 4 \times \frac{M - O}{P - O}
$$

$$
\beta = 1 + 4 \times \frac{P - M}{P - O}
$$

#### Step 2：单次随机抽样

每个阶段从PERT-Beta分布中抽取一个随机值：

```
随机值 = BETA.INV(RAND(), α, β, O, P)
```

重复 n 次（n ≥ 2000）。

#### Step 3：计算总工期

对于每次模拟，将所有阶段的值相加：

$$
\text{Total}_i = \sum_{j=1}^{m} \text{随机值}_{ij}
$$

#### Step 4：统计分组与频率密度

```
分组下限 = MIN × 0.96
分组上限 = MAX × 1.04
组距 = (上限 - 下限) / 分组数
```

频率密度 = 各组频数 / 模拟总次数

#### Step 5：累计概率

$$
F(k) = \sum_{i=1}^{k} f(i)
$$

其中 f(i) 为第 i 组的频率密度。

### 关键输出

| 指标 | 含义 | 用途 |
|------|------|------|
| **MIN** | 模拟中的最短总工期 | 理想情况 |
| **MAX** | 模拟中的最长总工期 | 最差情况 |
| **均值** | 总工期期望值 | 计划基准 |
| **中位数（P50）** | 50%概率完成 | 投标报价参考 |
| **P90** | 90%概率完成 | 风险缓冲 |
| **标准差** | 不确定性程度 | 风险量化 |
| **峰值位置** | 最可能工期 | 模态估计 |

### 蒙特卡洛 vs PERT

| 对比项 | PERT | 蒙特卡洛 |
|--------|------|---------|
| 计算复杂度 | 低（手工可算） | 高（需计算机） |
| 精度 | 近似值 | 高精度分布 |
| 输出 | 点估计+σ | 完整概率分布 |
| 适用阶段 | 1-5个 | 任意数量 |
| 偏态处理 | 近似 | 精确 |
| 多路径 | 不直接支持 | 天然支持 |

### 适用场景

| 场景 | 说明 |
|------|------|
| 多阶段复杂项目 | 5+个活动，路径多 |
| 高不确定性 | OMP跨度大 |
| 正式风险分析 | 需要概率分布 |
| 大型工程 | 建筑/基建/航天 |
| 投标与合同 | 需要P50/P90数据 |

## 方法五：CPM关键路径分析

### 基础概念

CPM（Critical Path Method）基于任务间的紧前关系，计算项目的最短工期。每个任务有四个关键时间参数：

| 参数 | 英文 | 含义 |
|------|------|------|
| ES | Earliest Start | 最早开始时间 |
| EF | Earliest Finish | 最早完成时间 |
| LS | Latest Start | 最晚开始时间 |
| LF | Latest Finish | 最晚完成时间 |
| TF | Total Float | 总时差（LS - ES） |

TF = 0 的任务即为**关键任务**，关键任务组成的路径为**关键路径**。

### 前向传递（ES/EF）

按拓扑顺序遍历：
- ES[任务] = max(EF[所有前置任务])
- EF[任务] = ES[任务] + 工期[任务]
- project_duration = max(所有EF)

### 后向传递（LS/LF）

按逆拓扑顺序遍历：
- LF[任务] = min(LS[所有后继任务])，终点任务LF = project_duration
- LS[任务] = LF[任务] - 工期[任务]

### 使用方法

```
输入: 任务列表 + 紧前关系（手动指定或自动规划）
输出: ES/EF/LS/LF/TF + 关键路径 + 项目总工期
```

## 方法六：多分布蒙特卡洛模拟

### 三种分布对比

| 分布 | 特点 | 适用场景 |
|------|------|---------|
| **PERT-Beta** | 基于OMP值，偏态灵活 | 项目管理标准方法 |
| **三角分布** | 简单直观，边界明确 | 缺乏历史数据时使用 |
| **泊松近似** | 聚焦M值附近 | 重复性高、波动小的任务 |

### 并行模拟流程

```text
每阶段:
  PERT-Beta随机值 = BETA.INV(RAND(), α, β, O, P)
  三角随机值 = TRIANGLE(O, M, P)
  泊松随机值 = POISSON(M)

每种分布 ≥2000 次 → 分位数 + 直方图 → 对比分析
```

## 方法七：任务重叠分析

基于扫描线算法，处理任务的时间区间事件：

1. 将所有任务的 start/end 转为事件列表（+1/-1）
2. 按时间排序后扫描，记录当前活跃任务数
3. 合并相邻且任务集合相同的区间
4. 提取：最大重叠数量区间 + 最长重叠时长区间
