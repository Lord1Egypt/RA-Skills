## Known Risks and Mitigations: <br>
Risk: A single install can enable a broad set of sensitive workflows, including finance, email, calendar, health, memory, shell, network, database, and recurring-job use cases. <br>
Mitigation: Install only the specific skills needed and review each selected skill's permissions, scripts, data handling, and external-access behavior before enabling it. <br>
Risk: Some included skills can process personal, financial, health, email, calendar, or workspace-memory data. <br>
Mitigation: Use least-privilege access, keep sensitive files local where possible, avoid unnecessary data sharing, and confirm deletion and retention behavior before use. <br>
Risk: Local-only and no-external-call claims may not fully describe runtime behavior if network features, scheduling, shell commands, or service-role credentials are enabled. <br>
Mitigation: Disable or require explicit approval for network access, scheduled jobs, shell execution, and privileged credentials unless they are necessary for the selected workflow. <br>
