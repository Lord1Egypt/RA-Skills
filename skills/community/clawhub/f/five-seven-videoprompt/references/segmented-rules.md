# 分段生成规则

当视频时长超过AI单次生成上限时，需要分段生成并保持一致性与连贯性。

---

## 一、核心原则（五必须）

| 必须 | 说明 |
|------|------|
| 角色卡全段共享 | 描述一字不改，每段完整写出 |
| 风格卡全段共享 | 渲染参数全段统一，每段完整写出 |
| 上段结尾 = 下段开头 | 动作延续不断，硬切+动作延续 |
| 每段结尾留钩子 | 为下段留出动作接口 |
| 帧率/比例/光追参数一致 | 全段技术规格完全统一 |

---

## 二、角色卡规则

**主体数量 × 角色卡数量 = 一一对应**

| 视频中主体数 | 角色卡 |
|-----------|--------|
| 1个 | 1张角色卡 |
| 2个 | 2张角色卡 |
| N个 | N张角色卡 |

角色卡格式：
```
角色名角色卡：
- 外貌：
- 发型：
- 装束：
- 武器/道具：
- 眼神：
```

**关键：** 每段必须完整写出角色卡内容，不写"同上"，方便用户独立复制每段提示词。

---

## 三、风格卡规则

风格卡格式：
```
风格卡：
- 比例：16:9 / 8K / UE5光追写实 / 24fps
- 升格：关键动作60fps
- 色调：[风格描述]
- 渲染：[渲染参数列表]
```

**关键：** 同角色卡一样，每段完整写出。

---

## 四、关键帧锚点法

每段开头必须明确三个锚点：

| 锚点 | 内容 | 示例 |
|------|------|------|
| 上段结尾帧 | 上一段最后一帧的确切画面和姿态 | 精灵右膝跪地左手撑法杖，恶兽右爪刚划过落空 |
| 本段开头帧 | 本段第一帧的确切画面和姿态 | 精灵顺势右脚发力站起，法杖横于胸前 |
| 动作延续 | 无缝/有间隔 | 无缝，侧身翻滚→站起是连续动作 |

**核心：不要只写"承接XXX"，而是写出具体画面——谁在哪里、什么姿势、什么状态。**

---

## 五、分段点设计

**分段点必须设计在"软切换"处：**

| ❌ 硬切（避免） | ✅ 软切（推荐） |
|----------------|----------------|
| 动作中途切断 | 动作完成后切换 |
| 镜头运动中切 | 镜头静止或匀速时切 |
| 场景突变 | 场景自然过渡 |
| 光线剧变 | 光线渐变过渡 |

---

## 六、衔接技巧表格

每套分段提示词末尾附衔接表格：

```
| 段落 | 上段结尾帧 | 本段开头帧 | 动作延续 |
|------|----------|----------|--------|
| 第一段→第二段 | [具体画面] | [具体画面] | 无缝/有间隔 |
| 第二段→第三段 | [具体画面] | [具体画面] | 无缝/有间隔 |
```

---

## 七、后期一致性三步

```
| 步骤 | 操作 |
|------|------|
| 1 | 去掉每段首尾黑帧，硬切拼接 |
| 2 | 全程色调统一，特殊节点注明色彩变化 |
| 3 | 音效全程连贯，不因分段断裂 |
```

---

## 八、一致性三锚点总结

| 锚点 | 内容 |
|------|------|
| 角色锚点 | 角色卡全程不变，AI每次生成同一个角色 |
| 风格锚点 | 风格卡全程不变，保证色调/光影/质感统一 |
| 动作锚点 | 段段之间动作承接不断，带动作钩子收尾 |
