## 🎯 原子动作动态更新机制（2026-03-07 10:40）

**用户要求：** 形成记忆，每次做事情时发现新动作就创建新动作

**核心原则：**
1. **动态发现：** 执行任务时，发现新动作 → 立即创建原子动作模块
2. **MECE 法则：** 所有原子动作必须满足
   - **Mutually Exclusive（相互独立）** - 不重复
   - **Collectively Exhaustive（完全穷尽）** - 不遗漏
3. **渐进穷举：** 逐渐完善原子动作库，不追求一次到位

**执行流程：**
```
执行任务 → 发现新动作 → 检查是否已存在
  ↓
不存在 → 创建新原子动作（MD + TXT）
  ↓
检查是否符合 MECE - 是否与其他动作重叠
  ↓
符合 → 纳入原子动作库
  ↓
更新总索引和分类
```

**MECE 检查清单：**
- [ ] 这个动作是否与其他动作有重叠？（相互独立）
- [ ] 这个动作是否还能继续拆分？（不可再分）
- [ ] 这个动作是否单一职责？（只做一件事）
- [ ] 这个动作是否可复用？（能被多个 Skill 使用）

**示例：**
```
任务：生成 HTML 专家点评
发现：需要"渲染 Mermaid 为 PNG"
检查：
  - 是否与现有动作重叠？ → 否（ATOM-VISUAL-006 只生成代码，不渲染）
  - 是否还能继续拆分？ → 否（单一动作）
  - 是否单一职责？ → 是（只负责渲染）
  - 是否可复用？ → 是（任何需要 Mermaid 图的场景都能用）
创建：ATOM-CANVAS-020 - 渲染 Mermaid 为 PNG
```
