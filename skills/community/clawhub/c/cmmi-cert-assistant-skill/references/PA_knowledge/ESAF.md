# ESAF — 确保安全性（CMMI V3.0 Domain-specific PA）

## PA 基本信息
- **缩写**：ESAF
- **英文**：Ensuring Safety
- **中文**：确保安全性
- **所属**：Domain-specific PA（领域特定实践域）
- **适用领域**：安全关键系统、航空航天、医疗设备、汽车电子

---

## CMMI V3.0 要求要点

### 核心目的
确保产品、服务、系统的安全性，防止安全事故。

### 关键实践（Specific Practices）
| 实践编号 | 实践内容 | 说明 |
|---------|---------|------|
| ESAF 1.1 | 识别安全需求 | 识别法律法规、标准要求的安全需求 |
| ESAF 1.2 | 分析安全风险 | 评估安全风险，确定风险等级 |
| ESAF 1.3 | 定义安全策略 | 制定安全策略和控制措施 |
| ESAF 1.4 | 设计安全机制 | 设计安全功能和安全机制 |
| ESAF 2.1 | 实施安全控制 | 执行安全策略和控制措施 |
| ESAF 2.2 | 验证安全机制 | 测试安全功能的有效性 |
| ESAF 3.1 | 管理安全事件 | 安全事件响应和处理 |

### 通用实践（Generic Practices）
- GP 1.1 执行特定实践
- GP 2.1 建立组织政策
- GP 2.2 策划过程
- GP 2.3 提供资源
- GP 2.4 分配责任
- GP 2.5 培训人员
- GP 2.6 管理配置
- GP 2.7 识别并记录干系人需求和接口
- GP 2.8 监督和控制过程
- GP 2.9 客观评估符合性
- GP 2.10 高层审查

## 用户文档 → PA 匹配规则

### 高概率匹配（主要PA）
- 文档名含"安全需求"、"Safety"、"安全案例" → **ESAF 主要匹配**
- 文档含安全风险评估、安全验证报告 → **ESAF 主要匹配**

### 中概率匹配（次要PA）
- 文档含安全架构设计 → 同时匹配 **TS**（技术服务）
- 文档含安全测试记录 → 同时匹配 **VV**（验证与确认）

## 差距检查清单（用于差距分析）

### ESAF 1.x — 安全策划
- [ ] 是否识别了安全需求和法规？

- [ ] 是否有安全策略和控制措施？

- [ ] 是否实施了安全控制？
- [ ] 是否有安全机制验证记录？

### ESAF 3.x — 安全事件管理
- [ ] 是否有安全事件响应计划？

## 评估访谈常见问题（模式D）

### 针对安全工程师的问题
1. "你的项目有哪些安全需求？如何管理的？"
2. "安全风险评估是如何进行的？"
3. "安全机制是如何验证的？有验证记录吗？"
4. "发生安全事件后如何处理？"

### 评分要点
- **ESAF 1.1-1.4**：是否系统识别安全需求并制定安全策略
- **ESAF 2.1-2.2**：安全控制是否实施，是否有验证记录
- **ESAF 3.1**：是否有安全事件管理流程
