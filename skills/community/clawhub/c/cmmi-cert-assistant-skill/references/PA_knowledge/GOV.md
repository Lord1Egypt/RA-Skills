# GOV — 治理（CMMI V3.0 Core PA）

## PA 基本信息#
- **缩写**：GOV
- **英文**：Governance
- **中文**：治理
- **所属**：Core PA（核心实践域）
- **对应 V2.0**：新增（V3.0 新增 Core PA）

---

## CMMI V3.0 要求要点#

### 核心目的#
确保项目和组织目标对齐，通过治理机制监督项目健康度，及时升级和解决问题。

### 关键实践#
| 实践编号 | 实践内容 |
|---------|---------|
| GOV 1.1 | 建立治理机制 |
| GOV 1.2 | 识别治理项和升级路径 |
| GOV 2.1 | 监督治理项 |
| GOV 2.2 | 解决治理项 |

## 对应模板文件（来自知识库）#

### 敏捷模型模板#
- 项目周报中的"需要升级的问题"部分 → **GOV 相关**

## 用户文档 → PA 匹配规则#

### 高概率匹配（主要PA）#
- 文档名含"治理"、"升级"、"项目健康度" → **GOV 主要匹配**
- 文档含升级记录、治理委员会会议记录 → **GOV 主要匹配**

### 中概率匹配（次要PA）#
- 文档含项目计划 → 同时匹配 **PLAN**
- 文档含周报/月报 → 同时匹配 **MC**（监控与控制）

---

## 差距检查清单#

### GOV 1.x — 策划#
- [ ] 是否有治理机制定义（治理委员会、升级路径）？
- [ ] 是否有项目健康度指标定义？

### GOV 2.x — 监督与解决#
- [ ] 是否有定期的治理审查记录（治理委员会会议）？
- [ ] 升级问题是否有跟踪闭环？

## 文档转换指导#

### 转换后的文档结构（治理报告模板）#
```markdown
# 项目治理报告

## 1. 治理机制#
（治理委员会组成、职责、会议频率）

## 2. 项目健康度#
| 指标 | 目标值 | 实际值 | 状态 | 趋势 |
|------|--------|--------|------|------|

## 3. 升级问题清单#
| 问题ID | 描述 | 严重等级 | 提出日期 | 负责人 | 解决日期 | 状态 |
|---------|---------|----------|----------|--------|----------|--------|

## 4. 治理委员会会议纪要#
（会议日期、参与者、讨论事项、决议、行动项）

## 5. 版本记录#
（报告版本、修订日期、修订内容）
```

## 评估访谈常见问题#

### 针对PM/高层管理者的的问题#
1. "项目的治理机制是什么？谁参与治理委员会？"
2. "项目问题如何升级？请展示升级记录。"
3. "治理委员会多久开一次会？有会议纪要吗？"

### 评分要点#
- **GOV 1.1-1.2**：治理机制是否明确定义
- **GOV 2.1-2.2**：治理项是否有监督并解决闭环
