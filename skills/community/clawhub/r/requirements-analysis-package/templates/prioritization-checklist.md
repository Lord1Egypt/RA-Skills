# Prioritization Checklist

## Value
- Does it solve a real user problem?
- Is the business value explicit?

## Urgency
- Is there a release or stakeholder deadline?
- Does delay create visible risk?

## Effort
- Can the team estimate implementation size?
- Are there cross-team dependencies?

## Confidence
- Are user needs validated?
- Are edge cases understood?

## Recommended Label
- Must have / Should have / Could have / Not now
