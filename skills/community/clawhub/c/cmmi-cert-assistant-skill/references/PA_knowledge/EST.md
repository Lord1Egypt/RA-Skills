# EST — 估算（CMMI V3.0 Core PA）

## PA 基本信息#
- **缩写**：EST
- **英文**：Estimating
- **中文**：估算
- **所属**：Core PA（核心实践域）
- **对应 V2.0**：相当于 PP（项目计划）中的估算部分

---

## CMMI V3.0 要求要点#

### 核心目的
确保项目估算基于可用数据，采用系统化方法，并随项目进展更新。

### 关键实践#
| 实践编号 | 实践内容 |
|---------|---------|
| EST 1.1 | 策划项目估算 |
| EST 1.2 | 建立和维护估算方法 |
| EST 2.1 | 实施估算 |
| EST 2.2 | 记录估算结果和依据 |
| EST 3.1 | 维护估算（随项目进展更新） |

## 用户文档 → PA 匹配规则#

### 高概率匹配（主要PA）#
- 文档名含"估算"、"规模估算"、"工作量估算" → **EST 主要匹配**
- 文档含故事点/功能点记录 → **EST 主要匹配**

### 中概率匹配（次要PA）#
- 文档含项目计划 → 同时匹配 **PLAN**
- 文档含历史数据对比 → 同时匹配 **MPM**（绩效与度量管理）

## 差距检查清单#

### EST 1.x — 策划#
- [ ] 是否有估算方法定义（故事点/功能点/专家判断）？
- [ ] 是否基于历史数据估算？

### EST 2.x — 实施#
- [ ] 估算是否有记录（含估算依据）？
- [ ] 估算结果是否得到干系人承诺？

### EST 3.x — 维护#
- [ ] 估算是否随项目进展更新（重新估算）？
- [ ] 估算偏差是否有记录和分析？

```markdown
（估算对象、估算方法、历史数据来源）

## 2. 规模估算#
| 模块 | 描述 | 规模单位 | 规模值 | 依据 |
|------|------|----------|--------|------|

|------|--------|--------|-------|----------|

## 7. 版本记录#
（估算版本、修订日期、修订内容、修订人）
```

## 评估访谈常见问题#

### 针对PM（项目经理）的问题#
1. "项目估算是怎么做的？用了什么方法？"
2. "估算是基于历史数据吗？请展示历史数据。"
3. "估算偏差如何管理？有记录吗？"

### 评分要点#
- **EST 1.1-1.2**：估算方法是否系统化
- **EST 2.1-2.2**：估算结果是否有记录（含依据）
- **EST 3.1**：估算是否随项目进展维护（重新估算）
