## D0 需求分析域范本

### D0-01 文档类型识别范本

**任务目标**：识别用户需要的文档类型

**执行步骤**：

1. **收集用户需求**
   - 询问用户文档的用途
   - 了解文档的目标受众
   - 确认文档的使用场景

2. **分析文档特征**
   - 报告类：正式、规范、有结论
   - 方案类：有目标、有计划、有预算
   - 合同类：严谨、规范、有法律效力
   - 简历类：简洁、专业、重点突出

3. **判断文档类型**
   - 根据需求匹配文档类型
   - 记录选择理由
   - 确认文档类型

**输出示例**：

```markdown
## 文档类型识别报告

### 用户需求
- 文档用途：年度工作总结
- 目标受众：公司管理层
- 使用场景：年终汇报

### 文档类型判断
- 文档类型：报告
- 选择理由：工作总结属于正式汇报文档，需要规范的结构和格式

### 文档特征
- 结构：封面→摘要→正文→结论
- 样式：正式、规范、层次分明
- 颜色：以蓝色、灰色为主色调
```

### D0-02 内容复杂度评估范本

**任务目标**：评估文档内容复杂度

**执行步骤**：

1. **统计内容元素**
   - 章节数量
   - 段落数量
   - 表格数量
   - 图片数量

2. **分析结构复杂度**
   - 层级深度
   - 交叉引用
   - 目录结构

3. **评估格式要求**
   - 特殊格式要求
   - 自定义样式
   - 嵌入对象

**输出示例**：

```markdown
## 内容复杂度评估报告

### 内容元素统计
- 章节数量：8章
- 段落数量：约50段
- 表格数量：5个
- 图片数量：3张

### 结构复杂度
- 层级深度：3级（章→节→小节）
- 交叉引用：无
- 目录结构：自动生成

### 格式要求
- 特殊格式要求：无
- 自定义样式：使用标准样式
- 嵌入对象：表格和图片

### 复杂度等级
- 复杂度等级：中等
- 评估依据：内容元素适中，结构清晰，格式标准
```

### D0-03 格式要求分析范本

**任务目标**：分析文档格式要求

**执行步骤**：

1. **确定格式类型**
   - 正式格式：用于正式场合
   - 非正式格式：用于内部沟通
   - 学术格式：用于学术场合

2. **分析格式要求**
   - 字体要求
   - 颜色要求
   - 间距要求
   - 页面布局要求

3. **评估格式复杂度**
   - 标准格式：使用预设样式
   - 自定义格式：需要特殊处理

**输出示例**：

```markdown
## 格式要求分析报告

### 格式类型
- 格式类型：正式格式
- 选择依据：年度工作总结属于正式汇报文档

### 格式要求
- 字体要求：标题使用黑体，正文使用宋体
- 颜色要求：以蓝色、灰色为主色调
- 间距要求：段前段后0.5行，行距1.5倍
- 页面布局：A4纸张，上下2.54cm，左右3.17cm

### 格式复杂度
- 格式复杂度：标准
- 评估依据：使用标准格式，无需特殊处理
```

### D0-04 输出格式选择范本

**任务目标**：选择合适的输出格式

**执行步骤**：

1. **分析需求**
   - 文档用途
   - 使用场景
   - 软件环境

2. **匹配格式**
   - 标准Word：兼容性好，易于编辑
   - 模板应用：效率高，风格统一
   - 自定义：灵活性强

3. **记录选择理由**

**输出示例**：

```markdown
## 输出格式选择报告

### 需求分析
- 文档用途：年度工作总结
- 使用场景：年终汇报
- 软件环境：Microsoft Word

### 格式选择
- 输出格式：标准Word
- 选择理由：年度工作总结需要正式格式，标准Word兼容性好，易于编辑和分享

### 格式特点
- 文件扩展名：.docx
- 兼容性：支持Microsoft Word 2007及以上版本
- 编辑性：支持全文编辑
```
