# RSK — 风险与机会管理（CMMI V3.0 Core PA）

## PA 基本信息
- **缩写**：RSK
- **英文**：Risk and Opportunity Management
- **中文**：风险与机会管理
- **所属**：Core PA（核心实践域）
- **对应 V2.0**：相当于 RSKM（风险管理）

---

## CMMI V3.0 要求要点#

### 核心目的
确保项目风险被识别、分析、应对，机会被捕捉和利用。

### 关键实践（Specific Practices）
| 实践编号 | 实践内容 | 说明 |
|---------|---------|------|
| RSK 1.1 | 识别风险与机会 | 风险登记册、头脑风暴记录 |
| RSK 1.2 | 评估风险与机会 | 概率/影响矩阵、风险值计算 |
| RSK 2.1 | 制定风险应对计划 | 应对措施、应急储备 |
| RSK 2.2 | 制定机会利用计划 | 机会捕捉措施 |
| RSK 3.1 | 监控风险与机会 | 定期风险审查记录 |
| RSK 3.2 | 实施风险应对/机会利用 | 应对措施执行记录 |

### 通用实践（Generic Practices）
- GP 1.1 ~ GP 2.10（完整覆盖）

## 用户文档 → PA 匹配规则#

### 高概率匹配（主要PA）
- 文档名含"风险"、"风险登记册"、"Risk" → **RSK 主要匹配**
- 文档含概率/影响矩阵 → **RSK 主要匹配**

### 中概率匹配（次要PA）
- 文档含问题跟踪（不只是风险）→ 同时匹配 **MC**（监控与控制）
- 文档含机会识别 → **RSK 独有**（V3.0 新增"机会管理"）

## 差距检查清单（用于差距分析）

### RSK 1.x — 识别与评估
- [ ] 是否有风险登记册（唯一ID、描述、类别）？
- [ ] 风险是否有概率和影响评分？
- [ ] 是否识别了机会（不只是风险）？

### RSK 2.x — 应对策划
- [ ] 每个高风险是否有应对措施？
- [ ] 是否有应急储备（时间/预算）？
- [ ] 机会是否有利用计划？

### RSK 3.x — 监控与实施
- [ ] 是否定期审查风险状态（每周/每双周）？
- [ ] 风险应对措施是否有执行记录？
- [ ] 风险状态变更是否有记录？

## 评估访谈常见问题（模式D）

### 针对PM（项目经理）的问题
1. "项目的主要风险有哪些？请展示风险登记册。"
2. "风险是如何识别和评估的？用了什么方法？"
3. "高风险有哪些应对措施？有执行记录吗？"
4. "风险审查的频率如何？谁参与？"
5. "V3.0 新增了'机会管理'，你们是如何管理机会的？"

### 评分要点
- **RSK 1.1-1.2**：风险识别是否系统化，评估是否有依据
- **RSK 2.1-2.2**：应对措施是否具体可落地
- **RSK 3.1-3.2**：风险监控是否有定期记录，措施是否执行
