## 二、股权合作Skill接口

### 2.1 接口内容

| 接口类型 | 输入 | 输出 | 说明 |
|----------|------|------|------|
| **三长老治理** | 项目信息、成员列表 | 三长老任命 | 复用三长老机制 |
| **信用台账** | 成员ID | 信用等级、分数 | 查询/更新信用 |
| **账务系统** | 分账方案 | 分账记录、流水 | 账务处理 |

### 2.2 数据流转

```python
# 信用查询接口
def query_credit(member_id):
    """
    查询成员信用信息
    输入：member_id - 成员ID
    输出：{
        'credit_level': 'A',  # S/A/B/C/D
        'credit_score': 85,   # 0-100
        'history': [...]      # 历史评价记录
    }
    """
    return credit_system.get(member_id)

# 信用更新接口
def update_credit(member_id, change):
    """
    更新成员信用信息
    输入：member_id, change - 变化值
    输出：更新后的信用信息
    """
    return credit_system.update(member_id, change)

# 分账记录接口
def record_distribution(project_id, distribution):
    """
    记录分账信息
    输入：project_id, distribution - 分账明细
    输出：分账记录ID
    """
    return account_system.record(project_id, distribution)
```

### 2.3 复用内容

| 复用内容 | 复用方式 | 说明 |
|----------|----------|------|
| **三长老角色** | 直接引用 | 复用其治理功能 |
| **信用评级体系** | API调用 | 查询/更新信用 |
| **账务处理流程** | 直接引用 | 复用分账流程 |
| **协议模板** | 模板引用 | 复用SPV协议 |

## 七、数据标准

### 7.1 统一数据格式

| 数据类型 | 格式 | 说明 |
|----------|------|------|
| **项目ID** | OPC-PJ-YYYY-NNNN | 年份+序号 |
| **成员ID** | OPC-MB-YYYY-NNNN | 年份+序号 |
| **时间格式** | YYYY-MM-DD HH:mm:ss | 标准时间格式 |
| **金额格式** | 保留2位小数 | 单位：元 |

### 7.2 接口认证

| 认证方式 | 说明 |
|----------|------|
| **Token认证** | API调用时携带Token |
| **权限验证** | 根据角色验证权限 |
| **数据加密** | 敏感数据加密传输 |
