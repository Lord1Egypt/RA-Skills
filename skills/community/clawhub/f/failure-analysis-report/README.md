# failure-analysis-report

A structured documentation skill for licensed forensic engineers (PE/SE/ME/EE) and expert witnesses that converts investigation findings into a court-ready failure analysis report for insurance claims and litigation matters.

---

## What It Does

Guides a forensic engineer through every section of a failure analysis investigation — scene documentation methodology, physical evidence inventory, applicable standards in effect at the time of construction or manufacture, failure mode hypothesis screening, root cause determination, and expert opinion — and produces a DRAFT report for licensed PE review before delivery to an attorney, insurer, or regulatory body.

## When to Use

- Documenting a structural, mechanical, fire, electrical, or product failure investigation for insurance or litigation
- Drafting an expert report for Rule 26 disclosure or a court-appointed engineering investigation
- Preparing a failure analysis memo for a product liability, construction defect, or casualty loss matter
- Any engineering investigation requiring a documented root cause opinion

## What It Produces

- Investigation identification and scope statement
- Scene documentation methodology (access, photo protocol, evidence preservation)
- Physical evidence inventory with chain-of-custody flags
- Applicable standards and codes table (timestamped to construction/manufacture date, not current codes)
- Failure mode hypothesis screening matrix (2–5 hypotheses, ranked)
- Root cause determination with supporting evidence and exclusions
- Expert opinion section
- Limitations statement
- Licensed PE authorization block for attorney/insurer delivery

## Scope and Boundaries

- Produces DRAFT documentation only — all content requires licensed PE review, signature, and stamp before delivery to any party
- Applicable-standards-at-time-of-construction is a hard rule: the skill always uses the edition in effect at the time, not the current edition
- Does not fabricate test data, measurement values, or failure observations — all data is entered by the engineer
- Does not constitute a regulatory filing or safety directive
- Chain-of-custody flags are reminders; actual CoC records are the engineer's responsibility

## Requirements

- Licensed Professional Engineer (PE) overseeing the investigation and reviewing the DRAFT
- Scene access notes, photographs, and physical evidence descriptions
- Project documentation (drawings, specifications, maintenance records) as available
