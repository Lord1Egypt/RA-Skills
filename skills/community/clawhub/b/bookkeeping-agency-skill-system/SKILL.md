# 代理记账行业十大功能集群技能体系

## 核心理念

每个能力单元是一个**可独立执行、可自由组合**的原子任务。AI拿到输入就能执行，输出可被任意其他单元消费。

代理记账行业的核心矛盾是：**高度标准化的服务 vs 极度个性化的客户需求**。本技能体系将代账全业务拆解为可原子化执行的Pipeline——从市场情报到获客签约、从票据采集到纳税申报、从合规审核到风控预警，每一步都有明确的输入、输出和依赖关系。不是"AI帮你记账"，是AI驱动整个代账机构的高效运转。

## 行业特性声明

1. **服务双重性**：代账机构既是服务提供方（面向客户），又是监管遵从方（面向税局）——Pipeline必须同时满足客户体验和税务合规
2. **强监管环境**：金税四期、全电发票、数电票改革持续深化——合规不是加分项是底线
3. **人效即利润**：代账行业人均服务客户数直接决定利润——标准化和自动化是生存命题
4. **季节性波动**：汇算清缴季(1-5月)、工商年报季(1-6月)产生极端峰值——Pipeline必须支持弹性调度
5. **信息不对称**：客户不懂财税、代账机构不懂客户业务——知识转化是核心能力
6. **信任经济**：客户一旦签约极少更换，但一旦出错信任崩塌——风控和差错管理是生命线

## 能力单元Schema

每个单元包含5个字段：

| 字段 | 说明 |
|------|------|
| **输入** | 启动所需的数据/上下文，标注必选/可选 |
| **输出** | 产物格式与颗粒度 |
| **依赖** | 前置必须完成的单元ID，无依赖=入口单元 |
| **AI自治度** | ⬛全自动 / 🟨半自动(需人工确认) / ⬜辅助(人工主导) |
| **组合接口** | 输出可被哪些单元消费（→单元ID列表） |

## 执行规则

1. **必读加载**：首次使用本技能时，必须用Read工具读取 `references/index-and-pipelines.md` 获取完整单元索引和Pipeline编排，这是执行所有后续操作的前提
2. **行业校准优先**：读取 `references/industry-calibration.md` 确定代账机构类型和业务模式
3. **按需加载**：识别集群后，用Read工具读取对应参考文件获取完整单元规格
4. **依赖检查**：执行前确认依赖单元的输出已就绪，缺失则先执行依赖
5. **自由组合**：无依赖冲突的单元可并行；有依赖的按序执行
6. **跨集群流转**：一个单元的输出可作为另一集群单元的输入，按组合接口串联
7. **合规优先**：涉及税务申报、发票处理、客户数据的单元，必须经过C6合规审核后方可执行
8. **季节调度**：汇算清缴季(1-5月)和工商年报季(1-6月)相关Pipeline享有执行优先级

## 行业校准

读取 `references/industry-calibration.md` 获取4种代账机构类型校准模板：
1. **传统代账公司**：线下为主，中小规模，人工+工具混合，以稳定续费为核心
2. **互联网代账平台**：线上获客为主，高度标准化，AI+自动化驱动，以规模增长为核心
3. **财税咨询公司**：高客单价，税筹+咨询为主，专家驱动，以价值创造为核心
4. **自动化代账平台**：持证SaaS+全自动报税，客户自助为主，以极低价格+规模效应颠覆传统代账（典型：自记账/伽玛账/自会计）
