# WildIdea 输出格式创新配方

2026-05-13 多轮迭代产出，经 vivo 手机影像 6 轮迭代验证。这些是可选增强，不改变 V4 核心流程（注入→并放→输出）。

---

## 1. 钉子压缩协议

将每条钉子标准化为 3 行，控制在 45 字以内：

```
〖方向〗核心约束（≤15字）→ 〖操作〗一个具体动作（≤15字）→ 〖锚点〗可验证数值/日期/名称
```

**示例：**
```
〖场景专注〗→ 只优化一个极端场景 → 水族馆隔玻璃消除反光
〖时间绑定〗→ 与节气同步更新算法 → 每年仅立春/芒种/白露/大雪
〖资源锁〗→ 砍6个低使用率模式 → 全景/文档/慢动作/延时/双视野/高像素
```

**效果：** 用户 3 行内决定"这个方向感不感兴趣"。减少阅读认知成本。

---

## 2. 用户画像感知输出

识别用户角色后，钉子按不同格式呈现（不改变钉子内容）：

| 用户画像 | 输出格式 | 示例 |
|---------|---------|------|
| **工程师** | 参数对照表 + 原理说明 | `方法B: 量子阱结构→暗电流从2.3e⁻/s降至0.8e⁻/s` |
| **设计师/产品** | 意象并置图 + 体验描述 | `像水族馆玻璃下的鱼：触到玻璃之前，你不知道中间隔着一层` |
| **管理者/VP** | 决策树 + ROI估算 | `选A: 半年工期, 2人, 预期用户感知提升30%, 但需要砍B功能` |
| **研究者** | 文献缺口 + 可验证假设 | `当前「隔物拍摄」公开文献零篇, 假设: 偏振滤光+深度估计可达85%` |

**实施方式：** 在 Step 0 阶段加 1 句角色检查。如果无法确定，默认用参数表格式。

---

## 3. 藏比乡配置

西藏那曲市比如县藏比乡：人口<2000、面积>1000km²，治理哲学是做精做准到每件事不可替代。

默认缩减运行时参数：

| 参数 | 标准模式 | 藏比乡模式 |
|------|---------|-----------|
| 🔵 预置远域 | 3个 | 2个 |
| 🟢 种子库 | 2个 | 1个 |
| 🟠 随机组字 | 1个 | 1个 |
| 总钉子数 | 6条 | ≤3条 |
| 输出风格 | 完整表+建议 | 高密度，一行是一行 |
| 保留条件 | — | 每条必须通过"无可替代性测试" |

## 4. 极端承诺模式

不是"6条你选"，是"1条你必须试试"。

| 条件 | 标准模式 | 极端承诺模式 |
|------|---------|-------------|
| 输出条数 | 6条 | 1条 |
| 舒适度 | VP皱眉即可 | VP剧烈反对才放行 |
| 最小启动成本 | 不要求 | 必须标注（±[X]元） |
| 止损线 | 不要求 | 自带：6个月无效果自动关停 |
| 验证门槛 | 未找到已实现 | 需搜索+1个具体竞品名称确认无相同方案 |

每次极端承诺输出后追加一句：
> 「这条如果 6 个月内没人反对，说明还不够野。到时再换。」

---

## 5. 先拆解再注入协议

标准 V4 流程是用户问题不动、旁边放远域。但有些问题"太结构化了"（如 vivo 影像这类行业竞争分析的题目），远域贴上去容易被惯性吸收。

可选前置步骤：

```
用户问题 → 溶解 → 问1个元问题 → 空白状态 → 注入远域 → 重新凝固输出
```

**元问题模板：**
> "在你当前的框架里，哪些方向你已经放弃了？"

用户回答后，把那些"被放弃的方向"也标记为隔离区（等同于已被ban），然后在"什么都不敢想了"的状态下注入远域。

**适用条件：** 跑完 3+ 轮迭代后，用户说"大部分的答案是行业内已经做到或考虑到的"时激活。
