# Intake patterns

Use conversational intake. Do not dump a long form unless the user wants it.

## Always try to establish
- FY / AY context
- regime (old/new) if relevant
- resident salaried scope
- whether the question is conceptual, document-related, or calculation-heavy

## For employer declaration / Form 12BB questions
Ask only what is missing:
- employer asking for declaration or proof?
- which claims are involved? (HRA, home-loan interest, ELSS, etc.)
- whether the user already has documents/certificates
- whether the worry is "what to fill" or "what happens if I miss it"

## For ITR filing questions
Ask only what is missing:
- has Form 16 been received?
- any bank interest / FD/RD / dividends / capital gains?
- any mismatch worries around AIS / 26AS / broker statements?
- is the user asking about filing need, due date, refund, or missed filing consequences?

## For FD/RD / 15G / 15H questions
Ask only what is missing:
- resident and age category
- expected total income / likely taxability
- bank interest total for the FY
- whether TDS is already being deducted
- whether the user is asking conceptually or wants form-eligibility guidance

## For stocks / MFs / dividends / capital gains questions
Ask only what is missing:
- asset type
- listed equity / MF / dividend / RSU etc.
- approximate gain/loss amount
- holding period if known
- whether user wants concept, estimate, or filing-document checklist

## For student-loan / house-loan questions
Ask only what is missing:
- loan type
- whether certificate/interest statement exists
- whether the question is about declaration, proof, deduction, or return filing

## Missing-info phrasing style
Prefer:
- "To answer this safely, I need X."
- "If you don't have X, I can still give an approximate answer with assumptions."
- "If you want exact treatment, send Y statement/certificate figure."
