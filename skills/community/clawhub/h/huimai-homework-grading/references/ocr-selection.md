# OCR 技术选型建议

## 适用场景分析

中小学作业批改场景的 OCR 有独特要求，与通用文档 OCR 不同：

| 场景特征 | 要求 |
|----------|------|
| 手写体为主（学生作答） | 手写识别准确率是关键 |
| 印刷体为辅（题目文本） | 印刷体识别成熟，门槛低 |
| 数学公式/化学方程式 | 需要公式识别能力 |
| 英语作文/语文作文 | 长文本手写识别 |
| 答题卡/填空/选择题 | 结构化版面分析 |
| 教师批改标记（勾/叉/划线） | 标记符号识别 + 语义理解 |

## 推荐路径

### 第一阶段（MVP 验证，0-1）
**PaddleOCR（开源）+ 自研版面分析**

- 成本: 0元，纯服务器资源
- 优势: 完全本地部署，数据不外传
- 不足: 手写准确率不如商业方案，需要调优
- 适合: 验证流程、小规模试点（<500学生）

### 第二阶段（规模推广，1-100）
**百度OCR教育版（主）+ PaddleOCR（辅，备用降级）**

- 百度OCR教育版的手写识别和结构化版面分析最成熟
- 支持私有化部署（数据安全合规要求）
- 按量计费，成本可控

### 第三阶段（深度定制，100+）
**自研微调模型（基于PaddleOCR/ERNIE）**

- 收集学校自有数据，微调手写识别模型
- 针对特定教材版本优化
- 逐步减少对商业 API 的依赖

## 数据安全与合规

- **中小学数据敏感**: 学生作业内容属个人信息，需谨慎处理
- **本地/私有云优先**: 推荐第一阶段开源方案 + 第二阶段私有化部署
- **数据脱敏**: 对接商业API前需进行学生姓名/学号脱敏
- **合规备案**: 涉及学生数据需符合《个人信息保护法》及教育部门相关规定

## 关键评估指标

| 指标 | 可接受 | 良好 | 优秀 |
|------|--------|------|------|
| 手写数字准确率 | ≥90% | ≥95% | ≥98% |
| 手写中文准确率 | ≥85% | ≥92% | ≥96% |
| 数学公式识别率 | ≥80% | ≥90% | ≥95% |
| 批改标记识别率 | ≥85% | ≥92% | ≥97% |
| 单页处理速度 | ≤3s | ≤1s | ≤0.5s |
| 价格 | - | ≤0.02元/页 | ≤0.01元/页 |
