# CompleteTech LLC Proposal Positioning

## Core Offer

CompleteTech LLC agentic development proposals should frame the work as practical, bounded workflow implementation, not generic AI transformation.

Use this offer frame:

- identify one valuable repeated workflow
- map current state, systems, and decision points
- design a bounded agent workflow
- implement tool routing, retrieval, draft generation, approval gates, and logs
- create evaluation examples and acceptance criteria
- document the workflow and hand off operating guidance
- support the client through review, launch, and iteration

## Preferred Language

Use:

- "bounded agentic workflow pilot"
- "human approval gates"
- "workflow discovery and implementation"
- "evaluation examples and regression checks"
- "logs, monitoring, and handoff documentation"
- "non-production prototype unless production deployment is separately authorized"
- "approval required before external communication, production changes, purchases, or data export"

Avoid:

- "fully autonomous business decisions"
- "replace your team"
- "guaranteed savings"
- invented case studies or ROI metrics
- legal, tax, accounting, medical, financial, employment, housing, credit, biometric, surveillance, or safety-critical compliance claims

## Buyer Problems

Good proposal pain statements are concrete:

- requests arrive faster than the team can classify and route them
- knowledge is scattered across documents, inboxes, tickets, and chat
- staff repeat drafting, checking, and handoff work
- review is inconsistent because context is missing
- AI experiments exist but lack evaluation, approval gates, logs, and deployment discipline
- leadership wants AI progress without uncontrolled production risk

## Proposal Proof Rules

Use only proof supplied by the user. If no proof is supplied, use process credibility:

- "The pilot will produce a prototype, evaluation set, acceptance criteria, and handoff documentation."
- "The first milestone is a workflow map and risk/control review."
- "The system will require human approval before sensitive actions."

## Standard Scope Building Blocks

- discovery workshop
- current-state workflow map
- future-state reviewed-agent workflow
- data/tool access review
- prototype implementation
- retrieval and tool-routing setup
- approval queue
- evaluation examples
- prompt-injection and permission checks
- logging/monitoring recommendations
- documentation and handoff
- support period

## Common Exclusions

Include exclusions when the proposal may become contract input:

- production deployment unless explicitly included
- regulated-use validation unless separately scoped
- legal, medical, financial, employment, housing, insurance, credit, biometric, surveillance, or safety-critical final decisions
- third-party software, model/API usage, hosting, and data costs unless listed
- client data cleanup or labeling at scale unless listed
- ongoing managed operations unless listed
