# Anti Generic Writing Rules / 反车轱辘写作规则

用途：防止报告停留在“身份、现金流、机会、路径、网络”等抽象词。所有默认用户报告都要通过本规则。

## 每段必检

每段生成后，检查 6 个问题：

1. 有没有具体年份或阶段？
2. 有没有具体生活/工作场景？
3. 有没有现实代价，例如钱、时间、健康、关系、签证、房贷、岗位或家庭照护？
4. 有没有情绪代价，例如不甘、焦虑、孤独、松一口气、安全感、错过感或羞耻感？
5. 有没有指出用户当年可能高估或低估了什么？
6. 有没有 7-30 天内能执行的验证动作？

缺 2 项以上，重写。删掉主题名后可以套到任何案例，也要重写。

## 抽象词处理

下列词可以使用，但后面必须跟具体解释：

- 身份感
- 现金流
- 机会密度
- 路径牵连
- 资源网络
- 稳定感
- 国际化
- 被看见
- 技术门槛
- 作品资产

失败写法：

> 北京会带来更高机会密度和资源网络。

合格写法：

> 2016-2019 年，北京内容和互联网团队更容易集中在平台会议、热点复盘、增长项目和同行饭局里。它会提高你被比较、被评价、被带进项目的概率，也会带来房租、通勤和睡眠代价。

## 事件冲击卡写法

每张卡合计 160-260 字，六段都要具体：

| 字段 | 必须出现 |
| --- | --- |
| 事件锚点 | 年份/阶段 + 事件或周期 |
| 生活场景 | 用户能看见的日常画面 |
| 现实代价 | 钱、时间、健康、关系、签证、房贷、岗位或家庭照护 |
| 情绪代价 | 不甘、焦虑、孤独、松一口气、安全感、错过感或羞耻感 |
| 误判点 | 高估/低估 + 误判对象 |
| 今天验证 | 人、材料、动作、反馈标准 |

## 禁用泛化句

看到这些句式时，优先重写：

- “这会影响你的职业发展。”
- “这会带来更多机会。”
- “你需要提升自己的能力。”
- “你需要扩大网络。”
- “这说明你的路径依赖很强。”
- “你要提高可见度。”

重写方向：

- 说清哪一年、哪类团队、哪类项目。
- 说清钱、时间、睡眠、关系、签证、房贷或岗位怎样变化。
- 说清今天找谁聊、写什么材料、发给谁、看什么反馈。

## 失败判定

出现以下情况，直接判定失败并重写：

- 删掉主题名后可以套到任何案例。
- 只有抽象词，没有具体年份和生活场景。
- 只讲影响因素，没讲当时用户会怎么过。
- 只讲机会与代价，没讲用户会怎么误判。
- 只给建议，没给现实验证动作。
- 判词来自因素分析，没来自自我叙事断裂。

## 事件体感规则

从事件库调用事件后，必须写出用户体感版解释：

| 字段 | 检查 |
| --- | --- |
| 事件 | 有没有年份或阶段。 |
| 用户当时可能看到什么 | 有没有生活画面。 |
| 它真正改变了什么 | 有没有现实代价。 |
| 它可能让用户产生什么情绪 | 有没有情绪代价。 |
| 它最容易被误判成什么 | 有没有误判点。 |
| 今天还能怎么验证 | 有没有低风险动作。 |

如果事件无法转成用户体感场景，只放附录。

## 判词规则

判词必须给遗憾命名。它可以尖锐，但不能审判用户。

合格判词通常包含：

- 一个具体承载物：城市、签证、房子、体制名额、上一家公司、AI 作品。
- 一个交换关系：安全感换可见度、熟悉环境换国际试炼、首付换流动性、收藏工具换作品空白。
- 一个情绪来源：不甘、孤独、松一口气、后悔、被落下、被看见。

## 30 天验证实验

每份报告最后必须给 4 周实验：

- 第 1 周：找一个真实人或真实材料。
- 第 2 周：写一页案例或做一个最小流程。
- 第 3 周：拿给 3 个可信对象看。
- 第 4 周：根据反馈决定加码、缩小或换题。

动作要低风险。禁止鼓励冲动离职、搬家、买房、投资、分手或借钱。

## 自检

最终输出前逐项打分：

| 维度 | 5 分标准 |
| --- | --- |
| 首屏命中感 | 用户能在 5 秒内看到自己的遗憾被命名。 |
| 分叉图 | 现实结果、假设结果、回补结果一眼能懂。 |
| 事件冲击卡 | 每张卡有年份、生活场景、现实代价、情绪代价、误判和验证。 |
| 反车轱辘 | 抽象词后面都有具体解释。 |
| 时代事件 | 4-6 个事件和主题高度相关。 |
| 反浪漫化 | 未选择结果同时写收益和代价。 |
| 30 天实验 | 4 周动作可执行，且能得到现实反馈。 |

任一维度低于 4 分，回到模板或事件引擎重写。
