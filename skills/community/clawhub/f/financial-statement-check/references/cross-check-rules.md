# 三表勾稽与跨表校验规则

## 概述

勾稽关系（Cross-check / Reconciliation）是指财务报表内部及报表之间数据的逻辑一致性关系。本文档定义了 **资产负债表 / 利润表 / 纳税申报表** 的：

1. **时间检查**（必输出）
2. **字段完整性检查**（仅异常时输出）
3. **原始指标输出**（必输出）
4. **核心指标检查**（仅异常时输出）
5. **跨表校验**（仅异常时输出）
6. **用户自定义检查项**（可选，按需输出）

## 容差规则

由于四舍五入、单位转换、税会差异等原因，校验允许一定容差：

| 容差级别 | 容差范围 | 适用场景 |
|---------|---------|---------|
| 严格 | ±0.01 万元 | 表内加减关系 |
| 宽松 | ±1.00 万元 | 表间对应关系 |
| 幅度型 | ±10% | 增值税应税销售额 vs 营业收入 |

**判定规则**：
- `|左值 - 右值| ≤ 容差` → ✅ 通过 / 符合预期
- `|左值 - 右值| > 容差` → ❌ 不通过 / 异常
- 任一值为 `null`（数据缺失） → ⚠️ 无法校验

## 一、时间检查（必输出）

### T-01：三表时间期一致性

| 项目 | 说明 |
|------|------|
| 检查内容 | 资产负债表、利润表、纳税申报表的**发生时间窗口**是否一致 |
| 输出 | 发生时间窗口（如 `2024-01-01 ~ 2024-12-31`） + `一致` / `不一致` |
| 是否必输出 | ✅ 是 |
| 备注 | 时间窗口取三表中实际发生业务所跨越的起止日期 |

### T-02：自然年错位检查

| 项目 | 说明 |
|------|------|
| 检查内容 | 报告期是否与自然年错位 |
| 输出 | `错位` / `无错位` |
| 判定 | 起始日 ≠ `XXXX-01-01` 或截止日 ≠ `XXXX-12-31`（且非标准月末/季末） → 错位 |
| 是否必输出 | ✅ 是 |

### T-03：申报类型识别

| 项目 | 说明 |
|------|------|
| 检查内容 | 纳税申报表的申报类型 |
| 输出 | `月度` / `季度` / `年度` |
| 判定 | 依据税款所属期跨度（见 field-mapping.md） |
| 是否必输出 | ✅ 是 |

## 二、字段完整性检查（仅异常时输出）

对三表中的**必填字段清单**进行缺失校验（清单见 `field-mapping.md`）：

- **资产负债表必填**：总资产、总负债、所有者权益、货币资金、短期借款、营业收入
- **利润表必填**：营业收入、营业利润、利润总额、净利润
- **纳税申报表必填**：应税销售额、销项税额、进项税额、应纳税额、营业收入、利润总额、应纳税所得额、应纳所得税额

### 输出规则

| 项目 | 说明 |
|------|------|
| 输出内容 | 财报三表数据是否完整 |
| 是否必输出 | ❌ **仅异常时输出** |
| 异常行格式 | 针对每个缺失字段，提供 **50字以内说明** + **查验引导（位置信息）** |

**示例**：
```
❌ 【资产负债表】货币资金 缺失：未能从报表中识别到该科目金额。
   查验位置：资产负债表·流动资产首行（PDF第3页）
```

## 四、核心指标检查（仅异常时输出）

以下检查仅在结果为"否 / 不通过"或存在异常时输出，输出格式：**判定结果 + 50字以内说明 + 查验引导（位置信息）**。

### C-01：增值税应税收入 = 会计收入？

```
tax_return.taxable_sales ≈ income_statement.revenue
```

| 项目 | 说明 |
|------|------|
| 左值 | 增值税纳税申报表·应税销售额 |
| 右值 | 利润表·营业收入 |
| 容差 | ±1.00 万元 |
| 输出 | `是` / `否` |
| 必输出 | ❌ 否（仅不通过时输出） |

### C-02：应纳税所得额 = 利润总额？

```
tax_return.taxable_income ≈ income_statement.total_profit
```

| 项目 | 说明 |
|------|------|
| 备注 | 二者的差异本质是"纳税调整"，完全相等属于特殊情况；此项用于提示需关注税会差异的显著偏离 |
| 输出 | `是` / `否`（一般为"否"，若为"是"也需在说明中提示） |
| 必输出 | ❌ 否（仅出现异常倾向时输出） |

### C-03：税法折旧 ?= 会计折旧？

```
tax_return.tax_depreciation ≈ income_statement.accounting_depreciation
```

| 项目 | 说明 |
|------|------|
| 左值 | 纳税申报表·税法折旧 |
| 右值 | 利润表附注·会计折旧 |
| 容差 | ±1.00 万元 |
| 输出 | `是` / `否` |
| 必输出 | ❌ 否 |

### C-04：税表可扣除职工薪酬 = 实际计提？

```
tax_return.deductible_employee_compensation ≈ income_statement.actual_employee_compensation
```

| 项目 | 说明 |
|------|------|
| 左值 | 纳税申报表·工资薪金税前扣除金额 |
| 右值 | 利润表附注·实际计提职工薪酬 |
| 容差 | ±1.00 万元 |
| 输出 | `是` / `否` |
| 必输出 | ❌ 否 |

## 五、跨表校验（仅异常时输出）

### X-01：增值税应税销售额 vs 营业收入（幅度校验）

```
| taxable_sales - revenue | / revenue ≤ 10%   且   taxable_sales 一般 ≥ revenue
```

| 项目 | 说明 |
|------|------|
| 规则 | 增值税应税销售额通常 ≥ 营业收入（含视同销售、价外费用），差异幅度一般在 ±10% 以内 |
| 输出 | `符合预期` / `异常` |
| 必输出 | ❌ 否（仅异常时输出） |
| 备注 | 若 `taxable_sales < revenue` 或差异幅度 > 10%，判定为异常 |

### X-02：企业所得税申报表 vs 利润表（口径一致性）

**X-02a：营业收入一致性**
```
tax_return.tax_revenue ≈ income_statement.revenue
```

**X-02b：利润总额一致性**
```
tax_return.tax_total_profit ≈ income_statement.total_profit
```

| 项目 | 说明 |
|------|------|
| 容差 | ±1.00 万元 |
| 输出 | `是` / `否` |
| 必输出 | ❌ 否 |

### X-03：应纳所得税额公式校验

```
income_tax_payable ≈ total_profit × legal_tax_rate - tax_reduction - tax_credit
```

| 项目 | 说明 |
|------|------|
| 左值 | 纳税申报表·应纳所得税额 |
| 右值 | 利润总额 × 法定税率 - 减免税额 - 抵免税额 |
| 法定税率 | 默认 25%（小微企业或高新技术企业需按政策调整） |
| 容差 | ±1.00 万元 或 ±5%（取较大值） |
| 输出 | `是` / `否` |
| 必输出 | ❌ 否 |

## 七、校验执行策略

### 输出规则汇总表

| 检查类型 | 是否必输出 | 输出触发条件 |
|---------|----------|-------------|
| 时间检查 | ✅ 是 | 始终输出 |
| 字段完整性 | ❌ 否 | 存在缺失字段时 |
| 原始指标 | ✅ 是 | 始终输出（缺失项标注异常） |
| 核心指标检查 | ❌ 否 | 结果为"否/异常"时 |
| 跨表校验 | ❌ 否 | 结果为"否/异常"时 |
| 用户自定义 | ❌ 否 | 用户提出附加问题时 |

### 缺失数据处理

1. 某科目为 `null` → 所涉校验规则标记为 ⚠️ 无法校验（不计入"通过/不通过"）
2. 某张表完全未识别到 → 跳过涉及该表的所有校验，并在"字段完整性检查"中给出整体缺失提示

### 异常说明模板（50字以内）

格式：`{简要原因} + 建议 {动作}；查验位置：{报表名·版块·页码}`

示例：
- `增值税应税销售额较营业收入偏低12%，建议核对是否存在未开票收入漏申报；查验位置：增值税申报表主表第1行、利润表第1行。`
- `应纳税所得额与利润总额完全相等，建议核对是否遗漏纳税调整项；查验位置：企业所得税年度申报表 A100000。`

### 常见不通过原因（参考）

| 原因 | 说明 |
|------|------|
| OCR识别误差 | 数字识别错误（6→8、1→7），建议人工复核原始报表 |
| 单位转换错误 | 表内单位不一致（部分科目为元，部分为万元） |
| 币种混用 | 三表币种不一致而未折算 |
| 合并报表差异 | 合并报表与母公司报表混用 |
| 报告期不一致 | 三表的报告期不同（月/季/年混淆） |
| 税会口径差异 | 税法口径与会计口径差异未做纳税调整 |
| 四舍五入累积 | 多科目舍入误差累积 |
