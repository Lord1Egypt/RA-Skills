# 语义分析引擎

> 本文件说明语义分析引擎如何工作：任务参数提取、任务类型分类、推荐方法映射。

---

## 概述

语义分析是本技能的核心智能组件。当用户输入一段描述（如"我有个软件开发项目..."），系统自动从中提取关键参数并推荐最合适的估算方法。

---

## 第一阶段：任务参数提取（5W2H）

采用5W2H维度提取方法：

| 维度 | 提取内容 | 
|------|---------|
| **What** 要估什么 | 任务/项目/活动描述 |
| **Why** 估算目的 | 投标/计划/预算/合同 |
| **Who** 谁参与 | 团队规模/角色 |
| **Where** 领域 | 软件/建筑/制造/科研/农业/其他 |
| **When** 时间框架 | 已知周期/没概念/大致范围 |
| **How** 是否有阶段 | 单阶段/多阶段/未知 |
| **How much** 数据充分度 | 有OMP/部分/无 |

### 约束强度标注

提取的参数进行三级约束标注：

| 标注 | 含义 | 示例 |
|------|------|------|
| 🔴 硬约束 | 用户明确指定的参数 | "A阶段4/6/8天" |
| 🟡 软约束 | 可合理推断的参数 | 没说依赖关系但多阶段通常FS |
| ⚪ 自由 | 完全不明确的参数 | "大概一两个月" |

---

## 第二阶段：任务类型分类

根据用户描述中的关键词匹配任务类型：

| 领域 | 特征关键词 | 
|------|-----------|
| **软件开发** | 软件/开发/编程/编码/测试/部署/迭代/版本/APP/Web/SaaS |
| **建筑工程** | 建筑/施工/土木/结构/水电/装修/地基/框架/混凝土 |
| **制造生产** | 制造/生产/流水线/装配/加工/组装/质检/批量 |
| **科研实验** | 科研/实验/研究/论文/试验/测试/分析/实验室 |
| **农业种植** | 农业/种植/养殖/播种/收割/育苗/灌溉 |
| **活动策划** | 活动/策划/会议/展览/演出/发布/推广 |
| **基础研究** | 一般性/学术/理论/方法/算法 |
| **其他/通用** | 匹配不到以上任何一种 |

### 分类逻辑

1. 扫描用户输入中的关键词，匹配以上分类
2. 如有多个分类匹配，取匹配度最高的
3. 匹配度相同 → 取更具体的分类
4. 无匹配 → 标记为"通用"，推荐保守组合

---

## 第三阶段：推荐方法组合

### 基础映射表

| 任务类型 | 推荐方法1 | 推荐方法2（可选） | 理由 |
|---------|----------|-----------------|------|
| 软件开发 | β分布（PERT） | 蒙特卡洛模拟 ⭐ | 非对称+不确定性累积 |
| 建筑工程 | β分布（PERT） | 蒙特卡洛模拟 ⭐ | PERT是建筑行业标准 |
| 制造生产 | 正态分布 | β分布 | 重复性好，对称性高 |
| 科研实验 | 蒙特卡洛模拟 ⭐ | β分布 | 高不确定性，概率分布 |
| 农业种植 | 直接估算 | β分布 | 周期固定但有自然变量 |
| 活动策划 | β分布 | 直接估算 | 经验估算为主 |
| 通用/不确定 | 蒙特卡洛模拟 ⭐ | β分布 | 保守推荐，覆盖所有可能 |

### 推荐逻辑细化

除了任务类型，还考虑以下因素来调整推荐：

1. **阶段数量**：
   - 1个阶段 → 三点估算就够，蒙特卡洛过度
   - 2-3个阶段 → β分布 + 蒙特卡洛都可以
   - 4+个阶段 → 蒙特卡洛强力推荐

2. **数据充分度**：
   - 有完整OMP → 所有方法可用
   - 只有部分 → 先搜索补充
   - 没有 → 必须先搜索

3. **不确定性**：
   - 低（OMP跨度<50%）→ 直接+β分布
   - 中（50%-100%）→ β分布+蒙特卡洛
   - 高（>100%）→ 蒙特卡洛必备

4. **已有知识**：
   - 同类任务大模型有充分数据 → 直接推荐
   - 非典型任务 → 标记需要搜索

## 输出格式

完成语义分析后输出以下信息供用户确认：

```
📌 任务分析结果

任务类型：[软件/建筑/制造/科研/农业/活动/通用]
  匹配依据：[匹配到的关键词]

参数提取：
  What: [描述]
  When: [时间框架] 🔴/🟡/⚪
  How (阶段数): [N个/未知] 🔴/🟡/⚪
  How much (数据): [有OMP/部分/无] 🔴/🟡/⚪
  依赖关系: [FS/SS/FF/SF/未指定] 🟡

推荐估算方法组合：
  🥇 首选: [方法] — [理由]
  🥈 备选: [方法] — [理由]

信息充分度评估：
  大模型自判: [✅足够/⚠️需补充/❌不足]
  建议搜索: [是/否]

❓ 是否按以上推荐执行？需要调整参数或补充信息吗？
```
