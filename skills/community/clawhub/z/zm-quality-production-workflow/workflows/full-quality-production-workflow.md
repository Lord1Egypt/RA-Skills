# 全流程｜高质量正式产出流水线

## 阶段 0：任务接入

- 识别任务类型：课程 / PPT / 漫画 / 文章 / 方案 / 代码 / 其他。
- 判断是否正式产出。
- 判断是否需要 SubAgent。
- 明确 main 不越权：正式正文、视觉、生图、代码、正式 Deck 成品必须交专业链路。

## 阶段 1：标准化输入

产物：`standard_input_card.md`

必填：

1. 任务名称；
2. 目标用户；
3. 使用场景；
4. 核心目标；
5. 输出清单；
6. 风格 / 口径；
7. 禁止项；
8. 必需素材；
9. 真实案例；
10. 数据来源；
11. 审核门槛。

## 阶段 2：依赖检查

产物：`dependency_gap_list.md`

检查：

- Logo；
- 二维码；
- 主角图 / 人物参考；
- 风格参考；
- 数据来源；
- 真实案例；
- 用户目标；
- 发布渠道；
- 输出格式；
- 可公开边界。

结论只能是：

- `READY`：可进入生产；
- `PARTIAL_READY`：可做文字 / 结构 / 样章，但不能全量；
- `BLOCKED`：关键依赖缺失，必须补素材。

## 阶段 3：结构设计

按产物类型输出：

- 课程：章节定位、学员课件结构、讲师包结构；
- PPT：内容大纲、页面合同、素材分级；
- 漫画：完整文字版、逐页脚本、分镜和角色设定；
- 文章：选题卡、大纲、素材卡；
- 方案：痛点、流程、交付边界、套餐。

## 阶段 4：样章 / 样页 / 样稿

硬规则：

- 漫画：只做 P1 样章；
- PPT：先做 1-3 页样页；
- 课程：先做一节样章或 1-2 页完整课件 + 讲师提示卡；
- 文章：先出标题、开头和结构样稿；
- 方案：先出一页方案结构或售前样页。

样章必须用于锁定：

- 风格；
- 信息密度；
- 角色 / 视觉；
- 口吻；
- 结构；
- 审核口径。

样章未确认，不得全量扩展。

## 阶段 5：全量生产

由专业 SubAgent 或专业 Skill 执行。

必须记录：

- owner；
- 输入；
- 输出路径；
- 版本号；
- 生成证据；
- 已知缺口；
- 自检结论。

## 阶段 6：执行者自检

执行者必须先自检，不得直接交付。

产物：`self_review.md`

自检至少包含：

- 是否完成全部输出；
- 是否遵守输入；
- 是否有缺失依赖；
- 是否有伪造风险；
- 是否符合样章标准；
- 是否可进入双审。

## 阶段 7：双 Agent 审核

见 `dual-agent-review-workflow.md`。

## 阶段 8：返工循环

默认最多两轮。

每轮必须写：

- 问题清单；
- 返工范围；
- 不改范围；
- owner；
- 新版本；
- 复审结果。

## 阶段 9：main 终审

产物：`final_acceptance.md`

结论：

- `PASS`：可交付；
- `CONDITIONAL_PASS`：可交付但有明确后补项；
- `NEEDS_REVISION`：必须返工；
- `BLOCKED`：缺素材 / 缺确认 / 工具不可用。

## 阶段 10：交付包归档

必须包含：

- README；
- 正式产物；
- 版本记录；
- 素材；
- 审核记录；
- 发布材料；
- meta / manifest。

## 阶段 11：发送 / 发布前核验

- 文件存在；
- 包结构正确；
- 预览可打开；
- 资源引用正确；
- 不重复发送；
- 外部发布 / 删除 / 付款先确认。
