# Unit Test Coverage Helper

## 需求

使用这个技能帮助以下用户群体：maintainers and users asking for software improvements

> feat:Add Unit Tests

如需查看来源证据、执行计划或评审标准，请阅读 `references/requirement-plan.md`。

## 工作流程

1. 重新说明用户想要的结果、限制、已有输入和成功标准。
2. 把请求转化为实用计划和可执行输出。
3. 只有当缺失信息会明显改变输出时才提问；否则先做合理假设并继续推进。
4. 产出用户需要的文档、流程、清单、分析、代码修改或决策支持。
5. 对照成功标准检查结果，并列出剩余风险或后续事项。

## 预期输出

- 针对用户当前情境的定制回答或产物。
- 当任务可复用时，提供检查清单或工作流程。
- 说明结果如何被检查的验证备注。

## 验证

- 输出直接回应发现的原始需求。
- 用户不需要阅读原始来源帖子也能采取行动。
- 假设、限制和所需输入清晰可见。
- 在有帮助时，最终回复包含简短的使用说明或下一步建议。

## 触发方式

关键词：`general-help`, `add`, `unit`, `tests`, `labels`, `gssoc`, `level`, `intermediate`, `type`

示例触发句：

- `Help me Add Unit Tests.`
- `I need a practical workflow for Add Unit Tests.`
- `Use $unit-test-coverage-helper to handle Add Unit Tests.`
