# 外部知识搜索集成

> 本文件说明外部知识搜索的两个阶段流程。

---

## 第一阶段：大模型自判

当语义分析完成后，系统判断大模型自身的知识储备是否足够：

### 判断矩阵

| 条件 | 结论 |
|------|------|
| 常见任务类型（软件/建筑/制造/农业/活动）且有明确OMP | ✅ **足够** — 直接执行估算 |
| 常见任务类型但缺少分布参数建议 | ⚠️ **需补充** — 大模型知识+搜索确认分布标准 |
| 非典型任务（深海/太空/特殊行业）或无OMP | ❌ **不足** — 必须搜索 |
| 任务类型和参数都不确定 | ❌ **不足** — 必须先搜索做规划 |

### 审查范围

大模型自判内容：
1. 该领域的典型工期范围
2. 标准的OMP比例（如软件通常在0.5x-2x之间）
3. 行业普遍接受的分布类型
4. 常用的估算粒度标准

---

## 第二阶段：搜索执行流程

### 第1步：询问用户

当自主判断结论为"⚠️需补充"或"❌不足"时，向用户发起询问。

**询问格式：**

```
⚠️ 当前信息不足以支撑完整估算，原因：[具体原因]

请选择：
[A] 明确指定信源 — 提供URL/文档/资料名称，我将直接获取
[B] 自觉搜索 — 我自行搜索公开资料补充
[C] 不搜索 — 按现有知识估算，结果可能偏差较大

🔔 提示：选择A或B后，执行搜索时应调用搜索相关的技能或MCP工具（如WebSearch/WebFetch等），
确保搜索质量和效率。
```

### 第2步：执行搜索

用户确认后的搜索执行原则：

| 场景 | 搜索策略 | 搜索关键词示例 |
|------|---------|---------------|
| 常见任务补充分布建议 | "典型 [任务类型] 项目工期分布" | "软件开发项目 PERT估算 标准工期分布" |
| 非典型任务规划 | "[任务领域] 项目阶段划分 WBS" | "深海探测项目 阶段划分 工期估算" |
| 参数范围确认 | "[任务类型] 典型工期 OMP" | "装配式建筑 标准层 施工天数" |
| 行业标准参考 | "[行业] 估算标准 规范" | "建筑工程施工工期定额标准" |

**执行工具建议**（由执行AI根据实际可用工具决定）：
- 通用web搜索
- 专业文档抓取
- 知识库查询

### 第3步：汇总并确认

搜索完成后汇总信息：

```
📋 搜索汇总

数据来源：[搜索到的参考来源列表]

基于搜索获得的行业参考：
- 典型工期范围：[结果]
- 推荐分布类型：[结果和理由]
- 建议阶段划分：[结果]

推荐方案：
分布类型：[根据搜索结果推荐的分布]
理由：[具体引用搜索结果参数]

估算计划：
[如需要则输出完整的阶段划分和参数建议]

❓ 是否按照此方案和参数进行估算？
```

### 搜索不成功的处理

| 情况 | 处理方式 |
|------|---------|
| 搜索无相关结果 | 告知用户未找到相关数据，建议手动补充参数 |
| 数据矛盾 | 列出不同来源的差异，由用户选择依据 |
| 搜索工具不可用 | 告知用户搜索工具当前不可用，建议手动提供信息 |

---

## 执行AI的职责

当使用此技能时，执行AI（大模型/智能体）应：

1. **开启搜索前先自判** — 非每次都必须搜索
2. **告知使用什么工具搜索** — 明确提及使用的搜索能力
3. **搜索后汇总不照搬** — 分析搜索结果后给出明确的推荐
4. **由用户确认后再执行** — 不跳过用户确认环节
