# Android SDK 接入流程

## 1. 前置条件

- 已在 DataNexus 平台注册应用并获取 `userActionSetId` 和 `appSecretKey`
- Android Studio 开发环境已就绪，最低支持 **Android API 14（Android 4.0）**

## 2. 引入 SDK

### 方式：手动集成

1. 下载 SDK 的 `.aar` 文件（例如 `GDTActionSDK.min.1.9.2.aar`）
2. 将 `.aar` 文件放入项目的 `app/libs/` 目录
3. 在项目的 `build.gradle` 中配置 flatDir 仓库，并在 app 模块的 `build.gradle` 中添加依赖：

```groovy
// 项目根目录 build.gradle 或 app/build.gradle 的 repositories 中
repositories {
    flatDir {
        dirs 'libs'
    }
}

// app/build.gradle 的 dependencies 中
dependencies {
    implementation(name: 'GDTActionSDK.min.1.9.2', ext: 'aar')
}
```

## 3. 添加 SDK 权限

在 `AndroidManifest.xml` 中添加以下权限声明：

```xml
<!-- 必要权限 -->
<uses-permission android:name="android.permission.INTERNET" />

<!-- 可选权限（提升归因效果） -->
<uses-permission android:name="android.permission.ACCESS_NETWORK_STATE" />
<uses-permission android:name="android.permission.READ_PHONE_STATE" />
<uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE" />
```

| 权限 | 是否必须 | 说明 |
|------|----------|------|
| `INTERNET` | **必须** | 网络通信，SDK 基本功能依赖此权限 |
| `ACCESS_NETWORK_STATE` | 可选 | 获取网络状态，提升归因匹配效果 |
| `READ_PHONE_STATE` | 可选 | 获取设备标识（IMEI 等），Android 6.0+ 需动态申请运行时权限 |
| `WRITE_EXTERNAL_STORAGE` | 可选 | 归因优化 |

> **注意：** `READ_PHONE_STATE` 在 Android 6.0+ 属于危险权限，需在运行时动态申请。

## 4. 初始化 SDK

在 `Application` 的 `onCreate` 中调用初始化方法。SDK 提供两种初始化方式：

### 方式一：基础初始化（3 参数）

```java
import com.qq.gdt.action.GDTAction;

public class MyApplication extends Application {
    @Override
    public void onCreate() {
        super.onCreate();
        // 初始化
        GDTAction.init(
            this,                          // Context
            "YOUR_USER_ACTION_SET_ID",     // 数据源 ID
            "YOUR_APP_SECRET_KEY"          // 数据源密钥
        );
        // 必须调用 start()，启动个人信息采集
        GDTAction.start();
    }
}
```

### 方式二：带渠道信息初始化（5 参数）

```java
import com.qq.gdt.action.GDTAction;
import com.qq.gdt.action.ChannelType;

public class MyApplication extends Application {
    @Override
    public void onCreate() {
        super.onCreate();
        // 初始化（带渠道信息）
        GDTAction.init(
            this,                          // Context
            "YOUR_USER_ACTION_SET_ID",     // 数据源 ID
            "YOUR_APP_SECRET_KEY",         // 数据源密钥
            ChannelType.TENCENT,           // 渠道类型
            "YOUR_CHANNEL_ID"              // 渠道 ID
        );
        // 必须调用 start()，启动个人信息采集
        GDTAction.start();
    }
}
```

### 参数说明

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `context` | `Context` | 是 | 传入 `Application` 或 `Activity` 实例 |
| `userActionSetId` | `String` | 是 | DataNexus 平台获取的数据源 ID |
| `appSecretKey` | `String` | 是 | DataNexus 平台获取的数据源密钥 |
| `channel` | `ChannelType` | 否 | 渠道类型枚举（仅 5 参数版本） |
| `channelId` | `String` | 否 | 渠道 ID（仅 5 参数版本） |

> **重要：** 初始化后必须调用 `GDTAction.start()`。`start()` 方法用于启动个人信息采集（设备信息等），不调用将导致 SDK 无法正常工作。

## 5. 按行为清单埋点上报数据

### 5.1 预定义方法（ActionUtils 类）

SDK 在 `ActionUtils` 类中提供了一系列预定义行为上报方法：

| 方法 | 说明 |
|------|------|
| `ActionUtils.onRegister(String method, boolean success)` | 注册 |
| `ActionUtils.onLogin(String method, boolean success)` | 登录 |
| `ActionUtils.onBindAccount(String type, boolean success)` | 绑定账号 |
| `ActionUtils.onQuestFinish(String questID, String questType, String questName, int questNumber, String desc, boolean success)` | 完成任务 |
| `ActionUtils.onCreateRole(String roleName)` | 创建角色 |
| `ActionUtils.onUpdateLevel(int level)` | 升级 |
| `ActionUtils.onShare(String channel, boolean success)` | 分享 |
| `ActionUtils.onRateApp(int rate)` | 应用评分 |
| `ActionUtils.onViewContent(String contentType, String contentName, String contentID)` | 浏览内容 |
| `ActionUtils.onAddToCart(String contentType, String contentName, String contentID, int contentNumber, boolean success)` | 加入购物车 |
| `ActionUtils.onCheckout(String contentType, String contentName, String contentID, int contentNumber, boolean isVirtual, String virtualType, String realType, boolean success)` | 发起结算 |
| `ActionUtils.onPurchase(String contentType, String contentName, String contentID, int contentNumber, String channel, String currency, int amount, boolean success)` | 付费 |
| `ActionUtils.onAddPaymentChannel(String channel, boolean success)` | 添加支付渠道 |

**示例 — 注册上报：**

```java
ActionUtils.onRegister("wechat", true);
```

**示例 — 付费上报：**

```java
ActionUtils.onPurchase("vip", "月卡", "item_001", 1, "wechat", "CNY", 3000, true);
```

### 5.2 通用方法

对于预定义方法未覆盖的自定义行为，使用通用上报：

```java
// 无参数
GDTAction.logAction("CUSTOM_ACTION");

// 带参数
JSONObject actionParam = new JSONObject();
actionParam.put("key1", "value1");
actionParam.put("key2", 123);
GDTAction.logAction("CUSTOM_ACTION", actionParam);
```

### actionParam 参数约束

- **Key：** 字母、数字和下划线组成，必须以字母开头，最大长度 255 字符，不能以 `ams_reserved_` 为前缀
- **Value：** 支持 String、Number、Boolean、JSONArray 类型（JSONArray 中元素类型需统一）

## 6. 自动采集事件

SDK 会自动采集以下事件，无需手动上报：

| 事件 | 说明 |
|------|------|
| `TICKET` | 票据事件 |
| `ENTER_FOREGROUND` | 应用进入前台 |
| `ENTER_BACKGROUND` | 应用进入后台 |

> **注意：** `START_APP` 在 Android 端默认由 SDK v1.9.4+ 自动采集，如已关闭自动采集（`GDTAction.setAutoStartEnable(false)`）则需手动调用 `GDTAction.logAction("START_APP")`。

## 7. 验证接入

使用 Logcat 过滤 TAG `gdt_action`（info 级别）查看日志：

- **初始化成功：** 日志输出 `GDTAction初始化成功（sdkv: 1.x.x, sdkvc: xx）`
- **上报成功：** 日志输出 `LogAction success xxxx`
- **上报失败：** 日志输出 `LogAction failed xxxx`

## 8. 混淆配置

如果项目开启了 ProGuard 混淆，需在混淆规则文件中添加：

```proguard
-keep class com.qq.gdt.action.** { *; }
```
