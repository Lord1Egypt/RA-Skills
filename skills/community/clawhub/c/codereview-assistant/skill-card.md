## Description: <br>
Code review best practices and workflow skill for reviewing pull requests, performing peer review, setting review standards, and improving code quality processes. <br>

This skill is ready for commercial/non-commercial use. <br>

## Publisher: <br>
[sydpz](https://clawhub.ai/user/sydpz) <br>

### License/Terms of Use: <br>
MIT-0 <br>


## Use Case: <br>
Developers and engineering teams use this skill to conduct structured pull request reviews, classify findings by severity, write actionable comments, and decide whether to approve or request changes. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Generated review comments may be incorrect, incomplete, or misleading if posted without human review. <br>
Mitigation: Review generated comments before posting them and verify findings against the actual code, tests, and project context. <br>
Risk: Private source code may be exposed if users share proprietary code with an assistant outside approved organizational controls. <br>
Mitigation: Use the skill only in environments permitted by the organization and avoid sharing private source code unless policy allows it. <br>
