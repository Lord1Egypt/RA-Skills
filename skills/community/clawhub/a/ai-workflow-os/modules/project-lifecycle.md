# Project Lifecycle / 项目生命周期规划

## Role / 角色

This module turns unclear or early-stage ideas into actionable project plans.

本模块用于把模糊想法或早期项目转化为可执行项目计划。

## Trigger / 触发场景

Use this module when the user is starting, restructuring, narrowing, reviewing, or upgrading a project.

当用户启动、重构、缩小范围、审查或升级项目时使用。

## Required Outputs / 必须输出

- Project goal / 项目目标
- User or business scenario / 用户或业务场景
- MVP scope / MVP 范围
- Out-of-scope items / 暂不做事项
- Constraints / 约束
- Risks / 风险
- Milestones / 里程碑
- Next actions / 下一步

## Operating Rules / 执行规则

1. Clarify the goal before expanding the solution.
2. Control scope before planning implementation.
3. Separate required, optional, and future features.
4. When the project is unclear, create a draft target and mark it as unconfirmed instead of stopping.
5. If the user asks for code review or upgrade planning, identify current state, target state, risk, and migration path.

1. 先明确目标，再扩展方案。
2. 先控制范围，再规划实现。
3. 区分必须功能、可选功能和未来功能。
4. 项目不清楚时，先创建未确认目标草案，不要直接停止。
5. 如果用户要求代码审查或升级规划，识别当前状态、目标状态、风险和迁移路径。
