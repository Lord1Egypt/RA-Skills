# 会话工作区

此目录是 `planning-with-files` 的唯一任务执行工作区，用于承接跨多轮会话的临时规划与执行记录。

## 目录职责

`docs/sessions/` 仅承接以下内容：

1. 正在进行中的任务拆解、调研与执行记录。
2. 需要跨多次会话持续推进的任务级工作文件。
3. 每个任务在执行期的阶段状态与验证笔记。

通用原则：

1. 会话目录只存放执行期材料，不作为长期知识的最终归宿。
2. 稳定结论应同步到项目的正式文档体系。

## planning-with-files 约定

每个任务创建独立子目录，例如：`docs/sessions/ai-chat-refactor/`。

任务目录固定包含以下 3 个文件：

1. `task_plan.md`：阶段计划、目标、关键决策。
2. `findings.md`：调研结论、证据链接、方案对比。
3. `progress.md`：执行日志、验证记录、问题与处理结果。

## 标准工作流

1. 初始化会话目录：创建 `docs/sessions/<task-slug>/` 与 3 个标准文件。
2. 阶段推进：每完成一个阶段更新 `task_plan.md` 状态。
3. 调研记录：发现重要信息时写入 `findings.md`。
4. 进度追踪：测试结果、错误处理写入 `progress.md`。
5. 任务完成：确认所有阶段标记为 complete 后，归档或清理。
