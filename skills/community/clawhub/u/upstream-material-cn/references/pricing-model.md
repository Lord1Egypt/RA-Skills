# 量价测算模板

## 需求侧测算

### Step A：找终端产品出货量

来源优先级：
1. 行业机构预测（LightCounting/TrendForce/Yole）
2. 券商研报共识
3. 公司指引

示例：2026年1.6T光模块预计出货2500万个

### Step B：计算单位材料用量

需要的参数：
- 单个终端产品的材料面积/重量/颗数
- 层数（如PCB 14层每层都要铜箔）
- 良率修正（如板材利用率70%→实际用量÷0.7）

示例：每个1.6T光模块PCB面积0.0023平米 × 14层 = 0.0322平米/个

### Step C：加总各应用场景

| 应用场景 | 终端出货量 | 单位用量 | 良率修正 | 材料需求 |
|---------|-----------|---------|---------|---------|
| 场景A | X万个 | Y平米/个 | ÷0.7 | Z万平米 |
| 场景B | ... | ... | ... | ... |
| **合计** | | | | **总需求** |

## 标的利润弹性情景

### 输入参数
- 材料售价：___ 元/单位
- 材料成本：___ 元/单位
- 毛利率：___%
- 净利率：___%
- 标的当前总归母净利润：___ 亿

### 四情景测算

| 情景 | 标的市占份额 | 年出货量 | 材料营收(亿) | 材料净利(亿) | 占当前利润比 |
|------|------------|---------|------------|------------|-----------|
| 保守 | 1-2% | | | | +?% |
| 中性 | 5% | | | | +?% |
| 乐观 | 10% | | | | +?% |
| 激进 | 20%+ | | | | +?% |

### 判断框架

- 如果乐观情景利润增量<10%→"锦上添花"，不改变投资逻辑
- 如果乐观情景利润增量10-30%→"有弹性"，可作为加分项
- 如果乐观情景利润增量>30%→"质变"，可能重构估值
