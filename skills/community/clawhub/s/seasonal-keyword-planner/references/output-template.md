# Output Template

Deliver the plan in three parts: a keyword classification table, the month-by-month action plan, and an early-mover summary.

## Part 1 — Keyword classification

| Keyword | Type (evergreen/seasonal/event) | Peak month(s) | Rising shoulder | Trough | Early-mover window | Evidence vs assumption |
|---|---|---|---|---|---|---|
| … | … | … | … | … | … | … |

## Part 2 — Month-by-month action plan

For each month, list actions keyed to specific keywords across listings, ads, and content.

```
### <Month>
- Listings: <which keyword listings to optimize/refresh and why>
- Ads: <budget shifts — which keywords up/down, target months>
- Content: <what to publish now to rank for an upcoming peak>
- Inventory note: <if relevant>
```

Repeat for all 12 months. Months in a trough should say so and redirect effort to off-season terms.

## Part 3 — Early-mover summary

A short list a manager can act on immediately:

```
- <Keyword>: act by <date/window> — <one-line action>
- <Keyword>: act by <date/window> — <one-line action>
```

## Part 4 — Measurement

State what to track to confirm pre-positioning worked:
- Organic rank by keyword, month over month.
- CPC and impression share by keyword during ramp vs peak.
- Conversion rate and revenue per keyword cluster.

## Worked snippet

```
### September
- Listings: refresh "packable down jacket" + "hiking base layer" copy/images (re-index before Oct peak).
- Ads: begin shifting budget to "down jacket"; capture early-mover window before competitors raise bids.
- Content: publish "best packable down jackets 2025" guide now so it can rank by late Oct.
- Inventory: confirm down-jacket restock landed.

Early-mover: "packable down jacket" — act by Sep 5 — optimize listing + start low-CPC ads.
```
