# ADR Template

Use this template when documenting architecture decisions during a grill-with-docs session.

```markdown
# ADR-NNNN: Title

**Status:** [proposed | accepted | deprecated | superseded]
**Date:** YYYY-MM-DD
**Deciders:** [who was involved]

## Context

[What's the situation? What problem are we solving?]

## Decision

[What did we decide? One clear paragraph.]

## Rationale

[Why this over the alternatives? List 2-3 options considered.]

## Consequences

[What does this mean for the codebase? Trade-offs, migration needs, etc.]
```
