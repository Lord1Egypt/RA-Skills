# phase-one-reserve.md — 阶段一：排期预占

> 数字员工接住 PM 预占请求 → 识别配置方式 → **按 `config-matrix.md` 收齐创建必填字段 + 4 前缀（死值或问 PM）+ 拼好推广名** → **交接① 委托技术那套 `promotion-add`+预占** → 写追踪表。
> ⚠️ **v3：推广名拼接、5 字段判定（项目类型+4 前缀）由数字员工负责整包传给技术那套。资源/投放量/人群等初始化字段可以阶段一有，也可以阶段二物料提交前补齐。**

## 总览

```
PM 说预占(DM/群) → 搜表防重复 → 识别配置方式(H列)
    → 按 config-matrix §2 收齐"阶段一必收"字段（含艺人ID；资源ID统一阶段二）
    → 按 config-matrix §1.2 定 4 前缀+项目类型（死值查表 / 可选值问 PM）
    → 按 config-matrix §1.1 算好已知初始化字段规范值（未知的阶段二物料提交前补齐）
    → 数字员工拼好完整推广名 name
    → 交接① 调技术那套 promotion-add+预占（整包传 §1.4 全部入参；技术那套只拆天/组人群JSON/写后台）
    → 拿回 promotionID → 写追踪表 → N=已预占 → 标准化完成话术通知 PM
```

**关键决策：**
- §1 群路由 + 搜表防重复
- §2 识别配置方式（写 H 列）+ 按 config-matrix 收齐阶段一必填字段 + 4 前缀(死值/问 PM)
- §3 交接① 委托技术那套预占（整包传齐 §1.4 入参）
- §4 写表 + 标准化完成话术

**红线：**
- 群聊发起 → 群内回「私聊对接」→ 主动 DM
- **不自己拼 CLI / 不自己组人群 JSON** — 交技术那套
- **自己拼推广名 + 5 字段（v3）**：照 config-matrix §1.2 死值查表 / 可选值让 PM 选，绝不脑补
- **初始化字段的业务正确值必须自己照 config-matrix §1.1 算好喂技术那套**；阶段一有就传，阶段一没有不阻塞预占，阶段二物料提交前必须补齐
- **阶段一收料集严格按 config-matrix §2**：所有资源ID/链接统一归阶段二，阶段一**只收资源子类型**（定 resourceType），空降/数专/标准均不收歌曲ID
- **禁猜必问**：config-matrix 没写的字段一律问 PM，不脑补
- 「专辑」「演出」有歧义 → 追问

## 1. 触发条件 + 搜表防重复

**谁说**：PM（项目负责人）
**在哪**：DM 优先。PM 在群里发起 → 群内回「收到，我们私聊对接」→ 立刻主动 DM 该 PM。
**说什么**：预占 / 占排期 / 建推广

**先搜追踪表 A 列防重复**：用项目核心名搜（去掉 `【…】` 前缀、`–【内容运营部】` 后缀、内嵌 `_MMDD`），匹配所有相关行。已存在则提示 PM 已预占过。

**DM 话术（第一轮，先拿够识别配置方式）：**
```
📋 我查了一下，你提到的项目：

{如有匹配：共 N 条记录}

需要预占的话，先告诉我：
• 项目名称
• 项目等级（S+/S/A/B）
• 时间段（起止）
• 投放类型（歌曲/歌单/专辑/MV/数专/H5/空降/云村有票）
```

> 识别出配置方式后，按 §2.2 **照 config-matrix §2 把"阶段一必收"字段一次性补齐**，不要分多轮挤牙膏。

## 2. 识别配置方式 + 收齐阶段一字段 ⭐数字员工核心动作

### 2.1 识别配置方式（写 H 列）

按 PM 描述映射（完整映射见 `config-matrix.md §6`）：

| PM 说的投放类型 | H 列配置方式 |
|----------------|-------------|
| 歌曲 / 单曲 / 普通专辑 / 歌单 / MV | **标准**（再细分歌曲/歌单/专辑/MV） |
| 数字专辑 / 数专 / 数字单曲 | **数专** |
| H5 / 策划 / 活动 / 实体专辑 | **H5** |
| 空降 | **空降** |
| 云村有票 / 演出有票 | **有票** |

**歧义保护：**
- 「专辑」未说清 → 追问：普通专辑(标准-专辑) / 数字专辑(数专) / 实体专辑(H5)
- 「演出」未说清 → 追问：云村有票(有票) / H5演出页(H5)

### 2.2 按 config-matrix §2 收齐"阶段一必收"字段 ⭐⭐

识别出配置方式后，**立刻对照 `config-matrix.md §2` 看这种方式阶段一要收什么**，缺的一次性问 PM：

| 配置方式 | 阶段一必收（除等级/时段外） | 还要问 PM 的 3 前缀 |
|---|---|---|
| 标准 | **艺人ID** + 资源子类型(歌曲/歌单/专辑/MV) | 内容大类 / 版权类型 / 战略类型 |
| 数专 | **艺人ID** + 是数字专辑还是数字单曲 | 内容大类 / 版权类型 / 战略类型 |
| H5 | （艺人不需要，等级/时段即可） | 内容大类 / 版权类型 / 战略类型 |
| 空降 | **艺人ID**（人群 66594/不包含 固定，不用问 PM） | **不用问**（4 前缀+项目类型全死值） |
| 有票 | **艺人ID** + 人群包ID + 包含 | **不用问**（4 前缀+项目类型全死值） |

> 💡 **所有资源ID/链接统一归阶段二**：标准方式歌曲/歌单/专辑/MV ID 在阶段二作为 `resourceInfo` 传 update-creative；空降/数专 ID 在阶段二拼跳转链接。阶段一只需知道资源子类型（定 resourceType），不要任何 ID。
>
> **禁猜必问**：config-matrix §2 标了要收的就问 PM；没标的（如空降的人群）按 config-matrix 固定值，不问也不猜。

**艺人ID 必填规则**：项目类型为歌曲版权宣发 / 艺人合作宣发时必填艺人ID（标准/数专/空降/有票都必填，仅 H5 不需要）。详见 config-matrix §1。

**v3 推广名前缀+项目类型铁律**（治"艺粉"错根）：
- **死值类（空降/有票全 5 字段、其他配置方式的项目类型+投放资源）**：数字员工查 `config-matrix §1.2` 直接填，不问 PM、不脑补
- **可选类（标准/数专/H5 的内容大类/版权类型/战略类型）**：数字员工**展示 §1.2 可选值让 PM 选**，**绝不替 PM 选**——错"艺粉"就是脑补可选值的产物
- 数字员工拼好完整推广名 `【内容大类/版权类型/战略类型/投放资源】项目核心名 –【内容运营部】` 整包传技术那套，技术那套不再补问

补字段话术（按配置方式动态生成）：

```
收到，这是「{配置方式}」类型，再补几个信息我就能预占：
{按 config-matrix §2 列出该方式阶段一缺的资源/艺人字段}

{若是 标准/数专/H5，追加：}
另外推广名前缀需要你选 3 项（按 config-matrix §1.2 列可选值）：
① 内容大类：头部 / 战略 / 潜力 / 品牌（选一个）
   提示：品牌=品牌合作；头部=A级且区域>A；战略=重点垂类非头部；潜力=非A非重点垂类
② 版权类型：自制 / 独家 / 首发 / 全网 / 后发（选一个）
③ 战略类型：艺粉 / Kpop / 嘻哈 / 欧西 / 日音/二次元 / 其他（选一个）
```

> 💡 空降/有票阶段一**不问 PM 任何前缀字段**——4 前缀+项目类型全是死值，数字员工查表填。这是 v3 治错"艺粉"的根本：没有"判断空间"就不会判错。

## 3. 交接① 委托技术那套预占

收齐字段后，把 PM 提供信息 + **数字员工照 config-matrix 算好的全部入参** 整包交技术那套执行 `promotion-add`+预占：

**交接给技术那套的整包入参（照 config-matrix §1.4）：**
- 项目核心名、项目等级、时间段、PM 邮箱（项目负责人）
- **环境默认线上**：用户明确预发布才用 `pre`；明确线上或未说明环境都用 `online`
- 位置：默认发现页 banner（`PAGE_DISCOVERY_BANNER`）
- **推广名 `name` 完整字符串**：数字员工拼好 =【内容大类/版权类型/战略类型/投放资源】项目核心名 –【内容运营部】
- **`projectType` 项目类型**（中文 label）：照 config-matrix §1.2
  - 标准/数专/有票 → 歌曲版权宣发
  - H5 → 活动推广
  - 空降 → 艺人合作宣发
- **4 前缀字段**（中文 label，照 config-matrix §1.2）：
  - 空降/有票 死值【潜力/全网/其他/其他】
  - 标准/数专/H5：投放资源死值（单曲/歌单/专辑/MV/数字专辑/H5），内容大类/版权类型/战略类型 PM 选
- **初始化技术字段**（照 config-matrix §1.1；阶段一有就传，阶段二物料提交前必须补齐）：
  - `fullType`（full / nofull）
  - `resourceType`（song/playlist/album/mv/activity）
  - `resourceTypes`（二维数组）
  - `resourceInfo`（标准方式=资源ID；非标=空串；阶段二 update-creative 前必须有）
  - 人群业务值（非全量：人群包ID + 包含/不包含；阶段二 update-creative 前必须有）
  - 艺人ID（项目类型需要时）
- **投放类型 PM 原话词**（如"空降""数专""有票"）：兜底信息，便于技术那套日志诊断

**技术那套负责（v3 退化为纯写后台）：**
- `plan-check` 校验人群包 → 组 `strategyInfo`/`crowdList`（包含=contain/include，不包含=notcontain/exclude）
- 跨天拆分（先与 PM 确认拆/不拆，最多 7 天）
- `promotion-add` 带 `autoPreoccupy` 创建即预占
- `admin-add` 补管理员权限（PM + 真实 @ 协作者）

> 🚫 **v3 红线：技术那套不再问 PM 任何字段。** 数字员工必须把 §1.4 清单全部传齐。若发现技术那套触发了"轻量补问"，说明数字员工漏传了，立即停手回到 §2.2 补收。
>
> 🚫 **数字员工绝不脑补 4 前缀的可选值。** 错"艺粉"就是脑补——空降已死值"其他"，标准/数专/H5 的战略类型 6 选 1 必须 PM 选。

**数字员工拿回：**
- promotionID（写 M 列）
- 后台生成的推广名（写 A 列；应与传入的 `name` 一致）
- 最终时段/端口（写 E/F/G 列）
- 跨天拆分结果（每条独立写一行）

**交接失败**：重试 2 次（5s/10s）→ 仍失败按 `hard-rules.md` 原理四通知王可 + 写 O 列。

## 4. 预占成功后

### 4.1 写追踪表

**追踪表：** docId `f45584742d5047b8a46f578702043d65`，Sheet「配置追踪」（A-O，15 列）

使用 `popo-doc` 技能通过 `fabric_call` 操作表格（读 `doc_execute_table` workbook.getFullData；写 sheet.setCell / batchSetCells）。

**阶段一写入列：**

| 列 | 字段 | 操作 |
|----|------|------|
| A | 项目名称 | 写后台生成的推广名全称；跨天拆分时核心名含 `_MMDD` |
| B | 项目负责人 | 写 PM **完整邮箱**（私聊取 sender_id 邮箱；阶段三 `--at @人` 依赖此值，不能只写昵称） |
| C | 群ID | 群里发起转私聊 → 写最初群消息 chat_id；纯私聊 → **留空不阻塞**（阶段三用帧位排期群刷新） |
| D | 更新时间 | 当前时间 |
| E | 开始时间 | 写入 |
| F | 结束时间 | 写入 |
| G | 端口/客户端 | 写入（技术那套返回的投放端） |
| H | 配置方式 | 写 §2 识别结果（标准/数专/H5/空降/有票） |
| I | 资源ID | **阶段一留空**（所有配置方式资源ID统一阶段二写入）|
| L | 人群包ID | 非全量写（空降=66594/不包含；有票=PM值/包含） |
| M | promotionID | 写技术那套返回的推广 ID |
| N | 状态 | →「已预占」（状态机见 SKILL.md §3.1） |
| O | 备注 | 交接失败时写日志（项目名/步骤/失败命令/错误码/错误信息/参数/时间） |

> J/K（跳转链接/角标）由阶段二配物料时写入；跨天拆分时每条推广独立写一行，A/E/F 各自不同，其余一致。

### 4.2 标准化完成话术 ⭐（解决错6）

预占成功后，**必须列出两段**让 PM 提前看见：(a) 阶段二还要交的素材；(b) 系统会自动填的默认值。按 config-matrix 动态生成：

```
✅ 已预占：{推广名称}
时段 {起} - {止}，配置方式：{配置方式}

📥 接下来请发我（阶段二素材）：
{按 config-matrix §2「阶段二必收」列出，如：大卡图1080×607、歌曲/专辑链接}

⚙️ 这些我会按规则自动填，你不用管：
{按 config-matrix 列默认值，如：角标=活动、人群=66594不包含、跳转链接自动拼 orpheus}
```

空降示例：
```
✅ 已预占：{推广名称}
时段 {起} - {止}，配置方式：空降

📥 接下来请发我：
• 大卡图（1080×607）
• 空降的歌曲/专辑链接或ID（我来拼跳转链接）

⚙️ 这些我自动填，你不用管：
• 角标：活动
• 人群：66594（不包含）
• 跳转链接：按你给的ID自动拼成 orpheus://…
```

跨天拆分时合并通知各条推广。

## 5. 边界情况

| 情况 | 处理 |
|------|------|
| 「专辑」有歧义 | 追问普通/数字/实体专辑（定配置方式） |
| 「演出」有歧义 | 追问云村有票还是 H5 |
| PM 在群里发起 | 群内回「私聊对接」→ 立刻主动 DM |
| config-matrix 没写的字段缺失 | 问 PM，**禁止脑补** |
| 标准方式 PM 没给资源ID | **正常，阶段二收** — 阶段一不需要资源ID，预占先进行 |
| 标准/空降/数专 PM 提前给了歌曲ID | 收下记到 I 列，注明「阶段二用」，**绝不填进 promotion-add 创建参数** |
| 项目已预占过（搜表命中） | 提示 PM，不重复预占 |
