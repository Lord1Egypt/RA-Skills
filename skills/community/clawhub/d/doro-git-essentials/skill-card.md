## Known Risks and Mitigations: <br>
Risk: Commands such as git reset --hard, git clean -fdx, branch or tag deletion, and force pushes can delete work or rewrite shared history. <br>
Mitigation: Review git status and git diff before running destructive commands, create a backup or safety branch when needed, and coordinate with collaborators before rewriting shared history. <br>
Risk: The skill is a broad Git command reference, so commands may be inappropriate for a repository's current state or team workflow. <br>
Mitigation: Treat commands as examples to adapt, verify the repository state first, and prefer reversible operations when uncertain. <br>
