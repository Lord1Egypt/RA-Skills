# Review Program Audit Checklist

Before redesigning, capture today's reality. Skip nothing.

## Volume and rate
- [ ] Reviews per SKU last 30 / 60 / 90 / 365 days
- [ ] Orders per SKU same windows
- [ ] Review rate per SKU (reviews / delivered orders)
- [ ] Review rate per channel (Amazon, Shopify, eBay, etc.)
- [ ] Star average per SKU
- [ ] Star average trend over last 6 months

## Source attribution
- [ ] What % of reviews come from organic (no prompt) vs solicited (prompted)
- [ ] If solicited, which channel solicited (platform-native, email, SMS, insert)
- [ ] Conversion rate from each ask channel
- [ ] Cost per review captured

## Timing
- [ ] Median days from purchase → review
- [ ] Median days from delivery → review
- [ ] Distribution of review timing (histogram)
- [ ] First-use window assumption per SKU

## Messaging
- [ ] Current templates inventoried
- [ ] Any rating-direction language present?
- [ ] Any incentive language present?
- [ ] Subject lines, open rates, click rates
- [ ] Reply rate (negative feedback intercept)

## Compliance
- [ ] Platform request windows respected (Amazon 5-30 days post-delivery, no second ask)
- [ ] No gating (no path that filters unhappy buyers away from reviews)
- [ ] No incentives tied to leaving a review
- [ ] Insert language reviewed against marketplace policies

## Negative feedback
- [ ] Existing support intercept rate
- [ ] Time-to-response on 1-3 star reviews
- [ ] Public response rate
- [ ] Conversion rate from intercept → resolution → review update

## Segmentation
- [ ] Are first-time vs repeat buyers split?
- [ ] AOV bands defined?
- [ ] SKU complexity tier defined?
- [ ] New-launch program separate from steady-state?

## Tooling
- [ ] Review aggregation tool in use (Yotpo, Stamped, Okendo, etc.)
- [ ] Email/SMS platform in use
- [ ] CRM tags for reviewer status
- [ ] Reporting cadence and owner

Run through this end-to-end before designing the new program. The audit findings drive the design — skip them and you're guessing.
