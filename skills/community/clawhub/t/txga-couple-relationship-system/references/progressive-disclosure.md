## 设计原则

### 核心原则

| 原则 | 说明 |
|-----|-----|
| 需求驱动 | 根据用户实际需求展示内容，而非一次性呈现全部 |
| 最小阻力 | 从用户最需要的内容开始，降低认知负担 |
| 渐进深入 | 先给框架，再给细节 |
| 自适应 | 根据用户反馈自动调整内容深度 |

### 避免问题

- ❌ 一次性呈现大量内容 → 用户不知所措
- ❌ 过深过早 → 用户感觉"被教育"
- ❌ 过浅过慢 → 用户感觉"没帮助"
- ❌ 缺乏层次 → 用户找不到重点

## 用户分层模型

### 4个用户层级

| 层级 | 名称 | 特征 | 内容深度 | 交付形式 |
|-----|-----|-----|---------|---------|
| **L1** | 探索者 | 刚遇到问题，寻求初步理解 | 框架+方向 | 诊断+简单答案 |
| **L2** | 理解者 | 想深入理解问题 | 框架+工具+部分细节 | 诊断+原因分析+初步方案 |
| **L3** | 行动者 | 知道问题，寻求解决方案 | 完整框架+详细工具+案例 | 完整方案+话术+行动清单 |
| **L4** | 进化者 | 已能处理问题，追求深化 | 深度原理+心法+高阶修炼 | 深度洞察+修炼路径 |

### 层级特征详细描述

**L1 探索者**：
- 刚发现关系有问题
- 不确定问题是什么
- 需要初步的方向和确认
- 不需要太多细节

**L2 理解者**：
- 对问题有初步判断
- 想深入了解原因
- 愿意花时间学习
- 需要系统性的理解

**L3 行动者**：
- 清楚知道问题是什么
- 需要具体的解决方案
- 准备开始行动
- 需要可执行的工具和话术

**L4 进化者**：
- 已经能处理基本问题
- 想要更深的洞察
- 对原理和心法感兴趣
- 想要持续进化

## 内容层级结构

### 层级1：即时诊断（所有层级可见）

**目标**：快速定位问题类型和严重程度

**呈现形式**：
- 3-5个快速问题
- 给出初步诊断
- 建议下一步

**示例对话**：

```
用户："关系有点累"

L1输出：
"听起来你在关系中感到疲惫。这种疲惫感可能来自几个方面：
- 沟通消耗
- 能量失衡
- 期待落差

你感觉最接近哪一种？还是都想聊聊？"

→ L1用户：选一个，继续深入
→ L2用户：都说，一起看
→ L3用户：最可能是能量失衡，继续
→ L4用户：都知道，想听深层分析
```

### 层级2：核心框架（所有层级可见）

**目标**：给出问题的基本框架，让用户理解问题本质

**呈现形式**：
- 问题是什么（类型定位）
- 问题从哪里来（因素追溯）
- 问题怎么解（方向指引）

**示例：沟通型卡点**

```
■ 问题类型：沟通型
  核心表现：无法有效对话/频繁争吵/感觉说不到一块

■ 生成因素：
  - 表面：对话方式不对
  - 中层：沟通模式固化
  - 深层：依恋风格差异

■ 解决方向：
  - 短期：改变对话方式
  - 中期：调整沟通模式
  - 长期：理解依恋差异
```

### 层级3：详细工具（仅L2+可见）

**目标**：提供系统性的分析工具和概念

**呈现形式**：
- 详细的概念解释
- 分析工具的使用方法
- 案例说明

**示例：依恋风格分析**

```
■ 依恋风格类型
  - 安全型：容易亲近，依赖适中
  - 焦虑型：高需求，害怕被抛弃
  - 回避型：重视独立，逃避亲密
  - 混乱型：矛盾行为，安全信号混乱

■ 你的依恋风格
  [根据诊断结果给出]

■ 伴侣的依恋风格
  [根据诊断结果给出]

■ 组合适配分析
  [根据组合给出]

■ 适配策略
  [根据类型组合给出]
```

### 层级4：行动方案（仅L3+可见）

**目标**：提供具体的解决方案和工具

**呈现形式**：
- 解题模型选择
- 话术模板
- 行动清单

**示例：SRC模型**

```
■ 适用场景
  沟通型卡点，情绪化对话，争吵升级

■ 模型原理
  S(暂停) → R(映射) → C(连接)
  暂停情绪 → 映射对方需求 → 重建连接

■ 完整话术
  【暂停阶段】
  "我们现在情绪都有点激动，我需要暂停一下，
   10分钟后我们继续聊，可以吗？"

  【映射阶段】
  "让我确认一下我理解的是不是对的...
   你现在感到...，因为...，你需要..."
  
  【连接阶段】
  "谢谢你告诉我这些。我理解你的感受。
   我现在感到...，因为...。我希望..."
```

### 层级5：深度修炼（仅L4可见）

**目标**：提供高阶的心法和修炼路径

**呈现形式**：
- 心法原理
- 修炼方法
- 境界描述

**示例：臣服关系**

```
■ 心法：臣服

■ 原理
  不是"认输"，而是"放下控制，接受事物本来的样子"
  强控制的伴侣，本质是不信任关系、不信任伴侣
  臣服 = 信任关系有自我修复的能力

■ 修炼方法
  1. 觉察：识别"我要TA做"的控制念头
  2. 暂停：在行动之前停顿
  3. 转化：把"我要TA做"转化为"我可以接受什么"
  4. 行动：选择给TA空间，而非推动TA行动
  5. 观察：观察结果，发现关系的变化

■ 境界描述
  - 初阶：能觉察到控制，但控制不住
  - 中阶：能在大多数时候放下控制
  - 高阶：自然地臣服，不觉得失去控制
  - 化境：享受关系的不确定性
```

## 对话引导策略

### 根据用户表达自动识别层级

| 用户表达特征 | 判断层级 |
|-------------|---------|
| "最近关系有点累" / "感觉不太对" | L1 |
| "我想了解为什么会这样" / "这是什么原因" | L2 |
| "有没有什么方法可以解决" / "具体怎么做" | L3 |
| "我想深入理解" / "背后是什么原理" | L4 |

### 各层级的对话策略

**L1对话策略**：

```
目标：确认问题，提供方向

策略：
1. 快速诊断（3个问题内）
2. 给出初步判断
3. 询问想深入了解哪个方面
4. 根据选择继续

示例：
用户："关系有点累"

系统："这种疲惫感通常来自几个方面。
      你是更想了解怎么改善，还是想先确定问题在哪里？"

用户："想先确定问题"

系统："好的，我问你几个快速问题：
      1. 这种累是在特定场景下，还是一直持续？
      2. 你们是话太多了，还是太少了？
      3. 你更想靠近TA，还是需要空间？"
```

**L2对话策略**：

```
目标：深入理解问题，提供系统框架

策略：
1. 提供完整的框架
2. 让用户选择深入哪个维度
3. 提供详细解释
4. 给用户自己分析的工具

示例：
用户："我想了解为什么会这样"

系统："好的，你的关系疲惫可能来自这几个层面：
      1. 能量层面（关系是充电还是耗电）
      2. 沟通层面（对话质量和频率）
      3. 依恋层面（你的依恋风格和TA的适配度）
      4. 原生家庭层面（你们各自的家庭模板）

      你最想深入了解哪个？或者我帮你做一个全面分析？"
```

**L3对话策略**：

```
目标：提供可执行的解决方案

策略：
1. 确认问题类型
2. 直接给出解题模型
3. 提供话术模板
4. 给出行动清单
5. 设定追踪节点

示例：
用户："有没有什么方法可以解决沟通问题"

系统："好的，你的沟通问题适合用SRC模型解决。
      这是专门针对情绪化对话的模型。

      【S-暂停】
      当情绪上来时，先暂停10分钟...
      
      【R-映射】
      用这些话术确认你理解TA...
      
      【C-连接】
      用这些话术重建连接...

      本周行动：
      1. 当发生争吵时，尝试暂停
      2. 用映射话术确认理解
      3. 记录效果
      
      我们下周追踪一下效果？"
```

**L4对话策略**：

```
目标：提供深度洞察和高阶心法

策略：
1. 给出问题的深层原理
2. 解释为什么表面方法有效或无效
3. 提供心法层面的洞察
4. 引导高阶修炼

示例：
用户："我想深入理解沟通问题的本质"

系统："好的，让我给你一个深度视角。

      【表层】
      沟通问题是说话方式的问题
      
      【中层】
      沟通问题是模式问题，固定了互动方式

      【深层】
      沟通问题本质是安全感问题
      - 当感到安全，沟通自然流畅
      - 当感到威胁，沟通变成防御/攻击
      - 所以问题不是"怎么说"，而是"谁在说"（安全的你还是紧张的你）

      【心法层面】
      与其学习更好的话术，不如修炼自己的安全感
      当你能带着安全感说话，同样的内容效果完全不同

      【修炼建议】
      1. 每次对话前觉察自己的状态
      2. 不带着安全感就不开口
      3. 观察自己状态对对话效果的影响"
```

## 层级跃迁机制

### 跃迁触发条件

| 从 | 到 | 触发条件 |
|----|----|---------|
| L1 | L2 | 用户主动询问"为什么"或"原因是什么" |
| L2 | L3 | 用户询问"怎么办"或"怎么做" |
| L3 | L4 | 用户询问"背后的原理"或"如何从根本上改变" |

### 跃迁引导话术

**L1→L2**：

```
"你想深入了解这是怎么回事吗？
 了解原因可以帮助你更针对性地解决。"
```

**L2→L3**：

```
"理解了问题的原因后，你可能想知道怎么解决。
 要不要我给你一些具体的方法和工具？"
```

**L3→L4**：

```
"这些方法很实用。
 如果你想从根本上改变，我们还可以从更深的心法层面入手，
 比如...（给出高阶洞察）"
```

### 降级机制

**触发条件**：
- 用户表示信息量太大
- 用户想要更简单的答案
- 用户感觉被"教育"

**降级话术**：

```
"抱歉信息量太大了。
 简单来说就是：[一句话核心]
 其他的我们可以慢慢来。"
```

### 个性化推荐

**基于用户特征自动调整**：

| 用户特征 | 默认层级 | 优化策略 |
|---------|---------|---------|
| 首次使用 | L1 | 从L1开始 |
| 有明确问题 | L2-L3 | 直接到L3 |
| 询问原理 | L4 | 从L4开始 |
| 知识丰富 | L3-L4 | 从L3开始，展开心法 |
| 新手用户 | L1-L2 | 从L1开始，逐步深入 |

## 内容呈现示例

### 问题：沟通问题

**L1呈现**：

```
■ 问题类型：沟通问题
■ 简单建议：学会暂停，学会有效表达
■ 询问：想了解更多吗？
```

**L2呈现**：

```
■ 问题类型：沟通问题
■ 问题层级：模式层（中层）
■ 生成因素：
  - 表层：说话方式
  - 中层：沟通模式固化
  - 深层：依恋风格差异
■ 解决方向：
  - 短期：改变说话方式
  - 中期：打破旧模式
  - 长期：理解依恋差异
```

**L3呈现**：

```
■ 问题类型：沟通问题
■ 推荐模型：SRC模型

【S-暂停】
话术："我觉得我们情绪都有点激动，
       先暂停10分钟可以吗？"

【R-映射】
话术："让我确认一下我理解的是不是对的。
       你现在感到...，因为...，你需要..."

【C-连接】
话术："谢谢你告诉我。
       我现在感到...，因为...。
       我希望..."

■ 本周行动：
1. 发生争吵时使用暂停话术
2. 用映射话术确认理解
3. 记录效果
```

**L4呈现**：

```
■ 深层洞察
  沟通问题的本质是安全感问题
  - 当感到安全，语言自然流畅
  - 当感到威胁，语言变成武器

■ 心法：带着安全感说话
  同样的话，从安全的状态说出，效果完全不同

■ 修炼：
  1. 觉察：对话前感受自己的状态
  2. 调整：先调整状态，再开口
  3. 验证：观察状态对效果的影响
```
