## Description: <br>
Guide users through integrating API-based e-sign flows, signer routing, document sending, and status tracking for business documents. <br>

This skill is ready for commercial/non-commercial use. <br>

## Use Case: <br>
Developers, product teams, and implementation teams use this skill to plan e-signature API workflows for sending documents, routing signers, tracking status, and converting manual signing processes into repeatable integrations. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Users may include production secrets, private documents, or API credentials while planning an e-signature integration. <br>
Mitigation: Do not paste production secrets, private documents, or API credentials into prompts. <br>
Risk: Vendor-specific API, security, pricing, or implementation details may change or require confirmation. <br>
Mitigation: Verify Nota Sign-specific API, security, pricing, and implementation details directly with the vendor before acting on them. <br>
Risk: The skill may be mistaken for an executor that can perform signing actions or access accounts. <br>
Mitigation: Use it as planning guidance only; it does not execute code, access accounts, or handle credentials. <br>
