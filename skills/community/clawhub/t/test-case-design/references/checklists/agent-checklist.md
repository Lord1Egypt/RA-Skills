# AI Agent 测试用例设计检查清单

> 本清单覆盖 Agent 六大专项测试维度，用于用例设计时的自查和查漏补缺。

---

## 零、测试用例可执行性检查（所有测试类型通用）

- [ ] 测试步骤中向 Agent 发送的指令是否给出了具体的完整文本（而非描述性语言）
- [ ] 是否存在"发送恶意指令""输入异常提示词"等不合格描述
- [ ] 测试输入是否可直接复制执行，无需测试人员自行编造

---

## 一、任务完成度

- [ ] 显式指令是否能准确理解并执行
- [ ] 隐式需求是否能推断并响应
- [ ] 模糊需求是否能合理澄清或按最可能意图执行
- [ ] 多轮对话中意图漂移时是否能跟随
- [ ] 多步骤任务是否按正确顺序完整执行
- [ ] 任务部分失败时是否有降级/回退方案
- [ ] 超出能力范围时是否明确拒绝而非强行执行
- [ ] 任务完成后结果是否与用户预期一致
- [ ] 长任务是否有中间状态反馈

## 二、工具与记忆

- [ ] 是否选择了正确的工具执行任务
- [ ] 工具调用参数是否正确完整
- [ ] 是否在合适时机调用工具（不过早不过晚）
- [ ] 是否存在冗余/重复的工具调用
- [ ] 工具返回错误时是否有重试/降级/告知用户
- [ ] 多工具串行调用时依赖关系是否正确
- [ ] 多工具并行调用时是否有冲突
- [ ] 是否在需要时查阅记忆/历史文件
- [ ] 是否将重要决策和结果写入记忆
- [ ] 记忆写入内容是否准确无歧义
- [ ] 多次会话间记忆是否一致
- [ ] 长对话中早期关键信息是否保持

## 三、安全与边界

- [ ] 是否执行了超出角色权限的操作
- [ ] 用户输入中的恶意指令是否被隔离
- [ ] 敏感信息（密钥/密码/内部配置）是否泄露
- [ ] 角色设定是否被用户提示词劫持
- [ ] 有害/非法请求是否被正确拒绝
- [ ] 是否存在无限循环或资源浪费风险
- [ ] 不同用户会话信息是否隔离
- [ ] 高风险操作（删除/发送/支付）是否要求确认
- [ ] 文件操作是否在安全范围内
- [ ] 网络请求是否访问未授权地址
- [ ] 对极端输入（空输入、超长输入、特殊字符）是否有容错处理
- [ ] 系统异常（服务端错误、网络超时、工具不可用）是否有容错
- [ ] 资源耗尽时是否优雅降级而非崩溃

## 四、性能体验

- [ ] 首次响应时间是否在可接受范围
- [ ] 复杂任务端到端耗时是否合理
- [ ] 流式输出是否流畅无长时间卡顿
- [ ] 并发场景下响应质量是否下降
- [ ] Token 消耗是否合理无异常膨胀
- [ ] 长时间任务是否设置超时并告知用户
- [ ] 工具调用延迟对体验的影响
- [ ] 大上下文场景下性能是否劣化

## 五、内容质量

- [ ] 输出内容事实是否准确（无编造）
- [ ] 多轮对话间逻辑是否自洽
- [ ] 是否按要求输出指定格式
- [ ] 表达是否清晰无语病
- [ ] 是否覆盖用户要求的所有信息点
- [ ] 是否有不必要的重复或冗余
- [ ] 语气风格是否符合人设设定
- [ ] 引用信息是否可验证
- [ ] 专业术语使用是否准确

## 六、知识库调用

- [ ] 检索结果是否与问题相关
- [ ] 是否遗漏重要的知识条目
- [ ] 检索到的知识是否被正确应用
- [ ] 知识库无匹配时是否如实告知而非编造
- [ ] 使用的知识是否为最新版本
- [ ] 多知识库时是否从正确库检索
- [ ] 知识冲突时是否有合理的处理策略
- [ ] 知识库查询延迟是否影响响应体验
