# 张小龙 · 三重验证报告

> 按照女娲造人标准，对张小龙核心心智模型进行三重验证

## 一、验证框架说明

### 三重验证标准
| 验证项 | 含义 | 通过标准 |
|--------|------|----------|
| 跨域复现 | 在≥2个不同领域/话题中出现 | 是/否 |
| 生成力 | 能推断此人对新问题的立场 | 是/否 |
| 排他性 | 不是所有聪明人都这样想 | 是/否 |

- **三重通过** → 心智模型
- **仅1-2重通过** → 降级为决策启发式
- **0重通过** → 丢弃

---

## 二、候选心智模型验证

### 模型1：用户价值第一

| 验证项 | 结果 | 证据 |
|--------|------|------|
| 跨域复现 | ✓ | 微信所有产品决策、公众号、小程序、朋友圈广告 |
| 生成力 | ✓ | 可推断张小龙对新功能、AI推荐、商业化的态度 |
| 排他性 | △ | 类似乔布斯、贝佐斯，但张小龙有其独特坚持 |

**验证结论**：**心智模型**（三重通过，可微调排他性）

**局限性**：
- 微信的成功可能使其更容易坚持
- 其他公司可能没有同样的条件

---

### 模型2：少即是多

| 验证项 | 结果 | 证据 |
|--------|------|------|
| 跨域复现 | ✓ | 微信界面八年不变、小程序去中心化、公众号设计 |
| 生成力 | ✓ | 可判断某功能是否该加 |
| 排他性 | ✓ | 大多数产品经理是功能堆砌逻辑 |

**验证结论**：**心智模型**（三重通过）

**局限性**：
- 边界模糊：什么算"少"？
- 竞争对手可能用多功能超越

---

### 模型3：善良比聪明更重要

| 验证项 | 结果 | 证据 |
|--------|------|------|
| 跨域复现 | ✓ | 不做开屏广告、隐私保护、拒绝骚扰 |
| 生成力 | ✓ | 可判断某商业决策是否符合道德 |
| 排他性 | ✓ | 大多数商业决策以效率为先 |

**验证结论**：**心智模型**（三重通过）

**局限性**：
- 善良的定义因人而异
- 长期善良与短期利益可能冲突

---

### 模型4：KPI是副产品

| 验证项 | 结果 | 证据 |
|--------|------|------|
| 跨域复现 | ✓ | 春晚红包决策、不追数据、给团队自主目标 |
| 生成力 | ✓ | 可判断某KPI是否合理 |
| 排他性 | ✓ | 大多数公司的KPI是目标而非副产品 |

**验证结论**：**决策启发式**（2重通过，降级）

**局限性**：
- 需要产品已经成功才能承受
- 不是所有公司都适用

---

### 模型5：去中心化

| 验证项 | 结果 | 证据 |
|--------|------|------|
| 跨域复现 | ✓ | 小程序、公众号、微信支付不收通道费 |
| 生成力 | ✓ | 可判断某平台政策是否合理 |
| 排他性 | △ | 类似互联网精神，但张小龙实践更彻底 |

**验证结论**：**决策启发式**（2重通过，降级）

**局限性**：
- 微信有足够的话语权
- 其他平台难以复制

## 三、验证总结

### 3.1 通过三重验证的心智模型

| 模型 | 核心要义 | 应用场景 |
|------|----------|----------|
| 用户价值第一 | 判断标准是用户是否需要 | 产品决策 |
| 少即是多 | 功能要克制 | 设计原则 |
| 善良比聪明更重要 | 道德底线 | 商业伦理 |

### 3.2 降级为决策启发式

| 启发式 | 应用场景 |
|--------|----------|
| KPI是副产品 | 管理决策 |
| 去中心化 | 平台策略 |

## 五、诚实边界声明

### 5.1 已知局限
1. **微信特殊性**：微信的成功可能使其原则难以复制
2. **信息滞后**：张小龙在不断进化，某些观点可能已更新
3. **数据缺失**：微信内部数据无法获取

### 5.2 不适用场景
- 初创公司需要快速增长
- 竞争激烈的红海市场
- 需要短期KPI的上市公司

### 5.3 使用建议
- 将此模型作为产品决策的参考框架
- 注意结合公司实际情况
- 优先考虑用户价值
