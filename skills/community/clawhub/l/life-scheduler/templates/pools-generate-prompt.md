--- IDENTITY.md ---
{identity_content}

请根据这个角色的性格、职业、气质、生活方式，生成一套完整的「生活状态创意池」。

要求生成以下 4 个池，每个池 8-12 个选项：

### 1. day_types（日程类型池）
分为 weekday（工作日）、weekend（周末）、special（特殊）三类。
- 要符合角色的职业和生活节奏
- 包含忙碌/轻松/特殊等不同强度的变体
- 具体到角色会做的事，不要太泛

### 2. moods（心情池）
- 要符合角色的性格特质
- 有正面、中性、轻微负面的自然变化
- 用角色会用的词汇，不要文绉绉

### 3. outfit_styles（穿搭风格池）
- 要符合角色的审美和身份
- 覆盖工作、休闲、约会、居家等不同场景
- 带一点具体描述（不只是"休闲风"，而是"休闲风（卫衣+运动裤）"）

### 4. events（随机事件池）
- 要符合角色的日常生活场景和职业
- 有好事、小意外、有趣的事、和用户相关的事
- 16-24 个，丰富一些

输出格式为标准 JSON，结构如下：
```json
{
  "day_types": {
    "weekday": [...],
    "weekend": [...],
    "special": [...]
  },
  "moods": [...],
  "outfit_styles": [...],
  "events": [...]
}
```

要求：
- 所有内容贴合角色人设，不要用通用模板
- 用中文
- 只输出 JSON，不要解释
