# SDM — 服务交付管理（CMMI V3.0 Domain-specific PA）

## PA 基本信息
- **缩写**：SDM
- **英文**：Service Delivery Management
- **中文**：服务交付管理
- **所属**：Domain-specific PA（领域特定实践域）
- **适用领域**：服务交付、IT服务管理

---

## CMMI V3.0 要求要点

### 核心目的
确保服务交付满足客户需求和服务水平协议。

### 关键实践（Specific Practices）
| 实践编号 | 实践内容 | 说明 |
|---------|---------|------|
| SDM 1.1 | 识别服务需求 | 识别客户服务需求、SLA要求 |
| SDM 1.2 | 策划服务交付 | 制定服务交付计划、资源计划 |
| SDM 1.3 | 建立服务交付能力 | 确保人员、工具、流程就绪 |
| SDM 1.4 | 管理服务交付 | 执行服务交付，监控服务性能 |
| SDM 2.1 | 管理服务请求 | 跟踪和处理服务请求 |
| SDM 2.2 | 管理服务质量 | 监控服务质量，确保符合SLA |
| SDM 3.1 | 改进服务交付 | 基于反馈和数据改进服务 |

### 通用实践（Generic Practices）
- GP 1.1 执行特定实践
- GP 2.1 建立组织政策
- GP 2.2 策划过程
- GP 2.3 提供资源
- GP 2.4 分配责任
- GP 2.5 培训人员
- GP 2.6 管理配置
- GP 2.7 识别并记录干系人需求和接口
- GP 2.8 监督和控制过程
- GP 2.9 客观评估符合性
- GP 2.10 高层审查

## 用户文档 → PA 匹配规则

### 高概率匹配（主要PA）
- 文档名含"服务交付"、"Service Delivery"、"SLA" → **SDM 主要匹配**
- 文档含服务目录、服务请求记录 → **SDM 主要匹配**

### 中概率匹配（次要PA）
- 文档含服务水平协议 → 同时匹配 **SAM**（供应商协议管理）
- 文档含事件记录 → 同时匹配 **IRP**（事件解决与预防）

## 评估访谈常见问题（模式D）

### 针对服务交付经理的问题
1. "你的服务交付是如何策划的？有服务交付计划吗？"
2. "服务水平协议（SLA）是如何管理的？"
3. "服务请求是如何管理和跟踪的？"
4. "如何保证服务质量？有质量监控记录吗？"

### 评分要点
- **SDM 1.1-1.4**：是否有系统化的服务交付策划和能力建立
- **SDM 2.1-2.2**：服务交付是否按SLA执行，是否有监控记录
- **SDM 3.1**：是否有基于反馈的服务改进机制
