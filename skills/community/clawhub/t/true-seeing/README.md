# True Seeing 🔮

> *6th-level Divination — See things as they actually are.*

AI 文章去幻觉校验工具。自动识别文章中的可验证事实，通过 Web Search 逐条校验，输出差异报告供用户审批后替换。

## 为什么需要它

AI 生成的内容中经常混入看似真实但实际错误的事实——日期偏移、金额夸大、事件虚构、技术架构张冠李戴。这些"幻觉"往往写得像模像样，人工逐条核实极其耗时。

True Seeing 把这个过程自动化了。

## 它能做什么

- **提取事实**：扫描文章，识别所有可验证的事实性陈述（金额、日期、事件、统计数据等）
- **并行校验**：将事实转化为搜索查询，通过 Web Search 逐条比对验证
- **差异报告**：输出清晰的三分类报告（有分歧 / 无分歧 / 未能验证）
- **审批替换**：用户选择要修正的事实，一键生成去幻觉版本

## 它不能做什么

- 不做语法纠错
- 不验证观点和预测
- 不做翻译质量检查
- 不替代专业领域（法律、医学）的人工审核

## 使用方式

触发方式（任意一种）：

```
"事实核查这篇文章"
"检查一下这些数据"
"fact check"
"去幻觉"
"核实这段话"
```

输入一篇文章（或文章片段），True Seeing 会：

1. 提取所有可验证事实并编号
2. 启动并行搜索验证
3. 输出差异报告，等待你确认
4. 根据你的选择生成修正后的文章

## 架构

三阶段 Pipeline，支持独立调用或全流程执行：

```
Stage 1: Extract → Stage 2: Verify → 🛑 用户审批 → Stage 3: Replace
```

### 状态机驱动的事实验证

每条事实经过严格的状态流转：

```
Unverified → keywords-generated → fact-fetch → Verified / Corrected / Remain
```

- 批次并行处理（每批 5 条，sub-agent 并行搜索）
- 搜索失败自动换策略重试（英文查询 / 去数值 / 扩大范围）
- 最多 5 轮循环兜底，确保每条事实都有终态

### 伪量化检测

自动识别看似精确实则虚构的数据（如"物理模拟准确度 99.9%"），优先搜索指标本身的学术来源而非仅搜索数值。

## 分类覆盖

| 优先级 | 分类 |
|--------|------|
| P0 核心必检 | 融资金额、产品定价、具体日期、重大事件 |
| P1 扩展 | 统计数据、人事变动、产品发布、排名 |
| P2 可选 | 地理位置、政策法规、其他硬事实 |

## 测试结果

在包含 18 条幻觉的 AI 生成文章中，True Seeing 识别出了全部 13 条有分歧事实和 2 条无法验证事实，准确率 100%。
