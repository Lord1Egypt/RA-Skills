# ВЕРИФИКАТОР v2.0

## Контекст

Этот скилл используется для проверки кандидатов на адаптацию стартапов для российского рынка. Каждый кандидат оценивается по 14 критериям. Результат определяет судьбу кандидата: GREEN / YELLOW / RED.

---

## КАК ИСПОЛЬЗОВАТЬ

1. Получить результаты от **Стратега** и **idea-reality-mcp API**
2. Применить 14 пунктов проверки
3. Посчитать ✅/⚠️/❌
4. Определить итоговый вердикт

---

## 14 ПУНКТОВ ПРОВЕРКИ

| # | Критерий | Что проверять | Оценка |
|---|----------|---------------|--------|
| 1 | **PMF за рубежом** | Продукт доказал PMF в другой стране: MRR/ARR, valuation, users, funding. Источники: Indie Hackers, YC batch reports, VC announcements. | ✅ = есть конкретные цифры; ⚠️ = признаки PMF без точных цифр; ❌ = только гипотеза |
| 2 | **Барьер для входа в РФ** | Существует объективный барьер: санкции, Visa/Mastercard блокировка, regulatory, заблокированные API, нет интереса к российскому рынку. | ✅ = барьер подтверждён документально; ⚠️ = возможный барьер; ❌ = барьер отсутствует |
| 3 | **Структурный барьер** | Western SaaS не работает в России без существенной переделки: зависимость от Western APIs, платёжных систем, инфраструктуры. | ✅ = требует полного ребилда; ⚠️ = требует частичной адаптации; ❌ = достаточно локализации |
| 4 | **Нет сильного российского аналога** | При проверке не найдено конкурентов с аналогичным функционалом. Проверять: vc.ru, habr.com, tenchat.ru, GitHub, Telegram. | ✅ = не найдено конкурентов; ⚠️ = 1-2 слабых конкурента; ❌ = 3+ конкурента с полным функционалом |
| 5 | **Для РФ нужен ребилд** | Western dependencies требуют замены на российские аналоги. Проверить: какой стек используется, можно ли заменить. | ✅ = требуется замена ключевых компонентов; ⚠️ = требуется точечная замена; ❌ = достаточно локализации текста |
| 6 | **Пригодность для vibe coding** | Оценить: 1-2 разработчика, 2-4 недели на MVP. Критерии: сложность стека, доступность библиотек, опыт команды. | ✅ = 1 чел, ≤4 недель; ⚠️ = 2 чел, ≤8 недель; ❌ = требует команды 3+ или >2 месяцев |
| 7 | **Кризисный профиль** | Продукт решает проблему экономии времени или денег. Приоритет: upskilling, automation, productivity, cost-saving. | ✅ = высокий кризисный профиль (решает реальную боль в кризис); ⚠️ = средний; ❌ = не связан с экономией |
| 8 | **CRDT-viability** | Доступные способы приёма платежей в России: ЮKassa, СБП, USDT, Telegram Stars, ЮMoney. | ✅ = 3+ способа доступны; ⚠️ = 1-2 способа; ❌ = платежи невозможны |
| 9 | **Команда/единомышленники** | Есть open-source библиотеки, сообщество, документация, готовые решения. | ✅ = сильное сообщество, много библиотек; ⚠️ = есть отдельные ресурсы; ❌ = всё с нуля |
| 10 | **Legal/Compliance** | Не требует специальных лицензий, разрешений, сертификаций. | ✅ = не требует; ⚠️ = требуется регистрация ИП/самозанятости; ❌ = нужны лицензии |
| 11 | **Монетизация** | Ясная модель: подписка, pay-per-use, freemium. Цена понятна для целевой аудитории. | ✅ = модель очевидна и протестирована; ⚠️ = модель неясна но возможна; ❌ = непонятно как монетизировать |
| 12 | **Масштабируемость** | Архитектура позволяет расти без полной перестройки. | ✅ = горизонтально масштабируется; ⚠️ = требует оптимизации; ❌ = вертикальный потолок |
| 13 | **Выход на рынок** |通道: Product Hunt, Telegram-каналы, Indie Hackers, VC-сообщества. Есть путь привлечения первых пользователей. | ✅ = каналы очевидны; ⚠️ = каналы возможны но не очевидны; ❌ = непонятно как выйти |
| 14 | **Время до MVP** | Реалистичный срок: от идеи до рабочего прототипа. | ✅ = ≤4 недель; ⚠️ = 1-2 месяца; ❌ = >2 месяцев |

## КРИТЕРИИ ОТСЕИВАНИЯ

### 🔴 АВТОМАТИЧЕСКИЙ RED

Если любой из этих пунктов = ❌ → RED:
- Пункт 4 (нет сильного российского аналога) → ищи других конкурентов, не останавливайся
- Пункт 6 (vibe coding) → слишком сложно для MVP
- Пункт 14 (время до MVP) → больше 2 месяцев

### 🟡 YELLOW (продолжить проверку)

Если:
- 10-13 ✅ → GREEN
- 7-9 ✅ → YELLOW (продолжить анализ)
- ≤6 ✅ → RED

## КЛЮЧЕВЫЕ ПРАВИЛА

1. **Не принимать решения на основе одного источника** — сверять: API + GitHub + Google search + Telegram
2. **Если конкурент найден** — проверить его функционал и цены; если аналог полный и дешёвый → RED
3. **0 HN posts** — не приговор; если в России конкурентов нет → это хорошо
4. **Время** — кризисный контекст = ускоренные решения; 2-4 недели нормально
5. **Vibe coding фокус** — если требуется 3+ месяца и команда 5+ → RED
