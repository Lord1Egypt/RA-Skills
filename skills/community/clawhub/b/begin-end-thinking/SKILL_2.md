# 分叉法思维

## Overview

分叉法是一种问题拆解思维，通过以终为始、从结果反推过程，找到影响处理方式和输出内容的分叉点，让复杂问题处理更清晰、沟通更顺畅。

**核心**：不是分类问题，而是梳理思路、抓主要矛盾、一步步简化问题。

## When to Use

- 问题很乱，不知道从哪里下手
- 对方请求不清晰，需要引导对方理清
- 沟通时说不清楚，想要更清晰的表达
- 面对投诉/威胁，不知道怎么回应
- 做教程/文档，想要输出更有针对性

**不适用的场景**：简单的二选一问题（如今天喝咖啡还是喝茶）

## Core Pattern

### 分叉法五步引导模板

适用于引导他人梳理问题（如客服引导客户）：

**第一步：以终为始**
> "您想解决这个问题，您是想要**快速完成**，还是想**了解整个流程**？"

**第二步：找影响因素**
> "要**快速完成**，您觉得有哪些因素会影响？"

- 让对方**自己说**，不提示、不给选项

**第三步：排序确认**
> "这些因素里，哪个最重要？其次是？"

- 让对方**自己排序**，不提供选项

**第四步：确认（输出思路链路图）**
> "根据您说的，我理解是：
> - 您的[因素1]是...
> - 您的[因素2]是...
>
> 这样理解对吗？"

**第五步：执行建议**
> "根据您的情况，建议您..."

### 分叉树结构

```
根节点（问题本身 / 最终想要的结果）
  ↓ 按影响程度排序
枝干1（最大影响因素）
  ├─ 子分叉A
  └─ 子分叉B
枝干2（次大影响因素）
  └─ ...
```

**特点**：
- 树形嵌套
- 同层互斥（选了A就不选B）
- 按优先级顺序执行
- 3-5层，不到步骤层

### 分叉点判断标准

当一个问题的不同分支需要：
- **不同的处理方式**？
- **不同的输出**？

两者有其一 → 这里就是分叉点

## 分叉法 vs 分类

| | 分类 | 分叉点 |
|--|------|-------|
| 本质 | 静态结果 | 动态过程 |
| 目的 | 分出种类 | 整理思路、抓主要矛盾 |
| 效果 | 知道是什么 | 简化并解决问题 |

## Quick Reference

### 遇到问题时的思考顺序

```
1. 以终为始 → 结果有哪些种？（最终状态/目的）
2. 取最优结果
3. 反推过程 → 找分叉点
4. 按影响程度排序（互斥，顺序关系）
5. 层层往下拆（3-5层，不到步骤层）
6. 验证：逻辑清晰 + 路径清晰
```

### 验证标准

分叉之后：
- 逻辑更清晰 ✓
- 路径更清晰 ✓

如果拆完发现逻辑还是乱/路径不清 → 分叉有问题

### 关键原则

- **我是引导者，不是分析者**
- **让对方自己说，不是让对方选**
- **我负责画图+确认，对方负责说清楚**

## 实战案例

### 案例一：投诉威胁

**情境**：对方说"你不帮我处理，我就投诉你"

**分叉分析**：
| 分叉 | 内容 |
|------|------|
| 威胁背后真正想要 | 不是投诉，是想让我帮他处理 |
| 分叉点1 | 是不是我的职责 |
| 分叉点2 | 我有没有能力/权限处理 |
| 分叉点3 | 投诉后果是什么 |

**回应方向**：
> "我理解您想解决这件事。让我先确认一下，这件事具体是..."

### 案例二：同事推任务

**情境**：同事说领导安排了一个任务给你，但你觉得这不是你应该做的

**你的结果**：不想做 + 不想翻脸

**分叉分析**：
| 分叉 | 内容 |
|------|------|
| 分叉点1 | 是不是我的职责（最重要） |
| 分叉点2 | 是不是领导的安排（其次） |

**分叉结果**：
| 情况 | 职责？ | 领导安排？ | 建议 |
|------|--------|-----------|------|
| A | 是 | 是/不是 | 考虑做 |
| B | 不是 | 是 | 找理由 + 建议找对人 |
| C | 不是 | 不是 | 拒绝 |

### 案例三：客服引导

**情境**：客户咨询"制作法人章怎么弄"

**五步引导**：
1. "您是想快速完成，还是想了解整个流程？"
2. "要快速完成，您觉得有哪些因素会影响？"
3. "这些因素里，哪个最重要？其次是？"
4. 确认思路链路图
5. "根据您的情况，建议您..."

## Common Mistakes

### 错误1：直接分析对方的分叉点

**错误**：我问客户的版本/权限是什么

**正确**：我设计分叉框架，让客户自己填写自己的情况

### 错误2：给选项让对方选

**错误**：问"权限和版本，哪个更重要？"（给选项）

**正确**：问"这些因素里，哪个最重要？"（让对方自己说）

### 错误3：拆得太细

**错误**：分叉到具体操作步骤

**正确**：3-5层，拆到能大概告知处理方式为止
