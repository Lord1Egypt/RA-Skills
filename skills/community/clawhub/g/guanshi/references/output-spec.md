# 输出规范

> Step 6 综合建议 + 最终报告输出前强制加载。本规范定义战略报告的格式、叙事逻辑、数据呈现和置信度体系，参照顶级战略咨询报告标准（德施曼组织诊断报告为基准模板）。

---

## 一、核心准则

### 1.1 标题即结论

**每节标题必须是完整论断，不是主题标签。**

| 错误（主题标签） | 正确（标题即结论） |
|---|---|
| "市场环境分析" | "广东啤酒市场年增8%，深圳外二三级城市存在显著未覆盖空间" |
| "竞争格局" | "青岛通过渠道下沉和纯生产品正在蚕食金威在深圳的领先地位" |
| "战略建议" | "推出六种核心产品覆盖全细分市场，优先从低价位绿金威和纯生起步" |

### 1.2 "但"字转折节奏

每 3-4 个分析模块之间插入一个"但"字转折，制造叙事张力：

```
肯定优势 → 但 → 揭示挑战 → 因此 → 提出方案
```

示例：
- "百威在深圳已建立领导地位...但广州与周边市场存在显著差异...因此需针对不同城市群差异化布局"
- "边际利润全国领先...但定价缺乏系统性，绿金威与老金威价位过近形成内耗...因此需建立科学定价体系"

### 1.3 每节闭环

每个分析节/模块必须在一屏内完成：**标题（结论）→ 数据/图表 → 推导 → 子结论**。

### 1.4 数据三源交叉

每条关键数据必须从三个来源中至少引用两个：
- **外部来源**：行业报告 / 第三方调研 / 公开财报
- **内部来源**：客户提供数据 / 内部财务 / 运营数据
- **分析团队**：模型推算 / 逻辑推导

每项数据标注来源格式：
```
资料来源：XXX行业报告；客户财务数据；观势分析
```

### 1.5 报告第一句话必须是一个判断

- 错误："市场进入是一个复杂的过程，需要多方面评估..."
- 正确："这个市场的窗口期只剩 6 个月，再犹豫就会被对手锁定。"
- 禁止用模糊开场，第一句话就是结论。

## 二、报告结构（PART 1-4 叙事驱动）

### 2.1 核心设计理念：叙事驱动而非框架驱动

最终报告采用 PART 1-4 叙事结构，以"成功经验回顾 → 核心矛盾诊断 → 系统解决方案 → 实施路线图"为主线层层递进。分析框架（五看三定、VRIO、7S、ADKAR、OHI、波特五力、安索夫等）的产出物按叙事线整合进对应 PART，而非按框架维度平铺。读者看到的是故事，不是框架罗列。

**基准模板**：德施曼组织诊断报告。所有诊断报告的输出结构和叙事节奏应以此模板为参照。

```
封面 + 机密声明 + 报告摘要
 └─ 前言（调研方式、访谈覆盖、分析方法论简述）
     │
     ├─ PART 1：过往成功经验与未来挑战
     │   ├── 1.1 过去为什么成功（核心能力 + 成功模型总结）
     │   ├── 1.2 未来挑战与机会（行业趋势 + 战略跃迁 + BLM 预判）
     │   │    ├── 五看三定（市场洞察核心方法）
     │   │    │   ├── 一看行业 → 二看市场(含TAM-SAM-SOM) → 三看客户
     │   │    │   ├── 四看竞争 → 五看自己(VRIO初判)
     │   │    │   ├── 三定（战略控制点 / 目标 / 策略）
     │   │    │   └── 机会与威胁清单
     │   │    └── BLM 一致性校验（战略维度末尾）
     │   └── 1.3 差距分析（业绩差距 + 机会差距 + 根因回溯 + 核心问题重述）
     │
     ├─ PART 2：当前核心问题——系统性诊断
     │   ├── 2.1 问题树 MECE 分解
     │   ├── 2.2 竞争定位（波特五力 + 战略群组）
     │   ├── 2.3 内部诊断（VRIO + 7S + ADKAR + OHI 四大工具交叉验证）
     │   └── 2.4 核心问题总结：系统性裂缝（裂缝之间相互强化机制 + 杠杆点识别）
     │
     ├─ PART 3：系统解决方案——从旧模式到新模式
     │   ├── 3.1 战略选项生成（安索夫矩阵 + 优先级筛选）
     │   ├── 3.2 BLM 新业务设计（六大要素重新设计）
     │   ├── 3.3 宪法审计（七维硬门控 + 整改时间表/门控节点）
     │   ├── 3.4 针对性解决方案（针对每道裂缝的举措表）
     │   │    格式：核心举措 | 具体内容 | 责任人 | 时间 | 产出
     │   └── 3.5 BLM 重构（战略-组织一致性修复 + 一致性校验）
     │
     └─ PART 4：实施路线图与组织保障
         ├── 4.1 情景规划（三种情景 + 各情景下的战略应对）
         ├── 4.2 三阶段突围路线（止血→造血→增长 / 含阶段里程碑表）
         ├── 4.3 战略放弃清单（必须放弃的执念 + 预期效果）
         ├── 4.4 举措优先级矩阵（P0/P1/P2）
         ├── 4.5 组织保障（陪跑机制 + 复盘机制）
         └── 4.6 结语（回到核心主张，给出路径总览）
```

### 2.2 报告摘要格式

报告摘要是全文的精缩版，必须包含：
1. **背景**：企业做了什么、当前面临什么局势（1-2句）
2. **调研基础**：访谈场数/素材量/数据规模（1句）
3. **核心发现**：过去成功模型是什么、为什么现在成为瓶颈（3-5句）
4. **核心矛盾**：一句话概括（加粗引用格式）
5. **战略放弃**：3个"必须做的放弃"（简短列出）

**示例格式**：
```
> **核心矛盾**：
> 过去的XX驱动增长模式已经触达天花板，但组织的系统能力还没有发育出来。

**三个必须做的放弃**：
1. 放弃"XXX"——聚焦XXX
2. 放弃"XXX"——接受XXX
3. 放弃"XXX"——给XXX窗口期
```

### 2.3 前言格式

必须包含：
- **调研方式表**：调研方式 | 具体内容 | 规模（可选，有原始调研数据时输出）
- **访谈覆盖表**：日期 | 访谈主题 | 涉及部门/角色（可选，有访谈数据时输出）
- **分析方法论简述**：2-3 句话说明采用的分析框架和方法论

### 2.4 BLM 一致性贯穿

报告中 BLM 一致性校验分布在以下位置：
1. **PART 1 五看三定末尾**：战略维度与战略意图的对齐度校验
2. **PART 2 内部诊断末尾**：组织/人才/文化与战略意图的对齐度校验（内嵌于裂缝分析中）
3. **PART 3 末尾**：BLM 重构后的六要素总体一致性校验

**各位置格式**：

**PART 1 末尾（战略维度）**：
```
> BLM 一致性校验：战略维度与战略意图的对齐度——[✓ 对齐 / △ 部分偏移 / ✗ 矛盾]
> 偏移点：[具体不一致之处]
> 对整体战略执行的风险：[评估影响]
```

**PART 3 末尾（BLM 重构后总体）**：
```
> BLM 总体一致性：[高 / 中 / 低]
> 过去→新战略的演变：[继承X项 / 抛弃X项 / 新建X项]
> 最脆弱的一致性环节：[维度-要素-风险]
```

### 2.5 核心矛盾定位规范

诊断报告必须定位到**唯一的核心矛盾**，不是"多个因素叠加"。核心矛盾是贯穿整个 PART 1-4 的叙事主线。

**格式**：
```
> **核心矛盾**：
> [过去的成功模式]已经触达天花板，但[新要求的能力]还没有发育出来。
```

**判断标准**：如果去掉这个矛盾，问题是否依然存在？是→不是核心；否→找到它。

### 2.6 成功模型总结规范（PART 1.1）

过去成功经验的总结必须以**模型公式**收束：

```
成功模型总结：
[要素1] × [要素2] × [要素3] × [要素4] = [持续时长]增长
```

并且必须紧跟一段"但"，说明每个要素在当前阶段面临什么挑战：
```
但这个模型的每个要素都在当前阶段面临挑战：
- 要素1 → 但[挑战]
- 要素2 → 但[挑战]
```

### 2.7 裂缝分析规范（PART 2.4）

核心问题总结采用"裂缝"隐喻——不是罗列问题，而是揭示**系统性裂缝**及其**相互强化机制**。

- 每道裂缝格式：`裂缝N：X与Y的裂缝` → 具体表现（≥3条） → 根因（一句话）
- 裂缝之间必须绘制**相互强化回路图**，标注**杠杆点**（打破回路的切入点）
- 杠杆点必须给出理由："必须从'裂缝X'入手，因为只有X解决，Y才能..."

**示例**：
```
### 裂缝一：战略意图与组织能力的裂缝
战略意图："XXX"
     ↕ 裂缝
组织能力：XXX
**表现**：...
**根因**：...

### 裂缝之间的相互强化
裂缝一 → 裂缝二 → 裂缝三
   ↑                  │
   └──────────────────┘
**打破回路的杠杆点**：必须从"裂缝X"入手，因为...
```

### 2.8 战略放弃规范（PART 4.3）

战略不仅是"选择做什么"，更是"选择不做什么"。报告必须包含"战略放弃"专节。

**格式**（每个放弃项独立小节）：
```
### 放弃一：放弃"[执念]"的执念
**现状**：[当前做法和数据]
**放弃内容**：
- 不再XXX
- 聚焦XXX
- 其余XXX定位为XXX
**预期效果**：
- 效果1
- 效果2
```

**数量要求**：至少 3 个放弃项。

### 2.9 针对性解决方案规范（PART 3.4）

解决方案必须针对 PART 2 识别的每道裂缝逐一给出。每个方案用表格格式：

```
### 解决方案X：修补裂缝X（... ↔ ...）
| 举措 | 具体内容 | 责任人 | 时间 | 产出 |
|------|---------|--------|------|------|
| ... | ... | ... | ... | ... |
```

### 2.10 门控节点规范（PART 3.3 宪法审计后）

宪法审计如产出整改要求，必须附带**门控时间表**：

```
Month 0   Month 3                Month 6
   │         │                       │
   ├─ 动作1 ────────────────────→ 指标1
   ├─ 动作2启动 ────────────────→ 指标2
   └─ 动作3建立 ────────────────→ 决策：继续或停止
```

### 2.11 结语规范（PART 4.6）

结语必须回到核心主张，给出路径总览，不引入新信息：

```
**核心主张**：[一句话，呼应核心矛盾]
**具体路径**：
1. 止血（M0-M6）：...
2. 造血（M7-M18）：...
3. 增长（M19-M36）：...
**战略放弃**：[简要复述]
```

## 三、数据呈现规范

### 3.1 图表使用规则

| 情景 | 推荐图表 | 示例 |
|------|---------|------|
| 市场规模/增长 | 柱状图（多年份多市场并列） | 广州130万吨/二级城市152万吨/三级城市99万吨 |
| 市场构成/份额 | 饼图 | 收入分布、消费场景占比 |
| 多对象多指标对比 | 结构化对比表 | 四城市十项购买因素对比表 |
| 二维变量定位 | 矩阵/坐标图 | 品质×市场矩阵、价格×年龄定位图 |
| 问题诊断 | 结构化问题表（问题→表现→原因） | 定价问题诊断表 |
| 实施计划 | 甘特图（阶段→月份→任务→负责人） | 一号产品实施工作计划 |
| 优先级排序 | 紧迫性×效益矩阵 | 六种产品建设重点顺序 |

### 3.2 关键数字呈现

- **锚定+对比**：三个以上数字并列对比，形成规模认知
- **百分比集约**：用百分比建立精确感，每个百分比标注基数
- **样本量必标**：所有调研数据标注 n=XXX
- **时间序列**：趋势数据用 年份→预测线，制造紧迫感
- **分级锚定**：用已知参照物锚定目标值（例："价格与喜力相当""略低于国外品牌""比XX略高"）

### 3.3 数据来源标注格式

每项数据/图表旁统一格式：
```
资料来源：XX第三方报告；客户XX数据；观势分析
```

### 3.4 TAM-SAM-SOM 市场量化规范

输出格式（来自 six-steps.md Step 2）：
| 层级 | 定义 | 规模 | 数据来源 | 置信度 |
|------|------|------|---------|--------|
| TAM | [总可触达市场] | [亿元/万件等] | | |
| SAM | [可服务市场 — 能力可及] | [亿元/万件等] | | |
| SOM | [可获取市场 — 资源可拿] | [亿元/万件等] | | |

SOM 必须拆分为：现有渗透份额 + 短期可获取增量，分别标注。

## 四、方案展开模板（安索夫矩阵驱动 + 4P 标准）

### 4.1 安索夫矩阵（选项来源）

每个战略选项必须从安索夫四象限出发推导，不能凭直觉生成。
如 Step 1 标记安索夫矩阵为"必须"或"部分"，必须在报告中输出完整矩阵。

**输出格式**：
```
安索夫矩阵：[客户公司] 战略选项来源

                    现有产品              新产品
              ┌──────────────────┬──────────────────┐
    现有市场  │  市场渗透         │  产品开发         │
              │  [选项X]          │  [选项Y]          │
              ├──────────────────┼──────────────────┤
    新市场    │  市场开发         │  多元化           │
              │  [选项Z]          │  [选项W]          │
              └──────────────────┴──────────────────┘
```

**四象限至少有一个象限有选项**。如果某象限确实无可行性，标注"不适用 — [理由]"。

### 4.2 4P 标准方案模板

每个战略方案必须按以下固定模板展开，禁止自由格式：

```
【方案名称】
├─ 价值定位：（一句话核心主张）
├─ 目标客群：
│   ├─ 人群画像：（年龄/性别/收入/职业/教育）
│   ├─ 行为特征：（消费场景/频次/决策因素）
│   └─ 人群规模：（基数 + 渗透率假设）
├─ 产品/服务设计：
│   ├─ 核心特征：（规格/参数/差异化点）
│   ├─ 品牌/标识：（命名/视觉/调性）
│   └─ 包装/交付：（规格/渠道适配）
├─ 定价策略：
│   ├─ 价格区间：（锚定参照品牌/品类）
│   ├─ 定价逻辑：（成本+/价值-/竞争参照）
│   └─ 价格体系：（与现有产品的价差/梯度）
├─ 渠道策略：
│   ├─ 主渠道：（类型/覆盖/管控方式）
│   └─ 次渠道：（补充覆盖）
└─ 推广策略：
    ├─ 广告立意：（核心信息/情感主张）
    ├─ 投放渠道：（媒体/场景）
    └─ 促销方式：（启动期/日常/节庆）
```

## 五、实施计划规范

### 5.1 优先级排序矩阵

```
          经济效益 →
紧迫性    高        低
  ↓    ┌────────┬────────┐
 高    │ 优先①  │ 次优②  │
       ├────────┼────────┤
 低    │ 观察③  │ 搁置④  │
       └────────┴────────┘
```

### 5.2 甘特图格式

| 阶段 | 月份 | 具体任务 | 负责人 |
|------|------|---------|--------|
| 产品开发 | 3月 | 配方测试 | 研发总监 |
| | 4月 | 小规模试产 | 生产总监 |
| 方案设计 | 4月 | 定价模型 | 市场总监 |
| | 5月 | 包装设计 | 品牌总监 |
| 推广测试 | 6-7月 | 深圳试点 | 区域经理 |
| 全面推广 | 8月起 | 珠三角铺货 | 渠道总监 |

### 5.4 情景规划（三种情景）

如 Step 1 问题树标记情景规划为"必须"或"部分"，必须在报告中输出。

**格式**：采用三种情景（基准/乐观/悲观），每种标注概率、行业环境、关键挑战。

```
### 三种情景设定
| 情景 | 概率 | 行业环境 | 关键挑战 |
|------|:----:|---------|---------|
| **基准情景** | 50% | [描述] | [挑战] |
| **乐观情景** | 30% | [描述] | [挑战] |
| **悲观情景** | 20% | [描述] | [挑战] |
```

**每种情景下的战略应对**必须单独展开：
```
#### 基准情景（概率50%）——"[主题标签]"
战略主题：[一句话]
关键动作：
  1. [动作1]
  2. [动作2]
成功条件：[一句话]
```

**情景规划结论**：明确基准情景是制定实施路线图的基准假设，但必须定期重新评估情景概率。

### 5.4b 三阶段突围路线（止血→造血→增长）

实施路线采用三阶段框架。每个阶段必须包含：核心目标、关键举措、门控节点。

**总览表**：
```
| Phase | 时间 | 核心目标 | 关键里程碑 |
|-------|------|---------|-----------|
| 止血(Stabilize) | M0-M6 | 活下来 | [里程碑1-3] |
```

### 5.5 Revenue Bridge（增长路径分解）

如 Step 1 问题树包含增长路径分支，必须在第三幕输出。

**输出格式**：
| 收入来源 | 当前占比 | 目标占比 | 变化方向 | 驱动因素 | 置信度 |
|---------|---------|---------|---------|---------|:---:|
| [业务线1] | X% | Y% | ↑/↓/→ | [驱动因素描述] | |
| [业务线2] | X% | Y% | ↑/↓/→ | [驱动因素描述] | |
| [新增业务线] | — | Z% | 新增 | [驱动因素描述] | |

Revenue Bridge 本质是回答"增长从哪来"——每个收入来源的变化必须对应到具体的驱动因素（价格提升 / 销量增长 / 新产品 / 新市场），不能只有数字没有逻辑。

## 六、去 AI 味规范

基于 humanizer-zh 原则，输出前逐条检查：

| 检测项 | 问题特征 | 修正方法 |
|--------|---------|---------|
| 夸大宣告 | "无疑将""必将""史诗级""颠覆性" | 改为量化判断或标注置信度 |
| 破折号滥用 | 每段超过 1 个"——" | 每段最多保留 1 个，其余改为句号 |
| 三段式法则 | "首先...其次...最后..."强行三段 | 按逻辑自然分段 |
| AI 高频词 | "值得注意的是""关键在于""更重要的是" | 删除，核心语义不变就删除 |
| 否定式排比 | "不是...不是...而是..." | 直接陈述判断 |
| 模糊归因 | "业内人士""普遍认为""众所周知" | 改为具体来源或标注"推断" |
| 修辞膨胀 | 形容词堆叠、比喻泛滥 | 删除修饰词，保留事实和判断 |

**锋利性约束**：
- 执行摘要框架表：≤ 8 维度
- 每个战略方案：按 4P 模板展开，不额外扩展
- P0 最多 2 个动作，P1 最多 3 个动作
- 每个动作一句话描述 + 一句解释"为什么是现在"

## 七、置信度体系

每条关键结论必须标注置信度：

| 等级 | 定义 | 标注示例 |
|------|------|---------|
| **高** | 外部调研 + 内部数据 + 分析团队三方交叉验证 | "（高 — URC调研+客户财务+观势分析一致）" |
| **中** | 两个来源交叉验证，或单一来源+逻辑推演 | "（中 — 行业均值推断+逻辑推演，需实地验证）" |
| **低** | 基于推断或类比，建议验证后再决策 | "（低 — 基于类比推断，缺乏直接数据）" |

**证伪条件**：每条中/低置信度结论，标注"什么证据能推翻它"。
