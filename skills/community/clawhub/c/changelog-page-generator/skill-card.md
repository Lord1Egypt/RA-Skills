## Description: <br>
Helps agents create, optimize, and structure changelog or release notes pages, including layout, entry templates, navigation, SEO, and integration guidance. <br>

This skill is ready for commercial/non-commercial use. <br>

## Use Case: <br>
Developers, product teams, and documentation writers use this skill to plan or draft changelog and release notes pages for SaaS, API, mobile, or similar products. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: The skill may read local project-context files to tailor changelog guidance, and those files could contain secrets or unrelated sensitive information. <br>
Mitigation: Keep project-context files free of secrets and unrelated sensitive data before invoking the skill. <br>
Risk: Broad update-related wording may trigger the skill when a user intended a different website or documentation workflow. <br>
Mitigation: Confirm the user wants changelog or release-notes page guidance when the request is ambiguous. <br>
