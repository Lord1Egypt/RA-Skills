# 测试用例输出模板

## 用例数量要求（参考值）

**关键指标**：根据项目复杂度调整

| 复杂度 | 倍数 | 示例（显性5条） |
|--------|------|----------------|
| 简单项目 | ×3 | 用例15条 |
| 中等项目 | ×5 | 用例25条 |
| 复杂项目 | ×7 | 用例35条 |

**复杂度判断标准**：
- 简单：单模块、低风险、无并发
- 中等：多模块、中风险、少量并发
- 复杂：跨模块、高风险、高并发

## 覆盖维度要求

**必须覆盖以下6个维度**：

| 维度 | 占比 | 说明 |
|------|------|------|
| 功能测试 | 40% | 核心业务流程、正常操作 |
| 异常测试 | 20% | 输入异常、状态异常、网络异常 |
| 边界测试 | 15% | 值域边界、格式边界、状态边界 |
| 并发测试 | 10% | 多人同时操作、数据竞争 |
| 安全测试 | 10% | 注入、越权、敏感信息 |
| 性能测试 | 5% | 响应时间、资源消耗 |

## 标准格式

### Markdown表格格式

| 用例编号 | 需求ID | 风险ID | 测试类型 | 功能模块 | 测试标题 | 用例级别 | 预置条件 | 测试步骤 | 预期结果 | 风险等级 |
|---------|--------|--------|---------|---------|---------|---------|---------|---------|---------|---------|
| TC_XXX_001 | REQ-XXX-001 | RISK-XXX-001 | 功能测试 | 模块名 | 标题描述 | P0/P1/P2/P3 | 条件描述 | 步骤1;步骤2;步骤3 | 结果描述 | 高/中/低 |

### 字段说明

| 字段 | 说明 | 格式 | 示例 |
|------|------|------|------|
| 用例编号 | 唯一标识 | TC_{模块}_{功能}_{序号} | TC_CART_ADD_001 |
| 需求ID | 关联的需求编号 | REQ-{模块}-{序号} | REQ-CART-001 |
| 风险ID | 关联的风险编号 | RISK-{模块}-{类型}-{序号} | RISK-CART-STOCK-001 |
| 测试类型 | 功能/接口/安全/性能/兼容性/并发/异常 | - | 功能测试 |
| 功能模块 | 所属业务模块 | - | 购物车 |
| 测试标题 | 简明描述测试目的 | - | 验证添加正常商品成功 |
| 用例级别 | P0关键/P1重要/P2一般/P3可选 | - | P0 |
| 预置条件 | 执行前需满足的条件 | - | 用户已登录 |
| 测试步骤 | 详细操作步骤，用分号分隔 | - | 步骤1;步骤2;步骤3 |
| 预期结果 | 每步对应的预期结果 | - | 结果1;结果2;结果3 |
| 风险等级 | 高/中/低 | - | 高 |

## ID编号规则

### 用例编号
```
格式：TC_{模块缩写}_{功能缩写}_{序号}
示例：
- TC_CART_ADD_001      购物车模块-添加商品-第1条
- TC_CART_DEL_002      购物车模块-删除商品-第2条
- TC_CART_PAY_003      购物车模块-结算-第3条
```

### 需求ID
```
格式：REQ-{模块缩写}-{序号}
示例：
- REQ-CART-001    购物车模块-需求1
- REQ-CART-002    购物车模块-需求2
```

### 风险ID
```
格式：RISK-{模块缩写}-{风险类型}-{序号}
风险类型：
- STOCK   库存相关
- CALC    计算相关
- CONC    并发相关
- TIME    超时相关
- DATA    数据相关
- SEC     安全相关
- PERF    性能相关

示例：
- RISK-CART-STOCK-001    购物车-库存风险-第1个
- RISK-CART-CALC-001     购物车-计算风险-第1个
- RISK-CART-CONC-001     购物车-并发风险-第1个
```

### 场景ID（来自qa-scenario-tree）
```
格式：SC-{模块缩写}-{序号}
示例：
- SC-CART-001    购物车场景1
- SC-CART-002    购物车场景2
```

### 边界ID（来自qa-boundary-deep-dive）
```
格式：BD-{模块缩写}-{序号}
示例：
- BD-CART-001    购物车边界1
- BD-CART-002    购物车边界2
```

### 评审ID（来自qa-expert-review）
```
格式：REV-{模块缩写}-{序号}
示例：
- REV-CART-001    购物车评审1
```

## 用例编号规则

### 格式
```
TC_{模块缩写}_{功能缩写}_{序号}
```

### 示例
```
TC_USER_LOGIN_001     用户模块-登录-第1条
TC_ORDER_PAY_002      订单模块-支付-第2条
TC_PRODUCT_LIST_003   商品模块-列表-第3条
```

### 模块缩写规则
| 模块 | 缩写 |
|------|------|
| 用户管理 | USER |
| 订单管理 | ORDER |
| 商品管理 | PRODUCT |
| 支付管理 | PAY |
| 库存管理 | STOCK |
| 营销管理 | MARKET |
| 系统管理 | SYS |

## 用例级别定义

| 级别 | 说明 | 占比建议 | 示例场景 |
|------|------|---------|---------|
| P0 | 关键：核心业务流程 | ≤20% | 登录、支付、下单 |
| P1 | 重要：主要功能 | ≤40% | 搜索、筛选、导出 |
| P2 | 一般：次要功能 | ≤30% | 设置、帮助、关于 |
| P3 | 可选：边缘场景 | ≤10% | 美化、优化、建议 |

## 测试类型分类

| 类型 | 说明 | 适用场景 |
|------|------|---------|
| 功能测试 | 验证功能是否符合需求 | 所有功能 |
| 接口测试 | 验证API接口 | 后端接口 |
| 安全测试 | 验证安全防护 | 敏感功能 |
| 性能测试 | 验证性能指标 | 高并发场景 |
| 兼容性测试 | 验证兼容性 | 多端适配 |
| 路由测试 | 验证页面跳转 | 前端路由 |
| 联动测试 | 验证跨页面联动 | 多页面业务 |
