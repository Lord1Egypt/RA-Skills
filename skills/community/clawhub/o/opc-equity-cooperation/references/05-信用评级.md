# 信用评级：5级信用体系、升降级规则

> 本文件详细阐述OPC联盟的信用评级体系、信用分计算规则、升降级机制及其对收益分配的影响。

---

## 一、信用评级体系概述

### 1.1 设计原则

> **重要说明**：不使用国家信用评级的名称和权限，建立联盟内部的"贡献履约信用体系"，只在联盟内部有效，对外不构成任何评级承诺。

| 设计原则 | 说明 |
|---------|------|
| **内部性** | 仅在联盟内部有效，不对外公开 |
| **动态性** | 信用分实时变化，反映最新履约情况 |
| **透明性** | 信用分变化有据可查，全员可查 |
| **激励性** | 高信用有奖励，低信用有约束 |

### 1.2 信用等级体系（5级制）

| 等级 | 名称 | 信用分值 | 标识颜色 | 权益说明 |
|------|------|---------|---------|---------|
| **S级** | 核心贡献者 | 90-100分 | 🌟金色 | 优先获得新项目份额、可担任监事/合规长老 |
| **A级** | 可靠伙伴 | 80-89分 | 🟢绿色 | 正常参与所有项目、享有提案权 |
| **B级** | 普通成员 | 70-79分 | 🔵蓝色 | 正常参与项目、部分高收益项目需担保 |
| **C级** | 观察期 | 60-69分 | 🟡黄色 | 限制参与高权重任务、需定期汇报 |
| **D级** | 警示 | <60分 | 🔴红色 | 暂停新合作、强制合规培训 |

## 二、信用分计算规则

### 2.1 基础分设置

| 情况 | 初始分 |
|------|-------|
| 新加入成员 | 75分（B级） |
| 从其他联盟转入（有历史记录） | 参考历史分，最高不超过80分 |
| 被动退出后重新加入 | 上次离场分 × 0.8 |

### 2.2 加分项（每次履约自动加分）

| 行为 | 加分 | 备注 |
|------|------|------|
| **按时按质完成任务** | +5分/次 | 每次里程碑验收 |
| **提前完成任务** | +8分/次 | 需盟主确认 |
| **主动承担额外工作** | +3-10分 | 盟主+监事确认 |
| **帮助解决争议** | +2-5分 | 视贡献大小 |
| **连续3个项目无争议** | +10分 | 一次性奖励 |
| **推荐优质资源/成员** | +5-15分 | 资源落地后确认 |
| **参与联盟公共事务** | +1-3分/次 | 如担任会议记录 |
| **积极培训新人** | +3分/人 | 新人通过考核后 |
| **连续6个月无扣分记录** | +15分 | 年度奖励 |

### 2.3 减分项（触发自动扣分）

| 行为 | 扣分 | 备注 |
|------|------|------|
| **任务延期交付** | -5分/天 | 最多扣30分 |
| **质量不达标返工** | -10分/次 | 需返工的情况 |
| **不配合对账/审计** | -15分/次 | 监事长老认定 |
| **违反议事规则** | -5-20分 | 监事长老判定 |
| **未经批准缺席会议** | -3分/次 | 每月最多扣10分 |
| **泄露联盟商业秘密** | -30分/次 | 视情节严重 |
| **恶意违约/毁约** | -50分/次 | 直接降至D级 |
| **误导客户/合作方** | -20分/次 | 合规长老认定 |
| **损害联盟声誉** | -10-30分 | 视情节严重 |
| **与成员发生冲突** | -5分/次 | 视情节严重 |

### 2.4 特别加减分

| 情况 | 加/减分 | 说明 |
|------|--------|------|
| 被选为监事长老 | +10分（任期内） | 履职奖励 |
| 担任培训讲师 | +5分/次 | 知识分享 |
| 提出重大改进建议被采纳 | +10-20分 | 贡献奖励 |
| 被有效投诉 | -10分/次 | 客户/合作方投诉 |
| 被除名 | 直接降至D级 | 最严重处罚 |

## 三、信用系数

### 3.1 信用系数对照表

| 等级 | 信用分值 | 信用系数 | 影响说明 |
|------|---------|---------|---------|
| **S级** | 90-100分 | 1.2倍 | 高质量履约奖励 |
| **A级** | 80-89分 | 1.0倍 | 正常分配 |
| **B级** | 70-79分 | 0.9倍 | 轻微影响 |
| **C级** | 60-69分 | 0.7倍 | 明显影响 |
| **D级** | <60分 | 0.5倍 | 严重限制 |

### 3.2 信用系数计算公式

```
实际分配金额 = 基础分配金额 × 信用系数

示例：
- A方基础分配10000元，信用系数1.2，实际到手12000元
- B方基础分配10000元，信用系数0.9，实际到手9000元
- C方基础分配10000元，信用系数0.5，实际到手5000元
```

## 四、升降级机制

### 4.1 升级路径

| 当前等级 | 升级条件 | 升级后等级 |
|---------|---------|-----------|
| D级 | 整改期3个月，通过考核 | C级 |
| C级 | 连续2个项目保持B级以上且无扣分记录 | B级 |
| B级 | 连续2个项目保持A级且无扣分记录 | A级 |
| A级 | 连续3个项目保持A级且累计贡献≥30分 | S级 |

### 4.2 降级触发

| 触发条件 | 降级幅度 |
|---------|---------|
| 扣分后分值跌破阈值 | 自动降级 |
| 恶意违约/毁约 | 直接降至D级 |
| 泄露商业秘密 | 直接降至D级 |
| 被除名 | 直接降至D级 |

### 4.3 升降级评定流程

```
1. 【系统自动计算】每月底自动计算信用分
        ↓
2. 【等级判定】根据分值确定当前等级
        ↓
3. 【升级公示】符合升级条件的，公示7天
        ↓
4. 【异议处理】如有异议，3天内提出
        ↓
5. 【生效确认】公示无异议后，生效
        ↓
6. 【上链存证】变更记录上链
```

## 五、信用权益体系

### 5.1 各等级权益

| 权益类型 | S级 | A级 | B级 | C级 | D级 |
|---------|-----|-----|-----|-----|-----|
| 参与所有项目 | ✓ | ✓ | ✓ | 部分 | ✗ |
| 提案权 | ✓ | ✓ | ✓ | ✗ | ✗ |
| 担任监事/合规长老 | ✓ | ✓ | ✗ | ✗ | ✗ |
| 优先获得新项目 | ✓ | ✓ | ✗ | ✗ | ✗ |
| 高收益项目参与 | ✓ | ✓ | 需担保 | ✗ | ✗ |
| 优先结算权 | ✓ | ✓ | ✗ | ✗ | ✗ |
| 培训资源优先 | ✓ | ✓ | ✓ | ✗ | ✗ |
| 联盟活动参与 | ✓ | ✓ | ✓ | 需审批 | ✗ |

### 5.2 信用承诺与约束

| 等级 | 承诺 | 约束 |
|------|------|------|
| **S级** | 持续高质量履约 | 可担任关键角色 |
| **A级** | 按时按质完成 | 正常参与 |
| **B级** | 基本履约 | 部分项目需担保 |
| **C级** | 改善履约 | 定期汇报、限制参与 |
| **D级** | 整改承诺 | 暂停新合作、强制培训 |

## 七、信用修复机制

### 7.1 D级成员修复路径

```
D级成员修复流程：

1. 【强制培训】参加合规培训，完成学习
        ↓
2. 【整改承诺】签署整改承诺书
        ↓
3. 【观察期】3个月观察期
        ↓
4. 【考核通过】完成至少1个小项目
        ↓
5. 【升级C级】通过考核后升级至C级
        ↓
6. 【持续表现】继续保持履约，争取升级
```

### 7.2 信用修复条件

| 修复类型 | 条件 |
|---------|------|
| **自动修复** | 连续6个月无扣分，加15分 |
| **主动修复** | 完成联盟指定任务，最高加10分 |
| **立功修复** | 重大贡献可申请额外加分 |
