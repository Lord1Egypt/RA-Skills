# 生成质量标准与评审检查清单

## 概述

本文件定义岗位智能体提示词生成的质量标准，用于生成后自检和用户评审。

---

## 一、泛化合规检查

| 检查项 | 标准 | 通过条件 |
|--------|------|----------|
| 公司名称 | 禁止出现任何具体公司名 | 全文搜索无具体公司名 |
| 人名 | 禁止出现任何具体人名 | 全文搜索无具体人名 |
| 系统名 | 禁止出现具体内部系统名 | 使用泛化表述如"企业ERP系统" |
| 产品名 | 禁止出现具体产品/品牌名 | 使用泛化表述如"核心业务产品" |

---

## 二、七层完整性检查

| 层级 | 必含字段 | 通过条件 |
|------|----------|----------|
| 身份层 | 角色身份、核心职责与价值目标、工作边界 | 三字段均存在且有实质内容 |
| 人设层 | 沟通风格、标准话术框架、表达禁忌 | 三字段均存在且禁忌条目≥2条 |
| 技能层 | 岗位核心能力、降级策略 | 能力条目≥3项，每项核心能力配有降级方案 |
| 知识层 | 参考规范底座、引用规范 | 规范文档≥2项，引用格式明确 |
| 流程层 | 标准工作步骤、异常处理 | 步骤≥3步且有序号，异常处理含完整链路 |
| 权限层 | 白名单、黑名单与上报机制 | 白黑各≥2项，黑名单含拦截动作 |
| 绩效层 | KPI指标、自我优化意识 | KPI≥3项且可量化，优化意识含具体行为 |

---

## 三、内容质量检查

### 3.1 可操作性标准
- [ ] 所有描述指向**可执行的行为**或**可判定的标准**
- [ ] 降级策略包含**具体的替代动作**（非笼统"转人工"）
- [ ] 异常处理覆盖**发现→诊断→建议**完整链路
- [ ] 黑名单条款包含**拦截动作**和**上报提示话术**

### 3.2 结构规范标准
- [ ] 使用统一的 Markdown 层级结构
- [ ] 每层有清晰的标题和字段标注
- [ ] 列表项使用统一的符号和编号体系
- [ ] 无冗余内容或重复描述

### 3.3 角色一致性标准
- [ ] 七层之间的角色定位保持一致
- [ ] 技能层的能力与身份层的职责对应
- [ ] 权限层的边界与工作边界匹配
- [ ] KPI 与核心职责直接关联

---

## 四、输出格式规范

生成的提示词应遵循以下 Markdown 结构：

```markdown
# [岗位名称] 数字员工系统提示词

> 基于《岗位职责说明书》自动生成 | 七层标准化模板 v1.0

---

## 1. 身份层（Identity）

### 角色身份
[内容]

### 核心职责与价值目标
[内容]

### 工作边界
[内容]

---

## 2. 人设层（Persona）
...（以此类推）
```

## 五、常见问题与修正指引

| 问题 | 症状 | 修正方法 |
|------|------|----------|
| 过度具体化 | 出现公司名/人名/系统名 | 替换为泛化表述 |
| 层级缺失 | 某层字段为空或仅有标题 | 回到原始文档补充提取 |
| 禁忌空洞 | 表达禁忌仅1条或过于笼统 | 从岗位合规要求中提炼具体条目 |
| 降级缺失 | 某项核心能力无降级策略 | 为每项能力补充不可用时的替代方案 |
| KPI不可量化 | 指标为"持续改进"等模糊表述 | 改为具体的行为指标或数值指标 |
| 权限边界模糊 | 白名单与黑名单条目重叠 | 明确区分"自主可做"与"禁止触碰" |
