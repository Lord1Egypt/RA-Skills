## ✨ 核心功能

### 1️⃣ 故事框架构思
从零生成完整大纲，包括：
- 基本信息（类型、主题、读者、篇幅）
- 核心设定（世界观、人物、冲突）
- 故事结构（三幕式/英雄之旅/七点法）
- 章节细纲（每章梗概）
- 人物成长弧（从XX到XX的变化）
- 伏笔与回收计划

**触发词**：`写小说`、`创作故事`、`构思情节`、`人物设定`

---

### 2️⃣ 续写/扩写
智能接续，风格完全一致：
- 分析上文风格（语言、节奏、情绪）
- 生成续写内容（衔接自然）
- 提供续写思路说明
- 支持多种策略（衔接、插入、分支）

**触发词**：`续写`、`扩写`、`继续写`、`接下来呢`

---

### 3️⃣ 文字润色
让文章更流畅自然：
- 轻度润色（保留原味，仅修复硬伤）
- 中度润色（平衡优化，提升可读性）
- 重度润色（深度改写，调整结构）
- 风格转换（古龙风、金庸风、网络小说风等）

**触发词**：`润色`、`修改文章`、`改写`、`改成XX风格`

---

### 4️⃣ 逻辑一致性检查
自动发现前后矛盾：
- 人物一致性（性格、外貌、背景）
- 时间线一致性（时间流逝、时间跳跃）
- 空间一致性（场景切换、地理位置）
- 因果一致性（伏笔回收、因果关系）

**触发词**：`逻辑检查`、`前后矛盾`、`人物不一致`、`时间线错误`

## 🔧 辅助脚本使用

```bash

# 检查多个文件
python3 ~/.qclaw/skills/novel-writer/scripts/check_consistency.py chapter1.txt chapter2.txt chapter3.txt
```

**输出**：人物、时间线、物品一致性检查报告

---

### 脚本2：大纲完整性验证
```bash
python3 ~/.qclaw/skills/novel-writer/scripts/validate_outline.py my-outline.md
```

**输出**：大纲完整性评分和改进建议

## ⚠️ 注意事项

### 1. 脚本准确度限制
- `check_consistency.py` 使用简单正则实现，人物提取准确度有限
- 建议：将输出结果作为线索，人工逐章检查

### 2. 时间线检查需要人工核对
- 脚本只能提取时间表达，无法判断顺序是否合理
- 建议：人工核对章节顺序和时间流逝

### 3. 物品检查可能有误报
- 脚本可能将对话中的内容误判为物品
- 建议：人工核对物品是否凭空出现/消失
