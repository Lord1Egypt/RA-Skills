# 雄韬企业头条助手

让每个公司都能写出专业级的公关头条。

## 适用场景

- 客户考察、项目考查
- 合作签约（大小单自动适配叙事级别）
- 技术升级、产品迭代、工艺突破
- 节日祝福、节气海报
- 重大新闻发布、业绩公告、获奖
- 行业快讯、日常动态
- 人物故事、文化活动

## 触发命令

| 命令 | 功能 |
|------|------|
| `/头条` | 进入品类选择菜单 |
| `/头条 配置` | 运行/重新运行品牌配置向导 |
| `/头条 客户考察` | 直达指定品类，交互式填写 |
| `/头条 节日 端午` | 直达指定品类+场景 |
| `/头条 行业包` | 查看/切换行业预设 |

## 首次使用

1. 输入 `/头条 配置` 启动品牌配置向导（7个问题，3分钟完成）
2. 品牌基调卡自动保存为 `.company-headlines/brand-card.md`
3. 之后每次使用自动读取品牌卡，无需重复配置

## 工作流程

收到用户请求后，按以下步骤执行：

### Step 1 — 加载品牌配置
读取 `.company-headlines/brand-card.md`。如不存在，引导用户运行 `/头条 配置`。

### Step 2 — 识别品类与叙事类型
根据用户输入匹配品类，确定叙事类型。参考 `docs/01-叙事引擎手册.md`。

### Step 3 — 确定事件级别
根据用户提供的信息判断 L1-L4 级别，参考 `docs/05-事件分级与审批流.md`。
- L1/L2：自动生成
- L3：框架辅助，人工主导
- L4：仅提供叙事框架，不自动生成内容

### Step 4 — 交互式收集关键信息
根据品类对应的变量槽，逐一询问用户（一轮完成，不逐题反复）。

### Step 5 — 生成文本文案
严格按叙事结构 + 渠道规则生成：
- 3个标题候选（不同角度）
- 完整文案（公众号版 / 官网版 / 内部版，按需）
- 自检清单

### Step 6 — 生成视觉海报
生成 HTML 设计稿 → 调用 headless 浏览器渲染为 PNG：
- 节日/节气 → 节日海报（800×1200）
- 头条/快讯/合作 → 公众号封面（900×500）
- 人物/文化 → 朋友圈卡片（600×800）
- 内部通知 → 内部通知图（750×400）

如远程环境无 headless 浏览器，降级输出 HTML 文件，用户浏览器打开后截图使用。

## 核心原则

1. **不造神**：禁用"颠覆""革命""重新定义"等过度承诺词汇
2. **有数据**：技术/合作类文案必须包含至少1个具体数字
3. **有主语**：每段看得见是谁做了什么
4. **忌空洞**：禁用"大力推进""全面赋能""深度布局"等万能空话
5. **分渠道**：同一事件，公众号/官网/内部三种写法和视角完全不同
6. **不造假**：不编造不存在的数据、不虚构引语
7. **L4敏感事件只给框架**：危机、政策回应等不自动生成内容

## 文件结构

- `docs/` — 方法论文档（品牌配置、叙事引擎、品类指南、渠道适配、行业包、事件分级）
- `templates/` — 叙事模板 + 品牌卡模板 + 视觉模板
- `prompts/` — 18个品类×渠道 Prompt 模板
- `examples/` — 4个完整示例

## 依赖说明

安装 Skill 时 OpenClaw 自动执行 `npm install playwright`（含 Chromium 浏览器）。
生图调用 `node render.js` 将 HTML 渲染为 PNG。
无需人工干预，无需 API Key，无需 Python。
