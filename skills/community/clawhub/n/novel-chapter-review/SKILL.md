# 逐章评价体系

## 使用场景

对已完结/已写完的小说，逐章进行质量审计，发现问题，及时修改。

## 评价流程

每章按以下顺序执行：**通读 → 六维评分 → 爽点审计 → 问题定位 → 输出报告 → 执行修改**

---

## 第一步：通读

逐章读取正文，不做评价，先建立整体印象：
- 这一章讲了什么？
- 读完什么感觉？（燃？闷？想跳？想翻页？）
- 最亮眼的一句话/一个场景是什么？

---

## 第二步：六维评分

每章打6个维度，每个维度 1-5 分：

| 维度 | 1分 | 3分 | 5分 | 权重 |
|------|-----|-----|-----|------|
| **爽感** | 闷，想跳过 | 有意思但不够 | 燃炸，忍不住重看 | 25% |
| **节奏** | 拖沓或仓促 | 基本流畅 | 张弛有度，停不下来 | 15% |
| **角色** | 工具人/人设崩 | 行为合理 | 行动有动机，对话有性格 | 15% |
| **文笔** | AI味重/语病多 | 通顺但不出彩 | 有记忆点的好句子 | 10% |
| **信息密度** | 大段废话/水字数 | 有干货但夹杂水分 | 字字有用，没有废句 | 15% |
| **翻页率** | 看完不想翻下一章 | 有兴趣继续 | 钩子够硬，必须翻 | 20% |

### 信息密度评判标准

- **高密度**：每句话都在推动剧情、展示角色、埋伏笔或制造氛围
- **中密度**：大部分有用，但有小段可删减
- **低密度**：存在明显水字数：重复描写、多余心理独白、不必要的场景过渡

### 翻页率评判标准

检查章末是否有钩子：
- **强钩子**：悬念/危机/反转/新信息揭示 → 读者必须翻
- **弱钩子**：情绪余韵/伏笔暗示 → 读者有动力翻
- **无钩子**：完整收束，没有悬念 → 读者可以放下

### 计算方式

```
章节总分 = 爽感×0.25 + 节奏×0.15 + 角色×0.15 + 文笔×0.10 + 信息密度×0.15 + 翻页率×0.20

满分5.0 | 及格3.0 | 优秀4.0
```

### 等级判定

| 总分 | 等级 | 处理方式 |
|------|------|---------|
| 4.5-5.0 | S | 保留，可作范文 |
| 4.0-4.4 | A | 微调即可 |
| 3.5-3.9 | B | 有优化空间，列出问题清单 |
| 3.0-3.4 | C | 需要重点修改 |
| < 3.0 | D | 建议重写或大幅删改 |

---

## 第三步：爽点审计（核心）

每章必须做爽点盘点。

### 爽点类型

| 类型 | 说明 | 示例 |
|------|------|------|
| **碾压** | 主角实力碾压对手 | 一招秒杀、降维打击 |
| **打脸** | 被看不起→实力证明 | "你算什么东西" → 碾压 |
| **反转** | 预期被颠覆 | 以为是弱者→是大佬 |
| **收获** | 获得重要资源/突破 | 升级、获得宝物、领悟 |
| **装逼** | 不经意间展露实力 | 随手一招满座震惊 |
| **复仇** | 对反派出手 | 等了10章的复仇 |
| **揭秘** | 真相大白 | 身份揭露、阴谋揭开 |

### 爽点三要素（缺一个就不爽）

每个爽点必须检查三个前置条件：

| 要素 | 含义 | 检查 |
|------|------|------|
| **压抑** | 爽之前有足够积累 | 读者是否已经被压抑了？压抑够不够久？ |
| **反差** | 前后反差够大 | 弱→强/小人物→大场面/被看不起→碾压 |
| **利落** | 执行干脆不拖泥带水 | 碾压不要啰嗦，打脸不要解释 |

> 常见失误：有碾压但没铺垫（一路平推没感觉），有反转但解释太多（爽感被稀释）

### 审计规则

1. **统计本章爽点数量**（0个/1个/2个+）
2. **标注爽点位置**（"第X段，主角做了XX"）
3. **评估爽点三要素**：
   - 压抑是否到位：✅/❌ + 说明
   - 反差是否够大：✅/❌ + 说明
   - 执行是否利落：✅/❌ + 说明
4. **综合爽点质量**：三要素全✅ = 满分，缺一个扣一分

### 爽点密度标准

| 间隔 | 判定 | 建议 |
|------|------|------|
| 连续3+章无爽点 | 🔴 危险 | 必须加爽点或删减拖沓内容 |
| 连续2章无爽点 | 🟡 警告 | 检查是否有足够铺垫 |
| 每1-2章有爽点 | 🟢 健康 | 保持 |
| 每章2+个爽点 | ⚡ 高能 | 注意不要审美疲劳 |

---

## 第四步：问题定位

逐章扫描以下问题类型，标注严重程度。

### P0 硬伤（必须修）

| 类型 | 检查方法 | 示例 |
|------|---------|------|
| 时间线矛盾 | grep时间词/年龄/季节 | "三十年前"变"前三十年" |
| 名字错误 | grep角色名全文比对 | "赵桂芬"应为"周桂芬" |
| 设定冲突 | 对照力量体系/世界观 | 未到境界使用了超纲能力 |
| 物件来源 | 道具是否已获得 | 没拿到就用了 |
| 逻辑硬伤 | 行为动机是否成立 | 没理由就叛变 |

### P1 软伤（建议修）

| 类型 | 表现 |
|------|------|
| 角色工具人 | 出场只为推动剧情，没有自己的动机 |
| 对话端水 | 角色说话没区分度，像同一个人 |
| 信息倾倒 | 大段设定说明打断叙事（应改为对话/行动揭示） |
| 情绪断裂 | 上一章愤怒→下一章突然没事了 |
| 节奏拖沓 | 一个意思重复了3段 |
| 爽点缺陷 | 有碾压但没铺垫 / 有反转但解释太多 |

### P2 优化（锦上添花）

| 类型 | 表现 |
|------|------|
| AI痕迹 | "不禁"、"然而"、"宛如宿命" |
| 比喻过多 | 一页3个比喻 |
| 形容词堆叠 | "深邃而忧郁的眼睛" |
| 上帝视角 | 作者跳出来总结人生哲理 |
| 西式句式 | 从句套从句，翻译腔 |
| 否定句 | "他没有说话"→"他沉默" |
| 心理锚点 | "他知道"→直接写事实 |

---

## 修改执行

### 修改优先级

```
P0硬伤 > 爽点真空 > 翻页率修复 > 爽点三要素补全 > P1软伤 > 信息密度 > P2优化
```

### 修改方式

- **P0 硬伤**：走变更通道（grep全定位→批量替换→同步记忆文件→验证）
- **爽点补充**：检查前置压抑是否足够，不够先加压抑；确保反差够大、执行利落
- **翻页率修复**：章末加钩子（悬念/危机/新信息）
- **爽点三要素补全**：缺压抑→前文加铺垫；缺反差→拉大差距；缺利落→删多余解释
- **P1 软伤**：逐章修改，改完校对
- **P2 优化**：批量调用 writing-polish + humanizer-zh
- **批量操作**：用 execute_code + Python 脚本，不要用 delegate_task+patch

### 修改后验证

每轮修改后重新评分，确认问题已解决、分数有提升。

---

## 批量审稿工作流

```
1. 确认审稿范围（全书/某卷/某几章）
2. 逐章执行 第一步~第五步
3. 输出全书质量报告（第六步）
4. 老板拍板：哪些要改、哪些跳过
5. 按优先级执行修改
6. 修改后重新审评分，确认提升
```
