# EVW — 工程价值交付（CMMI V3.0 Domain-specific PA）

## PA 基本信息
- **缩写**：EVW
- **英文**：Engineering Value Delivery
- **中文**：工程价值交付
- **所属**：Domain-specific PA（领域特定实践域）
- **适用领域**：硬件工程、软件工程、系统工程
- **新增说明**：V3.0新增实践域，关注工程实践如何交付业务价值

---

## CMMI V3.0 要求要点

### 核心目的
确保工程实践能够交付预期的商业价值。

### 关键实践（Specific Practices）
| 实践编号 | 实践内容 | 说明 |
|---------|---------|------|
| EVW 1.1 | 识别价值需求 | 识别干系人价值和业务价值需求 |
| EVW 1.2 | 定义价值度量 | 定义价值交付的度量指标 |
| EVW 1.3 | 策划价值交付 | 制定价值交付计划 |
| EVW 1.4 | 实施价值交付 | 执行价值交付活动 |
| EVW 2.1 | 验证价值交付 | 验证交付物是否满足价值需求 |
| EVW 2.2 | 管理价值变更 | 管理价值需求变更 |
| EVW 3.1 | 改进价值交付 | 基于反馈优化价值交付实践 |

### 通用实践（Generic Practices）
- GP 1.1 执行特定实践
- GP 2.1 建立组织政策
- GP 2.2 策划过程
- GP 2.3 提供资源
- GP 2.4 分配责任
- GP 2.5 培训人员
- GP 2.6 管理配置
- GP 2.7 识别并记录干系人需求和接口
- GP 2.8 监督和控制过程
- GP 2.9 客观评估符合性
- GP 2.10 高层审查

## 用户文档 → PA 匹配规则

### 高概率匹配（主要PA）
- 文档名含"价值交付"、"Value Delivery"、"业务价值" → **EVW 主要匹配**
- 文档含价值度量指标、价值验证报告 → **EVW 主要匹配**

### 中概率匹配（次要PA）
- 文档含需求规格 → 同时匹配 **RDM**（需求开发与管理）
- 文档含验证记录 → 同时匹配 **VV**（验证与确认）
