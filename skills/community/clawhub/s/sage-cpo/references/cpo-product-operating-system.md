# CPO Product Operating System

当用户讨论产品战略、北极星指标、OKR、路线图、产品团队节奏、连续发现、产品三人组或产品组织如何运转时，加载本文件。

## 1. 产品操作系统

```text
愿景 -> 战略赌注 -> 北极星指标 -> 机会空间 -> 实验 -> 路线图 -> 复盘
```

1-30 人团队的产品操作系统必须轻，但必须真实运转。

## 2. VSE 层级

| 层级 | 时间尺度 | 问题 |
| :--- | :--- | :--- |
| Vision | 5-10 年 | 我们想成为谁，为什么存在？ |
| Strategy | 1-2 年 | 我们押哪些赌注，靠什么赢？ |
| Execution | 季度/月/周 | 具体做什么，如何衡量？ |

## 3. 北极星指标

北极星指标是单一的、能同时体现用户价值和商业价值的核心指标。

判断标准：

- 反映用户获得核心价值。
- 能连接长期业务结果。
- 团队能通过产品动作影响它。
- 不只是收入本身。

## 4. 产品三人组

最小产品发现单元：

- 产品
- 设计
- 工程

即便团队很小，也要让关键执行者共同听用户、看机会、讨论方案，而不是产品写完需求再交给别人执行。

## 5. 双轨节奏

```text
发现轨道：访谈 -> 机会识别 -> 假设测试 -> 方案验证
交付轨道：实现 -> 测试 -> 发布 -> 观测
```

发现轨道至少领先交付轨道 1 个迭代。

## 6. 周节奏

- 周一：回顾北极星指标，确认本周 L 任务。
- 周二到周四：深度工作 + 用户访谈 / 实验。
- 周五：分享用户洞察，更新机会树和路线图。

每两周至少启动一个实验：访谈、假门、原型、手工交付、A/B 或销售验证都可以。

## 7. Now / Next / Later 路线图

- **Now**：本月必须做，已有证据和成功标准。
- **Next**：下季度可能做，方向已验证但细节未定。
- **Later**：机会池，不承诺。

进入 Now 的事项必须有：

- 用户问题
- 证据状态
- owner
- 完成标准
- 成功指标

## 8. 产品战略冲刺

季度策略可用四步：

1. 诊断：用户/业务痛点转成机会集群。
2. 聚焦：选出 3 个以内战略支柱。
3. HMW：把支柱转为“我们如何才能……”问题。
4. 路线图：HMW 进入发现和交付。

## 9. AI 产品判断

AI 产品机会先过三问：

1. 下一个更好的模型出来后，这件事还重要吗？
2. 模型升级后，这个功能会变得更好吗？
3. 竞争对手无法轻易复制什么？

永远先从真实用户痛点出发，不从 AI 能力反推功能。
