# 蒸馏流程详解

## L2/L3 周度蒸馏（每周日 10:00）

### 操作步骤

1. **回顾本周日志**
   - 读取 `memory/YYYY-MM-DD.md` 本周的文件
   - 如果没有 daily 文件，从当前 session 的对话中提取

2. **识别新模式**
   - 用户新的决策风格（行动导向？谨慎型？）
   - 新的沟通偏好（更简洁？更详细？）
   - 新的工作习惯（时间安排？批处理偏好？）
   - 新的约束或期望

3. **对比现有记忆**
   - 打开 MEMORY.md，对比 L2（思维画像）和 L3（行为习惯）
   - 检查是否需要更新、补充、或修正

4. **更新 MEMORY.md**
   - 有发现 → 写入对应段落，标注更新日期
   - 无变化 → 跳过

5. **写日记**
   - 记录蒸馏结果到 MemPalace 日记

### 蒸馏检查清单

- [ ] 本周对话中有无用户明确表达的新偏好？
- [ ] 用户的决策风格是否有变化？
- [ ] 是否有新的工作习惯或节奏？
- [ ] 之前的认知是否有误需要修正？
- [ ] 是否有经验教训需要从"临时"升格为"永久"？

## 月度晋升蒸馏（每月 1 日 10:00）

### 操作步骤

1. **扫描成长箱**
   - 读取 `成长箱/.learnings/叮当_errors.md`
   - 统计同类错误出现次数

2. **判断晋升条件**
   - 同类错误 ≥ 3 次 → 触发晋升
   - < 3 次 → 跳过

3. **执行晋升**
   - 如果是工作规范类错误 → 写入 AGENTS.md
   - 如果是底线规则类错误 → 写入 SOUL.md
   - 在 errors.md 中标记 "✅ 已晋升"

### 晋升分类指南

| 错误类型 | 晋升目标 | 示例 |
|----------|----------|------|
| 工作规范 | AGENTS.md | "写文件前先检查是否存在" |
| 底线规则 | SOUL.md | "绝不自行重启网关" |
| 用户偏好 | MEMORY.md L2/L3 | "老大不喜欢套话" |
| 技术经验 | MEMORY.md 经验教训 | "OverlayFS 写入需要验证" |
