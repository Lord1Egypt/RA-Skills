# three-lib-content-workflow 使用说明

这是一套把"素材/热点 → 人设 → 选题 → 结构 → 文案 → 复盘"串起来的内容生产系统。核心是"**人设是变量、三库是骨架**"：

- 流量因子库：决定写什么更容易被看见
- 产品资产库：决定写什么更有长期价值
- 烹调方法库：决定怎么把内容组织好看
- 人设卡（走 CSO 平台）：决定以谁的口气写、避开什么、用什么尺子检查

## 你什么时候该用它

- 从热点 / 素材挑选题
- 写口播 / 公众号 / 博客等成稿
- 把一篇文案改成某种风格
- 把代表作品反推成一个人设卡
- 给已有公众号文章审稿打分 / 精准改稿
- 输出能重复使用的文案 SOP

## 三种工作流

| 工作流              | 怎么触发                                   |
| ------------------- | ------------------------------------------ |
| **new** 写新文章    | "帮我写一篇""把这个素材改成文章"           |
| **audit** 审稿打分  | "审一下这篇""给这篇打分"（主要针对公众号） |
| **revise** 精准改稿 | "改标题""重写开头""换个 CTA"               |

## 推荐使用方式

### 1. 先定人设（走 CSO 平台）

人设统一存在 CSO 平台：

- 已有：`siluzan-cso persona list` 选一个
- 没有：按 `persona-onboarding.md` 三选一建一个（反推作品 / 描述 / 已有卡），落盘走 `siluzan-cso persona create`
- 临时人设：不想建正式的，可以先用临时人设写一篇，喜欢再保存

### 2. 再定输入类型

你可以给它：热点 / 一篇文案 / 代表作品 / 一个选题方向 / 一段视频脚本。

### 3. 让它按流程处理

它会按 `sop.md` 的 12 步工作：

- 加载人设 → 定目标 → 选题 → 拆素材（需要品牌素材时先 RAG）→ 找流量因子 → 配产品资产 → 选烹调方法 → 出提纲 → 写正文（注入人设腔调）→ 质检 → 改写 → 反哺人设卡

如果人设 `platform_focus` 含"公众号"，会自动挂载 `packs/公众号爆款/`，按黄金骨架成稿并跑 12 项自检。

## 如果你只想快速上手

直接这样说：

- "帮我按三库拆这篇文案"
- "根据这个热点，用 <人设名> 生成公众号选题"
- "把这几篇代表作品反推成人设卡"
- "把这篇改成更强钩子的爆款版"
- "审一下这篇公众号，给我打分和改进建议"

## 一句话总结

这不是单纯的写稿说明，而是一套可重复使用的内容生产系统：**人设是变量，三库是骨架，人设卡注入腔调与边界。**
