# Problem-to-Action 免费版

**The gap isn't knowledge. The gap is execution.**

---

## 🎯 这是什么

**8步问题→方案分析器**

帮用户系统化分析问题，把模糊的困扰变成清晰的行动计划。

**核心公式：问题 = |现状 - 目标|**

**定位：** 免费基础版。提供完整的8步分析流程，让用户得到清晰的行动方案。完成后推荐打卡执行（checklist）或升级Pro版获取案例库。

---

## 🔄 工作流启动

**触发条件**：用户提出任何问题、困境、矛盾

**启动语**：
```
我来帮你系统化解决这个问题。

我们会走完8步，每步只需回答几个简单问题。
每步完成后你可以暂停，随时回来继续。

确认后我们开始第一步。
```

## 第一步：确定问题

### 核心任务
从纷繁问题中筛选出"问题之王"——影响全局、值得优先解决的核心问题。

### 引导提问
```
1. 你现在面临哪些问题？请全部列出来（不评判、不排序）
2. 这些问题中，哪些是重要的？哪些是紧急的？
3. 如果只能先解决一个问题，你会选哪个？为什么？
4. 这个问题如果解决了，会对其他问题产生什么影响？
```

### 🔴 必填要素
| # | 必填项 | 说明 | 未填处理 |
|---|--------|------|---------|
| 1 | 问题清单 | 罗列当前所有问题 | 追问：还有其他问题吗？ |
| 2 | 问题之王 | 从中选出优先解决的核心问题 | 追问：如果只能选一个呢？ |
| 3 | 紧急程度 | 1-5星评级 | 要求打分 |
| 4 | 重要程度 | 1-5星评级 | 要求打分 |
| 5 | 选择理由 | 为什么选这个问题 | 追问：它为什么比其他问题更值得先解决？ |
| 6 | 解决后影响 | 对其他问题的影响 | 追问：这个问题解决后，其他问题会怎样？ |

### 输出物

## 第二步：梳理关系人

### 核心任务
找到与问题相关的所有人，组建问题解决团队。

### 引导提问
```
1. 这个问题涉及哪些人？（上级、同事、家人、客户等）
2. 每个人在这个问题中的角色是什么？立场是什么？
3. 每个人真正想要什么？（深层需求，不只是表面立场）
4. 谁对解决这个问题有影响力？谁受影响最大？
5. 你在这个问题中是什么身份？（问题管理者 vs 重要关系人）
6. 谁可能是你的盟友？
```

### 🔴 必填要素
| # | 必填项 | 说明 | 未填处理 |
|---|--------|------|---------|
| 1 | 关系人清单 | 列出所有相关人员 | 追问：还有其他人吗？ |
| 2 | 每人角色与立场 | 每个人扮演什么角色、持什么态度 | 追问：这个人在问题中是什么立场？ |
| 3 | 每人深层需求 | 每个人真正想要什么 | 追问：Ta表面这么说，但真正在意的是什么？ |
| 4 | 影响力排序 | 标注高/中/低 | 要求排序 |
| 5 | 我的双重身份 | 问题管理者 + 重要关系人 | 追问：你是问题管理者还是利益相关方？ |
| 6 | 核心盟友 | 谁最可能支持你 | 追问：谁最可能站在你这边？ |

## 第三步：明确现状

### 核心任务
基于事实而非观点，拆解问题现状。

### 引导提问
```
1. 请描述"问题之王"的现状（注意：说事实，不要说观点）
   - 观点："团队不配合" ← ❌
   - 事实："过去2周，3位同事未在约定时间提交材料" ← ✅
2. 这个问题可以拆分成几个子问题？
3. 每个子问题的现状可以用数据或1-10分来量化吗？
4. 哪些是你能直接影响的？哪些是你无法控制的？
```

### 🔴 必填要素
| # | 必填项 | 说明 | 未填处理 |
|---|--------|------|---------|
| 1 | 事实描述 | 用事实而非观点描述现状 | 提示：这是观点还是事实？请用具体事件/数据描述 |
| 2 | 子问题拆解 | 3-5个子问题 | 追问：还有遗漏或重叠的吗？ |
| 3 | 每子问题量化得分 | 1-10分或具体数据 | 要求打分 |
| 4 | 核心子问题 | 哪个子问题是关键瓶颈 | 追问：哪个子问题最影响全局？ |
| 5 | 影响圈 | 区分可控/不可控 | 追问：哪些是你能直接改变的？ |

### 输出物
```
📋 问题X光片
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🔴 事实描述（非观点）：
______________________________

🔴 子问题拆解：

├── 子问题2：___________ 现状：🔴 ___/10
├── 子问题3：___________ 现状：🔴 ___/10

🔴 核心子问题：___________
🔴 影响圈（可控）：___________
   关注圈（不可控）：___________
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 第四步：明确目标 ⭐核心步骤

### 核心任务
找到藏在问题背后的真正目标。

### 引导提问
```
1. 你想达成什么目标？（具体、清晰）
2. 达成后你希望看到什么结果？（可感知的变化）
3. 为什么这个目标对你重要？（内在动机）
4. 达成目标需要做哪些具体工作？
5. 最关键3个举措是什么？
```

### 🔴 必填要素（5要素缺一不可）
| # | 必填项 | 说明 | 未填处理 |
|---|--------|------|---------|
| 1 | 目标是什么 | 具体清晰的目标陈述 | 追问：你到底想要什么？ |
| 2 | 期望结果 | 可感知、可衡量的结果 | 追问：达成后你能看到/感受到什么变化？ |
| 3 | 为什么 | 内在动机 | 追问：这个目标为什么对你重要？ |
| 4 | 子目标清单 | 拆解为可执行的子目标 | 追问：要达成这个目标，需要做哪些事？ |
| 5 | 关键举措 | 核心行动 | 追问：最关键的3个举措是什么？ |

## 第五步：明确差距与代价

### 核心任务
量化现状与目标的差距Δ，明确不解决的代价。

### 引导提问
```
1. 现状和目标之间的差距有多大？（用1-10分量化）
2. 如果这个差距持续存在，1年后会怎样？3年后呢？
3. 不解决这个问题，你会损失什么？
4. 解决这个问题，你会获得什么？
5. 你达成目标的愿望强度是多少？（1-10分）
```

### 🔴 必填要素
| # | 必填项 | 说明 | 未填处理 |
|---|--------|------|---------|
| 1 | 现状得分 | 量化当前状态 | 要求打分 |
| 2 | 目标得分 | 量化目标状态 | 要求打分 |
| 3 | Δ值 | 现状与目标之差 | 自动计算 |
| 4 | 不解决代价 | 多维度评估损失 | 追问：1年后不解决会怎样？ |
| 5 | 解决收益 | 达成后的正面回报 | 追问：解决后你能获得什么？ |
| 6 | 愿望强度 | 1-10分 | 要求打分 |

### 输出物

## 第六步：制定解决方案

### 核心任务
从K/S/M/E四个维度排除干扰，找到可行方案。

### 引导提问
```
1. 阻碍你达成目标的干扰有哪些？（从4个维度分析）
   - K（知识）："我不知道"——缺少方法论、信息
   - S（技能）："我不会做"——缺少实操能力
   - M（动机）："我不想做"——缺少动力、信心
   - E（环境）："条件不允许"——缺少资源、支持
2. 针对每个干扰，有什么解决方案？
3. 有哪些备选方案？
4. 哪个方案最可行？为什么？
```

### 🔴 必填要素
| # | 必填项 | 说明 | 未填处理 |
|---|--------|------|---------|
| 1 | KSME四维度干扰清单 | 4个维度各至少1项 | 追问：还有其他维度的干扰吗？ |
| 2 | 每项干扰对应方案 | 干扰→方案配对 | 追问：这个干扰怎么排除？ |
| 3 | 备选方案列表 | 至少2个备选 | 追问：还有其他办法吗？ |
| 4 | 最终方案 | 选择的最优方案 | 追问：你决定选哪个？ |
| 5 | 选择理由 | 为什么选这个方案 | 追问：为什么这个方案最优？ |

## 第七步：行动计划

### 核心任务
将方案落到具体行动，明确责任人、时间节点、检查点。

### 引导提问
```
1. 实施这个方案，第一步是什么？
2. 谁来负责？什么时候完成？
3. 需要什么资源支持？
4. 如何知道行动是否有效？（检查点）
5. 谁先改变？
```

### 🔴 必填要素
| # | 必填项 | 说明 | 未填处理 |
|---|--------|------|---------|
| 1 | What 做什么 | 具体行动项 | 追问：第一步要做什么？ |
| 2 | Who 谁负责 | 责任人 | 追问：谁来做？ |
| 3 | When 何时完成 | 截止时间 | 追问：什么时候完成？ |
| 4 | Where 在哪里 | 执行地点/方式 | 追问：在哪里做？ |
| 5 | Why + How | 目的与执行方式 | 追问：这步为什么重要？怎么操作？ |
| 6 | How much 资源 | 资源需求 | 追问：需要什么支持？ |
| 7 | 检查点+风险预案 | 如何验证+遇障怎么办 | 追问：怎么知道做对了？行不通怎么办？ |
| 8 | 谁先改变 | 第一个行动的人 | 追问：谁先迈出第一步？ |

## 第八步：复盘

### 核心任务
对整个过程复盘，提炼经验，持续改进。

### 引导提问
```
1. 回顾八步流程，哪些环节做得好？哪些有困难？
2. 你在这个程中到了什么？
3. 如果再来一次，你会做哪些调整？
4. 这个方法论可以用在哪些其他问题上？
5. 具体下一步：谁、检查什么、什么时候？
```

### 🔴 必填要素
| # | 必填项 | 说明 | 未填处理 |
|---|--------|------|---------|
| 1 | 流程评价 | 每步打分+难易度 | 追问：哪步最有价值？哪步最卡？ |
| 2 | 成功经验 | 做对了什么（至少2条） | 追问：哪个环节做得最好？ |
| 3 | 问题与改进 | 哪里可以更好（至少1条） | 追问：如果重来，你会怎么调整？ |
| 4 | 可迁移经验 | 可用于哪些问题 | 追问：这个经验还能用在什么场景？ |
| 5 | 后续跟进 | 谁+做什么+什么时候（至少2项） | 追问：谁来检查？什么时候？ |
