# 支付产品对比速查

## 六大支付方式速查

| 支付方式 | 适用场景 | 是否需要openid | 运行环境 |
|----------|----------|----------------|----------|
| **JSAPI支付** | 微信内H5页面（公众号） | ✅ 是 | 微信浏览器 |
| **小程序支付** | 微信小程序内 | ✅ 是 | 微信小程序 |
| **Native支付** | PC网站二维码扫码 | ❌ 否 | PC扫码 |
| **APP支付** | iOS/Android原生App | ❌ 否 | 原生App |
| **H5支付** | 手机浏览器（非微信） | ❌ 否 | 手机浏览器 |
| **付款码支付** | 线下收银台扫码 | ❌ 否 | 扫码枪 |

## 选型决策树

```
你的用户在哪？
├── 微信小程序内 → 小程序支付
├── 微信内打开的H5/公众号 → JSAPI支付
├── PC网站 → Native支付
├── 原生App（iOS/Android）→ APP支付
├── 手机浏览器（非微信）→ H5支付
└── 线下扫码枪/收银台 → 付款码支付
```

## 准入条件

| 主体类型 | JSAPI | APP | H5 | Native | 小程序 | 付款码 |
|----------|-------|-----|-----|--------|--------|--------|
| 企业/事业单位/政府机关 | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ |
| 个体工商户 | ✅ | ✅ | ❌ | ✅ | ✅ | ✅ |
| 小微商户 | ✅ | ❌ | ❌ | ✅ | ✅ | ✅ |

## 下单API与调起方式

| 支付方式 | 下单接口 | 下单产物 | 调起方式 |
|----------|----------|----------|----------|
| JSAPI | `/v3/pay/transactions/jsapi` | prepay_id | WeixinJSBridge |
| 小程序 | `/v3/pay/transactions/jsapi` | prepay_id | wx.requestPayment |
| Native | `/v3/pay/transactions/native` | code_url | 生成二维码 |
| APP | `/v3/pay/transactions/app` | prepay_id | OpenSDK |
| H5 | `/v3/pay/transactions/h5` | h5_url | 跳转浏览器 |
| 付款码 | `/v2/pay/micropay` | 无（一步扣款） | 扫码枪 |

## JSAPI vs 小程序支付

**共同点**：
- 共用同一个下单API
- 都需要openid
- 都需要签名

**区别**：
- JSAPI：微信浏览器内，调起用 `WeixinJSBridge.invoke()`
- 小程序：微信小程序内，调起用 `wx.requestPayment()`
- 获取openid方式不同（网页授权 vs wx.login）

## 常见混淆

### H5支付 vs JSAPI支付
- **H5支付**：用户在微信**外部**浏览器支付，会跳转到微信完成后再返回
- **JSAPI支付**：用户在微信**内部**浏览器直接调起支付

### Native支付 vs 扫码支付
- **Native**：商户PC网站展示二维码，用户手机微信扫码支付
- **JSAPI扫码**：用户扫码后在微信内打开H5页面支付

## 小程序支付专属注意事项

1. ✅ AppID必须已认证（个人小程序不支持支付）
2. ✅ mchid和AppID必须在商户平台绑定
3. ✅ 需要获取用户openid（通过wx.login获取）
4. ✅ 支付目录无需配置（小程序场景特殊）
