# 贡献指南 / Contributing Guide

感谢你对 Session Branch 的关注！欢迎提交 Issue 和 Pull Request。

## 如何贡献

1. Fork 本仓库
2. 创建功能分支：`git checkout -b feature/your-feature`
3. 提交变更：`git commit -m "feat: 描述"`
4. 推送分支：`git push origin feature/your-feature`
5. 创建 Pull Request

## 代码规范

- Markdown 文件使用英文
- README.md 必须中英双语（中文在前，英文在后，用 `---` 分隔）
- 提交信息遵循 Conventional Commits：`feat` / `fix` / `docs` / `refactor`
- 变更必须同步更新 CHANGELOG.md

## 文档规范

- 新增 reference 文件必须在 SKILL.md 中引用
- 模板文件使用 `<placeholder>` 占位符，不包含真实数据
- 个人信息过滤规则见 `references/checklist.md`

## How to Contribute

1. Fork this repository
2. Create a feature branch: `git checkout -b feature/your-feature`
3. Commit your changes: `git commit -m "feat: description"`
4. Push to the branch: `git push origin feature/your-feature`
5. Create a Pull Request

## Code Standards

- Markdown files in English
- README.md must be bilingual (Chinese first, English second, separated by `---`)
- Commit messages follow Conventional Commits: `feat` / `fix` / `docs` / `refactor`
- Changes must update CHANGELOG.md
