## 二、客户价值分析

### 2.1 RFM模型应用

```
【RFM客户分层模型】

RFM定义：
├── R（Recency）：最近一次消费距今天数
├── F（Frequency）：消费频率（时间段内订单数）
└── M（Monetary）：消费金额（时间段内总金额）

计算方法：

┌─────────────────────────────────────────────────────────────────────────────┐
│                        RFM评分标准                                        │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  R（最近消费距今）              F（消费频次）              M（消费金额）    │
│  ────────────────────        ─────────────────        ─────────────────  │
│  1分：>180天                1分：1次                 1分：<$5,000         │
│  2分：90-180天             2分：2次                 2分：$5,000-20,000    │
│  3分：30-90天              3分：3-5次               3分：$20,000-50,000  │
│  4分：<30天               4分：>5次                 4分：>$50,000         │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘

客户分层矩阵：

┌─────────────────────────────────────────────────────────────────────────────┐
│                        客户价值分层                                        │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│            ┌───────────────────┬───────────────────┐                       │
│            │   R≥3, F≥3, M≥3   │   R≥3, F≥3, M<3   │                       │
│            │     价值客户        │    潜力客户        │                       │
│            │   (重要价值)       │   (重要深耕)       │                       │
│            ├───────────────────┼───────────────────┤                       │
│            │   R<3, F≥3, M≥3   │   R<3, F≥3, M<3   │                       │
│            │    维护客户        │    唤醒客户        │                       │
│            ├───────────────────┼───────────────────┤                       │
│            │   R≥3, F<3, M≥3   │   R≥3, F<3, M<3   │                       │
│            │    增长客户        │    新客户          │                       │
│            ├───────────────────┼───────────────────┤                       │
│            │   R<3, F<3, M≥3   │   R<3, F<3, M<3   │                       │
│            │    流失风险客户    │    低价值客户      │                       │
│            └───────────────────┴───────────────────┘                       │
│                      F<3                          F≥3                        │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘

客户分类策略：

| 客户类型 | RFM特征 | 策略 |
|----------|---------|------|
| **重要价值客户** | R≥3, F≥3, M≥3 | VIP维护、专属服务、账期支持 |
| **重要深耕客户** | R≥3, F≥3, M<3 | 提高客单价、新品推荐 |
| **一般价值客户** | R<3, F≥3, M≥3 | 保持联系、防止流失 |
| **潜力客户** | R<3, F≥3, M<3 | 激活消费、提升频次 |
| **增长客户** | R≥3, F<3, M≥3 | 提升频次、培养习惯 |
| **新客户** | R≥3, F<3, M<3 | 培育转化、首次复购 |
| **流失风险客户** | R<3, F<3, M≥3 | 紧急唤醒、专项活动 |
| **低价值客户** | R<3, F<3, M<3 | 降低维护成本 |
```

## 三、商品销售分析

### 3.1 爆款识别标准

```
【爆款商品识别模型】

爆款特征：
├── 销量：月销>500件 或 环比增长>50%
├── 利润：毛利率>20%
├── 复购：同客户重复购买>2次
├── 评分：客户满意度>4.5星
└── 评价：好评率>90%

爆款评分公式：

爆款指数 = 销量得分 × 0.3 + 利润得分 × 0.3 + 复购得分 × 0.2 + 增长得分 × 0.2

其中：
├── 销量得分 = (销量 - 最低销量) ÷ (最高销量 - 最低销量) × 100
├── 利润得分 = (毛利率 - 最低) ÷ (最高 - 最低) × 100
├── 复购得分 = (复购次数 - 最低) ÷ (最高 - 最低) × 100
└── 增长得分 = (增长率 - 最低) ÷ (最高 - 最低) × 100

爆款分级：
├── S级：爆款指数>90
├── A级：爆款指数80-90
├── B级：爆款指数60-80
└── C级：爆款指数<60
```

## 四、渠道效果分析

### 4.1 渠道ROI计算

```
【渠道ROI分析模型】

ROI定义：
ROI = (渠道收入 - 渠道成本) ÷ 渠道成本 × 100%

渠道成本构成：
├── 平台费用（年费+佣金）
├── 广告投放
├── 人员成本（按时间分摊）
├── 运营工具
└── 其他费用

计算示例：

┌─────────────────────────────────────────────────────────────────────────────┐
│                        渠道ROI对比                                        │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  渠道        收入       平台费    广告费    其他     总成本     ROI       │
```

### 4.2 归因模型

```
【客户归因分析】

归因模型选择：

┌─────────────────────────────────────────────────────────────────────────────┐
│                        归因模型对比                                        │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  【首次触点归因】                                                         │
│  ├── 逻辑：100%归因给首次接触渠道                                        │
│  ├── 适用：品牌认知驱动的转化                                            │
│  └── 问题：忽视后续培育                                                   │
│                                                                             │
│  【末次触点归因】                                                         │
│  ├── 逻辑：100%归因给最后一次接触渠道                                    │
│  ├── 适用：决策周期短的商品                                              │
│  └── 问题：忽视前期种草                                                   │
│                                                                             │
│  【线性归因】                                                             │
│  ├── 逻辑：平均分配权重给每个触点                                        │
│  ├── 适用：决策周期中等的转化                                            │
│  └── 问题：忽视渠道重要性差异                                            │
│                                                                             │
│  【时间衰减归因】                                                         │
│  ├── 逻辑：越接近转化的触点权重越高                                      │
│  ├── 适用：决策周期长的商品                                              │
│  └── 推荐：义乌外贸多渠道场景                                             │
│                                                                             │
│  【自定义归因】                                                           │
│  ├── 逻辑：根据业务自定义权重                                            │
│  ├── 适用：有明确营销策略的商家                                          │
│  └── 示例：首次30%+末次40%+中间30%                                      │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```
