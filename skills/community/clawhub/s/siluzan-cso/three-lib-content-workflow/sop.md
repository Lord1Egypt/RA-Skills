# 文案生成 SOP

> **前提**：第 0 步"加载人设"必须已完成（详见 `content-writer.workflow.md`）。没有可用人设卡前不得进入本 SOP。
>
> **人设 = styleGuide 的 6 个章节**（详见 `persona-schema.md`）：`一句话定位` / `目标受众` / `内容 DNA`（开场惯用方式·标志性表达·口头禅·节奏·价值观锚点）/ `推荐用词` / `禁用表达 · 禁忌` / `<平台> 平台适配建议`。平台**没有** `style_tags`、`core_task`、`opening_preferences`、`quality_checks` 这类结构化字段——需要时从章节**推断**，推不出来用库内**通用默认**兜底。本 SOP 引用的"人设"一律指准备阶段已加载的这份 styleGuide。

## 工作流分流

先判断意图再选分支。三种模式**都必须先完成** `content-writer.workflow.md` 准备阶段（读文档、加载人设、读取三库、RAG 按需）。**禁止**不加载人设就当普通聊天改字或直接输出。

| 模式            | 典型触发                                           | 走哪条 |
| --------------- | -------------------------------------------------- | ------ |
| **new** 写新稿  | 「帮我写一篇」「根据热点出稿」                     | 写新稿 |
| **audit** 审稿  | 「审一下这篇」「打分」「给改进建议」               | 审稿   |
| **revise** 改稿 | 「润色」「改改」「优化一下」「帮我改改」「口语化」 | 改稿   |

> 无原文先索要；有原文未指明段落 → **默认全文改稿**。「审一下」→ 只出报告。

## 写新稿（new）

写新稿是把"人设腔调 + 选题 + 三库素材"揉成成稿。下面每一项都给了**要产出什么**和**什么算过**——逐项产出、逐项自检，红的回去补，过了再下一项。

### 锚定人设与目标

**产出**：① 向用户复述 4 行——一句话定位 / 目标受众 / 内容 DNA 关键词 / 本次平台；② 用一句话写死本次目标，格式「对〔谁〕讲〔什么〕，让他〔信什么 / 做什么〕」。
**过的标准**：目标的动作类型（判断 / 方法 / 灵感 / 案例 / 转化）和「一句话定位」一致；不一致必须已取得用户确认才继续。复述有异议 → 回 `persona-onboarding.md` 修正。

### 定选题

**产出**：一句话选题，带**具体角度**（不是"聊聊外贸"这种宽话题），并写出它落在号哪条常讲方向（引用定位 / 受众原句作依据）。
**淘汰规则（命中任一就换题）**：

- 说不出它属于这个号的常讲方向；
- 触碰「禁用表达 / 禁忌」里写明的话题；
- 说不出目标受众此刻的**具体处境**（切身相关性不成立）；
- 反差 / 冲突 / 认知差一个都凑不出。

**附**：判断行业 → 挂"通用因子"或"通用 + 行业因子"包，并写明挂了哪个（推不出行业就只挂通用）。

### 列素材清单

**产出**：按 事实 / 数据 / 案例 / 风险 / 术语 / 场景 / 争议 分类列点，每类至少 1 条或显式写「无」。品牌 / 产品类事实**逐条标来源**（准备阶段 RAG 结果第几条 / 用户提供）。
**过的标准**：核心主张至少有 1 条事实或数据撑着——没有就回去补素材或补一轮 `siluzan-cso rag query`（详见 `references/rag.md`）。**无来源的品牌 / 产品事实不许进正文**，禁止编造、禁止直接粘贴 RAG 原文。

### 选流量因子

**产出**：列出本次要用的因子（趋势 / 情绪 / 认知 / 画面四类**至少凑齐 2 类**），逐个写清是哪条素材触发的。可选维度：反差感 / 冲突 / 焦虑·危机感·机会感 / 稀缺 / 趋势 / 认知差 / 数字冲击 / 对比 / 群体共鸣（TF-0028）。
**TF-0028 群体共鸣（要用就必须做实）**：先产出 4 个**具体词条**（不是抽象描述）——反派（具体到谁）/ 阵营（"我们"是谁）/ 暗号（自己人秒懂的那句话）/ 正当化（读完认同的那句）。凑不出具体词条就别用 TF-0028。

### 配结构

**产出**：从下表选 1 个模板名 + 一句话依据（这个号的内容 DNA / 一句话定位 哪点支持它）+ 开头 / 中段 / 结尾各一句话打法。

| 风格倾向（看内容 DNA / 一句话定位） | 选这些模板                           |
| ----------------------------------- | ------------------------------------ |
| 强判断、立场鲜明、先下结论          | 强观点开场型、风险预警型、趋势分析型 |
| 务实、步骤清晰、可执行              | 问题解法型、清单干货型               |
| 故事感、有画面、慢节奏              | 故事驱动型、案例拆解型               |
| 互动型、引发讨论                    | 评论互动型                           |
| 反差颠覆、打破认知                  | 认知重构型                           |
| 专业权威、知识密度高                | 趋势分析型、认知重构型               |

烹调骨架：开头按「开场惯用方式」、中段按「节奏」、结尾按「价值观锚点 / 标志性表达」；对应章节为空就用通用默认（结论先行 → 现象到本质 → 行动提醒）并提示用户补。公众号用其烹调方法库的黄金骨架。多倾向可组合模板；推不出明显倾向用通用模板并提示。
**过的标准**：选定模板的"必要结构"对应的素材手里都有——缺一块就换模板或回去补素材；长度 / 节奏对得上本次目标和平台。

### 出提纲（硬门：必须用户确认）

**产出**：5 段提纲，每段一句话——开头钩子（按「开场惯用方式」）/ 核心判断 / 事实·案例·数据支撑 / 对受众的启示 / 收口（按腔调与「价值观锚点」）。公众号改用 `packs/公众号爆款/` 黄金骨架（导语 → 对号入座 → 小节 → 写在最后 → CTA）。
**过的标准**：用户确认逻辑后**才能**写正文；未确认不准下笔。

### 写正文（对照注入清单逐条做到）

- **句式**：用了「标志性表达 / 口头禅」里的惯用句式（为空则用中性句式并标"建议补充"）；
- **关键词**：用了「推荐用词」，但没硬塞堆砌；
- **比喻**：「标志性表达」里有标志性比喻就挑 1 个贴本次主题的，别每篇套同一个；没有不强加；
- **禁忌**：全文扫一遍「禁用表达 / 禁忌」，命中即替换；
- **语气节奏**：按「标志性表达·口头禅」（语气）+「节奏」；
- **TF-0028**：把上一步的 4 个词条显式写进正文。

### 质检（逐条打钩，红的必须回改再过一次）

先按号定位提炼 1–2 条**专属标准**（如定位"给判断"→ 加"有没有给出明确判断"），再过通用清单：

```
[ ] 开头 5 秒落在受众处境上，说得出"和我有关"
[ ] 核心判断明确、不骑墙
[ ] 至少 1 处具体事实 / 数据 / 案例
[ ] 中段有推进，不是平铺
[ ] 结尾有一句能记住的话或一个动作
[ ] 全文无「禁用表达 / 禁忌」命中
[ ] 腔调与「内容 DNA」（语气 / 节奏 / 价值观锚点）一致
```

- 用了 **TF-0028 额外 4 项（缺一不可）**：反派具体（非"那些人"）/ 阵营清晰（"我们" vs "他们"边界明确）/ 暗号到位（自己人秒懂的金句·细节 ≥2 处）/ 正当化落地（读完感到"我做的事是对的"）。
- 公众号场景另跑公众号流量因子库 § 六的 12 项自检，过 10/12 才发（远端 `packs/wechat/`，见 `packs/公众号爆款/overview.md`）。

### 收尾：按用途改写 + 反哺人设

- **按发布场景微调**：爆款版（更强钩子 / 冲突）、转化版（更实操 / 落地）、口播版（更短句 / 口语，叠加附录 A）、视频号版（更稳更顺）、标题版（更短更有钩子）。
- **反哺人设（建议）**：写完主动问有没有新句式 / 口头禅、未记录的禁忌、新高频词，分别补到对应章节。CSO 平台只有 `persona create` **没有原地 update**——合并到对应章节生成完整 styleGuide（保持 6 章节）→ `siluzan-cso persona create --name <名称> --style-guide-file <path>` 存为新版本（名称加后缀如"外贸老炮-v2"）→ 告知用户"已存新版本，下次引用新名称"。用户不想新建则只口头建议、不落盘。

## 审稿（audit）

用户已有成稿要评分 / 建议 → **默认只出报告，不改全文**（「审完直接改」才转 revise）。要做的：识别体裁、核对品牌事实（用准备阶段已检索的 RAG 结果，不足再补一轮）→ 按上方质检标准过一遍（公众号加 12 项自检报 X/12）→ 标 **3–5 处**必改点，每条 **位置 → 问题 → 改法示例**（可复制的 1–2 句）。输出**禁止**泄漏三库编码 / SOP 过程，改法示例须符合 persona。

---

## 改稿（revise）

用户给原文 + 改稿意图，对**已有正文**重写表达、对齐人设。

- **范围**：默认全文（润色 / 改改 / 优化 / 口语化 + 未指明段落 → 直接全文改，别反问「改哪段」）；仅当用户明确「只改标题/开头/CTA/第 X 段别动」才局部。
- **怎么改**：拆块（标题 / 开头 / 小节 / 收口）默认全标【改】→ 按目标调三库 + 注入人设腔调（「内容 DNA」语气·节奏·标志性表达 +「推荐用词」，避禁忌），品牌事实改动以准备阶段 RAG 结果为准、不足再补检索 → **保原意 ≠ 保原文**：论点、数据不动，每句换说法，全文改稿段段重写（句式 / 节奏 / 钩子），不是换同义词。
- **禁止敷衍（任一命中不得交付）**：禁止原样输出或只改标点 / 1–2 个词；全文改稿每段须有可见改写，连续 3 句与原文相同 → 该段重写；与原文相同的长串（≥15 字）合计 ≤ 全文 10%（数据 / 专名词 / 用户要求保留句除外）；局部改稿指定块须整段重写。
- **交付**：过质检相关项 + 敷衍检测 → `siluzan-cso workflow validate`（详见 `references/validate-content.md`）→ 改后全文落盘（建议 `-改稿` 后缀）→ **改动说明**（对话里也要给）：范围 + 主要动作 + **至少 3 组**「原句 → 改句」对照，禁止只写「已润色」。

审后改稿、口播改稿分别叠加 audit 清单、口播专项自检。可选反哺人设。

## 附录 A：口播脚本专属方法论（写口播版时叠加）

> 口播与其他平台**共用同一套主库**（已是去人设化的通用骨架）。独有价值在方法论：**同频前提 / 焊点 / 拆解分支 / 开头三要素**。

- **同频是前提**：同频（感同身受的带入感）决定内容能否被看完。标题是同频第一关（对的人有没有点进来），开头是第二关（点进来的人有没有留下来）。带入感靠说准受众真实处境，不靠煽情；每处都问"这个人会觉得你在说我吗"。
- **输入判断**：热点 / 素材 / 选题方向 → 走主流程；已有他人高转化脚本 → 走拆解分支，拆完接回焊点。
- **拆解分支**（用户给源脚本）：提取流量因子（找火的原因 2-3 个，对照流量因子库匹配 TF 编号）→ 只选 1-2 个深拆（拆透 > 拆全，拆方法论不是内容本身）→ 以人设为锚点提炼焊点。**必须**针对所拆点提供至少 3 个细分品类的可套用句子，每句"先说买家处境，再说你做什么"。
- **焊点（口播核心）**：在拆素材 / 找因子 / 配资产之后、出提纲之前，从人设、流量因子、产品资产里挑 **1 到 3 个"钉子"**。焊点 = 人设契合 × 流量钩子 × 产品价值的交叉点，定义为「[人设受众] 关心 [TF 因子触发的情绪] × [能给到的价值]」。焊点类型决定结构：

| 焊点类型       | 对应结构                                     |
| -------------- | -------------------------------------------- |
| 火在"反常识"   | 观点先行 → 拆认知差 → 给可抄句               |
| 火在"强结果感" | 结果开头 → 倒推为什么 → 复刻路径             |
| 火在"场景代入" | 场景还原 → 拆带入感 → 给模板                 |
| 火在"争议立场" | 偏执观点开头 → 拆为什么有人骂有人爱 → 怎么用 |

出提纲前过焊点质检：自然不生硬？价值先给（内容在前、产品在后）？CTA 会不会太硬 / 像纯卖货？有问题回去重选。

- **开头三要素（至少命中 2 个）**：价值呈现（开头就告诉能得到什么）/ 看完即得（承诺即时回报）/ 失去焦虑（不看会错过什么，FOMO）。
- **写作风格**：短句、口语、有节奏可直接念；有判断、有画面；开头 3 秒必须有同频感；偏执性格（有立场、敢说、制造讨论）；默认 800–1200 字；腔调按「内容 DNA」+「推荐用词」注入，命中禁忌立即替换。
- **口播专项自检**（在通用质检外加查）：流量因子是否从库取并记录 TF 编号 / 同频是否真带入受众处境（今天正经历的事，不是泛泛情绪）/ 开头三要素是否命中 ≥2 / 是否只拆 1-2 个点没贪多 / 语言是否够口语有偏执立场 / 拆解类是否附 ≥3 个品类可抄文案。

## 章节消费速查

> 平台人设只有 6 个章节（见 `persona-schema.md`）。下表是各动作**读取哪个章节**或**从哪个章节推断**。

| 动作            | 读取 / 推断的 styleGuide 章节                                                  |
| --------------- | ------------------------------------------------------------------------------ |
| 锚定人设与目标  | 全部 6 章节（复述）+「一句话定位」（定位即核心任务）                           |
| 定选题 / 列素材 | 「一句话定位」+「目标受众」（方向 / 行业）；「禁用表达 / 禁忌」（排除）        |
| 选流量因子      | TF-0028 用「价值观锚点」+「目标受众」实时推导                                  |
| 配结构          | 「内容 DNA」+「一句话定位」（风格倾向）；「开场惯用方式 / 节奏」（结构偏好）   |
| 写正文          | 「标志性表达·口头禅 / 节奏 / 价值观锚点」+「推荐用词」+「禁用表达 / 禁忌」     |
| 质检            | 「价值观锚点」+「一句话定位」提炼专属标准；无则通用清单                        |
| 反哺            | 全部（合并后存为新版本）                                                       |
| audit           | 「价值观锚点 / 一句话定位」+「内容 DNA」+「禁用表达 / 禁忌」（+ 全部质检规则） |
| revise          | 从「内容 DNA」推断开场 / 推进 / 收口                                           |
