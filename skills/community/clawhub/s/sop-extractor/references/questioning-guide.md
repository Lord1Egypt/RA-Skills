# 提问话术库

> 萃取过程中使用的话术模板，按追问深度排列

---

## 一、开场破冰（建立安全感）

```
"今天我们不是考试，是帮你把重复性工作'打包'成AI助手，以后这些事就不用你亲自动手了。
 你只需要跟我聊聊你平时怎么做这些事的，就像跟新同事交接工作一样。"
```

```
"没有标准答案，你觉得怎么做就怎么说。如果我说的不对，你随时纠正我。"
```

---

## 二、场景回放（激活具体记忆）

### 标准开场
```
"能不能给我讲一个最近做[工序名]的具体例子？
 比如上次是什么情况触发的，最后结果是什么？"
```

### 追问细节
```
"那天你第一步做的什么？打开什么系统/表格/文档？"
"你当时先看了什么信息？在哪看的？"
"做完之后交给谁了？还是直接就结束了？"
"那次有没有什么不一样的地方？"
"整个过程花了多长时间？"
```

### 记忆模糊时的追问
```
"那上个月做的那次呢？"
"你手机/电脑里有没有相关的记录/截图/文件？"
"你一般在什么系统里操作这些？我们可以一起看看。"
"你旁边有没有同事也做这个？他们是怎么做的？"
```

---

## 三、步骤拆解（从案例到流程）

### 标准确认句式
```
"我理解下来是这几步，你看对不对：
 Step 1: [描述]
 Step 2: [描述]
 Step 3: [描述]
 有没有漏掉的？顺序对不对？"
```

### 用户说"差不多"时
```
"哪里'差不多'？具体差在哪？"
"差多少？是步骤漏了，还是顺序不对？"
"能不能举个例子说明这个'差不多'？"
```

### 用户说"对"时（深挖一步）
```
"Step 2 你说'检查一下数据'——具体检查哪些字段？怎么判断数据有没有问题？"
"这一步有没有什么格式要求？模板在哪？"
"做完这步之后，有没有什么标记或记录？"
```

---

## 四、决策点挖掘（找隐性判断）

### 标准追问
```
"这一步做完之后，接下来有几种可能？"
"什么情况下走A，什么情况下走B？"
"你判断走A还是走B，看的是什么？"
```

### 深度追问（隐性知识入口）
```
"这一步有没有可能需要返工？什么情况下会返工？"
"你做的时候有没有什么'潜规则'——就是你心里知道但文档上没写的？"
"新人来做这一步，最容易犯什么错？"
"你觉得这一步最容易出错的地方是什么？"
"有没有遇到过'这次不能按常规做'的情况？那次怎么处理的？"
```

### 逆向追问
```
"什么情况下你会换一种做法？"
"有没有什么情况是直接放弃/退回/找领导的？"
"这个流程有没有可能走到一半就停了？什么情况下会停？"
```

---

## 五、边界条件（什么时候不做/找人）

### 触发条件
```
"这个工序什么时候启动？有没有什么前置条件？"
"什么情况下是不能启动的？"
"启动之前需要确认什么信息？"
```

### 跨部门协作
```
"什么情况下你会找其他部门？找谁？怎么找？"
"找他们的时候需要提供什么信息？"
"他们一般多久能回复？不回复怎么办？"
```

### 拒绝/退回标准
```
"有没有什么情况是直接拒绝/退回的？标准是什么？"
"退回的时候怎么写原因？有没有模板？"
"退回之后对方应该怎么做？"
```

---

## 六、质量标准（什么算做好了）

```
"这个工序做完，你怎么判断做得好不好？"
"有没有检查清单？哪怕是你心里的也行。"
"有没有被退回过？因为什么原因？"
"你心里的'及格线'是什么？'优秀'又是什么标准？"
"如果给这个工序的完成质量打分，1-10分，你一般能做到几分？扣分的地方在哪？"
```

---

## 七、极端情况构造（AI主动构造，让用户判断）

### 通用极端情况
```
"如果这个工序做到一半，关键系统/数据出了问题，你会怎么处理？"
"如果同时来了3个紧急的[工序]请求，你怎么排优先级？"
"如果审批人出差/请假了，这个流程怎么办？"
"如果上游给的数据明显有问题，但对方坚持说没问题，你怎么处理？"
```

### 根据工序类型构造

**财务类**：
```
"如果金额刚好卡在审批线附近，多1块钱就升级审批，你怎么处理？"
"如果发票和合同金额不一致，差了几分钱，能过吗？"
```

**人事类**：
```
"如果候选人入职前一天突然说要改薪资条件，流程怎么走？"
"如果离职交接清单上有项东西找不到，怎么处理？"
```

**运营类**：
```
"如果巡检发现问题但不影响当前运营，记录还是立即处理？"
"如果客户投诉的问题你判断不是我们的责任，怎么回复？"
```

**项目管理类**：
```
"如果项目延期但客户没催，你主动汇报还是等客户问？"
"如果需求变更影响了预算，但项目经理说不用走变更流程，你怎么办？"
```

---

## 八、补漏确认（最后兜底）

```
"我再问几个问题，确保没有遗漏：
 - [极端情况A]，你会怎么做？
 - [极端情况B]，你会怎么做？
 - 有没有什么你一直想改进但还没改的地方？
 - 如果AI来做这个工序，你觉得最需要小心的是什么？"
```

## 提问禁忌

| ❌ 不要这样问 | ✅ 应该这样问 |
|-------------|-------------|
| "你的SOP是什么？" | "你平时做这个事情的步骤是怎样的？" |
| "有没有标准流程？" | "能不能讲一个最近做的例子？" |
| "判断标准是什么？" | "你做完之后怎么知道做得好不好？" |
| "边界条件有哪些？" | "什么情况下你会换一种做法？" |
| "这个工序有哪些步骤？" | "你第一步通常做什么？然后呢？" |
| "请描述你的工作流程" | "上次做这个是什么时候？当时怎么开始的？" |
| 一次问3个以上问题 | **一次只问1-2个，短小聚焦** |
| 问题写成一大段话 | **问题用编号列表，每条一句话** |
| 在同一轮里混合多个环节 | **一个环节聊透了再进下一个** |

---

## ⚠️ 实战经验（提问节奏）

**绝对不要一次问超过2个问题。** 实测用户反馈："每次给我仨问题我看不见，想不起来。"

正确节奏：
- 每轮只问 **1个** 主问题，最多加1个辅助追问
- 问题用 **简短单句**，不用长段落
- 用户回答后，**先确认理解**，再问下一个
- 一个环节聊透了，再明确告知进入下一个环节

错误示范：
```
❌ "我想确认几个点：1. JD一般是什么格式？2. 倾向性是谁告诉你的？3. 有没有拒绝的情况？"
```

正确示范：
```
✅ "JD一般是什么格式？"
   [等回答]
   ✅ "补充的招聘倾向性一般是谁告诉你的？"
```

### 信息变量化思维

用户说"看情况"时，追问出"看什么情况"——这些"变量规则"比固定规则更有价值。
