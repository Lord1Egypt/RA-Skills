# 企业 Web / SaaS GUI UAT 标准

本文件定义通用企业 Web UAT 标准。内容必须保持项目中立、隐私安全，不包含任何客户专属信息。

## 一、测试标准

### 1. 测试目标

UAT 的目标是证明 Web 系统在真实业务场景下满足验收要求。它不是截图集合，也不是随机点页面。

完整 UAT 应在适用范围内验证：

1. 来源资料中的验收准则；
2. 核心业务流程和数据生命周期闭环；
3. 角色、权限与数据范围边界；
4. 配置项是否真正生效；
5. 导入、导出、文件、通知、集成和后台任务；
6. 异常、空数据、非法输入、重复提交、超时、无权限等边界；
7. 系统可安全提供的证据、日志、审计轨迹或业务记录。

### 2. 执行前准备

| 类别 | 必要输入 |
|---|---|
| 范围 | 模块、流程、角色、设备、浏览器、排除项 |
| 文档 | 操作手册、PRD、合同/SOW、变更单、接口文档、验收准则 |
| 环境 | 访问地址、版本/构建号、测试组织/空间、测试账号 |
| 数据 | 允许使用的数据类型、清理规则、禁止使用的数据 |
| 风险 | 删除、外发通知、支付、生产配置、法律签署/证书等高风险边界 |

缺少任何关键项时，必须记录缺口及其对验收结论的影响。

### 3. 证据规则

有效证据应能识别：

- 用例编号和准则编号；
- 使用的角色/会话类型，但不暴露凭证；
- 页面 URL 或路由，但不泄露私密 Token；
- 当前页面、菜单或控件状态；
- 执行动作；
- 预期结果与实际结果；
- 截图、DOM、日志、网络观察等证据编号；
- 时间、环境和版本。

无效证据示例：

- 截图来自错误页面或错误角色；
- 截图暴露密钥或个人隐私；
- 接口成功被描述为完整 GUI 成功；
- 无执行记录却标记通过；
- 阻塞用例被计入通过。

## 二、验收标准

### 1. 需求/用例状态

- `Pass`：有充分证据证明符合预期。
- `ConditionalPass`：存在非阻断问题，但有可接受规避方案或修复计划。
- `Fail`：验收准则失败或关键行为错误。
- `Blocked`：因账号、环境、依赖或数据问题无法执行。
- `OutOfScope`：明确不在当前范围内。
- `ToConfirm`：预期行为缺失或不明确，需要确认。

### 2. 缺陷严重级别

- `Blocker`：阻断核心业务、造成严重数据/安全/权限失败，或导致无法验收。
- `Critical`：高影响功能失败，规避方案有限。
- `Major`：重要功能错误，但可能有规避方式。
- `Minor`：低影响的界面、文案、易用性问题。
- `Trivial`：轻微视觉或文档问题。

### 3. 最终结论

- `通过`：所有阻断准则通过，无未解决阻断/严重缺陷，证据完整。
- `有条件通过`：剩余非阻断问题已有接受的规避方案或修复计划。
- `不通过`：核心流程、安全/权限边界、数据完整性或阻断准则失败。
- `阻塞`：证据不足，无法做出验收判断。

## 三、执行标准

### 1. GUI 操作前的 DOM/控件清单

点击页面前，先检查当前页面/Frame 并建立清单：

- 表单与必填字段；
- 输入框、选择器、日期控件、上传控件；
- 主按钮、次按钮、危险按钮；
- 表格、筛选、排序、分页、行操作；
- TAB、弹窗、抽屉、提示；
- 空状态、错误状态、加载状态；
- 链接和导航目标。

控件可标记为：

- `SafeToOperate`：可安全操作；
- `ReadOnly`：只读观察；
- `NeedsConfirmation`：需要用户确认；
- `Destructive`：破坏性操作；
- `ExternalEffect`：可能产生外部影响；
- `Blocked`：当前不可操作。

### 2. 三条执行线

1. **业务流程线**：真实端到端场景和数据生命周期闭环。
2. **模块菜单线**：每个范围内页面的可见安全控件。
3. **角色权限线**：可见性、操作权限、数据范围、直接 URL/API 越权控制。

### 3. 自动化安全

默认禁止：

- 支付、退款、结算；
- 删除或修改真实数据；
- 批量通知；
- 发布、停用或覆盖生产配置；
- 具有法律效力的签章、证书、授权操作；
- 影响真实用户或外部系统的任务；
- 上传涉密文件。

优先使用安全测试数据和可回滚操作。需要清理时必须记录并验证。

## 四、报告标准

最终报告必须支撑验收决策，应包含：

1. 验收总览与最终建议；
2. 范围、环境、角色、假设、排除项；
3. 验收准则清单；
4. RTM 需求追踪矩阵；
5. UAT 用例清单与执行汇总；
6. 业务流程结果；
7. 模块/菜单结果；
8. 角色权限/数据范围结果；
9. 接口/后台/异步/集成补充验证；
10. 正式缺陷；
11. 阻塞/待确认/范围外事项；
12. 文档与系统差异；
13. 风险与遗留问题；
14. 证据索引；
15. 签署确认页。

## 五、最终审核清单

交付前检查：

- 不包含密钥、Cookie、Token、密码、私有 URL、浏览器存储文件、个人本机路径或客户涉密数据；
- 所有准则均有 RTM 映射；
- 所有通过/失败/阻塞状态一致；
- 每个问题都有可读复现步骤；
- 证据归属正确；
- 接口/后台验证与 GUI 验证明确分开；
- 阻塞项和待确认项没有被隐藏；
- 最终建议符合验收标准；
- 交付清单列出生成文件，并排除敏感材料。
