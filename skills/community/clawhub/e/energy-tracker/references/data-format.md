# 能量状态数据格式规范

## 概览
本文档定义了能量状态追踪器使用的数据文件格式，用于存储每日能量状态记录和辅助多日趋势分析。

## 数据结构

### 文件格式
- **文件类型**：JSON数组
- **文件路径**：`./energy_data.json`（用户工作目录，相对路径）
- **编码**：UTF-8

### 记录格式
每条记录是一个JSON对象，包含以下字段：

| 字段名 | 类型 | 必需 | 说明 | 示例 |
|--------|------|------|------|------|
| date | string | 是 | 记录日期，YYYY-MM-DD格式 | "2025-01-15" |
| answer | string | 是 | 用户选择的选项代码 | "A"、"B" 或 "C" |
| interpretation | string | 是 | 单天解读结果文本 | "你处于心理平衡期..." |

### 选项代码映射
- **A**：是（心理平衡期）
- **B**：勉强可以（心理代偿期）
- **C**：不，完全不想动（心理耗竭期）

## 验证规则

### 必需验证
1. **文件格式**：必须是有效的JSON数组
2. **字段完整性**：每条记录必须包含 date、answer、interpretation 三个字段
3. **日期格式**：date 必须符合 YYYY-MM-DD 格式
4. **选项值**：answer 必须是 "A"、"B" 或 "C" 之一

### 可选验证
1. **日期唯一性**：同一天不应有多条记录
2. **日期排序**：记录按日期升序排列（非强制，但建议）

## 常见操作

### 创建新文件
当首次记录时，创建包含一条记录的JSON数组：

```json
[
  {
    "date": "2025-01-15",
    "answer": "A",
    "interpretation": "你处于心理平衡期..."
  }
]
```

### 追加记录
读取现有数组，追加新记录，重新保存：

```python
import json

# 读取现有数据
with open('./energy_data.json', 'r', encoding='utf-8') as f:
    data = json.load(f)

# 追加新记录
new_record = {
    "date": "2025-01-16",
    "answer": "B",
    "interpretation": "你处于心理代偿期..."
}
data.append(new_record)

# 保存
with open('./energy_data.json', 'w', encoding='utf-8') as f:
    json.dump(data, f, ensure_ascii=False, indent=2)
```

## 约束与注意事项

1. **文件位置**：使用相对路径 `./energy_data.json`，确保文件位于用户工作目录
2. **编码要求**：必须使用UTF-8编码，确保中文正常显示
3. **日期格式**：严格遵守 YYYY-MM-DD 格式，避免其他变体
4. **数据持久化**：每次记录后必须保存到文件，避免数据丢失
5. **数据量**：建议至少积累7天数据后再进行趋势分析
6. **备份建议**：重要数据建议定期备份

## 错误处理

常见错误及处理方式：

| 错误类型 | 原因 | 处理建议 |
|----------|------|----------|
| FileNotFoundError | 文件不存在 | 首次记录时创建新文件 |
| JSONDecodeError | JSON格式错误 | 检查JSON语法，确保格式正确 |
| KeyError | 缺少必需字段 | 确保每条记录包含所有必需字段 |
| ValueError | 日期格式错误 | 检查日期是否符合 YYYY-MM-DD 格式 |
| ValueError | 选项值无效 | 确保answer字段值为A、B或C |
