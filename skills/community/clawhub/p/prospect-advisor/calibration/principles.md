# 惠迈有限算力极致化四原则

来源：DeepSeek提炼 + 老王确认采纳 + cn001编码实现

---

## 原则一：动态分配

**核心思想**：简单任务走轻量路径，复杂任务才调用深度推理

**校准框架对应实现**：
| 任务等级 | 校准级别 | 模型选择 | 算力消耗 |
|----------|----------|----------|----------|
| 基础查询 | 快速校准 | deepseek-flash | 低 |
| 标准咨询 | 标准校准 | deepseek-chat | 中 |
| 深度分析 | 深度校准 | deepseek-chat-v4-pro | 高 |

Skill在接收任务时，先通过快速校准判断任务复杂度，自动路由到对应级别。

---

## 原则二：上下文优先

**核心思想**：复用已有结论，避免重复计算

**校准框架对应实现**：
- `memory_get` 优先于 `API调用`：先查本地已有结论
- 校准结果缓存：30分钟冷却期内复用上一次校准报告
- 知识库优先：高频问答直接从知识库读取，不走模型推理
- hk001的使用：仅在需要海外数据/国际分析时才调用，减少跨国请求

---

## 原则三：拒绝无效请求

**核心思想**：对模糊、低质量、边界外的问题，快速给出"不处理"信号

**校准框架对应实现**：
- 输入过滤：快速检测提问质量，低于阈值直接返回"请重新描述"
- 边界检测：需求超出Skill能力范围时，明确告知不可处理而非强行回答
- 节省资源的拒绝模板：
  ```
  快速拒绝（轻量）: "这个问题超出了我的能力范围，建议咨询相关专业人士。"
  温和拒绝（标准）: "我理解你的需求，但这个场景需要更专业的信息，我建议..."
  ```

---

## 原则四：行为层缓存

**核心思想**：高频场景固化为例行响应，不反复推演

**校准框架对应实现**：
- 模板缓存：情感鼓励语、常见问题解答、标准免责声明 → 预编译模板
- 情境-动作映射：将"原则+情境"的常用组合写成固定响应规则
  ```
  情境: "我的分数能上XX大学吗？"
  动作: 查数据 → 匹配位次 → 生成冲稳保 → 模板包装输出（不走完整推理链）
  ```
- 校准结果快照：深度校准的完整报告可序列化存储，下次直接读取

---

## 四原则与校准框架的协同

```
用户请求
    │
    ├─ 原则三（拒绝无效）→ 模糊请求直接返回
    │
    ├─ 原则二（上下文优先）→ 查缓存/记忆/知识库
    │
    ├─ 原则一（动态分配）→ 快速校准定等级
    │   ├─ 轻量 → deepseek-flash
    │   ├─ 中等 → deepseek-chat
    │   └─ 深度 → deepseek-chat-v4-pro
    │
    └─ 原则四（行为层缓存）→ 高频场景模板输出
```

**最终效果**：80%的常规请求在1秒内完成，仅20%的复杂任务消耗深度算力。
