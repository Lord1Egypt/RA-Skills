# Candidate Screening Rubric

**Domain:** Recruiting · Talent Acquisition
**Platforms:** Claude · Codex

## Purpose

Turns a job description and a candidate's materials into a structured, bias-checked screening rubric: must-haves, nice-to-haves, role-fit signals, red flags, and an Advance/Hold/Decline verdict — all with cited evidence. Built for recruiters and hiring managers who need consistent, audit-ready screening that an ATS can attach to a candidate record.

## When to Use

- Reviewing a resume against a specific JD before the first call
- Debriefing a recruiter or hiring-manager screen call into a documented score
- Calibrating the next interview round by writing targeted questions for ambiguous areas
- Producing a defensible rationale for an Advance/Hold/Decline call

## What It Does

1. Collects the JD, candidate materials, screening stage, and decision threshold through one-question-at-a-time intake
2. Decomposes the JD into a confirmed rubric spine (must-haves, nice-to-haves, role-fit signals, red flags) before scoring
3. Scores each rubric item Strong / Met / Partial / Missing / Unknown with a required citation from the supplied materials
4. Runs a bias and fairness check (affinity, halo/horn, recency, demographic inference, pattern-matching, vague rationale)
5. Issues a verdict tied to the user's chosen threshold and writes 3–6 structured next-round interview questions targeted at Partial/Unknown items

## Notes

This skill produces a **screening rubric**, not a hire/no-hire decision. Final calls require a panel and structured interviews. The skill reminds the user to redact direct candidate identifiers before pasting materials and never infers skill or fit from name, photo, school alone, employer brand alone, tenure length alone, or any protected characteristic.
