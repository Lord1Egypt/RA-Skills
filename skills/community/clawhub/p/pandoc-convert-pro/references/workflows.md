# Pandoc Conversion Workflows

## Markdown to DOCX

```bash
bash ${CLAUDE_SKILL_DIR}/scripts/convert.sh README.md -o README.docx --toc
```

For custom Word styling, use a reference document:

```bash
bash ${CLAUDE_SKILL_DIR}/scripts/convert.sh report.md -o report.docx --reference-doc reference.docx
```

Use a `reference.docx` generated by Pandoc as the starting point, edit its styles in Word or LibreOffice, then reuse it.

## Markdown to PDF

```bash
bash ${CLAUDE_SKILL_DIR}/scripts/convert.sh paper.md -o paper.pdf --pdf-engine xelatex --toc --number-sections
```

For Chinese documents, prefer `xelatex`. If no PDF engine is installed, convert to HTML first:

```bash
bash ${CLAUDE_SKILL_DIR}/scripts/convert.sh paper.md -o paper.html --standalone --toc --embed-resources
```

Then open the HTML and print to PDF.

## Markdown with Citations

Document frontmatter can contain citation metadata:

```yaml
---
title: Research Notes
author: Your Name
bibliography: refs.bib
csl: ieee.csl
---
```

Conversion command:

```bash
bash ${CLAUDE_SKILL_DIR}/scripts/convert.sh thesis.md -o thesis.pdf --pdf-engine xelatex --citeproc --bibliography refs.bib --csl ieee.csl --number-sections
```

Citation syntax examples:

```markdown
Prior work discusses this pattern [@doe2024].
Doe [-@doe2024] also describes the limitation.
```

## Markdown to HTML

```bash
bash ${CLAUDE_SKILL_DIR}/scripts/convert.sh guide.md -o guide.html --standalone --toc --css style.css --embed-resources
```

Use `--embed-resources --standalone` when the HTML should be portable and include images/CSS inline.

## Markdown to EPUB

```bash
bash ${CLAUDE_SKILL_DIR}/scripts/convert.sh book.md -o book.epub --toc --css epub.css
```

Validate EPUB with external tools such as `epubcheck` if the document will be distributed publicly.

## DOCX to Markdown

```bash
bash ${CLAUDE_SKILL_DIR}/scripts/convert.sh input.docx -o output.md --extract-media media
```

Use `--extract-media` to preserve embedded images. Keep the generated media folder next to the Markdown file.

## Batch Directory Conversion

```bash
python3 ${CLAUDE_SKILL_DIR}/scripts/batch_convert.py docs --output-dir converted --to docx --retries 2 --report converted/report.md
```

Use `--fail-fast` when every file must succeed before continuing. Use the default tolerant mode for large mixed directories.
