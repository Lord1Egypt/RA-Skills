# Panel Intro Card / 专家出场卡

Format for the panel introduction printed in Step 3 after panel composition is finalized.

## Purpose

A scannable preview of who's about to speak. Lets the PM build context for unfamiliar experts before Round 1 begins. Each expert gets 4 lines max — no more, no less.

## Format rules

- One row per expert
- Each row: name + 1-line credential + 1-line lens + 1-line signature question
- Total card length: 4-6 expert rows, ~25-35 lines
- Use the same separator characters consistently

## Chinese template

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🎤 本次评审专家面板
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

▸ [姓名]
  [一句话身份背景，≤ 30 字]
  视角：[一句话框架名 + 核心维度，≤ 30 字]
  代表问题："[他典型会问的一句话]"

▸ [下一位专家 ...]

[...]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
PRD 类型识别：[新功能 / 迭代优化 / 商业模式 / 体验重构]
本次评审参与人数：[N] 位
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## Voice rules

- Credentials must be specific — book titles, role, company. Not vague ("renowned expert", "thought leader")
- Signature questions must reflect the expert's actual public framework
- No greetings, no "Hi I'm X, excited to be here" — these are not personal introductions
- For fictional roles (P9, Senior PM Director, The Closer), use credentials that match the archetype, not made-up résumés

## Order of experts in the card

1. Core experts first (in the order they were defined in the panel file)
2. Situational experts second
3. The Closer is **not** introduced in the card — he only appears at Step 7 as a surprise authority figure. Introducing him upfront would dilute the impact.
