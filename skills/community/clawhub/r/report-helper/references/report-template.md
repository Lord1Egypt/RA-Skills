# 报告正文结构模板

内部 Markdown 构建稿直接从正文开始。

---

## 先选成品体裁（必做）

写作判断工具只用于内部组织材料，不是固定给读者看的报告目录。正文结构必须先服从用户要的成品体裁。

- 用户要求「发展研究报告 / 产业研究报告 / 政策研究 / 趋势报告 / 2026-2030 报告」时，用 **A. 正式研究报告结构**
- 用户要求「深度研究某产品 / 公司 / 人物 / 概念」，且没有指定正式报告体裁时，用 **B. 叙事型深度研究结构**
- 用户明确指定目录或栏目时，用户指定优先

**硬禁区**：除非用户明确要求展示方法论，否则成品一级标题里不要出现「时间线检查」「同期格局检查」「判断合成」「竞品场景判断」这类内部工作词。

## 写作判断工具（不作为目录）

下面三组问题用于写作前组织材料和生成判断。它们不是正文目录，不能原样变成一级标题。写成品时，把答案放进对应体裁的读者友好章节里。

### 时间线检查

用于还原对象的发展全貌。必须看清：

- 起源追溯
- 诞生节点
- 关键阶段和转折
- 每个节点「为什么这么选」
- 每阶段的核心特征和核心矛盾

不要写成年表流水账。每个关键节点必须回答「为什么改变了格局」。过不了这一问的节点压缩成资料背景。

落位建议：

- A. 正式研究报告：放入「产业发展现状」「发展推演」「结论」
- B. 叙事型深度研究：放入「故事」

### 同期格局检查

用于理解当前时间切面的竞品、同类力量或产业链位置。

竞品场景：

- **A：无直接竞品**：分析为什么没有，未来竞争者可能从哪里出现
- **B：少量竞品 1-2 个**：逐一深入对比
- **C：充分竞品 3+**：选最具代表性的 3-5 个

对比维度：核心差异、用户视角、生态位、趋势判断。不要写成参数表文字版。

落位建议：

- A. 正式研究报告：放入「研究边界与产业链全景」「关键环节研判」「风险与观察指标」
- B. 叙事型深度研究：放入「地图」

### 判断合成

这是写作精华，产出新判断，不是前面材料的缩写。至少回答：

1. 发展历程中哪些决策决定了今天的位置？
2. 主要竞品或同类力量放到时间线上看，路径差异如何导致今天各自的特点？
3. 今天每个核心优势追溯到历史哪个节点？
4. 今天每个核心劣势追溯到哪个历史决策？
5. 未来推演：最可能 / 最危险 / 最乐观三个剧本，每个要有逻辑支撑

落位建议：

- A. 正式研究报告：放入「报告摘要」「关键环节研判」「发展推演」「结论」
- B. 叙事型深度研究：放入「判断」「推演」

### 公司定量检查（公司对象必做）

用于避免公司研究只剩叙事。至少回答：

1. 过去 1-3 年增长曲线如何？收入、ARR、GMV、订单、用户、客户、市场份额等哪个指标最能代表成长性？
2. 增长质量如何？毛利率、利润率、现金流、单位经济或费用结构是否支撑这种增长？
3. 最新融资、现金状况或资本市场表现如何？估值变化与业务增长是否匹配？
4. 与可比公司相比，估值倍数或经营指标处于什么位置？
5. 如果关键财务和估值数据未公开，可以用哪些替代指标观察？这些替代指标的局限是什么？

落位建议：

- A. 正式研究报告：放入「产业发展现状」「关键环节研判」「主要风险与观察指标」
- B. 叙事型深度研究：公司对象在「地图」之后增加「成长性与估值」章节

## A. 正式研究报告结构

适用于产业链、政策、趋势、年度展望、2026-2030 发展研究等对象。

```markdown
# {创作型或正式型标题：必须贴合用户要求的报告体裁}

> 研究时间：YYYY-MM-DD | 所属领域：... | 研究对象类型：产业链/政策/趋势/概念

## 目录

## 关于本报告的信息来源
正文中每条关键事实判断均以上标标注信源等级和编号，如 <sup>a1</sup> 表示 A 级官方/学术一手来源，<sup>b4</sup> 表示 B 级权威媒体原创报道。四级分类：a = 官方/学术一手，政府机构、监管机构、公司年报和原始研究报告均归入此类；b = 权威媒体原创报道；c = 行业媒体/工具商博客；d = 软文/服务商自述/不可验证。编号对应末尾「信息来源与分级」章节，可按编号查找具体出处。本报告目前不使用 c、d 级来源支撑正文判断。

## 一、报告摘要
[3-6 条核心结论，直接回答读者为什么要看这份报告]

## 二、研究边界与产业链全景
[定义研究对象、边界、产业链分层、关键参与者和价值流]

## 三、产业发展现状
[政策、市场、供给、需求、区域、资本、技术现状。用「时间线检查」筛出关键阶段，按读者理解格局所需排序，不按搜集过程排序]

## 四、关键环节研判
[芯片/基础设施/平台/应用/能源/渠道等关键环节。结合「同期格局检查」和「判断合成」，每节都要有判断，不写资料堆叠]

## 五、{年份区间或未来阶段}发展推演
[按阶段、剧本或关键变量推演。可使用「判断合成」里的三剧本问题。没有官方硬目标时必须标注，不写伪精确预测]

## 六、主要风险与观察指标
[风险、约束、验证指标、跟踪信号]

## 七、结论
[收束判断，不重复摘要]

## 八、信息来源与分级

### a · 官方/学术一手（x 条）

### b · 权威媒体原创报道（x 条）

### c · 行业媒体/工具商博客（x 条）

### d · 软文/服务商自述/不可验证（x 条）

> 所有来源访问时间：{YYYY-MM-DD}
```

## 硬规则

- **H1 标题必须匹配成品体裁**。产业发展报告可以正式，产品/人物深度研究可以更有故事感
- **目录之后、第一章正文之前必须有「关于本报告的信息来源」**，声明 `<sup>a1</sup>` 这类编号规则
- **末尾只有「信息来源与分级」一章**，**不要加「方法论说明」**。写作判断工具只服务成品写作，不需要在报告末尾解释
- **公司对象必须有成长性与估值定量分析**。不能只写发展故事、战略转向和竞争格局；缺少公开财务或估值数据时，要写明缺口和替代指标
- 非公司对象删除「成长性与估值」章后，必须顺延重排后续章节编号
- 正文关键事实判断必须带来源上标；末尾来源按 a/b/c/d 分级列出，编号与正文上标一一对应；没有 c 和 d 就不写对应小节
- 工具签名不属于正文内容，由 PDF 生成脚本在最终文件最末尾自动追加：`本报告由 report-helper skill 工具协助生成`、`开源地址：https://github.com/Jiaranbb/report-helper`、`交流和建议可联系作者：嘉然 Jiaran（+v: evadebot）`
- 章节编号用中文数字（一、二、三、四、五、六、七、八），子章节用阿拉伯数字（2.1、2.2）
- 篇幅指标见 SKILL.md 的各步骤说明，总字数 10,000-30,000 字
- 「写作判断工具」只用于组织材料和生成判断，不作为读者目录
