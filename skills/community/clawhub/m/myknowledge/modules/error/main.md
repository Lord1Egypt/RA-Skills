# 错误处理模块

> 💡 **恢复极其简单**：遇到任何问题，说一句"重新初始化 MyKnowledge"即可恢复——
> 无需手动操作任何文件，不会丢失知识库数据。
>
> ⚠️ **为什么不自动重试？** MyKnowledge 是本地离线 Skill，无法区分操作失败是因为
> 权限问题还是文件被占用。自动重试可能导致文件损坏或重复写入。失败后会主动给你修复选项，
> 你只需回复数字确认，安全可靠。
>
> 💬 **主动修复**：操作失败时，不要只报错。主动给出修复选项——
> 用户只需说"修"或"1"，AI 自动执行修复。比"重新初始化"更轻量。

当遇到错误时，按本模块指引处理。

---

## 错误分类与解决

### 1. 创建/写入失败

| 表现 | 怎么办 |
|------|--------|
| 无法创建知识库 | ① 换个文件夹试试（可能没写入权限）② 检查文件夹名有没有特殊符号 ③ 磁盘满了就清理一些空间 |
| 文件写入卡住 | 关掉正在用这个文件的程序（编辑器、终端），再试一次 |
| 无法创建需求 | 先对我说"创建知识库"；如果已有知识库，检查需求名是否重复 |

### 2. 状态更新失败

| 表现 | 怎么办 |
|------|--------|
| 无法修改需求状态 | 说"重新初始化"一键修复，知识库数据不会丢 |

### 3. 配置相关

| 表现 | 怎么办 |
|------|--------|
| 配置文件损坏 | 说"重新初始化"即可恢复，不会丢数据 |
| 平台/安装源不识别 | 按以下格式提示： |
| | ⚠️ 检测到安装方式异常 |
| | |
| | 我需要知道你用的是什么 AI 软件，以及怎么安装的 MyKnowledge： |
| |   • CodeBuddy？WorkBuddy？OpenClaw？还是 Claude？ |
| |   • 是通过应用市场安装的？还是从 GitHub 下载的？ |
| | |
| | 回复格式："我用 XXX，通过 XXX 安装的" |

### 4. 智能任务追踪相关

| 检测灵敏度不符预期 | 对我说"调整检测灵敏度"即可 |

### 5. 项目目录不存在

| 表现 | 怎么办 |
|------|--------|
| 读取项目状态失败 | 项目文件夹可能被手动删了，我帮你从列表清理，下次创建同名项目会重新记录 |

### 6. 极端情况

| 表现 | 怎么办 |
|------|--------|
| 所有操作都失败 | 试试：① 检查磁盘空间 ② 关闭占用 ~/.myknowledge/ 的程序 ③ 重启 AI |
| 以上都不匹配 | 我会列出最可能的原因（权限/空间/占用/版本），回复数字我来修 |

## 快速诊断清单

遇到问题时，先排查：

```
□ 文件夹有写入权限吗？（试试点右键→显示简介→权限）
□ 磁盘满了吗？（Mac：左上角苹果→关于本机→存储空间）
□ 有别的程序在占用文件吗？（关掉编辑器、终端再试）
□ 知识库创建过了吗？（说"项目进展如何"看看）
□ 配置文件被人手改过吗？（说"重新初始化"修复）
```

## 配置恢复

如果 skill-state.yaml 损坏：

```
1. 备份现有配置：
   cp ~/.myknowledge/config/skill-state.yaml ~/.myknowledge/config/skill-state.yaml.bak

2. 删除损坏文件：
   rm ~/.myknowledge/config/skill-state.yaml
   rm ~/.myknowledge/config/install-source

3. 重新触发 Skill，执行首次引导重建配置
```

## 错误响应格式

遇到错误时，按以下格式响应。**关键原则：直接给用户解决方案，不要让用户自己去查文档。主动给出修复选项。**

```
⚠️ 遇到问题：{通俗的错误描述}

原因：{一句话说清楚为什么}

修复选项：
  1 — {修复方案A}（推荐）
  2 — {修复方案B}
  回复数字即可，我来修
```

### AI 自检：回复前确认
- [ ] 用户看完回复后还需要去别的地方查吗？如果需要→把答案直接写进回复
- [ ] 用了技术术语吗？如果用了→换成日常说法
- [ ] 给出了修复选项吗？如果没给→加一个"回复数字即可，我来修"
- [ ] 用户知道下一步做什么吗？如果不知道→给出明确的一步操作
