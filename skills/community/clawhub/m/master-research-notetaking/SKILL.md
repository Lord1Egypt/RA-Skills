# 大师研究笔记工作流（v3 — Nuwa 增强版）

> **核心理念：提取思维框架，而非复制人物。** 捕捉此人如何思考，而非说了什么。

## 适用场景

用户要求研究某位人物（如"研究罗伯特·清崎"、"研究马斯克"等），尤其是当用户希望得到**可复用的思维框架**和**可落地的实操总结**时。

## 流程

### Phase 0：确认范围（30秒）

先用1-2个问题确认：
1. **研究谁？** 确保理解正确
2. **聚焦方向？** 全貌画像 vs 某个维度（如仅决策方式）
3. **用途？** 思维顾问？实操指南？知识库补充？
4. **新研究还是更新？** 此人的 skill/笔记是否已存在？

如果用户只说"研究XXX"无额外信息 → 默认全貌画像 + 实操指南，直接推进。

### Phase 1：六维并行研究（多智能体集群）

借鉴 Nuwa 的 6 维并行研究设计，启动 **6 个 delegate_task** 并行搜索，每个负责一个信息维度：

| 智能体 | 搜索目标 | 关键提取 | 产出 |
|--------|---------|---------|------|
| **① 著作体系** | 书籍、论文、newsletter、系统文章 | 核心论点（出现≥3次=真实信念）、自创术语、阅读清单 | 此人的系统性思想框架 |
| **② 对话思想** | 播客、长视频、AMA、深度访谈 | 压力下的反应、即兴类比、观点变化、拒绝回答的问题 | 即兴思维侧写 |
| **③ 外部视角** | 他人分析、书评、批评、传记 | 外部观察到的模式、批评与争议、同行对比 | 盲点和外部评估 |
| **④ 决策记录** | 重大决策、转折点、争议行动 | 决策背景与逻辑、事后反思、言行一致性 | 真实行为 vs 公开言辞 |
| **⑤ 时间线** | 出生至今 | 关键里程碑、思想转折点、**近12个月动态**（防过时） | 思想演化轨迹 |
| **⑥ 实战应用** | 此方法在具体项目/生意中的落地案例 | 实操步骤、可复制经验、失败教训 | 直接可用的战术参考 |

#### 来源可信度权重

| 来源类型 | 揭示什么 | 权重 |
|---------|---------|:----:|
| 此人自己的著作 | 系统性思维 | ⭐⭐⭐ |
| 长对话/深度访谈 | 即兴思维过程 | ⭐⭐⭐ |
| 实际决策记录 | 真实行为 vs 公开言论 | ⭐⭐⭐ |
| 他人评价/分析 | 外部视角、盲点 | ⭐⭐ |
| 二手转述 | 参考但需验证 | ⭐ |

#### 来源排除清单（禁止使用）
- ❌ 知乎、百度百科/知道、微信公众号文章
- ✅ **接受的中文优质来源：** 得到、36氪、极客公园、晚点、财新、第一财经、虎嗅、少数派等

#### 智能体失败处理
- **超时/无结果：** 不等，继续推进。Phase 2标记"信息不足"
- **信息太少（<10 个可用来源）：** Phase 0告知用户，降低预期，增加诚实边界
- **智能体结果冲突：** 保留矛盾——矛盾是有价值的信号

#### 需要提供的 context 需包含
- 用户基本情况（职业、地点、已有布局）
- 用户已有的相关笔记或认知基础
- 已完成的关联大师（清崎、Codie、马斯克）的认知定位

### Phase 1.5：研究质量检查点

**6 个智能体全部完成后，暂停并展示质量摘要：**

```
┌─────────────────┬───────┬──────────────────────────┐
│ 智能体          │ 来源  │ 关键发现                  │
├─────────────────┼───────┼──────────────────────────┤
│ ① 著作体系     │ 8     │ 核心论点：...             │
│ ② 对话思想     │ 5     │ 观点变化：...             │
│ ③ 外部视角     │ 6     │ 主要批评：...             │
│ ④ 决策记录     │ 4     │ 关键决策：...             │
│ ⑤ 时间线       │ 完整  │ 最新动态：...             │
│ ⑥ 实战应用     │ 3     │ 可复制经验：...           │
├─────────────────┼───────┼──────────────────────────┤
│ 矛盾点          │ 2     │ ①vs③：XXX               │
│ 信息缺口        │ 无    │                          │
└─────────────────┴───────┴──────────────────────────┘
```

用户确认质量 → 进入 Phase 2。
用户想补充某维度 → 追加搜索后再继续。

### Phase 2：框架提取（从信息到心智模型）

#### 2.1 核心心智模型（3-7个）

**三重验证**（借鉴 Nuwa 方法论）：

| 验证 | 标准 | 示例（以清崎为例） |
|:---:|------|-------------------|
| ① 跨域复现 | 同一框架出现在≥2个不同领域 | 现金流象限在投资、职业选择、教育中都出现 |
| ② 有生成力 | 能用此模型推断新问题立场 | 用"资产vs负债"能推断他对任何消费的态度 |
| ③ 有排他性 | 不是所有聪明人都这样想 | 不是所有财富专家都强调"现金流第一" |

- **通过 3 重验证 → 心智模型**（核心产出）
- **通过 1-2 重 → 降级为决策启发式**
- **0 重 → 特定场景言论，不纳入框架**

#### 2.2 决策启发式（5-10条）

此人做判断时使用的快速规则。可表达为"如果X，那么Y"，需有具体案例支撑。

#### 2.3 价值观与反模式

- **核心价值观：** 3-5 个排序后的核心价值
- **反模式：** 此人明确反对的行为/思维
- **内在矛盾：** 价值观之间的张力（这是深度的来源）

#### 2.4 智识谱系（思想地图）

```
影响过TA的人 → 此人的核心思想 → TA影响了谁
```

**处理三种矛盾**（借鉴 Nuwa 矛盾处理原则）：
- **时间性矛盾**（早期说A后来改B）→ 记录演化轨迹，标注「早期」「近期」
- **领域性矛盾**（工作中X生活中Y）→ 分领域记录，不强求统一
- **本质性张力**（既追求自由又重视纪律）→ 明确记录为核心张力

> ❌ 不要选一边忽略另一边 ❌ 不要编调和解释 ❌ 不要假装矛盾不存在

### Phase 3：关联用户已有笔记

```bash
ls /path/to/vault/ | grep -i "关键词"
```

避免重复，已有内容不做重复摘录，只做提炼和深化。

#### 4.2 质量自检清单（写入前必查）

| 检查项 | 通过标准 | 失败信号 |
|-------|---------|---------|
| 心智模型数量 | 3-7个，每个有证据 | <3或>10 |
| 跨域复现 | 每个模型≥2个来源 | 模型只有一个证据 |
| 局限标注 | 每个模型写了失效条件 | 只写优点不写局限 |
| 内在矛盾 | ≥1个矛盾对 | 观点高度一致（太假） |
| 一手来源占比 | >50% | 主要靠二手转述 |
| 对你的含义 | 每概念都有场景翻译 | 只有抽象议论 |
| 立即行动 | 具体到本周能做的事 | "好好学习"这类废话 |
| 参考来源 | 每条信息有出处 | 无来源 |

**全部通过 → 写入。有未通过 → 标记弱点，回溯 Phase 2 补充。**

### 核心原则

- ✅ **每一条都要翻译到用户的实际场景里**
- ✅ **"金句"必须有"对你的含义"这一列**
- ✅ **"立即行动"必须是具体到本周能做的事**
- ✅ **搜索后先展示中间发现（Phase 1.5 检查点）**
- ✅ **每篇报告末尾附带参考来源**
- ✅ **搜索和深读要克制**，够用就停，不穷尽
- ✅ **心智模型要经过三重验证**，不把随口一说当核心思想
- ✅ **保留内在矛盾**，反对"给人加滤镜"
- ✅ **明确标注诚实边界**，标注信息不足的维度
- ❌ 不使用知乎、百度百科、公众号文章
- ❌ 不写百科就能查到的生平事迹
- ❌ 不重复用户已有笔记的内容
- ❌ 不编造调和解释来掩盖矛盾
