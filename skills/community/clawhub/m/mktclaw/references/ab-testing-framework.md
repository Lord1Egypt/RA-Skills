## 一、实验设计核心原则

### 1.1 假设优先(Hypothesis-First)

```
❌ 错误做法: "我们 A/B 测试一下这两个素材吧"
✅ 正确做法: 
   假设: "在 [人群] 中，将 [变量 X] 从 [A] 改为 [B]，
        预期 [指标 Y] 提升 [Z]%，因为 [原因]"
```

### 1.2 实验类型光谱

```
观察性研究 ←─────────────────→ 准实验 ─────────────────→ 随机对照实验
(无因果推断)              (差分/DID/PSM)            (A/B 测试 = RCT)
```

| 实验类型 | 因果强度 | 适用场景 |
|---------|:------:|---------|
| **A/B 测试(RCT)** | ⭐⭐⭐⭐⭐ | 因果明确 / 高流量场景 |
| **A/B/n 测试** | ⭐⭐⭐⭐⭐ | 多方案 PK |
| **多变量测试(MVN)** | ⭐⭐⭐⭐⭐ | 多变量组合优化 |
| **切换实验** | ⭐⭐⭐⭐ | 中低流量 / 时间序列 |
| **差分中差分(DID)** | ⭐⭐⭐ | 政策 / 渠道变更 |
| **倾向得分匹配(PSM)** | ⭐⭐ | 无法随机时 |

## 二、样本量计算

### 2.1 公式

```
n = (Z_{α/2} + Z_{β})² × 2 × σ² / Δ²

其中:
- Z_{α/2}: 显著性水平对应 Z 值(α=0.05 → 1.96)
- Z_{β}: 统计功效对应 Z 值(β=0.2 → 0.84)
- σ: 指标标准差
- Δ: 最小可检测效应(MDE)
```

### 2.2 快速估算表(转化率场景)

**显著性 α=0.05, 功效 β=0.2(80% Power)**

| 基准转化率 | MDE=10% 相对提升 | MDE=20% 相对提升 | MDE=30% 相对提升 | MDE=50% 相对提升 |
|:--------:|:--------------:|:--------------:|:--------------:|:--------------:|
| 1% | 146,000/组 | 37,000/组 | 17,000/组 | 6,300/组 |
| 2% | 72,000/组 | 18,000/组 | 8,200/组 | 3,100/组 |
| 5% | 28,000/组 | 7,100/组 | 3,200/组 | 1,200/组 |
| 10% | 13,000/组 | 3,400/组 | 1,500/组 | 580/组 |
| 20% | 5,900/组 | 1,500/组 | 680/组 | 260/组 |

**使用示例**: 基准转化率 5%, 期望检测 20% 相对提升(从 5% 到 6%), 每组需要约 **7,100** 样本。

### 2.3 Python 样本量计算

```python
import numpy as np
from scipy import stats

def sample_size_cvr(baseline_cvr, mde_relative, alpha=0.05, power=0.8):
    """
    转化率场景样本量计算
    baseline_cvr: 基准转化率(如 0.05)
    mde_relative: 最小相对提升(如 0.2 表示 20% 相对提升)
    """
    p1 = baseline_cvr
    p2 = baseline_cvr * (1 + mde_relative)
    
    z_alpha = stats.norm.ppf(1 - alpha/2)
    z_beta = stats.norm.ppf(power)
    
    # 平均方差
    p_avg = (p1 + p2) / 2
    n = ((z_alpha + z_beta) ** 2 * 2 * p_avg * (1 - p_avg)) / ((p2 - p1) ** 2)
    
    return int(np.ceil(n))

# 示例: 5% 基准转化率, 检测 20% 相对提升
n = sample_size_cvr(0.05, 0.20)
print(f"每组需要样本: {n}")  # ≈ 7,100
```

## 三、统计显著性判断

### 3.1 p 值 vs 置信区间

| 方法 | 优点 | 缺点 | 建议 |
|------|------|------|------|
| **p 值** | 简单直观 | 易误用 / 二元思维 | 仅作初步判断 |
| **置信区间** | 显示效应范围 | 解释复杂 | **推荐** |
| **贝叶斯** | 可纳入先验 | 计算复杂 | 高阶场景 |

### 3.2 判断标准

| p 值 | 结论 | 行动 |
|:---:|------|------|
| <0.01 | 强显著 | 大规模推广 |
| 0.01-0.05 | 显著 | 推广(持续观察) |
| 0.05-0.10 | 弱显著 | 继续测试 |
| >0.10 | 不显著 | 不采纳 / 增加样本 |

### 3.3 常见陷阱

| 陷阱 | 描述 | 防范 |
|------|------|------|
| **窥探(p-hacking)** | 反复查看数据直到显著 | 预设样本量 / 不提前停止 |
| **多重比较** | 同时测多个指标 | Bonferroni 校正 |
| **选择性报告** | 只报告显著的 | 预注册所有指标 |
| **辛普森悖论** | 分组与汇总结果相反 | 检查分层数据 |
| **新奇效应** | 新方案短期效果好 | 延长测试周期 |

## 五、高级实验方法

### 5.1 CUPED 方差缩减

**目的**: 利用预实验数据减少指标方差, 提升实验敏感度(等效于增加样本量)。

```python
import numpy as np

def cuped_adjust(y_pre, y_post, treatment):
    """
    CUPED: 用预实验数据调整实验后指标
    y_pre: 预实验期间指标(协变量)
    y_post: 实验期间指标(目标)
    treatment: 0=对照, 1=实验
    """
    # 计算协方差
    theta = np.cov(y_pre, y_post)[0, 1] / np.var(y_pre)
    
    # 调整后的指标
    y_adjusted = y_post - theta * (y_pre - np.mean(y_pre))
    
    # 比较调整后的均值
    control = y_adjusted[treatment == 0]
    variant = y_adjusted[treatment == 1]
    
    return control.mean(), variant.mean()

# 效果: 可使所需样本量减少 20-50%
```

### 5.2 序贯检验(Sequential Testing)

**目的**: 允许在达到样本量之前查看结果, 同时控制误报率。

| 方法 | 优点 | 适用 |
|------|------|------|
| SPRT(序贯概率比检验) | 边界清晰 | 简单场景 |
| mSPRT(混合 SPRT) | 灵活 | Optimizely 默认 |
| Always-Valid p-value | 可随时停止 | 通用 |

### 5.3 Switchback 实验

**适用**: 流量不足 / 单用户无法随机化的场景(如物流 / 定价)。

```
时间窗 1:  城市 A=实验 / 城市 B=对照
        ↓ 切换
时间窗 2:  城市 A=对照 / 城市 B=实验
        ↓ 切换
时间窗 3:  ...
```

## 七、实验文化建议

### 7.1 团队实验成熟度模型

| 等级 | 特征 | 实验/月 | 建议 |
|:---:|------|:-----:|------|
| L1 萌芽 | 偶尔做测试 | <2 | 建立基础流程 |
| L2 初级 | 有固定流程 | 2-10 | 工具化 |
| L3 中级 | 实验驱动决策 | 10-50 | 文化建设 |
| L4 高级 | 全员实验 | 50-200 | 自动化 |
| L5 卓越 | AI 辅助实验 | 200+ | 个性化 |

### 7.2 常见反模式

❌ **"不显著就继续跑"**: 容易得到虚假显著结论
❌ **"p<0.05 就推广"**: 忽略效应大小和护栏指标
❌ **"测试越多越好"**: 多重比较未校正
❌ **"实验只是数据组的事"**: 缺乏业务团队参与
