# User Research Synthesis

Turns raw user research data — interview transcripts, usability test notes, survey responses, or diary study entries — into a structured insights report with themes, supporting evidence, and prioritized recommendations.

## When To Use

Invoke this skill when you have completed one or more user research sessions and need to synthesize the raw data into actionable findings. Works with any qualitative or mixed-method study type.

## What It Produces

- An executive summary of the study
- Numbered key insights, each backed by direct quotes or observed behaviors
- A theme breakdown with evidence
- Prioritized recommendations linked to specific insights
- Methodological notes (sample size, caveats, confidence level)

## Supported Study Types

| Study Type | Best For |
|------------|----------|
| User interviews | Goals, mental models, pain points, motivations |
| Usability tests | Task friction, error patterns, navigation issues |
| Survey responses | Frequency analysis, open-ended theme clustering |
| Diary studies | Longitudinal behavior, context of use, usage drift |
| Focus groups | Shared language, group consensus, dissenting views |

## Safety Notes

- Never fabricate quotes; use only text from the provided data.
- Anonymize participant identifiers unless the user explicitly opts out.
- Do not claim statistical significance for qualitative data.
- Do not share raw participant data beyond the conversation unless the user instructs you to.
