# M5 透明度控制域范本

## M5-01 默认隐藏策略

### 输入示例
任务类型：团队协作身份涌现分析

### 处理过程
1. 隐藏范围确定：场互动细节默认隐藏
2. 隐藏级别设定：中等
3. 隐藏原因说明：避免信息过载
4. 用户感知设计：展示共识结果和网络拓扑

### 输出示例
```markdown
## 默认隐藏策略文档

### 隐藏范围
- **隐藏内容**：场互动细节
- **隐藏级别**：中等
- **隐藏原因**：避免信息过载
- **用户感知**：展示共识结果和网络拓扑

### 隐藏级别
| 级别 | 描述 | 适用场景 |
|------|------|----------|
| 低 | 展示所有细节 | 调试模式 |
| 中 | 展示关键结果 | 常规模式 |
| 高 | 仅展示最终结果 | 快速模式 |

### 用户感知设计
- **展示内容**：共识结果、网络拓扑
- **展示格式**：结构化文档 + 可视化图
- **展示控制**：用户可切换详细程度
```

## M5-02 用户请求处理

### 输入示例
用户请求：展示网络拓扑与共识形成过程

### 处理过程
1. 请求接收：接收用户追溯请求
2. 请求验证：确认用户需求
3. 请求处理：展示网络拓扑和共识过程
4. 请求响应：提供详细分析过程

### 输出示例
```markdown
## 用户请求处理流程

### 请求接收
- **请求类型**：展示网络拓扑与共识形成过程
- **请求时间**：YYYY-MM-DD HH:MM:SS
- **请求来源**：用户界面

### 请求验证
- **验证内容**：确认用户需求
- **验证方法**：检查请求参数
- **验证结果**：请求有效

### 请求处理
- **处理内容**：展示网络拓扑和共识过程
- **处理方法**：调用共识网络模块
- **处理结果**：获取网络拓扑和共识数据

### 请求响应
- **响应内容**：网络拓扑图 + 共识过程时间线
- **响应格式**：可视化图表
- **响应时间**：≤5秒
```

## M5-03 网络拓扑展示

### 输入示例
网络拓扑：5节点、7边、强连通

### 处理过程
1. 展示目标：展示网络拓扑结构
2. 展示方法：可视化图表
3. 展示设计：节点表示身份场，边表示协作关系
4. 展示实现：SVG图形

### 输出示例
```markdown
## 网络拓扑展示方案

### 展示目标
- **目标**：展示网络拓扑结构
- **受众**：用户
- **用途**：理解身份场之间的关系

### 展示方法
- **方法**：可视化图表
- **节点**：表示身份场
- **边**：表示协作关系
- **颜色**：不同身份场用不同颜色

### 展示设计
```
N1(领导) ──→ N2(技术)
   │           │
   ↓           ↓
N3(创意) ──→ N4(质量)
   │
   ↓
N5(产品)
```

### 展示实现
- **格式**：SVG图形
- **尺寸**：800x600
- **交互**：支持点击查看节点详情
```
