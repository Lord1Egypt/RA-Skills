# 中医舌诊（Tongue Diagnosis）

## 触发条件

用户上传舌头照片，或明确要求进行舌诊分析时触发本技能。

**关键词触发**：舌诊、舌象、舌头照片、看舌、望舌、舌照、舌苔、舌质、健康评估

## 工作流程

### Step 1：接收并观察舌象照片

当用户上传舌头照片时，仔细观察照片中的舌象特征。加载 reference 知识库作为对照依据。

> **加载以下 reference 文件（按需加载，不需要一次性全部加载）：**
> - 分析舌质时 → 加载 `references/tongue-body.md`
> - 分析舌苔时 → 加载 `references/tongue-coating.md`
> - 分析舌下时 → 加载 `references/tongue-sublingual.md` + `references/sublingual-supplement.md`（补充内容）
> - 分区对应时 → 加载 `references/tongue-zones.md`
> - 可见舌纹（裂纹/纹理）时 → 加载 `references/tongue-lines-supplement.md`（舌纹诊病补充知识）
> - 综合判断时 → 加载 `references/syndrome-patterns.md`
> - 输出报告时 → 加载 `references/report-template.md`

**观察维度**：

1. **舌质（舌体）**
   - 舌神：荣舌（有神）还是枯舌（无神）
   - 舌色：淡红、淡白、红、绛、青紫、紫暗
   - 舌形：老/嫩、胖大/瘦薄、齿痕、芒刺、裂纹
   - 舌态：正常/痿软/强硬/歪斜/颤动/吐弄/短缩

2. **舌苔**
   - 苔色：白、淡黄、深黄、焦黄、灰黑
   - 苔质：薄/厚、润/燥/滑、腻/腐、剥苔、有根/无根
   - 苔分布：均匀、局部、偏某区

3. **舌下络脉**（如果照片能观察到）
   - 长度、粗细、颜色、分支

4. **分区观察**（按舌尖/舌中/舌根/舌边）
   - 舌尖→心肺、舌中→脾胃、舌根→肾、舌边→肝胆

### Step 2：综合评估（健康度评级）

基于观察到的舌象特征，参照 `references/syndrome-patterns.md` 进行综合辨证，给出健康度评级：

| 评级 | 判定标准 | 对应行动 |
|------|---------|---------|
| 🟢 健康 | 淡红舌、薄白苔，舌体正常 | 告知良好，无需担忧 |
| 🟡 亚健康 | 轻度舌象异常（如轻微齿痕、略胖大、薄黄苔等） | 提示功能失调倾向，建议调整生活方式 |
| 🟠 需关注 | 较明显的舌象异常（如明显齿痕、黄腻苔、裂纹等） | 提示存在潜在健康问题，建议关注并观察变化 |
| 🔴 建议就医 | 明显的病理性舌象（如绛舌、青紫舌、镜面舌、歪斜舌等） | 建议尽早就医检查 |

### Step 3：输出结构化报告

参照 `references/report-template.md` 输出完整报告。

**报告必须包含以下内容**：
1. 📸 舌象特征分析（舌质、舌苔、舌下络脉、分区观察）
2. 🟡 健康度评级
3. ⚠️ 可能存在的健康隐患（基于证候分析）
4. 💡 日常调理建议（饮食 + 生活方式 + 中医调理方向）
5. ⚠️ 医学免责声明（必须包含！）

### Step 4：重要注意事项

1. **必须包含免责声明**：每次分析结果都必须在最后附上医学免责声明
2. **不能替代专业诊断**：明确告知用户这是参考性分析，不能替代执业医师的诊断
3. **建议就医**：任何异常舌象都应建议用户结合自身症状咨询医生
4. **限于单次照片**：单张照片的舌象分析存在局限性，建议动态观察
5. **不能开药**：AI不能开具处方或推荐具体药物
6. **图片质量提醒**：如果图片质量不佳（光线不足、角度不对、模糊等），应提醒用户重新拍摄

## 输出质量要求

1. **数据驱动**：每个结论都必须有舌象特征作为依据，不能凭空判断
2. **保守为原则**：宁可判断为轻微异常，也不过度解读
3. **图文对应**：描述时应明确指出照片中哪部分表现出什么特征
4. **患者友好**：术语可适当解释，让非专业人士能理解
5. **分级明确**：评级区分度要清晰，不要模糊地带
