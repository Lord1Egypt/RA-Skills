## 品牌视觉资产生成清单与依赖拓扑

品牌视觉资产生成技能按"**核心管线 + 扩展层**"两层架构组织的任务清单。核心管线（BA1-BA4）生成62个必备品牌资产（21 SVG + 39 PNG + 1 ICO + 1 HTML），任何品牌项目均走此管线。扩展层（BA5-BA6）在核心完成后可选并行执行，共25个扩展文件（12 SVG + 12 PNG + 1 HTML）。全量87个文件（33 SVG + 51 PNG + 1 ICO + 2 HTML）。

**域间逻辑流**：BA1（需求）→ BA2（设计）→ BA3（转换）→ BA4（交付核心）→ BA5（品牌规范）‖ BA6（扩展应用）→ BA4-02（最终完整性）→ BA4-03（清理）→ BA4-04（交付）

---

### BA1 需求分析域

将用户输入转化为明确的设计规格。

| ID | 任务类型 | 说明 | 依赖 | 元操作映射 |
|----|---------|------|------|-----------|
| BA1-01 | 信息采集 | 采集公司名称、行业、个人信息（姓名/手机/邮箱/网址/地址）、设计偏好 | 无（入口） | S→I |
| BA1-02 | 设计方向校准 | 推导色彩方案、风格方向、Logo构成思路，获取用户确认 | BA1-01 | C→I |
| BA1-03 | 资产规格规划 | 规划7类资产文件清单、尺寸规格、目录结构（核心标志目录3SVG+21PNG+1ICO+商标12+数字化10+办公8+宣传6+规范7+扩展18） | BA1-02 | C→A |

### 依赖拓扑摘要

#### 一、域内链路

**需求分析链路**: BA1-01 → BA1-02 → BA1-03
**设计生成链路**: BA2-01 → BA2-02 → BA2-03 → BA2-04（BA2-05与BA2-03→BA2-04并行后汇入BA3-01）
**格式转换链路**: BA3-01 → BA3-02 ‖ BA3-02A ‖ BA3-02B → BA3-03 → BA3-04
**整合交付链路**: BA4-01 → BA4-02 → BA4-03 → BA4-04
**品牌规范链路**: BA5-01 + BA5-02 ‖ BA5-03 ‖ BA5-04（BA5-01依赖BA4-01；BA5-02/03/04依赖BA2-01/BA2-02，四任务可并行执行）
**扩展应用链路**: BA6-01 → BA6-02 → BA6-03（三任务按逻辑顺序执行）

#### 二、跨域协同链路

**主链路**: BA1-03 → BA2-01 → BA3-01 → BA4-01 → BA5-01 ‖ BA6-01
**需求→设计**: BA1-03 → BA2-01（规格规划完成后开始设计）
**设计→转换**: BA2-04 + BA2-05 → BA3-01（SVG和信息注入完成后开始渲染）
**转换→交付**: BA3-04 → BA4-01（渲染校验通过后生成展示页）
**交付→规范/扩展**: BA4-01 → BA5-01 + BA6-01（核心展示页完成后生成规范和扩展）
**规范/扩展→最终交付**: BA5-04 + BA6-03 → A-12(全量HTML) → BA4-02 → BA4-03 → BA4-04

#### 三、并行与迭代链路

**并行**: BA2-05 ‖ BA2-03→BA2-04（小尺寸优化版与SVG资产批量生成可并行）
**并行**: BA3-02 ‖ BA3-02A ‖ BA3-02B（纯图标多尺寸、横版/竖版多尺寸、批量SVG渲染可并行）
**并行**: BA5-01 ‖ BA6-01（品牌规范与扩展应用可并行）
**Logo迭代**: ↻(BA1-02 → BA2-01 → BA2-02, 直到用户确认Logo)
**渲染迭代**: ↻(BA3-01 → BA3-04, 直到渲染质量达标)
**HTML迭代**: ↻(BA4-01 → BA4-02, 直到完整性校验通过)
**规范迭代**: ↻(BA5-01 → BA5-02→BA5-03→BA5-04, 确保规范与Logo一致)

#### 四、管线模式

**主管线（核心必备 + 扩展可选）**:
```
[核心管线]
S-01(信息采集) → I-01(方向校准) → A-01(Logo生成)
↻(I-01 → A-01, 用户确认)
→ A-02(形态派生) ⇉ [A-03(SVG批量), A-04(小尺寸版)]
→ A-05(信息注入) → A-06(渲染)
⇉ [A-07(图标多尺寸), A-07A(横版/竖版多尺寸), A-07B(批量SVG渲染)]
→ A-08(ICO) → G-01(渲染校验) ?(通过) → A-09(核心HTML) → G-02(核心完整性) → O-01(核心整理) → O-02(核心交付62个文件)
[扩展层，可选并行]
→ ⇉ [A-10(BA5: 品牌规范), A-11(BA6: 扩展应用)]
→ A-12(全量HTML) → G-03(全量验收) → O-03(全面清理) → O-04(最终交付87个文件)
```

**文件统计**: 2 HTML + 33 SVG（3核心标志+18 VI系统+3品牌规范+9扩展应用；辅助简化版在BA4-03阶段清理） + 51 PNG（核心标志21：横版标准1+横版4多尺寸+竖版标准1+竖版4多尺寸+纯图标6多尺寸+ICO内嵌4+纯图标480展示版1+VI系统18+品牌规范3+扩展应用9） + 1 ICO = **87个文件**
