# v2.9.4 (2026-05-08)

**P0 续修：agent-api（HTTPS 路径）也撞 markdown_v2 40008，v2.8.32 误判**

## 用户反馈

赵博 5/8 私聊，BOT 回复"👆 李经敏简历 HTML 文件，点击下载 ⚠️ Media failed."
——LLM 以为发了带链接的消息，但企微 client 显示 Media failed，没有下载链接。

## 根因

v2.8.32 修 bot-ws 路径 markdown_v2 → markdown 时，**误判**说 agent-api（HTTPS）支持
markdown_v2 不动。但 5/7-5/8 gateway.log 实测：

```
[wecom-agent-api] sendMarkdown response account=default agentId=1000009
  toUser=PanDuoDuo errcode=40008 errmsg=invalid message type
[wecom-agent-api] sendMarkdown response ... toUser=HuangXuanRong errcode=40008 ...
```

agent-api 用 markdown_v2 一样撞 40008，从 v2.x 起就一直有问题，因为对外发非 admin 用户
消息时常有 markdown 内容触发 `Markdown features detected, sending as markdown_v2`
分支，全部失败。

## 企微官方文档

`/cgi-bin/message/send`（自建应用）+ `/cgi-bin/appchat/send`（应用群）
**msgtype 列表**：

| 支持 | text / image / voice / video / file / textcard / news / mpnews / **markdown** / miniprogram_notice / template_card |
|---|---|
| 不支持 | **markdown_v2** ❌ |

`markdown_v2` 仅在两条路径支持：
- 群机器人 webhook (`/cgi-bin/webhook/send`)
- ai-bot inbound 回调的 response_url（`bot-webhook/active-reply.ts` 走的就是这条，不动）

普通自建应用 `message/send` API 永远不支持 markdown_v2。

## 修复

```diff
  // src/transport/agent-api/core.ts
- const body = useChat
-   ? { chatid: chatId, msgtype: "markdown_v2", markdown_v2: { content: text } }
-   : {
-       touser, toparty, totag, msgtype: "markdown_v2",
-       agentid: requireAgentId(agent),
-       markdown_v2: { content: text },
-     };
+ const body = useChat
+   ? { chatid: chatId, msgtype: "markdown", markdown: { content: text } }
+   : {
+       touser, toparty, totag, msgtype: "markdown",
+       agentid: requireAgentId(agent),
+       markdown: { content: text },
+     };
```

## 不动的 3 条路径

| 路径 | msgtype | 理由 |
|---|---|---|
| `agent-api/client.ts`（upstream agent 跨企业） | `text` | text 是企微官方支持的最基本类型 |
| `bot-webhook/active-reply.ts`（ai-bot response_url） | `markdown_v2` | ai-bot 协议这条 endpoint 真正支持 markdown_v2 |
| `bot-ws/*`（智能机器人 SDK） | `markdown` | v2.8.32 已修 |

## 经验沉淀

**v2.8.32 复盘的"5 秒查 SDK 类型定义"教训这次又翻车了**——当时只查了 `@wecom/aibot-node-sdk`
SDK 类型，没查企微官方 message/send API 的 msgtype 列表。HTTPS 路径不走 SDK 类型，
必须查官方文档。

任何"msgtype 选择"决策都要**两条都查**：
1. 你用的 SDK / library 类型定义
2. 后端 API 官方文档实际支持的 msgtype 列表
