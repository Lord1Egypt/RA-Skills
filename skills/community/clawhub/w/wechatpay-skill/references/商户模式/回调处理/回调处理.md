# 回调通知处理

> 微信支付异步通知的正确处理方式

## 回调流程

```
微信支付
    │
    ▼ POST notify_url
支付结果 → 解密 → 验签 → 业务处理 → 返回200
    │
    ▼
微信支付重试 ← 超时未响应
```

## 回调通知类型

| 通知类型 | 触发时机 |
|----------|----------|
| **支付成功回调** | 用户支付成功后 |
| **退款成功回调** | 退款审核通过并完成 |
| **退款失败回调** | 退款失败 |
| **合单支付成功回调** | 合单所有子单支付成功 |

## 处理步骤

### 1. 接收回调

微信支付POST请求到notify_url，Content-Type为application/json

### 2. 获取必要信息

从HTTP头获取：
- `Wechatpay-Signature`：回调签名
- `Wechatpay-Nonce`：随机串
- `Wechatpay-Timestamp`：时间戳
- `Wechatpay-Serial`：微信支付证书序列号

从请求体获取加密的回调报文

### 3. 验签

```python
# 构造待验签串
sign_str = f"{timestamp}\n{nonce}\n{body}\n"

# 使用微信支付公钥验签
verify(public_key, sign_str, signature)
```

### 4. 解密

回调报文中的敏感字段（不含resource节点本身）使用AES-256-GCM解密：

```python
from cryptography.hazmat.primitives.ciphers.aead import AESGCM
import base64

def decrypt(resource: dict, api_v3_key: str) -> dict:
    key = api_v3_key.encode()
    nonce = base64.b64decode(resource["nonce"])
    ciphertext = base64.b64decode(resource["ciphertext"])
    associated_data = resource["associated_data"].encode()
    
    aesgcm = AESGCM(key)
    plaintext = aesgcm.decrypt(nonce, ciphertext, associated_data)
    return json.loads(plaintext)
```

### 5. 业务处理

| 步骤 | 操作 |
|------|------|
| 检查订单状态 | 只处理SUCCESS状态 |
| 校验金额 | 回调金额必须等于本地订单金额 |
| 幂等处理 | 使用out_trade_no作为幂等key |
| 更新订单 | 标记支付成功，触发后续流程 |

### 6. 返回200

处理完成后必须返回：
```json
{"code": "SUCCESS", "message": "成功"}
```

**重要**：返回200后才能标记业务处理完成

## 合单支付回调

合单回调与基础支付不同，解密后返回：
```json
{
    "combine_out_trade_no": "合单订单号",
    "sub_orders": [
        {
            "mchid": "子商户号",
            "out_trade_no": "子单订单号",
            "trade_state": "SUCCESS",
            "transaction_id": "微信订单号"
        }
    ]
}
```

**必须遍历sub_orders处理每个子单**

## 常见问题

| 问题 | 解决方案 |
|------|----------|
| 回调收不到 | 检查notify_url公网可达，返回200 |
| 验签失败 | 使用微信支付平台证书验签 |
| 重复回调 | 使用out_trade_no做幂等key |
| 处理超时 | 先返回200再处理业务 |

## 安全注意

1. **必须验签**：不验签不处理业务
2. **必须校验金额**：防止篡改
3. **必须幂等**：防止重复处理
4. **必须返回200**：否则微信会重复通知
5. **日志脱敏**：不要打印完整回调内容
