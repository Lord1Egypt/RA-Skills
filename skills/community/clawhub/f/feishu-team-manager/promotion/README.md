# 推广材料使用指南

## 概述
本目录包含 feishu-team-manager (HR大姐头) 技能的完整推广材料，涵盖多个平台和场景的宣传文案、公告模板和使用案例。

## 文件结构

### 核心推广文件
1. **DISCORD_ANNOUNCEMENT.md** - Discord社区分享文案
   - 5个版本：发布公告、功能解析、使用案例、更新通知、问题征集
   - 包含发布指南和互动技巧

2. **FEISHU_ANNOUNCEMENT.md** - 飞书群公告模板
   - 6个模板：发布公告、功能演示、更新通知、使用技巧、案例征集、问题解答
   - 包含发布时间建议和格式指南

3. **SKILL_INTRO.md** - 技能介绍文案
   - 6个版本：简洁版、标准版、详细版、营销文案、社交媒体文案、视频脚本
   - 覆盖不同平台和宣传需求

4. **USE_CASES.md** - 使用案例集
   - 4个完整案例：电商客服、教育机构、个人助理、企业内部
   - 包含案例模板和征集指南

### 辅助文件
- **本文件**：使用指南和发布建议
- **promotion/** 目录：所有推广材料的集中存放

## 使用建议

### 根据平台选择材料
| 平台 | 推荐文件 | 推荐版本 |
|------|----------|----------|
| Discord | DISCORD_ANNOUNCEMENT.md | 版本1（发布公告） |
| 飞书群 | FEISHU_ANNOUNCEMENT.md | 模板1（发布公告） |
| 技能页面 | SKILL_INTRO.md | 版本2或3（标准/详细版） |

### 发布时间规划
#### 第一阶段：初始发布（第1周）
1. **Day 1**: Discord发布公告 + 飞书群公告
2. **Day 3**: 分享使用案例（电商客服案例）
3. **Day 5**: 功能深度解析

#### 第二阶段：持续推广（第2-4周）
1. **每周1次**: 分享新使用案例
2. **每两周1次**: 更新通知和功能改进
3. **持续**: 回答用户问题，收集反馈

#### 第三阶段：社区建设（长期）
1. **案例征集**: 鼓励用户分享使用经验
2. **功能投票**: 让用户参与功能规划
3. **贡献者计划**: 奖励活跃用户

### 定制化建议
1. **添加具体数据**
   - 实际用户数量
   - 真实效率提升数据
   - 用户评价和反馈

2. **本地化调整**
   - 针对不同地区调整语言风格
   - 考虑文化差异和用户习惯
   - 遵守各平台的内容规范

3. **版本同步**
   - 随技能版本更新文案内容
   - 及时反映新功能和改进
   - 保持宣传材料的准确性

## 危机处理预案

### 常见问题应对
1. **技术问题**
   - 准备标准问题解答（FEISHU_ANNOUNCEMENT.md 模板6）
   - 提供故障排除指南
   - 建立技术支持渠道

2. **负面反馈**
   - 及时、专业地回应
   - 承认问题并承诺改进
   - 提供解决方案或补偿

3. **安全质疑**
   - 透明说明VirusTotal警告原因
   - 强调代码开源可审查
   - 提供安全使用指南

### 沟通原则
1. **专业性**: 准确描述功能，不夸大宣传
2. **透明度**: 如实说明限制和注意事项
3. **响应性**: 及时回应用户问题和反馈
4. **建设性**: 将批评转化为改进机会
