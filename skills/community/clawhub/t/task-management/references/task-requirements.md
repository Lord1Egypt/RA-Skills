## 任务管理方法论要求清单

任务管理领域每种任务类型的"零件清单"——必选/可选组件、组装顺序、领域约束。按清单逐项产出。

---

### 任务数据结构规范

#### 任务字段定义

| 字段 | 类型 | 必选 | 说明 |
|------|------|------|------|
| id | string | 是 | 任务唯一标识符，格式：TASK-{序号} |
| subject | string | 是 | 任务标题，简洁明确，最大100字符 |
| description | string | 是 | 任务详细描述，最少10字符 |
| status | string | 是 | 任务状态：pending/in_progress/completed/deleted |
| activeForm | string | 否 | 任务进行中表单，格式：正在{动作} |
| owner | string | 否 | 任务负责人 |
| metadata | object | 否 | 自定义元数据 |
| dependencies | object | 否 | 任务依赖关系 |
| subtasks | array | 否 | 子任务列表 |
| created_at | string | 是 | 任务创建时间，格式：YYYY-MM-DD HH:mm:ss |
| updated_at | string | 是 | 任务更新时间，格式：YYYY-MM-DD HH:mm:ss |
| completed_at | string | 否 | 任务完成时间，格式：YYYY-MM-DD HH:mm:ss |

#### metadata字段定义

| 字段 | 类型 | 说明 |
|------|------|------|
| priority | string | 优先级：low/medium/high/critical |
| deadline | string | 截止时间，格式：YYYY-MM-DD |
| tags | array | 标签列表 |
| estimated_hours | number | 预估工时 |

#### dependencies字段定义

| 字段 | 类型 | 说明 |
|------|------|------|
| blockedBy | array | 被阻塞的任务ID列表 |
| addBlocks | array | 阻塞的其他任务ID列表 |

### T0 任务协调域

#### T0-01 任务需求分析
- **必选组件**: 任务描述、任务复杂度评估、依赖关系分析、执行环境分析
- **可选组件**: 任务优先级、任务截止时间、任务资源需求
- **组装顺序**: 任务接收→复杂度评估→依赖分析→环境分析→需求汇总
- **约束**: 任务描述必须清晰明确；复杂度评估必须基于客观标准；依赖分析必须完整
- **格式**: 任务需求分析报告（Markdown）

#### T0-02 任务域选择
- **必选组件**: 可用任务域列表、任务域选择理由、任务域组合方案
- **可选组件**: 任务域依赖关系说明
- **组装顺序**: 需求分析→任务域扫描→选择决策→组合设计
- **约束**: 任务域选择必须基于任务需求，不可强行套用
- **格式**: 任务域选择报告（Markdown）

#### T0-03 执行顺序确定
- **必选组件**: 任务依赖关系分析、执行顺序方案、关键路径识别
- **可选组件**: 并行执行可能性分析
- **组装顺序**: 依赖分析→顺序确定→关键路径识别
- **约束**: 执行顺序必须尊重任务依赖关系
- **格式**: 执行顺序图（Markdown流程图）

#### T0-04 结果整合与优化
- **必选组件**: 各任务执行结果汇总、整合方案、优化建议
- **可选组件**: 整合效果评估
- **组装顺序**: 结果收集→整合分析→优化建议
- **约束**: 整合必须保持各任务输出的完整性
- **格式**: 整合报告（Markdown）

### T1 任务创建域

#### T1-01 任务识别
- **必选组件**: 任务来源、任务类型、任务特征
- **可选组件**: 任务背景、任务目标
- **组装顺序**: 任务接收→特征识别→类型判定→任务确认
- **约束**: 任务识别必须准确，不可遗漏关键信息
- **格式**: 任务识别报告（Markdown）

#### T1-02 任务定义
- **必选组件**: subject（标题）、description（描述）
- **可选组件**: activeForm（进行中表单）、owner（负责人）、metadata（自定义元数据）
- **组装顺序**: 标题设置→描述编写→元数据配置→任务定义确认
- **约束**: subject必须简洁明确；description必须详细完整；元数据必须符合规范
- **格式**: 任务定义文档（Markdown）

#### T1-03 任务分类
- **必选组件**: 任务复杂度分类、任务类型分类、任务优先级分类
- **可选组件**: 任务标签、任务分类依据
- **组装顺序**: 复杂度评估→类型判定→优先级设定→分类确认
- **约束**: 分类必须基于任务特征，不可随意分类
- **格式**: 任务分类报告（Markdown）

#### T1-04 任务注册
- **必选组件**: 任务注册信息、任务注册状态、任务注册时间
- **可选组件**: 任务注册日志、任务注册确认
- **组装顺序**: 信息准备→注册执行→状态确认→注册完成
- **约束**: 注册信息必须完整准确；注册状态必须及时更新
- **格式**: 任务注册记录（Markdown）

---

### T2 任务分解域

#### T2-01 复杂度评估
- **必选组件**: 任务步骤数、任务依赖关系数、任务参与角色数
- **可选组件**: 任务时间估算、任务资源估算
- **组装顺序**: 步骤识别→依赖分析→角色识别→复杂度判定
- **约束**: 步骤识别必须完整；依赖分析必须准确；复杂度判定必须客观
- **格式**: 复杂度评估报告（Markdown）

#### T2-02 步骤识别
- **必选组件**: 主要执行步骤、步骤顺序、步骤依赖关系
- **可选组件**: 步骤详细说明、步骤验证标准
- **组装顺序**: 任务分析→步骤识别→顺序确定→依赖确认
- **约束**: 步骤识别必须覆盖任务全流程；步骤顺序必须合理；依赖关系必须清晰
- **格式**: 步骤识别文档（Markdown）

#### T2-03 子任务创建
- **必选组件**: 子任务列表、子任务元数据、子任务依赖关系
- **可选组件**: 子任务分配、子任务时间估算
- **组装顺序**: 步骤分解→子任务创建→元数据设置→依赖配置
- **约束**: 子任务必须是可独立执行的最小操作；子任务元数据必须完整；依赖关系必须准确
- **格式**: 子任务列表（Markdown表格）

#### T2-04 依赖识别
- **必选组件**: 依赖类型、依赖关系图、依赖验证
- **可选组件**: 依赖风险分析、依赖变更处理
- **组装顺序**: 依赖分析→类型判定→关系图绘制→依赖验证
- **约束**: 依赖类型必须准确；依赖关系必须完整；依赖验证必须通过
- **格式**: 依赖关系图（Markdown流程图）

#### T2-05 关键路径识别
- **必选组件**: 关键路径、关键任务、关键路径时间估算
- **可选组件**: 关键路径风险分析、关键路径优化建议
- **组装顺序**: 依赖分析→路径识别→关键任务确定→时间估算
- **约束**: 关键路径必须准确；关键任务必须明确；时间估算必须合理
- **格式**: 关键路径分析报告（Markdown）

#### T2-06 并行识别
- **必选组件**: 可并行任务、并行条件、并行限制
- **可选组件**: 并行执行策略、并行结果合并方案
- **组装顺序**: 依赖分析→并行识别→条件确认→并行规划
- **约束**: 并行任务必须无依赖关系；并行条件必须满足；并行限制必须考虑
- **格式**: 并行执行计划（Markdown）

### T3 状态跟踪域

#### T3-01 状态定义
- **必选组件**: 状态类型、状态含义、状态转换规则
- **可选组件**: 状态图标、状态颜色
- **组装顺序**: 状态识别→类型定义→含义说明→转换规则制定
- **约束**: 状态类型必须完整；状态含义必须清晰；转换规则必须明确
- **格式**: 状态定义文档（Markdown）

#### T3-02 状态转换规则
- **必选组件**: 转换触发条件、转换执行动作、转换验证规则
- **可选组件**: 转换日志、转换通知
- **组装顺序**: 触发条件定义→执行动作设计→验证规则制定→规则确认
- **约束**: 触发条件必须明确；执行动作必须具体；验证规则必须严格
- **格式**: 状态转换规则文档（Markdown）

#### T3-03 进度计算
- **必选组件**: 进度计算公式、进度数据来源、进度更新频率
- **可选组件**: 进度可视化、进度预测
- **组装顺序**: 公式设计→数据确认→更新频率设定→进度计算
- **约束**: 计算公式必须准确；数据来源必须可靠；更新频率必须合理
- **格式**: 进度计算报告（Markdown）

#### T3-04 阻塞识别
- **必选组件**: 阻塞任务、阻塞原因、阻塞解决建议
- **可选组件**: 阻塞影响分析、阻塞优先级
- **组装顺序**: 阻塞检测→原因分析→影响评估→解决建议
- **约束**: 阻塞识别必须及时；原因分析必须准确；解决建议必须可行
- **格式**: 阻塞识别报告（Markdown）

#### T3-05 状态更新
- **必选组件**: 更新时间、更新内容、更新原因
- **可选组件**: 更新日志、更新通知
- **组装顺序**: 状态变化检测→更新内容准备→更新执行→更新确认
- **约束**: 更新必须及时；内容必须准确；原因必须清晰
- **格式**: 状态更新记录（Markdown）

### T4 依赖管理域

#### T4-01 依赖类型定义
- **必选组件**: 依赖类型、依赖含义、依赖使用场景
- **可选组件**: 依赖示例、依赖最佳实践
- **组装顺序**: 依赖分析→类型识别→含义定义→场景说明
- **约束**: 依赖类型必须完整；含义必须清晰；场景必须具体
- **格式**: 依赖类型定义文档（Markdown）

#### T4-02 依赖解析
- **必选组件**: 依赖链解析、可执行任务确定、依赖验证
- **可选组件**: 依赖解析日志、依赖解析优化
- **组装顺序**: 依赖收集→链路解析→任务确定→依赖验证
- **约束**: 依赖链必须完整解析；可执行任务必须准确确定；依赖验证必须通过
- **格式**: 依赖解析报告（Markdown）

#### T4-03 循环检测
- **必选组件**: 循环检测算法、循环处理策略、循环预防措施
- **可选组件**: 循环检测日志、循环处理示例
- **组装顺序**: 算法设计→循环检测→处理策略制定→预防措施设计
- **约束**: 检测算法必须准确；处理策略必须有效；预防措施必须可行
- **格式**: 循环检测报告（Markdown）

#### T4-04 依赖变更处理
- **必选组件**: 变更检测、变更影响分析、变更处理流程
- **可选组件**: 变更日志、变更通知
- **组装顺序**: 变更检测→影响分析→处理流程设计→变更执行
- **约束**: 变更检测必须及时；影响分析必须全面；处理流程必须规范
- **格式**: 依赖变更处理报告（Markdown）

#### T4-05 并行机会识别
- **必选组件**: 并行条件分析、并行任务识别、并行执行建议
- **可选组件**: 并行执行策略、并行结果合并方案
- **组装顺序**: 条件分析→任务识别→执行建议→并行规划
- **约束**: 并行条件必须满足；并行任务必须准确；执行建议必须可行
- **格式**: 并行机会识别报告（Markdown）
