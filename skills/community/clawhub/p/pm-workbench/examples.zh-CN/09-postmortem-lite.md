# 09 - 轻量复盘

## Input

> 我们上个月上线了一个功能，但采用率低于预期。帮我写一个轻量 postmortem，重点讲清楚预期和实际差距、原因、经验和下一步。

## What good output looks like

好的 `pm-workbench` 输出应该：

- 对比 expected vs actual
- 不写成流水账
- 区分原因和猜测
- 提炼能改变下次行为的经验
- 结尾有具体 next actions

## Example target output

# Postmortem Lite

## 1. 原始目标

这个功能原本希望提高重复使用，让用户有更多理由回到产品里完成核心动作。

## 2. 实际结果

上线后有曝光，但采用率低于预期，重复使用也没有形成明显趋势。

## 3. Expected vs actual

### Expected

用户会理解这个功能的价值，并在后续工作流中重复使用。

### Actual

用户看到了功能，但没有足够多的人把它纳入日常使用。

### Gap

我们高估了用户对这个功能价值的自然理解，低估了入口、场景和重复使用动机的重要性。

## 4. 可能原因

- 功能没有嵌入高频场景
- 用户不知道什么时候该用
- 上线指标过多关注曝光，过少关注重复行为

## 5. 经验

- 兴趣不等于采用
- 上线前必须定义重复使用场景
- 需要测采用质量，而不只是上线触达

## 6. 下一步

- 重新确认最高频使用场景
- 调整入口或上下文触发点
- 增加重复使用指标
- 4 周后再判断是否继续扩展
