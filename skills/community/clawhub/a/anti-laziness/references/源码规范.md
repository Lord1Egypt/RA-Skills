# 源码调研/梳理专项规范

## 一、源码阅读基本原则

### 1.1 搜索 vs 阅读

- **搜索结果只能作为发现线索**，不能替代源码分析
- grep/搜索是定位工具，read 是理解工具
- 找到关键文件后，必须逐行阅读理解

### 1.2 引用规范

- 所有结论必须有 `文件路径:行号` 引用
- 格式示例：`config.py:45-52` 表示 config.py 第 45 到 52 行
- **没有行号引用的结论 = 猜测 = 偷懒**

### 1.3 深度分级

| 级别 | 描述 | 标注 |
|-----|------|------|
| L1 | 只搜索了关键字 | ⚠️ 浅层 |
| L2 | 读了关键函数 | 需标注 |
| L3 | 理解了完整调用链 | 深度 |
| L4 | 验证了边界条件 | 完整 |

## 二、源码调研三阶段门控

### Phase 1：入口点与表面映射

**目标**：定位程序的入口点和关键配置

**操作步骤**：
1. 找到 CLI 入口或 API 入口
2. 识别主处理函数
3. 识别关键配置结构体/类

**产出**：

| 项目 | 文件路径 | 行号 | 证据片段 |
|------|---------|------|----------|
| 入口函数 | | | |
| 主处理函数 | | | |
| 关键配置类 | | | |

**门控条件**：
- 所有项目都有文件路径+行号
- 无空白项（未找到的写 `UNRESOLVED`）

---

### Phase 2：执行追踪（调用图）

**目标**：跟踪 Happy Path 的完整调用链

**操作步骤**：
1. 从入口函数开始，逐跳跟踪函数调用
2. 每一跳：找到定义位置，验证 import 路径
3. 记录数据流向：参数从哪来、返回值到哪去

**产出**：调用图证据表

| 步骤 | 动作 | 文件路径 | 行号 | 证据片段 | 数据流 |
|------|------|---------|------|----------|--------|
| 1 | 入口 | main.py | 10 | `def main()` | input → |
| 2 | 调用 | handler.py | 45 | `process(data)` | → 中间数据 → |
| 3 | 返回 | output.py | 20 | `return result` | → output |

**门控条件**：
- 每一跳都有 `文件路径:行号` + 源码片段
- 每一跳都验证了 import 路径
- 数据流从输入跟踪到输出

---

### Phase 3：风险与模糊性审计

**目标**：识别无法通过静态分析追踪的部分

**操作**：搜索动态模式
- `eval` / `__import__` / 装饰器动态注册
- `getattr(obj, method_name)` / 字符串分发
- 配置驱动：YAML/JSON 中的类名 → 运行时加载
- 插件系统：入口点注册 / 自动发现
- 继承多态：基类方法在子类中被覆盖

**产出**：

| 模糊点 | 文件路径 | 行号 | 无法追踪的原因 | 影响评估 |
|--------|---------|------|---------------|----------|
| 动态加载 | plugin.py | 30 | 类名从配置读取 | 🔶 中 |
| 装饰器 | auth.py | 15 | 运行时注册 | ⚠️ 低 |

**风险等级**：
- ⚠️ 低：可接受
- 🔶 中：需关注
- 🔴 高：需人工确认

## 三、自检清单

### 阅读前

- [ ] 明确任务边界：这次要分析什么？
- [ ] 锁定源码版本（commit hash）

### 阅读中

- [ ] 每读一个关键函数，记录文件:行号
- [ ] 遇到不确定的逻辑，标注 [待验证]
- [ ] 数据流是否从输入到输出？

### 阅读后

- [ ] 每个结论有 `文件:行号` 引用？
- [ ] 关键参数有具体数值？
- [ ] "不支持"/"未实现"是否经过全文搜索验证？
- [ ] 对比表每项有源码证据？

## 四、常见陷阱

### 陷阱 1：grep 就下结论

- ❌ 搜到 `TensorParallel` 就说"支持 TP"
- ✅ 追踪调用链，确认实际使用

### 陷阱 2：只读不动

- ❌ 读了很多文件，但没写调用图
- ✅ 每读一个关键函数，产出证据表

### 陷阱 3：假设继承

- ❌ 认为子类一定调用了父类方法
- ✅ 检查 super() 调用或重写逻辑

### 陷阱 4：忽略配置

- ❌ 只读代码，不看配置文件
- ✅ 代码+配置一起分析
