# 鸿蒙游戏 — 必报与选报事件

> 鸿蒙端必报事件与 Android/iOS 端基本一致，主要差异：鸿蒙端**仅有通用上报方法** `logAction()`，无专用方法。

## 必报事件

| action_type | 中文名 | 说明 |
|------------|--------|------|
| `START_APP` | 启动应用 | 鸿蒙端**自动采集**，无需手动上报 |
| `REGISTER` | 注册 | 用户完成账号注册时上报 |
| `LOGIN` | 登录 | 用户每次完成账号登录时上报 |
| `PURCHASE` | 付费 | **必须传 `value`（金额，单位：分）**，完整上报所有用户所有支付方式的全部付费行为 |

## 条件必报事件（无该行为可不报）

| action_type | 中文名 | 说明 |
|------------|--------|------|
| `CREATE_ROLE` | 创建角色 | 无该行为可不报 |
| `TUTORIAL_FINISH` | 完成新手教程 | 无该行为可不报 |
| `VIEW_CONTENT` | 浏览商城/活动 | 无该行为可不报 |
| `UPDATE_LEVEL` | 游戏等级提升 | 无该行为可不报 |

## 上报方式

鸿蒙端仅提供通用上报方法，无专用方法：

```typescript
logAction("PURCHASE", { "value": 600 })
logAction("REGISTER", {})
logAction("START_APP", {})
```

> 所有行为均通过 `logAction(actionType, actionParam)` 上报。
