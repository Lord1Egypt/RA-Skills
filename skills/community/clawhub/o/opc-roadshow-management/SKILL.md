# 季度路演管理 Skill

## 概述

本Skill专为OPC导师设计，用于系统化管理季度项目路演活动，实现从项目筛选到精准匹配的完整闭环。

---

## 一、赛道选择

### 1.1 季度主赛道确定

每个季度选择1个主赛道进行深耕：

| 赛道 | 关键词 | 适合阶段 |
|------|--------|----------|
| 人工智能 | AI、大模型、机器学习 | 早期-B轮 |
| 生物医药 | 创新药、医疗器械、生物技术 | 中期-B轮 |
| 新能源 | 储能、氢能、光伏、风电 | 成长期-B轮 |
| 新材料 | 碳纤维、石墨烯、半导体材料 | 早期-B轮 |
| 高端装备 | 工业机器人、航空航天 | 中后期 |
| 半导体 | 芯片设计、封测、材料 | 早期-B轮 |

### 1.2 赛道选择流程

```
收集项目池 → 统计赛道分布 → 选择项目最集中赛道
                    ↓
            考虑市场热点与导师资源
                    ↓
              确定季度主赛道
```

## 二、项目筛选标准

### 2.1 入选条件（必须满足）

- [ ] 符合本季度主赛道方向
- [ ] 有清晰的产品或技术定位
- [ ] 具备基本的商业模型
- [ ] 团队完整性（至少1人全职）
- [ ] 有明确的融资需求

### 2.2 评分体系

| 维度 | 权重 | 评分标准（1-10分） |
|------|------|-------------------|
| 技术壁垒 | 25% | 专利布局、独家技术、研发能力 |
| 商业成熟度 | 25% | 收入规模、客户数量、商业模式验证 |
| 市场空间 | 20% | 目标市场容量、增长率 |
| 团队能力 | 15% | 背景经历、行业资源、执行力 |
| 估值合理性 | 15% | 估值与阶段匹配度、融资轮次 |

### 2.3 筛选流程

```
Step 1: 初步筛选（满足入选条件）
    ↓
Step 2: 量化评分（按评分体系）
    ↓
Step 3: 排序分组
    - A类（25-30个项目）：优先安排路演
    - B类（10-15个项目）：备选池
    - C类：待培育项目
    ↓
Step 4: 最终确认20-30个项目进入路演名单
```

## 三、路演安排

### 3.1 季度路演规划

```
季度时间轴（13周）
Week 1: 路演启动会 + 项目分组
Week 2-12: 每周1场路演直播（12个项目）
Week 13: 季度复盘 + 重点对接会
```

### 3.2 每周路演安排

| 时间节点 | 任务 | 负责人 |
|----------|------|--------|
| 周一 | 确认本周路演项目 + 发送邀请 | 运营 |
| 周二 | 项目资料收集与审核 | 项目经理 |
| 周三 | 直播预演 + 问题预判 | 技术支持 |
| 周四 | 路演直播（晚8点） | 全部 |
| 周五 | 会议纪要生成 + 反馈分析 | AI自动 |
| 周六 | 投资人问题汇总 | 运营 |
| 周日 | 筛选决策更新 | 导师 |

## 四、AI自动化路演记录

### 4.1 会议纪要自动生成

每场路演结束后，AI自动生成会议纪要：

**核心内容：**
1. **项目基本信息**：名称、赛道、融资轮次、估值
2. **项目介绍摘要**：3分钟路演核心内容提炼
3. **核心亮点**：技术/商业/团队亮点（3-5条）
4. **投资价值**：市场机会、竞争优势
5. **风险提示**：已识别的主要风险

**生成模板：** 见 `references/会议纪要模板.md`

### 4.2 常见问题问答记录

**记录维度：**
- 问：投资人具体问题
- 频次：被问次数（反映关注度）
- 答：项目方回答摘要
- 质量评分：回答满意度（1-5分）

**模板：** 见 `references/常见问答记录模板.md`

### 4.3 关注度分析

基于问答数据自动统计：

| 分析维度 | 计算方法 | 阈值预警 |
|----------|----------|----------|
| 问题频次 | 单问题出现次数/总场次 | ≥3次为热点 |
| 追问深度 | 同一问题追问轮次 | ≥2轮为深度关注 |
| 并发关注 | 多投资人同时提问 | ≥2人为热点 |

### 4.4 重大问题识别

**识别标准：**

| 问题类型 | 触发条件 | 风险等级 |
|----------|----------|----------|
| 技术质疑 | 核心技术上被追问≥2次 | 高 |
| 商业验证 | 收入/客户数据被质疑 | 高 |
| 估值争议 | 估值问题被多位投资人提及 | 中 |
| 团队疑虑 | 团队稳定性/能力被质疑 | 中 |
| 合规风险 | 法律/政策风险被提及 | 高 |

## 五、筛选决策树

### 5.1 分层筛选机制

```
                    20-30个项目
                         │
            ┌─────────────┴─────────────┐
            ↓                           ↓
    路演表现良好                    路演反馈一般
    （投资人主动追问）              （被动回答为主）
            │                           │
    ┌───────┴───────┐                   │
    ↓       ↓       ↓                   ↓
  继续关注  深度尽调  精准对接      进入培育池
  (A类)    (B类)    (C类)         (D类)
```

### 5.2 决策判断标准

**进入"10个重点对象"标准：**
- [ ] 路演后有≥3位投资人有跟进意向
- [ ] 核心问题回答获得多数投资人认可
- [ ] 无重大风险预警
- [ ] 估值在合理区间

**进入"2-3个精准对接"标准：**
- [ ] 获得≥1家投资机构明确意向
- [ ] 核心亮点与特定机构偏好匹配
- [ ] 团队愿意配合深度尽调
- [ ] 估值有谈判空间

**滚动机制：**
- 未进入重点的项目 → 进入下一年度培育池
- 季度末复盘，重新评估进入新季度筛选

## 六、投资机构匹配逻辑

### 6.1 匹配维度

| 维度 | 分析要素 | 数据来源 |
|------|----------|----------|
| 机构偏好 | 投资阶段、赛道、金额区间 | 机构公开信息 |
| 关注点 | 技术/商业/团队侧重点 | 路演互动数据 |
| 决策风格 | 决策周期、尽调深度 | 历史合作记录 |
| 资源能力 | 投后赋能、产业资源 | 机构介绍 |

### 6.2 匹配算法

```
项目特征向量 = (赛道, 阶段, 估值, 亮点类型)
                    ↓
机构偏好向量 = (主投赛道, 阶段范围, 金额区间)
                    ↓
        相似度计算（余弦相似度）
                    ↓
            匹配度评分（0-100）
                    ↓
        输出TOP3匹配机构
```

## 七、季度复盘

### 7.1 复盘维度

| 维度 | 指标 | 目标 |
|------|------|------|
| 路演覆盖 | 完成场次/计划场次 | 100% |
| 项目曝光 | 触达投资人数 | ≥50人 |
| 意向转化 | 有意向项目/路演项目 | ≥30% |
| 重点培育 | 进入重点池项目数 | 10个 |
| 精准对接 | 完成对接项目数 | 2-3个 |
| 成交率 | 实际签约/对接项目 | ≥30% |

### 7.2 复盘输出

1. 本季度路演总结报告
2. 项目池更新清单
3. 下季度赛道建议
4. 投资人关系维护清单
5. 改进优化建议

---

## 八、使用流程

```
1. 季度开始前：
   - 确定主赛道
   - 筛选20-30个项目
   - 建立项目档案

2. 每周：
   - 执行路演直播
   - AI自动生成记录
   - 更新项目状态

3. 季度中：
   - 动态调整重点池
   - 启动精准对接

4. 季度末：
   - 完成复盘报告
   - 更新项目池状态
   - 制定下季度计划
```
