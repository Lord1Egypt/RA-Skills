# 依赖影响分析器 (Dependency Impact Analyzer)

基于.md描述的标准skill，使用大模型进行代码依赖关系分析和逻辑缺陷检测。

## Skill定义

### 基本信息
- **名称**: 依赖影响分析器
- **描述**: 分析代码改动对上下游方法调用的影响风险以及逻辑缺陷
- **风险类型**: Dependency Impact, Logic Defect
- **支持语言**: Java, JavaScript/TypeScript, Python

### 检测规则

#### 依赖影响分析
1. **方法签名变更**: 检测方法参数、返回值、访问修饰符的变更
2. **调用关系变化**: 分析方法被调用和调用其他方法的关系
3. **继承关系影响**: 检测父类方法变更对子类的影响
4. **接口实现影响**: 检测接口变更对实现类的影响
5. **构造函数变更**: 检测构造函数参数或逻辑的变更

#### 逻辑缺陷检测
1. **空指针异常**: 检测可能产生空指针异常的操作
2. **数组越界**: 检测数组访问时的边界检查缺失
3. **除零异常**: 检测除法操作中的除零风险
4. **资源泄露**: 检测资源未正确关闭的风险
5. **无限循环**: 检测可能导致无限循环的代码
6. **条件判断错误**: 检测条件判断中的逻辑错误
7. **并发问题**: 检测多线程环境下的并发风险

### 风险等级定义
- **高风险**: 可能导致系统崩溃、数据丢失或安全漏洞的逻辑缺陷
- **中风险**: 可能影响功能正确性或性能的依赖关系变更
- **低风险**: 对系统影响较小的代码改进或重构

## 分析方法

### 分析步骤（每个风险必须经过回溯校验）

**步骤1: 代码理解**
- 理解代码变更的内容和上下文
- 确定变更涉及的方法、类、文件

**步骤2: 依赖关系分析**
- 识别方法之间的调用关系
- 分析上下游依赖

**步骤3: 影响范围评估**
- 评估变更对上下游代码的影响
- 确定影响的方法和类

**步骤4: 逻辑缺陷检测**
- 检测潜在的逻辑错误
- 识别可能导致运行时异常的模式

**步骤5: 生成初步风险（每个风险单独处理）**
- 对每个识别的风险，生成包含以下字段的初步结果：
  - type, module, file, **line（数字类型）**, description, codeSnippet, suggestion, testCases, level

**步骤6: 回溯校验（强制执行）**
对每个初步风险执行以下校验：

```
## 风险回溯校验

### 待校验风险
- 文件: {file}
- 行号: {line}
- 描述: {description}
- 代码片段: {codeSnippet}
- 建议: {suggestion}

### 校验任务
1. 回溯到{file}第{line}行，验证问题代码
2. 验证描述是否准确反映问题根本原因
3. 验证建议是否可实施且最优
4. 验证行号是否精确（如不准确则调整）
5. 给出置信度：高/中/低

### 校验后处理
- 高置信度：直接输出
- 中置信度：根据corrections修正后输出
- 低置信度：重新分析该风险
```

**步骤7: 输出最终风险**
- 整合校验后的风险信息
- 确保所有字段完整准确

### 输出格式（经过回溯校验）
```json
{
  "type": "Dependency Impact",
  "module": "Upstream Impact|Downstream Impact|Logic Defect",
  "file": "文件路径",
  "line": 42,
  "description": "风险描述",
  "codeSnippet": "相关代码片段",
  "suggestion": "修复建议",
  "testCases": ["测试用例1", "测试用例2"],
  "level": "high|medium|low",
  "impactScope": {
    "affectedCallers": ["调用者列表"],
    "affectedCallees": ["被调用者列表"],
    "changeType": "变更类型"
  },
  "validation": {
    "validated": true,
    "confidence": "high|medium|low",
    "checks": {
      "descriptionAccuracy": true,
      "suggestionFeasibility": true,
      "lineNumberAccuracy": true,
      "impactClarity": true
    },
    "corrections": []
  }
}
```

**注意**: `line`字段必须为数字类型，如 `42`，不能是字符串 `"42"`。

### 行号定位规范
- **必须**：每个风险必须包含具体行号，格式为数字
- **精确性**：行号必须精确到引发风险的具体代码行
- **验证**：行号必须在文件实际行数范围内
- **关联性**：行号必须与codeSnippet中的代码片段完全对应
## 使用说明

### 触发条件
- 当检测到Java文件中的方法变更时触发分析
- 当检测到可能影响调用关系的代码变更时触发分析
- 当检测到潜在的逻辑缺陷代码模式时触发分析

### 配置参数
- **语言支持**: Java (主要), JavaScript/TypeScript, Python
- **文件过滤**: 仅分析源代码文件 (*.java, *.js, *.ts, *.py)
- **忽略规则**: 忽略测试文件、配置文件、构建产物
- **风险排除规则**:
  - **配置文件硬编码问题**: 跳过 `.yaml`、`.yml`、`.properties` 等配置文件中的硬编码内容，此类问题属于配置管理规范范畴，不纳入代码风险分析
  - **低级别日志打印不规范问题**: 跳过仅涉及日志打印规范的低级别（LOW）问题，包括日志级别选择不当、日志格式不统一、缺少占位符等轻微问题，不在风险报告中呈现

### 集成方式
此skill通过RiskForge CLI框架自动加载，执行分析时会使用大模型进行智能分析。

### 回溯校验集成
**重要**: 回溯校验已嵌入到分析步骤中，不是独立的后处理环节。

- 每个风险生成后立即执行回溯校验
- 校验内容包括：描述准确性、建议有效性、行号精确性、影响清晰度
- 根据置信度决定是否需要修正
- 低置信度风险需要重新分析

详细校验规则请参考：`references/risk-backtracking-validator.md`

## 注意事项
1. 分析结果依赖于大模型的理解能力
2. 对于复杂的继承关系和多态调用，可能需要进一步的人工审查
3. **每个风险必须经过回溯校验才能输出**，不可跳过校验环节
4. **行号必须为数字类型**，如 `42`，不能是字符串 `"42"`
5. 低置信度风险应重新分析，不应降低标准输出
