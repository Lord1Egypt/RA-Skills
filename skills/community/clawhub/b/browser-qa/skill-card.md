## Description: <br>
Automate visual testing and UI interaction verification across smoke testing, interaction testing, visual regression, and WCAG AA accessibility audit workflows. <br>

This skill is ready for commercial/non-commercial use. <br>

## Publisher: <br>
[djc00p](https://clawhub.ai/user/djc00p) <br>

### License/Terms of Use: <br>
MIT-0 <br>


## Use Case: <br>
Developers, QA engineers, and product teams use this skill to structure browser-based UI checks before shipping. It guides smoke tests, form and journey validation, screenshot review, and accessibility checks for web applications. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Browser QA may exercise forms, authentication flows, payment paths, or authenticated browser profiles against live systems. <br>
Mitigation: Run interaction tests in staging or with explicit approval, use test accounts and synthetic payment or form data, and avoid personal authenticated Chrome profiles in production unless intended. <br>
