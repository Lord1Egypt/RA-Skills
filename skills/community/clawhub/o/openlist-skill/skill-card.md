## Description: <br>
This skill helps agents execute auditable OpenList file operations through the OpenList HTTP API, including browsing, moving, renaming, single-path deletion, offline task creation, task management, and audit logging. <br>

This skill is ready for commercial/non-commercial use. <br>

## Use Case: <br>
Developers and operators use this skill to let an agent manage an OpenList instance while preserving a preview-and-confirm workflow for modifying actions. It is intended for controlled file-management, offline download, task inspection, and audit-review workflows. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: The skill can perform meaningful file-management actions on an OpenList instance. <br>
Mitigation: Install it only for intended OpenList administration workflows, use a least-privilege OpenList token, and review each generated preview plan before apply. <br>
Risk: Delete operations are irreversible. <br>
Mitigation: Require explicit user confirmation after showing the normalized path and object type, and only apply delete plans generated by preview-delete. <br>
Risk: Tokens, .env files, and audit logs may contain sensitive operational information. <br>
Mitigation: Keep credentials and audit logs private; the skill redacts sensitive fields in audit records, but storage access should still be restricted. <br>
Risk: Disabling TLS verification can expose OpenList traffic to interception. <br>
Mitigation: Leave TLS verification enabled unless using a trusted local or self-signed setup and the interception risk is accepted. <br>
