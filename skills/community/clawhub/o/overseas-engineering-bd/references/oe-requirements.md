# 海外工程BD结构要求清单

每个任务的槽位定义，用于UTOS内容轴清单法。按5域价值链组织。

---

## OE1 数据采集域

### OE1-01 网站爬虫脚本开发
- **必选组件**: 目标URL列表、页面解析规则、数据提取字段映射、异常处理逻辑
- **可选组件**: 代理IP池、请求频率控制、登录态维持
- **组装顺序**: 目标URL分析 → 页面结构解析 → 提取规则编写 → 异常处理 → 测试验证
- **技术约束**: 必须使用Playwright处理动态加载页面，BeautifulSoup解析静态内容
- **格式**: Python脚本

### OE1-02 PDF文档下载与OCR处理
- **必选组件**: PDF下载链接列表、文本提取逻辑、OCR识别配置、输出格式定义
- **可选组件**: 批量下载队列、下载进度追踪、OCR准确率优化
- **组装顺序**: PDF链接采集 → 下载存储 → 文本提取(pdfplumber/PyMuPDF) → 扫描件OCR(Tesseract) → 结构化输出
- **技术约束**: 必须同时支持文字版PDF(pdfplumber或PyMuPDF)和扫描版PDF(Tesseract OCR)；PyMuPDF速度更快，可作为pdfplumber的替代方案
- **格式**: 纯文本流/JSON

### OE1-03 文库文档搜索与下载
- **必选组件**: Google Docks搜索指令、目标平台列表(Scribd/Issuu/Yumpu/SlideShare/Academia.edu/ResearchGate)、搜索结果解析逻辑
- **可选组件**: 搜索结果去重、相关性评分、自动下载触发
- **组装顺序**: Docks指令库构建 → Google Search API调用 → 结果筛选 → 文档下载
- **搜索约束**: 必须使用site:限定平台，配合国家/行业关键词组合
- **Google Docks示例**:
  - `site:scribd.com "Uganda" AND ("Architects Registration Board" OR "Directory of Architects")`
  - `site:issuu.com "Tanzania" "Contractors" "WhatsApp" filetype:pdf`
  - `"Tanzania" "solar" "list of participants" (site:slideshare.net OR site:scribd.com)`
- **格式**: PDF文档集合

### OE1-04 异常重试机制
- **必选组件**: 重试策略(间隔/次数)、失败日志记录、告警通知逻辑
- **可选组件**: 指数退避算法、断点续传、备用数据源切换
- **组装顺序**: 异常类型定义 → 重试策略配置 → 日志记录 → 告警触发
- **运维约束**: 非洲网站宕机频繁，建议失败后间隔2小时重试，最多3次
- **格式**: 配置文件/代码模块

## OE2 智能处理域

### OE2-01 DeepSeek API集成
- **必选组件**: API密钥配置、请求封装函数、响应解析逻辑、错误处理
- **可选组件**: 请求队列、速率限制、Token计数、成本监控
- **组装顺序**: API文档研究 → 认证配置 → 请求函数封装 → 响应解析 → 错误处理
- **技术约束**: 使用DeepSeek-V3 API，支持长文本输入
- **格式**: Python模块

### OE2-02 结构化信息提取
- **必选组件**: 提取Prompt模板、目标字段定义(JSON Schema)、文本分块逻辑、结果验证规则
- **可选组件**: 多语言支持、字段置信度评分、人工复核队列
- **组装顺序**: Prompt设计 → 字段Schema定义 → 文本分块(2000字/块) → API调用 → 结果合并验证
- **提取字段Schema**:
  ```json
  {
    "project_name": "项目名称(翻译为中文)",
    "budget": "预算金额(如未提及写'未公布')",
    "funding_source": "资金来源(如世行、当地政府等)",
    "owner": "业主或招标机构名称",
    "contact_person": "联系人姓名(Procurement Officer/Project Manager)",
    "email": "邮箱",
    "phone": "电话号码",
    "requirements": "列出投标所需的核心资质要求(不超过50字)",
    "deadline": "截标时间(YYYY-MM-DD)"
  }
  ```
- **Prompt设计要点**: 指定角色为"国际工程投标数据分析师"，要求严格JSON格式输出，未找到信息填"NULL"
- **格式**: JSON数组

### OE2-03 数据清洗与过滤
- **必选组件**: 过滤规则(资质等级/企业类型)、去重逻辑、数据标准化规则
- **可选组件**: 数据质量评分、异常值检测、人工审核队列
- **组装顺序**: 过滤规则定义 → 资质等级映射 → 去重处理 → 标准化输出
- **业务约束**: 只保留Class 1/2级承包商，过滤小型施工队
- **格式**: 清洗后JSON/CSV

### OE2-04 背景调查报告生成
- **必选组件**: 搜索查询模板、搜索API集成(Tavily/Serper)、结果汇总Prompt、报告格式定义
- **可选组件**: 多轮搜索策略、历史数据对比、风险评级
- **组装顺序**: 查询词生成 → 搜索执行 → 结果筛选(前5条) → AI汇总 → 报告生成
- **输出约束**: 50字以内背景简报，包含资金状况、历史项目、风险提示
- **格式**: Markdown简报

### OE2-05 联系方式补全
- **必选组件**: 数据富化API集成(Hunter.io/Apollo.io)、已有联系人列表、补全策略、结果合并逻辑
- **可选组件**: 联系方式验证、优先级排序、数据源标记
- **组装顺序**: API认证 → 缺失字段识别 → API查询 → 结果验证 → 合并到主数据
- **业务约束**: 优先补全邮箱，其次电话，WhatsApp号码需单独验证
- **格式**: 补全后JSON

### OE2-06 伙伴画像生成
- **必选组件**: 画像维度定义(主营/规模/资质)、标签体系、简报模板
- **可选组件**: 匹配度评分、合作潜力评级、历史互动记录
- **组装顺序**: 背调数据输入 → 维度提取 → 标签生成 → 简报撰写
- **画像维度**: 主营业务匹配度、资质等级、资金实力、历史项目、风险等级
- **格式**: 结构化画像JSON

## OE3 数据管理域

### OE3-01 飞书表格数据导入
- **必选组件**: 飞书API认证(tenant_access_token)、表格ID配置、字段映射、批量写入逻辑
- **可选组件**: 数据去重、增量更新、导入日志
- **组装顺序**: API认证 → 表格创建/定位 → 字段映射 → 批量POST → 结果验证
- **技术约束**: 使用飞书Bitable API，支持批量写入
- **格式**: API调用脚本

### OE3-02 飞书多维表格设计
- **必选组件**: 表格结构定义、字段列表、视图配置、权限设置
- **可选组件**: 自动化规则、仪表盘、数据验证
- **组装顺序**: 业务需求分析 → 字段设计 → 视图配置 → 权限设置 → 自动化规则
- **表格类型**: 项目情报看板、伙伴动态跟踪舱、建筑师高价值核心库
- **格式**: 飞书表格配置

### OE3-03 数据推流与展示
- **必选组件**: 数据源配置、推流逻辑、刷新频率、展示格式
- **可选组件**: 数据过滤、排序规则、导出功能
- **组装顺序**: 数据源对接 → 推流逻辑编写 → 刷新配置 → 展示优化
- **技术约束**: 使用飞书API定时批量POST，支持每日更新
- **格式**: API推流脚本

## OE4 触达准备域

### OE4-01 WhatsApp号码预检
- **必选组件**: 号码列表、WhatsApp检测逻辑、结果分类(已开通/未开通)
- **可选组件**: 批量检测队列、检测频率控制、结果缓存
- **组装顺序**: 号码提取 → 格式标准化 → WhatsApp检测 → 结果标记
- **技术约束**: 使用开源WhatsApp检测脚本或Twilio/Wati API
- **格式**: 带标记的号码列表

### OE4-02 邮件触达序列设计
- **必选组件**: 邮件模板(多轮次)、发送间隔规则、个性化变量、SMTP配置
- **可选组件**: 打开率追踪、退订处理、A/B测试
- **组装顺序**: 邮件轮次定义 → 模板编写 → 间隔设置 → 变量配置 → SMTP对接
- **触达节奏**: 第1周(介绍) → 第3周(案例) → 第5周(询价)
- **格式**: 邮件模板集合+配置

### OE4-03 WhatsApp破冰话术生成与人设配置
- **必选组件**: 话术模板(Text+1 Media原则)、个性化变量、发送时机规则、WhatsApp Business人设配置、谷歌通讯录同步
- **可选组件**: 话术效果追踪、多版本测试、自动跟进逻辑
- **组装顺序**: WhatsApp Business人设配置 → 谷歌通讯录同步 → 目标画像输入 → 话术模板选择 → 个性化填充 → 媒体附件准备
- **WhatsApp Business人设武装**:
  - 准备2-3个业务员专属WhatsApp Business账号
  - 头像：必须是真实、穿着职业装的个人照片（最好在工地或工厂实景），不要用公司Logo
  - Catalog(产品目录)：上传钢结构厂房3D渲染图、光伏微电网系统架构图，附参数
- **谷歌通讯录同步**:
  - 将飞书表格里的名字和号码导出为Google Contacts支持的CSV格式
  - 导入到绑定的谷歌账号，手机WhatsApp里会瞬间出现好友列表，免手动添加
- **话术原则**: 短句+1张图/PDF，技术协助者人设，不群发
- **话术模板示例**:
  ```
  Hi [Name], I'm [Your Name] from [Company]. Saw your impressive portfolio in the Uganda Architects Directory.
  We specialize in pre-engineered steel structures and solar microgrids (manufactured in China).
  I noticed modern commercial designs in Kampala are demanding faster construction and green energy. Sharing a quick 2-page spec sheet on how our steel frames integrate seamlessly with rooftop solar.
  Might be useful for your upcoming structural designs. Cheers!
  ```
- **格式**: 话术文本+媒体文件+人设配置文档+CSV通讯录

## OE5 触达执行域

### OE5-01 触达节奏SOP执行
- **必选组件**: SOP流程定义(Day1/Day2/Day14)、触发条件、执行动作、状态追踪
- **可选组件**: 自动化触发器、人工干预节点、效果统计
- **组装顺序**: SOP流程设计 → 触发条件配置 → 执行逻辑编写 → 状态追踪 → 效果统计
- **SOP节奏**: Day1(破冰) → Day2(已读跟进) → Day14(干货分享)
- **人工介入**: 高价值客户沟通可由人工介入
- **格式**: SOP文档+自动化脚本

### OE5-02 触达效果追踪与优化
- **必选组件**: 效果指标定义(打开率/回复率/转化率)、数据采集逻辑、分析报告模板
- **可选组件**: A/B测试结果对比、策略优化建议、自动化调整
- **组装顺序**: 指标定义 → 数据采集 → 效果分析 → 优化建议 → 策略调整
- **核心指标**: 邮件打开率、WhatsApp已读率、回复率、会议转化率
- **格式**: 效果分析报告
