# investor-education-workflow: 投教工作流

基于 LLM Wiki 机制的投教内容生产与分发工作流。融合四专家思维，覆盖五大模块知识库。

## 快速开始

```
/投教 写篇投教文章：定投为什么能对抗择时焦虑？
/投教 解释什么是市盈率（PE），说人话
/投教 做个防骗投教卡片
/投教 客户问："最近科创综指涨得好，要不要追？"
```

## 核心流程

```
触发需求 → ① 检索知识库 → ② 若缺失，定向搜索 → ③ 转译大白话 → ④ 多形式输出 → ⑤ 反馈入库
```

## 五大模块

| 模块 | 内容 | 对应 Skill |
|------|------|-----------|
| 模块 1：投资基本功 | 金融基础/基本面/技术分析/财经素养 | ljg-learn, fund-analyzer-pro |
| 模块 2：市场认知与风险扫描 | 资本市场/风险识别/合规交易 | decision-checklist, content-compliance |
| 模块 3：行为管理 | 行为金融学/常见偏误/策略矫正 | decision-checklist, ljg-relationship |
| 模块 4：投资规划与生命周期 | 财务规划/资产配置/退休规划 | fund-allocator, ljg-rank |
| 模块 5：投资者陪伴与持续成长 | 分层学习/多元形式/日常陪伴 | companion-script, ljg-plain, ljg-card |

## 四专家思维

| 专家 | 视角 | 投教应用 |
|------|------|---------|
| 林奇 | 洞察发掘 | 生活化解释（"买店铺看租金"类比 PE） |
| 卡尼曼 | 行为纠偏 | 识别偏误，建立行为防御 |
| 芒格 | 逆向检查 | "太难"清单，安全边际教育 |
| 马利克 | 系统管理 | 六大账户系统，定期检视 |

## LLM Wiki 架构

```
原始素材层（raw/，不可变）
    ↓ AI 代理编译
Wiki编译层（Wiki/，可变化）
    ↓ 三层检索
应用查询层（问答/检索/内容生成）
```

## 多形式输出

| 形式 | 适用场景 | 制作要求 |
|------|---------|---------|
| 文字消息 | 1 对 1 答复 | 200 字以内，结构：问题→大白话→行动建议 |
| 长图文 | 群发/朋友圈/讲座 | 3-5 个关键点，搭配流程图/对比表 |
| 30 秒语音条 | 快速回复 | 先结论→生活例子→行动 |
| 1 分钟短视频脚本 | 短视频平台 | 开头提问 + 中间转折 + 结尾金句 |
| 知识卡片 | 线下沙龙/面谈 | A6 大小，关键词 + 图示 + 忠告 |

## 示例

### 示例 1：知识点转译

```
用户：客户问"什么是市盈率（PE）？能不能说人话？"
助手：[阶段①] 检索知识库 → 找到官方"一图看懂市盈率"
     [阶段③] 转译大白话 → "PE 就是你买这家公司，靠利润需要多少年回本..."
     [阶段④] 多形式输出 → 文字版 + 图文版 + 语音条
     [阶段⑤] 反馈入库 → 标记"客户接受度高"
```

### 示例 2：投教内容生产

```
用户：写篇投教文章：定投为什么能对抗择时焦虑？
助手：[ljg-learn → 概念解剖]
     [ljg-plain → 白话说]
     [ljg-writes → 深度文章]
     [ljg-card → 可视化卡片]
     [输出 Markdown 文章 + PNG 卡片]
```
