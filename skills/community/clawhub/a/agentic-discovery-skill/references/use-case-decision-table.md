# Discovery Use-Case Decision Table

| Situation | Template | Why |
|---|---|---|
| First-pass lead fit | `quick-qualification-checklist` | Determines whether discovery is worth doing. |
| New opportunity intake | `client-intake-questionnaire` | Gathers baseline business and workflow facts. |
| Live discovery call | `workflow-discovery-script` | Structures the conversation. |
| Multiple roles involved | `stakeholder-interview-guide` | Tailors questions by stakeholder group. |
| Existing process unclear | `current-state-workflow-map` | Documents current inputs, steps, systems, and handoffs. |
| Designing proposed workflow | `future-state-agentic-workflow-map` | Defines reviewed-agent path. |
| Access/data readiness | `systems-and-data-access-checklist` | Checks systems, documents, and permissions. |
| API/tool feasibility | `tool-api-readiness-checklist` | Checks integration constraints. |
| Approval model unclear | `human-approval-gate-checklist` | Identifies decisions requiring human review. |
| Sensitive or regulated concern | `risk-and-excluded-use-checklist` | Captures risk and excluded uses. |
| Buyer asks what success means | `success-criteria-worksheet` | Defines acceptance and outcome measures. |
| Need test cases | `evaluation-example-worksheet` | Captures representative examples. |
| Decide whether to propose pilot | `pilot-readiness-scorecard` | Scores readiness and recommends next step. |
| After discovery | `discovery-recap-memo` | Summarizes findings. |
| Handoff to proposal/SOW | `requirements-brief-for-proposal-sow-handoff` | Packages scope inputs for proposal skill. |
| Missing facts | `assumptions-and-unknowns-log` | Tracks unknowns and assumptions. |
| Ownership unclear | `stakeholder-responsibility-matrix` | Assigns accountable parties. |
| Dependencies unclear | `implementation-dependency-checklist` | Lists blockers and prerequisites. |
| Data sensitivity/retention | `data-sensitivity-and-retention-worksheet` | Captures privacy and retention needs. |
| Several workflows compete | `workflow-prioritization-matrix` | Ranks candidate workflows. |
| Logging/monitoring requirements | `logging-and-monitoring-needs-worksheet` | Captures observability needs. |
| Support/handoff expectations | `support-and-handoff-discovery-worksheet` | Captures post-delivery expectations. |
| Production readiness | `pilot-to-production-readiness-checklist` | Checks rollout requirements. |

Selection rule: choose the artifact that resolves the most important uncertainty blocking a proposal. If workflow fit is unclear, qualify first. If scope is unclear, map current/future state. If risk is unclear, run approval/risk/data readiness checks. If enough facts are known, create the requirements brief.
