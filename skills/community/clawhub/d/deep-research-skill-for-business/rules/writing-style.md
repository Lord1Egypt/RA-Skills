# 调研报告写作风格规范

## 核心原则

### 1. 论述体，非摘抄体

❌ **错误示范**（摘抄体）：
> 根据XX报告，2026年AI市场规模达到5000亿。另外YY机构认为增长率为30%。此外ZZ媒体提到...

✅ **正确示范**（论述体）：
> 2026年AI市场规模达5000亿 [1]。这个数字背后是三个驱动因素：A、B、C。值得注意的是，增长率虽然达到30% [2]，但增速已经开始分化——头部的增速是尾部的3倍。这意味着"普惠AI增长"的时代正在结束，"结构性增长"成为主旋律。

### 2. 故事体纵轴

纵轴（时间线/趋势线）必须是因果链条，不是年表流水账。

❌ **错误示范**：
> 2023年，XX发生。2024年，YY发生。2025年，ZZ发生。

✅ **正确示范**：
> 2023年政策X出台 [3]，行业进入了3年的快速扩张期。但2025年的拐点Y暴露了一个结构性问题：早期增长靠补贴驱动，而非真实的用户需求。2026年行业进入"挤泡沫"阶段，能够证明单位经济模型的企业将胜出。

### 3. 标注推测（事实/分析/判断不标注）

为保持阅读流畅性，**仅推测内容需要标注**，事实陈述、分析推理、判断结论均不标注标签，自然行文即可。

- **事实**：不标注。可验证的客观信息，自然陈述。
- **分析**：不标注。基于事实的推理、对比、因果分析，自然行文。
- **判断**：不标注。综合证据和分析后得出的结论，自然行文。
- **推测**：必须标注，超出证据范围的预判。格式为 `【推测，置信度：高/中/低】`。

示例：
> DeepSeek API 于 2026 年 4 月降至 ¥0.025/百万 Token [8]。这意味着中小 AI 应用商的成本门槛已降至接近零。
> 【推测，置信度：中】如果这一价格趋势持续，2027 年 C 端 AI 管家服务的 token 成本可能低于 ¥5/户/月。

**置信度指南**：
- **高**：有强相关数据或类比案例支撑
- **中**：有部分数据支撑，但存在较大不确定因素
- **低**：凭经验或弱信号推断，定量依据不足

### 4. 数据规范

- 每个数据必须注明：来源机构 + 报告名称 + 具体数值
- 禁止模糊表述：不使用"大幅增长""显著提升""较为明显"
- 替代方案：如果找不到精确数据，说"约X-Y区间（来源：XX，推算值）"或"该数据暂缺"

### 5. 源引用格式

正文使用编号角标引用，来源详细信息集中在文末「来源列表」章节。

```
正文：DeepSeek API 于 2026 年 4 月降至 ¥0.025/百万 Token [8]。
正文：Alexa+ 定价 $19.99/月，Prime 会员免费 [4][5][6][7]——四源印证。

文末「来源列表」章节：
| 编号 | 标题 | URL / 出处 | 信源级别 | 核心贡献 |
|------|------|------------|----------|----------|
| [8] | DeepSeek API 降价公告（2026-04-27） | 证券时报 | T3 | API 最新定价 |
```

- 编号按正文首次出现顺序分配 [1] [2] [3]...
- 同一来源多处引用，使用同一编号
- 多源印证的结论可并列引用 [4][5][6][7]

## 章节语气指南

| 章节 | 语气 | 说明 |
|------|------|------|
| 执行摘要 | 果断、不拖泥带水 | 直接给结论，不说"本文将通过..." |
| 背景与范围 | 中性、事实陈述 | 说"是什么"，不说"应该是什么" |
| 核心发现 | 分析性、有判断力 | 每个发现必须有"为什么重要"的分析 |
| 交汇洞察 | 独到、跨维度 | 不是摘要，是新发现 |
| 建议 | 可操作、具体 | 不说"应继续关注"，说"建议在X月前完成Y" |
| 缺口说明 | 诚实、不掩饰 | 不说"有待进一步研究"，说"以下N个问题数据不足：..." |
