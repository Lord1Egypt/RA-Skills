## 银行信贷生成要求

银行信贷每种任务类型的"零件清单"——必选/可选组件、组装顺序、领域约束。按清单逐项产出。

---

### 1 信贷申请

#### T1-1 客户申请

- **必选组件**: 申请渠道、申请材料、申请流程、申请记录
- **可选组件**: 申请咨询、申请指导、申请系统
- **组装顺序**: 确定申请渠道 → 收集申请材料 → 执行申请流程 → 记录申请信息
- **约束**: 渠道必须畅通；材料必须完整；流程必须规范
- **格式**: 客户申请记录（Markdown）

#### T1-2 资料收集

- **必选组件**: 资料清单、资料审核、资料补充、资料归档
- **可选组件**: 资料模板、资料指导、资料系统
- **组装顺序**: 列出资料清单 → 审核资料 → 补充资料 → 归档资料
- **约束**: 清单必须完整；审核必须严格；归档必须规范
- **格式**: 资料收集记录（Markdown）

#### T1-3 初步审核

- **必选组件**: 审核标准、审核内容、审核结果、审核记录
- **可选组件**: 审核工具、审核系统、审核报告
- **组装顺序**: 确定审核标准 → 审核内容 → 形成结果 → 记录审核
- **约束**: 标准必须明确；内容必须全面；记录必须完整
- **格式**: 初步审核记录（Markdown）

#### T1-4 受理登记

- **必选组件**: 受理条件、登记信息、受理编号、受理确认
- **可选组件**: 受理系统、受理通知、受理记录
- **组装顺序**: 确认受理条件 → 登记信息 → 生成编号 → 确认受理
- **约束**: 条件必须明确；信息必须准确；编号必须唯一
- **格式**: 受理登记表（Markdown）

---

### 2 信贷调查

- **必选组件**: 调查内容、调查方法、调查结果、调查记录
- **可选组件**: 调查工具、调查系统、调查报告
- **组装顺序**: 确定调查内容 → 选择调查方法 → 执行调查 → 记录结果
- **约束**: 内容必须全面；方法必须科学；记录必须完整
- **格式**: 客户调查报告（Markdown）

#### T2-2 资信评估

- **可选组件**: 评估工具、评估模型、评估系统
- **组装顺序**: 确定评估指标 → 选择评估方法 → 执行评估 → 生成报告
- **约束**: 指标必须全面；方法必须科学；报告必须准确
- **格式**: 资信评估报告（Markdown）

#### T2-3 风险分析

- **必选组件**: 风险类型、风险识别、风险评估、风险报告
- **可选组件**: 风险工具、风险模型、风险系统
- **组装顺序**: 确定风险类型 → 识别风险 → 评估风险 → 生成报告
- **约束**: 类型必须全面；识别必须准确；评估必须客观
- **格式**: 风险分析报告（Markdown）

#### T2-4 调查报告

- **必选组件**: 调查结论、风险提示、建议意见、报告说明
- **可选组件**: 报告模板、报告审核、报告系统
- **组装顺序**: 形成调查结论 → 提示风险 → 提出建议 → 撰写说明
- **约束**: 结论必须客观；风险必须提示；建议必须可行
- **格式**: 信贷调查报告（Markdown）

### 3 信贷审批

#### T3-1 审批流程

- **必选组件**: 审批权限、审批节点、审批标准、审批记录
- **可选组件**: 审批系统、审批工具、审批培训
- **组装顺序**: 确定审批权限 → 设计审批节点 → 制定审批标准 → 记录审批过程
- **约束**: 权限必须明确；节点必须合理；标准必须统一
- **格式**: 审批流程文档（Markdown）

#### T3-2 审批决策

- **必选组件**: 决策依据、决策结果、决策理由、决策记录
- **可选组件**: 决策工具、决策模型、决策系统
- **组装顺序**: 分析决策依据 → 做出决策 → 阐述理由 → 记录决策
- **约束**: 依据必须充分；结果必须明确；理由必须合理
- **格式**: 审批决策记录（Markdown）

#### T3-3 合同签订

- **必选组件**: 合同条款、担保条款、违约条款、签订流程
- **可选组件**: 合同模板、合同审核、合同系统
- **组装顺序**: 拟定合同条款 → 设计担保条款 → 制定违约条款 → 执行签订流程
- **约束**: 条款必须合法；担保必须有效；流程必须规范
- **格式**: 信贷合同（Markdown）

#### T3-4 放款操作

- **必选组件**: 放款审核、资金划转、放款确认、放款记录
- **可选组件**: 放款系统、放款工具、放款监控
- **组装顺序**: 执行放款审核 → 划转资金 → 确认放款 → 记录放款
- **约束**: 审核必须严格；划转必须准确；确认必须及时
- **格式**: 放款操作记录（Markdown）
