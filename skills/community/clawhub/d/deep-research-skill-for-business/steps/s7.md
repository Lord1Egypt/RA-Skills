# S7: 展开论述

## 目标

以"问题 → 证据 → 分析 → 判断"的论证逻辑，对每个子问题撰写论述。**不是搜索摘要的堆砌，而是有分析的研究论述。**

## ⛔ 红线

- 每个子问题必须有明确的"论证链"：问题 → 证据 → 分析 → 判断
- 禁止"摘要堆砌"——如果一段话只是重复搜索结果，重写
- **仅推测内容需要标注**，事实陈述、分析推理、判断结论均不标注，自然行文即可
- 推测标注格式：`【推测，置信度：高/中/低】`

## 操作

### 1. 论证结构

对每个子问题，按以下结构撰写：

```markdown
### [子问题 N]: [问题陈述]

**论证目标**: [这个问题要回答什么]

#### 证据呈现
来自来源 A（P0）的具体数据/事实，自然行文 [引用编号]。
来自来源 B（P1）的补充证据 [引用编号]。

#### 分析
将证据串联，分析因果、对比差异、揭示规律。自然行文，不标注标签。

#### 判断
基于证据和分析的明确结论。自然行文，不标注标签。
如果证据不足：该问题目前数据不足，结论置信度低，建议后续...
```

### 2. 论证质量自检

写完每个子问题后，问自己：
- 这段话是我**分析**出来的，还是只是**翻译**了搜索结果？
- 如果删除所有搜索引文，这段话还有独立价值吗？
- 结论是否从证据中**自然导出**，而非先有结论再找证据？

### 3. 标记完成

完成后执行：
```bash
bash orchestrator.sh --complete s7 --next s8
```

## 校验标准

- [ ] 每个子问题有完整的"问题→证据→分析→判断"链
- [ ] 内容不是搜索结果的翻译/堆砌
- [ ] 仅推测内容标注了【推测，置信度：X】，事实/分析/判断自然行文无冗余标签
- [ ] 推测标注有明确的证据缺口说明
