# Finance Analysis

财经市场分析工具，整合多个数据源，提供技术面 + 基本面 + 情绪面综合分析。

## ⚠️ 风险提示与免责

**本技能仅供参考，不构成任何投资建议。**

- **数据局限**: 本技能通过 web_fetch 抓取公开网页数据，可能存在延迟、错误或不完整。网页抓取行为可能违反某些站点的服务条款。
- **不保证准确性**: 所有数据、指标、观点均来自第三方公开来源，本技能不对其准确性、时效性、完整性负责。
- **不执行交易**: 本技能仅提供信息分析，不执行任何买卖操作，不代持资产，不涉及资金划转。
- **用户自负风险**: 任何基于本技能输出做出的投资决策，风险由用户自行承担。投资有风险，入市需谨慎。
- **非专业建议**: 本技能不能替代专业投资顾问、财务规划师或持牌金融机构的专业服务。

## 核心原则

1. 先确认标的，再分析。
2. 先跑完必需数据源，再写结论。
3. 数据缺失必须明示，不允许脑补。
4. 结论强度必须匹配证据完整度。
5. 默认给出可操作区间，而不是空泛观点。
6. **返回语言交给模型根据用户表达自然判断，优先顺着用户的语气和主要语言回复，不要被 ticker、指标缩写或外文数据源带偏。**
7. **不要把“拿到了某个网页”误判成“拿到了该指标”。**

## 标的确认

用户描述可能同时指向多种标的时，先判断并在回答里明确说明你按哪一种分析：
- 个股
- ETF
- 指数
- 现货/期货/大宗商品

如果存在歧义，默认采用最贴近用户语义的标的，并在首段注明；必要时提示可补充更具体代码。

## 强制执行门禁

### A. 必跑数据源
只要用户要求”分析走势 / 行情 / 是否能买 / 怎么看”，默认尽量补齐以下项目（主要面向美股市场）：
- CNBC 行情
- CNBC `.VIX`
- CurrentMarketValuation（巴菲特指数/市场估值）
- CNN Fear & Greed
- Benzinga
- 老虎社区
- 金十数据

**注意**: 以上必跑数据源主要覆盖美股市场。A股、港股分析需补充对应数据源（详见「参考资料 → 数据源详细说明」）。

### B. 输出前验收
写答案前先在脑中完成这张验收表：
- 行情：成功 / 失败
- RSI：成功 / 失败
- VIX：成功 / 失败
- 巴菲特指数：成功 / 失败
- CNN 恐惧贪婪：成功 / 失败
- 中文观点：成功 / 失败
- 快讯：成功 / 失败

只要有失败：
- 说明失败项
- 说明失败原因
- 不允许用其他来源“代替事实”假装补齐

### C. 先列 checklist，再动手
遇到新问题时，先内部按以下顺序执行，再生成报告：
1. 标的识别
2. 行情概览
3. 技术面
4. 情绪面
5. 基本面
6. 新闻/中文观点
7. 综合建议

### D. 数值成功标准
只有抓到**具体可引用的数值或明确结论**，才算成功。
以下情况一律不算成功：
- 只打开了页面
- 只抓到导航/说明文字
- 只抓到很长正文但没有目标指数数值
- 只能凭常识猜测
- 页面标题看起来对，但正文没有可引用数据

### E. 备选与降级
- CNBC 行情失败：可用 Benzinga 补行情主数据，但要明说是备选源。
- Benzinga 失败或只返回新闻标题：RSI/评级/目标价算失败，不得脑补。
- CNN / CurrentMarketValuation 只拿到页面说明、不含数值：算失败，不算成功。
- 老虎社区只作为情绪参考，不作为硬数据源。
- 金十数据主要用于新闻催化和宏观背景，不代替行情指标。

### F. 标的类型适配
- 个股：优先看价格、涨跌、成交量、RSI、估值、盈利、评级、目标价。
- ETF / 基金：优先看价格、涨跌、成交量、AUM、费率、跟踪对象、波动特征、持仓结构；没有 PE/盈利/目标价时不要强套。
- 大宗商品代理（如黄金 ETF）：把 ETF 自身数据和商品宏观驱动分开写，不要混成“普通公司基本面”。

## 分析流程

### Step 1: 行情概览
优先提取：
- 当前价
- 涨跌幅
- 当日高低点
- 52周高低点
- 成交量 vs 均量
- 市值、PE、股息率（若有）

### Step 2: 技术面
优先提取 Benzinga RSI。

**RSI 参考（快速3段规则）**
- > 70：超买
- 30-70：中性
- < 30：超卖

**注**: 本技能使用简化的3段RSI规则。更细粒度的6段划分（>80极度超买、70-80超买、50-70偏强、30-50偏弱、20-30超卖、<20极度超卖）详见「参考资料 → 技术分析详解」。

支撑/压力优先从以下位置推断：
- 52周高低点
- 当日高低点
- 整数关口
- 前高/前低

### Step 3: 情绪面
必须尽量拉取：
- VIX
- 巴菲特指数
- CNN Fear & Greed
- 中文讨论热度 / 观点

**快速判断**
- VIX 高 + Fear & Greed 低：偏恐慌
- VIX 低 + Fear & Greed 高：偏乐观 / 警惕过热
- 情绪与价格背离：重点提示风险

### Step 4: 基本面
适用于个股/ETF/产业链标的。
优先看：
- PE / Forward PE
- 盈利能力
- 成长性
- 债务负担
- 分析师共识与目标价

### Step 5: 新闻与观点
从金十、老虎等提炼：
- 影响价格的宏观事件
- 市场是否在交易地缘、利率、通胀、供需
- 散户讨论是否一致过热

### Step 6: 综合结论
必须包含：
- 趋势判断：看多 / 震荡偏强 / 震荡 / 偏弱 / 看空
- 关键支撑
- 关键压力
- 风险点
- 适合追涨、回踩、还是观望

## 本技能的局限

本技能通过 web_fetch 抓取公开网页数据，存在以下技术限制：

- **无法获取历史K线数据**: web_fetch 只能获取当前页面的即时数据，无法拉取多日历史价格序列。
- **不支持复杂技术指标**: MACD、布林带、K线形态等需要历史数据计算的指标无法直接提供。如需这些指标，建议使用 TradingView、同花顺等专业图表工具。
- **可用技术指标**: RSI（Benzinga 直接提供）、支撑/压力位（基于52周高低点等静态数据）、成交量对比。

详见「参考资料 → 技术分析详解」末尾说明。

## 失效处理

如果某个数据源抓取失败：
1. 先换备选源
2. 仍失败则明确写“失败”
3. 不允许用常识补洞当作事实
4. 不能把“页面打开了”当成“数据拿到了”

## 额外提示

- 对“黄金走势”“原油走势”“大宗商品走势”这类问题，优先当作宏观/商品分析处理，而不是误判成普通股票。
- 如果用户没给代码，必要时先按最常见标的分析，并在回答里说明假设。
- 当数据明显不完整时，结论要保守。
- 分析基金/ETF时，如果没有 PE、盈利等公司型指标，不要硬套个股框架；改看 AUM、费率、跟踪对象、成交量、折溢价/波动特征等可得信息。
