# Copy Template

## 结构

```
[主标题/Hook]
[副标题/价值主张，1行]

[引言段落，1句话设定场景]

[卖点1] emoji + 核心卖点描述
[卖点2] emoji + 核心卖点描述
[卖点3] emoji + 核心卖点描述
（如有模块信息，模块名称和描述必须逐字匹配）

[价格区块] 大字价格 + 原价划线（如无价格则省略此区块）

[行动召唤] 私信/留言咨询

[话题标签] #话题1 #话题2 #话题3（如无则省略）
```

## 规则

| 规则 | 说明 |
|------|------|
| 标题 | ≤20字，有力有温度 |
| 卖点 | 每条一行，以动词开头或利益导向 |
| 模块名称/描述 | **必须逐字对应用户提供的原始内容**，禁止改写 |
| 标点 | 中文标点，不用句号结尾卖点行 |
| Emoji | 每个卖点一个，相关性强 |
| 价格 | 数字要大，划线原价要小（如无价格则省略） |

**CTA 规则**：
- **图片**（有二维码）：CTA = "扫码咨询"
- **图片**（无二维码 + 有联系方式）：底部显示联系方式，无CTA文字
- **图片**（无二维码 + 无联系方式）：**省略整个CTA区**，不留任何占位
- **朋友圈文案**：始终有CTA，"私信咨询" / "欢迎私信"
- 两者 CTA 逻辑独立，copy 永远有CTA，图片CTA视情况而定

**朋友圈文案 CTA 参考**：
- ✅ "私信咨询，领取课程大纲"
- ✅ "欢迎私信，了解更多"
- ✅ "评论区留言，1对1答疑"

## 交互要求（核心规范）

**在收集用户输入时，必须逐项提问，不得跳过或假设：**

| 必问项 | 提问方式 | 如用户不提供 |
|--------|---------|-------------|
| 产品/服务主题 | "请提供产品或服务名称" | 生成通用主题 |
| 核心卖点（≥3） | "请提供3-5个核心卖点（每行一个）" | 减少卖点数量，生成简洁版本 |
| 价格信息 | "原价和优惠价分别是？" | **省略价格区块**，不在图片预留位置 |
| 讲师/专家 | "是否需要展示讲师信息？" | 省略讲师区域 |
| 二维码 | "是否需要展示个人微信二维码？" | **省略二维码区域**，不在图片预留位置 |
| 图片尺寸 | "图片比例？9:16 / 3:4 / 1:1" | 默认 9:16 |
| 风格 | "图片风格？minimal / notion / warm / blueprint" | 默认 minimal |

**交互时机**：非交互模式（参数全提供）时直接生成；否则必须先收集再生成，禁止用"扫码占位"等替代缺失信息。

## 图片文案一致性规则（关键）

**图片中的文字必须与 copy.md 中的文字100%一致。**

生成图片 prompt 时：
1. 从 copy.md 提取**逐字原文**，逐条填入对应 zone
2. 不得改写、缩写、意译任何卖点文字
3. 如 copy 中某区块省略（如无价格），图片对应 zone 也省略
4. 模块名称必须与用户原始输入完全匹配

**错误示例**（图片文案≠copy）：
- Copy: "内容营销AI全流程制作发布"
- Image: "AI可能的内容营销制片与发布" ❌

## 风格指南

| 风格 | 适用场景 | 文案调性 |
|------|---------|---------|
| 专业权威 | B端、企业采购 | 严谨、数据驱动 |
| 亲和友好 | C端、个人用户 | 温暖、有趣 |

## 禁忌

- ❌ 图片CTA使用"私信"（图片有二维码，应写"扫码"）
- ❌ "绝对化"词语（100%、一定、保证）
- ❌ 过多感叹号（!!!显得急躁）
- ❌ 堆砌卖点（最多5条）
- ❌ 图片文案改写用户原始输入
