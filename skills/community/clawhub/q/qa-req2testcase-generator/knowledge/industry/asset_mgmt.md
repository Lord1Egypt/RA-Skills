# 资管域（Asset Management Domain）

> 本文档覆盖证券资管核心业务对象的状态机、操作规则、异常模式与测试点，适用于公募/私募基金及资管产品场景。

---

## 产品申购赎回（Fund Subscription/Redemption）

### 状态全集
- 申请已提交（Submitted）：用户提交申购/赎回申请，等待系统受理
- 已受理（Accepted）：系统校验通过，申请进入处理队列
- 份额确认中（Confirming）：基金公司/TA 系统正在确认份额
- 已确认（Confirmed）：份额确认完成，申购份额到账或赎回资金到账（终态）
- 已拒绝（Rejected）：申请被拒绝（限额、黑名单等）（终态）
- 已撤销（Cancelled）：用户在确认前撤销申请（终态）
- 部分确认（PartialConfirmed）：大额赎回时部分确认、部分顺延

### 状态转移规则
| 当前状态 | 触发条件 | 目标状态 | 是否允许 |
|---------|---------|---------|--------|
| 申请已提交 | 系统校验通过 | 已受理 | ✅ |
| 申请已提交 | 校验不通过（限额/黑名单） | 已拒绝 | ✅ |
| 申请已提交 | 用户撤销 | 已撤销 | ✅ |
| 已受理 | 发送至 TA 系统确认 | 份额确认中 | ✅ |
| 已受理 | 用户在截止时间前撤销 | 已撤销 | ✅ |
| 份额确认中 | TA 确认成功 | 已确认 | ✅ |
| 份额确认中 | TA 拒绝 | 已拒绝 | ✅ |
| 份额确认中 | 大额赎回触发比例确认 | 部分确认 | ✅ |
| 部分确认 | 顺延部分在下一交易日确认 | 已确认 | ✅ |
| 已确认 | 撤销 | — | ❌ |
| 已拒绝 | 重新提交 | — | ❌（需新建申请） |

### 允许操作
- 提交申购申请：账户已开通基金交易权限，资金充足，在申购开放期内
- 提交赎回申请：持有该基金份额，在赎回开放期内
- 撤销申请：申请处于「申请已提交」或「已受理」状态，且未超过当日截止时间（通常 15:00）
- 查询申请状态和份额：任意时刻

### 禁止操作（反例知识）
- 在封闭期内提交申购/赎回：封闭式基金或定开基金封闭期内不接受申赎
- 申购金额低于最低限额：每只基金有最低申购金额（如 1000 元），低于限额应拒绝
- 赎回份额超过持有份额：系统必须校验可赎回份额（扣除冻结份额）
- 15:00 后提交的申请按下一交易日处理：必须明确告知用户适用的净值日期

### 异常模式
- 大额赎回：单日赎回申请超过基金净资产 10%，触发大额赎回处理 → 测试点：验证大额赎回的比例确认机制（按比例确认+剩余顺延 vs 全额顺延 vs 暂停赎回）
- TA 系统超时：份额确认请求发送至 TA 系统后无响应 → 测试点：验证 T+2 仍未收到确认时的超时处理和用户通知
- 净值异常：确认日净值计算异常导致份额计算错误 → 测试点：验证净值异常时的申赎暂停机制
- 重复提交：用户重复提交相同的申购申请 → 测试点：验证防重机制（同一用户+同一基金+同一交易日+相同金额）
- 巨额赎回导致流动性危机：基金持仓资产无法快速变现 → 测试点：验证暂停赎回的触发条件和公告机制

### 测试点模板
- TP-AM-001：提交申购申请 10000 元，验证 T+1 确认份额=申购金额/(T日净值×(1+申购费率))（优先级：P0）
- TP-AM-002：提交赎回申请 1000 份，验证 T+1 确认赎回金额=赎回份额×T日净值×(1-赎回费率)（优先级：P0）
- TP-AM-003：15:00 后提交申购申请，验证系统提示按 T+1 日净值确认（优先级：P0）
- TP-AM-004：单日赎回超过基金净资产 10%，验证触发大额赎回处理（优先级：P0）
- TP-AM-005：封闭期内提交赎回，验证系统拒绝并提示封闭期截止日期（优先级：P1）
- TP-AM-006：TA 系统超时 48h 未确认，验证系统自动标记异常并通知运营（优先级：P1）

### 中间态处理规则
- 「份额确认中」为中间态，通常持续 T+1 到 T+2 个交易日
- 「部分确认」为中间态，顺延部分在下一交易日继续确认
- 中间态期间资金/份额处于冻结状态，不可用于其他操作
- 超过 T+3 仍未确认，系统自动告警并通知基金运营

### 幂等/重试规则
- 申赎申请使用 userId + fundCode + tradeDate + applySeqNo 作为幂等键
- TA 确认请求使用 applyNo + confirmDate 作为幂等键
- TA 通信超时后重试：最多 3 次，间隔 5min/15min/30min
- 重试前检查是否已收到确认结果（可能是回报延迟而非丢失）

## 净值计算（NAV Calculation）

### 状态全集
- 待计算（PendingCalc）：交易日收盘后，等待启动净值计算
- 数据采集中（DataCollecting）：正在采集持仓、行情、费用等数据
- 计算中（Calculating）：正在执行净值计算（资产估值、费用计提、份额计算）
- 待复核（PendingReview）：净值计算完成，等待基金会计复核
- 已发布（Published）：净值已复核通过并对外发布（终态）
- 计算异常（CalcError）：计算过程中发现异常（数据缺失、估值偏差等）
- 已修正（Corrected）：异常净值已修正并重新发布（终态）

### 状态转移规则
| 当前状态 | 触发条件 | 目标状态 | 是否允许 |
|---------|---------|---------|--------|
| 待计算 | 定时任务触发（通常 16:00） | 数据采集中 | ✅ |
| 数据采集中 | 数据采集完成 | 计算中 | ✅ |
| 数据采集中 | 数据源异常 | 计算异常 | ✅ |
| 计算中 | 计算完成 | 待复核 | ✅ |
| 计算中 | 估值偏差超阈值 | 计算异常 | ✅ |
| 待复核 | 基金会计确认 | 已发布 | ✅ |
| 待复核 | 基金会计驳回 | 计算异常 | ✅ |
| 计算异常 | 修复后重新计算 | 数据采集中 | ✅ |
| 已发布 | 发现错误需修正 | 计算异常 | ✅ |
| 计算异常 | 修正完成并发布 | 已修正 | ✅ |
| 已发布 | 直接修改净值 | — | ❌（必须走修正流程） |

### 允许操作
- 触发净值计算：交易日收盘后，由定时任务或基金会计手动触发
- 复核净值：基金会计审核计算结果，对比估值表
- 发布净值：复核通过后发布至信息披露系统
- 修正净值：发现错误后走修正流程，需审批

### 禁止操作（反例知识）
- 跳过复核直接发布净值：净值发布必须经过基金会计复核
- 直接修改已发布的净值：必须走修正流程，留审计痕迹
- 在数据不完整时强制计算：缺少行情数据或持仓数据时应阻止计算

### 异常模式
- 行情数据缺失：部分持仓标的无收盘价（停牌等） → 测试点：验证停牌标的的估值方法（使用最近交易日收盘价或指数收益法调整）
- 估值偏差：计算结果与前一日净值偏差超过阈值（如 ±2%） → 测试点：验证偏差告警机制和人工确认流程
- 费用计提错误：管理费/托管费计提基数或比例错误 → 测试点：验证费用计提公式（日管理费=前一日资产净值×年管理费率/当年天数）
- 份额变动未同步：当日申赎份额未及时同步至净值计算系统 → 测试点：验证净值计算前的份额数据完整性校验
- 汇率波动：QDII 基金涉及外币资产估值 → 测试点：验证汇率数据源和折算时点的准确性

### 测试点模板
- TP-AM-010：交易日收盘后触发净值计算，验证 NAV = (总资产-总负债)/总份额，精度到小数点后 4 位（优先级：P0）
- TP-AM-011：持仓中有停牌标的，验证使用指数收益法调整估值（优先级：P0）
- TP-AM-012：净值与前一日偏差超过 2%，验证系统触发告警并暂停发布（优先级：P0）
- TP-AM-013：验证管理费计提：日管理费=前一日NAV×年费率/365，精度到分（优先级：P1）
- TP-AM-014：已发布净值发现错误，验证修正流程：修正申请→审批→重新计算→重新发布（优先级：P1）
- TP-AM-015：模拟行情数据源不可用，验证净值计算暂停并告警（优先级：P1）

### 中间态处理规则
- 「数据采集中」「计算中」为短暂中间态，应在 30min 内完成
- 「待复核」可持续数小时，但必须在 T+1 日 9:00 前完成发布
- 「计算异常」需人工介入，无自动超时机制，但应在 T+1 日开盘前解决
- 净值计算过程中需加排他锁，防止并发计算

### 幂等/重试规则
- 净值计算以 fundCode + navDate 作为唯一键
- 重复触发同一交易日的净值计算，返回已有结果或覆盖重算（取决于当前状态）
- 数据采集支持断点续传，已采集的数据源不重复拉取
- 净值发布接口幂等，重复发布相同净值不产生副作用

## 资管产品全生命周期管理

### 业务背景
证券公司资产管理业务是指券商接受客户委托，以客户资产进行投资管理的业务。资管新规后，券商资管全面转向净值化管理，打破刚性兑付，回归"受人之托、代人理财"本源。

### 产品生命周期状态全集
- 产品设计（Designing）：产品方案设计阶段
- 内部审批（InternalApproval）：产品方案内部审批
- 监管报备/审批（RegulatoryFiling）：向监管机构报备或申请审批
- 募集期（Fundraising）：产品公开或定向募集资金
- 成立（Established）：募集完成，产品正式成立
- 开放运作（OpenRunning）：产品正常运作，开放申赎
- 封闭运作（ClosedRunning）：产品在封闭期内运作
- 清算中（Liquidating）：产品到期或触发清算条件，进入清算
- 已清算（Liquidated）：清算完成，资金分配给投资者（终态）
- 募集失败（FundraisingFailed）：募集期未达到成立条件（终态）

### 状态转移规则
| 当前状态 | 触发条件 | 目标状态 | 是否允许 |
|---------|---------|---------|--------|
| 产品设计 | 方案完成提交审批 | 内部审批 | ✅ |
| 内部审批 | 审批通过 | 监管报备/审批 | ✅ |
| 内部审批 | 审批驳回 | 产品设计 | ✅ |
| 监管报备/审批 | 监管通过/备案完成 | 募集期 | ✅ |
| 募集期 | 募集达标（金额/人数） | 成立 | ✅ |
| 募集期 | 募集期满未达标 | 募集失败 | ✅ |
| 成立 | 开放期开始 | 开放运作 | ✅ |
| 成立 | 封闭期开始 | 封闭运作 | ✅ |
| 封闭运作 | 封闭期结束 | 开放运作 | ✅ |
| 开放运作 | 到期/触发清算条件 | 清算中 | ✅ |
| 封闭运作 | 到期/触发清算条件 | 清算中 | ✅ |
| 清算中 | 清算完成 | 已清算 | ✅ |
| 已清算 | 重新运作 | — | ❌ |
| 封闭运作 | 直接接受赎回 | — | ❌ |

### 允许操作
- 产品设计：设定投资策略、费率结构、风控指标、开放规则
- 募集：向合格投资者募集资金，发送募集公告
- 申购/赎回：开放期内接受投资者申购和赎回
- 分红：根据产品合同约定进行收益分配
- 清算：到期或触发条件后进行资产变现和资金分配

### 禁止操作（反例知识）
- 封闭期内接受赎回：封闭期产品不接受赎回申请（定开产品除外）
- 未经审批发行产品：产品必须完成内部审批和监管报备
- 超规模募集：募集金额超过产品合同约定的上限
- 向不合格投资者销售：私募资管产品仅面向合格投资者

## 公募资管

### 大集合资管计划
- 定义：参与人数≥200人或募集金额≥50亿的集合资管计划
- 转型要求：资管新规后，存量大集合需转型为公募基金或清盘
- 审批流程：参照公募基金审批，需证监会核准
- 信息披露：参照公募基金标准，定期报告+临时公告

### 公募REITs
- 定义：公开募集基础设施证券投资基金
- 底层资产：基础设施项目（产业园区、仓储物流、高速公路等）
- 结构：公募基金+ABS（资产支持证券）双层结构
- 交易方式：场内交易（二级市场买卖）+场外申赎
- 分红要求：可供分配金额的90%以上强制分红

### 审批流程
- 产品申报：向证监会/基金业协会提交申报材料
- 反馈意见：监管机构提出反馈，需补充材料
- 核准/备案：获得核准批文或完成备案
- 发行准备：确定发行日期、销售渠道、托管行

### 信息披露
- 招募说明书：产品成立前发布，包含投资策略、费率、风险揭示
- 定期报告：季报（15个工作日内）、半年报（2个月内）、年报（3个月内）
- 临时公告：重大事项（如基金经理变更、清盘预警）需及时公告
- 净值公告：每个交易日公布前一日净值

### 测试点模板
- TP-AM-020：大集合产品申购，验证参与人数≥200人时的合规校验（优先级：P0）
- TP-AM-021：公募REITs分红，验证可供分配金额的90%以上强制分配（优先级：P0）
- TP-AM-022：信息披露时效，验证季报在15个工作日内发布（优先级：P1）
- TP-AM-023：产品募集超过上限，验证系统自动截止募集（优先级：P0）

## 私募资管

### 定向资管计划
- 定义：券商与单一客户签订合同，为其提供资产管理服务
- 投资者：单一合格投资者
- 投资范围：根据合同约定，可投资股票、债券、衍生品等
- 灵活性：投资策略和限制由双方协商确定

### 集合资管计划
- 定义：券商设立集合资管计划，向多个合格投资者募集
- 投资者门槛：单个投资者≥100万元
- 人数限制：≤200人（超过200人需按公募管理）
- 备案要求：成立后10个工作日内向基金业协会备案

### 合格投资者认定
- 自然人：金融资产≥300万或近3年年均收入≥50万
- 法人/组织：净资产≥1000万
- 穿透核查：通过资管产品/信托等间接投资的，需穿透至最终投资者
- 合并计算：同一资管计划中，同一投资者的多笔投资合并计算

### 备案要求
- 备案时限：产品成立后10个工作日内
- 备案材料：产品合同、风控制度、投资者适当性材料
- 备案变更：重大事项变更需在5个工作日内更新备案
- 注销备案：产品清算后20个工作日内注销

### 测试点模板
- TP-AM-030：私募集合计划申购，验证单个投资者≥100万元门槛（优先级：P0）
- TP-AM-031：集合计划参与人数达200人，验证系统拒绝新增投资者（优先级：P0）
- TP-AM-032：合格投资者认定，验证金融资产≥300万的校验逻辑（优先级：P0）
- TP-AM-033：产品成立后备案，验证10个工作日时限提醒（优先级：P1）
- TP-AM-034：穿透核查，验证通过FOF间接投资的投资者人数合并计算（优先级：P1）

## 投资组合管理

### 资产配置
- 大类资产：股票、债券、货币市场工具、衍生品、另类资产
- 配置比例：根据产品合同约定的投资比例限制
- 再平衡：定期或触发式再平衡（偏离度超过阈值）
- 行业/个股集中度：单一行业/个股持仓比例限制

### 风控指标
- 杠杆率：总资产/净资产，不超过合同约定上限（通常140%）
- 集中度：单一标的持仓不超过产品净值的一定比例
- 流动性：高流动性资产占比不低于一定比例
- 信用风险：债券投资的信用评级下限
- 回撤控制：最大回撤不超过合同约定阈值

### 绩效归因
- 收益分解：总收益 = 资产配置收益 + 个股选择收益 + 交互效应
- 基准比较：相对基准的超额收益（Alpha）
- 风险调整收益：夏普比率、信息比率、最大回撤
- 归因频率：日度计算，月度/季度报告

### 异常模式
- 投资比例超限：市场波动导致被动超限 → 测试点：验证被动超限的宽限期处理（通常10个交易日内调整）
- 风控指标触发：杠杆率/集中度超过预警线 → 测试点：验证风控预警的实时性和处置流程
- 估值偏差：组合估值与托管行估值不一致 → 测试点：验证估值核对机制和差异处理

### 测试点模板
- TP-AM-040：投资组合单一股票持仓超过净值10%，验证系统预警（优先级：P0）
- TP-AM-041：杠杆率超过140%，验证系统禁止新增杠杆操作（优先级：P0）
- TP-AM-042：被动超限后10个交易日内未调整，验证系统升级告警（优先级：P1）
- TP-AM-043：绩效归因计算，验证收益分解公式的准确性（优先级：P1）

## 估值核算

### 净值计算规则
- 计算频率：每个交易日收盘后计算
- 计算公式：单位净值 = (总资产 - 总负债) / 总份额
- 精度要求：单位净值保留4位小数，四舍五入
- 估值时点：以交易日收盘时的市场价格为准

### 费用计提
- 管理费：按日计提，日管理费 = 前一日NAV × 年管理费率 / 当年天数
- 托管费：按日计提，日托管费 = 前一日NAV × 年托管费率 / 当年天数
- 业绩报酬：超过业绩基准部分按约定比例提取（通常20%）
- 销售服务费：按日计提，从基金资产中扣除
- 交易费用：实际发生时计入成本

### 分红派息
- 分红条件：产品有可分配利润且符合合同约定
- 分红方式：现金分红（默认）或红利再投资
- 分红频率：根据合同约定（年度/季度/不定期）
- 除息日处理：除息日净值相应下调
- 税务处理：个人投资者暂免征收所得税（公募基金）

### 异常模式
- 估值日无行情：节假日或停牌标的无收盘价 → 测试点：验证非交易日不计算净值，停牌标的使用最近收盘价或指数收益法
- 费用计提错误：闰年天数计算错误导致费用偏差 → 测试点：验证闰年/平年的天数处理（365 vs 366）
- 大额赎回影响净值：大额赎回导致被迫卖出资产影响净值 → 测试点：验证大额赎回的净值计算时点和公平性
- 分红后净值异常：分红除息后净值计算错误 → 测试点：验证除息日净值=前一日净值-每份分红金额

### 测试点模板
- TP-AM-050：净值计算，验证单位净值=(总资产-总负债)/总份额，精度4位小数（优先级：P0）
- TP-AM-051：管理费计提，验证日管理费=前一日NAV×年费率/365（平年）或/366（闰年）（优先级：P0）
- TP-AM-052：现金分红，验证除息日净值=前一日净值-每份分红金额（优先级：P0）
- TP-AM-053：业绩报酬计提，验证超额收益×20%的计算准确性（优先级：P1）
- TP-AM-054：红利再投资，验证再投资份额=分红金额/除息日净值（优先级：P1）

## 监管要求

### 资管新规核心要求
1. **净值化管理**：产品净值按公允价值计量，不得保本保收益
2. **打破刚兑**：不得承诺保本保收益，不得刚性兑付
3. **去通道**：禁止通道业务，回归主动管理
4. **降杠杆**：固收类140%、权益类120%、混合类140%的杠杆上限
5. **去嵌套**：资管产品最多嵌套一层（公募基金除外）
6. **期限匹配**：封闭式产品期限不得低于90天
7. **集中度限制**：单一标的投资比例限制

### 信息披露要求
1. **真实准确**：信息披露内容不得有虚假记载、误导性陈述或重大遗漏
2. **及时性**：定期报告和临时公告需在规定时限内发布
3. **公平性**：同时向所有投资者披露，不得选择性披露
4. **完整性**：披露内容应涵盖产品运作的重要信息

### 投资者保护
1. **适当性管理**：将适当的产品销售给适当的投资者
2. **风险揭示**：充分揭示产品风险，不得夸大收益
3. **冷静期**：私募产品设置不少于24小时的投资冷静期
4. **回访确认**：冷静期满后进行回访确认
