# 结构要求清单

每种写作任务的组件槽位定义，供UTOS内容轴清单法使用。标注 `[待补充]` 的字段需用户根据自身创作风格和平台要求定义。

---

## N1 素材与世界观

### N1-01 题材/流派定位
- **必选组件**: 题材大类（玄幻/仙侠/都市/言情/悬疑/科幻/历史/游戏等）、细分流派（如玄幻下的系统流/重生流/无敌流/种田流）、目标读者群定位、竞品对标作品(1-3部)、差异化卖点声明
- **可选组件**: 流派趋势分析、跨流派混搭策略、平台偏好匹配度评估
- **组装顺序**: 题材大类→细分流派→读者群→竞品对标→差异化定位
- **平台约束**: 不同平台的流量倾斜不同（起点偏玄幻/都市，晋江偏言情/耽美，番茄偏系统/无敌流），选题需考虑平台匹配度
- **格式**: Markdown

### N1-02 世界观设定文档
- **必选组件**: 世界基本规则（物理/魔法/科技）、地理格局（大陆/国家/城市）、历史时间线（关键事件）、文化/宗教/社会体系、核心矛盾来源
- **可选组件**: 语言系统、货币体系、历法、种族设定
- **组装顺序**: 基本规则→地理→历史→文化→矛盾→补充体系
- **平台约束**: 涉及宗教/政治/色情元素需注意平台审核红线；玄幻/仙侠世界观的暴力尺度需匹配目标平台
- **格式**: Markdown

### N1-03 力量/修炼体系设计
- **必选组件**: 等级名称与划分（至少5级）、每级晋升条件、战力表现特征、天花板设定、获取途径
- **可选组件**: 副职业体系、特殊能力变异、体系漏洞/禁忌
- **组装顺序**: 总览→等级详情→晋升路径→战力表现→边界约束→变体
- **平台约束**: 避免过于复杂导致读者理解门槛过高
- **格式**: Markdown + 表格

### N1-04 时代背景调研
- **必选组件**: 时代名称、政治体制、经济水平、技术水平、社会风气
- **可选组件**: 流行文化、服饰风格、饮食特色、日常用语
- **组装顺序**: 基础框架→政治经济→社会生活→细节填充
- **平台约束**: 无特殊
- **格式**: Markdown

### N1-05 原型档案库
- **必选组件**: 角色原型分类、情节原型分类、场景原型分类
- **可选组件**: 每个原型的经典作品参考、变体方向
- **组装顺序**: 分类框架→逐类填充→交叉索引
- **平台约束**: 无特殊
- **格式**: Markdown表格

### N1-06 文献/资料笔记
- **必选组件**: 来源标题/URL、核心摘录、关联任务标签
- **可选组件**: 个人批注、灵感触发点
- **组装顺序**: 按主题分组→按优先级排序
- **平台约束**: 注意版权，不可直接引用大段原文
- **格式**: Markdown

### N1-07 灵感板
- **必选组件**: 灵感类型（视觉/听觉/概念）、描述文字
- **可选组件**: 图片链接、音乐参考、关联角色/场景
- **组装顺序**: 按类型分栏→按使用场景排序
- **平台约束**: 图片需有版权或为原创
- **格式**: Markdown

### N1-08 素材优先级矩阵
- **必选组件**: 四象限分类（紧急重要/紧急不重要/不紧急重要/不紧急不重要）
- **可选组件**: 完成截止时间、关联章节号
- **组装顺序**: 全量列出→四象限归类→排序输出
- **平台约束**: 无特殊
- **格式**: 表格

## N2 架构与大纲

### N2-01 故事核心宣言
- **必选组件**: 一句话故事概括、核心冲突陈述、主题表达
- **可选组件**: 对标作品参照、差异化定位
- **组装顺序**: 概括→冲突→主题→定位
- **平台约束**: 用于简介和推广素材的源头
- **格式**: Markdown

### N2-02 三幕结构设计
- **必选组件**: 第一幕建置（世界观+主角现状+激励事件）、第二幕对抗（中点转折+危机时刻）、第三幕解决（高潮对决+新常态）
- **可选组件**: 幕间过渡策略、各幕字数比例
- **组装顺序**: 一幕总览→逐幕展开→比例分配→检查平衡
- **平台约束**: 网文第一幕需在3章内完成以符合"黄金三章"
- **格式**: Markdown

### N2-03 故事大纲
- **必选组件**: 核心冲突一句话、主线起因→发展→高潮→结局摘要、主要转折点标注、总字数预估
- **可选组件**: 支线概要、情感弧线摘要、伏笔位置标记
- **组装顺序**: 核心声明→主线展开→支线穿插→转折标注→统计汇总
- **平台约束**: 大纲应覆盖全书80%以上的关键事件
- **格式**: Markdown

### N2-04 英雄之旅映射
- **必选组件**: 12阶段对应的具体故事事件、缺失阶段说明及理由
- **可选组件**: 与三幕结构的对照表
- **组装顺序**: 12阶段逐一映射→缺失分析→结构对照
- **平台约束**: 不强制所有阶段都存在，缺失需有合理理由
- **格式**: Markdown表格

### N2-05 章纲设计
- **必选组件**: 章序号/标题、本章核心事件、出场人物列表、情绪基调、悬念钩子
- **可选组件**: 字数目标、场景数、伏笔埋设/回收标记、爽点/泪点标记
- **组装顺序**: 全书章目总览→逐章填写→依赖关系检查→节奏校准
- **平台约束**: 每章建议3000-5000字（视平台而定），章纲需覆盖全部计划章节
- **格式**: Excel/Markdown表格

### N2-06 时间线设计
- **必选组件**: 故事起始时间、关键事件时间戳、并行时间轴（如有多线叙事）
- **可选组件**: 年龄变化追踪、季节/天气同步
- **组装顺序**: 起始锚定→主事件串行→并行核对→一致性验证
- **平台约束**: 无特殊
- **格式**: Markdown/甘特图式表格

### N2-07 视角策略
- **必选组件**: 主视角选择及理由、视角切换规则（如有）、信息隐藏策略
- **可选组件**: 多视角分工表、视角切换信号词
- **组装顺序**: 选择决策→规则制定→切换方案→隐藏策略
- **平台约束**: 网文通常采用单一第三有限视角，多视角需谨慎
- **格式**: Markdown

### N2-08 叙事结构选择
- **必选组件**: 选定结构类型、选择理由、优势/风险分析
- **可选组件**: 备选方案对比
- **组装顺序**: 方案列举→评估→选定→风险预案
- **平台约束**: 网文读者偏好线性推进，非线性需有明确引导
- **格式**: Markdown

### N2-09 分卷/分部架构
- **必选组件**: 总卷数、每卷目标字数、每卷核心冲突/目标、卷间过渡方式、卷末收束策略
- **可选组件**: 各卷读者情绪目标、卷标题/代号、分卷里程碑时间表
- **组装顺序**: 总体规模→分卷划分→卷内目标→卷间衔接→节奏分配
- **平台约束**: 单卷不宜过短（<15万字读者感觉不完整）也不宜过长（>50万字节奏容易疲软）；各卷需有独立高潮和收束
- **格式**: Markdown + 表格

## N3 角色体系

### N3-01 主角档案
- **必选组件**: 姓名、年龄、外貌（3个辨识度特征）、性格关键词(3-5个)、核心动机、致命弱点、外在目标、内在需求
- **可选组件**: 口头禅、习惯动作、背景故事摘要、标志性物品
- **组装顺序**: 基础信息→性格→动机目标→弱点→细节填充
- **平台约束**: 主角需有明确的"可共情弱点"，避免完美人设
- **格式**: Markdown卡片

### N3-02 配角档案矩阵
- **必选组件**: 配角姓名、功能定位（推动者/阻碍者/衬托者/信息源）、与主角关系、出场频率估计
- **可选组件**: 性格亮点、退场方式
- **组装顺序**: 功能分类→逐个建档→关系网络检查
- **平台约束**: 配角数量控制在主线能承载范围内（通常≤15个主要配角）
- **格式**: Markdown表格

### N3-03 角色弧光设计
- **必选组件**: 起始状态（信念/行为模式）、触发改变的事件、中间挣扎、转折领悟、终态（信念/行为模式的改变）
- **可选组件**: 弧光失败版本（如果角色没有成长会怎样）
- **组装顺序**: 起点→催化剂→挣扎→顿悟→终点→反事实检验
- **平台约束**: 弧光需在全书长度内自然完成，避免突变
- **格式**: Markdown

### N3-04 人物小传
- **必选组件**: 出身背景、关键童年/青年事件、形成当前性格的关键经历
- **可选组件**: 重要人际关系史、秘密/隐痛
- **组装顺序**: 出生→成长关键节点→性格成型→现状衔接
- **平台约束**: 长度灵活，可在写作过程中逐步完善
- **格式**: Markdown

### N3-05 角色关系图
- **必选组件**: 所有主要角色的两两关系类型和强度
- **可选组件**: 关系演变轨迹、隐性关系
- **组装顺序**: 角色枚举→两两配对→关系标注→冲突检测
- **平台约束**: 关系复杂度需与故事规模匹配
- **格式**: 文本关系矩阵 / Mermaid图

### N3-06 对话语音板
- **必选组件**: 每个主要角色的说话风格描述、至少3句样本对话
- **可选组件**: 口头禅、语气词习惯、沉默时的表现
- **组装顺序**: 角色列表→风格描述→样本对话→差异对比
- **平台约束**: 不同角色的对话必须有可区分性
- **格式**: Markdown

### N3-07 反派动机分析
- **必选组件**: 反派的核心欲望、动机来源（合理化逻辑）、行动手段、与主角的根本对立点
- **可选组件**: 反派视角下的合理性论证、可能的救赎弧线
- **组装顺序**: 动机→来源→手段→对立→合理性自检
- **平台约束**: 反派不能是"为了坏而坏"，需有内在逻辑
- **格式**: Markdown

### N3-08 配角效用矩阵
- **必选组件**: 每个配角对主线推动的贡献评分、对支线的贡献、可合并/删除评估
- **可选组件**: 合并建议方案
- **组装顺序**: 全量评估→贡献打分→低效识别→优化建议
- **平台约束**: 无特殊
- **格式**: 表格

### N3-09 角色风险规划
- **必选组件**: 扁平化风险点、可能导致角色崩坏的情节、补救预案
- **可选组件**: 角色测试场景（极端情况下的反应预测）
- **组装顺序**: 风险识别→原因分析→影响评估→预案制定
- **平台约束**: 此项为质量保障工具，非必须产出物
- **格式**: Markdown

## N4 情节与冲突

### N4-01 主线情节设计
- **必选组件**: 激励事件、第一幕结束的"跨越门槛"、中点转折、"一无所有"时刻、高潮对决、结局
- **可选组件**: 各阶段的情绪基调、对应章节范围
- **组装顺序**: 七要素逐一展开→节奏检查→缺口填补
- **平台约束**: 高潮前需有足够的铺垫积累（网文通常占全书60-70%篇幅）
- **格式**: Markdown

### N4-02 支线情节规划
- **必选组件**: 支线名称/类型、起点、与主线的交汇点、终点
- **可选组件**: 独立完整度、对主线的辅助功能
- **组装顺序**: 列举所有支线→逐条规划→交汇检查→剪枝
- **平台约束**: 支线不应喧宾夺主，交汇时机需精心设计
- **格式**: Markdown

### N4-03 冲突层级设计
- **必选组件**: 内在冲突（主角内心）、人际冲突（主角vs他人）、社会冲突（主角vs体制/环境）、终极对抗（主角vs反派）
- **可选组件**: 冲突升级路径
- **组装顺序**: 内在→人际→社会→终极→升级链
- **平台约束**: 四层不必全有，但至少要有两层
- **格式**: Markdown

### N4-04 伏笔回收表
- **必选组件**: 伏笔内容、埋设章节、计划回收章节、当前状态（已埋/已收/待收）
- **可选组件**: 伏笔等级（明线/暗线）、回收效果预期
- **组装顺序**: 全面梳理→登记→定期更新→发布前清零检查
- **平台约束**: 长篇网文尤其需要此表防止遗忘
- **格式**: Excel/Markdown表格

### N4-05 转折点设计
- **必选组件**: 转折类型（反转/揭示/意外）、转折前的误导信息、真实情况、铺垫位置
- **可选组件**: 读者可能提前猜到的概率评估、备选方案
- **组装顺序**: 类型→真相→误导→铺垫→可猜测性自检
- **平台约束**: 转折需有合理铺垫（回顾时觉得"早该想到"而非"莫名其妙"）
- **格式**: Markdown

### N4-06 悬念设置方案
- **必选组件**: 悬念类型（信息悬念/命运悬念/时间悬念）、设置位置、强度等级、计划释放时机
- **可选组件**: 释放方式（揭晓/部分揭晓/转移）选择
- **组装顺序**: 类型分类→逐个设置→密度检查→释放规划
- **平台约束**: 悬念密度需与节奏曲线匹配，不能全程高悬也不能全程平淡
- **格式**: Markdown

### N4-07 爽点/泪点分布图
- **必选组件**: 事件名称、位置（章/卷区间）、情绪类型（爽/泪/怒/喜）、强度评级(1-10)
- **可选组件**: 前置铺垫标记、后续影响
- **组装顺序**: 按时间排列→强度分布可视化→密度调整
- **平台约束**: 爽点间隔不宜超过3章（网文留存关键指标）
- **格式**: 图表+Markdown注释

## N5 场景与正文

### N5-01 场景卡
- **必选组件**: 场景位置、时间、视角人物、核心动作（一句话）、感官要素（至少2种）、情绪基调、场景目标
- **可选组件**: 场景象征意义、与前后的连接方式
- **组装顺序**: 定位→人物→动作→感官→情绪→目标
- **平台约束**: 每章通常包含1-3个场景
- **格式**: Markdown卡片模板

### N5-02 对白设计
- **必选组件**: 对白的功能目的、参与者、潜台词（表面意思vs真实意图）、节奏（快/慢/交替）
- **可选组件**: 动作描写穿插、沉默/中断的使用
- **组装顺序**: 目的→参与者→内容→潜台词→节奏→修饰
- **平台约束**: 避免"说教式长对话"，网文读者耐心阈值较低
- **格式**: Markdown

### N5-03 环境描写模板
- **必选组件**: 场景类型、空间布局（上/下/左/右/远/近）、光线条件、气味/声音（至少1种）
- **可选组件**: 天气、温度、氛围形容词
- **组装顺序**: 类型→空间→光影→感官→氛围
- **平台约束**: 环境描写占比不超过场景总字数的20%
- **格式**: Markdown

### N5-04 动作序列设计
- **必选组件**: 参与者、动作分解步骤（至少3步）、力量/技能调用、结果
- **可选组件**: 受伤/消耗表现、旁观者反应
- **组装顺序**: 参与者→预备→交手→转折→结果→余波
- **平台约束**: 动作描写需与N1-03力量体系的战力表现一致
- **格式**: Markdown

### N5-05 感官描写清单
- **必选组件**: 五感调用清单（哪些感官被激活、各自的关键词汇）
- **可选组件**: 通感运用、感官对比
- **组装顺序**: 感官盘点→选择→关键词→嵌入检查
- **平台约束**: 避免连续使用同一感官超过200字
- **格式**: 清单式Markdown

### N5-06 过渡段落模板
- **必选组件**: 过渡类型（时间跳跃/地点转换/视角切换/情绪转换）、过渡方式
- **可选组件**: 锚定元素（用于帮助读者重新定位）
- **组装顺序**: 类型判定→选择方式→撰写→可读性检查
- **平台约束**: 过渡段落宜短（50-150字），不宜独立成段
- **格式**: Markdown

### N5-07 战斗场景设计
- **必选组件**: 交战双方、战斗目标、招式/技能调用、胜负结果、战后影响
- **可选组件**: 战斗中的心理活动、环境利用、观众反应
- **组装顺序**: 双方→目标→过程→结果→影响
- **平台约束**: 战斗需有战术层面的博弈，不仅是数值比拼
- **格式**: Markdown

### N5-08 情感戏设计
- **必选组件**: 情感类型、触发事件、角色反应层次（生理→情绪→认知→行为）、场景高潮点
- **可选组件**: 对白核心句（金句候选）、后续影响
- **组装顺序**: 类型→触发→层次递进→高潮→余波
- **平台约束**: 情感戏需有具体事件支撑，避免纯内心独白
- **格式**: Markdown

### N5-09 内心独白/心理描写设计
- **必选组件**: 心理描写类型（内心独白/自由间接引语/意识流/心理总结）、触发情境、认知变化路径（从什么认知→到什么认知）、与外部行为的呼应
- **可选组件**: 心理与对白的交叉节奏、潜意识暗示层、内心独白的叙事功能标注
- **组装顺序**: 类型选择→触发设定→认知路径→行为呼应→节奏控制
- **平台约束**: 网文心理描写宜短不宜长，单段内心独白建议≤300字；连续超过500字纯心理描写会打断叙事节奏
- **格式**: Markdown

## N6 叙事节奏

### N6-01 节奏曲线
- **必选组件**: X轴（进度百分比/章数）、Y轴（张力值0-100）、关键节点标注（高潮/低谷/转折）
- **可选组件**: 目标曲线 vs 实际曲线对比
- **组装顺序**: 全书分区→张力赋值→绘制曲线→检查波动
- **平台约束**: 网文推荐"小波浪+大波浪"模式：每3-5章一个小波峰，每20-30章一个大波峰
- **格式**: 图表/Markdown描述

### N6-02 情绪地图
- **必选组件**: 按章的情绪标签、强度值、主导情绪
- **可选组件**: 情绪连续性检查（相邻章是否跳变过大）
- **组装顺序**: 逐章标记→强度赋值→连续性检查→异常修正
- **平台约束**: 相邻章的情绪跳变幅度不宜超过3级（10级制）
- **格式**: 表格

### N6-03 高潮设计
- **必选组件**: 高潮级别（大/中/小）、定位（第几章/卷）、前置铺垫清单、核心冲突爆发方式
- **可选组件**: 后续回落策略、读者预期管理
- **组装顺序**: 级别→定位→铺垫→爆发→回落
- **平台约束**: 大高潮前至少3章铺垫期
- **格式**: Markdown

### N6-04 悬念密度表
- **必选组件**: 区间（章范围）、悬念数量、平均强度、未解悬念累计数
- **可选组件**: 密度过高/过低预警线
- **组装顺序**: 分区统计→密度计算→预警检查→调整建议
- **平台约束**: 每10章区间未解悬念累计不宜超过5个（读者记忆负担）
- **格式**: 表格

### N6-05 章节钩子策略
- **必选组件**: 钩子类型（悬念式/反转式/情感式/信息式/行动式）、适用场景、效果预估
- **可选组件**: 同类型钩子的变化策略（避免重复）
- **组装顺序**: 类型库建立→逐章匹配→重复检测→效果预估
- **平台约束**: 连续3章不能用同类型钩子
- **格式**: Markdown

## N7 修订与润色

### N7-01 一致性检查清单
- **必选组件**: 世界观一致（规则无前后矛盾）、时间线一致（事件顺序无误）、角色特征一致（性格/外貌/能力无突变）、力量体系一致（等级/战力无越界）
- **可选组件**: 设定变更日志
- **组装顺序**: 四维度逐一检查→记录问题→修正→复查
- **平台约束**: 发布前必须通过此项检查
- **格式**: 检查清单Markdown

### N7-02 风格校准指南
- **必选组件**: 文风统一度检查（是否像同一人写的）、用词偏好一致性、句式长度分布、人称/时态一致性
- **可选组件**: 风格参考基准文本
- **组装顺序**: 抽样检测→问题定位→批量修正→全文扫描
- **平台约束**: 长篇连载尤其容易出现前期后期风格漂移
- **格式**: Markdown指南

### N7-03 冗余删除清单
- **必选组件**: 废话段落的识别标准、重复描写的判定规则、无效过渡段的标记方法
- **可选组件**: 删除后可读性检查
- **组装顺序**: 扫描→标记→删除→通读确认
- **平台约束**: 删除幅度建议控制在总字数的5-15%之间
- **格式**: 清单式Markdown

### N7-04 互审流程
- **必选组件**: 自我修订轮次、审稿反馈收集模板、修改执行追踪、最终确认
- **可选组件**: 审稿人 checklist
- **组装顺序**: 自改→送审→收集→修改→确认
- **平台约束**: 如无可用的审稿人，可用"冷却24h后再读"替代
- **格式**: 流程文档

### N7-05 定稿检查表
- **必选组件**: 格式检查（分段/空行/缩进）、字数达标、标点规范、标题/章节名正确、无占位符残留
- **可选组件**: 移动端阅读适配检查
- **组装顺序**: 格式→内容→技术→发布预览
- **平台约束**: 这是最后一道关卡，不可省略
- **格式**: 检查清单

### N7-06 敏感词筛查报告
- **必选组件**: 违禁词命中列表、命中位置（章节+行号）、严重程度分级、替换建议
- **可选组件**: 平台特定词库版本
- **组装顺序**: 扫描→分级→替换→复扫→人工复核
- **平台约束**: 不同平台敏感词库不同，需匹配目标平台
- **格式**: 报告Markdown

### N7-07 设定修订
- **必选组件**: 修订类型（世界观补全/力量体系修正/角色设定调整/情节逻辑修正）、修订前版本、修订后版本、影响范围（下游受影响任务列表）
- **可选组件**: 修订原因记录、读者感知评估（是否需要作者说/补丁章）
- **组装顺序**: 问题识别→影响评估→修订方案→版本替换→下游同步
- **平台约束**: 已发布章节的设定修订需格外谨慎，避免读者感知到前后矛盾；连载中的补丁通常比回改更可取
- **格式**: Markdown修订日志

### N7-08 正文修订
- **必选组件**: 修订范围（章节号/场景号）、修订类型（场景卡调整/对白优化/描写增删/节奏微调）、修订前后对比
- **可选组件**: 修订理由、读者反馈关联
- **组装顺序**: 范围锁定→类型判定→修订执行→通读验证→更新章纲
- **平台约束**: 正文修订应在定稿检查(N7-05)之前完成；已发布章节的修订需权衡"改了更好"和"读者已经看了旧版"
- **格式**: Markdown修订日志

## N8 商业运营

### N8-01 出版方案
- **必选组件**: 变现模式选择（付费/免费+广告/混合）、定价策略、版权保留声明、分卷/分部策略
- **可选组件**: 海外发行考虑、有声书改编计划
- **组装顺序**: 模式选择→定价→版权→分卷→扩展
- **平台约束**: 需结合N8-02平台选品结果综合决策
- **格式**: Markdown

### N8-02 平台选品分析
- **必选组件**: 目标平台列表、各平台受众画像匹配度、竞争品类分析、推荐机制特点、收入分成比例
- **可选组件**: 平台政策风险评估
- **组装顺序**: 平台筛选→受众匹配→竞争分析→机制研究→综合评分
- **平台约束**: 需基于最新平台政策（政策变动频繁）
- **格式**: Markdown报告

### N8-03 读者画像分析
- **必选组件**: 年龄分布、性别比例、阅读时段偏好、付费意愿分层、核心诉求关键词
- **可选组件**: 读者评论情感分析、流失原因推测
- **组装顺序**: 数据采集→画像构建→分层→诉求提炼
- **平台约束**: 画像随连载进程动态更新
- **格式**: Markdown

### N8-04 IP改编潜力评估
- **必选组件**: 影视改编可行性、游戏改编可行性、动漫改编可行性、有声书改编可行性、核心IP资产清单
- **可选组件**: 改编优先级排序、潜在合作方类型
- **组装顺序**: 维度评估→可行性打分→资产清单→优先级
- **平台约束**: IP价值随追读数据和口碑提升而增长
- **格式**: Markdown

### N8-05 粉丝互动运营计划
- **必选组件**: 互动渠道（评论区/粉丝群/SNS）、互动频率、内容日历、危机应对预案
- **可选组件**: 粉丝UGC激励机制
- **组装顺序**: 渠道规划→频率设定→内容准备→应急预案
- **平台约束**: 互动是网文作者的核心竞争力之一，不可忽视
- **格式**: Markdown

### N8-06 数据复盘报告
- **必选组件**: 追读率趋势、收藏转化率、推荐票/月票数据、评论关键词提取、可执行改进建议
- **可选组件**: 同类竞品数据对比、A/B测试结论
- **组装顺序**: 数据采集→清洗→分析→归因→行动建议
- **平台约束**: 数据来自平台后台，需注意数据时效性
- **格式**: Markdown + 图表

### N8-07 日更计划/更新节奏
- **必选组件**: 日更字数目标、每周更新天数、存稿安全线（至少提前几章）、断更应急预案、休更/请假规则
- **可选组件**: 按卷分配的日更节奏（卷首加速/卷中稳定/卷末冲刺）、爆发/加更策略、节假日特殊排期
- **组装顺序**: 字数目标→更新频率→存稿规划→应急预案→特殊排期
- **平台约束**: 起点要求日更≥4000字才能上推荐位；断更超3天会掉追读；番茄对更新频率敏感度更高
- **格式**: Markdown
