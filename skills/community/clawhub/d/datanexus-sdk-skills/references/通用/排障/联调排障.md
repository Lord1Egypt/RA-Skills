# 联调排障

## 场景一：数据始终显示「数据上报（检测中）」

### 现象
联调页面中，数据上报状态一直显示"检测中"，不变为成功。

### 排查步骤

1. **确认数据已上报**
   - 与开发同学确认对应的 action_type 是否已成功调用 SDK 上报
   - 检查 SDK 返回的上报响应是否为成功

2. **确认数据匹配**
   - 上报的 clickid / 用户 ID 是否是点击联调预览广告的用户所对应的 ID
   - 上报的 action_type 是否与联调要求的转化行为一致

3. **确认在有效时间内**
   - 联调有效时间为 **1 小时**，超时后需重新创建联调任务

4. **确认链路一致**
   - 是使用**数据源（旧链路）**上报的，还是**新版转化归因**上报的？
   - 新旧链路不可混用，旧链路上报的数据不能走新版联调

5. **检查数据源绑定**
   - 在 DataNexus 平台确认数据源是否已绑定对应的 AppID

### 仍未解决
在联调有效时间内提供日志明细，联系SDK小助手排查。

## 场景二：联调点击转发失败

### 现象
联调页面显示「转发失败-接收点击失败」，失败原因：接收响应失败、超时、404、500 等。

### 排查步骤

1. **检查下发链接可达性**
   - 广告主的下发链接是否可以 ping 通
   - 直接用浏览器访问下发链接，确认能返回正常响应

2. **检查 HTTP 状态码**
   - 点击转发时，我们向广告主的链接发送请求
   - 广告主需要返回 **HTTP 200** 表示接收成功
   - **404**：链接路径不存在 → 检查 URL 是否正确
   - **500**：广告主服务器错误 → 广告主自查服务端
   - **超时**：响应太慢 → 优化服务端响应时间

3. **检查 URL 编码**
   - 确认 URL 中的参数已正确 URLEncode
   - 特殊字符未转义会导致请求异常

### 仍未解决
提供下发链接和 HTTP 响应状态码，联系SDK小助手。

## 场景三：联调始终显示检测中（点击转发环节）

### 现象
点击联调广告后，「已发送的广告点击数据」始终显示检测中。

### 排查步骤

1. 在观星台的**数据转发看板**查看数据是否成功转发
2. 确认网络环境稳定，点击行为已被正确捕获

---

## 场景四：显示「检测到你尚未回传数据」

### 现象
客户已回传转化行为数据，但联调页面显示未检测到数据上报。

### 排查步骤

1. 与广告主确认是否向接口**回传了联调要求的行为类型**（如：注册行为）
2. 确认上报后收到的**返回码是成功的**
3. 联调有效时间为 1 小时，确认在有效时间内操作

### 仍未解决
在有效时间内提供日志明细，联系SDK小助手排查。

---

## 场景五：联调任务添加失败

### 现象
点击"下一步"创建联调任务时，页面报错"联调任务添加失败"。

### 排查步骤

1. **检查版位权限**
   - Android / iOS APP 联调需要「腾讯平台与内容媒体」版位权限
   - 小程序 / 小游戏联调需要「微信朋友圈」版位权限
   - 确认账户是否具备对应版位权限

2. **检查账户状态**
   - 账户是否通过审核
   - 账户是否正常使用

### 解决方案
- 联系行业运营确认并开通对应版位权限

---

## 场景六：报错「您使用创意形式、版位和商品类型组合不支持当前品牌形象类型」

### 原因
账户缺少联调所需的版位权限。Android/iOS APP 需要「腾讯平台与内容媒体」版位，小程序/小游戏需要「微信朋友圈」版位。

### 解决方案
带着账户 ID 联系行业运营确认并开通对应版位权限。

---

## 场景七：联调服务器异常

### 排查步骤

1. 检查网络环境是否稳定
2. 尝试更换浏览器
3. 确认账户能否正常使用（如账户未通过审核会报此错误）
4. 检查点击 URL 填写是否正确

### 仍未解决
均排查通过后仍报错，请联系SDK小助手。

---

## 通用提醒

- 联调有效时间：**1 小时**
- 转化规则状态更新：约 **5 分钟延迟**
- 鸿蒙端暂不支持转化联调，请通过日志查询和质量看板验证数据上报
- 新旧链路不可混用：数据源（旧链路）和新版转化归因是不同链路
