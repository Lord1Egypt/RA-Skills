# 概念解释记录与用户档案存储规范

本文件定义 `my-study-pal` 的本地概念解释记录结构与用户档案结构。默认在**当前工作空间根目录**下创建并维护 `mystudy/`。

与其他文件的分工：
- `setup.md` 负责首次启用、补齐骨架和可运行状态检查
- `references/blueprint.md` 负责定义最小可运行骨架
- `study-storage.md` 负责定义目录结构、文件职责和模板内容

## 目录结构

```text
mystudy/
  study-summary.md
  study-detail/
    YYYYMMDD主题名.md
  user-profile.md
  runtime-profile.md
```

## 文件职责

### 1. study-summary.md

用于保存学习主题概览，适合作为长期总表。

至少记录以下字段：
- 时间（新写入记录精确到时分秒；历史缺失精确时间时，可保留原粒度，不强行伪造时分秒）
- 学习主题
- 回答方式（固定只写“直答模式”“讲解式”“引导式”“辨析式”“场景应用式”或“记忆巩固式”）
- 是否完成

### 2. study-detail/

用于保存单个学习主题的对话记录。

文件命名规则：
- `YYYYMMDD主题名.md`
- 文件名尽量简洁，去掉非法路径字符
- 同一天同主题若重复记录，可在文件名后追加序号，例如：
  - `20260410提示词工程-2.md`
- 作为 skill 内置模板文件时，优先使用 ASCII 文件名保存模板本体，例如 `study-detail-template.md`

每个详情文件至少包含：
- 学习主题
- 日期
- 回答方式（固定只写“直答模式”“讲解式”“引导式”“辨析式”“场景应用式”或“记忆巩固式”）
- 对话记录（直接保留完整对话过程，不做摘要版纪要）

### 3. user-profile.md

用于保存用户画像与长期教学偏好。

建议包含以下结构：

- 用户工作 / 专业 / 业务领域
- 用户爱好领域
- 用户最近的 5 个学习知识点
- 用户教学偏好
  - 直答模式
    - 追问偏好
  - 讲解式
    - 展开程度
    - 术语密度
    - 例子来源
    - 追问偏好
  - 引导式
    - 互动强度
    - 问题形式
    - 引导切入方式
    - 术语密度
    - 例子来源
  - 辨析式
    - 展开程度
    - 术语密度
    - 对照形式
    - 例子来源
    - 追问偏好
  - 场景应用式
    - 应用深度
    - 步骤粒度
    - 例子来源
    - 风险提示强度
  - 记忆巩固式
    - 记忆强度
    - 测试方式
    - 复习提示
    - 例子来源
- 语言风格偏好

运行时规则：
- 该文件是概念解释过程中优先读取的实际生效配置
- 如果该文件缺少某些偏好字段，再回退到对应 reference 文件中的默认建议
- 用户提出长期偏好时，必须在同一轮内实时更新该文件
- 用户提出临时偏好时，本轮可先临时生效，不必默认改写该文件
- 不允许出现“回复里说已记住，但用户档案未更新”的状态

### 4. runtime-profile.md

用于保存当前生效学习配置摘要，减少每轮临时合并固定规则和用户偏好的负担。

职责边界：
- `user-profile.md` 是用户长期偏好和画像源头
- `runtime-profile.md` 是从 `user-profile.md` 派生的缓存，不是源头
- 固定教学方法仍以 `references/method-*.md` 和 `references/response-contract.md` 为准
- 用户长期偏好变化时，先更新 `user-profile.md`，再刷新 `runtime-profile.md`
- 不要只修改 `runtime-profile.md` 来“记住”用户偏好

## 初始化与可运行状态

当 skill 第一次需要落盘概念解释记录或用户画像时，应先读取 `setup.md`，并按 `references/blueprint.md` 补齐骨架。

### 补齐要求

1. 检查工作空间下是否存在 `mystudy/`
2. 若不存在，则创建：
   - `mystudy/`
   - `mystudy/study-detail/`
3. 若 `study-summary.md` 不存在，则创建默认模板
4. 若 `user-profile.md` 不存在，则创建默认模板
5. 若 `runtime-profile.md` 不存在，则创建默认模板或由刷新脚本生成
6. 不要覆盖用户已有内容

### 可运行状态

概念解释系统至少满足以下条件时，才算达到可运行状态：
- `mystudy/` 存在
- `mystudy/study-detail/` 存在
- `study-summary.md` 存在
- `user-profile.md` 存在
- `runtime-profile.md` 存在
- `user-profile.md` 中至少包含直答模式默认配置、讲解式默认配置与引导式默认配置

核心原则是：**第一次需要记录概念解释内容或用户画像时，目录一定要自动补齐；未达到可运行状态前不要假装初始化已经完成。**

## 推荐模板

### study-summary.md 模板

```markdown
# 学习概览

| 时间 | 学习主题 | 回答方式 | 是否完成 |
|---|---|---|---|
| 2026-04-10 09:30:00 | 示例主题 | 讲解式 | 进行中 |

回答方式字段只允许填写：`直答模式`、`讲解式`、`引导式`、`辨析式`、`场景应用式`、`记忆巩固式`
```

### user-profile.md 模板

```markdown
# user-profile

## 用户工作 / 专业 / 业务领域
- 暂无记录

## 用户爱好领域
- 暂无记录

## 用户最近的 5 个学习知识点
- 暂无记录

## 用户教学偏好

### 直答模式
- 追问偏好：不追问（可选：不追问 / 轻追问）
- 适用主题：非常简单、直接、单点事实型问题优先

### 讲解式
- 展开程度：标准
- 术语密度：白话为主（可选：白话为主 / 白话夹术语 / 术语正常）
- 例子来源：自动选择最贴切的用户语境
- 追问偏好：轻追问（可选：不追问 / 轻追问）
- 适用主题：名词概念解释、术语辨析、缩写还原、快速听懂类问题优先

### 引导式
- 互动强度：轻互动（可选：轻互动 / 标准互动 / 强互动）
- 问题形式：选择判断（可选：开放回答 / 选择判断 / 混合）
- 引导切入方式：案例切入（可选：理论切入 / 案例切入 / 混合切入）
- 术语密度：白话为主（可选：白话为主 / 白话夹术语 / 术语正常）
- 例子来源：自动选择最容易帮助判断的用户语境
- 适用主题：原理机制理解、过程推演、因果链分析、问题深入拆解优先

### 辨析式
- 展开程度：标准
- 术语密度：白话为主
- 对照形式：短段对比
- 例子来源：自动选择最能暴露边界的用户语境
- 追问偏好：轻追问
- 适用主题：概念区别、边界判断、相近词辨析优先

### 场景应用式
- 应用深度：标准应用
- 步骤粒度：中粒度
- 例子来源：优先使用用户工作 / 专业 / 业务领域
- 风险提示强度：轻提示
- 适用主题：概念落地、工作应用、行动建议优先

### 记忆巩固式
- 记忆强度：标准巩固
- 测试方式：选择判断
- 复习提示：轻提示
- 例子来源：优先使用用户熟悉语境
- 适用主题：记忆、复习、巩固、避免遗忘优先

## 语言风格偏好

### 1. 扮演角色：默认风格（清晰、自然、像可靠同事）

#### 核心气质与立场
- 清晰、自然、可靠

#### 句式与措辞
- 直接表达，少绕弯，默认不过度风格化

#### 口头禅
- 暂无固定口头禅
- 若用户明确指定，应优先自然穿插，增强识别度

### 2. 其他语言配置
- 风格强度：中（可选：轻 / 中 / 强）
- 幽默程度：轻（可选：无 / 轻 / 中 / 强）
- 表情符号：少量（可选：无 / 少量 / 明显）
- 角色语境使用：允许少量用于开场、点睛、类比或收束
- 角色比喻来源：按角色设定自动选择；若无明确角色，则优先使用用户真实语境
- 主解释优先级：用户真实语境优先，角色语境辅助增强辨识度
```
