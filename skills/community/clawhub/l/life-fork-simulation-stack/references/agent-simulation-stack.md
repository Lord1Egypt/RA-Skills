# Life Fork Simulation Stack

用于把一个人生岔路拆成多角色、三种结果和多事件冲击的可审计复盘。多 Agent 的价值来自分歧和制衡，最终报告必须保留关键冲突。

## 运行方式

### 支持 sub-agent 的环境

可并行运行各 Agent，各自产出短结论，再由主 Agent 汇总。

建议顺序：

1. 用户画像 Agent 先用 2-3 轮问答建立基础画像。
2. 总控 Agent 整理输入、假设、置信度和问题边界。
3. 启动人生叙事、职业路径、资产现金流、关系家庭、健康能量、事件检索六个分析 Agent。
4. 时代事件 Agent 读取事件检索结果，判断外部冲击如何影响路径。
5. 总控 Agent 汇总各 Agent 的支持路径和风险冲突，形成判断来源表和冲突摘要。
6. 反方审计 Agent 读取所有分支，质疑过度浪漫化和单一路径光环。
7. 交付编辑 Agent 读取最终报告，生成可保存、可复读、可继续追问的报告交付包。

### 普通对话环境

没有 sub-agent 能力时，按顺序模拟十一个角色。每个角色只输出四个固定字段，避免篇幅失控。

## 角色定义

### 总控 Agent

负责：

- 判断触发模式。
- 管理用户画像、事件检索和三种结果推演。
- 汇总各 Agent 的冲突，不把分歧写平。
- 判断材料充分度和补充问题。

输出：

- 当前模式
- 已知事实
- 关键假设
- 判断来源表
- 综合判断
- 置信度

### 0. 用户画像 Agent

负责：

- 用 2-3 轮问答建立用户基本画像。
- 识别当前岔路、实际选择、想复盘的假设结果和主要情绪。
- 判断材料置信度。
- 生成下一轮最该问的 1-3 个问题。

调用：

- `references/onboarding-question-flow.md`
- `templates/intake.md`

输出：

- 核心判断
- 最担心的风险
- 最支持哪条路径
- 需要反方审计的地方

### 1. 人生叙事 Agent

负责：

- 把用户问题改写成一个可代入的人生岔路。
- 判断这个问题让用户反复想起的原因。
- 识别后悔、不甘、安全感、身份变化、漂浮感等叙事张力。

输出：

- 核心判断
- 最担心的风险
- 最支持哪条路径
- 需要反方审计的地方

### 2. 职业路径 Agent

负责：

- 行业周期
- 城市与平台资源
- 岗位路径
- 组织能见度
- 技能复利
- 履历路径依赖

输出：

- 核心判断
- 最担心的风险
- 最支持哪条路径
- 需要反方审计的地方

### 3. 资产现金流 Agent

负责：

- 收入曲线
- 生活成本
- 房租、房贷、负债
- 储蓄率
- 现金流安全感
- 城市选择对资产压力的影响

边界：

- 禁止给具体投资建议。
- 禁止推荐买卖资产。
- 只做现金流压力和选择承受力分析。

输出：

- 核心判断
- 最担心的风险
- 最支持哪条路径
- 需要反方审计的地方

### 4. 关系与家庭 Agent

负责：

- 父母距离
- 伴侣关系
- 婚育节点
- 家庭照护
- 地域距离
- 社会支持网络

边界：

- 禁止心理诊断。
- 禁止替用户处理亲密关系决定。
- 涉及安全风险时优先现实支持。

输出：

- 核心判断
- 最担心的风险
- 最支持哪条路径
- 需要反方审计的地方

### 5. 健康与能量 Agent

负责：

- 城市节奏
- 通勤
- 睡眠
- 压力
- 恢复时间
- 身体可持续性
- 长期消耗

输出：

- 核心判断
- 最担心的风险
- 最支持哪条路径
- 需要反方审计的地方

### 6. 事件检索 Agent

负责：

- 从 2008-2026 事件知识库中检索相关事件卡片。
- 根据用户当年年份、城市、行业、海外经历、资产状态、技术路径筛选事件。
- 提醒哪些事件需要用户确认年份和细节。

调用：

- `references/era-event-timeline-2008-2026.md`
- `references/event-impact-framework.md`

输出：

- 核心判断
- 最担心的风险
- 最支持哪条路径
- 需要反方审计的地方

### 7. 时代事件 Agent

负责：

- 国内大事件
- 全球大事件
- 行业周期
- 技术浪潮
- 城市变化
- 就业市场变化

边界：

- 已经发生的事件可用于过去复盘。
- 未来事件只作为情景变量。
- 不做宏观预测。
- 不用宏观事件制造焦虑。

输出：

- 核心判断
- 最担心的风险
- 最支持哪条路径
- 需要反方审计的地方

### 8. 反方审计 Agent

负责：

- 质疑假设结果的浪漫化。
- 质疑真实路径的过度否定。
- 检查是否把时代红利当成个人能力。
- 检查是否把外部冲击归因给个人选择。
- 检查是否出现“另一条路一定更好”的暗示。

输出：

- 核心判断
- 最担心的风险
- 最支持哪条路径
- 需要反方审计的地方

### 9. 交付编辑 Agent

负责：

- 把完整报告整理成终端用户可保存版本。
- 生成判词、三种结果、6 段报告、继续解释层和 3 个低成本验证动作。
- 保持人话表达，减少术语和复杂图表。
- 在支持文件的环境中，优先输出 HTML 用户报告，并把留档方式交给用户浏览器处理。

输出：

- 完整报告目录
- 判词型摘要
- 三种结果
- 附录
- 3 个低成本验证动作
- 5 个补充问题

## 冲突合成规则

每个核心 Agent 必须给出四个字段：

```text
### {{Agent 名称}}

- 核心判断：
- 最担心的风险：
- 最支持哪条路径：
- 需要反方审计的地方：
```

总控 Agent 汇总时必须生成冲突摘要：

| 冲突 | 支持路径 | 反对路径 | 需要验证 |
| --- | --- | --- | --- |
| 职业上限 vs 现金流承受 | {{路径}} | {{路径}} | {{信息}} |
| 城市机会 vs 家庭支持 | {{路径}} | {{路径}} | {{信息}} |
| 短期跃迁 vs 长期健康 | {{路径}} | {{路径}} | {{信息}} |

常见分歧：

- 职业路径 Agent 可能支持机会密度更高的路径。
- 资产现金流 Agent 可能反对高成本、高负债、高试错的路径。
- 关系与家庭 Agent 可能支持离支持网络更近的路径。
- 健康与能量 Agent 可能反对通勤过长、睡眠不足、压力过高的路径。
- 时代事件 Agent 可能指出某条路径吃到周期，也可能指出周期窗口已经关闭。
- 反方审计 Agent 必须质疑“另一条路天然更好”的想象。

## 汇总规则

- 先给判词和三种结果，再给多角度冲突摘要。
- 每个假设结果使用“可能、如果、在某些条件下、需要满足”等表达。
- 每种结果必须同时写收益和代价。
- 必须区分选择、时代、环境和运气。
- 最终结论落到今天仍可迁移的经验和 3 个低成本验证动作。

## sub-agent 并行优化

可并行组：

- 组 1：人生叙事 Agent、职业路径 Agent、资产现金流 Agent、关系与家庭 Agent、健康与能量 Agent。
- 组 2：事件检索 Agent、时代事件 Agent。
- 组 3：总控 Agent 生成判断来源表和冲突摘要。
- 组 4：反方审计 Agent，在冲突摘要之后运行。
- 组 5：交付编辑 Agent，在最终报告后运行。

主 Agent 必须检查：

- 各 Agent 是否围绕同一时间点。
- 外部事件是否落在用户当年路径相关时期。
- 各 Agent 的支持路径是否存在分歧。
- 是否出现单一路径美化。
- 是否存在诊断、投资建议或确定性命运判断。
