# PAD — 过程资产开发（CMMI V3.0 Core PA）

## PA 基本信息
- **缩写**：PAD
- **英文**：Process Asset Development
- **中文**：过程资产开发
- **所属**：Core PA（核心实践域）
- **对应 V2.0**：OPD（组织过程定义）+ OPF（组织过程焦点）的部分内容

---

## CMMI V3.0 要求要点

### 核心目的
开发和维护组织的过程资产，以提高组织的过程绩效和竞争力。

### 关键实践（Specific Practices）
| 实践编号 | 实践内容 | 说明 |
|---------|---------|------|
| PAD 1.1 | 建立组织标准过程 | 定义组织级的过程体系 |
| PAD 1.2 | 建立生命周期模型描述 | 定义适用的生命周期模型 |
| PAD 1.3 | 建立裁剪指南 | 为项目提供裁剪标准 |
| PAD 1.4 | 建立组织过程资产库 | 存储过程资产（模板、检查单、经验） |
| PAD 2.1 | 维护组织标准过程 | 基于反馈和变化持续改进 |
| PAD 2.2 | 维护过程资产库 | 更新和维护过程资产 |
| PAD 3.1 | 贡献过程经验到组织资产 | 项目经验教训反馈到组织 |

### 通用实践（Generic Practices）
- GP 1.1 执行特定实践
- GP 2.1 建立组织政策
- GP 2.2 策划过程
- GP 2.3 提供资源
- GP 2.4 分配责任
- GP 2.5 培训人员
- GP 2.6 管理配置
- GP 2.7 识别并记录干系人需求和接口
- GP 2.8 监督和控制过程
- GP 2.9 客观评估符合性
- GP 2.10 高层审查

## 用户文档 → PA 匹配规则

### 高概率匹配（主要PA）
- 文档名含"组织标准过程"、"过程资产"、"裁剪指南" → **PAD 主要匹配**
- 文档含过程体系文件、模板库 → **PAD 主要匹配**

### 中概率匹配（次要PA）
- 文档含过程度量数据 → 同时匹配 **PCM**（过程质量管理）
- 文档含培训材料 → 同时匹配 **OT**（组织培训）

## 差距检查清单（用于差距分析）

### PAD 1.x — 过程资产建立
- [ ] 是否有组织级标准过程体系？
- [ ] 是否有生命周期模型描述（敏捷/瀑布/混合）？
- [ ] 是否有裁剪指南（针对不同项目类型）？
- [ ] 是否有过程资产库（模板、检查单、经验教训）？

### PAD 2.x — 过程资产维护
- [ ] 组织标准过程是否有版本控制？
- [ ] 是否有定期的过程资产评审机制？
- [ ] 项目经验教训是否有反馈渠道？

### PAD 3.x — 过程经验贡献
- [ ] 项目结束后是否有经验教训总结？
- [ ] 经验教训是否纳入组织资产库？

## 文档转换指导（模式C）

当用户要求将自己的文档转换为CMMI合规格式时，按以下结构输出：

### 转换后的文档结构（Markdown模板）
```markdown
# 组织标准过程体系

## 1. 过程体系概述
（目的、范围、适用场景、与业务目标的对齐）

## 2. 组织标准过程清单
| 过程ID | 过程名称 | 过程类型 | 负责人 | 版本 | 上次更新 |
|--------|---------|---------|--------|------|----------|

## 3. 生命周期模型
### 3.1 敏捷生命周期
- **适用场景**：[需求不确定、需要快速迭代的项目]
- **阶段划分**：[Sprint规划 → 每日站会 → Sprint执行 → Sprint评审 → Sprint回顾]
- **关键交付物**：[产品待办列表、Sprint待办列表、产品增量]

### 3.2 瀑布生命周期
- **适用场景**：[需求明确、合同驱动的项目]
- **阶段划分**：[需求分析 → 设计 → 开发 → 测试 → 上线]
- **关键交付物**：[需求规格书、设计文档、测试报告]

## 4. 裁剪指南
| 项目类型 | 推荐生命周期 | 必选过程 | 可选过程 | 裁剪理由 |
|---------|------------|---------|---------|---------|

## 5. 过程资产库
### 5.1 模板库
| 模板ID | 模板名称 | 适用过程 | 文件格式 | 版本 |
|--------|---------|---------|---------|------|

### 5.2 检查单库
（评审检查单、审计检查单、验收检查单）

### 5.3 经验教训库
（项目总结、最佳实践、常见错误）

## 6. 过程改进机制
（过程反馈渠道、改进提案流程、改进实施跟踪）

## 7. 版本记录
（版本号、修订日期、修订内容、修订人、审批人）
```

## 评估访谈常见问题（模式D）

### 针对EPG（过程改进组）的问题
1. "组织级过程体系是如何建立的？包含哪些内容？"
2. "如何为项目提供裁剪指南？有裁剪记录吗？"
3. "过程资产库包含哪些内容？如何维护？"
4. "项目经验教训如何反馈到组织资产？"

### 针对PM（项目经理）的问题
1. "你的项目是如何裁剪组织标准过程的？"
2. "你使用过哪些组织级模板？有反馈过问题吗？"
3. "项目结束后有总结经验教训吗？"

### 评分要点
- **PAD 1.1-1.4**：是否有完整的组织过程资产和裁剪指南
- **PAD 2.1-2.2**：过程资产是否有版本控制和定期维护
- **PAD 3.1**：是否有项目经验贡献到组织资产的机制
