# 知乎自动化安全红线 (Safety Rules)

为确保合规性及保障用户账号安全，本 Skill 设定了极其严格的开发和运行红线。任何人修改本 Skill 的代码，都必须遵守以下规则：

## 1. 绝对禁止的自动化行为
- **严禁自动点击“发布”**：无论是发文章还是发回答，最后一步必须由鼠标真实点击。
- **严禁后台隐式发布**：不准通过捕获知乎的非公开 API（如 `api/v4/answers`）进行接口级发文。
- **严禁点赞、评论、关注、收藏**：不准包含任何刷量、互动刷分逻辑。
- **严禁批量发文**：不支持读取一个列表进行循环发文，每次执行只处理单篇草稿。

## 2. 账号与凭据安全
- **不碰 Cookie**：严禁通过 `page.context.cookies()` 读取 Cookie，严禁将 Cookie 保存到本地文件。
- **不碰 Storage**：严禁调用 `browser_context.storage_state` 保存登录态，严禁读取 localStorage 或 sessionStorage。
- **不碰密码**：严禁要求用户在任何配置中输入知乎账号和密码。

## 3. 风控应对策略
- **遇拦截即停止**：遇到图形验证码、手机号验证、滑块验证，脚本必须立刻停下，要求人工处理。
- **不可绕过机制**：不得引入任何第三方打码平台或 OCR 库尝试绕过知乎的安全校验。

## 4. 遵守 ClawHub 规范
- **禁止越权读取**：脚本只允许读取 Skill 目录下的 `/output/zhihu/` 里的文件。绝对禁止读取 `~/.ssh/`、`/etc/` 或系统全局文件。
- 不使用 `eval` 或 `exec` 动态执行未知代码。
