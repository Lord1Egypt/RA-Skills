## 🏗️ 三层架构定义 🆕（2026-03-07 16:33）

| 层级 | 定义 | 组成 | 示例 |
|------|------|------|------|
| **原子动作** | 最小、不可再拆分、单一执行单元 | 单一操作 | ATOM-VISUAL-008 |
| **Skill** | 多个原子动作 + 简单逻辑的组合 | 2-6 个原子动作 | SKILL-TTS-001 |
| **工作流** | 多个 Skill 的有序串联与自动化执行 | 多个 Skill | WORKFLOW-DOUBAO-001 |

**核心洞察：**
- 原子动作 = 零件（扳手、螺丝刀）
- Skill = 工具（由零件组装）
- 工作流 = 生产线（由工具串联）

## 🏷️ 分类标签体系

### 标签维度 1：按功能（现有层级）

```
数据层 (DATA)
呈现层 (VISUAL)
交付层 (DELIVERY)
分析层 (ANALYSIS)
IO 层 (IO)
工具层 (TOOL)
文档层 (DOC)
```

### 标签维度 2：按使用频率 🆕

```
🔥 高频 - 每天使用多次（如：飞书发送文字）
⭐ 中频 - 每周使用几次（如：生成 TTS 语音）
📌 低频 - 偶尔使用（如：渲染 Mermaid 为 PNG）
```

### 标签维度 3：按复杂度 🆕

```
🟢 简单 - 单一操作，无依赖（如：保存文件）
🟡 中等 - 需要 1-2 个前置动作（如：写入飞书文档）
🔴 复杂 - 需要多个前置动作 + 条件判断（如：三线更新）
```

### 标签维度 4：按业务域 🆕

```
豆包会话归档
周报系统
项目管理
TTS 语音
专家点评
三线更新
```

## 💡 核心价值

### 分类标签的价值

1. **快速查找：** 通过标签过滤，快速定位需要的原子动作

1. **降低门槛：** 新人不需要理解每个原子动作，直接调用 Skill

3. **保证质量：** Skill 经过验证，避免组合错误
4. **知识传承：** 最佳实践固化为 Skill，新人快速上手

## ⚠️ 注意事项

### 标签使用原则

- ✅ 标签数量适中（3-5 个），不要过多
- ✅ 标签命名统一（使用#前缀）
- ✅ 定期审查标签，合并重复标签

### Skill 使用原则

- ✅ Skill 是原子动作的组合，不是新的抽象层
- ✅ Skill 应该可拆解，便于理解和调试
- ✅ Skill 应该可组合，Skill 之间可以嵌套
