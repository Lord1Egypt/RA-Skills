# 题型格式规范

## 四种题型定义

### 单选题 (mc)
- 选项数：4个（A/B/C/D）
- 正确答案：1个
- answer 格式："A" | "B" | "C" | "D"
- 评分：5分/题（三套统一）

### 多选题 (mma)
- 选项数：4–5个（A/B/C/D 或 A/B/C/D/E）
- 正确答案：2–4个
- answer 格式：字母连排，如 "AC"、"BDE"（无空格无标点）
- 评分：6分/题
- 评分规则（写入模板页面说明）：全对6分，漏选2分，错选0分

### 判断题 (tf)
- 选项：无（options = null）
- answer 格式："正确" | "错误"
- 评分：3分/题
- explanation：必须说明判断依据或指出错误所在

### 简答题 (sa)
- 选项：无（options = null）
- answer 格式：分点作答，每点一行，前缀 "1. " "2. " "3. "
- 评分：10分/题
- explanation：给出关键词和主要评分点

## 考点分配策略

按 knowledge_map.points 的 importance 降序处理：

```
importance ≥ 0.8  →  套1 + 套2 + 套3（高优先级考点，三套必出）
0.6 ≤ imp < 0.8  →  套1 + 套2
imp < 0.6        →  仅套1
is_weak_point = true → 套1 额外将该考点的 tf 替换为针对最常见误区的判断题
```

题目槽位不足时：按 importance 降序截取，确保高分考点优先覆盖。
题目槽位有余时：高 importance 考点可在同套内从不同角度出多题。

## 题目来源策略（C方案 · 混合）

检查每个考点的 `evidence` 字段：

```
evidence 含"真题"/"历年"/"模拟题" →  source_type = "adapted"
  做法：调用 /knowledge-base 技能（note_retrieve）拉取对应原文 → 改编措辞/数据，保留核心考查点

evidence 不含上述关键词       →  source_type = "generated"
  做法：基于 point.name + 调用 /knowledge-base 技能（note_retrieve）获取相关上下文 → AI 生成
```
