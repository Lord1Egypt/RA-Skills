## Use Case: <br>
Developers and engineers use this skill to structure development work into measurable success criteria, checkpointed execution, quality gates, ranked findings, action plans, and concrete next steps. For SQL performance work, users should supply the query, schema, indexes, and execution plan because the artifact behaves as a general planning checklist rather than a database-specific diagnostic tool. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: The skill name and summary may lead users to expect database-specific SQL diagnostics, while the artifact is a generic planning and quality-gate checklist. <br>
Mitigation: Use it for structured planning unless the user supplies database context such as the query, schema, indexes, execution plan, constraints, and performance goal. <br>
Risk: Recommendations could be based on incomplete evidence when artifacts, logs, or constraints are missing. <br>
Mitigation: Record assumptions, require evidence-backed claims, and include fallback actions before finalizing recommendations. <br>
