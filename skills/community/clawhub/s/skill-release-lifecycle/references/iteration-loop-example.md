# Iteration Loop — Worked Example: waste-audit

This document shows how the Skill Release Lifecycle post-release loop applied to `waste-audit` based on real events from 2026-05-23.

---

## 1. Collect Feedback

Feedback items collected from user sessions and direct observation:

| # | Feedback | Source | Date |
|---|---|---|---|
| 1 | Activation phrase buried in SKILL.md caused routing to fail | User report / dogfood | 2026-05-23 |
| 2 | Two `## Install` sections appeared on public page | Public-page verification | 2026-05-23 |
| 3 | HTML comment blocks rendered visibly on public page | User report / public-page verification | 2026-05-23 |
| 4 | `## Features` required as first section instead of routing contract | User strategic direction | 2026-05-23 |
| 5 | "Use this skill when the user asks:" appeared in Activation | Public-page verification | 2026-05-23 |
| 6 | Top install block missing `--global` | Public-page verification | 2026-05-23 |

---

## 2. Classify Feedback

| # | Classification | Reason |
|---|---|---|
| 1 | Confusing activation / activation issue | Routing reliability affected actual use. |
| 2 | Public page mismatch | Duplicate source headers rendered publicly. |
| 3 | Public page mismatch / UX wording issue | HTML comments were visible to users. |
| 4 | Strategic repositioning | User explicitly changed the preferred public structure. |
| 5 | UX wording issue | Internal activation phrase leaked into public-facing page. |
| 6 | Not worth acting on / platform limitation | Top install block is generated by ClawHub and cannot include `--global`. |

---

## 3. Decide Action

| # | Decision | Reason | Impact Area |
|---|---|---|---|
| 1 | Minor version bump | Activation/routing behavior changed. | activation |
| 2 | Docs-only patch / patch bump | Source cleanup fixed public rendering. | public page |
| 3 | Minor wording patch / patch bump | Removed visible internal comments. | public page |
| 4 | Major repositioning | Public structure changed from routing-first to Features-first. | scope |
| 5 | Minor wording patch | Removed internal phrasing from public page. | public page |
| 6 | Ignore / log only | Known ClawHub platform limitation; document instead of churn. | install |

---

## 4. Apply Change

Smallest relevant patches were applied to the `waste-audit` SKILL.md.

`clawhub-auto-publish` was not modified unless the issue was about publishing workflow documentation.

---

## 5. Re-release

Re-release used the established ClawHub publish workflow through `clawhub-auto-publish`.

Version bumps were tied to concrete fixes rather than arbitrary churn.

---

## 6. Verify

Verification included:

```bash
clawhub inspect waste-audit
```

Public page verification checked:
- `## Features` appears first
- duplicate `## Install` section removed
- HTML comments absent
- internal "Use this skill when the user asks:" phrasing absent
- body install section contains the intended `--global` guidance

Top install block lacking `--global` was documented as a platform limitation, not a source-file failure.

---

## 7. Record Changelog

Good changelog shape for this loop:

- "Clarified activation and public structure after dogfood showed routing confusion." — impact: activation, scope
- "Removed duplicate Install section and visible internal comments from public page." — impact: public page
- "Documented `--global` install path in the body install section; top generated install block remains a ClawHub limitation." — impact: install

Bad changelog shape (avoided here):
- "Update."
- "Fix stuff."
- "Improve skill."

---

## 8. Report Evidence

Report should include:
- version before / after
- exact source sections changed
- `clawhub inspect` result
- public page status: PASS / FAIL / BLOCKED
- remaining limitations
- per-item impact area

Do not report PASS if public-page verification was blocked.
