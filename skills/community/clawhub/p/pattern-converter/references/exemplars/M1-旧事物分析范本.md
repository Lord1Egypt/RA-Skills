# M1 旧事物分析域范本

## M1-01 事物描述

### 输入示例
旧事物：传统出租车行业

### 处理过程
1. 事物名称：传统出租车行业
2. 事物特征：巡游载客、计价器计费、司机驾驶
3. 事物背景：城市交通出行的重要组成部分
4. 事物范围：全球范围内的城市交通

### 输出示例
```markdown
## 事物描述报告

### 事物名称
- **名称**：传统出租车行业
- **别名**：的士、计程车

### 事物特征
| 特征 | 描述 | 重要性 |
|------|------|--------|
| 巡游载客 | 在道路上巡游寻找乘客 | 高 |
| 计价器计费 | 使用计价器计算费用 | 高 |
| 司机驾驶 | 由专业司机驾驶 | 高 |

### 事物背景
- **历史**：19世纪末出现
- **发展**：逐渐成为城市交通的重要组成部分
- **现状**：面临网约车的竞争

### 事物范围
- **地理范围**：全球范围内的城市
- **服务范围**：城市交通出行
```

## M1-02 结构分析

### 输入示例
事物：传统出租车行业

### 处理过程
1. 结构识别：识别行业的组成部分
2. 结构组成部分：出租车、司机、乘客、调度系统
3. 结构关系：各部分之间的关系
4. 结构特征：结构的特征

### 输出示例
```markdown
## 结构分析报告

### 结构识别
- **方法**：识别行业的组成部分
- **结果**：4个主要组成部分

### 结构组成部分
| 部分 | 描述 | 功能 |
|------|------|------|
| 出租车 | 载客工具 | 提供出行服务 |
| 司机 | 驾驶人员 | 驾驶和提供服务 |
| 乘客 | 服务对象 | 接受出行服务 |
| 调度系统 | 管理系统 | 调度和管理 |

### 结构关系
- **出租车-司机**：司机驾驶出租车
- **司机-乘客**：司机为乘客提供服务
- **调度系统-出租车**：调度系统管理出租车

### 结构特征
- **特征**：线性结构
- **优势**：简单直接
- **劣势**：效率低下
```

## M1-03 原理分析

### 输入示例
事物：传统出租车行业

### 处理过程
1. 原理识别：识别行业的运作原理
2. 原理内容：供需匹配原理、计价原理、服务原理
3. 原理机制：各原理的运作机制
4. 原理验证：验证原理的有效性

### 输出示例
```markdown
## 原理分析报告

### 原理识别
- **方法**：识别行业的运作原理
- **结果**：3个主要原理

### 原理内容
| 原理 | 描述 | 作用 |
|------|------|------|
| 供需匹配原理 | 匹配司机和乘客 | 实现出行服务 |
| 计价原理 | 根据里程和时间计费 | 实现收益 |
| 服务原理 | 提供出行服务 | 满足乘客需求 |

### 原理机制
- **供需匹配**：司机巡游，乘客招手
- **计价**：计价器根据里程和时间计算
- **服务**：司机驾驶，乘客乘坐

### 原理验证
- **验证方法**：检查原理的有效性
- **验证结果**：原理有效
```

## M1-04 模式提取

### 输入示例
原理：供需匹配原理、计价原理、服务原理

### 处理过程
1. 提取方法：从原理中提取抽象模式
2. 模式内容：匹配模式、计价模式、服务模式
3. 模式特征：每个模式的特征
4. 模式验证：验证模式的有效性

### 输出示例
```markdown
## 模式提取报告

### 提取方法
- **方法**：从原理中提取抽象模式
- **原则**：提取可迁移的模式

### 模式内容
| 模式 | 描述 | 特征 |
|------|------|------|
| 匹配模式 | 匹配供需双方 | 实时、动态 |
| 计价模式 | 根据使用量计费 | 公平、透明 |
| 服务模式 | 提供专业服务 | 专业、可靠 |

### 模式特征
- **匹配模式**：实时匹配，动态调整
- **计价模式**：按量计费，公平透明
- **服务模式**：专业服务，可靠质量

### 模式验证
- **验证方法**：检查模式的有效性
- **验证结果**：模式有效
```
