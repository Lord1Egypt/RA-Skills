# phase-two-material.md — 阶段二：收料配置

> PM 发素材 → 匹配项目 → 判齐全 → **素材齐就算业务正确值 + 交接② 委托技术那套配物料（直接发审批）**。素材不齐就写表存着，灵渠助手会催料。

## 总览

```
PM 发素材(DM) → 匹配追踪表 → 读 H 列配置方式 → 解析物料 → 判齐全
        ↓ 匹配失败                    ├── 不齐 → 写表(已收的料) → N 保持已预占 → 结束(灵渠助手催料)
      问 PM 是哪个项目                 └── 齐全 → 算业务正确值(资源类型/拼好链接/角标/人群包)
                                              → 写表 I/J/K/L
                                              → 交接② 调技术那套 promotion-update-creative(直接发审批)
                                              → 过收尾门禁(对账后台真实态)
                                              → N=已配置 → 通知 PM
```

> 💡 **图片不经数字员工传 NOS**：PM 直接发图或图片 URL，数字员工把原始图片素材整包转给技术那套，由技术那套下载 URL、校验尺寸、必要时裁剪确认、重传 NOS、填模板。数字员工只负责"判齐全 + 算业务正确值（跳转链接/角标/资源类型/人群包）"。

**关键决策：**
- §2 匹配项目 — 唯一=静默处理，0或多=问PM
- §2.3 身份校验 — 发物料的人 ≠ 负责人，必须确认
- §3.3 配置方式对照 — 读 H 列，判齐全 + 算业务正确值（**防技术那套默认值出错的核心**）
- §3.4 齐全判断 — 不齐写表存着；齐全转 §3.5 配置

**红线：**
- **素材齐就配，配完直接发审批**（技术那套无暂存态，不做"提前配等排期"）
- **配物料前必须算好业务正确值**（尤其空降链接、数专链接、标准资源端维度默认链接），喂给技术那套，否则它用错误默认值（如空降默认链接不对）。标准资源和换物料不能沿用上游/旧创意 URL，也不能用资源 ID 手拼 `music.163.com`，必须让技术那套重新取资源默认值、读取模板 `preData` 并校验端维度链接：Web 才要求 http/https，非 Web 走客户端默认链接。
- **人工创建/历史推广缺资源初始化时先补齐**：如果后台详情缺 `fullType/resourceTypes/resourceInfo`，不要直接配物料，也不要用推广基础标签或旧创意反推；先让 PM 补实际投放资源类型、标准资源 ID、全量/非全量和非全量人群，再交技术那套按同一套逻辑继续。
- **配完必过收尾门禁**（回查后台真实态对账，见 §3.6），不过不通知 PM
- 物料不齐 → 只写表存着，不配置，**不催**（灵渠助手催）
- 不认识的项目不录入
- 还没预占就发物料 → 告诉 PM 先预占

## 1. 触发条件

**谁说**：PM（项目负责人）
**在哪**：DM（私聊）
**说什么**：发素材——图片/角标/资源ID/跳转链接中任意一种

⚠️ **归属判断**：PM 若没说是 banner 还是 minibanner，先按 `SKILL.md §5.1 裸素材兜底` 反查两表，确认是灵渠 Banner 才走本流程。

PM 在群里发素材 → 群内回「收到，我们私聊对接」→ 立刻主动 DM。

匹配到 0 或多条：

## 2. 匹配项目

### 2.1 提取关键词

从 PM 消息提取项目名关键词，搜追踪表 A 列。

### 2.2 匹配规则

搜表先过滤：跳过 N 列属于终态（已替代 / 已取消）的行。

| 搜索结果 | 处理 |
|---------|------|
| 1 条 | ✅ 静默处理 |
| 0 条 | 问 PM：「没找到匹配的项目，是哪个项目的素材？」 |
| 2+ 条 | 判跨天兄弟：核心名（去 `_MMDD`）相同 + PM 相同 + 日期紧邻 → 物料静默写入所有兄弟行 / 否则列出让 PM 选 |

**匹配后先判「换物料」** — 搜到项目后检查 I-L 列是否已有值。有值 = 换物料，按 N 列状态走 `phase-three-exceptions.md §2`：
- N=「已预占」→ 物料还没配，确认后更新列；若更新后齐全转 §3.5 配置
- N=「已配置」→ 已配并发审批，转换物料流程（exceptions §2，需重新配+重新发审批）

确认话术（匹配 1 条且发料人=负责人时）：
```
检测到「{项目名}」已有旧物料，本次按换物料处理，更新：{变化的列}
回复「是」确认。
```

PM 发图 → 直接收（不再要求云鹿）：数字员工把原图转给技术那套。只需确认「这是哪个项目的素材？」

### 2.3 身份校验

发物料的人 ≠ B 列（项目负责人）→ 必须确认：「这个项目负责人是 {B列}，你代发的素材确定要录入吗？」

## 3. 解析物料 + 算业务正确值

### 3.1 从 PM 消息提取

| 素材 | 识别 |
|------|------|
| 图片 | 图片文件或图片 URL（包括 NOS/NOSK/mb-mlbclaw-pub/nos-jd/p5 等地址），都作为原始图片素材直接转技术那套；数字员工不传云鹿、不写最终图片 URL |
| 资源 ID | 纯数字串，或 PM 说「ID 是 xxx」 |
| 跳转链接 | http/https/orpheus 开头 |
| 角标 | PM 说「角标 xxx」 |
| 人群包 ID | PM 说「人群包 xxx」或纯数字 |

图片 URL 和跳转链接都可能是 http/https，必须按语义区分：PM 说“图片/图/大卡/小卡”或消息附件里的 URL 属于图片素材；PM 说“跳转/链接/落地页/活动页/购票链接”才写入 J 列。图片 URL 不能写入 J 列，也不能作为最终创意图片 URL。

### 3.2 配置方式来源

**直接读追踪表 H 列**（阶段一已识别写入）。不在阶段二重新判断。H 列为空（异常）→ 按 §3.3 PM 描述临时判断并补写 H 列。

若这是人工已创建的推广或追踪表从历史导入，H 列/后台详情可能缺少实际资源初始化字段。此时按 config-matrix §6 只识别配置方式还不够，还必须补齐：

- 实际投放资源类型：标准/歌曲、标准/专辑、标准/歌单、标准/MV，或非标/活动等。
- 标准资源 ID：标准资源必填，作为 `resourceInfo`。
- 投放量：全量 / 非全量。
- 非全量人群：人群包 ID + 包含/不包含。

这些缺失项补齐后，再走 §3.3 计算业务正确值和 §3.5 交接。禁止用 `deliveryResource`、旧创意、上游 URL 反推实际资源类型。

### 3.3 配置方式 → 物料需求 + 业务正确值 ⭐核心（查 config-matrix）

> ⚠️ **完整字段定义见 `config-matrix.md`（唯一真相源），不在此重复。** 阶段二只用其中「§3 物料+业务正确值」部分判齐全、算值。下表是 config-matrix §3 的速查摘录，**以 config-matrix 为准**。

| 配置方式 | 必传图片 | 资源ID（阶段二用） | 跳转链接（业务正确值） | 角标 | 人群包 |
|---------|---------|--------|----------------------|------|--------|
| 标准 | 大卡607+小卡400+蓝框420(可选) | **歌曲/歌单/专辑/MV ID**（PM阶段二提供，直接作为 `resourceInfo`） | 技术那套按资源默认值/模板 preData 重新获取；Web 必须 http/https，非 Web 禁止手拼 Web 链接 | PM提供；未提供时可用资源默认角标 | — |
| 数专 | 大卡607+小卡400+蓝框420(可选) | 数专ID(拼链接) | **按端拼**(见下) | 数字专辑/数字单曲 | — |
| H5 | 大卡607+小卡400+蓝框420(可选) | — | PM提供H5链接 | 独家策划/活动/演出 | — |
| 空降 | **仅大卡607** | 歌曲/专辑ID(拼链接) | **拼orpheus**(见下) | 固定「活动」 | 固定66594/不包含 |
| 有票 | **仅大卡607** | — | PM提供购票链接 | 固定「演出」 | PM提供/包含 |

> ⚠️ **资源ID 在空降/数专只用来拼跳转链接，不是创建字段**（阶段一已确认 resourceInfo=空串）。标准方式的资源ID 阶段一已收进 I 列，阶段二不重收。

**资源类型（喂技术那套的 `resourceType`，见 config-matrix §1）：**
- 标准 → `song/playlist/album/mv`（按子类型）
- 数专/H5/空降/有票 → `activity`

**数专链接自动拼接（按端）：**
- 移动端/荣耀白牌/iPad新版/AndroidPad：`https://music.163.com/st/album/m/album/{数专ID}?nm_style=sbt`
- PC/Web：`https://music.163.com/store/newalbum/detail?id={数专ID}`
- iPadHD旧版/Mac：`https://music.163.com/#/album?id={数专ID}`

**空降链接自动拼接：**
- 空降歌曲：`orpheus://comment?threadId=R_SO_4_{歌曲ID}&autoPlay=1`
- 空降专辑：`orpheus://comment?threadId=R_AL_3_{专辑ID}&autoPlay=1`

> ⚠️ **防技术那套默认值出错的关键**：空降/数专必须喂拼好的链接，否则技术那套用错误默认链接。数字员工先算好写入 J 列，交接时一并喂。

**标准资源默认链接：**
- 数字员工不从旧创意、上游入参或对话记忆里沿用默认 URL；只把资源类型、资源 ID、用户本轮显式覆盖的链接交给技术那套。
- 技术那套必须重新调用资源默认值获取逻辑，并按每个端重新提取模板。Web 端最终链接必须是 http/https；如果资源默认值没有 http/https，数字员工向 PM 补问 Web 链接后再交接。非 Web 端必须使用资源/模板返回的客户端默认链接（如 `commonComposedData.url`、`orpheus`、模板 `preData`）；缺失时问 PM 提供端维度链接，禁止用 Web 链接兜底。
- 如果技术那套已调用 `plan-pack-resource-list` 但无数据，只有标准歌曲允许固定兜底：非 Web 端 `orpheus://song/{歌曲ID}`，Web 端 `https://music.163.com/#/song?id={歌曲ID}`。专辑/MV/歌单等其它标准类型先不使用默认兜底，继续问 PM 提供端维度链接。
- J 列可写端维度摘要，例如 `默认值由技术那套按歌曲ID获取；Web=http(s)链接；非Web=客户端默认链接`，不要把一个 orpheus 链接当作所有端通用链接，也不要把 `https://music.163.com/mv/{id}` 当作所有端通用链接。
- 技术那套组好物料后，必须先把最终端维度链接列表发给 PM 确认；PM 确认后才保存。

**图片尺寸（供判齐全）：** 大卡=1080×607 / 小卡=1080×400 / 蓝框小卡=1080×420
> 空降/有票只需大卡 1080×607；其余方式需大卡607+小卡400(+蓝框420可选)。

### 3.4 物料齐全判断

对照 §3.3，判断 PM 提供的是否齐全：

| 情况 | 处理 |
|------|------|
| 齐全 | 写 I/J/K/L 列 → 转 §3.5 配置 |
| 不齐 | 写已收的物料列，N 保持「已预占」，**结束**（不催，灵渠助手按"待配置+距上线"催料）。可选：回 PM 一句"收到部分素材，还缺 XX"（仅告知，非催促） |

缺料告知要写清参数（不是"缺图片"，而是"缺大卡图 1080×607"）。

## 3.5 配置物料（素材齐后立即执行，直接发审批）

物料齐全后，**不等帧位排期**，立即交技术那套配物料。技术那套配完物料会直接走发审批（带 `businessRemark`），无暂存中间态。

### 3.5.1 交接② 委托技术那套 `promotion-update-creative`

交接②不是在本文件里继续拼请求。素材齐全后，本 skill 只生成交接单并切到 `promotion-apply-skills` 执行；禁止在这里直接调用 `mws link promotion-update-creative`、`backend-nos-token-whalealloc`、`plan-pack-resource-list`、`data-template-extract-private`、`component-code` 或任何 Link HTTP 接口。没有进入 `promotion-apply-skills` 并跑它的 `validate_update_creative_payload.py`，就不能保存物料，也不能把 dry-run 结果当业务校验。

交接②前先清缓存防串料：页面预览里可能残留别人或上一轮物料，不能作为当前素材依据。交接给技术那套时必须写明“为当前 promotionID 新建唯一工作目录，重新查当前环境 promotion-detail，重新获取默认值/模板/图片上传结果，禁止复用旧 `/tmp`、旧 payload、旧 NOS URL 或旧表单”。

**交接给技术那套的输入：**
- promotionID（追踪表 M 列）
- PM 发来的**原始图片素材**（图片文件或图片 URL，整包转，不自己传 NOS；要求技术那套下载 URL 后重传 NOS）
- **算好的业务正确值**：资源类型、资源ID、**拼好的跳转链接或端维度摘要（J列）**、角标、人群包ID；标准资源必须要求技术那套重新获取默认值和模板 `preData`，禁止手拼 URL，Web 缺 http/https 时先补问，最终端维度链接必须回传给 PM 确认
- 全量/非全量（按配置方式）
- 人工创建/历史推广补齐后的 `fullType/resourceType/resourceInfo/resourceTypes/strategyInfo`（如有）
- 审批说明 `businessRemark`（如"物料配置完成，发起审批"）→ 让技术那套直接发起审批
- 清缓存要求：新建当前 promotionID 唯一工作目录，重新查当前环境详情，不能沿用页面缓存或旧临时文件

**技术那套负责：** 下载图片 URL、校验图片尺寸、传 NOS、按端提取模板、组装创意、`promotion-update-creative` 保存并发起审批。

**禁止交接成品创意 JSON：** 数字员工不能直接构造或转交 `promotionAliasCreativeList`、`promotionCreativeList`、`creative`、`templateSchema`、`creativeDetailDataVO`。这些字段必须由技术那套在 `promotion-apply-skills` 内部根据当前 `promotion-detail`、`promotionAliasTemplateList`、`data-template-extract-private`、图片上传结果和预检脚本生成。若上游已经给了这种成品 JSON，数字员工必须拆回业务输入再交接；不能原样提交。否则容易漏掉 `creative.name`、`tenant`、标准资源 `resourceInfo`、所有折叠端补齐、`--resource-defaults` 等校验，导致 500 或“素材不能为空2”。

**数字员工拿回：** 成功 / `needApproval`（技术那套会自动补 businessRemark 发审批）/ 失败。

### 3.5.2 收尾门禁（配完必做，不过不通知 PM）

交接②返回成功后，**回查后台真实态对账**，防"以为配了其实没配"或"A 项目素材配到 B 项目"：

委托技术那套调 `promotion-detail`（id = 本行 M 列），逐项核对：

| 核对项 | 期望值 | 不符处理 |
|--------|--------|---------|
| 推广 ID | = 追踪表本行 M 列 | ❌ 配错行 → **停，报警王可**，不通知 PM |
| 创意/物料已填 | detail 返回创意含本行物料 | 缺/不符 → 重做 §3.5.1 |
| 已发起审批 | status 已进审批流 | 未发起 → 重新交接②补发 |

全过 → N→「已配置」，通知 PM。硬性不符（配错行）→ 不改状态、不通知，报警王可 + 写 O 列。

> 授权（PM 能否审批）由技术那套 `admin-add` 在创建/复制时处理；本阶段门禁不再单独校验授权。

## 6. 边界情况

| 情况 | 处理 |
|------|------|
| PM 分批发物料 | 每次更新追踪表对应列，判断是否齐全 |
