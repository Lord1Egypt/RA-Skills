# Gestión y reportería

## Uso principal

Usar para:
- revisar dashboards del proveedor
- resumir pendientes y métricas operativas
- identificar licitaciones, OC y consultas relevantes sin entrar aún al detalle transaccional

## Paneles observados

- resumen de licitaciones y órdenes de compra
- licitaciones no ofertadas
- licitaciones publicadas en mis rubros
- licitaciones donde aún se pueden hacer preguntas
- órdenes pendientes de aceptar
- licitaciones en seguimiento
- consultas al mercado
- dashboard de gestión de órdenes

## Valor para el skill

Esta capa permite que el skill no sea solo navegacional, sino también:
- radar comercial
- resumen ejecutivo
- priorizador de pendientes
- punto de entrada para workflows posteriores

## Forma de responder

Cuando uses estos módulos, prioriza salidas como:
- “pendientes críticos”
- “oportunidades relevantes”
- “acciones recomendadas hoy”
- “anomalías o bloqueos”

## Regla

Usar reportería para decidir **qué abrir después**, no para reemplazar la revisión de detalle cuando hay una acción sensible en juego.
