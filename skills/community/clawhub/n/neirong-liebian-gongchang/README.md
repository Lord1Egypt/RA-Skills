# 内容裂变工厂

> 小红书热点追踪 × 爆文生成 × 知识卡片 × 公众号发布全链路自动化

## 🎯 业务场景

内容创作者、自媒体运营者、公众号编辑每天面临的核心挑战：

- **找选题太慢**：在小红书、微博、知乎之间来回刷，1-2 小时找不到好选题
- **创作效率低**：找到选题后还要手动整理素材、写文章、配图、排版
- **多平台分发难**：辛苦写完的文章发布到多个平台需要重复操作
- **内容难沉淀**：产出的内容分散在各个平台，无法形成团队知识资产

**内容裂变工厂** 将这套流程完全自动化：输入热点，输出可直接发布的公众号文章 + 精美知识卡片 + 飞书存档。

## 💔 痛点分析

| 痛点 | 影响 | 传统解法 | 本方案 |
|------|------|---------|--------|
| 热点查找耗时 | 每天浪费 1-2h | 人工多平台切换 | 多源聚合，一键抓取 |
| 内容创作慢 | 更新频率低，流量少 | 苦哈哈手动写 | AI 生成 3000-5000 字 |
| 配图制作麻烦 | 发布前还要找图 | 用现成素材（有版权风险） | AI 自动生成知识卡片 |
| 多平台分发 | 操作繁琐易遗漏 | 手动复制粘贴 | 结构化输出，格式适配 |
| 知识无法沉淀 | 好内容散落各处 | 不了了之 | 自动存档飞书文档 |

```
Agent-Reach (多平台热点抓取)

    └──────────┬──────────┘

               │
               │ doc_url
               ▼
        [团队共享/公众号发布]
```

### 各 Skill 职责

| Skill | 输入 | 输出 | 核心能力 |
|-------|------|------|---------|
| **Agent-Reach** | 小红书/微博/知乎 URL | 热搜榜原始数据 | 多平台内容抓取、加密参数自动处理 |
| **summarize** | 原始热搜数据 | 结构化分析报告 | 话题评分、角度建议、传播路径分析 |
| **card-renderer** | 热点话题标题 | 3种风格知识卡片图 | Mac/赛博朋克/包豪斯风自动渲染 |
| **wechat-article-pro** | 分析结果+卡片 | 完整公众号文章 | 深度文章生成、参考刘润风格 |
| **feishu_doc** | 文章+卡片 | 飞书文档 URL | 创建文档、写入内容、权限管理 |

## 📝 使用示例

### 场景：每天 16:30 自动生成内容

**Step 1: 配置定时任务**

```bash
openclaw cron add \
  --name "每日内容裂变工厂" \
  --schedule "30 8 * * *" \
  --skill 内容裂变工厂
```

**Step 2: 查看输出**

```
✅ 今日内容已生成
📄 飞书存档: https://feishu.cn/docx/xxx
📸 知识卡片: 3张（Mac/赛博朋克/包豪斯风）
✍️ 公众号文章: 3500字 | 标题: 【热点解读】为什么...
📅 生成时间: 2026-04-29 16:30 CST
```

### 单次手动执行

```bash
# 全流程：抓取 → 分析 → 生成 → 存档
openclaw run 内容裂变工厂

# 仅生成草稿（不存档）
openclaw run 内容裂变工厂 --draft true

# 指定行业领域
openclaw run 内容裂变工厂 --niche 科技
```

## 🔧 高级配置

### 自定义热点来源

编辑 `workflow.json` 中的 `sources`：

```json
{
  "sources": {
    "xiaohongshu": {
      "enabled": true,
      "categories": ["时尚", "美食", "科技", "职场", "旅行"]
    },
    "weibo": {
      "enabled": true,
      "types": ["hot", "fellow"]
    },
    "zhihu": {
      "enabled": true,
      "categories": ["热搜", "推荐"]
    },
    "rss": {
      "enabled": true,
      "feeds": ["https://www.zhihu.com/rss"]
    }
  }
}
```

### 指定卡片风格

```json
{
  "card_styles": ["mac", "cyberpunk"],
  "card_count": 2
}
```

### 定时执行配置

```json
{
  "schedule": {
    "cron": "30 8 * * *",
    "timezone": "Asia/Shanghai",
    "enabled": true
  }
}
```

## 🚨 注意事项

1. **选题时效性**：热点生命周期通常 24-72h，建议尽早创作发布
2. **内容原创**：AI 生成内容需二次加工，确保原创性，遵守平台规则
3. **频率控制**：建议每日执行 1 次，避免过度采集触发限制
4. **数据隐私**：抓取数据仅用于内容创作，不可商用
