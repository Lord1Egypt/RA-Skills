# 信息收集清单

当用户提交公司名称+岗位信息后，按以下优先级收集信息。

## 必须收集（用户可能已提供）

- [ ] 公司名称
- [ ] 岗位名称
- [ ] JD 关键内容（职责、要求）
- [ ] 标注薪资区间（如有的话）

## 需要搜索补充的信息

### 公司层面（web_search）

1. **公司基本信息**
   - 融资轮次、金额、投资方
   - 员工规模
   - 成立时间
   - 是否上市

2. **工作文化与口碑**
   - 搜索关键词：`[公司名] 工作体验`、`[公司名] 加班`、`[公司名] 脉脉`、`[公司名] 看准网`
   - 关注：加班情况、双休/大小周、管理层评价、离职率信号

3. **公司新闻与信号**
   - 最近是否有融资、裁员、业务调整等新闻
   - 行业趋势判断

4. **技术氛围**
   - 是否有技术博客/开源项目
   - 技术栈是否为主流方向

### 岗位层面（综合判断）

- 岗位在团队中的位置（核心业务线 vs 支撑部门）
- 技术栈匹配度（对照用户简历中的技术栈）
- 岗位是否为新增HC（增长信号）还是替换HC（中性/负面信号）
- 任职要求中是否有明显不合理之处（如要求既会前端又会算法还会运维 = 可能是填坑岗）

## 信息不足时的处理

如果某个维度无法判断：
- 不要猜测，标记为"信息不足"
- 向用户询问：列出需要用户补充的具体问题（不超过3个）
- 用户无法提供则该维度按中间分值处理（不加分不扣分）

## 用户补充信息的模板

向用户提问时使用简洁格式：

```
关于 [公司名] 的 [岗位名]，我还需要确认以下几点：

1. [具体问题1] — 你了解吗？
2. [具体问题2] — JD里有写吗？
3. [可选：具体问题3]

如果不太清楚也没关系，我先用已有信息评分。
```
