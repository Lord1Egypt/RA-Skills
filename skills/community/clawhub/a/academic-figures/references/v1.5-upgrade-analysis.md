## 5 Proposed Upgrades (v1.5.0)

### 1. Horizontal Bar Chart `-t hbar` (⭐ Easy)
**Pattern**: `fig1`, `fig2` — horizontal bars, model names on Y-axis, percentages at bar ends.
**Implementation**: Add `--horizontal` flag to `gen_bar()`, or separate `gen_hbar()`.
- Use `ax.barh()` instead of `ax.bar()`
- Value labels at bar ends (right side), not on top
- Good for many categories (model names are long)
**Medical use case**: Multi-drug comparison, multi-center study, lab panels

### 2. Ratio/Fold-Change Annotations `--show-ratio` (⭐ Easy)
**Pattern**: `fig6` — each bar pair shows multiplier like "4.96x" above.
**Implementation**: In `gen_bar()`, when ≥2 series, compute ratio of series[1]/series[0] per group, annotate above the taller bar.
```python
# Pseudocode
for j in range(n_groups):
    ratio = values_series2[j] / values_series1[j]
    ax.text(x[j], max_val + offset, f'{ratio:.2f}×', ha='center', fontweight='bold')
```
**Medical use case**: Fold change, OR/RR ratios, treatment effect multipliers

### 3. Multi-Panel Composite `-t composite` (⭐⭐ Medium)
**Pattern**: `fig4` — left panel (architecture flow) + top-right (line graph) + bottom-right (bar chart).
**Implementation**: Use `matplotlib.gridspec.GridSpec` with user-defined panel layout.
- JSON schema: `{"panels": [{"type": "bar", "position": [0,:], "data": {...}}, ...]}`
- Each panel reuses existing generators (`gen_bar`, `gen_line`, etc.)
- Shared legend option
**Medical use case**: Figure 1A+B+C (bar + KM + ROC), common in clinical papers

### 4. Dark Theme `--theme dark` (⭐ Easy)
**Pattern**: `fig5` — black background, bright colored blocks, high contrast.
**Implementation**: Add to THEMES dict:
```python
"dark": {
    "figsize": (10, 6),
    "dpi": 600,
    "font_size": 11,
    "colors": ["#009E73", "#E69F00", "#56B4E9", "#CC79A7", "#F0E442",
               "#D55E00", "#0072B2", "#FFFFFF"],
    "grid_alpha": 0.15,
    "spines": ["top", "right"],
    "colorblind_safe": True,
    "bg_color": "#0D1117",
    "text_color": "#E6EDF3",
}
```
- Set `fig.patch.set_facecolor()` and `ax.set_facecolor()`
- Invert text colors to light
**Medical use case**: Conference presentations, dark-mode reports

### 5. Architecture/Flow Diagram `-t diagram` (⭐⭐⭐ Hard)
**Pattern**: `fig5` — colored blocks with labels, arrows showing data flow, step annotations.
**Implementation**: `matplotlib.patches.FancyBboxPatch` + `FancyArrowPatch`.
- JSON schema: `{"boxes": [{"id": "A", "label": "MPT Layer", "color": "#009E73", "pos": [x,y,w,h]}], "arrows": [{"from": "A", "to": "B"}], "steps": [...]}`
- Math notation via matplotlib's mathtext (`$h_{t-1}$`)
**Medical use case**: CONSORT flow diagrams, treatment pathways, study design schematics

## Implementation Priority
1. **hbar** + **ratio** (fastest, highest ROI for medical papers)
2. **dark theme** (quick win)
3. **composite** (medium, very common in clinical journals)
4. **diagram** (hardest, lower frequency but high impact)

## Session Technique: Extracting Content from JS-Rendered SPA Blogs
When target page is a Vite/React SPA (curl returns `<div id="root"></div>` only):
1. Find the JS bundle URL in `<script src="...">` tag
2. `curl -sL 'https://site/assets/bundle-XXXX.js'`
3. Regex-extract: image URLs (`https?://...\.(png|jpg|svg)`), alt text (`alt:\`...\``), heading text (`t\.h2,\{children:\`...\``), body text (backtick-delimited strings >30 chars)
4. Match images to context by extracting text blocks before each `src:` reference
5. Download original images via curl for high-res VL analysis (much better than compressed Feishu thumbnails)
