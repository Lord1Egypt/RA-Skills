# 知识树结构

中学生编程启蒙的知识体系，按学习顺序组织。

---

## 第一阶段：入门体验（1-3小时）

### 核心目标
完成第一个项目，体验"我能做出东西"

### 知识点
- [ ] 什么是编程
- [ ] 运行代码的方法
- [ ] 变量（给数据起名字）
- [ ] 输入输出（和电脑对话）
- [ ] 条件判断（if/else）

### 项目
- 猜数字游戏
- 简单问答程序
- 文字冒险游戏

### 解锁成就
🎮 "第一个游戏" - 完成猜数字游戏
💬 "电脑会说话了" - 让电脑输出文字
🎯 "做选择" - 使用条件判断

---

## 第二阶段：基础能力（5-10小时）

### 核心目标
理解基本概念，能独立写简单程序

### 知识点
- [ ] 循环（for/while）
- [ ] 数据类型（整数、文字、布尔）
- [ ] 列表（存多个数据）
- [ ] 函数（把代码打包）
- [ ] 随机数

### 项目
- 猜词游戏（Hangman）
- 文字冒险游戏升级版
- 简单计算器
- 密码生成器

### 解锁成就
🔄 "重复大师" - 使用循环
📦 "收纳高手" - 使用列表
⚡ "快捷方式" - 定义函数

---

## 第三阶段：进阶应用（10-20小时）

### 核心目标
能设计并完成中等复杂度项目

### 知识点
- [ ] 字典（键值对）
- [ ] 文件操作（读写）
- [ ] 错误处理（try/except）
- [ ] 模块使用
- [ ] 基本调试技巧

### 项目
- 个人日记本（保存到文件）
- 单词学习工具
- 简单待办事项
- 网页数据抓取入门

### 解锁成就
💾 "数据保管员" - 操作文件
🛡️ "防御编程" - 处理错误
🔧 "工具收集者" - 使用3个以上模块

---

## 第四阶段：项目实践（20+小时）

### 核心目标
独立完成完整项目，培养解决问题的能力

### 知识点
- [ ] 项目规划
- [ ] 代码组织
- [ ] 注释和文档
- [ ] 基本测试
- [ ] 版本控制（Git基础）

### 项目方向（根据兴趣选择）
- 游戏：文字RPG、图形游戏（Pygame）
- 工具：自动化脚本、数据处理
- Web：简单网站（Flask）
- AI：调用API做聊天机器人

### 解锁成就
🏗️ "项目大师" - 完成完整项目
👥 "团队协作" - 使用Git
🚀 "发布者" - 分享你的程序

---

       │

## 概念掌握标准

### ✅ 已掌握
- 能独立使用
- 能解释给同学听
- 能在不同场景应用

### 🔄 学习中
- 知道是什么
- 需要看提示使用
- 偶尔出错

### ❓ 未学习
- 没接触过
- 或者接触过但忘记了

## 学习路径建议

### 路径 A：游戏方向
```
猜数字 → 文字冒险 → 猜词游戏 → 文字RPG → 图形游戏
```

### 路径 B：工具方向
```
简单问答 → 计算器 → 密码生成器 → 文件处理 → 自动化脚本
```

### 路径 C：数据方向
```
简单问答 → 数据收集 → 数据处理 → 可视化 → 简单分析
```

## 推荐资源

### 在线练习
- Codecademy（英文，互动式）
- LeetCode 简单题（有一定基础后）
- CheckIO（游戏化学习）

### 视频教程
- B站搜索"Python入门"（中文）
- 选择"项目驱动"类型的教程

### 书籍
- 《Python编程：从入门到实践》（有中文版）
- 《笨办法学Python》（适合自学）

---

## 注意事项

1. **不要追求完美** - 能运行的烂代码 > 完美设计的代码
2. **多动手少看书** - 编程是技能，不是知识
3. **遇到错误是正常的** - 调试是学习的一部分
4. **按兴趣深入** - 不感兴趣的内容可以跳过
5. **定期回顾** - 学新东西时复习旧知识
