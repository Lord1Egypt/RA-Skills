# 专业剧本杀作品分析框架

> **本文档是 murder-mystery-creator skill 的核心方法论。**
> 目标：像专业剧本杀研发团队一样，拆解任何一部作品，提取可复用结构。
> 适用：诡秘之主、东君书院、荒野大侠客、神秘复苏等所有参考作品。

---

## 分析维度总览

每部作品按以下 **6个维度** 拆解：

```
维度1：结构节拍 → 起承转合的精确时间节点
维度2：玩家体验地图 → 每个阶段的情绪曲线和认知状态
维度3：机制设计 → 线索/阵营/规则的工程实现
维度4：沉浸感工程 → 从"读剧本"到"活在故事里"的手段
维度5：复盘设计 → 结束后玩家带走的情感和认知
维度6：视觉素材分析 → 角色肖像/线索卡/场景地图/道具图/封面的设计规律
```

---

## 维度1：结构节拍分析

### 目标
把剧本拆成**可量化的节拍表**，标注每个节拍的功能、时长占比、信息传递量。

### 模板

```
## [作品名] 结构节拍表

| 节拍 | 内容 | 时长占比 | 信息释放量 | 玩家情绪 | 设计目的 |
|------|------|---------|-----------|---------|---------|
| 开场 | [发生了什么] | X% | 低/中/高 | 好奇/紧张 | [为什么这样安排] |
| 第一轮搜证 | [搜哪些地点] | X% | ... | ... | ... |
| 第一次质询 | [讨论什么] | X% | ... | ... | ... |
| ... | | | | | |

核心节拍（必须标注）：
- 第一个悬念点：出现在第几分钟？用什么手段制造？
- 第一次怀疑转移：玩家从怀疑A转到怀疑B，靠什么线索触发？
- 终幕反转：反转的信息量有多大？玩家能否在反转后复盘出自洽逻辑？
```

### 使用方式
生成新剧本时，先选定1-2部参考作品，套用其节拍表结构，再替换具体内容。

## 维度2：玩家体验地图

### 目标
站在**玩家视角**，画出从入场到离场的完整情绪曲线和认知曲线。

### 模板

```
## [作品名] 玩家体验地图

### 认知曲线（玩家知道什么）
入场时：知道[XX]，不知道[XX]
第一轮后：新增知道[XX]，误以为[XX]
第二轮后：开始怀疑[XX]，但[XX]仍是盲点
终极线索后：所有盲点消除，真相浮现

### 情绪曲线（玩家感受到什么）
0-10分钟：[情绪状态，如：好奇/兴奋]
10-30分钟：[情绪状态，如：紧张/怀疑]
30-60分钟：[情绪状态，如：争论/自信]
60-90分钟：[情绪状态，如：震惊/感动]

### 关键信息释放节奏
- 必须在第一轮释放的信息：[列出，说明为什么不能更晚]
- 可以保留到第二轮的信息：[列出，说明保留的戏剧效果]
- 必须留到终极轮的信息：[列出，说明为什么是终极揭秘的关键]

### "啊哈时刻"设计（玩家恍然大悟的瞬间）
[描述作品中玩家最强烈的"原来如此"时刻，分析它为什么有效]
```

## 维度3：机制设计分析

### 目标
拆解剧本的**可操作性机制**：线索密度曲线、阵营胜负条件、反转触发条件。

### 模板

```
## [作品名] 机制设计拆解

### 线索密度曲线
| 轮次 | 线索数 | 有效线索 | 误导线索 | 密度评价 |
|------|--------|---------|---------|---------|
| 第一轮 | X条 | X条 | X条 | [是否足够/过多] |
| 第二轮 | X条 | X条 | X条 | ... |
| 终极轮 | X条 | X条 | X条 | ... |

线索设计原则（从该作品总结）：
- 原则1：[如：每条线索必须有"表面解读"和"深层解读"两层]
- 原则2：[如：误导线索不能是完全无关的，必须是"对真相的错误理解"]

### 阵营/胜负机制（如有）
- 阵营划分依据：[玩家根据什么被分到不同阵营？]
- 阵营信息差：[不同阵营知道的秘密是否不同？如何体现？]
- 胜负判定条件：[什么情况下哪个阵营赢？是否允许多阵营共存？]
- 终幕可变结局：[哪些变量决定结局走向？]

### 反转机制
- 反转数量：[有几个反转？]
- 每个反转的信息基础：[玩家在反转前已经知道什么？反转后如何重新解释？]
- 反转是否可预判：[细心玩家能否在终极轮前猜到反转？应该/不应该？]
```

## 维度4：沉浸感工程

### 目标
分析作品如何把"读剧本"变成"活在故事里"——不是靠堆设定，而是靠**具体的工程手段**。

### 模板

```
## [作品名] 沉浸感手段拆解

### 代入感构建手段
1. [手段名称，如：用物件代替形容词]
   - 该作品怎么做：[具体例子]
   - 为什么有效：[认知心理学原理或玩家体验原理]
   - 可迁移到：[其他题材如何复用这个手段]

2. [手段名称，如：信息不对称]
   - ...

### "在场感"营造检查表
- [ ] 玩家是否有"我现在就在那个场景里"的感受？（如何验证？）
- [ ] 对话是否像"那个世界里的人"说出来的？（不是现代口语套古装）
- [ ] 玩家的决策是否影响剧情走向？（还是纯推理解谜？）
- [ ] 是否有"只有亲身体验才能理解"的瞬间？（而非读完剧本就能知道）

### 诡秘之主专属沉浸手段（参考）
- 短句+感官代替心理描写 → 玩家自己补充情绪=更强代入
- 物件质感（煤气灯/怀表）→ 具体物件比抽象形容词更能触发画面感
- 潜台词对话 → 玩家在游戏中"发现"隐藏含义，比直接被告知更有成就感
- 信息差 → "我知道你不知道的事"是极强的沉浸驱动力
```

## 维度5：复盘设计分析

### 目标
分析作品在游戏结束后，如何让玩家**带走情感余韵和讨论话题**。

### 模板

```
## [作品名] 复盘设计拆解

### 终幕情感落点
- 真相揭示后，玩家的第一反应是什么？[震撼/感动/遗憾/后怕？]
- 这个反应是如何被前面60-90分钟铺垫出来的？

### 可讨论话题（玩家离场后还会聊什么）
1. [话题1，如：如果是你，你会做同样的选择吗？]
2. [话题2，如：XX角色的秘密如果是我，我会怎么处理？]
3. ...

### "二刷价值"设计（如果有的话）
- 该作品是否值得二刷？为什么？
- 二刷时玩家会关注什么第一次忽略的细节？

### 情感余韵设计
- 最后一个画面/台词是什么？
- 它想让玩家带走什么感受？
- 这个感受是否在游戏过程中有充分铺垫？
```

## 维度6：视觉素材分析

### 目标
拆解剧本的视觉呈现设计规律，包括角色肖像风格、线索卡设计、场景地图、道具图和封面。

### 模板

```
## [作品名] 视觉素材分析

### 角色肖像设计规律
- 风格定位：[如：水墨淡彩古风立绘/维多利亚油画质感/现代插画扁平风]
- 构图规范：[如：半身像居中/全身像+场景背景]
- 视觉元素：[如：花魁头饰+团扇/维多利亚礼服+怀表/道袍+剑穗]
- 排版规律：[角色名位置/简介排版/尺寸规格]

### 线索卡设计规律
- 线索卡覆盖率：[如：30%的线索有配图，核心物证必配]
- 风格处理：[如：古纸泛黄质感/写实物证/羊皮纸做旧]
- 信息层级：[如：编号→物品名→描述→指向标注]
- 尺寸规格：[如：横向16:9 A5]

### 场景地图设计规律
- 是否需要地图：[封闭空间必备/开放空间可选]
- 俯视图规范：[如：水墨线条无填充/建筑蓝图色块分区]
- 标注体系：[如：中文标注关键位置/右下角图例]

### 道具图设计规律
- 哪些道具值得做图：[如：核心物证/有叙事功能的道具/可作为复刻品的周边]
- 风格处理：[如：实物照片风格/手绘插画风格]
- 排版规范：[如：物品居中+名称在底部]

### 封面/海报设计规律
- 核心视觉元素：[如：青楼夜景主体+花魁意象/维多利亚城市+煤气灯氛围]
- 排版结构：[如：上方大字标题+下方标签行]
- 尺寸规格：[如：16:9横向1920×1080]

### 可迁移视觉设计要素
1. [视觉要素名，如：古风角色用淡彩水墨立绘]
   - 本作实现：[具体方式]
   - 可迁移到：[新题材如何复用]
2. ...
```

## 综合应用：新剧本生成流程

当你需要生成新剧本时，按此流程使用本框架：

```
第一步：选择1-2部参考作品
  → 从本框架中调取它们的拆解结果

第二步：提取结构图谱
  → 节拍表、情绪曲线、线索密度曲线

第三步：提取视觉设计要素
  → 角色肖像风格、线索卡覆盖率、封面视觉主元素
  → 这些信息用于第七步"可视化素材生成"自动匹配风格

第四步：迁移到新题材
  → 保留结构，替换内容
  → 例：诡秘之主的信息差结构 → 古风悬疑的信息差结构

第五步：填入具体内容
  → 按结构图谱的每个节拍填入新剧本的具体内容

第六步：用框架自检
  → 对照6个维度的检查表，确认新剧本在每个维度上都有设计
```

## 剖析结果缓存机制

> **目标**：同一作品被剖析后，结果应被记录复用，避免重复剖析。

### 缓存读取流程

```
在 Mode B 剖析前：
1. 检查"已拆解作品索引"中该作品的状态
2. 如果已有"✅ 已拆解"标记：
   → 直接读取其"可迁移结构要素"（6条）
   → 跳过全维度剖析，直接进入生成流程
3. 如果"待拆解"标记：
   → 执行全维度剖析
   → 剖析完成后，将"可迁移结构要素"写入索引对应行
   → 将状态更新为"✅ 已拆解（YYYY-MM-DD）"
```

### 缓存记录格式

在"已拆解作品索引"中，每个作品追加以下字段：

```
| 作品 | 风格标签 | 拆解完成度 | 核心可学习方法 | 可迁移结构要素 |
|------|---------|------------|--------------|---------------|
| 诡秘之主 | H | ✅ 已拆解（2026-05-11） | 信息差结构... | [6条结构要素] |
| ... | | | | |
```

> 每次成功剖析后，更新对应行的"拆解完成度"和"可迁移结构要素"字段。
> 如需更新已有剖析（发现更好的理解），用新结果覆盖"可迁移结构要素"，并更新日期。

### 缓存失效条件

- 用户说"重新剖析"/"更新剖析"/"深入分析"→ 清除旧缓存，重新剖析
- 作品风格标签变更 → 清除旧缓存，重新剖析
- 超过6个月未更新的剖析 → 提示"是否更新"，由用户决定
