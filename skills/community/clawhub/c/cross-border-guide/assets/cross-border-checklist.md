# Cross-Border Quality Checklist

Run before delivering any cross-border plan or troubleshooting report. Unchecked = fix or disclose in Assumptions.

## Classification & valuation
- [ ] Customs-grade product description written (material + function)
- [ ] HS code reasoned from composition/function, not product name
- [ ] Destination national code (not just origin lookup) referenced
- [ ] Ambiguous classifications listed with alternatives; ruling recommended where volume justifies
- [ ] Valuation = transaction value; basis (CIF/FOB) confirmed per market
- [ ] Declared value consistency rule stated (invoice = receipt = payment record)

## Duties, taxes & thresholds
- [ ] De minimis checked separately for duty and tax, per market
- [ ] Channel difference (postal vs. courier) accounted for
- [ ] VAT/GST computed on the correct base (value + duty + freight where applicable)
- [ ] Brokerage/disbursement/handling fees included in landed cost
- [ ] Registration triggers checked (IOSS, UK VAT, AU/NZ GST); marketplace deemed-collector confirmed where relevant
- [ ] All thresholds/rates tagged "verify current"
- [ ] FTA preference only claimed with documentable origin

## Incoterms & customer experience
- [ ] DDP vs. DDU decided per market with stated rationale
- [ ] Checkout disclosure described (what customer pays, when)
- [ ] Refusal-risk handling defined for any DDU lane

## Carriers & restrictions
- [ ] Carrier class matched to AOV/speed/tracking per lane
- [ ] Delivery cost ≤25-30% of AOV guardrail checked
- [ ] Product checked against destination restricted lists
- [ ] Product checked against each carrier's own restrictions (batteries, liquids)
- [ ] Compliance gates (CPNP, type approval, permits) listed as blocking items

## Documentation
- [ ] Commercial invoice field spec complete
- [ ] Description ↔ HS ↔ value consistency rules stated
- [ ] Required certificates identified (origin, MSDS, UN38.3, ingredients)
- [ ] No gift-marking / vague-description practices anywhere

## Operations
- [ ] Customs-hold escalation SOP included (for programs)
- [ ] Returns policy defined (refund-without-return threshold, hubs, RTS economics)
- [ ] Per-market launch blockers separated from nice-to-haves

## Output quality
- [ ] Correct template used (market-entry vs. troubleshooting)
- [ ] Executive summary ≤120 words
- [ ] Estimates marked "(est.)"; sources typed (tariff schedule / calculator / quote)
- [ ] Broker/professional verification recommended for high-stakes figures
- [ ] Assumptions numbered in their own section
