# $ralplan — 共识规划工作流

## 触发条件
- 需求已明确，需要制定实现计划
- 用户说"规划一下"、"制定计划"
- 需要多方视角审查方案

## 执行流程

### Phase 1: Planner 制定计划
```
Agent: Architect (GPT-5.5)
输入: 需求文档
输出: 初步实现计划
- 模块划分
- 接口设计
- 技术选型
- 实现步骤
```

### Phase 2: Architect 审查
```
Agent: Architect (GPT-5.5) [不同视角]
输入: 初步计划
输出: 架构审查意见
- 可扩展性评估
- 性能瓶颈识别
- 技术风险评估
```

### Phase 3: Critic 改进
```
Agent: Reviewer (GPT-5.5)
输入: 计划 + 架构审查意见
输出: 改进建议
- 代码质量风险
- 安全隐患
- 边界情况处理
```

### Phase 4: 达成共识
```
综合三方意见，输出最终计划
```

## 输出格式

```markdown
# 实现计划

## 概述
[计划概述]

## 模块划分
### 模块 1: [名称]
- 职责: [描述]
- 接口: [API 定义]
- 依赖: [依赖列表]

## 实现步骤
1. [步骤 1]
2. [步骤 2]

## 技术选型
- [技术 1]: [选型理由]

## 风险评估
- [风险 1]: [缓解措施]

## 验收标准
- [ ] [标准 1]
```
