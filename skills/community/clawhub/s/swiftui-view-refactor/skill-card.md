## Use Case: <br>
Developers and engineers use this skill to refactor SwiftUI views into consistent, maintainable structure while preserving behavior and applying MV-first, dependency injection, and Observation guidance. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Proposed SwiftUI refactors could introduce incorrect behavior or misleading guidance if applied without review. <br>
Mitigation: Review proposed code changes before applying them and verify behavior remains intact. <br>
Risk: Examples or refactors that touch app data operations, including SwiftData deletes, could affect application data if copied directly. <br>
Mitigation: Check data-operation changes carefully against the app's intended behavior before deployment. <br>
