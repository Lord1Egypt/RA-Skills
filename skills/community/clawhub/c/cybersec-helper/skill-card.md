This skill is ready for commercial/non-commercial use. <br>

## Use Case: <br>
Developers, security reviewers, and bug bounty participants use this skill to structure authorized application security reviews, vulnerability classification, secure coding advice, and bug bounty report planning. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Security guidance can be misapplied to unauthorized or out-of-scope targets. <br>
Mitigation: Keep targets explicitly authorized and scoped; prefer local or lab reproductions when authorization or production impact is unclear. <br>
Risk: Vulnerability analysis can become misleading if source references are missing, stale, or invented. <br>
Mitigation: Use authoritative sources such as OWASP, CWE, CVE, NVD, Exploit Database, and vendor advisories; avoid inventing identifiers or vulnerability details. <br>
