# Couple Rules — 双人合盘规则

为"我和 xx 今天合不合"、"双人合盘"、"约会合盘"等查询服务。

## 通用流程

1. 收集两人生日(必)、姓名(可选,仅用于输出称呼)、性别(可选,用于话题建议侧重)。
2. 按 `fortune_rules.md` §1-2 分别计算 `B1, B2, R1, R2, S1, S2`。
3. 计算合盘 `R_couple` 与兼容度 `C`(见 §1-2)。
4. 按 §3 输出"今日相处氛围 / 共同宜做 / 共同忌做 / 沟通建议"。
5. 不要直接公开双方的"个人评级",只输出"合盘评级",避免一方被判读为"拖累"另一方。

## 1) 合盘 R 与 兼容度公式

```
R_couple = ((B1 + B2) * 131 + D * 17) mod 1000
gap      = abs(R1 - R2)
match    = clamp(50 + (R_couple mod 41) - (gap div 30), 0, 100)
```

### 生肖加分(可选)

若知道两人生年:

| 生肖关系 | bonus |
|---|---|
| 三合(鼠龙猴 / 牛蛇鸡 / 虎马狗 / 兔羊猪) | +8 |
| 六合(鼠牛 / 虎猪 / 兔狗 / 龙鸡 / 蛇猴 / 马羊) | +6 |
| 同生肖 | +3 |
| 六冲(鼠马 / 牛羊 / 虎猴 / 兔鸡 / 龙狗 / 蛇猪) | -8 |
| 相害(鼠羊 / 牛马 / 虎蛇 / 兔龙 / 申亥 / 酉戌) | -4 |
| 其他 | 0 |

```
C = clamp(match + zodiac_bonus, 0, 100)
```

兼容度评级:

| C | 评级 |
|---|---|
| 80-100 | 默契 |
| 65-79  | 顺畅 |
| 45-64  | 平稳 |
| 0-44   | 磨合 |

## 2) 沟通气场

由 `gap` 决定今日两人能量差距:

| gap 范围 | 描述 |
|---|---|
| 0-100   | 同频 — 几乎同步,对方一个眼神就能接住 |
| 101-300 | 互补 — 节奏不同但能拼起来,一方主动带节奏 |
| 301-500 | 错位 — 容易"我说东你回西",需要明确表达 |
| 501-999 | 反向 — 今天彼此状态差异大,以休息和不打扰为主 |

## 3) 输出模板

```
【双人合盘 | YYYY-MM-DD | <名 1> × <名 2>】

💞 今日兼容度:<评级> (C 分 / 100)
🌬️ 沟通气场:<同频 / 互补 / 错位 / 反向> — 1 句解释

🌟 相处建议
• 主调:基于 C 分,推荐"<热烈推进 / 平稳享受 / 留白克制 / 各自安好>"
• 适合一起做的事:<2-3 件>
• 不建议今天做的事:<2-3 件>

✅ 共同宜做
1. <动作>(基于 R_couple mod 9)
2. <动作>
3. <动作>

❌ 共同忌做
1. <规避>
2. <规避>
3. <规避>

💬 沟通建议
• 哪一方今天能量更高:<名 X>(S 较高的一方)— 由其主动开话题更顺
• 对方需要的方式:<倾听 / 鼓励 / 给空间>
• 一句话避雷:<基于 gap 给一句具体的"今天不要说..."建议>

- 禁忌事项:今日不宜<重大决定 / 冷战 / 当面争辩>
- 时间窗:今日全天

```

## 4) 共同宜池(12 条,索引 `(R_couple + i) mod 12`)

1. 一起吃一顿不赶时间的饭
2. 共同完成一件小事(整理 / 烹饪 / 散步)
3. 各自做事但同处一室
4. 一起看一部老电影或老剧
5. 把心里话写下来交换看
6. 一起做 30 分钟运动
7. 一起选一件即将共同使用的物品
8. 留出 1 小时单独陪伴(关手机)
9. 一起在户外走 20 分钟
10. 一起回顾过去一周做对了哪些事
11. 一起做一份明天的计划
12. 一起做一件 5 分钟以内的"对方喜欢但你以前没主动做"的事

## 5) 共同忌池(12 条)

1. 翻旧账 / 重提历史争议
2. 当对方面情绪化回复第三方消息
3. 同时刷各自手机超过 30 分钟
4. 谈钱 / 谈未来重大决策(若 C < 65)
5. 邀约第三方加入两人独处时间
6. 用"你总是""你从来不"句式
7. 把今天的累 / 烦带到对方身上发泄
8. 互相比较收入 / 工作压力
9. 在公共场合争论
10. 用消息冷战(C 低时尤忌)
11. 一方主导今晚所有选择(选哪个餐厅 / 看哪部剧)
12. 当面评价对方家人 / 朋友

## 6) 沟通建议生成规则

由 `gap` 区间 + 两人 S 差距决定一句具体提示:

| 条件 | 建议示例 |
|---|---|
| gap < 100 且 S 双方都 ≥ 70 | "今天默契度高,适合聊未来 1-2 周的小计划,不要谈太远" |
| gap 100-300 | "一方主动带节奏,另一方多给反馈;避免'随便''都行'" |
| gap 300-500 | "今天容易理解错位,重要事用文字写一遍再说" |
| gap > 500 | "今天彼此状态差大,以陪伴和不打扰为主,不要硬聊深度话题" |
| S1 - S2 > 20 | "状态低的一方今天主要被倾听,不要被劝太多" |
| S1 + S2 < 100 | "今天双方都偏疲,降低期待,完成一件小事就够" |

## 7) 强禁忌触发

`C < 55` 时必须输出 1 条强禁忌,从下列触发对应:

| 触发 | 禁忌事项 |
|---|---|
| `C < 45` | 不宜做重大共同决定(买房 / 同居 / 见家长安排) |
| `gap > 500` | 不宜安排长时间共同活动(超过 4 小时) |
| `(R_couple mod 5) == 0` | 不宜邀第三方加入独处时间 |
| 其他 `C 45-55` | 不宜在情绪未平静前讨论分工 / 钱 / 家庭关系 |

每条强禁忌包含:**禁忌事项 / 时间窗 / 原因 / 替代方案**(默认时间窗 = 今日 06:00 - 次日 06:00)。

## 8) 隐私约束

- **绝不**在输出里逐字重复对方的生日;输出时只用名字或"对方"代称
- 若用户没给对方名字,用"对方"
- 若用户没给生日,告知"需要双方生日才能做合盘,你可以让对方告诉你 ta 的生日(年月日就够)"
- 不存储第二人的档案,除非用户明确说"记住 ta"。即使存储,也只存生日 + 名字,不存性别/地点等。
