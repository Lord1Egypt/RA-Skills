# 用户反馈模板参考

所有反馈均通过 Claw 的微信响应通道发送。严禁发送原始错误堆栈。

---

## 标准反馈模板

```
⚠️ 任务中止：[任务简述]
原因：[错误类型]
建议：[可选，给出替代方案或说明限制]

已就绪，可以发送下一条指令。
```

---

## 场景化模板

### 场景 1：单文件/单操作权限不足

```
⚠️ 任务中止：读取 /var/log/system.log
原因：权限不足（需要 root 权限）
建议：可尝试在终端手动执行 sudo 后重试

已就绪，可以发送下一条指令。
```

### 场景 2：批量任务中部分文件跳过

```
完成：处理了 8/10 个文件
跳过：
  - /etc/hosts（权限不足）
  - /var/log/auth.log（访问被拒）

其余文件已正常处理。已就绪，可以发送下一条指令。
```

### 场景 3：连续多条指令均因权限失败

```
⚠️ 注意：最近 3 条指令均因权限问题中止。
Claw 当前运行环境可能缺少必要的文件系统权限。
建议检查：Claw 是否有访问目标目录的权限？
暂停执行，等待你的指示。
```

### 场景 4：微信通道本身不可用（极端情况）

AI 无法向用户发送反馈时，静默写入日志：
```bash
echo "[$(date)] 权限错误恢复：<操作描述> | <错误类型>" >> ~/claw_recovery.log
```

## 禁止的反馈格式（反面示例）

```
# ❌ 禁止：发送原始错误堆栈
Error: EACCES: permission denied, open '/var/log/system.log'
    at Object.openSync (fs.js:462:3)
    at Object.readFileSync (fs.js:364:35)

# ❌ 禁止：暴露系统路径给用户
无法访问 /Users/admin/.ssh/id_rsa，权限被拒

# ❌ 禁止：技术性描述
errno: -13, code: 'EACCES', syscall: 'open'
```

---

## 反馈语言原则

1. 用中文，简洁，不超过 3 行（场景 2 除外）
2. 不使用技术术语（用"权限不足"代替 "EACCES"）
3. 不暴露完整文件路径，只说"系统日志文件"等概括性描述
4. 始终以"已就绪，可以发送下一条指令"结尾（场景 3 除外）
