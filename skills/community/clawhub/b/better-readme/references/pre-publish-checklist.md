# Pre-Publish Checklist

Run through this list before launching your project on GitHub.

## Critical (Must Have)

- [ ] **README exists** — `README.md` in repo root
- [ ] **Project name is clear** — Not just "my-tool", has a real name
- [ ] **One-line description** — In first 5 lines, says what it does
- [ ] **Install instructions** — ≤3 steps to get running
- [ ] **Working example** — Code that actually runs
- [ ] **License file** — LICENSE or LICENSE.md exists
- [ ] **GitHub About** — Repo description filled in (not empty)
- [ ] **Topics** — 3-5 relevant topics added (e.g., `react`, `cli`, `ai`)

## Important (Should Have)

- [ ] **Screenshot or GIF** — Visual demo in first screen
- [ ] **"Why" section** — Explains the problem this solves
- [ ] **Features list** — Scannable, not buried in prose
- [ ] **Contributing guide** — CONTRIBUTING.md or section in README
- [ ] **Issues template** — `.github/ISSUE_TEMPLATE/`
- [ ] **Social preview** — GitHub repo settings → social preview image
- [ ] **First release tagged** — `git tag v1.0.0` + GitHub Release

## Nice to Have

- [ ] **Badges** — Version, CI, license, downloads
- [ ] **Changelog** — CHANGELOG.md or Releases page filled
- [ ] **Discussion board** — GitHub Discussions enabled
- [ ] **Funding** — `.github/FUNDING.yml`
- [ ] **Citation** — `CITATION.cff` for academic projects
- [ ] **Security policy** — `SECURITY.md`
- [ ] **Multi-language README** — At least EN + one other

## Scoring

| Items Done | Score |
|-----------|-------|
| All Critical + most Important | 🟢 Ready to launch |
| All Critical | 🟡 Ship, but add the rest soon |
| Missing Critical items | 🔴 Not ready — fix first |
