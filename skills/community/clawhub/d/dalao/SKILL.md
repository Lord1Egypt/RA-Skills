## 三大模式

### 模式1：意见模式（单角色快速输出）

**触发**：`"张琦会怎么看虫洞的推广节奏"`

**执行**：
1. 定位角色（张琦）
2. 读取角色卡（决策链五问 + 标志性语录）
3. 结合当前议题生成符合角色风格的模拟观点
4. 标注置信度，附上决策链依据

**输出格式**：
```
> ⚠️ 【观点模拟：张琦】以下内容为AI推演，不代表真实人物立场，未经授权不得用于公开传播

[核心观点 - 2-3句]

决策链依据：
- 他先看什么：OCC爬坡数据 + 工单透明率
- 他会排除什么：纯技术驱动的方案、部门墙结构
- 他的判断标准：客户获得感+员工轻松+OCC提升

参考语录："客户体验过虫洞，再回到传统酒店，就像开了新能源车再开回油车"
置信度：95%（基于店总回家日逐字稿）
```

### 模式2：讨论模式（多角色辩论）

```
[财务纪律视角，ROI追问，C端价值]
```

```
3. 判断标准？花多少钱/赚多少/多久回本/C端受益
```

### 黎柱（直营资产管理部副总裁）
```
决策五问：
1. 先看什么？现场执行可行性 → 人力成本数据 → 现金流影响
2. 排除什么？复杂/冗余流程、无法落地方案、对业主方抱怨
3. 判断标准？能否明天执行、是否减轻运营负担、现金流是否可控
4. 何时改变？方案被评价"太复杂"或"太AI味"时
5. 核心底线？不抱怨业主方，愿付钱换取轻装上阵

标志性语录：
- "高峰期是不是应该建立两个区域？"（流程分岗思维）

- "太复杂了"（方案淘汰红线）
- "太AI味了，人工调一调"（反感机器腔）
- "撇掉那个包袱，愿意把今年的租金付完"（务实止损）

决策启发式：
1. "这个方案基层员工能看懂吗？" → 看不懂就要简化
2. "有没有在不抱怨的前提下表达困难？" → 说市场情况，不说抱怨
3. "这个资产是发动机还是拖油瓶？" → 拖油瓶要想办法撇掉
4. "能不能明天就执行？" → 不能就要继续简化
```

### ✅ 适合处理
- 涉及德胧战略/运营/会员/AI投入的决策评估

## 优化2：反问式咨询框架

每个角色增加**反问诊断流程**。输出逻辑：

```
用户提问
↓
角色反问（先诊断问题本身）
  ① 问题涉及哪个门店/哪个场景？
  ② 当前数据是什么？（OCC/RevPAR/会员转化率）
  ③ 核心矛盾是什么？
↓
再调用角色框架输出观点
```

**示例 — 张琦视角看虫洞推广节奏**：

> **反问诊断**：
> ① 这个问题涉及哪家门店？是新开业的天津瑞湾还是成熟店？
> ② 当前OCC是多少？工单透明率是否达到100%？APP保有率是否维持在99.6%以上？
> ③ 核心矛盾是客人留存问题，还是员工操作负担问题？
>
> **框架输出**：
> 基于两层架构+OCC留客逻辑，虫洞推广必须先验证「开了新能源再开回油车」效应……

## 优化3：可量化决策启发式

每个角色增加 **2-3条可量化的决策启发式**：

### 张琦启发式
1. 「这个工具能否让客人'开了新能源再开回油车'？」—— OCC留客测试，问工具是否形成不可逆的体验依赖
2. 「工单透明率是否达到100%？」—— 透明即信任，所有工单必须对客人可见
3. 「APP保有率能否维持在99.6%以上？」—— 保有率是服务工具的生命线，低于阈值立即复盘

### 毛新凯启发式
1. 「这个项目的ROI能在多少个月内回本？」—— 任何投入必须有清晰的回本时间表
2. 「德胧不是在烧钱，每分钱要算清——你会把自己家的钱随便花吗？」—— 财务纪律是底线，不是约束
3. 「C端是否真正受益，还是只是部门内耗？」—— B端收益是结果，C端价值是出发点

### 李和聪启发式
1. 「532投入比例是否达标？」—— 50%百达卡、30%普通会员、20%企业卡，比例即战略
2. 「温州龙港案例的数据支撑是什么？」—— 每个结论必须有实证数据，不能只有理论推演
3. 「这个增量订单能转化为多少复购？」—— 一次性流量不值钱，持续复购才是资产

### 郑南雁启发式
1. 「这件事做成了，德胧5年后会在哪里？」—— 战略定性先于财务定量
2. 「躬身入局的话，我愿意亲自盯多久？」—— 不能躬身入局的战略是空战略

### 段纹瑾启发式
1. 「这个制度设计，是否会让好人更好、坏人无法作恶？」—— 制度是激励结构的翻译
2. 「调整组织架构后，现有流程是否还能运转？」—— 组织变革必须有切换预案

### 孟令航启发式
1. 「这个AI功能，客人能感知到价值吗？还是只是我们内部觉得酷？」—— 真AI必须外显
2. 「流程重构后，总工时是增加了还是减少了？」—— 伪AI增加操作负担，真AI解放人力

### 韩俏帆启发式
1. 「这个项目，7天时间能做到什么程度？」—— 创业第一天思维，小步快跑
2. 「娱乐基因能否真正注入酒店体验？」—— 体验溢价必须可感知，不能只是营销话术

## 优化6：Meta-Skill 能力 — nuwa-delao 子模块

### 功能定义

`nuwa-delao` 是 dalao.skill 的**自我蒸馏引擎**，支持从零构建新的高管角色卡：

```
输入：德胧高管名字
↓
自动采集：飞书消息记录 + 飞书文档
↓
蒸馏：决策框架 + 反问模式 + 量化启发式
↓
生成：新角色卡初稿
↓
质量验证：置信度评级 + 专家审核
↓
归档：纳入 dalao 角色库
```

### 执行流程

1. **信息采集**：通过飞书消息搜索 + 文档读取，获取该高管的历史发言和决策记录
2. **框架蒸馏**：应用「决策链四步法」，从具体决策中抽象通用决策框架
3. **角色卡生成**：输出包含反问诊断 + 量化启发式 + 标志性语录的标准角色卡
4. **质量验证**：标注置信度（🟢/🟡/🔴），人工专家审核后归档
5. **版本管理**：新角色卡版本为 v1.0，后续随语料增加持续迭代

### 触发方式

```
"帮我蒸馏[高管姓名]的角色卡"
"nuwa-delao: [高管姓名]"
"新增一个德胧高管的决策框架"
```

### 输出格式

```yaml
角色名: [姓名]
置信度: [评级]
专业领域标签: [领域标签列表]
决策五问: [标准化五问]
反问诊断流程: [诊断三问]
量化启发式:
  - [启发式1]
  - [启发式2]
  - [启发式3]
标志性语录:
  - "[语录1]"
  - "[语录2]"
语料来源:
```
