# 逻辑一致性检查清单

本文档提供逻辑一致性检查的系统方法，供 novel-writer skill 使用。

## 一、人物一致性检查

### 1. 外貌特征追踪

**检查项**：
- [ ] 头发颜色/长度是否变化（前文黑发 → 后文金发）
- [ ] 身高/体型是否变化
- [ ] 特殊标记（伤疤、纹身）是否前后一致
- [ ] 穿着是否连贯（前文脱了外套 → 后文又穿着外套）

**追踪方法**：
- 建立 `character_tracking.md`，记录每个人物的外貌特征
- 每次人物出场时核对

**常见错误**：
- 人物突然换了发型，但没有描写换发型的过程
- 伤疤位置前后不一致（左脸 → 右脸）

---

### 2. 性格一致性检查

**检查项**：
- [ ] 人物的选择是否符合其性格？
- [ ] 人物是否有不合理的性格转变？
- [ ] 人物的口头禅/说话方式是否一致？

**性格弧线设计**：
- 性格转变必须有**触发事件**（如：亲友死亡 → 变得冷漠）
- 转变应该是**渐进的**，不是突然的
- 转变后应该有**反复**（不是一下子就完全变了）

**常见错误**：
- 前文胆小如鼠的主角，突然变成无畏战士（没有铺垫）
- 冷静理智的角色，突然情绪化（没有触发事件）

---

### 3. 能力一致性检查

**检查项**：
- [ ] 人物的能力是否有明确限制？
- [ ] 前文不会的技能，后文是否会了（没有学习过程）？
- [ ] 人物是否太强（没有弱点）？

**能力清单管理**：
- 建立 `ability_tracking.md`，列出每个人物的能力和限制
- 每次人物使用能力时核对

**常见错误**：
- 前文不会魔法的主角，突然会了（没有学习过程）
- 反派太弱，主角太强，冲突不够激烈

---

### 4. 人物关系检查

**检查项**：
- [ ] 人物关系是否前后一致？
- [ ] 关系变化是否有合理铺垫？
- [ ] 人物是否记得之前发生的事？

**关系网追踪**：
- 建立 `relationship_tracking.md`，记录人物关系变化
- 每次关系变化时标注触发事件

**常见错误**：
- 前文敌对的两人，突然变成盟友（没有解释）
- 人物忘记之前发生的重要事件（记忆丢失）

---

## 二、时间线一致性检查

### 1. 事件发生顺序

**检查项**：
- [ ] 事件A发生在事件B之前，后文是否搞反了？
- [ ] 时间跨度是否合理？（前一天还在北京，后一天就在纽约，没有解释）
- [ ] "三天后"、"一个月后"等时间跳跃是否准确？

**时间线追踪**：
- 建立 `timeline_tracking.md`，记录每个事件的发生时间
- 用时间轴图示检查顺序

**常见错误**：
- 时间线跳跃不合理（没有交通工具解释）
- "三天后"与实际的章节时间间隔不符

---

### 2. 时间跨度合理性

**检查项**：
- [ ] 人物是否在不合理的时间内完成了某事？
- [ ] 年龄是否前后一致？（前文10岁 → 后文突然18岁，时间跨度不够）
- [ ] 季节/天气是否一致？（前文下雪 → 后文穿短袖）

**时间跨度计算**：
- 计算事件发生的总时间
- 核对人物的年龄变化

**常见错误**：
- 主角用一天时间从新手变成大师（不合理）
- 前文冬天，后文夏天，但没有交代季节变化

---

## 三、空间一致性检查

### 1. 物品位置追踪

**检查项**：
- [ ] 物品是否凭空出现/消失？
- [ ] 物品位置是否前后一致？（前文钥匙在桌上 → 后文钥匙在口袋，没有解释）
- [ ] 人物是否记得物品放在哪里？

**物品追踪**：
- 建立 `item_tracking.md`，记录重要物品的位置
- 每次物品移动时标注

**常见错误**：
- 前文放在桌上的枪，后面突然在主角手里（没有描写拿起的过程）
- 人物忘记重要物品的位置

---

### 2. 场景描述一致性

**检查项**：
- [ ] 房间的布局是否前后一致？（前文门在左边 → 后文门在右边）
- [ ] 环境描写是否矛盾？（前文描写城市繁华 → 后文描写荒凉）
- [ ] 距离/时间是否合理？（前文说两地相距100公里 → 后文主角1小时就到了）

**场景追踪**：
- 建立 `setting_tracking.md`，记录每个场景的布局
- 每次场景切换时核对

**常见错误**：
- 场景布局前后不一致
- 距离/时间不合理

---

## 四、因果一致性检查

### 1. 伏笔收回检查

**检查项**：
- [ ] 前文埋的伏笔是否都收回了？
- [ ] 后文的"惊喜"是否前文有伏笔？
- [ ] 是否有"机械降神"（突然出现的解决方法，没有铺垫）？

**伏笔追踪**：
- 建立 `foreshadowing_tracking.md`，记录每个伏笔和收回位置
- 定期检查未收回的伏笔

**常见错误**：
- 伏笔没有收回（读者会失望）
- "机械降神"（突然出现的解决方法，破坏逻辑）

---

### 2. 因果关系检查

**检查项**：
- [ ] 每个事件是否有合理的原因？
- [ ] 人物的选择是否有动机？
- [ ] 反派的行为是否有合理动机？

**因果链检查**：
- 画出因果链：事件A → 事件B → 事件C
- 检查每个环节是否合理

**常见错误**：
- 事件突然发生，没有原因（"机械降神"）
- 反派的行为没有合理动机（为了坏而坏）

---

## 五、逻辑检查工作流程

### 1. 建立追踪表

为长篇小说的逻辑一致性，建立以下追踪表：

- `character_tracking.md`：人物特征追踪
- `timeline_tracking.md`：时间线追踪
- `item_tracking.md`：物品位置追踪
- `relationship_tracking.md`：人物关系追踪
- `foreshadowing_tracking.md`：伏笔收回追踪

### 2. 分阶段检查

**初稿完成后**：
- 通读全文，建立追踪表

**修订阶段**：
- 对照追踪表，检查一致性

**定稿前**：
- 最后检查一遍追踪表，确保没有遗漏

### 3. 使用工具辅助

**手动检查**：
- 用不同颜色标注不同类型的不一致（红色=严重，黄色=轻微）

**自动检查**（如果有编程能力）：
- 用 Python 脚本搜索关键词（如人物名字、物品名字）
- 提取所有相关段落，检查一致性

## 六、逻辑漏洞修复方法

### 1. 添加解释

如果人物突然获得新能力，添加学习过程：
- 前文：主角去找师父学艺
- 后文：主角使用新技能

### 2. 修改前文

如果后文已经写好，无法添加解释，就修改前文：
- 前文：暗示主角可能有这个能力（伏笔）
- 后文：主角使用能力

### 3. 删除矛盾

如果矛盾无法修复，考虑删除矛盾的部分：
- 前文：删除矛盾的描述
- 后文：删除矛盾的事件

### 4. 接受不完美

有些小矛盾（如物品位置变化）可能不影响阅读，可以选择接受：
- 标注为"可解释差异"
- 在修订时考虑是否修复

## 七、逻辑检查清单总结

### 人物一致性
- [ ] 外貌特征是否一致？
- [ ] 性格是否一致（或有合理转变）？
- [ ] 能力是否有明确限制？
- [ ] 人物关系是否一致？

### 时间线一致性
- [ ] 事件发生顺序是否合理？
- [ ] 时间跨度是否合理？
- [ ] 年龄/季节/天气是否一致？

### 空间一致性
- [ ] 物品位置是否一致？
- [ ] 场景描述是否一致？
- [ ] 距离/时间是否合理？

### 因果一致性
- [ ] 伏笔是否都收回了？
- [ ] 因果关系是否合理？
- [ ] 是否有"机械降神"？
