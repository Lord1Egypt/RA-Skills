# 数据抓取最佳实践

## 合规数据获取原则

### 核心原则：尊重站点规则，合法获取数据

| 原则 | 说明 |
|------|------|
| 遵守 robots.txt | 先检查目标站点的 robots.txt 协议 |
| 控制请求频率 | 合理设置请求间隔，避免对服务器造成压力 |
| 使用公开API | 优先使用官方提供的 API 接口 |
| 认证授权 | 仅处理有合法授权的数据 |

## 基础数据获取配置

### L1: 基础配置（适用大多数场景）
- 设置合理的 User-Agent
- 添加 Referer 头部模拟正常浏览器访问
- 请求间隔 2-5 秒，避免高频请求

### L2: 动态内容处理（适用 JavaScript 渲染页面）
- 使用 Playwright/Selenium 进行页面渲染
- 等待页面完全加载后再提取数据
- 添加适当的页面等待时间

### L3: API 接口调用（适用官方 API 场景）
- 使用官方 API 获取结构化数据
- 遵守 API 调用频率限制
- 使用有效的 API Key 进行认证

## 数据质量检查清单

- [ ] 去重（基于唯一ID）
- [ ] 空值处理
- [ ] HTML标签清除
- [ ] 编码统一(UTF-8)
- [ ] 数字格式标准化("1.2万"→12000)
- [ ] 时区统一
- [ ] 字段完整性校验

## 常见问题处理

| 问题 | 建议方案 |
|------|----------|
| 连接失败 | 检查URL有效性，添加重试机制 |
| 超时错误 | 增加超时时间，等待后重试 |
| 选择器失效 | 检查页面结构，更新选择器 |
| 编码问题 | 指定正确编码，使用容错解析 |
