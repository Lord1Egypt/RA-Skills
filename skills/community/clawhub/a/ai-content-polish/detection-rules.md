# 中文 AI 内容去痕 — 检测规则库

> 版本: v1.0.0 | 更新: 2026-05-20

## 规则类别一：空洞过渡语

AI 最常见的特征是用大量无意义的过渡词连接段落。

### 高频标记词（每出现一次扣 3 分）

| 标记词/短语 | 说明 |
|------------|------|
| 值得注意的是 | 最经典的 AI 过渡，人类几乎不用 |
| 需要指出的是 | 同上 |
| 总而言之 | 过于正式的总结，除非写论文 |
| 综上所述 | 同上 |
| 首先...其次...最后 | 机械三段式过渡 |
| 一方面...另一方面 | AI 的辩证思维模板 |
| 与此同时 | AI 最爱的连接词之一 |
| 不仅如此 | 排比推进标志 |
| 更重要的是 | AI 认为什么都重要 |
| 事实上 | 空洞强调 |
| 毋庸置疑 | 过度确定 |
| 众所周知 | 套话开头 |
| 毫无疑问 | 同上 |
| 毋庸讳言 | 文言套话 |
| 某种程度上 | AI 的模糊表达 |
| 在某种意义上 | 同上 |
| 归根结底 | 总结套话 |
| 换句话说 | 过度解释标志 |

### 检测方法

- 精确匹配词表
- 正则：`(值得注意的是|需要指出的是|总而言之|综上所述|与此同时|不仅如此|更重要的是|事实上|毋庸置疑|众所周知|毫无疑问|归根结底|换句话说)`

## 规则类别二：排比与三段式结构

AI 特别喜欢对称的排比结构，人类写作很少如此工整。

### 检测模式

| 模式 | 示例 | 说明 |
|------|------|------|
| 不仅A，而且B，更是C | "不仅功能强大，而且体验出色，更是行业标杆" | 三段递进排比 |
| A化、B化、C化 | "智能化、数字化、自动化" | 后缀排比 |
| 既A又B还C | "既高效又便捷还安全" | 三重并列 |
| 一方面A，另一方面B | 机械辩证 | 二分法模板 |
| A是...B是...C是... | 连续定义式排比 | 定义排比 |
| 可以A，可以B，也可以C | 三重可能性排比 | 选择排比 |

### 检测方法

- 查找"不仅...而且"、"既...又...还"模式
- 检查连续 3 个以上结构相同的短语
- 检查段落是否严格遵循"论点-论据-总结"三段式

## 规则类别三：形容词堆砌

AI 倾向于用多个空洞形容词修饰名词。

### 高频堆砌词

| 形容词 | 说明 |
|--------|------|
| 强大的 | AI 万能形容词 #1 |
| 全面的 | AI 万能形容词 #2 |
| 高效的 | AI 万能形容词 #3 |
| 优秀的 | 无信息量的评价 |
| 出色的 | 同上 |
| 丰富的 | 描述任何数量都用 |
| 先进的 | 技术类万能词 |
| 专业的 | 同上 |
| 优质的 | 模糊褒义 |
- **强大的** — AI 万能形容词 #1
- **全面的** — AI 万能形容词 #2
- **高效的** — AI 万能形容词 #3
- **优秀的 / 出色的** — 无信息量的评价
- **丰富的** — 描述任何数量都用
- **先进的** — 技术类万能词
- **专业的** — 同上
- **优质的** — 模糊褒义

### 检测规则

1. 同一名词前堆叠 2+ 个上述形容词 → 扣分
2. 连续两句使用上述形容词 → 扣分
3. "强大的/全面的/高效的"修饰抽象名词（功能/能力/解决方案）→ 高概率 AI

## 规则类别四：机械列举

AI 喜欢用编号列举代替自然叙述。

### 检测模式

- 段落中连续出现 4+ 个并列项（编号或顿号分隔）
- 每个列表项结构完全相同（同句式、同长度）
- 列表后紧跟"以上几点..."式总结

### 不是 AI 列举的信号

- 列表项长短不一
- 列表项有关键细节差异
- 只列 2-3 项且有主次之分
- 用文字自然串联而非机械编号

## 规则类别五：过度总结

AI 几乎每个段落结尾都要来一句总结。

### 高频总结模式

- "总的来说，..."
- "由此可见，..."
- "这意味着，..."
- "因此，我们可以看出..."
- "这充分说明了..."
- 段落最后一句是前文内容的同义复述

### 检测规则

1. 每个段落最后一句是总结句 → 扣分
2. 总结句只是前文内容的同义复述 → 加倍扣分
3. 文章末尾有"总结段落"且与前文重复 → 扣分

## 规则类别六：口癖重复

AI 在同一篇文章中反复使用某些词语或句式。

### 检测方法

1. 统计全文每个词的出现频率
2. 出现频率 TOP 20 的词中，以下词出现 3+ 次 → 扣分：
   - "同时"、"此外"、"另外"、"当然"、"实际上"、"确实"
3. 相同句式出现 3+ 次（如连续 3 句以"通过...，..."开头）→ 扣分
4. 同一段落中"的"字出现超过 15 次 → 句式单一，可能 AI

## 规则类别七：缺乏个人声音

人类写作有个人色彩，AI 写作是"上帝视角"的客观叙述。

### 检测信号

- ❌ 全文没有第一人称（"我"、"我觉得"、"在我看来"）
- ❌ 没有任何主观评价（"我觉得不错"、"说实话"、"坦白讲"）
- ❌ 没有任何口语化表达
- ❌ 没有具体的生活场景或个人经历
- ❌ 全文没有疑问句
- ❌ 没有任何情感表达（惊讶、失望、兴奋）
- ❌ 全是陈述句，没有感叹、反问、设问

### 人类写作的信号

- ✅ 有个人代词
- ✅ 有口语词汇
- ✅ 有具体场景或例子
- ✅ 有情感词
- ✅ 有疑问句
- ✅ 句子长短不一
- ✅ 有停顿、省略、破折号等口语节奏

## 规则类别八：过度工整

AI 的文本在结构上过于完美。

### 检测模式

| 模式 | 说明 |
|------|------|
| 每段 3-4 句 | 段落长度过于均匀 |
| 句式高度统一 | 主谓宾结构占 80%+ |
| 段首都是主题句 | 典型 AI 段落结构 |
| 结尾都是总结句 | 同上 |
| 全文无插入语 | 人类说话会插入补充说明 |
| 无跳跃/转折 | 人类思维会有突兀的转折 |
| 标点符号过于规范 | 人类会用省略号、破折号、感叹号 |

### 检测方法

1. 计算各段落句数，标准差 < 1.0 → 过于工整
2. 段首句全是"XX是..."或"XX对于..." → 模式化开头
3. 无任何标点"错误"或非标准用法 → 过于规范
