# LEARNINGS — 纠正与洞察

记录用户纠正、新发现的洞察、值得固化的最佳实践。执行模式追加写入，反思模式聚类提升。

- **写入时机**：用户说"不对 / 不是这样 / 应该是…"，或对话中产生值得记住的新认知
- **写入格式**：严格按下方模板
- **area 字段**：必须复用已有类别——`路由规则` / `指数数据` / `大师框架` / `表达风格` / `用户画像`

---

## 条目模板

```
## [YYYY-MM-DD] <一句话标题（动宾结构）>
- **area**: <路由规则 / 指数数据 / 大师框架 / 表达风格 / 用户画像>
- **priority**: <high | medium | low>
- **status**: new
- **recurrence-count**: 1
- **cross-refs**: <相关条目锚点，可留空>

**触发情境**：<当时在聊什么、用户说了什么>
**原有做法**：<之前是怎么处理的>
**纠正/洞察**：<应该怎么做——一句可执行的规则>
**为什么**：<理由——反思时判断是否提升靠它>
```
