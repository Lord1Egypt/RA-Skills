# 项目输入卡（Project Intake）

> **用途**：每个任务/项目开始前，由 Boss 填写此文件，放入任务目录。Moss 读取后直接掌握完整上下文，跳过反复确认阶段，直接进入方案生成。
>
> **文件定位**：本文档为**不可变的启动卡**。任务进行中的动态追踪（Open Questions状态更新、进度记录）写入 `t-log.md`，不修改本文件。
>
> **位置**：`/home/huang/.hermes/mission_control/{YYYY-MM-DD}/T-YYYYMMDD-NNN/project-intake.md`

> - **临时任务**：单一可交付物，不需要持续追踪

收到此文件后，请勿直接开始生成方案。请先严格按照以下步骤执行：

3. **复述绝对红线**（直接引用 `[RED_LINE_N]` 标签）

**示例回复格式**：

```
✅ 自检完成：

- 核心交付物：生成 T1转录组分析报告.pdf

请Boss确认后进入方案生成阶段。
```

> **后续纠偏机制**：任务进行中，如对话过长导致 Moss 理解偏差，Boss 可重新发送本文件并说明"上下文已更新，请重新对齐目标"。
