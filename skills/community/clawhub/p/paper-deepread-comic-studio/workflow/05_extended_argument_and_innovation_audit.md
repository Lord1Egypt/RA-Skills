# Step 5: Extended Argument And Innovation Audit

After the baseline deep-read is complete, run an explicit second-pass audit focused on the following:

- title interpretation
- research equation audit: important setting, broken assumption, borrowed tool, new constraint, surrogate mechanism
- unavailable-mechanism audit: what ideal mechanism `Y` is absent, and what replacement `Z` the paper builds
- author-side discovery audit: valuable field, painful assumption, borrowed or emerging tool, unserved setting, blocking constraint, conceptual replacement
- story-construction audit: challenge -> failure mode -> design principle -> module -> evidence
- reverse citation audit: citation cluster -> narrative function -> inherited assumption -> paper modification
- experiments-as-story-evidence audit: claim -> counterfactual -> metric -> stress condition
- reusable story-making pattern audit: replacement story, two-axis empty cell, closed-loop contribution, hidden-assumption-to-new-idea conversion
- author subjectivity audit: where idea, proof strategy, module design, or engineering choices look author-specific rather than objectively forced
- paper-grounded specificity audit: whether the report stays close to the actual formulas, modules, datasets, baselines, figures, and findings instead of abstract generic phrasing
- frequently discussed prior papers and their remaining blanks
- claim-by-claim evidence support
- extra reviewer-lens audit inspired by strong GitHub paper-review skills: novelty, significance, soundness, methodology rigor, reproducibility, results-claims alignment, missing baselines/controls, figure/table clarity, limitation honesty
- baseline genealogy from related work to experiments
- innovation type: incremental, cross-domain, boundary-breaking, or hybrid
- limitations, unresolved questions, failure modes, and structural ceilings
- discipline-boundary crossing and its limits
- fresh follow-up directions and migration opportunities
- which directions may genuinely push a scientific frontier rather than only polish the current setting
- final plain-language story summary

When experiments contain tables, curves, or visual evidence, explain which claims they are meant to support and call out partial support or contradiction explicitly. If a LaTeX source exists, include figure-caption-aware analysis rather than relying only on rendered PDFs.

Prefer explicit judgments over vague praise. When evidence is missing, say so directly.
