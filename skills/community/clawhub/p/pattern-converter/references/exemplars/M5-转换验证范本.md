# M5 转换验证域范本

## M5-01 转换评估

### 输入示例
转换结果：传统出租车模式→共享单车模式

### 处理过程
1. 评估标准：转换效果、模式适用性、创新程度
2. 评估方法：多维度评估
3. 评估结果：转换评估结果
4. 评估结论：评估结论

### 输出示例
```markdown
## 转换评估报告

### 评估标准
| 标准 | 描述 | 权重 |
|------|------|------|
| 转换效果 | 转换是否成功 | 40% |
| 模式适用性 | 模式是否适用 | 30% |
| 创新程度 | 是否有创新 | 30% |

### 评估结果
| 标准 | 评分 | 说明 |
|------|------|------|
| 转换效果 | 高 | 成功将出租车模式转换为共享单车模式 |
| 模式适用性 | 高 | 匹配模式、计价模式等适用于共享单车 |
| 创新程度 | 高 | 模式创新，应用创新 |

### 评估结论
- **综合评分**：高
- **结论**：转换成功，模式适用，有创新
```

## M5-04 改进建议

### 输入示例
风险：技术风险、市场风险、运营风险

1. 问题识别：识别风险对应的问题

3. 建议优先级：建议的优先级排序
4. 建议实施计划：建议的实施计划
