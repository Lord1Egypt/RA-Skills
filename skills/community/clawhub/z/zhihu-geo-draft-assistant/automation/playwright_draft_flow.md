# Playwright 草稿辅助流程说明

本 Skill 中的 `scripts/draft_to_zhihu.example.py` 采用了非常保守的浏览器辅助策略。

## 流程概览

1. **环境准备**
   - 依赖项：`playwright`。
   - 用户在本地终端启动脚本。

2. **拉起浏览器 (Visible Mode)**
   - 脚本必须使用 `headless=False` 启动一个真实的 Chromium 浏览器界面。
   - 目的：让用户全程看见机器人的操作，并能够随时接管。

3. **用户手动鉴权**
   - 脚本导航至 `zhihu.com`。
   - 脚本在此挂起等待。
   - **用户必须亲手扫码或输入密码登录。**（脚本绝不可介入登录过程或保存登录态）。
   - 用户登录成功后，在终端敲击回车，允许脚本继续。

4. **导航至创作页**
   - 根据用户的选择（专栏文章 vs 问答），脚本导航至文章创作页 (`/write`) 或用户提供的问题链接。

5. **模拟内容填入**
   - 脚本从 `output/zhihu/` 读取指定的生成文件。
   - 使用前端插入文本（`document.execCommand`）的方式将 Markdown 文本填入编辑器中。
   - 考虑到知乎富文本编辑器的复杂性，这里通常只做基础文字的填充，复杂的格式（加粗、表格）可能在粘贴时丢失，需人工调整。

6. **人工接管与闭环**
   - 文本填入后，脚本停止一切自动化动作，并保持浏览器开启。
   - **人类介入**：用户检查内容，手动添加话题标签，调整排版。
   - 用户自行点击“保存草稿”或“发布”。
   - 用户关闭浏览器，流程结束。
