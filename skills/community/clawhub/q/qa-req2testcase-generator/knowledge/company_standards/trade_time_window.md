# 交易类用例时间窗口规范

> 文档编号：L3-TRD-001  
> 版本：v1.0  
> 生效日期：2026-04-18  
> 适用范围：所有涉及交易业务的测试用例

## 1. 目的

规范交易类测试用例的时间窗口标注，确保测试用例在正确的交易时段执行，覆盖跨时段和非交易时段的业务场景。

## 2. 交易时段定义

### 2.1 A股交易时段

| 时段编号 | 时段名称 | 时间范围 | 业务特征 |
|----------|----------|----------|----------|
| T1 | 集合竞价（开盘） | 09:15 - 09:25 | 可申报/撤单（09:20前可撤单，09:20-09:25不可撤单） |
| T2 | 集合竞价撮合 | 09:25 - 09:30 | 不接受申报和撤单，系统撮合确定开盘价 |
| T3 | 连续竞价（上午） | 09:30 - 11:30 | 正常交易，实时撮合 |
| T4 | 午间休市 | 11:30 - 13:00 | 不接受申报（部分券商可接受预埋单） |
| T5 | 连续竞价（下午） | 13:00 - 14:57 | 正常交易，实时撮合 |
| T6 | 收盘集合竞价 | 14:57 - 15:00 | 可申报不可撤单，确定收盘价 |
| T7 | 盘后固定价格交易 | 15:05 - 15:30 | 以收盘价成交（仅科创板/创业板） |
| T8 | 非交易时段 | 15:30 - 次日09:15 | 系统接受预埋委托（次日有效） |

### 2.2 港股通交易时段

| 时段编号 | 时段名称 | 时间范围 | 业务特征 |
|----------|----------|----------|----------|
| HK1 | 开市前时段 | 09:00 - 09:30 | 竞价盘/竞价限价盘 |
| HK2 | 持续交易时段（上午） | 09:30 - 12:00 | 增强限价盘/特别限价盘 |
| HK3 | 午间休市 | 12:00 - 13:00 | 不接受申报 |
| HK4 | 持续交易时段（下午） | 13:00 - 16:00 | 增强限价盘/特别限价盘 |
| HK5 | 收市竞价时段 | 16:00 - 16:10 | 竞价盘/竞价限价盘 |

### 2.3 夜市委托时段

| 时段编号 | 时段名称 | 时间范围 | 业务特征 |
|----------|----------|----------|----------|
| N1 | 夜市委托 | 当日18:00 - 次日09:15 | 接受次日委托，不实时撮合 |

### 2.4 特殊时段

| 时段编号 | 时段名称 | 触发条件 | 业务特征 |
|----------|----------|----------|----------|
| S1 | 临时停牌 | 交易所公告 | 停牌期间不接受该证券委托 |
| S2 | 熔断 | 触发熔断阈值 | 暂停交易或限制价格波动 |
| S3 | 节假日休市 | 法定节假日 | 全天不交易，夜市委托顺延 |

## 3. 用例时段标注规范

### 3.1 标注要求

每条交易类测试用例必须在用例属性中标注以下信息：

| 属性 | 必填 | 说明 |
|------|------|------|
| 适用时段 | 是 | 使用时段编号标注，如 T3/T5 |
| 时段依赖 | 否 | 是否必须在特定时段执行（是/否） |
| 跨时段标记 | 否 | 是否涉及跨时段场景（是/否） |
| Mock时间 | 否 | 非实盘测试时的模拟时间点 |

### 3.2 标注示例

```
用例编号：TC-TRADE-001
用例标题：连续竞价时段限价委托买入
适用时段：T3/T5
时段依赖：是（必须在连续竞价时段执行）
跨时段标记：否
Mock时间：2026-04-18 10:00:00
```

## 4. 跨时段场景用例要求

### 4.1 必须覆盖的跨时段场景

| 序号 | 跨时段场景 | 用例要求 |
|------|------------|----------|
| 1 | 集合竞价 → 连续竞价 | 集合竞价未成交委托在连续竞价时段的处理 |
| 2 | 上午连续竞价 → 午间休市 | 11:30未成交委托的状态变化 |
| 3 | 午间休市 → 下午连续竞价 | 午间预埋单在13:00的自动申报 |
| 4 | 连续竞价 → 收盘集合竞价 | 14:57时刻的委托类型切换 |
| 5 | 收盘集合竞价 → 盘后交易 | 收盘价确定后盘后交易的启动 |
| 6 | 交易时段 → 非交易时段 | 15:00后委托的拒绝/预埋处理 |
| 7 | 夜市委托 → 次日集合竞价 | 夜市委托在次日09:15的申报 |

### 4.2 跨时段用例编写规则

1. 每个跨时段场景必须作为独立用例，不与单时段用例合并
2. 用例步骤中必须明确标注时间切换点
3. 预期结果必须包含时段切换前后的状态对比
4. 涉及时间Mock的用例须说明Mock方式和恢复步骤

## 5. 非交易时段委托处理规则

### 5.1 必须覆盖的测试场景

| 序号 | 场景 | 预期行为 | 测试要点 |
|------|------|----------|----------|
| 1 | 非交易时段提交普通委托 | 系统接受并标记为预埋单 | 委托状态为"预埋"而非"已报" |
| 2 | 非交易时段撤销预埋单 | 允许撤销 | 撤销后委托状态变更正确 |
| 3 | 节假日提交委托 | 系统接受预埋，顺延至下一交易日 | 委托有效期标注正确 |
| 4 | 非交易时段查询持仓 | 显示T日收盘后持仓 | 数据为最新清算后数据 |
| 5 | 非交易时段银证转账 | 根据银行通道状态决定 | 转账时间限制提示正确 |
| 6 | 非交易时段修改委托 | 允许修改预埋单 | 修改后参数正确更新 |
| 7 | 清算期间提交委托 | 系统拒绝或排队 | 提示信息明确告知清算中 |

## 6. 各时段测试点模板

### 6.1 集合竞价时段（T1）测试点模板

| 测试点 | 操作 | 预期结果 | 断言条件 |
|--------|------|----------|----------|
| 正常申报 | 09:15提交限价委托 | 委托状态为"已报" | `order.status == 'REPORTED'` |
| 09:20前撤单 | 09:18提交撤单 | 撤单成功，委托状态为"已撤" | `order.status == 'CANCELLED' && cancel.time < '09:20:00'` |
| 09:20后撤单 | 09:21提交撤单 | 撤单被拒绝 | `cancel.result == 'REJECTED' && cancel.reason == 'NOT_ALLOWED_AFTER_0920'` |
| 市价委托 | 09:16提交市价委托 | 委托被拒绝（集合竞价不接受市价单） | `order.status == 'REJECTED' && order.rejectReason CONTAINS '集合竞价'` |
| 涨跌停价委托 | 09:15提交涨停价买入 | 委托正常接受 | `order.status == 'REPORTED' && order.price == stock.upperLimit` |

### 6.2 连续竞价时段（T3/T5）测试点模板

| 测试点 | 操作 | 预期结果 | 断言条件 |
|--------|------|----------|----------|
| 限价买入 | 提交限价买入委托 | 委托成功，等待撮合 | `order.status IN ('REPORTED', 'PARTIAL_FILLED', 'FILLED')` |
| 市价买入 | 提交市价买入委托 | 立即撮合或部分成交 | `order.filledQty > 0 \|\| order.status == 'REPORTED'` |
| 撤单 | 对未成交委托撤单 | 撤单成功 | `order.status == 'CANCELLED' && order.cancelledQty == order.orderQty - order.filledQty` |
| 部分成交后撤单 | 对部分成交委托撤单 | 剩余数量撤销 | `order.cancelledQty == order.orderQty - order.filledQty` |
| 涨跌停限制 | 提交超过涨停价的委托 | 委托被拒绝 | `order.status == 'REJECTED' && order.rejectReason CONTAINS '超过涨跌幅'` |

### 6.3 收盘集合竞价时段（T6）测试点模板

| 测试点 | 操作 | 预期结果 | 断言条件 |
|--------|------|----------|----------|
| 正常申报 | 14:57提交限价委托 | 委托接受 | `order.status == 'REPORTED'` |
| 撤单拒绝 | 14:58提交撤单 | 撤单被拒绝 | `cancel.result == 'REJECTED' && cancel.reason == 'CLOSING_AUCTION_NO_CANCEL'` |
| 收盘价成交 | 15:00撮合完成 | 以收盘价成交 | `trade.price == stock.closingPrice` |

### 6.4 非交易时段（T8）测试点模板

| 测试点 | 操作 | 预期结果 | 断言条件 |
|--------|------|----------|----------|
| 预埋委托 | 16:00提交委托 | 委托状态为"预埋" | `order.status == 'PRE_ORDER' && order.effectiveDate == nextTradingDay` |
| 预埋撤销 | 撤销预埋委托 | 撤销成功 | `order.status == 'CANCELLED'` |
| 预埋修改 | 修改预埋委托价格 | 修改成功 | `order.price == newPrice && order.status == 'PRE_ORDER'` |
| 次日生效 | 次日09:15系统自动申报 | 预埋单转为已报 | `order.status == 'REPORTED' && order.reportTime >= '09:15:00'` |

## 7. 时间Mock规范

### 7.1 Mock方式

1. 优先使用系统级时间Mock（如Java的 `Clock` 注入）
2. 禁止修改操作系统时间
3. Mock时间须在用例执行完毕后恢复

### 7.2 Mock实现示例

```java
// Spring Boot 时间Mock示例
@Configuration
public class ClockConfig {
    @Bean
    @Profile("test")
    public Clock testClock() {
        // Mock到连续竞价时段
        return Clock.fixed(
            LocalDateTime.of(2026, 4, 18, 10, 0, 0)
                .atZone(ZoneId.of("Asia/Shanghai")).toInstant(),
            ZoneId.of("Asia/Shanghai")
        );
    }
    
    @Bean
    @Profile("!test")
    public Clock systemClock() {
        return Clock.system(ZoneId.of("Asia/Shanghai"));
    }
}
```

## 8. 附则

- 所有交易类用例必须在评审时检查时段标注的正确性。
- 新增交易品种（如期权、期货）的时段定义由业务部门提供，测试团队据此更新本规范。
- 涉及交易所规则变更时，本规范须在规则生效前完成更新。
