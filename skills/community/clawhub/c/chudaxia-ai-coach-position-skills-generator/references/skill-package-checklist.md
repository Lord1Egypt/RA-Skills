# 岗位技能包质量检查清单

## 一、输入解析检查

- [ ] 已读取完整岗位智能体提示词，而不是只读取技能层片段。
- [ ] 已提取岗位名称、岗位职责、技能层、知识层、流程层、权限层和绩效层。
- [ ] 已建立岗位技能登记表。
- [ ] 技能数量与原提示词技能层核心能力数量一致；如不一致，已有用户确认记录。

## 二、知识萃取检查

- [ ] 每个技能都有独立的知识检索词。
- [ ] 每个技能都已检索腾讯 IMA 或用户指定的本地目录。
- [ ] 每个技能都有 `assets/knowledge/knowledge.md`。
- [ ] 每个技能都有 `references/source-map.md`，记录检索词、来源文档、采用/排除理由和可追溯标识。
- [ ] 所有关键规则、流程、模板和案例都有来源索引。
- [ ] `assets/knowledge/knowledge.md` 已内化技能运行所需知识，脱离原始 IMA 知识库、本地原始资料或上游提示词后仍可使用。
- [ ] `references/source-map.md` 仅作为追溯审计文件，不作为技能执行的必要知识入口。
- [ ] 证据不足的技能已创建 `assets/knowledge/knowledge-gaps.md`，并在 `assets/knowledge/knowledge.md` 对应章节标注 `[待补齐]`，没有虚构内容。

## 三、Agent Skills 规范检查

- [ ] 每个技能目录包含 `SKILL.md`。
- [ ] `SKILL.md` frontmatter 使用 YAML，且包含 `name` 与 `description`。
- [ ] `name` 与父目录名完全一致。
- [ ] `name` 只包含小写字母、数字和连字符，不以连字符开头或结尾，不包含连续连字符。
- [ ] `name` 不超过 64 个字符。
- [ ] `description` 非空且不超过 1024 字符，包含岗位、能力和触发关键词。
- [ ] 正文包含适用场景、输入、流程、输出、降级策略、合规边界和自检要求。
- [ ] 正文使用相对路径引用 `assets/knowledge/knowledge.md`。
- [ ] 如存在知识缺口，正文使用相对路径引用 `assets/knowledge/knowledge-gaps.md`。

## 四、提示词回写检查

- [ ] 已更新 `3. 技能层（Capability）`，每个岗位核心能力都标明生成技能目录。
- [ ] 已更新 `4. 知识层（Knowledge）`，列出生成的技能知识文档。
- [ ] 已增加或更新「岗位技能包映射表」。
- [ ] 已增加或更新「技能包同步记录」，包含生成时间、生成器名称、知识来源类型和更新摘要。
- [ ] 映射表中每个原岗位技能都只有一个对应技能目录。
- [ ] 映射表中每个生成技能目录都能追溯回一个原岗位技能。
- [ ] 如果存在 `assets/knowledge/knowledge-gaps.md`，映射表回写状态已标为「知识待补齐」。
- [ ] 如原提示词已有人工补充或旧映射表，已保留原内容或获得用户确认后覆盖。
- [ ] 回写没有破坏七层提示词结构。
- [ ] 如存在结构化 JSON 表示，已按 `chudaxia-ai-coach/assets/schemas/prompt-framework-schema.json` 校验。

## 五、交付检查

- [ ] 最终答复列出生成的文件和目录。
- [ ] 最终答复列出岗位技能包映射表。
- [ ] 最终答复列出知识缺口和待补资料。
- [ ] 最终答复说明已执行的校验，以及未能执行的校验原因。
