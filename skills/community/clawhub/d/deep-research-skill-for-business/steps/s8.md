# S8: 交汇洞察

## 目标

基于 s7 的所有论述，提炼出**跨子问题的新判断**。这些判断不能是前面章节的缩写或汇总，必须是"只有在看完全部论述后才能得出的新洞察"。

## ⛔ 红线

- **不可缩写**：本章内容不能是前面章节的概括
- **必须独立**：本章的每个判断都必须是在 s7 各子问题论述基础上产生的**新观点**
- **至少 2 条**：交汇洞察不少于 2 条

## 操作

### 1. 寻找交叉点

问自己以下问题来发现交汇洞察：
- 子问题 A 的结论和子问题 B 的结论放在一起看，有没有矛盾？有没有互相强化？
- 有没有一个现象/趋势贯穿了多个子问题？
- 有没有"看起来不相关，但实际高度联动"的发现？
- 不同来源的视角放在一起，有没有揭示新的张力？

### 2. 输出交汇洞察

```markdown
## S8: 交汇洞察

### 洞察 1: [一句话标题]
【分析】跨子问题的交叉分析...
【判断】新得出的独立判断...

### 洞察 2: [一句话标题]
...
```

### 3. 自检

- [ ] 每一条洞察，能否在"只读本章"的情况下被理解？（不能的话，说明它依赖前文太多，不是独立洞察）
- [ ] 每一条洞察，是否在 s7 中不曾明确提出？（如果是 s7 已有结论的换说法，说明是缩写，不是新洞察）

### 4. 标记完成

完成后执行：
```bash
bash orchestrator.sh --complete s8 --next s9
```

## 校验标准

- [ ] 至少 2 条交汇洞察
- [ ] 每条洞察是独立的新判断（不是 s7 的概括）
- [ ] 洞察之间有逻辑关联
