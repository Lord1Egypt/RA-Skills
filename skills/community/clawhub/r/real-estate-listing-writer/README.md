# Real Estate Listing Writer

Turn raw property details into a complete marketing package in one pass — no blank-page paralysis, no drip-feed questions.

## What it does

Paste your property specs and get four ready-to-use deliverables:

- **MLS description** — 150–220 words, Fair Housing compliant, strong hook, clear CTA
- **Instagram caption** — ~100 words, 2–3 emojis, 8 hashtags
- **Facebook post** — ~150 words, narrative agent voice, designed to drive comments/DMs
- **Email blast** — subject line + preheader + ~120-word body with `[SHOWING_LINK]` placeholder

Tone is adjustable: warm-professional (default), luxury, investor-focused, first-time-buyer-friendly, or fast-sale urgency. Tone affects sentence rhythm and voice — not just adjective choice.

## What you provide

**Required:** address, beds/baths, square footage, property type, key features, agent name and brokerage

**Optional:** asking price, year built, preferred tone, CTA contact info

Missing required fields? The skill asks for everything in one grouped message.

## Sample output

**Input:** 3/2 Austin single-family, 1,650 sqft, 2022 kitchen remodel (quartz + Bosch), refinished hardwood floors, mature oaks, detached insulated office/gym, $585k, warm-professional.

**MLS excerpt:** *"Tucked on a quiet lot shaded by mature oaks, this single-family home has been updated where it counts. The fully remodeled kitchen — completed in 2022 — features quartz countertops and Bosch appliances, anchored by refinished original hardwood floors that run throughout the main living areas..."*

## Requirements

No API keys. No external integrations. Runs entirely within your OpenClaw session using your own model.

## Compliance

All output is Fair Housing compliant. The skill will never include language referencing protected classes and will reframe any feature description that implies resident characteristics rather than property attributes.
