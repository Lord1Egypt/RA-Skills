## Description: <br>
在任何创造性工作之前必须使用 - 创建功能、构建组件、添加新功能或修改行为。通过协作对话探索用户意图、需求和设计。 <br>

This skill is ready for commercial/non-commercial use. <br>

## Publisher: <br>

## Use Case: <br>
Developers and other agent users use this skill to turn early creative ideas into a reviewed design through collaborative questioning, option exploration, and staged confirmation before implementation. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: The skill can activate broadly before creative work and may prompt repository review beyond the immediate task. <br>
Mitigation: Ask the agent to confirm scope before reading large parts of a repository. <br>
Risk: The workflow may write design documents under docs/plans or create git commits after design approval. <br>
Mitigation: Require explicit user approval before file writes or git commits. <br>
