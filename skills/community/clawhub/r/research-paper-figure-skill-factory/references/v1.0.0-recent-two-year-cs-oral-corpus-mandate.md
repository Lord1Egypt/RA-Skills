# v1.0.0 Patch — Recent Two-Year CS Top-Conference Oral Corpus Mandate


### 0.0.9 v1.0.0 recent-two-year CS top-conference oral corpus mandate

For the default Skill Builder path, B2 and B3 must use a stricter corpus scope unless the user explicitly overrides it:

> **recent two-year computer-science top-conference oral papers** for the chosen figure class.

Required behavior:

- Interpret "recent two years" dynamically at execution time. Prefer the two most recent complete conference years. If the current year's official oral list has already been published, include it and record why. As of 2026-04-27, the default complete-year window is 2024-2025 unless 2026 official oral lists are available.
- Use computer-science top conferences by default. For method-framework figure skill building, prioritize CVPR/ICCV/ECCV, NeurIPS/ICML/ICLR, ACL/EMNLP/NAACL, then KDD/SIGIR/The Web Conference unless the user narrows the field.
- `presentation_filter` defaults to `oral_required`. Spotlight, award, highly cited, or representative non-oral papers are fallback items only and require explicit user approval; they must not be counted as verified oral papers.
- Every included oral paper must have official oral-status evidence saved in the retrieval manifest. Acceptable sources include official conference programs/schedules, accepted oral-paper lists, OpenReview decisions, CVF/ACL Anthology/conference pages that mark oral status, or user-provided official evidence.
- Do not infer oral status from popularity, citations, social media, unofficial lists, or memory.
- Until oral status is verified and saved, call the set a `candidate oral corpus`, not an `oral corpus`.

Required state fields:

```yaml
corpus_scope:
  year_window: last_2_conference_years
  year_window_resolved: []
  field_scope: computer_science_top_conferences
  venue_set: []
  presentation_filter: oral_required
  oral_evidence_required: true
  official_oral_evidence_sources: []
  unverified_oral_items: []
  fallback_policy: user_approval_required
  fallback_items: []
```

Required manifest behavior:

- `presentation_status` must normally be `oral` for included default-corpus papers.
- `oral_status_verified` must be `true` before the row counts toward the default oral corpus.
- `official_oral_evidence_url` or `official_oral_evidence_local_path` must be populated.
- If official oral evidence cannot be found, move the item to `failed_or_blocked_items.md`, `unverified_oral_items`, or a user-approved fallback list.

This v1.0.0 mandate applies during the **builder-time acquisition** run. The generated specialized skill may later support corpus refresh, but the initial skill-generation corpus must be verified under this rule unless the user explicitly accepts a limited/pilot/fallback scope.

### v1.0.0 full-feasible corpus clarification

The oral-corpus mandate defines the default source type, but it does not authorize a tiny fixed sample when many relevant PDFs are already available. During specialized skill generation:

- enumerate all relevant local/indexed oral PDFs and any user-approved supplemental PDFs;
- process all accessible relevant PDFs as far as feasible, using chunked/resumable extraction if needed;
- record processed, skipped, failed, duplicate, metadata-only, and supplemental counts;
- use representative rendered pages only as audit examples, not as the corpus size;
- allow multi-label classification because one paper or diagram can support several figure-class categories.

If a small subset is used while many relevant PDFs remain available, the generated skill must be labeled `limited`, `pilot`, or `fallback` unless the user explicitly accepts the limitation and the package records it.
