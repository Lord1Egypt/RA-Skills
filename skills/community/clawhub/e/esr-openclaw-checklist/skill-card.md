## Description: <br>
Runs an ESR OpenClaw host security checklist, presents the script output, and summarizes discovered risks without automatically changing system configuration. <br>

This skill is ready for commercial/non-commercial use. <br>

## Publisher: <br>
[xpany37-max](https://clawhub.ai/user/xpany37-max) <br>

### License/Terms of Use: <br>
MIT-0 <br>


## Use Case: <br>
Developers, operators, and security teams use this skill to run and review local OpenClaw host security checks covering ports, listening addresses, tunneling tools, Node.js version, password mode, skill provenance, configuration permissions, and OpenClawd settings. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: The skill promotes recurring automated DingTalk reports to a hardcoded group and references a scheduled helper script that is not included. <br>
Mitigation: Use the manual audit path until the scheduled helper script is supplied and reviewed, the recipient group is verified, and recurring outbound reports are explicitly approved. <br>
Risk: The skill performs local OpenClaw and system inspection and writes local JSON report files. <br>
Mitigation: Run it only on hosts where local inspection and local report file creation are approved, and review results before taking remediation action. <br>
