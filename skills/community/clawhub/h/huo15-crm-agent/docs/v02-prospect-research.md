## 1. 探迹 Tungee（中国 · 智能销售云）

### 公司基本盘
- 总部：广州市；成立 2016 年。
- 核心产品矩阵：探迹拓客 + 探迹智能销售云 + 探迹 SalesGPT。
- 客户规模：累计服务超 5 万家企业客户；2025 年再次入选"中国独角兽企业"，创始人李展入选 2025 福布斯中国颠覆者。
- 客单价：标准化套餐基础配置约 16,800 元/年（10 账号），机器人模块约 9,800 元/年；客户反馈整体属偏中高定价。

### 核心拓客能力
- 数据来源：自研工商/招聘/招标多源知识图谱 + 公开数据爬取。
- 数据规模：1.8 亿+ 市场主体、4.1 亿+ 联系人/关系数据、覆盖 2000+ 细分行业。
- 检索维度：行业 / 规模 / 地域 / 招聘信号 / 招投标信号 / 融资 / 资质 / 上下游关系图谱 / 关联企业。
- 更新频率：动态更新（具体频次未公开）。

### AI 智能化能力（探迹 SalesGPT）
- 多模态大模型 + RAG，自研"太擎"大模型 + "旷湖"数据云底座。
- 已上线：智能筛选 / 潜客推荐 / 智能素材 / 智能话术 / 智能总结 5 大功能。
- 潜客推荐基于"产品关系图谱"——只给主营业务关键词，自动判断产业链上下游。
- 招聘 / 招标信号识别 → 推送高意向企业；电销话术 + 语音合成 + 通话画像自动提取。

### CRM 集成与触达
- 自带 CRM（探迹智能销售云）；支持钉钉 CRM 联动。
- 触达：内置外呼机器人、企微 SOP、邮件/短信。
- 开放平台/API：未在公开页发现完善的开放 API 文档（与企查查相比开放性偏弱）。

### 商业模式
- 年费订阅（按账号 + 数据深度）；标杆客户多在制造、外贸、SaaS、电销密集行业。

## 2. 企查查（中国 · 工商数据底座）

### 公司基本盘
- 总部：苏州；成立 2014 年。
- 定位：与天眼查、启信宝并称"三查一宝"中的第三方企业信息查询/数据 API 服务商。
- VIP 个人版约 388 元/年（与天眼查 360、启信宝 366 同档）；企业 API 调用按量计费。

### 核心数据能力
- 数据规模：3.5 亿+ 工商实体；300+ 数据维度。
- 数据维度：工商基本信息 / 法律诉讼 / 经营风险 / 经营信息（招投标、招聘、专利、商标、备案等）/ 企业发展 / 知识产权 / 关联关系。
- 数据特色：偏金融征信风控数据，与金融机构合作获取一手变更信息。
- 更新频率：动态变更监控（变更日推送/订阅）。

### AI / 智能化能力
- 数据为主、AI 为辅：提供企业身份认证、自动填单、深度关系挖掘、风险动态监测、合作风险评估等组件能力，但不涉及销售话术或意向信号语义识别。
- 关键能力：递名片、深度风险分析、财产线索（SVIP）。

### CRM 集成与触达
- 不做触达。定位是 **数据 + 风控** 底座。
- 开放平台 [openapi.qcc.com](https://openapi.qcc.com/) 提供企业搜索、明细、关联图谱、变更推送等接口；新注册 20 次免费测试调用。
- 调用受 QPS 限速 + 套餐计费（具体单价需联系商务）。

### 商业模式
- C 端会员（388/年）+ B 端 API（按调用次数 + 套餐）+ 企业户私有化。
- 财税行业典型场景：开户尽调、客户工商真实性校验、变更预警。

## 3. 卫瓴科技（中国 · 企微 SCRM 私域获客）

### 公司基本盘
- 总部：上海；成立 2020 年。
- 融资：A 轮亿元（云九资本领投，云启 / 光速中国 / 红杉跟投，2022）。
- 客户规模：400+ 企业，标杆含分众传媒、神州数码、明源云、Shopee、Shopify、畅捷通、分贝通、前程无忧、企查查、360、益海嘉里、得到 App。
- 价格：产品使用费 20 账户起 500 元/账户、40 起 400 元、60 起 360 元；首年技术服务费 5,000 元。

### 核心拓客能力
- 不做"从无到有挖企业"，定位是 **公域 → 企微私域 → 客户孵化** 的转化链路工具。
- 数据来源：广告投放数据 + 用户行为埋点 + 企微聊天记录；不自带 B2B 企业数据库。
- 检索维度：客户标签 / 来源渠道 / 行为打分 / 客户阶段。

### AI / 智能化能力
- 客户孵化 SOP / 商机感知（识别聊天意图）/ 销售赋能话术推荐 / 协同跟进。
- AI 含量在国内 SCRM 中中等偏上（v2026 已加大模型场景，但官网未单列大模型品牌）。

### CRM 集成与触达
- 自身就是协同 CRM；触达走企业微信（聊天 + 朋友圈 + 群发 + SOP 提醒）。
- 与广告投放系统打通（巨量、腾讯广告等）做归因。

### 典型用例
- B2B SaaS、高科技、教培、企服销售线索高频跟进型企业。
- 标杆案例："五格殿下半年内收入提升 15%"、"摩尔精英线索转化率翻倍"。

## 4. Apollo.io（海外 · AI 销售平台）

### 公司基本盘
- 总部：旧金山；成立 2015 年。
- 客户规模：宣称 100 万+ 用户、50 万+ 企业客户（PLG 路径）。
- 定价：免费版（100 credits/月）/ Basic $49 / Professional $79 / Organization $119 每席每月；实际综合开销常 $600–$1,000+/月（信用点消耗大）。

### 核心拓客能力
- 数据来源：自有数据贡献者网络（用户上传 + 互验）+ 公开网爬虫 + 第三方供应商 + 邮件交互 SMTP 反馈验证。
- 数据规模：2.75 亿+ 联系人；公司 + 联系人 + 意图信号一体；邮件准确率宣称 91%。
- 检索维度：行业 / 规模 / 技术栈 / 地域 / 职能 / 招聘动态 / 公司新闻 / 资金事件 / LinkedIn 关联。
- 弱点：北美外（尤其 APAC）覆盖明显下降。

### AI / 智能化能力
- 2026 年 3 月发布 AI Assistant，定位"首个 agentic GTM 操作系统"，自然语言驱动执行工作流。
- AI Research：基于 LinkedIn / 公司新闻自动生成个性化谈话切入点。
- AI Lead Scoring：基于互动信号 + 画像预测转化概率。
- Meeting Intelligence：通话录音 + 摘要 + 自动生成跟进任务。
- 用户参与 AI Assistant 后前 14 天预约率提升 36%。

### CRM 集成与触达
- 双向同步 Salesforce + HubSpot；多触点序列、A/B、自动跟进、活动回写 CRM 时间线。
- 自带邮件序列 / 拨号 / LinkedIn 关联 / 任务提醒。

## 5. ZoomInfo（海外 · 企业级数据 + 意向信号）

### 公司基本盘
- 总部：Vancouver, WA；成立 2000 年（前身 DiscoverOrg），2020 年纳斯达克上市（ZI）。
- 客户：典型企业级 / 中大型销售组织。
- 价格：Professional ~$14,995/年；Advanced ~$24,995/年；Elite $39,995+/年；每席附加 $1,500–$2,500/年。意向数据需 Advanced 起或额外加钱（$5K–$15K/年）。

### 核心数据能力
- 数据规模：2.3 亿+ 联系人、1.74 亿+ 验证邮件、7,000 万+ 直拨电话、5 亿+ 职业档案。
- 检索维度：行业 / 规模 / 技术栈 / 资金 / 招聘 / 高管变动 / 部门组织架构。
- 强项：北美数据精度，欧洲次之；APAC、LATAM 较弱。

### AI / 智能化能力（核心壁垒）
- **Intent Signals**：每月处理 10 亿+ 购买信号（内容消费、网站访问、研究下载）；按"Topic"打分。
- **Scoops**：调研 + 抓取生成的"项目情报"——招聘加速、PPP 资金、新项目启动等触发事件。
- **Champion moves**：识别高管 / 关键人换工作（"老粉去新公司"= 高意向窗口）。
- **WebSights**：站访客识别。
- **Copilot Signals**：把多源信号聚合为账户级触发器，驱动自动化序列。

### CRM 集成与触达
- 主流 CRM 全覆盖（Salesforce / HubSpot / Dynamics / Marketo / Outreach / Salesloft）。
- Workflows：信号触发 → 自动加序列 / 通知销售 / 启动 ABM 战役。
- 触达：邮件 + 拨号 + Engage（原 Chorus 收购线）。

## 7. 绿火 v0.2 启示

> 原则：绿火只做"专业化的财税销售员"，不做"通用拓客 SaaS"。能调用第三方就调用，能让 LLM 推理就推理；OpenClaw 非侵入原则下，**数据接入应通过 return-cliCmd / 配置外接，而非内嵌爬虫**。

### 1. ✅ **借鉴**：接入企查查 API 作为"工商底座" → v0.2 第一优先级
- 实现路径：新增 `sales_company_search` / `sales_company_detail` 工具，配置 `QCC_API_KEY` / `QCC_BASE_URL`，未配置时 graceful fallback 到 LLM 知识。
- 财税场景特别契合：开户尽调、变更预警、关联企业图谱（识别集团客户）、新设企业（潜在代账客户）。
- 不内置爬虫、不缓存大量数据；按调用付费，绿火只做"瘦客户端 + 提示词增强"。
- Why 优先：财税行业 90% 的获客信号都依赖工商数据（新设公司 / 变更 / 经营异常）。

### 2. ✅ **借鉴**：意向信号识别 → v0.2 中等优先级，先做"轻量信号"
- 三家头部都把意向信号当作核心壁垒（探迹招聘+招标、Apollo news+job change、ZoomInfo Topics+Scoops）。
- 绿火 v0.2 实现路径：新增 `sales_intent_signals` 工具，参数化输入"招聘 / 招标 / 融资 / 经营异常"四类信号源 URL 或 RSS（用户自配数据源），LLM 做语义抽取 + 财税相关性打分。
- 不自建爬虫——让用户接 RSSHub / 巨潮 / 中国政府采购网公开 RSS。
- 财税专属信号：注册资本变更、新设分公司、税务行政处罚、招投标中标（中标的需要做账）。

### 3. ✅ **借鉴**：sales_pitch / followup_plan 升级为"动态生成"——参考 Apollo AI Research
- v0.1 的 pitch 是模板化的；Apollo AI Research 演示了"基于公司新闻 + LinkedIn 即时生成切入点"的范式。
- 绿火 v0.2 路径：让 `sales_pitch` 接受 `signals` 参数（来自工具 2 的输出），LLM 基于具体信号生成针对性话术。
- 信号驱动的话术 = 比模板话术高一个量级的转化率（Apollo 数据：14 天预约率 +36%）。

### 4. ⏸ **暂搁**：自建联系人/邮件数据库 → v0.3+ 或永不做
- Apollo / ZoomInfo 都基于 2 亿+ 联系人数据库，绿火没有也不应有这种数据资产。
- 绿火定位是"销售员智能体"，不是"数据公司"——数据交给企查查 / 钉钉名片 / 客户自己的 Odoo。
- v0.3+ 可考虑接 LinkedIn / 脉脉 公开 profile 但仅用于"已知公司补充关键人"，不做反向挖掘。

### 5. ⏸ **暂搁**：自动外呼 / AI 通话 → v0.3+ 或不做
- 探迹 / Apollo / ZoomInfo 都有，但属于重资产能力（语音合成 + 电信线路 + 合规）。
- 财税合规风险高（电销骚扰投诉、个保法），绿火不应自己做。
- 替代：v0.2 输出"话术 + 通话提纲"，让用户在自家 CRM / 企微通话工具里执行。

### 6. ❌ **避开**：把 skill 内嵌爬虫 / child_process 调用 puppeteer 抓页面
- 红线第 4 条：不用 child_process（企业扫描器拦截）。
- 绿火所有"动态数据"必须走外部 API（企查查、用户配置的 RSS、用户自有 Odoo 数据）。
- 实现模式：return-cliCmd 让用户自己跑 curl / 数据准备脚本，绿火只做"基于已有数据做推理"。

### 7. ❌ **避开**：抄袭 SCRM 私域闭环（卫瓴模式）
- 龙虾 / openclaw 已有 wecom / wechat-service 插件做企微触达，绿火若再做就违反"不复制龙虾原生功能"红线。
- 触达交给 `@huo15/wecom`，绿火只生产"该发什么内容 + 何时发"的智能输出，不持有"怎么发"的基建。
- 协作模式：`sales_followup_plan` 输出 → wecom 插件执行 → 闭环交给 enhance memory 记录。

### 8. ✅ **借鉴**：行业知识图谱 / 信号词典——参考探迹"2000+ 细分行业"做法
- v0.2 路径：在 KB（`~/.openclaw/kb/shared/wiki/`）里建一份《财税行业获客信号词典》。
  - 招标关键词："代理记账服务采购"、"税务咨询服务"、"审计服务采购"、"IPO 辅导"。
  - 工商信号："新设公司无社保"、"注册资本变更"、"行业代码变更到 7233（会计服务）"。
  - 招聘信号："招会计 / 财务负责人"反映企业内审升级、可能外包代账。
- 绿火工具读取此 KB 做"信号 → 财税意图"映射。
- 这件事的护城河是"行业 know-how"，不是"数据规模"——绿火可以做出比通用拓客 SaaS 更深的财税信号识别。
