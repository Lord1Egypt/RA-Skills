## 技术发明生成要求

技术发明每种任务类型的"零件清单"——必选/可选组件、组装顺序、领域约束。按清单逐项产出。

---

### 1 问题发现

#### T1-1 需求识别

- **必选组件**: 需求来源（用户/市场/技术）、需求描述、需求优先级、需求验证
- **可选组件**: 需求调研、需求分析、需求文档
- **组装顺序**: 确定需求来源 → 收集需求 → 描述需求 → 排序优先级 → 验证需求
- **约束**: 来源必须可靠；描述必须清晰；优先级必须合理
- **格式**: 需求识别报告（Markdown）

#### T1-2 技术痛点

- **必选组件**: 痛点描述、痛点影响、痛点原因、改进方向
- **可选组件**: 痛点调研、痛点分析、痛点优先级
- **组装顺序**: 识别技术痛点 → 分析痛点影响 → 分析痛点原因 → 确定改进方向
- **约束**: 描述必须具体；影响必须量化；原因必须深入
- **格式**: 技术痛点分析（Markdown）

#### T1-3 创新机会

- **必选组件**: 机会描述、机会来源、机会价值、机会可行性
- **可选组件**: 机会评估、机会优先级、机会验证
- **组装顺序**: 汇总机会来源 → 描述机会 → 评估价值 → 评估可行性 → 排序优先级
- **约束**: 描述必须清晰；来源必须可靠；评估必须客观
- **格式**: 创新机会清单（Markdown）

#### T1-4 技术趋势

- **必选组件**: 趋势范围、趋势分析、趋势预测、创新启示
- **可选组件**: 趋势报告、趋势调研、趋势专家
- **组装顺序**: 确定趋势范围 → 收集趋势信息 → 分析趋势 → 预测趋势 → 提炼启示
- **约束**: 范围必须明确；分析必须深入；预测必须合理
- **格式**: 技术趋势报告（Markdown）

---

### 2 方案构思

#### T2-1 创意生成

- **必选组件**: 创新目标、创意方法、创意数量、创意草图
- **可选组件**: 创意工作坊、创意工具、创意评估
- **组装顺序**: 明确创新目标 → 选择创意方法 → 执行创意生成 → 记录草图 → 初步筛选
- **约束**: 目标必须明确；数量必须充足；方法必须多样
- **格式**: 创意生成记录（Markdown）

#### T2-2 方案设计

- **必选组件**: 技术原理、实现路径、技术参数、方案图纸
- **可选组件**: 方案评审、方案验证、方案优化
- **组装顺序**: 确定技术原理 → 设计实现路径 → 设定技术参数 → 绘制方案图纸
- **约束**: 原理必须可行；路径必须清晰；参数必须合理
- **格式**: 技术方案文档（Markdown）

#### T2-3 可行性分析

- **必选组件**: 技术可行性、资源可行性、时间可行性、风险评估
- **可选组件**: 可行性验证、可行性评审、可行性报告
- **组装顺序**: 评估技术可行性 → 评估资源可行性 → 评估时间可行性 → 评估风险
- **约束**: 评估必须全面；风险必须识别；结论必须明确
- **格式**: 可行性分析报告（Markdown）

#### T2-4 方案评估

- **必选组件**: 评估维度（创新性/实用性/经济性）、评估方法、评估结果、评估结论
- **可选组件**: 方案对比、方案评审、方案选择
- **组装顺序**: 确定评估维度 → 选择评估方法 → 执行评估 → 分析结果 → 形成结论
- **约束**: 维度必须全面；方法必须科学；结论必须客观
- **格式**: 方案评估报告（Markdown）

### 3 发明实现

#### T3-1 原型制作

- **必选组件**: 原型目标、原型类型、制作材料、原型成果
- **可选组件**: 原型测试、原型迭代、原型文档
- **组装顺序**: 确定原型目标 → 选择原型类型 → 准备材料 → 制作原型 → 测试验证
- **约束**: 目标必须明确；类型必须匹配；制作必须规范
- **格式**: 原型制作记录（Markdown）

#### T3-2 技术实现

- **必选组件**: 技术方案、实现步骤、技术参数、实现成果
- **可选组件**: 技术工具、技术文档、技术验证
- **组装顺序**: 确认技术方案 → 制定实现步骤 → 设定技术参数 → 执行实现 → 验证成果
- **约束**: 方案必须可行；步骤必须清晰；参数必须准确
- **格式**: 技术实现文档（Markdown）

#### T3-3 功能验证

- **必选组件**: 验证目标、验证方法、验证标准、验证结果
- **可选组件**: 验证工具、验证环境、验证报告
- **组装顺序**: 确定验证目标 → 设计验证方法 → 设定验证标准 → 执行验证 → 分析结果
- **约束**: 目标必须明确；方法必须可行；标准必须合理
- **格式**: 功能验证报告（Markdown）

#### T3-4 性能优化

- **必选组件**: 优化目标、优化方案、优化效果、优化验证
- **可选组件**: 优化工具、优化测试、优化记录
- **组装顺序**: 确定优化目标 → 设计优化方案 → 执行优化 → 验证效果 → 记录结果
- **约束**: 目标必须明确；方案必须可行；效果必须可验证
- **格式**: 性能优化报告（Markdown）

### 4 专利保护

#### T4-1 专利检索

- **必选组件**: 检索范围、检索策略、检索结果、新颖性评估
- **可选组件**: 检索工具、检索报告、检索分析
- **组装顺序**: 确定检索范围 → 设计检索策略 → 执行检索 → 分析结果 → 评估新颖性
- **约束**: 范围必须全面；策略必须有效；评估必须客观
- **格式**: 专利检索报告（Markdown）

#### T4-2 专利申请

- **必选组件**: 专利类型、申请文件、申请流程、申请费用
- **可选组件**: 专利代理、专利策略、专利布局
- **组装顺序**: 确定专利类型 → 准备申请文件 → 提交申请 → 缴纳费用 → 跟进审查
- **约束**: 类型必须匹配；文件必须规范；流程必须合规
- **格式**: 专利申请记录（Markdown）

#### T4-3 专利维护

- **必选组件**: 维护计划、维护费用、维护记录、维护提醒
- **可选组件**: 维护工具、维护团队、维护预算
- **组装顺序**: 制定维护计划 → 安排维护费用 → 执行维护 → 记录维护 → 设置提醒
- **约束**: 计划必须可行；费用必须到位；记录必须完整
- **格式**: 专利维护记录（Markdown）

#### T4-4 专利运营

- **必选组件**: 运营策略、运营模式、运营收益、运营风险
- **可选组件**: 运营团队、运营平台、运营合同
- **组装顺序**: 制定运营策略 → 选择运营模式 → 执行运营 → 评估收益 → 控制风险
- **约束**: 策略必须可行；模式必须匹配；收益必须合理
- **格式**: 专利运营方案（Markdown）
