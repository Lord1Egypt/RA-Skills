# 🎯 阶段 2：面试题目设计

## 设计原则

> 每道题必须三件套：**考察目标 / 评分标准 / 红灯信号**
> 题目要根据**岗位类型 + 职级**自动差异化
> 追问必须有结构（A/B/C/D 决策树），不能凭感觉

---

## 2.1 方法论选择矩阵

| 考察目标 | 推荐方法 | 不要用 | 原因 |
|---------|---------|--------|------|
| 过往业绩真实性 | Topgrading 时间线 + PEARL + TORC | 假设性问题 | 假设题无法验证真实经历 |
| 技术深度 | 6 层 How 追问 + 协作模拟 | STAR（太浅） | STAR 框架仅适合行为面试 |
| 管理能力 | 具体案例 + 归因追问 + 反向 reference | 价值观问答 | 价值观问答易被套话填充 |
| 文化匹配 | 行为面试 + 失败反思 | 智力测试 / IQ 题 | 智力题与文化无关 |
| 学习能力（转行） | 能力迁移图 + 学习样本 | 直接经验对标 | 转行候选人无直接经验可对标 |
| 抗压稳定性 | 离职原因深挖 + 冲突案例 | 性格测试 | 性格测试可被识破并伪装 |
| 创新能力 | 失败案例 + 反事实假设 | "你最有创新的事" | 表层问题只能获得套话 |
| 团队协作 | 跨部门冲突案例 + 360 反馈 | 自评 | 自评失真严重 |

## 2.2 岗位类型 + 职级 → 题库路由表

### 完整路由表

| 岗位类型 | 低职级题库 | 高职级题库 | 时长 |
|---------|-----------|-----------|------|
| 销售岗 | 销售题库（基础 + 实习应届） | 销售题库（全套 + TORC） | 低 60min / 高 120min |
| 技术岗 | 技术题库（基础 + 实习应届） | 技术题库（全套 + 系统设计加重） | 低 60min / 高 150min |
| 产品岗 | 产品题库（基础 + 实习应届） | 产品题库（全套 + 战略+商业判断） | 低 60min / 高 120min |
| 管理岗 | （管理岗多为高职级） | 管理题库（必做 12 题） | 高 120min |
| 职能岗 | 职能题库（执行型） | 职能题库（体系建设型） | 低 60min / 高 120min |
| 复合岗位 | （依据主类型 + 叠加） | 多题库交叉 | 高 150min |

### 职级深度差异

| 维度 | 低职级问题深度 | 高职级问题深度 |
|------|--------------|--------------|
| 追问层数 | 3-4 层 | 6 层及以上 |
| 案例要求 | 1 个核心案例 | 2-3 个跨场景案例 |
| 反事实思考 | 可选 | 必问 |
| 战略视角 | 不要求 | 必须有 |
| TORC | 可选 | 必问 |
| Reference Check | 视情况 | 建议做（如需要背调请参考行业TORC框架） |

## 2.3 A/B/C/D 追问决策树（核心工具）

```
候选人成果声明（如"提升销售 30%"）
 ↓
 ┌───┴───┐
 │ A 归因 │ → 防夸大
 └───┬───┘
 ↓
 ┌───┴───┐
 │ B 方法 │ → 防套话
 └───┬───┘
 ↓
 ┌───┴───┐
 │ C 度量 │ → 防虚报
 └───┬───┘
 ↓
 ┌───┴───┐
 │ D反事实│ → 防归因谬误
 └───────┘
```

### 维度 A：归因（Attribution）— 防夸大

**核心追问**：这个成果中，你个人的贡献占比多少？团队其他人分别做了什么？外部因素（市场红利 / 公司平台 / 运气）占多少？

**绿灯**：主动量化个人贡献占比（"我大约贡献 60%"）
**红灯**：始终用"我们"代替"我"；把所有功劳归自己或把所有问题推外部

### 维度 B：方法（Method）— 防套话

**核心追问**：你用了什么具体方法实现的？为什么选这个方法而非其他？实施过程中遇到的最大阻力是什么？

**绿灯**：能持续下钻到具体动作；解释方法选择的取舍逻辑
**红灯**：只能停留在战略层（"做了改造"无法下钻）

### 维度 C：度量（Measurement）— 防虚报

**核心追问**：30% 是怎么算出来的？基线数据是什么？这个数据是谁统计 / 审核的？

**绿灯**：数据来源可追溯；区分签约额 / 回款额 / 确认收入
**红灯**：说不清基线数据来源；"大概""差不多"等模糊表述

### 维度 D：反事实（Counterfactual）— 防归因谬误

**核心追问**：如果不做这个项目，结果会怎样？同期是否有市场 / 竞品 / 政策等外部因素影响？如果让你重做一次，你会改什么？

**绿灯**：主动思考"不做会怎样"；承认外部因素和运气
**红灯**："肯定不会成功"的绝对化回答；完全否认外部因素

## 2.4 PEARL 框架完整版

```
P - Problem : 你解决的是什么问题？为什么是问题？
E - Epiphany : 你的关键洞察 / 转折点是什么？
A - Action : 你具体做了什么？（按时间线）
R - Result : 结果如何？用数据说话
L - Learning : 你学到了什么？哪些可复用？
```

### PEARL 完整度评估

```
PEARL 完整度评分：
P：✅ 清晰 / ⚠️ 模糊 / ❌ 缺失
E：✅ 有洞察 / ⚠️ 表面 / ❌ 无洞察
A：✅ 按时间线 / ⚠️ 跳跃 / ❌ 流水账
R：✅ 量化+可追溯 / ⚠️ 量化无来源 / ❌ 无量化
L：✅ 深刻+复用 / ⚠️ 泛化 / ❌ "没什么"

总分 = (✅ 数量 × 1) + (⚠️ 数量 × 0.5)
5 分：完整深入 | 3-4.5 分：基本可信 | < 3 分：内容浅 / 可能造假
```

## 2.5 通用必问问题（所有岗位都问）

| # | 问题 | 考察目标 | 时长 | 红灯信号 |
|---|------|---------|------|---------|
| 1 | 用 2 分钟介绍你的职业经历，重点讲你最希望我了解的 3 件事 | 表达结构 + 重点把握 | 2 min | 流水账 / 无重点 / 超时 |
| 2 | 详细说说你每一次离职的原因 | 稳定性 + 真实动机 | 5-8 min | 全推外部因素 / 前后矛盾 |
| 3 | 你最大的失败 / 教训是什么？事后如何反思？ | 成长型思维 | 5 min | "没什么失败" / 包装失败 |
| 4 | 你的上级会如何评价你？他最不满意你哪里？ | 自我认知 + TORC | 3 min | 完美主义型回避 |
| 5 | 关于今天聊的内容，还有什么是我应该知道但没问到的吗？ | 挖掘隐藏信息 + 自我推介 | 3 min | "没有了"（草草结束） |

## 2.6 TORC 模块（Topgrading 杀手锏）

> **TORC** = Threat of Reference Check（背调威慑）

### TORC 5 大杀手问题（高职级强制问）

| # | 问题 | 考察 | 红灯信号 |
|---|------|------|---------|
| Q1 | "如果我打电话给你的上一任直接上级，他会怎么评价你？" | 自我认知 + 关系坦诚度 | 完美无瑕 / 回避 |
| Q2 | "他会给你打几分（10 分制）？为什么不是满分？" | 诚实承认不足 | 给自己 10 分 / 编造别人评价 |
| Q3 | "他会说你最大的优点和最大的需改进点分别是什么？" | 客观自评 | 缺点变优点（"我太追求完美"） |
| Q4 | "他会推荐我录用你吗？如果犹豫，他会犹豫什么？" | 自我认知 + 关系真实度 | 声称"会无条件推荐" |
| Q5 | "你愿意帮我安排和他的 reference call 吗？" | 行为真实性测试 | 找借口拒绝 / 推荐非直接上级 |
