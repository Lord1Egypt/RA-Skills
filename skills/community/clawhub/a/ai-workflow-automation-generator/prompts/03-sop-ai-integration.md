# Prompt 3: SOP with AI Integration Points

**Tier:** Paid  
**Input needed:** One specific workflow to automate (from your Prompt 1 output)  
**Output:** Complete rewritten SOP with AI touchpoints, prompts, human checkpoints, and time analysis

---

## The Prompt

```
You are an operations specialist rewriting business SOPs to incorporate AI automation.

The workflow I want to automate:
- Workflow name: [e.g., "Guest pre-arrival communication," "Client proposal creation," "Monthly report generation"]
- Current process (step by step): [describe each step as you currently do it]
- How often it runs: [daily / weekly / per transaction / triggered by X]
- Who currently does it: [owner / specific role / outsourced VA]
- Current time per occurrence: [X minutes or hours]
- Volume: [how many times per day/week/month]
- AI tool I'll be using for this: [from Prompt 2 output, e.g., Claude API, ChatGPT, Zapier + Claude, etc.]

Quality requirements:
- Output must sound like: [formal/professional / warm/conversational / technical / on-brand]
- Human review required for: [any outputs that must be checked before sending/publishing]
- Non-negotiables (never automate): [anything that must always have a human decision]

Please generate a complete AI-Integrated SOP with:

1. REWRITTEN PROCESS (step by step)
   For each step:
   - Step number and name
   - Who/what performs it (Human / AI / Human+AI)
   - Input required
   - Action
   - Output
   - Time estimate (before AI vs. after AI)

2. AI PROMPT FOR EACH AUTOMATED STEP
   For every step where AI does work:
   - The exact prompt to use (ready to copy-paste)
   - Variables to fill in (shown in [brackets])
   - Expected output format
   - Quality check: what a good output looks like vs. a bad one

3. HUMAN CHECKPOINTS
   - Every point where a human must review before proceeding
   - What to check for (quality, accuracy, tone, compliance)
   - Decision tree: if the AI output is off, what to do (edit / regenerate / escalate)

4. ERROR HANDLING
   - What to do when AI produces unusable output
   - What to do when an integration fails
   - Fallback manual process (so the workflow never fully stops)

5. TEAM TRAINING NOTES
   - How to explain this SOP to a new team member in 3 sentences
   - Common mistakes to avoid
   - How long until someone is competent at this SOP: [X days of practice]

6. TIME COMPARISON
   - Before AI: [X minutes/hrs per occurrence × Y occurrences/month = Z hours/month]
   - After AI: [X minutes/hrs per occurrence × Y occurrences/month = Z hours/month]
   - Time savings: [X hours/month = Y hours/year]
   - At [$hourly labor rate], this is [$X/month saved]

Format the SOP so it can be saved as a reference document and handed to a VA or team member to follow without additional explanation.
```

---

## Tips for Best Results

- Describe your current process in as much detail as possible — the AI can only improve what it understands
- If you don't know every step, describe the input (what triggers the workflow) and the output (what gets produced) and let the AI infer the middle
- Run Prompt 3 once per automation — don't try to automate multiple workflows in one prompt

---

## What to Do With the Output

Save each SOP in a shared document (Google Docs, Notion, etc.) accessible to your team. Then move to Prompt 4 to build your 90-day implementation roadmap.
