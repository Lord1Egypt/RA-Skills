## Description: <br>
Audits UI components for WCAG accessibility issues and helps improve ARIA labels, color contrast, keyboard navigation, and screen reader support. <br>

This skill is ready for commercial/non-commercial use. <br>

## Publisher: <br>
[hupmann86-cell](https://clawhub.ai/user/hupmann86-cell) <br>

### License/Terms of Use: <br>
MIT-0 <br>


## Use Case: <br>
Developers and UI engineers use this skill to audit interface components for accessibility issues, identify WCAG gaps, and apply fixes for keyboard navigation, ARIA labeling, contrast, and screen reader support. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Accessibility fixes can alter UI behavior, visual contrast, or navigation patterns. <br>
Mitigation: Review proposed changes before applying them and validate with accessibility tools, keyboard testing, and screen reader checks. <br>
Risk: The skill only needs access to the relevant UI project and could expose unrelated files if run too broadly. <br>
Mitigation: Use it in the target project scope and avoid granting access to unrelated repositories or secrets. <br>
