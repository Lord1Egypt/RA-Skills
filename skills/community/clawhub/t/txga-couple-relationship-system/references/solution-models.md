## 解题模型总览

### 六大卡点类型 × 解题模型映射

| 卡点类型 | 核心解题模型 | 核心机制 |
|---------|-------------|---------|
| 沟通型 | **SRC模型** | 暂停(Stop)→映射(Reflect)→连接(Connect) |
| 亲密型 | **3R模型** | 修复(Repair)→重连(Reconnect)→深化(Refine) |
| 权力型 | **GIVE模型** | 协商(Google)→投入(Invest)→验证(Verify)→调整(Adjust) |
| 成长型 | **共同进化模型** | 共同目标→协同行动→相互赋能 |
| 信任型 | **TRUST模型** | 透明(Transparent)→责任(Responsibility)→理解(Understand)→持续(Sustain)→信任(Trust) |
| 价值型 | **ALIGN模型** | 评估(Assess)→学习(Learn)→协商(Integrate)→达成(Negotiate)→落地(Ground) |

## 模型一：沟通重建模型（SRC模型）

### 适用卡点类型
- C04 情绪化对话
- C05 批评升级
- C06 防御机制
- C07 冷战模式
- C08 翻旧账
- C09 沟通回避
- C10 非暴力沟通失败

### 模型结构

#### S - 暂停（Stop）

**核心原理**：情绪高峰期无法有效沟通

**操作步骤**：
1. 识别情绪信号（心跳加速、声音变大）
2. 物理暂停（离开现场或做三次深呼吸）
3. 设定暂停时长（15-30分钟）
4. 约定后续时间继续讨论

**话术**：
- "我现在情绪有点激动，我们休息一下，30分钟后继续谈"
- "我需要冷静一下，这不是逃避，是为了更好地沟通"

#### R - 映射（Reflect）

**核心原理**：确保理解了对方在说什么

**操作步骤**：
1. 复述对方的话（不加入自己的解读）
2. 确认理解是否正确
3. 表达对方话语背后的感受
4. 询问是否准确

**话术**：
- "我听到你说的是...，对吗？"
- "听起来你感到...（情绪），因为...（需求）"
- "我理解得对吗？"

#### C - 连接（Connect）

**核心原理**：在理解的基础上寻求连接

**操作步骤**：
1. 表达自己的感受和需求
2. 寻找共同点
3. 提出解决方案
4. 确认对方是否愿意尝试

**话术**：
- "我的感受是...，我需要..."
- "我觉得我们可以..."
- "你愿意尝试...吗？"

### 应用示例

**卡点**：每次讨论家务分配都变成争吵

**应用SRC模型**：

```
S（暂停）：
"我现在有点激动，我需要休息一下"

R（映射）：
"我听到你说的是你觉得家务分配不公平，你做了大部分，对吗？"

C（连接）：
"我理解你的感受。我最近确实工作很忙，做得少了。
我想和你一起列一个清单，看看怎么分配更公平。"
```

## 模型二：亲密修复模型（3R模型）

### 适用卡点类型
- I01 亲密减少
- I06 情感疏离
- I07 浪漫消失
- I10 肢体语言冷漠
- I19 亲密压力
- I20 虚假亲密

### 模型结构

#### R1 - 修复（Repair）

**核心原理**：先处理伤害，再重建亲密

**修复清单**：
- [ ] 承认疏离/伤害
- [ ] 道歉（如有必要）
- [ ] 理解对方的感受
- [ ] 承诺改变

**操作步骤**：
1. 坦诚承认亲密缺失
2. 表达改变的意愿
3. 询问对方需要什么
4. 做出具体承诺

**话术**：
- "我注意到我们最近亲密少了很多"
- "我理解这让你感到...（感受）"
- "我愿意做出改变，每周..."

#### R2 - 重连（Reconnect）

**核心原理**：从小步开始重建连接

**重连阶梯**：

| 阶段 | 行为 | 频率 |
|-----|-----|-----|
| Level 1 | 微笑、眼神接触 | 每天 |
| Level 2 | 牵手、轻触手臂 | 每天 |
| Level 3 | 拥抱超过3秒 | 每天 |
| Level 4 | 亲吻、抚摸头发 | 每天 |
| Level 5 | 恢复亲密关系 | 商定 |

**操作步骤**：
1. 选择当前能接受的Level
2. 坚持一周
3. 逐步升级
4. 过程中持续沟通感受

#### R3 - 深化（Refine）

**核心原理**：从物理亲密走向情感深度

**深化路径**：
1. 分享脆弱（分享一个秘密或恐惧）
2. 深度对话（讨论深层话题）
3. 共同体验（一起做有意义的事）
4. 意义创造（建立共同的仪式/故事）

**话术**：
- "我想和你分享一件事..."
- "你还记得我们第一次...吗？"
- "我们能不能每年纪念..."

## 模型三：权力平衡模型（GIVE模型）

### 适用卡点类型
- P01 决策不平衡
- P02 家务分配不公
- P03 经济控制
- P04 时间霸权
- P07 情绪控制
- P09 隐私侵犯
- P11 决定后悔

### 模型结构

#### G - 协商（Negotiate）

**核心原理**：权力需要通过协商分配

**协商清单**：
| 议题 | 协商问题 |
|-----|---------|
| 财务 | 共同账户vs私人账户如何分配？ |
| 家务 | 谁做什么？标准是什么？ |
| 时间 | 家庭时间vs个人时间如何平衡？ |
| 决策 | 哪些事需要商量？哪些各自决定？ |
| 社交 | 与异性的边界是什么？ |

**操作步骤**：
1. 列出需要协商的议题
2. 各自写出理想方案
3. 寻找共同点和差异
4. 协商出双方接受的方案

#### I - 投入（Invest）

**核心原理**：权力平衡需要双方都投入

**投入承诺**：
- 我愿意在...方面做出让步
- 我愿意承担...责任
- 我愿意给另一半...自主权

**操作步骤**：
1. 明确自己需要什么权力
2. 明确自己愿意给什么
3. 做出投入承诺
4. 记录并执行

#### V - 验证（Verify）

**核心原理**：定期检查权力是否平衡

**验证问题**：
- 上周的决策中有多少是共同做出的？
- 家务分配是否按照约定执行？
- 我们是否都感到被尊重？

**验证频率**：每周一次简短的权力检查

#### E - 调整（Adjust）

**核心原理**：权力结构需要灵活调整

**调整触发**：
- 某方感到不满
- 生活阶段变化
- 外部压力变化

**调整步骤**：
1. 识别需要调整的地方
2. 提出调整建议
3. 协商新方案
4. 执行并验证

## 模型四：成长同步模型（共同进化）

### 适用卡点类型
- G03 目标不一致
- G04 成长停滞
- G05 单向成长
- G06 方向背离
- G07 共同活动消失
- G09 学习不同步

### 模型结构

#### 第一阶段：共同愿景建立

**愿景对话问题**：
- "5年后，我们希望关系是什么样的？"
- "我们共同想要实现的目标是什么？"
- "我们希望成为什么样的伴侣？"

**愿景金字塔**：

```
        人生愿景
      /          \
   关系愿景    事业愿景
   /    \       /    \
生活    成长  财务  家庭
```

#### 第二阶段：成长路径设计

**个人成长清单**：

| 维度 | 我的目标 | 我的行动 |
|-----|---------|---------|
| 职业 | ___ | ___ |
| 健康 | ___ | ___ |
| 兴趣 | ___ | ___ |
| 学习 | ___ | ___ |

**共同成长清单**：

| 维度 | 共同目标 | 共同行动 |
|-----|---------|---------|
| ___ | ___ | ___ |

#### 第三阶段：协同行动

**协同机制**：
1. 每周分享各自的成长进展
2. 每月评估是否同步
3. 每季度调整目标
4. 每年重新规划愿景

**话术**：
- "这周我在...方面有进展"
- "我需要你在...方面支持我"
- "我们一起做...吧，这样我们可以一起成长"

#### 第四阶段：相互赋能

**赋能清单**：
- 鼓励对方的成长尝试
- 在对方遇到困难时支持
- 庆祝对方的进步
- 接受对方的成长带来的变化

## 模型五：信任重建模型（TRUST模型）

### 适用卡点类型
- T01 小谎言
- T02 信息隐瞒
- T05 承诺不兑现
- T06 欺骗历史
- T07 出轨
- T10 财务隐瞒

### 模型结构

#### T - 透明（Transparent）

**核心原理**：信任建立在透明基础上

**透明清单**：
- [ ] 手机/社交媒体可查看
- [ ] 行程主动告知
- [ ] 财务状况公开
- [ ] 与异性的来往透明
- [ ] 决定前主动沟通

**透明程度约定**：
- 完全透明 vs 有边界透明
- 哪些需要报备 vs 哪些不需要

#### R - 责任（Responsibility）

**核心原理**：重建信任需要承担责任

**责任承担**：
1. 承认错误（不找借口）
2. 理解伤害（承认给对方造成的痛苦）
3. 承诺改变（具体的、可验证的承诺）
4. 持续负责（不是一次道歉就结束）

**话术**：
- "我承认我做错了..."
- "我理解这对你造成了...的伤害"
- "我承诺以后..."

#### U - 理解（Understand）

**核心原理**：需要理解对方的感受

**理解对话**：
- "你能告诉我这件事对你意味着什么吗？"
- "你现在对这段关系有什么担忧？"
- "你需要我做什么来让你感到安全？"

**理解表达**：
- "我理解你现在很难信任我"
- "我理解你需要时间来愈合"
- "我理解你的担忧..."

#### S - 持续（Sustain）

**核心原理**：信任重建需要持续努力

**持续行动**：
1. 保持承诺
2. 主动报告
3. 接受质疑
4. 创造新记忆

**信任检验点**：
- 1个月：建立基本透明
- 3个月：显示持续改变
- 6个月：开始重建安全感
- 1年：信任初步恢复

#### T - 信任（Trust）

**核心原理**：最终目标是重建信任

**信任恢复标志**：
- 不再需要时刻监控
- 可以处理对方独处
- 可以讨论未来
- 可以原谅过去

## 模型六：价值对齐模型（ALIGN模型）

### 适用卡点类型
- V01 金钱观差异
- V02 家庭观差异
- V03 育儿观差异
- V04 性爱观差异
- V06 人生观差异
- V14 职业观差异

### 模型结构

#### A - 评估（Assess）

**核心原理**：先了解自己的价值观

**价值观评估问题**：
- "这件事对我来说为什么重要？"
- "我的这个价值观来自哪里？"
- "它是我真正相信的还是被灌输的？"

**评估清单**：
- [ ] 我的金钱观是什么？来自哪里？
- [ ] 我的家庭观是什么？来自哪里？
- [ ] 我的育儿观是什么？来自哪里？

#### L - 学习（Learn）

**核心原理**：了解对方的价值观来源

**学习对话**：
- "你能告诉我你对这件事的看法吗？"
- "这个对你为什么重要？"
- "你是怎么形成这个想法的？"

**学习目标**：
- 理解对方观点
- 不评判对方
- 找到差异的根源

#### I - 整合（Integrate）

**核心原理**：整合双方的价值观

**整合策略**：

| 策略 | 适用情况 | 结果 |
|-----|---------|-----|
| 融合 | 有共同基础 | 创造新的共同价值观 |
| 妥协 | 都有一定弹性 | 双方各让一步 |
| 领域分离 | 某领域归一方决定 | 尊重各自领域 |
| 优先级 | 某些价值观优先 | 明确哪个更重要 |

#### N - 协商（Negotiate）

**核心原理**：通过协商达成一致

**协商框架**：
1. 确定核心需求（不可妥协的）
2. 确定可调整空间
3. 寻找双赢方案
4. 书面记录协议

**协商话术**：
- "我的底线是...，但我可以接受..."
- "你在这个事情上的底线是什么？"
- "我们能不能找到一个...的方案？"

#### G - 落地（Ground）

**核心原理**：协议需要落地执行

**落地清单**：
- [ ] 明确约定内容
- [ ] 约定执行标准
- [ ] 约定检查频率
- [ ] 约定调整机制

**执行检查**：
- 每周检查执行情况
- 每月评估是否有效
- 每季度重新协商

## 通用解题框架

### 解题流程图

```
发现问题
    ↓
识别卡点类型
    ↓
追溯生成因素
    ↓
匹配解题模型
    ↓
制定行动方案
    ↓
执行并追踪
    ↓
评估效果
    ↓
调整或升级
```

### 问题诊断流程

**第一步：识别问题**
1. 这个问题具体是什么？
2. 什么时候开始的？
3. 频率如何？
4. 影响多大？

**第二步：分类**
1. 这是什么类型的卡点？（沟通/亲密/权力/成长/信任/价值）
2. 属于哪个层级？（表层/中层/深层）
3. 严重程度？（轻度/中度/重度）

**第三步：溯源**
1. 什么因素导致了这个卡点？
2. 是外部触发还是内部结构？
3. 是动态问题还是深层问题？

**第四步：匹配**
1. 选择对应的解题模型
2. 按照模型步骤执行
3. 如有必要，组合使用多个模型

**第五步：追踪**
1. 设定评估时间点
2. 记录进展
3. 调整方案
4. 必要时升级处理

### 解题优先级

| 优先级 | 卡点类型 | 处理方式 |
|-------|---------|---------|
| P0 | 信任型-出轨/欺骗 | 危机处理，优先稳定 |
| P0 | 权力型-暴力/控制 | 安全第一，必要时离开 |
| P1 | 信任型-其他 | 信任重建模型 |
| P1 | 价值型-根本冲突 | 深度协商或接受差异 |
| P2 | 权力型-其他 | 权力平衡模型 |
| P2 | 沟通型-模式问题 | 沟通重建模型 |
| P3 | 亲密型 | 亲密修复模型 |
| P3 | 成长型 | 成长同步模型 |
| P3 | 沟通型-表层 | 直接沟通 |

### 模型组合使用

**常见组合场景**：

| 场景 | 组合模型 |
|-----|---------|
| 沟通问题+信任问题 | SRC + TRUST |
| 亲密问题+权力问题 | 3R + GIVE |
| 成长问题+价值问题 | ALIGN + 共同进化 |
| 复合问题 | 先处理信任/权力，再处理其他 |
