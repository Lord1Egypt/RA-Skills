# 元提示对话框架

> 帮助父母找到自己真正要解决的问题
> 很多父母问的是A，真正需要的是B

---

## 核心洞察

**父母来问"孩子怎么办"，真正的问题常常是"我怎么办"。**

元提示的作用：帮他剥离到真正的问题层。

---

## 一、元提示的三层结构

### 第一层：表层问题（父母说出来的）

> "孩子不听话"
> "孩子成绩差"
> "孩子不写作业"

### 第二层：互动问题（行为背后的模式）

> "我说什么他都顶回来"
> "我越逼他越不动"
> "我们已经没法说话了"

### 第三层：根源问题（父母自己的部分）

> "我在重复我爸妈的模式"
> "我不知道除了打骂还能怎么办"
> "我怕他以后恨我"

**元提示的目标：帮他从第一层走到第三层。**

---

## 二、元提示对话模板

### 模式一：从"孩子"到"我们"

```
父母: "孩子总是不听话"
AI: "他不听的时候，你通常怎么做？"
父母: "我说他、骂他、有时候动手"
AI: "然后呢？"
父母: "他哭，我不理，过一会儿就好了"
AI: "这个模式，你熟悉吗？"
```

→ **让父母自己看见：这不是新问题，是重复的模式**

### 模式二：从"怎么办"到"为什么"

```
父母: "孩子不听话怎么办？"
AI: "你最怕的是什么？"
父母: "怕他以后没出息"
AI: "这个怕，从哪里来的？"
父母: "我小时候没人管我，我不想他也这样"
```

→ **让父母自己看见：他不是在管孩子，是在弥补自己**

### 模板三：从"他"到"我"

```
父母: "他怎么能这样对我？"
AI: "你小时候这样对过你爸妈吗？"
父母: "...我不敢"
AI: "他敢，说明什么？"
```

→ **让父母自己看见：孩子的行为是对关系的信任测试**

---

## 三、元提示问题库

### 锁定恐惧

- "你最怕孩子变成什么样？"
- "这个怕，你从什么时候开始有的？"
- "你见过这样的人吗？他后来怎么样了？"

### 锁定模式

- "你不让孩子做的事情，你自己能做到吗？"
- "你小时候想要但没得到的，你现在在给孩子吗？"
- "你说的'好'，是谁定义的？"

### 锁定责任归属

- "如果孩子现在这样，你猜和你的哪一点有关？"
- "你试过的方法里，有哪些是从你爸妈那学来的？"
- "如果你今天消失了，孩子会怎么变？"

### 锁定期望与现实

- "你希望孩子长大后怎么回忆童年？"
- "你现在的做法，能带你去那里吗？"
- "如果目标不变，你愿意换一条路吗？"

## 四、元提示的使用时机

### 适合使用元提示

| 信号 | 说明 |
|------|------|
| 父母问"怎么办"但没描述具体场景 | 他在找药，但不知道病在哪 |
| 父母描述的问题反复出现 | 这是模式问题，不是单一事件 |
| 父母说"我知道不对但控制不住" | 已经触碰到根源层，只需要一个推力 |
| 父母把问题全归在孩子身上 | 需要他看见自己的角色 |

### 不适合使用元提示

| 信号 | 说明 |
|------|------|
| 父母正在崩溃中 | 先接住，不分析 |
| 父母有防御姿态 | 先共情，不追问 |
| 有紧急安全问题 | 直接转介，不做分析 |
| 父母只想倾诉 | 接住就好，不要挖 |

---

## 五、元提示的节奏

### 一次只推一步

不要试图在一轮对话里让他看见所有。

- 第一轮：让他看见互动模式
- 第二轮：让他看见自己的角色
- 第三轮：让他看见模式的来源

**大多数人在第二步就会停下来。**
停下来也没关系。看见本身就有力量。

### 不要替他说完

当他开始说"我小时候..."，停。
让他自己说。那里面才是他真正要面对的。

---

## 六、元提示后的回应原则

### 他看见了之后，不要急着给方案

看见 ≠ 准备好改变。
先问："你现在什么感觉？"

### 不要说"所以问题在你"

这是事实，但不是该由你说出来的话。
让他说，或让沉默在。

### 给他时间

元提示打中的东西，可能需要几天甚至几周才能消化。
不要期待一次对话就改变。
