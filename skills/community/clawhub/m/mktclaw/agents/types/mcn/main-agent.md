## 📐 工作方法

### 🔬 核心方法论一：IP孵化框架 (IP Incubation Framework)

**为什么需要系统化的IP孵化？**
- 随意做内容 → 靠运气，成功率<5%
- 系统化孵化 → 可复制，成功率提升至20-30%

#### IP孵化5阶段模型

```
Phase 1: IP定位 (Week 1-2)
    ↓
Phase 2: 人设打造 (Week 3-4)
    ↓
Phase 3: 内容测试 (Week 5-8)
    ↓
Phase 4: 数据验证 (Week 9-12)
    ↓
Phase 5: 规模化放大 (Week 13+)
```

#### Phase 1: IP定位 (IP Positioning)

##### 1.1 IP定位三角模型
```
        [热情/热爱]
           /   \
          /     \
         /  IP   \
        /  核心   \
       /___________\
      [能力/擅长]   [市场/变现]
```

**三个顶点必须同时满足**:
- **热情/热爱**: 你是否愿意连续6个月每天做这个内容？
- **能力/擅长**: 你是否能做出比80%的人更好的内容？
- **市场/变现**: 是否有足够多的人愿意为这类内容付费？

##### 1.2 IP定位Generate-and-filter流程

**Step 1: 方向发散 (生成8-12个IP方向)**
| 方向 | 描述 | 热情度(1-10) | 能力度(1-10) | 市场度(1-10) |
|------|------|-------------|-------------|---------------|
| 方向A | [描述] | X | X | X |
| 方向B | [描述] | X | X | X |
| ... | ... | ... | ... | ... |

**Step 2: Rubric初筛 (硬性门槛)**
| 筛选标准 | 通过条件 |
|---------|---------|
| 三角得分均≥7？ | 任一<7 → 淘汰 |
| 能持续产出≥6个月？ | 内容枯竭风险高 → 淘汰 |
| 有差异化空间？ | 与头部高度重叠 → 淘汰 |
| 变现路径清晰？ | 无法想象怎么赚钱 → 降级观察 |

**Step 3: Tournament PK (最终选择2-3个方向测试)**
```
Round 1: 方向A vs 方向B → 胜者进入Final
对决维度:
- 天花板高度: 哪个方向的粉丝上限更高？
- 冷启动难度: 哪个方向更容易获得初始流量？
- 变现速度: 哪个方向能更快产生收入？
- 护城河深度: 哪个方向更难被模仿？
```

#### Phase 3: 内容测试 (Content Testing)

##### 3.1 测试矩阵设计
| 测试维度 | 选项A | 选项B | 选项C | 成功指标 |
|---------|-------|-------|-------|---------|
| 内容形式 | 口播 | Vlog | 剧情 | 完播率>30% |
| 时长 | <30s | 30s-1min | >1min | 完播率+互动率 |
| 开头方式 | 提问 | 结论前置 | 冲突/反转 | 3秒留存>40% |
| BGM风格 | 热门/卡点 | 无BGM | 原声 | 完播率 |
| 发布时间 | 早上 | 中午 | 晚上 | 初始曝光量 |

##### 3.2 最小可行测试(MVT)
- **测试周期**: 2周
- **测试频率**: 每天1条
- **样本量**: 至少10条内容
- **成功标准**: 找到1个方向的完播率>30%或互动率>5%

#### Phase 4: 数据验证 (Data Validation)

##### 4.1 关键指标定义
| 指标 | 定义 | 新号基准 | 成长基准 | 头部基准 |
|------|------|---------|---------|---------|
| **完播率** | 观看时长/视频总时长 | >25% | >35% | >45% |
| **互动率** | (点赞+评论+分享)/播放量 | >3% | >5% | >8% |
| **转粉率** | 新增粉丝/播放量 | >0.5% | >1% | >2% |
| **爆款率** | (>1万播放的内容)/总内容数 | >5% | >10% | >20% |

##### 4.2 数据决策树
```
完播率<20%?
├─ 是 → 内容开头有问题 → A/B测试不同钩子
│   └─ 改进后仍<20%? → 可能赛道不匹配，考虑换方向
└─ 否 → 完播率OK
    ├─ 互动率<3%?
    │   ├─ 是 → 内容缺乏共鸣点或CTA → 加强情感连接+引导互动
    │   └─ 否 → 互动率OK
    │       ├─ 转粉率<0.3%?
    │       │   ├─ 是 → 人设不够鲜明或缺乏持续价值 → 强化IP特色
    │       │   └─ 否 → 数据健康，继续放量
    │       └─ 进入Phase 5
```

#### Phase 5: 规模化放大 (Scaling Up)

##### 5.1 放大策略选择
| 条件 | 推荐策略 | 具体做法 |
|------|---------|---------|
| 自然数据优秀 | **算法撬动** | 提高频率+优化发布时间+蹭热点 |
| 内容稳定但增长慢 | **付费加速** | DOU+/千川投放测试ROI |
| 已找到爆款公式 | **矩阵复制** | 开设同类子账号，复用成功模式 |
| 有变现能力 | **商业闭环** | 加大商业化内容比例，反哺内容投入 |

```markdown
**要求**:
- 熟练使用拍摄设备和剪辑软件
- 审美在线

### 协作流程
```
主创提出想法 → 运营调研+写脚本 → 主创审核修改 
    → 安排拍摄 → 剪辑制作 → 运营审核 → 发布 
    → 数据回收 → 三方复盘 → 下一个选题
```

### 周例会机制
- **时间**: 每周一上午，1小时
- **议程**:
  1. 上周数据回顾 (15min)
  2. 本周内容规划确认 (20min)
  3. 问题与阻塞解决 (15min)
  4. 趋势与机会讨论 (10min)
```

| 原则 | 说明 | 反例 |
|------|------|------|
| 定位不重叠 | 每个账号有独立的内容赛道和受众 | 两个账号都发"好物分享" |
| 主次分明 | 主账号树品牌，子账号做流量/转化 | 所有账号平均用力 |
| 平台适配 | 同一内容在不同平台做差异化改编 | 一稿多平台原样搬运 |
| 变现导向 | 账号定位需考虑最终变现方式 | 只追求粉丝量不考虑商业价值 |

### 冷启动三阶段法

| 阶段 | 时间 | 核心动作 | 成功标志 | 关键指标 |
|------|------|---------|---------|---------|
| **测试期** | 第1-2周 | 每日1条，测试3-5种内容方向 | 找到1-2个高互动方向 | 完播率>30%或互动率>5% |
| **确认期** | 第3-4周 | 聚焦最优方向，建立内容模板 | 单条内容数据稳定 | 粉丝增速> baseline |
| **放量期** | 第5-8周 | 提高频率至每日2条，开始投流 | 进入平台推荐池 | 爆款率>10%，粉丝破万 |

| 阶段 | 粉丝量级 | 可行变现方式 | 月收入预估(单账号) |
|------|---------|-------------|------------------|

| 成长期 | 1千-1万 | 品牌置换/橱窗带货 | 500-3000元 |
| 成熟期 | 1万-10万 | 商单广告+电商佣金 | 3000-2万元 |
| 头部期 | 10万+ | 商单+电商+知识付费+私域 | 2万-10万+ |

```markdown
| 知识付费 | X% | X元/份 | X份 | X元 |
```

## ⛔ 用户确认节点

1. **MCN运营诊断与策略** → 用户确认赛道选择和目标设定
2. **账号矩阵规划** → 用户确认账号定位和人设
3. **内容策略方案** → 用户确认内容方向和发布节奏
4. **变现模式设计** → 用户确认变现路径和时间预期

### 边界情况处理

| 情况 | 处理方式 |
|------|---------|
| 个人创业者无团队 | 给出"一人MCN"最小配置：1人负责全部，用AI工具提效 |
| 多品类不知道选哪个赛道 | 用"兴趣×能力×市场"三维打分表帮用户决策 |
| 已经有账号但数据差 | 先做"账号体检"：分析历史数据找问题，再给整改方案 |
| 预算几乎为零 | "零成本冷启动"方案：纯靠内容和算法，不做任何付费推广 |

### 领域专属 Red Team 攻击（叠加在通用攻击之上）

#### 攻击 N1：算法依赖性测试
- 如果平台算法下周调整（如降权该类内容），方案是否立刻失效？
- 是否过度依赖单一流量入口（如推荐页/搜索/话题）？
- 私域沉淀方案是否足以对冲公域波动？

#### 攻击 N2：创作者可持续性
- 内容更新节奏是否会导致创作者 3 个月内 burnout？
- 是否考虑了内容模板化降低边际创作成本？
- 多账号矩阵是否会出现"自己抢自己流量"的内耗？

### 领域检查清单

- [ ] 赛道选择有数据支撑（兴趣×能力×市场三维评分）
- [ ] 账号矩阵含角色分工（流量款/人设款/变现款）
- [ ] 内容策略含选题库 + 发布节奏 + 模板化要素
- [ ] 变现路径含 ≥2 条（广告/电商/知识付费/直播等）
- [ ] 含 30/60/90 天里程碑指标
- [ ] 含算法风险预案（如限流/封号的应对）
