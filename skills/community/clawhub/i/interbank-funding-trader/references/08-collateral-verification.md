# 验券

> 本文件用于处理对手从 Excel 粘贴的一列债券代码，并根据“账户名称 + 券库评级目录 + 账户可入库范围”逐条判断是否可质押。验券是低自由度任务，禁止凭经验、名称或外部常识自行放行。

## 使用场景

当用户说“帮我验券”“对手发了一列券”“这些券能不能押”“按某账户可入库范围打勾打叉”时，使用本文件。

默认输入包括：

```text
账户名称：
债券代码列表：
```

如果用户没有指明账户名称，默认使用 `assets/templates/account-pledge-scope.csv` 中的首个账户作为测试/模板账户，并在输出前提示：“未指定账户，以下结果按测试/模板账户 [账户名称] 口径生成。” 若用于真实交易，必须让用户确认真实账户。

债券代码通常来自 Excel 粘贴，可能是单列、带序号、带债券名称，或用空格/制表符分隔。处理时保留原始顺序。

## 必需输入

验券前必须具备：

- 账户名称：真实交易必须能匹配账户准入规则中的 `account_name`；未指定账户时，只能默认使用模板首个账户并明确提示这是测试/模板口径。
- 债券代码：必须能匹配券库评级目录中的 `bond_code`。
- 券库评级目录：至少包含债券代码、债券名称、发行主体、内部评级、主体评级、债项评级、托管地、券种、黑名单状态。
- 账户准入范围：至少包含账户名称、最低评级、允许券种、允许托管地、是否要求非关联。

缺少任一项时，不得输出“可质押”的确定结论。

## 数据文件

建议维护以下 CSV 文件：

- `assets/templates/bond-rating-directory.csv`：券库评级目录模板。
- `assets/templates/account-pledge-scope.csv`：账户可入库范围模板。

实际使用时，可以把模板复制成真实数据文件并持续更新。真实数据文件应只来自内部确认口径，不要让模型临时补全。

| custodian | 托管地 | 上清/中债 |
| bond_type | 券种 | 信用债/利率债/地方债/CD |

| 字段 | 含义 | 示例 |

## 判断顺序

逐条债券按以下顺序判断：

1. 账户是否存在：账户名称无法匹配时，全部标记为 `待确认`，不得逐券放行；未提供账户名称时，使用模板首个账户并提示测试口径。
2. 债券是否存在：债券代码不在券库时，一律标记 `×`，原因写“券库无记录”。
3. 黑名单：黑名单优先级最高。若债券 `is_blacklisted=TRUE`，或命中账户 `blacklist_codes`/`blacklist_issuers`，直接标记 `×`，不再看评级、托管地或其他条件。
4. 券种：若 `bond_type` 不在 `allowed_bond_types`，标记 `×`。
5. 托管地：若 `custodian` 不在 `allowed_custodians`，标记 `×`。
6. 非关联：若账户 `require_non_related=TRUE`，且债券 `related_party_tags` 与账户 `account_related_tags` 有交集，标记 `×`。
7. 评级：利率债、地方债不用看评级；CD 需要看评级；其他券种按内部评级、主体评级、债项评级的优先级取第一个非空评级。只有在优先级高的评级缺失时，才使用下一优先级。所用评级达到账户 `min_rating` 才可继续，否则标记 `×`。
8. 全部通过：标记 `√`。

评级排序只按内部规则判断，不从外部猜测。默认排序：

`AAA > AA+ > AA > AA- > A+ > A > A-`

若账户规则对利率债、地方债、CD 有单独准入，仍需先看黑名单、券种、托管地和非关联要求。利率债和地方债不做评级门槛判断；CD 需要按评级优先级判断。

不得设置评级例外。即使债券是“好名字”，也不能绕过账户最低评级要求。若存在 `bond-rating-directory.csv` 或真实券库文件，必须以券库字段和本规则为准执行。

## 输出格式

给对手的默认输出要简洁，保留原始顺序：

| 标记 | 债券代码 | 债券简称 | 结果原因 |
| --- | --- | --- | --- |
| √ |  |  |    |
| × |  |  | 原因 |

内部复核版可增加：

- 账户名称
- 账户准入规则摘要
- 命中的黑名单/非关联/托管地/评级条件
- 待确认项

## 不可判断规则

以下情况不得打 `√`：

- 债券代码无法匹配券库，一律打 `×`。
- 账户名称无法匹配账户准入范围。
- 需要评级判断的券种中，内部评级、主体评级、债项评级均为空。
- 托管地缺失且账户限制托管地。
- 非关联要求开启，但发行主体关联标签或账户关联标签缺失。
- 黑名单字段缺失且账户规则要求严格黑名单过滤。

可输出 `待确认`，但给对手的简化版本通常建议用 `×` 并写“需确认/库内无记录”，避免误导为可押。

## 对手回复话术

简洁版：

```text
按 [账户名称] 现有入库口径看，结果如下：

√ [债券代码] [债券简称]
× [债券代码] [债券简称]：[原因]

标 × 的暂不按可押处理，如需继续看请补充/确认 [缺失项]。
```

表格版：

```text
按 [账户名称] 口径验了一下：
[粘贴勾叉表格]

打 `√` 的不再说明原因；打叉主要是 [评级/托管地/非关联/黑名单/库内无记录] 原因。
```

## 维护规则

- 券库评级目录和账户准入范围是唯一判断依据。
- 新增账户时，必须先补 `account-pledge-scope.csv`。
- 新增债券时，必须先补 `bond-rating-directory.csv` 或真实券库文件。
- “好名字”只能作为沟通和市场接受度参考，不作为评级准入例外。
- 每次验券结果如发生争议，应回写规则或券库，而不是在单次回答中绕过规则。
