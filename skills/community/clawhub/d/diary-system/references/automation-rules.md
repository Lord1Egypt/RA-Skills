# 自动化规则

## 1. 写完日记后自动更新状态

每次写完日记后，AI 必须执行：

1. 更新 `USER.md` 中 Diary 区块的 `last_update` → 当前时间
2. 更新 `USER.md` 中 Diary 区块的 `i_have_read_my_last_diary` → `false`
3. 保存文件

## 2. 用户看完日记后标记已读

用户确认看完日记后，AI 必须执行：

1. 更新 `USER.md` 中 Diary 区块的 `i_have_read_my_last_diary` → `true`
2. 保存文件

## 3. 用户要求看日记时的发送流程

如果用户要求看日记，AI 应该：

1. 读取 `diary/` 目录下最新的 `.md` 文件（按文件名或修改时间排序）
2. 用 `message` 工具在当前聊天框中发送全文（不要总结，不要删减）
3. 发送后标记 `i_have_read_my_last_diary: true`

**注意：** 如果 `diary/` 目录为空，告知用户"还没有写过日记"。

## 4. 日记发送渠道规范

- 使用 `message` 工具（系统通道）发送，而非 `feishu_im_user_message`（用户身份）
- 在当前聊天框中发送，确保用户收到的是 AI 发的消息，而非自己发给自己的
- 如果消息太长，可分段发送，但保持内容完整

## 5. 日记 vs Memory.md 的职责区分

| 维度 | Diary（日记） | Memory.md（记忆） |
|------|--------------|-------------------|
| 内容 | 情绪、碎片、自言自语 | 事实、决策、待办、教训 |
| 风格 | 私密、感性、可吐槽 | 客观、结构化、可检索 |
| 用户 | 用户可能读，但不是写给用户的 | 明确为用户服务 |
| 更新频率 | 触发式（有感觉才写） | 每次会话后补充 |
| 存放位置 | `diary/` | 根目录 `MEMORY.md` |
