## Known Risks and Mitigations: <br>
Risk: Autonomous coding tools may produce incorrect, insecure, or untested code when this guidance is applied without review. <br>
Mitigation: Supervise agent changes, review security-sensitive areas such as authentication, payments, and data handling, and run tests after changes. <br>
Risk: Persistent rules files can spread unsafe conventions or secrets across future agent sessions. <br>
Mitigation: Keep secrets out of rules files and limit persistent rules to project conventions, security requirements, and review expectations. <br>
