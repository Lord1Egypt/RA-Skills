# 客户启动工作流

## 适用场景

当公司确认一个新客户或新项目，需要完成需求确认、资料收集、项目启动和内部排期时使用。

## 流程步骤

1. **确认合作范围**
   - 负责人：待填写
   - 输出：确认后的交付内容、价格、时间节点。

2. **收集客户资料**
   - 负责人：待填写
   - 输出：品牌资料、参考案例、历史文件、关键联系人。

3. **内部立项**
   - 负责人：待填写
   - 输出：项目负责人、协作成员、排期、风险点。

4. **启动沟通**
   - 负责人：待填写
   - 输出：客户确认的启动信息和下一步安排。

## 异常处理

- 如果客户需求不清晰：先补一次需求澄清，不直接进入执行。
- 如果资料不完整：记录缺口，并明确由谁在什么时间前补齐。
