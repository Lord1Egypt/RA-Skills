# li-bid-document-maker 使用ガイド

> **入札書作成エキスパート** — 入札説明書（ tender document ）を採点基準に基づいたプロフェッショナルな応札書に自動変換します

## インストール

```bash
clawhub install li-bid-document-maker
```

## クイックスタート

インストール後、AIアシスタントに以下のように話しかけるだけでスキルが起動します：

- 「入札書を作成して」
- 「この入札説明書を分析して応札書を生成して」
- 「応札書の準備を手伝って」
- 「RFPへの回答を生成して」

AI が自動的に6段階のワークフローを実行します：**入札書解析 → 戦略分析 → アウトライン生成 → 章ごとの作成 → 品質チェック → PDCA自動改善**

## 使用方法

### 方法1：ファイルをアップロード

入札説明書（PDFまたはWord）をチャットにドラッグ＆ドロップし、次のように言います：
> 「li-bid-document-maker を使ってこの文書を分析してください」

### 方法2：ファイルパスを指定

> 「li-bid-document-maker を使って、入札書は /projects/tender.pdf にあります」

## システム要件

| 項目 | 要件 |
|------|------|
| OS | Windows 10/11, macOS, Ubuntu 20.04+ |
| AI プラットフォーム | Claude, OpenClaw, Hermes、またはファイルI/O対応のLLM Agent |
| ファイル形式 | PDF（検索可能推奨）または Word (.docx) |
| 依存パッケージ | `python-docx`、`PyPDF2`（ファイル処理用、オプション） |

## 出力

- 完全な応札書（16章の標準構成）
- PDCA品質改善レポート
- 採点基準と配点のマッピング

## ワークフロー

```
段階1: 入札書解析       → プロジェクト情報、技術要件、採点基準を抽出
段階2: 戦略分析         → 採点权重分析、競争戦略の策定
段階3: アウトライン生成 → 採点基準に基づく構成案（ユーザー確認必須）
段階4: 章ごとの作成     → セクションごとに内容を作成
段階5: 品質チェック     → 6次元の完全カバレッジチェック
段階6: PDCA改善         → 3ラウンドの自動改善後、最終納品
```
