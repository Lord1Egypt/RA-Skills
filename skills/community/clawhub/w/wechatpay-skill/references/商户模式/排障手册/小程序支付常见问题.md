# 小程序支付常见问题

> 小程序开发者专属问题排查

## 小程序支付特殊流程

```
1. 前端：wx.login() → 获取code
2. 后端：用code换openid
3. 后端：调用微信支付下单API
4. 后端：返回prepay_id
5. 前端：wx.requestPayment() → 调起支付
6. 后端：接收支付成功回调
```

## 常见问题

### Q1: openid获取方式？

**公众号网页**：用户授权获取code → 后端用code换openid
**小程序**：wx.login()获取code → 后端用code换openid

**换openid接口**：
```
GET https://api.weixin.qq.com/sns/jscode2session?appid=APPID&secret=SECRET&js_code=CODE&grant_type=authorization_code
```

### Q2: prepay_id过期了？

prepay_id有效期2小时，超时后需重新调用下单接口获取新的prepay_id。

### Q3: 小程序调起签名怎么构造？

与JSAPI相同，签名串4行：
```
{appId}\n
{timeStamp}\n
{nonceStr}\n
{package}\n
```

**注意**：appId使用小程序AppID，不是公众号AppID

### Q4: 报"appid和mch_id不匹配"？

检查项：
1. 商户号(mchid)和小程序AppID是否在商户平台绑定
2. 绑定后需等待生效（约几分钟）

### Q5: 支付目录怎么配置？

小程序支付**不需要配置支付授权目录**（这是JSAPI/H5的场景）。

但需确认：
1. 小程序已认证（个人小程序不支持支付）
2. mchid和AppID已绑定

### Q6: 用户取消支付后怎么办？

1. 不自动处理，用户可重新发起支付
2. 可在前端监听用户取消事件提示重试
3. 已创建的订单不会自动关闭，超时后需调用关单接口

### Q7: 支付成功但回调没收到？

排查：
1. notify_url是否公网可达
2. 是否正确返回200
3. 服务器是否有防火墙拦截
4. 微信支付可能重试回调（最多24小时）

### Q8: 同一订单重复支付？

检查：
1. 前端是否做了防重复点击
2. 后端是否检查订单状态
3. out_trade_no是否唯一

## 小程序支付代码要点

### 获取openid（Python）

```python
import requests

def get_openid(code: str, appid: str, secret: str) -> str:
    url = "https://api.weixin.qq.com/sns/jscode2session"
    params = {
        "appid": appid,
        "secret": secret,
        "js_code": code,
        "grant_type": "authorization_code"
    }
    resp = requests.get(url, params=params)
    data = resp.json()
    return data.get("openid")
```

### 小程序调起支付参数

```javascript
// 前端调起
wx.requestPayment({
    timeStamp: '1600000000',
    nonceStr: 'C5BE8E6B1B7A5D3E',
    package: 'prepay_id=wx201410272009395522657a690289100',
    signType: 'RSA',
    paySign: 'xxxxx',  // 后端签名
    success: (res) => { console.log('支付成功') },
    fail: (res) => { console.log('支付失败') }
})
```
