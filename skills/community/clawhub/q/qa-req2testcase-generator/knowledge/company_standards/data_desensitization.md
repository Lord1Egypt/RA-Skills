# 测试数据脱敏规范

> 文档编号：L3-DES-001  
> 版本：v1.0  
> 生效日期：2026-04-18  
> 适用范围：全公司测试活动中涉及的所有测试数据

## 1. 目的

规范测试数据中敏感信息的脱敏处理方式，防止客户隐私数据泄露，确保测试活动符合《个人信息保护法》及证监会相关监管要求。

## 2. 基本原则

1. **禁止使用真实数据**：测试用例、测试报告、缺陷描述中严禁出现任何真实客户数据。
2. **最小化原则**：测试数据仅包含测试所需的最少字段，不复制完整生产数据。
3. **不可逆原则**：脱敏后的数据不可通过任何方式还原为原始数据。
4. **一致性原则**：同一客户在不同表/系统中的脱敏结果保持一致（使用确定性脱敏算法）。

## 3. 必须脱敏的字段与规则

### 3.1 客户姓名

**脱敏规则：** 保留姓氏，名字部分用 `*` 替代。

| 场景 | 原始数据 | 脱敏结果 |
|------|----------|----------|
| 两字姓名 | 张三 | 张* |
| 三字姓名 | 李小明 | 李** |
| 四字姓名 | 欧阳小华 | 欧阳** |
| 复姓两字名 | 司马相如 | 司马** |

**正则表达式实现：**
```python
import re

def desensitize_name(name: str) -> str:
    """客户姓名脱敏：保留姓氏，名字用*替代"""
    if not name or len(name) < 2:
        return name
    # 复姓列表
    compound_surnames = ['欧阳', '司马', '上官', '诸葛', '东方', '皇甫', '令狐', '尉迟']
    for surname in compound_surnames:
        if name.startswith(surname):
            return surname + '*' * (len(name) - len(surname))
    # 单姓
    return name[0] + '*' * (len(name) - 1)

# 批量脱敏正则（用于SQL/日志扫描）
# 匹配中文姓名（2~4个汉字）
NAME_PATTERN = re.compile(r'(?<=姓名[：:]\s?)[\u4e00-\u9fa5]{2,4}')
```

### 3.2 身份证号

**脱敏规则：** 保留前3位和后4位，中间用 `*` 替代。

| 原始数据 | 脱敏结果 |
|----------|----------|
| 310101199001011234 | 310***********1234 |
| 11010119850515123X | 110***********123X |

**正则表达式实现：**
```python
import re

def desensitize_id_card(id_number: str) -> str:
    """身份证号脱敏：保留前3位和后4位"""
    if not id_number:
        return id_number
    pattern = re.compile(r'^(\d{3})\d{11}([\dXx]{4})$')
    return pattern.sub(r'\1***********\2', id_number)

# 扫描检测正则（用于检测日志/文档中的身份证号）
ID_CARD_DETECT = re.compile(r'\b[1-9]\d{5}(?:19|20)\d{2}(?:0[1-9]|1[0-2])(?:0[1-9]|[12]\d|3[01])\d{3}[\dXx]\b')
```

### 3.3 手机号

**脱敏规则：** 保留前3位和后4位，中间4位用 `*` 替代。

| 原始数据 | 脱敏结果 |
|----------|----------|
| 13812345678 | 138****5678 |
| 19900001111 | 199****1111 |

**正则表达式实现：**
```python
import re

def desensitize_phone(phone: str) -> str:
    """手机号脱敏：保留前3位和后4位"""
    if not phone:
        return phone
    pattern = re.compile(r'^(\d{3})\d{4}(\d{4})$')
    return pattern.sub(r'\1****\2', phone)

# 扫描检测正则
PHONE_DETECT = re.compile(r'\b1[3-9]\d{9}\b')
```

### 3.4 银行卡号

**脱敏规则：** 保留前4位和后4位，中间用 `*` 替代。

| 原始数据 | 脱敏结果 |
|----------|----------|
| 6222021234567890123 | 6222***********0123 |
| 6217001234567890 | 6217********7890 |

**正则表达式实现：**
```python
import re

def desensitize_bank_card(card_no: str) -> str:
    """银行卡号脱敏：保留前4位和后4位"""
    if not card_no or len(card_no) < 8:
        return card_no
    return card_no[:4] + '*' * (len(card_no) - 8) + card_no[-4:]

# 扫描检测正则（16~19位数字）
BANK_CARD_DETECT = re.compile(r'\b[3-6]\d{15,18}\b')
```

### 3.5 资金账号

**脱敏规则：** 保留前2位和后3位，中间用 `*` 替代。

| 原始数据 | 脱敏结果 |
|----------|----------|
| 80012345 | 80***345 |
| 6600123456 | 66*****456 |

**正则表达式实现：**
```python
import re

def desensitize_fund_account(account: str) -> str:
    """资金账号脱敏：保留前2位和后3位"""
    if not account or len(account) < 5:
        return account
    return account[:2] + '*' * (len(account) - 5) + account[-3:]

# 扫描检测正则（根据公司资金账号规则调整）
FUND_ACCOUNT_DETECT = re.compile(r'\b[1-9]\d{6,9}\b')
```

## 4. 造数规则

### 4.1 测试账号段分配

| 数据类型 | 测试专用段 | 说明 |
|----------|------------|------|
| 资金账号 | 90000001 ~ 90999999 | 以90开头的账号段专供测试 |
| 客户编号 | T0000001 ~ T9999999 | 以T开头标识测试客户 |
| 手机号 | 13000000001 ~ 13000009999 | 130段前缀专供测试 |
| 身份证号 | 使用校验位合法的虚构号码 | 地区码使用999开头 |
| 银行卡号 | 6222090000000001 ~ 6222099999999999 | 622209段专供测试 |
| 股东账号 | A999000001 ~ A999999999 | A999前缀专供测试 |

### 4.2 造数工具要求

1. 测试数据生成工具必须使用上述测试专用段
2. 造数脚本须纳入版本管理，存放于 `test-tools/data-generator/` 目录
3. 造数脚本执行前须校验目标环境（禁止在生产环境执行）

```python
# 造数脚本环境校验示例
import os
import sys

ALLOWED_ENVS = ['dev', 'sit', 'uat']

def check_environment():
    """校验当前环境，禁止在生产环境执行造数"""
    current_env = os.getenv('APP_ENV', '').lower()
    if current_env not in ALLOWED_ENVS:
        print(f"[FATAL] 当前环境 '{current_env}' 不在允许列表 {ALLOWED_ENVS} 中，禁止执行造数！")
        sys.exit(1)
    print(f"[INFO] 当前环境: {current_env}，允许执行造数。")
```

## 5. 违规后果

### 5.1 违规行为定义

| 等级 | 违规行为 | 示例 |
|------|----------|------|
| 一级违规（严重） | 在测试用例/报告中使用真实客户数据 | 用例中出现真实身份证号 |
| 一级违规（严重） | 将生产数据未脱敏直接导入测试环境 | 全量复制生产客户表到SIT环境 |
| 二级违规（较重） | 脱敏不完整，部分字段遗漏 | 姓名脱敏但手机号未脱敏 |
| 二级违规（较重） | 使用非测试专用段的账号进行测试 | 使用真实资金账号段造数 |
| 三级违规（一般） | 造数脚本未纳入版本管理 | 本地脚本未提交Git |
| 三级违规（一般） | 测试数据未按期清理 | 上轮测试数据残留超过5个工作日 |

### 5.2 处罚措施

| 违规等级 | 处罚措施 |
|----------|----------|
| 一级违规 | 通报批评 + 安全培训 + 绩效扣分；情节严重者按公司信息安全管理办法处理 |
| 二级违规 | 书面警告 + 限期整改（2个工作日内） |
| 三级违规 | 口头提醒 + 限期整改（5个工作日内） |

### 5.3 自动化检测机制

```python
# 敏感数据扫描器（集成到CI/CD流水线）
class SensitiveDataScanner:
    """扫描测试用例文档和代码中的敏感数据"""
    
    PATTERNS = {
        '身份证号': re.compile(r'[1-9]\d{5}(?:19|20)\d{2}(?:0[1-9]|1[0-2])(?:0[1-9]|[12]\d|3[01])\d{3}[\dXx]'),
        '手机号': re.compile(r'1[3-9]\d{9}'),
        '银行卡号': re.compile(r'[3-6]\d{15,18}'),
    }
    
    # 测试专用段白名单
    WHITELIST = {
        '手机号': re.compile(r'130000\d{5}'),
        '银行卡号': re.compile(r'622209\d{10,13}'),
    }
    
    def scan_file(self, filepath: str) -> list:
        violations = []
        with open(filepath, 'r', encoding='utf-8') as f:
            for line_no, line in enumerate(f, 1):
                for field_type, pattern in self.PATTERNS.items():
                    for match in pattern.finditer(line):
                        value = match.group()
                        # 检查是否在白名单中
                        if field_type in self.WHITELIST and self.WHITELIST[field_type].match(value):
                            continue
                        violations.append({
                            'file': filepath,
                            'line': line_no,
                            'type': field_type,
                            'value': value[:4] + '****',  # 报告中也要脱敏
                        })
        return violations
```

## 6. 附则

- 本规范适用于所有测试环境（DEV/SIT/UAT/预生产），预生产环境执行最严格标准。
- 新入职测试人员须在入职培训中学习本规范并签署知悉确认书。
- 本规范每半年审查一次，根据监管要求和实际执行情况修订。
