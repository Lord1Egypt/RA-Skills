# 猫眼电影个性化推荐系统

> 每周从猫眼电影抓取正在热映 & 即将上映的全部电影，结合你的观影口味智能排序推荐，推送飞书周报。

## 功能介绍

### 这是什么？

一个自动化电影推荐系统。每个周末前自动扫描中国大陆院线新片，基于你填写的观影偏好档案，让 AI 帮你挑片——不用逐部翻简介，也不用被猫眼评分带偏。

### 核心能力

| 能力 | 说明 |
|------|------|
| 🎬 全量采集 | 抓取猫眼「正在热映」+「即将上映」全部电影，每部获取详细信息（类型/导演/主演/时长/制片国家） |
| 🧠 智能匹配 | LLM 读取你的偏好档案，对每部电影打分排序，非关键词匹配而是「理解你为什么喜欢某类电影」 |
| 📊 分级推荐 | 输出三级推荐：🎯 强烈推荐 / 👍 值得关注 / 👀 可观望，快速决策 |
| 🔔 自动提醒 | 如果你标记「想看」的电影上映了，周报开头会醒目提醒 |
| 📱 飞书推送 | 周报直接推送到飞书会话，手机上看也很方便 |

### 与猫眼 App 的区别

- 猫眼推荐依赖「大众评分 + 标签匹配」，你是被动的
- 本系统基于你**自己写的偏好档案**，AI 理解你的口味后做推荐
- 不用被高分烂片骗，也不用错过小众但对你胃口的片

## 快速开始

### 第一步：复制偏好档案模板

```bash
cd ~/.openclaw/workspace/skills/movie-rec-personal-cn/
cp profile-template.json profile.json
```

### 第二步：填写你的观影偏好

用任意编辑器打开 `profile.json`，按模板注释填写。**最重要的两个部分：**

**① `liked_movies`**——添加 5-10 部你真心喜欢的电影。每部写清楚：

- 片名、类型、主演、导演
- **你为什么喜欢**（这比评分重要得多）
- **特征标签**（如：硬核武侠、高概念科幻、暴力美学、游戏改）

示例：

```json
{
  "nm": "镖人：风起大漠",
  "cat": "动作,武侠,古装",
  "reason": "硬派武侠漫画改编，吴京监制+主演，袁和平执导，全明星阵容",
  "traits": ["动作驱动叙事", "漫改", "硬核武侠", "国风世界观", "全明星阵容"]
}
```

**② `core_traits`**——描述你的口味骨架：

```json
{
  "genre_affinity": {
    "动作": "核心底线——不是纯打斗，而是动作驱动叙事",
    "科幻": "偏好硬科幻/高概念设定，尤其IP改编"
  },
  "ip_affinity": "极高——偏好已有世界观基础的作品",
  "style_preferences": [
    "高概念/强世界观驱动",
    "类型混搭好于单一类型"
  ],
  "weak_signals": [
    "纯爱情片、文艺片几乎不看",
    "恐怖片 jump scare 类不看"
  ]
}
```

### 第三步：设置定时推送

在 OpenClaw 中创建 cron job（参见下方「Cron 配置」），或手动触发：

```
帮我做一期猫眼电影周报
```

## 偏好档案填写指南

### liked_movies（已喜欢的电影）

> **为什么重要：** 这是 AI 学习你口味的唯一样本数据。样本越多，推荐越准。

| 字段 | 必填 | 说明 |
|------|------|------|
| `nm` | ✅ | 电影名称 |
| `cat` | ✅ | 类型（用逗号分隔，如：动作,科幻,惊悚） |
| `star` | 建议 | 主演名单 |
| `dir` | 建议 | 导演 |
| `src` | 建议 | 制片国家/地区 |
| `rt` | 选填 | 上映日期 |
| `sc` | 选填 | 猫眼评分（可为 null） |
| `reason` | ✅ | **最重要**——用一句话说明你为什么喜欢这部电影 |
| `traits` | ✅ | **最关键**——3-5 个特征标签，描述这部片吸引你的特质 |

**traits 标签怎么选？** 想清楚「这片哪一点最打动我」：

- 动作驱动叙事 / 暴力美学 / 格斗竞技
- 硬科幻 / 高概念 / 太空生存
- 漫改 / 游戏改 / 小说改 / 原创
- 强世界观 / 架空设定 / 国风
- 类型混搭 / 黑色幽默 / 反转多

**常见错误：** traits 写成「好看」「精彩」「推荐」之类主观评价 → 没用

### want_to_watch（想看但还没看的电影）

> **为什么重要：** 这些电影出现在「正在热映」时，周报会醒目提醒你。

不需要写 reason 和 traits，标上片名和基本信息即可。可以持续更新。

### core_traits（核心偏好）

| 字段 | 说明 |
|------|------|
| `genre_affinity` | 对每种类型的偏好描述。用自然语言，AI 能理解「不是纯打斗，而是动作驱动叙事」这样的描述 |
| `ip_affinity` | 对 IP 改编的态度。极高/中等/无所谓/排斥 |
| `style_preferences` | 你喜欢的风格特征列表 |
| `weak_signals` | 你明确不看的类型。AI 会据此降级或排除 |

### recommendation_rules（推荐规则）

| 字段 | 说明 |
|------|------|
| `priority_logic` | 用自然语言描述排序逻辑。默认值适用于大多数动作/科幻向影迷 |
| `boost_keywords` | 出现这些关键词的电影会加分 |
| `downgrade_keywords` | 出现这些关键词的电影会降级 |
| `exclude_keywords` | 出现这些关键词的电影直接排除 |

## Cron 配置

### 推荐设置

```json
{
  "schedule": "0 18 * * 5",
  "sessionTarget": "isolated",
  "payload": {
    "kind": "agentTurn",
    "message": "执行猫眼电影周报：先运行 python3 skills/movie-rec-personal-cn/scripts/maoyan_movies.py 采集最新电影数据，保存到 /tmp/movies_weekly.json。然后读取 profile.json 和 /tmp/movies_weekly.json，基于 core_traits 和 recommendation_rules 为每部未看过的电影打分排序。输出结构化周报（🎯 强烈推荐 / 👍 值得关注 / 👀 可观望），推送到飞书。如果 want_to_watch 里有电影撞上正在热映列表，要在开头特别标注「你标记想看的电影已上映」。",
    "timeoutSeconds": 120
  },
  "delivery": {
    "mode": "announce",
    "channel": "feishu",
    "to": "你的飞书 open_chat_id"
  }
}
```

### 可选时段

| 时段 | 适合 | Cron |
|------|------|------|
| 周五 18:00 | 周末观影指南（推荐） | `0 18 * * 5` |
| 周一 09:00 | 新片首周上映提醒 | `0 9 * * 1` |
| 周四 20:00 | 提前规划周末 | `0 20 * * 4` |

## 常见问题

### Q: 脚本运行需要多久？

大约 30-120 秒，取决于当前有多少电影在上映。脚本内置 0.3-0.5s 随机延迟以避免触发猫眼反爬。

### Q: 需要安装什么依赖？

不需要。脚本只用 Python 标准库（`urllib` + `json`）。Python 3.6+ 即可运行。

### Q: 猫眼 API 会不会挂？

猫眼移动端 API 稳定运行多年，但偶尔会抽风。脚本内置了重试机制（最多 2 次），详情失败的个别电影会跳过但不会影响整体输出。

### Q: liked_movies 最少要填几部？

建议至少 5 部。如果刚开始用，填 2-3 部也能跑，但推荐精度明显下降。

### Q: 能不能同时用于家庭/情侣？

可以。修改 `core_traits` 的 `genre_affinity`，融合两个人的口味。例如：

```json
"genre_affinity": {
  "动作": "我偏好动作驱动叙事，伴侣偏好爆米花爽片——取交集：要有动作元素",
  "动画": "我俩都愿意看，尤其合家欢类型",
  "恐怖": "伴侣完全不看，排除"
}
```

### Q: 为什么不用豆瓣 API？

猫眼 API 更适合「中国大陆院线」场景：数据实时、接口稳定、不需要处理反爬。豆瓣数据更全面但抓取成本高、接口频繁变动。

### Q: profile.json 可以手动修改吗？

可以，而且随时可以。每次运行都会重新读取 profile.json，所以修改立即生效。

### Q: 怎么查看原始抓取数据？

```bash
python3 scripts/maoyan_movies.py > movies.json
```

输出全量 JSON，包含所有字段。

### Q: 「想看」电影上映了但没提醒？

检查 `want_to_watch` 里的电影名称和猫眼 API 返回的片名是否一致（尤其英文/数字片名可能略有差异）。
