# 03 - 季度优先级取舍

## Input

> 下季度只能做 3 件事。候选项有：AI 回答质量优化、企业审计日志、简化 onboarding、团队空间共享、管理员账单控制、历史搜索、增长裂变。CEO 想要增长，销售想要企业能力，客服一直反馈新用户 onboarding 很混乱。帮我排优先级，并说清楚哪些应该先不做。

## What good output looks like

好的 `pm-workbench` 输出应该：

- 先定义季度目标，再排序
- 真正限制在 3 个优先项
- 说明哪些在 below the line
- 讲清楚为什么不是所有利益相关方的诉求都能同时满足

## Example target output

# Prioritization Stack

## 1. 季度目标

下季度应该优先修复激活和核心信任问题，同时保留最低限度的企业可信度。不要在核心体验仍然漏水时先放大增长。

## 2. P0 / 必做

1. **简化 onboarding**  
   这是当前最直接的激活和客服压力来源。先让新用户更快进入有效使用。

2. **AI 回答质量优化**  
   这是产品核心承诺。如果回答质量不稳，增长只会放大失望。

3. **企业审计日志**  
   这是保留销售推进能力的最低企业可信度，不等于全面企业化。

## 3. P1 / 有价值但不进前三

- **历史搜索：** 对留存和深度使用有价值，但不如 onboarding 和回答质量紧急。
- **团队空间共享：** 对协作深度有帮助，但更适合在核心体验稳定后推进。

## 4. P2 / 本季度不做

- **增长裂变：** 当前不适合。产品核心体验还不够稳，裂变可能放大漏斗浪费。
- **管理员账单控制：** 对成熟企业能力重要，但不是当前最强商业瓶颈。

## 5. 主要取舍

我们不是忽视增长，而是选择先提高增长质量。这个季度的稀缺资源应该花在激活、核心信任和最低企业可信度上，而不是同时铺开增长、协作、企业、搜索四条线。

## 6. 给管理层的说法

建议把季度主题定为："先补激活和信任，再放大增长。"  
需要管理层确认：本季度不追求所有方向都有进展，而是接受一部分需求明确 below the line。
