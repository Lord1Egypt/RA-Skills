# 阶段5：质检整合与输出

## ⚡ 执行指引（Agent必读）

### 质检执行策略
1. **逐章扫描**：依次检查每章内容，不要跳跃
2. **对照招标原文**：每检查一项就回头对照招标文件原文，不要凭记忆
3. **优先级**：先修🔴错误（必须修复）→ 再修🟡警告（建议修复）→ 最后🟢建议（可选的优化）
4. **无法目测的格式项**（字体、字号）：标记为"请用户确认格式合规性"，因为AI无法精确判断字体渲染

### 覆盖度检查方法
1. 列出 `scoringCriteria` 中所有评分维度 → 在大纲中逐一找到对应章节 → 标注状态
2. 列出 `technicalParameters` 中所有技术参数 → 在标书正文中逐一搜索 → 标注状态
3. 缺失项标记位置："第X章 缺少对[参数名]的响应"

### 质检报告输出格式
必须包含以下三部分，缺一不可：
```
## 质检报告
### 总体评分：[通过/有条件通过/不通过]
### 问题清单（按严重级别排序）
### 修改建议
```

### 文档整合顺序
严格按照 `templates/bid-structure.json` 中 `chapters` 数组的顺序排列，不可调换。

## 目标
对所有章节进行全面质量检查，确保覆盖度、合规性、一致性和专业性，最终整合为完整的应标书文档。

## 输入
- 各章节已完成内容
- 阶段1的招标解析报告
- 阶段3的确认大纲

## 第一部分：全面质量检查

### 检查1：评分项覆盖度（错误级）

逐项核对评分标准与标书内容：

| 评分维度 | 满分 | 对应章节 | 是否覆盖 | 备注 |
|---------|------|---------|---------|------|
| [技术方案] | 40 | 第3章 | ✅ | 完整覆盖 |
| [施工组织] | 20 | 第4章 | ❌ 缺少X.X | 需补充 |

**判定标准：**
- 必须确保100%的评分项都有对应章节
- 每个评分项下面的细则点也要逐条核对

### 检查2：技术要求逐条响应（错误级）

| 技术要求编号 | 要求内容 | 响应位置 | 响应方式 | 状态 |
|-------------|---------|---------|---------|------|
| 技术参数1 | [要求] | 3.2节 | 逐条响应表 | ✅ |
| 技术参数2 | [要求] | — | — | ❌ 遗漏 |

**判定标准：**
- 招标文件"技术参数/技术要求"中每一条都必须有回应
- 响应方式可以是逐条响应表或正文描述
- 偏离须明确标注并说明理由

### 检查3：格式合规性（警告级）

- [ ] 字体是否符合招标文件要求（默认宋体）
- [ ] 字号是否符合规范（标题二号/三号，正文小四/四号）
- [ ] 行距是否统一（推荐1.5倍行距）
- [ ] 页边距是否符合要求
- [ ] 页码是否连续
- [ ] 目录是否与正文标题一致
- [ ] 标题编号是否连续无跳跃
- [ ] 表格格式是否统一（三线表）
- [ ] 图表编号是否连续

### 检查4：法律合规性（错误级）

- [ ] 投标函是否包含：投标意愿表达、投标有效期、保证金承诺、法律责任承担
- [ ] 法定代表人身份证明是否完整
- [ ] 授权委托书是否包含：授权范围、授权期限、代理人信息
- [ ] 诚信承诺函内容是否完整
- [ ] 所有需签字/盖章处是否已标注

### 检查5：内容质量（警告级）

- [ ] 是否存在空泛表述（"高度重视"、"全力以赴"等无实质内容词语）
- [ ] 技术方案是否有具体技术参数
- [ ] 是否每项承诺都可衡量、可验证
- [ ] 是否存在前后矛盾的内容
- [ ] 是否存在语病或错别字
- [ ] 各章节风格是否统一

### 检查6：评分导向一致性（错误级）

- [ ] 高分值章节是否获得了足够的篇幅
- [ ] 章节内容是否紧扣评分细则中的得分点
- [ ] 是否每个章节都明确标注了对应评分项

## 第二部分：问题处理

### 问题分类

| 级别 | 处理方式 | 示例 |
|------|---------|------|
| 🔴 **错误** | 必须修复，否则标书无效 | 遗漏技术要求、法律文件格式错误 |
| 🟡 **警告** | 建议修复，影响得分或专业性 | 空泛表述、格式不统一 |
| 🟢 **建议** | 可优化项，提升质量 | 增加案例、补充数据 |

### 处理流程
1. 列出所有问题清单（按严重级别排序）
2. 优先修复所有🔴错误级问题
3. 逐一修复🟡警告级问题
4. 选择性采纳🟢建议级优化

## 第三部分：文档整合

将质检通过的章节按以下顺序整合为完整文档：

1. **封面**：项目名称、投标人名称、日期
2. **目录**：自动生成，与正文标题一致
3. **投标函**：标准格式 + 项目特定信息
4. **法定代表人身份证明/授权委托书**
5. **资质证明文件**：逐项排列
6. **技术响应文件**：含技术参数响应表
7. **技术方案**：核心内容
8. **施工组织/实施方案**
9. **项目管理机构**
10. **质量保证体系**
11. **进度计划及保障**
12. **售后/培训方案**
13. **报价文件**
14. **诚信承诺函**
15. **其他材料**

## 第四部分：输出

### 质检报告
向用户输出一份完整的质检报告：

```
## 标书质检报告

### 总体评分
- 评分项覆盖率：[X]%
- 技术要求响应率：[X]%
- 格式合规性：[通过/不通过]
- 法律合规性：[通过/不通过]

### 问题清单
1. 🔴 [严重问题描述] → 建议解决方案
2. 🟡 [警告问题描述] → 建议解决方案
3. 🟢 [优化建议描述]

### 修改建议
[具体修改建议列表]
```

### 最终文档
输出完整标书内容（建议使用Markdown格式，便于后续转换为Word），并标注：
- 总字数统计
- 章节数量
- 技术参数响应条目数
- 图表数量（如有）

## 最终确认清单

交付前最后确认：
- [ ] 所有🔴错误级问题已修复
- [ ] 用户已确认质检报告
- [ ] 所有章节已整合到同一文档
- [ ] 文档格式可转换为Word
- [ ] 投标文件份数和格式符合招标要求
