# 📝 说服辅助之书 | Persuasion Support Template

> 当你需要说服一个人时，用此模板建立结构化的说服方案

---

## 第一步：揣情 —— 知己知彼

### 对方画像 | Target Profile

| 维度 | 填写 |
|:---|:---|
| **对方是谁**（角色/职位） | |
| **他最在乎什么**（核心利益） | |
| **他害怕什么**（核心恐惧） | |
| **他的决策权有多大**（最终决定者？影响者？） | |
| **他和你的关系**（上下级？平级？外部？） | |

### 你的定位 | Your Position

| 维度 | 填写 |
|:---|:---|
| **你想要什么**（具体可量化的结果） | |
| **你的底线是什么**（绝对不能接受的结果） | |
| **你的替代方案是什么**（如果失败，Plan B） | |
| **你给对方的价值是什么**（共赢点） | |

---

## 第二步：内揵 —— 建立联结

### 情感联结策略

1. **共同利益点**：你们有什么共同的目标或利益？
   > 例如："我们都希望这个项目成功。"

2. **情感共鸣点**：对方最近经历过什么，你能理解他的处境？
   > 例如："我知道你最近在推动的 X 项目压力很大……"

3. **信任锚点**：有没有共同信任的人或共同的经历可以引用？
   > 例如："上次和王总一起讨论的 Y 方案……"

---

## 第三步：捭阖设计 —— 攻守节奏

### 说服节奏表

| 阶段 | 动作 | 话术要点 | 时间分配 |
|:---|:---|:---|:---|
| **开场 (阖)** | 表达理解，建立信任 | "我理解你的顾虑……" | 20% |
| **过渡 (捭)** | 展示方案核心价值 | "这个方案可以帮你解决……" | 30% |
| **互动 (阖)** | 提问，倾听对方反馈 | "你怎么看这个方向？" | 30% |
| **收尾 (捭)** | 总结利益，给出行动建议 | "如果我们下周开始……" | 20% |

---

## 第四步：抵巇 —— 预判反对意见

### 反对意见预判表

| 对方可能说什么 | 我的回应 | 背后的真实需求是什么 |
|:---|:---|:---|
| "太贵了" | 重新框定价值："如果算上节省的时间……" | 预算有限？ROI 不明确？ |
| "时机不对" | "正是因为现在的时机，才需要……" | 优先级不够？有其他隐忧？ |
| "再考虑一下" | "你具体想考虑哪个方面？" | 决策权不够？还缺信息？ |
| "别人也在做" | "他们做的是 X，我们做的是 Y" | 担心风险？需要差异化？ |
| （自定义） | |

---

## 第五步：飞箝 —— 锁定行动

### 行动推动策略

1. **飞**（认可）：具体地认可对方的一个优点或贡献
   > "你的团队在效率上做得非常出色，这正是这个方案能发挥价值的土壤。"

2. **箝**（约束）：给出一个清晰、有时间限制、低门槛的下一步
   > "我们可以先用两周时间做一个 MVP，如果数据不符合预期，你可以随时叫停。"

3. **钩箝调和**：如果对方还在犹豫，给出一个双赢的中间方案
   > "我们不用全量铺开——先选一个团队试点，效果好再推广。"

---

## 第六步：忤合 —— 退出信号

### 何时停止说服

- [ ] 对方三次以上使用同一理由拒绝 → 那不是理由，是决定
- [ ] 对方开始回避沟通（不回消息、取消会议）→ 关闭沟通通道
- [ ] 说服成本超过收益 → 计算机会成本
- [ ] 你的替代方案突然变得更好了 → 重新评估忤合

> **鬼谷子提醒**：不是所有人、所有时机都适合说服。有时"忤"（退出）比"合"（坚持）更明智。

---

## 快速检查清单 | Quick Checklist

- [ ] 我了解对方的核心利益和恐惧？
- [ ] 我的方案解决了对方的什么问题？
- [ ] 我对反对意见有预案？
- [ ] 我的"飞箝"（认可+约束）是否自然？
- [ ] 我知道什么时候该停下？
