# $ralph — 持久完成工作流

## 触发条件
- 任务明确，需要持续执行直到完成
- 用户说"做完它"、"持续执行"
- 适合单 Agent 可以独立完成的任务

## 执行流程

### Phase 1: 接收任务
```
Agent: Coder (GPT-5.3-codex) 或 Executor (Mimo)
输入: 任务描述 + 约束条件
输出: 执行计划
```

### Phase 2: 持续执行
```
循环执行:
1. 执行当前步骤
2. 验证结果
3. 如果有问题 → 自动修复
4. 如果完成 → 输出结果
5. 如果需要帮助 → 求助用户
```

### Phase 3: 完成验证
```
Agent: Reviewer (GPT-5.5)
输入: 执行结果
输出: 验证报告
- 功能完整性
- 代码质量
- 测试覆盖
```

## 使用方式

```bash
# 持久执行
"用 $ralph 完成这个 REST API"

# 指定 Agent
"用 $ralph:coder 实现用户模块"

# 带验证
"用 $ralph + $code-review 完成并审查"
```

## 自动修复策略
1. 编译错误 → 自动修复语法
2. 测试失败 → 分析原因并修复
3. 依赖缺失 → 自动安装
4. 逻辑错误 → 重新分析并修复
