## Description: <br>
Comprehensive code documentation skill for maintaining inline file and function headers, subsystem documentation, architecture documentation, API references, and documentation audits after meaningful code changes. <br>

This skill is ready for commercial/non-commercial use. <br>

## Publisher: <br>
[encryptshawn](https://clawhub.ai/user/encryptshawn) <br>

### License/Terms of Use: <br>
MIT-0 <br>


## Use Case: <br>
Developers and engineering teams use this skill to create and update accurate codebase documentation, including source-file headers, function comments, subsystem guides, architecture docs, API references, and documentation audit findings. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Broad documentation and source-comment edits can introduce inaccurate descriptions or remove useful context. <br>
Mitigation: Use the skill on a branch and review generated diffs carefully before committing. <br>
Risk: Documentation generated from stale or wrong code can mislead future maintainers. <br>
Mitigation: Confirm the active branch and read current source and existing docs before updating documentation. <br>
