## Description: <br>
Helps maintainers and users plan, produce, and verify practical mobile responsive layout fixes through tailored guidance, checklists, analysis, or code-oriented support. <br>

This skill is ready for commercial/non-commercial use. <br>

## Use Case: <br>
Developers, maintainers, and contributors use this skill to turn mobile responsive layout requests into an actionable workflow with expected outputs, review criteria, and verification notes. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Broad activation terms may trigger the skill on general mobile or responsive wording. <br>
Mitigation: Prefer explicit invocation for targeted use, and narrow triggers if maintaining the skill. <br>
Risk: Generated responsive-layout guidance or code may not match the target application's design system or breakpoints. <br>
Mitigation: Review proposed changes against project constraints and verify them on representative mobile and desktop viewports before merging. <br>
