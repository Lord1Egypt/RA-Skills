## Description: <br>
产品经理专用的 PRD(产品需求文档)生成、优化与扩展工具。用于从自然语言描述创建结构化 PRD、对现有 PRD 进行澄清式审阅以补全模糊点、或向已有 PRD 追加新功能。 <br>

This skill is ready for commercial/non-commercial use. <br>

## Publisher: <br>
[xu4wang](https://clawhub.ai/user/xu4wang) <br>

### License/Terms of Use: <br>
MIT-0 <br>


## Use Case: <br>
Product managers, founders, and product teams use this skill to create structured PRD Markdown files, clarify missing requirements in existing PRDs, and extend PRDs with new features while preserving terminology, numbering, and priority conventions. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Clarify and extend modes can save edits directly to an existing PRD Markdown file. <br>
Mitigation: Confirm the target path before use and keep a backup or versioned copy for important PRDs. <br>
Risk: Generated PRDs may include assumptions when the initial feature description is incomplete. <br>
Mitigation: Review generated assumptions and run clarify mode to resolve high-impact missing requirements before relying on the PRD. <br>
