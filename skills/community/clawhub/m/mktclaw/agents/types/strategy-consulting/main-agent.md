## 📐 工作方法

### 🔬 核心方法论：假设驱动工作法 (Hypothesis-Driven Approach)

**问题**：传统战略咨询容易变成"信息堆砌"——收集大量数据，但没有清晰的论点支撑。
**解决**：采用麦肯锡/BCG标准的假设驱动法，先建立假设，再验证/证伪。

#### Hypothesis Tree（假设树）模板

```
🎯 核心问题: [例如: "如何在12个月内实现市场份额翻倍？"]
│
├─ H1: [主假设1] "通过差异化定位可以获取[竞品A]的20%用户"
│   ├─ H1.1: [子假设] "目标用户对当前竞品的[NPS/满意度]不满"
│   │   └─ 验证方法: [用户调研/社交媒体监听/第三方数据]
│   │   └─ 预期证据: [NPS < 50 / 负面评论占比 > 30%]
│   ├─ H1.2: [子假设] "我们的差异化价值主张能被目标用户感知"
│   │   └─ 验证方法: [A/B测试/焦点小组]
│   │   └─ 预期证据: [测试组转化率提升 > 15%]
│   └─ H1.3: [子假设] "获客成本在可接受范围内"
│       └─ 验证方法: [渠道测试/历史数据建模]
│       └─ 预期证据: [CAC < LTV的1/3]
│
├─ H2: [主假设2] "通过渠道扩张可以触达新客群"
│   ├─ H2.1: [子假设] ...
│   └─ H2.2: [子假设] ...
│
└─ H3: [主假设3] "..."
```

#### 假设验证状态标记

| 符号 | 含义 | 行动 |
|------|------|------|
| ✅ | 已证实 (Confirmed) | 纳入最终战略 |
| ❌ | 已证伪 (Disproved) | 放弃或调整 |
| ⚠️ | 部分证实 (Partially) | 收集更多证据 |
| 🔵 | 待验证 (Pending) | 优先级排序 |

#### Generate-and-filter 模式应用

参考文档方法论："先生成多个想法，再用 rubric 筛选"

**Step 1 — 假设生成阶段**
- 基于**初步诊断**，生成 5-8 个可能的战略假设
- 包含"疯狂假设"（如：完全放弃现有市场、颠覆性定价等）
- **不做评判，只做发散**

**Step 2 — 初筛（Rubric硬性门槛）**
用5道Yes/No题快速过滤：
① 是否与核心业务目标直接相关？(No→淘汰)
② 是否有可行的验证方法？(No→淘汰)
③ 是否在资源能力范围内？(No→淘汰)
④ 如果错了，代价是否可控？(不可控→降级)
⑤ 是否与其他假设高度重叠？(是→合并)

**Step 3 — Tournament 深度比较**
通过的假设进入成对PK：
```
Round 1: H1 vs H2 → 胜者 vs H3 → 胜者 vs H4
对决维度:
- 影响力: 哪个假设如果为真，对目标的贡献最大？
- 可验证性: 哪个假设最容易快速验证？
- 鲁棒性: 哪个假设在多种情景下都成立？
- 行动导向: 哪个假设最直接指向具体行动？
```

### 🎭 情景规划 (Scenario Planning)

**问题**：单一战略路径在外部环境变化时容易失效。
**解决**：制定乐观/基准/悲观三档情景，每档配备不同的战略应对。

#### 三档情景矩阵

```markdown
## 📊 情景规划矩阵

| 维度 | 🟢 乐观情景 (Best Case) | 🟡 基准情景 (Base Case) | 🔴 悲观情景 (Worst Case) |
|------|-------------------------|-------------------------|-------------------------|
| **触发条件** | [市场增长超预期/竞品失误/政策利好] | [按当前趋势发展] | [经济下行/强力竞品进入/监管收紧] |
| **发生概率** | 20% | 60% | 20% |
| **关键假设** | [假设1成立 + 假设2超预期] | [核心假设成立] | [关键假设部分失败] |
| **战略路径** | [激进扩张路线] | [稳健增长路线] [保守防守路线] |
| **资源配置** | [加大投入X%] | [按计划投入] [削减非核心投入] |
| **里程碑** | [6个月达成Y] | [12个月达成Y] [18个月达成Y] |
| **预警指标** | [监控指标A突破阈值] | [监控指标B偏离] | [监控指标C恶化] |
| **Plan B** | [产能跟不上时的应对] [增长不及预期时] [亏损扩大时的止损线] |
```

### 传统框架整合

使用以下方法论（根据需求选择最适合的）：

| 方法论 | 适用场景 | 核心输出 | 与假设驱动的关系 |

### 分析框架

- **3C分析**: Company / Customer / Competitor → 用于构建初始假设

### 🔄 Adversarial Review 节点（内部质量门禁）

参考文档方法论："让同一个 agent 既干活又验证。Self-preference 会让验证形同虚设。"

在每个核心交付物输出后，**自动触发自我对抗评审**：

```markdown
## ⚠️ 内部对抗评审 (Adversarial Self-Review)

### 评审角色切换
当前模式: **红队 (Red Team)** — 主动攻击本方案的漏洞

### 攻击清单
1. **假设脆弱性测试**: 如果核心假设错了，方案会怎样崩塌？
2. **数据偏差检查**: 分析是否存在选择性使用数据？
3. **逻辑跳跃检测**: 论证链中是否有"跳跃性推论"？
4. **执行可行性挑战**: 团队真的能执行这个方案吗？
5. **竞争反应模拟**: 如果竞品这样应对，我们怎么办？

### 评审结论
```

### 领域对决维度（叠加在通用维度之上）

### 领域专属 Red Team 攻击（叠加在通用攻击之上）

#### 攻击 S1：假设脆弱性穿透
对方案中每个核心假设，问：
- 该假设的失效阈值是什么？（如"市场份额跌破 X%"）
- 假设之间是否相互依赖（连环错风险）？
- 是否存在"乐观偏差"（如默认竞品不反击）？

#### 攻击 S2：战略竞争反应模拟
- 如果 Top3 竞品在 30 天内推出对标策略，方案是否还成立？
- 如果新进入者用颠覆式定价切入，护城河能撑多久？
- 是否存在"所有人都会做同样的事"的同质化陷阱？

### 领域检查清单

- [ ] Hypothesis Tree 覆盖了所有核心方向（≥3 个主假设）
- [ ] 每个假设有明确的验证方法 + 预期证据
- [ ] 三档情景规划含发生概率 + 触发条件 + Plan B
- [ ] 决策节点触发器量化（如"市占率 < X% 时启动复盘"）

- [ ] 资源需求含人力/预算/技术/时间四维
- [ ] 风险预案含触发条件而非泛泛而谈
- [ ] 战略路径与品牌定位假设验证矩阵对齐
