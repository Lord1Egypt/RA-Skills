# Content Rules — PPT Generator v3

内容密度和质量约束。Developer 子 agent 生成每页 slide 时遵循。

---

## 内容密度模型

废除旧版"每页≤50字"硬限制。根据页面角色选择合适的密度层级：

| 密度 | 代号 | 字数范围 | 适用布局 | 特征 |
|------|------|---------|---------|------|
| 精简型 | `minimal` | 10-30字 | cover, chapter-cover, ending | 大标题 + 1句副标题，极简导航 |
| 标准型 | `standard` | 30-200字 | toc, ca-architecture, ca-timeline, ca-table | 标题 + 结构化内容，适度展开 |
| 展开型 | `expanded` | 100-500字 | ca-bullets, ca-bullets-2col, ca-cards-3, ca-cards-4, ca-image-left, ca-image-right | 标题 + 段落 + 要点 + 数据说明 |

**原则**：结构页做减法，内容页做加法。不存在"每页均匀分配"的中间地带。

---

## 内容原则

1. **一页一主题**：每页只讲一件事
2. **要点优先**：用 bullet points，不用完整段落
3. **数据可视化**：能画图的不用数字列表
4. **中文为主**：英文仅用于副标题/标签，不超过 10 词
5. **不要废话**：去掉"的"、"了"、"我们"等无信息词

## 布局内容模板

### cover — 精简型 (10-30字)
```
标题: [主题名称]
副标题: [汇报场合/日期]
```

### toc — 标准型 (30-80字)
```
章节标题列表，带编号:
01  背景与痛点
02  产品介绍
03  产品功能
04  应用场景及价值
```

### chapter-cover — 精简型 (10-30字)
```
编号: [PART ONE / 01]
标题: [章节名]
简述: [一句话概括，可选]
```

### ca-bullets — 展开型 (100-500字)
```
标题: [本页主题]
要点:
  • [要点1，可多句展开]
  • [要点2，可多句展开]
  • [要点3，可多句展开]
  • [要点4，可多句展开]
  • [要点5，可多句展开]
  • [要点6，可多句展开]
可选: 底部数据亮点（1个数字 + 标签）
```

### ca-bullets-2col — 展开型 (100-500字)
```
每栏:
  图标: [material icon name]
  标题: [方案名]
  标签: [3个特征标签]
  优势: [3条优势描述]
  场景: [适用场景描述]
可选: 一栏标记为 featured（带推荐标签）
```

### ca-cards-3 / ca-cards-4 — 展开型 (100-500字)
```
左栏: 要点列表（3-5条）
右栏: 图表说明 或 数据可视化
```

### ca-image-left / ca-image-right — 展开型 (100-500字)
```
标题: [图表主题]
副标题: [说明]
数据: [json格式，按design-system.md §图表数据格式]
```

### ca-architecture — 展开型 (100-500字)
```
标题: [图表主题]
副标题: [说明]
数据: [json格式]
```

### ending — 精简型 (10-30字)
```
致谢: [感谢语]
联系方式: [可选]
```
