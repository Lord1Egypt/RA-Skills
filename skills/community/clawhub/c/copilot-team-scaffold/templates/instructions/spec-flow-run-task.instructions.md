## Spec-Flow 任务执行规则

当执行 tasks.md 中的任务时，遵循以下步骤：

1. **读取任务清单**：读取 `.spec-flow/active/{{PROJECT_SLUG}}/tasks.md`
2. **定位任务**：找到目标任务，提取描述、依赖、状态、复杂度和指派 Agent
3. **依赖检查**：如果依赖任务未完成（非 ✅ Done），停止并报告哪些任务阻塞
4. **读取模块 AGENTS.md**：读取任务涉及模块的 `AGENTS.md` 文件
5. **执行**：按根目录 `AGENTS.md` 和模块 `AGENTS.md` 的约束实现任务
6. **测试**：如果任务的测试模式是 TDD 或边写边测，在执行过程中编写/更新测试
7. **验证**：运行相关测试套件并确认通过
8. **报告**：总结完成内容、变更文件和后续事项
