# PPT 设计大师 Skill（增强版）

整合全球顶级咨询公司设计规范 + 锐普PPT十八年专业设计体系，
面向企业级专业PPT制作场景。

## 一、锐普四大设计思维（核心底层逻辑）

锐普PPT方法论的根基，贯穿所有设计决策：

| 思维 | 核心含义 | 应用场景 |
|------|---------|---------|
| **观众思维** | 以观众理解为中心，而非制作者视角 | 信息层级、术语解释、逻辑引导 |
| **导演思维** | 把PPT当电影来设计，起承转合 | 故事线、情绪节奏、视觉叙事 |
| **设计思维** | 视觉即传达，每个元素有意义 | 配色、字体、留白、图示化 |
| **营销思维** | 说服决策者，注重成果导向 | 结论前置、数据可视化、行动号召 |

## 二、PPT 设计风格分类体系（锐普分类法）

### 2.1 七大主流风格

| 风格 | 视觉特征 | 适用场景 | 代表配色 |
|------|---------|---------|---------|
| **科技风** | 深色背景、几何线条、粒子感、发光效果 | 互联网、科技产品发布 | 深蓝 #0A1628 + 科技蓝 #00D4FF |
| **政务风** | 红/黄为主、庄重严肃、图标密集 | 政府汇报、党课培训 | 中国红 #C00000 + 金色 #D4A843 |
| **大图风** | 全屏背景图+大标题、景深效果、遮罩处理 | 品牌发布会、开场封面 | 取决于图片主调 |
| **商务风** | 蓝色系为主、留白充足、图表驱动 | 商业汇报、方案提案 | 深蓝 #00338D + 白底 |
| **创意风** | 渐变、撞色、几何图形、动感构图 | 创意提案、品牌宣讲 | 多色渐变 #FF6B6B→#FFE66D |
| **简约风** | 纯色/白底、无图/少图、极简文字 | 学术报告、专业论坛 | 黑白灰 + 单色点缀 |
| **中国风** | 水墨、传统纹样、书法字体、留白意境 | 文化传承、政府活动 | 墨黑 #1A1A1A + 朱红 #B5451B |

### 2.2 风格选择决策树

```
PPT用途 → 听众类型 → 推荐风格
─────────────────────────────────
商业汇报（对决策层） → 企业高管 → 商务风（McKinsey蓝白系）
商业汇报（对专业层） → 行业专家 → 简约风（少即是多）
品牌发布/产品宣讲 → 公众/媒体 → 大图风 + 科技风
内部培训/党课教育 → 政府/国企 → 政务风（红金体系）
创意提案/路演 → 投资人/高管 → 创意风（渐变撞色）
学术/专业论坛 → 学术人员 → 简约风（纯色背景）
文化/品牌活动 → 公众 → 中国风（水墨意境）
```

## 三、配色体系（锐普配色心理学）

### 3.1 色彩情绪语义

| 色彩 | 情绪联想 | 适用场景 | 禁忌场景 |
|------|---------|---------|---------|
| **蓝色系** | 沉静、平衡、智慧、专业、科技 | 企业汇报、科技发布、研究结论 | 餐饮、儿童、教育 |
| **橙色系** | 热情、活力、温暖、创造 | 营销提案、创意展示、年会 | 严肃政府、哀悼场合 |
| **红色系** | 热烈、权威、紧迫、警示 | 政务党建、危机汇报、促销 | 需冷静的场合 |
| **绿色系** | 自然、健康、成长、希望 | 医疗健康、环保主题、新品 | 传统高端（与金色冲突）|
| **紫色系** | 高端、神秘、优雅、个性 | 奢侈品、女性品牌、艺术 | 严肃商务 |
| **黑白灰** | 严肃、简洁、专业、经典 | 学术、专业评审、简约风 | 单调演示（需点缀色）|

### 3.2 企业级标准色盘（扩展版）

```
主色体系（Primary）：
├── 企业蓝   #00338D  — 咨询/金融/科技首选
├── 企业红   #C00000  — 政务/党建/权威场景
├── 企业绿   #007C5D  — 医疗/环保/健康
├── 企业橙   #FF7A00  — 营销/促销/活力
└── 企业紫   #7030A0  — 艺术/女性/高端

图表配色（Data Palette）：
├── 蓝1 #0071E3  — 数据系列1 / 主强调
├── 蓝2 #34C759  — 数据系列2 / 正向指标
├── 蓝3 #FF9500  — 数据系列3 / 警示/中性
├── 红4 #FF3B30  — 数据系列4 / 风险/下降
└  灰5 #8E8E93   — 辅助/基线/参考线

中性色：
├── 深灰 #323130  — 正文
├── 中灰 #5F6368  — 副标题/标签
├── 浅灰 #E1E1E1  — 分隔线/网格
└─ 背景白 #FFFFFF — 页面背景
```

### 3.3 配色操作规范（锐普标准）

- 同一页面不超过 **3种主色 + 1种点缀色**
- 色块面积比：主色 60% / 辅色 30% / 点缀色 10%
- 渐变使用：不超过 3 个颜色节点，建议 45° 线性渐变
- 深色背景配色：文字必须为白色或高亮色（对比度 ≥ 4.5:1）

## 四、字体系统（锐普字体学）

### 4.1 字体性格分类

| 字体类型 | 代表字体 | 性格特征 | 适用场景 |
|---------|---------|---------|---------|
| **黑体** | 微软雅黑、思源黑体、Helvetica Bold | 严肃、端正、正式、中性、权威 | 企业汇报、政务党建、商务方案 |
| **圆体** | 方正圆体、幼圆体、思源黑体 Rounded | 柔美、温和、亲切、轻松 | 儿童、母婴、健康、消费品 |
| **宋体** | 思源宋体、衬线体、Cambria | 秀丽、文化、典雅、传统 | 文化活动、学术报告、高端品牌 |
| **书法体** | 方正舒体、华文行草、造字工房 | 力道、坚实、厚重、震撼 | 电影海报、演讲高潮、特殊场景 |
| **等宽体** | Courier New、Source Code Pro | 数据、专业、技术、代码 | 数据表格、代码演示、技术文档 |

### 4.2 字体使用禁忌

- ❌ 正文中避免使用全部大写（标题除外）
- ❌ 避免使用衬线字体在深色背景上（小字更差）
- ❌ 不要在同一页混用超过 2 种字体
- ❌ 不要在 PPT 中使用系统自带的花体/艺术字体

### 4.3 字号层级标准（16:9 比例）

| 元素 | 字号 | 字重 | 行高 |
|------|------|------|------|
| 封面主标题 | 48-60pt | Bold | 1.1 |
| 章节过渡标题 | 36-44pt | Bold | 1.2 |
| 页面标题 | 28-36pt | Bold | 1.2 |
| 副标题 | 20-24pt | Medium | 1.3 |
| 正文内容 | 18-20pt | Regular | 1.5 |
| 图表/数据标签 | 12-14pt | Regular | 1.2 |
| 页脚/来源/备注 | 10-12pt | Regular | 1.2 |

> ⚠️ **字号红线**：正文不低于 **18pt**；图表标签不低于 **12pt**

## 五、排版布局规范（锐普版式学）

### 5.1 排版三原则

| 原则 | 含义 | 实践方法 |
|------|------|---------|
| **重要性原则** | 信息按重要程度划分视觉层级 | 主信息放大/加粗/用主色，次信息缩小/灰色 |
| **呼吸性原则** | 留白让内容"呼吸" | 内容占页面不超过 60%，边距 ≥ 0.7" |
| **统一性原则** | 全程保持视觉一致性 | 锁定主色、字体、间距、图标风格 |

### 5.2 布局模板（5大经典版式）

**单栏居中型**（封面/总结/过渡）：
```
[         主标题（大/居中）          ]
[          副标题（较小）            ]
[           Logo / 日期              ]
```

**左文右图型**（内容页/产品介绍）：
```
[ 核心文字    ][    高质量图片     ]
[  40-50%     ][      50-60%      ]
```

**左图右文型**（对比/案例）：
```
[  图片/图表  ][  右侧文字说明    ]
[    50%      ][      50%        ]
```

**上下图文型**（数据页/报告）：
```
[         图表区域（60-70%）      ]
[  关键洞察 + 数据来源（30-40%） ]
```

**双栏并排型**（对比/多观点）：
```
[  左栏内容  ][  右栏内容  ]
[    50%    ][    50%     ]
```

### 5.3 页面网格系统（16:9 锐普标准）

```
上边距：0.6"  (~43px)
下边距：0.5"  (~36px，含页码区）
左右边距：0.7" (~50px)
内容安全区：距边距 ≥ 0.3"

网格：推荐 12 列（每列 ~0.7"）
内容区宽度：9.1"（约 655px）
内容区高度：4.425"（约 319px）
```

## 六、图示与图表设计规范

### 6.1 图示分类（逻辑型图表）

| 图示类型 | 适用场景 | 锐普设计要点 |
|---------|---------|------------|
| **框架图** | 展示体系/架构/层次 | 层级清晰，子项不超过7个 |
| **流程图** | 展示步骤/过程/生命周期 | 箭头方向明确，步骤不超过8步 |
| **时间轴** | 展示历程/发展/里程碑 | 线性为主，节点不超过10个 |
| **对比图** | 展示选项/方案/AB test | 左右对称，数据说话 |
| **矩阵图** | 展示四象限/优先级 | 象限标签明确，原点居中 |
| **鱼骨图** | 展示原因分析 | 原因分类不超过6类 |
| **甘特图** | 展示项目计划/进度 | 时间刻度清晰，进度可视化 |
| **雷达图** | 展示多维度对比 | 维度名称清晰，不超过8维 |

### 6.2 数据图表设计（数据型图表）

**通用原则：**
- 数据墨水比最大化（减少不必要的装饰线条）
- 坐标轴标签：不超过 12pt，但不低于 10pt
- 数据标签：14pt，直接标注于数据点上
- 图例：图表下方（首选）或右侧垂直排列
- 数据系列：不超过 6 条（超出合并为"其他"）

**柱状图规范：**
- 柱间距：柱宽的 0.5-1 倍
- 坐标轴线宽：0.75pt，灰色 #D9D9D9
- 柱形顶部可加圆角（咨询公司通常用直角）
- 同一页柱状图不超过 3 组

**折线图规范：**
- 线宽：1.5-2pt（主数据线）
- 数据点：实心圆，直径 4-6pt
- 参考线：0.5pt，虚线

**饼图规范：**
- 起始角度：12 点钟方向
- 标注：直接标签（禁止仅用图例）
- 最大 6 个扇区，第 6 个标注为"其他"

## 七、咨询公司设计规范对照

### 7.1 三大咨询公司 PPT 标准

| 维度 | McKinsey | BCG | Bain |
|------|----------|-----|------|
| 主色 | #00338D | #00A3E0 | #00205C |
| 强调色 | #FFD700 | #F57C00 | #C8A962 |
| 标题风格 | 结论陈述句 | 问题引导式 | 故事线串联 |
| 布局特点 | 数据密集左对齐 | 图表驱动 | 叙事优先 |
| 图表风格 | 极简坐标轴 | 创意图表 | 战略框架 |
| 单页原则 | 一图一结论 | 一页一信息 | 一页一故事 |

### 7.2 McKinsey 五大黄金法则

1. **标题即结论** — 用陈述句，不用疑问句
2. **每页一信息** — 禁止信息过载
3. **数据说话** — 结论须有数据支撑
4. **电梯法则** — 30秒内可理解
5. **格式服从内容** — 视觉服务信息

### 7.3 Bain SCQA 叙事框架

- **S（Situation）**：当前情况是什么
- **C（Complication）**：哪里出了问题
- **Q（Question）**：核心问题是什么
- **A（Answer）**：我们的建议方案

## 八、锐普设计趋势与前沿

### 8.1 2019-2026 PPT设计趋势

| 趋势 | 特征 | 应用场景 |
|------|------|---------|
| **流体渐变** | 多色渐变、年轻化、动感 | 互联网、科技新品发布 |

| **数据叙事** | 图表动画、数据故事线 | 数据报告、战略分析 |

### 8.2 锐普PPT质量评估检查清单

```
□ 风格是否与用途/受众匹配？
□ 配色是否超过 3 种主色？
□ 字体是否超过 2 种类型？
□ 正文是否低于 18pt？
□ 图表标签是否低于 12pt？
□ 是否有留白（内容区 ≤ 60%）？
□ 对比度是否 ≥ 4.5:1（正文）/ 3:1（大文本）？
□ 图表数据是否来源标注？
□ 页码是否统一右下角？
□ 同一页面是否超过 6 行正文？
```

```
用户：这页图表可以怎么优化

→ 给出优化方案（含具体参数：字号/线宽/色值）
```

## 十一、企业模板排版设计规范（模板来源分析）

> 来源：5766.pptx（活动策划执行方案）+ 企业工作汇报PPT模板（通用版）
> 通过 XML 解构两个企业模板的主题、配色、字体、布局、版式结构
> Miller 2026-05-13

### 11.1 模板1深度解析：活动策划执行方案（5766.pptx）

**基本信息：**
- 36张幻灯片 / 3个主题文件 / 12个布局类型
- 完整结构：封面→目录→章节过渡→内容页→结尾页

**主题配色体系（深青色系）：**

| 色值 | 名称 | 用途 |
|------|------|------|
| #294A5A | 深青蓝 | 主标题、背景色块、页眉 |
| #7AAE39 | 黄绿色 | 序号圆点、图标填充、点缀 |
| #F9C000 | 金黄色 | 高亮强调、进度指示、行动号召 |
| #ED5A00 | 橙色 | 次级强调、警示信息 |
| #484849 | 深灰 | 正文文字、辅助线 |
| #FFFFFF | 纯白 | 页面背景、留白 |
| #969696 | 中灰 | 来源/备注/次要信息 |

**配色布局规律：**
```
深青蓝(#294A5A) → 页面顶部/左侧色块（占约15-20%）
金黄色(#F9C000) → 进度条（01 02 03 04步骤指示）
绿色(#7AAE39) → 序号圆圈（01/02/03/04）→ 导航感
```

**版式结构特征：**

| 页码位置 | 类型 | 视觉特征 |
|---------|------|---------|
| 封面 | Cover | 大标题居中+副标题+日期+企业标识区 |
| 目录 | TOC | 四大章节编号（01/02/03/04）+ 章节名 |
| 章节过渡 | Divider | 全屏色块+章节数字+章节名（震撼感）|
| 内容页 | Content | 左侧色块+右侧内容（双栏/三栏）|
| 结尾 | End | 感谢语+联系方式 |

**内容页排版规律：**
```
左侧固定色块（#294A5A宽条约0.8"）→ 占约8-10%宽度
主要内容区（白底）→ 占90%宽度
右下角金黄色进度圆点 → 步骤指示（01/02/03/04）
页码 → 右下角，深灰色，10pt
```

**字体使用规范：**
```
主标题：Arial Bold，36-44pt，深青蓝#294A5A
副标题：Arial，20-24pt，深灰#484849
正文：Arial/微软雅黑，18-20pt，深灰#484849
序号：Arial Bold，24pt，白色（居中于圆形）
页脚：Arial，10pt，中灰#969696
```

**模板亮点：**
```
✓ 深青蓝+黄绿色彩对比鲜明（冷暖对比，专业+活力）
```

## 十四、高级图表设计体系

### 14.1 进阶图表类型与适用场景

| 图表类型 | 核心优势 | 适用数据 | 推荐工具 |
|---------|---------|---------|---------|
| **桑基图（Sankey）** | 流量/转化关系可视化 | 供应链、用户旅程、预算分配 | Plotly/Python（建议预生成图片）|
| **旭日图（Sunburst）** | 多层级占比关系 | 分类占比、组织结构、区域分布 | Excel/Power BI原生支持 |
| **华夫饼图（Waffle）** | 单指标达成率展示 | KPI完成率、目标达成 | 10x10网格色块（PptxGenJS可实现）|
| **仪表盘（Gauge）** | 单一KPI达成感强 | 销售完成率、转化率 | PptxGenJS原生支持 |
| **子弹图（Bullet）** | 实际vs目标vs历史对比 | 业绩分析、KPI汇报 | PptxGenJS支持 |
| **瀑布图（Waterfall）** | 增减分解 | 财务分解、成本变化分析 | PptxGenJS原生支持 |
| **热力图（Heatmap）** | 多维度交叉数据 | 用户画像、市场矩阵 | Python seaborn生成 |
| **堆叠面积图** | 趋势+构成双重信息 | 收入构成、时间序列 | PptxGenJS支持 |

### 14.2 华夫饼图（Waffle Chart）设计规范

**为什么用华夫饼图替代饼图：**
- 人类对角度的感知不如对面积敏感
- 华夫饼图用10x10=100格表示百分比，每个格子=1%
- McKinsey/BCG均在使用，比饼图更专业

**设计规范：**
- 格子数量：10x10（100格）或5x20（100格）
- 每个格子代表：1%（或2%）
- 填充颜色：企业主色（饱和度高）
- 剩余格子：浅灰色 #E1E1E1
- 标签：格子上方显示百分比数字，右侧显示绝对值

### 14.3 Dashboard设计原则

1. 一屏一故事：每个仪表盘只讲一个核心结论
2. 墨水比最大化：减少不必要的装饰线/刻度
3. 色彩有含义：红色=危险，绿色=达标，灰色=基准
4. 数字要大：核心KPI数字 >= 60pt
5. 布局有呼吸感：组件间距 >= 16px
