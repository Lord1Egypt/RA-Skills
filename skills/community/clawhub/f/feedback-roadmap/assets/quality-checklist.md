# Feedback Roadmap — Quality Checklist

Run this before publishing a roadmap. If an item fails, fix it or note why it's acceptable. Aim to check every box.

## Signal Coverage
- [ ] At least 3-4 independent sources included (reviews, surveys, support, returns, social)
- [ ] Both happy-path and unhappy-path channels represented (not only complaints, not only 5★)
- [ ] A neutral, non-self-selected source included (e.g. survey to all buyers, return codes)
- [ ] Time window stated and consistent across sources
- [ ] Known channel biases documented (e.g. reviews skew extreme)
- [ ] Any missing channel or under-sampled journey stage explicitly flagged

## Clustering Quality
- [ ] Each theme maps to exactly one owner and one decision
- [ ] 8-20 active themes (no mega-buckets, no unactionable fragmentation)
- [ ] Tagging taxonomy locked for the cycle so counts are comparable
- [ ] Journey stage tagged (pre-purchase / checkout / delivery / use)
- [ ] Symptom-vs-root-cause analysis done; themes scored on root cause
- [ ] Related symptom themes merged under a shared root cause where applicable

## Quantification
- [ ] Mentions counted as distinct issues, not raw rows
- [ ] De-duped so one customer contributes at most one mention per theme
- [ ] Severity assigned per theme using the written rubric (Blocker/Major/Minor/Cosmetic)
- [ ] Source-reliability weighting applied and disclosed (raw + weighted shown)
- [ ] Customer-value weighting applied only where it changes the answer, and disclosed
- [ ] Spam, bot, and incentivized reviews excluded and noted

## Prioritization Rigor
- [ ] One framework used consistently across all ranked items
- [ ] Framework name and weights/scale stated in the doc
- [ ] All factor inputs (reach, impact, confidence, effort, urgency) recorded
- [ ] Effort estimated by the team that would actually build it
- [ ] Urgency tied to objective triggers, not feelings
- [ ] Scores shown so anyone can recompute the ranking
- [ ] Ties broken by a stated rule (e.g. higher severity wins)
- [ ] Kano sanity check: no broken must-have ranked below a delighter

## Rationale & Evidence
- [ ] Every ranked item has a one-line rationale tying score to evidence
- [ ] At least 2 representative verbatim quotes per theme retained
- [ ] Revenue/return/churn figures sourced, not invented
- [ ] Confidence level noted where estimates are soft

## Ownership
- [ ] Every ranked item has a named owner or team that controls the fix
- [ ] Owners have the capacity/mandate to take the item (not just nominally responsible)
- [ ] No item left unassigned or assigned to "everyone"
- [ ] Cross-team dependencies identified and flagged

## Roadmap Clarity
- [ ] Exec summary leads with the decision, not the methodology
- [ ] Top 3 priorities and the headline metric at stake are explicit
- [ ] Parking-lot / monitor section captures low-n and low-priority items
- [ ] Each parked item has a promotion trigger
- [ ] Sequencing notes cover dependencies and items that should ship together

## Cadence & Maintenance
- [ ] Next review date set
- [ ] Refresh cadence agreed (monthly typical; quarterly minimum)
- [ ] Changelog vs last cycle present (shipped / new / promoted / re-ranked)
- [ ] Shipped items checked to confirm underlying complaints actually dropped
- [ ] Small-sample items re-evaluated against their promotion triggers
