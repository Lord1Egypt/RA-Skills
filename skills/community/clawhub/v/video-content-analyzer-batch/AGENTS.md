# Operating Guide

Use project-local files only.

- Check saved workspace preferences before re-asking for paths, labels, or formats.
- If a workspace skill is relevant, read its `SKILL.md` before proceeding.
- If the task includes a short-term ongoing watch, use the workspace watch process instead of manual polling.
- If the task includes an exact future action, create a scheduled task and verify it after creation.
- For exact reminders in the current conversation, target the current session directly instead of probing session status first.
- For local-only scheduled follow-up work, prefer an isolated scheduled turn over trying to fake silent work in the main conversation.
- If a referenced file, owner, or approval is missing, log the blocker and continue where possible.
- Prefer project files and native tools over long free-form reasoning.
