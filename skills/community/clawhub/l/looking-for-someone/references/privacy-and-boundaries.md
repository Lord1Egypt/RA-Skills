# Privacy and Security Boundaries

## Core Privacy Principles

### 1. Data Minimization
- Collect only necessary information for case management
- Default to excluding sensitive personal data
- Regularly review and purge unnecessary information

### 2. Purpose Limitation
- Use data only for missing person case management
- Do not repurpose data for other uses
- Delete data when no longer needed for the original purpose

### 3. Transparency
- Clearly communicate data practices to users
- Explain what information is stored and why
- Provide options for data management

## Data Storage and Security

### Storage Location
- **Primary Storage**: `~/.openclaw/skills-data/looking-for-someone/`
- **Backup Location**: User-defined (optional)
- **Format**: JSON files with basic structure

### Security Measures
- **Access Control**: Local file system permissions
- **Encryption**: Not implemented in current version (clearly communicated)
- **Backups**: User responsibility for regular backups
- **Data Integrity**: Basic validation checks on input

### Data Retention Policy
- **Active Cases**: Retained indefinitely until marked resolved
- **Resolved Cases**: Archived after 1 year
- **Inactive Cases**: Flagged for review after 6 months
- **Deletion**: User-initiated deletion available at any time

## Information Disclosure Boundaries

### Public vs. Private Information

#### Safe for Public Disclosure:
- Name (with consent)
- Age
- General physical description
- Last seen location (general area)
- Clothing description
- Distinguishing features (non-sensitive)
- Contact method for tips

#### Never Disclose Publicly:
- Home address
- Identification numbers
- Financial information
- Sensitive medical conditions
- Family contact details (without consent)
- Exact coordinates of last seen location

#### Conditional Disclosure:
- Photos (with explicit consent)
- Phone numbers (monitored line only)
- Specific medical needs (if relevant to safety)
- Mental health information (only to authorities)

## Platform-Specific Guidelines

### Social Media Platforms

#### WeChat:
- Use private groups for initial sharing
- Public posts only with family consent
- Include clear contact instructions
- Monitor comments for suspicious activity

#### Weibo:
- Use appropriate hashtags (#寻人 #missingperson)
- Include official case numbers when available
- Respond to credible tips privately
- Report and block suspicious accounts

#### Douyin:
- Use clear, recent photos
- Include location tags appropriately
- Short, factual video descriptions
- Monitor for misinformation

### Official Channels
- Follow police template requirements
- Include case reference numbers
- Use formal, accurate language
- Coordinate with authorities on timing

## Risk Management

### High-Risk Scenarios

#### Financial Scams:
- **Indicator**: Requests for payment for information
- **Response**: Report to police, do not engage
- **Prevention**: Never offer rewards without police guidance

#### Identity Theft:
- **Indicator**: Requests for personal documents
- **Response**: Verify requester identity through official channels
- **Prevention**: Never share IDs or sensitive documents

#### Emotional Exploitation:
- **Indicator**: Manipulative language, false hope
- **Response**: Refer to professional support services
- **Prevention**: Maintain factual, measured communication

### Safety Protocols for Users

#### Information Verification:
1. Cross-check all tips with known facts
2. Verify source credibility
3. Consult with authorities on significant leads
4. Document all communications

#### Communication Safety:
1. Use dedicated contact methods
2. Avoid sharing personal devices/accounts
3. Be cautious of unsolicited contacts
4. Report suspicious activity immediately

#### Emotional Safety:
1. Designate a spokesperson
2. Take regular breaks from case management
3. Seek professional support when needed
4. Maintain hope while being realistic

## Legal and Ethical Boundaries

### Compliance Requirements
- **Chinese Law**: Comply with PRC laws on privacy and personal information protection
- **Platform Rules**: Adhere to terms of service for all platforms used
- **Professional Ethics**: Maintain confidentiality and respect for all involved

### Ethical Guidelines
1. **Respect Autonomy**: Honor wishes of missing person's family
2. **Avoid Harm**: Minimize potential negative impacts of publicity
3. **Maintain Dignity**: Treat all individuals with respect
4. **Ensure Accuracy**: Verify information before dissemination

### Cross-Border Considerations
- Different jurisdictions have different privacy laws
- International cases require coordination with authorities
- Be aware of cultural differences in missing person responses
- Consult legal advice for complex situations

## Technical Limitations

### Current Version Limitations
- No end-to-end encryption
- No automated data backup
- No integration with official databases
- No real-time monitoring capabilities
- No image recognition or comparison

### Transparency Requirements
- Clearly communicate all limitations to users
- Provide alternative methods for secure communication when needed
- Regularly update users on capability improvements
- Document all feature requests for future development

## User Control and Rights

### User Rights
1. **Access**: View all stored personal data
2. **Correction**: Request corrections to inaccurate information
3. **Deletion**: Request deletion of personal data
4. **Export**: Export case data in readable format
5. **Consent**: Withdraw consent for data use at any time

### Implementation
- Provide clear instructions for exercising rights
- Respond to requests within reasonable timeframe
- Document all user requests and actions
- Maintain audit trail of data management activities

## Incident Response

### Data Breach Protocol
1. **Identification**: Detect and confirm breach
2. **Containment**: Isolate affected systems
3. **Assessment**: Determine scope and impact
4. **Notification**: Inform affected users
5. **Remediation**: Address vulnerabilities
6. **Documentation**: Record incident and response

### User Support
- Provide clear contact methods for concerns
- Respond to privacy inquiries promptly
- Document and address user feedback
- Regular review of privacy practices
