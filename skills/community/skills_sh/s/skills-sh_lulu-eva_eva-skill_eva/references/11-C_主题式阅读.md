# Eva Learn 11-C：主题式阅读

适用场景：用户提供多份资料，或围绕一个主题想比较多个作者、流派、案例和观点。

本文件是一条线性旅程：

```text
多资料存档 -> 共同问题 -> 谱系定位 -> 交叉路径 -> 逐讲对读 -> 掌握检查 -> 产出表 -> 完成判断
```

进入本文件前必须已经建档。若未建档，返回 `11_eva-learn.md`。

## Phase 1：多资料存档

先登记资料，不直接比较。

必须执行：

1. 每份资料编号：`S1`、`S2`、`S3`。
2. 保存或登记到 `sources/原始资料/`。
3. 更新 `sources/INDEX.md`：资料名、作者/来源、类型、核心主题、可读性状态、用途。
4. 标记资料状态：核心资料 / 补充资料 / 反对资料 / 案例资料 / 待判断。

资料不可读时，不使用该资料做结论。

## Phase 2：确认共同问题

主题式阅读的核心不是“读很多资料”，而是让多份资料共同回答一个问题。

按顺序判断：

| 输入状态 | 动作 |
|---|---|
| 用户已有明确问题 | 复述并写入 `01-探究问题卡.md` |
| 用户只有主题 | 从资料标题、目录、导言中提炼 2-3 个候选共同问题，让用户选 |
| 资料之间没有共同问题 | 说明无法主题式阅读，建议改为 `11-B` 分别带学 |

一次只确认一个共同问题。

共同问题未确认前必须停下。不得进入资料谱系定位、交叉学习目录或逐讲对读。

## Phase 3：资料谱系定位

为每份资料标注位置：

- 核心命题：这份资料主张什么。
- 证据来源：它靠什么证明。
- 反对对象：它在反对谁或修正谁。
- 使用边界：它解释不了什么。

不确定作者、版本、立场、背景时标注“待核实”。

资料关系低置信度时，只能标为“待判断”。不得编造正反关系、流派关系、时代关系或作者争议。

## Phase 4：交叉学习路径

根据资料关系选择一种路径：

| 资料关系 | 路径 |
|---|---|
| 主资料 + 反对资料 | 正反命题对读 |
| 理论资料 + 案例资料 | 理论到场景迁移 |
| 同主题不同时代 | 概念演变 |
| 同主题不同作者 | 分歧焦点 |
| 多份资料没有稳定关系 | 先回到共同问题，缩小范围 |

输出交叉学习目录，默认 3-5 讲；大型主题 6-9 讲。

## Phase 5：逐讲对读

每讲必须先输出对读掌握目标卡：

- 本讲比较哪两份或哪几份资料。
- 本讲共同问题的哪一部分。
- 本讲分歧点是什么。
- 用户要能区分什么。
- 掌握检查题。
- 通过标准。

“用户要能区分什么”只能从以下三类里选：

- 能区分不同资料的核心命题。
- 能区分命题和证据。
- 能区分作者立场和自己的判断。

每讲必须区分四类内容：

```text
作者命题：作者在主张什么
证据来源：作者靠什么支撑
反对意见：谁不同意，为什么
争议焦点：真正分歧在哪里
```

每讲只比较一个分歧点，不同时比较所有资料。

每讲结尾只问一个掌握检查问题：

```text
你现在更同意哪边？依据是什么？能不能用自己的经验举一个例子？
```

## Phase 6：掌握检查

掌握状态：

| 状态 | 判定标准 | 下一步 |
|---|---|---|
| 完全掌握 | 能区分各方命题、证据、分歧和自己的判断 | 即时沉淀后进入下一讲 |
| 基本掌握 | 能说出主要分歧，有少量混淆 | 纠偏，即时沉淀后进入下一讲 |
| 部分理解 | 只复述观点，分不清证据和立场 | 回到资料对照表 |
| 尚未理解 | 不知道各方在争什么 | 缩小到两份资料重讲 |
| 假性流畅 | 只说态度，没有资料依据 | 要求补证据定位 |
| 未验证 | 用户跳过检查 | 可浏览式推进，但不得生成思想种子卡 |

未完成即时沉淀前，不得标记本讲完成。

## Phase 7：主题式产出

每讲结束必须先做即时沉淀，追加到 `07-学习问答原稿.md`：

- 共同问题。
- 本讲资料范围。
- 各方命题。
- 用户判断。
- Eva 纠偏。
- 掌握状态。

阶段完成后，输出并保存四张表：

- 概念表：不同资料如何定义关键概念。
- 命题表：各作者/资料的核心主张。
- 证据表：每个主张的证据来源。
- 争议表：真正分歧点。

四张表分别追加到 `03-阅读任务记录.md` 或用户指定文件；关键问答仍追加到 `07-学习问答原稿.md`。

## 最小状态闸门

| 节点 | 未完成前禁止 | 唯一下一步 |
|---|---|---|
| 多资料未存档 | 禁止比较、禁止目录 | 登记资料并更新 `sources/INDEX.md` |
| 共同问题未确认 | 禁止谱系定位、禁止对读 | 让用户选择一个共同问题 |
| 资料关系待判断 | 禁止编造关系 | 缩小到两份资料或标注待核实 |
| 未完成掌握检查 | 禁止进入下一讲 | 判断掌握状态或要求用户回答 |
| 未完成即时沉淀 | 禁止标记本讲完成 | 写入共同问题、资料范围、命题、用户判断和掌握状态 |

## Phase 8：完成判断

输出判断版本，追加到 `05-判断版本.md`：

```text
判断版本 v{n}
- 共同问题：
- 我现在站在哪个判断上：
- 支持证据：
- 反对证据：
- 仍不确定处：
```

判断版本保存失败时停止，不读取 `11-D_思想种子卡与内容链路交接.md`。未写入 `05-判断版本.md` 前，不得生成思想种子卡。

满足条件时读取 `11-D_思想种子卡与内容链路交接.md`：

- 用户能说清共同问题。
- 用户能区分至少两方命题和证据。
- 用户形成自己的阶段性判断。

## 禁止

- 不把多资料阅读做成资料摘要拼盘。
- 不在没有共同问题时硬做主题式阅读。
- 不编造作者立场、争议关系、页码或引文。
- 不跳过掌握检查。
- 不在即时沉淀前标记本讲完成。
- 不把阶段性判断直接写成完整内容稿。
