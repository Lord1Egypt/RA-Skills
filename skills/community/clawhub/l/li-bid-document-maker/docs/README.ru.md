# li-bid-document-maker — Руководство по использованию

> **Эксперт по тендерной документации** — Автоматическое преобразование тендерных документов в профессиональные ответные заявки с ориентацией на критерии оценки

## Установка

```bash
clawhub install li-bid-document-maker
```

## Быстрый старт

После установки произнесите любую из следующих фраз вашему AI-ассистенту:

- "Создать ответ на тендер"
- "Проанализировать этот тендер и сгенерировать заявку"
- "Подготовить тендерную заявку"
- "Создать ответ на RFP"

AI автоматически выполнит 6-этапный рабочий процесс: **Анализ тендера → Стратегический анализ → Создание структуры → Написание разделов → Проверка качества → PDCA-улучшение**

## Использование

### Способ 1: Загрузка файла

Перетащите тендерный документ (PDF или Word) в чат, затем скажите:
> "Используй li-bid-document-maker для анализа этого документа"

### Способ 2: Указание пути к файлу

> "Используй li-bid-document-maker, файл тендера находится в /projects/tender.pdf"

## Системные требования

| Параметр | Требование |
|----------|-----------|
| ОС | Windows 10/11, macOS, Ubuntu 20.04+ |
| AI-платформа | Claude, OpenClaw, Hermes или любой LLM Agent с поддержкой файлового ввода/вывода |
| Формат файлов | PDF (рекомендуется с возможностью поиска) или Word (.docx) |
| Зависимости | `python-docx`, `PyPDF2` (для обработки файлов, опционально) |

## Результат

- Полный документ ответной заявки (стандартная структура из 16 глав)
- Отчёт о улучшении качества PDCA
- Соответствие критериям оценки с указанием баллов

## Рабочий процесс

```
Этап 1: Анализ тендера          → Извлечение информации о проекте, техтребований, критериев оценки
Этап 2: Стратегический анализ   → Анализ весовых коэффициентов, конкурентная стратегия
Этап 3: Создание структуры     → Структура на основе критериев оценки (требуется подтверждение пользователя)
Этап 4: Написание разделов     → Написание содержания по разделам
Этап 5: Проверка качества      → 6-мерная всесторонняя проверка
Этап 6: PDCA-улучшение         → 3 раунда автоматического улучшения, затем передача
```
