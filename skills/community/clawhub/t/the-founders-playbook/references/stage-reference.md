# Founder's Playbook — Quick Reference

## Stage Detection Cheat Sheet

When the user says... | Activate stage...
---------------------|------------------
"idea", "think", "validate", "research", "is there a market" | **Idea**
"build", "code", "MVP", "prototype", "first version", "architect" | **MVP**
"launch", "ship", "customers", "revenue", "investor", "fundraise", "GTM" | **Launch**
"scale", "enterprise", "hire", "growing", "institutional", "moat" | **Scale**

If ambiguous, ask: *"What stage are you at? Idea, MVP, Launch, or Scale?"*

## Core Philosophy Principles

1. **Validate before you build** — 42% of startups fail because they built something nobody wanted. AI makes it faster to build AND faster to fail. Don't let the speed of building outrun the quality of your validation.

2. **AI tech debt compounds** — Unlike traditional tech debt (linear, manageable), AI-generated code debt is exponential. Without architectural guardrails, each session drifts further from coherence. The context file (AGENTS.md, SKILL.md, or equivalent project guidelines) is non-negotiable.

3. **The founder bottleneck scales terribly** — Systematize yourself out of every loop before you need to. If a workflow stalls when you're unavailable for a week, it's a single point of failure.

4. **Data is your moat** — User interaction data is time-locked, context-specific, and impossible to replicate. Design feedback loops that turn usage into systematic product improvement.

5. **AI agents are co-founders, not employees** — They don't need hand-holding, but they need direction. The quality of your output is bounded by the quality of your context. Invest in instructions.
