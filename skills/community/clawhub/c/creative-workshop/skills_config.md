## 二、技能详细配置

### 2.1 TRIZ强化 Skill

#### 基本信息

| 项目 | 内容 |
|------|------|
| **Skill名称** | opc-triz-innovation |
| **显示名称** | TRIZ强化 |
| **描述** | 基于TRIZ理论的技术创新方法论，提供40个发明原理、39×39矛盾矩阵、物场分析等核心工具 |
| **版本** | 2.5 |
| **作者** | OPC导师团队 |

#### 调用配置

| 配置项 | 值 | 说明 |
|-------|-----|------|
| **调用时机** | 策划师阶段 | 用户确认创意方向后 |
| **调用方式** | 主动加载 | 进入策划师阶段时自动加载 |
| **调用参数** | 用户选定的创意方向描述 | 作为TRIZ分析的输入 |
| **返回格式** | 结构化创新方案 | 含矛盾识别、原理推荐、解决方案 |

#### 调用参数模板

```json
{
  "input": {
    "creative_direction": "用户选定的创意方向",
    "user_requirements": "用户的功能需求和约束条件",
    "resource_constraints": "资源限制（时间/资金/人力）"
  },
  "options": {
    "include_36ji": true,
    "include_dikwp": true,
    "max_principles": 5
  }
}
```

#### 输出格式

```json
{
  "contradiction_analysis": {
    "type": "技术矛盾/物理矛盾",
    "improvement_param": "改善参数",
    "worsening_param": "恶化参数",
    "matrix_result": ["原理编号列表"]
  },
  "principles_application": [
    {
      "principle_id": 1,
      "principle_name": "分割",
      "application": "具体应用描述",
      "solution": "解决方案构想"
    }
  ],
  "innovation_score": {
    "novelty": 4,
    "feasibility": 4,
    "comprehensive": 4
  }
}
```

#### 在Coze中的挂载方式

- **挂载到**：策划师子Bot或主Bot
- **触发词**：TRIZ分析、创新方案、技术矛盾
- **工作流集成**：作为策划师阶段的核心分析模块

|------|------|
| **Skill名称** | opc-novelty-search |
| **显示名称** | 查新检索 |

#### 调用配置

### 2.3 知识产权申请 Skill

#### 基本信息

| 项目 | 内容 |
|------|------|
| **Skill名称** | opc-ip-application |
| **显示名称** | 知识产权申请 |
| **描述** | 专利、商标、软著三类知识产权申请的全流程辅助 |
| **版本** | 1.0.0 |
| **作者** | 胡田 |

#### 调用配置

| 配置项 | 值 | 说明 |
|-------|-----|------|
| **调用时机** | 知识产权师阶段 | 查新检索完成后 |
| **调用方式** | 主动调用 | 基于查新结果制定IP策略 |
| **调用参数** | 创新点清单+查新结果 | 确定保护方式 |
| **返回格式** | IP策略报告 | 含专利类型建议、材料清单、时间规划 |

#### 三大申请类型

| 类型 | 适用场景 | 保护期限 | 审查周期 | 官费（个人） |
|------|---------|---------|---------|-------------|
| **发明专利** | 技术创新、产品改进 | 20年 | 2-3年 | ~1000元 |
| **实用新型** | 产品形状/结构改进 | 10年 | 6-8月 | ~500元 |
| **外观设计** | 产品外观 | 15年 | 4-6月 | ~500元 |
| **商标注册** | 品牌标识保护 | 10年（可续展） | 9-12月 | ~300元/类 |
| **软件著作权** | 软件产品保护 | 终身+50年 | 60日 | 免费（个人） |

#### 调用参数模板

```json
{

    "priority": "high/medium/low",
    "budget_range": "预算范围"
  }
}
```

#### 输出格式

```json
{
  "ip_strategy": {
    "patent_application": {
      "recommended_type": "发明专利",
      "application_timing": "立即",
      "material_checklist": ["材料1", "材料2"],
      "timeline": "12-18个月",
      "estimated_cost": "6000-20000元"
    },
    "trademark_application": {
      "recommended_classes": ["第9类", "第42类"],
      "application_strategy": "防御性注册",
      "material_checklist": ["材料1", "材料2"]
    }
  }
}
```

#### 在Coze中的挂载方式

- **挂载到**：知识产权师子Bot或主Bot
- **触发词**：专利申请、商标注册、软著登记、IP保护
- **工作流集成**：作为知识产权师阶段的核心指导模块

## 四、技能调用决策树

### 4.1 阶段启动决策

```
用户输入
    │
    ▼
【猎手阶段】
    │
    ├── 意图识别
    │   ├── 创意需求 → 启动猎手流程
    │   ├── 已有方向 → 跳过猎手，进入策划师
    │   └── 全流程 → 启动完整五阶段
    │
    ▼
【策划师阶段】
    │
    ├── 加载TRIZ强化Skill
    ├── 分析技术矛盾
    ├── 查矛盾矩阵
    └── 输出创新方案
    │
    ▼
【分析师阶段】
    │
    ├── 加载商业模式分析Skill
    ├── 市场分析
    ├── 财务测算
    └── 输出商业报告
    │
    ▼
【知识产权师阶段】
    │
    ├── 加载查新检索Skill → 技术特征分析
    ├── 加载知识产权申请Skill → IP策略
    └── 输出IP布局方案
    │
    ▼
【设计师阶段】
    │
    ├── 整合所有产出
    ├── 生成行动清单
    └── 输出完整方案包
```

### 4.2 异常分支决策

| 异常场景 | 检测条件 | 决策动作 |
|---------|---------|---------|
| 技术矛盾无法解决 | TRIZ分析无推荐原理 | 升级到所长决策 |
| 查新发现直接冲突 | 相似度>90% | 建议调整方向 |
| 商业分析否定方案 | 可行性评分<5 | 建议重新设计 |
| 用户需求超出MVP | 资源需求超出承受 | 触发方案裁剪 |
