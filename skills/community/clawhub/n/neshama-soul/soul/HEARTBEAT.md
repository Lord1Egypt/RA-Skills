# 定时心跳任务

> 定期检查用户状态和人格系统健康度
> **注意**: 此任务由 Agent 自动触发，无需用户手动执行

---

## 触发频率

- **间隔**: 每 30 分钟（或根据配置调整）
- **触发条件**: 每次用户交互后检查

---

## 检查清单

### 1. USER.md 健康检查

| 检查项 | 状态 | 操作 |
|--------|------|------|
| 文件存在 | ✅/❌ | 如不存在，提示用户重新初始化 |
| 格式正确 | ✅/❌ | 如格式损坏，使用默认配置 |
| 内容完整 | ✅/❌ | 如缺少关键字段，提示用户补充 |

### 2. SOUL.md 同步检查

| 检查项 | 状态 | 操作 |
|--------|------|------|
| OCEAN 配置有效 | ✅/❌ | 如数值异常，使用默认值 |
| 与 API 配置同步 | ✅/❌ | 如需要，调用 API 更新 |
| 人格印记清晰 | ✅/❌ | 如模糊，重新生成 |

### 3. 技术决策记录

- 检查是否有新的技术决策需要记录
- 检查是否有决策结果需要更新
- 检查用户偏好是否有变化

### 4. 系统状态

| 指标 | 状态 |
|------|------|
| API 连通性 | ✅ 正常 / ❌ 故障 |
| 本地配置 | ✅ 可用 / ⚠️ 需同步 |
| 情绪状态 | 💭 连续 |

---

## 自动行为

### USER.md 更新

当检测到新的用户偏好时，自动更新：

```markdown
## 偏好更新记录

| 日期 | 更新内容 | 原因 |
|------|----------|------|
| YYYY-MM-DD | [更新内容] | [原因] |
```

### 提醒用户

当检测到以下情况时，提醒用户：

1. **USER.md 过期**（超过 30 天未更新）
   ```
   💡 提示：你的用户偏好已经有一段时间没更新了。
   需要我帮你检查一下是否需要更新吗？
   ```

2. **新功能可用**
   ```
   🎉 Neshama Soul Engine 有新功能！
   需要我帮你了解吗？
   ```

3. **系统更新**
   ```
   🔄 系统已更新到 v[x.x.x]
   主要改进：[更新内容]
   ```

## 心跳报告格式

```markdown
## Neshama Soul 心跳报告

**时间**: YYYY-MM-DD HH:mm:ss
**状态**: ✅ 健康

### USER.md
- 状态: ✅ 正常
- 最后更新: YYYY-MM-DD

### SOUL.md
- OCEAN 配置: ✅ 有效
- 人格印记: ✅ 清晰

### API 状态
- 连通性: ✅ 正常
- 响应时间: xxx ms

### 技术决策
- 待记录: 0
- 最近记录: YYYY-MM-DD
```

---

## 异常处理

| 异常情况 | 处理方式 |
|----------|----------|
| API 不可用 | 使用本地缓存的 SOUL.md 配置 |
| USER.md 损坏 | 使用默认配置，提示用户检查 |
| SOUL.md 异常 | 恢复到默认 OCEAN 配置 |
| 全部配置失效 | 使用内置最小配置，保证基本功能 |
