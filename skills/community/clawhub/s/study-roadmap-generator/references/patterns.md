# 参考：学科模板库与典型路线示例

## 编程语言入门（Python / R / SQL / JavaScript / Go）

### 通用阶段主题

| 阶段 | 占比 | 典型主题 | 里程碑 |
|------|------|----------|--------|
| 基础 | 15% | 环境搭建、语法基础、首次输出 | 运行第一个程序 |
| 核心 | 35% | 数据类型、控制流、函数、文件 I/O | 完成基础数据处理 |
| 进阶 | 35% | 领域核心库 / 框架 / API | 完成目标场景的独立实现 |
| 综合 | 15% | 完整项目、调试、优化 | 提交可运行的完整项目 |

### 推荐学习栈模板

| 用途 | 资源类型 | 选择标准 |
|------|----------|----------|
| 交互式入门 | 在线教程（带 REPL） | 零基础友好、即时反馈 |
| 语法练习 | 习题平台 | 难度递进、覆盖全面 |
| 项目实战 | 数据集 + 项目模板 | 真实场景、有验收标准 |
| 面试准备 | 题库平台 | 按公司/难度筛选 |
| 参考文档 | 官方文档 / 速查表 | 权威、更新及时 |

### 常见卡点 — 按学科

| 学科 | 卡点 | 解决方案 |
|------|------|----------|
| SQL | JOIN 逻辑混淆 | 用 Venn 图理解集合关系后再写代码 |
| SQL | GROUP BY 使用规则 | SELECT 中的非聚合列必须出现在 GROUP BY 中 |
| SQL | 窗口函数 OVER 语法 | 先掌握 ROW_NUMBER，再扩展到 RANK / DENSE_RANK |
| SQL | 日期格式不一致 | 用 DATE() / DATETIME() 标准化后再比较 |
| Python | 列表推导式 | 先用普通 for 循环，熟悉后再替换 |
| Python | 包管理混乱 | 每个项目使用独立 venv / conda 环境 |
| R | 向量索引从 1 开始 | 与 Python/Java 习惯不同，写代码时检查边界 |
| JS | 异步编程 | 从 callback → Promise → async/await 逐步过渡 |
| JS | this 指向 | 优先使用箭头函数或 class 语法 |

### 动态调整场景

| 场景 | 调整 |
|------|------|
| 目标方向为后端开发 | 增加索引优化、查询性能分析和数据库设计章节 |
| 目标方向为数据分析 | 优先聚合查询、窗口函数和看板式查询训练 |
| 每日学习时间不足 1h | 路线延长至 5 周，每个阶段天数翻倍 |
| 有编程基础 | 跳过环境搭建阶段，直接从核心语法开始 |
| 目标为考证/认证 | 切换为考试路线，增加真题模拟比例 |

## 路线风格对比示例

### 同一目标（学 SQL），三种风格

| 维度 | 实战路线 | 学术路线 | 考试路线 |
|------|----------|----------|----------|
| 基础阶段 | 装好工具 → 跑通第一个 SELECT | 数据库原理 → ER 图 → SELECT | 直接看考点大纲 → 刷语法题 |
| 核心阶段 | JOIN / GROUP BY 做数据分析 | 关系代数 → 范式 → 复杂查询 | 刷遍 20 道高频语法题 |
| 进阶阶段 | 窗口函数 + 商业分析场景 | 查询优化 → 事务 → 索引 | 限时模拟题 + 错题复盘 |
| 综合阶段 | 电商数据分析完整项目 | 数据库设计报告 | 真题套卷 |
| 推荐学习栈 | SQLBolt + LeetCode 实战 | 教材 + 论文 + 基础练习 | 牛客真题 + 考点背诵 |
| 主要区别 | 一切围绕"能写查询分析数据" | 体系化建立数据库认知 | 以通过考试为核心标准 |
