## 领导力培养生成要求

领导力培养每种任务类型的"零件清单"——必选/可选组件、组装顺序、领域约束。按清单逐项产出。

---

### 1 领导力诊断

#### T1-1 能力诊断

- **必选组件**: 领导力测评工具（MBTI/DISC/360度评估）、能力维度清单（决策力/影响力/沟通力/执行力）、评分标准、诊断报告模板
- **可选组件**: 心理测评、行为面试、过往业绩分析
- **组装顺序**: 确定评估维度 → 选择测评工具 → 执行测评 → 收集数据 → 生成诊断报告
- **约束**: 测评工具必须专业可靠；评估维度必须覆盖领导力核心要素；结果必须客观量化
- **格式**: 领导力诊断报告（Markdown）

#### T1-2 潜力评估

- **必选组件**: 潜力评估维度（学习敏锐度/抗压能力/影响力/战略思维）、评估方法、评估结果、发展潜力判断
- **可选组件**: 发展潜力预测模型、成功案例对比
- **组装顺序**: 确定潜力维度 → 选择评估方法 → 执行评估 → 分析结果 → 判断发展潜力
- **约束**: 维度必须与领导力相关；方法必须科学有效；判断必须基于数据
- **格式**: 潜力评估报告（Markdown）

#### T1-3 发展目标

- **必选组件**: 短期目标（3-6个月）、中期目标（6-12个月）、长期目标（1-3年）、关键指标
- **可选组件**: 里程碑节点、激励机制、风险预案
- **组装顺序**: 分析诊断结果 → 识别发展重点 → 设定短期目标 → 设定中期目标 → 设定长期目标
- **约束**: 目标必须基于诊断结果；长短期目标必须衔接；指标必须可衡量
- **格式**: 领导力发展目标文档（Markdown）

#### T1-4 培养方案

- **必选组件**: 培养路径（诊断→训练→实践→辅导）、培养内容、培养方法、资源清单
- **可选组件**: 个性化方案、弹性安排、支持体系
- **组装顺序**: 确定培养路径 → 设计培养内容 → 选择培养方法 → 配置资源 → 制定实施计划
- **约束**: 路径必须符合发展规律；内容必须覆盖目标；方法必须有效
- **格式**: 领导力培养方案（Markdown）

---

### 2 领导力训练

#### T2-1 理论学习

- **必选组件**: 学习内容（领导力理论/管理知识/组织行为学）、学习资料、学习计划、学习笔记
- **可选组件**: 在线课程、专家讲座、学习社群
- **组装顺序**: 确定学习内容 → 获取学习资料 → 制定学习计划 → 执行学习 → 记录笔记
- **约束**: 内容必须系统完整；资料必须权威可靠；学习必须深入理解
- **格式**: 理论学习笔记（Markdown）

#### T2-2 案例研讨

- **必选组件**: 案例素材（经典案例/行业案例/企业案例）、研讨提纲、研讨记录、案例总结
- **可选组件**: 案例库、专家点评、小组讨论
- **组装顺序**: 选择案例 → 准备研讨提纲 → 组织研讨 → 记录讨论 → 总结经验
- **约束**: 案例必须典型有启发；研讨必须深入；总结必须有行动指导
- **格式**: 案例研讨记录（Markdown）

#### T2-3 模拟演练

- **必选组件**: 模拟场景（管理场景/危机场景/谈判场景）、角色分配、演练规则、演练反馈
- **可选组件**: 情景剧本、评估标准、录像回放
- **组装顺序**: 设计模拟场景 → 分配角色 → 讲解规则 → 执行演练 → 提供反馈
- **约束**: 场景必须贴近实际；规则必须清晰；反馈必须具体
- **格式**: 模拟演练记录（Markdown）

#### T2-4 实践锻炼

- **必选组件**: 实践任务（带领团队/负责项目/处理问题）、实践指导、实践记录、实践总结
- **可选组件**: 实践导师、实践平台、实践资源
- **组装顺序**: 确定实践任务 → 提供实践指导 → 执行实践 → 记录过程 → 总结经验
- **约束**: 任务必须有挑战性；指导必须及时；总结必须深入
- **格式**: 实践锻炼记录（Markdown）

---

### 3 实践发展

#### T3-1 岗位轮换

- **必选组件**: 轮换计划（轮换岗位/轮换周期/轮换目标）、岗位说明、轮换记录、轮换总结
- **可选组件**: 轮换导师、轮换支持、轮换评估
- **组装顺序**: 制定轮换计划 → 安排轮换岗位 → 执行轮换 → 记录经验 → 总结收获
- **约束**: 岗位必须有代表性；周期必须足够长；总结必须有深度
- **格式**: 岗位轮换记录（Markdown）

#### T3-2 项目领导

- **必选组件**: 项目任务（项目目标/项目范围/项目团队）、领导职责、项目记录、项目总结
- **可选组件**: 项目导师、项目资源、项目评估
- **组装顺序**: 明确项目任务 → 承担领导职责 → 领导项目执行 → 记录过程 → 总结经验
- **约束**: 任务必须有挑战性；职责必须明确；总结必须有启发
- **格式**: 项目领导记录（Markdown）

#### T3-3 团队管理

- **必选组件**: 团队管理职责（目标设定/任务分配/绩效管理/冲突处理）、管理方法、管理记录、管理总结
- **可选组件**: 团队建设活动、管理工具、管理培训
- **组装顺序**: 明确管理职责 → 学习管理方法 → 执行团队管理 → 记录过程 → 总结经验
- **约束**: 职责必须全面；方法必须有效；总结必须有深度
- **格式**: 团队管理记录（Markdown）

#### T3-4 危机处理

- **必选组件**: 危机场景（突发事件/重大投诉/团队冲突）、处理流程、处理记录、处理总结
- **可选组件**: 危机预案、危机演练、危机支持
- **组装顺序**: 识别危机场景 → 启动处理流程 → 执行危机处理 → 记录过程 → 总结经验
- **约束**: 场景必须真实；流程必须规范；总结必须有教训
- **格式**: 危机处理记录（Markdown）

---

### 4 导师辅导

#### T4-1 导师匹配

- **必选组件**: 导师选择标准（经验/风格/可用性）、导师库、匹配方法、辅导计划
- **可选组件**: 导师培训、导师激励、导师网络
- **组装顺序**: 确定选择标准 → 筛选导师 → 匹配导师 → 建立关系 → 制定辅导计划
- **约束**: 标准必须合理；匹配必须合适；计划必须可行
- **格式**: 导师匹配记录（Markdown）

#### T4-2 辅导实施

- **必选组件**: 辅导内容（经验分享/问题解答/职业建议）、辅导方式（一对一/小组/现场）、辅导记录、辅导反馈
- **可选组件**: 辅导工具、辅导资源、辅导评估
- **组装顺序**: 确定辅导内容 → 选择辅导方式 → 执行辅导 → 记录内容 → 收集反馈
- **约束**: 内容必须有价值；方式必须有效；记录必须完整
- **格式**: 辅导实施记录（Markdown）

#### T4-3 360度反馈

- **必选组件**: 反馈来源（上级/下属/同事/客户）、反馈问卷、反馈收集、反馈分析
- **可选组件**: 反馈平台、反馈辅导、反馈跟踪
- **组装顺序**: 确定反馈来源 → 设计反馈问卷 → 收集反馈 → 分析反馈 → 形成报告
- **约束**: 来源必须全面；问卷必须有效；分析必须客观
- **格式**: 360度反馈报告（Markdown）

#### T4-4 持续改进

- **必选组件**: 改进依据（反馈结果/实践总结）、改进目标、改进计划、改进效果
- **可选组件**: 改进支持、改进跟踪、改进激励
- **组装顺序**: 分析改进依据 → 确定改进目标 → 制定改进计划 → 执行改进 → 评估效果
- **约束**: 依据必须充分；目标必须具体；计划必须可行；效果必须可衡量
- **格式**: 持续改进计划（Markdown）
