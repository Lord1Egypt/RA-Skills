# 首次配置向导

此向导帮助新用户快速配置 `user_profile.md`，让评分引擎了解用户的求职偏好和底线。

---

## 流程概述

AI 将通过问答方式收集以下信息，完成后自动填入 `user_profile.md`。

```
配置流程：基本信息 → 薪资底线 → 技术方向 → 工作偏好 → 输出文件
```

---

## 配置步骤

### Step 1：收集基本信息

向用户逐一提问：

1. **称呼**：怎么称呼你？（可选，用于报告中的称呼）
2. **工作年限**：几年工作经验？
3. **期望城市**：你想在哪个城市工作？（如多个可列出）
4. **求职状态**：在职 / 离职 / 刚毕业 / 其他

### Step 2：收集薪资信息

5. **Base 底线**：你能接受的最低 Base 月薪是多少？（如 20K）
6. **期望薪资**：你理想的 Base 月薪是多少？（如 30K）
7. **是否接受 trade-off**：如果公司非常稳定（央企/国企/外企大厂），可以接受更低一些吗？低多少？

### Step 3：收集技术方向

8. **核心技术方向**：你的主要技术方向是什么？（如 Agent/LLM / 前端 / 后端 / 嵌入式 / 数据 / 其他）
9. **技术栈**：你主要用什么技术栈？（如 Python+PyTorch / React+Node / Go+K8s）
10. **成长方向**：你希望在哪个方向上持续成长？（如 AI 架构 / 全栈 / 高并发 / 产品化）

### Step 4：收集工作偏好

11. **公司类型偏好**：你优先考虑什么类型的公司？
    - 可选：大厂 / 外企 / 国企/央企 / 创业公司 / 中型科技公司 / 不拘
12. **工作节奏**：你能接受什么节奏？
    - 可选：必须双休 / 偶尔加班可接受 / 大小周可接受 / 不拘
13. **管理风格**：你偏好什么样的管理风格？
    - 可选：结果导向 / 尊重个体差异 / 避免微观管理 / 不拘
14. **稳定性要求**：你希望在一家公司待多久？
    - 可选：2年+ / 3年+ / 不拘

### Step 5：输出 user_profile.md

收集完以上信息后，AI 自动生成 `references/user_profile.md` 文件，使用 `user_profile.TEMPLATE.md` 的格式填入用户回答。

生成后告知用户：
> "配置完成！`user_profile.md` 已生成。你可以随时修改这个文件来调整评分偏好。"

---

## 错误处理

- **用户跳过某些问题**：标记为「未配置」，该维度评分使用默认值
- **用户提供模糊信息**：如"薪资差不多就行" → 追问具体数字
- **用户中途退出**：已收集的信息写入文件，未收集的用占位符，下次运行时提示用户继续配置
- **用户已存在 user_profile.md**：跳过向导，直接提示"已有配置，如需调整请修改 user_profile.md"

---

## 配置示例对话

```
AI: 我先帮你配置岗位筛选的参数。你的 Base 月薪底线是多少？
用户: 20K 吧
AI: 了解了。那理想的期望月薪呢？
用户: 30K
AI: 好的。如果你的技术方向是...（继续）
```

> 全程保持一问一答，不要一次性问太多问题。每次只问 1 个问题，等用户回答后再问下一个。
