### 原子动作 028：写入飞书文档

| 字段 | 内容 |
|------|------|
| **编号** | ATOM-FEISHU-028 |
| **名称** | 写入飞书文档 |
| **输入** | Markdown 内容 + 文档 Token |
| **输出** | 飞书文档（已写入） |
| **偏好设置** | **分块写入（≤200 blocks/块）**、第 1 块 write 后续 append、避免 400 错误 |
| **出现 Skills** | 统一专业词汇 Skill、原子动作文档同步 Skill、项目知识专家 Skill |
| **状态** | ✅ 已固化 |
| **最后更新** | **2026-03-07 14:48** |

**操作说明：**
1. 创建空白文档（`create -title`）
2. 分块（≤200 blocks/块，每块 30-50 blocks）
3. 第 1 块：`write` 模式（覆盖）
4. 第 2 块及以后：`append` 模式（追加）
5. 确认写入成功

**⚠️ 关键教训（2026-03-07 14:48）：**
- ❌ **禁止**：`create -content "长内容"`（会空白文档）
- ✅ **正确**：`create` → `write` → `append`（分块写入）
- **原因**：create 动作不支持直接写入长内容

**错误示例：**
```powershell
❌ feishu_doc -action create -content "101 blocks 内容"
   结果：返回 document_id，但文档空白
```

**正确示例：**
```powershell
✅ 第 1 步：feishu_doc -action create -title "文档标题"
   获取 doc_token

✅ 第 2 步：feishu_doc -action write -doc_token $token -content "第 1 块（34 blocks）"

✅ 第 3 步：feishu_doc -action append -doc_token $token -content "第 2 块（67 blocks）"
```

## 📋 使用指南

### 如何复用原子动作

1. **查找**：在清单中查找所需动作
2. **确认**：查看偏好设置和状态
3. **执行**：按操作说明执行
4. **记录**：在任务回复中标注用了哪些原子动作

### 如何新增原子动作

1. **定义**：明确输入/输出/偏好
2. **编号**：按分类编号（ATOM-分类 - 序号）
3. **记录**：添加到本清单
4. **测试**：执行一次验证效果
5. **固化**：确认无误后标记为✅已固化

### 如何纠正原子动作

1. **发现问题**：执行结果不符合预期
2. **标注偏好**：明确用户偏好
3. **更新清单**：修改偏好设置
4. **通知用户**：确认已更新
5. **应用新标准**：后续执行按新标准
