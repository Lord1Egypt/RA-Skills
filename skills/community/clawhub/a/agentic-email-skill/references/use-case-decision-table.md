# Email Use-Case Decision Table

Use this table when choosing the right template for a specific situation.

| Situation | Template | Why |
|---|---|---|
| Cold prospect, general fit | `cold-problem-pilot` | Opens with a workflow pilot idea and low-friction meeting ask. |
| Operations bottleneck | `cold-operations-bottleneck` | Focuses on intake, classification, drafting, routing, and review. |
| Technical buyer | `cold-technical-evaluation` | Leads with evaluation, architecture, approvals, logs, and production caution. |
| Executive risk concern | `cold-executive-risk` | Frames AI progress with controlled automation. |
| Sales workflow | `cold-revenue-team` | Targets research, recap, CRM, follow-up, and rep approval. |
| Support workflow | `cold-support-team` | Targets ticket triage, retrieval, draft response, escalation, and logs. |
| Knowledge scattered across systems | `cold-ops-knowledge` | Frames retrieval plus reviewed next action. |
| Founder or small team | `cold-founders` | Keeps the offer small, scoped, and practical. |
| First cold follow-up | `followup-workflow-map` | Adds value by suggesting a concrete workflow map. |
| Control/risk follow-up | `followup-risk-controls` | Explains human approval gates. |
| Example needed | `followup-example` | Shows the six-step agentic workflow pattern. |
| Prospect thinks scope is too big | `followup-proofless-value` | Reframes as a small test. |
| Need simple reply | `followup-breakthrough-question` | Gives multiple-choice diagnostic options. |
| Final follow-up | `breakup-close-loop` | Politely closes or asks when to revisit. |
| Referred lead | `warm-intro-context` | Uses referrer context. |
| Referred lead, second touch | `warm-intro-workflow` | Moves from trust to workflow review. |
| Inbound lead response | `inbound-fast-response` | Acknowledges and frames discovery quickly. |
| Need qualification details | `inbound-qualification` | Asks for workflow, systems, approvals, success, and constraints. |
| Ready to schedule inbound call | `inbound-booking` | Sends a discovery agenda and booking link. |
| Before discovery | `discovery-confirm-agenda` | Sets expectations and invites concerns. |
| After discovery | `post-discovery-recap` | Summarizes pain, workflow, controls, and next step. |
| Before formal proposal | `proposal-preview` | Confirms scope before writing the formal proposal. |
| Proposal sent | `proposal-sent` | Points to proposal and decision areas. |
| Proposal stalled | `proposal-followup-questions` | Asks about scope, budget, risk, timing, and ownership. |
| Budget objection | `close-objection-budget` | Offers smaller scoped pilot. |
| Risk objection | `close-objection-risk` | Re-centers on bounded pilot and approval gates. |
| Timing objection | `close-objection-timing` | Separates workflow assessment from build. |
| Internal build objection | `close-objection-internal-team` | Offers advisory workflow/evaluation handoff. |
| Close summary | `close-decision-summary` | Restates reason, pilot, controls, deliverables, next step. |
| Final close nudge | `close-final-nudge` | Asks whether to send agreement or close out. |
| Contract sent | `contract-sent` | Shares agreement and contract decision points. |
| Contract questions | `contract-clarifications` | Explains scope, approvals, acceptance, IP, and excluded uses. |
| Deposit or invoice | `deposit-invoice` | Moves payment and signature toward kickoff. |
| Waiting on signature | `signature-reminder` | Nudges without pressure. |
| Signature complete | `kickoff-after-signature` | Starts kickoff checklist. |
| Kickoff planning | `kickoff-agenda` | Defines meeting agenda and attendees. |
| Need access | `access-request` | Lists sandbox, docs, examples, APIs, approval rules, and logs. |
| Weekly project update | `weekly-update` | Standard delivery status. |
| Prototype ready | `review-ready` | Requests focused review. |
| Acceptance | `acceptance-request` | Asks for acceptance or specific deficiencies. |
| Handoff | `handoff-complete` | Confirms documentation, limits, support window. |
| Expansion | `expansion-next-workflow` | Suggests next workflow after success. |
| Referral | `referral-request` | Requests relevant introduction. |
| Testimonial | `testimonial-request` | Requests short project note. |
| Retention check-in | `quarterly-checkin` | Checks stability and adjacent needs. |
| Win-back with new event | `winback-new-trigger` | Reopens with a timely reason. |
| Hiring trigger | `trigger-hiring` | Connects hiring to repeatable workflow scale. |
| New tool/system trigger | `trigger-new-tool` | Connects system change to agentic workflow layer. |
| Growth trigger | `trigger-growth` | Connects growth to manual workflow strain. |
| Educational nurture | `nurture-educational` | Teaches workflow fit test. |
| Advisory/lightweight offer | `nurture-one-page-offer` | Offers workflow assessment instead of build. |
| Old opportunity | `reengage-old-opportunity` | Rechecks priority without forcing a new trigger. |

Selection rule: pick by current decision point first, then by buyer persona, then by trigger. If the recipient has not agreed to a problem, stay in cold/follow-up/discovery. If the recipient has agreed to a scoped pilot, move to proposal/closing/contract. If the project is signed, use kickoff/delivery/post-sale.
