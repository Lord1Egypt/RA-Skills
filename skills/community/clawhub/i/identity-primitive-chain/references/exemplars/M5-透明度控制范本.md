# M5 透明度控制域范本

## M5-01 默认隐藏策略

### 输入示例
任务类型：用户管理系统开发

### 处理过程
1. 隐藏范围：基元执行细节默认隐藏
2. 隐藏级别：中等
3. 隐藏原因：避免信息过载
4. 用户感知：展示执行结果和关键节点

### 输出示例
```markdown
## 默认隐藏策略文档

### 隐藏范围
- **隐藏内容**：基元执行细节
- **隐藏级别**：中等
- **隐藏原因**：避免信息过载
- **用户感知**：展示执行结果和关键节点

### 隐藏级别
| 级别 | 描述 | 适用场景 |
|------|------|----------|
| 低 | 展示所有细节 | 调试模式 |
| 中 | 展示关键结果 | 常规模式 |
| 高 | 仅展示最终结果 | 快速模式 |
```

## M5-02 用户请求处理

### 输入示例
用户请求：展示执行细节

### 处理过程
1. 请求接收：接收用户追溯请求
2. 请求验证：确认用户需求
3. 请求处理：展示基元执行过程
4. 请求响应：提供详细执行过程

### 输出示例
```markdown
## 用户请求处理流程

### 请求接收
- **请求类型**：展示执行细节
- **请求时间**：YYYY-MM-DD HH:MM:SS

### 请求验证
- **验证内容**：确认用户需求
- **验证结果**：请求有效

### 请求处理
- **处理内容**：展示基元执行过程
- **处理方法**：调用状态追踪模块

### 请求响应
- **响应内容**：基元执行过程时间线
- **响应格式**：结构化文档
```

## M5-03 执行细节展示

### 输入示例
展示内容：基元执行过程

### 处理过程
1. 展示目标：展示基元执行过程
2. 展示方法：时间线图
3. 展示设计：节点表示基元状态，边表示状态转换
4. 展示实现：SVG图形

### 输出示例
```markdown
## 执行细节展示方案

### 展示目标
- **目标**：展示基元执行过程
- **受众**：用户
- **用途**：理解任务执行流程

### 展示方法
- **方法**：时间线图
- **节点**：表示基元状态
- **边**：表示状态转换

### 展示设计
```
输入处理 → 处理执行 → 输出生成 → 质量验证
```

### 展示实现
- **格式**：SVG图形
- **尺寸**：800x200
```

## M5-04 链条可视化

### 输入示例
链条：用户注册→用户登录→(用户信息管理‖权限管理)

### 处理过程
1. 可视化目标：展示链条结构

3. 可视化设计：节点表示基元，边表示依赖

用户注册 → 用户登录 → ┬ 用户信息管理

## M5-05 透明度报告

### 输入示例
透明度控制结果：用户可切换展示级别

### 处理过程
1. 报告目标：汇总透明度控制结果
2. 报告内容：默认隐藏策略、用户请求处理、执行细节展示、链条可视化
3. 报告格式：结构化Markdown
4. 报告生成：生成最终报告

### 输出示例
```markdown
## 透明度报告

### 默认隐藏策略
- **隐藏范围**：基元执行细节
- **隐藏级别**：中等
- **用户感知**：展示执行结果和关键节点

### 用户请求处理
- **请求类型**：展示执行细节
- **请求验证**：有效
- **请求响应**：基元执行过程时间线

### 执行细节展示
- **展示内容**：基元执行过程
- **展示格式**：时间线图

### 链条可视化
- **展示内容**：链条结构
- **展示格式**：流程图

### 控制效果
- **用户满意度**：高
- **信息过载**：低
- **可追溯性**：高
```
