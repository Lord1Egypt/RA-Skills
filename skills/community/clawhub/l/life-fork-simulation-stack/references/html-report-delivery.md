# HTML-first 交付流程

用于把人生岔路复盘报告整理成可浏览、可保存、可转交的版本。

## 默认交付层级

1. Markdown：适合对话内继续追问和版本修订。
2. HTML 用户报告：使用 `scripts/render_html_report.py` 生成，适合手机和电脑浏览，也方便保存、转交和打印。
3. 浏览器留档：用户需要留档时，直接保存 HTML 文件，或在浏览器里自行打印。

## 使用脚本

先验证结构：

```bash
python3 scripts/validate_report_structure.py input.md
```

再渲染 HTML：

```bash
python3 scripts/render_html_report.py input.md --html output/report.html
```

HTML 是默认交付结果。脚本只负责生成 HTML；打印、分享或保存由用户的浏览器完成。

## HTML 版式要求

- 首屏只展示报告名、用户问题、强判词和下滑提示。
- 第二屏用简单分叉图解释现实结果、假设结果和回补结果。
- 第一屏不放置信度、今日行动和后台依据。
- 默认展开区按顺序保留：分叉图、三种结果卡、事件冲击卡、真正放不下什么、30 天验证实验。
- 三种结果卡必须解释“它给你的、它拿走的、它最大的误判”，每张卡只讲一种结果。
- 事件冲击卡只选 4-6 张，每张写具体阶段、生活场景、现实代价、情绪代价、误判点和今天验证。
- 判断来源、外部环境、已知信息和补充问题进入人话折叠层。
- 正文按卡片和 SVG 小图分区，适合手机浏览。
- 折叠层优先使用归因小图、外部时间线图、短卡片；避免大表格。
- 质量评分和修订记录留在源 Markdown 或开发者注释，不进入用户界面。

## 浏览器留档要求

- 标题层级清楚。
- 判词和分叉图靠前。
- 默认展开区不出现后台依据、评分和开发者检查。
- 4 周低成本验证动作靠前，方便用户执行。
- 最后一节用“最后提醒”说明报告不能替用户做决定。

## 交付前检查

- 打开 HTML，检查中文是否可读。
- 运行 `scripts/validate_report_structure.py`，确认正文和附录分层通过。
- 运行 `scripts/validate_html_first_skill.py`，确认 HTML 有主分叉图、归因小图和外部时间线图。
- 手机宽度约 390px 时，检查页面没有横向溢出。
- 检查用户界面不展示质量评分和修订记录。
- 检查折叠层说明哪些信息会改写结论。
- 检查是否有命运判断、诊断、投资建议或替用户决策的表达。
