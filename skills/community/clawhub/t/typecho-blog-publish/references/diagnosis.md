## 排查步骤

### 步骤 1：检查后台编辑器模式

1. 登录 Typecho 后台
2. 打开任意一篇测试文章（如 ID 918）
3. 查看编辑器右上角是否有 "HTML"/"可视化" 切换按钮
4. 如果当前是 HTML 模式，切换到可视化模式

**预期**：切换后应该能看到渲染后的效果

### 步骤 2：检查数据库内容

```sql
-- 查看文章实际存储的内容
SELECT cid, title, text, status FROM typecho_contents WHERE cid IN (918, 919, 920, 921);
```

**目的**：确认存储的是 HTML 还是 Markdown

### 步骤 3：检查 Typecho 设置

后台 → 设置 → 编辑器：
- 默认编辑器模式：可视化 or HTML
- 是否启用 Markdown 插件

### 步骤 4：测试不同组合

已准备 4 篇测试文章，分别测试：
- HTML vs Markdown
- description 字段 vs text 字段

## 可能的解决方案

### 方案 A：Typecho 后台设置问题

**症状**：所有文章在后台都显示源码，但前台显示正常

**解决**：
1. 后台 → 设置 → 编辑器
2. 默认模式改为"可视化编辑器"
3. 或者接受后台显示源码，只要前台正常

### 方案 B：需要特定字段名

**症状**：内容字段名不对

**尝试**：
```python
# Typecho 可能使用不同的字段名
post_data = {
    'title': title,
    'text': content,  # 主要字段
    'description': content,  # 备用
    'content': content,  # 某些版本
}
```

### 方案 C：Typecho 插件冲突

**症状**：启用了 Markdown 插件导致重复解析

**解决**：
1. 后台 → 控制台 → 插件
2. 暂时禁用所有 Markdown 相关插件
3. 重新测试

### 方案 D：XML-RPC 配置问题

**症状**：XML-RPC 接口需要特殊标记

**尝试**：
```python
# 添加 Markdown 标记
post_data = {
    'title': title,
    'description': f'<!--markdown-->{content}',
}
```
