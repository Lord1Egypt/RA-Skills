## `harness/models/state.py` — Full Implementation

```python
from __future__ import annotations
from dataclasses import dataclass, field, asdict
import json
from pathlib import Path


@dataclass
class SuccessCriterion:
    id: str
    description: str
    status: str = "pending"   # pending | pass | fail
    evidence: str = ""


@dataclass
class SprintContract:
    sprint_number: int
    goal: str
    success_criteria: list[SuccessCriterion]
    out_of_scope: list[str] = field(default_factory=list)
    confirmed: bool = False

    def all_criteria_met(self) -> bool:
        return all(sc.status == "pass" for sc in self.success_criteria)

    def failing_criteria(self) -> list[SuccessCriterion]:
        return [sc for sc in self.success_criteria if sc.status == "fail"]

    def save(self, path: Path) -> None:
        path.parent.mkdir(parents=True, exist_ok=True)
        path.write_text(json.dumps(asdict(self), indent=2))


@dataclass
class HandoffState:
    sprint_number: int
    project_dir: str
    completed_features: list[str] = field(default_factory=list)
    partial_features: list[str] = field(default_factory=list)
    known_broken: list[str] = field(default_factory=list)
    architecture_decisions: dict = field(default_factory=dict)
    next_steps: list[str] = field(default_factory=list)
    files_changed: list[str] = field(default_factory=list)
    run_commands: list[str] = field(default_factory=list)

    def save(self, path: Path) -> None:
        path.parent.mkdir(parents=True, exist_ok=True)
        path.write_text(json.dumps(asdict(self), indent=2))

    @classmethod
    def load(cls, path: Path) -> "HandoffState":
        data = json.loads(path.read_text())
        return cls(**{k: v for k, v in data.items() if k in cls.__dataclass_fields__})


@dataclass
class EvalResult:
    verdict: str          # "pass" | "conditional_pass" | "fail"
    rubric_average: float
    contract_results: list[SuccessCriterion]
    feedback: str

    def save(self, path: Path) -> None:
        path.parent.mkdir(parents=True, exist_ok=True)
        path.write_text(json.dumps(asdict(self), indent=2))
```

## Project Runtime Layout

The harness treats `project_dir` as a structured run root:

```text
project_dir/
  src/                         # app code generated by Generator
  harness-state/
    spec.md
    sprints.md
    contracts/ contract-sprint-N.json
    handoffs/ handoff-sprint-N.json
    evals/ eval-sprint-N-iter-M.json
  harness-logs/
```

Use helpers in `harness.py` so every artifact lands in the same place:

```python
def ensure_project_layout(project_dir: Path) -> dict[str, Path]:
    paths = {
        "app": project_dir / "src",
        "state": project_dir / "harness-state",
        "contracts": project_dir / "harness-state" / "contracts",
        "handoffs": project_dir / "harness-state" / "handoffs",
        "evals": project_dir / "harness-state" / "evals",
        "logs": project_dir / "harness-logs",
    }
    for path in paths.values():
        path.mkdir(parents=True, exist_ok=True)
    return paths
```

## When the Harness Writes a Handoff

The harness writes `harness-state/handoffs/handoff-sprint-N.json` at two points in
`run_sprint()`:

1. **After Generator completes** — before calling the Evaluator (always, every sprint)
2. **Mid-sprint on context pressure** — if you detect the Generator's message list is growing
   too long, interrupt and serialize current state

```python
# In harness.py run_sprint():
summary = run_generator(spec, contract, project_dir, handoff)
handoff = build_handoff(contract.sprint_number, project_dir, summary)
paths = ensure_project_layout(project_dir)
state_path = paths["handoffs"] / f"handoff-sprint-{contract.sprint_number}.json"
handoff.save(state_path)  # always write before calling Evaluator
```

## When the Harness Writes Contracts and Evals

Save every confirmed sprint contract and every evaluation result under `harness-state`:

```python
def save_contract(project_dir: Path, contract: SprintContract) -> None:
    paths = ensure_project_layout(project_dir)
    contract.save(paths["contracts"] / f"contract-sprint-{contract.sprint_number}.json")


def save_sprint_plan(project_dir: Path, sprint_plan: str) -> None:
    paths = ensure_project_layout(project_dir)
    paths["state"].joinpath("sprints.md").write_text(sprint_plan.strip())


def save_eval_result(
    project_dir: Path,
    sprint_number: int,
    iteration: int,
    result: EvalResult,
) -> None:
    paths = ensure_project_layout(project_dir)
    result.save(paths["evals"] / f"eval-sprint-{sprint_number}-iter-{iteration}.json")
```

Use `harness-state/sprints.md` for the human-readable sprint plan. If the harness needs
machine-readable seeds, either parse frontmatter from `sprints.md` or write derived JSON
contracts to `harness-state/contracts/`.

The `build_handoff()` function is a stub in `harness.py` — extend it to capture more detail
from the Generator's output by parsing its summary text or reading git status:

```python
def build_handoff(sprint_number: int, project_dir: Path, generator_summary: str) -> HandoffState:
    import subprocess

    paths = ensure_project_layout(project_dir)
    app_dir = paths["app"]

    # Get changed files from git
    result = subprocess.run(
        ["git", "diff", "--name-only", "HEAD"],
        cwd=app_dir, capture_output=True, text=True
    )
    files_changed = [f for f in result.stdout.splitlines() if f]

    # Get run commands from package.json or Makefile if present
    run_commands = ["npm run dev"]
    if (app_dir / "Makefile").exists():
        run_commands = ["make dev"]

    return HandoffState(
        sprint_number=sprint_number,
        project_dir=str(app_dir),
        completed_features=[generator_summary],
        files_changed=files_changed,
        run_commands=run_commands,
    )
```

## Generator System Prompt Section for Handoff Awareness

When resuming from a previous sprint, the harness injects the handoff state into the
Generator's system messages (Phase 3 `run_generator()`). Include this template in how
you format the handoff for injection:

```python
def format_handoff_for_prompt(handoff: HandoffState) -> str:
    lines = [f"## Handoff State — Sprint {handoff.sprint_number}"]
    if handoff.completed_features:
        lines.append("\n### Completed and verified:")
        lines.extend(f"- {f}" for f in handoff.completed_features)
    if handoff.partial_features:
        lines.append("\n### Partially completed:")
        lines.extend(f"- {f}" for f in handoff.partial_features)
    if handoff.known_broken:
        lines.append("\n### Known broken / feedback from Evaluator:")
        lines.extend(f"- {f}" for f in handoff.known_broken)
    if handoff.architecture_decisions:
        lines.append("\n### Architecture decisions (do not reverse):")
        for k, v in handoff.architecture_decisions.items():
            lines.append(f"- {k}: {v}")
    if handoff.next_steps:
        lines.append("\n### Next steps (start with step 1):")
        lines.extend(f"{i+1}. {s}" for i, s in enumerate(handoff.next_steps))
    if handoff.run_commands:
        lines.append("\n### To run the app:")
        lines.extend(f"  {c}" for c in handoff.run_commands)
    return "\n".join(lines)
```

Use it in `run_generator()`:
```python
if handoff:
    system.append({
        "type": "text",
        "text": format_handoff_for_prompt(handoff),
    })
```

## Session Resume Pattern

To resume a harness run after a crash or interruption:

```python
def resume_from_disk(project_dir: Path) -> tuple[str, HandoffState | None]:
    """Load SPEC and the latest handoff state from disk."""
    paths = ensure_project_layout(project_dir)
    spec_path = paths["state"] / "spec.md"
    if not spec_path.exists():
        raise FileNotFoundError("No harness-state/spec.md found. Run from the beginning.")
    spec = spec_path.read_text()

    # Find latest handoff file
    handoff_files = sorted(paths["handoffs"].glob("handoff-sprint-*.json"))
    if not handoff_files:
        return spec, None

    latest = handoff_files[-1]
    handoff = HandoffState.load(latest)
    print(f"Resuming from sprint {handoff.sprint_number} ({latest.name})")
    return spec, handoff
```

Add a `--resume` flag to `main()` in `harness.py`:

```python
import sys

def main():
    project_dir = Path(input("Project directory: ").strip())

    if "--resume" in sys.argv:
        spec, handoff = resume_from_disk(project_dir)
        sprint_number = (handoff.sprint_number + 1) if handoff else 1
    else:
        # ... run planner, get spec
        sprint_number = 1
        handoff = None
```

## What to Store in HandoffState

**Always include:**
- `completed_features` — what was built and confirmed working (from Generator summary)
- `known_broken` — Evaluator feedback items from failed sprints
- `run_commands` — exact commands to start the app from scratch
- `files_changed` — from `git diff --name-only HEAD`

**Include when non-obvious:**
- `architecture_decisions` — decisions a new Generator session could accidentally reverse
  - Example: `{"auth": "JWT in httpOnly cookie — XSS prevention, not localStorage"}`
  - Example: `{"deletes": "soft delete via deleted_at — audit log queries need deleted records"}`
- `next_steps` — ordered, starting with the first concrete action
- `partial_features` — anything started but not finished (prevents duplicate work)

**Do NOT include:**
- The Generator's full message history — too large, defeats the purpose
- Code snippets — read the files directly
- The SPEC — it's on disk at `harness-state/spec.md`; the harness reads it separately

## Common Mistakes

| Mistake | Effect | Fix |
|---|---|---|
| Not writing handoff before calling Evaluator | Evaluator result lost if crash | Always call `handoff.save()` before `run_evaluator()` |
| Storing code snippets in `architecture_decisions` | State file grows huge | Store the decision and file path, not the code |
| Using `HandoffState` for cross-sprint conversation history | Context blowup | Keep Generator's `messages` list scoped to one sprint call |
| Marking `completed_features` based on Generator claim, not Evaluator verdict | False state | Only move from `partial_features` to `completed_features` after a PASS |
| Not incrementing `sprint_number` on resume | Wrong handoff file loaded | Use `handoff.sprint_number + 1` as the next sprint number |
| Saving state in `project_dir/src` | Generator may edit/delete it | Keep all state under `project_dir/harness-state` |
