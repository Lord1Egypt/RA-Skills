# 结构要求清单

每种单文件产出类型的组件槽位定义，供UTOS内容轴清单法使用。标注 `[待补充]` 的字段需根据实际需求定义。

> **重要**：本技能的"结构要求"对HTML产物指HTML/CSS/JS代码结构要素；对Python CLI产物指标准库+argparse代码结构要素；对Shell脚本指Bash/PowerShell代码结构要素；对SQL脚本指DDL/DML语句结构要素。

---

## 家族级默认组装顺序与质量约束

每个产出类型的组装顺序和质量约束在下方逐条定义。当子域摘要表中标注"同家族默认"时，遵循以下定义：

| 家族 | 默认组装顺序 | 默认质量约束 |
|------|-------------|-------------|
| **HTML应用** | HTML骨架→CSS样式→核心数据结构→算法逻辑→交互处理→渲染→测试 | Chrome/Firefox/Safari最新版兼容 |
| **Python CLI** | argparse定义→核心逻辑→子命令实现→输入输出处理→错误处理→测试 | dry-run默认启用; 零第三方依赖; 错误时exit code非0 |
| **Shell脚本** | Shebang→安全设置(set -euo)→usage函数→参数解析→日志系统→主逻辑→清理→验证 | 可重复执行(幂等); dry-run模式完整; 彩色日志 |
| **SQL脚本** | 数据库声明→DROP IF EXISTS→事务BEGIN→DDL/DML→注释→COMMIT→验证查询 | 可重复执行(幂等); 约束名规范; 注释完整 |

### S2-B 经典街机

#### S2-06 俄罗斯方块
- **必选组件**: `<canvas>`游戏画布 + 分数/等级/下一个预览UI + 控制按钮
- **CSS组件**: canvas居中、UI面板布局、游戏结束遮罩、响应式画布缩放
- **JS核心模块**:
  - `Game`类：主循环(requestAnimationFrame)、状态管理(playing/paused/gameover)
  - `Board`类：棋盘数据结构(二维数组)、碰撞检测、行消除
  - `Piece`类：7种标准方块(I/O/T/S/Z/J/L)定义、旋转矩阵
  - `Input`类：键盘控制(方向键/旋转/硬降/暂停)、触摸支持
  - `Score`类：计分规则、等级递增、速度曲线、最高分(localStorage)
  - `Renderer`类：canvas绘制（方块颜色/网格/幽灵块阴影）
- **组装顺序**: HTML→CSS→Piece定义→Board逻辑→Input处理→Game主循环→Score→Renderer→测试
- **质量约束**: 60fps，移动端触摸操作
- **格式**: 单一.html文件

> - 含验证查询（执行后SELECT确认结果）
> - 脚本可重复执行不出错（幂等性设计）

### S8-A DDL结构定义

#### S8-01 建表脚本
- **必选组件**: 数据库声明 + DROP IF EXISTS + CREATE TABLE + 约束(PK/FK/UQ/CK) + 索引 + 注释 + 验证查询
- **约束组件**: PRIMARY KEY、FOREIGN KEY(含ON DELETE/UPDATE)、UNIQUE、CHECK、DEFAULT、NOT NULL
- **索引组件**: CREATE INDEX(含条件索引/部分索引)、CONCURRENTLY(PostgreSQL)、USING指定
- **幂等组件**: IF NOT EXISTS / IF EXISTS、事务包裹、验证SELECT COUNT
- **组装顺序**: 数据库声明→DROP IF EXISTS→事务BEGIN→CREATE TABLE→约束→索引→注释→COMMIT→验证查询
- **质量约束**: 可重复执行、约束名规范(表_列_类型缩写)、注释完整
- **格式**: 单一.sql文件

## 组装流程

按本文件组件清单逐项填充时的通用流程：

```
1. 确定目标ID（如S2-13数独）
     ↓
2. 读取本文件中对应条目的必选组件清单
     ↓
3. 提取必选组件清单
     ↓
4. 按组装顺序搭建骨架
   HTML: HTML结构 → CSS样式 → 核心数据结构 → 算法逻辑 → 交互处理 → 渲染 → 测试
   Python: argparse → 核心逻辑 → 子命令 → 输入输出 → 错误处理 → 测试
   Shell: Shebang → 安全设置 → usage → 参数解析 → 日志 → 主逻辑 → 清理 → 验证
   SQL: 数据库声明 → DROP IF EXISTS → 事务 → DDL/DML → 注释 → COMMIT → 验证查询
     ↓
5. 可选组件按需添加
```

**关键原则**：

| 原则 | 说明 |
|------|------|
| **骨架先行** | 先搭HTML/CSS骨架（或Python/Shell/SQL骨架），再填充逻辑 |
| **数据结构优先** | 先定义核心数据结构（棋盘数组/卡片列表/表结构等），再写渲染/查询逻辑 |
| **最小可用** | 先实现核心玩法/功能，再逐步加花 |
| **本地存储** | 凡涉及进度/设置/分数，一律用localStorage持久化（HTML产物） |
