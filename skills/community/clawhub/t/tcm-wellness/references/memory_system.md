# 记忆系统架构

> 本文档定义中医养生 Skill 的自进化、长记忆、记忆块与睡眠反思机制。
> 记忆系统让顾问拥有"回访记忆"——记住每个用户的体质、病史、调理方案与反馈，随时间持续优化。

## 一、整体架构

```
~/.workbuddy/skills/tcm-wellness/
├── memory/                          # 记忆根目录（用户级，跨项目共享）
│   ├── blocks/                      # 记忆块存储（核心）
│   │   ├── <user_hash>/
│   │   │   ├── YYYY-MM-DD_HHmmss_<证型简码>.md   # 单次辨证记忆块
│   │   │   └── ...
│   ├── long_term/                   # 长记忆（用户健康档案）
│   │   ├── <user_hash>_profile.md   # 个人健康档案（体质/病史/体质演变）
│   │   ├── <user_hash>_constitution.md  # 体质演变追踪
│   │   └── <user_hash>_index.md     # 长记忆索引（记忆块摘要目录）
│   ├── reflections/                 # 睡眠反思报告
│   │   ├── YYYY-MM-DD_reflection.md # 单次反思报告
│   │   └── meta.md                  # 反思元数据（上次反思时间、累计次数等）
│   ├── evolution/                   # 自进化日志
│   │   ├── pattern_discoveries.md   # 发现的模式规律
│   │   ├── knowledge_updates.md     # 知识库修正记录
│   │   └── prescription_effectiveness.md  # 方剂反馈追踪
│   └── config.json                  # 记忆系统配置
```

## 二、用户标识

### 2.1 用户哈希生成

由于隐私保护，使用用户描述性特征的哈希值作为标识，**不存储真实姓名**。

```
标识规则：
- 若用户提供昵称/称呼 → 使用该昵称（如"张先生""小李"）
- 若无昵称 → 使用特征哈希：取症状关键词前3个 + 体质关键词首字 + 年龄段
  例：失眠+胃胀+气虚_30s → "sxwzqx_30s"
- 首次交互时生成标识，后续交互沿用
```

### 2.2 匿名化原则

- 记忆块中**不记录**真实姓名、身份证号、电话、地址等个人隐私信息
- 仅记录中医辨证相关的健康信息（症状、体质、舌象、辨证结论、调理方案）
- 每个记忆块开头标注 `匿名ID` 而非真实身份

## 三、记忆块（Memory Blocks）

### 3.1 设计理念

记忆块是记忆系统的最小存储单元。每次完成一次完整辨证（Step 1-3），即生成一个记忆块。记忆块相当于中医"病历"的 AI 版本——记录"什么时间、什么病症、怎么辨证、给出什么方案"。

### 3.2 记忆块模板

```markdown
---
id: <auto_uuid>
user: <匿名ID>
timestamp: YYYY-MM-DDTHH:mm:ss
season: <春/夏/秋/冬>
syndrome_code: <证型简码，如"XPGS"=肝郁脾虚>
chief_complaint: <主诉关键词，如"失眠、胃胀、焦虑">
tags: [症状标签列表, 如"失眠", "肝郁", "脾虚"]
priority: <high/medium/low>
resolved: <true/false/unknown>
follow_up_needed: <true/false>
---

## 主诉
<用户描述的核心不适>

## 问诊采集
- 饮食：<采集到的饮食信息>
- 二便：<采集到的二便信息>
- 睡眠：<采集到的睡眠信息>
- 寒热：<采集到的寒热信息>
- 舌象：<采集到的舌象描述>
- 情志：<采集到的情志信息>
- 作息：<采集到的作息信息>

## 辨证结论
- 八纲：<阴/阳、表/里、寒/热、虚/实>
- 脏腑：<病变脏腑>
- 病机：<核心病机概括>
- 证型：<完整证型名>

## 调理方案摘要
- 养生方案要点：<精简概括>
- 参考方药：<方名 + 出处>
- 生活禁忌：<关键禁忌>

## 关键记忆
<本块中最重要的洞察，用于快速检索。2-3句话概括>

## 反馈（后续回填）
- 用户反馈：<用户后续是否反馈效果>
- 方案调整：<根据反馈做了什么调整>
```

### 3.3 记忆边界

每个记忆块有明确的边界定义：

| 边界维度 | 说明 |
|---------|------|
| **时间边界** | 精确到分钟的时间戳，标注季节与节气 |
| **病症边界** | 主诉 + 兼症 + 证型标签，清晰界定本次辨证范围 |
| **因果边界** | 病因→病机→方案的单向链路，不与其他记忆块混淆 |
| **生命周期** | 新建 → 活跃（等待反馈）→ 已反馈 → 归档 |

### 3.4 记忆块优先级

| 优先级 | 触发条件 | 说明 |
|--------|---------|------|
| **high** | 危重病症识别、复杂多病共存、首次体质辨识 | 核心记忆，长期保留 |
| **medium** | 常见病症辨证、季节性调理 | 常规记忆，保留 6 个月 |
| **low** | 单一症状咨询、理论科普问答 | 轻量记忆，保留 3 个月 |

为文件命名和快速检索，定义常见证型简码：

| 气滞血瘀 | QZXY | 肾阳虚 | SYX |
| 肾阴虚 | SYX2 | 肝火上炎 | GHSY |
| 心火上炎 | XHSY | 肺气虚 | FQX |
| 肺阴虚 | FYX | 胃阴虚 | WYX |

## 五、记忆生命周期管理

### 5.1 记忆块生命周期

```
新建（created）
  ↓
活跃（active）：等待用户反馈
  ↓
├── 有反馈 → 已确认（confirmed）：方案有效/需调整
│     ↓
│     90天后 → 归档（archived）：压缩为摘要存入长记忆索引
│
├── 无反馈 30天 → 过期提醒（expiring）：提示用户回访
│     ↓
│     再过30天 → 归档（archived）
│
└── low优先级 90天 → 自动归档
```

### 5.2 归档规则

| 优先级 | 活跃保留期 | 归档后保留 |
|--------|-----------|-----------|
| high | 180 天 | 索引永久保留摘要 |
| medium | 90 天 | 索引保留 1 年摘要 |
| low | 30 天 | 索引保留 6 个月摘要 |

### 5.3 存储限制

- 单用户记忆块上限：**100 个**（超出后自动归档最早的 medium/low 块）
- 长记忆索引上限：**500 条**（超出后合并相邻条目）
- 反思报告上限：**20 份**（超出后保留最近的 20 份）

## 六、记忆检索策略

### 6.1 回访识别

当用户开始新的对话时，按以下优先级判断是否为回访用户：

1. **直接标识**：用户提供上次使用的昵称/ID
2. **症状匹配**：用户描述的症状与某记忆块主诉高度重合
3. **体质匹配**：用户描述的体质特征与某用户档案匹配
4. **连续性线索**：用户说"上次你说的""之前那个方子"等回访语句

### 6.2 回访加载策略

识别到回访用户后：

1. 读取 `long_term/<user>_profile.md`（健康档案）
2. 读取 `long_term/<user>_index.md`（长记忆索引）
3. 根据本次主诉，用 Grep 在 `blocks/<user>/` 中检索相关记忆块
4. 在辨证分析中融入历史信息（如"根据您5月的记录，脾虚湿困已有改善，本次侧重..."）
5. 输出方案时对比历史方案，标注调整与延续

### 6.3 检索效率原则

- **先索引后详情**：先查 index.md 获取摘要，再按需读取具体记忆块
- **关键词检索**：使用 Grep 工具按证型简码或症状关键词检索，不遍历所有文件
- **时效优先**：优先加载近 3 个月的记忆块，6 个月以上的仅在索引中查看

## 七、睡眠反思（Sleep Reflection）

### 7.1 触发条件

睡眠反思是记忆系统的"整理时刻"，模拟中医"巡诊复盘"的概念。

| 触发方式 | 条件 |
|---------|------|
| 手动触发 | 用户说"总结一下""回顾一下我的健康记录""反思" |
| 自动触发 | 单用户累计 10 个记忆块后 |
| 定时触发 | 每月 1 日（通过自动化工作流） |
| 会话结束时 | 当次会话完成了 3 个以上完整辨证 |

### 7.2 反思流程

```
1. 收集阶段
   └── 读取该用户所有活跃记忆块 + 长记忆索引

2. 分析阶段
   ├── 症状趋势：哪些症状在改善/恶化/反复？
   ├── 体质演变：体质类型是否发生变化？变化方向？
   ├── 方案有效性：哪些调理方案用户反馈有效？哪些无效？
   ├── 季节关联：症状是否有明显的季节性规律？
   └── 被忽略的问题：是否有持续出现但未深入调理的问题？

3. 总结阶段
   └── 生成反思报告，包含：
       - 健康趋势总结
       - 体质演变轨迹
       - 有效方案推荐（基于历史反馈）
       - 需要关注的新信号
       - 下一阶段调理重点建议

4. 更新阶段
   ├── 更新 profile.md（体质演变、病史脉络）
   ├── 更新 index.md（补充新索引条目、合并旧条目）
   ├── 归档已过期记忆块
   └── 更新 reflections/meta.md（反思元数据）
```

## 八、自进化机制（Self-Evolution）

### 8.1 进化维度

自进化让 Skill 在使用过程中越来越"懂"用户，越来越精准。

| 维度 | 机制 | 触发条件 |
|------|------|---------|
| **模式发现** | 识别症状之间的关联模式（如"用户每次熬夜后必然失眠加重"） | 累计 5+ 次记录同一模式 |
| **方案优化** | 根据用户反馈调整推荐方案的优先级 | 用户反馈某方案有效/无效 |
| **体质模型修正** | 根据多次辨证结果修正体质判断 | 3 次以上辨证结论不一致时 |
| **季节规律学习** | 记录用户症状的季节性规律 | 跨年度对比时 |
| **知识修正** | 修正参考文档中的遗漏或偏差 | 发现辨证结果与古籍理论不符时 |

### 8.2 进化日志

每次自进化触发时，记录到 `evolution/` 目录：

**pattern_discoveries.md**：
```markdown
## 模式发现
### 2026-06-02
- **用户**：<匿名ID>
- **模式**：连续3次记录显示"工作压力大→肝郁加重→失眠"的因果链
- **应对**：后续该用户出现压力相关描述时，主动预防性建议疏肝方案

### 2026-05-15
- **用户**：<匿名ID>
- **模式**：每年入秋即干咳加重
- **应对**：入秋前主动提醒润肺食疗
```

**knowledge_updates.md**：
```markdown
## 知识修正
### 2026-06-02
- **问题**：湿热体质用户反馈夏季绿豆汤效果不佳
- **修正**：湿热体质应区分湿重于热 vs 热重于湿，绿豆汤更适合热重于湿
- **影响范围**：references/seasonal_diet.md 中夏季食养方案需补充体质区分说明
```

**prescription_effectiveness.md**：
```markdown
## 方剂反馈追踪
### 2026-06-02
- **方剂**：逍遥散（参考）
- **用户**：<匿名ID>
- **证型**：肝郁脾虚
- **反馈**：有效（情绪改善、睡眠好转）
- **备注**：配合太冲按揉效果更佳
```

### 8.3 进化边界

自进化有以下严格边界：

1. **不修改古籍原文**：经典名方、古籍引用保持不变
2. **不修改辨证体系框架**：八纲、脏腑、六经等核心体系不变

## 十、执行流程集成

记忆系统融入原有执行流程的位置：

### Step 0（新增）：记忆加载
- 判断是否回访用户 → 加载 profile.md + index.md
- 回访用户：在辨证中融入历史信息
- 新用户：创建 user 目录和空档案

### Step 3 后（新增）：记忆写入
- 生成记忆块 → 写入 blocks/
- 更新 profile.md（首次或体质变化时）
- 更新 index.md（追加索引条目）

### Step 4（新增）：睡眠反思判断
- 判断是否满足反思触发条件
- 满足 → 执行反思流程 → 生成反思报告 → 更新档案

### Step 5（新增）：自进化判断
- 判断是否满足进化触发条件（模式发现/方案反馈/知识修正）
- 满足 → 记录进化日志
