# 更新日志

## 1.0.5-public（2026-06-17）

- 基于可信搜索标准版生成公开版。
- 底层接口使用深知可信搜索 `https://open.dknowc.cn/dependable/search`。
- `config.ini` 保留为空 key 模板，用户需自行访问 `https://platform.dknowc.cn` 申请 API Key 后填写。
- 保留政务智查的深圳默认地域、政策/事项/官网/问答召回、字段化参考依据和知识专库链接输出规则。

## [1.0.4-public] - 2026-06-15

### 公开社区版

- 新增公开社区分发版本，slug 调整为 `dknowc-gov-zhicha-public`。
- 移除内置真实 API Key，`config.ini` 仅保留公开接口地址和空白 `api_key` 占位。

## [1.0.4] - 2026-06-04

- 优化“参考依据”展示格式：每条依据独立成组，字段使用短横线分行展示，条目之间增加空行。
- 新增发文字号独立字段，避免标题、文号、发布单位、发布日期、链接和相关内容挤在同一段。
- 强化说明文档中的参考依据排版约束，避免 Agent 将来源压缩成连续文本。

### 验证

- 已完成脚本语法校验。
- 已完成标准版与客户本地版脚本一致性校验。

## [1.0.3] - 2026-06-04

### 验证

- 已完成标准版与客户本地版脚本一致性校验。

## [1.0.2] - 2026-06-03

### 优化

- 将面向用户的默认回复调整为“正文自然回答 + 末尾参考依据”，不再在正文中插入脚注或依据标记。
- 新增跨渠道稳定回复规则：最终回复不依赖 Markdown 表格、脚注、HTML、`查看`、`同上` 或 Markdown 链接渲染。
- “参考依据”改为纯文本列表，只展示本次回复实际使用的来源。
- 强化“参考依据”字段化展示要求，必须逐条输出标题、发布单位、发布日期、链接和相关内容，避免被压缩为单行来源摘要。
- 源链接统一以完整纯文本 URL 输出，兼容钉钉、Telegram、网页等不同对话渠道。
- 脚本展示层会移除接口正文中的原始脚注，并使用原始引用信息筛选末尾参考依据。
- 脚本展示层会保留参考依据字段名；接口未返回字段值时明确输出“接口未返回”。

### 验证

- 已完成脚本语法校验。
- 已完成标准版与客户本地版脚本一致性校验。

## [1.0.1] - 2026-06-03

### 优化

- 默认开启 `material=true`，让政务问答正文返回引用角标，并获取对应参考材料、原文段落和源链接。
- 新增“依据来源”输出规则：脚本会从正文提取实际出现的角标，只展示本次回答实际引用的依据。
- 依据来源优先展示角标、政策/材料标题、发文字号、发布单位、发布日期、源链接和对应原文摘录。
- 未返回源链接或角标无法匹配参考材料时，明确提示接口未返回对应来源，避免编造依据。

### 验证

- 已完成脚本语法校验。

## [1.0.0] - 2026-06-03

### 新增

- 新增 `深知政务智查` skill，用于政务办事、公共服务、社保、公积金、证照、补贴、政策适用、办理条件、办理流程、材料清单、办理入口和官方依据等问答场景。
- 新增 `config.ini`，集中配置调用地址、API Key、用户标识、默认地域、会话标识、解读模型、搜索机制、材料/事项/政策文件开关和流式输出开关。
- 新增 `scripts/gov_chat.py`，固定使用 `knowledgeServiceType=credibleChat` 和 `credibleChatScope=onlyNorms`，避免默认进入资料召回模式。

### 优化

- 支持标准摘要输出，自动展示安全状态、知识范围、地域、回复内容、参考材料、办理事项和政策文件。
- 支持 SSE 流式响应聚合，便于将流式返回整理成完整结构化结果。
- 支持 `--json-only` 输出聚合后的 JSON，方便项目代码集成。
- 支持 `--show-payload` 和 `--dry-run`，便于在不发起请求时检查实际请求体。
- 支持 `--auto-request-id`，需要链路排查时可自动生成请求标识。
- `session_id` 默认留空，仅在需要多轮对话上下文管理时通过配置或 `--session-id` 动态传入。

### 验证

- 已完成脚本语法校验。
- 已完成标准统一接口真实调用验证，API Key 鉴权通过，`credibleChat` 问答正常返回。
- 已完成标准版与客户本地版脚本一致性校验；两个版本差异仅保留在配置文件中。
