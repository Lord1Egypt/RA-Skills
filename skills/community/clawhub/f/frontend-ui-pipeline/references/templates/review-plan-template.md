# Review Plan Template

Use this after the first UI pass, mockup, or implementation.

## 1. Review Context
- **What is being reviewed:** page / component / flow / full project
- **Current goal of the UI:**
- **Target user:**
- **Current maturity:** wireframe / mockup / prototype / implemented UI

## 2. Review Priorities
Choose the most important 2-4 first:
- Accessibility
- Usability
- Information hierarchy
- Visual consistency
- Responsive behavior
- Conversion clarity
- Interaction feedback
- Form quality
- Data density / readability

## 3. Findings

### A. Major Issues
- Problems that block clarity, trust, usability, or completion

### B. Medium Issues
- Problems that reduce polish or efficiency but do not block core use

### C. Minor Issues
- Nice-to-fix consistency or visual polish items

## 4. Checklists

### Accessibility
- Contrast
- Focus visibility
- Labels for controls
- Keyboard reachability
- Touch target size
- Reduced-motion support

### Usability
- Clear primary action
- Predictable navigation
- Reasonable information density
- Helpful empty/loading/error states
- No confusing jargon or unlabeled actions

### Visual Consistency
- Consistent spacing
- Consistent radius/shadows
- Consistent typography scale
- Consistent icon style
- Consistent CTA hierarchy

### Responsive Quality
- No horizontal overflow
- Readable layout on mobile
- Safe spacing for touch
- Important content remains visible
