# 错误输出格式（统一）

当 AI 检测到输入不满足约束时，按以下格式返回。

## 格式校验失败

```
【格式校验失败】
- 指令：{{/qa-prd | /qa-case | /qa-agent | /qa-bug | /qa-report | /qa-team}}
- 原因：{{具体原因}}
- 位置：{{出错章节或字段}}
- 建议：{{用户应采取的行动}}
```

### 示例

#### 缺少必填输入
```
【格式校验失败】
- 指令：/qa-case
- 原因：缺少必填字段「需求描述」
- 建议：请提供产品的功能需求说明，包括验收条件。格式示例：
  产品/模块：订单模块
  需求描述：用户可实时查看订单状态"已下单→已发货→已签收"
```

#### 信息不足
```
【格式校验失败】
- 指令：/qa-bug
- 原因：缺陷描述质量不达标（标题清晰度、复现步骤、环境信息 3 项不通过）
- 建议：请补全以下信息后重新提交：
  1. "没反应"的具体表现（页面无变化/报错/卡死）？
  2. 浏览器版本和测试账号？
  3. F12 Console 截图
```

#### 字段缺失
```
【格式校验失败】
- 指令：/qa-prd
- 原因：输出缺少「需澄清的问题」章节
- 位置：评审报告末尾
- 建议：AI 自查是否遗漏了该章节，补充后重新输出
```

## 使用说明
各 Prompt 中「输出格式错误返回【格式校验失败】」统一参考本模板。
