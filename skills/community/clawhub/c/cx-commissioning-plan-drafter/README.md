# cx-commissioning-plan-drafter

Draft a project Commissioning Plan aligned to ASHRAE Guideline 0-2019, LEED v4/v4.1 EA Prerequisite, or federal commissioning requirements, ready for CxA review before mechanical and electrical system installation begins.

## Overview

Commissioning Authorities (CxA, BCxP, ACG-certified), MEP engineers, and owner's representatives must produce a project-specific Commissioning Plan before building systems are installed. The Cx Plan is a contractually required deliverable under ASHRAE Guideline 0, LEED v4/v4.1 (EA Prerequisite: Fundamental Commissioning and Verification), California Title 24 NACM, and federal agency requirements (GSA, DoD UFC 3-410-11FA).

A complete Cx Plan defines: the commissioned systems scope, the Cx team with roles and responsibilities, the project schedule for all Cx activities, documentation and submittal requirements, the pre-functional checklist (PFC) framework, the functional performance test (FPT) approach, and the issues log protocol.

Without a structured Cx Plan, projects fail LEED prerequisites, experience delayed occupancy due to untested systems, and expose owners to post-occupancy mechanical failures that could have been caught during commissioning.

This skill guides the complete intake and produces a DRAFT Cx Plan labeled for CxA review and Owner acceptance before system installation begins.

## Use When

- A CxA or commissioning professional needs to draft a Commissioning Plan for a new construction or major renovation project
- A project team needs a Cx Plan to satisfy a LEED EA Prerequisite or Enhanced Commissioning credit requirement
- A federal facility project requires a Cx Plan per GSA P-100 or DoD UFC guidance
- An owner's representative needs to document the commissioning scope, team structure, and schedule before issuing construction contracts

## Out of Scope

- Full Functional Performance Test (FPT) protocols or detailed test scripts — these are separate system-specific deliverables
- Retro-commissioning (RCx) or existing building commissioning plans — this skill targets new construction
- TAB report preparation — this is a separate contractor deliverable
- Cx Report preparation — this is produced after FPT completion
- LEED Online submission documentation

## Workflow Summary

1. Project identification and commissioning scope driver (ASHRAE, LEED, federal, energy code)
2. Owner's Project Requirements (OPR) summary (energy goals, IEQ, reliability, sustainability)
3. Commissioned systems matrix (HVAC, BAS, plumbing, electrical, renewables, envelope)
4. Cx team roles and responsibilities table
5. Commissioning schedule (design, construction, occupancy phases)
6. Documentation and submittal requirements (deliverables table)
7. Pre-functional checklist (PFC) framework (triggers, blocking criteria)
8. Functional performance test (FPT) framework (authoring, execution, failure protocol, seasonal testing)
9. Issues log protocol (format, priority definitions, escalation)
10. DRAFT Cx Plan assembly with CxA and Owner acceptance block

## Output

A DRAFT Commissioning Plan with:
- Commissioned Systems Matrix table
- Cx team roles and responsibilities table
- Phase-mapped commissioning schedule
- Documentation and submittal deliverables table
- PFC framework with blocking criteria
- FPT framework with failure protocol and seasonal testing plan
- Issues log protocol with escalation path
- CxA and Owner acceptance sign-off block

All output is labeled **DRAFT — PENDING CxA REVIEW AND OWNER ACCEPTANCE** until the lead CxA reviews and the Owner accepts.
