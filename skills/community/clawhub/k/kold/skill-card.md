## Description: <br>
Guides agents building or refactoring pure HTML/CSS/JS frontends toward token-driven, accessible, responsive, dark-mode-ready UI. <br>

This skill is ready for commercial/non-commercial use. <br>

## Publisher: <br>
[cgartlab](https://clawhub.ai/user/cgartlab) <br>

### License/Terms of Use: <br>
MIT-0 <br>


## Use Case: <br>
Developers and agents use this skill when designing or refactoring pure HTML/CSS/JS interfaces, especially when they need consistent design tokens, dark mode, responsive layout, and accessibility checks. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: The skill is opinionated and may be too rigid for projects that already have a different design system or frontend framework convention. <br>
Mitigation: Use it for pure HTML/CSS/JS frontend work where token-driven styling, accessibility, dark mode, and responsive design conventions are desired; defer to a narrower project-specific skill when conventions conflict. <br>
Risk: Generated UI changes can still introduce accessibility, responsiveness, or visual hierarchy regressions. <br>
Mitigation: Review the produced markup and CSS against the skill's checklist, including named controls, image alt text, dark-mode token coverage, semantic HTML, and a 375px mobile viewport check. <br>
