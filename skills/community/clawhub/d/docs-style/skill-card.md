## Description: <br>
Core technical documentation writing principles for voice, tone, structure, and LLM-friendly patterns. Use when writing or reviewing any documentation. <br>

This skill is ready for commercial/non-commercial use. <br>

## Publisher: <br>
[anderskev](https://clawhub.ai/user/anderskev) <br>

### License/Terms of Use: <br>
MIT-0 <br>


## Use Case: <br>
Developers, documentation authors, and reviewers use this skill to choose the right Diataxis documentation type and apply consistent voice, structure, terminology, examples, and LLM-friendly writing patterns. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Generated or revised documentation can be technically inaccurate or misleading. <br>
Mitigation: Review generated documentation for technical accuracy before publishing. <br>
Risk: Documentation examples can accidentally include real credentials instead of placeholders. <br>
Mitigation: Keep credential examples as placeholders and remove real secrets before publication. <br>
