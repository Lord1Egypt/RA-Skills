# $deep-interview — 需求澄清工作流

## 触发条件
- 用户需求模糊、不完整
- 用户说"我不确定"、"帮我分析一下"
- 任务复杂度高，需要先理解再行动

## 执行流程

### Phase 1: 收集信息
```
Agent: Architect (GPT-5.5)
输入: 用户的初始需求描述
输出: 需求分析 + 澄清问题列表
```

### Phase 2: 澄清问题
```
向用户提出 3-5 个关键问题：
1. 核心目标是什么？
2. 边界条件有哪些？
3. 有哪些约束？
4. 预期输出格式？
5. 有没有参考实现？
```

### Phase 3: 需求文档
```
Agent: Architect (GPT-5.5)
输入: 用户的回答
输出: 结构化需求文档
包含:
- 功能需求
- 非功能需求
- 约束条件
- 验收标准
```

## 输出格式

```markdown
# 需求文档

## 背景
[项目背景描述]

## 功能需求
- [ ] 需求 1
- [ ] 需求 2

## 非功能需求
- 性能要求
- 安全要求
- 可维护性要求

## 约束条件
- 技术栈限制
- 时间限制
- 资源限制

## 验收标准
- [ ] 标准 1
- [ ] 标准 2
```
