# No-key fallback sources

Use these public pages when `web_search` is unavailable.

## arXiv

- https://arxiv.org/list/cs.AI/recent
- https://arxiv.org/list/cs.CL/recent
- https://arxiv.org/list/cs.CV/recent

Prefer papers with clear practical relevance to AI products, multimodal systems, agents, video generation, open-source tooling, or overseas business implications.

## GitHub

- https://github.com/trending
- https://github.com/trending/python?since=daily
- https://github.com/trending/jupyter-notebook?since=daily
- https://github.com/topics/artificial-intelligence

Prefer repos with obvious momentum, usable demos, or strong documentation.

## Hacker News

- https://news.ycombinator.com/
- https://news.ycombinator.com/newest
- https://news.ycombinator.com/front

Prefer posts with meaningful discussion depth, not just launch links.

## Hugging Face

- https://huggingface.co/models?sort=trending
- https://huggingface.co/spaces?sort=trending
- https://huggingface.co/datasets?sort=trending

Prefer models, spaces, and datasets with obvious product or workflow value.

## Mainstream tech media / official blogs

Use a small curated set instead of trying to cover the entire web:

- https://openai.com/news/
- https://www.anthropic.com/news
- https://blog.google/technology/ai/
- https://ai.meta.com/blog/
- https://www.theverge.com/ai-artificial-intelligence
- https://techcrunch.com/category/artificial-intelligence/

## Suggested fallback workflow

1. Fetch 1–2 pages from each of arXiv, GitHub, Hacker News, Hugging Face.
2. Optionally fetch 1–3 official/company/media pages if needed.
3. Pull only a few strongest items, not everything.
4. Merge repeated stories.
5. State that the run is based on curated public pages if broad search was unavailable.
