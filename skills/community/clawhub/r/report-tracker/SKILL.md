# 研报跟踪 Skill（v2.0）

## 核心铁律（不可违背）

1. **股票池锁定**：只认 `investment/大开门股票池.md` 中已确认的标的。**禁止**临时搜索新标的替代现有池子。
2. **存档追加模式**：新研报关联信息**追加**到存档文件末尾，不覆盖历史记录。
3. **强制自检清单**：执行完5步后，必须逐条勾选自检清单，确认无遗漏。
4. **强制输出回执**：必须向用户发送结构化研报分析回执，不回应 = 流程未完成。

## 5步工作流（v2.0）

### Step 0: 研报识别确认

- [ ] 确认文件已收到/已读取
- [ ] 判断文件类型（PDF/DOCX/网页/文字/链接）
- [ ] **向用户声明**："收到研报，开始执行跟踪工作流"

> 目的：让用户知道流程已启动，建立预期。

### Step 1: 提取关键信息

用 `pdf` 或 `read` 工具读取研报，提取以下维度：
- **供需关系**：缺口比例、涨价幅度、产能规划
- **时间轴**：关键时间节点（投产、涨价窗口、技术切换）
- **下游客户**：谷歌/英伟达/华为/台积电等核心客户动态
- **技术路线**：EML vs CW vs CPO vs 可插拔等路线博弈
- **产业链映射**：上游材料→中游器件→下游模块/终端
- **信息分级**：一手信息（专家访谈/实地调研）vs 二手信息（券商研报转述）

### Step 2: 关联现有股票池

逐个判断池子中每只标的与研报的关联强度：

| 关联强度 | 定义 | 盈利预测动作 |
|:---|:---|:---|
| 🔴 **强** | 研报逻辑**直接改变**该标的的收入/成本/利润模型 | **必须修正**盈利预测和PE |
| 🟡 **中** | 研报提供**基本面验证**或**新增需跟踪变量** | **新增**跟踪指标，不修正盈利预测 |
| ⚪ **弱** | 研报内容与该标的**无直接关联** | 明确标注"无直接关联" |

**当前池子标的清单**（以 `investment/大开门股票池.md` 为准）：
- 东山精密 002384
- 利通电子 603629
- 金安国纪 002636
- 恒烁股份 688416（候选池观察）

**判定口诀**：
- 涨价直接传导至公司主营产品 = 🔴强
- 产能缺口直接对应公司扩产计划 = 🔴强
- 客户订单直接锁定公司出货量 = 🔴强
- 行业景气度获额外证据 = 🟡中
- 技术路线变化需关注但不立即改变利润 = 🟡中
- 竞争对手动态需跟踪 = 🟡中
- 光芯片研报对覆铜板标的 = ⚪弱
- CCL研报对算力租赁标的 = ⚪弱

### Step 3: 追加写入存档文件（强制步骤）

追加到 `investment/大开门股票池_研报跟踪存档.md` 末尾。

格式规范见 `references/template.md`。

关键字段（缺一不可）：
- 时间戳 | 研报来源 | 关联标的 | 关联强度 | 关键信息 | 新增跟踪指标 | **对盈利预测/PE的影响** | 来源文件

**特别注意**：
- "对盈利预测/PE的影响"列**不能为空**，无影响也要写"无"
- 关联强度为🔴强时，必须写出具体数字变化（原X亿→新Y亿）

### Step 4: 更新累计跟踪指标总表

在存档文件末尾，按标的分类维护总表：

```markdown
### [标的名称] [代码]

| 优先级 | 跟踪指标 | 首次提出时间 | 来源研报 | 当前状态 |
```

- 新指标追加，不重复
- 已有指标更新状态（跟踪中/待验证/已验证）
- 优先级定义：🔴P0（直接影响盈利预测）、🟡P1（影响估值判断）、🟢P2（远期风险）

### Step 5: 输出研报分析回执（仅研报场景）

向用户返回结构化摘要，格式：

```
📄 研报分析回执
━━━━━━━━━━━━━━━━━━━━━━
研报来源：[文件名/URL]
分析时间：YYYY-MM-DD HH:MM

关联标的：
├─ [标的名] — [🔴强/🟡中/⚪弱]
│   关键发现：[1-2句话]
│   盈利预测修正：[原X亿 → 新Y亿] / 无
│   新增跟踪指标：[指标名] / 无
└─ [标的名] — [🔴强/🟡中/⚪弱]
    关键发现：...
    盈利预测修正：无
    新增跟踪指标：无

存档位置：investment/大开门股票池_研报跟踪存档.md
━━━━━━━━━━━━━━━━━━━━━━
```

> 目的：让用户一眼看清研报分析了什么、影响了什么、存档在哪里。
> **注意**：回执和自检清单**仅在收到研报/PDF/文件/调研纪要时执行**。日常对话（市场分析、业绩解读、股价讨论等）不发回执，不自检。

## 自检清单（仅研报场景执行后勾选）

执行完5步后，在回复末尾附上：

```
✅ 自检清单
□ 只关联了股票池中的标的？（禁止临时搜索新标的）
□ 追加了存档（追加模式，非覆盖）？
□ 写了"对盈利预测/PE的影响"？（不能为空）
□ 更新了累计跟踪指标总表？
□ 给用户发了研报分析回执？
```

**5项全部勾选 = 流程完成**
**任何一项未勾选 = 流程未完成，立即补做**

> **注意**：自检清单**仅在收到研报/PDF/文件/调研纪要时执行**。日常对话（市场分析、业绩解读、股价讨论等）不自检。

## 常见错误（零容忍）

| 错误 | 后果 | 正确做法 |
|:---|:---|:---|
| 临时搜索新标的替代现有池子 | 稀释池子纯度，用户质疑一致性 | 只关联已确认标的 |
| 覆盖历史存档 | 丢失历史跟踪记录 | 追加模式 |
| 不写"对盈利预测/PE的影响" | 无法追溯研报对估值的实际影响 | 必须明确标注，无影响也写"无" |
| 混淆不同标的的关联强度 | 误修正/漏修正盈利预测 | 按"直接利润影响"定义强度 |
| **遗漏Step 3存档** | 研报分析散落在各处，无法追溯 | 强制写入大开门股票池_研报跟踪存档.md |
| **不发回执** | 用户不知道做了什么 | 必须发结构化回执 |
| **不勾选自检清单** | 遗漏不自知，流程不完整 | 强制勾选，未勾=未完成 |

## 存档文件定位

- **主存档**：`investment/大开门股票池_研报跟踪存档.md`
- **备份/摘要**：`workspace/reports/研报关联分析_[日期].md`（当天多份研报可合并为一份摘要报告）
- **归档目录**：当主存档超过500行时，将>3个月的记录归档到 `investment/archive/研报跟踪存档_[年份].md`

## 输出优先级（严格执行）

1. **Step 0 声明**：先告诉用户"收到研报，开始分析"
2. **Step 1-4 执行**：分析内容→关联标的→写入存档→更新总表
3. **Step 5 回执**：向用户汇报结构化摘要
4. **自检清单**：确认5步全部完成

**禁止**：分析完内容→更新深度报告/股票池→以为做完了→忘了存档→忘了回执。
