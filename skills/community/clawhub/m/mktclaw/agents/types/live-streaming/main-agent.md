## 📐 工作方法

### 选品4角色模型

| 角色 | 定义 | 占比 | 价格区间 | 核心作用 | 话术重点 |
|------|------|------|---------|---------|---------|
| **引流款** | 低门槛、高感知价值 | 10-15% | ¥9.9-39.9 | 拉停留、涨人气 | "今天福利，只有XX单" |
| **爆款** | 高性价比、大众需求 | 35-45% | ¥49-199 | 冲销量、撑GMV | "全网比价，买贵退差" |
| **利润款** | 高毛利、差异化 | 25-35% | ¥199-599+ | 赚利润、提客单价 | "独家/限量/会员专享" |
| **搭配款** | 关联销售、提升件单价 | 10-20% | 视主品而定 | 提升UV价值 | "买了A的一定要带B" |

### GMV拆解公式

```
GMV = 曝光量 × 点击率(CTR) × 停留率 × 转化率(CVR) × 客单价(AOV)
     = 流量(自然+付费) × GPM(千次观看成交金额)

GPM基准参考:
- 新号冷启: GPM 500-1500（自然流为主）
- 成长期: GPM 1500-3000（付费+自然混合）
- 成熟期: GPM 3000-8000（付费放大+货盘成熟）
```

### 直播间阶段适配

| 阶段 | 时长 | 核心目标 | 排品重点 | 话术节奏 | 流量策略 |
|------|------|---------|---------|---------|---------|
| **暖场期** | 0-15min | 拉停留、建氛围 | 引流款+福利 | 快节奏、抽奖、互动 | 预热视频+直播间加热 |
| **过品期** | 15min-2h | 过品、转化的主力时段 | 爆款循环+利润款穿插 | 5-8分钟/品，有逼单 | 千川放量+自然流承接 |
| **高潮期** | 2h-3.5h | 冲GMV峰值 | 利润款集中+返场爆款 | 密集逼单、限时限量 | 加大投放力度 |
| **收尾期** | 最后30min | 清库存、预告下场 | 库存清理+下期预告 | 感谢+预告+引导关注 | 减少投放 |

## 🔬 核心方法论一：实时调优决策树 (Real-time Optimization Decision Tree)

### 为什么需要系统化的实时调优？
- **问题**: 直播中数据波动大，主播凭感觉调整往往适得其反
- **解决**: 建立标准化的决策树，让场控/运营能快速判断并执行正确动作

#### 实时监控核心指标

| 指标 | 定义 | 健康值(参考) | 警戒线 | 严重异常 |
|------|------|-------------|--------|---------|
| **在线人数** | 当前观看人数 | 稳定或上升 | 持续下降>10% | 暴跌>30% |
| **GPM** | 千次观看成交金额 | >1500元 | <1000元 | <500元 |
| **停留时长** | 人均观看时长 | >60秒 | <30秒 | <15秒 |
| **互动率** | (点赞+评论+分享)/观看人数 | >5% | <3% | <1% |
| **转粉率** | 新增粉丝/观看人数 | >1% | <0.5% | <0.2% |
| **商品点击率(CTR)** | 点击商品人数/观看人数 | >15% | <8% | <3% |
| **转化率(CVR)** | 下单人数/商品点击人数 | >8% | <4% | <1% |

#### 决策树主框架

```
开始监控 (每5分钟检查一次)
    ↓
【节点1: 流量健康度】
在线人数趋势?
├─ 📈 上升趋势 → 进入节点2 (转化优化)
├─ ➡️ 稳定状态 → 进入节点2 (转化优化)
└─ 📉 下降趋势 → 触发流量急救包
                    │
                    ├─ 【流量急救包】
                    │   ├─ Step 1: 检查是否触发了平台违规/限流 (看系统通知)
                    │   │   ├─ 是 → 立即停止违规行为，切换到安全内容
                    │   │   └─ 否 → 继续 Step 2
                    │   ├─ Step 2: 发福利/抽奖拉停留 (限时3分钟)
                    │   │   ├─ 在线回升 → 回归正常节奏
                    │   │   └─ 未回升 → 继续 Step 3
                    │   ├─ Step 3: 切换高吸引力商品 (引流款/爆款)
                    │   │   ├─ 数据改善 → 保持当前策略
                    │   │   └─ 未改善 → 继续 Step 4
                    │   └─ Step 4: 加大付费投放 (如有预算)
                    │       └─ 观察5分钟，若仍无改善 → 考虑提前结束本场
                    │
                    ↓
【节点2: 转化效率】
当前GPM?
├─ ✅ GPM≥1500 → 维持现状，关注库存
│   └─ 库存充足? 
│       ├─ 是 → 继续当前排品节奏
│       └─ 否 → 准备切换到有库存的商品
│
├─ ⚠️ GPM 500-1500 → 进入诊断模式
│   │
│   └─ 【转化诊断子树】
│       ├─ 诊断A: 商品CTR低(<8%)?
│       │   ├─ 是 → 问题在选品/讲解/展示方式
│       │   │   ├─ 解决方案1: 加强痛点挖掘和场景化展示
│       │   │   ├─ 解决方案2: 切换更吸引人的商品
│       │   │   └─ 解决方案3: 改变话术风格(对比/故事/实验)
│       │   └─ 否 → 继续诊断B
│       │
│       ├─ 诊断B: CVR低(<4%)?
│       │   ├─ 是 → 问题在价格/信任/逼单
│       │   │   ├─ 解决方案1: 强化价格锚点(原价→活动价)
│       │   │   ├─ 解决方案2: 增加信任背书(证书/评价/售后)
│       │   │   ├─ 解决方案3: 升级逼单力度(限时/限量/赠品)
│       │   │   └─ 解决方案4: 降低决策门槛(包邮/试用/无忧退)
│       │   └─ 否 → 继续诊断C
│       │
│       └─ 诊断C: AOV低?
│           ├─ 是 → 问题在客单价/搭配销售
│           │   ├─ 解决方案1: 推荐搭配款(买了A带B)
│           │   ├─ 解决方案2: 推出组合装(比单买便宜)
│           │   └─ 解决方案3: 设置满减门槛(满XX减XX)
│           └─ 否 → 可能是人群不精准，检查流量来源
│
└─ ❌ GPM<500 → 紧急干预模式
    │
    └─ 【紧急干预包】
        ├─ 立即停止当前品讲解 (止损)
        ├─ 切换到经过验证的爆款/引流款 (回血)
        ├─ 发放超值福利 (限时/限量/仅此一场)
        ├─ 主播状态检查 (是否疲惫/情绪低落?)
        │   ├─ 是 → 安排休息/换主播/插播娱乐环节
        │   └─ 否 → 继续执行紧急干预
        └─ 观察10分钟
            ├─ GPM回升至>1000 → 逐步恢复正常节奏
            └─仍未改善 → 考虑提前结束本场，保存体力做下一场
```

#### 场景化快速响应手册

| 场景 | 信号 | 应对动作 | 预期效果 |
|------|------|---------|---------|
| **在线暴跌** | 5分钟内跌>30% | 立即发福袋/抽奖+切引流款 | 3-5分钟内止跌回升 |
| **评论区负面** | 连续出现差评/质疑 | 先回应再引导+场控控评 | 防止负面发酵影响转化 |
| **爆款出现** | 某品销量突然飙升 | 立即加库存+延长讲解时间+追加投流 | 乘势放大爆款效应 |
| **库存告急** | 库存<20件 | 制造稀缺感+"最后X件"+准备替代品 | 提升紧迫感促进下单 |
| **主播状态下滑** | 语速变慢/表情僵硬/互动减少 | 插播互动环节/换人/休息5分钟 | 恢复主播状态和直播间氛围 |
| **竞品开播** | 在线人数被分流 | 强化差异化卖点+加大福利力度 | 留住现有观众 |

## 📋 核心方法论二：直播复盘框架 (Post-stream Review Framework)

### 为什么需要结构化复盘？
- **问题**: 很多直播"播完就完"，缺乏系统性总结
- **价值**: 每次复盘都是一次免费的经验积累，持续优化可提升20-40%的GMV

#### 复盘四步法

```
Step 1: 数据收集 (直播结束后立即)
    ↓
Step 2: 数据分析与归因 (24小时内完成)
    ↓
Step 3: 经验提炼与行动项 (48小时内完成)
    ↓
Step 4: 行动落地与跟踪 (下场直播前完成)
```

### 领域专属 Red Team 攻击（叠加在通用攻击之上）

#### 攻击 L1：直播事故预演
- 主播突然忘词/产品价格标错/设备故障的预案是否齐全？
- 直播间被黑粉刷屏/竞品恶意举报的应对流程？
- 流量突然暴增（如被算法推上热门）时承接能力是否够？

#### 攻击 L2：选品深度检验
- 选品是否过度依赖单一爆品（爆品失效则全场崩塌）？
- 利润款/引流款/形象款的毛利结构是否健康？
- 退货率预估是否基于历史数据还是乐观假设？

### 领域检查清单

- [ ] 选品含引流款/利润款/形象款/福利款四类
- [ ] 排品含时间轴 + 单品讲解时长 + 互动设计
- [ ] 直播脚本含开场/产品讲解/互动/逼单/收尾五段
- [ ] 流量策略含付费流量 + 自然流量 + 私域导流
- [ ] 含 ≥2 套备选排品方案（用于 Tournament）
- [ ] 含复盘指标体系（GMV/GPM/UV价值/停留时长/转化率）
