# 事实抽取规则

## 可验证事实的分类

### 优先级 P0（核心分类，初期必检）

| 分类标签 | 模式 | 示例 |
|---------|------|------|
| `funding` | 公司/项目 + 融资轮次/金额/时间/投资方 | "XX公司完成A轮融资3亿元" |
| `pricing` | 产品/服务 + 具体价格/定价策略 | "该产品定价199元/月" |
| `date` | 事件 + 具体日期/时间范围 | "2025年3月15日正式发布" |
| `event` | 主体 + 重大事件（并购、上市、合作、诉讼、监管处罚） | "XX公司于2024年被收购" |

### 优先级 P1（扩展分类，按需启用）

| 分类标签 | 模式 | 示例 |
|---------|------|------|
| `statistic` | 数据指标 + 具体数值 | "日活用户突破1亿" |
| `personnel` | 人物 + 职务变动/任命 | "张三出任CEO" |
| `product` | 产品 + 版本/功能/发布时间 | "GPT-5于2025年6月发布" |
| `ranking` | 排名/市场份额 + 具体数字 | "全球市场份额占比35%" |

### 优先级 P2（可选分类）

| 分类标签 | 模式 | 示例 |
|---------|------|------|
| `location` | 实体 + 具体地理信息 | "总部位于上海浦东" |
| `regulation` | 政策/法规 + 具体条款/生效日期 | "《数据安全法》2021年9月1日施行" |
| `other` | 不属于以上分类但包含可验证硬事实 | — |

## 抽取规则

### 必须标记为可验证事实

1. 包含**具体数字**的陈述（金额、百分比、数量、排名）
2. 包含**具体日期**的陈述（年月日、季度、上下半年 + 年份）
3. 包含**命名实体 + 动作**的陈述（公司融资、产品发布、人事任命）
4. 包含**比较关系 + 具体数值**的陈述（"比去年增长30%"）

### 不标记（跳过）

1. 纯观点/评价："市场前景广阔"、"用户体验优秀"
2. 推测/预测（无可验证基础）："预计明年将增长"
3. 定性描述（无具体数值）："发展迅速"、"规模庞大"
4. 常识性陈述："人工智能是当前热门技术领域"
5. 引号内的直接引用（应保留原文，不做事实核查）

### 边界情况处理

| 情况 | 处理 |
|------|------|
| 模糊数字（"数亿"、"近千万"） | 标记为可验证，category 按内容归类，search_query 用精确词替换模糊词 |
| 多个事实在同一句中 | 拆分为多条，各自独立编号 |
| 事实嵌套在长段落中 | 提取最小完整分句，保留上下文定位信息 |
| 条件式陈述（"如果X，则Y"） | 仅标记条件本身中的可验证部分 |
| 否定陈述（"未获得融资"） | 标记为可验证，search_query 改为肯定式 |

### 伪量化检测（pseudo_quantification）

**定义**：包含高精度百分比或数值（如 99.9%、99.99%、97.3%），但该指标在学术/行业内无统一定义或量化标准。

**识别特征**：
1. 数值精度异常高（小数点后 ≥1 位百分比）
2. 指标名称为模糊复合词（如"物理规律模拟准确度"、"语义理解能力"、"安全合规率"）
3. 该指标在标准评测基准（benchmark）中不存在

**处理规则**：
- 标记时在 category 基础上追加 `pseudo_quantification` 标签
- search_query 优先搜索该指标是否有学术/行业来源，而非仅搜索数值
- 搜索查询模板：`"{指标名}" 量化标准 学术来源` 或 `"{指标名}" benchmark definition`
- 如果搜索结果中无该指标的定义 → 判定为伪量化，附注"该指标无学术/行业统一定义，数值可疑"
