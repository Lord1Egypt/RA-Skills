# 外卖比价技能

多平台外卖/食品价格查询与对比助手。支持美团外卖、饿了么、京东、淘宝(折淘客/好单库)等平台的价格查询与对比，以及各平台红包/优惠券查询。

## 适用场景

| 场景 | 示例 |
|------|------|
| 外卖比价 | "麻辣烫哪个平台便宜"、"美团和饿了么比价" |
| 食品比价 | "可乐哪个平台便宜"、"淘宝和京东比价可乐" |
| 红包查询 | "查美团红包"、"饿了么优惠券" |
| 综合查询 | "比价外卖+查红包" |

## 工作流程

### Phase 1: 解析意图

从用户输入中提取：
1. **商品名称/食物名称**：用户想比价的具体商品（如"麻辣烫"、"可乐"、"鸡腿堡"）
2. **平台偏好**：用户指定了哪些平台（如只比美团和饿了么），默认全部
3. **查询类型**：价格对比 / 红包优惠券 / 综合（价格+优惠）
4. **定位信息**：如果用户提供了城市/区域，影响外卖价格

### Phase 2: 多平台价格搜索

用 WebSearch 依次搜索各平台价格信息：

**外卖平台（美团/饿了么/京东外卖）**：
```
搜索策略：
- "{食物名} 美团外卖 价格 {城市}" 
- "{食物名} 饿了么 价格 {城市}"
- "{食物名} 京东外卖 价格"
```

**电商平台（淘宝/京东）**（适用于食品/饮料等可配送商品）：
```
搜索策略：
- "{商品名} 淘宝 价格 折淘客"
- "{商品名} 京东 价格"
- "{商品名} 好单库 优惠"
```

**注意事项**：
- 搜索结果可能包含多个店铺/餐厅的价格，提取典型价格区间
- 外卖价格受定位影响，搜索时加入城市信息
- 优先搜索近期信息（注意时效性）
- 如果用户没有指定城市，默认搜索一线城市价格作为参考

### Phase 3: 红包/优惠券搜索

```
搜索策略：
- "美团外卖红包 2025 领取"
- "饿了么红包优惠券 2025"
- "京东外卖优惠券 2025"
- "淘宝折淘客 优惠券 {商品名}"
- "京东食品优惠券 2025"
```

提取信息：
- 红包金额/门槛
- 有效期
- 获取方式（链接/小程序/APP内领）
- 是否限新用户

### Phase 4: 数据结构化

将搜索结果整理为结构化数据，格式如下：

```json
{
  "product": "麻辣烫",
  "city": "上海",
  "query_time": "2025-01-15 18:30",
  "price_comparison": [
    {
      "platform": "美团外卖",
      "original_price": 25,
      "typical_range": "20-35",
      "delivery_fee": 3-5,
      "estimated_total": 28-30,
      "note": "多家店铺可选"
    },
    {
      "platform": "饿了么",
      "original_price": 22,
      "typical_range": "18-30",
      "delivery_fee": 2-4,
      "estimated_total": 24-26,
      "note": "部分店铺有满减"
    }
  ],
  "coupons": [
    {
      "platform": "美团外卖",
      "type": "红包",
      "amount": 5,
      "threshold": 20,
      "validity": "当日有效",
      "how_to_get": "美团APP首页领取"
    }
  ]
}
```

### Phase 5: 生成 HTML 报告

调用 `scripts/waimai_compare.py` 生成交互式HTML可视化报告。

**命令**：
```bash
python scripts/waimai_compare.py --data '<结构化JSON>' --output outputs/waimai_compare.html
```

**报告包含**：
1. 📊 价格对比表：各平台原价/优惠后价/配送费/总计对比
2. 💰 最优推荐：考虑优惠后的最低价平台标注
3. 🎫 红包/优惠券卡片：可用红包金额、门槛、领取方式
4. 📈 综合节省计算：使用最优组合可节省金额
5. 🎯 雷达图：价格/配送速度/优惠力度/品质/便利性五维评分
6. ⏰ 时效提醒：价格和优惠的时效性说明

## 交互规则

1. **信息不足时追问**：
   - 没有商品名称 → "你想比价什么食物/商品？"
   - 没有城市 → "你在哪个城市？（影响外卖价格）"
   - 没有明确意图 → "你想查价格对比还是红包优惠券？"

2. **搜索结果不可靠时**：
   - 告知用户价格是参考值，实际价格可能因定位/时段不同
   - 建议用户打开各平台APP实时确认

3. **无结果时**：
   - 尝试换搜索词（换关键词组合）
   - 如实在找不到，告知用户该商品在各平台的搜索方式

## 输出规范

- HTML报告文件输出到 `outputs/` 目录
- 报告标题格式：`{商品名}多平台比价报告`
- 必须包含时效性说明（价格可能随时变动）
- 红包/优惠券需标注获取方式和有效期
- 推荐使用 present_files 展示最终报告

## 技术依赖

- WebSearch：各平台价格和优惠券实时搜索
- Python 3.13+：数据处理和HTML报告生成
- 无需外部API配置，零门槛使用
