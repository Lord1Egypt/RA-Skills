# 交互式高质量 PPT 生产流程 v1

## 核心原则

PPT 生产不是闭门自动生成，而是交互式制作：在影响内容方向、真实素材、审美判断、业务事实、发布交付的关键节点，必须向用户确认。

## 需要用户确认的 8 个节点

### 1. 需求确认

必须确认：
- 使用场景：沙龙 / 课程 / 招商 / 培训 / 汇报；
- 受众是谁；
- 核心目标：科普、成交、招生、教学、品牌展示；
- 页数范围；
- 是否需要知识库归档；
- 是否需要交付包。

### 2. 内容大纲确认

必须确认：
- 页面数量；
- 每页主标题 / 副标题；
- 每页核心结论；
- 顺序是否正确；
- 哪些内容放页面，哪些留给讲师口述。

门禁：内容大纲未经确认，不进入视觉生产。

### 3. 制作规范确认

必须确认：
- 风格方向；
- 主色；
- Logo 使用方式；
- 页脚/页码/Part 标签规则；
- 是否沿用黑金商业风；
- 是否有品牌禁用项。

### 4. 真实素材确认

必须向用户索取/确认：
- Logo；
- 二维码；
- 产品截图；
- 讲师本人形象/视频；
- 客户案例；
- 真实数据来源；
- 录屏 / 演示视频；
- 可公开边界与授权边界。

缺素材时只能做占位，不得伪造。

### 5. 页面合同确认

每页必须明确：
- 页面任务；
- 观众必须记住什么；
- 最终可见文字；
- 主视觉来源：生图 / 真实素材 / 混合；
- 是否同构页；
- 后期元素有哪些。

### 6. 首批视觉小样确认

正式全量生成前，应先出 1–3 页样张，让用户确认：
- 风格；
- 高级感；
- 可读性；
- 真实素材处理；
- 是否符合业务表达。

### 7. 成稿审核确认

用户截图/反馈是问题样本，不是修复上限。收到用户反馈后必须扩展到全稿同类问题排查。

### 8. 上传/发布确认

涉及知识库上传、外部发送、发布、覆盖旧版本时，必须确认目标位置和版本策略；上传后必须记录 token/链接/节点。

## 测试任务也必须遵守交互门禁

即使是 smoke test / 3 页流程测试，只要会产生 PPT 标题、内容大纲、页面合同、视觉方向，就必须先提交给用户确认。

禁止用“只是测试”绕过以下门禁：

- 标题确认；
- 内容大纲确认；
- 页面核心结论确认；
- 页面合同确认；
- 首批视觉方向确认。

如果需要内部技术连通性测试，必须明确标记为“非内容测试/纯工具连通性测试”，不得使用未经用户确认的正式标题、正式内容或正式视觉方向。
