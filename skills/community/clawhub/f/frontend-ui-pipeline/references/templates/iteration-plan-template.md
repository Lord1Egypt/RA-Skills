## 3. What Needs Change
Group by impact:

### High Priority
- Issues that block clarity, usability, accessibility, conversion, or core task completion

### Medium Priority
- Issues that reduce efficiency, readability, or consistency

### Low Priority
- Visual polish, edge-case cleanup, or optional refinement

## 4. Type of Change Needed
Mark each issue as one of:
- **Strategy problem** — wrong product framing, wrong hierarchy, wrong task flow
- **Structure problem** — section order, layout, navigation, grouping
- **Implementation problem** — spacing, states, responsiveness, code-level UI polish
- **Content problem** — unclear copy, missing labels, weak CTA, poor messaging

## 5. Recommended Route
- **Polish current implementation**
- **Partially redesign selected screens/sections**
- **Return to design direction stage**
- **Re-brief the project before continuing**
