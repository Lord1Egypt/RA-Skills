## MF4 配置应用域范本

### MF4-01 配置导入范本

**必选组件**：

1. **配置文件**：

配置文件位置：`~/.workbuddy/models.json`

配置文件格式：
```json
[
  {
    "id": "agnes-2.0-flash",
    "name": "agnes-2.0-flash",
    "vendor": "Custom",
    "url": "https://apihub.agnes-ai.com/v1",
    "apiKey": "your_api_key_here",
    "supportsToolCall": true,
    "supportsImages": true,
    "supportsReasoning": true,
    "useCustomProtocol": false
  }
]
```

2. **导入工具**：

- **Windows**：使用文本编辑器（如 Notepad++、VS Code）打开配置文件
- **macOS/Linux**：使用文本编辑器（如 Vim、Nano、VS Code）打开配置文件

3. **导入步骤**：

1. 打开配置文件：`~/.workbuddy/models.json`
2. 添加新的模型配置
3. 保存文件
4. 验证配置是否正确

**可选组件**：

**导入验证**：

- [ ] 配置文件存在
- [ ] 配置格式正确
- [ ] API Key 已替换
- [ ] 接口地址可访问

**导入日志**：

```

[2026-06-19 04:25:04] 验证配置成功
[2026-06-19 04:25:05] 配置导入完成
```

**组装顺序**：配置准备 → 导入工具 → 配置导入 → 导入验证

**约束**：配置必须正确导入

**格式**：导入结果（Markdown）

### MF4-02 配置验证范本

**必选组件**：

1. **验证方法**：

| 验证项 | 验证方法 | 说明 |
|--------|---------|------|
| 配置文件存在 | 检查文件是否存在 | 使用文件系统检查 |
| 配置格式正确 | 检查 JSON 格式 | 使用 JSON 解析器 |
| API Key 有效 | 调用 API 测试 | 使用 curl 测试 |
| 接口地址可访问 | 测试网络连接 | 使用 ping 或 curl |

2. **验证步骤**：

1. 检查配置文件是否存在
2. 检查配置格式是否正确
3. 调用 API 测试 API Key 是否有效
4. 测试接口地址是否可访问

| API Key 有效 | {有效/无效} | {说明} |
| 接口地址可访问 | {可访问/不可访问} | {说明} |

**组装顺序**：验证准备 → 验证执行 → 验证结果 → 验证确认
