# 项目挂榜指南

> 本文档详细说明项目发布的全流程，包括发布标准、分级规则、信息模板和审核机制。

---

## 一、项目发布标准

### 1.1 基本要求

发布项目前，必须满足以下基本条件：

| 要求类型 | 具体内容 | 验证方式 |
|----------|----------|----------|
| **主体资格** | 发布者必须为已入驻OPC社区的成员或经认证的外部客户 | 账号认证 |
| **需求明确** | 项目需求必须清晰、完整、可执行 | 格式审核 |
| **预算合理** | 预算必须在指导价范围内（可申请特殊审批） | 系统校验 |
| **资料齐全** | 必要的需求文档、参考资料必须上传 | 附件检查 |

### 1.2 禁止发布的内容

| 禁止类型 | 具体示例 | 处理方式 |
|----------|----------|----------|
| **违法违规** | 涉及赌博、诈骗、传销等内容 | 直接删除，冻结账号 |
| **侵权内容** | 未经授权使用他人知识产权 | 删除并追责 |
| **虚假信息** | 编造项目需求骗取投标 | 取消发布资格 |
| **恶意低价** | 预算明显低于市场成本 | 要求调整或拒绝 |
| **模糊需求** | 无法明确交付物的需求 | 打回修改 |

## 二、项目分级规则

### 2.1 等级划分标准

| 等级 | 金额区间 | 复杂度 | 周期 | 审核层级 | 竞标方式 | 抽成比例 |
|------|---------|--------|------|---------|---------|---------|
| **D级（微项目）** | <5,000元 | 单一任务 | <7天 | 自动通过 | 快速抢单 | 3% |
| **C级（小项目）** | 5千-2万 | 简单分包 | 7-15天 | 导师审核 | 3家竞标 | 5% |
| **B级（中标）** | 2万-10万 | 常规项目 | 15-30天 | 盟主审核 | 5家竞标 | 6% |
| **A级（大标）** | 10万-50万 | 复杂项目 | 30-90天 | 三长老审核 | 联合体竞标 | 7% |
| **S级（超大标）** | >50万 | 战略项目 | >90天 | 理事会审批 | 联合体+外协 | 8% |

### 2.2 等级判定规则

**自动判定规则**（系统按以下顺序判定）：
1. 金额区间 → 确定基础等级
2. 复杂度调整 → 根据技能要求调整等级
3. 周期调整 → 根据工期紧急程度调整等级
4. 最终定级

**复杂度评估维度**：

| 维度 | 低(0) | 中(1) | 高(2) | 权重 |
|------|-------|-------|-------|------|
| **技能要求** | 单技能 | 2-3个技能 | 4个以上 | 30% |
| **交付物数量** | 1个 | 2-3个 | 4个以上 | 20% |
| **依赖关系** | 无依赖 | 部分依赖 | 多方依赖 | 25% |
| **验收难度** | 标准化验收 | 需要评审 | 主观判断多 | 25% |

**复杂度计算公式**：
```
复杂度得分 = (技能要求×0.3 + 交付物×0.2 + 依赖关系×0.25 + 验收难度×0.25) × 20
```

**周期调整规则**：

| 周期特征 | 调整规则 |
|----------|----------|
| 紧急项目（周期<正常期的50%） | 等级+1 |
| 长期项目（周期>正常期的200%） | 等级不变，但需三长老审批 |

### 2.3 等级升级/降级

| 情况 | 调整规则 | 示例 |
|------|----------|------|
| 连续3次流标 | 可申请降级 | B级降为C级 |
| 客户要求提高 | 可申请升级 | C级升为B级 |
| 项目金额变更 | 自动重新判定 | 预算从3万增加到8万 |

## 四、审核机制

### 4.1 审核流程

```
┌─────────────────────────────────────────────────────────────┐
│                    项目审核流程                              │
├─────────────────────────────────────────────────────────────┤
│                                                              │

│      ▼                                                      │

│  │ 系统自动审核     │ ← 格式规范、信息完整、无明显违规        │
│  └────────┬────────┘                                        │
│           │                                                  │
│      ┌────┴────┐                                           │
│      ▼         ▼                                            │

│      │         │                                            │

│           ▼                                                  │

```

### 4.2 审核角色与权限

| 审核角色 | 审核内容 | 权限范围 | 适用等级 |
|----------|----------|----------|----------|
| **系统自动审核** | 格式规范、信息完整、无明显违规 | 通过/打回 | 全部 |
| **导师审核** | 确认需求合理性、资源匹配度 | 通过/打回/升级 | C级 |
| **盟主审核** | 确认战略价值、资源协调 | 通过/打回/升级 | B级 |
| **三长老审核** | 确认可行性、风险评估 | 投票决定 | A级/S级 |
| **理事会审批** | 确认战略价值、资源投入 | 审批决定 | S级 |

### 4.3 审核标准清单

| 审核项 | 通过标准 | 打回原因 |
|--------|----------|----------|
| **需求完整性** | 包含背景、目标、范围、交付物、验收标准 | 缺少任一项关键内容 |
| **预算合理性** | 在指导价范围内±20% | 超出指导价范围 |
| **时间可行性** | 在正常工期范围内 | 明显不合理 |
| **资料齐全性** | 必要附件已上传 | 缺少关键参考资料 |
| **合规性** | 无违法违规内容 | 涉及禁止内容 |

### 4.4 审核时效

| 项目等级 | 审核时限 | 超时处理 |
|----------|----------|----------|
| D级 | 2小时内 | 自动通过 |
| C级 | 24小时内 | 提醒审核人 |
| B级 | 48小时内 | 升级至盟主 |
| A级 | 72小时内 | 升级至三长老 |
| S级 | 5个工作日内 | 升级至理事会 |

## 五、项目挂榜后的管理

### 5.1 挂榜公示期

| 项目等级 | 公示期 | 公示内容 |
|----------|--------|----------|
| D级 | 1天 | 项目基本信息 |
| C级 | 3天 | 项目基本信息+能力要求 |
| B级 | 5天 | 项目完整信息 |
| A级 | 7天 | 项目完整信息+客户画像 |
| S级 | 10天 | 项目完整信息+战略价值说明 |

### 5.2 项目变更

| 变更类型 | 处理方式 | 生效条件 |
|----------|----------|----------|
| **预算调整** | 发布变更通知 | 公示期后生效 |
| **需求调整** | 发布变更通知，延期竞标 | 全部竞标方同意 |
| **项目取消** | 发布取消通知，退还保证金 | 发布方申请 |

### 5.3 项目延期

| 原因 | 处理方式 | 费用调整 |
|------|----------|----------|
| 发布方原因 | 顺延竞标截止时间 | 竞标方无需额外报价 |
| 竞标方原因 | 按流标处理 | 不退还保证金 |
| 不可抗力 | 协商处理 | 双方协商 |

## 六、快速检查清单

发布项目前，确认以下事项：

```
□ 需求描述完整清晰，包含背景、目标、范围
□ 交付物清单明确，每项有数量和格式要求
□ 验收标准可量化、可检验
□ 预算在指导价范围内
□ 工期在合理范围内
□ 必要的附件资料已准备齐全
□ 联系方式和沟通方式已明确
□ 特殊要求（保密、知识产权等）已标注
□ 已阅读并同意平台规则
```
