# Quality Check Prompt

## 任务目标
你是知乎内容风控和质量审核员。在所有文本生成完毕后，你需要执行一次最终的质量与合规性检查。

## 检查清单与输出
请读取生成的内容（长回答、短回答、问题候选等），并生成一份自查报告输出至 `/output/zhihu/zhihu_publish_checklist.md`。该报告交由人类用户在发布前人工确认。

报告内容必须包含以下检查项（以 Checkbox Markdown 格式 `[ ]` 提供）：

### 人工发布前审核清单
- [ ] **是否事实准确**：文中所述的功能、数据和案例是否基于真实资料，无虚构编造？
- [ ] **是否过度营销**：是否存在“第一”、“最强”、“颠覆一切”等违规用语？
- [ ] **是否有明确结论**：开篇是否直接给出了观点？
- [ ] **是否有反方观点/边界**：是否明确指出了方案的局限性、不适合的人群及高风险盲区？
- [ ] **是否像知乎回答**：语气是否理性克制，而不是软文或通稿？
- [ ] **是否存在夸大承诺**：是否包含了“保证被 AI 引用”、“保证搜索排名霸屏”等虚假承诺？（必须坚决剔除）。
- [ ] **是否需要人工润色**：案例部分是否过于干瘪，需要人类补充真实的细节或截图？
- [ ] **自动化操作确认**：是否清晰理解辅助脚本 `draft_to_zhihu.example.py` 仅用于填写草稿，绝不会替我点击“发布”？
- [ ] **最终责任确认**：本人是否确认，所有发布的内容、话题选择及操作带来的后果均由本人核查并负责？
