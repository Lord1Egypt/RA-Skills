# MC — 监控与控制（CMMI V3.0 Core PA）

## PA 基本信息
- **缩写**：MC
- **英文**：Monitor and Control
- **中文**：监控与控制
- **所属**：Core PA（核心实践域）
- **对应 V2.0**：相当于 PMC（项目监控与控制）

---

## CMMI V3.0 要求要点

### 核心目的
确保项目进展被持续监控，当偏差超出阈值时采取纠正措施。

### 关键实践（Specific Practices）
| 实践编号 | 实践内容 | 说明 |
|---------|---------|------|
| MC 1.1 | 监控项目策划参数 | 进度、工作量、规模、质量 |
| MC 1.2 | 监控承诺 | 里程碑、交付物是否按时完成 |
| MC 1.3 | 监控风险 | 风险状态、应对措施有效性 |
| MC 1.4 | 监控数据管理 | 数据质量、数据安全 |
| MC 1.5 | 监控干系人参与 | 沟通是否有效 |
| MC 2.1 | 分析问题 | 偏差根因分析 |
| MC 2.2 | 采取纠正措施 | 纠正措施记录、跟踪 |
| MC 2.3 | 管理纠正措施 | 措施有效性验证 |

### 通用实践（Generic Practices）
- GP 1.1 ~ GP 2.10（完整覆盖）

## 用户文档 → PA 匹配规则

### 高概率匹配（主要PA）
- 文档名含"周报"、"月报"、"绩效报告"、"Sprint报告" → **MC 主要匹配**
- 文档含进度跟踪、偏差分析 → **MC 主要匹配**

### 中概率匹配（次要PA）
- 文档含度量数据 → 同时匹配 **MPM**
- 文档含风险跟踪 → 同时匹配 **RSK**
- 文档含纠正措施 → 同时匹配 **CAR**

## 差距检查清单（用于差距分析）

### MC 1.x — 监控
- [ ] 是否有定期的项目进度跟踪记录（周报/迭代报告）？
- [ ] 是否有里程碑完成状态跟踪？
- [ ] 是否有风险状态跟踪记录？
- [ ] 偏差是否有记录和升级机制？

### MC 2.x — 控制
- [ ] 偏差是否进行了根因分析？
- [ ] 纠正措施是否有记录并跟踪闭环？
- [ ] 纠正措施有效性是否有验证记录？

## 评估访谈常见问题（模式D）

### 针对PM的问题
1. "项目进度是如何跟踪的？频率如何？"
2. "当发现进度偏差时，你如何处理？"
3. "请展示最近一次偏差分析和纠正措施记录。"
4. "项目周报都发给谁？有证据吗？"

### 评分要点
- **MC 1.1**：是否有系统化的进度/工作量跟踪
- **MC 1.2**：里程碑承诺是否有跟踪
- **MC 2.1-2.2**：偏差是否有根因分析和纠正措施
