# 🐱 OpenClaw 多实例互联

> 让你的多个 OpenClaw 实例互相通信、传文件、共享记忆，像用一个一样。

## 这是什么？

如果你有多台机器（Mac + 服务器、家里 + 公司、多台 VPS），每台都跑了 OpenClaw，默认它们是互相不知道的。

这个项目让你的多个 OpenClaw 实例：

- 🔗 **互相通信** — 一个实例可以通过 API 直接跟另一个对话
- 📁 **互传文件** — SSH 免密传输，大小不限
- 🧠 **共享记忆** — 定期自动合并记忆文件，两边切换无感知
- 🤝 **协同工作** — 跟任何一个实例说，都能指挥其他实例

## 能做什么？

### 🗣️ 互相通信 — 一个入口，指挥所有实例

跟任何一个实例聊天，都能操控其他机器：

```
你 → Mac 实例："让服务器重启一下 nginx"
     Mac 实例 → [API] → 服务器实例执行 → 结果返回给你

你 → 服务器实例："帮我整理一下 Mac 桌面的文件"
     服务器实例 → [API] → Mac 实例执行 → 结果返回给你
```

### 📁 互传文件 — 随时随地收发

```
你 → "把服务器上的 /var/log/nginx/error.log 拉过来看看"
     Mac 实例 → [scp] → 从服务器拉文件 → 展示内容给你

你 → "把这个脚本部署到服务器上"
     Mac 实例 → [scp] → 推到服务器 → 远程执行部署
```

### 🧠 共享记忆 — 两边切换无感知

在 Mac 上聊的事，服务器也知道；在服务器上的操作，Mac 也记得。

```
周一在 Mac 上："我最近在学 Rust"
周二在服务器上："帮我写个 Rust 的 hello world"  → 它知道你在学 Rust！

每周日 00:00 自动对账：
  Mac 的记忆 + 服务器的记忆 → 叠加合并 → 两边同步到最新
```

### 🤝 协同工作 — 复杂任务多机协作

```
你："我要部署一个新项目"
     Mac 实例：在本地打包构建
           ↓ 通过 scp 传到服务器
     服务器实例：部署、启动服务、检查健康状态
           ↓ 结果回传
     Mac 实例：汇总报告给你

你："查一下我所有机器的运行状态"
     Mac 实例 → 查本地 CPU/内存/磁盘
     Mac 实例 → [API] → 服务器实例查服务器状态
     → 汇总成一份报告展示给你
```

### 🔧 互相学习 — Skills 和配置同步

```
你："服务器上也装一下 tavily 搜索"
     Mac 实例 → [API] → 告诉服务器实例 clawhub install

你："把我这边的配置同步给服务器"
     Mac 实例 → [scp] → 传配置文件 → 服务器实例应用
```

## 使用方式

### 方式一：作为 Skill 安装（推荐）

```bash
clawhub install https://github.com/<你的用户名>/openclaw-multi-instance
```

安装后，跟你的 OpenClaw 说：

> "我还有一台服务器也跑了 OpenClaw，帮我设置互联，地址是 xxx"

你的 OpenClaw 会读取 SKILL.md，一步步引导你完成设置。

### 方式二：手动搭建

按照 SKILL.md 里的步骤操作，四步搞定：

1. **开启 API** — 远程实例配置里加一行
2. **SSH 免密** — 一条命令搞定
3. **同步文件** — scp 把身份和记忆传过去
4. **设置对账** — cron 定期合并记忆

## 记忆同步原理

这是这个方案的核心——**叠加合并，永不覆盖**：

| 情况 | 处理方式 |
|------|---------|
| A 有 B 没有的记忆 | 补给 B |
| B 有 A 没有的记忆 | 补给 A |
| 两边都有相同内容 | 保留一份，去重 |
| 两边内容冲突 | 都保留，标记来源 |

默认每周日 00:00 自动执行，也可以手动触发。

## 安全提醒

- ⚠️ API Token 等于完全控制权，不要泄露
- ⚠️ 公网实例建议用 Tailscale 组网，不要直接暴露端口
- ⚠️ `peer-config.local.json5` 包含敏感信息，已在 .gitignore 中
- ⚠️ SSH key 建议加密码保护
