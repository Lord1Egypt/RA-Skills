# 改善项目定义与规划

## 项目定义流程

从诊断发现到可执行的改善项目，需要经过严谨的定义和规划。

### 项目定义5步法

```
Step 1            Step 2            Step 3            Step 4            Step 5
问题排序        →  改善方向映射     →  项目边界定义     →  效益评估         →  项目卡片输出
│                 │                 │                 │                 │
│ 按紧迫度+       │ 匹配改善方向     │ 定义范围         │ 量化预期效益      │ 输出标准化
│ 价值量评估      │ 确定方法论      │ 含/不含         │ 估算投入         │ 项目卡片
│ 确定优先级      │ (精益/数字化)    │ 里程碑          │ 投资回报        │ 归档入库
```

## 项目分类

| 项目类型 | 周期 | 投入 | 特点 | 典型例子 |
|---------|:----:|:----:|------|---------|
| **速赢(Quick Win)** | 1-3月 | 低 | 投入小见效快、建立信心 | 5S整理、SOP标准化、看板 |
| **重点改善** | 3-6月 | 中 | 系统性的改善、ROI明确 | SMED、TPM、线平衡、Andon |
| **数字化项目** | 6-12月 | 高 | 系统实施、需变革管理 | MES/WMS/APS/BI |
| **战略变革** | 12月+ | 高 | 组织级变革、影响面广 | 流程再造、新工厂规划、供应链重构 |

## 合作建议书框架

基于改善项目清单，编制合作建议书，包含以下部分：

### 建议书结构
1. **项目背景** — 调研概况、企业现状
2. **核心发现** — TOP5关键发现（痛点+数据）
3. **改善路线图** — 分阶段规划（速赢→重点→数字化）
4. **项目清单** — 详细项目卡片列表
5. **实施计划** — 甘特图、里程碑
6. **团队配置** — 我方团队构成+客户配合要求
7. **投入与回报** — 总投资、预期收益、ROI分析
8. **合作模式** — 合作范围、交付物、周期、商务

### 合作模式参考

| 模式 | 适合场景 | 优势 |
|------|---------|------|
| 诊断+方案 | 前期的合作切入点 | 低风险、建立信任 |
| 顾问辅导 | 企业有实施团队但缺方法论 | 知识转移 |
| 项目管理 | 企业需要外部PMO | 保障落地 |
| 总包实施 | 企业从方案到落地全外包 | 责任清晰 |
| 联合实施 | 双方团队共同推进 | 深度绑定 |
