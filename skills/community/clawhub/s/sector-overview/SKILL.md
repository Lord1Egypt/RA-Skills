# 行业概览 Skill

## 触发条件
- "行业分析" / "板块分析" / "sector overview" / "行业轮动" / "哪个板块有机会"
- "分析XX行业" / "XX板块怎么样"

## 输出位置
`<working_directory>/sector-overview/reports/YYYY-MM-DD-sector-{行业名}.md`

## 工作流程

### 步骤1：确定分析范围
从用户请求中提取目标行业，或默认覆盖5大集群：
- ①能源建材 / ②有色化工石油 / ③制造链条 / ④金融地产 / ⑤消费服务

### 步骤2：数据采集
使用 `neodata-financial-search` 逐行业查询：
- 各行业指数近5日/20日/60日涨跌幅
- 行业PE-TTM及历史分位（近5年）
- 行业ETF资金流向（近5日净流入/流出）
- 行业领涨股TOP3及领跌股TOP3

### 步骤3：轮动分析
- 计算各行业相对沪深300的超额收益
- 判断当前处于"进攻/防御/均衡"哪个阶段
- 识别强弱切换信号（如某行业从持续弱势突然放量走强）

### 步骤4：见底/见顶信号匹配
对照 workspace 中各集群配置文件的见底信号条件：
- 逐条检查是否触发
- 标注触发程度（强信号/弱信号/未触发）

### 步骤5：综合输出

```markdown
# 🏭 行业概览 — {YYYY-MM-DD}

## 行业涨跌排行
| 行业 | 近5日% | 近20日% | PE分位 | ETF资金 | 信号 |
|------|--------|---------|--------|---------|------|
...

## 轮动判断
- 当前市场阶段：[进攻/防御/均衡]
- 主线方向：[XX行业]
- 切换信号：[描述]

## 重点行业深度
### {行业名}
- 估值：PE xx，近5年xx%分位
- 资金：ETF近5日净流入xx万元
- 见底信号：[触发条件x已满足]
- 结论：[1句话]

## 行业配置建议
[基于轮动分析的简短建议]
```

### 步骤6：IMA 同步
如有见底/见顶信号触发，创建 IMA 笔记提醒。

## 约束
- 单行业分析控制在 500 字，全行业概览控制在 1200 字
- 数据必须标注来源（neodata-financial-search）
- 不做个股推荐
