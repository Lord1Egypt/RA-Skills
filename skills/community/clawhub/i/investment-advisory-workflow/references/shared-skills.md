# 共享 Skill 模块

投资顾问工作流的共享 Skill 模块，可在多个场景中复用。

---

## market-scan：市场扫描

**输入**：无
**输出**：市场快照（大盘/资金流/涨停池/情绪）
**复用场景**：1（市场解读）、2（事件分析）
**专家视角**：林奇（洞察发掘）

### 执行步骤

1. 获取大盘指数（上证/深证/创业板/沪深 300/科创 50）
2. 获取涨停池数据
3. 分析涨停行业分布
4. 生成市场情绪评分

### 数据源

- `data_layer.DataAPI.get_indices()` → 大盘指数
- `data_layer.providers.akshare.get_zt_pool()` → 涨停池
- `data_layer.providers.akshare.get_strong_pool()` → 强势股池

### 输出格式

```markdown
## 市场快照

| 指标 | 数值 | 含义 |
|------|------|------|
| 上证综指 | {数值}（{涨跌幅}） | {含义} |
| 沪深 300 | {数值}（{涨跌幅}） | {含义} |
| 科创 50 | {数值}（{涨跌幅}） | {含义} |
| 涨停股 | {数量} 只 | {含义} |
```

## industry-rank：行业降秩

**输入**：行业名称
**输出**：3-5 根独立生成器 + ASCII 关系图
**复用场景**：1（市场解读）、2（事件分析）、5（行为纠偏）
**专家视角**：林奇（洞察发掘）/ 芒格（逆向检查）

### 执行步骤

1. 列出行业的所有表象（品种/产品/服务）
2. 第一轮砍：砍品种留共性
3. 第二轮砍：砍到第一根线
4. 第三轮砍：砍到第二根线
5. 第四轮砍：砍到第三根线
6. 验证：用三根线推回全部现象
7. 生成 ASCII 关系图

### 输出格式

```markdown
## 降秩分析：{行业}背后有几根线？

### 推演起点
{行业描述}

### 第一轮砍：{砍的过程}
{推理过程}

### 第二轮砍：{砍的过程}
{推理过程}

### 第三轮砍：{砍的过程}
{推理过程}

### 验证：砍完还能生回来吗？
{验证过程}

### 结论
{行业}的秩 = 3 根独立的力：
{ASCII 图}
```

## plain-explain：白话说

**输入**：专业内容
**输出**：口语化解释
**复用场景**：1（市场解读）、5（行为纠偏）、6（陪伴服务）
**专家视角**：林奇（洞察发掘）

### 执行规则

1. 口语检验：读出声来，像跟朋友说话吗？
2. 零术语：专业词必须先用大白话解释
3. 短词优先：能用两个字说的不用四个字
4. 一句一事：每句只推进一步
5. 具体：名词看得见，动词有力气
6. 类比：抽象概念必须配日常类比

### 输出格式

```markdown
## {标题}

{口语化解释}

**一句话总结**：{一句话}
```
