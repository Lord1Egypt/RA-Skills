# 交互确认流程

## 概述

当从商业计划书中无法提取到完整信息时，需要通过交互向用户提问确认。本文件定义了缺口识别逻辑、提问模板和回答处理规则。

## 交互流程总览

```
┌─────────────────┐
│   文件解析       │
│   信息提取       │
└────────┬────────┘
         ↓
┌─────────────────┐
│   缺口识别       │
│   生成清单       │
└────────┬────────┘
         ↓
┌─────────────────┐
│   第1轮提问     │ ← 画布核心维度（P0）
│   (3个问题)     │
└────────┬────────┘
         ↓
┌─────────────────┐
│   第2轮提问     │ ← 画布支撑维度（P1）
│   (3个问题)     │
└────────┬────────┘
         ↓
┌─────────────────┐
│   第3轮提问     │ ← 画布渠道维度（P2）
│   (3个问题)     │
└────────┬────────┘
         ↓
┌─────────────────┐
│   第4轮提问     │ ← PESTL/SWOT补充（P3）
│   (3个问题)     │
└────────┬────────┘
         ↓
┌─────────────────┐
│   补充数据       │
│   生成报告       │
└─────────────────┘
```

## 缺口识别规则

### 缺口分类
| 类别 | 说明 | 处理方式 |
|------|------|----------|
| P0 | 核心维度缺失，影响评分 | 必须提问 |
| P1 | 支撑维度缺失，影响完整性 | 尽量提问 |
| P2 | 渠道维度缺失，影响细节 | 选择性提问 |
| P3 | 补充维度缺失，影响深度 | 可延后提问 |

### 缺口判断标准
```
如果：
- 原文中完全没有提及 → 标记为"未提取到"
- 原文中提及但不完整 → 标记为"部分提取"
- 原文中内容充分 → 标记为"已提取"
```

## 提问策略

### 第1轮提问：画布核心维度（P0）

**提问数量**：最多3个问题
**提问时机**：优先提问

#### 维度：价值主张
```
提问模板：
这份商业计划书中关于[产品/服务]的价值主张描述不够清晰。
请您帮助确认：
- 这个产品/服务主要解决客户的什么问题？
- 相比现有解决方案，有什么独特优势？
- 客户为什么愿意为此付费？

回答格式：[请简要描述]
```

#### 维度：客户细分
```
提问模板：
这份商业计划书中目标客户群体描述不够具体。
请您帮助确认：
- 您的首要目标客户是谁？（B端/B端/政府等）
- 目标客户的主要特征是什么？
- 预估市场规模大约是多少？

回答格式：[请简要描述]
```

#### 维度：收入来源
```
提问模板：
这份商业计划书中盈利模式描述不够清晰。
请您帮助确认：
- 主要收入来源是什么？（产品销售/服务费/订阅等）
- 定价策略是怎样的？
- 预计的收入规模？

回答格式：[请简要描述]
```

### 第2轮提问：画布支撑维度（P1）

**提问数量**：最多3个问题
**提问时机**：第1轮完成后

#### 维度：核心资源
```
提问模板：
请您帮助确认以下核心资源情况：
- 最关键的1-2项核心资源是什么？
- 这些资源目前是否已经具备？
- 资源壁垒主要来自哪里？

回答格式：[请简要描述]
```

#### 维度：关键业务
```
提问模板：
请您帮助确认关键业务：
- 业务开展必须做好的3件事是什么？
- 这些业务的优先级如何？
- 目前进展到什么阶段？

回答格式：[请简要描述]
```

#### 维度：重要合作
```
提问模板：
请您帮助确认合作伙伴：
- 是否已有确定的合作伙伴？
- 合作伙伴的作用是什么？
- 合作模式是怎样的？

回答格式：[请简要描述，如有]
```

### 第3轮提问：画布渠道维度（P2）

**提问数量**：最多3个问题
**提问时机**：第2轮完成后

#### 维度：渠道通路
```
提问模板：
请您帮助确认销售渠道：
- 计划通过哪些渠道触达客户？
- 哪些是主要渠道？
- 获客成本大约是多少？

回答格式：[请简要描述]
```

#### 维度：客户关系
```
提问模板：
请您帮助确认客户关系策略：
- 如何与客户建立联系和维护关系？
- 有没有客户留存或复购策略？
- 是否有相关数据（如满意度）？

回答格式：[请简要描述]
```

### 第4轮提问：PESTL/SWOT补充（P3）

**提问数量**：最多3个问题
**提问时机**：其他维度完成后

#### PESTL补充
```
提问模板：
请您帮助确认外部环境信息：
- 是否有相关政策支持或限制？
- 行业发展趋势如何？
- 主要竞争对手是谁？

回答格式：[请简要描述]
```

## 回答处理规则

### 回答记录格式
```
## 用户确认信息

### 维度：XXX
- 问题：......
- 用户回答：......
- 确认时间：[时间]

### 维度：YYY
- 问题：......
- 用户回答：......
- 确认时间：[时间]
```

### 回答处理原则
| 回答情况 | 处理方式 |
|----------|----------|
| 用户完整回答 | 直接采用，标注"用户确认" |
| 用户部分回答 | 采用已有内容，剩余标记"待补充" |
| 用户无法回答 | 标记"用户未提供"，使用保守评分 |
| 用户拒绝回答 | 标记"用户拒绝"，跳过该维度 |

### 评分影响
| 回答情况 | 评分处理 |
|----------|----------|
| 完整回答 | 基于信息正常评分 |
| 部分回答 | 评分上限下调1-2分 |
| 未回答 | 评分取该维度最低值 |
| 拒绝回答 | 不评分，标注"无法评估" |

## 提问示例

### 示例1：价值主张缺失
```
❓ 交互确认

商业计划书中关于价值主张的描述不够完整，需要您帮助确认：

**问题1**：您的产品/服务主要解决客户的什么问题？
> 请简要描述核心问题场景

**问题2**：相比现有解决方案，您的独特优势是什么？
> 请描述差异化特点

**问题3**：客户为什么愿意为此付费？
> 请描述客户价值

---
请依次回答以上问题，或跳过您不确定的问题。
```

### 示例2：客户细分模糊
```
❓ 交互确认

商业计划书中目标客户描述不够具体，需要您帮助确认：

**问题1**：您的首要目标客户是谁？
□ 企业客户（B2B）
□ 个人消费者（B2C）
□ 政府机构（B2G）
□ 其他：______

**问题2**：目标客户的主要特征是什么？
> 请简要描述（行业/规模/需求特点等）

**问题3**：预估市场规模大约是多少？
> 请提供数据或数量级估算

---
请选择或填写以上问题。
```

## 注意事项

1. **简洁明确**：每个问题不超过50字
2. **一次不超过3个**：避免用户负担过重
3. **提供选项**：尽量提供选项便于回答
4. **允许跳过**：用户可以跳过不确定的问题
5. **不催促**：用户有充足时间思考
6. **不编造**：用户未回答的内容不自行补充
7. **保守评分**：缺失信息按最低标准评分
