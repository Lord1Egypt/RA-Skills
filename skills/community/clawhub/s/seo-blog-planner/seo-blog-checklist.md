# SEO Blog Planner Quality Checklist

Verify every item before delivering the final content plan to ensure completeness and quality.

## Content Audit (7 items)
- [ ] All existing blog posts inventoried with URLs and publish dates
- [ ] Current organic traffic measured for each existing post
- [ ] Target keywords identified for each existing post
- [ ] Cannibalization issues flagged with specific keyword conflicts
- [ ] Thin or duplicate content identified with recommended actions
- [ ] Internal linking gaps documented for existing content
- [ ] Content freshness assessed — outdated posts flagged for refresh

## Keyword Research (8 items)
- [ ] Seed keywords generated from product catalog, reviews, and competitors
- [ ] Keyword universe expanded using modifier patterns
- [ ] Search volume estimated for all target keywords
- [ ] Keyword difficulty assessed with SERP analysis
- [ ] Search intent classified for every keyword (Info/Commercial/Transactional)
- [ ] Ecommerce relevance scored (1-5) for each keyword
- [ ] No keyword assigned to more than one article as primary
- [ ] Priority tiers assigned based on weighted scoring

## Topic Cluster Architecture (6 items)
- [ ] Minimum 3 topic clusters defined
- [ ] Each cluster has a clear pillar article targeting a head term
- [ ] Each cluster has 3-6 supporting articles targeting long-tail keywords
- [ ] Internal linking architecture mapped (cluster → pillar, pillar → cluster)
- [ ] Product/category page connections defined for each cluster
- [ ] No keyword overlap between clusters

## Article Briefs (9 items)
- [ ] Every planned article has a complete content brief
- [ ] Target keyword and secondary keywords specified per brief
- [ ] H2/H3 heading structure outlines all key subtopics
- [ ] Word count targets set based on competitive analysis
- [ ] Internal links specified: product pages, category pages, related posts
- [ ] Meta title drafted (under 60 characters, keyword in first half)
- [ ] Meta description drafted (under 155 characters, includes CTA)
- [ ] Content format matches search intent for the target keyword
- [ ] Schema markup type specified (Article, FAQ, HowTo)

## Editorial Calendar (6 items)
- [ ] 90-day publishing schedule created with specific dates
- [ ] Articles prioritized by impact-effort scoring
- [ ] Seasonal content opportunities identified and scheduled
- [ ] Publishing cadence is sustainable for available resources
- [ ] Quick-win articles scheduled in first 30 days
- [ ] Content production lead time accounted for in scheduling

## On-Page SEO Standards (7 items)
- [ ] Title tag format defined with keyword placement rules
- [ ] Meta description template created
- [ ] URL slug convention documented with examples
- [ ] Internal linking minimums specified per article
- [ ] Image alt text conventions established
- [ ] Schema markup requirements defined by content type
- [ ] Mobile formatting standards documented

## Product Integration (5 items)
- [ ] Every article connects to at least one product or category page
- [ ] Product links use natural, varied anchor text (not all exact-match)
- [ ] Links are placed contextually within relevant content sections
- [ ] No article feels like a forced product advertisement
- [ ] Category page links included where topically appropriate

## Performance Tracking (5 items)
- [ ] KPIs defined with specific monthly targets
- [ ] Tracking dashboard or template provided
- [ ] Article-level metrics specified (traffic, rankings, product clicks)
- [ ] Content refresh schedule established (quarterly minimum)
- [ ] Success criteria defined for scaling, holding, updating, or consolidating content

## Delivery Quality (5 items)
- [ ] All keyword data is organized in a clear, sortable format
- [ ] Cluster architecture is visually understandable
- [ ] Article briefs are actionable — a writer could start immediately
- [ ] Calendar is realistic and accounts for content production timelines
- [ ] Strategy rationale is documented — explains why, not just what
