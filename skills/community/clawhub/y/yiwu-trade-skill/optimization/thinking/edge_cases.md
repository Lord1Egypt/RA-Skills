## 一、支付场景边界情况

### 1.1 汇率剧烈波动

#### 边界情况清单

| 情况 | 触发条件 | 风险等级 | 影响 |
|------|----------|----------|------|
| 汇率单日波动>3% | USD/CNY、EUR/CNY | 🔴高 | 报价失效、利润蒸发 |
| 汇率持续下跌 | 连续3天跌幅>1% | 🔴高 | 收汇损失 |
| 汇率持续上涨 | 连续3天涨幅>1% | 🟡中 | 客户流失风险 |
| 小币种汇率异常 | TRY、RUB等 | 🔴高 | 结算困难 |
| 节假日汇率空白 | 国内节假日 | 🟡中 | 报价无参考 |

#### 解决方案

```
┌─────────────────────────────────────────────────────────────────┐
│                    汇率风险防控机制                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  1. 实时监控预警                                               │
│     ├── 汇率波动>2% → 自动推送预警通知                          │
│     ├── 24小时汇率走势图 → 用户可查看                            │
│     └── 极端波动提示 → "建议暂缓报价"                            │
│                                                                  │
│  2. 智能报价建议                                               │
│     ├── 自动计算汇率波动空间                                     │
│     ├── 提供含波动系数的报价区间                                  │
│     └── 默认有效期内置汇率保护                                   │
│                                                                  │
│  3. 分批锁汇建议                                               │
│     ├── <$5000：即期结汇                                       │
│     ├── $5000-$50000：分3批结汇                                │
│     └── >$50000：远期锁汇                                      │
│                                                                  │
│  4. 紧急应对方案                                               │
│     ├── 锁定汇率保护条款 → PI中注明                             │
│     ├── 价格有效期缩短 → 建议7天内                               │
│     └── 汇率保险产品推荐                                        │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 支付失败

#### 边界情况清单

| 情况 | 触发条件 | 风险等级 | 影响 |
|------|----------|----------|------|
| T/T转账未到账 | 超过正常时效(3-5工作日) | 🔴高 | 订单停滞 |
| 部分付款 | 金额与合同不符 | 🟡中 | 账目混乱 |
| 付款人信息不符 | 付款人与合同不一致 | 🔴高 | 诈骗风险 |
| 银行拒绝转账 | 合规审查触发 | 🔴高 | 资金冻结 |
| 第三方平台付款失败 | 信用卡、PayPal等 | 🟢低 | 客户流失 |

#### 解决方案

```
1. 付款状态追踪
   ├── 自动发送付款提示 → "请确认已转账"
   ├── 付款到账自动通知 → "恭喜，收款成功"
   └── 未到账主动查询 → "您的付款是否遇到问题"

2. 付款信息核对
   ├── 付款人姓名识别 → 核验与合同一致性
   ├── 付款金额识别 → 核对订单金额
   └── 付款银行识别 → 高风险银行预警

3. 异常处理流程
   ├── 付款人信息不符 → 要求提供授权书+关系证明
   ├── 部分付款 → 确认付款意图，明确尾款安排
   └── 付款失败 → 提供备选支付方案

4. 风控机制
   ├── 首次合作大额 → 要求预付>50%
   ├── 高风险地区付款 → 人工审核确认
   └── 频繁变更付款信息 → 触发反欺诈检查
```

#### 边界情况清单

| 情况 | 触发条件 | 风险等级 | 影响 |
|------|----------|----------|------|
| 客户拒付 | 货物到港不提货 | 🔴高 | 货物滞留 |
| PayPal争议 | 买家提起争议 | 🔴高 | 资金冻结 |
| 信用卡拒付 | chargeback | 🔴高 | 资金损失 |
| 退款金额争议 | 部分退款要求 | 🟡中 | 协商僵局 |
| 退款时效争议 | 超长账期 | 🟡中 | 资金占用 |

#### 解决方案

```
1. 预防措施
   ├── 出货前确认付款到账 → "定金+尾款到账再发货"
   ├── 保留完整沟通记录 → 对话存档、邮件备份
   ├── 货物验收标准明确 → PI中注明验收条件
   └── 第三方平台规则遵守 → PayPal/ECCI规则

2. 纠纷处理流程
   ├── 收到退款请求 → 24小时内响应
   ├── 收集证据材料 → 发票、物流单据、沟通记录
   ├── 评估解决方案 → 补发/部分退款/全额退款
   └── 达成一致 → 执行并记录

3. 备选争议解决
   ├── 平台仲裁 → 第三方介入
   ├── 银行调解 → 信用卡争议处理
   └── 法律途径 → 极端情况
```

## 二、物流场景边界情况

### 2.1 海关扣货

#### 边界情况清单

| 情况 | 触发条件 | 风险等级 | 影响 |
|------|----------|----------|------|
| 海关查验 | 随机查验 | 🟡中 | 延误3-7天 |
| 低报货值 | 申报价格异常 | 🔴高 | 罚款+扣货 |
| 货物侵权 | 知识产权问题 | 🔴高 | 没收+罚款 |
| 禁运货物 | 敏感品类 | 🔴高 | 整票扣货 |
| 单证不符 | 资料与货物不符 | 🟡中 | 延误+罚款 |

#### 解决方案

```
1. 合规预防
   ├── 货值申报合规 → 不低于实际成交价70%
   ├── 单证资料齐全 → 发票/装箱单/合同一致
   ├── 知识产权排查 → 出货前品牌筛查
   └── 清关文件规范 → 产地证/许可证

2. 查验应对
   ├── 提前准备 → 完整单证随时可查
   ├── 主动配合 → 及时响应海关要求
   ├── 专业代理 → 有经验的清关行
   └── 合理交涉 → 提供补充说明

3. 扣货处理
   ├── 原因分析 → 确认扣货原因
   ├── 资料补充 → 按海关要求补全
   ├── 申诉渠道 → 合规前提下申诉
   └── 备货方案 → 紧急补发

4. 成本分摊
   ├── 查验费预判 → 正常查验费用说明
   ├── 延误损失 → 合同中约定免责条款
   └── 不可抗力 → 保险覆盖
```

   ├── 包装规范 → 符合运输标准

   └── 预防建议 → 后续发货注意

4. 长期优化
   ├── 承运商评估 → 选择优质物流商

4. 合同约定
   ├── 交期条款 → 预留合理缓冲期
   ├── 延误责任 → 明确责任划分
   └── 不可抗力 → 免责条款约定

## 四、Agent协作场景边界情况

### 4.1 死循环

#### 边界情况清单

| 情况 | 触发条件 | 风险等级 | 影响 |
|------|----------|----------|------|
| 意图识别死循环 | 反复确认同一意图 | 🟡中 | 用户体验差 |
| 任务分发死循环 | Agent互相推诿 | 🟡中 | 任务无法完成 |
| 对话死循环 | 重复相同问题 | 🟡中 | 用户流失 |
| 决策死循环 | 反复计算同一方案 | 🟡中 | 响应超时 |
| 知识检索死循环 | 反复检索同一问题 | 🟢低 | 效率低 |

#### 解决方案

```
1. 技术层面
   ├── 对话轮次限制 → 单轮对话不超过20次
   ├── 去重机制 → 检测重复问题并跳过
   ├── 意图收敛 → 多轮确认后强制收敛
   └── 超时中断 → 超过设定时间强制结束

2. 逻辑层面
   ├── 分支限制 → 决策树不超过5层
   ├── 回退机制 → 支持返回上一步
   ├── 人工接管 → 复杂场景转人工
   └── 方案锁定 → 决策后不再更改

3. 用户层面
   ├── 清晰引导 → 明确告知用户如何操作
   ├── 进度提示 → 显示当前进度
   ├── 退出通道 → 随时可退出重试
   └── 帮助入口 → 提供手动帮助选项

4. 监控层面
   ├── 异常检测 → 监控异常对话模式
   ├── 实时告警 → 触发阈值时告警
   └── 自动恢复 → 尝试自动修复
```

| 资源访问冲突 | 同一资源竞争 | 🟢低 | 等待时间长 |
| 数据修改冲突 | 并发写入 | 🟡中 | 数据不一致 |
| 流程权限冲突 | 越权操作 | 🔴高 | 安全风险 |

#### 解决方案

```
1. 权限管理
   ├── 角色划分 → 清晰的权限层级
   ├── 操作授权 → 每次操作验证权限
   ├── 权限隔离 → 不同Agent权限独立
   └── 权限审计 → 记录权限使用

2. 冲突检测
   ├── 操作互斥 → 同一资源互斥访问
   ├── 版本控制 → 操作前检查版本
   ├── 锁定机制 → 操作时锁定资源
   └── 冲突预警 → 即将冲突时预警

3. 冲突处理
   ├── 优先级策略 → 高优先级优先
   ├── 时间戳策略 → 早操作优先
   ├── 人工裁决 → 无法自动解决时人工
   └── 操作回滚 → 冲突后回滚

4. 审计追踪
   ├── 操作日志 → 完整记录所有操作
   ├── 权限变更 → 记录权限变更历史
   ├── 异常告警 → 异常操作告警
   └── 定期审计 → 定期检查权限使用
```
