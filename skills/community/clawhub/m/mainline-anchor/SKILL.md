# 主线对齐工作法 · 技能文档

## Overview

解决"任务分岔后无法回归主线"的核心痛点。通过5段响应模板、任务分类机制、30分钟分支上限，确保多bot协作时各角色主线清晰、可持续推进。

## 核心机制

### 触发词
- 用户输入"主线对齐" / "对齐主线"

### 响应时限
- 1分钟内必须响应

### 5段响应模板

```
【主线对齐响应】

📍 主线：[一句话说明本bot的核心主线]
📍 当前阶段：[主线推进到了什么阶段 + 百分比]
📍 当前位置：[在做哪一步 / 卡在哪]

✅ 最近已完成项：[3-5条]
⏳ 待办项（按优先级）：[3-5条]

🎯 下一步建议：
A. [⭐推荐]
B. [备选]
C. [备选]

🔬 [bot名称] · YYYY-MM-DD HH:MM
```

## 任务分类原则

| 分类 | 标识 | 含义 | 处理方式 |
|------|------|------|----------|
| 主线 | 🔴 | 今天必须推进的核心任务 | 直跑，不设上限 |
| 分支 | 🟡 | 为主线服务的探索性任务 | 30分钟硬上限 |
| 后台 | 🟢 | 可延后处理的任务 | 暂搁，随时可捡起 |

### 分支任务规范
- **启动时声明**：目标 / 服务主线 / 预计耗时 / 回归触发 / 产出物
- **中途检查**：每30分钟一次"主线对齐检查"
- **结束**：主线推进计划

## session退出规范

- **必须更新 HEARTBEAT.md 收尾条目**
- 记录当前任务状态
- 记录下一步计划

## 跨bot主线冲突处理

1. 跨bot主线冲突 → 中枢bot裁决
2. 用户最新指令优先

## 验收标准

- [x] 用户输入"主线对齐"，1分钟内得到5段响应
- [x] 每次接任务必分类（🔴/🟡/🟢）
- [x] 分支任务 ≤ 30分钟，超时回归
- [x] session退出更新HEARTBEAT.md
