# 异常处理

做饭钟点工 Skill 的异常处理原则：先保证登录态，再保证地址、规格、时间格、订单参数完整一致。

## 通用错误

| 错误码 / 现象 | 展示给用户 | 处理方式 |
|---|---|---|
| 未登录 / `AUTH_REQUIRED` | 需要先授权天鹅到家账号，我会发你登录链接 | 执行 `login`，展示短链接 |
| 授权超时 / 授权取消 / 授权过期 | 授权未完成，请重新授权 | 执行 `login --force` |
| 登录失效 / Token 过期 / `10000` | 登录已过期，需要重新授权 | 执行 `login --force`，授权后重试原步骤 |
| 网络超时 | 网络请求超时，请稍后重试 | 保留已确认信息，重试当前命令 |
| 响应解析失败 | 接口响应异常，请稍后重试 | 重试一次；仍失败则停止流程 |
| 参数缺失 | 信息还不完整，需要补充后继续 | 只追问缺失的用户侧信息 |

## 地址相关错误

| 错误码 / 现象 | 展示给用户 | 处理方式 |
|---|---|---|
| 地址簿为空 | 暂未找到常用地址，请告诉我本次服务地址 | 展示地址填写模板，用户补充后调用 `poi_search` 校验 |
| POI 无结果 | 没找到这个地址，请补充小区名、楼栋、街道或地标 | 重新调用地址 POI |
| POI 多结果 | 找到多个相似地址，请选择一个 | 数字标识后让用户选 |
| 城市未开通 | 该城市暂未开通上门做饭服务 | 停止下单，建议更换城市 |
| 区域不可服务 | 该地址暂不支持上门做饭服务 | 让用户更换地址 |
| 坐标缺失 | 地址定位不完整，请补充更精确地址 | 重新 POI 搜索 |

## 规格错误

| 错误码 / 现象 | 展示给用户 | 处理方式 |
|---|---|---|
| 用户未选择规格 | 请先选择本次做饭服务规格 | 展示 1/2/3 规格档位 |
| 规格输入无效 | 请回复 1、2 或 3 选择规格 | 重新展示规格档位 |
| 规格列表为空 | 当前地址暂未找到可预约的做饭服务 | 停止下单或让用户更换地址/时间 |
| 规格已下架 | 该服务暂不可约，请选择其他服务 | 重新拉取规格列表 |
| 价格变化 | 服务价格发生变化，我会重新生成确认卡片 | 重新获取规格并生成卡片 |


## 服务时间不可用类错误

当查询时间格或生单接口返回以下类型错误时，统一按“重新选择服务时间”处理。

包括但不限于：

* 时间格为空
* 用户选择无效时间
* 服务时间不可用
* 服务时间已过期
* 未满足提前 2 小时预约
* 当前时间格库存不足
* 当前时间格已约满
* 当前时间格被占用
* 当前时间不可预约

| 错误码 / 现象              | 展示给用户                | 处理方式                     |
| --------------------- | -------------------- | ------------------------ |
| 时间格为空                 | 当前暂无可预约时间，请换个日期或稍后再试 | 重新获取时间格；仍为空则停止下单或让用户更换日期 |
| 用户选择无效时间              | 请选择列表里的可预约时间         | 重新展示可选时间格                |
| 服务时间不可用 / 当前时间不可预约    | 这个时间暂不可约，请重新选择上门时间   | 重新获取最新时间格                |
| 时间格已过期 / 未满足提前 2 小时预约 | 这个时间刚刚不可约了，请重新选择上门时间 | 重新获取最新时间格                |
| 时间格已被占用 / 已约满 / 库存不足  | 这个时间刚刚被约满了，请重新选择上门时间 | 重新获取最新时间格                |

处理规则：

1. 不得继续提交订单。
2. 不得重复使用原服务时间再次生单。
3. 必须重新查询最新可预约时间格。
4. 必须提示用户重新选择服务时间。
5. 用户重新选择时间后，重新生成订单预览卡片。
6. 用户再次明确回复“确认”后，才可以重新提交订单。

统一提示文案：

```text
这个时间刚刚不可约了，可能是已超过最晚下单时间，或时间名额已被约满。

我帮你重新看下可预约时间，请重新选择一个上门时间。
```


## 生单错误

| 错误码 / 现象 | 展示给用户 | 处理方式 |
|---|---|---|
| 未生成确认卡片就生单 | 需要先确认订单信息 | 禁止调用生单，先生成卡片 |
| 用户未确认 | 请确认后我再为你生成订单 | 等待用户确认 |
| 生单失败 | 订单生成失败，请稍后重试 | 保留确认信息，重试一次 |
| 库存不足 | 该时间已约满，请重新选择上门时间 | 重新获取时间格 |
| 支付链接为空 | 订单已生成但支付链接异常，请到天鹅到家 App 查看订单 | 展示订单号 |
| 风控拦截 | 下单信息异常，请检查地址和备注后重试 | 简化备注，确认地址后重试 |

## 登录失效自动处理

任何接口返回登录失效时，必须按以下顺序处理：

1. 停止当前业务步骤。
2. 执行 `login --force` 获取新登录短链接。
3. 展示短链接并等待用户授权。
4. 执行 `confirm_auth`。
5. 授权成功后，从失败的业务步骤继续。

不得跳过登录重试，也不得要求用户重新描述已确认的信息。

如果 `confirm_auth` 返回“本次授权已过期，请重新发起 login --force 获取新的授权链接，并引导用户重新完成授权”，应直接重新发起授权，不要继续使用旧链接。
