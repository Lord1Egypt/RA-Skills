## 一、四等级预测深度标注

每次预测输出**必须**标注当前分析的深度等级：

| 等级 | 名称 | 内容 | 输出格式示例 |
|------|------|------|-------------|
| **Level 1** | 领域识别 | 识别强/弱生命领域 | "第X宫领域处于强/弱状态" |
| **Level 2** | 时机判断 | 结合Dasha预测问题何时出现 | "在[时间段]内，X领域可能处于压力/机会期" |
| **Level 3** | 关联推测 | 行星-宫位关联分析 | "X行星在Y宫=Z类倾向（中等概率）" |
| **Level 4** | 综合验证 | 多系统交叉+分盘+Yoga | "三系统独立确认，高概率倾向，置信区间±X周" |

**默认最低等级**：Level 2。Level 1仅用于快速概览。

## 二、置信度评估标准

每条预测**必须**附带置信度：

| 置信度 | 条件 | 措辞 | 置信区间 |
|--------|------|------|---------|
| **高** | 3+独立系统（Parashara+Jaimini+KP/Tajika）交叉确认 | "高概率倾向" | ±2-4周 |
| **中** | 2个系统确认 | "中等概率倾向" | ±1-3月 |
| **低** | 仅1个系统/指标 | "低概率倾向，需更多数据确认" | ±3-6月 |
| **边界外** | 超出Jyotish精度边界 | "此维度超出星盘精度，无法判断" | — |

**独立系统列表**：
1. Parashara体系（Dasha+Transit+Yoga+Shadbala+AV）
2. Jaimini体系（Chara Karaka+Chara Dasha+Arudha）
3. KP体系（Sub-Lord+Significator）
4. Tajika体系（Varshaphala+Muntha+年主星）

```markdown
### [预测主题]

**分析等级**：Level X
**置信度**：高/中/低
**时间窗口**：[起始日期] ~ [结束日期]（±X周/月）
**激活宫位**：第X宫（[宫位名称]）从[Lagna/Chandra Lagna/AL/D9 Lagna]视角
**Dasha状态**：[当前大运]-[小运]-[微运]
**Transit触发**：[关键行星换宫/过境事件]

**承诺评估**：此盘[有/无/部分][主题]的结构性承诺
  → 依据：[Yoga/宫主星位置/Karaka状态/D9确认/ULA等]

**激活评估**：当前Dasha-Transit[是否]对齐以激活此承诺
  → 依据：[Dasha主星+Transit行星+宫位关系+多参考点]

**倾向性描述**：
  [用概率性语言描述能量场状态和时机窗口]

**精度边界声明**：
  ⚠️ 此分析描述的是[能量场/时机窗口/宫位活跃度]，
  不指定具体[项目/行业/角色/人物/事件形式]。
  具体落地取决于个人在窗口中的选择和行动。
```

## 四、禁用措辞列表

### 4.1 严格禁止

| 禁止措辞 | 原因 | 替代方案 |
|---------|------|---------|
| "完美吻合" / "100%命中" | Jyotish无此精度 | "与星盘指示方向一致" |
| "一定会..." / "肯定..." | 无确定性保证 | "高/中/低概率倾向" |
| "事业大爆发" / "事业大扩张" | 生活描述，非天象语言 | "第10宫能量在此窗口达到峰值" |
| "贵人会联系你" | 揣测具体行为 | "第9宫权威能量在此窗口活跃" |
| "你会在X月遇到Y" | 编写具体脚本 | "X月前后有[领域]的机遇窗口" |
| "AI电影/电视剧/App/[具体行业]" | 指定具体项目 | "由个人现实决定具体落地方式" |

### 4.2 谨慎使用

| 措辞 | 条件 | 约束 |
|------|------|------|
| "高概率" | 仅当3+系统交叉确认 | 必须附置信区间 |
| "倾向" | 2+系统确认 | 必须标注"中等概率" |
| 宫位现代措辞映射 | 仅用于描述能量性质 | 不用于预测具体事件 |

### 4.3 推荐措辞

| 场景 | 推荐措辞 |
|------|---------|
| 描述时机 | "X月前后，第Y宫处于高度活跃期" |
| 描述能量性质 | "此窗口的行星能量倾向于[增长/收缩/重组/扩张]" |
| 描述概率 | "中等概率倾向，±X周/月" |
| 描述不确定性 | "此维度超出星盘精度，需要结合个人现实判断" |
| 描述方向 | "多系统指向[领域]在[时间段]有显著变化倾向" |

## 五、分析流程规范

### 5.1 禁止：从Dasha反推事件

> ❌ 错误："我进入金星大运了，会结婚吗？"
> ✅ 正确：先判断婚姻承诺是否存在，再看金星Dasha是否激活它

**标准顺序**：
1. **先看承诺**（本命盘有无此主题的结构性潜能）
2. **再看激活**（当前Dasha-Transit是否对齐）
3. **最后看形式**（能量性质是什么，不指定具体形态）

### 5.2 禁止：用用户真实信息锚定星盘

- ❌ 先知道用户做AI电影，再从星盘找"依据"
- ❌ 把"协作能量场"锚定到"电视剧编剧"
- ❌ 把"Jupiter放大"锚定到"AI影片"
- ✅ 星盘只输出能量场和时机，具体项目由用户现实决定

### 5.3 强制：多系统交叉确认

- 单一指标 → 最多"低概率倾向"
- 两个系统确认 → "中等概率倾向"
- 三个系统确认 → "高概率倾向"
- 不允许在单系统确认时输出"高概率"结论

### 5.4 强制：回溯验证的诚实声明

- 回溯验证时，先输出星盘推导，再等用户提供真实事件
- 标注"命中/部分命中/未命中"
- 未命中的部分不强行解释，记录为"待研究"

## 六、输出检查清单

每次输出预测前，逐项检查：

- [ ] 是否标注了分析等级（Level 1-4）？
- [ ] 是否标注了置信度（高/中/低）？
- [ ] 是否包含承诺评估？
- [ ] 是否包含激活评估？
- [ ] 是否包含精度边界声明？
- [ ] 是否避免了禁用措辞？
- [ ] 结论是否基于2+独立系统？
- [ ] 是否避免了用用户真实信息锚定星盘？
- [ ] 时间窗口是否包含置信区间（±X）？
