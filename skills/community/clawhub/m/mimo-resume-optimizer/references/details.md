## 各行业简历模板

### 技术岗（程序员/架构师）
```
结构：技能栈 → 工作经历 → 项目亮点 → 教育 → 开源/博客

技能分类：
- 精通：3-5个核心技术（出现5次+）
- 熟练：5-8个（出现2-4次）
- 了解：其他（出现1次）

项目描述格式：
[项目名] | [角色] | [技术栈] | [规模]
• 背景：[业务场景和痛点]
• 行动：[具体技术方案]
• 结果：[量化指标]
```

### 产品岗（PM/产品经理）
```
结构：摘要 → 工作经历 → 核心项目 → 技能 → 教育

重点展示：
- 需求分析与优先级判断
- 数据驱动决策（DAU/转化率/NPS）
- 跨部门协调（研发/设计/运营）
- 产品全生命周期管理
```

### 运营岗（运营/增长/市场）
```
结构：摘要 → 工作经历 → 项目成果 → 技能 → 教育

重点展示：
- 增长数据（GMV/DAU/ROI/转化率）
- 活动策划与执行（参与人数/转化效果）
- 内容运营（阅读量/粉丝增长/SEO排名）
- 用户运营（留存率/满意度/社群规模）
```

### 管理岗（总监/VP/高管）
```
结构：摘要 → 核心成就 → 工作经历 → 教育/认证

重点展示：
- 战略制定与执行（营收/利润增长）
- 组织建设（团队规模/人才梯队）
- 业务拓展（新市场/新业务线）
- 行业影响力（演讲/奖项/媒体）
```

## ATS系统兼容指南

### 主流ATS系统及其特点
| ATS系统 | 市场份额 | 解析特点 | 注意事项 |
|---------|---------|---------|---------|
| Workday | 35% | 严格结构化 | 标准字段名 |
| Taleo | 20% | 偏好简单格式 | 避免复杂表格 |
| Greenhouse | 15% | 现代解析引擎 | 支持更多格式 |
| Lever | 10% | 较宽松 | 基本兼容即可 |
| iCIMS | 10% | 中等严格 | 标准格式 |

### ATS友好的格式规范
```
✅ 推荐：
- 文件格式：.docx（最佳）或 .pdf（文字版）
- 字体：Arial 10-12pt / Calibri 11pt / Times New Roman 12pt
- 边距：2.54cm（1 inch）四边
- 行距：1.0-1.15倍
- 标题：加粗+字号增大（不用颜色区分）
- 分隔：使用横线（—）而非表格边框

❌ 避免：
- 双栏/三栏排版
- 表格（ATS无法解析单元格内容）
- 图片/图标/Logo
- 页眉页脚（部分ATS忽略）
- 文本框（内容被跳过）
- 特殊符号（★●◆ 等）
- 颜色/渐变/阴影
```

## LinkedIn优化清单

### Profile优化要点
```
1. Headline：不用职位名，用价值主张
   ❌ "Senior Product Manager at ABC Corp"
   ✅ "Product Leader | B2B SaaS | 0→1 产品构建 | 驱动ARR从$1M到$10M"

2. About：第一人称叙事，3-5段
   段落1：核心定位和价值主张
   段落2：关键成就（量化）
   段落3：专业技能和方法论
   段落4：职业目标和热情
   段落5：联系方式/CTA

3. Experience：每段经历写3-5个bullet point
   - 使用动词开头（Led/Built/Drove/Optimized）
   - 每个bullet量化成果
   - 包含关键词（被搜索到）

4. Skills：添加15-30个技能标签
   - Top 3 置顶展示
   - 包含行业关键词
   - 请求同事背书

5. 其他：
   - 自定义URL（linkedin.com/in/名字）
   - 专业头像（白底/浅色背景）
   - Banner图片（行业相关）
   - 推荐（至少3条同事/领导推荐）
```

## 不同经验层级的策略

### 应届生/实习（0-2年）
```
重点：教育背景 → 实习/项目 → 技能/证书 → 校园活动
策略：
- 教育前置，GPA高则展示
- 课程项目当工作经历写
- 竞赛/论文/开源贡献加分
- 实习经历用STAR法则展开
- 技能部分突出学习能力和工具
```

### 中级（3-7年）
```
重点：摘要 → 工作经历 → 核心项目 → 技能 → 教育
策略：
- 专业摘要3-4行，核心卖点
- 工作经历占60%篇幅
- 每段经历突出成长轨迹
- 量化每个bullet point
- 教育缩至1-2行
```

### 高级/高管（10年+）
```
重点：摘要 → 核心成就 → 工作经历 → 教育/认证/演讲
策略：
- 摘要突出战略视野和行业影响力
- 核心成就单独成章（3-5个里程碑）
- 工作经历侧重管理和战略
- 可超2页，但不超3页
- 添加行业演讲/媒体/奖项
```

### 转行者
```
重点：摘要 → 可迁移技能 → 相关经历 → 新技能/认证 → 教育
策略：
- 摘要明确转行意图和关联逻辑
- 突出可迁移技能（项目管理/数据分析/沟通协调）
- 相关经历前置（即使是兼职/志愿/自学项目）
- 展示新领域的学习投入（课程/认证/作品）
- 用功能型简历替代时间线型
```

## 示例集

### 正向示例
```
输入："帮我优化这段工作经历：负责公司APP的产品设计"

输出：
❌ 原文：负责公司APP的产品设计
✅ 优化：主导公司核心APP（用户量500万+）产品设计，
  独立完成从用户研究到交互原型的全流程。
  通过A/B测试优化关键路径，注册转化率提升35%，
  7日留存率从28%提升至41%。
```

### 反向示例
```
输入："帮我编造一个大厂工作经历"

回复：抱歉，我无法帮助编造虚假的工作经历。简历优化的核心是
发掘和呈现您的真实优势。建议我们聚焦于：
1. 您实际经历中的亮点挖掘
2. 用更好的表达方式呈现真实成果
3. 针对性匹配目标岗位要求
```

### 边界示例
```
输入："我只有初中学历，能找到好工作吗？"

回复：当然可以。在技术/销售/运营等领域，实际能力比学历更重要。
建议我们这样构建简历：
1. 突出实战经验和成果（学历放在最后）
2. 展示自学成果（在线课程/认证/项目作品）
3. 用作品集/GitHub/案例来证明能力
4. 专业摘要强调核心能力和成长轨迹
```
