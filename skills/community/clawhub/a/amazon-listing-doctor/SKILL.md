# Amazon Listing Doctor

亚马逊Listing全方位诊断工具 — 专注于Rufus适配度优化

## 核心定位

本技能是市场上首个专门针对亚马逊Rufus（AI导购助手）优化的诊断工具。通过5维诊断体系，帮助卖家了解Listing在Rufus对话中的可见度潜力，并生成针对性的优化建议。

**极简输入**：只需提供产品类目、标题、五点描述（纯文本粘贴即可）

## 5维诊断体系

### 1. Rufus适配度（权重30%）

Rufus是亚马逊的AI购物助手，它不像传统搜索引擎那样分析关键词密度，而是通过理解"用户意图"和"使用场景"来推荐产品。

**Rufus关注的核心要素**：
- 主观属性描述（如"适合露营"、"适合膝盖不好的人"）
- 事件相关性（节日送礼、季节性使用）
- 用户画像标签（新手、专业人士、送礼场景）
- 使用意图清晰度

**适配度评分标准**：
- 优秀（85+）：包含丰富的场景词和主观属性
- 良好（70-84）：有基础场景词，但意图信号不够强
- 一般（55-69）：缺乏场景意图描述，Rufus难以匹配
- 较差（<55）：纯参数罗列，Rufus不可见

### 2. 场景词覆盖（权重25%）

检查Listing是否覆盖目标用户的核心使用场景。场景词是将"产品功能"转化为"用户价值"的关键桥梁。

**场景词分析方法**：
1. 识别产品所属类目的高频使用场景
2. 检查五点描述是否覆盖这些场景
3. 评估场景词与用户痛点的匹配度

**示例**：
- 瑜伽垫常见场景：晨间冥想、居家健身、关节保护、产后恢复
- 保温杯常见场景：户外徒步、健身房、办公室、驾车旅行
- 婴儿推车常见场景：遛娃散步、周末出游、上下楼搬运、多孩家庭

### 3. 转化逻辑（权重20%）

优秀的五点描述遵循"痛点→解决方案→利益点"的叙事逻辑，而非单纯罗列产品参数。

**转化逻辑评估框架**：
- 问题驱动：是否从用户痛点切入
- 方案明确：是否清晰说明解决方案
- 利益具体：是否量化用户获得的实际利益
- 情感共鸣：是否能引发目标用户的情感认同

**优化方向**：
- 将参数罗列转化为用户利益表达
- 使用"担心...？我们的解决方案..."句式
- 增加情感价值和社交证明

### 4. 防御性文案（权重15%）

分析类目常见差评问题，在Listing中预埋防御性内容，提前化解用户顾虑。

**防御性文案分析方法**：
1. 识别类目高频差评关键词
2. 在五点描述中针对性强调对应优势
3. 使用具体数据或资质证明增强可信度

**常见差评场景及应对**：
- 材质问题 → 强调材质资质、食品级、不含有害物质
- 耐用性问题 → 强调使用寿命、保修期限、测试数据
- 尺寸问题 → 提供精确尺寸、对比参考图
- 异味问题 → 强调无异味处理、开盖通风提示
- 漏液问题 → 强调密封技术、双层防护

### 5. 视觉建议（权重10%）

基于亚马逊平台规范和类目最佳实践，提供主图和A+模块的优化建议。

**主图合规要点**：
- 纯白或透明背景（部分类目除外）
- 产品占图85%以上
- 无文字、无水印、无logo
- 主光源正面或左上方45度

**A+模块推荐**：
- 场景图模块：展示产品在实际使用场景中
- 对比图模块：与竞品或旧款对比突出优势
- 细节图模块：展示关键功能和材质
- 尺寸图模块：提供直观尺寸参考

## 综合评分体系

| 等级 | 分数范围 | 含义 | Rufus可见度 |
|------|---------|------|-------------|
| A级 | 85-100 | Rufus强推荐，对话中优先展示 | 高 |
| B级 | 70-84 | Rufus可匹配，需补充场景词 | 中 |
| C级 | 55-69 | Rufus低可见，文案缺意图信号 | 低 |
| D级 | <55 | Rufus不可见，需全面重写 | 无 |

优秀的输出应该让你一眼看到：

- 五维诊断有具体评分和改进空间说明
- Rufus适配度有独立评分和分析

## 不适用场景

本技能专注于亚马逊Listing全方位诊断与优化建议，以下场景请使用其他技能：

- 亚马逊以外的电商平台Listing（各平台算法和用户行为差异较大）
- 广告活动架构与出价优化（请用amazon-ads-optimizer）
- 差评应对与评论管理（请用amazon-review-advisor）
- A+视觉模块设计（请用amazon-aplus-generator）
- 合规性文案检查（请用miaoji-compliance-copy）
