## Description: <br>
Create Solana-based USDC subscription checkout URLs with encoded parameters using the Tributary Payments SDK's subscription session manager. <br>

This skill is ready for commercial/non-commercial use. <br>

## Use Case: <br>
Developers and agents use this skill to generate and validate Lando checkout URLs for Solana USDC subscription payments, including line items, payment frequency, redirect URLs, recipient keys, tracking IDs, and memos. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Installing an unintended or unpinned Tributary package could change checkout behavior. <br>
Mitigation: Install only the intended Tributary package from a trusted npm source and pin or lock the dependency version. <br>
Risk: Incorrect payment parameters can route funds or subscription behavior incorrectly. <br>
Mitigation: Review generated checkout URLs before use, especially the recipient public key, fixed gateway, amount, billing frequency, success and cancel URLs, and auto-renew setting. <br>
Risk: Tracking IDs and memos may expose sensitive customer data because payment metadata can be visible or persistent. <br>
Mitigation: Avoid placing personal or sensitive customer data in tracking IDs or memos. <br>
