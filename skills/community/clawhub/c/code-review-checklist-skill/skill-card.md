## Description: <br>
Guides structured help for Code Review Checklist using clear templates, checks, and safe defaults (category: Software Quality). <br>

## Use Case: <br>
Developers and reviewers use this skill to turn a code review goal into a concise plan, numbered checklist, next actions, and optional editable draft copy. It is intended for structured software quality guidance without external scraping or improvised command execution. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Users may include passwords, keys, tokens, customer PII, or confidential logs when asking for review guidance. <br>
Mitigation: Redact secrets and minimize sensitive data before prompting; provide goals, constraints, and owned notes instead. <br>
Risk: Generated checklist guidance may be applied without human review. <br>
Mitigation: Review checklist items and draft copy for factual accuracy, reversibility, and fit to the specific repository before use. <br>
