## T0 任务协调域 范本

### 范本概述
- **域ID**: T0
- **域名称**: 任务协调域
- **任务数量**: 4
- **范本状态**: 可用

### 任务执行范本

#### T0-01 任务需求分析
**执行步骤**:
1. 接收任务请求，明确任务目标
2. 分析任务复杂度：简单/中等/复杂
3. 识别任务依赖关系：内部依赖/外部依赖
4. 评估执行环境：资源/时间/工具
5. 汇总需求分析结果

**输入示例**:
- 任务描述：创建一个用户管理系统
- 任务来源：产品经理
- 期望完成时间：2周

**输出示例**:
- 任务复杂度：复杂（预估15+步骤）
- 依赖关系：依赖数据库设计、前端界面设计
- 执行环境：需要开发环境、测试环境、部署环境
- 需求分析报告：详见附件

**注意事项**:
- 复杂度评估要客观，避免低估
- 依赖关系要全面，避免遗漏
- 执行环境要具体，避免模糊

#### T0-02 任务域选择
**执行步骤**:
1. 分析任务需求特征
2. 扫描可用任务域列表
3. 匹配任务需求与任务域能力
4. 选择最合适的任务域组合
5. 确定任务域选择理由

**输入示例**:
- 任务需求：需要项目管理、代码开发、测试验证
- 可用任务域：项目管理域、开发域、测试域、部署域

**输出示例**:
- 选择任务域：项目管理域 + 开发域 + 测试域
- 选择理由：任务需要完整的开发生命周期管理
- 任务域组合方案：项目管理域负责整体协调，开发域负责代码实现，测试域负责质量验证

**注意事项**:
- 任务域选择要基于实际需求
- 避免选择过多或过少的任务域
- 任务域组合要合理，避免重叠

#### T0-03 执行顺序确定
**执行步骤**:
1. 分析任务依赖关系
2. 识别关键路径
3. 确定执行顺序
4. 识别并行执行机会
5. 绘制执行顺序图

**输入示例**:
- 任务列表：需求分析、设计、开发、测试、部署
- 依赖关系：设计依赖需求分析，开发依赖设计，测试依赖开发，部署依赖测试

**输出示例**:
- 执行顺序：需求分析 → 设计 → 开发 → 测试 → 部署
- 关键路径：需求分析 → 设计 → 开发 → 测试 → 部署
- 并行机会：设计完成后，前端开发和后端开发可并行执行

**注意事项**:
- 依赖关系要准确，避免循环依赖
- 关键路径要识别，避免延误
- 并行机会要充分利用，提高效率

#### T0-04 结果整合与优化
**执行步骤**:
1. 收集各任务执行结果
2. 验证结果完整性
3. 整合相关结果
4. 优化整体输出
5. 汇总整合报告

**输入示例**:
- 需求分析结果：详细的需求文档
- 设计结果：系统架构图、数据库设计
- 开发结果：可运行的代码
- 测试结果：测试报告、缺陷列表

**输出示例**:
- 整合结果：完整的用户管理系统
- 优化建议：性能优化建议、安全加固建议
- 整合报告：详见附件

**注意事项**:
- 结果收集要全面，避免遗漏
- 结果验证要严格，避免质量问题
- 整合要保持一致性，避免冲突

### 范本特征摘要
- **结构特征**: 按域分组，每个域包含多个任务，每个任务有明确的执行步骤
- **风格特征**: 专业、清晰、结构化，使用表格和列表展示信息
- **逻辑特征**: 从需求分析到结果整合，逻辑清晰，步骤明确
- **格式特征**: 使用Markdown格式，包含标题、列表、表格等元素
