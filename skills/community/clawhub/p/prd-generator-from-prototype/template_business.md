# 一、公共约束（全局强制规则）
## 1.1 数值展示规范
- 金额：2位小数、右对齐、千分符
- 整数：无小数、右对齐、千分符
- 公里：3位小数（首页/小程序除外）
- 能耗/交易量：2位小数
- 非业务数据（序号）：不加分隔符

## 1.2 列表交互规范
- 行样式：选中#b8d2ff、偶数行#E0EDFF
- 表头/合计行固定，内容可滚动
- 横向滚动条锁定上次位置
- 按机构排序字段降序
- 导出：.xlsx、数值单元格为数字类型、不导出序号列

## 1.3 司机查询规范
- 统一下拉框：支持工号+姓名模糊查询、多选、首字母搜索
- 展示格式：工号 姓名
- 排序：按编号/姓名首字母升序

## 1.4 数据权限规范
- 集团用户：全量数据
- 分公司用户：本单位及以下
- 车队用户：本车队及以下

## 1.5 日期规则
- 默认月份：T-1月
- 禁止选择：当月及未来月份
- 重置逻辑：恢复默认值并自动查询

# 九、书面语言输出规范（强制约束）

> 本规范为PRD文档生成的**强制性要求**，所有输出内容必须符合以下标准。

## 9.1 禁止事项清单

| 禁止项 | 说明 | 正确做法 |
|--------|------|----------|
| Emoji图标 | 文档任何位置不得出现emoji符号 | 使用纯文字标记或编号 |
| Given-When-Then格式 | 验收条件不得使用BDD测试用例格式 | 采用自然语言描述 |
| 虚拟人设引用 | 不得出现"John审阅""AI优化"等字样 | 保持客观中立的文档语气 |
| 口语化表达 | 禁用"咱们""搞定""OK"等口语词汇 | 使用"应""需""须"等规范用语 |
| 感叹号滥用 | 正文叙述禁止使用感叹号 | 仅在警告提示文案中使用单个感叹号 |
| 叙述性竖线表格 | 指标/目标/规则/约束/验收标准/术语/附录等叙述性内容禁用`|列|列|`表格 | 使用书面段落或编号列表 |
| ASCII线框图及树形代码块 | 页面布局/弹窗结构禁用`┌─┐│└┘├┤`线框图；菜单层级/功能树/目录结构禁用`├──`/`│`树形代码块 | 均改为书面语言分层次描述组成和层级关系 |
| API响应JSON代码块 | 接口响应示例禁用```json代码块 | 使用书面语言描述字段结构、数据类型、取值含义及示例值 |
| 截图集中索引 | 严禁将界面截图/原型图集中放置在附录的"截图索引"章节中 | 图片必须直接嵌入对应功能章节正文中，使用`![描述](路径)`语法，紧邻功能说明或页面结构描述后插入 |

## 9.2 表格使用边界

**允许保留表格的场景（仅限以下3类）：**
1. 文档修订记录表
2. 数据库DDL级字段定义表（供开发建表参考）
3. API接口参数表/错误码表（供前后端联调）

**其余所有叙述性内容一律转为书面语言**，包括但不限于：
- 成功衡量指标、建设目标、不包含范围 → 分段叙述或编号列表
- 功能模块列表、菜单架构 → 加粗编号列表
- 字段定义（业务层面）→ "名称(英文名)，类型，必填，默认值，校验规则"连贯句式
- 验收标准 → "AC-编号 功能名：预期结果"自然语言格式
- 术语定义 → "**术语**：定义。"加粗格式
- 附录（权限矩阵/开放问题/风险事项/参考文档/截图索引）→ 书面段落

## 9.2 文档结构编码规则
