## Description: <br>
Sentient Memory provides a consent-governed memory governance framework for cryptographic dormancy, variance preservation, privacy safeguards, and revocable memory lifecycle controls. <br>

This skill is ready for commercial/non-commercial use. <br>

### License/Terms of Use: <br>


## Use Case: <br>
Developers, operators, and governance reviewers use this skill to guide implementation and evaluation of persistent agent memory systems that must preserve consent, revocation, privacy, and variance under adversarial or institutional pressure. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Readers may treat the documented cryptographic, sharding, differential privacy, and audit controls as implemented protections. <br>
Mitigation: Review any separate implementation before relying on those protections, and treat this skill as governance guidance until enforcement code is assessed. <br>
Risk: Persistent memory workflows can expose sensitive data if revocation, erasure, and break-glass authority are not clearly controlled. <br>
Mitigation: Define control authority, concrete thresholds, and user-controlled consent and erasure procedures before using the guidance with real sensitive memory. <br>
