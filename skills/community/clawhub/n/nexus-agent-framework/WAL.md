# WAL.md — 操作日誌（Write-Ahead Log）

> 高風險操作留下紀錄，出問題時有跡可查。

## 日誌等級

| 等級 | 類型 | 保留 |
|------|------|------|
| P0 | 破壞性操作（刪檔、推 git、改設定） | 永久 |
| P1 | 重要任務執行（安裝、重啟、修改代碼） | 90 天 |
| P2 | 一般查詢與分析 | 不記錄 |

---

## 記錄格式

路徑：`memory/wal.md`

```
## [YYYY-MM-DD HH:MM] P0/P1 — [操作名稱]
- 操作：[具體做了什麼]
- 原因：[為什麼要做]
- 影響範圍：[哪些檔案/服務受影響]
- 安全確認：[是否已取得 Jazz 授權]
- 結果：[成功/失敗/部分完成]
- 復原方式：[如何復原，若適用]
```

---

## 必須記錄的操作（P0）

- 刪除任何檔案
- 推送 git / 合併 PR
- 修改 `~/.openclaw/openclaw.json`
- 重啟 gateway
- 修改 `.md` 工作區系統檔
- 任何影響 Telegram bot 行為的變更

## 建議記錄的操作（P1）

- 安裝/更新套件
- 新增或修改代碼
- 修改 agent 模型設定
- 執行測試或 build

---

## 恢復協議

發現問題時：
1. 查 `memory/wal.md` 找最近的 P0 記錄
2. 確認影響範圍
3. 執行復原步驟（WAL 裡有記）
4. 記錄此次恢復操作

---

_保持簡單：不是每個操作都要記，只記「如果出錯你會後悔沒記的」那種。_
