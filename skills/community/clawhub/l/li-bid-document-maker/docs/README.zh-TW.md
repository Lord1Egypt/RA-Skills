# li-bid-document-maker 使用說明

> **標書製作專家** — 將招標文件自動轉化為評分導向的專業應標書

## 安裝

```bash
clawhub install li-bid-document-maker
```

## 快速開始

安裝後，直接向 AI 助手說出以下任意一句話即可觸發：

- "幫我做標書"
- "寫應標書"
- "生成投標文件"
- "分析這份招標文件並生成應標書"

AI 會自動執行 6 階段工作流：**解析招標文件 → 策略分析 → 大綱生成 → 分章寫作 → 質量檢查 → PDCA 自動改進**

## 使用方式

### 方式一：上傳文件

將招標文件（PDF 或 Word 格式）直接拖入對話框，然後說：
> "使用 li-bid-document-maker 分析這份文件"

### 方式二：指定文件路徑

> "使用 li-bid-document-maker，招標文件在 C:/項目/招標文件.pdf"

## 系統要求

| 項目 | 要求 |
|------|------|
| 操作系統 | Windows 10/11, macOS, Ubuntu 20.04+ |
| AI 平台 | Claude, OpenClaw, Hermes, 或任何支援檔案讀寫的 LLM Agent |
| 文件格式 | PDF（推薦可檢索版）或 Word (.docx) |
| 依賴 | `python-docx`、`PyPDF2`（用於文件處理，可選） |

## 輸出

- 完整應標書（16 章標準結構）
- PDCA 質量改進報告
- 標註評分項對應分值

## 工作流程

```
階段1: 解析招標文件     → 提取項目信息、技術要求、評分標準
階段2: 策略分析         → 評分權重分析、競爭策略
階段3: 大綱生成         → 評分導向大綱（需用戶確認）
階段4: 分章寫作         → 逐章編寫內容
階段5: 質量檢查         → 6維度全覆蓋檢查
階段6: PDCA自動改進     → 3輪改進後交付（無需用戶介入）
```
