## 风险模型任务清单

风险模型按子域组织的任务清单，附任务间关系和元操作映射提示。

**域间逻辑流**：风险识别 → 模型设计 → 模型验证 → 模型应用

---

### 1 风险识别

| ID | 任务类型 | 说明 | 依赖 | 元操作映射 |
|----|----------|------|------|----------|
| T1-1 | 风险源识别 | 识别金融业务中的风险来源和风险因素 | 无（入口） | S→C |
| T1-2 | 风险分类 | 对识别的风险进行分类，建立风险分类体系 | T1-1 | C |
| T1-3 | 风险评估 | 评估各类风险的发生概率和影响程度 | T1-2 | C→G |
| T1-4 | 风险画像 | 构建风险画像，描述风险特征和风险偏好 | T1-3 | C→A |

## 依赖拓扑摘要

### 一、域内链路

**风险识别链路**: T1-1 → T1-2 → T1-3 → T1-4
**模型设计链路**: T2-1 → T2-2 → T2-3 → T2-4
**模型验证链路**: T3-1 → T3-2 → T3-3 → T3-4
**模型应用链路**: T4-1 → T4-2 → T4-3 → T4-4

### 二、跨域协同链路

**完整风险模型链路**: T1-1 → T1-2 → T1-3 → T1-4 → T2-1 → T2-2 → T2-3 → T2-4 → T3-1 → T3-2 → T3-3 → T3-4 → T4-1 → T4-2 → T4-3 → T4-4

**反馈改进链路**: T4-4 → T2-1（模型迭代反馈到模型设计）

**验证驱动链路**: T3-4 → T1-3（模型评估反馈到风险评估）

更多组合根据具体任务动态推导。
