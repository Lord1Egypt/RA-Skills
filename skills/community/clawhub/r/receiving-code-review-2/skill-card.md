## Description: <br>
Guides agents to evaluate code review feedback with technical rigor before clarifying, pushing back, or implementing changes. <br>

This skill is ready for commercial/non-commercial use. <br>

## Publisher: <br>
[lovemymobilewebsite-dotcom](https://clawhub.ai/user/lovemymobilewebsite-dotcom) <br>

### License/Terms of Use: <br>
MIT-0 <br>


## Use Case: <br>
Developers and coding agents use this skill when handling code review feedback, especially unclear or questionable suggestions. It helps them verify feedback against the codebase, clarify scope, push back when technically warranted, and implement reviewed changes one item at a time. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: The agent may propose code edits, GitHub replies, commits, pushes, or merges while applying review feedback. <br>
Mitigation: Review proposed repository changes and external replies before approving them, especially in public or production repositories. <br>
