# gis-site-suitability-analysis

Drafts a multi-criteria site suitability analysis (MCSA) report for GIS analysts and planners — documenting spatial data layers, scoring criteria, weighting rationale, weighted overlay methodology, results, and limitations in a defensible, stakeholder-ready format.

## Use When

A GIS analyst, urban planner, environmental consultant, or site selection professional needs to document the methodology and findings of a multi-criteria site suitability analysis. Common use cases include renewable energy siting, transit-oriented development, facility location, conservation prioritization, logistics hub selection, and hazard zone mapping.

## What It Produces

A DRAFT site suitability analysis report containing:

- Suitability Objective Statement
- Criteria framework table (criterion, layer, type, scoring logic, weight, rationale)
- Exclusion layer summary (Boolean layers applied before weighted scoring)
- Weighted overlay methodology (scoring schema, normalization, composite formula)
- Spatial data catalog (source, date, resolution, format, limitations)
- Results narrative and suitability area summary table (or placeholder if analysis not yet run)
- Limitations and uncertainty section
- Next-steps and field validation recommendations
- Unsigned GIS analyst / project manager review block

## Not a Substitute For

- Regulatory determinations, zoning approvals, or environmental clearances
- Field verification of high-suitability areas before site commitment
- Legal or permitting review of applicable land use restrictions
- Formal stakeholder engagement or public review processes

## Scope

Covers any project type using weighted overlay, Boolean screening, or multi-criteria evaluation in a GIS environment. Platform-agnostic — methodology documentation applies to Esri ArcGIS, QGIS, CARTO, and similar tools. Recommends sensitivity analysis for any report using subjective weight assignments.
