# B2B SEO Audit Framework

## 核心诊断理念

B2B 官网 SEO 的核心问题不是"没做 SEO"，而是四个子系统彼此脱节：
- **品牌页**（首页/关于我们）→ 讲品牌故事，但缺乏关键词锚点
- **博客/资源中心** → 讲知识/趋势，但缺乏商机关联
- **产品/方案页** → 讲功能/规格，但缺乏搜索意图匹配
- **转化路径** → 表单/Demo/联系销售，但流量没有被有效导向

审计的终极目标：把这四个子系统串成一条完整的"搜索→发现→信任→转化"链路。

## 第一阶段：技术 SEO 审计 (Technical SEO)

### 1.1 可抓取性 & 可索引性

| 检查项 | 检查方法 | 通过标准 | 风险等级 |
|--------|----------|----------|----------|
| robots.txt 配置 | 访问 `/robots.txt` | 未阻止重要目录，sitemap 路径正确 | 高 |
| XML Sitemap | 访问 sitemap URL | 包含所有重要页面，更新时间 < 24h | 高 |
| noindex 标签误用 | Screaming Frog / Site Audit | 无重要页面被 noindex | 高 |
| 孤立页面 | Crawl analysis | 所有产品/方案页至少 1 条入链 | 中 |
| 抓取预算浪费 | Google Search Console → Crawl Stats | 低价值页面占比 < 20% | 中 |

### 1.2 网站架构 & URL 结构

| 检查项 | 标准 |
|--------|------|
| URL 可读性 | 包含目标关键词，不含参数和特殊字符 |
| URL 层级深度 | 重要页面 ≤ 3 层 (`domain.com/category/sub/page`) |
| 面包屑导航 | 所有内页有结构化面包屑 |
| 分页处理 | 使用 `rel="prev/next"` 或 `View All` 模式 |
| 多语言处理 | `hreflang` 标签正确配置 |

### 1.3 页面速度 & Core Web Vitals

| 指标 | 优秀 | 需改进 | 差 |
|------|------|--------|-----|
| LCP (最大内容绘制) | ≤ 2.5s | ≤ 4.0s | > 4.0s |
| FID/INP (交互延迟) | ≤ 100ms | ≤ 300ms | > 300ms |
| CLS (累积布局偏移) | ≤ 0.1 | ≤ 0.25 | > 0.25 |
| TTFB (首字节时间) | ≤ 800ms | ≤ 1800ms | > 1800ms |
| 移动端总体评分 | ≥ 90 | ≥ 50 | < 50 |

### 1.4 结构化数据 (Schema Markup)

必须检查的 Schema 类型：
- `Organization` / `LocalBusiness` - 公司信息
- `BreadcrumbList` - 面包屑
- `Article` / `BlogPosting` - 文章/博客
- `Product` - 产品页
- `FAQ` - FAQ 页面（可获富文本搜索结果）
- `WebSite` + `SearchAction` - 搜索框结构化数据
- `VideoObject` - 视频内容

### 1.5 内链结构

| 检查维度 | 标准 |
|----------|------|
| 导航结构 | 主导航覆盖所有核心产品分类 |
| 内链密度 | 每篇博客至少 2-3 条指向产品/方案页的内链 |
| 锚文本多样性 | 使用描述性锚文本，避免"点击这里"/"了解更多" |
| Hub & Spoke 结构 | 主题支柱页 + 辐射式子内容页 |
| 孤岛页面 | 0 个重要页面无入链 |

### 1.6 安全 & 基础设施

| 检查项 | 标准 |
|--------|------|
| HTTPS | 全站 HTTPS，HTTP 自动 301 跳转 |
| 移动端适配 | 响应式设计或独立移动站 |
| CDN 部署 | 静态资源 CDN 加速 |
| 404 页面 | 自定义 404 页，含导航和搜索 |
| 重定向链 | 无超过 3 跳的重定向链 |

## 第二阶段：内容 SEO 审计 (Content SEO)

### 2.1 页面级 SEO 元素

为每个重要页面检查：

| 元素 | 最佳实践 |
|------|----------|
| Title Tag | 30-60 字符，含核心关键词，品牌名在末尾 |
| Meta Description | 120-155 字符，含 CTA，唯一性 |
| H1 | 每页唯一 H1，含核心关键词 |
| H2-H6 层级 | 逻辑嵌套，无跳级 |
| 图片 Alt | 所有图片有描述性 Alt 文本 |
| 正文关键词密度 | 自然分布（1-2%），无堆砌 |
| 内容长度 | 信息类 ≥ 1500 字，产品页 ≥ 500 字 |
| Canonical URL | 自引用 canonical 标签 |

### 2.2 内容质量 & E-E-A-T

| 维度 | 评估标准 |
|------|----------|
| 专业性 (Expertise) | 有行业专家署名/引用权威数据/有方法论支撑 |
| 经验性 (Experience) | 有案例/客户证言/实操指南 |
| 权威性 (Authoritativeness) | 有外部引用/媒体报道/行业认可 |
| 信任度 (Trustworthiness) | 有联系方式/隐私政策/退款保证/资质证书 |

### 2.3 内容覆盖度矩阵

对照买家旅程（Buyer Journey），检查每个阶段的内容覆盖：

| 阶段 | 内容类型 | 覆盖率评估 |
|------|----------|-----------|
| 问题意识 (Problem Aware) | 行业痛点文章、趋势报告、白皮书 | ?/10 |
| 方案探索 (Solution Aware) | 对比指南、ROI 计算器、方案概述 | ?/10 |
| 产品评估 (Product Aware) | 产品页、功能对比、Demo 视频 | ?/10 |
| 决策购买 (Most Aware) | 案例研究、定价页、试用申请、客户推荐 | ?/10 |

### 2.4 关键词布局诊断

| 检查项 | 问题表现 |
|--------|----------|
| 关键词蚕食 | 多页面竞争同一关键词 |
| 关键词缺失 | 核心商业关键词无落地页 |
| 关键词意图错配 | 用信息型关键词做产品页 |
| 长尾覆盖不足 | 长尾关键词无对应内容 |

### 2.5 转化路径诊断

逐页检查 CTA（Call to Action）：
- 每页是否有明确的下一步行动？
- CTA 是否与页面内容意图匹配？
- 表单/联系入口是否在首屏可见？
- 移动端 CTA 是否易于点击？
- 是否有 Micro-conversion（如资料下载、邮件订阅）？

## 第三阶段：竞品内容差距分析

### 3.1 竞品选择

选择 3-5 个直接竞品，混合以下类型：
- 直接业务竞品（同一产品/服务）
- 搜索竞品（在目标关键词 SERP 中出现）
- 内容竞品（在行业内容领域领先）

### 3.2 分析维度矩阵

| 维度 | 我方 | 竞品A | 竞品B | 竞品C |
|------|------|-------|-------|-------|
| 收录页面数 | | | | |
| 月均自然流量 | | | | |
| 排名前10关键词数 | | | | |
| 博客更新频率 | | | | |
| 案例研究数量 | | | | |
| 白皮书/报告数量 | | | | |
| 视频内容数量 | | | | |
| 结构化数据使用 | | | | |
| 外链域名数 | | | | |
| Domain Rating (DR) | | | | |
| 核心功能页DA | | | | |

### 3.3 内容差距识别

对每个核心关键词，记录：
- 竞品有哪些我方没有的内容类型？
- 竞品内容的深度/长度是否优于我方？
- 竞品是否有富文本搜索结果（Featured Snippet/FAQ/Video）？
- 竞品的内容更新频率如何？

## 第四阶段：关键词机会地图

### 4.1 关键词四象限矩阵

将关键词按"商业价值 × 排名难度"分为四类：

| 象限 | 特征 | 策略 |
|------|------|------|
| 高价值/低难度 (速赢) | 搜索量适中，竞争低 | **立即优化** |
| 高价值/高难度 (战略) | 核心商业词，竞争激烈 | **长期攻占**，专题页+外链 |
| 低价值/低难度 (填充) | 长尾词，容易排名 | **批量覆盖**，模板化内容 |
| 低价值/高难度 (放弃) | 投入产出比低 | **搁置**，除非有战略意义 |

### 4.2 B2B 关键词分类

| 关键词类型 | 示例 | 意图 |
|-----------|------|------|
| 问题词 | "如何降低IT运维成本" | 问题意识 |
| 方案词 | "IT运维自动化解决方案" | 方案探索 |
| 对比词 | "Zabbix vs Prometheus" | 方案探索 |
| 产品词 | "运维监控平台" | 产品评估 |
| 品牌词 | "xxx公司 价格" | 决策购买 |
| 长尾商业词 | "制造业ERP系统多少钱" | 决策购买 |

### 4.3 关键词聚类与内容映射

将关键词按主题聚类（Topic Cluster），每个集群分配一个 Pillar Page（支柱页）和多个 Cluster Content（子内容）。

## 第五阶段：TOP 20 优化优先级

### 评分规则（总分 100）

| 评分维度 | 权重 | 评分标准 |
|----------|------|----------|
| 流量潜力 | 25% | 页面目标关键词的月搜索量总和 |
| 排名位置 | 20% | 当前排名越接近第1页，得分越高（第11-20位得分最高） |
| 商业价值 | 25% | 页面距离转化的远近（产品页>方案页>案例页>博客） |
| 优化难度 | 15% | 所需改动越小，得分越高 |
| 当前表现 | 15% | 当前 CTR、跳出率、停留时间等指标 |

### 输出格式

每页输出：
1. URL
2. 当前排名（如有）
3. 目标关键词
4. 优先级评分（/100）
5. 核心问题
6. 具体优化建议（3-5 条）
7. 预期流量提升
8. 建议实施时间

## 第六阶段：90 天执行计划

### 阶段划分

| 阶段 | 时间 | 重点 | 交付物 |
|------|------|------|--------|
| Phase 1: 修复期 | Day 1-14 | 技术 SEO 修复、紧急问题清零 | 技术修复清单完成 |
| Phase 2: 速赢期 | Day 15-30 | TOP 10 速赢内容优化 | 已完成优化的页面 |
| Phase 3: 建设期 | Day 31-60 | 支柱内容创建、关键词缺口填补 | 新内容上线 |
| Phase 4: 增长期 | Day 61-90 | 链接建设、内容矩阵扩展 | 流量增长报告 |

### 周报机制

每周输出：
- 本周完成任务
- 关键指标变化（排名/流量/转化）
- 遇到的问题与解决方案
- 下周计划

## 第七阶段：KPI 汇报框架

### 核心指标三层结构

**L1 - 成果指标（月度）：**
- 自然搜索流量（Organic Traffic）
- 自然搜索带来的线索/商机数（Organic Leads）
- 关键词排名分布变化

**L2 - 过程指标（周度）：**
- 新收录页面数
- Core Web Vitals 评分
- 页面优化完成数
- 新内容发布数

**L3 - 诊断指标（实时）：**
- 抓取错误数
- 404 页面数
- 重复 Title/Meta Description 数
- 损坏链接数

|------|------|------|------|
| 自然搜索流量 | | | |

### 亮点分析
- 哪些改动带来了最大提升？
- 哪些内容表现超出预期？
