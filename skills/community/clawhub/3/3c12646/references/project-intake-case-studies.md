## 案例1：T1转录组项目

**源文件**：`/mnt/d/transcriptome/PROJECT_SPEC.md`（v2.0）
**重构任务**：T-20260519-001

### 缺失的 mission-control 字段

| 缺失字段 | 原始状态 |
|---------|---------|
| 任务编号 | 无 T-YYYYMMDD-NNN |
| 期望完成时间 | 无 |
| 优先级 | 无标注 |
| 验收标准 | 只有科学目标，无具体完成判定 |
| 禁止事项 | Section 8 是"分析禁止行为"，非"任务禁止事项" |
| 产出要求 | 无明确路径/格式规范 |
| Moss填写区 | 无 |
| 执行日志关联 | 无 |

### 原文件优点（值得保留）

- Section 1.3 科学目标清晰
- Section 2 数据描述完整（样本数/批次/实验设计）
- Section 4 分析约束明确（禁止跨批次/WGCNA不可用等）
- Section 7 已确立发现可直接作为分析起点
- Section 9 命名规范实用
- Section 10 Open Questions 列表有价值

### 重构策略

以保留为原则，在原内容基础上补充缺失字段，不删科学信息。

| 任务编号 | 无 T-YYYYMMDD-NNN |

### 原文件优点（值得保留）

### 评分体系示例（五维度加权）

| 维度 | 权重 | 目标 | 当前 |
|------|------|------|------|
| IJSEM格式合规性 | 25% | ≥90 | 95 ✅ |
| 数据一致性 | 25% | ≥95 | 98 ✅ |
| 必填项完整性 | 20% | ≥95 | 95 ✅ |
| 参考文献 | 15% | ≥80 | 70 ⚠️ |
| 语言质量 | 15% | ≥85 | 85 ✅ |
| **加权总分** | 100% | **≥90** | **89 ⚠️** |

### 待补充项优先级分类

| 优先级 | 内容 |
|--------|------|
| 高 | 补充近缘种数据、参考文献、Acknowledgements、Figure确认 |
| 中 | Table S1/S2、补充材料图表、语言润色 |
| 低 | Cover Letter、Author contributions、Conflict of interest |

## 关键洞察

### 科研项目通用缺失

1. **数据不可变性未锁定** → 应明确标注"单一事实来源"
2. **评分体系缺失或散落** → 应结构化为五维度加权评分
3. **Open Questions 无优先级** → 应分级为 P0/P1/P2
4. **禁止事项缺乏分级** → 应拆分为红线vs约束
5. **上下文缺已研究边界** → 应明确Boss已完成vs未完成区域

### mission-control 模板改进方向

参见 `SKILL.md` Section 八.2 科研项目扩展字段。

### 使用建议

- **探索期项目**（如T1转录组）：重点填写 Open Questions 追踪 + 已研究范围
- **接近投稿项目**（如T1-newspecies）：重点填写评分体系 + Review 触发条件
- **新项目**：先判断阶段，再决定填写哪些扩展字段
