# 从「决策顾问」到「认知偏差顾问」——一次 AI Skill 的蜕变升级

> 它不再只是一个帮你做决定的工具，而是覆盖你人生六个关键领域的认知偏差教练。

---

## 一、起点：一个帮人决策的 AI

半年前，我开始做一个叫「认知偏差**决策**顾问」的 AI Skill。它的核心功能很简单——当你在纠结一个重大决定时，用一套基于认知偏差设计的十问框架，帮你逐题排查思维盲区。

确认偏误、认知失调、幸存者偏误、从众效应……10 道诊断题，一道一道来。答完生成一份「决策质量评估报告」，标出你的风险点和行动建议。

反馈不错。但用着用着，我发现一个问题：**认知偏差不只影响"做决策"这件事。**

---

## 二、顿悟：偏差渗透在生活的方方面面

用户跑来问的，远不止"要不要跳槽""该不该买房"：

> "我学 Python 三个月了，总觉得学不会，一打开教程就犯困。"
>
> "道理都懂，就是拖延，怎么才能真的动起来？"
>
> "我想跟老板谈加薪，一开口就怂了，有什么技巧吗？"
>
> "团队绩效评估每次都有人不满意，怎么才能公平？"

这些不是"决策"问题。这是**学习、行动、沟通、管理**的问题——但背後的底层逻辑，全都是认知偏差在作祟。

拖延不是懒，是**现状偏误**和**双曲贴现**联手让你觉得"现在不做也行"。学不会不是笨，是**达克效应**让你误以为自己懂了。谈加薪不是没勇气，是你不知道**框架效应**和**损失厌恶**怎么用来设计对话。

**认知偏差是个操作系统级的问题，不该只用在"决策"这一个应用上。**

---

## 三、核心升级：六大领域 × 策略 × 话术

于是有了这次重命名背后的真正变化——从 v1.x 到 v2.4，五次大迭代：

### v2.0：从 2 域到 6 域的全覆盖

| 领域 | 核心能力 | 覆盖偏差 |
|------|---------|---------|
| 决策 | 十问框架 + 自适应模式 + 评估报告 | 10种 |
| 学习 | 打破达克效应/后见之明/知识诅咒 | 5种 |
| 行动 | 克服拖延、即刻执行 | 5种 |
| 沟通 | 谈判/反馈/亲子/日常四场景策略话术 | 5种 |
| 影响 | 默认效应/社会证明/互惠原则，设计选择环境 | 6种 |
| 管理 | 结构化面试/红队蓝队/止损触发器 | 6种 |

每个领域不仅有偏差诊断，还配了**可直接套用的话术模板**。比如你直接说"我要去谈加薪"，AI 会把：

> "我先说一个数你看合不合理——[目标值+15%]。当然这个可以谈，你那边的预期是？"

这种现成话术直接给到你，不需要自己从偏差理论推导话术。

### v2.1：架构瘦身 45%

六个领域的内容从主文件拆分到 16 个独立 reference 文件，按需加载。这保证了 AI 不会一次性吞掉所有信息导致质量下降——只在用户触发具体领域时，才加载对应的偏差表和策略。

### v2.2：输出标准化

以前给策略是"说说重点"，现在每个领域都有精确的 Markdown 输出模板：偏差诊断卡、策略话术、反向自检三个模块一步不落，就像医生开的处方一样格式统一。

### v2.3 → v2.4：接入芒格"25 类人类误判心理学"

从 v2.3 开始，我们把查理·芒格《穷查理宝典》中的「人类误判心理学」逐个翻译成可直接使用的自检模块。

- **v2.3**：新增 **喜欢热爱倾向（Liking Tendency）** —— 横跨投资和管理两大领域，四重投资自检 + 三重用人自检 + 四道防线。
- **v2.4**：新增 **避免怀疑倾向（Doubt-Avoidance Tendency）** —— 个人决策 9 步自检 + 团队讨论 3 步自检 + 五道防线。

> ⚠️ 关于芒格 25 类中的两个特例：
> - **激励和惩罚超级反应（Incentive Superresponse）**：早在 v1.x 时代就是深度决策十问中的 **Q10**，是最早接入芒格体系的偏差之一。
> - **讨厌憎恨倾向（Disliking/Hating Tendency）**：本质是喜欢热爱倾向的镜像反面——喜欢让你看不到缺点，讨厌让你看不到优点——偏差机制对称，不再单独开设模块。

## 四、名字为什么变了

「认知偏差决策顾问」→「认知偏差顾问」

去掉"决策"两个字，因为**决策只是六大领域之一**。这个名字更准确地描述了它现在的样子：一个覆盖学习、行动、沟通、影响、管理的认知偏差全栈教练。

名字小了，格局大了。

## 五、怎么用

你现在随时可以在支持 Skill 的 AI Agent 中触发它：

| 你说 | 它做什么 |
|------|---------|
| "我想辞职创业，帮我分析" | 进入决策十问框架 |
| "学东西总是记不住" | 诊断达克效应 + 费曼自测策略 |
| "一直在拖延" | 两分钟法则 + 如果-那么预设 |
| "怎么谈加薪" | 给出锚定开场 + 损失框架话术 |
| "团队绩效评估老有人不满" | 结构化面试 + 绩效日志方案 |

三条底线贯穿始终：**不伤害、可透明、双向自检**。用认知偏差影响他人是一把双刃剑，正反都要清醒。
