# 提案/获客模块

生成提案、开发信、服务介绍、案例包装前必须先完成以下分析。

## 客户信号扫描

**🚨 危险信号**：

| 信号 | 风险 | 应对 |
|:---|:---|:---|
| 不愿签合同 | 无法律保障 | "所有项目都签合同，对双方保障" |
| 不愿付预付款 | 不信任或资金问题 | 预付款≥30%底线 |
| "先试做/看看效果" | 免费劳动力 | 试做收费，签约后抵扣 |
| "这个很简单吧" | 不尊重专业 | 用案例说明复杂度 |
| "做好了后面还有很多" | 画大饼压价 | 按单定价，不降价 |
| "你先做再谈价" | 做完没议价权 | 先报价确认再启动 |
| 换过多个供应商 | 客户可能难搞 | 问清换人原因 |
| "你看着办吧" | 需求会无限膨胀 | 先签SOW再开始 |

**⚠️ 注意信号**：预算模糊 / 决策流程不清 / 需求极度模糊 / 熟人介绍

⚠️ 以上是**待确认信号**，不是定罪。大企业用自有采购合同模板 ≠ "不愿签合同"；换过供应商 ≠ "难搞"，可能是前任供应商确实不行。每个信号都需要问清原因再判断。

**🟢 积极信号**：预算明确 / 愿签合同+付预付 / 需求清晰 / 有决策人和时间线 / 主动问你的建议

## 项目定性判断

根据信号扫描结果，给出定性建议（不输出具体数字评分）：
- 🟢 **积极争取**：多绿信号，三档定价最大化利润
- 🟡 **可以接，注意保护**：有黄信号，首付提至40-50%，合同加保护条款
- 🟠 **谨慎，加强条款**：有红信号，50%+首付+严格范围界定
- 🔴 **建议放弃**：多个红信号，风险过高

## 定价方式选择

| 场景 | 方式 | 理由 |
|:---|:---|:---|
| 需求清晰可预估 | 按项目报价 | 明确少纠纷 |
| 能量化业务价值 | 按价值报价 | 利润最高 |
| 需求可能反复变 | 按阶段报价 | 风险可控 |
| 长期持续合作 | 月/季retainer | 稳定现金流 |

> 💡 生成报价前，先用 web search 搜索该服务的最新市场价格。以下为粗略参考范围（以实际搜索为准）：
> - 软件开发：官网 3k-80k / 小程序 5k-150k / APP 30k-500k
> - 设计：Logo 2k-30k / VI 10k-100k / UI 8k-80k / 海报 300-5k
> - 内容：公众号 500-8k / 短视频脚本 500-10k / 代运营 5k-50k/月
> - 咨询：300-3k/小时 / 培训 3k-20k/天 / 翻译 100-500/千字
>
> 如提案中包含报价，同时加载 [quotation.md](quotation.md) 获取三档适用场景、价值定价法、日薪反推法、付款结构等定价策略。

## 文档生成

按 [proposal-templates.md](proposal-templates.md) 中的对应模板生成文档。三档报价模板必须用（>5000元项目）。

**主动建议**：
- 生成提案后 → "确认后我同步准备合同+SOW"
- 生成开发信后 → "建议同时准备服务介绍"
- 生成合同前 → "建议先签NDA保护方案细节"
