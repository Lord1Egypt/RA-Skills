# AI Agent 测试

> 本文件定义 AI Agent（智能体）的专项测试维度和策略要点。Agent 是跨平台能力，可嵌入 App/小程序/H5/桌面/PC Web 等任意宿主，与宿主平台测试叠加使用。
> 用例输出格式见 `references/examples/format-spec.md`，检查清单见 `references/checklists/agent-checklist.md`。

---

## 一、任务完成度

### 专项关注点
- 用户意图理解准确性：显式指令、隐式需求、模糊需求、多轮对话中的意图漂移
- 任务执行完整性：是否遗漏子步骤、是否跳过必要确认、是否提前终止
- 多步骤任务的依赖链：步骤顺序是否正确、前置条件是否满足
- 部分失败处理：某步骤失败后是否有降级方案或回退机制
- 任务边界识别：超出能力范围时是否明确拒绝而非强行执行
- 长任务/复杂任务的中间状态可观测性

### 常见缺陷
- 理解偏差导致执行了错误任务
- 多步骤任务中途跳步或遗漏关键环节
- 遇到异常直接放弃，无容错和降级
- 越界执行（用户未授权的操作）
- 任务未完成就报告完成

## 二、工具与记忆

### 专项关注点
- 工具调用准确性：是否选择了正确的工具、参数是否正确
- 工具调用时机：是否在合适的步骤调用、是否冗余调用
- 工具失败处理：工具返回错误时是否重试/降级/告知用户
- 工具链组合：多工具串行/并行调用时逻辑是否正确
- 记忆读取：是否在需要时查阅记忆文件、是否遗漏关键历史信息
- 记忆写入：是否将重要决策和结果写入记忆、写入内容是否准确
- 记忆一致性：多次会话间记忆是否一致、是否有矛盾记录
- 上下文窗口管理：长对话中是否丢失早期关键信息

### 常见缺陷
- 调用了错误的工具或传参错误
- 工具返回失败后无任何处理
- 重复调用同一工具获取相同结果
- 未读取记忆导致重复提问或忽略已知偏好
- 关键决策未写入记忆，后续会话丢失上下文
- 记忆内容过时未更新，导致错误决策

## 三、安全与边界

### 专项关注点
- 权限边界：是否执行了超出角色权限的操作（如删除数据、发送消息、访问未授权资源）
- 指令注入防御：用户输入中的恶意指令是否被正确隔离
- 数据泄露防护：是否在不应泄露的场景中输出敏感信息（密钥、密码、内部配置）
- 角色一致性：是否始终保持在设定角色内，不被诱导偏离
- 拒绝能力：对有害/非法请求是否正确拒绝
- 资源消耗控制：是否会产生无限循环、大量无效调用、资源浪费
- 跨会话隔离：不同用户的会话信息是否串扰
- 敏感操作确认：高风险操作（删除、发送、支付）是否要求确认
- 恶意输入防御：对 prompt injection、垃圾信息诱导等攻击的处理
- 极端输入处理：空输入、超长输入、特殊字符、异常格式的容错
- 系统异常容错：服务端异常、网络超时、工具不可用等异常场景的处理
- 资源耗尽处理：内存/Token 达到上限时的优雅降级

### 常见缺陷
- 被诱导绕过安全限制执行危险操作
- 在日志/输出中泄露 API Key 或内部信息
- 角色被用户提示词劫持
- 对明显有害请求未拒绝
- 高风险操作未经确认直接执行
- 同一实例服务多用户时会话数据串扰
- 对极端输入（空/超长/特殊字符）处理不当导致异常
- 服务异常时无合理降级直接失败
- 资源耗尽时未优雅降级导致崩溃

## 四、性能体验

### 专项关注点
- 首次响应时间：从用户发送到收到首个 token/首段回复的延迟
- 任务总耗时：复杂任务端到端完成时间
- 流式输出体验：是否流畅逐字输出、是否有长时间卡顿
- 并发处理：多任务/多用户场景下的响应表现
- 资源占用：CPU/内存/Token 消耗是否合理
- 超时处理：长时间任务是否合理设置超时并告知用户
- 工具调用延迟：外部 API 调用对整体响应的影响

### 常见缺陷
- 首次响应超过预期无反馈
- 复杂任务中间长时间无输出
- 流式输出中断或大段延迟
- 并发时响应质量下降
- Token 消耗异常（上下文膨胀导致成本飙升）

## 五、内容质量

### 专项关注点
- 事实准确性：输出内容是否与事实一致，是否编造数据
- 逻辑一致性：同一回答内、多轮对话间逻辑是否自洽
- 格式规范：是否按要求输出指定格式（表格/JSON/Markdown 等）
- 语言质量：表达是否清晰、是否有语病、是否夹杂不自然表达
- 信息完整性：回答是否覆盖所有要求的信息点
- 冗余控制：是否有不必要的重复、啰嗦、无关内容
- 语气与风格：是否符合设定的人设和语气要求
- 引用与溯源：引用信息是否可验证、是否标注来源

### 常见缺陷
- 编造不存在的事实或数据
- 前后回答自相矛盾
- 未按要求格式输出
- 回答遗漏关键信息点
- 过度冗余或偏题
- 语气不符合人设设定

## 六、知识库调用

### 专项关注点
- 知识库检索准确性：是否检索到最相关的知识条目
- 知识库检索召回率：是否遗漏重要相关知识
- 知识应用准确性：检索到的知识是否被正确理解和应用
- 空结果处理：知识库无匹配时是否有合理降级（如实告知而非编造）
- 知识时效性：是否使用过时知识回答当前问题
- 多知识库场景：是否从正确的知识库检索
- 知识冲突处理：不同知识源信息矛盾时的处理策略
- 检索效率：知识库查询对整体响应时间的影响

### 常见缺陷
- 检索到不相关的知识并基于其回答
- 遗漏关键知识条目导致回答不完整
- 知识库无匹配时编造答案而非如实说明
- 使用过时知识导致错误回答
- 多知识库场景下从错误库检索
- 知识冲突时随意选择而非标注矛盾
