# 经营分析模块

生成月度/季度/年度经营分析时加载此模块。

## 数据来源

读取 `data/deals.json` 和 `data/clients.json`，用实际数据填充报告。无数据时根据用户口述生成，并在报告开头标注"⚠️ 基于口述数据，建议使用项目管理功能记录实际数据"。

## 分析决策矩阵

### 客户集中度

| 集中度 | 判断 | 建议 |
|:---|:---|:---|
| <30% | 🟢 健康 | 维持现状 |
| 30-40% | 🟡 偏高 | 主动拓展新客户 |
| >40% | 🔴 危险 | "⚠️ 客户集中度 **X%**，最大客户一旦流失将严重影响收入。建议：①本周联系3个潜在客户 ②开发标准化产品降低依赖" |
| >60% | 🚨 极度依赖 | "🚨 客户集中度 **X%**，单一客户依赖严重。必须行动：①60天内开拓至少2个新客户 ②考虑将该客户的服务标准化，降低不可替代性" |

### 现金流

| 可支撑月数 | 判断 | 建议 |
|:---|:---|:---|
| >6个月 | 🟢 充裕 | 可考虑投入获客或提升服务 |
| 3-6个月 | 🟡 一般 | "建议保持，关注回款节奏" |
| 1-3个月 | 🔴 紧张 | "⚠️ 现金流仅支撑 **X个月**。建议：①催收所有应收款 ②推一单快钱项目（小而确定） ③考虑 retainer 客户稳定收入" |
| <1个月 | 🚨 危机 | "🚨 现金流告急！立即行动：①今天催收所有到期款项 ②联系老客户询问续约/追加需求 ③如有可提前交付的里程碑，加速推进回款" |

### 回款率

| 回款率 | 判断 | 建议 |
|:---|:---|:---|
| >80% | 🟢 良好 | 维持 |
| 40-80% | 🟡 需关注 | "回款率 **X%**，分析未回款原因：逾期客户？付款流程慢？针对性催收" |
| <40% | 🔴 严重 | "⚠️ 回款率仅 **X%**，大量应收未收。建议：①本周完成所有逾期催款 ②未来项目提高首付比例至50%+ ③考虑预付全款模式" |

### 复购率

| 复购率 | 判断 | 建议 |
|:---|:---|:---|
| >50% | 🟢 优秀 | "老客户复购率高，维护好关系就是稳定收入" |
| 20-50% | 🟡 一般 | 建议项目结束时主动提出后续需求，铺垫复购 |
| <20% | 🔴 偏低 | "复购率低，可能原因：一次性项目为主 / 交付后无跟进。建议：①项目完成后30天内回访 ②设计续约/维护套餐 ③每次交付附满意度调查" |

### 客单价趋势

| 趋势 | 判断 | 建议 |
|:---|:---|:---|
| 持续上升 | 🟢 好趋势 | 说明专业度在提升，继续保持 |
| 基本持平 | 🟡 停滞 | "客单价持平，突破方法：①提升报价 ②向高价值客户转型 ③提供更完整解决方案" |
| 持续下降 | 🔴 警惕 | "⚠️ 客单价下降，可能原因：接了太多小项目 / 被压价 / 定位模糊。建议：①设定最低接单价 ②用案例提升谈判力 ③放弃低价客户" |

## 月度分析

输出：核心指标表（营收/在手项目/新增客户/回款/应收）+ 现金流 + 项目进度表 + 风险汇总 + 下月计划。
每项指标自动对照上述矩阵给出判断和建议。

## 季度复盘

在月度基础上增加：
- 项目逐一复盘（目标 vs 实际 / 做对什么 / 教训 / 利润和利润率）
- 客户分析（集中度 / 复购率 / 客单价趋势，全部对照矩阵）
- 财务健康（收入-支出=利润率）
- 下季度核心目标（最多2个）+ 停止做的事

## 年度总结

在季度基础上增加：
- Top 5 成果（每项必须量化）
- 核心资产沉淀（今年积累了什么可持续利用的东西：方法论、案例、工具、人脉）
- 来年规划（收入目标及增长率 / 核心策略 / 开始做什么 / 停止做什么）
