# 列表项内容溢出：识别与修复（通用规则）

## 1. 适用场景

将静态页（index.vue）改造为 Custom.vue 时，若列表/卡片使用 **v-for 渲染动态数据**，而样式从 index.css 照抄了设计稿中**文本的固定 width/height**，会导致**列表项内标题、副标题、作者等文案溢出**卡片或容器，破坏布局。本文档给出识别规则、修复方法与需求文案，供改造与自检使用。

## 2. 问题成因

| 来源 | 说明 |
|------|------|
| **设计稿导出** | index.css 中文本常带固定 `width`、`height`（如 20px、14px），是按设计工具里**单行/固定字数**导出的尺寸。 |
| **静态页** | index.vue 中每条内容长度固定，与设计一致，不会溢出。 |
| **动态页** | Custom.vue 用 v-for 绑定列表数据，标题、作者等长度不固定，仍用固定 `width` 时内容会撑出容器。 |

因此：**凡由 v-for 渲染且展示可变长度文案的列表项，不能对文本节点使用从设计稿照抄的固定 width/height，须改为约束宽度并防止溢出。**

## 3. 识别规则（通用）

### 3.1 何时需要做「列表项溢出」防护

满足以下条件时，必须按 4 的修复方法处理：

- 存在**列表/卡片列表**（如古诗列表、单词列表、资源卡片等），且使用 **v-for** 循环渲染。
- 列表项内有**由数据驱动的文本**（如 `item.title`、`item.author`、`item.name` 等），长度不固定。
- 列表项或卡片容器有**固定宽高**（如 248px×150px），或所在网格/弹性布局有固定列宽。
- 在 index.css 中，对应文本类有**固定 width 或 height**（如 `width: 20px; height: 60px;`）。

### 3.2 识别步骤

1. **定位 v-for 列表**：在 Custom.vue 的 template 中找到 `v-for` 绑定的列表容器与列表项。
2. **找出动态文本节点**：列表项内哪些是 `{{ item.xxx }}` 或 `{{ poem.xxx }}` 等动态文案（标题、副标题、作者、描述等）。
3. **对照 index.css**：这些文本在源样式中是否带有**固定 width/height**（尤其是过小的 width，如 20px、14px）。
4. **确认容器尺寸**：列表项或卡片是否有固定宽高或 max-width，文本区域是否在固定框内。

若「动态文本 + 固定 width/height」同时存在，则必须按第 4 节修复。

## 4. 修复方法（通用）

### 4.1 列表项/卡片容器

- 为**列表项根节点**（或卡片根节点）增加：
  - `overflow: hidden`：防止子内容溢出卡片边界。
  - `box-sizing: border-box`：保证 padding/border 不撑大容器。
- 若列表项内部还有**固定宽高的内容区**（如 first-inner），该内容区也应设 `overflow: hidden`、`min-width: 0`（在 flex 子项中避免被内容撑开）。

### 4.2 动态文本节点（标题、作者、描述等）

对由 v-for 数据驱动的文本，**不得使用从设计稿照抄的固定 width/height**，应改为：

| 原写法（易溢出） | 推荐写法 | 说明 |
|------------------|----------|------|
| `width: 20px` | `max-width: 90px`（按可用空间取值） | 用 max-width 限制最大宽度，允许内容在空间内换行或截断。 |
| `height: 60px` | 删除固定 height，或改用 `min-height` | 高度由行高与行数自然撑开，避免内容被裁切或溢出。 |
| 仅 `overflow-wrap: break-word` | `max-width` + `overflow-wrap: break-word` + `overflow: hidden` | 保证超出部分不溢出容器；必要时加 `min-width: 0`（flex 子项）。 |

**取值建议：**

- **max-width**：根据列表项内容区「可用宽度」估算。例如卡片宽 248px，左侧图 109px、右侧装饰 16px，则文本区约 248−109−16−余量 ≈ 90px，标题可设 `max-width: 90px`，作者可设 `max-width: 50px`（按实际布局微调）。
- **多行标题**：若有两行标题（如主标题 + 副标题），每行都需设 `max-width`、`overflow-wrap: break-word`、`overflow: hidden`，避免某一行过长撑破布局。

### 4.3 自检清单

- [ ] 所有 v-for 列表项根节点（或卡片根节点）已设 `overflow: hidden`、`box-sizing: border-box`。
- [ ] 列表项内由数据驱动的文本（标题、作者、描述等）**未**使用固定 `width`/`height`，已改为 `max-width` + `overflow-wrap: break-word` + `overflow: hidden`（及必要时 `min-width: 0`）。
- [ ] 多行标题/多段文案每一段都有宽度约束与溢出处理。
- [ ] 在浏览器中拉长/缩短标题或作者文案，确认无溢出、无撑破网格或换行错乱。

## 5. 需求文案（可直接用于任务/规范）

以下可作为「静态改动态」任务或规范中的需求描述：

- **列表项内容不溢出**：凡使用 v-for 渲染的列表/卡片，其项内**由数据驱动的文本**（如标题、副标题、作者、描述等）**不得**沿用 index.css 中该文本的固定 `width`/`height`。须改为：列表项/卡片根节点设 `overflow: hidden`、`box-sizing: border-box`；动态文本节点使用 `max-width`（按内容区可用宽度取值）、`overflow-wrap: break-word`、`overflow: hidden`，必要时设 `min-width: 0`（flex 子项）。保证任意长度文案下内容不溢出列表项边界、不撑破布局。
- **自检**：改造完成后，在列表项中代入较长/较短标题与作者等文案，确认无溢出、无错位；列表项根节点与动态文本样式符合上述要求。

## 6. 与现有规范的关系

- **样式匹配标准**（style-match-standard.md）：在「完整匹配」或「简化匹配」列表项时，除尺寸、颜色、字体等与设计一致外，**动态文本的宽度约束必须按本文档处理**，不能为追求「与设计稿数值一致」而保留会导致溢出的固定 width/height。
- **循环渲染规范**（render.md）：v-for 列表的数据与 DOM 结构按 render 规范来；**样式**上若存在动态文本，则同时满足本文档的溢出防护要求。
