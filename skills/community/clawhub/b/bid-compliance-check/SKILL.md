# 投标文件合规检查

## 触发条件

用户提到以下意图时触发：
- "帮我核对一下标书"
- "投标文件检查"
- "帮我看看标书有没有漏的"
- "投标文件准备了，帮我过一遍"
- "核对招标要求"
- "废标项检查"

## 必需输入

| 输入项 | 必需 | 说明 |
|--------|------|------|
| 招标文件 | ✅ | PDF/Word 格式的招标文件原文 |
| 投标文件成稿 | ✅ | PDF/Word 格式的已完成投标文件（全部或分册） |
| 项目名称 | 推荐 | 用于输出文档命名 |
| 投标截止日期 | 推荐 | 用于检查时间类要求（如保证金缴纳时间、投标有效期等） |

## 执行流程

### 第一步：解析招标文件 → 生成专属要求清单

> **核心原则**：不从任何模板套用，必须从招标文件原文逐条提取。以下为引导你扫描的**检索方向**，不代表最终清单。

逐章节阅读招标文件，按四个维度提取每一条要求。对每条要求记录原文出处（章节/页码）：

#### A. 资格性要求

扫描方向（招标文件中常见于"投标人资格要求""供应商资格条件"等章节）：

- **企业资质**：营业执照、行业许可证、ISO认证等。注意有效期是否覆盖投标期
- **财务要求**：近X年审计报告、纳税证明、社保证明。注意年份要求
- **业绩要求**：类似项目合同数量和金额门槛、验收报告。注意"类似"的定义范围
- **人员要求**：项目经理资质证书、团队成员数量和证书、社保缴纳证明
- **信用要求**：信用中国截图、无违法记录声明、无行贿犯罪记录
- **联合体/分包**：是否允许、联合体数量上限、分包比例限制

#### B. 符合性要求 / 废标项（高危，逐条标注是否废标条件）

扫描方向（常见于"投标人须知""废标条款""无效投标情形"等章节）：

- **投标保证金**：金额、缴纳方式（电汇/保函）、到账截止时间
- **投标有效期**：要求天数，从开标日起算
- **文件格式与份数**：正本X份、副本X份、电子版X份（U盘/光盘格式要求）
- **装订要求**：胶装/活页、A4规格、页码编排
- **密封要求**：正本单独密封、副本单独密封、电子版单独密封，外封标注内容
- **签章要求**：法人签字/签章位置、授权委托书格式、骑缝章、公章类型
- **核心偏离条款**：标注"不接受负偏离"的技术或商务条款
- **报价限制**：最高限价、分项限价
- **进口/国产**：是否限定国产产品

#### C. 评分项

扫描方向（常见于"评标办法""评分标准"等章节）：

- 技术方案、商务报价、业绩案例、团队配置、服务方案、演示答辩
- 每条记录分值、评分细则（什么情况得满分/扣分标准）

#### D. 格式、封装与递交

- 正副本份数、电子版数量与格式
- 内外层密封方式、外包装标注文字要求
- 递交地点和截止时间（精确到分钟）
- 开标地点、是否要求现场参加
- 澄清/答疑截止时间
- 现场踏勘/答疑会时间地点（是否强制参加）

#### E. 其他必须确认项

- 投标有效期是否满足
- 是否允许分包、分包比例限制
- 是否允许替代方案
- 语言要求（中/英文）

### 第二步：逐项核对投标文件

对第一步生成的每一条要求，在投标文件中定位对应内容，标注状态：

| 状态 | 含义 | 处理原则 |
|------|------|---------|
| ✅ 满足 | 找到明确对应内容，且符合要求 | 无需操作 |
| ⚠️ 存疑 | 找到相关内容但不完全匹配或无法判断 | 必须人工确认 |
| ❌ 缺失 | 未找到对应内容 | 必须立即补充 |
| — 不适用 | 该条与本次投标无关（如不涉及联合体则跳过联合体条款） | 标注原因 |

> 要求：每条检查结果必须注明投标文件中的对应位置（页码范围、章节名或文件名），不能只写"有/无"。

### 第四步：递交前最终检查（可选，用户明确要求时执行）

如果用户在递交前要求最终检查，基于以下常见致命遗漏清单逐项确认（源自多个投标项目的踩坑经验）：

| 检查项 | 验证方式 |
|--------|---------|
| 投标保证金是否已到账 | 核对银行回单 + 金额 + 截止时间 |
| 所有签章是否完成 | 逐页确认：法人签字、公章、骑缝章、授权书签字 |
| 正副本份数是否对应 | 实物清点 |
| 密封是否按招标文件要求 | 逐层核对密封方式 |
| 外包装标注是否完整 | 核对标注文字 |
| 投标有效期是否满足 | 核对承诺函中的有效期表述 |
| 报价是否经双人复核 | 确认复核记录 |
| 递交时间地点是否确认 | 复查公告中的时间地点 |
| 备份电子版是否已存档 | 确认U盘/光盘内容 + 可读性 |

## 内嵌知识：常见踩坑经验（用于检查时的重点提醒）

以下来自实战经验，在生成检查表时自动对照提醒：

| 坑 | 后果 | 检查时要重点看的 |
|----|------|----------------|
| 投标保证金到账延迟 | 废标 | 核对保证金缴纳方式（电汇需T+1到账）、银行处理时间，建议提前3个工作日 |
| 密封不规范 | 当场废标 | 严格按招标文件"密封要求"章节操作，正本/副本/电子版可能要求分别密封 |
| 骑缝章遗漏 | 废标 | 多页文件必须加盖骑缝章，标书装订后漏盖是高频错误 |
| 报价计算错误 | 扣分或废标 | 报价表数字与分项明细之和不一致、大写小写不对应 |
| 业绩证明材料不匹配 | 评标扣分 | "类似项目"的定义范围、合同金额门槛、验收报告是否齐全 |
| 投标有效期不足 | 废标 | 承诺的投标有效期天数 < 招标文件要求天数 |
| 授权委托书法人章错误 | 废标 | 授权书上的法人名字/身份证号与营业执照不一致 |
| 电子版与纸质版不一致 | 废标 | U盘内容与纸质版页码不对应，或文件打不开 |

## 输出格式

- 默认输出 MD 表格，直接在对话中呈现
- 如用户要求 Excel，创建多 Sheet 表格（废标项 / 资格项 / 评分项 / 封装）

## 质量标准

- ✅ 必须从招标文件原文逐条提取要求，不套用任何固定模板
- ✅ 废标项必须逐条标注是否为废标条件，缺失即致命
- ✅ 每个缺失/存疑项必须有具体处理建议，不是泛泛的"需补充"
- ✅ 检查结果必须标注投标文件中的具体对应位置（页码/章节/文件名）
- ✅ 自动对照"常见踩坑经验"对高频遗漏点做重点提醒
- ❌ 不能遗漏招标文件中的任何一条要求
- ❌ 不能做出无依据的乐观判断——"应该没问题"不可接受
- ❌ 不能跳过不确定项，必须标注为存疑并给出人工确认建议
