## 一、支付方式详解

### 1.1 支付方式对比矩阵

| 支付方式 | 适用场景 | 费用 | 优点 | 缺点 | 推荐度 |
|----------|----------|------|------|------|--------|
| **T/T电汇** | B2B大额 | $25-50/笔 | 安全、无限额、直接到账 | 流程长、需核实水单 | ⭐⭐⭐⭐⭐ |
| **PayPal** | B2C/小额 | 3.5%+$0.3 | 全球覆盖、信任度高、买家保护 | 费用高、风控严、提现慢 | ⭐⭐⭐ |
| **PingPong** | 跨境电商 | 0.5%-1% | 费率低、到账快、阳光结汇 | B2B场景支持有限 | ⭐⭐⭐ |
| **连连支付** | 跨境电商 | 0.5%-0.7% | 生态完善、多平台支持 | B2B场景弱 | ⭐⭐ |
| **WorldFirst** | B2B收款 | 0.5%-1% | 大额友好、多币种支持 | 合规门槛高、审核严 | ⭐⭐⭐⭐ |
| **Escrow** | 大额/信任建立期 | 1%-3% | 资金安全、双方保障 | 费用较高、流程慢 | ⭐⭐⭐⭐ |
| **信用证L/C** | 超大额/高风险 | 0.5%-2% | 银行担保、风险最低 | 流程复杂、门槛高 | ⭐⭐⭐⭐ |

### 1.2 T/T电汇详解（首选方案）

#### 基本原理
T/T（Telegraphic Transfer）电汇是最常用的B2B外贸支付方式，通过银行将货款直接从买家账户汇至卖家账户。

#### 分阶段付款模式

**推荐结构：30%+70%（或20%+80%）**

#### 水单核实要点
```
【水单真伪核实清单】

1. 基本信息核对：
   ├── 汇款人名称是否与合同一致
   ├── 汇款金额是否正确
   ├── 汇款日期是否合理
   └── 附言/备注是否标注订单号

2. 银行核实：
   ├── 通过银行官网查询SWIFT代码验证
   ├── 电话联系汇出银行核实
   └── 要求客户提供汇款流水截图

3. 金额差异处理：
   ├── 金额少：绝对不能发货，需客户补汇
   ├── 金额多：原路退回或通知客户处理
   └── 汇率差异：按实际到账金额确认
```

### 1.3 不同场景的支付方案

| 客户类型 | 订单金额 | 推荐方案 | 说明 |
|----------|----------|----------|------|
| **成熟客户** | 任意 | T/T 30%+70% | 长期合作，信任度高 |
| **新客户** | <$5,000 | T/T 50%+50% | 适度风控 |
| **新客户** | $5,000-$20,000 | T/T 50%+50% 或 Escrow | 加强风控 |
| **新客户** | >$20,000 | 预付50%+托管 或 L/C | 高额必保 |
| **高风险国家** | 任意 | Escrow 或 全额预付 | 规避风险 |
| **小额样品单** | <$500 | PayPal 或 全额预付 | 低风险 |

### 1.4 PayPal使用指南

```
【PayPal外贸收款指南】

适用场景：
├── 小额样品单（<$500）
├── C端零售
└── 信任度不足的新客户

操作流程：
1. 注册PayPal企业账户
2. 发送PayPal收款链接给客户
3. 客户付款后资金到账
4. 提现至银行账户（约3-5个工作日）

费用计算示例：
订单金额：$500
PayPal费用：$500 × 3.5% + $0.3 = $17.8 + $0.3 = $18.1
实际到账：$500 - $18.1 = $481.9
费率约：3.62%

注意事项：
├── 保留完整的交易记录
├── 及时响应买家争议
├── 避免IP异常导致账号受限
└── 定期核对PayPal账单
```

## 二、收汇结汇流程（1039政策）

### 2.1 收汇流程SOP

```
【1039市场采购贸易收汇SOP】

Step 1: 银行账户准备
├── 开立义乌稠州商业银行/中国银行外汇账户
├── 确认账户支持1039贸易项下收汇
├── 开通网银外汇功能
└── 保存银行联系人电话

Step 2: 收款确认
├── 客户打款后，银行发送收汇通知
├── 登录网银查看收汇明细
├── 核对：金额、汇款人、附言
└── 记录收汇编号

Step 3: 收汇入账
├── 银行审核通过后入账
├── 查看外汇账户余额
├── 记录收汇日期和金额
└── 更新订单状态

Step 4: 结汇操作
├── 登录网银外汇业务
├── 选择结汇币种和金额
├── 确认结汇汇率
└── 提交结汇申请

Step 5: 人民币入账
├── 结汇成功后人民币到账
├── 核对入账金额
├── 转账至人民币账户
└── 归档收汇凭证
```

### 2.4 1039结汇注意事项

```
【1039收汇注意事项】

├── 借用他人账户收汇

├── 收汇被拒：检查外汇管理规定

├── 账户冻结：配合银行调查
└── 单证不全：补齐材料后重试

### 3.2 支付风控流程

```
【支付风控标准流程】

收到订单 → 客户背调 → 风控评估 → 方案设计 → 执行监控 → 异常处理
    │           │            │            │            │            │
    ▼           ▼            ▼            ▼            ▼            ▼
  接收询盘   工商核查     A/B/C分级    确定付款    跟踪水单     启动预案
   
Step 1: 客户背调（必须）
├── 核查公司工商信息（天眼查）
├── 查询制裁名单（OFAC/欧盟）
├── 历史合作记录（如有）
└── 邓白氏信用报告（如有大额）

Step 2: 风控评估
├── 评估结论：A/B/C级
├── A级（信用良好）：常规操作
├── B级（有一定风险）：加强风控
└── C级（高风险）：拒绝或全额预付

Step 3: 方案设计
├── 根据评估结果设计付款方案
├── 明确预付比例
├── 确定托管方式
└── 与客户确认条款

Step 4: 执行监控
├── 监控付款到账
├── 核实水单真伪
├── 确认正本提单
└── 全程保持沟通记录

Step 5: 异常处理
├── 未按时付款：催款+警告
├── 付款异常：暂停发货
└── 欺诈嫌疑：立即报警
```

### 3.3 客户背调清单

```
【客户背调标准流程】

1. 基础信息核查：
   ├── 公司名称（全称）
   ├── 注册地址
   ├── 注册资本
   ├── 成立年限
   └── 经营状态（存续/注销）

2. 信用信息查询：
   ├── 天眼查/企查查（国内客户）
   ├── 邓白氏报告（国际客户）
   ├── 海关信用记录
   └── 同行口碑调查

3. 制裁名单核查：
   ├── OFAC制裁名单（美国）
   ├── EU制裁名单（欧盟）
   ├── UN制裁名单（联合国）
   └── 中国制裁名单

4. 风险客户黑名单：
   ├── 行业协会黑名单
   ├── 同行共享黑名单
   └── 内部历史记录

高危信号：
├── 公司成立不足1年
├── 注册资本与订单金额不匹配
├── 注册地址为虚拟地址
├── 有法律诉讼记录
└── 银行征信不良
```

### 3.4 账户变更验证SOP

```
【账户变更防诈骗SOP】

⚠️ 触发条件：收到客户账户变更通知

Step 1: 立即暂停付款
└── 通知财务暂停该订单所有付款

Step 2: 多渠道核实
├── 电话核实（使用原始号码，非通知中提供的号码）
├── 视频核实（如可行）
├── 邮件核实（回复原始邮件地址）
└── 书面确认（要求提供书面变更申请+签章）

Step 3: 内部审批
├── 填写《账户变更审批表》
├── 财务+老板双重审批
├── 法务/风控会签（如有大额）
└── 通过后方可付款

Step 4: 记录归档
├── 保存全部沟通记录
├── 存档变更申请原件
├── 记录核实过程
└── 更新客户档案

Step 5: 差异处理
├── 如无法核实：拒绝变更，继续原账户
└── 如确认诈骗：报警处理
```

## 四、汇率管理

### 4.1 汇率管理工具箱

```
【汇率风险管理工具箱】

工具1：远期结汇
├── 原理：锁定未来某日汇率
├── 适用：订单周期>30天
├── 优点：确定性高，锁定利润
├── 成本：银行手续费约0.1%-0.3%
└── 操作：向银行申请远期结汇产品

工具2：外汇期权
├── 原理：购买结汇选择权
├── 适用：汇率波动剧烈
├── 优点：风险有限，收益无限
├── 成本：期权费约0.5%-1%
└── 注意：需评估是否值得购买

工具3：自然对冲
├── 原理：美元收+美元支
├── 适用：有美元采购渠道
├── 优点：零成本对冲
└── 条件：需配套采购渠道

工具4：汇率保险
├── 原理：保险产品兜底
├── 适用：大额订单
├── 优点：风险转移
└── 成本：保费约0.3%-0.5%

工具5：分批结汇
├── 原理：分散结汇时机
├── 适用：汇率波动期间
├── 优点：降低单一时点风险
└── 操作：按比例分批结汇
```

### 4.3 汇率波动应对策略

```
【汇率波动应急处理】

情况1：汇率大幅下跌（美元贬值）
├── 立即评估影响：利润损失计算
│   损失 = 原汇率 × 金额 - 新汇率 × 金额
├── 短期措施：与供应商谈判降价
├── 中期措施：提高新订单报价
└── 长期措施：增加非美货币结算

情况2：汇率大幅上涨（美元升值）
├── 短期措施：加速结汇
├── 中期措施：观望等待更高点
└── 长期措施：考虑美元采购对冲

情况3：汇率剧烈波动
├── 暂停大额结汇
├── 分批结汇分散风险
├── 咨询银行锁汇方案
└── 预留安全边际（报价加2-3%）

心理预期管理：
├── 设定目标汇率
├── 设置止损线
├── 避免频繁操作
└── 长期视角看待汇率
```

### 5.1 Escrow托管交易

```
【Escrow托管交易流程】

适用场景：
├── 新客户大额订单（>$10,000）
├── 买卖双方缺乏信任
└── 高风险商品/地区

操作流程：

┌─────────────────────────────────────────────────────────────────────────────┐
│                        Escrow交易流程                                      │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   Step 1: 双方约定Escrow托管                                              │
│   ↓                                                                       │
│   Step 2: 买家将款项支付到Escrow账户                                      │
│   ↓                                                                       │
│   Step 3: Escrow确认收款，通知卖家                                         │
│   ↓                                                                       │
│   Step 4: 卖家发货，提供物流单据                                          │
│   ↓                                                                       │
│   Step 5: 买家确认收货或验货                                              │
│   ↓                                                                       │
│   Step 6: 买家确认后，Escrow放款给卖家                                    │
│   ↓                                                                       │
│   Step 7: 交易完成                                                        │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘

推荐平台：
├── Escrow.com（国际通用，费用1%-3%）
├── Alipay Global（支付宝国际）
└── 连连支付托管服务

费用计算示例：
订单金额：$20,000
Escrow费用：$20,000 × 1.5% = $300
总成本：$20,300

优点：
├── 买家资金有保障
├── 卖家发货有保障
├── 争议时有中间方调解
└── 双方都放心

缺点：
├── 额外费用
├── 流程较长
└── 部分客户不了解

### 5.2 信用证（L/C）入门

```
【信用证基础知识】

适用场景：
├── 超大额订单（>$50,000）
├── 买卖双方首次合作
└── 高风险交易

基本原理：
├── 银行作为中间担保方
├── 买家银行开立信用证
├── 卖家按要求交单
└── 银行审核后付款

义乌实践建议：
├── 义乌稠州商业银行可开立信用证
├── 建议使用即期信用证（Sight L/C）
├── 建议要求买家承担全部开证费用
└── 建议指定义乌当地银行交单

信用证类型：
├── 即期信用证（Sight L/C）：付款交单
├── 远期信用证（Usance L/C）：承兑交单
├── 可转让信用证（Transferable L/C）：可转给供货商
└── 备用信用证（Standby L/C）：保函作用

注意事项：
├── 仔细审核信用证条款
├── 确保单证一致
├── 注意软条款风险
└── 建议通过银行专业指导
```
