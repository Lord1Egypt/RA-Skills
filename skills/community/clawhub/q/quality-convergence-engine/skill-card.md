## Description: <br>
Multi-dimensional Quality Acceptance and Problem Convergence Engine - Deeply deconstruct requirements, eliminate extreme defects, define absolutely objective acceptance and failure criteria. <br>

This skill is ready for commercial/non-commercial use. <br>

## Use Case: <br>
Developers and engineers use this skill to evaluate code, architecture, and solution proposals by identifying concrete failure conditions and measurable acceptance criteria. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: The always-on behavior can influence ordinary review requests even when quality acceptance is not explicitly requested. <br>
Mitigation: Use it in contexts where review-style risk analysis is desired, and check whether its conclusions fit the user's actual request before acting on them. <br>
Risk: The skill may shape code and architecture review responses with incorrect or overly strict acceptance criteria. <br>
Mitigation: Review the generated red light conditions and green light criteria against project requirements before using them for release or implementation decisions. <br>
