Mitigation: Confirm the Next.js version and hosting platform before applying recommendations, and review proposed fetch, cache, and revalidation changes against the current Next.js documentation. <br>
Risk: Changes around authentication, cache policy, and edge versus Node runtime can affect security, correctness, or deployment compatibility. <br>
Mitigation: Review any generated code or configuration before use, with special attention to user-specific data, middleware matchers, runtime dependencies, and deployment constraints. <br>
