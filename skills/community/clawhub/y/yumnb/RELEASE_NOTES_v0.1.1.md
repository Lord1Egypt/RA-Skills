# yumnb v0.1.1 — Positioning + README polish

This patch release tightens yumnb's public positioning and documentation.

## What's new

- New tagline:
  - **A local-first NotebookLM alternative for AI agents, built for real source capture.**
- README now emphasizes:
  - one request = one notebook folder
  - user-facing flow instead of internal implementation flow
  - local-first storage and opt-in external delivery
  - reliable source extraction for web pages and YouTube URLs
- Simplified top-of-page diagram for easier reading
- Added better GitHub / ClawHub page presentation
  - badges
  - concise tables
  - clearer artifact explanations

## Why this matters

yumnb is not just a summarizer. It is meant to help AI agents build a local, inspectable, reusable notebook library with notes, audio, and slides — while being more transparent about source capture than many hosted notebook tools.
