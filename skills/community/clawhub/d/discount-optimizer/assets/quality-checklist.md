# Discount Plan — Quality Checklist

Run before any promo goes live. Every box should be checkable with a number or a documented decision, not a feeling.

## Unit Economics
- [ ] Selling price (list) recorded
- [ ] COGS is **landed** (includes inbound freight + duties), not ex-works
- [ ] Platform commission included as a % of price
- [ ] Payment processing fee included
- [ ] Fulfillment / pick-pack cost per unit included
- [ ] Outbound shipping subsidy (the part you eat) included
- [ ] Affiliate / creator commission included where applicable
- [ ] Returns reserve included as a % of price
- [ ] Total variable cost summed and **contribution margin** computed in both $ and %

## Margin Floor
- [ ] Hard minimum post-promo contribution margin set (a % and a $)
- [ ] Floor is based on contribution margin, not gross margin
- [ ] Proposed discounted price clears the floor **after all fees**
- [ ] Floor is enforceable in the platform (won't be breached by rounding)
- [ ] Exceptions to the floor are documented and approved, not ad hoc

## Break-even Math
- [ ] Required volume lift computed via `required-lift = d/(m−d)`
- [ ] `m` used is the pre-discount contribution margin %
- [ ] Required lift compared against realistic channel demand
- [ ] Discount depth `d` is comfortably below contribution margin `m`
- [ ] Markdown % vs margin % not confused (used `(m−d)/(1−d)` to check remaining margin)
- [ ] Fee-adjusted (exact) break-even checked when fees are a large % of price

## Promo Mechanic
- [ ] Objective named in one sentence (acquire / AOV / clear / defend / launch)
- [ ] Mechanic matches the objective, not just defaulted to % off
- [ ] BOGO/B2G1 only used where CM covers the give-away unit's full cost
- [ ] Bundle blended CM computed across the whole bundle, post-discount
- [ ] Threshold (if used) set above current AOV to genuinely lift baskets
- [ ] Free-shipping subsidy treated as a variable cost

## Fees & Stacking
- [ ] Commission re-computed on the **discounted** price
- [ ] Voucher funding split confirmed (you vs platform)
- [ ] Affiliate/creator commission modeled on top of platform commission
- [ ] Maximum **combined** stacked depth capped
- [ ] Worst-case stack (coupon + auto + voucher + free ship) checked against floor
- [ ] Coupon vs automatic chosen deliberately for the goal

## Inventory Fit
- [ ] Days-of-cover calculated (units on hand ÷ daily sell-through)
- [ ] Fast movers not over-discounted (would sell anyway)
- [ ] Aging stock evaluated against carrying + obsolescence cost
- [ ] Clearance depth justified by salvage-value-later comparison, not full-price contribution
- [ ] Inventory / units cap set so the promo can't oversell

## Anchor-Price Integrity
- [ ] Credible list price held; not a permanent "sale" price
- [ ] Reference/"was" price is genuine and defensible
- [ ] Promo is time-boxed with a defined start and end date
- [ ] Post-promo return-to-list price defined
- [ ] Promo cadence varied to avoid training customers to wait

## Monitoring
- [ ] Success metric and target set before launch
- [ ] Realized unit lift tracked vs the break-even lift
- [ ] Blended contribution margin tracked vs the floor
- [ ] Sell-through and return rate tracked during the promo
- [ ] New-vs-returning customer mix tracked
- [ ] Kill condition defined (e.g. under break-even lift with no strategic payoff)
- [ ] Post-promo review scheduled with a decision/next-step owner
