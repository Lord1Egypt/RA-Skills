# 改造示例：情报分析雷达（intelligence-radar）

一个覆盖全部规则的完整 before → after，含：凭据型登录、无定时语义、4→5 占位符映射（{{where}} 不出现）、密码不回显。照此对齐风格。

---

## 输入（改造前 · 原始资料）

> 变量：`{company_name}`、`{user_input}`、`{employee_id}`、`{password}`

**场景简述摘要**：
情报分析雷达是一个AI驱动的公司情报采集和分析工具，主要用于情报采集、动态分析、策略生成、客户关联。使用步骤：首次登录（Token管理）→ 触发情报采集 → 等待采集完成 → 查看采集结果 → 客户关联处理 → 查看完整报告。

**原始安装提示词**：
```
请安装情报分析雷达，用于采集和分析公司情报，为销售提供策略建议。
```

**原始触发提示词**：
```
采集情报
分析情报
情报雷达
雷达分析
最近动态
最新动态
我要拜访{company_name}
我想了解{company_name}
帮我准备{company_name}
帮我分析{company_name}
```

> 注意：本场景原生即为手动触发，无定时推送逻辑，但需员工登录获取 Token，属于**凭据型**场景。

## 输出 1 · 场景介绍（改造后）

```
一句话摸清目标公司，让拜访有的放矢。

它会帮你做什么：
- 智能采集公开情报：从官网、招投标、招聘等公开渠道抓取目标公司最新动态，覆盖业务、产品、人事、市场多维度信息。
- 生成销售策略建议：基于采集数据提炼客户痛点、合作切入点和可执行话术，让销售拜访更有针对性。
- 关联客户档案：采集完成后检查该公司是否已录入客户库，未添加时一键快速建档。

怎么用：
1. 点「安装场景」
2. 填目标公司和关注动态/查询重点
3. 需要时点「采集情报」，即可获取策略摘要与完整情报报告
```

## 输出 2 · 安装提示词（改造后 · 带登录版）

```
## 任务
用户安装了「情报分析雷达」场景。请完成以下三步。

## 第一步：安装情报分析雷达 Skill
首先尝试从企业技能中心安装：
byted-skillhub-download intelligence-radar

- 如果安装成功，继续第二步。
- 如果安装失败，则尝试执行以下命令安装：
openclaw skills install intelligence-radar
  - 如果兜底安装成功，继续第二步。
  - 如果兜底安装仍然失败，告诉用户"情报分析雷达场景安装失败，请稍后重试"，并停止。

## 第二步：身份验证登录
使用 {{push_time}}（登录账号）和 {{push_channel}}（登录密码）调用登录接口验证并缓存 Token。
- 如果验证成功，继续第三步。
- 如果验证失败，告诉用户"账号或密码有误，请重新输入"，并等待用户重新输入。

## 第三步：确认并引导
安装并登录成功后，向用户输出以下内容：

---
✅ **情报分析雷达场景已安装成功！**

你的当前配置：
- 🧑‍💼 登录账号：{{push_time}}
- 🎯 目标公司：{{who}}
- 🔍 关注动态：{{what}}

**使用方式：**
在「我的场景」中点击「采集情报」，系统会采集目标公司的公开情报并生成销售策略摘要，同时提供完整情报报告的 H5 链接，支持拜访前快速摸底。

**你还可以：**
- 在「我的场景」点击「设置」，随时修改目标公司、关注动态
- 每次触发时也可以随时修改，适配不同需求
---

如果用户没有填写某项配置，对应行显示"未设置"。
```

## 改造动作对照表（发生了什么）

| 改造点 | 改前 | 改后 |
|---|---|---|
| 步数 | 未明确场景化步数 | 三步（安装 + 登录 + 确认） |
| 登录步骤 | 隐含在 Skill 内部 Token 管理 | 显性化为第二步身份验证 |
| push_time/push_channel | 无（原生手动触发） | 借作登录账号/登录密码 |
| 使用方式 | 多种自然语言触发词 | 统一为「我的场景」中点击「采集情报」 |
| 安装命令 | 口语化"请安装..." | 规范双重兜底 |
| 密码 | 隐含在首次登录环节 | 入参用，**不回显** |
| 场景介绍 | 原始场景简述（功能罗列） | 统一三段式营销文案（slogan + 能力 + 用法） |
| 变量展示 | 无统一占位符 | 全部映射到 {{who}}/{{what}}/{{push_time}}/{{push_channel}} |
