# Acceptance Criteria — Podcast Launchpad

## File Completeness
- [ ] `skill.json` exists and is valid JSON
- [ ] `SKILL.md` exists with valid YAML frontmatter
- [ ] `README.md` exists and summarizes launch-kit outputs
- [ ] `clawhub.json` exists for package publishing
- [ ] `ACCEPTANCE.md` exists (this file)

## Content Quality
- [ ] All content is English-primary
- [ ] Description is clear and actionable
- [ ] Prompt flow covers: concept, format, equipment, scriptwriting, recording, editing, hosting, distribution, launch QA, growth
- [ ] Directory guidance includes Apple Podcasts, Spotify, YouTube Music, Amazon Music, Overcast, Pocket Casts, and audience-relevant directories
- [ ] Skill tells users to verify current platform submission requirements before claiming listing status
- [ ] Target scenario matches: Aspiring podcast creators seeking end-to-end production guidance

## Safety Compliance
- [ ] Disclaimers present: no guarantee of success, not legal/financial advice, not audio engineering training
- [ ] Ethical boundaries for interviews and content stated
- [ ] No clickbait or manipulative growth tactics encouraged
- [ ] No unsupported claims about platform acceptance, rankings, or monetization
- [ ] Educational purpose clearly stated

## No-Code Compliance
- [ ] No executable code in any file
- [ ] No script, handler, or API references
- [ ] No code execution entry points
- [ ] `hasExecutableCode: false` in skill.json
- [ ] `requiresApi: false` in skill.json

## Structure Compliance
- [ ] Directory at `~/.openclaw/skills/podcast-launchpad`
- [ ] No handler/scripts fields in skill.json
- [ ] No executable script files in directory
- [ ] Pure descriptive/document-only content type

## Scene Ownership
- [ ] Skill scope does not overlap with existing skills
- [ ] Skill owns a clearly defined podcast creation and production scene
