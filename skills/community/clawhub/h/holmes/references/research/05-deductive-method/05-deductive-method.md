# AI 演绎法工作流：基本演绎法

> **来源：** 《血字的研究》chap_008–chap_020 + 三个经典案例（跳舞的小人 / 斑点带子 / 巴斯克维尔猎犬）
> **目的：** 将福尔摩斯演绎法蒸馏为 AI 可执行的精确工作流

---

## 一、核心公式：因果链条

福尔摩斯原理（chap_009）：

> "生活实则是根巨大的链条，观其一环便可知整体本质。"
> "即使没有亲眼目睹大西洋，逻辑学家也可以由一滴水推断出它的存在。"

**核心公式：**

```
已知 A → 推得 B → 推得 C
（每一步都基于前一步的逻辑必然，非统计概率）
```

**关键约束：**
- 因果链条是**必然性**推导，不是可能性猜测
- 局部细节（A）是链条的一环，通过环环相扣重建完整链条（B→C）
- 如果某一步出现矛盾，说明该环节另有解释，需修正而非放弃

## 二、AI 演绎法工作流（六步）

```
第一步：建立上下文
第二步：分析缺口
第三步：制定查找策略
第四步：执行查找
第五步：验证
第六步：结论
```

---

### 第一步：建立上下文

**目标：** 在分析前建立完整的"已知"基线。

**具体操作（AI 知道下一步做什么）：**

1. **收集所有可直接观察的信息**（痕迹、物品、言行、时间线）
2. **记录信息来源**（原文证据 vs 推断，避免混淆）
3. **识别已知的因果链片段**（哪些 A→B 已经确认）

**输出格式：**

```
已知事实 K：
K1: [观察/证据]
K2: [观察/证据]
K3: [推断，但需验证]

已确认链路：
K1 → K2（已验证）
K2 → K?（待填充）
```

**结束条件：** 已知事实不再增加，进入下一步。

---

### 第二步：分析缺口

**目标：** 明确"从已知到结论，还缺什么"。

**具体操作（AI 知道下一步做什么）：**

1. **列出从当前上下文到结论的所有必要环节**
2. **标记每个环节的状态：**
   - `已知`：已确认
   - `缺失`：无法从当前上下文得出
   - `矛盾`：与已知事实冲突
3. **找最大缺口**：缺失最多、阻断推理链的关键环节

**例：**

```
目标：从现场痕迹推断凶手特征

缺口分析：
├── 身高 → 已知（墙壁写字高度距地六英尺 → 凶手约六英尺多）✓
├── 年龄 → 缺失（需要从步幅/行为模式推断）
├── 职业 → 缺失（需要从烟灰/工具痕迹推断）
└── 当前位置 → 矛盾（脚印方向与证人证词不符）
    → 需验证矛盾来源
```

**结束条件：** 识别出 1-3 个关键缺口，进入下一步。

---

### 第三步：制定查找策略

**目标：** 针对每个缺口，制定获取信息的具体行动计划。

**具体操作（AI 知道下一步做什么）：**

1. **对每个缺口，选择信息来源类型：**

   | 缺口类型 | 优先查找方式 |
   |----------|------------|
   | 物理证据缺失 | 检索/请求读取文件、代码库、或执行命令获取 |
   | 外部事实未知 | web_search / web_fetch |
   | 历史上下文 | memory_search / lcm_grep |
   | 语义/密码含义 | 对照已观察到的符号规律进行解码 |
   | 矛盾点验证 | 设计"证伪实验"（模拟排除或确认） |

2. **优先级排序：**
   - 先查物理上可验证的（文件、代码、命令输出）
   - 再查外部知识（搜索引擎）
   - 最后用已有信息做逻辑推断

3. **制定具体查找指令：**
   ```
   缺口A → 操作：read[文件路径]，目标：获取[具体内容]
   缺口B → 操作：web_search，关键词："[精确短语]"，目标：确认[事实]
   ```

**结束条件：** 每个关键缺口都有对应的查找计划。

---

### 第四步：执行查找

**目标：** 按计划获取缺失信息。

**具体操作（AI 知道下一步做什么）：**

1. **按优先级顺序执行查找指令**
2. **每次获取后立即更新上下文：**
   ```
   原始缺口：嫌疑人职业
   执行：read /path/to/file
   获取：烟灰分析记录（印度特里其城雪茄）
   更新：K_new = 吸烟特征 → 缩小职业范围（非普通人）
   ```
3. **发现新线索时：**
   - 暂停当前查找计划
   - 将新线索加入上下文（回到第一步）
   - 重新评估缺口优先级
   - 决定是否调整查找策略

4. **查找失败时：**
   - 记录失败原因
   - 尝试替代信息来源
   - 如果所有途径都失败，标记为"不可验证"，用"可能"而非"确定"

**结束条件：** 所有关键缺口已获得信息，或已用尽可用查找手段。

---

### 第五步：验证

**目标：** 确认获取的信息与已知事实一致，无矛盾。

**具体操作（AI 知道下一步做什么）：**

1. **逐项验证新获取的信息：**
   - 与已知事实对照：是否矛盾？
   - 与其他新信息对照：是否互相印证？
   - 是否存在反例？（福尔摩斯："某个环节不能和整条推理链相吻合，这个环节必定另有解释"）

2. **证伪检验（核心）：**
   ```
   假设：凶手是德国人
   验证：德文字体写法 → 发现A字母是拉丁字体，不是德文
   结果：假设被推翻 → 转向"凶手伪装成德国人"
   ```

3. **构建完整的验证链：**
   ```
   新信息 + 已知事实 → 能否推出结论？
   若能 → 结论可提升为"确定"
   若不能 → 返回第三步，补充新的查找策略
   ```

**结束条件：** 关键结论无矛盾，或矛盾已解决（如发现"另一粒药丸有毒"）。

---

### 第六步：结论

**目标：** 输出有证据支撑的确定结论。

**具体操作（AI 知道下一步做什么）：**

1. **结论必须满足：**
   - 每一步都有已知事实支撑
   - 无矛盾点未解决
   - 结论是逻辑必然，非统计概率

2. **结论格式：**
   ```
   结论：[核心判断]
   支撑链路：
   K1 → K2 → K3 → ... → 结论
   置信度：确定 / 高度可能 / 可能（需进一步验证）
   ```

3. **保留不确定性：**
   - 如果还有缺口无法填补，明确标注"可能"而非混入确定结论
   - 如果假设被证伪，记录证伪过程和转向依据

4. **反向验证：**
   - 用结论反向推演现场/上下文，看是否与所有已知事实一致
   - 若出现新矛盾，返回第二步重新分析

---

## 三、工作流总览

```
上下文建立（第一步）
    ↓
    [缺口分析（第二步）]
    ↓
制定查找策略（第三步）
    ↓
执行查找（第四步）
    ↓
    ├── 成功 → 更新上下文 → 回到第二步分析新缺口
    ├── 发现新线索 → 回到第一步重建上下文
    └── 失败 → 记录不可验证 → 用"可能"标注
    ↓
验证（第五步） ←→ 若出现矛盾 → 返回第三步调整策略
    ↓
结论（第六步） ←→ 若出现新矛盾 → 返回第二步
```

## 四、福尔摩斯五大支柱（对应 AI 工作流）

| 支柱 | 原文 | AI 工作流对应 |
|------|------|-------------|
| **链条因果观** | "生活是根巨大的链条，观其一环便可知整体本质" | 第一步：建立完整上下文；第二步：识别缺失环节 |
| **系统观察法** | 手指甲→职业，靴子→行走区域 | 第一步：收集所有可直接观察的信息 |
| **知识管理哲学** | "阁楼论"——只存对工作有用的工具 | 第三步：选择最有效的查找策略，避免无关信息干扰 |
| **压缩推理链** | "瞬间闪过，历时不到一秒钟" | 第五步验证 + 第六步结论：确保每步有据，结论是必然链 |
| **主动验证** | 登失物启事；药丸实验 | 第四步：主动执行查找，而非被动等待；设计证伪实验 |

## 五、常见错误对照

| 错误 | 福尔摩斯案例 | 正确做法 |
|------|-----------|---------|
| 把"可能"当"确定" | "斑点带子"最初以为是吉普赛人 | 用第五步证伪，明确区分"确定"与"可能" |
| 忽略矛盾点 | 第一粒药丸无毒→大多数人放弃假设 | 矛盾是线索，不是障碍："这个环节必定另有解释" |
| 假设未被验证就下结论 | 跳舞小人最初以为恶作剧 | 每个假设都经过第三步→第四步→第五步验证 |
| 不主动查，只等线索 | 被动等待现场线索 | 主动制定查找策略，执行检索 |
