```markdown

## 1. 生态全景图（Mermaid）

\`\`\`mermaid
flowchart TB
    subgraph 上游
        A1[原料供应<br/>公司A<br/>公司B]
        A2[核心组件<br/>公司C]
    end
    subgraph 中游
        B1[核心制造<br/>公司D<br/>公司E]
        B2[平台运营<br/>公司F]
    end
    subgraph 下游
        C1[终端应用<br/>公司G<br/>公司H]
        C2[分销渠道<br/>公司I]
    end
    subgraph 横向支持
        D1[IT / 数据]
        D2[金融 / 资本]
        D3[监管 / 认证]
    end
    A1 --> B1
    A2 --> B1
    B1 --> C1
    B1 --> C2
    D1 -.-> B1
    D2 -.-> B1
    D3 -.-> B1
\`\`\`
```

## Mermaid 语法要点

1. 用 `flowchart TB`（自上而下）或 `LR`（左右）
2. 用 `subgraph` 分组（上游/中游/下游/横向支持）
3. 实线 `-->` 表示上下游供应关系
4. 虚线 `-.->` 表示横向支持关系
5. 节点文字用 `<br/>` 换行
6. 避免过深嵌套，复杂时拆成 2-3 个子图

## 示例（医疗行业片段）

```mermaid
flowchart TB
    subgraph 上游[上游：原料/器械/技术]
        A1[原料药<br/>恒瑞 / 复星]
        A2[医疗器械<br/>迈瑞 / 联影]
        A3[生物试剂<br/>药明康德]
    end
    subgraph 中游[中游：制造/平台]
        B1[药品制造<br/>恒瑞 / 百济神州]
        B2[器械制造<br/>迈瑞 / 微创]
        B3[医疗服务<br/>美中宜和 / 陆道培]
        B4[数字医疗平台<br/>微医 / 京东健康]
    end
    subgraph 下游[下游：分销/终端]
        C1[医药商业<br/>国药 / 上海医药]
        C2[医院诊所<br/>协和 / 华西]
        C3[零售药店<br/>老百姓 / 益丰]
        C4[互联网医院<br/>平安好医生]
    end
    subgraph 横向[横向支持]
        D1[IT/AI<br/>华为 / 阿里 / 百度]
        D2[保险<br/>平安 / 众安]
        D3[监管<br/>NMPA / 卫健委]
    end
    A1 --> B1
    A2 --> B2
    B1 --> C1
    B1 --> C2
    B2 --> C2
    B4 --> C4
    D1 -.-> B4
    D2 -.-> C4
    D3 -.-> B1
```
