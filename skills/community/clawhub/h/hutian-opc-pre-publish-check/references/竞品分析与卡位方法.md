# 竞品分析与卡位方法

## 1. 方法论概述

竞品分析是确保Skill/Agent在市场中找到精准定位的核心工作。通过系统性地扫描、拆解、对比同类产品，明确自身的差异化优势和市场卡位，避免同质化竞争和定位模糊。

**核心目标**：
1. 发现市场空白区（机会点）
2. 明确差异化方向（我方独特价值）
3. 评估可防守性（护城河分析）

## 2. 竞品扫描流程

### 2.1 搜索范围

| 平台 | 搜索关键词示例 | 备注 |

| GitHub | skill / agent / AI assistant | 开源参考 |
| 其他AI平台 | [功能]+assistant / bot | 间接竞品 |

### 2.2 竞品筛选标准

**直接竞品判定**：
- 功能相似度 ≥ 60%
- 目标用户有重叠
- 解决同类问题

**间接竞品判定**：
- 功能相似度 30-60%
- 目标用户部分重叠
- 可替代但不完全相同

## 3. 功能异同拆解

### 3.1 功能清单对比表

**格式**：
| 功能点 | 我方 | 竞品A | 竞品B | 竞品C |
|--------|------|-------|-------|-------|
| **核心功能** | | | | |
| 功能1 | ✅完整 | ✅完整 | ⚡部分 | ❌无 |
| 功能2 | ✅完整 | ❌无 | ✅完整 | ⚡部分 |
| **辅助功能** | | | | |
| 功能3 | ⚡部分 | ✅完整 | ❌无 | ✅完整 |
| ... | | | | |

**标注说明**：
- ✅完整：功能完整实现，效果好
- ⚡部分：功能有但不完整或效果一般
- ❌无：该功能不存在

### 3.2 差异化矩阵

**格式**：
| 差异化维度 | 我方 | 竞品A | 竞品B | 差异评价 |
|------------|------|-------|-------|----------|
| 技术实现 | 语义理解+向量检索 | 关键词匹配 | 混合模式 | 领先 |
| 场景覆盖 | 聚焦技术创新 | 通用场景 | 聚焦市场 | 差异化 |
| 输出深度 | 专业报告+建议 | 仅报告 | 报告+可视化 | 持平 |
| 用户引导 | 分步向导 | 单次执行 | 交互式 | 领先 |
| ... | | | | |

**短板功能（竞品有我方没有）**：

## 4. 卡位精准性评估

### 4.1 目标用户画像对比

| 用户维度 | 我方目标用户 | 竞品A用户 | 竞品B用户 |
|----------|--------------|-----------|-----------|
| 职级 | 研发经理/CTO | 研发人员 | 项目经理 |
| 行业 | 制造业/科技 | 通用 | 服务业 |
| 痛点 | 技术创新方法缺失 | 效率低 | 成本高 |
| 付费意愿 | 中高 | 低 | 中 |
| 技术水平 | 中高 | 中低 | 中 |

### 4.2 卡位类型分析

**价格带卡位**：
- 我方定价：[X元]
- 竞品A定价：[X元]
- 竞品B定价：[X元]
- 价格空白区：[X元区间是否有机会]

**功能深度卡位**：
- 竞品选择：做广度 → 我方选择：做深度
- 或：竞品选择：做通用 → 我方选择：做垂直

**场景卡位**：
- 竞品选择：通用场景 → 我方选择：[具体垂直场景]
- 示例：竞品做"通用查新"，我方做"TRIZ技术创新专项查新"

### 4.3 卡位评分表

| 评分维度 | 评分(1-5) | 说明 |
|----------|-----------|------|
| 卡位清晰度 | | 目标用户是否明确知道这是为他们设计的 |
| 差异化程度 | | 与竞品的差异是否可感知、可表述 |
| 市场空间 | | 该细分市场的规模是否足够 |
| 可执行性 | | 我们的能力是否支撑这个卡位 |
| 可防守性 | | 该卡位是否容易被竞品复制 |

**判定标准**：
| 总分 | 判定 | 建议 |
|------|------|------|
| ≥20 | ✅强卡位 | 可发布 |
| 15-19 | ⚠️中等卡位 | 可发布但需强化差异化 |
| <15 | 🛘弱卡位 | 需重新定位 |

## 5. 护城河分析

### 5.1 护城河类型

| 护城河类型 | 适用场景 | 建设成本 | 防御强度 |
|------------|----------|----------|----------|
| IP保护 | 有可专利的创新点 | 中 | 强 |
| 数据积累 | 用户行为数据越用越准 | 高 | 中 |
| 网络效应 | 用户越多价值越大 | 高 | 强 |
| 先发优势 | 最早进入细分市场 | 低 | 中 |
| 品牌认知 | 特定场景=特定品牌 | 高 | 中 |
| 转换成本 | 用户习惯后不易迁移 | 中 | 中 |

### 5.2 独家功能可模仿性评估

**评估格式**：
```
独家功能：[功能名]
可模仿难度：[高/中/低]
模仿时间预估：[X-Y个月]
护城河来源：[IP/数据/先发/其他]
可持续性：[长期/中期/短期]
```

**示例**：
```
独家功能：TRIZ语义理解查新
可模仿难度：高（需要TRIZ领域知识+AI技术结合）
模仿时间预估：6-12个月
护城河来源：先发优势+IP保护
可持续性：中期（3-5年）
```

### 5.3 护城河建设建议

**短期（<6个月）**：
1. 申请软件著作权保护
2. 积累用户反馈数据
3. 建立用户口碑

**中期（6-18个月）**：
1. 申请发明专利
2. 构建数据壁垒
3. 发展付费用户

**长期（>18个月）**：
1. 品牌建设
2. 生态建设（API开放、插件体系）
3. 社区运营
