# S-04 竞品对标分析

> 版本：v1.0 | 定价：¥49 | 适用对象：OPC创业者/需要了解竞争格局的创业项目

---

## 一、Skill概述

### 1.1 核心功能

本Skill为「途途创业咨询师」的核心分析Skill，通过系统化竞品研究框架，帮助用户全面了解目标市场的竞争格局、竞品优劣势、以及自身的差异化定位机会。

### 1.2 价值主张

> **不是让你盲目模仿竞品，而是帮你找到"只有你能做"的独特位置**

### 1.3 适用场景

- 用户准备进入某个赛道，想了解竞争对手情况
- 用户已有产品/服务，想找差异化突破口
- 用户融资BP需要竞品分析章节
- 用户想评估自身竞争力，制定竞争策略
- 用户想找对标学习的行业标杆

### 1.4 输入要求

| 字段 | 类型 | 要求 | 示例 |
|-----|------|------|------|
| company_name | string | 必须 | "我的公司叫XX科技" |
| product_service | string | 必须，≥50字 | "做面向Z世代的在线健身平台" |
| target_market | string | 可选 | "一线城市18-30岁年轻女性" |
| specific_competitors | list | 可选 | ["竞品A", "竞品B"] |

### 1.5 输出交付物

```
┌─────────────────────────────────────────────────────────────┐
│  📊 竞品对标分析报告                                        │
├─────────────────────────────────────────────────────────────┤
│  一、执行摘要                                               │
│  二、竞品矩阵（3-5个核心竞品多维对比）                       │
│  三、竞品画像卡（每个竞品详细档案）                          │
│  四、差异化定位建议（空白区域发现）                          │
│  五、竞争策略建议（四选一：正面/侧翼/差异化/蓝海）           │
│  六、护城河评估（六维度评分）                                │
│  七、行动建议                                               │
└─────────────────────────────────────────────────────────────┘
```

## 二、工作流程（完整8步）

### 步骤1：信息收集与需求确认

**目标**：明确分析范围和重点

**执行动作**：
1. 解析用户输入，确认：
   - 核心产品/服务是什么
   - 目标用户是谁
   - 目标市场/赛道

2. 若信息不足，按以下优先级追问：

**追问话术模板**：

```
💬 途途：
感谢分享！我来帮你做一次全面的竞品对标分析。

在开始之前，我需要再确认几个关键信息：

【必须确认】
1. 你的核心产品/服务是什么？解决什么具体问题？
2. 你的目标用户群体是哪类人？
3. 你是否已经有明确的竞品名单？（如果没有，我会帮你搜索发现）

【可选补充】
4. 你目前在市场中的位置？（刚起步/已有用户/想进入新市场）
5. 你最想了解竞品的哪些方面？（功能/定价/用户体验/获客策略）

信息越完整，分析越精准！
```

### 步骤2：竞品发现与筛选

**目标**：通过搜索发现并筛选3-5个核心竞品

**搜索策略**：

```
竞品发现搜索词组合：
- "[你的产品类型] AI工具"
- "[你的赛道] 创业咨询"
- "[你的用户群体] + 痛点 + 解决方案"
- "竞品名 + 融资/用户量/功能"

筛选标准：
1. 直接竞品（同解决同一问题）
2. 间接竞品（解决同类问题不同方式）
3. 潜在竞品（大公司可能进入）
4. 标杆产品（值得学习但定位不同）
```

**竞品筛选矩阵**：

| 筛选维度 | 必须满足 | 加分项 |
|---------|---------|-------|
| 市场地位 | 在目标市场有实际用户 | 融资/规模化 |
| 产品相似度 | 解决同类问题 | 目标用户重叠 |
| 数据可获取 | 能找到公开信息 | 有详细资料 |
| 代表性 | 能代表一种竞争类型 | 头部/创新/垂直 |

**输出格式**：

```

┌─────────────────────────────────────────────────────────────┐
│  直接竞品（3个）                                             │
│  ─────────────────────                                      │

│  2. [竞品名] - [一句话定位] - [市场地位]                     │
│  3. [竞品名] - [一句话定位] - [市场地位]                     │
├─────────────────────────────────────────────────────────────┤
│  间接竞品（2个）                                             │
│  ─────────────────────                                      │
│  4. [竞品名] - [一句话定位] - [与你的关系]                   │
│  5. [竞品名] - [一句话定位] - [与你的关系]                   │
├─────────────────────────────────────────────────────────────┤
│  潜在竞品（1个）                                             │
│  ─────────────────────                                      │
│  6. [大公司/平台] - [可能进入的方式]                          │
```

### 步骤3：六维数据收集

**目标**：收集每个竞品的核心数据

**数据收集维度**：

#### 维度1：产品功能

```
收集内容：
- 核心功能清单（列出前5个）
- 特色功能/差异化功能
- 技术亮点
- 产品更新频率
- 用户评价中的功能提及

信息来源：
- 官网功能介绍
- App Store/产品 Hunt
- 用户评价/评测文章
```

#### 维度2：定价策略

```
收集内容：
- 定价方案（免费/订阅/一次性）
- 价格区间（最低/最高/主流）
- 定价逻辑（按用户量/功能/时长）
- 促销/折扣策略
- 试用政策

信息来源：
- 官网定价页
- G2/Capterra等评测平台
```

#### 维度3：用户画像

```
收集内容：
- 目标用户类型
- 用户规模（月活/注册量）
- 用户评价摘要
- 典型用户场景
- 用户满意度/评分

信息来源：
- 官网案例
- 社交媒体讨论
- 应用商店评价
```

#### 维度4：渠道策略

```
收集内容：
- 主要获客渠道
- 内容营销策略
- 社交媒体布局
- 合作/分销渠道
- 用户增长策略

信息来源：
- 官网/社交媒体
- SimilarWeb数据
- 行业分析报告
```

#### 维度5：技术路线

```
收集内容：
- 核心技术架构
- AI/LLM使用情况
- 数据积累情况
- 集成/开放能力
- 技术团队规模

信息来源：
- 技术博客
- LinkedIn团队信息
- 技术评测文章
```

#### 维度6：融资阶段

```
收集内容：
- 融资金额与轮次
- 投资方背景
- 估值水平
- 资金用途
- 商业模式成熟度

信息来源：
- 官网/PR稿
- IT桔子/Crunchbase
- 36氪/创业邦报道
```

### 步骤6：差异化定位分析

**目标**：发现市场空白，找到差异化机会

**分析方法**：

```
1. 功能矩阵分析
   - 列出所有竞品的功能
   - 标记"已有功能"和"缺失功能"
   - 找出无人覆盖的功能组合

2. 用户群体分析
   - 各竞品主要服务哪类用户
   - 是否有被忽视的用户群体
   - 用户痛点是否被充分解决

3. 定价空档分析
   - 各竞品的定价区间
   - 是否有价格空白带
   - 是否有付费意愿未被满足

4. 体验差距分析
   - 用户吐槽最多的问题
   - 竞品普遍忽视的体验点
   - 可超越的机会点
```
