# 回复解析规则

## 概述

解析用户对意图澄清问卷的回复，支持多种格式。

## 支持的回复格式

### 格式一：结构化符号（推荐）

```
1→600519,2→B,3→A
```

| 分隔符 | 示例 |
|-------|------|
| `,` | 分隔不同字段 |
| `→` 或 `:` 或 `=` | 分隔序号和值 |
| `#` | 注释（忽略） |

**解析逻辑：**
```javascript
function parseStructuredReply(text) {
  const result = {};
  const pairs = text.split(',');
  for (const pair of pairs) {
    const [idx, value] = pair.split(/→|:=|=/).map(s => s.trim());
    result[parseInt(idx)] = value;
  }
  return result;
}
```

### 格式二：简短选择

```
B
```

仅回复单个选项时，匹配最近一次问卷的第1个问题。

### 格式三：自然语言混合

```
股票600519，技术面，近1月
```

**解析策略：**
1. 识别关键词映射
2. 上下文推断
3. 剩余字段标记为待填充

### 格式四：纯文本补全

```
600519
```

当问卷只有一个填空项时，直接作为答案。

## 回复状态机

```
用户回复
    ↓
判断格式类型
    ↓
┌─────────────┐
│ 结构化符号   │ → 直接解析
└─────────────┘
    ↓
┌─────────────┐
│ 短选择(A/B)  │ → 匹配最近问卷
└─────────────┘
    ↓
┌─────────────┐
│ 自然语言    │ → 关键词 + 上下文推断
└─────────────┘
    ↓
验证必填字段
    ↓
完整？ → 是 → 补全参数，执行
    ↓ 否
追问缺失字段
```

## 错误处理

### 格式错误

```javascript
const PARSE_ERRORS = {
  INVALID_INDEX: "序号无法识别，请回复如 '1→A,2→B'",
  INVALID_OPTION: "选项不存在，请从给出的选项中选择",
  MISSING_REQUIRED: "缺少必填字段：{field}",
  TIMEOUT: "回复已超时，请重新发起任务"
};
```

### 解析失败时

1. 尝试模糊匹配（部分匹配选项关键词）
2. 若仍失败，返回解析错误提示
3. 不自动放弃，等待用户重新回复

## 示例

### 示例1：结构化回复

**问卷：**
```
请补充：
1️⃣ 股票代码：______
2️⃣ 分析角度：A.技术面 B.基本面 C.全维度
3️⃣ 时间范围：A.近1周 B.近1月 C.近3月
```

**用户回复：**
```
1→600519,2→A,3→B
```

**解析结果：**
```json
{
  "stock_code": "600519",
  "analysis_type": "技术面",
  "time_range": "近1月"
}
```

### 示例2：自然语言

**用户回复：**
```
分析一下五粮液，技术面，近三个月
```

**解析结果：**
```json
{
  "stock_code": "000858",
  "stock_name": "五粮液",
  "analysis_type": "技术面",
  "time_range": "近3月"
}
```

### 示例3：部分补全

**用户回复：**
```
600519，技术面
```

**解析结果：**
```json
{
  "stock_code": "600519",
  "analysis_type": "技术面",
  "time_range": null  // 仍缺失
}
```

**追问：**
```
请补充时间范围：
A. 近1周  B. 近1月  C. 近3月
```

## 快捷命令

| 命令 | 效果 |
|-----|------|
| `r` / `重新来` | 重置当前澄清流程 |
| `取消` | 放弃当前任务 |
| `?` / `帮助` | 显示当前问卷 |
| `上一步` | 回退到上一问题 |
