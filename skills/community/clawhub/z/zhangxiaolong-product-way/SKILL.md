## Core Thinking Models

### 模型一：极简主义产品观
```
核心理念：少即是多，宁缺毋滥，做小做精
- 功能不是越多越好，每加一个功能都要问：去掉会怎样？
- 保持笨拙：好的产品不需要用户手册
- 克制比放肆更难，也更有价值
```

### 模型二：用户洞察三层次
```
第一层：满足用户已知需求（红海竞争）
第二层：发现用户潜在需求（蓝海创新）
第三层：创造用户从未想过的需求（改变世界）
→ 微信做的：第三层
```

### 模型三：需求优先级矩阵
```
                    用户需要
                    是        否
              ┌─────────┬─────────┐
        是    │   做    │  谨慎做  │
产品团队     ├─────────┼─────────┤
需要        否    │  观察   │   不做  │
              └─────────┴─────────┘
- 用户需要但产品团队不需要：考虑是否做
- 产品团队需要但用户不需要：绝对不做
```

### 模型四：迭代哲学
- 快速小步迭代：每两周最多三周发布新版本
- 追踪用户需求，而非追踪竞品
- "踏踏实实追踪用户的需求，而不是追踪市场报告"

### 模型五：产品气质论
- 好的产品有气质：有价值观、有性格
- 产品即人：做产品的人决定了产品的调性
- "把用户当人，见自己见众生，然后忘掉"

### 模型六：人文与技术双螺旋
- 技术是手段，人文是目的
- "在技术与人文的交汇处，构建符合自然规律的用户世界"
- 产品不应该是冰冷的工具，而是有温度的连接

## Workflow
```
【张小龙产品分析】
Step 1 - 需求诊断：用户真的需要吗？还是我们觉得用户需要？
Step 2 - 极简测试：去掉这个功能，产品会更好吗？
Step 3 - 气质检测：这个功能符合产品的核心价值观吗？
Step 4 - 迭代节奏：能否快速上线、小步验证？
Step 5 - 人文视角：用户用这个功能时，是开心的还是焦虑的？
```
