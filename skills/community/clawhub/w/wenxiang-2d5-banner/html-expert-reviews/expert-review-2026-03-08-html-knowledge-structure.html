💡 深度洞察

洞察 1：为什么 HTML 归档如此重要？

业务价值：知识沉淀与可视化

核心原因：

  • HTML 比 Markdown 更适合可视化展示（图表/配色/布局）
  • 统一归档便于查找和复用（html-expert-reviews/）
  • 项目关联形成知识网络（项目卡片 ↔ HTML）
  • 长期积累形成知识库资产（12 个 → 100 个 → 1000 个）

投资回报：极高（一次生成，永久复用）

洞察 2：项目关联的核心逻辑

关联原则：

  • 项目特定的 HTML → 关联到项目卡片
  • 跨项目/通用的 HTML → 标记为通用，无需关联
  • 关联方式：项目卡片中添加"📄 专家点评"章节
  • 链接格式：[HTML 标题](file://完整路径)

当前覆盖：5/12 = 42%（待提升到 80%+）

洞察 3：自动化流程的必要性

当前问题：

  • README.md 索引手动维护（易遗漏）
  • 项目关联手动更新（易忘记）
  • 缺少自动生成统计（无法追踪进展）

改进建议：

  • html-expert-review 技能自动更新 README.md
