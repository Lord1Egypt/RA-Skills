# 模块 1：盘面趋势

使用 `module1_market_trend.json`。

只分析：

- 上证指数
- 创业板指数
- `market.trend.sentiment`
- `market.temperature`

不要加入科创板、风格指数、融资、创业板 PE、外围资产或新闻解释。

分析框架：

1. 情绪趋势：结合量能温度、全市场换手率、情绪温度和广度温度。若情绪历史不可用，只保留当日盘面快照。
2. 上证趋势：判断趋势阶段、量价质量、均线位置和最近支撑压力。
3. 创业板趋势：使用同样四项框架，并与上证强弱对比。
4. 盘面定性：明确写成共振、背离或分化。

写作要求：

- 不要把单日涨跌写成趋势。
- 上证指数趋势、创业板指数趋势只写文字段落，不输出表格。指数数据用于支撑判断，关键数值直接嵌入句子，例如“收盘站上 MA20，成交额为近 20 日均量 1.2 倍，临近 20 日压力位”。这样避免指数部分挤占篇幅，也让报告重点回到盘面判断。
- 指数上涨但缩量时，写“反弹质量待确认”。
- 成交额与全市场换手率同步抬升、上涨家数同步改善时，写“良性放量加速，风险中性偏高”。
- 全市场换手率上升但上涨/下跌家数转弱时，写“换手加速但承接分化，风险升温”。
- 指数强但广度/情绪弱时，写“指数掩盖分化”。
- 情绪先修复但指数未确认时，写“情绪先行修复但指数尚未确认”。
