# 四层骨架写法详解

SKILL.md 的四层结构对应 Agent 读取时的四个递进阶段。每一层写错了，后面的效果打折。

---

## 第一层：头部（description）——触发器，不是摘要

### 作用

Agent 启动时扫描所有 Skill 的 description，判断"这个任务该不该用这个 Skill"。写不好，Agent 根本不会往下读。

### 写法

```
第一句：说清楚这个 Skill 做什么
中间：Use when user asks to + 触发短语列表（覆盖用户可能的多种表达）
最后：不适用于 XXX（划定边界，防止误触发）
```

### 正误对比

❌ **功能摘要式**（缺触发词，Agent 匹配不到）

```yaml
description: WeSpy的封装，支持微信公众号文章抓取和专辑批量下载。
```

❌ **过于宽泛**（没有边界，什么任务都可能误触发）

```yaml
description: 文章处理工具。Use when user asks to 处理文章、抓取内容、下载东西.
```

✅ **触发器式**（有触发词、有多表达覆盖、有边界）

```yaml
description: 获取并转换微信公众号/网页文章为Markdown的封装Skill。
  Use when user asks to 抓取微信公众号文章、抓微信、
  下载公众号文章、URL转Markdown、微信文章保存.
  不适用于通用网页抓取或非微信内容.
```

### 原则

- `Use when user asks to` 是给 Agent 的匹配信号，不是注释
- 同一件事用户可能有多种说法，都要覆盖（"抓取""下载""保存""转换"）
- 不适用边界防止误触发，和触发词一样重要

---

## 第二层：概述——画边界，不画地图

### 作用

Agent 通过了第一层筛选，快速确认能力范围是否匹配。

### 写法

```
一句话定位：说清楚能力边界
功能范围列表：只列"能做什么"，每条一行
```

### 正误对比

❌ **塞进操作细节**

```markdown
## 功能范围
- 单篇文章抓取，使用 python3 scripts/wespy_cli.py URL 命令
- 微信专辑文章列表获取，加上 --album-only 参数
```

问题：Agent 还没决定用不用你，就让它看命令行。

✅ **只画边界**

```markdown
## 概述

封装 WeSpy 的完整能力。

### 功能范围

- 单篇文章抓取（微信公众号 / 通用网页 / 掘金）
- 微信专辑文章列表获取
- 微信专辑批量下载
- 多格式输出（Markdown / HTML / JSON）
```

### 原则

- "能做什么"归概述，"怎么做"归操作指南
- 一句话定位要说清边界：就是对 XX 的封装、做 YY 类型的任务
- 功能范围列表是快速扫描用的，不需要解释

---

## 第三层：操作指南——给场景，不只是给命令

### 作用

Agent 确认要用这个 Skill 了，在这里找具体的操作步骤。是 Agent 逗留时间最长的部分。

### 写法

按场景给完整的命令/步骤示例。不同使用场景有不同的示例，不只是一个通用模板。

### 正误对比

❌ **参数说明书**

```markdown
## 参数说明
- URL：文章或专辑的链接
- --album-only：仅获取专辑列表
- --max-articles N：批量下载时指定数量
- --format FORMAT：输出格式

命令：python3 scripts/wespy_cli.py [URL] [OPTIONS]
```

问题：Agent 看到通用模板得自己拼命令。每多一步判断，多一次出错。

✅ **场景示例**

```markdown
## 使用

脚本位置：scripts/wespy_cli.py

# 单篇文章抓取
python3 scripts/wespy_cli.py "https://mp.weixin.qq.com/s/xxxxx"

# 专辑批量下载（下载前20篇）
python3 scripts/wespy_cli.py "URL" --max-articles 20

# 仅获取专辑文章列表（不下载）
python3 scripts/wespy_cli.py "URL" --album-only

# 输出为 HTML 格式
python3 scripts/wespy_cli.py "URL" --format html
```

Agent 直接对号入座，不需要自己拼。

### 原则

- 至少 2 个典型场景的完整示例
- 每个场景有注释说明 + 完整命令/步骤
- 不同类型的 Skill，场景示例的形式不同：
  - 工具封装型：给完整命令行
  - 生成器型：给输入→输出的示例对
  - 审查器型：给审查对象 + 审查要点
  - 流水线型：给每阶段的输入输出

---

## 第四层：补充说明——堵死岔路口

### 作用

Agent 执行中遇到问题来查这一层。把可能走错的判断点提前消灭。

### 需要覆盖的典型场景

| 场景 | 写什么 |
|------|--------|
| 依赖不存在 | 自动安装？报错？降级方案？给出具体路径和命令 |
| 输入格式不对 | 怎么判断？怎么修正？ |
| 输出目录不存在 | 自动创建？报错？ |
| 网络超时 | 重试策略？降级方案？ |
| 同名文件已存在 | 覆盖？跳过？追加？ |
| 多种输入源 | 优先级是什么？ |

### 正误对比

❌ **没有补充说明**

Agent 执行到一半发现 WeSpy 没安装 → 直接报错 / 自己猜 `pip install wespy` / 去搜索。

✅ **堵死岔路口**

```markdown
## 实现说明

- 优先使用本地源码路径 ~/Documents/QNSZ/project/WeSpy
- 若本地不存在则自动执行 git clone 到该目录
- 通过导入 wespy.main.main 直接调用上游 CLI，保持行为一致
```

### 原则

- 每一个踩过的坑都写进来——Skill 会越来越"聪明"
- 给出具体的兜底方案，不只是说"处理异常"
- 补充说明是沉淀层，会随使用逐渐丰富
