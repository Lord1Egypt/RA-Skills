## Description: <br>
Use Kadence Theme as a design system for WordPress delivery. Covers design tokens, layout system, header/footer/navigation setup, page/archive/single templates, and a practical Free vs Pro capability matrix so agents choose valid implementations before building. <br>

This skill is ready for commercial/non-commercial use. <br>

## Use Case: <br>
WordPress site builders and developers use this skill to plan Kadence Theme pages, templates, headers, footers, navigation, and design tokens while choosing Free-safe or Pro-verified implementation paths. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: An agent may recommend Kadence Pro-only features before the site's license and installed feature tier are confirmed. <br>
Mitigation: Confirm the active Kadence theme and Free vs Pro status before implementation; when status is unknown, use the Free-safe fallback path. <br>
Risk: Real WordPress site changes can affect production content, templates, navigation, or layout. <br>
Mitigation: Confirm the target environment and require explicit approval for production publishing, destructive bulk edits, or PHP/theme/plugin file changes. <br>
Risk: Kadence documentation and feature availability can change over time. <br>
Mitigation: Re-check the linked Kadence documentation before asserting current feature availability or Pro requirements. <br>
