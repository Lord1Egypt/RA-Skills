## 

### 名称
`core-bid-winner-profiler`

### Slug
`core-bid-winner-profiler`

### 定位（用户可见）
结构化汇总企业近期公开中标的核心标的，剖析其重点采购方特征及客单价，研判其近期营收基本盘与核心大客户资源。

### 描述（LLM识别）
精通政企采购规则与GTM（走向市场）策略的商务数据分析师。当用户询问某企业最近都在接什么类型的项目、赚谁的钱，试图了解某企业在政府端/大B端的真实落地能力和现金流状况，或要求分析目标企业的典型客户画像时，激活此技能。

执行逻辑：
1. 实体提取：明确指令中的“企业全称”（招投标信息通常要求全称匹配最准）。
2. 触发联网搜索：针对公开采购网、招投标公示平台及财经资讯聚合搜索。搜索逻辑包含：“企业全称” AND (“中标公示” OR “单一来源采购” OR “成交公告” OR “大单”)，优先关注金额较大的项目。
3. 结构化信息抽取：从非结构化的中标新闻中，强制解析出三个核心字段：【采购单位名称】、【中标金额】、【项目/标的名称】。
4. 客群画像归纳：对提取到的采购单位进行聚类分析（例如：发现其近期主要中标对象多为“某某市公安局”、“某某省交通厅”），推导出其核心收入依赖的行业或政务条线。
5. 输出格式：《核心标的与大单分析报告》，包含：近期亿元/千万级大单盘点、核心采购方画像（买单者是谁）、业务深耕区域推演。

### 适用场景
- 招商尽职调查中验证企业的真实营收来源与客户质量
- 投资人判断企业的增长可持续性及大客户依赖风险
- 竞品分析中了解对手的市场渗透策略
- 企业自身寻找可对标的标杆客户案例

### 输入参数
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `company_name` | string | 是 | 企业全称（建议使用工商注册全名，提高中标信息匹配度） |
| `time_range` | int | 否 | 搜索时间范围（月），默认12个月 |
| `min_amount` | string | 否 | 最小中标金额过滤，如“100万”、“1000万”，默认不设下限 |
| `bid_type` | string | 否 | 招标类型，可选`all`（全部）、`government`（政府）、`enterprise`（企业），默认`all` |

### 输出格式
输出一份《核心标的与大单分析报告》，包含以下模块：

1. **企业基础信息**：企业全称、统一信用代码（如有）、所属行业
2. **近期亿元/千万级大单盘点**（表格形式）  
   - 中标时间 | 采购单位 | 中标金额 | 项目名称 | 项目类型（硬件/软件/服务/集成）
3. **核心采购方画像（买单者是谁）**  
   - 按行业/部门聚类（如：公安系统、交通局、能源央企、地方城投）  
   - 各聚类的中标金额占比（估算）  
   - 代表性客户名称
4. **业务深耕区域推演**  
   - 中标项目地域分布热力图（按省份/城市累计金额）  
   - 优势“粮仓”区域（连续中标、金额大）  
   - 新开拓区域（近半年首次中标）  
   - 推演内容标注“【推演】”
5. **客单价与现金流健康度评估**  
   - 平均中标金额  
   - 大单集中度（前3大客户占总中标金额比例）  
   - 结论（如“对大客户依赖度较高，存在一定风险”）

### 执行流程（内部步骤）

1. **实体提取与名称标准化**  
   - 从用户输入中提取 `company_name`，建议用户使用工商注册全称。  
   - 若用户输入为简称，可先通过 `search_ent_simple.py` 查询全称，或提示用户提供全称以提高匹配精度。

2. **联网搜索策略**（使用 `web_search` 工具）  
   - 核心检索词：  
     - `"{company_name} 中标"`  
     - `"{company_name} 中标公示"`  
     - `"{company_name} 政府采购 成交"`  
     - `"{company_name} 单一来源采购"`  
   - 优先来源：中国政府采购网、各省级公共资源交易中心、天眼查/企查查招投标板块、剑鱼标讯、采招网。  
   - 时间过滤：限定在 `time_range` 内。  
   - 金额过滤：若指定 `min_amount`，优先展示高于该金额的项目。

3. **结构化信息抽取**  
   - 从每条中标公告中提取：  
     - **采购单位名称**：注意区分代理机构与实际使用单位（如“XX市公安局交通警察支队”）。  
     - **中标金额**：统一转换为“万元”或“亿元”单位，保留两位小数。  
     - **项目/标的名称**：提取核心内容，去除冗余描述（如“XXX项目中标结果公告”）。  
     - **招标方式**：公开招标、邀请招标、单一来源、竞争性磋商等。  
   - 若同一公告包含多个标段，合并计算总金额。  
   - 若金额未公开，标注“未披露”。

4. **客群画像归纳**  
   - 对采购单位进行聚类：  
     - 政府部门（按级别：国家级/省级/市级/区县级；按条线：公安、交通、教育、卫健等）  
     - 事业单位（学校、医院、科研院所）  
     - 国有企业（央企、地方国企、城投平台）  
     - 民营企业（行业分类）  
   - 计算各聚类的中标金额占比（基于可获取的公开数据，缺失部分标注“部分未披露”）。

5. **地域分布与区域推演**  
   - 提取采购单位所在地（省/市）。  
   - 累计各区域中标金额，生成分布表。  
   - 识别“粮仓区域”：连续2年以上有中标记录且金额排名前3。  
   - 识别“新开拓区域”：近6个月内首次出现中标的省份/城市。  
   - 推演企业下一步可能重点拓展的区域（基于周边辐射或行业政策）。

6. **输出报告**  
   - 大单盘点表格优先展示金额最高的3-5个。  
   - 客群画像用百分比或柱状图文本描述。  
   - 现金流评估给出明确结论（健康/一般/有风险）。

### 错误处理与注意事项

| 错误类型 | 处理方式 |
|---------|----------|
| 未找到任何中标信息 | 返回“近{time_range}个月内未公开查询到该企业的中标公示，可能企业未参与公开招标或以非公开方式获得订单” |
| 中标金额未披露 | 标注“金额未公开”，不计入统计总金额，但保留项目信息 |
| 企业全称不匹配 | 提示“请使用工商注册全称，例如‘XX科技有限公司’，以提高查询准确率” |
| 部分采购单位名称模糊 | 仅保留可识别的部分，标注“信息不全” |

### 扩展能力
- **竞争对手中标对比**：输入另一家企业，对比同期中标金额、客户重叠度。  
- **中标趋势图表**：输出按月/季度中标金额折线图（文字描述形式）。  
- **单一来源风险预警**：自动标注占比过高的单一来源采购项目，提示可能依赖关系或合规风险。
