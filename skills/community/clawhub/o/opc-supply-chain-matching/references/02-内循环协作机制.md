# 内循环协作机制

## 1. 社区分包网定位

### 1.1 核心价值

社区分包网是OPC社区内部的能力共享网络，解决"有人需要、有人能供"的信息撮合问题。

| 价值维度 | 具体表现 |
|---------|---------|
| **信任基础** | 社区成员相互了解，有信用记录 |
| **成本优势** | 无中间环节，价格透明 |
| **响应速度** | 社区内沟通快，响应及时 |
| **协作深度** | 可深度配合，非标准化需求也能处理 |

### 1.2 适用场景

- 小批量需求（<100件）
- 非标定制需求
- 需要多方协作的综合项目
- 保密要求高的技术资料
- 快速验证类需求

## 2. 内循环能力分类体系

### 2.1 能力大类

| 大类 | 涵盖能力 | 典型交付物 |
|------|---------|-----------|
| **设计类** | 外观设计、结构设计、UI设计、平面设计 | 设计图稿、渲染图、工程图 |
| **研发类** | 硬件开发、嵌入式软件、算法开发、APP开发 | 原理图、代码、算法模型 |
| **加工类** | 3D打印、CNC加工、手板制作、激光切割 | 实体样件、手板模型 |
| **电子类** | PCB设计、焊接贴片、电子测试、可靠性测试 | PCB板、焊接组件 |
| **软件类** | Web开发、小程序、系统集成、技术咨询 | 应用程序、接口文档 |
| **认证类** | 认证咨询、安规测试、EMC测试 | 测试报告、认证证书 |

### 2.2 能力标签体系

每位OPC可设置多个能力标签，便于精准匹配：

```
标签格式：[大类]-[细分领域]-[熟练度]
示例：
- 设计-工业设计-熟练
- 研发-嵌入式-精通
- 加工-3D打印-熟练
- 电子-PCB设计-精通
```

## 3. 内循环协作流程

### 3.1 完整流程（6步）

```
Step 1：能力登记
├─ OPC在平台登记能力
├─ 设置可承接范围（类型、数量、周期）
└─ 设置价格区间（参考定价）

Step 2：需求发布
├─ 有需求的OPC发布分包需求
├─ 填写需求详情（类型、参数、预算、时间）
└─ 选择发布范围（社区内公开/定向邀请）

Step 3：智能匹配
├─ 平台根据能力标签匹配候选方
├─ 结合负载状态（当前项目数）筛选
├─ 参考历史评价排序
└─ 推荐候选方列表

Step 4：双方对接
├─ 需求方与候选方直接沟通
├─ 确认技术细节和合作条件
├─ 明确交付标准和分账方式
└─ 达成合作意向

Step 5：签约执行
├─ 通过平台签署分包协议
├─ 约定交付节点和验收标准
├─ 进入执行阶段
└─ 定期更新进度

Step 6：验收结算
├─ 交付完成后按协议验收
├─ 确认交付物质量
├─ 按协议分账
└─ 双方互评，更新信用记录
```

### 3.2 快速撮合通道

对于紧急或简单需求，提供快速撮合通道：

```
需求发布 → 24小时内响应 → 直接对接 → 简化签约 → 执行交付
```

适用条件：
- D级微项目（<5000元）
- 单一能力需求
- 需求描述清晰

## 5. 质量保障机制

### 5.1 交付标准要求

所有内循环协作必须明确交付标准：

| 标准类型 | 要求 | 示例 |
|---------|------|------|
| **交付物** | 明确列出 | 设计稿源文件+PDF |
| **格式要求** | 说明文件格式 | AI/PDF/DWG |
| **验收方法** | 可执行的检验方式 | 图纸对比目检 |
| **修改次数** | 明确修改轮次 | 3轮免费修改 |

### 5.2 争议处理

| 争议类型 | 处理方式 | 责任人 |
|---------|---------|--------|
| 交付物不符合要求 | 协商返工或折价 | 双方协商 |
| 沟通理解偏差 | 重新明确需求 | 需求方承担 |
| 严重质量问题 | 按协议退款 | 三长老仲裁 |
| 违约行为 | 扣除信用分+赔偿 | 平台强制执行 |
