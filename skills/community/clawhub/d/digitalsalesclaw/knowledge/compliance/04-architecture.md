# 合规审核架构：语义优先

## 核心理念

**旧架构**：正则匹配 → 累加severity → 评分  
**本质**：字符串操作，不理解语义

**新架构**：语义解析 → 知识图谱上下文 → 结构化证据 → AI推理判断  
**本质**：证据提供器，AI是真正的判断者

---

## Guardrail 规则（仅8条绝对禁区）

Guardrail 是最后一道防线，不是主要审核机制。

| 级别 | 规则 | 触发条件 |
|------|------|---------|
| critical | 绝对化疗效承诺 | `根治\|治愈\|保证治愈\|彻底治愈\|包治百病` |
| critical | 绝对化用语 | `国家级\|最佳\|最优\|顶级` |
| critical | 处方药传播 | `Rx\|处方药.*购买\|凭处方购买` |
| critical | 无资质医疗行为 | `医生.*看诊\|在线开药` |
| critical | 封建迷信 | `算命\|风水\|迷信` |
| high | 无效承诺 | `无效退款` |
| high | 处方药代言人 | `专家推荐.*药品\|医生推荐.*药品` |
| high | 夸大疗效 | `永不复.?发` |

## 语义实体类型

| 类型 | 含义 | 示例 |
|------|------|------|
| `disease` | 疾病/症状域 | 糖尿病、高血压、骨质疏松 |
| `drug_category` | 药品类别 | 降糖药、降压药、OTC、处方药 |
| `efficacy_claim` | 疗效声明 | 具体有效率、临床数据、排名 |
| `authority_claim` | 权威背书 | 专家推荐、医生推荐 |
| `testimonial` | 患者证言 | 患者治愈经历、疗效对比 |

## 平台合规要求

| 平台 | 允许内容 | 限制内容 |
|------|---------|---------|
| 抖音 | 健康科普、用药常识、疾病教育 | 处方药推荐、在线开药 |
| 小红书 | 经验分享、健康笔记、用药记录 | OTC种草、处方药宣传 |

## AI 推理证据链

当内容涉及疗效声明时，系统自动构造如下证据供 AI 推理：

```
1. 【药品类别】降糖药合规表述：可说明血糖管理效果，不得声称'根治'
   审核要点：该药品类别的合规边界是什么？当前表述是否越界？

2. 【疗效声明分析】
   审核要点：声明的具体数据和措辞是否合规？需要哪些支撑材料？

3. 【平台合规要求】抖音 | 允许格式：健康科普、用药常识...
   审核要点：内容是否超出抖音允许的医疗内容范围？

→ AI综合以上证据，给出：(1)合规判断 (2)具体违规点 (3)修改建议
```

## 评分公式（仅参考，AI才是判断者）

```
guardrail_score = 100 - Σ(severity_weight × violations)
  severity_weight: critical=30, high=20, medium=10, low=5

等级：
  guardrail_score >= 90  → pass（参考）
  70 <= score < 90      → warning（参考）
  50 <= score < 70      → review_required（参考）
  score < 50            → fail（参考）
```

**注意**：评分仅供参考，真正的合规判断由 AI 基于语义证据链做出。
