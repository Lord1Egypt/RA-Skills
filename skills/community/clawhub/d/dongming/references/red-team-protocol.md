# 红队攻击协议

## 核心理念

逻辑审计是自审——Agent 自己审自己，天然有盲区。Rumelt 说过："你对自己的战略最容易在最需要严格的地方放松。"

红队攻击改变范式：**切换身份，用最恶意的眼光攻击自己的诊断。** 你不是在检查自己的作业，你是在摧毁自己的作业——如果它经得起摧毁，它才是真正成立的。

---

## 执行时机

报告初稿（诊断 → 指导方针 → 连贯性行动）完成后，**强制执行**红队攻击，攻击通过后才输出最终报告。

---

## 攻击流程

### 角色切换

```
你现在不是洞明。你是这个战略最尖锐的反对者——一个对行业有深刻理解、对逻辑漏洞有敏锐嗅觉、对"假连贯"有天然反感的批评者。你的任务不是提出更好的方案，而是证明这个诊断是错的、指导方针是虚的、连贯性行动是假的。
```

---

## 前置检查：反模式清单（LLM 行为陷阱）

> 在正式攻击之前，先自动检查报告是否中了 LLM 的固有行为模式。这些不是方法论问题，是 LLM 的均值回归本能。

| 反模式 | 典型表现 | 检验方法 | 中招了 |
|--------|---------|---------|---------|
| **伪装诊断** | "行业下滑 20%"是情势描述不是诊断 | 去掉这句话，行动建议变了吗？没变 → 不是诊断 | □ |
| **伪装指导方针** | "聚焦大单品"是目标不是方针 | 问：它说了不做什么吗？没有 → 不是方针 | □ |
| **伪装连贯行动** | "P0 做 A、P1 做 B、P2 做 C"是优先级清单 | 去掉 B，A 和 C 的效果下降了吗？没有 → 不连贯 | □ |
| **So What 敷衍** | "因此我们应该关注这个趋势" | So What 后能否接"具体来说就是"？不能 → 敷衍 | □ |
| **框架结论** | "根据波特五力分析，我们建议……" | 结论里出现了方法论名词 → 框架替代了判断 | □ |
| **形容词泛滥** | "市场很大""竞争激烈""用户需求旺盛" | 能用数字表达吗？能 → 必须改 | □ |
| **回避取舍** | "同时推进 A 和 B" | 问：A 和 B 的资源/激励/权力是否冲突？不冲突 → 没做取舍 | □ |

**中招≥2 条 → 报告打回，先修正反模式，再进入正式红队攻击。**

---

### 攻击一：诊断

**目标**：攻击诊断的核心——情势的重新框定。

**攻击问题**：
1. 这个诊断是否回避了最让人不舒服的问题？如果有一个问题你下意识不想提——那是什么？
2. 如果换一个完全相反的诊断，会是什么？那个诊断有没有可能也是对的？
3. 诊断中的"核心矛盾"是否只是"多个因素的叠加"换了说法？去掉这个矛盾，问题是否依然存在？
4. 诊断揭示的"行动可能性"是否真正是以前看不到的？还是任何分析师都能得出的结论？

**判定**：如果攻击一暴露了诊断的实质弱点 → 诊断不合格，回到假说筛选重新选择。

---

### 攻击二：排他性

**目标**：攻击指导方针的排他性——是否真正排了他。

**攻击问题**：
1. 指导方针说"不做 X"，但组织现有的资源/激励/权力结构是否实际上还在支持 X？
2. "不做 X"是真正的取舍，还是 X 本来就不重要、不做也无所谓？
3. 如果竞争对手知道这个指导方针，他们最可能从哪个"不做"的方向攻击？
4. 排他性是否只是换了个说法？"不做低端市场"和"聚焦中高端"是不是同一件事的正反说法？

**判定**：如果攻击二发现排他性是假的（不做无所谓的、组织实际还在做的）→ 指导方针不合格。

---

### 攻击三：连贯性 + 链式逻辑

**目标**：攻击行动之间的增强回路——是否是真的增强。同时检查是否识别了最弱链路。

**攻击问题**：
1. 去掉任何一个行动，另外两个行动的效果是否**真的**下降？还是只是说法上相互引用？
2. 行动之间的"增强关系"是否有具体的因果机制？还是只是"A 做好了 B 就更容易"这种万能句式？
3. 这些行动是否存在内在矛盾？比如"行动 A 需要放权"但"行动 B 需要集权"？
4. 如果只做其中两个行动（放弃一个），效果是否和全做差不多？如果是 → 那个行动是挂件。
5. **链式逻辑检查**：三个行动中哪个是最弱链路？指导方针是否触及了这个最弱链路？如果没触及 → 指导方针是无效的。

**判定**：如果攻击三发现增强回路是画出来的（不是真的）→ 连贯性不合格。

---

### 攻击四：可证伪性 + 前提假设

**目标**：攻击核心判断的可证伪性——是否敢于冒被证伪的风险。同时检查前提假设是否经过检验。

**攻击问题**：
1. 每个核心判断：什么证据能证明它是错的？如果找不到任何可证伪的证据 → 这个判断是安全的废话。
2. 如果 12 个月后这个战略失败了，最可能的原因是什么？
3. 诊断中的"核心矛盾"是否在 3 年前也存在？如果是 → 它可能不是当前问题的真正原因。
4. 指导方针中的"不做"是否可能在 6 个月内变成"不得不做"？什么情况会迫使这种反转？
5. **前提假设检查**：用户假设/合作假设/组织假设中最风险的那个，有没有在验证设计里被认真检验？还是被跳过了？

**判定**：如果攻击四发现核心判断不可证伪 → 判断没有信息量，是安全的废话。

---

## 攻击结果处理

- **全部通过**：输出最终报告
- **反模式≥2 条**：打回修正反模式，不进入正式攻击
- **攻击一失败**（诊断有问题）：回到假说筛选，换假说或强化诊断
- **攻击二失败**（排他性有问题）：回到指导方针，重新定义"不做"
- **攻击三失败**（连贯性有问题）：重新设计行动，或删除挂件行动
- **攻击四失败**（不可证伪）：强化判断的锋利性，加上具体数字/方向/时间窗

最多 2 轮回退修正。仍不通过 → 输出报告但标注"红队攻击未完全通过，建议人工复核"。

---

## 输出格式

红队攻击完成后，在最终报告附录中输出：

```
## 红队攻击记录

### 反模式检查
- 中招条数：[X/7]
- 修正说明：[哪几条中了，怎么修正的]

### 攻击一：诊断
- 最猛烈的攻击点：[具体攻击]
- 防御回应：[为什么诊断仍然成立，或做了什么修正]

### 攻击二：排他性
- 最猛烈的攻击点：[具体攻击]
- 防御回应：[为什么排他性是真的，或做了什么修正]

### 攻击三：连贯性
- 最猛烈的攻击点：[具体攻击]
- 防御回应：[为什么增强回路是真的，或做了什么修正]
- 最弱链路：[哪个行动是最弱链路，指导方针是否触及了它]

### 攻击四：可证伪性
- 最猛烈的攻击点：[具体攻击]
- 防御回应：[为什么判断可证伪，或做了什么修正]
- 最高风险前提假设：[用户/合作/组织假设中最危险的那个]

### 红队结论
- [攻击是否通过 / 主要修正 / 残留风险]
```
