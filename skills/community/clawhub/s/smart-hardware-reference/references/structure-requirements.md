# 结构要求清单

每种开发任务的组件槽位定义，供UTOS内容轴清单法使用。标注 `[待补充]` 的字段需企业根据自身产品和合规要求定义。

---

## H1 需求与市场

### H1-01 产品需求文档(PRD)
- **必选组件**: 产品名称/定位、目标用户画像、核心痛点(3-5个)、核心功能列表(优先级排序)、非功能需求（性能/成本/续航/尺寸）、成功指标(KPI)、竞品对标
- **可选组件**: 用户故事(User Story)、MVP范围定义、版本路线图
- **组装顺序**: 概述→用户→功能→非功能→指标→竞品→路线图
- **合规约束**: 无特殊
- **格式**: Word/Markdown

### H1-02 竞品拆解报告
- **必选组件**: 竞品名称/型号/价格、拆解BOM概览、优劣势分析、差异化机会、图片/截图
- **可选组件**: 成本估算、技术实现推测
- **组装顺序**: 基本信息→外观拆解→内部结构→BOM分析→SWOT总结
- **合规约束**: 注意商业秘密边界
- **格式**: Word/Markdown+图

### H1-03 用户场景分析
- **必选组件**: 场景名称、用户角色、操作步骤、期望结果、当前痛点
- **可选组件**: 场景优先级评分
- **组装顺序**: 用户定义→场景枚举→步骤细化→痛点提取→排序
- **合规约束**: 无特殊
- **格式**: Markdown

### H1-04 技术可行性评估
- **必选组件**: 技术点描述、难度评级(1-5)、风险评估(高/中/低)、备选方案、推荐方案及理由
- **可选组件**: PoC验证计划
- **组装顺序**: 技术盘点→逐项评估→风险汇总→方案推荐
- **合规约束**: 评估需基于可获取的资源和技术能力
- **格式**: Markdown表格

### H1-05 定价策略分析
- **必选组件**: BOM成本、制造成本、目标毛利率、建议零售价(MSRP)、竞品价格对比
- **可选组件**: 渠道分成模型、不同市场区间的定价策略
- **组装顺序**: 成本核算→毛利设定→定价→竞品对比→渠道影响
- **合规约束**: [待补充] 定价审批流程
- **格式**: Excel + Word

### H1-06 规格定义书
- **必选组件**: 功能规格表（含验收标准）、性能参数表、接口规格表、环境规格（温度/湿度/防护）、法规符合性要求
- **可选组件**: 否定规格（明确不做/不支持什么）
- **组装顺序**: 功能→性能→接口→环境→法规→否定清单
- **合规约束**: 所有规格必须可测试/可验证
- **格式**: Word/Excel

### H1-07 项目立项书
- **必选组件**: 项目目标、范围(SoW)、资源需求(人力/设备/预算)、关键里程碑、ROI预估、主要风险
- **可选组件**: Go/No-Go决策矩阵
- **组装顺序**: 目标→范围→资源→时间线→ROI→风险→决策
- **合规约束**: [待补充] 立项审批流程
- **格式**: Word/PPT

## H2 架构与规格

### H2-01 系统架构图
- **必选组件**: 模块框图、模块间信号流、数据流方向、电源分配路径、关键接口标注
- **可选组件**: 多视图（逻辑视图/物理视图/部署视图）
- **组装顺序**: 模块划分→信号定义→数据流→电源树→接口标注
- **合规约束**: 架构需覆盖H1-06全部功能规格
- **格式**: Visio/Draw.io/Mermaid

### H2-02 BOM初版（关键物料）
- **必选组件**: 料号、器件名称/型号、数量、关键参数、供应商（首选/备选）、预估单价、备注
- **可选组件**: 交期信息、MOQ
- **组装顺序**: 分类（MCU/电源/射频/传感器/连接器/被动件）→逐行填写→总价汇总
- **合规约束**: 关键芯片需有至少一家备选供应商
- **格式**: Excel

### H2-03 功耗预算表
- **必选组件**: 模块名称、工作电流(mA)、睡眠电流(uA)、占空比、平均功耗(mW)、电池容量(mAh)、续航时间(h)
- **可选组件**: 不同使用模式下的续航对比
- **组装顺序**: 模块枚举→电流值→占空比→计算→续航验证
- **合规约束**: 需留20%以上余量应对实际偏差
- **格式**: Excel

### H2-04 尺寸堆叠图(Stack-up)
- **必选组件**: 各部件尺寸(LxWxH)、堆叠顺序、总尺寸、公差预留
- **可选组件**: 多方案对比（紧凑型 vs 舒适型 vs 大电池版）
- **组装顺序**: 部件清单→尺寸确认→堆叠排列→干涉检查→总尺寸
- **合规约束**: 需考虑组装公差和壳体壁厚
- **格式**: CAD截图/示意图

### H2-05 接口定义文档
- **必选组件**: 接口名称、物理形式(连接器类型)、电气特性(电平/速率/协议)、引脚定义
- **可选组件**: 对接设备/线缆规格
- **组装顺序**: 接口枚举→物理定义→电气定义→引脚分配
- **合规约束**: 需考虑EMC和ESD防护
- **格式**: Markdown/Word

### H2-06 云端架构方案
- **必选组件**: 协议选择(MQTT/HTTP/CoAP)、云平台(AWS/Aliyun/自建)、数据模型(Device Shadow/Twin)、OTA架构、安全方案
- **可选组件**: 边缘计算策略
- **组装顺序**: 协议→平台→数据模型→OTA→安全→扩展
- **合规约束**: 需满足数据安全和隐私法规
- **格式**: 架构图+Markdown

### H2-07 技术风险矩阵
- **必选组件**: 风险ID、风险描述、概率(高/中/低)、影响(高/中/低)、风险等级、缓解措施、责任人
- **可选组件**: 应急触发条件
- **组装顺序**: 风险识别→评估分级→缓解→责任分工
- **合规约束**: 高等级风险必须有明确的缓解方案
- **格式**: Excel

### H2-08 规格变更追踪表
- **必选组件**: 变更ID、变更日期、变更内容(原→新)、影响范围、提出人、审批状态
- **可选组件**: 变更原因分类
- **组装顺序**: 时间序→变更详情→影响评估→状态追踪
- **合规约束**: 所有变更必须经过正式评审
- **格式**: Excel

## H3 硬件设计

### H3-01 原理图设计
- **必选组件**: 电源电路、主控及最小系统、外设接口电路、传感器电路、射频电路(如有)、调试接口
- **可选组件**: ESD防护电路、指示灯/按键电路
- **组装顺序**: 电源→主控→外设→传感器→射频→辅助
- **合规约束**: 原理图需经同行评审后冻结
- **格式**: Altium Designer/Cadence PDF导出

### H3-02 BOM完整版
- **必选组件**: 完整料号、厂商Part Number、完整描述、封装、数量、供应商、单价、总价、替代料(Alt)
- **可选组件**: LTA(Long Term Availability)标记、环保标志(RoHS/REACH)
- **组装顺序**: 按原理图位号排序→补充详细信息→价格→替代料→汇总
- **合规约束**: 所有元器件需确认供货周期和停产风险
- **格式**: Excel (CSV兼容)

### H3-03 PCB Layout设计
- **必选组件**: 层数规划、元件布局图、关键信号布线规则、阻抗控制表、孔径表、板框尺寸
- **可选组件**: DFM检查报告
- **组装顺序**: 层叠→布局→布线规则→阻抗→钻孔→板框
- **合规约束**: 需满足PCB厂工艺能力
- **格式**: Gerber文件/PDF截图

### H3-04 电源设计方案
- **必选组件**: 电源输入规格、各轨电压/电流需求、LDO/DC-DC选型、纹波噪声指标、启动时序、掉电保护
- **可选组件**: 电源完整性仿真结果
- **组装顺序**: 需求→拓扑选型→器件选型→时序→保护→验证
- **合规约束**: 电源设计是硬件稳定性的基础，不可省略验证
- **格式**: 原理图片段+Word说明

### H3-05 射频电路调试方案
- **必选组件**: 天线选型/设计、匹配电路、调谐流程、测试仪器清单、通过标准(回损/效率/辐射性能)
- **可选组件**: SAR评估(如为手持设备)
- **组装顺序**: 天线→匹配→调谐流程→测试计划→标准
- **合规约束**: 射频调试需在屏蔽室或暗室进行
- **格式**: Word+图表

### H3-06 EMC预评估报告
- **必选组件**: 干扰源识别(RE/CE/ESD/EFT/Surge)、风险等级、整改预案、整改优先级
- **可选组件**: 类似产品历史EMC问题参考
- **组装顺序**: 源识别→风险评估→预案→优先级排序
- **合规约束**: EMC问题在后期修改成本极高，前期预评至关重要
- **格式**: Word

### H3-07 热设计分析
- **必选组件**: 发热源列表(功率)、散热路径、材料导热系数、稳态温度预估、温升是否在规格内
- **可选组件**: CFD仿真结果
- **组装顺序**: 发热源→路径→材料→计算→判定
- **合规约束**: 关键器件温度需低于 datasheet 最大额定值的80%
- **格式**: Word+热分布图

### H3-08 可靠性设计评审
- **必选组件**: 元件降额清单(应力vs额定)、关键器件MTBF估算、失效模式分析(简化FMEA)
- **可选组件**: 加速寿命测试方案
- **组装顺序**: 降额→MTBF→FMEA→综合评定
- **合规约束**: 量产前必须完成此项评审
- **格式**: Excel+Word

### H3-09 设计评审(DR)报告
- **必选组件**: 评审时间/参会人员、评审范围、问题清单(ID/描述/严重度/负责人/状态)、决策记录
- **可选组件**: 行动项追踪
- **组装顺序**: 准备→评审→问题记录→决策→行动
- **合规约束**: DR问题必须在进入下一阶段前关闭
- **格式**: Word

### H3-10 生产文件包
- **必选组件**: Gerber文件(RS274X)、钻孔文件、坐标文件(Centroid/Pick&Place)、BOM(生产格式)
- **可选组件**: 钢网文件、装配图纸
- **组装顺序**: Gerber→钻→坐标→BOM→钢网→检查
- **合规约束**: 文件版本必须与最新冻结的原理图/PCB一致
- **格式**: ZIP压缩包

## H4 嵌入式软件

### H4-01 固件架构设计
- **必选组件**: 分层架构图（应用层/中间件层/HAL/驱动层）、任务调度方案、内存分区规划（RAM/Flash分配）、中断优先级表、模块间通信机制
- **可选组件**: 状态机总览、错误处理策略、日志系统设计
- **组装顺序**: 架构分层→调度方案→内存规划→中断表→通信机制→状态机→日志
- **合规约束**: 内存分配不得超过MCU可用资源的80%；中断嵌套深度需可控
- **格式**: 架构图(Mermaid/Visio) + Markdown

### H4-02 驱动程序开发
- **必选组件**: API函数定义（Init/Read/Write/Control/Deinit）、初始化流程图、错误码枚举及含义、寄存器映射表、典型用法示例
- **可选组件**: 中断服务程序(ISR)设计、DMA配置（如有）、功耗模式支持
- **组装顺序**: API定义→初始化流程→错误码→寄存器映射→示例代码→ISR→DMA
- **合规约束**: 驱动API需符合项目编码规范；寄存器操作必须加原子保护
- **格式**: C头文件(.h) + 技术文档(Markdown)

### H4-03 协议栈集成
- **必选组件**: 协议选型理由及对比（BLE/Wi-Fi/Zigbee/LoRa等）、配置参数列表、事件回调机制、数据包格式定义、连接状态机
- **可选组件**: 协议性能测试结果（吞吐量/延迟/功耗）、多设备组网方案
- **组装顺序**: 选型论证→参数配置→回调机制→数据格式→状态机→性能验证
- **合规约束**: 无线协议需符合目标市场的射频法规（FCC/CE/SRRC）
- **格式**: Markdown + 代码示例

### H4-04 应用逻辑实现
- **必选组件**: 业务状态机图（含全部状态和转换条件）、核心业务流程说明、数据处理算法、关键决策逻辑
- **可选组件**: 异常恢复流程、用户交互逻辑、数据持久化策略
- **组装顺序**: 状态机→业务流程→算法→决策逻辑→异常恢复→交互
- **合规约束**: 关键业务逻辑需有单元测试覆盖；状态机需覆盖所有边界情况
- **格式**: 状态机图(PlantUML/Mermaid) + C代码框架

### H4-05 OTA升级方案
- **必选组件**: 差分升级 vs 整包升级选择及理由、版本管理规则、回滚机制、包大小限制、签名校验方案、升级触发条件
- **可选组件**: AB分区方案、灰度发布策略、升级统计上报
- **组装顺序**: 选型→版本管理→回滚→大小限制→签名→触发→灰度
- **合规约束**: OTA过程中断电不能导致变砖；必须支持强制回滚
- **格式**: 流程图 + Markdown技术方案

### H4-06 低功耗设计
- **必选组件**: 各功耗模式定义及实测电流值、唤醒源表（外部中断/定时器/RTC/GPIO）、睡眠进入/退出策略、各模式切换条件
- **可选组件**: 低功耗调试经验记录、电池电量分级响应策略、RTC校准方案
- **组装顺序**: 模式定义→实测数据→唤醒源→睡眠策略→切换条件→电量响应
- **合规约束**: 待机功耗必须在产品规格书规定的范围内；唤醒时间需满足用户体验要求
- **格式**: 数据表格 + 代码示例

### H4-07 软硬件联调
- **必选组件**: 联调用例列表（ID/名称/前置条件/步骤）、预期结果、实际结果记录、Pass/Fail判定
- **可选组件**: 联调环境搭建指南、常见问题FAQ、调试工具使用说明
- **组装顺序**: 用例设计→预期→执行→记录→判定→FAQ整理
- **合规约束**: 所有Fail项必须有根因分析和修复记录；复测通过才能关闭
- **格式**: 测试用例表格(Excel/Markdown)

### H4-08 固件发布说明
- **必选组件**: 固件版本号（语义化版本）、变更Log（Feature/Bug Fix/Improvement分类）、已知问题清单、升级指引、固件MD5/SHA256校验值
- **可选组件**: 兼容性说明（硬件版本/旧版固件）、降级方法
- **组装顺序**: 版本号→变更Log→已知问题→升级指引→校验值→兼容性
- **合规约束**: 发布前必须通过冒烟测试；已知问题必须标注影响范围和规避方案
- **格式**: Release Note模板(Markdown)

## H5 结构与工业设计

### H5-01 ID工业设计方案
- **必选组件**: 产品渲染图（正/背/左/右/顶/底六视图）、CMF定义（颜色/材质/工艺/表面处理）、人机交互说明（按键布局/指示灯/显示屏位置）
- **可选组件**: 情景渲染图（使用场景）、竞品外观对比、材质样板编号
- **组装顺序**: 渲染图→CMF定义→交互说明→情景图→竞品对比
- **合规约束**: 外观设计不得侵犯已有专利；人机工程需符合目标用户群体
- **格式**: 高清渲染图(JPG/PNG) + Word说明

### H5-02 结构设计
- **必选组件**: 3D爆炸图（含全部零件）、装配关系说明、结构件BOM（含材料/数量/公差）、配合公差表（关键装配尺寸）
- **可选组件**: 运动机构仿真（如有滑动/旋转部件）、干涉检查报告
- **组装顺序**: 爆炸图→装配关系→结构件BOM→公差表→运动仿真
- **合规约束**: 装配顺序需考虑产线可操作性；公差累积需在允许范围内
- **格式**: 3D图(STEP/IGES) + Word

### H5-03 散热设计
- **必选组件**: 散热路径图（从热源到环境）、各散热材料导热系数、稳态温度计算或仿真结果、温升是否满足规格判定
- **可选组件**: CFD仿真详细报告、不同环境温度下的表现、自然对流vs强制对流对比
- **组装顺序**: 发热源识别→路径规划→材料选择→计算/仿真→判定→环境对比
- **合规约束**: 关键器件壳温需低于datasheet最大额定值的80%；用户可接触部位不超过45°C
- **格式**: 散热路径示意图 + Word分析报告

### H5-04 防水防尘设计
- **必选组件**: 目标IP等级（如IP67/IP68）、密封方式描述（硅胶圈/超声波焊接/胶水/一体成型）、对应测试标准及方法
- **可选组件**: 密封可靠性寿命预估、进水失效模式分析、成本vs等级权衡
- **组装顺序**: IP等级定义→密封方式选择→测试标准→可靠性评估→成本权衡
- **合规约束**: IP等级声明必须通过第三方实验室认证；密封件需考虑老化影响
- **格式**: 方案说明 + 测试标准引用(IEC 60529)

### H5-05 材料选型
- **必选组件**: 零件名称/用途、推荐材料牌号、供应商（首选/备选）、物性表（密度/强度/耐温/阻燃等级）、单件成本估算
- **可选组件**: 环保认证要求（RoHS/REACH/FDA食品接触）、可回收性评估
- **组装顺序**: 零件→材料牌号→供应商→物性→成本→环保认证
- **合规约束**: 材料必须符合目标市场法规（欧盟RoHS/美国FDA/中国GB）；食品接触材料需有相应认证
- **格式**: 材料选型表格(Excel)

### H5-06 DFM（面向制造的设计）评审
- **必选组件**: 出模角度检查（所有塑胶件）、分型面位置合理性、拔模斜度是否足够、组装可达性评估、注塑缺陷风险评估
- **可选组件**: 模具成本估算、注塑周期预估、二次加工需求（喷涂/镭雕/移印）
- **组装顺序**: 出模角→分型面→拔模斜度→可达性→注塑风险→模具成本
- **合规约束**: DFM评审应在开模前完成；评审意见必须逐条回应（接受/不接受+理由）
- **格式**: DFM评审报告(Word/PPT)

## H6 原型与验证

### H6-01 原型制作计划
- **必选组件**: EVT/DVT/PVT各阶段目标、各阶段样机数量、预算分配、时间表、验收标准
- **可选组件**: 外协厂商选择（如外包制作）、样机分发计划（内部/客户/认证）
- **组装顺序**: 阶段定义→数量→预算→时间表→验收→外协→分发
- **合规约束**: EVT阶段功能优先、DVT阶段可靠性和安规优先、PVT阶段量产一致性优先
- **格式**: 计划表(Project/Excel)

### H6-02 EVT（工程验证）报告
- **必选组件**: 测试项目清单、每个项目的测试方法、测试结果记录、发现的问题清单（ID/描述/级别）、结论（Go/No-Go）
- **可选组件**: 主要问题的初步改进方向、下一轮EVT调整建议
- **组装顺序**: 测试项目→方法→执行→结果→问题→结论→改进建议
- **合规约束**: EVT问题按严重度分级（致命/严重/一般/微小）；致命问题必须解决后才能进入DVT
- **格式**: 测试报告(Word)

### H6-03 DVT（设计验证）报告
- **必选组件**: 完整功能测试结果（覆盖率100%）、可靠性初步测试（跌落/振动/高低温循环）、安规预审（绝缘/耐压/防火）
- **可选组件**: 用户试用反馈汇总、EMC摸底测试结果
- **组装顺序**: 功能全覆盖→可靠性→安规→用户体验→EMC摸底
- **合规约束**: DVT阶段发现的架构级问题修复成本已很高，应尽量在设计阶段避免
- **格式**: 测试报告(Word)

### H6-04 PVT（生产验证）报告
- **必选组件**: 试产数量、直通率(FPY)/良率、Top3生产问题及改进措施、工艺参数确认、产能达标情况
- **可选组件**: 产线平衡率分析、操作员培训效果评估
- **组装顺序**: 试产信息→良率→问题→工艺→产能→线平衡
- **合规约束**: PVT良率需达到量产最低要求（通常>85%）；工艺参数需冻结并纳入SOP
- **格式**: 生产报告(Word/Excel)

### H6-05 可靠性测试
- **必选组件**: 测试项目及对应标准（如跌落IEC 60068-2-32）、测试条件（高度/次数/方向）、每项测试结果、判读依据（Pass/Fail标准）
- **可选组件**: 加速模型及等效寿命推算、与竞品的可靠性对比
- **组装顺序**: 项目→标准→条件→执行→判读→加速模型→竞品对比
- **合规约束**: 可靠性测试必须在 accredited 实验室进行（如SGS/TÜV/CTI）；测试样品需从试产批次随机抽取
- **格式**: 测试报告(Word)

### H6-06 认证跟进
- **必选组件**: 目标认证类型（如CCC/CE/FCC/RoHS）、申请资料清单、当前进度、已发现问题及解决方案、预计获证时间
- **可选组件**: 不同国家的认证差异对照表、认证费用预算
- **组装顺序**: 认证类型→资料准备→提交→进度跟踪→问题解决→预算
- **合规约束**: 计划销售的国家/地区必须列出对应的强制性认证要求；未获证产品不得上市销售
- **格式**: 进度跟踪表(Excel)

### H6-07 Bug List（问题追踪）
- **必选组件**: 问题唯一ID、问题描述（复现步骤）、严重程度（Critical/Major/Minor/Trivial）、负责人、当前状态（Open/In Progress/Fixed/Verified/Closed）、关闭验证记录
- **可选组件**: 问题归类统计（硬件/软件/结构/工艺）、根因分类（设计/物料/制程/人为）
- **组装顺序**: ID登记→描述→定级→指派→跟踪→验证→统计分析
- **合规约束**: Critical问题24小时内必须有响应；所有问题在PVT结束前必须Closed或经批准Deferred
- **格式**: Excel追踪表/Jira导出

## H7 量产与供应链

### H7-01 工艺文件
- **必选组件**: 标准作业指导(SOP)、检验标准(SIP)、工装夹具清单、节拍时间(Takt Time)、关键质量控制点
- **可选组件**: 作业平衡图、人机工程评估、防错(Poka-Yoke)设计
- **组装顺序**: SOP→检验标准→工装→节拍→质控点→平衡图→防错
- **合规约束**: SOP须经产线工程师和品质工程师共同审批；作业员须经过培训并考核上岗
- **格式**: SOP模板(Word) + 检查表(Checklist)

### H7-02 试产报告
- **必选组件**: 试产数量/批次号、各工序良率、Top3问题及根因分析(5Why)、改进措施及效果验证
- **可选组件**: 与目标良率差距分析、人员熟练度学习曲线
- **组装顺序**: 基本信息→良率→Top3问题→根因→改进措施→效果验证
- **合规约束**: 试产问题必须建立闭环追踪机制；改进措施需量化验证
- **格式**: 生产报告(Word)

### H7-03 供应商管理
- **必选组件**: 供应商基本信息（名称/地址/联系方式）、评分卡维度（质量/交期/成本/服务/技术）、审核发现项、绩效追踪趋势
- **可选组件**: 供应商准入流程、二阶供应商管理、备份供应商开发计划
- **组装顺序**: 基本信息→评分卡→审核→绩效→准入→二阶→备份
- **合规约束**: 关键器件(A类)必须有至少两家合格供应商；年度审核不可跳过
- **格式**: 评分卡模板(Excel)

### H7-04 QCP（质量控制计划）
- **必选组件**: IQC来料控制要点、IPQC制程控制点、OQC出货控制点、FQC最终检验要点、各环节抽样方案(AQL水平)
- **可选组件**: SPC统计过程控制应用点、MSA测量系统分析计划
- **组装顺序**: IQC→IPQC→OQC→FQC→抽样方案→SPC→MSA
- **合规约束**: AQL水平需参考客户要求和行业标准（通常常规0.65/严重0.4）；特殊特性需100%全检
- **格式**: 控制计划文档(AIAG PPAP格式)

### H7-05 包装设计
- **必选组件**: 包装结构设计（内盒/外箱/缓冲材料）、产品说明书、保修卡、条码（UPC/EAN/内部追溯码）
- **可选组件**: 包装跌落测试报告、环保包装方案（可降解/减量）、运输标签规范
- **组装顺序**: 包装结构→说明书→保修卡→条码→跌落测试→环保→运输标签
- **合规约束**: 包装标识需符合目的地国家法规（如欧盟包装指令94/62/EC）；条码需可扫描
- **格式**: 设计稿(AI/PDF) + Word

### H7-06 成本分析
- **必选组件**: BOM成本明细（含单价×数量）、制造成本（人工+制造费用）、NRE摊销（模具/治具/认证）、总成本汇总
- **可选组件**: 量产后成本下降曲线(Yield Learning)、规模效应分析、替代料降本机会
- **组装顺序**: BOM→制造成本→NRE摊销→总量→学习曲线→规模效应→降本机会
- **合规约束**: 成本数据需定期更新（至少季度）；NRE摊销需明确分摊基数和周期
- **格式**: 成本分析表(Excel)

### H7-07 产能规划
- **必选组件**: 日产能目标、爬坡计划（W1-W8每周产能）、人力需求、设备需求、瓶颈工序及解决方案
- **可选组件**: 倒班安排、加班策略、外包预案（产能不足时）
- **组装顺序**: 日产能→爬坡→人力→设备→瓶颈→倒班→加班→外包
- **合规约束**: 产能规划需匹配销售预测并有20%以上弹性；瓶颈工序必须有改善计划
- **格式**: 产能计划表(Excel/Project)

### H7-08 移交清单
- **必选组件**: 文件目录（全部交付物清单）、责任人签收确认、遗留问题清单（含解决时限）
- **可选组件**: 移交培训记录、知识转移计划（研发→量产团队）
- **组装顺序**: 文件目录→签收→遗留问题→培训→知识转移
- **合规约束**: 遗留问题必须有时限和责任人；移交后研发仍需提供约定时间的支持期
- **格式**: 交接清单(Word) + 签字页

## H8 运维与服务

### H8-01 云平台对接
- **必选组件**: 设备接入注册流程、数据上报协议和格式、远程控制接口定义、告警规则引擎配置
- **可选组件**: 边缘网关部署方案、多租户隔离设计、数据归档策略
- **组装顺序**: 接入注册→数据上报→远程控制→告警→边缘→多租户→归档
- **合规约束**: 设备通信必须加密（TLS/DTLS）；用户数据隐私需符合GDPR/个人信息保护法
- **格式**: 技术方案文档(API文档风格)

### H8-02 数据看板
- **必选组件**: 在线率监控、故障率统计、功能使用率分布、趋势图表（日/周/月维度的KPI曲线）
- **可选组件**: 异常自动告警（邮件/短信/webhook）、自定义报表生成、数据导出功能
- **组装顺序**: 在线率→故障率→使用率→趋势→告警→自定义→导出
- **合规约束**: 数据刷新频率需满足运营需求（实时/小时/每日）；敏感数据需脱敏展示
- **格式**: Dashboard设计稿(Figma/Axure) + 数据字典

### H8-03 售后支持体系
- **必选组件**: 常见问题解答(FAQ)、故障排查手册（按症状索引）、RMA（返修）流程、客服培训材料
- **可选组件**: 在线客服机器人知识库、备件库存管理、维修层级定义（L1/L2/L3）
- **组装顺序**: FAQ→排查手册→RMA→培训→知识库→备件→维修层级
- **合规约束**: 故障排查手册需覆盖80%以上的常见故障场景；RMA流程需有时效承诺
- **格式**: 支持文档集(Wiki/Markdown/Confluence)

### H8-04 固件迭代规划
- **必选组件**: 版本路线图（未来3-6个版本规划）、新功能列表及优先级、Bug修复队列、发布时间计划
- **可选组件**: 用户需求投票机制、技术债清理计划、废弃功能通知
- **组装顺序**: 路线图→新功能→Bug修复→发布计划→用户投票→技术债→废弃通知
- **合规约束**: 固件更新需向前兼容至少两个版本；重大变更需提前通知用户
- **格式**: Roadmap(产品路线图格式)

### H8-05 反馈闭环机制
- **必选组件**: 反馈收集渠道（App/客服/社群/电商评论）、分类规则（Bug/需求/投诉/夸奖）、根因分析方法、修复→验证→发布流程
- **可选组件**: 反馈响应SLA承诺、用户满意度回访、反馈数据分析报告（月度/季度）
- **组装顺序**: 收集渠道→分类→根因分析→修复流程→SLA→回访→数据分析
- **合规约束**: Critical Bug需在确认后48小时内出修复版本；用户投诉需24小时内首次响应
- **格式**: 流程文档 + SLA定义表

### H8-06 召回预案
- **必选组件**: 召回触发条件（安全风险/合规违规/批量质量问题）、召回执行流程、成本预估框架、公关应对策略、法律合规要点
- **可选组件**: 召回演练记录、保险理赔对接、渠道通知模板
- **组装顺序**: 触发条件→执行流程→成本→公关→法律→演练→保险
- **合规约束**: 召回预案需提前制定并在产品上市前评审；涉及安全问题需在24小时内启动
- **格式**: 应急预案文档(Word)

### H8-07 EOL（生命周期终止）计划
- **必选组件**: 计划停产时间、剩余库存处理方案、售后服务终止时间表、替代/升级产品指引
- **可选组件**: 库存报废财务处理、知识产权处置、环保回收方案
- **组装顺序**: 停产时间→库存→售后终止→替代品→财务→知识产权→回收
- **合规约束**: EOL需提前通知渠道和客户（通常6-12个月）；售后配件供应通常需延续3-5年
- **格式**: EOL计划书(Word)

### H8-08 知识库建设
- **必选组件**: 文档索引（按产品/类型/阶段组织）、经验教训库（历史项目踩坑记录）、最佳实践汇编、更新日志
- **可选组件**: 新人Onboarding指南、跨项目经验复用机制、知识贡献激励机制
- **组装顺序**: 索引→教训库→最佳实践→更新日志→新人指南→复用→激励
- **合规约束**: 知识库需指定维护人和更新频率（至少月度更新）；项目结束后2周内必须归档
- **格式**: Wiki/Markdown/Confluence空间

## H9 项目管理

### H9-01 主计划
- **必选组件**: 工作分解结构(WBS)、甘特图、关键里程碑清单、关键路径识别、资源负载图（人/设备）
- **可选组件**: 风险储备（时间和预算Buffer）、基线变更记录
- **组装顺序**: WBS→甘特图→里程碑→关键路径→资源负载→基线管理
- **合规约束**: 里程碑需有明确的验收标准(DOR/DOD)；关键路径上的任务延迟直接影响整体交付
- **格式**: MS Project / 甘特图(Excel)

### H9-02 风险管理
- **必选组件**: 风险登记册（概率×影响矩阵）、每个风险的应对策略（规避/转移/减轻/接受）、监控和审查频率、风险责任人
- **可选组件**: 应急资金预留比例、定量风险分析（蒙特卡洛模拟）
- **组装顺序**: 风险识别→登记册→应对策略→监控→应急资金→定量分析
- **合规约束**: 高风险项（概率高×影响大）必须制定缓解措施并每周跟进；风险登记册需动态更新
- **格式**: 风险登记册(Excel)

### H9-03 跨部门协同
- **必选要素**: 例会机制（频率/参会人/议程）、跨部门决策权定义、RACI矩阵（谁负责/谁批准/谁咨询/谁知会）、escalation升级路径
- **可选组件**: 协同工具选用（企业微信/飞书/Jira）、跨部门KPI挂钩机制
- **组装顺序**: 例会→决策权→RACI→升级路径→工具→KPI
- **合规约束**: RACI矩阵中每个决策事项只能有一个'A'(Accountable)；escalation路径需明确时效
- **格式**: 协同规程文档(Word)

### H9-04 里程碑评审
- **必选组件**: 各阶段Go/No-Go决策点定义、每个决策点的进入条件(DOR)和退出条件(DOD)、评审签字记录
- **可选组件**: 评审checklist模板、阶段复盘会议纪要
- **组装顺序**: 决策点→DOR→DOD→评审执行→签字→checklist→复盘
- **合规约束**: No-Go决策必须有书面记录和后续行动计划；禁止跳过任何里程碑评审
- **格式**: 评审记录(Word) + 签字页

### H9-05 成本控制
- **必选组件**: 预算分解（按阶段/科目）、实际支出记录、偏差分析（预算vs实际）、预警阈值设定、纠正措施
- **可选组件**: 挣值管理(EVM)指标（CPI/SPI/EAC）、成本优化建议
- **组装顺序**: 预算→实际支出→偏差→预警→纠正→挣值→优化
- **合规约束**: 偏差超过±10%需触发正式评审；成本超支需走变更控制流程
- **格式**: 成本报告(Excel)

### H9-06 复盘总结
- **必选组件**: 成功因素总结、失败教训深析（根因而非表象）、可沉淀资产清单（文档/代码/模具/专利）、后续改进建议
- **可选组件**: 团队成员个人成长回顾、客户反馈汇总、竞争对手动向分析
- **组装顺序**: 成功→失败深析→资产沉淀→改进建议→个人成长→客户→竞品
- **合规约束**: 复盘必须在项目结束后2周内完成；失败教训需写入组织知识库防止重犯
- **格式**: 复盘报告(Word/PPT)
