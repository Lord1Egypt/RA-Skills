# 定时任务冲突规则矩阵

## 标准任务类型

| 标准类型 | LUI 自然语言别名 | 操作范围 | 资源范围 | 风险等级 |
| --- | --- | --- | --- | --- |
| `auto_listing` | 自动铺货、上架商品、商品上架、每日铺货 | 写 | 商品/铺货 | 中 |
| `auto_follow_order` | 自动跟单、跟单、催发货、物流跟进、退款风险 | 写 | 订单 | 中 |
| `inventory_sync` | 库存同步、同步库存、库存管理 | 写 | 商品/库存 | 中 |
| `price_adjust` | 改价、调价、降价、低利润商品降价 | 写 | 商品/价格 | 高 |
| `auto_delist` | 自动下架、批量下架、无动销商品下架 | 写 | 商品/上下架状态 | 高 |
| `supplier_switch` | 智能换供、换供 | 写 | 商品/供应商 | 中 |
| `product_selection` | 选品、找品、商机 | 读/前置步骤 | 商品 | 低 |
| `inspection` | 巡检、风险巡检、店铺巡检 | 读 | 店铺/商品/订单 | 低 |
| `customer_service` | 智能客服、客服巡检 | 写/读 | 消息/订单 | 中 |
| `daily_report` | 经营日报、日报、任务汇总报告 | 读 | 报告 | 低 |

## 边界检查优先级

边界检查必须早于重复和冲突检查。没有有效执行店铺的任务，不应进入去重或调度逻辑。

| 边界类型 | 决策 | 用户感知 |
| --- | --- | --- |
| 无绑定店铺 | `block` | 告诉用户当前没有可用店铺，提供绑定/开店、了解支持任务或取消。 |
| 只有一个绑定店铺且未指定店铺 | `proceed` | 静默默认选择该店铺。 |
| 多个店铺且未指定店铺 | `ask_confirmation` | 要求用户选择全部店铺、某个平台子集、具体店铺编号或取消。 |
| 店铺授权失效 | `block` 或 `partial_create` | 阻断失效店铺并给出重新授权路径；有效店铺可继续。 |
| 平台能力不支持 | `partial_create` | 列出支持和不支持的平台，询问是否只在支持范围内继续。 |
| 缺少 ISV 高级版权限 | `block` | 说明 ISV 高级版权限限制，提供开通或降级为低频/基础任务的替代方案。 |
| 通知渠道不可用 | `warn_then_proceed` | 任务可运行；说明不可用通知渠道和可用兜底方式。 |

## 重复检测优先级

从上到下判断，命中最高优先级的可行动结果后停止。

| 重复类型 | 识别方式 | 决策 | 是否提示 |
| --- | --- | --- | --- |
| 完全重复 | 同店铺范围、同任务类型、同时间/频次、同策略 | `reuse_or_update` | 不提示。 |
| LUI 语义重复 | 与任一当前任务标准化后业务目标相同 | `reuse_or_update`，前提是策略等价 | 不提示。 |
| 流程重复 | 一个任务的前置步骤已包含在另一个流程中 | `silent_merge` | 不提示。 |
| 策略部分重复 | 同店铺、同目标，但阈值或策略不同 | `ask_confirmation` | 必须提示，默认建议修改原任务。 |
| 高风险重复 | 改价、下架、发货、订单写入等任务存在重叠，即使时间不同 | `ask_confirmation` | 必须提示，默认建议合并并加保护规则。 |

## 执行冲突

| 冲突类型 | 识别方式 | 运行时处理 | 创建时是否提示 |
| --- | --- | --- | --- |
| 无冲突 | 不同店铺/不同资源 | 并行执行 | 否 |
| 同平台限流 | 同平台、API 密集型任务、同时间窗口 | 平台级排队 | 通常否 |
| 同店铺批量写 | 同店铺、多个商品写任务 | 按优先级店铺级排队 | 通常否 |
| 商品字段覆盖 | 同商品、相同或相近字段 | 商品/字段级串行，并重新读取最新状态 | 高风险时提示 |
| 同订单处理 | 跟单、催发、异常处理命中同一订单 | 订单级串行和去重 | 否 |
| 高频任务堆积 | 每 5/10 分钟，或执行周期短于预估耗时 | 合并未执行轮次或跳过过期轮次 | 是，轻确认 |
| 时间窗口集中 | 多个任务集中在同一时间点 | 自动错峰或排队 | 否 |
| 运行时授权失效 | 创建后授权过期 | 暂停受影响店铺子任务 | 运行时强提示 |

## 提示模板

### 策略部分重复

```markdown
检测到相似定时任务。

已有任务：{existing_summary}
本次任务：{proposed_summary}

两个任务目标相同，但策略参数不同。{impact_sentence}

你可以回复：
1. 修改原任务
2. 仍然新建一个任务
3. 取消本次创建

默认建议：修改原任务。
```

### 高风险重复

```markdown
该任务存在高风险重复，暂时不会直接创建。

已有任务：{existing_summary}
本次任务：{proposed_summary}

风险：同一商品或订单可能被重复写入，导致价格、上下架状态、发货或订单处理结果被重复影响。

你可以回复：
1. 合并为一个任务，并设置保护规则
2. 保留两个任务，但开启同一对象每日只允许执行一次的保护
3. 取消本次创建

默认建议：合并任务。
```

### 高频任务提醒

```markdown
当前频次可能导致任务堆积。

{task_summary}

风险：上一轮任务可能尚未完成，下一轮已开始排队，导致延迟、失败率上升或平台限流。

你可以回复：
1. 按推荐频次创建：{recommended_frequency}
2. 仍按当前频次创建，接受排队风险
3. 取消本次创建
```

### 授权边界

```markdown
暂时不能为「{shop_name}」创建「{task_name}」任务。

原因：该店铺授权已失效，系统无法执行该能力。

你可以选择：
1. 去重新授权
2. 暂停这个店铺的任务
3. 取消本次创建
```

### 平台能力部分创建

```markdown
我可以为部分平台创建这个任务。

当前支持：
{supported_list}

当前不支持：
{unsupported_list}

你可以回复：
1. 仅为支持的平台创建
2. 取消本次创建
```
