# 简历撰写助手 · 使用说明

> **Skill 名称**：简历撰写助手（resume-customizer）  
> **版本**：1.0.2  
> **ClawHub**：[clawhub.ai/skills/resume-customizer](https://clawhub.ai/skills/resume-customizer)

---

## 一、功能介绍

简历撰写助手是一个面向求职场景的 AI Skill，帮助用户根据目标岗位 JD（Job Description）定制化撰写或优化简历。核心能力如下：

| 能力 | 说明 |
|------|------|
| **简历解析** | 从 PDF、Word、Markdown、HTML、纯文本中提取结构化信息（联系方式、工作经历、学历、技能、项目等） |
| **JD 解析** | 从文本粘贴、网页 URL、PDF、Word 中提取岗位名称、必需技能、优先技能、职责、关键词等 |
| **智能匹配** | 对比简历与 JD 的技能差异，输出匹配项、缺口项，生成优先级排序的改进建议 |
| **简历定制** | 根据 JD 自动重排章节顺序、技能排序、优化摘要、重组工作经历要点、植入关键词 |
| **ATS 优化** | 确保简历格式兼容企业招聘系统（Applicant Tracking System），避免因格式问题被自动过滤 |
| **多格式输出** | 支持导出为 PDF、Word (.docx)、Markdown、HTML、纯文本 |

**语言支持**：中文 & English，内置中英文双模板和行业关键词库。

## 二、适用场景

### 场景 1：已有简历，针对特定岗位定制

用户手上有一份通用简历，但投递不同岗位时需要针对性地调整内容。提供原始简历和目标 JD，助手自动完成匹配分析和内容定制。

### 场景 2：尚无简历，根据 JD 从零生成

用户没有现成简历，但知道自己想投什么岗位。提供目标 JD 和个人的背景信息（工作经历、学历、技能），助手生成一份面向该岗位的定制简历。

### 场景 3：简历投递无回音，排查 ATS 问题

简历投出后长期无面试邀请，怀疑被 ATS 系统过滤。提供简历和目标 JD，助手检测格式兼容性问题并进行 ATS 优化，同时补全 JD 所需的关键词。

### 场景 4：批量投递多岗位

同一份基础简历需要针对多个不同 JD 快速生成多个定制版本，避免重复手动调整。

## 三、如何触发

在对话中直接描述需求即可触发此 Skill，无需特殊命令。以下是三种典型触发方式：

### 方式 1：提供简历 + JD

> "帮我根据这个 JD 定制我的简历：[上传 resume.pdf 或粘贴 JD 内容]"

### 方式 2：提供背景 + JD（无简历）

> "我没有简历，请根据这个 JD 帮我生成一份。我的背景：3 年 Python 后端经验，熟悉 Django 和 PostgreSQL……"

### 方式 3：ATS 优化诊断

> "我投了很多简历没回音，怀疑 ATS 有问题，帮我检查并优化：[上传简历] + [JD 内容]"

**触发关键词**：简历定制、简历优化、JD 匹配、ATS 优化、简历撰写、resume tailoring 等求职简历相关表述均可触发。

## 四、输出内容说明

助手完成处理后，输出包含以下内容：

### 4.1 定制后的简历文件

用户可选择输出格式，默认提供 Word (.docx) 版本（ATS 兼容性最佳）：

| 格式 | 适用场景 |
|------|---------|
| `.docx` | **推荐**，ATS 友好，方便后续编辑修改 |
| `.pdf` | 正式投递使用，视觉效果佳 |
| `.md` | 便于版本管理和多版本对比 |
| `.html` | 在线简历、个人网站展示 |
| `.txt` | 部分企业系统在线表单直接粘贴 |

### 4.2 匹配分析报告

输出一份结构化的差异分析，帮助用户了解自身竞争力：

- **技能匹配项**：简历中已具备且 JD 要求的技能
- **技能缺口项**：JD 要求但简历中缺失的技能
- **额外加分项**：简历中有但 JD 未明确要求的技能
- **关键词密度**：JD 核心关键词在简历中的覆盖情况

### 4.3 改进建议

针对缺口的可操作建议：

- 建议补充的技能及优先级排序
- 可量化的成就表述建议
- 章节结构调整说明

### 4.4 简历模板（可选）

内置 4 种专业模板，用户可按需选择：

| 模板 | 风格 | 适合人群 |
|------|------|---------|
| `standard-en` | 英文标准 | 通用职位、金融/咨询等保守行业 |
| `standard-zh` | 中文标准 | 国内求职、国企/传统行业 |
| `modern-en` | 英文现代 | 互联网、创业公司、创意行业 |
| `professional-en` | 英文专业 | 外企、跨国公司、高级职位 |

## 五、可选参数

用户在触发时可指定以下参数，以获得更精准的结果：

| 参数 | 说明 | 可选值 | 默认值 |
|------|------|--------|--------|
| **输出格式** | 简历导出格式 | `docx`、`pdf`、`md`、`html`、`txt` | `docx` |
| **模板** | 使用的简历样式模板 | `standard-en`、`standard-zh`、`modern-en`、`professional-en` | 根据语言自动选择 |
| **ATS 优化** | 是否启用 ATS 兼容性优化 | 启用 / 不启用 | 启用 |
| **行业** | 目标行业，用于匹配专属关键词库 | `tech`、`data`、`product`、`marketing`、`finance`、`healthcare`、`education` | 从 JD 自动推断 |
| **关键词密度** | ATS 关键词植入的目标密度 | 数值（建议 1.5–2.5） | 2.0 |
| **语言** | 输出简历语言 | `zh`（中文）、`en`（英文） | 根据用户输入自动判断 |

**使用示例**：

> "帮我定制简历，JD 贴下面了，输出 PDF 格式，用 modern-en 模板，目标行业是 tech"
>
> "优化这份简历的 ATS，用中文模板，关键词密度设低一点"

## 注意事项

1. **隐私保护**：用户简历及个人信息严格保密，不对外泄漏
2. **内容真实**：只优化已有经历的表达方式，不编造或虚构任何经历和技能
3. **用户审核**：交付前请用户仔细核对所有信息，确保准确无误
4. **保留原稿**：始终提供原始简历版本供对比查看
