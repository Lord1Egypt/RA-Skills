# 路演直播流程SOP

## 概述

标准化路演直播全流程，确保每场路演高质量完成。

---

## 一、直播前准备（路演前5天）

### 1.1 项目确认

| 任务 | 负责人 | 完成时间 | 状态 |
|------|--------|----------|------|
| 确认本周路演项目 | 运营 | 周一 | □ |
| 与项目方确认时间 | 运营 | 周一 | □ |
| 收集项目简介/PPT | 项目经理 | 周二 | □ |
| 审核路演材料 | 导师 | 周三 | □ |

**项目方需提供：**
- [ ] 最新版BP（PDF+PPT）

- [ ] 产品/业务截图
- [ ] 核心数据指标

**运营方需准备：**
- [ ] 直播间背景图
- [ ] 主持人PPT模板

- [ ] 问答收集表单

### 1.3 直播技术测试

| 测试项目 | 标准 | 负责人 |
|----------|------|--------|
| 网络稳定性 | 上行≥10Mbps | 技术 |
| 画面清晰度 | 1080P以上 | 技术 |
| 声音清晰度 | 无回音/杂音 | 技术 |
| 屏幕共享 | 流畅无卡顿 | 技术 |
| 互动功能 | 弹幕/问答正常 | 技术 |

## 四、直播进行中

### 4.1 主持人话术模板

**开场白：**
```
大家好，欢迎来到【XX季度路演直播】第X场！

我是今天的主持人XXX，今天我们路演的项目是[项目名称]，
[一句话介绍项目]。

本场路演我们邀请到了[X]位投资人参与，
在正式开始之前，请大家把感兴趣的问题打在评论区。

现在，有请[项目方名称]的[路演人姓名]进行路演！
```

**过渡语：**
```
感谢[路演人]的精彩介绍！

接下来进入Q&A环节，请投资人们自由提问，
您可以在评论区留言，也可以举手申请连麦。
```

**控场语：**
```
这位投资人您好，请提问。
（等待回答）

感谢回答。还有其他投资人有问题吗？
（时间剩余5分钟时）最后还有2-3个问题机会。
```

**结束语：**
```
感谢[项目方]的精彩路演，感谢各位投资人的积极参与！

对[项目名称]感兴趣的投资人，可以扫描屏幕上的二维码
联系我们的运营同学进行深入对接。

下场路演预告：[简要介绍]
我们下期再见！
```

### 4.2 项目方注意事项

**路演准备：**
- [ ] 提前30分钟进入候场
- [ ] 测试网络/设备
- [ ] 关闭无关通知
- [ ] 准备好水/纸笔

**演讲要点：**
- [ ] 语速适中（每分钟180-200字）
- [ ] 重点内容重复强调
- [ ] 数据可视化呈现
- [ ] 时间控制（≤15分钟）

**回答问题：**
- [ ] 先复述问题确认理解
- [ ] 回答简洁有条理
- [ ] 不确定的问题不乱答
- [ ] 留出追问空间

### 4.3 运营监控要点

| 监控项 | 标准 | 处理方式 |
|--------|------|----------|
| 在线人数 | 目标≥50人 | 人少时引导转发 |
| 互动频率 | ≥10条/5分钟 | 冷场时主持人引导 |
| 提问质量 | 有效问题≥5个 | 运营协助提问 |
| 技术故障 | 0次 | 备用方案切换 |

### 4.4 技术应急方案

| 故障类型 | 应急措施 |
|----------|----------|
| 断网（项目方） | 切换为语音+PPT共享 |
| 断网（直播间） | 引导至备用平台 |
| 画面卡顿 | 降低画质/切换PPT模式 |
| 声音异常 | 重启音频/切换设备 |
