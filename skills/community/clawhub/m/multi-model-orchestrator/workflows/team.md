# $team — 并行执行工作流

## 触发条件
- 计划已审批，需要并行执行
- 用户说"并行执行"、"同时做"
- 任务可以拆分为独立子任务

## 执行流程

### Phase 1: 任务拆分
```
Agent: Architect (GPT-5.5)
输入: 实现计划
输出: 独立子任务列表
- 每个子任务有明确的输入/输出
- 子任务之间无依赖或依赖已解决
```

### Phase 2: 并行执行
```
使用 sessions_spawn 并行启动子任务:
- 子任务 1 [Coder/GPT-5.3-codex]: 实现模块 A
- 子任务 2 [Executor/Mimo]: 实现模块 B
- 子任务 3 [Coder/GPT-5.3-codex]: 实现模块 C

最多 3 个并行，避免 API 过载
```

### Phase 3: 结果汇总
```
Agent: Architect (GPT-5.5)
输入: 所有子任务结果
输出: 汇总报告
- 合并代码
- 解决冲突
- 验证一致性
```

### Phase 4: 质量验证
```
Agent: Reviewer (GPT-5.5)
输入: 汇总后的代码
输出: 审查报告
- 代码质量评分
- 问题列表
- 修复建议
```

## 使用方式

```bash
# 自动拆分并行
"用 $team 实现用户认证系统"

# 指定并行数
"用 $team 3 并行实现 REST API"

# 指定子任务
"用 $team:coder 实现模块 A，用 $team:executor 实现模块 B"
```

## 注意事项
- 并行子任务最多 3 个
- 每个子任务要有明确的输入输出
- 汇总阶段用主模型综合
- 子任务失败自动降级为串行
