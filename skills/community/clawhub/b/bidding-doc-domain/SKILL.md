# 招投标文档编制知识参考库

## 定位

本技能是 **Universal Task OS 的领域负载物仓库**，不包含任何执行框架。只提供招投标文档编制的"是什么"和"长什么样"——执行全部委托UTOS。

| 本技能提供 | UTOS消费方式 |
|-----------|-------------|
| 标书编制任务清单+依赖拓扑 | 内容轴·清单法的任务目录 + 执行轴·管线编排的依赖输入 |
| 标书各章节要求 | 内容轴·清单法的组件清单 |
| 优秀范本（待填充） | 内容轴·样本法的样本 |

## 三层结构

```
第一层：任务清单与依赖拓扑   →  references/bidding-catalog.md
第二层：标书编制要求清单      →  references/bidding-requirements.md
第三层：优秀范本库            →  references/exemplars.md
```

## 依赖声明

本技能**强依赖** Universal Task OS (universal-task-os)。没有UTOS，本技能只有参考查阅能力，无法执行任何标书编制任务。

**加载检查流程**（每次激活时执行）：

1. 检测 `universal-task-os` 技能是否已安装
2. **未安装** → 自动安装 `universal-task-os` 技能
3. **安装成功** → 同时加载UTOS，按本技能"使用规则"执行
4. **安装失败** → 降级为**只读参考模式**：
   - ✅ 允许：查阅任务清单、编制要求、范本索引
   - ❌ 拒绝：任何涉及标书产出、管线编排、废标检查的任务

**任务模式判定**：

| 任务类型 | 无UTOS | 有UTOS |
|---------|--------|--------|
| 查阅任务清单/编制要求/范本 | ✅ 只读参考 | ✅ 完整 |
| 按清单法产出标书文档 | ❌ 拒绝 | ✅ UTOS编排执行 |
| 依赖拓扑推导编制管线 | ❌ 拒绝 | ✅ UTOS执行轴 |
| 废标风控检查点插入 | ❌ 拒绝 | ✅ UTOS守护单元 |

## 使用规则

1. **依赖检查**：激活时按上述流程检测并安装UTOS
2. **首次加载**：读取 `references/bidding-catalog.md`，获取域分类、依赖拓扑
3. **按需深入**：确认目标任务类型后，读取 `references/bidding-requirements.md` 获取组件清单
4. **行业适配**：根据招标类型（货物采购/服务项目/工程施工）自动选择对应模板

## 招投标特有维度

| 维度 | 说明 | 影响 |
|------|------|------|
| 行业三分法 | 货物采购/服务项目/工程施工各有模板 | 技术标结构、报价格式、资质要求完全不同 |
| 废标红线 | 32项废标风险点必须逐项自查 | 漏项=废标=白干，是最高优先级约束 |
| 全局一致性 | 项目名称/编号/报价/工期全文同步 | 前后不一致是最常见的低级废标原因 |
| 合规硬约束 | 法律/安全/质量条款不可跳过 | 合规环节必须有人工确认 |
| 分片生成 | 50-500页标书需分章节生成 | 内容不丢、格式不乱、章节不串位 |

## 标书编制内容要求清单

### B1-01 招标文件解析
- **必选组件**: 项目名称、项目编号、预算金额、投标截止时间、投标地点、资质要求、工期/服务期/交货期、质保期
- **可选组件**: 联合体要求、保证金金额、开标时间、评标方法
- **组装顺序**: 文件通读→关键信息提取→信息结构化→缺失项标注
- **约束**: 所有信息必须从招标文件原文提取；不可推测或编造
- **格式**: 招标信息摘要（Markdown表格）

### B1-02 评分项提取
- **必选组件**: 评分大类（技术/商务/报价）、各类分值、评分细则
- **可选组件**: 加分项、扣分项、主观评分项
- **组装顺序**: 评分标准定位→大类拆分→细则提取→分值统计→策略建议
- **约束**: 分值合计必须等于总分；评分细则必须原文引用
- **格式**: 评分标准表（Markdown表格）

### B1-03 禁止条款识别
- **必选组件**: 废标条款列表、否决条款列表、限制性条款列表
- **可选组件**: 风险等级标注、应对策略
- **组装顺序**: 全文扫描→废标条款标记→否决条款标记→限制条款标记→风险评估
- **约束**: 必须逐条列出，不可遗漏；标注条款出处（页码/章节）
- **格式**: 禁止条款清单（Markdown表格）

### B2-01 投标函编制
- **必选组件**: 投标函正文（按招标格式）、投标报价（大小写）、工期/服务期承诺、有效期
- **可选组件**: 附加承诺、优惠条件
- **组装顺序**: 模板获取→信息填充→报价大写转换→格式校验→签字盖章位置
- **约束**: 格式必须与招标要求一致；报价大小写必须一致；必须有法定代表人签字/盖章位置
- **格式**: 投标函（Markdown→Word）

### B2-03 业绩表编制
- **必选组件**: 项目名称、合同金额、业主单位、完工时间、项目规模
- **可选组件**: 业主联系方式、验收情况、获奖情况
- **组装顺序**: 公司业绩库检索→同类项目筛选→信息核实→表格编制
- **约束**: 业绩必须真实可查；时间范围按招标要求（通常近3年）；必须是同类项目
- **格式**: 业绩表（Markdown表格）

### B3-01 项目理解与方案概述
- **必选组件**: 项目背景理解、需求分析、总体解决方案概述、方案优势
- **可选组件**: 项目重难点分析、创新点
- **组装顺序**: 招标需求梳理→项目理解撰写→需求分析→方案概述→优势提炼
- **约束**: 理解必须准确反映招标需求；方案必须对应评分项
- **格式**: 项目理解文档（Markdown）

### B3-02 技术方案编制
- **必选组件**: 技术架构/选型/设计、核心功能/性能指标、技术参数响应
- **可选组件**: 技术创新点、技术对比优势、技术图纸
- **组装顺序**: 需求映射→技术选型→架构设计→参数对标→优势论证
- **约束**: 技术参数必须逐项响应招标要求；偏离项必须标注
- **格式**: 技术方案（Markdown→Word）

### B3-03 实施方案编制
- **必选组件**: 实施计划（甘特图/里程碑）、人员配置、设备配置、进度安排
- **可选组件**: 风险预案、沟通计划、验收标准
- **组装顺序**: 工期倒推→阶段划分→里程碑设定→人员/设备分配→进度表编制
- **约束**: 工期必须满足招标要求；里程碑必须可验收；人员资质必须符合要求
- **格式**: 实施方案（Markdown→Word）

### B4-01 报价汇总表
- **必选组件**: 总报价数字、总报价大写金额、报价说明
- **可选组件**: 价格有效期、价格包含范围、价格不包含范围
- **组装顺序**: 分项汇总→总计计算→大写转换→大小写校验→说明撰写
- **约束**: 大小写必须一致；必须≤预算金额（除非策略性报价）；合计必须等于分项之和
- **格式**: 报价汇总表（Markdown表格）

### B4-02 分项报价明细表
- **必选组件**: 分项名称、规格型号、单位、数量、单价、合价
- **可选组件**: 品牌、产地、备注
- **组装顺序**: 需求清单映射→分项列举→单价确定→合价计算→合计校验
- **约束**: 合价=单价×数量；分项合计必须等于汇总表总价；规格必须响应招标要求
- **格式**: 分项报价表（Markdown表格）

### B4-03 偏离表编制
- **必选组件**: 招标要求条款、投标响应内容、偏离类型（正偏离/无偏离/负偏离）
- **可选组件**: 偏离说明、补充说明
- **组装顺序**: 招标要求提取→逐项对标→偏离标注→负偏离风险评估
- **约束**: 负偏离必须标红标注；关键参数不允许负偏离；偏离表必须逐项覆盖
- **格式**: 偏离表（Markdown表格）

### B5-01 废标红线自查
- **必选组件**: 32项废标红线逐项检查结果（通过/不通过/不适用）
- **可选组件**: 风险点标注、修正建议
- **组装顺序**: 红线清单获取→逐项检查→结果标注→风险点汇总→修正建议
- **约束**: 32项必须全部检查；不通过项必须修正后才能提交
- **格式**: 废标自查表（Markdown检查表）

### B5-02 报价一致性校验
- **必选组件**: 报价数字一致性（投标函/汇总表/分项表/偏离表）、大小写一致性
- **可选组件**: 不一致项修正记录
- **组装顺序**: 各表报价提取→交叉比对→不一致标注→修正→二次校验
- **约束**: 所有出现报价的位置必须完全一致；大小写必须对应
- **格式**: 一致性校验报告（Markdown）

### B6-01 全局信息同步
- **必选组件**: 项目名称全文一致性、项目编号全文一致性、报价全文一致性、工期全文一致性
- **可选组件**: 不一致项修正记录
- **组装顺序**: 关键信息提取→全文搜索→不一致标注→统一修正→二次验证
- **约束**: 关键信息在全文中必须完全一致；修正后必须重新校验
- **格式**: 信息同步校验报告（Markdown）

## 铁规则清单（AI必须严格执行）

> 优先级：最高。任何情况下不得绕过。

### A. 核心要素（缺一不可）

| 要素 | 缺失处理 |
|------|---------|
| 项目名称 | 追问「项目名称是什么？」 |
| 投标单位全称 | 追问「贵公司全称？」 |
| 报价总价 | 追问「报价总金额是多少？」 |
| 工期/服务期 | 追问「工期或服务期几天/月？」 |
| 质保期 | 追问「质保期多久？」 |

### B. 内容合规红线（严禁）

- ❌ 禁止虚假业绩、虚假人员资质、虚假设备信息
- ❌ 禁止出现违背招标文件要求的内容
- ❌ 禁止口语化、歧义表述
- ❌ 不得编造报价、工期、服务期、质保期
- ❌ 不得编造负偏离为无偏离

### C. 格式固定

- 目录顺序固定（投标函→法人授权→保证金→商务→技术→报价→偏离表→补充资料）
- 表格格式固定（业绩表/人员表/报价表）
- 报价大小写同步生成，不一致则提醒

### D. 偏离规则

- 默认：全部无偏离
- 正偏离：正常标注
- 负偏离：高亮提示废标风险

### E. 异常兜底

- 缺资料：主动追问，不编造
- 描述模糊：提示补充
- 报价冲突：高亮提醒
- 违规表述：拦截并提示

## 与UTOS的接口

当UTOS处理招投标领域任务时：

- **Step 0 三轴判定**：标书编制任务通常为复杂+高度结构化+合规驱动 → 执行轴+内容轴+守护轴
- **Step 1 领域校准**：R1(信息密度)=极高（招标文件+公司资料+行业规范）+R2(创造性)=低（标准化模板）+R3(交互性)=中（需用户确认关键数据）+R4(规范性)=极高（法律/合规约束）+R5(迭代性)=中（多轮修改）→ S极高C深A标准G极高
- **Step 2 内容轴**：清单法用本技能的bidding-requirements；样本法用本技能的exemplars
- **Step 3 执行轴**：管线编排基于本技能的依赖拓扑自动推导元操作序列
- **Step 4 交付**：G类守护单元自动插入废标风控检查点+全局一致性校验
