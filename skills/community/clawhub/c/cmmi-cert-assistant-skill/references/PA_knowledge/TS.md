# TS — 技术服务（CMMI V3.0 Domain-specific PA）

## PA 基本信息
- **缩写**：TS
- **英文**：Technology Solution
- **中文**：技术服务
- **所属**：Domain-specific PA（领域特定实践域）
- **适用领域**：硬件工程、软件工程、系统集成

---

## CMMI V3.0 要求要点

### 核心目的
设计、开发和维护技术服务解决方案，满足干系人需求。

### 关键实践（Specific Practices）
| 实践编号 | 实践内容 | 说明 |
|---------|---------|------|
| TS 1.1 | 分析需求 | 将干系人需求转化为技术方案 |
| TS 1.2 | 设计解决方案 | 架构设计、接口设计、详细设计 |
| TS 1.3 | 实现解决方案 | 编码、构建、配置 |
| TS 1.4 | 验证解决方案 | 确保解决方案符合需求 |
| TS 2.1 | 建立技术基线 | 控制技术方案版本 |
| TS 2.2 | 管理技术接口 | 协调技术接口 |
| TS 3.1 | 评估技术方案 | 评估技术方案的适宜性 |

### 通用实践（Generic Practices）
- GP 1.1 执行特定实践
- GP 2.1 建立组织政策
- GP 2.2 策划过程
- GP 2.3 提供资源
- GP 2.4 分配责任
- GP 2.5 培训人员
- GP 2.6 管理配置
- GP 2.7 识别并记录干系人需求和接口
- GP 2.8 监督和控制过程
- GP 2.9 客观评估符合性
- GP 2.10 高层审查

## 用户文档 → PA 匹配规则

### 高概率匹配（主要PA）
- 文档名含"技术方案"、"设计文档"、"架构设计" → **TS 主要匹配**
- 文档含代码、技术规格说明 → **TS 主要匹配**

### 中概率匹配（次要PA）
- 文档含测试用例 → 同时匹配 **VV**（验证与确认）
- 文档含配置项清单 → 同时匹配 **CM**（配置管理）

## 差距检查清单（用于差距分析）

### TS 1.x — 解决方案开发
- [ ] 是否有需求分析结果（需求追踪矩阵）？
- [ ] 是否有技术方案设计（架构、接口、模块）？
- [ ] 是否有代码实现记录？
- [ ] 是否有技术验证记录（单元测试、集成测试）？

### TS 2.x — 技术管理
- [ ] 是否有技术基线（设计基线、代码基线）？
- [ ] 是否有技术接口管理？
- [ ] 是否有技术变更控制？

### TS 3.x — 技术方案评估
- [ ] 是否有技术方案评审记录？
- [ ] 是否有技术方案适宜性评估？

## 评估访谈常见问题（模式D）

### 针对技术人员（开发/测试）的问题
1. "你的工作内容是什么？有设计文档吗？"
2. "你如何进行代码开发？有编码规范吗？"
3. "你如何进行单元测试？有测试记录吗？"
4. "技术变更是如何管理的？有变更记录吗？"

### 评分要点
- **TS 1.1-1.4**：是否有系统化的技术方案开发和验证
- **TS 2.1-2.2**：技术基线管理是否有版本控制
- **TS 3.1**：技术方案是否有评审和评估记录
