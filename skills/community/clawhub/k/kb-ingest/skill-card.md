## Description: <br>
Compile backend-uploaded source files into Research KB Markdown wiki pages, write them to Gitea, archive sources, and return structured JSON results. <br>

This skill is ready for commercial/non-commercial use. <br>

## Use Case: <br>
Developers and Research KB operators use this skill to convert backend-uploaded documents or source snapshots into traceable Chinese Markdown wiki pages in personal or team Gitea-backed knowledge bases. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: The skill can read backend-uploaded local files and persist source or configuration contents into Gitea. <br>
Mitigation: Install only in a controlled Research KB environment, review ingestion scope before execution, and avoid ingesting repositories or files that may contain secrets unless archival is disabled or secret scanning and redaction are added. <br>
Risk: The skill uses a Gitea admin token to write Markdown pages, archived sources, catalog data, and index files. <br>
Mitigation: Provide a least-privilege Gitea token limited to the intended repositories and keep credentials in environment variables rather than task JSON. <br>
Risk: Untrusted task JSON or uploadedPath values could cause unintended ingestion of files outside the expected upload set. <br>
Mitigation: Only pass task JSON generated by the trusted backend and enforce the documented protocol, single kbTarget, non-empty files list, and uploadedPath validation before execution. <br>
