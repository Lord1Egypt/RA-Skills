# 云助手测试用例专项风险图谱

## 来源基线

本清单来自云助手仓库历史和常见业务风险沉淀，用于评审测试用例是否覆盖真实高频问题。

代表性历史线索：

- 首页门店视角重复请求：`/workbench/yzs/user/subscribe/data`、`/notify/unread/count`、`/notify/popup/list`。
- 组织切换权限接口：`getViewPagesForYzs`按集团、门店、配送中心视角返回权限。
- 消息重复请求和弹窗展示。
- 收银明细表退款成功后返回自动刷新。
- 报表限流时`Network Error`提示。
- 关注商品支持商品属性，要求云助手与商品报表口径一致。
- AI入口、微海巡检、扫码、openClaw解绑、权限申请和多端能力差异。

## 专项检查表

| 风险项 | 评审问题 | 常见遗漏 |
|---|---|---|
| 组织视角 | 是否覆盖集团、门店、配送中心视角 | 只测一个默认门店 |
| 权限授权 | 是否覆盖无权限、授权过期、白名单未开、空权限 | 只测有权限用户 |
| 入口展示 | 首页、报表、应用、我的入口是否按权限和授权展示 | 入口误展示或不展示未测 |
| 状态闭环 | 退款、清台、上下架、消息已读是否覆盖成功、失败、重复操作 | 只测成功点击 |
| 数据刷新 | 操作后列表、详情、首页、报表是否刷新一致 | 返回上一页后仍是旧数据 |
| 接口异常 | 空数据、空字段、超时、限流、Network Error是否覆盖 | 只测200响应 |
| 重复请求 | 异步参数返回、切组织、切Tab、返回页面是否重复请求 | 未检查接口调用次数 |
| 报表口径 | 时间、门店、商品属性、订单状态、商品/套餐口径是否一致 | 云助手与报表口径不一致 |
| 多端兼容 | 微信小程序、H5、RN/App、扫码、相机、通知权限是否覆盖 | 只在H5验证 |
| 安全合规 | 退款、订单、会员、权限、消息是否有确认、审计、幂等 | 高风险动作无二次确认 |
| 埋点监控 | 关键入口、异常、失败是否有可观测路径 | 无法定位线上问题 |

## 云助手用例评审加分项

- 用例明确写出`viewCode`、`groupId`、`orgId`、门店/集团切换路径。
- 用例明确校验接口调用次数和触发时机。
- 用例明确区分入口展示、接口权限、操作权限三层。
- 用例覆盖历史Bug回归路径，并写出对应验证点。
- 用例覆盖小程序隐私授权、相机权限、通知权限、RN/H5差异。

## 云助手用例评审红线

- 订单、退款、会员、报表金额相关用例没有数据一致性校验。
- 权限变更只测入口，不测接口返回和操作拦截。
- 状态流转只测正向，不测重复提交、版本冲突、失败回滚。
- 报表类只测页面有数据，不测筛选口径和空数据。
- 首页或消息类不测重复请求和重复弹窗。
