## JSON 生成条件总结

| 场景 | 行为 | 生成新能力级 json？ |
|------|------|:---:|
| 命中规则级（完全） | 直接复用 | ✗ |
| 命中规则级（部分 + 能力级加载） | 拼装复用 | ✗ |
| 命中规则级（粗粒度 + 渐进补充） | 补充拼装 | ✗ |
| 命中能力级（完全/组合） | 直接复用 | ✗ |
| 命中 + 用户有调整 | 按调整执行 | 取决于用户选择 A/B/C |
| 不命中 + 用户确认执行 | 模型思考 → 执行 | ✅ 执行完成后生成 |
| 不命中 + 用户确认重新规划 | 模型思考 → 执行 | ✅ 执行完成后生成 |
| 不命中 + 用户选择仅本次使用 | 模型思考 → 执行 | ✗ |

## json 匹配规则

**如何判断是否"命中"**：

1. **规则级匹配**：用户任务的核心动词 + 对象类型 与规则级的 `tags` 或 `name` 匹配
   - 例：用户说"帮我做个PPT" → 匹配 `rule_ppt_v1`（tags 含 "ppt"）
2. **能力级匹配**：更细粒度，匹配能力级的 `name`、`description`、`tags`
   - 例：用户说"导出PPT" → 匹配 `export_ppt_v1`
3. **组合匹配**：多个能力级 json 拼装覆盖用户全部诉求
   - 例：用户说"做个PPT并导出为PDF" → 组合 `make_product_ppt_v1` + `export_ppt_v1`
4. **部分匹配**：规则级/能力级的前 N 步适用，后续步骤需加载其他 json
5. **完全不命中**：所有 json 均无法覆盖用户诉求的 50% 以上

### 脚本辅助匹配

AI 执行步骤 6 时，应使用脚本自动扫描匹配：

```bash
# ① 扫描规则级
python scripts/json_manager.py scan --keywords <任务关键词> --type rule --top 3

# ② 规则级不命中时，扫描能力级
python scripts/json_manager.py scan --keywords <任务关键词> --type capability --top 5

# ③ 均不命中时，检查归类统计是否需要凝练规则级
python scripts/json_manager.py categorize --threshold 5
```
