# Step 2: Extract Structure

Build a paper focus spec with:

- direction-native problems
- adjacent parent-field problems
- broader ML/AI scientific problems
- module-to-problem mapping
- graph node candidates
- graph relation candidates
- related papers
- idea-genesis trace

Do not jump directly from a direction-native problem to a remote ML/AI abstraction when a scientifically meaningful intermediate layer exists.
Prefer a continuous ladder such as `PFL -> FL -> AI/ML`.

Also extract the research-generative structure:

- research equation: `Important Setting + Broken Assumption + Borrowed Tool + New Constraint + Surrogate Mechanism`
- unavailable ideal mechanism `Y` and constructed surrogate `Z`
- valuable field / painful assumption / borrowed tool / unserved setting / blocking constraint / replacement mechanism
- challenge -> failure mode -> design principle -> module -> evidence map
- additive vs closed-loop story judgment
- hidden-assumption-to-new-idea candidates
