# 共享合规协议 (v5.1)

> **本文档是所有 Type Agent 的合规与品牌安全公共规范**。各 Agent 通过 `<skill-base>/references/_shared/compliance-protocol.md` 引用。
> **强制规则**：所有交付物在交付前必须通过本协议检查；不通过 = 🔴 Blocker，禁止交付。

---

## 🎯 设计目标

解决评估报告指出的"品牌安全护栏不足"问题：
- ❌ 缺乏广告法合规检查（极限词/功效宣称）
- ❌ 缺乏平台审核红线识别
- ❌ 缺乏文化敏感度检查
- ❌ 缺乏行业特殊资质校验

---

## 📐 三层合规检查框架

```
Layer 1：通用广告法合规（所有行业必检）
   ↓
Layer 2：行业专属合规（基于 industry 字段）
   ↓
Layer 3：平台审核红线（基于目标发布平台）
```

---

## 🚦 Layer 1：通用广告法合规

### 1.1 极限词检测（必检）

**《广告法》第九条禁用词**：

| 类别 | 禁用词清单 | 罚则 |
|------|----------|------|
| **最高级** | 最、最佳、第一、顶级、极致、极品、绝佳、首席、首选 | 20-100 万 |
| **绝对化** | 100%、永久、永远、彻底、万能、唯一、独家（无依据时） | 20-100 万 |
| **权威背书** | 国家级、世界级、国际级、全国、全球、宇宙级 | 20-100 万 |
| **虚假承诺** | 包治、100% 有效、永不脱妆、立竿见影、当天见效 | 撤下 + 罚款 |
| **医疗用语** | 治疗、治愈、处方、药用、医生推荐（非医疗器械/药品） | 撤下 + 罚款 |
| **贬低他人** | 比 XX 更好、超越 XX、XX 不行 | 民事赔偿 |

**检查方法**：对最终交付物做关键词扫描，命中即标红，必须替换或删除。

### 1.2 比较广告合规

| 场景 | 允许 | 禁止 |
|------|------|------|
| 与竞品对比 | 引用公开数据 + 注明来源 | 贬低、诋毁、未经授权使用竞品商标 |
| 自家产品对比 | 客观陈述差异 | 用旧款缺点衬托新款 |

### 1.3 数据引用合规

| 数据类型 | 必备条件 |
|---------|---------|
| 实验室数据 | 标注实验条件、样本量、第三方机构 |
| 用户调研 | 标注样本量、调研时间、调研方 |
| 销售数据 | 标注统计口径、时间范围 |
| 排名/奖项 | 标注颁发机构、年份 |

### 1.4 代言人/KOL 合规

| 场景 | 必备条件 |
|------|---------|
| 明星代言 | 代言人需实际使用过产品 |
| KOL 推荐 | 必须显著标明"广告"或"商务合作" |
| 真人出镜 | 需肖像授权 |
| 专家背书 | 需本人书面同意 + 资质真实 |

## 🏥 Layer 2：行业专属合规

### 2.1 引用方式

每个 Type Agent 在交付前，按 Brief 的 `industry` 字段加载对应行业文件：

```
industry = "beauty" → Read <skill-base>/references/industries/beauty-personal-care.md § 5
industry = "fnb"    → Read <skill-base>/references/industries/fnb.md § 5
industry = "3c"     → Read <skill-base>/references/industries/3c-digital.md § 5
...
```

### 2.2 高风险行业重点关注清单

| 行业 | 高风险合规点 |
|------|-----------|
| 美妆个护 | 功效宣称（特殊化妆品注册）、Before/After 过度 |
| 食品饮料 | 保健功能宣称（蓝帽子）、零糖零卡宣称依据 |
| 医美 | 医疗广告审查、效果承诺、价格透明 |
| 金融 | 收益承诺、风险提示、资质展示 |
| 教育 | 升学率承诺、教师资质、课程效果 |
| 房地产 | 价格承诺、面积、配套设施 |
| 医疗 | 处方药广告、医疗机构资质 |

### 2.3 功效宣称矩阵（重点）

**核心原则**：未取得相应资质的产品，**绝对不得**在营销中暗示相应功效。

| 产品类别 | 可宣称 | 不可宣称 |
|---------|-------|---------|
| 普通化妆品 | 清洁、保湿、香氛、修饰 | 美白、祛斑、抗衰、防晒（无证） |
| 普通食品 | 口味、原料、工艺 | 保健功能、治疗、预防 |
| 保健食品 | 已批准的 27 项功能 | 未批准功能、治疗 |
| 普通家用电器 | 功能描述 | 医疗效果、健康承诺 |
| 健身器材 | 锻炼辅助 | 减肥、治疗 |

## 📱 Layer 3：平台审核红线

### 3.1 平台通用红线

| 平台 | 重点红线 | 违规后果 |
|------|---------|---------|
| **抖音** | Before/After 过度、低俗擦边、虚假承诺、医疗暗示 | 限流 / 删视频 / 封号 |
| **小红书** | 软广未声明、夸大功效、医美推荐、虚假"自用" | 限流 / 删笔记 / 封号 |
| **微信** | 多级分销、诱导分享、过度营销 | 限制功能 / 封号 |
| **微博** | 数据造假、营销号炒作、负面压制 | 限流 / 封号 |
| **B站** | 利益相关未声明、数据无来源、低俗 | 删视频 / 封号 |
| **快手** | 虚假宣传、低俗、欺骗 | 限流 / 封号 |
| **视频号** | 同微信，更严格 | 限制功能 |

### 3.2 内容形式专属红线

| 形式 | 红线 |
|------|------|
| 短视频 | 未经授权使用他人素材、BGM 版权、虚假字幕 |
| 直播 | 假原产地、保质期虚假、价格欺诈、过度承诺 |
| KOL 内容 | 软广未声明、虚假测评、踩竞品 |
| 信息流广告 | 着陆页与广告承诺不一致 |
| 户外广告 | 内容未经审批、违反城市管理规定 |

## 🔍 检查清单（交付前必检）

每个 Type Agent 在交付前，必须执行以下检查并输出报告：

```markdown
## 🛡️ 合规自检报告

### Layer 1：通用广告法
- [ ] 极限词扫描：✅ 通过 / ❌ 命中 [词1, 词2]
- [ ] 比较广告合规：✅ / ⚠️ 含竞品对比，已注明来源
- [ ] 数据引用：✅ 所有数据均有来源 / ❌ 缺来源 [X 处]
- [ ] 代言人/KOL 合规：✅ / ⚠️ 已标注"广告"

### Layer 2：行业专属（industry = [行业名]）
- [ ] 功效宣称合规：✅ / ❌ 涉嫌越界 [具体描述]
- [ ] 资质要求：✅ / ⚠️ 需补充 [资质名]
- [ ] 行业特殊红线：✅ / ⚠️ [说明]

### Layer 3：平台审核
- [ ] 目标平台红线：✅ / ⚠️ [说明]
- [ ] 内容形式合规：✅ / ⚠️ [说明]

### 结论
- 🟢 全部通过：可交付
- 🟡 警示项：需用户决策（如"建议添加风险提示"）
- 🔴 Blocker：必须修改后重新检查
```

---

## 🚨 Blocker 红线（必须修改）

以下问题为零容忍，发现即拒绝交付：

| 类型 | 触发条件 | 处理 |
|------|---------|------|
| 极限词 | 命中《广告法》第九条禁用词 | 替换或删除 |
| 功效越界 | 普通产品宣称需资质的功效 | 必须删除 |
| 虚假数据 | 数据无来源且无法验证 | 必须删除或注明"估算" |
| 医疗暗示 | 非医疗产品使用医疗用语 | 必须替换 |
| 贬低竞品 | 未经授权贬低、诋毁 | 必须删除 |

---

## ⚠️ 警示项（建议处理）

以下问题不阻断交付，但必须告知用户：

| 类型 | 触发条件 | 告知方式 |
|------|---------|---------|
| 平台风险 | 内容存在被平台限流的潜在风险 | "建议修改 [X]，否则可能在 [平台] 被限流" |
| 文化敏感 | 涉及性别/地域/民族等敏感元素 | "建议补充多元化视角，避免文化争议" |
| 资质缺失 | 推荐内容涉及需资质的场景但 Brief 未提供 | "建议在投放前确认 [资质名]" |
| 时效性 | 数据/案例超过 12 个月 | "建议在投放前更新数据" |

## 📱 Layer 4：品牌安全（投放上下文安全）

### 4.1 内容环境黑名单（Blocklist）

任何投放预算不得分配到以下类别的内容相邻位置：

| 类别 | 示例 |
|------|------|
| 非法内容 | 毒品、武器、赌博、色情 |
| 极端政治 | 极端主义、仇恨言论 |
| 虚假信息 | 阴谋论、伪科学 |
| 暴力血腥 | 灾难现场、虐待动物 |
| 侵权内容 | 盗版、抄袭 |

### 4.2 品牌调性-内容场景匹配

| 品牌调性 | 适合场景 | 不适合场景 |
|---------|---------|---------|
| 高端奢华 | 艺术展、时装周、旅行 | 搞笑段子、土味视频 |
| 专业权威 | 行业报告、专家访谈 | 娱乐八卦、恶搞 |
| 年轻时尚 | 潮流穿搭、音乐节 | 严肃政务、老年健康 |
| 科技先锋 | 数码评测、极客文化 | 传统手工、迷信 |
| 亲民实惠 | 生活妙招、省钱攻略 | 奢侈品、高端社交 |

### 4.3 行业专属否定词

| 行业 | 必须添加的否定词 |
|------|---------------|
| 美妆 | 毁容、过敏、烂脸、激素 |
| 食品 | 食品安全、变质、中毒 |
| 母婴 | 事故、虐待、伤害 |
| 金融 | 暴雷、跑路、诈骗 |
| 汽车 | 事故、召回、自燃 |

## 📋 与 quality-protocol 的关系

| 协议 | 检查内容 | 触发时机 |
|------|---------|---------|
| quality-protocol.md § Adversarial | 方案质量（假设/逻辑/可执行性） | Tournament 后 |
| **compliance-protocol.md** | **合规与品牌安全** | **交付前最后一道关卡** |

**优先级**：compliance > quality。即使方案质量优秀，若合规不通过，禁止交付。
