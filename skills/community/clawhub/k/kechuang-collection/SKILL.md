# 科创线索采集 Skill

> **隐私与合规声明**
>
> 1. **仅采集公开信息**：本技能仅从各级政府平台、行业媒体等公开渠道采集信息，不访问任何需要登录或授权的非公开内容。
> 2. **不收集个人信息**：本技能不收集、不存储任何用户的个人敏感信息。所有采集内容均为公开公告。
> 3. **用户责任**：用户应遵守各目标网站的 robots.txt 和服务条款，建议监控频率≥60分钟以避免触发限流机制。
> 4. **仅供参考**：所有输出均为规划参考，高价值线索应在人工核实后再推进申报。
> 5. **Mock 模式**：本技能支持 `--mock` 参数，无需网络即可体验完整功能（详见下方 Mock 模式章节）。

> **Privacy & Compliance Notice**
>
> 1. **Public information only**: This skill collects data exclusively from publicly accessible government portals and industry media. No login or non-public content is accessed.
> 2. **No personal data collection**: No personal information is collected or stored.
> 3. **User responsibility**: Respect each site's robots.txt and terms of service.
> 4. **Planning aid only**: All output is for reference. Verify high-value leads manually before proceeding.
> 5. **Mock mode**: Use `--mock` flag for offline dry-run evaluation.

科创线索采集是一个自动化监控与采集工具，覆盖各级政府及官方平台的科创相关公告与新闻，智能筛选适配公司科创KPI考核的有效线索，替代人工筛取和排查的低效工作。

## 工作流程概览

```
用户输入 /kechuang-collection
  │
  ├─ scan <keywords>     → 实时全网扫描科创线索
  ├─ monitor             → 启动定时监控（后台循环扫描）
  ├─ report              → 生成近期科创线索汇总报告
  ├─ list-sources        → 列出当前监控的官方渠道
  ├─ add-source <url>    → 添加自定义监控源
  └─ (无参数)            → 交互式引导采集
```

| 类别 | 渠道 | 重点内容 |

| **人社厅/局** | 各地人社局 | 博士后科研工作站、人才引进计划、职称评审 |
| **省/市科技信息服务平台** | 各地科技大市场 | 技术需求对接、成果转化 |

### 二、全网行业动态（扩展渠道）

- 行业垂直媒体（36氪、钛媒体、雷锋网等）
- 重点企业科创合作新闻
- 技术攻关项目公示
- 科研课题立项公告
- 产业联盟、行业协会动态

## 核心能力

### 1. 按KPI维度智能筛选

根据公司科创KPI考核标准，将采集信息映射到以下KPI维度：

| KPI维度 | 匹配线索示例 | 权重大小 |
|---------|-------------|---------|
| 🏆 **科技奖/专利奖** | 省科技奖提名、国家专利奖申报 | 🔴 高 |
| 📋 **项目立项** | 重点研发计划、科技重大专项 | 🔴 高 |
| 🏭 **资质认定** | 高企认定、专精特新、企业技术中心 | 🔴 高 |
| 💰 **资金补贴** | 研发费用补贴、创新券、稳岗补贴 | 🟡 中 |
| 🎯 **人才计划** | 博士后基地、领军人才、创新团队 | 🟡 中 |
| 📐 **标准制定** | 国标/行标/团标征集、标准化试点 | 🟡 中 |
| 🔬 **平台建设** | 重点实验室、工程研究中心、新型研发机构 | 🟡 中 |
| 🤝 **产学研合作** | 校企合作技术攻关、成果转化需求 | 🟢 低 |
| 📊 **科创指标** | R&D统计、专利布局、技术合同登记 | 🟢 低 |

### 2. 线索分类标注

采集到的线索自动标记以下标签：

| 标签 | 含义 | 示例 |
|------|------|------|
| `[政策]` | 政策法规类 | 科创政策文件、管理办法、实施意见 |
| `[申报]` | 项目申报类 | 申报通知、征集指南、项目遴选 |
| `[公示]` | 结果公示类 | 立项公示、认定名单、奖补公示 |
| `[动态]` | 行业动态类 | 行业新闻、技术进展、企业合作 |
| `[会议]` | 会议论坛类 | 科创大会、培训会、路演 |

### 3. 优先级标注

| 优先级 | 判定标准 |
|--------|---------|
| ⭐⭐⭐ 紧急 | 距截止日期<15天，且高KPI权重 |
| ⭐⭐ 重要 | 距截止日期15-45天，或高KPI权重 |
| ⭐ 关注 | 其他有价值信息 |

## 使用方式


## 执行流程

当用户调用此Skill时，按以下逻辑执行：

> **Mock 模式**：如果命令带有 `--mock` 参数，所有模式将跳过 WebSearch/WebFetch，直接使用 `scripts/mock-data.js` 中的预置示例数据。

### 模式1: `scan` — 实时扫描

```
1. 解析用户输入的关键词和参数
2. [真实] 遍历所有监控渠道，使用WebSearch进行搜索
   [Mock] 从 scripts/mock-data.js 加载预置示例数据
3. [真实] 对每个搜索结果使用WebFetch获取详情
   [Mock] 示例数据已包含完整详情
4. 按KPI维度+分类标签自动标注
5. 计算优先级评分
6. 按[优先级][标签]排序输出结果表格
7. 结果保存至桌面 leads-output/ 目录
```

### 模式2: `monitor` — 后台定时监控

```
1. 使用CronCreate创建定时任务
2. 默认每120分钟执行一次全渠道扫描
3. 每次扫描后增量比对（去重）
4. 新发现的线索通过PushNotification推送通知
5. 运行日志保存至 leads-output/monitor.log
```

### 模式3: `report` — 报告生成

```
1. 读取 leads-output/ 目录下历史数据
2. 按KPI维度汇总统计
3. 生成结构化Markdown报告
   - 📊 KPI维度命中统计（饼图/Tables）
   - 📋 高价值线索TOP10
   - ⏰ 近期截止提醒
   - 📈 趋势分析（环比）
4. 输出至桌面和控制台
```

## 注意事项

- 采集的信息均为**公开信息**，不涉及爬取非公开/需登录内容
- 监控频率建议不低于60分钟/次，避免被目标网站限流
- 对疑似高价值线索，建议人工二次确认后再推进
- 支持自定义添加企业级监控源（如公司所在高新区官网）
- 合规使用：遵守目标网站的 robots.txt 和使用条款
