## 收集信息

### 基础信息

| 信息类型 | 收集问题 | 用途 |
|---------|---------|-----|
| 关系阶段 | 在一起多久？现在处于什么阶段？ | 确定差异化策略 |
| 能量状态 | 目前关系是充电还是耗电模式？ | 确定干预优先级 |
| 主要问题 | 最困扰的问题是什么？ | 确定核心议题 |
| 持续时间 | 这个问题持续多久了？ | 判断严重程度 |
| 尝试过什么 | 之前怎么处理的？效果如何？ | 避免重复无效方法 |

### 深度信息

| 信息类型 | 收集问题 | 用途 |
|---------|---------|-----|
| 触发点 | 这个问题通常在什么情况下出现？ | 设计预防策略 |
| 双方反应 | 你和伴侣通常如何反应？ | 设计干预方案 |
| 情绪强度 | 发生时你的情绪有多强烈？ | 判断是否需要危机处理 |
| 影响范围 | 这个问题影响哪些方面？ | 评估系统影响 |
| 改变意愿 | 双方都有改变意愿吗？ | 评估修复可能性 |

### 信息收集话术

**开场**：
- "我想了解一下你们关系的基本情况"
- "告诉我一些关于你们关系的背景"

**能量判断**：
- "下班后你更想回家还是不想回家？"
- "你觉得这段关系是在给你能量还是消耗你的能量？"

**问题聚焦**：
- "现在最困扰你的是什么？"
- "如果只能解决一个问题，你最想解决哪个？"

**深度了解**：
- "能给我一个具体例子吗？"
- "这个问题通常怎么开始？"
- "发生后你们通常怎么应对？"

## 问题归类

### 三层问题模型

| 层次 | 描述 | 举例 |
|-----|-----|-----|
| **表层问题** | 表面现象、具体事件 | "他经常加班不回家" |
| **中层问题** | 行为模式、互动方式 | "我们缺乏高质量相处时间" |
| **深层问题** | 需求、恐惧、依恋 | "我需要被重视但不知道怎么表达" |

### 问题分类框架

```
用户描述 → 识别表层 → 追问中/深层 → 确认核心问题
```

**示例**：

| 步骤 | 内容 |
|-----|-----|
| 用户描述 | "他最近总是不回我消息" |
| 追问1 | "不回消息时你的感受是什么？" |
| 回答 | "我觉得被忽视" |
| 追问2 | "被忽视对你意味着什么？" |
| 回答 | "感觉他不在乎我了" |
| 追问3 | "他在乎对你为什么这么重要？" |
| 回答 | "因为他是我最重要的人" |
| 核心问题 | 安全感/被爱的确认（深层） |

### 归类决策表

| 问题类型 | 表现 | 核心需求 | 推荐资源 |
|---------|-----|---------|---------|
| 沟通障碍 | 说不到一块就吵 | 被听见/被理解 | 话术模板-沟通障碍 |
| 亲密缺失 | 像室友无性 | 亲密/连接 | 话术模板-断供重启 |
| 信任受损 | 查看手机/不信任 | 安全/确认 | 场景-信任危机 |
| 边界冲突 | 婆媳/亲子问题 | 边界/尊重 | 场景-婆媳冲突 |
| 成长停滞 | 平淡无味 | 新鲜/意义 | 阶段策略-稳定期 |
| 依恋焦虑 | 担心被抛弃 | 安全/确认 | 依恋风格-焦虑型 |
| 依恋回避 | 逃避亲密 | 空间/自主 | 依恋风格-回避型 |

## 方案生成

### 个性化方案结构

```
个性化方案 = 阶段策略 + 问题适配 + 资源匹配 + 路径规划
```

### 方案生成步骤

**步骤1：确定关系阶段**
- 参考 relationship-stages.md
- 根据阶段选择差异化策略

**步骤2：匹配问题类型**
- 参考问题归类结果
- 从 references 中选择对应资源

**步骤3：考虑依恋风格**
- 如果提到依恋相关问题
- 参考 attachment-styles.md 加入适配策略

**步骤4：考虑原生家庭**
- 如果有深层议题
- 参考 family-mapping.md 溯源

**步骤5：设计具体路径**

### 方案模板

```
【个性化优化方案】

■ 关系阶段：[阶段名称]
■ 核心问题：[一句话描述]
■ 问题层次：[表层/中层/深层]

■ 即刻行动（本周）：
  1. [具体行动1]
  2. [具体行动2]

■ 本月重点：
  1. [重点目标1]
  2. [重点目标2]

■ 话术支持：
  - 场景：___ → 话术：___
  - 场景：___ → 话术：___

■ 资源推荐：
  - [references/xxx.md] - 用于：___
  - [scripts/xxx.md] - 用于：___

■ 预期进展：
  - 第1周：___
  - 第1月：___
  - 第3月：___

■ 评估标准：
  - 成功标志：___
  - 何时需要调整：___
```

### 方案示例

**用户情况**：
- 在一起3年，处于磨合期
- 主要问题：沟通障碍，经常为小事吵架
- 能量状态：轻度耗电
- 依恋风格：用户焦虑型

**生成方案**：

```
【个性化优化方案】

■ 关系阶段：磨合期
■ 核心问题：沟通障碍，冲突频率高
■ 问题层次：中层（互动模式问题）

■ 即刻行动（本周）：
  1. 约定一个"暂停词"，冲突时先冷静
  2. 每天晚安时说一句正向反馈

■ 本月重点：
  1. 学习"观察-感受-需求-请求"结构
  2. 每周一次深度对话练习

■ 话术支持：
  - 冲突要发生时：使用话术模板中的"暂停话术"
  - 表达需求时：使用"需求表达话术"

■ 资源推荐：
  - [scenario-solutions.md] 场景2：沟通障碍型
  - [conversation-templates.md] 冲突降噪话术
  - [attachment-styles.md] 焦虑型适配策略

■ 预期进展：
  - 第1周：建立暂停机制
  - 第1月：冲突频率降低30%
  - 第3月：形成新的沟通模式

■ 评估标准：
  - 成功标志：吵架后能主动修复
  - 何时需要调整：1月后无明显改善
```

## 方案校准

### 校准问题

**时机**：
- 生成方案后，先问用户确认
- 执行2周后进行评估
- 效果不佳时重新校准

**问题清单**：
1. 这个方案是否符合你的实际情况？
2. 有没有我遗漏的重要信息？
3. 某个行动对你来说是否可行？
4. 时间安排是否合理？

### 常见校准场景

| 情况 | 调整策略 |
|-----|---------|
| 用户太忙 | 减少行动数量，聚焦最关键的1-2个 |
| 用户抗拒某个方案 | 替换为替代方案 |
| 伴侣不配合 | 调整为自己单方面可做的 |
| 情况比描述严重 | 升级处理，加入危机干预 |
| 情况比描述轻微 | 简化方案，避免过度干预 |

## 执行与调整

### 执行原则

**小步快跑**：
- 每次1-3个行动
- 快速验证效果
- 及时调整

**持续反馈**：
- 每周回顾进展
- 记录有效/无效
- 分享给AI获取反馈

### 调整信号

**需要调整**：
- 方案执行超过2周无进展
- 出现新的问题
- 原有问题恶化
- 用户反馈方案不适合

**调整方法**：
1. 分析原因
2. 识别卡点
3. 替换/简化/强化
4. 重新执行

### 进度评估

| 时间 | 评估问题 | 判断 |
|-----|---------|-----|
| 第1周 | 行动是否执行？ | 能→继续；不能→调整行动 |
| 第2周 | 有何变化？ | 好转→强化；无变化→换策略；恶化→评估危机 |
| 第1月 | 核心问题改善了吗？ | 改善→继续；无改善→深入诊断 |
| 第3月 | 整体关系状态？ | 明显改善→进入维持；仍有问题→重新评估 |

### 升级路径

**从基础到深度**：
1. 基础诊断 + 场景方案
2. 依恋风格 + 原生映射
3. 心法修炼 + 高阶进化

**从单方到双方**：
1. 先处理自己可以做的
2. 引导伴侣参与
3. 共同参与方案
