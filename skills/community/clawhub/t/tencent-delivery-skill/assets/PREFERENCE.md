## 地址簿

> 本 skill 不查询用户微信/出行服务账号中的线上地址簿，所有地址匹配均仅基于此文件。
> 一条地址簿条目 = 别名 + 地址名称 + 经度 + 纬度 + 联系人 + 手机号。

| 别名 | 地址名称 | 经度 | 纬度 | 联系人 | 手机号 |
|------|----------|------|------|--------|--------|
| 公司 |  |  |  |  |  |
| 家 |  |  |  |  |  |

说明：
- 别名支持自由扩展（如"妈妈家"、"仓库"、"张三"等）
- 地址名称、经度、纬度均来自 `get_place_suggestion` 结果，经纬度必须保留完整精度（如 `116.273514`，不能四舍五入）
- 联系人和手机号为该地址的默认联系人
- 同一个别名既可以作为寄件地址也可以作为收件地址使用
- **经纬度齐全的条目匹配命中后可直接使用，无需再调用 `get_place_suggestion`**

## 匹配规则

**匹配流程（伪代码）**：
```
1. 从用户消息提取关键词（如"公司" / "望京" / "翁女士"）
2. 遍历地址簿每一行，任一命中即视为匹配：
   - 别名精确匹配（忽略大小写）
   - 联系人精确匹配
   - 地址名称包含匹配
   - 去前缀兜底（"回公司" → "公司"；"寄去望京" → "望京"）
3. 命中 0 条 → 走 get_place_suggestion 搜索
4. 命中 1 条 → 向用户确认
5. 命中多条 → 列出让用户选择
```

约束：
- 精确优先：`公司` 只匹配别名或地址名称为 `公司` 的条目，不匹配 `老公的公司`
- 地址名称匹配：`寄到永旺梦乐城` → 匹配地址名称含"永旺梦乐城"的条目
- 联系人匹配：`寄给张三` → 匹配联系人含"张三"的条目，同时带出地址与手机号
- 多匹配时列出选项让用户选，不自行决定；未匹配时向用户询问
- 条目信息不完整（如别名"望京"但无地址名称）→ 不能直接使用，需要补充

## 去重规则

判断地址簿中是否已存在某条记录，以 **联系人 + 地址名称** 联合匹配：
- 两者都相同 → 视为已存在（若经纬度为空则补充更新经纬度，其他字段不重复保存）
- 任一不同 → 视为新条目（可保存）

**注意**：
- 去重**仅**基于「联系人 + 地址名称」，**不参考别名**——即"同一地址+同一联系人但不同别名"也视为重复，不新增
- 命中后允许的字段级动作：
  - 经纬度为空 → 写入新经纬度（完整精度）
  - 手机号为空 → 写入新手机号
  - 其他列（别名、地址名称、联系人）**永远保持原值**，不覆盖
- 命中但所有字段已齐全 → 静默跳过，不修改文件
