# AI Startup MVP Factory

> 从一句话创业想法，到生产级可部署应用——全流程 AI 自动化。

---

## 业务场景

一位独立开发者或小型创业团队，需要快速将脑海中的产品想法变成**可演示、可运行的技术原型**，用于验证市场、参加路演、或向投资人展示。

**传统流程的问题：**
- 需求分析 → 产品设计 → 技术选型 → 写代码 → 搭 CI/CD → 部署
- 每个环节都要手动操作，学习成本高，耗时数天甚至数周
- 团队没有专职 DevOps，设计图还得找人帮忙

**现在的 AI 自动化流程：**
- 想法输入 → AI 生成 PRD → AI 生成完整代码 → AI Code Review → 容器化部署
- 全程无需手动写一行代码，1-2 小时完成从 0 到 1

---

## 痛点分析

| 痛点 | 描述 |
|------|------|
| 需求模糊 | 脑子里有想法，但不知道如何结构化表达 |
| 技术选型难 | 不知道用什么技术栈最合适 |
| 代码质量差 | 快速生成的代码缺乏规范，难以维护 |
| DevOps 门槛高 | Docker、CI/CD 对非运维人员来说太复杂 |
| 文档缺失 | 代码没有文档，后期交接困难 |

## 使用示例

### 完整流程演示

**输入：**
> 我想做一个"AI 会议助手"：
> - 自动录制 Zoom / 腾讯会议音频
> - AI 转写为文字并区分发言人
> - 自动生成会议纪要和待办事项
> - 支持导出 Markdown / PDF

**Step 1 — 生成 PRD：**
```
激活 prd-generator，输入以上需求 → 获得完整 PRD 文档
```

**Step 2 — 生成 MVP 代码：**
```
将 PRD 内容传给 ai-startup-mvp-launcher → 获得完整项目代码
技术栈推荐：FastAPI + React + PostgreSQL + Whisper（语音转写）
```

**Step 3 — 代码质量检查：**
```
将代码推送到 GitHub，传给 github-pr-knowledge-wiki-sync → 
获得 AI Review 报告 + 修复建议 + Wiki 文档
```

**Step 4 — 容器化部署：**
```
传给 docker-ci-release-pipeline → 
获得优化后的 Dockerfile + docker-compose.yml + 镜像
```

**输出：**
- ✅ 完整可运行的项目代码
- ✅ 结构化 PRD 文档
- ✅ 代码 Review 报告
- ✅ 可直接部署的 Docker 配置
