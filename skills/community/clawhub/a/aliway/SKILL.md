# AliWay — 阿里味战略文档写作术

## 核心理念

阿里味的本质不是词汇堆砌，是**思维方式**。真正的阿里 P10 文档有三个特征：
1. **从"道"出发** — 先讲为什么存在，再讲怎么做
2. **使命驱动** — 一切选择都指向一个比商业更大的意义
3. **组织穿透** — 文档即管理工具，读完要让人知道"我该做什么、不做什么、做不到会怎样"

---

## 写作框架：见自己、见天地、见众生

任何阿里味战略文档，都应该回答三个终极追问：

### 见自己（Who are we）
- **我们是谁** — 基因是什么？独特积累是什么？诚实面对强项和局限
- **为什么是我们** — 手上的牌面是什么？行业里有没有第二家能凑齐？
- **使命三问** — 有什么、要什么、为了要的东西打算放弃什么

写法要点：
- 不从竞品分析开始，从自我认知开始
- 诚实面对基因局限，不自吹自擂
- 强调"不可复制的积累"——时间换来的东西，钱买不到

### 见天地（What's happening）
- **时代级变革** — 不是"技术更新了"，是"生产关系变了"
- **终局判断** — 这个赛道的终局形态是什么？能活下来的有几家？
- **窗口期** — 为什么是现在？错过了代价是什么？

写法要点：
- 拉到文明级高度来看变革，不是"竞品做了什么我们跟"
- 用类比让人理解范式跃迁（蒸汽机不是让马跑更快，是让马退出历史舞台）
- 窗口期要有具体数字（X个月），不能模糊

### 见众生（Who do we serve）
- **用户的痛苦** — 他们今天的生存状态是什么？快乐吗？有成就感吗？
- **解放而非替代** — 我们要改变的是人的工作方式，不是消灭人
- **使命定义** — 一句话说清楚：我们要让谁、从什么状态、变成什么状态

写法要点：
- 从人文关怀出发，不从技术能力出发
- 清晰区分"解放"和"替代"，消除恐惧
- 使命必须比"做一个好产品"更大，要关乎一类人的命运

## 四层架构：道法术器

阿里味文档的内容组织遵循"道法术器"四层递进：

### 道（Why）— 第一性原理
- 从事物本质推导出不可违背的底层规律
- 不从竞品分析推导，从对世界的理解推导
- 每条原理要能用一句话表述，且能指导所有后续决策

### 法（What）— 战略选择与取舍
- **做什么** — 三个以内核心抓手，按优先级排列
- **不做什么** — 同样重要。列表呈现，每项说清底层逻辑
- 取舍的标准：是否指向使命，是否形成结构性壁垒

### 术（How）— 路线图与节奏
- 分阶段，每阶段有明确验收标准
- 验收标准必须是可量化、可验证的（不是"做了什么"，是"达到什么效果"）
- 资源分配要有具体比例

### 器（Measure）— 指标与考核
- 领先指标（方向对不对）、滞后指标（结果好不好）、健康指标（方法对不对）
- 指标不是用来看的，是用来考核的
- 必须带时间维度（6个月目标、12个月目标）

## 阿里味修辞术

### 核心心法：阿里黑话不是贴标签，是思维方式的自然流露

错误示范（贴标签）：
> 我们要拉通各个模块的底层逻辑，形成闭环的顶层设计。

正确示范（思维方式）：
> 这三个模块今天是散装的——各自独立演进，没有形成统一系统。散装的能力不叫生态，叫资源浪费。

### 六脉神剑的正确用法

不是口号式引用，是融入论证逻辑：

| 价值观 | 错误用法 | 正确用法 |
|--------|---------|---------|
| 客户第一 | "我们要客户第一" | "客户第一——这里的客户是 BP，他们的生产环境不能用来试错" |
| 因为信任所以简单 | "因为信任所以简单" | "我把这个判断给到你们，你们拿结果来验证。因为信任所以简单——但信任不是放羊" |
| 唯一不变的是变化 | "唯一不变的是变化" | "过去做的 feature 不是白做——是要把它的价值载体从按钮升级为 CLI。价值没变，载体变了。拥抱变化不是口号，是行为" |
| 今天最好的表现是明天最低的要求 | "今天最好的表现是明天最低的要求" | "这些指标不是用来看的，是用来考核的。做到了是及格，做不到复盘，反复做不到——组织会帮你做调整" |
| 此时此刻非我莫属 | "此时此刻非我莫属" | "不是因为我们最强——是因为我们的牌面在此刻最完整，而窗口不等人。If not now, when? If not us, who?" |

### 高频修辞模式

1. **追问式开篇** — 上来就问三个问题，逼读者自己想，不给舒适区
2. **类比降维** — 用操作系统、蒸汽机、翻译官等日常概念让抽象战略具象化
3. **对偶否定** — "不是X，是Y"结构制造张力（"不是技术升级，是生产关系变革"）
4. **量化锚定** — 所有判断带数字（"18个月"、"60%工程力量"、"6-12个月追平"）
5. **组织压力** — "组织会帮你做选择"、"变不了的人，组织会帮你做调整"
6. **格局对比** — "P7 的定位 vs P10 的定位"、"挂墙上的标语 vs 落到资源分配"

### 语气特征

- **审视感** — 从上往下看，不是平级讨论
- **笃定感** — 不用"可能"、"也许"、"建议考虑"，用"必须"、"不商量"、"这是战略选择"
- **责任感** — 所有判断都带"对结果负责"的兜底
- **使命感** — 让人觉得做的不是一个项目，是改变一代人命运的事业

## 文档结构模板

```
标题：[使命金句] — [具体方向的一句话定义]

开篇 callout：
- 这篇文章不讨论 X（降低预期）
- 这篇文章讨论的是 Y（升维定义）
- 读完如果只觉得"又要搞一个Z"，说明我没讲清楚

一、见自己 — 我们是谁，为什么是我们
二、见天地 — 时代在发生什么，终局是什么
三、见众生 — 我们要解放谁，使命是什么
四、道 — 第一性原理（2-3条不可违背的规律）
五、术 — 资源往哪里倾斜（3个核心抓手）
六、取舍 — 明确不做什么
七、节奏 — 分阶段路线图
八、衡量 — 指标与考核
九、组织 — 谁来 own，怎么考核
十、收尾 callout — 回归使命，形成闭环
```

## 绝对禁忌

1. **禁止空洞口号** — 每一句阿里味的话，后面必须跟具体论证或行动
2. **禁止纯技术视角** — 技术是手段不是目的，永远要回到"为谁创造什么价值"
3. **禁止面面俱到** — P10 的格局是取舍，不是列清单。砍比加更难
4. **禁止模糊表述** — 所有判断要有数字、有时间、有验收标准
5. **禁止自卖自夸** — 诚实面对局限，"牌好不等于赢"比"我们最强"更有说服力

## 质量检查清单

写完后用以下标准自检：

- [ ] 读完是否知道"我们为什么存在"？（使命感）
- [ ] 读完是否知道"时代在发生什么级别的变革"？（格局）
- [ ] 读完是否知道"我该做什么、不做什么"？（取舍清晰度）
- [ ] 读完是否知道"做不到会怎样"？（组织压力）
- [ ] 读完是否觉得"这件事值得我拿职业生命来赌"？（使命驱动）
- [ ] 阿里黑话是自然融入论证还是贴在表面的标签？（修辞质量）
- [ ] 有没有至少一处让人脊背发凉的判断？（战略穿透力）
