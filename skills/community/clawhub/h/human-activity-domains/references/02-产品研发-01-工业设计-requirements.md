## 工业设计生成要求

工业设计每种任务类型的"零件清单"——必选/可选组件、组装顺序、领域约束。按清单逐项产出。

---

### 1 需求分析

#### T1-1 用户研究

- **必选组件**: 研究目标、研究方法（访谈/问卷/观察）、样本选择、研究发现
- **可选组件**: 用户画像、用户旅程图、痛点地图
- **组装顺序**: 确定研究目标 → 选择研究方法 → 选择样本 → 执行研究 → 分析发现
- **约束**: 方法必须匹配目标；样本必须有代表性；发现必须基于数据
- **格式**: 用户研究报告（Markdown）

#### T1-2 市场调研

- **必选组件**: 调研范围、调研方法、市场数据、趋势分析
- **可选组件**: 行业报告、专家访谈、展会调研
- **组装顺序**: 确定调研范围 → 选择调研方法 → 收集市场数据 → 分析趋势 → 形成报告
- **约束**: 范围必须明确；数据必须可靠；分析必须客观
- **格式**: 市场调研报告（Markdown）

#### T1-3 竞品分析

- **必选组件**: 竞品清单、分析维度（功能/设计/价格/用户评价）、分析结果、设计启示
- **可选组件**: 竞品购买、竞品拆解、用户对比测试
- **组装顺序**: 确定竞品清单 → 设计分析维度 → 收集竞品信息 → 分析对比 → 提炼启示
- **约束**: 竞品必须有代表性；维度必须全面；分析必须客观
- **格式**: 竞品分析报告（Markdown）

#### T1-4 需求定义

- **必选组件**: 用户需求清单、设计目标、设计约束、优先级排序
- **可选组件**: 需求验证、利益相关者确认
- **组装顺序**: 汇总研究发现 → 提炼用户需求 → 定义设计目标 → 确定约束条件 → 排序优先级
- **约束**: 需求必须基于研究；目标必须可衡量；约束必须现实
- **格式**: 设计需求文档（Markdown）

---

### 2 概念设计

#### T2-1 创意构思

- **必选组件**: 创意方法（头脑风暴/SCAMPER/类比法）、创意数量、创意草图、创意说明
- **可选组件**: 创意工作坊、跨领域灵感、创意评估
- **组装顺序**: 回顾设计需求 → 选择创意方法 → 执行创意构思 → 草图记录 → 初步筛选
- **约束**: 数量必须充足（≥20个）；方法必须多样；草图必须清晰
- **格式**: 创意构思记录（Markdown）

#### T2-2 草图绘制

- **必选组件**: 设计草图（多角度/多方案）、草图说明、形态特征、设计亮点
- **可选组件**: 手绘板、数字草图工具、草图评审
- **组装顺序**: 选择创意方向 → 绘制草图 → 标注说明 → 评估可行性 → 筛选方案
- **约束**: 草图必须清晰可读；说明必须完整；方案必须多样
- **格式**: 设计草图集（图片+Markdown说明）

#### T2-3 方案评审

- **必选组件**: 评审标准、评审人员、评审记录、评审结论
- **可选组件**: 用户评审、专家评审、管理层评审
- **组装顺序**: 确定评审标准 → 邀请评审人员 → 展示设计方案 → 收集反馈 → 形成结论
- **约束**: 标准必须明确；人员必须相关；记录必须完整
- **格式**: 方案评审记录（Markdown）

#### T2-4 概念确定

- **必选组件**: 最终设计概念、概念说明、设计方向、设计原则
- **可选组件**: 概念验证、利益相关者确认
- **组装顺序**: 综合评审反馈 → 评估方案优劣 → 确定最终概念 → 撰写说明 → 获得确认
- **约束**: 概念必须符合需求；方向必须清晰；原则必须明确
- **格式**: 设计概念文档（Markdown）

---

### 3 详细设计

#### T3-1 3D建模

- **必选组件**: 3D模型文件、建模规范、尺寸标注、版本管理
- **可选组件**: 建模教程、材质库、插件工具
- **组装顺序**: 确定建模规范 → 建立基础模型 → 细化细节 → 标注尺寸 → 版本管理
- **约束**: 模型必须精确；规范必须符合；版本必须可追溯
- **格式**: 3D模型文件（STEP/IGES/STL）+ 建模说明（Markdown）

#### T3-2 效果渲染

- **必选组件**: 渲染效果图、材质选择、光影设置、环境场景
- **可选组件**: 渲染教程、材质库、渲染农场
- **组装顺序**: 设置材质 → 调整光影 → 选择环境 → 执行渲染 → 后期处理
- **约束**: 效果必须真实；材质必须准确；光影必须自然
- **格式**: 渲染效果图（PNG/JPG）+ 渲染说明（Markdown）

#### T3-3 人机工程

- **必选组件**: 人机工程分析、尺寸验证、舒适度评估、安全检查
- **可选组件**: 人机工程标准、用户模拟测试、专家评审
- **组装顺序**: 确定人机工程标准 → 分析设计尺寸 → 评估舒适度 → 检查安全性 → 形成报告
- **约束**: 标准必须符合；尺寸必须合理；安全必须保证
- **格式**: 人机工程分析报告（Markdown）

#### T3-4 设计定稿

- **必选组件**: 设计规范、设计文件、设计说明、版本记录
- **可选组件**: 设计评审、利益相关者确认
- **组装顺序**: 整理设计文件 → 撰写设计规范 → 编写设计说明 → 版本归档 → 获得确认
- **约束**: 文件必须完整；规范必须清晰；版本必须可追溯
- **格式**: 设计规范文档（Markdown）

---

### 4 设计验证

#### T4-1 原型制作

- **必选组件**: 原型类型（外观模型/功能样机/交互原型）、制作材料、制作工艺、原型说明
- **可选组件**: 3D打印、CNC加工、手板制作
- **组装顺序**: 确定原型类型 → 选择材料工艺 → 制作原型 → 检查质量 → 编写说明
- **约束**: 类型必须匹配需求；材料必须合适；质量必须达标
- **格式**: 原型制作记录（Markdown）

#### T4-2 用户测试

- **必选组件**: 测试目标、测试方案、测试用户、测试结果
- **可选组件**: 测试脚本、测试环境、测试工具
- **组装顺序**: 确定测试目标 → 设计测试方案 → 招募测试用户 → 执行测试 → 分析结果
- **约束**: 目标必须明确；方案必须可行；用户必须有代表性；结果必须客观
- **格式**: 用户测试报告（Markdown）

#### T4-3 设计修改

- **必选组件**: 修改依据、修改内容、修改方案、修改验证
- **可选组件**: 修改优先级、修改影响分析
- **组装顺序**: 分析测试结果 → 确定修改内容 → 设计修改方案 → 执行修改 → 验证效果
- **约束**: 依据必须充分；方案必须可行；效果必须可验证
- **格式**: 设计修改记录（Markdown）

#### T4-4 设计交付

- **必选组件**: 设计图纸、设计规范、品牌指南、交付清单
- **可选组件**: 设计培训、设计支持、设计维护
- **组装顺序**: 整理设计文件 → 编写设计规范 → 制作品牌指南 → 核对交付清单 → 正式交付
- **约束**: 文件必须完整；规范必须清晰；清单必须核对
- **格式**: 设计交付包（文件集合）+ 交付清单（Markdown）
