### 步骤 2：创建预订组件

**复制以下指令到 1D 后台**：

```
请创建行程预订组件 BookingPanel.tsx。

需求：
1. 文件位置：src/components/BookingPanel.tsx
2. 支持 3 个标签页：✈️ 航班、🚄 火车、🏨 酒店
3. 从 Mock 数据文件读取数据展示
4. "立即预订"按钮跳转到真实平台（飞猪/携程/12306）

组件 Props：
- destination: string（目的地）
- departureCity: string（出发城市）
- travelDate: string（出行日期）
- budget: number（预算）

功能：
1. 根据出发地和目的地筛选 Mock 数据
2. 按价格排序推荐（从低到高）
3. 每个类别显示 3-5 个推荐选项
4. 点击"立即预订"在新标签页打开预订链接

样式要求：
- 使用 Tailwind CSS
- 卡片式设计
- 移动端友好
- 与现有页面风格一致

请创建这个组件。
```

**预计耗时**：2-3 分钟（1D Agent 生成）

## 💡 常见问题

### Q1：Mock 数据不够真实怎么办？
**A**：Mock 数据只是 Demo 展示，评委关注的是产品思路，不是真实数据。后续可以接入真实 API。

### Q2：跳转链接参数不对怎么办？
**A**：飞猪/12306 的搜索页 URL 格式是公开的，按文档中的格式生成即可。即使参数不完全准确，用户也能在目标平台手动搜索。

### Q3：如果没有某个城市的酒店数据怎么办？
**A**：显示"暂无推荐，请前往平台搜索"，并提供跳转链接。

### Q4：需要支持多少个城市？
**A**：优先覆盖 20 个热门旅游城市（北京、上海、广州、深圳、武汉、成都、杭州、西安、南京、厦门、三亚、丽江、大理、拉萨、桂林、重庆、长沙、青岛、大连、哈尔滨）。
