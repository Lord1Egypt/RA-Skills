## Description: <br>
Guides architectural decisions for LangGraph applications. Use when deciding between LangGraph vs alternatives, choosing state management strategies, designing multi-agent systems, or selecting persistence and streaming approaches. <br>

This skill is ready for commercial/non-commercial use. <br>

## Use Case: <br>
Developers and engineers use this skill to make LangGraph architecture decisions, including when to use LangGraph, how to structure state, and how to choose persistence, streaming, multi-agent, and human-in-the-loop patterns. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Architectural advice could be applied incorrectly to production LangGraph systems, especially around persistence, retries, multi-agent flows, or credentials. <br>
Mitigation: Review any application code built from the guidance before deployment and validate production persistence, retry behavior, multi-agent control flow, and credential handling separately. <br>
