# phase-three-exceptions.md — 阶段三：异常处理

> 数字员工处理补位、换物料、取消等异常场景。后台写操作委托技术那套，催办交灵渠助手。

## 总览

```
自动发现（排期比对时）
├── 时段/端口不一致 → 交接③ 复制推广+改时段端口（phase-three-config §5）
└── 跨天 → 阶段一已处理，此处发现按不一致处理

主动指令（PM/王可说）
├── PM 说换物料 → 重新计算业务正确值并刷新默认值 → 更新追踪表 → 重新配物料（交接②）→ 重新发审批
├── 任何人说取消/延期/本期不上/黄了 → N→已取消（PM 自行去后台取消预占）
└── 补位 → 交接③ 复制推广调端口/时段
```

> ⚠️ **无独立"延期"状态**：本期不上 = 已取消。PM 后续要上需重新预占。
> ⚠️ **催办全交灵渠助手**：换物料/补位不再发催审批提醒。

**关键决策：**
- §1 补位 — 排期含「补位」→ 交接③ 复制推广调端口/时段
- §2 换物料 — PM 说 → 重新计算业务正确值并刷新默认值 → 更新追踪表 → 交接② 重配 → 发审批
- §3 取消 — 任何人说取消/延期/黄了 → N→已取消（PM 去后台取消预占）

**红线：**
- 跨天拆分已在阶段一处理，此处发现仍跨天 → 走不一致
- 换物料重配后由技术那套直接发审批（无暂存态）
- 配完必过收尾门禁

---

## 1. 补位

### 1.1 触发

帧位排期中某项目标注「（补位）」：
```
0点-12点 老番茄《乌云天》（补位）
```

### 1.2 处理

与 `phase-three-config.md §5`「不一致处理」流程相同：
1. 搜追踪表找原始推广（M 列拿 promotionID）
2. **交接③ 委托技术那套** `promotion-copy` + `promotion-update`（按排期端口/时段）+ 点预占
3. 同步追踪表：原始→「已替代」+ O 列备注；副本→新行 + I/J/K/L 继承原始 + N=已预占
4. 副本素材判断：按 phase-three-config §3.3 速查表检查继承过来的 I/J/K/L 是否已齐 → 齐则走 §2 配物料逻辑（交接②）；不齐保持「已预占」
5. 过收尾门禁 → DM 通知 PM
6. 不等 PM 回复直接处理

## 2. 换物料

### 2.1 触发

PM 在 DM 或群聊说「换物料」「换图」「换链接」「换角标」并发新素材。
群里说 → 群内回「私聊对接」→ 主动 DM。

### 2.2 处理

**搜表匹配：** 用项目名搜 A 列。

| 搜索结果 | 处理 |
|---------|------|
| 1 条 | 进下方分流 |
| 2+ 条（跨天兄弟） | 列出让 PM 确认改哪些，不默认全改 |

**确认话术（2+ 条）：**
```
搜到 {N} 条「{项目名}」：
• {推广名称A}（{日期/时段}）
• {推广名称B}（{日期/时段}）
这次换物料是全改还是只改某几条？
```

PM 确认后按 N 列分流：

**① N=「已预占」（物料还没配进后台）**

1. 更新追踪表物料列（I/J/K/L，含重算业务正确值）。若后台详情缺 `fullType/resourceTypes/resourceInfo`，先补问实际投放资源类型、标准资源 ID、全量/非全量和非全量人群。标准资源必须重新获取资源默认值；Web 端缺 http/https 链接时先问 PM，不得沿用上游/旧创意 URL。
2. 若更新后齐全 → 走 `phase-two §3.5` 配物料（交接②，直接发审批）→ N=「已配置」
3. 告诉 PM：
```
✅ 物料已更新并配置：{项目名}。审批提醒由灵渠助手通知你。
```
未齐 → 写表存着，保持「已预占」，灵渠助手催料。

**② N=「已配置」（已配并发过审批，可能在跑/待审批）**

1. 更新追踪表物料列（I/J/K/L，重算业务正确值）。若后台详情缺 `fullType/resourceTypes/resourceInfo`，先补问实际投放资源类型、标准资源 ID、全量/非全量和非全量人群。标准资源必须重新获取资源默认值；Web 端缺 http/https 链接时先问 PM，不得沿用上游/旧创意 URL。
2. 群聊发暂停通知：
```
⚠️ 「{项目名}」物料有更新，正在重新配置。@PM
```
3. **交接② 委托技术那套** `promotion-update-creative` 用新物料覆盖 + 带 businessRemark 重新发审批；交接时明确这是更新/换物料，要求技术那套重新跑默认值获取和保存前预检。数字员工只交接原始图片、资源/链接/角标/人群等业务字段，禁止把旧创意 JSON 改几项后直接交 `promotionAliasCreativeList` / `creative` 成品包。
4. 过收尾门禁（查 detail：物料=新值、已重新进审批）
5. 群聊 @PM 通知：
```
@PM
「{项目名}」物料已更新并重新发起审批，请留意灵渠助手的审批提醒。
┣ 配置方式：{H列值}
┗ 变更：更新物料
```

> 技术那套配物料即发审批，无"暂存等审批"中间态；换物料同理，更新即重新发审批。

## 3. 取消（含延期/本期不上）

### 3.1 触发

任何人（PM 或王可）说「XXX 取消」「XXX 彻底不做了」「XXX 黄了」「XXX 延期」「XXX 本期不上」等。

> 统一为取消：后台预占位需 PM 自行去灵渠后台取消。无独立延期态，后续要上重新预占。

### 3.2 处理

1. 搜追踪表匹配项目（跨天兄弟取核心名去 `_MMDD`；2+ 条列出让确认取消哪些，不默认全取消）
2. **取消预占（释放后台位）**：grp.bjoperation 无删除权限，后台预占位由 PM 自行去灵渠后台取消。数字员工只改追踪表。
3. N→「已取消」（终态）
4. O 列→「PM 说取消/本期不上 + 时间，请 PM 去后台取消预占」
5. 群聊/DM 确认：
```
✅ 已标记「已取消」：{项目名}，请你去灵渠后台取消预占。后面还要上的话，再跟我说一次预占就行。
```

---

## 4. 跨天

跨天拆分在阶段一（技术那套）已处理。阶段三比对若发现仍跨天 → 按 `phase-three-config.md §5` 不一致流程走（交接③ 复制 → 调时段 → 预占 → 配物料）。

---

## 5. 边界情况

| 情况 | 处理 |
|------|------|
| PM 换物料时群聊其他人插话 | 只认发物料的 PM |
| 取消的项目 PM 还在发物料 | 终态跳过，不收不配 |
| 任何人说取消但未 @数字员工 | 不处理（需 @数字员工 或说关键词） |
