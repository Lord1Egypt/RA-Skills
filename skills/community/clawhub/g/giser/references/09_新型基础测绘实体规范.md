# 第十四篇：新型基础测绘实体数据规范

> 来源：自然资源部国土测绘司 新型基础测绘与实景三维中国建设技术文件（2021年12月）
> **核心主题**——从传统"地理要素"到"地理实体"的范式转变，是整个测绘行业从二维符号化到三维语义化的底层变革。

## 14.1 范式转变：地理要素 → 地理实体

这是新型基础测绘最根本的概念革命：

| 维度 | 传统"地理要素"（Feature） | 新型"地理实体"（Geo-Entity） |
|------|--------------------------|------------------------------|
| **组织单元** | 图层（Layer）——按类型分层 | 独立地理对象——以实体为单元 |
| **表达形式** | 单一几何类型（点/线/面分开存） | 多种表达（二维/三维/多尺度共存） |
| **标识方式** | 无独立标识（靠分类码+图层名） | 全球唯一空间身份编码 |
| **属性管理** | 固定属性表（字段名+值） | 语义化属性（支持关系、时序、多源关联） |
| **关联能力** | 弱（仅空间位置关系） | 强（可关联物联网/IoT/互联网数据） |
| **应用模式** | 制图+分析 | 时空大数据平台的定位框架和承载基础 |

### 17个核心术语（文档1完整提取）

| 术语 | 英文 | 定义核心 |
|------|------|----------|
| 新型基础测绘 | new fundamental surveying and mapping | 以重新定义成果模式为核心和切入点，带动技术/生产/政策全面升级 |
| 实景三维 | 3D real scene | 对生产/生活/生态空间进行真实、立体、时序化表达的数字空间 |
| 时空大数据平台 | spatio-temporal big data platform | 以时空信息为基础，聚合分布式大数据资源的基础性开放式技术系统 |
| **地理实体** | geo-entity | 现实世界中占据连续空间位置、单独具有同一属性或完整功能的地理对象 |
| **基础地理实体** | fundamental geo-entity | 通过基础测绘采集和表达的地理实体，是其他实体和相关信息的定位框架与承载基础 |

### 地理实体组合类型

| 类型 | 英文 | 定义 |
|------|------|------|
| **组合地理实体** | combined geo-entity data | 权属一致、空间连续的地理实体数据集合（如长江干流由各河段组合） |
| **聚合地理实体** | aggregated geo-entity data | 为实现统一功能/管理目的形成的数据集合（如多个离散湖泊聚合） |

> **神经连接**：地理实体概念 → 第十一篇 数据组织中GDB要素类的再设计思路（从图层思维到实体思维）、第十七篇 GDB→DWG转换时的实体拆分/合并考量。

## 14.2 实景三维三个层级

这是国家新型基础测绘产品的标准化层级体系：

| 层级 | 核心定位 | 核心数据 | 空间粒度 |
|------|----------|----------|----------|
| **地形级** | 城市级和部件级的承载基础，重点对生态空间的数字映射 | DEM/DSM + DOM/TDOM + 实时感知数据 | 宏观 |
| **城市级** | 地形级的细化，重点对生产和生活空间的数字映射 | 倾斜摄影三维模型 + 激光点云 + 纹理 + 实时感知数据 | 中观 |
| **部件级** | 城市级的分解细化，满足专业化和个性化应用 | 精细模型 | 微观 |

> **神经连接**：地形级 ← 第七篇 DEM 数据（2米格网DEM上承地形级、下启城市级实景三维）；城市级 ← GB/T 39610-2020 倾斜摄影成果（城市级核心数据源）。

## 14.3 实体分类体系

### 三层分类结构

大类（一级类，6位代码前2位）→ 中类（二级类，6位代码前4位）→ 小类（三级类，6位代码）

### 10个大类

| 代码范围 | 大类名称 | 说明 |
|----------|----------|------|
| 110000 | 水系 | 河流、湖泊、水库、海洋、沟渠等 |
| 120000 | 居民地及附属设施 | 建筑物、城镇、工矿区等 |
| 130000 | 交通 | 道路、铁路、机场等 |
| 140000 | 管线 | 管道、电力线、通信线等 |
| 150000 | 境界与政区 | 国界、省界、市界等 |
| 160000 | 地貌 | 等高线、高程点、地形等 |
| 170000 | 植被与土质 | 林地、耕地、草地等 |
| 180000 | 地名 | 地名、地址等 |
| 190000 | 测量控制点 | GNSS点、水准点等 |
| 200000 | 地理实体标识 | 空间身份编码 |

> **与GB/T 20258.1八门类对比**：新增"地名"和"地理实体标识"两个大类，从8门类扩展为10大类。编码范围从6位数字扩展为10位（含图形码和细分码）。

### 粒度划分（I~IV级）

| 粒度级别 | 划分依据 | 适用对象示例 |
|----------|----------|-------------|
| **I类** | 面积大、结构简单 | 大型河流、大型湖泊 |
| **II类** | 面积中等、结构一般 | 建筑物、一般河流 |
| **III类** | 面积较小、结构较复杂 | 小型构筑物、路段 |
| **IV类** | 面积微小、结构精细 | 井盖、路灯、行道树 |

> **神经连接**：分类体系 → 第四篇 CASS 编码体系对照（CASS编码 vs 实体分类代码转换需求）、第六篇 数据组织（GDB中如何按实体模式重组）。

## 14.4 空间身份编码

### 编码结构

空间身份编码 = **位置码** + **类别码(4位)** + **时间码(8位)** + **顺序码**

```
完整编码示例：
NE104 J2525034 230101 0001 MA1001
│      │        │      │    │
│      │        │      │    └─ 扩展标识（MA1001）
│      │        │      └────── 顺序号（0001）
│      │        └───────────── 时间标识（230101=2023年1月1日）
│      └────────────────────── 实体分类代码（J2525034）
└───────────────────────────── 空间位置网格编码（NE104）
```

### 17级递归网格编码

| 层级 | 网格尺度(约) | 适用对象 | 层级 | 网格尺度(约) | 适用对象 |
|------|-------------|----------|------|-------------|----------|
| 1级(全国) | 4898 km | 国家级 | 10级 | 123.69 m | 建筑物级 |
| 2级 | 445 km | 省级 | 11级 | 61.84 m | 楼层级 |
| 3级 | 223 km | 地市级 | 12级 | 30.92 m | 房间级 |
| 4级 | 111 km | 区县级 | 13级 | ~7.73 m | 部件级 |
| 5级 | 55.56 km | 乡镇级 | 14级 | ~0.97 m | 精细部件 |
| 6级 | 18.44 km | 村社级 | 15级 | ~12.1 cm | 厘米级 |
| 7级 | 9.27 km | 大型实体 | 16级 | ~1.5 cm | 超精细级 |
| 8级 | 1.85 km | 中型实体 | 17级 | ~mm级 | 毫米级 |
| 9级 | 370 m | 小型实体 | | | |

> **神经连接**：空间身份编码 ← 第一篇 WKID编码（从WKID到空间身份编码的关联）、第十七篇 数据转换（转换过程中编码保持/映射）。

## 14.5 元数据体系

### 约束条件分类

| 标识 | 类型 | 数量 | 示例 |
|------|------|------|------|
| **M** | 必选 (Mandatory) | ~35项 | 数据名称、坐标系、生产单位等 |
| **C** | 条件必选 (Conditional) | ~15项 | 根据采集技术类型不同（倾斜/激光/建模） |
| **O** | 可选 (Optional) | ~5项 | 根据实际情况选择 |

### 8大类元数据

| 类别 | 主要包括信息项 |
|------|---------------|
| **标识信息** | 数据名称、所属空间(生产/生活/生态)、级别(国家/省/市)、版本、所有权、范围、面积、密级、格式、概略图 |
| **生产信息** | 数据源类型、现势性、生产方法、生产单位、更新资料源、LOD层级、标准编号/名称 |
| **时序信息** | 采集日期、生产日期、更新日期、时序化周期 |
| **精度信息** | 平面位置中误差、高程中误差、类别精度、纹理精度、属性精度 |
| **粒度信息** | 最小颗粒度(m²或m)、粒度划分方式(I/II/III/IV类) |
| **空间参考信息** | 坐标系统、长半径、扁率、地图投影、中央子午线、分带、带号、坐标单位、高程系统名、高程基准 |
| **质量信息** | 完整性、逻辑一致性、结论总分、质检单位/日期、质量总评价(优/良/合格/不合格) |
| **分发信息** | 分发介质、分发单位、电话、传真、地址、邮箱、网址 |

> **神经连接**：元数据体系 ← 第一篇 空间参考信息（长半径/扁率/投影参数）、第十三篇 质量信息关联。

## 14.6 三大处理过程：语义化/结构化/实体化

这是新型基础测绘区别于传统测绘的核心技术链条：

```
语义化 (semantic processing)
  → 用标准化范式对实体属性及关系进行一致性描述
  → 让计算机可识别、可理解、可操作

结构化 (structured processing)
  → 实现地理实体单体分割
  → 获取几何轮廓及组成结构

实体化 (objectified processing)
  → 完整实现语义化+结构化等处理的过程
  → 最终形成可管理的完整地理实体对象
```

**轻量化**：对实景三维数据进行多细节层次（LOD）构建、几何结构概化，以满足不同终端和网络的渲染需求。

> **神经连接**：三大处理过程 → 第十二篇 ArcPy 自动化工具链（可编写脚本辅助实体化处理流程）、第八篇 空间分析（实体级的空间关联分析）。

## 14.7 湖北省分类代码结构（10位）

根据湖北省DLG技术规程(2022)，实体分类代码扩展至10位：

```
门类码(2位) + 大类码(2位) + 中类码(2位) + 小类码(2位) + 图形码(1位) + 细分码(1位)
```

| 图形码 | 定义 | 图形码 | 定义 |
|--------|------|--------|------|
| 0 | 注记 | 3 | 面 |
| 1 | 点 | 4 | 边线 |
| 2 | 线/中心线 | | |

> **神经连接**：10位分类代码 → 第四篇 CASS 内部编码（CASS编码对照映射）、第六篇 GDB 属性字段设计。
