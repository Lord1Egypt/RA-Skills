# FlowSpeech Script Patterns

Use these patterns when the user asks for examples or when the source text needs more structure than the core workflow provides.

## Tagging Patterns

- Soft intro: `[warmly] Welcome to the walkthrough. [pause 0.5s] Today, we will turn a rough draft into a natural voiceover.`
- Emphasis: `The key difference is control. [pause 1s] You can guide emotion, pacing, and delivery without rewriting the whole scene.`
- Transition: `[pause 1s] Now let us compare the before and after.`
- Reflection: `[softly] That version sounds technically correct, but it does not feel human yet. [pause 0.5s]`

## Segment Types

### Product Demo

- Open with the user problem.
- Show one feature at a time.
- Use pauses before screen changes or result reveals.
- Close with a clear next action.

### Education

- Define one concept per segment.
- Add short recap lines after dense explanations.
- Use slower pacing and fewer emotion tags.

### Podcast Or Audiobook

- Use warmer voice direction.
- Preserve narrative rhythm.
- Mark chapter transitions with 1-2 second pauses.

### Dialogue

- Assign stable speaker names.
- Use Multi Speaker mode when each line needs a distinct voice.
- Keep speaker turns short enough to audition and revise independently.

### Short Ads

- Keep copy under 60 words for 20-30 seconds.
- Use one emotional shift at most.
- End with a short call to action.

## Timing

Estimate duration as:

```text
seconds = word_count / words_per_minute * 60
```

Use 130 wpm for careful narration, 145 wpm for normal product voiceover, and 160 wpm for energetic short-form content.
