# 主题与版式说明（wechat-publisher）

`wechat-publisher` 不是排版创作 skill，而是**公众号草稿推送执行层**。

所以这里的“主题”不是决定正文怎么写，而是提醒执行层：

- 输入的 HTML 必须已经是公众号兼容成稿
- 封面图与正文图职责要分开
- 最终成稿判断不能只看“能上传”，还要看“像公众号成稿”

## 执行层默认认知

### 1）封面不等于正文头图

公众号封面素材应作为封面上传使用，不能默认塞进正文第一屏冒充头图。

### 2）能进草稿，不等于能发

就算 `draft/get` 查得到草稿，也还要看：

- 标题是否对
- 正文是否完整
- 是否残留路径、备注、调试文本
- 版式是否像公众号成稿，而不是普通 Markdown 裸排

### 3）执行层不替代主编终审

`wechat-publisher` 负责的是：

- 提交链路稳定
- 核验链路完整
- 结果证据可追溯

它不负责决定文章写得好不好，也不负责替代内容终审。

## 推荐配合方式

在进入 `wechat-publisher` 之前，最好已经具备：

- 已清理过的公众号兼容 HTML
- 单独准备好的封面图
- 明确标题 / 作者 / 摘要
- 主编已确认当前版本允许进入推草稿层

## 一票否决提醒

即使发布链路成功，若出现以下情况，仍不应对外判定“可发”：

- 封面被错误塞进正文
- 正文仍像默认 Markdown 裸排
- 残留本地路径或编辑备注
- 标题和正文明显错配
- 正文图片缺失或阅读节奏明显断裂

## 结论

`wechat-publisher` 解决的是“**稳定推进去并核验清楚**”，不是“替你把文章变好”。

先有成稿，再进发布层；进了发布层，也必须核验后再报成功。
