# Packet Handoff Runbook

Use this runbook only after a match report or intro packet exists.

## Preflight

1. Confirm the user approved preparing an offline intro packet.
2. Confirm the operator is allowed to use the participant data.
3. Confirm the target people came from opt-in, consented, or intentionally supplied data.
4. Prepare a local log with timestamp, operator, target person id, intended action, and outcome.

## Review Mode

- Summarize only high-level signals: role, current project themes, relevant skills, server/community context, and likely fit.
- Do not copy long profile text, posts, comments, or private conversations into generated artifacts.
- Update the CSV with tags and notes only when the information is allowed for this workflow.

## Operator Handoff Mode

Only after approval:

1. Open the approved packet.
2. Re-read the exact draft.
3. Give the packet to the operator.
4. Log whether it was used, skipped, blocked, or changed.
5. Keep evidence out of public artifacts if it exposes private conversations or personal data.

The skill should never claim that it performed external communication. It only prepares local review packets.
