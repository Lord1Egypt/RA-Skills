# PLAN — 策划（CMMI V3.0 Core PA）

## PA 基本信息
- **缩写**：PLAN
- **英文**：Planning
- **中文**：策划
- **所属**：Core PA（核心实践域）
- **对应 V2.0**：相当于 PP（项目计划）+ RSKM（风险管理）的部分内容

---

## CMMI V3.0 要求要点

### 核心目的
确保项目活动被妥善策划、协调、跟踪，基于可理解的需求和约束。

### 关键实践（Specific Practices）
| 实践编号 | 实践内容 | 说明 |
|---------|---------|------|
| PLAN 1.1 | 估算项目需求 | 规模、工作量、成本、进度 |
| PLAN 1.2 | 建立项目定义过程 | 选择生命周期、裁剪组织标准过程 |
| PLAN 1.3 | 识别项目依赖和隐患 | 依赖关系、假设、约束 |
| PLAN 1.4 | 策划项目资源 | 人力、软硬件资源计划 |
| PLAN 1.5 | 策划项目干系人参与 | 沟通计划、干系人管理 |
| PLAN 1.6 | 建立项目计划 | 整合所有策划信息 |
| PLAN 1.7 | 建立项目计划承诺 | 获取干系人承诺 |
| PLAN 2.1 | 实施项目计划 | 按计划执行 |
| PLAN 2.2 | 评估项目计划是否符合组织过程资产 | 与组织标准对比 |
| PLAN 3.1 | 管理项目计划 | 维护计划的完整性 |

### 通用实践（Generic Practices）
- GP 1.1 执行特定实践
- GP 2.1 建立组织政策
- GP 2.2 策划过程
- GP 2.3 提供资源
- GP 2.4 分配责任
- GP 2.5 培训人员
- GP 2.6 管理配置
- GP 2.7 识别并记录干系人需求和接口
- GP 2.8 监督和控制过程
- GP 2.9 客观评估符合性
- GP 2.10 高层审查

## 用户文档 → PA 匹配规则

### 高概率匹配（主要PA）
- 文档名含"项目计划"、"总体计划"、"策划" → **PLAN 主要匹配**
- 文档含WBS、里程碑、资源计划 → **PLAN 主要匹配**

### 中概率匹配（次要PA）
- 文档含风险登记册 → 同时匹配 **RSK**
- 文档含估算记录 → 同时匹配 **EST**
- 文档含需求列表 → 同时匹配 **RDM**

## 差距检查清单（用于差距分析）

### PLAN 1.x — 计划制定
- [ ] 是否有规模/工作量/成本估算记录？
- [ ] 是否定义了项目生命周期（敏捷/瀑布/混合）？
- [ ] 是否有WBS（工作分解结构）？
- [ ] 是否有资源计划（人力/软硬件）？
- [ ] 是否有风险登记册？
- [ ] 是否有干系人沟通计划？

### PLAN 2.x — 计划实施
- [ ] 项目计划是否有版本控制（配置管理）？
- [ ] 计划变更是否有审批记录？
- [ ] 是否与组织标准过程对比过（裁剪依据）？

### PLAN 3.x — 计划管理
- [ ] 是否有定期的进度偏差分析记录？
- [ ] 计划修订是否有历史版本记录？

## 文档转换指导（模式C）

当用户要求将自己的文档转换为CMMI合规格式时，按以下结构输出：

### 转换后的文档结构（Markdown模板）
```markdown
# 项目总体计划
## 1. 项目概述
（目的、范围、假设、约束）

## 2. 项目估算
（规模估算、工作量估算、成本估算、进度估算）

## 3. 项目定义过程
（生命周期选择、裁剪依据、项目定义过程图）

## 4. WBS（工作分解结构）
| WBS编号 | 任务名称 | 负责人 | 工期 | 前置任务 |
|---------|---------|--------|------|----------|

## 5. 资源计划
（人力资源计划、软硬件资源计划）

## 6. 风险登记册
| 风险ID | 风险描述 | 概率 | 影响 | 应对措施 | 负责人 | 状态 |
|---------|---------|--------|------|----------|--------|--------|

## 7. 里程碑计划
| 里程碑 | 交付物 | 计划日期 | 实际日期 | 状态 |
|----------|----------|----------|----------|--------|

## 8. 干系人沟通计划
（沟通频率、方式、对象、内容）

## 9. 版本记录
（计划版本、修订日期、修订内容、修订人）
```

## 评估访谈常见问题（模式D）

### 针对PM（项目经理）的问题
1. "你的项目是如何策划的？请介绍项目计划的主要内容。"
2. "项目估算是如何做出的？用了什么方法？"
3. "项目风险是如何管理的？能展示风险登记册吗？"
4. "项目计划变更的流程是什么？有审批记录吗？"
5. "你如何保证项目计划与组织标准过程一致？"

### 评分要点
- **PLAN 1.1-1.3**：是否有系统化的估算和策划记录
- **PLAN 1.6**：项目计划是否整合了所有策划信息
- **PLAN 2.2**：是否对照了组织标准过程（裁剪记录）
- **PLAN 3.1**：是否有定期的计划维护和偏差分析
