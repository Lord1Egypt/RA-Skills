# 网络小说创作知识参考库

## 定位

本技能是 **Universal Task OS 的领域负载物仓库**，不包含任何执行框架。只提供网络小说创作的"是什么"和"长什么样"——执行全部委托UTOS。

| 本技能提供 | UTOS消费方式 |
|-----------|-------------|
| 写作任务清单（有哪些任务） | 内容轴·清单法的成品目录 |
| 结构要求（每种任务的组件槽位） | 内容轴·清单法的组件清单 |
| 经典范本（待填充） | 内容轴·样本法的样本 |
| 任务依赖拓扑（任务间先后关系） | 执行轴·管线编排的依赖输入 |

## 三层结构

```
第一层：写作任务清单 + 依赖拓扑   →  references/writing-catalog.md
第二层：结构要求清单              →  references/structure-requirements.md
第三层：经典范本库                →  references/exemplars.md
```

## 依赖声明

本技能**强依赖** Universal Task OS (universal-task-os)。没有UTOS，本技能只有参考查阅能力，无法执行任何写作产出任务。

**加载检查流程**（每次激活时执行）：

1. 检测 `universal-task-os` 技能是否已安装
2. **未安装** → 自动安装 `universal-task-os` 技能
3. **安装成功** → 同时加载UTOS，按本技能"使用规则"执行
4. **安装失败** → 降级为**只读参考模式**：
   - ✅ 允许：查阅任务清单、结构要求、范本索引
   - ❌ 拒绝：任何涉及写作产出、管线编排、内容生成的任务，并提示"需先安装 Universal Task OS"

**任务模式判定**：

| 任务类型 | 无UTOS | 有UTOS |
|---------|--------|--------|
| 查阅任务清单/要求/范本 | ✅ 只读参考 | ✅ 完整 |
| 按清单/范本产出章节/大纲 | ❌ 拒绝 | ✅ UTOS编排执行 |
| 依赖拓扑推导创作管线 | ❌ 拒绝 | ✅ UTOS执行轴 |
| 平台合规检查点插入 | ❌ 拒绝 | ✅ UTOS守护单元 |

## 使用规则

1. **依赖检查**：激活时按上述流程检测并安装UTOS
2. **首次加载**：读取 `references/writing-catalog.md`，获取域分类、依赖拓扑、UTOS元操作映射提示
3. **按需深入**：确认目标任务类型后，读取 `references/structure-requirements.md` 获取组件清单；如需样本法，读取 `references/exemplars.md` 获取范本
4. **委托UTOS**：将任务清单作为清单法输入、范本作为样本法输入、依赖拓扑作为管线编排输入，交给UTOS执行轴+内容轴处理
5. **用户填充**：范本槽位标注 `[待用户提供]` 的条目需用户提供脱敏真实范本后才能使用样本法；结构要求中标注 `[待补充]` 的字段需用户定义后才能完整使用清单法

## 与UTOS的接口

当UTOS处理网络小说领域任务时：

- **Step 0 三轴判定**：网文任务通常为中等+结构化 → 执行轴+内容轴激活；需要创新时额外激活创新轴
- **Step 1 领域校准**：网文=高创造性(R2)+高迭代性(R5) → A权重极高，C偏发散，G偏宽松，循环多
- **Step 2 内容轴**：清单法用本技能的structure-requirements；样本法用本技能的exemplars
- **Step 3 执行轴**：管线编排基于本技能的依赖拓扑自动推导元操作序列
- **Step 4 交付**：G类守护单元自动插入平台合规检查点（敏感词、版权红线、平台规则）

## 网络小说特有维度

与传统出版文学相比，网络小说有以下必须考虑的特有维度：

### 平台生态

| 维度 | 说明 |
|------|------|
| **连载机制** | 日更/周更/断更惩罚、字数门槛、更新频率与读者留存的关系 |
| **平台规则** | 各大平台（起点/晋江/番茄/飞卢/刺猬猫等）的内容审核标准差异 |
| **数据指标** | 追读率、收藏比、推荐票、月票、打赏、均订等关键指标 |
| **算法推荐** | 标题/简介/标签/封面对曝光量的影响、黄金三章效应 |
| **变现路径** | 付费阅读→IP授权→影视改编→游戏改编→周边衍生 |

### 创作节奏

| 维度 | 说明 |
|------|------|
| **爽点密度** | 每1000-3000字的情绪峰值分布、期待感管理 |
| **悬念钩子** | 章末留白策略、断章技巧、追读驱动设计 |
| **伏笔系统** | 长线/中线/短线的分层埋设和回收节奏 |
| **节奏曲线** | 开篇吸引→中期稳定→高潮爆发→收尾完整的全生命周期节奏 |
