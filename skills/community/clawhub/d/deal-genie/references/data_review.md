# 数据复盘模型 (data_review.md)

## 概述

本文件定义数据复盘（Step 12）的完整模型。没有复盘就没有成长——本节把推品结果变成下次推品的优化依据，形成「推品→复盘→优化→再推品」的自我进化闭环。同时，复盘结果会反哺 Step 1 的选品决策，让整个系统的选品能力越用越准。

---

## A. 数据输入模板

### 用户需要提供的数据

| 数据项 | 必填 | 说明 |
|--------|:---:|------|
| 推品商品 | ✅ | 商品名称 |
| 出单总量 | ✅ | 推品期间总出单数 |
| GMV（总销售额） | ✅ | 出单数 × 客单价，可自动计算 |
| 佣金总额 | ✅ | 实际到账佣金 |
| 佣金率 | ✅ | 用于计算预估和实际对比 |
| 推品达人数量 | ⚪ | 如有多个达人推同一品 |
| 各达人出单数 | ⚪ | 用于达人效率排名 |
| 寄样成本 | ⚪ | 样品费+快递费 |
| 人力投入 | ⚪ | 沟通/运营投入的时间成本估算 |
| 退货/退款数 | ⚪ | 用于计算净成交 |
| 推品日期范围 | ⚪ | 用于判断推品周期合理性 |

### 自动补全

如果用户只提供了部分数据，AI 自动补全：

- 有出单数+客单价 → 自动算 GMV
- 有 GMV+佣金率 → 自动估算佣金总额
- 无寄样成本 → 按行业均值估算（20元/样品）

## B. ROI 分析模型

### 核心指标计算

```
┌──────────────────────────────────────────────────────────────┐
│                         ROI 分析                              │
├──────────┬──────────────────────────────────────────────────┤
│ GMV      │ 出单数 × 客单价                                    │
│ 佣金收入  │ GMV × 佣金率（或用户直接提供）                      │
│ 总成本    │ 寄样成本 + 人力成本估算 + 其他费用                  │
│ 净利润    │ 佣金收入 - 总成本                                  │
│ ROI      │ (净利润 ÷ 总成本) × 100%                           │
│ 单均佣金  │ 佣金收入 ÷ 出单数                                  │
│ 退货率    │ 退货数 ÷ 出单数 × 100%                             │
│ 净成交    │ 出单数 - 退货数                                    │
└──────────┴──────────────────────────────────────────────────┘
```

### 投资回报阈值

| ROI | 评级 | 说明 |
|-----|------|------|
| > 300% | 🟢 优秀 | 暴利品，可加大投入 |
| 150% - 300% | 🟡 良好 | 正常利润，可继续推 |
| 50% - 150% | 🟠 一般 | 微利，需优化成本或提高佣金 |
| < 50% | 🔴 亏损 | 不值得继续推，检查问题环节 |

## C. 行业对标

### 对标规则

将本品的核心指标与同类目行业基准做对比（基准数据来自 `category_benchmarks.md`）：

```
┌──────────────────┬──────────┬──────────┬──────────┐
│ 指标             │ 本品     │ 行业基准  │ 对标     │
├──────────────────┼──────────┼──────────┼──────────┤
│ 佣金率           │ 30%      │ 25%      │ 🟢 高于  │
│ 转化率           │ 2.1%     │ 3.5%     │ 🔴 低于  │
│ 退货率           │ 5%       │ 8%       │ 🟢 低于  │
│ 平均推品周期     │ 5天      │ 7天      │ 🟢 快于  │
│ 达人推品转化率   │ 40%      │ 60%      │ 🔴 低于  │
└──────────────────┴──────────┴──────────┴──────────┘
```

### 对标结论

根据对标结果，给出综合判断：

- 🟢 3项以上高于基准 → 品类优势明显，可加大投入
- 🟡 2-3项持平 → 品类可行，需精细化运营
- 🔴 3项以上低于基准 → 品类或选品有问题，需要调整

## D. 环节诊断模型

### 诊断逻辑

根据数据异常特征，自动定位问题环节：

```
输入的异常信号 → 匹配诊断规则 → 输出问题环节 + 改进建议
```

### 诊断规则表

| 数据特征 | 诊断结果 | 问题环节 | 改进建议 |
|---------|---------|---------|---------|
| ROI < 50%（佣金率正常） | 成本过高或出单太少 | 选品/达人匹配 | 检查寄样成本是否可控；检查达人匹配度 |
| 佣金率低但出单多 | 利润薄 | 选品谈判 | 跟品牌方谈更高佣金率或阶梯佣金 |
| 出单数低于同类目均值 | 曝光不足或达人不对 | 达人匹配/素材 | 扩大达人触达数量；优化素材吸引力 |
| 转化率低于2% | 文案/价格/信任问题 | 文案/策略/定价 | 检查文案角度是否正确；定价是否合理 |
| 退货率高于8% | 产品质量/文案夸大 | 选品/文案 | 检查品控；降低文案中的夸张承诺 |
| 首日出单好后续断崖 | 追单没跟上 | 追单转化 | 加强推品后的社群追单和复购引导 |
| 达人拒推率高(>40%) | 品不够吸引人或佣金不够 | 选品/佣金 | 调整佣金结构或更换推品策略 |
| 寄样后沉默率高(>30%) | 达人体验不好或不感兴趣 | 选品/寄样后跟进 | 改进样品体验；强化寄样后的跟进节奏 |

### 诊断输出格式

```
🔍 问题诊断：

发现 2 个关键问题：

❶ 转化率偏低（2.1% vs 行业基准 3.5%）
   可能原因：文案角度偏"认知教育"，但本品属于快消品，用户不需要被教育
   改进建议：切换到"好物分享"或"价格锚定"策略，降低用户认知成本

❷ 达人推品转化率偏低（40% vs 行业基准 60%）
   可能原因：邀约了过多低匹配度的达人；或者样品寄出后跟进不到位
   改进建议：
   - 下次优先邀约测评型和好物分享型达人（见达人匹配模块）
   - 寄样后第1天、第3天各发一次跟进（见寄样管理模块）
```

## F. 偏好学习（自我进化机制）

### 什么是偏好学习

每次数据复盘后，系统自动更新用户的优势品类档案。下次选品时，系统会基于历史数据给出更精准的建议。

### 学习机制

```
本次推品数据 → 品类标签化 → 更新品类档案 → 影响下次选品权重
```

### 品类档案更新

```
品类档案 — 洗发护发

历史推品次数：5次
平均 ROI：210%
最优策略：利益驱动
最优达人：测评型
最佳客单区间：50-100元
最佳佣金率：25%-35%

→ 下次选品建议：
  同类目新品优先级提升，推荐佣金率≥25%的品
  优先匹配测评型达人
  预期 ROI 参考：150%-250%
```

### 下次推品改进清单

每次复盘后，自动生成 3-5 条可执行的改进建议：

```
📋 下次推品改进清单：

1. [高优] 切换到"好物分享"策略方向，文案角度从认知教育改为场景种草
   → 上次转化率低的核心原因是策略方向和品类属性不匹配

2. [高优] 寄样时附带"30秒试用体验指南"，降低达人的试用门槛
   → 上次有2个达人反馈"不知道怎么写推荐"

3. [中优] 增加1个"促销带货型"达人做补充，覆盖冲动消费人群
   → 上次只有测评型和好物分享型，渠道不够多元

4. [中优] 推品前1天增加朋友圈悬念预热
   → 上次预热只在社群做，朋友圈渠道没有利用

5. [低优] 收集用户真实反馈截图，丰富素材库
   → 减少从零做素材的成本，后续推品直接复用
```

## 复盘频率建议

| 复盘类型 | 频率 | 内容 |
|---------|------|------|
| 单品复盘 | 每次推品结束后 | ROI + 环节诊断 + 改进清单 |
| 周复盘 | 每周日 | 本周所有推品汇总 + 品类趋势 |
| 月复盘 | 每月初 | 品类档案更新 + 达人效率排名 + 策略调整 |
