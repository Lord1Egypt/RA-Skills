## 二、每个模块详解

### 1. Problem（问题）

**核心问题**：目标用户面临的Top 3痛点

| 填写内容 | 说明 |
|---------|------|
| 问题1 | [最痛的那个点] |
| 问题2 | [次要痛点] |
| 问题3 | [第三个痛点] |

**验证方法**：
- 用户访谈：直接询问用户当前的解决方案
- 痛苦程度量表：1-10分，8分以上才算真正的痛点

---

### 2. Customer Segments（客户群体）

**核心问题**：谁是你的目标用户？

| 填写内容 | 说明 |
|---------|------|
| 早期采纳者 | 最愿意尝试你产品的人（不是所有人） |
| 用户画像 | 年龄/职业/行为特征 |
| 规模估算 | 目标市场的用户数量 |

**验证方法**：
- 找到100个愿意为你的想法付钱的真实用户

---

### 3. Unique Value Proposition（独特价值主张）

**核心问题**：为什么用户要选你？

| 填写内容 | 说明 |
|---------|------|
| 目标用户 | [谁] |
| 产品名称 | [叫什么] |
| 核心价值 | [解决什么问题，提供什么价值] |
| 情感/功能 | [功能价值 or 情感价值] |

**优秀UVP公式**：
```
[产品名称] 是为了 [目标用户] 
谁需要 [核心价值] 
不同于 [竞品] 
我们的优势是 [差异化]
```

---

### 4. Solution（解决方案）

**核心问题**：你如何解决这些问题？

| 填写内容 | 说明 |
|---------|------|
| 功能1 | 对应问题1的解决方案 |
| 功能2 | 对应问题2的解决方案 |
| 功能3 | 对应问题3的解决方案 |

**优先级原则**：只做Top 3功能，不要贪多

---

### 5. Channels（渠道）

**核心问题**：如何触达用户？

| 渠道类型 | 示例 |
|---------|------|
| 口碑传播 | 用户推荐、KOL背书 |
| 有机获客 | SEO、内容营销、社交媒体 |
| 付费获客 | 广告投放、联盟营销 |
| 直销 | 电话销售、一对一沟通 |
| 平台分发 | 应用商店、第三方平台 |

**验证方法**：找到成本最低的渠道

---

### 6. Key Metrics（关键指标）

**核心问题**：如何衡量成功？

| 指标类型 | 示例 |
|---------|------|
| 获客指标 | 新用户数、获客成本（CAC） |
| 激活指标 | 首次使用率、激活时间 |
| 留存指标 | 留存率、流失率 |
| 变现指标 | 收入、客单价、 Lifetime Value (LTV) |
| 推荐指标 | NPS、推荐率 |

**北极星指标**：选择一个最核心的指标

---

### 7. Unfair Advantage（不公平优势）

**核心问题**：为什么你不会被轻易复制？

| 优势类型 | 示例 |
|---------|------|
| 资产壁垒 | 专利技术、独家数据 |
| 关系壁垒 | 独家合作、深度信任 |
| 认知壁垒 | 品牌心智、用户习惯 |
| 网络效应 | 用户越多越有价值 |
| 规模效应 | 规模越大成本越低 |

**注意**：不能说"执行力"或"团队"——这些可以被复制

---

### 8. Cost Structure（成本结构）

**核心问题**：做这件事需要花多少钱？

| 成本类型 | 示例 |
|---------|------|
| 固定成本 | 人员工资、房租、设备 |
| 变动成本 | 服务器带宽、客服人力、物流 |
| 规模经济 | 随规模下降的成本项 |

---

### 9. Revenue Streams（收入来源）

**核心问题**：如何赚钱？

| 收入模型 | 适用场景 |
|---------|---------|
| 订阅制 | SaaS、会员服务 |
| 交易抽成 | 平台型业务 |
| 一次性付费 | 工具类产品 |
| 增值服务 | Freemium模式 |
| 广告收入 | 流量型产品 |
| 授权费 | IP、专利授权 |

## 四、检查清单

```
✅ Problem：识别了真实存在的Top 3痛点
✅ Customer Segments：明确了早期采纳者画像
✅ Unique Value Proposition：清晰、可衡量、差异化
✅ Solution：功能与问题一一对应
✅ Channels：找到1-2个可测试的渠道
✅ Key Metrics：定义了可量化的成功指标
✅ Unfair Advantage：识别了真正的护城河
✅ Cost Structure：估算出了启动成本
✅ Revenue Streams：明确了收入模型
```

---

> **使用建议**：用这个框架梳理想法，确保每个模块都有清晰的答案。如果某个模块填不出来，说明想法还需要进一步打磨。
