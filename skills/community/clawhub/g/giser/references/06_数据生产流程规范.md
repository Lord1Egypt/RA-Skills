## 二、DLG（数字线划图）生产流程

### 2.1 适用标准

| 标准 | 用途 |
|------|------|
| GB/T 13923-2022 | 要素分类与编码 |
| GB/T 20258.1~4-2019 | 要素数据字典（各比例尺） |
| CH/T 9020.1~3 | 数字成果规范 |
| GB/T 33176~33178-2016 | 地形图精度标准 |

### 2.2 标准 SOP

```
步骤1：资料准备
  ├── 影像数据（DOM/航片/卫片）
  ├── 外业调绘成果
  ├── 已有矢量数据（更新项目）
  └── 技术设计书

步骤2：数据采集
  ├── 立体采集（航测立体像对）或
  ├── 单片采集（DOM底图数字化）或
  ├── LiDAR辅助采集
  └── 编码规则：GB/T 13923 六位数字编码

步骤3：数据编辑
  ├── 几何编辑：节点捕捉、悬挂处理、面闭合
  ├── 拓扑构建：Build Topology → 检查规则
  ├── 属性录入：要素编码 + 扩展属性
  └── 注记配置：字体/大小/位置规则

步骤4：数据接边
  ├── 相邻图幅几何接边（位置接边容差：图上0.1mm）
  ├── 属性接边一致性检查
  └── 跨带接边特殊处理（投影带边缘）

步骤5：符号化与制图
  ├── 符号库配置（按 GB/T 20257 图式规范）
  ├── 图廓整饰：图名/图号/比例尺/图例/接图表
  └── 输出：PDF/Plot

步骤6：格式转换与入库
  ├── → GeoPackage（推荐现代格式）
  ├── → File GDB（ArcGIS生态）
  ├── → Shapefile（兼容性交付）
  └── 坐标系：统一为 CGCS2000 / EPSG:4490
```

### 2.3 关键参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 坐标系统 | CGCS2000 (EPSG:4490) | 中国法定坐标系 |
| 高程基准 | 1985国家高程基准 | 正常高系统 |
| 要素编码 | GB/T 13923 六位码 | 如 310101 = 单幢房屋 |
| 拓扑容差 | 0.001m（投影）/ 0.0000001°（地理） | → 参见 29_避坑库110+.md |
| 接边容差 | 图上 ≤0.1mm | — |
| 数据格式 | GeoPackage (推荐) → Shapefile/File GDB | 格式对比→03_数据模型与格式.md |

## 三、DEM（数字高程模型）生产流程

### 3.1 适用标准

| 标准 | 用途 |
|------|------|
| CH/T 9009.2-2010 | DEM数字成果 |
| GB/T 24356-2023 | DEM质检标准 |
| GB/T 33176-2016 | 等高距/高程精度(以大比例尺地形图为参照) |

### 3.2 生产方法对比

| 方法 | 数据源 | 精度 | 适用比例尺 | 软件工具 |
|------|--------|------|-----------|---------|
| **摄影测量法** | 航片/卫片立体像对 | 高 | 1:500~1:10000 | ArcGIS Pro Image Analyst / 航天远景MapMatrix |
| **LiDAR法** | 机载/地面LiDAR点云 | 最高 | 1:500~1:2000 | GlobalMapper Pro / TerraScan / PDAL(开源) |
| **干涉雷达法** | InSAR数据 | 中 | 1:50000~1:250000 | SNAP / GMTSAR |
| **地形图矢量化** | 已有等高线数据 | 中-低 | 1:10000~1:100000 | ArcGIS Pro Topo to Raster |

### 3.3 标准 SOP（摄影测量法）

```
步骤1：资料准备
  ├── 空中三角测量成果（或RPC参数）
  ├── 影像数据
  └── 控制点数据

步骤2：影像匹配
  ├── 自动匹配生成点云
  ├── 匹配策略：半全局匹配(SGM) / 多视角匹配(MVS)
  └── 生成初始DSM

步骤3：点云滤波（提取地面点）
  ├── 渐进不规则三角网加密(PTD)
  ├── 布料模拟滤波(CSF) ← 建筑物密集区推荐
  └── 手动编辑特殊地形（陡坎/桥梁/水域）

步骤4：DEM内插
  ├── 方法选择：
  │   ├── 连续地形 → 双线性/三次卷积
  │   ├── 突变地形(TIN-based) → 自然邻域法
  │   └── 大面积 → ANUDEM(ArcGIS Topo to Raster)
  └── 格网间距：按比例尺和精度要求

步骤5：DEM编辑
  ├── 水域置平（保证水面水平）
  ├── 等高线生成验证
  └── 异常值检测与修复

步骤6：精度评定
  ├── 检查点法：预留20%控制点不参与计算
  └── 中误差公式：RMSE = √[Σ(Z实测 - ZDEM)² / n]
```

### 3.4 DEM精度指标

| 比例尺 | 格网间距 | 高程中误差（平地） | 高程中误差（山地） |
|--------|---------|-------------------|-------------------|
| 1:500 | 0.5m | ≤0.15m | ≤0.5m |
| 1:1000 | 1m | ≤0.3m | ≤0.7m |
| 1:2000 | 2m | ≤0.5m | ≤1.2m |
| 1:5000 | 5m | ≤1.0m | ≤2.5m |
| 1:10000 | 5m/10m | ≤1.5m | ≤4.0m |

## 四、DOM（数字正射影像）生产流程

### 4.1 适用标准

| 标准 | 用途 |
|------|------|
| GB/T 15968-2008 | 遥感影像平面图制作 |
| CH/T 9008.3-2010 | DOM数字成果 |
| GB/T 24356-2023 | DOM质检 |
| GB/T 39610-2020 | 倾斜摄影(含真正射) |

### 4.2 标准 SOP

```
步骤1：影像预处理
  ├── 辐射校正：辐射定标 → 大气校正(FLAASH/6S/QUAC)
  ├── 几何校正：系统几何校正 + RPC优化
  └── 影像融合（多光谱+全色）：Gram-Schmidt / NNDiffuse / Pansharp

步骤2：正射校正
  ├── 输入：原始影像 + DEM + RPC/定向参数
  ├── 方法：
  │   ├── 单片正射（有DEM即可）
  │   └── 真正射（需要DSM，消除建筑物倾斜）
  └── 重采样方法：双线性（连续值）/ 最邻近（分类影像）

步骤3：匀色与镶嵌
  ├── 单幅匀色：直方图匹配/Wallis滤波
  ├── 多幅镶嵌：自动生成镶嵌线（避开建筑物）
  └── 羽化过渡：消除镶嵌缝

步骤4：裁切与分幅
  ├── 按标准图幅裁切
  ├── 图幅编号：国家基本比例尺地图编号规则
  └── 输出格式：GeoTIFF（推荐）/ IMG / JPEG2000
```

### 4.3 DOM精度指标

| 比例尺 | 地面分辨率 | 平面中误差 |
|--------|-----------|-----------|
| 1:500 | ≤0.05m | ≤0.25m |
| 1:1000 | ≤0.1m | ≤0.5m |
| 1:2000 | ≤0.2m | ≤1.0m |
| 1:5000 | ≤0.5m | ≤2.5m |

## 五、生产环境配置建议

### 5.1 坐标系统一原则

> **所有生产数据统一到 CGCS2000 坐标系(EPSG:4490)**。这是不可协商的刚性要求。

```python
# Python 批量坐标系统一脚本
import arcpy
arcpy.env.workspace = r"D:\data\input"
target_crs = arcpy.SpatialReference(4490)  # CGCS2000

for fc in arcpy.ListFeatureClasses():
    desc = arcpy.Describe(fc)
    current_crs = desc.spatialReference.factoryCode
    if current_crs != 4490:
        arcpy.management.Project(fc, f"D:\\data\\output\\{fc}", target_crs)
        print(f"✅ {fc}: {current_crs} → 4490")
```

### 5.2 编码统一原则

- 所有属性编码使用 **UTF-8**
- ArcGIS Pro 设置环境变量 `ARCGIS_CODEPAGE=UTF8`
- 读取外部 Shapefile 时先尝试 UTF-8，乱码备选 GBK
- 参见 `29_避坑库110+.md` B.1.2 编码乱码坑

## 六、常见生产问题速查

| 问题 | 原因 | 解决 | 详见 |
|------|------|------|------|
| DLG属性编码缺失 | 未引用GB/T 13923 | 建立编码对照表 | 05 |
| DEM水域不平 | 未做水域置平处理 | 提取水域多边形→约束插值 | — |

| 影像色差明显 | 未做匀色 | Wallis滤波 / 直方图匹配 | — |
| 成果坐标偏移 | 坐标系不一致 | 全流程统一CGCS2000 | 04 |
