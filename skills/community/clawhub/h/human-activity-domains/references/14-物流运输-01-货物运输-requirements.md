## 货物运输生成要求

货物运输每种任务类型的"零件清单"——必选/可选组件、组装顺序、领域约束。按清单逐项产出。

---

### 1 运输规划

#### T1-1 运输需求

- **必选组件**: 货物信息、发货地、目的地、时效要求
- **可选组件**: 特殊要求、包装要求、保险要求
- **组装顺序**: 收集货物信息 → 确定发货地 → 确定目的地 → 明确时效要求
- **约束**: 信息必须准确；地点必须明确；时效必须可行
- **格式**: 运输需求单（Markdown）

#### T1-2 路线规划

- **必选组件**: 路线选择、距离计算、时间预估、路线说明
- **可选组件**: 路线优化、路线备选、路线风险
- **组装顺序**: 选择路线 → 计算距离 → 预估时间 → 撰写说明
- **约束**: 路线必须可行；距离必须准确；时间必须合理
- **格式**: 路线规划方案（Markdown）

#### T1-3 运力安排

- **必选组件**: 车辆调配、司机安排、装载方案、运力说明
- **可选组件**: 运力优化、运力备选、运力成本
- **组装顺序**: 调配车辆 → 安排司机 → 设计装载方案 → 撰写说明
- **约束**: 车辆必须匹配；司机必须合格；装载必须合理
- **格式**: 运力安排方案（Markdown）

#### T1-4 成本预算

- **必选组件**: 燃油费、过路费、人工费、保险费
- **可选组件**: 其他费用、成本优化、成本对比
- **组装顺序**: 估算燃油费 → 估算过路费 → 估算人工费 → 估算保险费
- **约束**: 费用必须准确；预算必须合理；成本必须可控
- **格式**: 成本预算表（Markdown）

---

### 2 运输执行

#### T2-1 货物接收

- **必选组件**: 货物检查、包装确认、签收确认、接收记录
- **可选组件**: 检查工具、检查标准、检查培训
- **组装顺序**: 检查货物 → 确认包装 → 确认签收 → 记录接收
- **约束**: 检查必须仔细；包装必须完好；签收必须确认
- **格式**: 货物接收记录（Markdown）

#### T2-2 装载运输

- **必选组件**: 装载操作、运输执行、安全监控、运输记录
- **可选组件**: 装载工具、装载培训、装载检查
- **组装顺序**: 执行装载 → 执行运输 → 监控安全 → 记录运输
- **约束**: 装载必须规范；运输必须安全；监控必须及时
- **格式**: 装载运输记录（Markdown）

#### T2-3 在途监控

- **必选组件**: 位置追踪、状态监控、异常预警、监控记录
- **可选组件**: 监控工具、监控系统、监控团队
- **组装顺序**: 追踪位置 → 监控状态 → 预警异常 → 记录监控
- **约束**: 追踪必须准确；监控必须及时；预警必须有效
- **格式**: 在途监控记录（Markdown）

#### T2-4 货物交付

- **必选组件**: 到达通知、货物卸载、签收确认、交付记录
- **可选组件**: 交付工具、交付标准、交付培训
- **组装顺序**: 通知到达 → 卸载货物 → 确认签收 → 记录交付
- **约束**: 通知必须及时；卸载必须规范；签收必须确认
- **格式**: 货物交付记录（Markdown）

- **必选组件**: 服务内容、服务标准、服务渠道、服务质量
- **可选组件**: 服务工具、服务系统、服务团队
- **组装顺序**: 确定服务内容 → 制定服务标准 → 建立服务渠道 → 监控服务质量
- **约束**: 内容必须全面；标准必须明确；质量必须保证
- **格式**: 客户服务方案（Markdown）

#### T4-2 异常处理

- **必选组件**: 异常类型、处理流程、处理措施、处理记录

- **组装顺序**: 识别异常类型 → 启动处理流程 → 执行处理措施 → 记录处理
- **约束**: 类型必须明确；流程必须规范；措施必须有效
- **格式**: 异常处理方案（Markdown）

#### T4-3 投诉处理

- **格式**: 投诉处理方案（Markdown）

- **约束**: 分析必须深入；措施必须有效；效果必须可验证
