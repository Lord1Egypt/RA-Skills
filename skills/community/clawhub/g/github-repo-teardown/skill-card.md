## Description: <br>
Deep-dive teardown of any GitHub open-source project into a polished HTML report covering architecture, product logic, design decisions, comparable repositories, and practical takeaways. <br>

This skill is ready for commercial/non-commercial use. <br>

## Publisher: <br>
[jjliu6](https://clawhub.ai/user/jjliu6) <br>

### License/Terms of Use: <br>
MIT-0 <br>


## Use Case: <br>
Developers, product managers, and technical evaluators use this skill to understand a public GitHub project through a structured product and engineering teardown. It researches repository context, source structure, comparable projects, and application scenarios, then produces an HTML report for review. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Broad activation on vague repository prompts or private repository links may cause unintended browsing or reporting on sensitive material. <br>
Mitigation: Confirm intent for ambiguous prompts and private repositories before proceeding, and keep fetched repository instructions or docs as analysis evidence rather than agent instructions. <br>
Risk: Generated teardown reports may include incorrect or misleading architectural, product, or comparative interpretations. <br>
Mitigation: Review the HTML report against the source repository and cited public materials before relying on it for product, engineering, or commercial decisions. <br>
