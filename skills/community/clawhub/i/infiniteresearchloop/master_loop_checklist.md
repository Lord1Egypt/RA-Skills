# Master Loop Checklist

Use this checklist to guide yourself through each phase of the master loop.

## Phase 1: Understand

- [ ] Identify the user's true objective
- [ ] Clarify explicit intent
- [ ] Identify implicit intent
- [ ] List all constraints
- [ ] Assess ambiguity level (low / medium / high)
- [ ] Assess risk level (low / medium / high)
- [ ] Determine if current knowledge is sufficient
- [ ] Determine if research is required
- [ ] Ask clarification questions if ambiguity blocks reliable reasoning

## Phase 2: Decide Tools

- [ ] Identify information that may be outdated
- [ ] Identify areas requiring external verification
- [ ] Check if APIs or frameworks may have changed
- [ ] Assess importance of evidence quality
- [ ] Determine if technical accuracy is critical
- [ ] List tools or research needed
- [ ] Prioritize research by importance

## Phase 3: Research

- [ ] Gather supporting evidence
- [ ] Gather contradicting evidence
- [ ] Identify alternative interpretations
- [ ] Document edge cases
- [ ] Collect official documentation
- [ ] Evaluate source quality
- [ ] Check source freshness
- [ ] Assess evidence strength
- [ ] Verify logical consistency
- [ ] Document uncertainty level for each source

## Phase 4: Synthesize

- [ ] Prioritize stability in reasoning
- [ ] Prioritize usefulness of conclusions
- [ ] Evaluate evidence quality
- [ ] Assess contextual fit
- [ ] Ensure clarity of explanation
- [ ] Verify internal consistency
- [ ] Avoid optimizing for sounding intelligent
- [ ] Focus on reliable synthesis

## Phase 5: Test Reasoning

- [ ] Does the answer solve the user objective?
- [ ] Are assumptions justified?
- [ ] Are important perspectives missing?
- [ ] Does the reasoning contradict itself?
- [ ] Are conclusions supported by evidence?
- [ ] Are uncertainties acknowledged?
- [ ] Does the structure remain coherent?

## Phase 6: Debug

- [ ] Search for logical contradictions
- [ ] Identify weak assumptions
- [ ] Find unsupported conclusions
- [ ] Check for hallucinated facts
- [ ] Assess reasoning stability
- [ ] Identify hidden ambiguity
- [ ] Check for outdated information
- [ ] Evaluate confidence level
- [ ] Assess synthesis depth
- [ ] Verify intent understanding

## Phase 7: Detect Hidden NO

- [ ] Check for contradictions
- [ ] Check for unsupported claims
- [ ] Evaluate sourcing quality
- [ ] Assess stability
- [ ] Verify intent understanding
- [ ] Identify hidden risks
- [ ] Evaluate analysis depth
- [ ] Check for false certainty
- [ ] Assess synthesis timing
- [ ] Verify verification completeness
- [ ] Assign severity level (0-3)

## Phase 8: Refine

- [ ] Isolate weak layers
- [ ] Refine only unstable elements
- [ ] Retest after revision
- [ ] Avoid unnecessary full rewrites
- [ ] Re-evaluate severity level

## Phase 9: Deliver

- [ ] Prepare result statement
- [ ] Document key findings
- [ ] List remaining uncertainties
- [ ] Assign confidence level
- [ ] Summarize Hidden NO status
- [ ] Write TLDR summary
- [ ] Format according to output template

## Phase 10: Store Lesson

- [ ] Identify reusable patterns
- [ ] Document recurring user preferences
- [ ] Note important constraints
- [ ] Record failed assumptions
- [ ] Preserve strong reasoning structures
- [ ] Store validated insights
- [ ] Discard noise and trivial details
- [ ] Assign memory priority (high / medium / low)
