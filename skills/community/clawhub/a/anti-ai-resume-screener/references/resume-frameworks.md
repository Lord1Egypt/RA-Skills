# 简历写作框架参考

## 一、弱词→强词替换表

将简历中的无力词汇替换为更有冲击力的表达。

### 领导/管理类

| 弱词 | 强词替换 | 适用场景 |
|------|---------|---------|
| 负责 | 主导、统筹、全面管理 | 体现决策权和领导力 |
| 带领 | 搭建、组建、从零到一建立 | 体现创始能力 |
| 参与 | 深度参与、核心成员、骨干 | 体现重要性 |
| 协助 | 独立完成、协同推进、联合主导 | 体现独立能力 |

### 执行/完成类

| 弱词 | 强词替换 | 适用场景 |
|------|---------|---------|
| 做 | 执行、落地、实施、交付 | 更专业 |
| 完成 | 超额完成、提前交付、高质量完成 | 体现超越预期 |
| 搞定 | 攻克、突破、解决 | 体现问题解决能力 |
| 处理 | 优化、重构、系统性解决 | 体现方法论 |

### 分析/研究类

| 弱词 | 强词替换 | 适用场景 |
|------|---------|---------|
| 看 | 分析、研究、调研、洞察 | 体现专业性 |
| 了解 | 深入研究、系统性分析、全面调研 | 体现深度 |
| 总结 | 提炼、归纳、输出方法论 | 体现思考深度 |

### 优化/改进类

| 弱词 | 强词替换 | 适用场景 |
|------|---------|---------|
| 改进 | 重构、优化、升级、迭代 | 体现系统性 |
| 提升 | 突破、跃升、显著提升 | 体现幅度 |
| 减少 | 降低、压缩、精简、降本 | 体现成本意识 |

### 创新/创造类

| 弱词 | 强词替换 | 适用场景 |
|------|---------|---------|
| 想 | 设计、规划、构思、策划 | 体现执行力 |
| 建立 | 搭建、构建、从零到一创建 | 体现创始能力 |
| 开发 | 独立开发、自主研发、创新开发 | 体现技术能力 |

### 协作/沟通类

| 弱词 | 强词替换 | 适用场景 |
|------|---------|---------|
| 联系 | 对接、联动、整合、协调 | 体现组织能力 |
| 合作 | 深度合作、战略合作、联合推进 | 体现合作深度 |
| 跟进 | 推动、驱动、持续推进、闭环管理 | 体现主动性 |

## 二、STAR 法则详解

- **S（Situation）**：项目背景 - 什么情况下做的？
- **T（Task）**：你的任务 - 你的具体职责是什么？
- **A（Action）**：你的行动 - 你具体做了什么？
- **R（Result）**：量化结果 - 取得了什么成果？

### 改写模板

> 「主导/负责 [项目/任务]，通过 [具体方法/工具/策略]，实现 [核心成果]，[提升/降低] XX%/XX元/XX天」

## 三、STAR 改写案例

### 案例1：销售岗位

**原文**：负责公司产品的销售工作，完成销售任务。

**优化后**：**主导**华东区域企业级 SaaS 产品销售，**独立开发**新客户 30+，**推动**季度销售额**提升 25%**，**超额完成**年度指标 120%。

**修改说明**：
- "负责" → "主导"：体现主动性
- 添加量化数据：30+ 客户、25%、120%
- "完成销售任务" → "超额完成 120%"：体现超额贡献

### 案例2：技术岗位

**原文**：参与公司后端系统的开发，使用 Java 和 MySQL。

**优化后**：**主导**公司订单系统后端重构，**采用** Java + MySQL 技术栈，**优化**数据库查询性能，**响应时间降低 60%**，**支撑**日均订单量**从 1 万提升至 5 万**。

**修改说明**：
- "参与" → "主导"：体现主导角色
- "使用" → "采用...技术栈"：更专业
- 添加量化成果：性能提升 60%、订单量 5 倍

### 案例3：运营岗位

**原文**：负责公司公众号运营，写文章，做活动。

**优化后**：**独立运营**公司微信公众号（粉丝 10 万+），**策划并执行**线上活动 20+ 场，**产出**爆款文章 5 篇（阅读量 10 万+），**推动**粉丝**增长 300%**，**转化率提升 15%**。

**修改说明**：
- "负责" → "独立运营"：体现独立能力
- "写文章，做活动" → "策划并执行...产出"：更具体
- 添加量化成果：300% 增长、15% 转化

### 案例4：项目管理岗位

**原文**：负责项目的管理和推进，协调各部门。

**优化后**：**主导**公司级 ERP 系统升级项目（预算 500 万），**协调**研发、财务、供应链等 5 个部门 30+ 人，**制定**项目计划和风险预案，**提前 2 个月**上线，**节省**成本 15%。

**修改说明**：
- "负责" → "主导"：体现领导力
- 添加规模数据：500 万预算、30+ 人
- 添加量化成果：提前 2 个月、节省 15%

### 案例5：应届生/实习经历

**原文**：在公司实习，做一些数据分析的工作。

**优化后**：**实习期间独立完成**用户行为数据分析项目，**使用** Python + SQL **处理** 100 万+ 条数据，**发现**用户流失关键节点，**提出**优化建议 3 条，**被采纳后**用户留存率**提升 8%**。

**修改说明**：
- "做" → "独立完成、处理、发现、提出"：体现能力
- 添加数据规模：100 万+ 条
- 添加量化成果：留存率提升 8%

### 案例6：转行求职者

**原文**：之前做销售，现在想转行做产品经理。

**优化后**：**5 年销售经验**，**深入理解**客户需求和市场趋势，**主导**客户反馈收集系统搭建，**累计收集** 1000+ 条需求，**推动**产品迭代 3 个版本，**客户满意度提升 20%**。**具备**产品经理所需的用户洞察和跨部门协作能力。

**修改说明**：
- "想转行" → 强调可迁移技能：体现匹配度
- 添加具体经历：需求收集、推动迭代
- 补充 PM 能力：用户洞察、跨部门协作

## 四、量化数据来源指南

如何找到可量化的数据？

| 数据类型 | 示例 |
|----------|------|
| 业绩数据 | 销售额、利润率、完成率、增长率 |
| 效率数据 | 时间缩短、成本降低、流程优化 |
| 规模数据 | 客户数量、项目金额、团队人数 |
| 质量数据 | 准确率、满意度、留存率、转化率 |
| 荣誉数据 | 获奖、排名、评级、认可 |

**如果没有精确数据，可以使用**：
- 量级：约 100 万+、数十人
- 比例：提升约 20%、降低近一半
- 排名：部门 Top 3、全公司前 10%
