## Use Case: <br>
Developers and engineers use this skill to drive Playwright browser sessions, inspect page state, debug web workflows, and generate or refine Playwright test code. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Browser automation can act on real sites and accounts. <br>
Mitigation: Use isolated test profiles and test accounts, and review proposed browser actions or run-code before execution. <br>
Risk: Saved state, traces, screenshots, downloads, and videos may contain cookies, tokens, credentials, PII, or network contents. <br>
Mitigation: Avoid sensitive login, payment, and PII workflows when recording or tracing, and protect or delete generated artifacts after use. <br>
Risk: Persistent profiles and saved storage state can preserve reusable authentication material. <br>
Mitigation: Prefer in-memory browser profiles for routine work and delete stale browser data or saved state when it is no longer needed. <br>
