# 胡田-OPC导师-大赛标准化打分.Skill

## 定位与价值

本技能专为商业大赛评委设计，实现从「人工逐份阅读打分」到「AI批量处理+标准化输出」的效率跃升。适用于：
- 创业大赛初筛/复赛评审
- 投融资项目海选
- 商业计划书大赛
- 创业训练营项目选拔

**核心价值**：处理100份商业计划书，从48小时缩短至2小时，保持打分标准统一、输出格式规范。

## 模块一：输入层

### 1.1 支持格式
| 格式 | 解析方式 | 提取内容 |
|------|----------|----------|
| PPT (.pptx/.ppt) | 提取每页文本、标题、备注 | 项目概述、商业模式、财务数据 |
| PDF (.pdf) | 文本提取、表格识别 | 全文结构化解析 |
| Word (.docx/.doc) | 段落/标题层级解析 | 章节内容、图表说明 |

### 1.2 自动识别结构
- **目录识别**：通过标题层级（如H1/H2）重建文档大纲
- **关键章节检测**：自动识别"团队介绍"、"市场分析"、"财务预测"等标准章节
- **信息抽取**：项目名称、团队成员、市场数据、融资需求等关键字段

### 1.3 批量导入
```
支持文件夹批量导入，自动识别所有支持的文档格式
支持按文件名排序，便于对照参赛名单
```

## 模块二：打分维度体系（5维度×20分=100分）

### 2.1 团队维度（20分）

| 评分项 | 分值 | 评估要点 |
|--------|------|----------|
| 创始人背景 | 6分 | 学历背景、行业经验、创业经历、过往成就 |
| 团队完整性 | 5分 | 核心岗位配置（技术/产品/市场/运营是否齐全） |
| 团队互补性 | 5分 | 技能互补、性格互补、资源互补 |
| 股权结构 | 4分 | 股权分配合理性、预留期权池、激励机制 |

**扣分标准**：
- 团队成员少于3人：-2分
- 核心岗位缺失：-3分
- 创始人无相关行业经验：-2分
- 股权过度集中（创始人>80%）：-1分

### 2.2 市场维度（20分）

| 评分项 | 分值 | 评估要点 |
|--------|------|----------|
| 市场规模 | 5分 | TAM/SAM/SOM三层市场分析清晰度 |
| 市场增速 | 4分 | 行业增长率、市场趋势判断 |
| 痛点精准度 | 6分 | 目标用户痛点描述的准确性和紧迫性 |
| 用户清晰度 | 5分 | 目标用户画像清晰度、市场定位明确性 |

**扣分标准**：
- 无具体市场数据支撑：-3分
- 目标用户模糊（泛指"所有人"）：-2分
- 痛点描述脱离实际：-2分

### 2.3 创新维度（20分）

| 评分项 | 分值 | 评估要点 |
|--------|------|----------|
| 技术创新度 | 6分 | 专利情况、技术壁垒、研发能力 |
| 模式创新度 | 5分 | 商业模式创新性、盈利模式可行性 |
| 差异化竞争力 | 5分 | 与竞品的差异化优势、护城河 |
| 竞品对比 | 4分 | 竞品分析完整性、自身优势阐述 |

**扣分标准**：
- 无技术创新点：-3分
- 差异化不明确：-2分
- 竞品分析缺失：-1分

### 2.4 商业可行性（20分）

| 评分项 | 分值 | 评估要点 |
|--------|------|----------|
| 产品成熟度 | 6分 | MVP状态、已有用户、营收情况 |
| 盈利模式 | 5分 | 收入来源清晰度、单位经济模型 |
| 获客路径 | 5分 | 获客渠道、获客成本、转化路径 |
| 现金流规划 | 4分 | 资金使用计划、盈利时间预期 |

**扣分标准**：
- 无产品/服务落地：-4分
- 盈利模式不清晰：-3分
- 获客成本严重偏高：-2分

### 2.5 法务财务（20分）

| 评分项 | 分值 | 评估要点 |
|--------|------|----------|
| 财务真实性 | 6分 | 财务数据合理性、数据来源可靠性 |
| 融资需求 | 5分 | 融资额度合理性、资金用途规划 |
| 估值合理性 | 5分 | 估值与业务阶段匹配度 |
| 法律风险 | 4分 | 股权合规、知识产权、合规资质 |

**扣分标准**：
- 财务数据明显不合理：-4分
- 融资需求与业务不匹配：-2分
- 估值严重偏离市场：-2分

## 模块三：输出层

### 3.1 单项目详细打分报告
每份商业计划书输出独立报告，包含：
- 五大维度得分雷达图
- 每维度详细得分与扣分点分析
- 核心优势提炼
- 改进建议（针对扣分项）
- 评委评语模板

### 3.2 批量项目排名总表
- 总分排名
- 各维度单独排名
- 分位数分布（Top 20%、中位数、尾部）
- 可导出Excel/PDF

### 3.3 量化分析仪表盘
- 各维度平均分/中位数统计
- 高分项目分布热力图
- 短板维度分析（哪些项目普遍弱）
- 异常值预警（得分过高/过低项目标记）

### 3.4 评委评语模板
自动生成结构化评语，支持：
- 一句话项目概括
- 核心亮点（1-3点）
- 主要不足（1-3点）
- 改进建议
- 综合评语

## 模块四：使用方式

### 4.1 批量处理模式
```
1. 将所有商业计划书放入指定文件夹
2. 调用技能，指定文件夹路径
3. 自动解析所有文档并打分
4. 输出：所有报告 + 排名总表 + 分析仪表盘
```

### 4.2 单项目精评模式
```
1. 上传单份商业计划书
2. 指定详细评估要求
3. 输出：详细打分报告 + 评委评语
```

|----------|------|----------|

## 模块七：忽悠识别模块（核心功能）

### 7.1 忽悠识别框架

**目的**：识别商业计划书中的过度承诺、虚假数据和夸大表述，确保评审质量。

### 7.2 夸张程度识别

**识别规则**：

| 夸张类型 | 识别关键词 | 严重程度 |
|----------|------------|----------|
| 绝对化表述 | 第一、唯一、最好、世界级、顶级 | 🟠 中 |
| 过度承诺 | 颠覆、革命、改变世界、重新定义 | 🟠 中 |
| 无依据排名 | 遥遥领先、绝对领先、市场第一 | 🔴 高 |
| 夸大市场 | 万亿市场、无限空间、蓝海市场 | 🟡 低-中 |

**输出示例**：
```
【夸张表述检测】
⚠️ "我们的技术是世界领先的" - 缺乏具体数据支撑
⚠️ "将成为下一个阿里巴巴" - 过度承诺，风险较高
⚠️ "万亿级市场" - 需验证市场数据来源
```

### 7.3 真实性验证

**验证维度**：

| 验证项 | 验证方法 | 异常阈值 |
|--------|----------|----------|
| 市场规模合理性 | 与公开行业报告对比 | 偏差>300% |
| 增长率合理性 | 与行业基准对比 | 远超行业平均5倍 |
| 利润率合理性 | 与行业均值对比 | 远超行业平均3倍 |
| 获客成本合理性 | 与行业基准对比 | 低于行业均值50% |
| 估值合理性 | 与融资阶段/营收对比 | 严重偏离行业基准 |

**行业基准参考**（部分）：

| 行业 | 早期项目合理估值 | 合理毛利率 | 合理CAC回收周期 |
|------|------------------|------------|-----------------|
| SaaS | 100-500万RMB/百万ARR | 60-80% | 12-18个月 |
| 消费 | 500-2000万RMB | 30-50% | 6-12个月 |
| 电商 | 视规模而定 | 10-30% | 3-6个月 |
| 硬件 | 500-3000万RMB | 30-50% | 6-18个月 |

### 7.4 大忽悠预警系统

#### 预警类型一：数据前后矛盾
```
检测场景：
- 市场规模描述前后不一致
- 团队背景描述与时间线矛盾
- 财务数据逻辑不自洽
- 用户数据与描述场景不匹配

示例：
"我们的产品已有10万用户" + "计划从0开始获客" = 矛盾

预警输出：
🚨 [高危] 数据矛盾：声称已有10万用户，但获客计划从0开始，建议核实
```

#### 预警类型二：过度承诺
```
检测场景：
- 时间线明显不现实
- 业绩承诺远超行业水平
- 技术突破承诺无依据

示例清单：
- "6个月后上市"
- "一年内实现100亿收入"
- "3年内成为行业第一"
- "革命性技术将改变整个行业"

预警输出：
🚨 [高危] 过度承诺：计划3年成为行业第一，缺乏可行路径分析
🚨 [中危] 时间线激进：6个月产品上线+盈利，时间节点过于乐观
```

#### 预警类型三：黑科技忽悠
```
检测场景：
- 技术描述模糊但声称领先
- 使用"量子""革命性""颠覆性"等词汇但无具体说明
- 专利/技术无具体信息

示例：
- "采用革命性的量子算法"（无具体技术说明）
- "我们的AI技术领先全球"（无技术细节）
- "拥有多项专利"（未列出具体专利号和内容）

预警输出：
🚨 [高危] 技术描述模糊：声称"革命性技术"但无具体技术路线说明
🚨 [中危] 专利信息缺失：提及"多项专利"但未列出具体专利号
```

#### 预警类型四：团队背景过度包装
```
检测场景：
- 职级夸大
- 经历美化
- 关联关系虚构

示例清单：
- "前阿里高管" → 实际只是P6
- "连续创业者" → 实际只有一次失败经历
- "斯坦福毕业" → 实际只是短期培训
- "10年经验" → 实际工作年限不足

预警输出：
🚨 [高危] 职级夸大风险：声称"前阿里高管"，建议核实实际职级
⚠️ [中危] 经历待验证：提及"连续创业者"，建议核实创业经历
⚠️ [低危] 背景描述模糊：学历背景描述过于简略
```

#### 预警类型五：财务数据造假风险
```
检测场景：
- 毛利率远超行业水平
- 增长率异常
- 成本结构不合理
- 现金流预测过于乐观

示例清单：
- 毛利率95%（行业平均30%）
- 用户增长曲线为完美指数曲线
- 成本预算为0

预警输出：
🚨 [高危] 毛利率异常：声称毛利率95%，远超行业平均水平30%
🚨 [高危] 成本数据缺失：未提供成本结构，无法验证盈利预测
⚠️ [中危] 增长预测激进：用户增长曲线过于平滑，缺乏阶段性数据支撑
```

### 7.5 忽悠指数计算

**计算公式**：
```
忽悠指数 = Σ(各预警项严重程度权重 × 存在该项的bool值)

严重程度权重：
- 🚨 高危 = 3分
- ⚠️ 中危 = 2分
```

| 11-20分 | 🟠 高风险 | 多项疑点，需重点核实 |
| 21+分 | 🔴 极高风险 | 严重存疑，建议一票否决或深入尽调 |

### 10.3 项目发展阶段自动识别

#### 阶段识别标准

| 阶段 | 定义 | 识别指标 |
|------|------|----------|
| **种子期** | 只有idea，无产品无团队 | 无产品描述、无用户数据、无团队介绍 |
| **天使期** | 有团队有原型，无营收 | 有团队、有产品原型、无收入或收入<50万 |
| **早期** | 有产品有少量用户，营收<500万 | 有产品上线、有用户数据、收入<500万 |
| **成长期** | 有稳定营收，用户快速增长 | 收入500万-5000万、有增长数据 |
| **成熟期** | 规模化盈利，营收>5000万 | 规模化收入、稳定盈利 |
| **Pre-IPO** | 准备上市 | 规范化运营、准备IPO |
