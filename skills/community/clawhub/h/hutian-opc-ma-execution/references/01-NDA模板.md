# 保密协议（NDA）模板框架

> ⚠️ **法律声明**：本模板仅供参考，实际使用需根据具体交易情况调整，并经专业律师审核。

---

## 标准保密协议框架

### 协议主体

```
【保密协议】

甲方（信息提供方）：[公司全称]
统一社会信用代码：[代码]
注册地址：[地址]
法定代表人：[姓名]

乙方（信息接收方）：[公司全称]
统一社会信用代码：[代码]
注册地址：[地址]
法定代表人：[姓名]

签订日期：[年]年[月]月[日]
签订地点：[地点]
```

### 核心条款框架

```
第一条 保密信息的定义

1.1 "保密信息"指本协议签署后，一方向另一方提供的、与本协议所述交易相关的所有信息，包括但不限于：

（一）商业信息
├── 公司战略规划、业务计划、市场分析
├── 客户名单、供应商名单、合作伙伴信息
├── 营销策略、销售渠道、定价体系
└── 财务数据、财务预测、预算信息

（二）技术信息
├── 专有技术、工艺流程、技术诀窍
├── 研发成果、专利申请、技术方案
├── 软件源代码、算法、技术文档
└── 产品设计图纸、技术参数、性能指标

（三）经营信息
├── 组织架构、人事信息、薪酬体系
├── 合同协议、商务谈判记录
├── 尽职调查材料、估值分析
└── 其他标注"保密"或"机密"的信息

1.2 例外情形
以下信息不属于保密信息：
（一）接收方能够证明在接收前已合法拥有的信息
（二）非因接收方过错成为公众可获取的信息
（三）接收方从有权披露的第三方获取的信息
（四）接收方独立开发的信息
（五）法律法规要求披露的信息（但应及时通知对方）
```

```
第二条 保密义务

2.1 接收方应采取不低于保护自身保密信息的谨慎程度保护披露方的保密信息，包括：
（一）指定专人负责保密信息的保管
（二）建立保密管理制度
（三）采取物理和技术防护措施

2.2 接收方仅可将保密信息用于本协议所述交易目的，不得用于其他任何目的。

2.3 接收方仅可向以下人员披露保密信息：
（一）需要知悉的员工（应限制在最小必要范围内）
（二）聘请的专业顾问（须签署不低于本协议约束的保密义务）
（三）法律或监管机构要求（须提前通知披露方）

2.4 接收方应对其知悉保密信息的人员的行为承担责任。
```

```
第三条 保密期限

3.1 本协议项下的保密义务自协议签署之日起生效。

3.2 保密信息的保密期限为[5]年，自各该信息首次披露之日起计算。

3.3 对于核心技术信息（包括但不限于技术诀窍、源代码），其保密期限不设上限。

3.4 本协议到期或终止后，接收方应在[30]日内返还或销毁所有保密信息及其载体。
```

```
第四条 陈述与保证

4.1 披露方陈述并保证：
（一）其有权披露保密信息
（二）保密信息不存在侵犯第三方知识产权的情形
（三）保密信息在重大方面真实、准确

4.2 接收方陈述并保证：
（一）将按照本协议约定使用和保护保密信息
（二）不会从事与披露方竞争的业务
（三）不会规避或试图规避本协议条款
```

```
第五条 违约责任

5.1 任一方违反本协议约定的保密义务，应向对方承担违约责任。

5.2 违约方应赔偿对方因此遭受的直接损失和合理维权费用。

5.3 鉴于保密信息的特殊价值，双方同意：
（一）若难以证明实际损失，可参照以下方式计算违约金：
   - 首次泄露：不低于[50]万元
   - 恶意泄露：不低于[100]万元或泄露信息价值的1.5倍（以较高者为准）
（二）违约金不影响受损方主张其他救济措施的权利
```

```
第六条 争议解决

6.1 本协议适用中华人民共和国法律。

6.2 因本协议引起的或与本协议有关的任何争议，双方应友好协商解决。

6.3 协商不成的，任一方可向[被告住所地/合同签订地]有管辖权的人民法院提起诉讼。

6.4 在争议解决期间，双方应继续履行本协议项下的保密义务。
```

```
第七条 其他

7.1 本协议自双方签署之日起生效，至保密期限届满或双方另行约定终止。

7.2 本协议一式两份，双方各执一份，具有同等法律效力。

7.3 本协议的任何修改或补充须经双方书面同意。

7.4 未经对方事先书面同意，任何一方不得转让本协议项下的权利或义务。
```

## OPC特别注意事项

### 技术方保护要点

```
【核心资产保护】

1. 技术参数分级披露
   - 建议采用"核心参数+非核心参数"分级方式
   - 核心参数仅在NDA签署后、有明确意向时披露

2. 限制竞争对手接触
   - 在NDA中明确约定：竞争对手及其关联方不得接触保密信息
   - 定义"竞争对手"范围应具体明确

3. 技术使用限制
   - 明确约定保密信息不得用于产品开发、生产或销售
   - 仅允许用于评估交易目的

4. 人员限制
   - 限制访问人员数量
   - 参与评估的人员应签署个人保密承诺
```

### 常见陷阱与规避

| 陷阱类型 | 风险描述 | 规避建议 |
|---------|---------|---------|
| 范围过窄 | 重要信息未纳入保密范围 | 扩大定义，采用"包括但不限于+概括"方式 |
| 期限过短 | 技术信息保护不足 | 核心技术建议5年以上或无期限 |
| 违约成本低 | 对方违约代价小 | 设置最低违约金门槛 |
| 无竞业限制 | 对方可用于竞争 | 增加"反向约定"条款 |
| 口头信息无保护 | 口头披露不受约束 | 明确约定口头信息同样受保护 |
