# li-bid-document-maker 使用说明

> **标书制作专家** — 将招标文件自动转化为评分导向的专业应标书

## 安装

```bash
clawhub install li-bid-document-maker
```

## 快速开始

安装后，直接向 AI 助手说出以下任意一句话即可触发：

- "帮我做标书"
- "写应标书"
- "生成投标文件"
- "分析这份招标文件并生成应标书"

AI 会自动执行 6 阶段工作流：**解析招标文件 → 策略分析 → 大纲生成 → 分章写作 → 质量检查 → PDCA 自动改进**

## 使用方式

### 方式一：上传文件

将招标文件（PDF 或 Word 格式）直接拖入对话框，然后说：
> "使用 li-bid-document-maker 分析这份文件"

### 方式二：指定文件路径

> "使用 li-bid-document-maker，招标文件在 C:/项目/招标文件.pdf"

## 系统要求

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10/11, macOS, Ubuntu 20.04+ |
| AI 平台 | Claude, OpenClaw, Hermes, 或任何支持文件读写的 LLM Agent |
| 文件格式 | PDF（推荐可检索版）或 Word (.docx) |
| 依赖 | `python-docx`、`PyPDF2`（用于文件处理，可选） |

## 输出

- 完整应标书（16 章标准结构）
- PDCA 质量改进报告
- 标注评分项对应分值

## 工作流程

```
阶段1: 解析招标文件     → 提取项目信息、技术要求、评分标准
阶段2: 策略分析         → 评分权重分析、竞争策略
阶段3: 大纲生成         → 评分导向大纲（需用户确认）
阶段4: 分章写作         → 逐章编写内容
阶段5: 质量检查         → 6维度全覆盖检查
阶段6: PDCA自动改进     → 3轮改进后交付（无需用户介入）
```
