# Multi-Agent 结果聚合策略（v4.0 新增）

> 用于 Multi-Agent Debate 后的共识聚合阶段

---

## 聚合模式

### 1. 共识聚合（默认）

当多个领域 Agent 完成分析后，Chief 按以下规则聚合：

| 类型 | 定义 | 处理策略 | 输出置信度 |
|------|------|---------|-----------|
| **共识点** | 所有 Agent 都同意的发现 | 直接采纳 | 高 |
| **分歧点** | Agent 间有争议的发现 | 标注各方理由，不强行统一 | 中 |
| **盲点** | 无 Agent 覆盖的维度 | 标注"分析不完整" | 低 |

### 2. 投票聚合（适用于量化指标）

当多个 Agent 对同一指标给出不同数值时：
- 取中位数（非平均值，避免极端值影响）
- 标注范围（min-max）
- 标注偏离中位数最远的 Agent 及理由

### 3. 加权聚合（适用于不同 Agent 可信度不同）

| 条件 | 权重 |
|------|------|
| Agent 使用了真实数据 | 1.0x |
| Agent 仅依赖推理 | 0.7x |
| Agent 标注了"数据不足" | 0.5x |

---

## 辩论规则

### 交叉辩论轮（1 轮）

每个领域 Agent 对其他 Agent 的结论提出 **1 条质疑**：

格式：
```
[Agent A] 质疑 [Agent B]：
- 质疑点：[具体哪条结论有问题]
- 理由：[为什么认为有问题]
- 建议：[应该如何修正]
```

### Chief 记录模板

```
辩论记录：
  共识点：
    1. [发现] — 所有 Agent 同意
  分歧点：
    1. [发现] — Agent A 认为 X，Agent B 认为 Y
       - Agent A 理由：[...]
       - Agent B 理由：[...]
  盲点：
    1. [维度] — 无 Agent 覆盖
```

---

## 失败隔离

- 单个 Agent 超时 → 降级为该领域"数据不足，建议补充"
- 不影响其他 Agent 的输出
- Chief 综合时标注"XX 领域分析不完整"
- 重试规则：失败 Agent 自动重试 1 次，仍然失败则降级
