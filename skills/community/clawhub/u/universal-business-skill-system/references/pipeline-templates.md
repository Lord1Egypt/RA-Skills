# 通用Pipeline模板

以下为跨行业通用的业务场景Pipeline。每个Pipeline由能力单元按依赖关系串联，可根据行业校准自动适配具体内容。

---

## 一、新品/新服务上市全流程

**适用场景**：推出新产品或新服务的完整上市过程

C1-01(市场数据收集) → C4-05(ABP撰写) → C7-12(客户分级) → C2-01(核心信息架构) → C2-03(内容故事线) → C3-02(活动方案设计) → C6-02(推广材料审核) → C6-03(审核SOP) → C3-14(活动执行) → C4-29(效果数据收集) → C3-21(效果评估)

---

## 二、竞品应对战

**适用场景**：竞品发起攻击或市场出现竞争压力

C1-01(市场数据) → C1-07(竞品监测) → C1-12(态势分析) → C4-06(SWOT) → C1-13(策略矩阵话术) → C2-13(内容转化) → C5-12(话术演练) → C8-31(跨部门协同)

---

**适用场景**：制定年度业务规划和预算

---

## Pipeline使用规则

1. **按需调整**：Pipeline是模板，可根据实际情况增删单元
2. **并行执行**：无依赖关系的单元可并行（如C1-01和C1-04可同时执行）
3. **行业适配**：每个单元的具体内容由行业校准决定
4. **循环迭代**：Pipeline可循环执行（如"预算管控闭环"按月循环）
5. **跨Pipeline组合**：不同Pipeline的单元可交叉组合
