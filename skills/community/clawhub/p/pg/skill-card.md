## Known Risks and Mitigations: <br>
Risk: Generated or suggested SQL can change database state, lock tables, drop indexes, affect vacuum behavior, or alter timeout and schema settings. <br>
Mitigation: Verify the target database and review database-changing SQL before execution; test risky commands in a non-production environment when possible. <br>
