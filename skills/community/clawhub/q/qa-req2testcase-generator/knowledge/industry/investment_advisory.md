# 投顾域（Investment Advisory Domain）

> 本文档覆盖券商投资顾问核心业务对象的状态机、操作规则、异常模式与测试点，涵盖投顾服务、投顾产品、签约管理等子域。

---

## 投顾服务

### 业务背景
投资顾问服务是券商向客户提供的证券投资建议服务，包括投资组合建议、调仓建议、市场分析等。投顾业务需持有证券投资顾问资格，并遵守相关法规要求。2022年起，基金投顾试点逐步推广，券商可为客户提供"买方投顾"服务。

### 投顾资质管理

#### 投顾人员资质
- 从业资格：持有证券投资顾问执业资格证书
- 注册要求：在中国证券业协会注册登记
- 继续教育：每年完成规定学时的继续教育
- 执业限制：不得同时从事自营交易、承销保荐等业务

#### 服务分级
| 服务等级 | 服务内容 | 适用客户 | 费用模式 |
|---------|---------|---------|---------|
| 基础服务 | 市场资讯、研报推送 | 所有客户 | 免费 |
| 标准服务 | 投资建议、组合推荐 | 签约客户 | 按资产规模收费 |
| 尊享服务 | 专属投顾、定制策略 | 高净值客户 | 按资产规模+业绩报酬 |
| 基金投顾 | 全权委托投资决策 | 签约客户 | 管理费+业绩报酬 |

#### 投资建议管理
- 建议类型：买入/卖出/持有/调仓
- 建议要素：标的、方向、目标价、止损价、持有期限、理由
- 合规审查：投资建议发布前需经合规审核
- 利益冲突：投顾人员不得与客户进行反向交易
- 留痕要求：所有投资建议需记录并保存不少于5年

### 投顾服务状态全集
- 待分配（PendingAssign）：客户签约后等待分配投顾
- 服务中（InService）：投顾正常为客户提供服务
- 暂停服务（Suspended）：因投顾离职/客户投诉等暂停服务
- 投顾变更中（Transferring）：正在进行投顾变更
- 服务到期（Expired）：服务期限到期（终态，可续约）
- 已解约（Terminated）：客户主动解约或违约解约（终态）

### 状态转移规则
| 当前状态 | 触发条件 | 目标状态 | 是否允许 |
|---------|---------|---------|--------|
| 待分配 | 系统/人工分配投顾 | 服务中 | ✅ |
| 服务中 | 投顾离职/客户申请变更 | 投顾变更中 | ✅ |
| 服务中 | 合规要求暂停 | 暂停服务 | ✅ |
| 服务中 | 服务期限到期 | 服务到期 | ✅ |
| 服务中 | 客户申请解约 | 已解约 | ✅ |
| 投顾变更中 | 新投顾确认接手 | 服务中 | ✅ |
| 暂停服务 | 问题解决恢复 | 服务中 | ✅ |
| 暂停服务 | 无法恢复 | 已解约 | ✅ |
| 服务到期 | 客户续约 | 服务中 | ✅ |
| 服务到期 | 未续约 | 已解约 | ✅ |
| 已解约 | 重新签约 | — | ❌（需新建服务合同） |

### 异常模式
- 投顾离职未交接：投顾离职后客户服务中断 → 测试点：验证投顾离职触发的自动交接流程和客户通知
- 投资建议合规违规：投顾发布未经审核的投资建议 → 测试点：验证投资建议发布的合规审核前置校验
- 利益冲突：投顾持有与建议方向相反的头寸 → 测试点：验证投顾个人交易与投资建议的冲突检测
- 服务质量投诉：客户投诉投顾服务质量 → 测试点：验证投诉处理流程和服务暂停机制

### 测试点模板
- TP-IA-001：投顾人员注册，验证执业资格证书有效性校验（优先级：P0）
- TP-IA-002：投资建议发布，验证合规审核前置校验（优先级：P0）
- TP-IA-003：投顾离职，验证名下客户自动触发变更流程（优先级：P0）
- TP-IA-004：投顾个人交易申报，验证与投资建议的利益冲突检测（优先级：P0）
- TP-IA-005：服务到期前30天，验证系统发送续约提醒（优先级：P1）
- TP-IA-006：客户投诉处理，验证投诉升级和服务暂停联动（优先级：P1）

## 投顾产品（组合策略）

### 业务背景
投顾产品是投顾团队构建的模型组合，客户签约后可跟随组合进行投资。基金投顾模式下，客户授权投顾代为做出投资决策并执行交易。

### 产品组合管理

#### 组合类型
- 股票型组合：以股票为主要投资标的
- 混合型组合：股票+债券+基金混合配置
- 基金组合（FOF）：以公募基金为投资标的
- 固收+组合：固定收益为底仓+权益增强

#### 组合要素
- 组合名称：简洁明了，体现投资策略
- 投资策略：价值/成长/均衡/行业主题等
- 风险等级：R1-R5，与客户适当性匹配
- 业绩基准：沪深300/中证500/中债综合等
- 持仓上限：单只标的持仓比例上限
- 最低跟投金额：客户跟投的最低资金要求

#### 组合状态全集
- 设计中（Designing）：组合正在设计和回测
- 审核中（Reviewing）：组合方案提交审核
- 已发布（Published）：组合已发布，可供客户签约跟投
- 运作中（Running）：组合正常运作，有客户跟投
- 暂停跟投（SuspendedNew）：暂停新客户跟投（存量客户不受影响）
- 清盘中（Winding）：组合正在清盘
- 已清盘（Closed）：组合已清盘（终态）

### 调仓策略

#### 调仓类型
- 定期调仓：按固定周期（周/月/季）调整持仓
- 事件驱动调仓：因市场事件或基本面变化触发调仓
- 止盈调仓：标的达到目标价触发卖出
- 止损调仓：标的跌破止损价触发卖出
- 再平衡调仓：持仓偏离目标比例超过阈值触发再平衡

#### 调仓执行流程
1. 投顾发起调仓指令（标的、方向、目标比例）
2. 合规审核（是否符合组合投资限制）
3. 调仓指令下发至跟投客户
4. 客户端执行交易（全权委托模式自动执行/建议模式客户确认）
5. 调仓结果汇总和偏差分析

#### 止盈止损规则
- 个股止盈：浮盈达到+20%触发止盈建议
- 个股止损：浮亏达到-10%触发止损建议
- 组合止损：组合净值回撤-8%触发全面止损
- 止损执行：全权委托模式自动执行，建议模式发送通知

### 异常模式
- 调仓执行偏差：客户资金不足导致无法完全跟随调仓 → 测试点：验证资金不足时的部分执行逻辑和偏差记录
- 调仓时标的停牌：调仓标的停牌无法交易 → 测试点：验证停牌标的的调仓延迟处理
- 大量客户同时调仓：调仓指令下发导致市场冲击 → 测试点：验证调仓的分批执行和市场冲击控制
- 止损与调仓冲突：止损触发时恰好有调仓指令 → 测试点：验证止损优先级高于普通调仓

### 测试点模板
- TP-IA-010：组合发布，验证必须通过回测验证和合规审核（优先级：P0）
- TP-IA-011：调仓指令下发，验证全权委托客户自动执行交易（优先级：P0）
- TP-IA-012：客户资金不足无法完全调仓，验证部分执行和偏差记录（优先级：P0）
- TP-IA-013：个股浮亏达-10%，验证止损建议/自动执行触发（优先级：P0）
- TP-IA-014：调仓标的停牌，验证延迟调仓队列和复牌后自动执行（优先级：P1）
- TP-IA-015：组合净值回撤-8%，验证全面止损触发和客户通知（优先级：P1）

## 签约管理

### 签约流程

#### 签约前置条件
- 客户适当性：客户风险等级≥组合风险等级
- 风险测评有效：客户风险测评在有效期内
- 最低资金：客户可用资金≥组合最低跟投金额
- 协议签署：签署投顾服务协议+风险揭示书

#### 签约状态全集
- 申请中（Applying）：客户提交签约申请
- 适当性校验中（Checking）：系统校验客户适当性
- 协议签署中（Signing）：客户阅读并签署协议
- 冷静期（CoolingOff）：签约后24小时冷静期
- 已生效（Active）：签约生效，开始跟投
- 暂停中（Paused）：客户主动暂停跟投
- 解约申请中（Terminating）：客户申请解约
- 已解约（Terminated）：解约完成（终态）

#### 状态转移规则
| 当前状态 | 触发条件 | 目标状态 | 是否允许 |
|---------|---------|---------|--------|
| 申请中 | 系统受理 | 适当性校验中 | ✅ |
| 适当性校验中 | 校验通过 | 协议签署中 | ✅ |
| 适当性校验中 | 校验不通过 | — | ✅（终态：已拒绝） |
| 协议签署中 | 签署完成 | 冷静期 | ✅ |
| 协议签署中 | 客户放弃 | — | ✅（终态：已取消） |
| 冷静期 | 24小时期满+回访确认 | 已生效 | ✅ |
| 冷静期 | 客户在冷静期内撤回 | — | ✅（终态：已撤回） |
| 已生效 | 客户暂停 | 暂停中 | ✅ |
| 已生效 | 客户申请解约 | 解约申请中 | ✅ |
| 暂停中 | 客户恢复 | 已生效 | ✅ |
| 暂停中 | 暂停超过90天 | 解约申请中 | ✅ |
| 解约申请中 | 持仓清算完成 | 已解约 | ✅ |
| 已解约 | 重新签约 | — | ❌（需新建签约） |

### 服务期限管理
- 合同期限：通常1年，到期可续约
- 自动续约：合同约定自动续约条款（客户可选择关闭）
- 到期提醒：到期前30天/7天/1天分别提醒
- 到期处理：到期未续约则进入解约流程

### 费用管理

#### 费用类型
- 管理费：按客户跟投资产规模的年化比例收取（通常0.5%-1.5%）
- 业绩报酬：超过业绩基准部分按比例提取（通常10%-20%）
- 服务费：固定金额的服务费（部分产品）

#### 费用计算规则
- 管理费计提：按日计提，日管理费 = 前一日跟投资产 × 年管理费率 / 365
- 业绩报酬计提：按季度/年度计提，超额收益 × 业绩报酬比例
- 高水位法：业绩报酬仅在净值创新高时计提
- 费用扣收：从客户跟投账户中自动扣收

#### 费用状态
- 待计提（PendingAccrual）：费用待计算
- 已计提（Accrued）：费用已计算入账
- 待扣收（PendingDeduct）：费用待从客户账户扣收
- 已扣收（Deducted）：费用已成功扣收（终态）
- 扣收失败（DeductFailed）：客户账户余额不足扣收失败

### 解约流程
1. 客户提交解约申请
2. 系统暂停新的调仓指令
3. 持仓清算（卖出跟投持仓）
4. 费用结算（计算并扣收剩余费用）
5. 资金返还（清算资金返还客户普通账户）
6. 解约确认（解约完成通知客户）

### 异常模式
- 冷静期内市场大跌：冷静期内组合大幅下跌，客户撤回签约 → 测试点：验证冷静期撤回的资金返还（是否承担冷静期内的损益）
- 费用扣收失败：客户可用资金不足以扣收管理费 → 测试点：验证扣收失败后的重试机制和服务暂停联动
- 解约时持仓无法卖出：解约清算时标的停牌 → 测试点：验证停牌标的的解约延迟处理
- 自动续约争议：客户未注意自动续约条款导致争议 → 测试点：验证自动续约前的明确通知和确认机制

### 测试点模板
- TP-IA-020：签约流程，验证适当性校验→协议签署→冷静期→生效的完整链路（优先级：P0）
- TP-IA-021：冷静期内客户撤回，验证签约取消和资金处理（优先级：P0）
- TP-IA-022：冷静期24小时期满，验证回访确认后签约生效（优先级：P0）
- TP-IA-023：管理费扣收，验证日管理费=前一日资产×年费率/365（优先级：P0）
- TP-IA-024：业绩报酬计提（高水位法），验证仅在净值创新高时计提（优先级：P0）
- TP-IA-025：解约清算，验证持仓卖出→费用结算→资金返还的完整流程（优先级：P0）
- TP-IA-026：费用扣收失败，验证3次重试后暂停服务（优先级：P1）
- TP-IA-027：合同到期前30天，验证续约提醒通知（优先级：P1）
- TP-IA-028：暂停超过90天，验证系统自动触发解约流程（优先级：P1）

## 合规要求

### 投资顾问法规核心要求
1. **持证上岗**：投顾人员必须持有证券投资顾问执业资格
2. **了解客户**：充分了解客户的财务状况、投资经验、风险偏好
3. **适当性匹配**：投资建议/组合风险等级与客户风险承受能力匹配
4. **风险揭示**：充分揭示投资风险，不得承诺收益
5. **留痕管理**：投资建议、服务记录保存不少于5年
6. **禁止代客理财**：传统投顾模式下不得代客户做出投资决策（基金投顾除外）

### 利益冲突管理
1. **个人交易申报**：投顾人员个人证券交易需提前申报
2. **反向交易禁止**：不得与投资建议方向相反交易
3. **利益输送禁止**：不得利用投资建议为自己或关联方谋利
4. **公平对待**：同一投资建议需同时向所有签约客户发布
5. **信息隔离**：投顾与自营、投行之间的信息隔离

### 信息披露
1. **组合业绩展示**：真实、准确展示历史业绩，不得选择性展示
2. **费用透明**：明确告知所有费用项目和计算方式
3. **风险提示**：历史业绩不代表未来表现的风险提示
4. **投顾信息**：公开投顾人员的资质、从业经历、管理规模

## 常见缺陷模式

1. **适当性绕过**：高风险组合向低风险客户展示并允许签约
2. **冷静期缺失**：签约后未执行24小时冷静期直接生效
3. **调仓通知延迟**：调仓指令下发后客户端通知延迟导致执行偏差
4. **费用计算错误**：管理费计提天数错误（节假日处理）或业绩报酬高水位计算错误
5. **解约清算不完整**：解约时未结算剩余管理费或业绩报酬
6. **投资建议未审核**：投顾直接发布未经合规审核的投资建议
7. **利益冲突未检测**：投顾个人交易与投资建议方向相反未被发现
8. **自动续约未通知**：合同自动续约前未明确通知客户

## 边界值与特殊场景

| 场景 | 边界条件 | 预期行为 |
|------|---------|---------|
| 冷静期 | 恰好24小时 | 冷静期结束，可回访确认 |
| 冷静期 | 23小时59分 | 仍在冷静期，不可生效 |
| 最低跟投金额 | 恰好等于最低限额 | 允许签约 |
| 最低跟投金额 | 低于最低限额0.01元 | 拒绝签约 |
| 止盈触发 | 浮盈恰好+20% | 触发止盈建议 |
| 止损触发 | 浮亏恰好-10% | 触发止损建议 |
| 管理费计提 | 闰年2月29日 | 日费率=年费率/366 |
| 暂停时长 | 恰好90天 | 触发自动解约 |
| 暂停时长 | 89天 | 不触发 |
| 到期提醒 | 到期前恰好30天 | 发送第一次提醒 |
| 费用扣收重试 | 第3次失败 | 暂停服务 |
| 业绩报酬 | 净值恰好等于高水位 | 不计提（需超过） |
| 业绩报酬 | 净值超过高水位0.0001 | 计提超额部分 |
