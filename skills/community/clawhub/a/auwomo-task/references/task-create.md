# task-create

在飞书任务中创建新的结构任务（主线或子线）。

## 场景

- 当前没有可承接工作的主线
- 用户明确要求新开主线或子线
- `check init` 返回 `missing`，需要初始化

## 两种交互模式

### 模式 A：用户提供了具体内容

触发："帮我建一个主线：论文写作" / "创建子线：数据收集，挂在 xxx 下面"

```
1. 根据用户描述拟定标题和描述
2. 展示草案："准备创建 [主线] 论文写作，确认吗？"
3. 确认后 → 创建（必须包含 description）
```

### 模式 B：用户未明确说要创建什么

触发："帮我建个任务" / check init 显示 missing / 从 record 流程改道过来

```
1. 先确认：这更像"会持续推进一段时间的长期任务"还是"一次性事项"？
   - 一次性事项 → 改走 task-record 流程
   - 长期任务 → 继续
2. 询问任务名称、大致内容
3. 可选：询问是否有相关文档/材料可参考
4. 用 context 检查是否已有类似主线（避免重复）
5. 拟定草案（含标题 + 描述）→ 确认 → 创建
```

## 标题前缀

- 新建顶层主线：`[主线] 论文写作`
- 新建结构子任务：`[子线] 数据收集`
- 不要在此流程中创建 `[记录]`（记录由 `task-record` 负责）

## 描述（description）要求

创建主线或子线时**必须**提供 `--desc`。没有描述的任务无法挂载记录。

### 主线描述（2-5 句）

说明：目标是什么、范围包括哪些、大致验收标准。

示例：
```
研究和评估 Claude API 在团队中的使用方案，包括额度管理、接入方式、成本优化。目标是形成一套可落地的使用规范，让团队成员能高效使用 AI 工具。
```

### 子线描述（1-3 句）

说明：聚焦点、预期产出。

示例：
```
调研 Claude 各套餐的额度和限制，对比团队实际用量，输出选型建议。
```

### 描述不需要

- 不要堆砌细节（不是文档）
- 不要写执行步骤（那是记录的事）
- 不要重复标题信息

## 操作命令

### 创建主线

```bash
auwomo task create --title "[主线] 论文写作" --desc "完成本学期毕业论文，包括选题、文献调研、实验设计、撰写和答辩准备。目标是6月前提交终稿。"
```

### 创建子线（挂在已有任务下）

```bash
auwomo task create --title "[子线] 数据收集" --parent <parent_guid> --desc "收集实验所需的训练数据集，包括公开数据集筛选和自有数据标注。"
```

CLI 自动行为：
- 无 parent 且无前缀 → 自动加 `[主线]`
- 有 parent 且无前缀 → 自动加 `[子线]`
- 以 bot 身份创建
- 默认 status = todo

## 创建原则

- 结构任务应由 app/bot 身份创建（确保后续 taskctx 可识别和挂接）
- 创建前建议先看上下文：`auwomo task context --duration 7d`
- 避免重复创建已存在的主线
- **必须包含描述** — 无描述的任务会被系统标记为不可挂载

## 与 context 的配合

创建前建议先运行：

```bash
auwomo task context --duration 7d
```

确认是否真的缺少结构，避免重复创建。

## 补充已有任务的描述

如果发现已有主线/子线缺少描述（check attachable 显示 `missing_description`），可用 update 补充：

```bash
auwomo task update <guid> --desc "补充的描述内容"
```

## 不要这样做

- 不要在未确认的情况下直接创建（必须与用户确认草案）
- 不要把一次性事项创建为主线（应走 record 流程）
- 不要手动管理标题前缀（CLI 自动处理）
- 不要忽略已有主线直接创建新的（先检查 context）
- 不要创建没有描述的主线或子线
