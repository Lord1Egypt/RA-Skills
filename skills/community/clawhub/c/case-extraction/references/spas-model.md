# SPAS 模型 — 案例萃取方法论详解

## 核心概念

**SPAS = Story → Picture → Audit → Structure**

来自《最佳实践萃取》（孙波，江苏人民出版社）

---

## 核心理念

SPAS模型的核心价值在于将原本依赖个人直觉与零星技巧的经验萃取，转变为具备**标准化流程**、**科学化工具**与**严谨逻辑**的系统性学科。

四阶段构成从感性素材到理性成果的递进转化链条：
```
隐性经验 → 情境还原 → 框架提取 → 深度挖掘 → 成果封装 → 显性知识产品
```

---

## S — 典型故事 (Story)

**目标**：找到符合场景的典型挑战性案例，获取真实、具挑战性且细节丰富的原始素材

### 故事三要素

| 要素 | 判断标准 | 引导问题 |
|------|---------|---------|
| **符合场景** | 和萃取的的业务目标相关 | "这个案例是在什么场景下发生的？" |
| **具有典型性** | 经常出现、有代表性 | "这种情况常见吗？" |
| **具有挑战性** | 过程有波折、最终成功 | "过程中遇到了什么困难？" |

### 故事曲线（叙事结构）

```
平静生活 → 偶遇挑战 → 踏上征途 → 化茧成蝶
```

引导用户按这个曲线讲述：
1. 刚开始是什么状态？
2. 遇到了什么挑战/问题？
3. 怎么应对的？
4. 最终结果如何？

### 工具：故事曲线

| 阶段 | 引导问题 |
|------|---------|
| 平静生活 | "刚开始是什么状态？" |
| 偶遇挑战 | "遇到了什么困难/挑战？" |
| 踏上征途 | "你怎么应对的？" |
| 化茧成蝶 | "最终结果如何？" |

### 引导话术

- "能分享一个具体的故事吗？从头到尾说一遍。"
- "当时是什么情况？"
- "你当时怎么处理的？"
- "有没有遇到什么困难？"
- "最后是怎么解决的？"

---

## P — 整体图片 (Picture)

**目标**：梳理专家解决问题的思维框架（≤7个步骤），形成可复用的解决方案框架

### 框架梳理要点

1. **不只是动作（Manual Task）**，还要分析**认知步骤（Cognitive Task）**
2. 步骤不超过 **7个**，颗粒度均衡
3. 可借助成熟模型/框架
4. 用图形/图表让信息更清晰

### 工具：工作任务分析

将故事分解为一级步骤时，兼顾两个维度：

| 维度 | 说明 | 示例 |
|------|------|------|
| **外在动作** | 做什么 | "询问客户现状" |
| **内在认知** | 怎么想、为什么 | "判断客户的真实需求" |

### 引导话术

- "回过头看，你觉得做这件事的关键步骤是什么？"
- "别人想学的话，你会先教什么？"
- "有没有什么判断标准或原则？"
- "你是怎么想到这个方法的？"
- "这个思路可以用在哪些场景？"
- "当时你是怎么判断/决策的？"

---

## A — 知识审查 (Audit)

**目标**：通过系统挖掘，实现从"做什么"到"如何做、为何如此做"的深度显性化

### 工具：结构化提问

| 问题类型 | 目的 | 示例 |
|----------|------|------|
| **判断性问题** | 判断情况、做决策 | "这种情况应该用A还是B？" |
| **审查性问题** | 审查操作诀窍、避坑 | "新手最容易在哪里出错？" |

### 审查清单

- [ ] 信息呈现有逻辑
- [ ] 步骤之间关系清晰
- [ ] 不超过7个步骤
- [ ] 每个步骤颗粒度均衡
- [ ] 有具体的判断标准或技巧
- [ ] 排除了"肢体相关"的内容（不适合萃取）

### 引导话术

**判断性问题：**
- "你觉得做得好和做得差的人，区别在哪里？"
- "这个方法有没有局限性？"
- "在什么情况下这种方法不适用？"

**审查性问题：**
- "你觉得新手最容易在哪个环节出错？"
- "有没有什么坑要提醒的？"
- "有什么注意事项？"

---

## S — 结构表达 (Structure)

**目标**：对前述成果进行逻辑重组、语言精炼与形式封装

### 工具：逻辑重组与格式化

产出可直接应用于工作指导或学习的标准化知识产品：

| 产出形式 | 适用场景 |
|---------|---------|
| 操作手册 | 需要标准化执行的场景 |
| 决策流程图 | 需要清晰决策路径的场景 |
| 培训课件 | 需要系统学习的场景 |
| 话术脚本 | 需要标准化沟通的场景 |
| 标杆案例 | 需要参考借鉴的场景 |

### 引导话术

- "这个案例是给谁看的？"
- "他们看完后需要能做什么？"
- "希望重点突出什么？"
- "有没有特别需要注意的地方？"
- "他们看完后需要达成什么目标？"

---

## 快速引导流程（完整版）

```
1. "请分享一个代表性的成功案例，从头到尾说一遍。"
   ↓
2. "过程中最大的挑战是什么？你怎么解决的？"
   ↓
3. "如果要让别人也能做到，你觉得哪几步最关键？"
   ↓
4. "新手最容易犯什么错？有什么坑要避开？"
   ↓
5. "你做了什么特别的事，让你赢而不是别人？"
   ↓
6. "这份案例是给谁看的？他们看完需要达成什么？"
```

---

## 不宜萃取的内容（经验法则）

- **肢体相关**：唱歌、踢球、游泳等身体技能
- **深度内隐**：创意、幽默、领导力直觉等
- **纯主观**：无法结构化描述的感受类经验

---

## 成功应用的关键条件

1. **明确的业务痛点与高层支持**
2. **合格的萃取师队伍**（需兼具深度访谈引导、逻辑结构化与业务理解能力）
3. **有效的专家激励机制**
4. **成果与系统的有机整合**
