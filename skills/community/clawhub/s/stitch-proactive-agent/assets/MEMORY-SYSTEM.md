# Proactive Agent Memory System

## Directory Structure
```
memory/
├── .abstract                    # Memory system overview
├── SESSION-STATE.md            # Current session state
├── insights/                   # User insights and patterns
│   ├── .abstract
│   ├── preferences.md         # User preferences
│   └── patterns.md           # Behavioral patterns
├── lessons/                   # Lessons learned
│   ├── .abstract
│   ├── successful.md         # What worked well
│   └── improvements.md      # What to improve
└── archive/                  # Archived memories
    └── .abstract
```

## File Purposes

### SESSION-STATE.md
Tracks the current session state:
- Active projects/tasks
- Recent conversations
- Current context
- Pending actions

### insights/preferences.md
Records user preferences:
- Communication style
- Preferred tools/methods
- Topics of interest
- Time preferences

### insights/patterns.md
Identifies behavioral patterns:
- Common workflows
- Recurring needs
- Problem-solving approaches
- Decision-making patterns

### lessons/successful.md
Documents successful proactive interventions:
- What worked well
- Positive user responses
- Effective timing/methods

### lessons/improvements.md
Records areas for improvement:
- Unsuccessful interventions
- User feedback
- Missed opportunities

## Usage Guidelines
1. **Regular Updates**: Update memory files after significant interactions
2. **Pattern Recognition**: Look for patterns across multiple sessions
3. **Continuous Learning**: Incorporate lessons into future behavior
4. **Respect Privacy**: Only store information relevant to proactive assistance
