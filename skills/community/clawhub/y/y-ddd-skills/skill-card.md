## Use Case: <br>
Developers and engineers use this skill to choose and apply DDD guidance across strategic modeling, layered architecture, tactical design, new project setup, legacy refactoring, and DDD-focused code review. It is especially oriented toward Java/Kotlin projects using Spring/JPA examples. <br>

### Deployment Geography for Use: <br>

## Known Risks and Mitigations: <br>
Risk: The skill may steer broad architecture or refactoring conversations toward DDD patterns. <br>
Mitigation: Review recommendations against the actual business domain, team constraints, and system complexity before adopting DDD patterns. <br>
Risk: Generated code, dependency examples, or configuration snippets may not fit the user's language, framework version, or runtime constraints. <br>
Mitigation: Review and test all generated code, Gradle dependencies, and Spring/JPA examples before applying them to a project. <br>
