# 深知政务智查

该 skill 用于政务问答场景。它不直接让接口生成最终答案，而是先调用深知可信搜索接口召回权威材料，再由 Agent 基于材料生成面向用户的政务回复。

## 使用边界

适合处理：

- 政务办事、公共服务、社保、公积金、证照、补贴等咨询。
- 政策适用、办理条件、办理流程、材料清单、办理入口、官方依据等问题。
- 需要引用政府网站、政策文件、公共事项或问答知识进行可溯源回答的问题。

不适合处理：

- 与政务、政策、公共服务无关的闲聊或创作任务。
- 没有材料支撑却要求给出确定法律结论的高风险判断。

## 配置

配置文件位于 skill 目录下的 `config.ini`：

- `endpoint`: 深知可信搜索接口地址，默认 `https://open.dknowc.cn/dependable/search`。
- `api_key`: 深知开放平台 API Key。使用前请访问 `https://platform.dknowc.cn` 注册/登录并申请 API Key，然后填入 `config.ini`。
- `area`: 默认地域。该 skill 面向深圳用户，默认 `深圳市`。
- `eff_time`: 生效时间过滤，支持 `2026年`、`2026年06月`、`2026年06月17日` 等格式，留空表示不限制。
- `policy`: 是否返回规范性政策文件清单。
- `item`: 是否返回公共事项和在线办理入口。
- `return_full_content`: 是否返回全文。默认关闭，避免材料过长。
- `segment_count`: 每篇材料返回的相关段落数量。
- `simplified`: 是否过滤低相关、过旧材料。
- `search_type`: 检索类型，可包含 `policy`、`affair`、`govSite`、`qa`、`private`。
- `search_channel`: 政府网站检索渠道，当前可用 `govSearch`，通常留空即可。
- `search_source_url`: 限定搜索网址，多个用英文逗号分隔。
- `search_source_id`: 限定搜索数据源 ID，多个用英文逗号分隔。
- `material_length`: 返回资料最大字符数，留空表示使用接口默认值。
- `return_md5`: 是否返回文章和段落 MD5，便于后续精确追踪。
- `query_top_area`: 是否同时搜索当前地域的上级地域政策。
- `know_base`: 是否返回知识专库链接，默认开启；只有命中专库时才会有结果。
- `max_articles`: 默认最多展开多少篇召回材料。
- `max_paragraphs_per_article`: 默认每篇材料最多展开多少段原文。

首次使用前必须完成：

1. 打开 `https://platform.dknowc.cn`。
2. 注册或登录账号。
3. 在平台申请 API Key。
4. 打开本 skill 目录下的 `config.ini`，将申请到的 Key 填入 `api_key`。

不要把他人的 API Key 写入公开仓库或公开分发包。

## 调用方式

基础调用：

```bash
python3 {baseDir}/scripts/gov_chat.py "深圳市办理社保转移需要哪些材料？"
```

常用调试：

```bash
python3 {baseDir}/scripts/gov_chat.py "深圳市保障性住房有哪些政策措施？" --show-payload
python3 {baseDir}/scripts/gov_chat.py "深圳市办理社保转移需要哪些材料？" --item --policy --search-type affair --search-type policy --search-type qa
python3 {baseDir}/scripts/gov_chat.py "深圳市保障性住房有哪些政策措施？" --return-md5
python3 {baseDir}/scripts/gov_chat.py "深圳市保障性住房有哪些政策措施？" --json-only
```

脚本默认输出人类可读的材料包；需要给其他程序消费时使用 `--json-only`。

## 地域规则

- 该 skill 面向深圳用户；当用户没有明确说明地域时，默认按 `深圳市` 处理。
- 当用户明确说明其他地域时，以用户指定地域为准，不要强行改回深圳市。
- 示例：用户问“公文会签需要哪些部门参与？”时，按“深圳市公文会签”理解；用户问“北京市公文会签需要哪些部门参与？”时，按北京市处理。

## 工作流程

1. 判断用户问题是否属于政务智查范围。
2. 提取地域、事项、政策主题和时间要求；未提地域时使用 `深圳市`。
3. 调用 `scripts/gov_chat.py` 获取可信搜索材料包。
4. 阅读材料包中的召回材料、政策文件清单和办理事项清单。
5. 基于材料生成简明、可执行的政务回答。
6. 在回复末尾输出字段化“参考依据”；如果材料包返回知识专库链接，在“参考依据”之后追加“知识专库”。

## 跨渠道稳定回复规则

政务智查的最终回复要优先保证用户能看懂、能核对、能复制链接。不要依赖特定渠道的 Markdown、表格、脚注或链接渲染能力。

- 面向用户回复时，正文保持自然表达，不插入 `[^1^]`、`[依据1]` 等标记。
- 可信搜索返回的材料只用于筛选末尾“参考依据”，不要把完整召回清单直接丢给用户。
- 默认在回复末尾输出“参考依据”，只列出本次回复实际使用的来源。
- “参考依据”是该 skill 的核心特色，必须逐条、逐字段输出，不得压缩成“《标题》- 发布单位，日期”这种单行摘要。
- 每条参考依据必须尽量保留以下字段：标题、发文字号、发布单位、发布日期、链接、相关内容。
- 每条参考依据内部字段必须换行展示，字段前使用 `-` 短横线；不同依据条目之间必须空一行，避免在聊天窗口中挤成一整段。
- 不要使用 Markdown 表格、HTML、`[标题](链接)`、`查看`、`同上` 等依赖渲染或上下文补全的表达。
- 链接必须以完整纯文本 URL 输出，用户复制或点击都应能使用。
- 如果接口未返回源链接或材料信息，明确写“接口未返回源链接”或“接口未返回对应材料”，不得补写或猜测来源。
- 如果某个字段接口未返回，保留字段名并写“接口未返回”，不要直接省略字段。
- 如果材料包返回“知识专库链接”，在“参考依据”之后追加“知识专库”小节；未返回时不要输出该小节。
- “知识专库”使用纯文本完整 URL，不使用 Markdown 链接。
- 如果用户追问“哪句话对应哪条依据”，再输出“观点与依据对应”清单；默认回答不做逐句标注。

推荐格式：

```text
回复内容：
深圳市保障性住房主要包括公共租赁住房、保障性租赁住房和人才住房等类型。符合条件的申请人可重点关注申请条件、配租规则、租金标准和线上办理入口。

参考依据
1. 《深圳市保障性住房管理办法》
   - 发文字号：接口未返回
   - 发布单位：深圳市人民政府
   - 发布日期：2023年...
   - 链接：https://...
   - 相关内容：...

2. 《...》
   - 发文字号：接口未返回
   - 发布单位：接口未返回
   - 发布日期：接口未返回
   - 链接：接口未返回源链接
   - 相关内容：...

知识专库
- 链接：https://...
```

如果用户追问“来源在哪”“依据怎么看”，优先重新列出本轮已使用的“参考依据”纯文本来源清单；只有用户明确要求“完整来源清单”时，才输出完整材料列表，并仍然使用纯文本列表，不使用表格。

## 回答原则

- 优先给用户可执行的办理步骤、条件、材料、渠道。
- 涉及线上办理入口时，重点查看脚本输出的“办理事项清单”。
- 涉及政策依据时，重点查看脚本输出的“政策文件清单”和召回材料原文段落。
- 如果脚本输出“知识专库链接”，将其作为可信搜索特色能力展示在回复结尾。
- 不要把“材料包/召回结果”作为默认用户体验。
- 不要编造政策、链接、文号、发布日期或办理入口。
