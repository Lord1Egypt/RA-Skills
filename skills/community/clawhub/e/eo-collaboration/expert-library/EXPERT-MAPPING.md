## Using the Mapping

### Direct Category Access (Deep Path)
```
Need a specific expert?
→ marketing/seo-specialist.md
→ testing/performance-benchmarker.md
→ specialized/blockchain-security-auditor.md
```

### Role-Based Access (Fast Path)
```
Need a general role?
→ /plan      → Planner experts
→ /architect → Architect experts
→ /verify    → QA experts
```

## Mapping Philosophy

1. **Roles are MAPPINGS, not copies** - Each role expert represents a category expert that fits that role
2. **One-to-many** - One role can map to multiple category experts
3. **Fast path wins for common tasks** - Use roles for 80% of tasks
4. **Deep path for specialists** - Use category experts for niche requirements

See [ARCHITECTURE.md](./ARCHITECTURE.md) for architectural context.
