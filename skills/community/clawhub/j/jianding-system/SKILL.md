### [2] 收集

向用户要：
- 📷 照片（至少6张，存到本地目录）
- 📱 手机号

**边界处理：**
| 场景 | 处理方式 |
|------|---------|
| 照片不足 6 张 | 明确提示「至少需要 6 张」，列出建议角度（正面/背面/底部/局部细节/款识/全貌） |
| 照片 > 20 张 | 只取前 20 张，提示「已选取前 20 张提交」 |
| 用户只发照片无手机号 | 追问手机号，说明「用于接收鉴定结果短信」 |
| 用户只给手机号无照片 | 追问照片，告知最小 6 张要求 |
| 用户发非图片文件 | 提示仅支持 jpg/png，请重新发送 |
| 纯文本模型 + 用户未提供描述 | 必须索要至少「名称」和「年代」两项，缺一不可进入下一步 |

藏品描述获取策略（⭐ 多模态优先）：

### [4] 登录（⭐ 按环境分叉）

#### 微信小程序环境

用户已在微信内，调用 `wx.login()` 获取 code，直接用 code 登录：

```bash
python3 scripts/submit.py login --code "wx.login()返回的code"
```

> code 通过 `wx.login({ success: res => res.code })` 获取。

#### 非微信环境（终端 / CLI / 浏览器）

**两步走：**

第一步：发验证码
```bash
python3 scripts/submit.py send-sms --phone 13800138000
```

第二步：验证码登录
```bash
python3 scripts/submit.py login --phone 13800138000 --sms-code 123456
```

> 登录成功后 token 自动保存到 `~/.jianding_token`，后续命令无需再传。

#### 已有 token

```bash
python3 scripts/submit.py login --token "xxxx"
```

#### ⚠ 兜底：全部失效时

当 `--token`、`--code`、`--phone`、`~/.jianding_token` 全部不可用时，脚本不会报错退出，而是：

1. 打印清晰的兜底提示
2. 引导用户微信打开 H5 链接自行提交

> 兜底时告诉用户：**微信打开 https://jiand.shoucangyaji.com/ 即可自助提交。**

**边界处理：**
| 场景 | 处理方式 |
|------|---------|
| 短信验证码 60 秒未收到 | 提示用户检查手机信号，确认号码无误后可重新调用 `send-sms` |
| 验证码输入错误 | 提示「验证码错误」，请用户核对后重新输入（最多重试 3 次） |
| 验证码过期 | 提示「验证码已过期（有效期 5 分钟）」，重新发送 |
| 登录接口返回 token 失败 | 自动触发兜底流程，推送微信 H5 链接 |
| Token 文件读取失败 | 静默跳过，按降级顺序尝试下一认证方式 |

### [5] 提交

```bash
python3 scripts/submit.py full \
  --images /tmp/a.jpg /tmp/b.jpg /tmp/c.jpg /tmp/d.jpg /tmp/e.jpg /tmp/f.jpg \
  --desc "清乾隆青花瓷盘，直径25cm，祖传，品相完好" \
  --phone "13800138000" \
  --category "瓷器"
```

> 认证：自动按优先级 `--token > --code > --phone + --sms-code > ~/.jianding_token` 读取。

脚本流程：登录→上传→下单→支付→出二维码。把二维码发给用户扫码支付。

**边界处理：**
| 场景 | 处理方式 |
|------|---------|
| 图片上传失败（单张超 10MB） | 提示用户压缩图片或减少分辨率后重试 |
| 订单创建失败（API 返回错误） | 展示具体错误原因（如「品类不支持」「参数缺失」），引导修正后重新提交 |
| 提交中途网络中断 | 脚本自动重试（3 次指数退避），若均失败则提示用户稍后重试 |
| 已存在未支付订单 | 告知用户订单号，调 `qr --order-id <ID>` 重新获取支付二维码 |

## 命令速查

| 命令 | 用途 | 环境 |
|------|------|:--:|
| `python3 submit.py info` | 查看专家和费用 | 通用 |
| `python3 submit.py login --code <code>` | 微信登录 | 微信 |
| `python3 submit.py send-sms --phone <手机号>` | 发送验证码 | 非微信 |
| `python3 submit.py login --phone <手机号> --sms-code <验证码>` | 手机号登录 | 非微信 |
| `python3 submit.py login --token <token>` | Token 注入 | 通用 |
| `python3 submit.py full --images ... [--desc ...]` | 完整提交（--desc 可选，多模态可自动生成） | 通用 |
| `python3 submit.py result --order-no <订单号>` | 查鉴定结果 | 通用 |
| `python3 submit.py qr --order-id <ID>` | 已有订单重新生成二维码 | 通用 |

## 故障排查

| 现象 | 原因 | 解决方式 |
|------|------|---------|
| 提示"请求超时，1s后自动重试…" | 网络波动，脚本已自动重试（最多 3 次） | 等待自动恢复；若 3 次均失败，检查网络后重试命令 |
| 提示"连接失败，已重试 3 次均失败" | 网络不通 / DNS 解析失败 | ① 检查是否能访问 jiand.shoucangyaji.com ② 切换网络环境（WiFi↔4G）后重试 |
| 提示"响应解析失败" | 网络中断导致响应不完整 | 脚本自动重试；持续出现则检查网络稳定性 |
| 验证码迟迟收不到 | 短信通道延迟 | 等待 60 秒后仍未收到可重新调用 `send-sms` |
| 支付后未收到短信 | 支付确认有延迟 | 等待 5 分钟后仍未收到，微信中打开 https://jiand.shoucangyaji.com/ 进入【个人中心】→【我的订单】查看 |
| Token 失效 | Token 有过期时间 | 重新登录：`login --phone ... --sms-code ...` 获取新 token |
| 提示 "--images 至少需要 6 张" | 图片数量不足 | 补充到至少 6 张清晰照片（建议多角度：正面/背面/底部/细节） |
