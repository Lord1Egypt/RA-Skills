# 机构模式 开发参数获取

> 本文档列出**机构**接入境外微信支付（跨境）所需的全部开发参数与获取步骤。
>
> ‼️ **境外境内严格隔离**：本节所有参数（`sp_mchid` / 私钥 / APIv3 密钥 / 平台证书）均与境内**完全独立**，请勿混用，混用会直接导致 401 SIGN_ERROR。
>
> ‼️ 境外接入有"机构"和“直联商户”角色（[llms.txt](https://pay.weixin.qq.com/doc/global/v3/zh/llms.txt)），所有路径都以 `/v3/global/` 开头。

## 1. 参数清单

| 参数 | 说明 | 用途 | 是否必填 |
|---|---|---|---|
| `sp_mchid` | **机构**商户号 | 所有 API 请求的身份标识，签名主体 | 必填 |
| `sp_appid` | **机构**自身的 AppID（机构在微信公众平台 / 开放平台注册的应用 AppID）| 下单请求体；JSAPI / 小程序场景下用于获取 `sp_openid` | 必填 |
| `serial_no` | **机构**API 证书序列号 | 请求签名头 `Authorization` 中携带 | 必填 |
| `private_key` | **机构** API 证书私钥（PEM，PKCS#8）| 请求签名 RSA 私钥；**机构维度** | 必填 |
| `apiv3_key` | **机构** APIv3 密钥（32 字节）| 回调 / 平台证书 AEAD-AES-256-GCM 解密；机构维度配置 | 必填 |
| `wechatpay_certificates` | 平台证书 / 平台公钥（机构维度通过 `GET /v3/global/certificates` 拉取）| 应答 / 回调验签 | 必填 |
| `sub_mchid` | 子商户号（每个子商户独立，**机构请求体必填**）| 请求体 `sub_mchid` 字段，路由资金到子商户 | 必填 |
| `sub_appid` | 子商户的公众号 / 小程序 / 移动应用 AppID | 下单请求体 `sub_appid` 字段；与 `sub_openid` 配套 | 可选（部分产品必填）|
| `sp_openid` / `sub_openid` | 用户 OpenID（按用哪个 AppID 取决） | JSAPI / 小程序下单时二选一必填，详见下表 | 见下表 |
| `notify_url` | 异步通知回调地址（HTTPS，公网可达，**无需 ICP 备案**）| 通过设置事件通知地址 API 配置；机构维度，所有子商户共用同一地址 | 必填 |
| `currency` | 三字母 ISO 4217 币种码 | 下单请求体 `amount.currency`，按机构结算款币种或 CNY 填 | 必填 |
| `Accept-Language` | 错误描述语言 | 请求头，控制响应错误的语言 | 推荐 |

### `sub_appid` / `openid` 适配表（按支付产品）

| 产品 | `sub_appid` | OpenID 字段 | 备注 |
|---|---|---|---|
| 付款码支付 | 可选 | 不需要 | 用户出示付款码 |
| Native 支付 | 可选 | 不需要 | 商户出二维码 |
| JSAPI 支付 | 可选 | `payer.sp_openid`（`sp_appid` 下取） **或** `payer.sub_openid`（`sub_appid` 下取，传此项时 `sub_appid` 必填）| OpenID 通过 OAuth2 网页授权获取 |
| APP 支付 | 必填（子商户 APP的AppID） | 不需要 | — |
| 小程序支付 | 必填（子商户小程序 AppID）| `payer.sub_openid`（通过 `wx.login` 获取）| — |
| H5 支付 | 可选 | 不需要 | **机构需先为子商户开通 H5 支付权限** |
| 委托代扣 | 视签约入口 | 视签约入口 | 与签约方式配套 |

## 2. 获取步骤（按参数）

### 2.1 机构号 `sp_mchid` 与受理关系

1. 注册境外微信支付机构主体（境外商户平台机构入驻入口）。
2. 通过子商户进件 API（详见 [📄 接口索引](../示例代码/接口索引.md)第 9 节）为每个子商户递交资料、获得 `sub_mchid`。
3. 进件审核通过后，**机构与子商户之间形成"受理关系"**，后续所有支付请求都基于该关系生效。
4. **常见踩雷点**：进件成功后立刻调用支付报"受理关系不存在"——多数是请求体里 `sub_mchid` 写错了，或受理关系尚未生效（详见 [📄 排障手册](../问题排查/排障手册.md)）。

### 2.2 机构 `sp_appid` 获取方式

1. `sp_mchid`入驻成功后会默认绑定一个`sp_appid`，即**wx7bc98d929da735fe**，机构可用在quick pay, native payment, 小程序支付，APP支付，以及H5支付的场景中；
2. 如果机构需要对接**JSAPI支付**，则需要在微信公众号平台mp.weixin.qq.com注册并生成**自己的公众号appid**作为`sp_appid`，申请好后可在商户平台操作sp_mchid和sp_appid的绑定，绑定入口→ 「 Dev Configuration → APPID Authorization → Add APPID」；
3. `sp_appid`只能是**公众号或小程序的appid**

### 2.3 机构 API 证书 `serial_no` + `private_key`

1. 登录境外商户平台→ 「account settings → API security → API certificate」。
2. 按平台指引下载证书工具（CertificateTool），**在机构号下生成**。
3. 工具会生成 `apiclient_cert.pem`、`apiclient_key.pem` 等文件。
4. 在商户平台提交 `apiclient_cert.pem`，平台返回 `serial_no`。
5. ‼️ **机构所有的请求都用机构自身的私钥与序列号签名**.

### 2.4 APIv3 密钥 `apiv3_key`

1. 登录境外商户平台（机构入口）→ 「account settings → API security → API key v3」。
2. 设置一个 32 字节的随机字符串。
3. 机构维度全局唯一；所有子商户产生的回调都用这把密钥解密。

### 2.5 平台证书 / 平台公钥

调用 [下载平台证书 API](https://pay.weixin.qq.com/doc/global/v3/zh/4013013878.md)：`GET https://apihk.mch.weixin.qq.com/v3/global/certificates`。

机构维度使用一份平台证书；用机构 APIv3 密钥 AEAD 解密 `encrypt_certificate` 字段，得到平台公钥用于响应 / 回调验签。机构内**只需缓存一份**，按证书序列号选择验签即可。

### 2.6 子商户 `sub_mchid` / `sub_appid` / `sub_openid`

1. **`sub_mchid`**：通过子商户进件 API 创建子商户后获得；保存在子商户档案表。
2. **`sub_appid`**：子商户在微信公众平台 / 开放平台注册的 AppID；机构需在境外商户平台「商户平台 → institution → application → 选择对应子商户 → config」中**绑定 `sub_appid` 到 `sub_mchid`**。
3. **`sub_openid`**：基于 `sub_appid` 通过 OAuth2 / `wx.login` 获取，**不能用机构 AppID 下的 OpenID**（不同 AppID 下的 OpenID 不通用）。如不传 `sub_appid` 而走机构 AppID 取 OpenID，请使用 `payer.sp_openid` 字段。

### 2.7 `notify_url`（机构配置方式）

1. **建议通过设置事件通知地址 API 配置**：每业务一份，对应业务下所有子商户共用。
2. **所有子商户共用同一回调地址**，业务侧必须按 `sub_mchid` 路由：
   - 错误：A 子商户的订单到 B 子商户系统处理 → 串单事故
   - 正确：解密回调 → 取 `sub_mchid` → 查到对应子商户上下文 → 入账
3. HTTPS + 公网可达，**不需要 ICP 备案**，URL 前后无空格。

### 2.8 多币种字段 `amount.currency`

- 默认情况下，currency字段仅允许传入sp_mchid对应的结算币种或CNY
- 如果有标价使用非结算币种的需求，可联系区域运营申请

### 2.9 `Accept-Language` 请求头

支持 `zh-CN` / `zh-HK` / `zh-TW` / `en`，控制错误响应的描述语言。**强烈建议显式设置**，否则默认语言可能跟团队期望不一致，导致排查不便。

## 3. 商户平台预先配置项

| 项 | 入口 | 影响 |
|---|---|---|
| 机构 API 证书启用 | account settings → API security → API certificate | 影响所有请求签名 |
| 机构 APIv3 密钥设置 | account settings → API security → API key v3 | 影响所有回调解密 |
| 子商户 AppID 绑定 | 商户平台 → institution → application → 选择对应子商户 → config | 子商户 小程序/APP下单 |
| 子商户 H5 支付权限 | 商户平台 → institution → H5 payment application → 选择对应子商户 → Apply，或在子商户进件时传入h5_payment_apply_info提交申请 | 子商户 H5 下单 |
| 跨境分账业务开通 | 提交业务申请 | 跨境分账下单与请求分账 |
| 海关报关业务开通 | 提交业务申请 | 报关推单 |

## 4. 与境内的差异速查

| 项 | 境内（机构 / 服务商） | 境外（机构）|
|---|---|---|
| 域名 | `api.mch.weixin.qq.com` | `apihk.mch.weixin.qq.com` |
| 路径前缀 | `/v3/pay/partner/...` 等 | `/v3/global/...`（统一前缀） |
| 平台证书路径 | `GET /v3/certificates` | `GET /v3/global/certificates` |
| 入驻入口 | 境内服务商入驻 | **境外机构入驻**（独立体系）|
| 证书 / 私钥 / APIv3 密钥 | 一套 | **另一套**（独立） |
| `notify_url` ICP 备案 | 必须 | **不需要** |
| 多币种 | 默认 CNY | 必填 `amount.currency`，默认只支持sp_mchid结算币种或CNY |
| `Accept-Language` | 默认中文 | 支持 zh-CN / zh-HK / zh-TW / en |
