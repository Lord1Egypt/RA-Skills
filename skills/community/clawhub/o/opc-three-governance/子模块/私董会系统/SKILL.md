# 胡田-OPC导师-私董会Skill

## 一、Skill概述

### 1.1 定位

这是一个基于"九贤私董会"方法论的战略决策辅助工具。通过调用9位跨时代思想家（王阳明、曾国藩、范仲淹、苏格拉底、维特根斯坦、张小龙、马斯克、任正非、苏轼）的智慧，对用户提出的任何战略决策问题进行多维度审视，最终输出结构化的"真-恒-心"统合框架。

### 1.2 核心价值

- **多视角审视**：同一问题，9种思维方式，9套决策逻辑
- **反直觉追问**：打破用户自身的认知盲区
- **统合输出**：从分散到归一，形成可执行的行动框架

### 1.3 适用场景

- 战略规划与重大决策
- 商业模式设计与验证
- 产品规划与创新方向
- 组织管理与团队建设
- 个人职业发展与人生选择
- 任何需要深度思考的复杂问题

## 三、私董会四步流程

### 第一步：知识蒸馏（可选加速）

**目的**：预先构建9位思想家的知识体系，供后续调用。

**操作**：
1. 为每位思想家提炼三层：
   - **世界观**：对世界的基本认知框架
   - **方法论**：面对问题的解决路径
   - **决策原则**：不可违背的底线

2. 9位成员的知识基座已预置在 `references/` 目录下：
   - `01_王阳明.md` - 致良知，知行合一
   - `02_曾国藩.md` - 拙诚为本，有恒第一
   - `03_范仲淹.md` - 先忧后乐，宁鸣而死
   - `04_苏格拉底.md` - 认识你自己，无知之知
   - `05_维特根斯坦.md` - 划定边界，保持沉默
   - `06_张小龙.md` - 洞察人性，简单极简
   - `07_马斯克.md` - 第一性原理，物理约束
   - `08_任正非.md` - 灰度，熵减，长期主义
   - `09_苏轼.md` - 此心安处，超然物外

3. 完整总纲参见 `references/私董会思想基座总纲.md`

**加速模式**：如时间紧迫，可跳过此步，直接进入第二步，由Agent按需调用相关思想家的知识。

### 第二步：圆桌论道

**目的**：9位思想家基于各自的知识体系，对用户问题从不同视角发表观点，并提出核心拷问。

**操作流程**：

#### 2.1 接收用户问题

清晰记录用户提出的核心问题，明确决策场景和背景。

#### 2.2 分发九贤发言

依次让9位思想家发言，每个成员包含两个部分：

**【观点阐述】**
- 基于该思想家的世界观和方法论
- 对问题的本质分析
- 独特的切入视角

**【核心拷问】**
- 该思想家抛出的最犀利问题
- 直指问题的核心矛盾
- 往往是用户未曾想到的角度

#### 2.3 发言模板

```
## [思想家姓名]篇：[核心主张]

### 观点阐述

[基于该思想家的核心方法论，对用户问题进行分析]
[指出现实中与该方法论相悖的现象]
[强调该视角对决策的重要性]

### 核心拷问

> **"[核心拷问内容]"**
```

#### 2.4 九贤视角分配

| 思想家 | 视角定位 | 核心关注点 |
|--------|---------|-----------|
| 王阳明 | 内在动机 | 这个决定是否发自良知？执行者能否知行合一？ |
| 曾国藩 | 长期坚持 | 热潮退去后能否坚持？有没有"日课"机制？ |
| 范仲淹 | 社会责任 | 受益的是少数人还是大众？利益如何分配？ |
| 苏格拉底 | 概念澄清 | 我们真的知道问题是什么吗？定义清晰吗？ |
| 维特根斯坦 | 边界意识 | 哪些事不该做？边界在哪里？ |
| 张小龙 | 用户价值 | 这对用户真的有益吗？会利用用户焦虑吗？ |
| 马斯克 | 本质追问 | 第一性的痛点是什么？成本结构合理吗？ |
| 任正非 | 组织活力 | 如何避免熵增？团队能否持续进化？ |
| 苏轼 | 生命意义 | 十年后回望，这件事值得吗？心安吗？ |

### 第三步：统合总结

**目的**：融合9位思想家的智慧，提炼出"真-恒-心"三维框架。

**操作流程**：

#### 3.1 提炼三维框架

**真**（Truth）
- 核心问题：什么是真导师/真方案/真价值？
- 追问者：王阳明、苏格拉底、马斯克
- 关键问题：
  - 这个决定是否经得起良知检验？
  - 我们对问题的定义是否清晰准确？
  - 这是不是解决本质痛点的最优路径？

**恒**（Persistence）
- 核心问题：如何持续为用户创造价值？
- 追问者：曾国藩、范仲淹、维特根斯坦
- 关键问题：
  - 能否坚持三年、五年甚至更久？
  - 受益者是否真正被赋能而非被收割？
  - 边界在哪里？哪些事不该做？

**心**（Purpose）
- 核心问题：这件事值不值得做？
- 追问者：张小龙、任正非、苏轼
- 关键问题：
  - 我们是在帮助人还是利用人的焦虑？
  - 组织能否保持创业早期的活力？
  - 做这件事，心安吗？

#### 3.2 统合输出模板

```
## 统合框架："真-恒-心"

### 真（Truth）—— 王阳明 + 苏格拉底 + 马斯克

[融合三位思想家的核心洞见]
[回答：什么是"真"？如何验证？]

### 恒（Persistence）—— 曾国藩 + 范仲淹 + 维特根斯坦

[融合三位思想家的核心洞见]
[回答：如何持续？如何划界？]

### 心（Purpose）—— 张小龙 + 任正非 + 苏轼

[融合三位思想家的核心洞见]
[回答：值不值得？心安吗？]
```

> **三个问题，检验一切决策：**
>
> 1. **王阳明之问**：这个决定，是否让我离目标更近一步？
> 2. **马斯克之问**：这个选择，是否用最低成本解决了本质痛点？
> 3. **苏轼之问**：做这件事，我的心，平静吗？
>
> 三个答案一致，则做；不一致，则停。

## 五、使用示例

### 示例：用户提出战略决策问题

**用户问题**：
> "我们计划推出一款面向中小企业的SaaS产品，定位是'一站式管理工具'，功能涵盖财务管理、CRM、项目管理等。请问这个方向是否正确？"

**Agent执行流程**：

1. **读取知识基座**（按需调用references目录下的文件）
2. **启动九贤论道**，让每位思想家发言
3. **提炼真-恒-心框架**
4. **输出三三三结构**

**输出示例**：

```
## 九贤论道摘要

| 思想家 | 核心洞见 | 核心拷问 |
|--------|---------|---------|
| 王阳明 | 产品的出发点是否真正为客户创造价值，而非自身商业利益？ | 客户用完真的能走吗？ |
| 曾国藩 | 功能大而全正是"拙"的反面，贪多求全是败亡之道 | 如何用笨功夫打磨一个核心功能？ |
| 苏格拉底 | 什么是"一站式"？这个词的定义清晰吗？ | CRM和项目管理的本质区别是什么？ |
| ... | ... | ... |

## 三三三输出

### 三个核心要点
1. **聚焦单点，突破阈值**：选择一个场景做深，不求全
2. **客户成功优先于产品功能**：先验证能为客户带来什么结果
3. **保持克制，拒绝诱惑**：敢于对扩展功能说"不"

### 三个核心难点
1. **功能蔓延**：做着做着就想加功能，最终四不像
2. **竞品焦虑**：看到别人做什么就慌，丧失定力
3. **盈利压力**：过早追求营收，牺牲客户体验

### 三个突破口
1. **从"记账"切入**：中小企业最痛的点是财税合规
2. **从标杆客户开始**：找3-5个愿意深度共创的种子用户
3. **从"用完即走"设计**：每个功能都要问"这个功能值得存在吗？"
```

## 七、注意事项

1. **真实性原则**：每位思想家的发言必须符合其核心思想体系，不可泛泛而谈
2. **多元性原则**：9位思想家的观点可能相互矛盾，这是正常的，矛盾本身是洞见的一部分
3. **实用性原则**：最终输出必须可执行，避免空泛的概念总结
4. **启发性原则**：核心拷问要够犀利，真正打破用户的思维定式
