===========================================
ATOM-FEISHU-031 - HTML 转飞书文档（含 Mermaid 截图）
===========================================

【这个动作是干啥的】
把 HTML 专家点评网页（含 Mermaid 图表）转换成飞书文档，Mermaid 图表会自动截图插入。

【什么时候用】
- 用户说"转化为飞书文档"
- 用户说"HTML 转飞书"
- 需要把本地 HTML 报告同步到飞书协作

【核心步骤】
1. 读取 HTML 文件内容
2. 用 Chrome 打开 HTML，等待 Mermaid 渲染
3. 截取 Mermaid 图表（PNG 图片）
4. HTML 内容转 Markdown
5. 分块写入飞书文档（≤200 blocks/块）
6. 上传截图并插入到对应位置

【举个例子】
用户：把架构图 HTML 转化为飞书文档
    ↓
调用：ATOM-FEISHU-031
    ↓
处理：
  - 读取 architecture-atomic-skills-workflow.html
  - Chrome 打开，等待 Mermaid 渲染
  - 截图架构图（PNG）
  - 内容转 Markdown
  - 分块写入飞书
  - 插入架构图截图
    ↓
输出：飞书文档（含架构图截图）
    URL: https://feishu.cn/docx/XXX

【对应阿福的 MD 文件】
atomic-actions/03-delivery/ATOM-FEISHU-031-HTML 转飞书文档.md

【关联动作】
- ATOM-VISUAL-005：生成 HTML 文件（前置）
- ATOM-FEISHU-028：写入飞书文档（分块）
- ATOM-VISUAL-009：Chrome 打开文件

【常见错误】
❌ 错误：Mermaid 未渲染就截图 → 空白图表
✅ 正确：等待 5 秒，让 Mermaid 渲染完成

❌ 错误：一次性写入飞书 → 空白文档
✅ 正确：分块写入（≤200 blocks/块）

❌ 错误：直接写入 HTML 代码
✅ 正确：转换为 Markdown 格式

【核心原则】
> HTML 转飞书 = 内容转换 + Mermaid 截图 + 分块写入

【关键点】
1. Mermaid 图表必须截图（飞书不支持渲染）
2. HTML 内容转 Markdown（飞书兼容）
3. 分块写入（避免 400 错误）

【触发词】
- "转化为飞书文档"
- "HTML 转飞书"
