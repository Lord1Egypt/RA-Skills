## 证券投资生成要求

证券投资每种任务类型的"零件清单"——必选/可选组件、组装顺序、领域约束。按清单逐项产出。

---

### 1 投资研究

#### T1-1 市场研究

- **必选组件**: 研究范围、研究方法、研究数据、研究结论
- **可选组件**: 研究工具、研究模型、研究报告
- **组装顺序**: 确定研究范围 → 选择研究方法 → 收集研究数据 → 形成研究结论
- **约束**: 范围必须全面；方法必须科学；数据必须可靠
- **格式**: 市场研究报告（Markdown）

#### T1-2 行业分析

- **必选组件**: 分析维度、分析方法、分析数据、分析结论
- **可选组件**: 分析工具、分析模型、分析报告
- **组装顺序**: 确定分析维度 → 选择分析方法 → 收集分析数据 → 形成分析结论
- **约束**: 维度必须全面；方法必须科学；数据必须准确
- **格式**: 行业分析报告（Markdown）

#### T1-3 公司研究

- **必选组件**: 研究内容、研究方法、研究数据、研究结论
- **可选组件**: 研究工具、研究模型、研究报告
- **组装顺序**: 确定研究内容 → 选择研究方法 → 收集研究数据 → 形成研究结论
- **约束**: 内容必须全面；方法必须科学；数据必须准确
- **格式**: 公司研究报告（Markdown）

#### T1-4 投资策略

- **必选组件**: 投资理念、投资风格、投资方法、策略说明
- **可选组件**: 策略回测、策略优化、策略评估
- **组装顺序**: 确定投资理念 → 选择投资风格 → 设计投资方法 → 撰写策略说明
- **约束**: 理念必须清晰；风格必须匹配；方法必须可行
- **格式**: 投资策略文档（Markdown）

### 2 投资决策

#### T2-1 投资分析

- **必选组件**: 分析维度、分析方法、分析数据、分析结论
- **可选组件**: 分析工具、分析模型、分析报告
- **组装顺序**: 确定分析维度 → 选择分析方法 → 收集分析数据 → 形成分析结论
- **约束**: 维度必须全面；方法必须科学；结论必须客观
- **格式**: 投资分析报告（Markdown）

#### T2-2 风险评估

- **必选组件**: 风险类型、风险识别、风险评估、风险报告
- **可选组件**: 风险工具、风险模型、风险系统
- **组装顺序**: 确定风险类型 → 识别风险 → 评估风险 → 生成报告
- **约束**: 类型必须全面；识别必须准确；评估必须客观
- **格式**: 风险评估报告（Markdown）

#### T2-3 投资决策

- **必选组件**: 决策依据、决策结果、决策理由、决策记录
- **可选组件**: 决策工具、决策模型、决策系统
- **组装顺序**: 分析决策依据 → 做出决策 → 阐述理由 → 记录决策
- **约束**: 依据必须充分；结果必须明确；理由必须合理
- **格式**: 投资决策记录（Markdown）

#### T2-4 投资组合

- **必选组件**: 资产配置、行业配置、个股选择、组合说明
- **可选组件**: 组合优化、组合回测、组合评估
- **组装顺序**: 设计资产配置 → 确定行业配置 → 选择个股 → 撰写组合说明
- **约束**: 配置必须合理；选择必须有据；说明必须清晰
- **格式**: 投资组合方案（Markdown）

### 3 投资执行

#### T3-1 交易执行

- **必选组件**: 交易指令、交易方式、交易确认、交易记录
- **可选组件**: 交易工具、交易系统、交易监控
- **组装顺序**: 下达交易指令 → 选择交易方式 → 确认交易 → 记录交易
- **约束**: 指令必须准确；方式必须合规；记录必须完整
- **格式**: 交易执行记录（Markdown）

#### T3-2 交易监控

- **必选组件**: 监控指标、监控方法、监控数据、监控报告
- **可选组件**: 监控工具、监控系统、监控预警
- **组装顺序**: 确定监控指标 → 选择监控方法 → 收集监控数据 → 生成监控报告
- **约束**: 指标必须关键；方法必须有效；数据必须准确
- **格式**: 交易监控报告（Markdown）

#### T3-3 交易记录

- **必选组件**: 交易明细、交易费用、交易收益、记录管理
- **可选组件**: 记录工具、记录系统、记录查询
- **组装顺序**: 记录交易明细 → 记录交易费用 → 计算交易收益 → 管理记录
- **约束**: 明细必须准确；费用必须完整；收益必须正确
- **格式**: 交易记录表（Markdown）

#### T3-4 交易结算

- **必选组件**: 结算流程、资金结算、证券结算、对账确认
- **可选组件**: 结算系统、结算工具、结算监控
- **组装顺序**: 执行结算流程 → 结算资金 → 结算证券 → 确认对账
- **约束**: 流程必须规范；结算必须准确；对账必须及时
- **格式**: 交易结算记录（Markdown）
