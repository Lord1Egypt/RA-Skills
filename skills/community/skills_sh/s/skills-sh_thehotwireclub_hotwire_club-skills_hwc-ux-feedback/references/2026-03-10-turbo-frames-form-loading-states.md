## Problem

When a form is submitted inside a `<turbo-frame>`, Turbo intercepts the request and swaps the response, but gives no visual feedback while the request is in flight. The submit button stays clickable and nothing changes on screen. The old Rails-UJS `data-disable-with` attribute is no longer the answer.

## Solution

Two complementary mechanisms:

1. `data-turbo-submits-with` on the submit button — Turbo 8's successor to `data-disable-with`. Turbo disables the button during submission and swaps its text (or value) for the supplied string, restoring it when the request completes.
2. The `[busy]` / `[aria-busy]` attributes Turbo sets on a `<turbo-frame>` while it has a request in flight — usable as a pure CSS hook for frame-level feedback.

## Implementation

### Submit button

For a `<button>`, the loading text replaces its text content:

```html
<form>
  <input type="text" name="message" required>
  <button type="submit" data-turbo-submits-with="Saving…">Sign guest book</button>
</form>
```

For an `<input type="submit">`, it replaces the `value`:

```html
<input type="submit" value="Sign guest book" data-turbo-submits-with="Saving…">
```

Turbo handles disabling and restoring automatically — no JavaScript required.

### Frame-level feedback with CSS

Turbo sets `busy` and `aria-busy="true"` on the frame element during the request. Style against the attribute selector:

```css
turbo-frame {
  transition: opacity 0.2s;
}

turbo-frame[busy] {
  opacity: 0.5;
  pointer-events: none;
}
```

### Rails view

In a Rails app the button is generated by `form_with`:

```erb
<%= form_with model: @entry do |form| %>
  <%= form.text_field :message, required: true %>
  <%= form.submit "Sign guest book", data: { turbo_submits_with: "Saving…" } %>
<% end %>
```

## Key Points

- `data-turbo-submits-with` is the Turbo 8 replacement for Rails-UJS `data-disable-with`.
- On a `<button>` it swaps text content; on `<input type="submit">` it swaps the `value`.
- Turbo disables the control during the request and restores it afterwards automatically.
- Turbo sets `[busy]` and `[aria-busy]` on a `<turbo-frame>` for the duration of any request — frame navigations and form submissions alike.
- The `[busy]` attribute is a CSS-only hook; no Stimulus controller is needed for frame-level feedback.
