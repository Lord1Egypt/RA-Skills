# B2B 官网 SEO 全维度审计

## 核心诊断理念

B2B 官网 SEO 的根本问题不是"没做 SEO"，而是四个关键子系统彼此脱节：

```
品牌页（首页/关于）──讲品牌故事，缺关键词锚点
    ↓ 断层
博客/资源中心 ── 讲知识/趋势，缺商机关联
    ↓ 断层
产品/方案页 ── 讲功能/规格，缺搜索意图匹配
    ↓ 断层
转化路径（表单/Demo/联系）── 流量未被有效导向
```

本审计的终极目标：将这四者串成一条完整的"搜索→发现→信任→转化"链路。

## 触发场景

当用户提出以下意图时自动激活：

- 对某个 B2B 官网（提供 URL）做SEO审计/诊断/体检
- 分析网站的搜索流量问题与优化机会
- 做竞品 SEO 内容差距分析
- 制定 B2B 网站的 SEO 执行计划
- 询问如何提升官网的自然搜索流量

## 工作流总览

采用 7 阶段递进式审计，每阶段输出中间结论，最终汇总为完整报告。

```
Phase 1: 技术 SEO 审计 → Phase 2: 内容 SEO 审计 → Phase 3: 竞品差距分析
    → Phase 4: 关键词机会地图 → Phase 5: TOP 20 优化排序
    → Phase 6: 90天执行计划 → Phase 7: 输出报告 (MD + HTML)
```

## 前置准备

在开始审计前，收集以下信息：

1. **目标网站 URL**：用户必须提供。若只给了网站名，先搜索确认 URL
2. **竞品 URL**（可选）：用户可提供 3-5 个已知竞品；否则通过网络搜索匹配的竞品
3. **业务上下文**：简要了解该网站的产品/服务、目标客户、客单价区间（B2B 中影响关键词策略）

如果用户未提供竞品，使用 WebSearch 搜索 `{行业关键词} top companies` 或 `{产品词} competitors` 自动识别 3-5 个相关竞品。

## Phase 1: 技术 SEO 审计

目标：发现所有阻碍搜索引擎正确抓取、索引和渲染网站的技术问题。

### 1.1 执行步骤

1. **基础可访问性检查**
   - 使用 WebFetch 访问目标 URL，确认网站可正常加载
   - 检查是否 HTTPS（浏览器自动升级则跳过）
   - 访问 `/robots.txt`，检查关键目录未被阻止
   - 若 robots.txt 中指定了 Sitemap URL，记录之

2. **PageSpeed / Core Web Vitals 分析**
   - 使用 WebSearch 查询 `site:{domain}` 了解收录情况
   - 使用 WebFetch 抓取关键页面，分析 HTML 结构线索
   - 关注：Title 长度、Meta Description 存在性、H1 唯一性、图片 Alt 属性
   - 若有条件，建议用户使用 PageSpeed Insights / Lighthouse 获取精确性能数据

3. **结构化数据检查**
   - 抓取首页、产品页、博客页 HTML，搜索 `application/ld+json` 或 schema 相关标记
   - 检查至少应包含 Organization / WebSite / BreadcrumbList 类型

4. **内链结构分析**
   - 抓取首页 HTML，提取主导航的链接结构
   - 抓取博客页 HTML，检查是否存在指向产品/方案页的内链
   - 识别是否有明显的孤岛页面（通过导航覆盖判断）

### 1.2 评分规则

按以下维度 0-100 评分（详见 `references/seo-audit-framework.md`）：
- 可抓取性 & 可索引性（20 分）
- 网站架构 & URL 结构（15 分）
- 页面速度 & Core Web Vitals（20 分）
- 结构化数据（10 分）
- 内链结构（15 分）
- 安全 & 基础设施（10 分）
- 移动端适配（10 分）

### 1.3 输出

该阶段输出技术 SEO 发现清单，每条包含：问题描述、严重程度（Critical/High/Medium/Low）、影响范围、修复建议。

## Phase 2: 内容 SEO 审计

目标：评估现有内容的搜索意图匹配度、关键词覆盖度和转化效率。

### 2.1 执行步骤

1. **页面级 SEO 元素抽样**
   - 抓取首页、3-5 个核心产品/方案页、3-5 篇博客文章
   - 逐页检查 Title Tag / Meta Description / H1-H6 层级 / 图片 Alt / Canonical URL
   - 判断 Title 是否包含核心关键词、是否超过 60 字符、品牌名是否在末尾

2. **内容深度 & E-E-A-T 评估**
   - 检查内容长度：产品页 ≥ 500 字、博客 ≥ 1500 字为合格
   - 评估专业性信号：行业专家署名、数据引用、方法论说明
   - 评估信任信号：联系方式、客户案例、资质认证

3. **买家旅程覆盖度矩阵**
   对照四个阶段，逐阶段评估内容覆盖度（1-10 分）：
   - 问题意识阶段：痛点文章、趋势报告、白皮书
   - 方案探索阶段：对比指南、ROI 计算器、方案概述
   - 产品评估阶段：产品页、功能对比、Demo 视频
   - 决策购买阶段：案例研究、定价页、试用申请、客户推荐

4. **转化路径诊断**
   - 逐页检查 CTA（Call to Action）的存在性和相关性
   - 博客页是否引导到产品/方案页或资料下载
   - 产品页是否有明确的 Demo/试用/联系入口
   - 移动端 CTA 位置是否合理（应在首屏可见）

### 2.2 评分规则

按以下维度 0-100 评分（详见参考文件）：
- 页面级 SEO 元素完整性（25 分）
- 内容质量 & E-E-A-T（25 分）
- 买家旅程覆盖度（25 分）
- 转化路径完整性（25 分）

### 2.3 输出

该阶段输出内容 SEO 诊断结果，包含：内容覆盖度矩阵、关键页面 SEO 元素清单、转化路径断点。

## Phase 3: 竞品内容差距分析

目标：识别竞品在内容策略、关键词布局和用户体验上优于我方的地方。

### 3.1 执行步骤

1. **识别竞品**
   - 优先使用用户提供的竞品列表
   - 否则通过网络搜索匹配（目标行业 + 产品/方案关键词）

2. **竞品内容扫描**
   对每个竞品，使用 WebFetch 抓取首页 + 博客页 + 产品页，对比：
   - 收录页面数（通过 `site:competitor.com` 搜索）
   - 博客更新频率（查看最新文章日期）
   - 内容类型丰富度（博客、案例、白皮书、视频、工具等）
   - 结构化数据使用情况（抓取 HTML 检查 Schema）
   - 核心功能页的页面体验（加载速度、移动适配等可观察质量）

3. **关键词覆盖分析**
   - 搜索 5-10 个核心行业关键词，记录 SERP 中竞品出现情况
   - 识别竞品有而我方无的排名关键词
   - 识别竞品获得 Featured Snippet / FAQ / Video 富文本结果的页面

4. **差距量化**
   使用对比表格（详见 `references/seo-audit-framework.md` 第3.2节），逐维度对比我方与竞品。

### 3.2 输出

该阶段输出竞品差距对比表，识别 3-5 个我方必须追赶的关键差距。

## Phase 4: 关键词机会地图

目标：按"商业价值 × 排名难度"对关键词分类，输出可执行的机会地图。

### 4.1 执行步骤

1. **种子关键词识别**
   - 从目标网站现有内容中提取核心关键词
   - 结合业务上下文扩展相关搜索词

2. **关键词扩展**
   - 使用 WebSearch 查询 `{种子词} 相关搜索` 或搜索相关行业术语
   - 通过 `site:{domain}` 搜索了解已覆盖的关键词
   - 通过竞品排名数据识别遗漏的关键词

3. **四象限分类**
   将扩展后的关键词按"商业价值（1-10）× 排名难度（1-10）"投入四象限：
   - **速赢区**（高价值 7-10，低难度 1-4）：立即优化，预计 30 天见效
   - **战略区**（高价值 7-10，高难度 5-10）：建设专题页 + 外链，90 天+
   - **填充区**（低价值 1-6，低难度 1-4）：模板化批量覆盖
   - **搁置区**（低价值 1-6，高难度 5-10）：除非有战略意义否则暂缓

4. **关键词聚类**
   将关键词按主题聚类为 Topic Clusters，每个集群标注一个 Pillar Page 主题。

### 4.2 输出

该阶段输出四象限关键词地图 + 3-5 个 Topic Clusters，每个集群含 Pillar Page 建议和子内容列表。

## Phase 5: TOP 20 页面优化优先级

目标：按综合权重评分，排出需要优先优化的前 20 个页面。

### 5.1 评分算法

每个页面按以下维度加权评分（总分 100）：

| 维度 | 权重 | 说明 |
|------|------|------|
| 流量潜力 | 25% | 目标关键词月搜索量总和 |
| 排名位置 | 20% | 当前排名越接近第1页，得分越高（第11-20名得分最高） |
| 商业价值 | 25% | 产品页 → 方案页 → 案例 → 博客 → 关于页 |
| 优化难度 | 15% | 改动越小得分越高（Title/Meta → 内容微调 → 架构改动） |
| 当前表现 | 15% | CTR、内容质量、转化路径完整性 |

### 5.2 输出格式

对每个 TOP 20 页面，输出：
1. URL
2. 当前排名位置（如有数据）
3. 目标关键词（2-3 个）
4. 优先级评分（/100）
5. 核心问题（1 句话）
6. 具体优化建议（3-5 条）
7. 预期流量提升（范围）
8. 建议实施阶段（Phase 1-4）

## Phase 6: 90 天执行计划

目标：将所有发现转化为分阶段、可追踪的执行计划。

### 6.1 四阶段划分

| 阶段 | 时间 | 核心任务 | 里程碑 |
|------|------|----------|--------|
| Phase 1: 修复 | Day 1-14 | 技术 SEO 紧急问题清零 | 404/301 修复、Sitemap 更新、结构化数据补全 |
| Phase 2: 速赢 | Day 15-30 | TOP 10 页面优化 | Title/Meta 优化、内容补充、内链加固 |
| Phase 3: 建设 | Day 31-60 | 支柱内容创建、关键词缺口填补 | Pillar Page 上线、簇内容创建、长尾覆盖 |
| Phase 4: 增长 | Day 61-90 | 外链建设、内容矩阵扩展 | DR 提升、品牌搜索量增长、转化率优化 |

### 6.2 输出

以周为粒度输出具体任务清单，每项含：任务描述、负责角色、预期耗时、完成标准。

## 审计原则

1. **可执行性优先**：每个发现必须附带具体的修复建议，而非模糊的"需要优化"
2. **商业导向**：所有优化建议必须与商业价值（线索/商机/收入）挂钩
3. **优先级排序**：始终标注紧急程度和时间投入产出比
4. **诚实评估**：不粉饰太平，客观指出问题，但也肯定已有优势
5. **用户设限礼貌处理**：若对网站/竞品的抓取次数过多受到限制，应尊重限制并表示理解，优先用缓存/替代来源完成分析

## 工具依赖说明

由于本技能无法直接调用 Lighthouse API / Google Search Console / SEMrush / Ahrefs 等 SEO 专有工具，审计采用以下替代方案：

- **页面分析**：WebFetch 抓取 HTML + WebSearch 搜索 `site:` 指令
- **关键词数据**：WebSearch 获取关键词思路 + SERP 分析 + 公开数据推断
- **性能数据**：基于 HTML 结构推断 + 建议用户用 PageSpeed Insights 补充
- **竞品数据**：WebFetch 抓取竞品网站 + WebSearch 搜索对比

报告中应明确标注哪些数据是推断的，哪些建议用户用专业工具验证。
