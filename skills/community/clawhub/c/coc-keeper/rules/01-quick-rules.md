# COC 克苏鲁的呼唤 - 快速规则参考

> 本文档基于《克苏鲁的呼唤》第七版守秘人规则书整理，供AI守秘人快速查阅核心规则。

---

## 1. 游戏概述

**克苏鲁的呼唤（Call of Cthulhu）** 是一款基于H.P.洛夫克拉夫特作品改编的恐怖主题桌面角色扮演游戏（TRPG）。

- **守秘人（Keeper/KP）**：游戏主持人，负责叙述故事、操控NPC和怪物、裁决规则
- **调查员（Investigator/PC）**：玩家扮演的角色，试图揭开恐怖事件的真相
- **核心体验**：调查员在追寻真相的过程中逐渐失去理智，面对宇宙恐怖，可能疯狂或死亡
- **没有传统意义的输赢**：游戏是合作性质的，参加者共同努力达到目标（通常是阻止邪恶阴谋）

### 游戏基本流程

1. 守秘人描述场景和事件
2. 玩家声明调查员要做什么（通常用角色口吻）
3. 遇到冲突/挑战时掷骰决定结果
4. 守秘人根据结果推进故事

**何时掷骰**：只有在存在紧张感和戏剧性的冲突时才需要掷骰。日常事务（开车去某个地方、正常交流等）不需要检定。

---

## 2. 核心骰子机制：D100 检定

### 基本原理

COC使用**百分骰（D100）**作为核心检定机制。由两个10面骰子组成：
- **十位骰**：标记为 00, 10, 20 ... 90
- **个位骰**：标记为 0-9
- 两骰相加得到 1-100 的结果
- 00 + 0 = **100**

### 检定方式

掷出D100，结果 **小于等于** 目标值即为成功。

## 3. 属性系统简介

调查员拥有以下核心属性（通常以3的倍数骰决定初始值）：

| 属性 | 缩写 | 说明 |
|------|------|------|
| 力量 | STR | 物理力量 |
| 体质 | CON | 健康与耐力 |
| 体型 | SIZ | 身体大小 |
| 敏捷 | DEX | 反应速度与灵活性 |
| 外貌 | APP | 外表与第一印象 |
| 智力 | INT | 推理与洞察力 |
| 意志 | POW | 精神力量与魔法抗性 |
| 教育 | EDU | 知识与经验 |
| 幸运 | LUCK | 命运眷顾程度 |

**派生属性**：
- **生命值（HP）** = (CON + SIZ) / 10（向下取整）
- **魔法值（MP）** = POW / 5
- **理智值（SAN）** = POW（初始值）
- **伤害加值（DB）** 和 **体格（Build）**：根据 STR + SIZ 总和查表

## 4. 难度等级与成功等级

### 三种难度等级

| 难度等级 | 要求 | 说明 |
|----------|------|------|
| **常规** | 骰出 <= 技能值 | 对胜任者构成挑战的一般情形（最常用） |
| **困难** | 骰出 <= 技能值 / 2 | 对专业人士构成挑战 |
| **极难** | 骰出 <= 技能值 / 5 | 对专家构成挑战，触及人类极限 |

### 六种成功等级（从高到低）

| 等级 | 条件 |
|------|------|
| **大成功** | 骰出 01 |
| **极难成功** | 骰出 <= 技能值 / 5 |
| **困难成功** | 骰出 <= 技能值 / 2 |
| **常规成功** | 骰出 <= 技能值 |
| **失败** | 骰出 > 技能值（非大失败） |
| **大失败** | 骰出 100；或当上限 < 50 时骰出 96-100 |

### 大成功与大失败

- **大成功（01）**：不仅达成目标，还有额外好运发生（如战斗中造成最大伤害）
- **大失败**：比单纯失败更糟糕的事情发生
  - 若需要骰出的上限 >= 50：骰出 100 为大失败
  - 若需要骰出的上限 < 50：骰出 96-100 均为大失败

## 5. 技能检定基础

### 基本流程

1. 玩家声明目标（"我要做什么"）
2. 守秘人决定使用哪个技能/属性，以及难度等级
3. 玩家掷 D100
4. 比较结果与目标值，判定成功/失败
5. 守秘人与玩家共同描述结果

### 对手/环境决定难度

当面对活物（NPC或怪物）时，根据对手的技能/属性值设定难度：

| 对手技能/属性值 | 难度等级 |
|----------------|----------|
| < 50 | 常规 |
| >= 50 | 困难 |
| >= 90 | 极难 |

### 人性极限

- 调查员是普通人，存在物理极限
- 极难成功的上限是 **100 + 调查员的技能/属性值**
- 超过此上限的事情对角色不可能，不允许检定
- 超越极限的唯一方法：**复数调查员协作**（多人属性值累加以降低对手属性值）

## 6. 特殊机制

### 6.1 孤注一掷（Push a Roll）

当检定失败后，玩家可以请求**孤注一掷**——再试一次，但失败后果更严重。

- 必须合情合理地描述如何额外努力
- 时间总会在检定间流逝（几秒到几小时）
- **不可用于**：幸运检定、理智检定、战斗检定、伤害/理智损失骰
- **成功**：目标达成，无失败后果
- **失败**：守秘人可自由施加后果（伤害、理智损失、装备损失、被俘等）

### 揭示预兆（Foreshadowing）

在孤注一掷前，守秘人可告知玩家失败的后果（"若你失败……"），让玩家衡量风险。

### 6.2 奖励骰与惩罚骰

当环境造成显著优势/劣势时使用：

- **奖励骰**：额外投一个十位骰，取**较低**的十位结果
- **惩罚骰**：额外投一个十位骰，取**较高**的十位结果
- 一个奖励骰与一个惩罚骰互相抵消
- 通常最多使用2个

**使用原则**：
- 技能检定：通过**难度等级**反映有利/不利因素
- 对抗检定：通过**奖罚骰**反映优势/劣势

### 6.3 对抗检定

双方同时掷骰，比较成功等级：

- 大成功 > 极难成功 > 困难成功 > 常规成功 > 失败/大失败
- 平手时，技能/属性更高者获胜
- 仍平手则僵局或重骰
- 对抗检定**不可**孤注一掷

### 6.4 组合技能检定

某些情况要求使用多个技能：
- 只掷一次骰子，与每个技能分别比较
- 守秘人决定是需要所有技能都成功，还是任一成功即可

### 6.5 幸运检定

当没有适用的技能/属性，由命运之手决定时使用：
- 骰出 <= 幸运值即成功
- 团体检定由幸运值最低者掷骰

### 6.6 智力/灵感检定

- **智力检定**：骰出 <= INT 值，用于破解智力谜题
- **灵感检定**：同样骰出 <= INT，用于调查陷入僵局时获取提示

### 6.7 知识检定

- 骰出 <= EDU 值
- 代表角色教育背景中储存的常识性知识
- 如有更合适的技能则优先使用技能检定

---

## 7. 失败处理原则

- **避免突然终结**：尽量不要让失败导致游戏无法继续
- 失败检定一般不造成伤害或理智损失（除非孤注一掷）
- 替代方案：挫折、被俘、交易、昏厥等
- 只有在孤注一掷时才考虑严重后果

---

## 8. 技能成长（幕间成长）

模组/章节结束后进行：

1. 检查角色卡上有标记（成功使用过）的技能
2. 每个有标记的技能掷 1D100
3. 若骰出 **大于** 当前技能值（或 >= 96），该技能增加 1D10
4. 清除所有标记

**特殊规则**：
- 技能达到 90% 或更高时，额外获得 **2D6 理智值** 奖励
- 克苏鲁神话和信用评级永远不会获得成长标记

---

## 9. 社交技能检定

社交技能包括：取悦、话术、恐吓、说服

**难度调整**：
- NPC 倾向支持玩家目标：直接同意，无需检定
- NPC 无强烈感受：标准难度
- NPC 强烈反对：提高1-2级难度
- 玩家提供了有力支持（如切实论点、展示武器）：降低1级难度

## 10. 快速参考表

### 检定结果速查

```
D100结果 vs 目标值:
  01        → 大成功（必定成功）
  <= 目标/5 → 极难成功
  <= 目标/2 → 困难成功
  <= 目标   → 常规成功
  > 目标    → 失败
  96-100    → 大失败（当目标值 < 50 时）
  100       → 大失败（当目标值 >= 50 时）
```

### 对抗检定胜负速查

```
大成功 > 极难成功 > 困难成功 > 常规成功 > 失败 > 大失败
同级比较：技能/属性更高者胜 → 仍同则僵局
```

### 骰子表示法

| 符号 | 含义 |
|------|------|
| D4 | 4面骰 |
| D6 | 6面骰 |
| D8 | 8面骰 |
| D10 | 10面骰 |
| D100 | 百分骰（两个D10） |
| 2D6 | 掷两个6面骰并相加 |
| 1D6+1 | 掷一个6面骰并+1 |
