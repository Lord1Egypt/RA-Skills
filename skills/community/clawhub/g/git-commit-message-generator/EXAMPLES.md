## Bug Fix Examples

### Fixing Race Condition

**Changes:**
```diff
# src/services/cache.ts
- async get(key: string) {
+ private locks = new Map<string, Promise<any>>();
+
+ async get(key: string) {
+   if (this.locks.has(key)) {
+     return this.locks.get(key);
+   }
+
+   const promise = this._fetch(key);
+   this.locks.set(key, promise);
+
-   return this._fetch(key);
+   try {
+     return await promise;
+   } finally {
+     this.locks.delete(key);
+   }
  }
```

**Generated Commit:**
```
fix(cache): prevent concurrent fetches for same key

Adds mutex locking to ensure only one fetch happens per cache
key at a time. Fixes race condition where concurrent requests
for the same uncached key would trigger multiple backend calls.

Reduces redundant API calls by ~40% under high concurrency.

Fixes #892
```

```diff
+   return { accessToken, refreshToken, expiresIn };

+   deviceId?: string;
+ }
+

+   accessToken: string;
+   refreshToken: string;
+   expiresIn: number;
+ }
```

**Generated Commit:**
```
feat(auth)!: change login API to return auth result object

Updates login function to accept credentials object and return
structured auth result with access token, refresh token, and
expiration time. Enables future extensions like device tracking.

BREAKING CHANGE: login() signature changed from
login(email, password) to login({ email, password }).
Return value changed from string to AuthResult object.

Migration:
- Before: const token = await login(email, password);
- After: const { accessToken } = await login({ email, password });
```

## Tips for Getting Better Messages

### ✅ DO: Provide Context

```
"Generate commit - this fixes a null pointer exception in the payment flow"
"Create commit for adding Stripe integration"
"Make commit message - refactoring to use TypeScript strict mode"
```

### ✅ DO: Stage Related Changes Together

```bash
# Good: All auth-related changes together
git add src/auth/login.ts src/auth/session.ts src/types/auth.ts

# Bad: Mixing unrelated changes
git add src/auth/login.ts src/components/Button.tsx
```

### ❌ DON'T: Stage Too Many Unrelated Changes

```bash
# Bad: Too broad, hard to describe
git add .

# Better: Stage by feature/fix
git add src/components/
git commit ...

git add src/api/
git commit ...
```

### ✅ DO: Ask for Specific Type

```
"Generate a 'fix' type commit"
"Create a 'refactor' commit message"
"Make a 'perf' commit for these optimizations"
```

## Summary

These examples show the wide variety of commit messages this skill can generate. The key is to:

1. **Stage related changes together**
2. **Provide context when needed**
3. **Review and refine the generated message**
4. **Keep commits atomic and focused**

The skill learns from your project's patterns and adapts its messages to match your style over time.
