# 机制本创作指南

## 机制本的定义

机制本的核心体验是"游戏性"——通过特殊的游戏机制驱动，让玩家的每个选择都有意义。

## 机制类型

### 探索机制

**特点**：玩家通过探索获取信息。

**设计要点**：
- 有地图或场景列表
- 玩家选择去哪个地点
- 不同地点有不同发现
- 某些地点需要特定条件才能进入

**示例**：
```
地点列表：
- 客厅：可发现信件、照片
- 书房：可发现日记、钥匙
- 厨房：可发现毒药、刀具
- 花园：可发现脚印、血迹
- 阁楼：需要钥匙才能进入，有关键证据
```

### 交易机制

**特点**：玩家之间可以交换资源。

**设计要点**：
- 有可交易的资源（线索、金钱、道具）
- 交易可以撒谎（假线索）
- 有稀缺资源，制造竞争

**示例**：
```
资源类型：
- 线索卡：可交易，但交易后对方知道你有这张卡
- 金币：可用来购买额外行动
- 道具：有特殊功能（如偷看别人的线索）

交易规则：
- 每轮可交易一次
- 交易不可撤销
- 可以用假线索交易，但被发现有惩罚
```

### 投票机制

**特点**：玩家通过投票做出集体决策。

**设计要点**：
- 投票结果影响剧情走向
- 有多种投票类型（处决、赦免、选择）
- 投票可以有策略（如投票给自己）

**示例**：
```
投票类型：
- 处决投票：每轮结束投票处决一人
- 任务投票：对某个决策投票
- 秘密投票：匿名投票，不知道谁投了谁
```

### 数值机制

**特点**：玩家有数值属性，影响游戏进程。

**设计要点**：
- 有生命值、能量值、信任值等
- 数值变化有规则
- 数值影响结局

**示例**：
```
属性系统：
- 生命值：初始100，被攻击减少，归零出局
- 信任值：初始50，被怀疑减少，影响投票
- 能量值：初始3，每回合恢复1，用于执行行动
```

## 机制本设计要点

### 机制与剧情的结合

机制不能脱离剧情，必须服务于故事。

**结合方法**：
- 机制是世界观的一部分（如"超能力"机制）
- 机制影响剧情走向（选择不同，结局不同）
- 机制本身有故事（机制的来源有解释）

### 机制平衡

**平衡原则**：
- 不能有"必胜策略"
- 随机性和策略性要平衡
- 弱者有翻盘机会

### 机制复杂度

**控制原则**：
- 规则要能在10分钟内讲清楚
- 有快速参考卡
- DM熟悉规则，能快速解答

## 机制本结构模板

### 标准结构（4-6小时）

```
第一幕：规则讲解 + 角色确认（30-40分钟）
  - DM讲解游戏机制
  - 玩家阅读剧本
  - 确认初始状态

第二幕：游戏进行（120-180分钟）
  - 多轮游戏，每轮包含：
    - 自由行动阶段（探索/交易/私聊）
    - 公共阶段（讨论/投票）
    - 结算阶段（更新状态）

第三幕：终局（30-40分钟）
  - 最终对决/投票
  - 胜负判定
  - 剧情揭示

第四幕：复盘（15-20分钟）
  - 揭示隐藏信息
  - 讨论策略
```

## 机制本示例：6人机制本

### 《超能力杀人事件》（示例）

**类型**：机制推理本
**人数**：6人（不限男女）
**时长**：4-5小时
**背景**：未来·2077年·超能力研究所

**故事梗概**：
2077年，超能力研究所发生命案，研究员被发现死在实验室。所有在场的六个人都有超能力，每个人的能力都可能是凶器。在互相猜忌中，他们必须找出真凶，同时隐藏自己的能力秘密。

**机制设计**：
- **超能力系统**：每人有独特超能力，每回合可使用一次
- **能量值**：使用超能力消耗能量，能量用完不能使用
- **能力鉴定**：可以花费能量鉴定某人是否使用了超能力
- **投票处决**：每轮结束投票处决一人

**超能力列表**：
| 角色 | 超能力 | 效果 |
|------|--------|------|
| A | 心灵感应 | 偷看一人的一张线索卡 |
| B | 时间暂停 | 跳过一次针对自己的行动 |
| C | 物质穿透 | 可以进入任何密闭空间 |
| D | 分身术 | 有不在场证明（分身在别处） |
| E | 记忆篡改 | 可以修改一人的记忆（改变证词） |
| F | 预知未来 | 可以看到下一轮的一个事件 |

**胜利条件**：
- 好人阵营：找出凶手并投票处决
- 凶手阵营：活到最后或只剩2人
