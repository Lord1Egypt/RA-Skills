# 阶段2-3：策略分析与大纲生成

## ⚡ 执行指引（Agent必读）

### 如何计算分值权重和分配篇幅
1. 从解析结果中获取 `scoringCriteria` 数组
2. 计算每个维度的权重百分比：`weight% = maxScore / totalScore * 100`
3. 按权重分配建议字数：`建议字数 = 总目标字数 × (weight% × 1.2)`（高分值章节额外倾斜20%）
4. 确保权重最高的3个维度至少占全文60%篇幅

### 大纲质量自检
生成大纲后，快速验证：
- 有无评分项遗漏 → 对照 `scoringCriteria` 逐条核对
- 有无技术要求遗漏 → 对照 `technicalParameters` 逐条核对
- 总分加起来是否等于招标文件规定的总分

### 输出规范
- 大纲必须使用 Markdown 层级结构（`#` `##` `###`）
- 每个标题后紧跟 `（对应评分项：[维度]，分值：[X]分，建议字数：[X]字）`
- 如果用户提供的自身信息不足→ 在策略报告中标注"以下策略基于通用假设，建议补充企业资质和业绩信息"

## 目标
基于招标解析结果，制定评分策略并生成评分导向的完整应标书大纲。

## 输入
- 阶段1输出的解析报告（含评分标准、技术要求等）
- 企业自身信息（用户提供的优势、资质、案例等）

## 第一部分：策略分析

### 评分权重分析
分析评分标准中各维度的分值分布，确定重点投入方向：

| 评分维度 | 分值 | 权重 | 优先级 | 建议篇幅占比 |
|---------|------|------|--------|------------|
| [技术方案] | [40] | [40%] | ⭐⭐⭐ | [40%-50%] |
| [施工组织] | [20] | [20%] | ⭐⭐ | [15%-20%] |

### 竞争优势分析
根据自身优势确定差异化亮点：

1. **优势识别**：哪些评分维度我司具备明显优势？
2. **差异化策略**：如何在标书中突出这些优势？
3. **劣势规避**：哪些方面是弱项？如何合理规避或弥补？

### 资源缺口识别
- 缺少哪些资质/证书/业绩？
- 能否通过联合体投标、分包等方式弥补？
- 是否需要提前准备补充材料？

### 报价策略建议
- 基于评分规则，分析价格分占比
- 报价高低对总得分的影响评估
- 建议报价区间

## 第二部分：大纲生成

### 大纲结构要求

基于解析结果和中国标准应标书格式，生成完整大纲。

**格式规范：**
- 使用 Markdown 层级结构
- 每个一级标题标注对应评分项和分值
- 每个章节标注建议字数
- 必须覆盖招标文件"技术要求"章节的全部条目
- 大纲深度至少到三级标题

### 大纲模板框架

```
# 第一章 [章节名称]（对应评分项：[评分项]，分值：[X]分，建议字数：[X]字）
## 1.1 [二级标题]
### 1.1.1 [三级标题]
  - 要点1：[内容提示]
  - 要点2：[内容提示]
## 1.2 [二级标题]
...
```

### 分值-篇幅对照规则

| 分值占比 | 建议篇幅占比 | 说明 |
|---------|------------|------|
| ≥30%    | 40%-50%    | 核心章节，重点投入 |
| 15%-29% | 15%-25%    | 重要章节，充分展开 |
| 5%-14%  | 8%-12%     | 一般章节，适度展开 |
| <5%     | 3%-5%      | 次要章节，简要说明 |

### 大纲验证

生成大纲后，请逐项验证：
- [ ] 是否每个评分项都有对应章节？
- [ ] 是否覆盖招标文件所有技术要求？
- [ ] 各章节篇幅分配是否与分值比例匹配？
- [ ] 大纲结构逻辑是否清晰（技术方案→实施方案→保障措施）？
- [ ] 是否有冗余或重复内容？

## 输出

向用户展示完整大纲，请用户确认后再进入写作阶段。
