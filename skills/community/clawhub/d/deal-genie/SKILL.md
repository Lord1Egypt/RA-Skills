# 带货精灵 - 商品全链路运营 v2.0

---

## 概述

输入一个商品 → 输出「该不该推 → 怎么推 → 找谁推 → 推出去 → 追回来 → 复盘优化」的完整闭环方案。

12 步全周期流水线，覆盖达人分销从选品到复盘的全部环节，支持分步调用或一键全案输出。

## 四阶段流水线

```
决策层(1-3)：选品决策 → 策略定位 → 合规审查
内容层(4-7)：多平台文案 → AI生图提示词 → HTML报告 → 素材工坊
执行层(8-10)：达人匹配 → 寄样管理 → 推品执行
增长层(11-12)：追单转化 → 数据复盘
```

## 阶段一：决策层

### 1. 选品决策

四维评分，给出明确结论（推 / 观望 / 不推）：

| 维度 | 权重 | 评估内容 |
|------|------|----------|
| 利润空间 | 30% | 佣金率、客单价、预估毛利 |
| 转化潜力 | 25% | 品牌力、类目竞争度、价格优势 |
| 合规风险 | 20% | 广告法违禁词、类目敏感度 |
| 操作难度 | 25% | 是否需要寄样、素材是否齐全 |

详见 `references/scoring_matrix.md`。

### 2. 策略定位

根据选品结论，确定推广主攻方向：

| 选品特征 | 策略方向 | 文案角度 |
|----------|----------|----------|
| 佣金高 + 品牌弱 | 利益驱动 | 强调 ROI、出单快 |
| 品牌强 + 佣金低 | 品牌溢价 | 强调正品、口碑 |
| 价格低 + 竞争激烈 | 价格锚定 | 对标竞品、性价比 |
| 成分/技术独特 | 认知教育 | 科普种草、差异化 |
| 新品类/新品牌 | 信任建设 | 检测报告、达人背书 |
| 佣金低 + 品牌弱 + 竞争高 | **不建议推** | 告知风险，停止后续 |

详见 `references/strategy_playbook.md`。

### 3. 合规审查

事前扫描违禁词，生成「合规红线清单」：广告法极限词、类目敏感词、功效夸大、虚假宣传。

详见 `references/compliance_rules.md`。

---

## 阶段二：内容层

### 4. 多平台文案

统一策略角度，适配各平台习惯：

- **抖音口播脚本**（2-3 分钟逐字稿）
- **小红书/图文种草**（200-300 字）
- **社群招商话术**（突出佣金和卖点）
- **朋友圈战报**（50-80 字）

详见 `references/copy_playbook.md`。

### 5. AI 生图提示词

只输出提示词，不调用生图 API。每种配图出中文版（即梦/通义万相）和英文版（Midjourney/DALL-E），覆盖种草图 + 促销图两个场景。

### 6. HTML 分析报告

打包为可视化 HTML，含选品评分表、合规审查、完整文案（每套带「一键复制」）。配图提示词用独立卡片区展示。主题色根据策略方向自动切换（利益驱动=橙金、品牌溢价=深蓝、价格锚定=红、认知教育=绿、信任建设=紫）。

### 7. 素材工坊

文案落地为可直接使用的素材：

- **短视频分镜脚本**（4-6 镜头，含画面描述、口播、时长、运镜、黄金前3秒钩子）
- **直播间贴片文案**（价格牌、卖点牌、福利牌、信任牌）
- **详情页优化建议**（标题、主图方向、10屏详情结构）
- **推广海报文案**（主标题+副标题+行动号召，适配 9:16 / 1:1）

详见 `references/material_workshop.md`。

---

## 阶段三：执行层

### 8. 达人匹配

基于商品特征，推荐最优达人类型 + 筛选标准 + 3 套邀约话术（初次触达/跟进/合作确认）。跟进话术覆盖「已读不回/犹豫中/讨价还价」三种场景。

详见 `references/influencer_matching.md`。

### 9. 寄样管理

寄样全流程标准化：自动生成寄样清单（含样品数量、随样物料）、5 节点跟进话术（寄出当天→签收后1天→3天→推品日→7天未推）、HTML 追踪表（含签收/推品状态标记）。

详见 `references/tracking_management.md`。

### 10. 推品执行

推品日的完整作战计划：类目 × 时间段发布策略、渠道优先级排序、5-8 分钟直播脚本大纲（钩子→痛点→展示→价格→逼单）、预热文案（前1天/1小时/10分钟）。

详见 `references/launch_execution.md`。

---

## 阶段四：增长层

### 11. 追单转化

推品后的二次转化和口碑裂变：4 节点追单话术（当天→1天→3天→7天）、4 种限时活动文案、晒单引导策略、4 类差评应对模板（品质/物流/期望/恶意）。

详见 `references/conversion_playbook.md`。

### 12. 数据复盘

用户输入出单数据 → 自动计算 ROI、行业对标、环节诊断（定位选品/达人/文案/节奏/追单哪个环节出问题）、生成改进清单。复盘结果反哺下次选品，越用越准。

详见 `references/data_review.md`。

## 飞书集成（可选）

> 用户提供飞书 webhook 地址后启用。未提供时功能完整不受影响。

### 四个触发点

| 时机 | 自动推送内容 | 接收方 |
|------|-------------|--------|
| 选品分析完成 | 商品名 + 选品结论 + 策略方向 + 核心数据摘要 | 飞书群 |
| 推品执行日 | 推品检查清单 + 分镜脚本 + 预热文案 | 飞书群 |
| 追单节点（1/3/7天） | 当天适用的追单话术提醒 | 飞书群 |
| 数据复盘完成 | ROI 数据 + 环节诊断 + 改进清单 | 飞书文档 |

### 配置方式

用户说"接入飞书"时，AI 引导输入 webhook 地址（飞书群 → 设置 → 群机器人 → 添加 → 复制 webhook）。配置后，后续分析结果自动推送。

详见 `references/feishu_integration.md`。

## 使用原则

1. **结论先行**：任何时候先给明确结论再展开
2. **文案和策略绑定**：不脱离策略生成文案
3. **合规内生**：构思时就避开雷区，非事后检测
4. **信息不全主动追问**：佣金/类目/品牌缺失时追问后再继续
5. **中文输出**：所有输出使用简体中文
6. **零依赖 + 可选飞书**：核心功能不依赖外部 API；提供飞书 webhook 即可自动推送报告和提醒
7. **模块可拆**：12 步任意触发，不必走全流程
