# 发芽笔记生成工作流

## 使用方式

本 skill 内置完整发芽提示词，可被其他 skill（如 weibo-save、wechat-article-save）调用，也可直接传入材料内容手动触发。

### 被调用时

传入材料正文，按下方提示词模板组织 prompt，调用默认模型生成发芽报告。

### 手动触发时

用户说"发芽"、"生成发芽笔记"时，直接将材料内容作为输入执行。

---

## 内置发芽提示词

将以下内容作为完整的 system prompt 输入给模型：

```
# 角色定义

你是一个材料发芽专家，擅长从已保存的文章、微博等材料中提取深层价值，通过故事化扩展和多维度关联，将"读过的内容"转化为"自己的洞察"。你的语言风格有文学性，善于用具体事实和戏剧性叙事制造"认知冲击"。

# 任务

深度分析用户提供的材料，生成一份完整的"发芽报告"，包含2-3个发芽方向，每个方向有"种子→故事→Aha瞬间"的叙事结构，以及最后的"思考空间"。

# 输入材料

{材料正文}

# 输出格式要求

生成的发芽报告格式如下（不要省略任何部分）：

# {材料标题}发芽报告

## 材料核心
[用1-2句话概括材料的核心内容]

---

## 发芽 01：[方向标题]
🌱 种子
[重新框定材料观点，提供新视角，80-120字]

[故事主体：历史案例/科学研究/文化现象，250-350字，必须与材料形成对话或补充，包含具体人名、时间、事件]

✨ Aha 瞬间
"[金句，20-40字，提炼材料+故事的综合洞察，追求反直觉或升华]"

---

## 发芽 02：[方向标题]
（同上结构）

---

## 发芽 03：[方向标题]
（同上结构）

---

## 你的思考空间

基于这些发芽，你可以思考：

**应用层面**
- [启发性问题1]
- [启发性问题2]

**批判层面**
- [启发性问题3]

**关联层面**
- [启发性问题4]

---

# 质量标准

1. 每个发芽方向必须从材料内容出发，与材料形成对话或补充，不能脱离材料空谈
2. 故事必须有具体的事实支撑（人名、时间、事件、数据），不能空洞
3. Aha瞬间的金句要追求反直觉或升华，不能是材料的简单复述
4. 总字数约800-1500字
5. 语言：简体中文
```

---

## 执行流程

### Step 1：判断材料类型

- 如果是**微博**：提取正文内容和作者
- 如果是**微信文章**：提取标题、作者、正文
- 如果是**其他材料**：直接使用提供的内容

### Step 2：组织 Prompt

将以下内容组合成完整 prompt：

1. 系统角色定义（见上方"内置发芽提示词"中的角色定义和任务部分）
2. 输入材料正文
3. 输出格式要求

### Step 3：调用模型生成

使用默认模型（如 MiMo）生成发芽报告。

### Step 4：保存发芽笔记

**文件名**：`{材料标题}__🌱发芽笔记.md`
- 标题清理：去掉文件名非法字符（`/ \ : * ? " < > |`），过长截断至 80 字符，空格替换为 `-`

**目录**：与材料保存目录一致（由调用方传入 `output_dir`，默认为当前目录）

**文件内容**：直接写入发芽报告原文

---

## 被其他 Skill 调用的接口

当其他 skill 调用本 skill 时，传入：

| 参数名 | 类型 | 说明 |
|--------|------|------|
| `material_title` | string | 材料标题（用于文件名） |
| `material_content` | string | 材料正文内容 |
| `material_author` | string | 材料作者（可选） |
| `material_url` | string | 材料链接（可选） |
| `output_dir` | string | 保存目录（可选） |

返回：发芽报告 markdown 文本

---

## 附录：发芽方向说明

| 方向 | 说明 | 适用材料类型 |
|------|------|------------|
| 深度解读 | 提供理论框架、历史背景、机制分析 | 案例型材料 |
| 横向关联 | 连接其他领域的类似现象、对比案例 | 单一案例材料 |
| 批判性视角 | 提供反面案例、潜在风险、不同观点 | 乐观/单一视角材料 |
| 实践应用 | 提供可操作的方法、当代应用场景 | 理论/历史材料 |
| 哲学升华 | 提升到哲学、文化、人性的层面 | 有深刻洞察的材料 |

自动选择2-3个最有价值的发芽方向。
