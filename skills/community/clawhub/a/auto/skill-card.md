## Known Risks and Mitigations: <br>
Risk: Generated automation may run commands repeatedly or on a schedule, so an incorrect script or schedule can repeat unwanted actions. <br>
Mitigation: Review generated commands, test manually or with dry runs where possible, and start with narrow schedules before enabling unattended execution. <br>
Risk: Automation examples may involve credentials for scripts, deployments, or notifications. <br>
Mitigation: Store secrets in environment variables or a secret manager, and avoid embedding passwords or API keys in scripts or task definitions. <br>
