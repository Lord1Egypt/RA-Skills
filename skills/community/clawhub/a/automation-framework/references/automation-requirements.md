## 自动化任务方法论要求清单

自动化任务领域每种任务类型的"零件清单"——必选/可选组件、组装顺序、领域约束。按清单逐项产出。

---

### 自动化任务数据结构规范

#### 自动化任务字段定义

| 字段 | 类型 | 必选 | 说明 |
|------|------|------|------|
| id | string | 是 | 自动化任务唯一标识符，格式：AUTO-{序号} |
| name | string | 是 | 自动化任务名称，最大100字符 |
| description | string | 是 | 自动化任务描述，最少10字符 |
| scheduleType | string | 是 | 调度类型：once/recurring |
| scheduledAt | string | 否 | 一次性任务执行时间，格式：YYYY-MM-DDTHH:mm:ss |
| rrule | string | 否 | RFC 5545格式的调度规则 |
| validFrom | string | 否 | 有效期开始时间，格式：YYYY-MM-DDTHH:mm:ss |
| validUntil | string | 否 | 有效期结束时间，格式：YYYY-MM-DDTHH:mm:ss |
| status | string | 是 | 任务状态：ACTIVE/PAUSED/COMPLETED/FAILED |
| errorHandling | object | 否 | 错误处理策略 |
| resources | object | 否 | 资源限制 |
| metadata | object | 否 | 自定义元数据 |

#### RRULE语法说明

```
FREQ=frequency;INTERVAL=interval;BYDAY=days;BYMONTH=months
```

**频率类型**：
- DAILY：每天
- WEEKLY：每周
- MONTHLY：每月
- YEARLY：每年

**示例**：
- 每天：FREQ=DAILY
- 每周一：FREQ=WEEKLY;BYDAY=MO
- 每月1日：FREQ=MONTHLY;BYMONTHDAY=1
- 每年1月1日：FREQ=YEARLY;BYMONTH=1;BYMONTHDAY=1

### AT0 自动化协调域

#### AT0-01 自动化需求分析
- **必选组件**: 自动化目标、自动化范围、自动化需求、自动化约束
- **可选组件**: 自动化优先级、自动化时间要求
- **组装顺序**: 目标确定→范围界定→需求分析→约束识别→需求汇总
- **约束**: 自动化目标必须明确；自动化范围必须合理；自动化需求必须全面
- **格式**: 自动化需求分析报告（Markdown）

#### AT0-02 自动化域选择
- **必选组件**: 可用自动化域列表、自动化域选择理由、自动化域组合方案
- **可选组件**: 自动化域依赖关系说明
- **组装顺序**: 需求分析→自动化域扫描→选择决策→组合设计
- **约束**: 自动化域选择必须基于自动化需求，不可强行套用
- **格式**: 自动化域选择报告（Markdown）

#### AT0-03 执行顺序确定
- **必选组件**: 自动化任务依赖关系分析、执行顺序方案、关键路径识别
- **可选组件**: 并行执行可能性分析
- **组装顺序**: 依赖分析→顺序确定→关键路径识别
- **约束**: 执行顺序必须尊重自动化任务依赖关系
- **格式**: 执行顺序图（Markdown流程图）

#### AT0-04 结果整合与优化
- **必选组件**: 各自动化任务执行结果汇总、整合方案、优化建议
- **可选组件**: 整合效果评估
- **组装顺序**: 结果收集→整合分析→优化建议
- **约束**: 整合必须保持各自动化任务输出的完整性
- **格式**: 整合报告（Markdown）
