💡 领域专家评点

阿福专家视角：这套 BOM 规则的本质是什么？

  • 核心目标：明确物权归属，划清责任边界，优化供应链协同效率
  • 设计哲学：主机厂抓大放小——只管控最终总成（B+HC），子零件放权给供应商
  • 商务智慧：通过 VAA 双经销解决专利/商务壁垒，通过 DB 指定锁定关键供应商
  • 财务影响：毛需求只算 B+HC，避免重复计算，精准反映真实采购需求
  • 风险管控：C/D/S 物权在供应商，主机厂不承担库存风险，只收最终成果
📊 最全总成件毛需求量 = B（外购总成） + HC（委外总成）

🔍 深度洞察

洞察 1：为什么 C 不算毛需求？

物权视角：C 的物权从头到尾都在供应商手中，主机厂从未拥有过 C，自然不需要为 C 做需求计划。
业务视角：主机厂要的是 HC 总成（装配完成的成果），不是 C 零件（中间原材料）。就像你买电脑，要的是整机，不是 CPU/内存/硬盘这些散件。
财务视角：如果 C 和 HC 都算，就重复计算了——C 的价值已经包含在 HC 里了。

洞察 2：VAA 双经销的本质是什么？

商务过桥：当专利/商务壁垒导致总成商无法直接采购零件 A 时，主机厂充当"中间商"——先买入 A，再卖给总成商。
风险隔离：通过主机厂经手，规避了总成商与零件 A 供应商之间的直接交易障碍。
财务处理：虽然 V 要经手，但最终归入 B 类需求（因为 V 本身就是采购级总成）。

洞察 3：DB 件的"指定"意味着什么？

权力博弈：主机厂通过指定供应商/定价，保留对关键子零件的控制权，但不承担采购执行责任。
供应链管理：防止总成商为了降低成本而选用劣质子零件，确保产品质量。
责任划分：虽然主机厂指定，但采购合同是总成商与 DB 供应商签的，出了问题找总成商。

🎯 行动建议

如何将这套知识应用到实际工作？

  • 需求计算：只统计 B+HC，其他物料类型自动过滤，避免重复计算
  • 供应商管理：对 C/D/S 放权，对 B/HC 严格管控，对 V 做好经手流程
  • 合同设计：DB 件签三方合同，VAA 件签买卖 + 转卖合同，CSMT 签委外加工合同
  • 系统配置：ERP 中设置物料类型自动识别规则，毛需求自动计算 B+HC
  • 风险管控：C/D/S 物权在供应商，主机厂不承担库存风险，但需监控 HC 质量

🧠 记忆口诀

阿福速记法

  • 毛需求：B 加 HC，其他不算
  • 物权：B/HC/V 归我，C/D/S 归供应商
  • 层级：只有 D/S/C 是子件，其他都是采购级
  • 流转：C 是委外专用，D 是指定专用，S 是随便买的
  • VAA：我买我卖我经手，最后归入 B 类走
