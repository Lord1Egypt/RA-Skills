## 📝 使用示例

### 示例 1：完整版分析（推荐新手）

**你这么说**：
```
帮我分析一下 Tailwind CSS 这个项目
https://github.com/tailwindlabs/tailwindcss
```

**我会这么做**：
1. 先把项目页面扒一遍（Star数、更新时间、README 啥的）
2. 去网上搜搜别人怎么评价它的
3. 找几个同类竞品（Bootstrap、Material UI 这些）
4. 按照 10 大模块给你整一份完整报告

**你会得到**：
- 一份超详细的分析报告，包含所有 10 个模块
- 大白话解释，连隔壁大妈都能看懂
- 每个模块末尾的「一句话结论」
- 适合快速扫一眼就 get 到重点

---

### 示例 2：快速版分析（老手/赶时间用）

**你这么说**：
```
快速分析一下这个项目
https://github.com/vuejs/core
```

**我会这么做**：
只输出 4 个核心模块：
- 模块 1️⃣：一句话总结
- 模块 2️⃣：项目名片
- 模块 5️⃣：上手难度评测
- 模块 9️⃣：综合评分

**你会得到**：
- 3 分钟就能看完的精简版报告
- 适合对技术有一定了解，只需要快速判断值不值得入坑
- 决策用报告，细节不多但核心信息全

---

### 示例 3：指定关注维度（你有特别想了解的）

**你这么说**：
```
帮我分析 Next.js，特别关注一下团队协作和多人开发场景
https://github.com/vercel/next.js
```

**我这么做**：
1. 正常分析流程走一遍
2. 重点在「目标用户」「落地建议」等模块里加戏
3. 专门聊聊多人协作怎么玩、有没有版本控制支持、团队开发体验如何

**你会得到**：
- 完整版报告
- 额外增加对你关注维度的深入分析
- 比如问「适不适合数据分析师」，我会重点看数据处理相关功能

## ⚠️ 注意事项

1. **我能做的**：帮你扒页面、搜评价、找竞品、写报告
2. **我做不到的**：
   - 实际运行代码（但能基于文档推断）
   - 帮你安装（但能告诉你怎么装、可能有哪些坑）
   - 预测未来（但能基于数据做合理推测）

3. **建议配合使用**：
   - 拿到报告后，自己去官方文档确认一下细节
   - 遇到具体问题，去 GitHub Issue 搜搜有没有人遇到过
   - 决定用之前，先跑个 Demo 试试水
