# DSTE 端到端战略管理体系

## DSTE 概述

**DSTE（Develop Strategy to Execution，开发战略到执行）** 是华为17个一级流程之一，是华为战略管理的核心框架。DSTE承载的是企业的"领导价值流"，使企业在未来竞争中处于领导地位。

### DSTE 的三种角色

- **作为流程**：完整澄清组织战略从制定到执行的业务脉络和管控视图
- **作为方法**：帮助理解现状，勾勒可实现的未来，将愿景变成可执行的任务
- **作为文化**：承载企业管理持续追求发展、追求卓越的价值导向

## DSTE 四大阶段

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  战略制定    │ → │  战略解码    │ → │ 执行与监控   │ → │  战略评估    │
│  (SP)       │    │  (BEM)      │    │  (IBP/KPI)  │    │  (复盘)     │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
```

### 阶段一：战略规划 (SP — Strategy Plan)

- **时间跨度**：从下一年度开始的5年滚动规划
- **启动时间**：每年4月（春季），至9月完成
- **核心方法**：BLM（业务领先模型）
- **主要输出**：机会点业务设计、中长期战略规划（3年方向/财务预测/市场策略等）
- **关键活动**：市场洞察→战略方向→战略机会点→业务设计→战略里程碑

### 阶段二：年度业务计划与预算 (BP — Business Plan)

- **时间跨度**：下一个财政年度
- **启动时间**：每年10月（秋季），至次年3月完成
- **核心方法**：BEM（业务执行力模型）
- **SP→BP六大输入**：
  1. 市场空间/机会 → 机会点到订货目标
  2. 战略优先级 → 投资组合指导
  3. 战略举措 → 年度重点工作
  4. 战略目标 → 年度KPI与PBC
  5. 人力规划 → 人力预算
  6. 战略预算 → 全面预算

### 阶段三：管理执行与监控

核心管理对象：
- **IBP**（集成经营计划）：销售/研发等业务滚动计划
- **重点工作**：关键战略举措的落地监控
- **KPI管理**：组织绩效指标闭环
- **运营绩效**：仪表盘可视化管控
- **战略专题**：未来关键战略课题深度研究

例会机制：
- **经营分析会**（双周/月度）：围绕目标，发现差距，分析问题，解决问题
- **BP季度审视**：各季度运营指标回顾，发现新机会，调整资源配置

### 阶段四：战略评估

- 审视战略目标正确性
- 根据环境变化调整重点工作
- 阶段性复盘，提炼经验教训
- 组织、流程、激励机制评估

## DSTE 年度运营日历

| 时间 | 活动 | 主要内容 |
|------|------|---------|
| 4-9月 | SP制定 | BLM驱动中长期战略规划（未来5年） |
| 9-10月 | BP启动 | 全面预算评审，SP向BP输入 |
| 10-12月 | 机会点到订货 | 预算约束条件制定，人力预算管控 |
| 12月 | BP制定 | 投资组合规划，导出重点工作 |
| 次年1月 | 预算评审 | KPI目标值、PBC制定 |
| 次年3月 | KPI评审 | BP与述职、PBC沟通签署 |
| 全年 | 执行监控 | 双周经营分析会+季度战略健康度审视 |

## DSTE 核心价值

- 解决"战略规划与日常经营两张皮"问题
- 实现"力出一孔，利出一孔"的目标聚焦
- 构建组织的战略执行力肌肉记忆
