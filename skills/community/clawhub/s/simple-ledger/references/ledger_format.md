# 账本文件格式说明 (.csv)

## 概述

`.csv` 文件是纯文本格式，设计目标：**人类可读、易编辑、结构化**。

简单直观的 CSV 格式，一看就懂。

## 文件结构

```
# 注释行（以 # 开头）
# 日期,类型,金额,分类,描述,账户

[账户初始余额]
[交易记录]
```

## 格式详解

### 账户初始余额

```
余额,账户名,金额
```

示例：
```
余额,微信钱包,1000.00
余额,支付宝,2000.00
余额,银行卡,50000.00
余额,现金,500.00
```

规则：
- 每个账户最多一行余额记录
- 余额行以 `余额,` 开头，后面是账户名和金额
- 后续交易会增减这个余额

### 交易记录

格式：`日期,类型,金额,分类,描述,账户`

示例：
```
2026-05-21,支出,50.00,餐饮,午餐,微信钱包
2026-05-21,支出,299.00,购物,外套,支付宝
2026-05-01,收入,15000.00,工资,5月工资,银行卡
```

字段说明：
- **日期**：YYYY-MM-DD，必填，如 `2026-05-21`
- **类型**：只能是 `收入` 或 `支出`，必填
- **金额**：正数，必填，如 `50.00`（支出时填正数，系统自动识别为支出方向）
- **分类**：一级分类名，必填，如 `餐饮`、`交通`、`购物`、`居住`、`娱乐`、`通讯`、`医疗`、`教育`、`社交`、`其他`
- **描述**：简短说明，可选
- **账户**：支付/收款账户，必填，默认 `微信钱包`

### 注释行

```
# 这是注释行
# 日期,类型,金额,分类,描述,账户
```

### 空行

空行会被自动忽略，用于分隔不同逻辑块。

## 完整示例

```

余额,现金,500.00

# === 2026年5月 ===
2026-05-01,收入,15000.00,工资,5月工资,银行卡
2026-05-01,支出,35.00,餐饮,午餐外卖,微信钱包
2026-05-02,支出,128.00,餐饮,火锅,支付宝
2026-05-03,支出,23.50,交通,打车上班,微信钱包
2026-05-05,支出,1200.00,居住,5月房租,银行卡
2026-05-05,支出,15.00,餐饮,早餐,微信钱包

```

## 余额计算

某账户当前余额 = 初始余额 + 收入合计 - 支出合计

示例（微信钱包）：
```
余额  +1000.00
  +35.00（午餐）
  -23.50（打车）
  -15.00（早餐）
  -45.00（午餐）
  -30.00（话费）
─────────────
= 921.50
```

## 分类系统

| 分类 | 说明 |
|------|------|
| 餐饮 | 吃饭、外卖、奶茶、咖啡、零食等 |
| 交通 | 打车、地铁、公交、加油、高铁等 |
| 购物 | 衣服、数码、日用品、超市等 |
| 居住 | 房租、水电、物业、网费、燃气等 |
| 娱乐 | 电影、游戏、KTV、旅游等 |
| 通讯 | 话费、流量、手机套餐等 |
| 医疗 | 看病、买药、体检等 |
| 教育 | 课程、书籍、考试等 |
| 社交 | 红包、礼物、请客等 |
| 其他 | 无法归类的支出 |

收入分类：工资、奖金、理财收益、兼职、报销退款、其他

## 与旧版 .ledger 格式的区别

| 特性 | 新 CSV 格式 | 传统双行格式 |
|------|------------|------------------|
| 文件扩展名 | `.csv` | `.ledger` |
| 账户格式 | 扁平中文名 | 层级路径（如 `Assets:银行`）|
| 记账方式 | 单边记账（一行一笔）| 双行格式（借贷两条线）|
| 金额方向 | 类型字段区分 | 正负号区分 |
| 学习成本 | 极低 | 中等 |
| 语法要求 | 宽松（CSV格式）| 严格（指令+posting）|
| 适用场景 | 个人随手记 | 传统双行记账 |
