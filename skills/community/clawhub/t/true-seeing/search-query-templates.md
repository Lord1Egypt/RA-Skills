# 搜索查询构造模板

## 构造原则

1. 提取核心实体（公司名、人名、产品名、项目名）
2. 保留关键数值和日期
3. 去掉修饰词、上下文依赖词、情感词
4. 查询长度控制在 5-15 个词/词组
5. 中文查询优先；如果无结果，尝试英文查询

## 按分类的查询模板

### funding（融资）

```
输入: "{公司名}于{日期}完成{轮次}融资{金额}"
主查询: "{公司名} {轮次}融资 {金额}"
备选查询: "{公司名} 融资 {年份}"
```

### pricing（定价）

```
输入: "{产品名}定价{价格}"
主查询: "{产品名} 价格 定价 {价格}"
备选查询: "{产品名} 官方定价"
```

### date（日期）

```
输入: "{事件}于{日期}{动作}"
主查询: "{事件} {日期} {动作关键词}"
备选查询: "{主体} {动作关键词} 时间"
```

### event（重大事件）

```
输入: "{主体}{动作}{对象}"
主查询: "{主体} {动作关键词} {对象}"
备选查询: "{主体} {对象} 最新动态"
```

### statistic（统计数据）

```
输入: "{主体}{指标}突破/达到/超过{数值}"
主查询: "{主体} {指标} {数值}"
备选查询: "{主体} {指标} 最新数据"
```

### personnel（人事变动）

```
输入: "{人物}{职务变动}"
主查询: "{人物} {新职务} 任命"
备选查询: "{公司名} {职务} 人事变动"
```

### product（产品发布）

```
输入: "{产品名}于{日期}发布/上线"
主查询: "{产品名} 发布 时间"
备选查询: "{公司名} {产品名} 最新版本"
```

## 查询优化规则

| 场景 | 处理 |
|------|------|
| 公司有常用简称 | 用简称 + 全称各搜一次 |
| 日期跨度大（"2024-2025年"） | 缩窄到具体年份 |
| 模糊数字（"数亿"） | 去掉模糊量词，只保留实体+事件关键词 |
| 搜索无结果 | 换用英文关键词重试 |
| 第二次搜索仍无结果 | 标记为 unverified |

## 重试策略（步骤 3e 使用）

当前批次中 keywords-generated 的事实搜索失败时，按以下优先级换策略：

| 优先级 | 策略 | 适用场景 |
|--------|------|----------|
| 1 | 换英文查询 | 中文查询无结果时，将关键词翻译为英文 |
| 2 | 去掉数值只搜实体+事件 | 模糊数字（"数亿"）导致搜索结果偏差时 |
| 3 | 扩大时间范围 | 精确日期无结果时，扩大到年份或季度 |
| 4 | 去掉公司后缀/产品全称 | 简称和全称不一致时，用最短名称 |

重试查询构造示例：
```
原始查询: "XX科技 A轮融资 2025年3月 5亿"
策略1:   "XX Tech Series A funding 2025 amount"
策略2:   "XX科技 融资 2025"
策略3:   "XX科技 A轮融资"
```
