# 参考角色映射 (Reference Role Map)

> 每个参考素材分配一个主角色，明确说明不转移什么。
> 参考 `model-mechanics.md` 机制 6：参考胜过文本。

---

## 核心原则

**参考素材是密集条件**：图像指定的外观信息比一段话更多。但参考也会"出血"——运动捐赠者想带外观，风格参考可能改变主体。

**必须明确**：
1. 这个参考的**主角色**是什么
2. 这个参考**不能转移**什么

---

## 8 种参考角色

### 1. 身份 (Identity)

**用途**：角色/产品的外观锁定

**转移内容**：
- 面部特征
- 体型比例
- 服装/道具外观

**不转移内容**（必须声明）：
- "不要转移姿势"
- "不要转移背景"
- "不要转移光照"

**示例**：
```
[Image1] 作为角色参考，保留面部特征和服装外观，
不要转移姿势和背景。
```

| 素材 | 主角色 | 说明 |
|------|--------|------|

## 常见错误

### ❌ 错误 1：一个参考要求多个角色

```
❌ [Image1] 作为角色和场景参考
```

**问题**：参考会冲突——角色和场景的要求互相干扰。

**修复**：拆分为两个参考，或选择最重要的角色。

### ❌ 错误 2：没有声明不转移内容

```
❌ [Image1] 作为角色参考
```

**问题**：模型会转移所有东西——包括你不想转移的姿势、背景、光照。

**修复**：明确声明"不要转移什么"。

### ❌ 错误 3：用参考重新描述参考已展示的内容

```
❌ [Image1] 是黑发女孩，穿红色连衣裙，她向前走...
```

**问题**：机制 6——文本重新描述参考创建冲突指令。

**修复**：只提示参考**不能携带**的东西（动作、镜头、声音、约束）。

```
✅ [Image1] 作为角色参考，保留外观，她向前走，镜头缓慢推进...
```

```
[Image1] 作为角色参考，保留面部特征和服装，不要转移姿势和背景。

[Video1] 作为动作参考，保留动作节奏和肢体运动，不要转移角色外观和场景。

角色执行参考动作，镜头按参考运镜运动。
保持角色外观不变，场景为现代城市街道。
```

### 示例 3：首尾帧过渡

```
[Image1] 作为首帧，保留所有可见内容。
[Image2] 作为尾帧，保留所有可见内容。

从首帧过渡到尾帧，角色缓慢转身，镜头从特写拉远到中景。
保持角色外观不变，光照从冷色调渐变为暖色调。
```

## 核心洞察

**参考不是万能药**：每个参考都有角色，必须明确角色和不转移规则。

**参考胜过文本**（机制 6）：所以不要重新描述参考已展示的内容。

**参考会出血**：所以必须声明不转移什么，否则模型会转移所有东西。
