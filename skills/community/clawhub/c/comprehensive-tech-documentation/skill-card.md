## Description: <br>
Creates layered, comprehensive technical documentation for projects, skills, or systems, producing 3-4 documents for different audiences and depths. <br>

This skill is ready for commercial/non-commercial use. <br>

## Publisher: <br>
[liuxxxu](https://clawhub.ai/user/liuxxxu) <br>

### License/Terms of Use: <br>
MIT <br>


## Use Case: <br>
Developers, technical writers, architects, and agent users use this skill to turn a project, skill, or system into layered Markdown documentation with deep technical explanations, architecture diagrams, quick references, and a navigation index. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: The skill reads project files to generate documentation, so sensitive files or private data could be summarized if they are included in scope. <br>
Mitigation: Choose the project or folder explicitly and exclude secrets, private notes, customer data, and environment files before using the skill. <br>
Risk: Generated documentation can contain incorrect, outdated, or misleading technical guidance. <br>
Mitigation: Review the generated Markdown, verify file paths, commands, and Mermaid diagrams, and scan or test the documentation before committing or sharing it. <br>
